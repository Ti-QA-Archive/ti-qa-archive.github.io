  • A couple of things:

    Don't include the .js extension. It's appended automatically. Your code is looking for a file called startUp.js.js as currently written.

    Don't include the slash before the file's name. require('startUp') not require('/startUp') Including the slash will cause the file to not be found on some platforms.

    I also recommend you don't open the window in your module. Instead, return a window object and open that in the file that loads the module. That's a better coding pattern, more reusable.

    // StartUp.js
var StartUp = function() {
  // all your current stuff, then
  return MainWin;
};
module.exports = StartUp;

// in index.js
var StartUp = require('StartUp');
var mainwin = new StartUp();
mainwin.open();

    • You could try the MobileWeb build – it will build your code to run in a browser though there are some caveats:

      • The resulting pages have to be run from a web server (you can't generate the HTML pages and load them from your local hard drive).
      • Realistically, you'll need to do the build inside Studio to MobileWeb (as opposed to using the command-line), as that will launch a local web server, load your web browser pointing to that local server instance.
      • Some of the APIs are not supported on MobileWeb, such as the Database, any platform-specific UI controls (something that's supported on iOS or Android only), etc.
      • Networking calls must be written differently because you must account for cross-origin security issues that native apps don't need to worry about

      Ultimately, since Titanium is about building native mobile apps for iOS, Android, and BlackBerry, having a system with fully functioning emulators is really where you want to be. Or, get a device and do all your builds to it. If you're looking for a general web framework, there are other systems better suited to your task.
