Sharing Projects Between Two Different Computers

Hi folks,

So this is more for some advice than anything.

My current developer setup (iOS and Android) is running on my iMac, but i'm frequently out and about or away from my desk for long periods and I have a few projects on my MacBook Air too.

I'm trying to find the best way of keeping the two in sync, so if i'm at home I can develop on the iMac and whilst I'm on the road I can carry on developing on my MacBook Air.

I did consider using Dropbox as a repository for the source code, but wasn't sure if that was the best solution.

Does anyone have any experience of running two Macs under the same dev profile and switching between projects?

Any suggestions would be greatly appreciated!

Simon

    Get yourself a real source repository. While Dropbox may work for a while, it can become tricky if there are ever any conflicts (trust me on this one). Also, you will benefit from many other features generally associated to using a source repository like version control, tagging, change requests, and many more.

    • You can take a private account on http://github.com (about 5$ a month)
    • Or you can also use http://bitbucket.org which also offers private repositories but completely free.

    Hope this helps.
