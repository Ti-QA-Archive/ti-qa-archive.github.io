How to stop Android textField from auto-focusing

Every textField I create focuses automatically when the window opens, forcing the soft keyboard to open. This does not happen on iPhone or iPad, and is the same on the Android emulator, HTC Incredible and HTC Thunderbolt. It has done it across multiple SDK versions (and still exists with 1.8.2).It is the same whether using textField or textArea. This is very annoying and not warranted in most situations I am dealing with.

I am able to temporarily stop it by setting textField.softKeyboardOnFocus = Titanium.UI.Android.SOFT_KEYBOARD_HIDE_ON_FOCUS when the field is created. That turns it off permanently, so after the window finishes loading, I can turn it back on. This is the only decent solution I have found. I have tried a bunch of variations on how the window elements are pieced together, but nothing changes this.

Typically, I will create a window (using url syntax), then in the new window create a scrollable view, and start attaching views, etc to that. The fact that this does not seem to be a widespread problem makes me think there is some setting somewhere that I am missing.

Has anybody solved this without doing kludgy workarounds? Surely this is not by design.

    Hello Burgiss,

    Hide your textfield at the Time of creation And show it in open event of window by setting Timeout function.

    Code is here.

    //hide
    textFieldname.hide();

    //show
win1.addEventListener('open', function(e) {

    setTimeout(function(e){
        textFieldname.show();
    },1000) ;
});

    It's working fine for me.

    I hope it will work For You.

    • Here's what worked for me, tested on several Android versions, SDK 3.1.1.GA:
      At TextField creation (I use Alloy), I set textField.visible to false. Upon Window focus I set textField.visible to true. The result is the text field is not initially focused, and works as expected from then on. No need for timeouts, no need to muck with softKeyboardOnFocus which always seemed to do weird things, no matter what I tried.

  • Keep android from auto-showing keyboard,

    textArea.addEventListener('focus', function(e){
    textArea.blur();
});

    or if you only want to stop the first occurrence,

    var first = true;
textArea.addEventListener('focus', function f(e){
    if(first){
        first = false;
        textArea.blur();
    }else{
        textArea.removeEventListener('focus', f);
    }
});

            txtFieldSearch.softKeyboardOnFocus = Ti.UI.Android.SOFT_KEYBOARD_HIDE_ON_FOCUS;
        txtFieldSearch.addEventListener('click',function(e)
        {
            txtFieldSearch.setSoftKeyboardOnFocus(Ti.UI.Android.SOFT_KEYBOARD_SHOW_ON_FOCUS);

        });

  • This worked flawlessly for me with no timeout waiting.

    var textAccountID = Titanium.UI.createTextField({
    width:Titanium.UI.FILL,
    value:'',
    keyboardType:Titanium.UI.KEYBOARD_NUMBER_PAD,
    bubbleParent: false,
    softKeyboardOnFocus: Titanium.UI.Android.SOFT_KEYBOARD_HIDE_ON_FOCUS
});

if (osname != 'android') {
    // This is for iOS and allows the user to click outside the textField to hide the keyboard.
    winLogin.addEventListener('click', function(e){
        textAccountID.blur();
        textPassword.blur();
    });
} else {
    // Be sure to place the following in the first text field of the view:
    // softKeyboardOnFocus: Titanium.UI.Android.SOFT_KEYBOARD_HIDE_ON_FOCUS    
    textAccountID.addEventListener('touchstart', function(e) {
        textAccountID.softKeyboardOnFocus = Titanium.UI.Android.SOFT_KEYBOARD_DEFAULT_ON_FOCUS;
        textAccountID.focus();
    });    
}

  • Actually, I am disabling focus during both window.open() AND during scrollingevents. Here is code snippet:

    var view = myScrollView();
win.close();

....

if ( _isAndroid ) {
    // keep stupid textfields from focusing
    // this is partially effective. The first visible field will still auto focus when scrolling.
    var stopFocus = function() {
        for (i=0;i<textField.length;i++) {
            textField[i].softKeyboardOnFocus = Titanium.UI.Android.SOFT_KEYBOARD_HIDE_ON_FOCUS;
            setTimeout( function() {
                for (i=0;i<textField.length;i++) {
                    textField[i].softKeyboardOnFocus = Titanium.UI.Android.SOFT_KEYBOARD_SHOW_ON_FOCUS;
                }
            }, 1500, [textField] );
        }
        win.open();
    }
    stopFocus();
    view.addEventListener('scroll', function() {
        stopFocus();    
    });
}

    What is not now working, is that during a scroll the first visible field focuses and the keyboard pops up automagically. This is very consistent unless scrolling very slooooowly.
