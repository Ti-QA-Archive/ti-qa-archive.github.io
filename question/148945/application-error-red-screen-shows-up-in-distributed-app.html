Application Error (red screen) shows up in distributed app

The headline says it all, an app we put in the App Store today sometimes show the red Application Error screen to some users, an issue we haven't been able to reproduce, probably related to trying to access data that hasn't been downloaded yet.

This is a soft fail, because the app will try to download the data again.

But my problem is that the red Application Error screen show, even though the error message says "Error messages will only appear during development. When your app is packaged for final distribution, no error screen will appear"

So why does it show in our distributed app? Have I done something wrong in the distribution?

  • I found a workaround to change Xcode source
    in main.m
    change
    TI_APPLICATION_DEPLOYTYPE = @"development";
    to
    TI_APPLICATION_DEPLOYTYPE = @"production";

  • The TI_APPLICATION_DEPLOYTIPE changes to "production" when you distribute it through Titanium Studio. I don't know why, but it doesn't change automatically with xcode. It used to (as far as I remember), but you would get those errors even on the appstore if you don't package it with Studio.

  • go to alloy.js file in plugin->ti.alloy->hooks folder

    in that you will find the following code

    deploytype: deployType || cli.argv['deploy-type'] || 'development',

    this will be inside run function

    replace it with the following

    deploytype: 'production'

    and bamm you are good to go (P.S you will need to keep switching between the lines when you want to test your build on device and this file refreshes on each build so just keep an eye out).

    Happy coding
