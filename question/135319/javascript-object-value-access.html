  • i think how to access the value is not the problem ?!

    am i right you want to do something like that:

    var test = {
   one : {
      top : 20
   }
   two : {
      top : get same value as in one.top
   }
}

    if thats the case you have to either declare a 'value'-variable and use this instead of the value itself : 

    var topValue = 20;
var test = {
   one : {
      top : topValue 
   }
   two : {
      top : topValue 
   }
}

    .. or declare/set the variable test 'partially':

    var test = {
   one : {top : 20}
};
test['two'] = { top: test.one.top};

    be careful when using object-types .. to a certain degree/level the reference idea applies but from a certain 'depth' of the object it doesn't and also 'deeper' properties become immutable .. exactly when and where and why i coludn't figure out over the last year :) .. just be aware of that and clone your objects before using/changing them if you want to use the unchanged source-object later ..

    var test = {
   one : {top : 20}
};
test['two'] = JSON.parse(JSON.stringify(test.one));
test.two['left'] = 10;

    .. the above example makes sure u get all the stuff from 'one' and on setting 'left' to 'two' 'one' stays unchanged… but don't get me wrong :) if you remove the cloning don't rely on 'one' getting the 'left'-value too .. sometimes it does sometimes it dont't
