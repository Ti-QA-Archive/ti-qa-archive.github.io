Titanium project edit tab path truncated / link not working in ubuntu

Using the titanium dev application, selecting the project from the list on the left, clicking the "edit" tab, and clicking the filesystem path where the project is located on disk, the pointer changes to a pointed hand, as though it is a link. Presumably it is supposed to open a file manager window showing the target, but this does not happen when running on ubuntu.

Furthermore, the path is displayed using an unnecessarily large font and is truncated when greater than 50 characters.

All of the above makes it difficult to determine where on the disk projects are running from.

I'd appreciate appcelerator's views on this, to determine which of these constitute a feature enhancement and which constitute a bug.

Thanks in advance
