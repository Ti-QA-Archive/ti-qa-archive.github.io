click event on Map.View does not fire if title property isn't set

I'm using SDK 3.1 and I don't want to display the full annotation when the user clicks on it, I just want to show the pin on the map and detect the click to show an overlay view. In order to do that, I create the annotation without the title property. It is shown on the map without problems and the event listener is added, but when it is clicked, no click event is fired. If I set the title property, the click event is fired normally but the full annotation is also shown, which is an undesirable behavior in my case. An example situation is shown below:

var map = Ti.Map.createView({});

var annotation = Ti.Map.createAnnotation({
        latitude: data.latitude,
        longitude: data.longitude,
        title: "Any title", //if not set, no click event is fired when the pin is clicked
})

map.addAnnotation(annotation);

map.addEventListener('click', function(e){
    if(e.clicksource == 'pin')
    {
        Ti.API.info('pin clicked');
    }
})

Anyone knows how to solve this problem? Is it a bug?
