"Length Required" error in Google Calendar Event POST request using HTTPClient & JSON

I am able to get the token and calendar entries from Google in my android application. Now, I am trying to post an event to Google calendar using the following code:

var xhr = Titanium.Network.createHTTPClient({
    onload : function(e) {
        alert(this.responseText);
    },
    onerror : function(e) {
        Titanium.UI.createAlertDialog({title:'Synchornization Error', 
            message:'Error occurred while posting: '+e.error}).show(); 
    },
    timeout : 5000 
});

var url = "https://www.google.com/calendar/feeds/default/private/full/?alt=jsonc";
xhr.open("POST",url);
xhr.setRequestHeader('Authorization','GoogleLogin auth=' + token);
//xhr.setRequestHeader("GData-Version", 2);
xhr.setRequestHeader('Content-type', 'application/json; charset=utf-8'); 
//xhr.setRequestHeader('Content-type', 'application/x-www-form-urlencoded'); 
//xhr.setRequestHeader('Content-Length', _entry.length); // throws 'header already present'
xhr.send({body:jsonEntry});
//xhr.send(jsonEntry);

jsonEntry is a valid json calendae event with title, start and end time as below:

{ "data": {"title": "AAA","when": [{"start": "2010-04-17T15:00:00.000Z","end": "2010-04-17T17:00:00.000Z"}]}}

However, I get "Length Required" error with the above code. If Content-Length header is specified explicitly as shown in the line that is commented, it throws "Content-Length Header already present" error. I have tried specifying GData-version and also tried removing the structure with Body property, but to no avail. Can someone identify where I am going wrong? Would appreciate any ideas. Thanks!
