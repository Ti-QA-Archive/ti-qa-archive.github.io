  • This is not much different than creating a website in Photoshop, slicing it up, and using the individual images in your html markup. The only difference is that instead of using the images in your markup, you'll be using them in your ui components.

  • It's not too bad to build a rich UI for iphone, where you effectively only have two screen sizes to worry about. It gets a lot more complicated if you try to make it work on all android devices. Factor in landscape vs portrait, and your complexity just went up even more.

    For an example of a rich UI built for all devices, see GrillTime. It's a long tutorial, so don't judge it by the screenshots in part 1. Finished product screenshot

    The tutorial takes you step-by-step through processing the images for different screen densities, and dynamically calculating the size of the views to optimally fit the display.
