DB Access Best Practice

I hope this is a simple enough question:

When an application is going to access the database many times on different windows, what's the best way to continually create (and then destroy) that access?

I'm used to .NET where the connection to the DB is defined in the web.config file, and then a connection variable can be defined in a global class which gets inherited in each page. So in Titanium Studio, is it possible to only use this line once:

var db = Titanium.Database.install('/data/MyDB.sqlite', 'MyDB');

And then have access to db globally? Opening and closing it each time?

Thanks,

Casey
