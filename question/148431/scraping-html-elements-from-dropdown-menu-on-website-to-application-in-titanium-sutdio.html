    Hi

    Keeping in mind the advice of both Nick and Christian - I agree with both of them, this is to be avoided if at all possible - there is a way to do it and reasonably easily.

    Using YQL

    This would be the query to retrieve the actual elements of the drop down you linked to above.

    select * from html where url="https://portaal.khleuven.be/index.php?page=147" and xpath='//select[@id="afdeling"]'

    Personally I prefer to call the YQL using httpClient as you are able to set timeouts and error handlers.

    This is the url property

    http://query.yahooapis.com/v1/public/yql?q=select%20*%20from%20html%20where%20url%3D%22https%3A%2F%2Fportaal.khleuven.be%2Findex.php%3Fpage%3D147%22%20and%20xpath%3D'%2F%2Fselect%5B%40id%3D%22afdeling%22%5D'&format=json&callback=

    Once you have retrieved the data using httpClient the following snippet would be placed inside the onload handler.

    var json = JSON.stringify(this.responseText);
var options = json.query.results.select.option;

var intOption = 0, intOptions = options.length, val = 0, text = '';
for (intOption = 0; intOption < intOptions; intOption = intOption + 1) {
    val = options[0].value;
    text = options[0].content;
    Ti.API.info(text + ': ' + val);
}

    This will list all the options in your app.

    If you want to test the query above copy it to a browser to see the results. If you want to play with it to see what you can do to tweak it then using this link you have access to the YQL console.

    • YQL Console

    Swap to JSON, remove cbunc form the text box and uncheck diagnostics. Press test when you are ready

    If it was me, I would create a web service of your own and your app can talk to that, you populate the data from the values each time they change. Worth checking the data load that will occur on the site portaal.khleuven.be if your app is successful.

    • Alright, I got it fixed by replacing the JSON.stringify() to JSON.parse().

      The final code before locking:

      var url = "http://query.yahooapis.com/v1/public/yql?q=select%20*%20from%20html%20where%20url%3D%22https%3A%2F%2Fportaal.khleuven.be%2Findex.php%3Fpage%3D147%22%20and%20xpath%3D'%2F%2Fselect%5B%40id%3D%22afdeling%22%5D'&format=json&callback=";
var client = Ti.Network.createHTTPClient({
    onload : function(e) {
        var json = JSON.parse(this.responseText);
        var options = json.query.results.select.option;

        var data = [];
        for (var i = 0; i < options.length; i++) {
            data[i] = Ti.UI.createPickerRow({
                title : options[i].content
            });
        };
        pickerDepartment.add(data);
    },
    onerror : function(e) {
        alert('Error: ' + e.error);
    },
    timeout : 5000
});
client.open("GET", url);
client.send();

  • Trying to scrape a website for content from within your app is always going to be a fragile solution.;

    1. Html is not alway well-formed (unlike say xml)

    2. The website could change their layout and therefore break your app with no notice.

    Much better to move the scraping to some server-side activity you can control, and then send that in a well-formed manner to your app by providing some form of REST/SOAP web service.
