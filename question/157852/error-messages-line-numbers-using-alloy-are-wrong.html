Error messages line numbers using Alloy are wrong

This question may have been answered somewhere in the forum and Q&A but I can't find it.

Simply why are error message line numbers, where the error has occurred, so off from actual code?

For example: if I receive an error 'null' is not an object (evaluating 'picker.setDate') at index.js (line 63) the error is actually in a function starting at line 77.

I'm just curious, is this what developers have to get used to and is this the way Alloy was designed to operate?

Is there a way to correct this?

I am using:
TS Build 3.1.3.201309132423
Alloy 3.1.0201309122332-773-7F7BJ7gBFFJJF

    Alloy is a pre-compiler so the actual line numbers refer to the code that is generated and can be found in the Resources directory of your project.
