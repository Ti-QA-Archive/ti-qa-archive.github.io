  • Hi

    It is common to have to do some house-keeping when updating to a newer SDK, however there are two main issues here;

    • In each of your cases you have moved from one major version to another major version
    • The most recent jump should have had a lot less changes

    Between SDK 1.x and 3.x there have been many changes, including many names space changes, some stuff has gone and some moved. Also advice from SDK 1.x days is not as common practice now.

    You should actively consider jumping up each minor SDK version as soon as is sensible, review the change logs for each version and tweak as required, this will give you very little to do each time but give you huge performance and reliability benefits.

    As Titanium has been an evolving product and the platforms have had increasing feature sets - everything will change, just follow along quickly and it is much easier.

    There is no reason you should be on any version other than Ti SDK 3.1.2 right now, unless you are targeting a device that is very old - and one that the provider themselves no longer supports.
