  • You will need a Mac to compile the iOS version. This is because the compilation process is ultimately done by Xcode, which Apple publishes for OS X only. You could use a service like MacInCloud or one of the host-a-mac-mini providers out there if you really don't want to buy one.

    • Commenting on my own answer because I can't edit it:

      While you could probably do everything on a Windows/Linux box, then transfer the files to a Mac and compile, you shouldn't. You should develop the Android & iOS versions in parallel, testing on both platforms as you go. If you wait till the end, you're just asking for troubles, incompatibilities, UI/UX problems, etc.

      Titanium is not a "write-once, run everywhere" tool. The native platforms are just too different to abstract completely (without making a completely minimalistic app dev tool). That means necessary API differences between the target platforms. Besides, you should be developing UI/UX experiences that match the respective platforms' UI guidelines and user expectations. Which means different screens, layouts, and flows across platforms.

      The differences are likely to be small, code-wise. Some apps can use the exact same code, some have small tweaks here and there to adjust sizes or placements, some apps have significant differences if you're getting into many platform-specific features & functions.

      I'm not sure if this is a hobby/side-thing for you, or you're going to be doing app dev on an on-going basis. If it's the latter, you really should make the investment in a Mac. It will make things so much easier in the long run.
