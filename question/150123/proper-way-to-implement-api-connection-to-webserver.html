Proper way to implement API connection to Webserver...

My app currently opens an "createHTTPClient" connection to my server. In order to try to secure the page from been accessed directly, I have my app send two 64 character strings via form post to the php page on the server. Then, on the server side, I right away check for both of those strings, and do an md5 of their combination. If the md5 of them matches a set string on the server, then the query to the database is made, json string generated and sent back to the app. If the md5 doesn't match, then redirect to another page.

Figured this would kind of make it pretty secure because you would need access to the source code of the app to get both keys that generate the exact md5 match on the server and on the server, the md5 is generated from the combination of both form post values.

The page will also be on the SSL side of my server.

I would like to somehow, on the server side, be able to detect via PHP if the page is accessed from an app in iOS. Not sure if that is possible at all, but would make it it that much safer to prevent access to the page directly.

Not sure if I am going overboard, but sometimes safe is better than sorry. Any ideas on how to beef this up?

  • You can check the user agent, should look like this:

    Appcelerator Titanium/3.0.2.GA (iPad/5.1; iPhone OS; de_DE;)

    But of course this can be faked too. With a network sniffer or tool like Fiddler almost every request can be detected and simulated via curl ;)
