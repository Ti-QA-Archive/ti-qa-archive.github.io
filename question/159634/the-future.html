    Hi

    Ti SDK 3.2 has loads of things coming, but most of these can be considered tweaks and fixes - but they amount to big changes/benefits.

    A few - no particular order

    • Managing Alloy themes beyond tss
    • iOS7 64 bit
    • Studio visual updates and refined interface
    • Direct to device compling for iOS - no need for iTunes in the middle
    • GenyMotion Android Emulator support - this is big
    • Lots of command line tweaks
    • New properties on common elements

    Ti SDK 4.0, this will be big. Often referred to as Ti.Next (just show it is beyond now), it includes hyperloop, this is a completely different SDK design - basically a whole new re-write of the original.

    Some of the benefits;

    • Much faster; compiling and app run times
    • Much smaller; the size of the supporting files used in the app files
    • Do everything exactly as you do today - unless you want to go further
    • Down down and dirty with native elements. This means you can right native code in JS - in the same code as your existing app. This means if you need something native - just add it, no need to create a module. You will need to learn the native syntax - but as it uses JS this will be fairly easy if you are comfortable in Titanium and JS and have a want to go further.
    • Native elements mean you can add that Android switch we do not have or use iOS iBeacons right inside your app.
    • Much easier to code for multiple platforms that you do not already code for
    • Windows mobile is expected
    • MUCH MUCH easier for Appcelerator to build out new functionality as they will be dealing with one core code rather than a duplicate for each additional platform supported.

    Read more about Hyperloop

    There is more, there is lots more. These are some big changes that we can expect for 4.0, but this is new ground and needs time to work correctly. This is going to be next year and likely somewhere around the middle - no dates I am aware of have been set in stone.

    This is a simple overview, I do not want to go further than anything you can access publicly.

    Just know that great gets better.
