Android Latest Nightly Builds

I am using the latest version of Ti Studio on Mac running Lion.

Firstly I will say that I have developed an app which runs on IOS 5 and successfully runs with the SDKS I am about to mention.

The issue is that I am now moving it over to Android and I have a few issues which I hope somebody else has come across or the guys at appcelerator can give me some guidance on. I am not going to post code unless requested.

Android SDK 1.8.01 and 1.7.6 latest nightly builds both using V8 and Rhino.

I am loading some files in the app.js file using Ti.Include defining functions within the application namespace, these load. I am then when required loading other other files. I use a global load routine and the commonJS require method. I load each file into its own sub namespace of the app namespace and export the functions, there are no globally accessible variable declared in these files.

In the above SDK's the required file functions cannot access the declared namespaces in the included files. In the 1.8.0 SDK everything runs as it should in either V8 or Rhino.

Next few issues…. This applies to 1.7.6 1.8.0 and 1.8.0.1
have not expected 1.7.5 to work but have the loading file issues so cannot check much after that.

Anywhere I do a show() or hide() on a component I have had to change it to visible = true or false. show() and hide() do not give an error they just do not work.

text areas. I have quite a few text areas declared throughout the code, on the input ones, they work as they should, but on the display only ones no (editable is false) actual text is displayed and although I have set the borderWidth to 0 they still have a border, also the backgroundcolor does not display correctly it is always yellow no matter what I set it to. Labels work as expected. Not setting the backgroundcolor makes it transparent.

I think this may have something to do with the text area focus but haven't tested that yet.

GPS Location.

Although I cannot fully confirm this yet as more testing and trials need to be done, it appears that the Geolocation services are not updating. I have had the same code without the file load issue running in 1.7.5 and geolocation works perfectly.

Animations.

It also appears that animations do not run. I have a couple of screens loading from the right and they do not appear. Again this maybe down to something else but no animations are working..

Data httpRequests

On one of my API calls it always fails. I have common code for all data calls which works for everything else. It appears that for some reason it returns the data as responseData instead of responseText.

Well I think that is about it.. Sorry for the length of the post but wanted to put all issues in one place. I hope somebody else has a solution or now that appcelerator are aware they can look at the nightly SDKS, and tell me where I am going wrong.

Thanks

T.
