  • First, test a default app. Create a new blank project, then run it. Also, check on the Dashboard page under Configure Native SDKs. Check for a green check mark next to Android SDK. If not, it's not configured properly. That should get you started.

    • On your test app, open tiapp.xml. Under deployment targets, is Android checkbox checked? Does the xml have a similar setup

          <deployment-targets>
        <target device="android">true</target>
        <target device="blackberry">false</target>
        <target device="ipad">true</target>
        <target device="iphone">true</target>
        <target device="mobileweb">true</target>
        <target device="tizen">false</target>
    </deployment-targets>

      Also, open a terminal window, type android. The android sdk manager should launch. Select Tools > Manage AVDs. Anything listed there that does not have a check mark next to it is corrupted. Even so, I would delete the one marked "emulator-5554" just to see if that's the problem. Then start a process of elimination. IIRC, Studio creates its own avd's. So, I would just delete them all and start over. But that's just me. Some folks have more patience than I.
