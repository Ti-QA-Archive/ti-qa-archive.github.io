Database update method, looking for expert opinion

I end up using this method for controlling database updates, I wonder if I miss something that might be a problem while switching to production?

I decided to use this method and not using dynamic name, because i've noticed that the simulator keep all databases and I don't want the user to keep unneeded data in the cache folder.

what do you think?

getDatabase: function(){

        var dbVersion = Ti.App.Properties.getDouble('dbVersion'),
            db;

        if(dbVersion !== null && dbVersion !== app.config.dbVersion){
            Ti.API.log('New database available, removing current database');
            db = Ti.Database.open('books');
            db.close();
            db.remove();
        }

        Ti.App.Properties.setDouble('dbVersion', app.config.dbVersion);
        return Ti.Database.install('books.sqlite','books');
    }
