Deployment App to iOS AppStore

First I use: Titanium Studio 2.1.2.201208301612
Titanium SDK 2.1.4
XCode 4.5.1
Application Loader 2.7.
Mac OS X 10.7.5

My App is ready for app store from Apple. Works fine on my devices and i love it ;-).

Now, I used Publish in Titanium Studio. That works fine and I found a archive in Xcode. Now, I validate this an I become the message: 

iPad: Appicon.png: icon dimensions (57 x: 57) don't meet the size requirements. The icon file must be 72x72 pixels, in .png format.

Ok, I check this and I see that XCode compromise my png files. First question is:

  1. What ist the way for set the property "Compress PNG Files" to "No" in Titanium. In info.plist is this not available.

now, I try and think about ist. Build a new archive in XCode and copy my origional icons into my archive. That works fine. After validate comes the next problem. 

iPhone/iPod Touch: application executable is missing a required architecture. At least one of the following achitecture(s) must present: armv6.

Now, I test and try. Titanium say in console that linked iOS 6.0 with minimum iOS version 4.3.

For every case i add in tiapp.xml:

<ios>
        <min-ios-ver>4.3</min-ios-ver>
</ios>

No way for a solution. After this, is set directly in XCode I set the architecture for armv7 and armv7s. That will not work. I try to set the deployment target from iOS 4.0 to 4.3 and many other try's. That ist dirty.

Now, the second question:

  1. What is the solution for deploy my App in the app Store from Apple? Everything ist ok. Apple only waits vor a Upload. Now, I'am tired an lost. Thank you for a tip or a idea. Thank you.
