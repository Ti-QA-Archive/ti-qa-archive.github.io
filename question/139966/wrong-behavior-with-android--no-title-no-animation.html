Wrong behavior with Android : no title, no animation?

I found some strange behavior on android development by Ti, I create a new project with default master/detail template, and modify it like this:
1, set '<navbar-hidden>true</navbar-hidden>' in TiApp.xml, to make splash screen no title
2, in ApplicationWindow.js, delete 'navBarHidden:false' parameter in both two window creation

If I didn't do the second process, when the application runs, a window with title (I hate title…) will appear, and when you click any row of list, a new window will show up with slid transition animation, then click 'back', the new window will be hidden with transition too.

But when I did the second process, the new window will show up WITHOUT any transition animation, and what's more worse, when you click 'back', the whole application will exit!

Is there any way to hide window's title and remain the normal behavior of application?

  • Solved.

    change followed text in AndroidManifest.xml

        <application android:icon="@drawable/appicon"
        android:label="xxx" android:name="Wxcs_klmyApplication" 
        android:debuggable="false">

    to 

        <application android:icon="@drawable/appicon"
        android:label="xxx" android:name="Wxcs_klmyApplication" android:theme="@android:style/Theme.Light.NoTitleBar"
        android:debuggable="false">

    and make sure ALL window created with title and navBarHidden:false, then everything will be ok
