Titanium Studio stops responding every few minutes

Working with Titanium Studio is frustrating me. I'm wasting a lot of time watching the Mac rainbow wheel but I'm still excited about this path towards mobile development. I would like some help or advice on getting Titanium Studio to be more responsive and to stop hanging while I'm editing my project files.

I can deal with the 1-2 mins build time while deploying to the iPhone Simulator and the 3-5 mins build time while deploying to the Android Emulator but even while editing files in my project I get interrupted by Titanium Studio doing something that must be really important and really urgent because it makes me stop and wait for a few seconds to a few minutes. It's really frustrating.

I'm using a Macbook Pro running OSX 10.6.8 with 8GB of RAM to develop with Titanium Studio, build: 1.0.7.201112281340, Build: jenkins-titanium-rcp-master-130 (origin/master), Date: 28 December 2011, 13:46:07

Incidentally, I just saw a boat-load of stuff in the Installation Details - Configuration screen, as if my Titanium Studio is configured to load 100 different things I may not need in this context like support for Ruby and Python and such. Or is there some way to increase the memory allocated to the VM that Aptana runs in, like Eclipse allows?

If there's some way to tune this installation, I would love to hear about it.

thanks for reading.

  • Accepted Answer

    Yes, you can increase VM memory. Go to TiStudio installation directory and edit "TitaniumStudio.ini" file:

    • –launcher.XXMaxPermSize
      256m
    • -Xms40m
    • -Xmx1024m

  • I spent a while searching for tips around the answer above and came up with the following gist: https://gist.github.com/1066477 (contents included below as well)

    Use this Titanium Studio ini file in place of your existing.

On OSX right click on the TitaniumStudio.app and choose 'Show Package Contents", then navigate to "Contents/MacOS/TitaniumStudio.ini" and edit the file.


<ini>
-startup
../../../plugins/org.eclipse.equinox.launcher_1.1.1.R36x_v20101122_1400.jar
--launcher.XXMaxPermSize
256m
--launcher.defaultAction
openFile
-vmargs
-XX:PERMSIZE=64M
-XSS1M
-server
-XX:+DoEscapeAnalysis
-XX:+UseConcMarkSweepGC
-XX:+UseCompressedOops
-XstartOnFirstThread
-XX:MaxPermSize=256m
-Xms256m
-Xmx1024m
-Declipse.p2.unsignedPolicy=allow
-Xdock:icon=../Resources/titanium.icns
-XstartOnFirstThread
-Dorg.eclipse.swt.internal.carbon.smallFonts
</ini>
