    • The xml file is for describing a Titanium app. I assume that since you're here, the app you've got is a Titanium app right? Second, the kitchen sink is source code, not a compiled app. When you take source code and compile it into an app, the end result is an .apk for Android or an .ipa for iOS. You CANNOT take an .apk file and open it in Titanium Studio to show you the source code or to deploy. Titanium Studio is for the SOURCE of the app, not the built app iteslf. An .apk is meant to be run in an emulator or device, which has nothing to do with Studio unless you have the original code that the app was built with. This is the same for iOS.

      If you want to install an .apk into the Android emulator, you first need to make sure that you have all the Android development tools and then you just need to run:

      adb install [apk file name]

      For iOS, you need to make sure that the app was provisioned to run on your device, then install your .ipa into iTunes. Once its in iTunes, you can sync your phone with iTunes to put it on your device. You CANNOT run an .ipa in the Simulator. The only way to run an app in the Simulator is to compile the source code yourself to launch into the sim.

      From the sound of it you do not have the source code for your app. Without that, Titanium is not going to do you any good. Titanium Studio is used for actually creating the app, not finding a way to run an existing app.
