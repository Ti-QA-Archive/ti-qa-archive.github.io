Using a retina sized image as backgroundImage for view/window doesn't work

I am building an iPad app and want to support the new iPad's Retina display, but I can't seem to get it to work. I'm running on SDK v2.0.1.GA2 and OSX Lion with Xcode 4.3.2 (just incase it matters).

I have a folder called uiassets in my Resources folder, and it contains 2 files: tabgroup_store_bg_ipad.png and tabgroup_store_bg@2.png

I tried the following VERY simple code in app.js:

var win=Ti.UI.createWindow({backgroundImage:'uiassets/tabgroup_store_bg_ipad.png'});
win.open();

When I run this in the iOS Simulator (setup for iPad Retina), it still uses the low-res (standard) version of the image. I thought this was handled by Titaniun - is this a bug, or am I doing something wrong ?

I also tried:

var win=Ti.UI.createWindow();
win.add(Ti.UI.createView({backgroundImage:'uiassets/tabgroup_store_bg_ipad.png'});
win.open();

and

var win=Ti.UI.createWindow();
win.add(Ti.UI.createImageView({image:'uiassets/tabgroup_store_bg_ipad.png'});
win.open();

with the same results - i.e. low-res image is used….
