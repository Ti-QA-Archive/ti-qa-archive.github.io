iOS tabGroup changes colour

Hi all,
I've found an issue I've not come across before, and I'm at a loss as to how to deal with it.

I have a simple app that includes tabs and a map. Everything works fine until you go to the map, at which point the tabs change colour. Once the tabs have changed colour, selecting a different tab produces yet a different colour.

I've included some before and after screenshots

Before: http://imgur.com/xE7fnUi

After: http://imgur.com/PCfxa1m

Selecting a different tab after: http://imgur.com/NZM94ef

I believe the issue stems from the map somehow, as it doesn't just happen when you go to the map tab. In the detail pages I have a button that shows the item on a map, which also causes the colour change.

Another interesting factor is that this only happens on a device. I have not been able to replicate the issue in the simulator.

I have looked through my code and there is nothing in there that even comes close to changing the colour of the tabs. I've performed project wide searches for background colour, tints, tabs and tabgroup and come up empty. I've attached focus events to the tabs and windows and logged out the properties, comparing them before and after.

The project is using SDK 3.2.3 and Alloy 1.0

If anyone has any ideas I would be grateful to hear them

  • It turns out it was a bug, it was just hard to pin down the existing tickets. For reference:

    https://jira.appcelerator.org/browse/TIMOB-15493

    https://jira.appcelerator.org/browse/TC-3032

    I set translucent:false on the window and tabGroup and the issue has gone away.
