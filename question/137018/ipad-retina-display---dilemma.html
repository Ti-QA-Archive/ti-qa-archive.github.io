iPad Retina display - dilemma

I have a beautiful set of images - which would look really cool on the iPad retina display…but of course the size of the app would be BIG when I use them! I don't think it is a big problem if the end user has iPad 3… but if end user has iPad 1 & 2 are those hight resolution images downloaded to their devices as well?

Is it possible to make a directory with images only for iPad 3 (@2x)?

The app works also on iPhone - I have 2 sets of images - low and high resolution.. already. Just for iPad I would like to show them BIGGER.

Is it better to make different app for iPad 3 only?

Any ideas would be appreciated. Thanks.

    The only way I see this is to detect what device the user has and download the images from a server based on this.
    If you don't want to download them, another solution would be to make 2 separate apps, one for ipad and one for iphone, so you get smaller sizes for each of them, but of course the ipad version will still be huge.

    I don't think you can limit an app to be downloaded only by the ipad3 model, as far as I know apple doesn't offer this kind of control.
