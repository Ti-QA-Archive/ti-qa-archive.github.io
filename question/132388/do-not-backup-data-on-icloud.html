do-not-backup data on iCloud

My app was recently rejected because I did not follow the guidelines for data storage. The reviewers said that "Data that can be recreated but must persist for proper functioning of your app - or because customers expect it to be available for offline use - should be marked with the "do not back up" attribute."

Is there a way to account for this? I am using a sqlite database that needs to be kept active for off-line use.

I'm using 1.8.1

    var db = Ti.Database.install(...);
db.file.remoteBackup = false;
