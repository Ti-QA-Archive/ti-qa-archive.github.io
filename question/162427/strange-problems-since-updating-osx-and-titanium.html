Strange problems since updating OSX and Titanium

Hi everyone. I've been having some strange problems since updating to Mountain Lion and the newest version of Titanium a few weeks ago. After figuring out a couple small iOS problems, I moved on to Android, and I'm stumped.

Previously, when using Snow Leopard and Titanium SDK 2.1.4, all my density specific images were working correctly. I've read many ways to do it, but the way it worked for me was to build the folder structure like this:

Android

    images

        res-ldpi

        res-mdpi

        res-hdpi

        res-xhdpi

In my code, I I didn't need to use 'dp'. Everything was working as it should, even though everything I had read said to use 'dp'. Here's an example of some image code:

var view1 = Ti.UI.createView({
    backgroundImage:'images/image1.jpg',
    width:174,
    height:130,
    left:10
});
scrollView.add(view1);

In my tiapp.xml file, I added the lines

  <android xmlns:android="http://schemas.android.com/apk/res/android">
        <manifest>
            <supports-screens android:anyDensity="true"/>
        </manifest>
    </android>

Now, I'm using SDK 3.2.0, and the screen density images aren't working. Actually, just the images within a scroll view aren't working. So I thought screw it, I'll just build it using SDK 2.1.4. Now here's where I'm really confused.

When the SDK is set to 3.2.0, beside 'Run' in the top left corner, the name of my device shows up. I can click run, and although the images aren't working properly, the app installs to the device.

When the SDK is set to anything other than 3.2.0, the name of my device disappears. And when I click 'Run', I get this strange error message:

[INFO] logfile = /Users/marc/Documents/Titanium Studio Workspace/SDK Test/build.log
[ERROR] Application Installer abnormal process termination. Process exit value was 1

Does anyone have any idea what's going on? Either to help me get SDK 2.1.4 working, or to get the image density working in SDK 3.2.0? Any help would be greatly appreciated.

Thank you.

  • Hi

    Seems you have a number of problems here - answers in no particular order;

    • Q: Why does the device name disappear when using SDKs lower than 3.2

    Because ONLY Ti SDK 3.2 supports compiling directly to devices. Anything earlier will not show the options because they were not available.

    • Q: Should you be using Ti SDK 2.1.4?

    NO - this is very old and you would have to have a very good reason to stick with it. For whatever minor pain you may have tweaking things for the latest Ti SDK 3.2 (at the time of writing) always stick with the latest stable version for each build (do not jump versions midway through a project unless you have to for a new feature or you have great backups - things can go wrong).

    • Q: Why are my images failing? (although not a greatly detailed question here of what is going wrong)

    I set my tiapp.xml to include these options, see if they improve things;

    <supports-screens
    android:anyDensity="true"
    android:largeScreens="true"
    android:normalScreens="true"
    android:smallScreens="false"
/>

    I also set my ti.ui.defaultunit as follows.

    <property name="ti.ui.defaultunit" type="string">dp</property>

    Make sure the android sdk versions are set to reasonable minimums;

    <uses-sdk
    android:minSdkVersion="10"
    android:targetSdkVersion="14"
/>

    These are examples and should be tweaked to match your own needs. Keep in mind that Android 2.3.3 is no the lowest supported android version.

    Make sure ALL of your images include a prefix forward slash so this;

    backgroundImage:'images/image1.jpg',
// becomes
backgroundImage:'/images/image1.jpg',

    This helps to normalise the iOS and Android differences. Your image folders appear fine.
