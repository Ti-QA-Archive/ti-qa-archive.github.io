  • Have you tried simply setting the ImageView height to 'auto'?
    It seems you're setting the width but unless you provide a height value, you'll end up with large margins at the top and bottom of your image as you have mentioned.

    • I ran across this issue today. Setting an 'auto' height is what causes the massive margins for me. Setting a fixed height, after getting the height back out of the image view, causes the image itself to disappear.

      SDK 1.7.6

      // gallery post is more recent, use that
var row = Titanium.UI.createTableViewRow({
    height:"auto",
    selectionStyle:'none'
});

// get the remote url for the image
var image = object.posts[count].imageURL;

var galleryImage = Ti.UI.createImageView({
    width: 300,
    height: 'auto',
    top: 10,
    bottom: 10,
    image: image
});

// return the actual height of the remote image            
Ti.API.info('Image height: ' + galleryImage.height);

// the next two lines correct the height issue, but the image is replaced by the default imageview placeholder                        
var forceHeight = galleryImage.height;
galleryImage.height = forceHeight;

// adds the image back in but borks the height
galleryImage.image = image;

// add the image to the tableview row
row.add(galleryImage);
