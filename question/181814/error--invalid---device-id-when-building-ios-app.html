[ERROR] : Invalid "--device-id" when building iOS app

I am running into some issues when trying to debug an app on certain iPods. I have 4 iPods that I am working with right now all running iOS 8.2. The iPod I have been using throughout the development process works just fine and I'm able to debug with it. However anytime I try to install the app onto any of the other 3 iPods, Titanium gives me the error message:

Available iOS Devices:
  xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx  Test iPod 1
[ERROR] :  Invalid "--device-id" value ""
  itunes                                    iTunes Sync
For help, run: titanium help build

The x's above are usually where it shows the UDID of the device I am trying to debug onto. I have made sure multiple times that I have added all of the iPods to the provisioning profile. I tried building the project through terminal and specifying the device UDID. Every time I go down that route, it does not load onto the iPod, but is ran in the simulator. I am using Titanium Studio build 3.4.1.201410281727 on Mac OSX 10.10.1.

Any help would be appreciated. Thank you in advance!

  • I actually solved this issue on my own. I have multiple Titanium SDKs installed. What I had to do is go into terminal and run 'titanium sdk select' and then I selected the SDK that corresponded to my projects. In my case it was 3.4.0 GA.
