  • > uncheck &quot;Include app symbols for your application to receive symbolicated crash logs from Apple&quot;

    I just ran into this same problem. The above solution works around a bug in Titanium Studio, but prevents us from including debug symbols in the App. I found the cause of this problem and came up with a hack which at least allows us to include the debug symbols (so we can leave the above checkbox checked).

    The problem is that Titanium Studio uses the wrong directory name for storing the debug symbols. Titanium Studio uses the subdirectory dSYM whereas Xcode expects dSYMs (notice the extra s at the end). This subdirectory in under the MyAppName.ipa directory in de Xcode Archives.

    The (still hacked) solution is to go to the Xcode Archive directory in Finder (you can do this from the Xcode Organizer), and then manually change the directory name.

    Of course this needs to be properly fixed within Titanium Studio.

    (I'm running on OSX 10.9.5, Xcode 6.1, Titanium Studio build: 3.4.1.201410281727, Ti SDK 3.4.0.GA)
