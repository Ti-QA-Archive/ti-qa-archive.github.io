Wildcard Bundle ID settings for iOS Distributing

I'm using Titanium Studio SDK 3.1.2 and I'm having trouble distributing my application …. I've taken all the necessary steps (App ID creation, iTunes connect settings currently waiting for upload,iOS distributing certificate and distribution provisoning profile; all matching the app ID previously created as wildcard ID bundle). I keep getting the following message (after the publishing process in Titanium Studio) when trying to validate from the Xcode organizer:

Invalid Code Signing Entitlements. Your application bundle's signature contains code signing entitlements that are not supported on iOS. Specifically, value 'MYTEAMID.com.xxxxx.appname*' for key 'application-identifier' in 'Payload/xxxxxxx.app/xxxxxxx' is not supported. This value should be a string starting with your TEAMID, followed by a dot '.', followed by the bundle identifier.

I've checked the app id and my settings in Titanium Studio, the match perfectly … I've also read this in the distributing docs:

If the Bundle ID is a Wildcard Bundle Identifier, you need to enter a Bundle ID Suffix, which you entered in the tiapp.xml file as the name portion of the app ID.

which I've also tried (writing 'appname' in the Bundle ID suffix textbox on iTunes Connect) without success ….. Any ideas?? I'm runniung out of time and this is a bit frustrating so I'd appreciate any help….

  • As far as I know, you can't use wildcards app ids for distribution. For distribution, an app_id needs to be totally unique.

    • Wildcards are permitted, but you cannot to in-app purchasing or push notifications

      Apple QA1713

      That being said, it is a hassle to have ids changed later on, in our shop, we believe best practice to be explicit app ids.
