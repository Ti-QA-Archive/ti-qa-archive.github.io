protect http request URL

Hi, i am getting remote JSON value as below.

var $Xhr = Ti.Network.createHTTPClient({
        onerror : function($e) {
            Ti.API.info("$e);
        },
        timeout : 5000,
    });

$Xhr.open("GET", "http://***********.json");
    $Xhr.send();
    $Xhr.onload = function() {
        if ($Xhr.status == 200) {
            try {
                Ti.API.info(this.responseText);
            } catch($e) {

                Ti.API.info($e);
            } finally {
                $Xhr = null;
            }
        }
    };

My json URL is static. i would like to protect this URL from stranger eyes after creating APK file or publishing for iOS.

  • i would like to protect this URL

    The URL represents the location of data, not the data. Even if you were to use SSL, this would only protect your data-in-motion, but expose the URL via widely available http observation tools.

    If your goal is to limit access VIA the URL to the DATA, then you need to implement a form of authentication or, as you have thought, encryption.

    MD5 is hashing. Hashing is NOT encryption.

    • Can i encode a string on the server side for example "teststring" can i decode it on appcelerator titanium app.

      The general answer is 'yes'. Both the client and the server need to support the same encryption algorithms. Both must have knowledge of the encryption key.

      Client side, I'd start with Ben's Securely module which can encrypt stings (think JSON.stringified data). You would need to find a module for your server that matches.
