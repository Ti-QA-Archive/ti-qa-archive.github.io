When ever I write a new module, then try to use require(), I get a script error informing me that it is unable to find the file.
If I click project->clean, then run the iOS simulator, it will run without issue. Because it can take anywhere between 2 and 5 minutes to rebuild after I use clean, I would REALLY like to not have to do that whenever I write a new file.

    George, you don't need to do a "full rebuild every time". Whenever you introduce a new source file or image to a project, you will need to do a "Clean" one time by going to the Project menu and choosing "Clean". Thereafter, as you change that new source file, you do not need to clean again.
