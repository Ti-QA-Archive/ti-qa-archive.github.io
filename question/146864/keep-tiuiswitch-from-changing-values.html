    Hi

    I have not seen any code yet but this is a very simple non-platform specific example;

    var sw = Ti.UI.createSwitch({
    value: Ti.App.Properties.getBool('changethispropertyname', false)
});
win.add(sw);

sw.addEventListener('change',function (e) {
    Ti.App.Properties.setBool('changethispropertyname', e.source.value);
    Ti.API.info('Switch value: ' + e.source.value);
});

    • You added your code 60 seconds before I added my example!

      Having had a quick look at your code you should find my example a simpler example to maintain.

      A couple of points about mine;

      Ti.App.Properties.getBool('changethispropertyname', false)

      This section asks for the current setting and if there is none it returns a default value (false). This does not need any logic attached to it as it is either a value that has been set or the default.

      Ti.App.Properties.setBool('changethispropertyname', e.source.value);

      This section allows the value of the switch to be used, as the code uses the e.source property you do not need to worry about the original value either, if you have a few you could go even further;

      var sw = Ti.UI.createSwitch({
    id: 'remember',
    value: Ti.App.Properties.getBool('remember', false)
});
win.add(sw);

sw.addEventListener('change',function (e) {
    Ti.App.Properties.setBool(e.source.id, e.source.value);
    Ti.API.info('Switch value: ' + e.source.value);
});

      This would allow you to name the switch and have a generic change handler - code for another day perhaps.

    • Exactly, you could even do this;

      function switchHandler(src) {
    // this is a generic handler
    Ti.App.Properties.setBool(src.id, src.value);
    Ti.API.info('Switch value: ' + src.value);
}
sw1.addEventListener('change',function (e) {
    switchHandler(e.source);
    if (something === somethingelse) {
        // code here as normal
    }
});
sw2.addEventListener('change',function (e) {
    switchHandler(e.source);
    // do nothing special
});
sw3.addEventListener('change',function (e) {
    switchHandler(e.source);
    if (up === down) {
        // code here as normal
    }
});

      This assumes your app had three different switches which you could call the same generic handler each time and then you can concentrate on the rest of your logic.
