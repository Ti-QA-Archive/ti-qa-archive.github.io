Prepare commonjs files/modules for 64 bit?

Hi
I have upgraded Titanium sdk to 3.5.0 GA as well as ios sdk (8.1) and xcode (6.1.1).

Now, when deploying on device (iPhone 6 or Ipad) I get this error message
"Script Error Couldn't find module: Data for architecture: x86_64"

I bet it is because I have not made the app 64 bit compatible.
I find instructions to make modules compatible, here

However, "Data" in the error message above, is a simple commonjs file (…commonjs/Data.js) alongside with a whole bunch of other ones. The instructions in the link above does not seem to apply in my case.

So how would I prepare an app, based on simple commonjs files, for 64 bit?
Any hints?

Thanks in advance /Tove

  • In xcode, make sure that you have set in Build Settings –> Architectures –> Architectures to $(ARCHS_STANDARD) for all targets and set set in Build Settings –> Architectures –> Valid Architectures to arm64 armv7 armv7s i386 x86_64

    From Xcode menu select Product –> Scheme –> ' Select Your project target'
    then From Xcode menu select Product –> Scheme –> Edit Scheme make sure all of the items in the list on the left are set to 'Release' in the pane on the right, select each item in turn and select from the 'Build Configuration' pulldown.

  • For anyone interested I have had this problem a couple of times. I don't know why at starts?

    However, I have found a simple solution that sort of "resets" the problem:

    1. In Studio on the project, right-click and select "Show in / Finder"
    2. Locate the "build" folder - and delete it
    3. Locate the "Resources" folder - and delete that as well
    4. Build your project again

    These steps seem to help in the cases where I have seen this problem :-)

    I have read about "architectures" in manifest.xml (i.e. iOS section of tiapp.xml in Titanium Studio) - however, so far I have not found anything conclusive…. And really, those articles I have found have been related to native iOS modules - whereas the problem here is with a simple CommonJS file in the lib folder…
