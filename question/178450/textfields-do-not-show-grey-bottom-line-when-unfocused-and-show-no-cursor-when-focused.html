TextFields do not show grey bottom line when unfocused and show no cursor when focused

Hi.

I am developing an Android app using Titanium SDK 3.4.0.201409261227-26092014122818 on Windows 7, testing on a HTC One device. My Titanium Studio build info is: Titanium Studio, build: 3.4.0.201409261227 Build: jenkins-titanium-rcp-master-197 (origin/master) Date: 26 September 2014, 12:28:18.

All of the text fields in my app do not display the default Android grey bottom line to show that they are text fields. If I select one of them then a blue (focused) line appears correctly. In addition, no cursor shows when the text field is focused though I am able to type ok.

I don't seem to have any styles that would overwrite the line/cursor behaviour.

Can anyone else re-produce this bug with the latest version of Titanium (3.4.0)?

Thank you.

  • Hi

    I had the same problem. I could not make titanium to apply my custom theme in app.

    I must mention that in my test project it was able to import custom theme, in my important more complex project theme was never imported.

    I ended up with this solution/workaround

    When it was not working with customtheme.xml in my more complex project.
    I tryed one more thing, I found theme.xml in build/android/res/values
    copy it and paste it into platform/android/res/values
    then I modify I to so it is looking like this: 

    <?xml version="1.0" encoding="utf-8"?>
<resources>
    <style name="Theme.AppCompat.Fullscreen">
        <item name="android:windowNoTitle">true</item>
        <item name="android:windowFullscreen">true</item>
        <item name="android:windowContentOverlay">@null</item>
        <item name="android:windowActionBar">false</item>
        <!-- AppCompat Compatibility -->
        <item name="windowActionBar">false</item>
        <item name="android:editTextStyle">@style/editText</item>
    </style>

    <style name="Theme.Titanium" parent="@style/Theme.AppCompat">
        <item name="android:windowBackground">@drawable/background</item>
        <item name="android:windowActionBar">false</item>       
        <item name="android:windowNoTitle">true</item>
        <!-- AppCompat Compatibility -->
        <item name="windowActionBar">false</item>
        <item name="android:editTextStyle">@style/editText</item>
    </style>

    <style name="Theme.AppCompat.Translucent">
        <item name="android:windowBackground">@android:color/transparent</item>
        <item name="android:colorBackgroundCacheHint">@null</item>
        <item name="android:windowIsTranslucent">true</item>
        <item name="android:actionBarStyle">@style/Widget.AppCompat.Base.ActionBar.Solid</item>
        <!-- AppCompat Compatibility -->
        <item name="actionBarStyle">@style/Widget.AppCompat.Base.ActionBar.Solid</item>
        <item name="android:editTextStyle">@style/editText</item>
    </style>

    <style name="Theme.AppCompat.Translucent.NoTitleBar">
        <item name="android:windowNoTitle">true</item>
        <item name="android:windowContentOverlay">@null</item>
        <item name="android:windowActionBar">false</item>
        <!-- AppCompat Compatibility -->
        <item name="windowActionBar">false</item>
        <item name="android:editTextStyle">@style/editText</item>
    </style>

    <style name="Theme.AppCompat.Translucent.NoTitleBar.Fullscreen">
        <item name="android:windowFullscreen">true</item>
    </style>

    <style name="editText" parent="@android:style/Widget.EditText">
        <item name="android:textCursorDrawable">@null</item>
        <item name="android:textColor">#000000</item>
    </style>

</resources>

    this seems to be working, so far.
    This can be helpful for someone who cannot make customtheme work.
    in this case leave in platform/android/res/values only file themes.xml
    in tiapp.xml is no need anymore to define application theme so you can delete it.
    you will end up with this (tiapp.xml)
    <android xmlns:android="http://schemas.android.com/apk/res/android"> </android>

    hope this helps you.

    I am developing on titanium 3.4.1. on mac.
