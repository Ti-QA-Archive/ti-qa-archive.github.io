Fatal Error Cannot Start Titanium Developer on Windows XP SP3

I'm new to Titanium Developer, and I can't get my fresh installation to run on Windows XP SP3.

Output of my JDK setup:

Microsoft Windows XP [Version 5.1.2600]

C:\>java -version
java version "1.6.0_24"
Java(TM) SE Runtime Environment (build 1.6.0_24-b07)
Java HotSpot(TM) Client VM (build 19.1-b02, mixed mode, sharing)

C:\>javac -version
javac 1.6.0_24

Contents of tiapp.log (using –debug) when it fails to launch and encounters a fatal error:

http://pastie.org/1658334

I've found this error in some other support Q&A posts, but no one has seemed to solve this.

I've uninstalled an reinstalled Titanium Developer multiple times including wiping all the Titanium application data under C:\Documents and Settings.

Additionally, since it seems to be a network related error, I've turned off my firewall entirely, with no success.

Thanks

  • Solved!

    I was able to get Titanium to launch correctly after disabling my firewall entirely.

    It seems ZoneAlarm and Titanium do not play nice together. Giving Titanium full outgoing and inbound (Trusted/Internet zones) access within ZoneAlarm was NOT enough. I had to completely shutdown ZoneAlarm for Titanium to startup.
