I'm running Titanium 1.7.3 developing for iPhone ios 5.

Ive run into a problem where i have created an audio streaming section in my app using AudioPlayer to stream preview songs from itunes. Everything is working fine except for one hiccup.

I have put an event listener for the AudioPlayer state that prints out the state into the debug console

When I play the first URL i get this print out:

[INFO] State: starting (1)

[INFO] State: waiting_for_data (2)

[INFO] State: waiting_for_queue (3)

[INFO] State: playing (4)

However when i change the URL with a next event listener and play the next preview (URL) this prints out into the log

[INFO] State: starting (1)

[INFO] State: waiting_for_data (2)

[INFO] State: waiting_for_queue (3)

[INFO] State: playing (4)

[INFO] State: buffering (5)

[INFO] State: playing (4)

Now the problem is that there is a slight start then stop then start of the audio when it goes from 'playing' to 'buffering' to 'playing'. Which results in the audio cutting in then out then in again over 1 second which is annoying. But it plays fine after that just the start of the audio is not working correctly

I know it has nothing to do with the data connection (wifi or 3g) as it happens on the simulator as well as when I test it on the device. (p.s the problem is inconsistent as it happens maybe 1 in 5 times on both device and simulator)

The sound cutting in and only happend on any audio played after the first one.

I have pretty much rewrote the code in every possible combination even removing everything except the audio player as I thought it was my code. However even with just the AudioPlayer in the code the problem was still happening

The only logical conclusion i can think of is some how releasing the AudioPlayer memory or some how resting it (which I don't know how to do). I have also tried the poorly documented setUrl property to no avail.

Any help will be appreciated.
