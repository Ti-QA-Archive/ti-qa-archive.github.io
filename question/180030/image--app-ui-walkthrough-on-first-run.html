    Another approach I've seen is to use an overlay atop the actual window. You'd show that overlay the first time the window is opened. The overlay would contain all the help text and arrows (and a transparent background). Tapping an "X" or something would close it.

    This approach lets you provide help info in context, when the user first loads the window. And it doesn't overwhelm new users by showing all the help info on the first launch. It would be useful if your app has a few windows, some of which the user might not reach on their first use. For example, after a few minutes or days of use, the user reaches WindowFoo at which time its overlay is shown.

  • I had used scrollable view with images inside it and skip button at bottom to skip the tutorial/help. When use login in first time this tutorial/help shown upfront to him. He can skip also if he don't want to go further with this tutorial/help.

    There are several ways, to check whether the user is first time launching the app.

    1. We have set variable at server side and on calling of login web service it returns which page to navigate. We kept this is because we want to render the user to password change screen on first login and then after this to tutorial/help screen. We don't want to put headache of checking the variable on each app launch and managing at client side.

    2. If your app does not have login screen. You can set Ti.App.properties and used that variable in app launch (app.js in Titanium classic/ alloy.js in Alloy).

    Hope this helps.
