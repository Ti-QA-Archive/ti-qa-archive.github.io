How can I view files Created?

Dear Forum members,

I have some code that creates a text file.

I'm pretty sure the file is created.

Of course, as you can imagine, its most likely going to be created in the iOS simulator's memory.

How can I access that? How can I examine the simulator's directories to see if my file was really created?

  • Hello E B,

    You can use Get File method to access your created file in your application. like :

    // resourcesDirectory is actually the default location, so the first 
// argument could be omitted here.
var file = Ti.Filesystem.getFile(Ti.Filesystem.resourcesDirectory, "textfile.txt");
if (!file.exists())
{
    alert('file not exists');
    return;
}

var blob = file.read();
var readText = blob.text;
// dispose of file handle & blob.
file = null;
blob = null;

  • Hi,

    you can use get directory listing method to list all the files created within the directory.

    for example…

    var dir = Titanium.Filesystem.getFile(Titanium.Filesystem.applicationDataDirectory);
Ti.API.info('directoryListing = ' + dir.getDirectoryListing());

    It will return all the lists of files of applicationDataDirectory.

    and as per your case…you can use your file name to check whether its available or not..like below…

    var file = Titanium.Filesystem.getFile(Titanium.Filesystem.applicationDataDirectory,<file name>);
alert(file.exists());

    so if you are creating file dynamically then this code will help you.

    There is no need to check whether the file is available in resource directory if you are create files dynamically.

  • The iOS Simulator folders can be accessed by going to

    \Users\{user name}\Library\Application Support\iPhone Simulator\{ios version}\Applications\{application id}

    Within there you have:

    • Documents: equivalent to Ti.Filesystem.applicationDataDirectory
    • Library: this contains
      • Caches: equivalent to Ti.Filesystem.applicationCacheDirectory
      • Preferences: where the Ti.App.Properties are stored in the {app id}.plist file
      • Private Documents: where the SQL lite databases are normally stored
    • tmp: equivalent to Ti.Filesystem.tempDirectory

    A bit quicker to check the content has been stored correctly, however always good to have code tests built as well so that you can confirm the content has been stored correctly on the device.

    Hope this helps
