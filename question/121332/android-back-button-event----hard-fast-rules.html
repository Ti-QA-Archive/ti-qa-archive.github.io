Android Back Button Event -- Hard Fast Rules?

I've been working on an Android app for a couple months now. It's a simple Geolocation app, and it works perfectly, except that I do not have complete control over when the application finally exits.

Toggling the window properties such as modal, fullscreen, hideNavBar, and exitOnClose do not seem to achieve what I'm trying to do. I simply want the app to close when the exit button is pressed. I have found a variation that works, but then the MapView misbehaves (doesn't redraw properly after zooming, etc.). At the simplest level, my app only needs the MapView, there are no menus, tabs, or anything else needed other than the map and the annotations.

Can somebody give me a brief list about the rules on heavyweight/lightweight windows, whether or not explicit handling of the android:back event is necessary, and any caveats about using Windows, Views, MapViews, etc.? Here are the UI window bits of code:

app.js:

cmc = {};

cmc.win = Titanium.UI.createWindow({
    title:'Geo App',
    modal:true,
    exitOnClose:true
});

Ti.include('./stations.js');
Ti.include('./map.js');

cmc.win.add(cmc.map.view);
cmc.win.open();

    You must be doing something fishy in other place.
    As a rule for Android, don't add windows to windows, just use open.

    A window with the specified properties to be heavyweight will create problems when it comes about maps.

    url,modal, fullscreen, navBarHidden will make a win heavy no matter if they are true or false, but the simply declaration in the constructor will do it.
