SOCKS proxy required for Titanium Developer?

I am trying to install and use Titanium Developer on Windows XP in an environment that does not allow direct Internet access, and no SOCKS proxy is available. Only pure HTTP and HTTPS proxy functionality is available.

Is it possible to get Titanium Developer to work in this environment? I have run through the –debug option process, and have managed to get the HTTP proxy server to be recognized and used by Titanium Developer upon startup ("[Titanium.Proxy] [Debug] Found proxy http://<blah,blah,blah>:1080) in environment" - proxy removed from the message).

However, the connection drops after a few seconds ("You are not connected to a network", "[Titanium.NetworkStatus] [Debug] ti.Network: Online status changed ==> 0").

So, I spun up Wireshark. I can see the request going out to api.appcelerator.net, and the 200 connection established coming back. After that, there's a fair amount of retrieving certificates going on, and then the connection appears to drop.

Any suggestions? Is it possible to get this working in a pure HTTP/HTTPS environment without SOCKS support?

  • Hi Joe

    Yes, it's certainly possible. Set the environment variables relevant to your situation:

    • HTTP_PROXY
    • HTTPS_PROXY
    • FTP_PROXY
    • SOCKS_PROXY

    The format for the value is http:&#x2F;&#x2F;username:password@proxy-name-or-ip:proxy-port or simply http:&#x2F;&#x2F;proxy-name-or-ip:proxy-port for proxies that do not require authentication.

    Hope this helps
