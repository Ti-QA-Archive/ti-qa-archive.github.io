Alloy TabGroup Width

I'm just getting started with Alloy (Converting app from regular) i'm struggling to figure out where to set the width of the tabgroup, I've got the below in my index.xml:

<Alloy>
    <TabGroup id="TabGroup">
        <Require src="Home" id="TabHome" class="Tab" />
        <Require src="MyStories" id="TabMyStories" class="Tab" />
        <Require src="Browse" id="TabBrowse" class="Tab" />
        <Require src="Me" id="TabMe" class="Tab" />        
    </TabGroup>
</Alloy>

I tried setting the below values in my app.tss file

"#TabGroup": {
    width: Ti.UI.FILL
}

But it doesn't seem to do anything, i want my tabgroup to be the width of the screen

    Hi

    A tabGroup will automatically fill the height and width of the screen - allowing your windows to fill the internal space remaining. So I suspect you have an issue in the Require's.

    This is a simple example of an Alloy tabGroup;

    <Alloy>
    <TabGroup backgroundColor="white" >
        <Tab id="tab1" title="Tab 1" icon="KS_nav_views.png">
            <Window id="win1" title="Tab 1">
               <Label id="label1" color="#999">I am Window 1</Label>
            </Window>
        </Tab>
        <Tab id="tab2" title="Tab 2" icon="KS_nav_views.png">
            <Window id="win2" title="Tab 2">
                <Label id="label2" color="#999">I am Window 2</Label>
            </Window>
        </Tab>
        <!-- Use the Require tag to include external Ti.UI.Tab views -->
    </TabGroup>
</Alloy>

    You will notice the first child nodes are Tab - these will need to the same inside your Requires, if they are not the first child then this is likely where the issues are.

    I would suggest backing up your index.xml file and then pasting the above code in-place - does this display as you would have expected - if so then work through the require file.

    BTW given a tabGroup is a container that would automatically fill the space normally, there is no need to include it in a TSS file, but if you do intend on doing that you do not require the id reference, simply TabGroup is sufficent.

    "TabGroup": {
    width: Ti.UI.FILL
}

    • OK - the screen shot cleared it up, you had not said it was specifically android nor version 4.0 with actionbar. That is OK.

      iOS and non-actionbar android (any version) does exactly as I said.

      However Android 4.x with ActionBar and tabs - is a different thing altogether. Firstly it is taking up the whole width - at least the container is, you will notice if you add a label to the centre of a window it will display in the middle of the device - it is not pushed out to the right.

      • Bad news - nothing you can do about this other than shorten the tab names
      • Good news - this is the expected behaviour.

      The tabs are meant to scroll off the screen if the captions plus padding of the UI means they do not fit. The scroll action allows over flows to occur. If your app supports landscape you will notice the tabs move into the actionbar directly with a spinner displaying them.

      You may be unhappy with this - but it is meant to be that way.
