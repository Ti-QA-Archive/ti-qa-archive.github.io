Native App for Honeycomb (3.x) always "zoomed/stretched"

Might be I looked at the wrong places or just overlooked the obvious… but it doesn't matter what I've tried, all my Apps done with the latest TitaniumStudio (SDK set to Android 3.1) look like "upscaled" 2.x Apps. There is always the "Screen Compatibility" Icon in the Statusbar indicating that the App is upscaled to fit the screen.

Same goes for the UI components (scrollbars, buttons etc). The Honeycomb-Look is completely missing. Just check out the KitchenSink example…

Is this normal? Do I have to "add" the Honeycomb-Style myself somehow? If so, how?

I'm planning to write an App especialy for Honeycomb Tablets and the Compatibility-Mode and Look&Feel realy gives me a headache… ;-)

Thanks for any Help

Oliver

    If you are using the anyDensity:False is your TiApp.xml then you automatically get upscaled on larger device screen sizes. Checkout this link for how to support multiple screen sizes http://developer.appcelerator.com/blog/2011/06/new-defaults-for-android-layouts-in-1-7.html
