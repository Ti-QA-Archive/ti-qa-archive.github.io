BUG: table row containing view with vertical layout results in blurry font

If you have a table row that contains a view with a vertical layout it displays blurry text. You can see an example of this in the "twitter" mashup in the kitchen sink.

Note: it only applies to the non-retina display. Text displayed on a retina display is always crisp.

  • This happens because a label will sometimes stretch text to fill the container. To avoid this, set a constant height for the container … not 'auto'. Not sure if this is a bug, or by design.

  • I'm pretty sure that this is a bug - but I've found a solution. Add 0.5 pixel to the "top" of the label (or other item) you are putting in the row and the text will look fine. Thanks to a colleague of mine for suggesting this. He took one look at it and recognised that it looked liked a 0.5 pixel misalignment problem.
