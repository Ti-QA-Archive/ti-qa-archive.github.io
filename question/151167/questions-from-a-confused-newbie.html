Questions from a Confused Newbie

I have spent the last two weeks installing and setting up Titanmium and watching numerous videos on this site.

I am confused about a few issue and was hoping I could get some answers to enable me to proceed with my training…

  1. Is there a document which describes the development timeline of Titanium since it was originally launched, I have found references to Titanium Mobile & Desktop, Titanium Studio, Titanium CLI and Alloy and am trying to understand how (and when) the difference components fit together and how the product has developed over time.

  2. All the Titanium training I have seen so far doesn't seem to mention the Alloy Framework [I have seen the videos from Codestrong2012 when Alloy was "launched" but no step by step training]. If I am learning to build Apps in Titanium from scratch now [April 2013] would it be better to start with the original framework or should I start with Alloy ? Which is better for beginners ? Will Alloy become the default way of coding within Titanium in the future ? Are they any videos that show how to create Apps using Alloy ? [or maybe I have totally misunderstood how it all works]

  3. The CLI - I have seen references to this across numerous videos - am struggling to understand whether this is something I need to know about - why would I want to use the command line when I can doing everything I need to within Titanium Studio ?

  4. This Q&A Section - whenever I search for answers here most of the search results show up Q&A's from 1-2 years ago - is there a different way of searching or maybe there is somewhere else I should be looking for answers, such as a Google Group ?

Apologies if I have posted this in wrong place or I have missed something obvious.

Cheers
David

  • Accepted Answer

    1: Not that I know of but here is a short explanation of each:

    • Titanium Mobile: The SDK that lets you build mobile apps
    • Titanium Desktop: This no longer exists, its now supported by the community and not appcelerator, the name has changed to TideSDK
    • Titanium Studio: The IDE/Code Editor built by appcelerator with tools to make Titanium development "easier" this is not needed to develop Titanium apps though, you can use any text editor
    • Titanium CLI: a command line tool to compile Titanium mobile apps, use this if you want to write code outside of titanium studio
    • Alloy: A framework built by Appcelerator to make Titanium development easier, it's a framework so it does not replace Titanium, it simply provides guidelines.

    2: There is no way to fully answer this without simply voicing opinions, me and my team are not fans of Alloy, many people live and die by it. Alloy will never replace non alloy development, but some features might bleed between the core and the Alloy framework.

    3: Studio is an editor + tools + a UI for the CLI, if you want to use studio, there is no real reason at the moment to "learn". If you have an editor you would rather use over studio, then learning the CLI would be a good thing.

    4: The Q&A sorts by relevance to your search not by date.

  • I can speak for #2…

    Almost everything you've asked regarding Alloy can be ascertained from the list of resources found here: https://developer.appcelerator.com/question/148610/alloy-kitchen-sink-documentation-samples

    In terms of its direction, Alloy is not meant to replace traditional Titanium development. It is a framework for Titanium development. It provides a standardized MVC structure, declarative UI via XML, styling, widgets, data adapters, compile-time optimizations, and a whole bunch of other stuff to make Titanium development faster and more performant, scalable, and maintainable. But if you don't like the approach, there's nothing binding you to use it, you can use as much or as little as you like (or none). That said, we use it for just about everything now at Appcelerator, both for apps developed internally and those for customers. It is in fact being integrated into the training curriculum. And if I can find a spare minute or two I probably should do a screencast or two. Though I think some might already exist if you search hard enough on google from the community developers.
