Titanium deletes the images while compiling.

Hi..

Titanium encounters a problem in adding images to default image folder in Resources. When I add new image to this folder, Titanium automatically deletes the image on compiling the app. Same problem is there when I tried to delete an image from the folder. when the app is compiled the folder is reverted to old state. Please help to solve the issue. I'd tried updating the titanium. It had worked well with me before.

    Hi

    Easy one to not know this;

    You need to place your images in the;

    {appname}/app/assets

    Folder and NOT the resources folder. Each time you compile your app the resources folder is wiped (as you have found) and the new code created from your alloy app is compiled into the resources folder. Any images are added to the folder above.

    This is the note from the documentation;

    • Contains image assets and other files that need to be copied into the Resources directory. Reference these files in the code without the 'app/assets' path.

    The Alloy documentation specific to this can be found in the Alloy - Convention over Configuration section.

    • @Josh, it is very much an Alloy specific thing and it is very much wiped.

      When you create an app using non-Alloy, you do all your code in the resource folder, build your app, it creates the build folder as you correctly point out. You clean the job and the build folder is emptied, but re-built on demand each time.

      In Alloy however the situation is VERY different, you put your code in the app folder, including the views, controllers, models, styles and views (plus assets). You do NOT place anything in the resources folder. You build the app, it pre-process the files from Alloy to normal code, all carefully considered for platform and mark-up. This code is placed into the resources folder and then same build process is done as the non-alloy projects.

      Please note the documentation on the subject;

      • Cleanup; The Resources folder is cleaned of any previous built files. If a platform-specific folder exists in the Resources folder, but not in the app/assets folder, it and its contents are not removed.

      You can read more about the Compilation Process in the documentation.

      But I assure you the code is wiped from the resources folder in an Alloy project during build. This was an Alloy question and the answer was given on that basis.
