Pause event only fires sometimes

I have programmed an application which needs to call a function (fncPause) when being paused.

For that reason I registered an event listener:

// add event listener for application pause to pause test
Titanium.App.addEventListener('pause', fncPause);

When pausing the app with the IOS home button the pause event listener fires - but only sometimes.

On some occasions the event listener does not fire when the pause event is pressed. Instead it fires when the application resumes. This behaviour is really a problem for my app.

I might have identified the reason for the missing fire on pause event: It seems that the event listener does not get fired properly when the application is already performing actions at the time the home button is pressed.

Does anybody have an idea how to make sure that the event listener gets fired immediately on "pause" event?

thank you for your help!
