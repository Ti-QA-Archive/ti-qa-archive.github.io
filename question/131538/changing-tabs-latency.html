Changing tabs latency

Changing from the first tab to the second tab (for instance) i notice a latency in the loading of the titleimage resulting in an bad looking behaviour:

1) change tab
2) the black bar is showed
3) the titleimage is loaded
4) the titleimage is showed

This problem starts when i divide my project in multiple files.

This is my structure right now:

->APP.JS

create the tabgroup, create the tabs, create the windows with url:"windows/SOMETHING.js" and titleimage:"SOMEIMAGE.png", adding the tabs to the tabgroup

->WIN_#.JS

all the stuff for each window.

How to speed up the loading time and/or preload the images?

  • You may have better luck using a single context app and pre-generate the window that will be associated with the tab so you get less latency. When using the 'url' property (and therefore a multi-context app) your window code is only getting evaluated (first time) after the first click on that tab

  • Hi Maurilio,

    You should consider doing something like this instead:

    // create a standard window
var win1 = Ti.UI.createWindow({ backgroundColor: 'white' });

// create a button
var button = Titanium.UI.createButton({
   title: 'Hello',
   width: 100,
   height: 50,
   top: 10
});

// add button to win1
win1.add(button);

// add an event listener to the button
button.addEventListener('click',function(e)
{
   win2.open(); // open win2 when required
});


// create and already place the views/buttons/objects/UI elements inside win2
var win2 = Ti.UI.createWindow({ backgroundColor: 'yellow' });

// create a view
var view = Ti.UI.createView({ 
    backgroundColor: 'red', 
    width: 200, 
    height: 200, 
    top: 10, 
    left: 10 
});

// add view to win2
win2.add(view);

// open win1 when app runs
win1.open();

    Hope this helps.
