  • Accepted Answer

    These issues should be addressed in Studio 3.0.1. You must have the correct versions of the SDK and CLI installed as well. If you are seeing these issues, please make sure you have the following installed:

    • Studio 3.0.1.
    • Titanium CLI 3.0.21 (titanium -v)
    • SDK 3.0.0.GA

    If you are still experiencing issues, check whether the CLI can see your provisioning profiles:

    titanium info -t ios

    This should list your distribution certificates. If you do <em><strong>not</strong></em> see any listed by the CLI, chances are you are running into a known issue with the CLI that causes it to fail when there are a large number of keys in the keychain. To check for this issue, run the following command:

    security dump-keychain | wc -c

    This returns the size of the keychain output in bytes. If the size exceeds 1M, you're seeing a known CLI bug which will be addressed in the near future.

    If the CLI does list certificates, try building from the command line. This indicates a problem between Studio and the CLI. If this is the case, please open a JIRA ticket so we can investigate ASAP.

  • Hi Ashok and Slim,

    We are actively investigating the cause of this issue, and there is currently a ticket open against it in https://jira.appcelerator.org/browse/TC-1616. We have not been able to reproduce the behavior locally, so would very much appreciate your help in tracking down the root problem. The last comment in ticket TC-1616 has steps to provide us more logging information on what is going on. I will copy them here:

    1. Update Studio to the beta stream using the update site http://preview.appcelerator.com/appcelerator/studio/standalone/update/beta/;
    2. After Studio restarts, clear the log via Help > Titanium Studio > Clear Log File;
    3. Open the Preferences > Titanium Studio > Troubleshooting page, set the Debug Level to all, and make the "com.appcelerator.titanium.mobile/debug/provisioning" and "com.aptana.core/debug/shell" entries checked;
    4. Restart Studio, and open the Preferences > Titanium Studio > Titanium > iOS Platform page;
    5. Click "Refresh" button in the "Provisioning Profiles" section and close the page;
    6. Attach the log content from Help > Titanium Studio > View Log File to the JIRA ticket.

    Thanks very much.
