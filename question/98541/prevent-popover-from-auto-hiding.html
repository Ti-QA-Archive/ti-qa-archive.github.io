Prevent popover from "auto hiding"

Messing around with popovers in an iPad app, and it works great.

Only trouble is that if I touch on any part of the screen outside of the popover, it automatically closes it. I need to prevent this behavior as I will have a Cancel button the user should click if they want to close the popover.

A good example is the iPad Calendar app. A popover is displayed when creating a new appointment. Touching outside of the popover does not close it.

I didn't see an obvious property setting that would prevent this, but I more than likely have overlooked it.

Many thanks!

  • As Mark mentions, you need to use the passthroughViews property which can be set using the setPassthroughViews method. It takes an array of one or more views which should be permitted to NOT auto close the popover. Try:

    var myPopover = Ti.UI.iPad.createPopover( ... );
myPopover.setPassthroughViews( [myCancelButton] );
myPopover.show(...);

    You can list other buttons or views as well if there are other things which should be permitted to accept taps.
