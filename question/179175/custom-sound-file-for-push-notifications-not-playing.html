Custom sound file for push notifications not playing

I copied the file pop.caf from Kitchensink to /Resources/pop.caf in my project. I cleaned the build, removed the app from the device and re-installed.

Then I sent a test push notification from the ACS console, specifying 'pop' in the custom sound field. When the notification was received on the iOS device, it played the default sound for push notifications instead of the pop.caf file.

Any ideas what's wrong?

  • I finally figured it out. Contrary to what I found in the docs, the file extension (.wav) should be included in the payload.

    Example

    curl -X POST -F "channel=<channel_name>" -F "to_tokens=<tokens>" -F "payload={\"alert\": "'Sample alert'", \"icon\": "'appicon'", \"sound\": "'boing.wav'", \"vibrate\": "'true'"}" "https://api.cloud.appcelerator.com/v1/push_notification/notify_tokens.json?key=<APP_KEY>&pretty_json=true"

    I converted the file to WAV file type as it has to be the same on both iOS and Android when I specify the the file extension in the payload.

    Also note the escaping and use of string in the payload object.
