  • Well, the Titanium compiler works pretty much like this.

    • It will take your JavaScript code and will generate native code for all the native components that your application will use. (Buttons, Labels, List Views, etc.)
      • But all the Javascript code used in your event handlers (when the user clicks a button for example) will remain in JavaScript and will be executed later on by the platform's JavaScript Runtime.
    • Once the native code is generated, it will then be compiled into a native application.
    • It will then package the application by assembling the generated (and compiled) native code, the JavaScript Runtime as well as the JavaScript code for the event handlers. (not that this code is usually obfuscated to avoid retro-engineering.
