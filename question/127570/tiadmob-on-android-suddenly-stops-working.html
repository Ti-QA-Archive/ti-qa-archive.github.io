  • I just figured it out, following what Mark posted in the comments.
    First, you'll need to download the latest AdMob Ads SDK, which is 4.3.1. You can download it here.

    Next, go to your project folder > modules > ti.admob > 1.0 > lib. Remove the current .jar file and replace it with the one you just downloaded.

    Next, go to your project folder > modules > ti.admob > 1.0 > timodule.xml. You'll need to change this line:

    <activity android:name="com.google.ads.AdActivity"
              android:configChanges="keyboard|keyboardHidden|orientation"/>

    With this:

    <activity android:name="com.google.ads.AdActivity"
              android:configChanges="keyboard|keyboardHidden|orientation|screenLayout|uiMode|screenSize|smallestScreenSize"/>

    Next, go to your tiapp.xml and add the following line, just after <android xmlns:androidâ¦.

    <tool-api-level>14</tool-api-level>

    I'm using Android 4.0 SDK, so that's the api level for it. Also, the admob docs recommend using the latest sdk for compiling. Go here for more info.

    And that's it. I already tested in the emulator using SDK 4.0 and 2.3.3 and it worked.

    Hope it can help! Thanks Mark for your post!

    • Douglas, try using the latest module in the Appcelerator Marketplace. Also, I don't use adMobView.requestAd() to request the ad. I just add the adMobView to the window or view and that's it. For me it works that way. Use the latest tool-api-level too. Here is a sample: 

      var win = Titanium.UI.createWindow();

var adMobView = admob.createView({
        publisherId:"YOUR ID HERE",
        testing:false, // default is false
        zIndex:2,
        //top: 10, //optional
        //left: 0, // optional
        //right: 0, // optional
        bottom: 0, // optional
        adBackgroundColor:"FFFFFF", // optional
        backgroundColorTop: "738000", //optional - Gradient background color at top
        borderColor: "#000000", // optional - Border color
        textColor: "#000000", // optional - Text color
        urlColor: "#00FF00", // optional - URL color
        linkColor: "#0000FF" //optional -  Link text color
        //primaryTextColor: "blue", // deprecated -- now maps to textColor
        //secondaryTextColor: "green" // deprecated -- now maps to linkColor

    });
//listener for adReceived
    adMobView.addEventListener(admob.AD_RECEIVED,function(){
       // alert("ad received");
       Ti.API.info("ad received");
    });

    //listener for adNotReceived
    adMobView.addEventListener(admob.AD_NOT_RECEIVED,function(){
        //alert("ad not received");
         Ti.API.info("ad not received");
    });

win.add(adMobView);

win.open();

      Hi Qing,

      Thanks for the attachment. I've tried to run it, but a few changes have to be made.

      timodule.xml needs to be changed, because the new AdMob SDK expects the following configChanges:

      android:configChanges="keyboard|keyboardHidden|orientation|screenLayout|uiMode|screenSize|smallestScreenSize"

      uiMode is only available as from Android 3.2. And this is what I cannot solve in Titanium. I've got the Android 3.2 SDK and changed the target in the manifest file of my project to 13.       Sincerly, Mark

      I figured it out. The Android SDK target can be altered by setting tool-api-leven in tiapp.xml. I did so, and it worked!!
