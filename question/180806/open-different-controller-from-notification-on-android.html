  • Don't know if you still need it.

    Thats how I do it:

    var intent = Ti.Android.createIntent({
    action : Ti.Android.ACTION_VIEW,
    packageName:"de.my.package",
    className:"de.my.package.ReminderRecieverActivity"
});
intent.addCategory( Titanium.Android.CATEGORY_LAUNCHER );
intent.putExtra( 'foo', 'bar' );

var pending = Titanium.Android.createPendingIntent({
    intent: intent,
    type : Ti.Android.PENDING_INTENT_FOR_ACTIVITY
});

var notification = Ti.Android.createNotification({
    contentTitle: 'TITLE',
    contentText: 'MESAGE',
    tickerText: 'MESSAGE',
    icon: '/images/logoSmall.png',
    contentIntent: pending,
    defaults: Ti.Android.DEFAULT_ALL
});

Titanium.Android.NotificationManager.notify( 0, notification );

    Additionaly I have a file under app&#x2F;assets&#x2F;with the name ReminderReciever.js

    And the tiapp.xmlhave

    <activities>
    <activity url="ReminderReciever.js">
        <intent-filter>
           <action android:name="android.intent.action.VIEW"/>
           <category android:name="android.intent.category.DEFAULT"/>
        </intent-filter>
    </activity>
</activities>

    The Content of the ReminderReciever.jsis:

    Alloy.createController('yourControllerName').getView().open();

    The Only issue I have with this Setup is that when you close your app, an another Instance of your app opens.
    So basicly I have to disable the Function that my App can be closed.

    I know its ugly but that I also got from trial and error because of a missing correct example :)
