I took a look at the samples posted on the website.

  • Hi Davide

    You cannot really have pages "remotely hosted", although you can open web pages where all the content is on the server. Be aware of the caveat to this approach, explained by this user. Anyway, you can view webpages within the app using the Ti.UI.WebView object, as demonstrated in this and this post. Also, search the KitchenSink repo for scripts with the name "webview" for examples. Particularly, check out local_webview.html. Also, mixing_views.js may be useful to you.

    Usually, you would have a webservice that, when queried using Ti.Network.HTTPClient, would return data that your local interface will act upon (and hence appear differently). Search the KitchenSink for examples in files with the filename containing "xhr" in this case.

    Hope this helps
