Events on fullscreen video

Hi,

I have problems with adding events to fullscreen video in Titanium SDK 3.2.0 GA.
If video is not fullscreen - event is fired, but when video is fullscreen - no events.
And if video is not fullscreen, but user click on fullscreen button on player - again no events.

Here is sample code (for fullscreen just change parameter fullscreen to true):

var vidWin = Titanium.UI.createWindow({
    title : 'Video View Demo',
    backgroundColor : '#fff'
});

var videoPlayer = Titanium.Media.createVideoPlayer({
    url: "http://movies.apple.com/media/us/ipad/2010/tours/apple-ipad-video-us-20100127_r848-9cie.mov",
    autoplay : true,
    fullscreen: false,
    scalingMode : Titanium.Media.VIDEO_SCALING_ASPECT_FIT
});

videoPlayer.addEventListener('click', function(e) {
    Ti.API.log('click');
    Ti.API.log(e);
});

vidWin.add(videoPlayer);
vidWin.open();

Is it even possible to add events on fullscreen video or do I have to use some tricks to do it?!

    • Ok, I have realised what is happening, by adding the following code you can see when you enter and exit fullscreen by checking the object for the 'entering' field.

      videoPlayer.addEventListener('fullscreen', function (e) {
    Ti.API.info('fullscreen ::: ' + JSON.stringify(e));
});

      Basically what happens is when you click the button to go fullscreen the videoPlayer moves from app level to OS level meaning your click event isn't available.

      As soon as you click 'done' in the controls, you exit fullscreen and return to app level again allowing your click event to work again.
