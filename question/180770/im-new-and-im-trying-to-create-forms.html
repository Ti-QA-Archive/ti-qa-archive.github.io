    I could not stand by and let the poor excuse of an answer provided by Papia Chowdhury be your entry into the world of Titanium.

    Appcelerator Titanium is a great product, the problem you are seeing now is that it is a product that has evolved over many years and techniques have changed, mobile device requirements have evolved. Because of this when you look upo information and what videos = some will be aimed at much older versions of the platform. Pretty much all of the ones provided in the other example are too old for you to start reviewing now.

    For clarity Titanium has evolved from straight code (often called classic) to a better more manageable framework (Still Titanium) but separates all the layout, style, models, controllers and more. For the purposes of this answer do not worry about what they all mean - as you will start to understand things as you try this example code I have provided below - and where required given you the reason behind the use of each part.

    Objective; Create a simple app with a Window, a couple of TextFields and an Button. When the button is pressed we will have an alert window show use the results of whatever was typed into the fields. This is an example app that you can then build on using the documentation I will provide links to - this is the best way to do things, add something, play with it and repeat.

    • Create the Base App

      • From Studio, choose the File Menu > New > Mobile App Project
      • Select Alloy & Default Alloy Project (usually the default options)
      • Choose Next
      • Enter a name for the project (can be anything) "Test-Form" is a good example
      • Select "Use Default Location" - you can place apps in other places - but for now, here is fine
      • Enter "com.yourname.testform" in the "App Id" box
      • Leave the other choices as they are
      • Choose Finish
      • Wait a few seconds for the app to be built

    • Edit {appname}/app/views/index.xml

      It will look like this;

    <Alloy>
    <Window class="container">
        <Label id="label" onClick="doClick">Hello, World</Label>
    </Window>
</Alloy>

    Change it to look like this;

    <Alloy>
    <Window id="win">
        <ScrollView class="container">            
            <Label class="gap">First Name</Label>
            <TextField class="gap" id="firstname" value="" />
            <Label class="gap">Last Name</Label>
            <TextField class="gap" id="lastname" value="" />
            <Button class="gap" id="submit" title="Submit" />
        </ScrollView>
        <Label id="result" class="result">Waiting</Label>
    </Window>
</Alloy>

    • Edit {appname}/app/styles/index.tss

      It will look like this;

    ".container": {
    backgroundColor:"white"
}

"Label": {
    width: Ti.UI.SIZE,
    height: Ti.UI.SIZE,
    color: "#000"
}

"#label": {
    font: {
        fontSize: 12
    }
}

    Change it to look like this;

    "Window": {
    backgroundColor: "#fff",
    height: Ti.UI.FILL,
    width: Ti.UI.FILL
}
".container": {
    contentHeight: Ti.UI.SIZE,
    contentWidth: Ti.UI.FILL,
    layout: "vertical",
    left: 20,
    right: 20,
    top: 50
}
"TextField": {
    borderColor: "#000",
    borderRadius: 5,
    borderWidth: 1,
    height: 50,
    left: 0,
    paddingLeft: 10,
    paddingRight: 10,
    width: Ti.UI.FILL
}
"Label": {
    color: "#000",
    font: {
        fontSize: 12
    },
    height: Ti.UI.SIZE,
    left: 0,
    width: Ti.UI.SIZE
}
".gap": {
    top: 10
}
".result": {
    bottom: 50,
    textAlign: "center",
    width: Ti.UI.FILL
}

    • Edit {appname}/app/controllers/index.js

      It will look like this;

    function doClick(e) {
    alert($.label.text);
}

$.index.open();

    Change it to look like this;

    $.submit.addEventListener('click', function (e) {
    var fullname = $.firstname.getValue() + ' ' + $.lastname.getValue();
    $.result.setText(fullname);
    alert(fullname);
});
$.win.open();

    These are the documentation links that will help you understand these different windows, views, scroll views, labels, text fields and buttons.

    • Ti.UI.Button
    • Ti.UI.Label
    • Ti.UI.ScrollView
    • Ti.UI.TextField
    • Ti.UI.View
    • Ti.UI.Window

    In Summary;

    • View XML is used to define how things are laid out
    • Style TSS is used to define how the View is to be presented
    • Controller JS is used to interact with the user and any View elements

    Also;

    • The View defines IDs so that the Controller can access the properties or react to the user pressing, moving or other interactions.
    • The Style property layout, allows things to automatically be placed one after - worth reading about that property - see View or ScrollView for a good explanation.
    • The documentation is very good and has lots of examples. You will learn by adding one new thing at a time and seeing what it does and then playing with the properties.

    This will NOT teach you everything you need to know about creating a whole app - but it will give you a good starting point to explain how windows, scroll views, labels, text fields and buttons work.

    I hope this will help you.
