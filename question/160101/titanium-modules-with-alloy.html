Titanium Modules with Alloy?

Hi everyone,

I have a very basic question on understanding Titanium modules. My company is currently working on something that I would call an "App Suite". The "App Suite" is a whole bunch of apps, all of which implement different parts of a business process. Some of the components we need to implement will be used multiple times in the same app, some others are needed in more than one app or even throughout the whole suite (e.g. adding events to a calendar). We know that this could be done using Alloy Widgets. However, on the one hand, we consider some of these components to be worth being implemented as modules and distributed via the marketplace, on the other hand we think that Alloy Widgets bear the disadvantage of storing the same source code multiple times (correct me if I'm wrong as I haven't yet developed a widget). So we'd rather go for Titanium Modules (i.e. packaged JavaScript modules), but as we value the benefits of using Alloy and wouldn't want to develop half-classic, half-Alloy style, we would only do this is if Titanium Modules could be implemented using Alloy. Is this possible?

Thanks,
Thomas

    Hi

    Titanium JavaScript modules are built using Titanium not Alloy and you have to package up your module twice - if you plan on one for Android and one for iOS.

    Widgets are useful as they are re-usable - but as you say they cannot be compiled so less suitable for sale to others.

    The other thing to keep in mind is that a packaged module would not have access to the parent apps Alloy style sheet even using createStyle, addClass or removeClass.

    Compiled modules (native or JavaScript) are for those things you want to separate out of the main app and maintain separately from everything else.

    My suggestion is - focus on the benefits to your own project first and then look to see if there is any merit on building versions for sale.

      Very easy way to split up app development over a team as well.
