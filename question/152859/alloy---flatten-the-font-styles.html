Alloy - Flatten the font styles

Hi,

I'm having the same problem as this thread:

Is it possible to use a default font for all labels troughout the app? my solution with themes in alloy doesnt work

I'm trying to use icon font in my project:

tss file

".icon": {
    font: {
        fontFamily: "iconfont"
    }
}

".icon-circle": {
    text: "\ue000"
}

view xml

<Label class="icon icon-circle"></Label>

It worked perfectly. But when I was trying to scale up the "icon" (which happens quite often for icon font approach):

".icon-large": {
    font: {
        fontSize: "30dp"
    }
}

The icon didn't show anymore because the fontFamily property was lost.

Because all the font properties already have the "font" prefix anyway, is it possible to flatten the font styles, like the "background*" or the "border*" properties. For example:

".icon": {
    fontFamily: "iconfont"
}

".icon-large": {
    fontSize: "30dp"
}

So one single font style change won't clear all the base font styles any more.

Thanks.

  • Yiqing, yeah this problem sucks. On 3.1.3, it is still an issue. The only kind of workaround I found is to override the class/append the styling using the xml attributes. So if in your app if you have:

    in .tss
".bg-default":{
    backgroundColor: 'white'
}
...
in .xml
<View class="bg-default" backgroundColor="blue" />

    The attribute in the XML will override the .tss class but the solution is just a workaround. It kind of defeats the purpose of having .tss if you have to use the XML attributes to override.
