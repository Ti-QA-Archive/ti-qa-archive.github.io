Webview Video on Android - NOT PLAYING

Hello All,

We want to display a video in our app by opening it in a WebView on Android and iOs app (built using Titanium SDK 1.8.2).

We create a webView and set its 'url' property to -

http://player.vimeo.com/video/42065651?title=0&byline=0&portrait=0&color=ffffff&autoplay=1

The video page opens properly on iPhone and iPad.

But on ANDROID the VIDEO DOES NOT PLAY on this page inside the webview of the app.

Although the SAME URL (above mentioned) if we try it out ON THE DEFAULT ANDROID PHONE'S BROWSER , THE VIDEO PLAYS.

We searched the forum for all possible existing solutions but of no help.

On Android we tried Setting

pluginState : Titanium.UI.Android.WEBVIEW_PLUGINS_ON

and in tiapp.xml -

<uses-permission android:name="android.permission.INTERNET"/>

But nothing seems to cause the video to play in the webview control of the android app.

Also there is this error displayed on the studio's console -

D/dalvikvm(  398): GC_FOR_MALLOC freed 4727 objects / 357424 bytes in 174ms
I/TiWebChromeClient.console(  398): (main) [20158,20158] TypeError: Result of expression 'h' [null] is not an object. (47:http://a.vimeocdn.com/p/1.4.16/js/player.mobile.js)
I/TiWebChromeClient.console(  398): (main) [537,20695] TypeError: Result of expression 'h' [null] is not an object. (47:http://a.vimeocdn.com/p/1.4.16/js/player.mobile.js)
I/TiWebChromeClient.console(  398): (main) [2,20697] TypeError: Result of expression 'h' [null] is not an object. (47:http://a.vimeocdn.com/p/1.4.16/js/player.mobile.js)

But there are no issues on the default android phone's browser.

What is then the difference between the native default android browser and the titanium's android webview control?

Is there any other property that needs to be set or anyother changes that need to be done?

Thank You.
