Heavywindow vs lightweight window, when should we use one and when the other?

Hello everyone :)

I've read a lot about the two types of window in Android.
But, is one better suited than the other? In which situation?

A heavywindow is a new Android activity with a new javascript context. A lightweight window is 'just' a new container that takes the whole screen.

Still, I don't understand what all that really means for my app!?
I can work more or less the same way with one or another. Except that heavywindows have more properties or "gadgets" like the action bar.

According to the Android API Reference, an activity is "a single, focused thing that a user can do." I could say the same of a lightweight window too!

The only problem I've seen with lightweight window is when the app is closed. The user will probably not get back the app with the same state it had when he opens it again.

Regarding performance, is there a difference?
If iOS has only one kind of window there must be a real good reason that Android has two.

I was wondering if someone could clarify the use of light and heavy windows.

Thank you :)

    The Android System gives you some opportunities when working with Activities (heavyweight windows).

    • It creates a back stack of Activities. That means, when the user navigates through you App like:
      from Activity A to Activity B to Activity C and so on, he always can go back in his navigation history with the back button

    • If you start another App (I should better say an Activity of another App) via an Intent (e.g. for sharing Content or scanning a barcode), then the Activity Stack will bring the User back to your App's Activity through the back button

    • This also allows other Apps to start your App when you define Intent Filters for your Activities

    • You have control over the Activity Lifecycle of heavyweight windows by reacting to its events (start, stop, pause, resume, destroy, create)

    • When you create OptionMenus, then you have to use heavyweight windows since each menu belongs to an Activity

    That are some reasons for heavyweight windows that just came to my mind, probably there are more.

    • Yes, it really depends on your needs. In most cases you can just go with lightweight windows.

      Note, that there is always at least one activity available that you can use for saving states, optionsMenus and those things. This Activity is accessible from every lightweight window with win.getActivity().

      I don't think that the window type influences the memory. There are other techniques to improve memory usage when working with images. See the recommendations for example.
