    • Yeah, sorry I did not provide much context. I was throwing it out there quickly to see if anyone else had issues like this before.

      I actually just fixed it.

      My app is working on iOS and I just submitted it to the App Store, so now I am seeing if I can build for Android also. It turns out that I had changed the default name of the icon from appicon.png to Icon.png. This was causing a build error since the file was not found.

      For some reason, the build log is not displayed in the console for Android like it is for iOS. I just realized this and was able to find the error in the build.log file that you can find in the file hierarchy of the project.

      Thanks for trying to help!

      PS I am using Android 2.1 API 7, latest Titanium Studio and SDK 1.7.3 on Mac OS X Lion.
