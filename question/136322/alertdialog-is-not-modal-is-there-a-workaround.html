AlertDialog is not modal, is there a workaround?

The AlertDialog object is defined as modal, but (on Android anyway, API version 1.8.2) when you click outside the dialog box, the box disappears and returns a bogus result. Here's some simple code:

var alertDialog = Ti.UI.createAlertDialog( { 
    title:'show title', message:'what to do', 
    buttonNames: ['OK','Cancel'] , cancel: 1
} );
alertDialog.addEventListener( 'click', function( event ) 
{
    Ti.API.debug( 'cancel=' + event.cancel + ', index=' + event.index 
            + ', button=' + event.button + ', type=' + event.type );
});
alertDialog.show();

Which produces the following output for clicks on OK, Cancel, and outside the box:

cancel=false, index=0, button=true, type=click
cancel=false, index=1, button=true, type=click
cancel=false, index=-268435457, button=true, type=click

It seems like index always has this same value -268435457. Note that the cancel value is never correct, but this is minor. The big issue is that the behavior is not modal. Does anyone know of a fix? Is there a missing parameter in the create call?

Thanks.
