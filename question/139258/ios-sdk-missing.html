IOS SDK Missing

Hi all,

I just downloaded Titanium last night (ver. 2.1.0.201206251749). After installing, under the configure tab on the dashboard, it says IOS SDK missing. I had Xcode 4.3.2 installed on Lion 10.7.3. I followed the instructions to update xcode and now have 4.3.3. I am able to create a new IOS project and compile/run it in the simulator without any problems, however Titanium still can't find the SDK. I checked for the IOS SDK which I believe (according to apples documentation in xcode) is suppose to reside in the /developer/platform folder, but I can't find it there. However I did find it in the xcode.app/contents/developer/platforms.iphoneOS.platform/developer/SDK. Its called iphoneOS5.1.sdk

Do I have a problem with my xcode, or is the problem with Titanium? Any help would be appreciated.

Thanks for your help,

Adrian

  • Hi all,

    I just found my answer. I went to the terminal and pasted the following in and it now works:

    sudo xcode-select -switch /Applications/Xcode.app/Contents/Developer

    I did find this earlier on a post at: http://developer.appcelerator.com/blog/2012/03/titanium-and-xcode-4-3-revisited.html but thought it was out of date since it was dealing with Titanium 1.8. I figured it would have been fixed by now in 2.1. I guess I was wrong. Anyhow, I found someone who had a similar problem but with a different development kit at (http://www.mosync.com/content/mac-osx-installation-problems) and noticed they had recommended the same fix.

    It hasn't been a very smooth start I must say. I've downloaded GBs of different xcode versions and wasted almost an entire day for something that should have worked first time. I guess you live and learn. Lets now see how it performs…

    Regards,
    Adrian

  • Studio does support Xcode 4.3. However, if you have used xcode-select in the past, it will take the value of xcode-select first, assuming it points to a valid path.

    See TISTUD-1196 for a description of the search algorithm.
