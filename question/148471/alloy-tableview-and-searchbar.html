Alloy tableview and searchbar

Hi,

A strange problem appears when I want to set "search" attribute to tableview. This is my Alloy xml:

<SearchBar id="searchbar" height="43" />
<TableView id="societies" dataCollection="society" search="searchbar">
    <TableViewRow title="{name}" id="{alloy_id}" />
</TableView>

an Java error says that tableview cannot bind "string" to "view" or something like this (I don't have log right now, I can edit my post later)… and application crashes. If I remove search attribute, there is no error, no crash… any idea ?

I tried to map search in controller:

$.societies.search = $.searchbar;

But that does nothing while I'm typing some text in searchbar…

Alloy 0.3.6, titianium GA 3.0.0 (Android)

  • Ok found a solution…

    I did this:

    var society = Alloy.Collections.instance('society');
society.fetch();
$.societies.search = $.searchbar

    But that cannot work… because tableview is filled before "search" attribute is set. I only had to switch my lines:

    var society = Alloy.Collections.instance('society');
$.societies.search = $.searchbar
society.fetch();

    And completion is ok. The only "bug" is that I cannot set "search" attribute in Alloy XML view…
