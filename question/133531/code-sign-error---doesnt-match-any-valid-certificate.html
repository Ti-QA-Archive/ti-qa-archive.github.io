    • Just FYI for anyone that runs into this problem, my certs were valid as I expected. I ended up compiling for distribution inside of xCode. Ti was trying to use the development cert for distribution. So the first thing that needed to change was the build settings for the project, change both of the release targets to the correct distribution certs. I also noticed it messed up my app icons, so I had to add those files into my resources folder and then add those to the info.plist file. Then under each project target (iphone, ipad, universal) make sure your new icon files are listed… (I named my icons, iphone-icon.png and ipad-icon.png just in case you were wondering). Next make sure you have iOS device selected from the schema. Once that is ready select Archive from the Product menu at the top. When that completes go to the organizer (if no errors occur), select the Archives tab (I think this happened automatically, I can't really remember I was too excited I got that far), and select Validate.

      Type in your iTunesConnect info, and pray to the gods (well at least that Apple won't reject you). If it validates submit it and you're just in a waiting game. If you get the error that no valid app was found on iTunesConnect, make sure you have set up you app on there, and the biggest kicker, make sure you click the "Ready to Upload Binary" button.
