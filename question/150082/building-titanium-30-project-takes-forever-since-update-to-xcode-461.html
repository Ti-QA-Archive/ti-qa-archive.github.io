Building Titanium 3.0 project takes forever since update to xcode 4.6.1

Hi!
For a week now my Titanium workflow has been severely slowed down and I have absolutely no idea what to do…

It all started when MaxOSX 10.8.3 update came out and a small Xcode 4.6.1 along with it. I haven't touched Titanium Studio nor any of the SDKs. I'm using latest 3.0.2.GA.

Also by mistake I upgraded node to v 0.10.0, but after reading blog post I downgraded to 0.8.22.

I'm still able to build.. but average build time now about 5 minutes(!). Sometime less (when it's pre-build). But as soon as I clean the project … it takes forever. Basically I'm unable to work as before. When I start building it goes up to the point where it says:

"Invoking xcodebuild"

After that I can go for a walk and have a coffee. Come back and it would still be building.

Also I noticed when building for device or distribution I have to wait additional 5 minutes until it pulls the dialog to select certificates.. And after long process of build it takes even more time to "copy itunes artwork" until the app actually appears in iTunes.

I thought Xcode was to blame, but after trying to build a native project - it's lighting fast I think it's something else.

Has anyone experienced this? I could really use some help. Maybe there is a way I can debug this? Check some logs? Clear some cache?

So far I tried:

  • Deleting / reinstalling Titanium Studio
  • Deleting titanium related folders in ~/Library
  • Deleting and reinstalling XCode from MAS
  • npm install titanium -g

I'm all out of ideas. Any help is greatly appreciated.

Thanks in advance.

    • Mauro,

      nvm, I was able to resolve the issue myself.
      Basically what I did was removed the following folders from home directory ~/

      .npm
      .titanium
      .eclipse

      Also I reinstalled titanium + alloy from command line:

      npm install titanium -g
npm install alloy -g

      All works fine now. Alloy even appeared in Titanium Studio on project creating (which wasn't the case before)

      I'm going to mark the answer as valid so others with similar issues are able to locate the answer more easily.
