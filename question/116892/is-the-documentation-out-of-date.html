    I wasn't even aware layout could be used on a tableviewrow. I thought it was only valid on a window or view. I suspect it would work the same on table row though – it is just another type of view anyway.

    When you use layout:'absolute' (the default), all your top/bottom position values are relative to the top or bottom of the size of the view/window. This lets you place objects side by side by repeating the same top/bottom values on multiple objects.

    When you use layout:'vertical', then each object's top value is relative to the object above it. So using top:0 on everything just stacks the objects with no additional padding between them, while top:5 would insert an extra 5 pixels of padding, etc. Note that you cannot place controls side by side while using layout:'vertical'.

    In order to place controls side by side in a vertical view, you must create another view which is layout:'absolute', place the controls in it, and add that view into the vertical layout view. Since view can be nested, that works out fine.

    I saw some references to layout:'horizontal' support on the roadmap, but AFAIK it is not available yet.

    Unlike most properties, the layout property MUST be specified when the view or window is created. You CANNOT change the layout property later. For example, Ti.UI.createView({layout:'vertical'}) is effective while myView.layout = 'vertical' is not.

    The subject question was if the docs are out of date. In my experience, they have always been incomplete. I have stumbled on many properties used in the kitchen sink which are not listed in the API docs for that object. And sometimes Q&A answers suggest a solution which is not in the docs either.

    I'm hoping that with the additional staffing in 2011, Appcelerator can devote some of it improving the documentation. In the meantime, I suggest looking at the kitchen sink examples, and doing Q&A searches related to any specific problem you are having. You may well learn of an undocumented technique.
