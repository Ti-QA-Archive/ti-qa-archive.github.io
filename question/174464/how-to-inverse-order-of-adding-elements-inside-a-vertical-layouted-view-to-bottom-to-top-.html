How to inverse order of adding elements inside a vertical layouted view to bottom-to-top ?

I've a ScrollView written on an xml layout which initially contains some views inside of it. The ScrollView has its layout property set to vertical :

<ScrollView id="ScrollView" layout="vertical">
    <View  id="view1" width="100%" height="37.5%" / >
    <View  id="view2" width="100%" height="37.5%" / >
    <View  id="view3" width="100%" height="37.5%" / >
    <View  id="view4" width="100%" height="37.5%" / >
</ScrollView>

Now i want to add other views by a dynamic way :

$.ScrollView.add(Ti.UI.createView({
    width:"100%",
    height:"37.5%",
    backgroundColor:'blue',
    top:0
}));

$.ScrollView.add(Ti.UI.createView({
    width:"100%",
    height:"37.5%",
    backgroundColor:'green',
    top:0
}));

$.ScrollView.add(Ti.UI.createView({
    width:"100%",
    height:"37.5%",
    backgroundColor:'red',
    top:0
}));

and i want that these new added three views will be above first other views added initially (on the xml) by setting the top property to 0 for all of them but that doesn't work and new added views always lay below initially added views.

Is there a way to inverse order of adding these view from bottom-to-top instead of the default way ?
