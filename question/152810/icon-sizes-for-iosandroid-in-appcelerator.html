Icon sizes for iOS/Android in Appcelerator

When we render icons in an Appcelerator app for Android or iOS, will the sizes be same as recommended by the native platform?

For e.g. the recommended size of icons are:

iOS Retina: 80x80 px, Normal: 40x40px
Android: hdpi: 72x72 px, xhdpi: 96x96 px

But I see for iOS, if we use the recommended size on the top navigation bar, the image looks too big! I had to then scale the image to a size so that it fits the navigation bar.

Is there something I am missing?

    Where do these recommended icon-sizes come from?

    Have a look into the offical guides and use the dimensions they mention there:

    iOS: http://developer.apple.com/library/ios/#documentation/userexperience/conceptual/mobilehig/IconsImages/IconsImages.html

    Android: http://developer.android.com/design/style/iconography.html

    e.g. NavigationBar icons for iOS are approximately 20 x 20 px (retina: approximately 40 x 40 px)
    and Android ActionBar icons are 32 dp x 32 dp

    hope this helps
