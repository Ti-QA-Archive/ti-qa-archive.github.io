Windows chained together

Titanium SDK 3.0.2, iOS 6

So I have my main window. I added it into a hidden tabgroup to make the titlebar appear. The app contains of three windows:
homewindow -> listwindow -> detailwindow

They should all keep the titlebar, so I tried to make this first with the hidden tabgroup hack. Doesn't seem to be particularly nice, but works.
To make it nicer I made listwindow and detailwindow a modal window. That works fine but I then get the warning that opening an other window from within a modal window is not good, but it seems to wrok.

I'm wondering what the correct way would be to handle such a situation.

    Hi

    As you have not provided any code this is a guess, but it sounds like you have not used tabGroup as part of your interface (I know you say you have used a hidden one as a hack) or a navigationGroup.

    You must use on of these two to "gain" the title bar you were expecting.

    It sounds like you should be using a navigationGroup to solve your problem it you do not have a need for the tabGroup.

    Follow the example code in the documentation;

    • Titanium.UI.iPhone.NavigationGroup

    You will note that the windows are opened as children of the main navigationGroup.

    And so you do not get confused a navigationGroup is added to a "main" window and then your actual windows are added into that. It may seem weird but it will make sense after a while.
