select childobject of view by index in nested views

i use the following code quite often, but i wonder if there is a better way.

When i create a view i always add the follwing custom function

var everyview=Ti.UI.createView({
    selectChild:function(index){
        return this.children[index];
    }});

this because i very often, need to manipulate a child view from code.

for examle i want to change the border of an imageview inside a view inside a scrollableview.

normally in javascript i would write:

scrollableview.children[2].children[4].setBorderColor('Red');

but this doesnt work in titanium for some reason,

with my custom function i can write:

scrollableview.getChild(2).getChild(4).setBorderColor('Red');

but then i have to remember to put in the custom function with every create. this messes up my codes readability, and i think the titanium guys would be smart enough to have built this in the core.
So i think i am overlooking something or doing something fundamentally wrong.

is there a better solution?
