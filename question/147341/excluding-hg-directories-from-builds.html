Excluding .hg directories from builds

I've recently started using Mercurial's subrepos feature to share common code between projects, including the subrepo in the resources directory, and I've noted that Titanium Mobile's hoovering up the contents of the subrepo's .hg directory.

Building for iOS produces an IPA containing the contents of the subrepo's .hg directory.
Building for Android the toolchain just falls over in a huff.

Is there a particular reason why the SDKs (2.1.4.GA & 3.0.0.GA) are not a good UNIX citizen and ignore the .hg directory as it should?
