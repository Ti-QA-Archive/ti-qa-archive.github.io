checking a server for new data, and showing a notification when new data is found?

Hi

I've been looking into appcelerator for a project. The client wants a iPhone and android app. And the app itself is fairly straightforward. He wants a notification on new data.

For android I can use appcelerator's service, running on interval, to poll for new data. For iPhone I can presumably use urban airship for this. However I've seen a lot of examples, but I have no idea which edition these examples use.

Will I be able to do this:
http://developer.appcelerator.com/question/121904/registerforpushnotifications-doing-nothing
or this:
http://urbanairship.com/blog/2010/05/26/appcelerator-and-urban-airship/
or this:
http://developer.appcelerator.com/blog/2010/05/push-notifications-with-titanium-and-urban-airship.html

In the community edition? Or are there any alternatives for me, wanting to do this very simple operation. Without paying a subscription fee for it?

  • I found the public (non subscription based) urban airship code examples work great. If you just need to do push notifications, urban airship is free for up to 1 million pushes per month, and the code we use is directly from the blog post you mentioned, found here:
    http://urbanairship.com/blog/2010/05/26/appcelerator-and-urban-airship/

    On the server side, we have a cron job that monitors for new data, and sends a push using urbanairship.com when new data come in.

    Hope that helps!
