Anddroid App crashing when installed through market app

This is more for information and for asking if anyone else has encountered this problem..

We had submitted a beta version of our app in one Android market (SmartAppFinder). The app works fine on its own, but when downloaded through the Android application of that store the app crashes.

We found that the problem is, that the market application would copy the downloaded APK to the SD card giving it the name of the application id (the package name).

Now Titanium always creates a folder on the SD card which also has the name of the application id. Because now a file with the name already existed Titanium was not able to create it's folder which our application requires to store sounds and images and the application now crashes when it tries to access files that should exists in this place.

Has anyone encountered this problem with any store or otherwise? Does anyone know how to possibly get around this problem? I don't seem to have influence on the name of the Titanium folder, now does it seem possible to write to a different custom place on the SD card..
