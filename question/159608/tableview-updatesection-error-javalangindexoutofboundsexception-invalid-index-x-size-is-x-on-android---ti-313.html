productTableView.updateSection(parseInt(productTableView.data.length-1), ingredient_section);

This works fine on iOS but not on Android. Tried on a few devices (Nexus 4, Galaxy S3) and a few versions of Android (4.3, 4.1.2). The number of rows in the section doesn't seem to affect the bug. Going from 2 rows to 9 rows to 4 rows would cause the bug as well as going from 2 to 4 to 9.

But it SEEMS (not quite sure) like the number of time updateSection is called could be the source of the problem. I do have some product view that works all the time and other that crash systematically.

Some time going back and forth between two sections work all the time but calling the third one crashes the app.

This is a real issue as I don't see how to replicate the same feature other than using updateSection…

I've tried Ti SDK 3.1.3, 3.1.2, 3.1.4 (beta), same behavior.

Any ideas?

  • I can't quite figure out what is the source of the issue but I did figure out a fix: recompile the Titanium SDK after editing: android/modules/ui/src/java/ti/modules/titanium/ui/widget/tableview/TiTableView.java:line 138

    public Object getItem(int position) {
    if (position >= index.size() || index.get(position) >= viewModel.getViewModel().size()){ 
        return null;
    }
    return viewModel.getViewModel().get(index.get(position));
}
