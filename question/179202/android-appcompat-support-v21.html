Android appcompat support v21

I'm able to set my Window's "theme" property to inherit from "android:Theme.Material.Light" which allows me to use the colorPrimary and colorPrimaryDark xml theme options etc (and looks great)

<style name="Theme.My.Custom" parent="android:Theme.Material.Light">
    <item name="android:colorPrimary">@color/themeblue</item>
    <item name="android:colorPrimaryDark">@color/themebluedark</item>
</style>

However inheriting the theme from AppCompat can apparently achieve the same results when using the support library v21 (which currently isn't working for me). For example people compilng with gradle in Android Studio are doing this by adding the following the their build files…

dependencies {
    compile "com.android.support:appcompat-v7:21.0.+"
}

Am I able to tweak a titanium compile command, or manifest variable to ensure the appcompat library is using the v21 libraries such as above? Or is this something that's more baked into the titanium sdk and needs to be updated internally at some point. Ultimately I'm hoping to use as much material design themed elements as I can whilst inheriting from AppCompat.

Appreciate any advice :)

Specs..
Application type: Mobile
Titanium SDK: 3.4.1
Platform: Android: 5.0
Device: Nexus 5
OS: Windows 8.1
Ti Studio: 3.4.1
