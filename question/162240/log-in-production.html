Log in production

Does the compiler remove the code about Ti.API.info() (or trace, …) when compile for production?
Is there a way to physically prevent to have "not production" code in the production binary?

Thank you all

    There was a question here last week on this same topic. Please search for that for all the specifics. The gist of it was that no, logging output is not stripped from your production builds.

    A rather simplistic solution (with Alloy) is a custom logging function like this:

    function log(msg) {
  if(!ENV_PROD) {
    Ti.API.info(msg);
  } else {
    return;
}
