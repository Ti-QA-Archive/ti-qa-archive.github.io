iOS screen resolution/density relationship

Can someone explain what is the relationship between screen resolution and density?
I understand that with retina display the density is set as 'high' however the resolution remain the same, how will this impact the way I write code that fills the view with a list of screen given by the screen height dynamically?

For example using basic iphone simulator ( 320 x 480 ), I was able to fit 20 items on the screen. But when I use iphone 3.5inch with retina display (320 x 480) i still see 20 items in the simulator.
Q1 - Is this what it is expected in retina display?
Q2 - Following Q1, will I see a difference between simulator and actual device for retina display?
Q3 - if not, with retina display activated, does this mean that I have to manually consider the screen height to double ie 480x2 to achieve 960 based on the density and then code it in such a way that it display 40 items instead?

thanks

    A1 - Yes, this is what is expected as the density is taken into consideration for retina displays. Although if you provide high res. imagery (which are twice the size of your originals and placed in the same location) and suffix them with @2x then these will be used instead in order to take advantage of the retina display. e.g. image.png becomes image@2x.png

    A2 - There should be no difference between simulator and device.

    A3 - Bearing in mind what I have already answered, if your intention is to see 40 items then yes you will have to take that into account when coding, but otherwise no additional consideration should be needed to code between standard and retina.

    Hope this helps.
