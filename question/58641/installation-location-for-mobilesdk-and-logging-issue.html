Installation location for mobilesdk and logging issue

In trying to resolve the logging issue (no logs show up) by following http://developer.appcelerator.com/blog/2010/07/introducing-titanium-mobile-1-4.html where it says to install the patch (a replacement logger.py) it indicates the file needs to go in 

/Library/Application Support/Titanium/mobilesdk/osx/1.4.0/iphone

Interestingly enough, on my machine, I see the following folders:

/Library/Application Support/Titanium/mobilesdk/osx/1.2.0/iphone
/Library/Application Support/Titanium/mobilesdk/osx/1.3.0/iphone

but 1.4 is installed as 

/Users/patakijv/Library/Application Support/Titanium/mobilesdk/osx/1.4.0/iphone

(the difference being 1.4 is under my user dir structure)

This install location for 1.4 is not only different from 1.2 and 1.3 but it is different from the instructions on the page about the patch.

Does anyone know how this installation location might have changed? I don't remember specifying anything different ever. The only thing that would have been different is that I installed 1.4 after upgrading to Snow Leopard.

What is the preferred location for all of this and where is it set so that all new sdk installations go to the correct place? Can I or should I attempt to move things around?

One reason I want to resolve this is because it appears that even though the log says I am using 1.4 SDK, the patch has not resolved the logging issue. Nothing I attempt to output to the log shows up in the Ti Developer.

Thanks for any insight.
