Going live with iPhone In App Purchases

Hi Everyone,

I've implemented the Store module into my app, I've completed all my in app purchasing testing using an iTunes test account, in an attempt to make sure I've done everything correctly I tried using a live iTunes account to perform a real world purchase but that isn't allowed - a message gets thrown back stating 'This is not a Test User account'. So my question is, once Apple has approved my app and it's in app purchases - will Apple automatically not use the Sandbox environment if the in-app purchases' 'Status' has been approved? Or is there some configuration setting I need to do my side with the store module to indicate to Apple that I'm going live, don't use the sandbox environment?

Many thanks.

    hello Paul,
    you don't have to do nothing.
    Once your iap are on itunes they will be immediately available. ( if you have then active and ready for sale )

    The difference between sandbox and production is only the username you use to connecto to iTunes Store.
    If you use your testing account to log in ( the one created in iTunes Connect - Manage Users ) you will entering in sandbox.

    Hope it helps.
