[Alloy] How to internationalize TabbedBar Labels?

How can I internationalize a Titanium.UI.iOS.TabbedBar?

I tried using the titleid property, but this doesn't work for Labels (BarItemType).

<TabbedBar id="interactiveBar" platform="ios" onClick="setView" >
    <Labels id="labels">
        <Label titleid="gallery" />
        <Label titleid="wiki" />
        <Label titleid="fb" />
    </Labels>
</TabbedBar>

My next attempt was giving them labels id's and set the title in the controller like:

_.each($.labels, function(_label){
    _label.title = L(_label.id);
});

but this also turns to an error because empty labels are not allowed.

How can I solve this? Thanks in advance for your advices!

    You can do this in a TSS file.
    Consider the following.

    Blah.xml:

<TabbedBar id="interactiveBar" platform="ios" onClick="setView" >
    <Labels id="labels">
        <Label id="gallery" />
        <Label id="wiki" />
        <Label id="fb" />
    </Labels>
</TabbedBar>



Blah.tss:
".gallery": {
    title: L('gallery')
},
".wiki": {
    title: L('wiki')
},
".fb": {
    title: L('fb')

}

    I haven't ran this code, but I have no reason to believe it is wrong. The take away is do that stuff in the tss.

    • Just to help others trying to do this. I could not make it work with "title" (I am on Titanium Studio 3.1.3 on Mac). I had to use "text" instead:

      "#gallery": {
    text: L('gallery')
},
"#wiki": {
    text: L('wiki')
},
"#fb": {
    text: L('fb')
}

      On my wishlist would be to be able to specify it directly on the label e.g. like this:

              <Label textid="wiki" />

      That would be a neat (and intuitively good) way of binding the translated texts into the xml ;-)
