Can't build project when Resource files are read-only (in Windows)

My source-control program marks all files in my project as read-only (at the file-system level…Windows file attributes) when I don't have them checked out for editting. When I try to build/run my app on the emulator, the Application Installer console window gives me this error:

[ERROR] IOError: [Errno 13] Permission denied: 'C:\Titanium\Workspace\Proj1\build\android\bin\assets\Resources\Default.png'

That file comes from C:\Titanium\Workspace\Proj1\android\Default.png, and it is marked read-only. I cannot check out all files every time I need to build the app.

Is there a way to tell Titanium to allow read-only on these files, or is there a way to tell it to remove the read-only attribute while copying to the build directory?

Titanium Studio 1.0.1.201106171427<br>
Titanium SDK 1.7.1<br>
Andriod 2.2 API

    • Spoke too soon…same problem…it hasn't gone away. If any of my image resources is read-only, I get the I/O error 13. We really need the copy process that occurs when building to remove read-only from the copies (under the build directories), or to not try to open them with write-access during the build process.

      — commented July 18th 2011 by Shawn Lipscomb

    • Running Titanium Studio under Windows, the build process calls the copy function in shutil.py (under plugins\com.appcelerator.titanium.python.win32_1.0.0.1312318466\python\Lib\shutil.py) to copy files into the build directory. That copy function calls copymode after copying the file. copymode (in the same file) calls os.chmod which, under Windows, can only copy the read-only attribute of a file. This is what's causing the problem. Since there's no need for the read-only attribute on any file in the build directory, this should be removed.

      I have reported this as bug TC-264.

      — commented September 27th 2011 by Shawn Lipscomb

    • And it has now been elevated to TIMOB-6164.
