Images do not show up in Android build

[Using Titanitum 3.1.3GA, up to date TI Studio on an uptodate Mac. Targeting to iOS and Android, iOS works well.]

I have an Alloy app that runs fine on iOS. I'm not porting it to Android and, in general it runs. But none of my image resources will display.

I have not setup 'high', 'med', 'low' folders. (I've seen references to them in this forum, but no actual Ti docs.)
Following the Alloy convention, all my images are .png's in:
app -> assets -> images [-> subfolder].

All paths have a leading '/' ("/images")

For example:

"#choiceContentView" : {
    top: "125dp",
    width: Ti.UI.FILL,
    height: 250,
    backgroundImage: '/images/find_overlay_paper.png', 
    backgroundRepeat: true
}

If I look at the android/build folder I see:
Android -> bin -> assets -> Resources -> Images
which contains my images.

But the images never show up and there are no errors on the console output.

If I put in a breakpoint and:

var testFile = Ti.Filesystem.getFile(Ti.Filesystem.resourcesDirectory);

And look at testFile.directoryListing()
I don't see images or any other folder.

It looks like Alloy (or TI) isn't copying over the resources to the correct place for Android?
I've tried this in both the emulator and on a G4.

What am I missing?

thx
