    Why don't you hide ActivityIndicator and show message in "setTimeout" function? And of course, remove "setTimeout" when long lasting action is done.

    So, different logic would be:

    var setTimeoutId = setTimeout(function() { hideActivityIndicator(); showMessage(); }, 1000);
showActivityIndicator();
var result = longLastingAction();
clearTimeout(setTimoutId);

    And yeah, I think "HTTPClient" have its own error handling mechanisms, you should be able to define timeout and use "onerror" (see in docs).
