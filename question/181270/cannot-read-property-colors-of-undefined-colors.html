  • Hi

    An error message is not helpful without the code it refers to - something in your code would carry something like;

    // this will be in the middle of a line of code
.colors &&
// this would be near it
typeof

    It is unlikely that Alloy is at fault as this would not have tweaked your code or object/property references. The down ground is more likely to not trip an error that still actually exists - you were not lucky enough to find it. Or coincidently the code when run using 1.5 did not receive the data in the same way.

    Whenever you get an error that is essentially *Cannot read property something of undefined something else then is would mean the left side is the issue. So if the thing is colors that is undefined the object you think it is related to is either wrong or has been redefined in the code.
