Emulator crashes with Animation with CURL_UP transition

I am creating an animation using default transition.
iPhone simulator crashes as soon as this code is executed.

var animation = Ti.UI.createAnimation({transition : Ti.UI.iPhone.AnimationStyle.CURL_UP});
animation.addEventListener('complete', function() {
Ti.API.info('Animation completed');
// Perform some action
});
view.animate(animation);

Is it invalid to create an animation using default transitions.
Any help would be appreciated.

    hi Prashanth,

    this should be like this..

    win.animate({
    view:view,
    transition:Titanium.UI.iPhone.AnimationStyle.CURL_UP
});
