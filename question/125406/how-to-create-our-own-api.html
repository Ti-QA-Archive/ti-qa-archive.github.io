  • It depends on what you mean by your own API in javascript. If you mean wrapping the existing Titanium SDK capabilities into your own components, then see Building Reusable Factories.

    If you mean extending the capabilities of Titanium by creating APIs to perform things not yet possible in the Titanium mobile SDK, then those cannot be coded in javascript. They must be coded for each target OS in the native language and toolset for that OS (e.g. objective-C for iOS). These are referred to as (custom) "modules" in the Titanium vernacular.
