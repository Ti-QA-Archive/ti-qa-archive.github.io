Does Titanium wrap modules (files) in an IIFE or is it global?

I was looking through some code and noticed that many of the CommonJS modules started declaring local variables in the file without being wrapped in a function to scope everything from the global space. Most systems (like CoffeeScript) automatically wrap a module file in an IIFE.

Does titanium also wrap all module files in a wrapper to prevent global namespace clobbering?

for example does the folowing:

var myFn, myVar = "foobar";
myFn = function() {
  return myVar;
};
exports.myFn = myFn;

convert to when compiled:

(function() {
  var myFn, myVar = "foobar";
  myFn = function() {
    return myVar;
  };
  exports.myFn = myFn;
})();
