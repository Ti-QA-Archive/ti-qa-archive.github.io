Android Support?

Hey,

I used Titanium recently to build an iPad app and was generally happy with it. I had some issues but nothing I couldn't circumvent using various workarounds. Besides that the app went live and the client was very happy with it.

I've been asked to evaluate doing another app this time it would be for iPhone and Android. I'm pretty sure my experience with iPad will be the same with the iPhone so no worries there. Instead Android is more the unknown.

I've read several tweets and posts mentioning that Titanium's Android support is quite buggy and not on par with its iOS support. The app itself wouldn't be too complicated, tabgroups, navigationgroups, geolocation/maps and content stored in an sqlite db, with a process to update the db when online.

So just looking for some opinions? Is it worth taking the plunge? Are there features I should avoid or are quite buggy? Pitfalls to be avoided? Also what versions of Android does Titanium support as we may have to support all the way down to 1.6.

Thanks,
Pat

  • Hi,

    I've been working on Android for a long time. One of the larger problems I face with using Titanium is that the tableView UI for android is clearly broken. If you're thinking of using a simple table with simple rows, it works fine. But try anything else fancy and the table view stutters and jerks while scrolling.

    So if your app is going to be table view intensive with lots of graphics and rich media, I suggest that you either use a scroll view (and end up having to rewrite all the table view functionality like what I did), or give up on Titanium.

    Anyone else has any other opinions?
