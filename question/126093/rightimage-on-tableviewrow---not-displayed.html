rightImage on TableViewRow - not displayed

I'm having some trouble getting a rightImage to show at all on a TableViewRow. I've tried various images in case the image I'm using has any "issues" but nothing is showing. Here's the code used to build the row from db results:

In my db.js module algorithms() function:

results.push({
                title: resultSet.fieldByName('label'),
                color:'#FFF',
                id: resultSet.fieldByName('id'),
                leftImage:resultSet.fieldByName('image'),
                //hasChild:true,
                rightImage:'../images/arrow.png',
                backgroundColor:'transparent',
                backgroundImage:'../images/list_bg.png',
                selectedBackgroundImage:'../images/list_bg_active.png',
                child: resultSet.fieldByName('child'),
                next_id: resultSet.fieldByName('next_id')
            });

The results array is then returned to my window code…

… then, back in my window code:

data = db.algorithms();

data is passed into the TableView and all is displayed fine, except the rightImage.

Switching hasChild on and off also has no effect. The backgrounds are working, as are leftImages (on iOS, not on Android, but I assume this is something else).

Any ideas?

  • OK, problem solved - although it seems to highlight some sort of inconsistency in the way the images are assigned to the TableViewRow in this case (and potentially in other cases).

    If anybody else has this issue, here's a way to avoid hours of pain:

    Changes to my code from it's initial state as posted in the question:

    At the top of my db.js module code:

    var path = Titanium.Filesystem.resourcesDirectory;

    In the algorithms() function:

    results.push({
                title: resultSet.fieldByName('label'),
                color:'#FFF',
                id: resultSet.fieldByName('id'),
                leftImage:resultSet.fieldByName('image'),
                rightImage:path + 'images/arrow.png',
                backgroundColor:'transparent',
                backgroundImage:'../images/list_bg.png',
                selectedBackgroundImage:'../images/list_bg_active.png',
                child: resultSet.fieldByName('child'),
                next_id: resultSet.fieldByName('next_id')
            });

    Note that I needed to prepend the &#39;images&#x2F;arrow.png&#39; with the resources directory path path variable EVEN THOUGH all images referred to in this script are present in the same directory. Go figure, as they say. Nothing else has been changed and this works, inexplicably.

    No doubt there's a reason and maybe somebody with knowledge of the compiler can explain why this is so. To me it's totally inconsistent and I seriously hope it doesn't change in a future release of Titanium Studio.

    PS: Also works on android emulator.
