Problem with tableView child windows in Modal Window

Hi All,

I have a pretty straightforward app with a 3 tab tabset. At one point I'm opening a modal window in front of that tabset, and trying to display a set of categories as rows in a tableView. Each of those rows hasChild, and they dispatch their click events properly. However, I can not go deeper into the child windows.

If i create the window NOT as a modal, it seems to work fine, however I cannot quickly close the modal window as it has been set to one of my main tabset views..

Any ideas? I could post code, but I'm using the parasitic inheritance model.. and the code snippets very modular and are a bit useless without the whole thing..

Is this just a limitation of modal windows?

(I'm having a similar problem with showing/hiding pickers in modal windows that has me suspicious)

Thanks!

  • This would not be a limitation of modal window, this is pretty much the definition of a modal window. If you need to be able to drill down into more windows, you should try either using a plain fullscreen window or maybe a navigation group depending on the controls you need to display on subsequent pages.
