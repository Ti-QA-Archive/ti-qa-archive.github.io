ondatastream.event only fires once and is always 1?

I have a progressBar with min=1 and max=1. During my call to the server, I want to show a loading bar to the user. So I tried this:

xhr.ondatastream = function(event) {
        Titanium.API.info(event.progress);
        Ti.App.fireEvent('app:myprogressBar', {data: event.progress});
    };


///in my main app:

var myprogressBar = Ti.UI.createProgressBar({
    min: 0, 
    max: 1, 
    value: 0, 
    message: "sending...", 
    width: 200, 
    color: "#fff", 
    top:'60%',
    bottom: '40%',
    shadowColor: "#64696e", 
    shadowOffset: {x:0, y:-1}, 
    font: {fontSize: 14, 
    fontWeight: "bold"}});

Ti.App.addEventListener('app:myprogressbar', function(obj){
           progressBar.value = obj.data;
});

But for some reason, ondatastream just fires once and event.progress is 1.
According to api, it should fired several times during the process and return values
between 0.00 and 1.00. What is wrong with my code here?

  • Hey Marcel,

    I notice your message for the progress bar is "sending". Is this progress bar for something you are uploading or downloading? If you're uploading you want to be using onsendstream, not ondatastream.

    Also, I would suggest trying your code on an actual device. I've had problems with the simulator only firing once from the simulator but working properly on an actual iphone.
