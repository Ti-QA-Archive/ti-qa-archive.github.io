createTableView does not fill table with empty rows since SDK 1.7

Quite new to this but previously my stopwatch type app launched with an empty table on the bottom half of the screen, createTableView automatically filled this with 5 empty rows (gap on screen for table). I then store times in this table from the stopwatch.

Anyway since SDK 1.7 the table has no empty rows on launch.

Interestingly if I add 5 times to the table (using stopwatch) and then reset the contents it shows 5 empty rows, as I'd expect on initial launch/creation.

// draw table code

var historyTable = Ti.UI.createTableView({
top: 30,
left: 20,
right: 20,
height: 220,
borderWidth: 1,
borderRadius:5,
borderColor:'#111'
});

// reset table code

historyTable.data = [];

Any ideas, is this an intentional change?

If I add 5 empty rows to the historyTable it behaves as it previously did.
