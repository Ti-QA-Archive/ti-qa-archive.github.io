How to set the Android SDK to 4.0

  • Out of curiosity, why do you want to increase the SDK level required by the app? That's just going to reduce the number of devices that can run your app. Typically, you would do this if you were going to take advantage of some API features that are only available in a higher SDK. But if your app runs fine with a minimum sdk of 8, why not leave it?

    At any rate, to answer your question, you can do this in the tiapp.xml file. Theres a manifest element that lets you control some of the android manifest file generation.

    I've truncated a lot of irrelevant and app-specific stuff from this example, so I hope this makes sense. I just wanted to provide enough of the elements so that you have some context to know where to put things.

    <ti:app xmlns:ti="http://ti.appcelerator.org">
    <android...>
        <manifest ...>
            <!-- android 4.0 and later -->
            <uses-sdk android:minSdkVersion="14"/>
        </manifest>
    </android>
</ti:app>
