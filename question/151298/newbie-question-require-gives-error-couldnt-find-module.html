Newbie question: require() gives error "couldn't find module"

Super basic question – still on my first day of Titanium – but is there some trick to ensuring a JS file is included and available? Here's my code:

var foo = require('test.js');

and later

var win2 = Titanium.UI.createWindow({  
    title:'Brokers',
    backgroundColor:'#fff',
    url: 'test.js'
});

test.js is at the root of Resources and contains nothing more than:


alert('here');

I get "[ERROR] : Script Error Couldn't find module: test.js".
What am I doing wrong?

  • Hi

    Correct syntax and location of file;

    var foo = require('/test');
    • {app name}/resources/test.js

    If you wish to locate the CommonJS file in a different folder then the syntax and location would change to this;

    var foo = require('/addons/test');
    • {app name}/resources/addons/test.js

    You should use the / prefix and you should ignore the file extension.

    When you have spent a little time coding you can take advantage of a great feature with CommonJS - platform dependant modules;

    Imagine you are creating an app that will work on Android and iPhone, but they have different requirements that the other does not share, but the code is essentially the same then you can create two versions, one for Android and one for iOS, the compiler then only installs the correct one for the platform. This way your code is efficient. This is NOT required, but pro apps tend to use this technique when they get bigger.

    Example;

    var foo = require('/addons/test');
    • {app name}/resources/iphone/addons/test.js (only run on ios devices)
    • {app name}/resources/android/addons/test.js (only tun on android devices)

    Have fun.
