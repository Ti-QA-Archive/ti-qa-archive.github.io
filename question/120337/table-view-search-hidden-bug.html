table view search hidden bug

hi,

This is the code that i used for creating a table view with search bar hidden, but the search bar will be made visible upon clicking a button that i plan to add at a later point of time…

my issue is that i have specified 'searchHidden:true' which works as expected(ie the tableview appears with searchbar hidden) when i launch the app for the first time(ie window creation) but when i go to the next tab and com back again to the tab1 the search bar is visible….

Is this a bug., how can i over come this issue …

    This is in fact a bug, here is a ticket that has been created to fix this issue that you can keep an eye on to know when this is fixed for you. http://jira.appcelerator.org/browse/TIMOB-4274

    for the time being, until this is fixed, maybe you could do something like:

    win1.addEventListener('focus', function() {
    tb.searchHidden = true;
});

    to force it to work.
