Aligning multiple lines of text in a Label

Hi guys,

I'm having some trouble aligning 2 lines of text in a Label:

introLabel = Ti.UI.createLabel({
        textAlign: 'TEXT_ALIGNMENT_CENTER',
        //center: {x: '50%'},
        width:'auto',
        height: 'auto',
        font: {fontSize: 25},
        color: 'black',
        text: characterName + '\n' + (levelsArray[levelData][0])/1000 + " seconds!"
    });

As you can see, I use an escape character for a new / second line (this works just fine) BUT the alignment is then lost.

The first line is perfectly horizontally aligned in the centre but the second line is then aligned to the left of the first line. You end up seeing something like:

CharacterName
30 seconds

…as it appears here in this screen rather than the "30 seconds" being in the middle below 'Character Name'

(I hope this makes sense!)

Cheers!

(I'm sure this is a 'user error' on my part and not an 'issue' but just fwiw, I'm using Android, the latest updated SDK, Google APIs 2.3.3 on the emulator, Win 7, also running on an XPeria Arc S LT18i)

    Hi

    From your description I am guessing your want the two lines to be in a centred block, because you have added both lines to the same label it simply follows the centred option for the entire contents. 

    var view = Ti.UI.createView({
    height: Ti.UI.SIZE,
    layout: 'vertical',
    width: Ti.UI.SIZE
});
win.add(view);
var lbl1 = Ti.UI.createLabel({
    color: 'black',
    font: {
        fontSize: 25
    },
    height: Ti.UI.SIZE,
    left: 0,
    text: characterName,
    textAlign: 'left',
    width: Ti.UI.SIZE
});
view.add(lbl1);
var lbl2 = Ti.UI.createLabel({
    color: 'black',
    font: {
        fontSize: 25
    },
    height: Ti.UI.SIZE,
    left: 0,
    text: (levelsArray[levelData][0])/1000,
    textAlign: 'left',
    width: Ti.UI.SIZE
});
view.add(lbl2);

    This works by adding two separate labels inside a container view. These two labels are left positioned and left aligned to the parent view. The parent view is simply added to the window (or other parent view) and because there are no other positioning elements it sites in the centre.

    This technique creates a centred block of left aligned text to that block.
