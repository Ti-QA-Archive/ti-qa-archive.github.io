    Hello, Kai, Why are you naming your Xcode as 5 or 6? If you want both of them in your Mac , First install Xcode5, which will install as Xcode.app in your application directory. Now then install Xcode 6. During installation, you would be asked to replace the other Xcode. Don't do that. Install keeping both, Xcode 6 will install as Xcode2.app.

    Don't use SDK 3.4.0.201409261227 or any other RCs. Use GA versions.

    Now on Terminal Run 

    sudo xcode-select -switch /Applications/Xcode2.app/Contents/Developer/

    for This configuration (SDK 3.4.0.GA, CLI 3.4.0, Alloy 1.5.0, iOS 8.0 And 7.1)

    Or Run 

    sudo xcode-select -switch /Applications/Xcode.app/Contents/Developer/

    For this Configuration( SDK 3.3.0.GA, CLI 3.3.0, Alloy 1.4.0, iOS 7.1,6.0)

    Check the selected SDK by

    titanium sdk select

    Select the Ga version if not selected.

  • I have found the answer to my very special problem. I will describe it below. Maybe 99% of you will already succeed by following the other comments and answers above.

    In my case, the "ti info" command doesn't like the alias in the Xcode package.

    (As one additional action, I changed the name of my Xcode 6 app, to "Xcode.app" instead of "Xcode6.app", followed by the xcode-select command. But I am not sure if this is necessary.)

    So, in my /Applications folder there are now Xcode.app (6.0.1) and Xcode5.app (5.1.1) in parallel.

    The Xcode.app contains after installation two SDK subdirectories:

    • /Applications/Xcode.app/Contents/Developer/Platforms/iPhoneOS.platform/Developer/SDKs/iPhoneOS.sdk

    • /Applications/Xcode.app/Contents/Developer/Platforms/iPhoneOS.platform/Developer/SDKs/iPhoneOS8.0.sdk

    The second is an alias on the first directory. This caused my problem…

    My workaround is the following: I replaced the alias by duplicating the first folder (about 150 MB) and I renamed it to the alias name. Now I have two identical SDK folders…

    When I type ti info -t ios I get the wanted result:

    Titanium Command-Line Interface, CLI version 3.4.0, Titanium SDK version 3.4.0.GA
    Copyright (c) 2012-2014, Appcelerator, Inc. All Rights Reserved.

    Please report bugs to http://jira.appcelerator.org/
