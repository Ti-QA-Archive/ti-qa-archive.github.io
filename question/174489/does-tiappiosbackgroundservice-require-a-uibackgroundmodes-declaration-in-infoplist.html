Does Ti.App.iOS.BackgroundService require a UIBackgroundModes declaration in Info.plist?

I'm assuming BackgroundSerive does not require a UIBackgroundModes declaration in Info.plist, BUT before I submit my first app for approval, I wanted to make sure that's the case.

I'm only using BackgroundSerive (bg.js) to hit the api once and notify the app has gone into background.

Ti SDK 3.2.3GA, iOS 7.1

  • Hi

    As far as I am aware (base on iOS documentation) you only need to state the background mode(s) if it affects other services that might be required whilst in the background mode;

    • iOS Keys - UIBackgroundModes

    If you match anything there then declare it.

    This is a snippet from an app I am working on that uses iBeacons;

    <ios>
    <plist>
        <dict>
            <key>UIBackgroundModes</key>
            <array>
                <string>location</string>
                <string>bluetooth-central</string>
            </array>
        </dict>
    </plist>
</ios>

    As this app uses both location and blue-tooth services - they are declared as you can see.
