XHR POSTing file to server (multipart problems)

Hi,

this is making me nuts. I am trying to send a client generated text file myfile.gpx from android to my server.

After generating the file I assign the file contents to the variable fcontent, which I'd like to POST together with an id (this is an authkey) to my server. The PHP server side is OK, I tried it with native code for blackberry, android and java.
Logging on the server side gets me the two parts (filecontent AND id / authcode)

But with Titanium instead of the filename I get what seems to be generated filenames like tixhr12345.txt instead of the original filename. How can I specify the filename before POSTing it?

var fcontent = gpxfile.read();

var xhr = Titanium.Network.createHTTPClient();

xhr.open('POST','http://www.myserver.net/subdir/?q=uploader);

xhr.onload = function() { handleAfterSentRouting(this.responseText); };

xhr.send({mFile: fcontent, id: 'code12xyz12xa'});

I am using Titanium 1.2.2 with the 1.4 SDK with Android

  • since you are passing in the file content in as a blob like that, the framework appears to be creating a temporary file based on the blob type…

    https://github.com/appcelerator/titanium_mobile/blob/master/iphone/Classes/TiNetworkHTTPClientProxy.m#L415

    I guess as a work around, you could pass the file name as another parameter
