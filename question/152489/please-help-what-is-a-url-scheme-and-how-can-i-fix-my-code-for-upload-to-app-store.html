  • Hi

    Not sure how much help this will be as you are going to have to dig through your own code or potentially a custom module.

    A URL scheme is a method of opening another app and optionally passing data to it.

    For example;

    Ti.Platform.openURL('http://www.apple.com/');

    Other scheme example can be found in the IPhone URL Schemes wiki.

    Yours indicates fb, chances are this is Facebook related. If you have added anything like the above or you have a facebook module - the problem is there.

  • If you are sure the problem is

    <property name="ti.facebook.appid">[the_app_id_here]</property>

    You can try to manually set this very early on in your application with a 

    Ti.App.Properties.setString('ti.facebook.appid', '[the_app_id_here]');

    (you should be doing this anyway, no hashes, api keys, oauth info should be in the tiapp.xml contrary to anything you read)
