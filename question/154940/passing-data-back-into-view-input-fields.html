Passing data back into view input fields

Hi All,

I see lots of examples of getting data from input fields (inputs, textareas) and have working code with those. The issue I am having is getting data back into those fields for editing. I have created an app that asks the user to create an account. I have stored the account data in local properties as strings:

Ti.App.Properties.setString(JSON.stringify(account));

I can, obviously, ask for that data via the getString(account) code … also fine. But where i get stuck is getting that data back into the form fields.

I have added a focus(loadData); event call to my tabgroup to load the fields in the alloy framework

function loadData(){
    var account = Ti.App.Properties.getString('account','empty');

    if(account != 'empty'){
        account = JSON.parse(account);
        $.first_name.value = account['first_name'];
        $.last_name.value = account['last_name'];
        $.email.value = account['email'];
    }


}

which in theory matches the view in the tabgroup

            <Tab id="tab2" title="Account" icon="111-user.png">
                <Window id="win2" title="Account">                    
                    <ScrollView id="scrollView" showVerticalScrollIndicator="true" height="90%" >

                        <Label top="1" class="formlabel">First Name</Label>
                        <TextField id="first_name" top="25" hintText="Johnny" ></TextField>

                        <Label top="70" class="formlabel">Last Name</Label>
                        <TextField id="last_name" top="95" hintText="Appleseed"></TextField>

                        <Label top="145" class="formlabel">Email</Label>
                        <TextField id="email" top="170"></TextField>
                        <Button id="save" top="745" class="smallButton" onClick="accountSetup">Save</Button>
            </ScrollView>
                </Window>
            </Tab>

environments:
Ti v 3.1
Mac OSX

App is for iOS and Android

But this doesn't seem to work. So how do I tackle this problem?
