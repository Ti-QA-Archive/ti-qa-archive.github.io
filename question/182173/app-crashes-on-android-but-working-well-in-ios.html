  • I'm a little surprised that this is all you are seeing in the logs. I would expect to at least see the titanium runtime spinning up.

    At any rate, you should probably try cutting down your app to something simpler so you can isolate the problem. For instance, if you have a complex main window with tabs, menus, buttons, controls, etc., try replacing the call to create that main window with a call to create a simple window with maybe a single label on it. Don't even load the code that builds your normal main window.

    If you have code that does other interesting things like network requests, location updates, third party modules, etc., try to eliminate them. Eventually, you will figure out which part of your code is to blame for the crash, and you can focus on fixing that.

    BTW – I always recommend that developers who want to build cross-platform Titanium apps should build them cross-platform from the very beginning. Run on both platforms every single day during development. If you let weeks of development pass by while you are only running on the ios simulator, you will experience a substantial amount of heartburn when you try to get your code running on android.
