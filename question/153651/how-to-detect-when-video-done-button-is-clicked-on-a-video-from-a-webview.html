How to detect when video "done" button is clicked on a video from a webview?

I have: a window with a webview. The webview contains a remote URL to a HTML page with a simple HTML5 video tag (mp4 video). Video works fine. Plays after clicking the play button.

The main window (thats contains the webview) is only allowed to be viewed in PORTRAIT mode.

When I open the video from the webview, the video is played fullscreen. Still in portrait mode. I change it to LANDSCAPE. The video changes to landscape. So far so good…

Problem: after viewing the video in landscape and clicking the DONE button, for some reason the parent window layout (which contains the webview with html5 video and has portrait-only allowed) is suddenly messed up and stretched half-way outside of the screen.

My conclusion: when the video opens from the html5 webview, it forces a layout change on the parent window, despite this window has portrait-only allowed.

Question: how can I prevent this window from being forced to change?
Question two: how can I track a html5 video (not an activeVideo object available!) is done playing?

I tried the "focus" event on the parent window. Doesn't fire. Tried orientationchange too; doesn't fire.

Any ideas?
