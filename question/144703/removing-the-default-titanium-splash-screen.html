  • Sorry – you can't do it. That screen is shown while the runtime is initializing itself. You can't get rid of that startup time, and you need to show something.

    I've heard of iOS developers showing a screenshot of the UI so that it looks like the UI comes up faster. Although you could argue that a UI that looks ready but is unresponsive is a bad user experience.

    At any rate, this technique won't work on android, with the wide variety of screen densities and skins applied to the UI controls.

  • This isn't entirely correct. I was able to remove the splash screen by overriding the AndroidManifest.xml file in the folder structure.

    Application theme = Some.Theme
    .MainActivity theme = Some.Theme.NoActionBar (this removes your splash screen)
    .TiActivity = Some.Theme.NoActionBar (this is your first activity, say a login screen)
    .TiActivityTranslucent = SomeTheme (this is every other <Window> of the application)

    This removed the Android splash screen from my sample app and went directly to my login screen.
