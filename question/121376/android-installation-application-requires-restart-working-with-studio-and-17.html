Android Installation "Application Requires Restart" working with studio and 1.7

After installing my android project from the market, If I select the open button on the app page, I receive a message "Application requires restart". This message is also displayed after installing the project from the .apk file, and choosing the "open" option. However If the application is first launched from the applications menu the message is not displayed. Any suggestions on what could be causing this?

  • I ran into this problem today, and short recompiling, there isn't much you can do to stop it. What I did however, was control the message that pops up so it doesn't scream "Bug!" so much.

    Add the following to your tiApp.xml:

    <property name="ti.android.bug2373.message">App initialized, Restart Required.</property>
