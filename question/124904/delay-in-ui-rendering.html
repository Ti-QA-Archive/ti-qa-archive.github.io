Delay in UI rendering

When switching windows through my tabbar I notice a render 'delay' of my UI elements. Is this something that is inherent to the use of Titanium and its performance, or is this something we can solve.

To give you an example:

  • When switching tabs. I'm trying to create a subbar below my navigationbar in the new Window. Eventually it will house a segmented switch.
  • When switching to that tab. The navigationbar and background image show up instant. The subbar however renders with a noticeable delay. Almost with a short flash. Note: It's really fast, but there is a delay.

  • I'm creating the window inside app.js with the following code:
var win3 = Titanium.UI.createWindow({  
    title:'About',
    backgroundImage:'more_detail-background.png'
});

var tab3 = Titanium.UI.createTab({  
    icon:'info.png',
    title:'About',
    window:win3
});

var label3 = Titanium.UI.createLabel({
    color:'#999',
    text:'This renders instantly',
    font:{fontSize:20,fontFamily:'Helvetica Neue'},
    textAlign:'center',
    width:'auto'
});

var subBar = Titanium.UI.createImageView({
     backgroundImage:'news-segmented_switch_bar_background-1px.png',
     width: 320,
     height: 57,
     top: 0,
 });    

win3.add(label3);
win3.add(subBar);

Any info or insights would be great to have. Even if this is because of the inherent parsing delay and performance trade off by using Titanium. Right now it's bugging me and I'd like to get a sense of why there's a delay.
