  • in titnaium setters and getters have not been implemented on every UI object.

    So in this case it maybe that there isn't a setVisible setter, without seeing your UI object code difficult to tell.

    However if your callback is firing, then using a setter is maybe not the best solution, it's good to meet coding practices but setters have a performance hit, and as already mentioned may not actually exist.

    it is better to use the object property directly, as the setVisible function only sets the visible property anyway.

    so to guarantee the best performance and conformity always use the object property. I.e.

    $.abort.visible = true;
