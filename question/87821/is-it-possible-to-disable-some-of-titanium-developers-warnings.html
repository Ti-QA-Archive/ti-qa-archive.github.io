Is it possible to disable *some* of titanium developer's warnings

It warns about many syntactic forms that are valid and useful, such
as:

if (condition)
   statement;

and:

x = (y
     + z);

I can avoid some of these (and be annoyed) but not all of them. The
result will be that I'll miss warnings I care about in the noise.

I'm hoping for a preference or config file somewhere that will let me
disable selected warnings. Most lint tools have this sort of thing.

Otherwise, I'll just ignore the warnings as I'm using other linty tools.

  • You can use this kind of comment at the begining of your script :

    /*jslint forin:true undef:false immed:false maxerr:999 */

    I've found this to know the available switches, but some don't work.
    For example, if you find a way to remove the message "[WARN] JavaScript compiler reported "'XXX' was used before it was defined." at xxx.js:441", I would be really happy to know how…
