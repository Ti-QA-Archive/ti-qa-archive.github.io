Error Validating App Using Older Titanium SDK and older XCode

We have an app that we are compiling using Appcelerator 3.2 using and older SDK (1.8.2) and XCode 5. Whenever we try to validate it we get the following error:

"This bundle is invalid. When supporting iPhone, the executable must include support for the armv6 architecture, unless the UIRequireDeviceCapabilities include the 'armv7' capability"

We have searched through the forums here and on Google and have tried editing the Info.plist and setting the UIRequiredDeviceCapabilities to armv7. We've also added the min iOS directive to tiapp.xml. Found a thread that suggested editing the build templates in the SDK (https://developer.appcelerator.com/question/128721/build-for-armv6-with-18-sdk-and-xcode-42) but they already support armv6 and armv7.

Any suggestions would be appreciated.

    Since you use the word any, I'll throw out the obvious and best solution. Update the app to a current SDK. The only reason not to do so, is if you have an overwhelming need to support ios 4.x and are on a private store. You cant even put a 1.8.x app into the apple store now.
