Trouble launching Android Genymotion device from Titanium Studio 3.4.1

I'm having some difficulties running my Android Genymotion devices directly from Titanium Studio.

Is there a special setup procedure to get this to work correctly?

I've downloaded the latest version of Genymotion and I have 2 devices within that that launch and run fine on their own when launched via Genymotion.

However, if I try and launch them via Studio (they do appear in my device list) I just get presented with the Run Configurations dialog. If I then hit "run" it vanishes for a few seconds, then reappears again.

I've updated my SDK and Studio to the latest 3.4.1 GA which I know has some fixes for Genymotion support, however it doesn't seem to have made any difference to my setup.

Can anyone suggest how I can get these devices running correctly on my Mac?

The two devices I have setup are….

Google Nexus 10 - 4.2.2 - API 17

Google Nexus 4 - 4.1.1 - API 16

  • OK, so it would appear there are two issues here.

    1. You can't build a project that has a space in the name (e.g "My Project Name" will not build, but "MyProjectName" will)

    2. The custom ADB in Genymotion settings clashes with the default Android one. I found a post on Stack Overflow that helped me out.

    The easiest way to fix seems to be to update your Genymotion Settings so it uses the same ADB as your Android SDK. Just check the "Use custom Android SDK tools" option and enter your desired location.
