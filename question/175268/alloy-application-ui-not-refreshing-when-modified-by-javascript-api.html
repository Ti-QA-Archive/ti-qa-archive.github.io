Alloy application UI not "refreshing" when modified by JavaScript API

Application Type: mobile
Titanium SDK: 3.2.3
Platform: Android 4.3
Device: Samsung Galaxy S4
Host OS: Mac OS X 10.10

I'm having a REALLY strange issue… Basically I have a button that when clicked opens a date picker. When the date picker is changed, the title of the button is changed to reflect the date they chose.

However, once the user picks a date, everything is triggered correctly and the actual JavaScript button object gets updated to say title = "05 / 06 / 2014" for example, but the actual button UI element does not reflect the change; the title remains at the original title "Select Date".

When you then select another field, that's when the button title actually displays correctly.

Is there some way to force the UI to "rebuild" from the JavaScript object? Because clearly the JS object is being updated but the UI isn't following suit until another field is selected.
