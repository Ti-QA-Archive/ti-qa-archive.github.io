registerForPushNotifications doesn't seem to work anymore on iOS7 in production binary

I packaged our app for iOS for production using our company's enterprise keys and to be distributed in our company obviously.

However, the problem I am facing right now is that the app is getting stuck at Ti.Network.registerForPushNotifications forever. If I compile the same code using a normal build (development), our app can work.

Anyone have observed this bug so far?

  • I found the fix for this bug.

    if Ti.Network.remoteNotificationsEnabled returns 'true', you can call Ti.Network.registerForPushNotifications no matter how many times you like. If it returns 'false', skip the call or else your app will just hang inside the Ti.Network.registerForPushNotifications function and you will never get a callback.

    Note again that this issue is particularly inherent in iOS7 and only when you build production binary.
