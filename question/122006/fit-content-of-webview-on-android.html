Fit content of WebView on android

I've got some in-line html formatted data that I display with the help of a webview. It renders the content perfectly fine, however it does surround it with alot of whitespace, which makes it possible to scroll in pointless areas.

The webview behaves the same in the kitchen sink example so I wonder if it is even possible to get the webview fit to it's content, as iOS's does?

  • If the content is .html files that are compiled into your app, consider using the viewport meta tag in your page headers. Keeps that horizontal scrolling from occurring on android. 

    meta name="viewport" content="width=300"

  • Yeah, thats about what ours looks like. Don't do anything else special.
    Thinking maybe it was the way we setup the views, but nothing special there. In one case, its set in a model window with a navbar. The other is just a webview pasted on a tabwindow.

    We set a DOCTYPE before the html tag:

    !DOCTYPE html PUBLIC "-//W3C//DTD HTML 4.01//EN" "http://www.w3.org/TR/html4/strict.dtd"

    Set the charset and viewport in the header:

    meta content="text/html; charset=UTF-8" http-equiv="content-type"
meta name="viewport" content="width=300"
