Facebook Module Issue

I updated titanium to the latest version (3.4.1.201410281727) and am now having a problem with the facebook module. It seems to function fine on iOS but I get the following error when trying to run the app on android.

11-20 20:55:18.631: E/dalvikvm(7285): dlopen("/data/app-lib/com.app.testing/libfacebook.so") failed: dlopen failed: cannot locate symbol "_ZN8titanium13TypeConverter23jsObjectToJavaKrollDictEP7_JNIEnvN2v85LocalINS3_5ValueEEEPb" referenced by "libfacebook.so"...

Has anyone found a solution to this problem? I have used the search and have been unable to find much of anything. If I comment out all instances where the facebook module is used and remove it from my tiapp.xml file the app will open up fine (but I lose the facebook functionality) I have tried cleaning the project and restarting to no avail.

Please advise.

Thanks

  • Possibly irrelevant but the latest stable SDK is actually 3.4.1.GA. It would be worth upgrading to that as the beta point releases often have weird bugs and at the very least, you would be ruling that out.

    Also, it's my understanding that the Facebook module hasn't been actively maintained for a while. There was an article yesterday on tidev.io about a fork of the Facebook module that someone has made and is fully up to date. Worth a try:

    http://www.tidev.io/2014/11/20/new-facebook-modules/
