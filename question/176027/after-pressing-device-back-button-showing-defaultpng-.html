After pressing device back button showing Default.png ?

Hi All

Whenever my app launches Default.png file loads after this index.xml window loads.

Issue: After pressing device back button in index.xml window then default.png showing and it is idle.

because of this my app is idle , I am closed my app and re-launched my app once again.

Is there any way to re-start my app while after press device back button default.png showing?

and also i want to show activity indicator as well while Default.png file is loading first time my app launches?

  • hello,

    Check this Window Property : exitOnClose

    Boolean value indicating if the application should exit when the Android Back button is pressed while the window is being shown.

    You can only set this as a createWindow({…}) option. Setting it after window creation has no effect.

    <Alloy>
    <Window backgroundColor="blue" exitOnClose="true">
        <Label id="label">I am a blue window!</Label>
    </Window>
</Alloy>

  • If you wish, capturing the android back event will prevent it going back to the splash screen. You may then react to the back button however you please in service of your app, but for example here:

    $.index.addEventListener( 'androidback', function(){
        Ti.API.info( 'back button pressed, but does nothing - including not going back to the splash screen' );
} );
