Android hardware acceleration and tabs width

Hi.

I'm trying to get hardware acceleration working (again :) ), but this time I have a problem with tabs width.

I'm using latest 1.9.0 build (Titanium SDK version: 1.9.0 (01/06/12 18:01 1c5a932)), Android 3.1, Thinkpad tablet, Window XP.

Test case can be found here: https://gist.github.com/1582755

To get this example work, you need to create new project named "Test", make "Resources/platform/android" directory with custom "AndroidManifest.xml" file with content that is present in gist I posted.

Also, "tiapp.xml" file needs to be modified according to gist.

Then, copy-paste "app.js" content from gist to "app.js" in project and try to run project on device running Android 3.1.

If you manage to compile project correctly, you'll see that there are two tabs, but their width is less than '50%' of screen width. That's different from usual behavior when there is no hardware acceleration enabled.

As you can see, I tried to set tab's width, but it had no effect. :(
Maybe it has something to do with theme selected, but I don't have previous Android experience.

Do anyone know how to fix this?

Thanks.
