    If you have existing Node.ACS applications created during the developer preview, you must must republish your existing applications to take advantage of the new environment. See the instructions in the next section.

    Migrating Applications to the Production Environment

    To migrate existing applications to the new production environment:

    Update to the latest acs npm package.

    If you are using Node.ACS from the command line, run the following command:
    sudo npm install â??g acs
    If you are using Node.ACS through Titanium Studio, you will be prompted to install updated acs package the next time you launch Studio.
    Note that you cannot use older versions of the acs npm package to create or publish new Node.ACS services.
