    Nope you read that just fine, i did not know the size of the static data so it could be a possability.

    Seeing you next question in the comments, you have static data in a tableview with alot of rows? Or something similiar.

    Using a .js file, you can create an array or object structure.
    In a database you will have a table with multiple rows representing the rows in a tableView.

    This is easy to recreate using an array or object.

    // your staticText.js file

Var staticText = {};
    staticText.section1 = [];
    staticText.section1[0] = 'Some text';
    staticText.section1[1] = 'Some more text';

    staticText.section2 = [];
    staticText.section2[0] = 'Another part with text';

var i = 0;
while( staticText.section1[i] )    {
    alert(staticText.section1[i];
    i++;
}

var j = 0; // or re-use i
while( staticText.section2[j] ) {
    alert(staticText.section1[j];
    j++;
}

  • Im no expert, but i guess if its text that will be static and only change with an update…

    I would create a seperate .js file (lang/eng.js) and then create lots of variables with text in there.

    Then if you decide to go multilingual you can, also you can change text pretty easily too.

    I have used a similar method for an app that i will be re-branding several times, so i have logo's, company name, company phone no's etc… all stored in a config.js file.

    You just have to include the lang.js file to get the text var's.

    I have a feeling a .js file will be quicker than a txt file or a database for static text, but i may be wrong.

    Adam.
