The bulk of the newer documentation says to create a 'classpathentry' in the module folder to indicate where any module .jar files you've included are located. No included .jar files, no entry in .classpath. The only other thing to do, according to docs, is to create a module node in tiapp.xml indicating the presence of the module to the mobile project. That is:
    <modules>
    <module version="(version #)" platform="android">com.example.modname</module>
    </modules>
    One comment posted here suggests that this node should be placed outside the "android" node in the tiapp.xml file. If you find a solution I would really appreciate hearing about it.

    One way of doing it (just for this app) is to create a folder in your project root folder called 'modules', then another within that called 'android', then copy the actual module folder ('com.what.ever') in there.

    Empty the the build/android folder to force a clean build and give that a try.

    (You did try a clean build, right?)

    Also, have you defined it in the tiapp.xml file in the <modules> section?
