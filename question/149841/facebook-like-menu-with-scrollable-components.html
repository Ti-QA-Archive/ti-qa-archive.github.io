    Hi

    Having just completed this myself for a large cross platform project, my solution is a simple one.

    Do not worry about it.

    By this I mean, leave the opening of the slide menus to the menu buttons that would be placed on the top bar. Then focus on getting your app complete. If you are not comfortable with handling the complexities of multiple events on the same views affecting different processes - simply hold off that until the end or version 1.1.

    There are lots of things you will have to overcome to ship the app and this is a "rat hole" that you mind yourself down for some time when you could be shipping your app.

    Most good apps that use the Facebook style UI, do not actually use the slide to open gesture in the main body as it DRASTICALLY reduces the content windows views you can use, no scrollableViews for example, no zooming/scaling images.

    If you want a great example of a Facebook style app, that takes the content section up a notch with other interactions, find the German McDonald's app, it is a great example of what you should focus on.

    Concern yourself more with getting the Android version to do the same basic thing.

    • i've solved the problem. ;)

      My key error in thnking about it and/or adopting other examples/Codes is to try handling the drag/drop Stuff on the whole view (or in my case window)

      i've build an environment that allows me to speciify each single item to be able to "drag over and move the whole view" or not.

      So, i can "flag" a list as "dragable" but maybe a single button as "not". That was my apporach earlier, but my structure was wrong so it won't work.
