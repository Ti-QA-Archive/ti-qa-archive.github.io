    Hi

    When you specify visible to be false it simply hides the "thing" from view - it does not remove it from the display other than its visibility. The top, left, bottom, right, width and height all remain exactly the same as they were before you made it invisible.

    So when you add a view to a parent view with layout set to vertical, simply saying "hide" does not change the layout, nor would setting horizontal.

    This is the normal behaviour and is not an error. If you have error worked with html/cc then the visible property there is the same here. Whereas the css display none would do what you what in html/css there is no direct equivalent here as the two environments are completely different.

    The only way to free up that space is to either remove the item in question from the parent view or be sneaky.

    The sneaky way (although not tested this) would be to add your child view and set the width and height to be zero, this would then remove the size of the view from the parent view. It might be more manageable to wrap the child view in a container and apply the width and height of that - this way the real child is simply clipped and should not affect any complicated layouts you may have inside the client.
