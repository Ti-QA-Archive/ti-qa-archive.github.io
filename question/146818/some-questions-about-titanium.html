Some questions about Titanium

Hello everyone,

I want to ask you some general questions about Titanium:

1- If I want to developp an application for Android, how should I take care my pictures's resolution used into my application, I mean what should I do to keep my images size unchanged no matter what is the deployment target device?

2- When should I assign a number for properties (length or widht for example) using px and when should I use % ? what is the best practise here?

3- Titanium has a dedicated IDE, but I wander what is the difference with PhoneGap?

Looking for your answers… Thanks ;)

    1 - Add this to your tiapp.xml

    <android xmlns:android="http://schemas.android.com/apk/res/android">
    <manifest>
        <supports-screens android:anyDensity="false"/>
    </manifest>
</android>

    Essentially makes all your images scale to the screen size.
    Check this reference for more information.

    2 - Assign a percentage if you want a fluid layout that can adapt to different sizes, assign an absolute pixel / point value when your dealing with specific sized UI.

    3 - Have you read this? Worlds of difference. PhoneGap is a webview, you write mobile webpages. Titanium is a pseudo native javascript interpreter with hooks in the platform..

  • Hi

    You have a perfectly good answer already, but I thought it worth noting that the use of dp as a dimension unit has greatly simplified my cross platform app design. I can still use normal positions they are just anchored to an imaginary grid that allows different android screen sizes to work as if they had the same virtual dimensions.

    I would not recommend this for games or apps that truly bridge small phone to large tablet without other logic in play but for most application layouts it can be a real performance boost.

    You can even change the default setting in the tiapi.xml file, it defaults to system but you can change it to dp or one of the others.

    It also means both android and iOS work with the same virtual dimension system.

    Not for everyone but works great for me.
