  • If you're referring to Titanium Mobile's Mobile Web platform, then yes it's a great idea!

    You can take the same code from your Android or iOS Titanium app and compile it into a mobile web version. This will produce a website that you can deploy on your web server.

    Mobile Web is much more sensitive to poor software design, so you really need to minimize the size of the app. Titanium Mobile Web supports a number of modern browsers and devices and supports around 90% of the Titanium APIs.

    It's definitely worth looking into.

  • I'm afraid I don't quite follow you there.

    What are you trying to achieve exactly? Are there some very specific needs that your project need to fill?

    Any idea can be good (or bad), it all depends on the goals and constraints of each project.

    One of the big advantages in using Titanium is that you delegate the support of multiple mobile browsers to the framework. So you can focus on the actual functionality of your site rather than working out the kinks depending on which device connects to your site.

    The other near feature is that on every new release, you can simply recompile your project and benefit from all the fixes an performance improvements.
