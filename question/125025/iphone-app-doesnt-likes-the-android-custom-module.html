iphone app doesn't likes the android custom module

hi there!

i'm building an app for android and iphone, that gets all the artists in your music library with two modules one for each plataform.

if i'm trying to run the app in android i didn't have any problems, but if i'm trying with iphone i get this error:

[INFO] Detected third-party module: com.nvivo.conciertos/0.1

[ERROR] Third-party module: com.nvivo.androidmusic/0.1 missing library at /Users/txarly/Documents/nvivo/modules/android/com.nvivo.androidmusic/0.1/libcom.nvivo.androidmusic.a

[ERROR] Error: Traceback (most recent call last):

  File "/Library/Application Support/Titanium/mobilesdk/osx/1.7.2/iphone/builder.py", line 638, in main

    sys.exit(1)

SystemExit: 1

in the tiapp.xml file i have these lines:

<modules>
    <module version="0.1">com.nvivo.conciertos</module>
    <module version="0.1">com.nvivo.androidmusic</module>
</modules>

but if I delete the second module i can run the app.. how can I run the app with the two modules in the same app?

and one more thing…

the android app only works if i put the module in the project folder not in Resources dir, and the iphone version only works if i put the module in:
project/Resources/modules/iphone

is normal to have the two modules separated?

thanks!

    Hola Miquel :)

    You have to set the platform for each module.

    <modules>
    <module version="0.1" platform="iphone">com.nvivo.conciertos</module>
    <module version="0.1" platform="android">com.nvivo.androidmusic</module>
</modules>
