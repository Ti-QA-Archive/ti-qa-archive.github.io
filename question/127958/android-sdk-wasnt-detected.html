  • Yesterday I got new machine with x64 Windows and I had to go through all horror of Titanium installation. :)

    Documentation miss things, for example, in my case, "javac" could not be recognized although I did everything from the docs (yes, I installed JDK, not JRE, put all env vars, etc.). I did some search and found that "CLASSPATH" environment system variable has to be set to point to Java's bin directory. As Paul mentioned, Java 32-bit is required, the best is to install JDK and JRE in "C:/Java" directory because there could be problems if directory name has spaces (like "Program files").

    Then, next thing was that Android SDK could not be recognized, but it was installed (all versions!) and everything worked fine (I could launch emulator with AVD manager), only Titanium studio could not recognize it. There was a bug reported regarding this and seemed to be solved, but it seems to me that it's still present. Solution in my case ('cause other people had some other solutions that didn't work for me) is funny. Really funny. I needed to move Titanium SDK to other location, so I changed that in preferences (Studio -> Window -> Preferences -> Aptana Studio -> Titanium -> Titanium SDK Directory) and then Android SDK was recognized. I was WTF-ucked, I lost couple of hours to get that work and then I managed to solve it accidentally. :)

    I hope Titanium Studio will become quality product as Aptana once was, I want to press "Next", Next", Next" … and not to loose whole day just to install basic stuff for development.
