Performance hit when putting labels in ListView

Hello

I'm currently testing out the new ListView, which appears to work much smoother than the TableView in a lot of cases. However, as I was testing I noticed that putting labels into the ListView quickly gave a performance hit on iPhone when having ~200 rows in a single ListView. Having just an image scrolls super smooth, but that was to be expected. Adding 1 label to each row is still fine, but having 2 or 3 labels you really start to see a performance hit. I tested on an iPhone 4 using version 3.1.0 of the SDK.

I've done some testing on an older Android device (running version 2.3.5) as well, which doesn't seem to take the same performance hit when adding multiple labels in each row. Here the performance appears to be quite stable, whether I use 1 or 3 labels in each row - overall a great performance upgrade from the TableView in similar use cases for me.

I tested with the sample code from the docs, see "List View with Custom Item Template". I simply added 60 extra rows to each dataset.

Do any of you guys have the same experience? And is there anything to do to optimize the performance of this scenario? I added a solid white background to the labels, which appears to have helped a bit but not by much - anything else you can think of?

  • Hello,

    how much rows do you see in your screen? 10? 20? The best practice says "lazy load stuff". Please, try to load whatever you will be able to see in the screen, and then cache another two screens (next and previous).

    In general, you can't expect to draw 200 or whatever number of rows without getting a performance impact. And think about that same app running on a cheap android. The UX will be horrible.

    Best,

    Mauro
