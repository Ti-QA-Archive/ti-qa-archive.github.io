iOS Module link third party framework and library

Hi,
I'm trying to link third party frameworks and Xcode standard library in an iOS module. So far I've only got frameworks linked successfully by modifying the module.xcconfig:

OTHER_LDFLAGS=$(inherited) -F"<path to the framework>" -framework <framework name>

However, I can't link the Xcode standard library libc++.dylib and libs.dylib (these were link from Xcode the same way as framework (Link Binary With Libraries)) . I've tried many ways but no success. Could somebody help?

Thanks.

  • Hey Stephen,

    I struggled with this issue too and I think I've found the solution. You need to include in your module.xcconfig the following:

    OTHER_LDFLAGS=$(inherited)  /usr/lib/libc++.dylib /usr/lib/libs.dylib
