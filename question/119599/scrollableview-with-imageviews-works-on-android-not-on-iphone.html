scrollableView with imageViews works on Android not on iPhone

I have a scrollableView containing views containing imageViews. The images come from a remote server as base64 encoded strings. The image dimensions are also retrieved from the server. I decode the base64 string, put the resulting blob in the imageView and then add the imageView to a view which is later added to the scrollableView like this:

var views = [];
for(images received) {// pseudo code of course
  var imageView = Ti.UI.createImageView({
     canScale: true,
     enableZoomControls: false,
     image: Titanium.Utils.base64decode(base64String),
     width: wdth,
     height: hght
  });
  var view = Ti.UI.createView({backgroundColor: '#fff'});
  view.add(imageView);
  views.push(view);
}

scrollableView.views = views;

Everything works as expected on an Android device (Android 1.6) and the Android Emulator (1.6 and 2.2) but on the iPhone (3G) all I see is a blank white screen.

Has anybody ever experienced similar problems with the scrollableView on iPhone? Any ideas? Or is it a problem with the base64 encoding?

I am using mobile SDK 1.6.2 btw.
