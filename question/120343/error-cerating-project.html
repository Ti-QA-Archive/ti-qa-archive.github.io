  • Manu

    To attempt to resolve your issue, please perform the following steps:

    • if you are not subscribed to one of the plans, and wish to use Android debugging while Titanium Studio is awaiting its official release, ensure that only the Titanium Studio update site named "Stable" is enabled. Customers with a subscription may also enable the Beta site. Then run Titanium Studio / Help / Check for updates
    • if you are having issues installing a new build of Titanium SDK 1.7.X, delete the $USER/.titanium/mobilesdk/platform/1.7.0 directory, and then paste the link to the package from the Continuous Builds site into Titanium Studio / Help / Install Titanium SDK from URL
    • restart Titanium Studio
    • close all emulators
    • using the Android SDK and AVD Manager, upgrade all packages
    • ensure you have at least the minimum Android SDK installed, as per Prerequisite Android SDK / API
    • using the Android SDK and AVD Manager, delete all existing AVDs via the Virtual Devices menu
    • delete all files from within $PROJECT/build/android
    • configure your project to use Titanium SDK 1.7.X and Android 2.2, or at least a compatible combination of Android and Titanium SDKs, as per Android SDK / Target Android Platform. Note the issues with 2.3.X explained in the Titanium Compatibility Matrix
    • launch app

    Please let us know if this resolves your problems.

    Cheers
