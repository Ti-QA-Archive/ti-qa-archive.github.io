Best way to update Tableview Rows when GPS location change is fired

I have a GPS application that builds a tableview with distance label from the then current GPS location.
I have a separate routine that kicks off an event viewer and updates the map as the user drives. I need to be able to update the tableview rows as well, am a little lost as to the best way to do this.

Everytime the location changes should I rebuild the entire tableview?

    Depending on how complex your rows are it may be easier to just rebuild the rows. If you have very simple rows, and not a whole lot of them, then go ahead and just rebuild the rows. If you have complex rows, or a large number of rows that you'll be updating, what you'll want to do is to create an array of references to the distance label within each row. You can key this array with something that you'll have when you get the new distances to make it easier. You can then loop over the array or the keys to update the labels for the distances.
