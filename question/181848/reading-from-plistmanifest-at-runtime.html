Reading from plist/manifest at runtime?

I'm working on a white label app which will load data dependant on a unique ID.

What I wanted to do was the following:

  • Add an "app_id" field to both the ios plist & android manifest sections in the tiapp.xml
  • Package for both iOS & Android
  • Pass the project files to a third party who would then open the projects, change the app_id, icons etc and build for app stores
  • The app will read the app_id on opening and deal with it accordingly

Now, I can add the properties just fine, but I'm unable to access them in any way I can think of. I've tried looking around but to no avail.

Is there any way of doing this that will work for both Android & iOS? Is there an alternative way to do what I'm looking for?

Please note: I cannot just allow the third party to build directly with Titanium - that is out of my hands so I need to find another way.

  • Hi

    Dangerous process - handing off code to third-parties and not keeping that in-house. Your security is compromised as is that of all other parties involved in this process as they all have access to the same code.

    Ignoring the point above. You do NOT need to add different properties to both the android and iOS tiapp.xml properties. You should instead simply use tiapp.xml level properties and read those back in the app using the "property" namespace already available in Titanium.

    You can also read the "app id" already set and use that within the app - keep in mind that each app should app its own guid. With the new versions of Appcelerator going forwards you will find your way to control the process will be limited by the new licences and how each "organisation" can be set.

    If you are using Alloy then much of this could and should be done within themes, this way you can localise all settings, images, styles and more.
