Layout horizontal causing overlapping issue on iPhone 6

I am using Ti SDK 3.4.1 GA and Alloy 1.5.1. I am using a simple layout horizontal for placing two buttons next to each other. They look proper on all other devices except the iPhone 6 which shows the Rename button below the Cancel button. Though i can make them proper if i dont use a layout horizontal. What is it that i miss that is causing the issue?

        <View height='50'  borderColor="red" left='20' right='20' layout="horizontal">
            <Button id='btnCancel' height='Ti.UI.FILL'   width='50%'  title="Cancel" color='#4B443C' backgroundColor="#F4F3F0"></Button>
            <Button id='btnRename' height='Ti.UI.FILL'  width='50%'  title="Rename"    color='#fff' backgroundColor="#f68c00"></Button>            
        </View>

    Hi Adnan

    Assign horizontalWrap property to false to parent view of button
    e.g

    <View height='50'  horizontalWrap="false"  borderColor="red" left='20' right='20' layout="horizontal">
            <Button id='btnCancel' height='Ti.UI.FILL'   width='50%'  title="Cancel" color='#4B443C' backgroundColor="#F4F3F0"></Button>
            <Button id='btnRename' height='Ti.UI.FILL'  width='50%'  title="Rename" color='#fff' backgroundColor="#f68c00"></Button>            
        </View>
