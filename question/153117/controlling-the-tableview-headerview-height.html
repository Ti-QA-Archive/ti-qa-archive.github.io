Controlling the TableView HeaderView height?

I am having issues with the HeaderView of my Table View. I can't seem to control the hight of the header. The header is blown out to about 400dp Any ideas on what I might be missing?

Alloy

<TableView id="locationTable">
  <HeaderView>
    <View id="topContainer">
      <Button id="completeButton">Complete</Button>
    </View>
  </HeaderView>
</TableView>

The Table rows are dynamic.

Style

"Window": {
  backgroundColor: "#fff",
  top: 0,
  bottom: 0

  layout: 'absolute'*/
},
"#locationTable": {
  top: 0,
  bottom: 0

"#topContainer":{
  top: 0,
  bottom: 0,
  height: "25dp"
}
"Button":{

    Hi

    Yes, you have accidentally triggered the default behaviour of the view whilst trying to get it to do something else - I think.

    You have;

    "#topContainer":{
  top: 0, 
  bottom: 0,
  height: "25dp"
}

    As this is a view the default behaviour is to fill the space available. You have overridden this with the height being specified. BUT by adding the bottom value you have told it to go back to the default behaviour.

    This is a precedence thing. I suspect you were aiming for no margins - which would be correct if the parent view of this one had a layout of horizontal or vertical.

    Try removing both top and bottom - as neither is actually doing what you think (I assume) you are after.

    • If you did this;

      "#topContainer":{
  layout: 'vertical',
  height: Ti.UI.SIZE
}
"Button":{
  top: "5dp",
  bottom: "5dp",
  height: "20dp"
}

      You would end up with an auto sizing header that had a single child button that had a margin of 5dp top and bottom.

      Adjust to suite your needs.
