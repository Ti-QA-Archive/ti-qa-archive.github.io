What is the recommended way to include a js file moving forward?

I have an Android app that works great using the current release SDK (2.1.3 GA) with the exception of a known ICS calendar bug that is fixed in SDK 3.0. In researching possible workarounds, I installed several versions of the 3.0 and 3.1 SDK continuous builds. When I compile my app using and SDK later than 3.0, none of my includes work. The KitchenSink includes, which appear to be similar to mine, do work. That has me concerned.

This obviously isn't the place to discuss future releases or bugs therein but I'm concerned my includes are somehow incorrect. What is the recommended way to include basic js files with simple variable definitions and functions?

Here's a simple example of how I'm including files now: 

var win = Titanium.UI.currentWindow;
Ti.include('/includes/theme.js'); //sets background variable
win.backgroundColor = background;

Thank you for your time.
