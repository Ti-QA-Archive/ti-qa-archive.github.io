Keeping Static Local Database

I have what is hopefully a simple question. I have a sqlite local database that I have in the Resources folder. I won't have ANY user local data stored in this DB. Thus, I don't need to worry about upgrades or anything like that. I simply want to use the database as is, as it is included with the app.

Today, of course, doing a normal Titanium.DB.install won't overwrite the existing database. The bottom line is, I want it to. Any suggestions to avoid doing all the overwriting of the new database tables manually and just have it overwrite the database en mass?

    The install() method doesn't overwrite and existing database on the device, so what you need to do is delete any existing copy first. Although there would be ways to check if the db exists and overwrite it, if you really wanted to always overwrite with the copy in the resources folder you could simply do this:

    var myDB = Ti.Database.install('myDB.sql', 'myDB' );
myDB.remove();
myDB = Ti.Database.install('myDB.sql', 'myDB' );

    In essence this will either open an existing copy or install one, then remove it. Effectively this is arguably the simplest way to delete an installed copy. Then you just install again, thereby forcing a new copy to get installed.

    For the sake of the Q&A history, let me point out this is NOT normally what you want to happen because it loses any changes made to the database on the device. But that is exactly what the OP requested, and this should accomplish that with minimal effort.
