Installing Android app on device for testing using latest Studio and SDK

Hi,

When developing Android apps I always test on the device. But since the recent Studio and SDK updates, it seems that it runs the app as an intent and doesn't actually install the app for testing, untethered to Studio.

How can I set it so that the app is actually installed, so I can unplug the device?

Thanks.

Ti Studio : 3.2.0.201303312345
Ti SDK : 3.1.0.GA

    Kosso, I've installed SDK 3.1.0GA apps to my Android phone from Studio 3.2. I can run them again once my phone is unplugged from the computer.

    New in this SDK is that Studio will launch the app on device. It never used to do that. It does so by firing a launch intent. That might be what you're seeing

    Are you getting errors? Is the app not in the app drawer? Check Settings -> Apps to see if it's installed.
