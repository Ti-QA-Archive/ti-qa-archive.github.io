Setting width of content of webview

hi,
I'm using webviews in a app where the content of the webview is set to 100%. The problem is, when I resize the webview, the view itself is scaled, but the content is not, so when set from a page fill(100%) to 50%, the webview itself is half a page in width, the content is still the full width, and horizontal scrollable in the view.

The thing is, the problem only resides in IOS, on android the content scales according to the parents width. Is there a way to let IOS behave like Android on this matter?

  • I found the answer to my problem. In the HTML is specified a meta tag viewport in wich I put widt=device-width. Removing this resulted in the content being spread over the webviews width instead of the devices.
