Disabling button while keeping opacity value to its original value

Hi, in an iOS project I noticed that if I code myButton.enabled=false, the button is disabled (and this is ok for me), but its opacity is reduced (as if i coded also myButton.opacity=0.5).

Is there any way to disable the button and keep opacity to 1?

  • Hi

    This is the normal behaviour of the phone and its operating system, to assist your users you should allow this process to remain as it is designed to provide visual feedback to the user that the button is no longer enabled, it also informs the visually impaired (because of the new state) that this button is now disabled.

    You must have a very compelling reason to decide that the visual and actionable state of a button should not reflect the change between enabled or disabled. The visually impaired should not suffer a "look" effect you are trying to achieve without some fall back.

    • Opinions vary, but your use case is stop the user from pressing the button whilst the next window is loading.

      So visually indicating to the user that something shouldn't happen (pressing the button) and something is about to happen (new window) should be done. This is what you would normally do in this situation.

      • Disable the button - so it is obvious they should not click
      • Place a transparent (or semi-transparent) view over the entire current window - so no objects receive clicks
      • Show a progress indicator to information the user has a short wait
      • load and display the window.

      Users get annoyed when things are slow, but react very well to being informed things are still working - perceived actions are an important UI factor for apps of all kinds.
