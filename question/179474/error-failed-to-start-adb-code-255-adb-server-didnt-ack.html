    Hello Ned,

    Thanks for your inquiry. "Error: Failed to start ADB (code 255): ADB server didn't ACK" indicates the ADB version shipped with Genymotion is out of date with the Android platform tools. To fix this issue, manually set the ADB tool to use in the Genymotion settings:

    1. Launch Genymotion.

    2. Click Settings to open the Settings dialog.

    3. Click the ADB tab.

    4. Select Use custom Android SDK tools.

    5. In the Android SDK textbox, enter the path to your Android SDK or click the Browse button to navigate to it.
      Click OK.

      The android path should be like "/Users/MAC.../Documents/android-sdk-macosx"
    6. Rebuild the application.

    Note that if you restart your computer, this setting may not persist in Genymotion and you will need to set the Android SDK path again.

    Hope this helps. Thanks.
