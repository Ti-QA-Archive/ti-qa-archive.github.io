  • Hi

    It looks like you have tried wrapping the sample code that does not have any tabs - with a tabGroup. What you have failed to notice is that did not actually work.

    Ignoring the extra tab for a moment - your app does not show a single tab, it shows a window and only a window. This is where the problem starts. Do not worry I can see how this might have confused you.

    index.js

    $.index.open();

    Your controller says open this id., the id is index. If you look back at your view;

    <Window id="index"

    This id is associated with the window. So when your open line is called THIS window is opened.

    You need to add an id to the TabGroup and then ask that to open instead.

    index.xml

    ...
<TabGroup id="tabs">
...

    index.js

    ...
$.tabs.open();

    This will then open the TabGroup which contains the window. Then as long as you make sure the next tab is contained inside another Tab node with a window below it - all should be well.

    BTW - given this app is running on an iOS device - can I assume you have removed the mobileweb part?
