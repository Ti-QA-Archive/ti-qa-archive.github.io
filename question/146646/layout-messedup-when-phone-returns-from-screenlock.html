Layout messedup when phone returns from screenlock

If the phone goes on the lockscreen and returns on again, the navbar will be shown for a split second and will disappear. Which is normal behavior. In win1 it works fine. My layout has been built 'under' the navbar so when it disappears, everything looks fine.

But in win2 it cause that top of my layout(a view) will be aligned to the bottom of the navbar. (so lets say, 20pixels lower than it should be) So instead the top is the border of the screen, it will be aligned to the bottom of the navbar. The navbar disappears, so you see the gray background of the window (so you see a gray bar with the height and width of the navbar of the phone).

I can use "keepScreenOn", but that won't prevent users lock the screen by them selves.

Because I can see the win2 background, means that the window has aligned itself good(width the border of the screen).

The first view in the window (win2View1) uses the bottom of the navbar as top somehow.

How can I fix this?
