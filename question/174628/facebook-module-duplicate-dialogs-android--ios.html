Facebook module duplicate dialogs, Android & iOS

Hey Everyone,

I am developing my application and currently I'm implementing the facebook module in my application as follow to share a news item:

In app.js: 

var fb = require('facebook');
fb.appid = <MY_APP_ID>
fb.permissions = ['public_profile', 'email'];

In my .js file where I use facebook:

// The variable data is set earlier in this file
var dialog = function() {
  fb.dialog("feed", data, function(e) {
    if(e.success && e.result) {
       alert("Success");
    }
  });
};
var login = function(e) {
   if(e.success) {
      dialog();
   } else {
      alert("Not succesfully logged in: "+e.error);
   }   
};
fb.addEventListener("login", login);
if(!fb.loggedIn) {
   fb.authorize(); 
} else {
   dialog();
}

What I do before this is, in the application I have a list of news items. When I click on an item it opens a new titanium window with the contents of the news item. When I press the share button in the window, it opens a dialog (by executing fb.dialog() to share the news item's content.

After I successfully shared an item, I go back to the list and open an another item. When I press share on that window, it opens the dailog box again. But when I click the X, cancel or share it shows the previous opened dialog as well.

Is anyone experiencing this issue?

Kind Regards,
Patrick van Vuuren

Titanium Mobile
