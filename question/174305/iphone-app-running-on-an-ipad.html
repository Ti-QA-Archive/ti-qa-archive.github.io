  • Hi

    Guessing your frustration is the reason for no code snippets or explaining what happened when you made a test app an do,aced a single image in it? You also have not started Titanium with the Alloy framework or Titanium without the Alloy framework. Neither cause the issue in this case but means I have to provide too different examples - typing code on an iPad is a pain.

    So guessing time. This 'may' be a very simple thing that gets most people to start with.

    Imagine these two code examples - one as Titanium the other using the Alloy syntax;

    • Titanium
    var image = Ti.UI.createImageView({
  backgroundImage: '/subfolder/image1.png',
  height: Ti.UI.SIZE,
  imagebackgroundImage: '/subfolder/image2.png',
  width: Ti.UI.SIZE
});
    • Titanium with Alloy framework TSS file
    "ImageView": {
  backgroundImage: '/subfolder/image1.png',
  height: Ti.UI.SIZE,
  imagebackgroundImage: '/subfolder/image2.png',
  width: Ti.UI.SIZE
};

    So we have two images referenced;

    • /subfolder/image1.png
    • /subfolder/image2.png

    Taking the first image as the example going forward. In your project this should be in any of these locations;

    • Titanium
      • {appname}/resources/subfolder/image1.png
      • {appname}/resources/iphone/subfolder/image1.png
    • Titanium with Alloy framework
      • {appname}/app/assets/subfolder/image1.png
      • {appname}/app/assets/iphone/subfolder/image1.png

    Now we know they are stored in the right locations as your app works on the simulator, so what else could it be?

    Chances are you have mixed up the 'case' of the references in code with the physical location and or file names. So if you image references are all lower case, none of these actual locations will work.

    • Titanium
      • {appname}/resources/Subfolder/image1.png
      • {appname}/resources/iphone/SubFolder/Image1.png
    • Titanium with Alloy framework
      • {appname}/app/assets/Subfolder/image1.png
      • {appname}/app/assets/iphone/SubFolder/Image1.png

    The file system on a physical device is very case sensitive. So for safety ALLOWS make your code references to file names and folder names lowercase, make sure the physical folder names and file names are also lowercase.

    Rename physical file and folders, do not try to have more than one rule - it will bite you. Your tasks now;

    • BACKUP YOUR PROJECT
    • Make those changes
    • Menu > Project > Clean > Choose project > Confirm
    • ONLY IF your project is using Alloy - MAKE VERY SURE IT IS
      • Only then delete the contents of the resources folder
