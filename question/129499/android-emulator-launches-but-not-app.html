  • Check to make sure the app is loading to the emulator.
    Just because the emulator loaded, doesn't mean Titanium is communicating with it.
    I find it's easier to use an actual device.

    Pop open a command prompt (run > cmd) and use 'adb devices' to see if the emulator is being seen, recognized and is online.
    If Titanium won't communicate with the emulator try shutting everything off, disabling any firewall software or antivirus you have running and then run the emulator.
    Once the emulator has loaded, check with adb devices.

    If the emulator is present, then load Titanium, and run the 'install to emulator' option.
    The funny thing about this process is that Titanium will start another emulator and NOT talk to it. You can shut off the new emulator right away, and Titanium will still conduct transactions with the original emulator you started before you ran Titanium.

    It's not the smoothest operation, but I wouldn't recommend using the emulator for more than academic purposes. Meaning to check a callback or behaviour in particular circumstances.

    The emulator will not force close the same way a device will, hence if you only test on the emulator, you may end up releasing a broken app all the same.

    I hope that helped.

      For example, on my computer, my SDK is located at c:\android\android-sdk and the platform-tools sub folder is where ADB is located.

      There are instructions on the Android developers network regarding the proper configuration of your path but if you know where the path variable resides for your version of windows (and how to add one) just do that and you'll be able to access adb anywhere.
