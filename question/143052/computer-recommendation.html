    • 8gb will be fine.

      As for storage, that gets eaten up quickly by any development process.
      If you are using git, you have the overhead from revisions sitting on your drive as well.

      I prefer 512gb-1tb, but the cost is crazy. You can get an external for side storage for a fraction of the cost. Do not build to the external, only use it for archiving stale project.

      Clean up your build directory on projects you aren't actively developing.

      Should be ok.

      — commented October 3rd 2012 by Stephen Feather

    • To give you an idea of space:

      android-sdk: 5.5gb

      android-ndk (optional): 600mb

      development (my titanium projects directory): 3.6gb - 37 projects

      xcode.app: 5gb

      ~/Library/../Titanium: 4.2gb - storage for various titanium sdks, modules ect

      That should give you an idea of usage for a heavily used system.
      Projects that we know we arent going to touch get zipped, and moved to a backup (remember to kill the builds directory before zipping).
