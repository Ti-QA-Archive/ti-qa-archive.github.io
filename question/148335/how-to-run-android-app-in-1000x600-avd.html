How to run android app in 1000x600 AVD

Titanium Studio, build: 3.0.1.201212181159
Platform: Android 2.3.3
Device: Android emulator
Host Operating System: Ubuntu 12.10 x64

Hi!

How do I setup Titanium Studio to run android app on AVD which I created in Android SDK previously?
It's a configuration with 1000x600 resolution using Intel system image for Android v2.3 which should emulate Motorola ET1 N2 7" Tablet - a physical target for Android Application I should write.

If this isn't possible how do I configure such a target from inside of Titanium Studio?
If I see correctly I only have usual handheld screen sizes to choose from when configuring run targets.

Please help!

  • Hi

    You can do this - it is not obvious though.

    • Close the current emulator if it is running
    • In Studio find the Dashboard tab
    • Select the Getting Started button
    • Press the Android button
    • Press the revealed Update Android SDK
    • Android SDK Manager will open, you are not playing with this
    • From the Menu select Tools > Manage AVDs…
    • This window is where you manage your AVDs (Android Virtual Devices)
    • (Tip; when this go wrong with AVDs - deleting them is a place to start)
    • Edit your AVD and assign a new configuration
    • Close everything related to the Android Manager and AVDs

    You then run the app and it "should" run in the new configuration.

    It is possible to create a new AVD and you can even create a new one based on a configuration from the pool added when you install Android updates. Be aware that the name that appears is not always obvious in your Run Configuration

    When you are ready you MUST choose the screen size in the run configuration to the right one, but as I said it is not always obvious. I would suggest you make a note of all the choices BEFORE adding a new one then work out which one is different from the previous list, select that and run.

    I am currently looking at my test app in the emulator based on the Nexus 7 tablet template provided in the Manage AVDs window of templates.

    The only obvious name change in the screen size drop down was WXGA800-7in, specifically the -7in suffix that was different to the similar WXGA800 that already existed.

    Good luck.
