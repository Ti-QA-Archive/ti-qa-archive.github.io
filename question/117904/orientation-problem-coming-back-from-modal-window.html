Orientation problem coming back from Modal window

Hello all, I have a problem with my iPad app. It all works inside a navigation group with forced portrait mode. At some point from the main app window I need to open a modal window in landscape mode. The new modal window opens perfectly but once I close it, the main window is rotated 90°. Mind you, it's not in landscape mode, it's still in portrait mode, simply rotated 90° in a landscape view. I can see 70% of the main window (the bottom is cut) and I have a black background showing on the right of the window.

Anyone can verify if this happens to everybody or it's just bad coding on my part? You just need to create a window with force portrait mode and from there open a modal window with forced landscape mode. Closing it should produce the error.

This is how I'm creating the window…

openwindowbutton.addEventListener('click', function(e)
{

var modalwindow = Ti.UI.createWindow({ modal: true, barColor: '#050505', 
orientationModes:[Ti.UI.LANDSCAPE_LEFT  , Ti.UI.LANDSCAPE ] }); 

var modalview = Ti.UI.createWebView({ url: 'http://www.myurl.ext' });

var close_modal = Titanium.UI.createButton({title:'Close it'});
modalwindow .rightNavButton = close_modal;

close_modal.addEventListener('click', function() { modalwindow.close(); });

modalwindow.add(modalview );
modalwindow.open();

});

Funny thing is, the app still thinks it's in UPSIDE_PORTRAIT mode, because to snap it out of this crooked view I need to put the iPad upside down so that it changes orientation, and then I can rotate it back 180°.

Thank you!
