How to detect high density (HD) android display

During the development process of my cross-platform app, I've noticed a few annoyances when it comes to Android devices. The most recent being that icons created for standard resolution iPhones appear too small on tabs when viewed on my android device. With TI, it's easy to fix this when dealing with retina displays by adding "@2x" versions of my images. Unfortunately, for my higher resolution HD android device, those larger version images are not automatically used.

Now, I've looked throughout this forum for direction on this, and this link is what I've found to be most helpful:

http://developer.appcelerator.com/blog/2010/08/dealing-with-multiple-screens-and-multiple-languages.html

I think that I should be able to style my elements with a jss file, and reference the appropriate jss file when an HD android device is detected. Perhaps this could work with my tabs?

The problem is, I don't know if an automated way to detect high density devices. I suppose I could evaluate the device screen's height and width, and based on some assumptions, determine whether it is using a high density screen.

Does anyone know a better way to determine this?

Thanks in advance - Joel

    Inside your resources folder you want an android folder and inside that an images folder and inside that have a 'low', 'medium' and 'high' folder. Just name the files the same and place your appropriate images in the respective folders and the OS will grab the correct ones.

    To verify this I created 3 separate background images for my views, at 3 separate resolutions and just photoshopped text to the bottom of each image saying if they were low, med or high. Try each resolution in the emulator and you should see the different bg's.

    -John

    • Remove the copies of those images that are in your Resources\images\ folder. (You really shouldn't even have that folder at all.) This way, the files in your Resources\android\images\high (medium, and low) folders will be used (the filename, for example, "70-tv.png" is the same in each subdirectory, even though they contain different size images, right?). Also, make sure that the "android" and "images" subdirectory names are lowercase. Make sure the case of your filenames matches the string in your code exactly. Delete your "build" directory and Clean your project, then run it again.
