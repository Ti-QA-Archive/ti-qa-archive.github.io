App Store Rejects Build: Device Family Issue

Hi all - I'm not an experienced developer, so your help will be appreciated. I have successfully submitted an app to the Apple app store. The build took place in Titanium studio, and Xcode uploaded it successfully. It is, however, immediately rejected for a missing screenshot.

I can see that somehow the build is being submitted for both iPhone and iPad. My target is iPhone only, and that is set in TS and visible in tiapp.xml. So, why is Titanium compiling it otherwise, and what can I do to correct it?

Thanks…very frustrating to be this close but…denied ;-)

    If you open the Xcode project (which you'll find in your build folder), you should see which device targets the project is setting up. iPhone iPad or universal. If you want the app to be only installable on an iPhone, simply select and delete the un-required targets, rebuild and resubmit.

    That should do the trick.
