  • The directories live in Resources/android/images, e.g. Resources/android/images/res-long-port-hdpi.

    You can probably save yourself some trouble if you simplify your resolution-dependent directories and just make them "ldpi", "mdpi", "hdpi" and "xdpi". Otherwise, you have a combinatorial problem, and you're going to have a ton of files to manage. I suppose some things like splash screens may be more important to have landscape and portrait versions, so you could probably selectively use the more specific directories for images like that. But for general buttons and logos, I stick to the simpler directory names.

    You'll reference the images as if they live directly in Resources/images. For example, if you have Resources/android/images/hdpi/foo.jpg, it would be referenced in your code as "/images/foo.jpg".

    I do not use the anyDensity directive. I believe that's for backward compatibility with legacy code (pre 1.7, which in Titanium terms is an eternity ago). If I understand it correctly, it makes display pixels the default unit on Android, which admittedly might make things simpler for cross platform UI design. But the Ti devs decided to make android units default to pixels, so I wouldn't try to fight that – you'll probably make your life more difficult.
