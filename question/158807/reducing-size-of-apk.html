  • Hi

    You will find that in order to provide the base level of functionality for Android that Titanium requires - libraries of code are used. For android those libraries are much larger than those required for iOS.

    An easy way to see the base difference is to create a default alloy project and without changing any code compile each so you can see the original file size of the installable files.

    You should find that once you take your common assets plus platform specific assets into account for each platform your contribution to the app size should be the same (again taking platform specific assets into account).

    This situation has been around for a long time and is unfortunate. However it may change in the future with Ti.Next and Hyperloop (both topics relate to upcoming internal changes to the Titanium system), but this potential size reduction will not be fully known until those projects are fully realised. I would not expect these to be fully available for at least 6 months.

    So for now, your app size for Android looks reasonable given the size of your iOS app once the base differences are taken into account.
