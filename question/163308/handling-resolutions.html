  • Hi

    The best way I have found to handle different resolutions in Android is to use the logicalDensityFactor in order to calculate your elements. It is used as a scaling factor for different displays to allow you to correctly scale your Density Independent Pixel (dip) units. This way you can make your app do all the hard work for you.

    As for images, you should see this question answered by @Malcolm Hollingsworth: images-in-android. The answer given tells you how to properly handle images for different densities, using the correct folder structure.

    Hope this helps!
