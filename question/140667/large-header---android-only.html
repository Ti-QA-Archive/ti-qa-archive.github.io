Large Header - Android Only

It seems like every time I start writing an app, the first window opened on android has a huge header, like three times the size it should be. It doesn't have any text in it either. This seems to happen when I start writing any application. This is how I usually start writing :

var tab, tab_group;

tab_group = Ti.UI.createTabGroup();

tab = Ti.UI.createTab({
  window: Ti.UI.createWindow({
    title: 'test'
  })
});

tab_group.addTab(tab);

tab_group.open(tab);

  • After messing around for a while, I guess the best method is this???:

    1. Create a Tab Group
    2. Create an Empty Window
    3. Create a Tab
    4. Assign the Empty Window to the Tab
    5. Add the Tab to the TabGroup
    6. Open the TabGroup
    7. Create a Useful Window
    8. Attach an event listener on the Useful Window to prevent Android from opening up the Empty Window when the back button is pressed
    9. Call the open method from the Tab to open the Useful Window with an animated property of true
    TabGroup    = Ti.UI.createTabGroup()
EmptyWin    = Ti.UI.createWindow()
Tab         = Ti.UI.createTab({
    window  : EmptyWin
});
TabGroup.addTab(Tab)
TabGroup.open()
UsefulWin   = Ti.UI.createWindow()
UsefulWin.addEventListener('close', function(e) {
    //close the app
});
Tab.open(UsefulWin, { animated : true})
