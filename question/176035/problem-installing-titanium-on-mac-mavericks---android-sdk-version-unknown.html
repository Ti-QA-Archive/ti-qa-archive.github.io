  • If you just downloaded the android sdk today, you can thank Google for once again changing file locations and other things. In this regard, they really are getting worse than apple. It is difficult to built automation scripts let alone the backend for cross platform development with the changes they make.

    In my case, I tend to install everything manually.

    • Start xCode download
    • While waiting, download and install node.
    • Still waiting, use npm to install titanium cli and android.
    • from the cli install the latest Titanium SDK
    • Still waiting, download android SDK (without an ide)
    • Place the android sdk where I want it to be (/sdks/android-sdk-osx usually, blackberry sdk sits along side it)
    • Add the android tools to my path.
    • run the android app, which on a new system, has osx prompt to install the correct version of java I need
    • download the API levels I need and the other tooling
    • (i now have to copy zipalign into the correct directory, thank you google!)
    • download and install virtuabox and genymotion
    • check xcode download, drat, another gig left
      -thanks to another change in android, you will only see fake AVDs (yeah, they mucked those up too) so with taht in mind, and the fact that genymotion is just so much better, thats why we use it. else you ahve to wait for 3.3.0 for the android avd fix.
    • install xcode. download the extra stuff after agreeing to the license.

    • run ti setup and answer the questions, point to the android install location

    • build apps

    • make money
