Events And Layered Elements

Hi

Hope someone could help with what should be a simple problem.

I have a single tap event listener added to a view that is full screen working as a toggle. The idea being when you click it and a submenu isn't already open, it opens up the sub menu. If you click it when the submenu is open it closes it.

All this is well and good, however when the user clicks one of the items in the submenu the event on the fullscreen view also fires triggering the toggle.

Is there a way to block the click on the menu item so it doesn't fire the event on the view below it?

thanks in advance

    Appcelerator states "Events automatically propagate to parent views." (this is one of the "notes" at the bottom of this page. So I think you'll have to keep a variable that tells whether the submenu is open, and in your click event, ignore it if necessary.

  • probably need to see some code Ron this. But I guess it is a layer issue, set the zindex on your pop up menu to greater than the screen view. this will mean that the menu will take precedence.

    If the pop up menu is part of the screen view, then no you won't be able to as the click event is on the whole view so both events will trigger. so put you pop up menu on the window with a higher zindex.

    Hope this helps.
