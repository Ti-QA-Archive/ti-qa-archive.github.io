  • David

    There is no need to launch an app in order to correct validation errors - they should be listed in the "Problems" view of Titanium Studio. If this is not the case for you, ensure that the correct validators are enabled in Preferences / Titanium Studio / Validation / JavaScript. Note that I have both the Mozilla and JSLint enabled in mine, but you may prefer a different combination.

    When you launch your app, ensure that the "Console" view is visible. You may need to use the Console selector drop-down list tool to view the output you are interested in:

  • This might also help:
    After configuring JSLint as already described in the above answers you can fix the not defined global variables errors in two ways:

    • Filter out the errors using a regular expression in the Preferences > Validation section (there is a screen shot above).

    • The bad thing about the first way is that it will still get to the point where it will say "Too Many Errors" because it is only the editor filtering the JSLint responses. The better way is to put the following comment in the beginning of your file to tell JSLint to ignore them.
      &#x2F;*global $$,Ti,Titanium,my_global_1,mb,etc*&#x2F;

    I have done that to make it usable. Hope that helps.
