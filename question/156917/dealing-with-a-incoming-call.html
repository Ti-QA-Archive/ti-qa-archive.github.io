  • As far as I know, using the 'pause' and 'resume' events are the only way to deal with incoming calls interrupting an app. Apple does not provide any other way.

    I have a similar issue with a recording app. I think the only way to deal with the calls might be to simply have the app pause the recording when sent to the background. Or make it a settable option. I haven't decided yet.

    But it's infuriating when a call comes in when doing a recording. ;)

    • All I'll do is set a Ti.Properties boolean when recording or not, then check for that if the app is sent the 'pause' event by an incoming call. If it is, I'll pause the recording, so the user can resume it once they get back to the app.

      If it's not caused by an incoming call, then carry on as normal. The top status bar should show red if it's still recording when paused.

      — commented September 7th 2013 by Kosso

    • Thanks Kosso just tested with the module now and this works great for my needs let me know if it works for you ;)

      var core = require('com.yydigital.coretelephony');
Titanium.App.addEventListener('pause', function(evt){
        var calls = core.getCurrentCalls();
        if (calls == null) {
            Ti.API.info('Paused without call');
        } else { // New Call In        
            Ti.API.info('Paused by call');    
            recording.pause();
        }
});
