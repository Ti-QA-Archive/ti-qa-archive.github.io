Persistent selections in subviews?

I have a navigation group application. In each view, the user makes selections in a tableview and when the next button is tapped, I save the selections to app properties.

http://pastie.org/1784090

The table view is constructed on the fly with a function that builds each row, adding a view with a checkbox background item.

As the items are tapped, I toggle the appearance of this view to display a "checked" checkbox. This all works well.

The problem is, if the user clicks the navbar "back" button, and then returns to the step, of course the table view is built again and the apparent choices are not "displayed". How would I handle something like this?

Note: in the screen shot i have disabled the back button

  • do you want to keep the choices when the go back?

    I would suggest a simple array of objects for each page.

    you push the information on the array when they complete the page and pop it off the stack if they go backwards.

    If you want to keep the information, then you just check the array to see if you have information for the page when you render it. If there is information in the array, then apply it to the page.
