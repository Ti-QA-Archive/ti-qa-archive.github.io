  • OK, so I experienced this problem when I installed the latest version of Titanium on a networked PC, which I think may be relevant, as I did not have any problem when using it on my laptop. I am using the 32 bit version of java with windows 8 64 bit and 32 bit version of nodejs.
    I found a very easy fix that seems to solve this problem. First, I reverted to an older version of Titanium Studio (3.1.0). When I ran it for the first time, the platform configuration dialog appeared and I selected Settings next to the Android platform to customize the installation configuration. This allowed me to browse my directories and manually select the location of the Android SDK. At the end of this process, the red X, became a green tick!!
    Finally, I closed the application and reopened it again and when prompted, I chose to update the current version of Titanium. Now, I have the latest version of Titanium Studio and it recognises the location of my Android SDK.

  • Open the terminal window within Studio Window > Show View > Terminal. From the command line, type android. If your $PATH is set correctly, you'll launch the Android SDK Manager. If not, you'll get a 'not found' message or something similar. Ensure that your-path&#x2F;android&#x2F;platform-tools is in your environment path.

    Also, open the Studio Dashboard Help > Studio > Show Studio Dashboard. Look to the bottom of the page for Configure Native SDKs. Android SDK may have a red X next to it. If so, click the link and ensure that the base API, version 10, is installed.

  • Hi all, I found a fix that worked for me. I had just performed a clean install of Studio 3.3.0 + android tools + Oracle Java under Ubuntu-gnome 14.04.

    What I discovered was that when Studio installed it had created some folders in my home directory as owned by root:root but not given read privileges to my actual account. Once I'd changed ownership back to my own account and set everything else up as below, the error message went away.

    First off, ensure that your environment variables are set properly AND that you have setup java using update-alternatives
    E.g. if Oracle java installed to /usr/local/share/jdk.1.7.0_65 (otherwise adjust path accordingly)

    sudo update-alternatives --install /usr/bin/java java /usr/local/share/jdk1.7.0_65/bin/java 1500
sudo update-alternatives --install /usr/bin/javac javac /usr/local/share/jdk1.7.0_65/bin/javac 1500

    Selected Oracle java as the alternative to use, i.e.

    sudo update-alternatives --config javac
sudo update-alternatives --config java

    Rather than launch TS directly I use a shell script to setup the environment

    #!/bin/bash
export UBUNTU_MENUPROXY=0
export JAVA_HOME=/usr/local/share/jdk1.7.0_65
#TS3.3.0 installs android to ~/android-sdk-linux - change path if you've moved it
export PATH=$PATH:/home/user/android-sdk-linux/platform-tools:/home/user/android-sdk-linux/tools
export ANDRROID_SDK=/home/user/android-sdk-linux
#TS3.3.0 is installed to /usr/local/share/Titanium_Studio - change path if its somewhere else on your system
/usr/local/share/Titanium_Studio/TitaniumStudio
