Error generating R.java from manifest with fresh install

I was having some problems previously with getting Kitchen Sink to run, so I did a fresh installation of the Android SDK. Currently I have SDK Tools, Platform-tools, and Build-tools installed, along with 4.2.2, 4.0, 3.0, and 2.2.

I have already gone through and created links for aapt.exe, and dx.bat from the build-tools folder to the platform-tools folder to resolve earlier errors I was having.

Now, I am stuck with "Error generating R.java from manifest". I am running Titanium Studio as an administrator, and I have made sure that the package name matches the application ID "com.appcelerator.kitchensink".

Considering I imported this directly from the Titanium Studio dashboard, I would assume it wouldn't be so hard to make it run.

Titanium Studio version 3.1.0
Titanium SDK version 3.1.0

Any advice would be appreciated.

  • Hi Kyle

    Hi Praveen,

    Can you try entering this command in the terminal/cmd

    titanium config android.sdkPath <AndroidSDK LOCATION>

    for example mine would be

    titanium config android.sdkPath /Applications/AndroidSDK

    And try installing the latest 3.1.X SDK from this address as it is required to support the version of the Android SDK that you are using.

    To install the SDK download it then Click on help in titanium -> Install Titanium SDK then select the location of the SDK.
