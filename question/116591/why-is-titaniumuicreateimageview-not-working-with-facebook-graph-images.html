why is Titanium.UI.createImageView not working with facebook graph images

I am trying to retrieve the user's facebook image using the graph and createImageView. It gives an empty image.

var fbpix =  Titanium.UI.createImageView({image:'http://graph.facebook.com/'+Ti.Facebook.uid+'/picture', width:50, height: 50, top:4,right:4});

win1.add(fbpix)

Please, what can I do?

  • that request returns a 302, which means you need to look in the request header to find the location/URL of where the actual image is located.

    Looking at the HTTP Response Header, from a Graph API call https:&#x2F;&#x2F;graph.facebook.com&#x2F;btaylor&#x2F;picture

    Cache-Control:private, no-cache, no-store, must-revalidate
Content-Length:0
Content-Type:image/jpeg
Date:Sat, 26 Feb 2011 17:55:32 GMT
Expires:Sat, 01 Jan 2000 00:00:00 GMT
Location:http://profile.ak.fbcdn.net/hprofile-ak-snc4/173665_220439_5054139_q.jpg
Pragma:no-cache
X-Cnection:close
X-FB-Server:10.28.10.114
X-Powered-By:HPHP

    As you can see the image is specified by the location property on the response
