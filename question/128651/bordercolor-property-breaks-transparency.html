BorderColor Property Breaks Transparency

It appears that the borderColor property is breaking the transparency of objects. Below is an example where simply adding the borderColor property prevents the button from being transparent as specified by the opacity value. If that value is commented out, the button is displayed as expected.

var win = Ti.UI.createWindow({ backgroundColor: '#fff' });
var lbl = Ti.UI.createLabel({ text: 'test' });

var btn = Titanium.UI.createButton({
   borderColor: '#FDB913', //transparency works if this line is commented out
   backgroundColor:'#005288',
   width:200,
   height:200,
   opacity: 0.5
});        
win.add(lbl);
win.add(btn);
win.open();

Is this a known issue, and if so, is there any workaround? I'm experiencing this problem both with Mobile SDK 1.7.5 as well as 1.8.0
