how do I sync local data with online data without downloading or uploading the entire database?

I have plans to develop an app that runs on multiple OSs as well as on the web. If the user installs the app on multiple devices (ipad and iphone) and also uses it on the web, I run the difficulty of how to store their answers (or update their data in general).
If the user is storing a list of items, and they add 20 items online, then when they install the ipad app it will download all 20 items (via an API). But if they add 2 more items and modify 2 items, how would the app know to download the data on JUST those 4 (because downloading/uploading the full list of 22 each time would be a waste).

The only solution I can think of is to add a timestamp to each row, and the app will store the last time it asked for an update (and send that along with the API call). Is there an easier/better way to do this?
