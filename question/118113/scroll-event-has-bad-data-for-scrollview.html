Scroll Event has Bad Data for ScrollView

Attaching a tableview to a scrollview forces the scroll event for either of them to have the same data.

Code

tableView.addEventListener('scroll', function(e) {
    Ti.API.info('TableView Scroll e: '+JSON.stringify(e));
    Ti.API.info('TableView Scroll this: '+JSON.stringify(this));
});

scrollView.addEventListener('scroll', function(e) {
    Ti.API.info('ScrollView Scroll e: '+JSON.stringify(e));
    Ti.API.info('ScrollView Scroll this: '+JSON.stringify(this));
});

Output

[INFO] [16,1112] TableView Scroll e: {"size":{"width":320,"height":569},"firstVisibleItem":0,"totalItemCount":21,"visibleItemCount":12,"type":"scroll"}
[INFO] [3,1115] TableView Scroll this: "[Ti.UI.TableView]"
[INFO] [6,1138] ScrollView Scroll e: {"size":{"width":320,"height":569},"firstVisibleItem":0,"totalItemCount":21,"visibleItemCount":12,"type":"scroll"}
[INFO] [3,1141] ScrollView Scroll this: "[Ti.UI.ScrollView]"

If someone can confirm this is a bug I will post a ticket.

  • Joe

    Be aware that a tableView is simply an enhanced scrollView. From a usability perspective, it is important that a scrollView has a stable "foundation", otherwise the user experiences one view scrolling around on top of another view that is also scrolling around. Thus, a scrollView should not be added to a scrollView and, it follows that, a tableView should not be added to a scrollView.

    I hope this makes sense

    Cheers

  • A Workaround

    Add the tableView to a scrollView. Set the height for the tableView, which can be computed by adding up the row heights. Make the scrollView have a higher zIndex than the tableView.

    Then, the scrollView scroll event tracks correctly.

    See the gist of my Tweetie-like pull to refresh for Android for an example.

    This, however, does not solve this bug–perhaps I should make a ticket. Also note what Paul says about the stability of this workaround. So far I've experienced no issues.
