Titanium Studio - Save password/Auto login

In Titanium Developer, the application would automatically save your credentials and auto log you in. When I upgraded to Titanium Studio, the program forces me to enter by username and password every time it starts up. My co-workers instance of Studio auto logs them in as expected.

Does anyone know how I might go about correcting the settings so that my instance of Studio saves my credentials and auto logs me in?

Thanks!

    • Hi Michael. On OS X, look in Applications -> Utilities and open the program called KeyChain Access. In the Category Pain (lower left on my computer), select All Items. Then, search for equinox.secure.storage using the search bar in the upper right of the window. Open the resulting item by double clicking on it. There should be two tabs in the resulting window, click on the one labeled Access Control. Click the radio button next to "Allow all applications to access this item" and click Save Changes.

  • I am running TitaniumStudio on OS X. One machine auto logged in and the other did not.
    This is how I fixed the one that didn't.

    This issue was with the eclipse secure data storage. I don't think it was updating it.
    Anyway, this is how I fixed it.

    First I ran TitaniumStudio using the following command (from the Terminal) and logged in as normal:

    /Applications/Titanium\ Studio/TitaniumStudio.app/Contents/MacOS/TitaniumStudio -eclipse.keyring ~/secure_storage

    When I exited, it created a new secure store file for me at ~/secure_storage.

    I made a back up of the existing default secure storage file (just in case):

    mv ~/.eclipse/org.eclipse.equinox.security/secure_storage ~/.eclipse/org.eclipse.equinox.security/secure_storage.old

    Then copied the newly generated one to the right location:

    mv ~/secure_storage ~/.eclipse/org.eclipse.equinox.security/secure_storage`

    Started up again and it just worked!

    I hope that helps.

    David
