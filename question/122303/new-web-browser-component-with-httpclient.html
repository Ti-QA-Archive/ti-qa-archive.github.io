New web browser component with HTTPClient

For some reasons, I've decided to implement a web browser component using HTTPClient connections. What I want is something like a webView, but more customizable and with low level functions (change useragent, access to cookies, and so on).

Connections would be managed with this new component and I would use the Titanium webView component only for rendering the HTML code.

I want to do it only using Titanium (not native modules). I think that it is possible to do, but if you think there is some good reason to don't do this, please tell me now (or forever hold your peace) :D

If someone finds this interesting and wants to participate actively, we can create an open source project on git.
