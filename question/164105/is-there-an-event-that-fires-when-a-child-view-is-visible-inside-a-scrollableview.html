Is there an event that fires when a child view is visible inside a ScrollableView?

I need to perform actions on views nested inside a scrollable view when the view scrolls into visibility, but I'm having a tough time finding an appropriate event to do this.

So far I've tried using the ScrollableView's scrollend event, then using the index of the current page to access that child, but for some reason, calling getChildren() on my ScrollableView returns no children. E.g., if I log children, I get:

  currentPage = e.currentPage;
  activeView = $.scrollableView.getChildren()[currentPage];
  Ti.API.info activeView;

I get <null> in the log.

My Alloy view looks like this:

<Alloy>
  <Window id="setup" class="container">
    <View id="pagingControl" layout="horizontal" height="20dp" />
    <ScrollableView id="scrollableView" showPagingControl="false"
      scrollingEnabled="false" >
      <Require type="view" src="welcome" />
      <Require type="view" src="login" />
      <Require type="view" src="setupWorkAddress" />
      <Require type="view" src="leaveWorkAt" />
      <Require type="view" src="chooseContact" />
    </ScrollableView>
  </Window>
</Alloy>

Any suggestions? (Thanks!)
