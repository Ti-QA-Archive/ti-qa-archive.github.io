    Hi

    The images for retina should be suffixed with '@2x', so if you had image.png then it would become image@2x.png

    For non retina, simply half the size of your imagery and remove the suffix.

    The images should be placed in the same location.

    Hope this helps

    • Yes, you should have two versions of each image, for example if we take 1024 * 768, then in order to support the iPad 2 as well as the iPad 3 and higher you would need to provide both retina and non-retina images.

      • image.png = 1024 * 768 (for iPad 2)
      • image@2x.png = 2048 * 1536 (for iPad 3 +)

      If your designer has given you images suitable for the 2048 * 1536 example, you will name these to include the '@2x'. However you will also need the non-retina images, so either reduce each image yourself by half or ask your designer to provide them as such.

      1024 * 768 is correct for iPad2 and is also the correct measure for Device Independent Pixels (DP) as one pixel = 1 dp, whereas the iPad 3 and higher have 2 pixels per dp. In your app's you should only consider dp's when measuring left to right or top to bottom and your images will display perfectly.

      The platform width and height represent the devices dp, so what you see in your alert is correct.

    • I believe your actual problem wasn't covered in the other answers. If your image is named "myImage@2x.png", then in your code you want to reference it as "myImage.png". iOS automatically grabs the file with the "@2x" suffix for retina devices.

      After that key clarification, the rest of the answer above is correct.
