Store GPS location in a variable

Hi,

I have a problem when I want to store the longitude and latitude within a variable.

my code is:

var longitude_;
var latitude_;
Titanium.Geolocation.getCurrentPosition(function(e){
    longitude_ = e.coords.longitude;
    latitude_ = e.coords.latitude;
    // First alert dialag
    var al = Ti.UI.createAlertDialog();
    al.setMessage('latitude: ' + latitude_ + " , longitude: " + longitude_);
    al.show();
});
// Second alert dialaog
var ale = Ti.UI.createAlertDialog();
ale.setMessage('latitude: ' + longitude_ + " , longitude: " + latitude_);
ale.show();

The problem is that on the First alert dialog the content of the variables are shown correctly but on the Second alert dialog the content of the variables is undefined

Do you know how can I store the location outside getCurrentPosition method?

Thanks!

PS: I am using Titanium SDK 1.7.5 with Android SDK 2.2

    Geolocation.getCurrentPosition is asynchronous, so the callback function where longitude and latitude are set to e.coords.longitude/latitude isn't yet completed before you try to access those values in your 'second' alert dialog.

    By putting the 'first' alert dialog inside the callback, it is correctly getting the lat/long since it isn't being run until the getCurrentPosition has a response. You can code it that way, or use custom events to handle notifying things when the lat/lng value is set.
