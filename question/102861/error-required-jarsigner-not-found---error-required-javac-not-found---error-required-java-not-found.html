  • follow that steps:

    1. if you windows, open the CMD window, and write SET
    2. search for the enviroment variable JAVA_HOME
    3. if this not exist, you could create.
    4. Go to "My Computer", rigth click and "properties".
    5. Goto "Advanced Options" Tab.
    6. Click on "Enviroment Setting" Button.
    7. Click on "ADD NEW" Button, and write JAVA_HOME, for the name and the Path to javac.exe file.
      Important, if you not know the path for javac could do a seach in the C:/ drive and copy the path.
    8. Normally the javac are in c:\program files\java\jdk1.7.0_09, (the version number could be diferent, you must use the major version number if are more than one JDK folder)
    9. Save the changes.

    For test is the configuration it's correct:
    a) open the CMD windows.
    b) write CD %JAVA_HOME%
    c) if you are in the directory java, are all correct.
    d) if you receive an error message, verify the correct path.

    Test in the titanium studio.
    i) open your project.
    ii) goto menu "Window", "Show View", "Terminal"
    iii) Then write: echo $JAVA_HOME
    iv) you could obtain the correct path home for java. if not is correct could restart you titanium, or check the path in the windows enviroment settings.

    NOTE: the Titanium could don't find the correct path because the windows use the large format for your paths.
    then you must obtain the short path,

    1. open the CMD window
    2. goto c:\ (cd c:)
    3. write dir /X /AD (obtain only folders, with short format), typically "Program Files" is showing as "Progra~1" or similary.
    4. Try with cd c:\Progra~1 to in the folder.
    5. in the Java folder
    6. write dir /X /AD and look for the short name for JDK folder.
    7. change the windows enviroment setting put all the short path. save the changes and restart the titanium
    8. realize the text for terminal. if return the correct path you could compile you app.

  • Note that you don't need to paste each line of the diagnostics commands in separately - just copy and paste the whole lot into the windows command line window, as the instructions explain.

    Regarding the information you have pasted - it's difficult to determine whether your system is configured correctly, because you have edited the output and hence I am not sure whether you have introduced the errors or they actually exist. If the information is correct, then I would advise the following:

    • the path value C:\Pro gram Files\android-sdk-windows\tools contains a space in the word "Program" that should not be there
    • the path to python is not in your system path
    • as Vadb is not a valid argument for python, the command has failed. Thus, it is not possible to determine whether the correct version of python is installed and properly configured
    • the output from adb version is missing, so it's not possible to determine whether adb works properly
    • the output from android list target is missing, so it's not possible to determine whether you have the correct android packages installed
    • the variable JAVA_HOME is not configured correctly. It should be set to C:\Java\jdk1.6.0_23

    Try to resolve these issues, and then follow the diagnostics instructions again to verify your configuration. Post the output to gist as per the instructions.
