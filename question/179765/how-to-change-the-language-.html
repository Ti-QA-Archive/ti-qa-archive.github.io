    Hi Hussein,

    You can create by two ways.

    Method 1: Using titanium localization

    Method 2: Using custom localization. according to this you can create separate language files depending on your requirement. this all language file having key value pair. and assign this file reference to one global variable and just change file reference on language change for example.
    e.g

    you want two language english and arabic then create two files english.js and arabic.js

    english.js

    exports.language = function() {
    return {
        appTitle : "SOME VALUE FOR ENGLISH"
    };
};

    arabic.js

    exports.language = function() {
    return {
        appTitle : "SOME VALUE FOR ARABIC"
    };
};

    for example initially you set english language as selected language in alloy.js

    Alloy.Globals.selectedLanguage = Alloy.createController(&#39;english&#39;).language();

    and after you can easily change the reference of language file on click event of any button from any where

    $.btnLanguageChange.addEventListener('click',function(){
    Alloy.Globals.selectedLanguage = Alloy.createController('arabic').language();
});

//AND USE IT SIMPLE

$.lbl.text = Alloy.Globals.selectedLanguage.appTitle;

    Hop this is helps you. :)

    Thanks
