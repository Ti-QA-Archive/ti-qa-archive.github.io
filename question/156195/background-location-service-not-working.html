Background Location Service Not Working

  • Application Type: Mobile

    • Titanium SDK: 3.1.2GA

    • Platform and Version: iOS 6.1.2

    • Device: iPhone 4S

All,

I have been trying in vain to get background location working on my phone again. This was working prior to upgrading both iOS and Titanium, so I am not sure which is the culprit. I am using the most current version of the Geoloqi module, which was working just fine. I don't think the module is the issue as I cannot get background to work in test apps either.

So far I have

  1. Added background mode to the info.plist file

  2. Copied info.plist into the root of my app folders

  3. Added background mode to the tiapp.xml file

  4. Messed around with significant change mode

Nothing is changing. I get a few location updates, perhaps 10 minutes, after the app is backgrounded and the phone is locked, then nothing until I start the app in the foreground again.

Anyone have any suggestions?

Thanks,
B

  • I'm certain that by default iOS will not allow a background service to run for more than 10 mins. That is why you get your updates, then it stops until you bring the app to the foreground again. I don't believe there is a way to extend that background time.
