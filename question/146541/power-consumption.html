Power consumption

Hi,

I am building an App (ios to start). The App is a standard hotels listings, 3 tab view, map with pins and sqlite db.

One concern I have is the power consumption of the App.. are there any things to consider?

https://wiki.appcelerator.org/display/guides/Managing+Memory+and+Finding+Leaks
This wiki talks about removing views when not needed… however I am currently converting the app to the Alloy framework, is this still needed?

  • Well, power consumption is not directly related to the content of that guide (memory leaks). Apps that make intensive use of Geo-Location and network related operations tend to consume more power so the answer is to not keep those operations active when not needed

    The basic rule is that if you use best practices, then you shouldn't have to worry about things like this unless you are creating an app that specifically has features that you are concerned about. Of the hundreds of hotel listing apps out there, Im sure none of them were concerned with power consumption.
    As Nick mentioned above, memory leaks and power consumption are not related in the slightest in this context. If you are using Alloy and/or CommonJS properly, then memory leaks should not plague your app.
