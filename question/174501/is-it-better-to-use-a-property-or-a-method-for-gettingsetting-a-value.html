Is it better to use a property or a method for getting/setting a value?

Is there any kind of difference between the two systems? For example, for getting a value from a text field is better this

a = textField.text;

or this

a = textField.getText();

Thanks for attention.

    Hi

    Yes there is a difference - speed. Does it matter, almost certainly not - unless you hundreds of them all being retrieved at the same time.

    This is a very simplistic explanation of what happens internally - for example purposes only.

    via Property

    // your code
a = textField.text;
// Titanium returns the answer using this sort of code
return textField.text;

    via Getter

    // your code
a = textField.getValue();
// Titanium returns the answer using this sort of code
function getValue() {
  return textField.text;
}

    So requesting using a property uses less code internally than using a getter. Thus the property way is faster due to less code being used. Whilst the "less code - is faster" rule works here - it is not always true.

    I ONLY use getters and setters. This is a convention I developed many years ago, back when Titanium was less evolved than it is now. There were some early bugs that meant certain properties dis not act as they should 'some of the time'. A good example was the label, depending on how it was created; using visible = true and visible = false would not always show or hide the label. Even today there is an issue with the view that can means that the only way to show and hide the view without concerns it could fail occasionally. Which is why I always use the methods; show() & hide() on views.

    The choice however is a personal one, these days I simply find getters and setters easier to determine their goal without even reading what it is getting or setting.
