Google Maps Annotation Image in Titanium 1.7.3 SDK

I don't know if it's 1.7.3 SDK specific, but that's the version I'm using. So I'm trying to use custom pin in Google Maps.
For that I create an imageView:

imageView = Titanium.UI.createImageView({
    image:'http://www.appcelerator.com/wp-content/uploads/2009/06/titanium_desk.png',
    width:261,
    height:178,
    top:20
});
A2B.win.add(imageView);

And image appears, so it's a valid image and path is right.

But when I try to use it as pin for Google maps:

var pin = Titanium.Map.createAnnotation({
    latitude:latitude,
    longitude:longitude,
    title:title,
    subtitle:subtitle,
    animate:true,
    image: imageView
});
A2B.mapview.addAnnotation(pin);

Pin doesn't appear, and for each missing pin there is an entry in console

D&#x2F;skia ( 537): --- SkImageDecoder::Factory returned null

Same thing happens with local files.

And if I don't use an imageView, and simply enter image path as string, it says

D/skia    (  644): --- decoder->decode returned false

Is there any undocumented specification, like image format, size, etc which is needed for annotation or is it simply a bug, or am I doing something really wrong?

    Hi,
    images in mapviews are not imageviews. Thatswhy the images must reside local. In other cases you have to cache it local in app folder. The size leave origin => you have to resize with imagemagick or with other tools.

    This are my experience in versions < 1.7.3 …

    latitude:parseFloat(Lat), //works
    longitude:parseFloat(Longitude), //works
    image:'../icons/marker.png', //*does* work in Android! Creates drop-shadow automagically
  });
