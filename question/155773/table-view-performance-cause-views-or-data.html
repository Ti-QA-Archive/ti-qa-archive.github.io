Table View Performance Cause: Views or data?

I have a problem with a tableview that gets quite unresponsive when I have more than 40 rows. The tableView includes rows with a lot of varying views and event listeners. A row can include 40 image views, 40 labels, 100 views and 40 event listeners on these views. The row has also some data that I need to be accessible when the row is clicked (10-40kb).

What is more likely to be the cause of the performance problem: the amount of data that a row holds or the views that need to be rendered each time?

iOS iPad, Titanum 3.1
