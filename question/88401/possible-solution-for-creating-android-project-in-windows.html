Possible Solution for Creating Android Project in Windows

I know a lot of people have had various reasons for not being able to create a mobile project for the Android SDK, so here was my resolution, hope this helps someone.

I'm using Windows 7, and was trying to get it to work all yesterday (very frustrating) and finally gave up, this morning I loaded Titanium and downloaded the update to V1.22, then the project error changed to "import site failed use -v for traceback" which is an error I see on the Blender 3D software for Python, so I downgraded Python to version 2.5.5 and reset the windows environment path to the Python root folder and it worked.

Also installed all the Google Api's in the SDK Manager.

— asked December 11th 2010 by Jonathan Willis
