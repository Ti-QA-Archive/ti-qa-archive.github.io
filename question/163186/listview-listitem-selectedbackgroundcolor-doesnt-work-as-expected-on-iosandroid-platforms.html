ListView ListItem selectedBackgroundColor doesn't work as expected on iOS/Android platforms

I am using Titanium Appcelerator build 3.2.1

I'm currently converting my code from a TableView to a ListView, and almost have everything working as expected. However, I can't seem to figure out how to have the immediate click feedback color change work as expected when setting selectedBackgroundColor.

On iOS, it stays selected (which is not what I want.) On Android, it only stays selected when a press event is active (which is great), but it only works when pressing on a part of an item that doesn't have any content on it. For example, if you press on an image (in a custom templated ListItem), then nothing happens. On iOS, you can click anywhere within the ListItem, but as I mentioned, the background state stays selected until you press another ListItem.

Eventually, I'm going to want to control whether a ListItem should stay selected or simply indicate a press for each platform, but at this point, I can't seem to get either way working consistently on both platforms at the same time.

As I'm using templates, etc. and there is a lot of code to make ListViews work, I'm not pasting it all here, but you can see the part where I'm creating the ListView and adding the selectedBackgroundColor in the ListSection.

    customTable[tableID] = Titanium.UI.createListView({
        top:tableTop+1,
        width:rowWidth,
        templates: { 'program': programListItemTemplate },
        defaultItemTemplate: 'program',
        left:slotID*subViewWidth,
        headerView:customTableHeader[tableID],
        separatorColor:'#888888',
    });

       // Sets the regular list data properties
        properties : {
            itemId: 'row' + (i + 1),
            selectedBackgroundColor:'#cccccc',
            selectedColor:'#cccccc',
            height: Ti.UI.SIZE,
            accessoryType: Titanium.UI.LIST_ACCESSORY_TYPE_DISCLOSURE,
        }
        });
    }

    listSection = Ti.UI.createListSection({items: data, headerView:blankPageSectionView});
    customTable[tableID].sections = [listSection];

    blankScreenView.add(customTable[tableID]);

    Try to set touchEnabled: false to the sub-elements (eg the image).
