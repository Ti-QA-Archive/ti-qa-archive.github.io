Custom Fonts in iOS Alloy.

I got trouble to set custom fonts in Alloy project specially in iOS platform for any other font other than default font. Android is running without any modification by same code.

Using:

####Mac OS X 10.8.2 (12C3103), Titanium Studio build: 3.0.2.201302191606, Titanium SDK Version: 3.1.0.v20130301200224, Xcode Version 4.6 (4H127)

Using latest Titanium SDK taken form http://builds.appcelerator.com.s3.amazonaws.com/index.html#master

Using simple project that demonstrate a Label demo. fonts folder placed in app > assets > fonts. select any suitable fonts for testing.

index.tss

".lblLabelDemo": { 
    font: { 
        fontFamily : 'Neo Tech Alt',
        fontSize : '25dp'
    }
}

index.xml

<Alloy>
    <Window class="container">
        <Label id="label" class="lblLabelDemo">Custom Font Demo !!!\n Please run this demo in iOS...</Label>
    </Window>
</Alloy>

I noticed that It compiles with alloy and info.plist file generated in the build folder for iphone, but there is no any entry or tag in the file. there is no any entry in the plist file, even if we put plist setting in the tiapp.xml file in to the project.

When we add "Fonts provided by application" key to info.plist file then It shows fonts in simulator, but it replace default plist file each time when we compile project from titanium alloy. There is no any key entry remaining in the new plist file.

Am I missing something or there is any problem in the project?
