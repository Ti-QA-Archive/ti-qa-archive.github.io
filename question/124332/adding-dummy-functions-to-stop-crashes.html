Adding dummy functions to stop crashes

We are coding on MAC OSX 10.6.8 with Titanium Studio SDK 1.7.2 for iPad running iOS 4.0 to 4.3.

We have a project with about 12 javascript files, each up to 1000 lines long. Often the app will crash, with little debug info apart from that in XCode organiser. When this happens, we either add or remove some of the following (where XX is a random number)

function test_XX(){}

to app.js, or at times into the other .js files.

A recompile (or several with different numbers of dummy functions) eventually gets the app running again.

Often the app will run in the iOS Simulator in debug, but not in normal simulation mode, or will run on the device but not on the sim. And various combinations thereoof.

There are 4 developers and we are all finding the same issues.
We have other apps that are fine. We have been using the suggested scoping of functions:

AppName.UI,Module = {};
(function(){
// do stuff
}();

  • We are now planning to move away from Appcelerator, reasons being:
    The unreliability on execution, and in particular the complete "opaqueness" of stack traces on the crashes.
    Lots of intricate SQL database processing needed in the app (> 100 tables).
    Slow execution speed when loading up > 100 images in a scrollable grid.

    We really like the speed of development and cross platform portability. We would really like to avoid the effort of re-coding in Objective-C.
