Best way to display more then 20 k table view rows in the table view.

Application Type : Mobile (iOS and android)
Titanium SDK : 3.0

We are getting performance issue on loading 20,000 table view rows in the table. Its taking too much time to display the table view rows in the table.

I have created the custom row which have title and its detail text of one line. Here is my code of row.

<Alloy>
<TableViewRow id="row" className="Custom">
<Label id="title"></Label>
<Label id="desc"></Label>
</TableViewRow>
</Alloy>

Its taking around 25 seconds to display these rows in the table.
We don't want the load more feature. We want to display all data in one go.

Any solution to improve performance ?

Thanks in advance !!

  • hi,

    You can use className property of tableview row, this will do lazy loading of table view, means it will reuse the previous row, and this in turn will increase the performance of tableview.
    There is a built in caching method you can invoke by using the property className when you define a row. This tells the OS that it can re-use this row style and just swap data. You should use this on all rows that you add any views to. Also if you have more than one style of presenting the row then have more than one className in the table.

    Example

    var row = Ti.UI.createTableViewRow({
...
    className: 'standard',
...
});
