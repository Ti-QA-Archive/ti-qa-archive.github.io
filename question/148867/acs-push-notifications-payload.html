ACS Push Notifications Payload

Ok. I'm trying to send a push notification payload through the NodeJS CocoaFish module. If I send the payload as a string, for instance
{payload:&#39;test from server&#39;, channel: &#39;testing&#39;}
it sends just fine. But, if I try to send the following custom payload so that I can have a default sound {payload:{alert:&#39;test from server&#39;, sound:&#39;default&#39;}, channel: &#39;testing&#39;}, I get the following error back from ACS: 

{
  "meta": {
    "status": "fail",
    "code": 422,
    "message": "Missing fields. Required: payload",
    "method_name": "Notify"
  }
}

Anyone have any insight to how to remedy this?

Thanks in advance.

    • http://docs.appcelerator.com/titanium/latest/#!/api/Titanium.Cloud.PushNotifications
      it should be something like this.

      ACS.PushNotifications.notify({
    channel: 'friend_request',
    payload: 'Welcome to push notifications'
}, function (e) {
    if (e.success) {
        alert('Success');
    } else {
        alert('Error:\\n' +
            ((e.error && e.message) || JSON.stringify(e)));
    }
});
