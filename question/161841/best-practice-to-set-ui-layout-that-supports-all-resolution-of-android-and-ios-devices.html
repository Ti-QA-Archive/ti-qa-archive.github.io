  • Hi

    Short answer; there are no best practices (short cuts) to support all devices and all resolutions - simply too many - especially when you include everything Android.

    Longer answer

    If you are building an app and you care about how it looks on multiple devices you need to spend time working out how you are going to design the app and how it will work across multiple aspect ratios as well as platforms and screen sizes.

    I have answered some similar questions in the past which will be helpful in part for this question.

    • How to achieve same layout for Android and iPhone using Titanium
    • images in Android (also iOS)

    But you have to realise that ranging apps from 240x320 (320x480 if we are being sensible all the way up to and beyond 2560x1600 cannot be done without serious thought and planning.

    • @Alberto - I am confused by your comment on dp as an unsuitable unit of measurement.

      40dp (or much better - tweak the tiapp.xml system unit value to dp) will be the same on all devices - dp takes into account the scale factor so a smaller device will appear differently to a tablet but the content space is the same.

      We use dp for all apps on all devices on all aspect ratios on all platforms all from the same alloy TSS files - all look great - no tweaking required.

  • Not sure if this is the best practise, but that's what I do:

    I usually divide the styling for 4 different targets:

    • android phone
    • android tablet
    • ipad
    • iphone

    It's necessary because there are big differences betweeb the four of them, not everything usually works using simple dp.

    I create a style sheet for each page, for each target:

    androidPhone \ homepage.js
androidTablet \ homepage.js
iPhone \ homepage.js
iPad \ homepage.js

    in the styling file (homepage.js) I have something like this:

    var theme = {
    labelText : {
        height : "150dp"
    },
 ...

module.export = theme;

    Then in the page I want to skin I get the device:

    var device = deviceUtils.getDeviceName();

    then get the right styling files, and apply the styling with _.extend

    var label = Ti.UI.createLabel(_.extend(theme.labelText,{
...

    this will ensure that I get the right dps no matter what kind of device I use.

  • I totally agree with Malcolm here but here are my two cents anyway.

    Supporting all form-factors of both platforms is one of the biggest complications in cross-platform development. I think implementing the UI is that part of Titanium development where you stop to work on a single code base. It is not a simple configuration thing.

    As Malcolm said, there is no real "best practice" but there are a few things I experienced that became part of my personal best practice.

    When we developed an App for iOS and Android that should also support tablets we started to design multiple versions of each part of the UI. As mentioned above, single code base works better for the logical part of your app turns out that the first point in my best practice is splitting UI and logic (MVC).

    I also adopted platform-specific navigation patterns. Sure, there were a lot of UI elements that works fine on every platform, especially contents. Creating re-usable and modular UI components is another best practice for me. Alloy is a great help at this. You can work with widgets, separate re-useable stuff in controllers, decide wich to use on what platform, form-factor etc. Alloy is awesome! When doing it right, you could start developing for one and add views for other platforms or form-factors later.

    Another technique that is definitely a best practice, primarily on Android, is using density dependency. That means using an abstract unit dp globally wich will be converted to the same physical size on target devices (both, iOS and Android). You will find good answers in this Q&A about this topic. (Also for using font sizes)

    When it comes to graphic assets, there are solutions to provide multiple images for every screen (link in Malcolms answer).

    I'm sure there are quite a lot of other things that I forgot to mention. There is no secret recipe, no universal solution for this. There are so many things to care about, so many banana skins. I hope the essence out of those answers you got here are a starting point.
