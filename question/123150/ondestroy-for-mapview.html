onDestroy() for mapview

Hi All,
Anyone know about onDestroy(). Is it still can use to release mapview?? I have notices that method from this
http://developer.appcelerator.com/question/46371/error-in-second-window-with-mapview

Unfortunetly, when I use onDestroy(), I got the error message as "Cannot find function onDestroy"

So, how can i use that one??? Please help me :)

    onDestroy() no longer exists for the MapView. The question that you referenced is over a year old now and the method is no longer supported. You can attach a map to an open window and simply close the window to release the mapview.

    • You can't close a tab window without closing the app, therefor, you have to attach the map to a window that is not attached to a tab. You can, for instance, add a button to that tab that opens a new window containing the map. This way you can close it, without closing the app.

  • Keep in mind that for Tweetanium style approaches, which have 1 window which shows/hides views instead of windows, this will not work. If you attach a mapview to a view, and then remove that view from the window or set the view or the mapview to null, the mapview will still exist in memory so when you create another mapview in Android, it will give you the "only one" error.

    Basically you have to use windows for each "view" and make sure you close the last window with a mapview before opening the new one. Not fun for apps like mine which have 3-4 windows with mapviews on them, but doable.
