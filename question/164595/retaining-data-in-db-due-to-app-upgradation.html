Retaining data in DB due to app upgradation

We were successfully able to post our first app to the AppStore last month. Now that we will be readying ourselves for an app upgrade in a month i am looking for a strategy where the user will be able to login automatically even if he upgrades the app. I am thinking of maintaining a version number and then do a compare with my local DB and do the necessary alterations. Is this a feasible approach that i am taking? I need to retain the previous data if the version is same and not otherwise. Could you please refer a link where i can read more about such techniques as to how normally database migration is handled due to app upgradation scenarios for iOS? Thanks

  • What if i wanted to make alterations to the existing DB tables? should i take the versioning approach?

    With Alloy models, you have the migrations that will help you out.
    http://docs.appcelerator.com/titanium/latest/#!/guide/Alloy_Sync_Adapters_and_Migrations-section-36739597_AlloySyncAdaptersandMigrations-Migrations

    If you did not use Alloy, then you will need to either

    • create a new database, create the new tables, open the old database, read the data, massage it, save to new database, delete old database.

    • rename old tables, create new tables, read in the data, massage it, save to new tables.
