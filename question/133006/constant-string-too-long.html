Constant string too long

Hi,

I'm trying to package my latest Android app for distribution, I've never had problems till today. I'm using 1.8.2 and V8.

During packaging I get the following error:

[ERROR] Error(s) compiling generated Java code
[ERROR] C:\Documents and Settings\[User]\My Documents\Titanium Studio Workspace\[app]\build\android\gen\com\[name]\test\AssetCryptImpl.java:19: constant string too long

assets.put("app.js", "then 81,060 worth of alpha-numeric characters");

Anyone have any idea on how to resolve this problem?

  • HI Joe!

    As a workarround it may help if you set the property

    <property name="ti.android.compilejs" type="bool">false</property>

    in tiapp.xml.

    The problem is the obfuscation code which was introduced in https://jira.appcelerator.org/browse/TIMOB-7553

    — commented March 8th 2012 by Dietrich Streifert
