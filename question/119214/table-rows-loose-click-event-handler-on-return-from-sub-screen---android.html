Table rows loose click event handler on return from sub screen - Android.

Hi,
I am listing a number of items in a table, each with it's own row.

When you click on a table row it opens a new screen and passes some data to it, its a detail screen so populates itself based on the data supplied e.g. id number.

When i hit back in emulator or on device i get back to the table rows, only this time no matter what i try and do i cannot open any of the other items to their detail page.

It works fine on iPhone but not on Android.

I am using hasChild on the table rows but a custom right button click handler as i need to pass data to the sub window… does any know of any problem with this approach, or an alternative to use table rows but pass data to the new window??

I can post some code, but which bit? The table row creation code: that makes a row and adds images and labels to it directly- not in a view, but tried and view and also not working.

Debug says nothing, like there was no click… but it works first time.

Thanks,

  • [solved]
    Put the event handler on the table itself and use the e.index value to refer to an array of the data you want e.g.

    My table row data was in a array called "currentRemoteData"
    So to pass the id i want to the detail window i refered to it like this;-

    var newwin = Titanium.UI.createWindow({
        backButtonTitle:'Back',
        url:'shop-details.js',
        title:currentRemoteData[e.index].shop_name,
        passData:currentRemoteData[e.index].id
        });
