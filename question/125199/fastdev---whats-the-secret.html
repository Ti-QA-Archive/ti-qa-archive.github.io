FastDev - What's the secret??

For some reason, I am not having luck getting this FastDev thing to work. I've followed all the steps/guides numerous times. I start my FastDev server:

$ titanium fastdev start
[INFO] [2011-09-05 23:28:12,577] Serving up files for com.burch.HowdyTitanium at 0.0.0.0:56842 from /Applications/MAMP/htdocs/titanium_apps/HowdyTitanium

But when I either run from Titanium Studio, or manually start my AVD and run from the command line by: 

titanium run --platform=android --android=/Users/PSB_Dev/Sites/assets/sdks/android-sdk-mac_x86

…I can't get Titanium to recognize and use the FastDev server. It doesn't see it running at all:

[WARN] Fastdev enabled, but server isn&#39;t running, deploying normally

Restarting the FastDev server doesn't work either:

$ titanium fastdev restart-app
Error: No Fastdev Servers found. The lock file at /Applications/MAMP/htdocs/titanium_apps/HowdyTitanium/.fastdev.lock does not exist, you either need to run "stop" within your Titanium project or specify the lock file with -l <lock file>

I can't work with a framework that makes me wait over a minute to preview changes. Please, somebody help me with this. I asked a similar question weeks ago, and after 130 views, nobody knows the answer. What am I doing wrong getting FastDev integrated into my workflow? Anyone? Thanks.

  • Found the answer somewhere else in the Q&A. Changing one line in fastdev.py under /android in Ti SDK fixed it for me. Line 320 in my fastdev.py sais:

    self.socket.connect((&#39;localhost&#39;, self.port))

    it used to say something else instead of a simple 'localhost' and it didn't work neither from CLI nor from the Studio. Now it does.

  • I did a trick.

    Following Pavel Veller solution, I checked what append…

    Here, at home, $HOSTNAME is patrice-desktop-home. gethostname() return this string. But pinging this doesn't work. We need "patrice-desktop-home.local". So, the problem is that the name without ".local" doesn't respond (DNS is not ok).

    The simple trick:
    edit /etc/hosts with root (or sudo) and append

    127.0.0.1    your_host_name

    where "your_host_name" is the value given by

    echo $HOSTNAME

    And it works like a charm.
