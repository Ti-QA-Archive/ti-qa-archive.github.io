How rename a Database?

Hey,

Our apps have to support multiple users / databases and we key our database names as follows:

appname_username_dbName.sql

However, it turns outs our usernames can be case-insensitive, so ADMIN and admin are the same user both are valid. (It is a legacy backend system).

On the simulator this is fine, as it is also case-insensitive, but on an IOS device, you'll end up with:

appname_ADMIN_dbName.sql
and
appname_admin_dbName.sql

So obviously depending on how the users types it, they'll get different data sets.

We're looking to resolve this, and the app is live with customer data. So I need to be able to rename the database files that are in the 'Private Documents' folder.

Tried, rename and it failed, I've tried copy or moves of the file to a different folder in an attempt to move it from Private documents, rename and then Db.install() from there. All of which doesn't seem to work.

I think its because Private Documents is read only?

Does anyone have any suggestions on how to resolve this?

Thanks!
Joe
