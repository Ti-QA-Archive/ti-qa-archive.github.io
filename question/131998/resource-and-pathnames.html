Resource and pathnames

Hi guys I'm new at Titanium, so I've a noob question that is driving me mad.
I'll try to be quick and simple:

First of all: I'm trying to deploy an Android App with a PC.

Then here's my folder structure & files

  • Resources
    • app.js
    • webpage.html
    • my_image_name.png
    • jquery.js
    • KS_nav_ui.png

Now here's my problem:

  1. When I'm trying to import 'my_image_name.png' it doesn't work.
    only 'KS_nav_ui.png' works and url like http://www.mysite/imgs/my_image_name.png

  2. The same is for jquery.js
    If I try to include it, the error is: "Can't find variable: $"
    It works instead, if I try to include it in this way:
    "http://www.mysite/resources/jquery.js"

If I separately open "webpage.html" in wich "jquery.js" is included, it works fine.
That means that the path is correctly declarated.

Why doesn't the same happens in Titanium?
The question is WHY? What I'm doing wrong?

Thanks in advance you all, Mathew.

  • Solved.

    THIS WORKS:
    <script src='http://www.mywebsite.com/resources/myscript.js'></script>

    THIS DOESN'T WORK:
    <script src='resources/myscript.js' type="text/javascript"></script>

    To make it work, be sure adding charset='utf-8', like that:

    <script src='resources/myscript.js' type="text/javascript" charset="utf-8"></script>

    No idea why.
