Tabbed bar forgets which view to show on the third try?

I'm using a tabbed bar to show three different table views. That's generally working fine. Each table view uses a click event to display a details window for the clicked row. That works fine, too.

The problem comes in when I use the "Back" button in the detail window on the THIRD try. I can view the details window properly on the first two clicks of any given table view. However, on the THIRD use of the "Back" button, my tabbed bar forgets which table view to show and reverts back to the first table view. Yet the selected tabbed bar tab is CORRECT…

tabbedBar.addEventListener('click', function(e) {
    if (e.index == 0) {
      businessScroll_view.animate({view:business_featured_tableView, transition:Ti.UI.iPhone.AnimationStyle.FLIP_FROM_LEFT});           
    } else if (e.index == 1) {
      businessScroll_view.animate({view:business_alpha_tableView, transition:Ti.UI.iPhone.AnimationStyle.FLIP_FROM_LEFT});           
    } else if (e.index == 2 ) {
      businessScroll_view.animate({view:business_category_tableView, transition:Ti.UI.iPhone.AnimationStyle.FLIP_FROM_LEFT});           
    }
});
