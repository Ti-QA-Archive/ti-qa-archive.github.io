TouchEnabled does not disable

Hello,
I have just upgraded to Titanium 3.3 and noticed that enabled is now deprecated. I previously was using enabled for a pair of buttons to create a pagination effect. When a button was disabled it would grey out and the user would be unable to click it. After installing 3.3 and moving to using touchEnabled, the expected behavior no longer occurs. To grey the buttons out I was setting the disabled color in my style sheet to be grey. When using the iOS simulator, the buttons are not disabled and there is no style change, but when using a physical android device the buttons do disable again without the style change. I am looking for any sort of help in better utilizing the touchEnabled property to accomplish this as I do not want to have to add and remove event listeners and change styles when the enabled property did that for me.

    Hi Logan,

    the Release Note only says "View.enabled" is deprecated for android, not for iOS. So you can continue to use the enabled property in iOS specific code. 

    Titanium.UI.View.enabled    property    This is deprecated on Android. Use touchEnabled property instead.
