Using alloy.jmk to customize different environment builds?

Can I use the alloy.jmk file to change the app icon and/or app ID based on a config.json environment variable before the application builds?

I have to manually change the app ID and app icon when I build the QA vs PROD version of my app and would love to automate this.

Thanks!

— asked May 20th 2014 by Demetri Ganoff

  • HI,

    if you take a look at

    https://github.com/rblalock/core

    you will see that it gives an example of a custom jmk file for alloy. This basically reads the project config file and then depending on what is set it will copy files into the main project.

    You should be able to use this as a starting point and create your own version to read the config file then overwrite the assets you need. Not sure if you can access just the app id though -it may be that you create a set of tiapp.xml files and just select the right one for the build you are doing
