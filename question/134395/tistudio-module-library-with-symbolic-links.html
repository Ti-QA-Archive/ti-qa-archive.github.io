TiStudio module library with Symbolic Links

I have been trying to use a symbolic link(MAC) to allow me to have a common modules dir that can be referenced in any project. I just add the symbolic link into the Resources dir of any project and refer to it as though it were a local directory.

This works fine for testing, developing and running in the emulator, but although a Deploy works fine the app does not run. I am assuming this is because the Deploy process(and TiStudio) does not like the symbolic link and the symbolically linked dirs do not get built in.

Has anyone else attempted this, come across the same problem or found a way round it? Currently I just have to actually copy the module dir into the Resources folder at deploy time, but it would be good if this was not necessary.
