win close variables gone

I am using win.addEventListener to observe close of the current win and once it is closed, I will do something. However, when it is closed, anything defined within the scope of current window is gong, meaning nothing is accessible.

For example

david.js

var myGlobal = {line:0
}
win.addEventListener("close",function(e){
  myGlobal.line // not accessible any more


});

Is there any solution available allowing me to access the variable? Thanks

— asked June 22nd 2011 by Jiale He

    You can declare the variable on the Ti.App namespace, this will always hold the variable

  • The event variable (e in this example) has a property named 'source'. This property is a reference to the object that triggered the event, so e.source.myGlobal.line will reference the variable correctly. After the close event is processed, however, it will be gone forever unless you follow Rasmus' suggestion and make it a property in the global Ti.App object.
