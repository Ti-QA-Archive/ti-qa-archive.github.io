Installing/Testing Multiple Devices

I'm trying to install my app on multiple devices… can anybody help me out with this…

To my understanding, I need to add the devices to the provisioning portal and I'm required to create a a provisioning profile for the testers. I just need to provide the tester with a copy of the provisional profile and the .app folder.
They then just drag and drop both these files into the itunes…

Are these the only things they need? Do they require the other certificate? Can somebody point me in a correct direction to help me test on multiple devices.

edit Also do I need to have the device attached to my machine or can I just email the profile and .app to them ?

Thanks guys

  • Accepted Answer

    It's worked for me to email the provisioning profile file and a zipped version of the app. My testers dragged & dropped the provisioning profile into iTunes first, then the same with the app zip, then synced their phones. For later updates, I just sent them the app zip again. You don't need to send them your certificate, and you certainly wouldn't send them your private key. And no, you don't need to have them attach their phones to your mac.

  • For testers which are at OS 4.x, by far the easiest thing for the user is an over the air install they can do with one click of a link in an email or your About box (if already installed and doing an upgrade). The trick is to create a *.ipa file with the provisioning profile embedded.

    OS 4.x added the ability to install the .ipa via a download provided the download link is to a special .plist file instead of the *.ipa itself.

    See my reply in this thread. If you don't have your own web server where you can host the file, you could consider using this to provide the download links.

    For OS 3.x users, I think you can just email the .ipa and they can do a single install via iTunes. Probably by just double clicking the .ipa attachment in the email.

    The *.ipa is compressed so tends to be half the size of the .app folder method besides.
