Remote jpeg images not displaying in imageview on iOS simulator

Titanium SDK 3.0, running in iOS SDK 6.1/iPhone simulator 6.1, latest titanium studio

I have a server-side php script which is outputting jpeg images like this:

header('Content-Type: image/jpeg');
header('Content-Length: ' . filesize($file));
readfile($file);

I have tested in Android emulator using SDK 2.2, and the images display fine. In iPhone simulator however, the image does not display.

I add the images to a horizontal-layout view like this:

var image = Ti.UI.createImageView({
  image: 'http://mysite.com/showimage?param1=1&param2=2&param3'=3,
  height: 100,
  width: 100
});

$.imagesView.add(image);

Several images are added to the 'imagesView', some are a local png image (those display correctly) others are like the above, a remote jpeg, and do not display in iOS simulator.

I think that maybe it's an issue with headers somehow? I've tried without the content-length header and that still works in android, still doesn't work in iPhone simulator.

I have also tried in the iPad simulator (same iOS SDK version) and the images don't show up there either.

I can also copy/paste the image url into browser and view the image correctly.

I can't give the image url as it's part of an api that is restricted by authorization token. However here are the http response headers when hitting the image url successfully:

HTTP/1.1 200 OK
Date: Fri, 15 Feb 2013 22:59:53 GMT
Server: Apache/2.2.23 (Unix) mod_ssl/2.2.23 OpenSSL/0.9.8e-fips-rhel5 mod_bwlimited/1.4
Expires: Thu, 19 Nov 1981 08:52:00 GMT
X-Powered-By: Fat-Free Framework (http://fatfree.sourceforge.net)
Pragma: no-cache
Cache-Control: no-cache, must-revalidate
Keep-Alive: timeout=5, max=100
Connection: Keep-Alive
Transfer-Encoding: chunked
Content-Type: image/jpeg

Any thoughts?
