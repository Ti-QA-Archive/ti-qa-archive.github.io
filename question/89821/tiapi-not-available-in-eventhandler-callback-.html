    • You should only increase the stacksize if you run out of stack space and you should increase it in small increments. These devices have limited memory and specifying an overly large stack that won't get used wastes space you might want for your app.

      — commented June 2nd 2011 by Don Thorp

    • I agree in most of what you've said Don, "128000" is not intended to be there for universal use, it's just an example. But I don't agree that stack size should be increased only when device run out of stack space, because (at least for me) error saying that stack run out of space didn't show many times. Rather, strange issues (wtf-issues :) ) would appear and increasing stack size solved that issues. I think increasing stack size is a good try to solve really strange issues.
