Ti.UI.SIZE doing a FILL on android

I have a view with a width set to Ti.UI.SIZE and it appears to be filling the whole screen. It is working fine on iOS, but not android. My code is as follows:

var tray = Ti.UI.createView({
    width:Ti.UI.SIZE,
    height:'100%',
    top:0,
    left:0,
    backgroundColor:'#ff00ff',
    layout:'horizontal'
});
win.add(tray);

var controlHolder = Ti.UI.createView({
    width:Ti.Platform.displayCaps.platformWidth * 0.9,
    height:'100%',
    top:0,
    left:0,
    backgroundColor:'#000',
    layout:'vertical'
});

var searchList = Ti.UI.createSearchBar({
    hintText:'Search'
});

var slideList = Ti.UI.createTableView({
    width:Ti.UI.FILL,
    height:Ti.UI.FILL,
    top:0,
    left:0,
    search:searchList,
    filterAttribute:'searchFilter',
    backgroundColor:(Ti.Platform.osname == 'android') ? '#000000' : '#ffffff'
});
controlHolder.add(slideList);
tray.add(controlHolder);

var grippie = Ti.UI.createView({
    backgroundImage:'images/grippie.png',
    width:21,
    height:50
});
tray.add(grippie);

The blatantly obvious pink (#ff00ff) fills the entire screen as if I was using Ti.UI.FILL on the view, tray I know for certain tray is not as wide as the whole screen…or at least it shouldn't be. As said before, this is working fine for iOS.

I am using TiDSK 2.1.4

    • I think the problem is due to "how" the Android implementation handles the Ti.UI.FILL on child views, in your case the slidLeft.

      However if you think about the problem a little differently you can create a cross platform solution with one set of code.

      Please accept my apologies for using a stupid set of background colours to show where everything is, but try this variation on your code;

      var tray = Ti.UI.createView({
    backgroundColor: 'green',
    height: Ti.UI.FILL,
    left: 0,
    right: 50,
    width: Ti.UI.FILL
});
win2.add(tray);

var data = [
    { title: 'One' },
    { title: 'Two' }
];
var searchList = Ti.UI.createSearchBar({
    hintText: 'Search'
});
var slideList = Ti.UI.createTableView({
    backgroundColor: 'blue',
    data: data,
    filterAttribute: 'searchFilter',
    height: Ti.UI.FILL,
    search: searchList,
    right: 21,
    width: Ti.UI.FILL
});
tray.add(slideList);

var grippie = Ti.UI.createView({
    backgroundImage: 'images/grippie.png',
    height: 50,
    right: 0,
    width: 21
});
tray.add(grippie);

      It uses the ability to set a dimension of width and still be able to play with left and right (or top and bottom) which creates a margin of sorts.

      Whilst the numerical values will not be your ideal, you can quickly tweak as required.

      Tested on iOS and Android. Let me know what you think.

      The technique used here has helped me solve many issues, on favourite is leaving a space above a table when I do not know the height of the screen, for example;

      var tbl = Ti.UI.createTableView({
    data: data,
    height: Ti.UI.FILL,
    top: 50,
    width: Ti.UI.FILL
});

      Now it always leaves a 50 pixel gap at the top for whatever I place there, but the balance automatically fills the remaining space. Perfect for the aspect ratio differences us poor developers have to constantly account for.
