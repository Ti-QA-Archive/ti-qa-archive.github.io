SDK 3.2.0 Camera Returns Inverted Image

Hello, I'm using Titanium Studio with the SDK version 3.2.0 for mobile. I'm developing the iOS version at the moment. I'm currently modifying an app for a customer and one of the complaints was the resizing of the image taken by the camera. At first I though the previous programmer had made and error but the bug resides in the Titanium SDK itself. It's also odd that I can't find anyone else with this problem.

Here's what happens when I call

Titanium.Media.showCamera({});

inside my function I just called success: function(e) and did var image = e.media and I got my image alright but the problem happens before this inside the function call.

It opens up the new view where you can take the photo then it asks you to retake or use this photo. So this all seems to be parts of Titanium.Media.showCamera. When I take the image my picture is inverted, flipped vertically. Lets say the sun was in the left top corner when I would take the picture the result would be the sun in the right corner instead, so it's like the whole image flipped. Then when I accept the photo I flipped it 180 degrees and my image is how it should be. I'm wondering if there is a fix to this and I'm also wondering why no one else has noticed this bug.
