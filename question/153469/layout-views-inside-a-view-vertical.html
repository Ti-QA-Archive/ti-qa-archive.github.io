Layout views inside a view vertical

I'm trying to make some views inside a view layout vertical. But I can't get it to work. Here is my code:

<Alloy>
    <View id="content" height="auto" width="auto" layout="vertical" top="150">
        <View height="auto" width="auto" layout="vertical">
            <View layout="horizontal">
                <Label id="time"    class="lecture">13:15 - 14:45:</Label>
                <Label id="room">R001<Label>
            </View>
            <Label id="subject" class="lecture">1. Lecture</Label>
        </View>
        <View height="auto" width="auto" layout="vertical">
            <Label class="pause">Pause</Label>
        </View>
        <View height="auto" width="auto" layout="vertical">
            <View layout="horizontal">
                <Label id="time"    class="lecture">15:15 - 16:45:</Label>
                <Label id="room">R001<Label>
            </View>
            <Label id="subject" class="lecture">2. Lecture</Label>
        </View>
    </View>
</Alloy>

The childViews are not arranged vertical at all. Instead they're are simply stacked over each over. The result is only the first childView being visible.

This is working: But I can't use this code because I have to make time and room to be displayed horizontal aligned next to each other:

        <Label id="subject" class="lecture">2. Lecture</Label>
    </View>
</Alloy>

Any help is appreciated!

  • Accepted Answer

    Couple things to try:

    • Don't use auto. The appropriate value you should be using is Ti.UI.SIZE
    • I'm betting this has to do with how many nested levels of auto-sized components you are using. This may not be practical for your app, but can you please try to use a static height and width for your inner views that have vertical layout? Let us know how that helps, if at all.

    • By default views are going to do a Ti.UI.FILL on height and width. You need to explicitly set them to Ti.UI.SIZE on all views contained in this hierarchy. When you are creating this long list of views that are dependent on the sizing of internal views, all it takes is one missed view size to throw the whole thing off.
