iOS7: showCamera in popover wrong preview

Hi Guys,

I'm having problem with the showCamera function in iPad when I put it in a popover. The preview is rotated 90degrees. Any idea why? I think it's a bug in Titanium.

You can check a screenshot here: Screenshot

Any idea on how to fix this and how to make the preview normal again? Any help is very much appreciated.

Here is what I'm using right now in building iPad apps.

Titanium SDK 3.1.3 GA OS X Mountain Lion.

Below is my code. 

Titanium.Media.showCamera({
    inPopOver:true,
    autohide:true,
    popoverView: menumenu,
    arrowDirection:Ti.UI.iPad.POPOVER_ARROW_DIRECTION_LEFT,
    saveToPhotoGallery:false,
    allowEditing: true,
    mediaTypes:[Ti.Media.MEDIA_TYPE_PHOTO],
    success: function(event){
        var image = event.media;
        if(event.mediaType == Ti.Media.MEDIA_TYPE_PHOTO){
             // do something here...
        }
    },
    cancel: function(){},
    error: function(){}
});

  • I now know how to temporarily fix it. To those who have same issue as I am you can add 

    transform: Ti.UI.create2DMatrix().rotate(-90)

    in the camera configuration.

    Titanium.Media.showCamera({
    inPopOver:true,
    autohide:true,
    popoverView: menumenu,
    transform: Ti.UI.create2DMatrix().rotate(-90),
    arrowDirection:Ti.UI.iPad.POPOVER_ARROW_DIRECTION_LEFT,
    saveToPhotoGallery:false,
    allowEditing: true,
    mediaTypes:[Ti.Media.MEDIA_TYPE_PHOTO],
    success: function(event){
        var image = event.media;
        if(event.mediaType == Ti.Media.MEDIA_TYPE_PHOTO){
             // do something here...
        }
    },
    cancel: function(){},
    error: function(){}
});
