New Unofficial API

A little test; I created a new API doc by analyzing the Kitchen Sink source code:

http://kitchen.srsly.co

This is helpful for me, because the Kitchen Sink code is the most up-to-date API documentation we have. And so this web app uses the source code to find out how the API is actually used: what parameters are available, and what values are allowed.

It's a rake script that parses the source code into objects, and then a rails app displays the results. It's pretty rough right now. I'll be improving it as I need to for my own reference.
