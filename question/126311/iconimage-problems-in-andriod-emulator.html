Icon/Image Problems in Andriod Emulator?

I am having trouble getting Titanium Studio to show icons/images on the Android emulator.

For example, when I create a new Titanium Mobile Project – which includes a two tab base program and the two tabs reference icons – those icons are not showing up on the tabs when I run the unmodified new app in the emulator for versions Android API/Screen 1.5/HVGA [app fails installation], 1.6/HVGA [app fails installation], 2.1-update1/HVGA, 2.2/HVGA, 2.3.1/HVGA, 2.3.3/HVGA, 3.0/WXGA, Google API 1.6/HVGA {emulator does not start}, Google API 2.1-update1/HVGA {emulator does not start} - however they do show up in 3.1/WXGA. Are icons in tabs a new Android API feature?

In addition to the icons, I am having similar problems with images, both leftImage/rightImage in TableViewRow and also createImageView. I am using the code from the Custom Table View Row example on the Ti documentation site. When I started with this problem I was not sure where they belonged – it seems that there are variations in the posts of people that are getting images go work or are having problems. I have pretty much put them everywhere Resources/, Resources/images, Resources/android/… etc but it does not seem to make any difference. I have also used many different "paths" including just the actual file name. When a file is missing I can see that the emulator throws an error to the Ti console; likewise I can see when the emulator seems to find the icon & image files - it shows the file get without an error - but no image or icon is showing up on the emulator screen.

As I am new to Titanium (first time coding for mobile device), I am sure I am doing something wrong – but hey, I cannot find it. I have been trying to get icons & images to show up for three days.

Running Kitchen Sink multiple times in a row seems to produce dissimilar results and never an icon on a tab. Some screens that say there are images there but no images are visible, others there are images. Inconsistent behavior at best.

I have used both url: and image: properties with createImageView – it does not seem to make any difference.

I have run the Android emulator update procedures several times – and I have deleted all emulators and redefined them through Ti Studio.

I am not sure why, but when the Andriod emulator is starting up, several command windows open and close – most I never see anything in, but at least one always appears to say “Error Loading Preferences”

I am using Titanium Studio 1.7.2 – all updates applied (well, it says there are no updates although I am hoping someone will tell me that I am missing something).
