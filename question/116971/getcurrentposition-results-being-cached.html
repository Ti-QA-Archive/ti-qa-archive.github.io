getCurrentPosition results being cached?

In my app, getCurrentPosition returns accurate results the first time. However, if I close the app, go somewhere else and open the app, getCurrentPosition will return my previous position. Anyone seen this behavior? Have a workaround?

Here's my code:

Ti.Geolocation.getCurrentPosition(function(e){
    if (!e.success || e.error){
        return {success:false, errorMsg:JSON.stringify(e.error)};
    }

    return {success:true, lat:e.coords.latitude, lng:e.coords.longitude};
});

Platform: iphone
