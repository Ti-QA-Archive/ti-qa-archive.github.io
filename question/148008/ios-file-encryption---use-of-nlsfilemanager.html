IOS File Encryption - Use of NLSFileManager?

Being new to app development:

We have an IOS/Android application created in Titanium, in which we import our generated HTML content files (created via XSL from XML) into, have our app.js created in Titanium and build .ipa file.

We need to have the html files encrypted on the device to prevent someone plugging in and grabbing those html files, as their content is sensitive. So, with that being said, one option I am looking into is the use of the 'NSFileManager' class mentioned in the 'iPhoneAppProgrammingGuide.pdf' under section for 'Protecting Data Using On-Disk Encryption' which states if using the 'NSFileManager' class to set some properties on the files, when passcode security is used, the files will be encrypted. My question is, where and in what piece of code to I add that? Is that something I create a script for to run against my html files?? Within the app.js?? Can I do this in Titanium or is this only for xCode? I cannot seem to find any examples of how and where to do this.

We've also talked about possibility of moving our html files to within the SQLite database and encrypting the database, not sure if that should be an option, worry over app performance of loading different html files using TOC menus and how fast they will load having to decrypt (and how from app.js to call to decrypt files anyway)? Any suggestion or help greatly appreciated.

Titanium: 3.0
iOS: 6.1

    You'd have to create a custom module for this or modify the Objective-C code for iOS SDKs for appcelerator. If you're good with Objective-C then updating the code is easier than writing the module. You can find the obj-c code in /Library/Application Support/Titanium/mobilesdk/osx/[SDK_VER]/iphone/Classes/

    Be advised that you might break up something if you dont pay attention, so better backup the folder before modifying the files.
