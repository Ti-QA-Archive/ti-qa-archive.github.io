[WARN] : New layout set while view [object TiUIImageView] animating: Will relayout after animation

Hi, I'm using this JS library: https://github.com/raulriera/Animator to animate some imageViews in my iPhone project.

I create/add an imageView instance (they a total of 10) this way:

var star1 = Ti.UI.createImageView({
        left:'1%',
        image:'/images/star1.png',
        width:113,
        height:291,
        top:-90
    });
self.add(star);

and I animate it this way:

var animation1 = new Animator();
animation1.loopSequence([
            { 
                type: "moveTo", 
                view: star1, 
                value: {x: 0, y: (Ti.Platform.displayCaps.platformHeight/8) }, 
                duration: 16000 }, 
            { 
                type: "moveTo", 
                view: star1, 
                value: {x: 0, y: 0 }, 
                duration: 18000 },  
        ]);

Problem is I don't know how to eliminate (in truth, I neither know what it is trying to tell me!) this warning I get once compiled and running the project:

[WARN] :  New layout set while view [object TiUIImageView] animating: Will relayout after animation.

Can this cause memory leaks? Is there any way to get rid of it?

  • As far as I can tell (on iOS) this does not actually cause any problems with the app. As the message states, the animation is re-scheduled to occurr after the existing animation completes. The reason for the message is that in Ti App you cannot have more than one active animation at a time, they should be chained by using the 'complete' event listener of the animation
