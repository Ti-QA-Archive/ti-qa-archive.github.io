Hide keypad.

I have a textarea field in a window. And I have write the code to close the window when user will click on android device back button. But now I have issue i.e. when I focused to that text filed and click on device back button it just close the window as per my code. So how to hide that keypad when I entered text to that textarea.

Please suggest. Here is my sample code.

var win = Ti.UI.currentWindow;

var txtArea = Ti.UI.createTextArea({
    height : 100,
    width : 300
});
win.add(txtArea);

win.addEventListener('android:back',function(e){
    win.close();
});

    • I see what you're saying. But if the problem you're having is already a jira issue, you'll have to watch the ticket to be notified when its fixed.

      Something that might work, is if you don't need the soft keyboard on the window you are going back to, then you can try to set the softKeyboardOnFocus property of that window like so:

      softKeyboardOnFocus:Titanium.UI.Android.SOFT_KEYBOARD_HIDE_ON_FOCUS

      That should hide the keyboard when that window comes into focus.

      OR

      A real ugly workaround might be to add an invisible text field on the page that you're going back to and focus on it when coming back from the form page. Once its focused, it will have control of the keyboard and you can blur that field to hide the keyboard again. Of course this is not the most elegant solution, but one of these 2 should work until the jira issue is fixed.
