Global variable usage and garbage collection

I am using global variable to hold a module and use the variable when I needed. I could not see it clear if the way I am using it will cause issue, such memory leak.

e.g.

In the alloy.js:

Alloy.Globals.moment = require('alloy/moment');

In the index.js:

$.label1.text = Alloy.Globals.moment().format('MMMM Do YYYY, hh:mm A');

$.label1.text = Alloy.Globals.moment.utc().format('MMMM Do YYYY, hh:mm A');

The Alloy.Globals.moment is also used in a similar way in other couple of places. Hope someone could give me some direction.

Thanks

  • Don't require it on alloy.js. Try to require on-demand basis it in controller. For e.g.

    index.js

    
var moment = require('alloy/moment');

$.label1.text = moment().format('MMMM Do YYYY, hh:mm A');
$.label1.text = moment.utc().format('MMMM Do YYYY, hh:mm A');

moment = null;
