iPad Ad Hoc builds in Xcode getting Failed to Install

Im having an issue getting my Titanium App installed on an iPad - specifically when building from Xcode. (I'm using the Ad Hoc distribution method so that can give the ipa file to a client).

What happens is after syncing the iPad with iTunes the app attempts to install then terminates with the message on the Device:

> "My App name" failed to install.

I have seen this error before and typically is the error you would get if the Device's UDID is not in the Provisioning File, which it is.

If I build the app for iPhone, and deploy to the iPad this DOES work. So very much iPad specific, I'm guessing its something to do with my Build Settings in Xcode.

Also, to note if I use Titanium Studios "Distribute app via Ad Hoc/Enterprise" for the iPad this DOES work, so again only Xcode builds not working. I've compared the Provisioning file that Titanium is building with and it is identical to what I have chosen in my "Code Signing" (Build Settings) in Xcode. So again - if anyone can tell me what Titaniums Ad Hoc distribution method is doing differently from how I'm building in Xcode?? As ideally I want to use Xcode to manually build the ipa as Titanium is doing. (A small point to note is that the ipa file size is slightly bigger after Titanium builds the ipa vs. Xcode doing it, as in 21mb to 19mb).

Note: I need to build the App via Xcode as I'm have to apply a bit of code to embed a custom push notification service.

Any help or recommendations on this topic would be very well received as this is causing me to tear my hair out!!

  • Hi,

    I was able to resolve the Original issue here and will post in case its of use to anyone else that hits this issue.

    In the end what I did was the following:

    • In Xcode click my project so that I get a list of my Targets (i.e. they were iPhone, iPad and Universal).
    • Duplicate the iPhone Target, and call it iPad or whatever.
    • For the duplicate Target change the Device to "iPad"
    • Change the Target (Top-left) to the my duplicated target (iPad) and build and archive.

    And that works.
