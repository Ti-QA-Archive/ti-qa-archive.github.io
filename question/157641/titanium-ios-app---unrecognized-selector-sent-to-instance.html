  • I found the solution to the problem (I think).

    I was having this issue as well and did a few tests to figure out what the error was causing.

    In my webview html I was adding an event named "loaded" to the titanium.app instance by doing:

    // Inside the html document.
Ti.App.addEventListener("loaded", function() { 
    // do your thing in the webview 
});

    and was firing the event on the webview.load event by doing:

    // Within my app
webview.addEventListener("load", function() {
    Ti.App.fireEvent("loaded", <data>); // <data> is an object with properties
});

    Every time I opened the webview, the event listener was added. So for me when I opened the webview the first two times it worked fine, but afterwards it crashed.

    Now I remove the event listener on the window.onunload event so instead of having 1 or more event listeners listening to the "loaded" event I only have one and now it works fine.

    I hope this helps and if not, please comment and I will try and review the problem.
