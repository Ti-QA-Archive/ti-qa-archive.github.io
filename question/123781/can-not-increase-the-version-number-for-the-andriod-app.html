Can NOT increase the version number for the andriod app

It is strange when I re-complile the Andriod App, and I try to replace the .apk at android market, it prompt the version is the same.

But I did edit the tiapp.xml file to change the <version>, it didn't update when re-compile the .apk.

Please help.

  • I'm not an expert but I went through the same thing.

    There is (or should be, or you can create) a file named "AndroidManifest.xml under "\platform\android" in Titanium.

    Here are the first few lines of mine:

    <?xml version="1.0" encoding="utf-8"?>
<manifest xmlns:android="http://schemas.android.com/apk/res/android"
      package="com.walkthelot.mobilelot"
      android:versionCode="213"
      android:versionName="2.1.3"
...continued...

    When I increase my version, I do it both in the tiapp.xml file and also in the two places in the AndroidManifest.xml. The first in AndroidManifest.xml has to be an integer, and the second is a more friendly version name that is shown in the market.

    Hope it helps. Let me know if you need anything else.

  • You don't need to create your own manifest xml file. You can customize the manifest (including version info) right in tiap.xml:

    <android >
    <tool-api-level>10</tool-api-level>
    <manifest android:installLocation="auto" android:versionCode="12" android:versionName="2.5">
        <uses-sdk android:maxSdkVersion="10" android:minSdkVersion="7" android:targetSdkVersion="8"/>
        <supports-screens android:anyDensity="false" android:largeScreens="false" android:normalScreens="true" android:smallScreens="false"/>
    </manifest>
</android>
