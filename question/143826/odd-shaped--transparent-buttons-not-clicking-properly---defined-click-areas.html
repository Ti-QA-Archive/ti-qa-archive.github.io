Odd Shaped / Transparent Buttons Not Clicking Properly - Defined Click Areas?

After much searching I haven't found any information about this topic. When you have standard image button code that uses an image that is not a rectangle, it appears that the "click area" is still the rectangle that encompasses the button no matter what the image’s actual shape is. Image transparency does not seem to factor into the click area of a button.

It there a way to make it so that the transparent portion of an image does not respond to touch events? So if I have a circle button, with the center cut out… can the actual circle detect the touch event while the cut out in the center ignores it?

The app I’m writing is for the iPad and the specific problem is that there is a cluster is buttons close together in a scrollableview. Because the image buttons are so close together (invisibly, not visually) it’s nearly impossible to actually scroll the view beneath. There are cut outs in the buttons that when ignored by touch events, would allow the interface to scroll beautifully. I’m on the latest version of Titanium with the latest SDK as of today. Thank you.

  • To elaborate on Pankaj's answer, you can use the x, y coordinates of the click to determine if it happened inside your odd shape:

    var iv = Ti.UI.createImageView ({ ... });

iv.addEventListener ('click', function (e) {
    Ti.API.debug ("x,y: " + e.x + ", " + e.y);
    // implement your logic to check whether x, y is inside the shape or not...
});

    • I'm pretty sure that click events are going to happen if you're inside the bounding box of the image. If you think about it, having the touch events not fire when you're touching a transparent pixel would not be very intuitive. I understand it would fit your situation, but it would also cause problems for other developers who would want it to fire.

      If your shapes are simple geometric shapes, you can compute whether you're inside or outside the non-transparent area.

      If they're just arbitrary shapes, you're right – it would be difficult. I don't think there's an easy way to access a pixel value in the image to determine whether it is transparent or not.
