Can you Cache a Component??

My question might seems odd but let me explain the scenario and this might make more sense.

I have a Tableview and for every row, i open the same window with the same elements. The problem is that because I have so many elements on this new window that it takes a while to load it up, not much but enough to annoys me.

To speed it up, I was wondering if anyone have a suggestion of how I could "cache" this window somewhere and maybe everytime I need it I wouldnt need to open and build it up again, but instead i would just bring it back and replace all the information I had it there before, like a "template".

Any suggestion?!

  • I have had poor results trying to reuse a window (under android).

    When I tried to reuse a window object, after I hit the hardware "back" button on android, I was unable to open the window again. The app would then become very unstable, usually crashing after 15-20 seconds.

    It's interesting to note that in the Master/Detail Application template in Ti Studio, the android code explicitly avoids reusing the DetailView and the window it's housed in, while the iOS code happily reuses it.
