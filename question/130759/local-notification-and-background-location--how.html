Local Notification and Background Location – how?

I've been researching this topic for a while, but I still didn't find any working solutions for this.

I have the following problem: I have to notify people at interesting places while my app is in the background.

I want to do this by download the list of interesting places to the memory, then by using Background Location (the location entry in the tiapp.xml) do the calculations on every location event callback. This works fine. What does not work is the Local Notifications, which somehow disappear shortly after displaying them.

On this forums I've found that the problem is caused by the program does not have any references to the notification. I've tried every trick: putting the scheduleLocalNotification in the app.js, making the returned var global, setting timeouts. The closest thing to working was my own solution, which used the bg.js to display the notifications. The problem with this is that bg.js runs only for 10 minutes (via Task Completion background run method).

Please tell me that there is a solution, only I did not find it :)
