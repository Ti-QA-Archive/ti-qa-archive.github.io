  • Well, to start with, debugging on genymotion is the same as debugging on a device. Use ddms or monitor to connect to the emulator/device and look at the logs. There is no psychic debug network that can magically tell you what 'your app has stopped' means.

    We had an S3 connected to a mac just decide one day they didn't like each other either. Took multiple reboots of both and flipping usb connection types back and forth to get it working again. Ugliness to be sure.

    My script is simply:

    ti build -p android -b
echo 'Installing APK to device if available'
adb install -r build/android/bin/app.apk

    No other emulators running, no other devices attached.
