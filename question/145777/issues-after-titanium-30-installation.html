  • To better debug your problems let's do a few things.

    First, launch the terminal and run titanium config cli.logLevel trace.

    Now, run titanium info -t ios. This will print out all sorts of iOS related dev environment information such as Xcode versions, iOS SDK versions, provisioning profiles, and so on. Using this info, you can verify that you have a iOS 5.1 Simulator installed and that the appropriate provisioning profiles exist.

    Next change into your project directory and run titanium clean. Then run titanium build -p ios --sim-version 5.1 to perform a build and launch the 5.1 Simulator.

    Regarding item 3, yeah "Reset Content and Settings" is your friend. For item 5, the new CLI (thus Titanium 3.0) require node.js 0.8+.

    Give this a try and lets see if we can get your issues resolved.

    • @Kiwi, you don't need iOS 5.1 SDK to build for a 5.1 device. You can build with the iOS 6 SDK and it will still run on a 5.1 device.

      Also, there was a bug in older versions of Titanium Studio where the iOS SDK dropdown would incorrectly list all the iOS Simulators, not the actual installed iOS SDKs. That means if you have iOS 6 SDK installed with iOS 5.1 and 6.0 Simulators installed, it would allow you to pick 5.1 or 6.0. Picking 5.1 would then tell the build scripts to use 5.1, but the build script wouldn't find 5.1 and would actually just use 6.0 anyways. Crazy, I know.

  • @Kiwi:
    For the first item in your list, I think this is what you are seeing: Xcode 4.5.x comes with iOS 6 SDK. Studio previously had a bug where "run on device" builds were showing the list of simulators instead of the list of SDKs. So since you can only compile with the iOS SDK 6.0 with Xcode 4.5.2, if you set the pop-up (in run-on-device builds in the older Studio) to anything other than iOS 6, say to 5.0 or 5.1, then that setting would be ignored, and the build would compile with iOS SDK 6.0. These apps should run on devices running iOS 4.3 or better.

    If you really need to compile against 5.0, the only way I know of is to install an older Xcode that comes with that SDK. If you do this, you need to run "sudo xcode-select -switch <your Xcode path goes here>" in terminal then restart studio to ensure the setting is used.

    For your second item:
    If your CLI installation is somehow not functioning, you may experience issues here. But aside from that, Studio will filter what you see based on your workflow. For example, if you want to run on device, you will only be presented with a list of development provisions. If you package, you will see only distribution provisions (Ad-hoc or app store). If you see none, try making a change to the list in Xcode, and this should force an update.

    If you are able to build, but the console outputs a problem with the provision, paste the info and I can provide more help.
