    The problem is on line 3 of your code:

    Playersdb.remove();

    which deletes any previous version of the database. This is a technique which is useful ONLY if you want to force the database install on the next line to ALWAYS install a fresh copy from the resources directory.

    Normally, you'd skip lines 2&3 (the open and remove) and just use line 4 (the install). The install command will create the database if necessary using the default values in the resource folder, but leave an existing database alone. That will preserve the user data.

    You may have seen another thread which used the open/remove/install sequence, but that is specifically ONLY when you want to force the resource folder copy to get used.
