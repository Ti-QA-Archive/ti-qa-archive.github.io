toImage() no respecting image transparency on Android

Using Framework 2.0.1
MacOSX Lion
Android SDK 2.2

The following code allows the pin to appear correctly on iOS, but appears with an unwanted black background on Android. Setting backgroundColor = 'transparent'. I tried changing the backgroundColor to other colors but it remains black, indicating that it has nothing to do with that setting.

// Creates a dynamic map annotation
        var v = Ti.UI.createView({
            width: 130,
            height: 39,
            backgroundImage: '/images/pin.png' //PNG with transparency
        });
        var label1 = Ti.UI.createLabel({  //Text that goes in the pin
            color: '#fff',
            width: 130,
            height: 27,
            top: 0,
            font: { fontSize:14, fontWeight: 'bold' },
            shadowColor: '#444',
            shadowOffset: {x:1, y:1},
            text: 'Pin text',
            textAlign: Ti.UI.TEXT_ALIGNMENT_CENTER,

        });
        v.add(label1);
        // This is the only way I could get the image to appear on android (setting toImage() to an ImageView first)
        var i = Ti.UI.createImageView({
            image: v.toImage(), // Takes a snapshot of the view
            width: v.width,
            height: v.height
        });        
        pin.image = i.toBlob(); //puts the image blob as an annotation
