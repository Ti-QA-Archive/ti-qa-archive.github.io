I've tried the Following and it works as expected:

Cloud.Objects.query({
        classname: ' ... ',
        page: 1,
        where: {
            user_id: .... 
        },
        per_page: 10
    },
    function (e) {
    }
);

Though in reality, every User has their own id, and I have named the custom Object when created by their user_id. If you follow the Documentation here: http://cloud.appcelerator.com/docs/api/v1/acls/show to show rather than query, I get an error from the server: 

code = 400;
error = 1;
message = "Missing object id(s)";

The Code I am using is:

Cloud.Objects.show({
    classname: Ti.App.Properties.getObject('account').username,
    },
          function (e) {
                if (e.success) {
                     Ti.API.info(e)

  • Hi Joseph,
    You need to give the id of the object you want to show. as in docs

    Cloud.Objects.show({
    classname: Ti.App.Properties.getObject('account').username,
    ids : [savedobject1.id, savedobject2.id],
    },
          function (e) {
                if (e.success) {
                     Ti.API.info(e)
                } else {
                     Ti.API.info(e)                        
                 }
    });

    If you do not give id you will get that error. It is required parameter to show an object from ACS.
    Hope it will help.
