  • I just did a project with the nApp module. I had two windows, a main window (called nav) and a left window (menuDrawer). In my case, the content on the main window is pretty sparse, but the menu could change/update, so I used 'require' to add it to the xml where needed. That is really the approach you should take with Alloy… Still developing the app, so if it turns out the main screen gets more complicated, I will extract the contents to another xml file and include it where needed in the main.xml file. Here is how I set it up:

    Main.xml

    <Alloy>
    <View id="nav" platform="android">
        <Menu id="menu">
            <!-- Cannot specify node text.  Use attributes only. -->
            <MenuItem id="Menu" itemId="1" visible="true" onClick="toggleMenu" showAsAction="Ti.Android.SHOW_AS_ACTION_ALWAYS">
            </MenuItem>
        </Menu>
        <ImageView image="/images/zlogo.png">
        </ImageView>
    </View>
    <View id="menuDrawer" top="20" platform="android">
        <Require src="menuDrawer">
        </Require>
    </View>


    <Window id="menuDrawer" top="20" backgroundColor="#fff" platform="ios">
        <Require src="menuDrawer">
        </Require>
    </Window>

    <NavigationWindow id="nav" onFocus="notify" platform="ios">
        <Window id="loginWin" >
            <LeftNavButton>
                <Button id="menu" image="/images/menu.png" onClick="toggleMenu">
                </Button>
            </LeftNavButton>
            <ImageView image="/images/zlogo.png">
            </ImageView>
        </Window>
    </NavigationWindow>
</Alloy>

    menuDrawer.xml

    <Alloy>
    <TableView>
        <TableViewRow title="My Forms" onClick="openOne"></TableViewRow>
        <TableViewRow title="My Tickets"></TableViewRow>
        <TableViewRow title="My Inspections"></TableViewRow>
        <TableViewRow title="My Procedures"></TableViewRow>
        <TableViewRow title="My Profile" onClick="showProfile"></TableViewRow>
    </TableView>
</Alloy>

    main.js

    var NappDrawerModule = require('dk.napp.drawer');

if (OS_IOS) {
    var drawer = NappDrawerModule.createDrawer({
        leftWindow : $.menuDrawer,
        centerWindow : $.nav,
        //rightWindow: Ti.UI.createWindow({backgroundColor:"blue"}),
        closeDrawerGestureMode : NappDrawerModule.CLOSE_MODE_ALL,
        openDrawerGestureMode : NappDrawerModule.OPEN_MODE_ALL,
        leftDrawerWidth : 200,
        //rightDrawerWidth: 220
    });
}

if (OS_ANDROID) {
    var drawer = NappDrawerModule.createDrawer({
    fullscreen:false,
    exitOnClose: true,
    leftWindow: $.menuDrawer,
    centerWindow: $.nav,
    fading: 0.2, // 0-1
    parallaxAmount: 0.2, //0-1
    shadowWidth:"40dp", 
    leftDrawerWidth: "200dp",
    animationMode: NappDrawerModule.ANIMATION_NONE,
    closeDrawerGestureMode: NappDrawerModule.MODE_MARGIN,
    openDrawerGestureMode: NappDrawerModule.MODE_ALL
});
}

function toggleMenu() {
    drawer.toggleLeftWindow();
}

Ti.App.addEventListener('App:closeMenuDrawer', function(e) {
    drawer.toggleLeftWindow();
});

drawer.open();

    • What do you have in your index file? The error you are getting is from your Index controller, so it's difficult to figure out what might be happening. In my index controller, I figure out if the user is logged in or not and then open the appropriate window. So, my code in index.js is:

      if ( typeof Ti.App.Properties.getInt('currentUser') == 'unknown' || Ti.App.Properties.getInt('currentUser') == null) {
    $.loginWin.open();
} else {
    var mainWin = Alloy.createController('main').getView().open();
};

      and loginwin is defined in Index.xml like this:

      <Alloy>
    <Window id="loginWin" backgroundColor="#fff">
        <ImageView id="zlogo" image="/images/zlogo.png"></ImageView>
        <TextField id="username" hintText="User ID" class="fields"></TextField>
        <TextField id="password" hintText="Password" passwordMask="true" class="fields"></TextField>
        <Button id="login" title="Login" onClick="doLogin"></Button>
        <Button id="signup" title="Sign Up"></Button>
    </Window>
</Alloy>

      So if you are getting the error above, it's because you don't have a 'top level object' like a window in your index.xml file to open.

      Ray
