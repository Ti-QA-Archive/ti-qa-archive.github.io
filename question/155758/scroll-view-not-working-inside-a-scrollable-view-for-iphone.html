    Hi

    As far as I can see you have a number of syntax issues that are likely causing the issues, some examples;

    <ScrollView scrollType:"horizontal">

    Should be

    <ScrollView scrollType="horizontal"> // not the : becomes =

    and

    <ScrollView id="scrollView" scrollType:'vertical' layout:'vertical' height:'auto' width:'auto'>

    Should be

    <ScrollView id="scrollView" scrollType='vertical' layout='vertical' height='auto' width='auto'>

    If you are adding properties to the XML then you use the property=value rather than the JSON way of property:value.

    Although I would include the following;

    <ScrollView id="scrollView" scrollType='vertical' layout='vertical' height='Ti.UI.FILL' width='Ti.UI.FILL' showVerticalScrollIndicator='true' showHorizontalScrollIndicator='false' contentWidth='Ti.UI.FILL' contentHeight='Ti.UI.SIZE' >

    Although I would really want to split out the code so the presentation is in a separate TSS file;

    View

    <ScrollView id="scrollView" >
...
</ScrollView>

    TSS

    #scrollView: {
    scrollType: 'vertical',
    layout: 'vertical',
    height: Ti.UI.FILL,
    width: Ti.UI.FILL,
    showVerticalScrollIndicator: true,
    showHorizontalScrollIndicator: false,
    contentWidth: Ti.UI.FILL,
    contentHeight: Ti.UI.SIZE
}

    • You still appear to be missing a number of properties I would have as standard and those you do have are using the older syntax that is ambiguous which is why it was change - try NOT to use auto.

      Use this as a starting point for your scrollView properties;

      "#scrollView": {
    scrollType: 'vertical',
    layout: 'vertical',
    height: Ti.UI.FILL,
    width: Ti.UI.FILL,
    showVerticalScrollIndicator: true,
    showHorizontalScrollIndicator: false,
    contentWidth: Ti.UI.FILL,
    contentHeight: Ti.UI.SIZE
}

      I would STRONGLY recommend that you create a general class for these scrollViews when they all do the same things. I also see no requirement to wrap a scrollView inside in a view as a scrollView and view are both container views - scrollableView does not require a simple view to work so ONLY use one as a container if you are trying to achieve something else - not to simply wrap the scrollView**.
