Assets of my widget not available

Hey Community

I created a clean Alloy app and added a widget which does simply contain a webview loading a local HTML file (it is located in app/widgets/mywid/assets). But the file doesn't load, if I move the file instead to /app/assets the view does load well. Though it seams that allow does not deploy my widget assets. Have I add some additional settings for that?

    I think you need to use the WPATH() macro… from the docs

    
Assets and Libs

For any files in the assets or libs folder, use the WPATH() macro to automatically map the path relative from the widget's root folder, as opposed to the Alloy project's app directory. For example, if you have a library located at app/widgets/foo/lib/helper.js, to require it, use require(WPATH('helper')); and if you have an image located at app/widgets/foo/assets/images/foo.png, to reference it, use WPATH('images/foo.png').
