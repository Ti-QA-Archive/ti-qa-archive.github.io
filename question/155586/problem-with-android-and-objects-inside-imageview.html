Problem with android and objects inside imageview

Hi,

I'm trying to place inside an imageview some labels…On IOS it works fine, but on android the labels are not showing… has anyone tried something like this?

this is the code that i'm using, at the end the image is added to a scroll ( that i created before)…

var view_image = Titanium.UI.createImageView({
image:<path to image>,
top:"15dp",
left:"15dp",
right:"15dp",
borderColor:"white",
borderWidth:"15dp"
});

var somelbl = Titanium.UI.createLabel({
text:"Some text",
backgroundColor:"white",
bottom:"14%",
right:"7%",
zIndex:100,
});

view_image.add(somelbl);

scroll.add(view_image);

thank you in advance!

    Hi

    The short answer is - this will not work.

    The longer reason is as follows;

    • All buttons, views, labels, imageViews, tableViews etc are either a container or a non-container
    • Views, ScrollViews are containers, but labels, buttons and imageViews are not containers
    • iOS (iPhone & iPad) allows containers and non-containers to be placed inside each other
    • Android will NOT allow containers to be placed inside non-containers.

    This is why it works on iOS but not on Android, this is the normal expected behaviour.

    Work-around

    • View
      • ImageView
      • Label

    The view is the container and both the imageView and label are placed inside it. The label is added last so it appears above the imageView.
