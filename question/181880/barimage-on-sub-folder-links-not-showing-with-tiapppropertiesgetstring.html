barImage on sub folder links not showing with Ti.App.Properties.getString

3.5.2sdk

i'm detecting ios platforms by using this

var platformHeight = Titanium.Platform.displayCaps.platformHeight;
        if (platformHeight==736){   //6+ 
                var navbar = '/navbar1080.png';
        }
        if (platformHeight==667){   //6 
                var navbar = '/navbar1080.png';
        }
        if (platformHeight==480) {   //4 
                var navbar =  '/navbar640.png';
        }
        if (platformHeight==568){   //5 
                var navbar =  '/navbar640.png';
        }
Ti.App.Properties.setString('navbar', 'navbar');

and then in the js files

var win1 = Titanium.UI.createWindow({  
    url:'link.js',
    barColor:'#000',
    barImage:Ti.App.Properties.getString('navbar'),

*snip*

this works fine for js files that are in the root but when the url is a sub folder the barImage doesn't show ?

i even tried putting the 2 navbar images in the folders and using

var navbar = 'navbar640.png'; etc without the forward slash

but that doesn't work either ?

    Ti.App.Properties.setString(&#39;navbar&#39;, &#39;navbar&#39;);

    You just set a property with the key navbar to literally be the string &#39;navbar&#39;
