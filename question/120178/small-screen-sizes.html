Small screen sizes

Hi,

I am trying to modify my app to work on an HTC Wildfire… which has a really small screen.

The first version of the app had fixed resolution;-

<screens small="false" normal="true" large="false" anyDensity="false" />

This was fine for everyone except the very small screens and was not available in the app store for my friend with the wildfire.

So, now i am trying to get to grips with the various screen sizes.

<supports-screens
                    android:largeScreens="true"
                    android:anyDensity="true"
                    android:normalScreens="true" android:smallScreens="true" />

In my Resources/android folder i now have images/low, medium and high. This doesn't seem to be picked up, but that is not the main problem.

In one screen i am using a scrollable view which just crashes the app with very little trace info.
It appears that it does not like scrollbars.

SO, now i am trying to use layouts for different screens.

On the Android docs it mentions using a layout folder in the 'res' folder.
To try and do this with Titanium i have put a layout folder in the Resources/android folder with a layout-small folder in it containing the page that is crashing… 

Resources/
         android/
                /images
                       /low
                       /medium
                       /high
                /layout
                       /layout-small

Still seem to crash with the same error relating to the scrollbars.. am i heading in the right direction.. is there a way to make the screen scroll without a scrollableView..

Thanks,

Jez

    use different style sheets for the different resolutions on android.

    name the style sheets like:

    windowfilename.android.low.jss,
    windowfilename.android.medium.jss,
    windowfilename.android.high.jss

    Put the specific code for each screensize in the file. titanium automaticaly uses the right one.
