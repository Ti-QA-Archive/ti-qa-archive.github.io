Filesystem: createTimestamp and modificationTimestamp are the same

Hi,

I'm currently working with Titanium.Filesystem. The problem is when I want to match if the file is older then 1 day, if so then update the file. When I used createTimestamp and modificationTimestamp they always get the same result.
response:

var f = Titanium.Filesystem.getFile(Titanium.Filesystem.applicationDataDirectory,klas + '.json');

Ti.API.info("Created: " + String(new Date(f.createTimestamp())));
Ti.API.info("Last modified: " + String(new Date(f.modificationTimestamp())));

result:
> [INFO] [2170,3044] Created: Tue Mar 15 2011 15:29:35 GMT+0100 (GMT+01:00)
> [INFO] [11,3055] Last modified: Tue Mar 15 2011 15:29:35 GMT+0100 (GMT+01:00)

result 2 (after the update):
> [INFO] [706,1651] Created: Tue Mar 15 2011 18:39:46 GMT+0100 (GMT+01:00)
> [INFO] [14,1665] Last modified: Tue Mar 15 2011 18:39:46 GMT+0100 (GMT+01:00)

How I update the file (or write if it doesn't exists):

.. more code ..
var j = JSON.parse(this.responseText);

// write JSON data to local storage
var file = Titanium.Filesystem.getFile(Titanium.Filesystem.applicationDataDirectory,'test.json');

file.write(JSON.stringify(j));
.. more code ..

It seems like file.write(..) just overwrites the file

I hope anyone has experienced the same thing ?
