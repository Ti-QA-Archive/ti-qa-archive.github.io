  • Hello, Arif. This is not the way to ask question in Community. Please next time follow the instruction in this LINK for asking questions.

    Now as i don't know what process you are following, So, According to Link "Bottom Tabgroup will be supported up to Titanium 3.0, it will not be supported for Titanium 3.X"
    The reason because
    Before SDK 3.0, Titanium used TabActivity to implement the TabGroup. Since TabActivity is deprecated, Titanium now use TabHost to implement the TabGroup for API level < 11 in SDK 3.0. We define the TabHost layout in the code. That's why the custom TabHost layout xml file does not work anymore. Based on the guideline of Pure Android, the bottom tab bars are not recommended. In addition, for API level >= 11 we use ActionBar to implement the TabGroup, in which case tab bars can only be on the top. In order to provide a consistent experience, we don't recommend using bottom tab bars.

    There is a JIRA Ticket on the related issue which is now closed with comments that it won't be fixed.link

  • Because on Android, tabs are at the top – http://developer.android.com/design/building-blocks/tabs.html

    They're on the top because the device buttons (home, back, menu, recent apps) are at the bottom. If your tabs were there, it would be too easy for a user to tap a device button accidentally when they were going for the tab.

    Besides, Android users don't want iOS apps (iOS apps have tabs at the bottom). You should target your design for the target platform.

    The Tweetanium app linked to is very old and currently represents an antipattern for coding organization.
