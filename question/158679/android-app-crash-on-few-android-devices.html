  • Hi Varma,

    A much better sollution (rather than logcat from console) is to find the      path_to_android_sdk\tools\ddms.bat (Dalvik Debug Monitor).

    You can do lots of cool stuff from there, view a live log on what goes on in either the emulator or on a real device via usb, you can make fake calls to the emulator/device, you can send text-messages to it, take screenshots, and loads of other things. It baffles me that no one in here promote it better for Android-debugging. Thanx.

    • At last I resolved this issue by adding few more lines of code to the imageView. 

      var picture = Titanium.Utils.base64decode(a.item(j).textContent); 

    var imageView = Ti.UI.createImageView({
         image:picture,
         left: 15,
        width:100,
        height:100,
      });

     if(picture.height >= 110) {
         imageView.size.height = 110;
    }
    if(picture.width >= 110) {
         imageView.size.width = 110;
    }
    picture = imageView.toBlob();

    var row10 = Ti.UI.createTableViewRow({
    height :120
     });

   row10.add(imageView);

      After adding the if condition for image width & height properties app is working fine without any memory leaks.
