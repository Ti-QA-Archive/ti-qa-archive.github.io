Ti.Map prompt access to user location.

It seems that just adding a simple mapView prompts the user for access to his or her location data. I don't wan't the users to think that I am using this data when I am not. Can anyone confirm that this is the normal/intended behaviour of Ti.Map, or perhaps come up with a workaround.

Note: The userLocation property is set to false.

Thanks :)

Application type: mobile
Titanium SDK: 3.2.0 GA

    Hi

    The userLocation property determines if the pulsing blue dot is shown on the map - it has nothing to do with the user giving permissions.

    Because the map has the ability to locate a user by the press of a map icon as well as routing a user from their current location - the map sub-system (at OS level) requests permission for those things.

    This is normal behaviour of the map. Whilst I can imagine some getting confused by this it must (hopefully) be a small percentage as the map can;

    • Centre to your current location
    • Provide a rout to or from your current location
    • Track the map to your location(so it moves with you)
    • When being routed it know where you are on the route

    These all require your location.

    • As the the user not knowing what you do with any information stored - great question - tell them.

      The app world and internet world are becoming increasingly aware that their information can be misused - so make a point of having a privacy policy in your app - make it important enough to be prominent and be honest in clear sentences.

      As an example; if you retain someone's phone number because your app manages contacts (not saying it does) - then explain that. If you app keeps phone numbers and it is a game - very little normally legitimate reason. So trust your user to understand what and why and they will trust to handle things correctly.
