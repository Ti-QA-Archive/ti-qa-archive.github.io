XML + XPath query to access attribute '@' triggers error

Hi,

First of all, I am testing the code in iPhone, using SDK 1.7.0.

I was trying some queries with XPath, using the xml string from Kitchen Sink, which is :

var xmlString = '<?xml version="1.0" encoding="UTF-8"?>\n'+
            '<response>'+
            '        <nodes id="nodes">'+
            '            <node id="node 1">'+
            '                <node id="node 2">'+
            '                    <node id="node 3"/>'+
            '                </node>'+
            '                <node id="node 4">'+
            '                    <node id="node 5"/>'+
            '                </node>'+
            '                <node id="node 6">'+
            '                    <node id="node 7"/>'+
            '                </node>'+
            '            </node>'+
            '            <node id="node 8">'+
            '                <node id="node 9">'+
            '                    <node id="node 10"/>'+
            '                </node>'+
            '            </node>'+
            '            <node id="node 11">'+
            '                <node id="node 12">'+
            '                    <node id="node 13"/>'+
            '                </node>'+
            '            </node>'+
            '        </nodes>'+
            '</response>';

I do the following in code:

var doc = Ti.XML.parseString(xmlString);    
var xpathQuery = "//node@id";            
var result = doc.evaluate(xpathQuery);

And I get the error : XPath error : invalid expression, with the arrow pointing to the '@' in the query….and that result = [NULL]…

What is wrong with this XPath expression ??
Or it is linked to the device, which do not allow access to node attributes…??

Any ideas, solutions ??

Thanks in advance !
