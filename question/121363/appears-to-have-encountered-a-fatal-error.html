    A little bit more insight to the problem.

    I have different user accounts on the mac, admin, developer, kids etc. And Titanium used to run on all of them. 3 months ago, when I had the premium membership, I was using titanium under the "developer" account. The developer account has the same admin privilidges as the "admin" account.

    After I hit that fatal error ( which started happening ith the latest download of titanium) , I wanted to try the one under "admin" account. This test is of course after my sending that original question.

    The admin version has worked. Then I tried the other accounts on the mac. and it turned out that every single failed, failed with the same fatal message. At this point, I don't see it as an error but as a licensing issue. It would be nice to get it working under my "developer" account but I'm going to move on with the temporary solution I found.

    Anyway, the bottom line is I was able to download the kitchensink and run it full fledged under 1.2.2 Titanium. The only error I encounter under the kitchensink app is when I click on the "Display Contacts". I believe "Display Contacts" has an error in its javascript cause without an exemption, kitchensink exists out as soon as I click the "display contacts".
