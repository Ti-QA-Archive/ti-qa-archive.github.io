  • Hi Ket :)

    This may be caused by this eclipse bug. Hence, it appears that there are two possible solutions, explained in the bullet points below.

    Important: Copy your whole workspace first, so that you are not risking your original files when you attempt these steps:

    • Rename the file workspace\.metadata\.plugins\org.eclipse.core.resources\.root\nn.tree, where nn is a number, and reduce it by 2. As an example, if you have the file 82.tree, rename it to 80.tree, and restart Studio
    • Delete the file .metadata&#x2F;.plugins&#x2F;org.eclipse.core.resources&#x2F;.snap, and restart Studio

    Let me know which, if either, of these work.

    Cheers

    • Ket

      Just to conclude this thread, I regret that there is unlikely to be a way for us to prevent this issue from happening altogether.

      It looks like it is normally caused by some sort of crash or other situation, where the product was interrupted in saving workspace data (I.e. power loss to the computer, permissions, full disk, corrupt disk segment etc). Normally it doesn't cause a problem to this extent, but it can on rare occasions.

      Studio is built on top of 3.6 currently, though we do plan to update to 3.7 at some point (no ETA there). It appears they made the message better (if not reducing the occurrence altogether) in Eclipse 3.7.

      So, to reiterate, deleting the .snap file will remove the projects from the workspace and resolve the error. Then they can be imported back in which, although is not ideal, is easy-enough to do. Note that the only metadata we have is in a .project file.

      Sorry this isn't a full resolution to the issue, but I hope has been enough to get you working again.

      Cheers
