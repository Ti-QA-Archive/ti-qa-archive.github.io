Rewrite detailWindow with a new one, instead of adding a window?

Hi, I'm developing a iPad app, with the latest Titanium. 1.6.1.

My masterView has a tableView, and when i click on a row, the detailWindow is supposed to open the "tableView-item". This works great. Then I wanted to add a basic navigation, so I can switch categories inside the MasterWindow. This also works great. Then I wanted to add so if I change category, the first item in the tableView opens in the detailWindow.

That also works, but here's the tricky part. My logic is that I add a new window, to the DetailWindow, and then open that window up. I don't want to do that, I want to rewrite the DetailWindow instead of append a new window inside of it.

Because: I don't know if it's a bug, but because of some reason an invisible window opens up above 80% of my screen so I can't use large parts of the app, but I can reach some spots (down left & right corners).

So I figure that maybe I could rewrite the window instead of append a new one inside of that one.

When I click on a row:

    itemWindow = Ti.UI.createWindow({
        url: "item.js",
        background: "#FFFFFF",
        width: "100%",
        height: "100%",
        id: item_id,
        detailNav: win.detailNav
    });

    detailWindow.add(itemWindow);
    itemWindow.open();

I've tried with detailWindow = itemWindow; but it doesn't work at all.

in my app.js:

Ti.include("settings.js");
Ti.include("func_lib.js");

var masterWindow = Ti.UI.createWindow({
    url: "pages/home_nav.js"
});

var detailWindow = Ti.UI.createWindow();

var masterNav = Ti.UI.iPhone.createNavigationGroup({
    window: masterWindow
});

var detailNav = Ti.UI.iPhone.createNavigationGroup({
    window: detailWindow
});

var splitView = Titanium.UI.iPad.createSplitWindow({ 
    masterView: masterNav, 
    detailView: detailNav 
});

// add the detailWindow-object to the MasterWindow
masterWindow.detailNav = detailNav;

splitView.open();

And at the top of home_nav.js (the MasterView):

var win = Ti.UI.currentWindow;
var detailWindow = win.detailNav.window;

Anyone who might know how to solve the "invisible-window"-bug?
Or rewrite the window? Is my logic idiotic?
