Xcode path problems

I just went to update Xcode to 4.3.1, which is now controlled via the App Store. It installed the Xcode app package into /Applications, and on first launch offered to remove older Xcode installations, which I did.

Xcode launches and runs perfectly on its own.

There is no longer a /Developer folder on my System; Xcode is not a package with the contents including several apps and folders, including the Developer folder.

I have used xcode-select to check the path, which was still /Developer.

After experimenting, I found that setting Xcode path to /Applications/Xcode.app/Contents/Developer would allow me to issue an "xcodebuild" command in terminal build a non-titanium project.

Finally, Titanium Studio sees that I have a valid iOS SDK installed. So I build an app to run on my device as usual. App builds and shows up in iTunes.

In the past, since this iPad is not synched to this iTunes Library, I would install the IPA with XCode's Organizer window. However, now this fails with

"No application exists in /Users/stevelombardi/Music/iTunes/iTunes Media/Mobile Applications/Tomato Match.ipa"

Ideas?

SDK 5.1, Titanium Studio, build: 1.0.10.201203061444, OS 10.7.3
