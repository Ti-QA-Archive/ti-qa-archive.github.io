Do I give tableViews unique class names or not?

There's a lot of talk on this QA board about giving rows in a table view a class name, but some people say the rows with the same formatting should all have the same class name (which makes sense to me), but others say the rows need their own unique name or the tableView will crash or render slowly. Also, a lot of the discussion about this is about year old, so I was wondering if someone might be able to give me updated info about what to do.

To give a bit more info, my tables have mostly very similar rows. They always contain the same number of labels, always placed in the same place, with the same font properties. The only thing that's different is a "Loading…" row at the bottom of the tableview that is inserted for pagination. All the rows except that "Loading" row have the same class name, called "tableRows". Is this the correct way to go about it, or should I be giving each row a unique classname like "tableRows1", "tableRows2", etc.?

    Tables that use similar formatting should have the same classname. This will increase the performance of your tableviews.
