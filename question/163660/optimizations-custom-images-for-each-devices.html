    Hi

    It looks like you are attempting to choose a file name based on the device dimensions. Whilst that is a way to achieve your goal there are easier ways.

    You should read the following answers I gave some time ago that explains how to manage different layouts across different devices and platforms.

    • How to achieve same layout for Android and iPhone using Titanium
    • images in Android (also iOS)

    The essence here is that you place your images into the correct density folders and then simply reference the image by name without alteration. The device will pick the correct image using the density info it has about itself. Everything is much simpler this way.

    This will give you all the information you need to get manage your cross platform and multiple device management.

    • I did not see anything about the images coming from a remote web service. But the logic remains the same.

      Store your images in different folders and name then the same as the density property returns. Return just the filename in your web service and then insert the correct folder.

      Something like this;

      var density = Ti.Platform.DisplayCaps.density;
var url = "http://www.something.com/mainfolder/";
var file = 'somethingfromthexmls.png';

var imagepath = url + density + '/'' + file;

      This is what I do in my apps. This assumes you are in control of the web service - as I imagine you are otherwise you would not be asking how to control it.

    • Your English via Google Translate is very good.

      OK you need to work out which is the best choice between two files (or images) to use as you do not want to create each specific resolution.

      The solution is to ignore pixels for width and height as they only contain half of the information. How close those pixels are to each other is the other half.

      var factor = Ti.Platform.DisplayCaps.logicalDensityFactor || (Ti.Platform.displayCaps.density === 'high' ? 2 : 1);
if (factor >= 2) {
    // one way
} else {
    // other way
}

      This will work out what the logical density factor answer is, think; 0.75, 1, 1.5, 2, 3 and so on. This property ONLY works on android though. So the logical branch finds the answers if not on android. An iOS device with high as an answer is a retina display so has an answer of 2, non-retina return 1.

      So the code above allows you to choose one or the other set based on the density factor. I have chosen 2 as the cut off - you can choose whatever fits your needs.
