  • Phone Gap and Titanium are very different technologies.

    Firstly lets cover phone Gap..

    It is a wrapper enabling the development of an application using web based technologies. HTML CSS etc.

    It is very effective for quick development and easy transition from a web development background. It does have quite a performance hit.

    Appcelerator Titanium is very different.

    As I am sure you are aware it's base coding language is JavaScript, with the Titanium name space providing the API's into the specific platforms requirements.

    It uses the platform (IOS, Android) SDK's to compile into a near native application. Very brief explanation.

    It is best described as a code once adapt everywhere platform, which means that all the business logic, data handling etc is fully usable across platforms, and most of the UI code is compatible.

    But different platforms have different requirements, so it enables within the single code base the ability to code specifically for the platform and device form factor, normally with a 80 - 90% single code and 10 - 20% being platform or form factor specific. It is however as mentioned a single code base.

    Having been developing Titanium applications for nearly 4 years, I have seen the good and the bad. It is now a very stable platform, ideal for Enterprise applications.

    So which is best…

    I can't actually answer that, it depends on your requirements. If you after a quick small cross platform application maybe Phone Gap, but if you're into full enterprise applications then it's Native or Appcelerator.

    Hope this helps

    T.
