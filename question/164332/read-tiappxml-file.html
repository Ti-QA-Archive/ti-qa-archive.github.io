  • Hello Ankita Gadhia,

    If you defined app properties in the tiapp.xml file then you can access using Titanium.App.Properties. An app property is defined inside the root-level <ti:app> element with the <property> element. Use the name attribute to define your property name and place the value inline with the <property> element. For example, to define an app property called foo with the value 42:

    <?xml version="1.0" encoding="UTF-8"?>
<ti:app xmlns:ti="http://ti.appcelerator.org">
    <property name="foo" type="string">42</property>
</ti:app>

    Then, you can retrieve the property during runtime with the following API call:

    var foo = Ti.App.Properties.getString('foo');

    As of Release 3.2.0, any application properties defined in the tiapp.xml file are stored in the device's secure storage, making them read-only. Additionally, external access to these properties is now restricted. Other iOS applications cannot access these properties and native Android modules must use the Titanium module API TiApplication.getAppProperties method to access these properties.

    If you need to change the values during runtime, initially create the property with these APIs rather than defining them in the tiapp.xml file.

    Prior to Release 3.2.0, application properties defined in the tiapp.xml file could be overwritten by these APIs and accessed externally by other applications and modules.

    For more information please visit Titanium.App.Properties

  • The tiapp.xml file gets converted during the pre-compile phase and does not get installed as part of the app.

    What I suggest is writing your own Alloy plugin that copies it to a folder within the app folder so that it gets distributed with the app, then you should be able to access it using the normal file system objects.

    There is a good blog about it here which should cover all you want. Plugins are written in Javascript and arent that difficult to write providing you can work out what objects/methods are available via Alloy.

    Ones I have used are:

    var path = require('path'); //object for doing folder path stuff
var afs = appc.fs; // appc is passed as a parameter to init, fs is an object similar to the file system object

...
//projFolder is where the tiapp.xml lives
var projFolder = cli.argv['project-dir'];

...
//appFolder now points to the app folder
var appFolder = path.join(projFolder, 'app');

...
//returns true or false as to whether the fully qualified path name exists
var cont = afs.exists(pathName);

...
//The following copies the source folder to the dest passing in the logger (parameter to init)
afs.copyDirSyncRecursive(source, dest, {
    logger: logger.info
});

    Hope this helps
