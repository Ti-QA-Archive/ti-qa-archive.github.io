How are localized applications listed on App Store?

This may be more a question of how the app store works, but I'm trying to get the groundwork done correctly before I'm too far in.

I'm working on a iPhone app that is being localized to a few different languages. I've found some very good information about localizing it. = (http://developer.appcelerator.com/question/92471/localization-not-displayed-on-the-appstore and http://developer.appcelerator.com/question/98921/how-to-localize-the-name-of-your-app-and-the-splashlaunch-image )

But how will the app appear in the app store itself? I would like the app to have one name in English and a completely different Chinese name (and one for Spanish) as well as completely different text descriptions for those languages. Will users be able to find my app by searching by the Chinese name and description? Or will I have to upload three different applications each with a different name and description? And what about screenshots of the application? I'd prefer to show the Chinese audience the Chinese screenshots, etc.

When submitting the iPhone app to Apple's app store I know I have the opportunity to select which languages it supports (or perhaps that will be auto-detected from the localization files that are part of the project). But I'm unclear on how the rest is handled.

Thanks,

Chris

  • When you submit the app to the app store you MUST introduce locale information for each language supported, what includes name, description, images and keywords. So, each language will have their own info completely separated from the rest. Users only will see info in their language if available (if not, they will see your default language).

    Appart of this, if you are using the i18n folder (I guess), on each language folder, you can include a file called app.xml with this content:

    <?xml version="1.0" encoding="UTF-8"?>
<resources>
    <string name="appname">You app name</string>
</resources>

    this is the name of the app once installed on device, customized for each language. So, you can show the app name in chinese for China and so on.
