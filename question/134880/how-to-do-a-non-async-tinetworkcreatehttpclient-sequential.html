How to do a Non async Ti.Network.createHTTPClient ..sequential

I have to do some servers calls into the same code block (into another onload of a HTTPClient after receiving a response)
So instead of a async request.. I have to do a sync request that won't call a function on done ..etc etc… (this already won't block the app because it's in another threat so no problem)

  • Create an array and use it as a queue for all of your request. Then create a recursive function that will loop through the queue. It should Array.pop() or Array.shift() each request, and the onLoad argument should have some logic that tests to see if the queue if empty. If the queue isn't empty, it calls your recursive function again. If the queue is empty, it simply returns.
