  • I found a way to do what I needed. Actually the toImage function use the dimension of the view. In the case of a scroll view that the same thing. Maybe that's a bug but the function does not use contentWidth and contentHeight.
    So just before apply the toImage function I change the height of my scroll view:

    myScrollView.setHeight(contentHeight);
var myImage = myScrollView.toImage();
myScrollView.setHeight(originalHeight);

    Now I can export the whole content of the view but that made an image with height >> width. Maybe I will try to segment my view and create several images with proportional dimensions.
