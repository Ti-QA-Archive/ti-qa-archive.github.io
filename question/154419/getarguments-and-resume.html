getArguments and resume

I want my app to receive files from other apps (for example mail attachments)
I've followed many snippets around and every one suggest using Ti.App.getArguments()

Ti.Arguments indeed works, if the application is initially closed.
However, if the application is in background or is paused, Ti.App.getArguments() (called within 'resume' event listener) is not well behaving:
sometimes it is working, sometimes the url is missing, sometimes the source is missing and sometimes getArguments returns null (I think this is completely a bug since inspecting getArguments during a clean start gives correctly no url or source, but gives an (almost) empty object, not a null). I also noticed that after getArguments returns null for the first time, it keeps failing.

my question is the following: should I really rely on Ti.App.getArguments() or should I look for another function? Because the documentation states that getArgument returns the agruments given during startup, and not during 'startup OR resume'. Otherwise, I've found many tickets regarding the fact that Ti.App.getArguments should return something different if called after a resume. Should I file a ticket or am I misunderstanding something?

Many thanks
