Hi *,

I try to realize a search in video DB:

self.search.addEventListener('return', function() {
        self.tv.setData([]);

            for (var i=0;i<_data.length && i<5;i++) {
                rows.push(require('modules/parts/movierow').create(_data[i],i));
            }
            self.tv.setData(rows);
            if (_data.length>=5) {
                for (var i=5;i<_data.length && i<100;i++) {

                }
            }
        };
    });

If I dont limit the count of rows the app is frozen. In this case over 1000 data sets comes from model.
The trick with first 4 rows accelerate the rendering ;-))

What can I do to show all entries … dynamic loading ..

    Hi

    If I read the question correctly, then you have LOTS of rows, but you are concerned with performance.

    My first thought would be;

    • Download the data as you are already
    • Cache the results in a local sqlite database
    • Show the first few records - 20-50 only
    • Detect the bottom position of the table (answer given just the other day by me in Q&A)
    • Show another "batch" as required.

    If you can control the results on the site, consider a paging mechanism and only download on need - using the "lazy load technique".
