Tabgroup reopening crashes app when using activity indicator

Hi.

With the help of this topic: http://developer.appcelerator.com/question/129341/tabgroup-reopen-crash I managed to isolate this issue: https://gist.github.com/1476642

To see the issue run the example (I use latest 1.9 SDK, Android 3.1, Window XP), click on "Click me!" button, press hardware back-button and then again click on button (so, you want to re-create tab group). App should crash. Comment out Activity indicator related code and app will run fine.

This is a problem of course if there is a lot of data in tabs and it takes some time to create tab group, tabs and tab content. Therefore, I was wondering if someone has a workaround how to let the user know something is going on and not to crash the app?

Also, tab events are not being fired. From the docs, tabs should support "close" event, but it seems it's not fire - at least not when using hardware back-button. So I wonder how to know if tab group is closed? "blur" event? But it doesn't guarantee tab group is actually closed.

Thanks for your help.
