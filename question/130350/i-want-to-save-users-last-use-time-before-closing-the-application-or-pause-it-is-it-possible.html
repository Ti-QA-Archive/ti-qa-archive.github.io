  • When working with iOS the pause event is your best strategy. There isn't an event fired when the app is terminated to my knowledge.

    You can look at the KitchenSink's app.js for examples on how to catch the pause and resume events. https://github.com/appcelerator/titanium_mobile/blob/master/demos/KitchenSink/Resources/app.js

    If you are going cross platform or focused on Android you will need to look at a different strategy. On Android each activity fires the pause and resume. So without an app level event you will need to structure your app to persist the data as you go. The one global for Android would be using the resume/pause events off the TabGroup. At least that is the only way I've found.
