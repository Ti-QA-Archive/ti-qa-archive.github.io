Doctype in webview causes ReferenceError: Can't find variable: Ti on android

This is more of an answer in case others are facing the same problem. I'm not sure if this is a bug and if I should file it on JIRA?

app.js

var win = Ti.UI.createWindow({});
var view = Ti.UI.createWebView({
    url:'sample.html'
});
Ti.App.addEventListener('hellothere', function() {
    alert('hello world');
});
win.add(view);
win.open();

sample.html

<!doctype html>
<body>
    <script>
        Ti.App.fireEvent('hellothere');
    </script>
</body>
</html>

The above returns "ReferenceError: Can't find variable: Ti" on android 2.2, SDK 1.8.2. Removing the !doctype "fixes" the error.
