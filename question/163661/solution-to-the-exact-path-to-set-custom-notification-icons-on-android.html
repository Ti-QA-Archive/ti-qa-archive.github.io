Solution to the exact path to set custom notification icons on Android

After wasting hours on a lot of tests, reading docs and searching in forum, I finally understood where the custom icon for notifications goes.

The docs explain it, but it is not so clear for noobs.

The EXACT path to icon must be

PROJECT_FOLDER\platform\android\res\drawable

and not APP_FOLDER\platform\android\res\drawable, that was I understood on docs.

I believed that all the icons and artwork must go always inside the app folder, under assets or something similar, but not in this case.

I saw a lot of questions about this on the forum, most unresolved, so I hope this tip can help someone to don't lose a whole day like me in a lot of tests.

    Hi

    Yes this is due to the differences in the aims of the files themselves.

    • {appname}/app/

      Files here are related to the Alloy and JavaScript code you create

    • {appname}/resources/

      Files here are EITHER the pre-processed Alloy files or the non-Alloy working files

    When you have folders like these;

    • {appname}/i18n/
    • {appname}/platform/
    • {appname}/models/

    They are for things that are outside your actual direct code, so international string, custom modules (do not confuse with CommonJS) and platform specific things that are OS level rather than code controlled level.

    A little bit confusing, but starts to make sense after a while. The docs should be updated a bit on this subject to provide examples so the app folder only used with Alloy are at least pointed out - so in this case they can be ignored for relevance.

    For this reason - I always write the folders out as above, helps to clear things up a bit.
