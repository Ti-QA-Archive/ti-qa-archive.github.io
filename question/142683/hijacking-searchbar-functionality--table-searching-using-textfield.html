Hijacking SearchBar Functionality | Table Searching Using TextField

I am attempting to mimic the functionality of the SearchBar component with a TextField for style reasons. Here is the basic idea:

  • Create my main table of information.
  • Create a hidden (visible : false) SearchBar and hook it up to the table via search : mySearchBar.
  • Create a TextField component and create a 'change' event listener on it which manually assigns the TextField value to mySearchBar via mySearchBar.setValue(e.value).

However this is having unexpected results. I believe there are a few issues.

  1. mySearchBar.setValue(e.value) does not seem to be working. It injects the object as a string as the value instead of the actual text.
  2. I don't fully understand how the filtering works. It seems as though there is a hidden unstyled, unpopulated table that shows behind my actual table when the mySearchBar is hidden, but not when it's visible. Kind of strange.

I understand this whole idea is a hack but it seems like it could work. Can anyone steer me a better direction or shine some light as to why this won't work? The filtering works fine when the bar is not hidden and not manually populated via the setValue method.

FWIW, the whole reason for this is to avoid having a background (barColor) on the search bar.

Thanks a lot.
