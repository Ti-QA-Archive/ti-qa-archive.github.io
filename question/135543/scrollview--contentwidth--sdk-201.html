scrollView + contentWidth + SDK 2.01

contentWidth seems to be broken. In 1.82 the 'views' added to scrollView would fall underneath each other create tiles going down the window. In 2.0.1 they extend past the desired 'contentWidth' of 320.

Had to roll back to SDK 1.82

var scrollView =  Ti.UI.createScrollView({
    contentWidth: 320,
    contentHeight: 'auto',
    layout: 'horizontal'
});

var things = [
    'item1',
    'item2',
    'item3',
    'item4',
    'item5',
    'item6' ];

for (var i = 0;  i < things.length;  i++) {

    var view = Ti.UI.createView({
        height: 90,
        width: 140,
        top: 10,
        left: 10,
        backgroundColor: '#CCC'
    });

    scrollView.add(view);

};
