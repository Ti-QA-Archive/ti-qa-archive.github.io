    • the only way is to set enabled to false then you will get click event but then opacity of switch will be set to near to 0.5

      the other is to create one view and add switch to that view and set switch's touchEnabled to false then on row's click event you can check that whether its a view so when user click on that view you can manually set switch's values like

      var view = Ti.UI.createView({
    width : Ti.UI.SIZE,
    height : Ti.UI.SIZE
});

view.swtch = Ti.UI.createSwitch({
    value : false,
    touchEnabled : false
});

view.add(view.swtch);
serviceTable.addEventListener('click', function(e){
    if(e.source.valueOf() == '[object TiUIView]'){
        var v = e.source;
        var swtc = v.swtch;
        if(swtc.value){
            swtc.value = false;
        }else{
            swtc.value = true;
        }
    }
});

    • I wouldn't be sure if this will be considered as good practice but here it goes.

      This expands on Gagan's answer so I will only include what he hasn't already mentioned.

          // You have checked the e.source and confirmed it's a button in the tableView's click handler

    e.source.callback(arg1,arg2,argN);

// When defining your buttons
    var button = Ti.UI.createButton({
        prop1: val1,
        prop2: val2,
        callback: function() {
                // Your code goes here
        }
        // Or do something like this so you can reuse the code for multiple similar buttons
        callback: predefinedNameFunction
    });

      That way, you needn't confuse yourself with e.row.index.
