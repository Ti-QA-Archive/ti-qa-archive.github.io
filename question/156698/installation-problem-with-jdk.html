Installation problem with JDK

Hi - I'm entirely new to Titanium and trying to install it for the first time on a Windows 8 machine. Having installed I get the following error when I try to run Titanium Studio for the first time -

"A java runtime environment (jre) or java development kit (jdk) must be available in order to run titanium studio. No java virtual machine was found after searching the following locations C:\Users\gordo_000\AppData\Local\Titanium Studio\jre\bin\javaw.exe
javaw.exe in your current PATH"

I know that I already have the JDK installed because I use the NetBeans IDE for other work.

Anybody any idea what's going on here and how I can solve it?

Gordon

  • This happens if your PC uses a 64x Operating System, while the Titanium Studio is looking into the "Program Files (x86)" for the javaw.exe. I bet you already have JDK, but it is in the "Program Files" because that JDK is registered as the x64 artitecture.

    What you need to do is go and download the JDK again, but for x86 architecture instead and install it. Re run the program.

  • This means that your JDK isn't accessible from your PATH environment variable. To make sure your environment matches the system requirements, simply run the following script from the Command Prompt. 

    cls
cd \
echo %PROGRAMFILES%
echo %PROGRAMDATA%
echo %USERPROFILE%
echo %PATH%
echo %JAVA_HOME%
java -version
javac -version
python -V
git --version
rem if the ANDROID_SDK variable is configured, please delete it and run this script again
echo %ANDROID_SDK%
adb version
cd %PROGRAMFILES%
android list target
cd \
rem --- The following are optional ---
rem ----------------------------------
scons -v
rem ----------------------------------
echo %HTTP_PROXY%
echo %HTTPS_PROXY%
rem ----------------------------------
echo %SOCKS_PROXY%
rem ----------------------------------
rem

    From there, you should be able to see if your JDK is accessible from your PATH.
