HTTPClient.responseText from SOAP call escapes the inner < and > characters

I have an HTTPClient (xhr) that is making a SOAP webservice call. When xhr.onload fires, I get the responseText from it. The responseText has a "return" element that contains the string returned from the SOAP webservice call. This string is an XML string.

The problem is that something in the HTTPClient is converting all of the < and > characters in the return string to &lt; and &gt; respectively. I have to translate all of these back to < and > in order to parse the XML return string. With large return strings, this is very time consuming. I have verified that the SOAP webservice is not doing this unwanted conversion. Other tools that I have to tap the SOAP webservice present the return string correctly, without the unwanted conversion.

Is there a way to prevent the HTTPClient from converting the < and > characters in the first place?

Android 2.2
