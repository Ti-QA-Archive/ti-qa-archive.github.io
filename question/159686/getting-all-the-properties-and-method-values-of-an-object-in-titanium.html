  • Are you looking to enumerate the properties of the Ti.XML.Document object so that you know what methods and properties are available to you? If so, the documentation is likely to be complete and accurate. Our docs team is pretty good these days.

    Or, are you looking to view the structure of your XML data? You're best bet is to use an XML inspection tool like http://apigee.com/ or checking with the provider of the XML API you're accessing.

    If you want to give a go at enumerating the object itself, you could try this function (by our esteemed CEO no less!):

    function getKeys(obj, keys) {
    keys = keys || {};
    Object.keys(obj).forEach(function(k){
       if (!k in keys) {
           var o = obj[k];
           keys[k] = 1;
           if (o instanceof Object) {
               getKeys(o,keys);
           }
       }
    });
    return keys;
}
getKeys({
    key1: "value1",
    key2: {key3: "value3", key1: "value4"}
});

    (From http://jsperf.com/get-keys-in-nested-object)

    And to answer your question included in your code comments, recursion is the technique you're looking for.
