Android ScrollView with large view

I use a scroll view and add a long view to it that contains many smaller views. The idea is to create something similar to tablesections on iOS. This works great if the view I add to the scroll view is less than a certain height (It most likely has to do with the height of the actual phone screen since it works on one phone and not on another, both running Android 4.1.2).

So if the view is too long, it takes up the space o nthe scrollview but it is invisible. Even the click events on its content work, but the view does not display.

here is code similar to what I am using:

scrollView = Ti.UI.createScrollView({
        top: 0
    });

view = Ti.UI.createView({
    borderRadius: 5,
    left: '10dip',
    right: '10dip'
    });

for (var i=0; i < 15; i++) {
    view.add(Ti.UI.createView({
        height: '45dip',
        width: Ti.UI.FILL
    });
}

//view.height = '400dip';
scrollView.add(view);

if I uncomment the line before last, the view shows just fine, but obviously it is cut off, otherwise the view "view" becomes invisible, but "scrollView" scrolls as much as "view" would be if it was not invisible. Tried setting view.visible = true as well.

Using SDK 3.10 on Android 4.1.2.

Anyone seen and got around this?
