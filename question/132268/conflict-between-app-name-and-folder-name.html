Conflict between App name and folder name

The problem is very simple.
My App is named "images" and I have a "resources/images" folder containing the images of my app.
This simple fact cause compilation to fail with Python errors.

I'm using Titanium mobile 1.8.1 and try to compile for iOS.

    • Yeah. If you really want your icon name to be "images" that would be fine, however your app name (physical name of the project) would need to be changed. Change your project to be called "ImagesApp" and then in your tiapp.xml file change the appname node to "images".

      <name>images</name>

      I think that should compile fine.
