Alloy localization in tss file

Hi guys,
I've read that starting from Alloy 1.2.0 you can use the localization method L() inside a TSS file, but this doesn't work for me…
I have this in my TSS file (not working)

"#btnPlay":{
    title: L("btnPlay", "Play")
}

But inside the controller it works fine

$.btnPlay.title = L("btnPlay", "Play");

What I did wrong? Do I need some parameters in tiapp.xml to enable this?

I use the latest Titanium SDK (3.1.2), problem is both on iOS and Android.

    Hi

    It should work as you will see from the answer I gave a few hours ago on the exact same subject;

    • Internationalize MenuItem

    In that answer the example code was;

    '#mnuNew': {
    title: L('new','New');
}

    Can you try performing a full menu > clean on your project - long shot but worth a go. I would also suggest manually deleting the contents of the resources folder in case old code is not removing itself automatically. Please BACK-UP before doing any of this.

    Make sure the TSS and XML have the same filename prefix or the TSS content is inside the app.tss file.

    Please double-check you are using the latest version of everything and you are using the very latest SDK (I know you have said you are) but it is easy to sometimes work on more than one project and think it is something it is not. We are ruling things out rather than calling your competence into account.

    It it still fails please provide the XML mark-up (relevant snippets only) ideally the window it sits in.

    • Problem found.

      <Button id="btnStats">S</Button>

      the "S" placeholder text is what doesn't let the tss works (but it works via controller code). Using

      <Button id="btnStats"></Button>

      without any text in it did the tricks.
