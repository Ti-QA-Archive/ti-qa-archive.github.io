  • OK so i seam to have fixed it. I don't know whether the solution is the proper way or whether it could cause more problems but for now all my modules are working again under 2.0.1GA2. These are the steps:

    • 1 . Download the latest continuous build of 1.8 (I am using 1.8.3v20120418104835)
    • 2 . Go to the SDK folder in your system where the 1.8 build is located
    • 3 . Open folder iPhone ( ~/Library/Application Support/Titanium/mobilesdk/osx/1.8.3.v20120418104835/iphone)
    • 4 . Copy libti_ios_debugger.a, libTiCore.a and libtiverify.a (note you must copy all three)
    • 5 . Open the folder containing the SDK 2.0.1.GA2 and open the iphone folder.
    • 6 . Paste and replace the 3 files from the 1.8.3 iphone folder to your 2.0.1.GA2 iphone folder
    • 7 . Go back to the project you are working on empty the build folder and rebuild it using the 2.0 build

    Like I said i don't know why this works and I don't know the repercussions of doing this but I cant wait for fixes a lot of projects to finish and 2.0 is a lot more stabler then 1.8.

    Any Titanium input to this problem might help with a more permanent solution.
