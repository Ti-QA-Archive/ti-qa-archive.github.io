Layout issue in TableViewRow when hasChild or hasDetail = true

Hi all,

I have a TableView with lots of TableViewRow's. Some of the rows have a property hasChild:true, some not. I have this problem on iOS as well as on Android.

I want to display a "value" on the right and a "title" on the left. The problem is, if a row is checked, the right position is different then a row which is not checked. That means my "value" is sometimes positions further left and sometimes not, depending on the hasChild flag.

Code looks like this:

var noChildRow = Ti.UI.createTableViewRow({hasChild: false});
var leftTitleLabel = Ti.UI.createLabel({text: 'My Title', left: 10, right: 90});
var rightValueLabel = Ti.UI.createLabel({text: 'My Val', width: 70, right: 10});

noChildRow.add(leftTitleLabel);
noChildRow.add(rightValueLabel);

Does anyone have a good solution for this? My only solution is to change the width position of the rightValueLabel depending on the hasChild flag. I don't think this is a good solution, especially on Android where the icon width can vary depending on the device though I'm working with dp units.

var childRow = Ti.UI.createTableViewRow({hasChild: true});
var leftTitleLabel = Ti.UI.createLabel({text: 'My Title', left: 10, right: 90});
var rightValueLabel = Ti.UI.createLabel({text: 'My Val', width: 50, right: 10});

childRow.add(leftTitleLabel);
childRow.add(rightValueLabel);

All titles and values have a different length and they may wrap.

Can anyone help me? Thx!

    Create a container view that is the same size regardless of the row having a child or not. Add your left and right titles to that container view to ensure that everything lines up properly. Then add the container view to the row. This will ensure that regardless of the row having a child, all the text will take up the same amount of space.
