query an empty sqlite database

I have the following model

exports.definition = {
    config : {
        columns : {
            "id": "INTEGER PRIMARY KEY AUTOINCREMENT",
            "body" : "TEXT",
            "image": "BLOB",
            "date" : "INTEGER"
        },
        adapter : {
            type : "sql",
            collection_name : "queue",
            idAttribute: "id",
        }
    }

}

Now when I run the app in the iPhone simulator it gets placed in

./Library/Private Documents/queue.sql

in the apps directory. If I view that with a sqlite editor it says its not a valid sqlite database. Also if I try to query it in my app before any data is saved there it doesn't work also..

I guess my question is how can i create an empty valid sqlite DB that I can query on before any data is added to it. I do it like

queueDB = Ti.Database.open('queue');
row = queueDB.execute('SELECT count(*) as num from queue');
queueCount = row.fieldByName('num');
