HTTP client : e.error undefined

Hello,
I'm trying to build an error message on the 'onerror' property of an HTTP client. Most of the time it works fine but sometimes i have a red screen with the message 'undefined is not an objet while evaluating e.error at authentication.js' . I've tried several fixes but the bug is still there. Here is my piece of code :

xhr.onerror = function(e) {
    errorHandler && errorHandler();
    var msg = "";
    try {
      // Une erreur récurrente survient sur un e.error avec e == null, malgré le test cela plante
      // => utilisation d'un try catch pour tenter d'éviter ce problème en récupérant les erreurs lors de la création de ce message
      msg += Alloy.Globals.tPadParameters.server.serverUrl + url + "\n";
      msg += JSON.stringify(Alloy.Globals.tPadParameters);
      msg += ", " + fctId;
      msg += ", " + errorMsg;
      msg += ", " + this.status;
      if(!!e) msg +=  ", " + e.error;
    }
    catch(err) { 
    }
    tPadLog.log('E', "Erreur : " + type + " " + msg);
  };

As you can see, i've put a try/catch to prevent this bug but it doesn't work well.
I'm running out of ideas, so i'm asking in case of someone has a better idea (or a solution !).

    Making your code something like this:

    if (e.error)
    msg +=  ", " + e.error;
