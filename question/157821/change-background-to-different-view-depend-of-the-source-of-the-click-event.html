    First off, this sounds a lot like you should be using the builtin Ti.UI.TabGroup, rather than reinventing the wheel. Having said that…

    e.source will always return the source UI element that triggered the event. Depending what is generating the click event, you should be able to check the properties of e.source to determine how you should make your changes in the click event, then you won't need a separate event for each permutation.
