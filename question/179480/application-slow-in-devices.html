  • 1) Your first problem is hard to find out without debugging. If done right the performance can be very good with Titanium.

    2) Font-size and other sizes: You can make your life much easier using density-independent pixels as default unit.
    You define this in the tiapp.xml like this:

    <property name="ti.ui.defaultunit" type="string">dp</property>

    That is standard in new alloy projects. Check if you have a different value there like px.
    See also:
    http://docs.appcelerator.com/titanium/3.0/#!/api/Titanium.UI.View
    You use it then like this in TSS:

    ".h1": {
    font: {
        fontSize: 24
    }    
}

    or in JS:

    ...
font: {
    fontSize: 24
}
...

    So you pass a number, not a string and it automatically is bigger on high-res devices.
    The same for sizes of views etc.
