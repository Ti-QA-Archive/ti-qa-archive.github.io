Incoming links to API docs should maybe redirect to latest version?

Several times, I've found that Googling for something gives you a link to an outdated version of the API. Not a big deal if you notice, of course, but it would make more sense to just redirect the user to the latest version (maybe with a notice at the top similar to "The link you followed goes to an outdated version of the documentation. This is the current version; if you really wanted the old version, click here.")

    • Actually, that's exactly why I think it would be a good thing. Here's a real-world example:

      The Titanium.Facebook module for 1.3 has an "isLoggedIn()" method. This method no longer exists in the latest version (the loggedIn property is still there, though).

      Unfortunately Googling for Titanium Facebook stuff turns up links to the 1.3 docs. Even worse, calling isLoggedIn() in the current SDK causes a hard crash. I lost some time because of this, actually, since I hadn't noticed the 1.3 in the browser's title bar (it's very easy to overlook).

      Sure, having the older docs around would be useful under occasional circumstances, but the overwhelming majority of people who are searching for stuff are going to want the current API, or so I see it.
