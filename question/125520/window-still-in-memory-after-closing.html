Window still in memory after closing

Dear all,

In my app, I have a table view with 4 rows, each with a window as child (with url).
When clicking an a row the window opens.
When hitting the back button it closes and takes you back to the main window.

But when I hit the same row again, a few components in the window are still on the screen and the new ones are added on top of them. This is not always the cases, something around 75% of the time.

If I close the app or hit another row that opens another window first, this problem doesn't occur.

Is their a way to completely get the window out of the memory?

Thank you
