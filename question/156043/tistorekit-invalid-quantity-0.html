  • I already fixed it, i was using old example code with the newest version of ti.storekit

    In the newer version the eventlistener changed when pruchasing a product, int he older version the event was not yet ready and that why it always had 0 products because the request wasnt ready…

    So my advice:

    1. Use the newest version of ti.storekit
    2. User the exact code from the newest ti.storekit example

  • I was also running into this issue. Make sure you upgrade to Storekit 2.1.3 and use the latest example code. The difference is in the event listener for purchasing products.

    I had immediate crashes with Storekit and iTunes connection problems as well (which was another problem). I solved this by a) upgrading to 2.1.3 and b) restoring my iPhone settings to its factory settings (not a FULL backup and restore, just the settings). Make sure you test it on the device and properly log out of iTunes before testing.
