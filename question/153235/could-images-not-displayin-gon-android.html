Could images not displayin gon Android

When setting the image property of an ImageView like so:

image = Ti.UI.createImageView({
    image: e.photos[i].urls.thumb_100
});

The images do not appear on Android devices, but display as expected on iOS.

I'm developing in Alloy with SDK 3.1 and getting the images with:

Cloud.Photos.search({
    user_id: "51aca70ea49ecc34ac0865bb"
}, function (e) {
    if (e.success) {
        var data = [];
        for (var i = 0; i < e.photos.length; i++){
            var row, image;

            row = Ti.UI.createTableViewRow({
                id: e.photos[i].id
            });
            image = Ti.UI.createImageView({
                image: e.photos[i].urls.thumb_100
            });
            row.add(image);
            data.push(row);
        }
        $.recentImageTable.setData(data);
    } else {
        alert('Error:\\n' +
            ((e.error && e.message) || JSON.stringify(e)));
    }
});

Is there a setting I need to enable to have the images display on Android, or a different value for the image property?

    I am not sure about your iOS output because When you use ACS photo Search Method tyhen it not returns the image urls as you write in code.

    • http://docs.appcelerator.com/cloud/latest/#!/api/Photos-method-search

    and if the doc is not updated then the problem is You are using thumb_100 instead of thumb only.

    Just try below code : 

    //Change e.photos[i].urls.thumb_100 to e.photos[i].urls.thumb.
 image = Ti.UI.createImageView({
     image: e.photos[i].urls.thumb  //e.photos[i].urls.thumb_100
});

    If above way not work then You can use Other ACS Photo method for that.

    Just try the Photo query and show methods Which may helps you to solve the problem.

    Try like below Way : 

    Cloud.Photos.query({
    page: 1,
    per_page: 20,

        user_id : "51aca70ea49ecc34ac0865bb"
    }

    if (e.success) {

                image: e.photos[i].urls.thumb

            data.push(row);
        }
        $.recentImageTable.setData(data);
    } else {
        alert('Error:\\n' +
            ((e.error && e.message) || JSON.stringify(e)));

});
