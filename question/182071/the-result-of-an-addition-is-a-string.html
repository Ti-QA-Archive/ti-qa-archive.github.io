The result of an addition is a string

Hi, i have a strange problem:

I have this function:

function ActualizaPantalla(){

    var AcumAnual=0;
    AcumAnual=0;
    Ti.API.info('*** Consumos diarios Actualiza 1: ' + AcumAnual+"/"+(AcumAnual+3)+"/");

        for (i=0;i<=6;i++){
            AcumAnual=AcumAnual+ConsDiaW[i];
        }

        Ti.API.info('*** Consumos diarios Actualiza 2: ' + AcumAnual+"/"+(AcumAnual+3)+"/");
        $.LMes.text=String.formatDecimal(ConsDiaW[midia-1]/10,'en-EN','#0.0')+" kWh";

        $.Indicador.animate({
            left : '22%',
            duration:50
        });    


}

In ConsDiaW[] i have several numeric values. The problem is sometimes in AcumAnual don't add the values of ConsDiaW[] like numeric but as string.
For example in ConsDiaW i have the values 130,140,150,160,170,180,190…. Normally the result in AcumAnual is 1190 (130+140+150+160+….), but sometimes the result is a string "130140150160170180190".
Why?? What i'm doing wrong?

    Welcome to the wonderful world of JavaScript! :-)

    JavaScript uses the + as both the addition and concatenation operator, and has a preference for working with strings. Thus, if any operand is a string, the whole operation is treated as a concatenation rather than addition.

    You could do:

    // if all whole-number values
AcumAnual = parseInt(AcumAnual) + parseInt(ConsDiaW[i]);

// or, if the values could be fractional
AcumAnual = parseFloat(AcumAnual) + parseFloat(ConsDiaW[i]);

// though JavaScript's handling of float numbers is problematic
// so, Google currency handling in JS
