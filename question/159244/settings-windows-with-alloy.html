Settings Windows with Alloy

I want to create a simple settings window and store/load the these settings via Ti.App.Properties

My view looks like this:

<Alloy>
    <NavigationWindow id="settingsWindow" platform="ios" modal="true">

        <Window title="Einstellungen" id="win2">

        <RightNavButton>
            <Button onClick="closeWin">Speichern</Button>
        </RightNavButton>      

        <ListView id="listView" defaultItemTemplate="template">



            <Templates>
                <ItemTemplate name="template">
                    <Label bindId="info" id="title"/>
                    <Switch bindId="setting" id="setting" onChange="updateSettings"/>
                </ItemTemplate>

            </Templates>

            <ListSection headerTitle="Push-Benachrichtigungen">
                <ListItem info:left="0" info:text="Benachrichtigung bei News" setting:left="260" setting:value="false" setting:id="pushNews"/>
                <ListItem info:left="0" info:text="Benachrichtigung bei Angeboten" setting:left="260" setting:value="false"  setting:id="pushPromotions"/>
            </ListSection>
        </ListView>    
        </Window>
        </NavigationWindow>
</Alloy>

Is it possible to load the settings from Ti.App.Properties and set according to these values the values of the two switches in the view before opening the window?
I open the settings view via:

    var settings = Alloy.createController('settings');
    var win = settings.getView();
    win.open();

Thank you

iOS 7, CLI version 3.1.2, Titanium SDK version 3.1.3.GA

  • Hi, you can pass an object argument to the settings controller with the values of two boolean properties 

    var settings = Alloy.createController('settings', {
    pushNews: Ti.App.getBool("pushNews"),
    pushPromotions: Ti.App.getBool("pushPromotions")
});

    and receive them in settings.js by

    var args = arguments[0] || {};
var newsSwitchValue = args.pushNews;
var promotionsSwitchValue = args.pushPromotions;

    the use the values to pre-set the switches.

    hth
