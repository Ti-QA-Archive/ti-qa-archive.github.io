Navigation Group duplicate child windows

Hi all, I'm having problems with a navigation group inside a tabgroup and duplicate child windows. Basically when I click on the tab which has the navigation group, then open a child window inside that navigation group by using

var childWindow = new ChildWindow(someArgs); // returns window object
tabGroup.activeTab.open(childWindow,{animated:true});

Now my navigation group has 2 windows, the first (parent) and it's child as opened using the above code. Everything appears fine to this point.

My Nav group at this point looks like:

> NavParentWindow -> childWindowX

The problem happens when I now click on a different tab in the tab group, then click back on the tab with the navigation group. Opening a child window now within the tab group proceeds to open it at the end of navigation stack after the original childWindow. So now I have a navigation stack with 2 identical childWindows (2 deep), when I only want one.

> NavParentWindow -> childWindowX -> childWindowX

My question is, how can I purge the original child window from the nav group so opening a new child window doesn't duplicate itself in the nav stack? I've tried childWindow.close(); to no effect.

  • Hello Mark, upon investigating my code, it turns out, that I had added the event listener twice to the element. Thus, every time, a 'click' event occurred on that element, it triggered opening a child window twice. Thus, the duplicate child window used to get created. I removed one listener and viola! Works as expected. Hope this helps.
