JSON.parse rises an exception if the Json string is not valid. Any way to capture the error?

I noticed that if the json string passed to 

var json = JSON.parse(this.responseText)

is not valid..
it will break the current block of code rising an exception..

What I want in this case is to protect the code from rising the exception and just capture the parsing error with an if or something…

  • The best way to check JSON code is to use JSONlint.com, paste your code in here and it will find any errors. You should consider this an essential part of your coding toolkit - saves lots of time debugging.

    There is one more error it may not pick up that Titanium will (due to how the internals work), unicode or special characters are not accepted. The amount of time I spent debugging a JSON feed to find out the GBP £ sign HAS to be encoding using the unicode escape sequance of \u0000 in-place - annoying.

    The answer provided Kami is correct and will create an exception so your code does not fail but unfortunately it will not track down the issue causing the problem in the first place.
