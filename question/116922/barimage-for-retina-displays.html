barImage for Retina Displays

I am currently working on an iPhone-App and successfully using the @2x naming convention to display images for Retina Displays. This works well for tabs, table views, but surprisingly not for barImages. Even if I link the high-res image directly the simulator, as well as the iPhone will not display the image correctly.

I am currently using Titanium SDK 1.5.1 and iPhone SDK 4.2
Any solutions out there?

  • Have you done a full rebuild yet? I can't really see what's wrong with out knowing your code….

    Make sure "barImage" is set without the @2x in it. It's done automatically for the devices that render it. Example….

    var win1 = Titanium.UI.createWindow({
    titleImage: 'images/logo.png',
    barImage: 'images/top.png',
    barColor: '#000'
});

    Also, you should give 1.6.0 a shot as it's now at RC level. That may clear a bunch of other things up as well.

  • Guys thanks for your support… but I found the problem and it is already working with SDK 1.5.1, no full rebuild necessary.

    I always thought that Titanium did not change the image according to the device, but it did! The image just looked messy and not sharp. The problem is that the barImage needs to have a specific width and height. A 640x88 image will be displayed correctly, while a 640x85 will not.
