First window orientation problems.

Hi, I have a problem and I hope you can help me!

I have my app in PORTRAIT mode compatible. When I open my app, after screen splash, I open a window with intro movie. I set the window in LANDSCAPE mode but it start in portrait mode and at the end of execution my tab group start rotated of 90° in my PORTRAIT status.

app.js 2.1.3GA

    var vidWin = Titanium.UI.createWindow({
        backgroundImage: Ti.App.sfondo,
        orientationModes: [Ti.UI.LANDSCAPE_LEFT, Ti.UI.LANDSCAPE_RIGHT]
    });

    var videoPlayer = Titanium.Media.createVideoPlayer({
        fullscreen: true,
        autoplay: true,
        sourceType: Titanium.Media.VIDEO_SOURCE_TYPE_FILE,
        scalingMode:Titanium.Media.VIDEO_SCALING_ASPECT_FIT
    });

    videoPlayer.url = 'intro.mp4';
    vidWin.add(videoPlayer);
    vidWin.open();

    videoPlayer.addEventListener('complete',function(){
        Ti.App.firstCustomTabGroup.open();
        vidWin.close();
    });
    videoPlayer.addEventListener('done',function(){
        Ti.App.firstCustomTabGroup.open();
        vidWin.close();
    });
    videoPlayer.addEventListener('fullscreen',function(e) {
        if (e.entering === false) {
                videoPlayer.stop();
                Ti.App.firstCustomTabGroup.open();
            vidWin.close();
              }
        });

  • technically the app doesn't know to start in LANDSCAPE until you set it in the middle of your code, and starts rotating only then.

    You should either restrict the available rotations to LANDSCAPE LEFT and LANDSCAPE RIGHT (if you don't need the PORTRAIT mode at all in your app), or maybe have a window on top of the other window that has to be in landscape, that waits until the window beneath finish loading (not sure if you know what I mean).
