  • Accepted Answer

    After spending some time troubleshooting Brad's Windows 7 64bit system, we discovered the following things:

    • the "xcopy is not recognized as an internal or external command" error was caused by the missing windows system paths. We added back "C:\Windows\system32;C:\Windows;C:\Windows\System32\Wbem;" to the start of the system PATH variable to resolve it
    • we found that Sun Java JDK 64bit was installed rather than the required 32bit version. We uninstalled 64bit and installed the 32bit version
    • Android SDK was already installed under C:\Program Files (x86), possibly causing Titanium Studio not to recognize its existence. Certainly Studio would not ordinarily have permissions to manage the Android packages when located here. Hence, we uninstalled it and allowed the Studio dashboard configuration wizard to install it under his home directory, which resolved this

    I have a feeling that the first point was the blocking issue.

    Hope this helps someone.
