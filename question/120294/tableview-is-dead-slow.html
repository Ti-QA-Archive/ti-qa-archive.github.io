  • Gorjan

    For Titanium versions that are affected by this issue, tableview performance will be degraded if a font is specified that does not exist on the system.

    For Android prior to 1.7.X (currently only in the Continuous Builds), only "monospace", "serif" and "sans-serif" are available, so you must stick to these. Please let me know if this works for you.

    That said, Continuous Builds 1.7.X onwards contain the fix and support for custom fonts, as per ticket TIMOB-3418.

    Cheers
