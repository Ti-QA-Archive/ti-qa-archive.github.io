Dashboard View Scrolls Vertically

Since I upgraded to Titanium Mobile SDK 1.7.2 the dashboard view in my application started having some erratic scrolling enabling scrolling up and down when that was originally not allowed. When the user stops scrolling sometimes the view bounces back to a wrong position where not all icons are visible. I looked through the documentation and didn't notice any changes. I was just wondering if anybody has run into a similar problem and if so if you could provide any solutions.

  • Accepted Answer

    And here's is the solution: you must set the height of the dashboard to anything >= 400. This causes the 1st row to be behind the title bar so you must also set the top property. My code looks like this and this doesn't scroll anymore. 

    var dashboard = Titanium.UI.createDashboardView({
    data:data, height:400, top:5
});
win1.add(dashboard);
