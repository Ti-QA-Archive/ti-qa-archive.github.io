iOS icon issues (iPad vs. iPhone icon sizes) when creating archive

When creating an archive for the App Store, my project compiles fine and Xcode opens. When I validate my project, I get an error message stating:

for iPhone/iPod Touch, the icon file must be 57x57 pixels

If I then use a 57x57 icon file and create a new archive, validating gives an error stating:

for iPad, the icon file must be 72x72 pixels

I have icons in both sizes (57x57 and 72x72) in my Resources folder. Is there a setting I need to change so that Titanium and/or Xcode knows that the 57x57 icon should be used for iPhone/iPod, and the 72x72 icon should be used for iPad, so that I can successfully validate the app?

    just change the name of the icons to:
    57x57 = appicon.png
    72x72 = appicon@2x.png
