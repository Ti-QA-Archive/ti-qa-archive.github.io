Issues with the App rolling back and deleting Databases

We have developed an app for our customers that allows them to work out in the yard. In the code there are databases that are created when pulling the information from the server via XML. The problem is that there have been a few occurrences with one of our customers getting booted from the app and losing all of their settings in the database. There's a button that is called "download settings" that dials out to the server and pulls the info. Once you pull that info then it stays in the databases, but when they get booted, it rolls back the app. I want to just blame the fact that they got the tablets from AT&T and there is something on the tablet causing this issue. Has anyone else been having a problem? Or any suggestions? Like I said, they got the tablets from AT&T because they are using the cellular service to transmit data instead of WiFi. I know carriers are notorious for putting a bunch of crap on their devices they sell to customers.

  • Hi

    I have never heard of a situation where an app is rolled back by the OS - with or without a crash.

    Are you sure this is not simply a result of the app not logging that the process to store the local copy of the settings has completed - before the "crash" occurs, so on next launch it tries again - thus resetting things back to original as it then calls out to re-download?

    If this was me I would add a diagnostic log, that tracks the pull request and the success or failure. Have this a file that is APPENDED to and then have a hidden option in the app that trigger that file being presented in a standard email dialog as an attachment and a pre-populated email address.

    Then if it happens again and you are able to speak with the customer - you can have them send you the log.

    Make sure to not re-created the file ever - only append.
