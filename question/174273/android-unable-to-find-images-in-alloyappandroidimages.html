Android unable to find images in Alloy/app/android/images

Android seems to be unable to find images in Alloy/app/android/images directory.

It gives a slew of errors like the following:

"[WARN] : TiDrawableReference:" "Could not open stream to get bitmap"

I'm sure I simply have the directory structure wrong, but I have been unable to find a doc that directly describes what structure to use other then the intro, by which explanation my current app should work.

    For assets, such as images, any references to the Resources folder in the Titanium SDK documentation should be replaced with the app/assets folder to use it for Alloy. For example, the Icons and Splash Screens guide tells you to place files in either the Resources/android or Resources/iphone folder. For Alloy, the files should be placed in either the app/assets/android or app/assets/iphone folder.
