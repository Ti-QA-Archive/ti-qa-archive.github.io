How to call a function just before app crashes

  • Hi

    Short answer - you can't.

    Assuming we are not talking about a simple syntax error.

    If your app has crashed at operating system level - then the OS has managed a crash event - your app is no longer running and has no opportunity to run more code - for whatever reason. If it could run more code it would not have crashed.

    You have to test your app often in many different ways and on as much kit as you can get.

    • You could use a different approach;

      • Log events of your own - for example; opened window, current memory, current storage
      • Run these either at each main app event - for example just before closing a window or opening another, making changes to a database
      • Save these in a file or a DB and marked them as not sent with a date/time
      • Just before the app closes mark the new entries as not required
      • When the app starts up - check the DB if there are any entries that are still marked as new - the app must have crashed
      • API call those over to your server at this point and reset them as processed (same as if app closed correctly)

      This was your next app start-up will send your own app log
