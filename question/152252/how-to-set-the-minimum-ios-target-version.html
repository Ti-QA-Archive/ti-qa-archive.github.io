how to set the minimum IOS target version?

I've released my app to testers and they have let me know that the app can't be installed on ios5 but requires ios6. Is there a way to set the deployment minimum target version lower or does this get set automatically based on certain features I'm using? I can't think of features that would require ios6 or even io5 so I'd love to set this version as low as possible.

PS: I'm using Titanium Alloy, but I don't think that would matter?

    Hi

    You set the minimum iOS target by editing the tiapp.xml file and adding the following if it does not exist or merging if it does.

    <ios>
    <min-ios-ver>4.3</min-ios-ver>
</ios>

    Example of merging;

    <ios>
    <min-ios-ver>4.3</min-ios-ver>
    <plist>
        <dict>
            <key>UIStatusBarStyle</key>
            <string>UIStatusBarStyleBlackOpaque</string>
        </dict>
    </plist>
</ios>

    4.3 is the current lowest iOS version supported by the most recent xcode version.

    • Apple themselves now only support 4.3 and higher - so this means iPhone 3Gs and higher (older phones cannot 4.3).

      Which version you support depends on the needs of the app features. If you are running Titanium only and no custom modules then you would go with 4.3. However if you add extra modules that expose features from say 6.0 - example would be the social sharing sheet (see safari share) then you have to make a decision. Make the app iOS 6.0 or higher or provide two ways to perform those tasks.

      My preference is to provide newer features where they add value to the app experience but add reasonable fall-backs for older OS versions.

      If your app MUST have newer only features then as long as your audience is iOS 6 only then that is OK. I write apps for restricted audiences as well as general audiences, for restricted audiences you can dictate.

      The choice is usually the users, if they have all moved, move with them - if they have not support the widest OS range you can.
