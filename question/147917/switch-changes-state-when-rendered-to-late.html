Switch changes state when rendered (to late)

My iOS example: I placed a Ti.UI.Switch in every table row of a table. If the state is set to "ON", the switch changes it state to "ON" with an animation as soon it gets rendered (scrolls into the visible screen) and unfortunately not before. This slows down the scrolling and looks very silly.

There are several very old posts regarding this problem, but no solution. Did anyone figured out a workaround? Because that is very annoying.
