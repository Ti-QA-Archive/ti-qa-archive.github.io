How to add views at the top of a normal View dynamically?

I'm working with Titanium Studio 3.1.3 and deploying for both iOS and Android.

I need a way to add elements at the top of another View with a vertical layout. With a TableView you only need to alter the data or call any of the method that allows to insert items at specific locations but a normal View doesn't have a similar behaviour.

I tried something like this:

    var childrenViews = $.AppViewWorkspace.children; // get the child views in AppViewWorkspace
    var tmpView = Alloy.createController("Stream_Item", response).getView(); // create new view to add initialized with the properties contained in response
    childrenViews.splice(0,0, tmpView); // insert the new element at the beginning of my collection of child views
    $.AppViewWorkspace.applyProperties({ // update the children property
        children : childrenViews
    });

And while it managed to display a new view at the top of the rest of the initial elements, it showed an empty view with only the backgroundColor I assigned to my Stream_Item views, without showing the rest of the elements contained in it.

What's the workaround to add elements at the top of a normal View? I don't want to remove all the children and add them again with the new element at the top because if there were many elements contained in the view, the rendering process for removing all of them and reinserting them would be a bother.

      There is one variation you can try based on your own code, but again not ideal. If the parent object to the children is a view then you have a method removeAllChildren. Then loop the array to add them back in.
