Large TableView old device performance

I'm having trouble finding a solution to my problem and I'm sure there much be a good way to achieve it.

I found that on older ios devices things such as button clicks that trigger a database update became unacceptably slow when the number of rows got too big. I've looked at using infinite scroll but that doesn't really resolve the issue, as it keeps adding extra rows which eventually just results in the same situation.

The ideal solution would be for the tableview to always have a limited number rows, and getting to the end or beginning updates its accordingly, but I'm finding it hard to make this happen smoothly and feel natural.

Or maybe I'm missing something, and there is a better way to do this, but I haven't come across it yet.

    • ListView, behind the scenes, does what you describe… it only draws a limited number of rows, and re-uses those rows as you scroll. It also supports infinite scroll (via markers).

      It won't solve poor coding or design choices, but it sounds like you probably would be able to tell the difference.

      — commented November 6th 2014 by Shannon Hicks
