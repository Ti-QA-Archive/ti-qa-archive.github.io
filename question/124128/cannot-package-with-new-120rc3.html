    Ray, Typically the packaging servers have not had release candidates on the, AFAIK, and you have needed to package locally to accomplish that. I'm not sure if that will change or not, but I wouldn't expect so…

    • There is a problem with this, when there is a fatal/crashing bug the turn around for online packaging is weeks/months.

      My suggestion would be to release two types of versions.

      1. Crash bug fixes that are included on the packaging servers
      2. Feature additions that are localised versions for testing.
