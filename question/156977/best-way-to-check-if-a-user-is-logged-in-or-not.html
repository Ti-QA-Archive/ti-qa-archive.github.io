Best way to check if a user is logged in or not?

I'm trying to check and see if a Ti.Cloud user is logged in or not. I want them to be able to log in once, then stay logged in when the app is in the background. But I need a way to check for login before some of the app's functions are able to run.

I see that the "hasStoredSession()" method has been deprecated. Is there some data that is stored locally in the app from when the user logged in so I don't need to make a network call to see if they are logged in? Like a client-side cookie in a web browser, etc?

I've been looking through the docs at http://docs.appcelerator.com/titanium/3.0/#!/api/Titanium.Cloud.Users to no avail. I guess I could use the "showMe()" method, but that doesn't seem like a best practice.

Thanks.

    • @kusuma
      You can store nearly any custom object in Ti.App.Properties. For example, if you want to keep track of logged in state do this where your login code resides:

      Ti.App.Properties.setBool('app:isLoggedIn', true);

      then to retrieve that value, simply get it wherever you need to use it.

      var loggedIn = Ti.App.Properties.getBool('app:isLoggedIn');

      hope that helps you. You can name the property whatever you like. I use the convention app:propertyName to remind me it's an app-level variable.
