Can't purge device of dummy data

While my app was in development I put some dummy data in the database and used it for testing. Now that I am ready to submit it to Apple I got rid of all the data in the original database, but when I push the app to my iPhone, that dummy data is still there.

I already did these things:

1) Went to 'MyUser > Library > Application Support > iPhone Simulator > 5.1' and deleted everything

2) In Titanium Studio: Project > Clean

3) Deleted the app from my iPhone in the usual way

4) Went to 'ProjectHomeDirectory > MyApp' where I renamed the 'Build' directory (sister to 'Resources') to build_old to force the creation of a new 'build' directory

None of those did the trick. When I fire up the app on my iPhone, the dummy data is still there. Where else could this data still be hiding or cached?

Thanks!

  • Accepted Answer

    I had a similar issue where an updated version of a database wouldn't be available on the device (the table structure was completely different to the DB in Resources). I tried various things but what finally worked was deleting the appropriate folder under ~/Library/Developer/Xcode/DerivedData (there should be one for each app you have built in Xcode). For some reason Xcode was caching the DB (but not any other assets AFAICT).
