Form validation using CommonJS

Hi

I'm a noob building an Android app on Titanium 3.0.1 with Windows 7 and I'm building it using commonjs. On my login screen I want to validate a form and I want to put the validation function into a seperate file, like so:

// Account.js
exports.validateLogin = function() {
    if ( loginEmail.value !== '' && loginPassword.value !== '' ) {
        // TO DO: AJAX request to login API
        // If successful, log in
    } else {
        alert('form empty');
    }
}

I then require Account.js in my login window file, but of course the loginEmail and loginPassword textfields cannot be accessed from this module. I'm wondering if it's possible to do this, or more to the point if this is the "correct" way to do something like this in Titanium? I know I could just keep all the code together in the login window file but I would like to seperate the view code from the underlying function code. Any suggestions?

Thanks
James

    exports.validateLogin = function(loginEmail, loginPassword) {
    if ( loginEmail.value !== '' && loginPassword.value !== '' ) {
        // TO DO: AJAX request to login API
    return true;
    } else {
    return false;
    }
}

....... then

var req = require ('path to your module');

var result = req.validateLogin(id,password);

    id and password an be replaced with the textbox controls for you id and password fields
