    Hi

    I would suggest you do the following;

    • Start xcode
    • Menu; Window > Organiser
    • Select provisioning profiles
    • Delete ALL entries for your app id
    • From developer.apple.com, download the latest profile(s)
    • Double click each you have downloaded
    • Confirm they appear in the Organiser provisioning profiles list

    Then in Studio

    • Menu; Project > Clean
    • Select your project
    • Clean
    • Reboot the computer - (this is most likely not required- but at this point worth a go)

    After re-boot

    • Compile for simulator - does this work?
    • Compile for Device - does this work?

    Report back.

    • The guy who created the jslint.com service is VERY VERY smart and this service is designed to get everything perfect. You do not have to follow everything so explicitly though.

      To turn off this as an error, tick this option many var statements per function.

      The reason it is being flagged is more of a safety notice. As JavaScript does not have as strong a variable scope as other languages - there is a convention that you place all the variables you use at the top of the function and then use them below. The logic is that you never use an undeclared variable and always have them in the same place so that you can tell what, when and where.

      This is VERY hard to stick to, but moving to it is a good aim.
