is it possible use data bindings in ??a Widget?

Hello

I have a window that uses a data model for building. I have no problems in using a model and its attributes to fill the content, but when i want to use the data binding a custom widget, the attributes are not considered

Here is the windows when sending hardcode text (spot.xml)

<Alloy>
    <Model src="spot" instance="true" id="spotObject"/>
    <Window class="container" model="$.spotObject">
        <Widget id="handlerWindows" src="windowHeader" title="this is a title"/>
        <Label class="title-header" text="{$.spotObject.title}"/>
    </Window>
</Alloy>

My widwet controller receives the data seamlessly (widget.js)

var args = arguments[0] || {}
$.titleLabel.text = args.title;
Ti.API.info(args.title); //print this is a title

But, when sending text via data binding (spot.xml)

<Alloy>
    <Model src="spot" instance="true" id="spotObject"/>
    <Window class="container" model="$.spotObject">
        <Widget id="handlerWindows" src="windowHeader" title={$.spotObject.title}/>
        <Label class="title-header" text="{$.spotObject.title}"/>
    </Window>
</Alloy>

I get the argument as undefined (widget.js)

var args = arguments[0] || {}
$.titleLabel.text = args.title;
Ti.API.info(args.title); //print undefined

Obviously when using the model data in any other element of the titanium markup, everything works perfectly.

finally, my question: is it possible in a titanium view use data bindings in ??a Widget?
