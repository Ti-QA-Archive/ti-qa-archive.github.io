Titanium asking for Android Platform 5.0, can't find any of the SDK

Fresh install of Studio, installed the Android SDK, but the Dashboard is saying that "An Android SDK is missing. Titanium requires Android platform 5.0."

Running Linux Minut 16.

Looks like other users have had similar issues but there is not yet a clear solution.

Steps I took:

Titanium was just installed and successfully updated to the latest 3.2.0 and CLI 3.2.0.

After clicking "Install or Update Android SDK" the "Platform Configuration" dialog appears (this looks new - it is NOT the AVD/SDK manager). Android is already selected by default, under the settings Android 4.1.2 is selected by default.

Android 5.0 is (obviously) not listed as one of the available platforms.

After clicking Configure with the default 4.1.2 selected, the progress bar runs for about a minute and then it comes to a screen with two green checks - Latest Version: Unknown, and Installed Location: /home/john/android-sdk-home

Closing this window and refreshing the Dashboard leaves the Android SDK still unavailable (red X). The Dashboard instructions need to be changed to mesh with the new Android SDK install dialog, but that's another issue..

Under Window-Preferences-Platforms-Android I get the message:
Problems detected with the SDK settings - [Android SDK Home] No Android SDKs were found under the specified SDK location

Clicking Install SDKs under this dialog brings up a similar dialog to the one under the Dashboard. Selecting Android SDK 4.1.2 proceeds with similar non-results.

Clicking Manage AVDs brings up the original AVD manager. I created several AVDs.

From here I switched over to the original SDK installer, and installed everything under Tools, the entire 4.4.2, 4.1.2, and 2.3.3 APIs.

I even checked the "Obsolete" box and I found API 5, Android 2.0.0 - thinking that this is possibly what might have been meant in the Dashboard where it originally told me I that needed Android Platform 5.0.

All of these APIs in there entirety have been installed by way of the AVD/SDK manager, and all show as checked on that screen. However Studio still shows a red "X" beside Android SDK in the dashboard, with the same message to install Android Platform 5.0.

Another poster recommended changing permissions of /opt, however the permissions of /opt were already at 755.

I had originally installed Titanium in /opt, but removed it and re-installed it under my home directory (/home/john/TitaniumStudio). The problem still persists.

Please help.

  • Android 5 doesn't exist yet, AFAIK. The latest release is KitKat 4.4. The minimum required install for Studio 3.1.3 is Android 2.3.3 (API10). Check if that minimum version is installed. If not, try installing that and see if the error goes away. I haven't installed 3.2 yet b/c I'm too deep into a project built on 3.1.x to introduce any unknowns. But if that doesn't help, I'd consider logging a bug. Hopefully someone can add something more constructive than this. Thought I'd throw it out there, though.

  • Found the answer.

    The problem was related to #TC-3399 - just follow the instructions in that post and it recognizes the installed SDKs and no longer asks for non-existent ones.
