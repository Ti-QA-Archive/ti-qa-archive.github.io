  • I had a similar problem but my network does not allow ICMP, so changing the rules to allow outbound pings was not an option. Here is a method to bypass the pings to yahoo.com:

    1. Browse to the hosts file, on windows it is located at:
      C:\WINDOWS\system32\drivers\etc\hosts

    2. Double click the file (or right click and click open)

    3. Choose a text editor such as Notepad or Wordpad

    4. Add the following line to the hosts file:
      127.0.0.1 yahoo.com

      Note: This line must appear AFTER the line:
      127.0.0.1 localhost

    This will redirect pings to yahoo.com to your localhost. Unfortunately, this means that you cannot access yahoo.com (although www.yahoo.com will still work). If you need to access yahoo.com, you will have to remove this line again.

  • I had a vexing time with this problem working behind a firewall.

    The app is spamming ICMP messages that are blocked by firewall and cause the firewall to block all my outbound traffic.

    If you are on Windows platform, this is a good guide to installing a IP Filter policy to drop all outgoing ICMP messages from your machine.

    http://www.petri.co.il/block_ping_traffic_with_ipsec.htm

    You can turn on and off the filter when you need to do any network troubleshooting.
