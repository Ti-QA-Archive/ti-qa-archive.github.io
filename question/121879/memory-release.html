Been around a month since I am struggling to get answers on how the memory release works in appcelerator / android. So here are simple questions:

  • Hi Emrah,

    I struggled with the application further, switched all my windows to views. Now my app is a single heavy weight window which calls views as and when neccessary. This reduced the memory footprint of the application significantly. While opening and closing windows, I see a constant rise of memory, the use of views has ensured the application stops taking in more memory after a certain amount of usage. This has been a big relief!!

    Just leaving you a message so you may want to consider views in the future…. its totally WORTH it!
