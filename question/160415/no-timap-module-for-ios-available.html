No ti.map module for iOS available?

I am trying the sample map from http://docs.appcelerator.com/titanium/latest/#!/api/Modules.Map
for iOS and Titanium 3.2.0/xCode 5.0.2.
In the manual it says I should put the following code into tiapp.xml, which does not work because there is obviously no module for iOS present on my Mac (only for Android)!?

<modules>
<!– Add this line to your modules section –>
<module platform="iphone">ti.map</module>
</modules>

The compiling fails with the error "Ld build/Debug-iphonesimulator/Maps.app/Maps normal i386" which is no wonder because the module does not seem to be anywhere on the machine - or do I miss something?

At the market I find "Google Maps for iOS" (https://marketplace.appcelerator.com/apps/5076) for $30,-. Is this a proper solution to get maps onto my iPhone? Any experience anybody?

Ronald

    There is a zip file there.
    When you unzip it should crate a folder "modules" containing the iphone/ti.map/etc.
    copy this "modules" folder in your project at the same level with tiapp.xml and Resources, so basically in the root of the project.

    Clean the project and build again.
    Don't forget to add it in tiapp.xml

            <module platform="iphone" version="2.0.0">ti.map</module>

    inside the <modules> tag
