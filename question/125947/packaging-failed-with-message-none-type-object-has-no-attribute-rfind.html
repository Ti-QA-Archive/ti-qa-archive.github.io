  • I had this problem today. Build: 1.0.6.201110251616. Maybe someone else has the same problem I did.

    My license was originally written on my Mac for a different product. I copied the text and pasted it into my Titanium app on Windows, adjusted the wording, and tried to package but got the no attribute 'rfind' error.

    Turns out the license included the word "user's" in it. Well, that artsy Mac apostrophe is not a valid character on Windows, so the license file had a binary character in it. Changing the artsy apostrophe to a Windows single quote solved the problem.
