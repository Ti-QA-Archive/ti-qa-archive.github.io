Alloy.Globals and NavigationWindow as index view.

Hi,

I'm having trouble getting Alloy.Globals to work when I setup a NavigationWindow as my primary view in index.xml.

In the index.js controller, I assign two values to Alloy.Globals; a reference to $.index, and a string.

Based on the Global Namespace documentation, I should be able to access those values on the first window (home), but I get 'undefined' for both.

Here's the code:

index.xml:

<Alloy>
    <NavigationWindow platform="ios">
        <Require id="home" type="view" src="home">
    </NavigationWindow>    
</Alloy>

index.js

$.index.open();
Alloy.Globals.parent = $.index;
Alloy.Globals.test = "test 2";

home.xml:

<Alloy>
    <Window backgroundColor="#fff">
        <Label>Test 2</Label>
    </Window>
</Alloy>

home.js:

var parent = Alloy.Globals.parent;
var test = Alloy.Globals.test;
Ti.API.info('>>> '+ (typeof parent));    // returns undefined
Ti.API.info('>>> '+ test);            // returns undefined

Now, if instead of using a NavigationWindow I use a standard Window, and open a child window from it, it works as documented, and I can access both values saved in Alloy.Globals from the child window.
