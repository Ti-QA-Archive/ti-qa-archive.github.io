Alloy.jmk change tiapp.xml before build

I've tried the pre:load and pre:compile events in my Alloy.jmk file, but neither of them change the file before the application is built. Is this just something that I cannot do?

My Goal is to change the app name, icon and ID based on the deployment type so my QA and Production builds are different.

  • As far as I know, you should be able to do this. Check the sample mapping app (importable via Studio, jmk file here). It modifies the tiapp.xml to add mapping keys. The same principle should apply to your case.

    Alloy 1.5 (which at the time of this posting won't be out for a while) will let you theme the i18n and platform directories. I'm not sure we'll add theming of the tiapp.xml since you should be able to do that with the jmk file.

    If you're still having troubles, please log a Jira ticket that includes a sample project and your jmk file.
