How to include relative css/js in Android Webview (dynamically created HTML)

Using WebView.setHtml('<html> /// </html>'), how can you include css/js files such that they work on Android?

The various solutions work fine on iOS simulator, some work on Android simulator, but nothing I've tried works on a deployed app on Android.

To explain in more details:

File structure is:

  • Resources

  • Resources/web (where the Titanium JS files sit that generates the HTML)

  • Resources/web/images (images to be included)

  • Resources/web/css (css to be included)

  • Resources/web/js (js files to be included)

Code is (paraphrased):

var myWebView;
var DynamicHTML;

myWebView = Ti.UI.createWebView({ url: '/web/loading.html' });
self.add(myWebView);    //construct UI

DynamicHTML = '<!doctype html>' +
'<html class="no-js" lang="en">' +
'<head>' +
'  <meta http-equiv="Content-Type" content="text/html; charset=utf-8" />' +
'  <title>Home</title>' +
'  <link rel="stylesheet" href="/web/css/screen.css">' +
'  <script src="/web/js/zepto.min.js"></script>' +
'</head><body>content</body></html>' +

myWebView(setHTML(DynamicHTML));

The css an js files are not loaded.

I've tried:

  1. web/

  2. /web/

  3. app://com.XXX.XXX/web/

  4. [nothing - i.e. relative]

  5. <base href="/web/" >

  6. <base href="web/" >

  7. <base href="app://com.XXX.XXX/web/" >

Has anyone created a solution that works (when deployed, not just in a simulator?) Or anything I've not tried?

Thanks.

  • I did find a solution that woked reliably for me. I created the default as src="/web/FILENAME" which worked perfectly on iPhone. Then I ran the following replace that made it work on Android.

    In my example, the "web" folder was situated directly under "Resources", and alos included the js files that built the page. Nasty structure as I'm mixing Titanium JS files with WebView JS files, but it worked.

    var html = pageout();

// If android, replace the external links in headers with the actual files.
if (Ti.Platform.osname == 'android') {
    html = html.replace(/"\/web\//g, '"web/');
}
myWebView.setHtml(html);
