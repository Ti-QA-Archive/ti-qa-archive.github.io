[WARN] : New layout set while view [object TiUIImageView] animating: Will relayout after animation.

I am not sure if I should be concerned, but in my app, I would like to have multiple animations going at the same time. In the current case, I have two imageView's that I am animating. Once slides down from the top of the screen, the other slides up at the same time. I would really like to have these happening simultaneously. The problem and concern I have is that I get a warning on the console that states:

[WARN] : New layout set while view [object TiUIImageView] animating: Will relayout after animation.

The warning happens every time that the application is triggered. Which could be several times through the use of the app. I have other similar animations with an imageView that slides to the right while another comes into view at the same time.

Should I be concerned with this warning? Could it cause memory problems? Is there a way to run multiple animations at the same time that is more efficient?

My code below for triggering the animation on an imagaView and a button I have in the app.

var SlideDown_App_Title = Ti.UI.createAnimation({
  top: 100,
  duration: 500
});

var SlideUp_btn_Create_Account = Ti.UI.createAnimation({
  top: 370,
  duration: 500
});

img_App_Title.animate(SlideDown_App_Title); 

btn_Create_Account.animate(SlideUp_btn_Create_Account);

  • Where did you put this snippet of code? If it's in say index.js the animation will start before there's time for it to finish when Titanium to execute the subsequent code that creates and lays out views, because JavaScript is single-threaded. The warning itself doesn't mean it's using more resources, just that you're trying to do something that can't be done. That's all.
