iPhone Emulator Shows Content On Top Menu Bar

  1. Application type: mobile
  2. Titanium SDK: 3.2.2.GA
  3. Platform & version: iOS
  4. Device: iOS simulator
  5. Host Operating System: OS7
  6. Titanium Studio: Titanium Studio, build: 3.2.1.201402041146

This is my first application attempt for the iPhone and Android platforms. When I run the project in the iPhone emulator the content covers the top menu bar. I was under the impression the top of the screen starts under the menu bar for iPhone that shows the date or time. This is the same problem for all content I add to the window. I could use top in the TSS but I do not know the command to auto translate the numbers per screen used. Any advice would be greatly appreciated.

index.xml

<VideoPlayer id="videoPlayer" ns="Ti.Media" url="video/Silly_Walks.mp4" autoplay="false"></VideoPlayer>

index.tss

"#videoPlayer": {
    top: 2,
    width: 300,
    height: 230,
    backgroundColor: 'black'
}

    In iOS7 statusbar is transparent. refer this this documentation iOS7MigrationGuide-StatusBar.

    and try to set :

    
//  statusbar height is 20 

"#videoPlayer": {
    top: 22,   // set only for iOS7
    width: 300,
    height: 230,
    backgroundColor: 'black'
}

  • Hi Dwayne,

    In IO& the default starting point of a window is from the status bar only, i.e the top of the window overlaps with the status bar content which shows the time and all. So if you are using a normal window which is not heavy weight , in that case you have to give your UI some units from top. Else you can use navigation window.

  • This seems to have worked!

    ".container": {
    backgroundColor:"white",
    statusBarStyle:Titanium.UI.iPhone.StatusBar.TRANSLUCENT_BLACK,    
    top: 20
},
