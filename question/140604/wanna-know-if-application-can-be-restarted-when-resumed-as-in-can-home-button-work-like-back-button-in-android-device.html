    • When I read android:clearTaskOnLaunch I noted that the first activity (which could be three tabs each with a window and a table full of data) would remain persistent.

      Anuj would still need to deal with managing these items - which was not wanted - it was all meant to reset.

      The android:clearTaskOnLaunch does not actually restart anything it simply returns the app to the first thing loaded 'activity P' in their example - basically anything until the back button would exit the app.
