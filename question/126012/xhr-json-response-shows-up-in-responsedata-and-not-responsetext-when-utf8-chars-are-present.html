xhr json response shows up in responseData and not responseText when utf8 chars are present

I have a simple web service that returns a json response. It was working as expected returning the json in the responseText property and I could easily parse it.
Once a two byte utf-8 character is present in the json it seems that the json text is then stored in the responseData property.

Does this sound correct?
How can I best convert / extract the json text from responseData to a string object so that I can use JSON.parse on it?
Using

var jsonText = clientConnection.responseData.text

returns null even though it shows that responseData contains correct response length.
