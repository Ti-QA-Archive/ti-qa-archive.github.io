Can Appcelerator inform us better about how to handle new Xcode releases please?

Yesterday Apple announced the new Xcode 4.3.1 and told developers to start using it now. So like a good boy I downloaded it, installed it, allowed it to uninstall my Xcode 4.2 … and then found out (or rather was reminded) that Titanium does not work with the new Xcode. So now I have to go back and re-download 4.2 and install it and get everything working again.

It's a bit painful, and as I am in the process of final steps of packaging my app for the App Store there were a few brief moments of panic. "Not now!" :)

Anyway. I guess I should check on these things first, and possibly know better, but it is all a learning curve and I just wondered if Titanium could help us out next time by maybe flicking out an email reminder to developers to NOT try and use the latest Xcode with Titanium - and that we need to hold onto Xcode version X.X for the moment.

That would be really helpful :)

Thanks for listening.

  • I've been playing around with this.

    If you're really needing to use this now:

    XCode is now installed in the Application's folder (read more here: https://developer.apple.com/library/ios/#documentation/DeveloperTools/Conceptual/WhatsNewXcode/Articles/xcode_4_3.html). So the dev path has changed. You can modify that via &#x2F;usr&#x2F;bin&#x2F;xcode-select -switch &#x2F;Applications&#x2F;Xcode.app

    If you go to XCode Preferences -> Downloads, you can select to download the older simulators. When you do this you can still build with Ti Studio. It still crash the simulator instance Studio tries to run but not before building and updating the app the older simulator looks at (iOS 5 was what I was using previous to the install, for instance). You can then run the sim manually and use it. When you run it manually, go to the menu at the top -> Hardware -> Select version of iOS. Boom, you'll see the app(s) you've used previously.

    Another way is to go a head and build in Studio and then let the sim crash like it currently does. Go to the build folder and open the xcode project. Select what you want to run (ipad, iphone ,etc) and click run. It will install on the sim and run it. You can select 5.1 and run and it will work too.

    Not ideal, but might get you through a touch spot until it's fixed.
