Android Emulator and custom host routing

I have a definition in my host file that forces internal routing of a URL to my localhost. I use this mainly because of redirect needs when using my web app to log into Facebook

The issue I am having is when running the emulator locally, I want to use this URL to debug locally and see what messages are being sent to my server. Instead, the emulator is sending the request to my EC2 instance.

Is there a config file somewhere that can be modified to get my emulator to route requests to the URL to my localhost?

    While not a titanium specific question, but more an android emulator question, you can change the network settings on the emulator just as you would a device.

    I'm surprised that its not already doing name look up based upon the hosts localhost.

    On OSX I have used the Charles Proxy app for a while now for debugging web and mobile network calls. (was a great session on debugging at the TiConf.us in Baltimore this summer)

    I can simply point my iOS simulator/devices and android emulators/devices to the proxy app for debugging.
