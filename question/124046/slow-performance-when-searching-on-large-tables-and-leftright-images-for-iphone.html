Slow performance when searching on large tables and left/right images for iPhone

Hi

I am having a problem when searching, using searchBar, on large table rows (rows greater than 100) and each row having a left and right image. For example, when you start typing letter "h" in a searchBar field, there is about a 5 second freeze on the phone. As soon as icons are removed from the tableRows, searching is much faster. The filtered results of typing "h" appear almost immediately. I am using mobile version 1.6.2. I have already tried setting className on the row, but this did not help. Has anyone else experienced this problem on the iPhone? Is there a suggested work around?

  • the problem usually is the caused by the complexity of the rows you are rendering. My first guess is that the images you are rendering are probably being scaled by the CPU which is causing the delay.

    Try removing the images from the row and see if that addresses the problem. If it does, create thumbnails of the images on the server and use them when rendering the table rows.

    Also always remember to use a classname on the rows, it increases the performance.
