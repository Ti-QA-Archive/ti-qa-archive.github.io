backgroundGradient causes "event listener" error with SDK 1.8.1

Just upgraded to the 1.8.1 SDK, and I am getting a weird Console error:

[ERROR] Adding an event listener to a proxy that isn't already in the context

I have been able to narrow it down to adding the backgroundGradient property to a view. Here is the code I am using:

backgroundGradient: { 
    type: 'linear', 
    colors: [{ color: '#cccccc', position: 0.0 }, { color: '#999999', position: 1.0 }] ,
    startPoint: { x: 0, y: 0 },
    endPoint: { x: 0, y: 23 },
    backFillStart: false
},

If I remove that snippet I no longer see the error. What is weird is the app still runs fine, and the gradient shows up correctly, I just get these log messages.

And if it helps, this is occurring on the headerView I am setting on a tableView section.

Thanks.

    Just ran into this myself with 1.8.2.

    Setting the backgroundGradient property after construction of the View doesn't result in an error being logged.

    Given that the documentation for this property shows it being used in the constructor one would hope that this will be fixed.
