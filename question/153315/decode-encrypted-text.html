  • Hi

    It is not possible to simply decode something encrypted using this method as the data returned does not actually contain the original value.

    When you MD5 something you take the original string run this function and it creates a "signature" of the original value. The signature is a mathematical result of an equation that if repeated again on the same original value will return the same result.

    This is how you test is something encrypted is the correct value as you repeat the process to compare the two, if they match it is correct, if not they are different.

    • VERY simple example;

      Apple > MD5 = abcdef123

      Someone gives you abcdef123 and you now test it against something else, for example a cache of the MD5 returned value in a DB.

      select * from users where hash = 'abcdef123';
// the result of this statement determines if the result is correct.
