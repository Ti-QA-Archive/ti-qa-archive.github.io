Ti SDK 3.0 ImageView now resizes images to fit

Is there a solution to the new behaviour of imageviews in Ti3.0? I used to use imageviews with the image property over views with a backgroundimage because the imageview would keep the image the size it actually is, without resizing to fit the width & height of the imageview itself.

Unfortunately I've just upgraded to 3.0 and now all imageviews scale the images to fit the imageview width and height.

Does anyone know a way round this or do I have to specify a new imageview with absolute width & height values instead of fluid?
