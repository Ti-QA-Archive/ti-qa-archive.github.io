  • Ti.App.myvariablename = myvalue;

    this will set a 'session variable' , i.e. it will be available while the application runs. If you restart the application this scope will be lost unless recreated.

    Ti.App.Properties.setString(&#39;firstName&#39;, &#39;David&#39;);

    this is like an 'application variable', i.e. the variable is alive even if you start the application. To get the value back you do it with the .getString() method.

    Now, the point of this post is to locate the best practice for storing your variables. Both Ti.App and Ti.App.Properties are good and easy to use but if you have plenty of variables you may start thinking of a different approach as you may experience performance and memory issues. Try the commonjs approach as described above by Trevor.
