    Hi

    Firstly - you are in the right place and other than providing the Ti SDK version you are using you nailed the requirements. I am guessing no code snippet as you are seeking ideas.

    You mention "table" - does "table" mean "table structure" or the view type of "tableView"?

    The reason I ask is that Android does not work well with textFields inside tableViewRows and as such should not be used.

    Given the limited space available in many devices - have you considered the hintText property?

    Here is an example;

    Ti.UI.backgroundColor = 'white';

var win = Ti.UI.createWindow({
    backgroundCOlor: '#fff',
    height: Ti.UI.FILL,
    width: Ti.UI.FILL
});

var textField = Ti.UI.createTextField({
  borderStyle: Ti.UI.INPUT_BORDERSTYLE_ROUNDED,
  color: '#336699',
  height: 60,
  hintText: 'hinted',
  left: 10,
  top: 10,
  width: Ti.UI.FILL
});
win.add(textField);

win.open();

    This will let you have the full width for the input and the caption is then built into the textField.

    Let me know what you think.

    • I have put some code together to give you a starting point, this handles the positioning in a field/value way with each question appearing on a single line. You can adjust the gutter position between the field and value using an optional gutter property.

      The full code is here, I suggest you create a blank new default project and then replace the code in app.js with the code below;

      Ti.UI.backgroundColor = 'white';

var win = Ti.UI.createWindow({
    backgroundColor: '#fff',
    height: Ti.UI.FILL,
    width: Ti.UI.FILL
});

var form = Ti.UI.createView({
    height: Ti.UI.SIZE,
    layout: 'vertical',
    top: 0,
    width: Ti.UI.FILL
});
win.add(form);

function addQuestion(obj) {
    var intWidth = Ti.Platform.displayCaps.platformWidth;
    var dblFactor =  (Ti.Platform.displayCaps.logicalDensityFactor || 1);
    var intDeviceSize = intWidth / dblFactor;
    var intPad = 12;
    var intViewSize = (intDeviceSize - (intPad * 2));
    Ti.API.info('intWidth', intWidth);
    Ti.API.info('dblFactor', dblFactor);
    var view = Ti.UI.createView({
        bottom: '4dp',
        color: '#000',
        height: Ti.UI.SIZE,
        left: intPad + 'dp',
        top: '4dp',
        width: intViewSize + 'dp'
    });
    var lbl = Ti.UI.createLabel({
        font: {
            fontSize: '14dp'
        },
        height: Ti.UI.SIZE,
        left: 0,
        text: obj.title,
        width: (obj.gutter || 100) + 'dp'
    });
    view.add(lbl);
    var txt = Ti.UI.createTextField({
        borderStyle: Ti.UI.INPUT_BORDERSTYLE_ROUNDED,
        color: '#336699',
        font: {
            fontSize: '14dp'
        },
        height: '40dp',
        idx: obj.idx || 0,
        right: 0,
        value: obj.value || '',
        width: (intViewSize - (obj.gutter || 100)) + 'dp'
    });
    view.add(txt);
    return view;
}
var data = [
    { title: 'First Name', value: 'Malcolm', gutter: 120 },
    { title: 'Last Name', value: 'Hollingworth' }
];
// gutter is optional, the default value is 100

var intQuestion = 0, intQuestions = data.length, question;
for (intQuestion = 0; intQuestion < intQuestions; intQuestion = intQuestion + 1) {
    data[intQuestion].idx = intQuestion;
    data[intQuestion].value = (data[intQuestion].value || '');
    question = addQuestion(data[intQuestion]);
    form.add(question);
}
form.addEventListener('change', function (e) {
    data[e.source.idx].value = e.source.getValue();
    Ti.API.info('change', JSON.stringify(e));
});

var btn = Ti.UI.createButton({
    bottom: '20dp',
    title: 'Test',
    width: '100dp'
});
win.add(btn);
btn.addEventListener('click', function (e) {
    alert(JSON.stringify(data));
});

win.open();

      You will see that you can set up a question array and report back on the results - including the value the user entered.

      This has been tested on iOS and Android so you should be good on both.

      As I say a starting point you will need to put in some effort of your own to fit this to your own app needs.

  • You can't think of it as 2 columns as in the table you posted. each row has to have 2 separate labels. The first label should take up a specific amount of space with a static size and the second label should take up the remaining space.

    The best way to accomplish this is to create a wrapper view. Add a label to that view that takes up, lets say, 25% of the space. Then add a second label that takes up the remaining 75%.
