  • Custom fonts in a webview would be done the same way you'd do them in a web page. After all, a webview is simply an embedded browser control loading HTML content. If you want more specific help, please post an example of what you've tried so that folks here can look at your code to determine what you're doing wrong.

    • You can still follow the same format. The src is relative to the project's home. 

      // I added heights to my previous code to prevent overlapping
// then added this
<WebView id="inline" height="100"/>

// in the controller
$.inline.html = "<style>@font-face { font-family: Firecat; src: '/fonts/FirecatMedium.ttf' } p { font-family: Firecat; text-align: center;} </style></head><body><p>Text added via html attribute</p>";

      New webview text shows up in the same custom font.
