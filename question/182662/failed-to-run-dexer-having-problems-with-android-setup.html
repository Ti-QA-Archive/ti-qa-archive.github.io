  • I had the same problem. When running dexer I saw the following:

    /System/Library/Java/JavaVirtualMachines/1.6.0.jdk/Contents/Home/bin/java
    "-Xmx1024M"
    "-XX:-UseGCOverheadLimit"
    "-Djava.ext.dirs=/Users/brownemt/Android_SDK/android-sdk-macosx/platform-tools"
    "-jar"
    ""
    "--dex"
    ...

    Comparing this to a good build using 3.5.0.GA it seemed that those empty quotes should contain a link to a dx.jar file. In my case, for 4.1.0.GA, it should be ~&#x2F;Android_SDK&#x2F;android-sdk-macosx&#x2F;build-tools&#x2F;22.0.1&#x2F;lib&#x2F;dx.jar

    So the fix:
    Open the Android SDK Manager and make sure the build tools for 22.0.1 of the Android SDK are installed.
