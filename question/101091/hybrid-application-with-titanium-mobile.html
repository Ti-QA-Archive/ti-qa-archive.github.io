  • While i experienced Titanium, basically a mix can be done , probably the way of window's form with some webview embedded.

    The only risk you have is related to the performances, webview pages (local or remote , pay attention where are some limitation due to security) are significantly slower than the native code, for the rest the mix can be done and solve some porting issue. if your engine is based on the web standard you can keep working easily the same code and work around the embedded code. just pass variables.

    For instance take a look here http://audiobox.fm/ . The core is html5 + javascript . the different mobiles keep the same code but embedded their native webview.

    Hope it helps.
