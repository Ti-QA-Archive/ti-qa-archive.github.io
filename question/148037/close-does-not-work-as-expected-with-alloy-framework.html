.close does not work as expected with Alloy framework

Hello,

I have read a lot of threads in the Q&A here as well as threads on Stack Overflow and other communities that the .close() method should completely close and remove a window.

As I understand the documentation and the threads and comments I have read on .close() method is that the window is being closed/killed/set to null when used. This however doesn't seem to be the case when working with the Alloy Framework.

Let me explain it with a very simple piece of code:

mainController.welcome.addEventListener('open', function() {
    console.log("Listener: welcome is open");
    mainController.main.close();
    Ti.API.debug("mainController.main");
});

outputs:

[INFO] :  Listener: welcome is open
[DEBUG] :  [object main]

Ofcause you could always manually set the main = null; which is a "solution" but it shouldn't be necessary?

Oh yea and running this example directly as it's written would not work ofcause as I don't add the $. namespace infront of the variables, but hopefully you get the point. Reason why I am not writing it in the specific example is that I've define a global alloy variable mainController to be able to access the root window from any controller.

Could anyone please explain why the .close() method does not act like it is suposed to, when working with the Alloy framework?

Kindly

    No, calling close on a window does not destroy it. You can always call open again at any time to re-use it.
