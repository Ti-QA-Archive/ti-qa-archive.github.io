Proxy Server Crashing Studio - Unable to Remove

On a Mac (10.6 Snow Leopard), after having used Titanium outside of our proxy for a long time, I noticed Studio had some new settings so I adjusted it to run behind our proxy server. After some tweaks, I was able to check for updates without error. However, as soon as I closed studio and started it again…..it crashes immediately. I cannot undo my settings as it crashes.

I uninstalled studio (using AppZapper) and am no longer behind the proxy, but it is still crashing and I cannot undo my settings. I was perfectly fine not running behind the proxy before, but I cannot revert back.

Here's the crash log: http://pastie.org/2449980

Any ideas? I can't even start studio to make the changes.

  • Accepted Answer

    Hi Mike, I recently had some issues with login on titanium studio. Following instructions from Paul Dowsett, this helped me:

    "Try closing TiStudio and confirming that its processes have stopped. Then rename its settings directory at ~&#x2F;.eclipse (the .eclipse folder in your home directory) and launching eclipse again."

    If in this folder are preferences saved, maybe force to create a new one may help you.

    • Sounds stupid but for anyone new to Mac OS like I was; the eclipse folder is hidden, so the you will need to either use the terminal (ls -a) to view the folder and it's content. Or you will need to Show Hidden files by using the following lines in the terminal and then rename the 'org.eclipse.equinox.security' folder:

      defaults write com.apple.finder AppleShowAllFiles TRUE

      killall Finder

      To hide the folders again use the lines below:

      defaults write com.apple.finder AppleShowAllFiles FALSE

      killall Finder

      Hopefully this will save some pain if anyone comes across this rather annoying issue. thanks for the answer Javier.
