TabGroup + CurrentTab + CreateWindow

I have a single context app that uses the 'window' property instead of 'url' to open and create a new window.

I do not know how to access the current window because the app does not have access to Ti.UI.currentWindow();

My app will create a new window, but when I click the back button bar, the content tthat was added is still present on the page, and overlayed over the previous tab's content.

My question is, how do add content to the newly created window, without having to add it to the tabgroup itself?

Is there an alias for 'Ti.UI.currentWindow'?

I basically have no way to cleanly add the new content to the newly opened window.

Will show code if needed.

    you can do something like that..

    buttons.addEventListener('click', function(e){
myApp.win = Ti.UI.createWindow({
    title: e.source.name,
    navBarHidden: false,
    backButtonTitle: 'Back',
    window: new Page()
});

myApp.tabs.currentTab.open(myApp.win);

});
var newWin = myApp.win;
//TEXTFIELD VIEW
var theView = Ti.UI.createView({

    width: '100%',
    height: 200,
    backgroundColor: '#f00',
    top: 100,
    zIndex: 0
})

newWin.add(theView);

  • If I understood well, you need a way to access the currentWindow. You can actually do two different things (not tested):

    // somehwere.js
var page = require('/ui/Page');
page.currentWindow = win;

// ...

    and then access to it inside your "page" in the following way:

    // /ui/Page.js
self.currentWindow.open(new Window());

// ...

return self;

    Another solution (and easier I think) is to do actually set your currentWindow directly when you open your Window, like this:

    // somewhere.js
Ti.UI.currentWindow = self; // where self = current window, where you are placed

    I'm not sure if you are using CommonJS way or Tweetanium way, as described on Forging Titanium 6, however you can take a look at this app as reference:
    https://github.com/appcelerator-developer-relations/Template.Tabbed

  • Ok it took all day but worked my way around this mess.

    SomeTab.js

    var Page = require('ui/Page');

buttons.addEventListener('click', function(e){
    myApp.active = new Page(e.source.name);
    myApp.tabs.currentTab.open(myApp.active);
});

    Page.js

    var newWin = Ti.UI.createWindow({
    title: name(variable passed in),
    navBarHidden: false,
    backButtonTitle: 'Back',    
    width: '100%',
    height: '100%',
    top: 0,
    zIndex: 0    
});

var theView = Ti.UI.createView({

    width: '100%',
    height: 200,
    backgroundColor: '#f00',
    top: 100,
    zIndex: 0
})

newWin.add(theView);

    So instead of creating the new window in the 'WhateverTab.js' which before happened on the click event, create the window inside of 'Page.js' and add whatever you need to with it. The backNavBar on iOS works as intended and the 'newWin' closes as needed. Thank you Mitul Bhalia for helping out!
