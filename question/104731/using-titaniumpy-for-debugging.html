using titanium.py for debugging

Howdy,

I was trying to use titanium.py (from the mobile sdk) to debug my code.

Now running from my project's folder the following command:

&#x2F;path_to_titanium_mobile_sdk&#x2F;titanium.py run --platform=iphone

makes the app be built/installed/run on the iOS simulator, the console keeps on logging output until I don't kill the simulator itself, great!

BUT running this:

&#x2F;path_to_titanium_mobile_sdk&#x2F;titanium.py run --platform=android

tells you that builder.py is being misused (it ends up printing the usage help).

So I ran:

&#x2F;path_to_titanium_mobile_sdk&#x2F;android&#x2F;builder.py ~&#x2F;my_proj&#x2F; ~&#x2F;my_andoid_sdk&#x2F;

The app buids, installs and runs but the output on the console ends after this line:

[INFO] Deployed Game Guides ... Application should be running.

No other errors/messages, just my console prompt…

How can I get the console keep on prinring debug info for android?

I tried with 1.5.1, 1.5.2 (continous build) and 1.6.0 (continous build).

    Federico

    If you just need to see the logcat/debug/trace output, simply use Android's adb tool:

    adb -e logcat

    Obviously, an emulator needs to be running at the time.
