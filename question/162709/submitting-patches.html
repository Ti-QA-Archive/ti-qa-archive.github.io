Submitting patches

Hi,

While working on our app, we've found several issues / bugs in the Titanium code on Android. Issues are mostly related to activity management, but also in the code generation (NULL pointer access in the templates).

After working the last couple of weeks on the various crashes that we experienced on Android, our app now looks stable, but we've had to make a few changes in the SDK.
We'd obviously like these fixes to be merged back onto the trunk as at the moment we've had to fork the code and we're compiling the SDK ourselves.

A few of these fixes are for some issues that have been raised as tickets in the past on the TIMOB Jira project, but were closed because they could not be reproduced. For some of the other issues we fixed we couldn't find a JIRA ticket.

What's the preferred way to submit these issues? Shall we ask to reopen tickets, shall we create new ones…? Should we talk to a TIMOB developer directly?

Thanks

Eric

    • Nothing will be accepted without the CLA signed.

      For existing tickets, reference the PR that applies JUST TO THAT TICKET.

      For non-existent tickets, create a ticket, with reproducible use case, then reference the PR.

      One PR, One Ticket. Else QE becomes unmanageable.
