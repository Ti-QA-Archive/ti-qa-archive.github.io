  • On osX, I use MesaSQLite open the database and make sure it is behaving as I expect. For iOS, databases are in the ~/Library/Application Support/iPhone Simulator/${Version}/Applications/${appid}/Library/Application Support/database directory. Harder on Android as you need to use ddms to get the database from the file system of the Android Emulator to see it.

In a single interface, can talk to a mysql, build a query, test it, then change frames and run it against the sqlite. Its $70 though for a single seat, but considered it an investment in tools.

  • As Jeff suggested, the database file (.sql) is located in ~/Library/Application Support/iPhone Simulator/${Version}/Applications/${appid}/Library/Application Support/database directory.

    To manage the database, I first rename it to (.sqlite) and open it with Firefox Extensions: "SQLite Manager" (https://addons.mozilla.org/en-US/firefox/addon/sqlite-manager/)
