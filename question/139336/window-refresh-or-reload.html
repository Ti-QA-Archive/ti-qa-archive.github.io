    Let me explain technique that I use. :)

    First, you need to write code that is testable (avoid privates, etc. Search for that).

    Second, I use quasi-test driven approach, I write "test" for every form/component I'm developing. So, for example if I want to develop form on win3, I write test (in Resources/tests) and run that test from app.js

    //app.js
var test = require('tests/win3');

test.run();

// tests/win.js

exports.run = function() {
    var win = ...create window ...
    var componentToTest = ... create component
    componentToTest.setSomething(fakeImplementation); //for example, you can set login data

    win.add(componentToTest);

    win.open();
};

    That way, you can run any form in your app by just replacing one line in app.js. Developing new forms is easier because all you have to do is to write new "test".

    I do have long term plan to replace this approach with better tests and automatization, but for now this is useful enough.

    Hope this helps.
