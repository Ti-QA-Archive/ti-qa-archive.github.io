Add views to repeater object in Alloy data binding

Hi!

Is it somehow possible to access the individual models in a dynamically populated TableRow using a Model? I use the following xml:

<TableView top="8dp" id="tableView" dataCollection="icd_result" dataTransform="updateRow">
    <TableViewRow>
        <ImageView left="0" height="14dp" image="{favourite_image}"/>
        <Label left="18dp" class="tableViewLabel" text="{code}"/>
        <View left="72dp" right="16dp" height="Ti.UI.SIZE" layout="vertical">
            <Label class="tableViewLabel" text="{text}"/>
            <Label class="tableViewLabel tableViewCommentLabel" height="{commentHeight}" text="{comment}"/>
            <Label class="tableViewLabel tableViewAlternativeLabel" height="{alternativeHeight}" text="{alternative}"/>
            <View id="hiddenView" width="Ti.UI.FILL" height="50dp"></View>
        </View>
    </TableViewRow>
</TableView>

and it works nicely, but I can't figure out how to access the individual repeater objects.

The thing is, some of my rows will contain an undefined amount of buttons (in "hiddenView") that depends entirely on the source, so I can't hardcode a certain number of buttons. The amount varies.

So, is it possible to access the repeater object during creation?

I tried using this in the XML:

<View id="hiddenView" width="Ti.UI.FILL" height="50dp">
    {childViews}
</View>

and this in the code:

var icd_item_data = {
    "code" : code,
    "text" : text,
    "favourite_image": favouriteImage,
    "alternative": alternative,
    "alternativeHeight" : alternativeHeight,
    "comment": comment,
    "commentHeight": commentHeight,
    "childViews": "<Label text=\"test\"/>"
};

but that turned up empty. The dataTransform only allows me to access the model, not the Views it create, it seems.

Any ideas?

    Solved it!

    Instead of this:

    <View id="hiddenView" width="Ti.UI.FILL" height="50dp">
    {childViews}
</View>

    I changed it to 

    <Require src="hiddenView" type="view"/>

    and in hiddenView.xml i did this:

    <Alloy>
    <View id="hiddenView" childData="{childViews}"/>
</Alloy>

    childViews was defined as:

    var childViews = JSON.stringify([{
    title: "test"
}]

    Further, in hiddenView.js I did this:

    var childData = JSON.parse($.hiddenView.childData);
for (var i = 0; i < childData.length; i++) {
    args = {
        title: childData[i].title
    };
    Alloy.createController('customButton', args);
}

    And of course customButton is a custom controller that handles the args to but the title where it should be.
