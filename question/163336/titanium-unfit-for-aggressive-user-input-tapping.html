Titanium unfit for 'aggressive' user input (tapping)?

Hi all,

I just started developing with Titanium Studio using Alloy for Android. For now I'm just creating a tableview with some data from a model. When tapping on a row, another window will open with some text displayed.

I noticed that when the app is loading on my android device and I 'aggressively' start to tap the screen to the point where the tableview is being shown. The app will crash… (device notifies: The application has stopped working.) and you have to kill the process.
Or sometimes if I aggressively tap the screen up to the point where the tableview is being shown; the app wont crash and the new window actually opened… but it opened twice!

On normal circumstances i.e. I 'calmly' wait for the app being loaded and I'm not tapping the screen all the time, everything is fine. It seems like the Alloy framework cant keep up when quickly tapping the screen. (Tries to submit the user input while the table is still loading which results in a crash?)

Do more people have this issue? Should I use something else than tableview for displaying data?
I just want my app to work fine, also for user who are more 'aggressive' with tapping the screen…
Maybe I can put a delay for when the app is allowed to process user input? But I wouldn't know how to do it.

Thanks in advance and excuse me for my english :),
Chris

    You can add a variable to set if a row was tapped, you execute the function in the event listener only in case no rows were tapped before, but you'll need to add a function to reset the variable once the child window is open.
