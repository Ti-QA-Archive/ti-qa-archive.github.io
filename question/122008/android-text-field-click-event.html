Android text field click event

Hi.

Using this code:

var win = Ti.UI.createWindow({
    modal: true
});

var tf = Ti.UI.createTextField({
    editable: false,
    width: 200
});

tf.addEventListener('click', function() {
    Ti.alert('Tf clicked!');
});

tf.addEventListener('focus', function() {
    Ti.alert('Tf focused!');
});

win.add(tf);

win.open();

On first click, "focus" event is fired, on second click, "click" event is fired (so, "click" is fired after text field has focus). Is this a bug or expected behavior (documentation has generic explanation)?
