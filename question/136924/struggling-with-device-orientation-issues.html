struggling with device orientation issues

My application should only work in the Portrait mode and it used to do so with older Titanium SDKs. However now, even if I've set only to support Ti.UI.PORTRAIT orientation in my tiapp.xml, and also programmatically set supported orientations for the windows to only contain Ti.UI.PORTRAIT,
for one of the windows it changes elements positioning when rotating device.

A couple of questions here:

  • is there any other ways to lock the orientation?
  • As a work around, I've created 'orientationchange' event listener, and it works fine when the window is first opened and then the orientation is changed. But if the orientation is not Portrait at the very beginning and the window is opened, I couldn't find a way to determine current orientation. Is there a way to do so in the window 'open' event?

Thank you for any help,

Olga

  • In your tiapp.xml try:

    <orientations device="iphone">
            <orientation>Ti.UI.PORTRAIT</orientation>
</orientations>
