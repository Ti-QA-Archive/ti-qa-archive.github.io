    • I'm glad you got it working, though it shouldn't have taken a total reinstall!

      Some things to try in the future (or for other readers):

      select the current SDK

      ti sdk select  # then make sure you have the right version selected

      Then, make sure to update the tiapp.xml file to also use that same SDK version.

      run configurations

      The old run configurations may not be removed on an update (they probably should be, I just haven't tested it). But you can delete them yourself. From the Run menu, choose Run Configurations. Select all of the run configurations under your target operating system (for simulators especially) then click the red X button. They will be automatically re-created the next time you do a build.

      Even without this, if you're running the new version of Studio, you should see the target iOS 8 simulators above the left (file tree) pane. Make sure Run is selected. From the devices list, you should see the various iOS simulators separated by version.

      confirm the right versions

      ti -v  # should be 3.4.0-rc4 or whatever you installed
alloy -v  # should be 1.5.0-rc2
xcode-select -v  # should be 6 or 6.0.1

      try the CLI

      You could try building from the CLI to confirm everything is working (even if Studio is confused)

      # turn off autoselection of simulator
ti config ios.autoSelectDevice false

# then do a build
ti build -p ios  
# you should be prompted for a sim and
# the iOS 8 sims should be listed

    • Depois de diversas tentativas, a única solução que encontrei foi baixar o XCode 5.1.1 e retornar o Titanium Studio, Alloy e Titanium CLI para a versão estável.

      After several attempts, the only solution I found was to download XCode 5.1.1 and return the Titanium Studio, Titanium Alloy and CLI for the stable version.

      Download XCode 5.1.1

      Download Titanium Studio 3.3.0 for MAC

      sudo npm remove -g alloy
sudo npm remove -g titanium
sudo npm install -g alloy
sudo npm install -g titanium

sudo xcode-select --switch /Applications/Xcode-511.app/Contents/Developer
