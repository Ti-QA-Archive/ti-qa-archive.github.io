Simulator Issues after Upgrading Titanium

I recently did a full update of my Titanium and SDK's and I am now running into an iOS 7.1 simulator issue, or at least I think that is the problem. It's weird because in the runtime configuration for iOS I only have one option 7.1, not included is 7.0. All of my projects old and new receive this error when I try to run the simulator:

[ERROR] :  An error occurred running the iOS Simulator
[ERROR] :  dyld: Library not loaded: @rpath/iPhoneSimulatorRemoteClient.framework/Versions/A/iPhoneSimulatorRemoteClient
[ERROR] :    Referenced from: /Users/user/Library/Application Support/Titanium/mobilesdk/osx/3.1.3.GA/iphone/ios-sim
[ERROR] :    Reason: image not found
[ERROR] :  
[ERROR] :  Project failed to build after 1s 498ms

    Hi

    This could be one of two problems;

    • The optional iOS simulator(s) have yet to be download;
      • Downloading and Installing Xcode Components found on the preferences menu.
    • Your app project does not have the run configuration set for the specific iOS simulator you wish.
      • Locate the "Run Configurations" section for this app
      • Choose the SDK and simulator choices you need and "run"
