scoll even broken for Titanium.UI.TableView on Android?

Hi there,

I have implemented this on iOS and it actually works pretty well: a auto complete search field that fills up a tableview. Once the user starts scrolling the table view, the search field is blured, so the keyboard is hidden automatically. I tried the same on Android and basically it seem like the scroll event from the TableView fires all the time. Tried with mobilesdk 1.6.1, 1.7.5 and 1.8.0.1, the keyboard comes up but is hidden immediately. Maybe somebody got a hint for me?

I tried to switch to different event listeners offered by Ti.UI.TableView like move, singletap, click, etc. but nothing gave the targeted result.
