TableView Footer not full-width after orientation change to landscape

Hi,

I'm having a problem with my iPhone/iPad app where if the orientation is changed to landscape, then the footer view has a white space to the left of it which is unclickable. I'm not really sure what to do here. Any ideas?

Here's an image of the problem: http://i.imgur.com/N0BP1.png

    Have you tried setting the width:&#39;100%&#39; on the footer? Percentage dimensions worked best for me when changing orientations in the past. If this doesn't help, can you post some code of your layout?

    • When I set width:'100%', the footer gets offset on both Portrait and Landscape orientations: http://up.andyh2.com/1Y3X3G3e1f2O2I3B3n0O

      Here's the code I use for the footer:

      var tableFooter = Ti.UI.createView({
    backgroundColor: "#FFFFFF",
    width:"100%",
    height:80
});
var loadMoreLabel = Ti.UI.createLabel({
    text:"Load more posts...",
    textAlign:'center',
    width:200,
    height:'auto',
    color:"#0088FF",
    font:{
        fontSize:16,
        fontWeight:'bold'
    }
});
var footerActInd = Ti.UI.createActivityIndicator({
    width: 30,
    height: 30,
    left:5,
    style:Titanium.UI.iPhone.ActivityIndicatorStyle.DARK
});
tableFooter.add(footerActInd);
tableFooter.add(loadMoreLabel);
tableView.footerView = tableFooter;
