Titanium not outputting correct versionCode when I package for distribution

Hi everyone,

I'm having a strange problem with updating one of my apps. This will be the 16th version, and this hasn't happened before.

In my tiapp.xml file I have the following code:

    <android xmlns:android="http://schemas.android.com/apk/res/android">
        <manifest android:versionCode="16" android:versionName="1.16"/>
    </android>

When I go to upload it, I get an error message saying "The APK code version number: 1 already exists. You must increase the code version in your apps AndroidManifest.xml".

When I go into the build folder inside my project folder and open AndroidManifest.xml, it says 'android:versionCode="1" android:versionName="1.16"'.

Does anyone know why It's outputting the versionCode as "1" instead of "16"? This is driving me nuts.

Thanks in advance.

  • Hi

    Have you tried a project clean?

    project -> clean... -> [select project || clean all projects]

    Also if you are using Alloy Framework, you can try deleting the contents of your resources folder found in the root. But ONLY if you are using Alloy

    It may be that your project isn't clearing out the previous compiled project resources etc.

  • I've solved my issue without having to make a custom manifest.

    In my tiapp.xml, I had this code near the top:

    <android xmlns:android="http://schemas.android.com/apk/res/android">
        <manifest android:versionCode="16" android:versionName="1.16"/>
    </android>

    but further down, I had this code in there:

     <android xmlns:android="http://schemas.android.com/apk/res/android"/>

    I don't know why it was in there twice, but whatever. I deleted that and it's working like a charm now.

    Thanks for your advice everyone.
