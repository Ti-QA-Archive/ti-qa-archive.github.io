Ti.API.info output buffer?

I don't see my Ti.API.info output in the console until I perform the action that generates the log calls more than once. Then the Ti.API.info output is displayed twice, as if it was waiting to fill some kind of buffer before displaying it.

I've encountered issues like this before with the console output. That is, output is delayed until another action that generates output has been done. It seemed to be inconsistant until it cropped up in this instance.

Is there a buffer settings somewhere I can turn off or am I barking up the wrong tree?

    Hi

    You should not be experiencing a "buffering" effect.

    If should be no more complicated than;

    Ti.API.info('something');

    or (iOS only)

    Ti.API.info('field', 'value');

    Whilst 2.1.4 is much older than normally used now - it behaved in the same way.
