Android Broadcast Receiver

I have a need to broadcast a custom intent (I'm using native Android to do this as) and this intent will carry extra data.
However the broadcast receiver in Titanium doesn't seem to expose the full intent i.e extras.
My receiver code is:-

var br = Ti.Android.createBroadcastReceiver({
    onReceived: function(e) {
        Ti.API.debug(JSON.stringify(e));
});

Ti.Android.registerBroadcastReceiver(br, ['uk.co.mycompany.broadcast.mydata']);

The object received in the OnReceived event is:

{"intent":{"bubbleParent":true,"action":"uk.co.mycompany.broadcast.mydata","type":null,"data":null,"flags":16}}

although my sender Android native code is:

      Intent broadcast = new Intent("uk.co.mycompany.broadcast.mydata");
      broadcast.putExtra("test", "hello world");
      sendBroadcast(broadcast);

Any ideas?

Thanks in advance.
Vic

  • Problem solved. The intent extras are being exposed - I should have used e.intent.hasExtra("…") instead of e.hasExtra("…"). In my defence, the documentation is very poor for this.
