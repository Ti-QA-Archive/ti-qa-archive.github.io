  • I had a similar problem and found it was a module that I was trying to load in that was causing the problem. Below was the code that I had in my tiapp.xml file.

    <android xmlns:android="http://schemas.android.com/apk/res/android"/>
<modules/>
<modules>
  <module version="0.1">ti.volumeslider</module>
</modules>

    I changed it to this and then I was no longer getting the error. 

    <android xmlns:android="http://schemas.android.com/apk/res/android"/>
  <module version="0.1">ti.volumeslider</module>
<modules/>

    The provision profiles were in the correct location for me so that wasn't causing any issue.
