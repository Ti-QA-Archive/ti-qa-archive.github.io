  • Here is the Solution which worked for me on a Mac.

    1.Open the Finder and go to the Titanium Studio App in Applications

    2.Right Click on the Symbol of Titanium Studio and choose to show the Package Contents

    3.Then go in Contents -> MacOS and open TitaniumStudio.ini

    4.Add in the last Line -Dtitanium.bypassAuthentication=true

    On the next Start the Server Authentication is bypassed and Titanium Studio works again.

    • On a PC, Govinda, check your path and make sure you are using a 32bt JDK there.

      If you have 1.6 32bit, you will need to get 1.7 or 1.8 32 bit, as this is a security problem which hasn't been updated in 1.6.

      Check your JAVA_HOME.

      I have 1.7.0_67 32 and 64 bits installed on my laptop and it is working without incident.

      Also, please have a look at your logs

      <Workspace path>/.metadata/.log

      This will give you information on which JDK/JRE is being used by Eclipse on startup.
