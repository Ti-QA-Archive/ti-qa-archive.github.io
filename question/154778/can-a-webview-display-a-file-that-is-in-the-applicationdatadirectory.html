Can a WebView display a file that is in the applicationDataDirectory?

I am trying to display in a WebView a file that I have downloaded from a server using xhr2 and saved to the applicationDataDirectory directory. The code follows:

var file = Ti.Filesystem.getFile(Ti.Filesystem.applicationDataDirectory, "SomeFile.pdf");
filename = file.resolve(); // get the fully qualified file path 
var webView = Titanium.UI.createWebView({
    url: filename
}); //specify the PDF path...
win.add(webView);

I have tried prepending "file:///" to the filename but that didn't work either.

I am sure the file has been saved because using the iphone emulator saves the file into the
~/Library/Application\ Support/iPhone\ Simulator/6.1/Applications/54ADDE9C-E072-47F6-9E88-C9C4CD1FAE96/Documents directory, where I can see it and can open it from the desktop.

My development environment is as follows:

  • Application type: mobile
  • Titanium SDK: version 3.1.1.GA
  • Platform: iOS, API 6.1
  • Device: iPhone emulator
  • Host OS: Mac OSX 10.8.4
  • Titanium Studio, version 3.1.1.2013061122350

Any help or pointers to documentation would be greatly appreciated.

Thanks,

Daryl

  • hi,

    you need to use .nativePath property instead of using .resolve() method.

    try this one..

    var webView = Titanium.UI.createWebView({
    url: file.nativePath
});

    or else you can also set data property for pdfs…

    var webView = Titanium.UI.createWebView({
    data: file.read()
});
