Table Search Glitch

I am having an issue with a search field on a tableview.

I'm using Titanium SDK 1.7.2 (although the issue appears in 1.7.3 CI builds) in iOS Simulators and devices running iOS 4.3 and 5b7.

Here's what's happening:
I have a table with about 40 dynamically appended rows. I have a search bar added to the table.

When I start searching, the table filters properly, but it looks like it's created another table ON TOP OF the existing table and filtering that table instead.

Everything is working fine, function-wise. This is strictly a UI issue.

Strange, right?

Getting a screenshot is difficult as I can't slide the table to show the overlapping while pressing the buttons to get a screenshot.

I don't know if it'll help, but here's my code:

The search bar

var SearchBar = Titanium.UI.createSearchBar({
  top:          70,
  left:         0,
  right:        0,
  height:       45,
  hintText:     'Search the list',
  keyboardType: Titanium.UI.KEYBOARD_NUMBERS_PUNCTUATION,
  showCancel:    false
});

The table

var Table = Titanium.UI.createTableView({
  filterAttribute:  'title',
  search:           SearchBar
});

  • The window is created in a tabgroup. I first create the window with this:

    var ActionWindow = Titanium.UI.createWindow({
    url: 'action_playerselect.js',
    backButtonTitle: 'Back',
    title: 'Screen Title',
    backgroundColor: Ti.App.BackgroundColor
  });

    Then open it like this

    Titanium.UI.currentTab.open(ActionWindow, { animated: true });

    Now, that Ti.App.BackgroundColor is defined elsewhere in the app (our app has a theming system of sorts) and is set to translucent in this case.
