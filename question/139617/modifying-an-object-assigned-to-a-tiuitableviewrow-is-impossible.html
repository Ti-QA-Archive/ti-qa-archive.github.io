Modifying an Object assigned to a TiUITableViewRow is impossible

I've got a strange problem that I'm hoping someone can answer for me:

If I create a TiUITableViewRow, assign an object to it and then try to modify that object, I can't!

Here's an example:

var myObj = { test:true, something:'hi' };
var row = Ti.UI.createTableViewRow(
{
    hasChild : false,
    testObj : myObj,
});

// Try to modify the object assigned to the Row:
// If you're running a debugger, this won't do anything to the object!
row.testObj.something = 'hello!';

// This on the other hand will, however the reference in the row won't get updated!
myObj.something = 'hi there';

The code above will create an object and after assigning a reference to the object to the row, attempt to modify it. If you have a debugger attached you'll notice that the value isn't set!

If you modify the original object, the value is correctly set, but the reference in the row isn't!

For some reason if you assign an object to a Row, it becomes read only?

Why is this?! Is this a bug?
