  • Accepted Answer

    Mobile web won't work with any browser that doesn't support Object.defineProperty.
    IE7 doesn't, and IE8 is limited only to DOM elements.

    • Actually, we did shim Object.defineProperty(), btoa(), atob(), JSON.stringify(), and JSON.parse(), however that's not why IE8 dies. IE8 dies when we shim the console.* object for browsers that don't support the entire console API. This is by accident. Of all the things we could die on, I bummed that's the reason. IE8 would have failed anyways because we don't shim ES5 array functions or addEventListener(), though we could have shim them all. Besides, do you really want to build apps for IE8 and older?

  • TItanium Mobile Web does not officially support desktop browsers for production use. You can do it, but your mileage may vary. The platform is designed for mobile devices. With that said, Mobile Web will mostly run on IE9 or newer, and even then, there's some things that IE9 can't do. We're hopeful IE10 catches up to Chrome 10.
