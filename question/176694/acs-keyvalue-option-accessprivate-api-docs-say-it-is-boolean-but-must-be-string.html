ACS Keyvalue option access_private: API docs say it is boolean, but must be string

I'm on SDK 3.2.1 and Titanium Studio, build: 3.2.3.201404181442

I've tried but could not open a TIDOC Jira ticket.

Problem is that the documentation says that access_private option is boolean, but when I set it to boolean I end up with a request like:

[INFO] :   GET: https://api.cloud.appcelerator.com/v1/keyvalues/get.json?key=.......&_session_id=.....&name=.......&access_private=1&user_id=......

and a response like: 

400 invalid key-value name

It only works when I set it to string, so the request is made like this:

INFO] :   GET: https://api.cloud.appcelerator.com/v1/keyvalues/get.json?key=.......&_session_id=.....&name=.......&access_private=true&user_id=......

To make this a question, what is the best way to get the docs corrected, so others don't loose their time like I did loose mine?
