    • Sure you can! Appc does:

      Your JS files / Resources -> 1st Compilation (generate native code)
      From Native code -> 2nd compilation (with native tools) -> Binary

      So, in theory, you can do:

      1. Modify the native code generated after the 1st compilation, and then just compile it with native tools.

      2. Go and modify the Appcelerator "Precompiler" so you can generate native code with your changes.

      Both ways will have their difficulties, will require knowledge from you about the whereabouts of each native platform, and it's not officially supported by appc.

      Other way to sightly change the behavior of components is the creation of custom modules, which are bits of native code created with the idea of being used in the normal/usual Ti Appcelerator javascript land.

      If you want to know more about the modules creation, check this document. It's pretty likely that what you need is not modify the titanium platform, but create modules so you can get those nifty details solved as you like.

      Remember that will be easier to port your modules to new versions of the SDK, than port your changes to the Ti platform.

      Best,

      Mauro
