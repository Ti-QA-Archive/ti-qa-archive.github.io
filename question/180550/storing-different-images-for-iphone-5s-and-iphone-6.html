Storing Different Images for iPhone 5s and iPhone 6

I apologize if this topic already exists- I searched the Q&A and did not find anything relevant.

I understand that as a Titanium developer you can store different versions of images optimized for different platforms. For example, you can resize an image using your favorite image editor such as Photoshop or Sketch and store specific optimized versions into Resources/android/images/res-hdpi, Resources/android/images/res-ldpi (for Android) and Resources/iphone/images (for iPhone). And then when you want to specify the image to use in your Javascript module, you can simply do something like:

image: '/images/My-Image.png'

and it will automatically select the correct image to load for whichever platform you are using.

While it is clear what you are supposed to do to achieve this effect across multiple Android screen densities (or aspect ratios, which is a completely different concept that should not be confused!), it is unclear how to achieve this effect across multiple iPhone aspect ratios. iPhone 5s has a different screen size (different width and height) than iPhone 6. My app works and looks great in iPhone 5s. Now when I try to use the iOS simulator for iPhone 6, all of my images are too small…they don't fit the screen well. Not that I necessarily want to just stretch them- this could end up looking bad. I want them to look crisp and formatted correctly for whichever aspect ratio or screen density I am using. I care about the way my apps look.

How do I specify different images in Titanium for different iPhone aspect ratios and screen densities? It seems nearly impossible to find anything on this searching either Google, Stackoverflow, or this very forum, without pulling up incredibly irrelevant search results. Thank you.

    If you want different images for iPhone6 and iPhone 6+. then use "-667h@2x.png" for iPhone6 and @3x.png for iPhone6+.
    for example.. your image is imgHome@2x.png (for iPhone5s) and same image with large size rename it imgHome-667h@2x.png (it use in iPhone6) and imgHome@3x.png (it is for iPhone6+).

  • Kari,

    What you really want is this. What this means is that you put all images in one directory for ios, but create three different densities. So if you have an image called 'a.png', you would have 3 in your directory.

    • a.png would get used for an iphone 4
    • a@2x.png would be used on a 5
    • a@3x.png would be used on a 6

    So, you create all three with the correct size and titanium will use the correct one based on the device. You simply reference a.png in your app and the correct version will be loaded.

    • The problem is not actually with image resolutions to use, of course this should be used as Nilesh stated. it's more with the layouts. You can use Ti.Platform.displayCaps.platformHeight and Ti.Platform.displayCaps.platformWidth to get screen sizes.

      You have multiple alternatives to tackle that. Branching is one of them: You can even create a separate module that returns different styles depending on the phone, and use applyProperties to apply them.

      
var style = {
    iPhone4: {height: 50, width: 100},
    iPhone5: {height: 80, width:100},
    iPhone6: {height: 120, width: 130}
}
function getDevice(){
    // here you check Ti.Platform.displayCaps.platformHeight to return the correct device
    // for example:
    return 'iPhone4'
}
function getStyle(){
    return(style[getDevice])
}
var s = getStyle();
s.image =  '/images/My-Image.png'


var img = Ti.UI.createImageView(s);

      Another way would be using percentages :

      
var img = Ti.UI.createImageView({
    width: '80%', height;'60%'
});
