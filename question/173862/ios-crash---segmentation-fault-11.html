iOS crash - Segmentation fault: 11

Starting with iOS 7 our Titanium application won't run and crashes with following exception:

Application 'UIKitApplication:com.abc.testapp[0x7c12]' exited abnormally with signal 11: Segmentation fault: 11

This happens when the app access memory that is no longer available. I am not sure what to fix in the JavaScript code as I do not know how the TI garbage collector works, but it seems that some object is deallocated too soon.

App was built using Titanium 3.2.3.201404181442 and SDK 3.2.3.GA.

  • I will answer myself. After 3 hours of wasting my time, trying to find some problem in JavaScript, I finally looked at line 1613 in TiViewProxy.m of SDK 3.2.3.GA and noticed this piece of code [self parentForBubbling] that returned ISA variable parent. Further debugging revealed that ISA variable parent was indeed released at this point. To fix the problem, I changed two methods in the offending file:

    -(void)setParent:(TiViewProxy*)parent_
{
    [parent_ retain];
    RELEASE_TO_NIL(parent);
    parent = parent_;

    if (parent_!=nil && [parent windowHasOpened])
    {
        [self windowWillOpen];
    }
}

    and

    -(void)dealloc
{
    RELEASE_TO_NIL(pendingAdds);
    RELEASE_TO_NIL(destroyLock);
    RELEASE_TO_NIL(parent);
    pthread_rwlock_destroy(&childrenLock);

    //Dealing with children is in _destroy, which is called by super dealloc.

    [super dealloc];
}

    And voila, no crashing anymore. I hope this helps someone.
