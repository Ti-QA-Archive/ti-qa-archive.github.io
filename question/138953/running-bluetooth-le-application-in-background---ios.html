Running Bluetooth LE Application in Background - iOS

I'm using Titanium to build a Bluetooth LE application for iOS however because Titanium doesn't support BTLE functions I've created an xcode module. Therefor any solutions via xcode or Titanium would work.

The Main problem I'm having is that I want my phone to be updated with the data from the Bluetooth device at a certain interval. Even when the app gets minimized to the background. I've been using This iOS Reference to understand the different app states. Now, as I understand it, when you add the UIBackgroundModes key to your info.plist file and specify bluetooth-central within that array the app will remain able to execute certain Bluetooth commands.

The problem I'm having is that, when the app goes into the paused state, I set a timer at a user defined interval where the app will then execute Bluetooth code which will connect to a device, get the data, etc. After the 10 minute cutoff mark however, the app gets suspended if no Bluetooth device has been connected to. I've found the in order for the app to remain active in the background a device must be connected to every 10 minutes.

Is there anyway around this problem?? I've been searching all over the web to try and find a solution but have turned up nothing.

Any help would be greatly, greatly appreciated!

Thanks
