iOs Cloud Storage & Database install path

Hi,

Since iOS 5 and iCloud integration the database should be placed in: 

Library / Caches

But when i install my database using Ti.Database.install, my files are placed in 

Library/Application Support/database

Does anyone know a tip for that problem ?

Thanks.

  • If your db stores critical data that cannot be regenerated, then it should not be stored in caches as that would make it subject to deletion by iOS. Apple wants this type of data to use iCloud, but that is not available yet in TiSDK.

    There is a ticket on the help desk that discusses this issue, and the fix that apple recommends is to use a 'do not back up' flag, explained here. The Jira ticket for implementing this in Ti is here.
