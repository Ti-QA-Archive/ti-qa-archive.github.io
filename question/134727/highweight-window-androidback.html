Highweight window android:back

I have one-window application and i mimic the window opening on android. Instead of working with more windows , i just hide views and show others.

Now , the problem is , when i make the window lightweight , all the contexts , variables and stuff mess up. My code is like this:

app.js
win1 = Ti.UI.createWindow({title:"TEST"});
category.addEventListener("click",function(){
Ti.include("/category.js");
alert(win1.categoryElement.left);
});
options.addEventListener("click",function(){
Ti.include("/options.js");
});
win1.open();

category.js
uiElement = Ti.UI.createView({
left:10,
backgroundImage:"categories.png"
});
win1.categoryElement = category;
win1.add(category);

And it works pretty well. Since i want to switch between these windows (home,category,options) i have a variable with current window position etc. And it works great. Now i need a back button so i can go back.

After switching win1.open() with win1.open({modal:true,fullscreen:true}); there are errors like win1.categoryElement is null etc. etc.

So , is there any way to bind android:home on a highweight window?

  • Ti.include() seems to be a very stupid feature because in some cases it creates a new context and in some it uses the current one. And defining variables is not working as it should.

    Even if i create a global context , win1.object it still creates an error.

    So i copy-pasted the code from all .js files instead of the Ti.include(); and it works as it should.
