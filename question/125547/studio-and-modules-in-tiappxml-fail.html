Studio and modules in tiapp.xml FAIL

TiStudio will not let me debug/run when any module references are in the tiapp.xml. I noticed that the default code has </modules> but not <modules>. 

    </iphone>
    <android xmlns:android="http://schemas.android.com/apk/res/android"/>
    </modules>
</ti:app>

If I modify this, TiStudio will not allow me to run the app in the simulator.
If the <module> reference is not in there, then my modules are not found in the compile and the build fails.

In TiDeveloper, everything works fine. Create a new project and the <modules></modules> section is there. I add in my <module></module> line and run, it comes up fine in the simulator.

    </iphone>
    <android xmlns:android="http://schemas.android.com/apk/res/android">
    </android>
    <modules>
        <module version="1.0">ti.opengl</module>
    </modules>
</ti:app>

There seems to be parts missing from TiStudio's tiapp.xml.
How do I fix it?

Mac OSX 10.6.8, SDK 1.7.2, Fresh TiStudio download and install.

  • This works for me:

        </iphone>
    <android xmlns:android="http://schemas.android.com/apk/res/android"/>
    <modules>
        <module version="1.0">ti.opengl</module>
    </modules>
</ti:app>

    Note there is no </android> tag.

    Interestingly though my default tiapp.xml has <modules/> not </modules>.

    • When a tag has / just BEFORE the closing > then it is a shortcut for an empty tag. That is:

      <modules></modules>

      is exactly the same as

      <modules/>

      In essence, the single tag is both the open and the close of the tag. Taking it one step further you can see the android tag is really the same thing:

      <android … />
