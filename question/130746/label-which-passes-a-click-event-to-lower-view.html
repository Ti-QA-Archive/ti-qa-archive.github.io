Label which passes a 'click' event to lower View.

Please give me advice.

When some views are piled up, is there good method to pass a 'click' event by upper view and receive a 'click' event by lower view ?

// Create mapview.
var mapview = Ti.Map.createView({
    top: '0dp',
    left '0dp',
    regionFit: true,
    animate:   true
});

// Create label on mapview. (Not react to touch operations.)
var label = Ti.UI.createLabel({
    text: 'Latitude = ' + latitude + '\nLongitude = ' + longitude,
    top:      '8dp',
    left:     '8dp',
    height: '128dp',
    width:   '96dp'
});

win.add(mapview);
win.add(label);

I would like to display on mapview a label in which touch operation is ignored.

In this code, a user can not control a mapview because a label overlapped on a mapview blocks operation.

Target: Both iOS(5.0) and Android(2.2)., SDK: Titanium mobile 1.8.0.1

  • Accepted Answer

    var label = Ti.UI.createLabel({
    text: 'Latitude = ' + latitude + '\nLongitude = ' + longitude,
    top:      '8dp',
    left:     '8dp',
    height: '128dp',
    width:   '96dp',

    touchEnabled:false

});
