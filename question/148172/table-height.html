Table Height

I'd like to display multiple tables vertically because I want to bind a different collection to each table. Unfortunately, each table sizes itself to be as tall as the containing view. Is there a way to stop a table from taking up all available space and instead to make itself exactly as tall as its elements?

If that won't work, is there something else I can bind a collection to so that I can control size?

  • You can set the height on a tableview to prevent it filling the entire window

  • use Ti.UI.SIZE in inner tableviews to use only required height or width 

    
height : Ti.UI.SIZE
    — commented February 12th 2013 by Muhammad Adnan

  • @Nick - I don't want each table to have a particular height value, I just want the table to be exactly as tall as it needs to be.

    @Muhammad - Everything within the TableView and the TableView itself have their height assigned to Ti.UI.SIZE. The table still sizes itself to the height of the entire parent.

    — commented February 12th 2013 by Matthew Carroll

  • Apologies, it looks like when I intended to give the table a height of Ti.UI.SIZE, I instead typed Ti.UI.FILL. That would explain the height issues.

    So to anyone who finds this later on, if a TableView and everything within that TableView are given a height:Ti.UI.SIZE, that table should only be as tall as it's elements.

    Also, on a related note, if you want to disable scrolling on your table (because there is no need to scroll a table that shows all of its content), just set scrollable:false on the TableView.

    — commented February 12th 2013 by Matthew Carroll
