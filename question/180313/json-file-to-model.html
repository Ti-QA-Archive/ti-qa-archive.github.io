JSON file to model

Hello,
I am reading a json file in my app, and populating a listView. In the back-end I have a mySql server, and my json file is generated by a php function that retrieves data from the DB. I also have a PHP function in the back-end that deletes a single entry form the DB, I can call this function from my app by sending the request to myURL+elemendID. The function deletes the element referred by the ID in the URL and return an updated list, this list populates the updated listView again.

Ok, sorry if confusing, I hope it’s clear enough.

Now, my question is: provided that I want to have some limited interaction with the elements form the listView (for instance, select the elements that I don’t want to see anymore, even if they are still in the database, or order my list by name, date, etc.) , should I load the json file in a model first? And in general, should I always use a model and an associate collection to handle loaded lists of elements?

Thank you for your help/opinion

    It is not a requirement, but using some sort of functionality to encapsulate your data is always a good thing
