Entitlements are not valid

I just don't get what I'm doing wrong.

Testing on my phone or distributing all bails telling me my Entitlements are not valid.

I'm working around the " Resources" bug by duplicating my "Resources" dir and making sure Entitlements.plist are the same in both places. All have get-task-allow to false

The strange thing is that this was working at some point. And then it all went downhill. I haven't changed anything big or anything.

I'm using a wildcard provisioning profile, but it doesn't work with a distinct one either.

Steps to reproduce:

  1. Wipe build directory
  2. Run in emulator (to rebuild and check)
  3. Install on device (Fails due to " Resources")
  4. Duplicate "Resources" to " Resources"
  5. Install on device - installs and dies at the end saying "Entitlements are not valid"

Same with packaging for distribution.

I've tried skipping Titanium and building it in xcode (4), build succeeds - I set the "Code Signing Entitlements" setting to "Entitlements.plist" but the Product > Archive menu is disabled.

I've been doing nothing but this for the past 48 hours and I'm about to pull my hair out.

  • I actually fixed it. 48 hours of staring at this and it's finally working.

    So first of all, I'm building in Xcode (4) instead of Titanium.

    I set my "Code Signing Entitlements" to "Entitlements.plist"

    Underneath that is "Code Signing Entity" - I had set that to my distribution profile… but… if you expand it, you have to set every single one to the same distribution profile. You have to do the both of those for the Project AND the TARGET as well.
