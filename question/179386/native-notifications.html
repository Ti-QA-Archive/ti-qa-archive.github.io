Or could we use other services for notifications?

  • You can achieve the native notification without any cost to the project. You don't have to pay anything if you are using titanium. Its open source framework. Titanium API is supporting native notifications in iOS and for android (GCM) you can use open source module (You can easily get it on github or gitio). Below us the sample to register for notification in iOS.

    Titanium.Network.registerForPushNotifications({  
   types:[  
     Titanium.Network.NOTIFICATION_TYPE_ALERT,
     Titanium.Network.NOTIFICATION_TYPE_SOUND,
     Titanium.Network.NOTIFICATION_TYPE_BADGE    
   ], 

   success: _onRegisterToIOSSuccess,  
   error: _onRegisterToIOSError,  
   callback: _onNotificationReceiveIOSCallback
});

  • Also keep in mind that Appcelerator's notification platform is just a proxy to what you call the "native" ones… Appc's servers just communicate with Apple/Google for you. If you don't like it, you can always swap out Appc for your own server code.

    • Hi J ZI,

      Appcelerator is the organization name. They have two products,

      1. Appcelerator Platform - (Paid one)
      2. Titanium - (Open Source)

      If you are creating your app using Titanium, they will not charge you for that even if your app is commercial.

      There paid Appcelerator Platform providing some cool features like automation testing tool, Live view, Analytics etc. which is not available in open source titanium version.
