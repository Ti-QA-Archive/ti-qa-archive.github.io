  • Thanks for this perfectly timed tip, just what I needed :)

    And great job, Yusuke!

    Worth mentioning that it uses AJAX long polling as its transport. Maybe it would be possible to write a websocket implementation using Titanium.Network.Socket.TCP (1.7+) that Socket.IO can use? At least while waiting for built-in websocket support in Titanium (see this issue).

    There is actually another GitHub project (by another Japanese!), TiSocketIO, which is a Socket.IO client for Titanium written in Cocoa (using socketio-cocoa). It obviously only works on iOS, whereas Yusuke's socket.io-titanium should theoretically (I haven't tried it yet) work on Android as well.
