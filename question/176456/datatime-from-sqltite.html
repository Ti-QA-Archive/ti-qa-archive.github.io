  • From the sqlite documentation:

    1.2 Date and Time Datatype

SQLite does not have a storage class set aside for storing dates and/or times. Instead, the built-in Date And Time Functions of SQLite are capable of storing dates and times as TEXT, REAL, or INTEGER values:

TEXT as ISO8601 strings ("YYYY-MM-DD HH:MM:SS.SSS").
REAL as Julian day numbers, the number of days since noon in Greenwich on November 24, 4714 B.C. according to the proleptic Gregorian calendar.
INTEGER as Unix Time, the number of seconds since 1970-01-01 00:00:00 UTC.
Applications can chose to store dates and times in any of these formats and freely convert between formats using the built-in date and time functions.

    Now, yours appears to be stored as a real or an integer (which only you can know, and you should have mentioned in your question).

    Use the momenJS library (built in to alloy projects) via var moment=require(&#39;alloy&#x2F;moment&#39;); or available from momenjs.com to format the date to something you can use and display. Moment documentation available via the link I provided.

  • Your time is being stored as Epoch time (seconds that have elapsed since January 1, 1970 midnight UTC/GMT). To construct a Javascript Date object from this, do the following:

    var mySqlDate = 1213788000;
var d = new Date(mySqlDate * 1000); //convert to miliseconds, and create a date object
alert(d.toString());

    The Javascript Date object does not contain many formatting helpers for displaying the date/time as a desired string. I'd recommend checking out XDate to perform that. Specifically check out the Formatting Documentation.
