  • Hi Steven

    I answered this just last week (and two weeks before for iOS). It is a popular question.

    Android and iOS they are basically the same answer - but each one has extra comments that are worth noting.

    The problem is that the points you are using are based on the direction changes not the actual route points. These are held in an encoded form in the property polyline.point and need to be decoded. Both exmaples provide code to smooth out the points automatically for you and works great on Android (need 2.1 or higher) and iOS.
