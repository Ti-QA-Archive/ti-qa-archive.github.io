Avoid Geolocation prompt when not using user's location?

I am using two map views in my app to show some locations to the user. I don't use the user's current position and set the userLocation property of the mapViews to false. Anyway, the prompt for allowing the app to use the user's location shows up at least once…
Is it obligatory to show this prompt when using maps even without usage of the geolocation services or is there a way to turn the prompt off?

  • Hi Aaron, thanks for the quick answer. I am not using any methods of the geolocation module, not even a reference of Ti.Geolocation is found in the code of the app. My mapViews are basically set up like this:

    var startPos = {latitude:46.9480943365053, longitude:7.44744300842285, latitudeDelta:7.5, longitudeDelta:7.5};
    var mapView = Titanium.Map.createView({
    mapType: Titanium.Map.HYBRID_TYPE,
    region: startPos,
    animate:true,
    regionFit:true,
        annotations:[],
    userLocation:false

    });

    They get added to the currentWindow, the annotations are then populated with locations after loading them from a web service.
