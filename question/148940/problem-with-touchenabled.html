Problem with touchEnabled

Hey,

i do have a container(view) containing a button. I want to set the touchEnabled of the container to false, but the button have to be work - of course^^

On Android this works fine, setting touchEnabled=false on the parent view. On ios however it does not work, cause the butotn isn't clickable anymore, too.

I've googled, but could'n find a solution. Any hint how to achieve this "android behavier" to ios?

Greetings, nIco

    Hi

    I think you will find the iOS behaviour is correct and the Android behaviour will be normalised to that of the iOS version soon.

    The problem can be explained like this;

    • Imagine you have a bike (your button) and it does not have a lock on it
    • You place your bike in a shed (your container) that does have a lock on it
    • You should NOT be able to get the bike as you cannot get into the locked shed.

    You will need to either;

    • Not worry if the container probably does not need any events processed anyway, so touchEnabled = true is fine
    • Place the button OVER the container and not in it

    If for whatever reason you cannot achieve this; expand on WHY you need a container that cannot be touched with a button inside that should be.
