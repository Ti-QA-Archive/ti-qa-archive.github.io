ti.admob working on iOS 8 for emulator but not in device (release 1.9)

Hi.

I have my app working right on Android, but on iOS I'm not having luck. The test banner appears right on the emulator, but on the real device I'm getting this message:

<Warning>: <Google:HTML> You are currently using version 6.12.0 of the SDK, which doesnt officially support iOS 8. Please consider updating your SDK to the most recent sdk version, 7.0.0, to get iOS 8 support, including a fix for smart banner rendering in landscape mode.   • I'm not sure what's up with that error message. Google's release notes say that 6.12.0 does support iOS 8.

    The good news is that you don't need anything special to build your own version of the module. You can fork the ti.admob repo and replace the contents of ios/GoogleMobileAdsSdkiOS with a newer version of the SDK.

    If you could find 6.12.2, you probably wouldn't have to change anything in the code. But all I can find is 7.0.0, which might require some changes to the code.

    • I think that if ti.admob was not 64-bit compatible, you would not have been able to build a 64-bit version of your app, and Apple would have rejected you immediately for not being 64-bit compatible (unless this is an update to an existing app, in which case you have until June to figure this out).

      I don't think that your missing ads are a symptom of non-64-bit support. More likely would be that your build would not work or if it did, your app would crash entirely (I'm not sure what happens in the iOS world if you try to build with 64-bit support and you use a module that doesn't have 64-bit support).

      Besides, if you have a look at the ti.admob changelog, you can see that 64-bit support was added back in November of 2014.
