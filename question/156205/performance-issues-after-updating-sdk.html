Performance issues after updating SDK

Hi,

i updated my SDK to 3.1.2 and was using 3.1.0 before.
Now i'm experiencing a decreased performance of my app when running it on an iPhone 4.
This shows e.g. when scrolling a ScrollableView which is now very stuttering or in very laggy page transitions.

I still have a build (exactly same code) using 3.1.0 which does not show this decreased performance.

Now i have two questions:

  • What is causing this decreased performance with the new SDK? Can i do anything about it or better where should i start to search for the problem?

  • Is it possible to revert back to SDK 3.1.0?
    I tried just switching to it, but get
    '[ERROR] : TypeError: Arguments to path.join must be strings'
    when trying to build it, even when using node.js 0.8.x

Thanks in advance, best regards Rob.

PS: The app is build with alloy, if this helps…

  • You can just change the SDK version you are using by editing the tiapp.xml, or doing so through the tiapp editor in Titanium Studio. If you'd like to identify or remove Alloy as a possible culprit, you can easily try different versions of Alloy using the instructions in this link: http://developer.appcelerator.com/question/155021/how-do-i-install-alloy-any-version-or-rollbacks
