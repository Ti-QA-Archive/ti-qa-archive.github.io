Map tiles missing in Samsung phones

I have an Alloy app that displays a map with annotations, using the method outlined here:

http://docs.appcelerator.com/titanium/3.0/#!/guide/Google_Maps_v2_for_Android

This works well in most tested Android devices. However, in Samsung phones, with Android 2.3.5 (and possibly other versions), the map view appears, with the annotation markers, but the map shows as a blank grid instead of map tiles.

On the problem devices, the built-in Maps app shows maps as expected.

The android part of tiapp.xml is below.

Any ideas?

    <android xmlns:android="http://schemas.android.com/apk/res/android">
        <manifest android:label="Golf" android:versionCode="14" android:versionName="1.0.2">
            <uses-library android:name="com.google.android.maps"/>
            <uses-permission android:name="android.permission.CALL_PHONE"/>
            <!-- Allows the API to download data from Google Map servers -->
            <uses-permission android:name="android.permission.INTERNET"/>
            <!-- Allows the API to cache data -->
            <uses-permission android:name="android.permission.WRITE_EXTERNAL_STORAGE"/>
            <!-- Use GPS for device location -->
            <uses-permission android:name="android.permission.ACCESS_FINE_LOCATION"/>
            <!-- Use Wi-Fi or mobile connection for device location -->
            <uses-permission android:name="android.permission.ACCESS_COARSE_LOCATION"/>
            <!-- Allows the API to access Google web-based services -->
            <uses-permission android:name="com.google.android.providers.gsf.permission.READ_GSERVICES"/>
            <!-- Specify OpenGL ES 2.0 as a requirement -->
            <uses-feature android:glEsVersion="0x00020000" android:required="true"/>
            <uses-permission android:name="redacted.permission.MAPS_RECEIVE"/>
            <permission
                android:name="redacted.permission.MAPS_RECEIVE" android:protectionLevel="signature"/>
            <application>
                <meta-data
                    android:name="com.google.android.maps.v2.API_KEY" android:value="redacted"/>
            </application>
        </manifest>
    </android>

  • In my case this was caused by an incorrect value in the API key. I was able to get maps working in debug and production on a Samsung S3 and Nexus 7 by following these steps.

    1. Make sure you're published app is using the same keystore as your debugger in your titanium configuration
    2. Log in to google developers console and select your application
    3. Check the application has an API key for Android Applications
    4. Check that the SHA1 key in your keystore matches the one listed in the API key 'Android Applications' list
    5. Check that the api key is the 'android:value' in your TiApp.xml in the following section

    …good luck, the fate of the empire is in your hands

    <application>
    <meta-data 
        android:name="com.google.android.maps.v2.API_KEY" 
        android:value="******api key here****"/>   
</application>
