HTTPClient response processing

Does the HTTPClient do any response processing?

I am using SDK 2.1.4 on iOS and the response I get when making the same HTTP call from Titanium is not the same as I see in the Chrome Developer Console.

The page I am calling, makes a JS call once it has been loaded to load some ajax content. When I do the call through Chrome, I can see page being loaded, and then the JS call loading the inner content. But with Ti's HTTPClient, I just seen the inner response.

is there anyway to debug this further?

  • Hi

    Calling HTTPClient will only return the raw html/data from the page you requested. It will not process anything that is run on or after the page load. This includes any Javascript calls on the page.

    If you are in control of the page, then separate the data from the html on the page and process that using the call to httpClient.

    If you are not in control of the page then you could always use YQL which returns data after a page has loaded. Not as elegant as a plain API call but it will work.

  • Actually I found the cause for this. The HTTPClient by default sends a header that caused this to happen. 

    'X-Requested-With: XMLHttpRequest'

    Once I set this header to null before sending my request everything worked as normal.

    client.open('GET', url, false);
client.setRequestHeader('X-Requested-With', null);
client.send();
