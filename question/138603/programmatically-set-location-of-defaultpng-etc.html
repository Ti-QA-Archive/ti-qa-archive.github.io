Programmatically set location of Default.png, etc

I am building an app that will have multiple "versions" within the AppStore. Most of the configuration and data used in the app will be downloaded from a server. However, I need the "branding" to be different; Splash Screen, icons, etc. I'm able to handle most of the branding (app name, version, etc) through multiple versions of tiapp.xml that I will just swap out, but swapping all these images might be a pain.

Is there any way to programmatically indicate these images rather than relying on Titanium to find them in their default locations?

  • Are you referring to multiple versions such as (for example) having a "Lite" free version and a "Full" paid version within the AppStore?

    If your talking about "Lite" and "Full" these need to be completely separate apps, as per Apple Guidelines, so you should probably just bite the bullet and have multiple projects, multiple images, multiple tiapp.xml. Otherwise when your distributing the app and testing it will get very confusing.

    • Ok that makes sense, well I'm not really sure you can do that through Titanium without a little hacking, since the file that manages the icons (the Info.plist in XCode) is constant per app…

      BUT :) You can specify your own custom Info.plist.

      So I would do the exact same thing your doing with your tiapp.xml: for each school, create their own set of images (in a subdirectory), tiapp.xml, and Info.plist (which you can drop in the resources directory and titanium will put it in the project). Go here to see how you can define your icon files for each plist: Apple Has a Page Here, go to "Add Icon files in Info.plist."

      So that handles all the images for app startup and icons, if you also want to differentiate between in-app icons thats just simple if-else style switching.
