    Hi

    This depends on how you are storing the date. You said text but that could be in many ways;

    • 26-07-2014 - dd mm yyyy
    • 07-26-2014 - mm dd yyyy
    • 2014-07-26 - yyyy mm dd
    • 17th of July 2014 - date string

    The problem with all but one of these is that they cannot be sorted correctly.

    • 26-07-2014 - all 26s will be grouped together with 07s as the second sort and then 2014s
    • 07-26-2014 - all 07s will be grouped together with 26s as the second sort and then 2014s
    • 2014-07-26 - all 2014s will be grouped together with 07s as the second sort and then 26s
    • 17th of July 2014 - I hope you see how bad this one will work

    The 3rd example is the correct way as the big to small display means they will always be in the correct order. So these examples;

    SELECT * from setting order by date;

    and

    select * from setting where date='2014-07-26';

    Will work perfectly.

    You can add the time to this as well as that is already capable of sorting as hh:mm:ss is big to small. So yuo can have date, time and date/time fields correctly stored.

    You can also store the data as a number, many JavaScript functions convert the date to a number, you can use that, the number always increments in time so is inherently sortable anyway.
