Listview - event.source doesn't provide the row

I want a popover to appear when a user touches a image on a row in a listview.

on itemclick, there's the usual 'source' property, but it seems to reference the entire listview area rather than the clicked button. The popover appears tom/bottom of the whole listview rather than the row/button itself

Is there any way to get a source object suitable for passing to a popover? The bindIDs are just text obviously, whereas I need the view itself so that popover.show can work correctly.

Any help appreciated!

    Hi

    This is by design;

    "Because of list view's approach, list view's performance is faster but limits access to items in the list, that is, you cannot directly manipulate the views, set view properties and bind event callbacks the same way as you can with table view rows. The next subsection describes how to use list items."

    • Ti.UI.ListView

    The ListView is meant to be used when you need raw performance WITHOUT direct interaction with the structure within in row.

    If you need specific events per item within the row, you need to be using Ti.UI.TableView.
