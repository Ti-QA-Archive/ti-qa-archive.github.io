I got info from TableView ScrollEnd is

[INFO] {

    Have you seen the demo in KitchenSink for the table pull header?

    I noticed something strange with the scrollEnd event too with a recent SDK.

    It seems it is not fired until the table has actually stopped 'springing' back to the top (to contentOffset.x == 0)

    I think in previous SDK versions, the event was fired as soon as the table view was released from the pull down.
