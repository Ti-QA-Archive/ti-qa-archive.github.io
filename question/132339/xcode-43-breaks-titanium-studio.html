Xcode 4.3 breaks Titanium Studio

So I installed last night Xcode 4.3 and when I ran it, it asked me if it can delete the old Xcode 4.2. So I said yes. It turns out it deletes the entire developer directory with Xcode in it because Xcode 4.3 is now an app in the /applications folder. When I ran Titanium Studio and tried to run an app, it said that my iOS SDK is not installed. I guess Studio looks only in the /developer/applications/xcode directory. The problem is that does not exist any more as of Xcode 4.3. I went into Studio->Preferences->Titanium Studio->Titanium and tried to "configure" the iOS SDK home there, but it says it does not exist. Any help would be greatly appreciated as for now, I can only build for Android.

  • The issue is that Xcode 4.3 removed xcode-select (unless you had it previously) and switched a number of paths. Please watch TISTUD-1196 for the Studio fix to find the new location, though there will still be related issues for platform (such as an issue with ios-sim). The fastest workaround is not use 4.2.1 in the meantime.

    Note that we will continue to use xcode-select if it exists, and then fall back to other options if that fails.
