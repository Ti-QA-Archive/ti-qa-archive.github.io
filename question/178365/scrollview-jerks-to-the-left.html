ScrollView Jerks to the Left

Image:

I’ve tried loads of things, but I seem to keep having this problem.
There is a ScrollView that is defined to be the width of the screen.
There is a modal window that pops up.

Now, on the iPhone 3.5" simulator (and sometimes on the 4" as well),
often (not always!), closing the modal window seems to shift the entire page to the left,
as if the scrollView is wider than the screen.

I have to drag the scrollview back again.
Yet the scrollView is explicitly defined to be the width of the screen, as is the window!

I’ve tried countless variations of properties, to no avail,
if anyone can point me in the right direction that would be great!

Cheers,

I’ve defined a scrollView as such:

var settingsScrollView = Ti.UI.createScrollView({
width: Ti.Platform.displayCaps.platformWidth,
height: Ti.Platform.displayCaps.platformHeight,
contentWidth: Ti.Platform.displayCaps.platformWidth,
left: 0,
right: 0,
bottom: 0,
top: 0,
layout: 'vertical',
showVerticalScrollIndicator: true,
showHorizontalScrollIndicator: false
});

And a modal window as such:

var citiesWindow = Ti.UI.createWindow({
    modal: true
});

The settingsWindow is added to the main window:

var settingsWindow = Ti.UI.createWindow({
    orientationModes: [Ti.UI.PORTRAIT],
    width: Ti.Platform.displayCaps.platformWidth,
    height: Ti.Platform.displayCaps.platformHeight,
    backgroundGradient: gradients.currentSettingsGradient(),
    navBarHidden: true,
});

settingsWindow.add(settingsScrollView);

  • I think you're not really using the ScrollView for its intended purpose. It looks like you're trying to use it to implement a set of tabbed panels.

    ScrollView is more for when you have a long list of things, and you want the user to be able to scroll through them with touch-and-drag.

    If you want to roll your own TabGroup (make sure you have a good reason for not using the built-in TabGroup), you can do so, but don't use a ScrollView for your main panel.

    There are a couple of ways to do this:

    • use a separate view for each tabbed panel; when the user clicks a tab button, build a new view and replace the main view of the app (I prefer this technique)
    • build one big view that is N times the width of the display; build your different views within this larger view, and then call setLeft() on it to bring the appropriate view on screen; you can even use animate() if you want it to slide

    Presumably, you were using ScrollView so that changing tabs would give you an animated effect. You should be able to achieve this without using ScrollView, using the second technique I describe.

    If you do go for this technique, I would recommend building a CommonJS module for each of the tabbed panels. My typical approach (others will disagree with me) is to use parasitic inheritance, and have my CommonJS module derive from the view object itself. Then you can just instantiate a new instance of your class and add it to the big multi-screen view.

    Hope this helps!

  • Man, I'm 0-for-2 on trying to figure out what you're doing.

    If you're just doing vertical scrolling, you shouldn't have a problem. I will mention that I never use contentWidth or contentHeight. Try doing this:

    var settingsScrollView = Ti.UI.createScrollView ({
    top: 0,
    left: 0,
    right: 0,
    bottom: 0,
    layout: 'vertical',
    showVerticalScrollIndicator: true,
    showHorizontalScrollIndicator: false
});

    Notice I'm leaving off width, height, contentWidth, and contentHeight. The top, left, right, and bottom settings should take care of the dimensions of the view. I would avoid trying to match view sizes to the display size whenever possible, opting for these kind of "gravity" parameters instead.
