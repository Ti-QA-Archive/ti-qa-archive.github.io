Module compilation on 1.8 RC1

I cannot get my module compiling on 1.8.0.1 RC1.

My extension is a TableViewProxy subclass which adds some extra functionality to tableviews (ie, removing separators). It isn't very big, consisting of around 30 lines of code.

I get the following log when running the ant script from Titanium Studio (latest beta).

It worked fine before upgrading from a previous nightly build.

I installed the NDK and added android.ndk=&#x2F;Users&#x2F;blanka&#x2F;Development&#x2F;android-ndk-r7 to the build.properties folder, and added the apiversion: 2 entry to the manifest file as described at http://wiki.appcelerator.org/display/guides/Android+Module+Porting+Guide+for+1.8.0.1

I have also removed references to TiContext from my extension as mentioned in the guide, and generally done what it says to do.

As you can see below, there are a lot of zip errors, which weren't happening before. I guess it can't open some jar files? There are also errors compiling the ndk stuff, seems like errors in the generated c++ code.

I have tried re-creating the module with the 1.8.0.1.RC1 titanium.py file and copying across the code, but the same thing occurs.
