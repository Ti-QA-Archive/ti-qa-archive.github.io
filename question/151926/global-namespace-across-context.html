Global namespace across context

Hi, I'm a newbie,
I followed the video tutorial provided by AppU and I have a question about namespaces.
In the video tutorial about cross-platform applications is suggested to use a global namespace in which you create sub-namespaces related to DB methods, application's custom UI ecc, the so called myApp namespace that you should declare in app.js
But if I create an application composed by different contexts (because I use commonJS module or the url property) how can I let myApp be visible for other context?
If I need to share db methods across the contexts how can I do? (I suppose that I have to pass the namespace as an argument to the module that need it).

Thanks in avdance,

Luca

    Hi Luca

    Firstly if your using the commonJS method as you should be, then global namespaces should NOT be used..

    Secondly, commonJS doesn't create new instances of the javascript interpreter but using the URL property on a window does.

    So effectively, use commonJS without namespaces, and without using the URL property on windows, and you will not have any issues with as you put it different contexts, but is different instances of the javascript interpreter.

    • Yes that is correct a commonJS module is a self contained context or scope.

      But by exporting the variables or functions required it makes them accessible by the parent (requiring) module.

      That is how you make the values available across applications.

      Also you really shouldn't combine global namespace with commonJS they are two very different programming techniques.

      So by creating a commonJS module with the variables you want to share, then exporting those variables you can require that module where required making the values available across the app.

      Hope this makes sense, spending time now understanding commonJS will really benefit you in the long term.
