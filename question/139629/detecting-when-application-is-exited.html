Detecting When Application is Exited

I need to be able to tell if a user has exited my application by interruption (phone, sms, other alert) or by pressing the home button on iOS. Is there an event that will let me listen to those two scenarios, BUT not fire when the device goes into sleep mode? I've looked into both Ti.App.pause and Ti.App.paused but they appear to have similar results.

  • http://docs.appcelerator.com/titanium/2.1/index.html#!/api/Titanium.App-event-pause
  • http://docs.appcelerator.com/titanium/2.1/index.html#!/api/Titanium.App-event-paused

Thanks for any help you can provide.

  • I was able to complete what I needed by combining the Ti.APP.paused event with Ti.App.idleTimerDisabled property set to true, simply preventing the device from going to sleep.
