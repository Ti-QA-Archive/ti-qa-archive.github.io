iPad Screen Orientation

I'm having difficulty getting Titanium to gracefully handle orientation changes in my iPad app, so I want to force certain (or all) screens into landscape mode.

I'm using:

Titanium Developer: 1.2.2
Titanium SDK: 1.7.0 (continuous, from March 22)
iOS SDK: 4.2

I have this in my tiapp.xml file:


    <iphone>
      <orientations device="iphone">
        <orientation>Ti.UI.LANDSCAPE_LEFT</orientation>
        <orientation>Ti.UI.LANDSCAPE_RIGHT</orientation>
      </orientations>
      <orientations device="ipad">
        <orientation>Ti.UI.LANDSCAPE_LEFT</orientation>
        <orientation>Ti.UI.LANDSCAPE_RIGHT</orientation>
      </orientations>
    </iphone>

I have this at the top of my app.js file:

Ti.UI.orientationModes = [Ti.UI.LANDSCAPE_LEFT,Ti.UI.LANDSCAPE_RIGHT];
Ti.UI.orientation = Ti.UI.LANDSCAPE_LEFT;

Every window that I create in my application is done like this:

Ti.UI.createWindow({
   left:0,
   top:0,
   width:1024,
   height:748,
   orientationModes:[Ti.UI.LANDSCAPE_LEFT, Ti.UI.LANDSCAPE_RIGHT]});

Yet when I run it in the simulator and rotate it the app rotates, even though everything on this guide says otherwise. Can anyone tell me why?

  • I've done some research on the "iPad must support all orientations" issue, and I've found this:

    http://developer.apple.com/library/ios/#qa/qa1689/_index.html

    According to that page, and I quote:

    If it is essential that your application run in only one orientation (for instance, portrait only), then you should:

    • Launch in your supported orientation, regardless of the current device orientation.
    • Avoid displaying any UI element that tells people to rotate the device.
    • Support both variants of an orientation.
