HTML5 Ti app using href sms/tel/mailto

simple urls to open Message, Phone, Mail (sms, tel, and mailto) don't work inside a Titanium wrapped app. Is there a way to get them to work without have to use Titanium-specific code? I just want my hrefs to open these apps like they did in PhoneGap wrapped html5 apps. My preference is to stick with pure html5.

Thanks,
Don

  • The short answer is no. You need to use the Ti API to achieve this, even if it is by including local js files in your HTML files. I would actually question whether you actually need to use Titanium at all if an HTML5 App is what you are after, PhoneGap fits that need perfectly. The main thing to remember is that Apps created using Titanium produce native Apps that utilise native controls specific to the platform you are targetting. PhoneGap just wraps the HTML in a WebView and you dont get the advantages of native controls that user's expect from their device.

  • I solved it using this technique
    https://wiki.appcelerator.org/display/guides/Communication+Between+WebViews+and+Titanium

    Basically, one line of JS in my web app

    $('.sms').on('click',function(){
    var number = $(this).data('number');
    Ti.App.fireEvent('app:fromWebViewSMS', { message: number });
    });

    and this in my app.js

    Ti.App.addEventListener('app:fromWebViewSMS', function(e) {

        Titanium.Platform.openURL('sms:'+e.message);

    });
