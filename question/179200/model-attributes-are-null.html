  • Accepted Answer

    Try using callback functions on the fetch statement. It may be that your console statements are executing before the DB select has completed.

    device.fetch({
    id: 1,
    success: function(model) {
        // Off the top of my head, I can't remember if the model param
        // passed in here is a simple JSON object or a Backbone Model object. 
        // Assuming the former for the sake of this example
        console.info(model);
        console.info('id -> ' + model.id);
        console.info('secondary_id -> ' + model.secondary_id;
    }, 
    error: function() {
        console.error("Yikes, an error has occurred!");
    }
});

      Even w/o the callback, it works by calling device.get("secondary_id") instead of device.secondary_id.

      Now, the question is why? Why does is the 'id' field accessible w/o the get() call but the 'secondary_id' field is not?

      — commented November 21st 2014 by Cord Awtry

    • Backbone models don't expose attributes directly. You need to use the .get() method to retrieve them.

      They do however have an id attribute which is mapped to id attribute of the model data. You could have an attribute called device_id which served as an ID and you can tell Backbone to use that attr by setting the attributeId. Accessing model.id would return the value of model.device_id.

      See http://backbonejs.org/#Model-idAttribute for a better explanation
