    • I hate this errors… :(

      Some things you can do:

      1- Clean build folder.

      2- Ensure you are using the correct provisioning profile: the DEVELOPMENT profile for testing on device and the DISTRIBUTION profile for deploy. Go step by step and use clear names to identify de provisioning profiles. Install on system when you download from Provisioning portal (double click them after download). My profile names convention:

      > MyApp_iPad_development_profile.mobileprovision

      > MyApp_iPad_DISTRIBUTION_profile.mobileprovision

      also, for quick-start development I have one, associated to my company domain (com.criteriastudio.*) that I only use for development:

      > Generic_iOs_development_profile.mobileprovision

      I always write the name of the app, the word "distribution" on caps to clearly differentiate, and the devices compatibles with the app. Sorry if I insist a lot on this, but is a very common issue.

      3- Ensure the same App Id (like com.mycompany.myapp) is in the project and in the provisioning profile.

      4- DO NOT use whitespaces on project name. If you have changed the name of the project, go to point 1 and start again :)

      hope this helps.

      Javier
