Android Permissions (WRITE_EXTERNAL_STORAGE)

Hi!

How can I prevent Titanium setting android.permission.WRITE_EXTERNAL_STORAGE as a permission in the AndroidManifest.xml?
I've made several changes to the AndroidManifest directly in the tiapp.xml. I've set all the permissions I need in the tiapp.xml but Titanium is still setting some by it's own! It's a clear thing that many Android Users dismiss Apps that needs permission they can't follow. For that point it's important to set only that what I need!

Is it possible to change the auto set of that permissions or is it like this Post much earlier: http://developer.appcelerator.com/question/118483/android-uses-permissions

Thanks for your help!

  • this worked for me:

    <android xmlns:android="http://schemas.android.com/apk/res/android">
  <manifest>
    <uses-permission android:name="android.permission.CALL_PHONE" />
  </manifest>
</android>
