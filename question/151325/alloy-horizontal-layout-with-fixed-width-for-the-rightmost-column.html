Alloy horizontal layout with fixed width for the rightmost column

Hi, I want to create a three column horizontal layout with Alloy. The first column has dynamic width 30%, the second column will fill the center area, and the third column is a fixed width column (for toolbar).
I try something like the code below. 

<Alloy>
    <Window id="winDashboard" layout="horizontal">
        <View id="viewProfile" left="0" width="30%" height="Ti.UI.FILL" backgroundColor="#093328">
            <Require src="userProfile"/>
        </View>
        <View id="viewExamList" width="Ti.UI.FILL" right="50" height="Ti.UI.FILL" backgroundColor="#E9E7E3">
            <Require src="examList"/>
        </View>
        <View id="viewToolbar" width="50" height="Ti.UI.FILL" backgroundColor="#628A37">
            <Require src="dashboardToolbar">
        </View>
    </Window>
</Alloy>

It made a 50 pixel gap on the right by using property right=&#39;50&#39; on the second column, but the third column is somehow not appearing. I've also tried left=&#39;-50&#39; on the third column, thinking that column is outside the screen, but its still has not showed up. It did showed up when I set the width of the 2nd column to 70%. However this is not what I want since the third column is layered on top of the second column.

Anyone knows how to achieve this correctly? Any help will be appreciated. Thanks.

FYI: I'm on Mac using Ti SDK 3.0.2 GA, TiStudio 3.0.2.201302191606, Alloy 1.0.0, develop a mobile app for iOS 6.1 and Android

    OK, first off, don't mix percentage, constraints, and static dimensions of elements all in the same layout. There's just about no way that's going to behave as you would expect. I took the liberty of creating your described layout in Alloy. In addition, I also included how you can use the Ti.Gesture API to readjust the dimensions of the elements on orientation change.

    app/alloy.js

    Alloy.CFG.leftWidth = Ti.Platform.displayCaps.platformWidth * 0.3;

    index.xml

    <Alloy>
    <Window class="container">
        <View id="left"/>
        <View id="middle"/>
        <View id="right"/>
    </Window>
</Alloy>

    index.tss

    "#index": {
    backgroundColor: "#fff"
}

"#left": {
    left: 0,
    width: Alloy.CFG.leftWidth,
    height: Ti.UI.FILL,
    backgroundColor: '#f00'
}

"#middle": {
    left: Alloy.CFG.leftWidth,
    right: 50,
    height: Ti.UI.FILL,
    backgroundColor: '#0f0'
}

"#right": {
    right: 0,
    width: 50,
    height: Ti.UI.FILL,
    backgroundColor: '#00f'
}

    index.js

    Ti.Gesture.addEventListener('orientationchange', function() {
    var width = Ti.Platform.displayCaps.platformWidth * 0.3;
    $.left.width = width;
    $.middle.left = width;
});

$.index.open();

    Also, if you want to be able to see the orientation changes in the ios simulator, make sure you add all the supported orienations in the tiapp.xml file:

    tiiapp.xml excerpt

            <iphone>
        <orientations device="iphone">
            <orientation>Ti.UI.PORTRAIT</orientation>
            <orientation>Ti.UI.UPSIDE_PORTRAIT</orientation>
            <orientation>Ti.UI.LANDSCAPE_LEFT</orientation>
            <orientation>Ti.UI.LANDSCAPE_RIGHT</orientation>
        </orientations>
        <orientations device="ipad">
            <orientation>Ti.UI.PORTRAIT</orientation>
            <orientation>Ti.UI.UPSIDE_PORTRAIT</orientation>
            <orientation>Ti.UI.LANDSCAPE_LEFT</orientation>
            <orientation>Ti.UI.LANDSCAPE_RIGHT</orientation>
        </orientations>
    </iphone>
