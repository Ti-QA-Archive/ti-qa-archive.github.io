3.2.0-beta ti cli unable to select avd

Titanium Command-Line Interface, CLI version 3.2.0-beta, Titanium SDK version 3.1.3.GA

According to this "fixed" bug
there should be an –device-id option to "ti build". There isn't. So, how might I select ans avd?
nb. I don't use studio, just pure cli.

    It may be a little confusing, but there is a Titanium CLI 3.2.0 and a Titanium SDK 3.2.0. In order to specify –device-id, you must have both Titanium CLI 3.2.0 and Titanium SDK 3.2.0, then set the <sdk-version> in your tiapp.xml to SDK 3.2.0 (or newer).

    The good news is we just released Titanium CLI, Titanium SDK, and Titanium Studio 3.2.0 tonight! Here's the blog post. Refer to the bottom of the post for instructions updating the various Titanium products using Titanium Studio.

    Simply set the the <sdk-version> in the tiapp.xml to "3.2.0.GA" and you should be ready to rock.

    If you do not specify a –device-id, Titanium will select the most appropriate device or emulator. If you specify a –device-id "all", then it will install your app to all connected/running devices/emulators.

    To disable the device auto selecting, run:

    ti config android.autoSelectDevice false
ti config ios.autoSelectDevice false

    If you don't specify a –device-id, Titanium will prompt you to select a device from a list.

    Please let us know what you think! :)
