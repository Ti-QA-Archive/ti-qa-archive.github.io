Titanium.map Issue

The Titanium Maps module uses version 1 of Google Maps. Google is no longer providing api keys for version 1. The Maps module mentions needing to use a new plugin for Android, in order to use version2 of Google Maps, but makes no mention of what to do on iOS.

Since the module recommended for use with Android is marked as "Android only" and since Google no longer provides API keys for the version used by the native Titanium module what should one do to implement maps in iOS?

— asked May 16th 2013 by Stuart Tannehill

  • Accepted Answer

    Use the normal Titanium.map. It works without any key iOS.

    As cou can see on the API page, the statement is for android only :

    Android Platform Notes

    This module provides native map support using Google Maps API v1. As of March 3, 2013, Google is no longer issuing Google Maps API v1 keys and this module can no longer be used to support native maps on Android unless you have these keys.

    For new Android applications, use the Google Maps v2 add-on module to add native maps support for your application. If you have an application that still uses the old keys, you can either transition to the new module or continue using the built-in Titanium.Maps module.
