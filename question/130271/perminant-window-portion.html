Perminant window portion

Goal :

build browser like interface for a portion of an app

Min-Requirements :

constant back & forward buttons, address input & refresh

attempted :

using static html wrapper with iFrame content area

problem :

various sites will choose to block/notload/display alternative content if displayed in an iFrame

possible solutions

most of which I can't figure out how to approach :(

1) extending the titanium UI or creating a component to include static navigation bar

2) finding a way to load two HTMLDOMs in the same window without frames

3) extending titanium to make iFrame this it is HTMLDOM root window

4) un-appealing : inject fixed navigation as iframe into each page after complete rendering

5) even less appealing : making it appear that a window is contained in another window by remoking the chrome for the 'contained' window and moving/sizing it as the 'wrapping/parent' window moves and sizes

I'm really not sure where to begin other than #4 or #5 which are really lame ways to accomplish what I am looking to do. a good iframe problem example would be to try loading http://www.google.com which will simply not load the page content if it detects iFrame embedding.

Any help would be greatly appreciated.

  • Hello,

    one approach that you can do is creating the address input with a textfield on top, and implement the buttons by yourself. Everytime you press the reload button or the "go" button, fire a custom event for the webview. You should create custom events in the webview: what to do when a go event happens, a reload, etc. and update the webview's url according to that. More info on the webview here.

    This functionality are not implemented in the webview component because is made just to show a page, not to implement all the browser functionality.
