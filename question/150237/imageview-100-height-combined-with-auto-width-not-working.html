imageView 100% height combined with auto width not working

For some reason, it looks like setting an imageView (with remote image) to 100% height and auto width is not working.

I want the image to fill the height of the parent container. This container has deviceHeight, 100% of the screen. But I need the image to be scaled correctly, so I tried width="auto". For some weird reason this result in 2 black spaces at the top and bottom. Image does not have 100% height.

Only when I set width to a static value, the image height is 100%.

Code:

img.width = 'auto';
img.height = Ti.UI.FILL;  // also tried '100%'

Result: http://postimg.org/image/sx7jbepxv/ (black spaces, image does not have 100% height)

Code 2:

img.width = 300;
img.height = Ti.UI.FILL; // or 100% height

Result: http://postimg.org/image/tnz64i213/ -> correct height of 100%, but with static width, so not preserving the ratio.

Any good ideas on how to get this done? So what I need is: 100% height of the image, auto width, so it scales correctly.
