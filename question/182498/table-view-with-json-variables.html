  • Hm, after first look at your code, without testing anything it seems, that you shouldn't use title property on a table view row in combination with child views. This is where you should consider using custom rows like:

    var row = Ti.UI.createTableViewRow();
var rowTitle = Ti.UI.crateLabel({
    text: servers[i],
    height: 25,
    left: 64
});
var timerLabel = Ti.UI.createImageView({
    image: points[i],
    width: 48,
    height: 48
});
var serverLabel = Ti.UI.createLabel({
        title: matchups[i],
        color : '#1B325F',
        left : 10,
        width:"33%",
        backgroundColor:"blue",
        height: Ti.UI.Size
});
var serverLabel1 = Ti.UI.createLabel({
        title : matchups[i],
        left : "33%",
        color : '#1B325F',
        width:"33%",
        backgroundColor:"red",
        height: Ti.UI.Size
});
var serverLabel2 = Ti.UI.createLabel({
        title : matchups[i],
        left : "66%",
        color : '#1B325F',
        width:"33%",
        backgroundColor:"yellow",
        height: Ti.UI.Size
});

row.add(rowTitle);
row.add(timerLabel);
row.add(serverLabel);
row.add(serverLabel1);
row.add(serverLabel1);

    And now just push your row to data set to display. You probably want to further customize styles for it, but this should give you a nice start.

    Generally, when you want a custom row layout, you should build it yourself entirely. row properties like title, image are only usable, when you wont't add any other view.

    I would recoment to package this row layout into js file, so you would just require it (in Classic Titanium), or make widget for it or row template (Titanium Alloy).

    Cheers.
