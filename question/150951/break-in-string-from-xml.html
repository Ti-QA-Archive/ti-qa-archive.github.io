Break in string from xml?

Hey,

i am pulling some text stuff from xml (localization) everything works fine - except the breaks :/

In my XML strings there is (for example)

<string name="hello_world_text">Hello \n World</string>

If i write this stuff "per hand" in a API.info call everything is fine. But the text from the xml does not converted "correctly" - my label field inside the app just shows the "\n" instead of showing a break.

Any idea?

SDK 3.1, ios

  • You can use regex to perform a global replacement and turn the '\n' character pair into the line breaks you need.

    var file = Ti.Filesystem.getFile(Ti.Filesystem.resourcesDirectory, 'test.xml');
    if (file.exists()) {

        var xmltext = file.read().text;
        var docObj = Ti.XML.parseString(xmltext);
        var doc = docObj.documentElement;
        var string = doc.getElementsByTagName("string");

        $.testLabel.setText(string.item(0).getText().replace(/\\n/g,'\n'));

    }    else{
        alert("file cannot be found");

    }

});
