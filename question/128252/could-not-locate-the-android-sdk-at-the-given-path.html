  • Matthew

    I am currently investigating some problems that seem to arise when users update their Android Tools packages to revisions 15 and 9.

    Would you try something for me, please?

    • close Studio, delete the system variable ANDROID_SDK
    • start Studio and install the latest continuous build from the master branch of our continuous builds site, using Studio's Help > Install Titanium SDK from URL function. Note that you don't need to use this version (1.8.0.1) when compiling your apps - you just need it for the python scripts that Studio will automatically use
    • restart Studio
    • try to configure the Android path again in Studio, being careful to click the apply button if necessary

    Let me know how you get on. I will check back frequently until this is resolved.

    Thanks

  • Are you on Windows, Mac OS, Linux ? Windows sometimes has difficulty with embedded spaces in paths so the default location for the SDK on windows should be something like c:\android_sdk
    Also, you need to ensure your environment variables for ANDROID_SDK are set (in your .profile for Mac, or System Environment settings for Windows). Close/Reopen TI Studio after any such change too

    2.- Execute again Titanium Studio . Change in Windows/preferences/Titatium and click "restore defaults" , close it . Open it again and browse to android sdk folder and it works.
