Android ICS - restart required on recent SDKs

Hi,

Having just released the latest version of an Android app I'm getting several reports of users getting the 'Restart Required' message on first run, and one user is reporting getting this message each time the app is run. The users reporting this are using ICS. Having done some emulator testing my self I can confirm this is impacting ICS. Further to that the issue does not appear with Titanium SDK version: 1.8.2 (02/23/12 17:46 59b3a90) but does with (for example) Titanium SDK version: 2.0.1 (04/12/12 16:33 999c68a) and later.

Having looked at Q&A and not having experienced this before I've altered the android manifest to include the below properties, but to no avail. Has anybody else been experiencing this or are there any suggestions on how to get around it. My immediate instinct is to rerelease with a 1.8.2 build

<property name="ti.android.bug2373.skipAlert" type="bool">true</property>
<property name="ti.android.bug2373.message"></property>    
<property name="ti.android.bug2373.disableDetection" type="bool">true</property>

    Those entries should be in the tiapp.xml not the androidManifest.

    The restart required is to be expected for a long drawn out reason that I don't agree with entirely, but am told it exists and supposedly we are to accept as inevitable despite only Titanium builds seemingly affected with work arounds that will end the world if we use them. </endrant>

    Yes, some ICS users seem to get the prompt over and over and over. We (not with appcelerator) don't have enough info/data to figure out what is unique about those users as follow up is difficult through the markets. Never seems to happen to beta testers. (grin)
