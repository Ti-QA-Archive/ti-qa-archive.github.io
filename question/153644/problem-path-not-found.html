  • Hi

    You question is confusing. You say "produits/KS_nav_views.png" generates path not found, but the only time you reference that image you do not include the path that is shown. You then explain you are using a httpClient call and parsing JSON but the image problem appears to be unrelated.

    If it is this image that is causing the problem, then you are better off including the full path for the location of the image as depending on the window creation technique the file could be referenced in relation to that file.

    If you have placed the file in one of the following folders;

    • {appname}/resources/images/KS_nav_views.png
    • {appname}/resources/android/images/KS_nav_views.png

    …then reference like this;

    image:'/images/KS_nav_views.png',

    This location…

    • {appname}/resources/KS_nav_views.png

    …would be referenced like this;

    image:'/KS_nav_views.png',

    My own personal preference is to change all filenames and file references to be lowercase, this way there are no accidental reference problems due to incorrectly typed case.
