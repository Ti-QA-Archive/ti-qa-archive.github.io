UnicodeDecodeError: 'utf8' codec can't decode byte 0x80 in position

I'm not actually asking a question… I'm posting on my problems with running into encoding problems with files.

Was receiving this when trying to run Android emulator:
[TRACE] UnicodeDecodeError: 'utf8' codec can't decode byte 0x80 in position 1818: unexpected code byte

There was way too many files in my project to hand change, so I came up with this terminal command (on Mac). First, make a backup. Then, cd to your Resources folder, and run the below command to batch convert all files to utf-8.

find . -name "*.js" -exec iconv -f ascii -t utf-8 "{}" >"{}" \;

Mine stumbled across one hidden file hiding down in a subfolder I never would have found. I deleted that hidden file (with rm command), and now it's compiling again. Phew.

This was the original thread I saw with people having the same problem. Comments are closed or I would have put it there:
http://developer.appcelerator.com/question/88591/unicodedecodeerror-utf8-codec-cant-decode-byte-0x80-in-position
