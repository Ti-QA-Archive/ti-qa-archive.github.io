Random Vertical Space when using NavigationGroup on iPad Windows

Hi all-

I am seeing a strange issue here. I am creating an iPhone / iPad application that creates a window, with a child NavigationGroup. The NavigationGroup is needed to navigate between several child windows. Within the context of an iPhone, all is well. Within an iPad though, I have an Alloy specified height/width, so the window does not consume the screen. What I have seen is that when the window is opened, above the NavigationGroup is empty vertical space, about the size of the status bar. I tried to remove the NavigationGroup from the window and add a simple ToolBar, and the vertical space disappeared.

I do not see any top, or other elements that could cause this random vertical shift to happen, and wonder if others experienced similar shifts when using the NavigationGroup on a window on the iPad?

Here is my code:

View:

<Alloy>
    <Window id='order' class="container">

                        <!--
                        <Toolbar id='orderBar'>
                    <Button onClick="cancel">Cancel</Button>
                </Toolbar>-->


                <Button id='previewOrder' enabled='false' onClick='previewOrder'>Preview Order</Button>

            </Window>
        </NavigationGroup>
    </Window>
</Alloy>

Style:


'#order': {
    //top: 0,

    layout: 'vertical',
    navBarHidden: true//,
    //modal: true
},
