  • @Rama Krishna I've also experienced this error after upgrading to the 3.2.0.GA SDK. Does the error persist on your end? Do you have Ti modules embedded to your app?

    After removing a Titanium Module the TiProject I was testing launched successfully. Looks like it wasn't built for 3.2.0.GA. I had also an issue into timodule.xml (wrong parameters) leading to "Unable to find generated R.java file" .

    Being said, you may want to remove Ti modules defined into tiapp.xml, if you have more than one in order to find out which the culprit is. Probably you do not even have any Ti modules so I just wanted to let you know what worked for me.

    Be also aware there were some breaking changes in 3.2.0.GA.

    If I can help any other way, let me know. Thanks.

  • I also faced same issue , maybe due to admob/gcm modules added in my project.

    What I did to make it work again :
- Reverted to old titanium sdk 3.1.3 ( from tiapp.xml )
- Uninstalled alloy( version 1.3.0 ) : npm remove -g alloy
- Installed alloy ( older version 1.1.3 ) : npm install -g alloy@1.1.3

    Maybe this could help someone.
