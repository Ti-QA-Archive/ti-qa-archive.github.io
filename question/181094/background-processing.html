Background Processing

Hi,

I'm looking to design a feature, that when a user enables it, is always on and periodically checks their location and notifies them of places nearby they are interested in.

My first question is, how do you launch a service like this without requiring the user to actually start the app? Is that possible?

Second, are there any recommendations or guidelines as to how to build such a feature such that it is not a major drain on the users battery?

BTW - I'm looking to do this for both iOS and Android.

Thanks for any advice.

  • Hi Frank,

    for this requirement you no need to create background service. just add location event listener of Titanium.Geolocation. it automatically fire when user location will change and you can refresh data according to current location..

    if you check it in background service by periodically then it consume so many battery of device.. so it's better to refresh location list on location change…
