Move a view from a view to another one

Hello,

I would like to move a view between views. I have two views "tagsViewOrigin" and "tagsViewDestination". There are some "tag"s in the "tagsViewOrigin", and I want them to "move" to "tagsViewDestination" when I click them. I've tried several things, but none of them works. The code I'm using right now is the next one:

var tagsViewOrigin = Ti.UI.createView({

});

var tagsViewDestination = Ti.UI.createView({
    layout: 'horizontal',
    touchEnabled: false,

    height: 80,
    borderColor: '#999',
    borderRadius: 3,

var tag = Ti.UI.createView({

});
tag.addEventListener('click',function(e) {
    tagsViewDestination.add(this);
    tagsViewOrigin.remove(this);
});

var tagLabel = Titanium.UI.createLabel({

tag.add(tagLabel);

Any of you have any idea on how to represent that "move" thing graphically?

  • I know you can remove a view from one parent and add it to another – that's how I deal with the android mapview limitation (only one mapview allowed per application on android).

    I also know that you can't add a view to one view while it's still added to another view. In your code, you're trying to add the view to the tagsViewDestination before you remove it from tagsViewOrigin.

    Try removing the view first, then adding it.

  • To part question, and Jason pointed you in the right direction on part one.

    Part two of the question was how to represent the move graphically:

    Take a look at toImage().

    • Create a new imageViewCopy that mimics the look of the viewToMove imageViewCopy.image = viewToMove.toImage() and has the coords of the existing view
    • attach imageViewCopy to viewToMove's parent container in same coords.
    • remove viewToMove, attach to new container but hidden
    • animate the movement of the imageViewCopy
    • on release hide() the imageViewCopy and show() the viewToCopy in its new location
