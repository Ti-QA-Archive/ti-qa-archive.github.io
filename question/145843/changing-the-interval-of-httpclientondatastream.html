Changing the interval of HTTPClient.ondatastream

Anyone knows how to set the interval, or how the built-in interval works, for HTTPClient.ondatastream?

I'm downloading a file and for this example, I'd like to get a progress update (to set a ProgressBar) every 1 second.

Anyone knows how to do it?

    Hi

    It is automatic, it fires when it is ready as it works out the best throughput of data and then updates you on the current progress. You are not able to change it.

    Trying to get it update every second is counter productive even if you could get it to work as you would have situations where it would fire back the same value for several fires as nothing new was ready to report.

    • Not sure why you do not trust httpClient. I am guessing it does return the data you ask of it?

      As the intervals are determined based on a multitude if factors, including connection type, connection speed, errors in transmission count, total amount of data being transmitted, available resources on the device, local storage for any caching and more - it then works out the best times to throw out a progress update. The intervals can increase and decrease during a single request as these factors change during a single request.

      I would and do trust it, if I did not then I would have to consider all the other parts of Titanium suspect and at that point give up coding altogether.

      I have have created many successful apps (slap on own back) and all use httpClient in some way, all work as expected and user feedback is handled using the progress callback.
