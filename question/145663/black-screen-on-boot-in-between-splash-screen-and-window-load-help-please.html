  • Hi

    Is this problem simply that you are loading all the UI and then eventually showing the window once everything is loaded, so your user has to wait until that completes before they see anything?

    You would be better served by creating a simple initial window show it, add an activity indicator if things really do take a while, adding items behind the scenes. Especially those elements that are not in the visual portion of the screen straight away.

    Plus I notice all your images end "…@2x.jpg" or "…@2x.png". You realise you are meant to supply two images; one standard res and the other retina res (ending @2x), but you only reference the simple file name as iOS handles the actual selection for you;

    You supply example1.png and example1@2x.png but you ONLY reference "example1.png" as iOS will automatically selection the @2x file and ignore the other when on a retina screen.
