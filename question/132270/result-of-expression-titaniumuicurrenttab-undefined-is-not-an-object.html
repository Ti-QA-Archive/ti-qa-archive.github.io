Result of expression 'Titanium.UI.currentTab' [undefined] is not an object.

I'm using the TabGroup template as a starting point for my application. I'm moving older code over and part of that code has me calling

Titanium.UI.currentTab.open(win,{animated:true});

This is resulting in an error of Result of expression 'Titanium.UI.currentTab' [undefined] is not an object.

Here's the code block

tbl.addEventListener('click',function(e) {
        var selectedRowId = e.row.children[0].text;
        var win = Titanium.UI.createWindow({
            url:'/ui/detail.js',
            title:"Detail",
            backgroundColor:'#b2dde3',
            infoId: selectedRowId
        });
        Titanium.UI.currentTab.open(win,{animated:true});
    });

In my app.js, I create the tabgroup like so:

var AppTabGroup = require('ui/AppTabGroup');
new AppTabGroup().open();

Here is the AppTabGroup code (from the template)

function AppTabGroup() {
    //declare module dependencies
    var AppWindow = require('ui/AppWindow');
    var InfoWindow = require('ui/info');

    //create module instance
    var self = Ti.UI.createTabGroup();

    //create app tabs
    var win1 = new InfoWindow(),
        win2 = new AppWindow(L('settings'));

    var tab1 = Ti.UI.createTab({
        title: L('Info'),
        icon: '/images/light.png',
        window: win1
    });
    win1.containingTab = tab1;

    var tab2 = Ti.UI.createTab({
        title: L('settings'),
        icon: '/images/KS_nav_views.png',
        window: win2
    });
    win2.containingTab = tab2;

    self.addTab(tab1);
    self.addTab(tab2);

    return self;
};

module.exports = AppTabGroup;

What should I be using instead of 

Titanium.UI.currentTab.open(win,{animated:true});

?

Thank you!

    var tabGroup = New AppTabGroup();

tabGroup.currentTab.open(...);

  • Surely it applies the same than for UI.Window.

    http://developer.appcelerator.com/apidoc/mobile/latest/Titanium.UI.Window-object

    > Sub-contexts

    > Windows can be loaded from another JavaScript file by specifying the property url, referencing a file relative to your application Resources folder. It's important to note that Titanium will refuse to load JavaScript files from a remote URL. Loading remote JavaScript from a URL and providing it with the full capabilities of the Titanium API would be very dangerous.

    > When your Window is loaded from a separate JavaScript file, the code will be executed in a separate JavaScript context (called a "sub-context") than your app.js global context. It will also execute in its own separate thread.

    > On Android, a new context is also created when you create a heavyweight window. See the section, "Android Heavyweight and Lightweight Windows" for more information.

    > The special property Titanium.UI.currentWindow is available inside a sub-context that points to the JavaScript instance by reference in the global context.

    So I think you just can use currentTab only when its window is defined by url. Otherwise you have to use the variable which it is set.
