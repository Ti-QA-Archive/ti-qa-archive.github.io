OAuth: 401 Unauthorized

###Hello,
I am trying to use the ACS API with "three-legged" OAuth. I am evaluating the API in a desktop environment, using cURL.

I have successfully obtained an access token, but when I try to use this token to make an API call, it returns a 401 Unauthorized error.
The response I get:

{
   "meta":{
      "status":"fail",
      "code":401,
      "message":"401 Unauthorized request. Please provide a valid App key or Oauth key."
   }
}

I couldn't find any examples on how oauth tokens should be passed, all examples show the URL parameter "key=<YOUR APP APP KEY>".

Making a request using my API key works fine. I would like to use OAuth instead.
I tried the following headers:

"Authorization: OAuth <token>",
"Authorization: Bearer <token>",
"Authorization: OAuth oauth_token=<token>"
"Authorization: <token>"

I've also tried the following URL parameters:

"access_token=<token>",
"oauth_token=<token>",
"key=<token>"

all with the same result.

Please tell me how am I supposed to authorize API calls with an oauth token.

Thank you.
