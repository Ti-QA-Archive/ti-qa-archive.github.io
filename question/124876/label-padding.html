    • Just a short note if you're targeting Android also. You shouldn't use opposite position properties together because then width/height property values are not respected.

      For example … let say that there is a label is in container view that has 100px height. If you define label position properties top: 10px, bottom: 10px … and height property 20px, it's defined height property (20px) won't be respected and label will be stretched and will have 100 - 10 - 10 = 80px height.

      Maybe iOS have similar issue … try to remove one of opposite position properties to see if there are some conflicts introduced when you use opposite position properties at the same time.

  • I used this code to have the label with padding to the left and right of the text.

    var headerLabel = Ti.UI.createLabel({
    top : 5,
    width: 'auto',
    height: 40,
    backgroundImage: 'images/labelbackground.png',
    text: "My test title",
    textAlign: 'center',
    backgroundPaddingLeft: 30,
    backgroundPaddingRight: 30
});
