embedded YouTube in WebView / remote source

We show a remote website in a webview in an android App.
Sometimes there are embedded YouTube videos on this site. Those videos are not working properly. The preview works fine but they play only sound, no video.
Also the controls for the video, once started, look wrong.
I also testet to show a simple link to bring up the mobile youtube website. This works, but the link opens youtube in a webview and also there the videos do not work.

For me it seems like a problem with the webview.

I'm testing this on a Sony Xperia ZL / Android 4.1.2
Titanium SDK: 3.1.0.GA

We use the embedded-iframe code

<div class="responsive-video">
<iframe width="305" height="240" src="http://www.youtube.com/embed/###URL###?rel=0" frameborder="0" allowfullscreen></iframe>
</div>

It works just fine in a desktop browser and on the android chrome browser.

    Hi, Have you tried using PluginState property of a web view? The Android web view supports native plugins such as Flash Player. To use plugin content, you must set the pluginState property to either WEBVIEW_PLUGINS_ON .
    To enable hardware acceleration, add the tool-api-level and manifest elements shown below inside the android element in your tiapp.xml file like:

    <android xmlns:android="http://schemas.android.com/apk/res/android">
    <tool-api-level>11</tool-api-level>
    <manifest>
        <application android:hardwareAccelerated="true"/>
    </manifest>
</android>
