HTTPClient response only return empty values

I'm running into an interesting problem.

I've got this app working nicely on iPhone. However, when I try to run it on android bring it over to android I'm not receiving any values when I do my HTTPClient Call.

Here is my connection code:

if ((Titanium.Network.networkType !=Titanium.Network.NETWORK_NONE) && valid ){ 
    var server = Ti.App.MyData[2] + "CheckUserServlet";
    xhr.open('POST',server);

    xhr.setRequestHeader("Content-Type", "application/x-www-form-urlencoded");
    xhr.setRequestHeader('User-Agent','');

    xhr.send(data); 
}

This part seems to work great. I've run traces and it looks good.

xhr.onload = function()
{
    Ti.API.info("Xhr status code is: " + xhr.status);
    Ti.API.info("text: -" + xhr.responseText+"-");
    Ti.API.info("Data: -"+ xhr.responseData+"-");
}

The status returns 200, so that's right. But my responseText and responseData are empty values.

Now, this code works like a dream on iPhone, and KitchenSink works for its image calls and the like, although they don't have additional info in send().

Other info: Titanium 1.2.1, Titanium SDK 1.4.0 (also on 1.2.0). JDK 1.6.22, tried on Android SDK 2.2 and 2.1 (not worrying about 2.3, rumor has it that its too new and buggy)

Can anyone see what I'm missing?
