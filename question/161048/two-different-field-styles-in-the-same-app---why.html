Two different field styles in the same app - why?

I've got two different windows in the same app which both have text fields. For some bizarre reason, each window's text fields have a different default style than the other. One shows the more recent Android style which are black background with the bracket/lines along the bottom. The other are just plain white rectangular field boxes.

And, yet, I'm not explicitly applying colors to the fields are either window. What gives? :-(

  • First, my response assumes this is an Alloy app.

    Styles can overwrite each other. Check the app.tss, if you have one, and the .tss for the files containing the fields. See if they all match. If so, and you still have this issue, try setting the properties explicitly in the app.tss. Those properties propogate to all like-named elements. Something like this example:

    "TextField":{
    left: 20,
    right: 20,
    top: 20,
    backgroundColor: "#fff",
    borderColor: "#000",
    borderWidth: "1dp"
}

      var textField = Ti.UI.createTextField({
  borderStyle: Ti.UI.INPUT_BORDERSTYLE_ROUNDED,
  color: '#336699',
  top: 10, left: 10,
  width: 250, height: 60
});
