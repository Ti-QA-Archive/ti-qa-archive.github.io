Ti.Filesystem and Android

Hello,
I have a problem to create a file in Android. The same code works on iPhone.

var newDir = Titanium.Filesystem.getFile(Titanium.Filesystem.applicationDataDirectory,'html');
newDir.createDirectory();

var fileErreurs = Titanium.Filesystem.getFile(newDir.nativePath,'erreurs.html');
fileErreurs.createFile();

When I start the application, i have this message :

Cannot find function createFile in object Ti.Filesystem.

Or in the documentation, this function is ok for android and iPhone.

Can you help me please ?

Thanks

  • Clément

    Although the createFile() method is documented in Titanium.Filesystem.File, no code exists for it in the Titanium Android SDK source, at TiBaseFile.java. Furthermore, no demonstration of its usage exists in filesystem.js.

    To create an empty file you can simply pass an empty string to the write() method. Hence, in your case, use the following:

    fileErreurs.write("");

    Note that if you don't pass the empty string, the file will not be created. Personally, I think write() should be sufficient.

    Alternatively, simply pass a string or an object:

    fileErreurs.write("This is a test");

    I will raise tickets for these two issues, and report back with the numbers when I have them. One possible solution would be to simply remove the createFile method from the documentation, and to make the write() method a little more user-friendly.

    Thanks for raising the issue. It's good to get these little quirks sorted! ;)

    Cheers

  • Using getFile is the preferred method. It allows you to get a reference to a filesystem object that may or may not exist. Once you have that reference, you can then do operations on it. If we do expose createFile it would have the same semantics as getFile, that is, returns a reference object to an object on the filesystem that may or may not exist but you can perform operations on it.
