Possible to toImage a scrollview that has content off screen?

I have a form with a lot of fields that extend well below the ipad screen. I was wondering if it is at all possible to call the toImage method on the scrollview and get the entire form, not just what is showing on screen? I tried it recently and only got what was onscreen, but was hoping there may be another way or an SDK hack?

I need to produce a pdf with pdf.js so its either this way or rebuild out the entire form again which would be a real pain in pdf.js.

Thanks

  • So here is what I ended up doing in case anyone runs into the same issue:

    Each section of my form was wrapped in a view, so I looped through all the children of the scrollview and toImage() them and save them to the data directory with an index ID and jpg file extension (0.jpg,1.jpg, etc).

    I then wanted to loop through and add them to a pdf so I used the modified jsPDF script from here:
    https://github.com/Core-13/jsPDF-image-support

    The problem I found is the toImage function does not actually encode the file as a jpg, just adds that file extension. So I grabbed the module com.sideshowcoder.jpgcompressor from here https://github.com/sideshowcoder/jpgcompressor which not only encoded my images correctly but compressed them also. I was then able to loop through each image and add them to the pdf!

    Easy right?
