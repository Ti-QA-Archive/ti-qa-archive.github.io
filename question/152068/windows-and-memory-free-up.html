Windows and memory free up

Hi, in my iPhone app I have a RootWindow.js containing the following code (any button in RootWindow.js opens a new Window with a transition, then, once the Window is open I can close it from the opened window itself):

// I omit repeated code but from my "home" I open 7 windows this way

      // one of my buttons
    var w1Btn = Ti.UI.createButton({
        top:'72%',
        left:'60%',
        width:123,
        height:86        
    });


    // my 7 windows
    var w1 = require('ui/handheld/w1');
    var w2 = require('ui/handheld/w2');
    var w3 = require('ui/handheld/w3');
    var w4 = require('ui/handheld/w4');
    var w5 = require("ui/handheld/w5");
    var w6 = require("ui/handheld/w6");
    var w7 = require("ui/handheld/w7");

    var a,b,c,d;

    // one of my buttons listener
    w1Btn.addEventListener("click", function(e) {
          if(a)a=null;
          a = new w1();
          a.open({transition:Ti.UI.iPhone.AnimationStyle.CURL_UP});
    });

In w1 commonJS module I have:

function w1() {

    var w= Ti.UI.createWindow({backgroundColor:'#ffcc00'});
    var label = Ti.UI.createLabel({text:"BACK!",top:0,zIndex:9,backgroundColor:'#ffffff',height:43});

    label.addEventListener("click", function() {
    var t = Ti.UI.iPhone.AnimationStyle.CURL_DOWN;
    w.close({transition:t});
    });

    w.add(label);

    return w;
    }

module.exports = w1;

Now, I want do to the following (but I don't know the right way to achieve it): anytime I close a window (and I reach my home page with my 7-buttons-menu) I want to free up the memory previously taken by that window. Is the way I'm doing the right way? Aren't there thing I can modify to improve the memory management?

Thanks in advance.

  • I think I understand what your asking.

    Firstly when you close a window the memory is cleared up at the next run of the garbage collection.

    But this is not always successful, so null is your friend here.

    If you are opening your window from the top level and also closing it there I would put my close event listener there. But you have two options …

    In your window module you can add a null of the window variable …i.e.

    //var t = Ti.UI.iPhone.AnimationStyle.CURL_DOWN;
    w.close({
        transition:  Ti.UI.iPhone.AnimationStyle.CURL_DOWN});
    });
    w.addEventListener('close', function(e) { w = null; });

    or you can add the close event on the parent object, which will null out a …

    w1Btn.addEventListener("click", function(e) {
          if(a)a=null;
          a = new w1();
          a.open({transition:Ti.UI.iPhone.AnimationStyle.CURL_UP});
           a.addEventListener('close', function(e) { a = null; });
    });

    If you are closing the window in the window module, then use the first example and after it is closed null out a.

    If you are closing the window in the main module use the second example.

    Hope this helps

    • When you null it out on close all the eventlisteners which are on objects of the window are removed.

      Any application level listeners will still be there.

      They will be removed when you close a window and the garbage collection kicks in. null just makes it happen instantly.

  • Hello,

    as Kiwi mentioned, if your item is linked via a global object (i.e., if it's referenced by another var, it wont be removed). If you add global events, like app events the objects won't be removed, because the event listeners weren't removed).

    BTW, you can't force the garbage collector; and there is two garbage collectors: one at JS level (quite efficient, and fast to run), and the native one (cleaning only the native objects, which could take time to run).

    I did an example about how to remove EVERYTHING from a window, and recursively remove all items. You can get it here.
