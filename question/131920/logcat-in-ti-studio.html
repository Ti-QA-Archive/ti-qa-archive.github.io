LogCat in Ti Studio

As I've been developing Android modules for Titanium, I noticed that they have a logcat view, which is very nice, as it allows you to see all logging occurring on your Android device and filter the log to only contain entries related to your app.

This begs the question, where is this logcat view in Titanium Studio?

    DDMS is still the best tool for communicating with an actual Android Device.

    If you haven't already installed them, take a look at the Java Development Tools and then the Android Development Tools. This will allow you to open a view that has the DDMS utility in TiStudio. This will show you all the log activity on the emulator and/or a device.

    Even when using the emulator, I use DDMS over the built in console logging as in DDMS I can see how the rest of the emulator is reacting. Often a JAVA error will not show up in the console yet will in DDMS.

    I prefer the DDMS displayed in TiStudio, as I can open code in a pane and browse through. It really comes down to how you operate.

  • Without any modification, Titanium Studio allows you to view three logs in the console. The first is for the fastdev process. The second is for the Android emulator process, which is what you wanted: the full log from the "device" filtered so that it only includes messages from your app.

    The third log is output that you explicitly send to the console via Ti.API.log.

    If you want more customization over filtering (though most people won't need it) you can use Android's built-in ddms tool. Just execute $ANDROID_SDK/tools/ddms.
