Coverflow Bug in Kitchensink

I think I discovered a kitchensink bug while I was trying to implement the coverflow in my project.

If you follow these steps you'll see what I mean:

  1. see the coverflow local files example.
  2. scroll all the way until the last image.
  3. Click any option in the tab group, like controls, phone, platform or mashups.
  4. click back the tab Base UI.
  5. See that the coverflow images get pushed to the side and the last focused option is not focused any more.

When you're trying to use less images like 7 or 8, it looks even worst.

Has anyone seen this and found a solution?
