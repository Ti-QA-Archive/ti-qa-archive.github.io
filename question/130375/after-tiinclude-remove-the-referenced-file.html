After Ti.include, remove the referenced file

Is there a way to remove a previously included file if it is no longer needed?

ex.
Ti.include("file.js");

Ti.remove("file.js"); // Is there a way to do this?

    Do you mean you want to delete the file from the filesystem?

    Ti.include just executes code. You can't then "remove" that code from your app. There's no reason to do so anyway. In and of itself, including a file doesn't use a significant amount of memory or anything. Separating code into files should be a way to organize code, not manage memory.

    If your included file creates an object or some views that you want to remove, that's a much different topic. In JavaScript, object variables are just references. If you set all the references to a particular object to null, you'll free some memory.

    With Titanium, objects (such as Ti.UI.View) should be reused if possible. If not possible, I believe the current way to free their memory is to add them to a new window and then close that window. I reuse all my Titanium objects, though, so I can't help you with that.
