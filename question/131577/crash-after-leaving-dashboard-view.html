Crash after leaving Dashboard View

I have a navigation group that goes down 2 levels. In the last level is a Dashboard view with 5 icons

var data = [];
var labels = [
    {label:'Video',icon:'video'},
    {label:'Case Study',icon:'case_study'},
    {label:'Slides',icon:'slides'},
    {label:'One Sheet',icon:'1_sheet'},
    {label:'Vforum',icon:'vforum'}
];

for (var i = 0; i < labels.length; i++)
{
    var item = Ti.UI.createDashboardItem({
        image:'',
        label:labels[i].label
    });
    var container = Ti.UI.createView({
        height: 100,
        width: 150
    });
    container.add(Ti.UI.createImageView({
        image: '../images/' + labels[i].icon + '.png',
        width: 'auto',
        height: 'auto',
        top: 0
    }));
    container.add(Ti.UI.createLabel({
        text: labels[i].label,
        textAlign: 'center',
        color: '#8cc44d',
        font: {fontSize:12},
        height: 14,
        bottom: 10
    }));
    item.add(container);
    data.push(item);
}
dashboard.data = data;

When I hit the back button on the top left, my app crashes shortly after. I see nothing in the console or anything. Are there log files somewhere that can give me an inkling as to what is going on? I am building for iOS and am using the lastest SDK 1.8.1

    In the Run Configuration settings in Ti Studio for your simulator have you set the log level to trace?

    Also, you can use the Crash Reporter built into OSX. Just change its mode to Developer, and whenever there is a crash in the simulator, it will pop up a window showing the error and stack trace.

    http://en.wikipedia.org/wiki/CrashReporter(Mac_OS_X))
