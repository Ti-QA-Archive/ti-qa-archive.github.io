Is it possible to create a Titanium Module and use it in a native Android Application?

Hi Guys,
In our company we are developing an application for a customer and we use Titanium for this. Now, the customer needs we to develop an SDK that Native Android Application's Developers could use in theirs apps.

We don't want to develop this SDK in Native Android. So one question arrived:

Can we develop a Titanium Module and publish it in a way that Native Android Developers can use?

— asked May 30th 2014 by Charles Schaefer

  • Hi

    Short answer - No

    Longer answer;

    An Android Titanium module is created using the native Android SDK. It is bridged to Titanium so that you can access any exposed methods and properties you design in. These modules once created only work inside Titanium anyway.

    So if you are hoping to avoid using the native Android SDK then you will not be able to create your custom Android Titanium module in the first place.

    Whilst you can create CommonJS modules and compile them - they are also only accessible inside Titanium.

    I understand the need you have and I would welcome this feature myself, but it is unfortunately there is current is no way around this.
