Failure [INSTALL_FAILED_INVALID_APK]

  • Application type: mobile
  • Titanium SDK: Titanium SDK version: 1.8.2 (02/23/12 17:46 59b3a90)
  • Platform & version: android 2.1 and android 2.2.1
  • Devices: Velocity cruz t301 (android 2.2.1) and Pandigital White (android 2.1)
  • Host Operating System: Windows 7 (64 bit)
  • Titanium Studio: Titanium Studio, build: 1.0.9.201202141208

I have two different devices: cruz t301 and pandigital white. I was able to upload and debug hello world app from eclipse IDE (app was written in java, not java script). So I assume that devices are ok to be used as a target device and I have correct drivers installed on my OS. I also can debug apps in titanium using android emulator (kitchen sink and so on). But when I try upload something from Titatium to real device I get an error: 

[INFO] Installing application on device
...
[ERROR] Failed installing com.test:     pkg: /data/local/tmp/app.apk
Failure [INSTALL_FAILED_INVALID_APK].

I tried to use different titanium SDKs (1.5.2, 1.6.3, 1.7.6, 1.8.2, 2.0.0). I tried to change android SDKs (Android 2.1, Google Api Android 2.1, Android 2.2, Google Api Android 2.2). Nothing helped me.

Am I doing something wrong? Please, advice.

  • [ERROR] Failed installing com.test:

    1. Be sure you are using the latest Android SDK (regardless of your target level)
    2. Uninstall any previous apps that might have the same id (com.test) from the device

  • It's a shame to write but I hope it will help somebody. The problem was in selected Android Runtime setting in Build section of tiapp.xml file. Device started to work once I changed V8 to Rhino. The reason I couldn't solve that for so long was in the difference of IDE I used to work with (MS Visual Studio) and Titanium. MS VS automatically saves all project files every time you run the project. But Titanium doesn't. You have to click save button and only then changes will be applied. So I was simply playing with settings but not saving them.
