Alloy Live Code Updating

One of the things I love about titanium is I can edit code live. Edit a js file and then navigate to that section of the app and my changes have taken place without a recompile. I think this is a real time saver.

It seems with Alloy you can't do this because the code is being generated. You can of course modify the generated code but that is not as powerful.

Am I missing something? Is there a way to do live code updates without a recompile?

If not is this on the road map?

Thanks.

  • Hi

    I suspect in the past you were using the old method of linking JavaScript files using the window url property. This itself soon lost favour due the requirement of a new JS context for every single window - this was not ideal for many reasons.

    The live updating you experienced was not so much live but a side-effect (in your case useful one) of this multi-context JS situation.

    CommonJS was and is the favoured approach for creating the modular coding methods that are employed with or without Alloy. Whilst you may not get the "live updating" you are used to - the CommonJS approach - when implemented correctly gives back much much more.

    CommonJS modularisation of code means that you (should be) create small single subject files of related code. You pick once section of your app and create a single CommonJS module to handle that single thing. From that point onwards each time you need that "thing" you reference the CommonJS module you created. This way you get many benefits - one of the biggest is that you focus on creating a solid re-usable module that is not dependant on anything that cannot be passed to it.

    Once you have perfected your library of CommonJS modules - you link them together using whatever code framework that suits you best.

    Later on you may refine your CommonJS module to; improve speed, increase error handling, better memory management, more features. But the interaction with your app should be the same or so close on minor tweaks are required.

    When you move between projects you can take your re-usable modules to the next project, with all that work done - your next app only need to create the framework and any new modules.

    Whilst "live updating" may have given you a sense of speed, it may (often does) have given you a false sense of speed as you will have spent a lot of time tweaking and checking visually - rather than working through the code, running it through in your mind - planning before you start to determine how it will link together. You will find the pre-planning, focused functions, careful use of variables and then finally testing will actually improve your speed.

    I do understand that learning a new MVC like Alloy means it would be better if you could quickly jump in and out but there is so much going on under the hood when converting the Alloy work into the live code - it is just not possible right now.
