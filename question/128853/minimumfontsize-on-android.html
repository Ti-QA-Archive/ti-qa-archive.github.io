'minimumFontSize' on Android?

Is 'minimumFontSize' supported on Android yet, or is there another way of doing the same thing for labels?

My current label code which works for iphone but only ellipsizes on Android:


        var location = Ti.UI.createLabel({
            top: 44,
            height: 64,
            left: 20,
            right: 20,
            color: '#ced3d8',
            text: BT.venue.name,
            font: {
                fontSize: 24,
                fontFamily: (BT.__isAndroid) ? 'Museo700' : 'Museo'
            },
            textAlign: 'center',
            shadowColor: '#000',
            shadowOffset: {x:0,y:1},
            minimumFontSize: 14,
            wordWrap: false,
            ellipsize: true
        });

  • Hi Daniel

    To make the font scale properly on small emulator and big emulator use the metric "sp".

    fontSize: '24sp'; there is also one another metric call "dp" so check which works for you.

    I think this can solve the problem what you looking for.

  • Actually minimumFontSize is best used when you don't know the length of the string in a restricted area where it should be displayed. So minimumFontSize automatically reduces the size of the font to hit the area specified for it. To date: no solution on Android.
