SDK 3.0 Error on iPad device with Ti.include

Dear reader,

I have noticed some strange bug while testing my app on iPad 2 (iOS 6.0.1) device.

/* Render view
 * @param properties
 */
exports.render = function(view, window, properties)
{

    if(require("includes/utils").isTablet())
        require("themes/" + require("includes/config").themeName + "/tablet/" + view).render(window, properties);
    else
        require("themes/" + require("includes/config").themeName + "/" + view).render(window, properties);
}

It will show this error while running the function above:

Can't find module themes/default/tablet/items/list

The funny thing is, this works perfectly on the iPad simulator and all other devices (android, android tablet, iphone) I have tested it on running SDK 3.0. I tried changing the path by adding a slash at the beginning, but the same error occurs.

When I run my app with SDK 2.1.4, on iPad it works.
I work on Mac 10.7.5.

    Hi Stefan,

    as there where JavaScript errors inside your code (missing {} for if and else) and SDK 3.0 is a bit problematic with that, try this code.

    /* Render view
 * @param properties
 */
exports.render = function(view, window, properties) {

    if (require("includes/utils").isTablet()) {
        require("themes/" + require("includes/config").themeName + "/tablet/" + view).render(window, properties);
    } else {
        require("themes/" + require("includes/config").themeName + "/" + view).render(window, properties);
    }
}

    Greetings,

    Matthias
