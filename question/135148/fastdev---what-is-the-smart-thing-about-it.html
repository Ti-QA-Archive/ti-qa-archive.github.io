FastDev - what is the smart thing about it?

Hi!

I've asked earlier about fastdev, but I'll try again.

On initial compile, fastdev spends approx. 2½ sec. pr. file it fetches to the emulator. This is subject to vodoo, 'cause some days it only uses a fraction of a second. I have no clue why some days it's slow and then suddenly it's fast, but I know that it's slow most days.

Is there anything I can do about this?

The command 'titanium fastdev restart-app' is supposed to restart the app which it does, however fastdev does NOT fetch the codechanges that you've made (at least not with 'require' files) - so even though you get a 'fast' restart, it's worth almost nothing, because in order to check code-changes you'll have to do a full recompile.

Can I do anything to make fastdev actually fetch the changed files?

  • Hi Esben,

    the only way is to stop fast dev process

    set this line in your tiapp.xml and clean build the project

    <property name="ti.android.fastdev" type="bool">false</property>
