getArguments() returns empty string on Android

Hi,

I am trying to retrieve the custom url that was used to invoke my app by using Titanium.App.getArguments(). The following code is supposed to do that, and it works fine in iOS, but not on Android:

var args = Titanium.App.getArguments();
Titanium.API.debug('args is: "' + args + '"');
Titanium.API.debug('stringified args is: "' + JSON.stringify(args) + '"');

In Android, the output shows args as an empty string "", and stringified version of it as blank "[]". In iOS, there are no issues, the data is properly retrieved for use.

On both platforms, the app invokes as desired using the custom url by clicking a link in the web browser, which means that the url should be accessible in Android through the arguments.

Many thanks in advance!

J
