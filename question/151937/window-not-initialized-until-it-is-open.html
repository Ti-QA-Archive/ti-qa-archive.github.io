Window Not initialized until it is open?

I found an interesting thing: The Ti.UI.window might not be initialized upon creation, until it was open.

I show my code here to see if anybody is kind to comment.

function RankWindow() {  
  var mainWnd = Ti.UI.createWindow(...);  
  ...  
  var datePicker = Titanium.UI.createPicker(...);  
  mainWnd.add(datePicker);  
  ...  
  return mainWnd;  
}  

...  

module.exports = RankWindow;  
...
var RankWindow = require('/RankWindow');  
var rankWnd = new RankWindow();
rankWnd.open();

As you have seen, I have a time-consuming operation in the constructor of RankWindow: Titanium.UI.createPicker ?I think because DatePicker is the most complicated UI control).

I thought the DatePicker would be created when I new the RankWindow object, and the window would have popup at once when rankWnd.open.

However, I observed that rankWnd.open took much longer than expected unless I remove the coe of createPicker. All seemed that my RankWindow was not initialized until it was open, similar to some mechanism of 'Lazy Loading' or alike.

This behavior confused me for days since it did not match my intuition at all.

Would you please tell me why?

thanks

  • Exactly - Lazy loading. This has always been the behavior since i've been using AppC. (since v 1.6) It makes sense to defer the creation of objects until necessary to keep the memory footprint as low as possible.
