    Hi

    You do not attach as a global - this is bad practice.

    Instead reference as a CommonJS module;

    // assumes moment.js is in {appname}/resources/moment.js
var Moment = require('moment'); 
Ti.API.info(moment(new Data(), 'YYYY-MM-DD HH:mm:ss'));

    • Attaching something to the global object can easily become the default and suddenly everything has to be included at the start- forcing the app to have way too much memory to simply function.

      Using a CommonJS approach means your modular code is only called into scope when first required, so if it is not used then the app never loads into memory for that app session. This is not the only benefit though. The documentation link will explain more.

      This is not for memory leaks so much as they are when you define something and it fails to release automatically when you no longer need it. This is more about bringing the winter clothes on a summer holiday "just in case".

      I understand your comment about String and JSON and agree/disagree at the same time. A better example would be the new Android Maps v2, this has now moved to a module that is ONLY referenced if actually called - thus saving memory. I think you will find many new features will be modules from now on; NFC and Facebook are two other examples of this.
