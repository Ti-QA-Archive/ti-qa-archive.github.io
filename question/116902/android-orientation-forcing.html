Android orientation forcing

Is this still the "best" way to force a specific orientation to an Android app? http://developer.appcelerator.com/question/40001/android-force-orientation

It seems a bit messy and a PITA. Is there an easier, more current way to address this?

  • In my current app using the 1.6.0 release, I still have to use the custom manifest file. Setting it within the app with orientationModes:[Ti.UI.PORTRAIT] has not worked for me. Also, AFAIK there's no way to set this within the tiapp.xml…or I should say I opened a ticket asking for the correct syntax for this and was told to do it in the Android manifest instead.

    Messy and PITA, but it works. Then again, to maintain version numbers the way I want, I've had to fiddle with a custom manifest every time anyway. I'm in that file anyway so perhaps not that big a deal.
