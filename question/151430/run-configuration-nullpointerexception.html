  • Thanks for your reply. In the end I found out that the manifest file was causing this error.

    When I arrived at work this morning I decided to check all my files again. During this check I saw that the contents of the manifest were completely different compared to before the update. Luckily I had an old copy of the manifest in my svn repository. The run configuration started working again after restoring the lines that had disappeared after the update.

    I updated my Titanium Studio to include the Ti.Map module for the android part of the app, so after updating I immediately added this module. So im not 100% certain if the Titanium Studio update is the cause of the manifest change or the module, but I think it was the implementation of the module.
