What's the best practice to test online/offline scenarios with Android apps?

I would like to test my implementation of offline mode, that is scenarios like being offline when the app starts or changes in connectivity status while the app is running.

Titanium provides Ti.Network.online and a change event. All nice and easy to test with the iOS simulator. However, I am wondering how to test this with Android.

  • cutting the connection of my dev box has no effect. The emulator pretends that Ti.Network.online = true
  • turning on airplane mode makes the emulator unusable with Titanium.
  • F8 to toggle internet connection in the emulator disconnects the emulator from Titanium and stops the debug session
  • using Ti.API.info and running in the emulator instead of debugging also stops logging after disconnecting via F8

What's the best practice to test scenarios where the device is online?

Am I down to the only options of putting alerts in my code?

Thanks for any pointers.

  • Hello,

    you should test at least this scenarios (in general):

    1. Device connected to a wifi
    2. Device on Edge Network
    3. Device on 3G network
    4. Device on 4G network
    5. Device in airplane mode.
    6. Device connected to a wifi without actual access to the internet (check your timeouts).

    Best,

    Mauro
