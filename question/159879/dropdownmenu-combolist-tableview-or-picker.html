  • Hi

    Not sure if you still have the problem?

    If so then here goes;

    The closest UI element for Android would be a optionDialog to show the list of choices; Mangos, Bananas, Apple and the use of buttonNames to show the; New Fruit & Cancel options.

    This way the UI is defined clearly that the choice is one of these or the action of "New Fruit" being distinct and always visible even if your list exceeds the visible display space.

    Something like this as a test;

    var dialog = Ti.UI.createOptionDialog({
  options: [
    'Mangos',
    'Bananas',
    'Apple'
  ],
  buttonNames: [
    'Create',
    'Cancel'
  ]
});
dialog.addEventListener('click', function (e) {
    Ti.API.info('button: ' + e.button); // button pressed
    Ti.API.info('index: ' + e.index); // list entry pressed
});
dialog.show();

    I hope this was of some help.
