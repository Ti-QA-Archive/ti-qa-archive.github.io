  • I found workaround with this bug - probably it's not a perfect solution but it's work.

    1) Delete your project from Titaium
    2) When it ask about remove files or no - select leave
    3) Save your Resources folder somewhere else (if you want you also can save tiapp.xml & manifest)
    4) Remove old project directory
    5) Create new project in TItanium
    6) Remove all default content from Resources folder
    7) Move your old content to Resources
    8) Press "Start in simulator"
    9) Wait
    10) Enjoy
    11) Get a coffee ;)

  • I can cause this problem reliably just by using "Titanium" in the name of my project.

    I found my clue by looking at the log file in build&#x2F;iphone&#x2F;build&#x2F;build.log which showed me the output of the failed build.

    In my case, it mentioned Unknown target: Hello2Titanium and it was true – the build target names had been mangled! I guess it's a bug in the iPhone project generation.

    I created a new project without the string "Titanium" in the name and everything is fine now.

  • I have a work around but it's a hack. After upgrading everything to the latest versions (as noted in comments below) a Run on Device and produces a new traceback error: mobilesdk/osx/1.5.1/iphone/builder.py line 1143, in main… builder.py, line 925, in execute_code… run.py, line 31, in run sys.exit(rc) SystemExit:1.

    Open the generated myproject/build/iphone/myproject.xcodeproj xcode project. Note, if you do, Build | Build and Archive, at this point you will get 'Command /bin/sh failed with exit code 2' error with no other warnings or errors and a non-functional myproject.app file.

    Click the Info button. Under General tab, change 'Base SDK for All Configurations' from blank to Latest iOS. Do a Build | Build and Archive. Ignore that you get the same error message as noted above and drag the myproject/build/iphone/build/Release-iphoneos/myproject.app to iTunes. iTunes Sync. Now the myproject.app runs on device.
