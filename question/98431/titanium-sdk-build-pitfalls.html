Titanium SDK build pitfalls

This is rather a tip than a question - I decided to share it as it costs me some hours:

I'm working with several SDK version, 1.5.0, 1.5.1, 1.6.0 and so on.
I ran into trouble, that my changes to the sdk where not reflected after rebuild.
Pro Tip: Never ever work in renamed directories!
I had a directory layout like this:

git/titanium_mobile <- this is the git clone of titanium

git/titanium_mobile-patched <- my own clone

Same for Library/Application Support/Titanium/mobilesdk/osx:
I had various SDK dirs like:

1.5.1
1.5.0
1.5.1-org
1.6.0

Titanium uses fixed directory names in a lot cases!
I.e. the scons build works on your current directory but later works on files out of the default directory (titanium_mobile).
