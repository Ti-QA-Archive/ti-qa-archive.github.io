Android images doesn't load, and app restarts

I have developed an app for iPhone and are now trying to make it work on Android as well. The first window in the app is the login screen. On this screen, all of the images (a background image and a logo) is rendered as supposed. However, when i login a new window (modal) is opened and then everything starts to go wrong. I can see that it tries to pull images from the path alloy&#x2F;controllers&#x2F;myimage.png, but the image is stored in assets&#x2F;android&#x2F;myimage.png. All my images is in this folder, and i am referencing the path to the images in the same way in my first window as in the other windows. Then the app is restarted instead of opening the window, and no error message is printed to the console.

I am referencing the images like this: backgroundImage:&quot;blackArrow.png&quot;, and then the image is stored in assets&#x2F;android&#x2F;blackArrow.png. I really can't see why Titanium is doing this to me…

SDK version is 3.1.1 and the project is an Alloy project. I have tried cleaning, and removing the build and resources folder without any luck..

  • Accepted Answer

    Prefix all your image paths with a &#x2F;. That will make the pathing work the same on iOS and Android. So instead of something like image: &#39;someimage.png&#39; you would do image: &#39;&#x2F;someimage.png&#39;.
