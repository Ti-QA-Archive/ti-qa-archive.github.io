Ti.File.exists() slow performance on Android

I just noticed that calling Ti.File.exists() on a file that is missing is quite costly in Android (on Galaxy S4).
The folder contains only one file, and I am trying Ti.Filesystem.getFile(&#39;wrongfile&#39;).exists(). It takes 500-600 ms for this call to complete. In comparison Ti.Filesystem.getFile(&#39;wrongfile&#39;).getSize() returns 0 immediately.

I am using SDK 3.1.3.

I've solved my problem, but it took me some time to nail it down, so i decided just to leave this as a note. Or perhaps to be taken for consideration if it has to be fixed in the SDK
