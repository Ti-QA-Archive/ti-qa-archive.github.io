ios app loses functionality in TestFlight

I have an app developed with Appcelerator, and the iOS version has been tested both in the emulator and on my iPod through a cable. My Titanium studio build is: 2.0.1.201204132053. My iPod Os version is 5.1.1. My xcode version is 4.2. My problem is: a crucial function in my app does not work when distributed through TestFlight. Basically, I press a button and nothing happens. I will instrument the code to see exactly what is going on, but i"m wondering if anyone has an idea what could cause this? It seems unlikely that it could be caused by the distribution process. Also, is there a log that I can get from the iPod that would help explain what is going on. Thanks.

    • I'm also using the TestFlight SDK from the Appcelerator Marketplace. It works great. It will take any errors or Ti.API.info() logs you make and store them in their web client.

    • May sound like 'me too' but the TestFlight module from the Marketplace has been a great investment for us. I have only occasionally noted a delay if the testflight servers are having an issue or the connectivity of the device is questionable.

      Testlflight in its most basic form is simply easing the process of ad hoc distribution. If that is all you are doing, then Testflight isn't actually adding any code to your project.

      If you are including the Testflight SDK which helps with monitoring, feedback, crash logging, then Testflight does have code in your app.

      Post some snippets of code relating to your button event in addition to the answers Simon has asked for.

    Exception Type: EXC_BAD_ACCESS (SIGBUS)
    Exception Codes: EXC_ARM_DA_ALIGN at 0x003fffff
