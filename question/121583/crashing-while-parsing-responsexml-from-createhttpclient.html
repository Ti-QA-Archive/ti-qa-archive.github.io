Crashing while parsing responseXML from createHTTPClient

I am trying to parse a Twitter XML document that gets returned from an httpClient request. I loop through up to maybe 5 or 10 requests in succession to the Twitter API.

I test for the existense of items in the document then start looping through them to get the tweet. After testing for "itemTweet &amp;&amp; itemTweet.length &gt; 0" the app will randomly crash when i try to access "itemTweet.item(0).text" even though itemTweet was just checked for a null value and to make sure it has at least 1 item.

In summary: what else can I do to make sure that "itemTweet.item(0).text" has a value so that I can avoid hard crashes in my application.

Here is some sample code:

var twitterdoc = responseXML.documentElement;
var items = twitterdoc.getElementsByTagName('status');
if (items && items.length > 0)
{
    //should be okay to continue parsing item.
    for(var i=0;i<items.length;i++)
    {
        //Lets get the tweet now...
        var itemTweet = item.getElementsByTagName("text");
        if (itemTweet && itemTweet.length > 0)
        {
        //App will crash on this call, but not every time...
            tweet = itemTweet.item(0).text;
        }
    }
}
