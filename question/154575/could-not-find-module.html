  • @Doc Jump I can't remember exactly what I did with the build file. But I've done it since and it worked first time round. Did you declare the module in your tiapp.xml? i.e. 

    <modules>
        <module version="2.1.0">ti.admob</module>
        </modules>

    Also another trick which was NEEDED (for Android), in the tiapp.xml was 

    <android xmlns:android="http://schemas.android.com/apk/res/android">
        <manifest android:versionCode="1" android:versionName="1.0"/>
        <uses-sdk android:maxSdkVersion="16" android:minSdkVersion="8" android:targetSdkVersion="16" />
        <tool-api-level>17</tool-api-level>
        </android>

    I would suggest you start a new project. When you are copying the module folder in, open up your 'project explorer' and paste it into the new project folder you created. The problem could be that your module folder isn't in the exact right place. Note it should not be in the 'resources' folder. After you paste the module folder. 'Clean' the project and run. If that doesn't help, I have no more advice to give. Good luck!
