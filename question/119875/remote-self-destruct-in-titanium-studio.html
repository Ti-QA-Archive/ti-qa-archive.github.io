Remote self-destruct in Titanium Studio?

The licence agreement for the new Titanium Studio 1.0 includes the following:

> 2. TIME-SENSITIVE SOFTWARE. The software will stop running on June 1, 2011. You may not receive any other notice. You will not be able to use the software when it stops running. Your license granted under this Agreement will terminate upon this date.

This deadline is less than two weeks away, but the developer blog says that Titanium Studio will have the status of "General Availability" in "Early June" but will apparently be split into "Standard" and "Premium" versions.

Does this mean that between 1 June 2011 and "Early June" I am not going to be able to access Titanium Studio or continue to develop my apps?

What exactly is going to happen to my current installation of Titanium Studio and my apps-in-progress on 1 June 2011?

Can you please confirm that if I use Titanium Studio to work on my apps now I am not going to have to suddenly pay a fee at some stage in order to finish them or any part of them?

We can trust you, Appcelerator… right?

  • Hi there,

    I always found strange that Titanium Studio doesn't work offline and after reading your post, I became a little worried, so I did a little research.

    I found this post on the developer's blog and I think I know what is going to happen.

    If you open the page and scroll all the way down to the title Titanium Studio, you can read all about the new changes to Titanium Studio.

    For summarize, Titanium Studio is going to be divided in two: the standard version, which is going to be available to everyone and the premium version which is going to be available to developers with paid subscriptions.

    I think that's why we need to login every time we want to use Titanium Studio. Because some features, like debugging, who are available to everyone today, are going to be exclusive for developers with paid plans, so the login is going to be a mechanism to filter developers with access to those features from the others (maybe!). But it really sucks having to login every time we want to use Titanium Studio.

    Hope it helps.

  • I certainly hope that Titanium Studio will work offline. I like to code on trains and planes, where connectivity is often non-existent.

    I'm still using Ti Developer and hope to be able to use it for some time.

    I'm not a big fan of the Studio IDE so far. (Which I've set up on a separate laptop to my main development laptop)

    There's a lot of things with the editor that I don't like, which I'm used to in other text editors. The code completion doesn't seem to always work. There seems to be a lot of overkill features. There are little niggling things too - like the folder icons in the side navigator being 'open', when the folder isn't open.

    I'm not saying it's not a very clever and useful product - it is (I used/tried Aptana for a bit, back when I started trying AIR and Nokia's WRT) but I actually like the simplicity of Titanium Developer.

    I seriously hope that we won't be 'forced' to use Studio (though, obviously support of Ti Dev will vanish at some point) to build and test apps. I guess that since we have command line methods to build and test stuff, there will always be a way. Recent updates and goings on with the very latest builds and KS demos have hinted that things won't work in Titanium Developer for much longer.

    This comment/response isn't to grow any 'FUD' at all, it's just my thoughts on these recent developments. I'm a huge and active supporter and admirer of the Appcelerator team, solutions, products and vision. The new product announcements the other day were incredible.

    Code strong, as they say! ;)
