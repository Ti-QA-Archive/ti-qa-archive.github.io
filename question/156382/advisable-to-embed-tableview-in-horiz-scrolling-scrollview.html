Advisable to embed TableView in horiz scrolling ScrollView?

  • Application type: mobile, iOS
  • Titanium SDK: 3.1.2 GA
  • Platform & version: iOS - iPhone and iPad - iOS version 5+
  • Device: iOS simulator and iPhone/iPad devices - both retina and non-retina, iOS version 5+, both orientations
  • Host Operating System: OSX 10.8.4

Referring to TableView guides, I see the following -
'Tables and Scroll Views
As a table view inherently scrolls, it creates a very poor user experience when one contains other scrolling views, such as a ScrollableView or TextArea. Thus, this layout is strongly discouraged.'

My requirement is that I need to have a vertically as well as horizontally scrolling table with rows and columns which supports both orientations. I understand from above that one should not have ScrollView / ScrollableView and TextArea nested within TableView - but is it okay to have a horizontal ScrollView and put a TableView / ListView (vertically scrolling) inside it so that i can achieve both horizontal and vertical scrolling?

    Hi

    As the documentation states it is discouraged, but that does not mean you should not do it.

    It actually means - because of the potential issue where the user interacts with one view and the other view responds at the same time making it difficult for the user to control either.

    If you can provide a user interaction that allows a simple sideways movement without an unsuitable vertical movement at the same time - then the user experience is fine. You are not told not to, just told if you do - make it work well as they are often bad.
