  • hi
    i have the same issue when i look for the solution i didnt find any so made one temporary solution by the below logic.

    We never knows what the size of the content which going to be displayed in the webview and so the scroll will always come if we give the height smaller then the content height.

    So i found out the way I calculate the size of the html page content which going to be displayed on the webview and set the height of the webview based on that. So as we match the content height and webview height it will not show the scroll bar.

    //As per content length, i am setting webview height so that it don't show scroll in webview content.

var fine_webview = Ti.UI.createWebView({
            html:this will be your data,
            top:0,
            left:5,
            height:135,
            textAlign:'left',
            backgroundColor:'#FFF2F7',
            width:Ti.Platform.displayCaps.platformWidth - 20,
            scalesPageToFit:true,
            touchEnabled:false,
            borderRadius:8,
            font:{fontSize:13}
        });
        fine_view.add(fine_webview);

        if(data.length < 195)
        {
            fine_view.height = 140;    // In "fine_view" i am adding "fine_webview".
            fine_webview.height = 135;
        }
        else if(data.length > 195)
        {
            fine_view.height += 20;
            fine_webview.height += 20;
        }

    I know its not what you looking for but this is currently the temporary way i worked out for my app i hope the same may you can apply for your app. If this seems good then do vote up.
