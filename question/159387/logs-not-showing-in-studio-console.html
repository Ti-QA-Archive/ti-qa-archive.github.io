    This is a known issue with 5.0.1

    Solution is to roll back to 5.0

    — answered November 12th 2013 by Stephen Feather
    permalink
    1 Comment

    • Specifically, up until 5.0.1, Apple has enforced a x.y versioning system, despite the rest of the world using an x.y.z. Was a major pain for those of us coming from other platforms that like major.minor.patch. Well, in 5.0.1, they pushed ios sdk 7.0.z and the TI scripts are looking for 7.y so not finding the log file on the simulator. You can dig down into the simulator file structure and tail the log file there if you want.

  • As suggested you can roll back or you can view the log with system log.

    "An alternative solutions is to view the simulator log directly:

    tail -n50 -f ~&#x2F;Library&#x2F;Logs&#x2F;iOS\ Simulator&#x2F;7.0.3&#x2F;system.log

    Obviously replace 7.0.3 with whatever version you are using.

    Also you can filter for just your app output using

    tail -n50 -f ~&#x2F;Library&#x2F;Logs&#x2F;iOS\ Simulator&#x2F;7.0.3&#x2F;system.log | grep AppName"

    Ref: http://developer.appcelerator.com/question/158781/mac-os-mavericks-titanium-console
