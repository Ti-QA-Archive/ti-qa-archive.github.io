Check username exists in Titanium Alloy

This is normally a easy task in other languages but can't seem to get anything to work in Titanium Alloy. How would I check if a user table already has a username. When a user registers I need away of determining if the username entered already exists via backbone models. Im not sure if I should obtain a collection first then check if the collection contains a specific username value??

Any help would be greatly appreciated, thanks…

  • As part of your server's API you'd need a sign-up (register) service, which takes care of checking if the registering username is already present, then either accept or reject the new user registration.

    Having a users collection fetched even before a new user has registered his/her account, just to be able to check for possible username pre-existence in the client's code, seems a bad approach to me.

    Also take care of not sending to the client any sensible data along with the users collection (if any), here I think about emails, password, addresses; include just the minimum required fields for the app's UI, i.e. id, username, profile-image-url …
