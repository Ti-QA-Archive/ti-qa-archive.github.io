different Application IDs for Android vs. iOS

I have an iOS provisioning profile for com.company-name.application. Unfortunately, the hyphen is invalid for Android, so my App ID for Android should be com.companyname.application. I tried using a custom Info.plist to override the App ID for iOS, but the Titanium SDK scripts use only the tiapp.xml setting.

Is there a way to set a different App ID for iOS vs Android that doesn't require me to modify tiapp.xml manually whenever I switch gears from one to the other?
