ajax doesn't work in iOS WebView

In my iOS app, a webpage in a webview uses ajax to implement a standard "read more" button. ajax requests additional contents dynamically. This works in Android, but in iOS XMLHttpRequest.status becomes 0 and ajax fails. I tried "crossDomain: true" in ajax but it did not help.

The JavaScript in the page is something like this.

<script type="text/javascript">
    var page = 1;
    $( "#read_more_txt" ).click( readMore );

    function readMore( e ) {
        page++;
        $.ajax( {
            type : "GET",
            url : "http://server's url/?page=" + page,
            cache : false,
            crossDomain: true
        } ).done( function( response ) {
            if ( typeof response.html !== 'undefined' ) {
                $( "#read_more_block" ).before( response.html );
            }
        } ).fail( function( XMLHttpRequest, textStatus, errorThrown ) {
            alert( XMLHttpRequest.status );
        } );
    }
</script>

The webview is an ordinary Titanium's webview, so I omit the code.
What is the most likely cause of this problem?
Thanks in advance.
