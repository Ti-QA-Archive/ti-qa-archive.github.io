iPad SplitWindow - Swapping out Windows in the DetailView?

I am trying to find the best way to implement the SplitWindow iPad control. I currently have a table view set as the masterView.

What I want to do is swap out the detailView with a new window or NavigationWindow each time the use selects a new tableview row.

What is the best way to structure my iPad app in order to accomplish this?

  • Nick,

    the only way I see is to create a SplitWindow setting a NavigationWindow as the detailView, and setting a blank window as the NavigationWindow.window and fill it with a View where you will place all your subviews.
    As you click on the tableView you change the View inside the window.

    SplitWindow[
    masterWindow[
        window[
            TableView
        ]
    ]
    detailWindow[
        NavgationWindow[
            window[
                View[
                    subviews
                ]
            ]
        ]
    ]
]

  • I struggled with that on my first iPad splitView app and finally came to a workable solution.

    http://developer.appcelerator.com/question/156348/trouble-rolling-back-multiple-steps-in-the-navgroup-stack-in-the-splitviews-detailview

    Note that this solution addresses the old navGroups. You will just have to replace them with the NavWindows (which I did with no trouble in my newest app versons). If your detailWndow hierarchy is one deep, you can reference the navWindow stack with a single variable; for a deeper hierarchy, an array will accomplish the same thing.
