Dynamic zindex Bug

So I've asked this on the IRC but couldn't get an anwser cause nobody knew why.

So i'm posting a project here which reproduces the problem (with comments for explanation)

In short: I'm changing the zIndex Value for a ImageView (or View for that matter) whenever I click on a thumbnail/button.

The view that changes doesn't get layer correctly but always above.
Then if you Debug the changed zIndex value, the Titanium Console.
gives me random numbers back that are ranged like these: 113304752

Here is the Project.

http://www.mediafire.com/?0n2y7j7zi68vccp

EDIT: As I was directed to the Lighthouse site of Appcelerator (first time I see this) to add a ticket as a bug
I noticed that the bug was already there

The workaround for now is instead of this line:

coloring.zIndex = bgbigs[e.source.backgr_name].depth;

Change it to:

coloring.animate({zIndex: bgbigs[e.source.backgr_name].depth});

This should fix it for now

It's good to see the Lighthouse app, It can help me even more with some stuff.

Sorry to bother and post it here, but maybe someone else can use this ^^
