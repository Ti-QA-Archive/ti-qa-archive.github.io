Changing image of an ImageView does not free memory

Hello,

platform : iOS5

I have an ImageView, clickin on the right, i want to change its image to the next image, clicking to the left, to the previous image, for browsing 50 images.

when I change the imageView image, the memory increases, increases, never decreases, with the new Titanium Mobile SDK 1.8.0.1

Everything was fine with 1.7.5

var win = Ti.UI.currentWindow;
var currentImage=0
var max = 50

var imView1 = Ti.UI.createImageView({
            image : 'img/large/IMG_'+currentImage+".jpg"
        });
win.add(imView1)

imView1.addEventListener('click', onclic)


 function onclic(e)
 {

     if(e.x>500)
     {
         currentImage++
        if (currentImage>max)currentImage=0
     }
     else
     {
         currentImage--;
         if (currentImage<0)currentImage=0
     }
     imView1.image= 'img/large/IMG_'+currentImage+".jpg"
    Ti.API.info("memory"+Titanium.Platform.availableMemory);
 }
