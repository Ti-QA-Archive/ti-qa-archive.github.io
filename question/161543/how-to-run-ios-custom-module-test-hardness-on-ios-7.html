How to run iOS custom module test hardness on iOS 7

I have a custom iOS module which I test by running "titanium.py run", which runs the test/app.js in iPhone Simulator. I use titanium.py because the new CLI doesn't support custom modules yet.

However, using the old titanium.py I can't figure out how to specify the iOS version/simulator, as I need to test it in iOS 7. The docs don't seem to show any way to set this, so my test harness always runs in the iOS 6 simulator.

http://docs.appcelerator.com/titanium/3.0/#!/guide/Legacy_Command-Line_Interface-section-29004835_LegacyCommand-LineInterface-Commands

Any ideas?

    Justin,

    I had to build a couple of shell scripts to do it.
    Have a module testing project in my default workspace.
    run a build.py each time.
    Copy the .zip file over, unzip it, then start the app using the new cli.

    It was nice being able to build up your app.js example as you worked through your module. Now I just build it up, then copy it over later if I'm going to push to github, or release.

    • for those wondering, the reason you don't do all this in the example directory, is that build.py will zip up everything, including your example/build directory, so its less time to move everything around.
