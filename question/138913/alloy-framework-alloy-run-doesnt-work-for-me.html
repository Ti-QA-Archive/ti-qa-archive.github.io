Alloy framework: "alloy run" doesn't work for me

I am trying out Alloy. I realize Alloy is in its early unstable stages, so I fully understand if there are glitches. Just trying to get started – create a project, compile it, and run it. The "run" part doesn't work for me.

I did this:

$ alloy new myproj
# (successful)
$ cd myproj
$ alloy compile
# (successful)
$ alloy run
2012-06-27 10:12:40 -- [DEBUG] /Users/mike/alloy/myproj/app does not contain a valid project

Is there something I'm doing wrong? I am up to sync at commit e241619 (Wed Jun 27 06:00:24 2012).

    Looks like the problem has something to do with the Titanium command-line tools which are invoked by Alloy – not with Alloy per se. The above command ends up invoking the following Python command, which is the one that actually reports the error:

    python /Library/Application\ Support/Titanium/mobilesdk/osx/2.0.2.GA/titanium.py run --dir=/Users/mike/myproj/app --platform=iphone
