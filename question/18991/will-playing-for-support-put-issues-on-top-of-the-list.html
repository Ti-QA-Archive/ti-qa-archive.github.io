  • Paul, after looking at that ticket I ran into the same problem when I upgraded from 1.0 to the 1.1 SDK (and then 1.1.1 and 1.1.2), also running on snow leopard. Hopefully what I did to fix it could help you in the interim:

    • First, even though I would get that "unknown or unsupported SDK version" error, the simulator would still pop up, but the app would fail to load. Clicking the home button on the sim and then clicking the app icon to manually start it seemed to have "located" the sdk, and on subsequent builds that error message was removed. It would come back if I changed the TI SDK I was using though.

    • Second, what fixed it permanently for me was trashing the 1.1.2 folder in Library/Application Support/Titanium/mobilesdk/osx and just using 1.0 for a while (which never gave me the problems). I didn't re-download 1.1.2 through TI like it wanted me to, and all was fine. At one point, I also completely uninstalled Titanium and re-downloaded it, allowing it to install 1.0, and then doing nothing further. This worked as well.

    • Since 1.2.1, I've had no issues with this error message, so perhaps doing a clean install and upgrading to that will work. I've found that testing your app just once on one of the problem SDKs (1.1.1 and 1.1.2 for me) will spoil it until I did a full re-install or managed to get the first workaround above to work.

    I hope this helps and please let me know if there is anything I can make clearer, as it sounds like we've had very similar experiences.
