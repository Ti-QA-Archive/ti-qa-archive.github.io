"An Application Restart is Required" Android w/3.0.0

New bug, same bug, old bug, or different bug? I've read all the explanations for this, but they don't seem to fit.

Steps to reproduce:

  • Remove all traces of app, check running applications (its gone)
  • Install from Play store
  • Navigate to home screen, launch from there
  • BOOM.

This message is 100% reproducible, and occurs every time the app is started, no matter how. I have yet to find a way around it. I have not tried the tiapp.xml falseroot thing yet, since it just went live a little while ago, but the reasoning why extra code is required to kill a second instance of an app that does not exist (best I can tell) seems like there is another explanantion. Maybe?

Tested on Droid Razr Maxx, Android 4.0.4

  • Hello Hal,

    The proposed solution to edit your tiapp.xml adding this:

    <property name="ti.android.bug2373.disableDetection" type="bool">true</property>
<property name="ti.android.bug2373.finishfalseroot" type="bool">true</property>
<property name="ti.android.bug2373.restartDelay" type="int">500</property>
<property name="ti.android.bug2373.finishDelay" type="int">0</property>
<property name="ti.android.bug2373.skipAlert" type="bool">true</property>
<property name="ti.android.bug2373.message"/>Initializing</property>
<property name="ti.android.bug2373.title">Restart Required</property>
<property name="ti.android.bug2373.buttonText">Continue</property>

    See that ti.android.bug2373.skipAlert is set to true, so it will happen silently on the background.

    1. You will need to save tiapp.xml.
    2. Clean the project
    3. Build your project.

    Tested on 2.1.4.GA, if issue persists please post logs to dig into it. Thank you.
