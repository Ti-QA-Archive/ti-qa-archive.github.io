App crashes an Android because of confliciting names

Ti SDK 3.4.1, Alloy 1.5.1

I have several modules residing in the app&#x2F;lib directory. One named form.js and another named filter.js.

It took me hours to find out the cause why my app didnt load these properly on Android. Apparently when i require() them in my code, another module gets used which seems to be part of the default framework. So these names are 'reserved'.

Are there any more of these names I should know of? I previously had issues with database.js and I think there are more that should not be used as a name.

I really dislike this setup (and the overal function of require(), in that it looks in several different directories)

I'm thinking of using a fix which basically adds subdirectory, so all my modules are in app&#x2F;lib&#x2F;unconflictingname&#x2F;** but this means I have to edit all my require() statements to prepend this directory (and it looks very unfriendly)

    This is going to be true for pretty much any language. As a seasoned developer, I can tell you, you want to stay away from naming files with generic terms. "form.js"? Try "SignupForm.js" or something that can tell another developer, without having to dive into the code, exactly what the file does.
