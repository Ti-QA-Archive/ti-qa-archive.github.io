Tweetanium Example Android Keyboard popup question

Hi,

I've be using the Tweetanium framework, very nicely written actually, to accelerate some app development.

I'm using a search bar and table view from the Kitchensink (table_view_layout_2.js).

All is good/working fine.

BUT! When I click in the search and the keyboard pops up, its moving up the lower tab system that Tweetanium uses.

I thought the keyboard would always pop over all views, but for some reason the view that holds the tabs gets pushed up and sits just above the keyboard…

I'd rather that the keyboard pops up and over that navigation tab as its a bit funky..

The tab view from tweetanium is created like

        tabView = Ti.UI.createView({
            bottom:0,
            height:tabHeight,
            backgroundImage:'images/tab_bg.png',
            width:platformWidth
        }),

I did try setting the zIndex: 10000 in the searchbar hoping that the KB popup would inherit or something..

But I'm a bit stuck here.

    Just a guess…try changing the Window defintion in styles.js. For android, it defines the softInputMode…I've changed from the original and forget what that was. Mine reads:

    Window: {
  backgroundImage:'images/background.png',
  navBarHidden:true,
  softInputMode:(Ti.UI.Android) ? Ti.UI.Android.SOFT_INPUT_ADJUST_PAN : ''
},

    I've been mostly testing with the iOS simulator. See the other soft input modes at http://developer.appcelerator.com/apidoc/mobile/latest/Titanium.UI.Android-module
