    • The Team profile is a good option for development but bear in mind certain features won't be available for use later.

      "Wildcard Bundle IDs are great for quick development – you don't have to create a new ID for each and every app you test. However, you cannot use some iOS features with a wildcard ID. You must create an explicit Bundle ID if you plan to use push notifications, iAd, or Game Center features."

      However looks like an App ID issue at Provision file. You may want to verify it again or better yet create a brand new one. Also try clicking on project's menu by right mouse button as the "Install to iOS Device" option can be found there.

      — commented November 13th 2012 by Eduardo Gomez

    • OK, so when I right clicked on the project I did get an Install to iOS selection. Strange. I also figured out the issue with the provisioning profile. Apple says my AppID is "DF94BB3ZPN.com.appliedx.ShowtimeDevelopment" but according to the profile in Titanium when I select it, it is actually "com.appliedx.ShowtimeDevelopment" with no bundle ID. I had my AppID entered into Titanium as "DF94BB3ZPN.com.appliedx.ShowtimeDevelopment" but when I took the bundle ID out and entered it as "com.appliedx.ShowtimeDevelopment", the red X went away.
