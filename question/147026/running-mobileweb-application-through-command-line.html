Running mobileweb application through command line

  • Accepted Answer

    You can't "run" a Titanium Mobile Web app from the command line. Mobile Web apps require a web server. When you build from Titanium Studio, Studio will start a web server for you.

    However, you can build a Mobile Web app from the command line. I recommend invoking Mobile Web's builder.py directly:

    # Mac OS X
python /path/to/Titanium/mobilesdk/osx/2.1.4.GA/mobileweb/builder.py /path/to/project

# Windows
python C:\path\to\Titanium\mobilesdk\win32\2.1.4.GA\mobileweb\builder.py C:\path\to\project

    That will generate the index.html and all the files into the /path/to/project/build/mobileweb directory. Either copy those files into the root of a web server or point your web server root to that build/mobileweb directory.

    Again, you currently MUST use a real web server (you can't drag and drop the index.html into a browser) and you really should put the Mobile Web site in the root of the served directory and not in a subdirectory.

    • You certainly have a lot of options. If you want to put your app in the app store, you can create a native Titanium app with a Ti.UI.WebView, that renders the Titanium Mobile Web app. You can also put the site on a web server.

      One pro of wrapping in a WebView besides being able to put the app in the app store is you can use the app while being offline. However, Ti.UI.WebViews do not have JIT-enabled JavaScript engines which means that a Titanium Mobile Web app will run faster in the Mobile Safari web browser than inside a WebView.

  • Another choice: use a standalone web server such as nginx, because titanium mobile-web is just one-page-app using backbone. refer to : http://siwei.me/blog/posts/titanium-titanium-web-run-titanium-web

    1. setup nginx in your PC
    2. configure your nginx as follow: 
    server {
     listen       3333;
     server_name  localhost;
     location / { 
         # here is your titanium projcet's folder. 
         root   /workspace/test_ti/build/mobileweb;  
         index  index.html index.htm;
     }
}
    1. restart nginx, and open in browser: http:&#x2F;&#x2F;localhost:3333
