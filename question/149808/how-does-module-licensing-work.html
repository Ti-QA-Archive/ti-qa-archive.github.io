How does module licensing work?

Hi,
I want to put a module I purchased within a "modules" folder in the root directory of a project me and my team are working on so it can be shared easily using git.

Will this work? I noticed most documentation mentions putting it in your "Library" (mac) folder instead.

This makes me wonder how licensing is enforced by appcelerator. Is my download of the module signed to me or can everyone on the team use it once I've assigned them a seat licenses to use it?

On another note the licensing checking is very frustrating. I've had it pop open the non compliance red alert window several times on me even though I have a valid license (and then it goes away later). It's starting to ruin the marketplace module experience for me.

— asked March 20th 2013 by Clark Endrizzi (work)

  • Hello,

    adding the modules to the project is good. It's actually the best practice, since you will be able to keep all the project in a working state (i.e., what happens if the Module's API changes in future version?). You can share it across all your developers, and I recommend putting all those developers under the same "group" in the appcelerator marketplace. That way, you will be able to share the license acrosss all your developers.

    Does that red screen happens on the same account, same machine? If you are logged with different accounts, this could be happening (one account not having the rights to use that module).

    Best,

    Mauro

    You can embed a module in you application project. and yes, the other developers getting the project from the repository will be able to use it.
