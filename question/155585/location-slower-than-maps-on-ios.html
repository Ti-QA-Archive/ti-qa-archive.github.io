location slower than maps on ios

We have an app with a std list view and map view of pubs.

On load of the app we use:
Titanium.Geolocation.getCurrentPosition()

We then generate the list view ( a table at present ) of the nearest pubs to the location gained from the getCurrentPosition.

However the getCurrentPostion will often be slower at returning the position than the map. Ie the map will display the blue current location before the Titanium.Geolocation.getCurrentPosition() has returned an accurate result.

Is there a way of using the maps current location pin instead?

    Dont use getCurrentPosition - it can returned cached results. Instead subscribe to the location event. Just remember to remove the event listener when done as it consumes lots of power
