Problem with Titanium Studio Debugger "suspend on error"

I am using Titanium Studio 1.0.7.201112281340 with Ti SDK 1.8.0.1. When debugging on MacOS in the iOS simulator, the debugger will stop on a certain line in one of my source files. There is no error or warning message that I can see, certainly not in the Console, nor on the simulator. If I resume execution, it keeps breaking on the same line. If I right-click on the stack trace, and un-check "Suspend on errors" and then resume execution then everything continues as expected, no errors in the simulator, Console, or anywhere. For reference, the file in question has only two JSLint errors, and they are in other locations in the file. (undefined global variables)

So, two questions: First, is there any way to see what error the debugger thought it was suspending for? Second, is there any way to persist the un-checking of "suspend on errors"? Right now I have to un-check this every time I debug.

Thanks all!

  • Ok, more info here: I got a non-minified version, and it turns out that the debugger is pausing when an exception is being thrown, even though the exception is being caught. Turns out this has already been discussed here: Why does Titanium Studio pause the debugger on CAUGHT exceptions?. I guess I will add my own request that the Ti debugger be changed to not suspend on caught exceptions. Thanks.

    — commented January 26th 2012 by David Southern

  • Turns out that you can persist the un-checking of "suspend on errors" in Preferences -> Titanium Studio -> JavaScript Debug. Still can't see how to tell what the error was short of letting it continue, and hope that something shows up in the console
