java.lang.IllegalArgumentException: bitmap size exceeds 32bits...

My app was built using Titanium SDK 3.1.2.GA and the latest Alloy at the time

Has anybody seen this / is it a known bug? Any help on this would be appreciated.

  • Hello David,

    I think I found the solution to this problem.
    When using the toImage() function on an imageview or view, it returns a Titanium.Blob. When it exceeds the available bitmap size, the application crashes.

    I suggest you to set the blob variables to null, so they can be garbage collected.
    Or use this module: https://marketplace.appcelerator.com/apps/7916?restoreSearch=true#!overview

    Yours sincerely,
    Patrick van Vuuren
