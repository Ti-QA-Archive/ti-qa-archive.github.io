HTTPClient Session not working on android

My app has the ability to login to my Drupal site. On IOS I can login and is returns the session information which I can save as properties. Also every time I open the app again and even completely restart it in recognises that I'm still logged in. I assume this is because the session is stored somewhere much like in your browser.

My problem is though this does not happen in Android. When I login the same session information is returned but the parts of my app that require the user to be logged in don't work, I just get errors that suggest the user is not logged in.

I'm using SDK 2.1 and have tried on a couple of versions of Android.

  • Session details are usually saved in a 'session cookie' that is returned to the Http client when the first response is received from the web server which should then be sent back on each subsequent request. There is a post that describes a work-around for android
