iPhone 5 splash and app not filling up screen

My app runs just like it were a pre-iphone5 app in the iPhone 5 simulator. It doesn't use the Default-568h@2x.png that is saved in the resources/iphone-directory. Instead it uses the regular splash image. I have tried removing the entire contents of the build directory manually, and cleaning the project several times. Why is my app running in "letterbox" mode? SDK version is 3.1.1.GA

    Hi

    The only requirements to enable Retina Tall (iPhone 5) are as follows;

    • Default-568h@2x.png file
      • Must be placed;
      • {appname}/resources/iphone/ folder (non-alloy)
      • {appname}/app/assets/iphone/ folder (alloy)
      • Must have the resolution of 640x1136
      • Must have the exact spelling and case as used above.
    • Your SDK must be set as Ti SDK 2.1.3 (but you really should always be on the latest version).
    • You should be using the latest xcode (or at least a more recent branch 4.5 or higher)

    Thoughts;

    • If you are creating an alloy app not correct location above.
    • Edit your tiapp.xml file manually and check your SDK version entry for confirmation

    • Solved it.

      The "default*.png" images are being cached in the resources folder and not being overwritten when re-compiling.

      Solution;

      Delete the ENTIRE contents of the resources folder. This will remove the files that are not being overwritten from the assets folder. I have tried this from scratch twice and worked both times. When I first started the app was letter-boxed, now fills the entire iPhone 5 (retina tall) device.

      Let me know how it goes.
