Unable to install SQlite db from file in iOS.

Application Type: Mobile

Titanium SDK: 3.2.3

Platform: iOS 7.1.x

Device: iPad Simulator

Host OS: OS X 10.9.3

Description:-

I am trying to have a feature to backup and restore the SQLite DB in my iOS app. I was able to successfully read the data from my "/Library/Private Documents/MYDB.sql" file and POST it to my applications Server.

I am able to GET the SQLite file content from the Server and create a dummy file (/Documents/DB.sql).

When I try to install the DB.sql I am getting an error Could not retrieve attributes. Please find my code below.

// Create an sqlite file with sqlite content.
var sqlInstallerFile = Ti.Filesystem.getFile(Ti.Filesystem.applicationDataDirectory, "DB.sql");

if(!sqlInstallerFile.exists()) sqlInstallerFile.createFile();

sqlInstallerFile.write(Ti.Utils.base64decode(data.sqlite));

// Delete existing MYDB. This step is irreversible.
DB = Ti.Database.open("MYDB");
DB.remove();
DB = null;

// Install this newly created sql file as the new DB.
DB = Ti.Database.install(sqlInstallerFile.resolve(), "MYDB");

Does anyone have a solution for this. I have read a few other Q&As, but I don't understand them.

Thanks,
Nirumal.
