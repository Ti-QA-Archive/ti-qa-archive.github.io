FREEBIE: TiSocket - a TCP Socket class for strings that reconnects if you lose connection

… but doesnt disconnect gracefully… yet.

i guess all the comments explain it pretty well. its build for string handling back and forth between server and client. it reconnects on disconnect and tries to connect and if it fails it retries in intervals. it also waits for the socket to get ready and fires events for almost anything including "ready" and "gaveup". ;)

if someone has the time to work over it, that would be nice because it got a bit messy with all the timers, counters and intervals.

for now it works pretty well. only downside is that it still tries to connect and sets the connection to ready when you disconnect. i think thats just fake and it only fires the handler but doesnt do anything but closing the socket.
workaround: disconnect() and kill the class.

anyway,

have fun!
