Source identification in table row event listener

Hey, guys –

I have a tableView that I'm filling with data where each row is built on the fly and made up of several stacked labels (name, address, description, etc.). I'd like for each row to launch a mapView. To that end, I've added custom properties to the row identifying the long/lat of the address represented in the row.

What I'm finding is that when my event listener is fired, the source element (e.source) is one of the labels that makes up the row, not the row itself.

How can I access properties of the row that contains the clicked element?

Thanks.
