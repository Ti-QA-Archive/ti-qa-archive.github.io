    No.

    First off, so far Titanium mobile has no support for Windows Mobile. Its Blackberry support is still in beta and the beta is only open to paid level subscribers. I hear the target date for the public release of Blackberry support is Q1 2011, but I don't know much else about what limitations it may have, etc.

    Secondly, in order to create the installable application for any given platform, you need the SDK for that platform installed so that Titanium can generate project source for that platform and compile it.

    The Android SDK is available for both Windows and Mac, so the android version of a titanium project can be compiled on Win7. However, Apple has not released a SDK for Windows so an iphone/ipad application can only be compiled using a Mac. I sincerely doubt Apple has any plans to release a SDK and compiler for Win7.

    It may be possible to run OSX in a virtual machine under Win7 and do it that way. However, the vast majority if not close to 100% of the folks will just use a real OSX machine to compile the iphone/ipad version of the program.

    Beyond Blackberry, I don't know what platforms Titanium mobile will eventually target. Some months ago I heard HP mention that they were interested in getting webOS support included in Titanium. But I have no idea if/when that will happen.

  • Technically this is possible. Titanium now has a Mobile Web Alpha that you can get if you pick up a developer subscription(free trial of it now :), and the Mobile Web will output to HTML5. Unfortunately it's still buggy at the moment on WebOS (it is an alpha after all), but you can take that and combine it with Phonegap's Windows Phone 7 implementation, and you can deploy it to WP7.

    Also BB beta is included in the above subscription, so that knocks that one out.
