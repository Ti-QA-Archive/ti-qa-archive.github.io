    Hello Morteza,

    Thanks for your inquiry. I think you are asking WHY is the APK file 10MB when you have yet to add any real code to your test app.

    If so there is a simple answer. The file size is based on payload of support files that ALLOW the app to functions.

    These include (not a full list);

    *Android Themes.

    *Basic graphics.

    *Library code.

    *JavaScript Engine - this is the largest part

    The size you are seeing is the overhead required. It can be seen as large, but only a small percentage of bigger apps of course. I believe that you can reduce this size depending on which Android API levels you target, but I would not consider this a major issue as the benefit of the Titanium framework easily out ways the file size issue.

    You can also try:

    *Deleting unwanted images. Delete the others platforms graphics images, If you build for only one or two platforms.

    *Don't put any waste space in your .js files.

    *Replace your code to another one new project.

    *Just clean and build again.
