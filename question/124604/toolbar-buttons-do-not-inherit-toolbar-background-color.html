Toolbar buttons do not inherit toolbar background color

When I create a toolbar with a button item, the button does not inherit the toolbar's barColor.

From Kitchsink: 

var toolbar2 = Titanium.UI.createToolbar({
    items:[change,flexSpace,revert],
    top:130,
    borderTop:true,
    borderBottom:true,
    barColor:'#336699'
});
change.addEventListener('click', function()
{
    toolbar2.barColor = '#000';
});

This works fine and when you click the toolbar changes colour and the buttons inherit that colour.

However, 

var toolbar2 = Titanium.UI.createToolbar({
    ...
    barColor:'#000'
});

does not give a black button, rather it stays the default blue.

So it appears that ONLY after some event - i.e. after MODIFYING the toolbar - does the button change colour.

  • Using the following code, the buttons inherit's the toolbar's backgroundColor. No events are necessary. Is it possible there is a problem in your button creation code? Could you post that?

    This was tested with mobile SDK 1.7.2, iOS 4.1, and the iPhone simulator

    var win = Ti.UI.createWindow();

var change = Ti.UI.createButton({title:'change'});
var flexSpace = Ti.UI.createButton({systemButton:Ti.UI.iPhone.SystemButton.FLEXIBLE_SPACE});
var revert = Ti.UI.createButton({title:'revert'});
var toolbar2 = Titanium.UI.createToolbar({
    items:[change,flexSpace,revert],
    top:130,
    borderTop:true,
    borderBottom:true,
    barColor:'#000'
});

win.add(toolbar2);

win.open();
