Android: Exception in getCleanUri

Our Android application keeps throwing the following Exception in DDMS, but the strange thing is that the images are loading

12-13 15:30:40.374: E/TiUrl(2419): (main) [7,212335] Exception in getCleanUri argString= /images/leftmenu/arrow.png
12-13 15:30:40.394: E/TiUrl(2419): (main) [16,212351] Exception in getCleanUri argString= /images/leftmenu/arrow.png
12-13 15:30:40.394: E/TiUrl(2419): (main) [4,212355] Exception in getCleanUri argString= /images/leftmenu/notifications.png
12-13 15:30:40.404: E/TiUrl(2419): (main) [7,212362] Exception in getCleanUri argString= /images/leftmenu/arrow.png
12-13 15:30:40.409: E/TiUrl(2419): (main) [4,212366] Exception in getCleanUri argString= /images/leftmenu/activity.png
12-13 15:30:40.414: E/TiUrl(2419): (main) [6,212372] Exception in getCleanUri argString= /images/leftmenu/arrow.png
12-13 15:30:40.419: E/TiUrl(2419): (main) [5,212377] Exception in getCleanUri argString= /images/leftmenu/location.png

We load the images like:

Ti.UI.createImageView({
    image: '/images/leftmenu/arrow.png',
    width: 12,
    height: 18,
    right: 15,
    touchEnabled: false
});

Is there anything we can do to prevent this Exceptions from throwing? I don't know what we are doing wrong.

It happens on line 335 in TiUrl.java: https://github.com/appcelerator/titanium_mobile/blob/master/android/titanium/src/java/org/appcelerator/titanium/util/TiUrl.java

SDK: 2.1.3GA
Samsung Galaxy SIII
Android 4.1.1

  • Hi

    Things to check;

    • your image url is all lowercase; are all folders and files in your development folder also all lowercase?
    • does the folder structure you are using appear more than once in your app? For example; if you have a images folder inside resources and/or one inside android and/or one inside android images resolution specific folders - with any filename duplications then this might be an issue
    • have you tried opening and then saving your images again using an image editor (small chance of image corruption - unlikely given the number of files)
    • has your device run out or running close to using all the local device storage?
    • are there old files in the emulator or the device? Un-install from both and re-compile
