Is Titanium affected by the iOS SDK4.0 api calling through intermediary translation?

I am planning to get my hands on Titanium development for building apps but i am a bit apprehensive as i have come across the iOS 4.0 SDK saying - Applications may only use Documented APIs in the manner prescribed by Apple and must not use or call any private APIs. Applications must be originally written in Objective-C, C, C++, or JavaScript as executed by the iPhone OS WebKit engine, and only code written in C, C++, and Objective-C may compile and directly link against the Documented APIs (e.g., Applications that link to Documented APIs through an intermediary translation or compatibility layer or tool are prohibited). Is Titanium affected by this?
