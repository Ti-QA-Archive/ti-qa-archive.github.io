ACS Stored Procedure?

I know with parse you can create stored procedures that can run on the sever which is awesome and lowers alot of heavy overhead. Unfortunately parse does not integrate as smoothly with titanium as ACS does and I cant rely on Aaron Saunders living forever and always finding a way to make parse work good with Appcelerator. When deciding whether to use parse or ACS Im really at a crossroads because with parse I can do bulk updates and deletes which is huge and as far as I can tell ACS does not do that.

The problem I am facing is that I don't see a way to update multiple records in ACS. If I have 200,000 photos in ACS how would I delete 150,000 that I see as "unneeded" without having to do a query to get those 150,000 records and then run 150,000 Cloud.Photos.remove calls?
