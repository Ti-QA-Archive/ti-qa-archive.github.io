  • After doing everything that's mentioned in previous comments it still didn't work for me. Using freshly installed Titanium Studio with SDK version: 2.0.1 (04/12/12 16:33 999c68a) and iPhone SDK version: 4.3.

    So I dug into the code, it appears that the identifier is only tested for when:

       __IPHONE_OS_VERSION_MAX_ALLOWED >= __IPHONE_5_0

    The problem was introduced in this commit:

       https://github.com/appcelerator/titanium_mobile/pull/600

    So I figured I just get rid of that code. In Titanium > Preferences > Titanium Studio > Titanium look up the Titanium SDK Home, go there in a terminal and search for the file NetworkModule.m (for me it was in ~/Library/Application Support/Titanium/mobilesdk/osx/2.0.1.GA2/iphone/Classes). Look up the two blocks with: 

        #if __IPHONE_OS_VERSION_MAX_ALLOWED >= __IPHONE_5_0 
   yada yada 
#endif

    and remove or comment out. Exit Titanium Studio, remove the build directory from your project, start Titanium and try again. Worked for me.

    I would suggest there must be an easier way to configure in Titanium which iPhones version you want to target, couldn't find it though.

  • What version of XCode do you have installed?

    After some more searching I discovered someone had logged a Jira 

    User had both Xcode 4.0 (/Developer) and 4.2 (/Developer-4.3) installed. And Xcode 4.0 was used by default.
Titanium does not support 4.0 since 1.8.0 release. 
Solution: switch to Xcode 4.2 and above.

sudo xcode-select /Developer-4.2

    If you're not up to XCode 4.2+ then you might want to try updating.

  • I fixed this issue by following the instructions on this page:
