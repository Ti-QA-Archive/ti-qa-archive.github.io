Persistent Keyboard

I was wondering if there is a way to have a persistent keyboard inside a window.

Let's say, I have three TextFields in a window. The first two have the return key set to "NEXT", such that on tapping that, the next field in the chronological order is focused.

When I tap next, the next field is focused, however, there is this animation happening: The keyboard goes down a few pixels and then comes up back to it's original position. Any way to avoid this animation from happening?

TiSDK: 2.1.1.GA
on iPhone 4, running iOS 5.1.1

  • Accepted Answer

    Set suppressReturn to false on the text field (seems backwards, but false is correct, trust me), and handle the return event to focus the next text field. That will remove the stutter.
    http://docs.appcelerator.com/titanium/2.1/index.html#!/api/Titanium.UI.TextField-property-suppressReturn
