negative positioning no longer works

I used to be able to position a child view within it's parent using a negative top position, and the child would appear directly above the parent. For example:

var bar = Titanium.UI.createView({
        width:36,
        height:50,
        backgroundColor:'#000'
        });

var view = Titanium.UI.createView({
        backgroundImage:'/images/test.png',
        height:22,width:36,left:0,top:-22
        });

bar.add(view);

This would position "view" directly above "bar" because of the negative top value. Since an update to 2.1.1, "bar" now hides any overflow, meaning "view" wont show up.

Is there any way around this (while still keeping "view" as a child of "bar")?

    This is because they now use view "clipping".

    If you need it turned off globally modify the following file in your mobilesdk for iphone: TiViewProxy.m.

    Change line 152 from [self view].clipsToBounds = YES; to [self view].clipsToBounds = NO;.

    Warning: it might have some other undesirable effects.
