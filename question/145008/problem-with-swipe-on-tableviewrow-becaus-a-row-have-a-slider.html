    Hi

    Still no code - so this is a guess based on your screen shots - which did help.

    As you are the one telling the window to slide after responding to the swipe event for the table you can tell it to slide or not based on what fired the event.

    I am going to assume that the slider does actually move when you touch it, but you ALSO have your slide animation as well.

    One way to resolve this would be to add an extra "custom" property to the slider and then check for that property in the swipe event and ONLY slide if the property was not found.

    var slider = Ti.UI.createSlider({
...
    ignoreSwipe: true
...
});

    Then in the swipe event code add this check

    table.addEventListener('click', function (e) {
    if (!e.source.ignoreSwipe === true) {
        yourSwipeFunction();
    }
});

    As I have not seen your code I cannot put everything in place but you should be able to work out how to place things from this.

    The basic idea would be that the slide function is ONLY called if the event was not fired by an object that has the extra property and that property is set to true. e.source... is used to access that actual object that fired the event that the table is responding to.

    This is ONE way of solving it, there are a few others ways, but should be enough to get you started.
