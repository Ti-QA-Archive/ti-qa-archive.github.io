iOS module error related to sub-projects

I made an iOS module which includes several subprojects as libraries. It works fine as a Xcode module. However, when I tried to convert it to Ti module, it seems it is not loading the subproject, thus results in errors at build time. build.py tells that it cannot find the header files of subprojects.

What should I do to Titanium to find the header files of the iOS subprojects ?

  • Thanks a lot. Framework in xcconfig is rigth, but what I really wanted to know was how to set path subproject library file such as "xxxx.a" and header "xxx.h". First i built it as one project, but it was hard to set path header when i package it by "builder.py". Now I solved this problem. I changed like the following.

    I built subprojects each and then get the header and lib(.a)file each. Then just added those titanium module project in XCODE. Then, set path Header search path in XCODE's Building Setting.

    Also added path such as .dylib in module.xconfig like this.
    other_ldflags = $(inherited) -framework CFNetwork /usr/lib/libiconv.dylib

    eg) CFNetwork is framework, libiconv.dylib is usr/lib libarary file.
