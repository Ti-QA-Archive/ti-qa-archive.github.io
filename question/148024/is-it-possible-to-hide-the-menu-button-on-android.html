  • You can actually remove the software menu button by adding the following to your manifest:

    <uses-sdk android:targetSdkVersion="17" android:minSdkVersion="8" />

    MinSdkVersion can be anything you want it to be. Target Sdk of 11 of higher should hide the menu button. TargetSdk does not affect who can install your app, so I think for most cases it should be whatever the latest API level is.

  • If this is what you are talking about:

    You can make it go away by bumping the targetSdkVersion in your Android manifest or tiapp.xml (for example, this overflow was always appearing in my app, even though I have no menu set up, until I changed from targetSdkVersion 8 to 17)

    If you don't want it to disappear entirely, but instead want it in the ActionBar, then take a look at the docs for ActionBar and menus.
