Mobile App with External SDK

I am new to mobile app development and working on an app for my company that needs to work on both Android and iOS phones/tablets. Currently I am writing the program in a GWT project, using Java, and then compiling it into Titanium Mobile and from here, building it to the separate platforms. The problem I am having is I need to use an external SDK for my app. This external SDK is written platform specific, using the function calls to either os(Android or iOS). Is there a way that I can write the app once, using this external sdk, and then deploy it to both devices? Or do I need to either re-write the sdk, or write two apps, one for each platform, using each specific sdk?

  • Yes I am mixing a lot of technologies but that was the only way I could see to do it, other than writing two apps.

    There is a framework available (Emitrom) that will take a java GWT project and compile it into Javascript, which I can then import into Titanium Studio to deploy to both iOS and Android.

    I am writing my app in Java, compiling it through GWT and Emitrom into Titanium Studio, and then using Titanium Studio to push it to iOS or Android.

    I have a credit card scanner that attaches to mobile phones/tablets. This scanner came with two separate SDK's, one for Android and one for iOS. I would like to write one app, that utilizes one of the SDK's (the Android SDK since I am programming the main part of the app in Java), and deploy it on both Android and iOS.
