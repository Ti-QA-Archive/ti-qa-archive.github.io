    just one more note on this:

    Titanium supports only 2 of 4 Subclasses of the native CompoundButton: ToggleButton and CheckBox, but not RadioButton and Switch. They're represented as style properties (SWITCH_STYLE_TOGGLEBUTTON and SWITCH_STYLE_CHECKBOX) of Ti.UI.Switch which you can see here.

    That's why you are not able to get a "real" switch or radio-button just by providing theme resources. You need a module to wrap the native components, I don't know any. Otherwise you could create a custom element or use an alloy widget.

    I also think that it is very important to give users the best UX on whatever version they are. This became an often discussed topic… Should you support Holo.Theme on Android 4.0? Yes you should! By now, you could also ask yourself if you even should support Android 2.3.

    Android 4.X got over 80% as seen at the dashboards

  • Hi

    I may be wrong here - it has been know to happen, but the switch you referring to is not the same switch as Titanium actually supports (Android has more than one) so it is not such much that Titanium is ignoring you - it is more that it thinks you are talking about something else.

    • Toggle Buttons

    Here you can see that the two buttons types are shown side by side, however whilst each appears to do the same job they are two different beasts.

    Whilst Android 4.0 has access to the switch and toggle, prior versions only have access to toggle. They both return back on/off or true/false - so the result is the same.

    As far as I know it takes more low level Android API calls to allow the switch version to become a new view inside Titanium - which would sit alongside the existing toggle version. I would imagine it is for this reason you cannot have the cool looking one as time has been focussed on other things instead of giving a second version of a switch that has an existing version and would only benefit Android 4.0 users (which itself is only just getting around/over the 50% mark).

    However I am with you - the old one makes apps look - well - old.

    I would suggest you create a ticket for it and hopefully Appcelerator will listen.
