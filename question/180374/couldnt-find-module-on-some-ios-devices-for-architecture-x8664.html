Couldn't find module on some iOS devices for architecture: x86_64

I need a user agent module to modify the user agent of my webview in titanium on iOS.
Only iPhone 5 and iPad 2 are able to load this module correctly. Other iOS devices output this error:

> "Script Error Couldn't find module: tv.harukaze.ti.webviewuseragent.ios for architecture: x86_64"

and it stops the app from progressing further/running.

I'm currently using Alloy.
The module was extracted from a zip file here:
root/modules/iphone/tv.harukaze.ti.webviewuseragent.ios

I'm requiring it like this:

var useragent_module = require('tv.harukaze.ti.webviewuseragent.ios');

For example it does not work on iPhone 6 but it works on iPhone 5.

Do I need to rename the folder in which the module is in or something?

This is the github link for the module.

    The reason for this is because the module has no 64-bit support yet, as you can see here. On gitTio you can find modules that support a 64-bit architecture. This one doesn't yet.

    Try to reproduce the steps as described here, to change the module itself to a 64-bit architecture.

  • Just ran into this issue myself. I think it's important here to note that when doing a require on a commonJS file and it can't be found, looks like you'll get the "can't find 64 bit" message. Unfortunately, in the case of the app not being able to find the JS file trying to be require'd, the 64 bit message comes up. If you get this message with a commonJS file, the first thing you should check is if the path is right and if the case is right. In my situation, a single character was the wrong case.
