Android Manifest and Google Maps API Key

In the " Kitchen Sink" demo for Titanium SKD 3.1.1. GA, there's a folder called KindleFire_AndroidManifest. Read-me doc states to copy it to resources folder.

Question: Does using a Google Maps API in a Ti app require the creation of an Android Manifest with Google Map API key listed in manifest ?

    The map API key in the manifest is deprecated – it was required for our Google Maps API v1 module. Unless you have an old project with a v1 API key from Google, you'll need a new v2 key and you'll need to use the new Maps API v2 module. See http://docs.appcelerator.com/titanium/latest/#!/api/Modules.Map as a starting point for implementing this module. Despite what that page might say, you need to download the module and include it in your project. Get it from GitHub (click the View Raw or follow the steps I posted in Ranier's question of a couple of days ago).

    Finally, any Kindle examples you find in the Kitchen Sink or elsewhere would apply to building for a Kindle device only, not for a general Android device.
