Win32 installation

Is it possible to install Titanium sdk components to a directory other than %ALLUSERSPROFILE%\Application Data\Titanium (i.e. Network Share) ?

  • My instinct tells me that's a very bad idea. What is your objective and reason for wanting to do this?

    I imagine you want to save Internet bandwidth on a network with more than one person developing with Titanium. If so, download the SDKs once to a network share, and allow other users to copy them from there to the local directory you've mentioned. To automate the process, use a batch file (ie using xcopy) or sync software to mirror the files periodically. Obviously, the users will still need to restart Titanium Developer to be able to utilize the new SDKs.

  • My initial objective was to install it to a different drive as I was short of space on C-drive at the time.
    I assume that the answer is "no" at this stage.

    On the same note, every new iPhone/iPad Titanium application on a MAC consumes over 200 Mb of disk space.

    I am suggesting to create a symbolic link to "/Library/Application Support/Titanium/mobilesdk/osx/1.X.Y/iphone/libTiCore.a" instead of copying it to "build/iphone/lib" (this is for the next mobilesdk release).
