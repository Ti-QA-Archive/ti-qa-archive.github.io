  • Set autoLink property before setting value. For multiple types linking use | separator 

    var win = Ti.UI.createWindow({
    backgroundColor : "white"
});

  var contactInfo = Ti.UI.createTextArea({
    width:'90%',
    autoLink: Ti.UI.AUTOLINK_ALL,
    textAlign:Titanium.UI.TEXT_ALIGNMENT_CENTER,
    value: '123 Fake St. \n Some City, US 12345 \n \n Phone: (123) 456-7890 \n Email: someone@aplace.com \n www.google.com',
    editable:false,
    backgroundColor: 'transparent',
    font:{fontFamily:'Arial',fontSize:16},
    bottom: 10
});
win.add(contactInfo);
win.open();

  • Here's what ended up working for me (using the Kitchen Sink example):

    var contactInfo = Ti.UI.createLabel({
    autoLink : Ti.UI.Android.LINKIFY_ALL,
    width: '90%',
    bottom: 10, 
    textAlign: Ti.UI.TEXT_ALIGNMENT_CENTER,
    font:{fontFamily:'Arial',fontSize:16},
    backgroundColor: 'transparent',
    text: '123 Fake Street \n Some City, US 12345 \n \n Phone: (123) 456-7890 \n Email: someone@aplace.com \n www.google.com'
});

    It would be nice if there was a clear distinction that specified whether we should be using Ti.UI.AUTOLINK, TI.UI.AUTODETECT, or Ti.UI.Android.LINKIFY; it would really save a lot of frustration.

    No luck on getting this to work for a text area on Android. So, I just use a platform check. If it's Android, I use the code above, if it's iOS, then I create a text area with Ti.UI.iOS.AUTODETECT_ALL set.

    Also should point out that the addresses can't have an abbreviation in them because they won't be recognized as an address (if you use 'St.' instead of 'Street' in the code above, only the zip code is linked.
