Universal App Submission Error:

I was just trying to submit my universal app the app store and hit a snag. I get this error when compiling with both 1.3 beta and TiStudio preview. Cant seem to figure it out

I followed the directions on this page link to add the icons for the iPad and iPhone. But I get an error when uploading to app store from xcode that says 

iPad: Icon.png: icon dimensions (57x57) don't meet the size
requirements. The icon file must be 72x72 pixels, in .png 
format.

The thing is, the Icon.png file is the 57x57 file, Icon@2x is the 114x114 file and Icon-72.png is the iPad image. It seems its not recognizing the images. Here is a Screenshot of Plist section. I tried changing the order so the ipad one was first but for some reason it keep only looking at the one Icon.png file.

Any ideas how to solve this? any help is GREATLY appreciated.

  • A M had one solution above… here is a more complete solution:

    Open the master template file in the SDK you are using.

    example on the mac: /Library/Application Support/Titanium/mobilesdk/osx/1.7.1/iphone/Info.plist

    You will see the following block

        <key>CFBundleIconFiles</key>
    <array>
        <string>__APPICON__.png</string>
        <string>__APPICON__@2x.png</string>
    </array>

    Add a line and make it look like

        <key>CFBundleIconFiles</key>
    <array>
        <string>__APPICON__.png</string>
        <string>__APPICON__72.png</string>
        <string>__APPICON__@2x.png</string>
    </array>

    Each time you compile an iphone project using the 1.7.1 sdk it will generate the appicon72.png reference based on the changes above.

    As of Titanium Studio Build 1.0.2.201107130739 it appears to build the info.plist based on the above referenced master template file on every build. …meaning you don't have to delete it for each change.

    Hope that helps someone who's been struggling with it as I had in the past.

  • I believe Titanium regenerates the Info.plist on each build and does not specify the -72.png file which you need to submit a Universal App.
    Here's how I resolved it using 1.3 beta and submitted successfully to Apple:

    Copy the Info.plist out of your build/iphone directory and place it in your project directory. That will force Titanium to use your Info.plist instead of generating it each time. Make sure you have the icons listed. Here's an example:

             <key>CFBundleIconFile</key>
    <string>Icon.png</string>
    <key>CFBundleIconFiles</key>
    <array>
        <string>Icon-72.png</string>
        <string>Icon.png</string>
        <string>Icon@2x.png</string>
    </array>

    By moving the Info.plist be sure to manually make any changes later.

  • So with iOS7 some problems appeared again, related to new icon sizes and older Ti.SDK builds! I think the newest versions of Ti.SDK (latest 3.x's) automatically add them in the build of your app. However if you are like me and still work with older versions of the Ti.SDK you need to change some stuff in the ~&#x2F;Library&#x2F;Application Support&#x2F;Titanium&#x2F;mobilesdk&#x2F;osx&#x2F;SDK_VERSION&#x2F;tiapp.py :

    Find the line which says:

    iconorder = list(iconname+".png" ... etc
and complete this with the following new icons (60.png, 60@2x.png, 76.png, 76@2x.png):
iconorder = list([iconname+".png",iconname+"@2x.png",iconname+"-60.png",iconname+"-60@2x.png",iconname+"-72.png",iconname+"-72@2x.png",iconname+"-76.png",iconname+"-76@2x.png",iconname+"-Small-50.png",iconname+"-Small-50@2x.png",iconname+"-Small.png",iconname+"-Small@2x.png"])

    This part in the tiapp.py generates the bundleiconfiles property in the info.plist file. IT DOESN'T WORK if you add them manually to a custom info.plist file as this tiapp.py will still overwrite them. So do it (carefully) in the tiapp.py file.

    Now be sure to add the new icon files (with maybe a flat design, that's up to you) to your project Resources&#x2F;-root or project's Resources&#x2F;iphone folder.

    Good luck!

  • I know this thread is a little old, but I just went through this and managed to get a universal app out the door successfully. Basically, just remember to include appicon-72.png and appicon-144.png and it should build correctly. XCode for some reason doesn't show the iPad icons but they will be in there.

    Also, although I had all of my defaultxxx.png files named correctly and in the right place, XCode didn't seem to recognize them, so I had to drag them all into the correct places that appeared when I switched the "Devices" selection to "Universal". So I dragged in default.png, default@2x.png, default-portrait.png, default-portrait@2x.png, default-landscape.png and default-landscape@2x.png in their places and they all appeared (and the yellow '!' icon went away).

    Lastly, I checked the info.plist file from inside XCode and for some reason, it was listing all of those defaultxxx.png files as icons, which was causing another validation error! No idea why it did this, but I removed all of those references plus a bunch of copies of references to the icon files and then validated and it all went through. Whew!

    I'm betting there is a way to configure Titanium so you don't have to do all of this, so I'll update if I can figure that out.
