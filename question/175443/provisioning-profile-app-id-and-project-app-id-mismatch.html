Provisioning Profile app id and Project App Id mismatch

Using Titanium Studio on MacBook Pro, trying to run app on USB attached mini iPad. I get the following message:

Selected Provisioning Profile app id "com.nextgenerationsys" does not match project app id "com.nextgenerationsys.Word2"

So obviously I created them, one with the .Word2 prefix and one without. Is there any way to manually change this or do I have to start over? I can't find any way to edit either one of these to make them identical.

Thanks,

Dave

  • When you created your App ID (part of the provisioning profile in the Apple DevCenter portal), you set the app's bundle ID to be com.nextgenerationsys but in your tiapp.xml, you entered the app ID as com.nextgeneration.Word2. Those have to match.

    If you meant to be using a wildcard provisioning profile (so you can use one profile with multiple apps), you need to update the associated App ID (in the Apple DevCenter portal) so that the app ID is com.nextgeneration. (I'm guessing you left off the '.' part).

    You might want to check out the Apple docs, for example When should I use a wildcard App ID?
