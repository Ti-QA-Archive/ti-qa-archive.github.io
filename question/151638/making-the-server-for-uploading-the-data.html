    Hi

    When you say "make it yourself", do you mean;

    • specify it and host the server yourself?
    • specify it and have someone else host it?
    • choose a server platform and have another company host it?

    Unless you have a lot of bandwidth - get someone else to host it and if there is budget pay for a managed server.

    As to what sort of server software - this one will have lots of answers, there are loads and they all have advantages and disadvantages, ideally choose one you are comfortable with.

    Personally I flip-flop between PHP and ASP.net, but it always project dependant and budget dependant as well.

    If you are just hosting files then you might be better served simply doing that - use something like Amazon S3 and even a CDN to host the files as static objects, there is no server required and you only pay for what you are storing and retrieving. The costs can be low - but constant access will increase those values.

    You really need to work out how much data in what sort and how many times it will be uploaded and downloaded.

    Does the server need scripts to be run to pass back data or even a database? All factors.
