A little SDK help

I keep getting the following error message when trying to launch Kitchen Sink. Any suggestions?

[INFO] One moment, building …
[INFO] Titanium SDK version: 1.5.1
[INFO] iPhone Device family: iphone
[INFO] iPhone SDK version: 4.2
[ERROR] Error: Traceback (most recent call last):
File "/Library/Application Support/Titanium/mobilesdk/osx/1.5.1/iphone/builder.py", line 673, in main
fd = open(source_lib,'rb')
IOError: [Errno 2] No such file or directory: '/Library/Application Support/Titanium/mobilesdk/osx/1.5.1/iphone/libTiCore.a'

  • I have the file that you are missing at that location. However, recently I've seen some Appcelerator documenation (in the native module guides) that says that sometimes the lib gets installed at ~/Library/Application Support/Titanium/mobilesdk/osx/1.5.1/iphone/libTiCore.a. At least a quick check. Also, check if you have /Library/Application Support/Titanium/mobilesdk/osx/1.5.0/iphone/libTiCore.a.

  • If you build using a later version of the apple SDK, does the problem still occur?

    This used to be an issue for people (see this post and Jeff's blog post here).

    However, the problem has been corrected in SDK versions after the date of Jeff's blog post.

    Hence, ensure that you have 1.2.2 of Titanium Developer installed, install the latest Continuous Build, and clean out the contents of the build&#x2F;iphone&#x2F;subdirectories. Then try again.

    Hope this helps
