android .project permission denied and R.java error

Titanium SDK: 1.5.0
Andriod SDK: APIs 2.3

Hello,
I have a project that I have been working on for iPhone and I now wanted to get it running on Android. On 64-bit windows 7, I have my Titanium Developer all set up, and the Android sdk ready, and when I create a new test app, it runs just fine on the emulator. But, when I try to run my app, first I get this error regarding Permissions denied on the .project file:

[ERROR] Error in compiler. <type 'exceptions.IOError'>, [Errno 13] Permission denied: 'C:\Carls\Projects\mobile\ushud\USHUD.com\build\android\.project'; Traceback (most recent call last): File "C:\ProgramData\Titanium\mobilesdk\win32\1.5.0\android\builder.py", line 1548, in <module> s.build_and_run(False,avd_id) File "C:\ProgramData\Titanium\mobilesdk\win32\1.5.0\android\builder.py", line 1300, in build_and_run self.android.create(os.path.abspath(os.path.join(self.top_dir,'..')), True, project_dir=self.top_dir) File "C:\ProgramData\Titanium\mobilesdk\win32\1.5.0\android\android.py", line 266, in create self.render(template_dir, 'project', app_dir, '.project') File "C:\ProgramData\Titanium\mobilesdk\win32\1.5.0\android\android.py", line 108, in render f = open(os.path.join(dest, dest_file), "w") IOError: [Errno 13] Permission denied: 'C:\Carls\Projects\mobile\ushud\USHUD.com\build\android\.project'

I will continue to get the R.java error each time I run it until I go to the edit tab, and hit "Save Changes" on my project properties, then the next run I will get the permissions denied error again.

I have tried all I know to try including adjusting the permission settings, opening it to everyone, but still get this error. Very weird because the test program I made works great, but my main one, not so much.

Anyone have any ideas??

  • Looking at the trace log, I believe it is actually barfing because of your project name (despite the file permissions issues in the log file). Our build scripting will attempt to generate Java identifiers with a . character, which is illegal Java syntax. If you create a new project with the name 'USHUD', will it run?
