  • Since there is not fix as of yet. I used a work around. Since I am using a tabGroup and my tabGroup is created by a separate method outside of my app.js file. I added an event listener to check for the Ti.Android.currentActivity resume event. When that event fires, I call the close() method of the tab group and then call my tabgroup creation method to recreate the tab group. So far this has been working well for me. I hope this can help someone else
