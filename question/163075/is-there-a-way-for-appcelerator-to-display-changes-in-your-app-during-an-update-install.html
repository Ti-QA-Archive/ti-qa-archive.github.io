  • Is there a built-in Titanium method to display changes you put in the changelog.txt when the user starts the app?

    In short, no. Titanium exposes native functionality. This behavior is not native to either the android sdk nor the ios sdk.

    You could create a view, and load the changelog into it for display at first run of a new versions. This would require writing the version to a Ti.Property. Comparing new apps version to saved version and if first time we have run this new version (possibly saved in a property as well) throw up the view. You would need to move the changelog into Resources or into app&#x2F;assets to be able to access it however.
