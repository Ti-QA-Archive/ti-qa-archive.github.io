How to sharing Safari cache with WebView and Network.HttpClient the way Android Does with Chrome

I'm using the Modules.Facebook framework to log into the application that is being released on both iOS and Android.

On Android, after logging into Facebook successfully via Modules.Facebook, both webviews and HttpClient calls recognize Facebook is active because the same cache seems to be shared with Chrome.

Unfortunately, in iOS that doesn't seem to be the case. The Modules.Facebook uses Safari to log into Facebook, but that cache isn't shared via the webviews or HttpClient. Is there a way to do that?

Thanks!

Titanium SDK: 3.1.0.GA,

Device: iOS Simulator, iPhone 4, RazrM (Android 4.1.2)
