    Hi

    Without you providing any code to check HOW you are referencing images or information on where the images are actually stored this will need to be a guess.

    But is may well be one of these two issues;

    • You have placed the images in a location that the simulator allows but the device does not
    {appname}/resources/iphone/example.png
{appname}/resources/iphone/folder/another.png

    If you have placed the images in the iphone folder like ones above - then you would reference the images like this;

    /example.png
/folder/another.png

    This is because the platform folders are virtual and are designed for you to place your platform specific imagery inside. When the app is complied these folders are collapsed down as if they were not there and the other platforms folders are not included in your project. The simulator does allow either reference but that is a quirk and not something you should rely on.

    • You have image names that do not match the physical filename because of upper and lower case differences.
    Default.png !== default.png
example.png !== example.PNG

    For safety and sanity I ALWAYS convert my physical filenames to lowercase versions so I know in my code they will ALWAYS match. Again the simulator does not care - but the device certainly does.

    • This is happening because you told it to;

      The iphone width is 320, the ipad is 768, so you only have one set of layout dimensions your iPhone version is also used for the iPad verion.

      You tell the text field to be 300 wide and start from 10 in from the left, so the iPad does the same thing;

      var email = Titanium.UI.createTextField({
    color:'#336699',
    top:'110dp',
    left:'10dp', 
    width:'300dp',
    height:'40dp',
    hintText:'Email ID',
    keyboardType:Titanium.UI.KEYBOARD_DEFAULT,
    returnKeyType:Titanium.UI.RETURNKEY_DEFAULT,
    borderStyle:Titanium.UI.INPUT_BORDERSTYLE_ROUNDED 
});

      You tell the button is has a width of 90 and no left or right assigned so it sits in the middle of the screen.

      var loginBtn = Titanium.UI.createButton({
    title:'Login',
    top:'240dp',
    width:'90dp',
    height:'35dp',
    borderRadius:1,
    font:{fontFamily:'Arial',fontWeight:'bold',fontSize:14}
});

      What you could do is this;

      var email = Titanium.UI.createTextField({
    color:'#336699',
    top:'110dp',
    left:'10dp', 
    width:'300dp',
    height:'40dp',
    hintText:'Email ID',
    keyboardType:Titanium.UI.KEYBOARD_DEFAULT,
    returnKeyType:Titanium.UI.RETURNKEY_DEFAULT,
    borderStyle:Titanium.UI.INPUT_BORDERSTYLE_ROUNDED 
});
if (Ti.Platform.osname==='ipad') {
   // example tweaks only - you set your own
   email.applyProperties({
      left: '50dp',
      top: '150dp',
      width: '500dp'
   });
}

      This will let you tweak the layout if the platform is iPad, but leaving all the original values there.

      OR you could write a helper function do achieve this inline;

      function adjustForDevice(iPhone, iPad) {
    return (Ti.Platform.osname==='iphone') ? iPhone : iPad;
}
var email = Titanium.UI.createTextField({
    color:'#336699',
    top: adjustForDevice('110dp', '150dp'),
    left: adjustForDevice('10dp', '50dp'),
    width: adjustForDevice('300dp', '500dp'),
    height:'40dp',
    hintText:'Email ID',
    keyboardType:Titanium.UI.KEYBOARD_DEFAULT,
    returnKeyType:Titanium.UI.RETURNKEY_DEFAULT,
    borderStyle:Titanium.UI.INPUT_BORDERSTYLE_ROUNDED 
});

      This lets you provide both values at the same time with; iPhone first and iPad second.
