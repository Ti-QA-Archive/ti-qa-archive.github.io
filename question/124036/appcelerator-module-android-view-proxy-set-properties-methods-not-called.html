Appcelerator module (android): view proxy set properties methods not called!

I'm making a module for android, and i have a view proxy on which i wrote a setTestProp method to set the property "testProp". The method is using the @Kroll.setProperty notation and @Kroll.method.

The problem is that this method is only called when i explicity set the property like this: 

var view = module.createView({
    top:0,
    left:0,
    width:320,
    height:50,
    backgroundColor:"#00ff00",
});

view.testProp = "something"; //this works (setTestProp called on ViewProxy)

but i doesn't call setTestProp when i set the property inside the createView like this: 

var view = module.createView({
    top:0,
    left:0,
    width:320,
    height:50,
    backgroundColor:"#00ff00",
    testProp: "Hello", //setTestProp method on ViewProxy not called
});

Here is my viewProxy class:

@Kroll.proxy(creatableInModule=AdwhirlModule.class)
public class ViewProxy extends TiViewProxy {

    AdWhirlView adWhirlView = null;
    public ViewProxy(TiContext tiContext) {
        super(tiContext);
        // TODO Auto-generated constructor stub
    }

    @Override
    public TiUIView createView(Activity arg0) {
        // TODO Auto-generated method stub

        adWhirlView = new AdWhirlView(this, arg0);
        return adWhirlView;
    }

    @Kroll.setProperty @Kroll.method
    public void setTestProp(String value) { //called only when i explicitly set the property (not inside createView)
        adWhirlView.changedTestProp(value);
    }

    @Kroll.getProperty @Kroll.method
    public String getTestProp() {
        return "tmp";
    }

}

I hope someone can help me! i'm going crazy… i saw that examples source code does exactly what i do and they should works, but my code doesn't!!

Thanks!
