[Android] WebView inside a TableView => scrolling inside the WebView is not working

Hello,

I'm stuck with this issue since hours and I didn't find anything relevant, so …

I've a simple view, with a tableView, a TableViewRow and a WebView inside with a predefined height.
My problem is that, whatever I try, I'm not able to scroll the content of the webview on Android, but it works perfectly on IOS (at least 7.X).

I've tried with Ti SDK 3.2.3 and 3.3, Android 2.3, 4.1 and 4.4, the issue is always the same.

Here is the (simple) code to reproduce this issue.

var self = Ti.UI.createView();

    var table = Ti.UI.createTableView({
        showVerticalScrollIndicator : false,
        top : 10,
        height : 'auto',
        backgroundColor : '#31434C',
        separatorColor : 'transparent',
    });
    var data = [];

    var row = Ti.UI.createTableViewRow({
        selectionStyle : 'none',
        height: 160,
    });

    var webView = Ti.UI.createWebView({
        top: 10,
        bottom: 10,
        enableZoomControls: false,
        showVerticalScrollIndicator: true,
        showHorizontalScrollIndicator: false,
        height: 150,
        width: '90%',
        html: '<!DOCTYPE html> <html> <head><meta name="viewport" content="width=device-width" /><meta http-equiv="content-type" content="text/html; charset=utf-8" /></head> <body><div>Line 1<br/>Line 2<br/>Line 3<br/>Line 4<br/>Line 5<br/>Line 6<br/>Line 7<br/>Line 8<br/>Line 9<br/>Line 10<br/>Line 11<br/>Line 12<br/>Line 13<br/>Line 14<br/>Line 15<br/></div></body> </html>',
    });

    row.add(webView);
    data.push(row);

    table.setData(data);

    self.add(table);

Any help/suggestion is very welcome :)

Thanks.
Cédric.

  • If you modify your code to put the webview directly into your self (Ti.UI.View) object then it scrolls fine. I think you've just hit one manifestation of why we say what you're doing is an anti-pattern. We recommend you don't put scrolling containers inside scrolling containers. Furthermore, WebViews are among the most expensive components, computationally speaking and generally should not be used in repeating-views like ListViews and TableViews.

    • Cedric, right, if you have HTML to display the WebView is the way to do it. I guess I figured that if you were putting it in a tableview, you were probably going to have multiple rows, each with a webview, which could get real bad.

      As for your layout, the webview itself supports scrolling, as you know. So if the help text & form is too long, users should be able to get to all of it. You could retrieve the HTML using the HTTPClient, do some string parsing to modify it to your needs, save it to a local file, then load your webview's content from that local file rather than the remote URL. You could set up a webservice to do the same, sending the processed HTML to the webview. Sounds like you'll need to get creative to solve the layout issue. Just don't create another problem in your app by contorting Titanium to do things that aren't in your best interest.
