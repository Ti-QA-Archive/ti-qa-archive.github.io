Titanium Tabgroup bottom iOS bug

Hello,

I've been stuck at this problem since two days ago. I can not figure out how this happens. I've got an iOS7 application which uses the Tabgroup in Titanium Appcelerator. It always worked, and not it just gives a space at the bottom of the application on my iPhone 4S.

I've tried cleaning my project, restarting my Macbook and restarting Titanium several times. I have even created a new clean project with simple tab group code and it gives the same issue. I really don't know how this happens. Is this a bug, or am I doing something stupid?

Simple code that I've used to create a tab group:

    // create tab group
    var tabGroup = Titanium.UI.createTabGroup();

    // create base UI tabs and  windows

    var win1 = Titanium.UI.createWindow({  
    title:'Tab 1',
    backgroundColor:'#fff'
    });

    var tab1 = Titanium.UI.createTab({  
    icon:'KS_nav_views.png',
    title:'Tab 1',
    window:win1
    });

    var win2 = Titanium.UI.createWindow({  
    title:'Tab 2',
    backgroundColor:'#fff'
    });

    var tab2 = Titanium.UI.createTab({  
    icon:'KS_nav_views.png',
    title:'Tab 2',
    window:win2
    });

    //  add tab
    tabGroup.addTab(tab1);  
    tabGroup.addTab(tab2);  
    // open tab group
    tabGroup.open();

It gives a black margin at the bottom of the screen. Can't figure out how to place the tab group at the bottom of the screen. Is this a bug with the new appcelerator version or do I need to do something in the settings? I appreciate any help!

Some info:

  • I'm running on an iPhone 4S with iOS 7.1
  • Using titanium Appcelerator 3.2.1

Thanks!
