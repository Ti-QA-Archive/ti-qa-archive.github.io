  • Does the app work in Safari or Chrome? What version of Firefox are you running?

    To debug code in mobile web, you just use the normal debug tools that ship with the browser. In Safari, these can be brought up by going to Preferences->Advanced->Show Develop menu in menu bar, and then selecting Develop->Start Debugging Javascript. In Chrome, bring up the chrome menu (wrench icon) and then select Tools->Developer Tools. In Firefox, install the Firebug extension. In Internet Explorer, press F12.

    The code that is breaking is in an internal helper method used for event listening. In your developer tools, enable "break on exceptions." In Safari/Chrome, open the scripts tag, enable debugging, and select the pause on exceptions icon (octagon with the pause sign in it). In Firebug, enable the script tab, and then go to the console tab and enable the break on all errors icon (pause sign with a read crossout over the corner). This way you can get a stack trace that will tell us what the offending event handler is.

    I suspect that mobile web is trying to add an event listener to an object that doesn't support that event.
