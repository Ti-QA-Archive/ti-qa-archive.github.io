    Hello Deepan,

    something like :

    // create object

var win = Ti.UI.createWindow();
var button = Ti.UI.createButton();

win.add(button);
win.open();

// remove object and release memory 

win.remove(button);
button=null;

//the object could be removed by GC

    You can find more info about Memory Management see this :

    Managing Memory and Finding Leaks.

  • Hi

    The documentation gives you some great examples and how-tos to improve your garbage collection needs.

    • Managing Memory and Finding Leaks

    Unlike a web page a mobile app memory usage remains during the life cycle of the app itself. A web page - even a really cool one with lots of server side stuff) is mostly self contained.

    So in a mobile app there may be very good reasons why a routine in one section can continue even after the window it is attached to may have been closed. Event listeners are prime examples of this.

    With a few modifications to your code to handle your apps specific needs - you can clean things up fairly automatically.
