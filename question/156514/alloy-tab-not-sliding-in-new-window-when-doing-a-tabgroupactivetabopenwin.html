Alloy Tab not sliding in new Window when doing a TabGroup.activeTab.open(win)

Application type: mobile;
Titanium SDK: 3.1.2.GA;
Platform & version: iOS 6.1.3;
Device: iOS iPhone simulator;
Host Operating System: OSX 10.8.4;
Titanium Studio: 3.1.2.201308091617

I have a very simple Alloy Application, where I have 3 controllers:

index, login, dashboard.

index.xml

<Alloy>
    <TabGroup id="tabGroup">
        <Tab>
            <Require src="login">
        </Tab>
    </TabGroup>
</Alloy>

login.xml

<Alloy>
    <Window class="container" id="window">
        <ScrollView id="scroll">
            <ImageView id="image"></ImageView>
            <Label id="label"></Label>
            <TextField id="username"></TextField>
            <TextField id="password"></TextField>
            <Button id="button" onClick="signIn"></Button>
            <Button id="button2" onClick="openDashboard"></Button>
        </ScrollView>
    </Window>
</Alloy>

dashboard.xml

<Alloy>
    <Window class="container" id="win">
        <Label text="Testing"></Label>
    </Window>
</Alloy>

index.js only has the following to open the Tab Group

$.tabGroup.open();

login.js has the following:

if(!OS_ANDROID){
    $.window.setBarColor('#CC6600');
    $.window.setTabBarHidden(true);
    $.username.setBorderStyle(Titanium.UI.INPUT_BORDERSTYLE_ROUNDED);
    $.password.setBorderStyle(Titanium.UI.INPUT_BORDERSTYLE_ROUNDED);
}


function openDashboard(){
        var tabGroup = Alloy.createController('index').getView("tabGroup");
        var win = Alloy.createController('dashboard').getView("win");
          tabGroup.activeTab.open(win);
}

Now, all of the above code seems fine, but when the openDashboard function is triggered, the new Window is opened in the Active Tab, but it doesn't slide in….it just opens straight. Yet, there is a back button on the top left and when clicked, slides out the new Window that was opened.

I've done this multiple times in Appcelerator prior to alloy.js and it always worked fine. I can't understand what's different now.

  • Accepted Answer

    You are right var tabGroup creates a new instance of your index view. You can fix the problem by adding the tabGroup to a global scope in your index.js file as below.

    Alloy.Globals.tabgroup = $.index;

    and in your dashboard button click event open the window using the tabGroup from the globals.

    function openDashboard(){
        var win = Alloy.createController('dashboard').getView("win");
        Alloy.Globals.tabgroup.activeTab.open(win);
}
