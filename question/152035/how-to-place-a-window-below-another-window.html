How to place a window below another window

I have an alloy application with (simplified) 2 controllers and 2 views. View 1 = index, View 2 = settings. I'm trying to detect a swipe on the index window and then show the settings window BELOW the index window, by using an animation.

I've used the zIndex property (which is documented on the website here) on the windows, but they are ignored. The window I open last (in this case the settings window) will always appear on top of the index window.

Can somebody tell me how I can open the settings window BELOW the index window? I'm using alloy.

Thanks for helping out!

    Hi

    Whilst I have not done this in Alloy, the principle should be the same;

    • Create and open your settings window, leave the visibility as false
    • Open the main window above the main window
    • On the event of open for the main window - make the settings window visible

    Then simply use your animations. The idea here is the windows are added in order so the first one is the lowest in the visual stack, the most recent is the topmost.

    • I agree with the logic of memory handling, but I balance this with speed of UI.

      The app I use this on keeps a left window and right window "hidden" under the main window. These are revealed as required - but left in context. These "hidden" windows have a rich UI, but a considered one at the same time - nothing too heavy, well presented standard UI elements for example.

      The delay in loading and the animation of sliding starting is too much - particularly on older devices. Whilst the delay is minimal when the window is not loaded, it is still not perfect.

      Remember you can remove the contents of the windows when not in use if you want - that will remove most of the memory.
