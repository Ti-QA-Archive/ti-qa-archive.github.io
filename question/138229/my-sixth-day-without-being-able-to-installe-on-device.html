  • I finally solved this issue and I can now build to device. In my case, I investigated build.log and found the following error being thrown (but it was never printing to the debug console in Titanium Studio): UnterminatedRegularExpression() jspacker.UnterminatedRegularExpression.

    My offending code was:

    /regex_pattern_here/.test(subject);

    Changing it to this fixed the issue:

    var pattern = new RegExp(/regex_pattern_here/);
pattern.test(subject);

    Maybe Appcelerator changed something recently with how javascript is minified? In any case, if you're having this problem I suggest you review the bottom of iphone/build/build.log right before ** BUILD FAILED ** because it contains additional error messages that do not make it to the Titanium Studio debug console.

  • Last time I renewed my apple license, both the old and the new were active and installed simultaneously in my mac. This is due to the license is renewed a few days before the other expires, and this causes a lot of problems, because if you try to use a duplicated certificate compilation fails.

    Ensure you do not have duplicated certificates installed in your key chain. Even better, delete both (if both are installed), create a new certificate and regenerate the provisioning profile.

    Keep the keychain tool open and just before compile from Titanium Studio be sure that the old certificate is not installed again (I found that sometimes is installed automatically I don't know how). If you see that the old one appears, delete it ASAP. This way the compiler will use the new (and unique) certificate. A bit tricky, but this is what I had to do.
