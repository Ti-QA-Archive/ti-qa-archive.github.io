However, the tab bar is still on top. Am I missing something?

I am building my app using the 3.3.0.GA SDK, towards Android 4.4 (Genymotion + Galaxy S4).

  • Hello, Anton, Thanks for your Query. According to Link "Bottom Tabgroup will be supported up to Titanium 3.0, it will not be supported for Titanium 3.X"

    The reason because

    Before SDK 3.0, Titanium used TabActivity to implement the TabGroup. Since TabActivity is deprecated, Titanium now use TabHost to implement the TabGroup for API level < 11 in SDK 3.0. We define the TabHost layout in the code. That's why the custom TabHost layout xml file does not work anymore.
    Based on the guideline of Pure Android, the bottom tab bars are not recommended. In addition, for API level >= 11 we use ActionBar to implement the TabGroup, in which case tab bars can only be on the top. In order to provide a consistent experience, we don't recommend using bottom tab bars.

    There is a JIRA Ticket on the related issue which is now closed with comments that it won't be fixed.link

    You can Create a Custom tab. Here is a sample Tweetanium app that has custom tabs created in it. Take a look specifically at ApplicationWindow.js for the tab designs.

    Hope that helps. Thanks.
