  • I agree with Rainer. I think that Titanium may have gotten a bad reputation early on, probably due to a number of issues:

    • immaturity of the platform
    • lack of guidance for application architecture
    • the misguided perception that if you've ever written a few lines of Javascript in a crappy web page, you could now build robust mobile apps because Titanium makes it that easy

    I think that these issues have been largely resolved. Titanium has definitely matured and CommonJS and Alloy have emerged as two solid application architectures.

    That is not to say that building a large, complicated mobile app with Titanium is painless. You will certainly experience your share of frustration. But if I had to guess, I would say that you could build a cross-platform (android + ios) app with about 120% of the effort of building one of them natively.

    • If you want a cross platform app, you should build it cross platform from the very beginning, and you should test on both platforms daily. Otherwise, it will be extremely difficult to take your iphone-only app and get it to run on android. Not impossible, but painful. I've seen many developers bad-mouth Titanium because they built the entire app for iphone and then tried to port to android.

      Building for android takes a whole different mindset, which iphone users aren't accustomed to – you have to accomodate all sizes of displays, which can be very tedious. But this is something that native android developers are already used to doing.

      Given a choice between phonegap and Titanium, I would choose Titanium. Titanium uses true native UI elements. Your application logic is in Javascript, but the UI is native. This leads to a more natural experience for the end user, and likely better performance.

      As I understand it, phonegap relies on other mobile JS frameworks for the UI. I have found that such frameworks (like jquery mobile) are enticing, but ultimately disappointing. I love the idea, but in reality, there are too many weird artifacts, like footer bars scrolling with content, flickering on transitions, etc. I think it would be very hard to build an app that would really feel like a native app with such toolkits.
