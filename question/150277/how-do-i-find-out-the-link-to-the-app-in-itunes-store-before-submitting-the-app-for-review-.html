  • I prefer not to use app name for the link because it is subject to change and that would break someday.

    You can use this link:

    https://itunes.apple.com/app/id361285480?mt=8&at=10l6dK

    Change the number here "361285480" to the Apple ID you get from iTunes Connect after creating the app there.

    Pay attention that if you want to directly open the App Store/iTunes you must not remove the "at=…".
    Moreover, If you know this link would be open on a Mac only you can replace the "https" with "macappstore" so it won't open safari first and open directly the store (in case that's a mac app). on iOS 7 this problem was solved if you are using this link on older versions of iOS you shall try "itms-apps" instead of the "http" to not open safari before the app store…

    On your case I think "https" is the best (would work and open the store everywhere). Also, Pay attention not having country code on the link (saw other posts with it) that would make it direct to the specific store (would ask user to change store if he defined a different one there he would probably won't be able to download because his account is for different store).
    This is probably clear it al, if not comment here.
