A definintion of Single Context

I have been reading a lot about using a single context approach to avoid problems with your titanium apps.

All these articles make the assumption that you know what Single Context is.

I have, sort of, derived a definition based on the description.

Does single context refer to an app that has the entirety of its code reside within the app.js file?

This would mean that any object declared within that code would be global to the app?

That would mean reducing the use of modules, specially, 3rd party modules, to a minimum?

  • The code would not all have to be in app.js physically, but would be essentially pulled in using Ti.include at runtime.

    Yes, object declared within the code would be available throughout the app. To keep this from getting messy, one would declare a unique namespace to store them under.

    • A problem occurs with large apps, because everything is loaded at runtime, all at once. Slow startup, memory issues, etc.

      A common namespace cleans up the organizational mess, but doesn't help with the startup/memory issues.

      CommonJS can solve the slow startup and memory issues. But you then need to work around 'global variables' (which really isn't all that tough to do)
