    Ok very good start you are handling your files really well and creating the namespaces and using anonymous functions.

    You don't need the include in the params test receive file…. You have already included the file in the app.js file. What happens is if you include the file using the anonymous function it puts the contents available to the rest of the app.

    Your abstracting the code into functions which is also good.

    Also put a return in your receive file at the moment it will return the slider and you need to return the window so return win;

    apart from that if this is your first attempt.. nice.

    Just don't pollute app.js with code create a controller file and control from there.

    Hope this helps.

  • Well to answer this.. Studio is a little buggy but yes it should show your functions as you type. But it doesn't always…. but it does sometimes. Studio is still under 6 months old so they are ironing out the bugs like this.

    Now the control file is actually no different to the files you have already created. It just has a function which is called from the app.js and this function actually does the initial load of the app, then when you press buttons etc you call a function in the control file to handle the processing you need.

    It depends how far you want to go I am in the process of writing a series of blog posts on how to format and abstract your code into an MVC architecture, it will be a few weeks though as I am a little busy at work.

    SO just keep going as your going and create separate functions as required in what seems like logical files for them to be in…

    Something like the slider in MVC would be a helper file.
