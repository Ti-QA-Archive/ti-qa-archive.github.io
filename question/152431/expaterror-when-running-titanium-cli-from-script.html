    • In your tiapp.xml, there is probably an <android> tag. You need to make sure this tag has a namespace like this:

      <android xmlns:android="http://schemas.android.com/apk/res/android"/>

      or you can set a number of things that go into the Android manifest like this:

      <android xmlns:android="http://schemas.android.com/apk/res/android">
    <manifest>
        <supports-screens android:anyDensity="false"/>
        <application>
            <activity android:alwaysRetainTaskState="true"
                android:configChanges="keyboardHidden|orientation"
                android:label="testapp"
                android:name=".TestappActivity" android:theme="@style/Theme.Titanium">
                <intent-filter>
                    <action android:name="android.intent.action.MAIN"/>
                    <category android:name="android.intent.category.LAUNCHER"/>
                </intent-filter>
            </activity>
        </application>
    </manifest>
    <activity android:name="ti.modules.titanium.ui.android.TiPreferencesActivity"/>
    <tool-api-level>10</tool-api-level>
    <services>
        <service type="interval" url="testservice.js"/>
    </services>
</android>

    • That's possible. I recommend only modifying the XML nodes you need to and not touch the top level nodes.

      FWIW, we created a command "titanium project" to get/set certain tiapp.xml properties including the id and deployment-target. Unfortunately, due to technical complications and time constraints, the command is currently read only. The first pass at modifying the tiapp.xml caused all comments in the file to be wiped out. No good. Maybe someday we'll be able to fix it.

      On a side note, you may want to consider writing a Titanium CLI plugin. Plugins can contain one or more commands and/or hooks. Hooks can tie into the build process and you have direct access to the parsed tiapp.xml object, which is JSON. For example, in your project directory, create a folder "plugins/myplugin/1.0/hooks", then create a JS file called "myhook.js" or whatever you want. Put this in it:

      exports.cliVersion = '>=3.X';

exports.init = function (logger, config, cli) {
    cli.addHook('build.pre.construct', function (finished) {
        // this before the build has even started
        logger.info('changing app name');
        cli.tiapp.name = 'My new app name';
        finished();
    });

    cli.addHook('build.pre.compile', function (build, finished) {
        // build state is created, but we haven't started compiling
        // yet.
        logger.info('changing app name again');

        // notice we now reference the tiapp via "build", not the
        // "cli" variable
        build.tiapp.name = 'My new awesome app name';
        finished();
    });

    cli.addHook('build.post.compile', function (build, finished) {
        // build is done!
        finished();
    });

    cli.addHook('build.finalize', function (build, finished) {
        // last chance to do any cleanup
        finished();
    });
};

      Then you can enable your new plugin in your app's tiapp.xml:

      ...
<plugins>
    <plugin version="1.0">myplugin</plugin>
</plugins>
...

      Disclaimer: this code is not tested, but probably works. Use at your own risk!
