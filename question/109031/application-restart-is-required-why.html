  • The issues each platform are equally easy to fix, because the troubleshooting procedure is the same. The iOS simulator, which is only possible to install on the mac, is quicker than the android emulator, which you may prefer. Let's dedicate this thread to the mac issues, then.

    Firstly, install android packages SDK 1.5 to 2.2, and APIs 3 to 8. Remove any honeycomb packages, as they are not supported.

    Then copy and paste the following into your terminal/shell so that all the commands execute successfully. Copy the output and paste it to your preferred pastebin service. It's important that you do not modify it in any way, as this could mask problems with your configuration. Do NOT paste it here! This is because the system will change the formatting.

    clear
cd /
uname -a
echo "$PATH"
echo "$JAVA_HOME"
java -version
javac -version
python -V
echo "$android_sdk"
adb version
android list target
git --version
echo " --- The following is optional ---"
scons -v
echo "$HTTP_PROXY"
echo "$HTTPS_PROXY"
echo "$SOCKS_PROXY"

    Also, take a screenshot of your android SDK manager "installed packages" screen, and share it.

    Once you have provided this information, I will try to help you further.

    Cheers

    • Hi David

      I am sorry for the delay. Regarding your changes, you seem to have lots of duplicates in your path now. You should correct this.

      Is your android sdk really in the directory &#x2F;android_sdk&#x2F;? If not, you need make sure your paths reflect its true location.

      Bearing in mind the previous point, you need to create the environment variable android_sdk set to &#x2F;android_sdk&#x2F; (if the path in the previous point is correct).

      Is your Java JDK really in the directory &#x2F;java-jdk&#x2F;? If not, you need make sure your paths reflect its true location.

      Bearing in mind the previous point, you need to create the environment variable JAVA_HOME set to &#x2F;java-jdk&#x2F; (if the path in the previous point is correct).

      If none of the above problems existed, I would have advised you that your PATH variable should have been optimized by reorganizing the following:

  • If the message "application restart required" is popped up after installing the application, there is the following work around described in https://jira.appcelerator.org/browse/TIMOB-9285: add the following to your tiapp.xml:

    <property name="ti.android.bug2373.finishfalseroot" type="bool">true</property>
