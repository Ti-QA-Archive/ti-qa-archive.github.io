Image from Ti.Filesystem as button background

Hi,

I have a button which calls Ti.Media.openPhotoGallery on click and lets me select image from my gallery.
On success handler, I am trying to set that image as the same button's background. It is working perfectly in IOS, but not in android.
This is my openPhotoGallery success handler :

//////////////////////////////////////////////////////////////////////////////////////////////////////////////////////////////////////////////////////////////////
Ti.Media.openPhotoGallery({
success : uploadImage
});
//////////////////////////////////////////////////////////////////////////////////////////////////////////////////////////////////////////////////////////////////

function uploadImage(e) {
var media = e.media;

var file1 = Ti.Filesystem.getFile($.datefield.text + '.png');
if (file1.exists()) {
    file1.deleteFile();
}
file1.write(media);

$.selectimg.backgroundImage = $.datefield.text + '.png';
$.selectimg.title = 'Change..';

}

What do I have to make it work for android too.
I have tried
$.selectimg.image = $.datefield.text + '.png';
No luck.

Any help is appreciated. Thanks..

  • Accepted Answer

    Use the nativePath property of the file1 object for both platforms:

    $.selectimg.backgroundImage = file1.nativePath;

// untested, but per the docs, you can access the path
// to the media selected from the gallery with the following
// though you might have permission issues accessing it

$.selectimg.backgroundImage = media.file.nativePath;

    Keep in mind that it will be drawing the entire full-size image at the scale of your button's dimensions, which is memory inefficient. You could run into out-of-memory issues. 

    var tmpThumbFile = Ti.Filesystem.createTempFile();
tmpThumbFile.write(media.imageAsResized(btnWidth, btnHeight));
$.selectimg.backgroundImage = tmpThumbFile.nativePath;

  • Hi Vanisree, i am not sure why are you writing the image first & then setting it as background of the button.

    However, if you just want to set the image from file gallery as button background, you could do this by removing few lines from your code:

    // open gallery to take image
Ti.Media.openPhotoGallery({ success : uploadImage }); 

function uploadImage(e) {
    var media = e.media;

//var file1 = Ti.Filesystem.getFile($.datefield.text + '.png');
//if (file1.exists()) {
//    file1.deleteFile();
//}
// file1.write(media);

    $.selectimg.setBackgroundImage(media);       // just use media variable as it will be holding the gallery image in blob format
    $.selectimg.title = 'Change..';
}

    • Button.backgroundImage accepts a string per the docs. In my experience, iOS will work with a blob but not Android. I was working with images from the camera not the Gallery. So, maybe it works differently with the Gallery picker.

      If you do use the media blob, however, you are shoving the entire large image into the small button. Essentially, you're taking an 8 megapixel image (typical digital camera file these days) and putting it into a button that's a few hundred or thousand pixels. This, as I said above, is memory inefficient and will lead to crashes. Don't do that.
