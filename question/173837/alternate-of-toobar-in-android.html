    Hi

    There is no direct alternative available into to Android. There are other ways of course using things that are available to Android but the use of which one is dependant on your app needs, user interface wishes and user interaction requirements.

    Here are a few ideas;

    • Buttons in a view

      The most obvious alternative is to simply create a view with the same number of buttons that the iOS would have - or would have if you were building for iOS and using the toolbar. You would need to handle the responses and pass them back as something simple, but that is not too hard.

      In this example and in Android apps - these would normally be placed at the bottom of the window and evenly spread out, if two buttons each taking up 50% of the space allowing for margins of course.

    • OptionDialog

      Allow the simply presentation of a small number of choices (although more - thinking specially for this right now). The choice is returned in the same simple way as the toolbar. However the UI for this requires the user to actively make this dialog appear - the toolbar is always visible, two entirely different things.

    • Picker

      Like the OptionDialog but I am not a fan, same issues as OptionDialog - not being in-line.

    The answer will be down to your apps final usage requirements. The most common would be the buttons in a view - the choice is your though.

  • For all based on my above answer I have created a custom view and put two buttons. It works as accepted.

    XML

    <View id="buttonView" backgroundImage="/Images/btnImg.png" platform="android"> 
    <Button id="cancel" title="Cancel" left="0%" height="100%" width="35%" onClick="ClosePicker_type" />
    <Button id="Done" title="Done" right="0%" height="100%"  width="35%" onClick="SelectPickerValue_type" />
</View>

    TSS

    "#buttonView[platform=android]": {
    width:"100%",
    bottom:"60%",
    height:"25%",
    top:"0%",
}
