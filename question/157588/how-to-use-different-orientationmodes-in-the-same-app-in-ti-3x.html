How to use different orientationModes in the same app in Ti 3.x

I have an app that is mostly used in PORTRAIT mode, but there is one specific window that must be in LANDSCAPE, I've tried using the code bellow, but it does not work, and setting a orientation for manifest.xml is not an option in this case as I need 2 orientations, but only one enable for each window.

ps: It used to work on TI 2.x, after 3.x it's not working anymore.

var win1 = Titanium.UI.createWindow({  
    title:'Win 1',
    backgroundColor:'#fff',
    orientationModes: [Ti.UI.PORTRAIT],
});
//win1.orientationModes = [Ti.UI.PORTRAIT];

    I have an app I have just finished updating to 3.x on both Android and iOS and it works like a charm. On the window I have:

    fullscreen: true,
navBarHidden: true

    Maybe that can make a difference?
    Otherwise you might have to play with setting the orientation for each activitity in the manifest section of tiapp.xml.

  • I was having this issue as well. The code above helped me out when I added it to the display calling the window I wanted opened. Since I am creating the window on that page, I added the orientationMode and fullscreen: false to the createWindow and was able to lock in portrait mode for the opened window. It wasn't until I added the fullscreen: false that it started working.

    sigCapBtn.addEventListener('click', function(e)
{
    Ti.UI.Android.hideSoftKeyboard();
    var newView = Ti.UI.createWindow(
    {
        title : 'Sig. Capture',
        url : "sigCapture.js",
        woID: Ti.App.currentWorkOrderId,
        orientationModes: [Ti.UI.PORTRAIT],
        fullscreen: false
    });

    newView.open();
});
