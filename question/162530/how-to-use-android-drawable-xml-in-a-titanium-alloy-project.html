How to use Android drawable xml in a Titanium Alloy project?

I'm using Titanium Studio 3.2.0 and Titanium SDK 3.2.0.GA, my OS is MacOSX and my deploy target is Android.

I want to use the Buy with Google drawable resource provided by Google into my project. I added it to my res folder in my project (ProjectRoot -> platform -> android -> res) and then I tried to add it to an ImageView like this:

    var googleWalletButton = Ti.UI.createView({
        width : Ti.UI.FILL,
        height : '30dp',
        backgroundColor : 'red'
    });

    var googleWalletDrawable = Ti.UI.createImageView({
        image : Ti.App.Android.R.drawable.wallet_button_background
    });
    googleWalletButton.add(googleWalletDrawable);
    $.BuyCreditsWorkspace.add(googleWalletButton);

But the drawable isn't loading. I checked the documentation on Ti.App.Android.R and the documentation on ImageView, and while my access to the drawable is correctaccording to the Ti.App.Android.R documentation, it seems ImageView doesn't support drawables defined in a xml like this.

Is there a way to use drawable resources defined in xml, like the Buy with Google, in a Titanium project? How can I achieve this?

  • Hi,
    Under the platform folder you can include an "android" subdirectory, and then any of the resource directories such as ("res/") layout, values, color, etc. You can place your resources similarly, as placed in this module under "platform/res" directory:
    https://github.com/appcelerator/titanium_modules/tree/master/facebook/mobile/android/platform/android/res

    Implementation of this can be seen in the LoginButton class, for example look into the LoginButton.java class at line 223 in below link:
    https://github.com/appcelerator/titanium_modules/blob/master/facebook/mobile/android/src/com/facebook/widget/LoginButton.java
    Here color and size are set using the resources. Now in the Utility.java class that you can find below:
    https://github.com/appcelerator/titanium_modules/blob/master/facebook/mobile/android/src/com/facebook/internal/Utility.java
    At line 415 you can see the properties are being set through getting the resource values using getIdentifier method which return a resource identifier for the given resource name.
    http://developer.android.com/reference/android/content/res/Resources.html#getIdentifier(java.lang.String,%20java.lang.String,%20java.lang.String)

    Regarding the question with setting the custom layout I have prepared a sample code that sets the basic button included in the module under platform/res/layout directory

    platform/android/res/layout/samplelayout.xml

    <?xml version="1.0" encoding="utf-8"?>
<LinearLayout xmlns:android="http://schemas.android.com/apk/res/android"
              android:layout_width="fill_parent" 
              android:layout_height="fill_parent" 
              android:orientation="vertical" >

    <Button android:id="@+id/button"
            android:layout_width="wrap_content"
            android:layout_height="wrap_content"
            android:text="Hello, I am a Button" />

</LinearLayout>

    ExampleProxy.java

    @Override
    public TiUIView createView(final Activity activity)
    {           
                class MyImageView extends TiUIView 
                {

                    public MyImageView(TiViewProxy proxy) 
                    {
                        super(proxy);


                        LayoutInflater in = (LayoutInflater) getActivity().getSystemService(getActivity().LAYOUT_INFLATER_SERVICE);
                        int id = getActivity().getResources().getIdentifier("sample_layout", "layout",
                                getActivity().getPackageName());
                        View v = in.inflate(id, null);
                        setNativeView(v);
                    }

                    @Override
                    public void processProperties(KrollDict d)
                    {
                        super.processProperties(d);
                    }

                }

                MyImageView imageView = new MyImageView(this);
                return imageView;
    }

    Take reference from https://github.com/rupeshsharma/customresourceintegration and change the following mentioned above and see.
