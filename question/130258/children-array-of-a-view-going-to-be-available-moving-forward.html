.children array of a view going to be available moving forward?

It seems pretty well documented that the .children array is gone in 1.8.X on tableViews and tableRowViews. This has caused a large delay in time and cost a good deal of money for my companies project. As a workaround we are no longer using tableViews or tableViewRow and have moved forward building our own solution with by adding multiple views to a scrollView. On the dynamically added views we add the necessary UI elements such as imageViews and labels. The solution seems to be working but it depends on myView.children[x].text being able to be manipulated where children[x] is a label.

I have seen many examples and tutorials where people are telling users to use something like below to get a particular child of row in a tableView.

tableView.data[x].rows[y].children[z].text

I don't see children listed as the property of a view, nor do I see getChild or getChildren or the like as methods.

Will view.chidren[] be supported moving forward?

Thanks!

  • I missed that in the tutorial that Aaron was using an undocumented API - I edited that tutorial so I should have caught it. Rather than using .children, I would suggest looking at the Kitchen Sink example where various source elements of the table view row are being clicked, and a table view row is swapped out for a new one.

    https://github.com/appcelerator/KitchenSink/blob/master/Resources/examples/table_view_layout_2.js#L172

    If that doesn't satisfy your use case, we can explore other techniques. But given that it's an undocumented API, I think that's why it hasn't been uniformly implemented. It looks like there's also an open ticket on this issue:

    http://jira.appcelerator.org/browse/TIMOB-6710

    • You can make a custom property, for example called HiddenDeleteImage, of the TableViewRow as you're creating it. Then instead of iterating over "children", just address the property directly.
