    It can be interpreted this way: If "b" equals (==) "true", evaluate second operand (the one after "AND" condition), that is - call "b" as function with parameter "c" and I think that statement will return what comes as a result of "b" call.

    The thing is that interpreters and compilers in many programming languages try to optimize things. For "AND" operator, second operand doesn't need to be evaluated if the first one returns "false" (that's obvious from "AND" logical operator's truth table - "false" && anything = "false").
    For "OR" operator ("||"), second operand is evaluated if the first one is equal to "false".
