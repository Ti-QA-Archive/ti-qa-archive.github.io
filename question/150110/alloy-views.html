Alloy Views

I am trying to make a simple slideover menu with a menu button to call over the menu View in Alloy. I have tried to get the menu view to animate with no success. I can get it to work if I include all my code within one controller, but then that defeats the whole appeal of Alloy to me. My code:

index.xml:

<Alloy>
    <Window id="container">
        <Require src="menu" />
        <Require src="home" />
    </Window>
</Alloy>

index.js:

$.container.open();

home.xml:

<Alloy>
    <View class="fullview">
        <View id="navbar">
            <Label id="menuLabel">MENU</Label>
        </View>
    </View>
</Alloy>

home.js

$.menuLabel.addEventListener('click', function(e){
    var menu = Alloy.createController('menu').getView();
    menu.animate({ left:0, duration:500});
});

menu.xml

<Alloy>
    <View id="menuwrapper" />
</Alloy>

It appears to create the controller object alright, and it recognizes the function animate on the resulting object, it just doesn't do anything.

  • Accepted Answer

    You are creating an instance of the "menu" in your index.xml which is automatically getting added to your UI. When you programmatically create an instance of "menu" in home.js, you are creating a second instance that you are never adding to the UI. You aren't referencing the existing instance when do that.

    Here's a simple example of two loosely-coupled controllers communicating to achieve what you are trying to do here. Notice that I included the style properties inline in the XML. This is not best practice, they should be in the TSS files, but I did that here to make the example more succinct.

    index.xml

    <Alloy>
    <Window backgroundColor="#fff">
        <Require src="home" onOpenMenu="openMenu">
        <Require src="menu" id="menu"/>
    </Window>
</Alloy>

    index.is

    function openMenu() {
    $.menu.getView().animate({left:0,duration:1000});
}

$.index.open();

    home.xml

    <Alloy>
    <Label color="#000" height="Ti.UI.SIZE" width="Ti.UI.SIZE" onClick="doClick">SHOW MENU</Label>
</Alloy>

    home.js

    function doClick(e) {  
    $.trigger('openMenu');
}

    menu.xml

    <Alloy>
    <View backgroundColor="#500" height="200" width="200" left="-200"/>
</Alloy>

    • To keep it loosely-coupled, yeah. At any point you could decide to expose a particular component globally so that you could access it directly, but this is not something I would suggest for a couple reasons:

      • Just in general, globals should be avoided unless necessary
      • Creating global references to UI components leads to the potential of memory leaks. If you forget to clear these references, they will not be freed by garbage collection.
      • You are undoing the loose coupling inherent in the process of breaking your views/controllers into separate files.
