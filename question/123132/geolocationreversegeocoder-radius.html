Geolocation.reverseGeocoder Radius

Hi
I would like to know if it's possible to define a radius for reverseGeocoder. My goal is to generate random reachable points in a circle around the user.

Currently, I generate 50 random points and for each point I do reverseGeocoder to find a reachable location but it's too slow and it generates too many queries.

So here is my question. Is it possible to define a radius for reverseGeocoder ? The idea would be to do a reverseGeocoder using the user location and take random index of the result to have XX reachable points.

If you have another idea to find XX random reachable point around a target i'm interested :)

Thanks
