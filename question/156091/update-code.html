  • Hi

    Firstly be aware that most Android apps are now automatically updated by the Android OS automatically and usually within a week most of the users of the previous version will be on the new version. You can check this out in the Android developer console and watch as the first and second versions swap over installation numbers in the next few days.

    There are two ways I use to achieve what you are trying to do, neither of which you can do now as you need to plan ahead for these to work. You can however make the changes in your app and future users will get these changes.

    • Push Notifications
      If you use push notifications, add a tag with the version number included in addition to whatever data values you use. Then when you have a new version you simply send out a push notification to the tag you added for the previous versions. So if you have a "v1" and then a "v2" - you send a push notification to "v1" saying there is a new version.

    • Update Poll checking
      Have a web service API that you call from your app periodically that returns the current public version of the app, if the returned value is greater than the one on the current app - tell the user there is a new app.

    Both of these techniques are simple, but both require work. It may turn out that the automatic updates that occur by Android itself are sufficient.

    Keep in mind that NONE of these solutions will force an update as soon as you publish one, there is always going to be some that take a while and some that never update.
