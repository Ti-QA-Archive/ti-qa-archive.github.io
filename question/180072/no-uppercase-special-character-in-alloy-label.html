No Uppercase Special Character in Alloy Label

Hi,

In my App I have a label which displays the name of a person all capitalized.

To do this, i use code like this:
var name = 'tonio';
$.labelId.setText(name.toUpperCase());

If the name contains german special characters like Ü, the label converts it to U.
Example:
var name = 'toniü';
$.labelId.setText(name.toUpperCase());

Label is like: TONIU

If no toUpperCase() is called, everything looks fine.
.setText('Ü'); does not work as expected
.setText('ü'); does work as expected

If I print out the name on the console, it displays the correct result:
Ti.API.info(name.toUpperCase());
= TONIÜ

This used to work in earlier Versions.

Any ideas???

Application type: mobile
SDK: 3.4.1
iOS Simulator 8.1, ipad retina
Host Operating System: OSX

    Are you sure that it converts Ü to U? I believe it may only cuts the dots off because the text it higher than the labels height. Could that be a possible reason? Have you tried logging the label value after applying the name? like

    $.labelId.setText(name.toUpperCase());
Ti.API.info($.labelId.text);

    I'd suggest to increase the labels height in that case (if it logs the correct string). Or use a smaller font?

  • Hi Tonio,
    I am not sure if this defect is solved for you , but if it isnt here is the fix for a similar problem i had .
    PROBLEM : I was a using custom Font - GothamItalic,Bold . Any uppercase display of the in spanish , truncated the diacritics .
    THEORY(Boring !!!) : There is some theory you need to understand before solving this . The problem is some custom fonts need to have the ASCENDER set to a higher value for the diacritics to display . so most of them might assume you use english always and hence the discrepanices. Your character belongs to "DIAERESIS" category and hence it needs a higher ascender value

    SOLUTION : 1) Download a software called GLYPHS(https://www.glyphsapp.com/) , it comes with a 30 day trial.(TrueType based fonts are converted into cubic béziers by glyph )
    2) Open your ttf file in GLYPHS
    3) Click on File->Export
    4)It generates a new ttf file (you can customise it for PostScript name , destination etc)
    5)Open the ttf in "Font Book"(prebuild mac software) to get the new PostScript Full Name
    6)Put this new ttf in your project and use the new PostScript Full Name .

    Thats it !!!

    My code snippet looks like this now :

    "#topBandTitle":{
    font:{
        fontSize: 22,
        fontFamily: Alloy.Globals.Language === 'en' ? 'Gotham Bold' : 'Gotham Bold Spanish',
    },

    'Gotham Bold Spanish ' - Thats the generated
