    It is currently incomplete, but the way I saw this being implemented starts here.

    When that event is fired, it should also pass data from the tweet along with it, because in the table view click event it should have access to all the data in the row. Then in the drawer view here, you would need to pass some data into the TweetDetailsView. That's what currently isn't implemented - the TweetDetailsView needs a function on it which will update it's self with the given tweet details.

    Also, generally speaking, you do not want to attach event listeners to every individual row in a table, as that may not perform well for large numbers of rows. Use the click event listener on the table instead.

    • well, not exactly - I think you have some trial and error ahead of you on the plumbing code (of transferring tweet data around), but here's how you add a function to the object:

      tt.ui.createTweetDetailsView = function(_args) {
    var view = Ti.UI.createView({backgroundColor:'blue'});

    view.update = function(tweetData) {
        //update the labels or whatever you've added to view
    };

    return view;
};

      And then inside the drawer view, you can call the update function on the TweetDetailView object you created, passing in the tweet data needed to update the detail view.

  • I recommend using some global variables. In your app.js just create an object that is global.

    var global = {};

    Then you can put data into it from anywhere for later use anywhere else. 

    (function() {
    // This is a window js file.

    global.something = 'something';
    global.an_object = {
        blue: '#0000ff',
        red: '#ff0000'
    };
})();

    Then from another window you should have access to that info via 

    (function() {
    // This is another window js file.

    Ti.API.info(global.something);
})();

    This method has been most versatile for me.

    Warning: using Ti.App.something … has weird limitations (like only supports one level deep). Ti.App.Properties would work but you have to convert objects to strings for storing and reverse them on reading (with JSON.stringify), this is not so in my global method above.
