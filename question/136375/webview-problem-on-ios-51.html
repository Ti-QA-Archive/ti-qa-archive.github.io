WebView problem on iOS 5.1

Hi.
I have a problem. I'm trying to use webView in Titanium mobile sdk 1.8.2, HTML for which is sended from server. By default height of this view is 0, and after receiving I calculate it like this:

    var myTimer = setInterval(function () {
        var actualHeight = webView.evalJS("document.height;");
        if(actualHeight > 1)
            {
                webView.height = actualHeight; 
                myTimer = null;          
            }
    },1000);

Everything is ok, and I can scroll it vertically, but when I'm trying to scroll it horizontally, application crashes. When I'm trying to debug it in xCode, console displays message "Received memory warning", without any level of warning. It works fine on Android and iOS 4.3, but problem appears on 5.1.
Thanks in advance

  • I think you should clear timer when calculation finish:

    if(actualHeight > 1) {
    webView.height = actualHeight;
    clearInterval(myTimer);
    myTimer = null;
}
