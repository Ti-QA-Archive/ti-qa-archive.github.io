Titanium IDE Build Configurations - iOS 7.1 Unavailable

Versions:

  • Titanium Studio: 3.3.0.201403071406
  • Titanium SDK: 3.2.2
  • iOS SDK 7.1

Issue:
When using the IDE to try to build for iOS, the only iOS SDK available in the build dialog is 7.0.3. If I try to build using the IDE, I receive an error

[ERROR] : Invalid "–ios-version" value "7.0.3"
Accepted values:
7.1
For help, run: titanium help build

I have all recent Appcelerator updates installed, and have all updates from Apple. I can build using the CLI, but I cannot build using the IDE. I have read as many QA posts as I can, but I still am unable to get the build dialogs to recognize iOS SDK 7.1.

Any and all help is greatly appreciated.

Thanks,
Bruce

  • Accepted Answer

    This is the resolution: https://jira.appcelerator.org/browse/TISTUD-6250.

    In my case it was a step more involved as the cached files were not visible, which I resolved (on OSX Maverick, differs in earlier versions), in a Terminal window with:
    defaults write com.apple.finder AppleShowAllFiles -boolean true
    killall Finder

    If files are still not visible, hover over Finder and alt>right click>refresh

    In Studio:
    Project > Clean,
    Run Configurations > delete (Titanium iOS simulator)
    Close Studio

    deleted (the now visible) files at <titanium_studio_workspace>/.metadata/.plugins/com.appcelerator.titanium.core/*

    restarted studio.

    Note that if Studio is not closed prior to deleting the files, cleaning the project causes the issue to reoccur.

  • In TI Studio IDE go to Menu item Run >> Run Configurations and choose the Configuration you would like to build for. Check to see if 7.1 is set in the Select SDK Version box. If it is not, change it to 7.1 and click the Apply button. You should be able to build for iOS SDK 7.1. Don't forget to run a "Project Clean". I also delete the Resources folder in my project. I'm not sure this is necessary but it has helped in the past. The Resources folder will be recreated during your next build.

    Cheers
