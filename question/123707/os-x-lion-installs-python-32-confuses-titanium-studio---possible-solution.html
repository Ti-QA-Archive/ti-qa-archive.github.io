OS X Lion installs Python 3.2, confuses Titanium Studio - possible solution

After installing Lion I had trouble building and running Titanium Studio. I would see errors similar to this:

File "/Library/Application Support/Titanium/mobilesdk/osx/1.7.2/project.py",
line 13 print "Usage: %s <name> <id> <directory> [iphone,android]
[android_sdk]" % os.path.basename(args[0]) ^ SyntaxError: invalid syntax

After searching I think I discovered a solution and things are working again for me. Here's what I did…

  1. To confirm which version of Python you're running open up a command shell (search in Spotlight for Terminal) and type "python". Under Lion you will probably see "Python 3.2"

  2. Check to see if you have other versions by going to /usr/bin and searching for python with "ls -lt python*" Hopefully you will see a symlink to a 2.6 version and the path to where it is.

  3. Go to your home directory via "cd"

  4. Edit your .bash_profile ("vi .bash_profile")

  5. Comment out the existing PATH setting for the newer Python and change it to the version that Titanium likes (2.6) with a line similar to:

    PATH="/System/Library/Frameworks/Python.framework/Versions/2.6/bin:${PATH}"

Restart Titanium and rebuild a project. This originally started as a question but after getting things to work it became a question-and-answer. If you have any comments or suggestions for doing this better, please let me know. For now, though, I'm happy to be building again.
