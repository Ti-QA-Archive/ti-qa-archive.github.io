  • Accepted Answer

    Mobile provision files are automatically installed on system when you double click after downloading them from provisioning portal, or by titanium if you did not manually. You can see all installed profiles with xCode Organizer and delete them from there (also, you can see the files doing righ-click over the profile and selecting "reveal on finder").

    Ensure you are using a DISTRIBUTION profile, and not a DEVELOPMENT profile and that you have changed it properly on titanium. If not, go to provisioning portal, create a distribution profile and install it correctly. And be sure you are using the DEPLOY option, and not the RUN option if you are using Titanium Studio. These are common errors, but maybe is not your case… tell me something about.

  • I hit this yesterday… while the provision has a human rememberable name like myproject.mobileprovision, it appears Ti is looking for it by the UID it contains, so rename the profile file that it should be looking at, to the name its not finding and it will go. (ie from myproject.mobileprovision to f2eca437-def….mobileprovision)
