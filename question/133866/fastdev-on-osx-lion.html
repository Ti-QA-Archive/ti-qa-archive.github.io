FastDev on OSX Lion

Hi,

Seems like for some people FastDev just works.. Has anyone who it didn't just work for found a way to get it to play nice?

Myself and my colleague have never been able to get FastDev to work.
I've been converting my app from iOS > Android for some time now, and in the end I gave up on FastDev and turned it off in Tiapp.xml.

I am using a MacBook Air, I've tried 1.8.1 and 1.8.0.1.

FastDev does connect, I can see that it is pulling in the files etc.. But I assumed it was like LiveTanium which I have setup on iOS. That is if I change something in a file which is a commonJS module, if I go back an Activity / window and then click on a button that effectively 're-requires()' the module and it should load the new code. This never happens.. I always have to re-run the entire app which takes about a minute and is really slowing me down.

On images it often says ':not found' but I have read other people getting this in Q&A and most report that FastDev still works fine for them.
I have re-build, cleaned the project many many times. I have re-created the project many times, as mentioned it handshakes etc.. fine, it just isn't loading the new files.

I also notice that I have to re-start my app 3 times to get it to re-run but when I have FastDev switched off, if I press Home on the Android simulator and then re-run the debugger it re-loads the app first time. I get a black screen twice when I try to re-run with Fast-Dev turned on and then on the third attempt it seems to work.

Any advice much appreciated as I'd love to speed things up a bit..

    • Nope - I love my Pro ;-)

      regards windows / activities - It depends. Single context Apps in the CommonJS style will not get reloaded due to the caching layer that CommonJS applies. New contexts (loaded via the url window property) will also re-evaluate and reload javascript. Light weight windows wont. that seems to be the rule of thumb for me.
