Alloy model to get JSON from web service and display it in TableView

Is it ok if I implement the XHR calls in a utility file and then populate the data in a model file attached to a table view? What would be the correct way to implement it.

  • You'll likely want to make a custom sync adapter to handle your remote JSON. Take a look at the existing Alloy sync adapters to get an idea of how they should be structured. The properties and localStorage adapters are much simpler than the sql one, so you'll probably want to start there.

    [https://github.com/appcelerator/alloy/tree/master/Alloy/lib/alloy/sync](https://github.com/appcelerator/alloy/tree/master/Alloy/lib/alloy/sync)

    Aside from that, some of our community developers have already developed remote sync adapters for their own purposes. Here's 2 more from Aaron Saunders (ACS) and Mads Moler (REST) respectively:
    www.clearlyinnovative.com/blog/post/34758524010/titanium-quickie-a-simple-integration-of-alloy-and-wordpress-json-plugin#.UTIqH3xARi8
    https://github.com/aaronksaunders/ci.alloy.adapter.two/blob/master/app/assets/alloy/sync/acs.js
    https://github.com/viezel/napp.alloy.adapter.restapi
