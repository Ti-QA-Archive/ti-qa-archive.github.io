if statement

hello, i made an app with a game, but the game only works for the iphone. so i created a window which has a standard launch button, but when it is launched on mobile web or android it should give another button with other text on it, so i created this piece of code:

    var operatingsys = Ti.Platform.osname
    var mobileweb = "mobileweb"
    var android = "android"
    var ipad = "ipad"
    if (operatingsys=mobileweb)
    {
      win4.add(weberrorbutton);
    }
    else if (operatingsys=android)
    {
      win4.add(anderrorbutton);
    }
    else if (operatingsys=ipad)
    {
      win4.add(ipaderrorbutton)
    }
    else
    {
        win4.add(lbutton); 
    }

(i am not a great javascript developer)
but every time i launch the app (even on the iphone) it gives the weberrorbutton button, what did i do wrong?

    • +1 @ Ken

      Or more correctly ===

      if (operatingsys=mobileweb)
// says SET "operatingsys" to be the same value as "mobileweb"
if (operatingsys==mobileweb)
// says COMPARE if "operatingsys" EQUALS "mobileweb"
if (operatingsys===mobileweb)
// says COMPARE if "operatingsys" EQUALS "mobileweb" 
// AND they are both the same type

      Whilst the second and third both compare ONLY the third one === is the safest.

      The reason for this can be seen here;

      5 == 5 // true
5 == "5" // true
// but
5 === 5 // true
5 === "5" // false

      The number 5 and the string "5" are NOT the same type. Using === allows you to avoid logic bugs - which are VERY hard to find and get harder the bigger the app gets.

    switch(Ti.Platform.osname){
