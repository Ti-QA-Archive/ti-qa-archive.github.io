CFBundleShortVersionString is always 1.0,

Hey,

we got an app in the store built with the 1.8 SDK and wanted to push our first update. Unfortunately, the CFBundleShortVersionString always states "1.0" even though the version in tiapp.xml is 1.1.

I found the part in builder.py that sets the CFBundleShortVersionString, and it is correct in ~/Library/Developer/Xcode/Archives/…my_project…/Info.plist:

<key>CFBundleShortVersionString</key>
<string>1.1</string>

However, if I descend from there into the Products/Applications/My.app/ directory, the Info.plist contains a CFBundleShortVersionString of 1.0 instead of 1.1.

Can anyone here successfully build for the app store with xcode 4, titanium 1.8 and a CFBundleShortVersionSTring other than 1.0?

  • A workaround to my problem above was to

    • build the app
    • convert the binary Info.plist from Products/Applications/My.app/ in the archive directory to XML
    • store that xml in the project directory
    • fix the short version number
    • rebuild the app

    Funky…
