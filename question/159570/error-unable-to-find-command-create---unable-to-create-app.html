    • It almost sounds like the SDK is:

      1. Missing
      2. stored under bad permissions

      The following should force the reinstall of the latest (as of this post) and use it as the default.

      titanium sdk install 3.1.3.GA --force --default

      — commented November 18th 2013 by Stephen Feather

    • An issue similar to yours, was the missing 'build' command that occured for users that had already working ti installs but upgraded.

      Per Chris Barber, try to set owner ship of the Titanium support files to yourself using this:

      sudo chown -R `whoami` ~/Library/Application\ Support/Titanium

      — commented November 18th 2013 by Stephen Feather

    • glad we could figure it out. Sometimes the installers miss something, and with a system as complex as Titanium, it happens. Mark the answer correct so others can find your question w/ the correct answer quickly.
