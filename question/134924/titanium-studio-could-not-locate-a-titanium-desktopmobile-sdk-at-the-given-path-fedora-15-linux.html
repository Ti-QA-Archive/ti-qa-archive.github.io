Titanium Studio: Could not locate a Titanium Desktop/Mobile SDK at the given path (Fedora 15 Linux)

I installed Titanium Studio build: 2.0.0.201203291340 (Build: jenkins-titanium-rcp-master-169 (origin/master) - Date: 29 March 2012, 13:42:27) on my Fedora 15 Linux system. When run for the first time, Titanium Studio downloaded and installed Titanium SDK 1.8.2 under ~&#x2F;.titanium&#x2F;mobilesdk.

When I open the preferences dialog with the Window -> Preferences -> Aptana Studio -> Titanium command, the dialog displays this error at the top:

[Titanium SDK Home] Could not locate a Titanium Desktop SDK at the given path

and it marks in red the name of the Titanium SDK Home text field, which by default includes the string &#x2F;home&#x2F;paolo&#x2F;.titanium&#x2F;. The dialog also has the Apply and OK buttons grayed out. If I set the Titanium SDK Home field to &#x2F;home&#x2F;paolo&#x2F;.titanium&#x2F;mobilesdk&#x2F;, the error at the dialog's top changes to:

[Titanium SDK Home] Could not locate a Titanium Mobile SDK at the given path`

Also, the Titanium SDK Home field name remains red and the Apply/OK buttons are still grayed out. I tried providing different directories for the mobile SDK home, but the error is still there.

How can I fix the Titanium Studio configuration and get rid of the errors?
