Returning the currently displayed position of a mapView

Hi,

I'm having trouble getting the currently displayed position of a mapView. What I need is for a user to select a point on the world map, in order for the application to send requests to a remote server. I have tried using click/tap/doubleclick… listeners on the mapView but apparently, these events only work when targeted at annotations. I have thus turned my efforts to an external button, but then I have found no existing function enabling me to get the map's displayed position.

The following code is how I built my listener, with just an alert dialogue that doesn't show.

mapview.addEventListener('click', function(evt){
    Titanium.UI.createAlertDialog({title:'Click',message:'Click intercepted'}).show();
});

My Mobile SDK is 1.7.1, and the target Android SDK is 2.2

Is there any way of returning the mapView's displayed position ?

Thank you

Clement

  • Accepted Answer

    var current_latitude;
var current_longitude;

mapview.addEventListener('regionChanged', function(evt){
    current_latitude = evt.latitude;
    current_longitude = evt.longitude;
});

    this way the 2 vars for lat and lng will always have the center points lat and lng for you to use at will.
