My views are build with horizontal and/or vertical layout.

Any tips or best practices to increase perf ?

  • I would suggest you to create your views only when you need them. It is bad design to block the UI by drawing views that you may need later in application history.

    For example, the TableView is designed to only render rows that are displayed. When the user scrolls down, it render more rows, the upper rows may become destroyed.

    A bad descision would be to create thousands of rows first before displaying the table. Also the creation of detailViews would not make any sense before an element is selected in the master view.

    I recommend to create only the needed parts of the UI, then display that and in the background create more parts without draining the presented UI.

    This is just some thoughts. I probably have additions to make later.

  • Hello Pascal,

    there is no silver bullet to optimize/fine tuning any Titanium app. However, you can use this guideline in order to see if you can get a perfomant app (there are some nice tricks related to Graphic performance).

    In addition to that, you need to spend some time checking WHY your app is slow. Something to have in mind:

    • Data. Cache as much data as possible. Store them either in SQLite or JS arrays. Remember: Using JS data is cheap, the expensive part in memory consumption is using native components (like the graphical ones).
    • Use optimized images for your app - Try not to use rounded corners, but a background instead, by example.

    Best,

    Mauro
