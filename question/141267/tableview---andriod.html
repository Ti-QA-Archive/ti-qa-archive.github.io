  • Hi Steven

    I think part of the problem is that Android tables are slower than iOS and as there are so many different types of Android hardware is it easy to generalise.

    There are techniques to make the experience faster for the user, but at the end of the day until the underlying tables are improved at an operating system level there is only so much Appcelerator can do.

    For example; only show as many rows as are required at that time, use staggered, lazy or request methods to add to the table. Only use custom row layout is actually required. Make use of the property className if you are not already. Filter results, drill down results, if you are using images only use an image view if the standard leftImage cannot solve the problem. Provide default images for any view you use so the table can lazy load better.

    Also what hardware are you testing on? I have a few tests Android devices and get different results, I should expect my htc desire S is faster than my htc wildfire, so I should also expect my wildfire to be slower - seems logical but not all phones are equal. My Nexus 7 is faster than both of those, but my iPhone is faster than all of them. And none of those observations are using Titanium code - using the common apps - like; twitter, facebook, imdb etc.
