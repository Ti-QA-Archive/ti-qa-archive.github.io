Landscape only app not possible in Titanium

When building a landscape only app there seems to be no way to load the app in landscape mode and making the launch screen stay this way until the first window can be shown.
The launch screen rotates 90 degrees, placing it partly outside the screen, leaving a black area (in my case) on the right side.

For me this is a huge bug and failure. It means you cannot make landscape only apps in Titanium that follows Apples guidelines.

There is a workaround, but I'm not sure it will be approved by apple.
Launch the app in portrait mode, open a temporary window with an exact image copy of the splash screen. Set orientation to landscape and at the same time open up the real window. This looks pretty decent, but you can clearly see that the whole screen is rotated from portrait to landscape, as happens when the user rotates the device when having support for both orientations. This is fine when done by the user but doesn't look good when launching the app.

Any ideas or workarounds?
This is clearly a bug in Titanium since according to Apple you CAN launch an app in landscape mode and keep it that way.. Is there going to be a fix? Theres many threads about this problem, some up to 1 year old and nothing has been done about this.

  • But I think I know what it is…..
    Go to your [App]/build/iphone/Classes/ and locate the TiRootViewController.m
    Edit and locate around the line 144 the following code:

    [self manuallyRotateToOrientation:newOrientation duration:0];

    Comment that line by adding // to the beginning… save…delete your build folder (the one at the same level as the classes [App]/build/iphone/build

    And run…. I think this is what causes the App to change rotation when launching….

    win.orientationModes = [Ti.UI.LANDSCAPE_LEFT,Ti.UI.LANDSCAPE_RIGHT];

    This is all I did and it worked.
