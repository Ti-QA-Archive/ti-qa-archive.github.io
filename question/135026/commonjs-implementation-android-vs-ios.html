CommonJS implementation: Android vs iOS

Hi.

We created some sort of AppFramework for creating apps. We use this framework as a core to build different apps. Currently this AppFramework is used only in iOS. We are now working on version 2 which adds support for Android.

config.js in the AppFramework

// in app.js
var App = {
config:require("framework/config");
};

// in config.js
exports.applicationID = 0;

then in our project specific config file we override the 'framework's applicationID property, like this:

App.config.applicationID = 1;

After this, we call: App.start() and from that point we can always refer to App.config.applicationID and the value will always be 1.
On Android however, this does not work. The global var 'App' is not available from inside a CommonJS module.

What would be de perfect solution for this problem?
