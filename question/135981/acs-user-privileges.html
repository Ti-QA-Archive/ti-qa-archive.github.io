ACS User Privileges

I'm just starting to use the User objects of ACS, and it seems like there is no built in way to handle priviledges. For example, any user can register and then search the entire set of users with arbitrary data, exposing all of the other users data. I can understand that an admin type role could possibly have this type of access, but this pretty much makes storing user information in the User object equivalent to making it public and therefore useless if it needs to be controlled (as most users would expect). Unfortunately we HAVE to use the Users objects to get basic things like Push Notifications working with ACS, so this is kind of a blocker right now. Any known workarounds?

— asked April 25th 2012 by Ankur Oberoi (Developer)

    Currently you can goto app setting to disable user creation from API, we will have an website interface up soon that allow you to disable query/show to non-registered users.

    In the mean time, we are working on Access control list to allow custom access control. which will be released in a month or so.
