    I've never run into a charting library for Titanium that uses native views. That doesn't mean there isn't one.

    But I have one question: why are you opposed to a webview?

    There are so many HTML/CSS/JS frameworks for charting that produce very good looking charts. It would be hard to imagine a Titanium module generating charts as nice as jqPlot or Highcharts.

    You could try to roll your own, but you'd probably spend 3-4 weeks working on a general purpose library for charting, and then you probably would only have 5% of what these other libraries can bring you. And I think you'd either need SVG rendering or some other 2D graphics library, which kind of brings you back around to the WebView, which already has SVG support.

    There's no shame in using WebView. It's 5 million lines of code for a reason! :-)
