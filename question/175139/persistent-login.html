Persistent Login

I have set-up a user authentication system that interacts with a MySQL database to verify a users credentials and I am looking to store this information similar to a cookie so that I can implement a persistent login system. I looked at storing the information in a local database but it seems cumbersome for such a small amount of data. Any ideas as to how to approach this without using a local database? It would be ideal if the storage could have a expiration in the same way that a cookie would.

Thanks,
James

    There is something that acts in the same way;

    // check - as boolean (has default value if never set)
var loggedin = Ti.App.Properties.getBool('loggedin', false); 
// set - as boolean
Ti.App.Properties.setBool('loggedin', newstate);
