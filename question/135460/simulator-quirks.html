Simulator Quirks

Hello All,

I have an iPhone Titanium app that is pretty much ready to be tested adhoc. I have run into one or two simulator issues that I would like to try and understand before I go any further

My app is a hybrid - a Ti app which handles local notifications and has a two way conversation with a WebView that contains one of a collection of HTMTL5 jQuery Mobile Web Apps.

The issues I have run into

a.For some strange reason sometimes, not always, geolocation fails in the webapp with the Geolocation eror 2 - like there is no geolocation available on the "device"

b. Given that Ti does not let me repeat a local notification more than once a day I am using the last notification to schedule a new one. This was not working consistently so, in the webapp, I started using a setTimeout to schedule the next notification after a slight delay. For some reason - this repeat loop works x times and then the "application exits the simulator". Why on earth could this be happening.

Finally, I have come across an aricle that describes how to adhoc a Ti app

http://spitfiresky.com/blog/ad-hoc-distributing-a-titanium-mobile-app.html

However, he looses me at the Project -> TEst & Package -> Distribute step. My Ti setup does not appear to have any distribution options at all!

I'd much appreciate any help

  • I can't speak to your other issues, but as far as packaging an app for ad-hoc distribution, I've always been successful with the following:

    • Delete the build directory in your project directory
    • In Titanium, right click on your project, select "install to device"
    • Go through the steps of adding the provisioning file, etc.
    • After the process is complete, navigate to the build/iphone/build/Debug-iphoneos/ in your project directory
    • Your .ipa file will be in this folder ready for adhoc distribution
