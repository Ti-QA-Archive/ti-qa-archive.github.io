Listing of res-xxx directory on Android

Hi,

Using SDK 3.4.1 & 3.5.0 for a mobile project (iOS & Android), I would like to manage file icons, using file extension for each MIME type. For this, I have structured assets for Android as following :

  • assets/android/images/res-hdpi/mime types
  • assets/android/images/res-xhdpi/mimetypes
  • etc…

I have no problem when I use path like images/mimetypes/xls.png for an ImageView. But now, I would like to display a default image if the MIME type is unknown. No problem for iOS, using the defaultImage property, but on Android, this property does not exist. So I tried to use following code to discover all managed files :

var dir = Ti.Filesystem.getFile(Ti.Filesystem.resourcesDirectory + 'images/mimetypes');
var files = dir.getDirectoryListing();

But files always an empty String array.

PS : I also tried Ti.Filesystem.getFile(Ti.Filesystem.resourcesDirectory + &#39;images&#x2F;res-hdpi&#x2F;mimetypes&#39;); without success.

When I list images directory, I found only default.png file, the res-xxx directories are not found.

How can I list files in dependent res directories?
