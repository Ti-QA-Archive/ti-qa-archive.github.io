Dragging a row away from a TableView? ( ScrollView eats all touchmove events )

Hi,

I have seen some 10 related questions, all ultimately unanswered. But they are all too specific. I am going to make a more general question: How can I create a TableView from which I can drag Rows. I know that you can't actually move the row out, but you can draw some sort of ghost/proxy that can then be dragged around on touchmove.
The main problem with most failed approaches is that the ScrollView "eats" all the touchmove events whenever the touchstart event was originated within its bounds.
This makes it impossible to listen for touchstart or longpress events on a row and then use the touchmove events to move the ghost/proxy. They are nowhere to be found.

Thanks!

  • Hi Aldo,

    I'm having now the same problem, and seems that nobody can give a good answer.
    Still, I can give you a temporary solution(hack) that for me works:

    • on each tableViewRow, you have to place it's content inside a button (I'll explain below why a button);
    • you add the 'touchstart' event listener to that button, every time you create a new tableViewRow(probably inside a for loop, where you can create all rows);
    • inside the 'touchstart' callback you create a "shadow row" (a label that looks just like the table row) and position it just above the tableViewRow (you have the coordinates inside the callback);
    • you add 'touchmove' and 'touchend' listeners to the main window and you just check inside the callback functions a certain property of that lablel, just to be sure you react only when moving the desired label (Ex. e.source.name == 'dragLabel').

    This works for me at the moment, but I detected a very strange behavior if I use a label instead of the button, inside the tableViewRow: as you said, you cannot drag the "shadow row", because the scroll of the tableView "eats" the touchmove event. But this does not reproduce if you use a button….

    I'm still looking for a solution to this, since I don't want to have a button on each table row.

    If you don't really understand what I tried to explain here, let me know, I can provide you with some code snippets.

    Best Regards, Daniel
