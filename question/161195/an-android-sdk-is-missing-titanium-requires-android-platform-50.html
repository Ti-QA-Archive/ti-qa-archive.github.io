  • Hi, I had the same problem today since I updated to SDK 3.2.0GA.
    I am running Titanium Studio on Windows.
    When running 'titanium setup' I get the same message:
    selected Titanium SDK "vundefined" is not installed.

    I don't have a fix, but there is a workaround. It has to do something with the Titanium SDK Home.

    When I open Titanium Studio and start running the app on an android device, the 'preferences' box opens and it shows the path to my Android SDK, but it cannot find any 'default devices'.
    The path to the Titanium SDK home says: C:\Users\mvdijk\AppData\Roaming\Titanium\

    Then change that to C:\Users\mvdijk\AppData\Roaming\Titanium\mobilesdk\3.1.1GA and click 'OK' to close the preferences box.

    Then close Titanium Studio. Now start it up again. Of course it cannot find 3.2.0GA now, so it will ask if it must be installed. Click on 'Remind me later'.

    Then I go to 'Window -Preferences - Titanium Studio - Titanium.
    Hey, the preferences box now actually has found the Android devices and shows a list in the dropdown box 'Default Android SDK' ! Select the one you need.
    Now restore the Titanium SDK home to C:\Users\mvdijk\AppData\Roaming\Titanium\ again. Click on OK to close the preferences box.

    Run your app now and it will work just fine.

    This solution only works until you close Titanium Studio. The next time you start it up, you have to do the workaround all over again. But hey, a workaround is better than not being able to develop at all. :-)

    I don't know if this is a bug in the latest update, but I hope it will be fixed in the next update.
    Good luck!
