Zipfile?

Hello,

I have the zipfile module https://github.com/TermiT/ZipFile and it says it is installed and loaded when I do a quick debug build. However when I open the window with the

var zipfile = require(&#39;zipfile&#39;);

at the top, I get an error there? but it doesn't say anything just error at that line?

Any idea why?

  • Accepted Answer

    You shouldn't be referencing libzipfile.a directly. In your tiapp.xml you should have something similar to this (depending on what version you are running):

    <module platform="iphone" version="0.1.21">zipfile</module>

    Then make sure the zipfile folder is at:

    /Library/Application Support/Titanium/modules/iphone/zipfile/0.1.21

    Inside that folder should be the libzipfile.a (among other things such as module.xcconfig). If you are using a different version than 0.1.21, then replace my examples above with the version you are using.

    When you say "when you open the window with … at the top", are you repeating this from multiple windows which use the url property (*.js) to create a multi-context app?

    If so, put it in app.js and see if app.js is able to correctly unzip a file. If it can, then you are hitting some error from using it in multiple contexts.

    Make sure you can get it working from app.js first, then we can address multi-context issues if you still get errors in other windows.
