Android focus events (again) - onload?

Hi,
I have an app in the Android store which loads remote data into different tabs.

The first page gets 'latest' another tab gets 'nearby' etc etc

This to worked on the tab 'focus' event.

I am now working on the next version that addresses some minor bugs. But it seems that the focus event is no longer working.

In the meantime I have had to replace my computer due to a hardware failure but created a new app in Titanium and copied code into it.

The version of titanium has increased to 1.7 and the Android SDK stuff is up from 10 to 11.

Since focus events are so flakey what is the best practise for loading data when a tab or window is first shown…. i.e. before ANY USER INTERACTION as it would change the design alot to put a big button in the middle and arguably make it a pretty poor user experience.

Thanks,

Jez

  • To make this fire on load of the apps / tabs, i simply removed the event listener wrapper around my call to get the data.. .this means that the data does not update when you leave and return to the tab but it also means that the app functions and does not leave the user with a blank screen on certain tabs (not blank, but data free).

    A workaround, but really tab focus events need fixing up ..
