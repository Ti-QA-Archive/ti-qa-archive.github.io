binding model to tableview

target: simulator
Building using iOS SDK: 6.1
iOS iPhone Simulator: 6.1
device family: iphone

Hello,

I was going through the bounty hunter tutorial and trying to create a similar app for notes application using alloy. I am not able to update the status of the task to done using the set method. This is a portion of the view file I am using (todotask.xml).

<TableView id = "tasktable" dataCollection="tasks" backgroundColor = "transparent">
                    <TableViewRow id="row" dataid = "" backgroundColor = "transparent" model = "{alloy_id}">
                        <View id="tskrowid"/>
                    </TableViewRow>
                </TableView>

The model creation and saving it to storage part : 

function saveToDatabase(){
    Ti.API.info("Save button clicked!");

    var task = Alloy.createModel('tasks',{
        title : args.title,
        detail : $.detail.value,
        isdone : 0
    });

    tskcollection.add(task);
    task.save();

    tskcollection.fetch();
}

My table event listener code is as follows.

$.tasktable.addEventListener('click', function(p) {
    var dialog = Ti.UI.createAlertDialog({
    cancel: 2,
    buttonNames: ['Done', 'Delete', 'Cancel'],
    message: 'Change status of task?',
    title: ''
  });
  dialog.addEventListener('click', function(e){
    if (e.index === e.source.cancel){
      Ti.API.info('The cancel button was clicked');
    }
    Ti.API.info('e.cancel: ' + e.cancel);
    Ti.API.info('e.source.cancel: ' + e.source.cancel);
    Ti.API.info('e.index: ' + e.index);

    if(e.index == 0)
    {
        p.rowData.model.set({"isdone":1});
    }
    else if(e.index == 1)
    {
        p.rowData.model.destroy();
    }
  });
  dialog.show();
});

I am not sure about the line - p.rowData.model.set({"isdone":1}). It keeps giving me the error undefined
is not an object. I have tried p.row.model.set({"isdone":1}) also. Still its the same.

    Couple things:

    1. You probably don't need to fetch() after you add and save the task to the database. The UI should update without that step.
    2. id is irrelevant in a repeated row, since it will be used over and over again so you shouldn't really specify it. The only time that is useful is if the row itself was in its own controller.
    3. I don't know what model=&quot;{alloy_id}&quot; is supposed to be doing in your markup. If it is the id of a model, then you wouldn't call it like this:
      p.rowData.model.set({"isdone":1});
      you would instead get a reference to the model from your collection, then make the set call. Again this is assuming that alloy_id refers to the ID of the model.
      var model = taskcollection.get(p.row.model);
model.set({isdone:1});
