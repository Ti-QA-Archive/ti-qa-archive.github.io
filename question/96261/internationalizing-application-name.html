Internationalizing application name

Hi!

I'm struggeling with one important issue: The name of my application should be language-dependent. In German it should be "Küche", in English "Kitchen" (maybe there will also be a Spanish version).

Is it possible to internationalize the application name?
By the way: It seems that special characters in the application name (here: "Küche") lead to build problems.

Thanks for your support!

    • Unfortunateley it doesn't work for me.

      I added 2 directories in my "Resources" directory. "English.lproj" and "German.lproj". For each directory I created 1 file named "InfoPlist.strings". The file in the "German.lproj" directory contains:

      CFBundleDisplayName = "Küche";
CFBundleName = "Küche";

      I deleted all files in "Build" directory and started a full rebuild.

      — commented January 6th 2011 by Stephan Lenhart

    • Ok, I found the problem. The files have to be UTF-16 encoded. Now it works. Thanks for support!
