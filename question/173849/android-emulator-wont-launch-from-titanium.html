Android Emulator Won't Launch from Titanium

Application type: mobile
Titanium Studio: 3.2.3.201404181442 - 4/18/14
Device: Android emulator
Host Operating System: OSX 10.9.2

This is an odd one. Everything was working fine before the OSX Mavericks upgrade. Now that I've upgraded, I'm trying to run the Kitchen Sink (or anything that previously ran). When running on any version of iOS emulator it builds and launches the emulator as expected. No problems there. When I switch to android…

  1. I click Run
  2. It pulls up the "Run Configuration"
  3. I confirm that there's an Emulator selected.
  4. I confirm that there's a log level of info.
  5. No keystore, since it's just a test.
  6. Click "Run".
  7. Little bar motion in the bottom right for a fraction of a second as the window goes away. "Launching: Invoking 'Studio Builder' on '/KitchenSink'."
  8. "Run Configuration" window comes back.
  9. Rinse and repeat.

No output in the console window… no nothing. No build log file either. So I try running the emulator outside of Titanium and it works like a charm. Comes up no problem. Go back into Titanium and it looks as me like I'm speaking Greek. I would expect there to at least be a log with an error message but there's nothing. I've tried:

  1. Running a fresh download of KS
  2. Cleaning and then running in Droid
  3. Cleaning, running in iOS, and then running in Droid
  4. Cleaning, running in iOS, cleaning again, and then running in Droid
  5. Launching the emulator outside of TI, then running in Droid

Nothing works.

  • All the android SDK's are installed and up to date
  • The latest version of Java is installed and up to date
  • The emu launches outside of TI

Help? :)

Thanks! :)
