Saving blob object to filesystem

Hi guys,

I'm retrieving photo from device photo library and the result i get is this:

{
          availableFormats =         (
              "public.jpeg"
          );
           creationDate = "2013-03-27T01:24:24.000+0000";
           image = "[object TiBlob]";
           location =         {
               latitude = 0;
              longitude = 0;
          };
          orientation = 0;
          thumbnail = "[object TiBlob]";
        type = photo;
           waysToAccess =         {
               "public.jpeg" = "assets-library://asset/asset.JPG?id=37E76D2D-3A9E-42A1-9EF1-D660B560349B&ext=JPG";
         };
       }

Because the image is a TiBlob, if there is lots of photos, it will cause a memory issue and crash. So now I want to put the Ti.Blob inside a sub-folder in the app filesystem. I need an example of how would i create the directory and how to save it into the directory. I would also need to know how to call back a particular blob image so that I can set it to my image view.

If there is any other way that you think can solve my problem other than saving the blob to app, please post it here. I am building for iPhone using 3.0.2.GA.

  • Hi Chung Furn,

    See this Sample for create Directory : 

    var dir= Ti.Filesystem.getApplicationDataDirectory();
var new_folder = Titanium.Filesystem.getFile(dir, 'folder_name');
if(!new_folder.exists())
{
  new_folder.createDirectory();
}
