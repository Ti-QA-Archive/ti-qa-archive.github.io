"Xcode Installation is Missing" When Running in App in Studio

Having a frustrating time configuring Studio to access xcode 4.5.1. When I originally downloaded xcode 4.5.1, I ended up having it along with 4.3 (not on purpose). Ti Studio kept accessing 4.3. My quick workaround was to launch 4.5.1 and then run Studio, which resulted in the 6.0 SDK being picked up.

Since I recently distributed my app, I wanted to clean up my environment, so I got rid of 4.3 and try to point Studio at my 4.5.1 installation. I followed the instructions in the docs by getting into a terminal session and running sudo xcode-select -switch <xcode_folder_path>.

After that didn't work, I tried re- installing xcode 4.5.1. Still not found by Studio. Issued the sudo command again, and that didn't help. Tried pre-launching xcode and that no longer works.

Now, every time I try to run on the simulator I get the message "Xcode installation is missing". I would greatly appreciate any help as I'm completely unable to work on my app.

Thanks….
Sam
