  • Check your run and debug configurations in Titanium. You can see where it says Android API and a dropdown. You can't use Android x.x with KitchenSink because it uses Google Maps and some other stuff, that is only available if you select 'Google APIs Android x.x' instead.

    If you change your emulator settings manually with AVD manager and they don't match EXACTLY with what's in the debug/run configurations in Titanium, Titanium will create a new emulator that does.

    • Troubleshooting suggestion:

      1. Any chance you have spaces in your user folder? Can be weird on Win, unless you're handy with Python.
      2. Can you compile a sample Android project in Eclipse (rules out the SDK or JDK being the issue).
      3. The emulator starts, but where does it end? Can you run other apps? One problem I had was the Emulator being corrupted, and trying to use too high of resolutions…For some reason Windows can't even really use the tablet emulator, no idea why. That's why I suggested moving your Android version down to something like API Level 4 or maybe Droid1.6, rinse and repeat to test it out.
      4. If all else fails, and you can definitely run a sample project in Eclipse (just a normal Android hello world thingy), IF and only if all that doesn't work, reinstall Titanium. I had to do it more than once during troubleshooting and I think it may have helped at some point.
