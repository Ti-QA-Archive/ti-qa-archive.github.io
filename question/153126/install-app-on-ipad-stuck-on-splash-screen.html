  • Ok so I guess I really needed to ask the question to find the answer…

    If someone runs into this issue too, here's the simple solution:

    The problem was that I forgot to set the -D flag in CLI to development, it was building with test by default which apparently doesn't work on device.

    Bad:
    titanium build -p ios -T device -F ipad

    Good:
    titanium build -p ios -T device -F ipad -D development

    That's it.

  • Sorry for the late answer. This sounds unusual. By default, when you build for device or distribution, your JavaScript code gets minified and encrypted. The red screen of death should never appear in production builds which gives the appearance that the app has hung.

    I would re-run a build, but set the --log-level trace as well as launch Xcode and from the organizer, view the console for your device. Keep your eyes peeled for errors, especially those referencing the app.js.

    If you are getting 'app.js not found' errors, then usually it's because of a syntax error in your JavaScript code. In Titanium 3.1.0, this should be reported and the build should fail.

    Also, you should be using the --target app-store option if you're ready to build your app for distribution.
