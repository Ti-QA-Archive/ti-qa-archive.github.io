Titanium Studio strange behaviour

Hi guys,
I found some problems with Titanium Studio with code completion function:
1) when I try modify properties of element (for example borderStyle of TextField) Studio doesn't show me the value, just "No proposal". But no problem with adding GUI objects (Ti.UI.Label etc.) How to fix it?

2) when I adding non-existing constant (for ex. Ti.UI.KEYBOARD_NUMBERS_PAD instead of Ti.UI.KEYBOARD_NUMBER_PAD) my code marked no mistake! It can compile but no effect on GUI. How to mark error in code?

Thanks, Dmitry

  • Accepted Answer

    Titanium Studio grabs its content assist from a documentation file provided by the Titanium Mobile SDK. In the case of a property like "borderStyle," all that the documentation currently provides is a description of "Number" and not the constants a developer might like to see. We've created a ticket to help refine that process. Please follow along here for more information.
