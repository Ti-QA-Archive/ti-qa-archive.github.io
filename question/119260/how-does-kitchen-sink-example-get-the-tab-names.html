    1) There are no stupid questions :-)

    In line #6 and #10 titleid references the key in the locale file.

    There should be a folder named i18n with subfolders for some languages, this is how you can add internationalization to Titanium applications.

    Every language has it's own Strings.xml file, which specifies the key (#6,#10) and the corresponding translation:

    <string name="controls_win_title">Controls</string>

    • If you start a new project, you will have to add these folders by yourself (localization is optional).

      You don't need to include the strings.xml file, since it will be included automatically. If you have a closer look at your build log you will see

      [INFO] Compiling localization files

      This is the stage where the strings.xml file is beeing converted in the platform specific localization files. Titanium takes care of including it.

      If you want to use localized names somewhere else, you can use the L() function:

      L('key') which aliases to Titanium.Locale.getString('key').
