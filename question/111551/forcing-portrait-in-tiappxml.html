Forcing Portrait in tiapp.xml

I've read a number of posts that show how to force portrait layout in Android via a custom AndroidManifest, then other posts that imply that this can now be done via the tiapp.xml. I've tried a number of approaches in tiapp.xml. Can someone point me to the right direction?

Current attempt:

    <android xmlns:android="http://schemas.android.com/apk/res/android">
        <manifest>
            <application>
             <activity
                android:name=".TvfoodmapsActivity"
                android:label="TVFoodMaps"
                android:theme="@style/Theme.Titanium"
                   android:configChanges="keyboardHidden"
                   android:screenOrientation="portrait"
                >
                <intent-filter>
                <action android:name="android.intent.action.MAIN" />
                <category android:name="android.intent.category.LAUNCHER" />
            </intent-filter>
            </activity>
        </application>
        </manifest>
    </android>

I've also just having &lt;activity&gt; android:screenOrientation=&quot;portrait&quot;&lt;&#x2F;activity&gt;

No luck.

  • I'm using 1.7.5 SDK and my solution to this issue was order of operations.
    I futzed about with the manifest for a while before deciding enough was enough.

    // make a window
var mWin = Ti.UI.createWindow({
    title:"A Beautiful Landscape",
    fullscreen:true,
    touchEnabled:true,
    exitOnClose:true
})

// define the orientation of said window
mWin.orientationModes = [Ti.UI.LANDSCAPE_LEFT,Ti.UI.LANDSCAPE_RIGHT]

// now that we're rotated, we grab the dimensions
var mWidth = Ti.Platform.displayCaps.platformWidth;
var mHeight = Ti.Platform.displayCaps.platformHeight;

// create our view using those dimensions
var mView = Ti.UI.createView({
    title:"A View of A Beautiful Landscape",
    width:mWidth,
    height:mHeight
})

// down to business
mWin.add(mView);
mWin.open();
