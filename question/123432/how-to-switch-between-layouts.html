  • I don't know what is inside the .js files you include, but I'm pretty sure that this is not the way. If you create the UI elements on each .js files, the second time you call Ti.include, the elements already exists and exists in the actual window so including them again will produce unexpected results.

    Ok, so, you have at least two choices:

    1. Create two "big" views with both layouts, and switch between them hiding and showing the views.

    2. Other way, if difference between both layouts are only aesthetics (the same components but disposed different) is re-style each component.

  • What do you mean by separate window? I am guessing by the use of the "" it is not separate windows but just layouts for the same window.

    A quick and dirty way would be to encapsulate the content of your js files into functions say drawContacts() and drawContactsAvatar(), include them both at the start of the main js file and then use your detection logic to call the appropriate function.

    Another solution would be to create all your UI elements used in your window, use shared objects where ever possible and the use change all the parameters when you switch modes, even animate it for niceness.
