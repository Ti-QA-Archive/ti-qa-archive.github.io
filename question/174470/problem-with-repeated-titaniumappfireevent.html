problem with repeated Titanium.App.fireEvent

I have encounter a repeated action when I called Titanium.App.fireEvent. I found out that each time when I call Titanium.App.fireEvent, it will repeat all previous action by the same event name and shows the event data by the number of times i call that event. Suppose, this is my appEventListener - 

Titanium.App.addEventListener('HorseboardFilterSearch', function(e) {
        Ti.API.info(e.x);
    });

and this is my calling function -

Ti.App.fireEvent('HorseboardFilterSearch', {
                x:6
            });

when i first time call 'HorseboardFilterSearch' event it shows only 6 then for 2nd time call it shows 6 and 6 , then for 3rd shows 6, 6 and 6 and goes on. why this happening ? May I know how can I avoid from repeating the same previous actions and avoiding same value multiple times when the same event is called? please help me out.

    You are getting multiple results, because you probably have added to the EventListener multiple times.
    Solution: add the eventlistener only once.

    One more advise: Don't use Global EventListeners( like Ti.App.addE…) , they can cause memory problems.
    Assign Eventlisteners to views, instead to the Ti.App namespace, or use callback functions instead.
