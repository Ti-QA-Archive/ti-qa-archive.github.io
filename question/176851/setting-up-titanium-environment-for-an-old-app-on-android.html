Setting up Titanium environment for an old app on android

Hello all, I am fixing bugs on a Titanium app that was written in 2012. I primarily work on front-end web development and this is my first exposure to mobile app development.

I have it working on iOS just fine but I am having a heck of a time getting my titanium studio environment set up to work on android.

The tiapp.xml says: <sdk-version>2.1.4.GA</sdk-version>. The Titanium Studio version number I'm using is 3.3.0. Do I need to use an older Titanium Studio version to use the old SDK? Do I also need an android SDK and tools installed? What is the difference between Titanium SDK and android SDK?

Thanks for help

  • So, the sdk-version is the Appcelerator SDK version, NOT the android sdk. So, when the app was written, it was done with that older version. You CAN download and install earlier versions of the SDK, check out this page.

    Just know that there have been MANY changes over time to the SDK and that you may find bugs/issues that cannot be fixed without going to a newer sdk, which may require you to change your code… good luck!

    And to answer Q2, YES you do need the android sdk's. Easiest way to get them is to:

    • Start studio, you should get the dashboard opening by default.
    • Click on the 'Get Started' Tab
    • Scroll to the bottom of the tab where you can configure SDK's. Click on Android SDK and follow the instructions to either confirm it is installed properly or install for the first time.

    If you don't see the dashboard, you can load it by clicking Help/Studio/Show Studio Dashboard from the menu.
