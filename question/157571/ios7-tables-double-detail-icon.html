iOS7 tables double detail icon?

As much as we're all trying to get to grips with the (often strange) Apple UI decisions for iOS 7, I can't believe I'm supposed to be seeing the new version of the Detail Disclosure button (a blue circle with a thin 'i' in it) followed to it's right by the Disclosure indicator (a grey arrow) in the same table row. I would expect one or the other, depending on if I set Titanium.UI.LIST_ACCESSORY_TYPE_DETAIL or LIST_ACCESSORY_TYPE_DISCLOSURE.

I can't find any mention of both being the norm in the new iOS HIG doc issued by Apple. So there appears to be a bug somewhere or I'm missing something. Which wouldn't be the first time!

Mobile /Titanium SDK version 3.1.3.GA/ iOS 7/ Mac OSX 10.8.5

  • Accepted Answer

    Quick workaround: search for UITableViewCellAccessoryDetailDisclosureButton and replace with UITableViewCellAccessoryDetailButton

    for ListView replace in UIModule.m:

    MAKE_SYSTEM_PROP(LIST_ACCESSORY_TYPE_DETAIL,UITableViewCellAccessoryDetailDisclosureButton);

    with

    MAKE_SYSTEM_PROP(LIST_ACCESSORY_TYPE_DETAIL,UITableViewCellAccessoryDetailButton);

    for TableView replace in TiUITableViewRowProxy.m

    cell.accessoryType = UITableViewCellAccessoryDetailDisclosureButton;

    with

    cell.accessoryType = UITableViewCellAccessoryDetailButton;
