Styling issues when opening windows

I've made a topic on it and now I've figured out a way to get it to behave the way I want, I'd like your opinion on this workaround.

Okay, here's the issue.. when opening windows on my ipod 4 touch, the styles get applied after the window opens, there's a split second where I see the elements unstyled, sitting in the middle of the view or at the top and quickly shift to their position, it really doesn't look professional.

My Solution was to set the container view's tss as 'visible : false' and add an event listener to the window on 'open' to show() the container view… this workaround works but I'm wondering if anyone has ever dealt with similar issues.

Thank you.

  • Hi

    This seems like a logical solution.

    Usually in my apps, I will create a loading view with an Activity Indicator. This will sit over the top of my window until everything finishes loading and then I would remove this view to reveal the main window.

    This informs the user that something is loading and allows you to load everything in the background. Of course if you make the view slightly transparent, then the user will still be able to see the page as it loads. But because the user is informed that this is meant to be happening, this isn't usually a problem.

    Now you could take the best of both these solutions, as your solution I'm assuming, will leave an empty page until the container finishes loading, so showing a loading view might help and the user won't see any un-styled elements until you show them.

    Hope this information is useful.
