Empty ScrollableView causes app to crash in Android since 3.2.3

Since 3.2.3 if I create an empty ScrollableView in causes the app to crash.

Example:

<Alloy>
    <Window class="container">
        <ScrollableView>
        </ScrollableView>
    </Window>
</Alloy>

  • Hi

    Seems very unlikely.

    Are you sure this is the cause of the crash and not some related code or styles used that are doing something (not expecting the last one)?

    • If you create a brand new test app solely to check this the theory what happens?
    • What code is associated with loading this controller?
    • If you comment out ALL the code [Except any win.open()] does it work?
    • If the controller is being called from else where and affected in that place - comment out the code that affects this section and then what happens?

    Hopefully either of the last two will provide some movement - if it works without issue then start uncommenting code until it triggers and work from there.
