  • The correct way to calculate status bar height

  • According to the link, in Android you can tell the status bar height based on the display density. That seems safer to me then testing specific display dimensions because new devices may come along with different screen sizes.

    In Titanium, you can easily get the display density with Ti.Platform.displayCaps.density, so untested, the code could be something along these lines:

    var statusBarHeight;

switch ( Ti.Platform.displayCaps.density ) {
case 160:
    statusBarHeight = 25;
    break;
case 120:
    statusBarHeight = 19;
    break;
case 240:
    statusBarHeight = 38;
    break;
case 320:
    statusBarHeight = 50;
    break;
default:
    statusBarHeight = 25;
    break;
}

    Based on the above, this should also work and may be self-adjusting even when new densities come along:

    var statusBarHeight = Math.Round(25*Ti.Platform.displayCaps.density/240);
