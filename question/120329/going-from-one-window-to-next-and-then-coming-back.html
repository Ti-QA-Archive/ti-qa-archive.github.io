Going from one window to next and then coming back

I seem to be stuck trying to implement this.

Initially I'll have a map displaying certain locations. The top left part of the map has a button, on clicking which it will move to another window. This is working fine. In the next window, the user will enter a form, and then click another button. Then, the flow will move back to the map, and display locations according to the filled form. It doesn't really matter to me if the same mapview is used as was used earlier, as the locations can change.

I'm using Mobile SDK 1.6.2 and Android 2.2

Since I can't use more than 1 mapview, the first time I'm creating a mapview (mapview= Ti.UI.createMapView()), and setting Ti.App.map = mapview. In the other cases (when it comes back from the form), I'm using the same mapview by setting mapview = Ti.App.map instead of creating a new mapview.

In doing this, when I come back to the map the second time, it displays the button, but not the map. It continues to display the elements of the form page.

I could also try to remove the mapview on going to the form window, and creating a new mapview each time. Could anyone help me out on how this can be done?
