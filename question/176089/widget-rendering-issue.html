Widget rendering issue?

I am trying to create a widget, and having an issue with I believe the rendering. Please advise. Nothing appears to be rendered. Currently using 3.2.3, testing on IOS7.

Widget.js

var args = arguments[0];
var layout, controller, views;

initUI();

function initUI() {
    var childNum = parseInt(args.children.length);

    switch (childNum) {
        case 1:
            controller = Widget.createController('one');
            layout = controller.getView();

            break;
        case 2:
            controller = Widget.createController('two');
            layout = controller.getView();

            break;
        case 3:
            controller = Widget.createController('three');
            layout = controller.getView();

            break;
        case 4:
            controller = Widget.createController('four');
            layout = controller.getView();

            break;
    }

    views = controller.init();

    for (var i = childNum - 1; i >= 0; i--) {
        views[i].add(args.children[i]);
    }

    $.view.add(layout);

}

one.xml

<Alloy>
    <View layout="horizontal" width="Ti.UI.FILL" height="Ti.UI.SIZE">
        <View id="one" />
    </View>
</Alloy>

one.js


exports.init = function () {
    return [$.one];
};

I am then calling the widget as followed:


<TableViewRow>
    <Widget src="com.my.widget">
        <ImageView image="http://http://lorempixel.com/320/200/fashion" />
    </Widget>
</TableViewRow>

    When you say nothing is rendering, what exactly do you mean, as none of your styling will actually indicate the view is there, i.e. no border, or image, or background color.

    Just for information, the top level object in a view takes the id of the name of the view, if it isn't given an id, so

    <View layout="horizontal" width="Ti.UI.FILL" height="Ti.UI.SIZE">

    means that view has an id of "one".

    You have then created another view with the same id:

        <View id="one" />

    So now you have two views called one. Here is part of the code that Alloy generates so you can see:

    ...
    $.__views.one = Ti.UI.createView({
        layout: "horizontal",
        width: Ti.UI.FILL,
        height: Ti.UI.SIZE,
        id: "one"
    });
    $.__views.one && $.addTopLevelView($.__views.one);
    $.__views.one = Ti.UI.createView({
        id: "one"
    });
    $.__views.one.add($.__views.one);
...

    I suggest changing the name of the inner view, and putting some styling to indicate which view is which for debugging purposes.
