Keyboard over textfield

I add textfield and some chat view to scroll view, because without scrollview soft keyboard show over textfield. It's known problem. But i have a next problem after adding scrollview. Please, see problem at video of my screen: http://youtu.be/qbj7pr55QPYd.

My code:

    var localChatView = Ti.UI.createView({
        top : 0,
        width : '100%',
        height : '100%',
        backgroundColor : '#ffffff'
    });
    chatView.add(localChatView);

    var scrollView = Ti.UI.createScrollView({
        contentHeight : 'auto',
        top : 0,
        bottom : 0,
        backgroundColor : 'red',
        layout : 'vertical'
    });
    localChatView.add(scrollView);

    var viewOnScroll = Ti.UI.createView({
        top : 0,
        left : 0,
        right : 0,
        bottom : 0,
        height : 'auto',
        backgroundColor : '#f8f8f8',
        zIndex : 5
    });
    scrollView.add(viewOnScroll);

    var newMessageView = Ti.UI.createView({
        height : 31,
        width : '100%',
        bottom : 0,
        left : 0,
        zIndex : 6
    });
    viewOnScroll.add(newMessageView);

    var newMessageTextField = Ti.UI.createTextField({
        color : '#336699',
        height : 23,
        width : '80%',
        bottom : 4,
        left : 0,
        borderWidth : 1,
        borderRadius : 5.0,
        zIndex : 10
    });
    viewOnScroll.add(newMessageTextField);

    var sendMessageButton = Ti.UI.createButton({
        title : '????.',
        right : 0,
        bottom : 4,
        height : 23,
        width : '20%'
    });
    viewOnScroll.add(sendMessageButton);

  • I found solution! Thank's to all answered! I just add code below:

        newMessageTextField.addEventListener('focus',function(e){
        scrollView.setContentHeight(0);
    });

    newMessageTextField.addEventListener('blur',function(e){
        scrollView.scrollTo(0, 0);
        scrollView.setContentHeight(0);
    });

    I reaaly don't know how but it work's!

    • Hi Alexander,

      it sounds good for your answer, you decrease the content Height to restore your current view. Means you are shrinking your Content view to 0 so your scroll view man not remain scrollable for even if they have more controls in it.

      Scroll view always scroll the content to its content width. when you give (0,0) condition then you scroll to top most position and disable the scrolling by applying content width 0 so your control become stable. this is preferable when you to pull down view only.
