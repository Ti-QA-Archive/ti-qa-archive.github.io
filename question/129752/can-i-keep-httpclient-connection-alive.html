Can I keep httpClient connection alive?

I am using httpClient to get data from the server. Initialization happens only once, the second time I send the request to the server, I only use send without reopening , resetting callback etc. It works nicely.But I am not sure if doing this has potentials….

  • Keeping a http connection alive sometimes is a good solution but it is not really good practice.

    In titanium an asynchronous connection is made when you create the network connection. This then remains open until it is automatically closed. This happens after it stops being used.

    So to answer your question the connection is open and stays open. The pitfalls, are that the connection can get overwritten by the next request and this practice is not advised.

    I have hit major problems in this, and effectively you need a new variable declared for each connection. not just a new occurrence of the variable.

    Hope this explains more, but as Rainer says to give an answer specific to your requirements please provide code. Or if this answer your question mark it as correct.
