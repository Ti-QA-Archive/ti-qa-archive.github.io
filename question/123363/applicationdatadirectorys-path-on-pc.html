  • With the iphone simulator, the path will be something like

    /Users/(username)/Library/Application Support/iPhone Simulator/(version)/Applications/(app id)/Library/Application Support/

    However, this statement concerns me:

    "I would like to insert a picture there that I can use in my application."

    You normally don't want to add a file there using methods on your PC, because the same thing can't be used in production mode on the device. The main benefit to knowing where the simulator puts the files for an application are to let you independently examine things your application creates. For example, I use this to examine SQLite databases using the Firefox SQLite Manager plugin.

    In your application, you always want to use the normal syntax to the folder:

    Ti.Filesystem.applicationDataDirectory
