Locale isn't working anymore

Hi all,

I'm having some trouble right now using localization for my application. I have two folders for english and german language. It was working fine but out of a sudden, it stopped working and I'm just seeing the string which should be replaced. Here is a snippet of my strings.xml:

<string name="head_pref">Preferences</string>
<string name="head_offline_map">Offline map</string>
<string name="head_load_offline_maps">Load offline map</string>

If I'm using
L(&#39;head_load_offline_maps&#39;)
or
Ti.Locale.getString(&#39;head_load_offline_maps&#39;));
the app shows me 'head_load_offline_maps' instead of 'Load offline map'.

While compiling the app, the log says that the localization files have been compiled successful. I've also used Project > Clean and cleaned the workspace folder manually without any success.

My Titanium SDK is 1.7.2 and I'm building on iOS 4.2 Simulator.

Any help and hints would be appreciated :)

  • I found it. What a stupid copy - paste mistake ! In one of my strings I had some quotation marks. It looked like 

    <string name="missing_value">"Missing values"</string>

    instead of 

    <string name="missing_value">Missing values</string>

    After removing the quotation marks and compiling my app it works again.
