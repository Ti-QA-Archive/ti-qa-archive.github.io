  • A TableViewRow is one of the rows within a TableView. What might be confusing you is that you can create a TableView by passing generic data to it. Implicitly, that's creating TableViewRows for you.

    But a TableViewRow is much more interesting than the rows created by this generic data property of the TableView. You can create a TableViewRow and add subviews to it to create all sorts of interesting rows.

    I'm not sure what showClickEventInfo() does, but when you get a click event, you can use e.row to reference the row that was clicked, and your code can do anything based on the properties of that row. If you want it to open a remote URL, add that URL as a property of the row when you create it, and it's available for you when the user clicks on the row. It's really powerful stuff.
