javascript variable and function scope

win 7 android 4.2 Titanium 3.0. I was sure that this question would have been asked/answered before but couldn't find it. On the other hand, it is strictly a very basic javascript question. If inappropriate for this forum, I'm sure someone will tell me! Anyway, I'm writing an android app for a galaxy s3 using appcelerator. I've read all the information on scope that I've been able to find online, but my question is still not answered. I've read that a global variable (on defined outside of a function with the var keyword) can be accessed from anywhere in the current file and that a local variable (one defined in a function without the var) can only be accessed from within the function. I've also read that it is next to horrific to use globals. I understand the reasons but I've always found the need for them in C. My program uses multiple .js files, one of which houses the globals I use. I want to access those globals in another .js file. But it doesn't work even though defined with 'var'. So, how do I access variables defined in one .js file, in other files, all local? Also, how do I do the same with functions? Any help would be appreciated. Thanks.

  • If you're searching the web for the answer you're going to get alot of different and aging answers. For your specific needs, you'll need to be searching for commonjs modules. commonjs is a way of modularizing your javascript code and the module spec is built into Titanium. The basic idea is this:

    app.js

    var globals = require('globals');
Ti.API.info(globals.foo);
globals.bar();

    globals.js

    exports.foo = "Bar";

exports.bar = function() {
    alert("Foo");
};

    No matter which file you require globals.js in, the variables within will always maintain their value. It may really be a bad idea to call or even consider these as globals, but its the way you get around using actual global variables using commonjs.
