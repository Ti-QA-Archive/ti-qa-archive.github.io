Parse error while using .jss file

I need to run the IBS office app on all platforms (iPhone, iPad and android). I found two ways to create platform specific app, one using .jss files and other using if else condition.

I created two jss files (app_iphone.jss and app_ipad.jss). The file structure of these two files are:
> Resources->ui->ipad->app_ipad.jss
> Resources->ui->ipad->app_iphone.jss

In my app.js file (bootstrap file), i gave 'id' property to imageview and use this id property in .jss files for giving different width and height according to iphone and ipad, but after executing i am facing error "Parse error at app_iphone.jss (line1)"

  • jss was never well supported by Titanium and it seems that it will be going away as development moves forward.

    What you can do however is to have the same basic approach using specific .js files depending on implementation as explained on the this blog post from the Appcelerator Blog.

    Another approach would be to encapsulate your UI properties in a different file depending on the platform.
