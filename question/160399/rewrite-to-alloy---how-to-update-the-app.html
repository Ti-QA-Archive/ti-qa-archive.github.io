Rewrite to Alloy - how to update the app?

Hi,

I have rewritten my app in Alloy. The version that is currently live in the appstore is made in 'standard titanium'. To do the rewrite, I created a new project with a different app id etc. I am now ready to roll out the rewritten app as an update to the current app.

But what do I need to adjust in the rewritten app so that it is recognised as an update to the same app? Should I copy the old <id> and <guid> contents from the old tiapp.xml to the new alloy tiapp.xml? Is that enough or are there more things to think about?

Or can I just copy the directory contents of the alloy-version over the old directory contents?

  • In order to publish the App as an upgrade to an existing one, you would still need to use the original provisioning profile. that is linked to your App id, so yes you need to re-use the same App Id. If you have enabled Ti Analytics and want them maintained in the new version then you also need to copy your guid, else that isnt neccessary
