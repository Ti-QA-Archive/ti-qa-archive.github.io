Considering using Titanium. Is this a viable platform?

Hi guys

After seeing Gartner's magic quadrant report for mobile app development platforms, we're leaning toward doing a proof of concept app using Titanium. Everything I've read so far seems favourable. The problems I have been able to Google mostly date back to 2011 articles and forums. These include things like:

• Random crashes due to memory leaks

• Difficult aligning GUI elements on screen

• Focus event called repeatedly on specific devices

• Poor documentation

• Large binaries

• Slow performance

In addition to this, I notice that many of the apps listed on their App Showcase page are only available for iOS. As Android is the most popular OS, I can only assume these showcased apps weren’t published for Android due to some difficulties in getting a stable build.

I don’t know if these issues persist today so was hoping to get some feedback from developers currently working with Titanium and see if they would recommend or not.

Tks

  • Hello,

    disclosure: I work for Appcelerator. This is my personal opinion, and has nothing to do with my employer.

    Time warp – before working for Appcelerator

    Being said that, before I knew Titanium (2009?), we were creating native mobile apps. We were having trouble to develop apps as fast as the customer required them, so we were able to produce less apps than we needed to (we were an only iOS little shop).

    Once we found Titanium, there was a lot of changes in our methods to create apps. If we were having small apps, we just did it in Titanium, since it was A LOT faster than in native. And we did start creating android apps as well!

    Then, Appcelerator hired me.

    Time warp here

    Nowdays, I see some BIG companies using Titanium for all kind of apps (either for internal and external usage), supporting either Android, iOS or mobile web (covering Blackberry OS and Windows Phone via mobile web) and they save a bunch of time. Not all the apps should be made in Titanium: some of them need really fine tuning, so you do it in native (or create a module).

    Best,

    Mauro

  • Accepted Answer

    Titanium has improved a lot in the past couple of years especially for Android and the best is yet to come.

    I have been using it for my clients for a couple of years now and I heard mostly positive feedback. Of course development is not easy and you'll face a lot of unfavorable surprises but once you master it you'll find your way through. Using Alloy MVC framework and CommonJS module specs will make it even easier for you and will drive down the possibility of memory leaks, and the XML and TSS syntax is familiar to web developers. When it comes to image intensive apps on android, large heap is your friend.
    The other great thing about Titanium is the supportive community. Ask clear questions, add enough source code and feel the love.

  • • Random crashes due to memory leaks

    Crappy programming results in crappy memory management.

    • Difficult aligning GUI elements on screen

    You can place any object pixel by pixel.

    • Focus event called repeatedly on specific devices

    Cannot answer as 'specific devices' is generic.

    • Poor documentation

    This rumor has got to freaking die a painful, horrible death. As dev who has been using Titanium for over 3 years, I don't want to even hear from new folks complaining about documentation. Yea, it was horrible in 2010-2011, because the engineers were building out the framework, not writing documentation. In 2011-2012 a change in best practice occurred, leaving some outdated docs floating around. Its 2013, the docs are not perfect, but definitely not lacking.

    • Large binaries

    At the moment, your ipa or apk contains a javascript engine. This is due to restrictions of the target platforms. At one point, for compatibility, on android, two javascript engines were packaged. This could very well be changing in the next 6 months.

    • Slow performance

    Other than tableviews (and listviews resolves a bunch of this) this is another lie/myth that needs to die. Write good code, get good results.
