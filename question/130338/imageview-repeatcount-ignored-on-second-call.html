Imageview repeatcount ignored on second call

This question was brought up earlier here:

(http://developer.appcelerator.com/question/4831/imageview-repeatcount-bug)

and here:

(http://developer.appcelerator.com/question/110991/imageview-animation-repeats-endlessly)

And I was hoping there was a fix to the problem available somewhere.

Basically I'm calling a series of images to animate sequentially 

var aniFrames = [];
for(var i=1; i<4; i++){
    aniFrames.push('img/frame_' + i + '.png');
}

var sequenceAnimate = Ti.UI.createImageView({
    images:aniFrames,
    height:135,
    width: 250,
    bottom:25,
    left: 50,
    duration:300,
    repeatCount:1   
});

Works fine the first time, however the second time the animation is called it behaves as if the repeatCount were set to 0, and the animating loops endlessly.

Are there any fixes or work arounds anyone can suggest?
