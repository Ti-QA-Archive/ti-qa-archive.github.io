To separate design from code

Hello,

I'm trying to separate my logic code from my design code. A little bit like a mvc framework, but not that complex.

I looked at some mvc framework on titanium and I didn't like any of them, some of them had great idea that I incorporate in my developpement structure but that pretty much it.

So the main two reasons why I would like to separate my logic code from the design code, is first: it's easier to read
Then, I want to be able to support multiple screen size. I know we can use percent, but in my experience, using percent is only a "good enouph hack", it never really give you exactly what you want. Further more, some bigger screen should be able to have more options, and be optimized.

So I tried to include / require a file, I tried using a normal include, or using it as a commonJs module, but could not make it work.

My graphics variable aren't passed to the logic file.
So is there a way to do so, without having to pass every single graphics variable to some function?

Thank you

  • I do a bit of this, not MVC, just trying to separate things. I think it's going to come down to your style. My apps so far have been fairly easy on the resources so my process may not work on larger apps.

    I usually create a namespace for each group of activities and separate each name space into different files. So my ajax calls are in a call.js file in the app.ajax namespace. Then I define all my ajax calls as so:

    app.ajax.getThisData = function(){
//CODE
}
app.ajax.getThatData = function(){
//CODE
}

    I do the same thing for different parts of my application. I also have a general.js file that has constructors for various UI parts. So:

    app.general.createTableSection = function(){
//CODE
}

    This helps because then for all platforms I can simply call:

    var tableSection = app.general.createTableSection();

    and it will build the section platform/device specific based on which general.js file was included or other logic in that file.

    Then in my app.js file I include different files based on platform/device/etc. So that way I can have an altered general.js file for each platform etc and build things a bit differently. Other parts of my app such as calls, events, etc can be the same across all platforms.

    That's my way and is probably in no way the best. I also have an events.js file to handle all App level events, and an init file to handle initial loading os things.
