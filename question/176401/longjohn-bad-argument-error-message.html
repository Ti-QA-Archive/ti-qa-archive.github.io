  • There have reports of a couple of Android issues that we're investigating. The primary problem seems to involve Android tooling version problems. Apparently some developers are having troubles when using Android tools R23. This ticket has been resolved, which could be the cause of this problem.

    Open the android app (part of the Android SDK) and see what it reports on the "Android SDK Tools" line. If it says R23, you have three choices: 1) downgrade your Android tools (a manual and not very easy process), 2) use the nightly build of the SDK which includes the fix for TIMOB-17286, or 3) wait for our next SDK release which will officially contain this fix.

    To use the nightly build: visit http://builds.appcelerator.com/ then next to "install" under the heading, copy the ti sdk install command. Paste that into your terminal window and press Enter. Update your tiapp.xml to use the resulting SDK version.

    • I think this is related to this issue: TIMOB-18340

      Also, the solution step by step is (for others with the same problem):

      • Check in your error the path to the file "windows.js", in my case, it was in: C:\ProgramData\Titanium\mobilesdk\win32\3.5.1.GA\node_modules\titanium-sdk\lib\windows.js

      • Open that file

      • Go to the line described in the error, for example, in my case it was line 386: "…lib\windows.js:386:14"

      • Remove the line: if (!selectedVisualStudio) return next();

      • In the same place, add the line: if (true) return next();
