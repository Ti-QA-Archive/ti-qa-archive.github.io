    • Ok Karray,

      on the sqlite3 download page there is a link to command-line shell where you will find a getting started for sqlite3.

      After downloading the sqlite-shell-win32-x86-3071401.zip please unzip and put the sqlite3.exe into a new folder. For example: c:\sqlite3

      Add that c:\sqlite3 to the end of your path with ;c:\sqlite3

      From now you will work from a windows prompt (cmd) with sqlite3.

      A little Windows 7 cmd crash course for navigation inside prompt:

      • open a command prompt: press start, type cmd and click cmd.
      • you can navigate to root: in the prompt type cd \ and hit enter
      • you can list files and folders: type dir and hit enter
      • you can navigate inside a folder: type cd folder name and hit enter
      • you can navigate one level up: type cd .. and hit enter

      hint: If a folder name is very long or contains spaces you can type the first letter(s) and press tab to get the folder name.

      So lets assume we want to navigate down the folder structure you provided.

      "c:\myTitaniumStudioWorkspace\appName\Resources"

      • open a command prompt: press start, type cmd and click cmd.
      • navigate to root: enter cd \ and hit enter
      • type cd ( cd and one space), type my and hit tab until myTitaniumStudioWorkspace displays
      • type \ (backslash)
      • type a and hit tab until appName displays
      • type \ (backslash)
      • type R and hit tab until Resources displays
      • hit enter and there you are!

      (for more info on navigation and working with the windows command shell please use google or study a good book)

      Now if we are in the desired folder, we use sqlite3 to create the database:

      • type sqlite3 test.sqlite and hit enter. This will start sqlite3 command line shell and create a new or open an existing database (new db will be created physically after executing the first create statement)
      • type your sql commands like CREATE TABLE testdb (name TEXT, surname TEXT); and hit enter

      • for exiting the sqlite3 command line shell type .quit and hit enter.

      • before copying or moving that database file you should always exit sqlite3 using .quit!

      Hope that helps.

      Matthias
