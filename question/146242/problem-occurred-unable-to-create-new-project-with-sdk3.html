"Problem Occurred" "Unable to create new project" with SDK3

I'm on OS X Mountain Lion with a fresh install of Titanium SDK3. Android SDK is installed. All updates installed. Workspace has been deleted then recreated with a new Titanium install.

If I try to create a new project with Titanium SDK2, no problem. If I try to create one with SDK3, at the point where it's trying to create a new folder, I get..

"Problem Occurred" "Unable to create new project"

The project folder is created but empty. I see no strange permissions problems and nothing in the Console.

Please help, I'm quite stuck.

Thanks,
Stuart.

    Let's start by doing a few things from the command line:

    First run node -v and verify that you're using 0.8.x or newer.

    Next run titanum. It should say:

    Titanium Command-Line Interface, CLI version 3.0.22, Titanium SDK version 3.0.0.GA

    If it doesn't say CLI version 3.0.22, then run:

    sudo npm install -g titanium

    If it doesn't say Titanium SDK version 3.0.0.GA, then let's check that 3.0.0.GA is installed.

    titanium sdk

    Verify that you have 3.0.0.GA installed and it's the "active" SDK.

    If it doesn't list 3.0.0.GA, then run titanium sdk install 3.0.0.GA.

    If 3.0.0.GA is If it's not the "active" SDK, run titanium sdk select 3.0.0.GA to make 3.0.0.GA active.

    Next, let's run setup.

    titanium setup --advanced

    This will ask you a bunch of questions. Most of them you can just hit the enter key to accept the default value.

    Next, let's create a project from the command line:

    titanium create

    It will ask you what target platforms you want, where the project should be created, the app id, and the name.

    If the project is created successfully, then that's good! Try creating a project again from Titanium Studio.

    If the CLI fails to create the project, then you probably have a file permission error.

    Try the above and let me know if that fixed it or if we need to dive a little deeper. Best of luck!
