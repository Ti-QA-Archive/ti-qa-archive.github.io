Android distribute package still contains javascript source

If you have a look at this QnA: http://developer.appcelerator.com/question/100201/enable-android-byte-code-compile Marshall mentions that the bytecode compiler runs when it is in distribution mode.

Once a run the package process I do get an apk file generated. When I rename the apk file to zip, and unzip it. I can see inside the assets/resources folder my whole javascript source files.

Should these be compiled?

I am using Mac OSX 10.6.6, Titanium 1.2.2 I've tried with SDK: 1.6.1, 1.7 nightly and 1.7 preview.

I've also tried with and without this tag in my tiapp.xml

<property name="ti.android.compilejs" type="bool">true</property>

    • Thanks for posing back about this-it works! I grabbed todays 1.7 SDK build and everything works great. No more JS files in the APK.

      FWIW, I have these two tags inside the android node in the tiapp.xml file:

      <property name="ti.deploytype">distribution</property>
<property name="ti.android.compilejs" type="bool">true</property>

      I'm not sure if you need the deploytpe line, but it's in mine, and it's working, and I ain't touching it!

  • you can delete "assets" folder with 7z:

    mv app.apk app.zip
7z d app.zip assets
mv app.zip app.apk
zipalign -f -v 4 app.apk production.apk
