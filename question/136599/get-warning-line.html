Get warning line

Hello, with transition to SDK 2.0 we are all getting tons of warnings about width and height auto sizing of views, labels ecc..

....
[WARN] [object TiUILabel] has an auto height value of 0, meaning this view may not be visible.
[WARN] [object TiUILabel] has an auto height value of 0, meaning this view may not be visible.
[WARN] [object TiUIView] has an auto width value of 0, meaning this view may not be visible.
[WARN] [object TiUIView] has an auto width value of 0, meaning this view may not be visible.
[WARN] [object TiUILabel] has an auto height value of 0, meaning this view may not be visible.
[WARN] [object TiUILabel] has an auto height value of 0, meaning this view may not be visible.
[WARN] [object TiUILabel] has an auto height value of 0, meaning this view may not be visible.
[WARN] [object TiUILabel] has an auto height value of 0, meaning this view may not be visible.
[WARN] [object TiUIView] has an auto width value of 0, meaning this view may not be visible.
[WARN] [object TiUIView] has an auto width value of 0, meaning this view may not be visible.
....

I know, many times warnings are not dangerous for app stability but I want to fix them before to release a final build.

Is there any simple way to print the warning affected code line when testing the app? I can do a search and manually check all related elements (labels, views ecc..) but it's not a very fast way, specially for apps with a lot of code.

Thank you, Stefano.

  • I also fed up from these warnings.
    but i would like to add some solution of these warnings.
    in below code Something can be a label or window or else.
    just assign toImage.width of Something to the width of Something. 

    var Something = Ti.UI.createSomething({
    backgroundColor:'yellow',
    text:'This is a test...',
    left:10
});
parentView.add(Something);
Something.width = Something.toImage().width

    because Something is better than nothing :)
