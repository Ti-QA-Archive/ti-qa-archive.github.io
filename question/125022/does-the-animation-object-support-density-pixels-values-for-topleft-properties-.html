Does the animation object support density pixels values for top/left properties ?

Hi, is it possible to pass density pixel values (eg: '-45dp') to the top/left property of the animation object? It seems to just accept floats which I assume is just pixels.

  • It appears there is inconsistency with Titanium's layout api, some components only allow integer (pixel) values for top/bottom/left/right/height/width whereas others accept integer or strings (so allow density pixels).

    To get around the issue I'm calculating the density pixel value using the pixel value and the displays DPI. See here for solution:
    http://developer.appcelerator.com/question/125317/calculate-apps-density-pixel-width–height-using-titaniumplatformdisplaycapsdpi
