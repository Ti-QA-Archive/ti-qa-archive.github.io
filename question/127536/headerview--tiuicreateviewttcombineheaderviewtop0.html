  • In Tweetanium, they just happened to choose the characters '$$' to act as an abbreviation for 'tt.ui.properties'. It's essentially just a shortcut to save characters.

    This is briefly explained the top comments header section of the app.js, &#x2F;&#x2F;There is one additional global variable, $$,

    This is actually defined in styles.js as &#x2F;&#x2F;global shortcut for UI properties, since these get used A LOT. polluting the global &#x2F;&#x2F;namespace, but for a good cause (saving keystrokes) var $$ = tt.ui.properties;

    • To answer the first part… this line just creates a view with all of the properties passed into it instead of defining each property manually in a hard-coded fashion as you might see in the KitchenSink. Top : 0 is the only property not defined from the pre-built shortcut of '$$'. TT.combine, defined in tweetanium.js, just takes all of the params passed in to it and returns a new object with those params. So these other functions such as 'TT.combine' and 'mixin' are just 'helper' functions that save keystrokes and time, since they need to be done repeatedly in the same fashion. Hope that helps.
