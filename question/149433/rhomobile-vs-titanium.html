    • Pros and Cons on RhoMobile:

      Pros:

      1. Great support by motorola and rhomobile team. Really big plus, they're fast and know what they are doing.
        1. It's easy to learn. I found Ruby very easy to learn and if you want, you can avoid it almost entirely and do just the basics. The framework is also very straightforward and simple and using any library usually just requires calling a function.
      2. Supports out of the box a lot of devices hardware features (camera, barcode scanners, etc.)
      3. Rhohub, Rhoconnect, RhoGallery, check them out, they work great and gives a lot of value to the platform.

      Cons:

      1. Rhostudio (still) sucks. It's a tool that lacks all the great features of any modern IDE. Luckily you can develop using any other editor (like Sublime text) and use the command line. I use Rhostudio just for building and some quick editing to avoid the command line.
      2. Like every framework, when you want to do something it's still not prepared to do, it's a pain.
      3. Check the features and libraries already included and make sure it has everything you need. It allows you to write extensions but that's not really what you are looking for.
      4. Building and deploying (not using rhohub) for windows mobile and blackberry can be a pain if it doesn't work like described in the documentation.
      5. A lot of outdated documentation. Many things doesn't go as described in the documentation, or you can't find the detail you want on something. That's when the support in launchpad and google groups shines.
      6. Don't know why but i constantly get that feeling that mobile cross-platform development it's still not there yet but it's getting there…
