    • No the WebView interprets the HTML at runtime every time, this isn't cached (as far as I know) instead browsers usually cache script docs and images. Scripts and images can be external to the web-page, so a url is a valid src path. Therefore you can also append a fake query string, this to the browser will look like the path is different, therefore it won't try to load the cached script / image it will re-grab it. I use this in my Android app where I have a web-view.

      var randomStr = Math.random().toString(36).substring(7); //random string so that webview won't cache the image!

var img = new Image();
img.onload = function() {
    $('#bg').css({
        background: "url(" + e.path + "?" + randomStr + ") top left no-repeat",
        width: this.width,
        height: this.height
    });
};
