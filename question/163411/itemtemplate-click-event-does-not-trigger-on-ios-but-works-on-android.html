Itemtemplate click event does not trigger on IOS but works on Android

I'm making an app with the latest titanium sdk with alloy for both iOS and Android

I've set a trigger on my listview's itemtemplate

    <ListView id="listView" defaultItemTemplate="appointment">
                <Templates>
            <ItemTemplate name="appointment" onClick="clickEvent" >
                  <Label bindId="title" id="title" />
            </ItemTemplate>
            </Templates>
            <ListSection>
            <ListItem title:text="Blood Collect"
                          template="appointment" />
                    </ListSection>
       </ListView>

and the controller

function clickEvent(e)
{
   alert("test");
}

clickEvent is working fine on android, but it does not work in iOS, any ideas why?

    In general, you listen for the itemclick event on the ListView, not the click event on the template/items. See the code sample on the ListView docs page. Is there a specific reason you're using the onClick with the template instead?

    • Sorry I wasn't more explicit – the event names don't have the "on" but you use that if you want to associate an event in the XML markup. Hence, the event name is itemclick but you use onItemclick="function" in your XML. If you attach the event in the controller, you don't use the "on".

      $.yourList.addEventListener('itemclick', function() {
  // do stuff
});
