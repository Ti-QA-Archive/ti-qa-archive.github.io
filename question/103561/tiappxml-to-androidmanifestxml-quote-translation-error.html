Tiapp.xml to AndroidManifest.xml Quote Translation Error

I'm trying to add code to AndroidManifest.xml through Tiapp.xml and getting a translation error.

Here's a snippet from my tiapp.xml:

<android xmlns:android="http://schemas.android.com/apk/res/android">
    <manifest>
        <activity>
                 android:name="buildpath" 
        </activity>
    </manifest>
    </android>

Where 'buildpath' is the path to the activity.java file I want.

When I build the project with Titanium, the code that gets added to AndroidManifest.xml is:

<activity>
         android:name= & q u o t ;buildpath  & q u o t ; 
</activity>

Bear in mind there are no spaces in the actual error. They are needed to display the error properly here.
I get the feeling that & q u o t ; should not be there. In Titanium, the error I get is:

[ERROR]
C:\Users\myname\Experiment\build\android\AndroidManifest.xml:15:
Tag <activity> missing required attribute name.

What's happening here? Please help!

  • I managed to work out the answer to this problem. To fix it, you have to put your activity tags within application tags in tiapp.xml.

    There should be some kind of error that outlines the problem rather than titanium launching and then doing nothing.
