Looking for a way to handle multiple app targets in a single Titanium project codebase

In Xcode we use "Targets" to create multiple app output destinations from a single code base. For example, an Magazine eReader type app that contains defined targets for a dozen different publications. This allows us to make changes to a single code base, and then change the selected target to distribute updates to each of the application. We use header files to flex various UI changes and text between the different targets.

How would we do the same thing in a Titanium project? We have ported one of our magazine eReader projects to Titanium, but I can't find a way to specify multiple output targets. I know I can just duplicate the project once for each published app, but I'd rather not have to manage updating multiple project code bases as we tweak ongoing versions of the platform. Any suggestions?

    • Theming of i18n & platform folders is not yet done (tickets linked to in that article), and since then there's been a request to theme the tiapp.xml file as well. Once those are implemented, you should be able to use themes to accomplish your goal. These changes are not trivial and have lots of potential ramifications (like, we could break lots of apps if we do it wrong). So, we are taking our time implementing the changes to be sure we get it right.

      Fokke's article doesn't mention is using Alloy jmk (make-like) files to do the pre-processing. These files are nodejs scripts that are executed before (or after) the Alloy compilation process. You could use these in place of the sort of scripts that Fokke mentions to achieve the same goals. So, his overall technique, just implemented differently. See the Alloy build configuration files) page in the docs.
