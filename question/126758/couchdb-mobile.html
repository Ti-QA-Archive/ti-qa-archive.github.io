    I've written a free, open-source Titanium module that wraps Couchbase Mobile for iOS. You can download the module from Github here. Usage instructions are also online, but it's pretty straightforward to include in your project.

    I plan to create an Android version, but at the moment I don't have a lot of free time for that project. Pull requests are welcome!

    • Personally, I've only used replication in Apache CouchDb running on Linux servers, but since Couchbase Mobile is CouchDb underneath, I would expect replication to work fine for mobile as well. The Grocery Sync example app from the Couchbase Mobile site is an iOS native app that does a lot of syncing; I saw it demoed at CouchConf SF this year and it worked perfectly. Also, you should definitely subscribe to the mailing list for the latest information about the mobile framework.

      Just to be clear, the hard work and heavy lifting on this module was done by the folks at Couchbase, particularly Jens Alfke and J. Chris Anderson for the iOS version and Dale Harvey and Marty Schoch on Android. These guys are the real rock stars!

      — commented October 12th 2011 by Paul Mietz Egli

    • I've recently updated the Couchbase Mobile module so it works with Android, and have started another module based on TouchDB called TiTouchDB. TouchDB is a lightweight database for mobile devices that is compatible with the CouchDB replication protocol. Jens Alfke, the author, says

      > Think of it this way: If CouchDB is MySQL, then TouchDB is SQLite.

      My TiTouchDB module wraps TouchDB and provides both an HTTP and in-process API for common database operations. Right now, it runs on iOS, and I will have an Android version going very soon.

      Both TiTouchDB and TouchDB are open source and released under the Apache License 2.0.
