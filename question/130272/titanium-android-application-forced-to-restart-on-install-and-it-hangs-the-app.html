Titanium Android application forced to restart on install and it hangs the app

When I install my titanium application to an android device I get a dialog saying an application restart is required. When the app restarts, it seems that several event handlers are not attached to UI components - e.g. the app is pretty much dead - when I kill the app and restart it a second time (or more) the app works fine. It's just that first time after an app restart is forced that we see the problem. Our client will pay a bounty to the first person that can solve this for us. Thanks in advance.

  • The "application restart is required", we are told, is expected behavior because of a bug in Android. I don't think we have ever received an answer on why not EVERY app experiences this problem.

    [TIMOB-4941[(http://jira.appcelerator.org/browse/TIMOB-4941) and TIMOB-1559. There are 'fixes' available in the TIAPP.xml to make the workaround look a bit less amateurish.

    This is most likely not directly related to your problem (although, might be) as we all deal with this.

    Mauro is right, break out DDMS, see where there is a breakdown in your app.
