Internationalization of default UI elements - More tab, table row Delete button

Hello,

I'm in the process of internationalizing my iOS app using the i18n directory structure and strings.xml file and using the L() macro in the app. All has gone well until I got to the automatically generated More tab (when app has more than 5 tabs), and red Delete buttons on editable tables. These UI elements are automatically generated by Titanium and still appear in English even when the locale is set to Spanish and other custom texts display correctly. I cannot figure out how to get them to properly internationalize to Spanish.

I've read several articles in the Q&A section, but am not finding a satisfying solution. One article called for configuring the tiapp.xmsl file, another for hacking the Info.plist file in the build directly, but these approaches don't seem right. I don't want to preset these at compile time, there's only one app version which should pick the correct set of UI texts depending on user locale. Is there a way to set these More tab/delete button texts using the i18n/locale/strings.xml framework? How the heck do I get these to show up in the correct language at app startup?

Thanks in advance!

    Hi

    The more button is a system button as far as I know, so no idea how to change that.

    The tableView delete buttons that appear on tableViewRows when request can however have their captions changed. There is an undocumented property, no idea why, that allows you to change the caption, here is how;

    var tbl = Ti.UI.createTableView({
    data: rows,
    deleteButtonTitle: 'xxx', // enter caption here
    editable: true,
    height: Ti.UI.FILL,
    width: Ti.UI.FILL,
});

    Obviously you use the L macro on the indicated property.
