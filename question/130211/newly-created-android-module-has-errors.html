Newly created Android module has errors

I'm new to using this system, but I followed the steps so I don't know what the problem is. I ran the following command in my terminal:

titanium create --platform=android --type=module --name=*NAME* --id=*PACKAGE NAME* --android=*PATH*

Which created the module I wanted to make. I imported that project into Eclipse and there are errors in both the example proxy and module files:

The type org.appcelerator.kroll.KrollProxySupport cannot be resolved. It is indirectly referenced from required .class file.

and

Cannot find the class file for org.appcelerator.kroll.KrollProxySupport. Fix the build path...

There is no problem with the build path, it references the titanium.jar just fine, but I looked in that jar file and there is no class file for KrollProxySupport so I am a little confused on what I need to do to fix this issue.

Thanks.

  • Is this using 1.8 or 1.7.5? You may want to change your Titanium path in bash_profile to 1.8 and then create a new module, as there were a lot of breaking changes to Android modules due to the change from Rhino to V8. Sometimes I get errors in Eclipse about it not resolving class names, however when I actually do the build it works fine. Have you tried yet building the module?

    Either run the command "ant" from terminal when in the module directory or from Eclipse right click the build.xml > run as > ant build.
