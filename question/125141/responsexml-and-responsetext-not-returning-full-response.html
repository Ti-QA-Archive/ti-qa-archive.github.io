.responseXML and responseText not returning full response?

HI - I'm not sure if this is a bug, or what the issue could be. But I'm using the createHTTPClient to hit a web page that will return some XML. If the web page has a problem (bad parm for instance), it will return this XML: 

<result>
   <field name="success" id="success" value="false"/>
</result>

However the value in .responseXML and .responseText of the request only contains

<result></result>

which I confirmed both in the debug and doing a simple alert(this.responseText).

The createHTTPClient seems to work fine with this same page for a response nested a child lower. For instance, a valid response from the same page produces the xml below, and I can iterate through the fields using doc.getElementsByTagName(&#39;field&#39;) just fine there and all seems well. 

<result>
  <record>
    <field name="first_name" id="first_name" value="Bob"/><field name="last_name" id="last_name" value="Smith"/>
  </record>
</result>

Any idea why I can't see the first result? Does anyone else know if this has been identified as a bug? I'm really surprised that even the .responseText value is empty. Anyone?
