  • Make sure that the file is in the folder: <android-sdk>/tools

    If it's there, try adding that folder to the path on Windows or .bash_profile on iOS.

  • Hi

    Take a look at this guide from the Appcelerator Docs Installing the Android SDK

    If you don't already have the Android SDK Manager then you can download it from this link: http://developer.android.com/sdk/index.html, also found in the guide.

    Once you have the manager installed you can use it to update and manage your Android SDK's, which could help resolve your issue. Please note that certain updates can severely affect compiling to Android and break your builds. This again could be the issue if you are using an SDK which is causing your compiler to error and break.

    Hope this helps.

  • After much searching, the answer is that zipalign needs to be moved from build-tools to tools:

    cp /usr/local/Cellar/android-sdk/23.0.2/build-tools/20.0.0/zipalign /usr/local/Cellar/android-sdk/23.0.2/tools/
