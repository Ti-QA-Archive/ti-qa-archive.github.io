Images - Android vrs iPhone

I've been testing my app today. I've put the final touches on and everything shows up on iPhone, but not on the Android Platform.

Any guidance on what the heck I'm doing wrong. Thx


var win = Titanium.UI.currentWindow;

if (Titanium.Platform.name == 'android') {
    win.backgroundImage = 'images/home.jpg';
}

var photo = Ti.UI.createImageView({
        image:'images/home.jpg',
        top:0,
        left:0,
    width:320,
    height:480
    });

win.add(photo);

    George

    What Android SDK version are you using? There have been improvements to image paths in 1.6. Install the latest Continuous Build 1.6.X (master) which may resolve your issue.

    Paths should be either absolute (leading slash to denote the Resources folder) or relative to the js file context. If android still isn't working, you could always resort to:

    if (Titanium.Platform.name == 'android') {
    win.backgroundImage = '/images/home.jpg';
}

    …where the file is located in your project's Resources&#x2F;images directory.

    See my post in ticket #2585 which may be of interest to you.

    Hope this helps
