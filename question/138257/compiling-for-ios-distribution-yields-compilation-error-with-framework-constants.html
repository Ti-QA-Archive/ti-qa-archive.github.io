Compiling for iOS Distribution yields compilation error with framework constants

Environment

Application type: mobile

Titanium SDK: 2.0.2 (05/22/12 18:05 0f37aba)

Platform & version: iOS 5.0,

Device: iOS Distribution

Host Operating System: OSX 10.7

Titanium Studio: Titanium Studio, build: 2.0.2.201205311912

Building an application for iTunes Store stops with error while compiling the Application.

Android and iOS Development: Runs ok on ios simulator, and in Android Devices and Android Distribution. Fails to compile for iOS Distribution.

The application runs fine on the emulator and Android, but won't compile for iTunes Distribution. Changing the empty character string at line 28 (defines.h) to something not empty doesn´t work as the file also gets regenerated with XCode build.

I Tried changing back to 2.0, then 1.7.6, 1.7.5, 1.74, and all give the same error whe I try to publish for distribution on iOS

Any pointers on what might be wrong? Does everybody have this issue?

    This is related to the Objective-C compilator. You should review your code and change single-quote empty strings to double-quote ones. I found this myself just few minutes after reading your question.

    It is strange that this error pops only when building for distribution while debugging and testing is OK. Let's hope they will fix this in a future version (e.g. to translate '' to "" automatically when building for iOS)
