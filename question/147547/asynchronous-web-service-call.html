  • I have two suggestions:

    • if you can control the back-end web service, modify it to send smaller chunks. Do you really want to send 5MB directly to a mobile app anyway? Have the app request 50KB or 100KB slices of the data. It will introduce some more overhead, but I don't think you will get too much performance hit.
    • or instead of using Ti.Network.HTTPClient, try using Ti.Network.Socket.TCP and implement the basics of the HTTP protocol yourself. Then you can read the response directly and react to it as it comes in. In fact, the Ti.Network.Socket.TCP docs have a perfect example of making an HTTP request and using a Ti.Stream to read 1024-byte chunks.
