click event has x,y coordinates of child view

I have a scroll view with multiple image views in it - the 'background' view and various foreground 'sprites'. A click event handler for the scroll view returns the local coordinates (in the x and y properties of the event) for the topmost image view tapped on instead of for the scroll view.

For example, if the user taps on a sprite, the click event handler returns get the x, y of the tap relative to the sprite. If the user happens to click where there is no sprite, the click event handler returns the x and y of the tap relative to background image view. There seems to be no way of distinguishing the target. I am using Titanium SDK version 3.0.0 on Android 2.3.5 (Samsung Galaxy Player).

    Hi

    As you have not provided any code to check against I am not sure if you are;

    • trying to get the x/y of the scrollView instead of the sprites?
    • or simply how to determine which object returned in the event is a sprite or a scrollView?

    If you only need to get the scrollView x/y then you simply add touchEnabled: false to the sprites.

    If you wish to determine which is which then try this code.

    var scrollView = Ti.UI.createScrollView({
    contentHeight: Ti.UI.SIZE,
    contentWidth: Ti.UI.SIZE,
    id: 'scrollView',
    showHorizontalScrollIndicator: true,
    showVerticalScrollIndicator: true,
    height: Ti.UI.FILL,
    width: Ti.UI.FILL
});
var sprite = Ti.UI.createImageView({
    height: Ti.UI.SIZE,
    id: 'sprite',
    image: 'sprite.png',
    width: Ti.UI.SIZE,
});
scrollView.addEventListener('click', function (e) {
    switch (e.source.id) {
    case 'scrollview':
        // do something
        break;
    case 'sprite':
        // do something
        break;
    }
});
