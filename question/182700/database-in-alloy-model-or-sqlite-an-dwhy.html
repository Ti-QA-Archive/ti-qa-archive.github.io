    Hi,

    with Alloy you define and configure data structures as models in app/models (i.e. books.js); you can choose the persistence layer (local sqlite-db or properties, remote server rest-api) and use a 'sync adapter' to access and store data using Alloy's Models and Collections APIs.

    In case of local database the adapter is called "sql" and it gives you all the CRUD actions on single models and fetch (select * from …) and filter (… where..) actions on the collection itself.

    Refer to Alloy Sync Adapters documentation for all the details. Hth.
