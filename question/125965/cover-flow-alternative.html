Cover Flow alternative

Hello there,

(I now its a lot of text for some basic questions but I don't speak english very well so I have to write a lot to compensate my lack of vocabulary.
Thank you for your understanding).

First of all I want to insist on the fact that I'm a complete noob.
This is my first steps within Titanium AND Javascript.
So far so good, I have been able to create a nice interface which is orientation dependent, with views, frames, images and even a cover flow.
Appcelerator is impressive.

However, I'm not that impressed regarding the embedded Cover Flow.
I find it quite rigid and somehow…"ugly":

-The covers on the sides pixellate a lot even if my pictures have a decent resolution.

-The height of the covers on the sides is way to small compared to the selected cover.

-The mirror effect is too massive.

-I can't modify to my taste the display angle of the covers on the sides.

It doesn't seem that I can do much about these 4 complains as there is no parameters I found which allow me to do something about it and there is no source code I can mess with.

On the other hand, I tried the Cover Flow proposed by "JQTouch" and even if I don't like its default setup, I managed to edit the source code in order to achieve exactly what I want.

Now, the trick is that it seems I can't integrate natively JQTouch within Appcelerator. According to what I read, I must create a web view…
I'm not that fond of the web view because of the display latency and because it also implies that the user is connected to the internet…

Here come my questions:

1) According to this all, is there a way to customize the embedded Cover Flow?

2) If not, is there a way to use JQTouch without the need of a web view?

3) If a web view is the way to go, is it possible to trigger local Titanium windows from the web view html links? In other words, would I be able to click on a particular cover on the web view and expect it to lead me to a local view on my iPad?

Thank you for your help!

  • (1) Beyond the CoverFlow properties and methods exposed by the current API, the only way you'll be able to modify and extend the CoverFlow is by either creating a native module that has the new functionality you need, or by submitting a pull request to the mobile SDK github repository. Titanium mobile is open source and available for community contribution.

    (2) No

    (3) Yes, by using what we call "application level events" from Titanium.App. Here's a very basic example that we use a lot for opening links from a webview in the native browser. You can extrapolate it to your needs:

    yourwebview.html

    <html>
  <body>
    <a href="http://www.appcelerator.com" onclick="Ti.App.fireEvent('app:openURL', { url:'http://www.appcelerator.com' }); return false;">This link will open in the native browser</a>
  </body>
</html>

    app.js

    Ti.App.addEventListener('app:openURL', function(e) {
    Ti.Platform.openURL(e.url);
});
