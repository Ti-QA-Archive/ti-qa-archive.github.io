Check for image size before letting the user load it from a local path

Hi, one question. I want to load an image from the device (iOS) into an imageView (or as a background). I don't know what the image size will be, so the user can insert a 200x200 image, a 400x250, etc. Before the user can do so, I want to check for the image size and only after this check allow him to insert the image. Is it possible (and eventually how) to check for a local image size (w and h) before inserting it into an imageview or component background?

  • Hi, You can use following code 

    var imageTemp = Ti.UI.createImageView({ image: 'images/star.png' });
var starWidth = imageTemp.toImage().width;

    After this based on the condition you can decide whether you want to add the imageView or not .

    If you want to use image saved on you device gallery, then you can use 

    Ti.Media.openPhotoGallery({});

    Using this you can set the image for the imageView and the use the same code as mentioned above.

  • Yes. Ti.Blob has height & width properties. How do you get a Ti.Blob? Well, that's what the gallery & camera return. Or, you can use Ti.Filesystem.getFile(&#39;full&#x2F;path&#x2F;to&#x2F;your&#x2F;image.png&#39;).read()
