Titanium version upgrade problem from 3.1.3 to 3.2.0

Hi

I had upgraded my titanium sdk version from 3.1.3 to 3.2.0 and I found some problems in my currently running app.

In tiapp.xml file(for both 3.1.3/3.2.0)

1)&lt;navbar-hidden&gt;true&lt;&#x2F;navbar-hidden&gt; in tiapp.xml file

2) android:screenOrientation=&quot;portrait&quot; is set for portrait mode only

Above same setting was working fine in 3.1.3 but it's not hiding nav-bar and set orientation once i had upgraded titanium sdk to 3.2.0

I am using alloy framework

    For navBarHidden, try setting it in your controller or app.tss. In the tiapp.xml, the property would apply to iOS only (it's a legacy setting that I think is deprecated).

    $.index.navBarHidden = true;

// or in the app.tss or index.tss
"Window": {
  navBarHidden: true
}

    For the orientation, confirm that you're setting that attribute on the correct activities, that you have all activities listed, and that you're setting the property on all activities.

    Keep in mind that all windows are now heavyweight (ie. activities) in 3.2 so there are probably new activities in your app since you generated the manifest back with 3.1.2. You will need to re-generate the AndroidManifest.xml file (remove all your customizations from tiapp.xml, build, copy the pertinent sections from the generated file back to tiapp.xml, edit to set orientation).

    In general, the very last thing you should do before publishing is to set manifest customizations so that you're sure you're getting all the activities and relevant tags.

    If after doing that, you still see an issue, create the smallest possible example app that demonstrates the issue and file a Jira ticket.
