    Hi

    In answer to your actual question "what happens if I do not assign a classname" - nothing, not requirement to set a classname (not to be confused with Alloy class) for Android iOS or any other platform.

    The negative of not assigning one correctly for each unique layout (not data) is the list will not be assisted by your hint as to how to handle row caching on the platform. So your table may be slower for small tables and increasingly slower for larger tables. This is more important on Android as iOS does a lot of smoke and mirrors to allow tables to speed past that Android does not.

    Your errors are not related to this. As Ashish has said you need to provide some code to help diagnose that. Please make sure you use the correct code format this time though.
