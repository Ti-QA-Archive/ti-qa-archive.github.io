Setting components in a window before opening

I have a ListView populated with some tasks. When I click on an item, a window containing the details of that task is shown. I'm doing it as follows:

function itemclick(e){
     var ntarefa = Alloy.createController('detalhes', { tarefa: $.tarefasList.sections[0].getItemAt(e.itemIndex) }).getView();
    if(OS_IOS){
        $.navGroupWin.openWindow(ntarefa);
    }
    if(OS_ANDROID){
        ntarefa.open();
    }
}

I also have an onOpen listener attached to detalhes.js's window, which gets the tarefa object sent and sets the textfields in detalhes.js.

The problem with this is that on both the iOS simulator and Android emulator, the setting process happens slow enough that I am able to see individual textfields being set after the window opens.

This might just be the nature of the slowness of the si/emulator, but incase it is not, is there a way to create the window and set the textfields before calling .open()?

    Move the code that "gets the tarefa object sent and sets the textfields in detalhes.js." to outside the onOpen function, example:

    var args = arguments[0] || {};

function onOpen(e) {
    //Your code...
}

    change to 

    var args = arguments[0] || {};

//Your code...

function onOpen(e) {
}

    You are seeing the individual text fields being built because you arent populating them until the window opens, the code in the second part populates them before the window is opened.

    without you showing your code in detalhes.js, thats the best you are likely to get.
