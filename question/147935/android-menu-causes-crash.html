Android menu causes crash

I'm developing the app, currently for the android, using the Alloy. It has simple window with a webview in index.xml and if worked nice untill I've added the simple android menu according to Alloy Docs:

<Menu id="menu" platform="android">
  <MenuItem id="logoutMenuItem" title="Sign out" icon="ic_menu_sign_out.png" onClick="doLogout" />
</Menu>

Now the App throws the exception:

[ERROR][TiExceptionHandler(  428)] (main) [1696,4762] ----- Titanium Javascript Runtime Error -----
[ERROR][TiExceptionHandler(  428)] (main) [1,4763] - In ti:/window.js:314,34
[ERROR][TiExceptionHandler(  428)] (main) [0,4763] - Message: Uncaught TypeError: Cannot call method 'extend' of undefined
[ERROR][TiExceptionHandler(  428)] (main) [1,4764] - Source:             this.window._internalActivity.extend(this.cachedActivityProxy);
[ERROR][V8Exception(  428)] Exception occurred at ti:/window.js:314: Uncaught TypeError: Cannot call method 'extend' of undefined

I've also tried an old-school method (based on an example from Docs):

    var activity = $.index.activity;
    activity.onCreateOptionsMenu = function(e){
    var menu = e.menu;
    var menuItem = menu.add({ title: "Sign out" });
    menuItem.icon = "ic_menu_sign_out.png";
    menuItem.addEventListener("click", function(e) {
    Ti.API.debug("I was clicked");
  });
};

But with same result.

I've found just few relevant mentions in Google about the issue, however it looks to be fixed about a 3-4 months ago. I've used SDK version 3.0.0.GA as well as I've tried 3.0.2.v20130201161712 (latest), but have same result. So is there any solutions, patches or something? Or I just do something wrong?
