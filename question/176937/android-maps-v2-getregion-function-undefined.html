Android Maps v2 getRegion function undefined

It seems something broke in 3.3.0.
Calling getRegion or accessing the mapview.region getter results in undefined.

Works as expected on iOS. All Android devices either the function is undefined or the getter returns undefined.

Any ideas anyone?

  • Sure. It is very simple. 

    <Alloy>
    <Window class="container">
        <Module id="mapView" module="ti.map" method="createView" />
    </Window>
</Alloy>

    ".container": {
    backgroundColor:"white"
},
'#mapView': {
   mapType: Alloy.Globals.Map.NORMAL_TYPE
}

    function fireAfterOpen(evt) {
    alert('region is:' + JSON.stringify($.mapView.region));
};

$.index.addEventListener('open', fireAfterOpen);
$.index.open();

    On iOS, the alert will display the region information. On android it return undefined.

    You can change the event to be fired on a setInterval too. It isn't an issue where the map hasn't been instanced yet. Region just comes back undefined at any call now.

    This was not an issue with 3.2.3 and Alloy 1.3 Something is amiss with the new SDK.
