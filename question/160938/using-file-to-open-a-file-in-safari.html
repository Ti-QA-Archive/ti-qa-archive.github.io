Using File:// to open a file in Safari

Dear All,

I'm trying to get Safari to open a non HTML file. Here's the code I'm using:

var myFile = Ti.Filesystem.getApplicationDataDirectory() + fileName;
Ti.Platform.openURL(myFile);

Safari doesn't load when Ti.Platform.OpenURL is called. However, it launches Safari when I change 

file:// to http://

Now, here's the interesting part:

Safari opens my URL (when using the http:// method) - and I get the obvious error:

"SAFARI COULD NOT OPEN THE PAGE BECAUSE IT COULD NOT CONNECT TO THE SERVER"

Here's the "fake url"

http://Users/username/Library/Application%20Support/iPhone%20Simulator/6.1/Applications/66508D69-C14A-4605-826C-EEAB5E54F8D1/Documents/asfasf.cml

Now, here's the amazing thing:

If I changed the above link to file:

file://Users/username/Library/Application%20Support/iPhone%20Simulator/6.1/Applications/66508D69-C14A-4605-826C-EEAB5E54F8D1/Documents/myFile.cml

… then Safari opens my file EXACTLY the way I want.

So - what I'm trying to say is that this isn't a question of the file being in some sandbox where it cannot be opened. Its a problem of making Safari respond to url's beginning with File://

Now, i've tried: "file://", and "file:///" and "file:////"

  • I've also tried URL schemas, to add "file://" as a method or scheme, and this hasn't worked. Have I done this wrong?

Is there anyone who has done this before?

Thanking you in advance,
EB
