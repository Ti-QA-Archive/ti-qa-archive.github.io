    • Go into your iphone build folder <my_project>/build/iphone
    • Run the file _<my_appname>.xcodeproj which should launch xcode with this project opened.
      • If it's just a folder or xcode doesn't start up automatically go to /Developer/Applications/ and run xcode manually for it's first run setup stuff it does. Now clicking on the .xcodeproj file should work now.
    • On the window that opens top left you'll see "Run", "Stop" and a dropdown. Change this to be _<your_appname>-universal | iOS Device (will be a device name if one is plugged in). Basically make sure it isn't set to a "simulator".
    • Now if you don't actually have a device connected (you have the iOS Device showing up in the dropdown) then go to the main menu at the top of your screen for xcode and select "Product" then select "Build" this will do a build but not actually run it. If it cannot build the code it will let you know.
    • This doesn't test the issue of the wrong production certificates being installed. To test those you'll need to setup itunes connect for this app first. Then there's other steps I can give you to validate/upload the binary.
    • Also check the Organizer (button on the far right of the project window) Devices tab to ensure that your Provisioning Profiles are valid (they will have notes in the Status column if they do).
