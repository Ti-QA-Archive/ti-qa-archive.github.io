Language Switch Option

Hello, am looking at a scenario where I need to do a language switch in my mobile app. The languages switch is between Arabic and English. I came up with two views to manage it. However, as a best practice am not supposed to multiple views and restrict myself with a single view. However for English-Arabic conversion the entire layout and the language changes. How are we supposed to balance this challenge?

  • This isn't an answer to your question, but you should know that there does not appear to be the correct 'right-to-left' ability of labels, to display Arabic text correctly. (textAlign:right is not correct, as you probably know).

    I recently submitted a bug report about this on Jira : http://jira.appcelerator.org/browse/TC-1469

  • You can access the layout and the language settings using your view as an object.
    Example view.font.fontSize = 13; or view.text = &#39;my Arabic text&#39;;

    I hope this helped.

    • Hello Mike, I understand this. However am pondering if its fair to do this as the entire layout is affected. So for every element I need to reverse this. So am kind of using a recursive call to make this work for me.

      function change(x){
      if(x.children){
      for(var r = 0; r<x.children.length;r++){
      change(r.children[r]);
      }
      changedirection(x);
      } else {
      changedirection(x);
      }
      }
