    • I have the same problem, and it occurs on almost every new app I create. Has to do with path name character case mismatch between Titanium and Xcode, I think, but I've never been able to fix Xcode to make it work. There is a simple, one-line fix to the build script, but it must be made with each release (wish Ti would fix the script once and for all). In Ti 1.6.1, fix the line in /Library/Application Support/Titanium/mobilesdk/osx/1.6.1/iphone/builder.py (line 1029):

      if target_build_dir!=build_dir:

      to

      if target_build_dir.lower() != build_dir.lower():

      and it won't show up any more for that release.

  • Seems like this is a bug from if you ever ran Xcode 3 on your system and/or changed the build directory.

    Here's how I solved it:

    • Download latest Titanium (I used SDK 2.1.0)

    • Download latest Xcode (I used 4.4)

    • Navigate to ~/Library/Preferences and trash everything that has Xcode in it.

      For me, that was com.apple.dt.Xcode.LSSharedFileList.plist, com.apple.dt.Xcode.plist, and com.apple.Xcode.plist
