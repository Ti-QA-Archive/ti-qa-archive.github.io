Image upload over HTTPS

I've been successfully uploading an image with a simple HTTPClient POST request to our web server. We recently switched to using HTTPS and found that on iOS this no longer works. The web server receives a content length of 0.

Is this possibly a bug in the iOS Titanium implementation?

var loader = Titanium.Network.createHTTPClient({timeout:30000});  
// setup the client  
var url = "https://www.ourserver.com/api/photos/";
loader.open("POST",url);  
Ti.API.debug("Preparing httpclient for url " + url);
// setup the callback handlers
var callback = function(ev) {
    Ti.API.debug("HTTPClient responseText = " + this.responseText);
    if (!ev || ev.error) {
        Ti.API.debug("Something went really wrong!");
    } else if (this.responseText) {
        var data = JSON.parse(this.responseText);
        if (data.errors) {
            Ti.API.debug("Error response " + JSON.stringify(data.errors));
        } else {
            Ti.API.debug("Success!");
        }
    }
};

loader.onload = callback;
loader.onerror = callback;

loader.onsendstream = function(e){
    Ti.API.info('ONSENDSTREAM - PROGRESS: ' + e.progress);
};
loader.send({image:media});

In the above example media is an image blob. Like I said this worked perfectly with a http request, and all our other HTTPS requests (login, various other posts/gets) all work fine. But when doing this upload, the content-length header is 0.
