View containing a ScrollableView containing buttons

Hi,

So, I have a View ("categories"), which containes 2 ImageViews (for more left and more right images), and a ScrollableView with 2 views containing 3 buttons each. Each of the buttons is for a "category". This is all created in a "utils,js" file, which is included in every other javascript file in the project.

So, on the main window have Tabs on the top. Each Tab has a window associated with it, calling its own javascript file. The "categories" view is created on every page, and in the respective Javascript file, there is a "categories.addEventListener(…) method added, to load the respective data for the selected Category.

So, the problem is that somehow, when I click on a particular category or change it, the event listener is not triggered, though the "selected Category" index is updated.

The categories.addEventListener(…) is called only when I click on the More Right and More Left images.

I tried recreating the views in different ways, but still the same result. Is it because the buttons are not on the Categories View directly? This same code works in iOS.

Can someone help me or give me a hint, please? I can post code if needed.

Sri

  • I got it to work. The code in the pastie.

    The "bottom:0" for the views is the trick.

    Also, on the calling page, the view had a top:30, which was causing a band of height 30, above the data view, which made me this that it was this.

    The third thing that did it was - on the calling page, add the data view first and then the categories. Earlier, I was adding categories first and then the data view, so even after you change bottom:0, you still cant see the categories, as the data view covers them. I could figure this out only when I gave different colors to the windows and views.

    Hope this helps someone!
