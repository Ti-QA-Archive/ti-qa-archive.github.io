Including all the modules. OK or bad tactic?

Hi,

I'm thinking to do the following in order to shorten a little bit my code. Let's say I have 10 module js files:

var m1 = require('js/m1');
var m2 = require('js/m2');
var m3 = require('js/m3');
var m4 = require('js/m4');
var m5 = require('js/m5');
var m6 = require('js/m6');
var m7 = require('js/m7');
var m8 = require('js/m8');
var m9 = require('js/m9');
var m10 = require('js/m10');

I use some of them each time in every js file logic, even inside these modules. I was thinking to hold these references I created above in a separate js file which I will include every time:

Ti.include('js/global.js');

in order not to type the requires all the time but just call the short named reference. (so for example m1.myMethod(); )

Do you think this is an ok practice? Or is it risky memory-leeks-wise?
