Tutorial (Favebook) bugs?

Doing one of the first tutorials from the Quick Start: Creating your first Titanium app (FaveBooks) I run into multiple problems. It seems this tutorial was never tested. filenames change halfway (bookdetail becomes bookview). The detailview is not showing the details because it seems not to be linked to the database. I had a more experienced developer looking over my shoulder, but we both thought it was strange how the project is implemented. The rowdata is send to the detailview, not the actual database entry, so the author data can never be retrieved.
This seems one sloppy tutorial, and it probably drives a lot of beginners away.
(Or am I completely missing something here…?)

    Full Sample Code

    You will need to have this files in order to run the sample:

    Views

    app/views/index.xml

    <Alloy>
    <Collection src="books"/>
    <Window class="container">
        <TableView dataCollection="books">
            <TableViewRow title="{title}" onClick="showBook"></TableViewRow>
        </TableView>
        <Menu id="menu" platform="android">
            <!-- Cannot specify node text.  Use attributes only. -->
            <MenuItem id="addBook"
            title="Add book"
            onClick="addBook"
            showAsAction="Ti.Android.SHOW_AS_ACTION_IF_ROOM" />
        </Menu>
    </Window>
</Alloy>

    Controllers

    app/controllers/index.js

    var myBooks = Alloy.Collections.books;

var book = Alloy.createModel('books', { 
   title : 'Great Expectations', 
   author: 'Charles Dickens' 
});

myBooks.add(book); 
book.save();

function showBook(event) {
    var selectedBook = event.source;
    var args = {
        title: selectedBook.title,
        author: selectedBook.author
    };
    var bookview = Alloy.createController("bookdetails", args).getView();
    if (OS_IOS) {
        $.navGroupWin.openWindow(bookview);
    }
    if (OS_ANDROID) {
        bookview.open();
    }
}      

function addBook(){
    var myAddBook = Alloy.createController("addBook",{}).getView();
    if (OS_IOS) {
        $.navGroupWin.openWindow(myAddBook);
    }
    if (OS_ANDROID) {
        myAddBook.open();
    }
}

if(OS_IOS) { 
   $.navGroupWin.open(); 
} 
if (OS_ANDROID) { 
   $.index.open(); 
}

    app/controllers/bookdetails.js

    var args = arguments[0] || {};
$.titleLabel.text = args.title || 'Default Title';
$.authorLabel.text = args.author || 'Default author';

    app/controllers/addbook.js

    var myBooks = Alloy.Collections.books;

function addBook() {
    var book = Alloy.createModel('books', {
        title : $.titleInput.value,
        author : $.authorInput.value
    });
    myBooks.add(book);
    book.save();
    $.addbook.close();
}

  • I really think the tutorial is incomplete. For completeness it would be very beneficial to have a delete example in addition to an add example. I submitted a pull request to the documentation project: https://github.com/appcelerator/Documentation-Examples/pull/5 but here is the code in case you want to add it:

    app/controllers/index.js

    function deleteBook(event) {
    var alertDialog = Titanium.UI.createAlertDialog({
        title: 'Remove',
        message: 'Do you want to remove this row?',
        buttonNames: ['Yes','No'],
        cancel: 1
    });
    alertDialog.show();

    alertDialog.addEventListener('click', function(e){
        // YES
        if(e.index == 0) {
            myBooks.at(event.index).destroy();
        }
    });
}

    app/views/index.xml

     <TableView dataCollection="books" onSingletap="showBook" onLongclick="deleteBook">
      <TableViewRow title="{title}" author="{author}" color="black"></TableViewRow>
</TableView>

    app/vies/ios/index.xml

    <TableView dataCollection="books" id="bookTable" onClick="showBook" onLongpress="deleteBook">
      <TableViewRow title="{title}" author="{author}"></TableViewRow>
</TableView>
