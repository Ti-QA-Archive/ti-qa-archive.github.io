    As Manuel says, the recommendation about JSON probably doesn't apply in your situation. Then again, are you re-fetching and parsing that JSON data each time the screen loads? If so, that would certainly cause a big slow-down. Cache the data locally (in a file or database, or just in memory).

    Screen transition slowness is likely due to other factors, notably complexity of your UI. You didn't mention whether you're developing for iOS, Android, or both. Optimization will vary a bit, with Android being more sensitive to UI complexity. The optimization steps you could take really depend on what you're doing in your app. Without knowing more, I can't offer too much more than broad generalizations:

    • Limit the number and size of images
    • Avoid transparency (set solid background colors instead)
    • Avoid rounded borders, especially on Android
    • Avoid gradients, especially on Android

    I'm not saying never use these things. They're all valid app components. But the more complex your UI, the more work the device must do to render your UI. You have very tight memory constraints with mobile. And mobile devices generally lack dedicated GPUs.

    Finally, what the JSON recommendation is saying is that given a choice between hard-coding variable values in one of your JavaScript files versus shipping a JSON configuration file that gets parsed at runtime, choose the JavaScript file approach.

  • I'd also add to that - if you're fetching JSON data from a remote server for a Window (within a tab) then ensure you're making that remote server call AFTER the tabbed Window has loaded. You can either use the 'focus' event or if you must, a setTimeout(). This ensures that any remote data fetching won't be blocking your UI.
