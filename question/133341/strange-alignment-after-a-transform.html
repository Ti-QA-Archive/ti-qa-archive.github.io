Strange alignment after a transform

Android: 2.2
SDK: 1.8.2
Host: Ubuntu 11.10

I have the following code which I want to use to draw some text rotated and aligned against the left hand edge of a view.

Ti.UI.setBackgroundColor('#000');
var win = Ti.UI.createWindow({});

var view = Ti.UI.createView({
    backgroundColor:'#202020',
    width:75,
    height:150,
    top:50,
    left:10
});

var lbl = Ti.UI.createLabel({
    text: "text",
    width: 100,
    height: 31,
    backgroundColor: '#6d6d6d',
    color: '#3e3e3e',
    textAlign: 'center',
    left: 0,
    transform: Ti.UI.create2DMatrix({rotate:-90})
});

view.add(lbl);
win.add(view);
win.open();

As it stands this fails because the label is offset from the left hand edge of the view.

If I set the left value of lbl to -34.5 this then aligns the left hand edge of the label with the view -> as I require.

Analysis shows this value can be calculated from (31/2)-(100/2). Which seems to indicate the bounding box of the label has not been updated to reflect the post transform coordinates.

Is this a known issue or a bug? Or am I making some incorrect assumptions?
