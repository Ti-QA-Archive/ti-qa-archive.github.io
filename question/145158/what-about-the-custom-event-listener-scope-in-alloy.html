What about the custom event listener scope in Alloy?

I have found some source code with CodeStrong sample project, it's defined a custom event in main view, like following:

Ti.App.addEventListener('app:logout', function() {
    // do something...
});

I just want to know whether it's a global event listener?( just like define in a app.js? ) If yes, then should we release it to avoid Memory leaks ?

  • Accepted Answer

    Any event listener attached to App is an app level event listener. If you find that you are leaking memory with your event listeners, then you should plug those leaks regardless of whether or not its a global event listener.
