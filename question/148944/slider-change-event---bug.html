Slider change event - bug?

Hi, guys. Unfortunately I have big problem with slider "change" event ( Just on real device, on emulator is app working without problems ).

var win = Ti.UI.createWindow({
    backgroundColor: 'yellow'
});

var slider = Ti.UI.createSlider({
    width: '90%',
    min: 0, max: 110,
    minRange: 5, maxRange: 105,
    leftTrackImage: 'images/sliderLeftTrack.png',
    rightTrackImage: 'images/sliderRightTrack.png',
    thumbImage: 'images/sliderThumb.png'
});

slider.addEventListener('change', function(e){    
    alert("Pos: "+e.value);
    // calculations...
});

win.add(slider);
win.open();

Problem is that when I start application "change" event listener automatically fire twice alert (Pos: 5) and this is big problem for me because I have a lot of calculations in function which are uncorrect in this case. On emulator is application working without problems so "change" listener fire just when I touch and move with thumb of slider.
