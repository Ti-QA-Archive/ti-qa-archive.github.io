How to access a view from another controller without to create the controller corresponding to the xml file

I've a View with id="basewin" and an ImageView with id="menubtn" which looks like a button and when i click it the basewin's left will be animated to another position in the screen (from -10dp dp to 254dp).

this is my index.xml file where i've written all these Views: 

<Alloy>        
        <Window class="indexwin">

            <View id="quickmenu">
                <Require src="quickmenu" type="view" />
            </View>            

            <View id="basewin">
                <View id="shadow"/>
                <View id="basewincontent">

                    <View id="header">
                        <ImageView id="menubtn" image="images/menubtn30x42.png" />    
                        <ImageView id="booknightlogo" image="images/booknightlogo.png" />                            
                    </View>
                    <ImageView id="ads0" image="images/ads0.png" />

                          <View id="accueilscreen">                    
                                 <Require src="accueilscreen" type="view" />
                           </View> 


                         <View id="detailsoireescreen">
                             <Require src="detailsoireescreen" type="view" />
                         </View>  

                         <View id="lessoireescreen">
                             <Require src="lessoireescreen" type="view" />
                         </View>                    
                </View>                            
            </View>        

        </Window>    

</Alloy>

the initial View left value was declared in the .tss file as : 

"#basewin":{
    left:-10,
    width:330,
    height:Ti.UI.FILL,
    backgroundColor:"transparent"
}

When the basewin View moves, another View(with id="quickmenu") which was hidden behind it appears and inside the quickmenu View (which its content is written in the quickmenu.xml and added with the <Require src="quickmenu" type="view" /> ) there is a TableViewRow id="lessoires" for which i've added the following eventListener on the quickmenu.js controller :

$.lessoires.addEventListener("click", function() {

    var index = Alloy.createController('index');

    index.basewin.animate({
        left : "290dp",
        duration : 2500,
        curve : Ti.UI.ANIMATION_CURVE_EASE_IN_OUT
    });

So, after that the first left animation has moved the basewin from (-10dp to to 254dp), clicking lessoires's TableViewRow will move it again to 290dp (this is what the added eventListener is supposed to do).
But what actually happened is that a new instance of the whole content of index.xml appears in the screen (then i see a new instance of the basewin view on its initial position (-10dp) and then this new instance will be moved from -10dp to 290dp. (i've set duration : 2500 in the eventListener to understand what is happning slowly).

So instead of that the first instance will continue to move from 254dp to 290dp as i was expecting to happen, the first instance will remain at 254dp and the new added instance will appear and will move from -10dp to 290dp.

I think that Alloy.createController("index") has created the new view instead of getting the old view and what i need is something like Alloy.getController("xxx") but this doesn't exists on the framework API and now i'm blocked as i don't know another way to access the basewin view (to make the second move correctly) from the quickmenu controller.

Thanks for help.
