Windows 7 (x86) Titanium Only Runs Once Successfully

First off, I like what I've seen so far, Titanium seems like a really slick middleware solution to a painful development problem.

Okay, now for the real problem. After getting a successful installation of Titanium on 32-bit Windows 7 Professional (running as a virtual machine on Oracle VM VirtualBox from an Ubuntu 10.10 host OS), I can run and build projects to my heart's content.

However, as soon as I close the application, I'm locked out forever with the message:

Titanium Developer appears to have encountered
a fatal error and cannot continue.

The application has collected information about
the error in the form of a detailed error report.
If you send the crash report, we will attempt to
resolve this problem.

The only way to get back in is to delete the following folder:

C:\Users\user.name\AppData\Roaming\Titanium

Then I have to set it up all over again and reimport any projects. Any ideas?

    • Christopher

      Bearing in mind you have MS SQL Server running on your system, I doubt 2GB total memory is going to be adequate, due to the Titanium SDK Minimum Memory requirement. Raise it to 3GB.

      Your configuration is almost perfect, but you should remove the following obsolete Android packages (close Titanium Developer and the emulator first):

      • SDK Platform Android 2.3.1 API9, revision 2 (Obsolete)
      • Google APIs by Google inc, Android API9, revision 2

      Once you have done this, remember to delete all your existing AVDs using the Android tool, then start Titanium Developer again and use, as you normally would.

      Let me know the outcome.

      Cheers

      — commented April 14th 2011 by Paul Dowsett

    • After bumping the RAM up to 3GB and removing the obsolete packages and AVDs, it appears that Titanium is starting every time now. I guess that Java Heap issue is a pretty big deal after all! SQL Server was a non-issue, because I don't have any instances running, but I can see where that would have contributed to the memory problem as well.

      Fast service with a smile, thanks Paul!
