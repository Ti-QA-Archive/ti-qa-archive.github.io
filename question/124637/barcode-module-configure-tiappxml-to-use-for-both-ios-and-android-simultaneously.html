Barcode Module, Configure TiApp.xml to use for both iOS and Android Simultaneously

So I have an app that will run on both iOS and Android devices. This app uses the ti.barcode module. I an successfully run the app on either platform, but am having issues figuring out how to get both to run without having to change the module version manually when I switch os types.

everything is deployed correctly (I think :)). I have my Android module at: /modules/android/ti.barcode and my iOS modules at: /modules/iphone/ti.barcode

when I go to the tiApp.xml to add these in, if i have:

<modules>
    <module version="1.0">ti.barcode</module>
    <module version="1.1">ti.barcode</module>
</modules>

I receive the build error on iOS: [ERROR] Third-party module: ti.barcode/1.0 missing library

The same thing, just flipped for Android. now if I remove the Android version when running iOS (and vice versa), everything runs great. so for example, this is my iOS config:

<modules>
    <module version="1.1">ti.barcode</module>
</modules>

I can make my builds for now manually, but it really seems like this is a big hole in a multi-platform solution like Appcelerator, so i figure i must be missing something.

any help?

    Try changing your tiapp.xml to read:

    <module platform="android" version="1.0">ti.barcode</module>
