Android Density Images?

I added all the density images to the "android/images/", but whenever I launch the App in Android I get this in the console:

[WARN] For your density-specific images (android/images/high|medium|low|res-*) to be effective, you should put a <supports-screens> element with anyDensity="true" in the <android><manifest> section of your tiapp.xml or in a custom AndroidManifest.xml

But I'm not really that… Efficient in the "tiapp.xml" or "AndroidManifect.xml" files and not really sure what to do with the "anyDensity" flag. Any help? Seems I just have to add that element between <android><manifest></manifest></android> and the densities will work?

PS - If it helps, I am using Titanium Mobile SDK 1.6.0 (Latest nightly build) and Android API 2.3, as well as Titanium Developer 1.2.2.

  • Due to the various sizes, resolutions, orientations available for Android screens and how Titanium sets up your application….by default you code you app and it's assuming the screen size is a standard 320 x 480 resolution. If the actual screen is smaller or larger android will do it's best to scale to the appropriate size. If you want to have multiple resolution images for Android you need to disable the automatic scaling and code for all the different screen sizes on your own (and test them). It might not be worth the effort to you. Anyways, to enable said properties open up your tiapp.xml file and make sure it looks something like:

    <?xml version="1.0" encoding="UTF-8"?>
<ti:app xmlns:ti="http://ti.appcelerator.org">
...
    <android xmlns:android="http://schemas.android.com/apk/res/android">    
        <manifest>
            <supports-screens android:smallScreens="false"
                        android:normalScreens="true"
                        android:largeScreens="true"
                        android:anyDensity="true"
            />
        </manifest>
    </android>
    <modules>
    </modules>
</ti:app>

    You can read more about android screen sizes and scaling here.

    Note that due to the standardization getting the higher resolution images on iOS is much simpler.
