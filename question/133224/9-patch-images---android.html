9 patch images - Android

For a project I'm working on, I would like to use 9 patch (.9.png) images. I've tried many things myself but it seems that Titanium/appcelerator crashes when it starts compiling if there are .9.png images in the project. I have not been able to get this working. When I 'Googled' the problem I found that some people had the same issues with the 1.7.1 and 1.7.2 versions of appcelerator, but I never saw or found any solution (also not as a response to those issues).

Is there anybody who knows how to use and handle 9 patch images in appcelerator?

    Tymen

    Place the following images in your &#x2F;images&#x2F; directory:


    Which are available from the KitchenSink:

    https:&#x2F;&#x2F;github.com&#x2F;appcelerator&#x2F;titanium_mobile&#x2F;raw&#x2F;master&#x2F;demos&#x2F;KitchenSink&#x2F;Resources&#x2F;images&#x2F;custom-slider-right.9.png

    https:&#x2F;&#x2F;github.com&#x2F;appcelerator&#x2F;titanium_mobile&#x2F;raw&#x2F;master&#x2F;demos&#x2F;KitchenSink&#x2F;Resources&#x2F;images&#x2F;custom-slider-left.9.png

    You will note that they are simply standard png images, with a transparent border of a single pixel. Wherever you want the images to be stretched, color that border with a solid color (black is used here). You can use a image editor, or use the draw9patch utility in the android sdk tools directory.

    Then replace your app.js file with the following code, and clean your project to ensure that the images are copied over.

    Note the images are referenced in the code without the .9 part of the filename.

    var win = Ti.UI.createWindow({
    backgroundColor: 'white',
    exitOnClose: true,
    fullscreen: false,
    title: 'Click button to test'
});

var button = Ti.UI.createButton({
    backgroundImage: '/images/custom-slider-right.png',
    backgroundSelectedImage:'/images/custom-slider-left.png',
    title: 'Click me!',
    top: 10,
    width: 300,
    height: 200
});
button.addEventListener('click',function(e){
    Ti.API.info("You clicked the button");
});
win.add(button);
win.open();

    This has been tested and works with Titanium 2.0.0 (03/01/12 14:01 0ccdf63) - continuous builds master branch with v8 and Android 2.2 APIs.

    Good luck!

    • Just to add to the above, although I stated that you could use any solid color, it should indeed be black (#000). Also, the top and left guides, which determine how the image is scaled, should be filled, whereas bottom and right guides, which determine the content area, are optional.
