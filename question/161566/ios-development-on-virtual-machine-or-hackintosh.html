  • Hi

    The short answer is no.

    The longer answer is;

    The ability to run virtual machines running OSX on windows is limited at best and simply unworkable (at this level) at worst.

    The OSX licence does not allow you to run it on anything other than a Mac device (think Apple created only), this also means running on a "hackintosh" is also prohibited.

    This does not mean it cannot be done on a hackintosh - but it does mean that you may very well have issues in the future with certificates even if you are happy to ignore the licence issues. The reason I state this is certificates are created based on hardware and the hardware is not from Apple so at some point (not sure if already) this will cause problems.

    XCODE also has licence restrictions so that it can only be run on an Apple computer running a suitably licensed OSX.

    You could rent a virtual could based mac - although the costs for full time development very quickly become prohibitive. Mac in Cloud is one such example.

    So the only real and sensible solution is to own an actual Apple Mac. I understand cost is a factor and a large factor but not all Macs are expensive any more. You have more than one avenue to explore in sourcing an inexpensive Mac.

    • Buy a reconditioned Mac from Apple - they come with the same warranty and are usually around 20-25% cheaper than normal
    • Buy a previous generation from eBay, you do not need the latest Mac just something that can take Mountain Lion at least and ideally suitable from Mavericks (which is a lot of macs for the last few years)
    • Think MacMini - the smallest desktop and still the best price to performance value of ALL the macs - it is not the fastest of all the Macs but it is more than fast enough to do any and all development - I have an office full of them doing all development on them. The current model is £500/$599 much cheaper than all other macs and you can add your own keyboard, mouse etc.

    I hope this helps answer your question.
