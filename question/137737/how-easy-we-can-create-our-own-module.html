  • It's actually very easy to create your own modules, using the CommonJS. Consider the following example which we are creating a module that creates a view.

    /src/common.js

    exports.createArticleView = function(e) {
    var articleContainer = Ti.UI.createView({
    layout:"vertical"
    });
    var articleTitle = Ti.UI.createLabel({
        text:e.title
    })
    var articleText = Ti.UI.createLabel({
        text:e.content
    })
    articleContainer.add(articleTitle);
    articleContainer.add(articleText);    
    return(articleContainer);
}

    now, you can call the above module from your app.js (or any other controller) like this:

    var ui = require('/src/common');  //remember not to include the .js !!!!
var content= ui.createArticleView({
   title:"This is a nice title!",
   text:"This is the main content of this article. Modules is a nice way to get your code tidy"
});

content.open();

    The great advantage of this is that whatever you define within a module it is not exposed to the global context. Only whatever is assigned into the 'exports' struct will be available to the outside world. This will help you avoid polluting the global scope and reduce memory issues.
