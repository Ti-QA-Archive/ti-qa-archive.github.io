  • I'm not certain why the module isn't working. I abandoned Ti for this project and switched to Java (largely because of this issue). When I downloaded the Java version of Zxing there were errors in its code as well. With a little research I found that JDK 1.7 changed a requirement for switch/case statements that requires the argument be static. When I changed these to if/else in the Zxing code I got it to work. Since I can't see inside the Ti module, I can only assume this might be the issue. However, it did seem more than coincidental the switch statements were in classes dealing with capture. One way to check would be to verify if you're using JDK 1.7 and try regressing it to see if it changes things.
