    Quick update, this has been fixed and will be available in Titanium SDK 3.1.1.

    Here's the Jira ticket for reference: https://jira.appcelerator.org/browse/TIMOB-13692.

    You can test drive the fix by downloading the latest 3_1_X build. Note: this is an CI build and is not 100% tested yet. It is not meant for production use, but should be fine for development.

    ti sdk install --branch 3_1_X

  • Generally speaking, Titanium SDK 3.1.0.GA should work with both Node.js 0.8 and 0.10. If you're using Node.js 0.10, many, many things could cause a "Maximum call stack size exceeded" error. Downgrading to Node.js 0.8 will usually show you the correct error message.

    I recommend using Titanium SDK 3.1.0.GA, so update the <sdk-version> in your project's tiapp.xml to 3.1.0.GA.

    Next, open the Terminal and change to your project's directory. Clean and build your project:

    cd /path/to/my/project

ti clean

ti build -p ios -Y ipad --log-level trace

    I'm interested to see the complete output of your build. It will tell me what plugins are being loaded and give me clues to where the problem is coming from.
