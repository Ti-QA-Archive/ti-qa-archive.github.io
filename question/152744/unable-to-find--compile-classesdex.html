  • Solved it for Android.
    It is not enough to copy the files aapt and dx from build-tools to platform-tools. I had to copy everything from build-tools to platform-tools.

    This does not solve the iOS-simulator problem, but at least I can work now.

  • I resolved this issue by Installing "Android Platform-tools 18.0.1" & "Android Build-tools 18.0.1" after "Android SDK Tools 22.0.5"

    Now when I look the folder <Android_SDK>/build-tools so there are 2 folders as "android-4.2.2" & "18.0.1"

    I removed symbolic link which initially I created in "platform-tools" point to "android-4.2.2" (http://developer.appcelerator.com/question/152497/titanium-sdk-310-error-typeerror-argument-of-type-nonetype-is-not-iterable-on-building-android-app) and again created the same which points to the "18.0.1"

    That resolved my problem. :)
