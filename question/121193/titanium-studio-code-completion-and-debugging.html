Titanium Studio Code Completion and Debugging

I am trying out Titanium Studio with the intention of getting the indie subscription and using it to build desktop apps for mac and pc, but not all is peachy in never-never-land.

Problem #1

Code completion for "Titanium" or "Ti" does not seem work.
To reproduce:

  1. Install Titanium Studio
  2. Install the SDK update (Help -> Check for Titanium SDK Updates)
  3. Create a new desktop application (File -> New -> Titanium Desktop Project)

  4. Add some JavaScript code into index.html, eg:

    <script type="text/javascript">
 Ti(ctrl+space)
</script>

  5. Notice that the inserted code is this: 

    setTimeout(function() {}, 10);

    Notice no mention of "Titanium" anywhere in the list of namespaces.

Problem #2

How to enter debug mode, or show traces.
The only debug perspectives seem to be for web and for mobile. I have also tried using traces as a hackish work-around using something like Titanium.API.trac(&quot;...&quot;); but the traces don't appear anywhere. I have checked the "Console" and "Terminal" window, as well as the console window which opens when the application is launched.

It feels like I left out something somewhere but trolling the forums wasn't very helpful. Is any special voodoo needed to get this stuff working?

  • At this point Studio's debugging and code completion features are focused on the mobile SDK. When debugging your desktop app think of it more like working in a traditional browser. You can use Webkit's inspector to interact with the DOM and your JavaScript code.

    However, when using Titanium.API.(debug, trace, log, etc) you should see the messages appearing in Studio's console.
