Ti.API.firevent vs Ti.App.fireevent?

With iPhone I can setup an event handler and a fireevent under Ti.API, however Android will not fire it. If I change the handler and the fireevent to Ti.App, Android will fire it, however iPhone declares the input value of tab undefined.

    Ti.App.addEventListener('openTabWindow', function(obj) {
        obj.tab.open(obj.win,{animation:true});
        pp.ui.winStack.push(obj.win)

        Ti.API.info("Current Window Stack: "+JSON.stringify(pp.ui.winStack));
    });

.....

    obj = {tab:pp.ui.tab3,win:pp.ui.myView()};
    Ti.App.fireEvent('openTabWindow', obj);

Any help or explanation would be appreciated!

  • When you dispatch a Titanium.App. event, the data payload should be JSON parseable. Here you seem to be sending a UI object ( a proxy object) , so the data payload is not serialized. I don't know how is that you made that work with Titanium.API on Android (a platform parity bug?) . Also, for Application level events (cross context), you should use Ti.App. Hope this helps
