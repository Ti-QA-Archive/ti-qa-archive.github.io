KitchenSink web_views.js has back, fwd, reload toolbar in source - invisible?

This code found in KitchenSink web_views.js (pasted below) seems to anticipate the often-requested functionality of being able to go back, reload, forward once one is navigating for example by links on a remote web page.

However, this "back, forward, reload" toolbar does not seem to actually be displayed on any demo in the KitchenSink.

I tried adding a line just above this to set the var rowdata.controls = true;
But when loading the app onto (Samsung Galaxy Tab, where it otherwise runs great) - accessing any web_views.js link now gives an error:

TypeError: Cannot find function setToolbar in object [Ti.UI.Window]. (app://examples/web_views.js#191)


        if (rowdata.controls)
        {
            // test web controls
            var bb2 = Titanium.UI.createButtonBar({
                labels:['Back', 'Reload', 'Forward'],
                backgroundColor:'#003'
            });
            var flexSpace = Titanium.UI.createButton({
                systemButton:Titanium.UI.iPhone.SystemButton.FLEXIBLE_SPACE
            });
            w.setToolbar([flexSpace,bb2,flexSpace]);
            webview.addEventListener('load',function(e)
            {
                Ti.API.debug("url = "+webview.url);
                Ti.API.debug("event url = "+e.url);
            });
            bb2.addEventListener('click',function(ce)
            {
                if (ce.index == 0)
                {
                    webview.goBack();
                }
                else if (ce.index == 1)
                {
                    webview.reload();
                }
                else
                {
                    webview.goForward();
                }
            });
        }

  • I see that people say Android does not have a toolbar. Instead on Android we press the hardware back button ( some call it the curvy arrow button, I think of it like ESC key).

    The problem is that once my app opens a web page, and I click even one link deep into the remote web page - hitting the Android hardware back button will not navigate back one web page. It closes exiting Titanium App window and returns to the previous Titanium App window.

    This is understandable to me (finally) - it seems the only solution is to design the remote web page with standard javascript back button. (Assuming you have control of the remote web page code.) Or - download each page to the device and manipulate it prior to displaying it to the user.

  • Or you could try something like this to get the hardware back button working in an Android web view:

    myWin.addEventListener('android:back', function() {
    if (myWebview.canGoBack()) {
        myWebview.goBack();
    } else {
        myWin.close();
    }
}
