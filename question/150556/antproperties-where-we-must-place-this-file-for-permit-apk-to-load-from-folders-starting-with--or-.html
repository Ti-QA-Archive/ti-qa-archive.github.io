ant.properties: where we must place this file for permit APK to load from folders starting with "_" or "."

Hello everybody (I'm on Windows)!

I'm new at Titanium Studio (essentially for write some mobiles apps!).

I figure how I can load files (images/sounds/others files) from a folder starting with a "_" or a "."

Actually I have the path to load from folder (at the root of the project): _dataSystem_&#x2F;GFX&#x2F;gameBackgrounds&#x2F;
In that path I have a 'brick.png' file !

Then now, at each time I want compile the APK in the console Titanium Studio throw me that error:

[WARN]     (skipping dir '_dataSystem_' due to ANDROID_AAPT_IGNORE pattern '_*')

So, after some search on Internet I can see it's an Android default settings…
Now, solutions preconizes to create a "ant.properties" file in the project and add to that file this line:

aapt.ignore.assets="!.svn:!.git:<dir>_*:!CVS:!thumbs.db:!picasa.ini:!*.scc:*~"

SO now… even after create that file in my ROOT project folder (where tiapp.xml is localized) and adding the line aapt.ignore.assets… things don't work.

I have always at each APK rebuild that error.
And all files I want load from that folder don't load.

So, actually not really all fildes.. for make the things short :
I use a createWebView in ApplicationWindow.js that invoke the html page load a file localized in: _dataSystem_&#x2F;GFX&#x2F;game_index.html

Then in Android emulator I can show up that html page ! It works..
But in that game_index.html page I have created some piece of HTML code :

<img src="../_dataSystem_/GFX/gameBackgrounds/brick.png">

But this simply don't work ! It show nothing… (nothing as image of course)..

but that code below works as expected
(I have the same brick.png file in an images folder localized into project's root path..):

<img src="../images/brick.png">

Where i'm wrong please :( ?
