Date object returning wrong date?

I tried storing the current date to get a timestamp when my application started, like this:

Ti.App.Properties.setList('updateTime', [new Date()]);

But when I output this somehwere else, like this:

var lastUpdate = Ti.App.Properties.getList('updateTime');
Ti.API.info(lastUpdate[0]);

It seems that I'm getting a wrong time, I started my application at 20:04, but my output is saying it was 18:04
[INFO] 2011-08-08 18:04:41 +0000

What would cause this problem? Or is this a normal behavior? (like another timezone)

Not that this really maters, because if this happens every time it's no problem, but this seems kinda odd …

    The +0000 after your date is the timezone offset. Therefor you are actually getting the time according to GMT. If you want your time, subtract your timezone offset. This may only happen on the simulator and not on the phone, but you'd have to test that.
