Frustrated with module installation under Titanium 1.8 under Mac OsX

I have tried numerous times to install TI.paint from both marketplace and zips found on the net.

Firstly, I tried copying to /Library/Application Support
I did not find a Titanium in this folder, so I created it.
Then modules/iPhone/ti.paint/1.0
Into this folder I pasted the contents from the market folder.
There is a libti.paint.a in this folder.

I then tried to compile simple mobile app.

app.js:
var paint = require('ti.paint');

Into tiapp.xml I add…
<modules><module version="1.0">ti.paint</module></modules>

When attempting to run on iPod simulator returns:
Application Error
Couldn't find module: ti.paint

Ok, so I next, I find the module in downloads.
I drag and drop the entire module folder into the project root folder in titanium. This copies the files into the project, and I see the module folder at the same level as the resource folder.

I therefore see /testpaint/modules/iphone/ti.paint/1.0
And by the same token, I see /testpaint/resources
Run again on iPhone simulator, same error.

Maybe I misunderstood the instructions.
So I move the modules folder to under resources using titanium studio.
Try again, still no go.

What gives?

Details.
Apple MacBook 2010 4gb Mac OsX
Titanium 1.8 all updates current
Have compiled an run apps for both iOS and android.

  • From what I read of the documentation, we were supposed to install to /Library/Application Support/Titanium/Modules. I was suspicious as I did not see a Titanium folder. Nikunj suggested looking in the ~/Library/Application Support/ folder. Yes, Titanium was there. A far cry from /Library. I copied into the modules folder. A restart of Titanium was required. It works now, thank you Nikunj.

    So, short answer, install modules for mac osx into
    ~/Library/Application Support/Titanium/modules

  • I think I can answer my own question that the best way now adays is, for Mac, to turn off automatic unzipping in Preferences, I think that's the last checkbox in General. Then download the Zips of the Modules. Then use Help | Install Module, find the Zip, install to SDK. Then it becomes a pick and choose option when setting up a new program. Much simpler.
