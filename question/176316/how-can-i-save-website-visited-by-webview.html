  • Hi

    If you are trying to create an off-line cache of your locally visited website then you have to do some leg work yourself. The API does not have an option for this built in as it is neither a standard feature/need or something provided in the OS level web view either (from IOS or Android). I am not aware of any third-party modules that would do this either.

    So you will need to use httpClient to download the html and then parse the HTML to locate all images, scripts and css files. You will have to translate all local path references are very few paths have full URLs - usually just relative. Then loop those files and download them. Finally go through the CSS file to find any more image references and download those as well. It is unlikely you will be able to determine images from the JS files as depending on how they are created it would be very difficult to resolve the full name.

    You then have to store all of this in a file system structure that you hold locally in a way that you do not duplicate files and cater for the external libraries.

    This is not considered a simple task and is only normally feasible in a live browser as it can tell all the files that were asked of it.

    Good luck, I am sorry to say you will need it.
