Can't build Android module

When trying to build the Android example module, the build fails with the following error:

C:\Users\me\AppData\Roaming\Titanium\mobilesdk\win32\2.0.1.GA2\module\android
\build.xml:236: C:\sandbox\workspaces\Titanium\Android\ExampleModule\build\
generated\libs does not exist.

If I manually create the libs folder, packaging works from within Titanium Studio, but not with Ant. Furthermore, when trying to use the module in another project, I am getting "Uncaught Error: Requested module not found".

I have exactly followed these guides:

Creating_a_New_Titanium_Module

Android_Module_Development_Guide

What am I doing wrong?

I am working on SDK 2.0.1.GA2 on Windows and Android 2.3.3.

  • You need to use the Android ndk folder directly in a drive. not inside any subfolder like C:\Program files\Android ndk

    You need to have gpref installed in your system.

  • I was finally able to find the reason for this issue. It turned out that the PATH environment variable didn't contain a the correct path to my Python installation. Having fixed the path, the libs folder was generated and the module was found as well.

    • I also thought it was an error in the Python build script and altered it to create the libs folder. However, this did not work as well because the necessary libraries were not included.

      Open a command prompt and type "python –version" to see if python is correctly installed. If the command is not found, there is an error in the environment variable.

      As said above, this was the reason for my issues. Having fixed the path in the environment variables, it worked perfectly fine.
