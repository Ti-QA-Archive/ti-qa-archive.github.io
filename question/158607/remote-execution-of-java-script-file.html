Remote execution of Java Script file

Hello All,

I want to build one app where all the UI is dynamic. And these UI will change based on the type of user.Based on the web service response we are drawing the UI control. But if we follow this approach then web service will be heavy as it needs to send all the UI control name, its validation rule etc for all the UI control(Currently we have 20 UI and 20 user so all together 400 UI) . So is it possible to have any other approach of the same.

  • The short answer is: No you cannot

    A longer answer might be to have some sort of mechanism where you still build your UI based on files that you access remotely (JSON files where you describe your objects as well as their properties). Then your code will have to interpret those description files and build the interface. Although I sincerely think that will have issues on your app's overall performance.
