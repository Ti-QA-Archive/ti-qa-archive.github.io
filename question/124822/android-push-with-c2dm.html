Android Push with C2DM

I have had lots of help on this forum and wanted to give a little back.

If you want to know how to do push with C2DM on Android without having to use Urban Airship then…

1) Go get this awesome titanium module from IGx89:
https://github.com/IGx89/titanium-c2dm

2) If using a Titanium module seems daunting to you, post your question on this ticket and I'll respond with whatever I can do to help

3) Here's my complete tiapp.xml file
https://gist.github.com/1173879

Note you will need to change com.company.acme to be your real package name and that down in the android manifest section you'll find com.company.acme used in several permissions, change those to be the same as well.

4) You'll need to signup with Google to be allowed to use their C2DM service, when you do you'll do so with an email address at gmail and with a package name , e.g. com.company.acme, the package name of your app (the value of the <id> element of the tiapp.xml element) must match the package name you gave google.

5) When you download IGx89's module, look at the provided example and change the senderId to be the email address you signed up for C2DM with Google

I had questions on how to get this to work and IGx89 helped me out, so if you have questions put them here and I'll do what I can.

Props to IGx89 for the great work!

  • I was able to get this working when the app is closed as well as when you first boot your device. There is also an 'Unregister' function now. I didn't come up with the full code myself, i got it from other forks on github, i just combined them and then added some minor tweaks. So props goto digidens and cashlo.

    you'll need to open C2DMReceiver.java located in src\com\findlaw\titanium\c2dm and update the parts with your own package name and activity name (places in between the brackets). You'll then need to recompile using ANT.

    To get this to work on boot you need to add the following to the tiapp.xml right inside the 'receiver' node. (You can see this in the timodule.xml file on git).

    <intent-filter>
        <action android:name="android.intent.action.BOOT_COMPLETED"/>
        <category android:name="android.intent.category.HOME"/>
</intent-filter>

    To unregister from the c2dm server use:

    c2dm.unregister();

    Here's the link.

    https://github.com/johnnnysc22/titanium-c2dm

    • Hi Sean,
      Have a look at line 52 at the pull code: if (C2dmModule.getInstance() == null)
      What happens is that when the app is closed (either from task manager or by restarting the device) the Instance returns a null object. What they actually did in the code is a singleton. I debugged it before I found the pull and got the same result (only what I tried to do first was warp the class with another static class which didn't really work for me).

      But back to our little problem… what they did was check if the object is null - if so recreate a instance object and notification from native Android SDK. If it's not null then 'else' part continues as regular.

      This actually means that you have to maintain to codes for the push notification - one in Titanium and one in Java. What I'm planning on doing (haven't tried it yet) is to make the Java code create the notification in each case so I wont have to maintain it twice.

      Hope that helps…

  • And in the spirt of giving back. Here is the php code I used for my 3rd party server. I didn't write this, I found it online but I did verify it to work.

    <?php
/*
*
*    $username - your gmail email address you white listed with the c2dm service
*    $passwors - password for above gmail account.
*/
function googleAuthenticate($username, $password, $source="Company-AppName-Version", $service="ac2dm") {     

    session_start(); 
    if( isset($_SESSION['google_auth_id']) && $_SESSION['google_auth_id'] != null) 
        return $_SESSION['google_auth_id']; 

    // get an authorization token 
    $ch = curl_init(); 
    if(!$ch){ 
        return false; 
    } 

    curl_setopt($ch, CURLOPT_URL, "https://www.google.com/accounts/ClientLogin"); 
    $post_fields = "accountType=" . urlencode('HOSTED_OR_GOOGLE') 
        . "&Email=" . urlencode($username) 
        . "&Passwd=" . urlencode($password) 
        . "&source=" . urlencode($source) 
        . "&service=" . urlencode($service); 
    curl_setopt($ch, CURLOPT_HEADER, true); 
    curl_setopt($ch, CURLOPT_POST, true); 
    curl_setopt($ch, CURLOPT_POSTFIELDS, $post_fields); 
    curl_setopt($ch, CURLOPT_RETURNTRANSFER, true); 
    curl_setopt($ch, CURLOPT_FRESH_CONNECT, true);     
    curl_setopt($ch, CURLOPT_HTTPAUTH, CURLAUTH_ANY); 
    curl_setopt($ch, CURLOPT_SSL_VERIFYPEER, false); 

    // for debugging the request 
    //curl_setopt($ch, CURLINFO_HEADER_OUT, true); // for debugging the request 

    $response = curl_exec($ch); 
     echo $response;
    //var_dump(curl_getinfo($ch)); //for debugging the request 
    //var_dump($response); 

    curl_close($ch); 

    if (strpos($response, '200 OK') === false) { 

        echo "200 Not Ok";
        return false; 
    } 

    // find the auth code 
    preg_match("/(Auth=)([\w|-]+)/", $response, $matches); 

    if (!$matches[2]) { 
        echo "No Match";
        return false; 
    } 

    $_SESSION['google_auth_id'] = $matches[2]; 
     echo $matches[2]; 

     return true;
} 


/*
*
*    $authCode - the authcode returned when you ran the googleAuthenticate function
*    $deviceRegistrationId - registrationID returned by the device receiving the message
*    $$msgType - can be anything i.e. 'chat message'
*    $messageText - text to be sent
*/


function sendMessageToPhone($authCode, $deviceRegistrationId, $msgType, $messageText) { 

        $headers = array('Authorization: GoogleLogin auth=' . $authCode); 
        $data = array( 
            'registration_id' => $deviceRegistrationId, 
            'collapse_key' => $msgType, 
            'data.message' => $messageText //TODO Add more params with just simple data instead            
        ); 

        $ch = curl_init(); 

        curl_setopt($ch, CURLOPT_URL, "https://android.apis.google.com/c2dm/send"); 
        if ($headers) 
            curl_setopt($ch, CURLOPT_HTTPHEADER, $headers); 
        curl_setopt($ch, CURLOPT_SSL_VERIFYPEER, false); 
        curl_setopt($ch, CURLOPT_POST, true); 
        curl_setopt($ch, CURLOPT_RETURNTRANSFER, true); 
        curl_setopt($ch, CURLOPT_POSTFIELDS, $data); 


        $response = curl_exec($ch); 

        curl_close($ch); 

        echo $response; 
} 
?>
