Android-App: Reminder functionality - best way to alert multiple future dates?

Helloooooo dear community :)

I'm developing an iPhone & Android app, that contains a list with various future dates/events, like:

  • 01.12.2011 19:00h - Event 1
  • 01.12.2011 19:45h - Event 2
  • 02.12.2011 15:45h - Event 3
  • …

The user can "flag" multiple list-entries to create his own timetable. The flagged dates are stored in a database. The app notifies the user with a local notification when the date/event starts.

So, on iOS it was very easy to develop this functionality.
I simply fetch all stored dates from the database and schedule the dates in a loop:

storedNotifications[i] = Ti.App.iOS.scheduleLocalNotification({
    alertBody: 'Reminder: Lorem ipsum',
    alertAction: "Open App",
    date: futureDate
});

Now i want to create this functionality for the android version of my application. I read the API-documentation and the wiki, too. As I understand it, there no easy way to create notifications with a future date. The wiki contains an article about future notifications.

But how can I store multiple future notifications? Should I create a service-intent for each date? Or should I create one service-intent, that runs in one-minute interval and fetches all upcoming dates within the next 60sec? An interval-solution sounds inefficient and battery-consuming to me.

Any ideas?
