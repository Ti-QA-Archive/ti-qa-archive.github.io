Nice VideoPlayer IOS bug with 2.1.2

Hello,

Found a "nice" bug in 2.1.2 SDK (with iPhone SDK 5.0), try this simple code:

// this sets the background color of the master UIView (when there are no windows/tab groups on it)
Titanium.UI.setBackgroundColor('#000');

var mainWin = Titanium.UI.createWindow({
    orientationModes:[Ti.UI.LANDSCAPE_LEFT,Ti.UI.LANDSCAPE_RIGHT],
    exitOnClose:true,
    navBarHidden:true,
    fullscreen:true,
    title:'Slide Box',
    backgroundColor:'#ffffff',
    zIndex:0
});

var    VideoPlayer=Ti.Media.createVideoPlayer({
    zIndex:3,
    top:350,
    left:300,
    url:"",
    backgroundColor:"#000000",
    mediaControlStyle:Titanium.Media.VIDEO_CONTROL_EMBEDDED,
    scalingMode:Titanium.Media.VIDEO_SCALING_ASPECT_FIT,
    width:308,
    height:208,
    visible:true,
    url:"http://movies.apple.com/media/us/ipad/2010/tours/apple-ipad-video-us-20100127_r848-9cie.mov"
});
mainWin.add(VideoPlayer);

mainWin.open();

Now wait for video to load, play it, put it on full screen (with pinch or button control). Then, minimise the video…
As you'll see the "mainWin" has been resized ?!

Can someone pull this bug request on the JIRA ? (or maybe it is already done ?)
