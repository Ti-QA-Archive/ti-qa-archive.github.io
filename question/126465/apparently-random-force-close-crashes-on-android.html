Apparently Random Force Close Crashes on Android

Hi there, we are facing severe issues with apparently random force close errors in Android. There's one App which is quite large, so we thought it could be a memory issue, but then on a different App which is really lightweight, 4 screens and one barcode scanner, the same appears as well.

We cannot reproduce the bugs accurately, they happen sometimes and sometimes they don't.

We've gone through every single currently open bug in Jira, but none sounded like it's applicable / related. We also went through every single post in the Forum as well, but without clear answer.

The closest we came to was this post http://developer.appcelerator.com/question/120771/ddms-error-called-layout-cleanup

We're basically sometimes getting the LayoutCleanup error in the log, but it doesn't say where or which line of code or what is actually the problem.

Chances are we have done something wrong or didn't interpret the log output correctly, so my question is:

How can we effectively debug Force Close crashes on Android, and what 'tool' gives us information where to home in on the issue? Sorry if this is a bit unstructured, if there was some sort of chatroom or so we could give more info.

— asked October 5th 2011 by Anthony Hsio

  • Typically viewing the logcat logs in ddms around a 'force close' will give you a pretty descriptive stack trace describing your errors. You can usually start to track down your issues based on this Java stack trace. Often times the content will point to a class that maps to a Titanium API, making it more clear what is going on.
