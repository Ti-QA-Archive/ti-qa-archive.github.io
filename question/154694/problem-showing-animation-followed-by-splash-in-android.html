Problem showing animation followed by splash in Android

I am developing for iOS and Android.
My aim is to play an animation when the app opens and then (after the animation) show a splash screen/image.

I followed this answer, it works fine on iPhone and iPad but not on the Android.
On Android, I will get white screen for a while (till the animation is supposed to last).

Can anyone help me out on this? I was considering using ImageView.animate(), will it be the only solution?

    Replace var t = setInterval(function(e) { and following lines with this:

    var image_number = 1;
img.addEventListener('change', function(e) {
    if(image_number >= door_array.length)
        loadMainScreen();
    image_number++;
});

    • I am doing the same project in that question, so the basic idea is the same.

      var door_array;    //array with image paths
var counter = 0;
var img = Ti.UI.createImageView({
    image : door_array[counter]
});
win_door.add(img);
var t = setInterval(function(e) {
    if(counter < door_array.length) {
        img.image = door_array[counter];
    } else {
        clearInterval(t);
        loadMainScreen();
    }
    counter++;
}, 50);

      In loadMainScreen() method I just load the splash and the screen. Screen opens up fine but no splash or door animation.
      Interestingly, the animation plays fine if I use the ImageView.animate() but it is tricky (there is no tidy way to run B after A finishes).
