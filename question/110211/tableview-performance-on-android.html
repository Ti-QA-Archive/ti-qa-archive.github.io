  • I tried your code out, and had to do the following to get it to show anything (on iPhone first): removed layout='vertical' at the top (got blank screen with this left in); removed 'row.title=' as it overwrote rest of text on row; removed 'bottom: 60' to prevent leaving a big gap on bottom of page (you may want that for something…?); removed the tmpView (per your comments, and it is not needed); removed the backgroundImage and SelectedImage (didn't have those); removed the initial var rd[] and tView.data=rd (not needed); and removed the timer code (no timer.js).

    After that, the code worked fine on both iPhone and Android sims, using Android APIs 2.2, Ti Developer 1.2.2, and Ti Mobile 1.5.1. Android Sim takes less than 2 seconds to display a 25-row table; 100 rows takes about 4.3 seconds.

    Keep in mind that unlike the iPhone Sim, the Android Sim tends to run slower than the code on the device. I timed the same 100 row creation on a Droid 2 and it took 2.1 seconds.
