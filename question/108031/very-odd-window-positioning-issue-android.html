Very odd window positioning issue (Android)

I'll try to keep things simple:

  • My app has a tab group. In some cases it opens a new window (call it Info window). From there a button opens "more info" which is designed to look like a popup (it has some padding and a border).

  • Testing on API 1.6 everything works fine for me.

  • Testing on API 2.2.1 the more info "popup" is located relative to the info window views not the overall screen (and thus is basically positioned out of view).

  • If I alter the code to remove the tab group and just have it open the info window directly things work again.

So as far as I can tell there seems to be some issue with the combination of the tab group + opening the window + 2.2.1. I'm pretty much out of ideas.

Here is the code to open the "more info" window:

borderSpace = 40;
var w = Titanium.UI.createWindow({
    height:styles.ui.platformHeight-borderSpace,
    width:styles.ui.platformWidth-borderSpace,
    backgroundColor:'white',
    borderWidth:3,
    borderColor:'#336699',
    left:borderSpace/2,
    right:borderSpace/2,
    top:borderSpace/2,
    bottom:borderSpace/2,
    layout:'vertical'
});
w.open({animate:true});

UPDATE:
Originally I was opening the Info window using Titanium.UI.currentTab.open(newwin,{animated:true}); this was opening the window over the entire screen (e.g. covering the tabs) - which I liked. However my more info window placement issue goes away if I change that code to newwin.open({animated:true});. However, now both the info and more info windows are located within the tabs (not preferable but if its the right way I'll deal).

All in all I think I need some guidance on how to properly "stack" windows.

    • There is indeed a proper way to submit a bug, and the first rule of it is not to assume you have found a bug simply because you cannot get something to work.

      You need to create a usecase, and post it here for others to test and confirm on their systems. The code above is not a usecase, because it does not allows someone to paste it into a default app and for it to run without modification. If you want to read more about use-cases, see Creating Good Use-cases.

      Most of the time you should post proper usecases to the Q&A also.
