Updating iOS app. Bundle is invalid.

Application type: mobile
Titanium SDK: 3.0.2 (have built the app with multiple sdk's)
Platform & version: iOS SDK 6.1
Host Operating System: OSX 10.7.5
Titanium Studio: 3.0.24

I am trying to make an update to an iOS app. I was able to do this just a few weeks ago but now am no longer able to. Since then I've received updates for the TI SDK (I've tried multiple TI SDKs though) and an Xcode update. The Xcode update put me on iOS SDK 6.1. So now when I try and build the app for distribution I get this error on validation:

This bundle is invalid. The application-identifier entitlement is not formatted correctly; it should contain your 10-character App ID Seed, followed by a dot, followed by your bundle identifier: xxxxxxxxxx.com.xxxxxx.AppName*

I've read a few threads about this. I know there's a ticket in JIRA related to it. I've seen that other people have said to create a provisioning profile without the wildcard. I did just that with ANOTHER app and it resolved the issue. With this one, however, the error persists. What am I missing?
