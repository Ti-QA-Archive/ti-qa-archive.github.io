Custom AndroidManifest and tiapp.xml are overridden by the build,

I already spent hours and hours on this and I just give up. I followed the recommendations of how to add the custom manifest besides of the existing manifest and for some reasons it leave the custom manifest on the build folder but then it rollback to the original values. So I tried with the second option of adding the version number on the tiapp xml with same results. 

<android xmlns:android="http://schemas.android.com/apk/res/android">
        <manifest xmlns:android="http://schemas.android.com/apk/res/android" package="com.interacttivo.tunerslab" android:versionCode="2" android:versionName="1.1" android:installLocation="auto">
            <application android:hardwareAccelerated="true">
                <activity android:alwaysRetainTaskState="true"

                    <intent-filter>

  • This is how you do it -

    • create a directory on the same level directory where the resources folder resides as follows
      Platform>android
    • copy your android manifest file from your build folder and put it back into the android folder you just created above
    • now the manifest file will never gets overwritten when you build again,
      Hope this helps!

  • Lets focus on tiapp.xml.

    • back up your <android> section.
    • double check to be sure you don't have a custom android manifest hanging around anyplace.
    • clean your project (go so far as to physically delete the build folder)
    • lets start with a very simple change

        <manifest android:versionCode="2" android:versionName="1.1"/>
