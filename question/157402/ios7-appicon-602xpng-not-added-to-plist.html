IOS7 appicon-60@2x.png not added to plist

I've created the appropriately sized and named (I think) appicons for my app to be both ios6 and 7 compatible, placed in the Resources/iphone folder. I'm building with Titanium 3.0.0GA, and the icons ARE getting copied into the built app package. BUT they are not all in the plist file - missing the appicon-60@2x.png one, even though the file was copied and is present in the same file location in the built package.

<array>
        <string>appicon.png</string>
        <string>appicon@2x.png</string>
        <string>appicon-72.png</string>
</array>

iTunes throws a warning when I try to upload: "Missing recommended icon file. The bundle does not contain an app icon for iphone/ipod touch of exactly '120 x 120' pixles, in .png format."

Yes, I can add the file to the plist manually, but this seems like either I'm missing something or it's a bug. (Same thing happens under 3.1.1, btw, and we're using 3.0.0 for compatibility reasons with a couple modules)
I'm on OSX 10.8.5 with xCode 4.6.3 and Titanium Studio 3.1.3.2. Since my system is stable, I'd love to find a solution that does not involve upgrading :-)

Anyone else experiencing this?

Thanks,
David

  • We had a project, built out under 2.x, submitted to the store just fine. We had to make a minor change for the client, and suddenly, their icons are being rejected. The very same icons that had been accepted in the ipa previously. The solution was loading up the icons, shifting the all white background to a slight off white, rerendering them out and bingo, darn if it didn't work.

    The other thing that apple likes to screw with is the application loader. They have updated it from time to time, requiring that you download and update it manually to accommodate changes on their end.

    Now, regarding the ios7 icons, you will need to be using the ios7 compatible sdk and tooling for it to automatically copy and populate your info.plist. You could try to manually add them to your tiapp.xml in the section for the plist and see if that works for you.

  • Yes! I think newest versions of Ti.SDK (latest 3.x's) automatically add them in the build of your app.

    However if you are like me and still work with older versions of the Ti.SDK you need to change some stuff in the ~&#x2F;Library&#x2F;Application Support&#x2F;Titanium&#x2F;mobilesdk&#x2F;osx&#x2F;SDK_VERSION&#x2F;tiapp.py :

    Find the line which says: 

    iconorder = list(iconname+".png" ... etc

    and complete this with the following new icons (60.png, 60@2x.png, 76.png, 76@2x.png): 

    iconorder = list([iconname+".png",iconname+"@2x.png",iconname+"-60.png",iconname+"-60@2x.png",iconname+"-72.png",iconname+"-72@2x.png",iconname+"-76.png",iconname+"-76@2x.png",iconname+"-Small-50.png",iconname+"-Small-50@2x.png",iconname+"-Small.png",iconname+"-Small@2x.png"])

    This part in the tiapp.py generates the bundleiconfiles property in the info.plist file. IT DOESN'T WORK if you add them manually to a custom info.plist file as this tiapp.py will still overwrite them. So do it (carefully) in the tiapp.py file.

    Now be sure to add the new icon files (with maybe a flat design, that's up to you) to your project Resources/-root or project's Resources/iphone folder.

    Good luck!
