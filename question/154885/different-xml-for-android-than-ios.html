Different xml for android than ios

Hello,

Is it possible, when using Alloy, to have different markups for different platforms? For example, I'd like to use ScrollableView in Android and TableView in iOS. How can i accomplish this?

    Hi

    There a re several ways, the two most common are;

    • Inside the mark-up

    You can add one or both of the following attributes to the definitions; platform & formFactor.

    Example;

    <Alloy>
    <View id="view" platform='ios' formFactor='handheld' backgroundColor='red' />
    <View id="view" platform='ios' formFactor='tablet' backgroundColor='green' />
    <View id="view" platform='android' formFactor='handheld' backgroundColor='blue' />
    <View id="view" platform='android' formFactor='tablet' backgroundColor='yellow' />
</Alloy>

    Only one will show up depending on the target and device tpe.

    • In CommonJS

    Create a module in each platform folder with the same methods and exports, but internally the code returns platform specific code. When you compile the app ONLY the specific platform code is included and run.

    Notes

    In Alloy, ONLY the platform specific code is compiled, any code that is platform conditional will not be included in the build.

    • Got you - no this will not affect the performance of the other platform, so if you have lots of android mark-up but compile for iOS only the iOS code is actually compiled into the app, so no performance hit. There will be a slight speed hit during compile as it is working out which to include and which to ignore, but it is unlikely you notice that in real terms.
