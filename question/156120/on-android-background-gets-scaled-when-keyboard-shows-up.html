On Android background gets scaled when keyboard shows up

I have a screen with the following hierarchy:

Window
    View (having bg of my screen)
        <some image views and stuff>
        ScrollView

Now, The problem is that, on Android, when I focus on a TextField, the background will scale (spoils the looks of the screen).
I have tried:

  1. this solution but (as stated by the questioner) it moves all of the screen above when the TextField is about to hide below the keyboard.
  2. putting the bg in the windows but it stretches the image (vertically), although my image is of 320x480 for a 320x480 device.

How can I fix this?

Titanium Studio SDK: 3.1.2
Android SDK: 2.3.3 (API 10)

  • Solution #1 is the typical UX on Android, so it would be my recommended solution. Since most folks ask how to get the window to scroll, perhaps the default window behavior has been set to SOFT_INPUT_ADJUST_PAN in the recent SDKs. In which case, you could set windowSoftInputMode = SOFT_INPUT_ADJUST_UNSPECIFIED.

    Do you have an explicit height set on the view (that has the background image)? If not, set one. Its height is probably adjusting to account for the keyboard's height.

    Really, the window is the appropriate place to set your background graphic. Are you accounting for the height of a title bar or tab bar in your graphic's height? The window's height might not be 320x480 if those are present. Also, keep in mind that unless you're writing this app for yourself and your specific device, the end user's device is not likely to be 320x480 and the graphic you're supplying is going to be stretched.
