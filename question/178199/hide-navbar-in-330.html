    • Hi, to clear up the answer, you can hide the nav bar/ action bar in your Java script code or using a custom theme.

      - Hide NavBar In JavaScript Code:

      win.activity.actionBar.hide();

      - Add Custom Theme:

      1. Add a custom theme file to the project:

      Directory: platform/android/res/values/custom_theme.xml

      <?xml version="1.0" encoding="utf-8"?>
<resources>
    <style name="Theme.NoActionBar" parent="@style/Theme.Titanium">
        <!-- Depending on the parent theme, this may be called android:windowActionBar instead of windowActionBar -->
        <item name="windowActionBar">false</item>
    </style>
</resources>
      1. Modify the tiapp.xml file to use custom theme:
      <android xmlns:android="http://schemas.android.com/apk/res/android">
    <manifest>
        <application android:theme="@style/Theme.NoActionBar"/>
        <!-- Need to specify at least API level 11 for Titanium SDK 3.2.x and prior -->
        <uses-sdk android:minSdkVersion="14" android:targetSdkVersion="19"/>
    </manifest>
</android>

      N.B: navBarHidden property is deprecated from the release of Ti SDK 3.3.0. So to hide nav bar in 3.3.0 and 3.4.0, there is no use of navBarHidden property.

    • Your error was because, "win" is a previously opened window, you can only get the activity after the window appears on screen.

      Use it like,

      win.addEventListener('open', function() {
        win.activity.actionBar.hide();
});
