NavigationGroup is SDK 3.2.0 GA

Hi,

Why is that NavigationGroup is no longer supported on SDK 3.2.0 GA? I really like how NavigationGroup works because it can be added on views or windows. My app uses a lot of NavigationGroup and can't afford to rebuild it from scratch. Any idea how we can put back the NavigationGroup?

Thanks - Mark

  • Hi

    NavigationGroup has been deprecated and NavigationWIndow has replaced it. This now uses the iOS7 swipe back navigation when on iOS7.

    Keep in mind the structure of the window is now very different to how it worked before - there is no parent window required any more.

    • I am not sure if you realise that NavigationGroup == NavigationWindow, to the end user they are exactly the same in look and feel. The underlying code in Titanium changed to support the additional iOS7 features. To you the only real difference is the lack of parent window requirement to house the NavigationWindow.

      Having both a NavigationGroup and NavigationWindow would be an unnecessary duplication in the framework.

      I know there are a few code changes you will need to do to remove the current NavigationGroups parent window (I had to do the same), but once it is done - it is well - done (sorry tailed off there).

      One of the things we have to get used to (me as well) is our world is a constantly moving target, what we develop today may need feature tweaks in 6 months. The whole Android 2.x to 4.x jump and before iOS7 and after it are good examples of that.

      I say these things not to teach you how to suck eggs, but more of a reminder that we are the experts because we can evolve and evolve quickly so our own apps and our clients apps are always fresh.
