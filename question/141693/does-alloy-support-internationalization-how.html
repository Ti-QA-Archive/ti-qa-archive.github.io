    • Ah, I see. Right now, no, you can't set localization for strings in markup, but you can in controllers and styles. In the controllers you would do it just as you normally would. In styles, you can use the shorthand L() method anywhere you want to use a localized string. Something like this:

      "#someLabel": {
    text: L('localizedString')
}

      — commented August 30th 2012 by Tony Lukasavage

    • Two additional notes:

      The correct link for the doc Tony mentioned is: http://docs.appcelerator.com/titanium/latest/#!/guide/Internationalization.

      Also, many UI elements have special properties for specifying a localized string, such as Button.titleid. Where these properties are available, I believe you can specify them directly in the markup. For example:

      <Alloy>
    <Window class="container">
        <Button id="label" onClick="doClick" titleid="button_title"></Button>
    </Window>
</Alloy>
