Access ListView component from another view

Hi All,

I am working on ios app using Alloy framework 1.3.0, Titanium sdk 3.2.0 and mac OS. And using ios simulator for testing.

I have 2 views. View1 has button and view2 has listView with 2 rows having 2 labels each. On the click of button I want to access the labels from listview.
ListView looks like follow:

<ListView id="listView" defaultItemTemplate="template1" onItemclick="questionClick"> 
                    <Templates>
                        <ItemTemplate name="template1" height="50">
                            <Label bindId="label1" id="label1"/>
                            <Label bindId="label" id="label2"/>
                        </ItemTemplate>                    
                    </Templates>

                    <ListSection>
                        <HeaderView>
                            <Label backgroundColor="#a88">Questions</Label>
                        </HeaderView>
                        <ListItem label1:text="question" label:text="answer"></ListItem>
                        <ListItem label1:text="question" label:text="answer"></ListItem> 
                    </ListSection>
            </ListView>

On the click of button, I want to change the value of 'label1:text'. Now I have got the handle of ListView with

e.source.parent.parent.children[1].children[0].children[0]

above code has the handle of above ListView object. But I am unable to access the labels.

Can anyone please help in this? Thanks in advance.

  • One of the key differences between Tables and Lists is that Lists are data oriented, not View oriented. Thus, you can't access the child elements like you want. They don't necessarily exist in a container that you can probe. While you could swap out the ListView for a TableView to make your approach work, I'd suggest instead you keep a reference to the data and use it instead.

    By that I mean, in your controller, when you populate the listview, also save the label strings into an array. Then, on the button click, access the array member corresponding to the target list item rather than trying to access the item's child.
