  • I don't have such a folder in Windows XP: "/configuration/.settings/org.eclipse.core.net.prefs".

    But I got it running by editing the following file:
    "%Titanium_Studio_Path%\jre\lib\net.properties".

    You simply have to edit the following lines:

    http.proxyHost=yourProxy.com
http.proxyPort=8080
http.nonProxyHosts=localhost|127.0.0.1

https.proxyHost=yourProxy.com
https.proxyPort=8080

    Now it's running. :)

    Good luck!

  • OK, so this is what I did for setting a proxy that required authentication with the latest release of Titanium Studio.

    Titanium Studio, build: 1.0.2.201107130739
    (c) Copyright 2011 by Appcelerator, Inc. All rights reserved.

    Build date: 13 July 2011, 07:42:08

    Modify the following file
    <INSTALL_DIR>\Appcelerator\Titanium Studio\jre\lib\net.properties

    Add the following lines:
    http.proxyHost=<PROXY>
    http.proxyPort=<PROXY_PORT>
    http.proxyUser=<YOUR USER>
    http.proxyPassword=<YOUR_PASS>

    https.proxyHost=<PROXY>
    https.proxyPort=<PROXY_PORT>
    https.proxyUser=<YOUR USER>
    https.proxyPassword=<YOUR_PASS>

    • OK, I have discovered that the proxy settings are stored in the file:

      <WhereYouInstalled>/configuration/.settings/org.eclipse.core.net.prefs

      You will probably be able to edit this manually to configure your proxy settings and workaround the issue.

      This is the contents of mine:

      org.eclipse.core.net.hasMigrated=true
proxyData/HTTP/host=www.something.com
proxyData/HTTP/port=888
proxyData/HTTP/hasAuth=false
nonProxiedHosts=localhost|127.0.0.1
eclipse.preferences.version=1
proxyData/HTTPS/host=www.something.com
proxyData/HTTPS/port=888
proxyData/HTTPS/hasAuth=false

      This uses the (bogus) host www.something.com:888 (port 888) for the HTTP and HTTPS proxies.

      Please let me know if this helps.

      Cheers
