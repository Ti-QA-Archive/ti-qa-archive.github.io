  • Hi Shekhar Saraf,

    Please check the code below,

    presentWin.addEventListener('android:back', function(e){
    presentWin.close();
});

    Or you can use,

    presentWin.addEventListener('android:back', function(e){
    presentWin.close();
    var previousWin=Titanium.UI.createWindow({
        url:'/previousWin.js'
    });
    previousWin.open();
});

    It'll navigate to the previous window by using Android Back button.

  • Hi

    Given the property you mentioned it looks like you are talking about iOS - you have not said which is not very helpful.

    If this is correct then you have to have a window as a root of a navigation controller and opened up at least one window opened up as a child.

    Navigation controllers are built into tabs of a tabgroup and the navigationWindow, both are well documented, but if you have a specific question let me know.

    Without a navigation controller you do not have a navigation bar containing a left or back button a title or an optional right button.

    The navigational controllers are iOS specific, so will not work on android. It has its own UI equivalents.

    • Titanium IS cross platform

      You are mistaking cross platform for everything is the same.

      Titanium allows you to create apps using the same base language and APIs. However where the different OS are visually, structurally or progammatically different it gives those specific things to you as well.

      Otherwise iOS would not have buttonbars, toolbars or navigationWindows because android does not have them - or Android would not have action bars and menu items because iOS does not have them. If Titanium attempted to make each OS specific thing available on each other platform then it would look bad, act poorly and confuse the user of the device as other apps honour the target platform OS.
