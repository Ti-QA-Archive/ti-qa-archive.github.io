  • Hi

    Seeing a map container with controls but no map tiles is almost always the api key not being set correctly. This is most likely either the generated sha1 is not correct or the app name (com.company.appname) appended to the Google console does not match that of the app tiapp.xml.

    The key piece of your own debug info is;

    MapModule.isGooglePlayServicesAvailable() returns 0

    This means that the device/emulator you are testing on does not have the correctly installed version of Google Services - a good indication of support is having the Google Market Place app on the device - although it would need to be a more recent version as earlier ones are no longer supported. Most devices have upgraded the Market Place app automatically - so a physical device should have no problems if updated have been done in the last 6/12 months.

    This means there are no Google Services on the device you are trying to test on. I assume you are aware that in order to test the maps v2 you can only do this on a physical device or a correctly configured GenyMotion emulator - the standard Android emulator simply will not work when testing maps.
