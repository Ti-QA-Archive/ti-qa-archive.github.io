move strange in scrollableView in android

Application type: mobile

Titanium SDK: 2.0.2

Platform & version:Android 2.3

Device: Android emulator

In my case, I add a view into a ScrollableView and want to move left and right. but when i moved the view to left, the view moved a little right and fire the event touchcancel, if i moved the view to right the view moved a little right and fire the event touch cancel. If I move the view a little up or down then i could move the view correctly. below is my test code.

By the way, I test the code in iOS , it works fine.

var win1 = Titanium.UI.createWindow({  
    backgroundColor:'#fff'
});

var W = Ti.Platform.displayCaps.platformWidth;

var sview = Ti.UI.createScrollableView({
    height: '100%',
    width: '100%',
    showPagingControl: true
});
var view = Ti.UI.createView({
    height: '100%',
    width: '100%'
});

var view1 = Titanium.UI.createView({
    height: '20%',
    width: '20%',
    top: W * 0.2,
    backgroundColor: 'red'
});
view1.addEventListener('touchmove', function(e){
    var point = e.source.convertPointToView({x: e.x, y: e.y}, win1);
    view1.left = point.x;
    label.text = point.x;
});
view1.addEventListener('touchcancel', function(e){
    label.text = 'cancel';
});

var label = Ti.UI.createLabel({
    height: '20%',
    width: '20%',
    bottom: '20%',
    left: '20%',
    color: 'black',
    font: {fontSize: W * 0.04}
});
view.add(label);

view.add(view1);

sview.views = [view];

win1.add(sview);

win1.open();

  • I'm not sure what you're trying to accomplish, but I think you might want the ScrollView, not the ScrollableView.

    ScrollView: A view that contains a horizontally and/or vertically-scrollable region of content.

    ScrollableView: A view that encapsulates a horizontally-scrolling set of sub-views, navigable using its built-in horizontal swipe gestures.

    If you only have one view in a ScrollableView, you're not going to be able to do anything interesting with it.

    You also don't really have to manage the scrolling at the level you're trying to manage it. Just let the UI manage the scrolling for you.
