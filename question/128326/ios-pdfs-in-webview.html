iOS: PDFs in WebView

When I load a PDF in regular ole safari mobile, the hyper links in the PDF work. When I load a PDF in a WebView, I cannot click the hyper links anymore. Any idea how to make them work? TouchEnabled is true

  • I had this problem too. I discovered that the webview will throw an error on PDF's. Then I learned that with in the error it contained the URL to the PDF, however it contained the prefix 'app://com.mycompanyname.myappname/'.

    So what I did to get it to work is set up an error handler and captured the returned error message

    webview.addEventListener(&#39;error&#39;, function(e) { alert(e.message); });

    Then I used the javascript method "slice" to grab the url out of the error text string

    var newURL = e.message.slice(e.message.indexOf(&#39;app:&#x2F;&#x2F;com.mycompanyname.myappname&#x2F;&#39;),e.message.indexOf(&#39;,&#39;));

    This grabs the URL to the PDF which as of right now is wrong cause it has the prefix "app", so one more step is required to replace the "app" prefix with the proper URL path through javascript replace method

    newURL = newURL.replace(&#39;app:&#x2F;&#x2F;com.mycompanyname.myappname&#x2F;&#39;, &#39;http:&#x2F;&#x2F;www.myproperurlpath.com&#x2F;&#39;);

    and then loaded "newURL" in a webview.
