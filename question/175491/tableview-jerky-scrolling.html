TableView jerky scrolling

SDK 3.2.3, IOS 7.01, Iphone 4s

I have read many (if not all) of the posts about jerky scrolling on a tableView.

Most of the posts regarding this problem are more than 3 years old and many ended in someone saying this was going to be resolved in a future release.

I have tried:
Add class name to each row
Checking for memory leaks
Removing any images or background colors
Removing background images
Removing everything from the row until there is just one simple label
Removing custom fonts
Removing any processing tasks from each row
Removing imported modules from the row controller
Set absolute height on each row and any containing elements

When I stripped down the row to just one label, it got a little better, but the problem still persisted.

My table has under 100 records in it and I have implemented the continuous scrolling solution so I am only loading 20 at a time.

I have downloaded the Kitchen Sink demonstration to see if it has the same issues and it does, so I am guessing this is a limitation of this framework?

My native apps on my phone do not respond this way when scrolling. They are silky smooth.

Any other suggestions?

    • As I was working through the process of stripping my app down so I could identify the cause of the jerky scrolling, I did notice one thing that increased the performance of the scrolling tremendously.

      I had a view within the row that contained custom icon fonts. When I removed this view, the scrolling got noticeably better. Not completely smooth, but better than it was.

      Hope this helps anyone who is dealing with the same problem.
