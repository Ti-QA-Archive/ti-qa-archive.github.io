Disable a tab on iOS until I enable it?

I have an iPad app with 4 tabs. On launch, I want to prevent users from going to tabs 2 and 3 until they make a selection on the initial tab view.

I would like the disabled tabs to look disabled, not just "not work". Failing that, can I hide the tabs in question and then add them once the appropriate selection is made on the main screen?

iOS 5.1, iPad, Studio 2.02, SDK 1.8.2, OS 10.7.3

  • Accepted Answer

    1) Set the touchEnabled for these tabs = false
    2) create an eventListener that sets these to true
    3) after they click your selection you're talking about, fire the event listener.

  • While the accepted answer is perfectly valid, I would suggest you maybe rethink your approach and UI for what you are trying to accomplish.

    TabGroups are not meant for hierarchical navigation and dynamically disabling / enabling a certain tab based on user's selections in another tab is bad UX design and counter-intuitive for the user.

    If you need a user to select or enter some intitial data before you can display something meaningful in a certain tab, consider using a modal window for that instead. Alternatively, instead of disabling a tab, leave it enabled, but display a general "no data available" view explaining the user how to make the data available.
