fieldCount() and fieldCount

If ti is an abstraction layer to enable shared code between platforms then why are there discrepancies in the data base API such as this:

fieldCount() is used iOS to return the field count

but on Android it is not the same, and in calling this on Android you throw errors.

Surely as an abstraction layer i should be able to call either and the abstraction layer figure out which method to call.. i now have to write and abstraction for the abstraction which seems to make little sense!

  • The platform is not completely at parity between all supported OS's. This is for a variety of reasons; Some UI elements have no exact match in all platforms. Equally true of some of the other implementations ( Gelocation, Communication between apps [ intents vs openUrl ] are two examples). In the case of getter/setter properties and methods there has always been some slight differences. In my experience it is better to always use the properties as they seem to fully implemented between platforms
