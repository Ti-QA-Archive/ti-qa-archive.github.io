  • Here is an example I have that seems to work (Tested with 1.7)

    if(!desertMonkey) {
    var desertMonkey  = {};
}
desertMonkey.includeAbsolute = function(absolutePath) {
    if (Titanium.Platform.name == 'iPhone OS') {
        return Ti.includeAbsolute(absolutePath);
    }else{
        return Ti.include(absolutePath);
    }
}

    Example use(notice the leading"/"):

    desertMonkey.includeAbsolute('/desertMonkey/desertMonkey.service.js');

  • It is due to the fact Android and iPhone behave differently.

    Here is my hack for now (but it only works for some files):

    var resources = Titanium.Filesystem.resourcesDirectory;
if (Titanium.Platform.name == 'iPhone OS') { resources = '../../'; }
Titanium.include(resources + 'settings.js');

    I opened a ticket in the help desk and thankfully they have escalated the issue: https://appcelerator.lighthouseapp.com/projects/32238-titanium-mobile/tickets/2585-include-paths-for-iphone-and-android-are-not-based-on-the-same-starting-point#ticket-2585-3
