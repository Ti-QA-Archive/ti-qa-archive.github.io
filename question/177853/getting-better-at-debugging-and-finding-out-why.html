  • The error messages are usually pretty self explanatory:

    [ERROR] :  TiExceptionHandler: (main) [0,83] - Message: Uncaught TypeError: Cannot read property 'latitude' of undefined

    That means that you are trying to read latitude of an undefined variable. In your case this would be: e.coords in:

                    latitude: e.coords.latitude,

    The manual for getCurrentPosition says:

    If success is true, actual location data for this update.

    Which means you SHOULD check if the success property is true before reading the coords ;)

  • Hi Denny,

    the problem is, that you are trying to get a synchronous response from a function, which itself gets its data from an asynchronous callback. It sounds complicated but it isn't.

    see my comments for explanation

    var mapview = Ti.Map.createView({
        maptype: Ti.Map.STANDARD_TYPE,
        region: getLocation(), // here you call the getLocation function below which immediately returns undefined
        animate: true,
        regionFit: true
    });

function getLocation(){

        Ti.Geolocation.getCurrentPosition(function(e){ 
        // the callback function will fire after the getLocation function has returned
            var region = {
                latitude: e.coords.latitude,
                longitude: e.coords.longitude,
                animate: true,
                latitudeDelta: 0.005,
                longitudeDelta: 0.005
            };
            Ti.API.info(region);
            return region;
        });

    // the function will return immediately before the callback of the getCurrentPosition is executed
    // you don't specify a return value. to this automatically leads to
    return undefined;
    };

    to get the right execution order you'll have to modify you code like this, basically you have to pass your own callback to the getLocation function, when the callback is called, we set the region of the mapview

    var mapview = Ti.Map.createView({
    maptype: Ti.Map.STANDARD_TYPE,
    animate: true,
    userLocation: true,
    regionFit: true
});

getLocation(function(_region) {
    region = _region;
    mapview.setRegion(region);
});

function getLocation(callback) {

    Ti.Geolocation.getCurrentPosition(function(e) {
        // the callback function will fire after the getLocation function has returned
        var region = {
            latitude: e.coords.latitude,
            longitude: e.coords.longitude,
            animate: true,
            latitudeDelta: 0.005,
            longitudeDelta: 0.005
        };
        Ti.API.info(region);
        if (callback && e.coords) { // we also check if e.coords is given
            callback(region);
        }
    });
};

    btw. there is also a "userLocation" property of the mapview, which you can set to show the native user location view of the mapview, already included in the last code snippet

    Hope this helps
