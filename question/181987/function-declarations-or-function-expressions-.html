Function Declarations or Function Expressions ?

Hello,

Is there any advantage of using any type of function ?

Function Declaration

    function destroyDeathStar() {
        Ti.API.log("Stay on target, stay on target");
    }

Function Expression

var destroyDeathStar = function() {
    Ti.API.log("Stay on target, stay on target");
}

I know that declarations are created on runtime before any other code of file, and that expressions are created on the specific line. I am just wondering there are any advantages/disadvantages other than that.

  • In a controller.js file, Alloy will put your function declarations (function xxxx() {} ) before the Alloy View things.
    It will put every other controller code AFTER the Alloy view creation code.

    So, this code will fail because "eventHandler" doesn't exists

    // controller.js
var eventHandler = function() {};

    // controller.xml
<View onClick="eventHandler"></View>

    The allow generated code will look like following

    var win = Ti.UI.createView();
win.addEventListener("click", eventHandler);

var eventHandler = function () {};

    For other considerations like speek or memory usage, I'll let you to search on the Internet, but I guess it doesn't worth it (but I'd be happy to read a good explanation if there is a way to gain something.
