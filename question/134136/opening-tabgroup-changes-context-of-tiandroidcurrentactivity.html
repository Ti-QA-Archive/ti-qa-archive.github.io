Opening tabgroup changes context of Ti.Android.currentActivity

Does anyone else have the issue where the opening of a tabgroup causes the context of the current activity on android is changed. After the tabgroup is opened, proper activity events can't be called (if you hit the home button, it doesn't capture the pause event, instead, it pauses the context before the tabgroup is opened). It does call get the destroy event properly though.

    Each tab is associated with a particular window so I think you have to call the pause, resume event listener on that particular window (win.activity.addEventLister..) instead of calling Ti.Android.currentActivity.
