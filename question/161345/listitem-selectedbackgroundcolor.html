ListItem SelectedBackgroundColor

Previously in a table view I have been setting the row selectedBackgroundColor to 'transparent' so that when an item is selected it does not get highlighted (this method works for both Android and iOS).

In 3.2.0 if I do the same for a listItem when the item is selected it also applies this 'background color' to the separator colour which in effect removes the top and bottom lines around the listItem. Is there any way of preventing this or is there another way of achieving the same thing that work on both platforms?

    According to the docs, the selected background color is 'transparent' so you shouldn't need to set that. For iOS, try setting the selectionStyle to NONE
