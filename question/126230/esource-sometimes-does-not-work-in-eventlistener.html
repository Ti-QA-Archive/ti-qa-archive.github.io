e.source.* sometimes does not work in eventListener

OK, I'm prepared for the profound silence, but I'm running into situations (more than one) where when I do

foo = Ti.UI.createButton({...});
foo.bar = 'string';

foo.addEventListener('click',function(e) {
Ti.API.info(e.source.bar);
}

Then e.source.bar is printed as 'undefined' - yes, I know this is impossible but I'm having to change code right now where this happens. The same code that a week ago worked fine enough to be in the App Store.

Anyone know of any changes that can cause this to happen?

Thanks, Joe

  • Seems the e.source may reference the object and not the instance.
    I attach a user_id property to my button and catch it in the click event.
    Try using 'this' to reference the source…
    Ti.API.debug('Click!: User Id: = ' + this.user_id_str);
