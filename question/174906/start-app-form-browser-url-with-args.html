    • Ok, this is what I did:

      1. Brand new project

      2. App.js has in it only

        alert(Ti.Android.currentActivity.intent.data);

      3. Build it at this stage (so almost empty)

      4. Open the build/android/AndroidManifest.xml and copied the first activity there. It looks like this, but for you will be a little different
        <activity android:name=".CustomurlActivity" android:label="@string/app_name" android:theme="@style/Theme.Titanium" android:configChanges="keyboardHidden|orientation|screenSize">
 <intent-filter>
     <action android:name="android.intent.action.MAIN"/>
     <category android:name="android.intent.category.LAUNCHER"/>
 </intent-filter>
</activity>
      5. I pasted in tiapp.xml the previous activity in the android -> manifest -> application tag
        ~~~

      <android xmlns:android="http://schemas.android.com/apk/res/android">
      <manifest>
      <application>

      <activity android:name=".CustomurlActivity" android:label="@string/app_name" android:theme="@style/Theme.Titanium" android:configChanges="keyboardHidden|orientation|screenSize">
      <intent-filter>
      <action android:name="android.intent.action.MAIN"/>
      <category android:name="android.intent.category.LAUNCHER"/>
      </intent-filter>
      </activity>

              </application>
    </manifest>

      </android>

      
6. I added a new **intent-filter** under the first one already existent

      <intent-filter>
      <action android:name="android.intent.action.VIEW"/>
      <category android:name="android.intent.category.DEFAULT"/>
      <category android:name="android.intent.category.BROWSABLE"/>
      <data android:scheme="myapp" />
      </intent-filter>

      
so this is my new android tag now

      <android xmlns:android="http://schemas.android.com/apk/res/android">
    <manifest>
        <application>
            <activity android:name=".CustomurlActivity" android:label="@string/app_name" android:theme="@style/Theme.Titanium" android:configChanges="keyboardHidden|orientation|screenSize">
                <intent-filter>
                    <action android:name="android.intent.action.MAIN"/>
                    <category android:name="android.intent.category.LAUNCHER"/>
                </intent-filter>
                <intent-filter>
                    <action android:name="android.intent.action.VIEW"/>
                    <category android:name="android.intent.category.DEFAULT"/>
                    <category android:name="android.intent.category.BROWSABLE"/>
                    <data android:scheme="myapp" />
                </intent-filter>
            </activity>
        </application>
    </manifest>
</android>

      ~~~

      1. Clean project. Rebuild.
        You will get an empty alert when the app starts, which it's normal, there is nothing to show.
      2. Open this jsfiddle in the phone's browser
      3. Click on the "test myapp" link in the right.
        You app should open and the alert should say what the url was - in this case myapp://view?doc=123

    • You did duplicate the activity yourself.

      As I said at this point

      Open the build/android/AndroidManifest.xml and copied the first activity there. It looks like this, but for you will be a little different you don't need to copy what I sent, but your own activity in the generated AndroidManifest.xml file

      So now you have (this is mine)

       <activity android:name=".CustomurlActivity" android:label="@string/app_name" android:theme="@style/Theme.Titanium" android:configChanges="keyboardHidden|orientation|screenSize">

      while it should be

       <activity android:name=".MyAppNameActivity" android:label="@string/app_name" android:theme="@style/Theme.Titanium" android:configChanges="keyboardHidden|orientation|screenSize">

      However, I suggest you to clean the tiapp.xml completely - well, the android tag - and rebuild the app to generate a new manifest file, repeat the steps above to have the url working, then populate with the rest of the properties, meta, etc.
