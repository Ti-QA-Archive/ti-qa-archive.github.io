    Hi

    I am assuming you are using iOS as the navigation element is not available on the Android.

    In order to show a nav bar you need to either have a tabGroup or a navigationGroup. Your example code appears that you have no interest in tabs so guessing a straight navigationGroup.

    The example code below is taken directly from the online docs for the NavigationGroup

    navgroup.xml

    <Alloy>
    <Window>
        <NavigationGroup id="nav" platform="ios">
            <Window id="win2" title="Red Window" backgroundColor="red">
                <Button title="Open Blue Window" onClick="openBlueWindow">
            </Window>
        </NavigationGroup>
    </Window>
</Alloy>

    navgroup.js

    function openBlueWindow(){
    var win3 = Alloy.createController('bluewindow').getView();
    $.nav.open(win3, {animated : true});
}

    bluewindow.xml

    <Alloy>
    <Window title="Blue Window" backgroundColor="blue" />
</Alloy>

    This should get you moving.

      If you are pointing to this window_navbar it actually uses the tabGroup it is a child of to run as that is its parent when opened inside - that is how it inherits the nav controllers functionality. Whilst there is no code in there to show a tabGroup the calling methods are part of the tabGroup so the window is automatically a child of that controller when opened inside it.

      navgroup is the closest example within the kitchen sink app for a non-tabgroup way.

    • Unfortunately the NavigationGroup is not supported in Android, this is where you will need to split your UI between the two platforms. You could use the new ActionBar extension but this means your devices will need to be Android 3.0 or higher for it to function.

      The alternative is to create a synthetic ActionBar version (visually acts like a real one but made of view ans ImageViews) in a re-usable fashion and use that.
