HTML5 Boilerplate style.css does not validate

I'm working on my first project using the Aptana Studio 3 plugin for Eclipse (Indigo). I used the "HTML5 Boilerplate" option. The boilerplate includes a css file with the following path: <project root>/css/style.css.

When I open this file in the editor, four errors appear:

Errors 1-2. "Property box-sizing doesn't exist" at lines 180 and 181 (input statements with [box-sizing:] options).

Error 3. "The pseudo-element ::selection can't appear here in the context css2" at line 46. ("::selection { background: #fe57a1; color: #fff; text-shadow: none; }")

Error 4. "Value Error : clip (http://www.w3.org/TR/REC-CSS2/visufx.html#propdef-clip) Invalid separator in shape definition. It must be a comma. :" at line 241 (".visuallyhidden { border: 0; clip: rect(0 0 0 0); height: 1px; margin: -1px; overflow: hidden; padding: 0; position: absolute; width: 1px; }")

This are in the original boilerplate files, which can be downloaded from the original boilerplate web site at http://html5boilerplate.com/. But it seems strange that Appcelerator would distribute "stock" code that throws errors.

My questions are:

  1. Is there some way to fix the problems so that the editor shows no errors? Even if these errors will not be fatal in css2 contexts, having the IDE show them as errors makes finding true errors more difficult.

  2. What else can be done about these errors?

  3. What causes them?

Thanks.

  • You can suppress this error. Go to Window > Preferences > Validation > CSS and add this filer: .*\::selection.* That should remove the warning from the problems window.
