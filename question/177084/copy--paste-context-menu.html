    Hi,

    in a terminal window, just try to go to the root directory of your project (the one with the tiapp.xml file) and issue the command

    mdkir -p i18n/de

    then recompile and test. The actual wordings are part of iOS and I don't think you can change them at all. What I show you here is one way to let iOS know that your app supports German.

  • Hi,

    I (almost) fixed this for my app not an hour ago. Am I right in assuming that you don't use the default i18n process from Titanium ?

    Because, assuming you have en (english), (ru) russian and fr (french), just add at the root of your project the following directories:

    i18n/en
i18n/ru
i18n/fr

    recompile and test. No need to put a strings.xml file in there.

    It "works for me" (tm) on simulator with iOS 7 (you didn't say which version you are using, btw). I have not yet tested on device.

    Be wary of one thing: switching back and forth between defined languages work as expected but switching to a language for which you don't have a matching directory will keep the menus in the latest language used, even if you uninstall and reinstall the app. Not sure yet if it is a real production case, but it may mean I am missing something.

    Hope this helps

    PS. See here to understand where directories need to be set if I am not clear enough.
