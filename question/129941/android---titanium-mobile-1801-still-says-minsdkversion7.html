Android - Titanium Mobile 1.8.0.1 still says minSdkVersion=7

Hey all,

I'm not sure but generating an App with 1.8.0.1 or 1.9 (nightly builds) still says minSdkVersion=7. According to the 1.8 release blog post Titanium requires at least Android 2.2 (minSdkVersion=8) now.

The builder.py script defines the minimum API Level 8 but android/templates/AndroidManifest.xml still defines minSdkVersion=7.

If I publish an App generated with minSdkVersion=7, Android 2.1 Users would get the update and then the App wouldn't work.
