ImageView does not open image when called just by filename

In this example an imageview is created and just references to the filename of the image, instead of the path.

This worked (althoug not in high/low/res dirs but i'll figure that out later) when I created views by just creating a window and then adding sub views to it like this:

var win = Titanium.UI.createWindow();    
var imageview = Titanium.UI.createImageView({
    image: 'image_name.png',
});
win.add(imageview);

But now I create windows like this (with an url, which loads a js file):

var win = Ti.UI.createWindow({
    url: '/src/views/ProgramView.js',
    App: App,
});

In programview.js i have:

var win = Ti.UI.currentWindow;
var imageview = Titanium.UI.createImageView({
    image: 'image_name.png',
});
win.add(imageview);

Here (ProgramViews.js) I create an imageview, and reference the filename, but that doesn't work anymore. It searches for an image in the directory of the javascript file. Is this desired behaviour? If so, how does this work with density specific images?

I'm running Ubuntu 11.4 and titanium mobile 1.7.2.
