    • Hello Ramesh, you're welcome. So your development system is up and running, as you can compile and test the app on the simulator. The problem you describe make me wonder if the project you try to update could be an Alloy project instead of a standard Titanium.

      In order to understand this could you please check in the directory tree for the presence of an app directory (and an alloy.js file inside it, with other dir as models, views, style, controllers ..)?

      If they exists, the proj is using the Alloy MVC framework, and you will want to edit the code in the app dir, NOT in Resources dir, which is overwritten by alloy compilation before each run. So, for example, as a test-modification you could put an alert at the very beginning of alloy.js, as:

      alert("Updated by Ramesh");

      then check if the app shows it running in the simulator.

      In case the app dir is not present, try adding the alert line at the beginning of app.js, and again run the project in simulator.

      Hth.
