Reading 'height' property in 1.6 (Android)

Hi all,

I'm trying to write a pull up panel (like an android status bar, but at the bottom). Below is the essence of the code I'm using, but I'm encountering a funny bug. When I uncomment the alert, everything works fine - but the moment I comment it, the pull-up panel is too far down the screen (so its header just lips the footer of the main window).

Any clues or workarounds gratefully received.
M.

var win = Titanium.UI.currentWindow;

var header = Titanium.UI.createView({
    top:0,
    height:50,
    backgroundColor:'blue'
});
win.add(header)

var body = Titanium.UI.createView ({
    top:50,
    bottom:50,
    backgroundColor:'white'
});
win.add(body);

    var pullupPanel = Titanium.UI.createView({
        backgroundColor:'green',
        height:100
    });
    body.add(pullupPanel)

    var pullupPanelHeader =Titanium.UI.createView ({
        backgroundColor:'red',
        top:0,
        height:30        
    })
    pullupPanel.add(pullupPanelHeader)

var footer = Titanium.UI.createView ({
    bottom:0,
    height:50,
    backgroundColor:'blue'
});
win.add(footer);

win.addEventListener('open', function () {
    //alert (win.height);
    pullupPanel.top = body.height-30;
    pullupPanel.height=body.height;
})

  • Is there any reason for using alert? I'm not confident, but is the function alert the same as creating an alert dialog?

    Maybe you can try creating an alert dialog first, setting the message to be win.height.

    If you're trying to debug, then I suggest you use Ti.API.info() or Ti.API.Debug(). Its must easier that way.

    Another workaround is to create a notification, i.e. Ti.UI.createNotification and set the message to be win.height.
