iPhone buid complaining of an Android module. How to fix it?

I'm trying to run my project on iPhone Simulator but I'm getting an error asking for the .a file of a module on the android directory (which obviously there's isn't .a in). This is the error:

[ERROR] Third-party module: com.obscure.keychain/1.0 missing library at /Users/felipe/Library/Application Support/Titanium/modules/android/com.obscure.keychain/1.0/libcom.obscure.keychain.a

The libcom.obscure.keychain.a exists in this directory: /Users/felipe/Library/Application Support/Titanium/modules/iphone/com.obscure.keychain/1.0/

How can I fix the project so it looks in the correct directory?

    You can make your tiapp.xml file only include modules for your specified platform, using the 'platform' attribute. That way it won't try to include the same module file for Android when building.

    In this case:

    <module version="1.0" platform="iphone">libcom.obscure.keychain</module>

  • Iphone modules cannot be used on Android.
    Android Modules cannot be used on iOS.

    Some developers released versions for both platforms.

    If it is an Android specific module, you will need to wrap your code that uses that module in if specific platform code so it won't be called on the unsupported platform.
