Titanium.UI.createSwitch on iOS 4.2 iPad

I have an app deep in development and have found an oddity. Essentially, a simply Titanium.UI.createSwitch call works and displays great on an iPad with 3.2(.2) – however the same build on an iPad 4.2.1 will not display the switch at all. It has worked otherwise properly for months on 3.2(.2).

Has anyone else encountered this? I've tried a number of build combination's and nothing seems to be working. There are multiple switches in the app and this scenario is identical for all of them.

TI SDK 1.4.2 || iOS SDK 4.2 || Code to create:

    var rememberSwitch = Ti.UI.createSwitch({
        value: Ti.App.Properties.getBool('remember'),
        right: 0
    });

Then of course added to the view.

Tried setting height/width/backgroundColor etc to try and see where its going. Nothing has been working.

  • I had to rewrite all iPad switches for iOS 4.2 - I found that picking up the initial switch value on load/create as you are doing, did not work.

    If you try getting the 'remember' value from Properties early on in your window load and then assign it as the switch value later when you create the switch, it might help.
