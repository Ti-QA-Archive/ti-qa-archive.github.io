  • I have found that little can be executed on the pause activity, due to the app is closing down and will eventually halt.

    Adding code to the resumeHandler however, gives you all the time you need to reset your application:

    //resumehandler
Titanium.App.addEventListener('resume', function()
{
    Titanium.API.info("App resumed!");
    //Add code to reset all your variables and views here.
});

    The garbage collector will eventually kill your app in time/when/if the memory is needed for another app. But I have found it neccessary to manually reset all variables and set the application up as on a new fresh start on resume.

    Reset all your variables, properties and views on the resumehandler, and it should be pretty much what you look for I think.

    Hope this helps

    Harald
