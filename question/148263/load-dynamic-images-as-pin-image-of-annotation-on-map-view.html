  • Hi

    You can achieve this, you just need to think about it a little differently.

    The annotation view has a property called leftView, and if you remember that there is a view designed for images called imageView you can marry the two things together. An imageView has built in support for the remote images.

    Here is a a working example for you, the image is a windows flag - do not shoot me for that.

    var url = 'https://twimg0-a.akamaihd.net/profile_images/2535822544/secztwqh31xo8xydi4kq_normal.png';
var pin = Ti.Map.createAnnotation({
    animate: true,
    latitude: 51.5171,
    //leftView: leftImage,
    leftView: Ti.UI.createImageView({ image: url, height: 20, width: 20 }),
    longitude: -0.1062,
    pincolor: Ti.Map.ANNOTATION_RED,
    subtitle: 'London',
    title: info.title
});

  • I don't think you can set a URL for an annotation image. But one thing you can do is to download those images (using the HttpClient) and save those images in the Titanium.Filesystem.tempDirectory directory. From there, you can use those same images.

    Hope this helps.
