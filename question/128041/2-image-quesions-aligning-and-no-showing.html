2 image quesions: aligning and no showing

HI All,
I have 2 image related questions which I have you can help me with:

Is is it possible to center align an image, so that it's always in middle of the screen?

So I have an image view

var myImageFlag = Titanium.UI.createImageView({
    image:'landingPage_IMG.png',
    top: 5,
    left: 5,
    width: 320,
    height: 'auto'  
});
win.add(myImageFlag);

Can I say something like: imageAlign: 'center'?

And my 2nd question is about a bug. Using the above example, the PNG does not appear when viewing the app in the android emulator (2.1+ and 2.2). (I have the images in the same resources folder as the .JS folder). However, when I install the app on the real phone, the images do appear. I have tried putting the images in another folder within Resources/Android/Images but no luck. Is this a known bug? Or am I making a silly mistake?

Much appreciated!
LEe

    I am using the following code with mobile SDK 1.7.5 and the android 2.2 emulator. It shows the image directly in the center of my screen. 

    var win = Ti.UI.createWindow({
    backgroundColor: '#fff',
    fullscreen: false
});
var image = Ti.UI.createImageView({
    image: 'KS_nav_ui.png'
});
win.add(image);
win.open();

    When you give no positioning properties to your views, in this case ImageView, they will be centered within their containing view. Because I added image to my main window, it is centered within that window.
