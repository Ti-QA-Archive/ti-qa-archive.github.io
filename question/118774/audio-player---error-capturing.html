Audio Player - Error Capturing

Im working on getting a streaming radio client working in an Iphone App. everything works great, except for this little issue:

is there any way to have valid error capturing for the Audioplayer.

i really don't want the user to see the technical error messages that pop up in the alert box when: 

 - a user doesn't have an internet connection
 - the feed went dead. (lets hope that never happens)

  • You can detect the internet connection like this:

    Ti.Network.addEventListener('change', function(e) {
    Ti.API.log('is online? ' + e.online);
});

    I don't think there is a direct way to check if the feed went dead, but you can check the property STATE_WAITING_FOR_DATA using a setInterval() every second and if the feed is not available inside a timeframe you specify then show an error message.
