Type of Save & Edit Later system?

Alright so I have an app with my contractors, and they are asking me to add a "Save & Edit Later" system to the app.

The app is pretty much: Fill out a whole shit load of textFields, Switches, and take 3-6 pictures, and at the end dump it into a remote MySQL Database and refresh the entire app, and start over. All the values are stored via the Ti.App.Properties module and, at the end, set to "null".

How would I store all this information on the device if the user clicks a button and then view all this information later, and then show it all in a view and they could click it and load it all back into the app. I was thinking of storing it all in a file somehow, but I blank everytime I want to program it. Does anybody have a project available that shows how this would be done?

  • Depending on how much data you need to store, you can do this using Ti.App.Properties to store a certain amount of data of various types.

    http://developer.appcelerator.com/apidoc/mobile/latest/Titanium.App.Properties-module

    Or you should have a look at creating and using an sqlite database to store things.

    There are a couple of demos for each in the KitchenSink demo.
