Best practice for loading images in tableview (iOS)

I'm looking to load people's Facebook friends images next to their friends names in TableView rows. I have it doing the names fine right now, but it appears leftImage doens't support remote URLs (and, if I do try to use leftImage, it slows down the 300+ friends a LOT).

I'm looking for a way that is:
1.) Quick
2.) Smooth

I'd like to know what the best methods you've seen for making it happen in Titanium, in a non-intrusive way (like jarring up the UX by making scrolling through the tableview choppy.)

Thanks!

  • You can create a TableViewRow and explicitly add views to it. One of those views might be an ImageView.

    Two things:

    • don't use high-res images that are scaled down to small sizes. Figure out a way to deliver small images to the client application. Some of the work of building a good mobile client app happens on the server side.
    • don't use 300 rows if you don't need them. Filter this list down, show 20-30, then provide a means for the user to load more
