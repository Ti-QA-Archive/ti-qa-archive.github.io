Error: The result of 'Ti.Gesture' [undefined] is not an object (only happens on the device!)

Hi all,

I just created my first app for iPad, developing on Mac OSX Lion. It runs fine in the simulator (as long as I keep cleaning the project build folder). However, when I try to run it on the iPad, i get the error

Error: The result of 'Ti.Gesture' [undefined] is not an object

in this piece of code ripped from KitchenSink:

Ti.Gesture.isLandscape = function(orient)
{
orient = orient || Ti.UI.orientation;
return orient == Ti.UI.LANDSCAPE_LEFT || orient == Ti.UI.LANDSCAPE_RIGHT;
};

The same error occasionaly occurs in the simulator when i do NOT clean the project and do a full rebuild, but on the ipad it happens all the time.

It seems that the full rebuild does not help when deploying to ipad. Perhaps it builds twice and overrides some code?

I tried running from XCode but the result is the same.

This error is in the knowledge base more than once, but the result that has been posted does not help for me (full rebuild) on the device.

How can I force a full rebuild when deploying to the iPad? Or is there another way to compare or control the build process when deploying?

I find it frustrating that an error that clearly pertains to some linking problem in the code can happen 'sometimes' even when not changing any code… :/

Any advice or tips are welcome!

-Erik

  • Fix: 1. Make sure XCode has all of the targets set for SDK version 4.0. 2. Make sure you call all of the Ti.XXX functions in fake calls in the beginning of app.js. (Dirty Hack!). I used:

    // Hack to make sure modules are loaded...
//
var bla = Titanium.createImageView();

    After that, clean your build, run on sim from Titanium.
    Then, Finally, don't run on the device from Titanium Studio, but from XCode directly.

    If you still get errors, include these calls in the app.js in similar way.

    E.
