resize imageview gives bad quality

Hi guys,

I wonder if somebody has a solution for this problem. Added an image via createImageView works, but when I resize it to half the size the quality of the image displayed is really bad. It becomes even worse with <supports-screens android:anyDensity="false"/> probably because it get resized more?

So here is my demo for everyone to try:
(it also includes my createButton work around, but that gives bad performance in scroll/tableviews)

// this sets the background color of the master UIView (when there are no windows/tab groups on it)
Titanium.UI.setBackgroundColor('#000');

//
// create base UI tab and root window
//
var win1 = Titanium.UI.createWindow({  
    title:'Tab 1',
    backgroundColor:'#fff'
});

var imageurl = 'http://dl.dropbox.com/u/63372320/default_logo_small_de.png';

var image2 = Titanium.UI.createButton({
    backgroundImage: imageurl,
    backgroundSelectedImage: imageurl,
    title: '',
    left: 5,
    top: 5,
    width: 171,
    height: 99
});

win1.add(image2);

var imageView2 = Ti.UI.createImageView({
    image: imageurl,
    left: 5,
    top: 110,
    width: 171,
    height: 99
});

win1.add(imageView2);

var image3 = Titanium.UI.createButton({
    backgroundImage: imageurl,
    backgroundSelectedImage: imageurl,
    title: '',
    left: 200,
    top: 220,
    width: 85,
    height: 50
});

win1.add(image3);

var imageView3 = Ti.UI.createImageView({
    image: imageurl,
    left: 200,
    top: 280,
    width: 85,
    height: 50
});

win1.add(imageView3);

win1.open();

What is strange, if I add the image as button, the resizing gives way better quality for some reason, check out the following screenshots:

http://dl.dropbox.com/u/63372320/device-2012-02-21-141332.png

and here with supports-screens android:anyDensity="false"/> set:
http://dl.dropbox.com/u/63372320/device-2012-02-21-141655.png

I would use the createButton method as workaround, but that gives way worse performance within a scrollview, so I hope there is a way to make the createImageView look better. Anyone?

Thanks for your time :)

PS: seems like the mobilesdk version you run doesn´t really matter, for reference: screenshots made with app built on mobilesdk version 1.8.1 on a Samsung Galaxy S2.

  • This may or may not help you, but in my situation, I had success with only changing width or height of the view after it was already added, but not both. My experience was, if you change one, the other gets changed maintaining the same aspect ratio as the original. But that was on an older SDK. It might be worth a try.
