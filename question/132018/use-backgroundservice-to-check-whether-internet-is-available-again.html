Use backgroundservice to check whether internet is available again

Hi,

in my App users can modify some data without the need to have internet available. For my service it is important, to receive the modified data via webservice as soon as the user's phone has internet available again.

In the mean time the user most probably has backgrounded my App, so I see two possible ways:

  • Using the backgroundservice - but the docs say it will be terminated by the os (i. e. iOS) sooner or later
  • Using localNotifications on a daily basis - but I don't want to bother the user with messages not relevant for him (same for push notifications…)

Under certain conditions the backgroundservice will be kept alive by iOS, e. g. using geolocation or streaming audio. Is there any chance to keep the backgroudservice active, until internet is available again and the modified data is sent?

Regards,
Christoph

    • Thanks for your feedback! Exactly these limitations (copy from docs) is what I try to overcome.

      • The OS limits the total amount of time the background service can run for after the application is suspended, typically to no more than 10 minutes.
      • The OS may terminate the background service at any point to reclaim resources

      It is possible to prevent the OS from stopping the backgroundservise. E. g. playing/streaming audio or using geolocation. I'm curious if there might be another way, e. g. trying to connect to the webservice in a given (probably short) interval.
