Density Pixels, differences between Android and iOS retina, please confirm

Hello,

In my projects im working quite alot with density pixels. Sometimes however i have to do calculations with both dp and px. For this to work i found a function that converts dp units to pixels and vica verca (here).

However just before publishing the first app using this function in the apple app store, I noticed that this function does not work properly with retina displays. I think the reason behind this is that iOS does the scaling of pixels to density pixels all by itself, which means that when you want to calculate the pixel value of a certain dp value, it should be just the same. The function i used however would double the size.

Old function according to this:

DPUnitsToPixels = function(TheDPUnits)
{
    return (TheDPUnits * (Titanium.Platform.displayCaps.dpi / 160));
}

A 3.5" retina display should have a resolution of 960 x 640 pixels with a DPI of 320. What happens when you are developing for retina is that you have only half the resolution, but the same DPI.

What would happen in the old function is that when you want to calculate the pixel value of a certain dp value, double the expected value is calculated.

This has caused some problems for me on retina displays with iOS, and im not quite sure if i have solved this the correct way, so I'd like to verify my solution. I think that it is not needed to convert density pixels to pixels, because on iOS the resolution is always based on a medium density (so it behaves as if the dpi is always 160). For this reason i assumed that it is not nessecary to calculate the density pixels to pixels because it is always a 1 to 1 comparison.

Therefore my new improved function:

DPUnitsToPixels = function(TheDPUnits)
{
    if(Ti.Platform.osname == 'iphone')
        return TheDPUnits; 
    return (TheDPUnits * (Titanium.Platform.displayCaps.dpi / 160));
}

I hope someone can verify my solution for iPhone retina displays, or maybe ive done something wrong and one can point that out for me.

Hylke
