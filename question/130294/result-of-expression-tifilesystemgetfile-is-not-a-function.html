Result of expression Ti.Filesystem.getFile is not a function

Titanium 1.8, OS X 10.7, iPhone Simulator

Sigh. It's partly my own fault. I broke the cardinal rule of Titanium development: "Don't ever touch a working program." But I dared to try adding a feature, so here I am.

When I run my application, almost immediately I get a Titanium runtime warning:

Application Error
Result of expression
'Ti.Filesystem.getFile' [...] is
not a function. at file.js (line 9)

My file.js is included, and the function at line 9 is called a few times during setup, although the call causing the error seems to be unpredictable.

The […] changes occasionally, and is a string from my program that has not been encountered yet.

And sometimes, of course, the program starts up properly.

So based on those forensics, my first guess is that there's some sort of memory corruption going on. Based on the fact that nothing has changed with program startup, I'm left to guess that the memory corruption might actually be caused during…compilation? …setup? …something else?

Anyway, I'm at a lost as to where to even start looking for where things are going wrong.

Can someone give me a small hint as to what would cause that sort of thing?

  • I just found the solution specific to my problem. Even though I didn't use any Titanium File I/O directly, I've been employing rjss feature of redux.js, which used Ti.Filesystem.getFile to parse the rjss data to process. My blunder was using the unsupported feature of rjss, specifically, commenting out a property of style element using the pair of / … / The comment-out worked alright for Android port, but iOS, for it generated the error and just stopped right away. I guess your app might read a resource file or something somewhere without your notice, which generates File I/O error or something you might not be aware of due to permissions or etc.,
