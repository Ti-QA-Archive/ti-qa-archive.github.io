iOS5 NavigationGroup 'back' bug

This is really a response to Raul Riera's question 'Bug in iOS5 : Tab group windows' asked on 14th October 2011 which appears to be 'locked and cannot accept new answers.'

Anyway just to say I have also hit this one where on a device NavigationGroup windows animate nicely when they open but close with no animation when the 'back' button is clicked. This appears to be a bug associated with iOS5 as animation works both ways in the Simulator.

I'm also getting many warnings when building in Xcode including one in TiUIiPhoneNavigationGroup.m which flags up a wonderful comment in the code which states "TODO: This is a hideous hack, but NavGroup needs rewriting anyways"

Which just about says it all!
