Large file download on mobile(iOS & Android) Memory Issue.

Application type: Mobile
Titanium SDK:3.2.0.GA
Platform & version: iOS(7.0),Android(KitKat)
Device: iOS simulator, Android emulator or physical device model
Host Operating System: OSX (10.9)
Titanium Studio:3.2.0.201312191547

Hi , I am trying to download multiple files from remoteServer using HTTPClient. I am getting memory issue while downloading large files(more than 200 MB). Please suggest me what will be the solution for this? I have followed the KitchenSink app, for file download. Still Getting the same Memory Issue , when trying to download some larger file. Titanium HTTPClient (this.responseData) saves the data to memory before committing it to the destination like applicationDataDirectory.

What will be the good way to solve this memory Issue? How to use the FileStream operation while download under progress instead of using the complete internal memory? Is there any API available, where i can download multiple files with no memory Issue?

  • If you use the file property this should resolve some of your issues as this writes the content directly to file, bypassing some of the in-memory storage.

    However, you or your users may still encounter memory problems due to other apps that are running on the devce using memory.

    I suggest that you split up your large files on the server and download them as smaller chunks.
