Android module: Unable to find explicit activity

I'm developing an Android module that just needs to start a new activity like so:

@Kroll.method
public void doIt()
{
    final Context ctx = getActivity();
    final Intent intent = new Intent(ctx, MyActivity.class);
    intent.addFlags(Intent.FLAG_ACTIVITY_NEW_TASK);

    ctx.startActivity(intent);
}

But however I try, I always get the error "Unable to find explicit activity class {com.my.application/org.my.androidmodule.MyActivity}; have you declared this activity in your AndroidManifest.xml". And guess what, I also don't see any way to declare it in AndroidManifest.xml (neither of my application nor my module) even if I add the &lt;application&gt;&lt;activity ...&#x2F;&gt;&lt;&#x2F;application&gt; element in timodule.xml/tiapp.xml - they're just ignored and not put into the output manifest file "build\android\AndroidManifest.xml".

  • Okay solved it fortunately, the manifest element was missing around the application element.

    So this is how it should look like in timodule.xml:

        <android xmlns:android="http://schemas.android.com/apk/res/android">
        <manifest>
            <application>
                <activity
                android:name="com.example.foo.MyActivity"
                android:configChanges="fontScale|keyboard|keyboardHidden|locale|mnc|mcc|navigation|orientation|screenLayout|uiMode|touchscreen"
                android:theme="@android:style/Theme.NoTitleBar.Fullscreen" />
            </application>
        </manifest>
    </android>

    The good thing is that I don't even have to copy it to my application's tiapp.xml.
