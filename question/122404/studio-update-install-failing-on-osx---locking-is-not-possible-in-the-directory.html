Studio Update Install failing on OSX - "locking is not possible in the directory"

Hi all, I need some help. After a few weeks of vacation inactivity, I started Studio and was told my version was expired. Nothing was in my upgrade url's so I downloaded and attempted to install the latest version of studio. After downloading and dragging the icon to the application folder, I received an error that I didn't have permissions on the configuration directory under studio. I ensured my account and admin had read/write and now I'm getting a bizare error dialog:
"Invalid Configuration Location", "Locking is not possible int he directory "/Applications/titanium Studio/configuration/org.eclipse.osgi", "A common reason is that the file system or runtime enviornment does not support file locking for that location…"
Worse, Studio will not start at all and I can't go forward or back - I'm dead in the water. Any suggestions? Appreciate any advice.

  • You can trash the Studio install and start from scratch, just make sure that you don't trash your workspace. The default workspace is usually ~&#x2F;Documents&#x2F;Titanium Studio Workspace&#x2F;. This should make it so you can reinstall Studio without losing any of your work.

  • I've run into this issue before on another application built ontop of eclipes, while deleting the application and then reinstalling it is a solution, an easier fix would be to open terminal an enter in the command sudo chmod 755 -fR /Applications/titanium Studio (Use 777 if 755 doesn't work for you, just note it is not as secure) as this will allow all the folders to have full read write permissions. The next launch you might get a message stating something along the lines of can't open workspace /root/(name of workspace location) just change the workspace to the ~/Documets/(Location of the workspace you've been using) I know this sounds like a lot, but is really easy to do and a lot faster than having to reinstall the applicaiton.
