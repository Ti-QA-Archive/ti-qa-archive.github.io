Alloy framework and Memory Leaks

Hi,

I just wanted to ask if the Alloy framework prevents memory leaks? Does it null widgets and remove event listeners as soon as a window/controller is closed? Is there something to take into consideration?

  • Accepted Answer

    Alloy itself isn't engineered in a way that retains objects or creates memory leaks. However, depending on how you program you can, yourself, create memory leaks inside of alloy (like any other Javascript program / framework, etc.).

    For instance, your examples of removing events when a window is closed: That's up to you. There are cases where a window is closed but still needs to be held in memory for later use. There are cases where a window and it's objects should go away once they're closed. That control is up to you.
