  • Banepalli,

    The function is valid only within the orientation event listener and is accessed via the event object and it takes no arguments. 

    
Ti.Gesture.addEventListener('orientationchange', function(ev) {
    if (ev.source.isLandscape()) {
        Ti.API.info('landscape');
    } else if((ev.source.isPortrait()) {
        Ti.API.info('portrait');
    }
    Ti.API.info('changed');
});

    It's best to check for both landscape and portrait as shown above. The orientation can switch to an "undefined" state as the device rotates. If you don't specifically test for both portrait & landscape in your code, you could end up executing the else block with bad orientation data.
