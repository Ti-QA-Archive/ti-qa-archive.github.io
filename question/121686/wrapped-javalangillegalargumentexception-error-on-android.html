Wrapped java.lang.IllegalArgumentException error on Android

My app is targeted for iPhone and Android. I am using Titanium Studio, build: 1.0.1.201106171427, Titanium SDK 1.7.1 Google API Android 2.2.

The app retrieves category data from a local database and populates a table. When one of the categories are selected the clients are retrieved from a database and displayed in a table. When a client is selected, a window opens with more detailed information from the database.

The app works as expected on iPhone, but when run on my Sprint EVO, it works as expected until I select a client. The device throws this error:

>Wrapped java.lang.IllegalArgumentException:Expected org.appcelerator.titanium.proxy.TiViewProxy type for argument "child" in "add", but got undefined (app://clientinfo.js#99)

I do not know how to read this error unless in clientinfo.js which is the file for the window with info, and if #99 is the line containing the error then line 99 is: clientWin.add(closeButton);

My closeButton code is:

if (Ti.Platform.osname === 'iphone'){
    var closeButton = Ti.UI.createButton({
        title:'Close',
        height:40,
        width:300,
        top:300,
        left:10,
        font:{fontSize:20}
    });
    win.add(closeButton);

    closeButton.addEventListener('click', function()
    {
        win.close();
    });

Can someone help me understand what is wrong here and how I might fix it?

    David

    The error references app:&#x2F;&#x2F;clientinfo.js#99, which indicates a problem with either line 99 of clientinfo.js or, as is more often the case, the line of code immediately before line 99. What is just before clientWin.add(closeButton) in this case?

    Cheers

    • David

      My apologies, I got it the wrong way around - app:&#x2F;&#x2F;clientinfo.js#99 would indicate an issue with the line of code that follows line 99.

      I think the following block:

      clientWin.add(clientLogo);
clientWin.add(clientAddress);
clientWin.add(clientLgImage);
clientWin.add(button);
clientWin.add(closeButton);

      should be replaced with:

      win.add(clientLogo);
win.add(clientAddress);
win.add(clientLgImage);
win.add(button);
win.add(closeButton);

      Also, clientWin should be in created and opened within an eventListener, so that it is properly destroyed when closed.

      If you are still having trouble, could you describe which of the objects you would like on the first screen/window, which object the user clicks to get more info, and which objects you would like on the screen/window that opens. Then I can provide the full code.

      It's a little tricky, because the tableview you have mentioned does not appear in your code. However, I hope the following demonstration, based on your code, will help you towards a solution:

  backgroundColor:"white",
  exitOnClose: true

clientLogo.addEventListener('click', function(){
  var clientWin = Ti.UI.createWindow({
    backgroundColor:"#fff"
  });

  var clientAddress = Ti.UI.createLabel({
    text:'12345 E. Main St.',
    top:25,
    left:130,

  button.addEventListener('click',function(e){
    if (Ti.Geolocation.locationServicesEnabled===false){
