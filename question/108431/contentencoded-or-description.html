content:encoded OR description

Hi!

I'm working on a simple rss reader. Some feeds use the 'description' tag instead of 'content:encoded', and I can't figure out how to make my app choose 'description' if 'content:encoded' isn't available…

If i call for 'content:encoded' when it's not available, the app won't even show the tableview.

I hope you understand what i mean :P

I tried something like this, but I could almost understand that this isn't right:

var title = null;
        var desc = null;
        var date = null;
        var content = null;

            // Item title
            title = itemList.item(c).getElementsByTagName("title").item(0).text;
            desc = itemList.item(c).getElementsByTagName("description").item(0).text;
            date = itemList.item(c).getElementsByTagName("pubDate").item(0).text;

            if(itemList.item(c).getElementsByTagName("content:encoded").item(0).text == null)
                {
                    content = itemList.item(c).getElementsByTagName("description").item(0).text;
                }
            else
                {
                    content = itemList.item(c).getElementsByTagName("content:encoded").item(0).text;
                }

EDIT:

I'm using the rss.js from kitchensink. It works perfectly when loading feeds that uses 'content:encoded'. I want to be able to load both types of feeds.
