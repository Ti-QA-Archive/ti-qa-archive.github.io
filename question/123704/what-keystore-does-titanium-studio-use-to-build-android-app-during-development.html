What keystore does Titanium Studio use to build Android app during development?

Ok, so I messed up a little (maybe a lot).

On my initial deployment to the Android Market I used the last "Run -> Android Device" generated .apk file found in my projects "build/android/bin/app.apk".

I now know that I should have used "Deploy -> Distribute - Android" to build the final apk but I didn't.

Anyway, the Android Market took that bad apk and now when I do the apk the proper way with a keystore it says the key doesn't match the existing app.

So my question is: what keystore did Titanium Studio use when it was building the "Run -> Android Device" apk?

  • Accepted Answer

    You didn't mention your OS, but on Mac you can find the development keystore for Android at:

    &#x2F;Library&#x2F;Application Support&#x2F;Titanium&#x2F;mobilesdk&#x2F;osx&#x2F;1.7.2&#x2F;android&#x2F;dev_keystore

    I'm sure its in a similar path on Windows if that is your OS.
