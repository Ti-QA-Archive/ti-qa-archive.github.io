When should I use Ti.Database ?

My application collects data and stores it in objects over the course of a year and pulls a month worth of data out and into the cloud when that month has exceeded one year's time. I am worried about the amount of local data that might accumulate but cannot continually call it from the cloud because it will be used regularly. Would it be beneficial to store it in another manner such as a Ti.Database? Alternately, I have attempted to work out methods of storing past months in directories, releasing the local data after periods of dormancy, calling upon it when requested and repeating the process. Does this all sound like best practice?

  • I think you can treat the local database as caching, I havent heard anyone hitting a size limit, per this thread, somebody hit 250mb http://developer.appcelerator.com/question/145074/limit-of-data-that-can-be-stored-in-the-sqlite-database-in-the-titanium-application

    As for storing it in different directories, you are practically sharding it based on month, yes it will work but it is limiting if you want to "query" spanning different months. I think it is better if you store it in a sqlite database, have a timestamp there, then you can query and manipulate the data however you want.
