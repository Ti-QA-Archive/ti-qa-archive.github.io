  • Accepted Answer

    I was getting the EXACT same error (SystemExit: 65 and build.py line 1222) and I figured out what the cause was in my particular case. I had a directory in my Resources directory named the same as my project. My project was called twitterclient in Titanium Studio and had a directory called Resources/twitterclient. Apparently, since iOS SDK 5.0 and/or Titanium Mobile SDK 1.7.5 this became an issue. I renamed Resources/twitterclient to Resources/twitterclient_include and the problem was RESOLVED!

    I was banging my head on the desk for many hours on this one. I hope this helps some of you guys.

    Dave

  • I had the exact same problem here. I checked out my build.log file and read the following line:

    UnicodeDecodeError: 'ascii' codec can't decode byte 0xcc in position 69: ordinal not in range(128)

    I figured out, that I had the german umlaut 'ü' in the project name. Creating a new project and rewriting the 'ü' with 'ue' within the project's name solved the issue. The project's content (tiapp.xml, Resources folder, etc.) I simply copied. So I don't think the issue was caused by it.

    Be careful with special characters - especially within tiapp.xml … I once had line breaks in the description field causing the build process to fail.

    Chris
