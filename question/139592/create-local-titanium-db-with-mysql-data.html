  • Here is how I would look at the steps involved with the app you mention…..

    Use HTTPClient to request data from your php/MySQL server in JSON format.

    Use the parsed JSON to fill your local DB (sqlite).

    These are oversimplified, but it gives you an idea. I would watch some videos. I would try to get some functioning code going that stores data locally first. Or, try getting remote data to display in the console or in an alert window first. Pick one task, get it to work first, then work on getting both to work together.

    Working With Local Data - Video

    Working With Remote Data - Video

    Working With Local Data Sources - Written Guide

    Working With Remote Data Sources - Written Guide

    And here is an older and deprecated guide that helped me get started with Titanium Mobile, and a remote server running PHP and MySQL. From what I can tell, the only reason it's deprecated, it's not "best practice" with structuring code, but it should still work.

    HTTPClient With PHP and MySQL

  • Hi Tareq dowla!
    it is an easy way to store mySQL (php) Data into your SQLite (local) Database.
    here are some simple steps.

    Simple Steps

    1. create webservice from php end
    2. write functions to fetch data from MySQL through SOAP call
    3. make SOAP call from Titanium end
    4. fetch data and store it into SQLite (local) DB

    if you got any problem regarding SOAP call, We are there to help you.
