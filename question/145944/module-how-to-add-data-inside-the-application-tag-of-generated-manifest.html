Module: How to add data inside the <application> tag of generated Manifest?

Application Type: Mobile
Titanium SDK: 3.0.0 GA
Platform : Android
OS: Ubuntu Linux 12.04LTS

Hi,

I have been trying to write a module which can integrate Airpush with my appcelerator application.
Airpush requires me to add a few data (meta-data) inside the <application> tag of the manifest file along with other permissions which become a part of <manifest> node.

I have added all those things in timodule.xml like this:

<?xml version="1.0" encoding="UTF-8"?>
<ti:module xmlns:ti="http://ti.appcelerator.org" xmlns:android="http://schemas.android.com/apk/res/android">
    <!--
    Similar to tiapp.xml, but contains module/platform specific
    configuration in <iphone>, <android>, and <mobileweb> sections
    -->
    <iphone></iphone>
    <android xmlns:android="http://schemas.android.com/apk/res/android">
        <!-- Mandatory settings for Airpush -->
        <uses-permission android:name="android.permission.INTERNET" />
        <uses-permission android:name="android.permission.READ_PHONE_STATE" />
        <uses-permission android:name="android.permission.ACCESS_NETWORK_STATE" />

        <!-- For Push Notification -->
        <uses-permission android:name="android.permission.RECEIVE_BOOT_COMPLETED" />

        <!-- To create icons on homescreen -->
        <uses-permission android:name="com.android.launcher.permission.INSTALL_SHORTCUT" />
        <!--application-->
            <meta-data
            android:name="com.NffESQhryA.aWHFewFUdE126926.APPID"
            android:value="some-id" />
            <meta-data
            android:name="com.NffESQhryA.aWHFewFUdE126926.APIKEY"
            android:value="some-key" />
            <!-- This activity is required for all ads. This activity is also used for Dialog ads. -->
            <activity
            android:name="com.NffESQhryA.aWHFewFUdE126926.OptinActivity"
            android:configChanges="orientation|keyboardHidden"
            android:exported="false"
            android:theme="@android:style/Theme.Translucent" />
            <!-- End of required declarations for all ads -->
            <!-- Start declaration for push notification ads. -->
            <service
            android:name="com.NffESQhryA.aWHFewFUdE126926.PushService"
            android:exported="false" />
            <receiver
            android:name="com.NffESQhryA.aWHFewFUdE126926.BootReceiver"
            android:exported="false" >
                <intent-filter>
                    <action android:name="android.intent.action.BOOT_COMPLETED" />
                    <category android:name="android.intent.category.HOME" />
                </intent-filter>
            </receiver>
            <!-- End declaration for push notification ads. -->
            <!-- Required activity for AppWall and Landing page ads. -->
            <activity
            android:name="com.NffESQhryA.aWHFewFUdE126926.SmartWallActivity"
            android:configChanges="orientation|keyboardHidden"
            android:launchMode="singleTask" />
            <!-- End of Airpush Manifest declaration -->
        <!--/application-->
    </android>
    <mobileweb></mobileweb>
</ti:module>

(I have tried adding and removing application tag with the same effect)

Once built, the manifest file under build folder reads this:

<?xml version="1.0" ?><manifest android:versionCode="1" android:versionName="1.0" package="com.example.app" xmlns:android="http://schemas.android.com/apk/res/android">
    <uses-sdk android:minSdkVersion="8"/>

    <!-- TI_MANIFEST -->

    <application android:debuggable="false" android:icon="@drawable/appicon" android:label="HelloWorld" android:name="HelloworldApplication">

        <activity android:configChanges="keyboardHidden|orientation|screenSize" android:label="HelloWorld" android:name=".HelloworldActivity" android:theme="@style/Theme.Titanium">
            <intent-filter>
                <action android:name="android.intent.action.MAIN"/>
                <category android:name="android.intent.category.LAUNCHER"/>
            </intent-filter>
        </activity>
        <activity android:configChanges="keyboardHidden|orientation|screenSize" android:name="org.appcelerator.titanium.TiActivity"/>
        <activity android:configChanges="keyboardHidden|orientation|screenSize" android:name="org.appcelerator.titanium.TiTranslucentActivity" android:theme="@android:style/Theme.Translucent"/>
        <activity android:configChanges="keyboardHidden|orientation|screenSize" android:name="org.appcelerator.titanium.TiModalActivity" android:theme="@android:style/Theme.Translucent"/>
        <activity android:configChanges="keyboardHidden|orientation|screenSize" android:name="ti.modules.titanium.ui.TiTabActivity"/>
        <activity android:name="ti.modules.titanium.ui.android.TiPreferencesActivity"/>
        <service android:exported="false" android:name="org.appcelerator.titanium.analytics.TiAnalyticsService"/>
    </application>

    <uses-permission android:name="android.permission.ACCESS_NETWORK_STATE"/>
    <uses-permission android:name="android.permission.WRITE_EXTERNAL_STORAGE"/>
    <uses-permission android:name="android.permission.ACCESS_MOCK_LOCATION"/>
    <uses-permission android:name="android.permission.ACCESS_WIFI_STATE"/>
    <uses-permission android:name="android.permission.ACCESS_COARSE_LOCATION"/>

</manifest>

As it can be seen, the permissions have been added successfully, but the stuff that I needed to add under application tag (the <meta-data> part) can't be seen anywhere, and thus when I run it, Airpush throws exceptions saying it can't find that meta-data in manifest.

Any clue where I might be going wrong?

    I'm not sure how it works for modules, but for Titanium apps you make a copy of the generated AndroidManifest.xml in the build folder and add it to your project under /platform/android/. The file there is the one that will be used and you can modify it as you need.
