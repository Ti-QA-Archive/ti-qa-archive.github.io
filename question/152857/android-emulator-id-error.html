  • the path to the Android SDK [default: /opt/android-sdk]
  • the id for the avd [default: 7]
  • the skin for the avd [default: HVGA]

It appears to be the id for the avd that is causing the problem. Running 'android list targets' does not display an available target with an id of 7. There is one with an id of 2 (that has the correct API level) and so if I try 'titanium build –platform android -I 2', the emulator opens and my app displays.

Is there an easy way for me to change the default id for the avd?
