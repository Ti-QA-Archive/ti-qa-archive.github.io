Remote ImageView doesn't understand variable on Image property.

Hey guys,

I'm trying to use a remote ImageView that will be configured dynamically. That is:

    var strSource = _dbNews[i].thumbSrc;
    Ti.API.info('My image source is: ' + strSource);
    //This info returns a valid url such as http://address.com/image.jpg or .png

    Ti.API.info('strSource stype: ' + typeof strSource);
    //This info returns 'String'

    var postThumb = Titanium.UI.createImageView({
        image: strSource,
        left: 10,
        top: 40,
        hires: true,
        preventDefaultImage: true,
        height: 136,
        width: 308
    });

But it doesn't work.

I'm positive that strSource is holding a valid url, because I've tested the url on my browser, however, my ImageView doesn't show me anything!

If instead of using a variable I just paste the static url string, it fetches the image correctly.

What's going on? How can I load images dynamically into my ImageView?

Thanks!

  • the code is ok and, if strSource has a valid url, it should work. Check that you have internet connection from the emulator, I have seen some related problems (specially in android emulator). Try using the emulator browser.

    Also, sometimes a remote image can't be loaded even if it exists. It has happened to me very few times and I don't know if is a problem with the server (maybe refuses connections from other apps than browsers) or with the file format. Check another url to ensure that it is not your case.
