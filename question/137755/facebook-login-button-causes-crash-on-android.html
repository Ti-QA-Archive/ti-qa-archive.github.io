Facebook LogIn-button causes crash on Android

Hej, as we say in Swedish :)

I have this weird issue i haven't been experiencing before. I have a tableview, and in the first tableviewrow i have a "Facebook login" button that is created like this

fbRow.add(Ti.Facebook.createLoginButton({top: 38*h/480}));

Problem

When i open up the app, if i was logged into Facebook when i closed the app, the app crashes if device is on WiFi. If i turn of WiFi and are connected through 3G instead it works fine.
When i press "LogOut" (still the button that is created through createLoginButton), the app crashes but when i start it again i am logged out of facebook.

I am experiencing this issue on Android 4.0.2, Ti SDK 2.0.1 GA2.

  • How about if you create the login button first, on a separate line, and then add it ti the row after?

    Also, I would be careful specifying a top position parameter like that with a calculation - since it may not always be an integer (whole number) in pixels. Wrap the calculation result in Math.floor( … )
