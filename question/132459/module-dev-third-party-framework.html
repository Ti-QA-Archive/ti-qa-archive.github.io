Module dev third party Framework

Hi,

I am developing a module which needs a third party framework. This third party framework has a dependency of the SystemConfiguration framework which comes with XCode.

I am following the steps of the module dev guide but somehow this third party frameworks is giving problems.

This is how I am setting up the module.

  • I am adding to the xcode project the two frameworks I need, SystemConfiguration and the third-party one (called comScore).
    On the other hand I add the dependencies to the module.xconfig file, so that it looks like this:
OTHER_LDFLAGS =$(inherited) -framework SystemConfiguration -framework comScore
  • I then execute: 
./build.py

which works without problems, so message is: BUILD SUCCEEDED

  • When I try to run the module test 
titanium run
  • I get this error:

  • Hello, There.

    I was having a similar problem which all you guys are have had here.

    I've added dependencies I need to build into module.xconfig as you mentioned first of your post before like this.

    //NOTE: AdStir is the 3rd party library I am trying to link.
OTHER_LDFLAGS =$(inherited) -framework Foundation -framework QuartzCore -framework SystemConfiguration -framework AdStir

    Even though I added some compiler options manually, compiling ended up giving a failure code.

    Then I've discovered a thing which is explained here and all we need more is probably adding the -F compile options to specify where to load your additional frameworks files.

    //NOTE: AdStir is the 3rd party library I am trying to link.
OTHER_LDFLAGS =$(inherited) -F"PATH TO YOUR FRAMEWORK" -framework Foundation -framework QuartzCore -framework SystemConfiguration -framework AdStir

    At least this worked for me.
    Hope this works for all you guys.

  • Hello, Tamara.

    The my post above explaing how to resolve path lookup was not perfect becuase the way to specify was still static and once if you had changed your dev-environment, it would haven't worked anymore.
    I've just figured out the new way to specify a platform dependencies in the "module.xcconfig" and so I'm suggesting you trying this.

    Here is a little trick of how.

    1) Find out what is your module ID and version number that you've been working on and copy it.

    2) Add two lines in module.xcconfig like this:

    // Here is the little hack of how to resolve environmental path dependencies.
MODULEID=jp.foobarmodule
VERSION=0.1.0

    3) Define a base path where your modules are located in your computer.

    EXT_MODS_BASEDIR=$(SRCROOT)/../../modules/iphone/$(MODULEID)/$(VERSION)/platform/iphone

    4) Add each build path with the path, which is part of your 3rd party framework.

    // Add options to resolve framework paths.
OTHER_LDFLAGS=$(inherited) -F"$(EXT_MODS_BASEDIR)" -F"$(EXT_MODS_BASEDIR)/AdStir-Dependencies/3rdParty/Ad lantis" -F"$(EXT_MODS_BASEDIR)/AdStir-Dependencies/3rdParty/Ad Maker" -F"$(EXT_MODS_BASEDIR)/AdStir-Dependencies/3rdParty/adPapri" -F"$(EXT_MODS_BASEDIR)/AdStir-Dependencies/3rdParty/AMoAd"  ...... (and so on)

    5) The result will be like this: 

    
// Here is the little hack of how to resolve environmental path dependencies.
MODULEID=jp.foobarmodule
VERSION=0.1.0
EXT_MODS_BASEDIR=$(SRCROOT)/../../modules/iphone/$(MODULEID)/$(VERSION)/platform/iphone

// Add options to resolve framework paths.
OTHER_LDFLAGS=$(inherited) -F"$(EXT_MODS_BASEDIR)" -F"$(EXT_MODS_BASEDIR)/AdStir-Dependencies/3rdParty/Ad lantis" -F"$(EXT_MODS_BASEDIR)/AdStir-Dependencies/3rdParty/Ad Maker" -F"$(EXT_MODS_BASEDIR)/AdStir-Dependencies/3rdParty/adPapri" -F"$(EXT_MODS_BASEDIR)/AdStir-Dependencies/3rdParty/AMoAd"

//
//
// How to add a compiler define:
//
// OTHER_CFLAGS=$(inherited) -DFOO=1
//
//
// IMPORTANT NOTE: always use $(inherited) in your overrides
//

    This might work and I hope this helps you.

    Thank you.
