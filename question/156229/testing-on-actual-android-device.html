    There are a few things that could be going wrong:

    • You could be using a charge-only cable (not all USB cables have the right wiring, e.g. my kids' feature-phone USB cables won't enable a USB connection with my Nexus)
    • On Windows or Linux, you need to enable drivers which are provided by your phone maker (see here for more info)
    • You have not enabled USB debugging on the device (under developer options in Settings, see here for how to do this with a JellyBean device)
    • You have not enabled "non-Market apps" in the security page of Settings
    • With a Jelly Bean device, you have not authorized the connection to the computer. You should see a prompt on the home screen asking you to accept the laptop/desktop's connection. Make sure to tap yes.
    • The adb process is hung and needs to be restarted

    To reset adb, at a command prompt / terminal window (though obviously without the JavaScript-y comment lines):

    // to show attached devices/emulators:
adb devices  

// if it shows nothing, even though your device is attached
// confirm your device is attached with a proper cable
adb kill-server

// wait a few moments, then
adb devices
// this should restart ADB and then should show your device attached
// now you can build from within Studio

    BTW, this is all stuff outside our control. These are all Google/Android controlled tools and configuration steps.
