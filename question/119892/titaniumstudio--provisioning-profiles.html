TitaniumStudio & Provisioning Profiles

Hi,

I have been trying out TitaniumStudio, and in general I am very pleased that we are now getting a dedicated IDE.

Apart from the widely reported issue of not seeing debug messages, I have found that as soon as I add a provisioning profile to my project in order to test on a device (iPad 1 in this case), the app ceases to work in the simulator and hangs on startup screen on the device.

I can then no longer even work on the simulator.

Having completely removed and reinstalled Titanium SDKs etc. my app will start working again in the simulator. Adding the profile again has the same affect … therefore this si a repeatable issue that I am having.

I have tried using 1.6.2 and 1.7 SDKs and the same thing occurs in both.

I appreciate this is still a RC build but wonder if other have had the same issue.

Kind regards

Mark

  • When you build for a device Titanium works out what modules you need and only includes them. Whereas simulator builds include all modules.

    I've found that Ti Studio is very bad at detecting required modules - namely for the UI.
    To beat it I've had to write to write in some 'pre app' objects.

    eg: at the top of my app.js I have

    preWin = Ti.UI.createWindow({})
preView = Ti.UI.createView({})
preWin.add(preView)

preLabel = Ti.UI.createLabel({})
preView.add(preLabel)

//etc etc etc

    After building for device I find it's more efficient to delete the contents of the /build/iphone directory to force a full rebuild before trying to run in the simulator again.

    Debug Issue:
    Ti Studio is pointing to the 4.3 sdk folder
    You need to create a symbolic link from there to the 4.3.X sdk folder.
    There is a great post on this somewhere in the Q&A.

    The debug will reveal that it can't find a module if it hasn't been included with the build.
