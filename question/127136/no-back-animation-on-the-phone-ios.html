No back animation on the phone? (iOS)

I have a Tabgroup and I'm opening new windows within tabs. I get the default backbutton in the left upper corner as I should. But when I try my application on the phone I'm not getting any back animation? It just jumps back without animation even though I have animate:true when I'm opening the window and that works, the new window are animated in from the right.

I've tried SDK 1.7.2, 1.7.3 and latest 1.8.

It looks correct in the simulator, but it doesn't do any animation when I try it on the phone.

I'm opening the new window like this within the tabgroup:

            var win2 = Titanium.UI.createWindow({
                barColor:'#000000',
                tabBarHidden:false,
                  url:'newwindow.js',
                  title:"My Title"
            });

            Titanium.UI.currentTab.open(win2,{animated:true});

Why doesn't it animate the window again when I'm going back?

  • Accepted Answer

    I already entered a bug in Jira for that: http://jira.appcelerator.org/browse/TIMOB-5793. Only happens for apps built for 4.3 or less and runned on a iOS5 device.

    • Working: building for iOS5 using SDK 1.7.3 and XCode 4.2.
    • Not working: building for iOS4.3 or less using ANY SDK version and ANY XCode version.

    All the old apps built with Titanium and already in the App Store have that bug.

    Hope this help.
