Tableview: Sections vs Data array

Hello

I was wondering, is there a performance difference between using sections to insert data to a tableview, or using an arrayList?

I currently use a data array and push all rows into that array. When I'd use a section, and add all data into that section, will it be a difference? Or will it be the same just doing it by another way?

Will i gain performance when using the tableview section? Could anyone explain me the difference between the 2 methods?

I'm developing on the iPhone with the SDK 4.2 and titanium SDK 1.6.2.

Thanks!

    In fact the table creates one section( it does this by default) if you use simple data arrays.
    If you don't need the data to be presented in sections ( add headerViews or use grouped table) you don't have to use sections.

    I don;t think there is any performance difference between the two methods, maybe only more memory allocated if you create the sections by yourself, but this occurs only if you need more than one section or a custom headerView.
