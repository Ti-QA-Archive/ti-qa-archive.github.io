  • Jicks

    Thanks for starting a Q&A thread before raising your ticket, as this is a prerequisite for community member tickets. I am sorry you had not received a response, however.

    The reason why you have not been able to get to the bottom of your issue is because you have not simplified your usecase. When you complicate your code with unnecessary functions and setTimeouts, it makes it impossible to determine whether specific Titanium functionality really is not working or whether your own code is causing your issues. Also, there is no need to include comments and superfluous properties.

    Personally, I think it is a lot to expect casual Q&A helpers to wade through long, complicated, code to get to the bottom of your issues. Honestly, by simplifying your usecase, you increase the chances of receiving help by a great deal!

    Hence, the following usecase works as expected on the latest Continuous Build of both Android and iOS 1.7.X:

    Titanium.UI.setBackgroundColor('#000');

var win1 = Titanium.UI.createWindow({
    title:'Win 1',
    backgroundColor:'#fff'
});

var scrollableViews = [];

for (var i=0; i<4; i++){
  var tableData = [];
  for (var j=0; j<15; j++){
    var row = Ti.UI.createTableViewRow({
      height:60,
      title:"Test table "+ i + " ,row " + j
    });
    tableData.push(row);
  }
  var table = Ti.UI.createTableView({
    data:tableData
  });
  scrollableViews.push(table);
}

var scrollView = Ti.UI.createScrollableView({
    views: scrollableViews,
    showPagingControl: false
});

win1.add(scrollView);
win1.open();

    Hope this helps
