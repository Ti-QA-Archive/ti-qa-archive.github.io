Android: which code lines in android:back handler do THE SAME as the default back?

Hello, we are stuck on this apparently simple issue on Android:

  • app has a single Javascript file
  • app has tabs
  • when opening child windows in a tab, we use tab.open(win) and get all the nice animation effects sliding to the left etc.
  • if we let default back button handler close our child windows, they slide to the right as expected
  • but if we define an android:back handler to do some things and THEN close, we could not find a way to invoke the same behavior as above.

We tried the following, all on real devices:

  1. using tab.close(win): works on iPhone but not on Android, that refuses to accept the win parameter

  2. removing param and using tab.close(): works ok for the first win, but no further event is ever accepted by the app (not surprisingly, as we mangled the tab structures)

  3. using plain win.close(): flashes to a white screen, THEN slides back to parent win

Please help!
