Problem with splashscreen : removed before animation is finished.

Hello,

Here is my app.js :

Ti.UI.setBackgroundColor('#fff');

// VARIABLES
var dbNameActive = Ti.App.Properties.getString("dbName");
if (!dbNameActive) {
    dbNameActive = Ti.App.Properties.setString("dbName", "test1");
}

// INSTALL DATABASE
var db = Ti.Database.install('/db/test.db', dbNameActive);

var menu = Titanium.UI.createWindow({
    backgroundColor:'#fff',
    url:'views/menu.js'
});

menu.open({transition:Ti.UI.iPhone.AnimationStyle.CURL_UP});

The problem is sometimes my splashscreen image is removed before the "CURL_UP" animation is finished. So sometimes, during or before the animation, the splashscreen disappears then I got a white blank page with a topBar (the title of this topBar is the title of my first titleView in menu.js) that is animated instead the splashscreen. Sometimes it works fine, I've got no problem.

I hope you know what I mean, sorry for my bad english…

  • Instead of setting the background color on your app to white, try changing it to set the background image to the same image as your splash screen. I think what is happening is that the app bg color is getting set to white as the animation is in progress.

    Change this:

    Ti.UI.setBackgroundColor('#fff');

    To this:

    Ti.UI.setBackgroundImage('Default.png'); //or whatever your splash image is
