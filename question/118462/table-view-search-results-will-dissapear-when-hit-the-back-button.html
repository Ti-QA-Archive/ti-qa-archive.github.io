Table View Search results will dissapear when hit the Back button

Hi All,

Finally i managed to make my profile search app and everything seems to be fine except a major issue with search.

Here is my scenario:

I have 50 profile pages, each name.js files linked up to Table View.

So when i do a seach in the header, i get the following results.

Eg:
If i search for the name jhone, im getting 3 results

1) Jhone cameron
2) Jhone Ford
3) Jone Adamns

So imaging i dont know which Jhone am really looking at but the detailed information is in the profile page. That means i have to click each 3 one buy one.

My problem is, when i click the first one (Jhone cemeron in my example) and hit the back button, it will go to my 50 list. instead of the searched results.

Because of this, i have to type the word Jhone 3 times if the real Jhone am looking is in the 3d position.

So please any one who has a better solution please drop some lines.

All i need is to go to my search result page when i hit the back button form a new window (profile page)

Thanks

  • 1) save the search query as a property on the window object when the window gets a blur event; loses focus.

    2) when the window gets a focus event, apply the search query to the window, if one exists and update the table.

    3) also add an eventListener to clear the query property, this way if the user clears the search box the query property can also be cleared out.
