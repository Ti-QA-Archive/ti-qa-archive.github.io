Obaining source code from already compiled app

Quick question, I wanted to know if it were possibly for anyone to get access to my source code that I used to build my iPhone app? Meaning… If I have already distributed my app on the App Store, can someone take that program and find the JS functions I have created and are using? I would not see how this is possible from considering I compiled this into a program already, but I am not sure…

Let me explain…

I have an app that I am saving a file to the Filesystem to. The string I am inputting into this file is an ID number that is being encrypted using Salt and And Algorithm that I created. I have the salt and algorithm both inside of my app.js file. I want to make sure nobody gains access to my decryption algorithm, if they do then I need to do things differently in my app, but it will mean eliminating functionality and I would prefer not to do that.

I appreciate any insight, thanks in advance.

Rob

  • hello,

    I have not sensed the breaking of security and even explain how it's done, but it's simple to get the source code of an application made by titanium, so do not go into details.

    Just install the application on a jailbroken device, copy the application already installed to the mac/pc open in an editor, it will be possible to find for each block of base64 js with their name, and decode.

    With it will have the exact same files with the same code. Only no longer have comments in the js, which is removed in the build.

    This apparent lack of security is the downside of having the advantage of creating applications in javascript. However it is possible to increase the security by using tools to obfuscate js.
