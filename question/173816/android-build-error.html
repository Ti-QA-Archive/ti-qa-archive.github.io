    Hi

    This an unfortunate side effect of time. As the speed of change is very fast in our world - so is the need to keep up with it.

    Have you tried changing your tiapp.xml SDK value to an earlier one for your project? Make sure you perform a Project Clean after changing the value.

    This might be something that is specifically related to an SDK. Checking the SDK release notes (and those of any SDKs in between original and current) is the best starting point if the roll back on SDKs is not the answer.

    I understand there were some Android underpinning changes related to Android in Ti SDK 3.2.3 - might be related. But as they are so low level that detailed information neither be passed on or useful in assistance.

    If these things fail to set you straight start ruling things out;

    • Create a brand new project and compile - does this work?
      • Yes - nothing wrong with the install or SDK generally
      • No - install issue, try installing over the top
    • BACKUP you app
    • Reduce the app down to basic start up things - does it start up?
    • Start adding stuff back in to continually check
    • Keep in mind your removal of code may show different errors - keep those in mind as those are not the problem you are trying to detect.

    • More things to check;

      • Does you app have any modules?
      • BACKUP YOUR APP
      • If so, try to disable them all (might need to comment out code or even remove sections for this test)
      • Does it work, if so put each one back in one at a time until it stops again (with the same style of error - not one related to missing module)
      • If it fails after putting one or more back - check you have the latest versions by searching on Google.
      • Try again.

      Next steps;

      • Brand new app, create a default one using the latest SDK you have enabled and see what happens
      • If it works, not an install issue, something else.
