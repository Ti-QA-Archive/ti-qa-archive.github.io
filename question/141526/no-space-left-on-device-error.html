    Two things come into play here:

    Default memory size of the AVD itself

    In your mobile SDK, you can increase the default partition size from 128M to whatever you want by modifying this line &#39;128&#39; # in between nexusone and droid in your android/builder.py file. It appears after this line &#39;-partition-size&#39;,

    Default size of the SDCARD created by Titanium.

    In your mobile SDK, you can increase the default SDCARD size from 64M to whatever you want by modifying this line run.run([self.sdk.get_mksdcard(), &#39;64M&#39;, self.sdcard]) in your android/builder.py file

    The defaults work fine for most folks, but if you are working on a graphic heavy or with multimedia files, you can quickly run out of space. Additionally, Android AVDs can get 'stuck' and do need to be deleted from time to time, simply to refresh them.

  • I had this error in the past (it happens from time to time).

    The way I found around it was to launch the emulator, go into the Setting-> Applications menu. From there, there is a list of every application installed on the device (emulator). Simply uninstall your application and relaunch the run from Titanium Studio.

    Your application should re-install without any problems.

    Hope this helps
