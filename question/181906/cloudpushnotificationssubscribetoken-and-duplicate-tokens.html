Cloud.PushNotifications.subscribeToken and duplicate tokens

When I'm using subscribeToken to subscribe a device to a push channel, is there anyway to check if the token is already subscribed to that channel?

As is, every time I build my app and run it again, the device token gets added and not replaced/ignored. Which means when I tell my server to notify all, the devices gets one notification for each time I subscribed.

I've tried unsubscribing before subscribing but it doesn't seem to work.

  • I'm pretty sure this is the problem described in MOD-1870.

    If so, you can take some small comfort in the fact that this will only affect users who are doing a lot of installing and uninstalling. This is a common scenario for developers who are testing things and uninstalling/reinstalling a lot, but probably not as common for real-world end users.

    I would really like to see this issue fixed, but it's not as bad as some other bugs out there.
