how to create a good grid with thumbnails which is portrait/landscape-proof?

In short: I am trying to re-create the photoroll design. A grid with thumbnails, which you can view in both portrait and landscape. When you click on an image a window is launched with full view to the image. Images are remote images, coming from a JSON feed.

What I have now:

  • a tableView
  • with 3 imageViews per row

Screenshot: http://postimg.org/image/4rhshwnoj/

This works fine, but on a orientation change I need to refresh the feed if I want the images to get resized. If I don't refresh it I get the following problem: http://postimg.org/image/lukkdf4df/

Question: How can I manage to fix a 33% width for these 3 images, without reloading the feed?

What I've tried:

  • giving the imageViews a width of '33%' (doesn't work)
  • a container around the imageViews with 33% width and setting imageView width to 100% (doesn't work)

Possible solution: loop through al child objects and update width to deviceWidth / 3, but that doesn't feel right when showing a high amount of images (performance?).

I also read about using a DashboardView. Is this recommended for a grid with thumbnails?

Any ideas are welcome!
