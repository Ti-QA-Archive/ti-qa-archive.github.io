Scrollview issue Ti2: View with H/V layout and auto size

Using titanium 2 intensively last days, one of the most breaking changes in iOS I've found is the new scrollview behavior.

CASE 2 - View with vertical/horizontal layout and size = auto

var win = Ti.UI.createWindow({backgroundColor: 'white'});

var scroll = Ti.UI.createScrollView({
        top:0, left:0, right:0, bottom:0,
        contentWidth:'auto',
        contentHeight:'100%'
    });

var itemsView = Ti.UI.createView({
    layout:'horizontal',
    height:'100%',
    width: 'auto'    //Ti.UI.SIZE does not achieve the same result
});

for(var i = 0; i < 10; i++) {
    itemsView.add(Ti.UI.createLabel({
        width:80, height:80,
        textAlign:'center',
        text: 'dodoah'
    }));        
}    

scroll.add(itemsView);

win.add(scroll);
win.open();

In 1.8, itemsView used to grow while labels were added, so we had a horizontal scroll with all items. This does not work on Ti 2. Replacing ìtemsView width: &#39;auto&#39; to Ti.UI.SIZE or .FILL does not work as expected. The workaround for this is to set itemsView.width to a fixed width (in this example, 10 labels * 80 pixels = 800), but sometimes is not possible to know the width or the items.

The same for vertical scrolls.

bug or normal behavior? how to work with dynamic sizes inside scrollviews?
