    Hi

    I have been using the following syntax that was exposed a short while back that means you do not need to have a custom plist file to add fonts. The custom plist is the likely cause of the problem - so lets remove that from the equation first.

    <ios>
    <min-ios-ver>4.3</min-ios-ver>
    <plist>
        <dict>
            <key>UIAppFonts</key>
            <array>
                <string>/fonts/BebasNeue.otf</string>
                <string>/fonts/burnstown_dam.otf</string>
                <string>/fonts/SpicyRice-Regular.ttf</string>
                <string>/fonts/WEST____.TTF</string>
            </array>
        </dict>
    </plist>
</ios>

    This is added the tiapp.xml file in the root of your app folder. Place your fonts in the following device specific location;

    • {app name}/platform/iphone/fonts
    • {app name}/platform/android/fonts

    Create the folders as they are unlikely to already exist there. Then;

    • Close the Simulator
    • Use menu > Project > Clean > {app name}
    • Re-run

    Your existing fontFamily properties should be the same.
