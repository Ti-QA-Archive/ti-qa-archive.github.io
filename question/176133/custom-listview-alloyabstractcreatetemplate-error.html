Custom ListView Alloy.Abstract.createTemplate error

Hi.

I'm trying to create a custom listview component but i get the following error:

"'undefined' is not an object (evaluating 'Alloy.Abstract.createTemplates')";

Anybody got an idea of what i am doing wrong?

My customListView.js file:

exports.createCustomListView = function(args) {
    var listView = Ti.UI.createListView(args);
    return listView;
};

My alloy file:

<Alloy>
    <CustomWindow id="logbookWindow" ns="require('common/customWindow')" class="container">
        <CustomListView id="meterLogList" ns="require('common/customListView')" onItemclick="logClicked" onDelete="deleteEntry">
            <Templates>
                <ItemTemplate name="meterLogTemplate" id="meterLogTemplate">
                    <Label bindId="timeStamp" id="logTimestamp"/>

                </ItemTemplate>
                <ItemTemplate name="meterLogHeaderTemplate" id="meterLogHeaderTemplate">
                    <Label bindId="timeStampHeader" id="logTimestampHeader"/>
                    <Label bindId="valueHeader" id="logValueHeader"/>
                </ItemTemplate>
                <ItemTemplate name="addNewMeterLogItemTemlate" id="addNewMeterLogItemTemlate">
                    <ImageView id="addNewImage"/>
                    <Label bindId="addNewItemLabel" id="addNewItemLabel"/>
                </ItemTemplate>
            </Templates>
            <ListSection id="addNewMeterLogSection">
            </ListSection>
            <ListSection id="meterLogSection">
            </ListSection>
        </CustomListView>
    </CustomWindow>
</Alloy>

  • You're confusing the parsers! :-) Templates are an abstract component, hence the Alloy.Abstract part. And the parsers generally take the tag name and prepend 'create' to access the associated native API to create UI components. That's obviously failing here.

    I think you'd be more successful using Widgets or using Fokke's technique for creating custom view components using the ns attribute.
