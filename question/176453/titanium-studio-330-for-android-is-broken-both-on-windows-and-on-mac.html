Titanium Studio 3.3.0 for Android is broken both on Windows and on Mac!

Hello,
this is a cry for help with sadness and disbelief.

Sincerely… what's going on?

I was using previous versions of the app both on Windows and on the Mac and everything worked (not smoothly, but at least it worked!)… then something happened and I wasn't able to test my android apps on Windows because Titanium started to say that the Android SDK wasn't configured. I tried to reconfigure it, redownloaded it, I've followed most advices (even the craziest ones) both on StackOverflow and the Appcelerator's forums but nothing seemed to fix the problem… someone pointed to some apparently fixed Jira bugs… someone to some nodejs versions… but at the end, at least for me, no success…

At least it still worked on my Mac…

Well that was, of course, before I decided to let Titanium update itself to the latest release… then even my Mac was broken… and after 10 hours wasted trying to fix the problem I used TimeMachine to restore my files back to where they were and I was able to return to work…

Today I decided to retry on Windows, because I can't believe, that this basic platform functionality, running Android, could be broken in the latest and shinest version of Titatium Studio…. yet it's still painfully broken….

I uninstalled everything related to Titanium, downloaded and installed Studio, then configured it to use Android… everything seemed ok, it connected and downloaded the Android SDK, then it told me that it was configured….I smiled…
I restarted Titanium and it told me it wanted to update…I let it update and then everything was again broken…

Android SDK red –> ok let's configure (It says "Not yet configured") so let's click on Configure –> Ouch it starts to download Android again (didn't it do it just 10' ago?), nethermind let's wait —> Mmm… something must have gone wrong because it still doesn't know the downloaded Android SDK version —> Let's try again pressing "Settings" instead of configure –> version 4.1.2 is checked but if I press configure I'm back to the download loop…

Ok, what now? I really don't understand…

I thought that the Appcelerator solution was the only viable mobile multiplaform solution but if I'm not even able to start developing with it, what should I use it for?

I hope someone here will be able to explain to me what I've done wrong and why I'm so wrong about all this thing….

Thank you

Stefano

    • I solved!!!
      I noted that executing "ti info" it throws an Exception due to a not found path on my PC!!
      I'm working with Italian language version of Win7 and it re-defines some system path in Italian using non-working link to english default paths.
      I recreated these links and the "ti info" command worked and finally Titanium Studio configured Android correctly!
