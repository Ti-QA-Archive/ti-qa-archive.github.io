Cannot run a mobile android app on the android emulator

Hi, I'm new using the Titanium IDE, I use the Eclipse environment normally for creating Android Apps.
So after I installed the Titanium IDE (3.2.1.1391541866-7T7848-f6gZIs6MY) in a Windows 8 OS, I configured the paths for the Android SDK installed in order to run the android emulators on the Titanium.
I checked this docs for configuring the Android-SDK in Windows 8:
http://docs.appcelerator.com/titanium/latest/#!/guide/Software_Locations_and_Environment_Variables-section-29004844_SoftwareLocationsandEnvironmentVariables-WindowsSoftwareLocations
http://docs.appcelerator.com/titanium/latest/#!/guide/Installing_Platform_SDKs-section-29004842_InstallingPlatformSDKs-Androidsettings

The app I'm creating is for development. I have created many Android Emulators which work correctly in my Eclipse IDE, this is really weird.
At work I got installed Titanium IDE(same version as the one described in the question) and configured the JDK, SDK, paths, created emulators on the AVD Manager and works fine. My laptop at work is on Windows 7.

I thought it could be the SDK's - JDK system variables configured in the environment paths but I got the same configuration as on the Windows 7 machine.
The Keytool utility and Jarsigner utility are available on the SDK build tools and I have checked that both of these tools are available in the JDK. So I have set the SDK build tools to find these utilities by setting the JAVA_HOME environment variable so it references a suitable JDK.
Could be any specific issues with the OS Windows 8?

So after configured the Android SDK, JDK (32 bits) and system variables paths, I tried to run a Mobile App and I get the "Run configurations" screen where I am asked to browse a "Keystore Location".
This issue happens everytime I try to run any mobile app on the Android Emulator.
Any thoughts? Thanks.

  • Not entirely sure about Windows, but on OSX the default keystore ships in the SDK.

    So in my case, in

    $HOME/Library/Application Support/Titanium/osx/3.2.2.GA/android

    I have a dev_keystore.
    If my memory hasn't failed me, the password is tirocks and the alias is tidev.

    You could manually point to this file within your sdk install location through the run configurations, but I'm a bit more concerned about why it isn't being picked up from the SDK being used automagically.
