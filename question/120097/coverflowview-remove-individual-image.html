coverFlowView remove individual image

Hi,

How do we go about removing just one image from the coverFlowView? The coverFlowView.remove(image) does not work (why is it in the documentation then?)

I would also settle for removing the cover flow and adding a new isntance but it seems that if i do something like:

var myCoverFlowView = Ti.UI.createCoverFlowView({
    images: [1, 2, 3, 4]
});

window.add(myCoverFlowView);

window.remove(myCoverFlowView);

myCoverFlowView.images = [5, 6];

window.add(myCoverFlowView);

The images remain in the object and the two new ones only replace the first two images from the previous list?

Any ideas how I can just empty the coverFlowView and fill it up again with more or less images than were in it previously, without opening a new window?

Thanks.

    James, yo dont need to remove the coverflow view, you can simply call:

    myCoverFlowView.images = [5, 6];

myCoverFlowView.images = [];

myCoverFlowView.images = [1, 2, 3];

    removing and re-declaring is a very large waste of memory

  • Ok, I just had a brain wave….

    var myCoverFlowView = Ti.UI.createCoverFlowView({
    images: [1, 2, 3, 4]
});

window.add(myCoverFlowView);

window.remove(myCoverFlowView);

myCoverFlowView = Ti.UI.createCoverFlowView({
    images: []
});

myCoverFlowView.images = [5, 6];

window.add(myCoverFlowView);

    You have to call Ti.UI.createCoverFlowView({}); again, I assume this over writes or creates a new instance in memory, which therefore gives you a fresh (empty) array of images.

    I dont believe for a second that Titanium with handle the constant stack of coverFlowView's in memory well and with eventually crash the app.
