Prevent double-click events when using a nav group

I've got an application that uses a tab group where the first tab has a nav group.

The nav group pages just have tableviews in, and you can navigate through a tree using these tables.

However, if you tap many of these rows at once very quickly, the app gets confused and pops a new window where the title is correct, but it's contents never load. If you press back, none of the rows can be selected.

They flash blue, but the new window is never opened. I've tried to implement all kinds of locks, but nothing is ridding my application of the problem.

Has anyone else encountered a similar situation, and if so, how did you fix it? Surely this is something Appcelerator should handle itself for you?

  • Fast tapping can cause all sorts of trouble by queueing up multiple events. If the event listener doesn't limit what it will respond to then it could be creating extra windows, views, etc.

    This is the technique I've been using. The constant of 500 will reject clicks that arrive sooner than 1/2 second apart - adjust this to the control type and responsiveness you need.

    object = Ti.UI.createOBJECT();

object.clickTime = null;
object.addEventListener('click', function(e){

    // double click prevention
    var currentTime = new Date();
    if (currentTime - e.source.clickTime < 500) { return;  };
    e.source.clickTime = currentTime;

    ... event code
