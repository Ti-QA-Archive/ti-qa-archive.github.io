How to use XCode 6 shared library frameworks with Titanium Studio?

Hi there,

I'm using Titanium Studio 3.4.0 and XCode 6, targetting iOS 7 / 8 devices. Using this combination, I develop a native iOS module to be used by another team for a third-party Titanium app. Everything is working as expected now and I can even include linker flags for framework dependencies (like -framework AVFoundation) following the docs:

Titanium 3.0 - iOS module development guide - adding special compiler directives

This way, I can create a *.ZIP file using "python ./build.py" that includes my own module code as static library. If I want to use third-party static libraries / frameworks (like opencv2.framework), I can distribute that separately to the app developers and include the relevant linker flags into the module.xcconfig. All fine.

But now I need to use an XCode 6 shared library framework from the iOS module. Using the method specified above, I can only link to static library frameworks. A shared library framework (as available since XCode 6 only) has to be copied and code-signed during a "Copy Files"->"Frameworks" build phase of the app project. How can I instruct Titanium studio to do that? Also, can I include the shared library somewhere (similar to the "assets" directory) in my module *.ZIP?

Thanks in advance,

Mark
