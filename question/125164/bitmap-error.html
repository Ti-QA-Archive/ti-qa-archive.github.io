  • Hey, sometimes this happens with referencing any asset on Android, versus iOS, so I figured I provide a quick breakdown.

    Whenever loading in assets, ensure your path is prefixed with a forward slash; iOS is slack on this, where it will still link with the asset just fine if no forward slash is provided. For example:

    //This will work on iOS, but fail on Android
Ti.UI.createImageView({
  image: 'images/button.png'
});

//As opposed to this, which will work on both iOS and Android
Ti.UI.createImageView({
  image: '/images/button.png'
});
