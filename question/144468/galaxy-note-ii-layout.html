Galaxy Note II Layout

Hi all,

Has anyone tips of implementing a layout for the Galaxy Note II device?

displayCaps.platformWidth is reported as 360 - which means layouts based on 320 will end up with 40dp of space to the right.

Is the only option to use a computed width/height figure depending on the platformWidth?

Thanks,

Si.

  • Don't base your design on fixed sizes. There are so many android devices that you simply cannot make all of them happy.
    Try to go with percents where you can, and where not, calculate based on the platformWidth/Height

    • Thanks for the reply Dan, I see this answer a lot, but to be quite honest I haven't found any rational/practical approach to working within this, given inevitable exceptions.

      Sometimes interface elements can't be based on entirely fluid principals. For example, in one case we have three buttons which span the width of the screen, 33% width each. These can be easily scaled horizontally, however it is important these buttons maintain the correct aspect ratio, in this case, having to calculate the height, based on a multiplier which is derived from the platformWidth is just so clunky. In order to calculate the width/height of these elements we would have:

      element = Ti.UI.createButton({
  ..
  height: 50+(50*(Ti.Platform.displayCaps.platformWidth-320)/320)),
  width: '33%'
});

      (This is before we add a conditional for platformWidths < 320)

      So, given this I'd be keen to know any superior approaches to this.
