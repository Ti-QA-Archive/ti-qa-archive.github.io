Mobile web poor performance/user experience and Safari bookmark on desktop

I have a little problem of performance/user experience in my mobile web application.

When user from iphone safari saves the page as a desktop bookmark, each time the user opens the application from the desktop, it reloads the application from the beginning.

That's because when the page is saved on desktop, what is stored is the index.html, which loads the splash screen page ever. That means that every time the user opens the application, even if you come from the background, the load is very slow.

Can i do something to add the application to the desktop without loading the splash forever?

If the user opens the app from safari, as it's stored in memory, it keeps the app last status.

  • Titanium Mobile Web requires a web server. You cannot view a Titanium Mobile Web app using the file:/// protocol. The reason the splash screen is displayed for so long is because the app has actually crashed due to a cross-origin resource sharing violation. There's nothing we can do about this, so you must use a web server.
