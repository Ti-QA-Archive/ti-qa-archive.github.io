  • There are passive sockets and active sockets. listen() is for passive sockets, they wait until receive a request for connection, working like a server. You use connect() with active sockets, to connect like a client to a server (to a socket that is already listening).

    So your socket should be active or passive, not both. Once the connection is stablished between client and server, then you use write() and read() to write or read a buffer if you think there's something to read from the other side, no matter if you created or received the connection.

    If the same app can work as client (create connections) and as a server (receive connections request) then you should use one socket for each kind of connection.
