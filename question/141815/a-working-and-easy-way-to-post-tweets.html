    • Here's my guess: It's not working because you're using the stored access token from the default credentials with your newly provided credentials. Wipe the app from the simulator or device, change the OAuth stuff to be your own, and try again. If that doesn't fix it, you probably need to double check you got the keys right, and have write access enabled,…and that my code is bug free. ;)

      Also, possibly relevant, is a recent tweet I got on this. Maybe this is what you're seeing. Hack it, and let me know: https://twitter.com/jbermudez5/status/247023546998611969

      I'll explain a bit more: each request you make is signed to 1) prove its you (with the oauth key), and 2) prove that the user granted you (with your oauth key) access to act on their behalf (with the access token). So you tested with my oauth keys first, and got an access token, which is stored so the user doesn't have to authenticate again. But when you change to use your oauth keys, Twitter sees that you are making an invalid request (the access token is specific to the oauth key), and you see the above error.
