  • Thanks for all of your help. Stephen's link is what I needed. I ended up modifying the builder.py and the emulator is now running.

    In the end I changed this:

    # start the emulator
        emulator_cmd = [
            self.sdk.get_emulator(),
            '-avd',
            avd_name,
            '-port',
            '5560',
            '-sdcard',
            self.sdcard,
            '-logcat',
            "'*:d *'",
            '-no-boot-anim',
            '-partition-size',
            '128' # in between nexusone and droid
        ]

    To this:

    # start the emulator
        emulator_cmd = [
            self.sdk.get_emulator(),
            '-avd',
            avd_name,
            '-port',
            '5560',
            '-sdcard',
            '\"' + self.sdcard + '\"',
            '-logcat',
            "\"*:d,*\"",
            '-no-boot-anim',
            '-partition-size',
            '128' # in between nexusone and droid
        ]

  • Are their spaces in the path to your android SDK? Like for example:

    C:\Program Files\path\to\android_sdk

    If so, this has caused problems for other Android developers, not just Appcelerator ones. Check out this post on StackOverflow for more details. If this is the case, try troubleshooting by moving your Android SDK to a new path that has no spaces.
