Use only HD images

Hi,

I am new to mobile and titanium development. I target both iOS and Android.
I use "dp" as the default unit.
My concern is regarding using different image sizes for the same image.

I want to aford the need to deal with several image resolutions and several folder images in Titanium for ldpi, mdpi, hdpi.
Is it possible to use only HD images (hdpi) and let Titanium resize them for lower resolution devices using ImageView ?
Is it a good solution to put hdpi images in the root ressource folder and use dp units to have the same layout for all devices ?

    Hi

    The short answer is; no.

    Whilst you can technically do this - you will be providing a poor experience to your users and memory management problems to smaller devices.

    In order to show and image or any size it needs to be rendered - so if you are showing smaller versions of larger images the larger image still needs to be in memory - so smaller devices with lower amount of memory will run out much much sooner as they have a much high burden to simply display a single image.

    Making your app look and act the same way across iOS and Android is also not a good plan as the users of those devices have expectations created by the operating system they are using in how things should be done. Tabs, navigations, how windows are shown and back buttons are all examples of things that appear they same but are very different in operation and experience.

    Using dp (which is the best plan) will give you a consistent method to layout out your apps but it will NOT make the apps have the same aspect ratio or even sizes - just the same measuring method. For example the iPhone 4s, 5s, 6 & 6+ have different aspect ratios from each other and varying different resolutions. The dp width of the iPhone 4s & 5s is 320, but the iPhone 6 width is 375 - you have 55 more dp units to fill - stretching is a bad idea - you should make use of the extra space available.

    Doing these thing and doing them properly does take planning - but it will make your app much better for your users and a more robust and pleasant experience for them.

    Keep in mind using Titanium with the Alloy framework is now the preferred method, so the 3rd link updates the references in the first 2. Also the iPhone 6 & 6+ have been released since so update yourself on the revised resolutions and that there is now a retina x3 requirement as well.
