  • Hi

    I am guessing you have not actually tested this yourself as this is very basic JavaScript 101.

    var one = 'Malcolm';
var two = 'Hollingsworth';
var three = 'Malcolm';

if (one == two) {
    // nope
}
if (one == three) {
    // yes
}
if (two == three) {
    // nope
}

    However I always use the === version rather than ==, basically because;

    ==   says does the left side equal the right side
===  says does the left side equal the right side and BOTH sides must be the same type

    So when using numbers;

    if (1 == 1) {
    // yes
}
if (1 == '1') {
    // yes
}
if (1 === 1) {
    // yes
}
if (1 === '1') {
    // no
}

    The last example makes sure that when you compare things that a number equals another number and both are actual numbers - this makes your code safer from unexpected bugs in the future.

    So in your example;

    var input = textfield.value; //A string data
var data = responseObject[0].value; // A string data

// if input = "something"
// and if data = "something"

if (input === data){
    // will always work
}

    Really not sure where you read or heard about .equals() but this is not part of JavaScript itself, not a standard method. So it would not be supported Titanium either. It is often created as a prototype - but given your knowledge level on == that is for a lesson in the future.

    So in short;

    • == does compare
    • === is a safer compare
    • .equals() is not even close to being relevant
