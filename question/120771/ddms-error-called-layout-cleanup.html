  • Hi,

    this issue can also be seen in Android Emulator console. The full error message is simply: E/TiBaseActivity( 1019): (main) [298,23316] Layout cleanup.

    In my case this always happens when closing the app (e.g. in a "heavyweight" window with exitOnClose: true). There are to other effects related to this:

    1. The app does not really exit. The process is always in the background.
    2. When you restart the app there is more memory allocated than on initial start up. Allocated memory grows on every restart of the app and will never drop back to inital value (you can see this in ddms).

    For me it seems, that the "Layout cleanup" error will cause this memory leak.

    Any help??

  • Annkiit, Tobias,

    Recommend not to use heavyweight windows unless absolutely needed. I was using the same method ie. 1 heavyweight window for each screen of the application, the memory keeps on rising (probably due to the error posted but not sure). Switched all the code to 1 heavyweight window and a view for each screen and no more memory leaks.
