Can I use Titanium to create desktop apps in addition to mobile apps?

As I understand it, originally Titanium could build apps that ran on Windows, Mac OSX, iPhone and iPad, and Androids. Blackberries seem to have been added to the list, but I don't see mention of using Titanium to build desktop apps any more. I would really like to use one platform to cover both desktop and mobile. Does Titanium still do that?

    Unfortunately, Appcelerator has discontinued work and support on Titanium Desktop:

    Appcelerator Blog: Future of Titanium Desktop

    The project will be made open source for the community to develop. However, as a few people state in the comments of that blog post, if the community doesn't work on it, it'll just die. If I were to be asked, I'd have to say Desktop isn't dead, but it's pretty close. :/

    • Hi Tom. The Desktop Project is not dead and we are working toward a release candidate that we may be in a position to release a candidate for evaluation over the next week. We currently have builds on all platforms we support and have been working through tests to determine their quality. We have some testing and rebranding work left before we release.

      I had recently taken over as the Project Lead to see this occur. As part of this I prepared a budget for submission to Appcelerator yesterday as we need a Continuous Integration system to develop a project like this destined for multiple OS's. Appcelerator had its own system in place and it surprises me that they believe we could continue without such a system.

      We needed $2104 to start this up and $797 per month to run our servers which could also potentially double for network packaging of apps. This would allow you to send requests for packaging fromthe platform you are developing on to get packages for them all.

      Appcelerator flatly denied the request in an email I received indicating our expectations for the type of support they were offering was excessive and beyond their capabilities. Kevin Whinnery advised he would be making a public statement on this on the Titanium Desktop Transition mailing list on Google groups yesterday which has not yet been seen. Needless to say this is disappointing to the small group of dedicated folks that are working to see this continue. You will see our plans at:

      https://github.com/TideSDK/TideSDK/wiki

      We will assess Kevin's statement when it comes and determine next steps. The code base is large and complex and we need a combination of legal, financial and some short term technical consultation with Appcelerator as we work through this.

      — commented June 12th 2012 by fairwinds dp
