Ti.App.Properties.setString() not working for existing property

Application type: mobile,
Titanium SDK: 3.3.0GA,
Platform & version: Andriod 4.4,
Device: Galaxy Nexus,
Host Operating System: Windows,
Titanium Studio: 3.4.0.201409261227

I am trying to simply set property userID to '2' instead of '1'

tiapp.xml files has:
<property name="userID" type="string">1</property>"

app.js file:

var windowStart = require('/windowStartingTabs');
windowStart.newWindow().open();

Ti.App.Properties.setString('userID','2');
alert(Ti.App.Properties.getString('userID'));

The alert always returns '1' - the original value. I have tried placing the alert on other windows in case it was the order it ran but it always alerts '1'. If I don't have userID as a property in tiapp.xml, it properly creates a new property with value of 2, and alerts 2. It is having trouble setting the string of a property that is already created. Any ideas?

    So if you set the property id as 'usedIDA' it will work.

    You have set an application property in your tiapp file which is conflicting with this property, and the one in the tiapp file is taking precedence.

    • You shouldn't declare any application properties in your tiapp.xml it is your application configuration file.

      If you want to persistently store the user id, then using the properties mechanism is correct. If it does not exist when you read it you will get a null return, it will create it at set time.

      If you only want it for the running time of the app then use config.json file to set fixed data (not variable).

      You shouldn't really set application global variables, but you can use alloy.
