Custom UI Component

Hello there,

I have a custom component defined like this:

/app/article.js

exports.createArticleViewContent = function(title) {
    var articleTitle = Ti.UI.createLabel({
        text:title
    })
    return(articleTitle);
}

and I am calling it from my app.js like this

var ui = require('/app/article');
var content= ui.createArticleView("Hello World");

This works fine! Now, what I want is to make my call look like Ti calls, i.e.

var content= ui.createArticleView({
title:"Hello World"
});

In this case, the whole JSON value is passed into my label instead. How do I need to re-write my module in order for the title value to be passed through instead?

Hope my question makes sense… lol

George

    exports.createArticleViewContent = function(e) {
    var articleTitle = Ti.UI.createLabel({
        text:e.title
    })
    return(articleTitle);
}

exports.createArticleViewContent = function(args) {

  • You can reference the object property passed through the argument like in the other answer, or if you are going to be passing multiple properties, you can avoid having to hard wire each property by using the very handy 'extend' function in the underscorejs library. 

    var _ = require('/lib/underscore');

exports.createArticleViewContent = function(args) {
    return Ti.UI.createLabel(_.extend({
        text:'Default text'
    }, args || {}));
}

    Additionally, _.extend will let you put in default values for the properties, only overwriting them when you pass in a different property. I left 'default text' in as an example. This way, you can also set up defaults for your custom components. Color, size, or whatever will be the same across multiple uses of your component.

    More info on underscorejs here. You can also see other examples in action on the Titan Community app.
