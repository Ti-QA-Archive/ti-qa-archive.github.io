  • What do you mean by link? You should be able to add the button to the parent view and add a listener that opens the new window.

    Something like this:

    var window1 = Titanium.UI.createWindow({
url:'window1.js'
});

button.addEventListener('click',function(e){
window1.open();
});

    Is it variable scope you're having an issue with?

  • Use events for this.
    Declare in app.js an event

    Ti.App.addEventListener('open_win1', function(){
win1.open();
})

    and call this event from you other file.

    Ti.App.fireEvent('open_win1',{});

  • Ah, then your issue is variable scope, I recommend you take a look at https://github.com/kwhinnery/Tweetaniumfor how to structure your apps code.

    Using window URLs is good in some instances, but not in all.
    The reason we use them all over the place in the kitchen sink is that it creates a clean workspace for each of our examples.

    As a general rule when developing JS based apps (using titanium, node.js… or any other true application) you want to be sure to keep things in your scope. Creating a new URL is like creating many apps inside your one app, not only can doing this make development with things like what you are trying to do hard, but it also makes your app run slower.
