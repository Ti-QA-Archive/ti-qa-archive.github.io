Image upload constantly failing over 3g

Hi, I am using the code from the Snapost example to upload an image file to a server. It works over wifi, but does not post reliably over 3g. Obviously, this is a user experience show-stopper. Are there any techniques that have worked for anyone experiencing similar issues?

    When uploading over 3g it is obviously a lot slower, most servers default settings will timeout before the image can upload. You might want to compress the image before upload too as images can be rather large.

    An example on how to do this would be to create ImageView and set it's width and height in order to resize your image according to your desired dimension. This will compress the image size.

    example: 

    var _img = Ti.UI.createImageView({ image: 'image_your_sending.png', top: 5, left: 5, width: 100, height: 100 });
var url = '<your server url here>'; 
var xhr = Ti.Network.createHTTPClient(); 
xhr.open(url); 
xhr.send(_img.toImage());
