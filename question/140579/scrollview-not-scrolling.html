    Hi Matt

    Adding a tableview directly to the window, rather than in a scrollview is the easiest (as you have been shown).

    By doing this, it should only use the space available to its parent as a sizer.

    I have created a very basic tableview example for you to try, add this to a window directly. win1 in this case.

    var data = [
    { title: 'One' },
    { title: 'Two' },
    { title: 'Three' },
    { title: 'Four' },
    { title: 'Five' },
    { title: 'Six' },
    { title: 'Seven' },
    { title: 'Eight' },
    { title: 'Nine' },
    { title: 'Ten' },
    { title: 'Eleven' },
    { title: 'Twelve' },
    { title: 'Thirteen' },
    { title: 'Fourteen' },
    { title: 'Fifteen' },
    { title: 'Sixteen' },
    { title: 'Seventeen' },
    { title: 'Eighteen' },
    { title: 'Nineteen' },
    { title: 'Twenty' }
];
var tbl = Ti.UI.createTableView({
    height: '90%',
    data: data,
    width: '90%'
});
win1.add(tbl);

    I would personally not use 90% when adding views, Android apps usually go to the edges, so the table would change to;

    var tbl = Ti.UI.createTableView({
    height: Ti.UI.FILL,
    data: data,
    width: Ti.UI.FILL
});

    Does this allow you to view these your rows when you scroll them interactively?
