javascript parasitic inheritance

Hi,

New to titanium, I am learning from all resources into titanium web site. Especially forging titanium video series and wiki documentation.

I found contradictory informations about how to extends titanium proxy objects (those returned by Ti.UI.createXx methods).
In this page Tony Lukasavage says that we should use parasitic inheritance to extends proxy object, while in this page, he says : "it can result in unpredictable behavior in Titanium Mobile". Those two wiki page were created the same day :~)

What are the right paradigm, and why ?

Thanks a lots

  • In JS environments there are two types of objects: native and host. Native objects are defined in ECMAScript specification, while host objects are provided by host environment (Titanium in this case). Host objects, by specification, can behave in any manner, while native objects have their functionality defined in specification. Because of that, usage of standard JS tools for manipulation of host objects can result in unpredicted way (as many of us have seen in DOM environment :) ). That behavior is valid (like it or not) because it's not against specification.

    Because of that, better approach is to write and use wrappers around host objects that provide controlled way of accessing host object.
