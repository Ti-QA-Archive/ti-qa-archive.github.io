Quit an app when the android:back button has been intercepted ANDROID

I got this issue where i have 2 views in 1 tab, and both the views behave like they were a window. I got it all figured out, but for 1 thing. in view 1 in can press a button and open a "window" (its actually a view) called view 2. When im in view 2 and i press the android:back button i will simply hide view 2 and show view 1.

But when i am already in view 1, i want to close the app when the android:back button is pressed. I cant let the android:back button behave normally since i already intercepted it, since it arent actual windows but just some views to create the same behaviour. In my event i have the following code:

win.addEventListener('android:back', function(e){
    if(currentWindow == "shoppicker")
    {
        win.remove(bodyShoppicker);
        win.add(bodySettings);
        currentWindow = "settings";
    }
    else
    {
        Ti.UI.currentWindow.close();
    }
});

The thing is, Ti.UI.currentWindow.close(); (or win.close(); for that matter) wont exit the application like i want it to.

If there really aint a solution for this, all i can do is remove the eventlistener from the android:back event so it will behave normally when it should, and add the eventlistener to the android:back event when i want to return to the "previous window", but i was hoping there was a method to force close the app.

thanks in advance.
