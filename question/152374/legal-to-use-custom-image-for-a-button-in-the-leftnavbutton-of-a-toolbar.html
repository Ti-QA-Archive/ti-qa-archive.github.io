Legal to Use Custom Image for a Button in the LeftNavButton of a ToolBar?

Hi all-

I have the below:

<NavigationGroup>

  <Window title='Test'>

    <LeftNavButton>
      <Button id='myButton' />
    </LeftNavButton>

  </Window>

</NavigationGroup>

Within my Alloy .tss file, I am attempting to set my button to use an image, rather than the default button. I have a 40x40 as well as a 20x20 image file (with the Retina using the @2 postfix naming convention).

In my tss file, I define the style as such:

'#myButton': {
  backgroundImage:  '/images/buttons/power.png'
}

This image is a typical power/logout image (a circle with a bar in the upper section, intersecting the top). What I am finding is that the image is found, and placed on the button, but the button still contains an assemblance of a border, and completely distorts the image, almost making the circular aspect of the image into a rectangle. I have tried to set the width/height, but to no avail (and with the Retina/non-Retina version, I wasn't sure if this would be valid). I have also attempted to set the style to Ti.UI.iPhone.SystemButtonStyle.PLAIN, again, to no success.

In the GIT examples, I have seen no samples in Alloy that have used buttons in the Toolbars with images, minus the ToDo application…but this application creates a custom Toolbar, and does not use the Appcelerator Toolbar with the LeftNavButton control. I was not sure if this is possibly something I am missing in the styles, or perhaps something in the was I am attempting to use the button to customize it in the LeftNavButton in a true and not custom ToolBar?

Thanks for any help and hope all is well!

Chris

  • Accepted Answer

    Try using a Label rather than Button within the LeftNavButton. In my quick test, it shows up without a border. You can use your graphic as a background on that label. But better yet, use a custom font like Icomoon (http://icomoon.io/app/), which has a power symbol as one of the glyphs. That way, it can scale cleanly for various sizes and can be easily colored.
