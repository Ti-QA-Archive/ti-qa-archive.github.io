Switch not working on camera overlay

I am trying to add a switch to a camera overlay on iphone. The switch is displayed, but I can't change it. I also got a button on thet overlay that works perfect. Is this a bug in appcelerator?

If I apply the overlay to the current window and do not use showCamera the switch works fine.

// The switch
var switchLiveView = Titanium.UI.createSwitch({
    value:false,
    top:12,
    right:12
});

// Adding it to an optionsView which is added to the overlay. If I add the switch directly to the overlay it doesnt work either. Tested that allready.
optionsView.add(switchLiveView);
overlay.add(optionsView);

// The showCamera
Titanium.Media.showCamera({    
    error:function(error)
    {
        var a = Titanium.UI.createAlertDialog({title:NoCameraHeadlineTextArray[lang]});
        if (error.code == Titanium.Media.NO_CAMERA)
        {
            a.setMessage(NoCameraTextArray[lang]);
        }
        else
        {
            a.setMessage('Unexpected error: ' + error.code);
        }
        a.show();
        win.close();
    },
    overlay:overlay,
    showControls:false
});

  • I set the touchEnabled property for the switch to 'true'.
    Also I implemented an alert if switch is changed and an additional button that changes the switch.

    // Button event lsitener that changes the switch
buttonChange.addEventListener('click',function()
{
    if (switchLiveView.value === false)
    {
        switchLiveView.value = true;
    }
    else
    {
        switchLiveView.value = false;
    }
})

    The switch itself can not be touched on iOS 5 beta 6 like before.
    But the additional button I implemented changes the switch. The alert also only fires when the button is changing the switch. If switch is being pressed directly the alert isn't firing.
