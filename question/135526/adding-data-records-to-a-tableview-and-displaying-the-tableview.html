Adding data records to a TableView and displaying the TableView

For my first app creation, I am following along from this example:

http://mobile.tutsplus.com/tutorials/appcelerator/titanium-mobile-database-driven-tables-with-sqlite/

It's exactly what I need, but I am not seeing any results. Ie: the page shows an empty TableView.
No errors, just an empty TableView. I have a pre-populated database with a couple of rows in it. Here's my code:

// create var for the currentWindow
var currentWin = Ti.UI.currentWindow;

// set the data from the database to the array  
function setData() {  
    // create database
    var db = Titanium.Database.install('/data/GTBdb', 'GTBdb');
    var rows = db.execute('SELECT * FROM Groups'); 
    var dataArray = [];
    while (rows.isValidRow())  
    {  
        dataArray.push({title:'' + rows.fieldByName('GroupName') + ''});  
        rows.next();  
    };  
    tableview.setData(dataArray);  
    rows.close();
    db.close();
};  

// create table view  
var tableview = Ti.UI.createTableView({  
});  

// add the tableView to the current window  
currentWin.add(tableview);  

// call the setData function to attach the database results to the array  
setData();

Again, no errors, but also no data is displayed. I double-checked in SQLLite Manager and the query does return data.

Guess: Could it be that the first time I compiled this app that the database was empty, and I added rows after the fact, and now it's not getting the updated database with data? If so, how would I force a re-fresh?

Thanks,

Casey

  • You can either remove the db before using your install line OR you can go to the installed version of the DB in finder and delete it, forcing the new db to install again.

    I am going from memory, but I believe the location in finder would be
    USER>Library>Application Support>iPhone Simulator>VERSION>YOURAPP>Private Documents
