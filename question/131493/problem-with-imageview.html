Problem with ImageView

Hello,

I'm developing a mobile application with appcelerator, and I want to add a ImageView to a view. I'm creating the ImageView so:

var logo = Titanium.UI.createImageView({
    image: 'logo.png',
    width: 253,
    height: 96,
    left: _logoMarginLeft,
    top: 0
});
view.add(logo);

I'm not sure what's the matter, but the image is not shown. I know that I'm creating the ImageView well.

I've read in the forum that there is some kind of problem with 'fastdev', and that if the 'fastdev' is turn off, the 'ImageView' is shown. So I turn off the fastdev, and it works, the ImageView is shown.

<property name="ti.android.fastdev" type="bool">false</property>

But I think that is not a good solution to turn off the 'fastdev'. Is there any other solution?

Could anybody help me?

Thanks in advance.

Application type: mobile; 
Titanium SDK: 1.8.0.1; 
Platform & version: Android 2.2; 
Device: Android emulator; 
Host Operating System: Windows XP

  • Hello,
    I tested this code:

    
var win = Ti.UI.createWindow();

var logo = Titanium.UI.createImageView({
    image: 'logo.png',
    top: 0
});
win.add(logo);

win.open();

    in Ti 1.8.1, works like a charm without having to disable fastdev. Tested on 2.2 emulator. If you added your images after building, maybe you will need to clean your project and try again.

    Let me know if this works for you. BTW, my image is in Resources/

    Best,

    Mauro
