Calling a service from module to application in alloy

Hi, I have a android push module which calls the service using super(name_of_the_file). This works well in classic project as it is able to find a file under Resources folder. But in alloy project there is no Resources folder so it is giving error as no file found. Any thoughts of how to achieve this?

  • On receiving android push message following code is run - 

    protected void onMessage(Context context, Intent messageIntent)
  {
TiApplication tiapp = TiApplication.getInstance();

    Intent intent = new Intent(tiapp, GcmjsService.class);
    for (String key : messageIntent.getExtras().keySet())
    {
      String eventKey = key.startsWith("data.") ? key.substring(5) : key;
      intent.putExtra(eventKey, messageIntent.getExtras().getString(key));
    }
    tiapp.startService(intent);
}

    And GcmjsService.js class is -

    public final class GcmjsService
  extends TiJSService
{
  public GcmjsService()
  {
    super("gcm.js");
  }
}

    Now in classic project we can add gcm.js file under Resources folder and aplication specific task can be done. But in alloy there is no Resources folder so I am not sure how to call from module to application file.
