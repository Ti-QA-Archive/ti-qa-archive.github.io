Multiple Options Dialogs Open on Button Click - iPad

I have a button in my Top Navbar. On click it opens the dialog directly below with the ^ pointing right below the button. Works awesome. Except…

When I click the button again it opens dialog box right over it. It seems that the option dialog screen overlay that disables buttons doesn't cover the top toolbar (which I guess makes sense).

There is no close method for the optionDialog so I can't call that programatically by toggling the function on the button. And there's no listener for when the optiondialog closes so I can't disable the button till the optiondialog closes.

Anyone know if there's a solution here.

  • For those of you who don't mind adding a quick method to the objective-c class within the titanium SDK, this is actually pretty easy to fix.

    inside the sdk folder, open the file '/iphone/Classes/TiUIOptionDialogueProxy.m' (I'm using sdk 1.6.3 so the location on my mac is '<username>/Library/Application Support/Titanium/mobilesdk/1.6.3).

    Once in there, add a 'dismiss' method, it doesn't matter where as long as it's loose and not inside any other brackets!! (I put mine underneath the closing bracket of the -(void)show:(id)args method).

    Here's the code for the new method:

    -(void)dismiss:(id)args{
    ENSURE_SINGLE_ARG_OR_NIL(args,NSDictionary);
    ENSURE_UI_THREAD(dismiss,args);

    BOOL disanimated = [TiUtils boolValue:@"animated" properties:args def:YES];

    if (actionSheet && [TiUtils isIPad]){
        [actionSheet dismissWithClickedButtonIndex:actionSheet.cancelButtonIndex animated:disanimated];
        actionSheet = nil;
    }
}

    Save the file.

    Inside titanium, you'll now find that any options dialogue can be dismissed by calling options.dismiss(); For example, if you want to stop the multiple dialogues bug, you can now dismiss each dialogue before you show it:

    var dialogue = Titanium.UI.createOptionDialog({
    options:['one','two','cancel'],
    destructive:2,
    cancel:2    
});

var openDialogueButton = Titanium.UI.createButton({
    systemButton:Titanium.UI.iPhone.SystemButton.ACTION
});

Ti.UI.currentWindow.setRightNavButton(openDialogueButton);

openDialogueButton.addEventListener('click', function(e){
    dialogue.dismiss({animated: false});
    dialogue.show({animated: true});
});

    You'll probably also find you need to dismiss on window close as well to stop the dialogue hanging there when you don't want it. Don't worry about checking whether the dialogue is actually on screen before dismissing, the objective-c function does that for you!

    Hope this helps.
