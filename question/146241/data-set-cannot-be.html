Hello all

Environment : Titanium SDK 3.0 GA and Android
I work on Mac OSX Lion

Here's the code I use :

var win = Titanium.UI.currentWindow;

var data010 = [
{ordre:"1", number:"010", title:"Helen", corresp:"0A00", icF:"deb-ligne-1", icR:"fin-ligne-1"},
{ordre:"2", number:"010", title:"LL Cool J ", corresp:"00", icF:"1", icR:"1"},

];

var data = data010; // this is necessary for me in my app

// table 
var tableviewtemp = Titanium.UI.createTableView({
    data:data
});

alert(tableviewtemp.data[0].rows[1].corresp);

win.add(tableviewtemp);

I set the data in a tableview and I want to get the custom value called "corresp".

The alert shows nothing but if I put this code

alert(tableviewtemp.data[0].rows[1].title);

What did I do wrong ?

  • Hi

    When you access data out of the data parameter it only contains the data used to display the actual table.

    You can access it from a listener, so you can react to it.

    tableviewtemp.addEventListener('click', function (e) {
    alert(e.rowData.corresp);
    alert(e.rowData.title);
});

  • Hi Pat, to answer your question, any property which is directly related to table view row object defined in Titanium can be accessed using row and any other custom property which we define can be accessed using rowData.

    Thats why you are able to access title but not other custom properties.

    another thing is behavior of row and rowData is different in iOS and Android, so better to use rowData for common code.
