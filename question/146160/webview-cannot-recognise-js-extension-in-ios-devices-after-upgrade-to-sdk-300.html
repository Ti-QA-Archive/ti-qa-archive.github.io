WebView cannot recognise .js extension in iOS devices after upgrade to SDK 3.0.0

Hi,

I had compiled my app with latest SDK 3.0.0 and executed it in iOS Simulator. This work fine, as expected. However, the moment I deployed the app into iPad with iOS6, the app is no longer loaded. Basically the app is using Titanium.UI.Webview to load local HTML and javascript files. Strangely, the app loaded successfully in android devices

After some googling, I found that this could be due to .js extension optimisation by XCODE. I renamed all the javascript files' extension to .ajs, recompiled the app, and executed it again in iOS devices. The app is loaded and run as per normal.

This is not the case in SDK 2.1.4 where my app is running well in iOS devices while still using javascript files with .js extension

I presume this is a bug?

Application type: mobile
Titanium SDK: 3.0.0.GA
Platform & version: iOS 6.0.1
Device: iPad 3rd Generation
Host Operating System: OSX 10.8.2
Titanium Studio: build: 3.0.1.201212181159
