C2DM deprecated - New Android Push solutions?

Hey there,

Today i noticed, that the C2DM-service for Android, which is used for push-notifications with Urban Airship, is deprecated by Google since last week.

Are there any solutions to implement Android push-notifications with UA, yet?

Thank you!

  • Doesn't look like UA has started to work with GCM, yet. They haven't even released a guess at a timeline. However, if you have an app currently working with UA doing C2DM, it will continue to work.

    In short, you can't do a new app with UA that does android push-notifications. However, any existing apps will continue to work. There's no set date when C2DM will stop working, though. My guess is, from looking at the steps to migrate, that UA will be ready to switch to GCM before C2DM stops working.

    • If you're creating a new app, there's a couple of options.

      1) Wait until UA adds support for GCM. The problem with this is that you'll be waiting an unknown amount of time for a 3rd party to complete something. That's the problem you get when you rely on a 3rd party, though.

      2) Use Appcelerator's cloud services to do push notifications. This is great if you also want to do things like have them manage your users, etc. I'm not sure what they're using for their Android backend, or whether it works or not, so that would require some research, first.

      3) If you have access to a server, you could do the server component yourself. I'm using PushSharp for my server part. I also plan on using Kuraturpa as my Titanium component, and then converting that to GCM. Not sure how that's going to go, though.
