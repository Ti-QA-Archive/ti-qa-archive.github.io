  • Well, your title is kind of misleading…

    For starters, there is no JSON on the page you mention of.

    Secondly, if you ever want to parse an html page, I would recommend against doing it on the mobile device itself. The size of the page could become an issue if it ever gets bigger and trying to strip all the unnecessary stuff (script tags, css, forms, etc) would be too complicated using JavaScript alone.

    The most viable option in these cases it to simply make a screen scraping program (using any language you choose) that does all the heavy-lifting for you.

    The thing could go like so:

    • The screen scraping program retrieves the hml code from the url.
    • It parses the parts that are relevant to your application.
    • It then provides the extracted information in JSON format through an url (your server, your code, your data)
    • Your mobile application retrieves the JSON string from your server (whic is much lighter that the html page itself) and uses it.

    Hope this helps.
