  • iOS SDK 6.0 only supports armv7 (iphone 3gs, 4, 4s and iPad) and armv7s (iphone 5). Since iOS SDK 6.0 does NOT support armv6, then it's impossible for the app to target iOS 4.2 and older, thus we set the minimum version to 4.3.

    If you want to target 4.0 through 4.2, then you need to build using iOS SDK 5.1.

    This minimum iOS version is used by the AppStore. You can set it to whatever you want in the tiapp.xml via the <ios><min-ios-ver>4.3</min-ios-ver></ios> section. If you put in crazy values, they'll be overwritten back to 4.0 or 4.3 depending on the iOS SDK version used to compile.

    • I just tested this by setting in my tiapp.xml:

      <ios>
    <min-ios-ver>5.0</min-ios-ver>
</ios>

      Towards the top of the output it says:

      [INFO]  Minimum iOS version: 5.0

      Based on the iOS version and the min iOS version, we are able to figure out what architecture we should compile for. In this case, I'm building for iOS 6.1 and a min ver of 5.0 and it says the following:

      [DEBUG] Building for the following architectures: armv7 armv7s i386

      This list of architectures is directly passed into Xcode's xcodebuild command via the VALID_ARCHS argument.

      The last thing we use the min iOS version for is to set xcodebuild's IPHONEOS_DEPLOYMENT_TARGET value.

      I ran an adhoc distribution build which generated an ipa file. After renaming my testapp.ipa to testapp.zip, then unzipping the file, I was able to find the binary-formatted Info.plist in Xcode and the MinimumOSVersion is properly set to 5.0:

      So, as you can see, everything should work as expected.
