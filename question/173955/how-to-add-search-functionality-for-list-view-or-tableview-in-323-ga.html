  • Hi

    There are two ways to add search to either the TableView or ListView - both act in basically the same way.

    • Filter Mode

      If you add a SearchBar directly to the TableView or ListView then the searchbar will filter the items/rows automatically. If you have 100 items then entering a a value into the SearchBar will reduce those 100 items to those that match the search value to the items in the TableView or ListView.

      Filters are used when your list already has all the items it will ever have for this session and the user needs to filter that list down quickly. Removing the value from the SearchBar shows all previous results.

    • Search Mode

      If you add a SearchBar but do NOT set it as a property of the TableView or the ListView then you add EventListeners" to the SearchBar to respond to changes in the value and then update the contents of the TableView or ListView yourself.

      Searching is used when you need to call either an external API or an internal DB to then show just those results. Changing the value would make additional calls to the API or DB.

    • Filters

      Titanium

      var data [
    { title: 'One' },
    { title: 'Two' },
    { title: 'Three' },
    { title: 'Four' },
    { title: 'Five' }
];
var search = Ti.UI.createSearchBar({});
var tbl = Ti.UI.createTableView({
    data: data,
    height: Ti.UI.FILL,
    search: search, // used for filters
    width: Ti.UI.FILL
});
// you do NOT add search to the parent
//assumes win is the parent
win.add(tbl);

      Titanium with Alloy FrameWork

      // View
<TableView id="tbl">
    <SearchBar />
    <TableViewRow title="One" />
    <TableViewRow title="Two" />
    <TableViewRow title="Three" />
    <TableViewRow title="Four" />
    <TableViewRow title="Five" />
</TableView>

    • Each ListItem has a property property you set that allows the searchView to filter the ListView;

      • searchableText

      Think of this as another title property but it can contain more than a single value if you choose to join them together. You set this for every single item as you go;

      Non-Alloy

      // snippet from help documentation with property added;
var fruitSection = Ti.UI.createListSection({ headerTitle: 'Fruits'});
var fruitDataSet = [
    {properties: { title: 'Apple', searchableText: 'Apple' }},
    {properties: { title: 'Banana', searchableText: 'Banana Fruit' }}
];
// the second entry would match f and B even though F does not exist inside Banana

      Alloy

      // snippet from help documentation with property added;
<ListItem title="Apple" searchableText="Apple"/>
<ListItem title="Banana" searchableText="Banana Fruit"/>
// the second entry would match f and B even though F does not exist inside Banana

      Alloy bounds items;

      // simplified example
<ListItem  searchableText="Apple" info:text="Apple" es_info:text="Manzana" />
<ListItem searchableText="Banana Fruit" info:text="Banana" es_info:text="Banana"  />
