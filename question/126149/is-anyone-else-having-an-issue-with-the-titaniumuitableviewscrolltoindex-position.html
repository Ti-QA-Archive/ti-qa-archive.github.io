Is anyone else having an issue with the Titanium.UI.TableView.scrollToIndex 'position'?

When I have a tableview in a window I'm opening and I want to pre-select a row, I have it set fine and I want to scroll the selected row in to the middle of the table view.

This all used to work fine, a long time ago, where you can set the position using : Ti.UI.iPhone.TableViewScrollPosition.TOP|MIDDLE|BOTTOM to decide where in the tableView you want the selected row to be.

However, what appears to be happening is the row is getting selected fine, but it's always scrolling to the top of the tableview BUT setting the TOP|MIDDLE|BOTTOM with respect to the ROW and not the actual tableView.

eg: If I have MIDDLE set, the row will fly to the top of the tableview and hide half of the row, since it's using the MIDDLE of the row and not showing half way up the tableview, as it should.

Anyone else have this problem or noticed this?

Could be a bug.
