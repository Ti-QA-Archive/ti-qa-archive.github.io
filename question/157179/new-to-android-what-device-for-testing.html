New to Android, What device for Testing?

To my understanding there is problems with fragmentation, to what extend is this true? When they talk about fragmentation is it because the various hardware OEMs? or the numerous Android versions out there? Can my app target all Android versions or should I focus on an specific release?.

I know my questions are a bit vague, but I would like to hear personal experiences, what Android device actually works well with Titanium for testing.

    Hello Omar,

    the reason for fragmentation is not the variety of manufactures. Because of some OEMs that don't provide updates for all of their Android devices to newer versions. So, as you can see in the Android Dashboards there are still a lot of Gingerbread devices out there. It would be nice if you target them too but IMHO it's more important to target the last version.

    When I develop Apps for Android, I test them on my Nexus devices with stock Android (latest version). I also test my Apps on my old HTC with Android 2.3.3 Gingerbread - just to be sure it runs and looks okay. I finally go through some virtual devices on the Android emulator to see my App on common screen sizes and resolutions.

    You can also see the most used screen sizes and resolutions on the Android Dashboards. That doesn't seem to be that many tablets. Anyway, the Android system is not designed to distinguish between phones and tablets, it's more flexible across all screen sizes. I mean, there is no borderline between all possible screen sizes from 2 inch to 10 inch where Android decides "this is a tablet" etc. You can read about supporting multiple screens here: http://developer.android.com/guide/practices/screens_support.html

    You don't need profiles like on iOS to install Apps, you can install them straight from Titanium Studio on your device. When you release an App on Google Play, then you have to sign it with a private key. But this is also done by TiStudio (including the creation of the key) - very easy!

    OEM Firmwares don't effect the UI of Apps. So, you don't have to worry about developing for multiple manufactures ;)
