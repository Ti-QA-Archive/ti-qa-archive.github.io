Titanium difficult to use over a network

I'm really struggling with this. I've been developing apps locally with no problems. Now I try and store all files on a network drive that gets backed up. Yet Titanium seems to hate it! It's so slow in building and testing etc, as well as sometimes not even updating files in the file tree.

Is this common? Does Titanium just not like network workspaces? The only way I'm getting around it is working locally, then at the end of the day copy & paste all files to the network. Next day, copy & paste all files from the network to local.

This seems a bit of a pain. Any reason why it's so painful working over a network?

Cheers!

  • However you proceed, you might want to create a Dropbox account and copy your code into that (assuming it's not particularly sensitive). Whatever OS you're using, it would be easy to set it up to do this automatically.

    (I don't recommend using the same directory for local work as you use for Dropbox, because the Dropbox process has the ability to delete any local files without warning. It's never done it to me, but they could introduce a bug that causes that to happen. Better safe than sorry.)

  • I would personally use a SC system (SVN, GIT) and commit my working copy back into a branch of the repo at regular intervals. The advantage is that you work entirely locally and when you commit you only transmit the "diff" of the changes so is much faster then performing all your I-O off a network share or manually backing up your entire project to one.

      — commented January 12th 2012 by Nick Milner

    • As Nick mentioned, you can also host your own SCM inside your network. Think of it like your corporate email server: Everybody's mailboxes get backed up from the email server, not from their local pc. They then pull email locally from the server.

      Any corporate environment that is doing software/web development of any kind will be using a SCM, and that may be internally or externally hosted. Externally hosted SCM, such as github or others are popular for independent developers because it reduces the work involved in managing the SCM. If you have IT staff to handle infrastructure support, it might be better to look at internal hosting.
