  • I read I can put the IPA on testflight, which should avoid all of that

    Untrue. Testflight does not magically remove provisioning profiles, signing certs and device registration from the mix. It merely makes it easier to distribute binaries and collect/manage device ids.

    In a simulator build, an ipa isn't actually created. The yourApp.app is created and placed into the simulators working structure.

    To generate an ipa from studio, you would work through the 'packaging' build flow.

    From the cli ti build -p iphone -T dist-adhoc -O &#x2F;directory&#x2F;where&#x2F;you&#x2F;want&#x2F;the&#x2F;app.ipa and on newer cli builds that will try to match your signing cert and appropriate provisioning profile with your app's id
