Persist ASP.NET Membership beyond Application session

Looking to develop a Mobile App which contains a members only area. The aspect I am not sure about is how to persist an ASP.NET Membership login beyond an Application session.

What options are they for storing membership information locally and how can I check this information for each HTTPClient request?

    This is outside the scope of Titanium, but now that you're developing for mobile, you can't think in the same terms that you did for ASP.NET, where the web application handled the authentication. In mobile apps you're calling web services, and in the modern .NET world specifically WCF services, so you want to look into WCF authentication schemes.

    That being said, once you do authenticate with the web services, let's say they give back a secure token guid. You could then store it in Titanium like so:

    Ti.App.Properties.setString("AuthToken", token);

    And then you could access it, to pass in to future web service calls:

    var token = Ti.App.Properties.getString("AuthToken");
