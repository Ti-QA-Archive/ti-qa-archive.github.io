    Hi

    Some time ago I answered a couple of questions on a similar basis they have been used as the standard for this sort of question, they are shown below.

    Before you jump into those I want to hold you back a bit. The questions were asked prior to Alloy so whilst the structures are correct and the answers are perfect ;-) I need to tweak your understanding based on what you have read.

    • Do not use Ti.include. Certainly not if you use Alloy and ideally not if you have not yet moved to Alloy (this bit for anyone else reading this later).
    • Paths to images should ALWAYS be /folder/image.png note the prefix of the / that is always required - trust me. If the image is in the root then /image.png - again always use the /.
    • The root to your images is different depending on Alloy or non-alloy;
      • Alloy; 
    {appname}/app/assets/
{appname}/app/assets/android/ // platform specific
{appname}/app/assets/iphone/ // platform specific
    • Non-Alloy
    {appname}/resources/
{appname}/resources/android/ // platform specific
{appname}/resources/iphone/ // platform specific
    • The only time you should ever ignore the leading / in code is the require method;
    var x = require('something');
var x = require('something/somethingelse'); // when in folder

    For your image example to work it must be in one of the following places;

    {appname}/app/assets/images/glic.png
{appname}/app/assets/android/images/glic.png // platform specific
{appname}/app/assets/iphone/images/glic.png // platform specific

    When using Alloy you NEVER add anything to resources or below - this is done automatically. If your app is not doing things you expect it to when showing images do the following to reset things to normal so your next compile starts fresh - may not work but at least previously tried attempts will not cloud determining the fault.

    • If you are not sure what you are doing backup your project right NOW
    • Stop the app from running; Studio > Red stop button in console window
    • Studio > Menu > Project > Clean > Select Project > Confirm
    • WARNING the following instruction must ONLY be performed on actual ALLOY projects;
      • Remove the content of resources - these are recreated every single time.

    SO those two links that give you lots of information about images - their locations and more, just remember to swap the base paths as instructed above.

    • How to achieve same layout for Android and iPhone using Titanium
    • images in Android (also iOS)

    This should point you in the right direction.
