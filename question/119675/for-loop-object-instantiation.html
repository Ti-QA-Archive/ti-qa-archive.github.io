for loop object instantiation

Hey,
I'd like to use TI.HttpClients to download several images using a for loop:

// this function is used later to handle the httpclient readystatechange
function loadImageToView(view,httpclient)
{
if(httpclient.readyState == httpclient.DONE)
{
view.image = httpclient.responseData;
}
}

// the following code is inside another function

// … some code hear for imageView instatiation etc. this code works

for(var url in urlArray)
{
var callfun =function() { loadImageToView(imageView,imageLoader); };

var imageLoader = Titanium.Network.createHTTPClient();


imageLoader.onreadystatechange = callfun;
imageLoader.open('GET', url);
imageLoader.send(null);

}

The above code adds only the image to the last image view. I think what happens is, that the function callfun is overwritten every time and the imageview changes multiple times compared to the desired multiple imageviews are changed once.

  • The httpclient is asynchronous.
    This means you will have to wait for each call to end, use the image and start the next one until you finish the array.

    Idea: why don't you simply make an imageView with the remote url, on the load event use toImage() and save this.
    ( Not tested :) , but I think less stress )
