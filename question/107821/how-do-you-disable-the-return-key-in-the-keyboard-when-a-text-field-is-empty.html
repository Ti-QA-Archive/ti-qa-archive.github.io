  • You didn't specify a platform or target SDK; generally speaking that is a good idea when asking for help.

    If using iOS and the textfield can be replaced with a searchbar object, it will behave that way automagically. Searchbars in android do not.

    For cross platform support or if you need to stay with a text field, the closest thing I can think of is to just redisplay the keyboard:

    field.addEventListener('return',function(e) {
  if ( ! field.value || ( field.value == '' )) {
    field.focus();
  }
});

    Untested and just typed into the reply, but should at least provide the general idea.

  • Sorry, I'm such a newbie, should have thought about that..I'm working on an iPad application.

    Anyhows, after a little bit more searching, i was looking for was this property "enableReturnKey:true" for textfields.

    This will disable the return key (fade it slightly in the keyboard display) and only enable it once the textfield is no longer empty. Found this information at appcelerator.lighthouseapp.com
