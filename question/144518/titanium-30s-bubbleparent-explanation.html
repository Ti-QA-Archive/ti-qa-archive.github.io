Titanium 3.0's bubbleParent explanation?

Can anyone explain how bubbleParent event works? For example I have picker control on a view and bubbleParent is set to true for my picker. Does that mean:

  1. Picker's "change" and "click" events would be "bubbled" to the view?
  2. Picker will be "transparent" for events, so when I touch the screen over the control, the view will trigger "touchstart" event? (while there is no "touchstart" event for spinner)
— asked November 8th 2012 by Artem Pervenyonok

    If you have picker added to the view, bubble parent will mean that if you click on picker, its parent view will receive click event also.
