TableView - force "touchEnabled" while "moving" is true

I have a table in an iPad app where the user needs to be able to reorder the rows. The user also needs to be able to click on an element inside of that row however when myTable.moving = true, the row element doesn't respond to click events.

Any known solutions or workarounds for this?

var myTable = Titanium.UI.createTableView();
myTable.width = 600;
myTable.height = 570;
myTable.separatorColor = "transparent";
win.add(myTable);

///////////////////////
myTable.moving = true;
myTable.touchEnabled = true;
///////////////////////

var row1 = Titanium.UI.createTableViewRow();
row1.width = 600;
row1.height = 105;
row1.backgroundColor = "transparent";                
myTable.appendRow(row1);    

var rowImage = Titanium.UI.createImageView();
rowImage.image = "images/rowBackround.jpg";
row1.add(rowImage);

rowImage.addEventListener("click", function(){
    alert("touched");
});

  • There is no way to do this from the JS side of things… you might be able to do this by changing the native code (have not looked at that) but would recommend a change to the UX of the app to not need this. As far as I am aware this is not something that even apple does in its apps for UX reasons… The issue here I would think is that the user could accidentally start dragging when they mean to click or the other way around. Typically the UI would have a clickable row that an edit button makes draggable… or would this not work for you?
