Module preventing iOS app from installing on device

Hi Everyone. I'm having a strange issue with a module I've been using for over a year with no issues. I haven't done anything with this particular app in a few months, and I believe I've updated Titanium since then, so this might be an issue with 3.2.3.

The module I'm using is PSPDFKit. It used to be on the marketplace, but I believe the developer sells it independently now. I've gone back and forth with him a few times, but we can't figure it out.

When I try to install to the device from Titanium, I get this error message:

[TRACE] :  ** BUILD SUCCEEDED **
[INFO] :   Installing app on device: iPad 3
[ERROR] :  Failed to install app on device (0xe800002d)
[INFO] :   Project built successfully in 55s 846ms

If I remove PSPDFKit.bundle from my Resources folder, it will install fine, but when I go to view a PDF in the app, it tells me I'm using bundle 3120 and I should be using 3700. This is with no bundle at all in the app. I guess the iPad still has previous data saved. But if I put in any bundle, whether it's the new one (3700) or the old one (3120), it won't install to the device.

It will run in the simulator, and if I make an IPA using my Enterprise profile, it builds and installs to my device through iTunes with no issues. That is a workaround, but I do a lot of testing and it's just easier to install it through Titanium.

Does anyone know what's going on?

iOS 7.1
Titanium 3.2.3
PSPDFKit 3.7.5 (I've also tried 3.5.0 and the same thing happens).
