File Not Found!

I have this file in this path:
Resources/suds.js

When I use include, I do:

Titanium.include('../suds.js');

// '../' because I'm including it from Resources/Controllers/, 
//so go back one folder and find file suds.js.

It does find the file suds.js in the Resources folder!
It's also in the build directory:
PROJECT/build/android/bin/assets/Resources

But when I run it on Android (Device) I get this:

Wrapped java.io.FileNotFoundException:
Resources/android_asset/Resources/suds.js (file:///android_asset/
Resources/Controllers/ConfigController.js#10)

Where #10 is the line of the 

Ti.include('../suds.js');

Any clue of what's going on?
I've been losing time with this for a while now. This doesn't make any sense.

Can anybody help, please?

  • I have not long started using titanium and have also come across this error.

    In the iphone emulator it all works but in the android emulator it doesn't.

    I have actually found that iPhone relates to Resources path so for a file in the config directory Ti.include('config/setit.js');

    But for android it is a relative path so if requiring to use setit.js from a file in the same directory the path is relative to your current location Ti.include('setit.js);

    This has meant that I have to test the type of OS before including files. It all works the same for both android and iPhone is your main file resides in the Resources directory.

    Hope I have explained this correctly and it helps
