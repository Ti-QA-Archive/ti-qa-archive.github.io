ACS photo upload on user object returns old photo

Hi there,

I've implemented the uploading of a new user photo when updating the User object using the REST api. It gets accepted and the User object is returned, however, it contains the old photo that was already present prior to updating. When I do another call to retrieve the user, I do get the new photo, also when not already processed. Along with the user_id and photo I'm also sending photo_sync_sizes[]=medium_500 to synchronously process the image.

Does anyone know what could be going on? I really want to prevent having to do another API call as the single call is slow enough of itself. Cheers!
