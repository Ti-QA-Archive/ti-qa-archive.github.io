UI problem after upgrading to 1.6

Rebuilt my iPad project using the new 1.6 SDK.

Experiencing a couple of issues:

  1. My grouped TableView is no longer the default grey background, but black.
var customers = Titanium.UI.createTableView({
        style:Titanium.UI.iPhone.TableViewStyle.GROUPED
    });

If I apply a backgroundColor to the tableview, it also applies it to the row background color.

  1. I have a grouped TableView in a Modal window. When the window is shown, the table appears in the center of the window.
var checkInTable = Ti.UI.createTableView({
        data:data,
        style: Titanium.UI.iPhone.TableViewStyle.GROUPED,
        top:0
    });

checkInWindow.add(checkInTable);

checkInWindow.open({
        modal:true,
        modalTransitionStyle: Ti.UI.iPhone.MODAL_TRANSITION_STYLE_FLIP_HORIZONTAL,
        modalStyle: Ti.UI.iPhone.MODAL_PRESENTATION_FORMSHEET
    });

Any ideas on how to fix? If I switch back to 1.5.1, the behavior is restored.

Thanks!

  • Anyone else having this issue?

    I've narrowed it down to a custom grouped tableview with a row that contains a TextField or TextArea.

    The table appears at the top of the modal window as it should. But, when you touch inside the text field, the table floats to the middle of the window.

    Not happening in 1.5.1.
