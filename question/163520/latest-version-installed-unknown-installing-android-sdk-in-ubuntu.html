Latest version installed unknown installing android sdk in ubuntu

I am trying to set up the Titanium development environment for the first time. I downloaded the most recent version (3.2.1.201402041146), installed libwebkitgtk-1.0-0, and downloaded the most recent android sdk (adt-bundle-linux-x86_64-20131030).

I ran the android program (sdk/tools/android) and installed the sdks for api versions 19, 16, and 10. Then I clicked "Install or Update Android SDK" under "Configure Native SDKs." For the android sdk, I used the sdk directory in the adt bundle (adt-bundle-linux-x86_64-20131030/sdk).

When click "Configure" it goes through an install process and ends with a dialog saying "Latest Version Installed: Unknown" and "Installed Location" pointing to the sdk directory I specified. When I try to create a new app it says the android sdk isn't installed.

What should I be doing differently?

  • In ubuntu the command node it's called nodejs to override collision with node package. Copy, not link, &#x2F;usr&#x2F;bin&#x2F;nodejs to &#x2F;usr&#x2F;bin&#x2F;node and try to set the android sdk again. With this method I have fixed my installation.

  • Had the same Problem with titanium 3.4 and windows 7

    I've tried so many things: - reinstall titanium - reinstall android sdks - reinstall nodejs - setting environment paths - reinstall java - setting paths with cli and ti command

    what finally helped was removing nodejs (i had the newest version 10.32) completely (also from path) and install Titanium. Titanium then installs node version 10.13 and after that I install the android sdk and they were correctly detected by titanium.
