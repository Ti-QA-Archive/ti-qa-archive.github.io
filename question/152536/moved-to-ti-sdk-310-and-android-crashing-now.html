Moved to Ti SDK 3.1.0 and android crashing now

I have an app that was built using SDK 2.1.4. For the past week, I've been getting my iOS version of the app up to date with 3.1.0 SDK. I am now ready to do this for android. I compile to the device and I get this error http:&#x2F;&#x2F;pastie.org&#x2F;7921720

Pretty sure the Invalid Int is the culprit here. The only problem is these error outputs don't tell you where this error is!

The order of events are compile, splash screen launches on device and then crashes after a few seconds never loading the first window.

  • I tracked down the error. It was this line in my ActivtyIndicator style:(Ti.Platform.osname != &#39;android&#39;) ? Ti.UI.iPhone.ActivityIndicatorStyle.DARK : &#39;&#39;

    What worked in SDK 2.1.4 no longer works in 3.1.0 (for good reason I suppose). Fixed it by doing style:(Ti.Platform.osname != &#39;android&#39;) ? Ti.UI.iPhone.ActivityIndicatorStyle.DARK : Titanium.UI.ActivityIndicatorStyle.PLAIN

    It'd be nice if the errors told you where they stemmed from like EVERY OTHER DEBUG TOOL OUT THERE
