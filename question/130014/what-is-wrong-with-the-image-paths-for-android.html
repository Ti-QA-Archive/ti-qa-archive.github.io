Hi All.
I cannot find out why my image paths don't work in Android. I have tried android&#x2F;images&#x2F;image.png and &#x2F;android&#x2F;images&#x2F;image.png and ..&#x2F;android&#x2F;images&#x2F;image.png but none of them works. What do you do? I am using the new 1.8.0.1 Titanium SDK. I am almost ready to package and distribute, only this thing is troubling me now…

  • Accepted Answer

    We resolved this by just putting a another forward slash prior to the images folder name
    i.e.,

    use:
    &#39;&#x2F;images&#x2F;someimage.png&#39;

    instead of:

    &#39;images&#x2F;someimage.png&#39;

    • I think we are chasing two different issues.

      1. images in the android directory

      2. referencing images from javascript source files in subdirectories.

      3. Regarding images in the android folder. The contents of the platform specific directory are merged at build time. The 'android' directory no longer exists so you cannot reference it. (my original answer to Louise's question.)

      4. As Brian has stated (in answer to Zack's question), if you are trying to reference files from subdirectories on android, where the source javascript is NOT in the Resources directory and is being Ti.included into the project, a root reference is required.
