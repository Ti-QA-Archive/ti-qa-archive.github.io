Redraw view on orientation Change on ipad

The screen does not get adjusted on the orientations change. I have tried two generic ways of adding views

First One - using multiplication factor
width : 20 * (multiplication factor based on size of device)

Second one - using %
width : '10%'

In both cases. the screen draws perfectly well if u make initial layout as landscape for portrait. The trick works well with the first draw in both orientation.

But, the issues arises when u do orientation/rotation of the device. That Time, the views are not getting draws again based on the (multiplication factor or %). The view just does not redraw already rendered views.

The last option is to change the properties of these views/object as below to make it work
labelView.width = 20 * (multiplication factor based on size of device)

If we use this way, then it rendered the view again.But my view and layout is complex. There are many objects/views on just one screen. It's not feasible to set the property for each view/object to make the rotation works.

Does anyone have answer whey the first two methods are not working and how to redraw/render a view on orientation change.

    Try to add delay for setting width in orientationchange Event.

    Titanium.Gesture.addEventListener('orientationchange', function(e) {
    setTimeout(function(){
        myView.width = "100%"
    },2000);
}
