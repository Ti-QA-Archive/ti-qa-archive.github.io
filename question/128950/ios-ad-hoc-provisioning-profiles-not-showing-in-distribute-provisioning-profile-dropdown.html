iOS: Ad Hoc Provisioning Profiles not showing in Distribute "Provisioning Profile" dropdown

Is anyone else seeing an issue where only App Store provisioning profiles are being displayed in the "Select Provisioning Profile" dropdown in the "Distribute" dialog?

We used to see both Ad Hoc and App Store profiles, but now we only see our App Store profiles in the latest build of Titanium Studio.

This means we can no longer use this dialog to create Ad Hoc builds for internal testing, even though the dialog text says "Distribute app via Apple App Store or Ad Hoc".

The workaround we are using is to build in XCode, but this is a longer and more involved dev cycle than being able to build directly from Titanium Studio.
