Adding ImageView to mutiple TableViewRows?

For memory saving purposes I'm currently trying to add an ImageView to multiple TableViewRows instead of creating this ImageView over and over again.

Is this impossible? I have a variable containing that ImageView, let's call it imageView for now and some TableViewRows in a TableView. All I want to do, is to do something like:

var imageView = Ti.UI.createImageView(...);

tableRow1.add(imageView);
tableRow2.add(imageView);
tableRow3.add(imageView);

But if I try it this way, the image is shown only in tableRow3…

Is there something I'm missing?

Thanks for your help!

    • The problem is that in this case the imageView is one object only.
      When you are trying to reuse it is the same object and not a clone of it.

      Don't worry about images - they are loaded in the memory only when they are in view and pretty soon after the go out of the screen (you scroll) they are freed.

      A good thing would be to server the device the image in the exact size(no downscaling to appear).

      Having so many rows (hundreds) would imply though some tweaks - try to load only 10-20 rows (depending on complexity) and load the rest by request (a "more" button) or when the table scrolls close to the bottom.
      Also avoid transparent backgrounds, gradients and rounded corners if possible.

  • your imageView var is a reference to an object. what you are doing is putting that object in row 1, so the actual object is being associated with row 1, then you tell your app "add this object to row 2" so what it does is take the object and change it's container to row 2 and so it goes until the last row.

    What you need to do is 3 different objects. If you were looping some data then you would have created your object inside the loop and you wouldn't have that problem (because the loop has it's own scope).

    Also, I also recommend paying attention to Dan's tip on className
