Events firing multiple times

Using a dashboard view similar to kitchensink, and the left and rightNavButton are changed according to the selected item. These navbuttons typically take you to another window using the Ti.UI.currentTab.open(newwin); Others open an optionsdialog w/ 3 options, for navigation. First click on rightnavbutton, ok. But second click, navigations accumulate, the second time opens the new window twice, the third time 3 times. Using 1.6.0 SDK iPhone. Any solution? Thanks. http://pastie.org/1824247.js

  • From the sounds of your explanation and looking at your code, what's happening is that everytime the dashboard is clicked you're essentially adding EventListener on-top of EventListener. This is why everytime you click it replicates the number of times you've clicked total. So if you clicked 5 times, you'll get 5 responses.

    You can either move all EventListeners so they are by themselves, this is probably the safest way. Or if that's not possible you can remove the EventListener's after you close/change window, you do this by doing something like:

    Adding: event_variable.addEventListener(&#39;click&#39;, event_function);

    Removing: event_variable.removeEventListener(&#39;click&#39;, event_function);

    If you do that correctly for the dashboard EventListener, it should fix your problem.
