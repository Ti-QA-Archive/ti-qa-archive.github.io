windows works not the same as linux(ubuntu) with the backgroundImage.Bugs?

Ti SDK 1.8.x
i found that there was something wrong with the backgroundImage in different os.

In linux , some images works fine in emulator, but they are missing in the real android device. Plus, a wrong image url will easily lead a crash.

In windows , images in Resource/android never show up , even if i clean and rebuild the project.

any ideas?

    This sometimes is related to lower cases and paths

    cases: be sure your references to files in code are exactly the same as the filenames. Emulator is less strict in this sense than real devices.

    Paths: use constant Ti.Filesystem.separator when building paths. And I suggest always to use absolute paths (starting with &#x2F;) instead of relative paths.
