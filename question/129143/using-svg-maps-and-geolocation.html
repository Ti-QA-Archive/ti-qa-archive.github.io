Using SVG maps and geolocation

All,
I'm less worried about the code involved (for now), and more wondering if this is even a possibility. Our mobile app (for a class) will involve using a SVG of an area map. We're trying to see if it's possible to use the geolocation API to pinpoint an area on the map to reflect where the user is (a park map).
I'm new to mobile development, so I'm relatively unsure how I'd even have the map as an overlay, or have the coordinates reflected with portions of the map.

Anyone able to take a stab and let me know if it's even feasible?

    I did some work on this kind of thing a couple of months ago. In the end, the way I accomplished it was, create the image of your area to map, load it into google earth and use the overlay function to in earth to size/position your image to match the actual location on a map, then take the top,left and bottom,right latitude and longitude of the overlay. Next I took the JPG of that overlay and used it in Appcelerator and then transposed the user's latitude and longitude (using the geo-locate Appcelerator API) to an x,y coordinate on the jpg. Kind of a pain, but it did work. Let me know if that helps.

    Ray
