  • Hi

    You are calling this line of code as if it is a function (method) but it is actually and event you listen for. When you get this event fired (and you are listening for it) you can then react to the action.

    Try these lines of code in your app and then press the home button and then come back in again.

    Ti.App.addEventListener('pause', function (e) {
    Ti.API.info('Pause');
});
Ti.App.addEventListener('paused', function (e) {
    Ti.API.info('Paused');
});
Ti.App.addEventListener('resume', function (e) {
    Ti.API.info('Resume');
});
Ti.App.addEventListener('resumed', function (e) {
    Ti.API.info('Resumed');
});

    Add your code in-place of the Ti.API.info lines which will then run when the event is fired.

    • You are getting confused with what events are and how and what pause does.

      The events define code that will run when that event fires. For example code on a click event runs when you click the "thing".

      You are NOT able to force the app into a pause state, this is only initiated by the user when they either press the home button or the device places the app into a pause state due to an action that overrides your app - for example; incoming phone call.

      The problem you need to solve is to stop whatever code is running inside your gpsstart function. If you are using the Ti.Geolocation.addEventListener('location', …); you would need to use the corresponding Ti.Geolocation.removeEventListener('location', …);
