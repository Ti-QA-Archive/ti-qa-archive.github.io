listview items in ios overlap

Hi,

I have a problem where I cannot set horizontal layout for listview rows in iOS. And if I don't set the left property of each item in the row they overlap.

For Android the same code works well.

This is how the row item looks like:
var row = { id: customerModel.id, orderBy: name, name: {text: name}, mobile: {text: mobile}, gender: {text: gender}, birthDate: {text: birthDate}, properties: {height: Ti.UI.SIZE} };

and this is the template:

  • You can do vertical/horizontal layout in a listview, but you don't want to. To do so, wrap you components in a view which is set to the layout you want.

    However, this will slow down rendering of your list items as well as scrolling. That's because the layout subsystem is going to have to figure out where to put components for every item it renders. And since native list items are reused as you scroll and add/remove list rows, the subsystem will have to re-figure out that positioning information frequently.

    You'll get far better performance if you use absolute positioning. You'll need to specify this positioning only once, on the template. It might take some trial and error. But the performance gains will be worth it.

    BTW, next time, please use three ~ before and after you code to properly format long blocks of code.
