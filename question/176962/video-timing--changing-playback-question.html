Video timing / changing playback question

Hi - I've nearly completed an app for iPad which makes heavy use of video playback, playing a single file stored on the device. The game has to start video (usually) when the user initiates it and play short sections of the video, which then 'loop' - i.e., once the video reaches the end of the defined section, the program sets the currentPlaybackTime back to the start of the current section. Key frames in the video allow instant jumps, which all works nice and smoothly.

However, currently the safest and most reliable way I've found of keeping track of where the video is up to is by using a setInterval set to roughly 24fps (41.66) and to keep checking the currentPlaybackTime. This works almost perfectly, which is really annoying because just when you think it's never going to go wrong you see a flash frame of the next part of the video - presumably when the script has failed to catch the end and restart the video quick enough. I've tinkered with the timing, for instance it actually resets the video position 350ms before currentPlaybackTime reaches the end, and I know I could tinker with the video and increase this buffer to above 350ms, but the problem is the randomness of this - if I trim off too much sometimes it cuts in very noticeably.

So does anyone know of a more native / better way of checking when a video reaches a certain point and resetting it to another point? It's almost something the native player should do, but I can't see any way of setting those parameters and in fact I might be wrong and there is no way of doing this natively.

Thanks for reading this long question!!
