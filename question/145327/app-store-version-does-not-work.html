App store version does not work

We have a strange problem. During testing everything in our iOS version works perfect: in the simulator, on multiple devices and even if we make an ad hoc version and test that one it still works. However, for some reason when we build it for the app store (and Apple approves it every time) things go wrong. Then all of a sudden there are 2 bugs (as far as we know) in the market version.

The problem is that we can't replicate the bugs, because during testing everything works perfectly. We have tried to do some changes, but we have to wait 1-2 weeks to get reviewed and see if that changes worked. We have submitted it 4 times now with the same problems over and over again.

Does anyone know how a way to test the 'store version' or another solution to our problem?

Thanks in advance!

  • Hi

    I had a similar issue some time ago and the answer was because of an errant comma in a third-party library, for example ONLY not the code that had the error;

    var view = Ti.UI.createLabel({
    height: Ti.UI.SIZE,
    text: 'Label',
    width: Ti.UI.SIZE, // this comma was OUR problem.
});

    We also thought there was no way to test under the basis of an app-store app rather than a local device app - there are subtle differences - as I found out.

    However, compile for ad-hoc (set up certificates) and this is the same end compile as the app store app, you can put this version on your device and it works the same as if you had compiled for the app store (or as far as I can tell).

    This way you should be able to replicate the bugs, build in some test code, retest and keep going until all is well. Then put back in the store.

    A programmers life is not an easy one :(
