Ti.storekit restore non-renewing subscription identify user

I´m building an iphone app with inApp purchase and non-renewing subscriptions. To verify and restore receipts I want to use my external server. I save a few parameters like transaction id, macaddress, purchase date in my DB. Restoring a subscription on the same device can be done with the device macaddress but apple requires the restoring over all devices the user owns. Which parameter can I use to identify not only the device but also the current user (apple id) to restore the subscription over all the users devices.

Can I get the current apple id associated with the device?

Did anyone have a working example for these external verification and restoring?

Or is it not needed that the subscription should be restored over all devices?

Thank you for your help.
Mirco

  • After the purchase, I am asking the user twice if he wants to save this purchase on our servers so he can restore it later or use it in other devices. If he agrees, then I ask him to log into facebook and store the user's uid on my server. When he restores, I ask him to log into facebook and then I can match his subscription to the uid.

    I just submitted today, hopefully Apple OKs this, I have read that as long as you don't force the user to log into facebook to make a purchase, they approve.
