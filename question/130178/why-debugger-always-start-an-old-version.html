Why debugger always start an old version

Hi! Here's my annoying problem. I modify the code but each time the iPhone simulator launch an old version of an include. Why? I'm having the latest version 1.0.7. I'm working for customer here and I'm tired of this and I consider getting back to xCode. So please, help me with this.

Thank you!

    Sometimes the Titanium SDK has an issue with telling that updates have been done. If it doesn't thing there has been any updates, it skips the build process.

    You can clean your project in Titanium Studio ( or delete the build/iphone directory ).

    I'm running the CI Build and it doesn't seem to be an issue. You can get the CI Builds from http://builds.appcelerator.com.s3.amazonaws.com/index.html
