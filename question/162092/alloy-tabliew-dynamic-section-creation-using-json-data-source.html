    Hi

    You need to create a loop, to loop through your posts array from your JSON object.

    Inside the loop you would call a controller that creates the TableViewRow and pass the data through to that controller. This controller would return a view, in this case you TableViewRow.

    Once you call your controller, you add the returned view to an empty array. You then use 'setData' to set this new array as your TableView data.

    Heres some code to get you started:

    var data = [returned JSON],
    views = data.posts.length,
    row,
    array = [];

_.each(views, function(item, index) {
    row.add(Widget.createController('row', item).getView());
    array.push(row);
});

$.tableview.setData(array);

    • I would just keep it simple for now.

      Yes, the files need to have code in them, here is some example code:

      row.xml

      <Alloy>
    <TableViewRow id="row"/>
</Alloy>

      row.js

      var args = arguments[0] || {};

$.row.title = args.title;

      row.tss : this can be left empty as no styling is necessary at this point.

      For the sake of just getting you to grips with Alloy, I wouldn't bother with placing these files inside of a widget.

      • row.xml should be placed in this location '/app/views'
      • row.js should be placed in this location '/app/controllers'
      • row.tss should be placed in this location '/app/styles'

      Now when you call Alloy.createController(&#39;row&#39;, item).getView(), it will search from the root of your app for the files related to 'row'.

      'item' is the data you are passing to your controller and inside your row.js, args captures this data.

      getView() is a method used to retrieve the parent view of your xml, which in this case would be TableViewRow.

      Try this first and let me know how you get on, I can then help you understand widgets afterwards :)
