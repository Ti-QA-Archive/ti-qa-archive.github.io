Trouble changing backgroundGradient colors in a navbar button

I am having trouble changing the backgroundGradient of a button in the Navbar.

window.xml

<Alloy>
    <Window id="win" class="container" layout="vertical" height="100%" width="100%" barColor="black">
        <LeftNavButton>
            <Button title="Cancel" id="leftNavButton" onClick="doCancel"/>
        </LeftNavButton>
        <RightNavButton>
            <Button title="Send" id="rightNavButton" onClick="doSend"/>
        </RightNavButton>
</Alloy>

window.tss

'#rightNavButton': {
    backgroundGradient: {
        type: 'linear',
        startPoint: { x: '0%', y: '0%' }, 
        endPoint: { x: '0%', y: '100%' },         
        colors: [ 
            { color: '#A6CBFF', offset: 0.0 }, 
            { color: '#286DD9', offset: 0.5 } 
        ]
    },
    height: '25',
    width: '80',
    backgroundImage: 'none', 
    borderRadius: '5',
    font: {
        fontWeight: 'bold',
        fontSize: '14'
    }    
}

This does nothing to change the colors of the button. I can't figure out why. I also tried using setBackgroundGradient with same result.

window.js

$.rightNavButton.addEventListener('touchstart', function(e){
    e.source.backgroundGradient = {
          type: 'linear', 
        startPoint: { x: '0%', y: '0%' }, 
        endPoint: { x: '0%', y: '100%' },
        colors: [ 
            'black', 
            'gray'
        ]        
    };
});

Thanks in advance for any help!

    Hi

    Buttons in the Nav bar inherit a system style for their UI - this is by design, so changing the background color or gradient will not normally have the outcome you desire.

    You could try setting the style to Ti.UI.iPhone.SystemButtonStyle.PLAIN but I have never tries this in a nav bar as I have not had a use case for this.

    By setting the plain style it tells the button to NOT inherit system UI when displaying - this may have the outcome you are looking for.

    If not try adding a view to the property of left or right nav button and then add the actual button inside that. As this property actually accepts a view you can add things other than views inside it. There are limitations to this but might get you further if the plain often did not.

    • Setting the style did not work. However, wrapping the button in a view does work.

      window.xml

      <Alloy>
    <Window id="win" class="container" layout="vertical" height="100%" width="100%" barColor="black">
        <LeftNavButton>
            <Button title="Cancel" id="leftNavButton" onClick="doCancel"/>
        </LeftNavButton>
        <RightNavButton>
            <View>
                <Button title="Send" id="rightNavButton" onClick="doSend"/>
            </View>
        </RightNavButton>
</Alloy>

      Then using these two event listeners I get the behavoir I was looking for. Button changes to black/gray when clicked and then back to it's original color.

      window.js

      $.rightNavButton.addEventListener('touchstart', function(e){
    e.source.backgroundGradient = {
        colors: [ 
            'black', 
            'gray'
        ]        
    };
});

$.rightNavButton.addEventListener('touchend', function(e){
    e.source.backgroundGradient = {
        colors: [ 
            '#A6CBFF', 
            '#286DD9'
        ]        
    };
});
