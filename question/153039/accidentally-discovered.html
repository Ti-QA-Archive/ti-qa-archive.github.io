Accidentally discovered!

This is more an observation but I will try to turn it into a question :)
I have a table and each row is a link to another page and when clicked the user is taken to another page. That is all fine. What I unintentionally discovered is that if I do a double touch on a row [that is I touch a row with two fingers at the same time] the row gets selected rather than transferring me to the linked page. Now is that a bug or a feature that we can take advantage of?

Details for reproduction:

  • Titanium studio 3.1
  • Test device: Nexus 4 Android 4.1 I think
  • two fingers

It is not the easiest thing to reproduce but trying couple of times achieves it. Hope I am not the only one observing this :)

  • Hi

    I think you will find that is just one of those things rather than a specific objective. It may very well be a "feature" (quotes intentional) something that should not occur or simply does irrespective of right or wrong - the end result being it is now a feature.

    If it fires the two finger event then yes you can take advantage of it.

    However as you noted yourself you accidentally discovered this - I have never tried it before either. So what are the discover-ability chances of your users also finding this. If you do plan to benefit from it - understand two things;

    • Most of your users will never find it
    • It may very well go away in future versions of the Android OS

    • You normally detect a twofingertap event by adding an event with this name;

      // where tbl = the name of the variable used to create a table (for example)
tbl.addEventListener('twofingertap', function (e) {
    alert('twofingertap fired');
});
// same logic as the click event
tbl.addEventListener('click', function (e) {
    alert('click fired');
});
