Ti.include('/src/myFolder/myFile.js')

  • Hi Michael,
    Yes you can use easily "Ti.include".
    Please put you file in
    app –> assets –> {folder name "abc"} –> filename.js

    and this file we can add this type

    Ti.include("abc/filename.js");

    • if you want to access something as Common/name.js, or lib/something.js, or utilities/another.js, you would put them in the following paths respectively:

      • app/lib/Common/name.js
      • app/lib/lib/something.js
      • app/lib/utilities/another.js

      We will not be supporting arbitrary folders inside the app folder structure, which would in turn undo its structure.

    • app/assets and app/lib do the same thing, they just copy these files to Resources at compile time. app/lib/somefile.js would overwrite app/assets/somefile.js at compile time. The only reason both exist is as a form of organization, allowing you to put JS libraries in app/lib and stuff like images, videos, and other files in app/assets.
