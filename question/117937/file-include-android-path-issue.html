File Include Android Path Issue

I know there are now quite a few questions about this issue.

Here is my latest issue, I can include files in iPhone without issue, but in Android I keep getting file not found, yet the file exists and is identical path as running in the iPhone emulator.

OK so what is the issue, in Android emulator file is not found when, I have a .js file in a sub directory of the Resources and I include a file in a different sub direcotory by relative path i.e. Ti.include('../dir_2/file.js'). Now this works on some files and not on others.

The files which fail are ones which also have a Ti.include in them.

So if I include in app.js a file in tmp and this file includes another file it will fail. if it does not include another file it loads great.

I hope I have explained this correctly, I am hoping somebody from appcelerator will come onto this and see if we need to raise a bug, or provide a solution as I haven't fixed it yet.

Thanks

  • You need to use the 'absolute' path from the Resources dir. So in if you have file.js in /Resources/dir2/ you refer to it in Android like Ti.include('/dir2/file.js')

  • On Android, Ti.include() is relative to your Resources directory. On iOS, the path is relative to the current document's location. So you can use the absolute path as suggested by Jeroen. Or look into the CommonJS approach, which uses paths relative to Resources on both platforms.
