Hide ActionBar on 3.3.0

After going from 3.2.2 to 3.3.0, I can't get the ActionBar to hide on Android. Previously, it was working flawlessly as a full screen, no actionbar application.

Per http://docs.appcelerator.com/titanium/latest/#!/guide/Android_Action_Bar and a lot of internet searching, I've tried various variations of the following::

platform/android/res/values/custom_theme.xml:

<?xml version="1.0" encoding="utf-8"?>
<resources>
    <style name="Theme.AppCompat.NoActionBar" parent="@style/Theme.AppCompat">
        <item name="android:windowActionBar">false</item>
    </style>
</resources>

<?xml version="1.0" encoding="UTF-8"?>
<manifest xmlns:android="http://schemas.android.com/apk/res/android" package="xxx" android:versionCode="1" android:versionName="1">
    <application android:hardwareAccelerated="true" android:largeHeap="true" android:icon="@drawable/appicon" android:label="xxx" android:name="xxx" android:debuggable="false" android:theme="@style/Theme.AppCompat.NoActionBar">
        <activity android:name=".xxx" android:label="@string/app_name" android:theme="@style/Theme.Titanium" android:configChanges="keyboardHidden|screenSize" android:screenOrientation="portrait">
            <intent-filter>
                <action android:name="android.intent.action.MAIN"/>
                <category android:name="android.intent.category.LAUNCHER"/>
            </intent-filter>
        </activity>
        <activity android:name="org.appcelerator.titanium.TiActivity" android:configChanges="keyboardHidden|screenSize" android:screenOrientation="portrait"/>
        <activity android:name="org.appcelerator.titanium.TiTranslucentActivity" android:configChanges="keyboardHidden|screenSize" android:screenOrientation="portrait" android:theme="@style/Theme.AppCompat.Translucent"/>
        <activity android:name="ti.modules.titanium.ui.android.TiPreferencesActivity" android:configChanges="screenSize"/>
        <service android:name="com.appcelerator.analytics.APSAnalyticsService" android:exported="false"/>
    </application>
    <uses-permission android:name="android.permission.ACCESS_WIFI_STATE"/>
    <uses-permission android:name="android.permission.ACCESS_NETWORK_STATE"/>
    <uses-permission android:name="android.permission.WRITE_EXTERNAL_STORAGE"/>
    <uses-permission android:name="android.permission.INTERNET"/>
</manifest>

Hiding works programmitcally via activity.actionBar.hide() but trying to do this on the first window causes it to flicker for a second since it launches with the actionbar showing.

Anyone had success with this on 3.3.0 and is there a simpler way to make an app always fullscreen (no title bar or action bar)?

  • Hi,

    I have been having the same problem but setting the theme to parent=&#39;@style&#x2F;Theme.Titanium&#39; fixed it for me.

    I did have a problem with a 'type' being required for the <item> so had to add that to. Check here http://developer.appcelerator.com/question/176470/android-theme—a-type-attribute-is-required-for-item#comment-215911

    Below is the code I used:

    <?xml version="1.0" encoding="utf-8"?>
<resources>
    <style name="Theme.NoActionBar" parent="Theme.Titanium"/>
     <!--Depending on the parent theme, this may be called android:windowActionBar instead of windowActionBar-->
        <item type="bool" name="android:windowActionBar">false</item> 
        <item type="bool" name="android:windowNoTitle">true</item>
    </style>
</resources>

    I had a problem with targeting SDK's as well so I removed the below line from tiapp.xml.

    <uses-sdk android:minSdkVersion="14" android:targetSdkVersion="19"/>

    I am also using NAPP Drawer and removed the reference to SDK targeting there as well, I removed the line from timodule.xml located in modules/android/dk.napp.drawer/1.1.3/.

    The final thing I noticed that is that one of my pages was still showing the actionbar, the var that was creating the window had modal set to true, I removed this and that removed it from that window.

    Hope this can of help to someone

  • Hi Josh,

    Thanks for your concern.

    I've tested your issue and successfully remove action bar and make an app to always support fullscreen with Titanium SDK 3.3.0.GA. It works fine for me.

    Please check my test case below:

    To modify the theme to hide the action bar:

    1. Add a custom theme file to the project:

    platform/android/res/values/custom_theme.xml

    <?xml version="1.0" encoding="utf-8"?>
<resources>
    <style name="Theme.NoActionBar" parent="@style/Theme.Titanium">
        <!-- Depending on the parent theme, this may be called android:windowActionBar instead of windowActionBar -->
        <item name="windowActionBar">false</item>
    </style>
</resources>
    1. Modify the tiapp.xml file to use the modified theme:
     <android xmlns:android="http://schemas.android.com/apk/res/android">
    <manifest>
        <application android:theme="@style/Theme.NoActionBar"/>
        <!-- Need to specify at least API level 11 for Titanium SDK 3.2.x and prior -->
        <uses-sdk android:minSdkVersion="14" android:targetSdkVersion="19"/>
    </manifest>
</android>

    For Fullscreen Support:

    For fullscreen support use <fullscreen>true</fullscreen> in tiapp.xml

    By following this procedure i've successfully removed my actionBar and able to make the app fullscreen.

    For more details, please check the link to hide the actionBar
