Titanium Mobile SDK 1.7.2 vs. 1.7.1

I've been developing with studio since it has been available. I have it installed on an XP machine, virtual hackintosh machine, and now a MacBook Pro. The virtual hackintosh machine will be deleted as soon as I can get studio working with my apps on the new MacBook Pro. The virtual machine and the XP machine are both sitting at Titanium Mobile SDK 1.7.1

I've been working with three apps. 1) Kitchen Sink 2) Database App 3) Dessert App.

All three I have successfully emulated and installed on to an Android device, more specifically an old school Droid, using the XP machine and the virtual machine mentioned above. Enter the MacBook Pro.

I downloaded and installed Studio to the new MacBook Pro. Out of the box, the Titanium Mobile SDK is 1.7.2.

All three apps mentioned above run on the android emulator fine. I get mixed results when installing to to my droid. The database app force closes on the droid but not the emulator. If I take the code back to my 1.7.1 machines, it installs and runs fine. The database app is an app that I am using to follow along on one of the database tutorials (I'll post a link later if someone asks). The kitchen sink app "fails to install" do to memory constraints never seen before. If I take the same code back to the machines that run SDK 1.7.1, all is good.

Can someone point me in a direction to look for what might be happening?

Thanks!

  • I've run into a few issues with 1.7.2 but they all seem to have been addressed in the recent CI builds http://builds.appcelerator.com.s3.amazonaws.com/index.html

    Hope this helps.
