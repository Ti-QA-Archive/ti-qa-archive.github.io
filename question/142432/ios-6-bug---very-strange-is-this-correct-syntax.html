iOS 6 Bug - Very Strange, is this correct Syntax?

So this was working before iOS6. It doesn't seem strange to me. I am adding 2 labels to a TableViewRow. If I remove the 2nd label app runs, if 2nd label is there, app crashes. Here is my code.

var description = Ti.UI.createTableViewSection({
    headerView: cf.general.createSectionHeaderLabel(courseList[0].TITLE)
});
var desc = Ti.UI.createTableViewRow({
    layout: 'vertical',
    selectedBackgroundColor: '#fff',
    selectedColor: '#000'
});
var descContent = Ti.UI.createLabel({
    text: courseList[0].COURSEDESCRIPTION,
    font: {fontSize: 12}
});
var notes = Ti.UI.createLabel({
    text: courseList[0].INSNOTES,
    font: {fontSize: 12},
    top: 10
});
desc.add(descContent);
desc.add(notes);
description.add(desc);

If I remove the desc.add(notes); The app runs, If it's there it dies. Studio is not throwing an error, or logging anything whatsoever at the moment, no idea why. All of this worked fine prior to iOS 6.

  • Apparently this came down to labels having an empty string as their text. In my example, on both my apps actually, I display information if it exists and if it doesn't oh well. No where in the documentation does it say this is not ok, but apparently in iOS6 if a label has an empty string as its text it will crash. I tested it even with having text: '' and that failed to. I had to make at least a space in there.

    Sure sure I could do a check and only add the label if the label is needed and that indeed in this case would have prevented the problem, but I don't see anything wrong with the way I did it. Others may come across the same error since it worked before.

    Labels must have at least 1 char as text or the app will crash. Even if that 1 char is a space.
