Android Webview - Body With 100% Height

Application type: Mobile App - Android

Titanium SDK: Titanium SDK version 3.2.3.GA

Platform & version: Android 4.4.2

Device: Nexus 5

Host Operating System: Windows 8

Titanium Studio: Titanium Studio, build: 3.2.3.201404181442

Hello,

This might be a simple problem, and I think its pretty straightfoward to emulate.

I created a new project, I only changed the index.xml to:

<Alloy>
    <Window class="container">
        <WebView id="webview" url="http://mywebsite/"/>
    </Window>
</Alloy>

and index.tss to:

".container": {
    backgroundColor:"white",
    height: Ti.UI.FILL,
    width: Ti.UI.FILL,
    contentHeight: Ti.UI.FILL,
    contentWidth: Ti.UI.FILL
}

My webpage has html,body and a div with height 100%. The page is rendered properly in chrome, but in the webview inside titanium, all those divs appear to be with height auto…

From what I can see in the comment section here: http://www.appcelerator.com/blog/2013/11/android-kitkat-now-available/ Ingo Muschenetz says that the only open ticket for webviews in android kitkat is https://jira.appcelerator.org/browse/TIMOB-16695

Anyone had a similar problem or knows how to solve this?
