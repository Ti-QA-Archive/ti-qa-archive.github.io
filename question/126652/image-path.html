Image Path

This is a crazy beginner question, I know, but it's shocking how little information is out there (or maybe my Google Fu is weak today). I'm just trying to create an imageView and I can see that the image is conventionally loaded relative to the Resources directory, but I'm developing for iPhone and Android. I have a bunch of subfolders beneath my Resources directory. Do I have to detect the platform and display for each image or is that something that Titanium does for me?

If I add a body background image to Resources&#x2F;iphone and reference it as &#39;body.png&#39;, the iPhone simulator doesn't appear to pick it up, so I'm looking for some general guidelines with respect to resources.

  • Check out the folders android and iphone that exist on the same level as Resources. You will find that for android there exist subfolders for a lot of resolutions and for iphone you will use normal images or images with the @2x suffix on the file name for Retina.

    You can reference these images with the root path - Titanium will do the rest grabbing the correct image for the specific platform/resolution.

    Hope this helps a little?

    • Yes, it would probably be easiest if you posted a non-working example of your code.

      BTW.. I actually meant the Resource/iphone and Resources/android folder - not parallel to the Resources folder.

      For me this works:

      backgroundImage : &#39;&#x2F;images&#x2F;overlaybox.png&#39;,

      And the image lies under:

      Resources/android/images/<res…>/overlaybox.png

      Resources/iphone/images/overlaybox.png

      Resources/iphone/images/overlaybox@2x.png

      (so I added an images subfolder to the Resources/iphone folder)

      Let me know if you still have problems.

      — commented October 10th 2011 by Kathrin Holweger

    • The above comment by Kathrin is the absolute & correct answer for iPhone. For Android, it should be:

      • Resources/android/images/low/overlaybox.png
      • Resources/android/images/medium/overlaybox.png
      • Resources/android/images/high/overlaybox.png

      where those are 3 different sized images, but all named the same. The high one should be twice the size of the low one.

  • Within your code you can define an image path like 'images/icons/icona.png';

    This lives in your resource folder in images/icons

    That should work on both iPhone and Android.

    When the applications built it will include the images directory in the build for use within the application.

    Hope this explains it, there have been a lot of issues over paths and differences between Android and iPhone but latest release resolve these.

    • Not to be pedantic, but for the sake of others reading this I think you should probably add a leading / to make the path absolute, i.e., use &#39;&#x2F;images&#x2F;icons&#x2F;icona.png&#39;; instead. As coded you have it as a relative path, so if your current *.js was not in the Resources folder, you would have path problems.

      As of SDK 1.7.0, both Android and iOS should now treat paths the same, as described in this blog which discusses using absolute vs relative paths.
