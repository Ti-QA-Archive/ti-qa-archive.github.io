Memory and garbage collection

I have been running a few tests, as it has been well documented that it is possible to crash apps dues to the way that the memory is handled.
It would appear that although when you close a window or null an object, it is added to the stack of memory to be freed, it does not get deleted immediately. This is where the problem lies, as it is when the memory gets low that it becomes possible to crash the app… This leads to my question… Is there no way to force the memory to be freed immediately?
