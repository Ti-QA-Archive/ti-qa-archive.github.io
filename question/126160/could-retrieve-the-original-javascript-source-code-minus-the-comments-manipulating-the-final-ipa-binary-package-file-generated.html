Could retrieve the original Javascript source code (minus the comments) manipulating the final IPA binary package file generated

As the entire code of the Mobile Application is written in Javascript as required by Titanium, an external entity could have access to all the business logic utilized in the application (UI, Controller and Model components).

These are the steps to to visualize the source code from the IPA file:

1) Change the extension of the package file from .IPA to .zip
2) Unzip the file
3) Load the .app file into Notepad++
4) Search for "_JS" until a name of a project source code module is displayed (see picture B for controller/navigationController_js)
5) Copy the string before the "controller/navigationController_js" until the previous NULL
6) Convert the string from Hex to ASCII selecting the string and using the option Plugins -> Converter -> "HEX->ASCII" from Notepad++
7) Compare the file obtained with the original source code listed in file.

Is there is a workaround for this?

    Enhanced source code security is definitely an area where we can improve. We're working on solutions internally on the iOS platform side, but a good intermediate step is to run an obfuscation tool (like the Google Closure Compiler) on your application if you'd like to add an additional layer of security. The Juicer command line utility is really nice for this type of thing.
