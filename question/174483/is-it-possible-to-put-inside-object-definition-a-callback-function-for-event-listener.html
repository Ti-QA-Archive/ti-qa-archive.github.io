Is it possible to put inside object definition a callback function for event listener?

Seeing the use of Ti.Network.createHTTPClient seems that can be possible to define a property of the object as event listener. For example it is possible to handle the onload and onerror events writing this

onload: function(e){...},
...
onerror: function(e){...},

So I was wondering if this way of writing event listeners for HTTPClient can work for all JSON definition objects. I tried with an imageView object putting inside its definition this

onclick: function(e){alert('Hello!');},

but nothing happend. That way is a prerogative of HTTPClient object only? Thanks for the attention.

    For UI elements you need to use addEventListener.

    On the other hand the onload of the HTTPClient is not an event, it's a callback that gets called at the native level.
