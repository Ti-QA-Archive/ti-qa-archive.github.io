unable to set the webview url to an anchor inside a local webpage in iOS

I am trying to open a local webpage with an anchor in its html code.
On Android I can do someting like:

webview.url='/localhtmlfile.htm#htmlAnchorName'

and the webview opens just fine and scrolls to the htmlAnchorName position in the web page
However if I do the same thing on iOS (exept for the leading '/') i.e

webview.url='localhtmlfile.htm#htmlAnchorName'

The weview opens but I get the same result as if I had done

webview.url='localhtmlfile.htm'

I have also tried webview.evalJS('localhtmlfile.htm#htmlAnchorName') but then I get a spinner in the webview as it was trying to load the file over the internet instead of accessing the local file.

Any ideas on how to make iOS take notice of html anchors in local files.

    Webview is often a bad idea. The seam of titanium is avoiding of webview. Anyway: If this special URL mit # doesnt work, then you can start an evalJS process (after loading event!). This javscript scrolls to your id in DOM. Hope this helps.
