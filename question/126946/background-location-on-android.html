Background Location on Android

Hi,
I want to have my application (on Android) run a background service that every hour checks the users current location. I've created the service but the issue I'm having is with the location. If I use getCurrentPosition it doesn't turn on the radio so I get stale data. If I use the location event listener it doesn't seem to fire in the background (I see the service run but the callback never runs).

Thoughts?

  • Sample code
    app.js:

    var service;

function registerService(){
    if (service!=null){
        Ti.API.info("Service already running");

    } else {

        service = Titanium.Android.createServiceIntent({
          url: 'testService.js'
        });
        service.putExtra('interval', 15 * 1000);
        Ti.API.info("Starting android service");
        Titanium.Android.startService(service);
    }
}

    testService.js

    
Ti.API.info("Test Service running");

//Titanium.Geolocation.getCurrentPosition(handleBackgroundLocation);
Titanium.Geolocation.addEventListener('location',handleBackgroundLocation);

function handleBackgroundLocation(e){

        Ti.API.debug("background location event responded "+JSON.stringify(e));
        if (e.success){
            var lat = e.coords.latitude;
            var lng = e.coords.longitude;
            Ti.API.info("Got location "+lat+" "+lng);

        }
        Titanium.Geolocation.removeEventListener('location',handleBackgroundLocation);

    }

    What I found is that while my app is running the background service works fine, once i back out of the app the background service runs but the callback never gets called (probably due to the error).
