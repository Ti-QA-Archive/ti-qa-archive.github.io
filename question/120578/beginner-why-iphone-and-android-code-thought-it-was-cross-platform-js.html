  • Titanium is not designed to be a "write once, run everywhere solution" and this is intentional. Android, iPhone and BlackBerry are all very different devices, and OSs, each with there own UI styles, and native OS APIs. If Titanium were to work cross platform for all devices, many features, simply could not exist.
    That being said, you have 2 options for writing apps that work on many platforms:

    1. Write your code with if statements to distinguish your code on the diff platforms you are compiling for.
    2. Use the device directories in the resource folder to separate your code. Details of how to do this can be found here: http://wiki.appcelerator.org/display/guides/The+Application+Project+Structure

    For the most part option 1 is best for small-ish apps where there are not as many changes between platforms, and option 2 is best for larger apps. Or you can use a mixture of both those solutions.

    If you have any questions about any of this, please feel free to ask :)

    Objective-C and Java is for advanced module-creating and adding your own functionality to the mobile SDK.

  • Hi Gerald.

    I think it's not cross-platform JS, it's rather cross-platform-compiled JS. The difference is that JS is not interpreted by JS interpreter at runtime, it's converted to native source code (Objective-C & Java) and then compiled for native platform. Otherwise, Titanium application would not be native application, it would be web applications that runs inside (web) browser.

  • A number of issues quickly arise when you do cross-platform projects, so you need to be aware of that. In example, YouTube movies in a Webview won't play in Android. There's no left or rightnav button location in Android, so you need to plan for that. Objects have no auto .width and .height readable attributes, so you will need to account for that in Android (Android will return 'auto' and not the width or height of the element). Images are not automatically scaled by one of the two factors as they are in iOS. And much more… Not necessarily all about Titanium, a lot is about limits of the Android system.
