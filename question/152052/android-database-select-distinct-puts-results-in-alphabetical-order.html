Android Database SELECT DISTINCT puts results in alphabetical order

I am calling a database file and pushing the results into a TableView.
This is how I'm calling my code:

var db = Ti.Database.install('/categories.sqlite', 'list');
var rows1 = db.execute('SELECT DISTINCT cat FROM items');
while(rows1.isValidRow()) {
    //My code to create the tableview here
}

The categories which it retrieves are Lorem Pages, Ipsum Pages, Dolor Pages and Sit Pages. As you can imagine, these are the names I am testing my app with.

The only problem is, on Android, when I do a DISTINCT query, it sorts the results into alphabetical order before outputting them…

On Android, my table rows are: Dolor Pages, Ipsum Pages, Lorem Pages, Sit Pages

On iPhone (Which displays them how I want, which is the order they have been entered into the database): Lorem Pages, Ipsum Pages, Dolor Pages, Sit Pages

How do I go about getting Android to display them in their original order? It seems to sort the results as it does the SQL query, is there a way to stop it from doing so?

I am using: Titanium SDK: 3.1.0.GA and Android Version: Google APIs (Android 2.2) [armeabi]

Cheers,
Ben

    I hope you will have id for all records in your table. So you could use order by class in your query to prevent sort by category name.

    db.execute('SELECT DISTINCT cat FROM items ORDER BY id');
