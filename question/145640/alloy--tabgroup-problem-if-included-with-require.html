Alloy | TabGroup problem if included with Require

[INFO] :  Building for target: simulator
[INFO] :  Building using iOS SDK: 6.0
[INFO] :  Building for iOS Simulator: 6.0
[INFO] :  Building for device family: iphone
[INFO] :  Building for iOS 6.0; using 4.3 as minimum iOS version

Titanium Studio: build: 3.0.0.201211301903

Hi,

I wanted to build a tab bar example with following structure:

controllers

  • index.js
  • home.js

views

  • index.xml
  • home.xml

home.js

$.index.open();

index.xml

<Alloy>
    <Require src="home"/>
</Alloy>

home.xml

<Alloy>
    <TabGroup>
        <Tab id="home">
            <Window id="homeWin">
                <Label>Home</Label>
            </Window>
        </Tab>
    </TabGroup>
</Alloy>

Running this example in the ios simulator I get following Error

[INFO] :  iPhone Simulator log:
[ERROR] :  Script Error = 'undefined' is not a function (evaluating '$.__views.home.addTab($.__views.home)') at home.js (line 24).
[INFO] :  Application has exited from iOS Simulator

It only works if I put the TabGroup directly into the index.xml.
As I'm quite new to Appcelerator it could be that I'm doing something totally obvious wrong.

I get the same error if I try to include the tabgroup view in the index controller:

...
function initApplication(){
    var home = Alloy.createController('home');
    home.getView().open();    
};
...

Thanks for any hint on this!

Christoph

  • Accepted Answer

    All you need to do is change your index.js to this:

    $.index.getView().open();

    The <Require> element does not return a UI component, it returns a controller. To get the views from within your controller, you need to call getView(). With no parameters, it returns the first top-level view inside the controller, in this case your TabGroup. You can optionally give it an id as a parameter, and the view inside the controller with that id will be returned,
