Version vs Build

I recently came across an interesting issue that either I don't understand, or if I do, Titanium needs to change the way they work.

In Xcode you have version numbering and build numbering. My understanding is that during development, while I may be working on version 1.0 of my app, I may have various builds during that process. When I am finished with 1.0 and release it, it may be build 15, who knows. Test Flight App understands it this way as well.

However in Titanium Version = Build. You can have incremental builds in the same version, as far as I can tell anyway. This is a problem. The main reason is because I think it is incorrect, especially since the build number is in the info.plist file and always matched the version number. Unless there is a way to change it that I don't know about, but in the 2.1.0 SDK I don't see anything.

The second reason is that my current app that I am updating from PhoneGap to Titanium is currently in the app store as Version 1.0.2 Build 14. Well, now in Titanium I need to release version 15 of my app so that way it's higher than build 15. Other wise I need to generate a custom info.plist file.

Please fix this in future versions of Titanium unless I my understanding of builds is incorrect, in which case I apologize before hand. It seems easy enough to add "Build Number" as a setting in the tiapp.xml file.

    Apple Developer Core Foundation Keys

    Two strings that actually mean anything when submitting:

    CFBundleVersion - specifies the build version number of the bundle

    CFBundleShortVersionString - specifies the release version number of the bundle

    Until recently (within the last 6 months) apple only cared about one of them. They then made a submission change and required BundleShortVersionString be present and incremented on submission.

    In titanium, they are equal to each other.

    TIMOB-6558 TIMOB-7674

    If you think its a bug or needs enhanced, submit a JIRA for it.
