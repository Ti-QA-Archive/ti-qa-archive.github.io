iOS7 translucent title bar difference between simulator and device.

When i'm testing my app on the simulator und actual device there is a strange difference.
The title bar on the iOS7 simulator is translucent but when i run the same app on my iphone 5 (iOS 7.0.3) the title bar isn't translucent any more.

This are the window settings

"Window": {
    color: "#FFF",
    barColor: "red",
    backgroundColor: "white",
    navTintColor: "white",
    extendEdges:[Ti.UI.EXTEND_EDGE_TOP],
    autoAdjustScrollViewInsets: true, 
}

has anyone an idea how to fix that?

    Hi

    I have spent some time recreating your problem and looking for a solution. Here are some notes;

    • I do experience the same problem you are
    • The normal settings that would make this work appear to work well on the simulator but not on the device - you know this part though
    • The problem exists for coloured or white navigation bars
    • There is an issue with tableView, although I have not fully explored this
    • There is a jira related to this although I was not able to resolve the problem using the information provided.

    I have a work-around though. You use opacity on the barColor property.

    • Dark Red example; #99c60000
    • White example; #99ffffff

    To use opacity use a two character hex value before your normal hex triplet and be aware you must use the full 6 hex value letters - no simplied (so FFFFFF not FFF).

    #FFxxxxxx is no transparency, where as #00xxxxxx is full transparency. 

    "Window": {
    autoAdjustScrollViewInsets: true,
    color: "#FFF",
    backgroundColor: "#fff",
    barColor: "#99c60000", // this
    extendEdges: [
        Ti.UI.EXTEND_EDGE_TOP
    ],
    height: Ti.UI.FILL,
    includeOpaqueBars: false,
    navTintColor: "#fff",
    statusBarStyle: Ti.UI.iPhone.StatusBar.DARK_CONTENT,
    width: Ti.UI.FILL
},
    • note: Sorry if some of this you understand (colours for example) but you will not be the only one ready this.
