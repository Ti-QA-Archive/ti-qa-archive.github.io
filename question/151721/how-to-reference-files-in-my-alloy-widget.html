How to reference Files in my Alloy widget?

I built a Alloy Widget. The widget (for the moment) just loads a Webview located in assets. I just learned that I have to prefix the filename with the name of my widget. That's how the widget looks like:

<Alloy>
    <WebView id="webview" url="/testwidget/test.html"></WebView>
</Alloy>

My test.html webview contains a external js files such that:

<!DOCTYPE html>
<html>
    <head>
        <title>Test</title>

        <script src="test.js"></script>

    </head>
    <body>
        Hello
    </body>
</html>

the test.jsfile just shows an alert such that:

alert("Hey);

unfortunately the js file is never loaded, it is placed in the same folder as the test.html webview (/app/widgets/testwidget/assets).

  • Accepted Answer

    All local resources, including your test.js in this case, will need to be referenced relative to the widget directory. This is to ensure that there are no collisions when using multiple widgets, or collisions with your own project assets. You would need to modify your script tag to reflect this:

    <script src="/testwidget/test.js"></script>
