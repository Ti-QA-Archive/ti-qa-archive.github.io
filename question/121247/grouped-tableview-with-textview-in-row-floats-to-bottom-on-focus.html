Grouped TableView with TextView in row floats to bottom on focus

Here's the scenario:

SDK 1.6.2, building for 4.3 on iPhone

I have a grouped TableView that contains one row. The row contains a TextView.

When the app starts, the table is presented properly with the row being at the top of the screen. When the TextView receives focus, the keyboard opens and at the same time, the row moves to the bottom of the screen, just above the keyboard.

This only happens when using a grouped TableView with a control that will open the keyboard when it receives focus.

Code to replicate this behavior:

var win = Ti.UI.createWindow({
    backgroundColor:  '#cccccc'
});

var row = Ti.UI.createTableViewRow({
    height:50
});

var data = [];

var dataName = Titanium.UI.createTextField({
    height:35,
    left:5,
    width:150,
    hintText:  'test me',
    borderStyle:Titanium.UI.INPUT_BORDERSTYLE_NONE
});

row.add(dataName);
row.selectionStyle = Ti.UI.iPhone.TableViewCellSelectionStyle.NONE;
row.className = 'control';

data.push(row);

var editTable = Ti.UI.createTableView({
    data:data,
    style: Titanium.UI.iPhone.TableViewStyle.GROUPED
});

win.add(editTable);

win.open();

Is this behavior intended or is there a way I can prevent it? Many thanks!!

    I know that this behavior is normal for scroll views, and a table view also scrolls, so I guess they are similar. If you put this code for your table view, then it won't scroll down when you go to edit the box; it won't scroll down at all actually.

    var editTable = Ti.UI.createTableView({
    data:data,
    scrollable: false,
    style: Titanium.UI.iPhone.TableViewStyle.GROUPED
});
