IOS stopped building: SystemExit: 65

Hi All,

Yesterday my project was working perfectly on both iOS and Android. Today I refactored a couple of methods and tried to build and suddenly my IOS build is no longer working.     • Weirdest thing ever - I was playing around with my tiapp.xml file, and if I change the name field, the project will now build to the iOS simulator!

      Even adding a space to the name that was there "Pashash" -> "Pashash " seems to fix the issue.

      <name>Pashash </name>

      As soon as I revert the name the problem comes back! My guess is that the name element must somehow correspond to something that broke in xCode, and changing that makes it seem like a different project.

      I didn't pick this up when I started a new project and copied my resources accross, since I used the same name (even if I create an entirely new project, and change its name field to "Pashash" it refueses to build). Thanks for all your help Gaurang! If you post this as an answer, I'll mark it as correct, so others can see how it was finally resolved.

  • Finally found what the issue was for those that have this problem in the future. The issue was that inside my resource folder, I had a folder with the same name as my project name (project called Pashash, folder resources/pashash). This caused the xCode project to fail. The error the xCode process cited was :

    ld: can't open output file for writing: /Users/lawrencewebley/Library/Developer/Xcode/DerivedData/Pashash-fzgpnmlmabcarfgpjjxuvtvwygaw/Build/Products/Debug-iphonesimulator/Pashash.app/Pashash, errno=21 for architecture i386

    Some googling on this error finally led me to find that if their is a folder in your resource folder of the same name as your project, this error occurs.
