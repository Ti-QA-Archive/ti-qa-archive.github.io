Alloy Theme - User selected, dynamically switch

In the Alloy 1.2 update, the team released Dynamic Style, and from what others have said this functionality should allow for a user to be able to select a theme from the app/themes/themename area. The app would then update with the new theme styles and assets.

I'm working on a project where this would be extraordinarily helpful. All current documentation references changing individual styles on click actions but not the whole theme.

This question seems to ask the same question and, from what I can tell, ran into the same issues of not being able to get it done. Any ideas?

    Not having used Themes, what I understand from the documentation is that themes overwrite the main assets/styles in the resources folder before the app is compiled which means that the original styles are lost, any other styles are not included in the resources folder, so changing themes at runtime is not possible.

    It would be best to have separate classes instead of themes and apply those at runtime using the dynamic styling you mention.

    Hope this helps

    • This is accurate. Themes are currently determined at compile time. Dynamic styling does not support the wholesale changing of a theme at runtime, and I'm pretty sure no where in the docs does it indicate that this is expected functionality.
