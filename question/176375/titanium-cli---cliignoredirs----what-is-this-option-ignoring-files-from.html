Titanium CLI - cli.ignoreDirs -- What is this option ignoring files from?

I am trying to ignore some files and directories from being included in my build but no changes to cli.ignoreDirs seems to be used. In fact, it seems that the only directories being ignored is the one (.svn) that is hard-coded into the Alloy utils.js file.

Is there a different way to pass options to the alloy compiler using the cli?

    I've never played with the cli.ignoreDirs option, so I can't speak from direct experience. Perhaps by the time it would be acted upon, Alloy might have already copied all the files. While hacking the Alloy compile process might not be the best solution it is apparently effective.

    You could put in a feature request that Alloy should let you specify folders/files to ignore in the build process. It could honor the cli.ignoreFiles directive. Or, perhaps there should be a field in config.json in which you specify files & folders to ignore. I think I prefer the latter option because it would fit well with the workflow developers already use with setting themes, etc.

  • Well… while not the answer I was looking for, this solved my problem.

    ###Modify the file: alloy\Alloy\common\constants.js in your global npm_modules folder:

    // folders/files to exclude when copying and processing files
// RegEx format: must escape special chars - so use \.svn not .svn
exports.EXCLUDED_FILES = [
    '\\.svn',
    '\FOLDER_YOU_WANT_TO_EXCLUDE'
];

    I will add it to my regex replacement utility that I use to patch the sdk's and post it to the community.
