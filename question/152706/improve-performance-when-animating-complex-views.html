Improve performance when animating complex views

Hi,

i asked something similar a while ago but still have no satisfying solution.

The current state is:

I have, at least, two complex views (each has 10 Subviews, 1 ScrollableView, 50 Labels) and two animations (one for every view) running "in parallel" to get a custom transition between those views.

Both animations have a duration of 350ms. In simulator the time between starting the first animation and ending in the callback is around 360ms.
But on the iPhone 4 this time is between 600 and 1000ms. This leads to a very unresponsive ui feeling…

When performing the animations with empty views the duration gets near 400ms, which would be ok.

So i tried different things to speed up the animations, but couldn't find a real improvement.

  • I set both views to touchEnabled=false -> no big improvement.

  • I put all the content of the views in a wrapper-view and called $.wrapper.hide() before starting the animation. Here i was hoping for a big speedup, because the content doesn't need to be updated while animating, but it seems this has very little effect and i'm a little bit confused why.

Are there any tips how to speed things up? How do you handle such cases?

Thanks in advance, regards Joe.

PS: the app is alloy based and i'm using the latest versions of the sdk and alloy.
