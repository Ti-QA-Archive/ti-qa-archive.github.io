Monitoring Region on iOS

I need to monitor when the user enters a specific city while the app is not running. Some folks suggest background services, but that won't work if I understand correctly since background services will eventually be stopped. The native solution is described here: http://developer.apple.com/library/ios/#documentation/UserExperience/Conceptual/LocationAwarenessPG/CoreLocation/CoreLocation.html#//apple_ref/doc/uid/TP40009497-CH2-SW13

I've searched both Q&A here and SO, and basically my question is a duplicate question of this:
http://stackoverflow.com/questions/5616997/appcelerator-titanium-startmonitoringforregion which is >2 years old.

As the user on SO writes, there are properties "ERROR_REGION_MONITORING_DELAYED", "ERROR_REGION_MONITORING_DENIED", "ERROR_REGION_MONITORING_FAILURE" as described here: http://docs.appcelerator.com/titanium/latest/#!/api/Titanium.Geolocation, but at the same time there's no sign of capability to start monitoring region in the Appcelerator docs.

Perhaps there's a module on the marketplace that can do this? I searched but didn't find anything relevant.

Thanks.
