Projects Won't Build for Android

  • Application type: android mobile
  • Titanium Studio: 3.2.3.201404181442 - 4/18/14
  • Host Operating System: OSX 10.9.2 (Mavericks)

I recently upgraded to Mavericks. Here's the current situation:

  1. iOS always builds fine when "Run"

  2. Previously programed projects won't build for Android when "Run" from Titanium GUI

  3. All projects build for Android when "Run" from command line

  4. Opened a brand new project with no edits called "Testing" and it built for Android from Titanium GUI

  5. I replaced the entire resources folder of the "testing" app with a previously built app and it built fine for Android from Titanium GUI. (Yay!)

  6. I replaced the entire resources folder of the "testing" app with a the resources folder from the KitchenSink and the build failed within the Titanium GUI. (Ugh!)

  7. Made another fresh new project with no edits called "Test" and it no longer built for Android from Titanium GUI (Super Ugh!)

There are no errors and no console activity at all when the build "fails". It just doesn't attempt a build.

Instead it shows me the "Run Configuration" screen every time. I click run, the screen pops away and then pops back. I can build form the command line, but it's much slower for me and it's a lot of typing instead of just clicking "Run" like I'm used to.

Any idea about what could be happening here would be appreciated it. Hopefully I'm not the only one with this issue. Thanks guys.

  • Accepted Answer

    When you mentioned that Studio just doesn't attempt a build - I assume that the app is not launched at all. One of the potential reasons that can happen is when the RUN button can get retrieve the active selected project in the project explorer. Can you make sure by manually selecting any file/folder in the project and then, click the RUN button?

    Also, as you have pointed out, there seems to be a potential bug in Studio that if a project is renamed to an existing older project, then Studio still remembers the devices/emulators of the older projects. The only workaround for now is to restart the Studio to reload the new devices/emulators for the project. I have created a ticket for issue at https://jira.appcelerator.org/browse/TISTUD-6414 - please do add any additional info if required.

    • This comment led me to the solution. For the record, yes I tried everything including manually selecting the folder first. It was remembering all the old run configurations, which I had no idea it did. I also had no idea that there was a delete option for run configurations! So for all who come after me here's how it gets fixed.

      Delete your old run configurations one by one.

      After that, everything starts working again like magic. Everything except KitchenSink which apparently no longer works on any Droid emulator. :)

      Thanks Praveen!
