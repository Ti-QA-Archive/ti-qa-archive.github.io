zIndexing issue

Can you not set zIndexing with tss?

My alloy:

<View id="viewWrapper">
    <View id="view1">I am view 1</View>
    <View id="view2">I am view 2</View>
</View>

My tss

"#viewWrapper":{
    layout: 'vertical'
}
"#view1":{
    backgroundColor: 'red',
    zIndex: '1'
}
"#view2":{
    backgroundColor: 'green',
    top: '-5px',
    zIndex: '1'
}

When view2 is positioned top: -5px, it sits over top view1. That shouldn't happen in theory

  • Hi

    Your hack is not really hack as the experienced behaviour you found originally is by design. Layout property trumps other positioning entries.

    Personally I never use the zIndex property, partially as this is an attempt to shoehorn web css into mobile but mostly due to managing the layouts myself. If you need one thing over another - then place the bottom entry before the next layer and so on.

    This can easily be performed in apps and correct use of one or more parent views to group two or more layers is the right way to go.

    BTW the zIndex property is a number and not a string.

    "#view2":{
    backgroundColor: 'green',
    top: '-5px',
    zIndex: '0' // with your update
}

    Should be

    "#view2":{
    backgroundColor: 'green',
    top: '-5px',
    zIndex: 0 // with your update
}

    I would also suggest you take a look at your explicit dimensioning using 'px' as it is not the most efficient method as most devices these days are using densities greater than x1. So '5px' on iOS retina or higher will mean uneven positioning as 5px on retina is twice as small as on non-retina layouts. Take a look at the following links and focus on the dp elements.

    • How to achieve same layout for Android and iPhone using Titanium
    • images in Android
    • Images path - Android vs iOS (Alloy)
