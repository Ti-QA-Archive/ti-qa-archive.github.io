tableview and allowsSelection

Hello everyone, I make a part of my app into a tableview with a single row to make possible the pull to refresh method like the kitchen sink example.
Into the tableview there are 3 view with differents background color.

Now, I added the property allowsSelection:false to my tableView because I hate the change of color when the inside view is tapped by the user.
I tryed with backgroundColor:&#39;transparent&#39;, SelectedBackgroundColor:&#39;transparent&#39; etc.. but I can't succeed to avoid this noise.

My actual problem is this, with the allowsSelection:false the user can't press any button or make any action in the elements in the tableview.

So is there a way to bypass this problem?
I'm searching 2 option:

  1. Make buttons 'clickable' if the table view is set to allowsSelection:false

or

  1. Disable this option and succeed to avoid the change of color in the single row with 2+ views inside.

Thank you, Stefano.

  • ok I found the answer myself, the DOC is not very helpfully.. if someone interested set
    selectionStyle:Titanium.UI.iPhone.TableViewCellSelectionStyle.NONE instead of a row property
