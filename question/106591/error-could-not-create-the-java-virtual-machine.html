    Jason

    Actually, this is starting to make sense. I have a feeling your software configuration is correct.

    However, the error you see is:

    [ERROR] Could not create the Java virtual machine.

    This is generated after the following command, which specifies a heap space of 1GB:

    [DEBUG] "C:\Program Files\Development\Java\jdk1.6.0_23\bin\java.exe" -Xmx1024M ... [etc]

    Hence, how much physical RAM do you have installed on your system? If it's at least 1GB, how much is available (free) after the emulator is booted?

    • Jason

      OK, this and this information explains the reason for the issue, and it is not a Titanium-specific one.

      There are a number of things you could try. The results of each of them would probably be useful for other users, if you are prepared to do it.

      • restore the original builder.py value, configure the WinXP /3GB switch, explained here, in boot.ini, explained here, reboot your machine and try to launch the app. If it doesn't work, reboot without the /3GB switch (actually, from the depths of my memory I seem to recall that SQL server requires the /3GB switch anyway)
      • the heap memory must be contiguous, as explained in one of the posts on stackoverflow. Install Process Explorer to work out which applications, dlls, and drivers are not using your memory efficiently (are occupying the middle of the address space), and consider removing them
      • incrementally increase the Xmx value in builder.py, testing your app with each value, until you find the "limit" where the app stops working

      Not fun, admittedly, but interesting nonetheless! :)
