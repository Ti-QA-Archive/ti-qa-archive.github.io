  • Accepted Answer

    If you are planning on distributing your application via TestFlight you will need an Ad-Hoc provisioning profile generated through your Apple Developer account.

    Every application I build has at least 2, often 3, different provisioning profiles assigned to them.

    1. Development
    2. Ad-Hoc for outside testing
    3. Distribution

    • No, I believe each apple developer account is restricted to 100 devices per year. If you remove some devices, those devices are no longer able to install the app but you still can't add more devices. To get past 100 devices, I think you need to use another developer account.

      An alternative for those who qualify is an Enterprise Developer account. Enterprise profiles are similar to ad-hoc distribution profiles but with two major differences:

      • It can be installed on an unlimited number of devices
      • The profile does not need to contain the UDIDs

      In essence, an app with a valid enterprise profile can be installed on virtually any iphone/ipod using testflightapp or other OTA means, or iTunes or whatever.

      The caveat is that the Enterprise license is designed for internal use by a corporation. It cannot be used for general distribution outside of a qualified enterprise setting. You can read more about it here.
