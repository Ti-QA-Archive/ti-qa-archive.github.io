Brandable iPhone app

Hi, I'm about to embark on my first iOS/Android app using titanium and before I get started I'd just like to verify that a couple of things will be possible using the platform:

1) My client requires that I create an app that will be brandable/configurable by various different businesses, to use as their own. Is it possible to do this by, for example, dynamically creating a different app key depending on which business is using the base app?

2) If this is possible, will apple allow the various users to add their branded app to the app store. Or would they consider this spam because all the apps are essentially the same/similar?

Thanks in advance for any advice or pointers.

Carl

  • Accepted Answer

    There are plenty of horror stories around about this. Apple will eventually pull your app as essentially being spam if you keep on re-releasing it rebranded but with the same functionality. Apple insist that either you use a main app and add the other brandings as "in-app" enhancements (usually not possible), or you ensure that each rebrand of the app is done under a separate IOS developer license, paid for by each fo the rebranding customers.
