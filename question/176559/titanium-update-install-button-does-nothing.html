  • Hi,

    some good news

    I have been able to make the installation (no more install button doing nothing). I don't know what was the exactly step that it made it work but here is my procedure:

    1) Reinstall titanium (maybe is not necessary)
    2) My problem seemed to be a node / npm conflict. So I clean my system (ubuntu 12) from them… I followed some guide of some page… maybe this: http://gangmax.me/blog/2013/05/05/how-to-remove-nodejs-manually-from-ubuntu/
    3) I execute titanium like this:

    sh ~/TitaniumStudio/TitaniumStudio.sh

    and installed nodejs using these commands:

    sudo apt-get install python g++ make
    mkdir ~/nodejs && cd $_
    wget -N http://nodejs.org/dist/node-latest.tar.gz
    tar xzvf node-latest.tar.gz && cd ls -rd --color=no node-v*
    ./configure
    sudo make install

    (this was taken from http://jetsmi.blogspot.com.es/2013/05/instalar-titanium-3x-appcelerator-en.html but is in Spanish).

    So… finally I did the installations, but now my titanium seems not recognize my android sdk… i am working on it…

    good luck!

  • I had Titanium Studio working fine on Ubuntu 14.04, but stumped upon this problem too after I had to do a fresh install of Kubuntu. After several days, and many installs of different flavors of Ubuntu, trying to get it to work I finally got all the necessary steps together to fix all the issues, so I decided to write my own walkthrough from start to end, to maybe help people with the same problem.

    1. Uninstall OpenJava:
      Titanium Studio won't work with OpenJava, so you have to uninstall it and install Oracle Java instead.
      In a terminal window, type

      java -version

      If you it shows that OpenJava is installed type this to remove it:

      sudo apt-get purge openjdk-\*

    2. Install Oracle Java:

      sudo apt-get install software-properties-common
sudo add-apt-repository ppa:webupd8team/java
sudo apt-get update
sudo apt-get install oracle-java7-installer

    3. Install Node.js:

      sudo apt-get install nodejs

    4. Symlink nodejs executable as node:
      Type node -v in a terminal. If you get command not found, type this:

      sudo ln -s /usr/bin/nodejs /usr/bin/node

      This will make sure TS can find the node executable.

    5. Install npm:

      sudo apt-get install npm

    6. Install jpeg and and webkitgtk libraries:
      If you open TS now, you'll get an error about the internal browser and will be suggested to close it. Type this in a terminal to fix it:

      sudo apt-get install libjpeg62 libwebkitgtk-1.0-0

    7. Install dependencies:
      If you open TS, this time it will ask you to install some updates, and here's where it gets locked. If you click the Install button nothing happens.
      So close TS and type these lines in a terminal:

      sudo npm install -g titanium
sudo npm install -g alloy
sudo npm install -g acs

    8. Install Titanium SDK:
      Open TS now and it will show you again the update window with only the Titanium SDK, but this time the Install button should work, so click it.

    9. Install Android SDK:
      After installing the Titanium SDK, TS will prompt you to configure Android SDK.
      Click the settings button, choose a folder, check the versions you want and install them.

    10. Add Virtual Devices and link avd folder:
      Now in TS go to Window > Preferences > Studio > Platforms > Android.
      Click Manage AVDs… it will open Android AVD Manager.
      Add the devices you want and try to start one of them, you'll probably get an error like this: PANIC: Could not find emulator.ini file in $ANDROID_AVD_HOME nor in $HOME/.android/avd.
      Setting the android avd home variable didn't work for me, so I had to take a workaround.
      In a terminal try to move to the .android folder in your home folder:

      cd /home/username/.android

      If the directory doesn't exist yet, try to execute android manager with your user (without sudo):

      /your_android_sdk_path/tools/android

      Now you have to access your root folder and symlink the files in the avd folder with the one in your home folder:

      sudo su
cp -s -d -R /root/.android/avd/* /home/username/.android/avd/

      Now the virtual devices should be starting normally.

    11. Check if it compiles:
      At this point TS was fully working on Ubuntu 14.04, but for Kubuntu 14.10 I had to take some addicional steps.
      First thing you'll notice is that the list of emulators in TS won't show the AVDs you've created, but a list with all of them instead.
      Trying to run a project might get you an error saying that Titanium SDK is not installed (even though it should have done it), so you'll execute the command to do it manually. I did this one especifically from a terminal within Titanium studio (Commands > Shell Script > Open Terminal), just to make sure:

      titanium sdk install

      Restart TS and it should be showing the correct AVDs now.

    12. Install x32 libs:
      If you're using a x64 Linux, you might get one last error when trying to run a project. Just run the command it asks you to on a terminal window:

      sudo apt-get install libc6:i386 libncurses5:i386 libstdc++6:i386 zlib1g:i386

    After all that I could, at last, develop with Titanium Studio on Kubuntu.
    Hope it helps some of you.
