  • Allright I'll give you a hands-on the way we do it currently:

    1 - Setting up push notifications

    Use the guide given by Raju to setup push notifications. Though you can skip the whole user login stuff, just retrieve the device token instead. That's enough.

    For Android push notifications is kind of buggy due to a few bugs in the ti.cloudpush module. I found a stable version at 3.3.5 for now.

    Remember the channel you're subscribing to, you need this on your server

    2 - Sending a push notification from your server.

    • Create an admin user on your ACS panel in Manage ACS in my.appcelerator.com/apps (It's needed to use the everyone option on notify_tokens)
    • Remember the login and password and store them somewhere on your server where you want to send push notifications from.

    Do the following curl requests (Can also be done with http post) when you want to send an update, with <appkey>, <login>, <password>, <channel> being your own data.

    1. CURL Post Admin login

      $ curl -b cookies.txt -c cookies.txt -F "login=<login>" -F "password=<password>" "https://api.cloud.appcelerator.com/v1/users/login.json?key=<appkey>&pretty_json=true"

    2. CURL Post Notify Tokens

      curl -X POST -F "channel=<channel>" -F "to_tokens=everyone" -F "payload=My Pushnotification Data, which can also be an object" "https://api.cloud.appcelerator.com/v1/push_notification/notify_tokens.json?key=<appkey>&pretty_json=true"
