Connect to soap web service through proxy

I am writing an app that needs to communicate with (not one, but two!) web services that are sitting behind a proxy login.

Unfortunately I have no say in the matter regarding the SOAP services, they are already written and being used for the web based version of my app (a reporting app for a local council).

I have read dozens of blog posts and forum comments about this, and so far have not been able to find one that addresses my issue.

When I try and connect to one of my services with suds.js my app fails and the debug console fills up with the HTML from the login form that you get to if you try to browse to a page behind our proxy from the web.

How do I pass a username and password to authenticate past the proxy so I can use the web service?

my last attempt:

var suds = new SudsClient({
    endpoint: url,
    targetNamespace : 'defaultNameSpace',
    authorization: 'myUserName:myPassWord@domain:443'
});

Any help would be greatly appreciated.
