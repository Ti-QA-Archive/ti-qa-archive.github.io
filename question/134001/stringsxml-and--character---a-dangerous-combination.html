strings.xml and ":" character - a dangerous combination

Just noticed that for some reason if you include : in a word in the strings.xml file, you cannot get the string converted on an Android emulator, it returns an empty string. E.g. "Username:" does not work. Moreover, even if you delete ":" character, you must delete the Android build directory and then recompile to get it working.. Not sure why it works like this but it is what it is.. Just wanted to prevent time loss in case someone bumps into the same problem.
