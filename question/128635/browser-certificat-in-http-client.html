  • Here's a few questions which might help us find a solution:

    • What communication are you trying to secure? Frequent? Highly confidential? Financial?
    • Which end needs to know who they're communicating with? Your devices need to know they're talking to YOUR server? Your server needs to know it's talking to a particular device?

    If you want to guarantee that you're communicating securely and that you're actually talking to your web service then you can use regular SSL (certs on your site) and make sure you get the phone to verify the certs received from the server. This is what web browsers do as standard. If you need the server to verify who the device is (like in the SSH protocol where both ends check each other's certs) then you'll need to install a cert onto the device via some other channel. You probably don't need to.
