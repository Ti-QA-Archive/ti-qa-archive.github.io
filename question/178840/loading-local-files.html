  • Hi

    I am always surprised when people come here and decide to blame (and in your case insult) the tools they use - then ask for assistance. "…why titanium is pulling this out of their butt…"

    In your case you have misread the error message as it does not say what you think it does. It does not suggest line 8592 is in your code but in the code that is processing your code. This sort of error message where it needs to refer to itself is uncommon - mostly because this sort of error is also uncommon. It is rare that this sort of value is ever provided as people at this level of capability have normally read the docs for the example and would then not have made such a strange leap towards providing a value that is simply bad.

    Your question provides this as the location to your source file;

    app/Resources/web/streaming.html

    To avoid confusion in the future it might be best to adopt this format when providing path details. Yours is fine but could be confused for the top level folder app used in an app benefiting from the Alloy framework.

    {appname}/Resources/web/streaming.html

    Given your app root runs from;

    {appname}/Resources/

    As do your files; so you exclude any references below that point as they are already known to Titanium. You only provide the details relevant to those things you have provided.

    So as has been stated in the question comment the path you use should be;

    url: '/web/streaming.html'

    All questions welcomed - but consider holding off on the blame and insults until you have determined something is wrong and only then if once telling Appcelerator via the jira ticketing system. they do not fix it. Anything up until this point it is; not a bug, if it is a bug it is not a known one and then only if it becomes an ignored bug.
