    • I do like this:

      // app starts
// but this is the first thing I do, no window opening, nothing.
// this way until the app opens, we have better  chances to have a gps signal.


Titanium.Geolocation.addEventListener('location', do_the_stuff); 

getCurrentPosition()  //get the cached position and do_the_stuff until the first location event triggers.

      You might want to set accuracy to best and distanceFilter to 0 to force the device trigger as fast as possible. After the first location event you can change the distanceFilter to a bigger value to save some battery.

  • You cannot influence the accuracy directly. The ACCURACY_BEST constant will Tell the GPS Sensor to work more precise (needs more Power) but in fact it will Take some time until the GPS Sensor established the Connection with satelites. More of them means better results, at the very start youll get the estimated coords through the cell Towers (triangle).

    You could out a loading indicator and Tell the User that the los is bring aquired and internally you listen to the lication Event and check the acciracy of the location Events.
