    There are few possible reasons to run into this kind of situation.

    AFAIK - Check for your Java runtime environment(In terminal type > java -version). It should be 1.6.0_51(I am running on this) or less. Since, titanium doesn't support 1.7

    Also, I heard about this may leads because of Python build. Where the Python version should be less than 3.0. where in my case it was 2.7.2 (In terminal type > python) installed.

    Still if you got into problem - here are few more suggestions that you should look into it
    http://developer.appcelerator.com/question/152497/titanium-sdk-310-error-typeerror-argument-of-type-nonetype-is-not-iterable-on-building-android-app#comment-175782

    • I have never been able to get to the android virtual device manager through the home screen or dashboard. If you search in spotlight for "monitor" it should find the "android device monitor", sometimes it just shows as monitor but it is in your android install folder.

      If you launch that you can then go to your virtual device manager and sdk manager, they are both under the window menu.

      Finally if you are using the logcat monitor make sure to add a filter of "TiAPI". It will make the stream of info much more digestible.
