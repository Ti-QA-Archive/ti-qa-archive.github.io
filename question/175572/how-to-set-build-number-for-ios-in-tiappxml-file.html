How to set Build number for IOS in tiapp.xml file?

Hi,
For Android, we set the app Build number using VersionCode attribute as follows.

<android xmlns:android="http://schemas.android.com/apk/res/android">
<manifest android:versionCode="1016">
<uses-sdk android:minSdkVersion="14"/>

We manage the app version with Version number in tiapp.xml.

I could not find any attribute to set the Build number for IOS. But IOS needs the version number and the Build Number.

Can you help me in this?

Regards,
Chella

  • Hi,
    Thank you for your quick reply. I have already gone through the docs and got the below information: This says that CFBundleVersion is auto generated and cannot be overridden in tiapp.xml file. In IOS, CFBundleVersion = Build number. Currently, CFBundleVersion is auto generated with Version number specified in tiapp.xml. It means, there is no possibility to set the BuildNumber for IOS from Titanium, which is different from Version number in tiapp.xml.

    Thank you so much for confirming this. If there is any contradictory in my understanding, you can comment on this.

    Conclusion:

    I will provide this information to my client stating that the BuildNumber for IOS cannot be manually set through Titanium. It gets auto generated with the Version number specified in tiapp.xml file.

    Unsupported Info.plist Keys

    The following Info.plist properties are ignored in the <plist> section. The values for these properties are automatically generated by Titanium and may not be overridden in the tiapp.xml file or custom Info.plist file.

    CFBundleDisplayName. The name in the generated file always reflects the value of the <name> element
    CFBundleExecutable
    CFBundleIconFile
    CFBundleIconFiles. Automatically includes all application icon files from project directory.
    CFBundleIdentifier
    CFBundleInfoDictionaryVersion
    CFBundleName
    CFBundlePackageType
    CFBundleSignature
    CFBundleVersion
    CFBundleShortVersionString
