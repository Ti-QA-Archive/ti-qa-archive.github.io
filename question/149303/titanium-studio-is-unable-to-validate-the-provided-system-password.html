Titanium Studio is unable to validate the provided system password

Titanium SDK: 3.0.2.GA?
Host Operating System: OSX 10.8.2
Titanium Studio: Titanium Studio, build: 3.0.2.201302191606
Build: jenkins-titanium-rcp-master-372 (origin/master)
Date: 19 February 2013, 16:09:49

For 2 months now I keep getting this error when I launch Titanium and it wants to install two updates being "Titanium CLI 3.0.24" and "Alloy 1.0.0".

I click "Install" button and it says "Titanium Studio wants to make changes. Provide your system password to allow installation." Under "User:" it says Lou and is greyed out and I can not change it. It is "Lou" as a Standard user user and it is "Lou Cipher (Admin)" as the Admin. However, I work under a Standard user account not an administrator account for security. So I try and enter my Standard user passord and it does not work at all.

Then this screen pops up:

"Titanium Update"
"Titanium Studio is unable to validate the provided system password.
Please provide the system password in the terminal view to proceed with installation."

So, under the terminal view it shows the following:

lou-ciphers-mac-pro-2:Titanium_Studio_Workspace Lou$ sudo npm install -g titanium alloy

WARNING: Improper use of the sudo command could lead to data loss
or the deletion of important system files. Please double-check your
typing when using sudo. Type "man sudo" for more information.

To proceed, enter your password, or type Ctrl-C to abort.

Password:

I try and enter my Standard user password giving me the following error message:

Lou is not in the sudoers file. This incident will be reported.
lou-ciphers-mac-pro-2:Titanium_Studio_Workspace Lou$

When I try and enter my Admin user password I get three tries then this message:

Password:
Sorry, try again.
Password:
Sorry, try again.
Password:
Sorry, try again.
sudo: 3 incorrect password attempts
lou-ciphers-mac-pro-2:Titanium_Studio_Workspace Lou$

So, it seems it does in fact want the Standard user password but says my account is not in the sudoers file.

Any advice…??

If I have to run a certain instruction, what is the exact directory that I am supposed to run the instruction from…? Do I go to Terminal and cd to a specific directory and run a specific command…?

Please advise as I would love to get started programming - have not programmed in 17 years and I am so out of date and awkward at this point. Please help stabilize and get me going :)

Thanks in advance,

Joe

  • I was also stuck on this issue on a new Mac with OS X 10.9.3 (Mavericks). The chown suggestion on this thread did not work for me, possibly because the /usr/local folder was moved between versions of the OS. There was no /local/ folder under /usr/ on my machine.

    However, I came up with a workaround that solved it for me. Apparently I never set a Master Password for my Mac. Here's how I solved the problem.

    1. Go to Apple menu > System Preferences > Users & Groups.
    2. Click on "Click the lock to make changes" and provide your current user password.
    3. Click the tiny gear icon (below Login Options on the left) and select Set Master Password.
    4. Enter the password you want to use. Click OK on all prompts.
    5. Next time you try to run Titanium and try to install everything (node.js, Alloy, etc.) the installers should work.
