  • Hi,
    you are probably accessing the wrong instance of your database file. Indeed, when you call Ti.Database.install you are actually copying the file in the user directory for the app, where every modification takes place.

    What about performing a select from your app code and printing records on the debug console?

    However, on Android you can also directly access the db file installed in the emulator as explained here: http://stackoverflow.com/questions/2531540/how-mount-userdata-img-or-userdata-qemu-img-in-osx

    With the iPhone emulator is kind of simpler, since your database resides on the MacOS filesystem in a directory that looks like: ~/Library/Application\ Support/iPhone\ Simulator/4.2/Applications/<app identifier>/Library/Application\ Support/database/db.sql
