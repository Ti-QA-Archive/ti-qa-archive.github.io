EmailDialog not working if no mail accounts is configured

Hey guys, I have a number of test users who said that my email functionality (a EmailDialog) is not working. They don't seem to be using the built-in email client functionality of iOS, but rather prefer to mail by using mailto: and going to their gmail account in safari.

Is there a way for me to detect wether I should trigger an EmailDialog or wether I could just send them to a mailto: link?

    Hi

    To test if email dialog is supported you simply test with isSupported

    var emailDialog = Ti.UI.createEmailDialog();
alert(emailDialog.isSupported());
