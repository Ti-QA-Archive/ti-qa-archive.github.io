Android backbutton navigation

I know this was asked before several times and I did my homework to try all proposed solutions with Titanium SDK 3.4.1GA. :)

I've got 3 windows (A, B and C) in my application with a menu to switch between them.

I want backbutton to behave exactly as back button in browsers. For example if I navigate :

A->B->C and start pushing backbutton I get C->B->A->closeApplication.

Or more complex example A->B->C->B->C navigation path should reverse to C->B->C->B->A.

What happens now is that no matter what was my navigation path I get to the starting screen (A).

This is the code of opening windows:
Alloy.createController(&#39;SecondWindow&#39;).getView().open();

This is the definition of window

<Window  exitOnClose="false" navBarHidden="false" layout="vertical">

The spec mentions something called task stack.

From backbutton event description: By default this event would trigger the current activity to be finished and removed from the task stack.

Is there a standard way (API) to add a window to this stack? Or should I implement the stack myself?

    the task stack isn't something you can control… it's the order in which Activities (what a window is called in Android) are opened. The only way to "manipulate" the stack is to open or close windows.

    Hint: You do not always need to close the top-most window. If you're keeping track of your windows somewhere, you can close something other than the current window.
