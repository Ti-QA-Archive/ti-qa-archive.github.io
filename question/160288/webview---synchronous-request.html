Webview - Synchronous request

Webviews in iOS 7 do not support making synchronous ajax requests. This issue was brought up on StackOverflow and a solution suggested. Does Titanium support a way of exposing native code to javascript running in a webview?

  • Using of webview in titanium is a bad pattern. OK. synchronous calls are not supported … You can make synchronous calls from titanium (only under iOS) and you can communicate between titanium namespace and the webview. WTF: why do you need sychronous call?

    Rainer

    • We are using a webview because the app is basically a wrapper around a collaborative editor we've built using HTML5 technology. See http://www.lucidchart.com/demo. The reason the call needs to be synchronous is because at anytime a shape can be added to the document (from a copy and paste, or another collaborator). If the shape belongs to a plugin that hasn't been previously loaded, it needs to be loaded before the drawing can occur. By the time we know the plugin hasn't been loaded we are already several layers deep into stack calls, hence the need for a blocking call. This only became an issue in iOS 7.
