  • Alberto

    What happens when you using Titanium SDK 1.4 rather than 1.5+? If it works, then you have probably encountered a common problem between java and Windows XP.

    Java requires contiguous (non-fragmented) memory (and Titanium 1.5 needs 1GB, as opposed to 1.4 which only needs 512MB) while Windows XP has very poor memory management capabilities exacerbated by a memory limit of 2GB (less overhead) for applications.

    The ideal solution would be for you to upgrade to Windows 7, which manages memory better. If this is not possible, you need to clean up your runtime memory, by preventing unnecessary applications and services from running at startup (it's not sufficient to stop them after the OS has loaded, because this may not necessarily free memory properly).

    It would be interesting to know how you get on with this.

    Cheers
