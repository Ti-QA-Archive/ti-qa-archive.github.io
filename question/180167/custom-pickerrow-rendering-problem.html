  • Little update:
    it seems that the problem is on picker width property. If you don't set it and you try to add only a single column to picker like this code

    var picker = Ti.UI.createPicker({
        top: 20,
        columns: [column1],
        selectionIndicator: true
});

    The problem persist. If you force width property to a fixed width like 

        var picker = Ti.UI.createPicker({
        top: 20,
        width: 200,
        columns: [column1],
        selectionIndicator: true
    });

    and set the textAlign property on label that will be added to a picker row to

    textAlign: Ti.UI.TEXT_ALIGNMENT_CENTER,

    the problem is solved.

    It's a known bug? I have to open an issue on jira?
