no initial regionchanged / Android maps v2

No regionchanged-event is fired when a mapView is opened for the first time, while this is the case for iOS5+6 + Android maps v1. I'm not sure if this is a bug or a feature in v2.

A workaround is a fireEvent(&#39;regionchanged&#39;) after setting up the mapView. But then all lat/lng/deltas are undefined, until you manually zoom/swipe. A workaround for this again, is to read the region-properties, which hold the initial lat/lng and deltas. If someone has a more elegant solution, please let me know.
