Facebook Login stopped working, more people with the same experience?

Hi Guys,

I love appcelerator and us it for a Iphone app which connects to Facebook.

I run Ti.version.1.5.1 against an old 3.1.3 iOS SDK, and had a working Facebook connect / login button (Titanium.Facebook.XXX) button. I implemented this Facebook connect part over a month ago and had it running without any problems, it was really cool to see it working, sharing stuff on my wall. Yeaj!

Problem

However now I find that my FBlogin is not working anymore, and I didn't change anything about this specific code, nor the iOS SDK, nor the Ti.version (1.5.1) .

Visually it looks like the Connect menu pop's up, shows the loading indicator (depending on which Ti.version) and closes again. (At that point the Ti.Facebook.createLoginButton fires a 'Cancel' event)

I reckon it has something todo with recent API changes on the Facebook side (around 7 januari 2011). However while debugging I noticed that FBlogin kept on working in Kitchensink, even after rebuilding it.

Here is my all to regular code snippet, which used to work:

fbButton = Titanium.Facebook.createLoginButton({
        'apikey':'<APP ID>',
        'sessionProxy':'<SESSION PROXY>'
    });

I also tried it with the less secure method, which obviously also used to work.

var fbButton = Titanium.Facebook.createLoginButton({ 
    'style':'normal', 
    'apikey':'<APP ID>', 
    'secret':'<API SECRET>'
});

Just mentioning this again: both methods worked like 1 month ago or so.

What did I do to debug:

  • Tried switching to the Kitchensink API credentials: did not work
  • Rebuild the Kitchensink App: FBlogin does still work
  • Changed the Kitchensink fbcredentials to my app and another rebuild: FBlogin does work (!)
  • Created an empty new project (with the regular default files) in Titanium with my app's FBcredentials only running the FBloginbutton: did not work
  • Changed credentials in this empty project to Kitchensink's FBcredentials: did not work either
  • Deleted the APPDIR/build/iphone/build/* contents as well for almost everything

In short: pretty weird. In the old Kitchen Sink it keeps on working.

My questions

Would be great if you have any experiences here, or tips on how to debug more…

The FBlogin / Titanium Developer App doesnt return any clues in the logfield. (even on 'INFO' level it doesnt return any information)

  1. Do you have any tips on further debugging or maybe had the same experience?
  2. Are there any logs written on the backside by Titanium
  3. I'm on mac, do you have a tip on good network monitoring software, to get some clues on what's happening between the Iphone Simulator and Facebook site?

Thanks very much in advance,
Would love to finish this app. and show it to you all!

Greetings from Amsterdam, Roeland
