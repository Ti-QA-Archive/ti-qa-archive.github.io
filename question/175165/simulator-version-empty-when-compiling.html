Simulator Version empty when compiling

Hello,

I have a fresh simple installation of XCODE and Titanium.

Ti -version = 3.2.3
Xcode = 5.0.0

My Xcode installation is /Applications/Xcode.ap
My /Developer directory is EMPTY ?? (is that normal?)

When I go to compile I go to the following:

–> RUN MENU –> Debug Configurations –> Titanium iOS Simulator (Side Menu)

The Settings section displays the following:

* Project Name: My Project
* Settings:
iOS SDK = 7.0
Simulator Version = EMPTY
Device Profile = iPhone Retina
Log Level : Info

Why is my simulator version empty?

I restarted my Mac so many times, I've tried everything, the simulator version is still empty.

I did a quick unix find for the term "simulator" , and I found the following:

/Applications/Xcode.app/Contents/Applications/iPhone Simulator.app/Contents/MacOS

in MacOS directory, there's a binary called iPhone simulator; but that's all I have. Is there anything missing?

Many thanks for any support you can give,

I'm really stuck.

  • OK - I got the answer

    Here's what happened.

    Type:

    xcode-select -version
xcode-select -print-path

    This information showed me that the path where my Xcode was set did not exist. I changed it to where the Xcode directory was.

    I hope this helps someone else.
