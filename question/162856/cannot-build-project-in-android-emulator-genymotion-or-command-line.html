Cannot Build Project in Android Emulator, Genymotion, or Command Line

I have recently updated to version 3.2.1.201402041146 on my Mac. I have tried running my project on an Android emulator but continue to get the following message:
[ERROR] : Failed to install apk on "emulator-5554"
[ERROR] : Error: INSTALL_FAILED_MISSING_SHARED_LIBRARY
[ERROR] Application Installer abnormal process termination. Process exit value was 1

I verified that I have ALL on the Android SDKs installed and even created a new AVD but still end up with the same error.

I tried going through the steps to make the Android emulator faster as outlined here: http://www.appcelerator.com/blog/2013/07/speed-up-your-android-emulator/. I hit some additional bumps, but once I got that all squared away, I tried running my project again and got the same error.

I have tried cleaning my project then running again, but still no luck.

So, I finally broke down and followed the steps to get Genymotion set up as outlined here: http://www.appcelerator.com/blog/2013/12/using-genymotion-with-titanium-3-2/

I have my Genymotion emulator showing up as a device in Titanium, but whenever I try to run my project, I get the following error:
[ERROR] : Failed to install apk on "192.168.56.101:5555"
[ERROR] : Error: INSTALL_FAILED_MISSING_SHARED_LIBRARY
[ERROR] Application Installer abnormal process termination. Process exit value was 1

Very similar to the one before.

One more bit of information, if I try to build my project using the command line and Genymotion emulator, I get an error that says [ERROR] Invalid "–project-dir" value "." I'm not sure if that is related or not, but thought I'd throw it in there just in case.

Any help is greatly appreciated.

    [ERROR] : Error: INSTALL_FAILED_MISSING_SHARED_LIBRARY

    This usually means that the google APIs are not installed on the emulator image.
