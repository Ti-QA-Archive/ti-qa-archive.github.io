Android Map View Event Listner

Hi

I will try and explain this to see if anybody has come across this before.

I have a map view with annotations and on each annotation I add an event listener for when selected. This displays an additional panel with info in.

On iPhone this works exactly as I would expect.

On Android the event listener is not triggered, or is not processed.

On both platforms the annotation title is displayed correctly.

Currently using Mac with Titanium Studio and SDK 1.7.1. Android SDK google API 2.3.3

var annotationArray =[];

for (iPos = 0; iPos < getVouchersByLocationCallDataIn.length; iPos++)
{
var tmpID = getVouchersByLocationCallDataIn[iPos].VOUCHERLOCATIONID;
var voucherItem = Ti.Map.createAnnotation
({
latitude: getVouchersByLocationCallDataIn[iPos].LAT,
longitude: getVouchersByLocationCallDataIn[iPos].LNG,
title: getVouchersByLocationCallDataIn[iPos].NAME,
pincolor: Titanium.Map.ANNOTATION_GREEN,
ID: tmpID
});
voucherItem.addEventListener('click', mainNS.view.showSelectedMapFunc);
annotationArray.push(voucherItem);
};
var voucherMapView = Ti.Map.createView
({
mapType: Ti.Map.STANDARD_TYPE,
region: {latitude: globalCurrLat,
longitude: globalCurrLong,
latitudeDelta: 0.00095,
longitudeDelta: 0.00095},
animate: false,
regionFit: true,
annotations: annotationArray,
userLocation: false
});

To clarify all data is returned correctly.

  • Accepted Answer

    Try adding the event listener to the mapview instead, and then check for specific annotation info in the event data:

    mapview.addEventListener('click', function(e) {
    Ti.API.info("CLICKED: "+e.annotation.ID);
});
