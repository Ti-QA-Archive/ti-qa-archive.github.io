Loading index.js always shows the splash screen

Hi all,

I'm using 3.1.3 with Alloy. I've got the index.js to load another controller, and that controller's back button will load back the index.js controller. Code is as follows:

$.page2.addEventListener('android:back', function(){
    var index = Alloy.createController('index').getView();
    $.page2.close();
    $.page2 = null;
    index.open();
});

It works, but loading index.js will always show the splash screen before rendering the window. Is there any way to disable the splash screen after the initial 'splash'?

Alternatively I can consider having index.js always load another controller straight away. But that doesn't sound like the right thing to do…

    windows are stacked one after another sequentially - the last one is displayed at the top.

    That way, you do not need to open the root window - index.js again when closing the page2 win.

    • Open page2 DOES NOT close the index. Instead, page2 is pushed into stack and shown on the screen. Once page2 is closed, it is popped out leaving the index displayed.
