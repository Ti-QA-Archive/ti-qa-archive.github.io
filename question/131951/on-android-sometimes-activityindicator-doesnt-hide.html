  • I fixed this by setting the activityWin.show() in a setTimeout. This is solution have used for other titanium problems. I'm guessing that the Ti.UI.currentWindow on android is changing faster than the framework notices, and the framework gets confused about which window it actually added the activity indicator to.

    Setting it as:

    setTimeout(function() { activityWin.show() }, 200);

    should solve your problem.
