DIP not working correctly on Genymotion Emulator

dp is my default system unit.

in my tiapp.xml (not supporting tablet)

<supports-screens android:anyDensity="true"
                android:largeScreens="true" android:xlargeScreens="false"/>

Things look fine on devices, but not Genymotion emulators. On an xhdpi emulator, views with width/height specified to a number look much smaller. It seems "dip" doesn't work on Genymotion emulator.

Running on SDK 3.3.0. Any idea?

  • Hi

    Can you confirm that your tiapp.xml file has the following entry with the following setting?

    <property name="ti.ui.defaultunit">dp</property>

    Also fine

    <property name="ti.ui.defaultunit">dip</property>

    They are the same thing, I prefer the first as it is less to type.

    Can you also confirm that you are using numeric dimensions and not sting sufficed versions;

    "#something": {
    width: 100
}
// not
"#something": {
    width: "100dp"
}

    Using the property and numeric should keep everything on an even playing field. Obviously more screen size is available on larger devices, but smaller is not a phrase I would expect to hear.
