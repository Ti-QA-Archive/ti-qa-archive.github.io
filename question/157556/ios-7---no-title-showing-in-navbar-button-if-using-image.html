iOS 7 - No title showing in navBar button if using image

Hi,

I am trying to emulate the back button on Titanium 3.1.3/iOS 7

When I create a button with image and title properties and set it as leftNavButton, the button appears fine but only the image is displayed.

If that same button is placed in a window, it works OK.

Has anybody had this issue found a workaround for this ?

I have opened JIRA Ticket 3019 but need to find a solution until this is fixed.

Thanks

  • I believe the apple docs reference an image mask if you are going to be using an image for your nav buttons. Something about the way the image will slide into the chevron or something. Not sure how that applies in Titanium though.

    • That's right. Apple allows customization of the back button using the backIndicatorTransitionMaskImageproperty. This allows keeping the animation where the tilte slides into the image. However, this property is not implemented in Titanium (afaik).

      But furthermore, this is really for the back button, not for a standard button. As far as graphics are concerned, Titamium is ok if you use the backButtonTitle property. The chevron is here and the title appears. The issue is that this button triggers the close event and I handle the navigation and window stacks directly in the app. So I have to use a customized button (because I don't want the standard close event).
