    Hi

    It is possible to mix glyphs with text in most things including tabs (at least iPhone - not checked Android).

    Just look up the unicode value for the one you are wanting to use.

    For example;

    title: '\u2665'

    Would provide a red heart on the iPhone/iPad.

    Take care though as this MAY increase your chances of having an app rejected if your use of the glyphs is excessive where they should not be, distracting or contrary to UI guidelines for that platform.

    • Right I have a few solutions for you;

      1. Specifically to your question

      function createGlyph(strCode, strFilename) {
    var glyph = Ti.UI.createLabel({
        font: {
            fontSize: 30,
            fontWeight: 'normal'
        },
        height: 30,
        text: strCode,
        textAlign: 'center',
        width: 30
    });
    var file = Ti.Filesystem.getFile(Ti.Filesystem.applicationDataDirectory, strFilename);
    file.write(glyph.toImage());
    file = null;
}
createGlyph('\u2661', 'glyph.png');
var tab = Ti.UI.createTab({
    icon: Ti.Filesystem.applicationDataDirectory + 'glyph.png',
    title: 'Tab',
    window: win
});

      This will allow you to create a file version of your Glyph, you can then reference this within your app. My example shows it in use as the icon of a tab.

      There are problems with this technique;

      • iOS icons work on light and dark and the Glyph you provide may not give you those regions in the measures you require.
      • You are creating an image file every time the app runs, when that image is unlikely to change, even if it does it is unlikely the number of different images will be too small.

      Whilst this specifically answers your question - it would not be my advice, check the others. If you ask so why bother showing you - why not, it shows you how as you can use this technique for other things and also shows I may know and understand your question.

      2. Render the Glyph in an image application

      If you have the font, you should create the images in each resolution and supply with the app, if you need several variations, create them all, this is much more efficient on all counts.

      3. You need colour in your icons - and blue doesn't do it for you

      If you need icons that have more than the standard blue hue then check out TiCustomTab](https://github.com/mpociot/TiCustomTab) from Marcel Pociot. This seems to provide the ability to have your own images for both selected and unselected states.

      My Preference

      Do number 2 and benefit from 3 if you need colourful tabs.
