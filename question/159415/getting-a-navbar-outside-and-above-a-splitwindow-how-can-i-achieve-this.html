    Hi Esau,

    don't trust the documentation on the "SplitWindow" UI Component. It is full of wrong information. 

    The SplitWindow inherits its methods and properties from Titanium.UI.Window.

    No it does not. Basically it doesn't even inherit from "UISplitViewController" (The real native component).
    The "SplitWindow" is based on MGSplitViewController, so the Apple Documentation does not help in this case. And the MGSplitViewController does inherit from UIViewController. 

    >The SplitWindow is a top-level window and cannot be contained within another window or view.

    The "SplitWindow" ( the TiUIiPadSplitWindow Class ) does inherit from TiUIView. And what it does is basically to take the view of the[MGSplitViewController and add it as subview. So SplitWindow is a normal TiUIView, and therefore can be used like that.

    I made a proof of concept, it adds a splitWindow to a scrollableView of the root Window. The ScrollableView is only used to cast the SplitWindow to UIView. Don't use this! It is stupid, you can easily write a 3 Line Module to do this cast operation, instead of loading a ScrollableView into Memory.

    Proof of concept: (Ti Classic ) 

    var self = Ti.UI.createWindow({title:"rootWindow"});

var masterWindow = Ti.UI.createWindow({
    title:"masterWindow",
    backgroundColor: "white"
});

var detailWindow = Ti.UI.createWindow({
    title:"detailWindow",
    backgroundColor: "white"
});

var splitWin = Ti.UI.iPad.createSplitWindow({
    detailView: detailWindow,
    masterView: masterWindow
});

var nextWindow = Ti.UI.createWindow({
    title:"nextWindow",
    backgroundColor: "green"
});

var nextButton = Ti.UI.createButton({title:"nextWindow"});

nextButton.addEventListener('click', function() {
    outerNav.openWindow(nextWindow);
});

self.setRightNavButton(nextButton);

var outerNav = Ti.UI.iOS.createNavigationWindow({
    window: self
});
// stupid to do it like that basically just a cast to UIView
var scrollableView = Ti.UI.createScrollableView({views:[splitWin]});

self.add(scrollableView);

outerNav.open();

    • i've updated the module to version 1.1, the wrapper now subclasses the "add" method, which doesn't check i the view that will be added is a window. Use it with care. I only recommend using it with SplitWindow.

      i think a syntax like this should work now
      Classic

      var ViewWrapper = require('ti.viewwrapper');

var self = Ti.UI.createWindow();
var splitWindow = Ti.UI.iPad.createSplitWindow();

var viewWrapper = ViewWrapper.createView();
viewWrapper.add(splitWindow)
self.add(viewWrapper);

self.open();

      Alloy

      <Alloy>
    <Window>
        <Module id="viewWrapper" module="ti.viewwrapper">
            <SplitWindow>

            </SplitWindow>
        </Module>
    </Window>
</Alloy>

    1. Titanium maps to native iOS UI components.

    2. From apple docs:

    The UISplitViewController class is a container view controller that manages two panes of information. The first pane has a fixed width of 320 points and a height that matches the visible window height. The second pane fills the remaining space.
    1. Split view controllers are top level controllers.
    A split view controller must always be the root of any interface you create.
