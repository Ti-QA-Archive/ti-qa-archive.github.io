Android only module in cross platform app, error when trying to build for iPhone

Hi,

I have created a module that fills in some of the functionality not currently available for Android, but is available for iPhone. Therefore while the app is cross-platform, the module is an Android module written in Java.

It works fine when I build for Android, but when I try and build for iOS, I get an error about missing module. If I remove the entry from the tiapp.xml (modules section) it compiles OK.

How do I stop this error? I only want the module to be available for Android (all the files that actually use it are in my /Resources/Android directory)

Any help much appreciated.

    Specify the platform of the module in tiapp.xml
    Something like this:

    <modules>
  <module version="0.1" platform="android">com.androidmodule</module>
</modules>
