Invalid URL scheme generated from appname - fails iTunes validation and I can't fix it!

I'm using Titanium Studio 2.1.2, SDK 2.1.3RC2 targeting iOS6

So, my app has an odd name with an '@' in it. It's per the client's choice, that's just how it is. I've built the app and it's on the iTunes app store built with SDK 2.1.2. I'm rebuilding it with SDK 2.1.3 to fix an iOS6 bug and to support iPhone5.

Trouble is, the SDK generates an URL scheme for me and the '@' in the appname causes the app to fail validation in Xcode. So now I have an app which can't be uploaded to iTunes Connect.

I've tried editing the Info.plist to make a valid URL scheme, but when I build again in Titanium Studio, my edits to Info.plist get clobbered.

I'm HOPING there's a way to skip URL schemes in my tiapp.xml (I don't need one for my app, only for facebook integration), or barring that, some way of overriding the default one created for my app.

Right now, my app's no longer submittable to iTunes connect!

    In your tiapp.xml try:

    <ios>
    <plist>
        <key>CFBundleURLTypes</key>
        <array>
            <dict>
                <key>CFBundleURLName</key>
                <string>your.app.id.goes.here</string>
                <key>CFBundleURLSchemes</key>
                <array>
                    string>mycustomurlscheme</string>

                </array>
            </dict>
        </array>
    </plist>
</ios>

  • Hey guys,

    here is a dirty but working way to do this (use this only for final compiling or publishing):

    Go to the default Info.plist file there :

    [/Library or ~/Library]/Application Support/Titanium/mobilesdk/osx/[SDK you are using]/iphone/Info.plist

    In the file, modify the CFBundleDisplayName entry with your custom name :

    
    <key>CFBundleDisplayName</key>
    <string>Name-4'example</string>

    Compile or publish then reset default value.
