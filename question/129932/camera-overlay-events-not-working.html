Camera Overlay events not working

I'm writing an application that takes photos and stores them. I have all of my code working properly with the standard camera/camera controls, but when I try to use a camera overlay any event listeners I attach to my buttons in the overlay view don't work.

I've tried messing with z-indexing (which obviously doesn't work), different combinations of touchEnabled in both the overlay view, child buttons, and the camera itself. I've tried using a window instead of a view for the overlay. I've tried setting a 'touchmove' listener on the overlay view itself. I've tried copying the camera overlay sample straight from Kitchen Sink, none of it ever make the event listeners work.

I've only been able to find scarce few posts about this online, the ones I do find seem to be pointing toward the change from iOS 4.3 to iOS 5.0.

I'm using Titanium v1.8.0.1/v1.7.5 and my development iPhone has iOS 5.0.1, I don't have access to anything pre-5.0 to test on at this time.

If anyone has any insight as to what I could be doing wrong, or wants to see any code snippets let me know? Or is this just a bug that I can't overcome myself at this time? Thanks!

  • My code is in Coffeescript, I'm cutting out some of the code that doesn't have to do with the issue. This is also just a simple example, I've had many iterations of this, including the Kitchen Sink example.

    OpenCameraMethod = () ->
  overlay_view = Ti.UI.createView()
  cancel_btn = Ti.UI.createButton
    title: 'Cancel'
    bottom: 10
    width: 297
    height: 50
  overlay_view.add(cancel_btn)

  cancel_btn.addEventListener 'touchend', (e) ->
    Ti.API.debug('cancel button was touched')

  Ti.Media.showCamera
    autohide: true
    saveToPhotoGallery: false
    allowEditing: true
    mediaTypes: [Ti.Media.MEDIA_TYPE_PHOTO]
    showControls: false
    overlay: overlay_view
    success: (e) ->
      # success method does app specific stuff, this works fine with default camera.

    cancel: ->
      # camera was cancelled by user

    error: (error) ->
      # display error
      a = Ti.UI.createAlertDialog(title: 'Camera')
      if error.code == Ti.Media.NO_CAMERA
        a.setMessage('No camera detected on this device!')
      else
        a.setMessage("Unexpected error: #{error.code}")
      a.show()
  # end of Ti.Media.showCamera()

    Sorry for not including any sample code with my original post.

  • hey dudes i had this issue as well i know it over a year old but i figured out the solution when using overlays

    Ti.Media.showCamera({  allowEditing: false });

    this is required to be false otherwise the touch events for focusing will still be able for the camera which seem to override any other touch events on the overlay.
