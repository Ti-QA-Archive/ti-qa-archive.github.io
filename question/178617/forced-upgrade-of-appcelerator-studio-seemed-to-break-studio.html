Forced upgrade of Appcelerator Studio Seemed to Break Studio

All-

Due to a project deadline, I was delaying upgrading to the 3.4.0 SDK and new Appcelerator Studio, etc packages until the project went out. Yesterday, within studio, studio forced the upgrade and once the upgrade occurred, whenever in studio/opening studio, I now get an immediate error dialog stating:

Multiple problems have occurred
An internal error occurred during: “Retrieving platform status…”.

In the details it states:
Computing SDK info
Retrieving platform status…
Retrieving platform status…
Retrieving platform status…
Retrieving platform status…
Retrieving platform status…
Retrieving platform status…
…

Which I now have to kill studio to remove, making studio unusable. Has anyone else encountered this with the latest upgrade and gotten around this? I've seen other threads with a similar issue on start of a project, but this occurs literally at the startup of studio itself.

Hope all are well, and thanks for any advice.

  • I have not seen this, nor have I heard of any reports of it. It could be that one of the required components didn't get upgraded properly and Studio is not handling that gracefully. Try this at a terminal window / command prompt:

    [sudo] npm install -g titanium
[sudo] npm install -g alloy
ti sdk install
ti sdk select  # make sure 3.4.0.GA is selected

    You didn't say which OS you're running; sudo is required on Mac/Linux but not on Windows. If you're on OSX, make sure you're running Xcode 6.x. There are issues with Google's latest Android tooling (we put a note on our blog). You might need to downgrade to the v20 or v19.1 build-tools version (not SDK level).

    Sorta last-ditch, you might try uninstalling Studio then reinstalling it.
