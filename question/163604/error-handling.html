  • If you want to catch the error event the you need to code using try catch block, if any error comes then it will be handle by catch block.
    For example:

    try{
// do code in try block

}
catch(e){

alert(e); // if error comes the control comes to catch block.

}

    Another thing is whenever you will make the production build, red screen will never come, but in this case you or user will unable to find the problem. to make the production build you can go to publish then distribute and can create android store build.

    So best way is wherever you think exception or error may come write that code in try block so that you can handle exception to, and also when your development get finished you can distribute production build to users.

    Hope this will help you!
