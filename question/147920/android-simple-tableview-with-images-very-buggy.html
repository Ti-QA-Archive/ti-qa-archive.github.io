Android: Simple Tableview with images very buggy!

I have built a simple tableview with 25 rows that each contain a scollableview. In each scrollableview I have 3 pages that each contain an imageview of an remote image that takes up the width of the screen. The idea is to create something similar to photo posts in Facebook's news stream (When there are many images you can swipe right and left to view more images). When a row contains more than 3 images I dynamically load them via a modified version of this code: https://gist.github.com/dawsontoth/810171.

On the iPhone this code works beautifully but on Android it's terrible. Tableview scrolling is choppy and slow and most of the times images don't even appear until I have scrolled down and back up again. I have set the className as described in the documentation.
After some scrolling up and down the app finally crashes with a IndexOutOfBoundsException.
I found some issues in JIRA that seem to address this: https://jira.appcelerator.org/browse/TIMOB-11282

I am using Titanium SDK 3.0 with Alloy and testing on a Samsung Galaxy S3 with Android 4.1.2. As this is the top of the line Android device I shouldn't be having performance issues. I can't even imagine how it will work on an older 2.3 device.

I tried installing the latest 3.0.2 SDK and that made scrolling a little better and images seem to appear faster but still my first table row is blank! But now my dynamically loaded scrollable pages flicker when re-ordered. This worked fine in 3.0.

If there is any interest I can post some sample code but my setup is pretty basic and standard and shouldn't result in poor performance. My images are only 300x300px. From what I have understood the imageview will load remote images once and then save them to cache. This should make the second launch of the app superfast and load the images instantly? As my remote images are on my local laptop even they should appear almost instantly. Are there any alternative ways to load images that would be faster and more stable?

I have just started using Titanium and like the setup and idea of fast development but if it's this buggy it will be faster to build a true native app. Everything went pretty smooth on the iPhone but on Android it's a big disappointment as I have spent more time debugging and trying alternative code than actually building the app.

  • For my simple tableview with only an imageview I just discovered that when using the latest 3.0.2 it's not refreshing the images correctly when scrolling. I can all of a sudden see an image from a previous row. Maybe I should be writing my own code to load the images?

    Here is the code for the scrollableview scenario. As I am using Alloy the code is very modular but it should be enough to view the tablerow code.

    Table row view:

    
<Alloy>
    <TableViewRow id="row">
        <ScrollableView id="photos">
            <View class="photoContainer">
                <ImageView class="photo" />
                <Label class="count" />
            </View>
            <View class="photoContainer">
                <ImageView class="photo" />
                <Label class="count" />
            </View>
            <View class="photoContainer">
                <ImageView class="photo" />
                <Label class="count" />
            </View>
        </ScrollableView>
        <ImageView id="avatar" />
        <Label id="title" />
        <Label id="date" />
      </TableViewRow>
</Alloy>

    Table row controller:

    var args = arguments[0] || {};

$.row.link = args.id;
$.avatar.image = args.user.avatar_url + '_tc.jpg';
$.title.text = args.message;
$.date.text = args.date_created;

if (args.data.photos != null && args.data.photos != '') {

    // We load our views to the scrollableview dynamically for performance/memory reasons. e.g. only show three views at a time.
     var currentPhoto = 1; // Keeps track of which photo we are currently viewing.
     var lastPage = 0; // Keeps track of which page we were last on.
     var containers = []; // Temp array to store the pages.

    function loadView(view, index) {

        //Ti.API.info(JSON.stringify(view.children));
        //Ti.API.info(JSON.stringify(data));
        var data = args.data.photos[index];
        view.children[0].image = data.url + '_mc.jpg'

        if (index == 0 && args.data.photos.length != 1)
        {
            view.children[1].text = args.data.photos.length + ' photos';
        }
        else
        {
            view.children[1].text = '';
        }
    }

    // On scrollend we re-order and load views to make it seem infinite.
    $.photos.addEventListener('scrollend', function(e) {

        // If we have triggered an event and are still on the same page.
        if (e.currentPage == lastPage) {
            //Ti.API.info('Exit Scrollend event');
            return;
        }

        // Set current views into temp container if empty.
        if (containers.length == 0)
        {
            containers = $.photos.views;
        }

        // If we have scrolled to a new page we re-order.
        switch (e.currentPage) {

            case 0: // Scrolled to the left.
                currentPhoto--;
                //Ti.API.info('Current photo0 '+ currentPhoto);

                // If we have reached the beginning of the photo array we do nothing and stay on the 0 page.
                if ((currentPhoto - 1) == 0) {
                    lastPage = 0;
                    return;
                }

                lastPage = 1;

                // If we have more photos to show we pop a view off the end, and put it at the start.
                containers.unshift(containers.pop());       
                // Reset the image so that we don't accidentaly show an old image.
                containers[0].image = '';

                // Set the page so we are back in the middle.
                $.photos.currentPage = 1;
                // Set our new views array to our scrollble view.
                $.photos.views = containers;

                   // We now reload the 0 page with the correct content.
                loadView($.photos.views[0],  currentPhoto - 2);
                break;

            case 1: // We are on the 1st page we don't reorder anything as we are in the middle.

                  if (currentPhoto == args.data.photos.length) {
                     currentPhoto--; // If we come from the end of the photo array back to the 1st page we subtract the photo counter.
                } else {
                    currentPhoto++; // If we come from the 0 page we increase the counter.
                }
                //Ti.API.info('Current photo1 '+ currentPhoto);
                lastPage = 1;
                break;

            case 2: // Scrolled to the right

                currentPhoto++;
                //Ti.API.info('Current photo2 '+ currentPhoto);
                //Ti.API.info('No photos '+ args.data.photos.length);

                // If we have reached the end of the photo array we do nothing and stay on the 2 page.
                if (currentPhoto == args.data.photos.length) {
                    lastPage = 2;
                    return;
                }

                lastPage = 1;

                  // If we have more photos to show we shift a view off the start, and put it at the end.
                containers.push(containers.shift());
                   // Reset the image so that we don't accidentaly show an old image.
                containers[2].image = '';

                // Set the page so we are back in the middle.
                $.photos.currentPage = 1;
                // Set our new views array to our scrollble view.
                $.photos.views = containers;

                   // We now reload the 2 page with the correct content.
                loadView($.photos.views[2], currentPhoto);
                break;
        }
        Ti.API.info('Available Memory: ' + (Ti.Platform.availableMemory / (1024 * 1024)).toFixed(2) + ' MB');

    });

    // Load content for first three views and setup events. We always have at least one photo.
    for(var i = 0; i < args.data.photos.length && i < 3; i++) {
        loadView($.photos.views[i], i);

        // Add event for click on photo.
        $.photos.views[i].addEventListener('singletap', function(e)
        {
            Ti.App.fireEvent('open.overlay', { view: e.source, data: args.data, index: currentPhoto - 1 });
           });
    }
    // Remove views if we don't have 3 photos.
    if (i < 3)
    {
        $.photos.removeView($.photos.views[2]);
    }
    if (i == 1)
    {
        $.photos.removeView($.photos.views[1]);
    }

    Ti.API.info('Available Memory: ' + (Ti.Platform.availableMemory / (1024 * 1024)).toFixed(2) + ' MB');

}

    Thanks for taking your time!
