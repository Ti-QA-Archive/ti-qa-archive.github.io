  • To create an AppID, you do not need to have xCode or Titanium. You just need to enter a unique identifier. "The recommended practice is to use a reverse-domain name style string ". Like com.mydomain.appname.

    May be, the best thing to do, is to declare your developper as an admin temporarily. He must know how to handle the complex mechanism of Apple provisionning profile.

  • Hi,
    Follow the folowing steps to create Provisioning Profile :

    To distribute an app, your team must have a distribution provisioning profile you create using the
    http://developer.apple.com/ios/manage/overview/index.action

    1. After logging in to the iOS Provisioning Portal, click Provisioning in the sidebar and click the Distribution tab.
    2. Click New Profile.
    3. Select the distribution method and enter a profile name.
    4. Confirm that your team’s distribution certificate is displayed.
    5. Choose the app ID for the app you want to distribute.
    6. For ad hoc distribution only, select up to 100 devices you want to be able to run the app.
    7. Click Submit.

    On application side what you need to do is provide the same bundle name in Application Id in tiapp.xml File.
    While running on device you need to select the provisioning profile with bundle name as pass in tiapp.xml file.
