Querying User custom fields does not seem to work

Hi, I am trying to query for users based on custom define fields but for some reason ACS is not adhering to my where clause in my query statement.
For example I have

Cloud.Users.query({
page : 1,
per_page : 1000,
where : {
role : "Employee",
store_id : Ti.App.Properties.getString("userStoreId"),
deleted : false
}
});

for some reason it does adhere to the deleted : false property. It returns a user even if deleted is set to true.

  • i think you should not use deleted: false as query condition, because now Users.query always return all non-deleted users.

    You can use predefined fields to query users, we just support the following predefined fields of Users as query condition, you can find them at http://docs.appcelerator.com/cloud/latest/#!/api/Users, we list them in Fields section.
