Will my user files in the applicationDataDirectory get overwritten when the app updates?

I am making a mobile application and the users will be reading and writing to a JSON file in the filesystem applicationDataDirectory to store some data. Before going any farther, I wanted to ensure that this file doesn't get overwritten when I update the application. If it does, is there a way to back it up before they update and then re-download it after the update?

I could also use a SQL-Lite database if this avoids this issue. Thanks!

    No. Provided the app has the same ID (which will be the case if you are just recompiling the project), the resources folder gets replaced but folders like applicationDataDirectory remain. In fact, that is the reason a SQL database also retains its existing state as well.
