APK size large BUT should it be THIS large?

I had a dev in India make an app out of a book for me. The size of the html/css and images is about 7MB. The size of the app built with Titanium is 23MB downloaded to my Windows computer and 29.7MB installed on my Android phone.

I've read others on here who have questions about the size but most seem to be talking about maybe 5MB - 10MB added on. This is 22MB added to an original 7MB! Is Titanium or my dev at fault here? Any opinions would be appreciated.

  • Titanium's set of libraries that it packages with will take up a bit of space. To be honest it really depends how the app was built. You say html/css, so if this app is essentially a lot of web views, wrapped in Titanium, then i'm not sure why the APK could be quite that big. However, if native components are being used and not just webviews, then it could potentially be that big.

    On Android there are many sets of folders for each different resolution/screen-dimension that Android supports. If a developer takes the time and puts optimized images in each of those folders for the different devices, then that can substantially add to the APK size.

    One of my cross platform apps is roughly 30MB for the iPhone/iPad version. Compared to the 49MB for the Android Phone/Tablet version.

    The speed of the app could be an issue where they may have used the older Rhino Android engine, and may also depend on what device you are using.

    Hope this helps a little.

  • 7MB for an application is pretty standard, so don\'t worry about that. Please note that the application's limit size on Google Play is 250MB if I remember correctly. So adding a few more items won't go over that limit.

    Another thing though, know that the final packaged application will be smaller than the one you compile to test. That's because packaging will remove some unused libraries. But don't expect a big reduction in size though.

    Hope this makes it clearer for you.
