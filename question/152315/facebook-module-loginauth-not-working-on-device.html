Facebook Module login/auth not working on device

Hi!

I have 2 very big problems with Facebook Module on Titanium:

I created a small login script. And I have 2 situations:

  1. If I request this perms: ['email', 'user_birthday', 'read_friendlists', 'user_hometown', 'read_stream', 'publish_stream', 'user_likes'];
    when I try to login using Facebook it opens the Facebook App, asks me for permission, no matter what I choose it does not redirect back to the app
  2. If I remove the "publish_stream" permission when I click on the login button, I no longer get redirected to the Facebook App, but instead I get an alert with "com.facebook.sdk error 5"

The facebook app is ok, because the ios app works perfect when fb.forceDialogAuth = true; both on simulator and on device. Since the simulator has no Facebook App I need to test the fb.forceDialogAuth = false; on the device and I get the above problems…

Anyone can help? Thanks in advance!

Some data:

Titanium sdk 3.1 with Facebook Module +
Xcode 4.6 +
Iphone 4S 6.1.3

  • The big problem is not the module, but a failure to read the docs:

    The permissions property MUST NOT contain any of the following: offline_access, publish_actions, publish_stream, publish_checkins, ads_management, create_event, rsvp_event, manage_friendlists, manage_notifications, or manage_pages
Use the reauthorize method to request write permissions if needed.

    http://docs.appcelerator.com/titanium/latest/#!/api/Modules.Facebook
