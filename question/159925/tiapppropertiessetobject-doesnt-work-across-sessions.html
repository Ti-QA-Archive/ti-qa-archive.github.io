Ti.App.Properties.setObject() doesn't work across sessions?

3.1.3.GA / iOS 7

This function behaves as expected during a single session - loading the correctly saved object when a new instance of the controller is created. But if I kill the app and start again, my object in Properties is lost. Is there something else I'm supposed to do here?

function requestHide() {

    var hiddenRoutes = Ti.App.Properties.getObject("hiddenRoutes", {});

    if (!hiddenRoutes[args.siteId]) {
        hiddenRoutes[args.siteId] = [];
    }

    hiddenRoutes[args.siteId].push(args['label']);

    Ti.App.Properties.setObject("hiddenRoutes", hiddenRoutes);

    Ti.App.fireEvent("HIDE_ROUTE", { route:args.label });

}

  • This was my mistake. I had code in alloy.js to reset the value. So the object was saved correctly until the app started again, which executed alloy.js.
