  • I find two of your sentences paradoxical:

    • why does it say; iPhone SDK not found?
    • Obviously I can't get the iPhone SDK under Windows

    Doesn't the second answer the first?

    I am not sure what led you to believe Titanium could build an iOS application under Windows. Since Titanium works by creating a source project in the target platform's native language and automagically passing it through the native compiler, it follows an iOS target cannot be built when the native SDK is not available.

    As such, it seems to me there are only two possible alternatives:

    • Run an OSX virtual machine under Windows (and while technically possible, is against Apple's OSX license terms and conditions, so is not legal even if you purchase a license to OSX)
    • Cloud compile the project, i.e. ship the Ti project source off to a server which does run OSX, have it perform the build and code signing, then return the distributable

    The first option is illegal, so obviously not something Appcelerator can endorse.

    The second option leaves you without a simulator, and with horrendous debug/test cycle times.

    I think the only reasonable way to develop iOS apps is with an "Apple branded" computer, to use the terminology of the OSX license terms.

    IMHO. YMMV.
