How to use the SearchBar with a TableView in Alloy

I've bound a TableView to a collection and added a SearchBar as below. What is the best way to apply the search to the collection? Should I handle the change event on the SearchBar and use setData to rebind the data to the table? I haven't been able to find many examples for Alloy.

<TableView dataCollection="players">
    <SearchBar hintText="search"></SearchBar>

    ....
</TableView>

    Check out this test app from the alloy repo: https://github.com/appcelerator/alloy/blob/master/test/apps/ui/tableview/views/index.xml

    In short, you have the XML setup fine, but you need to set the filterAttribute property of the TableView that you are searching in order for the SearchBar to know how to filter your data. This is done client-side with no impact on the underlying collection itself.

  • hi please use the code i am posting– This one in your .xml file

    <Alloy>
    <TabGroup id="tabGroup">
        <Tab id="tab">
            <Window id="mywin" title="Hello">
                <TableView id="table">
                    <SearchBar hintText="search"></SearchBar>
                </TableView>
            </Window>
        </Tab>
    </TabGroup>
</Alloy>

    This one in your .js file

    var items = ['Television', 'Music System', 'Car', 'Bus', 'Train', 'Computer', 'Lap Top', 'I-Phone', 'Tablet', 'Electronics', 'Watch', 'Mouse']

function fillData() {
    var data = [];
    for (var i = 0; i < items.length; i++) {
        var row = Ti.UI.createTableViewRow({
            height : 50,
            title : items[i],
            color : 'gray',
            hasChild : true,
            font : {
                fontSize : 11
            },
        });
        data.push(row)
    }
    $.table.setData(data);
}
fillData();
$.tabGroup.open();
