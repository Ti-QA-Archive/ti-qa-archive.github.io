    I have had the same problem. I have solved right now.

    Solution:

    1. In tiapp.xml use a version number with 4 digits. Ejem: 2.0.0.1 (the first numbers MUST match with the version number you already has submitted to iTunesConnect, i.e., in the case before I have had uploaded the version 2.0, matching the version of my APP in iTunesConnect -2.0 too-)

    2. This tiapp.xml generates a Info.plist with different values for the params CFBundleVersion (2.0.0.1) and CFBundleShortVersionString (2.0.0.1). If you use a shorter version value in tiapp.xml, both params have the same value in the Info.plist and in the submitting process.. meeeeckkk!! Error: redundant bla bla bla…

    3. Build for iTunes.

    4. On Xcode: The Validation process in Organizer doesn't validate the APP (missmatch version number), but I have submit the APP and the submit was OK!!! I can see the APP in iTunesConnect. Now is "Waitting for review".
