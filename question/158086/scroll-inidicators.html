scroll inidicators

Is there any way to indicate to the user that a table is scrollable on Android. I see in the API there is a 'showVerticalScrollInidicator' boolean property which can be set for iOS but not Android. Any way around this?

  • Hi

    The showVerticalScrollIndicator property is available on; iOS, Android, Tizen and mobile web.

    I suspect you think it does more on iOS than it does on Android. So to be clear this is what is does on Android and iOS;

    • iOS; provides a bar down the edge that is moveable to show the current position and size of the current view-port onto the larger view available outside the view-port. This is not visible unless you are moving the view.
    • Android; (most but not all versions of android) provides a bar at the top of the view-port and the bottom view-port, this appear as a shaded colour when there is no more content outside the view-port that can be positioned within the view-port (you are at the end of the scroll).

    Whilst they two methods are platform dependant - neither provides any initial visual clue to the user that the view (scroll or table) has more content. This is left to user discovery, the idea being that most users even after a few minutes of exposure to the platforms will discover this feature and if not immediately obvious there is more content (cut off content showing more available) the user is likely to check themselves.
