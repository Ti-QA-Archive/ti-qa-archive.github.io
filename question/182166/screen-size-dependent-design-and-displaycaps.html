Screen size dependent design and displayCaps

hello,

id like to make the layout of the apps as much as possible screen-size dependent and use percentages where i can.

so in alloy.js i define the basic size definitions like so:

Alloy.CFG.height = Titanium.Platform.displayCaps.platformHeight;
Alloy.CFG.width = Titanium.Platform.displayCaps.platformWidth;
Alloy.CFG.bigText = Alloy.CFG.height / 20;
Alloy.CFG.mediumText = Alloy.CFG.height / 26;
Alloy.CFG.normalText = Alloy.CFG.height / 32;

and in app.tss im calling them:

"Label":{    
    font: {
     fontSize: Alloy.CFG.normalText,
     fontFamily: "OpenSans-CondensedLight"
     },
     color:"#151624",
}

now my question: the iphone simulator shows this height/width for the iphone 5S

[INFO] :   Device height: 568 //Alloy.CFG.height
[INFO] :   Device width: 320 //Alloy.CFG.width

while on http://www.iphoneresolution.com/ it says the screen dimensions for the iphone 5S are 640 * 1136.

what am i missing here?

  • Hello Markus,

    There is no problem here. In this link here it says that any height and weight measurement is dp/dip in Titanium unless you mention % after that that it become relative measurement.

    And "1dip=2px on retina devices". The link you provide shows the pixels value for the iOS devices. and with Titanium you get the dp/dip values.

    So just double the value you get to get the pixels value. Hope this helps, Thanks.
