  • Hi Richard

    I have downloaded the app to find the section you are talking about but I am not sure where the section you are talking about is to see if there is anything special.

    Can you provide a screenshot - use something like droplr.com.

    But I think Shannon is right, apps like this (I produce a few myself) usually create a feed to a webserver for 'offers', in that feed it includes at least; a title, graphic, value and expiration date. The device simply reads that and will only show it until the expiration date has exceeded. If the offer is redeemed then the the app store the used ID or marks the offers database table row as redeemed. That way the app will no longer offer the single use voucher again - if that is what we are talking about.

    It is also possible for the app to fire back to the server the redemption of that offer and the location of where it was redeemed. It will normally do this next time it seeks data. It is good for checking success criteria for marketing people.
