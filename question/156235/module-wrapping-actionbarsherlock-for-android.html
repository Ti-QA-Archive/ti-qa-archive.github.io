    • BTW there is NOTHING wrong with creating an app that has visual differences between Android 2.x and Android 4.x. This is expected behaviour, some companies try their best to make everything look the same - but you have to remember your 2.x user does not have 4.x so they are expecting 2.x UI not 4.x UI.

      This is the same for iOS/Android. The common wisdom is to create an app that performs the same functions but honours the UI of each device. It is not fair to the Android users to thrust iOS on them and the same is true of iOS apps looking like Android apps.

      The same code is used underneath, the actionbar/menu code is almost the same. As long as you create modular code, split up into functions and call as required you really have very little extra effort required.
