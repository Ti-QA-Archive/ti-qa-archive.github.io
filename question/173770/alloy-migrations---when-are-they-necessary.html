Alloy Migrations - When are they necessary?

According to the guides, it is possible to write migrations for the local sqllite database. However, when developing, it doesn't seem to be necessary. When are the migration files necessary? Should I be specifying them straight from a fresh app? Are they only necessary when doing something custom that alloy doesn't give you?

Thanks!

    Migrations allow you to update the database in a staged fashion, usually applied to applications that have already been released. For example, you might release version 1 of an application, then ver 2 and 3 are released, each with a migration to the database. However, users might never have loaded version 2, and upgrade right to version 3 of the app. The migrations provide a staged, known path from one version of a database to another.

    So you are right, usually might not apply while developing, until after you release something into the wild.
