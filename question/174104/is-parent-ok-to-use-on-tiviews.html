  • Hi

    Firstly the documentation or lack of it - good question, it is not being deprecated anytime soon as far as I am aware.

    The benefits of using it - it does make it easy to know where this child was placed;

    • how many times did you forget where you added something?
    • how many times does the knowledge of the parent affect the behaviour of the child?

    Ideally speaking you should never use this property, not because it is wrong, more that your code should know where things are and children should not need to understand their parents in order to function.

    So in short the use of the patent property in my opinion is not necessary and at worst used rarely when all other techniques have been considered first.

    As to a better way if doing things - keep in mind you only just found out about it so you already know how to work without it. There are so many ways in which to maintain links between parents and children to manage the scope - that they are simply too many to explain here.

    I am not sure what your example was trying to achieve - I always work on solving a problem rather than providing an answer. They are rarely the same thing. So explain a situation where you think the patent property would assist you that cannot be resolved any other better way.

    • thanks Malcolm for your reply,
      You are absolutely right, ofcourse there are alternatives but, at least for my case, .parent is the simplest, but maybe not the best with performance in mind.
      here is my case for parent usage:
      i have a multi-window app that all windows share a single view(a custom drawer with same functionality and look returned by a commonjs module that should appear on all windows on demand),
      so each time a window gain focus(its activity starts or resumd) im gonna remove the drawer view form its old window(current parent) and add it to the newly opened window as below:

      // drawer.js
var drawerView = Ti.UI.createView({
// some props
)};
extends.addToWin= function(newWin) {
    var oldWin = drawerView.parent;
    oldWin.remove(drawerView);
    newWin.add(drawerView);
}

// anyWindow.js
var win; // createWin

win.addEventListener('focus' , function(e) {
    require('drawer.js').addToWin(win);    
});

      it works as expected but im not sure it will do so in the future or on all devices or conditions, 'coz of use of parent.
