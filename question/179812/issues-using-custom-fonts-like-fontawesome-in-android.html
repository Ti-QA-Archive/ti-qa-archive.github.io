Issues using custom fonts like fontawesome in Android

I'm using custom fonts like fontawesome with this code (fontawesome font downloaded from their website):

var os = Titanium.Platform.osname;
var window = Titanium.UI.createWindow({ ... });
var label = Ti.UI.createLabel({
    text: String.fromCharCode(0xf003),
    color: '#000',
    font: {
        fontFamily: (os == 'iphone') ? 'FontAwesome' : 'fontawesome-webfont',
        fontSize: "40dp"
    }
});
window.add(label);
window.open();

This is working perfectly on iOS but generates an Exception with the message native typeface cannot be made on Android:

[ERROR] :  TiApplication: (main) [1299,1299] Sending event: exception on thread: main msg:java.lang.RuntimeException: Unable to start activity ComponentInfo{webintex.android.test/org.appcelerator.titanium.TiActivity}: java.lang.RuntimeException: native typeface cannot be made; Titanium 3.4.0,2014/09/25 16:42,b54c467
[ERROR] :  TiApplication: java.lang.RuntimeException: Unable to start activity ComponentInfo{webintex.android.test/org.appcelerator.titanium.TiActivity}: java.lang.RuntimeException: native typeface cannot be made
[...]

Using Batasoft's AppIcons is working for both operating systems (and they are using the same fontawesome font). I'm just testing with this font, because I want to use another one.

I am clueless, can anyone help?

(Titanium SDK: 3.4.0.GA)

    To ensure compatability with Android please copy(dont delete) this fonts folder to your app assets directory.

    The resulted path should be app&#x2F;assets&#x2F;fonts&#x2F;FontAwesome.ttf

    For me fontFamily: "FontAwesome" is enough to get it working on both operating systems. And i use this as label text, text: unescape("%u"+"f105");
