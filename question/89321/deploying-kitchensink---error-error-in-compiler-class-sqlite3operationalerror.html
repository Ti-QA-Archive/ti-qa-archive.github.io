  • Anthony

    OK, let's try this again. :)

    Are you actually using a database in your application?

    Is your system configured to use by default 32bit Sun JDK 1.6 revision 10 or above? (not 64bit, not version 1.7). run java and javac at the command line to find this information.

    Is your system configured to run python 2.6 (not another version)?

    Do you have Titanium Developer 1.2.2 installed?

    Titanium 1.5.X from the continuous builds website should work, but you can always try 1.4.X, which is very stable and so useful in finding the "baseline" of what should work.

    Close all emulators.

    Close Titanium Developer. Rename the directory in your system's user dir named something like .titanium. This is where TiDev stores all its settings for the projects you have created (it also stores some data in some sqlite databases). Note, if the approach I am explaining does not work for you, you can always rename the directory back later to recover all your settings. If you can't find the directory, search your user profile for the file application.properties, which is inside it.

    Delete the android directory in your user profile. On my system this is named .android. Inside it, there is a directory named avd.

    Delete the android directory in your system's temporary directory. On my system this is &#x2F;tmp&#x2F;android.

    Delete the adb.log file in your system's temporary directory. On my system this is /tmp/adb.log.

    In your project directory, delete everything in the following directories:

    $PROJECT/build/android/<subdir>/*

    Hence, anything inside subdirectories assets, bin, gen, lib, res and src.

    Now launch Titanium. You will need to reconfigure it, as you deleted the titanium directory. Ensure that the path to the android SDK is configured correctly, On my system it is &#x2F;opt&#x2F;android-sdk. This directory contains the tools directory which holds the adb or adb.exe executable.

    In "Test & Package" / "Run Emulator" Android SDK "APIs 2.2" should be selected, as version 2.3 is not supported at this time. Note this is "APIs 2.2" not "2.2".

    Launch the app. Give it 3 attempts.

    If an emulator does not boot successfully, from the command line run the command android. Select the "Virtual Devices" category on the left, which should show you a list of AVDs. There should be at least one with a name that starts with titanium_7_. Boot this manually using the "Start…" button on the right. Does it boot?

    When an emulator has booted, try launching the app again.

    If you provide all the information requested here, and it still doesn't work, I will try to think of some other suggestions. :)

  • Happened to me when I had non-english user profile folder name.

    Fix is easy - look up builder.py and deltafy.py (in /mobilesdk/(version) folder) for strings like

    os.environ['USERPROFILE']

    and replace them with something like 'D:\'…

    E.g. change

    self.home_dir = os.path.join(os.environ['USERPROFILE'], '.titanium')

    to

    self.home_dir = os.path.join('D:\\', '.titanium')

    Don't forget to save a backup and make a .titanium folder on D:\

    Hope this makes sense.

  • Richard, I feel your pain too… I just went through the same process of agony and pain… Here’s my account just to encourage everyone to keep trying:

    About 10 days ago, I decided to give Ti a shot after seeing a friend doing some nifty development work on his Mac. He said it should be as easy doing it on Win (since I am a windows guy). I figured I'll spend an afternoon to check it out and create a simple hello world app. I literally spent one week glued to my Win computer just trying to get a proper install, and create the first "I am a window" app, and almost gave up many times. I read and re-read messages relating to the errors I was getting. Many people tried to help, some with success others not. I was disgusted but it became a challenge to get this platform up and running. I finally got the whole thing working on XP, Vista, and Win7. Each has its own quirks and what works on one does not necessarily solve problems on another. (For example, I still have some lines in the XP version’s index.js code commented out since otherwise Ti thinks I am not online.)
    My distilled wisdom boils down to this:

    1- Make sure all required products (other than Ti Dev) are installed properly and they all work from the DOS prompt (for Android work: Java, Android SDK, Python, Scons, Git) by trying the tests specified in getting-started document to make sure all path are correctly defined… failure to do this will send you in days of agony and unrelated error messages.

    2- For Android, make sure SDK 1.6 and Google API 1.6 are installed along with at least one more recent version (2.1 or 2.2 but not SDK 2.3).

    3- Install Ti Dev and make sure it points to the correct Android SDK folder… this will require copying some files from the default install folders to the platform-tools folder like adb.exe, AdbWinApi.dll, etc… Ti will tell you which files are missing and you can copy them manually

    4- Do not mix and match versions i.e. when using Ti SDK 1.4, do not use KitchenSink 1.5, but rather use KitchenSink 1.4.

    5- To get your message to the appropriate eyes in Q&A (VERY IMPORTANT), make sure you follow the guidance listed in: http://developer.appcelerator.com/guides/en/contribute.html

    6- If you can afford it, sign up to the professional version where you will get help quickly, but if you don’t (like me) hang in there in the Q&A and keep trying: once you get the whole thing working, you’ll be amazed all the stuff you’ll be able to do with just a couple lines of code!!

    Good luck.
