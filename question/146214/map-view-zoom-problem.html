    Hi Drew

    The result you are seeing is by design and out of your control. The mapping system is based on image tiles and they are tuned several zoom levels.

    When you zoom in and out the map system will do its best to keep in the same level of dynamic zoom - but if it is close to a more detailed set of tiles it jumps to the closest level.

    This is true when you use the zoom method, touch or latitudeDelta & longitudeDelta parameters.

    It can also nudge the map in any direction to better match the tiles that appear.

    When all the maps use the vector based data rather than image tiles this problem will disappear. This is outside of Appcelerators control of course.
