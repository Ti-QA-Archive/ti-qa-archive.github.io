    I have this same issue and I just hacked it to work. I believe it stems from the latest android sdk moving the aapt and dx files. They used to be in a different directory, but they were moved to the build-tools directory so if you happen to be using the latest build tools this will be broken. Just like you I searched high and low and could not find the answer so I plodded through the code and just changed the paths to the direct paths to those files in androidsdk.py. There should be a bug report about this if there isn't already.

    In that file basically you will need to change any reference where it does a self.get_platform_tool to be a simple string containing the file path instead. This is obviously only a work around until the Appcelerator guys can fix this in the SDK.
