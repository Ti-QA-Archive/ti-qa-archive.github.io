Titanium.Platform.displayCaps.dpi gives wrong value on Samsung Galaxy Mega 6.3"

dpi should be 233 while it gives 160. Is there a way to force it do go to hdpi and use the proper folder for assets?

    Ti.Platform.displayCaps.dpi represents Androids densityDpi which breaks the value down to the density (low, medium, high, etc.)

    To be treated as high the DPI must be at least 240. Because 233 is < 240 it is broken down to use medium density. In fact, there is no way to tell Android to use hdpi resources, because it won't scale up, always scaling down.

    If you want to retrieve exact values for DPI then you should use xdpi and ydpi which returns you the physical pixels per inch of width and height of the display. (Be careful with the height: platformHeight does not include the navigation bar)
