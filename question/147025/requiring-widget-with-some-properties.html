Requiring widget with some properties

Hi,
I am creating a widget and want to require in controller. I have done that very well.
Also I have created a function in widget.js which I do call from the controller which needs to be passed some arguments which is constant.
Is there any possibility that I can pass arguments to widget at the time of requiring in xml or tss rather than in js?

<Require type="widget" src="newWidget" id="myWidget"/>

    Yep. All attributes on the <Require>/<Widget> element get passed to the corresponding controller as an object. Bear in mind that this is XML, so all values will be in the form of a string unless you convert them to a different type.

    This should work for you if I'm understanding correctly.

    index.xml

    <Alloy>
    <WIdget src="newWidget" id="myWidget" someCustomProperty="123" anotherProp="hi there"/>
</Alloy>

    myWidget/controllers/widget.js

    var args = arguments[0] || {};
var someCustomProperty = args.someCustomProperty;
var anotherProp = args.anotherProp;

// now you can use those properties from the XML in your widget's controller.

    Also, <Widget> is just shorthand for <Require type="widget">. They behave exactly the same.
