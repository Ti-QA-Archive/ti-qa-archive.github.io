  • Hi

    I answered a similar question a few months back about Chinese and the differences between traditional and simplified as it also has this sort of issue to overcome.

    Use this to see what the device is returning back;

    Ti.API.info('currentLanguage: ' + Ti.Locale.currentLanguage);
alert('currentLanguage: ' + Ti.Locale.currentLanguage);

    Also take a look at my previous answer as it points out the difference that Android and iOS have, you may not get exactly what you want with Android. There was no choice on Android for specific Chinese choice, but there was on iOS.

    • So now follow the example in the linked answer to test IF those files are being returned, first created the files and folders;

      {appname}/i18n/en/strings.xml
{appname}/i18n/nb/strings.xml
{appname}/i18n/nn/strings.xml
{appname}/i18n/nb-NO/strings.xml
{appname}/i18n/nn-NO/strings.xml

      Then test WHICH is returned by having a single test string that is displayed in your app;

      i18n/en/strings.xml

      <?xml version="1.0" encoding="UTF-8"?>
<resources>
    <string name="test">English</string>
</resources>

      i18n/nb/strings.xml

      <?xml version="1.0" encoding="UTF-8"?>
<resources>
    <string name="test">nb</string>
</resources>

      i18n/nn/strings.xml

      <?xml version="1.0" encoding="UTF-8"?>
<resources>
    <string name="test">nn</string>
</resources>

      nb-NO/strings.xml

      <?xml version="1.0" encoding="UTF-8"?>
<resources>
    <string name="test">nb-NO</string>
</resources>

      nn-NO/strings.xml

      <?xml version="1.0" encoding="UTF-8"?>
<resources>
    <string name="test">nn-NO</string>
</resources>

      In your app add a label you can easily see and use the L macro to show the test keyword;

      ...
text: L('test')
...

      Now your app should show WHICH one is being matched on each respective platform.
