Ti.Database.open(file.nativePath) fails - "contains a path separator"

In my app I need to be able to open sqlite files that I have downloaded to the device. Once the download has finished (verified to work OK), I have a file handler named file. file.nativePath resolves to this value:

file:///mnt/sdcard/com.lucion.nexgen/dl-1380037701175.sqlite

Which I then use to attempt to open using Ti.Database:

Ti.Database.open(file.nativePath);

Looking at the Titanium.Database.open documentation I'm led to believe that we can open a database with a full path, but in practice I get this Runtime Error:

Uncaught Error: File file:///mnt/sdcard/com.lucion.nexgen/dl-1380037701175.sqlite contains a path separator.

If anyone can help shed light on this issue, I will be very grateful.

  • You splendid man! That worked a treat.

    // My original code (failed)
Ti.Database.open(file.nativePath);

// What you suggested (worked)
Ti.Database.open(Ti.Filesystem.getFile(file.nativePath));

// My final solution (refactored)
Ti.Database.open(file);
