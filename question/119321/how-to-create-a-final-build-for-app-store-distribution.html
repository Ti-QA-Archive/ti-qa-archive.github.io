How to create a final build for app store distribution

What is the recommended way to create a final build to submit to the app store? Apple's instructions seem to be for a previous version of XCode which is confusing.

This is what I've done so far: Performed a clean build in Titanium then opened it in XCode 4. Set the code signing identity to Distribution.

What is the next step? Do I manaully have to change the plist and xml files before creating an Archive in Xcode and then use the archive file for submission? Or am I totally on the wrong track here! Many Thanks.

  • In your build settings, make sure its picking up your distribution profiles. If so, then in the top right, change it from 'simulator' to 'device'. After that, Archive it, and when its done building, it will open the Organizer. Click your app, and then you can validate and submit it from there.

    Make sure you have the app created in iTunes Connect on Apple's site and that you've gone through the steps and its "Waiting for Upload".

    For me, to get my icons working, I had to manually delete the existing Default.png file in the Resources folder in XCode, and drag and drop my own.
