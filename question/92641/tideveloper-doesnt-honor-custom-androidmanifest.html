Ti.Developer doesn't honor custom AndroidManifest

[SOLVED]

Ti.Developer 1.2.2 defaults to using Google's SDK version 8 (corresponds to APIs 2.2 in the GUI). I'd like to use an earlier SDK to support older phones. I'm trying to use Google's "APIs 1.6" sdk, which in the XML file would be SDK 4. What I'm finding is that the custom manifest changes are not being honored.

The old method was to create a PROJECT/build/android/AndroidManifest.custom.xml file. The new method, per this page is to specify the minSdkVersion in the PROJECT/tiapp.xml file.

If I set the Titanium SDK to 1.5.1, then setting the Android SDK version in the tiapp.xml file is honored. But my app is very slow with the Titanium 1.5.1 SDK, so I want to try using the older Titanium SDK.

If I set the Titanium SDK to 1.4.1, then the tiapp.xml file is NOT honored. Neither is the AndroidManifest.custom.xml file. In either case, the generated manifest contains the line: android:minSdkVersion="8" and my app fails to install with the INSTALL_FAILED_OLDER_SDK error.

Anyone find a workaround for this?
