[sdk 1.7.0] How to run fastdev on Windows XP?

Hello,
I try to run fastdev server on Win XP (for android project), but I'm obtaining some problems…
When I'm runing 'titanium fastdev start' in command line it's throwing an error:

C:\Documents: can't open file 'and': [Errno 2] No such file or directory

So the point is that path to titanium contains some spaces ('documents and settings', 'all users' etc.).

Any idea how to work it around? For 'docs and settings' it's easy, tilda works, but for the rest… :/

  • This not a solution but information I discovered while investigating the issue.

  • I managed to finally get it working on Window 7! (still not on XP though)

    As one of my other message stated, the problems lies in the fact that Python is installed in a directory that contains a space character. (ie. C:\Program Files\Python27)

    What I did was, I uninstalled Python altogether and re-installed it in the default installation directory (C:\python27). From there, it worked like a charm.

    The problem seems linked to the way Titanium retrieves the Python path and then used it further down the road. The scripts don't seem to put quotes around everything, but this is just speculation.

    I'll keep investigating for Windows XP…

  • After making it to work on Windows 7, I finally found a way to make it work on XP. The key phrase to remember under Windows XP and Fastdev:

    ###You cannot have spaces in any of your paths. NONE!
    That means that:

    • Python must be installed in the default folder (c:\python27)
    • The titanium SDK must be copied from C:\Documents and Settings\All Users\Application Data\Titanium to a path that will not contain spaces (I copied mine in C:\Titanium)

    The last piece of the puzzle is: Don't forget to update your paths and environment variables! Otherwise, the scripts will still call the wrong ones.

    If you are using Titanium Studio, make sure to make point to the new Titanium SDK path (the one without spaces).

    Hope this helps everyone!
