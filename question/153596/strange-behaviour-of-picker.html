Strange behaviour of Picker

I am using Titanium 3.1 + Alloy 1.1

if ($.currencyPicker.columns.length > 0){
        var col = $.currencyPicker.columns[0];
        if(col.rowCount > 0){
            for (i=0, l=col.rowCount; i<l; i++) {
                var _row = col.rows[0];
                col.removeRow(0);
                _row = null;
            };
        };
    };

There is error at the line. But col.rowCount is showing correctly in the debugger

        if(col.rowCount > 0) {

After I change the code snippet to the following snippet and it works perfectly

    if($.currencyPicker.columns[0]){
        var _col = $.currencyPicker.columns[0];
        var len = _col.rowCount;
        for (var x = len-1; x>=0; x--){
            var _row = _col.rows[x];
            _col.removeRow(_row);
        }
    }

What's the problem?

    I can't see how the code snippet in the first example would work. All it is doing is removing the first item in the column and not iterating over the rows. The second code snippet actually iterates over the columns (via the x variable). I think this may be the reason the first code is not working.
