New project has no simulators

Hi,

I have just started a new project in titanium studio and for some reason I cannot get an iphone simulator to show up in the debug or run options.

I did have this on a previous app I started on, but since I removed that and started another, all I get is the 'Debug configurations' and 'Run configurations' options and I have no idea what to do in here.

I tried look on Google for answers but have come up with nothing, which is weird. Can someone point me to documentation somewhere on how this is setup?

I have already been here: http://docs.appcelerator.com/titanium/latest/#!/guide/Quick_Start and that is not what I am after, I have done all that.

Thanks!

  • I had a similar problem. The only way I was able to solve it is by changing the Titanium SDK version in config.xml to something else, and then changing it back.

    In my case, the run command was disabled for an existing project that I work on regularly. The project is running on SDK 3.2.3. I changed the SDK to 3.4.1 and saved it. The run command then became enabled. I then changed it back to 3.2.3, and saved it. The run command remained enabled.

  • For some reason, there are two folders at the same level with similar folder and file structure. These were both created by titanium when I created the project. One is the name of my app and the other is the name in which I originally intended the project to sit in.

    The project folder did not have a tiapp.xml but the other folder (the one that is named the same as the project) did.

    I copied it across and it worked. So it was the missing tiapp.xml file. Not sure why I get two folders created though. And why they are slightly different. But I have found the problem.

      APP_NAME
