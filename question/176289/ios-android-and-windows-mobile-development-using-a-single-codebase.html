    • Native Win phone will be coming with Hyperloop towards the end of year if all goes well. At what level is another question.

      I am going to take a look at the hybrid stuff myself in the next few days, but not expecting anything near Android/iOS support currently available.

      Also keep in mind "single code base" can be misleading. Some take it too literally;

      • "the same code does the same thing on all platforms"

        This is wrong, Android has tables, buttons and labels - so does iOS, but iOS has Navigation Windows not available on Android, Action Bars are only available on Android not iOS.

      • "the same core code base - that you can then honour the uniqueness of each platform"

        This is the right thing, your business rules, APIs, messaging all are the same, the UI and some UX will be different so that your users know they are using the iPhone, iPad, Android or other version of your app.

        Trying to make an iOS app look the same as Android puts off users, so does making your Android app the same on iOS. But making the same database, apis and network calls, business rules, internal functions and logic are all the same. You then move to accessing and viewing the same information following the rules of the target platform and that is the route to success.
