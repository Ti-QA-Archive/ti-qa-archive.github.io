Listening to a 'android:back' on a tabGroup

I have a tabGroup with one of the tabs consisting of a tableView. From what I've done, the tableView acts as a linked list where a user click changes the list to a different one. Currently, if the user presses the hardware back button, the application window is closed. I don't want that. What I want to achieve is to go back to previous list.

Basically, what I want to know is where do I listen for the 'android:back' event. From what I've read in the documentation and the QA's here, you need a modal window for that. But I don't have any root window to listen to. I just have the tabGroup as the root which does not seem to receive the 'android:back' event.

Any workarounds greatly appreciated.
