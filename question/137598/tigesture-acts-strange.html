Ti.Gesture acts strange

Hi.

In my app i have different backgrounds for Landscape and Portrait mode.

It works fine when you start in portrait mode but if i am in landscape and opens a new window it acts like i am in portrait until i change to portrait and then back to landscape.

Any ideas on why it doesn't "feel" that i am in landscape to begin with?

I have tried both in emulator and on device.

This is my code. 

Ti.Gesture.addEventListener('orientationchange', function (ev) {


  if (Ti.Gesture.isLandscape(ev.orientation)) {

    logoImage.visible=false;
    bgImage.image = 'images/landscapesf.jpg';
    bgImage.top = -60;
    tableview.top = 12;
    tableview.height = '100%';


  } else
{

    logoImage.visible=true;
     bgImage.image = 'images/11.jpg';
      tableview.top = '110';
    tableview.height = '70%'

  }
});

    • See this…..

      
Titanium.Gesture.addEventListener('orientationchange', function(e) {
    switch (Titanium.Gesture.orientation) {
        // For LandScap.......
        case Titanium.UI.LANDSCAPE_LEFT:
        case Titanium.UI.LANDSCAPE_RIGHT:
                // code...
        break;

        // For Portrait.......
        case Titanium.UI.PORTRAIT:
        case Titanium.UI.UPSIDE_PORTRAIT:
                // code...
        break;
    }
});

Titanium.Gesture.fireEvent('orientationchange');
