Re-opening window

Hi

Quick question: I'm opening a new window (win2) using a navigation group so that I will get a back button in the upper left corner in the new window.

This works great.

However, if I open my app, navigate to win2, go back, navigate to the win2 again it seems to be cached.

I've defined some Ti.App.addEventListener( 'custom', function() {}); events in my win2 that my webview is using to communicate between the JavaScript in the webview and win2.

The second time I open win2 the "custom" event is undefined. JavaScript isn't able to fire the event.

Why is this? How do I make sure that when I open win2 it starts from scratch everytime.

  • John

    I have created ticket #3501 for Android relating to webview.reload(). Although I haven't tried it on iOS yet, I will do later today and create a ticket accordingly.

    A workaround is included in the ticket, where instead of using webview.reload(), you reassign the url property in the eventListener, for example:

    Ti.App.addEventListener("reload", function(){
    Ti.API.info("Reloaded Webview");
    alert("fireEvent from webview - reload Webview");
    webview.url = "webview.html";
});

    Hope this solves your issue.

    Cheers
