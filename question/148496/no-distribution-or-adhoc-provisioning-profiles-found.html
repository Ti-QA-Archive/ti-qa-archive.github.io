    • One thing you can try is run a build from the command line and omit the provision profile UUID.

      cd /path/to/project

ti build -p ios -T device

      It will prompt you for any missing arguments including the provision profile UUID. Simply hit <enter> and you'll see a list of all valid provisioning profile UUIDs. Copy and paste and it should build.

      — commented February 23rd 2013 by Chris Barber

    • Excellent. This did the trick:

      ti build -p ios -T dist-appstore

      Any reason why it would work on the CL but not via Ti Studio?

      — commented February 23rd 2013 by Ben Hill

    • While Studio calls the CLI to do the build, it doesn't always call titanium info -o json to discover certs and such, which means the two get out of whack. We'll figure it out someday.
