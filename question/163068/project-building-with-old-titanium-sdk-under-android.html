Project building with old Titanium SDK under Android?

For some reason, when I create an Android build of my app, it uses an older version of the SDK on my work machine than my home machine. And I can't figure out where to change this. I know the machine itself has the most up to date SDK available, but my builds aren't taking advantage of it, for some reason.

Where do I go to look for this and set to use most current?

    • 1) Try to clean project first ( delete build folder) then rebuild it. If it does not work, try step 2

      2) Open tiapp.xml editor (tiapp.xml tab)
      try to add this after </iphone> tag

       <android xmlns:android="http://schemas.android.com/apk/res/android">
        <manifest>           
            <uses-sdk android:minSdkVersion="11" android:targetSdkVersion="12"/>
            <supports-screens android:anyDensity="false"
        </manifest>
 </android>

      you have to use midSDKVersion = 11 (which is Android 3.0) to get new black-blue tabbed UI

      3) and if it has some thing like this:
      android:name=&quot;.YourAppActivity&quot; android:theme=&quot;@style&#x2F;Theme.Titanium&quot;&gt; you need to delete it cause it uses custom theme
