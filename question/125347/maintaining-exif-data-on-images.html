Maintaining EXIF Data on Images

In my (iPhone) app, users can snap a photo (or grab one from their library). After adding a caption, I send it to the server through XHR. On the other end, the image is received (for either function: camera or photo gallery) but there is no EXIF geolocation accompanying the image. There's orientation, size, and such but no geolocation.

I've tried sending both the raw event.media as given to the success method of both showCamera and openPhotoGallery, but neither of those have geolocation available. I'm not resizing the image.

I've also tried this method to see if something might show up, but nothing. I've also tried this with both toImage() and toBlob(), but both fail.

I don't need to read the EXIF data in the app, I just want it there when I POST it to the server. I've seen suggestions of grabbing the current geolocation when taking a new photo, but that's really a last-resort-type solution for me.

My questions: why is this data getting stripped out? How do I prevent this from getting stripped out? Has anyone built a module that can do this accurately?
