Rotate animation not working on iOS... fine on Android!

Hi all,

Could anyone please explain why this works just fine on Android and the view spins as expected but on iOS it won't budge! The image IS displayed but it doesn't spin.

I have tried using an imageView instead - no change. I have tried taking all the properties out of the declaration eg:

var rotation = Ti.UI.create2DMatrix({});
rotation.rotate = 2880;

I have tried reducing all the properties down to a bare minimum… no change.

Like I say - it all works perfectly on Android… There are several things I 'spin' but nothing I animate using rotations works properly on iOS!

var workingView = Ti.UI.createView({
        right: -10, 
        top: bannerHeight + (((screenHeight - bannerHeight - adMobView.rect.height) * 0.5) - Math.floor((bannerHeight * 0.4))),
        width: Math.floor((bannerHeight * 0.4) * 2),
        height: Math.floor((bannerHeight * 0.4) * 2),
        backgroundColor: 'transparent',
        zIndex: 900000006,
        backgroundImage: '/images/brown.png'
    });

    var rotation = Ti.UI.create2DMatrix({
        rotate: 2880
    });

    var animation = Ti.UI.createAnimation({
        transform: rotation,
        duration: 7000,
        autoreverse: true,
        repeat: 999
    });
    if (android) {
        animation.anchorPoint =  {x: 0.5, y: 0.5};
    } else {
        workingView.anchorPoint = {x: 0.5, y: 0.5};
    };

    gameWindow.add(workingView);
    workingView.animate(animation);

Many thanks in advance,

Robin

  • Hi Robin Williams

    If you want a continious circular rotation like android in iOS, it is not possible. Because Android make your workingView to rotate 8 complete cycle when you set 

     var rotation = Ti.UI.create2DMatrix({
        rotate: 2880
    });

    But in iOS the rotation angle is normalized to 0 degree (8360+0=2880). So set rotation angel not like **n360** and see the change
