  • A typical web authentication mechanism will use a token stored in cookie to authenticate each subsequent request after a username and password have been provided. This is more secure than repeatedly passing the username/password as it can the token value can be refreshed on each request and can also be set to expire.

    It looks like the mobile API does not support cookies, but the same mechanism can be easily replicated by returning a token with each XML/JSON response and passing it on each request as a post or get parameter. By storing the token on the device DB instead of as a property you can build local expiration of the token by including an expiry field with the token record. I'm in the process of modifying a PHP authentication module for Symfony to do exactly this and had been working out nicely.
