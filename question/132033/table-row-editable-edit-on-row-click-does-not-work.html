Table row editable "edit" on row click does not work.

Hi,

So I am replicating an existing iPhone app for a client, the app right now is native so I am converting to Titanium but have the following issue which is possible in native iOS but I am currently struggling with Titanium to create the same functionality.

They currently have a table view where the default on click functionality selects a record and goes back to the last screen. On making the table editable you can delete a record from the sqlite DB or clicking on the row and not the delete icon allows you to edit the record by opening the next screen. I have tried doing this with Titanium and thus far the click event is not fired at all in the editable mode. Is there something I am missing?

Right now the only way round it that I can think off is to add a button to the left of each row in the non editable mode which is less than ideal but a way round it if this is a limitation of Titanium.

Thanks in advance,

Alex

    you can set the property
    allowsSelectionDuringEditing:true

    so that you can select the row at edit time
