  • Joshua

    You have not stated your platform or SDK versions. Also, without all the pertinent parts of your code, particularly the CREATE TABLE statement, it's difficult to give you a definitive answer. It's for this reason that you should include this information as the bare minimum for any question you post to the Q&A in future.

    Have you seen the SQLite Databases guide? This explains that string substitution should be used for queries. Hence:

    var db = Titanium.Database.open('categories');
db.execute('INSERT INTO main.categories (category_fname, category_lname, category_dob, category_address1, category_address2, category_city, category_state, category_zip, category_gender, category_homePhone, category_workPhone, category_cellPhone, category_title) VALUES (?,?,?,?,?,?,?,?,?,?,?,?,?)', fname.value, lname.value, dob.value, address1.value, address2.value, city.value, state.value, zip.value, gender.value, homePhone.value, workPhone.value, cellPhone.value, title.value);

    As per the KitchenSink database_2.js script, there is no need to assign an insert statement to a variable.

    Hope this helps
