iOS/Android Mobile: Fire WebView doubletap event programmatically

I'm using a build of Titanium Mobile SDK 1.7, and the latest iOS and Android SDKs. This is targeted specifically to mobile devices.

I want to "fake" the doubletap event on a webview in order to programmatically zoom+scroll the user to a specific part of a page. I've been able to do work arounds on iOS utilizing a ScrollView, but zooming isn't available for ScrollViews on Android.

The code I'm trying below attempts to fire the doubletap event after 3 seconds, and I'm passing in the coordinates I'd like to zoom+scale to, but it's not working, no errors – it just doesn't do anything.

The placeholder object I'm passing in as a param is from listening to the doubletap event and logging the returned event object.

var win = Titanium.UI.createWindow({  
    title:'Tab 1',
    backgroundColor:'blue'
});


var webView = Ti.UI.createWebView({
    url: 'blah.html',
    scalesPageToFit: true, 
    setZoomScale: 0.5, 
    top: 0,
    left: 0,
    zIndex: 1,
    disableBounce: true,
    scrollsToTop: false
});

setTimeout(function(){
    webView.fireEvent('doubletap', {
        globalPoint: {
            x: 159,
            y: 385
        },

        source: webView,
        type: 'doubletap',
        x: 159,
        y: 365
    });
}, 3000);

win.add(webView);

/*
webView.addEventListener('doubletap', function(e){
    Titanium.API.info(e);
}); 
*/


win.open();

Any thoughts on what I can do? There is no possibility to change my UI to find an alternative solution. This is a very integral part of my application.
