  • I don't think there is yet a good MVC framework for Titanium. That being said, you don't actually need one.

    Adding a framework to a framework would increase processing and app size..

    So a solution is to build your own based on namespaces and commonJS.

    Separate you code into separate Controller, Model and View files and create a helper section.

    A quick way define you Namespaces.

    var APPNS = {};
    APPNS.CTL = {};
    APPNS.MDL = {};
    APPNS.VIW = {};

    Then you can have a helper NS

    var APPHP = {};
    APPHP.LO = {}; for layouts

    etc etc

    Then you can create a file in the proper directory structure for each one which is loaded using an anonymous function and name each function to meet your requirements.

    Abstract the code into these files and name the functions you have an MVC architecture without using a framework, without adding event listeners everywhere as you can call your namespace functions from anywhere and one which should reduce the size of your code base as you abstract the code and reduce duplication.

    Hope this helps
