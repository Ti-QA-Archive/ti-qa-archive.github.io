softkeyboard covers textfields in android but not iphone

I set a scrollview on a window and put my elements in the scrollview. This allows the iphone to scroll up the screen when the soft keyboard pops up. On the android however, it only scrolls up a little bit, getting worse and worse as I move down to more fields. On the device and not the simulator. My client brought it to my attention as it is doing this on his HTC. I dont recall this being a problem before. I am using the 1.8.0.1 sdk, on a mac, newest studio. Its like its 20 pixels off on the first field, then 40, then 60 etc. (I tested on an archos) Any ideas?

    set to the window:

    windowSoftInputMode: Titanium.UI.Android.SOFT_INPUT_ADJUST_RESIZE,

    it should work.

    • Problem is, when using _RESIZE the textfield loses focus (when in a tableview) - if you use SOFT_INPUT_ADJUST_PAN, it does not lose focus, but fails to scroll properly. This has been an issue for over a year now, with no resolution.
