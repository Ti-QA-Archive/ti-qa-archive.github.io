Geolocation: difference between blue dot and location events

Hello all,

I'm working on a geoloc based app. Instead of displaying the built in blue dot (that's another topic but pay attention it's not I18n) I'm displaying the user location based on location events and the use of Titanium.Geolocation.getCurrentLocation().

I've tried a lot of settings combinations and finally went back to basics (coffeescript):

Titanium.Geolocation.purpose = "Real purpose here"
Titanium.Geolocation.accuracy = Titanium.Geolocation.ACCURACY_BEST
Titanium.Geolocation.addEventListener 'location', (e) ->
  Titanium.App.fireEvent("app:user_moved", e)

Seeing that it wasn't working great I decided to set userLocation to true on my maps to compare.
The blue dot is always at the right place and updates frequently and correctly whereas my own pin is almost never right and takes lots of time to update.

Even if I could consider using the blue dot to show the user it's location it still means that I don't have the correct location listening to the events or calling Titanium.Geolocation.getCurrentLocation().

Are there any other ways to get the location of the built-in dot?
Any means to check Titanium.Geolocation.distanceFilter does it's job (tried with 0/default and 1 but same results)?
Something else?

Any ideas/thoughts would be greatly appreciated!
Thank you,
Jb
