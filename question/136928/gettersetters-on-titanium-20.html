Getter/setters on Titanium 2.0

Just downloaded 2.0 and suddenly my project is showing "unexpected token" errors for all my prototype getter/setters. The project still debugs successfully despite the errors. Is there something wrong with using these? Other prototype functions do not show errors:

SHOWS ERROR
MyClass.prototype = {
get foo(){
return "bar";
}}

DOES NOT SHOW ERROR
MyClass.prototype = {
getFoo : function(){
return "bar";
}}

    You can use ECMA5 features:

    Object.defineProperty(MyClass.prototype, 'foo', {
    get: function() {

    },
    set: function(value) {

    },
    enumerable: false,
    configurable: true
});
