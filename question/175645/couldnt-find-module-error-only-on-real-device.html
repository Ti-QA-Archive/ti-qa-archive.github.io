Couldn't find module error only on real device

Hi,
I am new to Titanium so excuse me if this is noob. I have an Alloy project with the latest sdk. I have created a simple JS file in "app\lib" folder. For example; "app/lib/Person.js". It's a very simple code;

function Person(){
   this.Name = 'person name';
   this.Age = 0;
};
module.exports = Person;

Then I call this from a controller like this;

var oPerson = require('Person');
var prs = new oPerson();
$.labelAge.text = prs.Name;

This works in iPhone 4 simulator. But when I deploy this to an iPad Air real device, it says "Couldn't find module: Person".

I'm not sure what to do. I have found a similar case in Q&A but that was because the dev put the module file in a folder called "iPhone". This is not the case for me. (https://developer.appcelerator.com/question/136031/couldnt-find-module-error-on-device)

I will be happy to provide any additional info if required. Thanks.

  • I solved the problem and would like to share it for future reference.

    Strangely, in the trace log, I saw a line including "…person_js…" starting with "p" not "P". Don't know if the realdevice somehow tries to search the module with small case first letter. No clue on this as I told before, the code was working on the simulator just fine.

    I have renamed all my "Person" to "person" including the module name, the JS file name and all the references. It worked great both in simulator and the iPad device.

    Note: I tried several clean/build cycles and that did not work. This was solution in some of the similar cases I found in the Q&A.
