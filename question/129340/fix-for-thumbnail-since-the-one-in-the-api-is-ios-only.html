Fix for thumbnail (since the one in the API is iOS only)

I had an app where loading remote images (and sizing them) was being a pain (for Anrdoid, naturally).

Below is a snippet of code I use that works flawlessly on both iPhone and Android.

What it does is get the remote image (in this example, I have it passed from a previous window, so change the image on the 'temp' view to match your needs.

Next, it converts it to a blob to get the height/width, and for my purposes, divides it by 2 (since these images are rather large).

I decided to do it this way, because in Android, after the image was loaded, I would still get 'auto' for the width/height, in iOS it worked fine.

var productImageTemp = Ti.UI.createImageView({

   height:'auto',
   width:'auto',

   image: win.imagepath //this is the remote image
});


var blob = productImageTemp.toImage();


var productImage = Ti.UI.createImageView({
   top:60,
   left:10,
   height:blob.height/2,
   width:blob.width/2,
   image:win.imagepath //still the remote image, with smaller constraints
  });

win.add(productImage);
