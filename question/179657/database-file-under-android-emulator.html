  • The database goes in the same place whether you're testing with the emulator or a device. For Alloy projects, put it in the assets folder. For a Classic project, put it in Resources.

    There is no "isEmulator" function. However, you can differentiate by using 

    if(Ti.Platform.model == 'google_sdk') {
  // running with the stock Google emulator
} else if(Ti.Platform.manufacturer == 'Genymotion') {
  // running the Genymotion emulator
} else {
  // running on a device
}

    • Sorry I was not clear, you don't need to do anything special. Put the db file in your assets folder and use the same command to open it regardless of whether you're running on the emulator or device. Don't add the file extension.

      var db = Ti.Database.open('mydb');
