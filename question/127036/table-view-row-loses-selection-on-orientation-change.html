Table view row loses selection on orientation change

When I have a row selected and then rotate my iPad, the row and child labels lose all of their selected values (selectedBackgroundImage, highlightedColor, etc…). Basically the row is deselected. Is there a way to retain the selection when the orientation changes?

— asked October 18th 2011 by George Hess

  • Bump, same issue. Even manually selecting and scrolling to a row with the following does not work on the iPad:

  • Regarding the selectRow function. I had to edit the TiUITableView.m file found in /mobilesdk/osx/1.7.3/iphone/ to get it to work.

    Change:

    if (initiallyDisplayed==NO && [indexPath section]==[(TiUITableViewProxy *)[self proxy] sectionCount]-1 && [indexPath row]==[section rowCount]-1)

    To just:

    if (initiallyDisplayed==NO)

    Make sure you edit the SDK files and not the ones in your app's build folder. You will have to delete your existing build so that the changes copy over next time you run it.

    — commented November 4th 2011 by George Hess
