Titanium Studio 2.0 Autocomplete - Local Variables

Local variables do not appear in content assist. I believe this has been happening since upgrading to Titanium Studio version 2. Example:

function myFunction(myParm){
    var newLocal = my

}

With the cursor at the end of var newLocal = my I would expect autocomplete to suggest myFunction and myParm. However, it only shows myFunction. If I were to continue the example and later look for newLocal in autocomplete, it would not be there.

I've cleaned my projects, and tested with different versions of the SDK, it makes no difference.

Anyone?
