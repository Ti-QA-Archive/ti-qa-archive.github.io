ListView's itemclick event doesn't work after updateItems

Hello, i have a listView inside a tab in my android app with custom template.

index.xml

<Tab id="contactTab" title="L('tab_contact')">
<Window id="contactWindow" title="Tab 3">
<ListView id="contactListView" defaultItemTemplate="template1">
<Templates>
    <ItemTemplate name="template1" height="60">
        <ImageView bindId="bindImage" id="iconImage"/>
        <Label bindId="bindLabel1" id="tLabel1"/> 
        <Label bindId="bindLabel2" id="tLabel2"/>
    </ItemTemplate>
</Templates>
<ListSection id="phoneSection">
    <HeaderView><Label id="header_phone"/></HeaderView>
    <ListItem itemId="phoneItem" bindImage:image="/images/ic_action_call.png" bindLabel1:text="-" bindLabel2:text=""/>
.
.
. (more listSections and listItems)
.
.
</ListSection>
<ListSection id="emailSection">
    <HeaderView><Label id="header_email"/></HeaderView> 
    <ListItem itemId="emailItem" bindImage:image="/images/ic_action_email.png" bindLabel1:text="-" bindLabel2:text=""/>
</ListSection>
</ListView>
</Window>
</Tab>

and the listener for itemclick.

index.js

$.contactListView.addEventListener('itemclick', function(e) {
var item;
    if (e.itemId == "phoneItem") {
        Ti.API.info("phoneItem clicked");
    }else if(e.itemId == "emailItem"){
        Ti.API.info("emailItem clicked");
    }
}

The problem is that when the app starts, listView and itemclick listener works ok, but when i fill listView with data, event listener "itemclick" doesn't work. The code that fills the listView is:

index.js

function setData(phone1, ..., mail) {
    $.phoneSection.updateItemAt(0, {
        bindImage : {
            image : "/images/ic_action_call.png"
        },
        bindLabel1 : {
            text : phone1
        },
        bindLabel2 : {
            text : ""
        }
    });

    $.emailSection.updateItemAt(0, {
        bindImage : {
            image : "/images/ic_action_email.png"
        },
        bindLabel1 : {
            text : mail
        },
        bindLabel2 : {
            text : ""
        }
    });
}

I've also tried to put itemclick listener like this:

index.xml

<Tab id="contactTab" title="L('tab_contact')">
<Window id="contactWindow" title="Tab 3">
<ListView id="contactListView" defaultItemTemplate="template1" onItemclick="listViewItemClick">
.
.
.
</ListView>
</Window>
</Tab>

index.js

function listViewItemClick(e){
var item;
    if (e.itemId == "phoneItem") {
        Ti.API.info("phoneItem clicked");
    }else if(e.itemId == "emailItem"){
        Ti.API.info("emailItem clicked");
    }
}

But the result is the same. I suspect that when an item has been updated, it turns off the itemclick event listener of the whole listView. Is this a bug or a normal behavior? and how can i avoid it?
I am using Titanium SDK 3.3.0 GA

  • The itemId should be unique for each data element being loaded into your list. It looks as though you're setting them all equal to phoneItem (or emailItem). Thus, the event listener is getting "confused." See the ListItem docs page for more info on eventing with list views.
