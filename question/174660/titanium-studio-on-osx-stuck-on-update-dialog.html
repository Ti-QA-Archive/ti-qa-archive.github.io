Titanium Studio on OSX stuck on update dialog

I was working with Titanium Studio on my MacBook Pro for almost a day without problem. Then the update dialog pops up and I was stuck there. The problem was that all the buttons, checkboxes on the dialog has no response, and it can't be closed. I could however access the Studio preference menu, tried to disable auto update, Command-Q, restart Studio. Same dialog problem! Now my studio is unworkable.

I've tried manual install the mentioned module updates. Deleted the app folder from /Applications, removed ~/.titanium and ~/Library/Application Support/Titanium/ . Reinstalled few times, nothing changed.
I also tried from another user account in the same OSX and still got stuck.

However I tried update Titanium Studio on my other Mac Mini, there's no such issue.

Any help ?

  • The issue still persisted on my other workstation for sometime even with manual install.

    Finally I reached the true solution to the issue: http://stackoverflow.com/questions/16151018/npm-throws-error-without-sudo/24404451#24404451

    It's the ~/.npm folder ownership was messed up by previous improper CLI install of npm components. Probably caused by the missing of '-g' parameter.

    Now use:

    sudo chown -R `whoami` ~/.npm

    Then the problem is gone.

  • Have you checked your $PATH with "echo $PATH". I believe I had a similar issue because titanium was looking in /usr/bin while the files were actually in /usr/local/bin.

    To fix this I just created a symlink in /usr/bin "sudo ln -s /usr/local/bin/npm /usr/bin/npm"

    Hope this helps.
