    Hi

    You are not going to like the answer; iOS support any view being a parent to any other view (rightly or wrongly) - but Android ONLY allows child views to be place inside container views.

    Examples of container views;

    • View
    • ScrollView
    • Window

    Examples of things that are not container Views;

    • Label
    • Button
    • ScrollableView (this takes views as a parameter but not as children

    In order to get around the problem you have to think slightly differently;

    <View class="container" >
    <Button id="btn" />
    <View id="line" />
    <Label id="day" />
</Button>

    In this example you use a view to be the container and simply stack the other items on top of each other, things are absolutely positioned so you have control over the placement of course.
