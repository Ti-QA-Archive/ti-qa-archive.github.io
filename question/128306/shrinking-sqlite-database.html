  • Hi Shlomi,

    you can simply execute the SQL "vacuum" and SQLite will shrink the database. As Nick told this is only necessary in case you inserting/updating/deleting data in the database at large.

    If you run serious queries on the SQLite and if you use indexes you should run the SQL statement "ANALYSE" as well.
    The analyse statement will update the statistics used by the SQLite query optimizer.
