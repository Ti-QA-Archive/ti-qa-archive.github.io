PubNub Background Service

Hello All,

In an Android app that I am writing I need to subscribe to PubNub messages, receive them even when the app is nto running and reactivate the app when a message is received. From what I have read it sounds like this is possible but I am getting lost in a quagmire of Android Intents, background services… . Are there any good sources of information, comprehensive examples, on the subject out there? All my Googling has not turned up anything that is solid and reliable.

  • Hi Et! You have an excellent question: "How to Receive PubNub Messages when the app is Not Running." Good news! This is possible. You are correct that you need to receive messages via a Background Service. You will subscribe to PubNub in the Background Service via PUBNUB.subscribe({...}) function. Then you will use Android Intents to communicate the message between your app and the background process. Note that the background process can be run on Android Boot-up, so it will always be connected to PubNub.

    Next you will use the background process to wake-up or launch the non-running app and send notifications "intents" to your real app. I have some code already that you can use to register the Android Background Service.

    Example Background Process Code:
    https://github.com/pubnub/pubnub-api/tree/0dfd8028b803332f5641adc909b1a26f87bd7ff1/android/PubnubAndroid/src/com/aimx/androidpubnub

    Use the code you see in this directory to better understand how to connect PubNub and Android with using a Background Service. Note that this is a working example, however you need to morph this example code to match your needs with Android Push Notifications using PubNub.
