Does using JS patterns while writing the code translate into better running apps?

Hi

Just started using Titanium and was thinking how, or if, using Javascript patterns in the code would translate into better running Android/iOS code (if nothing else is wrong in the code)?

What I'm thinking is for instance using a prototype pattern to minimize the memory usage, but would that translate into less memory usage on the mobile platform?

If anyone knows and would like to share, or has some reading/watching resource that might cast a light on this issue I'd really appreciate it.

  • Accepted Answer

    Since Titanium's JS code still runs inside JS engine (v8 on Android and JSC on iOS), then of course that good JS practices makes your code run faster.

    Take a look at performance tips for v8, writing JS code that can be optimized by v8 makes your code run very fast.

    • Yeah, a lot people assume that Titanium compiles JS code to native (Java or Objective-C) code. But that is not true, JS code is still interpreted at run time. From that point of view, Titanium is not different from PhoneGap (Callback, or whatever its name is now) and similar products, but still Titanium is different (read - better) because in the background it uses native components.

      See this article to get more depth on this subject: http://developer.appcelerator.com/blog/2012/05/comparing-titanium-and-phonegap.html
