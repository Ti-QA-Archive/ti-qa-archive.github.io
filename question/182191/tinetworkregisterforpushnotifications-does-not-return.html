Ti.Network.registerForPushNotifications does not return

Ti.Network.registerForPushNotifications are not working. None of the callbacks are working. Is there anyone who is successfully using this code? almost all of the questions I can find on this site are more than four years old?

Our project in using pushwoosh, and once we can successfully retrieve a valid device token from APNs everything is fine.

I have pretty much followed the documentation to the letter, but the following behaviours in iOS8 seem inconsistent.

  1. iOS8 seems to cache UserNotificationSettings so the usernotificationsettings event never fires
  2. calls to Ti.Network.registerForPushNotifications do not call the callback

is this expected behaviour if there is an existing device token?

presumeably Ti.Network.registerForPushNotifications is sending some kind of UUID to APNs? if so what is it sending, and should I be calling unregisterForPushNotifications first.
It seems to me that there are alot of unanswered questions on this topic on this forum and on stack overflow, if it is possible to get some clear information about this part of the Titanium API it would be good to clarify exactly how to use it for the good of the community.
