    • Hi Moritz

      Thanks for your help! I did some testing - starting out from the standard sample that comes with the nl.fokkezb.drawer widget. And I then found out when it breaks.

      If I use this tss (with the above markup) - then it works:

      "Window": {
    navGroup: true,
    backButtonTitle: '',
    translucent: true,
    navTintColor: '#fff',
    statusBarStyle:Titanium.UI.iPhone.StatusBar.LIGHT_CONTENT,
       backgroundImage:'images/background.png',
    backgroundColor: 'transparent',
    extendEdges: [
        Ti.UI.EXTEND_EDGE_TOP
    ],
    hideShadow:true,
    barImage:'images/transparent.png'
}

      However, I do not want all windows to have this layout (specifically the drawer window). So I tried to use an id:

      "#winContent": {
    navGroup: true,
    :
    :

      … and then the statusbar turned black. Using a class instead made no difference - like:

      ".container": {
    navGroup: true,
    :
    :

      So that brought me on the right track to a solution. I now use two "selectors":

      "Window": {
    statusBarStyle:Titanium.UI.iPhone.StatusBar.LIGHT_CONTENT,
}
"#winContent": {
    navGroup: true,
    backButtonTitle: '',
    translucent: true,
    navTintColor: '#fff',
    backgroundImage:'images/background.png',
    backgroundColor: 'transparent',
    extendEdges: [
        Ti.UI.EXTEND_EDGE_TOP
    ],
    hideShadow:true,
    barImage:'images/transparent.png'
}

      Not intuitively easy to see why this is needed - but hey, now it works! Thank you for pointing me in the right direction ;-)
