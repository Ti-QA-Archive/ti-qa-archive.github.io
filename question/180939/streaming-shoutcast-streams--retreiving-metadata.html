  • Something like this will get the icy-metaint value as described in that smackfu article

    
 var url = "http://stream.dradio.de/7/530/142684/v1/gnl.akacast.akamaistream.net/dradio_mp3_dkultur_m";
 var client = Ti.Network.createHTTPClient({
     ondatastream : function(e) {
         Ti.API.info("Received icy-metaint header value : " + this.getResponseHeader('icy-metaint'));
         this.abort();
     },
     onerror : function(e) {
         Ti.API.debug(e.error);
         alert('error');
     }
 });
 client.open("GET", url); 
 client.setRequestHeader('Icy-MetaData', 1);
 client.send();

    Then, I suppose it would be a question of setting something up (maybe using the FileStream IOStream methods) to use that buffer value to read the data?

  • Depending on the version of the ShoutCast server being used, there's a file called 7.html which you can grab and parse.

    eg: http://someserver.com:someport/7.html

    I've never had any luck with extracting any metadata using the standard Ti AudioPlayer. But then, that whole functionality needs updating, since it doesn't even have an 'error' event, let alone anything to detect metadata.

    Maybe there's a module which can?
