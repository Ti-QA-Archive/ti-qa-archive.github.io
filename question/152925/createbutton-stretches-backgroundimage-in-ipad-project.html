createButton stretches backgroundImage in iPad project

Hello, in my iPhone/iPad project I've some buttons created this way:

var homeBtn = Ti.UI.createButton({
     top:'25%',
     left:'1%',
     title:" ",
     backgroundImage:'/graphics/Home_off.png',
     backgroundSelectedImage:'/graphics/Home_on.png',
     width:331,
     height:236
     });

No problem when I run the code in the iPhone simulator (and device too): it perfecly works.

The same code (of course, except for the "@2x" part and width/heigth part) in iPad creates a problem: the image appears tremendously stretched and the effect is terrific.

I tried to change the button to an imageView but in that case, even if the image looks the way it should, I lost the selected state, so when I press the image it does not change.

Can you help me?

  • Hi

    Is the button image you used the exact same pixel width and height as the space shows on the screen? If not this is the problem as the button will automatically stretch to fit.

    Are the filenames of the two resolutions case matched? For safety I ALWAYS name my images using only lower case as the simulator does not care but the devices are very specific.

    Check; Home_off.png & Home_off@2x.png both have ONLY the initial caps.
