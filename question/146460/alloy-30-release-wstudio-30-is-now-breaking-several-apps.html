Alloy 3.0 release w/Studio 3.0 is now breaking several apps

My project uses similar code as in the alloy_fugitive example app. Specifically, I have a loop creating rows that uses a data model. This use to work in the previous versions of Alloy and Titanium Studio 2.1.3GA and 2.1.4GA. Now that I have upgraded, the projects are breaking the iOS simulator hard without error. I upgraded the Alloy fugitive app to be sure it wasn't my code and the same error occurs in fugitive. I narrowed it down to this line of code see where the arrow is.

var rows = [];
for (var i = 0; i < capCollection.length; i++) {
    var model = capCollection[i];
    var row = Alloy.createController('FugitiveRow', model.toJSON()).getView();
    row.model = model;   // <--------This line is now causing an error
    rows.push(row);
}

Any help is much appreciated.

      captured.xml

      <Alloy>
    <Tab id="capturedTab" title="Captured">
        <Window id="capturedWindow"  title="Captured" class='container'>
            <TableView id="table" model=""/>
        </Window>
    </Tab>
</Alloy>

      captured.js
      inside the function populateData() method change this from

      for (var i = 0; i < capCollection.length; i++) {
    var row = Alloy.createController('FugitiveRow', model.toJSON()).getView();
    row.model = model;
    rows.push(row);
}

      to 

      for (var i = 0; i < capCollection.length; i++) {
    var model = capCollection[i];
    var rowCtrl = Alloy.createController('FugitiveRow', model);
     rows.push(rowCtrl.row);
}
