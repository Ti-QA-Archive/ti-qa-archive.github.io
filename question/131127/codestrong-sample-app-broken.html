Codestrong Sample App Broken

Can tonylukasavage or somebody please update the GIT repository for the Codestrong sample app to the point where it works correctly?

I created a project in Titanium studio directly from the GIT repository today (1/24/2012) and tried to run it using an Android 2.3.4 phone and almost nothing works correctly. The dashboard icons don't show up, switching tabs causes a Runtime error in cs/ui.js, random freezes, "Web page not availiable" errors for file:///android_asset/ urls, etc.

I think most, if not all of the problems are due to missing/misplaced image/resource files. I downloaded the app from the Android Marketplace and it works just fine. Why can't the GIT repository at https://github.com/appcelerator/Codestrong.git reflect the code-base that works properly?

Learning how to program effectively on the Titanium platform is quite difficult with the existing state of Documentation and it's 10 times harder when the sample app that works correctly is Kitchen Sink and it's a terrible example of how to design a Titanium app.
