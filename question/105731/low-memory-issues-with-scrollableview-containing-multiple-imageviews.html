Low memory issues with ScrollableView containing multiple ImageViews

I have a ScrollableView which contains 8 ImageViews. Each ImageView is linked to a local image. The image sizes are between 400-500kb each, so we're looking at a little under 4mb total.

Everything works wonderfully on the Simulator. However on an actual iPad, I see all kinds of strange behavior.

  1. Images flicker rapidly, even when the iPad isn't being touched.
  2. The ScrollableView jumps between pages seemly at random.
  3. The application crashes entirely. The iPhone Configuration Utility console shows various kinds of Low Memory errors in this case.

I can reproduce this behavior on several different iPads running iOS 3.2 and iOS 4.2.

Is this a known issue? If so, what memory limitations do I have to deal with and what are the recommended solutions?
