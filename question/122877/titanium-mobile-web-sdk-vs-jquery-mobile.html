Titanium mobile Web SDK vs jQuery Mobile

Hi,

for a real assignment, I have to develop a HTML5 mobile web. All the web content is static (no web services nor server logic). I'm just considering JQuery Mobile (beta) and Titanium Mobile Web SDK (in alpha state).

Not sure how these technologies can complement each other, if they are both exclusive alternatives (one or the other) or if can work together. Any idea?

Finally, when the web is finished, I will have to do an app for iOs, which will encapsulate the files an show them in a webView. So there will be an HTML5 version for all platforms and an app for iOs. Do you think this an important point to take in consideration now? I think both Jquery and Titanium mobile web should work the same in a web view. Any suggestion?

thanks,
Javier

  • The clear advantage of using Titanium Mobile Web SDK would be that you can easily compile it to a native app (for both iOS and Android, even though you might only need iOS at this point).

    The extra benefit is that it's the same API and therefor the same syntax (and basically the same code) for using Titanium Mobile Web and Titanium Mobile, so packaging it to an app after you've built the site will be a piece of cake.

    So it turns out the cake is not a lie after all.
