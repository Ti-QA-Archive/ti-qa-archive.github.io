App name internationalization not working in Finnish language (Android)

I followed instructions here with a new project, and tested it also with AppNameLocalization project.

I added values-fi -folder, since Finnish language code is 'fi'. ISO language codes.

When running with Android 2.3 device and an Android Emulator, the app name is listed as "com.appcelera…", when using the device in Finnish language. Other languages seem to work, tested also with Swedish and English.

App name localization works perfectly on iOS and all strings inside i18n -folder are localized correctly also on Android.

  • Solved! Seems to work if you put &lt;string name=&quot;app_name&quot;&gt;app name&lt;&#x2F;string&gt; to i18n&#x2F;fi&#x2F;strings.xml instead of platform&#x2F;android&#x2F;res&#x2F;values-(language code) if you have i18n strings localization in your application.

    • That's for iOS. For Android the only way it worked for me was by following these instructions from "Android App Name Localization", but instead of using /res/values/fi/strings.xml I used /i18n/fi/strings.xml. Actually, the whole /platform/android -folder with AndroidManifest.xml is not needed, if you put your custom manifest directly to tiapp.xml.
