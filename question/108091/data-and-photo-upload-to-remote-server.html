    • Forrest – I finally see what you are trying to do. You don't need the temporary file at all. The problem you are hitting is that you define variable contents within the success callback, lines 192-215 of the original pastie but then try to reference it in another code block around line 121. However, variable "contents" is going out of scope when the success callback completes.

      Add this near the top of the file, say around line 2:

      var contents;

      Now around line 213 change:

      var contents = tempFile.read();

      to

      contents = image;

      And get rid of the temporary file stuff – you don't need it. Actually, you could just define var image at the top of the file and use it instead of "contents". The whole problem is what is known as the "scope" of variables. Those you define within a function are only valid during that function call. Those you define at the top of the file persist their value across calls and are available throughout the *.js file.

      For the same reason, you could not use "event" around line 121 either. Here is one explanation of scope.
