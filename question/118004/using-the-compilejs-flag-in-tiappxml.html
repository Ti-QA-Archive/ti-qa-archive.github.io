Using the compilejs Flag in tiapp.xml

Hi everyone! I've been working with appcelerator for a few weeks now and I'm getting ready to do some Beta testing for my Android app.

I can run the app in the emulator, and I can package the project for the phone successfully under the Distribute tab.

However, as I get ready for Beta testing distribution, I'm curious about "distribution" mode and bytecode generation for the final app. From what I've found online, and in the Q&A section, I just add the following line to the tiapp.xml:

&lt;property name=&quot;ti.android.compilejs&quot; type=&quot;bool&quot;&gt;true&lt;&#x2F;property&gt;

I added it at the end of the tiapp.xml file, right before the closing </ti:app> tag.

The problem that I'm having is that the final apk is the same size with or without this flag set. Also, if I unzip the apk, I can see and view all of the js files. So, either I'm doing something wrong, or I'm not understanding what this flag is supposed to do.

The heart of the matter is that there are certain bits of code that I don't necessarily want public–or at least in plain text format in my APK. For example, I'm making calls to my REST server using API keys that I don't want to share with the world.

I'm looking for a definitive answer that tells me that there is or isn't a way, or some sort of distribution mode, that will create an APK that contains "compiled" JS files and not all of my code in plain text. I've seen many people asking this question, but with contradicting thoughts, and code bits (as in the above xml property) that aren't having an effect in my application. Like I said though, maybe my expectations are off base.

UPDATE: I am seeing this in my trace file when I run in an emulator followed by a list of each JS file, so it looks like it's doing something:

[DEBUG] Found ti.android.compilejs=true, overriding default (this may take some time) [INFO] Compiling javascript: foo.js ...

The problem is that I'm still ending up with all of the plain text JS files in an assets folder in the APK.

Thanks for the insight!

Matt

  • I think you are meant to add it in the <android> section of the tiapp.xml:

    <android xmlns:android="http://schemas.android.com/apk/res/android">
    <property name="ti.android.compilejs" type="bool">true</property>
</android>
