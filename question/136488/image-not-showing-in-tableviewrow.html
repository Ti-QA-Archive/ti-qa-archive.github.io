Image not showing in TableViewRow

Hi,
I have a typical tableviewrow setup. (A table is being build from an array of data).

var row = Ti.UI.createTableViewRow(
                {
                    height: 190,
                    selectedBackgroundColor:'#eeee33'
                }); 

var myImage = Ti.UI.createImageView(
                {
                    image: 'images/logo.png',
                    width: 30,
                    height: 190,
                    left: 45,
                    canScale: true
                });

row.add(myImage);

I tried to change that problematic line in my tiapp.xml file (android - fastdev = false) - still no luck.

What is odd, is that if I put the same 'images/logo.png' as a leftImage: 'images/logo.png' in the tableviewrow, the image appears. It isn't the right place so I cannot use leftImage (I need it in a specific location on the row).

Has anyone else had this problem?

Here is my Environment:

SDK Ti: 2.0.1
SDK XCODE: 4.3.2
Target iOS SDK : 4.3

many thanks for any (ANY) assistance you can provide.

I even tried to change the ImageView into a button, and it didn't display the image. I have the same setup on another part of the app, and it works just fine.
