Application Not Installed message

My android app builds fine in Titanium and it says it was successful. However, when the user attempts to install it on their phone for testing (outside store), they get a message "Application Not Installed." I'm using a self signed certificate generated by Studio.

When I use the Run config to install it directly on a device, the app works fine. This issue is with the app store distribution option.

Some posts indicate that it is related to the cert. I assume when I use the New button to generate the certificate in Studio, that it is creating a valid keystore?

Any ideas?

Thanks,
Don

  • Hi Don Kerr,

    Your installation process on device is right but I think your device don't get USB connection properly at first check it, or your phone don't have enough internal memory .For this reason you can install it in your SD card. And past this code in your tiapp.xml file then select your project and click install to android device . Then it will be installed in your SD card.

    <android xmlns:android="http://schemas.android.com/apk/res/android">
            <manifest android:installLocation="preferExternal" />
        </android>

    I think it would be work.

  • I resolved this issue myself. JDK 1.7 was the issue on my Mac OSX 10.8.5 . The keystore generated by Studio was not compatible. Unforrunately, Titanium doesn't throw an error. :( The signed apk does verify the signature, but Android still doesn't like it. I removed JDK 1.7 and recompiled the apk and it worked.
