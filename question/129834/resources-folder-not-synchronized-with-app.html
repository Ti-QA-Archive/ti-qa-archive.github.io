Resources folder not synchronized with .app

My folder structure:

Resources/images/

in images folder I have images and I read their names in code below:

var imagesPath = Ti.Filesystem.resourcesDirectory + Ti.Filesystem.separator + 'images';
var imageFiles = Ti.Filesystem.getFile(imagesPath).getDirectoryListing();

Ti.API.info(imagesPath);

for(var i = imageFiles.length - 1; i >= 0; i--) {
    Ti.API.info(imageFiles[i]);
};

If I delete some images from that folder, application doesn't know about that.
E.g. I have image1.png in it. I run the application and I get only one print in that for loop. Then I add image2.png and image3.png and I get 3 prints. Then I delete image2.png and image3.png and still I get 3 prints and later when I try to load that image I get an error because it obviously does not exist.

Project -> Clean is not helping. Only thing that I figured is to delete .app file from Terminal. That print before for loop shows me the application location.

How to fix this? It is frustrating to develop like this.
