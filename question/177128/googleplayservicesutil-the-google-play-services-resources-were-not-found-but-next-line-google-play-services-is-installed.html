    • After some more investigation and trial and error i found the solution, hopefully ;)

      So first of all the Jira Ticket doesn't describe a bug, but a feature Custom AndroidManifest.xml is being overwritten. Because Android specific entries from tiapp.xml and AndroidManifest.xml will be merged into AndroidManifest.xml.

      To protect your custom AndroidManifest.xml from overwriting with no values, you have to remove the android tag

      This custom AndroidManifest.xml will be overwritten:

      <?xml version="1.0" encoding="UTF-8"?>
<android xmlns:android="http://schemas.android.com/apk/res/android">
    <manifest xmlns:android="http://schemas.android.com/apk/res/android" package="com.freshpoint.freshpointv1" android:versionCode="501" android:versionName="0.0.0">
        <uses-sdk android:minSdkVersion="14" android:targetSdkVersion="14"/>
    </manifest>
</android>

      to this nearly empty one

      <?xml version="1.0" encoding="UTF-8"?>
<manifest xmlns:android="http://schemas.android.com/apk/res/android">
</manifest>

      This custom Androidmanifest.xml will be merged correctly:

      <?xml version="1.0" encoding="UTF-8"?>
    <manifest xmlns:android="http://schemas.android.com/apk/res/android" package="com.freshpoint.freshpointv1" android:versionCode="501" android:versionName="0.0.0">
        <uses-sdk android:minSdkVersion="14" android:targetSdkVersion="14"/>
    </manifest>

      So please try modifying your custom AndroidManifest.xml by removing the android tag:

      <?xml version="1.0" encoding="UTF-8"?>
       <manifest  xmlns:android="http://schemas.android.com/apk/res/android"
        package="com.freshpoint.freshpointv1" android:versionCode="501" android:versionName="0.0.0">
            <uses-sdk android:minSdkVersion="14" android:targetSdkVersion="14" android:maxSdkVersion="19"/>
            <supports-screens android:anyDensity="true"/>
            <!-- Allows the API to download data from Google Map servers -->
            <uses-permission android:name="android.permission.INTERNET"/>
            <!-- Allows the API to cache data -->
            <uses-permission android:name="android.permission.WRITE_EXTERNAL_STORAGE"/>
            <!-- Use GPS for device location -->
            <uses-permission android:name="android.permission.ACCESS_FINE_LOCATION"/>

            <!-- Specify OpenGL ES 2.0 as a requirement -->

            <!-- Replace com.domain.appid with your application ID -->
            <uses-permission android:name=".MAPS_RECEIVE"/>

      2) In Jira Ticket TIMOB-16510 Android: The Google Play services resources were not found Ingo Muschenetz suggests to use a newer version of the ti.map module.

      You can grab version 2.1.6 (ti.map-android-2.1.6.zip) from appcelerator-modules/ti.map branch: master

      • Download and extract the zip file
      • copy the folder 2.6.6 to /Users/yourUserName/Library/Application Support/Titanium/modules/android/ti.map
      • inside Studio open tiapp.xml
      • in TiApp Editor Overview select ti.map module, edit and select Version 2.1.6 for Android.
      • save tiapp.xml, clean your project and run your app on device from Studio
