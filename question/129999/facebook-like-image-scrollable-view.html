Facebook-like Image Scrollable View

Hi All,

I'm trying to create a scrollableView similar to the one that Facebook uses to display the recent photos (http://gigaom.files.wordpress.com/2010/08/androidhome.jpg?h=450&w=604).

Using the scrollableView component only displays the current image rather than the list of images with a centered, selected, image.

Does anyone know how to accomplish this?

Thanks,

    The dimensions are iPad sized.

    Should give you an idea, has 16 elements added via the loop to give you an idea.
    Just add the blogScroller to a window.

    Note that the contentWidth is much wider, this is like having a virtual canvas outside of the viewable area. 

    blogScroller = Ti.UI.createScrollView({layout:'horizontal', left:0, height: 171, bottom: 0, backgroundColor: 'black',contentWidth:2736,contentHeight:171});

for (var x=0; x<16; x++){
        var testviews = Ti.UI.createView({backgroundColor:'#262626', width: 171, height: 171, borderColor: 'black'});
        var testlabel = Ti.UI.createLabel({text: ' View '+x, color:'white'});
        testviews.add(testlabel);
        blogScroller.add(testviews);

    }

    • Since it is Christmas, I'll give some more away:

      updatePublicationsUI = function(orientation){
    if (orientation === 'portrait'){
        Ti.API.info('pub portrait');
        blogScroller.contentWidth=2736;
        blogScroller.contentHeight=171;
        blogScroller.layout='horizontal';
        blogScroller.height=171;
        blogScroller.width='auto';
        blogScroller.bottom=0;
        blogScroller.top=null;

    } else {
        Ti.API.info('pub landscape');
        blogScroller.contentWidth=171;
        blogScroller.contentHeight=2736;
        blogScroller.layout='vertical';
        blogScroller.width=171;
        blogScroller.height = Ti.Platform.displayCaps.platformHeight-(50+50);
        blogScroller.top = 50;
        blogScroller.bottom = 50;

    }
};

      Throw this function in an orientation change event, and it will move from horizontal scroll across the bottom, to vertical down the left side.

      — commented December 29th 2011 by Stephen Feather

    • I'm in a good mood, so here is sample code for your gesture event, the switch is to sort out only 90 degree changes, the ones we need to redraw for:

      Ti.Gesture.addEventListener('orientationchange',function(e){
    currentOrientation = '';
    switch (e.orientation){
        case Titanium.UI.PORTRAIT: 
            currentOrientation =  'portrait';
            break;
        case Titanium.UI.UPSIDE_PORTRAIT: 
            currentOrientation =  'portrait';
            break;
        case Titanium.UI.LANDSCAPE_LEFT: 
            currentOrientation =  'landscape';
            break;
        case Titanium.UI.LANDSCAPE_RIGHT: 
            currentOrientation =  'landscape';
            break;
    }

    Ti.API.info('got a orientation change event: '+currentOrientation);
    updatePublicationsUI(currentOrientation);
});

      — commented December 29th 2011 by Stephen Feather

    • I know your question was specific to creating a scrolling view, but wanted to add some usability thoughts into the mix as I feel that they go hand in hand. The code above is sized for an iPad, but the same concepts apply to all devices.

      We redraw ALL tabs at the time of orientation change behind the scenes.

      Reason 1:
      I can't trust the focus events to fire correctly.

      Reason 2:
      I personally don't like changing orientation, switching to a new tab, and then waiting for that tab to redraw its view. After I have made the decision to turn my tablet (iPad OR Android), I shouldn't have to wait again for a redraw every time I open a new window.
