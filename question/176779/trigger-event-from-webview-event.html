Trigger Event from Webview Event

Hi, I have a web application that I control. It has a page that is displayed in my app via a remote webview. On the page is a button that says "map this". On the web the button displays Google Maps via JS. When displayed in my mobile app I want to trigger some code to run the native map control. I know that I can't Use Ti.* from within my HTML, so here is my proposed approach:

  1. Conditionalize the button click so when it happens from within my mobile app it instead redirects to - mobileappmap.jsp (I did this already)
  2. In my appc code listen for the load event with e.url == &#39;mobileappmap.jsp&#39; then open my map window and redirect the webview back to the original page.

Does this general approach make sense? Is there a way to notify the appc code of the click without the extra page and redirect?

Note: the solution needs to work on iOS and Android.
