    Hi

    There was a weird old bug that meant that TextFields on iOS could not support the internal buttons (they would not show) unless the both internal button default were set.

    • Example TSS;
    "TextField[platform=ios]": {
    leftButtonMode: Ti.UI.INPUT_BUTTONMODE_ALWAYS,
    leftButtonPadding: 0,
    rightButtonMode: Ti.UI.INPUT_BUTTONMODE_ALWAYS,
    rightButtonPadding: 0
},

    This shows the important values to set as default.

    • In-line - same as your example (these should be in a TSS)
    <TextField
    id="locationfield"
    backgroundColor="#e6e6e7"
    borderRadius="8"
    color="black" 
    height="28"
    hintText="Location" 
    left="5" 
    leftButtonMode="Titanium.UI.INPUT_BUTTONMODE_ALWAYS"
    leftButtonPadding="0" <!-- new -->
    paddingLeft="45"
    paddingRight="5"
    right="165" 
    rightButtonMode="Titanium.UI.INPUT_BUTTONMODE_ALWAYS"
    rightButtonPadding="0" <!-- new -->
    top="25" 
>

    I am not sure if this was ever fixed - but by default now - if we add a TextField that needs one or both button, we use those settings.

  • Hi Paul,

    Your code is fine, but you have to add something to the left button such as height, width, title like this sample code. Try this

    <Alloy>
    <Window backgroundColor="red">
        <TextField id="locationfield" backgroundColor="#fff" borderRadius="8"  hintText="Location" width="200" height="Ti.UI.SIZE" top="30">

            <LeftButton>
                <Button onClick="sayHi" class="textButton" title="left" width="50" height="60" backgroundColor="green"></Button>
            </LeftButton>
            <KeyboardToolbar>
                <Toolbar>
                    <Items>
                        <Button title="Use Current Location" />
                        <FlexSpace/>
                        <Button title="Clear"/>
                    </Items>
                </Toolbar>
            </KeyboardToolbar>
        </TextField>
    </Window>
</Alloy>

    If you have any question, then let me know.
