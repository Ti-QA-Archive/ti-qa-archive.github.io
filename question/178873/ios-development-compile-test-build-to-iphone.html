iOS Development, compile test build to iPhone

I'm trying to install a development app on a iPhone, but my boss isn't here, he send my a UUID of his phone.

But in Titanium the only option that request a development certificate is: Run > iTunes Sync.

The other two Package > iOS Adhoc/Enterprise / Package > iOS iTunes Store require a production certificate.

I've run the project in Titanium with the option iTunes Sync, but I don't find any .ipa or .app file.

Here on Q&A I've found a topic with an answer "You have the .ipa file in project-folder/build/iphone/build/Debug-iphoneos", can I use that?

  • If you're not installing to a locally-connected phone, you'll need to do an ad-hoc build then use a service like Installr, TestFlight, or HockeyApp to get the IPA and provisioning profile to your boss. You could possibly email him the IPA and .mobileprovision files, and have him do an iTunes Sync – with the device connected and apps screen showing in iTunes, drag & drop both the IPA and mobileprovision files into the iTunes window then initiate the sync.
