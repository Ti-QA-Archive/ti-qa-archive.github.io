  • HI Steven

    Not enough information here to give any advice specific to your situation and realistically speaking you will have way too much code to provide for any one to have a stab at it.

    My suggestion would be work out when it last ran successfully on the simulator and try and determine what changes or sections of the app you add afterwards. Then where possible remove or hide those sections and see what happens. Ideally rolling back a few versions of any back-up code for test compiles would be a good plan.

    If this does not work - then you have two main ways;

    1. Plenty of use of Ti.API.info(&#39;debug&#39;, &#39;info&#39;); At least one at the start and end of every code file and module. Ideally start and end of every function, this will tell you if the code got that far, then find the last place and work from there.
    2. I always use a 50/50 approach when all else fails, remove 50% of the code from running and try the rest, if it works try the other 50%, when one fails repeat within that 50%. Really only useful though if you have broken your project up into CommonJS and/or Alloy techniques.

    I am guessing you have been testing as you have gone along so it must be recent - so wherever you were last is most likely.
