Getting Started Question

I was hoping you might be able to give me some advice on Mobile Application development. I have inherited a legacy system that comprises a mobile application running on Windows Mobile 6.5 devices that synchronizes with a windows desktop applicationvia a physical connection using a USB cable, the architecture is below..

Application Architecture
1)On the mobile device, a .exe program written in Visual Basic that executes against a Microsoft SQL Server 2005 Mobile Edition using .Net Framework
2)To synchronized data, a physical connection using a USB cable and Active Sync to push and pull data from the Desktop Application.
3)The Desktop Application is an extended version of the mobile running SQL Server Express, VB Application and .Net Framework.

As Windows Mobile 6.5 is now no longer available the legacy system can no longer be used commercially. Therefore there is a need to recreate the application that will run on Android devices (and possibly iPhone? or is Windows 7)

Can I use Titanium to build both the desktop and mobile application, what databases are used on the mobile and desktop to sync. How is the sync managed offline, and when online? I could really do with a getting stated for both mobile and desktop solution. Any help would be most appreciated.

  • > If I have mobile workers in the field who are collecting data the likelihood is the online access will fluctuate and they need to record data live as it happens, to then shared that data with a the mobile team and office based staff.

    sounds like online is a more convenient step. with that, what's the expected uptime? the interweb do fails, and it happens at the most inconvenient time. the mobile worker stepped out of carrier friendly range, server went down, hosting company doing maintenance, etc.

    now, I assume it's a corporate app. the employees will have to use it, everyone will have to live by using the app via mobile browser. that depends on the time requires to run the queries, I've heard from friends who work with .net stuff, they have queries that need to ran overnight. most browsers will timed out in 30-60 seconds range.

    having both web & native generally works for end user facing products (4square, twitter, tumblr, facebook, etc), so the product will have a permanent real-estate on as many screens as possible.

    if you really need offline syncing, and willing to step away from your comfort zone, that is Microsoft, checkout couchdb. afaik, couch has master-master sync out of the box with just minor setups. and it also has a module for Ti.

    Otherwise, imho, avoid syncing… just stop the app from working without internet connection.

  • Few ideas:

    • Forget about Windows mobile. Titanium does not have support for it (donno if will ever have)

    • I'm not sure this is possible, at least not o iOs.

    • If you use Titanium Desktop you will basically create a "self contained" webpage, with more capabilities (filesystem access, network access, zip, start a builtin webserver, etc).
      You can also use a "backend" in PHP,Ruby or Python that will do the hard job :)

    • The db system used by Titanium is SQLLite, on both desktop and mobile, so a sync between this would make sense to make use of, but you will have to build the actual sync algorithm.

    • You can even integrate nodejs in the desktop app if js and node are something u are confortable with.

    What you want to do is not easy, but is doable :)

    The best resources to read are:
    http://wiki.appcelerator.org/display/guides/Home

    And then on the github the kitchensink - actually here is where you really learn

    https://github.com/appcelerator/titanium_mobile/tree/master/demos

    There are other resources, blogs, github repos and gists that will help you, but you need to dig a little :)

    Hope this helps you a little.

  • the word sync gives me the chill. I assume your old system everything is handled by ActiveSync? tbh, syncing is a bitch. I'm still working on a decent algo with a simple conflict resolution to work with Dropbox.

    imho, depending on the budget, if you can afford to start from scratch, then go for it. or just make everything into webapp, force everything to be online, forget offline syncing altogether.

    you still can have native apps if you wish. Ti can help with desktops (Mac / PC), iOS, android, and soon, blackberry. it sounds a lot of work but trust me, it's mostly grunt work, which most likely you'll need to do anyway.
