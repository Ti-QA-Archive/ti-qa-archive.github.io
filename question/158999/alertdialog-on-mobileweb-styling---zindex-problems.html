alertDialog on Mobileweb styling - zIndex problems.

Is there any way to add styling values to an alertDialog on mobileweb? Or to force the device to use its built in alert box instead of the modal style web one?

I have the following alert dialog:

var dialog = Ti.UI.createAlertDialog({
   message: errors,
   ok: "Okay",
   title: "Sorry, the form contains errors",
}).show();

Which display the alert box fine but since I have other elements on the page which have a zIndex more than 0, those elements seem to appear above the alert box. Any solutions to this?

  • I ran into this exact same problem.

    What I had to do to get this to work was give negative zIndex values for all my views, so that zIndex of 1 was reserved for the alert box, zIndex of 0 was reserved for the top layer of views, and everything below that had a negative zIndex.

    It is a bit hackish but it works!
