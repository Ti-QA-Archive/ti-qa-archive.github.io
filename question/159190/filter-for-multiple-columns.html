filter for multiple columns

Hi I want filter on multiple columns at a time in android. my code is as follows

<Alloy>
    <Window onOpen="doOpen" id="win">
         <TableView dataCollection="contacts" dataTransform="transformFunction" id="tableview" searchAsChild="false" dataFilter="filterFunction" filterAttribute="filter" onClick="getDetails">
            <TableViewRow class="contacts_data" layout="vertical" filter="{firstName}{mobilePhone}" color="#11ceb5" selectedBackgroundColor="#F27935">
                <Label class="firstName" text="{firstName}" ></Label>
                <Label class="phNo" text="{mobilePhone}"></Label>
                <Label class="contactId" text="{id}"></Label>
            </TableViewRow>
        </TableView>
    </Window>
</Alloy>

In TableViewRow

when I use filter="{firstName}{mobilePhone}" then my app crashes.

when I use filter="{firstName},{mobilePhone}" search not working.

when I use filter="{firstName, mobilePhone}" then my app crashes.

when I use filter="{firstName mobilePhone}" then my app crashes.

I want to give search option on first name and mobilePhone both. Please help me.
Any help will be appreciate.

Is there any work around??

    Hi

    The SearchBar and TableView combination does not directly allow more than one field to be filtered.

    It is however possible with a custom property on the tableViewRow as you have tried. This is possible if you hard code the values in the XML and if you assign the values in the controller. So the problem appears to be related to the use of the collection and it's ability to combine fields (you are right in your attempts to combine the fields though).

    I would have hoped you first attempt would have been correct so it must be how alloy manages the row by row swap of data from a collection. I suspect it requires a single field per set of quotes.

    You could try creating two new properties; firstname and phone and then in the controller loop through the rows that have been created to combine this two into the row as the filter custom property.

    This is probably the easiest thing to do without starting fresh in the controller.
