  • Accepted Answer

    You should put all manifest customizations into the tiapp.xml. With current versions of the SDK, there are few if any times that you would need to use a custom AndroidManifest.xml file. Keep in mind that if you do use your own custom file, you must update it any time you make a change to activities. In fact, I think we've removed pretty much all instructions on how to use a custom file since the tiapp.xml method is strongly preferred.

    Instructions for adding custom values to the tiapp.xml are covered in many places in the docs, such as orientation handling, themes, intent filters, etc. No place I know of talks of a .gen file but that could be there someplace and is probably a mistake.

    • You're link goes to the 3.0-specific section of our docs. The current link would be http://docs.appcelerator.com/titanium/latest/#!/guide/Maintaining_a_Custom_AndroidManifest.xml

      I see the reference to the .gen version in that doc. After each build, there should be one of those files present in your build/android directory. If you use a custom manifest, you're going to need to keep an eye on that generated file to make sure that you copy any required auto-generated nodes (such as new activity nodes) from that generated file to your custom manifest file.

      Because of this extra maintenance overhead, which will lead to serious app errors if you don't do it faithfully and correctly, I recommend you do not use a separate, custom manifest file. Or if you do, create it as late in your development process as possible. You should be able to do everything with tiapp.xml customizations instead.
