OrientationChange Event Triggered When User Tilts iPad Device to Slight Angle

I have a very strange issue that a user just encountered and I witnessed. The user has an iPad case which folds in the back, creating a stand, allowing the iPad to rest at approximately a 30 degree angle in Landscape mode. This angle somehow triggers/retriggers the OrientationChange event between portrait and landscape mode continuously in my application.

I first noticed this with a background image. I have the following listener/handler:

function orientationChangeEvt(e) {
  if (Alloy.isTablet) {
    if (e.orientation == Ti.UI.LANDSCAPE_RIGHT || e.orientation == Ti.UI.LANDSCAPE_LEFT) {
      $.main.backgroundImage = '/images/Default-Landscape.png';
      $.logout.setLeft(680);
    }
  }
  else {
    $.main.backgroundImage = '/images/Default-Portrait.png';
    $.logout.setLeft(420);
  }
}

Ti.Gesture.addEventListener('orientationchange', orientationChangeEvt);

Typically, this code works as expected, but if you hold your iPad up, and slowly tilt the iPad down until it is in Landscape at a 30 degree angle, it triggers the orientation change event, moves my logout button to the Portrait position and changes to the portrait image (stretched though, since I am still actually in Landscape mode). For my user, since his case is at this angle, his logout button continually bounces between the Landscape and Portrait mode, and the background image is always stretched to the Portrait image.

Has anyone experience this? I am able to reproduce this behavior on my own iPad, and just unsure how to ensure the Landscape holds true for this type of angle.

    I think the orientation changes to FACE_UP and because your code is looking for landscape and switches to portrait for anything else it breaks.
