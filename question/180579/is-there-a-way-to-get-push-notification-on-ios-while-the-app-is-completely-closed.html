Is there a way to get push notification on iOS while the app is completely closed?

Is there a way to get push notification on iOS while the app is completely closed (has stopped, not even in the background)?

I'm sure there is a solution out there I've seen it working but can't find any lead on how to make it work. My notification callback is not getting called unless the user hit the notification. and it can be easily missed.

The new background services introduced in iOS 7 is not helping when the app is not in the background.

Any solution? Anyone?

  • If I understand what you're asking, you're asking whether you can build an app that receives the push notification callback as soon as the notification arrives on the device, correct?

    I don't think it works that way. The OS talks to the APNS server, gets any relevant push notifications, and displays them in the notifications view. Your app doesn't get activated until the user taps the icon in the system notifications view.

    Can you imagine how annoying it would be to the end user for various apps to be activated, possibly jumping to the foreground, every time a notification came in?
