  • I tried comparing a .project file from at project that had the clean option enabled with the .project file from one that didn't and apparently the difference was the content of the <buildSpec> tag, so I added the following to the <buildSpec> tag:

    <buildCommand>
    <name>com.appcelerator.titanium.core.builder</name>
    <arguments>
    </arguments>
</buildCommand>
<buildCommand>
    <name>com.aptana.ide.core.unifiedBuilder</name>
    <arguments>
    </arguments>
</buildCommand>

    And the clean option is availiable again, and seems to be working even though I'm not really sure what clean actually does.

  • I just had this problem using TS build: 3.1.4.201310290744

    Project Clean… was grayed out. All the appropriate check marks were made in the: Project > Properties > Project Natures Window.

    Solution was: Uncheck Mobile then click OK click YES at the "Nature Change" Popup. Then go back again and Recheck Mobile in Project > Properties > Project Natures Window. Click YES again at the "Nature Change" Popup. Cleaning functions again :)
