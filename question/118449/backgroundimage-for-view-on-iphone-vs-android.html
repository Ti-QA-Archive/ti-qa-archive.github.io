backgroundImage for view on IPhone vs Android

I am trying to build an app that works on iphone and android, but keep running into path issues on different types of UI components. I spend more time trying to work thru these issues than building the app itself!

I am creating a view in a file which is located in Resources/views/file.js.
All my images are stored in Resources/images/…

On Android:
This works:

var button = Ti.UI.createView({
    backgroundImage: "/images/button.png",
    top:10,
    left:10,
    width:48,
    height:48,
});

but the same doesn't work on iPhone. On iPhone it expects the backgroundImage to be path relative to the JS such as ../images/button.png

Is there a way for me to specify path for a backgroundImage for a view which doesn't require it to be relative to where the js file??
