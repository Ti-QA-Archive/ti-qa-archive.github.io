  • I have a similar problem where changing links are not always active.

    Option 1, Publish from xCode:
    Open the .xcodeproj in your build folder.
    Navigate to Thirdparty > AudioStreamer > AudioStreamerCUR.m
    find and edit or comment out instances of 

    [self presentAlertWithTitle:NSLocalizedStringFromTable(@"File Error", @"Errors", nil)          
message:NSLocalizedStringFromTable(@"Unable to configure network read stream.", @"Errors", nil)];

    Option 2, Write a scheduled check on the server side when new links are added and flag errors so that they are not served.

    In any case I listen for the 'change' event's 0 state and if it never reaches 3 I shut it down and get another, as so..

    var success = false;
audioPlayer.addEventListener('change', function(e) {
        switch(e.state){
            case 0: 
                if( (Ti.Platform.osname === 'iphone') &&  !success ){
                    // I nullify the player here and get another link
                }
            break;
            case 3: 
                if( Ti.Platform.osname === 'iphone' ){
                    success = true;
                }
            break;
        }
});
