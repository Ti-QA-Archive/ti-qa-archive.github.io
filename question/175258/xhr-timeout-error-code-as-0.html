XHR timeout error code as 0

Hi,

I have an xhr network call in my app. where I want to check for timeout. But I am getting the status as "0" instead of "408" in both iOS and android.
Please any one help me in this issue. It is happening in Titanium 3.2.2 SDK in iOS7 and Android 4.0+

  • There are several types of timeout, and a HTTP 408 response is a server timeout, which means that the web server running the request has decided that the request has taken too long and so returns that status.

    A status of 0 with a timeout means that the HTTP client object has run out of time trying to connect to the server, so you wont get a server response as it hasn't been reached yet.

    Hope this helps.
