How to prevent web view that handles big files from crashing

Hello,

The case below is about only when the web view is alive and on the screen (not after it closes and needs to be released from memory).

I'm trying to let the user load and preview files (images, docs & pdfs) through a webview. There are problems with iPad 1 when the files are too big and make the app crash. How could I prevent this? The only way I figured out till now it to check the size property of the Filesystem.File with which I feed the web view. Isn't there any other better way? Something like monitoring the web view and when I get a memory warning to make it stop loading. Because even big files start displaying ok but as the user scrolls the web view on the device, it crashes a bit later. For the current project it is sufficient if the user will scroll for a couple of pages. So, I could live with making the web view stop loading after a while. But I cannot figure a way to detect how much content has the web view loaded.

var swapFile = Ti.Filesystem.getFile(Ti.Filesystem.getApplicationDataDirectory(), "filePreview." + $extension);

var webView = Ti.UI.createWebView({
    data: swapFile
});

PS: I cannot use the document viewer component (which is memory-wise more reliable) for this app cause I cannot customize it the way I want (it needs to be a small view, not full screen)

iOS 5.1 on iPad. Ti SDK 2.1.2

  • There is no way to monitor memory usage on a webview. How big are the previews that you are loading? Even on a first gen ipad the files would have to be fairly large to cause a crash. Does this happen with just 1 file loaded, or does it happen after you load a second or third file?
