  • Ok, I think I have resolved that issue by myself. Somehow the installation path got mangled and Ti3.1 could not find the new locations of XCode.

    Now, when setting the path with xcode-select --switch &lt;path&gt; is is ultimately important to no set the path down into the Xcode.app but only to the folder itself.

    Do NOT set 

    /Applications/Xcode.app/Contents/Developer/Platforms/iPhoneOS.platform/Developer/SDKs

    but DO set

    /Applications/Xcode.app

    Now if you call xcode-select --print-path it shoudl correctly say

    /Applications/Xcode.app/Contents/Developer

    instead of 

    /Applications/Xcode.app/Contents/Developer/Platforms/iPhoneOS.platform/Developer/SDKs

    Unfortunately Ti3.1 hides almost all of the error message inside the Preferences>Studio>Platform panel in one single line at the top so that you don't see that the full path actually contains double elements int the path.

    Also there were some problems with rights. I had to install Ti from scratch which then asks for permission and then I could switch back to my old installation which still complaisn about some permissions denied while the newly installed doesn't …

    Phew … any advices anyone in addition to this?
