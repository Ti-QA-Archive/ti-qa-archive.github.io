I'm almost ready to deploy my first app but sometimes I get 'Unfortunately, your application stopped working'

Hi, I wrote a little app that sometimes crashes and I'm not able to understand the reason. It is an Android app developed using Titanium Studio and the last SDK on a Windows 7 64 bit machine. It is a very simple app where there are few imageViews (12): when a user touches an image a very short sound is reproduced.The problem is that sometimes app crashes: when it happens it could be happen after few seconds or after a lot of minutes. The strange is that sometimes it crashed during startup. I gave the APK for testing to few friends of mine and two of them got the problem. Usually, as I observed, the problem could happen when an imageView is touched so it is probably that the problem is related to the way click event is managed (but this doesn't explain the crashes on startup). I use a AudioPlayer object:

var effetto = Ti.Media.createAudioPlayer({});

The function called by clicking an imageView is this:

function effettoSonoro(obj){
    effetto.release();
    effetto.url = obj.suono;
    effetto.start();
};

Thanks for any help.

  • so it is probably that the problem is related to the way click event is managed

    You don't have enough data to make this assumption. It may be correct, but simply an uneducated guess.

    The strange is that sometimes it crashed during startup.

    So, you either have 2 problems (the one you assume is related to touch events, and another one) or you have 1 problem that you have misdiagnosed.

    the last SDK

    Means nothing. Versions have numerical values, we expect numerical values when we ask for a version.

    Tie your devices to the computer with a USB cable. Load up monitor/ddmsand see what the actual errors are. Anything else is simply a guess and a waste of my time and yours.
