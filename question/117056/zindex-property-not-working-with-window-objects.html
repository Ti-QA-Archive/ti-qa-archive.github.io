zIndex property not working with Window objects?

Hi,

I've been working on an app that will have a large number of content screens, and so am looking for a way to manage them efficiently.

I'd like to have each content screen in a separate file, loaded in to a Window via the url property. Then, display each as needed, removing the ones that are no longer in view.

However, I'm noticing a couple things, and would really appreciate some help confirming that I'm not missing anything.

First, it seems to me that the zIndex property is ignored when used with a window. I've tried setting the property in a number of ways (including using animate() to set it as recommended by another forum user), but the only thing that holds true is that the most recently opened window is on top.

Second, it seems that nesting windows doesn't work as I'd expect either. For example, if I open a window with a left:300 property, then open a window from that window with a left:0 property, I would expect it to still be 300 left as per it's parent, but… it shows up at 0, 300 pixels left of it's parent. This is not how a View would behave.

So, long note I know. But if anyone could confirm that my assumptions are correct, or (hopefully!) prove me wrong with a way to use zIndex or nested windows.

Here's my example code: https://gist.github.com/864415

Thanks a ton everyone.

    • Hi Roman, you just need to open your windows in the order of how you want them arranged.

      var win1 = ...;
var win2 = ...;
var win3 = ...;
win1.open();
win2.open();
win3.open();

      The above would result in like win1 in the way back, win2 in the middle, and win3 would be on top.
