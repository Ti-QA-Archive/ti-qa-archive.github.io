Urban Airship for Android - not working when device is restarted

Hey guys,

we have an issue with the UA module for Android that we are trying to resolve already for some time.

We are using Ti+ Urban Airship push module. It is working normally when the application is running - all pushes are received normally.
However, when the device is restarted no notifications are received. The application has to be manually restarted in order to be able to receive notifications again.

I suspect that the application is not able to restart properly (it is not listed under running applications).

The devices we used for testing are:
HTC Sensation 2.3,
HTC Desire 2.3.3 and
Droid 2.2.1.
All showing the same behaviour.
We are using Titanium Mobile SDK 1.7.2.

Anyone having similar issues?

I saw a ticket regarding this problem: https://support.appcelerator.com/tickets/APP-799638.
It is however not resolved and Titanium claims that they have successfully tested this scenario with no issues.

Would appreciate any feedback.

Thanks!

Lech
