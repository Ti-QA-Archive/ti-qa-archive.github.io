ACS Won't log user in after upgrading to iOS6

Hello.

Everytime my App starts it checks the current user (Cloud.Users.showMe), and if the users has logged in once, it automatically log the user in again when starting the app untill the user Log out by pressing the Log out button.

Here is the problem:
Ever since I upgraded to iOS6, the user won't get log in automatically after starting the app. It worked fine with iOS5. Anyone have a solution for this problem?

Thanks.

  • Hi Nicolai,

    This is because of the security enhancement for ti.cloud 2.3.0: http://docs.appcelerator.com/titanium/release-notes/?version=2.1.2.GA#cloud

    In ti.cloud 2.2.0, the module would persist the sessionID data and would automatically use
    that sessionID to make ACS calls across different application sessions i.e. starting the app
    again.

    Now in ti.cloud 2.3.0, the module does not persist the sessionID (the security enhancement).
    But, the sessionId property can be used by the developer if they wish to persist
    it in a secure location and restore on next app startup. Here is the api doc: http://docs.appcelerator.com/titanium/2.1/index.html#!/api/Titanium.Cloud-property-sessionId

    I hope this helps answer your question.
