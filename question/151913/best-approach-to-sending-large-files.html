Best approach to sending large files

Hi,
I'm building an app having a feature to shoot and send 6-10 images from client app to server. What strategy should I use? Imagine the scenario: one image is around 3MB, 10 images around 30MB. The app uses 3G, sometimes it's really slow. If a connection is lost, the app should continue sending it later. I was wondering if Titanium has some already prepared tactics for doing this. Should I encode images to base64 and send them one by one? Or use some services, such as Dropbox or Google Drive?

  • Hello,

    if you are sending 30 MB, it will take forever, no matter what. You can use zip module in order to try to minimize the transfer, or wait until wifi is available, or simply try to reduce the size of the files (if you are using an image, load it in a view and then save the view image, it will be a lot smaller).

    If you absolutely need the high quality files, then be patient (even if you use box, drive or dropbox, it will take time to sync).
