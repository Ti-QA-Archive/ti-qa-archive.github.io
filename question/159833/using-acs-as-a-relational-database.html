Using ACS as a relational Database

Hi everybody,
not considering any performance issue, do you think that ACS with custom
objects and custom fields can be used as a normal
relational database, or do you think, in your experience, it would be a misuse of it and
a big pain?

For example, from what I see, It looks like every object must be owned by an user but sometimes
this doesn't make sense and would require the usage of fake users.

For example, think of a restaurant reviewing app. A custom object Restaurant could exist, but
it wouldn't make any sense to say that it is owned by any user, unless we introduce a fake
user for that purpose.

I'm asking this because I'd like to use ACS for my application, since it is easy to use and wel integrated, but I wouldn't like to use it for something it has not been thought for.

Thanks!
Daniele

    Well, using ACS as you would a normal relational database would be a very bad idea. This is always the case when you are trying to use something that is created for a purpose and try to force it into what you want it to do.
    Now that being said, there is no reason what you could store your restaurant review information into ACS and have links between some of your objects. You are also perfectly correct when you say that you will need to use a technical account if you want your information to be shared across all your users. This makes sense for ACS since it was created as a cloud service for users and not as a central server database. But there are a lot of applications out there that use a single user for their entire application and it works like a charm.

    Hope this clears this up for you.
