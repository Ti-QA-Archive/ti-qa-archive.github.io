    • In answer to your earlier question about globals;

      A suitable method these days is in the use of CommonJS.

      With this you can create your own modules of re-usable code and depending on implementation they can serve very well as global equivalents.

      Quick example;

      test.js

      function create(o) {
    var obj = o || {};
    var tbl = Ti.UI.createTableView({
        data: obj.data || [],
        height: Ti.UI.FILL,
        width: Ti.UI.FILL
    });
    return tbl;
}
exports.create = create;

      Usage;

      var data = [
    { title: 'One' },
    { title: 'Two' },
    { title: 'Three' }
];
var table = require('/test').create(data);
win.add(tbl);
