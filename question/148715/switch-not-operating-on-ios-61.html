Switch not operating on iOS 6.1

I created a switch in my app using the following syntax : 

var RemindersSwitch = Ti.UI.createSwitch({
    value:false
});

unfortunately it renders Switch which is off, and it is not possible to change its state to on. I would have thought that switch element would allow to change itself, and then I would be able to catch the "change" using event listener, but at the moment this listener has nothing to catch, since the UI element is not responding. Is there something I am missing, or maybe I am using something that is not supported?

I am currently using MAC OS X 10.8.2
Titanium Studio, build: 3.0.2.201302191606
with
SDK 2.1.3 GA

and iOS 6.1

2 Comments

  • I solved this problem myself. It was not an issue with iOS 6, but with a view blocking the touch controls. (overlapping views). I solved the issue by adding "zIndex: 1000000," to the Switch declaration
