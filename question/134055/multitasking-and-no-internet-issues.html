multitasking and no internet issues

Hi

I have pretty much completed this IOS app, however I have found one major issue.

If the app is closed and then reopened without internet access before the app has timed out I get errors. The app checks for internet on window open, however as the app was just in the background the window was never closed.

Is there a way to detect when an internet connection fails in real time so I can throw an event that will stop trying to access external resources.

I am currently using on window open:

Titanium.Network.networkType === Titanium.Network.NETWORK_NONE

This app must allow background as it plays audio so turning off multitasking is not an option.

Is there a way to fire an event when the app is brought back from the background.

Any help would be appreciated.

    this is from the kitchen sink app in the app.js file

    Ti.App.addEventListener('resume',function(e){
        Ti.API.info("app is resuming from the background");
    });
