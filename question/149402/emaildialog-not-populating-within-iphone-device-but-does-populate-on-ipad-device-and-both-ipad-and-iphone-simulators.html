emailDialog not populating within iPhone device, but does populate on iPad device, and both iPad and iPhone simulators

  • hi,

    first thing is if your mail account is not configured then may be it causes the problem so you can check first if mail is configured in your device by emailDialog.isSupported().

    var emailDialog = Titanium.UI.createEmailDialog();

    emailDialog.setToRecipients([emailaddress]);
    emailDialog.setSubject('ROS - Patient ID: ' + patientid + ' - Created on ' + completetimestamp);
    emailDialog.setHtml(true);
    emailDialog.setMessageBody(emailbody);

   if(emailDialog.isSupported()){

     emailDialog.open();
    }else{
    alert('Email is not configured.');
    }

    Ti.API.info('emaildialog should be open now');

    second thing is if you are opening your window with modal property true then i does not open because emailDialog is one kind of window which opens with modal so you can not open emailDialog within the window which is opened with modal property.

    but i think second suggestion is not for problem because as you said that this is working in iPad and iPhone simulators. so go for first suggestion
