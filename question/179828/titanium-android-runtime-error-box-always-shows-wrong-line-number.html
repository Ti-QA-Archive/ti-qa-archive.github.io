i am developing an cross platform application in OSX mavericks environment with titanium sdk 3.3.
While debugging on android the runtime error caught always show [1,69] for line number.So i can't spot exact position of actual error.
The same environment run smoothly on IOS application.

  • Hello Jithin,

    Thanks for your inquiry, which project environment are you using, Alloy or classic?

    Alloy takes your code, builds a classic and you need to go look at it in the Resources/{platform}/alloy/controllers folder.

    If it's a classic, if you are deep in a callback, line numbers may not match. If the code has been minified then it might have the wrong line numbers.

    Run the project in debug mode.
