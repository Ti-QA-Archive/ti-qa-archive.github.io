I'm using Samsung Galaxy Tab (as I wrote) and 1.7.1 final ([INFO] Titanium SDK version: 1.7.1 (06/21/11 14:28 1293a6d)). I don't use emulator.

Also, I don't agree that it's logical. I think is other way around, click should generate "click" event by the definition of DOM events no matter is text field focused or not:

http://www.w3.org/TR/DOM-Level-2-Events/events.html

click
The click event occurs when the pointing device button is clicked over an element. A click is defined as a mousedown and mouseup over the same screen location. The sequence of these events is:

    mousedown
    mouseup
    click

I haven't found any info regarding click event and text field focus, so I think that click is not affected with the issue is text field focused or not.
You can test this in the browser with following code (please, don't use IE :) ):

<!DOCTYPE html>
<html>
<head>
Testing


window.onload = function() {

    var tf = document.getElementById('tf');

    tf.blur();

    tf.addEventListener('click', function() {
        alert('Clicked!');
    }, false);

}


</head>
<body>
    <input type="text" id="tf" value="Click me!">
</body>
</html>

Of course, mobile and tablet devices don't have mouse, but same logic should apply, IMHO.
I agree that core team should decide what will be implemented in Titanium. Titanium doesn't have to follow the same rule, but I think that click event behavior should be documented otherwise and especially should point out the difference.
