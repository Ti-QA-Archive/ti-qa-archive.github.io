  • Accepted Answer

    Ideally the request logic to send the email needs to be on the Server side; Otherwise the App needs to be running when the date to send is set in this case, check everytime the App is launched, and check if the currentDate is after the date set, then send the email.

    Another way, is to develop that feature on the Server side with Node.ACS, which is probably the correct way of implementing this. This would send the email on time regardless if the User's App is running or not.

    For playing with dates & times, I highly recommend using moment.js

    As you might have already known, the docs for email are here. As for wanting to sent an automated email after a specific interval, there are numerous ways to accomplish this. One way could be to save the date that you want to send the email to a specific user (possibly when that user signed up) and do a check to see if the date is a year past the sign up date.
