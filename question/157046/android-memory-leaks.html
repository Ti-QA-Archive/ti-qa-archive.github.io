  • Before we jump to any conclusion regarding memory. It is important to identify how much memory your application is actually using.

    ###Here are some thing to consider:

    • How many rows are you displaying at any given time?
    • Have you used the className property for each row in order to gain performance?
    • How many controls (Labels, Images, etc.) are contained in each row?
    • Are you loading the whole set of rows in a single call or are you loading as the user scrolls the table.

    If all these factors are addressed and you are still experiencing slowdowns, you may want to have a look at the new ListView component. Which is way faster than the regular TableView.
