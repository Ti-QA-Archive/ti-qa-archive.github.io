How to get the number of files in a remote directory (without server-side scripts)

Hi!

I'm wondering if anyone knows whether it's possible to retrieve and store
all the URLs of the files in a remote directory (i.e. not on device) in an array, or otherwise how to get the number of files in that directory. The thing is that I have a gallery in which i use imageViews with remote URLs (saved in a folder on my server as 0.jpg, 1.jpg etc),
but I need to get the number of images (which of course may vary over time)
of the directory to be able to set the condition of the for loop (e.g. "i<numberOfFiles.length").

I know how to solve this by using either json and/or a php script, but I won't be hosting the image content on my own server when the app is done, so a non-script solution would be preferable!

Thanks for your help guys!

Application type: mobile
Titanium SDK: 3.0.0
Platform & version: iOS x.x
Device: iOS simulator,
Host Operating System: OSX x.x

  • Hi

    The short answer is no.

    The only way to get information about a remote file system is to have code on the remote system you can query.

    There is another way if you really cannot run code on the server. It is not the best solution and it is inelegant but it will work. This does depend on the files being either consecutive or at least being predictable.

    Using hhtpClient you can try to download a file and determine if you get a 404 status code. By using some simple routine you can determine when the files stop. Ideally you would keep the last number in memory and then the next time skip a few ahead and check again, if not 404 skip again until it is.

    As I say - this is not recommended and should be a last resort only.
