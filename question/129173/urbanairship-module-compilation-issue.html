UrbanAirship module compilation issue

I've been trying to get the iOS UrbanAirship module working and have encountered a strange issue.

I have installed the ti.urbanairship module into the "/Library/Application Support/Titanium/modules/iphone" folder and during the compilation process I am seeing the "Detected module for iphone: ti.urbanairship 1.2 @" message so I know its being found.

I have also added the "<module version='1.2' platform='iphone'>ti.urbanairship</module>" definition to the tiapp.xml file.

However, when I build the app on iOS I receive the following error message:

[ERROR] Error: Traceback (most recent call last):
  File "/Library/Application Support/Titanium/mobilesdk/osx/1.7.5/iphone/builder.py", line 1222, in main
    execute_xcode("iphonesimulator%s" % link_version,["GCC_PREPROCESSOR_DEFINITIONS=__LOG__ID__=%s DEPLOYTYPE=development TI_DEVELOPMENT=1 DEBUG=1 TI_VERSION=%s %s" % (log_id,sdk_version,debugstr)],False)
  File "/Library/Application Support/Titanium/mobilesdk/osx/1.7.5/iphone/builder.py", line 1140, in execute_xcode
    output = run.run(args,False,False,o)
  File "/Library/Application Support/Titanium/mobilesdk/osx/1.7.5/iphone/run.py", line 39, in run
    sys.exit(rc)
SystemExit: 65

If I then remove the "<module>" tag from the tiapp.xml file and build again (without doing a Project->Clean) then the app builds and runs successfully and the module is found.

If I clean the project and try and build again, then it builds successfully, but the module is not found when the application loads.

What's going on here? It sounds like there's something wrong with the build process and the module files are not being copied into the right location, hence the failure after cleaning the project.
