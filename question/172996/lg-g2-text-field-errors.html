LG G2 Text Field errors

I have recently updated my LG G2 to the latest version of KitKat and am noticing issues with textfields. Has anyone else noticed that the text typed in text fields is appearing above the actual text field and cut off? Here is a snippet of the code I am using if that helps, but it seems to be completely systematic.

var signupFirst = Titanium.UI.createTextField({  
    color:'#336699',  
    top:133,    
    width:135.5,
    left:22,  
    height:40,  
    hintText:'First Name',
    autocorrect: false,
    autocapitalization: Titanium.UI.TEXT_AUTOCAPITALIZATION_NONE,  
    keyboardType:Titanium.UI.KEYBOARD_EMAIL,  
    returnKeyType:Titanium.UI.RETURNKEY_NEXT,  
    borderStyle:Titanium.UI.INPUT_BORDERSTYLE_ROUNDED  
});
if (Ti.Platform.osname == 'android'){
    signupFirst.backgroundColor = "white";
}

  • I have found what I believe to be a satisfactory solution to this problem without having to go back and completely rewrite my app. It requires 2 changes both of which are in tiapp.xml. The first is what several have stated and making sure the anyDensity value is set to be true. The second is to find the ti.ui.defaultunit property and set that value to be dp. This will keep all of the correct formatting in your app while doing away with the field issues that have cropped up.

  • Setting the text field's sizes in 'dp' and settind height to Ti.UI.SIZE worked for me.

    var signupFirst = Titanium.UI.createTextField({  
    color:'#336699',  
    top:'133dp', // FIX    
    width:'135.5dp', // FIX
    left:'22dp', // FIX  
    height:Ti.UI.SIZE, // <-- FIX TO FIT TEXT TO FIELD  
    hintText:'First Name',
    autocorrect: false,
    autocapitalization: Titanium.UI.TEXT_AUTOCAPITALIZATION_NONE,  
    keyboardType:Titanium.UI.KEYBOARD_EMAIL,  
    returnKeyType:Titanium.UI.RETURNKEY_NEXT,  
    borderStyle:Titanium.UI.INPUT_BORDERSTYLE_ROUNDED  
});

    This makes the text fields a little bit bigger in height, and the text fits in the text field.

    I think, LG's Kit Kat theme has bigger text padding values than others..
