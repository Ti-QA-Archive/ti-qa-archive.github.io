userLocation marker (blue dot) disappears BUG STILL EXIST !?? [Android]

Hi everyone,

Does anybody have a solution for the userLocation marker (blue dot) disappearing on Android when a mapView window loses focus (i.e. switching around tabs and phone autolock)?

Here's my code.


var win = Titanium.UI.currentWindow;

// This is my current region longitude and latitude
var lat = 5.32235;
var lon = 100.234223;

var mapview = Titanium.Map.createView({
    mapType: Titanium.Map.STANDARD_TYPE,
    animate:true,
    region: {latitude:lat, longitude:lon, latitudeDelta:0.150, longitudeDelta:0.150},
    regionFit:true,
    userLocation:true
});

win.add(mapview);

Blue dot does appear on the first run, but disappear after mapView window loses focus…

Tested it on both emulator and an android 2.2 phone.
Titanium SDK 1.7.1, 1.7.2, 1.7.3, 1.7.5 and 1.8.0 >>> ALL NOT WORKING
Android Google api 2.2

P.S I've found a raise ticket for this issue in JIRA here : TC-105
They said this is fixed in 1.7.2, but THE BUG IS STILL THERE…

Please help.
Regards,
Neti.

  • I haven't checked in the last Ti SDK build, but I had to get a workaround for my android projects, which is very annoying.
    The thing I did to solve this:

    I add an annotation to the map that has the same visual style of the blue marker and I add an eventListener('location') that updates the annotation's latitude and longitude.
    Keep in mind that removeAllAnnotations and AddAnnotations didn't work as expected in Android, just in case you want to add some annotations to the map, apart from the "blue marker"

    Jose
