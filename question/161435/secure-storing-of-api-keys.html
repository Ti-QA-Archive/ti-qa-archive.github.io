Secure storing of API keys

Hi all.

Hoping to tap your collective brains & experiences for the following questions…

Question 1:

Is it considered secure to store API keys within the JS of a Ti App?

From what I've researched, Ti will obfuscate the JS code when compiling, however the code & keys can conceivably be reversed engineered. Even when using ACS, API keys are added to the tiapp.xml, which are even more clearly visible in an compiled app.

Question 2:

So, what's the best way to secure API keys within a Titanium app?

Question 3:

If the only solution is a custom encryption module or a custom implementation of something like AES, won't that cause problems when submitting the app to Apple and Google given their encryption policies?

Thanks in advance to everyone who lend some advice!

Titanium SDK: 3.2.0 GA
Platform & version: iOS 7, Android

    From what I&#39;ve researched, Ti will obfuscate the JS code when compiling

    Your search is resulting in OLD information.

    Your js files are more than obfuscated, they now are combined and encrypted. These are stored within the compiled native binary and decrypted at runtime. I cant find the reference, but 128b AES I believe.

    Understand that anything you write to a property is available from the .plist file created on the device.

    Understand that nearly anything written in bytecode (android) can be r-engineered.
