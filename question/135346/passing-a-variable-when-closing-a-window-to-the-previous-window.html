Passing a variable when closing a window to the previous window

Hi everyone, i think i have a small problem. I'm sure isn't a big issue but i haven't find a nice workaround for that.

I have tabgroup based navigation layout (one single tab with tabbar hidden). So basically you can open many level windows and you can return to the previous one.

My question is that isn't a nice way to come back to the previous window with a variable when closing current window?

I have a textfield on a window, onclick opens a new window, select a value from there, close that window and come back to that value in the previous window to complete in the textbok. It's simple.

Isn't a more simple method then add a focus event to the parent window and get a persistent data that was set on the new window?

I mean it's something simple, a windows opens another one, when closing it come back to the previous window with some data.

Thanks a lot.

    One way or the other, you need to notify the existing window that something in the world has changed and it needs to take action.

    Even if there was a magic callback on closing a window that passes variables back to an existing object, you will still need to demand action of some sort from the object.

    Enter events. You mentioned the focus event on the existing window and that is one option.

    The other is to create an app level event that calls a function to update your existing window.

    Either way your persistant data (stored in a App.Property or in a global variable) has to be brought to the attention of the other objects somehow.

    • The problem with a focus event on the window, is that the focus event would be fired when the window is initially created and interacted with or on a tab change.

      You should add a close event to your sub window that calls the function to update your first window.
