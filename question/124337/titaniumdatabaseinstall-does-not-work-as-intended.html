Titanium.Database.install does not work as intended...

After fighting with my project for most of the day because I changed a data value in my database and just couldn't get Titanium Studio to pick it up when rebuilding my project, I have concluded that Titanium.Database.install does NOT work as per the documentation:

The documentation says "install a database from the application Resources folder (at build time) and return a reference to the opened database".

It doesn't. If you have run your project before then a copy of the then-current database will be cached in either the iPhone simulator or android emulator. You'll need to reset this cache (iPhone simulator "Reset Contents an Settings", android emulator just delete the virtual devices you've set up and start again), delete your build folders in your Titanium workspace, probably also restart Titanium Studio and then run again.

You should then pick up the latest copy of your database from the Resources folder.

Apart from suggesting that the powers that be reword the documentation AND remove the fluffiness about the difference between Titanium.Database.install and Titanium.Database.open, I hope that this post will be of use to anybody else who's found themselves pulling their hair out cleaning their projects and deleting to no avail.

This seems like a huge fail in the design of Titanium Studio and the API: If it's not actually possible for install to work as documented then just give us a simple open statement that does and include notes on how to empty cache.
