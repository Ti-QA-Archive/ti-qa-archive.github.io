App runs in simulator and devices, but hangs at splash screen in Xcode Instruments Profiler

My app hangs at that splash screen when I try to check for leaks in Xcode Instruments Profiler, but otherwise works fine in device and simulator. Any ideas what is going on here? I am not sure where to even start.

  • Hello Michael,

    there is a fix/workaround from Vladimir at TIMOB-15215. Quote the comment:

    
I just found when it happens, maybe that is your case too:
App freezes if it was previously compiled in Debug Mode (i.e. Cmd + F11). Re-compile it in Normal Mode before run in Instruments (i.e. Cmd + Shift + F11) – now working for me and I'm very happy.

Maybe it not happened with me before because I usually used Normal Mode to run application, but after updating to Xcode 5 Titanium Studio doesn't show console logs properly anymore – very very annoying. It shows me logs only in Debug Mode (btw, some parts of logs are missing there anyway, also filtering by log type is not working anymore). And I started using Debug Mode to run application since that time and then realized what Instruments are not working and only now I found why (see above). I think it hangs because waiting for Ti Studio debugger to connect, so need to compile in Normal Mode to prevent that.

    Let us know if that workaround works for you.

    Best,

    Mauro
