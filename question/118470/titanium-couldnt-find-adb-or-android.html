Titanium couldn't find adb or android

Hello World

I've installed the last version of Titanium 1.2.2 and Android SDK.

And actually, Titanium tells me
"Couldn't find adb or android in your SDK's "tools" directory. You may need to install a newer version of the SDK tools."

However I can see android in the tools directory and I also can launch adb or android via the terminal. So how is it that Titanium couldn't find it ?

I've search throught several forums and found that I had to make a symbolic link from platform-tools/adb in tools, I have to put the path to the platform-tools directory before the path to the tools directory in my .bash_profile which give me this actually :
"export PATH=${PATH}:/Users/Name/Library/android-sdk-mac_x86/platform-tools:/Users/Name/Library/android-sdk-mac_x86/tools"

So how can I make this work ? Do I have to uninstall newer versions of Android SDK ? Do I have to change the path ? Do I have to rename my Android SDK folder ?

Thanks in advance for any help :)

    • Karine

      Make sure you have followed the instructions at Preparing for Android development regarding adb.

      Then, close Titanium Developer and the emulator, and remove the following obsolete packages:

      • SDK Platform Android 2.3.1, API 9…
      • Google APIs by Google Inc, Android API 9…
      • SDK Platform Android 1.5, API 3…
      • Google APIs by Google Inc, Android API 3…

      Important: afterwards, delete all existing AVDs.

      Keep in mind which Android SDK versions are supported by which Titanium SDK versions, as per Android SDK / Target Android Platform.

      Use this tool to configure the ANDROID_SDK environment variable with the path to the Android SDK directory (the directory that contains the tools directory).

      Then please post again all the information I request before.

      Cheers
