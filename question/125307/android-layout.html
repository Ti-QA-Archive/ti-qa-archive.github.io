Android layout

I have a label, which may have the text of different length. So, it can occupy many lines and it means that its height may vary. I add it to a view and want to place another control below this label. In iPhone it works fine, if I set the label's height to 'auto', add it to a view and then check its height. But as I know, on android all sizes are resolved when the window is rendered. Is there a way to place controls correctly not in 'open' event? It looks ugly when controls are added at runtime. And one more question: how can I get the height of control if its size is measured in 'dp'? I'm parsing this string to int now, but I think that it should be a better way.

    something like this:

    var wrapper = Ti.UI.createView({
    layout:'vertical',
    height:'auto'
})

var lbl = Ti.UI.createLabel({
    height:'auto',
    text:'variable length text'
});

var btn = Ti.UI.createButton({
    height:30,
    title:'the button'
});

wrapper.add(lbl);
wrapper.add(btn);
