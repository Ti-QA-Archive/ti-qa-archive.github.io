    I can reproduce this and looking through the code in TiUITableViewCell.h where the code is crashing:

    -(void)keyboardDidShowAtHeight:(CGFloat)keyboardTop
{
    int lastSectionIndex = [(TiUITableViewProxy *)[self proxy] sectionCount]-1;
    ENSURE_CONSISTENCY(lastSectionIndex>=0);
    CGRect minimumContentRect = [tableview rectForSection:lastSectionIndex];
    InsetScrollViewForKeyboard(tableview,keyboardTop,minimumContentRect.size.height + minimumContentRect.origin.y);
}

    it seems that the lastSectionIndex is being set to -1, the ENSURE_CONSISTENCY is throwing the error, but the code is continuing to get the rectForSection with a value of -1 which is obviously wrong.

    Looks like you need to raise a JIRA ticket to get this one fixed.

    For now, I suggest you hide the field when the table row count is zero so that it cant get focus.
