Forcing refresh on window after clicking tab

Howdy y'all; got a question that's stumping me.

I'm building an iPad app that's heavily reliant on external data. I've essentially built an API on an external server to which this app interfaces and pulls data in using a series of XHR calls.

The starting 'page', or 'Dashboard' (the first tab) is a list of items that is pulled in via XHR. As the user moves through the app, they would have the opportunity to add an item (using a form which submits data to the server). After submitting the form, I present the user with a couple of routing options, one of which is a choice to return to the 'Dashboard'. If the user clicks this button, everything works fine… as in the API is called and the list is refreshed to show the new entry. However, if the user clicks the first tab ('Dashboard'), the XHR call is not fired. I do not have the request in any sort of event listener, though it's essentially the first thing in the file (list.js). Is there anything I can do to force a 'refresh' of the dashboard when a user clicks a tab? Or, to put it another way, treat a tab click the same as when navigating to the same window via a button click.

(If it matters, I'm running 1.7.2)

Thanks,
Sam
