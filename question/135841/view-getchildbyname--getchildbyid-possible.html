View getChildByName / getChildByID possible?

Coming from a Flash background I'm starting to miss the functionality to get a views child directly by calling getChildByID or getChildByName.

The only way that i can see this being possible at the moment is by getting the views children and looping through them, which feels a bit dirty.

Are there any easy ways or tips and tricks that people use?

    hi Martyn,

    you can try like this

    
var win = Ti.UI.createWindow({
    backgroundColor : 'red',
});

var btn = Ti.UI.createButton({
    systemButton : Ti.UI.iPhone.SystemButton.REFRESH
});

win.button = btn;

win.rightNavButton = btn;
var imgView = Ti.UI.createImageView({
    backgroundColor : 'white',
    top : 0,
    width : 320,
    height : 30
});
win.img1 = imgView;
win.add(imgView);

var view = Ti.UI.createView({
    height : 175,
    width : 320,
    backgroundColor : 'red'
});

scrollView = Titanium.UI.createScrollableView({
    views : [view],
    showPagingControl : false,
    currentPage : 0,
    top : 30,
    height : 145,
    width : 320,
    backgroundColor : 'black'
});

win.scroll = scrollView;

win.add(scrollView);
var imgView1 = Ti.UI.createImageView({
    backgroundColor : 'white',
    top : scrollView.top + scrollView.height,
    width : 320,
    height : 30
});

win.add(imgView1);
win.img2 = imgView1;

win.open({
    modal : true
});
alert(win['img2']);

  • Hello Martyn.

    IMO, depending directly on "children" array is not the best practice. First thing, it's not reliable because "children" array in some SDK versions (on Android) was gone (I'm not sure what's the current status). And the other thing is that relying on component's position in parent container component can make maintenance harder (change position and you'll have to change your code).

    I solved this by using my own wrappers around Titanium components. In the background, I use JS Object object to register added components by string name. I think that it is easier to remember component's name, than component's index.

    New version of that wrappers should be open-sourced some time in the future. You can see first glace of them in this gist: https://gist.github.com/2028016
    If you're interested, stay around, or if you want to contribute you can contact me. ;)

    Cheers, Ivan.
