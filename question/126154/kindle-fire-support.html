  • OK, I have successfully run titanium mobile apps on the Kindle Fire. You need to configure the android emulator with this info:

    Width: 600px
Height: 1024px (the device will reserve 20px of the height to display a soft key menu, yielding a height of 1004px when in full-screen mode)
Abstracted LCD Density: 169
Target: Android 2.3.3 - API Level 10  (NOT Google APIs)
RAM: 512 MB

hw.gps: no
hw.camera: no

    instructions for doing this inside Titanium studio can be found in the Appcelerator wiki: http://wiki.appcelerator.org/display/guides/How+to+resize+the+Android+Emulator

    You also need to set up the adb ports for the Kindle Fire. Under Mac OS X:

    1. Open the ~/.android/adb_usb.ini file for editing
2. Add 0x1949
3. Add 0x0006
4. Save the file

    Using Appcelerator Studio, you can now "Run on device".

    I have found that Studio seems to crash often and I do not always get a "good" load of the app to the Fire. You should quit any open android emulator, quit Studio and kill any "left-over" android emulator processes. A clean build and deploy then seems to work OK.

    LMK if these instructions are clear enough – and have fun developing for the Fire!

    Andy

    • Hi Brian,

      Make sure you have the Andoid SDK installed and that you have run Appcelerator Studio and the Android emulator at least once! You should then have this file ;-) Use the Mac Terminal app (in Utilities) to get to a command line & do:

      cd
ls -a           #should see ".android"
cd .android     #enter android prefs dir
ls              #should now see "adb_usb.ini"
vi adb_usb.ini  #or use other editor to add ports
