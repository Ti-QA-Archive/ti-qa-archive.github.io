Cancel Button in Camera randomly goes black

Hello all! I'm doing a very basic camera call in iOS (5.0.1 on a 4S) on SDK 1.7.5 as such:

    Titanium.Media.showCamera({

        success:function(event)
        {
            alert("Success");
        },
        cancel:function()
        {
            //Titanium.Media.hideCamera();
        },
        error:function(error)
        {
            // create alert
            var a = Titanium.UI.createAlertDialog({title:'Camera'});

            a.setMessage('Unexpected error: ' + error.code);

            // show alert
            a.show();
        },
        saveToPhotoGallery:false,
        allowEditing:false,
        mediaTypes:[Ti.Media.MEDIA_TYPE_PHOTO]
    });
}

For the most part it works just fine but every so often hitting the Cancel button once the camera starts goes to a full screen black screen and I'm forced to force kill the app to get it back to a normal state.

As you can see in my cancel event in the code I tried adding a hideCamera call but that didn't work either. The tough thing is it's random.. I haven't been able to reproduce it consistently but it happens at least once in 4-5 times of trying. Otherwise the rest of the camera works just perfectly… it's just the Cancel button.

Anyone else experience this? I'm trying my best not to lean towards it being a bug. Thanks for any insight anyone may have!
