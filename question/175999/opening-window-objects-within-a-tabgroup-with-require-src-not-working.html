Opening Window objects within a TabGroup with <Require src=""/> not working

Why does this:
(index.xml from the default 2-tab starter project)

<Alloy>
    <TabGroup>
        <Tab title="Tab 1" icon="KS_nav_ui.png">
            <Window title="Tab 1">
                <Label>I am Window 1</Label>
            </Window>
        </Tab>
        <Tab title="Tab 2" icon="KS_nav_views.png">
            <Window title="Tab 2">
                <Label>I am Window 2</Label>
            </Window>
        </Tab>
    </TabGroup>
</Alloy>

returns the windows correctly but this:
(index.xml refactored with <Require>)

<Alloy>
    <TabGroup>
        <Require src="tab1"></Require>
        <Require src="tab2"></Require>
    </TabGroup>
</Alloy>

(tab1.xml)

<Alloy>
    <Tab title="Tab 1" icon="KS_nav_ui.png">
        <Window title="Tab 1">
            <Label>I am Window 1</Label>
        </Window>
    </Tab>
</Alloy>

(tab2.xml)

<Alloy>
    <Tab title="Tab 2" icon="KS_nav_views.png">
        <Window title="Tab 2">
            <Label>I am Window 2</Label>
        </Window>
    </Tab>
</Alloy>

(index.js)

$.index.open();

Returns blank Windows.
It is the same code just broken out into separate xml files and included with the Require statements. I diff'd what I am doing against the github.com/appcelerator/alloy/tree/master/test/apps/ui/tabgroup example and it looks the same. Any help/explanation of what I am missing would be much appreciated.

    Hi

    When you say "it returns blank windows" - do you mean the windows appear to be empty and black?

    If so the answer is easy - and you will kick yourself. I guess your only TSS file is index.tss, so when you broke things up and did not create a tab1.tss or tab2.tss they no longer have any window styles that would have been set inside the index.xml.

    I suggest you create a file call app.tss and add the following style code - this will set the app wide default for the window style, you can always override later if you wish.

    "Window": {
    backgroundColor: '#fff'
}

    I have tried your code in both versions (iOS) and this solves the problem.
