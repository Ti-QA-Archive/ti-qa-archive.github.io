ACS Event Update - Custom Data Field Not Being Updated

All,

I have the following code: 

var custom_data = {
    toLawEnforcement:true //boolean custom field in ACS
}
Cloud.Events.update({
    event_id : eventID,
    custom_data: custom_data
    }, function(e) {
        Ti.API.log(JSON.stringify(e));
    }
);

I receive success each time, but the custom field fails to be updated. Here is the ACS log information

  • My workaround, for what it's worth…

    Since all I needed was an event status, I created a public KeyValue object setting the name to the EventID and the value to either ACTIVE or LAWENFORCEMENT. When I need the status of the event, I get the KVP with the associated EventID. If I need to change the status for the event, I append the new status to the KVP with the associated EventID. It is kludgeriffic, but it works.

    It would really be nice if I actually got a response either here or in JIRA after posting the bug. Oh well. At least my app is now functionally complete for iOS.
