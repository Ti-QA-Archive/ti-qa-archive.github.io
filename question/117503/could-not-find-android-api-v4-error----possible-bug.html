"Could not find Android API V4" error -- possible bug?

I was stuck trying to setup titanium developer. Whenever I try to set the path to android sdk, it gives an error "could not find Google API V4 (or 1.6) in your platforms directory".

I am using: -Windows xp sp2, -android-sdk_revision8, -android_platform_tools_r3, -android_api_v8(2.2)

>I have already copied adb.exe and the dll file into "tools" directory and Updated mobile sdk to 1.6.1. >This problem was resolved by copying the "android-8" directory to "android-4" in android sdk platform folder (pretty much a hack!!).

So, my question is why does it need an V4 api when i have the later API v8( 2.2) installed? Is this a bug I encountered?

  • Pradyumna

    Be aware that Windows XP SP2 is not supported. See the Titanium Compatibility Matrix. You should upgrade to Service Pack 3.

    Note that it is not apt.exe that needs to be copied into the tools folder, it is adb.exe and its associated dll.

    You should not copy the android-8 directory to android-4. Instead, install the missing package using the Android SDK and AVD Manager. The two packages you need/mention are SDK Platform Android 1.6, API 4, rev x and SDK Platform Android 2.2, API 8, rev x.

    The package, SDK Platform Android 1.6, API 4, rev x, is simply a required package. See Android SDK / Target Android Platform of the compatibility matrix. If you need to target other versions of Android also, you will need to install them in addition to this.

    Hope this helps
