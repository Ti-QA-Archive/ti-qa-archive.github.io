Multiple Workspaces in Titanium Studio

Currently I have Titanium Studio configured to use /Users/myname/Documents as the workspace. This means that whenever I create or import a project, it automatically copies that project into the root of the workspace directory. The essence of my question is, how do I create/import a project that is in a location other than the root of my workspace and have it not copy it into the root of the workspace?

My situation is that I have a hg repository called "Coach", which has directories under it named "Web" and "Titanium". I can only clone the whole repository (Coach), not just the Titanium directory. So within my Workspace I clone Coach, which has the child directory Titanium. I then try to import the Titanium directory into Titanium Studio as an existing project, which then copies the Titanium code into the root of the Workspace. This obviously breaks my source control structure, as I then have the Titanium directory under {workspace}/Titanium as well as {workspace}/Coach/Titanium.

This is an annoying issue, how do I get around it?
