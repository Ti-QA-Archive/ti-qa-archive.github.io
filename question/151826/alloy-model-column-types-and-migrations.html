Alloy Model Column Types and Migrations

Hi there-

I am looking through the Alloy model documentation, and just trying to piece together some things that I cannot find in full when generating Alloy Models and Migration files:

  1. What is the full set of types supported for Model columns (I've noticed in documentation, examples, questions differences between string, TEXT, real, etc…just looking for a list of the fully supported list of types)

  2. Is there better documentation/tutorials on how to create a migration? I saw a command line on creating a migration, and understand the concept of the migration file being the bridge between an old version of a model and a new, but just unclear on how to actual build the migration files, and migrate data between the old and new models…is this done via a command? Manually? Does this need to be done whenever adding new columns to a model or extending model functionality?

Thanks in advance

    1. The set of datatypes supported by a model in Alloy depends on what type of adapter you are using. the "sql" adapter uses SQLite, therefore it supports the types specified in the SQLite documentation. Same goes for all other adapters, it relies on the datatypes supported by the underlying data storage.
    2. Here's the official Alloy migration docs for the sql adapter. In addition, there's a few test apps in the alloy github repo that make use of multiple migrations. They should all be listed in the models folder. One particular example that shows how to create a table and then pre-seed it with rows in 2 different migrations can be found here: https://github.com/appcelerator/alloy/tree/master/test/apps/models/sql_queries
