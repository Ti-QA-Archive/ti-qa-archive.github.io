FileNotFoundException on Android - Not building all files.

Hey, my application is running great on iPhone but on Android I keep getting this error whenever the file is called: 

Wrapped java.io.FileNotFoundException: 
Resources/android_asset/Resources/Controllers/DataBaseController.js
(file:///android_asset/Resources/Controllers/LoginController.js#64480)

Where LoginController calls DataBaseController.
Both files are in the path Resources/Controllers/

I went to build/Android/bin/assets/Resources/Controllers/ and the file DataBaseController is there.

Why isn't the application seeing it in runtime, then?
Maybe Titanium is not building all the files?

PS: I'm running it on device.

Can someone give me a hand, please?

  • I experienced this error on the android when I changed the case of some files (for example changed include.js to Include.js)

    The solution for me was to delete the entire contents of the build directory. (not just deleting the contents of build/android). This forces a full re-compile.

    It appears that for android it doesn't detect the changes to the file when you change the case.
