    The require function is busted in SDK 1.7.2., which is why you get the IllegalArgumentException: size must be &gt;= 0 error. Download the in-progress 1.7.3 SDK from here: http://builds.appcelerator.com.s3.amazonaws.com/mobile/1_7_X/mobilesdk-1.7.3.v20110808134952-win32.zip

    or wait until it is officially released on Monday 22 August 2011.

  • Just try to change module name:

    From:

    ~/.titanium/modules/android/com.codeboxed.audiorecorder/0.1/audioRecorder.jar

    To:

    ~/.titanium/modules/android/com.codeboxed.audiorecorder/0.1/audiorecorder.jar

    This should helps in case Titanium could not load module.

    I guess this is general problem for all 3rd party Titanium modules under Linux systems as this systems are case sensitive for file names.
