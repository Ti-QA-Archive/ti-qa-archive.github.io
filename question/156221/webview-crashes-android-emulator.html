WebView crashes Android Emulator

Very simple code. index.xml:

<Alloy>
    <Window>
        <WebView id="myWebView" />
    </Window>
</Alloy>

index.js:

$.index.open();
$.myWebView.url = 'http://www.google.com';

Google site displays for about 5 second, then crashes with following output:

...
[ERROR][InputDispatcher(   62)] channel '407b4fa8 com.blah.AppName/com.blah.AppName.AppNameActivity (server)' ~ Consumer closed input channel or an error occurred.  events=0x8
[ERROR][InputDispatcher(   62)] channel '407b4fa8 com.blah.AppName/com.blah.AppName.AppNameActivity (server)' ~ Channel is unrecoverably broken and will be disposed!
...

I have tried downgrading my AVDs from API 2.3.3 to 2.2 in the Android SDK Manager, but the emulator won't even run then.

Any ideas?

Your basic data:

Application type: mobile

Titanium SDK: 3.1.1.GA

Device: Android simulator

Host Operating System: OSX 10.7.5

Titanium Studio, build: 3.1.2.201308091617
