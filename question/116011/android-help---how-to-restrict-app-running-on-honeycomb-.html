  • Answer to my own questions. Setting sdk version in between manifest should take care of the problem but that's not the case.

    http://developer.android.com/guide/appendix/api-levels.html

    
 <uses-sdk android:minSdkVersion="4" android:targetSdkVersion="9" android:maxSdkVersion="10" />

    Supposedly if i were to limit maxsdkVersion, it should prevent users from installing the app. The problem is i was still able to install maxSdkVersion="7" (2.1) on 2.2 device. Also, on emulator, it's still attempt to run the app with honeycomb SDK despite maxSdkVersion set to 10.

    Any pointer?

    • From the AndroidManifest.xml documentation "Future versions of Android (beyond Android 2.0.1) will no longer check or enforce the android:maxSdkVersion attribute during installation or revalidation. Android Market will continue to use the attribute as a filter, however, when presenting users with applications available for download."
