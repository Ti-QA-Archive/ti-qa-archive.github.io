Including third party framework when distributing a custom module

I've built a custom module for iOS to support an ad delivery network and everything is working as expected.

I now want to contribute this module to the titanium network but I can't figure out how to distribute it, the problem is the 3rd party framework delivered by the ad network.

In my module.xconfig I have added this line to include the framework at compile time:

OTHER_LDFLAGS=$(inherited) -F"<path to the custom framework on my disk>" -framework <Framework name>

It works totally fine, but when I distribute my module won't the other users need to put the framework (which they'll get from the ad network) in the same location to build their app? Is there any best practice on where the put the downloaded framework or should the user edit the path once the module is distributed?
