Missing icon file when submiting IOS app

Hi,

I have two blocking errors when I submit my app to Apple :

"Missing required icon file. The bundle does not contain an app icon for iphone / ipod touch of exactly 57x57, in png format for IOS versions < 7.0"

and more or less the same message with "72x72 pixels" icon.

Yet, I generated my icons with ticons.fokkezb.nl.
I copied the appicon.png file in ios/ and Resources/ directories.
I renamed them to icon.png, event Icon.png.
I recreated the icon from scratch to resize them to 57px.
I create a appicon-57.png file…
I even tried a transparent background, as well as a blank one !

Now I'm really running out of options. Would anyone have an idea ?

An help would be welcome,
Florence.

    Florence, I had a problem like this earlier. The appicon-* should be automatically added to the app's plist file on a build. Make sure you do a clean before rebuild. If it fails again try opening the app's plist file and see if the appicon-57 is added.

    • Got it !!!

      In my tiapp file, I had a custom logo name instead of appicon.png.

      <copyright>2014 by Florence</copyright>
      <icon>logo.png</icon>
      <persistent-wifi>false</persistent-wifi>

      Obviously, Titanium gets confused and does not recognize the usual other appiconXXX file in Resources/iphone.

      Using directly appicon.png in the icon element did the trick.

      Thanks a lot for your help !!!
