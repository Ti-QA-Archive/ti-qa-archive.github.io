Android - What is the best way to create window?

Hi all,

Please see my below example:

//METHOD 1
//windows.js 
exports.Window = function(e){
    return Ti.UI.createWindow(e);
};
//app.js
var UI = require('window');
var win = UI.Window({backgroundColor:'#999'});
win.open();

//METHOD 2
//windows.js
function create_window(e){
    return Ti.UI.createWindow(e);
}
//app.js
Ti.include('windows.js'); // This Ti.include will be deprecated, so any other way to include a js file?
var win = create_window({backgroundColor:'#999'});
win.open();

Base on http://docs.appcelerator.com/titanium/3.0/#!/guide/Coding_Best_Practices-section-30082362_CodingBestPractices-Deferscriptloading we should follow method 1. But what is the different here? Actually the two methods are same each other, why we need to follow method 1?

Another small question is "Ti.include will be deprecated, so any other way to include a js file?"

    Hi Iker, your question is completely valid about using the first method.

    My explanation to your question is that Titanium recommends to use the CommonJS way to include the files in your project, instead the Ti.include method.
    It is just because that any Ti call is actually a cross-brodge call because whenever we make any Ti call, titanium has to cross the bridge to complete the request from native/core modules of platforms & hence it is not considered as safe process to make Ti calls just to import some files.
    Read more about this at Ti.include() Deprecated

    This is also somewhat the same case with using the Ti.App event listeners

    • You are right about that code except one thing:

      • whenever we define variables inside controllers (.js) file, those variables are never accessed by $.
      • $ is used only to access the elements from views file(.xml) in Alloy for id attributes.

      Hence above code will be more precise this way:

      var winMain = Ti.UI.createWindow({

      // to open the window,iker_tran holds the current window now
