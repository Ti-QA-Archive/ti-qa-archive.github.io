Building and Deploying Alloy Apps on iPhone and iPad Devices

Hi all-

I have a question on deployment of Alloy apps. In the past, I have created apps specifically for iPads or iPhone via the Titanium framework. With Alloy, I have one Alloy project, and using the conditionals within the tss/xml view/controller functions, I utilize the single project to generate views specific to the iPad or iPhone platforms (retina/non-retina), which all work well in the simulators. Also, when opening the builds up in XCode and running the apps on my physical devices with the Device settings set to iPhone or iPad as appropriate, I can run/install the dev release to my device and also run as expected.

Here is the question. When I build my production application, can I build one app, which will work on both devices, or do I need to build one app, targetting the iPhone within the XCode iOS Application Target settings, and another targetting iPad, to pick up the iPad views/settings? I was wondering if there were docs/guidance for others building/deploying for universal targets out there. I'd love to build one app that users could grab that would work as expected on iPhone/iPad since it is all compiled within the one project.

Thanks for any guidance.

Talk soon
Chris

    Hi

    When you initially create the app project you are asked if you want to target; hone, iPad, android (and now Tizen & blackberry). Concentrating solely on iOS for this conversation.

    • If you choose, JUST iPhone, then you app is only going to compile as iPhone, if you run it on an iPad if will appear as an iPhone app.
    • If you choose JUST ipad then your app will not work on iPhone.
    • If however you choose BOTH iPhone and iPad, then your app is compiled as a universal app and will run as iPhone on iPhone and iPad on iPad. As you have already noted the alloy framework enables you to make light(re) work of targeting sections of you UI to each platform relatively easily (you still need to be a good coder).

    So if you want to have one code base and target both iPhone and iPad, simply choose both targets when you start the app.

    Two important notes;

    • you can change your target platforms AFTER you have started the project, just remember to code for both of course.
    • this ability has nothing to do with Alloy, you can do this with or without using Alloy. Alloy just makes managing the differences easier from a UI stand-point and a few other tricks of course.

    Have fun
