Building Scripts

I have a bunch of different apps that use the same application core folder.

When I make an improvement or update to the core every app needs an update in the app store (both android and iOS). I am trying to scale the operation by creating a single script to just run and then send up all the new versions…

Any one have tips for this?

  • You could use symlinks, if you're on Mac/Linux, and then have all the common files in one directory and then symlink the files into each app's directory.

    You could also use git submodules.

  • This is an old question that I figured out. You cannot use symlinks (and really any variation hardlink, virtual folder, etc).

    With symlinks the projects will have issues when building for device.

    I am currently using a program called FolderWatch and it works great. Keeps my core folder in sync.
