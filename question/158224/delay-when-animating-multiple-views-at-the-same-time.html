Delay when animating multiple views at the same time

I have this code in my app to animate these two views at the same time:

listContainerFinal.animate({
    left: 0,
    duration: 250,
    curve: Ti.UI.ANIMATION_CURVE_EASE_IN_OUT
});

pageContainerFinal.animate({
    left: contWidth, //contWidth is '300dip'
    duration: 250,
    curve: Ti.UI.ANIMATION_CURVE_EASE_IN_OUT
});

Which works fine, but I am testing it on my device (iPhone, but I assume Android/iPad will have the same issue), and there is a random short delay between each animation.

Is there a way to force the device to animate both at exactly the same time?

I need them to stay perfectly in sync with each other or else it looks weird because they don't animate at the same time, and you can visually see the delay between execution of each animation.

Cheers,
Ben
