Disappearing barImage in tabGroup

I have a tabGroup with a couple of windows, and each window has a barImage (which is actually the same for most windows). If you click once on any tab, everything behaves as expected, but when you click on a tab that's already active, the barImage disappears. Tested in both the nightly and the current stable versions (2.0.1 GA2). Tested only on iOS, but is easily replicable in both the simulator and a real device.

var tabGroup = Titanium.UI.createTabGroup();

var tab1 = Titanium.UI.createTab({  
    icon   : 'KS_nav_views.png',
    title  : 'Tab 1',
    window : Titanium.UI.createWindow({  
        title           : 'Tab 1',
        backgroundColor : '#fff',
        barImage        : '/KS_nav_ui.png',
    }),
});

var tab2 = Titanium.UI.createTab({  
    icon   : 'KS_nav_ui.png',
    title  : 'Tab 2',
    window : Titanium.UI.createWindow({  
        title           : 'Tab 2',
        backgroundColor : '#fff',
        barImage        : '/KS_nav_ui.png',
    }),
});

tabGroup.addTab(tab1);  
tabGroup.addTab(tab2);  

tabGroup.open();

Try clicking on any tab twice. The images are from the default Titanium project, but any images will do.
