ListView rows do not fire many events (most importantly postlayout)

We have a relatively complex template for list view items, these items consist of handling a number of events such as touchend, dragend, scroll, and postlayout. We originally used TableView objects to display this content but have found the initial rendering of the TableView (even with the use of className) containing 25 rows takes a great deal of time (6 to 9 seconds). When we converted the layout to ListView the rendering was instant. Obviously we want to use ListView because we could never release an app that takes 6 seconds to render a list of 25 items, but we can not get all of the events to work that we need, specifically touchend and postlayout.

We are assigning the events through the use of the events template property, an example of the template code looks like this (this code is coffeescript so ignore the syntax difference but it's pretty straightforward that we are assigning a method of handlePostLayout to the postlayout event:

templates =
      properties:
        height: 204
      events:
        postlayout: handlePostLayout
      childTemplates: [...]

Our ListView displays beautifully but unfortunately the handlePostLayout method is never called. As well when we assign an event to touchend the handler is never called.

To get around this we were attempting to gain some sort of access to the generated views but it doesn't look like there is any access to these child views.

Any help would be appreciated, we really would like to gain the performance benefits of ListView.
