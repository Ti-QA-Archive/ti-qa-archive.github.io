  • I remember seeing a comment somewhere about this from Appcelerator. Something to the effect of it being frozen. It's not going anywhere, but it's also not going anywhere :)

    Currently I am doing most of my mobile development (testing / playing around with the platform) through the console, without the use of Studio or Developer. This is a personal preference because I use custom scripts to automate and install custom SDK builds from the latest source branch. I will plug Studio as a huge leap forward in Titanium development. The debugging is a welcomed breath of fresh air.

    Here's what I do on OSX. Add a couple things to your enviornment (.bash_profile, or whatever):

    alias titanium="/Library/Application\ Support/Titanium/mobilesdk/osx/1.8.0/titanium.py"
export ANDROID_SDK="/Developer/android-sdk-mac_86"

    Obviously replacing the paths above with those for your installation. Now you are free to do most things from the command line:

    Create New App

    titanium create --platform="iphone,android" --name=testapp --id=com.zaplabs.testapp

    Build and Run Project (from root of project directory) 

    titanium run --platform=iphone
titanium run --platform=android

    Keep in mind, I don't think any of this is officially supported or documented as it's intended to be used internally by Titanium Studio/Developer. Everything may change at any time, and the command arguments and flags are not documented in many cases.
