HTTPS Client Call

Hi all,

I was asked to create a mobile app for an organization that demands secure connections between their web services and the mobile app HTTP calls.

As far as I know, using an HTTP Client call and just calling the website (or web service) using https://…. does not encrypt the data.

How can I use a secure http connection/call between a website/webservice and the mobile app?

Please provide an example or something because I could not find any on the internet.

    Hi

    You can use HTTPS in place of HTTP, as long as the domain has a valid SSL, and it will be sent encrypted.

    HTTPS is encrypted by design, so any data that is sent using this protocol will be encrypted. If you wanted you could also use your own encryption on the data before it is sent, but that is your choice.

    Hope this helps

    • So that means if I use https in my web services, I will have the same level of security as using https in my browser when calling those web services?

      — commented September 3rd 2014 by Michael Massalas

    • HTTPS is simply a communications protocol, it provides bidirectional encryption of communications between a client and server. It will work the same way regardless of what client you use.

      The only difference between an app and browser is that you are afforded the opportunity to inspect the certificates to decide whether you trust it, whereas through an API you are not.
