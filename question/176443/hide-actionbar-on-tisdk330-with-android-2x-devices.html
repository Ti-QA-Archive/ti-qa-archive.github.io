Hide ActionBar on TiSDK3.3.0 with Android 2.x Devices

From 3.3.0 on, the navBarHidden property of a window no longer works. To hide the Titlebar/Actionbar, you'd have to create a custom theme as described in the docs.

<?xml version="1.0" encoding="utf-8"?>
<resources>
    <style name="Theme.AppCompat.Light.NoActionBar" parent="@style/Theme.AppCompat.Light">
        <item name="android:windowActionBar">false</item>
        <item name="android:windowNoTitle">true</item>
    </style>
</resources>

This works nicely on devices with Android >=3 (with native AcrionBar support). On Android 2.3.3, the (compat) ActionBar is still shown and there seems to be no way to disable it. Any clues?

  • You should be able to hide the action bar with JavaScript code after the window is opened (and the action bar displayed);

    win.addEventListener('open', function(e) {
   if(e.activity) {
      e.activity.actionBar.hide();
   }
});

    Or, you can modify the theme to automatically hide the action bar altogether:

    First, add a custom theme file to your project, and set the android:windowActionBar item to false and android:windowNoTitle to true.

    platform/android/res/values/custom_theme.xml

    <?xml version="1.0" encoding="utf-8"?>
<resources>
    <style name="Theme.NoActionBar" parent="@style/Theme.AppCompat">
        <item name="android:windowActionBar">false</item>
        <item name="android:windowNoTitle">true</item>
    </style>
</resources>

    Then, modify your tiapp.xml file to use the modified theme. This will globally hide the action bar in all of your activities:

    <android xmlns:android="http://schemas.android.com/apk/res/android">
    <manifest>
        <application android:theme="@style/Theme.NoActionBar"/>
        <uses-sdk android:minSdkVersion="14" android:targetSdkVersion="19"/>
    </manifest>
</android>

    Though, you might want to use the techniques described at overriding an activity's theme unless you want your entire app to be without an action bar.
