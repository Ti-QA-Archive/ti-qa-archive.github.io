  • Hi

    We need to rethink your question.

    • If you simply need a copy of your database exactly as it stands with no changes then YES you can do this - you simply copy the db file using normal file system commands as long as the database is closed.

    • If you wish to make alterations to the data as it is being copied then you will need to create your own function for this as there is no such thing as an automated approach for a custom process.

      Note: Your structure does NOT need to be the same as you are in control of the export itself - so the source can come from one or more tables with zero or more filters and end up in any mixture you wish. If you want an exact copy then my first point is the solution.

    As this is a sql style db you use a select statement, loop the data and insert as required. But I guess you know this part.

    Your specific questions;

    • 1 = no
    • 2 = no
