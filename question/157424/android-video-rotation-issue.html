Android Video rotation issue

I'm playing streamed videos using the Ti VideoPlayer in an Android app where its only window is set to [Titanium.UI.PORTRAIT]. When the video starts to play it does so in portrait mode and one has to physically rotate the device to shift to landscape, however in doing so the video stops, sometimes reloads (can take two minutes to do so) and continues playing - but sometimes throws an error and crashes.

I've tried forcing the window to landscape just before creating the videoPlayer using Ti.Android.currentActivity.setRequestedOrientation(Ti.Android.SCREEN_ORIENTATION_LANDSCAPE) and that works, but the device will still not stay in landscape as on rotation it shifts back to portrait and exhibits the behaviour as described above. The error message in the log at this point is E/ARTSPConnection(176): Server unexpectedly closed the connection.

I also tried using the 'orientationchange' event to catch when a rotation was taking place during a video play to force it back to landscape, but the server still seems to want to close the connection.

What is the trick here? Ideally I want to change to landscape on starting a video and return to portrait only on its exit it with no pauses or crashes. Too much for Android?

Mobile /Titanium SDK version 3.1.3.GA/ Android/ Nexus 7/ Mac OSX
