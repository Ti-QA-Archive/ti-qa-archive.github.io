  • Bug Found!!!

    So that what I expected if you read ma last comment just above.
    If you try to set a property and pass something undefined as the 1st parameter, it flush all your other properties.

    Try this on a new project, run it at least twice:

    app.js :

    (function(machin){


    Ti.API.info(Ti.App.Properties.listProperties());

    Ti.App.Properties.setString(machin,'chips');
    Ti.App.Properties.setString('thisIsARealString','veggies');

    Ti.API.info( Ti.App.Properties.getString('thisIsARealString'));

    Ti.API.info(Ti.App.Properties.listProperties());

})();

    The 1st time you run it, it is normal that the listProperties() only returns Titanium own properties, but it should list my 'thisIsARealString' the 2nd time, but it doesn't.

    Have you the same behavior?

    • For the little story behind my undefined variable, here's what my code looked like:

      MyParentClass.prototype.setLocalHistory= function (data) {
  Titanium.App.Properties.setList(this.constructor.HISTORY_PROPERTY_ID, data);
};

      When my JS code is loaded, it looks like it is parsed once before it is attached to my child object/class, so this.constructor.HISTORY_PROPERTY_ID is undefined

      Corrected code:

      MyParentClass.prototype.setLocalHistory= function (data) {
    if (this.constructor.HISTORY_PROPERTY_ID) {
        Titanium.App.Properties.setList(this.constructor.HISTORY_PROPERTY_ID, data);
    }
};
