Can't publish app update

Hello!
I get this error when im trying to upload my app update to ITC.

This bundle is invalid. The key CFBundleShortVersionString in the Info.plist file must contain a higher version than that of the previously uploaded version.

The CFBundleShortVersionString for the previous version is, according to ITC, 1.0. The bundle version i 1.0.1.

Now, if i look at CFBundleShortVersionString in the archive for the new app version, it is 1.0.2, just like the new version i prepped in ITC. I have been struggling with this for days now, and can't seem to figure it out. What is causing this? I have tried to add BundleVersion to info.plist with a higher number than the previous version without any luck…
Titanium SDK 1.8.3 V8. XCode 4.2

Thank you!

  • Anton,

    There was a problem with the CFBundleShortVersionString not being increased and always being 1.0 TIMOB-7674

    This was resolved, then it was discovered that for older products that never had a CFBundleShortVersionString added when the project was created, the value was 1.0.

    Both issues have been resolved in 2.0.x. I would suggest you download the update if you can (from 1.8.x to 2.0.1 should require minimal refactoring and modifications)

    • You know, there are two info.plist files in each xarchive bundle.
      The one at the root of the xarchive only ever has the cfbundleshortversionstring while the one inside the .app itself should have both version strings in it.

      Be sure you are looking at the info.plist in the .app for the xarchive you are trying to submit and for the one you already submitted in the past. So looking for 4 data points:

      Published xarchive

      • CFBundleShortVersionString
      • CFBundleVersion

      New xarchive

      • CFBundleShortVersionString
      • CFBundleVersion
