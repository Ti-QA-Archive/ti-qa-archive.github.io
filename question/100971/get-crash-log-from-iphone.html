  • What I do is attach a USB cable to my device, then watch all the errors in the console window of either XCode Organizer, or the iPhone Configuration Utility. You see everything you would normally see in the Titanium Developer console window when using the simulator.

    You already have XCode installed – it is used by Ti to compile apps. In fact it should already be loaded if you recently did a build. Switch to it, and use menu Window > Organizer, select your device in the left frame, then display the Console tab.

    Or if you have Apple's iPhone Configuration Utility installed, you could do the same thing.

    SOLUTION?!

    I moved all code inside app.js to an include file app-window.js

    # app.js
Titanium.include('app-window.js');
// this sets the background color of the master UIView (when there are no windows/tab groups on it)
Titanium.UI.setBackgroundColor('#000');

    # app-window.js
var win1 = Titanium.UI.createWindow({ 
// ...
});
// ...

    Now works fine on iPhone. Maybe Titanium libraries UIModule are slow to load? Include file helps to waiting the libraries loading?
