  • As my project, i face the same situation as yours. And our team decided that we will place the logic in server side ( As Shannon commented ) Why we have to do that:

    • Server side should know all users information as: Name, email, push notification config,… So that it will be easy for server to decide send or not send notification to user
    • If you let's app to decide what is the notification can be showed, it will be wasted the connection between your server and Apple/Google push notification server because you will send notification to user's device, but our app did not show this notification, Why waste 1 push notification?

    I hope that you will make the good decision! Thank you.
