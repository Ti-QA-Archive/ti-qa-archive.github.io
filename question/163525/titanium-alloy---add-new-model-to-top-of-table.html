Titanium Alloy - Add new model to top of table

Part of my Alloy (SDK 3.2.2.GA) project includes a mailbox controller that naturally adds messages received from an HTTPClient request to a collection and displays them in a TableView.

However, when receiving new messages, the added models are appended to the bottom of the table. After refreshing they are then correctly displayed at the top.

Is there a way to specify reverse table row rendering, or something to that effect? I thought a comparator that gets the created UNIX timestamp would do it automatically.

  • Sorry, I realized I just created the comparator incorrectly (didn't supply the attribute name as a string). It's all good now.

    Just in case anyone needs to know, just put the following in your model definition:

    extendCollection: function(Collection) {
    _.extend(Collection.prototype, {

        /* 
        *  In this instance, 'created' is an integer timestamp
        *  The '-' operator reverses the order (ie. newest first)
        */
        comparator: function(model) {
            return -model.get('created');
        }

    });

    return Collection;
}
