ImageView gives error - Bitmap bounds could not be determined

I'm seeing an odd issue on Android in Windows 7 x64. Here is the sample code:

app.js:

Ti.UI.setBackgroundColor('#fff');

var win = Ti.UI.createWindow();
win.open();

win.add(Ti.UI.createLabel({ text: "test", top: 0, width: 100 }));

var path = Ti.Filesystem.resourcesDirectory + "images/logo.png";
Ti.API.info("image path: " + path);
var image = Ti.UI.createImageView({ image: path, top: 30, left: 0, width: 200, height: 59 });
Ti.API.info("image is null ? " + image == null);
win.add(image);

And here is the output:

I/TiAPI   ( 1019): (kroll$1: app://app.js) [328,1741] image path: app://images/logo.png
I/TiAPI   ( 1019): (kroll$1: app://app.js) [9,1750] false
D/skia    ( 1019): --- decoder->decode returned false
W/TiDrawableReference( 1019): (main) [287,2037] Bitmap bounds could not be determined.  If bitmap is loaded, it won't be scaled.
D/skia    ( 1019): --- decoder->decode returned false

The image is not shown, and it gives the error above about bitmap bounds not being determined. It works fine on OSX. I tested on both 1.7.2 and the latest 1.7.3 CI build. I can provide the png file if necessary.
