Is the "call" function method supported?

I am using a compiler that emits javascript that calls functions like:

var h = null;
Titanium.UI.setBackgroundColor.call(h, "#000");

instead of the "normal":

Titanium.UI.setBackgroundColor("#000");

This will give me a runtime error with Titanium, telling me that the "call" method is not implemented… which was kind of surprising as I thought that was part of the standard JS-language (?).

Then browsing thru the Q&As, I found that the following "trick" seems to work:

Titanium.UI.setBackgroundColor.call = Function.prototype.call; 
var h = null;
Titanium.UI.setBackgroundColor.call(h, "#000");

Is the observation correct, that "call" is not implemented as a function method in Titanium mobile?
If so, any specific reason?

Is the workaround that I found the right one?
Are there other, maybe better solutions?

Thanks for your help,
FrankS.

  • Hello Frank. Titanium objects are not native JS objects, therefore, they can implement any behavior they want in (bad) theory (similar thing can be found in DOM environment). I wish that they would act more like native JS object, then you could do cool things directy on Titanium object, this way you need to wrap them with native ones. :)
