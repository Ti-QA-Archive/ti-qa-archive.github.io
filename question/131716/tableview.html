    Because you have one section in table. :D

    No, really.

    "table.data" contains array of section. Each element of that array (one section) contains "rows" property that contains array of rows (in that section).

    If you haven't created section, one is created by default.

    So, if you have only one section, you can get number of rows like this:

    var sections = table.data;
var oneDefaultSection = table.data[0];
var rowsInOneDefaultSection = table.data[0].rows;
var numberOfRowsInOneDefaultSection = table.data[0].rows.length;

    • "data.length" will return number of section.

      Each section has its own number of rows. "data" property looks like this:

      [
    0: { rows: [ TiUITableViewRow, TiUITableViewRow ] },
    1: { rows: [ TiUITableViewRow, TiUITableViewRow ] },
    2: { rows: [ TiUITableViewRow, TiUITableViewRow ] },
    3: { rows: [ TiUITableViewRow, TiUITableViewRow ] }
]

      when there are 4 sections with 2 rows in each sectionn (that means 4 * 2 = 8 rows in table view).
