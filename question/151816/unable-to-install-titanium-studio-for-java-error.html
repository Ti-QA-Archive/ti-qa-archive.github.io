  • Here's what was going on: installer dug up some obsolete jdk home path from the windows registry - was not in my env variables, could only find it in the registry - and then did a check to verify the location by looking for c:\jdk_dir\bin\java.exe. Since I had long ago deleted the jdk the check failed and the install was aborted. I put a java.exe in the expected location and tried again. The installer finished successfully. Interesting thing is that when I start up, titanium studio seems to be using the jdk found in my path or java_home env variables, not the one for which the installer was checking.

  • That is usually due to permission at the Windows level. My best suggestion to you is that you install the necessary components yourself.

    • The JDK
    • The Android SDK
    • The Python Interpreter (if needed)

    Once all these are in place (as well as the right environment variables). Then relaunch the Titanium Installation program. All the necessary items will be detected and properly configured.

    You could always try and hope that Titanium will do this correctly for you. But from experience, I always had to do this manually on Windows.
