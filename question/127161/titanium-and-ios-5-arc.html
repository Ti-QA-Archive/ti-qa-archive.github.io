Titanium and iOS 5 ARC

Reposted from http://developer.appcelerator.com/question/126363/titanium-and-ios-5-arc that was closed due to iOS5 being in Beta. XCODE 4.2 and iOS 5 have now been officially released.

Will Titanium-generated iOS code (and compilation) use the new XCode auto reference counting? It is a iOS 5 SDK compile-time feature that is supposed to work on iOS 4.x devices. I think that would solve the leaking problem so many people complain about on Titanium - that would make Titanium a safer bet to me.

  • In a way Titanium does not really benefit from ARC because ARC is just a compile time helper. It really to save the coder from having to remember to retain/release. As all the Titanium code is generated it should be releasing and retaining properly already. The only reason ARC would be useful to Titanium is if Titanium was leaky.
