Downloading JSON from remote Server to render UI

Hi,

We are planning to implement a logic where-in a screen will be rendered based on a JSON response received from a remote server. It is something like we will have a parser, which will parse the JSON and create the UI dynamically.

Hence we can change the view anytime we want without going through the process of pushing the APK/IPA to production.

The drawing logic will be present in the client side but only the structure will be sent from the web service response.

For example:
{
controls : [
{
type : "textfield",
hintText : "enter username",
label : "User Name"
},
{
type : "textfield",
hintText : "enter password",
label : "Password"
},
{
type : "button",
title : "Login",
}
]
}

Our query is that, whether Apple will approve this process of displaying the UI, that is, based on the JSON received from a web service response. Is there any chance of the application getting rejected or has anyone done anything similar earlier for Apple.

We will be using 3.4.0.GA SDK and targeting iOS 7 onwards.
