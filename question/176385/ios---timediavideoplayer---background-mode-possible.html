iOS - Ti.Media.VideoPlayer - background mode possible?

I saw it here

I have the correct setup:

Java Script file

Ti.Media.audioSessionMode = Ti.Media.AUDIO_SESSION_MODE_PLAYBACK;

tiapp.xml file:

<key>UIBackgroundModes</key>
                <array>
                    <string>audio</string>
                </array>

However, I don't see the audio of a videoplayer continue to play in the background (press Home or Lock). In my case the video player just pauses. How is it possible as the author of the above post claims?

Fyi, in my case, the video player (Ti.Media.VideoPlayer) is added to a view, which is added to a window.

Running on iOS 7, SDK 3.3.0

  • Hy, Shawn Lan, Yes it is possible in iOS. not in Android. Try this, it will work.

    For iOS:

    In your app, put

     Ti.Media.audioSessionMode = Ti.Media.AUDIO_SESSION_MODE_PLAYBACK;

    In your tiapp.xml, add

    <iphone>
        <backgroundModes>
            <mode>audio</mode>
        </backgroundModes>
</iphone>

    If you have problem, please comment.

    Thanks
