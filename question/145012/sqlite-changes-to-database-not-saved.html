SQLite changes to database not saved

Type: mobile
TSDK: 2.1.4.GA
Platform: IOS SDK 6.0
Device IPhone simulator
Host OS: Mountain Lion 10.8.2 Xcode 4.5.2
Titanium Studio Version: 2.1.2.201208301612

var db = Titanium.Database.install('../database/mobile_database.sqlite','mobile_database');
db.execute('CREATE TABLE newtable (ID INTEGER PRIMARY KEY AUTOINCREMENT, TITLE TEXT, LINK TEXT, DESCRIPTION TEXT)');
db.close();

the file: mobile_database.sqlite does exist but is empty.

when the simulator is run there is no error, however the table is not created in the mobile_database.sqlite file.
If i close the simulator and re run i get a message saying the table already exists but mobile_database.sqlite is still empty.
When i delete the app from the simulator and re run, there is no error.

So the table does get created but the .sqlite file is not updated

  • According to the documentation:

    http://docs.appcelerator.com/titanium/2.1/#!/api/Titanium.Database-method-install

    The "install" method just copies the database to the default location. The .sqlite file is basically just a starting point for the database. In your case you would most likely want to go ahead and create your "newtable" in the.sqlite file, otherwise you might as well just use the "open" method.
