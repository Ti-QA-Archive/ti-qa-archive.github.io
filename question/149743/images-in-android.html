    Hi

    A few things to note;

    • defaultImage is only required if the image you have in the image is a remote image (starts with http) and not one locally on the device.
    • Devices images should be placed in specific folder locations for them to work as you want them and for good reasons.

    Device specific images should be placed inside the android folder of android specific images and iphone for iOS images (iPhone, iPod and iPad).

    If you are creating an app that support both main platforms and some image are the same no matter the device (think photos) then you can and should create a folder outside the device folders.

    If you place an image here

    • /resources/
    • /resources/android/
    • /resources/android/test1.png

    Then your code would be;

    var myImageView = Ti.UI.createImageView({
    height: Ti.UI.SIZE,
    image : '/test1.png',
    width: Ti.UI.SIZE
});

    However if you place an image here;

    • /resources/
    • /resources/android/
    • /resources/android/images/
    • /resources/android/images/test1.png

    Then your code would be;

    var myImageView = Ti.UI.createImageView({
    height: Ti.UI.SIZE,
    image : '/images/test1.png',
    width: Ti.UI.SIZE
});

    This may seem strange but you should keep in mind the device folders are virtual, which means they will collapse down when actually on the device as only the device folders images would be copied over.

    With android you can also have a different resolution image for each of the different densities that Android supports. In this case you would place an image converted to the correct resolution in each of the folders below. When on the device ONLY the correct one is actually displayed - again without the extra folder information.

    • res-ldpi = low res 75% size of normal
    • res-mdpi = medium res 100% size of normal
    • res-hdpi = high res 150% size of normal
    • res-xhdpi = xtra high res 200% size of normal

    An Android folder structure with these images would look like this;

    • /resources/
    • /resources/android/
    • /resources/android/images/res-ldpi/
    • /resources/android/images/res-mdpi/
    • /resources/android/images/res-hdpi/
    • /resources/android/images/res-xhdpi/

    And your image would be placed as follows;

    • /resources/
    • /resources/android/
    • /resources/android/images/res-ldpi/test1.png
    • /resources/android/images/res-mdpi/test1.png
    • /resources/android/images/res-hdpi/test1.png
    • /resources/android/images/res-xhdpi/test1.png

    Your code will still look like this;

    var myImageView = Ti.UI.createImageView({
    height: Ti.UI.SIZE,
    image : '/images/test1.png',
    width: Ti.UI.SIZE
});

    And the correct image will be displayed depending on the density.

    Now in a cross platform app supporting all image densities (or at least the main ones - your folders would look like this with the images inside;

    • /resources/
    • /resources/android/
    • /resources/android/images/res-ldpi/test1.png
    • /resources/android/images/res-mdpi/test1.png
    • /resources/android/images/res-hdpi/test1.png
    • /resources/android/images/res-xhdpi/test1.png
    • /resources/iphone/
    • /resources/iphone/images/test1.png
    • /resources/iphone/images/test1@2x.png

    With these two being

    • /resources/iphone/images/test1.png = non-retina
    • /resources/iphone/images/test1@2x.png = retina

    In order to get the Android auto sizing images to work, edit your tiapp.xml to include this code block.

    <android xmlns:android="http://schemas.android.com/apk/res/android">
    <manifest android:installLocation="auto" android:versionCode="1" android:versionName="1.0">
        <supports-screens android:anyDensity="true"
            android:largeScreens="true" android:normalScreens="true" android:smallScreens="true"/>
        <uses-sdk android:minSdkVersion="7"/>
    </manifest>
    <tool-api-level>8</tool-api-level>
</android>

    • @Gowtham - not sure what you mean by ratio but the iPhone retina is twice the width and height of iPhone non-retina.

      • iPhone = 320x480
      • iPhone Retina = 640x960
      • iPhone Retina Tall = 640x1136

      Unless you change the system units from the default for iOS they are reported to Titanium as follows for layout purposes;

      • iPhone = 320x480
      • iPhone Retina = 320x480
      • iPhone Retina Tall = 320x568

      For retina and retina tall you can use the image suffix @2x (the image being twice the width and twice the height) so image.png becomes image@2x.png.

      For retina tall where you require an image to be different from a retina image (backgrounds are a good example) then image-568h@2x.png is the file name change.
