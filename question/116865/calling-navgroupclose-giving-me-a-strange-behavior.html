Calling "navGroup.close" giving me a strange behavior

Hi All,
I'm writing an iPhone application which is utilizing NavigationGroup heavily. I have a button in one of the screens which when pressed by the user, takes the user to the previous screen. Here is the code I wrote to do that:

navGroup.close(currentWindow, {animate: true});

Here "currentWindow" is the existing screen and "navGroup" references to NavigationGroup

This is taking the user back to the previous screen BUT it first makes the existing screen black and then animates the previous screen from left to right. Ideally, it should not make the existing screen black AND should do its animation for the existing screen from the right to he left coming the previous screen from the right to the left.

I know it's a simple thing to do but I'm unable to get it to work. Please let me know how can I solve this issue?

Thank you very much in advance.

  • I have the same issue so I posted the same here with sample code: http://developer.appcelerator.com/question/118350/navigation-group-close-strange-behaviour-on-iphone

    The funny is that I haven't tried with pre 1.6.0 SDKs, but in 1.6.x and 1.7.x contains this bug. The bug is very important I think and it was a showstopper for me. However the fix is really simple just like everything in Ti:

    1. Open TiUIiPhoneNavigationGroup.m in the build/iphone folder (it is somewhere inside a directory called Classes).
    2. Search for the close:properties: method.
    3. Fix the ridiculous Ti implementation

    The problem is that they take the stack of views (windows in Ti term), remove the top one and set the new stack with animation on.
