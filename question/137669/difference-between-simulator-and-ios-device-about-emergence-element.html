Difference between simulator and iOS Device (about emergence element)

Hi,
it will be a little complicate to explain you my problem : because i am french, and because my probleme only appear on a real device (iPhone 4)

But i try !

I have a window with few view : simple view, image view, and table view.

On my simulator (retina screen and normal screen) it is ok. No problem.
In my device, the "emergence" (not sur about this word) of my view are "weird".

For exemple, my tableView will first appear at the top of the screen and, only one little 0,1 seconde later, position themself correctly (just below the previous view).

So, i have like a "flash" when i open my window, and it's not cool visualy.

Here is my view.js : http://sebastiendelrue.fr/appli/Resources/views/servimovie.js
The currentWindow is created with a vertical layout.

I think, maybe i am wrong with my way of use the layout, top, bottom, etc !

    Using vertical layout will cause "flicker" most of the time, because it recalculates everything once all the elements have been loaded.

    You don't see this "flicker" on the simulator because the Mac's processor is a lot more powerful than any mobile devices out there.

    Knowing that, my best advice would be to get rid of the vertical layout and to position your components using precise coordinates. Since you already know the size of your components, There is little value in using a layout manager.
