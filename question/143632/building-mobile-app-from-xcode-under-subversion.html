Building mobile app from xcode under subversion

Hi All

I am having trouble building my app in Xcode, when I put it under version control (using Versions on OSX). Everything works fine when launching the simulator from Titanium and all. But when I want to compile and sign my app using Xcode the problems begin (but only under SVN version control).

Xcode aborts the build process with the following message:

[ERROR] While reading /Users/ [some long local path] /._startup.png pngcrush caught libpng error

I don't have an image named "._startup.png", I do however have one named "startup.png". So that made me look in the .svn/entries file located next to my image, and that has the string "._startup.png" in it.

It looks like on of the build scripts copies the Resources directory into Xcodes compilation and somehow gets the .svn files mixed up with actual image files?

My question is; How do I solve this?
