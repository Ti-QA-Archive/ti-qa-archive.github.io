getting label height at runtime with Ti.UI.SIZE

Hi everybody,
the new 2.x Ti SDKs are driving me nuts. I have a lot of labels with an 'auto' height property, now migrated to 'Ti.UI.SIZE' after the upgrading to Ti SDK 2.0.2GA and 2.1.0GA.

The issue for iOS is that the label.height property at runtime is always set to "SIZE" and NOT to its numeric value.

All the solutions I red so far do not works. Even the label.toImage.size() trick does not returns correct value all the time (it depends upon the label content, carriage return chars and so on…).

It's not a timing problem either, so trying to read the height property after a timeout like somebody suggested gives always the "SIZE" hated value.

Does anybody have a solution that really works?

Thanks

Alessandro

    • "postlayout" seems to be inconsistent, see this issue that I reported: https://jira.appcelerator.org/browse/TC-1037

      A workaround can be used. For Android, listening just to window's "postlayout" event should be enough, for iOS, "postlayout" event should be removed once the values are read.

      — commented July 4th 2012 by Ivan Škugor

    • Hi Ivan,

      postlayout seems to work in the end.

      The right code is finally this one:

      label.addEventListener('postlayout', function(e) {
        var label_height = e.source.rect.height;
        alert(label_height);
    });

      Thank a lot

      Alessandro
