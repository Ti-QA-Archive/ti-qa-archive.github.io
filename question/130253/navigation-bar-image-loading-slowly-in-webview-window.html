Navigation bar image loading slowly in webview window

Greetings,

My application has four tabs, each one with a window attached to it. The navigation bar in each of that windows is a custom image. I am loading the images when calling the createWindow function as specified below: 

var dashboardWindow = Titanium.UI.createWindow({
    title : "",
    barImage: "/images/application_barImage.png",
    backgroundImage: "/images/emptyBackgroundImage.png"
});

The image is loading just fine in all of the windows, except for one, where I load a webview from a local URL. In this case, the navigation bar image loads but it takes a lot of time. Before the image loads, the default blue navigation bar appears. I tried to make an easy example to show some code but it appears to work fine. I guess the problem might be my webview being too heavy. I create the webview only after all the other components are created but the problem persists.

I need to find a workaround because the webview is very important and is the first screen in the application.

Titanium Studio: 1.0.7.201112281340
SDK Version: 1.8.0.1

  • Well, the code is here but I think it will not help you understand what was going wrong. It is working fine because the creation of the webview is made in the focus event as Sindre suggested. If I take the code out of that event, the webview flickers when loading and the navigation bar image takes a lot of time to appear. So, putting the code into the focus event is the solution and it was Sindre Sorhus who gave it.
