TableView filterAttribute and SearchBar change event

Is TableView filter fired before SearchBar "change" event attached to TableView?

I'm creating table rows in SearchBar "change" event dynamically and using TableViewRow filter attribute properly. But TableView does not show anything.

Here is the quick sample of what I did:

var search = Ti.UI.createSearchBar({ ... });
var table  = Ti.UI.createTableView({ filterAttribute: 'test', search: search, ... });

search.addEventListener('change', function(e) {
    // getting row data from database filtering with e.value
    var row = Ti.UI.createTableViewRow({ test: 'some text', .... });
    data.push(row);
    table.setData(data);
});

If I press arrow keys (up/down/left/right) after writing something in SearchBar on simulator I get results.
