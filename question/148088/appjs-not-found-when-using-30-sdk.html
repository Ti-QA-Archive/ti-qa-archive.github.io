app.js not found when using 3.0 SDK

On my iPhone, whenever I compile apps (even after clearing out the build folder), whenever I try and run an iOS app in the simulator or device, I get the red screen that says app.js not found. If i change it to a lower sdk, it works fine.

Ideas?

  • Hi

    This is a known error with 3.0.0. The reason it is happening is because you have an error in your Javascript file(s) that is preventing the minify process that occurs during the build process. 3.0.0 should fail back to the non-mini field code but doesn't.

    You have a couple of choices;

    • Find the error best plan
    • use a newer version, it will fail back

    The most common reason for the error is a trailing comma in a property declaration. Example

    var img = Ti.UI.createImageView({
  height:Ti.UI.SIZE,
  image: 'test.png',
  width:Ti.UI.SIZE, // trailing comma
});

    Start with app.js and work through all. Consider checking your code with something like jslint.com.

    Good luck, it can be sorted and you will succeed.
