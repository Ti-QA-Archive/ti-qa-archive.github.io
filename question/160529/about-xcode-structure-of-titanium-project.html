About xcode structure of titanium project

I want to know in XCode Project Structure, I am unable to see any View Controller or UI classes, after step-by-step debugging of Titanium generated XCode project, I found that
Titanium stores all the required files (HTML /CSS / Images) in application main bundled.

So, from the above observation, I have some queries.

  1. Can we forcefully store all these required files in somewhere else (not in main bundle)?
  2. Why Titanium application generates 3 different application bundled for iPhone, iPad and Universal devices?

It will be a great help if anybody could make me understand about this. Thanks.

  • Titanium code is not cross-compiled to Objective C code. You will not find the same sort of project structure with a Titanium project as you would with one you create yourself.

    No, you cannot "forefully store those required files somewhere else"

    I'm not sure why Ti generates three bundles as you describe.

    By the way, there's no HTML/CSS in a typical Titanium project. Titanium is not PhoneGap.
