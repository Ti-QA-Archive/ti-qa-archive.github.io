setTimeout and Window Context

Hi,

I'm dealing with a setTimeout to lock / unlock a password checking in my app, however, it seems that setTimeout stick to the window context is has been launch.

Example: Win 1 > Win 2 : Launch setTimeout > Win 1: the timer is lost :(

Any one as a clue on this ?

Thanks :)

  • Here is a working example. How does your code differ?

    ####app.js

    
var win1 = Ti.UI.createWindow({
    backgroundColor : 'blue'
});
win1.open();
var win2 = Ti.UI.createWindow({
    backgroundColor : 'yellow'
});
var button1 = Ti.UI.createButton({
    title : 'Set Timeout'
});
win2.add(button1);
button1.addEventListener('click', function() {
    setTimeout(function() {
        alert('Called using setTimeout');
    }, 2000);
    win2.close();
    delete (win2);
});
win2.open();
