height in Ti.UI.SIZE on label much slow then height in fixed units

This is the case: have 6 boxes which contains several labels and image. If i set height of labels to Ti.UI.SIZE after opening controller user have to wait for 2 or 3 seconds before interaction with page is possible(i mean scrolling page, calling click handlers and so on)

If I set height of labels to fixed units for example: "80dip" everything happens faster and user can interact with page in less than a second.

Any ideas why this happens ? It is much more convinient to use Ti.UI.SIZE, also setting labels in fixed height do not allow me to achieve desired layout

  • Hi Georgi

    Yes this will always be the case.

    If you set a fixed height there is no requirement for the height to be calculated, so will always be faster than using SIZE.

    SIZE is basically an instruction to set the height automatically, so various elements needs to be used to calculate that height at runtime.

    No real way around it, except use a height and changed it after rendering.

    T.
