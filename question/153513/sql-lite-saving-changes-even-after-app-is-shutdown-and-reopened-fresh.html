SQL Lite saving changes even after app is shutdown and reopened fresh...

Hi there,

I have my app writing to a SQL Lite database as the user interacts with the app.

The changes remain in the SQL Lite database when the app is running in the background but a full close, reset and open looses the changes and reverts to the original database.

Is it possible to keep the changes if the user shuts the app down and then re-opens?
If not what other options are there to consider?
If possible I want to avoid saving anything remotely.

Thanks for any pointers,
Dave

    Dave, sqllite is persistent and you won't lose data you save between instances of using the app. Could it be that you are testing a new installation each time? Or could you be deleting the database or forcing a new database install when you close or launch your app?
