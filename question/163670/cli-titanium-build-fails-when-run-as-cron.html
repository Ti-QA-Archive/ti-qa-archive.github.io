CLI titanium build fails when run as cron

CLI version 3.2.1, Titanium SDK version 3.2.2.GA, MAC OSX 10.9.2

The following commands work fine when run in command line but give an error when run as a cron job. Does anybody know what could be causing this? I have removed the actual paths and passwords.

Android Build

titanium build --build-only --platform android --target dist-playstore --project-dir [dir] --android-sdk [sdk] --alias [alias] --keystore [keystore] --output-dir [out_dir] --store-password [pass] --key-password [pass]

The above returns the following error:

###[ERROR] : Invalid "–key-password" value "[pass]"

iOS Build

titanium build --platform ios --build-only --device-family universal --target dist-appstore --project-dir [dir] --distribution-name [dist] --pp-uuid [uuid]

The above returns the following error:

###[ERROR] : Invalid "–distribution-name" value "[dist]"

Again both of these commands when run through command line compile the apps perfectly but when run as a cron job (i have to automate this process on a mac) they fail with the errors given above. Any help would be greatly appreciated

    • I was running the cronjob as the same user as I log in with. I have changed the way it runs as a task on the mac and this has fixed the issue. I am now using launchctl instead of cron. Here are some links that show how to use launchctl on the mac.

      http://www.davidlanier.com/story/cron-and-launchctl-on-mac-os-x-105-leopard
      http://arunxjacob.wordpress.com/2008/08/28/mac-launchd-and-launchctl-the-osx-alternative-to-cron/
      https://developer.apple.com/library/mac/documentation/Darwin/Reference/ManPages/man1/launchctl.1.html#//apple_ref/doc/man/1/launchctl

      Though most likely it had to do with the paths not set up correctly, here is what i did to set up the paths on mac:

      # fix path issues for launchctl, will load the following path for all users on startup
sudo vi /etc/launchd.conf
# paste the following into the launchd.conf file
setenv PATH /opt/local/bin:/opt/local/sbin:/usr/local/bin:/usr/bin:/bin:/usr/sbin:/sbin:/usr/local/bin
# save the file and run the following command to load the path
grep -E "^setenv" /etc/launchd.conf | xargs -t -L 1 launchctl
