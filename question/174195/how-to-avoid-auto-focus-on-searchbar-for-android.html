How to avoid auto-focus on SearchBar for Android

Hi,

I have a searchBar added to a TableView which is placed inside a View as below :

<Alloy>
    <View id="centerView">
            <TableView id="mainTable" onClick="openDetail">
                <SearchBar id="searchBar" class="searchBar"/>
            </TableView>
        </View>
</Alloy>

My problem is that when the View is displayed for the first time, the 'searchBar' has auto-focus and the soft keyboard is displayed. I don't want this to happen, I want the searchBar to get focus only when a user clicks it and then the soft keyboard is displayed. How can I achieve this?

  1. I have tried to use the searchBar.blur() function but it did not work.

  2. I have also set "softKeyboardOnFocus" in the .tss file for the searchBar as below : 

    softKeyboardOnFocus : Ti.UI.Android.SOFT_KEYBOARD_HIDE_ON_FOCUS,

    and when the searchBar gets focus to show the softkeyboard as below - 

$.searchBar.addEventListener('touchStart', function(e) {
        $.searchBar.softKeyboardOnFocus = Ti.UI.Android.SOFT_KEYBOARD_SHOW_ON_FOCUS;
        $.searchBar.focus();
    });

Other details :
Titanium CLI version 3.2.3, Titanium SDK version 3.2.3.GA, Alloy version 1.3.1
Developing mobile app for android.

Really appreciate any help on this as I can't seem to work this out since a long time.

Many thanks.

  • Hi

    This is a very old cheat I used to avoid the first field focus Android does - I have not tried it recently but maybe it will help here;

    The idea is to add a TextField that will should have gained the focus, it is placed before any other TextFields - but it is hidden. The important property was to say enabled: false. Then user select the search when they are ready.

    Again - I have not tried this in ages - but worth a try;

    <Alloy>
    <View id="centerView">
        <TextField id="txthack" class="txthack" />
        <TableView id="mainTable" onClick="openDetail">
            <SearchBar id="searchBar" class="searchBar"/>
       </TableView>
    </View>
</Alloy>

    TSS

    ".txthack": {
    enabled: false,
    height: 0,
    touchEnabled: false
    visible: false
}

    See what happens.
