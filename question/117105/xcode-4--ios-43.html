  • I had some hard time after upgrading because the global setting of the build path led to a build error in Titanium Developer, which expects the output path to be under the project's root, and all settings in the "Locations" panel of Xcode preferences didn't help.

    Finally the solution of amattn (the 2nd at the time of this writing) in http://stackoverflow.com/questions/5599587/strange-behavior-of-xcode-build-products-path-under-build-locations helped me out by getting rid of an (obsolete?) SYMROOT entry under the IDEApplicationwideBuildSettings key in ~/Library/Preferences/com.apple.dt.Xcode.plist, which probably stemmed from the migration of old preferences. (If you edit this file from within Xcode make sure to close all projects first.)
