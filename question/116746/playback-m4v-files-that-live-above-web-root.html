Playback m4v files that live above web root

This is directly related to Ti, but I know this forum has some pretty knowledgeable people.

I have some m4v files that live above web root. The reason is I don't want people that aren't logged in able to download the videos if the link was acquired. The way I feed them to the user when they are logged in is, I have a file called fileserve.php.

<?php
include("config.php"); // just includes session_start and db connection
if ($_SESSION['user']['authed'] == true)
{
    session_write_close();
    $id         = stripslashes(mysql_real_escape_string($_GET['id']));
    $query         = mysql_query("SELECT filename FROM episodes WHERE id = '" . $id . "'");
    $num        = mysql_num_rows($query);

    if ($num == 1)
    {
        $row         = mysql_fetch_array($query);
        $filename     = "../../media/" . $row['filename'];

        header('Accept-Ranges: bytes');
        header('Content-Type: video/x-m4v');
        header('Content-Length: ' . filesize($filename));

        ob_clean();
        flush();
        readfile($filename);
        exit;
    }
    else
    {
        echo "invalid video";
    }
}
else
{
    header("Location: /");
}
?>

When I view it on the ipad and iphone, I get the big play button with a cross through it and an alert saying "Cannot Play Movie: The server is not correctly configured".

What do I have to do to fix this?
