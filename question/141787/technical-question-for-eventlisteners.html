Technical question for eventListeners

Hey there,

I'm currently developing an app which has a kind of constructor-function which runs everytime the app is new opened (so not only one time, but on every startup). But now i have an eventListener which reacts to the push-notifications-callback and then runs the same function.

Code example :

var App = function() {
   var _this = this;
   this.init = function() {
      this.do(args);

      Ti.App.addEventListener("push:callback", function(e) {
          _this.do(e.args);
      });
   }

   this.do = function(args) {
      alert("function fired");
   }
}

var _app = new App();
_app.init();

So here is my question: How can i react, that the running of the do()-function is ignored at the startup when the startup is given by the push-callback?

Thank you!

  • This doesn't have much to do with event listeners as basic order of operation. Try this:

    var doRan = false;

this.do = function(args) {
  if (doRan) {
    return;
  }
  alert("function called");
  doRan = true;
}
