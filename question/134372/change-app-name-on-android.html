  • This is very simple. The method referenced by Minh Nguyen may have worked in the past and may still work today; but it didn't work for me. Besides, creating and maintaining a custom AndroidManifest.xml file is a painful task. You should do it only when absolutely unavoidable.

    Here is what you need to do to localize your App name for Android (assuming that English and Français/French are the languages of interest).

    I have tried this method successfuly on Titanium SDK 3.1.3.GA and Titanium SDK 3.2.0.GA

    1. Create a folder called platform along side your app folder so that the structure will look similar to the following. Then create a folder called android in the platform folder.
        app/
    platform/
    modules/
    ...
    plugins/
    ...
    1. In the newly plateform/android folder, create a folder for each language and name them values-en for English and values-fr for Français … then create a file called strings.xml in each of those folders (the name of the file itself doesn't matter since any xml file will be parsed for Android resources. That's an android thing). Now, the structure will look like …
        app/
    platform/
        android/
            values-en/
                strings.xml
            values-fr/
                strings.xml
    modules/
    ...
    plugins/
    ...
    1. Assuming that your App's name is My Application in English and Mon Application in French, place the following in platform/android/values-en/strings.xml …
    <?xml version="1.0" encoding="UTF-8"?>
<resources>
    <string name="appname">My Application</string>
</resources>

    and the following code in platform/android/values-fr/strings.xml

    <?xml version="1.0" encoding="UTF-8"?>
<resources>
    <string name="appname">Mon Application</string>
</resources>
    1. Inside tiapp.xml, make sure that the Android resource called appname is being used for the label of the application. The label is the human readable name of your App. The actual machine name is quite different something. For that, you need to locate the Android section of tiapp.xml. You may need to add it if it is not in there already. It's a section that will roughly look like the following …
      <android xmlns:android="http://schemas.android.com/apk/res/android">
 <manifest>
     <application ... android:label="@string/appname" ... />
 </manifest>
</android>

    For more information, read these too pages: Maintaining a Custom AndroidManifest.xml and tiapp.xml and timodule.xml Reference

    After completing step 4, the name of your App will be localized in all the languages that you will have configured.

    As a side note, the app name in /i18n/en/app.xml and /i18n/fr/app.xml does not work for Android. It's worked only for the iOS version of the App.

    Cheers
