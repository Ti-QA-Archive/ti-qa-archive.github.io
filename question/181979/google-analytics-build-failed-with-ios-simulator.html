Google Analytics Build Failed with iOS Simulator

Is there an issue with using MattTuttle's analytics.google module downloaded from the Appcelerator Marketplace with the iOS Simulator? I successfully added the modules to the {project_loc}/modules path as instructed and updated my tiapp.xml file accordingly for <modules/>. Then I cleaned the project and rebuilt under several of the iOS Simulators, such as iPhone 5s or iPhone 6, and I got the following build error at the console:

[ERROR] :  2015-04-12 18:53:14.828 xcodebuild[13926:4872472]  DeveloperPortal: Using pre-existing current store at URL (file:///Users/{my_user}/Library/Developer/Xcode/DeveloperPortal%206.3.db).
[ERROR] :  ** BUILD FAILED **
[ERROR] :  The following build commands failed:
[ERROR] :      Ld build/{app_name}.build/Debug-iphonesimulator/{app_name}-universal.build/Objects-normal/i386/{app_name} normal i386
[ERROR] :  (1 failure)

where {my_user} and {app_name} are private data I wish not to share here.

I am running:

Mac OS X 10.10.2 64-bit
Titanium Studio, build: 3.4.1.201410281727
Ti SDK 3.5.1 GA
Node.JS version 0.10.13.

This looks similar to the following open JIRA ticket: https://jira.appcelerator.org/browse/TIMOB-17694. But the thing is I don't have a folder named after my app in the Resources folder or anywhere else. I only have {app_name} as the name of the root project folder, so I don't know what gives.

I doubt the problem is lack of 64-bit support in the module because I see the GitHub README for the module actually says it has 64-bit support: https://github.com/Sitata/titanium-google-analytics

This is related to the following Q&A posting, but of course my case doesn't fit the solution provided: https://developer.appcelerator.com/question/131687/project-cant-run-on-simulator
