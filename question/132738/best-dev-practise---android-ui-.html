Best Dev Practise - Android UI ?

Hi there,

I have been developing Titanium apps for quite some time now and have about 10 apps in both app stores.
Throughout the years I discovered that I always start by working on the iPhone version due to the fast compile time to the simulator. When starting working on the equivalent Android app, I find myself struggling to get the Android layout of views to be placed like the iPhone.

Even though everything works great in the iPhone version, the view parameters specified are not correspondent in a similar fashion. How come?
I end up spending must time correcting layout, so it looks correct on android.

Let me give you an example:

var view = Ti.UI.createView({left:"10dp", right:"10dp", height:"auto", backgroundImage:”/images/bg.png”});
var label = Ti.UI.createLabel({text:"hello world", width:"auto", height:"auto"});
view.add(label);

This is very basic code, but it will not look the same on iPhone and Android.

I’m looking on great "Best Practise" tips on layout - so I due to need to spend must of my time figuring out Android layouts.

1. What do to guys do to make your layout 100% cross platform?

2. Any "no goes" I should stay away from?

3. Any clever UI framework to unify iPhone/android layout for appcelerator?

  • Hej Mads :)

    Unlike you, I start out on Android … and sometimes it IS a pain. When I say sometimes I mean oh … just about 85% of the time :)

    I'm not too experienced wth iPhone yet, but I've noticed something that seems to work for me.

    In the android section of tiapp.xml put this:

    <supports-screens android:anyDensity="false"
android:installLocation="preferExternal"/>

    (the install-location bit is not needed, but it's a nice tip on the side)

    And then use percentages of Ti.Platform.displayCaps.platformWidth and Ti.Platform.displayCaps.platformHeight

    Like this 

    Ti.UI.createView({
    top : (Ti.Platform.displayCaps.platformHeight / 100) * 10
    // For some reason using real % doesn't work as well
    width : '80%'
})

    Of course you want to wrap the Ti.Platform … etc … into an object or a function so that you don't have to write that long line all the time.

    This is my best bid at the moment, others might have better ways of doing it :)
