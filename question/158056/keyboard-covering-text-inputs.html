Keyboard covering text inputs

After the upgrade to iOS7 I noticed the system keyboard covers the text inputs in my webview instead of scrolling the content up to put the keyboard below them as it did in iOS6. Any ideas how to workaround this? My users can't see what they're typing.

This is a mobile application that is being developed in Titanium Studio on OS X 10.8.5 using Titanium SDK 3.1.3 for IOS7. I am testing on an iPhone 4s.

  • Adding "height=device-height" to the viewport meta tag did the trick, strange it was never needed before. Hope this saves someone the headache.
