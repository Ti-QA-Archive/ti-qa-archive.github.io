GPS. Wrong coordinates. Twin Falls, United States.

Hello,

Titanium SDK version: 1.6.2
iPhone Device family: iphone
iPhone SDK version: 4.2

I am developing an iOS app that uses GPS data on a map. There is a strange problem occuring. The first GPS data registered are always Twin Falls, United States. I live in Sweden so this is totally wrong. Is this a default behavior of some sort? When I get the GPS coordinates the next time it is correct.

Does anyone have a solution for this?

  • I think the answer is that GPS is a little awkward. It is rare to get the correct co-ordinates first time.

    I resolved this by calling the get co-ordinates function 2/3 times before setting the event listener for position change. Always waiting for at least the second call to return before continuing.

    Set a time out just incase the call hangs, which it can do.

    Hope this helps

  • This is the way the GPS works. It needs a lot of time to "cold start" so it takes the data it has cached form a previous usage.
    I use the location event started with a very small distance filter, then after I get the location I close this event (of course if I do not need it ).
    Keep in mind that everything is async and you have to let the user know that the app is working or it will think it's a bug :)
