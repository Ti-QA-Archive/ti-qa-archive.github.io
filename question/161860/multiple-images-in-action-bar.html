Multiple images in action bar

can i add multiple images in action bar for android ? In iPhone, i can set images as leftNavButton, roghtNavButton of window property and in the middle of navigation bar i can set image by titleControl. is it possible to set images in the middle and right of the action bar for android ? It seems android only allow left image as appicon in action bar. how can i do that for android ? and also want know, if i can set images in those position then can i fire a eventListener for those image click ? please help me.

    Hi

    The ActionBar is much more than a bar with stuff - it changes what and how it displays depending several factors. The density of the device, dimensions, if there are tabs and device orientation all determine how the ActionBar chooses to display the information you have asked it to show.

    In order to answer your question you really need to know the make-up and logic of the ActionBar.

    ActionBar has zones where things can appear, in simple terms (from left to right) they are;

    • Small icon to represent;
      • this will open menu from the side
      • this will close this window (essentially similar but not identical the iOS back button)
      • nothing displayed
    • App icon - you can change this but normally it is to represent the branding of the app, commonly (but not required) it is a smaller version of the app icon
    • Title or Spinner
      • This will normally show the title of the app or whatver you have set the title to display as
      • A Spinner - If the app has tabs and the orientation, density or pixel dimension determine a need to show the tabs inside the ActionBar - it will - you do not determine this - it is automatic
    • Menu Icons and/or text
      • This is the right 50% of the total action bar - no additional space can be used
      • Whilst you choose the menu items that appear here - you have limited control in how they should appear and again - the orientation, density or pixel dimension determine a need to show them differently
      • All menu items should have an icon and optionally text
      • You can choose if a menu item should appear as text - but for the same reasons above it may be displayed as just an icon
      • If there are too many menu items the furthest right icon will display as an overflow icon - again this is automatic and you do not control this from happening when it "has" to, but you can control items appearing here rather than the main bar - but again the action bar may add other menu items from the main bar if there is insufficient space to display them.
      • If a menu item appears in the overflow drop down it should have both an icon an text when you define it - a good rule of thumb is always set both

    So to answer your question;

    • Left icon: you can set and change the left icon
    • Right icon: you take mimic the iOS right button by creating a single menu item and setting it to display as icon only - this will persist as an icon and not create an overflow menu as there are no situations (that I know of) that would force a layout change automatically as no trigger will have been even close to firing.
    • Center - no, this is either a title or a spinner (if you have tabs and if it determine they should appear here)
    • Events;
      • Left Icon - yes onHomeIconItemSelected
      • Right icon (and any menu items) - yes Android.Menu

    I hope this helps.
