[Doubt]Auto size views containing text.

Hello everyone! I have some little doubts about graphic design :)

The first one:

I want to create a text (with a label). That text I want it to insert in a view and after doing that I need te view to adjust its size to fit with the text and make it look nice, but I have no idea how to do it since the layout system has been changed…

For example:
I create a view

var view = Ti.UI.createView({});

then a label

var label = Ti.UI.createLabel({
    text: 'This is some epic text. So epic that will make the view fit its size :)',
    width: Ti.UI.FILL,
    height: Ti.UI.FILL,
    font: {
        fontSize: '16dp',
    },
});

view.add(label);

And I want the label to change its height at least to make the text fill the entire view. I've tried in iPhone with some 'auto' properties in the view but in Android is a little tricky and I wonder if there is possible to make this generic.

Thanks for your time and code strong!

    var vectorial = Ti.UI.createView({

    backgroundColor: '#AAA'

var label = Ti.UI.createLabel({
    text: 'random things are random! random things are random! random things are random! random things are random! random things are random! '+
        'random things are random! random things are random! random things are random! random things are random! random things are random! '+
        'random things are random! random things are random! random things are random!',
    font: {
        fontSize: '12dp',
    },
    height: 'auto',
    color: '#000',
    top: 0,
});

vectorial.add(label);
win.add(vectorial);

    As promised, there is a little workaround for get the views height and we are going to need the Ti.Blob object.
    As you can possibly imagine, the Ti.Blob has its own height and width properties. So the idea is to turn the desired view into a Ti.Blob.

    How can this be possible? You have a method in view that turns it into a picture, represented by a Ti.Blob.
    That TiBlob will be generated with his own height and width properties, but when you see both of then, there wont be 'size' or 'fill' values, instead, you will receive them as numeric values :)

    win.addEventListener('open', function(){
    alert(view.toImage().height);
});

    Obviously, it's better to save that value to make some nice variable UIs.
