Validate page before closing

I have a preferences page and I want to validate the info before the user can navigate away from the page. I want to alert the user if they did not enter information into a field before leaving the page. I know that I can use a close event listener, but that happens after the page has already closed. I don't want the user to have to navigate back to the page if there is a problem since that is a horrible design. I also know that I can use a button to handle this, but I'd rather the user be able to use the built in navigation. I am using iOS and Andriod, and I have a NavigationWindow in iOS with the built in back button to deal with. I can do it on Android because I can catch the Android back button. I am using iOS 7.1, Android 4.2, OSX 10.9, Studio 3.3, Titanium 3.3 and Alloy 1.4.

    Logan,

    As you can see, you cannot 'block' the back button. In the case where you need to validate, open a new window instead of adding it to your navWindow hierarchy. That is the way I do it with all my apps.

    Ray
