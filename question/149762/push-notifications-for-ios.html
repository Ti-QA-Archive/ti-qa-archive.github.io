Push Notifications for iOS

Hi,

Our client is working with a Software Company that specialises in Push Notifications. To integrate the 3rd parties push notification library we have had to edit our Titanium projects (iOS version) in Xcode, make the necessary code amendment to the Tiapp.mm file (literally 2 lines of code), and add in the 2 files supplied to us, build and that's it done.

The only snag is that if we re-build the Titanium project i.e. a clean build, all the custom code changes are lost, and we have to re-apply the push code again.

I'm guessing this is the way it has to be, and its just a post-build exercise we will have to manually do after re-building our Titanium project.

Has anyone else had experience of make Xcode mod's to their Titanium project? Is this bad practice for a Titanium project? I would welcome any recommendations on this subject.

Thanks,

    You guys need to make an iOS module for the stuff you are doing natively in objective-c. This way, in your Titanium project, you just need to require your module and never have to worry about losing the code. I highly recommend creating a module over modifying the Xcode project each time you want to deploy. This will also speed up deployment time, because with a module, you will be able to distribute from Titanium Studio, instead of taking the extra step of going into Xcode and modifying code.
