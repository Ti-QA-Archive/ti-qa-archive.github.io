TextField has bottom padding that shouldn't be there

I'm creating a simple text box that always has this bottom padding that I don't want. I create the text box like this:

var bodyHeight = Math.ceil(Ti.Platform.displayCaps.platformHeight * 0.905);
label = Titanium.UI.createLabel({
    top: '0',
    left: '20dip',
    right: '20dip',
    font: {fontFamily: defaultFont.fontFamily, fontSize: '16dip', fontWeight: 'bold', fontStyle: 'normal'},
    textAlign: 'left',
    color: '#fff',
    text: 'Title'
})

var textBox = Titanium.UI.createTextField({
    top: Math.floor(bodyHeight * .05),
    bottom: '6dip',
    left: '12dip',
    right: '12dip',
    paddingLeft: '10dip',
    font: {fontFamily: defaultFont.fontFamily, fontSize: '16dip'},
    backgroundImage: 'Images/NewSkin/category_bg.png',
    color: '#46c2e7'
});

row = Titanium.UI.createView({
    height: Math.floor(bodyHeight * 0.15),
    width: '100%',
    backgroundColor: 'transparent',
    data: {title: 'Title', type: 'text', label: label, textBox: textBox}
});

row.add(label);
row.add(textBox);

...

Then the row get's added to the current window. It displays but this is what I see:

What am I doing wrong?

  • Hi, I've also experienced this and what fixed the issue for me was the mix of @Vishal's answer and @Malcolm's comment at TextField top/bottom padding, namely setting:

    <supports-screens android:anyDensity="false"/>

    and

    <property name="ti.ui.defaultunit">dp</property>

    on tiapp.xml.

    A lot of us would have supports-screens anyDensity="false" because there was a bug on the documentation as pointed out in TIMOB-13567.
