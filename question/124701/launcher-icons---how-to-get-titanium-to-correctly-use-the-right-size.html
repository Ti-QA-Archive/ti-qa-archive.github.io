Launcher Icons - How to get Titanium to correctly use the right size

We have developed 3 sizes of our launcher icon (36, 48, and 72), per the Android guidelines. I have placed them in the low, medium, and high subdirectories under \Resources\android\images\ respectively. Titanium refuses to build my app unless I place one of these in the Resources directory directly. In tiapp.xml, I have <icon>MyAppIcon.png</icon>.

What is the correct way to set this up so that the correct icon is used on different density devices?

  • You will need to put them in the platform/android/res/drawable/drawable-DENSITY folders. (e.g. drawable-mdpi, drawable-ldpi, drawable-hdpi). Files under images/android/[low|medium|high] have their names mangled to prevent collisions and to remove folders if any appear. Which works fine when accessed from a Titanium API, but not from the Android OS.

      1. The name of the launcher icons in platform/android/res/drawable… must be appicon.png
      2. Whatever the name of your launcher icon is listed as in TiApp.xml, a file with that name must exist in Resources\android (even though it is not used)
