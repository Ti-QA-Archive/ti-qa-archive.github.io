registerForPushNotifications at every startup ?

Hey everybody,

I have a simple question regarding the Ti.Network.registerForPushNotifications usage.

Should I call the method at EVERY startup or should I store (in an app propertie for instance) the fact that my registration succeeded ? This last solution could create issues if a device is unregistered (remotely) so any advice would be greatly appreciated.

FYI, I'm using the Urban Airship way to implement the Push Notifications (and it works great).

Thanks.

    You should call it at every startup, as that is how it is done in a native iOS app. It is basically reaffirming to the Apple Push Notification Server that yes this app is continuing to be registered for notifications. You don't have to store any property to remember that it succeeded.
