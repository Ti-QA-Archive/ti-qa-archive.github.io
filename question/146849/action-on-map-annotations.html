    Hi

    Unfortunately not, this is a design feature of iOS UI and the behaviour expected by the user - which is the main reason why it is not possible.

    Whilst the new Google Maps has a different approach it is not part of the iOS UI guidelines for maps.

    Even if this was not true, as you have found the click event does not fire unless you have at least one property that triggers the annotation to appear - as the pin is not actually the annotation - just a position marker for one.

    You have used an event on the annotation rather than the map, both are valid ways - I prefer the map method as I only need one event. However having just checked both operate under the same rules - looking for a way out for you.
