Remote images in backgroundImage property

Because of scaling issues I want to use the backgroundImage property in an imageview instead of the image property, however I'm using remote images and it seems backgroundImage can't take remote images. Is this true or has anyone got this to work?

  • I was working on this just today as well. At first I just had them stored locally, but I realized the images (Both regular and @2x) were causing the app to be quite large, so I dumped them onto my WebServer and tried setting them as backgroundImages, but it didn't work.

    What I did was simply add an ImageView to the row (Same height & width as the row), then set the zIndex of the Image to 1 and the zIndex of all the other content that is supposed to be ontop of the image to 2.

    Seemed to work quite good :) I also created a 1x1 image and set the defaultImage of the ImageView to it so that when it loads it doesn't have that ugly grey default "no image" image that Apple supplies.

    Note that I do not believe resizing of a remote image is very well supported. It seems I can downsize the image, but upsizing it is quite difficult and annoying. Also, if you need to figure out whether to use @2x or not, use the Ti.Platform.DisplayCaps.density API. Retina = "high", regular = "medium".
