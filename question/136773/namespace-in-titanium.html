  • Hi Prabu,

    Namespaces in javascript help protect your code from getting confused with other peoples code by "protecting" it in its own namespace. Without using a namespace your javascript code ends up in whats called the "Global Namespace" and can easily be overwritten by another piece of javascript elsewhere in the project.

    Consider this function.

    function SayHello(){
    Alert('Hello');
}

    Using this in my own project by itself is fine. I can call

    SayHello();

    And all is good. But just suppose I find a really nice piece of javascript written by someone else that I want to use in my project so I include it. Part of this really nice piece of javascript contains a function;

    function SayHello(){
    Alert('GoodBye!!!!!!');
}

    Now, depending on where I include this new piece of javascript (either before my own, or after it). When I now execute;

    SayHello();

    I might get the result "Hello" or "Goodbye!!!!!"

    So, if I namespace my function eg

    
mynamespace.SayHello = function(){
    alert('Hello');
}

    I can be more confident my function is not going to get overwritten by someone elses.

    CommonJS does not stop you from namspacing your functions, but because it only imports the function names declared in the export section it hides any namespace declared.

    So, I can still do this;

    in a new file (myhellomodule.js)

    exports.MySayHellofunction = function SayHello(){
    alert('Hello');
}

    I can just do this;

    var sayhello = require('myhellomodule').MySayHelloFunction;
sayhello();
