iOS publish build not copying resource images

I am using titanium studio 3.0.1.201212181159 and I am building a universal iOS app for iphone and ipad. When I build the app to publish to the appstore, the build folder does not contain any of the images I've included in Resources/iphone folder. I only get stock appcelerator images in the build folder. This causes xcode validator to fail because I don't have separate icon.png and icon-72.png files.

I've read previous messages about errors with older versions of appcelerator needing to include a info.plist file that points to the icon files. I have tried including that in my app but I get the some lack of images in the build folder. This is after cleaning and removing and re-adding my images to appcelerator multiple times.

  • There were bugs in Titanium SDK 3.0.0.GA regarding the copying of the custom splash screens. This has all been fixed in 3.0.2.GA. I recommend updating to the latest version.

  • I'm coming back to answer my own question. My solution was to open the .xcodeproj file from the build directory of my app. From there, I dragged in the respective images to the image fields on the summary page. I still don't know why I this isn't being done from Titanium, but it finally worked. I also had to change the deployment target on each app's build settings in xcode to 4.3 or I would get errors about arm6.
