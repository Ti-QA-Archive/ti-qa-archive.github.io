Extending base objects. What is the current best practice.

Ok, I'm pretty new to titanium and an not using Alloy.

I've been researching for the last couple of days the best way to extend base ui objects (e.g., make a custom button based off the view class).

So far it appears that the community is somewhat confused on this topic, largely because Appcelerator themselves have been changing their answers on this over time.

Here's a blog post talking on two of the most common options, a wrapper class and parasitic inheritance: http://smorgasbork.com/index.php?option=com_content&view=article&id=133

There seems to be much disagreement on exactly what method you should use, many saying that parasitic inheritance is fraught with issues because you are extending native bridges, while others say you can use them, just be aware of the naming limitations.

This is represented in this oft-referenced Q&A post here: http://developer.appcelerator.com/question/137094/parasitic-inher%20itance-and-prox%20y-objects

In this Kevin Whinnery suggests parasitic inheritance is fine for the aforementioned reasons (and says they are working on new way of dealing with this, is this Alloy or something else???).

Meanwhile older documents addressed this issue straight on (https://wiki.appcelerator.org/display/guides/Mobile+Best+Practices) but I can't seem to find clear guidelines in the newer docs for this case (does this mean the old way is not recommended anymore?).

All this has left me in complete confusion. I need to create my own widgets based off existing widgets. Ideally this would be done through extending the older widgets somehow, but how?
