Handling a lot of annotations on a mapview

Anyone have some good solution on how to handle a lot of annotations on a mapview, let say 2000-3000, I've seen for example in Apples on Map-app they merge the the annotation pins when the map i zoomed out and splits them when you zoom, anyone managed to do something similar for Titanium?

Been looking for a module or something similar to handle this.

Annotations are representing a lot of fans spread out all over the world.

The "solution" I'm running now is a local database that gets queried with the current DeltaLongitude/DeltaLatitude and just shows the annotations for current region, but it can still be a lot of annotations and if you scroll a lot the loading times still gets long.

All suggestions are welcome.

  • A possible solution is to view the map at a given zoom level as a grid, and to aggregate all the annotations falling in each rectangle of the grid as a single annotation. As the user zooms in or out, you recompute the grid and aggregate/deaggregate the annotations for each rectangle, thus showing a reasonable number of annotations at each zoom level.

    You want these operations to be quite fast, so, depending on your requirements, you may need to reconsider your underlying model and possibly leverage the concepts from the geohash and geomodel models.
