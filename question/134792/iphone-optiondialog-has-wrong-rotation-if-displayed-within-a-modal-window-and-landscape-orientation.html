iPhone: OptionDialog has wrong rotation if displayed within a Modal Window and landscape orientation

Hi all,

I think I found a bug in the OptionDialog. If one rotates an app to landscape and will then open a ModalWindow and then an OptionDialog. The OptionDialog is displayed centered and in portrait mode instead of landscape.

Example:

var win1 = Ti.UI.createWindow();
win1.open();

setTimeout(function() {

    var win    = Ti.UI.createWindow({title: 'MyApp'});
    var button = Ti.UI.createButton({title: 'open'});

    win.open({modal: true});
    win.add(button);

    button.addEventListener('click', function () {

        var dialog = Ti.UI.createOptionDialog({title: 'test', options: ['1', '2', '3']});
        dialog.show();
    });

}, 5000);

Just put this into app.js. I just set a timeout cause you need a few seconds to rotate the device.

Steps to reproduce:

  • Open app and rotate the iPhone or Simulator to landscape (cmd + left) while the window is black.
  • After 5 seconds it'll open a Modal Window with white background and a button.
  • Open the OptionDialog by pressing the 'open' Button.

Result:

The option dialog is centered and not correct displayed. See this image: http://imageshack.us/photo/my-images/36/screenshot215d.png/

Environment:

  • Titanium Mobile 1.8.2 and 2.0.0.v20120328101644
  • Mac OS X 10.7.3
  • iOS SDK 5.1 (I installed the iOS 5.1 patch as described in your blog)
  • Titanium Studio 1.09

  • Hi Markus, this is not a bug, you need to add the orientation support at your window in order to view correctly in a landscape mode. I hope this code helps you!

    var win=Ti.UI.createWindow({title:'MyApp',orientationModes[Ti.UI.PORTRAIT,Ti.UI.LANDSCAPE_LEFT,Ti.UI.LANDSCAPE_RIGHT]});
