Titanium Module for (Accessory)Hardware

Hello everyone.

I have an app which uses the LineaPro barcode and credit card reader. LineaPro comes with it's own Objective-C API so it's very simple to use on the iPhone.

Now I want to rewrite this app with Titanium and keep using LineaPro.
Since LineaPro is an iPhone/iPod-only accessory, there is no need for it to work on the Android.

I've been doing some research on Titanium Module development but couldn't find anything about developing a module to interact with an existing (Obj-C)API for a hardware. Or maybe write my own (Obj-C)API to interact with this hardware and a Titanium Module to use this (Obj-C)API.

Is it possible?

Here's the link to LineaPro's website so you know what I'm talking about.
http://www.ipcprint.com/mobile/

    Hi Bernardo,

    I have a working enterprise grade app written in Titanium Developer for my current company. I wrote a custom LineaPRO Titanium Module for the Linea by following Appcelerator's excellent documentation on doing so.

    This is all you need.
    http://developer.appcelerator.com/doc/mobile/iphone/module_sdk

    Be sure when you build your module you are including Linea's libLineaSDK.a

    Cheers,
    Paul
