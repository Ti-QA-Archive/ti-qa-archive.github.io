    Hi Michael

    Lets see if we can explain this if not look HERE)

    A callback is a function, which is passed to another function / module to execute in that area of code.

    i.e.

    Module A.

    
var moduleB = require(moduleB);

function b () {
  // Do something within this module
}

function c() {
    moduleB.returnhere(b);
}

    Module B.

    function return here(callback) {
    if (callback) {
    callback();
    }
    }
    export returnhere = return here;

    ~~~

    This very basic example shows 2 commonJS modules interacting. Module a calls the function in module B, passing the function b. Module B can then run this function.

    The reason it is called a callback is that function b still runs within the scope of Module A, and program flow continues from this point.

    I hope this has explained it. Please ignore the node.js reference above, not a criticism just that it might be a bit confusing.

    Titanium uses commonJS, the reference to node is that it also uses commonJS. But what is commonJS and that is probably what you need to understand first.

  • In javascript functions can be passed to other functions as parameters. This means that a function defined in one file(or function) not necessarily be called in the same file rather it could be passed to some other file and called there. Such passing function is called a callback function.

    For example in your code the function GetString is defined in app.js but its not used (called) any where in app.js instead passed to another function as callback where it could be execute on call.

    Probably you should first learn the basics of javascript (In node.js style especially) and then dive to Titanium then it would be easy to grip on Titanium otherwise you may feel it very difficult.
