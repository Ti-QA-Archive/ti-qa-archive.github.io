    • There is a degree of risk you take on when you use a module from the marketplace. You have to trust in the module developers to support their work. I understand how this could make someone nervous, and similarly understand the desire to control your own destiny with your own code.

      Titanium has it's problems - mostly around API parity (an API behaves one way on one platform, and differently on another) and feature coverage (we don't support everything the native platforms do out of the box). We also have specific UI components we know need to perform better (like custom table views with large data sets). We know about these issues and have prioritized them in our dev process.

      However, that blog posting references memory leaks that can happen in any JavaScript application, in the browser or otherwise. Over 33,000 apps have been published with Titanium, by companies large and small. By following general JavaScript development best practices, memory problems can be avoided. We put together a guide about memory use in Titanium - I would recommend consulting this page in the wiki.

      At the end of the day, we will plug any memory leak found in the platform. People have found them in the past, and they are resolved immediately as priority one issues. If you believe you have found such an issue, please report it in our JIRA system and alert community@appcelerator.com - we will move to resolve it immediately.
