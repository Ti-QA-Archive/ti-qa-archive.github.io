  • Accepted Answer

    You can add as many views as you want to your windows. Obviously, the more you add, the more memory you'll eat up, but that doesn't mean you need to restrict yourself to one. I'm thinking what was meant by one view per screen was to add a main view to your window then add all of your other views to that main view, that way you could easily clean up the entire window if you needed to. Of course, this isn't really necessary. Keep your code structured commonjs style and when you close any window, all its children will clean up properly.

  • Hi Robert

    In answer to your question - I have created some extremely complicated layouts compared to the 3 or 4 you mention, with deep nested navigation with many tables, images etc.

    I have NEVER had a memory issue or app crash that was not related to poor code (happens to the best of us) or a known bug (that is usually fixed within a few short weeks).

    The Appcelerator platform is very stable and provides for a very rapid application development solution.

    If you are new to the platform, have a play - use the KitchenSink app to see how many of the things work. If you are already using the platform - play more, expand your scope and see how easy it can be.

    BTW using CommonJS is a great way to structure your code and will make extending and re-using your own code much easier.
