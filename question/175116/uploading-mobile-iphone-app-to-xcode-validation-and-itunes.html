Uploading mobile iPhone app to Xcode validation and iTunes

I am a new developer trying to get a better understanding of coding mobile app for the iPhone. I have recently ran into a problem with packaging my app and validating it on Xcode to publish it on iTunes. I have already created the app on iTunes Connect but the app will not pass validation on Xcode. Please I really need some help fixing this issue. I am using iOS7 and Titanium SDK 3.2.2. I do have a launch file named under Resources/Default-568@2x.png

–Error Message–

This Bundle is invalid. $message.

Your binary is not optimized for iPhone 5 - New iPhone apps and apps updates submitted must support the 4-inch display on iPhone 5 and must include a launch image with the -568h size modifier immediately following the &amp;It;basename&amp;gt; portion of the launch image's filename. Launch images must be PNG files and located at the top-level of your bundle, or provided within each.lproj folder if you localize your launch images.

Version Mismatch. Neither CFBundleVersion ['1.0.0'] nor CFBundleShortVersionString['1.0.0'] in the Info.plist match the version of the app set in iTunes Connect['1.0'].

  • The file name is spelled wrong. It should end with 568h not just 568.
    (The file name should be Default-568h@2x.png with 640 x 1136.)

    Here is the list for Slash screen documentation.
    http://docs.appcelerator.com/titanium/latest/#!/guide/Icons_and_Splash_Screens

    In the later case, I think the version format is mismatch. Follow either X.Y.Z or X.Y formats. Where those stands for X - Major, Y - Minor, and Z - Patches.
