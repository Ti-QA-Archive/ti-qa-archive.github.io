Ho wto reset the getArguments() value

Hi,

I am using the Ti.App.getArguments() on the "resumed" Event Listener to retrieve data from an Custom URL handler.

It Works perfectly but once I got the data from the getArguments() I want to reset the data in the Arguments.

Is there anyway to reset the getArguments() value?

Thanks

  • Not that I have found. What I now do is save the value when the app first launches, and during the pause event. If the value is the same during the resumed event, I treat it as if the user resumed the application as opposed to another launch from the browser or other application.

    This is not accurate, but I have not figured out a way to determine whether the resumed event is due to a user resume or a custom URL launch that repeats the same URL string as the previous launch. For my purposes, it will come close enough for government work.

    YMMV.

  • So following your logic you could than append an additional random variable to your external website with the url schema which would tell your app that a new call has been made:

    e.g. setting a random string value at the end of your url schema by using since unix epoch;

    myapp://?r=RANDOM_STRING_HERE

    This ofcourse would still leave the addressbook calls (e.g.) to your app unnoticeable, because they can't have a dynamic changing variable appended.
