[ERROR] [BEROR]Code Sign error: The default keychain doesn't have an identity matching the profile X and identity Y (Z)

Hello, I am currently trying to get my application to run on an iPhone device. It works in the simulator, and on my Android device.

Here is the error I am getting when I choose Run on iOS Device from Titanium Studio:

[ERROR] [BEROR]Code Sign error: The default keychain doesn't have an identity matching the profile 'XXX' and identity 'iPhone Developer: YYY (ZZZ)'
[ERROR]
[ERROR] Error: Traceback (most recent call last):

I have signed up as an iOS developer, and have created iPhone Developer and iPhone Distribution certificates. I downloaded the .mobileprovision file as well.

Using these files, I can successfully get a hello world app running Xcode.
I thought that if I got something running on the device from Xcode, then the certificates would be correct, and should work on Titanium as well. However, It just will not get past that error message posted above. Are there any Titanium configurations I need to look at?

My initial thought was that the login keychain might not be default, but going into Keychain access, Preferences, First Aid, there is a checkbox in "set login keychain as default".

Any ideas on where I should look to find the problem?

  • I think I found the problem! In Titanium, I had selected my downloaded Distribution Provisioning Profile file instead of my Development Provisioning Profile.

    After downloading the development provisioning profile and selecting it after clicking Run on iOS Device, I was able to finish compilation, and the app appeared in iTunes. Now to get the app over to my phone..

  • I wanted to update these answers because I keep getting caught by this as well - if you're using a distribution profile such as AdHoc, you need to use the Package Build command or Distribute for Apple App Store. This will create the ipa archive and sign correctly. You can then share the ipa and drop into iTunes. If you're just using dev certs then use the regular Run/Debug commands.

  • I was having the exact same problem, so what I did was to open the project in XCode, and select the option to run on iOS Device. That worked fine, so then I selected Product > Build for > Testing. After not seeing anything in iTunes, I finally noticed that my app was installed on a secondary window, and seems to be running fine. I still don't know what the problem is with Titanium, but this workaround seems to do it.
