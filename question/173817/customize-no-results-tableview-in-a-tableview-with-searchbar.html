Customize "No results" TableView in a TableView with Searchbar

Hi there,

I've spent several hours trying to find how to do something that seems to be quite simple…

I'm working with a TableView which has a Searchbar and black backgrounded rows. Then, when I write something in the searchbar, the existing results are displayed fine, but if the results doesn't fill all the TableView, the blank automatic generated rows have white background. In addition, when my search doesn't have any results, all rows have that background color white.

I would like to know how to change the background color of the automatic blank generated rows, how to change the "No results" rows color and how to change the "No results" text, so I could translate it.

Now i'm working with iPhone, using CLI version 3.2.1, Titanium SDK version 3.2.2.GA.

Thanks!

  • Hi

    I have just tried many things including;

    • Setting BackgroundColor to black on the table itself - no change
    • Adding a fake FooterView, this stops the empty rows effect but only on the Main Table - no change on the search version

    I did not expect either of these two to work - but worth a go. I understand that some time ago there was a thought that Apple made a concious choice to have the filter results appear simple. Whilst it seems that filter rows are now honouring the parent the rest of the table styles are not.

    A work around is to create a hidden table that sits over the top over the exiting one. When the SearchBar gains focus you make the hidden TableView visible and perform the filter drill down yourself manually by updating the rows in that table. When a row of that table is selected you can choose to leave the filter TableView visible until the cancel choice is selected (assuming you enable that) or to hide ready for the next focus event. You will need to set the styles for the hidden TableView the same as your main table.

    This is not ideal and you will need to remove the SearchBar from the TableView and add it to the parent Window - as the SearchBar will now have no direct connection to the main TableView.
