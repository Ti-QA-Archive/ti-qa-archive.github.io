Read logcat in Android app?

I have an Android app out that is being used by about 20 Technicians and a few are complaining of a random crash here or there. I have been unable to pin it down to a certain action or a specific device. I was wondering if there is a way or module to read the logcat? Say if a users app crashes, they then start it back up and I can then read the logcat and send it to my servers for review?

    ACRA is an open source for sending bug reports including X lines from logcat.
    ACRApper module bind it with Titanium.

    • as far as I understand the module, the amount of the log is limited by the modules initialization. I tried the android version of the module today and not had reliable results with Logger.getDeviceLogs() method. (mostly "JAVA BINDER FAILED" errors).

      But the "error" event seems to work more reliable, i would set a small php or node script on a server. Then send each payload of the "error" event, which triggers on crashes, to this script.
