Android KitchenSink - Disable Autocorrect not correct working when Password Mask is enabled

Hi all,

I'm using the final version of Titanium Mobile 1.6 under Ubuntu 10.10.

My users are reporting issues regarding the password field. T9/auto correct is enabled although I've disabled this. I tried several methods too disable autocorrect when passwordMask is set to true but it does not work. It is also reproduceable in KitchenSink:

To reproduce, do the following:
Select tab "Controls" -> "Text Field" -> "The Rest"

Now press the "Autocorrect" button, than press the "Password Mask" button (once or twice, it doesn't matter).
It also doesn't matter whether you press the "Password Mask" button first or the "Autocorrect" button.

Now focus the text field. The keyboard will appear. Type some letters for example "hello". T9 is still enabled.

The autocorrect works at the time when the user enter's the first letter and then again click the "Autocorrect" button. But you have definitely wait till the user has entered the first letter.

Is there a workaround? It is a critcal issue for me.

Edit: I forgot to mention that I used Simulator Android 2.1 update 1

Edit2: Maybe I should better describe that it seems the autocorrect is correct enabled, but the suggestion-bar where usually the autocorrect words is displayed is still visible and displays plain the entered password. As soon as the user selects the entered word from this bar, it adds a whitespace to the password. I found out that this issue exists even if passwordMask not activated.
