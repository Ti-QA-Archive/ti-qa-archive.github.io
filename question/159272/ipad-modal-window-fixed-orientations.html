iPad Modal Window Fixed Orientations

Let me begin with my setup: iOS 7.0.3, TiSDK 3.1.3GA, Xcode 5.0.1

Problem

I have a Universal application on iOS. The problem only pertains to the iPad. On the iPad I'm opening a splitView which is allowed to rotate freely (all 4 possible orientations, set via tiapp.xml)

<orientations device="iphone">
            <orientation>Ti.UI.PORTRAIT</orientation>
 </orientations>
<orientations device="ipad">
            <orientation>Ti.UI.PORTRAIT</orientation>
            <orientation>Ti.UI.UPSIDE_PORTRAIT</orientation>
            <orientation>Ti.UI.LANDSCAPE_LEFT</orientation>
            <orientation>Ti.UI.LANDSCAPE_RIGHT</orientation>
</orientations>

Now, when I try to present a modal window with only two orientations: Ti.UI.LANDSCAPE_LEFT, Ti.UI.LANDSCAPE_RIGHT, the modal and the splitView also rotate to the PORTRAIT modes.

Can this be prevented? Is this an iOS or a Ti specific issue if the above is possible?

Possible Solution

I know one must set possible orientations for a window before opening it. However, would it be possible to prevent rotation completely after the window is opened?

Example: if the window is opened in PORTRAIT mode, it stays that way. Same for the LANDSCAPE mode. Once opened, it cannot be rotated, until it is dismissed?
