  • You don't need currentWin here. You only have to do 

    win2.setRightNavButton(edit);

    Ti.UI.currentWindow only can be used in a new different execution context, for example, if you uncomment the url:'recipes.js' line

    var win1 = Titanium.UI.createWindow({
    title : 'Recipes',
    backgroundColor : '#fff',
    url: 'recipes.js'
})

    then, INSIDE recipes.js, you can call

    var currentWin = Ti.UI.currentWindow;
currentWin.setRightNavButton(edit);

    but not in the same execution context

  • I can't tell which .js file the second link is to, but I would guess your variable for currentWin is out of scope, if you are using a separate js file for this window. Declaring currentWin as a var in app.js will not allow you to access it from a window created from a url .js file. If you want to go with this structure, you will have to re-declare the variable in the second .js, or call it something else.
