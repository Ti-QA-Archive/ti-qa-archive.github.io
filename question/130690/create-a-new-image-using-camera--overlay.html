create a new image using camera + overlay

Morning all,
I hope you can advise me with a camera question.

My goal is to use the camera to create a compiled image by sticking a .png on top of the photo.

Example: Let's say I own a hat shop (not really ;-) and I want the user to be able to see what wearing a hat looks like on themselves. In realtime I could use a hat.png overlayed in the camera viewfinder. But what i really want to be able to do is save the image away.

When the user takes the picture, I want that hat superimposed onto the image and save the new image away. So in e.g. your gallery there is a photo of you wearing the hat.

Any general thoughts? Is it possible to add the 2 together in the overlay view, or should I e.g. use a Screenshot?

Any thoughts much appreciated.
THanks,
Lee

  • Hi!

    here is my hint:

    make a container view, for example myViewToScreenshot. Add your png to backgroundimage, which is come from camera. than create a new view: theHat. myViewToScreensot.add(theHat), of course position it.

    than there is a method, called toImage(), wich returns an image of the rendered view, as a Blob.

    so myViewToScreensot.toImage() returns with the camera image+the hat, in principle:)
