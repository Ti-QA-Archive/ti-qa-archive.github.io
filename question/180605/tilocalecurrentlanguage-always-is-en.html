Ti.Locale.currentLanguage always is "en"

Hi guys!
I'm developing multilanguage application.

MacOS 10.10
Titanium SDK version 3.4.1.GA
iOS Simulator version 8.1 (550.3)

I have tried to change different language on iOS simulator, but always at start application i got 'Ti.Locale.currentLanguage' is "en".
But on emulator's screen my app name has been changing depend on the actual emulator language settings. So emulator shown me app name in right localization, but Titanium return always english locale.

I've done "Reset Content & Settings" on emulator, deleted project "build" folder but with out any results.

    The Xcode 6.1 sim has the locale broken, is an Apple bug.

    https://developer.apple.com/library/mac/releasenotes/DeveloperTools/RN-Xcode/Chapters/xc6_release_notes.html

    Localization and Keyboard settings, including 3rd party keyboards, are not correctly honored by Safari, Maps, and developer apps in the iOS 8.1 Simulator. [NSLocale currentLocale] returns en_US and only the English and Emoji keyboards are available. (18418630, 18512161)
