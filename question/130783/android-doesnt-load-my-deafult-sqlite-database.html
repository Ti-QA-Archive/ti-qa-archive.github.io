Android doesn't load my deafult SQLite database

Hello everyone! I've been following the tutorial to manage local data sources and I'm interested in SQLite. I have this line:

var db = Ti.Database.install('', 'prueba.sqlite');

That line on iOS load perfectly the database (even better, I can manage the data with the SQLite Manager plugin for Firefox and kept saved in the next run) but with Android, it's a whole different story: It creates an empty database.

I tried to set multiples paths in install() but with no result. The last one I tried with is:

var db = Ti.Database.install('prueba.sqlite', 'prueba.sqlite');

And I've tried with app://, app-private://, databases, app://databases… A lot of them.

In the project, my SQLite file is located in the Resources folder

My SDK version: 1.8.0_1

and I've also been looking in https://wiki.appcelerator.org/display/guides/Working+with+a+SQLite+Database

  • You're initial database should be in the Resources directory of your app for pre-loading into the data directory of your android device.

    One thing I do is not use a file extension for the database name (second property) when creating the database as iOS and android deal with the database files within the app differently.

    That means I would use something like this:

    var db = Ti.Database.install('/db/prueba.sqlite', 'prueba');

    This implies that the file prueba.sqlite has been stored in a folder called db within the Resources folder of your app project.

    from here on if I want to run queries or open the database for queries you only need to use :

    var db = Ti.Database.open('prueba');

    If you can browse the emulator or device filesystem, the actual database file used by the app will be located here on the filesystem: &#x2F;data&#x2F;data&#x2F;&lt;your-app-id&gt;&#x2F;databases&#x2F;prueba

    It's possible that the app has a database with a blank file in the location with your previous attempts, I would

    Clean your build but also remove/delete the cache or completely remove the app from the emulator/device and then re-run the project on either, once you've checked the above.

    There are no known problems with using Ti.Database.install - i have a project on android that uses it heavily. Let me know if it helps/how you get on.
