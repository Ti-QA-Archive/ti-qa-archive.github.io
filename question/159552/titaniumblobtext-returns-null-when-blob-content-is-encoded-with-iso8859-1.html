Titanium.Blob.text returns null when BLOB content is encoded with ISO8859-1

Hi,

I've got a xml file that I read from a http webservice. This file is encoded with ISO8859-1.
If I try to read the file content with the blog method getText or the property text, the Titanium.Blob.text returns null.

I'm using Titanium 3.1.3 and XCode installed is 5.0.1 (also tried with 5.0.2).

What's the supposed method of reading a non UTF8 content?

Regards
Joao

    When you get the XML back from the http client you need to capture it the responseXML.documentElement.textContent property like so:

    
    var xhr = Ti.Network.createHTTPClient({

        onload : function(e) {

           var xml = this.responseXML.documentElement.textContent;
        console.log(JSON.stringify(xml));        

        },

        onerror : function(e) {

             Ti.API.debug("STATUS: " + this.status);
               Ti.API.debug("TEXT:   " + this.responseText);
                Ti.API.debug("ERROR:  " + e.error);
        },

        timeout : 5000

    });
