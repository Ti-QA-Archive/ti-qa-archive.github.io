button background image not showing up when button defined in a url based view file

Hi,

(this is for an iPhone app).

When defining in a separate file (using the URL based view technique) a windows containing a button, the button background image does not show up.

Clicking on the button works (it pops up the alert defined in the click listener).

If I use a background color instead of a background image, the background color correctly shows up.

If I define the window and the button directly in app.js, the button background image correctly shows up.

Looks like a Titanium bug for me, but maybe I made some error?

I've setup a demo repository, the url-view branch contains code with the button background image not showing up, here is the view file

The master branch contains the code with the button background image showing up (when defined in app.js), here is app.js file

Any help would be appreciated :)

    since the view is in the "views" folder, the path for the image is no longer correct. It should be something like

    backgroundImage: '../images/connect.png'
