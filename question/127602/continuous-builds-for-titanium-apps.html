  • You can also check out: http://guilherme.pro/2011/04/06/titanium-mobile-hack-execute-your-projects-from-the-command-line-using-make/, but it's a bit out of date.

    My experience with automating Appcelerator builds typically involves modifying the builder.py script. First, because the documentation is out of date and doesn't provide the proper options. Second, not even the builder.py script provides the clearest documentation on what the proper options are. So instead, I hack the builder.py script to tell me what the GUI is using as options when I perform a build using Studio. I add something along the lines of:

    for arg in args:
      print arg

    Once I have that, the next challenge is to edit the builder.py script so that instead of attempting to build AND install the app into iTunes, I bypass the "open -b com.apple.itunes" lines. This can be done in a few ways. Commenting the lines out or adding a whole new option to builder.py.

    Then, I create a whole other script that will run builder.py with all the proper options. This script need to parse the output of the builder.py to check for errors and grep for "IPA file should be at" to determine where the IPA got stored. It will then copy the IPA to the place I configured Jenkins to pick it up.

    It's a long and complex process, and I'm a little new to Appcelerator so there may be easier ways that I am not aware of. Keep in mind that the builder.py changes with each new iteration of the SDK, so you'll have to do this for every new version you build for. Evidently the entire build system is up for a rehaul, but it just started in April/May so it might take some time until progress is visible.
