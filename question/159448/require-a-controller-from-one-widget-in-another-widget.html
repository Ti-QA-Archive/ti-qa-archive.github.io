Require a controller from one widget in another widget?

In one of my widgets I am exporting a couple functions:

var addWidget = function(e) {
    $.win.add(e.getView());
};

exports.addWidget = addWidget;

var openWindow = function() {
    Ti.App.myVariable.getActiveTab.open($.win,{animated:true});
};

exports.openWindow = openWindow;

I would like to use these functions in another widget:

var open = require('what should this path be?');

open.addWidget(view);
open.openWindow();

I don't know what the path should be for the require statement. Any help would be appreciated.

  • Hi

    Firstly; exports are much quicker with Alloy;

    $.addWidget = function(e) {
    $.win.add(e.getView());
};
$.openWindow = function() {
    Ti.App.myVariable.getActiveTab.open($.win,{animated:true});
    // not sure what you use this for Ti.App.myVariable... though
};

    If these are true widgets - as in you created them as a widget in the widgets folder then you can create a widget programmatically;

    var something = Alloy.createWidget("widgetname", "controller", { arg1: 'value' });
something.addWidget(view);

    Notes;

    • I have no idea if your code will work as you have only provided a snippet
    • widgetname = the name you used to create the widget you want to reference
    • controller = the name of the controller file containing the logic you want
    • "widget" is the default name if you do not have a specific one
    • arg1… = arguments your controller requires to be setup

    Doing this is only worthwhile if the code you are using inside the widget is specifically useful to the widget. If the code you want is general create a commonJS module and place it in the {appname}/app/lib folder instead and use a normal require.
