Is it possible to run ON DUPLICATE KEY UPDATE on Ti database?

Im having a hell of a time to get ON DUPLICATE KEY calls to work on my database. Im wondering if this might not be supported on the SQLlite that titanium uses?

Thanks!

    sqlite doesn't support this at all. In order to accomplish this, sqlite has the IGNORE resolution. It works like so:

    INSERT OR IGNORE INTO mytable VALUES ($id, 0);
UPDATE mytable SET count = count + 1 WHERE id = $id;

    The id field requires a unique or primary key to make this work
