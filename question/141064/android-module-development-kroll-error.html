    Hi Nilesh,

    Following are my answers:

    • If you are facing issues with kroll libraries, then update your SDK to 2.1.1 and change titanium sdk path to the latest sdk as this is the problem with pre 2.0 SDKs. Now, create your module again and you will not face any issues w.r.t the kroll libraries.

    Note: if you have already written your module code then port the code to the new module created and it will work just fine.

    • If you are facing error w.r.t to ANDROID_SDK/ANDROID_NDK environment variables then you can do two things:

      • First, if you are using windows then define these two environment variables in system settings or if you are using Mac/Linux then create export variables in ~/.bash_profile file
        where ANDROID_SDK pointing to the android sdk home directory and ANDROID_NDK pointing to android ndk home directory

      • Secondly, you can add path to Android SDK and NDK to build.properties file present inside the module directory (%MODULE ROOT%/build.properties) which gets created upon module project creation from command line.
        Add following three variables (i hope, first two android sdk path variables are already present in this file):

        android.platform=/Applications/android-sdk/platforms/android-8
google.apis=/Applications/android-sdk/add-ons/addon-google_apis-google_inc_-8
android.ndk=/Applications/android-ndk

    do check path w.r.t your platform(Mac/Windows/Linux)

    • Nilesh,

      The download link provided was of the source code and you have to compile it for your machine. Follow the following simple steps:

      • 1 ~ 'cd' to the directory containing the package's source code and type
        './configure' to configure the package for your system. If that does not work then type
        'sh ./configure' instead.

        Running `configure' takes a while. While running, it prints some
        messages telling which features it is checking for.

      • 2 ~ Type `make' to compile the package.

      • 3 ~ Open <gperfdir>/src/Makefile and change path to ~/gperf for both prefix and local_prefix variables

      • 4 ~ Open <gperfdir>/doc/Makefile and change path to ~/gperf for both prefix and local_prefix variables

      • 5 ~ Type `make install' to install the programs and any data files and
        documentation.

      after above steps you will obtain gperf executable in ~/gperf/bin directory which you have to add in PATH variable
