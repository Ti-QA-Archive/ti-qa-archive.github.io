[Ti.Media.systemMusicPlayer] nowPlaying property is always an empty object

I am building an app that needs to know if the user is listening to music on his device and show data about currently played music (artist, song title and artwork).

I looked several time to documentation and I think I understood needs to be done… I wrote the function below to test the interface between Titanium and the iOS system music player.

function whatMusicIsPlaying() {

    var systemMusicPlayer = Ti.Media.systemMusicPlayer,
        nowPlaying = systemMusicPlayer.nowPlaying;

    Ti.API.info( JSON.stringify( systemMusicPlayer.playbackState ) );
    Ti.API.info( JSON.stringify( Ti.Media.audioPlaying ) );
    Ti.API.info( JSON.stringify( nowPlaying ) );

}

The problem is I execute (on physical device) this function with a call to setInterval( whatMusicIsPlaying, 5000 ); so every 5 seconds I expect to have results in the app log, but I alway get an empty result for the nowPlaying object (both with the "iPod" music player playing something or not!)

Is this a bug? Am I missing something?

I'd like to get title, artist and artwork from the currently played media.

Would you mind to point me in the right direction to solve this issue, please?

Titanium SDK: both 3.2.0 GA and 3.3.0 (continuos build)

    The nowPlaying property is a Ti.Media.Item and it's NOT a javascript object. That means it cannot be serialised (this is what it's happening when you try to stringify it)

    Try something like

    Ti.API.info( nowPlaying.title );
