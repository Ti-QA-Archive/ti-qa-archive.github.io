  • The "preview in browser" performs a "development" build and is not optimized. Packaged builds perform a "production" build that are somewhat optimized. I agree it's confusing and way too many clicks.

    I exclusively build mobile web projects from the command line on my Mac:

    python /Library/Application\ Support/Titanium/mobilesdk/osx/2.1.0/mobileweb/builder.py /path/to/project production

    To do a development build just like what "preview in browser" does, then just change production to development:

    python /Library/Application\ Support/Titanium/mobilesdk/osx/2.1.0/mobileweb/builder.py /path/to/project development

    Replace 2.1.0 with whatever version you have installed that you want to use. I find it to be faster too.

  • Which version of the SDK are you using?

    There have been a lot of performance improvements made to Mobile Web that are not present in the 2.0.x versions of the SDK. Try using a nightly build of the SDK to see if that helps improve performance.

    It's also possible that there are some large images/scripts causing latency. Check the Network tab in the WebKit Inspector to see if there are some network requests hurting performance.

    In the upcoming 2.1 release, we will release a debugging guide for Mobile Web that talks in more depth on how to use these tools, so stay tuned!
