    • The problem with this approach is you need to pull in each record, compare/process its coords with the current location, and then sort them.

      On MySQL there has been some work done to do on-server functions that will do this, but haven't seen anything for sqlite, or to use points that can be queried.

      Optionally, place in the cloud, acs/parse and allow their backend to feed closest to you based upon geopoints.
