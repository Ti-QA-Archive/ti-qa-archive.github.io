Backbone JS 1.0 support for Titanium Alloy models - Tables issues with binding

I know currently titanium supports Backbone 0.9.2
Is there a plan / feature on Jira / Version you plan to move to Backbone 1.0 or later?

I have some issues with table view binded to backbone collection, I think the table is redrawing every fetch call I make, even when it is just for new items. (there is no smart set method on Backbone 0.9.2)

The performance is lousy and the table seems to not be responsive when the collection returns from the API.

I can write a new binding method and wait for success on the fetch call, but I don't want to re-write the binding mechanism.
I can also listen on the change event and work from there but again it goes to creating the rows dynamically and not via the XML bind.

Any help would be greatly appreciated!

  • Accepted Answer

    • Backbone will be updated to at least 1.0 (probably 1.1) in Alloy 1.4.0: https://jira.appcelerator.org/browse/ALOY-576
    • Alloy built-in view-binding is aggressive, ensuring that it doesn't miss any changes. This will improve when Backbone is updated.
