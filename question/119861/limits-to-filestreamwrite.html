Limits to Filestream.write

Is there a limit to the amount of data you can write with Filestream.Write. I'm trying to write a large amount of HTML to a text file and it seems to be cut off before the end.

— asked May 18th 2011 by Nathan Jones
  • desktop
  • filestream

  • Could you please post a code sample for us to take a look at? How big is this large HTML file?

    — commented May 18th 2011 by Matt Apperson

  • It's around 57000 characters. Here's the code I'm using to write it out:

    var fileSystem = Titanium.Filesystem;
var sep = fileSystem.getSeparator();
var docs = fileSystem.getDocumentsDirectory();
var currentFile = fileSystem.getFile(docs,"my_accounts","current.txt");
var writeFile = currentFile.open(Titanium.Filesystem.MODE_APPEND);
bodyTxt = $("section#accountsSheet table").html()
writeFile.write(bodyTxt);

    I've done a console.log on the bodyTxt variable before I write it out and all the data is there, but when I write it out to the text file, it seems to get cut off a little before the end.

    — commented May 30th 2011 by Nathan Jones

  • I think I've solved it. The filestream object has to be closed (fileStreamObject.close()) after use, otherwise it doesn't finish writing the file.
