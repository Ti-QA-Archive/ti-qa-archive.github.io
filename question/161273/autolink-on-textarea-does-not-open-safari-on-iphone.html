AutoLink on TextArea does not open safari on iPhone

Hi,
Is there anyone who got that issue before?

I am using autoLink Property in TextArea to wrap all website link and Phone number by using Ti.UI.AUTODETECT_ALL or Ti.UI.AUTOLINK_ALL . It shows the link and the phone number in color but when I click on link, it does not run browser to open this link

My code looks like this:

var myView = Ti.UI.createView({});
var myText= Ti.UI.createTextArea({
        value: "Mywebsite: http://mywebsite.com",
        top:15,
        left:10,
        right:10,        
        textAlign:'center',
        editable:false,
        autoLink: Ti.UI.AUTOLINK_ALL,  // I have tried : Ti.UI.AUTODETECT_ALL Ti.UI.AUTODETECT_LINK ... but it does not work at all
        font: {fontSize:15},
        tintColor:'#9400d3',

    });
myView.add(myText);

I am using SDK 3.2.0 , iPhone Simulator: 7.0.3

I would appreciate any help

  • Hello Nta,

    After testing a modified version of your code, we found that it's a bug happening with Mobile SDK 3.2.0.

    Testcase

    var myView = Ti.UI.createView({});
var myText= Ti.UI.createTextArea({
        value: "Mywebsite: http://mywebsite.com \n phone: 555-55-5555",
        top:15,
        left:10,
        right:10,       
        textAlign:'center',
        editable:false,
        autoLink:  Ti.UI.AUTOLINK_ALL,
        font: {fontSize:15},
        tintColor:'#9400d3',

    });
    alert( Ti.UI.AUTOLINK_ALL);
myView.add(myText);
var win = Ti.UI.createWindow({backgroundColor:'white'});
win.add(myView);
win.open();

    JIRA bug report

    I reported this bug into jira, with the number TIMOB-16143. If you visit the bug and click on "Watch", you will be getting notifications on any update we have into it.

    Workaround

    None yet for the textarea.

    Thanks for your help us to improve the platform.

    Best,

    Mauro
