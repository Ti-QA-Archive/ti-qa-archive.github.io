Android webview ping is not stopping

Hi,

I have local webview page which pings server url on regular interval using javascript inside the page.When I am clicking device back button I am calling this function to clear webview from memory.

$.window.addEventListener('androidback', function(e){

$.webview.release();    
$.webview     = null;
$.window.close();
$.window     = null;
Alloy.createController('subject').getView().open();

});

even after redirecting to subject page I am able to see ping is continuously coming to server side.
Is there any other way to stop the ping/release the webview on clicking of the back button.???

SDK version: 3.2.3

Thanks in advance

Vivek P

  • Try to remove it from its parent component. Also set html property to empty string.

    Here is sample code

    $.window.addEventListener('androidback', function(e){
    $.window.html = "";
    $.window.remove(webview);
    $.webview.release();    
    $.webview   = null;
    $.window.close();
    $.window    = null;
    Alloy.createController('subject').getView().open();
});

    Also if there are any global events, remove it like Ti.App.removeEventListner. I think global event listener are causing issues in this case. Otherwise its quite strange after setting null its still working.
