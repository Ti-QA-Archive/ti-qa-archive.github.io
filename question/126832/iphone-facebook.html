  • Just tested the update, and it worked! Thanks for the patch.

    Here's few observations with the update :

    –Facebook feedback on post
    on the case of user approves app then de-authorize or change password later, token is not longer valid. It used to return e.error, now appears on e.result. An example of result feedback with error 

    "error_code=190&error_msg=Error+validating+access+token%3A+User+10xxx2234723xxx+has+not+authorized+application+27xxx9812760xxx."

    an example of good result

    post_id=10000223472xxxx_2046110896xxxx

    To catch such event, i try these

             var res_ = {};
         var res = Titanium.Network.decodeURIComponent(e.result.replace(/\+/g, ' '));
           res.replace(new RegExp( "([^?=&]+)(=([^&]*))?", "g" ),function($0, $1, $2, $3) { res_[$1] = $3; });
             var post_id = res_['post_id'];
             var error_code = res_['error_code'];
             var error_msg = res_['error_msg'];  

 if(post_id || res.indexOf("post_id")>0 ){
  //post is good

 }

  else if (error_code && res.indexOf("Error validating access token")>0 ) {
   //errors

   Ti.Facebook.logout(); //log user out since the token is not longer valid, then prompt message to re login again.
  }

else ........ //trap the rest of who-know-what error
