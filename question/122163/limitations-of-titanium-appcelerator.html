Limitations of Titanium Appcelerator

Hi. I'm newbie in the programming business. I know only html, css, some javascript and some python. I wanted to start developing ios apps, but first of jumping into learning something I wanted to know 'where' to jump. I don't know what to do, If start learning Objetive-C and build native apps or use titanium. I know that the path of learning Objective-C is a longer one, but I have the sensation that using a 3rd party in the developing process will limit me. So I wanted to know, how true is this. What do you recommend?
Specially: what are the limitations nowdays of using appcelerator if I don't know writing a native module??
Thanks for the help!!

  • Mateo,

    I can only tell you about my own experience.

    It took me 3 months to build my 1st app in Objective-C (including 1 month full-time to start understanding how Apple Dev Tools work), and my app was rather simple (no communication for instance - search 'clostermann' in the App Store). I had done some C in the past and I had OOP experience.

    It took me 1.5 month to build my 1st Titanium app, and it was much more complex (some Ajax, RSS feed) with a nicer interface (search 'corinnelepage' in the App Store).

    My feeling is that Titanium offers (almost) every features native languages (Obj-C, Java…) offers, but is easier to use and much more productive (thanks JavaScript!). Also, you don't have to care about low-level technical aspects (think of the impact of the modified memory management in iOS 5 for programers :-( ).

    Since TitaniumStudio has been released a few weeks ago, we have a nice IDE, based on Eclipse, with more to come (including a GUI designer tool). And there is always the possibility to add native modules if one function you need is not available in Titanium.

    Best regards,

    Laurent
