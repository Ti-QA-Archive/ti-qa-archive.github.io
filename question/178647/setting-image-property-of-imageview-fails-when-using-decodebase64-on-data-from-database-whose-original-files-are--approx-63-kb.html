Setting image property of ImageView fails when using decodeBase64 on data from database whose original files are > approx 63 KB

W're building an app for android. We have an external CMS which encodes uploaded images as base64 and stores that data in a table. The base64 encoded image data is then queried (via Node.js) and pulled into our app. This all works perfectly - that is, for image data whose source image is under about 60 KB (as reported by Mac finder window.) For image data whose source images are over (about) 60 KB, however, the imageView does not display the image.

I do get the following warning when the failure occurs:

[DEBUG] skia: --- decoder->decode returned false
[DEBUG] skia: --- SkImageDecoder::Factory returned null

My hunch is that the base64Decode function is giving up before reading all the image data for larger images, but I have no way to back that up.

Some system specs:

  1. Android App.
  2. tiapp.xml <sdk-version> set to 3.4.0.GA, but current Titanium SDK set to 3.3.0.GA
  3. My machine is Mac 10.9.5 and the device is an MOMOMini.

And, a sample of the code which produces the behavior:

imageBlob = Titanium.Utils.base64decode(offerData[slideNumber].offerMainImage);
mainImage.setImage(imageBlob);

Oh, and normally I would store image paths in the database, rather than the image data, but this particular project requires the image data be stored in the DB.

Many thanks,

Josh
