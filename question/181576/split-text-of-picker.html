  • Hi

    It is likely the selectedValue you are retrieving is an array. Even though you only have a single column the result is returned as an array so the value you seek is actually the first element of the selectedValue array not a selectedValue string.

    // change your code as follows
var Loquetengo = e.selectedValue[0] || '';
Ti.API.info('User selected: ' + e.rowIndex + "/"+Loquetengo + "/");
Loquehay=Loquetengo.split(":");

    This assigns the first array element of selectedValue to Loquetengo, if it is not an array then an empty string is used instead.

    This means Loquehay will have the correct string value provided the returned value was an array rather than a string.

    Hopefully this will work but I agree with Mark and Stephen though, it is more common to provide two columns as it lets the user quickly jump through the hours.
