I realize that this problem was posted in earlier questions. However, solution was partially built and it was possible to "amend" this code of TiUIiOSAdViewProxy.m file, from code generated by Titanium in the Xcode project file:

-(NSString*)SIZE_320x50 
{
    if ([TiUtils isIOS4_2OrGreater]) {
        return ADBannerContentSizeIdentifierPortrait;
    }
    return @"ADBannerContentSize320x50";
}

-(NSString*)SIZE_480x32 
{
    if ([TiUtils isIOS4_2OrGreater]) {
        return ADBannerContentSizeIdentifierLandscape;
    }
    return @"ADBannerContentSize480x32";
}

What let me run the project on simulator was putting both returned variables as strings (adding "@" and quotations).
Thing is: I do not know if this is proper way of fixing that problem or just band-aid - I am no expert in native iPhone development. Testing it on KitchenSink application, I found out that the "Window Events" example does not work and application shuts down - is this code in any possible way related to it?
