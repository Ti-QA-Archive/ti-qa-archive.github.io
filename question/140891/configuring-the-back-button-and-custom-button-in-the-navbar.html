    Hi John

    This is a great and simple example of how to play with and react to the top nav bar buttons.

    var btnChoice = Ti.UI.createButton({
    title: 'Choices'
});
btnChoice.addEventListener('click', function (e) {
    var opts = {
        title: 'Quick Jump',
        options: [
            'Daily Reports',
            'Snapshots',
            'My School',
            'My PDA'
        ]
    };
    if (Ti.Platform.osname === 'iphone') {
        opts.options.push('Cancel');
        opts.cancel = opts.options.length - 1;
    }
    var dialog = Ti.UI.createOptionDialog(opts);
    dialog.addEventListener('click', function (e) {
        Ti.API.info('index', e.index);
        Ti.API.info('cancel', e.cancel);
        if (e.index !== e.cancel) {
            win.tabGroup.setActiveTab(e.index + 1);
        }
    });
    dialog.show({
        view: btnChoice
    });
});
win.setLeftNavButton(btnChoice);

var btnWindow = Ti.UI.createButton({
    title: 'Window'
});
btnWindow.addEventListener('click', function (e) {
    var newWin = Titanium.UI.createWindow({
        backgroundColor: '#fff',
        title: 'New Window'
    });
    win.tabGroup.getActiveTab().open(newWin, { animated: true });
});
win.setRightNavButton(btnWindow);

  • Hi John

    Just in case you are not aware, when you add a window to tabView then navBar functionality is automatic included as well.

    This line controls the process;

    tab.open(winNew, { animated: true });

    A full example below; create a new app and then replace the contents of the app.js with these.

    Ti.UI.setBackgroundColor('#000');

var tab;
(function () {
    var win = Ti.UI.createWindow({
        backgroundColor: '#fff',
        title: 'Test'
    });
    tab = Ti.UI.createTab({
        icon: 'KS_nav_views.png',
        title: 'Tab 1',
        window: win
    });

    var btnMain = Ti.UI.createButton({
        title: 'Main'
    });
    btnMain.addEventListener('click', function (e) {
        alert('Main Button');
    });
    win.setRightNavButton(btnMain);

    var btnNewWindow = Ti.UI.createButton({
        height: Ti.UI.SIZE,
        title: 'New Window',
        width: Ti.UI.SIZE
    });
    win.add(btnNewWindow);
    btnNewWindow.addEventListener('click', function (e) {
        var winNew = Ti.UI.createWindow({
            backgroundColor: '#fff',
            title: 'Another Window'
        });
        var btnTest = Ti.UI.createButton({
            title: 'Test'
        });
        btnTest.addEventListener('click', function (e) {
            alert('Test Button');
        });
        winNew.setRightNavButton(btnTest);
        tab.open(winNew, { aniated: true }); // used here
    });

})();

var tabGroup = Ti.UI.createTabGroup();
tabGroup.addTab(tab);
tabGroup.open();

    If you specifically need a navBar equivalent to this then let me know, but if you have a tabBar then this should help you out.

    • Having reviewed and run your code, I cannot see why a back button would appear in the context of your app.

      You have several buttons sitting over the top of a background image (not sure if this will become a webview), clicking each of these takes me to the associated tab sharing the same name. This code functions correctly and swaps the tabs. However a back button would not be required as by changing tabs you have swapped visual contexts - the user would not expect to to jump back using the standard UX/UI.

      The logic being that the tabs are distinct and subjects so there is no drill down navigation to return back up from. It is also not usual to present the user with a duplicate set of choices that match those presented in tabs. Can you explain why you wish to have two navigation sets to get to the same places?
