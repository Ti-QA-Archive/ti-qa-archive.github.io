Want to skip blackberry installation

Hi, I would just like to develop for Android and maybe later, iphone. I do not plan to develop for Blackberry. During the installation, when I am creating a New Titanium Mobile Project, under Deployment Targets, there is a link to Setup/Configure SDKs. When I click on that, I can setup the Titanium directory and the android directory, but I don't have a blackberry installation and I don't want one. However Titanium will not let me save the Titanium and Android directories and continue until I have successfully installed a Blackberry directory. So I try the little link that says "Get SDK" (the blackberry one), and I get an error message "Could not complete the installation because one or more items could not be installed".

Cannot complete the install because one or more required items could not be found.
Software being installed: BlackBerry Java Plug-in Documentation 1.5.0.201110141512 (net.rim.ejde.doc 1.5.0.201110141512)
Missing requirement: net.rim.ejde 1.5.0.201110141512 requires 'bundle org.eclipse.jdt.launching 3.5.0' but it could not be found
Cannot satisfy dependency:
From: BlackBerry Java Plug-in Documentation 1.5.0.201110141512 (net.rim.ejde.doc 1.5.0.201110141512)
To: bundle net.rim.ejde 0.0.0

So I can not work with Android because the blackberry installation is messed up and Titanium does not allow me to get past that. How can I resolve this? Remember I have no intention to use the blackberry setup, so I think Titanium should not force me to get it right before allowing me to continue.

  • You should install the Eclipse Java Development Toolkit.

    Go to Help -> Install New Software.
    If you have done this already, add the Eclipse Helios Update Site via the Available Software Sites link. Choose this site via the Work with dropdown menu, wait and select Eclipse Java Development Tools, which is located in the Programming Languages submenu.

    Restart Titanium Studio after installation of the JDT.

    Then you are able to install the BlackBerry plugin.
