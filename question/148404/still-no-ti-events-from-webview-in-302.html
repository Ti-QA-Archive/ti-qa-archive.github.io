  • Hi

    As you have not said what you actual problem is, I am going to guess. Obviously you have an issue with your onload event failing - but I am sure that event is not your actual need.

    Here is some code to add to your app.

    function webTest(e) {
    //Ti.App.removeEventListener('test', webTest);
    alert(JSON.stringify(e));    
}
Ti.App.addEventListener('test', webTest);

var web = Ti.UI.createWebView({
    height: Ti.UI.FILL,
    url: '/test.htm',
    width: Ti.UI.FILL
});
win1.add(web);

    This is the test.htm file.

    <!DOCTYPE HTML>
<html lang="en-US">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=320, initial-scale=1.0"/>  
    <meta name="format-detection" content="telephone=no">
    <title>Test</title>
    <link rel="stylesheet" href="styles.css" />
<script type="text/javascript">
    function test(boolAnswer) {
        Ti.App.fireEvent('test', {
            test: boolAnswer,
            somethingfixed: 'hello'
        });
        return false;
    }
</script>
</head>
<body>
    <p><a style="text-decoration: underline; font-weight: bold" onclick="test(true);">
        Click Me 1
    </a></p>
    <p><a style="text-decoration: underline; font-weight: bold" onclick="test(false);">
        Click Me 2
    </a></p>
</body>
</html>

    If you are truing to receive events from a HTML page inside your webView and beable to respond to them - that is how to do it.
