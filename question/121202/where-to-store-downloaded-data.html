Where to store downloaded data

Hi,

I was downloading some images locally on the device and storing it in the Resources/images directory after deleting the previously downloaded images. It was working but sometimes some random images were not getting downloaded on device.

So I changed the storage location to Ti.Filesystem.applicationDataDirectory and now it is working fine on simulator and device both… problem is solved but I don't understand the difference between two storage locations.

Does anyone have any idea why the images were missing sometimes in Resources directory but works with applicationDataDirectory?

Ti SDK: 1.6.1 | Platform: iOS 4.x

  • On a device (but not the simulator), the Resources folder is read-only. So you can't download to it while running on a real device. I suggest you use Ti.Filesystem.applicationDataDirectory instead. Or create your own subfolder off the above folder.
