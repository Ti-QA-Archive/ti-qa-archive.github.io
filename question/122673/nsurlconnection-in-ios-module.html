  • I ran into a similar problem integrating a library that uses NSURLConnection into my module. I don't have a confirmed answer as to why the normal NSURLConnectionDelegate methods are not being called, but it could be because the thread which calls your module's method goes away before the URL request is completed.

    Instead of creating an NSURLConnection object with connectionWithRequest:delegate:, try using sendAsynchronousRequest:queue:completionHandler:. That method sets up an NSOperation in the operation queue that you specify which exists for the entire lifetime of the URL connection, and it has the advantage of simplicity in that you only have to write a completion handler block to get the entire response.

    This workaround actually didn't help me as I was calling a method in a library that set up the connection. In my case, I just wrapped the library call in an NSBlockOperation and added it to the main operation queue.
