Test the presence of an application on the devise

Hi, my purpose is to test if I have an application installed on my devise. e.g.: I have a button on my app that on click event, test if an application named "AppX" is installed on my iPhone or Android, if yes, it will open it. Else it will open the Apple Store / Android MArket to install it.
Does this possible with titanium?
Thanks.

    For any Apple created device this can not be done, apple does not allow an app to use the private APIs needed to do this (aka this is not a titanium issue, but rather an apple issue).

  • With apologies to Matt…

    Actually, on iOS devices if the other application has registered a custom URL (as all Titanium apps do) and you know that applications URL scheme, you can use the Ti.Platform.canOpenURL method to determine if the app is installed.

    If true, then you can use it with Ti.Platform.openURL method to start the app.

    This is iOS only.

    If these are you own apps, you can override the custom URL schema name that Titanium registers. See also this and this thread.

    • You can find the iOS URL scheme for any app by doing the following:

      • Download the app in iTunes
      • Open the app in Finder
      • Use Terminal to cd to that location
      • From terminal, unzip the app unzip Third\ Party\ App.ipa
      • You should see a Payload folder. Go there and cd again into the Third Party App.app folder
      • Copy the Info.plist file to your desktop cp -a Info.plist ~&#x2F;Desktop&#x2F;
      • Open the Info.plist in Xcode
      • Drill down to URL Types > Item 0 > URL Schemes > Item 0
      • If the app has a URL scheme, you will see something like myapp:&#x2F;&#x2F; or fb12341234:&#x2F;&#x2F;
      • Note: the number after the fb is the Facebook App ID

    • For Android,

      You can try to open the 3rd party app in an intent, and fall back to Google Play in a catch statement.

      try {
    var intent = Ti.Android.createIntent({
        action : Ti.Android.ACTION_VIEW,
        className : 'com.foo.bar.SplashActivity',
        packageName : 'com.foo.bar'
    });
    intent.addCategory(Titanium.Android.CATEGORY_LAUNCHER);
    Ti.Android.currentActivity.startActivity(intent);
} catch (EE){        
    Ti.Platform.openURL( 'https://play.google.com/store/apps/details?id=com.foo.bar');
}

      You will need to replace the three items in the above example:

      • Google Play URL (just search for that app and grab the URL)
      • packageName (usually the id URL parameter in Google Play)
      • className (see below)

      To find the className, open Android Debug Monitor. Then launch the 3rd party app from your Android device and try plugging in things you find.

      /Users/<username>/Library/android-sdk-macosx/tools/monitor

      iOS has Ti.Platform.canOpenURL where you can test for the presence of a 3rd party app without actually launching it. Android doesn't have the equivalent (well, it does in native android – queryIntentActivities on the package manager). The only way in Appcelerator seems to be to use a 3rd party module.
