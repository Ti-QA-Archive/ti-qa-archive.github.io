How to scrollTo content in a ScrollView before it is shown?

Hi y'all!

So how do you move the content to be viewed in the right position before you show it?

I have tried the:

  • contentOffset property
  • center property
  • scrollTo method - looking at the 'scroll' event, it does scroll but then another event is fired which sets it back at 0.
  • ?

You can check it out with the KitchenSink example:

scroll_views_tabs.js

My changes:


var scrollView = Titanium.UI.createScrollView({
    contentWidth:500,
    contentHeight:50,
    top:10,
    height:50,
    width:230,
    borderRadius:10,
    backgroundColor:'#13386c',
    center: [250, 0],
    contentOffset: [250, 0]
});
scrollView.scrollTo(300, 0);

I'm probably doing something wrong, but what is it?

Any light would be much appreciated, thanks.

  • What I Tried

    I wasn't able to get the contentOffset to work either on Ti Mobile 1.6.1 and Android 1.6. I tried all kinds of combinations of top properties and contentWidth and contentHeight, all to no avail.

    Solution

    Here is the solution I am using. Because there is no window onload event, I use an interval and check if the scrollview is offset where I want it ever 100 ms until it finally gets there, and then ends the timer. You can simplify it more, but this will show you what is going on.

    var offset = 0;
scrollView.addEventListener('scroll', function(e) {
    if (e.y!=null) {
        offset = e.y;
        Ti.API.info('offset: '+offset);
    } else {
        Ti.API.info('offset: null.. no idea why');
    }
});

var init = setInterval(function(e){
    Ti.API.info('check if '+offset+' = 50');
    if (offset==50) {
        Ti.API.info('we are done');
        clearInterval(init);
    }
    scrollView.scrollTo(0,50);
},100);
