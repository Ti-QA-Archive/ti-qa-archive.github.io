Custom fonts working in 3.4.0.RC2 but broken in 3.4.0.GA ?

I can't be sure if this is a bug or not, hence my posting it here prior to filing a bug report. I ran the sample:

var customFont = 'DS-Digital'; // use the friendly-name on iOS
if(Ti.Platform.osname=='android') {
   // on Android, use the "base name" of the file (name without extension)
   customFont = 'DS-DIGI';
}

var label1 = Titanium.UI.createLabel({
   color:'#000',
   text:'I am Window 1',
   font:{
      fontSize:40,
      fontFamily: customFont
   },
   textAlign:'center'
});

It works fine in RC2 but in 3.4.0.GA it defaults to the standard font. I've reviewed the release notes and not found anything referencing custom fonts, but as I am a little rusty I wanted to confirm I wasn't missing anything before filing a bug report.

Any input greatly appreciated.

(Running this in the latest Xcode simulator, various iOS devices, 8.1)

Thank you

    I'm not having any issues with custom fonts in 3.4.0.GA

    Sometimes they're just annoying…

    http://docs.appcelerator.com/titanium/latest/#!/guide/Custom_Fonts has the tip of renaming the file to match the PostScript name, so you don't need different strings per platform.
