Android Problem with "index" in TableViewRow

Hi all,

I have a problem with the "index" of tableviewrow only with android.
On IOS, when i click on one row it's possible to know what is the row who is clicked (with "e.index"). On android it's not possible.

But how to do ?

My tableview is editable , so it's impossible to attribute manually a ID .

myRowButton.addEventListener("click",function(e){
    Ti.API.info(e.index); // For iOS it's ok, but nothing on Android
    //with "e.rowIndex" i've nothing .
});

    Add the event to the table, not to the button inside the row and check for the e.source if it's the button you clicked on.

    You will get this way the index and also as a bonus one event only in the whole table.
