Android Version Number

Hello,

I am sorry, to annoy you again. First off: I Deleted the build/android folder AND in the Studio selected Project > Clean…

Still, in Android (which really starts to fuck me up) I log:

  • Accepted Answer

    I'll agree that Titanium has a number of bugs and inconsistencies. I've blogged about it extensively.

    But I don't see the problem you're seeing. I just ran a quick test myself:

    tiapp.xml:

    <ti:app xmlns:ti="http://ti.appcelerator.org">
...
    <sdk-version>2.1.0.GA</sdk-version>
    <name>Test</name>
    <version>1.0.2</version>
...
    <android xmlns:android="http://schemas.android.com/apk/res/android">
        <manifest android:versionCode="1" android:versionName="1.0"/>
    </android>
...
</ti:app>

    app.js:

    Ti.API.info(Ti.App.name + ' ' + Ti.App.version);

    Console output:

    [INFO][TiAPI   (  311)] Test 1.0.2

    Note that the android versionName has nothing to do with what's returned by Ti.App.version.

  • In my experience, Ti.App.getVersion() returns the string that is in the tiapp.xml in the &#x2F;ti:app&#x2F;version element. You can use more or less any string (like "1.0.0rc1"), but for best results, I would stick to three-segment integers, like 1.0.0.

    Maybe you should try posting your entire tiapp.xml.
