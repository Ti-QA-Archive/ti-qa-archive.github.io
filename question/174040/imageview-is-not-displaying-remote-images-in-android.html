  • ok, please try setting width and height of that imageView. If also nothing is shows up, the image probably is already downloaded and cached with a zero size, because ImageViews are automatically caching remote images. To force the imageView to download the image again, add a random component suffixed by "?". If this also doesn't work, check if the image is corrupted in any way, and check the httpheaders if they return the right mimetype.

    example (add random component):

    var image=Ti.UI.createImageView({
    width:200,
    height:200,
    image:url+json.item[i].photo+'?'+Math.random()
)};
win.add(image);
