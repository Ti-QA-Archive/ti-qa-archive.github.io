  • Right now there isn't a great way to create inheritance in Alloy-style models, due to the way they are structured. I know this isn't exactly the "answer" you are looking for, but in the near future I'll be working on documenting a new structure for Alloy models, likely to work along with the existing format. The new format will have little to no Alloy-specific code around it, allowing you to use Backbone models just as you would without Alloy, but still leverage things like data binding in your markup. I don't have details on this yet unfortunately, but if you subscribe to the Alloy Google Group I'll be sure to be talking about it there when it occurs.

    That said, this community always surprised me, maybe somebody has thought of something clever that I haven't yet.
