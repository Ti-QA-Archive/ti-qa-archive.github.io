Android resolutions - strategy?

Hi,

I'm currently working on an Android app that's suppose to work on both phones & tablets.

I've already finished developing for the WXGA resolution (although it's not really 1280x800 but more 1280x752 but that's a different issue).

Is there a 'smart' way to make my app look good on all other Android resolutions (tablets & phones)? Or do I simply need to design the graphics for each resolution by checking the Titanium.Platform.displayCaps.platformWidth and Titanium.Platform.displayCaps.platformHeight values?

Thanks,
Idan

  • Accepted Answer

    There is no need for mathematical calculations since Titanium supports dimensions defined as percentages.

    //set view's width to half of it's parent width
var view = Ti.UI.createView({
    width: '50%'
});

    You can also check this link on how to handle images: https://wiki.appcelerator.org/display/guides/Supporting+Multiple+Platforms+in+a+Single+Codebase#SupportingMultiplePlatformsinaSingleCodebase-Resolutionspecificimages

  • You can code your interface using dp unit, such as

    Ti.UI.createView({ width: '120dp' });

    In your tiapp.xml, set to support all density & size

    <android xmlns:android="http://schemas.android.com/apk/res/android">
  <supports-screens
    android:anyDensity="true" android:largeScreens="true"
    android:normalScreens="true" android:smallScreens="true"/>
</android>

    Tested on emulator of all size type
