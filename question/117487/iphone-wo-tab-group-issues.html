iPhone w/o Tab Group Issues

In order to make my app as cross platform as possible I didn't use the standard tab group but rather one posted here in the forums. When building for Android it met my needs quite well. Now that I'm finally testing on iPhone I'm finding a lot of things don't work quite right when not using a tab group. Am I doing something wrong or is this a known fact?

The two major things I'm hitting right now:

  1. I can't get any type of navigation bar without a tab group. Thus I have to manually build a back/close button on screens that load open other windows.

  2. When opening window to window {animated:true} doesn't seem to do anything.

Any advice is appreciated.

FYI, the "custom tab" solution I'm using can be found here for reference:
http://developer.appcelerator.com/question/46801/giving-to-the-community-custom-tabgroup-class-working-on-android-and-iphone

  • Hi.,

    In-order to make your app cross platform you can use Ti's default Tab group itself as Tab group is available on both iPhone and Android.

    You can use Ti.UI.currentTab.open(wnd2,{animated:true}); to open child windows and this will work on both platforms but the difference is that on iPhone it will create a Child window with a navber+back button while on android it opens a new window and you can always go back using the back button on Android.

    Hope this helps …
