Titanium.App.Properties not working on android

Hi,
i am new on Titanium,facing some problem related to android,
my application is tab based,
and currently structure of my app as below

- app.js
 - tabWindow
   - window1.js
   - window2.js

currently on window1.js i am saving the values in app properties, code as below.

Titanium.App.Properties.setString('Center_Id', e.rowData.id);
    Titanium.App.Properties.setList('Stations',stations);

and on the next page window2.js using the following code for the values.

var centers = Titanium.App.Properties.getList('Stations');
var center_id = Titanium.App.Properties.getString('Center_Id');

but it's not getting values that i saved from previous window.

this code perfectly working on iPhone but not on android.

from going window1 to window2, i am using the following code.

var next_window = Titanium.UI.createWindow({
        url : 'window2.js',
        title : 'Express Centers'

    });
    //Ti.UI.currentTab.open(next_window, {animated:true});
        next_window.open({animated : true});

can any one help me with this issue?
thanks.

    If you're building a custom tableViewRow you should use e.row instead of e.rowData

    Titanium.App.Properties.setString('Center_Id', e.row.id);

  • Hi,

    I do not know how you create row but if you are creating row with tableViewRow object then you can get it by e.row.id or else if you are creating with javascript object then you can get it by e.rowData.id as you did in your example.

    for debugging purpose, you can put alert on click event so you can debug your problem.

    tableView.addEventListener('click',function(e){
    alert('ID >> ' + e.row.id + + ' Array >> ' + stations);
});

  • Maybe you have a timing problem. Please try:

    var centers = (Ti.App.Properties.hasProperty('Stations'))? Titanium.App.Properties.getList('Stations') :[] ;

    On the other hand: what is your goal? I think you will communicate between activities (windows). Properties is for persisting between sessions/runtimes of app. In you case maybe Event/Observer- or the Callback pattern a better solution.
