Performance Problem on Android

  • Application type: mobile
  • Titanium SDK: Titanium Command-Line Interface, CLI version 3.4.1, Titanium SDK version 3.5.0.GA
  • Platform: Android
  • Device: physical device (all)
  • Host Operating System: Windows 7
  • Titanium Studio: Titanium Studio, build: 3.4.1.201410281727

My app contains a search function based on an SQLite-Database. On the resultpage I show images for every entry. The app has to work offline, so I have to publish the apk with all images for the entries.

There are more than 10.000 images I have to include in the app. As soon as I include these images, my whole app starts to feel stumple. Everytime I open the app and initialize the database with the first "Select"-Statement, the app freezes for 20-30 seconds and the search need 3 times longer and the whole app gets slower, as if I dont include the images.

I don't know, how I can optimize this … on iOS there are no problems with that amount of images. I spend a lot of time in optimizing these, nothing worked.
