I am trying to replicate the Kitchen Sink "Pull to Refresh" example using a ScrollView instead of a TableView. According to the API, there is not a &#39;scrollEnd&#39; event, so how would I detect that the user has released the view? I tried &#39;touchEnd&#39;, but it doesn't seem to fire. Thanks!

  • My current solution is to open a new window when the scroll event's offset reaches a certain point. The new window will efectively force a scrollend because the user will no longer be able to touch the ScrollView. The new window can either be completely transparent, or in my case it serves as an activity indicator. I would love to know a more appropriate way to solve this so that the user doesn't lose any interactivity.
