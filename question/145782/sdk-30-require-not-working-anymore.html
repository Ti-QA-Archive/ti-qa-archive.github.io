SDK 3.0: require() not working anymore

Hi,

I just upgraded to SDK 3.0 and now my Ti.include() works but not my require();

I have Node.js installed and everything, etc. When I start my app all I get is:
[ERROR] Script Error = Couldn't find module: UI/PinGridView at app.js (line 0).

I changed NOTHING since yesterday and it was working fine.

I read across the changeslog and found nothing about require anywhere.

Anyone has an idea?

Thanks and have a nice day!

    Try to put the in the require the absolute path of the module:

    require('/UI/PinGridView');

    The leading / means the Resources folder.

    • I HIGHLY recommend not using the beginning forward slash. That's actually a bug and incorrect behavior on our part. If a module ID begins with a forward slash, we should be treating it as a resource, not a module. If you do use a leading forward slash, then when the bug is fixed, you'll also need to add .js to the end.
