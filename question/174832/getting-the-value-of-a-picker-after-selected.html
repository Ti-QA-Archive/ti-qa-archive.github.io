  • Accepted Answer

    Referring to the API doc, selectedValue stores an array of selected values, one element per column in the picker.
    Applicable to Titanium.UI.PICKER_TYPE_PLAIN picker types.

    <Alloy>
    <Window>
        <Picker id="compPicker" selectionIndicator="true">
               <PickerRow title="fee"/>
               <PickerRow title="fie"/>
               <PickerRow title="foh"/>
               <PickerRow title="fum"/>
               <PickerRow title="other"/>
        </Picker>
    </Window>
</Alloy>

    given the above style, in controller you may get the value of the selected item like this:

    $.compPicker.addEventListener( 'change', function( event ){
    Ti.API.info( event.selectedValue[0] ); 
} );

  • Could be this has changed with Titanium 3.x, but according to the API and my own experience the correct way is to use getSelectedRow( index ). If you have only one selection, the index will be 0.

    Also the correct way to define a picker is: 

    <Alloy>
    <Window>
        <Picker id="compPicker" selectionIndicator="true">
            <PickerColumn>
                <PickerRow title="fee"/>
                <PickerRow title="fie"/>
                <PickerRow title="foh"/>
                <PickerRow title="fum"/>
                <PickerRow title="other"/>
            </PickerColumn>
        </Picker>
    </Window>
</Alloy>

    to get the selected value would be:

    $.compPicker.getSelectedRow(0).title;
