New to Titanium: Alloy or Not?

I'm new to Titanium. "Traditional" Titanium development looks easy enough and there are tons of code samples. Alloy appears to be the new hotness but it seems to be more complicated (node, file creep, etc). Should I stick with "traditional" Titanium development approach or force myself to use Alloy?

    This very much describes the scenario I'm in too (but I'm a bit further ahead with my stuff). I went the older, js route.

    My reasons:

    • Way more examples and useable code out there.
    • Usually when I end up doing stuff using code anyway (like in Cocoa, and other UI frameworks in the past). It often is the only way to achieve the dynamic UI's that I want to create.
    • Alloy is VERY new.
    • Alloy certainly has a steeper learning curve.

    I know there are are some good reasons to use alloy. I'm personally a huge fan of node.js so that would be nice (and wish even their older stuff would incorporate it). Alloy makes your code more structured by default, otherwise you need to be careful (especially if you haven't written applications in js before).

    In the end I'm happy with my decision. The biggest challenge so far has just been unexpected behavior across the platforms (mostly Android really).

  • I've been using traditional Titanium for about a year now. I made a switch to Alloy a month ago. After the initial learning curve, I find Alloy 100% easier for cross-platform (iOS and Android) code and form-factor (handheld and tablet) specific code. Nearly all of the platform and form factor specific UI code that used to be in my JS files are now in xml and tss files.

  • I studied Alloy for a couple of weeks and it seems fine..
