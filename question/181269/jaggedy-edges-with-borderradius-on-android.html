Jaggedy edges with borderRadius on Android

Example here

I am using a widget to draw circles with loading bars (this one: https://github.com/manumaticx/circularprogress)
On iOS because of the native anti aliasing the edges look very good but on Android devices the edges are very jagged as there is no AA being done here.

I want to try get it to smooth out without touching the widget code itself (to keep it at least somewhat independent from our code)

Is there a way to force titanium to use anti aliasing on a View (which holds the widget) to get it to smooth the edges.
Looked at the other questions on this topic here but they were quite different from this I think.

    Hi

    This is how you solve the problem;

    tss

    ".avatar": {
    borderColor: 'transparent',
    borderRadius: 50,
    borderWidth: 0,
    height: 100,
    image: "http://www.fillmurray.com/200/200",
    width: 100
}
".border": {
    borderColor: 'orange',
    borderRadius: 50,
    borderWidth: 10,
    height: 100,
    width: 100
}

    xml

    <Alloy>
    <Window class="container">
        <ImageView class="avatar" />
        <View class="border" />
        <Label id="label" onClick="doClick">Hello, World</Label>
    </Window>
</Alloy>

    This is a quick Alloy version (tested) if you need a non-Alloy version just say and I will provide one.

    In this above example I use the fact that not providing positioning centers the views. For yuor needs you will need to wrap the example as follows;

    <View class="wrapper">
    <Window class="container">
    <ImageView class="avatar" />
</View>

    Then simply position wrapper as you require.

    This works by ignoring the jagged issue caused by providing a border radius that has a width and then overlaying a simple view with the border.
