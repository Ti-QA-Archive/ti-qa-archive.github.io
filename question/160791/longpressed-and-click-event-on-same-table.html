Longpressed and click event on same table

I need two event listeners on a table. If the 'longpressed' event fires I don't want the 'click' event to fire and vice verse. From other references I've seen that's supposed to be the behavior, but with the below code (cut down) it always fires both events when I press long on the screen. First the 'longpressed' and then the 'click' event. A quick touch correctly fires only the 'click' event.

How can I get only the 'longpressed' event to fire?

myTable = Ti.UI.createTableView({
    backgroundColor: ‘#fff’
});

var row = Ti.UI.createTableViewRow({
    className : 'row',
    touchEnabled : true,
});
.
.
.
tableData.push(row);

myTable.addEventListener('longpress', function(e) {
  Ti.API.info("table row " + e.index + " longpressed");
});

myTable.addEventListener('click', function(e) {
  Ti.API.info('Row clicked');
});
  • Ti 3.2.0GA
  • OSX Mavericks
  • Android 4.1

    Hi Marten :

    Try this event longclick.

    myTable.addEventListener('longclick', function(e) {
  Ti.API.info(" >>>>> longclick event fired <<<<<");
});

myTable.addEventListener('click', function(e) {
   Ti.API.info(" >>>>> click event fired <<<<<");
});
