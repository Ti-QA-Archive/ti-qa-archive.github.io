HTTPClient.open async has no effect?

I want to wait for my HTTP request to return with an auth token before trying to use it – but I can't get the async flag to open() to do anything.

A minimal example to demonstrate my problem:

var xhr = Ti.Network.createHTTPClient();
xhr.onload = function() {Ti.API.info('got response ' + this.status);}; 
Ti.API.info("before open");
xhr.open('GET', 'http://www.shishnet.org/', false);
Ti.API.info("before send");
xhr.send();
Ti.API.info("after send");

I/TiAPI   (14208): (kroll$2) [18,4831] before open
I/TiAPI   (14208): (kroll$2) [4,4840] before send
I/TiAPI   (14208): (kroll$2) [9,4849] after send
I/TiAPI   (14208): (kroll$2) [8,6783] got response 200

Shouldn't send() block, and so the output is "before send", "got response", "after send"?

This is using mobile SDK master from git.

I have tried writing it all asynchronously with layers and layers of callbacks calling other callbacks's callbacks, but it just ended up as a mess; being able to do a couple of things synchronously would make the code much cleaner.

Grepping through kitchensink, it doesn't look like the async flag is ever used, so I can't see an example of how it's supposed to work.
