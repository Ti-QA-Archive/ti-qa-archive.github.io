    pic.png-100x50px(normal)

  • Just as a follow up here. I actually wrote that post and put together the code for the test app. There are no 'theme conventions' used to create a theme for an app. In the marketplace, all that existed was a PSD file which I then sliced to the correct sizes for each of my components. There is no way to simply name an image something like 'header_image' and have it automatically be used as the header for the app. You have to create the images yourself, name them whatever you want, and then use those images in the creation of each component.

    Now as far as the orientation change making some of the custom components look skewed or stretched, the fix for that is to use the 'caps' for the custom images. backgroundTopCap and backgroundLeftCap can be used to keep the edges/corners of your images from stretching while the interior of your images scale accordingly. It works like 9-slice scaling for your custom images.
