Implement Google Map V2 in Android

I have create an Android and iphone application in which I have used the create map as:

var mapview = Titanium.map.createview();

I have not used the ti.map module for this which is newly proposed as Google Map V2. Also for this I have not used any key for that. Please Suggest, Is that require any API key? If yes, Is that require Google Map V1 key? it is working fine without any key.

When I am going to implement the same code with Google Map V2 with proper API key. I am using the ti.map module. On execution in device the application is getting crash, with giving error in DDMS as no class found for TiUIMapview. Presently I am using Titanium Mobile SDK 2.1.3. Please Suggest.

  • Hi

    The first thing you have to be aware of is that you CANNOT use the built-in Android maps Ti.Map for any NEW apps that you have not ALREADY obtained a Version 1 API key.

    Sorry for all the uppercase, but you (and others) should be aware of the current situation.

    Google has not STOPPED issuing version 1 API keys, this is how the built-in Android Maps support worked.

    A few things to note;

    • iOS apps do NOT require an API key of any sort from any version.
    • iOS apps can simply use Ti.Map for utilise Google Maps for devices prior to iOS 6 and Apple Maps for iOS 6 and higher.
    • iOS can run Google Maps on iOS 6, there is a paid for module in the marketplace at a reasonable cost to make that work.
    • Android apps already created and with an API key already issued will CONTINUE to work for some time (although how long is unknown, but think at least a year - most likely more [guessing])
    • Android apps that have NOT yet been issued a version 1 API key, will not work, these would be used with Ti.Map
    • Android apps that want to use version 2 of the Google maps API can start using this right now and full details can be found
      • Google Maps v2 Module info
      • Google Maps v2 for Android
    • A version 2 API key MUST be used with this module
    • Google Maps version 2 module MUST (according to the docs) be used with Titanium 3.0.2 or higher SDK.

    So if you are writing a cross platform app right now, you can and should be using Titanium SDK 3.0.2 (and higher) with iOS maps coming from Ti.Map (or the iOS Google Maps module) and Android should be using the Google Maps v2 module.

    You can achieve this by writing two CommonJS modules to handle the interactions and place the iOS one in the iPhone folder and the Android version in the Android folder in resources. Then when the app is compiled your device will be using the correct version.

    This may seem a little complicated - but it is very simple.

    Good luck.
