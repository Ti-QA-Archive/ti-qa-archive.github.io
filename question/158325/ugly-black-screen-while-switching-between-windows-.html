  • I assume you mean switching tabs.

    Make sure you have at least the latest 3.1.3 GA release:

    From the release notes:

    "TabGroup Loads All Windows Upon Start up

    In this Release for the iOS platform, all Windows that are added to a TabGroup get loaded and their context is invoked when the app launches to allow a smooth UI transition when tabs are changed. If you need to wait for a Window to appear on screen before invoking any context, use the Window's focus event."

    • so this seems to be what you're experiencing.

  • Hi Alex

    I just figured out the reason being - the tab that flashes black when switching loads the backgroundColor that is set in app.tss - Global theme. I fix this issue by setting the backgroundColor directly in markup file.

    app.tss

    "Window": {
    backgroundColor : "#95B9C7"
}

    home_tab.xml

    
<Alloy>
    <Window backgroundColor="#95B9C7"/> 

 </Alloy>

    Hope this helps

    I believe it is a Titanium Bug.
