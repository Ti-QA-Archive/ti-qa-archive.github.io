  • We have identified the issue and have created a/another ticket to fix it: https://jira.appcelerator.org/browse/TIMOB-17034.

    I'll be working on fixing it real soon and the fix is scheduled to be available in Titanium SDK 3.3.0 whenever that ships. You'll be able to preview the fix in an upcoming RC build.

    You may have some luck if you open the Xcode project, then go to the Build Phases and find the "Pre-Compile" script. Edit the call to "ti build" and add:

    --developer-name "Your developer name" --pp-uuid "your pp uuid"

    To get the correct developer name and pp uuid, run ti info -t ios and find the exact values. You may also need to specify the --distribution-name too just in case you do an app store or ad-hoc build from Xcode.

  • Hi,
    I had the same problem after upgrading my handset os to 7.0.4 (from 7.0.3 which was working perfectly).
    Here's what i did to get it working …

    Went to Run Configurations
    (clicked on) Selected the relevant Titanium iOS Device config
    clicked on sdk version
    did a refresh on certificates
    did a refresh on Provisioning Profiles
    clicked ok

    back on the configuration of the Titanium iOS Device - made sure the provisioning profile matched the app name (the profile matching the app common name)

    Hope this helps

  • Wrestled with this for hours. A combination of two things worked for me:
    1.) I was using developer certificates for a developer account that wasn't signed in as an account in Xcode (my clients account). I signed in to that account in xCode Preferences (Not sure if this made the difference or not, but it's something I did. May have been the next steps based on what's above.)
    2.) Created brand new ID certificate and mobile provisions through the Apple Dev Member Center and installed them (after deleting the old ones). Double clicked on both the ID and provision to install into Keychain and Xcode. Went into xCode Prefs > Accounts > View Details… and refreshed everything to be sure it was all in there. Then went into Titanium > Run Configuration > Titanium iOS Device, refreshed the panel and selected the new ID and certificates. Run. Now go back into and select your new Developer Profile on the project Build Settings, but don't select a Provisioning Profile (leave it as "None").

    It should now build without the red flag. It's probably something coming from Titanium regarding the certificates or provisions. Hope this works for you
