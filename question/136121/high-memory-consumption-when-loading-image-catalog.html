High memory consumption when loading image catalog

Application will crashed when loading catalog images in scroll-view. I am using titanium sdk 1.8.0.1 and average image sizes are around 35 to 45 KB. One scroll view has maximum of 12 images right now.

The blob data of image are got from local db of the device and those data attached to image view before adding into main view. Scroll view has 3 views attached initially when scrolling it remove left or right view and attach new view with images in order to reduce the load.

The database has 1000+ blob image data stored.
Please be advice us how to manage the high memory consumption and crash issue of this mobile app developed via Titanium studio with sdk version 1.8.0.1
