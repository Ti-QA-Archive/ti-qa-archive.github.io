Same app version crash on startup and runs on different pcs

This seems a very specific situation.
I'm working with a colleague and we are using Svn.
Our app is for both iOS and Android, but the following situation was triggered while working for Android.
We are using 3.2.2GA
There are no problems on compilation.

We both have the same version on our machine, I've confirmed that we both updated the project root, and not the app folder. Svn confirms it with a visual "no arrow >" feedback.

When I try to run or debug the app in my pc the app crashes.
ON MY PC, I've tried to:

  • uninstall the app from the android
  • clean project
  • delete project resources folder
  • restart titanium studio
  • force update to latest svn version
  • restart cellphone

and I did it all at once.

the app crashes on splash screen.

however, i gave the cellphone to my colleague and ask him to run the app once from his pc.
It has the same version and it seemed to work fine.

After this initial deployment, on his pc, I can come back to my pc and run/debug my application again without any problems.

I believe that if I clean the project with the steps I've described i will follow into the same problem again.

Since the app crashes on splash screen i have no error feedback.

Its like something is missing on the first deployment. I'm afraid to do the same steps on my colleague pc and we both lose a working version of the app. For the moment I must have a functional working environment running.

Any ideas what might cause similar behaviors? This is the second time it happens.

Thank you for your support once again

  • Try using the Monitor tool (included with the Android development tools) to monitor the device/emulator logs. It typically shows more details than the abbreviated logs you see in Studio and may show problems that aren't causing displayable runtime errors. If the error messages are unclear, post them here and someone might be able to help you interpret them.
