Orientation switches back to portrait on network call

Hi,
In my application, when I hold the iPhone device in landscape position, and then make a network call, the orientation switches back to portrait mode. I found a workaround in this Q&A section, which asked to manually add

Ti.UI.orientation = Titanium.Gesture.orientation;

But in this case, if you are in landscape mode and you make a network call, it'll try to go back to portrait mode until it hit's the line above, then switch back to landscape.
Is there a better solution to this? This happens only on iPhone. Android works fine.

Am using Ti 1.7.2, and iphone 4.3

Thanks in advance.

  • This issue got fixed by adding the 

    <orientation>Ti.UI.PORTRAIT</orientation>
<orientation>Ti.UI.LANDSCAPE_LEFT</orientation>
<orientation>Ti.UI.LANDSCAPE_RIGHT</orientation>

    to the tiapp.xml.

    Initially we were setting this orientation in app.js to fit one of our requirements (as per the solution given by appcelerator team). But for some reason, it was not recognized during the network call. Now its working fine.
