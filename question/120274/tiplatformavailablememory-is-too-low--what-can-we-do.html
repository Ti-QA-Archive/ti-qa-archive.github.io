  • Closing a window will release it from memory. Hiding will make it not visible, in which case its code will still be running in the background. A common way to "hide" a window is if you have a list of entities and you click on a row to open up a modal window showing the specific entity. You can still have the parent list window running in the background, which could be possibly refreshing every 60 seconds and could fire an event (caught by the child window) to update it. Hopefully that makes sense… If you don't need that sort of behavior then you can always close the current window when you open a new one.
