function CurrentPosicion() {
            this.coordenadas = {

function Data(){

Data.prototype.getData = function() {
        var position = myPosition.coordenadas;

module.exports = Data;

Well, there´s an error in var position = myPosition.coordenadas cause says me that myPosition is undefined. Why? In iPhone works…

  • There's NO global variable in CommonJS approach. I think this wont work with new Ti SDK in the future. Avoid to use global variable, pass it as parameter when you want it.

    Data.js

    Data.prototype.getData = function(myPosition) {
        var position = myPosition.coordenadas;

};

    app.js

    var myPosition = new CurrentPosicion();
var myData = new Data();
myData.getData(myPosition);

  • Marcos -

    For global objects, I create a globalObjects.js. I then use:

    
// globalObjects.js

exports.myGlobalVariable = ""; // make it available for consumption

    …inside of globalObjects.js, then I make sure to include that globalObjects.js in all .js files that need access to those variables. The variable is then accessed by:

    // mycode.js

globalObjects.myGlobalVariable = some_value;

    I don't know if this is the BEST solution, but so far it has worked for me.
