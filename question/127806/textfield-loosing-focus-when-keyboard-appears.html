  • I gave up in the end - the only way I found a real solution to this annoyng problem was to rewrite my code to use views inside a scrollView (with layout:'vertical') - this mimicked the tableview I had previously used and I was able to reference click events using children[n] to know which view was being accessed.

    Hope this helps anyone else who has been struggling with "Android TableViews: Keyboard causing textField to lose focus" issue!

    Has the added bonus of being waaaay faster (especially as 1.8.1 and v8 runs like treacle with complex tableViews).
