"titanium build --build-only --platform ios" is launching xcode and installing the archive even though it should not

Titanium Studio, build: 3.2.0.201312191547
Titanium SDK: 3.2.0 GA

after the latest update when you run "titanium build –build-only –platform ios", instead of just packaging the application code into .app and waiting for xcode to be run to create .ipa it is putting the archive into xcode to be uploaded. This is not the behavior it had before the update.

Has anybody else ran into this and know of a solution?

    • I guess I did not give the full command.  It seems to be happening when you set the target to dist-appstore. 

titanium build --build-only --platform ios --device-family universal --target dist-appstore --project-dir /code/ --distribution-name 'cert' --pp-uuid id

here are the results:

I know it would be difficult to test as a distribution certificate and mobile provisioning is required.

Originally when you ran the command it just created the app and then you could run xcode to create the ipa.  now it fires off xcode and shows the Organizer - Archives.  It seems to build correctly but is very annoying.
