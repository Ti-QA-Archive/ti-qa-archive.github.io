Alloy iOS 7 Device Status bar - iOS 6 SDK - orientation change

Hi,

I am using the latest 3.1.3 Titanium and using ALLOY.

My app is compiled using the iOS 6.1 SDK.

When I run it on an iOS 7 device the status bar appears as in iOS 6 and my window is at the correct position.

My apps root window is set to only show in portrait mode

index.tss

"#mainWindow": {
    exitOnClose: true,
    fullscreen: false,
    orientationModes:     [    Ti.UI.PORTRAIT,
                            Ti.UI.UPSIDE_PORTRAIT
                        ],    
}

When I change the app to a different window that has landscape orientation enabled it still displays correctly

textWindow.tss

"#textWindow" : {
  fullscreen: false,
  orientationModes:     [    Ti.UI.LANDSCAPE_LEFT,
                            Ti.UI.LANDSCAPE_RIGHT,
                               Ti.UI.PORTRAIT,
                            Ti.UI.UPSIDE_PORTRAIT
                        ],
}

If I then close that window (textWindow) and it returns to the root window (index) the orientation changes to portrait as that is the only orientation that window allows. The problem however is that the windows then moves under the Status bar by 20px

Now the problem is - If I set the statusbar style as suggested here
http://docs.appcelerator.com/titanium/latest/#!/guide/iOS_7_Migration_Guide-section-37533766_iOS7MigrationGuide-StatusBar

It does the same thing - if I add the top 20 value on iOS 7 devices it goes crazy and adds in 40 pixels on the top in some instances.

Please let me know what I can do!

Thanks
