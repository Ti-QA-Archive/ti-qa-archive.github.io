Inconsistencies with Alloy Model documentation

Not so much a question but hopefully this will save someone else some time and frustration.

I spent half a day pulling my hair out over this error

&#39;undefined&#39; is not an object (evaluating &#39;config.adapter&#39;) at alloy.js (line 16)

It turns out the docs for working with Alloy Models are wrong. These docs say that in your export.definitions there is an object labeled 'configuration', however this is wrong and upon further inspection of the examples that come with alloy, the object in the export.definitions that sets up your tables defaults, and storage should actually be 'config' not 'configuration'
