Unable to update Studio, script requires sudo?

os - mac osx 10.9.4

studio/sdk - 3.3.0.201407100905

  • There was a bug in Studio 3.3.0 related to sudo handling. Depending on your system's configuration, when you run sudo the first time (or every time) it will output a message warning you that using sudo is bad, your actions will be logged, etc. Standard *nix security warning stuff. Unfortunately, that confused Studio, as it thought sudo failed.

    To work around this, use sudo at the command line at least once to suppress that message. If your system is configured to show that warning every time you run sudo, you'll need a sysadmin's help to turn that off temporarily. Once sudo works silently (other than prompting you for the password), then you should be good to go.

    Matthias's command line technique would get the upgrade installed too, and is a perfectly valid way to handle the upgrade.

    You can also install one of the prerelease versions of Studio, as this bug has been address since release.
