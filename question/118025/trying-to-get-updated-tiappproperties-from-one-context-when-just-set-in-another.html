Trying to get updated Ti.App.Properties from one context when just set in another

Hi everybody,

I'm trying to build a mobile app (iphone) with four windows, and a tabgroup. Each window is a .js file and the app.js is the hub.

I've got two properties which I can modify in one of the contexts (a settings screen/window), but, I can't get the freshly modified property appear on another window.

However, when I re launch the app, the property appears modified.

Anyone any idea for a clean way to do so ? So basically, one window modifies the settings, the others display them.

    Xavier, I'd use application level events to pass the information between the windows. Better yet, avoid using the separate execution contexts of the individual windows (don't create them with the url:yourfile.js notation). My suspicion is that the properties are not actually serialized and stored synchronously so that you can't rely on them as an instant means to transfer information between window contexts.

    Oh, and what Paul suggested is good practice too.
