  • Are you trying to access Ti.Android.currentActivity.intent.data from within a CommonJS module?

    I've found that when the app is running, and another app resumes it via an intent, Titanium executes your app.js code from the beginning, but it "remembers" that the CommonJS modules have already been require()-ed, so it doesn't reload those. It seems that a stale reference to Ti.Android.currentActivity.intent.data (left over from the first time the app was launched) can get stuck in a CommonJS module.

    If you reference Ti.Android.currentActivity.intent.data from within app.js, it works.
