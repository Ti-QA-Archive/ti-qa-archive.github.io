  • This can occur for a variety of reasons:

    • the app wasn't actually signed properly
    • a file inside the app was signed properly
    • there is already an installation with the same appID installed on the device/emulator that was signed differently (this is a big one when you go from testing locally, and you pull your signed install from the market the first time)
