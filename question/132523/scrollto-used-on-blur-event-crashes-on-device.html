scrollTo used on blur event crashes on device

Hi, I have a tabbed application for iOS.
One of the windows in the stack is a scrollview that contains a table view. In the main view of the app there is a table and when you click a row in the table it goes to this scrollview. As the scrollview opens it populates its table view with a number of rows. So the table could have 2 rows and other time it could have 20 rows. The contents of the scrollview live in a view container so the scrollview can extend its height as needed. All that is ok.

My problem is that if you scroll down, go back to the main view and select a different row it opens the scroll view and scrolls it to the point where the previous one was left. I've tried adding scrollTo(0,0) in the open event as it opens the view but it starts scrolling to the previous point and then to 0, so it makes an ugly movement.

The only thing I managed to get working was adding the scrollTo(0,0) in the blur event when the scrollview is left while going back to the main view. That works great and as it closes you can see how it scrolls back to the top. Except that is crashes on the device…

I have the latest SDK, XCode and all that.

Any hints? I though I could detect the back button and try how the the scrollto 0 behaves there, but I should then create a custom button so I can attach the event listener, and I'm afraid it might crash the same way…

  • Hi, I created a reduced example of my app and I've been able to reproduce the problem also there. It is either a bug in my code or in Titanium.

    The problem is that it only happens on the device, not the emulator. To reproduce it you must enter a list from the main view, scroll down then leave the view with the back button and enter the other view. Usually when opening the second view it crashes, if not the first time then the next one. No crash if you don't scroll down.

    The line that crashes the device is: 

    scrollView.scrollTo(0,0)

    When used inside the blur event of the window.
