Android Fragment like Master-Detail

Hi!

I would like to ask for some tips on a Master-Detail app on Android. I've already checked the new master-detail app template in TiStudio, but as far as I can see it doesn't perfectly mirror Androids fragment based approach (which is no problem, I am willing to implement the behaviour by myself).
But one big question is how to handle the Android back button in such master-detail apps. On Android when adding a new fragment to the stack, it is also added to the back stack, i.e. something like this:

Master -> Detail, click button in Detail fragment => switch to Detail2 fragment. Back button would go back to Detail fragment.

I would like to know how others handle this using Titanium, are there some best practices for a "back stack"-like behaviour? Currently my app would just exit, as I am still in the main "exit on close" window, just switched some views.

Thanks for your help.

    • Sorry, I think my question wasn't clear enough.
      I am creating a tablet app, built on top of the master detail template in TiStudio. It has a master navigation view on the left and a detail view on the right. Opening a new window everytime the user navigates deeper in the hierarchy tree wouldn't be very tablet-like. I want a deeper item to replace the detail view (maybe a nice slide transition or something), but the nav view should stay in place. Just like when using android fragments. But then I have the problem with the back button.

      But I think I found a solution for this special app, I will just keep a manual array of views (like a manual back stack of fragments) for the heavyweight window and on hardware back button press will slide out the last item to the right.

      — commented March 26th 2012 by Tobias Reike

    • Ah, yes.

      You can create heavyweight window and listen to "android:back" event. In that listener you can do what you described, creating stack of views shouldn't be too hard.
