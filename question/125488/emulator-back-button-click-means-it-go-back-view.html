  • Typically, new windows are added to a stack. Tapping the back button pops the top-most window off that stack, so that the next one down becomes the active window. You can override this behavior by "hijacking" the back button.

    You can set an event listener on the android:back event on a "heavyweight" window. In that event handler, you'd define the custom functionality you want. Once you do, you'll need to manually handle the back button behavior for all your windows so that actions are consistent within your app. Be aware that users expect certain functionality from the back button.
