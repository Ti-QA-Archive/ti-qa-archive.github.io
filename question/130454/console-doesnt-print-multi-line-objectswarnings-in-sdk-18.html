Console doesn't print multi-line objects/warnings in SDK 1.8

Since upgrading to SDK 1.8, the console only prints the first line of output that should span multiple lines.

When printing an object, such as:

var obj = {
thing1: 1,
thing2: 2,
thing3: 3
};

Ti.API.info(obj);

The output is: 

[INFO] {

I've worked around this by creating a custom function to print objects.

More annoyingly, this problem applies to warnings. I'm currently getting an exception that only displays:

[WARN] Exception in event callback. {

This isn't terribly helpful.

Is anyone else having this problem?

Anyone know of a fix?

  • Accepted Answer

    That's because log level was changed in new TS.

    To grab full error message back: Run > Run Configurations > Change log level to Trace or Debug.

    If it does not help, you can run/debug directly using XCODE: YourProject > build > iphone > YourProject.xcodeproj
