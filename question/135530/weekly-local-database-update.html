Weekly local database update

Is it possible to create a weekly db update, instead of providing a weekly update to the app.

So… every week, the local db would be overwritten with the new content of a remote mySQL db.

I am not concerned about how to actually get the contents of the remote db and overwrite the contents of the local db, as I currently have this working (but it requires the user to click for the update). I am only concerned about the ability to force this request every week, so that it requires NO user input. They just see the updates seemlessly.

    Andrew,
    Just set a Ti.App.Properties when you do the update and set the current date, then on resume of the app, do a check to see if its been 7 days. If it has, start the update. You may need to create a view that sits on top of the other windows that will block the user from touching the app during the update. You could also possibly do the update as a background service on the pause event so the user never sees it, but then they could be using out of date data during their current session. Does this make sense?
