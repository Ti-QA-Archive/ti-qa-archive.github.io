Identifying camera orientation

        var capturedImg;  

        Titanium.Media.showCamera({ 
            success : function(event) {                 
                /* Holds the captured image */
                capturedImg = event.media;                  
                /* Condition to check the selected media */
                if (event.mediaType == Ti.Media.MEDIA_TYPE_PHOTO) {
                          // passed to a function where it shows it in imageView
                        ImageViewWindow.init(capturedImg);
               }
            },
            cancel : function() {
                /* While cancellation of the process */

            },
            error : function(error) {
                /* called when there's an error */
                                }
        });
    }

    Hi Karanveer,

    I'm assuming you are using running the app on Android. Which TiSDK version are you using?
    This issue has been reported to be happening on some Android devices. We have a Jira bug that has been fixed: http://jira.appcelerator.org/browse/TIMOB-3887

    The fix it's included into latest 3.x branch (still not the GA but you can download it from here: http://builds.appcelerator.com.s3.amazonaws.com/index.html choosing 3.0 or Master from the branch menu), you might also have to set the 'autorotate:true' property in the ImageView (see Jira bug comments too).
