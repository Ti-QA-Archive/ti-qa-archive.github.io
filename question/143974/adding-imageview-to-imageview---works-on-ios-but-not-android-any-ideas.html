Adding imageview to imageview - works on iOS but not Android, any ideas?

Hi all,

I'm building an app where I've put a camera button inside the photo's imageview.

This works perfectly on the iPhone simulator but not on an actual Android handset

Here's the code

var photoField = Ti.UI.createImageView({
top : "20%",
left : "5%",
width : pWidth 0.25,
height : pWidth 0.25,
backgroundColor : "#fff",
borderWidth : 2,
borderColor : "#e29300",
});

var photoButton = Ti.UI.createImageView({
width : Ti.UI.SIZE,
height : Ti.UI.SIZE,
image : "/icons/camera@2x.png",
zIndex : 1,
});

photoField.add(photoButton);

  • Accepted Answer

    There are lots of places where there is not complete parity between iOS and android. Most of these are undocumented, so they can be difficult to deal with.

    You might need to be creative here. How about using a standard View and setting its background image to the photo? Then add your photo button to that view?

  • We have found a number of significant issues bringing up the Android version of a released iOS app where we are layering buttons onto ImageViews for a camera-based app where we displayed the resulting image and wished to display an edit button on the imageView. One cannot add a button to an ImageView on Android and have the button show up. Adding a button to a view will work.
    Unfortunately we are not amused by how creative one has to be to code around the incompatabilities between the iOS and Android SDK implimentations - it gets very painful very quickly.
