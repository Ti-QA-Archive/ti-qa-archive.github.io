Navigation in in a <WebView>

SDK 3.3.0GA / iOS 7.1.2

I have an html page with an iframe. The iframe src is a non-local website with a login form. If I view the html page in desktop browsers including Safari, the website shows and I can login and use it normally. But if I load the html from a local asset file in the app, the website shows but submitting the login form just reloads the page.

When I submit the login form, I see two traces: One that would be the intended logged-in page and one that is just the login form again.

[DEBUG] New scheme: <NSMutableURLRequest: 0x17d65700> { URL: https://website/correct-logged-in-page }
[DEBUG] New scheme: <NSMutableURLRequest: 0x17e8bf40> { URL: https://website/login-form }

How can I prevent the iframe from immediately reloading the login form?
