Versioning the android app from Tiapp.xml fails, why?

Hi again. Another question from the newbie; I need to make an update to Android Market and I have followed the guidelines from the programming guide, which says that versioning can and should be done from the Tiapp.xml file instead of making a custon manifest.xml file. Below is what I have included in my Tiapp.xml, but on build time it goes back to version 1, and my .apk file is dismissed upon upload because the version number is wrong. How can I change that? (I am not familiar with making a custom manifest file if that is what is needed…) This is what I have been writing in Tiapp. xml; 

  <android xmlns:android="http://schemas.android.com/apk/res/android">
    <manifest>
        <application>
            android:versionCode="2"
        android:versionName="2"
        </application>     
    </manifest>
    </android>

I would be most happy to have a helping hand on this (as well ;-))
Many thanks,
Dearest Louise

  • Hi louise you made a mistake here. You don't need the custom manifest you can do it in the tiapp.xml but u've put it in the wrong place.

    following works fine:

    <android xmlns:android="http://schemas.android.com/apk/res/android">
        <manifest android:versionCode="14" android:versionName="1.4">
