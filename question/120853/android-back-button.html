Android Back Button

My app uses namespaces and each is called to load different windows.

When a window is opened it then closes the previous window. It is a circular app so it is done this way to resolve a large window stack.

I have created a toolbar with a back button. Details stored in an array. This works really well on both iPhone and Android.

The problem is when I press the actual back button on the phone, it loads a blank screen with a heading.

I am going round in circles with adding an event listener for the 'android:back' it doesn't work like this…

if (Ti.Platform.osname == 'android')
{
    Ti.App.addEventListener('android:back', function() 
    {
        mainNS.event.loadBackScreenHandler({});
    });
};

but it does work

Ti.App.addEventListener('android:back',mainNS.event.loadBackScreenHandler({}));

Well I say it works but because the back button closes the window before running the event listener and the previous window does not exists then I get a JavaScript error because it cannot find the function to load the new window.

Any ideas if I have explained this correctly…

  • In your example that doesn't work:

    Ti.App.addEventListener('android:back', mainNS.event.loadBackScreenHandler() );

    you are immediately executing the mainNS.event.loadBackScreenHandler function.

    Omit the parens so mainNS.event.loadBackScreenHandler is run only when the event fires:

    Ti.App.addEventListener('android:back', mainNS.event.loadBackScreenHandler );
