  • Hello John Kortis,

    This problem occurs for android SDK folder structure. Recently, Android SDK folder structure was changed.

    Please follow my instructions.

    For MAC create symbolic link 

    
ln -s /Applications/Android-sdk/build-tools/17.0.0/aapt aapt ln -s /Applications/Android-sdk/build-tools/17.0.0/dx dx

    For Windows

    Simply copied all the files from build-tool/17.0.0/ to platform-tools or build-tool/18.0.1/ to platform-tools

    And for more help please visit following link: -
    http://developer.appcelerator.com/question/152497/titanium-sdk-310-error-typeerror-argument-of-type-nonetype-is-not-iterable-on-building-android-app#comment-175782
