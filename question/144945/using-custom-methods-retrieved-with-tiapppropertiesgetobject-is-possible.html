Using custom methods retrieved with Ti.App.Properties.getObject(); is possible?

Hi there,

In a general object like this:

function myObject()
{
this.myVariable = 5;

this.MyMethod = function(){}
}

When i use the Ti.App.Properties.setObject(); to store the object and then i retrieve it with and Ti.App.Properties.getObject(); i can get acces to the variable, but i can't execute the method.

It says Object [object Object] has no method 'MyMethod'.

What am i doing wrong?

    I can't think of any reason that this should ever be necessary …. ever. If you want to store executable code, create a javascript file and write your code in there. You can then require that file and execute that code. If you need to store values, then use the Properties API. The code you need to run on those variables should still be stored in a javascript file.

  • Hi

    Ti.App.Properties... is designed to save a snapshot of an object variable at the time they were stored and then retrieve them later on, but not functions.

    You are mistaking the method setObject as an ability to cache functionality, when it is designed to store variable objects instead.

    This is suitable as an object;

    var obj = [
    {
        firstname: 'Malcolm', 
        lastname: 'Hollingsworth',
        appearance: {
            general: 'scruffy',
            hair: 'balding'
        }
    },
    {
        firstname: 'Someone', 
        lastname: 'else',
        appearance: {
            general: 'neat',
            hair: 'blonde'
        }
    }
];
Ti.App.Properties.setObject(obj, {});

    You should be able to store any JSON data structure with any valid nesting of array and nodes, but not functiond/method.
