Clash between iPhone-4 and iPhone-5 UI design

I want to design a UI commonly for both iPhone4 and iPhone5. I know that iPhone 5 display resolution is 1136 x 640 pixels, measuring 4 inches diagonally screen size and iPhone 4 display resolution is 640 x 960 pixels, measuring 3.5 inches diagonally screen size. In simulator (with iOS SDK 7 and Titanium 3.1.3 ), i got screen size 320 x 480 for 3.5 inches screen(Display type : Retina) and 320 x 568 for 4 inches screen (Display type : Retina & Tall) . Now my problem is when i design a ui/window like below code, for iPhone5/4' screen size, it is perfect but for iPhone4/3.5' screen size, login button does not appear because it exceeds the window size. Now how can i make a common UI for both iPhone4 and iPhone5 to see the every component of a window/UI properly that belongs to the window/UI or should i make them differently ? is it possible to use a scroll view when screen size is 3.5' such that i can see the login button with scrolling ohttp://www.appcelerator.com/r any better suggestion ? please help me out.
