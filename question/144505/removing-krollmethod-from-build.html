Removing @Kroll.method from build

I can't seem to figure this annoying little thing out. When I create a method that Titanium can see…

@Kroll.method
public void register_Receiver(){
    //Some code
}

…and then delete it, Titanium returns this error…

 [javac] Compiling 3 source files to C:\Users\LocalUser\Titanium_Studio_Workspace\android_bluetooth\build\classes
    [javac] C:\Users\LocalUser\Titanium_Studio_Workspace\android_bluetooth\build\generated\java\com\dearinggroup\adbt\AndroidBluetoothModulePrototype.java:215: cannot find symbol
    [javac] symbol  : method register_Receiver()
    [javac] location: class com.dearinggroup.adbt.AndroidBluetoothModule
    [javac]         proxy.register_Receiver();
    [javac]              ^
    [javac] 1 error

It's like the module is still looking for the method. But I don't want the method anymore. Anyway to fix this?

    • To be precise, the problem is not that the method is still present. It's that your method is still being called somewhere else in the code.

      So when the compiler tries to process that call, it cannot find the method (since you deleted it).

      My best suggestion is to do a full scope search for the method name and delete every line referencing this method.
