false data return on tableViewRow click event

Hi,

I'm using a Collection to fill my tableView. Here is the code: 

<Alloy>
    <Collection src="partners" />
    <Window class="container">
        <View>
            <TableView class="table" id="table" dataCollection="partners" onClick="selectPartner"  allowsSelection="true">
                <TableViewRow class="tableRow" id="rowId" rowId="{tidPartner}">
                    <ImageView class="imageRow" id="image" image="{imageUrl}" />
                    <Label class="cellLabel" id="museumName" text="{name}" />
                </TableViewRow>
            </TableView>
        </View>
    </Window>
</Alloy>

In my controller, I have the code: 

function selectPartner(e) {
    console.log(e.rowData);
    console.log(e.section.rows[e.index]);
    console.log(e.section.rows[e.index].rowId);
    var res = parseInt(e.section.rows[e.index].rowId);
    partnerID = res;
    console.log("partner selected: " + res);
    callEventChangePartner();
}

The goal is to get the id of the selected "partners". The id is stored in "rowId" property of the tableViewRow.

Sometimes, when I run for the first time the application, e.rowData return the last row and not the clicked row. On the second click, the rowData gives the correct data. I have also tried to get the rowId from another waye.section.rows[e.index].rowId but the same problem occurs.

I have two similar windows that contains a tableViewRow and both have this problem.

What is wrong with the code?

Thank you.
