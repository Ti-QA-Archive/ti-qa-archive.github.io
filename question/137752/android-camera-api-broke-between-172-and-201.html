Android camera API broke between 1.7.2 and 2.0.1+

I'm seeing a consistent crash when using the camera on Android:

  • Install your application by downloading the .apk (not by deploying to the Android device via Eclipse)
  • The first time you use the camera, after you've taken the picture, as soon as you hit "Save", the app will crash before your success callback is called

Once you restart the app the camera save function works fine - ie. the crash only occurs on first use of camera.

To isolate this I created the simplest possible app that only fires up the camera and takes a picture. The camera code is taken directly from KitchenSink.

The app works fine when built using the 1.7.2 SDK, and crashes on first use when builg with the 2.0.1 or 2.0.2 SDK.

So it seems somewhere between 1.7.2 and 2.0.1 the camera code for Android broke.

Any ideas as to how I'd go about getting this fixed? Or reporting it to Appcelerator?
