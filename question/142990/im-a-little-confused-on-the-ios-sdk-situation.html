    No, always work backwards. If you want to support iOS6 and iPhone 5, it will require that you have the proper Xcode and SDKs installed. Then you can work backwards compatibility up to a certain extent. Currently, if you are building for iOS 6, then the minimum required SDK is 4.3 I believe.

    Build for the latest release, and test on earlier version of hardware/software

    • You're saying 3G and 4G, but the device has nothing to do with it. Its the version of the OS installed on those devices that you need to support. For instance, you'll be able to support a 3G device running 4.3, but not 4.1.

      Everything has a version, so by '4.2.1' I assume you mean Xcode, in which case the answer is no. Again, the only way to support iOS 6 and the iPhone 5 is with Xcode 4.5 and Titanium SDK 2.1.3RC. Grab the latest Xcode and Titanium SDK and you will have a minimum supported version of iOS 4.3
