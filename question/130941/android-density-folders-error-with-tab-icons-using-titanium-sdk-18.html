Android Density folders error with tab icons using Titanium SDK 1.8

Hi,

I'm developing a app for iPhone and Android and have had issues using the different density folders with the 1.8 SDK. The images aren't copied to the build -> android -> images folder.

I've had tried using both the high/medium/low folder structure and the res-long-land-hdpi etc structure and still the images don't work.

I have placed &lt;supports-screens android:anyDensity=&quot;true&quot;&#x2F;&gt; in the tiapp file and have tried writing the path to the image in various different ways, but still no luck. I wish to use the images for different tab icons.

Is this a bug in the 1.8 SDK? I have downloaded one of the 1.9 SDK versions from the master branch and the icons worked as expected using their different densities. I would like to continue to use the 1.8 SDK as the 1.9 version I downloaded seemed to make the app less stable.

Any help would be appreciated!

Thanks!

  • Hi Lisa.. i do this approach to manually check for density and output specific icon

    save it as "filename.js"

    exports.iconMenu = function(menu,name){
    switch(Titanium.Platform.displayCaps.dpi){
        case 120:    //low
            menu.setIcon('/menuicon/' + name + '_25x25.png');
        break;
        case 160:    //medium
            menu.setIcon('/menuicon/' + name + '_60x60.png');
        break;
        case 240:    //high
            menu.setIcon('/menuicon/' + name + '_110x110.png');
        break;
    }
}

    to use this you will need to require the "filename.js"..

    var win = Ti.UI.createWindow({
    fullscreen:false
});
win.activity.onCreateOptionsMenu = function(e) {
        var menu = e.menu;
        var menuButton = menu.add({ title: "Sample Menu"});
        require("filename").iconMenu(menuButton,'menu_icon_file');
};
