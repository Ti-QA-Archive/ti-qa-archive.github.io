Keep animating when app is loading other content

Hi all, I've just spent quite some time getting a view to continuously animate (as a loader, instead of using ActivityIndicator), it looks perfect. But only to now find out that when anything is loading on the app, any animations freeze whilst other content is loaded.

Is there a way to keep other functions running whilst the app is doing something else? it works if I set the duration on the animate to something like 10000 so the first animation works, its just the repeat calls that dont. Like as soon as any other content is loading, all timeouts or function calls just stop.

My loader: (in an module export)

function loadingBall(){
    var _loading = null;
    var _loading2D = null;
    _loading = Ti.UI.createImageView({ 
        width:50, height:50, image:'images/loading.png', zIndex:1
    });

    var _animStep = 10;
    var _animCurStep = 0;

    var _transform = Ti.UI.create2DMatrix();
    var _anim = Ti.UI.createAnimation({ duration:30 });

    function rotate(){
        _transform = _transform.rotate(_animStep);
        _anim.transform = _transform;
        _loading.animate(_anim);
        setTimeout(function(){ rotate(); },30);
    }

    rotate();

    _loadWrap.add(_loading);
    return _loadWrap;
}

Where the loader is created:

var _loadData = function(){
    var _loadView = new ui.loading();
    _self.add(_loadView);

    // All other content for this window is loaded here
    // The loading view animates once but stops when other
    // views etc start being added. If I use setTimeout here before
    // loading content I can allow the animation to run
    // how ever long the timeout is set for, but still, as soon as
    // the timeout finishes and content starts loading the animation stops

    _self.remove(_loadView);
    _loaded = true;
}
