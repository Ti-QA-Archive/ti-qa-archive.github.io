Android Soft Keyboard Resizing View

On android when I use a Textfield, after the user focuses on the field it brings up the soft keyboard. The problem is when the soft keyboard comes up it resizes all of the layout items on my screen. I have attempted to solve the issue by putting the following into tiapp.xml…

    <android xmlns:android="http://schemas.android.com/apk/res/android">
          <manifest>
              <activity android:windowSoftInputMode="adjustPan"
                        android:name="Broken{(-_-)}App">

              </activity>

          </manifest>
    </android>

but this didn't solve the issue. Does anyone know of a fix for this that will cause the keyboard to overlay the layout, like it does on iOS, rather than resize everything? I would imagine it is a simple flag that needs to be set, I just cant seem to find it…

    • For anyone using alloy having an issue getting the current window, what I did was in each of the windows that I wanted this to occur, I accessed a view that was a direct child of the window and used the getParent() method to retrieve a reference to the window. Here is what I did.

      test.xml

      <Alloy>
    <Window class="container">
        <View id="entireScreen">
        </View>
    </Window>
</Alloy>

      text.js

      if(Ti.Platform.osname == "android"){
    var softInput = Ti.UI.Android.SOFT_INPUT_ADJUST_PAN;
    $.entireScreen.getParent().windowSoftInputMode = softInput;
}

      There may be a cleaner way to do it, but Ti.UI.currentWindow was returning null and I got frustrated and just did it like this.
