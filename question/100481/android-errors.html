    Colton

    You have broken the cardinal rule of the Q&A - you haven't mentioned the Titanium Android SDK version you are using! ;-p What is it?

    So, interestingly, I encountered this issue myself today. It seems that, if you delete everything from the build&#x2F;android directory (including hidden files), when using Titanium SDK 1.6.0, the above exception is generated and the app will not launch.

    The workaround that worked for me was to do the following:

    • delete everything from build&#x2F;android, including hidden files
    • launch the app using the latest 1.5.X CI Build (currently at version 1.5.2)
    • after app successfully launches, delete everything from the build&#x2F;android subdirectories, namely the contents of the assets, bin, gen, lib, res and src directories.
    • launch app using 1.6.X CI Build (currently at version 1.6.0)

    The third step is not mandatory, but explains the correct procedure of how to delete the build files when using Ti SDK 1.6.0 (while this bug/issue exists with it).

    Come to think of it, I have found this issue when using previous Ti SDKs, where all the build directories and files were not generated properly but, for some reason, it does not exist in 1.5.2 at the moment but has been reintroduced in 1.6.0.

    So, for the core devs, to reproduce this issue, delete everything from the build&#x2F;android directory, including hidden files, and boot the app using 1.6.0.

    Hope this works for you, Colton.
