Selectively hide actionbar on windows - Android

Hi,

Ti SDK 3.3.0 GA, OSX 10.9.4 - moving from Holo themes to AppCompat themes and subsequent "cya" to navBarHidden property.

I've been doing a lot of reading trying to get this working. The best link I found was —> JIRA ticket. I have managed to amend my app that has a custom AppCompat theme on my action bar for my tabs and all other windows without any action bar by setting the windows I want to have no action bar as "opacitiy:1.0" and adding the "NoTitle" theme to the TiTranslucentActivity per the link. (My custom theme has been set at the application level)

This screams work around to me and I would like to know if there is anyone with a true method of determining on an app whether a given window will use a theme with no Action bar or not. Setting the opacity even to 1.0 gives you some wierd results like text fields having text less than the opacity level of 1.0.

The final comment in the link suggested that each window has a theme property which I cannot find.

Kind regards,
Brad.

    Currently, you can use the opacity workaround or just call actionbar.hide();
