Do I have to develop for Retina from the beginning of a project?

WARNING: I have VERY limited knowledge of dev!

We are facing a problem with the dev now concerning the Retina display. It seems the team has been developing for the old (standard) resoltuion. Do we need to be developing in a "Retina" mode? Really, I am not familiar with the Titanium software, and the lead dev guy doesn't know if we should be doing something different.

Help please! Thanks!

  • 'Retina mode' at the base level are graphic assets that are 2x as big.

    So, if you have a button graphic that is 300x50 named btnImage.png, its retina pairing would be btnImage@2x.png which would be 600x100.

    Easiest method in hindsight is to develop the larger images and work down, as you can upscale an image and get better quality out of it.

    • If the retina is missing, the low res is used on retina device.
      If the lowres is missing, no image is shown on a non-retina device.

      You can ship your app, with the @2x named as low-res, so that both devices can find use them and save space. The problem with this, is that non-retina devices are a few gen back, so it takes a lot of work for them to process the hi-res graphics.

      Fonts are scaled automagically. As for a chart to compare point to mobile, well, never found one I could trust reliably.
