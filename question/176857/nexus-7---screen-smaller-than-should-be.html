Nexus 7 - Screen smaller than should be?

Work just got me a Nexus 7 tablet to use for development. I took one of the small nothing apps that I have and ran it on the tablet and the screen sizes are off. I am using some code to give me the screen height and width so I can then do various positioning.

var pHeight = Titanium.Platform.displayCaps.platformHeight; var pWidth = Titanium.Platform.displayCaps.platformWidth;

These return values that they should. 1824x1200. But when I try and do positioning with these values, everything is way off.

The variable _logoMarginLeft puts the logo image in the center of the screen on my other test device, but the new one is far to the right. Doing some testing, it looks like the screen is only giving me approx 600 pixels in width.

Am I missing something here? 

var pHeight = Titanium.Platform.displayCaps.platformHeight;
var pWidth = Titanium.Platform.displayCaps.platformWidth;

//create the window 
var win = Titanium.UI.createWindow({
    width: pWidth, 
    height: pHeight, 
    top: 0, 
    left: 0,
    backgroundColor: Ti.App.Properties.getString("backgroundColor")
});

//we will give the logo a left margin so it centers neatly within our page 
Ti.App.Properties.setString("_logoMarginLeft", (pWidth / 2) - 248);
var _logoMarginLeft = Ti.App.Properties.getString("_logoMarginLeft");

var logo = Titanium.UI.createImageView({ 
    image: '/images/Logo.jpg', 
    width: 248,
    height: 53,
    left: _logoMarginLeft,
    top: 50 
});

win.add(logo);

//finally, open the window to launch the app 
win.open();

  • That's because by default, units are density-independent pixels (DIP), rather than pixels. If you want to use pixels, either specify it as a string… width:&#39;248px&#39; or change ti.ui.defaultunit.
