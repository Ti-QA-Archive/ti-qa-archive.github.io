Retrieve production keystore created automatically by Titanium

I published an app on Google Play using the apk created by Titanium in the /build/andoid/bin folder.

I didn't create a keystore for production (great mistake!). I think my app is now signed with an automatic keystore created my Titanium.

Now I need to create a google maps api and to do that I need a SHA1 fingerprint.

But to get the SHA1 (for distribution not for debug) I need the production keystore.

Is there a way to retreive it?

I don't know how to do :(

Thanks!!

  • If you did not use the distribution option in Titanium Studio to package and sign the app but just uploaded the apk from within the /build directory then it is signed with the debug keystore. You can retrieve this by selecting "Preferences" in Titanium Studio and selecting "Android" > "Build" (There you find the location of "default debug keystore" as well as its SHA1 and MD5 fingerprints)

    My advice: Follow the distribution guide here: Distributing Android apps

    • create a fresh, new production keystore
    • build and sign your app with that
    • upload it as a new app to google play
