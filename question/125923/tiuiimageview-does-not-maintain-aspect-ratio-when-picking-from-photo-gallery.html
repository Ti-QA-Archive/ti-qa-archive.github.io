Ti.UI.ImageView does not maintain aspect ratio when picking from Photo Gallery

If you set the image property of an ImageView to a remote photo on the web, then the image is scaled to fit in the image view whilst maintaining the aspect ratio, even if the aspect ratio of the image view is different to the photo. (Equivalent of iOS Aspect Fit).

However, if you pick an image from the photo gallery, and use the event.media to set the imageView's image property, the image aspect ratio is not maintained.

You can see this in the Kitchen Sink > Phone > Photo Gallery example if you modify the example to set allowEditing to false so that it does not crop photos to be square.

Anyone managed to find a workaround to make this work?

Thanks
Martin

  • When loading a image from gallery, original image size is in event.media.width and event.media.height. You can easily determine image aspect ratio with this properties.

    If you want to avoid that jump, use a temporal imageview object to resize the image:

    1, Selected image is in event.media

    2, Copy image to tmpImageView

    3, Resize tmpImageView

    4, Load tmpImageView image on final imageView
