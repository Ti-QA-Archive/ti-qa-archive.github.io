Table view rows become separated after upgrade to Titanium 1.8.5

I just upgraded to the latest Titanium SDK 1.8.5 and table views in my application now looks differently: every row looks like a separate table view. I'm wondering if someone else has experienced that and how do I fix it?

Thank you in advance,
Olga

  • Sure, the SDK version is 1.8.2, sorry for the confusion. Apparently, I've found out what was the issue. Just in case someone else will bump into it. In the old sdk (I think that was 1.6.5) this code was returning one tableview with all rows "attached" to each other:

    
var row1 = Ti.UI.createTableViewRow({title: 'Row 1', hasChild: true, header: null});

var row2 = Ti.UI.createTableViewRow({title: 'Row 2', hasChild: true, header: null});

var tableview = Ti.UI.createTableView({
        scrollable: false,
        data: [row1, row2],
        top: 10, 
        style: Titanium.UI.iPhone.TableViewStyle.GROUPED,
        backgroundColor: 'transparent'
});

    But with 1.8.2 the rows looked like they belong to different tableviews. All I had to do was to remove header: null and it worked as desired again.
