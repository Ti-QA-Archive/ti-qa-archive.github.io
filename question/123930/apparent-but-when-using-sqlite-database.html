Apparent but when using sqlite database

I'm building a desktop app and using a sqlite DB that is prebuilt with some default data.

The table, "info" has 3 fields: "tname", "twidth", "theight"

I originally declared "twidth" & "theight" as NUMERIC fields. When I launched the app & made my db.execute call I got nothing. I don't get an error in the log but all the javascript after that call didn't execute. After quite a few failed attempts (recreating DB from scratch on assumption it was corrupt, renaming tables & fields in case I'd corrupted a namespace somewhere, etc.) I tried changing the field declarations from NUMERIC to TEXT and now the app can read the DB without problem.

I'm not sure what's going on but it seems that Titanium Desktop doesn't play well with sqlite NUMERIC fields.

I'm running Titanium Desktop 1.1 with a sqlite3 database.

  • For an sqlite database NUMERIC is a type affinity used for compatibility and not a built in storage class. Try changing the field to either INTEGER if you are using signed integers or REAL if you need them to be floating point values.
