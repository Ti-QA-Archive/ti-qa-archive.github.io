setContentInsets Reset on TextField Focus

Hi,

I am having a problem using setContentInsets in iOS7.

I have a window that is extended under the translucent NavBar and Status Bar.

I set the cententOffset so that my TableView content does not start behind the NavBar, but the tableview can still be scrolled and can be seen through the translucent NavBar.

This all works as expected until a text field is focused. As soon as the textfield gains focus the contentOffset is reset and the top of the tableview disappears behind the NavBar.

According to other questions this seems to be happening with autoAdjustScrollViewInsets as well:
iOS7 autoAdjustScrollViewInsets gets overwritten when focusing textfield

I was wondering whether anyone else has experienced this issue or has found any work arounds.

Any help would be greatly appreciated.

  • iOS SDK: 7.1
  • Titanium SDK: 3.2.2 GA
