  • Hi Yoav,

    Let me give you a quick refresher on Titanium Android Module development to build/run/install (on device) module. To begin with, there are different targets available in build.xml to perform all the activities required w.r.t module, but we require four main targets to work upon:

    a. build (creates distribution file of your module)

    b. run.emulator (runs emulator)

    c. run (creates a sample application for your module using example/app.js and installs and runs it in emulator)

    d. install (creates a sample application for your module using example/app.js and installs it on device)

    Step 1: In order to first test your module run the basic target, dist (this target is also default) using build.xml -> Run As -> Ant Build

    Step 2: If this target gets executed without any issue, run run.emulator target to start emulator using build.xml -> Run As -> Ant Build… and selecting run.emulator

    Step 3: After emulator is started, run run target using build.xml -> Run As -> Ant Build… and selecting run

    Step 4: if you are able to see your application and test your application then run the final target after connecting your device to install and test on your device using build.xml -> Run As -> Ant Build… and selecting install

    while running these targets do ensure only there targets are selected.

    In your case it seems there is some configuration issue in setting up module development environment on Windows machine. It would be helpful if you can provide separate logs of eclipse environment after executing dist and run targets separately.
