Finding js files in Resource dir

I have a problem where something works fine in the iOS simulator but fails when running in an app. I am attempting to detect some .js files within a dir located under the Resources directory. They are module files that I want to load depending on what is found. An example of the code is shown below:

var dir = Ti.Filesystem.resourcesDirectory+'libs';
var f = Ti.Filesystem.getFile(dir).getDirectoryListing();
alert(f);

If I have 3 .js files in the libs dir there file names are displayed in the simulator, but nothing is found when running on the app.

I am currently assuming that this is because the simulator version is not compiled but the release version is. Am I correct in my assumption? and if so is there any way of obtaining the .js file names at runtime in a dir (or sub dir) in a compiled app? or is this lost as far as the filesystem is concerned after compile.
