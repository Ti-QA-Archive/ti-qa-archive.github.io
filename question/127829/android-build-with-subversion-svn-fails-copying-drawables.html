Android build with Subversion .svn fails copying drawables

Hi,
I have a mobile app that builds and runs in iOS emaulator but fails when building for Android because it tries to copy the .svn files while copying the drawables resources. The end result is that it does not show up in the emulator.

If I export the project from subversion (no .svn files) it builds and runs in the Android emulator.

I am running on Mac OS X Version 10.6.8. I have tried several different SDK version 1.6.2, 1.75 and in both Titanium Developer and TitaniumStudio. The nice thing is Titanium Developer stops at the error so I know what went wrong. TitaniumStudio logs the error and continues but my app isn't there. I added the relevant part of the build log. I am surprised I didn't find anything about this in the forum so I am wondering if there is something I can do to work around.
