Specify filename in multipart file upload

I need to upload a JPG image file to an existing service. The service requires an authentication string which is a hash that includes the file name. Therefore, the filename in the hash (determined BEFORE I make the request) must match the file name IN the request which is generated by the HTTPClient().

I've tried a number of ways of uploading the file. The closest I've come is to create an object with the parameters:

var multiPart = {
        api_sig : digest,
        img : fileLeafName,
        time_stamp : timeStamp,
        json : 1,
        api_key : api_key,
        img : resizedBlob
    };

xhr.send(multiPart);

The problem is that when the header is generated (verified by Wireshark), Appcelerator generates a random name for the ing : resizedBlobl element. It sends it correctly as a JPEG mime type, just with the wrong name. (The name I start with is 'test.jpg'.) Of course a blob has no name associated with it. So I tried passing the file object, but then it just URL encodes the file, it doesn't read it and send it as JPG mime type.

I've tried putting the other parameters on the URL and just sending the file object (which presumably Appcelerator knows it's leaf name)


xhr.setRequestHeader('enctype', 'multipart/form-data');
xhr.setRequestHeader('Content-Type', 'image/jpg');

var multiPart = { img : imageFile };

but this also just sends a URL encoded version of the file.

I found one old tutorial but that uploads a text stream and I'm not sure how to get a JPEG file into the stream manually.

I don't mind patching an Appcelerator file, but so far I can't find where the filename is generated.

Any help, pointers or suggestions would be appreciated!
