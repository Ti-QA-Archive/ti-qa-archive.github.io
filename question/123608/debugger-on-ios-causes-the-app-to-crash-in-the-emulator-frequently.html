Debugger on iOS causes the app to crash in the Emulator frequently

Hi there

I've been pulling my hair out for ages now trying to figure out why my app seemingly crashes at random points every time I debug it, and I've finally found the root cause, and it's the damned debugger itself.

Every time I run the app in the iOS 4.3.2 emulator without the debugger enabled, the application runs perfectly every time. As soon as I turn on the debugger, even without any break points set, the app crashes at random points (I have tried debugging and tracing to find where it crashes and every run seems to crash at a different point) in the code base. The error message that is returned is completely useless and is usually something like "[DEBUG] Session did end with error (null)" and nothing else. No exception is raised, no indication of where the execution stopped, no traceback. I do sometimes see that there appears to be quite a few threads running at the point of the crash, but the debugger itself provides no hints as to what is running in those threads so I can't diagnose that.

I am running OS X Lion, Titanium Studio, build: 1.0.2.201107130739, Titanium SDK 1.7.2, iOS emulator 4.3.2.

Is anyone else experiencing the same issue? Does this really mean that in order to build my app I have to now stop using the debugger. This seems like madness that the tool I need to use to diagnose issues with code is in fact introducing new issues. Help Appcelerator!

Any advice would be appreciated.

Matt
