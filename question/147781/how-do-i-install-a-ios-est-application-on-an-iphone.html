    Hi

    I have never seen that article and that is not the way I do it.

    • Run toolbar button (same one you use to run in simulator
    • iOS Device
    • Confirm the following either on a single screen (if you have done this before on this app) or over three screens if this is the first time)
      • SDK version (most likely - 6.0)
      • Select Development Certificate (most likely you only have one to choose from)
      • Select KeyChain (unless you have a GOOD reason - use System Defaults)
      • Select Provisioning Profile, this should be a development certificate you have created in the iOS Provisioning Profile, it should have as many devices assigned to it as you are going to use the app on for testing
    • Finish

    When the app has compiled it will be installed into iTunes;

    • find the app icon
    • right click it
    • select Show in Finder

    To pass to another person;

    • Copy that file
    • Copy the same certificate you used to provision the app
    • Zip them up (safest if you are emailing)

    Send to the other person

    Make sure they drag the app to iTunes (it will indicate where you are to drop it) then do the same with the app.

    They will be able to install the app now and will not need another certificate unless things change on the existing one.

    Once you have done it a couple of times this is very fast.

    BTW there is another way to install an app to a device using the xcode organiser if for some reason the other device does not connect to iTunes.
