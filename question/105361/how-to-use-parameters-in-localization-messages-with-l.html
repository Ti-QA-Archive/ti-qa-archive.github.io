How to use parameters in Localization messages with L().

Not a question, just a helpful tip that I couldn't find by googling…

To use parameters in localized messages (which is needed quite frequently since word-order changes between languages), just put "%s" in the message itself, then use String.format.

EXAMPLE TIME!

in /i18n/en/strings.xml:

<?xml version="1.0" encoding="UTF-8"?>
<resources>
    <string name="artist_byline">This song is by %s!!</string>
</resources>

in app.js or other .js file:

alert( String.format( L('artist_byline'), 'Cee-Lo' ) );

Would pop an alert dialog with the text:
"This song is by Cee-Lo!!"

Hope that helps!

  • Note there is currently a bug in JIRA when using sequences with parameters in the format "Parameter %1$s", so avoid using sequenced parameters.

    Also, if you are interested, json.i18n which is an alternative JSON based localization system providing more flexibility than the default Titanium localization system, provides a shortcut method which automatically supports parameters i.e. assuming you have a resource key "object.key" with the value "Parameter %s", then you can simply call I('object.key', 'value');
