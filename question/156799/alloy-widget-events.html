Alloy Widget Events

I am trying to replicate (if it's possible) the way that in an XML definition you can trigger a custom event and pass it back to the container. For example:

<Widget id="test" src="test" onDone="processDone" />

Then in the controller for the container view/window I simple need to create 

function processDone(e){
    // my code
}

This is great ,but I am also attempting to do the same thing with a controller instantiated widget. So my code would be:

var test = Alloy.createWidget('test').getView();
$.mainWin.add(test);

This works fine, but I am currently unable to figure out how I would detect the widget $.trigger('done'). I guess my actual question is can it be done, and if so how?

Thanks,

  • As usual, 5 mins after posting and reading what I post I find an answer(not sure if it the correct one).

    Rather than return the view of the widget access the controller directly, i.e.

    var test = Alloy.createWidget('test');
test.on('done', function(e){
    alert('Done Triggered');
});
$.mainWin.add(test.getView());

    Well it seems to work anyway.
