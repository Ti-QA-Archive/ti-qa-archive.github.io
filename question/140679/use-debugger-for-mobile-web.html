Use debugger for mobile web

I see titanium is doing a good job for debugging IOS or Android app, but I try to set a breakpoint for mobile web app, and it does not work. I am using the mobile web SDK CI download from July, 2012. Is it because it's not supported yet?

  • Currently, you cannot use Titanium Studio's debugger to step through your code. You need to use the web browser's debugger. Each browser's debugger has it's own quirks.

    I don't think we'll have browser debugging integrated into studio for quite some time and if it does, then it will probably be limited to Studio's built-in browser or a handful of browsers that support remote debugging.

    There isn't much debugging for browsers on device. Chrome on Android 4 and iOS 6's Mobile Safari have remote debuggers.

    We have plans to at a minimum to expose console/Ti.API log calls from device browsers, but probably won't be available for a while. You can do this today with Android's adb logcat. Othewise, you can use Weinre (requires node.js and a Webkit based browser (Chrome, Safari) and then hack the generated index.html to include Weinre's script.
