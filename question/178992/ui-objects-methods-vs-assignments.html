UI objects: methods vs assignments

I was wondering if there is any difference between:

$.view_object.height = '50dp';

and

$.view_object.setHeight('50dp');

which one is recommended and are there any pros or cons, perhaps for other properties?

    Hi

    Both of these will do the same thing. The "assignments" are generally used for initial creation. The setter/getter methods are used afterwards as they allow for cleaner, readable code.

    The only difference between the two is that the "assignment" way is marginally faster. This wouldn't be noticeable to your users though without a significant number of methods being used.

    The most important way to improve your example would be to remove the "dp", as this should be the default unit inside of your tiapp.xml (if it reads 'system' instead, change it to 'dp'), and then also change your value from a string to an integer. This change would be of more significance to you, but still wouldn't be noticeable enough to the users.

    Finally, personally I use the *.tss for defining initial objects, so there wouldn't be any assignments inside of my controllers or JavaScript. I would use the methods to alter my objects afterwards if and when needed.

    Hope this helps.
