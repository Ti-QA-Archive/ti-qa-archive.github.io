Android fastdev

Hi,

I installed Titanium mobile sdk 1.7.0 RC1 and I have Titanium Studio 1 running. But when I want to use the fastdev option for android I always keep getting the same error, for example: In the terminal of Titanium Studio I type following command:

$ titanium fastdev status

BUT I get following error:
_Error: .fastdev.lock found in /Users/sandra/Documents/Mobile/VT4_CitiesApp, but couldn't connect to the server on port 52463: [Errno 8] nodename nor servname provided
, or not known. Try manually deleting .fastdev.lock.

What could be the problem here?

  • By tracking down where the exception was getting thrown in fastdev.py, I found that on my MBPro, it was trying to connect to a hostname that wasn't localhost. I hand edited the python script to change (line 279):

    self.socket.connect((socket.gethostname(), self.port))
    to:
    self.socket.connect(('localhost', self.port))

    I'm not sure why this wasn't working, because it worked at some point in the past as recently as an hour before it stopped working. But after I made that edit to the script it started working again. Obviously this isn't an optimal solution but it let me run fastdev again which is a necessity for developing an android app in Titanium.
