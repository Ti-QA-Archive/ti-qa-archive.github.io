Windows Overlapping

I followed this tutorial: [http://mobile.tutsplus.com/tutorials/appcelerator/titanium-user-authentication/][http://mobile.tutsplus.com/tutorials/appcelerator/titanium-user-authentication/] to create a user authentication app which was the initial building block on what my app was going to be.

Now I was trying to grab invoice data from a mysql database which I did successfully but when I a test message on the invoices tab window which I used:

var win = Titanium.UI.currentWindow;

I also based the invoices tab on the login tab.

So when I coded a simple test message to appear like on the main.js file in the tutorial. in doing this i deleted the like 

win.close();

so that when the message was displayed it wouldn't close the window. After doing this is when my problem came to air itself.

I built the app and tested in in the iOS Simulator and it displayed the message but the login window didnt disappear. So I went back to my invoices.js file and changed

var win = Titanium.UI.currentWindow;

to

var win = Titanium.UI.createWindow;

Relaunched the simulator but the login window still persisted on the invoices window. I then returned all the code to what it was before and relaunched the simulator thinking the problem would be fixed but no the login window still persist.

So this is where I ask, how do I fix this?

I hope I explained my issue well enough

  • var win = Titanium.UI.createWindow;

    so NOW win = function that can be used to create other windows.

    Maybe you are looking for:

    var win = Titanium.UI.createWindow(); // which will create a window with little or no useful value until you modify its properties.
