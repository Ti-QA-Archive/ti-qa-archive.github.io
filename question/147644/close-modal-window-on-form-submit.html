  • Hi

    Here is a simple technique that does not depend on your Ti SDK version or Platform choice.

    function stateChoice(obj) {
    var cancelButton = Titanium.UI.createButton({
        title:'Cancel',
    });

    cancelButton.addEventListener('click', function() {
        if (obj.action) {
            obj.action({
                state: textField.getValue()
            });
        }
        modalwindow.close();
    });

    modalwindow = Ti.UI.createWindow({
        width: 500,
        height: 300,
        top: 0,
        left: 0,
        borderRadius: 10,
        title: 'Search State',
        rightNavButton: cancelButton,
        orientationModes: [
            Ti.UI.PORTRAIT, 
            Ti.UI.UPSIDE_PORTRAIT
        ]
    });

    var submitButton = Titanium.UI.createButton({
        title: "Search",
        height:24,
        width:60,
    });

    var textField = Titanium.UI.createTextField({
        top: 20,
        left: 20,
        height: 24,
        width: 260,
        backgroundColor: '#fff',
        boarderStyle: Titanium.UI.INPUT_BORDERSTYLE_ROUND,
        hintText: "Enter the State",
        keybaordType: Titanium.UI.KEYBOARD_DEFAULT,
        rightButton: submitButton,
    });

    submitButton.addEventListener("click", function (e) {
        if (textField.value !== "") {
            var newWindow = Titanium.UI.createWindow({ 
                url: 'search.js',
                MyState: textField.value,
            }); 
            newWindow.open(newWindow); 

        } else {
            alert("Please enter a state.");
        }
    });

        //modalwindow.add(submitButton);    
    modalwindow.add(textField);

    modalwindow.open({
        modal: true,
        modalTransitionStyle: Ti.UI.iPhone.MODAL_TRANSITION_STYLE_COVER_VERTICAL,
        modalStyle: Ti.UI.iPhone.MODAL_PRESENTATION_FORMSHEET
    });
}

imageView01.addEventListener('click', function(e) { 
    stateChoice({
        action: function (e) {
            alert(e.state);
        }
    });
});

    This uses a function to handle the modal window and is called by that function which uses a callback to return data from inside the function and thus the modal window.

    You will see this is how you call it.

    stateChoice({
    action: function (e) {
        alert(e.state);
    }
});

    You can extend the fields returned by tweaking this section;

    obj.action({
    state: textField.getValue(),
    anothervalue: 'hello',
    more: 'choices'
});

    Let me know how it goes.

    • Use my previous function, Change your submit handler inside to this.

      submitButton.addEventListener("click", function (e) {
    if (textField.value !== "") {
        modalwindow.close();
        if (obj.action) {
            obj.action({
                state: textField.getValue()
            });
        }
     } else {
        alert("Please enter a state.");
    }
});

      Then call it like this.

      stateChoice({
    action: function (e) {
        var newWindow = Titanium.UI.createWindow({ 
            url: 'search.js',
            MyState: e.state
        }); 
        newWindow.open(newWindow); 
    }
});

      This will call the function, show your window, on successful entry of a state return the state in the callback and closes the modal window. The callback then opens the new window as you need.

    • The problem you have is that the emailDialog addAttachment method requires a local file. You have correctly tried the local file system, but you have passed a remote URL which will not work.

      You will need to either;

      • use the httpClient to download the file to your device, save the response locally and then attach the file
      • Add a HTML anchor tag to the the remote PDF so the recipient can download the PDF themselves

      You can add a print option to an iOS app but you need a custom module to achieve it. The AirPrint module available in the market place will do what you need (it works I use it). You can tell it to print a PDF document or a view available in your app. It then connects to any local AirPrint enabled printer.
