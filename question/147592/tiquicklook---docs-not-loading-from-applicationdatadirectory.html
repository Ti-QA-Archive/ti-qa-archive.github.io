Ti.quicklook - Docs not loading from "applicationDataDirectory"

Hi,

I'm using Ti SDK 3.0, Module version 1.2 (latest) on iOS6.

I got booth a ".doc" and a ".pdf" documents working smootly under the path "/Resources/assets/myDoc.pdf". However, when I retrieve the docs from the applicationDataDirectory anything cames out.

Working:

var docs = [];
docs.push('/assets/myDoc.doc');
var quickView = quicklook.createView({
    data : docs,
    index : 0
});
win.add(quickView);

None of this Works:

var docs = [];

var myDir = Ti.Filesystem.getFile(Ti.Filesystem.applicationDataDirectory, "myDir");
var f = Titanium.Filesystem.getFile(myDir.resolve() + '/myDoc.doc');

docs.push(f.resolve()); // .canPreviewItem(f.resolve()) returns "true"
docs.push(f);
docs.push(f.read());

I tried all the workarounds I could think of but no luck…

could it be that's not supported by 3.0 SDK?

thanks!

      I know it's 4 month now from your question but I hope it helps, because yes, It did work in the end. 

      var quicklook = require('ti.quicklook');
var docs = [];
docs.push('/assets/corp/noImage.png'); // I always put an image by default just in case
var quickView = quicklook.createView({data : docs});
self.add(quickView);


var docPath = filename + '.' + ext;
docPath = replaceAll(docPath, " ", "%20"); // replaceAll is a self-made function

var f = Ti.Filesystem.getFile(Ti.Filesystem.applicationDataDirectory + '/images_' + Ti.App.Properties.getString("companyName") + '/' + docPath);
if(f.exists())
{
    docs.push(f.getNativePath());
}

f = null;

if(docs.length < 1)
{
    docs.push('/assets/corp/noImage.png');
}
quickView.data = docs;
