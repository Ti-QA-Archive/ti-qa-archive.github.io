HTTPClient is encoding url in 1.8 despite autoEncodeUrl set to off

Is anyone else having this problem?

When using HTTPClient and framework 1.8, my url requests are always encoded…meaning instead of a '~' there is a '%7E' and so on. I've tried using POST and GET, both with same result.

I am setting autoEncodeUrl to false like so:

mc.dm.httpClient = Ti.Network.createHTTPClient({autoEncodeUrl:false});

When I reverted back to 1.7.5 my urls were not automatically encoded and worked fine.

    This is because autoEncodeUrl is not properly initialized at creation time. You'll need to set it after the object is created:

    mc.dm.httpClient = Ti.Network.createHTTPClient();
mc.dm.httpClient.autoEncodeUrl = false;

    I actually opened a ticket for this a few minutes ago, as it still occurs in 1.9.0.
