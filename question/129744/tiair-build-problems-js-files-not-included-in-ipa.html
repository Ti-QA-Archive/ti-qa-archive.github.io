TiAir build problems, js files not included in ipa

Has anyone used TiAir successfully? https://github.com/dawsontoth/TiAir

When I publish the app to the iPhone Simulator it works fine, but once I publish to the device, I noticed that the controllers, models, navigator and views classes are never included in the ipa file. I find it odd that TiAir doesnt use Ti.include to pull in these files but rather just runs through all the files in the directory to see if they exist.

  • Hi Mark, I use this Library too. On loading process the files will be read from the filesystem which will only work in simulator mode because the files are not "compiled" (= minimized and packed). To get this run on a device you have to rename your files to "._js" (or something similar, because .js will graped by TiStudio).
    That worked for me.
    You can minimize, pack and encode your files before building.
    Greetings, André
