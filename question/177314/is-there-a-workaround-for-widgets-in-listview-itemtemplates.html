  • Accepted Answer

    Create the widget instances in the controller code, then add them to the listview within the JavaScript. As long as the resulting objects are instances of Ti.UI.View (or its subclasses), then they will work in a listview. It's just the Alloy doesn't process those tags in XML when generating the code for your list.

    • Sorry, I didn't fully read your original note and my initial reply before adding today's comment. I got it totally backward!

      You should be able to declare your listview and define your template in XML. Make sure to include a View container in that template to hold the view exposed by the widgets. But then, you'll need to populate your listview in the controller code, instantiating and initializing widgets as part of whatever looping code you use to create your data set. You won't be able to use Alloy's built-in data binding mechanisms.

      You might need to modify the starrating widget to update how it handles user interactions. It was not designed with the ListView's eventing model in mind.
