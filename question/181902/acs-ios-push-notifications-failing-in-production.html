ACS iOS push notifications failing in production

We have a high profile mobile Titanium app built with the following versions and deployed for iPhone

Titanium Command-Line Interface, CLI version 3.4.2, Titanium SDK version 3.3.0.GA
[INFO] :   Found Titanium plugin id=ti.alloy version=1.0 
[INFO] :   Building using iOS SDK: 7.1
[INFO] :   Building for device family: iphone
[INFO] :   Minimum iOS version: 6.0

There are currently more than 10K users registered for push notifications on ACS. Push notifications have been working through all levels of our acceptance testing and the registration process is clearly still working, however push notifications sent from the ACS console are not reaching the devices.

I have double checked that the ACS App Key, Oauth Consumer Key and Oauth Secret shown in the Production Console settings match those in my tiapp.xml file and that the valid Production APN certificate is uploaded in ACS.

Push notification logs in the console for the latest attempt show success for only 309 iOS devices but no iOS failures. (ref: Push Notification Logs for 551bf62054add893d5d16be1)

The app is also deployed to android where there are >1k subscribed clients. The console reports individual failures against the android clients (I think for a different reason) but could these failures affect iOS pushes?

  • What are the possible reasons for a registered device not receiving a push notification?
  • Is 10K users a problem for the ACS platform?
  • How can I debug this issue when the ACS console is indicating success?
  • Is support available for the ACS platform?

Any help much appreciated.
