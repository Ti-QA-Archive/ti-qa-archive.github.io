Module License Violation - How to Fix?

Hi guys,

I have two iOS modules purchased from the marketplace (Testflight and Sharekit) which allow for 1 seat each.

I normally use my iMac for all dev work, but wanted to do some work on the road on my MacBook Air and installed Ti Studio and downloaded all my modules from the marketplace.

It would seem now that whenever I run an app using either Testflight or Sharekit on my iMac, I get the dreaded message:

[ERROR] : License violation detected. Unlicensed module(s) detected. You must be a subscriber to use one or more of the modules included with this project.. Please contact Appcelerator Support.

How on earth do I 'unregister' my MacBook Air from the Titanium dev environment (therefore registering just the 1 seat) and to unblock this error? I can't develop anything at the moment, it's super frustrating!

Simon

    • I had this problem to and to fix (on my mac) i did:

      • Exit titanium
      • uuidgen (from command line and copy and paste the uuid)
      • edit tiapp.xml with another edit (vi tiapp.xml in my case)
      • Replace the uuid with the one I copied (save, exit).
      • Start titanium again and clean the project
      • Then build and run.

      Problem fixed.
