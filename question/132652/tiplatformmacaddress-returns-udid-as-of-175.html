Ti.Platform.macaddress Returns UDID As of 1.7.5

Hi,

we've been using Ti.Platform.macaddress to uniquely identify devices in a couple of our Titanium mobile apps targetting iOS 5+. This morning it was noticed that the value that is returned by this property is actually the UDID of the device. This behaviour appears to have changed in 1.7.4/1.7.5 as 1.73 seems to return a hash of the macaddress (which is what we expected). We have concerns now as to whether we can reliably use the macaddress to identify devices (our plan is to hash the returned macaddress). Can anyone shed any light as to why the behaviour of this property changed (I cannot find anything in the release notes for any builds)?

Thanks in advance,

Graham
