In App Purchase media - storage guidelines

I have an app which has in app purchase downloadable videos. The first time I submitted to the app store it was rejected because I was writing it to applicationDataDirectory. The guideline told me that

  1. Data that can be downloaded again or regenerated should be stored in the <Application_Home>/Library/Caches directory. Examples of files you should put in the Caches directory include database cache files and downloadable content, such as that used by magazine, newspaper, and map applications.

Ok so I changed the directory to the following code. I checked the simulator's cache directory and found that there is a default folder within the Caches folder with the name of the Bundle Id of the app. So I used the following code.

                var f = Titanium.Filesystem.getFile(Titanium.Filesystem.applicationDataDirectory+'../Library/Caches/<Bundle Id>/', videosArr[i].title+'.mp4');                

f.write(this.responseData);
f.remoteBackup = false;

Ran this in the sim and seems fine. However, when I uploaded it, it was rejected. The problem seems to come from writing the data.
Question: Is there something wrong with this directory? Should I just store it in the Caches root folder?

Thanks.

    You may not be away there is another constant to do exactly what they are after, applicationCacheDirectory;

    var cachefolder = Ti.Filesystem.applicationCacheDirectory;
var f = Ti.Filesystem.getFile(cachefolder, videosArr[i].title + '.mp4');

    Use any folders you like under this references.
