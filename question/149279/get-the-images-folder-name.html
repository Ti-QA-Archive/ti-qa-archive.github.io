  • Hi

    My question to you is why?

    The process is automatic - as long as you have set the correct multi density values in the tiaap.xml.

    If you are just trying to confirm that the process is working then you only have to create one imageView that shows a different image to test. Simply place an image in each of the folders and make one red, another green and so on.

    This way if you are testing it will be very obvious. Then you put images exactly where you to have them.

    The folder names based on platform and size are "flattened" when on the device as it manages the path automatically - so what you see when developing is NOT the structure when on the device.

    If you want a simple app to confirm things, I found this simple one a while back. github.com/jeffbonnes/dipscheck - should provide you with confirmation and a simple folder structure layout.
