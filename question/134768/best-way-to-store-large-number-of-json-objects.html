    • Yah you can also store to File. For more efficient, you should use both File & Properties:

      • Use JSON.stringify to encode JSON object into String, then store to File. If your JSON object has unique key/id, then use that key/id for file name. Else, generating a random string for file name.

      • Store file name ( key/id or random string ) and file path to Properties file.

      • When you want to read JSON object, determines file path based on file name, then read file content into String.. And finally use JSON.parse to parse your string into JSON object.
