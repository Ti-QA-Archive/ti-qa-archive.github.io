  2. applicationDataDirectory

  • I believe the resourcesDirectory is readOnly so you cannot cache images there.

    So it pretty much leaves you with the applicationDataDirectory; that is what I have used in the past

  • Yes the applicationDataDirectory is correct.

    I usually use it like this:

    var file = Ti.Filesystem.getFile(Ti.Filesystem.applicationDataDirectory, directoryName, filename);
