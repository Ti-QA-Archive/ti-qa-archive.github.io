Not receiving GCM when app not running

When I switched to Titanium 3.3.0 GA my GCM pushes (using builtin the titanium cloudpush-module) don't reach the device if the app is no longer running. The message is received if the app is in the foreground or running in the background but is not delivered the app has been terminated by the user or by Android due to memory shortage.

I have not had these issues with earlier versions of the Titanium SDK.

The push is sent using the HTTP api (or manually through the ACS console) and the logs give response code 200 in both when the push actually reaches the device and when its not.

The cloudpush code used is nothing more than this:

var CloudPush = require('ti.cloudpush');

// These events monitor incoming push notifications
CloudPush.addEventListener('callback', function (evt) {
    Ti.API.info("Receiver GCM notification");
    Ti.API.info(evt.payload);;
});

CloudPush.setShowAppOnTrayClick(true);

Anybody experienced similiar problems and/or have any solutions to it?
