  • Steven,

    There have been 2 different scenarios that have popped up a few times, and, its a rare occurrence.

    Scenario 1 is the result of projects with extremely large numbers of .js files.

    Scenario 2 is the result of projects with extremely large .js files.

    Because these two scenarios themselves are anomalous, it took a while for the bugs in the tooling to surface. I'll try to dig up the JIRA tickets that references the issues.

    To keep you building, you can use the --skip-js-minify as a build param on the command line, and this should bypass the munging of the js files. Do NOT do this for a production apk, as your js source will be available to anyone. You can also try to force a test build (-D test) which I believe also skips the munging process.

    https://jira.appcelerator.org/browse/TIMOB-16093 - Closed as unable to reproduce, but user had a bad export

    https://jira.appcelerator.org/browse/TIMOB-16077

    If you have information that you could share, please append to the first ticket if it looks like it applies. Let me know if it does, and one of us will see if we can raise some attention to it.
