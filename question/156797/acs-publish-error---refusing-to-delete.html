  • I've also run into this error twice:

    Error: Refusing to delete: /private/var/folders/cl/...

    I've tried manually deleting the file paths listed using sudo but it just gives me a different path that it needs to delete.

    To fix it, I've had to go into the /node_modules/.bin folder and delete these files:

    • _mocha
    • express
    • mocha

    And replace them with symlinks:

    lrwxr-xr-x   1 robertchen  staff   19 Mar 20 15:00 _mocha -> ../mocha/bin/_mocha
lrwxr-xr-x   1 robertchen  staff   22 Mar 20 15:00 express -> ../express/bin/express
lrwxr-xr-x   1 robertchen  staff   18 Mar 20 15:00 mocha -> ../mocha/bin/mocha

    Syntax for creating symlinks is: 

    ln -s ../mocha/bin/_mocha _mocha
ln -s ../express/bin/express express
ln -s ../mocha/bin/mocha mocha

    Not sure what the mechanism was for this to work, but it seemed to do the trick twice.
