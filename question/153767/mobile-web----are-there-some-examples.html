    • There aren't really any samples to show that are specific to Mobile Web. Mobile Web differs very little from iOS and Android, so there simply isn't much to show.

      To convert an existing web site to Titanium Mobile Web is a pretty major undertaking, and the process will be specific to the web site you are converting. The UI will need to be rewritten to use TItanium APIs instead of the DOM/jQuery/Dojo/whatever. Interacting with data sources, either via local storage or networking, will likewise need to be rewritten to use our APIs. Some concepts won't transfer over either. Titanium Mobile Web apps are single page apps only, and if the site you are porting is not single-page, you will need to re-architect it to be single page.

      I don't mean to scare you off though. Even though it's a lot of work, there are a lot of advantages to the work as well, namely the ability to easily target native iOS and Android app development. Once you've done the initial Mobile Web work, you are already most of the way towards being able to submit a native iOS to the App Store.
