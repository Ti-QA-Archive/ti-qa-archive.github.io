Ti.Gesture.addEventListener('shake',...) works on device, hangs on emulator

Hi all,

I'm facing a problem with the android emulator using mobile SDk 1.8.2 and Android 2.2 APIs.

This code works on the device, but makes the emulator stall forever without any notice in the logs, even not verbose. I have tried full rebuild, fastdev on/off, clean project, and some other stuff. Still I can get it only to work on the emulator if I comment out this section. The variable suwLogo is an image view with a click listener attached.

Ti.Gesture.addEventListener('shake',function(e) {
    if(Smash.__trialBtnVisible){
        mainWindow.remove(suwLogo);
        Smash.__trialBtnVisible = false;
    } else {
        mainWindow.add(suwLogo);
        Smash.__trialBtnVisible = true;
    };
});

Is this a known issue with the gesture module and the emulator? Couldn't find anything useful about it… Any suggestions what is going on here?

Just in case I need to work around this: Is there any way to tell Ti to ignore this code when running on an emulator?

Thanks,
Christoph

    The shake gesture doesnt work on the emulator. if you really need to fire that event on the emulator just do a Ti.Gesture.fireEvent(&#39;shake&#39;);
