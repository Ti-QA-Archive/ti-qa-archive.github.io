    • So you are seeing a message suggesting you need a newer version of XCODE but you are only targeting Android? As you probably have guessed this may not be a positive outcome.

      Couple of things to check;

      1. Have you used Project menu > Clean - fixes more than you would think.
      2. Changes the project you are working on to use 2.1 not 2.1.1
      3. Make sur eyou project ONLY specifies Android, if any iOS is checked you may be getting the error because the app THINKS you may want iOS.
      4. If you have NO iOS apps, have you tried installing the latest version, you may have a need for pre-requisite - weird but possible.

      And if all else fails then these two.

      a. If you have an external hard drive you could spare - then your Mac should be able to boot from that, you could install the old version on to it and during boot swap to the Lion external for booting. This is a good way to test as well. May also let you restore your time machine to the external Lion drive.
      b. Parallels for Mac allows you to install a virtual operating system and the Mac allows the Lion and higher to be installed in a virtual environment, you could run a virtual mac side by side your actually mac (at the same time).

      This way you will have the best of both worlds - albeit with extra hassle right now.
