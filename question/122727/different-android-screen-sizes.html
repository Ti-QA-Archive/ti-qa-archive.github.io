Different Android screen sizes

I developed my app for iPhone originally and have just finished optimizing it for Android. However testing on an Orange San Fransisco revealed the app literally fits in the top left corner of the screen as the screen size is so large.

What's the best strategy for dealing with this? Obviously my icons and fonts are now way too small, do I have to calculate screen width/height and alter my font sizes and spacings accordingly, or is there some sort of view scaling function that would make life easier? I'm guessing probably not :)

thanks for any advice.

  • Accepted Answer

    You can set the UI to scale to the device, in which case Android will simply enlarge everything to fit. It won't be optimal, but it will work. See this Developer Blog post for more info.

    You can provide screen density specific graphics in the Resources/android/images folder. See this guide. You can also use JSS to provide density-specific stylesheets, such as yourfile.android.high.jss.

  • I had some similar issues until I put 

    <manifest>
    <supports-screens 
        android:anyDensity="false" 
        android:largeScreens="false" 
        android:normalScreens="true" 
        android:smallScreens="false"/>
</manifest>

    in my tiapp.xml…
    I'm not sure this is THE answer but it seemed to take care of most phones, but the jury is still out.
