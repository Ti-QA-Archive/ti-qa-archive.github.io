  • Appcelerator is a native solution for iOS and Android app (BB coming soon). Like any programming language, there are pitfalls especially when poorly designed and implemented. The greatest advantage the platform has on the competition is that it produces Native Applications using Javascript, and not HTML5 apps wrapped in a webview.

    It is highly extensible, and there are solutions for most problems that are not part of the core Titanium using modules.

    1. There are modules for doing barcode/qr - check the marketplace or online (google)
    2. It supports most REST APIs, and even SOAP implementations. If it is XML/JSON, you should be able to implement it.
    3. This is a difficult version to answer - the core system addresses most use cases
    4. I am not well versed in pricing, but there isn't a standard edition for 195/month. Before the new prices and plans were announced a few weeks ago, the Indie level package was 600/year.
    5. Using best practices can mitigate this. It is native code, and has the best performance in its space, when code is well implemented
    6. Appcelerator may not be your choice for games, i'd advice Corona for that. But for all other types of apps, it should be your #1 choice
    7. It integrate natively with Facebook, and there are numerous Twitter implementations out in the wild.

    I hope i helped answer your questions.

    Prior to version 2, the latest yes it was possible to have memory leaks easily, but it is down to coding. Version 2 now has the commonJS specification implemented into the framework and it is much harder to code serious memory leaks, but still very possible.

    a) There is no inbuilt Bar code scanner, but it is JavaScript so you can plug in one of the open source ones or use one of the available modules.

    b) You can link to any website and incorporate most JavaScript open source items.

    c) Yes standard issue is sufficient, unless you have a lot of developers.

    d) Not sure on the cost but I think what you have been told is high, you can sign up and start coding for free, you don't need to buy a specific service. You would then just pay for the modules you may need on top. I haven't as yet had to use any modules, and I have also produced bar cde and qr code scanning apps.

    e) Actually I think Appcelerator is now slightly faster than corona although I don't think there is much in it. It is a lot faster than phoneGap. The memory issues are down to coding.

    f) Yes is the answer. Although I think titanium is much more suited to business apps, there is a growing number of games coming out using it.

    g) Yes integrates to social apps, and even open source examples are provided.,

  • So to go further.

    a) I don't work for Appcelerator so I cannot comment on pricing.. but if you contact sales@appceleraotr.com they should get straight back to you.

    b) Yes, the code doesn't change at all.

    c) Studio and SDK version 2 have recently been released they are the ones that are currently downloadable.
