  • Here an example for Mituls hint:

    var starter = Ti.UI.createWindow({
    backgroundImage : '/assets/Default.png',
});
var actInd = Titanium.UI.createActivityIndicator({
    style : Titanium.UI.iPhone.ActivityIndicatorStyle.PLAIN,
    message : 'Initialisierung …',
    backgroundColor : '#D4A017',
    width : '200',
    height : '40',
    color : 'black',
    padding : 10,
    opacity : 0.87,
    bottom : 20,
    borderRadius : 8,
    borderColor : 'black',
    borderWidth : 1
});
starter.add(actInd);
actInd.show();
setTimeout(function() {
    actInd.animate(Ti.UI.createAnimation({
        right : -300,
        duration : 700
    }));
}, 1500);
setTimeout(function() {
    starter.close({
        transition : Titanium.UI.iPhone.AnimationStyle.FLIP_FROM_RIGHT,
        duration : 1500,
    });
    tabGroup.open();
}, 3000);

    • This is because Flip Animation takes usually a 'view' property, Transition between 2 Views. Thats why it is called Transition. Now the 2nd view is empty. Furthermore you execute visible = true directly after the Animation Start, but the Animation will Take 1,5s to complete. Therefore you will need a second win for the Transition and make your Tabs visible after the Animation has finished.
