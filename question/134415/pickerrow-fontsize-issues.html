PickerRow FontSize Issues

I'm using Titanium Mobile 1.8.2 and iOS Simulator Patch 5.0, Xcode 4.2.

I have a picker view with 1 row in it saying 'Loading…'. The picker is then emptied and new rows added from an ajax call.

for (var key in data) {
    picker.add(Titanium.UI.createPickerRow({
        fontSize: 12,
        title: _data[key],
        value: key
    }));
}

Yet when I open up the picker, the first of every 7 rows has normal font size and the rest the correct font size. When I run getFontSize() on all the rows, they all return a correct font size of 12, but this clearly isn't happening.

Anybody else had this same issue and found a way to solve it? I'd prefer to use fontSize of pickerRow than inserting a label due to android compatibility.

Thanks,

ise

  • Hi Gary,

    Try this

    var row = Ti.UI.createPickerRow();
    var label = Ti.UI.createLabel({
        text:text,
        font:{fontSize:24,fontWeight:'bold'},
        color:text,
        width:'auto',
        height:'auto'
    });
    row.add(label);
    picker.add(row);
