Alloy TableViewRow Class vs Classname

Hi

Just a quick question, when defining a TableViewRow in Alloy XML is the class property different to className? In other words, is class used to define the TSS styles, but 'className' makes the rows reusable, therefore saving on memory? Or are they the same thing?

Thanks

James

    Hello James,
    Class is different from className property of tableViewRow. If you define a class instead of an ID for an object in alloy XML it suggests that you can provide a single tss style for different tables having the same class by using .class property. You know the rest. Thanks…

  • class refers solely to Alloy's style processing. className refers solely to TableViewRow's row optimization property. The two are not related.
