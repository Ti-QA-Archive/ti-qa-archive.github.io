backgroundPadding failed after upgrade to 3.0

Hi,

I'm developing an iOS 6 mobile application using the latest Titanium SDK (3.0) and can't get the backgroundPaddingLeft and backgroundPaddingRight to work. Earlier versions of the SDK had no problems with this.

Symptoms are that the background image is cut off exactly where the text starts and ends. Looks like the image is stretched outside the text as it should, but it is not painted outside the text (like it is transparent outside text range)

Any one having the same problems and perhaps a solution?

My code:

var containerView = Ti.UI.createScrollView({
    contentWidth:"auto",
    contentHeight:"auto",
    top:0,
    showVerticalScrollIndicator:true
});

var headerLabel = Ti.UI.createLabel({
    top : 100,
    height: 50,
    backgroundImage: "images/album_header.png",
    text: L("loading_stories"),
    textAlign: "center",
    backgroundPaddingLeft: 20,
    backgroundPaddingRight: 20,
    font: { fontSize: 20 },
    color: "#333"
});

containerView.add(headerLabel);

Platform info:
Application type: mobile
Titanium SDK: 3.0.0.GA
Platform & version: iOS 6.0
Device: iOS simulator and iPhone/iPad
Host Operating System: OSX 10.7.5
Titanium Studio: build: 3.0.1.201212181159

  • Hi

    Works for me.

    var label = Ti.UI.createLabel({
    backgroundColor: 'orange',
    backgroundImage: 'strip.png',
    backgroundPaddingLeft: -40,
    backgroundPaddingRight: 40,
    height: 50,
    width: 280,
    text: 'This is a label',
    textAlign: 'center',
    color: '#fff',
    font: {
        fontSize: 24,
        fontWeight: 'bold'
    }
});

    The difference between mine and yours is you have not provided a width, so it is auto-sizing the the content size. Try setting a fix width and see if it gets better.
