    • Sean,

      Just follow this instruction step by step and see if it work.

      1. Go to http://builds.appcelerator.com.s3.amazonaws.com/index.html and download the latest build of mobilesdk for Window. Extract it, navigate to your Titanium studio installation directory and do either of;

      IF (mobilesdk OR Mobilesdk directory exist in Ti installation dir) THEN
      Delete existing one in Ti dir
      Extract the latest build to Ti dir
      Rename the latest build from "mobilesdk" to "Mobilesdk"

      ELSE
      Extract the latest build to Ti dir
      Rename the latest build from "mobilesdk" to "Mobilesdk"

      ENDIF

      1. Locate your Android SDK installation directory. Navigate into "platform-tools" directory copy both adb.exe and AdbWinApi.dll. Go out of that directory and navigate to "tools" directory. Paste both file into the "tools" directory. Navigate to "platforms" directory. If empty just create a empty folder called "android-7", "android-8" e.t.c, this depends on Android platform(s) you installed through the Android SDK Manager.

      2. Make sure your environment variable is set for both "tools" and "platform-tools" directory. To do this you have to edit the Path variable.

        i) Go to Start and type "system variable"

        ii)It will open a window, towards the bottom you will see Environm ent Variables, click on it. Locate Path under the System variab les pane/window. Click on it once, you will notice 3 buttons be low, Click on Edit. Type the following (Make sure the path conforms with the way it is on your system as the one here is t he way it on my on system) ;C:\apps\android-sdk\platform-tools\ ;C:\apps\android-sdk\tools.

        iii)Just hit OK(s).

      3. Restart your Titanium studio and configure it this time.

      Hope this help. Keep me posted. Cheers

  • It's most likely the bug described here and fixed for the next release http://jira.appcelerator.org/browse/TISTUD-422

    It's throwing this error because Studio expects the directory $ANDROID_SDK/platforms/android-7 and its contents to exist.

    To fix this, install "SDK Platform Android 2.1, API 7" using tools/Android.bat (on windows. There should be a similar tool on the Mac as well). I had this issue, and this fixed it for me.
