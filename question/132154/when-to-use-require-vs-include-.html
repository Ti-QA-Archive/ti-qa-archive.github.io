When to use Require Vs. Include ?

Hi All,

I'm a newbie to appcelerator to JavaScript Design Patterns in general and I've been studying all the excellent docs & videos on using Ti.Include() and CommonJS Modules.

My understanding is that Ti.Include() is the same conceptually as copy & paste and by using anonymous functions & namespacing we can avoid polluting the global scope (e.g. I've looked at the Tweetanium example).

CommonJS Modules though seem to do something similar albeit with less work on a developers side through the exports object, which guarantees public/private access.

Can someone explain why and when I would use one over the other? Is it that CommonJS Modules are a more standardised approach that has only come along recently through open Community efforts and that there is a general move towards it? (Remember I'm new to appcelerator :).

Are there some cases where you would use one over the other? e.g. encapsulate UI code in a Include
files while Business Logic and Database Logic might be included as CommonJS Modules using require.

Am I right in saying a CommonJS Module runs in it's own context and this can have pros/cons from a memory perspective?

Thanks for the help

    • Yes, exactly. (IMO) Bad part of CommonJS is that every CommonJS module creates new JS context, which means new JS environment. There is no direct access to global objects. Also, if you do something with native prototypes (like adding methods to "Array.prototype"), that won't be reflected in module's context. Everything you do needs to be re-done, or re-required (because, you'll write a module for every type of functionality).

      Although I like "Ti.include" and JS namespaces, CommonJS is better choice in long term because Titanium framework will be moving in that direction.
