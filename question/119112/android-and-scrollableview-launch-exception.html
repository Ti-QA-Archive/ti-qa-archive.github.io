Android and ScrollableView launch Exception

Hi,

I use Titanium SDK 1.6.2,

I created a Scrollable View, which simply has 2 views.

I open the scrollable view as a window when a button is clicked.

With Android, when I click on the back button, and then re-open this scrollable view, I get the following exception (just the beginning of the trace) :

java.lang.IllegalStateException: The specified child already has a parent. 
You must call removeView() on the child's parent first.
    at android.view.ViewGroup.addViewInner(ViewGroup.java:1970)
    at android.view.ViewGroup.addView(ViewGroup.java:1865)
    at android.view.ViewGroup.addView(ViewGroup.java:1822)
    at android.view.ViewGroup.addView(ViewGroup.java:1802)

...........

Here is my code : 

app.js :

var sViewWindow = Ti.UI.createWindow(
     {
         url: "scrollableWin.js",
         title:"scrollableWin",
         fullscreen:true
     }
);

//button to open the scrollable Window
var openSViewButton =Ti.UI.createButton(
     {
         title: "open Scroll View",
         width: 200,
         height:40,
         top:100
     }
);
//click on button to open window with scrollableView
openSViewButton.addEventListener("click", function(e){

     sViewWindow.open();
});

scrollableWin.js:

var win = Ti.UI.currentWindow;

var view1 = Ti.UI.createView(
     {backgroundColor: "gray"}
);

var view2 = Ti.UI.createView(
     {backgroundColor: "red"}
);

var scrollableView = Ti.UI.createScrollableView(
     {
     views:[view1,view2],
     showPagingControl:true,
     pagingControlHeight:20,
     maxZoomScale:2.0,
     currentPage:0
     }
);

win.add(scrollableView);

I also ran the Kitchen Sink demo (and with it the back/open works without problem…), and I do not see why it doesn't work here, too…

Someone has an idea ?? (bug ? )

Thanks !

  • I found a way to avoid the exception (and get the expected UI interaction, i.e go back and re-open a scrollableView)…

    It to add an Event Listener to the "android:back" button, and manually remove the view by doing 

    win.addEventListener("android:back", function(e){
   win.remove(scrollableView); 
   win.close();
});

    To my opinion this must be a BUG, since win.close() has to do all of these by itself….(and since it was not removing properly, I was getting the exceptions…)
