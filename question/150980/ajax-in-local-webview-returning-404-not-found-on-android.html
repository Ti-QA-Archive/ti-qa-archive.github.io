AJAX in local webview returning 404 not found on Android

My application features a webview that loads local HTML from the Resources folder. That HTML features JavaScript that uses jQuery to make an AJAX request for HTML content also hosted locally. The content doesn't seem to load on my Android device, instead returning a 404 not found error. This issue seems to be isolated to Android SDK Version 16. Has anyone else encountered similar issues?

  • @Stephen Feather, I find that long-winded questions with too much custom code often deters viewers from offering assistance.

    Developing on Mac OSX 10.8.3 with Titanium Studio 3.0.2 using Titanium SDK 3.0.2GA and Android SDK 4.2.2 (16) using a Google Nexus 10 to debug. The Resources folder is structured as follows:

    web
     - index.html
     - static
          - html
               - content.html
          - js
               - index.js

    The webview loads web/index.html, that file looks something like this:

    <!doctype html>
<html>
<head></head>
<body>
<script src="http://code.jquery.com/jquery-latest.min.js"></script> 
<script src="static/js/index.js"></script>
</body>
</html>

    static/js/index.js (referenced above) looks something like this:

    $(document).ready(function(e) {
    $.ajax({dataType:"html",type:"GET",url:"static/html/content.html"}).fail(function(jqXHR,textStatus,errorThrown) {
        alert(jqXHR.status);
    }).done(function(html,textStatus,jqXHR) {
        alert("Success");
    });
});

    The result is a 404 error returned via jqXHR.status per the fail method. Note, this works as expected in iOS on various devices.
