Error Generating R.Java from manifest

Did a fresh install of Titanium Developer 1.2.2 on my macbook pro. I was previously using Developer 1.2.2 with mobile SDK 4.2.1. I'm now experiencing this [error] upon compiling an app for use on the android emulator. I've tried importing the old project with no luck. I've even attempted to create a brand new project with no such luck..

Does anybody have a solution for this?

note: the kitchen sink did compile with no problems

  • Ok, I had this problem because I've simply named a new project "test" well in AndroidManifest.xml tries to declare "test" package I suppose this does not feet with the whole things.

    By naming my app "info.esion99.my_test" (using my reversed domain name) the app can be compiled.

  • Using Titanium Studio 1.0.4.2#### with SDK 1.8.0.v##### I too received "Error generating R.java from manifest" when creating an app with the Application ID "Scrolling". The problem was solved by changing the Application ID in tiapp.xml to "com.Scrolling"

    • Pete's answer worked for me. Somehow running on W7, 2 spaces got added to tiapp.xml; one in the icon name and one before the name of the app.. Eliminating them worked. So the problem had nothing to do with the manifest but tiapp.xml
