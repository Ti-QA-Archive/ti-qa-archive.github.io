Imageview seriously broken for Android, sdk's > 1.7.X

Titanium Studio

Android Only

Google API 2.2 Titanium mobile 1.6.1 the following works fine (loading image from local resource)

var f = Ti.Filesystem.getFile('images/sm_logo.png');

var imageLogo = Titanium.UI.createImageView({
    image:f,
    top:220,
    height:141,
    width:200,
    canScale:false,
});

loginwin.add(imageLogo);

This gets broken in Titanium Mobile SDK 1.7.2, 1.7.3, and 1.7.4.

In fact, if you create a new application under Titanium Studio Mobile SDK 1.7.3, and run it in the emmulator, the icons do not render for the tabs. They do render for 1.6.1. It (Mobile SDK's > 1.7.2) do not render graphics for Kitchen Sink.

The following ALWAYS appears for sdk's > 1.7.2

D/skia    (  746): --- decoder->decode returned false
W/TiDrawableReference(  746): (main) [356,356] Bitmap bounds could not be determined.  If bitmap is loaded, it won't be scaled.

It does not matter if I use the following in any of the 1.7.X SDK's

var f = Ti.Filesystem.getFile('images/sm_logo.png');
var f = Ti.Filesystem.getFile(Ti.Filesystem.resourcesdirectory,'images/sm_logo.png');
var f = Ti.Filesystem.getFile(Ti.Filesystem.resourcesdirectory,'/images/sm_logo.png');
var f = Ti.Filesystem.getFile(Ti.Filesystem.resourcesdirectory,'sm_logo.png');
var f = Ti.Filesystem.getFile(Ti.Filesystem.resourcesdirectory,'/sm_logo.png');
var f = Ti.Filesystem.getFile('app//sm_logo.png');
