Specific theme appicon ignored on android

Hi,

My application support several themes, so i created the app&#x2F;themes&#x2F; folder and put on it several directories, each one contains a theme specific style files and assets.

My problem is that the theme specific app icon file located at app&#x2F;themes&#x2F;mytheme&#x2F;assets&#x2F;android&#x2F;appicon.png does not replace the default project app icon located in app&#x2F;assets&#x2F;android&#x2F;appicon.png.

I only have this problem on android, on ios it works fine. I tried to rename, delete the default project app icon, but non of this work.

Thank's for your help.

  • Titanium SDK : 3.4.1.
  • Titanium Studio, build: 3.4.0.201409161931.
  • Host Operating System: OSX 10.10.1 (Yosemite).

  • Hello Ayoub Zamouchi,

    By default, a newly created Titanium application uses the assets/android/appicon.png file. To use density specific versions of the appicon you will need to create a folder structure in the root of the app that looks something like this:

    "<app>/platform/android/res/drawable"

    "<app>/platform/android/res/drawable-hdpi"

    "<app>/platform/android/res/drawable-ldpi"

    As you can see, you'd place density-specific versions of the appicon.png in the respective folders.
