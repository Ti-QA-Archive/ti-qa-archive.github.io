Demo mobile app: Google weather API

This is a sample mobile app that I wrote to retrieve Google's undocumented and unsupported international weather API. It's still very rough. You can find it here on GitHub.

Change the value of the wx_location variable in Resources/ui.js to change the default location.

This app might be helpful as a component of larger app, such as a conference or "brochure"-type app to show local weather conditions. I've tried to use some of the global-scope protection techniques outlined in the recent video, but still need to learn more about modularizing my code.

This app created using Titanium Studio. Any feedback or suggestions from experienced devs are most welcome.

Known issues:

Currently no error handling.

  • Hey Bob Sims,

    First, great work. I've probably spent close to four fours trying to wrestle with Yahoo! Weather API, WorldWeatherOnline.com JSON and XML, Weather.com RSS, Google's undocumented API, etc. But your code finally made everything coming together nicely, at least for Android.

    I loathe iOS, but I added the platform's functionality to your code after forking your GitHub code and making a few changes. I'm having difficulty install Git on this thing, but I was able to copy and paste my changes in the Text Editor. I changed the tiapp file, app.js, and ui.js. You can see what I adjusted, but essentially I made the Weather window appear within the app.js file's tab groups (like on Kitchen Sink), so ui.js doesn't create an overriding window.

    I defined your function and called it at the end, I guess for organizational purposes, and added a few variables above that definition, like in the twitter.js Kitchen Sink example. This removed the error messages I was getting on iPhone and Android upon application launch.

    Known issues:
    iOS background image is overridden by TableView. I have a few hunches, but I'll be making that change soon.
    iOS styling is broken slightly. Might resolve with an if='platform' or something along those lines and define two separate styles for that.
