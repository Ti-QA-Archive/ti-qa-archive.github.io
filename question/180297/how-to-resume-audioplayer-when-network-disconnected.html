How to resume AudioPlayer when network disconnected

Hi.

I used Ti.Media.AudioPlayer.
If I disconnect network, AudioPlayer fires "unable to configure network read stream" error.
Reconnected network, It cannot be resumed, and I cannot execute play() method.

When disconnected, AudioPlayer fires following "change" event.

[INFO] :   State: stopping (6)
[INFO] :   State: stopped (7)
[INFO] :   State: initialized (0)

This event is fired when Music normally finished. so I cannot distinguish between both.

How can I resume the music in Ti.Media.AudioPlayer.

thanks.

  • I had the same problem with an application I was building. The solution I use for this is to check for network changes using Ti.Network.addEventListener( &#39;change&#39;, networkChangeHandler ); Then check wether you have internet or not and start/stop the audioplayer based on that.

    
function networkChangeHandler( e ) {
    isOnline = e.online;

    switch( isOnline ) {
        case true:
            // Resume audioplayer
            break;

        case false:
            // Pause audioplayer and display error message?
            break;
    }
}
