Krollmodule c2dm notifications is null when application is closed

I am using IGx89-titanium-c2dm module to receive and handle android push notifications. When a notification is sent it is handled by the C2DMReceiver which calls the instance of the Krollmodule which sends the callback to my javascript code.

This works perfectly if the application is either running on the foreground or background. If the application is closed, I no longer receive the notifications since my Krollmodule (which is handling the received intents) is null.

I have extended the Java code in the module to test if the Krollmodule instance is null. If this is the case I create the notification within Java and create the intent to start it.
This is also working. The problem now, is that I still want to be able to run the callbacks functions in titanium since I need to use the sent variables in my javascript code.

To summarize:

  • C2DMReceiver receives c2dm intent and runs an onMessage()-function.
  • The onMessage()-function calls the Krollmodule instance to handle the data:
    -> C2dmModule.getInstance().sendMessage(data).
  • Within the Krollmodule (C2dmModule) messageCallback.callSync(data) is called and the sent variables can be used in the javascript callbacks.
  • Now, because my Krollmodule is null (since the application is closed) I am not able to call "sendMessage(data)"…

How can I activate my Krollmodule to run the needed method?
