"Swiping" tabgroup on Android

Hi,
I try to use the new feature:

var tabGroup = Ti.UI.createTabGroup({
    fullscreen : true,
    exitOnClose : true
});

and here my tiapp.xml snippet:

<android xmlns:android="http://schemas.android.com/apk/res/android">
        <tool-api-level>14</tool-api-level>
        <manifest android:versionCode="17" android:versionName="1.17">
            <uses-permission android:name="android.permission.INTERNET"/>
            <uses-sdk targetSdkVersion="14"/>
                <activity android:configChanges="keyboardHidden"
                    android:name="org.appcelerator.titanium.TiActivity"
                    android:screenOrientation="portrait" android:theme="@android:style/Theme.NoTitleBar.Fullscreen"/>
                <activity android:configChanges="keyboardHidden"
                    android:label=""
                    android:name="ti.modules.titanium.ui.TiTabActivity"
                    android:screenOrientation="portrait" android:theme="@android:style/Theme.NoTitleBar.Fullscreen"/>
            </application>
        </manifest>
    </android>

But I see the old tabgroup with short (linebroken) tabs.

What can I do?

    A few things…

    • your manifest seems to be malformed, I do not see a opening <application> - tag
    • since SDK 3.2.0, Titanium automatically targets the latest Android SDK - if you want that, don't specify one
    • since SDK 3.2.0, Titanium doesn't change the theme specified for all activities at <application /> -> you don't have to put all of your activity tags into tiapp.xml
    • here is a ticket you should watch for swipable / scrollable tabs

    • And I forgot to mention something important:

      Theme.NoTitleBar IS the old Theme. You should only use this on API Levels < 11

      Within {Project}&#x2F;platform&#x2F;android&#x2F;res&#x2F; you can add these different files (as an example):

      • values/styles.xml

        <style name="Theme.Default" parent="@android:style/Theme"></style>
<style name="Theme.NoTitle" parent="@android:style/Theme.NoTitleBar"></style>
<style name="Theme.FullScreen" parent="@android:style/Theme.NoTitleBar.Fullscreen"></style>

      • values-v11/styles.xml

        <style name="Theme.Default" parent="@android:style/Theme.Holo"></style>
<style name="Theme.NoTitle" parent="@android:style/Theme.Holo.NoActionBar"></style>
<style name="Theme.FullScreen" parent="@android:style/Theme.Holo.NoActionBar.Fullscreen"></style>

      • values-v14/styles.xml

        <style name="Theme.Default" parent="@android:style/Theme.Holo.Light"></style>
<style name="Theme.NoTitle" parent="@android:style/Theme.Holo.Light.NoActionBar"></style>
<style name="Theme.FullScreen" parent="@android:style/Theme.Holo.Light.NoActionBar.Fullscreen"></style>

      Then in your tiapp.xml:

      <application android:theme="@style/Theme.NoTitle" />

      (Android chooses the correct theme according to its current API Level)
