  • Hi Balu, Ilan, the solution to this would be threefold:

    1. First you would update the SQLite table's content with the new settings as extracted from the 'edit screen' as you call your input field on the correct row you want to update in the SQLite table.
    2. Once the update of the SQLite table has been performed a callback should be triggered, eg by forms of calling a function, by firing an app-wide event, or by inline coding a function which all have the same execution in common, namely it should first empty the tableview with yourTableview.setData([],&#39;OPTIONAL ANIMATION EFFECT (FOR IOS)-CONSTANT (or leave empty));
    3. Immediately after emptying the tableview, you can recompile the tableviewrows (just as you did in the inital listing of the tableview) and use .setData() again on the 'yourTableview'-TableView-object.

    good luck!
