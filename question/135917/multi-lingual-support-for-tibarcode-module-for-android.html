Multi-lingual support for ti.barcode module for Android

Hi all,

I am using ti.barcode module for android. It is working well on device when the English(default) language is used, but as soon as the language is changed to another language say German or French, its scanning the barcode but it doesn't match the contentType. The reason that I found for this is that the contentType returned on success callback is 'UNKNOWN', which is of course not the type of the content we will be using in callback. With default language, it returns contentType as "CONTACT", "URL", etc… So after changing it returns "UNKNOWN"..

What is the possible reason or solution for this issue ? What else I can check in order to check the type of content scanned by the barcode scanner ? Anyone is facing the same issue ?

Please provide any suggestions.. Thanks in advance for any help provided…
