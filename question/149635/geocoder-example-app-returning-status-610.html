    Hi

    I notice your original question did not include any code to show what you were actually trying. This might be why your question got little attention. People are not happy about guessing.

    You are right about the URL, this is an example;

    • http://maps.google.com/maps/api/geocode/json?region=uk&sensor=true&address=London

    This will return back the full geo information and address details. Note the extra optional parameter region, this allows you to target results to a particular territory if required - think how places match Melbourne for example.

    The order of the parameters is not important, I prefer address at the end ONLY so I can quickly swap the value if testing in a web browser.

    Now the important stuff;

    • There are usage limits on the use of this API
    • There are restrictions on how you use this API

    You can ONLY use this API when resolving information to a map that Google supplies. So you should consider the Google Maps module from the marketplace if this is going to be for iOS.
