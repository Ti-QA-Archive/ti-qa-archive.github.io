Installing on Droid 2 device - doesn't recognize SD card (60 second timeout)

I recently got a used Droid 2 w/SD card (8G) for development - the first Android I've used. Whenever I do an installation to the phone, it spends 60 seconds trying to find the SD card before actually installing. This is the repeated "adb.exe -d shell mount" call one sees other posts about.

Since I actually do have an SD card installed (I can see it under Setup), why is ADB failing to see the SD card? I did notice that if I plug in the phone to the computer via USB, and view the Storage settings, it states the card is unavailable. But if I unplug the USB, then it shows as available.

Any thoughts?

  • Accepted Answer

    But if I unplug the USB, then it shows as available.

    This is expected behavior if the USB access is 'mass storage' in Froyo or GingerBread.

    Trying changing it to 'charge only' or similar.

    Some of the newer devices allow a MTP mode which often allows access from device and from attached system at the same time.
