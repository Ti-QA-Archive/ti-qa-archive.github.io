Hello World Android app won't start in emulator on Fedora Linux

I have been trying for a couple days now to simply run a standard, unmodified new Titanium Developer Mobile project in the Android emulator. I have followed the getting started instructions closely, and everything seems to work as expected up until a certain point in the app launch process, where it always hangs. I have tried all the troubleshooting suggestions in the Getting Started guide, to no avail. Details below:

  • Titanium version: 1.2.2
  • Titanium SDK version: 1.5.1
  • Java JDK version: 1.6.0_21
  • Operating System: Fedora 14 (Linux)
  • Message in emulator: "Sorry! Process system is not responding. [Force Close] [Wait] "
  • Trace Messages: https://gist.github.com/817501

    Hi Justin

    In order to troubleshoot this, copy and paste the following into your terminal/shell so that all the commands execute successfully. Copy the output and paste it to your preferred pastebin service. It's important that you do not modify it in any way, as this could mask problems with your configuration. Do NOT paste it here! This is because the system will change the formatting.

    clear
cd /
uname -a
echo "$PATH"
echo "$JAVA_HOME"
java -version
javac -version
python -V
echo "$android_sdk"
adb version
android list target
git --version
echo " --- The following is optional ---"
scons -v
echo "$HTTP_PROXY"
echo "$HTTPS_PROXY"
echo "$SOCKS_PROXY"

    Also, take a screenshot of your android installed packages.

    Once you have provided this information, I will try to help you.

    Cheers

  • Ok, thanks to Paul Dowsett for getting me on the right track. The error I was actually getting was a "No such file or directory" error, because Titanium couldn't find the jarsigner utility. I added a symbolic link to jarsigner in my /usr/bin directory and everything works now.
