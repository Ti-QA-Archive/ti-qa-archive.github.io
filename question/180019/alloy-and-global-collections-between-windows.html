Alloy and Global Collections between Windows

I want to setup an app to use global collections between windows. From my understanding from reading the documentation, using the Collection element in the markup will create a global instance. However, I can't seem to verify this.

I can verify data is loaded into the collection in my main window, however, if I open a new window and try to access the data via another Collection element, I get no data. I've tried a number of ways: with/without the Collection element in the secondary window, copying the collection into Alloy.Globals and specifying that in the dataCollection of the TableView, and specifying the Alloy.Collections.[collection] directly in the dataCollection attribute.

What is the proper pattern for using global collections between windows?

    GAH! It turns out I had created a secondary model and had inadvertently set the adapter.collection_name property the same, which was clearing out the data.

    File this one under PEBKAC/ID10T.
