Images in a Webview

I am having issues figuring out how to reference images from HTML in a WebView for iOS and Android using a standard syntax.

This works in iOS
app://Resources/HTML/images/Logo.gif

This works in Android
file:///android_asset/Resources/HTML/images/Logo.gif

What syntax will work for both?

The tricky part of this is that the HTML that is being loaded can be loaded from one of two places. I have a default HTML file that is located at "/Resources/HTML" but when the app is opened I am downloading an updated version and saving it to the application-data folder. When I load the webview I check to see if a newer version exists in application-data and if so I load that instead. So far all the images are located in "/Resources/HTML/images/" and I need a way to reference that location that will work in both iOS and Android.

Thanks,

David

    • The reason I was doing this was because I have a default copy of the HTML built with the app and then I download updates to the applicationDataDirectory folder when the app is opened. I was not downloading the images so they were all stored in the 

      /Resources/HTML/images

      folder and built with the app. From the applicationDataDirectory folder I couldn't find syntax that worked for both Android and iOS (at least in the simulators) to reference the images.

      This worked for Android 

      file:///android_asset/Resources/HTML/images/logo.jpg

      and this worked for iOS 

      app://Resources/HTML/images/logo.jpg

      To make it work for both, I am now also downloading the images to an "images" subdirectory with in applicationDataDirectory and I changed all my image reference to be like below so everything is relative to the current path

      <img width="96" height="64" src="images/logo.jpg" />

      It was a simple fix but strange that I had to do that, so maybe there is better way. I hope that helps.
