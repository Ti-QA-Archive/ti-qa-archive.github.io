  • Your question is extremely light on details, so this is a shot in the dark.

    in your AndroidManifest.xml your application and primary activity should have something similar:

    <application android:debuggable="false" android:icon="@drawable/appicon" android:label="@string/appname" android:name="YourApplication">

    and

    <activity android:configChanges="keyboardHidden|orientation" android:label="@string/appname" android:name=".YourApplicationActivity" android:theme="@style/Theme.Titanium">
  <intent-filter>
    <action android:name="android.intent.action.MAIN"/>
    <category android:name="android.intent.category.LAUNCHER"/>
  </intent-filter>
</activity>

    note the android:label settings

    • at the root of your project you should have an i18n folder. In this folder you will have your language specific folders (ex: en, es, pt)

      In those folders are your language specific language files (strings.xml and app.xml)

      ex: i18n/es/strings.xml:

      <?xml version="1.0" encoding="UTF-8"?>
<resources>
    <string name="appname">Centro de Conversión</string>
</resources>

    • Now, for string translations in your code you will use the L() function.

      ex: i18n/es/strings.xml

      <?xml version="1.0" encoding="utf-8"?>
<resources>
  <string name="info">Info</string>
</resources>

      In your code:

      var winInfo = Titanium.UI.createWindow({  
        title:L('information'),
        backgroundColor:'#262626',
        barColor: '#262626'
    });

      title:L(&#39;information&#39;), will check the appropriate language file for the phone (defaults to en if no match is found) and replace the tag with the appropriate text strings.

      You can hardcode a fallback by adding a second param to the function:

      title:L(&#39;information&#39;, &#39;My Fallback Text)

  • Well come out it was definitely not what the other guy here recommended(doesnt work with Alloy projects). The proper way to use localization in an alloy project is in your styles file you use "textid" for your labels. So for example: 

    "Label": {
    width: Ti.UI.SIZE,
    height: Ti.UI.SIZE,
    color: "#000",
    textid: 'welcome_message'
}

    Sucks it was something that simple, there was a tutorial I followed but they used "titleid" which is why it wasnt working.
