Using Appcelerator as a Site-Specific Browser

Hey all,

I'm trying to use Appcelerator just to make a site-specific browser for my existing web app. I had previously been using Fluid and Prism, but for a variety of reasons (tough to distribute a self-contained binary, cross-platform issues, etc), I've had to look for other options. It seems like Appcelerator may be the answer, but I'm running into a few issues.

I should preface this by saying that I realize I'm not using Appcelerator for what it was designed to do, and I hope over time to take better advantage of it.

Basically, when my Appcelerator app launches, it should go immediately to the external URL I provide. I've been able to accomplish this (using an iframe or window.location or just setting the url in tiapp), but past that I can't find a way to access any of the Titanium functions from the web app itself. I would like, for example, to be able to use the Titanium notification API.

I was hoping to be able to stick something like this in my web app's JS:

parent.Titanium.Notification.createNotification(parent);

However, no matter what I try, Titanium is undefined. The last response on http://developer.appcelerator.com/question/14781/iframe-on-desktop seemed of interest, but I couldn't figure out a way to make it work.

Any advice is greatly appreciated. I'm not even totally sure if this admittedly unorthodox approach is even possible, but it would really be great if there was a way.

Thanks!
