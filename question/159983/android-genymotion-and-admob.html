Android Genymotion and Admob

Hi all,

This week I discovered the Genymotion emulator, what an improvement! After some hassle I finally got it to work, however, I have an app that uses Admob and this app keeps crashing when I use the Genymotion emulator. I get the message "Unfortunately, the app has stopped".

The error message in the console is the following:
Couldn't load ti.admob from loader dalvik.system.PathClassLoader[DexPathList (etc etc) findLibrary returned null; Titanium 3.1.3

Seems to be a clear issue but it appears not. The odd thing is, if I use the default Android emulator the Admob module works fine and the app loads. (The module is present in the global module folder and the project folder).

I start the regular emulator via Titanium Studio. For Genymotion I build the apk file with the command:

ti build -p android -b

And then I copy the file to the virtual device with:
adb install -r build/android/bin/app.apk

I've set the API level in the tiapp.xml file. The API level is set at 18 for the HTC One virtual device in Genymotion. Im using Titanium 3.1.3GA. The modules are declared in the tiapp.xml. Basically, all works correctly on the slow emulator but not in Genymotion.

Am I missing something obvious here? What could I be doing wrong?
