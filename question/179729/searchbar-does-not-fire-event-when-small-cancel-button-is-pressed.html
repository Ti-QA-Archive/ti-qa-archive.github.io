SearchBar does not fire event when small cancel button is pressed?

I need to catch an event when the small cancel button is pressed on a <SearchBar/>.

The large cancel button (showCancel="true") fires the cancel event just fine, but the little one does not.

I cannot find an event in the documentation that fires when the little button is clicked.

What am I missing here?

  • OK, it is called UISearchBarIconClear and, as far as I can see in the Apple documentation, it does not seem to play any part in the SearchBar delegate. So we are not meant to interact with a user clearing the search bar.

    Well, I personally would like to have events attached to every posible user interaction. But when Apple doesn't, I can hardly ask for it to be available in Titanium, can I now :)
