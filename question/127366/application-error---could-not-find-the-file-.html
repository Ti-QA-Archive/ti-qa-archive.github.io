Application Error - Could not find the file .....

Can anyone help on this:

I have XCode 4.2 installed with iOS5 SDK and Titanium SDK 1.7.3 all running on Lion.

My app builds and runs fine on the simulator but when deployed to device (iPhone 4 iOS5) it boots but cannot/does not load any files I link to from the navbar.

ie:

  • app.js boots and loads

  • following code in this file calls a js file in a subfolder like this:

    var win2 = Titanium.UI.createWindow({
    url:'iphone/screens/myfile.js',
    titleid:'Set #1',
    barColor: 'black'

      });
  win2.senderID = 1;

  • when this file loads, I get the red 'Application error' with the message 'Could not find the file myfile.js'

This is driving me nuts, can anyone help? I've tried putting all the .js files in the root in case there was an issue referencing subfolders but this makes no difference. I must be doing something wrong but what! Ahhhhhh!!!!!!

    • The issue with needing to change the platform specific file paths on iphone is logged in this JIRA ticket: TIMOB-5114

      Unfortunately for now, yes, if you want to use the platform specific project directory for iphone, you will need to change this path when running simulator or installing to device. An easier alternative, if appropriate for your project, would be to simply put the file somewhere else in your project's structure, so that it is not part of the platform specific directory.
