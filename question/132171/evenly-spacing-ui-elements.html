Evenly spacing UI elements

I'm starting out with Titanium Mobile and began building a prototype of the UI I wanted. I began with a window that had four buttons stacked vertically and centered horizontally. I wanted the vertical spacing between the buttons and the top/bottom of the screen to be even, but there didn't seem to be an easy or built-in way to do this. The KitchenSink app doesn't really show a cohesive UI anywhere, just uses of single elements.

Here were some of my ideas:

1) If I just set layout:&#39;vertical&#39; on the buttons' container, the buttons stack up without being spaced vertically in the container. No good.

2) If I set the top/bottom properties on the buttons, I'm going to get things wrong unless I do math based on the button heights and screen size. No good.

3) If I make one view per button and set each view to a percentage of the container, it's not quite right (the space between buttons is not the same as the space above the first and below the last). Close, but no good. It's also kind of annoying to make the views.

4) Make n + 1 spacer views for n buttons. Works but is even more annoying.

What am I missing? I could write my own code that takes lists of UI elements and spits me back a view that holds everything (n elements, n views for the elements, n + 1 spacer views), but shouldn't this be a common issue that everyone doing a UI runs into? What's the way you're supposed to do it with Ti?
