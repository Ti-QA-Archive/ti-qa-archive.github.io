    • Ti.Database.install will copy your database from your app Resources and install it onto the phone… if it does not already exist. So "persistent through builds" would be correct, as long as you don't delete the app on the test device.

      The install location is different on iOS & Android, as is the actual end-result filename. If you're running this in the iOS Simulator, you can find the "installed" version at ~&#x2F;Library&#x2F;Application Support&#x2F;iPhone Simulator&#x2F;[the iOS version you&#39;re running]&#x2F;Applications&#x2F;[some sort of GUID]&#x2F;Library&#x2F;Private Documents&#x2F;
