What's the right way to use those modules?

    • Okay I figured it out guys. The problem is related to this thread: https://developer.appcelerator.com/question/155815/already-added-lcomgoogleandroidgmsgamesachievementachievement

      In my case I had to remove the "google-play-services.jar" from the lib folder of the google analytics module. Same goes for the ti.cloudpush module if you use it in combination with the ti.map module.

      So: make sure that from all the android modules you include, only one has the google-play-service attached.

      — commented September 2nd 2015 by Achim Hoth

  • Not 100% sure that this is the same issue, but i have had trouble with conflicting dependencies when using several modules. For example when i use ti.ga in combination with ti.map and another module that i created my self, that all 3 uses google-play-services, i cannot build the app. The solution to the problem is to copy the google-play-services.jar file from one of the modules to the others, so that they all use the exact same version (the versionnumber need to be an exact match).
