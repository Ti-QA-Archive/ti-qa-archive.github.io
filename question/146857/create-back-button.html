    Hi

    The back button found on iOS (iPhone, iPod and iPad) apps is a system generated button that will automatically appear when a new window is created and opened as a child window of either a "navigation group" or "tab Group".

    If you have a navigationGroup or a tabGroup in your app you can either open a window over the top of other windows or as a child of either of the groups.

    For example, if you simply opened a window above all others;

    var win = Ti.UI.createWindow({
   height: TI.UI.FILL,
   width: TI.UI.FILL
});
win.open();

    This will simply open up over any other window displayed (unless one or more were modal - but that is a different problem).

    However if you had either a navigationGroup or tabGroup you could do this;

    var navGroup = ...
// create nav group
var win = Ti.UI.createWindow({
   height: TI.UI.FILL,
   width: TI.UI.FILL
});
navGroup.open(win, { animated: true });

    or

    var tabGroup = ...
// create nav group
var win = Ti.UI.createWindow({
   height: TI.UI.FILL,
   width: TI.UI.FILL
});
tabGroup.open(win, { animated: true });

    Both of these code examples will create a window and then slide the new window in from the right of the screen, pushing the existing window out to the left. A back button will automatically appear on the top left of the titleBar. Pressing this button will return the user to the window opened previously by pushing the new window out to the right. The back button then goes away.

    A back button is presented as a lozenge shape with a left arrow on the left side of the button - directing in a back direction. It is possible to change the caption of the back button on a window by window basis.

    You cannot however create a button that acts or looks like a back button that isn't created automatically. But you can add a normal button and have that close a window - this will perform the same function but not be the same thing "technically".

    Code examples to build both types are available in the links above.
