http status code 0 on iPhone

When doing a send() for an opened Titanium.Network.createHTTPClient, the status returned is 0 and the responseText is null. This happens about 10-20% of the time, even when doing identical server calls and is repeatable on both the emulator and iPhone. The server has been verified to have returned a valid response in all cases. The same code works 100% of time on Android. I have added a changing dummy parameter in case it was a cache issue, but this has no effect.

  • Relevant code is as follows. It's pretty simple. (target URL) is the URL of a dedicated jsp on another server.

    
var sURL="http://(target URL)?parameter1=123456789&parameter2=abcdefg";

var request = Titanium.Network.createHTTPClient(); 

request.onerror = function(){ 
        alert("Sorry, your request could not be completed, please click find flights again, status="+request.status);
        ta1.value = request.responseText;
};
request.onload = function(){ 
        ta1.value = request.responseText;
        ta1.show();
};
request.open("GET",sURL); 
request.send();
