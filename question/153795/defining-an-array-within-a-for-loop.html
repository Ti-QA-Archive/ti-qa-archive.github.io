  • Hi

    OK, firstly I have no idea which app or what code it contains, so this answer whilst (hopefully) informative you will need to take to the code to work out what next.

    • question['answers'] implies there is an object called question that has a child simple or object array
    • question['answers'].length simply returns the number of array elements so the loop knows how many times to actually loop round
    • var answer = question['answers'][i] is a helper line, it simply lets the coder access each array element simply, without the need to re-type question['answers'][i] each time, it also makes the code more readable
    • Another way to write question['answers'][i] is question.answers[i], which is a little easier to read

    Some examples for you

    • As a simple array, imagine this data;
    var question = {
    answers: [
        'Answer 1',
        'Answer 2'
    ]
};

    This would then loop as follows;

    for(var i = 0; i <; question.answers.length; i++) {
    var answer = question.answers[i];
    Ti.API.info('Answer: ' + answer);
}

    This would display the following in the console;

    Answer: Answer 1
Answer: Answer 2
    • As an object array;
    var question = {
    answers: [
        { title: 'Answer 1', score: 10 },
        { title: 'Answer 2', score: 5 }
    ]
};

    This would then loop as follows;

    for(var i = 0; i <; question.answers.length; i++) {
    var answer = question.answers[i];
    Ti.API.info('Answer: ' + JSON.stringify(answer));
}

    Note the small change here as each array element is an object - the *JSON.stringify() method allows us to display the value as a string.

    This would display the following in the console;

    Answer: { title: "Answer 1", score: 10 }
Answer: { title: "Answer 2", score: 5 }

    What you do with this information depends on your app needs of course.
