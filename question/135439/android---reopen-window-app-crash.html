    • Ti.App.addEvent creates global event - and listener function for that event remains in the memory all the time (along with everything that function references!). On the other hand, component.addEventListener is a listener that is bind to that component and will automatically be removed once component is destroyed.

      Regarding not working … you are probably doing something wrong. There are some thing missing in your code, for example: createwinSearchResults method is not defined anywhere. You probably add listener to wrong window or something like that. Do your homework. ;)

    • "js" files are not created, to be technically correct.

      Those self-invoking(!) anonymous functions makes no difference, you can remove them and you will have same result! Those functions needs to be created in situations when you don't want to pollute global scope. Since you just add a property to global namespace object, global scope won't be polluted:

      var NS =  {};

NS.method = function() {
    //do something
};

(function() {
    NS.method = function() {
        //do something
    };
})();

      This two are identical in their final result.

      Note that those methods are properties of global object - that is, object defined in global scope, and not the scope that is created by self-invoking anonymous function.

      You don't need to communicate between two JS files, you need to communicate between two JS objects. Read this topic: http://developer.appcelerator.com/question/129451/refresh-a-parent-window-from-a-child
      Hopefully it will make some things clearer to you.
