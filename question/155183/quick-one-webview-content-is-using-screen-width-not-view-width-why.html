    You have window <– main view <– browser

    Usually if you set your width to main view then in browser you can change width property to 

    
var main = Ti.UI.createView({
    backgroundColor: '#495059',
    opacity:1,
    width: Titanium.UI.FILL,     
    zIndex:10
});

    You didn't set any width to main so main fills window and you didn't set any width to browser so browser fills main view.

    Also you have this platform code going which forces platform width..Not sure if you are testing this app on an iPad.

    if(Ti.Platform.osname === "ipad"){
    main.width = 528;
    main.left = 0;
    nav.width = 528;
    webNav.width = 528;
    browser.width = 528;
    rightMenu.opacity = 1;
    rightMenu.zIndex = 6;
} else {
    main.width = Ti.Platform.displayCaps.platformWidth;
    nav.width = Ti.Platform.displayCaps.platformWidth;
    webNav.width = Ti.Platform.displayCaps.platformWidth;
    browser.width = Ti.Platform.displayCaps.platformWidth;
    rightMenu.opacity = 0;
    rightMenu.zIndex = 4;
}

    • You can set width Titanium.UI.SIZE for browser and set main width to Titanium.UI.FILL. Webview doesn't control the size of what's in the html,css, javascript of what you are displaying.

      Titanium.UI.SIZE The view should size itself to fit its contents.
Titanium.UI.FILL The view should size itself to fill its parent.

      If it's on android you can use userAgent property to tell the browser that it's a iphone or ipad even though it's an android device. search on the web for "iphone 5 user agent string"

    • Sorry webview does control the width of the content.

      
browser = Titanium.UI.createWebView({
 url:site_url,
 backgroundColor:'#EDE6E9',
 top:0,
 width: Titanium.UI.SIZE, 
scalesPageToFit: true ,
 zIndex:10 
});

      http://docs.appcelerator.com/titanium/latest/#!/api/Titanium.UI.WebView-property-scalesPageToFit
