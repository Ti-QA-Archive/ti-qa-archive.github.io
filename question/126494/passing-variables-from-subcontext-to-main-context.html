Passing variables from subcontext to main context

Can anyone suggest me with an example how could be possible to pass variables created in a subcontext (i.e. win.js) to the main context (app.js) and use these variables in the main context? Thank you in advance.

  • For simple cases I've also used the properties feature:

     Titanium.App.Properties.setBool("XYZ",true);
 res = Titanium.App.Properties.getBool("XYZ");

    Probably not a good idea for sensitive information or for lots of variables but for me it's worked well.

    • Properties are basically persisted values across all contexts (and even across application runs) (think of it like writing a value to a file and reading from one, with 1 line of code). So the setBool can be done in context1 and the getBool in context2. There are other methods for other types besides Bool as well.

    • There is one thing that can cause confusion in that blog post:

      > We refer to windows that have their own execution contexts as “heavyweight” windows

      That is not completely true, because heavyweight windows can be created without creating new execution context (by defining "modal", or "fullscreen", or "navbarHidden" property when creating new windows, but without defining "url" property).

      Also, I think global events (Ti.App.fireEvent) are not good practice for the same reason global variables are not good practice.
