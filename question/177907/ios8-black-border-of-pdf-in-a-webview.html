iOS8, black border of PDF in a WebView

Upgrading an older iPad app to iOS8
(Titanium CLI version 3.4.0-rc, SDK version 3.4.0.v20140909222515)
I found a black border around pdf pages displayed in a WebView, not appearing in previous build with older SDK an iOS versions.

I put together a test app from scratch to reproduce the layout and try to understand the origin of the new black border of pdf pages in webViews.
For the test I'm using two pdf pages, t1.pdf and t2.pdf, placed in the Resources dir.

Here is the code:

var mainView = Ti.UI.createWindow({
    backgroundColor: "yellow"
});

var webView1 = Ti.UI.createWebView({
    url: "t1.pdf",
    width: 695,
    height: 975,
    scalesPageToFit : true,
    disableBounce: true,
    backgroundColor : "blue"
});

var webView2 = Ti.UI.createWebView({
    url: "t2.pdf",
    width: 695,
    height: 975,
    scalesPageToFit : true,
    disableBounce: true,
    backgroundColor: "blue",
    borderWidth: 10,
    borderColor: "cyan"
});


var scrollableView = Ti.UI.createScrollableView({
    backgroundColor: "magenta",
    top: 20,
    views: [webView1, webView2]
});

mainView.add(scrollableView);
mainView.open();

In webView2 I tried to use the view's border in order to hide the black one which is instead visible in webView1.

It seems to work in hiding the black border, but I'm not happy with it, as the black border reappears as soon as the user zooms out in the pdf page.

Has anyone found the same issue with pdf in webViews with iOS8, and possibly other solutions? (maybe I'm just missing the right property for that …)

Thanks in advance for any hint or help.
