  • Your post is vague at best.

    • You mention memory problems and ask for help, but you have absolutely no code to show for.
    • You give absolutely no explanation of what you application does.
    • Let me guess, you developed the entire application for iPhone and then hoped it would work just the same on Android?
    • Memory problems come from bad code and Android is just less forgiving that iOS when it comes to memory usage.
    • This is not an issue specific to Titanium. There are a lots of good Android apps out there that work just fine. This is an issue related to your application.
    • If you ever decided to redo the entire application using native Android SDK, you would be faced with the same problem if the code quality is poor.

    Listen, I'm not giving you a hard time here. But you'll surely admit, that simply asking questions without code won't help you solve your problem. Of course, if you go through your code and identify sections of code that may cause some issues, feel free to post it on the Q&A and we will be glad to help you out.

    In conclusion, there is no silver bullet when dealing with software development. My best advice to you would be to look at the code line by line and identify ways you can optimise it. It's little things like that may get your app to run smoothly in the end.
