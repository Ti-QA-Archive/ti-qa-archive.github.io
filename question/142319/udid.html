    Hi Jigar,

    Apple policy now dictates that you may not use the phone's UDID in your software, as they do not want a device to be uniquely identifiable. For that reason, Ti.Platform.id now generates an ID that will be unique for that installation of the app.

    What I have done to maintain the identity of a user is to allow Titanium to generate the ID upon the first install, and then store that ID in the user's keychain. Any time the app is installed, it checks the keychain for that identifier. If present, it will use that. Otherwise, it will use the platform-generated ID. There is a good keychain module in the Appcelerator markatplace here: Keychain module.

    One added benefit to this method is that it ties the application to the user rather than the device. so even installing on another device with the same iTunes account should generate the same ID.

    Hope this helps!
