    Hi

    Yes; but you should be aware that the getCurrentPosition method may not be exactly what you should be using depending on your needs.

    The name getCurrentPosition implies the results are current, however if you read the description you will see that whilst the information is correct it may be older than you would need for your app.

    Take a look at event location, with this you can listen into the current locations position as it changes.

    A quick example;

    var lbl = Ti.UI.createLabel({
    text: 'Waiting'
});
var locationHandler = function (e) {
  if (e.success === undefined || e.success) {
    lbl.setText(JSON.stringify(e.coords));
  }
}
Ti.Geolocation.addEventListener("location", locationHandler);
