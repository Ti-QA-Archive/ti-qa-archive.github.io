SDK 1.8.1 - remote image cache without retina display

Hi,

I just noticed that the remote image cache does not show cached images with retina display.

If the image is loaded the first time, everything looks good. But on second load (now from cache folder /Library/Caches) the graphic is pixelated, seems like "hires" parameter is ignored.

Code works perfect with SDK <= 1.8.0.1 but fails with latest SDK 1.8.1

Any workaround for this bug?

Best
Danny

  • For quick fix to current mobileSDK (my case is 1.8.2)
    Just replace 2 files "ImageLoader.m" and "TiUIImageView.m" in mobileSDK directory

    get files from here
    https://raw.github.com/sptramer/titanium_mobile/db0702eb19f0ac0777c824e12982413e43f2e1b3/iphone/Classes/ImageLoader.m

    https://raw.github.com/sptramer/titanium_mobile/db0702eb19f0ac0777c824e12982413e43f2e1b3/iphone/Classes/TiUIImageView.m

    then replace here: /Library/Application Support/Titanium/mobilesdk/osx/1.8.2.v20120221114636/iphone/Classes

    rebuild
    work!

    (please back up before any replace)

  • The mentioned bugfix does not work as expected, there is no remote image caching at all for hires images (like in 1.8.0.1)

    Each time the remote image is loaded and saved to cache folder, just overriding the existing @2x file there. With no internet connection, no "cached" remote image will be displayed.

    But I found a quick workaround: you have to save the remote image additionally as regular file without @2x suffix.
