android invalid app id (with dashes)

Hi everybody,

I have the following problem: I have an iOS provisioning profile with an app ID which starts like this "de.company-domain–name-with-dashes" and I cannot change this id. If I want to build the app for Android, Titanium shows the following error:

[ERROR] :  tiapp.xml contains an invalid app id "de.company-domain-name-with-dashes.appname"
[ERROR] :  The app id must consist of letters, numbers, and underscores.
[ERROR] :  The first character must be a letter or underscore.
[ERROR] :  The first character after a period must not be a number.
[ERROR] :  Usually the app id is your company's reversed Internet domain name. (i.e. com.example.myapp

Is this "dash restriction" defined by Android or by Titanium? Is there any way to use this domain name as app id for the android app or do I have to change it for android?

Thanks!

    The App ID must conform to the Android Package Name specs, which are really just the Java package name rules as defined originally by Sun Micro. Dig around and you'll find that hyphens (dashes) are not one of the permitted characters in a Java package name, thus are not permitted in your app ID.

    • http://blog.eisele.net/2010/09/hyphen-ussage-in-package-names.html
    • http://stackoverflow.com/questions/3678520/hyphenated-company-name-in-java-packages/3960773#3960773
    • etc…

    You will have to substitute another character. The underscore is commonly used, but unfortunately, it looks like the underscore is an illegal character for an iOS bundle ID. Sorry, we have to play by the platform rules.
