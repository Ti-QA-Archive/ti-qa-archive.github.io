Titanium iOS Module development Bug?

If you create a Titanium Module with the id which contains "Module" in it, the Proxy, View, and View Proxy class name will not trigger the original <module_id> prefix name when using createXXX() method, but will remove ALL the "Module" string in the class name and trigger it.

Example:
module_id: com.test.module
Module File: ComTestModuleModule
—> This is fine
Proxy FIle: ComTestModuleTestProxy
—> will not trigger with createTest(), also the m file not found so will build fail; but will find ComTestTestProxy.m with success build.

— asked October 6th 2011 by Benjamin Peng
