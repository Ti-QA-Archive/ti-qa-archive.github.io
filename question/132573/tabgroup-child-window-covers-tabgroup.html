TabGroup child window covers tabgroup??

Hi there..

Im working on a cross platform app. Everything is working in the iPhone version, but the tabGroup in Android will not let me open a child window without it covers the tabGroup.

When the child window is opened, it covers all screen real estate besides the statusbar, because fullscreen:false.

I want to have the TabGroup visible at all time, making the app user able to change tab anywhere in the app. How is that possible?

Ti.UI.currentTab.open(childwin,{animated:true});

  • Accepted Answer

    Remove "fullscreen" property from window definition (and "navBarHidden", "modal", "url").

    The thing here is that defining "fullscreen" property (even defining it as "false") cause window to be "heavyweight" on Android, which creates new activity.

      See this example:

      var tabs = Ti.UI.createTabGroup();

var win1 = Ti.UI.createWindow({
    backgroundColor: '#fc0'
});

win1.addEventListener('click', function() {
     var twitterDetailWindow = Titanium.UI.createWindow({
        id: 'twitterDetailWindow',
        backgroundColor: '#FFF',
        modal: true,
        barColor: '#414444'
    });
    twitterDetailWindow.open();
});

var win2 = Ti.UI.createWindow({
    backgroundColor: '#3f9'
});

var win3 = Ti.UI.createWindow({
    backgroundColor: '#cf3'
});

var tab1 = Ti.UI.createTab({
    title: 'Tab 1',
    window: win1
});

var tab2 = Ti.UI.createTab({
    title: 'Tab 2',
    window: win2
});

var tab3 = Ti.UI.createTab({
    title: 'Tab 3',
    window: win3
});

tabs.addTab(tab1);
tabs.addTab(tab2);
tabs.addTab(tab3);

tabs.open();

      — commented February 23rd 2012 by Mads Møller

    • No, memory is released when window closes, no matter is it created as heavyweight or lightweight.

      Let's say that there is situation like this:

      There is one root window that opens new modal window:

      root
    ----> modal

      root window won't release memory after modal is opened. Modal window will release memory after it is closed (and if nothing references it from JS code).

      root
    <---- modal

      After modal has closed:

      root
