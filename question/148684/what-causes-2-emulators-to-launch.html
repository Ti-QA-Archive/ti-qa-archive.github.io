What causes 2 emulators to launch?

This has pretty much happened since I started with Titanium a couple years ago. Sometimes when I launch, two iOS emulators launch. One will say "You cant have two running at the same time." with a quit button. It doesn't effect anything. It's just starting to get annoying. It happens like 1 out of 5 launches.

    Hi

    This used to happen to me in the past, no longer though. I am not sure if it stopped because I now have a faster mac or a newer Studio version and Titanium SDK version.

    You are correct it does not cause any issues, it comes down to the sub-system initiating a start process on the simulator but in case it fails to launch it listens out for a success. If that is not detected it tries starting a new one so you end up with a simulator. Sometimes when the first fire command is called there is a delay long enough for the system to think that process request failed so it starts the second request process. However the first one did not fail and was simply slower than expected so it finishes and you end up with two.

    Weird but not a problem.
