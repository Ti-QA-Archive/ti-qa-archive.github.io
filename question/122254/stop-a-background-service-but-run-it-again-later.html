Stop a background service but run it again later

I'm trying to set local notifications using a background service, but I'm having trouble unsetting, and then resetting the background service when the app resumes.

The problem is that, if the background service runs, then I resume the app, things just seem to crash, I press any button and the app closes with an null error. the only way around it is to call Ti.App.currentService.stop(); at the end of the bg.js file and the app no longer crashes when resumed. But the problem then, is that the background service won't run any more, until the app is completely closed and restarted.

I'm wondering if there is a way to run a background service more than once, each time the app closes? As at the minute, my local notifications set ok the first time the app closes, but only that once.

in my app.js I have:

var service = Ti.App.iOS.registerBackgroundService({
    url:'bg.js'
});

and in bg.js:


Ti.App.iOS.cancelAllLocalNotifications();

notification = Ti.App.iOS.scheduleLocalNotification({
    alertBody:notificationBody,
    alertAction:"Open",
    userInfo:{"test":"test2"},
    sound:'pop.caf',
    date:notificationDateTime
});

Ti.App.currentService.stop();

I've removed bits that dont apply.
