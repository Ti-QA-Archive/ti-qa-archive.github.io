Writing app version to Settings.bundle

Is it possible to get a value (i.e., Ti.App.version) into a Settings.bundle DefaultValue so that the app version can be displayed in the iOS settings panel?

Presumably this could be done via a build-time script, but I can't figure out how to read the version without actually parsing tiapp.xml myself.

Has anyone done this?

  • Hello,

    Have you tried using the Properties instead?

    Titanium.App.Properties.setString("version","1.2");

    and in order to get this back you use 

    Titanium.App.Properties.getString("version");

    I am not 100% sure if the Properties are the ones that are found at the tiapp.xml. However with this piece of code you can loop all over your properties defined.

    var props = Titanium.App.Properties.listProperties();
for (var c=0;c<props.length;c++)
{
    var value = Titanium.App.Properties.getString(props[c]);
    Titanium.API.info(props[c]+" = "+value);
}

    Good luck!
