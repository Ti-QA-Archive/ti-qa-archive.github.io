WebView Titanium object access restriction through evalJS viewing HTML files in cache directory

I've got a standard mobile app I'm developing for Android and iOS, uses a webview to display a portion of content from a webpage, pulled from a proxy, and then downloaded to the Ti.Filesystem.applicationCacheDirectory as a hash, then the fully resolved path of this file is loaded within the WebView. This is all fine and dandy, and correctly loads (with the WebView "beforeload' event properly triggering). The problem comes when I want to disable / catch events on the links within this content.

I should be able to simply modify the links with WebView.evalJS (which does work) and then trigger Ti.App.fireEvent, which does not work.

In android at least (haven't tested iOS yet) it seems that the mobile browser gives an error saying that the [INFO][TiWebChromeClient.console( 5005)] (main) [9791,60545] ReferenceError: Can&#39;t find variable: Titanium (80:file:&#x2F;&#x2F;&#x2F;...).

Example code to test, given an html file with links, loaded from the app cache directory into the WebView:

    var js = "document.body.innerHTML = document.body.innerHTML.replace(/<a.*href=/gi,'XX<a onclick=\"Ti.API.info(12345)\" href=\"#\" _url=');";
    webview.evalJS(js);

If the code ran without errors, there will be two "XX" in front of each link. Clicking a link will bring up the error in the console, showing that either TI or Titanium are not around. This would make sense for a remote file, but this is a local cache file at file:&#x2F;&#x2F;. Any ideas?

Using Studio v 2.1.2.201208301612 and latest SDK 2.1.3
