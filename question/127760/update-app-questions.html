Update App Questions

I've been reading whatever posts I can find, but was hoping that somebody might be able to offer some clarification on this matter.

I have an app on Android MArketplace and App store that I'd like to update. Am I right in assuming that I just rebuild my updated app for distribution using the same methods as I originally built the file, and then just upload this as a new version in Android Publisher and iTunesConnect?

I know that I have to increment the Android versions in the Tiapp.xml, do I have to do anything similar for the iPhone version?

Also am I missing anything from this process?

Any help would be massively appreciated…

  • You are assuming right. On ITC (iTunes Connect), go to the app you want to update and select "Add version". ITC will ask you for the new version number (which must be the same as in your tiapp.xml file), revision notes and new screenshots. So you must increment iPhone app version as well in Tiapp.xml, and the same for android. Remember mark the app as "ready to upload binary" on ITC and send de ipa file from xCode organizer, as you did the first time you published the app.

    It's a safe process, I mean, you can't "broke" anything, but be sure you upload the right file and that you have tested it deeply, because revision time for ios is about 7-10 days.

    • Yes, that's it. But be sure the new version is higher than the old one and that you can only use one "dot" in your version number. 1.2 is correct, but 1.0.2 is not.

      At least when I've tried to update an app with more than one dot, upload to ITC fails saying the new version must be higher than the old one. Solution, increment always in one (1.0, 1.1, 1.2 …)

      — commented November 4th 2011 by Javier Rayon

    • Yes true, don't use 1.2.x, ITC will fail, so versioning should be
      one_digit_major_version[dot]one_digit_minor_version
