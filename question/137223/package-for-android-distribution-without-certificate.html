Package for android distribution without certificate

I'd like to package my Android app for release to the Market.
However, I'm not the publisher of the app, just the developer and my client doesn't want me to know his private key for signing the application for distribution.

Is there any way to get an unsigned app in "distribution mode"?

I already know that I can take the apk from the build folder, but that apk is in DEBUGGING mode, so that's not the best solution.

If there is no possibility to do this, could someone scalate this as a feature request?

  • I understand you're working on an app that was already released on the market, but was not developed by you ?

    As Christian said, you can't export an app without signature.

    There are three options :

    • Use your customer's keystore
    • Create a new one
    • Send your code to the customer and let him export the app with his keystore

    If your client does not want to give you his keystore + password, tell them you'll have to create a new one. But creating a new one will not update the old app, it will create a new one. You client has to make a choice about this.

  • You cannot publish an unsigned application on the Android Market. Every single application on the market must be signed for security purposes (makes sense when you think of it)

    But you simply want to distribute your app to our customer (usually for testing), you can send him the unsigned app so he can install it on his device. You can send the file by mail, Dropbox or any other means you see fit.

    ####You can find the app file at the following location:
    [APP_DIR]&#x2F;build&#x2F;android&#x2F;bin&#x2F;app.apk

    In order for this to work, you have to keep one thing in mind

    • Your customer must allow non-market applications on his device. (in the applications settings)
