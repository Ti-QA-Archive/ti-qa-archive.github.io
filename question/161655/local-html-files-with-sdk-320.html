Local html files with SDK 3.2.0

Hi all, I finded a weird behavior with SDK 3.2.0.

This script works well in iOS, but with Android the file index1.html doesn't load.

webview = Titanium.UI.createWebView({
    disableBounce:true,
    hideLoadIndicator: true,
        url: 'html/index1.html'
    });

I tryed to modify the url so: url: '/html/index1.html' and now works fine in Android but not in iOS.

What is the correct path? Thanks.

    Hello Henry,

    In short, since they are different platforms the way the refer to the url filename it's different as well.

    Workaround

    If you are using that pattern a lot, I recommend to define constants with the file names for the HTML. For example:

    if(Ti.Platform.osname === 'android'){
   URL1 = '/html/index1.html';
}  else {
  URL1 = 'html/index1.html';
}

    JIRA Bug Report

    We reported this issue here in TIDOC-1469. If you visit the bug, there is a "watch" button, if you click the button, a notification will arrive to your mail when the bug is updated.

    Thanks for helping us improve our platform!

    Mauro
