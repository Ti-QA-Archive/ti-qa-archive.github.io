Remote server timeout - retry when signal sucks?

Hi Guys,

Is there a best practice in terms of getting your app to re-try a remote server call in the event of bad connectivity or timeout?

I have a generic function that handles all my remote calls;

function doQuery(url) {
    var xhr = Ti.Network.createHTTPClient({
    onload: function(e) {
            if(xhr.responseText != '0') {
                handleResponse(xhr.responseText); // Success
            }
        },
        onerror: function(e) { 
            // Failure...  Can I call myself again?  settimeout doQuery(url) ?
        },
        timeout:5000
    });
    xhr.open("GET", url);
    xhr.send();
}

  • Hi

    The best practise is to return a failure status value from the function to whatever called it. Then announce to the user there was a problem and offer a retry dialog.

    Unless you are truly doing things in the background then you should not auto retry as the user should be offered the opportunity to find a better signal or enable wifi or whatever the user might do in this circumstance.

    It is also sensible to test for a net connection before trying to use one so you do not attempt anything that will not work anyway.
