Some localized strings isn't fetched

I have a strings.xml file that holds all of the text strings for my iOS app. The strings.xml look like this

<resources>
    <string name="registerEmailHint">Mailadress<string>
    <string name="registerNameHint">Namn</string>
    <string name="registerPasswordHint">Lösenord</string>
    <string name="registerPasswordRepeatHint">Repetera lösenord</string>
    <string name="registerCancel">Avbryt</string>
    <string name="registerDoRegister">Registrera</string>
    <string name="registerError">Kunde inte registrera</string>
</resources>

And this is the part of my view that uses the strings.

<View id="registerView">
    <TextField id="registerUsername" hintText="Ti.Locale.getString('registerEmailHint')"/>
    <TextField id="registerName" hintText="Ti.Locale.getString('registerNameHint')"/>
    <TextField id="registerPassword1" passwordMask="true" hintText="Ti.Locale.getString('registerPasswordHint')"/>
    <TextField id="registerPassword2" passwordMask="true" hintText="Ti.Locale.getString('registerPasswordRepeatHint')"/>
    <Button id="registerCancel" onClick="cancel" title="Ti.Locale.getString('registerCancel')"/>
    <Button id="doRegister" onClick="doRegister" title="Ti.Locale.getString('registerDoRegister')"/>
                </View>

The problem is that only the first textfield gets the value from strings.xml, the other one has hinttexts like "registerPasswordHint" instead of the strings.xml-value. Why is that? I have tried to remove the build and resources folder, as well as cleaning the project and reinstalling the app on the simulator.

SDK is 3.1.1, and this is an Alloy-project for iOS

    <string name="registerEmailHint">Mailadress</string>
