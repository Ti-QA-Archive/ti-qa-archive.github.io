  • When we launched 3.0 of Titanium Studio and Titanium SDK, we introduced the new Titanium CLI. Both Studio and the CLI require you to log into the Appcelerator Network to create and build projects. I'm not sure what causes it, but I suppose it's possible for Studio to get confused and not properly authenticate the Titanium CLI.

    I recommend just logging in manually. Open up the Command Prompt, then run titanium login. It will prompt you for your email and password.

    You can verify your login state by running titanium status. Once you log in, you will never be logged out unless you call titanium logout, which you don't ever really need to do.

      In the meantime, you can use the Titanium CLI to create and build your projects.

      To create a project:

      titanium create --name myapp --id com.company.myapp --platforms android,ios,mobileweb --workspace-dir ~/Desktop

      You may want to adjust the log level to see more info:

      titanium config cli.logLevel trace

      Use the titanium build command from your project folder to build it. You'll need to specify the –platform argument. Run titanium build --help for a complete list of arguments.

    1. Open Preferences > Titanium Studio > Troubleshooting page, change Debug Level to All, and make the entry com.aptana.core/debug/core checked;

    5. Look toward the end to find the command to create the project; it should be something like "titanium create …", and run the same command from the command line to see if it works.
