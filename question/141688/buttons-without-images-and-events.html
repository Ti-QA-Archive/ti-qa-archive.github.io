Buttons without images and events

Hi

Linux
Android 2.3.3/2.2
Ti 2.1.1GA

I am doing an app where the entry-window shows a vertical row of buttons.

The buttons are created like this:

A view
add a button to the view
add a label and an icon to the view

All of this is done in a CommonJS module

With NO change of code, suddenly only the first button is shown. All labels and icons are shown exactly where they need to be - but the buttons are gone.

A debug run confirms that all code is executed, and that the button objects are created, they are visible, they have the right coords each and every one,

One element on the page is draggable, using the touchstart, touchmove and touchend events. This element is an image and has nothing to do with the buttons.

removing the draggable code from this element restores Ti's ability to show more than one button, and the buttons are back to normal.

You get this? Removing eventhandlers from a non-related image fixes this error. How on earth can eventhandlers on an image object prevent Ti from showing more than one button? How?

My only luck is that the draggable image is out-of-spec, something I added as an 'extra'. I'll silently not mention it. But if it HAD been in-spec, my hind would be roast tomorrow.

Appcelerator - it's time to stop the feature-run and start the 'lets make this into a solid and reliable product'-run

I'll be leaving the community really soon if nothing happens.
