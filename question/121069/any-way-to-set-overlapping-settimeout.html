Any way to set overlapping settimeout()?

I have a pretty simple question - is there any way to use multiple settimeout() calls? For instance, I want to be able to say 

settimeout(somefunction(), 10000);
settimeout(somefunction(), 20000);

and have both timeouts function. I can only get the last one to fire, as it appears the second overwrites the first.

I have also tried assigning each call to an array with a different index, and that doesn't work either.

Is this possible? If so, how? If not, how can I schedule multiple future tasks at once?

Also, I'm on Titanium mobile 1.70 and in this case, experiencing with android, but haven't tried yet in ios.

  • Made a discovery of my own - my somefunction code was calling an android notification, that was overwriting itself. First notification showed, then was replaced by the second. Appears that the problem isn't in the settimeouts at all. Sorry for the bother.

  • It's been my experience that multiple timers within the same context will interfere with one another. Not as noticeable in the Simulator as on the device so don't assume something will work until you've tried it "live".

  • This worked for me:

    var mytimeout1 = settimeout(somefunction(), 10000);

    var mytimeout2 = settimeout(somefunction(), 10000);
