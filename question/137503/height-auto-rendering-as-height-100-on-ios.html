Height auto rendering as height 100% on ios

Since I recently upgraded to xcode 4.2 i've had rendering issues on iOS. Setting height to 'auto' results in the element filling the parent - equivalent to setting height to 100%.

In the example below, i'd expect the element 'item' to be 40 units high - the height of the child element. Instead it fills the screen. This is only happening in IOS.

So is this a bug or what?

view = Ti.UI.createScrollView({
        layout:'vertical',
        bottom:0,
        top:0,
    });

    item = Ti.UI.createView({
        backgroundColor: '#ff0000',
        height:'auto'
    });

    label = Ti.UI.createLabel({
        backgroundColor:'#00ff00',
        height:40,
        width:'100%'
    });
    item.add(label);

    view.add(item);
