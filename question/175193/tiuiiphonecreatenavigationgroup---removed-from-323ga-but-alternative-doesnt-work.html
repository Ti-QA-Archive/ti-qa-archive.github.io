Ti.UI.iPhone.createNavigationGroup - removed from 3.2.3GA, but alternative doesn't work

Dear All,

Im trying to compile an app i created in 2012 on the latest and greatest Ti/Xcode release (3.2.3GA)

I'm getting an error message indicating the Ti.UI.iPhone.createNavigationGroup is not recognised, andI've seen from the docs that it has been removed from releases > 3.2.0.

It has been replaced by Ti.UI.iOS.createNavigationWindow. But I tried that, and it doesn't work.

Here is my original code:

 navWindow = Ti.UI.createWindow();

        myApp.navGroup = Ti.UI.iPhone.createNavigationGroup({
            window: mainWindow

        });

        navWindow.add(myApp.navGroup);

If I change Ti.UI.iPhone.createNavigationGroup to .iOS.createNavigationWindow, I get an error saying that I cannot add a window to a view.

Can someone suggest what I should do?

Is there a magic button I can press that can make my 2012 successful Titanium app just work on 3.2.2.GA?
Thanks

  • Accepted Answer

    You cannot make a navigation window a child of a window object anymore. You are going to have to redesign the way you present your window stack so that the only top level windows are navigationWindows instead
