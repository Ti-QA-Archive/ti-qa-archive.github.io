iOS Doesn't Respond to Touch Events While Working

I have an app working on iPhone and Android. When loading a time-consuming screen, I can't interact with the app until the screen loads. As in, I select an item in the menu. The item starts to load. I have an activity indicator on the screen. I try to open the menu during the three-ish seconds it takes to load the screen. It doesn't respond. The screen eventually loads after those three-ish seconds. THEN the menu button's click handler happens and opens the menu again. On Android? The menu is opened right away. I even put console.log(…) calls so I could see when the event handler for opening the menu gets called and when the screen's external call is returned, etc. I definitely click the button before the call returns, but the console.log literally hangs onto the button click event until the screen renders.

Any ideas? Is there a way to force an event handler to interrupt something? The XHR call I'm making is asynchronous, right? Are they not asynchronous on iOS?

    XHR.prototype.get = function(url, onSuccess, onError, extraParams) {
    …
    extraParams.async = (extraParams.hasOwnProperty('async')) ? extraParams.async : true;

  • @Allen - the httpClient object is asynchronous by default in all versions. From the documentation;

    • Asynchronous vs. Synchronous HTTP Requests

      By default, the HTTPClient makes asynchronous requests. Asynchronous requests do not block the application and use callbacks to process responses when they are received.

      Synchronous requests block the execution of the application until it receives a response. On the iOS and Mobile Web platforms, you can make synchronous requests by setting the optional async parameter to false when calling the open method.

      The Android platform does not support synchronous requests.

    This is why the "failure" did not occur on Android as sync calls are not supported - so again - wholly down to the code by the author of this question - who had made a concious decision to specifically specify sync that caused all of his problems.
