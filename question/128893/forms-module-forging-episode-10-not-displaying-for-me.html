Forms Module (Forging episode 10) not displaying for me

Having watched the screencast, I loved this implementation for all of the obvious reasons and would like to use it in my app. My first "form" is a really simple one (a search form) for which this is probably overkill, but I'm a sucker for consistency so decided to use it in this case as well as future cases. What I'm seeing is that it just doesn't display.

I get the usual auto height/width warning, but I've inspected the fieldRefs property and the field that does exist and its height/width are the default values set in the module, so the form it exists within should have non-zero dimensions.

The one thing that comes to mind is that, unlike the module's demo app, I'm not opening the form in a window, but returning it from a factory for display within a scrollView. I haven't read anywhere that this should matter, but the implementation is one key difference.

I'm also not sure about the final form's container property. I don't quite get how that is rendered in the display so I'm not sure how it may impact the lack of visibility.

For whatever it's worth, I've compiled the standalone code and it works fine. Something in my app is failing me, but I'm having trouble finding it.

Any insight would be much appreciated. Here's my factory code:

Client.ui.createSiteAccessView = function( props ) {
    props = props || {};

    var forms = require( '/lib/forms' );
    var fields = [
        { title: 'Search', type: 'text', id: 'search' },
        { title: 'Submit', type: 'submit', id: 'searchClient' },
    ];
    var form = forms.createForm({
        style: forms.STYLE_LABEL,
        fields: fields,
    });

    Ti.API.debug( form.fieldRefs.search.width );

    form.addEventListener( 'searchClient', function( e ) {
        Ti.API.debug( e );
    });

    return form;
};
