    The way you're approaching this is not ideal. You should store that value in the app properties using Ti.App.Properties instead. That way, your value can persist across app launches, too.

    Pro tip: Ti.App.Properties.getBool() takes a second argument, the default if it hasn't been set yet. So, in your code where you need it, use Ti.App.Properties.getBool(&#39;autoShowChart&#39;,false);. Then, in your preferences screen, put Ti.App.Properties.setBool(&#39;autoShowChart&#39;,switchChartOption.value); in your change event handler.
