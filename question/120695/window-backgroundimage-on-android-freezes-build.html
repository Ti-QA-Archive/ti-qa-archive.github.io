Window BackgroundImage on Android Freezes Build

1.7 Release Candidate 1 - Android - Mac OS X.6.7

I'm setting a tabgroup window's background image to a 266kb file in Android. VERY rarely do I have a successful build on Android if I include this image, the build will freeze unless I comment out the line of code that sets the window backgroundImage. I found this problem using Android fastdev and can visually see the fastdev log stop at the bg.png file. Thinking it was a fastdev issue, I turned off the fastdev server, and still experienced the app failing to fully deploy. If I comment out the backgroundImage from the window I can consistently get the Android build to succeed, both with fastdev and without.

Anyone have any issues with window background images? Any ideas Titanium team why a 266kb file would stop the build from completing on Android?
