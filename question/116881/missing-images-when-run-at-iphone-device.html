Missing images when run at iPhone device

My app runs without problems under the iPhone simulator, but when I install the app at the actual device, all images were missing - icon, background images, all pics gone.

What could be the cause of the problem? Anyone help would be much appreciated!

I am using the 1.6.0 SDK with Titanium Developer 1.2.2.

Jimmy

    I had this same problem and was happy to see that the solution was a simple one in my case. I had placed my files in a folder called 'Images' with an uppercase 'I' in my project folder but had referenced it as 'images' with a lowercase 'i' in my code. As soon as I changed the folder name to 'images' with the lowercase 'i' everything worked fine.

  • One possible reason for this discrepancy is that image and file names on the iOS device are case-sensitive, whereas they are not on the Mac OS filesystem the simulator uses. This is one potential cause for the problems you're experiencing.

    It could also be that our build scripts are running into problems processing your images - the most common culprit for this in the PNG Crush utility we use to try and minimize the final size of your binary by compressing .png images. However, if an image has a .png extension but is in fact just a renamed file of another type (JPEG, GIF, etc), that will cause PNG crush to fail, which may cause problems.

    Before diving in to either case, the first thing to try is clearing out the contents of the build/iphone directory (leaving the directory intact), and allow Titanium to regenerate all of the native files necessary for your application. If that doesn't work, check the two items listed above. If THAT doesn't work, pastie.org the contents of your build/iphone/build/build.log file to give the community a little more insight into what could be amiss.
