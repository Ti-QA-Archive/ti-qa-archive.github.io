  • App type: mobile, Android-only
  • Titanium SDK: 3.1.0.GA
  • Titanium Studio: 3.1.0.201304151600 (jenkins-titanium-rcp-master-417)
  • Platform: Android 2.x, Android 4.x
  • Host OS: Ubuntu 13.04

I have an Android-only app I made with Titanium & Alloy. In development, we have tested it with no issue on the following hardware/software configurations:

  • Samsung Galaxy S2 (T-Mobile) running Android 4.2.2 (CM 10.1-20130423-NIGHTLY)
  • HTC Desire Z (T-Mobile G2) running Android 2.3.7 (CM 7.2.0)
  • Samsung Galaxy S running Android 2.2 stock ROM
  • Samsung Galaxy Note 2 running Android 4.1.1 stock ROM

Also tested w/ no issues on the following AVDs (creating using device definitions that come with AVD, only changes made are to the target API level that the AVD is running):

  • Nexus 4 on API level 16 & 17

  • Thanks for your response, Mauro.

    With regard to the back button, I have an androidback event listener on $.index that performs my back nav event. The function for this event pops the name of the last view off of the breadcrumb stack, creates a new copy of that view, adds it to the window, and removes the current view from the window. When the back nav event determines that it can't go back anymore, it is time to exit the app, and it does this by calling Ti.Android.currentActivity.finish();. This works fine on all of the emulators and other devices I have tested so far.
