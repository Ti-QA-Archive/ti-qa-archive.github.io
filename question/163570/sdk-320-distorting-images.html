SDK 3.2.0 distorting images

Titanium SDK 3.2.0 and above, iOS 7.1, Working on a Mac, Titanium Studio 3.2.1

Hi folks,

I have recently rebuilt our app using 3.2.0 and 3.2.2, some images have now become distorted. The images are double the size in pixels than specified in the code, I did this because the same image could be used for retina and pre-retina devices. I have spent most of the day trying to figure out what would be the cause but I'm at a loss.

What it looks like now

What it should look like

Below is a sample of the code I am using for the icons and the view that it sits in:


var botline = Titanium.UI.createView({
    left:59,
    top:35,
    height:17,
    width:225,
    layout:'horizontal'                                 
});
post_inner_view.add(botline);            

var carpark_distance_icon = Titanium.UI.createLabel({
    backgroundImage:'images/list/icons/distance_icon.png',

    height:12,

});
botline.add(carpark_distance_icon);

Its happening here and also in another part of the app, it only seems to be small icons from I can see, the rest of the images are fine.

If anybody has any ideas I would greatly appreciate your advice.

    Hi Paul,

    assuming that you are targeting iOS, I have two points which could help you with this issue:

    1) "The images are double the size in pixels than specified in the code, I did this because the same image could be used for retina and pre-retina devices."

    –> Retina images must use a @2x ending in their filename

    so rename your (double sized image) "distance_icon.png" to "distance_icon@2x.png" and generate a new image from that with half the resolution and name it "distance_icon.png" and put them in the same folder

    In your code just keep the plain name, on retina devices it will automatically search for a @2x version

    2) I don't understand why do you use a Label to show an image

    –> use an ImageView or View instead

    Hope this helps
