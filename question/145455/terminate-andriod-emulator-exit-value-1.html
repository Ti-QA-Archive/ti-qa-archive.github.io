  • Hello,
    don't disable the fastdev. Do you have an android device around? If you have, go for testing on the device. The emulator is usually super slow.

    On the other hand, is this Windows 7 64 bits? Are you using Java 32 bits or 64 bits? You should use 32 bits.

    In addition to that, one trick that usually works is restart the compilation process, the second time, the emulator will be there.

    And, when you finish testing. Don't close the emulator. Keep it open and will be reused in the future testing.

    Best,
    Mauro
