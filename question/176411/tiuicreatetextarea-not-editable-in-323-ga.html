Ti.UI.CreateTextArea not editable in 3.2.3 GA

I'm using Ti.UI.CreateTextArea with success in 3.1.3 GA (iOS SDK 5.02). No problems with this functionality. I can edit, pre-populate, and save without an issue.

In testing with 3.2.3 GA and OS SDK 5.11, I lose all functionality of the textareas. They render, but are not editable and I can't enter text.

Same code, different results. Have not been able to find any bugs, or other issues with TextAreas in the 3.2.3 GA release.

Any help would be appreciated.

      In 3.1.3 GA, the textfield was being added to a label. For some reason it worked.

      — commented July 23rd 2014 by Mark Smith

    • You have stumbled upon an old issue that let the dev add kids to parents that were not good parents.

      iOS would allow it (I think this has been normalised now - likely given what you found).

      You can only add views to containers, so; window, view, scrollview are containers, but; labels, buttons, switches are not containers.

      You can of course nest containers so view inside view inside view is perfectly fine.

      So it was hidden/covered ;-)
