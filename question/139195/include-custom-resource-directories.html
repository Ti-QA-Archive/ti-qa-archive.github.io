Include custom resource directories

I have an app that I would like to "skin". For that purpose I'd like to have several resource directories holding a skins graphic assets. Dependent on a configuration I'd like be able to "switch" between the skins, so I can build the app choosing any of the skins.

Is it possible in Titanium Mobile to tell the build process, which directories to include (and which not to). Can this be done by an entry in the tiapp.xml?

Anybody a solution for this?

  • Entries in tiapp.xml can be accessed via the Ti.App.Properties methods.

    This would allow you to switch between packaged assets.

    However, you are looking for prepackaging solution. You can build your projects from the command line, so could implement a script that would copy over your resources at build time.

    execute your projects from the command line using Make

    Matt Apperson's MakeTI
