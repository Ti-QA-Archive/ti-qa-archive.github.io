Map Precision

Want to draw this point on a map, with matching zoom.
I have got the data from the 'regionChanged' event previously.

location.lat = -36.08032061668828;
location.lng = 146.91621780395508;
location.zoom_lng = 0.054931640625;
location.zoom_lat = 0.04439524504623904;

Now I use this code to render the map:

if (scale_map) {
        var town_name = central.getTownAttrByID(town_id, "Title");
        var centre_pt = central.getTownCentre(town_name);
        var location = new Object();
        location.lngDelta = centre_pt.zoom_lng;
        location.latDelta = centre_pt.zoom_lat;
        location.lng = centre_pt.lng;
        location.lat = centre_pt.lat;
        central.centreMap(map_view.the_map, location, location.latDelta, location.lngDelta);
    }

And I have the region changed event:

the_map.addEventListener('regionChanged', function(e) {
        central.printObject(e);
    });

Just prints out regionChanged data:

As you can see the latitudeDelta is not the same as the one I supplied to the centering function, so the map is not precise.

  • Hi Steven

    It is possible and likely that either of the delta values will change. There are a couple of reasons for this;

    • The latitudeDelta and longitudeDelta are related to the rectangular region the map itself occupies, so if your scale ratio between the two axis is not the same as the ratio between the width and height of the map rectangle then the maps internal deltas are adjusted - so the map visual remain with a 1 to 1 scale.

    • The map itself can also adjust the center point - even if you supplied one - if the content of the map requires it to adjust it position. For example if you add a push pin or a pop up window from a push pin. The display of these may require the map to move slightly to show the pin/pop-over within the visual area.

    This situation is normal and expected.

  • Hi Stephen

    I have a solution (that almost always works), a reason (that I am not sure I understand completely why) and some code that will show you how I resolved it.

    Firstly, I must say that I thought you were seeing things - you were not, this problem is mental, so for that I apologise.

    The problem has nothing to do with adding the 'Event Listener' regionChanged, this does not affect the map simply by using it. That said the event does report values that appear to be way off from what was previously supplied. This situation is not caused by minor adjustments to accommodate any pins or pop-overs.

    The reported delta values are much larger than the ones supplied, these do not appear at first to have ANY relation to the difference between supplied and returned. I spent some time trying to resolve this, playing with regionFit and allowing for the fact that the mapView will try and size to the nearest zoom level and not always the actual match you supplied.

    I then started looking for any reason why and found a Q&A titled Why do I have to divide latitudeDelta and longitudeDelta by 5?. This was never answered by anyone at the time - I will be adding a reference back to this answer and a comment to see if the author Kenneth Lewis ever got further.

    His comment about dividing by 5 set me trying to work out if there was any actual relationship to this problem. Using his factor it sometimes worked and other times it did not, but it was close.

    I was using Iceland as a test region (trying to fit the whole country in the window), and I realised that if you divide lat by lon you get a number that when multiplied by 10 gave a figure very close to 5. This was close to the number Kenneth came up with. Different locations have different factors results, this seems to swing from north to south poles. But each time the ratio is used to divide against each delta, the results are usually (mostly) right, simple tweaks to allow for the nearest Google zoom - can resolve any remaining issues.

    So to calculate the adjustment you do the following;

    var ratio = Math.abs(e.latitudeDelta / e.longitudeDelta) * 10;
var longitudeDelta = e.longitudeDelta / ratio;
var latitudeDelta = e.latitudeDelta / ratio;
