GPS Not working the first time...

okay here is my code. I have timers set so the GPS has time to find where it is at so I don't get an old position or a undefined location.

My problem is when I hit the button ~cmdGetCords~ I get Undefined terms the first time. then the second time I hit the button, it gives me the coords and then shoots out an error saying "Cannot read property "latitude"…. here is my code

cmdGetcords.addEventListener('click', function()
{        

        Ti.Geolocation.distanceFilter = 0;
        Ti.Geolocation.accuracy = Ti.Geolocation.ACCURACY_BEST;

        Ti.Geolocation.getCurrentPosition(function(e)
        {
            latitude = e.coords.latitude;
            longitude = e.coords.longitude;
        });    

        Ti.Geolocation.addEventListener('location',function(e)
        {
            latitude = e.coords.latitude;
            longitude = e.coords.longitude;
        });

        for(timer = 0; timer <= 5000; timer++)
            {
                    txtCords.text = "Please wait...";
            }

        Ti.Geolocation.removeEventListener('location',function(e){});

        for(timer = 0; timer <= 5000; timer++)
            {
                txtCords.text = "Please wait....";
            }

        txtCords.text = 'latitude: ' + latitude + ' longitude: ' + longitude;

});
//GPS CODE ENDS

let me know if you have questions about the code I will answer them….

    OK the issue is that the location does not always update on the first shot through.

    On the Titanium training course Mr Whinnery said always get your location twice first time… This is not a titanium bug, just how it works..

    So code accordingly. I apply this to our app and it works every time.
