Titanium Bug on Android with Ti.UI.SIZE ?

Hi all, I have a ScrollView and a Tableview in it.

To keep the size of the Tableview at the minimum I write Ti.UI.SIZE in the height property.

At this point I run the App and I start to add rows dinamically.

Teorically when I add a Row, the height of the Tableview will increase with each row added.

Effectively it works like I said, but when the Tableview reach the same height of the screen (more or less), stops increasing the height.

All other rows will be viewed only scrolling the table, but there the user is not able to scroll the TableView.
Someone knows why this happen?

I think it is a bug.
The behaviour of the Ti.UI.SIZE in the documentation is described as follow:

The SIZE behavior means the view will constrain its size fit its contents.

But this is not true, in this case.

In iOS the same code works perfectly.
This is the code of my TableView:

var tableView = Ti.UI.createTableView({
    data: [],
    top: 15,
    width: "94%",
    height : Ti.UI.SIZE,
    border: 4,
    borderColor: "orange",
    borderRadius: 6,
});
