Button click fails on Android/Alloy if View has no height

function actionLogin(e) {
    alert('Login Action');
}

function actionRegister(e) {
    alert('Register Action');
}

$.winLogin.open();

<Alloy>
    <Window id="winLogin" class="container" fullscreen="false" exitOnClose="true">

        <View id="loginView" layout="vertical">
            <TextField id="inputUsername" hintText="Username" />
            <TextField id="inputPassword" hintText="Password" />
            <Button id="buttonLogin" onClick="actionLogin" title="Login" />        
        </View>

        <View id="doRegisterView" height="46">
            <Label onClick="actionRegister">Click here to register</Label>            
        </View>

    </Window>
</Alloy>

The above code runs as expected in Android emulator 4.0.3, but if I remove the height="46" on the 2nd view, the login button click fails to execute (the button style on click doesn't change neither does the alert fire).

Previously I had the 2nd view position set to the bottom of the screen, but the login button would not fire without the height being set on the doRegisterView.

I'm not sure if this behaviour is a bug or a side-effect of me not adhering to an expected way of working with views?
