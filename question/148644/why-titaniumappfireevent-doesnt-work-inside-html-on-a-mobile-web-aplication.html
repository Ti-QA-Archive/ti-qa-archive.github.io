Why Titanium.App.fireEvent doesn't work inside html on a mobile web aplication?

I have a project for mobile web created with Titanium 3.0.2.
The application has a header with a webview, and the content with a tableview.

On app.js I have the next lines for catching the event named menuSelected:

Ti.App.addEventListener('menuSelected',function(e){
    alert('selected!');
});

Inside of the ui/common/HeaderView.js I have a webview like this:

var menu = Ti.UI.createWebView({
    height: 35,
    backgroundColor: 'transparent',
    url: 'html/menu.html'
});
self.add(menu);

Then, inside of the html/menu.html have the next function:

<html>
<head>
    <script type="text/javascript">
        function selectMenu(name) {
            Titanium.App.fireEvent('menuSelected',{});
        }
    </script>
</head>
<body>
    <a href="#" onclick="selectMenu('link name')">Menu 1</a>
</body>
</html>

When I run the project on "Mobile Web Preview in Browser" and select the link for "Menu 1" nothing happens but the Firebug's console display the next message:

ReferenceError: Titanium is not defined
Titanium.App.fireEvent('menuSelected',{});

The same project executed on iPhone simulator works fine, but on the web browser (even on safari over iPhone) always have the same error.

If anyone have a solution or an alternative way to fire events from html embeded and handle those events in the main application, I willl be very grateful.

Thanks in advance.
