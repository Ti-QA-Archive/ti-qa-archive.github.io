Prevent Window close on Back button

In an android application, is this not supposed to prevent the window close?

win.addEventListener('close', function(event)
{
Titanium.API.log('ERROR', "close fired");
event.preventDefault();
event.stopPropagation();
return false;
});

    Hi Ravi

    No, the event for the back button is 'android:back' rather than 'close', as the following demonstrates:

    win.addEventListener('android:back', function(e) {
    Ti.API.Info("Pressing Back Will Not Close The Activity/Window");
});
