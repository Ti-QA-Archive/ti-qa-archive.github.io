PNG image crashes application unexpectedly

I am working with android development at the moment and have been having some wierd issues with animations.

I have been creating animations for my application and everything has been going fine. I have been drawing images in GIMP and exporting the PNG images to my images directory in titanium, then assigning the images as the backgroundImage property in my view that is being animated. All was going well until my application unexpectedly crashed.

The old images I made in GIMP and exported to my project still work fine, but now if I create an image in GIMP the same way as before and export it into my project, then assign that new image as my backgroundImage property then my application quits with a 'unfortunately your program has stopped'

I thought maybe this was a memory issue, but I downloaded images from the internet and put them into my project folder and they all work just fine. I have tried exporting my images from GIMP as jpg, png, bmp, gif and nothing. I also tried the same process in paint, but that doesn't work either.

Here is my working code:

var view3 = Ti.UI.createView({
            backgroundImage : '/images/animation1.png',
            height : '45%',
            width : '50%',
            right: 0,
            top: 0,
            opacity : 0.0
        });

and here is my broken code:

var view3 = Ti.UI.createView({
            backgroundImage :  '/images/animation2.png',
            height : '45%',
            width : '50%',
            right: 0,
            top: 0,
            opacity : 0.0
        });

where animation1.png was created last week and animation2.png was created today. I can open all of these images in numerous photo viewer applications and they open and appear fine. I tried exporting different pixel sizes and everything. This is causing quite a headache, any ideas?

    • I figured it out!
      You can't use 000000 html black as a background color in your image file, titanium doesn't like that for some reason. You can use this color as text color, but not as the background of the image. I did notice a proprietary three digit color notation in titanium instead of standard html 6 digit color codes. This only crashes on android for some reason, possibly a bug?
