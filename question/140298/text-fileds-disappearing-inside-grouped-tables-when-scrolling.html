Text Fileds Disappearing inside grouped tables when scrolling

I have written a much simplified version of what is happening:


var win = Ti.UI.createWindow();

var allFields = Ti.UI.createTableView({
    style : Ti.UI.iPhone.TableViewStyle.GROUPED,
});

allFields.data = [makeSection(), makeSection(), makeSection(), getEmptyViewSection(), getEmptyViewSection(), getEmptyViewSection(), getEmptyViewSection(), getEmptyViewSection()];

win.add(allFields);
win.open();

function makeSection() {
    section = Ti.UI.createTableViewSection({
    });
    var aRow = Ti.UI.createTableViewRow({
        selectionStyle : Titanium.UI.iPhone.TableViewCellSelectionStyle,
        title : "Name",
        height : 30
    });
    var editableTextField = Ti.UI.createTextField({
        clearButtonMode : Titanium.UI.INPUT_BUTTONMODE_ONFOCUS,
        textAlign : "right",
        value : "May disappear when scrolled!",
        width : "80%",
        right : 10,
    });
    aRow.add(editableTextField);
    section.add(aRow);
    return (section)
}

function getEmptyViewSection() {
    section = Ti.UI.createTableViewSection({
    });
    aRow = Ti.UI.createTableViewRow({
        height:80
    });
    section.add(aRow);
    return (section);
}

Basically, if I scroll the page up, the fields on the top come back empty when I scroll back down. (Actually, they are there, just in a weird invisible state). It seems to be sporadic, not always the same fields disappear.
Has anyone else encountered this? Any help?

Thanks in advance.

  • Accepted Answer

    One very important thing that you need to do, is declare you variables.
    Like this

    var section = Ti.UI.createTableViewSection({});

    instead of this:

    section = Ti.UI.createTableViewSection({});

    As for the problem you're seeing, it looks like the title is actually covering the text field when scrolled. If your rows are going to be custom, you should create labels for the titles instead of using the "title" property (btw, in iOS the title is really a UILabel as well). Commenting out the "title" property in your example makes the problem go away.
