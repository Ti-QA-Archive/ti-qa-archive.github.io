Titanium 1.2.2 + Titanium 1.5.1 sdk + Tibar 4.2.0 = cannot install into device

Hello,
i red all the posts but I think there is no current clean solution to this issue: if I am using Titanium 1.5.1 with Tibar 4.2.0 I am able to compile and run my project into the emulator (with some tricky work, as stated for example here: http://developer.appcelerator.com/question/94391/issues-with-barcode-scanning-module-tibar-in-sdk-15x) but there's NO WAY to have it running on device from Titanium. I always get this error:

Install Error

Error: Traceback (most recent call last):

execute_xcode("iphoneos%s" % iphone_version,args,False)

My workaround is currently to open the project directly from Xcode and to build for the device and to manually move [myProject].app into iTunes. This seems to work. One side effect is that if my application uses localization I have to manually fix the missing localization files (see here for this other workaround: http://developer.appcelerator.com/question/98811/i18n-titanium-vs-xcode).

I am guessing if somebody was able to find a cleaner solution to this problem.

I hope the great Titanium team will fix this in the next release, as it is very annoying.

Thank you
