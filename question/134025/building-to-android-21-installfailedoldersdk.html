Building to Android 2.1 [INSTALL_FAILED_OLDER_SDK]

Hi,

I've got Titanium up and running with SDK version 1.8.2. After creating a plain new project with the Tabbed template, I can build to an IPhone simulator fine, no problems. I can also build to an Android 2.2 simulator with no problem. As soon as I try to build to Android 2.1 simuator, I get an error message saying:

[ERROR] Failed installing com.xxx.xxx: pkg: /data/local/tmp/app.apk
Failure [INSTALL_FAILED_OLDER_SDK]

Some things I've already tried/made sure of:

  • I'm building to android runtime: Rhino, not V8 (as I understand it only 2.2 and above support V8
  • I have Android 2.1 and 2.2 SDK's installed, as well as Google API's 2.1 and 2.2
    -I've tried setting the default android SDK to both Google APIs Android 2.1 and just plain Android 2.1.
  • I've tried creating a new project with the default Android SDK pointing to both of these. The same thing happened.

Is their some hidden build setting I'm missing? Is it just not possible to build to Android 2.1 with Titanium SDK 1.8.2?
If it helps, I'm working on OSX Lion.
