Custom AndroidManifest problems...

Alright so according to Maintain a Custom AndroidManifest.xml, the "androidManifest.custom.xml" file is no longer required to update your Android app in the Android Marketplace (Changing the versionCode & versionName).

But whenever I try changing it via the tiapp.xml file, it always defaults back to 1. Same with the other things they say (Putting a "androidManifest.xml" in the "platform/android" directory).

So I decided to try the old way (androidManifest.custom.xml) and it worked perfectly, allowing me to update my App through the Android Marketplace in one go, rather than the 10+ times I tried the other ways.

So just saying… It seems your guides are a little dated or the coding is wrong?

  • You've uncovered a bug, namely this: If the only thing you set in the android->manifest section of tiapp.xml is an attribute of that element – and not any child elements – then we're missing that. We must have some bad logic in there that first checks to see if you've put in any custom child elements for "manifest", and only then does it process your customizations. I've created a ticket for that:

    https://appcelerator.lighthouseapp.com/projects/32238-titanium-mobile/tickets/2973
