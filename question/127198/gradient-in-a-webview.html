The file t.html I want to use runs fine in a desktop browser, but not in the webview

<!DOCTYPE HTML>
<html>
    <head>
        <meta http-equiv="Content-Type" content="text/html; charset=utf-8">

  • More specifically, it shows a black background in the iOS Simulator (with XCode 4.2). Putting this on my device it shows properly the gradient and also animates the chart.

    Related to this, I had the html code in a string, but 

    var webview = Ti.UI.createWebView({
    top: 0,
    height: '100%',
    html: html
  });

    does not work on my device (no gradient and no animation), whereas 

    var webview = Ti.UI.createWebView({
    top: 0,
    height: '100%',
    url: 't.html'
  });

    does work

    So I had to put my html string in a file in Ti.Filesystem.applicationDataDirectory

      var f = Ti.Filesystem.getFile(Ti.Filesystem.applicationDataDirectory,'web.html');
  result = f.write(html, false); // write to the file

    and reference the file with a complex url

      // http://developer.appcelerator.com/question/81571/what-is-the-url-for-applicationdatadirectory
  webview = Ti.UI.createWebView({
    top: 0,
    height: '100%',
    width: '100%',
    url: 'app://resources/../../../../../../../../..' + f.nativePath.replace(/file:\/\/localhost/, ''),
    backgroundColor: template.backgroundColor
  });

    But it works, and will see what Android brings us.
