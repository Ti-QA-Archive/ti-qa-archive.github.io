Feature Request: Add "current titanium version" in forum posts

Here is a feature request that some of us have been wanting for a while. Please vote this UP if it makes sense so we can get some visibility to the Powers That Be.

These Q&A Community Forums are so well laced with great answers and bits of knowledge often critical to success, that they are the primary documentation in some ways for people. When searching for answers, we get some 2+ year old questions and responses in here, which is generally great once you know to look at the "posted 2 years ago", before you jump back and start changing your code. ;)

So The Feature:
What would a GREAT HELP would be to have the webserver here dynamically add a "Posted when current Titanium release was 1.2" to that date when displaying the articles.

This would help you determine at a glance how relevant a certain bug or behavior is, to give some context for the answers. Seems like a simple lookup thing to add on server side to the date formatter code.

— asked September 29th 2011 by Jeff Askew

  • You make a good point when it comes to finding relevant issues to the version of Titanium your using. For this we do provide, and encourage the use of tags.

    Im not sure how good a system that used dates to determine Titanium version would be, as not everyone is using the latest released version… as an example, the current version is 1.7.2, we are about to release 1.8 (that some users are using) and yet some people are still posting about 1.6 (that we still support).

    But you do make a good point that tags (or some slightly altered system) is needed to help users find the content they need.
