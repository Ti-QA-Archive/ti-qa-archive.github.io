eval_js works, then it doesn't

I have an app that will load a map view, then upon clicking on an annotation will open a web view, that has this attached to it: 

 profile_view.addEventListener('load', function(m) {

    m.source.evalJS("alert('load fired')");
    m.source.evalJS("set_variables('blahg')");

});

Now, the profile_view is a web view 

profile_view = Ti.UI.createWebView({
    url: '../profile_view.htm'
});

That calls some javascript to set some variables inside it..

(in profile_view.htm)

<script type="text/javascript" src="web_js/profile.js"></script>

where the function set_variables() looks like this (for experimentation purposes)

function set_variables(profile_info) {
    alert("Set Variables Called");
}

So my problem is that this works perfectly expected on the emulator.. (iphone 4)
but on a development device (iPad) the only thing that gets alerted is "load fired".

Why is it functioning differently on device versus the emulator?
