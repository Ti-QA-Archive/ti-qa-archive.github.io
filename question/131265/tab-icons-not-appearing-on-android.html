Tab Icons Not Appearing on Android

I have no idea why. I've cleaned the build, restarted the emulator, turned off fastdev and I think I even rebooted yesterday (for another reason, but still). I'm only supporting one Android configuration right now, so the tab images, like all of the other images for that platform are stored in &#x2F;Resources&#x2F;android&#x2F;res-long-port-hdpi&#x2F; (in a tabs&#x2F; subdirectory). I'm loading the images like so:

    var tab_locations = Ti.UI.createTab({
        icon: '/images/tabs/locations.png',
        title: 'Locations',
        window: Client.Views.Locations.createIndexView(),
    });

All of the other images in that path (and referenced similarly) load fine. Likewise, the iPhone tab icons load fine from &#x2F;Resources&#x2F;iphone&#x2F;images&#x2F;tabs&#x2F;[image_name].

Any idea what I'm missing here?

Thanks.

  • In 1.8.0.1, multi-density tab icons are not being recognized in the android/images folder (there is a Jira ticket already). Other images are being recognized. I had to create a workaround by adding my images to the Resources/images folder with separate file name suffixes (e.g., radar-hdpi.png, radar-mdpi.png, etc.) and then check for density to determine which file to use.
