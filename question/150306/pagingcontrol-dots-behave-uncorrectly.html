PagingControl dots behave uncorrectly

Hi all, I have a strange behavior with the pagingControl dots in a scrollableView. I simply need to know if this is a known bug or something that I can solve.

The problem is: the dots show and lights correctly but NOT if it's selected the last view. In this case the last dot lights but then returns to the previous one.

Practically, my scrollableView contains 3 views, the dots behave correctly for the first 2 views but the third one returns the dot to the second (the view is however shown with no problems).

I've tested with 2 views and when I select the second view the light remains on the first dot only, so the problem seems to pertain to every last view.

I have no error/warn in the console and no useful log.

var matchView = Ti.UI.createView({ 
    layout:'vertical'
});
var voteView = Ti.UI.createView({ 
    layout:'vertical',
    backgroundImage:'images/grass_tile.png'
});
var tacticsView = Ti.UI.createView({ 
    layout:'vertical',
    backgroundImage:'images/grass_tile.png',
});
var cont = Ti.UI.createScrollableView({
    top:'3%',
    bottom:'3%',
    right:'3%',
    left:'3%',
    borderRadius:8,
    backgroundColor:'white',
    views:[matchView,voteView,tacticsView],
    showPagingControl:true
});

Application: mobile => iOS simulator and device (iPhone 4S)

Titanium: last

Thanks for every advice!
