  • Hi

    The getCurrentPosition method may no be what you think it is.

    Retrieves the last known location from the device.

    On Android, the radios are not turned on to update the location, and a cached location is used.

    On iOS the radios may be used if the location is too "old".

    This method reports any updates back to the app.

    var lbl = Ti.UI.createLabel({
    color: '#000',
    text: 'Waiting',
    textAlign: 'center',
    top; 10,
    width: TI.UI.FILL
});
win.add(lbl);
var locationHandler = function(e) {
  if (e.success === undefined || e.success) {
    lbl.setText(JSON.stringify(e.coords));
  } else {
    lbl.setText('Problem');
  }
}
Ti.Geolocation.addEventListener("heading", locationHandler);

    You should turn this off when you are not using it;

    Ti.Geolocation.removeEventListener("heading", locationHandler);
