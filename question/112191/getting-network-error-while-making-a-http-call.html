    Simplify your usecase:

    Make sure you can connect to a simple webpage, preferably static html that has no need for authentication. Also, try using the actual IP address, like http:&#x2F;&#x2F;127.0.0.1&#x2F;webpage.htm or http:&#x2F;&#x2F;your-local-ip-address&#x2F;webpage.htm instead of using localhost.

    Disable your local firewall and antivirus temporarily.

    Once you have it working on a simple level, add back in the complexity.

    Hope this helps

    • Correction to my previous post about localhost and 127.0.0.1. Your emulator and device are tcp/ip network hosts, and hence 127.0.0.1 simply refers to them.

      To connect to the machine that the emulator is running on, you need to use the machines IP address, that it uses to communicate with any other host on your local network.

      — commented February 14th 2011 by Paul Dowsett
