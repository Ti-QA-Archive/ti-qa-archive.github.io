Best way to handle a join model with ACS

With ACS, it appears you can't query a custom field that is an array. With that said, if you had a join model, what's the best way to handle the structure on the ACS side?

Simple example. You have a custom object called "Box" and another called "Apple". Boxes have apples in them, and can have more than one.

For the purpose of this discussion, let's also pretend that Apples can be in multiple Boxes (despite what physics tells us), so setting a box_id on Apple isn't a prudent choice. It's a true many-to-many relationship.

What would be ideal is to create a custom field on Box with an array of Apple ids, since Box owns Apples. Then, query on Box to get the array of related objects.

My understanding is that this isn't possible with ACS right now. Instead, it seems like the only real way to do it is to use a join model, wherein I have another object called BoxApples, where I store the relationship between them as discrete records. That feels like hacking SQL on top of a NoSQL store, which, well, feels a little dirty.

Is there a better way? Am I missing something here?

  • I've been looking at this over the last week and came to the same conclusion. The only way was to use a CustomObject to define the relationship, possibly using Relational Fields which would return the related objects in the same API call.

    I posted this Q&A thread related to my specific use case. I wanted to enable the owner of a PhotoCollection to set access to it on on a per-user basis. I can set permissions to a PhotoCollection to protect it on a per-user/global basis using an ACL. But I couldn't/can't find a way for enabling a user to query which PhotoCollections it has access to based on these defined ACLs alone. So I need to define the relationship between PhotoCollection(s) and User(s) myself.

    But PhotoCollection doesn't provide a query() method - only search() - and even if it did, and I set the custom_fields to something like: custom_fields = {"user_ids_read": [id1,id2,id3]} you can't query() on an array value for custom_fields.
