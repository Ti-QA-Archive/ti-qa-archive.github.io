    • Here's how to downgrade (At least How I did it..)

      1) Download 11.1.5 from here http://mac.filehorse.com/download-itunes/4050/

      2) Go to Applications, and Delete itunes… If you get an error message saying something like "This app is require by OSX" just go to Get Info > Sharing & Permissions > Everyone … it has to say Read & Write.

      3) Install the itunes you just downloaded

      4) Open it (You might need to hold you "alt" key while opening so you can create a new library (11.2 changed something in your library that is not compatible with 11.1.5)

      And, that's all.

  • Please check if Version (CFBundleShortVersionString) and Build (CFBundleVersion) parameters in your .plist file are filled in and your Build number is a single number. Also read on how numbering should be handled.

    In our product we had similar problem. Build and Version were both set to full build number. When I changed Build to single iterable number installation worked.

    • Further to Pawel's answer I compared the build script from Ti 3.2.3 GA to that of Ti 3.3.0 RC where the problem has been fixed according to JIRA.

      I narrowed it down to a single line in /Users/<user>/Library/Application Support/Titanium/mobilesdk/osx/3.2.3.GA/iphone/cli/_build.js

      Opening the script in a text editor, I modified line 1805 from:

      plist.CFBundleVersion = appc.version.format(this.tiapp.version, 3, 3) + &#39;.&#39; + (new Date).getTime();

      to:

      plist.CFBundleVersion = String(+new Date);

      After overwriting the existing file I then cleaned the project in Ti Studio and did a Run > iTunes Sync and iTunes was happy, no errors.

      Hope this helps.

  • Same problem here. My workaround doesn't involve downgrading iTunes (it's a Titanium problem, not an iTunes problem - so why waste your time rebuilding your library). First, ignore the message about it being a bad app. Open Xcode, and then Organizer (if it's not already open). Select your device, and then it's applications. You'll see your app listed there. Add it again using the Add button at the bottom (no need to delete). It will open a Finder window and then navigate to your build (something like ../yourapp/build/iphone/build/Debug-iphoneos/yourapp.ipa … Select it and it will re-add the app to your device (you'll know it's done when the version number of the app changes).
