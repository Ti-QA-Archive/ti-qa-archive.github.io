  • Hey Huang,

    I've solved this as follows:

    Android:
    Create in platform/android/res/ following folders:

    • values = default language
    • values-zh = chinese language

    iOS:
    Create in platform/iphone/ the i18n folder:

    • First folder in i18n is default language
    • zh-hans = simplified chinese language

    You have to maintain your localizations twice, but you dont need to rename anymore.

    Hope this will help.

    • Hi Huang and Christoph,

      A simple solution which doesn't need to maintain two locale folders for the same language is modifying Titanium Mobile SDK build script, localecompiler.py, directly as below:

      def compile_for_ios(self,file):
                locale = self.get_locale(file)
>                if locale == 'zh-TW': locale = 'zh-Hant'
>                if locale == 'zh-CN': locale = 'zh-Hans'
                build_dir = self.get_ios_dir()

      So that you only need to maintain zh-TW for Traditional Chinese and zh-CN for Simplified Chinese in your project.

    • Unfortunately this is a long standing known issue, that I too am suffering from, see this JIRA ticket: https://jira.appcelerator.org/browse/TIMOB-3639

      Can you put a comment and a 'watch' on it, so that they prioritise the issue for the next release.. Thanks!

      To fix, you are going to have to swap your folders out when you build.. An interim solution if you are startin and app is to use:

      L('myKey, 'Default Value') for each localisation. And rename the l18n folder to something else so it isn't included in the builds.. The second argument will display the default for your general development. Then when you build a release you copy the correct folders into L18N and do the build for one platform then swap them out for the other..

      Annoying that it is like this though!
