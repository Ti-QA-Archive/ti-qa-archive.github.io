Label's text doesn't change while it's in a tableViewRow

Hello all,

I've a serious problem and I don't understand what's going on :-(

I have a tableViewRow that contain's a label, and I have a text field it the window it self but not in the tableView, what I want to do is changing the label's text to match the textField's value on change event to the textField, it works normally on iOS but never worked on android, I have to remove the label from the tableViewRow in order to work on android, one of the tests I made was putting the label in the window outside the tableView and it works, but I need it inside the tableViewRow.

I tried a lot of other things rather than the textField, but with zero results, the label's text can not be changed on android in any eventListener if it's inside a tableViewRow.

Here's a sample of my code that explains what I want to achieve

.xml

<Alloy>
    <Window id="win" title="Complete Your Profile">
        <TableView>
            <TableViewRow height="140">
                <View layout="vertical">
                    <!-- Some other objects -->
                    <Label id="lbl" text="Test"></Label>
                </View>
            </TableViewRow>
        </TableView>

        <TextField onChange="updateLabelText"></TextField>
    </Window>
</Alloy>

.js

function updateLabelText(e) {
    $.lbl.text = e.source.value;
}

This is simply what I want to achieve, this code works fine on iOS, but never on android.

Note that: I could only change the label's text outside the eventListener but I don't know why.

    We have the same issue and I had to analyse some code at:
    https://github.com/appcelerator/titanium_mobile/blob/master/android/modules/ui/src/java/ti/modules/titanium/ui/TableViewRowProxy.java

    You can found that this code 

    TiUITableView table = getTable().getTableView();
table.setModelDirty();
table.updateView();

    runs only in case you add/remove view from/to row or change one of TableViewRow property.

    so the easiest workaround is 

    <Alloy>
    <Window class="container" layout="vertical">
        <TableView id="tab" height="200">
            <TableViewRow id="row" height="140">
                <View layout="vertical">
                    <Label id="lbl" text="text">Text</Label>
                </View>
            </TableViewRow>
        </TableView>
        <TextField id="txt" hintText="Enter Text" onChange="txtChange"></TextField>
    </Window>
</Alloy>

    and

    function txtChange(e)
{
    $.lbl.text=$.txt.value;
    $.row.title = $.row.title == " " ? "   " : " "; 
    //two spaces or one spaces - NEW value MUST BE not equal OLD value
    // or even
    //$.row.title = new Date();
}
$.index.open();

    also you can try to do $.tab.updateRow(0, $.row);

  • Looking at the actual code you provided in your comments, and assuming your event listener is on the table, the issue would be that e.source is not the label on Android like it is on iOS. BTW, the event listener should be on your table, not the individual rows or labels within. Adding lots of event listeners to the rows/labels would be memory-inefficient.

    $.table.addEventListener('click', function(e) {
  // e.row is the row tapped
  // its first child should be your view
  // the view's first child should be the label
  e.row.children[0].children[0].text = 'foo';
});
