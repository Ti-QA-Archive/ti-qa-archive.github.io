Parts of a button control disappear when a user scrolls the page...

I'm having an interesting problem…I have a Scrollable View with a widget inside one of the views and parts of a button control disappear when a user scrolls on this page…Here's the XML for the scrollview: 

<ScrollableView id="scrollableView" showPagingControl="true">
            <View borderColor="black">
                <Require type="widget" src="myWidget" id="myWidget" />
            </View>
</ScrollableView>

In the widget I just have tableview with buttons in the tableview rows like so:

<TableView layout="vertical">
        <TableViewSection>
            <HeaderView>
                <View class="blue_backgroud" id="headerView1">
                    <Label id="section1" />
                </View>
            </HeaderView>
            <TableViewRow class="row" id="row1">
                <Button class="checkbox"/>
            </TableViewRow
        </TableViewSection>
</TableView>

In the tss a checkbox class is defined as follows: 

".checkbox": {
title: '',
width: 40,
height: 40,
borderColor: '#666',
borderWidth: 2,
borderRadius: 3,
backgroundColor: '#aaa',
backgroundImage: 'none',
color: "white",
font:{fontSize: 25, fontWeight: 'bold'},
value: false, //value is a custom property in this casehere.
},

Here's the screen shot of what the user sees (if you notice parts of the last control are missing):

screen shot

This only occurs for rows that are in contact with the heading after a user scrolls and only for a few rows. Anybody have any idea why this is happening?
