Is this document miss?

http://developer.appcelerator.com/apidoc/mobile/latest/Titanium.UI.Window-object

>softInputMode
>int
>One of Ti.UI.Android.SOFT_INPUT_ADJUST_PAN, Ti.UI.Android.SOFT_INPUT_ADJUST_RESIZE, or Ti.UI.Android.SOFT_INPUT_ADJUST_UNSPECIFIED. Note: MUST be passed in the creation options. (Android Only) Android Doc: Window.setSoftInputMode

I specified softInputMode at Ti.UI.createWindow(), but it doesn't work.

But I found an example code at KitchenSink:
<https://github.com/appcelerator/titanium_mobile/blob/master/demos/KitchenSink/Resources/examples/android_window_soft_input_mode.js>

In this code, windowSoftInputMode is written instead of softInputMode.

opts.windowSoftInputMode = state;

I fixed my code from softInputMode to windowSoftInputMode, and my code worked well.

I think this is document miss.
