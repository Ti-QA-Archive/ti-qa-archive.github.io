Store files in applicationDataDirectory instead of resources/images

Hi, I'm using some icons in my application and many of them will go with the app installer. I have them into resources/images folder and when you install the app they are immediately available for use. But in my app there might be new icons needed after a while and I want to store them locally when they come.

Stored data is usually located at applicationDataDirectory and that's where I would store the new icons. But the problem is that I'd like to use the same path for both the ones that will come later on and the ones that I already have.

So, how can I instead of using the resources/images folder use the one that applicationDataDirectory points to? The idea is that when installing the app they would reside there. I can only thing of copying the files manually after first installation. Any hints?

Thanks.

    If you don't like the idea of copying them over from the install, then you could download them on first run.

    I personally think copying them over is a better idea, so the user has a good "first run experience"
