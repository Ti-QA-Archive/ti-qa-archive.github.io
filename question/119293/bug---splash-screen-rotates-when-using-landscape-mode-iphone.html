Bug - Splash screen rotates when using Landscape mode (iPhone)

This is a known bug and there are several threads about it, none receiving any attention by developers.

Since we got no way of creating issues in the new Jira system, I'm trying here to either report the bug or ask if anyone has found a workaround.

The problem is when you got an landscape only app, the splash screen first loads default.png correctly, but just as the first window is opened, the splash image will rotate 90 degrees in a distorted way before the window appears correctly in landscape. The shortly rotated image is clearly scaled as to fit a portrait window - it's partly outside of the screen and theres a black empty area..

There is no difference if I remove all orientations from tiapp.xml or info.plist, or set the orientation through code..

I think this is a critical bug since there is no way to create a landscape only app with Titanium as it is now.

Anyone that has any ideas at all?
