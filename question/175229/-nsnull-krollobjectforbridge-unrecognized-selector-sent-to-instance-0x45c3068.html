    • Matej, would you please file a ticket? Include a minimal app the demonstrates the issue. Please include a link to this discussion thread. We'll be glad to look into this.

      In general, we have always recommended against putting a scrolling component inside a scrolling component – doing so creates potential usability issues and in some cases causes eventing issues (which component should get the scroll event?) or even race conditions that cause an app crash. I'm not sure if the latter two are still valid concerns. But the UX issue is.
