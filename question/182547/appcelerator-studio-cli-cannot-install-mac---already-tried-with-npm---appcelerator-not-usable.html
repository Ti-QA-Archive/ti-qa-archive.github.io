Appcelerator Studio CLI cannot install (Mac) - already tried with npm - Appcelerator not usable

When launching Appcelerator Studio (on Mac OS 10.5 Yosemite) after a fresh install:

A window pops up (new updates) asking to install "Appcelerator CLI Core" (version N/A).

If I click "Install", an error appears:
(details is: "Appcelerator CLI is currently upgrading or installing. Please wait until it has finished..").

If I click "Remind me later" or if I continue:
Appcelerator cannot be used as I cannot sign in after that
(it says: "Appcelerator CLI is currently installing or upgrading").

Appcelerator Studio is just broken now.
Reinstalling does not fix it.

I have already tried the npm workarounds (to install alloy and titanium).
These say that titanium CLI 4 is installed.

I have already updated NodeJS just in case.
I have already set the chown to usr/local/bin, just in case.

None of these worked.

Problem is different than this:
https://developer.appcelerator.com/question/145844/cant-install-titanium-cli-and-alloy
.. because mx problem is with Appcelerator Studio, the new 21st of May version, which doesn't work here.

Problem is different than this:
https://developer.appcelerator.com/question/145775/mac-os-x-cant-install-titanium-cli–alloy—invalid-password
… because it is not a password problem.

Please help, it's really important that I get it working soon.

  • Thank you people for your answers!
    I have to fix a minor typo: I was on Mac 10.10 Yosemite (not 10.5).

    Now let's get to the solution.

    From the URL there:

    https:&#x2F;&#x2F;web.appcelerator.com&#x2F;product&#x2F;cli

    … I did already step 1 and 2, which are mentioned in other related topics.

    Before that I unfortunately didn't read about :

    appc setup

    … so I instead launched appcelerator to end installation, as they mention there too as step 3.
    This is what I've done before I had these answers.

    Note that Appcelerator CLI confirmed it was installed with a version 4 already on Terminal commands.
    It only never confirmed so in Appcelerator application, and the Appcelerator application was bugged.

    What worked for me, before I received these answers and after days trying things, was to:

    ` - erase the entire hard disk (!)

    • reinstall Mac OS
    • and then install Appcelerator on this entirely new disk. `

    It worked.
    It was a real pain to go to that just because of installation bugs, but at least it worked.

    So, for other people googling about this issue, as I did, you may either use Roberto / Juan / Chris solution, or install on a clean disk.
