Problem working through iOS Module Development Guide

Hi everyone,

I'm working my way through the iOS Module Development Guide. Everything's working fine until I get to the "Testing Your Module" section. I'm in the tmp/test directory in terminal. com.test-iphone-0.1.zip is in there as well, but when I type "titanium run", I get "-bash: Library/Application Support/Titanium/mobilesdk/osx/2.1.4.GA/titanium.py: No such file or directory".

After trying to deal with that for a few hours, I decided to start from the beginning. So I get to part 6 of Step 0, type in "titanium", and get the same message as above.

Does anyone know what's going on here or have the same problem?

  • Hi Marc, there are few things to tell you… if you have installed Titanium CLI then titanium is by default for CLI uses and will create problem.

    do few things…

    1: run vi ~/.bash_profile

    2:: edit your .bash_profile and make titanium alias to something like

    titanium.py ="Library/Application Support/Titanium/mobilesdk/osx/2.1.4.GA/titanium.py"
    // path for system library

    or

    titanium.py="~/Library/Application Support/Titanium/mobilesdk/osx/2.1.4.GA/titanium.py"
    // path for user library

    //Here titanium.py path actually depends on whether its in Used Library or System Library.

    i would say to check this and most of the time it is in User Library, which is by default.

    3: run source ~/.bash_profile

    4: run titanium from console and check what it display now, and whether run as an option is given or not with titanium.
