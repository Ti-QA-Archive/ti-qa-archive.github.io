App with required login, when to call device registration for push?

I have Urban Airship working with my app for push notifications. I added the register code to app.js, but i have a few questions:

1) It seems to me I should only need to register a device after a successful login (so i can attach the userid Alias). My question is: can i call the register function outside of app.js?

2) …And if yes: If you include files in app.js (i.e "urbanairship.js") are they accessible throughout the application or do you have to include them again in each JS file? Or should i just avoid any push code in app.js altogether?

  • 1 . Yes

    2 . Depends on how you are making the code. If the code is "one big file" type, like tweetanium, yes the urbanairship will be accessible along the whole app.
    If you use separate contexts ( windows with url ) you will have to find a way to access the methods and vars you need.
    One way is to include the same file over and over again in each file.
    Another way is to use cross-app events. Like you declare in app.js on event and you trigger it from any other point within the app.

    The last version is more complicated but is better in my opinion.
