Default iOS 8 behaviour is to hide the Status Bar in landscape mode - but does not work in Ti?

In iOS 8 SDK the native/default behaviour for iPhone apps is that the status bar will disappear when the device is rotated to landscape, and show again in portrait. Try the Mail app or Notes app on an iPhone with iOS 8 and you will see what I mean.

I want this default behaviour. It is perfect for the status bar to disappear to optimise the usable landscape screen real estate.

However when building for iOS 8 using Ti SDK 3.4.0 I cannot seem to get this behaviour. I can turn on the status bar app wide, or disable it app wide, but I cannot get this default behaviour where it will show in porter and hide in landscape.

If I create a clean project in Xcode and build for the iPhone 6 Plus simulator I can see this behaviour in action.

Any ideas on how I can get this working for my Ti project? A setting I can tweak maybe?

I will investigate further but thought I would check if anybody had noticed / solved this issue already…
