Remote Database Synchronisation

I am starting from zero and I have a requirement to build both a cross-platform mobile app and a web app that sycnchronizes data both ways.For example the mobile app will have a contacts form that once completed can be pushed to the parent web app. Subsequently changes made to the parent web app must be pushed to the child mobile app.

The web app will be asp.net, IIS and SQLServer 2005 DB.

There is a possibility of users updating the same record at the same time so record locks must be avoided, also managing data when the mobile device is off line, storing the updates until connection is available and then push/pulling the data. This is a very high level request, I have seen mention of soap,json and webservice options but which is the best way forward to research? Thanks

  • To manage the issue of locking and updating the same record, you should use the ability of ADO.Net to version the dataset rows and only update if the version has not been raised since the record was originally read. That way "the last person wins" in terms of being able to update the record
