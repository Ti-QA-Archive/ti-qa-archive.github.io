Bug: Button file path

I believe I found a bug. It has to do with the image path for images of the button.

var deleteButton = Titanium.UI.createButton({
        color:'#fff',
        backgroundImage:'../Resources/images/BUTT_red_off.png',
        backgroundSelectedImage:'../Resources/images/BUTT_red_on.png',
        //backgroundDisabledImage: '../images/BUTT_red_off.png',
        //backgroundColor: '#CC0000',
        top:303,
        width:300,
        height:60,
        font:{fontSize:20,fontWeight:'bold',fontFamily:'Helvetica Neue'},
        title:'Delete',
        //style:1
    });

The above code displays the button in the simulator but not on the device. I had to add the "Resources" in order to make it work in the simulator.

I haven't tried but I believe that if I remove the "Resources" it will display on the device but not in the simulator.
