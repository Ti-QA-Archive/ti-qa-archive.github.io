Detecting if remote assets is an audio file

Hi folks,
I try to build a twitter app. The app shows all tweets and if in a tweet a audio file is embeded, then a player/downloader should appear.

This my code snippet:

exports.getContenttype = function(_url, _callback) {
    var xhr = Ti.Network.createHTTPClient({
        autoRedirect : true,
        onload : function() {
                var type = this.getResponseHeader('Content-Type');
                if (type)
                    _callback(type.split('/')[0]);
            }
    });
    xhr.open('HEAD', _url);
    xhr.send();
};

It works and give me 'text' or 'audio'. But: the code need a very long time for answer. I know, there is a redirect …

Any other ideas how I can decide between webview for normale link and audioplayer for mp3?

Rainer

    f so, there is a limited number of valid extensions for mp3 files. Why not just test the filename extension instead ?

    Trying to go off elsewhere and retrieve the Mime Type is going to place a lot of battery strain on your App and you cant control the latency to get the info.

    Alternatively, do all this work on a server you control and relay them to your App
