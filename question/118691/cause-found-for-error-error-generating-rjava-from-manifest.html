cause found for "[ERROR] Error generating R.java from manifest"

Hi,

I found a cause for the error "[ERROR] Error generating R.java from manifest":

When/if you fill out just a single word/name (like "myapp") in "Application Id" in the Titatium Studio wizard (or in the tiapp.xml), compiling will result in "[ERROR] Error generating R.java from manifest".

The error goes away if you fill out a FQDN-like name like "com.myrealdomain.myapp" in the wizard.
NB: Changing it in the tiapp.xml afterwards still results in some (other) error on my system, but results in a running app on the emulator.

If others can confirm this, I would say it's a bug in the syntax checking of the Titanium Studio wizard; it should require the FQDN, and thus it should refuse just a name without dots in it. I'm not sure if you need a real existing domain.

  • I have experienced this problem using Studio 2.1.2 when compiling an Android build.
    The problem was resolved when I changed all the "Name" values in my strings.xml files to make sure that no hyphen (-) characters existed in them (e.g. name="Geo-Code" was renamed to name="GeoCode"). Obviously I had to change the string names throughout my code to reflect these changes. Xcode 4.5 does not complain when hyphens are used.

    Comments are allowed in the string.xml file so long as they follow the normal xml comment syntax <!– comment –>.

    It has been reported (but I have not personally experienced this problem) that the applicationId in TiApp.xml also needs to be in the format "com.yourcomany.yourproject"
