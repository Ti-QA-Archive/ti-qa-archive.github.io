    Hi

    If you want to play an MP4 in a webview inside your app (I am sure you have a good reason not to use the native player) then the resources directory is the root of the device, relative to the html page.

    This is an example which works fine in webView using the html5 video tag.

    <!DOCTYPE HTML>
<html lang="en-US">
<head>
    <meta charset="UTF-8">
    <title>Video</title>
</head>
<body>
    <img src="KS_nav_views.png" alt="KS_nav_views.png"/>
    <video src="example1.mp4"></video>
</body>
</html>

    • OK so this is a new problem.

      You cannot download files to the resources folder - it is read-only.

      You have to download them to the applicationDataDirectory(). You should then be able to reference files by using file:// prefix.

      You will need to parse your HTML file to pass in the full file path and video file name.

      Use f.nativePath() to access that.
