"With Titanium you have to customize your app to the different platforms" - TRUE?

Hello,

since I'm a newbie to this app development thing I've checked before the Internet about the different app developer opinions. One opinion which pops up very often is "With Titanium you have to customize your app to the different platforms". As reason for this they mention that Titanium creates NATIVE code and not web views. And the APIs on the platforms implement if differently. I hope I remember those reasons correctly. Otherwise please correct me. So when this statement is true what can I, not very comfortable with Javascript, do that I don't have to change my code for every platform? Is there a solution to make this step more comfortable for me?

Thanks a lot for answer.

    Hi

    The problem here is the phrase "cross-platform".

    • Appcelerator is a native cross-platform solution that uses JavaScript to achieve this
    • Many others are cross platform but only because the apps work on different platforms but mostly do not honour native UI. Instead (as you note) are based on web pages rather than actual native elements.

    When you create an Appcelerator Titanium app you are creating a true native cross platform solution.

    Most of the code you are going to create will be the same, irrespective of the platform or platforms you are targeting. This is because the actual logic of the application will be the same. So when you create a table of rows that is the same for each platform, create a window or a button that code is the same.

    I am going to restrict this conversation to just iOS and Android to make examples simple.

    • iOS has a navigation controller, Android does not
    • Android has an action bar, iOS does not
    • iOS has button bars and toolbars, Android does not
    • Android supports intents, iOS does not

    However they both support;

    • Windows
    • Views
    • Buttons
    • Images
    • ScrollViews
    • ScrollableViews
    • and many more

    The important thing to take into account is that the users of each platform expect it to work the way that platform is designed to work. So a good developer honours that and tweaks the application to suit those differences.

    Many of the tweaks are nothing more than a few lines of code to make each work with respect to each platform. Some however take more effort, that effort will pay off in the long run as your app will not feel rushed or cheap.

    If you are finding your way as a new user, then the best thing to do is not to worry about the differences in UI and some in the core platform interfaces - but instead focus on getting things to work. Once you have got the basics sorted then you can revise your code if required to make the most of each platform.

    If you are not as comfortable with JavaScript as you think you should be then take some time to get the basics down as knowing the more important things such as this will make your life much easier and faster;

    • Variables
    • Functions
    • Loops
    • Strings
    • Arrays

    There are many site on the web to help with this.

    Welcome to the platform and I hope you enjoy it as much as the rest of us.
