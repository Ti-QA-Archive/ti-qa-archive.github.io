Crossplatform include for multiple views

Hi,

I'm using Titanium Studio 2.0.2 with Titanium mobile SDK 2.1.0 on linux and I use the latest version of Titanium on OSX.
I have a mobile app, which has to include a js file and use the variables/functions in 2 views.
iOS platform (both in emulator and the device) I were able to include my JavaScript in ui/common/handheld/ios/ApplicationWindow.js:

Ti.include(Ti.Filesystem.resourcesDirectory+'JayData.js');

This include in the ApplicationWindow.js makes me able to call functions from this JS in both each views.

I couldn't achieve this on Android platform, so I had to include the JS in all the views.

Is there any solution for for this to avoid hacking and keeping the number of platform checker statements at a minimal level?

Thanks,
Robert

  • You should stop using include.
    Now Ti supports very well require.

    Besides this instead of trying to detect the resources directory, using an absolute path will solve the issue.

    Ti.include(/'path/to/file/JayData.js');

    The leading / is the Resources folder and you don't have to take care of anything.
