Uncaught TypeError: undefined is not a function at /index.html (line 90)

Working on Windows 7
Titanium SDK 3.5.1 GA

Was revisiting an HTML based app I was working on. Ran it in mobile web preview in browser. First thing to come up was the error Uncaught TypeError: undefined is not a function at /index.html (line 90). Line 90 in index.html is CSS, no functions are being executed. I decided to create the template HTML based app and run that, sure enough I had the same issue. Thought it was an issue with my install so I re-installed titanium, still the same issue. Installed titanium on another machine and ran a fresh template HTML based app on that… still same issue. I'm guessing its an issue with setting up titanium however I don't remember running into this issue when I started working on this application earlier last year. There might be a new step involved however I am not seeing anything I am missing in the startup guides. Any help on how to get this running again will be much appreciated.

  • I'm having the same issue.

    Running the webview in Firefox shows a more detailed information about the javascript error.
    (I'm trying to run the basic HTML View example app)

    Turns out that the file ui/ApplicationWindow.js is accesible via 127.0.0.1:8020 but mobileweb/ui/ApplicationWindowPlatform.js not. (Returns 404 on direct reference using http://127.0.0.1:8020/mobileweb/ui/ApplicationWindowPlatform.js

    So, as ui/ApplicationWindow.js loads the later one, this is the cause of the problem. (ApplicationWindowPlatform() function defined in his .js is the 'undefined' thing the error referres to.

    The solution? I don't know. Maybe a misconfiguration of the server not setting the correct permissions to this mobileweb path, but how can we change it?
