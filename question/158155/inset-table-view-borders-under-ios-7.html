Inset table view borders under iOS 7

Having a tough time upgrading an app to mimic what it looks like under iOS7 when it comes to table view rows.

I've set the left image using setLeftImage, and am using the build in title property.

My first instinct was to create an image view and insert it, as well as manually inserting a label. While it worked, it causes noticeable flickering when the user taps on a row.

If I use the built in title and insert an image, the title overlaps the image. (perhaps theres a way to set the left of a label's title view?)

Is there a way I can get a full, ios6-like border under my rows, or am I better off removing the row separators completely?

  • I wondered about this too, and it appears the inset is part or the new iOS7 look (you can see it in the Mail app). Check out iOS 7 UI Transition Guide under "TableView" and the description table below "By default, the separator is inset from the left edge of the table view."

    Not sure if there is a way to change this with Ti, or a good workaround. Maybe a hack to try would be to put items in a view inside the row, and then set left:&#39;-15dp on the TableView?

  • As noted by Edward, the inset is standard behavior for iOS 7. There is a property to override this that the SDK doesn't currently expose. I believe that's on the feature list for a future version.

    See https://jira.appcelerator.org/browse/TIMOB-14986 for some additional information.
