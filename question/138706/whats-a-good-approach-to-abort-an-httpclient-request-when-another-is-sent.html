What's a good approach to abort an HTTPClient request when another is sent?

I'm using Maps 'regionChanged' event to load location information from a server. Every time the map is scrolled or zoomed, the event triggers a HTTPClient request to the server. So, while moving the map around, multiple requests are being sent.

I'd like to know of any best practices or slick ways others have found to abort the first request before sending the next.

thanks,
M

  • Obviously you need to utilise the abort() method to achieve this (http://docs.appcelerator.com/titanium/2.0/index.html#!/api/Titanium.Network.HTTPClient-method-abort). I would start by ensuring that your http requests were queued in an array of some sort and were chained so that the next one was only executed after a response had been received from the previous one in the queue. Then you could remove all pending requests when you next received a "regionChanged" event. Not tried this - just a thought..
