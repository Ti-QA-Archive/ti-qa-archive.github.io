  • Yes best practice says to null the object, after its use is over. When you assign any object to null it is eligible for GC, but if you don't assign it null then GC may or may not free its memory, or it may free the object after some time.

    To check what I said above you can use memory tool like instrumentation. There you will find the difference when you assigned it to null and when you don't

    • Hi Moiz,

      Thanks for the reply, but if you see in

      "http://stackoverflow.com/questions/8467350/how-to-free-up-the-memory-in-javascript"

      you can refer the variable to some other object. to make older object elegible for garbage collection.

      For example, if you had a 10,000 element array held in a global variable:

      var largeDataArray = new Array(10000);
      And, you had filled most elements with data, then you could allow that memory to be eligible for garbage collection by assigning it some other value like:

      largeDataArray = null;
      or if you still want it to be an array:

      largeDataArray = [];
