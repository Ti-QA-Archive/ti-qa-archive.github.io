  • Hi

    Have you actually tried it?

    The underscore library does an amazing job and proving many extensions to common JavaScript objects. If you only use this library for the each command then it is worth its weight in gold.

    But back to the main points;

    • Lag; no, the file is so small, much smaller than your own code - this is nothing in the grand scheme of things
    • Assuming you are storing the the library locally to the device - if you are not - then you MUST
    • Performance of functions; this is harder to answers as there are so many positions;
      • If you are new the functions help you solve problems that might take much longer to learn
      • Each function is obviously tuned to provide the most efficient performance possible
      • HOWEVER the methods are generic solutions to general problems, so the speed is relative to that premise
      • There is NO substitute for understanding the core JavaScript objects and how to interact with them, not learning them is lazy
      • Lazy is however relative, using a sat-nav to get from a to b makes much more sense than using paper maps
      • However using the paper maps shows you about the journey much more than the sat-nav could and that journey may bring additional knowledge and understanding
    • Back to should you use it
      • Yes, I do
      • No, I don't
      • OK - that sounds stupid; know when to use it, as I said earlier it creates performance capable methods to extend base JS, however your specific requirement may have a more suitable non underscore way - keep in mind underscore is just a library of JS commands.
      • If you are performance tuning your app then consider both using and not using, but have the knowledge to not need it.
      • But have the smarts to know when to use it as well. For example - you cannot reasonably performance tune a loop of 20 array items that builds another array used only once in your app. Yes you CAN make it faster but I said reasonably - performance tune something that happens 1,000 times before you get back to this one.

    So yes I use it, many smart people do, many smart people don't, if you use it - remember you should have an understanding of what it is doing and do not rely on it rather than learning these techniques without a library.

    So the best example I can give is this; I use and oven to cook my food, I use a washing machine to clean my clothes, I use motorised transport to get me from a to b when the journey is long. If we lost electricity and fuel; I better know how to start a fire, kill some animals, find fresh water and get from a to b. But the car is still better until that happens - I will just be prepared if that changes.
