Is it possible to play a streaming mp3 from a webserver with the MusicPlayer.

Hello,

I saw an example at KitchenSink which loads mp3 from the directory,

sound_file_url.js
var file = Titanium.Filesystem.getFile(Titanium.Filesystem.resourcesDirectory,'cricket.wav');

// load from file object but use the nativepath
var sound = Titanium.Media.createSound({url:file.nativePath});

I would like to know if the mp3 can be loaded from webserver. e.g from http://www.somewebsite.com/mp3/filename.mp3

Thank you in advance

  • As far as I am aware, Ti.Media.Sound is used to play local sound files (.wav, .mp3, etc.) while Ti.Media.AudioPlayer is used to play remote sound files (.wav, .mp3, etc.)

    I've tried loading remote sound files with Ti.Media.Sound but they would not load, and I have tried loading local sound files with Ti.Media.AudioPlayer and they do not load either. So I'm guessing they are just different media players - Both play sound but one is for local and one is for remote.

  • Yes, you can load remote audio using Sound, but you won't be able to access it until the entire sound file has loaded into memory. This is why its probably advisable to use AudioPlayer which streams the sound for remote audio.
