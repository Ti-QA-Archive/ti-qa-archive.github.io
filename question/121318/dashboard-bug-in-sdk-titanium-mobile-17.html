    • I have found a kind of custom & dirty fix….but this has to be corrected I think.

      Once the dashboardView is created, simply fire the "commit" event on the view using dashboardView.stopEditing(); and the vertical scroll problem will be solved temporary…

      I discovered that when you enter the edit mode (by clicking 1-2 seconds on an item) and leave the edit mode after having deleted items (or not), the vertical scroll "disappeared"
