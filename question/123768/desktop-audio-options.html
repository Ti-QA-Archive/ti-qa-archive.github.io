    Also, I just realised you're adding the src attribute to the audio tag.
    You might try adding a child tag for 'source' within the audio tga and setting the src and type attributes to that.

    Then you should be able to use audioElement.play() , audioElement.pause() and audioElement.currentTime to do what you need to control it, as well as adding the 'playing', 'pause', 'durationchange', 'seeked' and 'ended', etc events to it.

  • Hey Kevin pinged me about this thread and to offer my advice, I work on our Desktop
    platform. I figured out a workaround to get the audio tag to work.
    It appears it is unable to locate media files inside the Resources folder.
    For example: &lt;audio src=&quot;test.mp3&quot;&gt; should resolve to test.mp3
    in Resources/ for your application. My workaround was to use the
    File object in our API to get a file:// URL to the media file. I then
    set the src attribute for the audio tag.

    I created an example here. I also opened a ticket for this bug here. Let me know if you find a better way or have questions!
