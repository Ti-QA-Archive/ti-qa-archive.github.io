    • Hi, I created a project in github, ti-db-rdump where I'm developing a possible solution, beginning with the client side.

      The proposed scenario:

      • A simple test db, with one table people(firstname, lastname).

      • Dump db data in a json object as 

        {
db: "dbtest",
tables: {
  people: [
    {firstname: "Walter", lastname: "White"},
    {firstname: "Jesse", lastname: "Pinkman"}     
  ]
}
}

      • Post the people dump with a Ti.Network.HTTPClient request

      The server app will take care of parsing the json dump and persist it.
      But this part is not included here.

      [!] Work in progress, pull frequently if interested ;)
