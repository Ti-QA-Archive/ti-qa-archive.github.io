Unlicensed module error after upgrading

I am using the Barcode Scanner module by Scandit (https://marketplace.appcelerator.com/apps/3907?916047513), which is free. After being notified by Appcelerator that a new version is available, I downloaded it, put it into my project directory and changed the version in tiapp.xml to match the new version.

Immediately on starting the app I get an 'Unlicensed Modules Detected' error.

This is a free module that I was prompted to upgrade, and after the upgrade it no longer works. I followed some advice in these threads:

http://developer.appcelerator.com/question/125625/unlicensed-module
http://developer.appcelerator.com/question/126870/unlicensed-modules-detected-on-plus-modules

And got it to work again, but that solution involved removing the app from my products and adding it again, which subsequently broke the live version on iTunes!

Is this a problem with the module? Or am I doing something wrong in the upgrade process? This seems like either a serious gap in the whole upgrade procedure or a poorly documented option that is impossible to find.

What is the best process for upgrading a module to its newest version?

Thank you!

    • Mohammad's answer doesn't work for me either. Scandit have rolled back their changes, so if you didn't change anything in terms of guid, just stick the new (or old) module into your folder and it will work again.

      Scandit are awaiting advice from Appcelerator I believe.

      I personally did these steps to get the version working (the new one before the rollback):

      • Downloaded the new module and placed them in the android / iphone directories.
      • Removed Scandit Barcode Scanner from 'My Products' on the Appcelerator website.
      • Added Scandit Barcode Scanner back to 'My Products' and assigned team members to it.
      • Created an empty project in Titanium Studio.
      • Copied the guid from the new empty projects' tiapp.xml file and into my project's tiapp.xml file.

      This allowed the new version to work. There must be a better solution for all involved, because this kind of 'upgrade' is a right pain.
