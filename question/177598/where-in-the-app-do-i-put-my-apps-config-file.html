Where in the app do I put my apps config file?

I have an app that does some on the fly configuration based on what country the user is in. The data is simple JSON file but I am having some issues working out where in the app's folder structure I should place the file to be able to access it.

The app itself uses alloy, so the resources file is out from my understanding, and the app errors out when I try to use the DataDirectory (inside the current app folder structure, I've placed the file in the assets folder for now) I've also tried using a lib/ folder and the root folder in the project with similar failures. The docs are not clear where a file like this should be placed.

    var file = Titanium.Filesystem.getFile(Ti.Filesystem.resourcesDirectory, "/assets/bullybuddies.json");

    var contents = file.read();

        Ti.API.info('Output text of the file: '+ contents); alert(contents); //debug

    var configJSON = JSON.parse(contents);

Any clues would be appreciated.

  • Bastien,

    Just to clarify, Alloy does NOT change the accessibility of any of the directories. The best description from the Alloy Documentation states The files in the assets and lib folders, as well as the files in the theme's assets folder, are copied to the Resources folder.. So, your files are copied to the Resources directory on compile. The resources directory is available using the property you have in your code Ti.Filesystem.resourcesDirectory. So, you can really place it anywhere you want, as long as you know you are starting with the Resources Directory as your root.

    Does that make sense?

    Ray
