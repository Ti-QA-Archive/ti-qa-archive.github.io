Adding more than one event listener to a view crashes the app?

I've been tracking down some weird crashes in my app. In some cases it crashes the iPhone simulator on startup and I've traced them to some calls of the type view.addEventListener(&#39;signal&#39;,function(e) { });. The problem is, the crashes don't depend on the functions themselves, just on the existance or not of the event listeners. In this particular case, having just an event listener for 'singletap' works, but not with two (for 'singletap' and 'doubletap').

Are there some known bugs or problems with addEventListener, or some recommended practices? This isn't a build error because I've already deleted the build directory a few times.

— asked July 26th 2011 by Joao Silva

  • I added the following code to the default Titanium mobile app (when you create a new mobile project). It's working just fine (with windows).

    win1.addEventListener('singletap', function() {
    alert('1 click');
});

win1.addEventListener('doubletap', function() {
    alert('2 clicks');
});

  • Some things to note about events

    1. You can only fire events to window above below (But still finicky use one window structure with views instead of windows)
    2. If you fire an event and then fire another before the other has completed..crash
    3. If an event is fired and then the window is closed …crash
    4. Basic usage is call the fire event, lock events, event is completed, unlock events
