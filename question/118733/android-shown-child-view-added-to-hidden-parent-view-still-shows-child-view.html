Android: Shown child view added to hidden parent view still shows child view

I have 2 views, a parent and child. The child view (in this example an activity indicator) is shown, then added to the parent view. The parent view is hidden. I then add the parent view to the window.

On iOS this works as expected and does not display the parent (or child) view initially since the parent is hidden. However on Android, the child view is shown. This seems like a bug, or am I doing something wrong?

As a work-around I can hide the child view before adding it to the parent but then every time I want to toggle and show the parent, I have to show both views (same goes for hiding). This works but isn't ideal.

var dimView = Ti.UI.createView({
  backgroundColor: '#000',
  opacity: 0.85,
  touchEnabled: false,
  zIndex: 1
});

var activityIndicator = Ti.UI.createActivityIndicator({
  color: '#fff',
  message: 'Blah',
  height: 'auto',
  zIndex: 2
});

activityIndicator.show();
dimView.add(activityIndicator);
dimView.hide();
Ti.UI.currentWindow.add(dimView);
