Change Tab Window in TabGroup

Hello guys,

I'm trying to change a tab group's window in iOS.
As of tab.setWindow(), it should be possible to set the root window. However it doesn't seem to work, even if the method is not stated to be read-only. The tab simply does not refresh.
I've seen this post already, the thread posters think it might be a bug.

Can any official member confirm this to be a bug?

Thanks.

    Hi

    I do not know if I am considered an "..official member…", but I am number 3 on the experts list :-)

    A few days ago I spent quite some time investigating this exact same subject for another persons question.

    Despite the documentation providing both a property called window and a method called setWindow, neither of these actually have any effect on the tabs or the window stack within the tabs. Strangely neither the property or method reports an error during use.

    The property window is not denoted as read-only in the documentation and if it was it would then not have the matching method getWindow.

    At this stage it is unclear if this is a bug or a documentation error.

    I have tried this in Ti SDK 2.1.4 and Ti SDK 3.0.0 with the same results.

    text: 'Label 1'

    title: 'Remove'

    tabGroup.removeTab(tab3);

      Thanks again for the code. I had some time to test it.
      However I think that for my purpose this solution won't be sufficient (Though I will mark it as the best answer, since it actually solves the problem I described). I will briefly explain what I want to acomplish so that others having similar questions will find a way to solve this.

      So my app's structure will be very dynamic. I have a webinterface where I can add category and pages in a hierarchy (which are stored as a tree in a sqlite database). When the app launches, it fetches the root node and its direct children (to save bandwidth). Is the type of the root node a "tab", so will all direct children be the tabs. Since they describe only the tab, the 2nd children level of the root node (children of children of the root node) for the first node needs to be loaded as well (of course the first tab needs to have some content since it will be shown automatically on startup). This tab may contain a dashboard, a list or be a page itself (webview) as well.

      Now the thing is, I want content to "time-out" after a previously defined amount of time. Let's say after 30 minutes the content of a branch will be reloaded. This may happen if the user switches to another tab and then switches back to the previous tab. The reload will be necessary so that news can be displayed or other realtime content. Of course it is unlikely that a user really has the app open for 30 minutes, but I want it to stay dynamic (think of a ticker, where a much shorter amount of time will be necessary). Only that specific branch and its direct children will be loaded, so that not the whole application structure has to be fetched.

      For now I've done it the following way: Each time a selection is made (tab content clicked, list item clicked, dashboard item etc.) a new window will be pushed onto the navigation stack. This makes it easy to let windows be dynamic (I have functions for creating a dashboard window, list windows and pages for instance).
      But imagine the user switches to one tab and back to the other again after its content has timed-out. Now I'd need to pop the root window and set it to another window (for instance when I would change the application structure, let's say I don't want to use a dashboard anymore, but a list view).

      So that it seems to be impossible to re-set a tab's window, I will have to work with views. I will not create dashboard windows or list windows or page windows, I will simply create views of them and manage my own windowing system with pushing windows onto a stack and popping them.
