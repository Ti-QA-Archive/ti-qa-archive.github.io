Save push notifications in local database

Hi there,

I know that in order to get push notifications you have to

  1. register for them,

  2. subscribe to channels,

  3. handle the "clicked" push notification by the user on the callback function.
    So in this third step you can do whatever you what with the push data.

What I intend to do is to save a local copy (in a database) of all the push notifications that my app receives, read or not, in order to present them as a history list inside my app, in a custom notifications area.

With the callback mentioned above I can only store the read ones. What about the other push notifications?

Can I do that? Any example please?

Actually, I should handle push messages on receiving them even if my app is closed or in background. Maybe using a background thread or something?

Titianium SDK: 3.3.0, Platform: iOS 6+
