    • First I'd define a model, then in order to fetch data from remote db and persist changes onto it, I'd implement a custom sync adapter: i.e. https://github.com/viezel/napp.alloy.adapter.restapi

    • Hi Brian,

      you defined the 'books' model using the standard alloy sql adapter, so connected to a SQLite local database. This allow you to make your persist books models locally on the device.
      When you create a model instance in index.js and save it, the sql sync adapter takes care to insert the new record in the proper db table, named after the model, as 'books'.
      This is a way to input records. No need to define a collection at this step.

      Most of the times the table already contain a number of records and you could be needing to display them in some way in the app (say in a tableView).

      Collections comes in help to fetch models' sets from your persistence layer (here the local db).

      As you can see in the docs, there is a simple view's xml markup syntax to define the binding collection, and in your controller trigger data-binding with the tableView:

      http://docs.appcelerator.com/titanium/latest/#!/guide/Alloy_Data_Binding

      Just to test model creation and collection fetch (no databinding yet), try with:

      var book = Alloy.createModel('books', { 
   title : 'Great Expectations', 
   author: 'Charles Dickens' 
});
book.save();


var books = Alloy.createCollection('books');
books.fetch({
    success: function(coll) {
        alert("You have " + coll.lenght +  " book")
    },
    error: function() {
        alert("An error occurred"}
    }
});


$.index.open();

      P.S. Alloy's model/collection method are detailed in Backbone 0.9.2 docs

      hth, regards

    • This line 

      [ERROR] :      message = "'undefined' is not an object (evaluating 'myBooks.add')";

      says that myBooks is undefined, the error rises when you try to call its method .add().

      myBooks is undefined because you are doing:

      var myBooks = Alloy.Collections.books;

      This do not create the collection, it is a reference to a global singleton instance previously created.

      So you should instead use 

      var books = Alloy.createCollection('books');

      to create the collection instance in index.js to be able to access its methods, in my example the method fetch().
