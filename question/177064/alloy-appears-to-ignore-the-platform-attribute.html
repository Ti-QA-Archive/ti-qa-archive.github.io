Alloy appears to ignore the platform attribute

I have the following view:

<Alloy>
    <Window class="container">
        <NavigationGroup id="nav" platform="mobileweb">
            <Window id="navRootWindow" class="container"/>
        </NavigationGroup>
    </Window>
</Alloy>

Whenever I run the app with the mobileweb target Alloy emits

[WARN] :   Ti.UI.iPhone.NavigationGroup (line 3) is deprecated as of Titanium 3.1.3
[WARN] :   Use Ti.UI.iOS.NavigationWindow instead

I've tried using the platform attribute as shown here as well as using the ns attribute, i.e. ns=Ti.UI.MobileWeb, but the result is the same.
The browser does open with the app loaded, but it doesn't look like I expect it to, so I don't know if it's somehow using the iPhone version or if it's using the MobileWeb version and I can ignore the warning indicated.

Titanium SDK version: 3.3.0 GA
Titanium Studio, build: 3.3.0.201407100905
OS: OS X Mountain Lion

  • It sounds like our error message logic is simply assuming that if you're using NavigationGroup, you're on iOS. MobileWeb doesn't have a huge usage rate, especially compared to iOS, so I guess that mistaken assumption has slipped by everyone's notice until now. Please file a ticket to update that error message. Thanks for the catch.
