Using a back button in local HTML WebView file?

First question here, but have really learned a lot from reading other questions and answers. I'm just having a little issue with this one. This is a mobile app for the iPhone, and I'm currently using Titanium Studio, build: 2.0.1.201204132053.

Here is how I have it set up currently:

  • I have a scrollableView with two views, and I'm pulling in local HTML files for them.
  • I'm hiding the status bar so that I can have a more customized looking "status bar" using a png and CSS3, http://cl.ly/1D0U0i0j3f072Y282G0Y.
  • A link, in blue, on one of my scrollableViews opens another local HTML file, http://cl.ly/2h2g0g2u1x25172S150O, but I can't go back to my previous view after clicking the back button, http://cl.ly/1D0U0i0j3f072Y282G0Y.

Here is the code I am using on that HTML file for the Back button: 

<html>
    <head>
        <link rel="stylesheet" type="text/css" href="main.css" />
            <script>
                Ti.App.addEventListener("app:fromTitanium", function(e) {
                    goBack();
                });
            </script>
    </head>

    <body>

        <header>
            <button onclick="Ti.App.fireEvent('app:fromWebView', { goBack });">Back</button> 
            <strong>Athlete Parent Orientation</strong>
        </header>

        <div class="content">

            <p>
                Lorem ipsum dolor sit amet, consectetur adipiscing elit. Sed vitae ligula lectus. Prae sent bibendum laoreet faucibus. 
            </p>

        </div>

    </body>
</html>

Questions dealing with this:

  • Is my back button code correct, as it's currently not working how I think it should?
  • I want to eventually add some MapView's, so should I even be doing a status bar back button this way, or should I use the built-in status bar?
  • What's your favorite color?

Thanks for any help you can provide. I can give more details, code, or screenshots if needed.
