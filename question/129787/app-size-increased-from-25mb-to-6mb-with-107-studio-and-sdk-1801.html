App size increased from 2.5mb to 6+mb with 1.0.7 studio and sdk 1.8.01?

Upgraded to 1.0.7 with sdk 1.8.0.1 (Rhino runtime), and noticed my app increased from 2.5mb to 6mb

Is this normal?

  • The V8 runtime is a bit larger than the Rhino .jar file - we're looking at ways to potentially mitigate this, but shipping the V8 shared library does result in an increased binary size.

  • Also, if you're targeting Rhino, you're getting the Rhino interpreter, as well a two versions of the V8 interpreter, each targeting a separate ARM7 ABI. All packed into your .apk. For whatever unknown reason.

    If you're targeting V8, you might only be getting the two V8 libraries – I'm not sure if Rhino is still included.

    You can look at the compiled .apk to see what I mean. There's a metric ton of stuff bundled in there that you didn't have/need before.

    No clue whatsoever what Appcelerator is thinking with this one. Even two ARM7 versions shouldn't be necessary: simply target the lower ABI.
