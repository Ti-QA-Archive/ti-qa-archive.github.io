setInterval timer stops when changing apps?

My setInterval timer works fine while my app (iphone) is the front app. But it stops cold as soon as I hit the home button to switch apps. My code is very simple:

setInterval(function() {

    Titanium.API.log('INFO', 'Fired');    
    refreshOffers();

},1000);

    This is a "feature" of iOS. If you want to actively do things while paused, look at the kitchen sink background example and how it is configured to take effect at the end of the kitchen sink app.js.

    • More than likely you just need to force a full compile. Ti uses an incremental compile paradigm that does not necessarily detect when it needs to add a new module you are using for the first time. If you are using Ti Studio, use menu Project > Clean. If you are using Ti Developer, you can first try just going to the Edit tab and clicking Save. If the project still does not build, empty the project's build/iphone folder (but leave the empty folder).

    • When you suspend an iOS app from the foreground, it gets "paused" which is why your original setInterval also paused while not in the foreground.

      Now that you have a background service, it can do whatever you want while in the background. So just do another setInterval from within the background service.
