Sample only one alert

I have several alerts but only sample 1


alert("I'm first alert");
alert("I'm second alert");
alert("Another");

When run my app only sample last alert

Any idea of because done that?

    Hi

    Because that is how Android handles them. It determines that only one active alert can be visible at any single time and others must be ignored - the logic would be that an app might accidentally fire many in a loop for example. The Firefox browser has a similar protection - but it asks if it should ignore the subsequent ones.

    The solution is obvious, do not fire successive alerts with the first being dismissed. Firing unmanaged alerts with annoy the user and app would not interact well - if it was allowed of course.

    The correct way to handle alerts is to not use them - or at least do not use the short hand alert use the Ti.UI.createAlertDialog function instead and use event listeners to move on from an alert if your app has to wait for user interaction.
