app's view shrinks and button positioning goes wrong on mobile(android)

    Hi Hamza

    This may be related to the DPI of the simulator and that on your phone.

    When you set values for left, right, top, bottom, width and height - which of these number formats do you use?

    width: 50
width: '50px'
width: '50%'
width: '50dip'
width: '50dp'

    If for example you use

    width: 50

    Android will use pixels (as if you entered 50px), if your simulator density and phone densities are different then things will look very much as you have.

    You can test this by change any numbers as above from simple number to their string equivalents.

    So try using

    width '50dp' // or left, right, top, bottom, height as required

    The put it on your phone. If it works - great, if not - provide a snippets of code and let me know what screen resolution your phone has and what screen resolution you simulator has.

    Further information on the changes to the layout system and how the dimension system works are shown in Titanium 2.0 Layout Changes

    Hope this helps.
