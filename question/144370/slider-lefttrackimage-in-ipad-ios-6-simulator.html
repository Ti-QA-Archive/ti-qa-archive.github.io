Slider leftTrackImage in iPad iOS 6 simulator

Hi All

I don't know if this is a bug but I am experiencing strange behaviour when trying to specify images for the left and right tracks on a slider in the iOS 6 simulator.

I can reproduce the issue by creating a single window project from the create project wizard, and add a slider "FirstView.js" using the following code:

var slider = Ti:UI.createSlider({
min:0, max:200, leftTrackImage:"/images/slider_red.png", rightTrackImage:"/images/slider_white"
});
self.add(slider);

When dragging the slider, the red image for the left track isn't displayed until the value of the slider is approximately 1/5th of the way up. And when you drag the knob to about 4/5ths the slider displays the leftTrackImage on the entire track.

The above example is using Titanium SDK 2.0.1 GA2, in the iOS 6 simulator. But it also occurs in SDK 1.8.2

The problem only occurs in the iOS 6 simulator.

  • Ok I just tested the "slider basic" sample in Kitchen Sink, and there's no problems there. I then tried replacing my images with the ones used in Kitchen Sink, and there are no problems. So no bugs.

    The problem is my images and the way the 9 patch scaling works. So the solution was to use the image used in Kitchen Sink as a template.

    Just strange that the problem doesn't occur in the iOS5 simulator (or on any iPad device I've seen running the app on iOS 5).

    Thought I'd post it if anyone else is running around with wrongly applied 9-patches that start acting up in iOS6 :)
