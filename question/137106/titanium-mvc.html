  • If you actually want to structure your code MVC style, then you'll actually need a framework to do so. There have been a few 3rd party frameworks created on the matter, check out the framework list on the wiki to see if any meet your needs: https://wiki.appcelerator.org/display/guides/Third-party+Tools

    Note: we don't maintain or offer support for the libraries in the list

    If you just want to see the best way to structure your mobile applications, you can generate and play with the sample applications or templates from within Titanium Studio. To use a template, simply create a new mobile project, and on the creation screen, choose Next instead of Finish and you'll see the available templates to build from. The samples window should appear in the bottom left corner of your workspace and contains a list of a few samples built using the previously mentioned templates. Both of these resources should give you a good idea on how to structure your applications for a number of scenarios.

    • Two options. You can create a commonjs module that holds certain global type variables and access and update them from anywhere in your code. But for things like user state, I would say to just use the Properties API. You can easily track and update any values and they will be persistent across app launches.
