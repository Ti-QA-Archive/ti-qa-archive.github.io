Continually monitor the GPS position - but not as fast

Hi,
I'm continually monitoring GPS location with the code as follows:

if (Ti.Geolocation.locationServicesEnabled) {
    Titanium.Geolocation.purpose = 'Get Current Location';
    Titanium.Geolocation.addEventListener('location', function(e) {
        if (e.error) {
            alert('Error: ' + e.error);
        } else {
            Ti.API.info(e.coords);
        }
    });
} else {
    alert('Please enable location services');
}

However if the device is moving on a freeway for instance, it's updated every second or so. Is there a way way to slow it down to every 10 or 20 seconds?

    Hi

    Yes you need the accuracy property.

    Then you can use one of these settings;

    • ACCURACY_BEST (highest accuracy and power consumption)
    • ACCURACY_HIGH
    • ACCURACY_NEAREST_TEN_METERS
    • ACCURACY_HUNDRED_METERS
    • ACCURACY_KILOMETER
    • ACCURACY_LOW
    • ACCURACY_THREE_KILOMETERS (lowest accuracy and power consumption).
    • ACCURACY_BEST_FOR_NAVIGATION (Available in iOS 6.0 and above)

    Like this;

    ...
Titanium.Geolocation.purpose = 'Get Current Location';
Titanium.Geolocation.accuracy = Ti.Geolocation.ACCURACY_HUNDRED_METERS;
...

    • Try to be more descriptive with this line;

      Titanium.Geolocation.purpose = 'Get Current Location';

      It is very likely all calls to Geo will want to know "current location". Apple can get a bit specific about this.

      A few of quick examples;

      Ti.Geolocation.purpose = 'Determine your route for this journey';
Ti.Geolocation.purpose = 'Mapping your horse trail';
Ti.Geolocation.purpose = 'Track your performance in this bike race';

      Just a thought.
