Scrollable View shows up empty

Hello

I've created a scrollable View using Titanium.UI.createScrollableView and added 2 Views with the addView method.

If i open the window which contains the scrollableView the scrollableView appears to be "empty".

If i scroll to the right using arrow keys within the android emulator the second view i've added appears.

I can also scroll back to the first view using the left arrow key .

But why is my first view not shown when the window is opened with window.open()?

  • Ok, i fixed it.

    It seems to be a bug. If you are adding the views using the addView method and the scrollableView is not visible yet.

    Now i had to break my code convention to be able to add the views using the "views" property directly on creation of the scrollableView. Now it works as expected.

    Calling scrollToView() even doesn't fix it. Maybe the index is internelly set to the first view, but because the first view is not known at this point it isn't shown. scrollToView() won't work in this case because the internal index already is set to the first view.

    Tested with Titanium SDK Version 1.6.2 and 1.7.0-RC1.
