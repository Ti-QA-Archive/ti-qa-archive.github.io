Sqlite DB modifications fail

Hola

I create a tableview populated with a sqlite database.

When i clicked on a row, it must deleted the cell ID from de sqlite db.

Here my code example :

table.addEventListener('click', function(e) {

var db= Titanium.Database.open('dbname');
db.execute("DELETE FROM dbname WHERE columnID LIKE '+e.row.rowID+'");
});

It do nothing but when i copied the row value to the clipboard, it's the good rowID ! Normally, everything should work without problems!

When I changing the rowID my an "exesting cell ID" (changing "e.row.rowID" by CellID)

db.execute("DELETE FROM dbname WHERE columnID LIKE 'CellID'");

it delete the row on the tableview and in the database.

What's the problem ?

Thanks.

  • Accepted Answer

    The quotes in your statement are incorrect. You are literally searching for the string +e.row.rowId+. It should look like this:

    db.execute("DELETE FROM dbname WHERE columnID LIKE '" + e.row.rowID + "'");

  • Are you using the desktop or mobile SDKs? Please always specify the platform(s) and SDK levels being used. Without them, it can be hard to give an accurate answer.

    Your mention of the clipboard makes me wonder if you are using the desktop SDK. If so, I have no experience with that. However, if using the mobile SDK, I think you want e.rowData.rowID instead of e.row.rowID.

    But rowID is not a standard mobile SDK property either. So in order for e.rowData.rowID to work you would need to be assigning that value yourself as you populate the table. Are you setting rowID?
