  • Hi Moiz

    That is a HUGE undertaking. The math functions are relatively easy, the biggest problem will be the actual interactions and displaying of the data. You will have a huge number of labels and views being displayed at any one time. You will need to create a selection object so you can select more than one cell at a time.

    If you are looking for a simpler interface then you could start;

    • with a few rows and columns of labels
    • add the formatting style to each label
    • have a generic click handler that when clicked sets the current cell
    • do not aim for in-app-editing, place the edit at the top or bottom of the screen
    • manage the data in a behind the scenes 2 dimensional array
    • you have the clipboard object to provide the cut/copy/paste
    • the existing modules you have found will contain the logic on sums etc, pull these as require

    Good luck :-)

    • You cannot save the formatting information in a plain text format like CSV or TAB (txt).

      Good news, excel support a hybrid HTML table formatting so you can include formatting.

      <table>
    <tr style='border-bottom: 1px solid #000;'>
        <td width='200'><b>First</b></td>
        <td width='200'><b>Last</b></td>
        <td width='250' align='center'><b>Email</b></td>
        <td width='100' align='right'><b>Age</b></td>
    </tr>
    <tr>
        <td>Malcolm</td>
        <td>Hollingsworth</td>
        <td align='center'>someone@somewhere.com</td>
        <td align='right'><b><i>39</i></b></td>
    </tr>
    <tr>
        <td>John</td>
        <td>Doe</td>
        <td align='center'>john.doe@somewhere.com</td>
        <td align='right'><b>25</b></td>
    </tr>
</table>

      BTW this also includes alignment columns widths and borders. You can also merge cells - but I imagine that is too far for your app model at this stage.

      To test; save this on your computer with the file name test.xls and then open it, you may see a confirmation dialog - accept it.

      This should solve the problem.
