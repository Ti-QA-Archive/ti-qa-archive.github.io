    • If you turn off the "Inherited" option (under the Show menu on the right) in the docs, you'll see that backgroundImage for Tab goes away. It's being inherited from a generic View and is apparently not directly supported on tabs. You have to set the tab backgrounds using the properties of the TabGroup. Check the docs for that component to see the full range of properties – there are options for various active, inactive, and selected states.

      You can set the foreground image using the icon and activeIcon property as Joseph pointed out.

      I further suggest you move all your styling to the TSS file. Think of it like HTML – you don't want to mix HTML and CSS because that makes later changes difficult (you override the cascade with directly assigned style properties on HTML elements). XML/TSS works a similar way. You want your XML to set ids and classes, then put all your styling into the TSS. Personally, I put all titles/text values there as well.

  • Make sure you are using at least Ti 3.1.1

    var isAndroid = Ti.Platform.Android;

var someTab = Ti.UI.createTab({
    title: 'someName', window: someWindow,
    icon: isAndroid?'/images/tabBtnAndroid.png':'/images/tabBtn_inactive.png',
    activeIcon: '/images/tabBtn_active.png',
});

    Works well in iOS, make sure the image covers the full tab area…

    For Android… the title will show, and the icon would have to be smaller. Another caveat for Android… you can't set the activeIcon. The title will highlight though. Android is a nuisance to skin.
