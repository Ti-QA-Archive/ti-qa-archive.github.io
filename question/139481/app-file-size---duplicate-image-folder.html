App file size - duplicate image folder

Hello,
Our application follow the structure of kevin w's best practices example with an image folder located in Iphone/images. When we compiled the ipa file seemed unusually large, so we unziped it and noticed two images folder (/images) and (iphones/images). There was also an 8.5 mb unix executable file titled the name of our app.
Any idea on how to prevent the duplicate image folder, or how to reduce ipa file size ?
Thanks !

  • if studio is duplicating the folder it does so for a reason. (remember it's cross platform environment, there should be a valid reason they do this).

  • We was found the similar issue on 2.1.0GA which got duplicate image folder (/images) and (iphones/images). This was happen when putting the resource inside the Resources/iphones/images.If putting resource folder into Resources/images seem solve this issue.
