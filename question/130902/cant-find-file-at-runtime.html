Can't find file at runtime

I have a Titanium Mobile project where the Resources folder looks like:

Resources
-javascripts
 -views
  login.js
  main.js
 my_project.js
app.js

in app.js:

Ti.include('javascripts/my_project.js')

and in my_project.js:

Ti.include('views/login.js');

which works fine. However when I modify it to include the main.js file:

Ti.include('views/login.js', 'views/main.js');

I get a runtime error that the main.js file cannot be found. I had a similar error when I first tried to get the login.js file to be included however it started working and I'm not quite sure what I did that caused it to be recognized.

I am using Titanium Mobile SDK 1.8.01, Titanium Studio 1.0.7, and testing with the iOS 5 simulator.

Any ideas on what is going on?

  • Accepted Answer

    actually should be

    Ti.include('/javascripts/views/login.js', '/javascripts/views/main.js');

    whenever you get this, try to clean the project and build again. Sometimes when you add new files the app is already cached and the new file is not visible.
