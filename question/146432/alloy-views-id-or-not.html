Alloy views id or not

In my application all of my views are loaded into tabs and so have Windows as their outer element e.g.

<Alloy>
    <Window>
        ...
    </Window>
</Alloy>

If i don't give my window a name I can access that window using $.controllerName but if I do give it an ID I have to access it via $.windowID

I understand that this is how it is supposed to work, and my question is: Is there any reason to give my windows an ID? is there any advantage to using $.controllerName?

For readability sake it seems to me that not giving my windows an ID and just using the controller name is best. Is this wrong?

Any information or opinions would be greatly appreciated. Thanks.

-Paul

    Absolutely up to you. There is no technical difference between assigning an ID yourself or letting Alloy do it for you based on the controller name.
