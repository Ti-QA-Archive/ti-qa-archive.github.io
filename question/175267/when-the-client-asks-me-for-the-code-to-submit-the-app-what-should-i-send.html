    Hi

    My first question would be to ask them why.

    Titanium does the entire process for you and you are the developer. I have to be honest I have never actually submitted an app via xcode directly as there is simply no need. Having run the app through xcode to benefit from instruments I can tell you the app works just fine.

    The one inside the build folder will be using your profiles, certificate and other identifying elements - so they may have trouble configuring those to perform a build for production without your assitance - so again seems strange.

    It might be;

    • They do not like you and want to walk away - chances are you are great - so unlikely this one
    • They want to hold up payments - if likely this one; we never submit an app to the store until final payment has been made - suggest you have the same rule - including providing them with sufficient code to do the same for themselves.
    • They may not think you will provide them with the code, your contract will state if you should do or not. The general rule is - you did it for them, they paid you - their code. Some developers charge extra - seems weird to me.
    • They think they are great or an employee wants to show off - not usually a good reason.

    Ask them why and get a good reason - one that does not cause problems as I mentioned above.

    Keep in mind there is a huge difference between the code in xcode to build an app and source code - the client may think asking for the code ready to submit is both those things - they are wrong.

    • Yes and no, whilst all Titanium APIs are mapped directly onto iOS (or Android) APIs not all parts of the built app are compiled straight down at compile time. Many are mapped in real-time when the app is running. Your client is much better off with the Titanium source. It is as industry standard as you can get now, so many developers, so many custom modules, so many app using it already in the stores.
