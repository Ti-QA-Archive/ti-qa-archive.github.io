Ti.App.Properties doesn't work in iOS

  • Application Type : Mobile
  • Titanium SDK : 2.0.1
  • Platform : iOS
  • Titanium Studio : 2.1.0
var loginParams;

if (!Ti.App.Properties.hasProperty('loginParams')) {
    loginParams = {
        driverName : "",
        driverPhone : "",
        customerId : "",
        password : "",
        routeCode : "",
    };

} else {
    loginParams = Ti.App.Properties.getObject('loginParams');
}

var driverName = Titanium.UI.createTextField({
    left : 0,
    top : topVar + 150,
    color : '#336699',
    width : width,
    hintText : 'Driver Name',
    value : loginParams.driverName

});

I get an error saying 'undefined' is not an object (evaluating 'loginParams.driverName'). This same

snippted of code works perfectly in Android. But I have issues trying to run it in iOS.

Any line of code with Ti.App.Properties.setProperty() or Ti.App.Properties.getProperty() doesn't work in iOS.

Can anybody shed some light as to why this might be occurring?

Thanks,

Sham

  • I assure you that Ti.App.Properties works fine in iOS. There are no methods called setProperty or getProperty. Ti.App.Properties.

    You should take advantage of the optional second parameter in getObject…

    var loginParams = Ti.App.Properties.getObject('loginParams',{driverName:'',...});
