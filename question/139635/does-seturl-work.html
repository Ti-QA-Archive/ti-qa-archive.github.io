Does setURL work?

I have a webview and create the window, then create a webview as view1 and set it to index1.html. Shows the index1 file. When the user swipes left I do:

    view1.setURL('index2.html');
    view1.repaint();
    view1.reload();

Try a repaint and a reload neither work. The emulator in the diags shows index1.html still is being loaded. If I do:

     win.remove(view1);
     view1 = null;
     view1 = Titanium.UI.createWebView({url: 'index2.html'})
     win.add(view1);

it works fine and index2 shows. Not sure if either method is better, however why doesn't the setURL work?

Thanks.

JR

  • One better way of doing this is:

    view1 = Titanium.UI.createWebView({url: 'index1.html'})
win.add(view1);
view2 = Titanium.UI.createWebView({url: 'index2.html'})
win.add(view2);
view2.hide();
view1.show();

    on swipe event you can write

    view1.hide();
view2.show();

    This method will take less time and memory as compare to others.setUrl and reload() are not reliable and depends on device performancs.
