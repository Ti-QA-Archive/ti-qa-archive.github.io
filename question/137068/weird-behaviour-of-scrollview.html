Weird behaviour of scrollview

I have a scrollview, on which I add a view on which I have various labels.

The problem is that the scrollview's contentHeight doesn't work as it should and it's set as auto. However if I set it as an integer value, say 1000, and then later on (after adding everything to my view, I set contentHeight of scrollview to 'auto') it will show everything fine but it isn't a clean way right?

Here is my code snippet:

var scr=Ti.UI.createScrollView({
    contentWidth:appWidth,
    contentHeight: 5000,
    touchEnabled:true,
    scrollType:"vertical",
    verticalBounce:true,
    showVerticalScrollIndicator:true,
    bottom: 10,
    top: 0
});

var detailViewXtra = Ti.UI.createView({
    height: 'auto',
    width: winDetailXtra.width,
    top: frontViewTop,
    bottom: frontViewBottom
});
scr.add(detailViewXtra);
winDetailXtra.add(scr);

After adding labels to my view, I write this line to change scrollview's height to required (smaller than 5000 of course):

scr.contentHeight= 'auto';

PS: If I don't do like this, then the labels at the bottom will start overlapping.
