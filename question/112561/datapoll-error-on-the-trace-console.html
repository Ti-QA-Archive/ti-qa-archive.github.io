Data__poll error on the trace console

We see this error on the titanium developer console when we fire up our app…

[TRACE] I/global ( 1001): Default buffer size used in BufferedReader constructor. It would be better to be explicit if an 8k-char buffer is required.

[TRACE] E/QemuSensors( 947): data__poll: len=-1, errno=9: Bad file number

This was found when we tried to track why the app was slow.. A number of the same messages get printed.

Any one have any idea about these errors?

Thanks!

    [TRACE] I/global ( 1001): Default buffer size used in BufferedReader constructor. It would be better to be explicit if an 8k-char buffer is required.

    This message is informational (hence the I&#x2F; prefix), and hence is nothing to worry about and can safely be ignored. Some part of the Titanium code is using a default, rather than explicitly defined, buffer size, which is a perfectly reasonable thing to do.

    If you want to read more about it, see this page

    [TRACE] E/QemuSensors( 947): data__poll: len=-1, errno=9: Bad file number

    This message is an error (hence the E&#x2F; prefix), which is generated because the emulator has no sensors. This will not cause you any problems and can be ignored.

    You can access logcat's output directly using the following at the command line:

    adb logcat

    In the logcat documentation it explains how you can filter out these messages if you wish.

    Cheers
