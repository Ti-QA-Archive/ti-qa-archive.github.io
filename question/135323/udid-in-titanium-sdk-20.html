    • To avoid confusion, and ensure that new apps do not use a deprecated API, we replaced the Apple UDID with our own. This means that “Ti.Platform.id” now returns a Titanium-created ID that uses, in part, the device MAC address.

      From there I understand that Ti.Platform.id returns a perfectly valid ID but not the Apple UDID anymore. To get the Apple UDID you need to use the module which let's you access the Apple deprecated UDID, right?

  • According to Release Notes of Titanium Mobile 2.0.1.GA (Rev. 2) - 4/17/2012: Titanium.Platform.id now returns a per-application unique ID, instead of the Apple UDID.

    So you can use Titanium.Platform.id in your project with Ti SDK 2.0.1
