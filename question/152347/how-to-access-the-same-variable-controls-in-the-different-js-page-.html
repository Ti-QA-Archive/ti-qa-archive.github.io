  • OK so your immediate problem is this line

    url:Ti.Filesystem.resourcesDirectory + 'FeedBackFilesPad/CustomerQA.js',

    in the createWindow.

    Using the URL like this creates a new instance of the JavaScript interpreter.

    This makes it incredibly difficult to access data across windows, also has major potential performance issues and memory leaks. The basic problem is variable scope.

    have a look HERE it is an article introducing commonJS.. Their really isnt much you need to do to bring it into that architecture. But doing so will make passing data around very easy once you have got it.

    and although written a while ago THIS will give you a heads up on using a common module to handle data and variables.

    Hope this helps

    T.
