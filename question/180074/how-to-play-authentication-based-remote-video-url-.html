  • I will suggest prompt user to enter credentials when user click on play button to watch the video. Check those credentials are correct on server side by using web service call. If it success, send remote url video to app. Now play the video.

    Note - If you want to achieve more security send encrypted url to app and decrypt it on the fly inside app logic and then open/play the video using decrypted url.

  • Hi

    I'm sure you know how to do a httpClient call, here is the example from the docs

     var url = "{your url}";
 var client = Ti.Network.createHTTPClient({
     // function called when the response data is available
     onload : function(e) {
         Ti.API.info("Received text: " + this.responseText);
         alert('success');
     },
     // function called when an error occurs, including a timeout
     onerror : function(e) {
         Ti.API.debug(e.error);
         alert('error');
     },
     timeout : 5000  // in milliseconds
 });
 // Prepare the connection.
 client.open("POST", url);
 // Send the request.
 client.send({
    password: {somePassword},
    user: {someUser}
});

    So here we carry out a POST to your server passing an object containing the parameters your server expects in order to authenticate the call. On success this should pass back the URL you need to pass to your video player.

    Of course without any example code I am unaware if this is what you are doing already, but if not then I hope this helps.
