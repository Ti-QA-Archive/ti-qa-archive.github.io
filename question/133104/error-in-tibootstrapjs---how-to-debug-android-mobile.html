  • If you by any chance mix Common JS setup with regular JS files defining objects, and the including them through the app.js file, then the error can be an error in how Titanium interprets its code.
    It seems that JS objects defined in included files are not in the scope of Common JS definitions, but only when the code is interpreted for Android, because I have no problems with that on iOS.

    Hope it gives some clues.

    • In my case the problem was with the way clearTimeout() function was handled on specific platforms. clearTimeout(undefined) -> Works fine on iOS, while on Android you will get the Uncaught Error: Invalid value, expected type Number error.
      This is lack of consistency across different platforms. Hopefully the Appcelerator team will make the behavior more consistent on all platforms in future.
