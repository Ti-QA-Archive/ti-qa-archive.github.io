  • I received this error and was able to fix it by simply removing a JavaScript library (in my case it was jquery.mobile-1.0a3.min.js) from my resources directory.

    The day before I ran into issues with Run on Device I downloaded jquery mobile (and put it in the js directory of my resources directory) to test in a webview. I actually hadn't used it yet.

    Try removing any third party JavaScript libraries that you include anywhere in the resources directory (even if you don't use them) and then see if Run on Device works.

    I stumbled on this fix after I had tried many other fixes mentioned in threads about install errors.

  • The solution to this problem is to go to line 989 in

    /Library/Application Support/Titanium/mobilesdk/osx/1.5.1/iphone/builder.py

    And change

    args+=["CODE_SIGN_ENTITLEMENTS = Resources/Entitlements.plist"]

    to
    args+=["CODE_SIGN_ENTITLEMENTS=Resources/Entitlements.plist"]

    The whitespace after the = is being included in the Entitlements.plist path causing it to be incorrect and for the file not to be found.
