View inside TableViewRow uses wrong Ti.UI.FILL height

Hi,

I'm trying to achieve a Layout like the one shown below in a Tableviewrow:

| Bananas
| Carrots
| Potatoes
| Apple

This is the code I'm using:

<Alloy>
<Window backgroundColor="white">
    <TableView>
        <TableViewRow>
            <View id="container" layout="horizontal" height="Ti.UI.SIZE" top="0">
                <View id="coloredbar" backgroundColor="red" left="0" width="10dp" height="Ti.UI.FILL"></View>
                <View top="0" layout="vertical" height="Ti.UI.SIZE">
                    <Label text="Bananas"/>
                    <Label text="Carrots"/>
                    <Label text="Potatoes"/>
                    <Label text="Apple"/>                            
                </View>
            </View>    
        </TableViewRow>
        <TableViewRow title="Haddock"/>
    </TableView>
</Window>
</Alloy>

A (varying) number of labels of varying height (Bananas, Carrots, …) with a colored line next to them, spanning the entire height of the resulting table row.
The layout works as intended when I set a fixed height for the "coloredbar" view.

Note that this is dummy code, the actual table will be filled via a collection, thus the varying elements.

Since the table rows will be of varying height I cannot set a fixed height for it.
Ti.UI.FILL should do the trick, in the documentation it says: "The Ti.UI.FILL behavior represents growing a view to fill its parent's dimensions"

However, when setting the height of coloredbar to Ti.UI.FILL the view's height extends to fill the entire screen, the result looks like this:

| Bananas
| Carrots
| Potatoes
| Apple

Removing the "container" view gives the same result.
In my opinion, the parent of the coloredbar is the container view, which has the desired height.
Is this a bug in the Ti.UI.FILL behaviour or am I missing something else? Any suggestions on how to achieve the desired layout?
Thanks!

    • I'm guessing you want to keep the TableView fill up the full screen?

      In that case, the coloredbar forms indeed the issue here. No parents of it have a fixed height (Ti.UI.SIZE is dynamic, and added after the rest is calculated), so the FILL takes automatically the full screen space.
      If you add the TableViewRows in code, there is a trick to get the fixed height of a view. This height you can then apply to the coloredbar.

      var viewHeight = view.toImage().getHeight();

      Another (dirty) trick i can think of is by adding empty labels:

      <Alloy>
    <Window backgroundColor="white">
        <TableView rowHeight="Ti.UI.SIZE">
            <TableViewRow layout="horizontal">
                <View width="10dp" backgroundColor="red" height="Ti.UI.SIZE">
                    <View layout="vertical" height="Ti.UI.SIZE">
                        <Label />
                        <Label />
                        <Label />
                        <Label />
                    </View>
                </View>
                <View layout="vertical" height="Ti.UI.SIZE">
                    <Label text="Bananas" />
                    <Label text="Carrots" />
                    <Label text="Potatoes" />
                    <Label text="Apple" />
                </View>
            </TableViewRow>
            <TableViewRow title="Haddock"/>
        </TableView>
    </Window>
</Alloy>
