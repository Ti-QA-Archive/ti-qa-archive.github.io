Minimum target iOS for apps submitted to App Store from 1st February 2015?

Hi.

The following blog says iOS apps submitted to the App Store after 1st February 2015 need to be compiled with at least iOS 8 SDK and support 64-bit architectures.

http://www.appcelerator.com/blog/2014/11/alpha-version-of-titanium-sdk-64-bit-support-now-available/

The Xcode page on Wikipedia, if I’m interpreting/understanding correctly, details the iOS 8 SDK was first available in Xcode 6.0.1, and had a minimum deployment target of iOS 5.1.1. So I presume this means with iOS 8 SDK I can compile apps that will run on devices with at least iOS 5.1.1 installed (to some degree)? I understand some features won’t work on devices running pre-iOS 8 but can be worked around at runtime.

http://en.wikipedia.org/wiki/Xcode

The following page details Titanium SDK 3.4.0 is the earliest version that supports iOS 8.0.x SDK with a minimum deployment target of iOS 7.1.x. Therefore, is this the earliest version of the Titanium SDK I can compile iOS apps against to submit to the App Store from 1st February 2015? If correct, does this mean that those apps will not run on devices running pre-iOS 7.1? Whereas if I wrote the app in Xcode rather than using Titanium, they would run on devices running iOS 5.1.1 (apart from particular features not supported in iOS 5.1.1).

http://docs.appcelerator.com/titanium/latest/#!/guide/Installing_the_iOS_SDK

Thanks.

  • Apple says you can't SUBMIT apps/updates that aren't 64-bit ready starting Feb 1. That doesn't mean that un-updated apps suddenly will be removed or stop working.

    Titanium 3.6.0, in testing right now, is 64-bit compliant.
