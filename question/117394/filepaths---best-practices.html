Filepaths - Best Practices?

When referring to files (includes, images, URLs) what is the best practice regarding relative vs. absolute? If relative how does it work? My app original built for android is having issues with how I reference images and other JS files and instead of trial and error I'd like to better understand the best practice here.

Thanks.

    There are slight differences between android and iPhone. iOS isn't so strict and can understand where you're most likely referencing, usually relative to the resources folder. However I found Android doesn't.

    After a lot of forum searching I opted for this solution. which will adjust the image URL

    var resDir = Ti.Filesystem.resourcesDirectory;
var sep = Ti.Filesystem.separator;
var dir =Ti.Platform.osname == 'android'?'/':resDir+sep;
var theImage = dir+'images/imageName.png';
Ti.API.debug(theImage);
var imageView = Ti.UI.createImageView({
        image:theImage,
    });

    I can now create an images sub-directory within Resources, and both android and iPhone will find my reference to them.
