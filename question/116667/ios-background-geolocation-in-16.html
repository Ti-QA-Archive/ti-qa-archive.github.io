iOS Background Geolocation in 1.6

Hi,

I'm experiencing some problems with background geolocation in my app.
I don't know if I've missed something or what the problem might be.

The app uses the 1.6 mobile sdk release.
What I'm doing is listening to the location events in Titanium.Geolocation.
It fires correctly (or as correct at the crappy GPS in my 3GS is) when the app is active and running in the foreground.
If I press the home button or the top sleep/lock-button it doesn't update anymore.

But when I add a background service that also listens to location events it works when I press the home button.
If I press the top button so the display is locked it doesn't update, UNLESS I've pressed the home button before that.
This is on my iPhone 3GS with iOS 4.2 on it.
On iPhone 4 it sometimes works when I press the top button right away. But not always.

I have added the following in tiapp.xml:

<backgroundModes>
    <mode>location</mode>
</backgroundModes>
<requires>
    <feature>gps</feature>
    <feature>location-services</feature>
</requires>

And I've also tried manually adding it in Info.plist, but still no success.

Does anyone have an working example of background geolocation on iOS?

Thank you!

  • In order to get GPS notifications while my app is in the background, I had to do the following:

    • apply Marcel's patch (see remark above)
    • add code like Ti.App.iOS.registerBackgroundService({url: &#39;&#x2F;background.js&#39;});
    • in background.js, add code like Ti.Geolocation.addEventListener(&#39;location&#39;, event_handler_geolocation);
    • add &lt;backgroundModes&gt;&lt;mode&gt;location&lt;&#x2F;mode&gt;&lt;&#x2F;backgroundModes&gt; to TiApp.xml

    To my surprise, event_handler_geolocation() could just be some dummy function. Merely adding a geo-listener to the background service kept my main app receiving the same events! So it appears that app.js keeps handling geo-events, as long you do the same in a background.js.

    Used Titanium SDK 1.6.3

    • It appeared to be a bit more complicated than I thought. What I didn't realize is that when a app is backgrounded, it keeps running for some time (5-6 seconds) before it gets suspended. So I was under the wrong impression that I still got foreground location events by merely registering a background service (sounds foolish after all…)

      Also, be aware that when you test background services, test for at least 11 minutes. This is because iOS kills a background service after 10 minutes if it is not properly marked in Info.plist as such.
