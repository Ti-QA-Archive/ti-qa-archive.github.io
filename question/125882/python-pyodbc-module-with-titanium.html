python pyodbc module with titanium

i discovered my project in titanium studio sdk 1.1 crash in presence of the statement "import pyodbc" into a python function (from python prompt run regular).
I am in windows 7 platform and python 2.7. Do you have a solution to use pyodbc or can you indicate me an alternative to connect titanium studio to MS SQL?
Thanks

  • Francesco,

    I suspect as Matt mentions below, that the version which ships with the SDKs is currently 2.5.4, you should see it in \ProgramData\Titanium\modules\win32\python\1.2.0.RC4 if that is the SDK your using.

    I am not 100% positive, but if the module was compiled with Python 2.7, it might be incompatible with Python 2.5.4.

    What you can try doing is to build/install using the python located in the above directory (or for the SDK you are using) by pathing it in your environment before your python 2.7.

    pyodbc says it will work with version of python from 2.4, so it should be compatible with 2.5.4.

    The other thing you could do is remove python 2.7 and install python 2.5.4 to be compatible with the SDK. Python will be upgraded in a future release, but I have some other stuff to get off my plate first.

    — commented September 21st 2011 by Alan DuBoff
