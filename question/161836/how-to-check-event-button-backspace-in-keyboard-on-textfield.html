    • Why keypressed is not handle event

      — commented January 23rd 2014 by apichart apinyasudja

    • The underlying platforms deal with things differently. Android has a keypress event that will provide you with this value (without any further work) where as iOS does not and requires more work to extract it.

      I guess that the number of cases where people use this is small and therefore it is not in the Ti SDK core for iOS. It is one of those areas where you would have to write your own iOS module to do it.
