Best practice for encrypting and decrypting data within mobile app

Is there a resource that outlines best practices for safely encrypting data that is being passed back and forth via a web service over https in titanium?

  • Accepted Answer

    Neither base64 nor MD5 count as encryption. In the case of MD5, it is just a hash total of the original value. It cannot be used to decrypt back to the original. Its sole purposes is to compare the MD5 of two things. When they match you have a near guarantee the original was not modified in any way.

    Base64 can be used to "decrypt" back to the original, but it is not designed as an encryption at all and does not use ANY encryption key. Its sole purpose is to convert binary or other non-ASCII text values into text characters which can be safely sent and received using text protocols such as SMTP, POP, and some HTML.

    That said, if you are using HTTPS, then it provides the encryption although you should avoid putting sensitive information in the URL query string itself. Put all sensitive data in the payload, such as POST values.

    I don't know what you are referring to regarding Apple encryption requirements for the App Store. By itself, a program need not use any encryption unless it is desirable or necessary for its own needs such as PCI compliance or HIPAA rules.

    • This answer is misleading. Neither MD5 and Base64 are encryption methods.

      MD5 is a one way hash function. It should not be used in new applications if you follow best practices. Base64 is an encoding standard. Neither of these can be used for encryption.

      If you are building high secure apps you need to secure the whole data chain. The data at rest in your application and server and data in transit. The first is hard (build your own) the latter is easy (built in using https).
