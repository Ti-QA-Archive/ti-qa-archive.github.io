    A module is basically an extension of the Titanium platform. Most of the APIs that are available by default are actually just modules that extend our core platform. We allow you to use that exact same architecture to extend the platform using your own code. This is almost similar to module classes in AS3.

    Check out Module Development Guides for each platform for more information.

    • Modules are primarily written in the native language for each platform. While we do allow for pure js modules to be created, for the most part, modules for iOS are written in Obj-C and modules for Android are written in Java.

      If you don't understand modules classes in your primary language, then its going to be hard to understand modules in another language. Modules simply extend the platform to include functionality that is not already included in the core platform, much like a 'plugin' in other languages. For example, we have modules that allow you to add advertising to your apps, or the box2d physics engine to your apps. These things would not be available in Titanium by default, so we allow them to be added as modules. These modules can both be added to an app, but they dont relate to eachother in any way. You can add as many modules as you'd like.

      Creating modules is a different story. In the end, module creation is something Titanium offers to advanced developers. For the most part, you need to be someone proficient in a native platform language to create them, and/or quite proficient in js to create a native javascript module.

      You don't need to understand modules to use the Titanium platform.
