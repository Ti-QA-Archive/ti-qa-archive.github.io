ACS - Querying Users (Total Number of Users) > 1000

Say a Cloud Server has over 1000 Users.

With ACS Querying lots of Users, usually we make a Query with params page and per_page, though we cannot know the Total Numbers of Users unless we make several calls incrementing page and adding them to the previous array.length of users.

Is there a way to know the total number of users without making several calls. The more users, the more calls we have to make. not very scalable.
