How to localize the name of your App and the Splash/Launch Image

This is actually an answer for a problem that I was looking for, I am working on an App that will be available in 5 languages and the name of the app will be different for some Languages and with that my Launch/Splash Image needs to change accordingly.

To do that you need to create some folders in the Resources folder of your project.

1) Create a folder following the ISO 639 convention for each language you want to support following this format:
language_region.lproj

Then to support Spanish you will create a folder named:

es.lproj

To support Traditional Chinese you will create a folder named:

zh_Hant.lproj

and if you will support English don't forget to create the en.lproj folder

2) Once you have created the language folders, create a new Text file inside each folder named: InfoPlist.strings (pls pay attention to the name as it is case sensitive) and save it using format UTF-16

3) Now edit the InfoPlist.strings file:
Add the following lines, for instance for English

CFBundleDisplayName = "Thanks app";
CFBundleName = "ThanksApp";

The CFBundleDisplayName is the name that will appear in the iPad/iPhone springboard and CFBundleName must have the same value as the NAME field on your tiapp.xml

Now let's create the Spanish version, in the InfoPlist.strings in the es.lproj folder, add the following lines:

CFBundleDisplayName = "Gracias app";
CFBundleName = "ThanksApp";

As you can see you don't need to change your CFBundleName only the Display name

OK, now you are all set with the name of you app, clean up you build folder and Install the App in your iPhone and iPad via the Titanium Developer. Yes, you can only see it working in the device itself, unfortunately this feature doesn't work in the simulator. To test if the name changed just change the language of your iPhone/iPad in the Settings>General>International>Language preference screen

4) How to change the Splash/Launch image:

This is pretty easy, you just need to add the Localized images in their respective lproj folders, for instance here is my file structure for an iPad Landscape only App.

Resources
|- en.lproj
   |- InfoPlist.strings
   |- Default-Landscape.png
|- es.lproj
   |- InfoPlist.strings
   |- Default-Landscape.png

Don't forget to remove the Default-Landscape.png file from your /Resources/iphone/ folder. The launch images once localized should be only inside the language folders
