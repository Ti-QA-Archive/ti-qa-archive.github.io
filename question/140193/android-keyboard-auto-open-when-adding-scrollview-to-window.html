android keyboard auto open when adding scrollview to window

when my app started searchbar auto focus , this is due to adding scrollview in windows

i used following code in app

var win=Ti.UI.createWindow();
win.backgroundColor='green';

var serach=Ti.UI.createSearchBar({

height:30,
width:200,
top:20,

});

win.add(serach);

var scoll=Ti.UI.createScrollView({

height:100,
width:300,
contentHeight:'auto',
contentWidth:'auto',
top:100,
backgroundColor:'red',
softKeyboardOnFocus:Titanium.UI.Android.SOFT_KEYBOARD_HIDE_ON_FOCUS,

});

win.add(scoll);

var i=0;
win.addEventListener('focus', function(){

i++;
Ti.API.info('focuse————————-' +i);

});

win.open();

    You can Hide your search bar at the Time of creation And show it in open event of window by setting Timeout function.

    win.addEventListener('open', function(e) {

    setTimeout(function(e){
        serach.show();
    },1000) ;
});

  • SOLVED:
    in tiapp.xml 's Android tag <activity> set the android:windowSoftInputMode=&#39;stateHidden|adjustResize&#39;

    <activity android:configChanges="keyboardHidden"
                    android:name="org.appcelerator.titanium.TiActivity"
                    android:screenOrientation="portrait" 
                    android:windowSoftInputMode="stateHidden|adjustResize" />
    — answered October 12th 2013 by Joseph Sachs

  • Hi, I'm using Titanium SDK 2.1.3 and setting the following attribute to the window that contains the scrollView seemed to do the trick.

    windowSoftInputMode = Ti.UI.Android.SOFT_INPUT_ADJUST_PAN;
