Working cross-platform custom fonts

Hi all,

This is not a question but some useful information on getting custom fonts working (SDK 1.7.1). There is a very good video on the topic posted here, but for those not wanting to sit through a video and just get the rundown on the steps which need taking, as well as some further information worked out in the process:

On Android

You just copy the font file into your project directory under Resources&#x2F;fonts. You then set the font.fontFamily property of a control to the filename of the font file (case sensitive), minus the extension. Note that Android's fonts don't seem to like spaces in the filename, and won't work if present.

On iOS

You can put the font file anywhere (but naturally, you'd put it in the same place as on Android :p). The trick is getting it registered.

  1. Build your app for iPhone/iPad to generate some files
  2. copy info.plist from build&#x2F;iphone&#x2F; to the root of your project folder (in the same folder as build and Resources)
  3. open up the plist file, and add a new key UIAppFonts. Make it an array value.
  4. Add any fonts you want imported under this key, storing the full path to the font file, relative to your project's Resources folder. Using the Android-safe location, this would be fonts&#x2F;myFontFile.ttf. Remember that the font paths are case-sensitive.
  5. Now, in your app JavaScript, you need to set the font.fontFamily property to the font name of the font.
  6. Delete everything in your build&#x2F;iphone folder (you only need to do this each time you edit the info.plist file)
  7. Compile and run!

As you can see, the font family values you must specify are different for iOS and Android. You could either conditionally code them, or just name the font file the same as its internal font name. But of course, you won't be able to rename the font file like this if the font name has spaces in it!
