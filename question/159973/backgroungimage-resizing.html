  • Hi

    The reason there is no such properties as; backgroundImageWidth or backgroundImageHeight to scale the backgroundImage using backgroungRepeat should be obvious; if you are changing the width and height of a repeated tile image then you are either;

    • stretching the tile means the image is distorted
    • shrinking the tile means the image is distorted

    Whatever you do will distort the image which is a really bad idea and means your app will look amateurish which I hope is not your intention.

    You have not mentioned which platforms you are developing for but I can only imagine you are trying to manage tiles sizes across different density platforms. If this is your reason you should manage this using the correct tile size for each density using density image folders for android and understanding how images density file names are used for iOS.

    If you are not sure how to manage this folders check out the following Q&A answer;

    • How to achieve same layout for Android and iPhone using Titanium
