Open Titanium.UI.iOS.NavigationWindow from a Titanium.UI.iOS.NavigationWindow

Hello, I received an app design from a UX architect: he devides this app into two sections, a login one and a second one containing the rest of the app. Either the first and the second "live" under a (different) Ti.UI.iOS.NavigationWindow.

So, after the login process, I need to "jump" from the first Ti.UI.iOS.createNavigationWindow to a second one.

Problem is: I don't know how to do it because if I try to open the second NavigationWindow as a regular window, the second NavigationWindow does not substitute the first, but it gets loaded as a regular section of the first NavigationWindow.

Question: is there any way to transition to a Ti.UI.iOS.NavigationWindow from another Ti.UI.iOS.NavigationWindow? If so, how?

  • Setting aside the technical issues for a moment, this doesn't sound like good UI design to me.

    What if instead you built a single NavigationWindow with the "rest of the app", and then handle your login as a view that is attached to the NavigationWindow? You can cover the entire window with the view so that the user cannot see or interact with its contents while the login view is visible.

    Once the user logs in, you can remove the login view and the user can use the controls in the main window of the app.
