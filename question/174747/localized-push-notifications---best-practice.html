Localized Push Notifications - Best Practice

What is the best practice to push notifications to a multi language app?

For instance, the app supports 2 languages: EN and IT

Should I subscribe the user to a specific channel (mychannel_en or my channel_it) and then send two different notifications?

Or is there any automatic mechanism managed through the notification payload itself, using the language code to retrieve the right alert text?

The target platform is Android but it could be useful for iOS too.

    Subscribe the users to the correct channel for their language and then send one to each channel (or tags if using Urban Airship).

    • With iOS you can use "loc-key" and "loc-args" within your payload data, instead of unlocalized texts; With Android you can do the same:

      "alert" : {
    "loc-key" : "GAME_PLAY_REQUEST_FORMAT",
"loc-args" : [ "Jenna", "Frank"]
}
