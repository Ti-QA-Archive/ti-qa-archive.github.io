Animation start() is buggy?

I have an imageview showing a helicopter. When pushing the image an animation should start, looping through a set of images. The problem is, when pushing, the image disappears briefly only to show up and start the animation after that. I really don't get it why it disappears.. I've watched several examples and I can't see anything wrong with my code.

Code:

var animationArray = [];
animationArray.push('heli1.png');
animationArray.push('heli2.png');

var animation = Titanium.UI.createImageView({        
   duration:0,
   repeatCount:0,
   width:100,
   height:200,        
   top:100,
   left:100,
   image: animationArray[0]
});
animation.addEventListener('click', function(e){
   animation.images = animationArray;
   animation.start();    
});

If i set the images variable when creating the animation, the helicopter won't show up at all before starting the animation. It's clickable though and the animation will start directly, showing the image.
