horizontal ScrollView in a ScrollableView Android & iOS

Hey, I have a ScrollableView which contains an array of Views (content pages). In one of the views I have a horizontal scrolling filmstrip style photo gallery, it's only small and occupies a about 25% of the vertical space. I have given the user tab controls to move between the views, but I'd also like them to be able to swipe the content views also.

The problem I'm having is that on Android the ScollView doesn't respond to touch swipes from left to right as the scrollableView captures the action and moves the page left or right instead. On iOS it operates correctly - the user can scroll the photo gallery to the end, and once their it passes the continued swipe down to the scrollableView.

Possible solutions:

1: make the android horizontal scrollview work as it does on iOS which would allow the user to scroll the photo strip, but then also get to scroll the next scrollable view once they reach the end of photos. This is the ideal solution.

2: disable touch events from being passed to scrollableView when they are registered on the scrollView. This would mean that the user would not be able to move between the main views from the scrollview/photo-strip. This would be a fine solution.

3: Disable touch events on ScrollableView entirely - the user would habe to use the tabs. If neither of the above are achievable I could disable touch events from the ScrollableView. The problem I've had here is that android doesn't honour touchEventsEnabled and on iOS this property stops the inner scrollView/photo-strip from working/scrolling.
