Perfromance issue question regarding scroll view and table view in a scrollable view.

The set up

I am developing a horizontal news feed using the scrollable view. In each view, there is a background image that can be scrolled horizontally in a scroll view. Additionally, overlaying the image is a table view with about a max of 10 rows.

Since this is a news-type feed, it will continually load new views as the user scrolls. I can remove views if the load becomes too great.

The question

What sort of performance can I expect to get with this setup of a scrollable view with each view containing a scroll view and a table view? Testing has showed no issues so far, but I am using the simulator and an iphone5 - I'll be testing with a 3gs soon.

The background

Application type: mobile
Titanium SDK: 3.x
Platform & version: iOS for now but soon Android
Device: iOS simulator and iPhone 5 physical device
Host Operating System: OSX 10.8.3
Titanium Studio: 3.1

  • Hi

    You do not provide enough information about the numbers of views within the main scrollable view to be able to determine total size.

    Your reference of "…feed, it will continually load…" does not reference WHAT it is, so it could be the scroll view, the scrollable view or the table view.

    You will come across a problem with Android which does not like to mix up scroll views and table views in the same context - so depending on what and how you are adding what to what - this may be a problem.

    You may want to provide a sketch of what things are going where or the code you are using (usable snippet only - not the whole thing) to help determine exactly what you are aiming for.

    BTW to perform real tests, spend at least an hour but hopefully longer actually using the app on a device - make sure you enter and exit the windows and views and constantly load new feed entries (fake them if required). This will help determine memory leaks, also check out the use of xcode instruments to help determine memory leaks.
