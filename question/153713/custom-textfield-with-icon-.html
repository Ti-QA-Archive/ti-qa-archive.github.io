    Possible on iOS using the add() function. On android you'll need to add both to a view and make use of zIndex

    here's a link to add funciton http://docs.appcelerator.com/titanium/latest/#!/api/Titanium.UI.TextField-method-add

    • iOS example

      var win = Ti.UI.createWindow({
    backgroundColor:'white'
});

var tf = Ti.UI.createTextField({
    backgroundColor:'blue',
    width: 100, height: Ti.UI.SIZE
});

var img = Ti.UI.createImageView({
    image: '/KS_nav_ui.png',
    right:0, top:0, width: 30, height: 30
});

img.addEventListener('click', function(){alert('image clicked')});

tf.add(img);

win.add(tf);

win.open();

    • for android (and ios too)

      
var win = Ti.UI.createWindow({
    backgroundColor : 'white'
});
var v = Ti.UI.createView({
    top : 20,
    width : 200,
    height : Ti.UI.SIZE
})
var tf = Ti.UI.createTextField({
    backgroundColor : 'blue',
    width : 200,
    height : 40,
    paddingRight:40
});

var img = Ti.UI.createImageView({
    image : '/KS_nav_ui.png',
    right : 0,
    top : 0,
    width : 40,
    height : 40
});

img.addEventListener('click', function() {
    alert('image clicked')
});

v.add(tf);
v.add(img);

win.add(v);

win.open();

  • Hi

    You have a working example from Hani, but there is a built in version for iOS, you use the leftButton and rightButton properties;

    • leftButton
    • rightButton
    • paddingLeft
    • paddingRight

    The first two accept button views as the parameter and the second two control the space they leave between the button and the text entry section.

    Keep in mind that there is a bug that means if you wish to use the right button but not the left button, you actually have to specify the left buttonPadding anyway - weird but true, just set it to zero.

    For Android, you do as Hani suggests, you create a parent view and add the text box and button to that with suitable spacing.

    Do not try the method Hani suggests for android (works for iOS of course) as Android does not allow non-container views to contain child views.
