  • You're using Ti.include() to get code from someother files. So, you're only running into ONE execution context. The lack of memory it's caused by the constant opening and closing of your windows, let me explain you:

    • The Garbage Collector(GC) only remove objects from memory when there is not a single reference to the object.
    • Maybe you call win.close() into every .js file that you include.
    • BUT because sentences like var winreposito = Ti.include(&#39;reposito.js&#39;);, the reference to the window NEVER dissapears and the GC never free that memory.
    • If you open and close the windows serveral times, the memory will got full and a crash will occur.
    • Ti.include() is basically a COPY/PASTE technique. Not good enough for many applications.

    I recommend to you use another point of view, CommonJS is probably the best one for an app like yours. Take a look at this video for more info. Which is happens into your application it's explained here, take a look to this too.

    I hope this helps,
    Ivan
