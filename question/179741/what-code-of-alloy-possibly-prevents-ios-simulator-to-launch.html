What code of Alloy possibly prevents iOS simulator to launch?

[INFO]: Finished building the application in ...
[INFO]: Launching iOS Simulator

App B is able to build and launch in Genymotion.

I've tried many times of clean but still have no idea what could be the cause.

Any ideas?

      Here's what I found by creating a clean project and add my stuff bit by bit:

      I have a folder structure like this:

      /app/assets/contents/some_folders/some_asset_files

      1. The App could launch in iOS simulator by removing "some_asset_files", but that doesn't serve my purpose.
      2. It turns out I can't use the folder name "contents". If I change it to say "entries", everything is fine as my previous Apps. I just happen to change to use "contents", thinking it's better for this new App.

      So the questions is really what folder names are reserved inside "assets" folder? Is there a explicit reserved name list for folders in the doc. Is this a bug ?
