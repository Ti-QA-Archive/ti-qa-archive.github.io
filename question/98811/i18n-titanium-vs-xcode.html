i18n: Titanium vs Xcode

Hi,

I think there is a problem with the i18n.

When I compile my application with Titanium Developer, it gets the i18n files correctly.

But when I use Xcode, i doesn't work anymore. However, I use to delete:

  • project/build/[projectName].build

  • project/build/iphone/build/

I have checked my generated files (projectApp.app):

  • in the one compiled by Titanium, I have the files:

  1. en.lproj/Localizable.strings

  2. fr.lproj/Localizable.strings

  • in the one compiled by Xcode, I only have the file:
  1. strings.xml

Do you know why? How may I compile correctly with Xcode?

Thanks!

  • I have found an hack, but it's of course not a long terme solution.
    You just have to create in your Resources folder the files (don't forget to save them in UTF-16):

    • en.lproj/Localizable.strings
    • fr.lproj/Localizable.strings
    • …

    During the compilation, these files will be placed in the correct place of your application and the i18n will work.

    But still, it's just an hack. Is there someone who has a better solution?

    • it IS working for me!

      Just remember to manually reformat your XML files to Apple String Resources file format:
      http://developer.apple.com/library/mac/#documentation/Cocoa/Conceptual/LoadingResources/Strings/Strings.html#//apple_ref/doc/uid/10000051i-CH6

      (I use BBedit grep feature.)

      AND, make sure to A) erase the /build/iphone folders' content, B) compile your project once with TitaniumStudio and C) build to a device in xcode (choose the right target!)

      This is handy when testing on other devices than yours. You don't want to make a mess in your friends' iPhone….

  • i had the same problem.

    just test your app with titanium and the simulator.
    next go to build folder and search for ".app"
    right mouseclick show package
    copy the en.lproj / de.lproj or whatever
    paste into [yourProject]/]Resources/ (not the build/resources the other one where your app.js is)

    now it should work and you get now error when you compile it with xcode :)

    • Hello, I got the same problem and the suggested workarund worked for me!
      You don't have to create the Localizable.strings file with the same XML code of strings.xml! Just open your [application].app form the build compiled from the simulator and you will find the [language].lproj/Localizable.strings filed that are created by Titanium. Just copy them into [myProject]/Resources and build from Xcode. Localization will work.
      Hope this can be useful.

    • Ok…I just found it out for myself.
      If anyone needs to fix this:

      Titanium Studio expects Xcode to be located at Userdir/Developer/ (at least for me)
      But my original Xcode was at /Developer/
      So i did:

      mv /User/username/Developer/ /User/username/Developer-beta/
ln -s /Developer/ /User/username/Developer

      Deployed from Titanium Studio and uploaded my update.
      Hope the localization is online soon!
