Issues in Using Class Styles with Alloy in Latest Alloy Update

Hi all-

Has anyone seen issues in the latest update to the Alloy framework (released the week of April 22, 2013) in using classes within tss files? I use classes extensively to style my application, and upon upgrading my framework, I noticed that my views suddenly changed dramatically. Upon investigation, it appears that my class styles are no longer being applied from my tss files to my views. If I take the same properties from my tss files and apply them inline in the controls, the styles apply, or if I change the classes to ids, the styles apply, it is simply when I have a view, with a corresponding tss file, the class is not taking styles and applying the styles (such as top, backgrounds etc of a view for example).

Has anyone else upgrade and seen this behavior?

Thank you in advance for any help

Talk soon
Chris

    I think this was a regression (sorry) when I was solving ALOY-434 in Alloy 1.1.0. I'm fairly certain that it is resolved in 1.1.2 which we are testing this week. Could you do me a huge favor and test your app against the alpha version we are currently testing? It would be great to have your app as another data point to confirm out testing.

    You can download the 1.1.2 alpha via npm like this:

    on mac or linux

    sudo npm install -g alloy@1.1.2-alpha

    on windows

    npm install -g alloy@1.1.2-alpha

    Please give 1.1.2-alpha a shot and let me know if it solves your issue.
