Ti.Barcode doesn't exit properly on Android

I have a tabbed interface which has ti.Barcode on the second tab.

When tab[1] is triggered, Barcode starts a scan. When the user clicks on the "cancel" button, the scan is cancelled and the user is returned to tab[0]. I do this with the following code:

cancelButton.addEventListener('click', function () {
        self.endScan();
        self.tabGroup.setActiveTab(0);
    });

(self.tabGroup has been passed into the scope of this window from the tab object)

This works fine on iPhone simulator, but when I run it in Android emulators, I get a very strange situation. The barcode scanner goes away, and I am returned to tab[0], but the whole app is "greyed out" as if the screen is asleep, and I cannot interact with the app any further. I have to force stop the app and restart to get back in again.

Any suggestions?
