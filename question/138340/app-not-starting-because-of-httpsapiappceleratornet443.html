App not starting because of https://api.appcelerator.net:443

Hello, I'm currently using the latest master sdk 2.1.0.v20120613210250.c8ddb29 (Jun 13).
I think there is a problem related to titanium analytics or some other service called by the API when at the app's startup.

I turned analytics off in my tiapp.xml

<analytics>false</analytics>

When the app starts, Titanium SDK tries to call https://api.appcelerator.net:443 See this image.

I'm connected to the internet but my provider has locked connections to external addresses.

So, if I try to intercept the connection with a connection manager, like "Little Snitch", and I set it up to "Deny" connections, the app will load without any problem because the connection to https://api.appcelerator.net:443 does not start (and has therefore no chance to timeout).

If I set "Allow", the app holds on the splash screen for about 40-50 seconds because the connection to https://api.appcelerator.net:443 lays open until timeout.

Is this a know issue? Is there any way to avoid this https call when the app is starting?

Thank you, Stefano.
