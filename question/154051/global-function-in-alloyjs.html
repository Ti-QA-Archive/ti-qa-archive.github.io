Global function in alloy.js

I was just wondering if a function is going to be used through out the app, would it be best practise to declare this function in alloy.js? The function I have is as follows

Alloy.Globals.windowTransition = 
    function windowTransition(previousWindow, nextWindow){
        //Animations will be added
        nextWindow.open();
        previousWindow.close();
    }

Is this the best way to do it, or will I just create the function in each of the view controllers?

    • You can create a global function, or you can create a commonjs module that holds your shared functionality. That commonjs module should be placed in "app/lib" and can be required with the require() function. So for example, let;s say you put a JS library named example.js in "app/lib/example.js", you would then access it in code via require(&#39;example&#39;)
