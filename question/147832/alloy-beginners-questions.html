    • Well there's the 50 or so examples in the repo and the Alloy Framework docs should be extremely helpful. If that is too few, what number would you consider to be enough?

      Adding a custom property to a view is performed the same as with non-Alloy development. The only difference is in how you reference the elements from the markup.

      index.xml

      <Alloy>
    <!-- add a custom property right in markup -->
    <Window id="win"/ customProperty="123">
</Alloy>

      index.js

      // add a custom property from the controller
$.win.anotherCustomProperty = 'hi there';

      As for sqlite integration, that is by far the most covered adapter in the test apps. Every example in https://github.com/appcelerator/alloy/tree/master/test/apps/models with the exception of "properties" and "binding_no_persistence" uses the sql adapter in varying ways. This leaves you with 9 different tests apps, including the more complete "todo" app, to learn how to use the built in sqlite integration. Your exact use case may not be accounted for, but surely this gives enough documentation and samples to get you going.
