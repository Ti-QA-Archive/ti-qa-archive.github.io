Light Weight Window in a multi tab navigational mobile app

Hello All

We want to build multi tab navigational mobile apps using Titanium sdk v1.8.0.1 on iphone and android similar to kitchen sink UI - A set of tabs with sub menus in table lists with more options.

The Best Practices Wiki says that it is not advisable to use a heavy weight window(multiple context) approach

Also the Titanium.UI.Window API description says that -
A heavyweight window is always created when you open a new window from inside a TabGroup.

Now with the above 2 facts it seems its not possible to create a multi tab navigational mobile with a SINGLE CONTEXT.
So are there any other possibilities.

Also the MEMORY MANAGEMENT details of apps/code is not clearly specified in the wiki.

The "Mobile Best Practices" wiki says "the windows can be closed." and "proxy objects can be set to null".It is confusing - TO DO or NOT TO DO.What does this mean.How is memory freed/managed?

How is memory managed for windows and other UI controls?
Is the memory GCed after a window is closed?

On DDMS for android apps built with titanium it is observed that the "Memory Usage" keeps on increasing.

A lot of information is not clear.

How should i begin and what architecture to follow for our app design.

  • A quick answer is single context is better, ignore for the start light and heavyweight windows.

    Use the commonJS architecture approach, and if you want to code for both IOS and Android with limited code changes Don't use tab groups, unless that is the way you want to add the navigation buttons.

    If you are planning on using a custom menu the that is a good option, but write a way to close and open new windows which keeps the window stack to a minimum or it may blow memory.

    Hope this helps

    T.
