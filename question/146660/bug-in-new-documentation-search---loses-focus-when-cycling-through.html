Bug in new documentation search - loses focus when cycling through

When using the search-bar on the new documentation (v3.x.x) the focus is lost after clicking the > (next) button to get the next set of results. I think it only occurs after the first click of the > button.

Steps to reproduce:

  • Search for something, say "size"
  • Click the next arrow >
  • Click the next arrow > again
  • search results disappear…

In order to get the next page of results you need to blur the search text, then refocus it… rather annoying if the result you need isn't on the first 2 pages…

http://docs.appcelerator.com/titanium/latest/
