Which Android SDK Version use you to package an Android distribution apk?

If you have an application that you want to make it compatible with both Android 2.3.3 that the latest versions of Android (eg Android 5), which Android SDK Version use you to package it?

  • Accepted Answer

    You should generally use the newest version of the Android tooling to build and package apps. This is different than the SDK version (targetSDK/minSDK from Harish's answer). So, in the packaging dialog box, choose the newest version.

  • Hi Vittorio Sorbera,

    If you want your application to be compatible with 2.3.3 through 5.0, then in your tiapp.xml file define your minSDK version to 10(for 2.3.3) and targetSDK version to 21(for 5.0) and build it against Ti SDK 3.4.1.GA (it's the latest stable version). Then create distribution APK.

    Hope it helps.
