Tableview scroll stuttering and performance issues

Titanium SDK - 1.7.2
Platform - iOS

We have a huge problem in tale view scroll performance in iPhone. The table view scrolling stutters frequently on scroll down and then sometimes are not responding. The rows are complex with lot of views and labels, images etc added to it. Unique id is given to the rows with different templates. Please see the sample code AT http://pastie.org/2734106

While scrolling down we get close to 5 to 7 FPS. We also have a huge memory problem where the scrolling decreases the available memory. While we close the window we manually release the memory and is working ok. Since the scrolling is not releasing memory the app is crashing too often.

Please give advice

  • How many records ?….

    Looking at it, this could simply be that the device will not handle that much data easily, and it is struggling to run properly.

    Rather that downloading all the records at once and trying to display them, maybe a paging system would work well.

    Or as a lot of apps work as you scroll down it then loads the next set of data, and removes the earlier set.

    (I cannot see anything obvious in the code).

    Hope this helps.
