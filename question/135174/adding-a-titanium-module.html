  • Hi all,
    There is some configuration issues with the Zooz module for iOS.
    The issues are that the generated iOS xcode project does not find all the relevant ZooZ files in path.
    The simplest way to solve this is as follows:
    1) Place the module as defined in titanium, and build the project once.
    2) It will fail to build but it will generate the xcode project under the build directory.
    3) Open the xcode project in xCode
    4) Right click on the project tree and choose "add files"
    5) Add the zooz embedeed sdk folder that is under the module/assets folder.
    6) Choose all targets and choose not to copy files

    That's it, now all should work

    These steps are also documented in the module documentation file with screenshots

    If you need more help simply
    email me ronen@zooz.com

  • Place the zip file (do not uncompress it) in the root of your project, where the tiapp.xml file lives.

    Make sure the correct module version number is in tiapp.xml.

    Delete the content of the /build directory and build again.

    When building, the zip file will uncompress automatically and install in its correct path.
