Setting Tags/Alias for Rich Push Users Using Urban Airship Module

The Problem: No tags/alias for rich push users

Appcelerator's Urban Airship module works great. The push users being created have my custom tags and alias. But my rich push users do not have any tags or alias. Without tags/alias assigned to my rich push users, I cannot send a rich push to a defined group of users (a most valuable feature – some might say crucial)…the only rich push I can do is a blast to everyone.

I could update the rich push user tags/alias myself (without the module) if I had the Urban Airship user_id (which is only returned when the user is first created - API documentation on this is located here). But alas, I do not have a way (from the app) of obtaining this infamous user_id. The user_id is not exposed in the Urban Airship module.

It seems to me that one of three things needs to happen:

###1. Appcelerator exposes the user_id in their module

This would allow developers the freedom to do what they want with rich push users in their app.

###2. Appcelerator adds a public function in their module to modify rich push users

This would fit the bill nicely (at least for my situation, although the first option adds more flexibility).

###3. Develop some crazy work-around

Needless to say, this is not the best solution – and it is what I am currently doing.

Please let me know if this is already in the works (updating the module) or if you have discovered a work-around or if I am blind and have missed something!

— asked July 11th 2012 by Eric Whitlock
