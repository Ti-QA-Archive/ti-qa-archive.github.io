Android intent getData() returns the same value

A have the following intent filter set up under Titanium SDK 3.1:

<application android:debuggable="false"
                android:icon="@drawable/appicon" android:label="App" android:name="AppApplication">
                <activity
                    android:configChanges="keyboardHidden|orientation|screenSize"
                    android:label="App" android:name=".AppActivity" android:theme="@style/Theme.Titanium">
                    <intent-filter>
                        <action android:name="android.intent.action.MAIN"/>
                        <category android:name="android.intent.category.LAUNCHER"/>
                    </intent-filter>
                    <intent-filter>
                        <action android:name="android.intent.action.VIEW"/>
                        <category android:name="android.intent.category.DEFAULT"/>
                        <category android:name="android.intent.category.BROWSABLE"/>
                        <data android:scheme="http"/>
                        <data android:mimeType="video/*"/>
                    </intent-filter>
                </activity>
            </application>

This filter works correctly if the application is not running. When the application is in background, then the getData function always returns the previous url value. Here is the sample code for reading the url from the intent:

$.tabGroup.addEventListener('focus', function (e) {
    if (e.index === undefined && e.source.id == 'tab') {
        var activity = Ti.Android.currentActivity,
            intent = activity.getIntent(),
            url = intent.getData();

        if (url !== null) {
            Ti.API.info('URL: '+url);
        }
    }
});
