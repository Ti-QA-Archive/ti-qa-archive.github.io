    Hi

    In the root of your app there should be an 'i18n' folder, if not then you should create one. Inside this folder you create another folder for each country you want to localise for, for this example I will use English. So the code is en, therefore the folder will be en.

    ../ -> i18n -> en -> *.xml

    You will then need to create your strings files, inside mine I have 2 files. One is called 'app.xml' and the other 'strings.xml' and here is some example code:

    // app.xml

<?xml version="1.0" encoding="UTF-8"?>
<resources>
    <string name="appname">App Name Goes Here</string>
</resources>

// strings.xml

<?xml version="1.0" encoding="UTF-8"?>
<resources>
    <string name="cancel">Cancel</string>
    <string name="done">Done</string>
</resources>

    Now, there are various ways to use the localisation in your app. In your JavaScript you can use L(&#39;&#39;) to wrap the word you are localising,eg. L(&#39;done&#39;)

    Or for certain elements, there are certain properties suffixed with 'id' such as titleid or hinttextid which can be used to localise the word passed also.

    So depending on whether you are using Alloy or not, here are a few examples or how to use localisation in your code:

    var button = Ti.UI.createButton({
    titleid: 'done'
});

// Or

<Alloy>
    <Button id="btn" titleid="done"/>
</Alloy>

// Or

button.title = L('done');

// Or

$.btn.title = L('done');

    Hopefully you understand everything I have explained and I hope this helps.
