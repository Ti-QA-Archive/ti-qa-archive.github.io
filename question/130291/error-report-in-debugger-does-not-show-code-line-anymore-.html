Error report in debugger does not show code line anymore ?!

Hello,
ever since I updated the SDK and Ti Studio through the latest update 2 weeks ago the error reporting in the debuger (console of Ti studio) is pretty useless! , it used to show (for most of the errors) the file, the line number producing the error and sometimes it was nice enough to mention the reason.
Right now it only says [ERROR] Script Error = {
and lets you shoot in the dark to find the error. I am relatively new to Titanium but I love it, so I don't want to sound complainting but I really miss the old the functionality it saved lots of time to just show you the line number, the rest is luxury, but line number and file of the error source is pretty crucial i think.
Thanks alot,

  • Make sure your default log level is set correctly: Right Click on your project > Run As > Run Configurations > Main tab > Set Log Level to Trace.

    In one of the recent updates, it seems to have gotten reset to a less robust level.
