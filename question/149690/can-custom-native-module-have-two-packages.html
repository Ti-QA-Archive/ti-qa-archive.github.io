Can custom native module have two packages?

Hi, I am writing custom native android module. I want two separate packages and package them to a single module. Both package have a class that extends KrollModule.How can i package them? When i am packaging this module to a sample class , i am getting only 1 package exported to that sample application, other is missing. How to achieve this?

  • My comprehension is that you can only have one class that extends KrollModule per module.

    You can however have many objects that extend the KrollProxy class. So you can get around your limitation by having a module that then exposes different proxies. Actually, that's roughly the way Titanium works. For example, all the UI controls contained in Titanium.UI are contained in a same UIModule class.
