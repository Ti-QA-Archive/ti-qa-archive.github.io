zIndex effect vertical positioning on Android

I have a vertical layout view which works correctly on iOS. However on Android, the elements that have higer zIndex values are placed at the bottom of the lower zIndex elements, rather than below.

What i am trying to do is to place the element at the bottom under the element at the top by giving it a negative top value in the vertical layout mode. Below is the code snippet.

Titanium SDK 3.0.0.GA
Platform: Android 4.1.2
Device: Emulator
Host: Mac OS X

var win = Ti.UI.createWindow({
    backgroundColor: "white"
});

var container = Ti.UI.createView({
    top: "0px",
    left: "0px",
    height: "100%",
    width: "100%",
    layout: "vertical",
});

var above = Ti.UI.createLabel({
    zIndex: 1,
    top: "1%",
    left: "10%",
    height: "5%",
    width: "30%",
    backgroundColor: "red",
    color: "white",
    text: "Above"
});
container.add(above);

var below = Ti.UI.createLabel({
    zIndex: 0,
    top: "-2%",
    left: "5%",
    right: "5%",
    height: "25%",
    backgroundColor: "blue",
    color: "white",
    text: "Below"
});
container.add(below);

win.add(container);
win.open();

However this code yields to this: http://quizapps.azurewebsites.net/ss/zIndex-mess.png

Is zIndex behaviour is broken in vertical layout on Android, or am i missing something?

    • Thank you for your answer Muhammad. Although i can achieve what i want with absolute posititoning as you've stated, I feel that this should not be the correct behaviour, for 2 reasons;

      1. Quoting the documentation chapter Transitioning to the New UI Layout System; zIndex: Stack order of UI component in its parent. Higher values are rendered towards the top, in front of components with lower values. If no zIndex value is set, components stack in the order in which they are added to the parent, with the last child added displayed on top of earlier children. Any component with a defined zIndex value is displayed in front of any components with undefined zIndex.
        Does not affect the actual layout of this component.

      This tells me that horizontal or vertical layout should not be effected by zIndex.

      1. It works as stated on iOS.
