The right way to create listviews - alloy

I'm new to Alloy and Titanium. I cannot figure out the 'alloy' way to create a listview in a controller, and the associated xml in the view. I feel like a lot of examples in the docs have controller logic bleeding into the view and vice versa. So here is my example: create some data items:

var a_data = [];
for(var i=0; i<10; i++) {
    var item = {
        rowTitle: {text: 'Item number' + i },
        searchableText:'everything',
        template:'normal'
    };
    a_data.push(item);
}

then, create the sections:

var sections = [];
var aSection = Titanium.UI.createListSection({
    // properties
    items: a_data,
    headerTitle: "Section A"
});
sections.push(aSection);

then, push the sections out to a list:

$.list.setSections(sections);

The xml is:

<Window title="Charleston">
            <ListView id="list" filterAttribute="rowTitle" defaultItemTemplate="odd">
                <SearchBar id="search" barColor="#000" showCancel="true" height="43" top="0" />
                <Templates>
                    <ItemTemplate name="normal" height="60" accessoryType="Ti.UI.LIST_ACCESSORY_TYPE_CHECKMARK">
                        <Label bindId="rowTitle" id="rowTitle"/>
                    </ItemTemplate>
                </Templates>

                <ListSection id="section">
                </ListSection>

            </ListView>
        </Window>

My questions:

  1. is it better to write $.list.setSections(sections) or var list = Ti.UI.createListView? What is the difference?
  2. How do you get (in the view AND the controller) the search field to search the data rows?? Searchabletext??
  3. What's the right syntax for listDataItem? Some examples have simply title, others have a more abstract JSON.

-Titanium noob, Rails intermediate
