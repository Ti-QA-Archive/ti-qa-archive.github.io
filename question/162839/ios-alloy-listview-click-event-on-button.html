IOS Alloy ListView click event on button

I can't understand why an event defined with alloy doesn't, when the quite equivalent code in pure js works.
Can someone help me to understand this? thank you. I precise that I found every thing in the guide

Here is my alloy code where the click event is NOT triggered

index.xml

<Alloy>
    <Window>
        <View class="container">
            <ListView id="cart" defaultItemTemplate="def">
                <Templates>
                    <ItemTemplate name="def">
                        <Button onClick="myClick">Click me</Button>
                    </ItemTemplate>
                </Templates>
            </ListView>
        </View>
    </Window>
</Alloy>

index.js

var args = arguments[0] || {};

var myClick = function(e) {
    alert('test');
};


$.cart.setSections([
    Ti.UI.createListSection({
        headerTitle : 'Test section',
        items :  [{},{}]
    })
]);
$.index.open();

Here is a pure js code where the click event is triggered

index.js

var args = arguments[0] || {};

var myClick = function(e) {
    alert('test');
};

var plainTemplate = {
    childTemplates: [              
        {
            type: 'Ti.UI.Button',
            events: { click : myClick }
        }
    ]
};

var listView = Ti.UI.createListView({
    templates: { 'def': plainTemplate },
    defaultItemTemplate: 'def'               
});

listView.setSections([
    Ti.UI.createListSection({
        headerTitle : 'Test section',
        items :  [{},{}]
    })
]);

$.index.add(listView);
$.index.open();

    I found it by myself!
    The code that alloy generates puts the definition of my callback AFTER the items creation : so it put "undefined" as a click callback.
    I found a way to avoid it : I change the way to declare the callback (I used a variable declaration, I changed it with a real function declaration.

    Thank you all!
