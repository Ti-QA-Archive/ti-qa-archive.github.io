    • I think we're both in the same boat with respect to Eclipse-based IDEs. Other debuggers certainly would be cool, especially a CLI one.

      Getting a feature like this added is not super easy. We have already planned what we're going to work on for at least the next 6 months. There is not much room for adding something of this size.

      Traditionally you would create a Jira ticket, then it would have to be promoted from a community ticket to a official TIMOB ticket, then get scheduled, and finally someone to work on it. To accelerate this, you could become a customer and then you have access to the support team. They have the ability to escalate issues. If you are a big customer, then you could gain even more traction.

      Now, if you're a bit of a hardcore hacker, there might be an easier way. The Titanium CLI has a hidden option for iOS and Android builds called –debug-host. For iOS, it's value is a colon separated list consisting of <ip>:<port>:<airkey>:<hosts> (see here). I'm not sure about Android since it's currently all in the old builder.py scripts (this will change for 3.2.0 when I port it from Python to Node.js).

      You could write some Node.js (or whatever) TCP server that listens on a port and then pass it into the build command via the –debug-host param. When the app is launched, it will connect to the server and pause the JavaScript engine to wait for you to click step through the code.

      Now the downside. I have no clue:

      • what protocol it uses
      • if it's encrypted
      • how you set breakpoints
      • how you send commands
      • tell what file you're in
      • if this functionality is apart of the open source Aptana Studio or if it's in Titanium Studio
      • what an "airkey" is
      • what "hosts" is
      • how Android differs from iOS
      • if this is using some Eclipse based debugging system
      • etc, etc

      You could dig through the iOS and Android source to uncover most of these and possibly some packet sniffing. I for one would be interested to know. I wish I had the time to hack on this. It would be cool and fun. :)
