  • Can you explain what you mean by "load"? Are you talking about putting all those rows into a tableview?

    That would be painful.

    Take a look at the kitchen sink dynamic scroll example for how to handle this. It loads just a small portion at once – which is fast – then if / when the user scrolls down it goes and grabs enough more rows to let them view more.

    Fast initial response time plus the ability to just scroll.

    • BTW, the kitchen sink example shows how to dynamic scroll but does not use a database to populate the rows. What you want to do is use a SELECT statement which also contains the LIMIT and OFFSET clauses.

      The LIMIT xxx clause tells sqlite to only return xxx rows (however many you want to add each time you increase the size of the table) and the OFFSET xxx clause tells it to start at row xxx of the database result set – where 0 is the first record.

      See the sqlite Select statement documentation for how that works.
