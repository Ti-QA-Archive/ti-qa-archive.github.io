  • As documentation says here the contents of Ti.Filesystem.resourcesDirectory directory which corresponds to your app Resources directory is a read-only.

    so i think you can do this by ignoring the splash screen from custom Theme.xml file and make a top-level activity like fullscreen window having an image view with platform's height & width and it's image property is set to the url of the required image from Ti.Filesystem.applicationDataDirectory that you can write to it easily each time you want change it after app installation.

    i didn't tried this idea and i think not having a native splash screen will make your app startup slow and unresponsive till the loading complete especially if you're using modules.
