I can't get custom fonts to work.

I'm using alloy, Titanium Studio, build: 3.2.1.201402041146

I followed the documentation and I have a .ttf font in assets > iPhone > fonts
in Font Book the Post Script name for this particular font is "Rockwell"

here's the tss:

"#rockwell": {
    width: Ti.UI.SIZE,
    height: Ti.UI.SIZE,

    color: "#000",
    font: {
    fontFamily: 'Rockwell',
    fontSize: "50%"

}

and the xml:


<Alloy>
    <Window class="container" navTintColor="#CE171E" backgroundColor="#CE171E">
        <Label id="rockwell">Hello, World</Label>

    </Window>
</Alloy>

I've tried the example listed in the documentation and it works. I've tried other types of my own fonts, both .ttf and .otf and I can't get either to work..

Does anyone have any idea why these custom fonts won't work? Is it just the file? It's the right file size and the font works on my computer otherwise so I don't understand why it won't here.

    Hello Kerrin,

    You need to set font in info.plist like:

    <key>UIAppFonts</key>
    <array>
        <string>Lobster_1.3.otf</string>
    </array>

    for more information about custom font Look this Document/Example : Alloy-Custom-Fonts.

    i hope this helps. thanks.

    • With recent versions of the SDK (like you're using), there's no need to update the info.plist file. You need to follow the guidelines described in the custom fonts guides. Basically, those are A) put the font file in the assets/fonts folder and B) for iOS, use the PostScript name as the fontFamily name in your TSS. See that guide for more info and tips.

      — commented April 4th 2014 by Tim Poulsen

    • I did both those things and couldn't get it to work. I put this code in the tiapp.xml and it now works 

      
   <ios> <plist> <dict>
        <key>UIAppFonts</key>
    <array>
        <string>ufonts.com_rockwell__2_-2.ttf</string>
    </array>
    </dict>
    </plist>
    </ios>
