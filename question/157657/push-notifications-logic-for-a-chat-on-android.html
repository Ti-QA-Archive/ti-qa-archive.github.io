Push Notifications logic for a chat on Android

Hi everyone,
I'm developing a simple chat application for Android just to study titanium.
You can think of it as a small clone of whatsapp. I won't add any detail about sdk or software versions,
since they are irrelevant for the kind of question I'm posting.
I've got some general questions about the logic that the app should use to manage notification of received messages. I know it's a long post, but I really think this could help a lot of people dealing with this very common scenario.

So basically this is how I was thinking to make it:

  • I will use che ACS users and chat services
  • When a user sends a message to another user, an automatic push notification is sent to that user from ACS on a 'chat' channel.

My goal is:

  • The user will see a list of users he can chat with (just like whatsapp)
  • When the user clicks on another user, a page allowing to chat with that user will open (just like whatsapp)
  • When a user enters a chat page, the history of the last N messages will be retrieved by the server

  • If User A receives a chat from User B:

    • if user A is not running the app, A receives a notification in the tray that he can click to start the app.
    • if user A is running the app and he's not in the page showing the chat with user B, A receives a notification in the tray that he can click to open the page showing the chat with user B.
    • if user A is running the app and he's in the page showing the chat with user B, he just gets the message appended to the list of messages.

Questions

  1. Do you think my approach could stand? Does it make sense or is there any other standard approach?

  2. Since a user must be subrscribed to receive push notifications, to get notifications when the app is not running I only see two ways, am I wrong?

    • Having a service always running in the background even when the app is not running
    • Avoid unsubscribing the user when closing the application. (This looks really weird!)

  3. About the showTrayNotificationsWhenFocused property…I thought the "focused" was referred to the whole application, meaning that at least 1 heavyweight window was on top of the stack, but from my tests it looks like it is somehow behaving differently depending on which window I'm in.

More precisely this is what I do:

  • showTrayNotificationsWhenFocused is left to the default false value,
  • in the users list page I enable the CloudPush and subscribe to the PushNotifications,
  • when in the chat page I register the notification callback. Strange thing is that while i'm in the chat page I keep receiving notifications in the tray instead of getting the callback directly called. Then If I click the notification, the callback is correcly called. How is that showTrayNotificationsWhenFocused property supposed to work then?

Thanks a lot!
Daniele
