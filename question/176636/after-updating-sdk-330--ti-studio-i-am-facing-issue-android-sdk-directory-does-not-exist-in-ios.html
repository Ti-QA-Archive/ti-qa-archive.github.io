    • Long john errors usually mean that the titanium cli is unable to parse or find a config file that it expects.

      Try a ti setup and walk through the process. If that doesnt work, try renaming the config file in the hidden .titanium directory in the current user's home folder and try again.

      Did you open up the android app and download the required/updated android tooling? Google went and changed the a couple of things. Moved some binaries, and now create 'fake' AVD references that don't actually exist, just kind of placeholders. Appcelerator has addressed all of these, but its a cat and mouse game.

  • We believe this is still related to Genymotion, even if you've disabled it. See TIMOB-17344

    There are three possible fixes (but I only recommend the first two):

    1. Use the 3.3.1 SDK build from the CI: ti sdk install --branch 3_3_X
    2. Delete your Genymotion emulators and re-add them
    3. Follow the workaround steps in the ticket.

    Please let us know if that helps.
