Android Scrollable View Problem with 3 views

Hello. I'm very nervous because I have to send a project tomorrow and, in Android, if I add more than two views to a ScrollableView, it fails!!!

This is an example.


var win = Titanium.UI.currentWindow;
win.backgroundColor = 'black';

var vA = Ti.UI.createView({
    backgroundColor: 'transparent',
    top:0,
    width:'100%',
    height: '100%',
    borderRadius:5,
});

var vB = Ti.UI.createView({
    backgroundColor: 'transparent',
    top:0,
    width:'100%',
    height: '100%',
    borderRadius:5,
});

var vC = Ti.UI.createView({
    backgroundColor: 'transparent',
    top:0,
    width:'100%',
    height: '100%',
    borderRadius:5,
});


var scrollView = Titanium.UI.createScrollableView({
    top:0,
    showPagingControl : true,
    views: [vA,vB,vC]
});

win.add(scrollView);

The window opens, but when I slide to the second page, the Apps fails and the execution stops. :(

But if I change the code like this:

var scrollView = Titanium.UI.createScrollableView({
    top:0,
    showPagingControl : true,
    views: [vA,vB]
});

It works!!! It's very weird. I need 3 views.

Somebody help me, please?

Thanks.

    Hi

    I found the problem, strange as it might seem the problem is borderRadius, remove this and everything works perfectly.

    Here is a revised version of your code (with minor modifications) - colours added to to make sure things swap between pages.

    var vA = Ti.UI.createView({
    backgroundColor: 'red',
    height: Ti.UI.FILL,
    top: 0,
    width: Ti.UI.FILL
});

var vB = Ti.UI.createView({
    backgroundColor: 'green',
    height: Ti.UI.FILL,
    top: 0,
    width: Ti.UI.FILL,
});

var vC = Ti.UI.createView({
    backgroundColor: 'blue',
    height: Ti.UI.FILL,
    top: 0,
    width: Ti.UI.FILL
});

var scrollView = Ti.UI.createScrollableView({
    top:0,
    showPagingControl : true,
    views: [vA,vB,vC]
});

    I have tested this on iOS and Android and the problem with borderRadius only exists on Android.

    If you really need the borderRadius then you can add child views with borderRadius and this works without a failure.

    Revised code to show you what I mean;

    var vA = Ti.UI.createView({
    backgroundColor: 'red',
    height: Ti.UI.FILL,
    top: 0,
    width: Ti.UI.FILL
});
var vABR = Ti.UI.createView({
    backgroundColor: 'orange',
    borderRadius: 50,
    height: Ti.UI.FILL,
    top: 0,
    width: Ti.UI.FILL
});
vA.add(vABR);

var vB = Ti.UI.createView({
    backgroundColor: 'green',
    height: Ti.UI.FILL,
    top: 0,
    width: Ti.UI.FILL,
});
var vBBR = Ti.UI.createView({
    backgroundColor: 'orange',
    borderRadius: 50,
    height: Ti.UI.FILL,
    top: 0,
    width: Ti.UI.FILL
});
vB.add(vBBR);

var vC = Ti.UI.createView({
    backgroundColor: 'blue',
    height: Ti.UI.FILL,
    top: 0,
    width: Ti.UI.FILL
});
var vCBR = Ti.UI.createView({
    backgroundColor: 'orange',
    borderRadius: 50,
    height: Ti.UI.FILL,
    top: 0,
    width: Ti.UI.FILL
});
vC.add(vCBR);


var scrollView = Ti.UI.createScrollableView({
    top:0,
    showPagingControl : true,
    views: [vA,vB,vC]
});

    • Quick Tip; remove trailing commas, they will come back to bite you in the future as they are an incorrect coding syntax and can cause apps to fail when being distributed.

      Example;

      var vA = Ti.UI.createView({
    backgroundColor: 'transparent',
    top:0,
    width:'100%',
    height: '100%',
    borderRadius:5, // remove these trailing commas
});
