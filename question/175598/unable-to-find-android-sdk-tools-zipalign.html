    • Just for reference: Here is the according ticket TIMOB-17221 which provides these two workarounds (solved it for me):

      1. "Create "android.executables.zipalign" flag in ti.config:
        Create a "android.executables.zipalign" ti.config flag in CLI & set it to the path for zipalign.
        The new path for the zipalign tools is "android-sdk-folder/build-tools/19.1.0/zipalign".
        Make sure you include tool name zipalign in the path.

      2. Copy zipalign from "build tools/19.0.1" to "tools" folder:
        Copy & paste the zipalign tool from "android-sdk-folder/build-tools/19.1.0" to "tools" folder."

  • Copying files from "build-tools folder and past to tools folder." as the commentor (Luiz Eduardo Eugenio Lopes) above suggest will work. but the most throughall way to do it is to manually config their locations and use Check Environment to see which are not working properly

    appc ti config android.executables.dx "/users/owengerig/Android/tools/dx"
appc ti config android.executables.aidl "/users/owengerig/Android/tools/aidl"
appc ti config android.executables.aapt "/users/owengerig/Android/tools/aapt"
appc ti config android.executables.zipalign "/users/owengerig/Android/tools/zipalign"
