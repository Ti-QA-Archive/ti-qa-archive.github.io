More on "Module Not Found"

I saw in one document where a 'modules' folder was created when the main project containing a module was run with Ti Studio. Inside that 'modules' folder at modules/android was the module id folder, i.e., modules/android/com.example.modulename. I've built my main project with a module project several times and a modules folder etc., has never being created. Since I have an ongoing problem with 'Module Not Found' when running my project, I'm wondering if this could be part of the problem and how I can successfully have the Studio 'run' find my module. All help will be very much appreciated. Ti 3.1, Android SDK 4.2 Win 7
