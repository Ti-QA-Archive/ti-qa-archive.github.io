crossplattform app (iOS & Android) which unit to choose? (px, dpi, percentage) with alloy

Hi guys,

I'm wondering about a general strategy for units for a crossplattform app.
I have already read about the characteristics of pixels, dpi's etc. As far as I understood the best practice is to use pixel for iOS and dpi for android.

Converting dpi to pixel or the opposite is easy (https://developer.appcelerator.com/question/125317/calculate-apps-density-pixel-width–height-using-titaniumplatformdisplaycapsdpi) but what should I do, if I have alloy integrated? Calculating inside TSS is not possible, is it? And for each height, width, top, left, etc. property doing javascript based on the plattform is not really the thing…

So what's the best practice here? Which strategy should I follow?

Other solution:

  • Make plattform based TSS rules
  • Choose pixel for iOS
  • Choose dpi for android

  • Accepted Answer

    I use dip. SInce I plan to use that throughout the project, I change tiapp.xml from

    <property name="ti.ui.defaultunit" type="string">system</property>

    to

    <property name="ti.ui.defaultunit" type="string">dip</property>

    Then you can save some typing and do something like

    "#wMain": {
    top: 20
}

    or

    <Window id="wMain" title="Red Window" backgroundColor="black" top="20">

    I'm all for less typing. :)

  • Hi emre,

    No, you can't do calcuations in TSS files.

    Just refer this link :

    http://developer.appcelerator.com/question/149637/math-in-tss .

    i suggest you to use dp for both platform because in ios it's only accept in pixel.

    for ie: top : &#39;10dp&#39; then in ios take only 10 it can not take dp,px,sp etc. but in android it takes all. if you work in both then use :

    "Button" :{
    top : '10dp',
    height : '30dp',
    width : '100dp',
    left : '10dp'
}

    as above example is working in both and display same UI in all resolution in android and ios. i hope this is helpful to you. thanks.. :)
