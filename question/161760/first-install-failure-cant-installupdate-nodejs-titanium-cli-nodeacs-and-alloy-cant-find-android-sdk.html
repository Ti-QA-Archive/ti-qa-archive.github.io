First install failure: can't install/update NodeJS, Titanium CLI, Node.ACS, and Alloy; can't find Android SDK

I am really new to Titanium and I have problem with installation. At the first run of Titanium Studio (placed in &#x2F;opt&#x2F;Titanium_Studio&#x2F;), it told me there are updates for NodeJS, Titanium CLI, Node.ACS, and Alloy as expected. After selecting all of them and clicking Next, there was message "We were unable to find NodeJS in your PATH …", so I entered &#x2F;usr&#x2F;bin&#x2F;nodejs (it had been installed with sudo apt-get install nodejs). After clicking Next, the license text was only a blank. After accepting, clicking Next, and entering sudo password, it seemed nothing was happened (there was no downloading process etc). Now, I cannot see any Alloy template when creating new Mobile App Project.

Secondly, in Preferences > Studio > Platforms > Android, the Android SDK Home was &#x2F;opt&#x2F;android-sdk, but it said "[Android SDK Home] No Android SDKs were found under the specified SDK location". Now I cannot see any Android choices in Run in the toolbar. Actually I've the following installed:

  • Android SDK Tools rev 22.3
  • Android SDK Platform-tools rev 19
  • Android SDK Build-tools rev 17, 18.0.1, 18.1, 18.1.1, 19, and 19.0.1
  • SDK Platform API 7 rev 3, API 8 rev 3, API 14 rev 3, API 16 rev 4, and API 17 rev 2
  • etc

Help me please.

Info

  • Titanium Studio: VERSION 3.2.0.201312191547, BUILD 201312191547
  • Host OS: Ubuntu 13.04 (raring)

  • on Debian-based distros you need to install nodejs-legacy package which symlinks "nodejs" to "node". Titanium actually looks for the old "node" executable rather then "nodejs".
