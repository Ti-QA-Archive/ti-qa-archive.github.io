    There are different way of doing this all depends on how are you programing you app. Are you using single context or multi context?

    Single context is when you used the Ti.include or CommonJs module. Commonjs is the one that is been push by appcelerator as best practices) you can find examples of this on you Ti studio templates.

    Multi context is the one used in the Kitchen Sink application where you used the url property of you window elements to access other js files. An example will be:

    var win = Ti.UI.createWindow({
    title: 'new win',
    url: 'win_content_file.js'
});
win.open();///<<<this will execute the content on the 'win_content_file.js'

    • The biggest problem is that you're not instantiating the window properly. The return value from require() is a reference to the Categories() function, not a new window object. You have to actually call the function to get an instance that you can open.

      var Categories = Ti.require ('Categories');
var win = new Categories ("some title");
win.open();
