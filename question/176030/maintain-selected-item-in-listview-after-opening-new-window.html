Maintain selected item in ListView after opening new window?

I have a window containing a ListView that allows users to select an item, after which a view to the right of the list updates with details of the selected item. However, if I open a new window to perform some other task unrelated to the list (e.g. a system configuration window), then close that window, the selected item is now deselected. I'm pretty sure this is due to the fact that Titanium assumes a ListView is used to navigate to a new window in order to display details about a selected record, and that when returning to the list the selected item should no longer be selected (I'm getting this from this bug ticket: https://jira.appcelerator.org/browse/TIMOB-13499). The assumption that the new window was opened due to selecting an item in the list is wrong in this case. I've tried working around this issue by listening for the new window's close event and reselecting the selected item, but it causes an animation that seems out of place. I also tried handling setting the item's background color myself when an item is selected, but when the item is deselected as a result of closing a window it just gets set back to white (I even tried putting a View inside the ListItem and setting its background color instead but I get the same behavior). Any suggestions?

  • Just tested that and see what you mean… grr!

    It looks like they addressed the iOS guidelines issue by attaching the reset of the selected item to the re-showing of the window regardless of the action that caused a new window .

    However if the iOS guidelines really state this (i cant find anything after a casual search) then regardless your App would be rejected as they seem to mean ANY navigation away from the active window should reset it state, so your app would fail as well.

    So I would bear this in mind unless you want to have to argue your case with the fickle App approvers
