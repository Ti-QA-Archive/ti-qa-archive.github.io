Apply Autocorrect Value on Event

Is there a methodology for applying the suggested autocorrection on an event?

The use case is this:
A user types a message into a textfield and clicks a button to send the message to the server. If the last word they typed has an autocorrect suggestion, the suggestion isn't applied when the send button is clicked.

Having autocorrect apply automatically when the button is clocked is the assumed functionality, as this is how the native Messages app works, among others.

  • I just figured it out, at least for my variation of your issue (see my comment on your original post).
    Instead of binding the submit function to "return," I changed it to "blur." As I only have one field, that works. Now, when I press enter it will apply the autocorrect, then blur the field, which fires my handler.
    If you just press the button though without somehow accepting the autocorrect suggestion it assumes you are rejecting the suggestion and I don't know any way to do otherwise. But if you're tying it to the return key, it works.
