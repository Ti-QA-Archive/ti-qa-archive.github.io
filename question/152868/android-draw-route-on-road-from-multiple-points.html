  • Hi

    I end up answering this one a lot. If you have the google route then you actually receive two sets of points

    • The directions points, simply a set of point a to point z via b, c & d.
    • The polyline points, these are an encoded set of points that follow the route in a smooth true route following way.

    Here is one of the answers I provided the code for;

    • Android generate route on mapview

    You will see that it decodes and uses the poly lines on the map. The decoded route can be used on iOS and android, although the answer was provided a while a go so on android you should be using the map module now and not the old built in version.
