  • I found this tutorial and now it actually works!!

    Custom fonts tutorial appcelerator

    By including my ttf fonts in a custom plist I can use my fonts in labels and(!) in webviews!

    I tried it out with a very significant font from the internet as well, here's example code:

     var label5 = Titanium.UI.createLabel();
 label5.font = {fontSize:30,fontFamily:'CrazyCrazy'};
 label5.text="1234567890";

    var description = Titanium.UI.createWebView({
        html:"<div style=\"font-family: CrazyCrazy;
                font-size:30px;\">1234567890</div><br><br>",
        height:500,
        width:400,
        scalesPageToFit:false,
        backgroundColor:'white',
        touchEnabled :false,
            left:50,
            top:50
          });

    You can get the font here, it has very significant numbers so you'll instantly know it's the right font

  • The answer is unclear. The Kitchen Sink demonstrates a custom font in use, but AFAIK, not in a web view. Rather, in a label. There are two issues with this, however:

    1. Many people report problems using custom fonts (myself included). It's not clear that they can be used reliably.

    2. Even if you solve #1, there's no example of a custom font in a web view, so it may not be supported.

    Suggestion: modify the Kitchen Sink to attempt what you need. If you can get it working there, you know it's theoretically possible.

    • On the iPhone, you can use the "font-face" CSS declaration and link it to a font file stored in your project. For example, with the otf file in the Resources directory

      @font-face { font-family: 'Kingthings Trypewriter 2' ; src: url('/kingthingstrypewriter2.otf'); } 

body {
     padding: 5px;
     font-family: 'Kingthings Trypewriter 2', Helvetica,sans-serif;
    font-size: 14px;
}
