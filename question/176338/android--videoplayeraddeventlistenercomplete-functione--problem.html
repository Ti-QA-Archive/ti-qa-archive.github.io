Android + $.videoPlayer.addEventListener('complete', function(e) = Problem

I am trying to have a video listen to when the video completes then shut the window.
This works fine in iOS but I cannot get anything to fire on Android. Does anyone know why?

Here is my code…

$.videoPlayer.addEventListener('complete', function(e){
  if (!killEverything){
    index += 1;
    if (args.videoPaths && e.reason == Titanium.Media.VIDEO_FINISH_REASON_PLAYBACK_ENDED && index < args.videoPaths.length) {
      $.videoPlayer.setUrl(args.videoPaths[index]);
      $.videoPlayer.play();
    } else {
      $.videoWindow.close();
    }
  }
});

I have then simplified this to just 

$.videoPlayer.addEventListener('complete', function(e){
  if(e.reason == 0){
    $.videoWindow.close();
  }
});

Can anyone help me out please?

  • Is the event even firing? Have you put a Ti.API.info() call inside the event handler function (outside of your if block) to be sure? If not, then I'd log this as a bug in Jira because our docs say this event should fire and return the constants you're attempting to reference.

    If it is firing, log out JSON.stringify(e) and see whether e.reason is giving you the response code you expect.
