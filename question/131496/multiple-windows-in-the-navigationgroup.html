Windows in the NavigationGroup behaves as a stack, as in when the back button is hit the window is closed (pop from the navigation stack). Is it possible to maintain multiple window session parallely inside the NavigationGroup (navigate using the tabbedbar, etc)? If yes, how?

  • Add all of your stuff to a tiView and add this view to the window. Then just before you pop the window, assign the view to a variable outside of the window (take a reference to it) and you can add it to a new window later.

    So for example (pseudo code really) in the parent window:

    var winView = getWindowAsView();
myWin.add(winView);
navgroup.open(myWin);
navgroup.close(myWin);

var myNewWin = //new window code;
myNewWin.add(winView);
...
..

    I use this technique to get a popOver to pop back up again after doing something like taking a photo which has to be fullscreen and doesn't work with popOver zIndex's. Good luck ;)
