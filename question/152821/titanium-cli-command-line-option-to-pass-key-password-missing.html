Titanium CLI: Command line option to pass Key Password missing

I have created Key Store for Android with different password for keystore and key. I want to use my Keystore to sign my TItanium Android app. But I didn't see option for Key password (keypass)

I'm using below command to generate apk file for play store.

titanium build  --platform android -d $PROJECT_PATH -O "$BUILD_OUTPUT_PATH/" -A $ANDROID_SDK_PATH -K $ANDROID_KEYSTORE_PATH -L $ANDROID_KEYSTORE_ALIAS -P $ANDROID_KEYSTORE_PASSWORD -T dist-playstore

anyone know , How to pass keypass in above command?

Android singing - Keytool Option

-keypass <password>
The password for the key.

As a security precaution, do not include this option in your command line. If not supplied, Keytool prompts you to enter the password. In this way, your password is not stored in your shell history.

-storepass <password>
A password for the keystore.

As a security precaution, do not include this option in your command line. If not supplied, Keytool prompts you to enter the password. In this way, your password is not stored in your shell history.

Titanium Android Build Options

-L, –alias <alias>
Alias for the keystore

-A, –android-sdk <path>
Path to the Android SDK.

-I, –avd-id <id>
ID for the AVD. The ID specifies the API level and supplementary APIs for an AVD. Run the command android list targets to see a list of AVD IDs.

-S, –avd-skin <skin>
Skin for the AVD. Defaults to HVGA.

-K, –keystore <path>
Location of the keystore.

-O, –output-dir <dir>
Output directory (used when target is dist-playstore).

-P, –password <alias>
the password for the keystore

-T, –target <value>
Target to build for. Target can be emulator, device, or dist-playstore.

  • I have the same issue and I solved that !!!

    Chris Barber, you right about the fact that Titanium doesn't support it in the UI, but when checking the Terminal Trace log, you can see the last line (when it stop forever waiting for… nothing?) - at that point, it waits for the keypass which Titanium doesn't know to provide. So what I did is:

    1) Copy the last line (starts with: "jarsigner -sigalg ….")

    2) Stop the build (you might see then the error about missing key password)

    3) Open Titanium Terminal

    4) Past that line (WITHOUT THE STRING - " -storepass * ") and execute it.

    It will ask you for both 2 password.

    That's it, your file is ready in the target folder you submit before in the UI.

    Enjoy ;-)

  • One more thing, if you try to upload to googles market, it will ask you to "zipalign" your APK.

    To do that, you can see the command from using an internal new generated key.

    So after you run - jarsigner -sigalg … , do that:

    /Applications/android-sdk-macosx/tools/zipalign -v 4 "/Users/…../<YOUR FILE>.apk" "/Users/…../<YOUR FILE>.apkz"

    Done!
