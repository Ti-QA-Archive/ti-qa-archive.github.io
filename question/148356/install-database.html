  • Hi

    You appear to have tried installing your database from two different locations in order to solve your own problem;

    Ti.Database.install('../products.sqlite','productSpecs');
// and
Ti.Database.install('bdsample.sqlite', DATABASE_NAME);

    The first technique will ../ is not recommended as it requires that the file is relative to the script being run, however depending on the context the database may not be relative to the script you think it is.

    The second technique assumes the file is located EITHER in the root or relative to the script and for the same reasons as the first technique should be avoided.

    If you locate your database in the root, as in;

    /{appname}/resources/bdsample.sqlite

    Then you should reference it as follows;

    Ti.Database.install('/bdsample.sqlite', DATABASE_NAME);

    If you choose to locate the database in a sub-folder then;

    // database is the root folder with your db
Ti.Database.install('/database/bdsample.sqlite', DATABASE_NAME);

var db = Ti.Database.open(DATABASE_NAME);
