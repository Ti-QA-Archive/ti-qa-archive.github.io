Android - Decimal Pad Keyboard not showing symbols

Hi,

Following is the style which I have defined in TSS:

"#dataView": { 
top : "15%", 
left : '30dip', 
right : '5dip', 
height : Titanium.UI.SIZE, 
color : '#5d6770', 
borderColor : "red", 
borderWidth : "1dip", 
textAlign : Titanium.UI.TEXT_ALIGNMENT_RIGHT, 
backgroundImage : 'none', 
keyboardType : Titanium.UI.KEYBOARD_DECIMAL_PAD, 
font: { 
fontSize: '12dip' 
} 
}

When building the application using 3.2.0.GA and launching the keyboard, the symbols are getting displayed in Android. But when building the application using 3.4.0.GA the symbols are not getting displayed. This issue is observed in Android Samsung devices. It is working properly in Nexus devices.

Any one faced this issue and know how to resolve it?

  • One of our users has recently complained about the same. I've researched a bit and found input type="number" on samsung phones have no decimal point and What models of Samsung smartphones have missing period for html5 input type=“number”?. So it does seem to be an issue with Samsung's keyboard itself on SDK 4.0.

    We couldn't find any solution, but we are using the following workaround: 

    keyboardType: Ti.Platform.manufacturer.toLowerCase() === 'samsung' ? Ti.UI.KEYBOARD_NUMBERS_PUNCTUATION : Ti.UI.KEYBOARD_DECIMAL_PAD
