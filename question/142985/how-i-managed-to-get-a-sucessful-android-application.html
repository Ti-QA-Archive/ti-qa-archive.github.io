How I managed to get a sucessful Android application

Version

I use 1.7.6 because it supports everything and doesn't have major bugs. It's much faster than 2.1 for me , and my application size is just 2MB (unlike 15MB with the new version).

Window control

Appcelerator windows are bad. They don't even support DOM copying. Let's say you want a menu , and you don't want to rewrite it for every window. Not possible.

So i managed to make my own window system. It has a function which decides when to delete UI elements from the screen , when to remove them from RAM , when to add new UI elements. It also has events for the back button and it has activity events making it seem perfect.

Design

I used a custom manifest to specify the target version , which is 4.0 (ICS). So i have a great holo theme. For older versions , i made fallback ICS pngs so it looks the same.

Maps+GPS

I don't use the native maps because they aren't supported by tablets that don't have Google APIs in them (you can see that if you run standard emulator , not Google APIs one). That means app crashing. I used the Google Static Maps API which generates a PNG for some coordinates. That , plus a touchmove event deciding when to scroll a PNG.
The GPS has a fix , it makes 3 passes to make the precise location.

Dragging UI elements

Since in the most versions of Titanium the touchCenter variable is undefined , i made my own interpolation. It drags so smooth.

Fastest HTTP image fetch

I use custom XHR to load an image , then i use a module which creates a new thread to save the image+return it , and uses the titanium proxy to interact with my app.
