    • Sagar,

      I use install exclusively at this point. This is how I use it:

      var dbPath = Ti.Filesystem.getResourcesDirectory();
var db = Ti.Database.Install(dbPath + "fileName.sqlite","file")

      Where "fileName.sqlite" is the exact file name of the db file in the resources directory. The second parameter is the name of the file as it is (or will be) copied into the database folder by the install function. If the db resides in the database folder already, the install process will not override it but will open the database (from the database folder) and return the handle.

      I created the database structure in the SQLite Manager extension for Mozilla Firefox.

      Hope that helps.
