  • "Error message: Cannot set property 'backgroundImage' of null."

    The object you are referring to, Ti.App.win1, is null. It has no backgroundImage property because it is not a Ti.UI.Window. You need to assign to Ti.App.win1 the instance of the window you are attempting to modify.

  • I don't know if that currentWindow stuff works anymore. If you build your apps in a modular fashion using CommonJS modules, you shouldn't need it anyway.

    Build a module that creates the window and all the controls within it:

    MyWindow.js:

    function MyWindow (params)
{
    var _self = null;
    var _logo = null;
    var _messaggioIntro = null;
    var _buttonInizia = null;

    _self = Ti.UI.createWindow ({
        backgroundImage: '/images/sfondo.jpg'
    });

    _logo = Ti.UI.createLabel({...});

    _messaggioIntro = Titanium.UI.createLabel({...});

    _buttonInizia = Titanium.UI.createButton({...});

    _self.add(_logo);
    _self.add(_messaggioIntro);
    _self.add(_buttonInizia);

    return _self;
}

module.exports = MyWindow;

    app.js:

    var MyWindow = require ('/MyWindow.js');
var w = new MyWindow ();

// note that this is overly simplistic -- if you're using a NavigationController
// or a TabGroup, you need to open the window differently
w.open ();

    More tips:

    • I'm not sure what you're doing with Ti.App, but you shouldn't be tacking on random properties to it. If you need the concept of global variables, do it with a simple CommonJS module, not by adding to SDK objects

    • watch your case – you're inconsistent with the naming of your variables (MessaggioIntro and buttonInizia); there are situations where you can make a case error and it will work on iOS, but not on android; you will be very frustrated with such a bug

    • avoid the use of relative paths when referencing your assets (don't use "../images", use "/images"). Again, there are situations where this will work on one platform, but not another
