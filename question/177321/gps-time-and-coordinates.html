GPS time and coordinates

Dear Fellow Developers

Titanium 3x ; Android ; Mobile ; NFC ; Camera ; GPS

I'm working on an App for Mountain Bike Orienteering … and I need to record accurate time and place at the moment a BLE Beacon, NFC tag or maybe barcode is scanned. I have already established and tested Bar Code and NFC [soon Beacon] … this part of the tech I'm familiar with.

However using the Device time is not good enough as this can be set by the user. I need to use something more universal and accurate to nearest second. I was thinking of GPS time signal … i.e. devices with GPS will record the same time in a given location. How is this possible in Titanium?

  • Accepted Answer

    For any non-rooted carrier-connected device, the device's time will be synced to the carrier's signal clock automatically. Few users would be savvy enough to override that. And most devices would drift from that clock only fractions of a second per day if out of signal range.

    I'm pretty sure that the GPS time is returned in the payload of a Geolocation location event but you'll have to check

    If you expect 3G/4G connectivity in the field, you could do a query to a server-side script that queries the NTP time servers. Depending on how accurate you need the result to be, you will have to make some calculations to account for transmission time and server delays, but that would probably be on the order of fractions of a second.
