  • Hi

    The best idea and the one I use on ALL apps is to start using the i18n abilities of the platform. I create these files at the very start of any project and if/when I choose a different name - you simply change the value you are about to see.

    Create any folders and files as noted below - or amend if already created

    {appname}/i18n/en/app.xml (used for iOS apps)

    <resources>
    <string name="appname">New Name</string>
</resources>

    {appname}/platform/android/res/values-en/strings.xml (used for Android apps)

    <?xml version="1.0" encoding="UTF-8"?>
<resources>
    <string name="app_name">New Name</string>
</resources>

    For those who do not know; i18n is the short form name for internationalisation. Your app does NOT need to support multiple languages in order to benefit from using i18n. There are many other benefits including locating all your fixed content into a single location - more about that in the new year.

    Clean the project and run again and your app name should now honour the value entered in one of both of the platforms above.

    This is identical for both Titanium apps benefiting from Alloy and those that do not.
