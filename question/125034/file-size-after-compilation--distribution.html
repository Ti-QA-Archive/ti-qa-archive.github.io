File size after compilation / distribution

Hi,
I've just compiled a simple Android app for distributing (so I could test it on my device).
My program didn't contain much (no resource file either) and I noticed that the file size received is 1.7MB.
I had some who developed something for me for Android that contained a lot more code and did a lot more then the simple program I wrote, and he did it with "native" code (using java + eclipse), and his file size was 170kb.

I was wondering why the file size is so big in comparison? my guess is because it includes all the Titanium SDK.
Second, is there a way to make it smaller?
And, does it causes any limits on performance?

Thank you

  • This is normal and your guess is right (AFAIK, not all sdk is included, but only the APIs your app needs). I think there is no way to do it smaller.

    About performance limitation, file size is not a performance limitation by itself. Titanium is not a platform for real time signal processing and things like but, appart of this, the performance limitations you can find woking with Titanium are very specific (and probably will disappear with next hardware generations).
