Possible to link to application resources (JS and CSS) from hosted page contained in IFrame within App

Hi All,

I would like to know if it is possible to link to application resources in a hosted HTML contained within an IFrame in the app.

The scenario is as follows - I've got an HTML5 android app with a linked html on an external webserver contained within an IFrame as well as several html pages within the app itself. postMessage API being used to send messages from the IFrame back to the application.

Is it possible to save some bandwidth by hosting the CSS & JS includes required by the external page locally in the app rather than on the external server. I have tried using "<script src="app://full_in_app_path_to_resource" within the hosted html to no avail.

Kind regards,

Willem

  • As far as I know, webviews that load remote URLs do not have access to the Application (and it's resources) and vice-versa. This isn't a bug, it's for security.

    You could, however, download the remote HTML, and put it into a local webview, as a workaround.
