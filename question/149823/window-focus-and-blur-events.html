Window focus and blur events

Environment: Mobile sdk, all current versions, testing on iPhone simulator, playing on OSX (current).

I would like to have some dynamic changes made any time the root window (window 0), comes into focus. But the best I can tell the window focus event only fire when window 0 is opened. So it fires once, and anytime navigation removes a window and we are back at window 0, nothing happens. This is not the case with subsequent windows. If 3 windows are opened, and I close window[2], window[1] focuses and a focus event fires as it should. But when window[1] closes, window[0] is silent.

Unless … I force it, with another call to .open(), which does cause focus event to fire. Does anybody have any insight? I've tried some variations on how the listener is defined, eg as a global function, to no avail.

Per docs, focus event is defined after window creation, and prior to opening (same code for all window) …

self.addEventListener( 'focus', function() {
    alert('Boo!'); 
});
