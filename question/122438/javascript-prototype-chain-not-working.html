JavaScript prototype chain not working

Hi all.

I'm using Titanium SDK 1.7.1, Windows XP, Android 2.2 (on Samsung Galaxy Tab v7)

I've noticed that there is a bug regarding JavaScript prototypes and prototype chain in Titanium mobile SDK 1.7.1 and possibly older version (I've noticed this bug a long time ago, but couldn't isolate it/haven't figured out what's causing it).

Here is simple example that shows the issue:

var win = Ti.UI.createWindow({
    backgroundColor: '#fff'
});

var Conf = function() {
    this.height = 50;
};

Conf.prototype.width = 100;

var cnf = new Conf();

cnf.backgroundColor = '#f93';

Ti.API.debug(cnf);
Ti.API.debug(cnf.width);
Ti.API.debug(cnf.height);
Ti.API.debug(cnf.backgroundColor);

//workaround is to copy properties
//cnf.width = cnf.width;

var view = Ti.UI.createView(cnf);

win.add(view);

win.open();

Output:

{"backgroundColor":"#f93","height":50}
100.0
50.0
#f93

So, the bug is that Titanium doesn't check for values in prototype chain when creating components (as you can see if you run the code). As you can see in the output, "width" is defined and it exists in prototype chain (and not as it own property).

Any thoughts before I report a bug? :)
