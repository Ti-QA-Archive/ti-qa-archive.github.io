Build Efficiency and other time save tips

Hello everyone,
My question here, is about tips and other methods to test an iPhone app as fast as possible.

I've been using Titanium for a year now. I am currently using Navigation Group system on my App and to save and test changes , I have to build the application all over again.

When I just use window open/close I can close a window, change it's .js file and open again and the changes are applied…that was just great :)

Is there anyway I can do that with Navigation Group and the require() method?
(I tried Ti.include, cause if you change the included .js and include him again, the changes are also applied. But you can't have a Navigation Group with Ti.Include())

Thanks!

  • I assume you're talking about iOS apps.

    The best tip is to upgrade to Titanium SDK v3.0.2.GA. There was a bug in 3.0.0.GA where simulator builds were doing full rebuilds when it shouldn't have. In 3.0.2.GA, subsequent iOS Simulator builds with only JS file changes take a couple seconds.

    Device and package builds will always do a full rebuild. The reason is complicated and unfortunate. We are working to fix that in a future release.
