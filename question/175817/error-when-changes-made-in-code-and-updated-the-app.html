Error when changes made in code and updated the app

Hi Iam working on ios used sdk 3.1.3. My question is I have uploaded my app into app store and now I want to make changes in the code and update. I have added a table in sqlite in the newer version. When Iam simulating from titanium studio from the older version to newer version where I have added an extra table. Iam getting an error table doesnt exists. So I have deleted the app from simulator and run the app. Now it works. Now does the same error shows if I uploaded the newer version into app store. I have updated in the mobile with the old to new version I got the error. So Iam thinking if I got the error when I uploaded into app store that might be a problem . Please answer.
Thanks

    You have to create the new table via script. Otherwise every user has to delete the app and reinstall it.

    • var db = Ti.Database.open('your db name');
db.execute('CREATE TABLE IF NOT EXISTS "favourite" ("id" INTEGER PRIMARY KEY  AUTOINCREMENT  NOT NULL , "entryid" INTEGER, "type" VARCHAR);');
db.close();

      — commented July 7th 2014 by Sebastian Klaus

    • IF NOT EXISTS is important for further updates. If you ask not for this, you'll get an error on updating your app
