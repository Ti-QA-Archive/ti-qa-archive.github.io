  • There are seemingly dozens of reports here in the Q&A section about the the shake gesture listener no longer receiving events after <insert situation involving native UI widget here>. In my particular case, if I registered a shake event with Titanium.Gesture.addEventListener('shake') … then someone clicked on a link in a WebView, I would stop getting the shake event. Deleting and re-adding the event listener, changing all sorts of properties, etc, did nothing to solve the problem.

    Another issue here in Q&A points at this Jira issue, which contains a workaround: http://jira.appcelerator.org/browse/TIMOB-2977 … it is in the comments on the issue, and basically calls for you to re-implement your own shake gesture using the "update" event on Ti.Accelerometer. This does work … though you may have to play around with the magic numbers a little to get the sensitivity you want, and to keep from getting too many shake "events" in a row.

    However, note that this WILL NOT WORK in the iOS simulator, the "shake" event you can simulate there does not also cause accelerometer output. But it works well enough on my iPhone 3G running iOS 4.2.1.

    I am using Titanium SDK 1.8.0.1 and iPhone SDK 4.3.
