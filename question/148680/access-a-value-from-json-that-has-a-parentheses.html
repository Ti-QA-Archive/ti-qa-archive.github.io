Access a value from JSON that has a parentheses

I'm using YQL to parse a website and then display the information in my app. It was all working really well until I came across something weird. This is what the JSON looks like:

{
    a =     {
        content = 40082;
        href = "/public/caseView.do?csIID=7933";
    };
    p =     {
        a =         (
                        {
                content = 40258;
                href = "/public/caseView.do?csIID=8109";
            },
                        {
                content = 40476;
                href = "/public/caseView.do?csIID=8327";
            },
                        {
                content = 43478;
                href = "/public/caseView.do?csIID=11329";
            },
                        {
                content = 56602;
                href = "/public/caseView.do?csIID=24239";
            },
                        {
                content = 56793;
                href = "/public/caseView.do?csIID=24931";
            }
        );
        content = ", \n              , \n              , \n              , \n              , \n              ";
    };

I need to get the value in each of those "content"s. I'm fine with getting the first one (a.content), but the other ones are throwing me off. Using p.a.content gives me null because after the a there is a parentheses. Any idea on how to access these values?

    Sarah,

    Here's a cool trick I use to work with JSON feeds.

    Assuming that your data feed is http://query.yahooapis.com/v1/public/yql?q=select%20p%2Ca%20from%20html%20where%20url%20%3D%20%22http%3A%2F%2Fcaseinfo.nvsupremecourt.us%2Fpublic%2FcaseView.do%3FcsIID%3D23622%22%20and%20xpath%3D%22%2F%2Fp[.%3D\%27Short%20Caption%3A%27]%2Fancestor%3A%3Atable[1]%2F%2Ftd%22&format=json&diagnostics=true, do this.

    Using Google Chrome, install the JSON View Extension.

    Now, open the feed URL in Chrome, and you'll see it neatly formatted. Then, notice that by hovering over the area you which to extract, you'll get a "path" on your browser's status bar. For example the area that says "NYGREN VS NYGREN" is under query.resutls.td[2].p. This path is given to you by JSONVIew in "Array Format", meaning that you could pretty much grab it from there and use it on your code.

  • Hi

    This will show you how to loop the entries and the inner loop that contains the content and the href. All the values will go to the console.

    Ti.Yahoo.yql('select p,a from html where url = "http://caseinfo.nvsupremecourt.us/public/caseView.do?csIID=23622" and xpath="//p[.=\'Short Caption:\']/ancestor::table[1]//td"', function (e) {
    Ti.API.info('yql' + JSON.stringify(e));

    var intElement = 0, intElements = e.data.td.length, element;
    var intContent = 0, intContents = 0, content;
    for (intElement = 0; intElement < intElements; intElement = intElement + 1) {
        element = e.data.td[intElement];
        Ti.API.info('element: ' + JSON.stringify(element));
        if (element.p.a) {
            intContents = element.p.a.length;
            Ti.API.info('length: ' + intContents);
            for (intContent = 0; intContent < intContents; intContent = intContent + 1) {
                content = element.p.a[intContent];
                Ti.API.info(content.content + ': ' + content.href);
            }
        }
    }        
});
