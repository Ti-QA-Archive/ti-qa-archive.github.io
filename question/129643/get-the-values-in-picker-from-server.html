  • 4 days on same problem for an iPhone & Android app with SDK 1.7.5…

    I found a part of problem : XHR call is async by default an you receive data after picker is return to view or window…

    I test with third parameter of xhr.open to false and it work on iOS but not on Android and not supported on SDK 1.8.0.1

    I search now method to send empty array to picker column and populate after the xhr is ready with data.
