Set application specific locale

Application type: mobile

Titanium SDK: 2.0.1 (04/12/12 16:33 999c68a)

Platform & version: iOS 5.1

Device: iOS simulator

Host Operating System: OSX 10.7.4

Titanium Studio: build, 2.0.1.201204132053

Hi guys,

I just want to know if it is possible to set the application locale. I don't want to use the language configured for the device and need to enforce a specific language, which is determined by the application, at runtime. The problem is that I need to use the internationalization mechanism given by Titanium and to be able to get the localization strings through Ti.Locale.getString() a.k.a. L(). Is this possible?

P.S.: I also don't want to change the device's locale configuration. I want to change ONLY the application settings. I suppose this is not even possible, but I decided to make it clear, just in case.

  • I have a similar problem, but the solution Alberto provides cannot help me directly.

    The i18n in combination with L(…) functionality of Titanium Internationalization will provide language items for the languages which are present in the i18n directories.
    In my case I have English and Dutch.
    When the device language is not present in there, the provided L(…) function will return the English variant. But suppose a user has a German device language and for whatever reason he wants to use the Dutch translations in stead of the English?

    In that case I would want to switch to Dutch locale…
