Requirements for the app

I have the following requirements for the client. Can anyone suggest me how it is possible in appcelerator? Or how many requirements are possible out of the below requirements.

Mobile Application (iOS, Android) with features such as Augment Reality, beacons, Route map etc.. integrated into it.

  • Navigate to Place (Navigator)

  • When user points his/her mobile towards a certain hotspot which can be a chandelier, glass window, Calligraphic art works on walls of house then application should show contextual information about the particulate object, let us say I am pointing my phone towards chandelier then it should give me information in the form for text overlay about the chandelier like ( It was imported from Morocco, it is handmade, it has 3 meters in diameter etc.)

  • Application should Beep when he/she is in the hot zone so that user know he has something to look at or user can be also notified with image so that they know what they have to look at along with text overlay etc.. (using beacons)

  • Application needs to have multilingual support English, Arabic.

  • Contents should be downloaded in real-time and contents need to be managed using content management system.

  • Integrated with Analytics.

  • Audio Video integration based on Hotspot/marker.

  • Built in guided tour based.

    • Whilst I appreciate you are new to the platform you need to understand that you will need to spend some time learning how to achieve what you/your client are hoping for. This is not because it is difficult - nothing you listed is difficult and again; all are possible.

      If I was doing this then it would take a few days. Then more once you take into account the requirements of an app beyond that of the raw code.

      You are asking for the unanswerable; "how long will it take you" - no idea. You say you are new to the platform, likely you are new to many of the items on the list you may be new to JavaScript, you may be new to creating apps.

      If someone was asked how long would it take to run a mile - some would saw 4 minutes, some would brag less than that, some might say I could do it in 6 to 8 minutes. I on the other hand am the wrong side of sport, the wrong side of age and the wrong side of flat feet. I would walk the mile and it would take me 15 minutes if the old 4 miles per hour logic works.

      I could do it in a couple of days; but then I having been using the platform for years, I understand JavaScript well I have been building on and improving my techniques each and every day. Some call me an expert, I am certainly a professional.

      This would take others a week or two, some more, some less.

      But you will have to judge for yourself how long you will take. I spent a year learning the platform, how apps worked on actual devices, what different screen resolutions and aspect ratios changed in the app and the user experience. I learnt how different iOS is from Android, how phones are different from tablets. How structured code is a must not an aim. How much work needs to be achieved outside of raw code to complete an app.

      After I spent that year learning I then started accepting paid contracts. I continue to learn every day, because it continues to change every day.

      You will know how long it will take once you have finished it. If you are unable to advice your client right now as you are unable to guess for yourself - then maybe you need to learn "how" before you can say you "do".

    1. Navigate to Place - Use Geolocation and navigation service eg google maps
    2. Object recognition - Nothing inbuilt but there are libraries that could be wrapped at Ti Modules
    3. Beacons - Yes, on both platforms
    4. Multilingual - Yes, inbuilt support for localisation of Apps
    5. Downloadable content - Yes, http support available
    6. Analytics - Yes, inbuilt and third party available
    7. Hotspot - Another candidate for a Ti Module
    8. Built in guided tour based.- Sure, start coding
