    Go to the dist folder within the GitHub repo to find the zip. The direct link (at the time of this posting) is: https://github.com/appcelerator/titanium_modules/blob/master/map/mobile/android/dist/ti.map-android-2.1.2.zip the click View Raw.

    • I don't believe you can use the debug certificate to generate the API key and have it work on device. I'm pretty sure that would work on the emulator but not a device. You have to sign your app and get its SHA1 fingerprint (which is created based on your app ID) from the keystore. Then, you can use that to generate your API key. At least, that's what I had to do to get maps working in my test app (for the training class).

      To confirm, though, you added both the API key and the application ID where indicated in the tiapp.xml, right?
