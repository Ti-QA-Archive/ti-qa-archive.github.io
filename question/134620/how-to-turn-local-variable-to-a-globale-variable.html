  • Please don't use application level variables unless you absolutely have to.

    If you don't want to declare the variable outside of the function then using persistent data is a good solution for small pieces of data.

    as above Ti.App.Properties.setString and getString.

    Really it's not good practice to have Global variables if your using commonJS architecture, and going forward commonJS is going to be the solution to use.

    There is a solution to using global variables in commonJS, but you will have to define them upfront.

    Have a look at http://www.thewarpedcoder.net/2011/12/apccelrator-titanium-commonjs-global-variables/

    Hope this helps

    T.

  • hi ruoli,

    if you are using commonJS style then you can follow the method as Inzori said..

    if mySearchWin is your current window object and you want to use variable name in same window then you can declare like var name=''; in your window or you can also use like what you are using right now mySearchWin.name then no need to declare any variable…

    or if you are not using commonJS method and do not want to use that variable in only your current window then you can set that variable as

    Ti.App.Properties.setString('name',<value>);

    or

    Ti.App.name = <value>;
