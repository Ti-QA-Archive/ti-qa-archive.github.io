What is the preferred way to pass a variable to a view file?

Hello,

I'm starting using Titanium (I'm also a JavaScript beginner).

I try to build a basic app where I display from app.js a new windows defined in a separate view file (using the "URL-based windows described in http://guides.appcelerator.com/en/architecture.html#windows). In my view file, I have a button and add to it a event listener.

I would like to define the listener callback function in app.js and not directly in the view file. Is there a preferred way to pass a variable (in my case the listener callback function) from app.js to the view file?

Thanks!

    • OK, thanks Paul for the guidance!

      Here is the solution that worked for me: 

      // in app.js

var win = Ti.UI.createWindow({url:"views/my_view.js"});        
win.open();
Ti.App.addEventListener('my_action_handler',function(e){
  // code
});

// in views/my_view.js

view.add(button);    
button.addEventListener('click', function() {
  Ti.App.fireEvent('my_action_handler', my_data);
});

  • Probably the simplest thing to do is create an event listener on the new window you've opened, then inside the window fire that same event when the time is right. Nice thing about events is that you can pass data back at the same time like so:

    Ti.App.fireEvent('MyEventString',{myVar:'foo'});

    Because the data you're sending back is a JS object you can add as many different variables as you like:

    Ti.App.fireEvent('MyEventString',{myVar:'foo',myOtherVar:100});

    In your app.js you just need to listen for this event being fired:

    Ti.App.addEventListener('MyEventString',function(e){

     // add yr logic in here
     // e.myVar & e.myOtherVar are now available by simple object reference like this

});

    Note: I've written this from memory but I'm pretty sure it's correct.

    Good luck
