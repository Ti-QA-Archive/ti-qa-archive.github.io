    • i THINK i may have found it. I had _self.layout = null when creating a view, to override the layout:&#39;vertical&#39; that was set by default on a reusable view:

      
var _self = new ui.BasicView();
_self.layout = null;

      I hadn't realised that layout cannot be null, but if not setting it to vertical or horizontal it has to be set to 'composite'.

      So that seems to have fixed it… Although now I have another issue… It looks like Android cant handle large images. So now I have to replace a large majority of all my background images with solid colours. that sucks.
