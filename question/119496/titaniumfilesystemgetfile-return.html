  • Which platform(s) are you targeting? It is best to always include that in the question or tags, because it can make a big difference in the reply. At this stage we don't even know if you are building a desktop or mobile app (which have very different feature sets).

    Also, your question is not clear. Aside from the fact the second example is missing the closing quote on the filename, the two examples should produce similar results.

    The getFile() should never read the data of the file. The getFile() method returns a reference to the file object. With that object reference you can then use the read() method to get the file contents as a blob, or read().text to get the contents as a string.

    But read() always returns the entire contents of the file. If the file is too large for that to be practical, mobile SDK 1.7.x will introduce the ability to use streams and read just portions of a file at a time.
