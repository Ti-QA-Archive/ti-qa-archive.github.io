TI Desktop can't find my Android SDK

Just installed Windows 7 Home Advanced.
Installed Android SDK latest version in e:\Program Files\Android\android-sdk.
When trying to locate the Android SDK in TI Desktop, it says:

Couldn't find adb.exe or android.bat in your SDK's "tools" directory. You may need to install a newer version of the SDK tools.

I checked my Android SDK installtion directory, the tools dir is there and there is android.bat file but no adb.exe. Something wrong with my installation of Android SDK? Bujt my Eclipse seems working fine with my current Android SDK installation.

Any hints?

    You need to "trick" Titanium in seeing the files in the &#x2F;tools directory. The best way to do this is to create a logical link between &#x2F;tools and &#x2F;platform-tools.

    Under Windows 7 or Vista

    cd C:\Program Files (x86)\Android\android-sdk-windows\tools 

mklink adb.exe ..\platform-tools\adb.exe 
mklink AdbWinApi.dll ..\platform-tools\AdbWinApi.dll
