ImageView and Remote Image caching

I have an iPhone app that uses a Imageview to display a remote image to the user. The image is a CAPTCHA graphic used as part of a sign on process. The image is unique each time (with a different name).

The problem I have is that the remote image seems to be stored in the applications tmp directory and, after having the app penetration tested it was pointed out that if a keylogger had been installed and that was related back to the local images in the tmp dir it is possible to deduce what the user response to the CAPTCHA was.

Now I know there are a lot of if and buts about this scenario, but I was wondering if there was any way of either stopping this tmp caching or at least removing the image from the cache when it was finished with?

The image URL is determined via a REST call to a server and I was wondering if I ensured that the REST server sent back a no-cache directive in the HTTP headers, if this would prevent this problem ?
