fastdev not sending source files

When I start up fastdev and launch my app with titanium run –project=android all the files are served with fastdev.

But later when I do code changes nothing is sent.
When I switch between views in my ScrollView the images are sent, but no source files. So my code changes are not reflected in the emulator.

Can this have anything to do with how Im including files? They are all included with Ti.include() in my app.js file.

  • Hey Niklas

    Fastdev doesn't actually "send" files, but the app requests them directly from the Fastdev server. This means any files that have changes, and are included from your app.js, will require an app restart to see. The quickest way to do that with Fastdev is:

    $ titanium fastdev restart-app

    You can read more details by checking out my Fastdev Reference for Android
