Fast clicking a view (android)

Hi all.

I've found that in android when you double click a view, if you haven't defined the dblclick event for it, the click event is triggered twice. Even the click events appear to be executed in different threads, because adding a one thread "semaphore" won't stop the double execution of the function.

Is that normal behaviour?

I ask because this forces me to always add a listener to the dblclick event to avoid things being executed twice.

Am i forgetting something?

  • The typical scenario I have encountered is this:

    • user taps a button
    • event listener starts opening a window; once the window is open, it will cover up the button, so you shouldn't have to worry about a second event on the button, but…
    • user taps screen again before the window opens up, and the event listener tries to open another window

    I think I've even seen somebody tap three times and get three windows open. BTW, it affects iOS as well, but it's possible that your iOS devices are fast enough that you haven't noticed it yet. I first discovered the problem on a slower android phone. I would imagine it might be hard to trigger this problem on a Galaxy S3, for example.

    I use an "event throttle" to deal with this. I check with the event throttle to see if it's OK to open the window. If so, I register the new window with the event throttle. The event throttle won't allow another window to open until that one is closed.

    See the code in the TitanUp library.
