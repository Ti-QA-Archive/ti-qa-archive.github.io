    Couple of typos in your button declaration. Should be createButton, not createbutton. Try:

    var button1 = Titanium.UI.createButton({
    title : 'button1',
    borderRadius : 2,
    right : 'auto',
    bottom : 50,
    width : 250,
    borderColor : '#cccccc',
    backgroundColor : '#eeeeee'
});

  • First, please always specify the platform(s) you target – or at least mobile vs desktop – and the SDK level being used when you have a problem. Also, "the sim will run then crash" is less helpful then telling us what message you get on the simulator or console.

    Second, object properties are case-sensitive, so be careful how you name them. In general, all Titanium object properties begin with a lowercase letter so BorderColor and Backgroundcolor should by borderColor and backgroundColor respectively.

    When you use a name which is not a real property, you are in essence creating a custom property by that name. That doesn't create a problem by itself, but it does mean the Ti module will not be expecting a property by your name and will thus ignore it. That can lead you to think something isn't working right when really it is just spelled wrong.

    Third, right: 'auto' is not really buying you anything here. You have a specific width given (250), so if you leave off both left and right it will just get centered horizontally, even when you change orientation. If that is what you want, just drop the right property.

    Fourth, you don't say what error you get. One possibility is it complaining that Ti.UI.createButton() is undefined. If that is the error you got, just force a full rebuild by using menu Project > Clean followed by trying the project again.

    This is a concept worth remembering when using Titanium: when you get an "undefined" error, verify you spelled it right (e.g. createButton and not Createbutton). Then use menu Project > Clean and try again.

    The reason this happens is Titanium optimizes the build by only including the modules you reference. But then to save time during debug/test cycles, it only looks to see if you need a new module when you do a "full rebuild".

    So if you started without any button listed, the Button object code would not have been included in the debug build. Then when you added the lines which require the button module, without a full rebuild the module would be undefined. Forcing a full rebuild will let it realize the button module is needed, and no more "undefined" error.

    If this does not clear it up for you, please be more specific about what platform you are using, what SDK level you are trying, and exactly what error you are getting.
