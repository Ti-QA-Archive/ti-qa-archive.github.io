    • Just so I understand, you are running Mac OS X 10.7.4 (Mountain Lion) in a virtual machine on a Windows host? What VM software? What version of Windows? How much RAM did you give the VM? Maybe I can replicate your environment.

      I noticed you are using the old Titanium CLI (3.0.23) and old Titanium SDK (3.0.0.GA). The latest CLI is 3.0.24 and the latest SDK is 3.0.2.GA. TONS of bugs were fixed. You can install these really fast and easy by running:

      sudo npm install -g titanium

titanium sdk install --default

      Update the <sdk-version> in your project's tiapp.xml to "3.0.2.GA".

      You have Xcode 4.5.2 installed and it has the iOS SDK installed which is good! I'm not quite sure why it's the "GMSeed". Your version is build 4G2007a and the final 4.5.2 release is 4G2008a. I'm not sure what the difference is. You're safe to update the official release.

      I don't think it's a problem for simulator builds, but you may want to download and install the Apple WWDR cert from here: http://developer.apple.com/certificationauthority/AppleWWDRCA.cer. Download and double click to install.

      xcode-select output looks good.

      When you run ti build -p ios --log-level trace, you need to be in your project's directory. After it builds, it will launch the iOS Simulator. If it still errors, then before closing the simulator, click the "iOS Simulator" menu and select "Reset Contents and Settings". Back at the command line, run ti clean and then run that build command again. Let me know what the few lines say starting with "Removing old log file" and ending at "Found iPhone Simulator log file".
