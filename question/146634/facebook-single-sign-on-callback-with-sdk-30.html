Facebook single sign on callback with SDK 3.0

I'm currently working on a project where a user signs in with his or hers facebook account using the facebook functions built into Titanium. I wanted to use the single sign on (SSO) functionality, and added the ti.facebook.appid property to my tiapp.xml.

When I tried it, facebook tried to send us back to the app after i accepted but it generated a "Safari cannot open the page because the address is invalid" message. When I checked out the info.plist that had been generated, it said the following under CFBundleURLSchemes

<string>fb[object Object]</string>

What it should say is something like:

<string>fb495338853813822</string>

Any idea why it would spit out [object Object] instead of the string?

This only seems to occur when building the project with SDK 3.0. In SDK 2.1.4 this works just fine.

P.S. One can also solve the issue by using your own info.plist instead of the tiapp.xml, just like Kitchensink does. Still, it should work from tiapp.xml as well… ( source: old article on this topic by appcelerator)

Specs: I am developing on OS X 10.8.2 on a retina MacBook Pro for iOS using the latest XCode and Ti Studio.
