  • Accepted Answer

    Just gone through the process and it's all pretty straightforward.

    Simply uninstall your old version of XCode by running:

    sudo [xcode-path]&#x2F;Library&#x2F;uninstall-devtools --mode=all

    Then go the Mac App store, download the latest version of Xcode which is free and version 4.1

    Install that and voila, everything works perfectly with the latest version of Titanium Studio and Titanium Mobile 1.7.1+ (Lion support was added in 1.7.1)

  • i have lion gm working here on both iOS4.3 SDK and android with xcode 4.1gm/xcode4.2b3. i had also iOS5, itunes 10.5b3 and xcode 4.2 running without any problems when compiling things

    please note, that even if you are able to run code on iOS5 simulator and on the device (you will also be able to compile for 4.3 using sdk 5.0) you can't use xcode 4.2 for validating and submitting an app (also doesn't work when choosing sdk 4.3 in titanium studio when creating a distribution package), apple doesn't seem to accept apps signed with xcode 4.2 at the moment. you will have to remove itunes 10.5 beta and xcode 4.2, and believe me, this isn't a procedure you want to run through

    i also tried to use snow leopard and xcode 4.1 in an virtual box image, and couldn't compile an app there, however this could be related to having not enough memory inside the virtual machine, or not having waited long enough for the code to have finished compiling, xcode seemed to hang with 50% cpu-usage for half an hour when i finally stopped trying ;-)
