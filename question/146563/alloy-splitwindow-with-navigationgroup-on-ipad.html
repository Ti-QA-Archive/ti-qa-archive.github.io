Alloy SplitWindow with NavigationGroup on iPad

Hi,

I'm trying to get a master/detail view working in Alloy on an iPad. The split window works but it doesn't have navigation groups (in simulator or on the device). From the docs it looks like the navigation groups are added to both master and detail windows automatically.

<Alloy>
    <SplitWindow id="index" formFactor="tablet" platform="ios">
        <Require src="master" id="master"/>
        <Require src="detail" id="detail"/>
    </SplitWindow>
</Alloy>

Master is a table view and detail is just a few labels. (basically a direct copy from https://github.com/appcelerator/alloy/tree/master/test/apps/advanced/master_detail).

I can't see how I can add the navigation group to the xml definition.

I've tried this on Titanium SDK 2.1.5 and 3.0.0.GA with the same results.

    • The github example provides a split window but it doesn't cater for navigation groups in the master and detail views. My aim is simply to have the navigation groups visible in both master and detail and to be able to handle rotation to show / hide the button in the detail navigation bar which presents the master view in a popover (when in portrait orientation).

      The problem is that the markup for split window can only accept windows for the master and detail sections. Navigation groups are views, not windows, so you have to wrap them in windows (as seen above) go get valid markup. That means that when you try to use the navigation button for showing the master view in a popover it doesn't work (it doesn't get added to the screen).

      You may be able to add the navigation button to the container window and have it show up and work but that's messy and would require knowledge of the hack in all views that are pushed onto the navigation stack.
