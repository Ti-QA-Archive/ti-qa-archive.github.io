Suspected bug with orientation change when using percentage width/height

Only been using Titanium for a couple of weeks, very impressed so far, but I think i've discovered a bug in the orientation handling when switching between landscape and portrait. Only tested this on Android but my app does NOT correctly layout when switching orientations IF i use percentage based widths/heights (which I really want to use).

The problem can be worked-around to a limited extent by using 'right' and 'bottom' properties instead of width/height. But this doesn't work for more complex layouts where width/height are needed. I'm not exactly sure how to log this issue with Appcelerator but I though I'd post it up here first in case anyone's come across the same issue and knows a decent workaround.

The code for a simple demonstration of the problem can be found here (http://pastie.org/2057826). If you try this in the Android emulator and switch orientations you'll see how the 'width/height' tab layout gets messed up, whereas the 'right/bottom' tab is correct in both portrait and landscape.
