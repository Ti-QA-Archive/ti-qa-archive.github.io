How to hide a <TableViewRow> - or alternative for form layout?

Ok, this is (or ought to be) straightforward - unless I am overlooking something (which probably is the case).

I have created a very simple example:

index.xml

    <Alloy>
        <Window>
            <TableView>
                <TableViewSection>
                    <TableViewRow>
                        <Label>Row 1</Label>
                    </TableViewRow>
                    <TableViewRow visible="false">
                        <Label>Row 2</Label>
                    </TableViewRow>
                    <TableViewRow id="row3">
                        <Label>Row 3</Label>
                    </TableViewRow>
                </TableViewSection>
            </TableView>
        </Window>
    </Alloy>

… and index.tss:

    "#row3":{visible:false}

    "Window": {
        top: 60
    }

I would have expected that row 2 and row 3 were NOT displayed. However, all rows display (in the iOS Simulator).

I have read about similar issues in earlier versions of Titanium Studio. The workaround from these 2-4 years old posts were to set the height of the tableviewrow to 1 (as 0 is ignored). However, this still shows part of the label. Then I can hide the label (visible="false" workst for that!) and then I only see a "thin" row. But with the standard view row separators you still see that there is a row….

So, what am I doing wrong here??

I use the TableView layout to create a form for entering data - and some of the rows should only be shown to the user if a switch is set. Is there a better pattern for doing this?

I am on Studio 3.4.1GA, Mac OS X 10.10.1 and using the 8.1 iOS in simulator ;-)

Thank you in advance!

/John

    I had the same problem, I fixed it using the setData() method of table.
    So with your code, put an id for each of the TableViewRow

        <TableView id="table">
                <TableViewSection>
                    <TableViewRow id="row1">
                        <Label>Row 1</Label>
                    </TableViewRow >
                    <TableViewRow id="row2">
                        <Label>Row 2</Label>
                    </TableViewRow>
                    <TableViewRow id="row3">
                        <Label>Row 3</Label>
                    </TableViewRow>
                </TableViewSection>
    </TableView>

    Then put those ids in an array.

    var visibleRows = [$.row1, $.row3];
$.table.setData(visibleRows);

    • This is a very good suggestion. I had come around a similar solution by adding a <View> tag inside the <TableViewRow> (after advice from John Jardin).

      The markup looks like this:

      <TableViewRow>
      <View id="clubRow1">
            <Label id="label_club" />
            <TextField id="club1" class="club" editable="false" onClick="onClubClick"></TextField>
      </View>
</TableViewRow>

      And then to show/hide it I had to set the height (and hasChild attribute) of the row to avoid a lot of "empty" space.

      if(show) {
      if(!obj.visible){
            obj.parent.hasChild = true;
            obj.height = '30dp';
            obj.visible = true;
      }
} else {
      if(obj.visible){
            obj.visible = false;
            obj.parent.hasChild = false;
            obj.height = 0;
      }
}

      I just do a little sanity check to see if the row is already in the state I want (to avoid any flickering).

      As this particular case has 20+ rows (a form with fields) and only a few of these need to be controlled depending on the switch I thought it was a little too much to name all the rows and add them all every time (using setData).

      I guess that I could use insertRow/deleteRow to just work on the handful of rows I need to hide/show.. - following the principles in your reply. That would make a cleaner approach without having to fiddle with row heights and hasChild attribute.

      I think I'll give that a go :-)
