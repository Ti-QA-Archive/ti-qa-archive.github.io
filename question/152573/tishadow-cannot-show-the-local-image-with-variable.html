TiShadow cannot show the local image with variable.

I need to dynamic to show the view's(or button) backgroundImage, so I use a variable to set it, for example:

var vbg = '/images/' +  Alloy.Globals.CurrentLang + '/btn-submit.png';
$.btnSubmit.backgroundImage = vbg;
Ti.API.info($.btnSubmit.backgroundImage);

when I print the button's backgroundImage, the result is:

file://localhost/var/mobile/Applications/81D4BE05-3594-4234-A6D1-4DF75AF86C64/Documents/MyApp/file://localhost/var/mobile/Applications/81D4BE05-3594-4234-A6D1-4DF75AF86C64/Documents//MyApp//images/en/btn-submit.png

I don't know why TiShadow will add a duplicate file:&#x2F;&#x2F;localhost... path in it, but if I don't add the variable it's work, like following:

$.btnSubmit.backgroundImage =  '/images/en/btn-submit.png';
Ti.API.info($.btnSubmit.backgroundImage);

in this way, the backgroundImage result is correct:

file://localhost/var/mobile/Applications/81D4BE05-3594-4234-A6D1-4DF75AF86C64/Documents//MyApp//images/en/btn-submit.png

another issue is cannot show the image in my widget, I use WPATH(&#39;bg-tab.png&#39;) to set the image in widget.

Is this a bug with TiShadow?

  • TiShadow is in fact an app running multiple apps within. Each app you run over TiShadow, gets it's own folder within the TiShadow app. To make sure references like image = &#39;&#x2F;images&#x2F;image.png&#39; end up in the right place, it prepends the path dedicated to that specific app you run to all strings that begin with a slash.

    To workaround this, you could do:

    $.btnSubmit.backgroundImage =  'TS/images/en/btn-submit.png'.substring(2);

    I'm sure David will find a better way of handling these replacements :)
