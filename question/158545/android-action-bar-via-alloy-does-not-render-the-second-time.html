[Android] Action Bar via Alloy does not render the second time

I've got the Action Bar Up navigation and static menus working together via Alloy:

// page2.js
function OnPrevWindow(e){
    $.page2.close();
    $.page2 = null;
}

function OnReady(e){
    var activity = $.page2.getActivity();
    var ActionBar = activity.actionBar;
    ActionBar.title = "I'm in page 2";
    ActionBar.displayHomeAsUp = true;
    ActionBar.onHomeIconItemSelected = function(){
        alert("I'm home!");
    };        
    activity.invalidateOptionsMenu();
}

// page2.xml
<Alloy>
    <Window class="container" fullscreen="true" onOpen="OnReady">
         <Menu id="menu" platform="android">
            <MenuItem id="menuItem" title="Save" icon="Ti.Android.R.drawable.ic_menu_save" onClick="OnButton1" showAsAction="Ti.Android.SHOW_AS_ACTION_ALWAYS" />
            <MenuItem id="menuItem" title="Image" icon="Ti.Android.R.drawable.ic_menu_gallery" onClick="OnButton2" showAsAction="Ti.Android.SHOW_AS_ACTION_ALWAYS" />
            <MenuItem id="menuItem" title="About" onClick="OnButton1" showAsAction="Ti.Android.SHOW_AS_ACTION_NEVER" />
            <MenuItem id="menuItem" title="Prev Page" onClick="OnPrevWindow" showAsAction="Ti.Android.SHOW_AS_ACTION_NEVER" />
        </Menu>
        <Label id="label" onClick="OnButton1">You're in page 2'</Label>
    </Window>
</Alloy>

With this code I can traverse from the first page to this page (page2.js) and back. But the second time I traverse to page2.js, the action bar does not render: I get only the app icon. No title text, no menus, no up navigation.

I tried to follow Richardo's helper class but that didn't seem to do much either.

Currently I'm on Ti 3.1.2. I can't move forward to 3.1.3 because I'm on Windows 8.

Ideally I'd like to use the menu item description in Alloy if that's possible. I realise that dynamically creating the menu items in code seems to make the menu items very unresponsive. But that's another problem for another day…

Any insight will be appreciated.
