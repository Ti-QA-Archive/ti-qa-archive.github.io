Calling web service method, from my Titanium Mobile application

Hi all.

Im trying to do a very simple task. (at least i think it is very simple)
I have created a normal asp.net webservice using Visual Studio on a windows machine.
The webserice contains 1 method which takes an integer as argument, and returns the number multiplied by itself. Looking like this:

[WebMethod]
public int Multiply(int number)
{
return number * number;
}

I have also created a simple iPhone/Android application using Titanium Mobile. For this purpose im using a mac running OS X version 10.7.3.
The Titanium SDK is version 1.8.2.

Now all i want to do is to call this "Multiply" function from my app.

So my questions is:

  1. Can i take my webservice and copy it to my Mac computer and having it running locally on this (similarly to IIS on windows) ? Or does it need to be available online on the web in order for me to contact it ?

  2. How do I invoke this webservice method from my Titanium app?

Hope someone out there can help me, as I am quite new to this (obviously)

  • It's been a while since I've messed with ASP.NET web services, but it looks like there are some techniques you can use to access them from client-side Javascript. See this example and this one.

    It looks like you have to set some attributes of your web service to allow JSON-serialized data (I think the default for ASP.NET web services is SOAP), and there are some special considerations in the way you make the HTTP requests. Those articles have a lot of detail.

    Note that if you're not using jquery in your app (and I don't recommend bringing such a large framework into your Titanium app), you'll have to use the HTTPClient to make the requests (as opposed to the $.ajax method used in the examples).

    As to running your web service on your Mac, the easiest thing to do would be to run a virtual windows machine with IIS. I can't imagine how you would possibly get .NET web service code to run on a Mac, unless you used some sort of Mono-based solution, which sounds like going down a rabbit hole.

    Good luck – you'll have a fair amount of learning to do, but once you get the pieces in place to make the first web service call, everything will be easier from that point forward.
