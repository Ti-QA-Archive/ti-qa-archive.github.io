Alloy Models and Sync Adapters and Using Set vs Save

Maybe someone can clear this up for me. What I would like to do is update a model and have only the attributes that get changed updated on my server, I don't want to send all the attributes across. I have a model and a custom sync adapter.

lets say I declare my model like so:

Alloy.Models.instance("book");
Alloy.Models.book.fetch();

Which successfully goes to my server and retrieves the single book information for me.

Now I need to update that book. (here is where I am confused)

If I update the book title like so:

Alloy.Models.book.set({"title": "title_updated"});

This changes the book models title locally but does not sync it to the server at this point. My understanding is that I need to call:

Alloy.Models.book.save();

But in my SYNC adapter how do I get only the attributes that changed? I have tried using backbone.js calls such as:

model.changedAttributes();

But it returns false. Does the model keep track of changed attributes when SET is used?

The only way I can get it to sync to my server is by calling.

Alloy.Models.book.save({"title": "title_updated"});

Which in my sync adapter I can update using the attributes being passed in. This works but I would like to SET a few different attributes and a little later on do a save to have it sync only the changes.

Hopefully this makes sense, and maybe someone can point out how I can get only the attributes that have been changed via the SET command once I try to SAVE.
