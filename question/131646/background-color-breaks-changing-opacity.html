Background Color breaks changing opacity

It appears that adding the backgroundColor property to a button prevents the ability to dynamically change the opacity. Or rather, changing the opacity resets the opacity to a value of 1. I am using Android and Titanium SDK 1.8.1. The code below highlights the problem. Removing and then adding the button again to the window does seem to be a workaround, but should it really be necessary?

var win = Titanium.UI.createWindow({  
    backgroundColor:'white'
});

var btnTest = Titanium.UI.createButton({
    height: 100,
    width: 100,
    backgroundColor: '#005288',
    opacity: 0.50
})

btnTest.addEventListener('click', function(){
    //win.remove(btnTest);
    btnTest.opacity = 0.25;
    //win.add(btnTest);
})

win.add(btnTest);
win.open();
