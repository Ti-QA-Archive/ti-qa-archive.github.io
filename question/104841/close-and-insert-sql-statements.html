close() and Insert SQL statements

So I've spent most of the afternoon trying to track this issue down. I am having seriously weird SQL behavior that is causing my app to stop processing. When I enable Trace as the filter in Ti Developer, I notice the following error:

[TRACE] W/SQLiteCompiledSql( 1517): Releasing statement in a finalizer. Please ensure that you explicitly call close() on your cursor: INSERT INTO node (nid, vid, type, title, changed, data) VALUES (?, ?, ?, ?, ?, ?)
[TRACE] W/SQLiteCompiledSql( 1517): android.database.sqlite.DatabaseObjectNotClosedException: Application did not close the cursor or database object that was opened here

Followed by a huge stack dump.

But that makes no sense whatsoever.

1) A select query should return a result set, and I can see the need to close() that result set but realistically it should self-close when the variable goes out of scope. So far I have been unable to determine if that's even the case.

2) An insert statement does not return a result set, and I've confirmed that the return value from execute() on an insert statement is null. So there's nothing for me to close(), even if Ti would let me!

I am highly confused, but this is blocking further development on my app. Any idea what is going on here?

If it matters, I'm using Ti 1.5.1 in the Android emulator with Android API 2.2.
