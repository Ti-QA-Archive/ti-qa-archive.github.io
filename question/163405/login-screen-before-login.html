Login Screen before Login

Hey Guys, i'm a newb for Titanium Alloy but a veteran in Rails.

I'm building a mobile app and here is the logic that i'm looking for and am struggling with how to get this implemented.

  1. If the user is not logged in display a login screen that will communicate with my site to authenticate
  2. Else display the contents of the application in a tab group

  • Because your app will be in a tab group, you'll need to launch your login window as either a modal atop the tab group, or open it as an either/or sort of window. Like this:

    <!-- index.xml -->
<Alloy/>

    // index.js
if(isLoggedIn) {
  var main = Alloy.createController('maintabgroup');
  main.getView().open();
} else {
  var login = Alloy.createController('loginwindow');
  login.getView().open();
}

    Then, in loginwindow, you'd open the main tab group controller and close the login window controller on a successful login.

  • If this will be iOS only, you can also add/remove tabs as needed. So could present a tab with your login/register tabs, upon successful login, add some new ones, remove the old.

    This does NOT work (the removing part) for android as android does its tab layout VERY early on in its bootstrap.

    If you take Tim's approach, and open a modal over over the tab group, be sure to set the exitOnClose property or they can simply close the window and be at the tabgroup.
