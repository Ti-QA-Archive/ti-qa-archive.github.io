Android Hardware Back Button

Hi…!!!
I am working on app which has lots of navigation from one window to another and that too in Tabs.. What I am doing is on 1 tab I am opening a window which has got a tableview inside of it.. And on clicking of a row in that tableview we are closing our window 1 and opening another window which is window 2.. The problem here is that "Android Hardware Back Button" seems to struggle with tabs.. I want to add an event listener to window 2 so that we can go back to our window 1 and then from window 1 after navigation from window 2 I want to close the app..

Please can any one help me in that.. I'll post the code if needed…!!!

  • You shouldn't have to explicitly manage this sort of thing. If you open windows via the Tab's open() function, the back button will do exactly what you expect - it will close child windows, and if you hit it while on the main tabgroup window, it will close the application.

    • Not if you're using anything greater than the 2.15 SDK. With newer ones, the Titanium.UI.currentTab.open(window) is broken for Android from windows that are setup PAST the ones that are initialized in app.js.

    • I don't use Ti.UI.currentTab – I don't think that's been reliable for some time. You have to track the current tab with your own code (you can set a "containingTab" property on each window you add to the TabGroup, or you can use some more elaborate mechanism)

      I'm not 100% sure I understand what you mean by "windows set up after the ones initialized in app.js". Titanium on android doesn't support manipulation of the tabgroup after it's created. You can wholesale replace the tabgroup, but you can't modify it.
