Titanium vs Java for Android

OK so I have given Titanium a 3-4 day evaluation and really like the concept because we are a small shop and need to support both iOS and Android and our app is a fairly basic data-driven interface.

However, I have noticed in this short time that I am having a significant amount of trouble getting things to work on Android. That includes features not being supported (Toolbar, NavigationGroup, etc) to more severe things like events not firing properly. All of these things work great on the iPhone, but I am fighting the Android build every step of the way with poor results.

We do have in-house Java skills but of course it's preferable to have a single codebase for all mobile devices. I'm just wondering if this is still a fantasy?

I'd probably at this point considering using Titanium for our iPhone app and just going with Java for Android. I really don't want to do this but nor do I want to spend a ton of time learning all the Android/Titanium glitches if the product is going to wind up being poor quality.

Any thoughts or opinions would be appreciated. Thanks!

    • Yes, currently android support lags behind iOS, but the gap has narrowed considerably. Prior to mobile SDK 1.5.x, the difference was more pronounced. So they are moving in the right direction.

      I suspect there will always be numerous differences, because the native UI components available are not the same in each OS.

      That said, my opinion is that Titanium is still sufficient for many types of cross-platforms app, even when building an Android version. However, I suppose it will depend on what you are trying to achieve.
