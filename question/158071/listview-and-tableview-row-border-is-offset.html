listview and tableview Row border is offset

With ios7 and Ti 3.1.3 GA, I am noticing that the ListView and TableView row border is offset by a certain amount on the left. It appears as if the row border color starts at offset ~20px from the left of the table. This happens even with KitchenSink sample code for ListView and TableView. Any reason why this is the case?

How can the row border color extend through the entire width of ListView or TableView (starting from offset 0px from the left)?

Thanks.

    it's the default setup in ios 7, there is a property which is not exposed in Titanium to configure the Inset.

    refer to UITableView Class reference

    separatorInset

    Specifies the default inset of cell separators.

    @property (nonatomic) UIEdgeInsets separatorInset

    Discussion

    In iOS 7 and later, cell separators do not extend all the way to the edge of the table view. This property sets t>he default inset for all cells in the table, much like rowHeight sets the default height for cells. It is also used formanaging the “extra” separators drawn at the bottom of plain style tables.

    For example, to specify a table view where the default left separator inset is 3 points and the default right separator inset is 11, you would write:

    tableView.separatorInset = UIEdgeInsetsMake(0, 3, 0, 11);

    Special Considerations

    Only left and right insets are honored.

    Availability

    Available in iOS 7.0 and later.
