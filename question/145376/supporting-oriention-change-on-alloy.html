Supporting Oriention change on Alloy

I have noticed that Alloy just copy the styles into any created views, however, if I need to change the view's style after an orientation update event, I don't have any way to reach these styles. Is there any way to support it withing the styles folder without having to create the separate styles on the controller with Alloy?

    Supporting dynamic style changes and application at runtime is definitely a priority for Alloy. We have a (vague) ticket here for queueing it up: https://jira.appcelerator.org/browse/ALOY-210

    This will become top priority once Titanium 3.0 is officially released. We could implement before 3.0, but it can be done much more efficiently with the new applyProperties API that was added in 3.0.
