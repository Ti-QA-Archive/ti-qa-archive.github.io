    Hi, that console message is a just a warning, and it is likely not causing the app to stop at splash screen. Is it possible you are just not opening the index window with $.index.open() in your index controller?

    Also, if you could paste code snippets of your model/view/controller it would be easier to spot the cause of the behavior you are describing (to properly display code in your questions/comments/replies remember to place it in between two distinct lines of ~~~ , as the Markdown Syntax here adopted suggests)

    • I am just following tutorial at:
      http://docs.appcelerator.com/titanium/latest/#!/guide/Creating_Your_First_Titanium_App

      I created a Model with name "books". It (books.js) had following code:

      exports.definition = {
      config: {
      columns: {

              "title": "text",
        "author": "text"
    },
    adapter: {
        type: "sql",
        collection_name: "books"
    }
},
extendModel: function(Model) {
    _.extend(Model.prototype, {
        // extended functions and properties go here
    });

    return Model;
},
extendCollection: function(Collection) {
    _.extend(Collection.prototype, {
        // extended functions and properties go here
    });

    return Collection;
}

      };

      Controllers/index.js had following code:

      var myBooks = Alloy.Collections.books;

      var book = Alloy.createModel('books', {
      title : 'Great Expectations',
      author: 'Charles Dickens'
      });

      myBooks.add(book);
      book.save();

      And in views/index.xml, I copied following code:

      <Alloy>
      <Collection src="books"/>
      <Window class="container">
      <TableView dataCollection="books">
      <TableViewRow title="{title}" author="{author}"></TableViewRow>
      </TableView>
      </Window>
      </Alloy>

      That was fairly simple. But it did not allow my sample app from tutorial to move forward
