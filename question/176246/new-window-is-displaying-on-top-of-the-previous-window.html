New window is displaying on top of the previous window

I’ve just started to learn Titanium and have encountered an issue when I try and open a new window from index.js the new window is drawn on top of the index.xml tablerview. My index.xml contains a tableview, when a certain row is clicked I have an onClick event. I’m using the following code in index.js to navigate the the window1 view which contains the new window:

index.js

function openWindow(e){
            window1 = Alloy.createController('window1’);
}

The window1 view contains the following code:

<Alloy>
  <Window id="child_window">
    <Label>This is the new window</Label>
  </Window>
</Alloy>

However when I window1 is displayed the label is shown on top of the previous windows tableview. Do I need to hide or close the index.xml tableview?

    Hi

    Unfortunately you have not provided the code that actually tells the window to display. This means I have to guess.

    The most likely situation is the following;

    • your app is iOS only, android new windows always appear over the top.
    • you sound like you expected a slide animation and a back button to appear.
    • instead the window just appeared and the label appears to be part of the previous window.

    If this is the case we have a few simple things to fix;

    • use the NavigationWindow Alloy markup in your index
    • the child example in your window1
    • the use of the method openWindow on the NavigationWindow opens the child window into the a NavigationWindow and not over the top of it.

    You will almost certainly not have provided a white (or any other colour) setting to the backgroundColor property. So it is currently using the default which is transparent. This is why it appears to have the label over the tableView.

    If you do not need a NavigationWindow then use the Open method on the child which you must be doing now and just set the backgroundColor to white.
