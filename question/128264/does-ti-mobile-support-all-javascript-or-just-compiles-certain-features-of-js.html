Does Ti Mobile support all javascript, or just compiles certain features of JS?

I am still getting my head around how Titanium uses javascript to compile into native code. One thing i am not sure about at this point, is whether i can actually use virtually any javascript code i want, or are only certain parts of javascript incorporated into the compiler?

There are certain ways of doing things with Titanium Mobile such as setting up views and transitions etc using the Titanium API, and i understand that this code is compiled into native code, but if this code is turned into native code, then how are pure javascript functions and code run inside a native app?

For example, if i create an app using the API to create the views and events etc, are my functions to manipulate data, which are written in pure javascript, going to work? Or do i have to use the API functions only to perform calculations etcetera?

Please let me know if you need me to elaborate.

  • The simple answer is yes. The core javascript language is available in Titanium so calculations should all work, and things like date functions etc. There is even some of the commonJS module available.

    The difference is that Titanium uses a set of API's to access data and device sensors etc, it also has a built in set of components for the UI.

    So although the base code is JavaScript you the uses the Titanium API's and components to create your app.

  • "how are pure javascript functions and code run inside a native app?"

    By using a javascript interpreter as it runs. :) It is a common misconception that Titanium apps have the js converted to completely native code and the js no longer exists. That isn't true, and is the reason implementing the V8 engine will benefit apps running on android.

    You can use pretty much any pure javascript you want. What you can't use are things which rely on browser provided things like DOM. So js libraries which utilize DOM are not compatible. Pure javascript is.

    I suggest you read this and this for a better understanding on how Titanium works.

  • You can write pure javascript to do pretty much anything in your app. You can use other libraries such as CoffeeScript and Underscore. The only significant difference between Titanium and your previous experience with javascript is that the client is different. You won't want to use jQuery or other libraries that are specific to a web browser (although Titanium does support using web views if necessary). The purpose of Titanium (in my amateur opinion) is to allow you to interact with native UI components and device features that you can't access using a browser.
