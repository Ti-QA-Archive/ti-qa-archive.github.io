  • Hi

    First of all, I noticed that you confused the heavyweight / lightweight issue. Setting navBarHidden makes a window heavyweight, not the other way round ;)

    So, by default, every heavyweight window gets closed by the back button and you get back to the previous heavyweight window. If your first window (the index window) is also heavyweight, then you should give it an exitOnClose: true to prevent getting back to the "splash screen" (the root activity).

    If you're using a custom navigation controller with lightweight windows, then you should store your navigation history in your own back stack (LIFO) and override the back event to close those windows manually.

    I recently came across a similar issue finding a solution for resuming behavior. I don't use Ti.App.resume on android. To manage the activity lifecycle correctly I did some tests with activity events (have a look at my question here). To resume on both, iOS and Android, I use this code (note, that my index window is heavyweight):

    if (OS_IOS){
    Ti.App.addEventListener('resume', onResume);
}
if (OS_ANDROID){
    $.index.addEventListener('open', function(){
        $.index.activity.addEventListener('resume', onResume);   
    });
}

    I hope this helps you a little bit. Otherwise, you can always ask if something is not clear!

    Manuel
