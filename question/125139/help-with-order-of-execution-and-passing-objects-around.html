Help with Order of Execution and passing objects around

I am a newbie programmer still struggling somewhat with order of execution. Allow me to explain.

I am writing a mapping application that does several things.
I have been writing the funtionality of the app in modules or functions with the hope that at then end I could put it all together like lego blocks.

My struggle and need for understanding is best illustrated by an example:

So I write my user interface as a function, let's say var drawui = function drawui( filter); I'll include this code in a javascript file say… ui.js.
Nice and simple to edit my UI in one place.
Part of the UI by the way is going to include a mapview.

Now I have two popover controls that are included in the UI as well.
Inside each popover is a tableview list of "actions".

When a user selects an item (action) I want to call a function… say for this example mapPins( filter ). When I call this function I am going to pass a simple string for example mapPins( "all" ) to the function and I want the function to drop the pins and annotations onto the mapview.

The problem I "think" where I am getting lost is the mapview I created is inside another function I used to draw the gui and thereby inaccessible (I think) to me to drop pins on.

See my problem? I'm keep trying to modularize my functions, but I'm not clear on how to access controls(objects) that were created in different functions?

I know that I can pass some things around via function parameters but I'm not sure what if anything this does to memory. For instance if I need to map 2000 pins, in the example above, how would I get the instance of the current onscreen mapview to place them on when I'm running the code inside my mapPin function?

I don't know if this makes sense or not, but I'd appreciate it greatly if anyone could help shed some light here.

Thanks!

  • You can use custom events. Fire an event from any function or any window and put a listener whatever you want, it will run when the event occurs:

    To fire an event:

    Ti.App.fireEvent('action_filter', object)

    To listen the event

    Ti.App.addEventListener('action_filter', function(object){
    Ti.API.info(object.toString());
});

    object is any object you want to pass through the event (for example, to know which pin was clicked).

    Take a look to this link on how to create custom components.

    http://wiki.appcelerator.org/display/guides/Building+Reusable+Factories
