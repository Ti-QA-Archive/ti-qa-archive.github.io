App update in app store.. Erase users custom data?

I'm about to submit an app that relies heavily on custom user data. The user will store custom data in the app. However, when i submit updates to the app at what point will it overwrite the custom data inside of sqlite on the users phone?
Do i need to be aware of some sort of best practice of this? Ie. Never rename a db or something of that matter?

— asked September 22nd 2011 by Anthony james

  • The database on the user's device will not be overwritten. As long as you don't change the database name in the Ti.Database.install function, it will reuse the one already deployed on the device. (even if you include a newer version in the Resources directory)
