  • Accepted Answer

    After encountering some other users that are having the same problem, it turns out the problem is because you probably have unicode characters in your JavaScript and/or i18n strings.xml files.

    We realize this is a huge oversight and this will be fixed once we find a replacement for the Python-based Mako template system.

    In the meantime, you'll have to remove all unicode characters.
