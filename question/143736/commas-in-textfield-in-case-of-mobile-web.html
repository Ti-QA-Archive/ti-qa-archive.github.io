Commas in Textfield in case of mobile web

I am developing a Titanium application which contains a form with text fields and labels and a button to submit the data. In the form there is text field which should accept only numbers. I did that using
keyboardType:KEYBOARD_NUMBER_PAD
It is working fine in all platforms. But in mobile web when this textfield gets focus I am able to input only numbers. But when the focus of this textfield is left special characters like commas are getting inserted between the numbers of the textfield.

For example when textfield gets focus I am entering a number like

123456789

when the textfield lost its focus the number in the textfield looks like

1,234,567,89

Can anyone share the solution why the textfield is behaving like this ASAP. This is the problem in mobile web only.

Thanks in Advance.

  • Hi,

    I'm investigating on the browser behavior, but it looks like a specific behavior of the browser itself and not due to Titanium (I've tried on iOS Safari so far).
    However, this is only an esthetic visualization. The textField value it's exactly the number without any special character or punctuation (only numbers).
    You can check this by running this simple code:

    var win = Ti.UI.createWindow({
});

var tf = Ti.UI.createTextField({
    top:2,
    width : '80%',
    height : 50,
    borderStyle : Ti.UI.INPUT_BORDERSTYLE_ROUNDED,
    keyboardType : Ti.UI.KEYBOARD_NUMBER_PAD

});

var label = Ti.UI.createLabel({
    top:120,
    width : '80%',
    height : 50,
    borderColor:'white',
    text:'Click on the button to display TextField value',
    color:'white',
});

var button = Ti.UI.createButton({
    top:220,
    title:'click',
    width:100,
    height:50,
});

button.addEventListener('click', function(){
    label.text=tf.value;
});

win.add(tf);
win.add(label);
win.add(button);

win.open();

    Is this helping?

    Thanks,
    Federico
