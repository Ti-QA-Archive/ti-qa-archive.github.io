New page on clicking an annotation

Right now I'm displaying certain annotations on a map. I want to be able to go to a new page on clicking an annotation. This page would contain information about the annotation.
I tried adding an eventListener to the mapview, just to start off things : 

//create the mapview and the annotations first
//now add eventlistener
mapview.addEventListener("click", function()
    {
        Ti.API.info("map clicked!")

    })
//now add mapview to win and open win

However it never seems to display anything on clicking the map, or any of the annotations.

I'm basically going in the direction of checking the id in the event listener of mapview, and then maybe going into another window. Is this approach a good one?

I'm using android 2.1 and mobile SDK 1.7.0RC1

    A longer winded example (for the sake of yourself and future readers of this question)
    you need to use e.annotation.myid.

    This is because each event that you fire has event properties.

    each event listed in the API docs has the available properties the event gives you access to.

    For example, the map gives you access to:

    annotation - the announce source object
clicksource - the source of either annotation, leftButton or rightButton to indicate to location of the click
globalPoint - a dictionary with properties x and y describing the point of the event in screen coordinates
index - the annotation index
map - the map view instance
source - the source object that fired the event
title - the annotation title
type - the name of the event fired
x - the x point of the event in receiving view coordinates
y - the y point of the event, in receiving view coordinates

    Each event has its own properties available to it, so you will have to look to the API docs for each elements event
