Why does Titanium Studio pause the debugger on *CAUGHT* exceptions?

Hi!

I'm using a library called "date.js" in my project, which internally throws and catches a ton of exceptions on startup. I'd like to reiterate that none of these are uncaught exceptions… for whatever reason, their framework bootstraps itself this way.

The issue is that when I run my project in debug mode, Ti Studio pauses the debugger at the point where each exception is thrown, even if there is a catch handler for it.

My options are:
1) Click "resume" 50 times on startup
2) Disable "pause on exceptions/errors" in the Ti Studio preferences

With option 1, it takes me a few minutes each time I start the debugger.
With option 2, I miss other uncaught exceptions in my own app code

Both options make debugging much harder. It would be great to have the option to pause only on uncaught exceptions. In the meantime, has anyone found a workaround for this?

Thanks!
Dave

  • For anyone else hitting this issue in the future, you can reduce the number of debugger pauses from 50 to 1 by using:

    require('src/date');

    instead of:

    Titanium.include('/src/date.js');

    For whatever reason, "require" seems to eat the exceptions and just pause once.
