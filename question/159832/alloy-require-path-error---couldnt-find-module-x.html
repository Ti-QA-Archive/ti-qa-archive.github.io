Alloy require path error - "Couldn't find module: X"

Hi all,
I have a file I need to require in my controllers. The absolute path is app/lib/Piwik/Tracker.js. The require statement is this var tracker = require(&#39;Piwik&#x2F;Tracker&#39;); But when I try to run the app with that require statement, I get the error "Couldn't find module: Piwik". I don't know what I'm doing wrong here. The folder and file exist as stated in the /lib folder. Another file I use in /lib causes no problems. I've tried /Piwik/Tracker and ../Piwik/Tracker.     Hi

    As Alloy is a pre-processor of file into resources before the actual compile starts it is worth checking your resources folder once the app has launched to check where the file has ended up.

    If it does not appear in…

    {appname}/resources/Piwik/Tracker.js

    …do a search to see if it made it through somewhere else in resources.

    Other things

    • personally I prefer to lowercase all files as case sensitivity can cause unexpected issues
    • check you do not have another file/folder combination that could overwrite the the file
    • does the CommonJS file actually work? If not it may have been excluded
    • does the file appear on the pre-processing list that is performed in console?
    • try a very simple CommonJS module with the same name to confirm the process works.

    If all else fails; close the simulator, emulator or device, cancel the current build, clean the project AND empty the resources folder contents. This will make sure no pre-existing files that were in different folders are causing conflicts or read only rights to files. The resources folder will recreate on next build.
