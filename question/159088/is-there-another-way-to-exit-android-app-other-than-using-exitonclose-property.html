Is there another way to exit Android App other than using ExitOnClose property?

My app will open a login page first and than open a TabGroup after login.
What I want is to quit the App if the Android back button is pressed when the tabGroup is opened. So far, using the ExitOnClose will do the job.

However, my app has a Logout option. If logout, TabGroup.close() will be used to close the tabGroup and show the Login page again. So in this case, if I use the ExitOnClose:true, the app will quit when the TabGroup is being closed.

Is there another way to exit Android App other than using ExitOnClose property?
So that I can differentiate the close of tabGroup is initiated by the Android back button or by the program code tabGroup.close(); for different handling.

Besides, I have tried using Ti.Android.getCurrentAcivity().finish(); but no luck. It said Ti.Android.getCurrentAcivity() is undefined.

Appreciate any suggestion or advise.

    • Yes, just tried it and works fine (in the emulator)

      app.js

      var tools = require('bencoding.android.tools');

var platformTools = tools.createPlatform();

var w = Ti.UI.createWindow();

var b = Ti.UI.createButton({
    title : 'Click me',
    width : 200,
    height : 50
});

b.addEventListener('click', function(e) {
    platformTools.exitApp();
});

w.add(b);
w.open();

  • Not sure if I'm stealing other people's answers here but I've just come across the same scenario with a login screen and I've tried a combination of the snippets above, and this works okay for me:

        var activity = Ti.Android.currentActivity;
        activity.finish();
