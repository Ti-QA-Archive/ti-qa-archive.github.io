The naming rule for App ID?

I am using Titanium Studio 1.0.5 with Mobile SDK 1.7.2 and Xcode 4.0.2. When I created a Titanium Mobile project named "LearningTitanium" with App Id of "com.example.LearningTitanium", the build process will complain that:

[ERROR] xcodebuild: error: The project 'LearningTitanium.xcodeproj' does not contain a target named 'LearningTitanium-universal'.

I then checked the LearningTitanium.xcodeproj under build/iphone and found that there are three targets in the project file, and they are LearningLearningTitanium, LearningLearningTitanium-ipad, and LearningLearningTitanium-universal. No wonder the build process failed!

The same problem will occur if I chose other project name/App Id like "ExperimentingTitanium" or "HelloTitanium". However, the build process will succeed for project name/App Id like "Learning", "Hello", or "Experiment".

Why the choice of project name/App Id matters? Are there specific naming rules for project name/App Id?

    • Based on your comment, I tried a few experiments and found that indeed "Titanium" can not be any part of a project name, otherwise the build process will throw out errors and refuse to run.

      For example, if I name my project "Learning-Titanium", "TitaniumSandbox", or "MyTitaniumApp, they will all fail to build & run. Having "Titanium" in part of App Id is OK (e.g. com.example.LearningTitanium is all right), but it matters to project name.
