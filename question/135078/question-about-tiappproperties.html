question about Ti.App.Properties

Hello,

I have a scenario like this , When i run the app for the first time and click on the Button A , a view appears with Button B . If i click on Button B, then view is hiding . Then if i click on Button A again the view should not come . So thats why i want to use Ti.App.Properties but seems that not working . It works for the one session but when i restart the app its again showing the view on click of the button A, which it should not be.
My logic is as following .

I think bcoz i m always initializing myBool as true in the beginning thats why!!!!!!!

Ti.App.Properties.setBool('myBool',true);
buttonA.addEventListener('click',function(){

if(Ti.App.Properties.getBool('myBool')==true){
//Show view function called
}
else {

}
});


buttonB.addEventListener('click',function(){
Ti.App.Properties.setBool('myBool',false);
//Hide view
});

    • Ti.App.Properties.getBool(&#39;myBool&#39;, true) will return true if property myBool does not exist. So it'll show the view first time.. After buttonB is clicked, myBool is set to false and the view wont show. It'll return false even when you restart the app.
