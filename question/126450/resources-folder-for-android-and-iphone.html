  • You guys are way overcomplicating this. Bastien, instead of putting your images in Resources/iPhone and Resources/android, put them here:

    • Resources/iphone/images/
    • Resources/android/images/

    Notice that the iphone directory is lowercase.

    Now, in Resources/iphone/images/, you'll want to have 2 files for each image: (1) for iPhone 3 class devices, a file like MyGlyph.png; (2) for iPhone 4 (retina) class devices, a file like MyGlyph@2x.png which is twice as big as MyGlyph.png. But you just reference these images as &#x2F;images&#x2F;MyGlyph.png in your code.

    In Resources/android/images/, you need to have 3 subdirectories (if you want to support all 3 current screen densities):

    • Resources/android/images/low/
    • Resources/android/images/medium/
    • Resources/android/images/high/

    You drop files named MyGlyph.png in each of these directories, but each contains a different size image. If you consider the "medium" image to be your base or standard size, then the "low" version of the image should be 75% of that size, and the "high" version should be 150% of that (medium) size.

    After doing all of this, if your images don't show up in either the iPhone simulator or the iOS device itself, use a more current 1.8.0.v.xxxxx CI build of the SDK, which fixes this bug (the 1.7.3.v.xxxx builds may also fix this, but I'm using the 1.8s).
