Planning a login-script. How to use gobal variables?

Hi guys! I'm currently planning some sort of a authentication-script for my app. Since the app is based from a remote server (getting info from mysql) I figured I'll manage all the authentication the same way.

No problem I thought. But what about storing it inside the app as a global variable? So if the login accepts, could I somehow make for instance the variable "logged_in = true", and use it all over the app?

Because, the way I see it work is that each window with a url entitied to it will not "earn" the variables from the app.js?

In each window-url i include at the beginning some files (settings.js, func_lib.js), and I also do this in the beginning of the app.js.

My question is, are there any way that I can store something like a cookie, otherwise I dont know how to make it possible to "earn" variables?
