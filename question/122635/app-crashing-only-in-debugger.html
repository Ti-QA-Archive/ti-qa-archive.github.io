My app has several UI functions that begin with Ti.API.info(...); or Ti.API.debug(...);. If I run it in the iPhone simulator alone, it does not crash. If I run in the Studio debugger, the app crashes on these Ti.API functions. I don't know how to pause and catch the exception in the debugger, but if I comment out these functions, there is no crash. I have also cleaned my project, and a clean build does not help.

  • To pause on one of the info or debug statements you should double click on the line number in the main page code view (in the left hand side gutter). A blue dot will appear, that is a breakpoint.

    When your app runs in debugger mode, it will stop at the point. You can then use the Resume button (the image icon with a bar and play icon combined (it is next to the pause button) and it will run the app through to the next logical breakpoint. Keep a note of where it is and see which statement is crashing the app.

    Put a few breakpoints in at these info & debug statements - it looks like you are trying to echo out something that can't be calculated. the breakpoints will allow you to find where that is hopefully.
