Custom Default.png is displayed correctly on simulator - but not on iPhone - using Alloy

I am running Titanium Studio (3.0.2) on a Mac and am developing an App using Alloy (2.2.0).
I added custom Default.png (in all those different sizes and names) and everything is displayed correctly in the iPhone simulator.
But when I package the whole thing (via Titanium Studio - Run - "iOS Device") and install it on an iPhone or iPad (via Testflight) the splash screen is the default Appcelerator splash screen.

I have really no clue what I am doing wrong! I read something here hat there was an Appcelerator problem about this which was supposed to be fixed in Version 3.0.2.

    Hi Eric,

    So here's the link to the issue you mentioned: https://jira.appcelerator.org/browse/TIMOB-12652

    It was indeed supposed to be fixed for 3.0.2 GA. We are currently reviewing user reports and if we are able to come up with a further fix, it may be available in the 3.1 release. If you could participate in the issue to help us out with testing, that would be great.

    Before that, please also try the latest 3.1 CI build to see if that helps.

    If you need an emergency workaround, you can replace the default Appcelerator splash screens in the Ti SDK folder at:

    /Library/Application Support/Titanium/mobilesdk/osx/3.0.2.GA/iphone/Resources
