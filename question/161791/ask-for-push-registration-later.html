Ask for push-registration "later"?

Hello, I've created an iPhone app with the latest Ti 3.2 and so on.

What I've learned is that Im supposed to put everything about "push" in the top of app.js, therefor when a new downloader opens the app for the first time, he/she gets prompted with an push-request.

Is it possible to "save" this request, until he/she logs in, in the app, instead? But still maintain the functionality from the push, but just ask for push-request later?

    You can put the push registration wherever you want.
    Let the user do whatever he wants, and when you really need to register the app to receive push messages, start the registration process.

    • "later" you put the whole registrtaion part

       Ti.Network.registerForPushNotifications(....

      You don't need to put the registration code on top of everything, but only when you need it, so basically when your app starts for real (we assume the login part is not the real deal and after login it won't be there anymore)
      So try to find the point where your app starts to a fully working state and put it there.

      Makes more sense?
