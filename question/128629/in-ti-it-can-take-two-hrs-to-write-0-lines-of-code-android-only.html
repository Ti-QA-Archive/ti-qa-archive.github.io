  • Well, after logging into this site, using the only webform in the entire internets that refuses to remember your login, I thought I'd tell you the conclusion of yesterday:

    In order to have a Android Emulator running that is usable, here is what I did:

    In Titaniums 'run configurations' choose a suitable API for your program - Google APIs Android 2.2 seems to work, I had hell with 2.3.3, although that might not have been the API but something else.

    In the Android SDK (android.bat) choose 'manage AVDs' and delete them all.

    Run your program once, but kill the emulator immediately

    In Android SDK edit the new AVD that Ti created:

    SD-Card: size: 512Mib

    Snapshot: Enabled If you run into trouble you'll have to start the AVD from the SDK with the 'wipe user data' option. You can also go with 'disabled' and wait for the boot, you'll only have to wait once

    Skin: HVGA (the higher resolutions runs slower and takes up a lot of screen)

    Hardware: Press New and add 'Device ram size' with a value of 4096 (less would probably do, but I have 16Gigs of the stuff)

    Click 'Edit AVD' - go back to Titanium and run your program, the first run will be slow, but then it should pick up speed.

    I'll now go searching the Ti API using the only searchform on the entire internets that require you to delete some words ('search documentation') before searching …
