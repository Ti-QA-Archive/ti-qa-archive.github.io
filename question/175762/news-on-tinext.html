    • Previous Window support for Titanium has not worked out as well as hoped as far I know.

      Whilst there were ambitions to add support during 2013 and then Q1-Q2 2014 - the task has not been as simple as hoped, partly (guessing some of this) distinct differences on some core navigational elements and beyond.

      However given the new way of thinking about the core engine (which is hyperloop) and how it can now work with SDK APIs much more easily - logically including it into the HL is the smartest plan.

      As I guess you know currently each platform supported has 90% unique and 10% shared effort to include it. HL turns that on its head to more like 95% shared and 5% unique effort. Thus adding platforms 'should' be much easier.

      Obviously the transition will take the existing platforms first and then the new ones. Windows being a new one. But as you can see the beta allows you to build Windows Mobile apps AND windows metro style apps.

      It is also hoped that the iOS version will allow desktop apps in the future as well.

    • This is very difficult to explain exactly how these layers work now and will work in the future as I need hands gesticulating all over the place ;-)

      In short the words you are saying are close to correct but the meanings are not quite right.

      • JavaScript-to-native conversion, rather than a JavaScript Runtime/Bridge/Native

        This actually means the bottleneck in the middle is removed - this does not change how your side of things works.

      In essence; You will use Ti 4.0 in almost exactly the same way as you use it right now, same JavaScript, same APIs (some tiny changes where absolutely required). Things like studio will change; because things get better.

      You will not lose any of the things you currently have with the JavaScript you use right now. You do not have to think ahead to wonder what might happen during translation/compiling - this is not anything you will ever be part of.

      There are many goals of Hyperloop, app size reduction is one, but not top of the first stage priorities - but average app size will decrease and more so as 4.1, 4.2 etc are rolled out.

      Some of the many goals;

      • Massively reduced amount of platform specific effort required by Appcelerator
      • Significantly increased core code re-use
      • Adding new platforms no longer adds another 90% of the time, often around 10% is the hope
      • App performance - will be much faster
      • For pros, the ability to use platform specific APIs but with the JavaScript syntax in-line with all other code

      So in short, when Ti 4.0 comes out your coding life will barely change, same JS, same Ti. However your apps, will be faster, app size will reduce and more over time. You will gain new platforms - win phone being one of them. Also you code beyond Ti and still be in Ti.
