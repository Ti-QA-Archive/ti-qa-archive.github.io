Titanium Studio on Ubuntu: doesn't keep changes

Hi,

following investigating after this answer, I finally figured out this issue: my instance of Titanium Studio doesn't keep changes.
Basically, what happens is: once I started the IDE for the first time, it asks me to install updates such as CLI, Alloy, ACS (better said: it forces me to do such installation). Everything works fine, I can create my apps in alloy, deploy them both in emulator and device, and so on. Then I close the IDE and, once I open it again, it asks me to re-install updates like it lost previous configuration - even appearance settings are lost.

My system is Ubuntu 12.04 LTS, I work on my user folder so I should not face permission problems (or maybe yes, but why?).

Please help me figuring out what is wrong, that's kind of frustrating ;)

thanks

Emanuele

  • If you are running with a normal user you can't update cli,alloy,acs using the update dialog cause you don't have permissions. You have to open a console, switch to root and do the update using npm. Then you can start tistudio again and it won't ask you again. At least thats how I do it under fedora
