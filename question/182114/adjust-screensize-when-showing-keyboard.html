Adjust screensize when showing keyboard

Hi,

I am building a mobile application for Titanium. In the mobile app, I have a screen with multiple input fields and a button at the bottom of the view. When the user wants to input text in the textfields, the screen scrolls to make sure the contents of the textfield is visible. However, I would like that the entire screen is visible at all times to make sure that, when the user inputs data in the lowest textfield and the keyboard is expanded, he can immediately click on the button beneath that, without having to first collapse the keyboard.

I have searched a lot online for a solution for this and always found these two solutions:

  1. putting the android:windowSoftInputMode to "adjustPan"

    <activity url="app.js" android:windowSoftInputMode="adjustPan" />

  2. adding the windowSoftInputMode property to the window itself

    window.windowSoftInputMode = Titanium.UI.Android.SOFT_INPUT_STATE_VISIBLE;
window.windowSoftInputMode = Titanium.UI.Android.SOFT_INPUT_ADJUST_PAN;

Both solutions have been tried, but they all don't fix my issue. They only make the screen scroll upto the textfield that is in focus, and hide the buttons below it.

For reference: this is my entire android xml configuration:

<android xmlns:android="http://schemas.android.com/apk/res/android">
        <manifest>
            <application android:theme="@style/Theme.Titanium"/>
        </manifest>
        <activity url="app.js" android:windowSoftInputMode="adjustPan" />
 </android>

And this is a graphical representation of what the problem is:
http://imgur.com/oG5gCKv

Does anybody know how to solve this? Thanks!

  • Accepted Answer

    The user shouldn't have to collapse the keyboard to get to the next field. The user should be able to scroll within the reduced-height upper area.

    You can put your form into a ScrollView, and this will all happen automatically for you. If it's not in a scrollview, I could imagine that it might behave like you're describing (have to hide the keyboard to see the other controls); I'm not sure – I almost always put forms into ScrollViews. With any substantial form, you really can't be sure that the form will fit on every screen, so the ScrollView helps you deal with those tiny old iPhone 4s and low-end Android phones.

  • Try to put an animation to make the view goes up whenever the user focus on the last field, somewhat like this:

    function viewUp(){
    $.lastTextFieldParentView.animate({
        duration : 300,
        top  : -200 // set the amount here according to your designs, -200 in this case
    });
}

function viewDown(){
    $.lastTextFieldParentView.animate({
        duration : 300,
        top  : 0 // set the amount to zero to reset the parent view to its initial position when the layout is set to vertical, or set this value according to your designs
    });
}

$.lastTextField.addEventListener('focus',viewUp);
$.lastTextField.addEventListener('blur',viewDown);

    This is the only way i use to make whole view visible at the same time.
    You can also calculate the values to make the correct position of your form view.
