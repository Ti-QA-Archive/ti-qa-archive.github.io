  • Just a long shot but one thing that I got bitten by recently was a function call failing only on a release build but not in a dev/adhoc simulator. The path names to included or required modules is CASE SENSITIVE ONLY on a release build. Check that your file names are exactly the same case as your include/require statements.

  • As I promised, now that my app is in Store, i can tell you what went wrong.
    The problem wasn't at the http connection, it was when I create the annotations.
    All the details are in JSON format. So the latitude and longitude were parsed as string instead of float. Big bug!!
    Second I'm using the rightButton property like this: rightButton : Titanium.UI.iPhone.SystemButton.DISCLOSURE,. Here the app crash. So I put at the begining of the app.js this code : 

    var img_test = Ti.UI.createButton({
    style : Titanium.UI.iPhone.SystemButton.DISCLOSURE
})

    and the everything goes well…
