Android GPS issue

Once the app loads i have to get the users GPS location to pass to the server. I was originally using getCurrentPosition but it does not actually turn on the gps to get your location. I changed it to Ti.Geolocation.addEventListener(&#39;location&#39;, locationCallback); which after it gets the location, it sends it to the server and turns the gps off. The problem is it can take 15 seconds to return a gps location the first time. Is there a way to get it to return faster?

  • Accepted Answer

    Try:

    var mapview = Titanium.Map.createView({
    userLocation:true,
    width:1,
    height:1
});


mapview.hide();
win.add(mapview);

    With the hidden mapview in the background, it should turn on GPS on Android.

  • Start the location set to get the best accuracy the very first thing when the app starts. While the user is playing with the app you should have time to get the location.
    A "cold boot" of the gps usually takes a good amount of time, even with a very strong gps signal. There is nothing you can really do about, except what I just said: try to get the location as early as possible.
