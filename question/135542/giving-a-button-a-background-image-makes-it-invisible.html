Giving a button a background image makes it invisible

http://developer.appcelerator.com/blog/2011/10/forging-titanium-episode-10-forms.html

This is a great tutorial and a fantastic module. I just have one question: When I try to modify the button that the module creates to give it a background image, the button just vanishes. In the unmodified module, lines 114 to 119:

        var button = Ti.UI.createButton({
            title: title,
            height: '40dp',
            width: '100dp',
            top:'10dp'
        });

That creates a very boring white button that is begging to be styled up a little. So I downloaded a few button background images from the kitchen sink, put them in the resource folder, and tried to use them:

var button = Ti.UI.createButton({
            title: title,
            height: '40dp',
            width: '100dp',
            top:'10dp',
            backgroundImage:'BUTT_grn_off.png'
        });

No matter what I try, the button just simply vanishes. I can get rid of or play with the height and width to no avail. Of course I've also added the Selected and Disabled buttons. I've even found sample buttons that use these images and tried them. Same result. No button.

I've used this sample:

http://ks.kanna.asia/controls/button/

and made the module look like this:

var button = Ti.UI.createButton({
            color:'#fff',
            backgroundImage:'BUTT_grn_off.png',
            backgroundSelectedImage:'BUTT_grn_on.png',
            backgroundDisabledImage: 'BUTT_drk_off.png',
            top:10,
            width:301,
            height:57,
            font:{fontSize:20,fontWeight:'bold',fontFamily:'Helvetica Neue'},
            title: title
        });

No love. What could I be doing wring here?

Thanks!

  • there are many options to solve this problem , try :
    if your image is in resource folder.

    backgroundImage:'/BUTT_grn_off.png',

    or

    backgroundImage:Ti.Filesystem.resourceDirectory +'/BUTT_grn_off.png',

    or try clean and build and full uninstallation. Or may be your application not getting the media changes , you can try after doing changes in tiapp.xml.

    <property name="ti.android.fastdev" type="bool">false</property>
