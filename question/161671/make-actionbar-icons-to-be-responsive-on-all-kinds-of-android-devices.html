  • The documentation doesn't directly address Allen's question. He is looking for a reference list of recommended icon pixel sizes for the Action Bar in Android.

    I haven't found a list of sizes for Appcelerator, but here is the native Android SDK documentation:
    Android Iconography Guide. BTW, there is also an icon size reference guide for iOS.

    Google says that Action Bar icons should be 32x32 display independent pixels (dp), although the icon should be centered inside the middle 24x24dp with a 4dp transparent border for padding. Depending on the screen size that you wish to target, the true pixel sizes should be as follows:

    • LDPI (120 DPI) = 0.75X = 24x24
    • MDPI (160 DPI) = 1X = 32x32
    • HDPI (240 DPI) = 1.5X = 48x48
    • XHDPI (320 DPI) = 2X = 64x64
    • XXHDPI (480 DPI) = 3X = 96x96
    • XXXHDPI (640 DPI) = 4X = 128x128

    Here are the options to target these 6 sizes using Appcelerator:

    1. Create high/med/low icons - Under your /app/assets/android/ folder, create sub-folders named "high", "medium" and "low" and place your icons into these folders as appropriate. Android will do its best to match the device to one of these categories.
    • Pros: Only requires 3 graphics.
    • Cons: The actual result can be somewhat unpredictable. Based on the Android Iconography guide, there are 6 different sizes that you should accommodate. I'm not sure how Android would map to these sizes. I suspect that low = 120 DPI, medium = 160 DPI, high = 240 DPI, and everything else (320 DPI and higher) is scaled up. It may take some experimentation.
    1. Create density specific icons - Under your /app/assets/android/ folder, create sub-folders as detailed in the documentation. If you target all of the orientations (landscape vs portrait) and aspect ratios (long vs not long) then you would need 4 variations for each of the 6 sizes. Altogether, you could end up with 24 sub-folders.
    • Pros: This offers the most granularity so that the icons can look perfect on any display.
    • Cons: You have to create and test multiple versions of every icon, potentially up to 24 variations of each one.
    1. Use one icon for all devices - Place a single image directly in the /app/assets/android/ folder. Add an entry to tiapp.xml inside <android><manifest> which says <supports-screens android:anyDensity="false" />.
    • Pros: Definitely the easiest option. Only one file is required per icon.
    • Cons: All UI objects will be scaled to display independent pixels instead of using the device's native pixels. Scaling is based on a 160dp screen size. Google doesn't recommend this option since some UI elements can look stretched and blurry, and the application won't take advantage of the device's higher resolution.

    I recommend a simplified version of option #2. Looking at Google's Action Bar Icon Pack, I see that they only created 4 versions of each icon: MDPI, HDPI, XHDPI and XXHDPI. They didn't bother targeting different orientations or aspect ratios. Based on that pattern, 4 icon variations should be enough.

    I suggest using the following sub-folders under /app/assets/android/images/:

    • res-mdpi (32x32px)
    • res-hdpi (48x48px)
    • res-xhdpi (64x64px)
    • res-xxhdpi (96x96px)

    If you follow that pattern, Android knows exactly which densities you've provided. It can correctly scale up or down as needed to support the other resolutions (LDPI and XXXHDPI).

    Whenever possible, save yourself some work. Try to use the built-in icons from Titanium.Android.R-property-drawable whenever possible. Unfortunately the list of icons is pretty limited.

    On a related note, I ran across a great utility called TiCons that handles creation of multiple startup screens and application icons for both Android and iOS. Perhaps you could find something similar that could be used for Action Bar and Tab Group graphics.

    Happy coding!
