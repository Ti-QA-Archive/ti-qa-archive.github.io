Kitchen Sink fails to find any SDKs - a newly asks!

Hello!

\ OSX 10.7
\ XCode (latest from App Store)
\ Android SDKs (all!)
\ Titanium Studio (latest)
\ Kitchen Sink app (latest)

Today I've followed the Getting Started tips at http://developer.appcelerator.com/get_started which sounded straightforward enough. I've downloaded & installed XCode via the Mac's App Store together with all available Android SDKs.

To import the Kitchen Sink project I've launched Titanium Studio and followed: Import Project > Titanium > Import Existing Titanium Project > Next > Browse {locate Kitchen Sink folder] > Open

All I ever get from Titanium Studio is: "The following warnings were detected:
Missing an Android SDK required for this project.
Missing the iOS SDK required for this project"

Clicking on the Titanium Mobile Preferences link from the import screen in Studio there's the option to specify where my Android SDK is but nothing about iOS? In any event I've tried pointing the preferences at every conceivable folder inside 'android-sdk-macosx' but Studio fails to find an Android SDK in any of them.

As for iOS I'm at a loss as to why Studio can't find XCode and there doesn't seem to be any options for specifying where it is!

So I've fallen at the first hurdle. Please can anyone give this newby a little nudge in the right direction?

Many thanks,

Steve

  • Hi Steve

    Check that you have, at a minimum, all the packages shown in the screenshot of the Summary of Installed Android Packages section of the documentation.

    Install the latest continuous build, by selecting master from the drop down menu, right-clicking and copying the URL of the archive for your OS, and pasting it into Titanium Studio's Help > Install Titanium SDK from URL dialog. Note that you don't have to actually use this version of the SDK for your apps; it just needs to exist on your system.

    Note: if you want more information about continuous builds, see the Installing Titanium SDK Continuous Builds guide.

    Make sure that Studio is configured with the location of the Android SDK, via its Preferences > Titanium Studio > Titanium dialog. See Configuring Titanium Studio with the Platform SDKs.

    Then restart Studio and try again.

    If this doesn't resolve it, please let me know and I will help you further.

    Cheers
