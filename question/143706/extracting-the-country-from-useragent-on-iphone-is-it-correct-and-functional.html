Extracting the country from UserAgent on iPhone. Is it correct and functional?

Hi there!
I am creating an app that I want to run on iOS in different countries in the World. I need to know the country where each users lives (I don't mind if I'm wrong about some users, but most of them should be localized ok).
I have struggled with Titanium.Locale to no avail: the country appears as "undefined".

var country = Ti.Locale.currentLocale;

I have tried different ways, but suddenly I've found the following in the console log:

Oct 19 10:08:21 iPhone-Name KitchenSink[41014] <Warning>: [DEBUG] user agent established in Appcelerator Titanium/2.1.3.v20120915120319 (iPhone/5.0.1; iPhone OS; es_ES;)

So, this is the language and country!!! es_ES, I mean.
Would it be ok to use the user agent data to know what country the user belongs to?
Thanks in advance!

    • It doesn't work on iPhone. The property definition states that the country code is sent. However, that is not the case. It only returns the current language in a two-letter string.
      If I have my iPhone in Spanish, it returns "es". If I set its language to English, it returns "en" alone.
