Database Update not persistent (SQLITE)

Hi,
I have a function that adds a row in a database.
The row works fine. It is added in the database correctly.
When I close the app, and open it, the entry is STILL there.

However, if I close the app by double clicking the iPhone home button and close the app from the Apps that are open, and then launch the app again, the data i have added in the database is not there.


var db = Ti.Database.open('mydatabase');    

var dbrows = db.execute('UPDATE myTable SET COL1 = null where STATION_ID=?', 'x');

db.close();

As I mentioned, the row addition works perfectly well. It just doesn't last long enough for me to be able to charge money for this app.

I know in Oracle, you need to execute a commit to save the changes to a database. What is the equivalent for Titanium/SQLite Database updates?

Thanks
