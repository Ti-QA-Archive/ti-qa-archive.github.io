Guide or best practice

I have been playing with the examples and reading the documentation and was wondering what is recommended or a best practice for when to: "include" a javascript file or load a javascript file via "url".

I understand from the documentation that a file loaded via "url" is in a subcontext
and thus requires all data to be passed via events. Between the global and subcontexts.

In which instances would it be best to use one contra the other?

I have been modulizing my code into separate include files:

Example code in external file:

var BrowserPanel = {};
BrowserPanel.view = Ti.UI.createView();

BrowserPanel.init = function(){

var webdoc = Ti.UI.createWebView();

var searchBar = Ti.UI.createSearchBar({top:0,height:44,barColor:'#333'});

BrowserPanel.view.add(searchBar);

searchBar.addEventListener('return',function(e){
    searchBar.blur();
    webdoc.url = 'http://' + searchBar.value;
});
BrowserPanel.view.add(webdoc);

}

BrowserPanel.init;

  • This topic is touched in in multiple videos in the Appcelerator training courses, so reviewing those would wise. You'll find a mountain of other best practices in their as well.

    The short version is that you should likely use a single context in most cases so that data can be more easily shared in your application. Loading separate contexts using url is ideal only when a clean slate is the goal with each window. A good example of this is the Kitchen Sink. Each section of the Kitchen Sink contains stand alone examples.
