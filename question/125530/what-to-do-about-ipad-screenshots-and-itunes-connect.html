What to do about iPad screenshots and iTunes Connect

This is my first app and I'm having trouble understanding some of what iTunes Connect is asking of me. I can't find a solid answer in my searches.

I've validated and submitted my iPhone app to iTunes Connect and the status immediately changed to 'Missing Screenshot'.

I've learned that I must have iPad screenshots uploaded as well because my app is a 'Universal' app. I also understand that for apple approval, my app must also run on the iPad.

My questions are:

  • How can I get iPad screenshots? Do I actually have to install this on an iPad test device?

  • How did my app get set as a 'Universal' app? Is there anyway to set it to be 'iPhone' only?

Thanks so much to anyone who can provide guidance!

  • Are you using Titanium Studio? Do you have iPad set as a 'Deployment Targets' in your tiapp.xml file? You should be able to create an iPhone only app that doesn't need iPad screenshots.

    — commented September 13th 2011 by Jeff Bonnes

  • When I had this problem I was trying to submit an iphone-only app. But you need 2 things for that:

    in the 'manifest' file you need:

    #type: mobile

    and in 'tiapp.xml' you need:

    <target device="ipad">false</target>

    Once I fixed both of those and resubmitted the app, my 'Missing Screenshot' error went away.

  • HI,

    As Jeff said while during creation of the project you can opt to target specific platform, or try changing the Boolean values for targets in the Tiapp.xml(but remember to clean the project and rebuild again).

    Yup if you can run it on an iPad you can test how the App actually works on iPad(see if some random crashes occur) and at the same time you can take screen shots using the xcode's organizer tool or a simple work around is to use Cmd+shift+4 to select the simulator area to take a screenshot.
