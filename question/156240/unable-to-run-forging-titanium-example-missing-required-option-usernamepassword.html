  • I have it working but it seems like a hack. I got the idea from a few other vague posts I've seen here in the forum. I downgraded to Node.js v 0.8.22, which is older than the 0.10.13 version that Titanium says it needs.

    I went to http:&#x2F;&#x2F;nodejs.org&#x2F;dist&#x2F;v0.8.22&#x2F; and downloaded the node-v0.8.22.pkg file (remember that I'm on Mac OS X 10.8.4). I then ran the installer but first had to update my Mac's security preferences (System Preferences –> Security and Privacy –> General) to allow installation of applications downloaded from anywhere.

    The node.js installer did it's thing. After restarting Terminal, I did titanium login xxxx from the command line, entered my password, and it said I was logged in.

    I restarted Studio and it immediately said there were upgrades available. It was prompting me to install Node.js 0.10.13, but I declined to do so.

    I don't know what problems I'll have using an old version of node.js. Now that I'm logged in (and the login appeared to survive a system restart) I'm wondering if I can upgrade back to node.js 0.10.13, but I'm scared to do so.

    Better yet, I was finally able to run the Forging Titanium episode 2 example code (cross platform navigation controller) which is why I started this in the first place!

    Clearly there is something wrong with the Studio installation. Please fix.
