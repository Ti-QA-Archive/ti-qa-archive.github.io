Searching in web API -> results in view

What would be the best approach of building a searchfield (with eventually extended options) which submits its value to a REST API (returning JSON) and putting the results in a view? I've seen quite some issues around the web and the documentation is very limited on how to implement this.

I'm using as much Alloy as I can because I like the view-setup (too bad the docs seem to focus a lot on regular Titanium which makes the Alloy-variant sometimes vague), so i'd like to use SearchBar/TableView where possible

Any directions/indications?

    Well, when the user hits "search" you take the value of the searchfield, send it to the server, meanwhile you show a "loading…" or whatever visual indicator to the user.
    When the httpclient loads, you get the json and populate the listview/tableview and then hide the "loading" indicator.

    To make this as fast as possible

    • gzip the server reply
    • send only the data you need, don't send a verbose json from which you use 2 fields only
    • send "pages", like 10-15 rows in a set, if the user wants more you can always load the next page with a new request

    Further optimisation would be to make a manager for the http requests based on priority and concurrency, so you can ask the server the most important request as soon as possible, and not to have lot of data downloading in the same time.
