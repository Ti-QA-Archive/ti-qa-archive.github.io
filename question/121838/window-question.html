Window Question

Hello-

I am hoping someone can tell me the best practices for opening and closing windows for android/titanium. I am able to open windows based on criteria (logging in, registering, etc) but whenever I go back to a window, all of the functionality on the window stops working.

Example.

I come to the login screen on my app and click on the register button. This event takes me to the registration screen. I click the cancel button here to go back to the login screen. Everything works, I am sent back to the login screen. However, now nothing on the login screen works. I cannot enter a username or password or click a button.

Thanks in advance for your help.

Josh

    > However, now nothing on the login screen works. I cannot enter a username or password or click a button.

    I've already had this "problem" once : it was because of an invisible (i.e. with no background and no elements) window or view that was put foreground and received all the events.
    Therefore, the window displayed (in background) couldn't receive any event. I think you should check you have no "ghost" window or view at this time.

  • Hi Josh.

    I use heavyweight windows without "url" property (and with "modal" property). Every component is in it's own JS file which I load with "include" function. Component is created with function constructor.

    My typical JS file looks something like this (super-short version :) ):

    //object based namespacing
Ad.createNamespace(Ad, 'modules.geo');

//autoloader -> maps component names to JS files
Ad.util.register(
    'Ad.modules.geo.models.db.Regions',
    'Ad.modules.geo.ui.Regions',
    'Ad.modules.geo.Region'
);

//build component
Ad.modules.geo.Regions = function(config) {

   var regions_model = Ad.modules.geo.models.db.Regions();

   var regions_data = regions_model.getAll([ 'id', 'name' ]);

   var ui = Ad.modules.geo.ui.Regions({ items: regions_data });

   Ad.events.subscribe(ui.TableView.Element, 'click', function(e) {
       var id = (e && e.rowData && e.rowData.id) ? e.rowData.id : 0;
       var region = Ad.modules.geo.Region({ id: id });

       //HW window
       var context = Ad.context();

       Ad.events.subscribe(region.Toolbar.CancelButton.Element, 'click', function() {
            context.close();
       }

       region.addTo(context);

       context.open();
   });

  return ui;
};

    All functions in "Ad" namespace are wrappers around Titanium functions and components, in event part I register and store all event handler functions because before window closes, I unregister all events and remove all window children (because Titanium is lazy to do that :) ).

    I can send you relevant source code so you can see implementation details if you wish (it's still in early development phase and some concept parts can be better, but it's useful for things and problems like yours). It doesn't follow Titanium best practices, because I like different coding style. It took me a lot of time to get this working. :(
