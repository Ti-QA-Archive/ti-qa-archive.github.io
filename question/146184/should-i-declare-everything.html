Should I declare everything?

Hi all,

Looking at examples and videos, and they all declare variables before using them in the .add() method.

Is this nessasary?

Eg.: couldn't I just go:

oRow.add(Ti.UI.createLabel({
text : "Projekt titel"
});

instead of having to declare my label first, and then appending it?

I would really like to just add simple objects to the .add() method, without declaring them first. But I'm not entirely sure if this will "clutter" my scope?

  • Great examples from Malcolm. To your point about clutter, if your code is constructed properly (with CommonJS modules), you don't have to worry about cluttering scope. Yes, there will be more variables within the scope of your module, but that shouldn't be a big deal.

    I find that it's easier to assign my views to variables almost all the time. A common problem with not assigning them to views is that you find out later that you need to reference the view (to set a property, retrieve a property, add an event listener, etc.). Then you have to go back and retrofit a variable in there.

    That irritates me enough that I just go ahead and assign almost every view to a variable as I'm setting up my views.

    But a lot of this comes down to personal preference.

    Hi

    The only place you should declare everything is at Customs :)

    It is common to declare variables but not required. Examples commonly do this as the created view is often then altered in script. You do not have to explicitly declare variables before adding them to the context in question UNLESS you want to access them in script outside an Event Listener.

    Example 1

    var win = Ti.UI.createWIndow({
    height: Ti.UI.FILL,
    width: Ti.UI.FILL
});
var lblTitle = Ti.UI.createLabel({
    height: Ti.UI.SIZE,
    text: 'Title',
    top: 20,
    width: Ti.UI.SIZE
});
win.add(lblTitle);
win.open();

    Example 2

    var win = Ti.UI.createWIndow({
    height: Ti.UI.FILL,
    width: Ti.UI.FILL
});
win.add(Ti.UI.createLabel({
    height: Ti.UI.SIZE,
    text: 'Title',
    top: 20,
    width: Ti.UI.SIZE
}));
win.open();

    Example 1 and Example 2 are functionally and essentially identical. However the following variation of Example 1 would not work in Example 2.

    Example1b

    var win = Ti.UI.createWIndow({
    height: Ti.UI.FILL,
    width: Ti.UI.FILL
});
var lblTitle = Ti.UI.createLabel({
    height: Ti.UI.SIZE,
    text: 'Title',
    top: 20,
    width: Ti.UI.SIZE
});
win.add(lblTitle);
var btn = Ti.UI.createButton({
    title: 'Press Me',
    top: 120
});
btn.addEventListener('click', function (e) {
    lblTitle.setText('Changed'); // would only work if declared
});
win.add(btn);
win.open();
