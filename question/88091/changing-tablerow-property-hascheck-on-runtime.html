changing TableRow property: hasCheck on runtime

hi,

i try to change the hasCheck property when a row is clicked.

on the TableView object i have a "click" eventlistener.

tableView.addEventListener("click", function(e){
    Ti.API.info(e.row.id); // traces the correct row id i provided
    e.row.hasCheck = true; // does nothing
});

the trace gives me the corect row, but the check icon doesn't show.
when i set the property when creating the row, the icon is visible.

any ideas?

i'm using the sdk 1.5.0 for android

  • I have the same problem running SDK 1.5.1 and Android. The change is reported in the row object, but the tableview is not being "repainted". Here is my code.

    tableView.addEventListener('click',function(e){
     Ti.API.info(e.rowData.hasCheck);
     if (e.rowData.hasCheck){
          e.rowData.hasCheck = false;
     }else{
      e.rowData.hasCheck = true;
     }
});

    On the debug it correctly reports the previous hasCheck property.
