Bug for TableView ClickEvent on iOS 6

Greeting All,

I got one issue for TableView ClickEvent on iOS6. Previously my app is working fine and live on app store. After iOS6 upgraded, app is not working. But not crash also. I got a lot feedback from my users so that when I trace on coding, I saw something at below:

Previously, we can create like this for TableView

var myTableRow = Titanium.UI.createTableViewRow({height:46, className:'myTableRow'}); 

var myLabel = Ti.UI.createLabel({color:'#000000', text:"Hello", font:{fontSize:17, fontWeight:'bold'}, top:8, left:12, height:24, width:170});

myTableRow.add(myLabel);

array.push(myTableRow);

var tableView = Titanium.UI.createTableView({ 
         backgroundColor:'transparent',     
         data:array,      
         top: -17, 
        style:Titanium.UI.iPhone.TableViewStyle.GROUPED });
win.add(tableView);


tableView.addEventListener('click', function(eventObject){     
if (eventObject.rowData.className == "myTableRow")      {    
    //do something at here.
});

Last time eventObject.rowData.className is working but now it is returning as TiUITableViewRowProxy. How can I fix it?

Note: I already installed 2.1.3 RC according this blog post. http://developer.appcelerator.com/blog/2012/09/titanium-sdk-2-1-3-rc-is-released-support-for-ios-6.html

If you all don't mind, please give some suggestion for it.

  • Thanks for you guys answering at my post. Yes after I changed something according Aaron's reply, that issue was solved. Here is what I did.

    //add my own property
var myTableRow = Titanium.UI.createTableViewRow({height:46, className:'myTableRow',myRowName:'hellorow'});

    //when I check at IF condition, I used it.
//Don't use className at here
if (eventObject.rowData.myRowName == "myTableRow")      {   
    //do something at here.
});

    I think that is quick fix only. By right Ti SDK should be managed that kind of issue. Hopefully, 2.1.3 final release would be fixed that kind of small small issues.
