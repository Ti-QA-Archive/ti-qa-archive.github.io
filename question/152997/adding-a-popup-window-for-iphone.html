    Hi

    The standard alert box has this option when the actual AlertDialog is used instead of the alert shortcut and the extra style attribute introduced back in iOS 5.0.

    var dialog = Ti.UI.createAlertDialog({
    cancel: 1,
    buttonNames: ['Login', 'Cancel'],
    message: 'Enter your details to continue?',
    style: Ti.UI.iPhone.AlertDialogStyle.LOGIN_AND_PASSWORD_INPUT,
    title: 'Login'
});
dialog.addEventListener('click', function(e){
    if (e.index === e.source.cancel){
        Ti.API.info('The cancel button was clicked');
    }
    Ti.API.info('e object: ' + JSON.stringify(e));
});
dialog.show();

    The responses are passed back in the event object parcel, these include; text, login and password.

    The key is the style property, other variations;

    style: Ti.UI.iPhone.AlertDialogStyle.PLAIN_TEXT_INPUT // just text
style: Ti.UI.iPhone.AlertDialogStyle.SECURE_TEXT_INPUT // password style just text
style: Ti.UI.iPhone.AlertDialogStyle.LOGIN_AND_PASSWORD_INPUT // username and password
style: Ti.UI.iPhone.AlertDialogStyle.DEFAULT // normal with no boxes
