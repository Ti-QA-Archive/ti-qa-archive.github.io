  • It would help if you could reduce this down to a smaller example, but here's a few things I spotted:

    • You haven't really explained what is and isn't happening with this example, which makes it really hard to help when you haven't given us a small reproducible test case.
    • You should attach new properties directly to Alloy. Any additions you make the to Alloy namespace should either be on Alloy.Globals or Alloy.CFG.
    • Your index.js should not create another controller, since the settings controller already exists (you defined it in the XML). Defining it in the XML creates it and has already made it part of the view hierarchy.
    • You've got 2 components with the same id: settingsCTRL. This will cause all kinds of unexpected issues.

    You've got a lot going wrong here, so it would probably be best to reduce this to a smaller reproducible test case that exhibits one issue at a time that we can test on our end.
