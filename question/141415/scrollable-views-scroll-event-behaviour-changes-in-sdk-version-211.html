Scrollable View's scroll event behaviour changes in SDK version: 2.1.1

While making a demo I noticed that a Scrollable View behaves different in 2.1.1 SDK, It calls multiple time while page scroll in scrollableView.

I am using Titanium SDK version: 2.1.1 (07/27/12 14:01 0fd84a2)

I tested following code for demo and realize that getting multiple time scrollview event calling. At very first scroll of the page I got muliple call for event

I tested this code in Titanium SDK version: 1.8.2 (03/28/12 17:04 36d96f1) it runs perfect and gives output only once when page scrolled.

Is it ok or something wrong in this case?

var loadedViews = [];
var win = Ti.UI.createWindow({
    title : "Test"
});

win.open();
var scroller = Ti.UI.createScrollableView({
    showPagingControl : true,
    maxZoomScale : 1,
    minZoomScale : 1,
    backgroundColor : 'gray'
});
scroller.views = loadedViews;
var i = 0
for ( i = 0; i < 3; i++) {

    var v = Ti.UI.createView({
        height : 200,
        width : 200,
        backgroundColor : "blue"
    });
    var lbl = Ti.UI.createLabel({
        text : i,
        font : {
            fontSize : 50
        },
        color : 'whte'
    });
    v.add(lbl);

    loadedViews.push(v);

}

scroller.addEventListener('scroll', function(e) {
    Ti.API.info("Current Index is : " + e.currentPage + "\tView Scrolled....");
});
win.add(scroller);
scroller.views = loadedViews;

  • Yes it is always better to have another event in scrollableview pageChanged event. I created pageChanged event like this below with help of scroll event

    var viewNumber = 0; //This is default selected view index, i just pass 0 because it is first view
        scroller.addEventListener('scroll',function(e){
        if(viewNumber != e.currentPage)
        {
            viewNumber = e.currentPage; // And saving for next time change notification
            scrollableView.fireEvent('pageChanged',e) // And I am fire my pageChanged event on scroller
        }
    });

    /*Creating custom event for page changed event using scroll event above*/
    scroller.addEventListener('pageChanged',function(){
        alert('Event fired...'+ e.currentPage);
        //Do something here
    });

    I hope this will be helpful for you in iOS. In android we can use dragStart and dragEnd events
