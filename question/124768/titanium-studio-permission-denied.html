Titanium Studio Permission Denied

Hi Team, i have installed the Titanium Studio Free version in mi Administrator account of my computer.

Now, i have created other account on my computer the new account is the type admin.

In the new account of my computer, i opened the Titanium Studio, i created the hello world application, for compiling the application, the titanium show to me the following error:

[INFO] One moment, building …
[ERROR] Error: Traceback (most recent call last):
File "/Library/Application Support/Titanium/mobilesdk/osx/1.7.1/iphone/builder.py", line 588, in main
versions_txt = read_config(os.path.join(template_dir,'..','version.txt'))
File "/Library/Application Support/Titanium/mobilesdk/osx/1.7.1/iphone/builder.py", line 86, in read_config
contents = open(f).read()
IOError: [Errno 13] Permission denied: '/Library/Application Support/Titanium/mobilesdk/osx/1.7.1/iphone/../version.txt'

Would you help to me a solve it.
Thanks Jorge Luis

    Jorge

    I am able to reproduce this issue on my system. The problem is that permissions of the files in the &#x2F;Library&#x2F;Application Support&#x2F;Titanium&#x2F;mobilesdk&#x2F;osx&#x2F;1.7.1 directory (or the directory that holds the version of the SDK you are using to build your app) are only accessible by the user who was running TiStudio when the SDK was downloaded/installed.

    As a temporary workaround, you can use the following steps. However, please be aware that if followed incorrectly they can cause serious issues with your system. Proceed with care!

    • insure current your user has the "Allow user to administer this computer" OSX privilege
    • start the Applications/Utilities/Terminal application
    • run sudo -i and enter your password, to gain admin access to the filesystem
    • run cd &#x2F;Library&#x2F;Application Support&#x2F;Titanium&#x2F;mobilesdk&#x2F;osx&#x2F;1.7.1 (make CERTAIN you are in the correct directory before running the following command!)
    • run find .&#x2F; -exec chmod 777 {} \; (make CERTAIN that you use a dot and forward-slash as the first argument!)

    Once this is done, your app should run successfully.

    I will bring this up with the TiStudio team, and report back with any more relevant information.

    Thanks
