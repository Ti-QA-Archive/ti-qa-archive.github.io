do you use checks so events don't fire multiple times when button clicked?

I notice that sometimes when a button is clicked an event might be fired multiple times. It happens randomly so I can't reproduce it. Does this happen to you, and do you use any checks for this?

  • I have seen this behavior, especially when the button click event listener is opening a window – a quick double-tap of the button can fire the event more than once.

    For situations like this, I have built an "event throttle" class that I use to ignore all taps until the child window is closed. The class can also use time-based throttling (e.g., ignore taps within 500ms of the first tap).

    See the TU.UI.EventThrottle class in the TitanUp library for details.
