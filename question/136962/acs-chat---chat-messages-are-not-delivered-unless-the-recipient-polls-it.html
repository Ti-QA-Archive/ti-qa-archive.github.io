ACS Chat - Chat messages are not delivered unless the recipient polls it

I am using ACS Chat API in my mobile application. When User A sends a chat message to User B, it is delivered to User B. But the recipient of the message is required to make a query to check for the incoming messages. Is there a way to notify the recipient that he/she has a message to be read? Is there a callback mechanism to achieve the purpose without using any Push Notification mechanism?

    We always return paginated results. You should see in the meta that how many results and how many pages, by default, we return 10 mesages per query, you can ask for up to 100 message per query and specify page number if you want to get more history
