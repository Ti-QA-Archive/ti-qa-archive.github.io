How do I notify users without tab badge in Android

Hi,

I am using Ti 3.2.3 on OSX 10.9.3. I am currently adjusting an iPhone app for Android. My iPhone app utilises the tab badge to notify the users that there are things to look at. I know that Android doesn't support this property so I have tried adding a simple view containing a label to emulate the badge to the following with no success:

  1. The tabGroup (ActionBar)
  2. The tab element itself
  3. The window for the tab with a negative top value
  4. The base window the tab group is added to (controls app exit)

None of these things actually end up showing the view even with a zIndex of 10000. The tab group seems to appear on top no matter what.

I am formatting the action bar using a custom Holo Theme so perhaps this is interferring.

Is there a way to emulate the tab badge icon successfully using action bars? Given that I would like to utilise native Android as much as possible is there a way that this sort of notification should be done in an Android app?

Please note I have also looked into changing the icon for the tab group and found that is not possible once the tab group has been drawn.

Kind regards,
Brad.

    Android doesn't have an equivalent. It's not part of their UI design standards. You're going to have to look to standard techniques (see http://developer.android.com/design/patterns/notifications.html or other apps for inspiration. You could build yourself a custom tab controller component that lets you add notification badges. There's this (old & unmaintained) project that might get you started on that https://github.com/patrickseda/Badge. However, you're breaking design conventions by implementing this UI. Users on Android don't expect or want iOS-style apps.
