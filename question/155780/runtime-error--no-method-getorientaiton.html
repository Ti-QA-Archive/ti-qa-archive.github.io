  • Because whereas on Android there is an orientation property, there is no getOrientation getter method (even though the docs says there is!). This is one reason why I never use the getter/setters and always use the properties instead

    • No.There are two ways of accessing a property. One, by using the property directly eg win1.orientation, or by using the getter method call to access the property eg getOrientation(). There are setter methods too eg setOrientation(). HOWEVER, this pattern is not fully implemented across iOS and Android (even though in places in the docs, it says its supported)

    function getOrientation() {
    if (longueurDevice > largeurDevice){
        orientationDevice = "portrait";
    }
    else if (longueurDevice < largeurDevice){
        orientationDevice = "landscape";
    }
    return orientationDevice;
