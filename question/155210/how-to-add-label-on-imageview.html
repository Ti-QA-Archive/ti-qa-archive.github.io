    • As this seems to be the lead answer of the four identical answers I am going to add to it to explain the reason WHY the technique you have tried fails on Android but works on iOS. I do not need to add another answer to achieve this a simple comment will do.

      Adding a child object to a parent object is supported in both Android and iOS, both platforms consider some views to be containers and the rest are not containers. However only Android enforces the logic of ONLY containers can contain child views.

      For example; imageView, label and buttons are not containers, so Android enforces the rule of only containers can accept child views so following list shows what happens (not full list - just a guide);

      • view inside label; Android = No, iOS = Yes
      • view inside view; Android = Yes, iOS = Yes
      • label inside view; Android = Yes, iOS = Yes
      • label inside label; Android = No; iOS = Yes
      • label inside image; Android = No, iOS = Yes

      Have fun
