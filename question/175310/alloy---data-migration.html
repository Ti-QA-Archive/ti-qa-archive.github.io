Alloy - Data Migration

docs

It is confusing. How does Alloy know which version the previous database is? The docs only says Alloy applies mirgrations from the oldest to the newest.

If the previous version is, for example, version 3, then Alloy should apply from version 4 to the newest, not from version 1 to the newest.

How does Alloy know?

    Alloy maintains a table in your database noting the date/time of installation of the database and which was the last migration applied, if any. From that information, it can calculate which migrations to apply. If you want the details, you can check out the code at https://github.com/appcelerator/alloy/blob/master/Alloy/lib/alloy/sync/sql.js
