i18n not working on Alloy application

Hi,

I want to add i18n on my application.
The guide in the documentation has no reference to how to do it on alloy, but i've seen some questions about it and people say it works the same way.
Unfortunately, my app doesn't seen to be working as expected with the i18n strings.
Here is what I've done:

Created a i18n folder in the root of the project
Inserted two folders inside: en and pt
Created the following strings.xml inside then:

en/strings.xml

<?xml version="1.0" encoding="UTF-8"?>
<resources>
    <string name="user">User</string>
    <string name="password">Password</string>
    <string name="login">Sign In</string>
    <string name="forgot-password">Forgot my password</string>
</resources>

pt/string.xml

<?xml version="1.0" encoding="UTF-8"?>
<resources>
    <string name="user">Usuário</string>
    <string name="password">Senha</string>
    <string name="login">Entrar</string>
    <string name="forgot-password">Esqueci minha senha</string>
</resources>

In my index.xml I have a piece of code like this:

<View class="text-input-wrapper">
    <TextField id="loginInput" class="text-input-element" hintText="L('user')"></TextField>
</View>
<View class="text-input-wrapper">
    <TextField id="passwordInput" class="text-input-element" hintText="L('password')" passwordMask="true"></TextField>
</View>

Now, here is what is happening:
When I try to run it in a Nexus 5 4.4.4 Genymotion emulator it runs but it only shows translation in the first input and it completely ignores the string name, it always show the first item in the translation xml. If I have the first item as

<string name="password">Password</string>

The first input will show 'Password' written. The second will show nothing at all.

When I try to run on my device (Nexus 5 android L) the application doesn't even compile with an error on the java compilation process. No error information given.

Any ideas?
