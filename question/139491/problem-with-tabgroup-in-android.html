Problem with TabGroup in Android

I'm building a Ti mobile app for iPhone and Android and having an issue with tab groups.

var win = Ti.UI.createWindow();
    var    tabGroup = Ti.UI.createTabGroup({
            id: 'tabGroup1'
        }),
        win1 = Ti.UI.createWindow({
            backgroundColor: "red"
        }),
        win2 = Ti.UI.createWindow({
            backgroundColor: "blue"
        }),
        tab1 = Ti.UI.createTab({
            title: 'Tab1',
            window: win1
        }),
        tab2 = Ti.UI.createTab({
            title: 'Tab2',
            window: win2
        });

tabGroup.addTab(tab1);
tabGroup.addTab(tab2);

tabGroup.open();

win.add(tabGroup);
tabGroup.setActiveTab(0);

win.open();

That works just more or less as expected for iPhone simulator, but in the Android emulator it bombs with the following error:

[ERROR][AndroidRuntime(  332)] FATAL EXCEPTION: main
[ERROR][AndroidRuntime(  332)] java.lang.RuntimeException: Unable to start activity ComponentInfo{com.sandbox.app/com.sandbox.app.SandboxActivity}: java.lang.IllegalStateException: call to getView on a Window

This is latest Ti Studio (2.1 the 6/25/2012 build) and Ti Mobile SDK 2.1 GA (although I've also tried 2.0.2)
I'm building on Lion 10.7.3

TIA

    don't add the tabGroup to the window and don't open 'win' just open the tabGroup itself. tabGroup is its own 'window' and shouldn't be added to any other element.
