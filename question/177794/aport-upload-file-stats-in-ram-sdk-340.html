Aport Upload File Stats in Ram SDK 3.4.0

With Titanium SDK 3.4.0 and I think this happened in 3.3.0, specifically I'm using 3.4.0.v20140917120000 on iOS 8 if your upload a file and abort the file upload, the file stays in ram. If you let the file upload finish the ram is cleared no problem.

When aborting the upload and letting the file stay in ram it can and does cause the app to crash depending on the file size.

Has anyone else noticed this or found a way to work around it to get the file to clear when the upload is aborted?
