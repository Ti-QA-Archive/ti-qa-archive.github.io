Clicking annotations on iOS 7.1 Maps

Hi all,

I'm adding an annotation to a map, on iOS 7 application, and I want to check when the user clicked on an annotation on a map. When I click the pin, the annotation appears, but after when clicking on the annotation itself, nothing happens!

I'm using Titanium Studio SDK 3.2.1, using the iOS 7 Simulator on OSX Mavericks (Titanium Studio 3.2.1.201402041146).

The project uses Alloy… the view file (index.xml) is quite simple:

<Alloy>
    <NavigationWindow id="beachMapWindow">
        <Window class="container" fullscreen="false" title="SiUinde"layout="vertical">
            <LeftNavButton>
                <Button id="backBtn" onClick="clickBackButton">Back</Button>
            </LeftNavButton>
            <View id="mapview" ns="Alloy.Globals.Map" animate="true" regionFit="true" userLocation="true" height="Ti.UI.FILL" top="0" left="0" />
        </Window>
    </NavigationWindow>
</Alloy>

And the controller has the following code to detect the clicks on the annotation:

$.mapview.addEventListener('click', function(e) {
    console.log('detected click on the map');
    console.log(e.clicksource);

    if (e.clicksource == 'rightButton') {
        Ti.App.Properties.setString("placeId", e.annotation.placeId);
        Ti.App.Properties.setString("placeName", e.annotation.title);
    }

});

However, I can't seem to detect the clicks inside the annotations!

Is this something I'm doing wrong?!?

Thank you for your help.

  • Hola Carlos,
    You dont show the code used to create annotations in the provided snippets, but I guess you are not defining any right button for them. I.e.:

    var annotation  = Map.createAnnotation({
    [..]
    rightButton: '<an_image_for_your_button>'
});

    by setting this property a right button will be visible in the annotations and you'l be able to tap it and trigger the desired behavior.

    • Hi, quoting the docs

      "The click event includes a value, clicksource, which describes the part of the annotation that was clicked. Note that the possible values for clicksource differ between platforms.

      On iOS, if the user clicks on the pin or annotation, the clicksource is one of: pin, annotation, leftButton, rightButton, leftView, rightView, title, or subtitle. If the user deselects the annotation by clicking elsewhere in the map view, clicksource is null."

      It's a matter of choosing the correct 'clicksource' for the desired event. You have a condition matching a 'rightbutton' click in your event listener, so or add a right button or change the condition itself to "annotation", "title", "subtitle" … :)
