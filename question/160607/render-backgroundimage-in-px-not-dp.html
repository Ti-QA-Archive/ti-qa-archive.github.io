Render backgroundImage in px not dp

It appears that background images are rendered in dp not px units. This makes them larger than their actual size and blurry. How can I have a background image render at it's true pixel size using px units.

".stripedBackground":{
    backgroundImage: "/images/gold_stripe.png",
    backgroundRepeat: true
}

This problem is on iOS and Android. I'm using Alloy 3.1.3.GA

  • Try using the SIZE and FILL constants. For example:

    ".stripedBackground":{
    backgroundImage: "/images/gold_stripe.png",
    backgroundRepeat: true,
    height: Ti.UI.SIZE,
    width: Ti.UI.FILL
}

    If that doesn't help. Try setting those properties to the image's actual height/width values. Check here for further discussion on the UI layout system. Hope that helps.
