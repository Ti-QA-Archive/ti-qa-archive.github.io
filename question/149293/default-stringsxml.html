Default strings.xml

Hi,

I am developing a cross platform application that will support 17 languages including English. I have created a strings.xml file for each language and placed it under the proper folder such as ´i18/en´for English or ´i18/de' for German, as the documentation suggests. However how to set the default strings, when the device language is none of the i18n languages, is missing from the documentation.

So my question is what language would Titanium default to if the language of the device is none of the i18n languages? In iOS simulator, I see that the application somehow defaults to English, which is what I would want but I don't know if this is the expected behavior or not, since I couldn't find any means to configure the default language. Also, although defaulting to English is the behavior I want, I don't think that this is globally desired and feel that there should be a way to configure the default language, for example via placing a strings.xml to the root of i18n, just like the language suffix free platform&#x2F;android&#x2F;res&#x2F;values folder.

Thanks.
