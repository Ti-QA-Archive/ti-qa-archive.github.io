  • hmm… difficult to say from the logs. Looks like your parent activity is null when the framework calls the API setActivity() ( check https://github.com/appcelerator/titanium_mobile/blob/master/android/titanium/src/java/org/appcelerator/titanium/proxy/TiViewProxy.java). This normally means you are trying to do something with your Titanium Window even when it is not ready.

    Remember, in Android, the window open call is asynchronous which means if the window is a heavyweight window, it will go through the following Android activity creation states: create, pause, resume (see http://docs.appcelerator.com/titanium/latest/#!/api/Titanium.Android.Activity-event-pause) before it is ready to be used. So if you are doing something with the window (like adding or updating a view to it after creation), I recommend you do it after the "resume" event or on "focus"
