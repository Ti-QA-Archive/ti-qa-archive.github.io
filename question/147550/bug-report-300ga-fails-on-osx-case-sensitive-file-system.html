Bug report: 3.0.0GA fails on OSX case sensitive file system

(What's the place to submit a bug report? Didn't see one in the tabs on the website.)

I installed Titanium Studio fresh, created a new app, and could not build for iPhone with this error:

Error: ENOENT, no such file or directory '/Users/robb/Library/Application Support/Titanium/mobilesdk/osx/3.0.0.GA/iphone/Resources' 0 [ 'Error: ENOENT, no such file or directory \'/Users/robb/Library/Application Support/Titanium/mobilesdk/osx/3.0.0.GA/iphone/Resources\'',

I changed into that directory, and saw that there was a /resources directory. I renamed it to Resources, but then I got this error:

Error: ENOENT, no such file or directory '/Users/robb/Library/Application Support/Titanium/mobilesdk/osx/3.0.0.GA/iphone/resources' 0 [ 'Error: ENOENT, no such file or directory \'/Users/robb/Library/Application Support/Titanium/mobilesdk/osx/3.0.0.GA/iphone/resources\'',

I created a symlink resources -> Resources, and it now works.
