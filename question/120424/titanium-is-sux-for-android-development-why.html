  • I think its important to applaude Appcelerator team for what they have achieved. You create a window and a window appears on two (soon three!) platforms. You can create alerts, tableviews, buttons, textviews, checkbox, bullets, text fields, tabs, progress bars, and so much more! Also, most logic is 100% cross platform! Both platforms each have specific ui advantages to them, so no, not everything is going to be cross platform. While I do agree that Android has in the past been a little neglected, they are almost up to speed with the ios development. Sure you have to put if and else statements to differ from the two platforms, but have you looked at some sample code that have been provided in apps like the KitchenSink and tweetanium.

    Consider the following function taken from the tweetanium app:

        tt.os = function(/*Object*/ map) {
        var def = map.def||null; //default function or value
        if (typeof map[osname] != 'undefined') {
            if (typeof map[osname] == 'function') { return map[osname](); }
            else { return map[osname]; }
        }
        else {
            if (typeof def == 'function') { return def(); }
            else { return def; }
        }
    };

    and a sample usage of it:

        tt.ui.theme = {
        textColor:'#000000',
        grayTextColor:'#888888',
        headerColor:'#333333',
        lightBlue:'#006cb1',
        darkBlue:'#93caed',
        fontFamily: tt.os({
            iphone:'Helvetica Neue',
            android:'Droid Sans'
        })
    };

    Also note that they only have 1 set of code for both iphone and android for the tweetanium apps and kitchensink.

    So, I'm not sure what your specific issue as you don't say, but I've found titanium to be quiet easy to maintain 1 set of code.
