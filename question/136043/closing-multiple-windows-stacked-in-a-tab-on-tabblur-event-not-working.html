Closing multiple windows stacked in a tab on tab.blur event not working

Hi,

I've read multiple posts about closing windows stacked on a tab in a tabgroup. The only reliable solution seems to be to maintain a stack manually and closing windows (probably most recent window first). The problem that I now have is the following. Suppose I have two tabs with 2 windows in tab1 and 5 windows in the tab2 and the user selects tab2 and navigates to the 5th window, but then clicks on tab1. What I want to accomplish is that if the user clicks tab2 again, he starts at the first (root) window of this tab.

I thought I would be able to do this by attaching a "blur" event handler on the 2nd tab. However, this event is only triggered if the user is navigating away from the root window of tab2 and not from any subsequently opened windows. Is this a bug or should I try some other way to get this working?

How would I otherwise distinguish between these events: 1. user navigating back and forth between windows within one tab, and 2. user switching tabs?

It would also be very convenient if there was a method that would close any opened windows in a tab, except the root window (feature request?).

Please let me know what you think,

Regards,

Jeroen
