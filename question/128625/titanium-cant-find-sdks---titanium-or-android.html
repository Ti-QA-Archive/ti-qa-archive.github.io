Titanium Can't Find SDKs - Titanium or Android

I have Titanium Studio installed on a new Macbook Pro (running OSX Lion). All SDKs are exactly where they should be according to the docs. But no go. When I try to run a project, it says that the SDK can't be found.
When I try to update the Titanium SDK it says 'No such file or directory'.

Per the Appcelerator Wiki, the Android SDK is located in the Applications directory.

/Applications/android-sdk-macosx

The Titanium SDK is located in the Application Support directory.

/Library/Application Support/Titanium/mobilesdk

In the TiStudio preferences, I can browse directly to them and, while I'm looking at them in place, Titanium says that it can't find them. I reinstalled everything last night - same deal. It's probably something exceedingly simple, but I don't know what it is. I've checked folder and system permissions, versions, etc. And all the more frustrating, Titanium works perfectly on my other Macbook Pro and on my Windows laptop - no idea what I've done differently.

What'd I miss?

Thanks in advance! (he says optimistically)

  • Hi Eddie

    Note that the Titanium SDK is simply &#x2F;Library&#x2F;Application Support&#x2F;Titanium rather than &#x2F;Library&#x2F;Application Support&#x2F;Titanium&#x2F;mobilesdk. You will need to confirm this is correctly set in Titanium Studio.

    To attempt to resolve this, check that you have, at a minimum, all the packages shown in the screenshot of the Summary of Installed Android Packages section of the documentation.

    Install the latest continuous build, by selecting master from the drop down menu, right-clicking and copying the URL of the archive for your OS, and pasting it into Titanium Studio's Help > Install Titanium SDK from URL dialog. Note that you don't have to actually use this version of the SDK for your apps; it just needs to exist on your system.

    Make sure that Studio is configured with the location of the Android SDK, via its Preferences > Titanium Studio > Titanium dialog. See Configuring Titanium Studio with the Platform SDKs.

    Then restart Studio and try again.

    Let me know if it continues to be a problem.

    Thanks
