Retain Sub Window State in Tab Group

I would like my iOS app's tab's sub windows to retain their state when they are closed via the "back" button and then reopened. The best example of what I am trying to emulate is the iOS Messages application.

The main window shows a list of items linked to dynamically populated sub windows. When a sub window is opened, the user may change the scroll position of the chat thread, open the keyboard, begin typing a message, then click the back button, and reopen the sub window at a later time. When the sub window is reopened, the user expects the state to be identical to where they left it.

The obvious solution to this is to not close the window and instead leave it open in memory, however, I am unable to find out how to prevent the default "back" button functionality from closing the sub window, and simply hiding it instead.

Being that sub windows are dynamically generated and many could exist simultaneously, saving the window state in the prefs isn't very practical, nor would the UI feel smooth; some state-refreshing (like the keyboard being brought back into the view) firing after the window has come into the view.

Any solutions?
