  • Anyone else encountering a similar problem, the issue appears to be with passing in data parameters for the request when calling fetch.

    defining the URL parameter as: 'http://www.someurl.com'
    and doing, collection.fetch({ data: { user_token: 123 } }), will not work.

    However, explicitly setting the URL to: 'http://www.someurl.com?user_token=123' and then calling, collection.fetch(), will work.

    This is not ideal, since performing any other type of request will still use the same url. But at least I have found the issue.
