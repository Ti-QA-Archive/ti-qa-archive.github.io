Photo collection with ACL public write cannot be written to

Hi everyone,

I'm trying to access a photo collection by several users. Each user should be able to write to the collection (upload a photo to it). For that purpose I created an ACL with public_read and public_write set to true and attached that ACL to the collection. If I try to upload an image with the users who created the collection everything is fine - but another user can't upload to it. Every other user gets the error message: "Invalid collection id or collection cannot be modified by the current user".

Any ideas?

    Hi,

    I found the same issue, turns out its the way it's supposed to work. Collections are used as "photo albums for a user" - http://cloud.appcelerator.com/docs/api/v1/collections/create

    If you want to share multiple photos across multiple users I would suggest adding a custom field to all photos you upload with a similar collectionid you can then do a query search for.

    Dave
