    Hi

    First thing I would do is stop using the zIndex, you are better off removing that property and simply change the order of the views in the XML - you are just making work for yourself and adding a layer of complexity that could (and might be) going wrong.

    I do this often in apps, but you should not use 100% as it does not do what you think it does and you should use DP values and not percentages for the new position (at least in my experience.

    Setting a width to 100% says use the full width, when you add the view to a window that width is the full width - so it matches this. When you change the left to also be 100% it reduces the width of the view you are moving.

    You say you are moving windows, which in Android will not happen for heavyweight windows - which is the new standard, however your code suggests you are animating views - which is the right thing to do.

    When I animate a view out of position I make sure the width of the thing is set as a real width as I mentioned - so if I wanted the full devices width I would use this;

    var deviceWidth = parseInt(Ti.Platform.displayCaps.platformWidth / (Ti.Platform.displayCaps.logicalDensityFactor || 1), 10);

    This returns the full width in DP units. You could then use the following to tweak your code;

    $.view1.setWidth(deviceWidth + 'dp');
$.view2.setWidth(deviceWidth + 'dp');
var animateRight = Ti.UI.createAnimation({
    left: parseInt(deviceWidth * 0.2, 10) + 'dp',
    duration: 500
});
$.view2.animate(animateRight);

    • I have just noticed you have; layout="horizontal" in window declaration. This is very likely causing the problem as right now one will be NEXT to the other and will not be able to sit over the top of each other.

      Remove this to see if it at least overlaps, then work from there.

      Right now your views look like this, with views next to each other.

      +-------++-------+
+       ++       +
+   1   ++   2   +
+       ++       +
+-------++-------+

      By removing the layout property they will look like this;

      +-------+
+       +
+   1   +
+       +
+-------+

      Then animating they should look this;

      +--+----+
+  +    +
+  +  2 +
+  +    +
+--+----+

      Give it a go.
