Rotated views in box2d not realy works

Hi,
in my box2d project I'am using rotated views, because box2d implementation doesnt support polygones.
The object is the symbol of St. Pauli – it is a skull with four bones in cross order. The bones are rotated. ;-))
In parts are all properties of skull and bones.

for(var i = 0; i < parts.length; i++) {
    var part = parts[i];
    var left = position.x + part.x;
    var top = position.y + part.y;
    skullparts[part.name] = world.addBody(Ti.UI.createImageView({
        image : '/assets/skull/' + part.name + '.png',
        width : part.width,
        left : left,
        transform : Ti.UI.create2DMatrix().rotate(part.angel),
        top : top,
        height : part.h
        }), {
                    density : part.density,
                    friction : 0.7,
                    restitution : 0.6,
                    type : "dynamic"
    });
}
world.start();

Problem: after starting of world all bones are horizontal and fall down.

Any ideas?

Rainer

  • Box2DWeb supports polygons, and it is a pure JS implementation: Box2DWeb

    You might be able to pull off some interesting integration with the lower level b2Math classes and Titanium's transformation, but polygons are pretty fundamental to Box2D bodies and fixtures, and it's not clear why you're trying to do this.
