TableView Longpress, how to deselect previous selection?

I have an iPad/Android app in which we are implementing the TableView for a slide out menu (TOC). We are calling the same 'loadPage' function for events 'click' and 'long press'. The problem is that occasionally for whatever reason, when doing 'long press', the previous menu selection is still highlighted when select new one, and the link for 'loadPage' doesn't happen, but now both menu links are highlighted but not on right page. Should I explicitly be doing a 'deselectRow' call? Could this just be a clicking timing issue on the iPad maybe?

  • I possibly have taken care of the issue by replacing my TI.UI.createView object listener from a 'click' event listener to a 'touched' event listener.

    We were using:

    myWindow.addEventListener('click', loadPage);

    was changed to:

    myWindow.addEventListener('touchend', loadPage);

    What was happening was we have a slide out Table of Contents menu. Most of the time it worked fine, but occasionally after selecting a section and it opening up in the main window, you might try to select another section from the menu to open, and you would do a long press of the item and let go, it would highlight (change color) to indicate which section was selected, but it wouldn't open the content in the main window, and also the previous section would also still be highlighted. So it was as if it didn't do the loadPage, but it did see that it was selected. It would then take a couple of clicks on another section to start working again. Did try using the 'long press' action listener as well, but still sometimes didn't load and both would be selected.

    Titanium SDK Version: 2.1.4
