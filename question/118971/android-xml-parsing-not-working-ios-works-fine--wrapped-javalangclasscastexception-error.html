Android XML parsing not working, iOS works fine : Wrapped java.lang.ClassCastException error

Hello, Im porting my finished titanium iphone app to android and I am gettign errors on loading XML. The code works flawlessly in iOS but generates a runtime error in Android with the following message:

Wrapped java.lang.ClassCastException: ti.modules.titanium.xml.NodeListProxy (file:&#x2F;&#x2F;&#x2F;android_asset&#x2F;Resources&#x2F;app.js#79)

The code i have on line 79 is:

        Ti.API.info('Stamp XML ' + this.responseXML + ' text ' + this.responseText);
    var docStamp = this.responseText;
    var xmlDocStamp = Ti.XML.parseString(docStamp);
    var stamp = xmlDocStamp.getElementsByTagName("timestamp");

    var tempStamp = Titanium.App.Properties.getString('timestamp');
    Ti.API.info('Timestamp is ' + tempStamp);

    if(tempStamp == null){
        Ti.API.info('Timestamp is not Available');
        Titanium.App.Properties.setString('timestamp',stamp);

The error is happening when im trying to setString the timestamp.
Looking at the traces from the console, the xml being parsed from Android is different than from iOS. The "stamp" element is showing up as a [Ti.NodeList] in android, while its just a string in iOS.

This is the XML its loading:

<?xml version="1.0" encoding="UTF-8"?>
<root>
    <timestamp>2011-03-30 16:12</timestamp>
</root>

Ive tried modifying the dependency.json like others had mentioned to no avail. Tried running titanium sdks 1.6.1,1.6.2,1.6.3,1.7.0 android SDK 2.2 & 2.3, no difference.
