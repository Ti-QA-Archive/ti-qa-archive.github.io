Titanium Studio Debugger issue

Hi All

I read all the old posts regarding the strange behavior of the Debugger and I want to share my experience.

  • As a "Indie" licence I have the premium features of the Studio so the debug icon in ON!
  • I'm on a Mac and Studio version is the 1.0.0.20110527 1822
  • the SDK is the 1.7.0.RC1

1) I created a blank project in a brand new Workspace and added a few breakpoints
2) I launched the Debugger!
3) At the first run the breakpoint was ignored
4) At the second I received a message regarding the Debugger cannot connect etc…
5) starting and stopping the Studio a few time and full rebuild
6) than some crashes <terminated, exit value -1>

–than some light–

7) I removed all the breakpoints and I just added one "inside an event" and the debugger started to work a little

8) Breakpoint in the first windows generate crashes and the cursor does not follow the execution in other .js files.

–an experiment–

9) I recreate the Workspace "into" the Document folder and magically everything worked marvelous .

10) I put back the workspace in a different local location and the erratic behavior started again

11) I'm in the process to put back the WS in the Document folder crossing the fingers

Stay Tuned for more experiments!

If someone is so kind to give me any hint to "really" fix this I will appreciate

Vincenzo

  • Vincenzo

    It's important that you update your Titanium SDK to the latest Continuous Build 1.7.X to obtain all the fixes with the debugger. If you are updating it using the "Install Titanium SDK from URL" menu item, then confirm that it has properly replaced the previous version, by comparing the old version/date/hash with the new.

    Remember that the Titanium Studio Stable Updates should be installed for all users.

    With regards to the debugger, Nightly and Beta updates may be installed by Premium customers without affecting debugger functions. They may also be installed by Standard (non-paid) users who do not need the debugger functions.

    Note that, Beta updates may introduce important features and increase stability, but they have not been widely tested. Thus, in some circumstances, they may actually reduce stability.

    Nightly updates should be installed with caution by anyone in production, as they are considered very untested and unstable, and could potentially in rare cases delete your files. That said, I install Nightly updates, and they are usually fine.

    Please bear in mind that Titanium Studio is at a pre-release stage of its development, and hence it should be considered unstable until the official release.

    After installing the latest 1.7.X SDK, you should clean your build folders before launching the app.

    Also, bear in mind the issue described in Android emulator refuses to connect to Fastdev, even though it's running, which is the same for the debugger.

    Let me know how you get on.
