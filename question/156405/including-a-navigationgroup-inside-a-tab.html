Including a NavigationGroup Inside a Tab?

In my Alloy (iOS and Android) app I'm trying to implement a NavigationGroup for iOS devices. The main reason is to include "Back" buttons in the upper left hand corner for navigation. I'm running into issues when trying to include a NavigationGroup inside a tab:

policies_worklist.xml:

<Alloy>
    <Tab title="Work List" icon="policies_worklist.png">
        <NavigationGroup title="Work List" id="nav" platform="ios">
            <Window class="container">
                <Label id="policy1" onClick="openPolicy">Worklist...</Label>
            </Window>
        </NavigationGroup>
    </Tab>
</Alloy>

This fails to compile because: "[ERROR] : Tab must have only one child element, which must be a Window"

Am I going about this the wrong way? Is there a different solution to implement a navigation system within TabGroups/Tabs? Thanks for any help.

    Hi

    The reason you are running into to trouble is two fold;

    • You should NOT place a navigation controller inside a tabgroup
    • A tabgroup actually contains a navigation group

    If you open a window as a child of the current tab it will automatically gain all the benefits of the navigation group including the back button.
