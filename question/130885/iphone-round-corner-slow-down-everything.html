iPhone round corner slow down everything

Simply setting borderRadius to any value significantly slow down animation of all views on screen as long as the view with round corner is visible. Any workaround? Or it is a bug and only can wait for a fix in new titanium version? Thanks in advance!

    Is not a bug, is the way the UI works. Anything that implies transparency requires an image composing using the current element and the one under it.

    The more elements to combine, the slower the UI will be.

    If you are using this in a tableview, take care to use the same className for the rows with the same layout. It will speedup the things a lot.

  • I back Dan's take on the issue, and I add that sometimes it's more efficient to simulate the feature by using an image mask, i.e. an almost completely transparent png (with only the four corners painted with the background color) to position over the view that needs rounded corners.

    This trick is particularly efficient with tableviews, where even if you set the className property, the scrolling animation is heavily affected by the real-time compositing needed for handling rounded images/views.
