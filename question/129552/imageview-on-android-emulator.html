ImageView on android emulator

Hi all. I am new to Titanium studio and I'm trying to put an image in a emulator window, but when I pass the value where picture is, it's not showing on emulator, it shows only when I use image from internet. So is this a bug or am I doing something wrong?
I'm using windows 7, titanium studio sdk 1.7.5, Android platform 2.2.
this is an example what doesn't work as it should:

var IMAGE = "C:\worksheet\andro1\Resources\KS_nav_ui.png";

var win = Ti.UI.createWindow({
backgroundColor: '#fff',
fullscreen: true
});
var imageView = Titanium.UI.createImageView({
image:'http://www.liberalbaptistrev.com/wp-content/uploads/2010/04/compass.jpg',
});

win.add(imageView);

var imageView2 = Titanium.UI.createImageView({
image:IMAGE,

});

win.add(imageView2);
win.open();

Android emulator shows only the picture which is from internet…

    Fastdev is known to prevent imageview from showing image, try to disable it. In tiapp.xml

    <property name="ti.android.fastdev" type="bool">false</property>

  • Three things:

    1. One: As implicitly stated above, local paths are not appropriate for the ImageView image property.
    2. Two: FastDev is not the problem. But keep in mind that whenever you add a new image file to your project, you'll have to do a "clean" build.
    3. Three: Images should be placed in the appropriate density-specific folder under your project:
    • Resources\android\images\low\
    • Resources\android\images\medium\
    • Resources\android\images\high\
    1. and then referenced simply as &#39;&#x2F;images&#x2F;MyImageFile.png&#39;
