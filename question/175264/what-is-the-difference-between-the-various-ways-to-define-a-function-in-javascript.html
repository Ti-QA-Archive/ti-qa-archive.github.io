What is the difference between the various ways to define a function in javascript?

Hi, it is possible to define a function in various ways

var myFunc = function(){... do something.. };

function myfunc(){... do something.. };

var myFunc = new Function(...);

Are there any kind of benefits using one instead of others? Thanks for attention.

  • be aware that

var foo = function(){

  • To explain a little why it's this:

    foo();

var foo = function(){
    alert('bar');
};

    The function it's defined at runtime, that being said it will be available only after the script starts, so this is why it will trigger an error, foo not being yet defined.

    foo();

foo = function(){
    alert('bar');
}

    The function it's defined at the compile time, what it means that when the script starts to execute it has already foo defined.

    Keep in mind that in Alloy inside the controllers you need to use the second version.

    The new Function it's a different story, it's more like an eval

    new Function("return 1+2;");

    So basically you evaluate a string.
