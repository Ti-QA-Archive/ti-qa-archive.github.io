ul li ul li in webview font-size problem (only on iPhone)

I'm having a weird problem with html and css in a webview.

In short, there is a nested list like:

<ul>
<li>item 1</li>
<li>
  <ul>
  <li>item 2</li>
  </ul>
</li>
</ul>

<style type="text/css">
body{font-size:13; } 
ul li{ font-size: 13; } 
li li{ font-size: 100%;  }
</style>

I'm running the latest SDK and iOS 6 on both devices. On an iPhone 4 the second list item has a font-size which is bigger than the first list item. On the iPad, all is fine.

Is there a difference in webviews on both platforms, even when the iOs version is the same?N

  • Even though this isn't a Titanium issue, why don't you do

    li { font-size:100%; }

    This will set all "li" elements.
    Or if you want to explicitly specify the second "li" element you can do

    ul li ul li { font-size:100%; }
