Mac address issue from Apple

  • Application type: Ipad
  • Titanium SDK: 3.0.1
  • iOS SDK: 5.0
  • Titanium studio: 3.1.0.201304151600

From Apple:
"MAC addresses in iOS 7 August 22, 2013 If your apps use the MAC address to identify an iOS device, the system will return the same static value for all devices running iOS 7. Please update your apps to use the identifierForVendor property of UIDevice. If you need an identifier for advertising purposes, use the advertisingIdentifier property of ASIdentifierManager."

But current, my ipad app use Mac Address to track user log in, one user only use one ipad device. With this issue, what are things i should be change to this feature work without any problem?

Notes: Use Mac Address for this feature is a bad solution, but Apple don't publish any device's identify, so i used it as temporary instead.

Thanks,
VyPhan

    Hi

    You should be using installId. This does not provide a unique id for the user but for the life of the app being installed. If this app is installed on a different device for the same user then it will have a different id at that time, if the app is uninstalled and reinstalled it will have a different id.

    Apple are making sure that privacy is much more important than apps ability to track a person without their opt it. If you must track a user and the installid is not sufficient you will need to use a login process by the user to start using the app. This is generally seen as an intrusion to far if the only reason to login is so you can track someone though.
