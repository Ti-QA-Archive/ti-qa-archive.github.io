  • Lets run through a couple scenarios, because the same error can mean different things.

    1. Did a binary actually get created in {project_root}&#x2F;build&#x2F;android&#x2F;bin (app.apk)?
    2. Be sure you have appropriate sdk+vms available for the android API you are targeting (downloadable from within the android sdk manager) and the required 2.3.3 API. (this should have been caught by the studio setup)
    3. What version of the android tooling do you have installed? (there was a problem in the past with android changing tooling location and breaking the build scripts)

  • There are "so many freaking download buttons and folders" because Titanium uses and requires the native development toolkits. This means, for Android development, you will need:

    • Oracle's Java Development Kit (JDK) – download from Oracle
    • Android Development Kit (ADK) – download from Google
    • Android API version 10, plus the associated Google API add-ons for API version 10 (you can install additional versions of the Android API, but you need at version 10 as a base-minimum) – download using the android tool included with the ADK
    • Node.js – though this should be installed as part of Studio's install routines
    • The Titanium tools, including the command-line interpreter (CLI), Studio, and the Titanium APIs – all installed as part of the Studio installation routine

    For legal/licensing and practicality reasons, we cannot distribute all of this software. In some cases, we must point you to the original vendors to obtain. Additionally, each requires specific configuration steps that we can't always automate.

    Be sure to read the setup guide, especially the Windows-specific section and the Installing Android Packages with Android SDK Manager. Also check out the CLI setup guide, especially make sure that you can run titanium info at the command line without errors. You might want to run titanium config and step through all its prompts to make sure the CLI is configured correctly.

    Please keep in mind that you would need to install many of these same components to do Java/Android development. You wouldn't need node, and you'd substitute Eclipse and the ADT plug-in for Studio but the rest would be similar. As complex as it is, our setup is not that much more difficul than Google requires. (Translation: it sucks either way!)

    One last note, your app is named "Halsten's App" – the apostrophe and space might be causing problems. Have you tried another app named without those special characters?

    The fix is to go into the Android SDK Manager and install "Android 3.0 (API 11)". Make sure the app uses emulator "Google APIs (Android 2.3.3)" and "WVGA854". I assume there's a Titanium bug because you have to install a higher API level (3.0) than is actually used (2.3.3).
