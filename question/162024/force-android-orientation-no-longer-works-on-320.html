Force Android Orientation no longer works on 3.2.0

I have an app build oroginally on the 2.xmobile SDK and orientation locking worked. As I am updating this and using the latest sdks I cannot lock orientation any longer. I have tried everything I can see in thesee forums. Please help

I am using the following
3.2.0GA Mobile SDK
Host OS: Mac OS X
OS Version: 10.9.1
OS Arch: x86
JRE Version: 1.6.0_65
Titanium Studio Version: 3.2.1.201401081514
Android SDK Version: 4.4.2

My app.js is as follows

// this sets the background color of the master UIView (when there are no windows/tab groups on it)
Ti.UI.orientationModes = [Ti.UI.LANDSCAPE_LEFT,Ti.UI.LANDSCAPE_RIGHT];
Ti.UI.orientation = Ti.UI.LANDSCAPE_LEFT;
Titanium.UI.setBackgroundColor('#fff');


var win = Ti.UI.createWindow({
    fullscreen:true,

  • Hi Laney,

    to force the orientation of android you must set the attributes in manifest section of your tiapp.xml:

    <activity android:configChanges="keyboardHidden"
                    android:name="org.appcelerator.titanium.TiActivity" android:screenOrientation="portrait"/>

    You can do this on app level or for every activity. Hope this helps you.
