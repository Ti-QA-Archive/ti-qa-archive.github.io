  • Hi Jorge – there's a workaround for now, using a hidden webview (haven't tried the 'hidden' part, buta webview has access to gyroscope.)

    You can use a webView to load a local htm5 page that gets (html5) deviceorientation gyroscope data.. (perhaps make the webview transparent, or not visible?) You can then pass the deviceorientation to Ti event listener. Your html5 page will should fire off data to a Ti.App event listener…

            // on deviceorientation event...
        Ti.App.fireEvent('orientationData', { data : {
            "alpha": alpha,
            "beta": beta,
            "gamma": gamma
            }
        });

    Your Ti event listener will look something like:

    Ti.App.addEventListener('orientationData', function(e){
    Titanium.API.info(e.data.alpha);
    Titanium.API.info(e.data.beta);
    Titanium.API.info(e.data.gamma);
    onGyro(e.data.alpha, e.data.beta, e.data.gamma)
    });
