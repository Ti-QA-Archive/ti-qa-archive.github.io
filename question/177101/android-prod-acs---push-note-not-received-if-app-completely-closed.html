Android Prod ACS - Push note not received if app completely closed

Hi,

I'm using TiSDK 3.3.0 and OSX 0.9.4. I have observed the following on Android OS 4.0.4, 4.1.2 and 4.3 (so far).

I have setup my app to use ACS for Android Push notifications. Using the development portal it was working fine. I decided that prior to going to the Play Store I would test out the push notifications via the production portal of the ACS and I have found a differences in operation.

When I send a notification via the portal (or my own php backend) to the device and my App is either open or listed in the "Recent apps" screen, the push notification is received flawlessly. Even when the device is locked. If I swipe the app out of recent apps, then the push notifications is no longer received. I didn't have this behaviour when I was using the development portal.

I would really like to know why this is occuring in Prod and not Dev ACS portals and also will this operate like this when I submit to the Play Store. It is rather annoying and sort of flys in the face of why you would enable push notifications in the first place,

Key setup points:

1) I changed the "development" keys to be the same as the productions keys as explained here - ACS Dev vs Prod

After doing this I was able to see my device registered successfully within the Production portal of ACS

2) In order to be able to send push notifications to individual devices, for each device id, I’m generating a channel with the same name. So when I deliver a push notification all I have to do is target the channel and set the "to-ids" to "eveyone". I am also registering them using "subscribeToken" like so:….

Cloud.PushNotifications.subscribeToken({
    device_token: deviceToken,
    channel: deviceToken,
    type: Ti.Platform.name == 'android' ? 'android' : 'ios'
}, function (e) { .....success / failure tests .... });

3) I am actually sending push notifications via my php backend using this technique ricardoalcocer.com

As pointed out though, sending push notifications from php or from the console have the same effect. If the app is not open, or doesn't appear in the recent apps list, the notification is not received.

So, any genius ideas out there?

Kind regards,
Brad.
