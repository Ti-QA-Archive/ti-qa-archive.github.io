Ti SDK 3.0 - Black Screen caused by labels inside a TableViewRow firing click events twice

We're having an issue where users periodically see a black screen that cannot be escaped in our application without a force quit. We spent a lot of time trying to debug this and eventually found that it's an issue with labels inside table view rows. For example, if clicking the label is supposed to open a new window, the open event is fired twice in rapid succession. When the user then closes the window, two close events on the same window will be called which will result in a black screen.

I filed a JIRA report (link) but I was wondering if anyone had any insight or guidance? We're dead in the water waiting for a fix since we re-wrote a lot of code to be 3.0 compliant and cannot release with this issue.

  • While waiting for this bug to fix, probably what you can do is to code a workaround - 'open' event should not be fired very rapidly.. say 0.5 seconds. If second event happened after 0.1s from the first, then ignore it.
