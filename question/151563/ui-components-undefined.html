I would like try and see in the "Mobile Web Preview in Browser" the behaviour of all Titanium UI components as listed in http://docs.appcelerator.com/titanium/latest/#!/api/Titanium.UI.

In the below app.js code I commented all components which raise the "Undefined" at creation time.

My questions are:

  1. How can I test these UI components?

  2. Can I assume that a component undefined in the Mobile Web Preview in Browser will be not executed correctly on real device? Or there is the chance that a UI component undefined in Mobile Web Preview in Browser will be executed correctly on real device (of course it's "mandatory" for me test on real device any code, but if I know in avance that a UI Component undefined in Web Preview is not implemented everywhere, I spare a lot of time.

  • Accepted Answer

    The UI components you have commented out are all ones supported on specific native platforms. For example, buttonbar, coverflow, dashboarditem, etc. are all iOS-only controls. This means you cannot use them in a WebView. You'd have to create a native project and view them in the appropriate simulator/emulator.
