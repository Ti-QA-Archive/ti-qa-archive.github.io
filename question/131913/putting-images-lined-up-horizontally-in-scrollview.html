  • When you use the layout property, you're telling child views to position themselves relative to the last view added. This differs from the normal behavior, which is absolute positioning based on the parent view.

    Try removing the left property from the child image:

    
                var fbImage = Ti.UI.createImageView({
                      image: photos.data[i].images[2].source,
                      //left: -100,
                      top: 0,
                      height:100
                });

    • I have a couple other ideas for what this could be. Try making the size of your scrollView much larger than it is, so that you can find out for sure whether the images are being added.

      Sometimes when you use a layout and the view doesn't fit, it'll get "wrapped" and not show up at all.

      This might also be a bug (this bug: https://jira.appcelerator.org/browse/TIMOB-3256 ?)

      Here's how you can find out if this is a bug or not. Comment out everything in your app.js (for now) and try the following experiments.

      1. Find out if horizontal layouts work at all for you by adding a view (with layout: horizontal) to a window and then adding regular views to it. Make sure they all have different background colors, and that the container view is much larger in both height and width than the child views.

      If that works…

      1. Find out if it's just a problem with image views. Repeat the test above, but use imageViews instead of regular views. Make sure you load images from your Resources directory so you can be 100% sure they're loading properly.

      If that works…

      1. Try the test above using remote images or, as in your example code, images from Facebook.
