How to store app resource content in 'Documents' folder

IOS: 6.1
Titanium: 3.0

Is it possible to have my iOS app store the content files (html, images, etc) in the IOS 'Documents' or the 'Private Documents' folder, vs. inside the xxx.app bundle? I see my database files automatically go to the 'Private Documents' folder, and each app creates a 'Documents' and 'Private Documents' folder. The reason I want to do this is that I have a custom module that sets the NSFileProtectionKey attribute value on all of my resources to force security on them, but everything under the app bundle is read only so it is failing.
I use entitlements to set this value for my database which works, but not for other app bundle content.
