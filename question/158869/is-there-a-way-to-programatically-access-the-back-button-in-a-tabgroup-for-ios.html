Is there a way to programatically access the 'back' button in a TabGroup for IOS?

I know that for IOS when you open a new window in the current tab (like Ti.UI.currentTab.open(newWin)) you get automatically a button for the previous Tab. My question is, can I access that button programatically? or it can be accessed only automatically because I haven't found a way to access it programatically in Titanium.

In Iphone native development can be done by calling UINavigationController&#39;s popViewControllerAnimated method (here is a link about that).

I tried as an alternative to close the current window but it's the root window of the tab and I can't close it. What I'm trying to accomplish is a logout functionality in a tab, so I want to be able to set a global variable like Ti.App.loggedIn = false and then to go back to the previous Tab.

PS:. I also want to know if I can access the 'More' Tab of a TabGroup programatically. On IOS it's generated automatically a 'More' Tab if you have more tabs that can be showed in the main window and I also want to access that Tab programatically and I don't know how.I tried with tabGroup.setActiveTab(number) but I didn't find the proper number for the 'More' tab.

Thank you !

  • I'm not entirely sure what you're trying to do since you can close the window as people have said, or attach a handler to the window close event to do something when it's clicked (albeit after the event has occurred).

    However, if you want to say, do something after the button is clicked but before closing the window then you could create your own back button, a click handler to handle the close of the window and then use the win.setLeftNavButton(button) method on the window.

    This would then replace the default button, allow you to intercept the click and do something before you close the window.
