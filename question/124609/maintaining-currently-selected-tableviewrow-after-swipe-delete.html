Maintaining currently selected TableViewRow after swipe-delete

Say I have a TableView that is editable=true with 3 rows in it and row #3 is currently selected (and highlighted). If I swipe row #1 to bring up the "Delete" button and delete the row, I'm left with 2 rows in the TableView, but neither is currently selected. I would like to be able to make row #2 (which was the currently selected row #3 before the delete) to remain selected.

My use case is that I'm using a SplitViewController for an iPad app and the TableView is in the left master window. Keeping the right detail window sync'ed with the currently selected row after deletion is a bit of a challenge.

Does anyone have an example of how this is done exactly?
