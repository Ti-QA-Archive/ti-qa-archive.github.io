Device Caching App installed via Titanium?

Hello, I am starting a new mobile project using the 'Master/Detail Application' template. There are two views: master and detail. I don't really need the detail view, and I have removed every reference to the detail view. The results are as expected in the emulator. Trying to run the exact same code directly on 2 android test devices gets 2 different results: one that matches the emulator, and one with code that is an hour or two old (at least). Despite running 'clean', and manually checking the build folder (always is empty after a clean). I am convinced the device is holding on to an older image. Is there any other explanation? I have tried removing the app and re-installing twice, but that does not help either. I've tried upping the version string (out of desperation). How to really, really uninstall an android app or force a new version?

    add the following to your tiapp.xml and see if the problem disappears:

    <property name="ti.android.fastdev" type="bool">false</property>

    If this works, your build times will be slower.
