Can't call Titanium.API from specific JS file.

Hey guys,

I just created a new file, called HTTPClientController.js. It's basically this:

function HTTPClientController(data) {

    this.url = data.url || null;
}

HTTPClientController.prototype = {

    loadFeed: function() {

        Ti.API.info('Im in!');

        //Do Stuff
    }    
}

However, it keeps crashing on the Ti.API.info line, with the following message:

[ERROR] Script Error = -[__NSCFDictionary setObject:forKey:]: attempt to insert nil value (key: API) at HTTPClientController.js (line 15).

I can trigger Ti.API.info on other files with no problem, but in this one I get this error. I also tried moving it to the function constructor, but it's still happening.

I tried this to validate my Titanium object:

if(Titanium) alert('Titanium Exists');

And my alert gets called. But when I try this:

if(Titanium.API) alert('API Exists');

I get the error again.

Does anyone have any ideas on this?
Seems like lots of new warnings have been showing up and this is an error I've never seen before in Titanium.

Any help will be much appreciated!

Thanks!
