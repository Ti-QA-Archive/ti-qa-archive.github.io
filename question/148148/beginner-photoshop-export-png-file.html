[Beginner] Photoshop export PNG file

I use the photoshop to export the image : File>Scripts>Export Layers to Files> PNG-8

And then , When I positioned a image in titanium , the image size and position is totally mismatched in the simulator. The image seems larger than that in photoshop. It is nothing wrong with the iOS version and hardware config.

Can someone help me to figure out this problem? Thanks !

  • Hi

    Firstly you do not need to set the DPI that high, 72 DPI is fine.

    The second problem is WHERE you are using the image. The iPhone 5 does have a resolution of 640 x 1136 (portrait), but this is only important if you have a full screen window with no; status bar, navigation/title bar or tab bar.

    Also the images you supply are actually based on two resolutions with a naming convention. Which even if you are not targeting devices less than the iPhone 5 you HAVE to provide if you want the app in the app store.

    Example naming conventions;

    • back.png [320x568]
    • back@2x.png [640x1136]

    back.png and back@2x.png are the same image but with two resolutions. You only ever reference back.png and iOS will automatically use the higher resolution image if available.

    This is because the iPhone uses a device width of 320 when laying things out. You can address every pixel (640) if you need but for most apps that is not required and inhibits the ability to support non-retina devices. Retina devices will gain the correct imagery when supplied with a @2x suffix and fonts will look much better automatically.

    If your image is going to be displayed as the background to a window and remain fixed in-place, you must determine the internal space.

    So subtract (if you have each);

    • status bar [20/40]
    • navigation/title bar [44/88]
    • tab bar [50/100]

    So the height would be;

    • iPhone = 480 - 20 - 44 - 50 = 366
    • iPhone Retina = 960 - 40 - 88 - 100 = 732
    • iPhone Retina Tall = 1136 - 40 - 88 - 100 = 908

    Assuming the original filename is back.png these "should" be named as;

    • back.png
    • back@2x.png
    • back-568h@2x.png

    Keep in mind that support for back-568h@2x.png is not available until Titanium SDK 3.1 which is due out within the next week.
