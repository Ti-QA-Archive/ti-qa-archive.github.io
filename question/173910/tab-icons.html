    • This may well end up being something as simple as your folder management - you do not need to and should not try to manage your apps image files for densities - at least not here for these things (photos etc play all you like - but usually only for external content).

      I would start very simply and place your images like this;

      {appname}/resources/images/res-mdpi/tab_search_grey.png
{appname}/resources/images/res-hdpi/tab_search_grey.png
{appname}/resources/images/res-xhdpi/tab_search_grey.png
{appname}/resources/images/res-xxhdpi/tab_search_grey.png

      I have renamed your image so that you do not need lots of tiny folders for images - I have moved the images into density folders (the links below give lots of details to this) and given the file name a simple prefix - this way you can quickly find the image in the folder. I have also made sure everything is lower case. Whilst this is not a requirement - having a single case standard means you can never go wrong.

      You do NOT manage your image size swapping. Whilst the again the links below have lots of details here - a quick logic to the folders above.

      • mdpi = 100%
      • hdpi = 150%
      • xhdpi = 200%
      • xxhdpi = 300%

      So if your baseline image is 36x36, then hdpi = 54x54, xhdpi = 72x72 and xxhdpi = 108x108. So you simply drop the same image with a different wxh in each folder - but with the SAME file name. A quick tip - not required but useful, image files should be divisible by 4. This way the scale above can easily scale. Technically you only need a factor of 2 - but if you support ldpi (most do not now - it was 75%).

      I have provided some very detailed answers about how to manage images and density folders for iOS and Android and manage them so they are cross platform (should you need it). Rather than rewrite all the information - review these two to see exactly how you can simplify your life - only problem is they are VERY detailed - so you will learn much more than you need for this - but everything you read will be useful.

      • How to achieve same layout for Android and iPhone using Titanium
      • images in Android (also iOS)

      Give these a go and see what happens - chances are everything slots into place.
