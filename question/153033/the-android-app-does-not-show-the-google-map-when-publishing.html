  • Hi

    As your problem appears to be Android related and you have a cross platform app, can you confirm you are using TWO different sets of map code as the Android platform now requires Google Maps v2 which is now NOT the same as the iOS creation code.

    • iOS code;
    var mapview = Titanium.Map.createView({
    mapType: Titanium.Map.STANDARD_TYPE,
    region: {latitude:33.74511, longitude:-84.38993, 
            latitudeDelta:0.01, longitudeDelta:0.01},
    animate:true,
    regionFit:true,
    userLocation:true,
    annotations:[mountainView]
});
    • Android code;
    var MapModule = require('ti.map');
var mapview = MapModule.createView({
    mapType: MapModule.NORMAL_TYPE,
    region: {latitude:33.74511, longitude:-84.38993,
            latitudeDelta:0.01, longitudeDelta:0.01},
    animate:true,
    regionFit:true,
    userLocation:true,
    annotations:[mountainView]
});

    The main differences are these two lines;

    var MapModule = require('ti.map'); // a new require as this map is a MODULE outside of Ti
var mapview = MapModule.createView({ // NOT Titanium.Map.createView

    For v2 maps you sign them as before but the process requires a SHA1 encryption and not the MD5 version.

    If you are not seeing the map and only a blank background with a location dot - then the problem is the API key. Which means either you have the wrong type of key, the key is not entered in the tiapp.xml file or your new SHA1 key app id does not match that in your actual app.
