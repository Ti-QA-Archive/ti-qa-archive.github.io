App Icons and Splash Screens

I zipped up the demo icons and splash images from my project:

find Resources/android Resources/iphone -name '*.png' -print | zip images -@

And the designer gave me back a file with those images with the icons and pages for our app, all with the same names, all the same dimensions and everything.

So I replace the images in the project with the new images and now I get no icons or splash screen on iphone, and android errors out with:

ERROR] libpng error: Not a PNG file
[ERROR] ERROR: Failure processing PNG image

These files look fine in finder and safari, so I highly doubt the images are not png files.

I 'rm -rf build' a couple times just in case, but no dice.

Any ideas what is going on here?

    • Todd, they should be simply copied over from your Resources directory.

      After you opened and resaved them using a text editor, did you then process them with the convert program? If so, maybe it is this that is creating problems.

      Try simply creating some new images using your photo editor (just images filled with color, with the same dimensions of the previous images) and testing whether they work. If so, then the problem is with the format of your files.

      Cheers

      — commented June 3rd 2011 by Paul Dowsett

    • indeed the assets I was given were not valid png files. ImageMagick's identify command gave me the info I needed to figure out how to convert them. If I'd have used that first I wouldn't have needed this post.

      — commented June 8th 2011 by Todd Wade

    • Thanks for the update, Todd.

      For those who need to use ImageMagick identify, and example of the command in use is as follows:

      > identify chat.png
> chat.png PNG 29x23 29x23+0+0 8-bit DirectClass 1.55KiB 0.000u 0:00.000
