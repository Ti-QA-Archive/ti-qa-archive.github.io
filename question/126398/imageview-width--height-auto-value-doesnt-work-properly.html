ImageView width / height 'auto' value doesn't work properly

Just submitted this as a bug, but wanted to throw this out to everyone as well.

Mobile SDK Version 1.7.2

Loading an image into a Ti.UI.ImageView with a property of height/width set to auto sizes the image correctly, but the size of the ImageView will remain the full size of the original image.

Ex.

Original Image = 864 x 768
desired scaling = width 200, dynamic height

Code

Titanium.UI.setBackgroundColor('#000');
var win = Ti.UI.createWindow();

var view = Ti.UI.createView({
    bottom: 0,
    left: 0,
    top: 0, 
    right: 0
});

var img = Ti.UI.createImageView({
    image: 'pic.png',
    width: 200,
    height: 'auto',
    border: 1,
    borderColor: '#f00'
});

view.add(img);

Ti.API.info('Img Height: ' + img.height);
Ti.API.info('Img Width: '+ img.width);

win.add(view);
win.open();

Console

Note width will be set correctly as declared by the API.info output, however, height will be the original height of the image.

iPad Simulator

Note that the image will appear to be sized correctly albeit the positioning may be off.

Adding a border to the image reveals the true size of the image.
