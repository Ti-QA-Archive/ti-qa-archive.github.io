How to disable keypad from displaying when calling a URL

I am new to Titanium and mobile development in general. I've made considerable progress on my first app, but I have one problem for which I can find no references on the web. Any time I make a call to a URL, the soft keypad on my device (Nexus 7) displays for a moment before the page is displayed. It appears to happen regardless of the technique I use to call the URL:

Example 1:

 tableview.addEventListener("click",function(e) {
           var w = Ti.UI.createWindow({title:doctitle});
           var wb = Ti.UI.createWebView({url: e.row.url});

           w.add(wb);
           w.open({modal:true});
 });


 $.news.add(tableview);

Example 2:

function dispAthletics() {
    var win = Ti.UI.createWindow();
    var webview = Ti.UI.createWebView({
    url: 'http://www.govandals.com'});
    win.add(webview);
    win.open();
}

Any ideas on how I can keep the keypad from displaying? Apart from that, the app works exactly as desired.

  • Hello Brian,

    Use this Window Property : Titanium.UI.Window-property-windowSoftInputMode..

    like,

    // use bitwise OR to combine two settings for the windowSoftInputMode property
var softInput = Ti.UI.Android.SOFT_INPUT_STATE_HIDDEN;

function dispAthletics() {
    var win = Ti.UI.createWindow({
        windowSoftInputMode: softInput
     });
    var webview = Ti.UI.createWebView({
           url: 'http://www.govandals.com'
     });
    win.add(webview);
    win.open();
}
