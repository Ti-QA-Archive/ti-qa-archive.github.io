Facebook Login Dialog Issues

Greetings,

I've been working on integrating Facebook with my app and have run into a troublesome issue when testing on my iPhone device. Once I press the Facebook Login button, the login dialog pops up and apparently one of the text fields is focused automatically, which causes the keyboard to launch prematurely. The Facebook login ends covering up the keyboard and making it inaccessible. If I close and reopen the FB login dialog, the text fields are unresponsive and I can't input my email and password. Any ideas or suggestions on how to deal with this issue would be greatly appreciated. Thanks in advance!

  • Currently, this happened with Titanium SDK 1.7.3 on an iPhone 4 and 3GS (both on iOS 4.3). The usecase from how I understand it would be: User wants to login to Facebook through my app. User presses Titanium's Facebook.LoginButton and Facebook dialog pops up for user to input email and password to log in to FB. Once logged in, user can can access facebook friends or post on his wall, etc. Problem here is that the Facebook dialog that pops up gets in front of the iPhone keyboard. And if I close the window, then try to open again the dialog, I can't make my touch cause the fields to focus and show the keyboard. After hours of trying and leaving it alone (not even restarting or force shutting the app worked) it decides to work as expected. Has anyone else run into this problem?
