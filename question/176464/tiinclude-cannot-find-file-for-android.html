    Hi

    No magic involved. I have an explanation for you and then some news that means you need to take action sooner rather than later.

    There are certain 'reserved' names that cannot be used otherwise they cause conflicts with other actions the OS performs. This looks like one of them. Often during the compile the console will flash up with a warning about any found - however as they can change - it might not know about this one. It might also be it does and you simply never spotted the warning.

    Now the news that says you need to take action;

    • Titanium 3.3.0: Ti.include() deprecated

    For those who do not know what deprecated means; this function, feature, property or service is now marked for removal. This normally occurs in the next release following this one. Once deprecated this method will no longer be available and you cannot use Ti.include() form that point going forwards.

    Ti.include() is an old relic that should have been retired years ago. It also most certinaly should NOT be used with Alloy as its use can create unstable apps simply by what include is technically doing inside a framework.

    Normally I would say you need to start moving towards Alloy and CommonJS instead. The first yuo already have - chances are the second as well.

    The library you are using will need to be updated to a CommonJS version if it will be usable in the future beyond Ti.SDK 3.3.
