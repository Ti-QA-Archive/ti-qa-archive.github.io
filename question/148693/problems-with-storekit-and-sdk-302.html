Problems with StoreKit and SDK 3.0.2

I can't make StoreKit module work with SDK 3.0.2. I always get "Couldn't find module: ti.storekit" when the app launches in the simulator or device. If I simply change the SDK version in the tiapp.xml to 3.0.0 it works flawlessly. I'm using the StoreKit module version 1.6.2. I notice that the 3.0.0 SDK shows the message "Detected third-party native iOS module: ti.storekit version 1.6.2" in the build time but the 3.0.2 doesn't. My app is a Titanium Classic app (not Alloy) and the StoreKit module is in the modules folder inside the app folder. Any suggestions to make it work?

      <modules>
        <module version="1.6.2" platform="iphone" >ti.storekit</module>
    </modules>

  • Accepted Answer

    The final solution to this problem is to add the platform to the xml tag. In 3.0.0 this is not needed, but in 3.0.2 it seems that it is. So try this:

    <modules>
        <module version="1.6.2" platform="iphone" >ti.storekit</module>
    </modules>
