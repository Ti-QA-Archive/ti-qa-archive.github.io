Second Storekit.purchase call results app crash.

I've got quite the same problem with storekit as there

Although the suggestions there do not help me.
First time in app purchase goes well. Second time I try to purchase it'll crash the app right away when calling Storekit.purchase

My code:

Ti.Storekit.purchase(product, function (r) {
        if (r.state == Ti.Storekit.FAILED) {
            setTimeout(purchaseFailed,500);
        }
        else if (r.state == Ti.Storekit.PURCHASED || r.state == Ti.Storekit.RESTORED) {
                purchaseSuccess();
            }
    });

I tried to add info statements everywhere but it'll crash immediately after purchase call.

This crash happens on both, simulator and device.

I wonder what could cause this. I've ran out of options to try and read all posts here about the subject. I could submit my app like this (as first purchase will work), but it would be awesome if it would work every time. Please help.

SDK: 2.0.1 | Storekit: 1.5 | iOS iPad v.5
