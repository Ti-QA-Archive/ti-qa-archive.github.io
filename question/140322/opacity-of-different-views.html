Opacity of different views

I have two views. Parent view has opacity 0.8 and child view has opacity 1.0 .

I add some labels in child view and then add the child view to parent view.

In iPhone the child view's opacity also becomes .08 while android is working perfectly fine.

  • In 2.1.0GA the child view is inheriting the parent view's opacity. In 2.0.2 it is not inheriting.

    I want that the child view should not inherit the parent view's property.

    I want two different view's with different opacity one top of another?Could you please tell me if it is possible?

    • You can add them to the same parent window/view, so not one to the other. This way there is no child view and no opacity is inherited.
      Makes sense?

  • The normal behavior is that the child view to inherit the parent's opacity, so in fact the iOS is behaving the right way.

    The fact that on android the opacity of the child is not affected is a bug that has been fixed in a CI version.

    https://jira.appcelerator.org/browse/TIMOB-5799

    What SDK are you using?
