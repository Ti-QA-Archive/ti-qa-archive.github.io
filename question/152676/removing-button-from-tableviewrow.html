Removing button from tableviewrow

I've recently been updating an app of mine for iPhone 5 while also adding new features, and I've noticed a bug has emerged that wasn't there.

I currently have tableviews that, when a row is swiped, a button is added. If the button is clicked, or the row swiped again, the button is removed. While this seems to still work, in that the tableviewrow reverts to it's original size, the button is still partly visible "behind" the row. So while the app seems to be removing the button in behaviour, it is still visible which is confusing to the user.

The current code I'm using to remove the button is:

var neededRow = tableRows[rowIndexNumber];
neededRow.remove(neededRow.children[1]);

(Note neededRow.children[0] is a view containing text).

Any ideas why this is no longer working for iOS 6.1?

(Titanium SDK version 3.1.0.GA)
