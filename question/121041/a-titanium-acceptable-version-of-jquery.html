A titanium-acceptable version of jquery?

I have a 2-page iPad app that is a conversion from a 2-page web site that did all it's animations, ajax, show/hide operations with jquery. When compiling for the simulator Titanium tried to parse all of jquery in advance (and failed at 67% or something, mostly due to missing semicolons), so I renamed my jquery file to jquery.txt, and called for it from the webview as jquery.txt, and that seemed to work.

Now Titanium does not try to evaluate (and fail at xx%) jquery at compile time, and in the simulator the webview uses jquery.txt perfectly.

Next I created my .ipa file (no easy task!) and sent it to the client who has a physical iPad to try it out. For him, jquery does not load, and therefore none of the animations or other functions work.

Does anyone know why it doesn't work, and if it's because on a physical iPad it has to be named "jquery.js", then is there a version of jquery out there that will survive the Titanium's pre-processing?

Thanks in advance if anyone knows this!

  • Some additional info:

    For me, I have the jquery 1 folder down from the html, it did not seem to cause a problem. But I did put the name back to jquery-1.6.1.js, and as you may be able to tell from the name, it is NOT the minified version.

    Happily Titanium has no issues with jquery-1.6.1.js (not minified) and it loads in the final webview application and runs just fine.

    So, short answer = "use the non-minified version and all will be well." :)
