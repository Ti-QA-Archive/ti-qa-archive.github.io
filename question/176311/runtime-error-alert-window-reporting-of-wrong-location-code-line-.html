Runtime Error alert window reporting of wrong 'Location' code line ??

Hi,

I noticed an error on 'Runtime Error' Alert window that show a wrong code line at 'Location' that shows [1,69]. The 'Message' and 'Source' seems OK. This has caused problem in code debugging.

This was OK before I updated to the latest Ti.SDK 3.3.0. Even now, I changed back to Ti.SDK 3.2.3, it is still happening. I debug my code on android device with SDK 4.4.2.

Why is it so? Is there anyone facing the same problem? Appreciate help on this matter.

Thank you.

    Alloy or classic?

    Alloy takes your code, builds a classic and you need to go look at it in the Resources/{platform}/alloy/controllers folder.

    If its a classic, if you are deep in a callback, line numbers may not match. If the code has been minified then it might have the wrong line numbers.
