  • More than likely you only need to force a full rebuild. In Ti Studio, use menu Project > Clean, then try again.

    The reason this happens is Ti only includes the modules (classes) you use when it builds a project. This keeps the app size smaller and leaner but not including extraneous stuff. However, to keep debug / test cycles faster, a normal rebuild will just keep the modules previously included.

    So this error often indicates you are using a new module for the first time in your code. A full rebuild (after a clean) forces it to include the module.
