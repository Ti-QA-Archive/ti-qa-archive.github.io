  • To expound on Matt's answer, the app id is used by iOS in conjunction with the provisioning profile used. When an iOS application is installed or run, the system verifies there is a valid (unexpired) profile which matches the app id in the program.

    If you are targeting iOS devices, you must create and register the app id in the provisioning portal section of your developer member center at Apple. This can either be a single specific program (e.g. com.appmaker.appname) or a wildcard (e.g. com.appmaker.*). Using an id with a wildcard allows the same profile to match multiple programs.

    The disadvantage to wildcards is they cannot be used for things like in-app purchasing or push notifications.

    You may want to refer to Apple explanation once logged into your developer account at Apple. If the link doesn't work once logged in, follow this path: login > iOS Provisioning Portal > App IDs
