Alloy: Style Elements Based On Model State Change

I need to make UI changes based on the state of the model and collections which create the UI elements. For example, I have a list of clients that need to be picked-up. This list is a set of TableViewRows created from a collection, and who's model attribute is set to a model instance from the collection. When a client is picked up, the driver clicks a "picked-up" button on that client instance. This event changes the state of the model so that its 'is_picked_up' field is set from false to true.

I would like to update the state of the UI to reflect the updated state of the model, for example grey out the appearance of elements who's underlying model's 'is_picked_up' field is set to 'true'. Additionally, as the TableView is built from the associated collection, I would also like to style each row based on the same condition, so that when the window first loads all rows who's associated model's 'is_picked_up' field is set to 'true' is also appropriately styled.

Thanks!
