Android Push Notification re-launch application

  • Mobile Application
  • SDK 2.0.1.GA2
  • Android 4.0.1
  • Google Nexus One
  • Mac OS X 10.7
  • Titanium Studio, build: 2.1.0.201206251749

Hi!

My application get relaunch each time I get push notification with the application fully closed. If the application is in the background it's ok.

Does anyone have encounter this behavior?

Easy case:

  1. I fully close the application
  2. I receive a push notification
  3. I open the notification center
  4. I click the message
  5. The application launch
  6. A message appears saying: "An application restart is required"
  7. I click continu, but the window that should open when I receive a push doesn't open.

In the dalvik debug monitor, I get this line that could help:

Android issue 2373 detected (missing intent CATEGORY_LAUNCHER or FLAG_ACTIVITY_RESET_TASK_IF_NEEDED), restarting app. com.illicocard.illicocard.IllicocardActivity@418c8ca0

How can I set this intent filter with the UrbanAirship module?

Thank you!
