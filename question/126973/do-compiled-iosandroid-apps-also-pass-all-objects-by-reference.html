Do compiled iOS/Android apps also pass all objects by reference?

In JavaScript, objects are passed by reference, so creating new pointers to objects is not an issue.

When Titanium compiles my scripts for iOS and Android, are objects still handled this way?

  • Simple answer Yes.

    There is a JavaScript engine built into the compiled app. which is why it is slightly bigger than a native app.

    And as you correctly say, objects are passed by reference, which is a good practice, and is handled well by titanium.
