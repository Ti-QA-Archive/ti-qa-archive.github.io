Programmatically showing/hiding of master view in splitwindow

Hi,
in a map appI want to hide the masterview in start. If user clicks on annotation in map the masterview should appear.
Here my concept:

if (Ti.Platform.osname === 'ipad') {
    var masterwindow = Ti.UI.createWindow({visible : false});
    masterwindow.add(Ti.UI.createTableView({}))
    var splitwindow = Ti.UI.iPad.createSplitWindow({
        detailView : mapwindow,
        masterView : masterwindow
    });
    splitwindow.open();
    Ti.App.addEventListener('app:showmaster', function() {
        masterwindow.show();
    });
}

And in map widget:

self.addEventListener('click', function(_e) {
    Ti.App.fireEvent('app:showmaster');
});

But (other as I hope) the master view is on start not invisible – it is black. Any idea to solve?

  • Rainer, unfortunately, the SplitWindow control is a fixed layout – meaning the size and placement of the two views are not changeable. Your best option is to create your own version of split window.

    I'm not sure if it will meet your needs exactly. But I've posted a sample from an old project of mine at https://gist.github.com/skypanther/5446027 You would need to extend it to support the hiding/showing of the master pane.
