Deleting a table from Application data directory

Hi! I'm using SQLite databases on my app. Depending on the country of the user a different database is installed on the device and everything works perfectly. The problem is that when the user logs out of the app and logs in again but with another country (This means I need to overwrite the tables installed before), I am not able to delete the tables that have been already "installed" on the application data directory. I tried dropping the table, deleting the table.sql as a file… but I haven't manages it to work. Is there a way to "uninstall" the tables and start again installing new tables from scratch?

  • try this one

    var path = Ti.Filesystem.applicationSupportDirectory;
var f = Ti.Filesystem.getFile(path, 'database', <table.sql>);
if(f.exists()){
    f.deleteFile();
}
