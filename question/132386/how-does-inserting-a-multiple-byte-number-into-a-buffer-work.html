How does inserting a multiple-byte Number into a Buffer work?

Going from the Titanium Intro Series, I'm trying to send a series of 4-byte integers into a buffer. The example code and API seem to imply that buffers are byte-accessible, and my concern is that by inserting an encoded Number into the buffer as such

// Encode a 4 byte integer into position 10 using BIG_ENDIAN encoding
var buffer = Ti.createBuffer({ length: 100 });
Ti.Codec.encodeNumber({
    source: 0x3456789a,
    dest: buffer,
    position: 10,
    type: Ti.Codec.TYPE_INT,
    byteOrder: Ti.Codec.BIG_ENDIAN
});

we are trying to fill 8 bits of memory with 32 bits! Can anyone shed some light? My hope is that that code above would fill positions 10, 11, 12 and 13 sequentially, or that the buffer actually has 4 bytes available in each position.

  • If anyone's curious, it turns out that we are indeed filling positions 10, 11, 12, and 13 sequentially with 0x34, 0x56, 0x78, and 0x9a. encodeNumber returns the position of the next buffer position, in this case 14.
