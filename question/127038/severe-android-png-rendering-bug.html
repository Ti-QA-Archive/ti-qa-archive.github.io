Severe Android PNG rendering bug?

Hi,

While working on a cross platform app I've run into a problem with the visual rendering of the graphical UI elements on the Android version. I've made a new test on a clean project and it looks like there might be a serious problem in the way PNG and transparency are looking on Android.

The problem is very obvious in this test image: TI PNG Problem

The test is made up of a Window with a background PNG, and two child views with unscaled background PNG's with both solid and semi-transparent content. The PNGs are 32bit (RGBA) and looks fine in iPhone simulator. On Android both the emulator and device (HTC Legend) are affected.

Am I doing something wrong, or is this a (rather severe) Titanium bug?

var win = Ti.UI.createWindow({
    backgroundImage:'bg.png',
});

var shadow = Ti.UI.createView({
    backgroundImage:'shadow.png',
    width:'179dp',
    height:'179dp',
    top:'25dp',
});

var glow = Ti.UI.createView({
    backgroundImage:'glow.png',
    width:'189dp',
    height:'189dp',
    bottom:'15dp',
});

win.add(shadow);
win.add(glow);
win.open();

  • I've found a "solution". In my simulator all png images with gradients looks "blocky".

    I've tried the application on my Samsung Galaxy S2 and most of my images looks perfect on the device. Some images didn't, but I found out why… all images that didn't have any transparency looked bad, while all images that had at least one pixel of transparency looked good. So I made sure that atleast 1 pixel in all my images had a transparent pixel, and voila it looks good on the device, but it still looks bad in the simulator.

    • Here's some code that illustrates the problem (would like black background to be transparent)

      //create object instance
var self = Ti.UI.createWindow({
    title:'Products',
    exitOnClose:true,
    navBarHidden:false,
    backgroundColor:'#ffffff'
});

var webview = Titanium.UI.createWebView({
    backgroundColor: 'transparent',
    url:'local.html'}); 

self.add(webview);
self.open({modal:true});



var button = Titanium.UI.createButton({
width:  180,
height:  50,
bottom: 100,
left: 100,
title: 'Save View ToImage()'
 });

 button.addEventListener('click',function(e){
//save your view TOIMAGE()    
var imageData = webview.toImage();  


//add a temporary imageView and use the view's toImage() data blob as the image property
var tmpImageView = Titanium.UI.createImageView({
    width: webview.width,
    height: webview.height,
    image: imageData,
    top: 20,
    left: 20,
    borderColor: 'transparent',
    borderWidth: 0,
    backgroundColor:'transparent'
});
webview.add(tmpImageView); //you must add it to your window!

//save the temp imageview TOBLOB()
var saveImageData = tmpImageView.toBlob();

//now you can save and it works!
var file = Titanium.Filesystem.getFile(Titanium.Filesystem.applicationDataDirectory, 'test.jpg');
file.write(saveImageData); // this is where the magic happens!

Ti.API.info('size = ' + file.size);

      });

       webview.add(button);
