Renewing of images in coverFlowView

Hi *,
in my app app comes the list of issues and images of every image via xhr from net. A picker select the volume of magazine.
What is the best way to refresh the coverflowview?

win.remove(coverFlow);
var coverFlow = create...

or

coverFlow.images = [newimages];

The problem is the flashing of view, because the changing needs time to get images from net…

Best Regards
Rainer

  • I would say the second option. Removing and re-creating a coverFlow would be memory intensive and I think more visible to the user (in terms of the 'flashing' you describe). Plus your code on top would just remove the view from the window, not necessarily remove the view from memory. If you haven't already, I'd suggest looking at 'coverflow_replace.js' in KiSink. It basically changes the coverFlowView.images property from one array to another and there is no 'flashing' - the images are just replaced. Just make sure you don't use the exact syntax you have for the 2nd option above - again see the KiSink example where it's like:

    newArray = [];

// Put some code wherever to add to the newArray by populating it...

// finally get down to using the newArray..

coverFlowView.images = newArray;
