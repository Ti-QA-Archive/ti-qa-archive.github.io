ScrollView.getChildren() return nothing

Hi there,

I am working on iOS. SDK v1.8.2

On window's focus event, items are added to the window's scrollView using scrollView.add() under a for loop.

If invoke scrollView.getChildren() right after this for loop, it return nothing. but if invoke it a moment later, it can return the children array correctly. I tried in debug mode, setting breakpoint and run it line by line, it is normal.

So does it mean ScrollView.add() take significant time to process?

Anyone encounter the same thing? Anyway to get around this?

Since there is no event like 'complete' for ScrollView. There is no way to tell when did the add() really completed and effective.

Thanks in advance.

            y : 0.5
        }

    tp += 160;
    scrollView.add(view1);
}
var t = true;
var c = 0;
while(t) {
    if(scrollView.getChildren().length == 100) {
        t = false;

    }
}
win.add(scrollView);
