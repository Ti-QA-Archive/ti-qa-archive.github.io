Localization defaults without using hints, and an alternative JSON i18n solution

Firstly, this is not really a question, but more of a solution to a problem I encountered when using Titanium's localization features.

The problems I had with the Titanium.Locale i18n solution are as follows:

  • I really don't like the way there is no support for a default language strings.xml file, meaning that I need to use hints (i.e. L('key','default value')). I hate this solution for two reasons: firstly I need to store actual text in my app files where logic is container so all changes to copy down the line require changes to the various strings.xml, but also need changes to wherever that default is used in code file. Secondly, I duplicate copy, so if I have a button with the word Cancel on it used in various places in my application, I need to have L('cancel', 'Cancel') numerous times, hardly a DRY solution.
  • I simply don't like using XML for storing resources if I can help it. Firstly it does not support object hierarchies so it very quickly becomes unmanageable with large amounts of text, and XML is just ugly where all my other code is in Javascript / JSON.
  • The Titanium solution is buggy, see http://jira.appcelerator.org/browse/TIMOB-4191
  • There is no way of overriding the current locale even if you need to.

So I've come up with a simple JSON based solution, which I believe solves all the issues above. I'd love your feedback and comments if you have any. Hope you find my json.i18n solution a help.

Go and take a look at https://github.com/mattheworiordan/json.i18n-for-Titanium-Mobile for the small library, and a working example of how it has been implemented.
