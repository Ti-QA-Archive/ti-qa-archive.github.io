App failed to resume in time

We are having a lot of users who are experiencing big problems with our app. The app is using background location and the crash logs say "App failed to resume in time". As far as I can understand, there is something in the Titanium SDK (3.1.2) that is causing this, since the javascript processing should reasonably not disturb the wakeup process. So far, everyone who has had problems have been using iOS7.

I am not sure how to file this possible bug, since we have not managed to produce a small code example that securely can reproduce this problem.

Since there is no support email address on Appcelerator that I know of, I have no other option that posting this problem here.

  • I will have to reply to my own question: One thing that can go wrong (and what is probably the problem in my particular case) is to do lots of operations on UI elements while in background. Those operations are queued up and executed as soon as app resumes. If there are many such things to do, the app might not resume in time, and thus crash.

    There are probably more things that can go wrong, but this is probably a quite common mistake.
