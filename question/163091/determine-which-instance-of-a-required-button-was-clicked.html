Determine which instance of a <Require>'d button was clicked

If I have the following button view (btn_add_item.xml):

<Alloy>
    <View class='button_container'>
        <Button id='add_item_btn' class='add_item_btn' onClick="addItem" >
            add
        </Button>
    </View>
</Alloy>

And create 2 instances of it (index.xml):

<Alloy>
    <Window id='main' class="container">
        <Require id="add_notebook" src="btn_add_item" type="view"/>
        <Require id="add_note" src="btn_add_item" type="view"/>
    </Window>
</Alloy>

How can I determine which instance was clicked in the addItem() function in btn_add_item.js

function addItem(e) {
    Ti.API.info(e.source);
    Ti.API.info(e.source.id);
}

    I'm assuming that once you know which button is clicked, you would have to include conditional logic (if/else) to handle each button click differently. In that case, would something like this work (better?)-

    In index.xml

    <Require id="add_notebook" src="btn_add_item" onClick="addNotebookHandler" />
<Require id="add_note" src="btn_add_item" onClick="addNoteHandler" />

    note the onClick on the <Require>

    Then add the click handlers in index.js

    function addNoteHandler(e){
    // we know that note button is clicked    
    Ti.API.log("addNoteHandler");
}

function addNotebookHandler(e){
    // we know that notebook button is clicked    
    Ti.API.log("addNotebookHandler");
}

    Then, to make all this work, add the trigger in btn_add_item.js

    $.add_item_btn.addEventListener('click', function(e) { 
    $.trigger('click', e); 
});

    This triggers the click even for the parent view (index) to receive and the hence the addNoteHandler() and addNotebookHandler() will be called on respective button clicks.

    If this solution works for your case, would you please accept the answer / leave a comment so I know that I'm on the right track (or not).

    Good luck.
