Using retina version of image for ipad

I have an universal iphone/ipad app and would like to load the @2x version of the image in the ipad version. Is this possible? Would be great so I wouldn't have to create a third version of the image just for ipad use.

Trying to set image property of imageView to 'picture@2x.png' doesn't work. 'picture.png' works of course but the retina version isn't loaded..

  • the way I do it, but this may not be the correct way. Is to have a images file with all the images loaded into variables (global). Then I find out the screen resolution and adjust the height and width down depending on screen width.

    Or I load the variable to be used with the actual image within a function called at run time, then use that variable in the image or background options.

    The @2 does work for the splash screen.

    The solution I have works for iPhone, iPad and android resolutions.

  • Actually I have no idea why it didn't work to set the image url to "image@2x.jpg".
    Yesterday (6 weeks after i posted the question) I tried this again, and had no problems doing it this way!

    So, for future readers, I would suggest creating image.jpg and image@2x.jpg and setting the url with something similar like this:

    image:&#39;image&#39; + (Titanium.App.ISIPAD ? &#39;@2x&#39; : &#39;&#39;) + &#39;.jpg&#39;,

    ISIPAD is not a real variable, I created it for ease of use:

    Titanium.App.ISIPAD = Ti.Platform.osname == 'ipad';
