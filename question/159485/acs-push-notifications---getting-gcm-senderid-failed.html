ACS push notifications - getting GCM senderId failed

I am trying to get the token id using the code provided in the Appcelerator documentation - I've set up GCM, got the senderId and API key, set them in the Apccelerator PN settings, and finaly - in my app i'm using this code to retreive the token:

var CloudPush = require('ti.cloudpush');

var deviceToken = null;

// Initialize the module
var CloudPush = require('ti.cloudpush');
CloudPush.retrieveDeviceToken({
    success : function deviceTokenSuccess(e) {
        Ti.API.info('Device Token: ' + e.deviceToken);

    },
    error : function deviceTokenError(e) {
        alert('Failed to register for push! ' + e.error);
    }
});

But all i see in the console is
11-14 17:59:06.916: W&#x2F;com.appcelerator.cloud.push.CCPushService(10356): Getting GCM SenderId failed. No response area in returned JSON. Will try again in 5 seconds.

After 5 retries it throws an error

11-14 17:59:11.930: E/com.appcelerator.cloud.push.CCPushService(10356): Getting GCM SenderId failed. Max retry time reaches.

Any ideas what this might be?

    SOLVED - So the problem was that the credential in the APP settings were not saved properly (the Google Cloud Messaging (GCM) API Key and SenderId). I doublechecked, re-saved and the problem is solved.
