Studio System 65 Error on iPhone Build

Hi

I am getting a strange error. Latest TI Studio and SDK 1.7.3 on Mac.

If I build the app for the iPhone device it works, but when I then build it for the emulator I get a System 65 error.

I resolve this by doing a clean, or deleting the build/iphone directory contents. But I think this may well be a bug.

Also in the latest version of studio my clean project isn't working :(…

Anybody else getting this?.

  • I had this as well and found in the end it turns out the problem was with a custom Info.plist file. I had used custom fonts in my app, and had copied a Info.plist file but instead of putting it in my root directory, had dumped it into Resources.

    Removing the plist file from Resources and putting it back into the root directory, then doing a clean and re-build on my project and it was fixed straight away!

    So if you are using custom fonts/plist files those would be well worth checking.
