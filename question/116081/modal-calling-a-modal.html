Modal calling a modal

Hello Everyone,

I'm hoping someone can help me out with my problem.

My app is made up of a tab group with 4 windows. These windows use the 'url' property to get the code for each of the windows. Once one of the windows is loaded the user can click on a button to add a new record. This causes a new window to load as a modal. The modal is nice because it provides the 'back' button without to much effort.

On this modal if the user clicks a button it will open another modal. When the second modal is launched it causes the first modal to automatically close. This is my problem. I wanted the second modal to overlay the first modal and when the second modal is closed then the first modal will be displayed. This is similar functionality to the adding an event to the calendar on the iPhone.

I've tried everything I can think of to prevent the first modal from closing when the second is opened, but I can't stop it.

Is there an issue of having a window modal call another window modal? If so is there a work around available?

I'm using titanium sdk 1.6.0, but I've tried other versions and they all have the same problem.

Thank you for help.

  • Well, isn't that the way it goes? Once you post something you find the answer.

    I believe I've figure out the functionality that I wanted without using modals.

    This came from finding some similar functionality in the Kitchen Sink app.

    The trick is to use Titanium.UI.currentTab.open(w,{animated:true}); The animated:true is just for fun. This gives me the functionality I was looking for without having to use modals.
