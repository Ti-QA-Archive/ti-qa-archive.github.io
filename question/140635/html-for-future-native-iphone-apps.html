HTML for future native iPhone apps

Hi!

I have a background in web programming which I haven't used for some years, but I am now considering making an iPhone app. I saw this interesting alternative to create apps, and since I already know web programming it would be a smooth transition for me. However, there seem to be some disadvantages with these kinds of apps. Can't just Apple forbid all other ways of making apps other than the native iOS programming language, and then all of a sudden this knowledge is worthless when it comes to making "native" apps? It would be interesting to hear your opinion on this matter. Thanks in advance.

  • Viktor, there is no way to do a promise related to any technology you want to use to develop apps when is about Apple. They can do this even with objC, is their company, their product, and they do take this kind of (stupid) decisions sometimes.

    The way Titanium does the apps is approved by Apple because Ti is respecting exactly the guidelines and rules imposed by this so it should be no issue in the future.

    Related to your web knowledge, you must understand that Ti is not using HTML or JS or CSS to make the app. The code you write in javascript is split in 2 parts - the UI is converted in native components and the "background" part is executed using the javascript core that exists on the os.
    The difference between Ti and Phonegap is that you get a native app (UI, sql, hardware access, etc) that is driven by a javascript logic. Phonegap is a webview to wich are exposed some hardware functions.
    For Ti a button means a real objC button while for Phonegap it means some html styled element.

    So from all your prior web knowledge the only thing that will help it will be JS (and by this I mean pure JS and not jQuery or whatever other framework).

    The transition is not easy, you need to really understand how the way the mobile apps works which is different from a web page (is more close to the way a desktop app works) but I can tell you that it worth every single sleepless night or hardtime you might have (I've been there myself, a former web dev converted in a fulltime mobile developer) :)
