Require in Alloy project - seems to fail

Ok this is an easy one. Since working with Alloy, I've had issues with requiring files. So this is my folder hierarchy (relative to Resources):

app/
  |_ assets/
  |_ controllers/
  |_ lib/
 ....etc

I have a controller (in the controller folder), called win3.js. I have library I made, in my lib directory, called icons.js.

app/controllers/win3.js:

var icons = require('/app/lib/icons').icons;

I've tried "/app/lib/icons", "../lib/icons", "/Resources/app/lib/icons".

[ERROR] : Script Error = Couldn't find module: ./app/lib/icons.

The paths for require should be relative to the Resources directory, so shouldn't app/lib/icons be expected? What simple fact am I overlooking?

Thanks!

    Everything in "lib" (and "assets") gets dropped into the Resources folder at compile time. This means you would reference your icons lib like this:

    var icons = require('icons').icons;

    Check out the Alloy guides on conventions for more details: http://docs.appcelerator.com/titanium/latest/#!/guide/Alloy_Concepts-section-34636240_AlloyConcepts-ConventionoverConfiguration
