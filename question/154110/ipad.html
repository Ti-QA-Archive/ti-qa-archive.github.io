    Hi

    The hint text will not disappear until the user enters one or more characters - this is by design so that someone filling out a form can still read the hint next when they get to the next field. This makes the user experience much better and is the expected behaviour for iOS.

    clearOnEdit will only clear text entered by the user or by you programmatically, as the hint text is a system accessibility value and not the value of the text box - this has no affect.

    You could programmatically remove the hint text on the focus event and put it back on the blur event this would take away a useful and expected interface element.

    • // assumes text field value = txt
txt.addEventListener('blur', function (e) {
    e.source.setHintText('');
});
txt.addEventListener('focus', function (e) {
    e.source.setHintText('Hello again');
});

      For this to work the user will have to be able to focus away from the current text box, so you will need at least two text boxes or one text box and one button as a minimum.
