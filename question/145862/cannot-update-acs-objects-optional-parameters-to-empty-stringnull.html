    • The PHP class that I use is a slightly modified version of this https://github.com/benedmunds/PHP-Appcelerator/blob/master/appcelerator.php

      The admin log tab says:

      Request Name    PUT /v1/events/update.json
Response Status    200
Parameters    { "name" => "NAME", "place_id" => "ID", "details" => "", "event_id" => "EVENT", "custom_fields" => "{"acts":[{"name":"ACTNAME","url":"google.com"}],"genre":"Acoustic","coupon_text":null}", "duration" => "21600", "start_time" => "2012-12-19T13:00:00+0000", "key" => "APPKEY", "controller" => "events", "action" => "update", "version" => "v1", "format" => "json" }

      Notice that the coupon_text fields is null, yet details is empty string. If i var_dump the array just before I send it via curl I get ["details"]=> NULL.

      To change empty strings to null I use this function

      <?php
private function _toNull($data){
        $temp = array();
        if (!empty($data)){
            foreach($data as $key => $val){
                if ($val === ""){
                    $temp[$key] = null;
                } else {
                    $temp[$key] = $val;
                }
            }
        }
        return $temp;
    }
?>

  • To anyone who is interested, with the help of my boss we created a 'hack' so to speak.

    Instead of setting empty strings to null I used a null byte "\0", after that the return values from Appcelerator was a UTF encoded null byte "\u0000". So before I pass the results back from our Appcelerator class, I use

    // Get output from curl
$output = str_ireplace('\u0000', '', $output);

    I'm OK with this work around but I dont understand why Appcelerator can't accept null values for non custom fields parameters.

    Thank you to Wei for your help in resolving this.
