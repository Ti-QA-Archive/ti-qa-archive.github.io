    This is actually a good question!

    I experienced some struggles when creating apps for multiple platform and multiple formfactors. I found that it is quite easy to create a single code base for either phone and tablet devices of a one platform or one code base for both platforms but only one formfactor.

    Anyway, one for all is hard but possible. There are different approaches to do so:

    When you use Alloy,

    • use the platform and formfactor attributes in your XML layouts to dispatch views to several targets
    • use platform-specfic styles and themes

    when working on Titanium classic,

    • bootstrap your application like the templates for Single Window, Master/detail and Tabbed Application
    • in the ui directory create folders for common, handheld and tablet ui files
    • require the needed views / windows from there
    • little differences can be branched with if-else within these files
    • create platform specific templates for your UI components and include them in common views

    and last but not least: TEST, TEST, TEST :-)
