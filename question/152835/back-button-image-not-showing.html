    • You cannot use the image in place of the button but you can use an image on the button in place of the text.

      To test this code create a new "classic default app" and replace the contents of app.js;

      var win1 = Titanium.UI.createWindow();

var win2 = Titanium.UI.createWindow({
    backgroundColor: 'red',
    title: 'Red Window',
    backButtonTitleImage: 'KS_nav_ui.png'
});

var win3 = Titanium.UI.createWindow({
    backgroundColor: 'blue',
    title: 'Blue Window',
    backButtonTitleImage: 'KS_nav_ui.png'
});

var button = Titanium.UI.createButton({
    title: 'Open Blue Window'
});
button.addEventListener('click', function(){
    nav.open(win3, {animated:true});
});

var nav = Titanium.UI.iPhone.createNavigationGroup({
   window: win2
});

win2.add(button);
win1.add(nav);
win1.open();

      The documentation is incorrect in the suggestion it is only tabGroups - more correctly it is anything that utilises the navigation controller, both the tabGroup & navigation controller do.

    • You can use a button a view, or almost all other objects in the left (or right) button locations;

      // button example
var btn = Ti.UI.createButton({
    image: 'KS_nav_ui.png'
});
win3.setLeftNavButton(btn);
// image example
var img = Ti.UI.createImageView({
    height: 30,
    image: 'KS_nav_ui.jpg',
    width: 100
});
win.setLeftNavButton(img);

      If you use a button - it will act as a button, which means you may not get the display you are seeking. You can use an image and it will act as an image, which means you can get it to look anyway you wish.

      Remember both a button and an image can have click listeners - so it does not matter which you choose.

      You can also have a view with child views, buttons and images (an anything else) so you can combine things as you need.

      The key here is the leftNavButton property (when defining windows) and setLeftNavButton when setting the property afterwards.

      You will have to manage the closing of the window yourself if you use this method as the automatic method is overridden by replacing it with this technique.

      This is how you manually close the window in the same way that the automatic way did.

      btn.addEventListener('click', function (e) {   
    nav.close(win3);
});
