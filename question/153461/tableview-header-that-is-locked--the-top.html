    • Then you can simply create a view with a fixed height and place the table underneath it using the same fixed height as the top value, this allows the table to leave enough space at the top for it. By using the height value of Ti.UI.FILL the table will use the entire space less that of the top value.

      Example XML;

      <Alloy>
    <Window id="win">
        <View id="header">
            <Label id="label1">
                First Label
            </Label>
            <Label id="label2">
                Second Label
            </Label>
        </View>
        <TableView id="table">
            <TableViewRow title="Apple"/>
            <TableViewRow title="Bananas"/>
            <TableViewRow title="Carrots"/>
            <TableViewRow title="Potatoes"/>
            <TableViewRow title="Cod"/>
            <TableViewRow title="Haddock"/>
        </TableView>
    </Window>
</Alloy>

      Example TSS;

      "#win": {
    backgroundColor: "#fff"
},
"#header": {
    height: 50,
    width: Ti.UI.FILL
},
"#table": {
    top: 50,
    height: Ti.UI.FILL,
    width: Ti.UI.FILL
}

  • I figured out how to make a heading that does not move. You create the TableView but then immediately under it create a table view section and create the additional rows. The heading is then locked @ the top…like this:

    <TableView top="0">
    <TableViewSection id="heading">
            <HeaderView>
                <View id="headerView1" borderColor="black">
                    <Label text="I'm a heading" />
                </View>
            </HeaderView>
    </TableViewSection>
    <TableViewRow>

    </TableViewRow>
</TableView>
