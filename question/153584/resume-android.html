    • When you press home button then state will change to pause and it will stop current activity. To understand this have a look at this example:

      var win = Ti.UI.createWindow({
    title : "test",
    backgroundColor : 'white'
});
win.open();
Ti.Android.currentActivity.addEventListener('destroy', function(e) {
    Ti.API.info(">>>>> Destroy");
});
Ti.Android.currentActivity.addEventListener('pause', function(e) {
    Ti.API.info(">>>>> Pause");
});
Ti.Android.currentActivity.addEventListener('resume', function(e) {
    Ti.API.info(">>>>> Resume");
});
Ti.Android.currentActivity.addEventListener('create', function(e) {
    Ti.API.info(">>>>> Create");
});
Ti.Android.currentActivity.addEventListener('start', function(e) {
    Ti.API.info(">>>>> Start");
});
Ti.Android.currentActivity.addEventListener('stop', function(e) {
    Ti.API.info(">>>>> Stop");
});
Ti.Android.currentActivity.addEventListener('newintent', function(e) {
    Ti.API.info(">>>>> New Intent");
});

      Put this code in newly created project. and install it on device. After application launch press home button and open task manager. You can see that your application is still running in device. It will be running till system or device doesn't need more memory to run.

      May be your application will be still running in the background after three days, but It is device dependent. If your device or system does not allow background application running then It will not let it go in background. It will ask you are forcefully close that application which is going in background, basically this type of functionality is used in low memory devices.

      But System has authority to kill any background Process at any time when it require more memory to run, basis on priority it will kill background processes.

      for more reference you can see this link:

      http://developer.android.com/reference/android/app/Activity.html#onPause())
