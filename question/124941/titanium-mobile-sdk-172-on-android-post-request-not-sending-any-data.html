Titanium Mobile SDK 1.7.2 on Android: POST request not sending any data.

Hi everyone,

I'm making an app for a client, where I have to sent a post request to the server (Google App Engine)

However, the data being sent is always empty.
I'm using this code to send the image:

function sendImage(attrs) {
    var xhr = Titanium.Network.createHTTPClient();
    xhr.onreadystatechange = function() { };
    xhr.onload = function () {
        Ti.UI.createAlertDialog({
            title: 'Success',
            message: 'Status code: ' + this.status + ' Server Response: ' + this.responseText
        }).show();
    };
    xhr.open("POST", 'https://<appName>.appspot.com/store');
    xhr.setRequestHeader('Content-Type', 'text/html; charset=utf-8');
    imageB64 = Titanium.Utils.base64encode(attrs.image);
    debug("Sending image now: imageB64: " + imageB64);
    xhr.send({
        a: "storeImage",
        image: imageB64,
        name: username,
        password: password,
        fish: '',
        location: ''
    });
}

The response of the server is this:

Status code: 200
Server Response: Started storing
Action: 'null'

A part of the code on the cloud's side is: 

resp.getWriter().println("Started Storing\n");
String action = req.getParameter("a");
resp.getWriter().println("Action: '" + action + "'");
Enumeration params = req.getParameterNames();
while (params.hasMoreElements()) {
    String reqName = (String) params.nextElement();
    String[] toWriteArr = req.getParameterValues(reqName);
    resp.getWriter().println(reqName + ": " + toWriteArr[0] + "\n");
}

I'm sure this has nothing to do with the server code, on GET reqeust it performs just fine, but with POST request it just sends nothing with the request.

The reason I switched from GET to POST is because the request is too large to send as a GET request.

I hope someone can help me and that I informed you enough.
Thanks anyway!

    I was having the same problem as you… battled it for four hours tonight. Finally as a last ditch effort before going to bed, I did this…

    //xhr.setRequestHeader('Content-Type', 'application/json; charset=utf-8');

    Commented out the line where I sent the content type. Sure enough, now it works. I have no idea why… seems like a bug to me, but what I do I know.

  • Android doesn't need User-Agent to be defined for posts to work. It does, however, require the method type be UPPERCASE.

    Sent as POST:

    xhr.open('POST', 'http://example.com');

    Sent as GET:

    xhr.open('post', 'http://example.com');
