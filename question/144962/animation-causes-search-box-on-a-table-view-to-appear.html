Animation causes search box on a table view to appear

Weird issue I'm having, I'm animating the window to the left (250) to show another window behind it. Whilst the animation is occurring, the search field appears and disappears when the animation is complete.

Is this something I will have to live with or can I do something to prevent it from happening.

    //
    //
    //
    var classes_table_view_search_bar = Titanium.UI.createSearchBar({
        showCancel: false
    });

    //
    //
    //
    var classes_table_view = Ti.UI.createTableView({
        bottom: 0,
        maxRowHeight: 100,
        minRowHeight: 0,
        top: 42,
        visible: false,

        backgroundColor: 'transparent',

        filterAttribute: 'class_title',
        filterCaseInsensitive: true,
        search: classes_table_view_search_bar,
        searchHidden: true,
    });
    self.add(classes_table_view);

Animation

    //
    //
    //
    animate_left = Ti.UI.createAnimation({
        curve: Ti.UI.ANIMATION_CURVE_EASE_OUT,
        duration: 320,
        left: 0
    });

    classes_window.animate(animate_left);

iOS
SDK 2.1.3.GA

    Better still rather than using 320 which is iPhone specific get the devices height & width using:

    Ti.Platform.displayCaps.platformWidth

    and

    Ti.Platform.displayCaps.platformHeight

  • I fixed the problem !

    By setting the window width to 320 instead of 100%, and the table view to 320 instead of 100%. Now the strange animation doesn't appear when I slide the window to the right of the screen.
