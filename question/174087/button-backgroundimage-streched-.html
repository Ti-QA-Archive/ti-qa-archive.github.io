    Hi

    The backgroundImage allows you to provide an image and have that image occupy the width and height of that thing - a button in this case.

    So if you create a button and make it 40x40 in size and provide a backgroundImage that has a size of 30x20 it will stretch to fit the space. The property backgroundRepeat only says do not use more than one of this image to make up any differences in dimension.

    If you want to have an image in the LeftNavButton - you can;

    • use the image property of the Button - however there may be predetermined border areas that you cannot reach - these will be OS level and change between iOS 6 and iOS 7.
    • make the image the same size as it will be used - even if that means adding transparent padding pixels as a border to make up the differences in sizes.
    • use an ImageView and use its image property. You can attach the same Event Listeners as you would a button.

    • Thanks, I used the image property instead of backgroundImage now it tint's my image and it looks blue, why is that?

      — commented May 11th 2014 by Dan Peleg

    • On iOS 7 the image property on the button will automatically inherit the tint color of the;

      • NavigiationWindow
      • TabGroup
      • WIndow

      It would not do this in iOS 6 - this is by design.

      This is why I gave three suggestions; you need to work out the best one for your needs.

      Keep in mind the UI guidelines for iOS 7 is to have a solid image with a tint - check out the AppStore app and see the examples of those apps that have been redesigned to honour the iOS 7 UI - lots of examples. The idea is to have a clean consistent look and feel, ideally textual versions are meant to be used - but I still prefer icons - plus there is normally a requirement to use as little space as possible.
