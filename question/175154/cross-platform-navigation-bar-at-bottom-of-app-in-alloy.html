      • Shouldn't this be easy?

        Yes and no. Yes; it should be easy in so far as Titanium is a cross development solution that enables you to write common code and then compile to many platforms. No; Titanium does not create identical applications from one set of code. All business logic and functions work the same way - but as each device has unique ways of handling many things there are differences. You note yourself that only iOS has the navigation group - so Titanium cannot make that up for another platform - as that is simply trying to make all platforms the same - they are not.

        Developing software is hard, you can tell that by the facts; not everybody does it and also makes a fortune, you have to learn new things constantly just to keep up. It is also not just about the code - you have to have a complete understanding of how the Android, iOS (and any other) platforms work from the users perspective.

        Without this understanding you will never be able to deliver an app from the same base code that presents itself to each platform by honouring the UI guidelines for each, the differences in inter-app communication, managing of graphics and all the other things.

      • Am I missing something?

      • What does cross platform really mean then?

        With WhatsApp - you look at it know it is not identical in look but it is identical in function. This is cross platform development. You write core code that works across each platform that performs all the things your app needs to do. But when it comes to presenting to or interacting with you must take note to do that in the way each platform should. Some times they are identical, many times almost the same the rest completely different.

      • What are your responsibilities as a newbie?

        Learn - and learn lots. Make sure you have each platform you are developing for. If you choose both iOS and Android then understand that both of those has two distinct UIs from different OS versions. iOS pre 7 and 7+, Android 2.3x and 4.x (3.x should be ignored). Each of these UI are distinctly different - not only in looks but in how the user interacts with each and how the OS themselves interacts with the user.

        Make sure you have each device - USE it do not play with it, make it your normal phone for awhile and understand the differences. Know that in Android there are small but important differences between; 4.0, 4.1, 4.2, 4.3 & 4.5.

        Understand how the platforms handle graphics, learn about density and retina variations, never assume you know - it always changes. Understand how local storage differs and how apps talk to each is different. Keep in mind that Android !== iOS - you may not have like the phrase but it is the best way to describe it.

        Learn today, learn tomorrow and learn the day after. Everything changes, priorities swap and needs of the develop evolve. I have been writing commercial software for 25 years. During that time I have written two apps that became the industry standards in their fields. I have learnt 20+ software languages, written 3 scripting languages of my own. I still have to learn new things every single day. I have three R&D proof of concept projects running right now using things that were not available 12 months ago - you never stop learning.
