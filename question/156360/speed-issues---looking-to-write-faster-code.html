    • OK, what follows is a list of suggestions and improvements that can make things better. The code you have provided is incomplete as there are many functions that are not provided (I understand why) so anything here is based on what I can see.

      • Move the vibrate line to the last thing the event, this will make sure that anything before it actually starts in case their is a delay;
      moreButton.addEventListener('singletap', function(){
    screenProtector.on();
    clearMainMenu();
    setTimeout(function(){ moreScreen(); }, (buttonAnimationDuration + buttonDelay + 100) );
    if (foregroundSoundFlag){ Ti.Media.vibrate(); };
});
      • Try adding a vibrate after the app has started and before anything occurs - this "may" cause the vibration function to be primed ready for first use.
      • You appear to determine the instructionsSmallScreen or instructionsLargeScreen state each time a more button is pressed. Instead create an instructions view once, set visible to false, base the layout on decisions made at the start and once only. When required use show/hide to display it, this will save some time.
      • Stick to a single filename case Images/MyButton.png this will bight you in the future, better = images/mybutton.png, much better = /Images/mybutton.png
      • NEVER use a space in a filename /Images/MyButton Fade.png < asking for trouble
      • Animation event listeners are badly documented, the documentation suggests using a second parameter to the animate method - ignore this, use an eventlistener on the animation itself;
      onScreen.addEventListener('complete', function (e) {
   // do something
});
      • Add touchEnabled: false to anything that should not receive a touch event itself, this includes the label on each button.
      • As you noted, wrapping the button and label creation into a single function and returning the button is the smartest move, simplifies the whole process, makes the code easy to maintain and read.
      • Read an earlier answer I gave about cross platform support for images - which includes how to manage dimensional units and image folder locations, you are trying to manage something that can and should be automatic if you set things correctly;
        • How to achieve same layout for Android and iPhone using Titanium
      • You define the delay of the timeout at the bottom of the script AFTER the script has run menuDelay = 1000;, move this above the timeout function

      This should give you some stuff to work on.
