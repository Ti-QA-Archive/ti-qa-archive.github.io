"TiBaseActivity layoutCleanup" causes hanging in Android App in Titanium SDK 1.7.5

This is not a question, but since I can't submit a bug report or support ticket, I thought it might be useful to post it somewhere so others can find it. "A problem shared…" and all that! :-)

I have observed that when moving between Heavyweight windows in Android app (particularly when closing the window that is being switched from), log cat reports that TiBaseActivity fires 'layoutCleanup'. In versions of the Titanium SDK prior to 1.7.5 this does not cause any issue, but under 1.7.5 it causes the app to hang up and the UI becomes unresponsive. There is no other information output to log cat, no errors, debug or anything. The only solution is a manual Force Close and restart of the app.

The window switching is being handled by Kevin Whinnery's excellent Navigation Controller module code as it's the best way I have found of having a properly working hierarchical navigation system in Android apps under Titanium.

I have reverted to using 1.7.3 for now as it seems stable for supporting this approach.

I would love to submit this as a bug report or ticket for the Titanium team to look into but as an Indie subscriber I can't as far as I am aware?
