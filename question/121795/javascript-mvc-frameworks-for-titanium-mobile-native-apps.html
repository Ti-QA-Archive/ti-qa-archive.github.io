  • For my current project I'm using PureMVC with Prototype (of which I changed the Class-method a little). First I tried PureMVC with MooTools, but the last version of MooTools would not run in Titanium. I've added some of the MooTools features to Prototype, and am very satisfied with the result. Finally, I've also built a basic ActiveRecord-class for the models. So, in the end I kind of built my own framework (with copy-paste and writing some original code). It needs a lot of finetuning, but I am very happy with the result so far.

    PureMVC is great: it is lean but powerful! I could have finished my first project much faster if I had used it, and PureMVC leads to clearer code and better architecture of the app.

    I'm not sure whether ExtJS will work in Titanium. You should try both iOS and Android: Rhino is not WebKit JavaScriptCore. I'm using PureMVC in combination with Prototype works on both platforms.

    • I don't know what the value is in using jQuery: it is meant to manipulate the DOM – which is something one does not use in Titanium (which is actually a wrapper for the native API's), with the exception of webviews, of course.

      Prototype works fine. It is a very simple framework (of which you only need a part); unfortunely it's OO-features are less advanced that those of MooTools – I don't know exactly about Ext (I once used it for a web-project, and was quite impressed by it). I will look into the changes I made to Prototype's Class-method (which were somehow important to me).

      You will have to get used to PureMVC – it takes some time to 'get' how it works. Examine the examples: that helps a lot. Once you start to use it for your own project, PureMVC becomes clearer. The documentation is great: especially the simple rules.
