  • Accepted Answer

    As of Titanium 3.0, Titanium has a new CLI that no longer generates the build.log file. The build.log mostly contained the output from the xcodebuild process. Now that the CLI outputs the complete xcodebuild output, there really is no urgent need for a build.log file.

    You can see the full output by setting the log level to "trace". This can be done in Titanium Studio from the console pane's log level menu (on the right side). Otherwise you can set it via the CLI by running:

    ti config cli.logLevel trace

    Now, with that said, I'd like to add file logging back to the CLI. This is mostly for support reasons. If you have a problem, you can just copy the build.log into a gist and share the link. But being this isn't a huge priority compared to some of the other issues, it probably won't happen anytime soon.
