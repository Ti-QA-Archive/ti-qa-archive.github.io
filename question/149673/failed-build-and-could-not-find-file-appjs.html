    Hi

    Firstly you were right to move to Titanium SDK 3.0.2, there was an issue and normally it would have gone away with the SDK upgrade and a full clean.

    The problem you had found was due to an error in your code, this is likely because of something really small and even more likely could be as simple as a comma badly placed. Most of the issues for 3.0.0 users was just that (not always of course). It would be smart to try and find those issues and resolve them.

    This is an example of what could be the cause and you may have this problem more than once.

    var view = Ti.UI.createView({
    backgroundColor: 'red',
    height: Ti.UI.FILL,
    width: Ti.UI.FILL, // notice the trailing comma - this SHOULD not be here
});

    I would suggest using jslint.com to test your code on a regular basis, however you should not that you will need to add a few common library names at the bottom of page otherwise you will get lots of warning about things that are not actual issues. I would also suggest using the toggle switches to turn off some of the more harsh rules - like white-space checks.

    However before all of this, I would suggest you firstly rule out a bigger issue - can you build using Ti. SDK 3.0.2.

    To test for this, create a new default project, confirm it is using the latest SDK and make sure you change NO code. Does this work? If yes then, if you are using any modules, if so add them to the test app and a quick test code block to make sure all is called. If this works - then it is likely a code syntax problem.

    • OK we shall start by assuming it is the certificates - I had not noticed your previous question related to the certificates.

      Make sure you do not have BOTH the old and new certificates assigned in your computer. You should delete the old ones. You can find these using either xcode organiser (window menu in organiser) or keychain. If not sure, delete anything that matches the same device id and re-install all of them.

      Have you tried creating the new default project without any code changes yet?

      If this works, try compiling for device using the new certificate - remember to set the device id to the same as the certificate.

      If this fails, create a NEW certificate with a test name, adjust your app device settings for this name and try for device again.
