IOS device code sing error

Hello, I have been trying to test on IOS Devices and I get the following errors

[ERROR] : Error details: Code Sign error: No codesigning identities found: No codesigning identities (i.e. certificate and private key pairs) that match the provisioning profile specified in your build settings (“iOS Team Provisioning Profile: *”) were found.

[ERROR] : Error details: CodeSign error: code signing is required for product type 'Application' in SDK 'iOS 7.0'

I am using MAC Maverics OS and Xcode version 5.0.1 (5A2053)

Any idea what may cause the problem?

Thank you in advance

    I ran in the exact same problem when I upgraded to OS X Mavericks. This happens when we do not backup signing certificates and update or reinstall the OS. Pretty common.

    Steps to Fix

    • Open Keychain, and ensure you have Apple's Intermediate Signing Certificate (Skip to step 5 if you have it)
    • In case you don't, Xcode probably has created new ones for you. Head over to dev connect
    • Under Certificates, you'll see your old certificates and the new ones (new ones will have a further expiry date)
    • Download and install them
    • Open Xcode preferences and under Accounts, open your account. You'll see a small refresh button in the bottom right corner. Hit it!

    In a few seconds, you'll be good to go.
