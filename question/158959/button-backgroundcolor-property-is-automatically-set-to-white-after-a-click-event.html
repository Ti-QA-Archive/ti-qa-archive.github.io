Button backgroundColor property is automatically set to white after a click event

I've got a button defined like so in my .tss file: 

"Button":{
    width: 50,
    height: 50,
    bubbleParent: false,
    color: "white",
    font : {
        fontFamily : 'Helvetica Neue',
        fontSize : 18,
        fontWeight : 'bold',
    },
    backgroundColor: "blue",
},

In the xml I have the following: 


<Alloy>
    <TableView id="table">
        <TableViewRow id="row1">
            <Button id="my_button"/>
        </TableViewRow>        
    </TableView>
</Alloy>

In the javascript, I have an event listener on the button, like so: 


$.my_button.addEventListener("click", function(e) {
    alert("I was clicked");

});

I also have an event listener on the row, like so that fires a button click when the row is clicked: 


$.row1.addEventListener("click", function(e) {
    $.my_button.fireEvent("click"); 
});

If you notice I have the bubbleParent property of a Button set to "false" but when a user clicks on the button the background color of the button completely disappears (I think it's set to white and blends in with the white background color).

I can set the backgroundColor property of the button back to 'blue' after a button click but I would have to do this for each of my buttons. I just would like to know why this is happening. I suspect it's somehow related to bubbling events but I'm not really sure. Thanks in advance.
