Alloy Android slider e.value undefined

All,

  • Titanium Command-Line Interface, CLI version 3.2.0
  • Titanium SDK version 3.2.0.GA
  • Device: Nexus 4
  • Device OS: Android 4.2.2

The following code shows e.value as undefined. If anyone has any suggestions as to what I am missing, I would greatly appreciate it.

JS

$.slider.addEventListener('touchend', function(e) {

        var currentValue = Math.round(e.value);
        Ti.API.log('slider currentValue: ' + currentValue);
        Ti.API.log('slider touchend e: ' + e.value);

        if (currentValue < 1000) {
            currentValue = 500;
        } else if (currentValue < 1500) {
            currentValue = 1000;
        } else if (currentValue < 2000) {
            currentValue = 1500;
        } else if (currentValue < 2500) {
            currentValue = 2000;
        } else if (currentValue < 3000) {
            currentValue = 2500;
        } else if (currentValue < 3500) {
            currentValue = 3000;
        } else if (currentValue < 4000) {
            currentValue = 3500;
        } else if (currentValue < 4500) {
            currentValue = 4000;
        } else if (currentValue < 5000) {
            currentValue = 4500;
        } else {
            currentValue = 5000;
        }
        $.slider.value = currentValue;
        if (currentValue < 1000) {
            $.sliderLabel.text = currentValue.toString() + ' meter alert radius';
            Ti.API.log('sliderLabel.text: ' + $.sliderLabel.text);
        } else {
            $.sliderLabel.text = (currentValue / 1000).toString() + ' kilometer alert radius';
            Ti.API.log('sliderLabel.text: ' + $.sliderLabel.text);
        }
    });

    • As a follow up to this answer; the reason that e.value did not work is because the event listener touchend does not provide that property in the returned object. Only a few event listeners actually do for the slider - including; change (which is the most commonly used event for this view), start and stop.
