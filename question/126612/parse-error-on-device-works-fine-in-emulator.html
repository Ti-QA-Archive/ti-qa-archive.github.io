  • Accepted Answer

    The problem was this:

    var x = 1,
var y = 2,

    I missed this when moving out of the json object.
    The js validator wasn't catching the commas.

    I suspect this is not the only thing that might potentially cause the issue though.

    Hope this helps someone at some point.

  • I had a very similar problem. Code ran fine in the simulator but created a JS parse error on the device.
    After using binary chop for the last 3 hours I found that this line was causing the problem.

    var someString = 'text' + variable + ' more text ' + + ',' + ' some other text';

    Notice the double plus + + ? that's what was doing it.
    It would have been nice to get at least a compile warning for that. Any plans for that feature?

  • I had the same issue. To make it worse it worked on my iphone when run from Titanium, but when distributed as an IPA file it gave errors such as these in the xCode console:

    <Notice>: [ERROR] Syntax Error = Parse error at tab1_position.js (line 7)
<Notice>: [ERROR] application received error: Parse error at tab1_position.js (line 7)

    Eventually it turned out to be this line that was the problem:

    var dLong = (180 - previousLongitude) + (180 - - issPredict.longitude);

    The two minus symbols created the parse error. And this wasn't line 7 but at line 264! Hopefully this will save other people some time debugging. It took me many hours to debug this.
