Dynamic positioning of views

I have an app with four tabs. Each tab is loaded via data from web services. If one of my users views tab 1, then changes the backend data, switches to tab 2, then back to tab 1, tab 1 should refresh with the changed data from the backend.

Part of the data in tab 1 is a list of phone numbers. If the count in the list of phone numbers changes, I want to expand or retract that list as needed with views around the phone number list adjusting as necessary.

The way I have it implemented now is that tab 1 calls the web service upon a focus event. The web service function then fires an event when it is done. Tab 1 listens for the web service completion event and then populates tab 1 with the necessary views.

So I have several questions:

If I save handles to the original views, remove the views, and create new views upon each focus event, will the original views be released in memory? Or will memory keep accumulating each time a user goes back to tab 1?

I tried creating all the necessary views first, but because the views are showing before the web service completion event is fired, changing the 'top' and 'left' properties of the views in the event listener have no effect. Is there a way to keep a view from displaying until I have updated the positioning properties? And, if a user comes back to tab 1, how can I adjust their positioning properties appropriately?

Thanx in advance for your help.
