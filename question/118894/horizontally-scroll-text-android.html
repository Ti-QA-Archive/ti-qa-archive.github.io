Horizontally Scroll Text (Android)

Hi All:

Working on a media player for Android. Since I can't parse the stream for the metadata (and please PLEASE correct me if I'm wrong there), I'm going out to a URL to get the artist/album/song info.

That's all working well enough… but there are instances where the resulting string is longer than the allocated display area.

Would like to horizontally scroll the text (left and right), as you see in many media player interfaces.

I've been googling, reading these forums, and experimenting most of the day, all to no avail.

Right now, I've got a Ti.UI.label inside of a Ti.UI.ScrollView. The label is populated with the data… but how to scroll it?

The issues as I see them are:

1) How do I even know if I need to scroll? how can I test that the width of the label (which is set to "auto" on creation) is wider than the width of the containing ScrollView?

2) If I do determine that the label is wider than the scrollview, how would I go about scrolling 1 px left until I reach the end? Specifically… how would I know where the end is?

Then, the same 2 questions in reverse :)

Any help/suggestions/info etc would be GREATLY appreciated.

  • I think you can get the width of the auto-sized label in the postlayout event. Then you should be able to use the scrollTo() method to move the scrollview along. Not sure what you'll do when you hit the end of the label; you may just have to pause at the end and quickly jump to the start again (rather than a continuous scroll like I've seen in some music players).
