ListView instead of Table view for data binding

Hi Friends,

i am creating an android app using alloy framework. i would like to use "ListView" instead of "TableView" in my code.

please see my current code below

controller.js


        var threads= Alloy.Collections.restModel;
    var thread = Alloy.createModel("restModel",payload);
    threads.unshift(thread);



function threadtransform(model) {
       var transform = model.toJSON();
       transform.title =  transform.topic;
       transform.id =  transform.id;
       transform.image = transform.image;
       transform.careprovider = transform.careprovider;
       transform.date = transform.date;
       return transform;
}

xml

    <TableView dataCollection="restModel" dataTransform="threadtransform" id="table">
            <TableViewRow id="row" title="{title}" onClick="openChats" accessibilityLabel="{id}" selectedBackgroundColor="#000080">
                <Label id="title" text="{title}"/>
                <Label id="date" text="{date}"/>
                <ImageView id="avatar" image="{image}"/>
                <Label id="careprovider" text="{careprovider}"></Label>
            </TableViewRow>
       </TableView>

But if i use ListView instead of table view i cannot do model binding like this.

Also currently when a new model comes to collection it automatically display as a row in table. but this is not possible in ListView

Please give some suggesion

Thanks

  • This is not possible in the latest stable Alloy (1.1.3). It is supported in the upcoming 1.2.0 release. You can test it out early by installing alloy straight from github:

    sudo npm install -g git://github.com/appcelerator/alloy.git

    and you can find a test app that shows listview binding in action here:

    https://github.com/appcelerator/alloy/tree/master/test/apps/models/binding_listview
