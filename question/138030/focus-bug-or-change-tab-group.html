Focus Bug or Change (Tab Group)

//Create Tab Group
tabGroup = Titanium.UI.createTabGroup();

tabGroup.addTab(home_tab);
        tabGroup.addTab(trip_tab);
        tabGroup.addTab(map_tab);
        tabGroup.addTab(search_tab);
        tabGroup.addTab(more_tab);

tabGroup.addEventListener('focus', function(e) {
                Ti.API.info("tab index:" + e.index);
            });

//Display Tabs
tabGroup.open();

Now if I am on the home tab, and click the home tab the 'focus' event is not fired.
It only gets fired if I change tabs.
This is only for Titanium SDK: 2.02.GA

Under 2.01.GA2:

It fires the 'focus' event when you are on the home tab, and you click this tab.

So we are switching to 2.0.1.GA2 until this bug is fixed, or another way is suggested for this.

Thanks.

  • If you are on the home tab and then you click the home tab, there will be no 'focus' event because the window is already focused. A focus event is only fired once when the tab comes into focus. Clicking on the tab that you are already on should not re-fire the focus event. So Im going to go out on a limb and say that the bug was in the previous version and was fixed in this version.
