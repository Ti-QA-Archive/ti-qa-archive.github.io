iOS: Combine Tableview -SwipeToDelete and ScrollableView

Hi all,

i am not able to combine scrollableView and swipe event on a TableViewRow.

For example:

var globalScrollView = Ti.UI.createScrollableView({
    views: [myView],
    showPagingControl: false
});
var tableView = Ti.UI.createTableView({editable: true, data: [{title: 'test'}]});
myView.add(tableView);

I tried via Swipe-To-Delete (editable = true) and via an EventListener 'swipe' on a TableViewRow. Also tried several touchEnabled combinations.

Swipe works as soon as I remove the scrollableView. The only workaround is to set editing=true but I won't do this.

Does anyone know a workaround?

  • haven't had the time to code up an example, but my suggestion to you is to have an "edit mode".

    the tables should not be editable until they are switched into edit mode, this would allow the swipe to function normally when in edit mode, the tables will work, when not in edit mode, the scrollview will work.

    i also believe it would be a better user experience because it would be clear to the user what their action is doing to the content, otherwise a swipe to change a page could accidentally delete a row
