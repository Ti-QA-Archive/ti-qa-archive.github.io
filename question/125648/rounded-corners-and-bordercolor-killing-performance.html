Rounded corners and borderColor killing performance.

I've seen a few posts here where people have commented about the performance hit with rounded corners, but only a few small comments about it (from what I've found so far). I'm experiencing huge performance problems when using either borderRadius or borderColor on a view, and even worse performance when combining the two. This is on the iPhone 4 with Titanium 1.7.x. Windows containing views with these property sets feel sluggish when scrolling or when alerts pop up over them. This is not the case, however, when rendering tables with the "GROUPED" style, which automatically come with rounded corners. These render perfectly smooth. Is anyone else experiencing similar issues? If so, are there any solutions (besides avoiding rounded corners and borderColor, of course)?
