    Hi

    On the basis this is the AirPrint module from Appcelerator, then I have used this in a few of my apps.

    It will print anything that is an image or a PDF. You have not provided either.

    The url property is used to point to a local file which is either a PNG, JPG or PDF. You are passing event.media - this is a blob.

    Save the event.media to the file system, the reference that file in the url property.

    • This is not a reference to a url;

      url: file

      It is a reference to a file, the file being an object. As you can see in your own code the file object has method write is used in your own code. This is because you correctly created a file reference object and then wrote a file using it. But that files URL is the same path as you used in getFile.

      Ti.Media.takeScreenshot(function(event){        
    var filepath = Ti.Filesystem.applicationDataDirectory + 'test.png';
    var file = Ti.Filesystem.getFile(filepath);
    file.write(event.media);
    file = null;

    AirPrint.print({
        url: filepath,
        showsPageRange: true,
        view: ClearButton
    });
});
