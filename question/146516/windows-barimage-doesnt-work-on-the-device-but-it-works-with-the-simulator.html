windows barImage doesn't work on the device (but it works with the simulator)

Hi,
I have specified an image for the bar, like this :

var window = Ti.UI.createWindow({
    barImage:'./iphone/title.png'
});

it works with the simulator,
but on the device (iphone 5), the image doesn't appear.

An idea ?

ios:6, titanium SDK:3.0.0

    Hi

    Yes this is an easy one - once you know.

    Change your code to the following;

    var window = Ti.UI.createWindow({
    barImage:'/title.png'
});

    I know what you are going to say "but i put my image in the iphone folder", you are right - you did - but the iphone and android folder are special.

    If you place an image (or ANY file) in the iPhone folder then it is actually placed into the root of the device when compiled. So in short pretend the iPhone folder is not there when on the device.

    This is great if you need device dependant imagery (or ANY file), the iPhone versions are available when on iPhone and the Android versions only on Android.

    These paths

    /iphone/example1.png
/android/example1.png
/example1.png

    And each are referenced as 

    barImage:'/example1.png'

    Are all the same, the OS will look for the example1.png in the the iphone folder first and if there is not one there it will look in the root.

    Even more - but you might be getting bored now.

    Imagine you were creating an app and it was going to be available for BOTH iPhone and Android, you had one image but the sizes were different when on Android and on iPhone.

    You would then place your android image in;

    /android/example.png

    And the iphone one goes in

    /iphone/example.png

    But as these are special folders the actual path is truncated to the root.
