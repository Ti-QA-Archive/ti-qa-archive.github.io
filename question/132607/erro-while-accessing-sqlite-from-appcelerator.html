Erro while accessing sqlite from Appcelerator

Hi,

I am trying to access a SQLite database from appcelerator.
I have added the sqlite file under the resources folder of my project. In my .js file, here is the code I use to access the DB - 

var db = Titanium.Database.install("myDB.sqlite","myDB");
db.execute('SELECT * from member');

However, what happens here is that, it gives me this error when I run the application on the iPhone simulator- 

invalid SQL statement. Error Domain=com.plausiblelabs.pldatabase Code=3 "An error occured parsing the provided SQL statement." UserInfo=0xa010840 {com.plausiblelabs.pldatabase.error.vendor.code=1, NSLocalizedDescription=An error occured parsing the provided SQL statement., com.plausiblelabs.pldatabase.error.query.string=SELECT * from member, com.plausiblelabs.pldatabase.error.vendor.string=no such table: member}  in -[TiDatabaseProxy execute:] (TiDatabaseProxy.m:186)

However, the member table does exist in the database.

Would someone be able to help me out with this?

  • hey,
    i just had a similar problem.

    what's weird is that calling Titanium.Database.install will either install a given database or install a new blank one. if your getting an SQL error, that probably means that the table (member) does not exist. i use this as a test query:

    var db = Titanium.Database.install("myDB.sqlite","myDB");
db.execute('SELECT * FROM sqlite_master');

    the table 'sqlite_master' is always in the db. even a brand new blank one. if that query works and has no results, then it's a new blank database.

    i had troubles making a working sqlite db.
    my solution was using the freeware, cross-platform sqlite studio
    sqlite studio download

    make sure your database file name is .sqlite (i could NOT get them to work with a .db extension). make sure your using sqlite version3. make sure your filename and database name are exactly the same in sqlite studio and your titanium code. any variance will make them not work.

    hope that helps!
