Storing User Data - Best Practises

Hello there,

I am working on a simple application for which I will need to collect from the users some data. It's really 2-3 figures that I am going to need from them. I have three options to consider.

  1. Storing the user's data on a remote server. I can do that since my app will be exchanging data with the remote server all the time. Pros is that data will be fully backed up and if my user's phone is replaced nothng will be lost. Cons is data usage and performance.

  2. Storing the user's data on a local SQLite database. Pros is that both performance and data usage issues are eliminated. Cons is too much code to store 2-3 values.

  3. Storing the user's data on an application variable within the Titanium.App.Properties scope. Pros is that it's too simple to implement. Cons, not sure if this is a good practise! :-)

What would you do and why?

George

    I'd go with the properties.
    If you need db like behaviour take a look at TaffyDB and TiTaffyDb to see how to use it (json database).
    You still have to take care about the size of the properties. I discovered that on my 3GS in properties I cannot store more than 204k in one variable.
