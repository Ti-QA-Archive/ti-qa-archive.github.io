hello world results in 275 objects?

Hi. With Titanium Developer 1.2.2, your "New Project" that creates the 2 tab app - has a app.js that fits on one screen - roughly 2 dozen lines of code. This compiles into an iPhone app that has 275 separate objects. A similar app built with Apple's XCode results in 3 objects. I am concerned about the proliferation of objects - this would seem to create a debugging nightmare. Can you please comment on this? Thanks.

— asked June 21st 2011 by Keith Wood

  • Titanium doesn't translate that couple of dozen lines of code into the corresponding native implementation. Instead it packages them along with a javascript runtime and a set of classes that implement the Titanium framework API, resulting in a quite big Xcode project.
