Code Sign error: No matching provisioning profile found

Hi,

I'm on OSX 8.5 and I'm trying to run my Titanium application on a physical iOS device. When I run the project, I get the following message :

[ERROR] :  Error details: Code Sign error: No matching provisioning profile found: Your build settings specify a provisioning profile with the UUID “0DA316A1-D67A-42AF-34A6-F2EA181D12BF”, however, no such provisioning profile was found.
[ERROR] :  Error details: CodeSign error: code signing is required for product type 'Application' in SDK 'iOS 7.1'

In the Xcode organizer window, I can see the correct provisioning profile, and I can also see it in the ~/Library/MobileDevice/Provisioning Profiles folder, so I don't understand why Titanium Studio can't use it.

For your information:

  • I use Titanium SDK 3.2.3.GA and Titanium Studio 3.2.3
  • I'm trying to deploy my app on an iPhone running iOS 6.0
  • I don't find the build/iphone/build/build.log file so I can't paste you its content…

Can you tell me what i'm doing wrong? What should I do in order to deploy my Titanium app on my device? Your help would be greatly appreciated! Thank you!

  • Accepted Answer

    Lets try to deploy to the device from the command line and see what happens.

    In the project directory, with the device plugged in

    ti build -p platform -T device -C all

    Should be prompted for the certificate to use to sign with, followed by a list of matching profiles.

    • Oh you were right, it was an install problem. I finally found the solution: the problem came from file system permissions.

      I ran Disk Utility, selected my hard drive on the left panel, and clicked Repair disk permissions under the tab First Aid.

      Now everything is fine: I don't need sudo to get ti info and Titanium finds the provisioning profile and it is able to deploy my app on a physical device.
