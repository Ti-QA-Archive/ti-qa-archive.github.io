How do you reload a different tab from another window?

Hi folks,

Hopefully a simple one here!

I have a settings screen with a simple button that will remove a setting.

Once this button is tapped it'll perform a tiny SQL update, but when I tap back on the tab I want, it'll still show the original content rather than my default content for first time users.

What I don't want to do is have a listener that looks for window focus as I don't want that screen to be making HTTP requests every time it's tapped.

How can I make the app animate to the tab in question and then perform my reload function?

Here is the button on 'settings.js', it needs to animate to 'my tab.js' after it's pressed and then load a function called loadAll() on the 'mytab.js' tab.

button.addEventListener('click',function(e) {
    var db = Ti.Database.open('TWv1');
    var sql = db.execute('DELETE FROM my_tab');

    alert("You have removed your setting");

    // close the db
    db.close();

});

Simon

    Check out an example for doing this with custom events in the Todo Sample, also available as a template project in Studio.

    Specifically, when a 'todo' is added to the DB via the add window, it will fire an event called 'app:updateTables' that is being listened for on the list window.

    If you also want programmatically switch tabs, you can check out the setActiveTab method.

    • I can't really say without seeing how you structured your code. If you are using a global namespace variable, you can add the tabgroup to that, so app.tabGroup would be accessible everywhere. Careful to watch for memory leaks if you are planning on closing the tabgroup. If you are using the new CommonJS standard for development (like the Todo sample) or have some other setup, you may want to set up an eventlistener, or perhaps listen for the same event, to handle the tab changing.

      You may also want to think about if you actually want to programmatically change the tab. Users are accessing a particular tab for a reason, and maybe moving them to a different tab might not be in their best interests in terms of UX. I know that doesn't really answer your question, just some thing to consider.
