  • You don't need the NDK. You would need that only if you were making extension modules for the Titanium framework. You have some other Android configuration issue.

    Check the setup guides. They're pretty long and there are lots of steps to go through. You might have missed a step. Or, something might be misconfigured.

    Run the android tool (comes with the Android SDK, not Studio). Do you have both the v19+ build tools and the API (SDK) for the target OS version installed, plus the associated Google APIs? Can you launch the emulator through the AVD manager (again, part of the Android tools).

    Search these Q&A forums for other "exit code was 1" messages. Unfortunately, the Android tools are a bit of a pain to configure so there are many such posts here (though at least you're on OSX; most of those posts are from Windows users where setup is even more challenging).

  • Hello Wallace,

    Thanks for your inquiry. Here are two things you do first.

    1. You need to make sure that the phone is in debug mode. Go to Settings –> Applications –> Development and turn on Debug Mode. This will allow Titanium Developer to install the app on your phone.

    2. Increased the timeout by modifying "/Users/Mac../Library/Application Support/Titanium/mobilesdk/osx/3.4.1.GA/android\builder.py"

    def wait_for_device ...
...
        max_wait = 120
        max_zero = 30

    Now try building your app on device.
