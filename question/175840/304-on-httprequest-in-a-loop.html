304 on HTTPRequest in a loop

Hi

I need to call an API in a loop to update multiple reports in the server. When I call the API using HTTPRequest, the first record gets updated. I am getting a 304 for the following records.

I have tried:

xhr.setRequestHeader('Cache-Control','no-cache');
xhr.setRequestHeader('Cache-Control','no-store');

after opening the connection, but it does not seem to work. Need some advice.

Thanks,
Anish

    Just a thought, add all of these;

    xhr.setRequestHeader("Cache-Control","no-cache, must-revalidate"); //HTTP 1.1
xhr.setRequestHeader("Pragma","no-cache"); //HTTP 1.0
xhr.setRequestHeader("Expires","Sat, 26 Jul 1997 05:00:00 GMT"); // Date in the past

    If that fails provide your entire httpClient code.
