Toolbar and buttons in 1.8.0.1

I've been using 1.7.5 for a while now with no issues. Just switched over to 1.8.0.1 and noticed I am getting some warnings about things are going to be deprecated in 1.9 so I began making the suggested changes in the warning.

The first was an animation change. Changed it to the new syntax and good that works. The second was Ti.UI.createToolbar(). The warning says the new syntax is Ti.UI.iOS.createToolBar() which is actually wrong. It is Ti.UI.iOS.createToolbar(). Notice the lowercase 'b'.

Once I changed it back to a lowercase B, it was then making the toolbar again. Now the issue I am having is setting the enabled property on the buttons in the toolbar. Changing a buttons enabled property to false, does disable it, however, it no longer dims the button. In other words it still looks active. I've tried changing the opacity on the button to 0.5 and that doesn't work. I think this is broken as of 1.8.0.1

code:

var backBtn = Ti.UI.createButton({
    backgroundImage:'../images/back.png',
    width:30,
    height:30        
});

var pauseBtn = Ti.UI.createButton({
    backgroundImage:'../images/pause.png',
    width:30,
    height:30    
});

var playBtn = Ti.UI.createButton({
    backgroundImage:'../images/play.png',
    width:30,
    height:30    
});

var slideListBtn = Ti.UI.createButton({
    backgroundImage:'../images/list.png',
    width:30,
    height:30
});

var linkBtn = Ti.UI.createButton({
    backgroundImage:'../images/link.png',
    width:30,
    height:30
});

var flexSpace = Ti.UI.createButton({
    systemButton:Titanium.UI.iPhone.SystemButton.FLEXIBLE_SPACE
});

var controls = Ti.UI.iOS.createToolbar({
    items:[flexSpace,backBtn,flexSpace,playBtn,flexSpace,slideListBtn,flexSpace,linkBtn,flexSpace],
    bottom:'base',
    borderTop:true,
    borderBottom:true,
    translucent: false,
    barColor:'#336699'
});
win.add(controls);

doing linkBtn.enabled = false, does NOT dim the button like it did in previous version of Ti SDK. Any suggestions?

  • Hello, I had the same problem with the 1.8.0.1 Ti SDK version and I found out that for every button which has a background Image you have to set another 2 images for every state:

    btn.backgroundImage = &#39;images&#x2F;button.png&#39;; &#x2F;&#x2F;normal state btn.backgroundFocusedImage = &#39;images&#x2F;button.png&#39;; &#x2F;&#x2F;focused state btn.backgroundDisabledImage = &#39;images&#x2F;button_disabled.png&#39;; &#x2F;&#x2F;disabled state

    If you set these properties, then when you set btn.enabled = false; the button will seem disabled (according to your backgroundDisabledImage) as it did in the previous SDK.
