HTML Webapp - download data and store locally, without sqlite3 dbs

Is it possible to 'fetch' a (html?) file from a webserver, store it locally and use it in the app?

I'm not a javascript programmer, and would need a working example (all the way) of how to achieve this, if even possible.

What I'm trying to do is:

  • make a small Webapp for Android and iPhone (WinPhone maybe later…)
  • have a check for 'am I online or not' and if online, fetch the latest html data (3-4 files) from a webserver (constructed from php/mysql)
  • use the fetched html data in 3-4 pages (iframe or direct?) to (offline) display the data.

There would be no to-be-synced/fetched pictures involved, only raw html data.

If someone could please instruct me how to achieve this, by means of a (full) example?
I do not want to use sqlite (for now), as it seems to be much more difficult to fetch, update and use the data in a html webapp on both platforms for what I want to achieve, no?
