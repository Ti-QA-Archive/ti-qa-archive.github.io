Push notifications to everyone with REST API

Hello,

I setup push notification services in dashboard and want to broadcast send push notifications to all devices subscribed to same channel via REST API.

Docs says:
Application admins can set the to_tokens parameter to everyone to send to all devices subscribed to the identified channel.

http://docs.appcelerator.com/cloud/latest/#!/api/PushNotifications-method-notify_tokens

I subscribed 2 devices (1 - iOS, 1 - Android) to same channel via Titanium code and can able to see it in dashboard under "Show Subscribed Devices" section.
I tried send notification to this devices from my server via ACS REST API using 'everyone' as value of 'to_tokens' parameter.

POST /v1/push_notification/notify_tokens.json?key=<my app key> HTTP/1.1
Host: api.cloud.appcelerator.com
Cache-Control: no-cache

----WebKitFormBoundaryE19zNvXGzXaLvS5C
Content-Disposition: form-data; name="channel"

test_channel
----WebKitFormBoundaryE19zNvXGzXaLvS5C
Content-Disposition: form-data; name="payload"

{"title":"Test title", "alert":"Test alert", "vibrate": true, "badge": 2}
----WebKitFormBoundaryE19zNvXGzXaLvS5C
Content-Disposition: form-data; name="to_tokens"

everyone
----WebKitFormBoundaryE19zNvXGzXaLvS5C

and get this response, saying status is ok, but with error message

{
    "meta": {
        "status": "ok",
        "code": 200,
        "method_name": "NotifyTokens",
        "message": "Unable to send notification to everyone"
    }
}

Notifications not received by devices after this request. If I change 'everyone' to real device token copied from dashboard, then push notification successfully received by device.

Can you check this? Maybe I missed some header or parameter or this is ACS REST API issue?

Thanks.

  • Hi,
    have you created an admin user under App Management > Users ?
    I had similar problem and solved sending _session_id parameter like:
    v1/push_notification/notify.json?key=<APP_KEY>&_session_id=<SESS_ID>

    You can get the session id through: v1/users/login.json?key=<APP_KEY> and sending login and password via POST

    Take a look: http://docs.appcelerator.com/cloud/latest/#!/guide/rest

    Hope it helps.

  • Hi, everyone is rule if your login Curl use Cookies,(CURLOPT_COOKIEJAR and CURLOPT_COOKIEFILE). For Example:

    $this->curl_opt = array(
        CURLOPT_URL => $this->acs_api_url.'/users/login.json?key='.$this->key,
        CURLOPT_RETURNTRANSFER => true,
        CURLOPT_SSL_VERIFYPEER => false,
        CURLOPT_COOKIEJAR => $this->tmp_fname, 
        CURLOPT_COOKIEFILE => $this->tmp_fname, 
        CURLOPT_POST => 1,
        CURLOPT_POSTFIELDS => 'login='.$this->creditentials['login'].'&password='.$this->creditentials['password'],
        CURLOPT_FOLLOWLOCATION => 1,
        CURLOPT_TIMEOUT => 60

    );
