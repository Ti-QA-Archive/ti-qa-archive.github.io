Uncover window "living" under the main navigationWindow

Hi, in the Titanium.UI.iOS.createNavigationWindow docs, I found this kind of window is not meant to be added to other windows. That's ok, but I found the Facebook iPhone app does a very strange thing: to reach the chat window, it moves the entire navigation group to the left and displays (uncovers) a window that apparently is under the navigationWindow. How they do this thing? Can Titanium do the same?

    • Hi Prashant, in the meantime (I don't use Alloy!) I found two ways to achieve a similar result.

      FIRST ONE: I created a "middle" Window the app launchs when it loads. When this window loads its content it's invisible: this same window opens the navigation window. IF I open the nav this way, if I use simply open() the nav opens but I can't see it (I know it opens because I can read the Ti.App.info coming from there). If I open this window with modal:true it opens correctly.

      SECOND ONE: directly in the app.js I open two top level windows, one containing tha nav, the other one the "popunder" (I don't know how to call it). This works too, even if I "like" (!) the first solution more (if it would work without modal:true, it would be better).
