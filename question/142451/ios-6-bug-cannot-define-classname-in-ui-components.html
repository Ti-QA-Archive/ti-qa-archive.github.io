I created a button that had a "className" attribute, but when I clicked it and it turned out to be a unexpected className "TiUIButtonProxy" .Somebody help me ,thanks!
I use mac OS 10.7, Titanium SDK 2.13, IOS SDK 6.0.

    var button1 = Titanium.UI.createButton({
        title : "test",
        width : '60dp',
        height : '35dp',
        bottom : '5dp',
        className:'testClassName'
    });
button1.addEventListener("click", function(e) {
    Ti.API.info(e.source.className);
})

here is the console output:
[INFO] TiUIButtonProxy

    This has been logged as TIMOB-11620, and I linked it to a deeper problem TIMOB-10577. Basically, Apple implemented a className method that shows up as a property. For tableViewRows, we work around it by making a write-only property which stores it under a different name. Unfortunately, the fix is deep, because just overriding the values could cause crashes or app store rejections. Only thing I can suggest for now is to use a different property name, sorry.
