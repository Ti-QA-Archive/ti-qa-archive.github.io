Issues with any screen density

Hi All

I am having issues with any screen density i built my whole app with <supports-screens android:anyDensity="false"/> in the manifest which is the simple option to cater for multiple screen sizes.

But when i try to use the following code.

var currentTop = 0;

var win = Ti.UI.createWindow({
    navBarHidden : true,
    fullscreen : false,
    backgroundColor : '#202020'
});

if (Ti.UI.Android) {
    win.windowSoftInputMode = Ti.UI.Android.SOFT_INPUT_ADJUST_PAN;
}

var scrollView = Ti.UI.createScrollView({
    contentHeight : 'auto'
});

var inputTextField = Ti.UI.createTextField({
    backgroundColor : 'white',
    hintText : 'Enter value ',
    bottom : 0,
    left : 0,
    width : '80%',
    height : 90
});

scrollView.add(inputTextField);

var send = Ti.UI.createLabel({
    backgroundColor : 'white',
    width : '20%',
    height : 90,
    right : 0,
    color : '#900',
    font : {
        fontSize : 18
    },
    text : 'Send',
    textAlign : Ti.UI.TEXT_ALIGNMENT_CENTER,
    bottom : 0
});

scrollView.add(send);

win.add(scrollView);

win.open();

It will not work as the sizes seem to be wrong. But as soon as change android:anyDensity to true it works fine.

So i am thinking i am going to have to change all my code and remove the dp's set for all my height and width values.

Also been reading the docs http://developer.android.com/guide/topics/manifest/supports-screens-element.html

and it says

android:anyDensity
Indicates whether the application includes resources to accommodate any screen density.
For applications that support Android 1.6 (API level 4) and higher, this is "true" by default and you should not set it "false" unless you're absolutely certain that it's necessary for your application to work. The only time it might be necessary to disable this is if your app directly manipulates bitmaps (see the Supporting Multiple Screens document for more information).

So i am thinking whether i should have never set android:anyDensity to false in the first place.

Can someone please shed some light on this. How can i manage multiple screen sizes without setting android:anyDensity to false ????

Thanks

  • I use anyDensity=false and dp on all my units and it works like a charm. Looking at your code I can see that you don't use dp.
    You can also try to set dp as default in the tiapp.xml with:

    <property name="ti.ui.defaultunit">dp</property>
