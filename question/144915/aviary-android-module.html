Aviary Android module

Hi,
I'm trying to add Aviary SDK functionalities (http://www.aviary.com) in my mobile application.
I've found a module for iOS but not for Android.
So I'm trying to create it myself. Problem is that the SDK comes with a library project which needs to be include in module project (http://www.aviary.com/android-documentation).
I've read that library projects, for now, are pretty laborious to include : http://jira.appcelerator.org/browse/TIMOB-11360 (basically, it's not possible to use xml layout resources to inflate views), especially for a project like Aviary, which contains hundreds of calls to findViewById or inflate functions…
So my question is simple : did someone faced the same problem for Aviary and found a quick way to achieve these modifications ? And/Or do you think it's pertinent to do so or should it be better to create a native Android project instead ? (code maintainability after aviary sdk update etc.)
Thanks (and sorry for my English).
