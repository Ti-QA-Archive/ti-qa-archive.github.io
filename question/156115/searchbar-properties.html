  • Hi

    When you use a searchBar and link it directly to a tableView a syou have done, you have a very limited set of properties that you can change - this is because the searchBar becomes part of the tableView so it uses the standard UI.

    The choices you have set will not work at they would affect the placement or standard UI of the searchBar.

    If you need this level of control then you cannot link it directly with a tableView, but if you do separate them, you will need to manage the filtering that is currently occurring automatically.

      • remove the search reference in the tableview
      • add an event listener to the searchbar change
      • revise the array you originally added to the table based on the searchbar value (less values than previously)
