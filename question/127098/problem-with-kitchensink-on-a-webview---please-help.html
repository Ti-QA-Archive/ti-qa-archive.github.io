Problem with KitchenSink on a WebView - please help

i am trying to implement the 'show_indicator', 'hide_indicator' to my project.
but i think there is a problem i cant find how to solve it.

please go to KitchenSink in the 'Base UI' tab click on the 'views' and than on 'Web Views' - now click on 'Local URL' and very fast on the 'back button' in left navbar so its not finish to load the webview.. the indicator not hide.. i think the event listener 'load' not called cause we click on the back button very fast.

wat should i do to fix it… thank you all

  • Accepted Answer

    You could use a timeout as a fallback. If the load event isn't called is some set amount of time then the timeout will close the indicator.

    You could also listen for the click event on the left nav button on the window and if the activity indicator is still showing, then hide it.

    You could also listen for the close event on the web view window, and if the activity indicator is still showing, then hide it.

    There are any number of ways to handle this. One of the above should work just fine.
