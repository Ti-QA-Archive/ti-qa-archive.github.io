Error when requiring custom classes when using iphone simulator.

Hi there,
I'm trying to build a project that already works in android (works in emulator and device) for iOS. The problem is that custom modules (just classes i've created) are not recognised by the iphone when i require them.
The message is: [WARN] "Exception in event callback. Couldn't find module: buttons/custombutton"
My code is:

var customButton = require('buttons/custombutton');

and the call:

var mouthButton = new customButton();

The buttons folder path is "Resources/views/buttons".
The require line is in a file in the "views" folder.

Application type: mobile.

All sdks etc are last ones because its pretty new installation.

Device: iOS simulator.

Host Operating System: OSX 10.8.

Titanium Studio: build: 2.1.2.201208301612.
