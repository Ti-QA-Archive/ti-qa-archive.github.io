Titanium SDK 3.0 borderRadius problem

Hello,
So today i decided to use SDK 3.0 (mostly because of the applyProperties method and On-Device debugging), however, i noticed that rounded borders on controls (textField and textArea to be specific) are not acting as they should.

The problem is that editable area of the controls is going over the controls' border thus hiding the round part as shown in the screenshot.

What's even weirder, is that if i put borderRadius on their parent, the problem is fixed for those controls too. :/

Parent view code:

    var llView = Ti.UI.createView({
        width : "250dp",
        height : "250dp",
        backgroundColor : "#e0e0e0",
        borderWidth : 1,
        borderColor : "silver"
    });

First textField code:

var txtUsername = Titanium.UI.createTextField({
        height : "auto",
        left : 5,
        right : 5,
        top : "45dp",
        maxLength:32,
        borderWidth:1,
        borderColor:"black",
        backgroundColor:"white",
        font:{
            fontSize:"15dp"
        },
        borderRadius:7,
    });

Second textField code:

var txtPassword = Ti.UI.createTextField({
        //width : "90%",
        height : "auto",
        left : 5,
        right : 5,
        top : "90dp",
        passwordMask : true,
        maxLength:32,
        borderWidth:1,
        borderColor:"black",
        backgroundColor:"white",
        font:{
            fontSize:"15dp"

        },
        borderRadius:7
    });

The previous worked flawlessly on Titanium SDK 2.1.2.

I'm developing Android application using Google API 2.2 using Windows 7 64bit (if that matters).

  • The problem was with the "Force GPU rendering" option (in Developers settings) which was turned ON.
    After turning it OFF, the problem was resolved.
    However, there is a noticeable performance drop (lag) while scrolling a view with multiple textfields having rounded corners.
