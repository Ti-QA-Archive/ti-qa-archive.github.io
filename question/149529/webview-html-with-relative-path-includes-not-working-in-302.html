Webview HTML with relative path includes not working in 3.0.2

Going from Titanium SDK 3.0.0GA to 3.0.2GA, relative paths in my webview's HTML are no longer working (the path doesn't seem to point to the same place anymore).

Here's what I'm working with (simplified):

my webview:

var myWebView = Ti.UI.createWebView({
    html: htmlSource
});

htmlSource variable:

var htmlSource = '<html>\
        <head>\
        <script src="../html/js/jQuery.min.js"></script>\
        </head>\
        <body> SOME STUFF THAT USES jQUERY </body></html>';

The html folder is located in my resources directory in the project.

So, when I switched to Titanium SDK 3.0.2GA the path to my jQuery.min.js file can no longer be found by the simulator (or my device).

Is there a way I can change this path so it will again point to my html folder in the resources directory of my project?

Thanks!

  • I got it!

    I need changed the path to: app:&#x2F;&#x2F;Resources&#x2F;html&#x2F;js&#x2F;jQuery.min.js

    This is only for iOS (my app is only made for iPads), so I think it would need to be different for Android devices if you're wondering.
