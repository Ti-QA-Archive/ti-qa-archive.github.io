  • I don't see why you think the two techniques would differ in performance, nor why you're choosing to worry about this for optimizing the performance of your app. In either case, the performance cost of loading the image file from storage, decoding in memory, and rendering the pixels on the screen must be incurred. If anything, setting the background would probably take longer as the component would need to be rendered, then its background filled. There's no reason to set the background of an (empty) image view when you could just use the component properly and set its image attribute.

    Surely there are many other ways you could optimize the performance of your app than worrying about techniques like this.
