    You must be using a custom Info.plist file and the iOS build is having trouble parsing it. In order to help better, would you mind emailing me your Info.plist and I'll run some tests on it this asap. My email is cbarber@appcelerator.com. Thanks!

    • UPDATE: There is a bug in Titanium 3.0's iOS build script that affects people with <ios><plist> sections in their tiapp.xml file or custom Info.plist where there are settings that have <dict> values such as CFBundlePrimaryIcon.

      A fix is pending review and release in v3.0.1.

      Diff here -> https://github.com/cb1kenobi/titanium_mobile/commit/2e21d9e1cafe47c8d9ab5e3eec102e38632f4962

      Ticket here -> https://jira.appcelerator.org/browse/TIMOB-12105

      — commented December 15th 2012 by Chris Barber
