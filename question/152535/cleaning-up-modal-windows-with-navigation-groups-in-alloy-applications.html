Cleaning up Modal Windows with Navigation Groups in Alloy Applications

Hi all-

I have an application which launches a modal window. This modal window contains a form, and within the form, I have a NavigationGroup which a user can navigate between fields to select multiselect options within other views. The form also contains a cancel button, and the onClick handler performs the following:

$.myForm.close()

Here is the question…since the modal, and children windows of the NavigationGroup, contain eventlisteners, windows, models, etc, how much cleanup is necessary to do manually, or does closing the modal window close all children of the modal, inclusive of listeners? If not, would is there a suggested way on a close to traverse windows and unsubscribe from listeners, close children windows of a NavigationGroup?

I noticed that the form essentially creates a new model, and on launching the form multiple times, after close, the former listeners are in fact still hanging around and not closed from simply closing the modal window parent. I had presumed closing a parent closed children of that parent incorrectly, so just looking for best practices here on cleaning up windows and children/handlers from a window so I don't spin out of control with memory leaks.

Thanks for any guidance.
Chris
