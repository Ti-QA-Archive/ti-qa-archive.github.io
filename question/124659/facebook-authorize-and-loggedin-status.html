facebook authorize and loggedIn status

hi,

shouldn't checking the loggedIn status also check if the app is authorized on facebook or not? for example, say i login with facebook on my app then i go to facebook and deauthorize my app. if i go back to my app it should ask me to authorize again but it doesn't. the loggedIn always shows 1 even if the app has been deauthorized on facebook.com app privacy settings.

for example:

Titanium.Facebook.appid = "12345";
Titanium.Facebook.permissions = ['publish_stream', 'read_stream', 'user_photos', 'email'];

Ti.API.info('is Facebook LoggedIn: ' + Titanium.Facebook.LoggedIn);

if (Titanium.Facebook.loggedIn == true)
{
  userLoggedIn();
}
else
{
  showLogin();
}

if i deauthorize my app on facebook.com then try my app again it will let me go in because loggedIn is still true even though deauthorized?

HELP!

    Your app is using its own access token, so logging out using safari or the Facebook app will not make any changes to your application, as these are seperate logins and tokens.
