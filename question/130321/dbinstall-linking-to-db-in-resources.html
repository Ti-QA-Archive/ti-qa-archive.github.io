db.install linking to db in Resources

Hi All,

I have strange issue that just appeared tonight relating to DB. I use a db.install to install a preconfigured DB into a mobile app. This has been working great. Tonight after reseting the contents of the iPhone simulator (a number of times) it seemed that data was persisting in the simulator. I've gone through and checked that the reset deletes the application content in the Application Support directory, so it is blowing away all of the files correctly.

What I've discovered though is that the run in iPhone simulator no longer installs the DB as a file. It is now creating a SymLink back to the DB file in my resources directory. This is bad.

Up until tonight this worked fine and I can't see what I have done to change this (I've only been working on UI code), I haven't changed tiapp.xml settings and don't see anything that make this work like that.

Anyone else seen this behaviour?

Nick
