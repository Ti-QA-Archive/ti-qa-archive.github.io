Why loads the backgroundImage of createButton 3 times?

Hi,

If I got a backgroundImage for a button, it loads 3 times.

I think that is because, if you use a backgroundImage, a button also need an image for backgroundSelectedImage and backgroundDisabledImage. Well, that makes some sense.

But it also loads the backgroundImage 3 times when you've also used backgroundSelectedImage. That makes no sense. Why it should load 3 times the same backgroundImages and a backgroundSelectedImage?

That makes a total of 4 connections to the resources when only 2 are needed (well maybe 3 if it just needs an backgroundDisabledImage). The connection to the resources / finding the images one by one, takes the most time. So 50% more connections to make, makes my app 2 times slower loading time.

Now I got 21 buttons, so 42 images. Each images is 3.3kb. So lets say a total of 140kb. It's not the size of the images that makes it slow, it's the loading time, the time to find the images, the connection.

My app needs only 42 connections to the resources to find all the images needed. But now it makes 84 connection because of the backgroundImage. 50% less speed is alot.

So, can I do something about this? It's really annoying to know that the loadingtime of your app is unnecessary slow…

Here a log about this all

Here is the code I use for the buttons:

for(var x = 0;x<(maxbtn);x++){
    var newBtn = Ti.UI.createButton({
        pagelink: x, 
        width:btnWidth, 
        height:btnHeight,
        backgroundSelectedImage: 'images/buttons/btn'+x+'s.png',
        backgroundImage: 'images/buttons/btn'+x+'.png',
    });

    newBtn.addEventListener("click", btnCallback);
    v0cont.add(newBtn);
    data.push(newBtn);
}
