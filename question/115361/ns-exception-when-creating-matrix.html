  • These responses were v. helpful. Amazing how many bugs the FULL REBUILD can erase. But my experience of how to get that rebuild to happen is a bit different, so i thought i'd list the steps in the event that it may help others.

    I'm working with OSX and SDK 1.6.0 > i routinely do the following to force a full rebuild:

    1. Drag the iPHONE folder out of the BUILD folder onto my desktop. (If I delete the outer build folder, the bottom buttons including LAUNCH disappear - so i can't test anything)

    2. Go to the edit tab and click SAVE CHANGES (doesn't matter if you change anything on the panel or not - but is a necessary step as deleting the folder alone in #1 doesn't force a FULL REBUILD - the one that says "Hang on, this may take a few minutesâ¦"

    3. Quit the emulator, quit and relaunch Appcelerator Developer. Navigate to your project and click LAUNCH from the testing panel.

    This has fixed NS Exception errors when calling Ti.App.Properties as well as 2D Matrix transitions (standalone window animation with wobble)
