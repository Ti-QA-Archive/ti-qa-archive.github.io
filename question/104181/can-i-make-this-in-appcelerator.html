  • This is an old question for me, but for those interested in the app i created for iPhone with features and stuff here goes.

    CRM data sending request to webserver which reponds with an json string. Totally in love with this principle, send a HTTP request to a server and get the data you need.

    CRM data sync Instead of a sync, i created a hashtables, which represents all data a user recieved from the webservice. Reducing the amount of request to the server by ALOT. The more the user uses the app, it will use the webservice alot less and speeds it up alot more.

    This does gives 1 disadvantage, the data in hashtables can become outdated. To resolve this i added a refresh button that simulates the first time you get data and adds it to the hashtables.

    Social Media Twitter, facebook, linkedin and foursquare updates. These are all shown in 1 list of updates. Only user related info is being shown, so that the list is still manage able.

    Social Media 2 Showing comments on updates, in facebook you can have comments. These are shown and in addition you can leave a comment yourself. Meaning you can reply/comment on an update.

    Social Media 3 Creating new Updates for a social platform, this is something i'm going to work on in the future.

    LinkedIn Viewing a linked in profile in full details AND connecting with the person in question. Meaning in this app you can add LinkedIn users to your LinkedIn account.

    Foursquare like places An window which shows something similiad to foursquare like places.

    Skype Verry important was the ability to use skype, after long long searching i finnaly was able to use call a skype account. Though its external just like calling an phone number.

    Catch being called Is not possible on iPhone, sadly.

    Speech As this was lowest priority, i havent looked into this anymore.
