    • @Amri; sorry for jumping in; as you will see from the documentation on Ti.App.Properties.getBool it shows an optional second parameter. This is used to provide the default value if one does not exist.

      So using;

      // assumes 'firstTime' has never been set

// will show undefined as no property with that name exists
alert(Ti.App.Properties.getBool('firstTime'));

// this will show false as it uses the default when no property exists
alert(Ti.App.Properties.getBool('firstTime', false));

      This is how the comparison part of the if statement works. You should always provide the optional second parameter to give a default. The only time you should not is if you truly want undefined as the result.
