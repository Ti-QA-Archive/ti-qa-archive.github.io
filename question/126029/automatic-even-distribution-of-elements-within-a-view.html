Automatic even distribution of elements within a view

I have seen code samples where people deal with positioning of elements and orientation changes by explicitly re-specifying locations of elements after an orientation change. Is there a better way to do this if my goal is to (for example) have a view with 5 views as children of it. Each of the sub-views is set to occupy 16% of the width and starts at a set distance from the top of the screen, which will be the same in either orientation. I want them to be evenly spaced horizontally and distributed to occupy the full width of the parent view in either orientation. Is there a property that can be set on the parent view to do this or is the best way to detect the orientationchange event and give all the children views new 'left' properties?

    For each of the 5 children use a width of '16%', and for the left property use '2%', '22%', '42%', '62%', and '82%' respectively.
