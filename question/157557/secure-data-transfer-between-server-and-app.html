Secure data transfer between server and app

Hi guys,
I'm developing an app with the classical "remote data loading" (from a server).
I was thinking about a way to make sure that data can't be read clearly during the transmission (I mean an encryption method).
The app need an user account to be used and this the idea that I would like to realize:

When the user sign in the app passes to the server the device id of the user's smartphone (that is secret) and the server passes to the app an identification code (that is public) .
The server creates a new account with the relation public-code / secret-code.
Well… since the secret code (the device code) is passed just once time (during the registration) nobody won't be able to know it anymore.
So everytime the app needs data it passes to the server the request with the user identification code (that is public) and the server returns encrypted data (encryption on the basis of secret code) .

In this way nobody who is listening on the conversation between the app and the server could be able to understand what the server is telling to the app (that is confidential).

The question is: "How can I do the decryption of the server response on the basis of the secret code?

Is there any Titanium tool ?

Thank you in advance,

Luca
