Map Annotation w/custom image not draggable

Application Type: iPhone
Titanium SDK: 3.1.0 to 3.1.3
Platform: iOS 7 SDK
Device: iOS simulator & physical device
Host OS: OS X 10.9
Titanium Studio:T itanium Studio, build: 3.1.4.201310290744

If you have a Map View and add an annotation that has the "image" variable set with a custom image icon instead of a built in pin and you also have "draggable" set to true, you can not drag the pin around.

If you drag the pin around on the map it will pick it up and let you drag it to a new location. However, if you then drag the map around (not the pin) the annotation is locked into a pixel location on your screen and doesn't move with the map.

It is not locked into a GPS location on the Map View.

Here is a link to a video showing the bug in action.

Does anyone have a work around?

  • Just solved this, seems to work.

    Add the following code to line 609 in the didChangeDragState function of TiMapView.m:

    if (newState == MKAnnotationViewDragStateEnding) annotationView.dragState = MKAnnotationViewDragStateNone;

    Pullrequest added here: https://github.com/appcelerator-modules/ti.map/pull/36
