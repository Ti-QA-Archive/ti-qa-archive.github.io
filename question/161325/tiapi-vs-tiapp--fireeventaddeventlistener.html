Ti.API vs TI.App ( fireEvent/addEventListener)

Okay, it seems these two prefixes are interchangeable but they're not equivalent, one listener created with Ti.API will not answer to events fired by a Ti.App method.

Is this a bug or a feature?

  • If you read the Titanium documentation you will be able to understand the difference, I have linked you to both sections.

    • Ti.APP
      > The top level App module. The App module is mainly used for accessing information about the application at runtime and for sending or listening for system events.

    • Ti.API
      > The top-level API module, containing methods to output messages to the system log.

    Hope this helps!
