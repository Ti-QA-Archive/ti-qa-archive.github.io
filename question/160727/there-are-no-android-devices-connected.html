    I think this can be easily solved by killing adb and restarting it using the Command Prompt.

    adb kill-server

adb start-server

    If you don't have adb in your path, you can find it in C:\path\to\android-sdk\platform-tools\adb.exe.

    You can also run "ti info" to see if Titanium can see your devices:

    ti info

    If you have not setup Titanium yet, run:

    ti setup

    Generally, you only need to go through the "quick" setup. There's only 6 questions, one of which is where is the Android SDK.

  • As some of you, I was having exactly the same issue on Mac OS X 10.8.5.

    After upgrading to 3.2.0.GA I found that I couldn't run the application into Android. What I've found is that my Titanium CLI was attached to v3.1.2, even after upgrade.

    So, check ti info for the CLI version. If it's < 3.2, then you need to upgrade.

    ti info // shows cli 3.1

where ti // this will give you the path
// remove that path (I had to remove it manually, couldn't make it with npm)

sudo npm install -g titanium

ti info // now shows cli 3.2

    Hope it helps anybody.
