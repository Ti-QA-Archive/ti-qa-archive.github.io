Maintain session between httpclient and UIWebview

I am POSTing username and password to a login page for authentication to a website (using httpclient) but then I want to show a page from that website in a UIWebview.

I seem to lose the session in my UIWebview. How can I authenticate the user through httpclient but then also be able to see page from that site in UIWebviews?

What I am doing:

var xhr = Ti.Network.createHTTPClient();
xhr.open('POST', 'http://site/login.asp');
xhr.send({
userid: 'sdfsdfsd',
password: 'sdfsdfsdfsdf'
});

and then I am creating a UIWebview with the url: http://site/page/product45

but I am thrown back to the login page :(

  • I'm assuming that the login session in your web app is maintained using some sort of session cookie. The problem you will face is that you won't be able to set the cookies used by the WebView.

    But if you have control over the web app, you might be able to make some sort of gateway page that can log the user in and redirect him to the desired page.

    Example:

    http://site/applogin?key=XXXXXXXXXXXX&desturl=http%3A%2F%2Fsite%2Fpage%2Fproduct45

    The page at /applogin would read the key and set it into the appropriate cookie, and it would then redirect to the url specified in the desturl CGI parameter (in this case, http://site/page/product45). At that point, the user should be logged in and can view the product page.
