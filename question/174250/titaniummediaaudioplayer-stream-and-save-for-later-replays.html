Titanium.Media.AudioPlayer stream and save for later replays

Hi everyone,

I've been using the AudioPlayer to stream larger audio remote files in my application.

However, this application is to be mainly using 3G (or equivalent mobile data plan), and thus, must reduce the amount of transferred data to a minimum.

This turns out to be an issue since, on one hand I have large audio files that have to be played multiple times, and AudioPlayer doesn't give me an option (that I know of) to download the audio to a file, nor it allows to perhaps replay the whole thing without retrying the download.

I cannot use Media.Sound, since I require the file to be playing as soon as possible (instead of having to wait for a full load).

Is there a third approach I am missing to solve my issue? Has Titanium perhaps overlooked/oversimplified key features?

    First, if you are streaming large files, I am surprised that Apple (wait, you didnt mention your platform, thus this quite nearly qualifies as an incomplete question…) hasn't forced you to use HLS. HLS is supported in the iOS VideoPlayer. We have a series of apps that can stream and download audio files in the background for offline play.

    • Android simply streams the mp3 from a web server (com.kcwdev.audio module as we customize the controls).
    • iOS streams via HLS from the web server (videoPlayer).
    • Both can download the mp3 for local playback (com.kcwdev.audio /videoPlayer).
    Has Titanium perhaps overlooked/oversimplified key features?

    • we have a custom json feed to the app. In that feed, each item has a stream url and an mp3 url.
      The stream is an HLS that by default the ios videoplayer is used to play. The android client by default streams the mp3. We could have used the build in audioPlayer, however, because our videoplayer on ios has a custom set of rw, pp, s, ff buttons, to match the interface on android, we used a custom module purchased through the marketplace.

      The user can choose to download the MP3 for offline playing. We use a downloader module from the same guy that made the android audioplayer, to pull the mp3s down. When a user chooses an audio file, we check to see if we have a cached version. If the version is local on the filesystem, we play that instead of the stream.
