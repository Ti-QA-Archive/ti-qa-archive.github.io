2x Images in IOS

Question about image sizes with retina displays.

I know you can do the thing where you save two versions of a file.

my_image.png & my_image@2x.png

Of course the "@2x" version is shown if you have a retina display.

So for example my_image.png is 100x100 & my_image@2x.png is 200x200.

I often do not bother with the my_image@2x.png version. I just save my_image.png as 200x200. Then in my code I do this.

Ti.UI.createImageView({ width: &#39;100dp&#39;, height: &#39;100dp&#39; });

I realize there could be a performance issue on non-retina displays. As it has to load an image that is twice as big and scale it down.

Other than that is there any reason I am missing to save 2 versions of the file instead of one image twice as big?

Really a question to learn more about how and why to see if I am doing anything wrong.

Thanks.

  • Besides performance, there's also a visual penalty for image reduction. Having the ability to fine-tune your image is nice, if you care about what your app looks like on those devices.

    Also, Android is more stingy memory, and high-res images on devices that don't need them could run you into trouble.

    The only code feedback I have is that you should set hires:true too, if a single image is your approach.

  • The problem comes from a misunderstanding of the value of retina displays.

    Retina support is NOT creating 2x sized images and then cutting their resolution in half for the other devices.

    Nor is it taking your standard graphics and doubling the resolution.

    It is about creating graphics whose resolution is appropriate for the target platform.

    It is about offering retina users a better experience with enhanced graphics, more details, sharper lines, better shades.

    • So let's tweak Spicer's procedure just a little bit – rather than just upscaling his 100x100 and saving as 200x200, what if he started with a true 200x200 image and then used the 

      Ti.UI.createImageView({
    width: '100dp',
    height: '100dp'
});

      technique to make it the right physical size on screen. On a retina display, 100x100dp would show the full 200x200 pixels. On a non-retina display, presumably, the device would downsample the image automatically.
