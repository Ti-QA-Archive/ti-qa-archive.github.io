Whats the use of using classname

I want to know whats the use of using class name in a table view. How it is important from memory point of view? and how to use them efficiently. If I have a table view that displays various country names.So in this case how class name makes any difference becoz all of them have the same purpose.

var table1 =  Titanium.UI.createTableView({
    data:[
        {title:"ENGLAND",className:"layout_one"},
        {title:"IRELAND", hasChild:true,className:"layout_two"}

    ]    
});

win1.add(table1);

Thanks

  • The benefit is performance. I don't know if it affects memory usage as well.

    Ideally, table rows which have the same layout should share common className values. That is, rows with objects added should have the same types of objects and same placement on other rows with the same className – it doesn't matter that the values are different (e.g. title, label text, image used, etc).

    Rows that use a different number or placement of objects should not share a className with rows of another styling – but should with rows of similar style.

    This helps to optimize performance.
