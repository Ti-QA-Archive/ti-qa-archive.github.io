TableViewRow

I have 10 tableViewRow and each row different number of label and textboxes and buttons.
var row1 = Titanium.UI.createTableViewRow ({
height : 45,
selectionStyle : 'NONE',
});

and other rows are also declared in same pattern except the number of labels,buttons and textbox..
Will it be suitable to use a Single className property for all the rows.

i am Using Titanium Studio 3.0
and i am trying to build an android application.

    Hi

    You are right to use the className it really helps. You do need to have a different className for each different style of tableViewRow. In this case a style would be the collection of object.views inside.

    So if you have a tableViewRow with a single label, you would have one class name, another tableViewRow with two labels and an image - even if inside the same tableView - because they are two different sets of views and types of those views.

    Example 1;

    row1 = Ti.UI.createTableViewRow ({
    className: 'single',
    height : 45,
    selectionStyle : 'NONE'
});
var lbl = Ti.UI.createLabel({
    text: 'Title'
});
row1.add(lbl);

    Example 2;

    row1 = Ti.UI.createTableViewRow ({
    className: 'standard',
    height : 45,
    selectionStyle : 'NONE'
});
var lblTitle = Ti.UI.createLabel({
    text: 'Title'
});
row1.add(lblTitle);
var lblSubTitle = Ti.UI.createLabel({
    text: 'SubTitle'
});
var img = Ti.UI.createImageView({
   image: 'example.png'
});
row1.add(img);

    Example 3;

    row1 = Ti.UI.createTableViewRow ({
    className: 'standard',
    height : 45,
    selectionStyle : 'NONE'
});
var lblTitle = Ti.UI.createLabel({
    text: 'Something else'
});
row1.add(lblTitle);
var lblSubTitle = Ti.UI.createLabel({
    text: 'Different'
});
var img = Ti.UI.createImageView({
   image: 'test.png'
});
row1.add(img);

    Example 1 if different from 2 & 3 as their intervals views are different, but 2 and 3 are the same className as their structures are the same even if their information is different.
