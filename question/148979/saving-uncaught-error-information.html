Saving uncaught error information

I would like to know if there is a way to save information about runtime/uncaught errors that happen on a device.

In Javascript I usually add a global error handler to my apps. When an uncaught error happens, I send that information to the servers so that we can then study the error. In Javascript, I would use the onerror funcion as follows:

window.onerror = function() {
    //Send error information to server
};

However this does not seem to work in Titanium. Any idea how can I make it work? Or other good practices to handle uncaught errors?

Many thanks

  • HI,
    To create global error function you can do by creating app level object.

    var App = {};
App.onerror = function() {
    //Send error information to server
};

    create this App object in app.js at the top and it will be available through out your app.
