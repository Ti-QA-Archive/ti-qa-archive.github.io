  • The test case above is a mess, I'm not really sure what I'm looking at. I tried to clean it up but I'm not even sure what it's supposed to look like. Without a true test case, the best I can do is guess. And my best guess is that you shouldn't be using that "divClear" class with a bunch of empty views. It's inefficient and makes the markup far more cumbersome than it needs to be. It appears that you are only using it for spacing in your scrollview's vertical layout. For that you should instead use the "top" property to set the padding between elements. For example, instead of doing this:

    <Alloy>
    <ScrollView layout="vertical">
        <Label id="label1">label1</Label>
        <View class="divClear"/>
        <Label id="label2">label2</Label>
    </ScrollView>
</Alloy>

    do this instead and achieve the same result much cleaner:

    <Alloy>
    <ScrollView layout="vertical">
        <Label id="label1">label1</Label>
        <Label id="label2" top="14dp">label2</Label>
    </ScrollView>
</Alloy>

      • You're already using a vertical layout to position things, why are you adding a zIndex to everything?
      • Try using an ImageView for regular button instead of a View
      • I've seen issues using borderRadius in vertical layouts before with Android.
