  • You're biting off a big piece as your introductory project. This would be true regardless of the platform.

    XML processing is a separate issue from what you do with the data once you extract it. See our XML guide and the Ti.XML API docs for information on processing XML documents. By the way, traversing XML and extracting data is not specific to Titanium. The XML specs essentially describe the API that any language implements. So, a web search for XML manipulation should net some valuable information.

    As for models, once you get done processing your XML, I assume you'll have an array of objects representing your data. So, start with our Alloy Models guide, and make sure to read up on Backbone which serves as the basis for our models/collections implementation. There are examples there covering how to create/save models from arrays of data.

    Our KitchenSink application does not represent the best architecture for your app. In fact, it doesn't use Alloy at all. However, there are XML examples that you might find useful. You can import this sample directly from Studio. There are many other example apps in our docs and on our various GitHub repos.
