    Hi

    You have a lot of questions there, I will do my best to answer them all. But in order for you to appreciate what you can do and how - you have to actually start using the system.

    You created two different lists, although not that easy to read I will try and keep the answers to the same points.

    Questions

    1. Can the app comply with Android guidelines - yes certainly.
    2. Compatible with 2.x the minimum supported is 2.3
    3. You can have Action Bar and Tab Group
    4. Need apps to look consistent between 2.x, 4.x and 5.x
      • You currently fake it for the ActionBar on 2.x - most do (including me)
      • As 5.x does not exist it is impossible to say what changes if any will occur on the UI. ActionBar was introduced in 3.x and refined in 4.x, so there is no telling what will happen in 5.x
      • You can take advantage of Android themes
    5. You can give up pre-Android 4.x - many have, this reduces problems, not only in the UI

    Confusions

    1. Difference between AppCompat and Holo?
      • Holo and related themes are designed for Android 4.x
      • AppCompat is not so much a theme, more a way to gain the ActionBar available in 4.x
      • So they are not equal enough to compare
      • Yes AppCompat is supported from Ti SDK 3.3
    2. How will the AppCompat affect your app
      • Chances are very little, but of cuorse you will need to do some work to allow for it
    3. Which Ti Studio should you start with?
      • Start with the current GA, currently 3.2.3
      • There are many more things you have to code beyond the ActionBar - start on something and learn the platform
      • Stability is better when you are learning, if there is a problem - is it yours or the beta SDK?
    4. Which theme should you choice?
      • Your choice it is a visual thing, so either choose a pre-created one or customise a them for yourself
      • Read this - Action Jackson
      • Themes do not make an app more or less compatible - they change the appearance

    Suggestions

    1. Download the Ti SDK and start learning, why wait. Unless your app is just an ActionBar, you have lots to create within your app
    2. Review Android apps for guidance
      • Notice how most new ones have the same framework UI, but different colour schemes
      • NOT all android apps are identical
      • NOT all android apps are good - in fact most are really bad
    3. Design yours to follow the UI
      • But learn to make it different at the same time
    4. Learn from Googles Android apps
      • There are loads
    5. Understand that handling multiple aspect ratios, screen densities and scaled images will affect your development time much more than ActionBar will
    6. Create lots of test apps
      • Give each a purpose and over come the issues
      • Learn to write tight code
    7. Follow tidev.io all the cool kids are.
