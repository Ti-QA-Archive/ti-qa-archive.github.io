  • Yes. You need to fire the event from javascript like:

    <script type="text/JavaScript">
function fireLink(url){
    Ti.App.fireEvent('openLink', {
        url: url
    });
}
</script>

  • Your event listener looks good.

    The website should fire the event like this to get it working.

    <script type="text/JavaScript">
function fireLink(url){
    Ti.App.fireEvent('openLink', {
        url: url
    });
}
</script>

...


<a href="http://www.appcelerator.com/" onClick="fireLink('http://www.appcelerator.com/');">appcelerator</a>

    If you are not able to manipulate the website, you can inject this script before opening the webview. I've done this before, too and it works - it's just a little more complicated. Basically, you have to:

    • request the website with a HTTP request
    • parse the responeText with e.g. soupselect
    • alter the <a /> tags to trigger your function
    • set the manipulated body as your webview content

      Ti.App.fireEvent('app:openUrl', {url: this.getAttribute('href')});
