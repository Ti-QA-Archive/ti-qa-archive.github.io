mapview.setResion() bug?

I am developing iOS App using map view.
The following code is no problem on iOS5,

var mapview = Titanium.Map.createView({
    mapType: Titanium.Map.STANDARD_TYPE,
    animate:true,
    userLocation:false,
    top:0,
    left:0,
    width:320,
});
var region = {
    latitude = "30.63797";
    latitudeDelta = "18.73291";
    longitude = "-8.428525000000008";
    longitudeDelta = "298.80545";
};
Ti.API.debug(region);
mapview.startLayout();
mapview.setRegionFit(true);
mapview.setRegion(region);
mapview.currentRegion = region;
mapview.finishLayout();

but on iOS Simulator(Xcode4.5) and iOS6, The following warning was occurred.

[DEBUG] {
    latitude = "30.63797";
    latitudeDelta = "18.73291";
    longitude = "-8.428525000000008";
    longitudeDelta = "298.80545";
}
[WARN] Exception in event callback. {
    line = 234;
    message = "Invalid Region <center:+30.63796998, -8.42852497 span:+178.38431915, +450.00000000>";
    sourceId = 344842080;
}

Is it bug?
Do not you know a solution?

    latitudeDelta : "18.73291",
