Using getString in Alloy XML markup

I am currently running build 3.1.0.201304151600 and trying to get the Ti.Locale.getString (or L() for short) call to work in the XML markup for an Alloy widget. So far I have discovered the following information:

This previous question seems to imply that it cannot (as of 10 months ago) be done in the XML markup: Does Alloy support internationalization? How?

I then found the following, which allows the use of Titanium constants in XML markup: Accept Titanium SDK constants in the XML markup

I have tried the following methods of making the localized string show via XML markup:

<Label id="lblTest" class="headerLabel">{L('test_string')}</Label>

As expected, this only shows the string "{L('test_string')}" in the label.

<Label id="lblTest" class="headerLabel" text="{L('test_string')}" />

This fails compilation with the following error: ERROR: Unexpected token name «test_string», expected punc «]»

At this point, I am at a loss for what to try next, or if this is even supported. Some guidance would be greatly appreciated. Let me know if any more information is required to assist in solving this.
