keyboard issue

This is happening for android but it probably will happen for iPhone also.
I have a table view open and i click on post to Facebook.
this opens another window (moda in iPhone) which has a text box. If you are logged in it focuses on the text box and the keyboard opens.
If you are not logged it ip goes to your Facebook app and asks you to authorize the app. Once you come back to the text box to post the keyboard is closed but the text box is still there. It looks kinda stupid and they have to know to click on the text box to bring up the keyboard.

Any idea on how to get the keyboard to show back up when the window is showing again. I have added an focus event to the window but it did not work because the text area is already focused. I tried to add an on blur to the window to blur the text area when we leave to go auth the app but it also blurs when you type on the keyboard.

    Ti.App.Properties.setBool('checkLogin',true);

    put this condition in your window focus event.

    if(Ti.App.Properties.getBool('checkLogin'))
