Settings screens does not work with 1.8.0.1 RC1

I think I've found a critical bug in Ti SDK 1.8.0.1 Release Candidate.

On iOS, setting screen is not installed on device (but works on emulator). I use the Settings.bundle/Root.plist file included with KitchenSink. It used to work with sdk 1.7.5. Same result with various projects.

Can anyone confirm before open JIRA ticket?

To try this: uninstall KitchenSink (Titanio) from your iPad. Build kitchensink with SDK 1.8.0.1 RC and install on device, check that Titanio is installed on device, goto your iPad settings screen and look for the Titanio settings page.

thanks,
Javier

  • Accepted Answer

    Javier, with 1.8, the settings bundle files need to go in the project/platform directory rather than project/modules. So, move project/modules/iphone to project/platform/iphone, clean build and you should be good to go. You might have to uninstall the app from the simulator / device first.
