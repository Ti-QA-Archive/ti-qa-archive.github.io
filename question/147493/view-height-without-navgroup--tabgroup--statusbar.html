View height without navGroup / tabGroup / statusBar

Hi, what is the best solution for obtain accurate view height ? In some windows I have navGroup + tabGroup somewhere only statusBar and I need accurate view height size because view is moving ( From down to top: 0 and then back down [hide] ).

Problem is that my application si for iPhone, iPad ( Also Retina ) and also for PORTRAIT / LANDSCAPE orientation. Still set / calculate pixels for view is not good idea…

    Please look into api doco

    The rect property is a read-only dictionary with the properties x, y, width and height. It provides the rendered size and position of the view, and is only available once both it and its ancestors have been fully drawn.
