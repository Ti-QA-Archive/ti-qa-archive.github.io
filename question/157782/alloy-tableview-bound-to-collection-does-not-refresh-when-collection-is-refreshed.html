Alloy tableView bound to collection does not refresh when collection is refreshed

I have a collection of data based on an entity called a case. That data is displayed in a tableview. The user can change the caseid to view data from a different case. When the user returns to the window containing the data, I am initiating a fetch on that collection with the new case id as the criterion for the fetch. However, the table does not update. What am I missing?

Any insight would be greatly appreciated.

Application Type:Mobile
Titanium SDK: 3.1.2GA
Platform: iOS 6.0 - physical device
OS = OSX 10.8.4
Studio: 3.1.1.201306112235

View Markup

        <TableView id="tbleventlist" dataCollection="events" dataTransform="transformFunction" dataFilter="filterFunction">
            <Require src="eventrow" />
        </TableView>

Controller Code

function filterFunction(collection) {
        return collection.where({
            caseid : Alloy.Globals.currentcase.id
        });
    }


    $.analyze.addEventListener('focus', function() {
        alert('before fetch');
        Alloy.Collections.events.fetch({
            caseid : Alloy.Globals.currentcase.id
        });
        alert('after fetch');
    });
