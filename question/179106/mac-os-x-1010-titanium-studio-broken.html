  • Hello Petr,

    Thanks for your inquiry. What Ti SDK you are using?. Download Ti SDK 3.4.1.GA from here. Extract the zip file and put the file in "Library/Application Support/Titanium/mobilesdk/osx". Restart the studio.

    Check your "Studio>Dashboard>Get Started>Configure Native SDKs". Is there a red cross before "Android SDK" and "iOS SDK"? If so, try configuring each SDKs from there. The option should display on the right.

    If you want to use Xcode 6.1 in addition with Xcode 5.0 then try installing Xcode 6.1 as "Xcode 2.app" in your application. And run 

    sudo xcode-select -switch /Applications/Xcode2.app/Contents/Developer/

    in terminal to configure iOS SDK. Or, if you are only using Xcode 6.1 in your Mac try runing 

    sudo xcode-select -switch /Applications/Xcode.app/Contents/Developer/

    Hope this helps. Feel free to response. Thanks.

  • I get the same issue with a brand new build of 10.10 and loading the new Appcelerator Studio. I try to install the SDK as it prompts me to and it opens the "login to grant permissions" but then it does nothing. The problem I see is that Appcelerator USED to ask for write permission before it would open (similar to Windows "Run as administrator" rights. Since Appcelerator no longer runs with elevated rights, the system fails to do anything even after it prompts the user to update. This isn't an issue with 10.10 as I had this working just fine before I reimaged my laptop, this has to be some update in the Appcelerator Program.
