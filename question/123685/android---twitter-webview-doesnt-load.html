Android - Twitter Webview Doesn't Load

Since Twitter changed their mobile website, it has often failed to load from within a webview but succeeded in the browser. You just get a continuous spinning wheel, that the twitter site displays.

After some searching, I found on StackOverflow that several users recommended either setting Javascript enabled on the webview OR less preferably faking an iPhone user agent string (see here).

Is there any way I can accomplish this in Titanium? While it would be trivial natively, the properties I need don't look exposed from the source.

  • I did find some workarounds, for Android I had to modify the Titanium source to expose the userAgent and fake it out so that it believes it is an older iPhone so it still displays the older website. My pull request only recently made it in (taking about 90 days for Appcelerator to approve my exposing a native property).

    For iOS, I was finding if the script took over 10 seconds to load I would have to stop it. However, that now seems better and I don't use any special code for iOS anymore.

    I provided a lot of details to my solution in the answer to this question, which surprising someone voted down the answer. Vote it back up to zero if you feel it helps.
