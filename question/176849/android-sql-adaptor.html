Android SQL adaptor

Just trying to develop my iOS app into an Android app, unfortunately getting halted at installing the database.

I am getting the following error:

[ERROR] V8Exception: Exception occurred at alloy/sync/sql.js:290: Uncaught config.adapter.idAttribute "id" not found in list of columns for table "properties"
[ERROR] V8Exception: columns: []

Installation is invoked by:

Ti.Database.install(&#39;master.sqlite&#39;, &#39;properties&#39;);

I have an alloy model of:

exports.definition = {
    config: {

        "adapter": {
            type: "sql",

            collection_name: "properties",

            db_file: "/master.sqlite",

            db_name: "properties",

            idAttribute: "id",

            remoteBackup: false
        }
    }
};

Do I need a columns object with Android? Why cant these things be the same. This is all working on iOS :(
