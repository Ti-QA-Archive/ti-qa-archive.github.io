How to emulate android horizontal layout behaviour in iPhone when using labels and images combined.

Hi there,

Android works like a web browser. You add words and images in the same line and everything works as expected. Words that don't fit in a line go to the next one, too big words are cut and continued in the next line, images appear between correct labels…

In iphone words are often ellipsized (if the text height is set) even if there is no need for it. When a word is too big for a line it creates a column for that word between images… i've tried many options but none works correctly.

This is a code you can try in both platforms to see the differences:

    var base = Titanium.UI.createWindow({
    backgroundColor: '#ffffff'
    });

    var container = Titanium.UI.createView({
    layout: 'horizontal',
    });

    base.add(container);

    for(var i =0; i<60;i++)
    {
    var redbox = Titanium.UI.createView({
    width:30,
    height:30,
    backgroundColor:'red',
    top:0,
    left:0
    });

    container.add(redbox);


    var label1 = Titanium.UI.createLabel({
    text: 'rgbgngurewgvrewv grevwgrewgrewcgrewvgrewyres ',
    top:0,
    left:0,
        height:Ti.UI.SIZE,
    });

    container.add(label1);

    var label2 = Titanium.UI.createLabel({
    text: 'rsgfds ',
    top:0,
    left:0,
        height:30,
    });

    container.add(label2);
    }

base.open()
