Android: How to Move Image to SD Card on Emulator?

Sorry for the dumb question but I can't seem to figure it out. Titanium Studio starts up the Android 2.2 emulator with this (according to the console output) on my Mac OS X development machine:

/Users/ryan/dev/android-sdk-mac_x86-1.6_r1/tools/emulator -avd titanium_11_WVGA854 -port 5560 -sdcard /Users/ryan/.titanium/titanium_11_WVGA854.sdcard -logcat *:d,* -no-boot-anim -partition-size 128

I want to push an image to the emulator's SD card so that it's available in the gallery app. With the emulator running, I use this command:

./adb push /Users/ryan/Desktop/b-70-rollout.jpg /Users/ryan/.titanium/titanium_11_WVGA854.sdcard

But it fails with this messasge:

failed to copy '/Users/ryan/Desktop/b-70-rollout.jpg' to '/Users/ryan/.titanium/titanium_11_WVGA854.sdcard': No such file or directory

I've verified the path to the image is correct (I use the tab to auto-complete to the image in Terminal) and I see that the "/Users/ryan/.titanium/titanium_11_WVGA854.sdcard" file exists also.

What am I doing wrong? Why won't the image push to the SD card image? If it helps, I didn't do any of the SD card setup; I assume Titanium did it when the emulator image was created. Also, with the emulator running, I see there is a "/Users/ryan/.titanium/titanium_11_WVGA854.sdcard.lock" file, which makes me think the emulator is using it (as expected).

Thank you!!!!

-Ryan
