  • Hello, Alloy models are built on top of Backbone.js; Viezel's restapi adapter overrides the default sync method of Backbone for models and collections which use it (adapter.type = 'restapi') thus enabling to perform CR(eate) U(pdate) D(elete) operation consuming a restful api on you server.
    The adapter maps CRUD operation to the standard HTTP verbs (GET, POST, PUT, DELETE) at line 98, and the choice is made upon the Backbone method you are using on your model or collection. I.e. when you create a new model and save it it will perform a POST, when you fetch() a collection or a single model it will use a GET method.
    I don't clearly understand your need to change this mapping, or the way you intend to use the adapter. Could you try to describe what you want to achieve, to understand better your problem? Regards.

    • Hi Mahmoud,
      I'm afraid the 'type' member you defined in model's config will not work as you wish.

      Let's say you have a model 'book' that is using the restapi adapter for persistence on a remote db through a restful api implemented by your server application:

      In your client app you can create a new book record with

      var book = Alloy.createModel('book', { title: 'Ubik', author: 'Philip K. Dick' });

      then you save() it in order to store the new record in the remote db:

      book.save();

      The save() method will send to the server you new book record, using a POST request.
