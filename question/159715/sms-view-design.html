    • Ok a few pointers;

      // this is not cross platform compatible
chatView.add(chatTable, toolbar);
// this is MUCH better
chatView.add(chatTable);
chatView.add(toolbar);

      You should not add a tableView to a scrollView it will not work as well as you probably hope it will unless you really know what you are doing and again cross platform wise - Android will have an issue with this - when/if you get to it.

      You are better off creating this with a scrollView and removing the tableView. I would also consider creating the toolbar as a custom view containing a textField and any other buttons, removing the system toolbar - then changing its position on textfield focus and blur. Add the custom view directly to the window.

    • How about this module?

      • titanium-keyboard-listener

      It is designed to listen for the keyboard position change so you can respond to it with your own code to move things around. I think it will solve your problem.

      You can use it like this;

      container.addEventListener('keyboard:show', function (ev) {
    Ti.API.info('* keyboard height is ' + ev.keyboardHeight + ', and my height is now ' + ev.height);
});
container.addEventListener('keyboard:hide', function (ev) {
    Ti.API.info('* keyboard height was ' + ev.keyboardHeight + ', and my height is now ' + ev.height);
});
