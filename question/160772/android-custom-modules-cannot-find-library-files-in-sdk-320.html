  • I am having the same issue here (and it sounds like you're using the sql encryption module that we wrote). It looks like a problem with the build.py script for Android in 3.2.0. It's not including libraries from the ./platform/android/libs folders into the final apk file.

    — commented December 23rd 2013 by Mark Burggraf

  • Here's a workaround we have found for now:

    In the module, copy the *.so files from the /platform/android/libs folders into the /libs branch. There should be 6 files total (2 files for each platform.) This fixes it.

    — commented December 23rd 2013 by Mark Burggraf

  • To clarify for future readers, directories he is referencing are found on OS X at ~&#x2F;Library&#x2F;Application Support&#x2F;Titanium&#x2F;modules&#x2F;android&#x2F;com.dmarie.sql&#x2F;&lt;version&gt; (assuming your Titanium modules are stored in the default locations).
