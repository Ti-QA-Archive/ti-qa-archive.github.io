Mac GenyMotion Google API Install Help Needed

I'm about at my wits end here. Currently I have the following installed:

Mac OSX 10.9.5
VirtualBox 4.3.20
Genymotion 2.3.1

I've got several virtual machines installed but the one I'm currently working with is for the Galaxy S4. Since native Google support was dropped from Genymotion a while back, I've seen people install the apis on windows boxes so Genymotion loads apps like the Kitchen Sink. On mac though, nothing's working.

I've downloaded multiple versions of the google api's as well as an ARM translation file. On Windows, you can drag and drop these files into the open emulator and they will auto install. On Mac however, it just says this:

Files successfully copied to: /sdcard/Download/

When I try to access the download area to potentially manually install, they aren't even there. I've also opened up VirtualBox and changed the following settings:

General / Advanced / Drag'n'Drop: Bidirectional
Shared Folders: [path to unzipped google apis]

Rebooted and still no installed Google Apis. Tried dragging and dropping both zipped and unzipped versions again and still got the same "Files copied" dialog.

I would like to run the KitchenSink on Android and I've also got some previously deployed Android apps that need updating, which use the Google apis. Without the emulator working, I can't get in there and update them properly. I've really got to get this app updated.

Any and all assistance would be greatly appreciated. Thanks guys! :)

  • I figured it out. For anyone else who is having this problem, this is what happened:

    My mac auto unzips zip files when downloaded. If I re-zipped them and dragged or tried dragging the unzipped files, they wouldn't flash. I had no idea how to turn off this feature, so I dl'd them on a windows computer and transferred them over. Once the new files were on the mac I dragged them over and they flashed. Kitchensink is now working. Hope this helps someone else.
