    Hi

    As Muhammad Wahhab Mirxa points out the web page you reference populates the contents of that tab dynamically in script - so initially the element is empty and thus cannot be accessed by YQL.

    Bad news over - good news begins.

    Because the content is provided in script it is actually a JSON asynchronous call to another page that provides that data. So you can simply call this URL and you will see a JSON result set with the data you are after.

    http://www.rccgna.org/DesktopModules/mspCalendar/GetJsonAllEvents.ashx?pid=0&tid=118&mid=491&_search=false&nd=1356260033327&page=1&rows=20&sidx=&sord=asc

    Click the link you will see everything you need.

    You can even increase the rows parameter to give you all results not just the first 20. The returned JSON includes the HTML formatting so if you do need a webView then you do not have to do anything else. If you want to parse the content then a few well placed regular expressions will solve that.

    You will of course need to add an Ajax call (web based httClient) inside the page, jQuery, some other library or your own version will solve that.

    Enjoy

    • Below is a complete test app to take advantage of the JSON feed you get back.

      It will parse the results into re-usable data and populate a tableView - this way you do not need a webView and can fully integrate with your app;

      To give this a test, simply create a new default app project and replace the entire app.js with the contents of this example code;

      var win = Ti.UI.createWindow({

function filterNode(node) {
    var temp = node;
    temp = temp.replace(/<strong>([^>]+)<\/strong>|<b>([^>]+)<\/b>|<a[^>]+>([^>]+)<\/a>/gim, '');
    temp = temp.replace(/<[^>]+>/gim, '|');
    temp = temp.split('|');
    function hasVal(element, index, array) {
        return element !== '';
    }
    var data = temp.filter(hasVal);
    return {
        start: data[0],
        end: data[1],
        allDay: ((data[2].toLowerCase()) === 'yes' ? true : false),
        location: data[3],
        description: data[4]
    };
}
function addRow(obj) {
    var row = Ti.UI.createTableViewRow({
        backgroundColor: obj.isOdd ? '#fff' : '#eee',

        width: Ti.UI.FILL

    var outer = Ti.UI.createView({
        bottom: 5,
        height: Ti.UI.SIZE,
        layout: 'vertical',
        left: 10,
        right: 10,
        top: 5,
        width: Ti.UI.FILL

            left: 0,

            top: 5,

            top: 5,

}
function addEvents() { 
    var url = "http://www.rccgna.org/DesktopModules/mspCalendar/GetJsonAllEvents.ashx?pid=0&tid=118&mid=491&_search=false&nd=1356260033327&page=1&rows=20&sidx=&sord=asc";
    var client = Ti.Network.createHTTPClient({
        onload: function (e) {
            var data = JSON.parse(this.responseText);
            var intEvent = 0, intEvents = data.rows.length, element, cell, event, rows = [];
            for (intEvent = 0; intEvent < intEvents; intEvent = intEvent + 1) {
                element = data.rows[intEvent];
                cell = element.cell[0];
                event = filterNode(cell);
                event.isOdd = intEvent % 2;
                rows.push(addRow(event));
                Ti.API.info(intEvent.toString(), JSON.stringify(event));
            }
            tbl.setData(rows);
        },
        onerror: function (e) {
            Ti.API.debug(e.error);
            alert('error');
        },
        timeout: 5000  // in milliseconds
    });
    client.open("GET", url);
    client.send(); 
}

var btn = Ti.UI.createButton({

    title: 'Load Events',
    top: 5,
    width: 200
});
btn.addEventListener('click', function (e) {
    addEvents();
});
win.add(btn);

var tbl = Ti.UI.createTableView({
    height: Ti.UI.FILL,
    top: 50,
    width: Ti.UI.FILL
});
win.add(tbl);

win.open();

    • The extraction method is one of many - in this case Regular Expressions. They can be your friend and your enemy - when you realise you can do so much but are just not sure how.

      Places to learn;

      • Tutorial
      • Object Reference
      • Examples
      • Web based test bed - I use this all the time

      What my versions did.

      • My aim was to reduce the html snippet to end up with just the data
      • http[^"]+ finds any content that matches a simple url synatx
      • <a[^>]+>([^<]+)<\/a> finds any tags containing an a anchor
      • then filter that down to the first match [0]
      • then remove all the html content <[^>]+> leaving just the title
      • <strong>([^>]+)<\/strong>|<b>([^>]+)<\/b>|<a[^>]+>([^>]+)<\/a>
        • <strong>([^>]+)<\/strong> strong tags or
        • <b>([^>]+)<\/b> bold tags or
        • <a[^>]+>([^>]+)<\/a> anchor tags
        • replaced with empty strings
      • <[^>]+> removes any html tags and swaps with | (simple marker)
      • finally runs a filter over the remaining content and ignores all empty strings

      The balance is the fields we are after, this technique does rely on the position of the content not changing order. But works very well.

      This looks very complicated but if you deal with one at a time it really starts to get easy (easier).
