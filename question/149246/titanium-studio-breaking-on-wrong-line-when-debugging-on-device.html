Titanium Studio Breaking On Wrong Line When Debugging On Device

Application Type: mobile
SDK: 3.0.2
Platform: android 4.0.3
Device: Android tablet (motorola xoom)
Host OS: Win 8
Titanium Studio: build: 3.0.2.201302191606, jenkins-titanium-rcp-master-372 (origin/master), 19 February 2013, 16:09:49

Case 1)
I add a breakpoint to the editor in titanium studio. It stops there but debug information is referring to the wrong line of code (in the last case, it stopped on the right line, but debug info (variables, etc.) was related to the line above where it stopped).
