  • Accepted Answer

    If you're using a TabGroup, you need to open the window by getting the active tab and calling open() on it. See more details in this post.

    I would also recommend that you don't create your windows with the url property – that will run the code in another context; better to design your other windows as CommonJS modules and use require() to access the classes you define.

    — answered July 10th 2012 by Jason Priebe

    • Sorry – I should have given you a little more info. If you don't open windows in the appropriate fashion, you will run into strange behavior like this, where event listeners stop responding, or the app becomes unstable.

      With Ti mobile, I find that you need to gain a good understanding of these nuances, and you need to stick as closely as possible to the best practices outlined by Appcelerator. Unfortunately, there are a lot of bad code examples out there (most notably, Kitchen Sink).

      I think if you adjust your code appropriately (use Tab.open() instead of Window.open() and use CommonJS modules), you'll find that some of the strange behavior will go away.
