New to Titanium on the Mac? Here are a few tips that might save you some time and frustration

/*
tl;dr; 
just skip to the dark patches below for the answers to the issues i had over
the past couple weeks while getting started with Titanium Studio.

hope this helps someone else.
*/

I've seen a few of these tipsheets. After searching a couple of them I couldn't find anything specific to installation and slow performance on a Mac which has been my experience over the past two weeks. I ended up piecing this together from several posts here. I think this list would have helped me break into what I believe is a reasonable workflow so I thought I'd share.

Getting Set Up and Started

slow performance after installation?

oh man, this one was killing me for about a week. i can't stand waiting for computers because they make you wait in these useless little chunks of time that you can't do anything with.

when I installed Titanium Studio 1.0.8 under my normal user account on my Mac I had all sorts of problems. i left 2 posts here that were answered pretty quickly (thank you!) about how dog slow the thing was and I read about another dozen or two other posts with quotes like "how can you people work like this!?!" and what not.

i'm talking like EVERY click took 2-3 minutes watching the rainbow wheel or whatever. it was crazy, but i don't want to talk about it or else i'll get worked up again.

well, at some point over the past couple weeks i landed on a simple, elegant suggestion here: factor out any kind of corrupt files that my user account may be introducing to the mix by creating a new user account. :) sweet.

(FYI, i had used Aptana Studio before it was bought by Appcelerator as well as Eclipse to mess around with Android development last year so there's probably like 23 million orphan configuration files all over the place on my laptop under this account.)

/*
once i created a new user account on my Mac and installed Titanium Studio 
from there suddenly everything started working like a charm. (it warned me 
about overwriting the one already in Applications but I ignored it, turns out 
that wasn't what made the difference, I believe it was the configuration 
files under my account that disappeared under the new account)

i'm now creating new projects in under 10 seconds (note about a quirk 
there later) and working happily.  so glad i found that post ... now where 
was it again?
*/

multiple user accounts on a Mac

so some of this is dead obvious to folks who love their command line on sunday mornings but for me it took almost an hour of fiddling to figure it out.

once i got past the slow performance issue above by creating another user account on my mac, i ended up getting python errors when trying to build a shiny new mobile project. i changed my Run Configuration (see below) to show Debug messages and found that it was a security issue. the new account couldn't access a .txt file in the titanium mobilesdk installation folder under my original account.

i spent about 20 minutes messing with chown and chmod before i admitted i had no clue what i was doing.

/*
then i decided to copy the entire titanium mobilesdk folder to the Shared 
folder and set its permissions wide open using chmod a+rw or something and 
that seemed to allow me to build on both accounts just fine.
*/

Visibility

by this i mean, "what's really going on when i click this button in Titanium Studio?"

the best single switch to flip on visibility when you get started (as far as I've found) is under Run Configurations (and Debug Configurations if you're debugging).

/*
you just set the level of Run/Debug Configuration logging to DEBUG instead of 
INFO (the default) and watch the juicy details fly by.

ouch, how obvious was that right? i shoulda known.
*/

not a big deal but just watching that python script output (i think) and other details has saved me hours of wondering what happened and chasing my tail.

Quirks

python errors when you build?

after a while i ended up corrupting the /Library/Application Support/Titanium/mobilesdk folder again somehow. i saw the path as part of a python script that had exited prematurely during a build so I just decided to blow away the whole folder (above) and Titanium Studio automagically re-downloads it when it starts.

so that wasn't such a big deal either.
