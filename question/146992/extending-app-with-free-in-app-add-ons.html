Extending app with free "in-app" add-ons

What is the best development strategy for creating an app with basic functionality, which is then extensible with more features post installation?

So for example, the basic application allows you to create a user profile and retrieve some data from a service. But then, the app would list plugins one could download, to retrieve further data from the service, or do something nifty with that data, like trend it, graph it, etc.

How would one go about architecting such an app? Can "in-app" downloads from the marketplace be free? How does one then implement the basic app to reflect new functionality from "in-app" downloads.
Advice or experience anyone can relate would be extremely appreciated :)

    Hi

    This response is biased towards iOS as that is what you are starting with, the basic principles are the same for Android but should research this with both stores yourself when signing up.

    An "in-app-purchase" (IaP) is an actual purchase so you cannot offer them for free. There is an exception though but ONLY for free subscriptions, but this is ONLY for news-stand apps.

    In order to add IaPs you add them to the App Store as a list of products that can be purchased and assign at least a name and price to each - you can have 10,000 of these for each app - so go wild.

    You are responsible for making your app IaP capable and this includes creating the required setting changes for your app certificate.

    You then access the list of IaPs and present them to the user as a list of things they can purchase, which they do with their own iTunes Account details. The AppStore validates the purchase and returns the sale receipt back to you - MAKE SURE you validate this as some naughty people exploit this communication.

    There a few types of IaP but two of the main ones are;

    • Consumable; this is something that can be bought more than once; think gold coins for points to get to the next level in games

    • Non-consumables; these are for things like extra access to the app, example; pro version or a free app or to remove the ads in a freemium model app.

    If you offer non-consumables you MUST allow the user to restore those purchases without further charge.

    Choosing the wrong type of IaP will likely get your app banned - so make sure you understand the differences.

    To make things easy you can use the appcelerator marketplace module, this comes with example code to show you how to how to set it up. You will need an intermediate knowledge of appcelerator and iTunes Connect, as well as patience for your first run. It gets easier - there are just lots of steps when you include proper tests into the mix.

    Plan, plan - plan some more, make notes - it makes it easier for the next time.

    Good luck.

    • So a few ways for you to look;

      If the extra features are files, data or anything you could download in a browser

      • Add the content to a server (or CDN service)
      • Create a features API that pulls down the current list of features available in this technique
      • Assign each feature a title and ID at the least
      • When the user chooses a feature from the list you present them - cache the IDs they chose and download the content
      • Next time the list is shown either hide or disable the features already chosen

      If the content is logic;

      • Consider moving some code to the cloud in the form of node.js support
      • If you do not want or can not use the cloud logic style, you will need to add the features to the app and re-submit to the store
      • Use the same cached IDs logic as in the previous block.

      This is very project dependant and there are many combinations.
