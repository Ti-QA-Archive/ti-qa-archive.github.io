setRemoteBackup(false) for local files?

There's already similar questions, like this, but it doesn't exactly answer my question.

If I place a file (PDF in my case) to my project's Resources / iphone directory, will it be backup up to iCloud / local iPhone backup file? In other words, will my PDF file be accessible to a end user in any way?

In my case is important to copy protect the PDF file in way that it's accessible only via iOS App.

For downloaded files, I'm able to set the remoteBackup to false, but how to do it with file in Resources directory, since it's never opened as a file object? (Titanium.Filesystem.getFile)

    Files in your resources iphone directory are not backed up to iCloud or any other place, they are only available from within your app.
