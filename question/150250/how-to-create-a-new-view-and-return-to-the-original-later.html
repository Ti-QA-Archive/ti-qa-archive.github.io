How to create a new view and return to the original later

Hi all,

// creating a main window, adding a view, opening it
var win =  Ti.UI.createWindow();
var view1 = Ti.UI.createWebView( { url: url } );
win.add( view1 );
win.open();

// display a new view using a third-party module
var view2 = thirdPartyModule.createView();
win.add( view2 );

I'm using a third-party module as above. It displays the second view correctly and all UI on it work, but when I close the second view, the app returns to the first view but it no longer responds to any touch.

It seems that the app is still running (not crashed) because on Android the hardware-buttons respond correctly as I coded.

My question is, what is the right way to switch the view so that I can return to the first view without losing control of it?

  • very possible that the the second view is still "there" just hidden, blocking any events to the main view behind it. This could be because the view is not closed properly.

    you should be able to test this by assigning a z-index to the windows - the higher the number the higher the view is on the stack - so for example, if you were to swap the main view z-index with the second view after you close it you could see if this is the case.

    Addtiionally - you can cleanup the view by doing the following:

    win.remove(view2);
view2 = null;

    as for capturing an on close event for your modules view/window - i'd love to help out, but not knowing what 3rd party module your using, i can't check out the documentation to help :)
