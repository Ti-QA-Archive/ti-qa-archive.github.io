Why aren't the generated Titanium source files precompiled in a static linked library?

Often times when I'm working on an iOS project, I will open the "build" directory generated by Titanium Studio and open up the Xcode project to do some advanced debugging and profiling.

However, compile times are very very slow, even for small projects because Xcode has to compile many many source files on each build (all of the Ti* classes, as well as ASI, AudioStreamer, FBConnect, JSON, etc.)

This seems unnecessary. Why aren't these source files bundled into a static linked library, or at least included in a precompiled header? Is there a specific reason for this? It seems like this is a simple optimization that should have been done already, so I wanted to ask the community about this to see if I'm missing something.

If there are no major roadblocks in doing so, I want to customize my installation of Titanium to do this so I am not so heavily taxed each time I want to compile.

— asked October 2nd 2012 by Donald Ness
