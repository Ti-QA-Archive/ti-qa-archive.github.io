How to upgrade an Alloy project

I started working on an Alloy project a few months ago. I recently updated Titanium Studio and all related packages. I am running the latest Titanium SDK (3.4.1 GA) which includes the newest version of Alloy (1.5.1).

I already updated my project's tiapp.xml to use the latest SDK. I'm not sure if there is anything else that I need to do in order to upgrade Alloy in my project.

I am concerned that some of the project files may be outdated. In particular, I see that nothing has been changed in the \plugins\ti.alloy\ folder and the \hooks\alloy.js file still says "Copyright (c) 2012".

Aside from updating the SDK version in tiapp.xml, are there any other steps needed to upgrade my Alloy project? Thanks!

    Alloy versions are usually released in step with the SDK (though the version numbers differ). Updating, either via Studio's prompts & dialogs or with the npm update command, will install the latest version of the Alloy libraries. You have the latest released version of Alloy. So there's nothing there for you to update.

    As an aside, (and I'm not a lawyer) by international convention, copyrights last the life of the author (Appcelerator, Inc) plus 75 years. Thus there's not an imperative for the project's author to go through each of the many files in a project to update a copyright statement for each release. You're better off looking to the changelog file, or the readme file for version info. Keep in mind that we don't update any files we don't have to with each release, to minimize the necessary testing we must do.
