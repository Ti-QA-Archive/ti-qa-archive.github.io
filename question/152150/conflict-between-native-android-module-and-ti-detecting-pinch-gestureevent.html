Conflict between native Android module and Ti detecting pinch gesture/event

I have a native Android module that renders PDFs, and it's working great, with one issue. I have an onTouchEvent in my .java file that detects a pinch and redraws the PDF based on the zoom level of the pinch. If I run my code as a native app, this works fine, but when I run it within Ti as a module, the pinch to zoom only works for a slight adjustment, and then stops, requiring many short pinch gestures to zoom in.

I started logging in several areas to figure out what was going on. I added logging to my android java code, as well as adding logging in an eventlistener for "pinch" in Ti.

Oddly enough, I am seeing some conflicting behavior. While the zoom is working, I see the log messages firing from within my android java code, and after a little more movement, the zoom stops working, and then Ti starts firing the "pinch" log events. It seems something in Ti is "hijacking" the pinch event from my native android code. As soon as I lift a finger, the Ti event listener stops detecting pinches and my android java code starts to catch touch events again.
