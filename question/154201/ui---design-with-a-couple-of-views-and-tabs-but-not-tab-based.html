UI - design with a couple of views and tabs, but not tab-based?

I'm considering Titanium for building an app that needs to run on iOS and Android, in both phone and tablet layouts. The app has already been designed visually and needs to have a consistent set of tabs on top rather than following the iOS conventions of tabs on the bottom. Essentially it looks like a fairly simple traditional web site - but it needs to run offline, so we can't just build a mobile web site. I spent a LOT of time trying to modify a tab-based Titanium example, but since by default tabs are displayed on bottom for iOS it seems like it might be a lot simpler to just build the app with HTML, CSS and Javascript. Is Titanium still the best option for such a project? If so, can anyone point me to some sample code to help me get started?

  • Hi

    Whilst you do not have to follow the UI conventions for iOS and Android when designing apps (otherwise we would have no innovation) it does make your life much easier - as I think you have been finding out.

    Something to consider before you move forwards - if your design is better than the existing ways (not necessarily technically - visually works) then move ahead with your design. If not you may end up annoying users by trying to get them to interact with a new way of doing things that runs differently than they would expect.

    For example a good cross-platform app does NOT look the same, but has the essence of the same and the differences suitable to the platform. This way you honours the users and the app at the same time.

    But as I say - you may have very good reasons for not doing this and that is valid.

    No the problem - straying from the path of normal methods makes life a little harder to achieve, and even harder if you are aiming for both iOS and Android.

    Given your explanation I would suggest that you create imageViews that represent your tabs and then place a scrollableView underneath. Each page of the the scrollableView can act as the tab container and you add your content in that. Click events on the imageView tabs can tell the scrollable view to swap pages (with or without animation - there are methods for both). This will give you app a good base start.

    You will have to solve how you add windows though as these will cover the main tabbed window - however if you only use lightweight windows on Android (iOS is good about this no changes needed) you can set the left, right, top and bottom of a window to not cover the tab strip.

    I would not recommend using HTML to solve the problem - it will give you too many compromises down the road when you want extras in the app that HTML5 will not give you access to.

    You can do this is Titanium and many people do.

    Good luck and have fun.

As you can programattically set the page to show - you have lots of control.
