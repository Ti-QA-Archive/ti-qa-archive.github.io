Issues deleting UI-Proxies

Hello,

I've got a multi platform application built with Titanium, and I am facing some performance issues in the
iOS-version. To be more precise, the app gets really slow after a certain time of usage and even crashes sometimes at the end. On older iPhones, these problems are more considerably. They do not show neither on iOS-Simulator nor on Android. The iOS Developer Instruments showed, that nearly all of the UI-Proxies (TiUIViewProxy, TiUILabelProxy,…) are not deleted as they should. This was previously asked at: Previous Question.
I've implemented this work-around, but the results are the same as before, so I'm running out of ideas. The suspected part is a NavigationWindow. I can open and close the same Window over and over again, the UI-Proxies stack up without any byte marked as Transitory. The Window is closed by NavigationWindow.closeWindow(window) and I'm invoking the clearMemoryPool-method I implemented of the previous question.

Can you help me with this?

    A global eventhandler prevented these proxies from closing. So if anyone is facing the same problem, look out for global eventhandlers.
