new date object fails when beyond cell reception

So this is a weird bug. On several occasions my app has failed to generate a valid date object with new Date() when outside of cell reception. I can't seem to replicate this when I put my iPhone in airplane mode or turn off 3G and cellular data. It only seems to happen when I power up my phone and there really isn't any cell reception, then start my Ti-based app and do something that tries fetch the current datetime, which fails.

Strangely, the error seems to stop after the phone gets cell reception, even if it loses reception again.

Has anyone else experienced a bug like this? Can anything be done? I'm using the Ti SDK 1.6.2.

  • On the surface, this sounds like whatever device you are using won't report its clock time after a power up until it has verified it with a cell tower to get the correct current local time and timezone information. Once it has done that, it continues to report the time even if it can no longer see a tower.

    This sounds to me like a device / OS specific problem instead of a titanium problem per se. But all of this is just a SWAG based on your description.

    What device / OS version are you using?
