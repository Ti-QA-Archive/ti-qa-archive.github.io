Can't make button with background image in alloy project

Hey, I'm new to titanium and I'm trying to make a button with a background. My project is an alloy, here's my code, as simple as it can get:

"#button1":
{
    backgroundImage:"images/paw.png"
}

<Alloy>
    <Window>
        <ImageView id="imageView" onClick="clickImage"/>
        <Button class="button1" top="25%" left="5%" id="boton1" onClick="button1Click">MyButtonName</Button>
    </Window>
</Alloy>

The thing is that the image doesn't show up, If I put text in the button, the text shows up and image doesn't.
I've tested my code in my Android phone, android tablet and android emulator.

Hope I can get help because I've tried adding backgroundSelectedImage with no effect. Background or any kind of image doesn't show up.

    Android and iOS reference paths differently. This is primarily due to the differences in the javascript engines they use. iOS reference files from the root of the application. Android references them from the calling file's location.

    Those that have been around Titanium for a while, tend to throw a preceding forward slash in before their file image references. So an image in app/assets/images would be references as '/images/' instead of 'images/'. So in your case, give &#x2F;images&#x2F;paw.png a try.
