  • you can try the approach that is used in the KitchenSink-iPad application; what they are doing there is creating the views before hand,

    First they are including them

    Ti.include('popover.js','video.js','option_dialog.js');

    Then calling the init method on the view objects that were created

    Popover.init();
Video.init();
OptionDialog.init();

    the view objects are assigned to the table data rows

    MainTests.tableVewData = [
    {title:'Popovers', v:Popover.view},
    {title:'Option Dialogs', v:OptionDialog.view},
    {title:'Embedded Video', v:Video.view},
];

    and then on the click event for the detail window, they are assigning the appropriate view to the window through the row data.

    
MainTests.tableView.addEventListener('click', function(e)
{
    MainTests.detailWindow.animate({
        view:e.rowData.v,
        transition:Ti.UI.iPhone.AnimationStyle.CURL_UP
   });
});

    Or you can continue with your approach but you would need to create the new window in the event listener

    addView.addEventListener('click', function(e) {
        var _window = Titanium.UI.createWindow({
            url : 'addlocation.js'
        });
        SplitViewNav.detailWindow = _window;
    };
});
