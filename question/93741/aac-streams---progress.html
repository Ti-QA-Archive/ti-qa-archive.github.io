AAC Streams - progress?

Since asking this question several months ago others have also stated they wish to be able to play Shoutcast style aac format streams. However the current TI AudioPlayer API is unable to oblige.

We then learned that John McKnight had solved the problem (to do with file extensions) and included it in his own build of the 1.5.0 SDK. At the time it I didn't try to download it for various reasons but it did seem odd that this wasn't being rolled into the general 1.5.0 or 1.5.1 release and it still hasn't from what I can tell.

Maybe John can build a player 'module' using his fix? I'm sure some would be willing to pay something towards his work. The inability to play a radio station's main stream is really crippling my app building right now as customers - certainly in the UK - expect it to able to play the aac format.

— asked December 24th 2010 by Patrick Mounteney

  • I have put an app together with Titanium that streams an AAC shoutcast using the following:

    for iOS

    Use a videoPlayer and not audioPlayer:

    var streamer = Ti.Media.createVideoPlayer({
                url : 'http://198.144.148.12:8000/'
            });
            streamer.play();

    for Android

    audioPlayer or videoPlayer do not support AAC at all so I developed the following module:

    https://marketplace.appcelerator.com/apps/5728
