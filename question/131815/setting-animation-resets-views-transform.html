Setting animation resets view's transform

I have a view that has a transformation (rotation), when the user performs an action, the view's opacity starts pulsating. When the user is done, the animation needs to stop. I got the animation to stop but, it's resetting the view's transformation thus losing the transformation.

SDK:

[INFO] Titanium SDK version: 1.8.1 (01/27/12 17:31 a24502a)
[INFO] iPhone Device family: ipad
[INFO] iPhone SDK version: 5.0
[INFO] iPhone simulated device: ipad
[INFO] Launching application in Simulator
[INFO] Launched application in Simulator (4.40 seconds)
[INFO] Found 5.0 patch installed
[INFO] Application started

When the timeout executes, the view will stop pulsating and lose it's transformation.

var win = Ti.UI.createWindow({ backgroundColor: '#444' });
var view = Ti.UI.createView({ borderColor: '#fff', width: 200, height: 200, backgroundColor: '#aaa' });
var label = Ti.UI.createLabel({ text: "90 Degree Text", textAlign: 'center' });
var transform = Ti.UI.create2DMatrix({ rotate: 90 });
var animation = Ti.UI.createAnimation({opacity: 0.30, duration: 1000, autoreverse: true, delay: 300, repeat: 1000});

view.setTransform(transform);
view.animate(animation);

win.add(view);
view.add(label);

win.open();

setTimeout(function(){

    Ti.API.log("stopping animation");

    view.animate({ opacity: 1, duration: 300 });

}, 2500);

Same thing happens with 1.9.0 (nightly build), 1.8.0.1, and 1.7.5
