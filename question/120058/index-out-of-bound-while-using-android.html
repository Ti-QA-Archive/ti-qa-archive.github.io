  • Without some code it is difficult to say.
    Well I had the same exception, but the code did not point to something out of bounds. In my case the solution was the className of the TableViewRow.

    I use classNames to speed up the recycling, the issue was that all the rows had the same className but actually had different views in it, I mean the views were not aquivalent (not just different content). The problem was solved by giving the right classNames to those different types of rows I had.

    Hope this helps.
