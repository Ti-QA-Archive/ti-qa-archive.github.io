    • You do not need to do anything in Titanium, it has had support for iPhone 5 before 3.0.0 GA, all you have to make sure you have if a single extra file and that trigger support (strange but true).

      You already provide

      • default.png = 320x480 start up splash screen for non-retina
      • default@2x.png = 640x960 start up splash screen for retina

      The new one is

      • Default-568h@2x.png = 640x1136 start up splash screen for tall retina (iPhone 5)

      As soon as you put this in the same folder as the other two you will gain iPhone 5 support.

      If your app is a game or avoids standard UI elements then support will be based on determining device height and adjusting accordingly.

      If your app is based on standard UI with tableViews and scrollViews then for the most part you do not have to do anything. Just make sure that things do not fall of the screen if you are overly targeting iPhone 5 as when you swap back the extra height will not be there.

      A quick tip when designing interfaces and you do not want to know about the height but you need two items on a screen one is fixed height and the other is flexible.

      var viewLedge = Ti.UI.createView({
    height: 50,
    width: Ti.UI.FILL
});
win.add(viewLedge);
var tbl = Ti.UI.createTableView({
    heigth: Ti.UI.FILL,
    top: 50, // this will push the height down 50 from whatever the fill height would have been
    width: Ti.UI.FILL
});
win.add(tbl);

      Using those sort of tricks means you have one set of code that will show the table bigger or smaller on the device without having to determine or fix anything in place.
