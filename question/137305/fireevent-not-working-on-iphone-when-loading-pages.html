FireEvent Not Working On iPhone When Loading Pages

I have a small web document that I am 'wrapping' in Titanium and releasing on iPhone and Android. It needs to play some sounds so I am using a bit of code to fire an event and trigger the audio playback (I want it to work on Android 2.2 so can't use HTML5 audio):

<p>Listen to this sound:</p>
<p><a href="#" onclick="Titanium.App.fireEvent('playSound',{url:'test.mp3'});">Click</a></p>
<p><a href="./page2.html">Page 2</a></p>

This works on both platforms for the "index.html" page, and on Android for all following pages loaded. However, on the iPhone the fireEvent is only triggered on the index.html. On the pages that index.html links to (eg. page2.html) it doesn't. The pages are all in the same directory on the device.

Any idea what's going on? It's rather infuriating!

Sean
