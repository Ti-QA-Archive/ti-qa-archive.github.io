Serious (and weird) issue with SDK 3.0.0 GA

Since ever I upgraded to the SDK 3.0.0 GA my imageViews cannot display remote images. However I have noticed that this is only true only from a specific web server that we currently have. An example image from this server can be found here:

http:&#x2F;&#x2F;www.lillikas.com&#x2F;assets&#x2F;image&#x2F;image&#x2F;simos-aggelides.jpg

When I compile to 2.1.4 GA it works perfect. When I switch to the 3.0.0 GA it does not show at all. Even when I place a border to the imageView , nothing is displayed.

Here's my code:

// this sets the background color of the master UIView (when there are no windows/tab groups on it)
Titanium.UI.setBackgroundColor('#000');

// create base UI tab and root window
//
var win = Titanium.UI.createWindow({  
    title:'Tab 1',
    backgroundColor:'#fff',
    layout:'vertical'
});

var imageView = Ti.UI.createImageView({
    image:'http://www.lillikas.com/assets/image/image/simos-aggelides.jpg',
    borderWidth:1,
    borderColor:'red',
    height:100,
    width:50
})

var imageView2 = Ti.UI.createImageView({
    image:'http://i2.cdn.turner.com/cnn/dam/assets/121218084850-debunking-doomsday-mayan-calendar-00005515-video-tease.jpg',
    borderWidth:1,
    borderColor:'red'

})

win.add(imageView);
win.add(imageView2);

win.open();

imageView2 works great, but imageView, no ..

any ideas on what might have possibly goes wrong?

Thank you,
George

  • After excessive testing, debugging but also sniffing the traffic, I have ended up that the problem is the specific caching value of the cache-control header. To be more precise the following header is causing the issue.

    Cache-Control: no-cache

    Caching is not an issue, but the value 'no-cache' is causing the problem. (perhaps it's the dash?!) It's a bug that needs to be resolved.

    George
