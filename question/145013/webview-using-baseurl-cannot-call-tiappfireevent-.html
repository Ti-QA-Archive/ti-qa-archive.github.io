Webview using baseURL cannot call Ti.App.fireEvent ??

Hi.

After much confusion about why I couldn't call Ti.App.fireEvent from my webView I have found the reason is the webView has baseURL defined which is breaking the fireEvents.

But how do I get round this? Please help!

  • My guess is that setting a base href makes the document effectively a remote document, and thus, you lose access to the Titanium API, and that there wont' be any workaround. You can use evalJS() to interact with the content in a limited way, but you're not going to be able to call Ti.App.fireEvent().

  • Why chime in if you don't have an answer? Sorry, but your answer added nothing that my question didn't already state.

    I have found this can be solved thus:

    • Do NOT use the baseURL attribute on the webView.setHtml function
    • Instead, add a <base href=""> tag to your HTML code

    This will keep your links and image references relative to the 'base' but will not break JS calls to `Ti.App.fireEvent'.
