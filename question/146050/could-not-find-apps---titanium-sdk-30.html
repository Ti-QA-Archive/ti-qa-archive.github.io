  • Accepted Answer

    In Titanium 3.0, if any of your files has a JavaScript typo, the build process fails despite saying it was successful. That's a bug. We fixed it and the fix will make it into the upcoming 3.0.1 release.

    In the meantime, you can download the latest CI build of 3.0.1 and use that to find your typo. Simply run the following from the command line:

    titanium sdk update --branch 3_0_X

    Then in Titanium Studio, select 3.0.1.v201212[whatever] and rebuild. You'll see exactly where the problem is happening.

    • @Stephen, are you sure that you don't have a JavaScript errors in any of your JavaScript files? Even files that you aren't require()'ing or Ti.include()'ing? Each JavaScript file MUST not contain any syntax errors.

      The only other known thing that would cause that error is building your app for the iOS Simulator, then going into the yourproject/build/iphone directory and opening the Xcode project and building directly for device. That doesn't work in 3.0.0.GA, but has been fixed for the upcoming release.

      Would you mind emailing me the build output? From your project directory, run the following:

      titanium clean

      titanium config cli.logLevel trace

      titanium build -p ios -T device

      It will prompt for your provisioning profile uuid, so just hit <enter> to list them all and then enter the correct one. It may also ask for the name of your iOS development certification. Again, hit <enter> to see all possible names, then enter your cert name. Copy the complete output of the build command and send it to cbarber@appcelerator.com. I'll take a peek and see if I can track down the issue. Thanks!
