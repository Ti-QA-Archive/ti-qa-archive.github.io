Sync data from Web to mobile and vice-versa.

Hi,

I am looking for some existing API call for syncing my web-app data to mobile and also the other way round. Does something exists in titanium. I don't want to have my own login for syncing the data, if it already exists. CRUD operations can be done from both ends i.e either in web-app or in iphone. So if a record is created on device, it would be saved on local database and later on synced up with server. Same thing should happen when any operation is carried on web-app. Is httpclient the only available option for it? Any sample application which does same thing?

  • I have done this - but ours is a large project so would be very difficult to provide any code that makes sense.

    But basically we have a database on the server and one on the device. When you add data to the device it gets an "id" of "NEW". Then when you sync, you first send the app data and the web server code checks for any id's of "NEW" and INSERTS the new data. It then returns the data with a "last insert" id along with any pre-existing items. The returned data overwrites the local app data - so we now have a dataset with std numerical id's.

    Next time you sync, any updates made locally on the app will have a real id and so get UPDATED on the server and any NEW ids get INSERTED, then everything gets returned to overwrite the local app data again.

    Make sense?
