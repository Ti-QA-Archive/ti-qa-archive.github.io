Facebook - login dialog without using Titanium.Facebook.LoginButton

I'm sorry if somebody asked this before, but I just couldn't find it anywhere.

What I need is quite simple, my app has a sharing menu, so the users can select a social network (twitter, facebook, etc) to post that he is using the app. For a better user experience, I wanted to go by the simplest possible way, the app should work like this:

1- The user chooses a social network from the menu

2- The system checks if the user is already logged, then a dialog appears, asking for his login or the permissions to post (if he is already logged in).

3- It's done, tells the user that everything worked fine

But I just can't do that, the only way I found out to do the login is using the 'Titanium.Facebook.createLoginButton', but if I use it, when or if the user is already logged, the button turns itself into a 'logout button' - it would be terrible to force him to logout and then login again. I know I could hide the button and show another in its place, but its seems like a workaround, it would mess the with whole style of the menu, and I dont want that text neither 'Connect with Facebook', I just want it to be a normal row with a simple text: 'Facebook', 'Twitter', and so on.

In fact, I dont need the button at all, I just need to know what method does the button call, or something like 'Ti.Facebook.createLoginDialog'.

Does anyone knows about it? Is it possible to login without that button?

I'm being forced to make another window with a lonely button…

    You can make a call with the facebook object to see if the user is logged or not.
    Titanium.Facebook.isLoggedIn
