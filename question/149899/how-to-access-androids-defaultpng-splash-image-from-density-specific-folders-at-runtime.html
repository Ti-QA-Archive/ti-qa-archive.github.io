How to access Android's default.png Splash Image from density specific folders at runtime

Titanium Mobile 3.0 SDK
Android 4.x
Alloy 1.0

Hi,

Is it possible to access the density specific version of the splash screen image at runtime?

I have several splash images at different densities e.g:

/app/assets/android/default.png
/app/assets/android/images/res-sw600dp-mdpi/default.png
/app/assets/android/images/res-sw600dp-tvdpi/default.png
etc

The app correct displays the appropriate splash image when you first load the app. What I want to do is access the same density appropriate version at runtime later on (to overlay my own loading/init sequence).

If i try to load the splash image within an ImageView e.g. imageView.image = &#39;&#x2F;default.png&#39;; it displays the generic mdpi one from &#x2F;app&#x2F;assets&#x2F;android&#x2F;default.png. How can I access the ones from my res-sw600-dpi folders when appropriate?

I notice that default.png gets renamed to background.png in the compiled Android app inside the &#x2F;build&#x2F; folder. I can get the Resource ID using Ti.App.Android.R.drawable.background however I don't know how to then request/load the resource using the Resource ID? Is this only for modules combined with the TiRHelper class?

I also notice that within the compiled build folder it leaves a copy of &#x2F;app&#x2F;assets&#x2F;android&#x2F;default.png inside the &#x2F;bin&#x2F;assets&#x2F;Resources&#x2F;default.png, as well as moving and renaming a copy into &#x2F;res&#x2F;drawable&#x2F;background.png which I guess makes sense why that particular version loads when I request "/default.png" within an ImageView.

Should I be able to access the density specific folder or will it not work considering Titanium doesn't append a random token to the end of the filename like it does for all other images in the /build/res/drawable folders?

Cheers,

Tim
