Android Emulator will not open

Problem: the Android emulator will not open when launching a Ti Project, or from the Android Tools AVD manager (this is true of both AVD's created by Titanium or manually created by me). Oddly, AVD's seem to function normally if started via command line (eg. $ ./emulator-arm -avd test -partition-size 128 -no-snapshot-load -verbose -cpu-delay 0 -no-boot-anim -debug all -show-kernel &).

There is an emulator-arm process started in the problem scenarios, but it never "displays", and starts taking all CPU, and eventually has to be manually killed.

System: Ubuntu 10.04 LTS 32 bit, using the Ti mobilesdk. I have verified the recommended Java version is being used. This system has previously had a fully working Ti Studio / Android Tools environment without issue. (I have another 10.04 LTS system that works just fine.)

Titanium and Android SDK and Android tools are fully updated to latest stable versions.

Things I've tried:

Multiple 'project cleans'

Killing and restarting all components (verifying they are indeed killed.), eg "adb kill-server; adb start-server", same with android tools and Ti Studio.

Manually deleting the .android/avd and .android/cache folders

Fastdev both true and false.

Tried several versions of Android SDK.

What I get in the console window:

[INFO] Titanium SDK version: 3.0.0 (12/12/12 18:49 a5894b3)
[DEBUG] Waiting for device to be ready ...
[TRACE] adb devices returned 0 devices/emulators
[TRACE] adb devices returned 0 devices/emulators
[TRACE] adb devices returned 0 devices/emulators
[TRACE] adb devices returned 0 devices/emulators
[TRACE] adb devices returned 0 devices/emulators
[TRACE] adb devices returned 0 devices/emulators
[ERROR] Application Installer abnormal process termination. Process exit value was 1
[ERROR] Timed out waiting for emulator to be ready, you may need to close the emulator and try again

There are several AVD's available according to Android Tools AVD manager. All have green check marks to indicate they aren't broken.

While an emulator process is started, nothing really "opens", so "closing" does not help in this case.

I'd like to get better logging but haven't found a way to do that. I see a few other people have had something similar, but I don't see any resolutions.

I know this isn't a Titanium issue per se, but am hoping someone has been here before or has ideas for finding a solution. Thanks.

    Hi

    This is a common problem but not the easiest to solve, here are some things to try;

    • Make sure you are not testing against Android SDK 4.2 - this is not currently supported.
    • Confirm your Java is 1.6 and NOTHING higher
    • Delete ALL AVDs, Titanium will automatically create a new one
    • In "Run Configurations", change the Android Emulator SDK version to a different version
    • In "Run Configurations", change the Android Emulator SDK version to include the suffix [armeable] or [armeable-v7a], for example; Android 2.3.3 [armeable] or Android 4.1.2 [armeable-v7a]
    • If you hold your files on a non-local file-system (network drive for example), create a locally stored test app, run that into the emulator, leave that running and then run your own app - this actually works for me - no joke
