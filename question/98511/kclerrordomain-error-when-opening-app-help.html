  • Danno , I get the error on my simulator when it's trying to get the current location ( and doesn't find it ).

    Do you try to get the gps on start ? If yes you have to check if they have the location enabled first, and use try/catch in the geo part of the code to avoid the alerts.

    The second part with the gps not accurate is because the getCurrentPosition it's not so accurate when the app starts and the gps was not used - it's some sort of "caching" of the last detected position. To avoid this you should ask for the geo position when the app starts, and attach a function( even an empty one ) to the 'location' event to force the device refresh it.
