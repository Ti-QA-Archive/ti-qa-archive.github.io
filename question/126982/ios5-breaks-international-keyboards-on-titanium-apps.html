IOS5 breaks International keyboards on Titanium apps

ALL my present and previous Titanium apps no longer recognise International keyboards after devices update to IOS5. Is anybody else having this problem? It is easy enough to test, just add a keyboard e.g. Thai and then see if your app recognises it.

    • Hi, it looks like the first 4 versions of iOS - or the first million versions of Titanium - ignored the keyboardType parameter when users changed to an International keyboard. So anyone like me who simply copy-pasted the US-Centric Kitchen Sink or API Docs examples that specify an ASCII keyboard now has to go through the code of all their previous apps and remove any text field or text area references to a specific keyboardType and recompile with 1.7.3+. That seems to allow the International keyboards to be recognised again. I hope this is right anyway, one corrected and recompiled app has been approved this morning.
