Problems using a folder structure for icons in different resolution which works on Android without duplication

I am having problems getting icons to work using the following folder structure

Under the folder: Resources->android->images

Each icon folder is placed under one of the following 3 folders

  • res-hdpi
  • res-mdpi
  • res-ldpi

All splash and backgrounds images are placed in one of the following 10 folders

  • res-long-land-hdpi
  • res-long-land-ldpi
  • res-long-port-hdpi
  • res-long-port-ldpi
  • res-notlong-land-hdpi
  • res-notlong-land-ldpi
  • res-notlong-land-mdpi
  • res-notlong-port-hdpi
  • res-notlong-port-ldpi
  • res-notlong-port-mdpi

My problem is that if I place a copy of my icons folder under each of the 10 specific folders it works fine but I have a redundancy problem.

if I place them under one of the 3 less specific folders it seems that images are sometimes only taken from the res-mdpi folder. I noticed this when trying to update an icon file, where on a res-hdpi device icons were taken from res-mdpi folder

Any suggestions would be appreciated

    Resource > android > images > low folder
