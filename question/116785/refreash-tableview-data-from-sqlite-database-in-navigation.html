Refreash tableview data from SQLite database in navigation

I'm using navagation groups in an iPhone application. I have the table data populated from a sqlite database. Thats all fine.

To edit an item, a user can select that item, it opens to a child navagation window and I do my editing logic there. When a user clicks the save button on the child window, I update the database and close the window back to the parent window.

How do I get the data reloaded into the table? should there be an eventlistener running that looking for me comming back to the window?

Not sure how to go about this.

  • Assuming winB opens from winA, you can add a 'close' event listener to the winB which you opened to edit the item.

    When the event is fire, i.e. when you close the window, it basically updates the table which is present in winA.

    Beware of bugs like these though:
    http://developer.appcelerator.com/question/88031/bug-tinetwork-dead-thread-exception
    There are rather ridiculous workarounds for the problem anyway, so no worries.

    Gd luck!
