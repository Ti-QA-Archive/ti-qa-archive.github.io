How to use Grand Central Dispatch in Appcelerator Mobile?

Is there any way to use Grand Central Dispatch or NSOperations in Appcelerator Mobile?

There are some heavy calculations we need to do when receiving a new GPS point and we'd like to dispatch that to a separate thread.

At the moment we notice delays of up to 5 seconds between the moment the iPhone receives the point and we receive the point through the eventlistener. We expect that this happens because the listener is waiting for our code to finish. To test this and to optimize our code, we'd like to use multi threading and making optimal use of the multi-core processor in the 4S.

  • I think you should write a native module for that: https://wiki.appcelerator.org/display/guides/iOS+Module+Development+Guide

    This way your background task will be fast.

    Other solution is to separate your background task into peaces and process them with setTimeout. This way your UI remains responsive (although it will be slow).
