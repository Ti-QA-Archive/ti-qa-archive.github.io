BlackBerry ready yet?

Hi everyone. I realise the bb target is still under closed beta. However I have a potential corporate directory project for my client that initially needs to be bb-only (5.0 and late 4.x) before going public on iOS and Android next year. To secure the project I'd need to be able to guarantee internal deployment of the bb binary in the next 4-6 weeks. It would be a simple corporate directory but with the ability to check for and download personnel updates when the information changes (server side developed separately). There would be a search function in the app.

If the bb components aren't quite there yet, then fine, I will have to use a native sdk approach, but I would of course love to use Titanium to reduce future dev effort when the app goes public next year. If the bb target is stable enough, I would be looking to take a Pro subscription and enlist one or more developers with experience of the platform. Many thanks.
