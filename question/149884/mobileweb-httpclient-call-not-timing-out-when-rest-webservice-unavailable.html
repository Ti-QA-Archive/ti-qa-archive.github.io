Mobileweb: HTTPClient call not timing out when REST webservice unavailable

Titanium SDK: 3.0.2.GA
Studio Platform: OSX 10.7.5
Device: Mobileweb (using Chrome for web browser)

I am trying to call a REST web service via a mobileweb app (works fine in iOS). Everything works fine when the REST web service is available. I am trying to figure out how to code it for mobileweb platform to have the timeout create an on error event.

I understand by reading the documentation that the timeout specified below will not work in mobileweb. How should I code this so that the timeout will fire?

Here is my code:

var xhr = Ti.Network.createHTTPClient({
        enableKeepAlive : false,
        onload : function(e) {
            Ti.API.debug("HTTPClient returned onload");
        },
        onerror : function(e) {
            Ti.API.alert("HTTPClient Failed!");
        },
        timeout : 6000
    });
