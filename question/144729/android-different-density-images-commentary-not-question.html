Android different density images (commentary not question)

So, I just started playing around with Titanium for iPhone and Android, using DropBox to keep the files the same between the two. Going back and forth and testing Android on my laptop (the AVDs run better on Windows) and iOS on a Virtural Box.

Titanium: 2.1.1
Android: 2.2 (mainly because I haven't bothered anything serious just the defaults)

In order to make the applications, Google provides pretty sweet utility that creates different resolution images for you to use, you can find that here:
http://android-ui-utils.googlecode.com/hg/asset-studio/dist/index.html

Note, it only works in Chrome.

Anyhow, so you get your images created and put them into the folder stucture, by default you'll have:

resources/images/
resources/iphone/
resources/android/images/res-ldpi
resources/android/images/res-mdpi
resources/android/images/res-hdpi
resources/android/images/res-xhdpi

So I had 4 images for 4 tabs in the /images/ folder as well as the /android/images/res* folders. Obviously the images were different resolutions for different densities. And for the life of me I couldn't figure out what was going on. I read that Titanium copied the images required to the base image folder, which is awesome!

EXCEPT!

It doesn't when this folder is filled already. So you can edit the manifest, and change folder names, etc until you're blue in the face. But as long as resources/images has images in it, those images are going to be used.

Not until I removed the resources/images (and copied those to) resources/iphone/images (note, I haven't actually tried to compile the iphone version yet) did it work. As soon as this images folder was empty, Titanium now copies the correct version into the folder when compiling. I've now confirmed this on L, M and H dpi screens.

So if you're having the same problem I was.. give this a shot.
