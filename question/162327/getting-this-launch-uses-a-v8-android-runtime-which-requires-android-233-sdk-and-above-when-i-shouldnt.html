Getting "This launch uses a V8 Android Runtime, which requires Android 2.3.3 SDK and above." when I shouldn't!

I am attempting to make my Android app work for Android 2.2 and above. Based on Titanium's documentation, I found the following (documentation link):

Titanium SDK Version
3.1.0 ;
Min Android/SDK Version
2.2 (API 8)

Based on that, I installed and am using Titanium SDK version 3.1.0 GA. And I'm using Android API 8. In the tiapp.xml, they look like the following:

<uses-sdk android:minSdkVersion="8" android:targetSdkVersion="8"/>

<sdk-version>3.1.0.GA</sdk-version>

My virtual device is configured as:
Target : 2.2 - API Level 8

So everything seems configured correctly, but when I try and run the emulator, I get:

"This launch uses a V8 Android Runtime, which requires Android 2.3.3 SDK and above."

This just doesn't seem to match up to the documentation of Titanium. Anyone have any clues on what's happening and how to get Titanium SDK 3.1.0 work with 2.2 (API 8) as it is supposed to?

Thanks,
Chris

    • There are 3 different SdkVersions available to you.

      <uses-sdk android:minSdkVersion="integer"
          android:targetSdkVersion="integer"
          android:maxSdkVersion="integer" />

      minSdkVersion should be set to the lowest you wish to support, this must be 10 or higher now and you must level 10 installed.

      targetSdkVersion is the version you will build against, you must have this level installed.

      maxSdkVersion is the highest level you wish to support.

      http://developer.android.com/guide/topics/manifest/uses-sdk-element.html
