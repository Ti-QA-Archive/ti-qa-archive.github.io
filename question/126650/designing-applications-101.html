Designing applications 101

I'm trying to write a cross-platform app for Android and iPhone. I wonder what the best way is to set up my windows/views is. Right now I'm using modal windows but I'm getting the feeling this isnt the right way to go about things.

Right now I have a couple of windows displaying a tableview (with different data depending on where you are) and then normal views for the last windows. The user pretty much follows a set path: click type, click product, enter information, confirm, return to beginning.

Should I be using something else than modal windows? I'm very grateful for constructive input and links to relevant documentation/tutorials.

    Hi,

    I started with the same questions and if you look at the kitchensink it'll actually put you on the wrong track.

    If you need to understand a point/reference implementation of an API then the kitchensink is great (as it usually contains undocumented stuff).

    If you use it as a framework it's very poor (each new .js is a new url and then runs in its own window/Javascript context).

    Tweetanium is a very good sample application but unfortunately it is a little bit "middle ground". Yes, it runs on both Android and iPhone but it doesn't look native on either. So if you are doing a very custom UI then great, go for it, if not then you'll have some grief.

    You should check out

    http://codecanyon.net/searches?term=titanium&type=files

    For some reference sample apps, spend 100$ or so on a few to get some code examples. Note, I've not used any of these, after a year on Titanium I've figured out the pain/avoiding pain points..

    Just remember that Titanium shouldn't be thought of as "write once, run anywhere (mobile)" but "write once, adapt a few times, run anywhere".

    Hopefully they come up with a design tool and some form of coded layout manager where you can pass down something like XIB/NIB and the ObjC/Java code will instantiate all the objects.. today it can be a little slow (especiallllllllly on Android). Their NDK V8 JS engine should help there but that's still sometime off.

    In short, but some sample apps, the ones from Titanium aren't the best for building world class apps unfortunately. They should release/buy & release some better samples.

    Code for iPhone first, its far quicker dev cycle, then adapt for Android.

    Cheers

    Chris

    • There are best practices for any given situation. You need to consider the complexity of your applications, the number of windows / views in them, and the logical connection between them.

      I think that for some cases it's good to open windows in different context because and for some not. Keep in mind that you don't want to load all your includes when the app starts which makes it slow, and when not working properly pollute the global scope.

      Have a look at this thread and it's discussion: http://developer.appcelerator.com/question/126492/mass-includes—fixed
