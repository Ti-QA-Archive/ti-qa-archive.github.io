fireEvent: event value undefined

Hi,

I have an XML file in my mobile App which I read within a screen. From there I want to fire an event sending the content of the file as parameter. Something like this:

var xmlContent = service.getXml(filePath);
Titanium.App.fireEvent('addXmlEvent',{"data":xmlContent, "testdata":"test"});
Ti.API.info("xml content "+xmlContent);

until now everything is correct. The output is something like this:

[INFO] :  xml content <?xml version="1.0" encoding="UTF-8"?>
[INFO] :  <kml xmlns="http://www.opengis.net/kml/2.2" xmlns:gx="http://www.google.com/kml/ext/2.2" xmlns:kml="http://www.opengis.net/kml/2.2" xmlns:atom="http://www.w3.org/2005/Atom">
[INFO] :   <Document>
[INFO] :     <name><![CDATA[Some Data]]></name>
[INFO] :     <open>1</open>
[INFO] :     <Snippet maxLines="0"><![CDATA[]]></Snippet>
...........

The corresponding event listener is within a WebView:

var addXmlData = function(params){
  Ti.API.info("Adding xml data "+params.data+ " test "+params.testdata);
};
Ti.App.addEventListener('addXmlEvent', addXmlData);

And this is the output from within the WebView addXmlData function:

[INFO] :   Adding xml data undefined test test

So somehow my Xml content within the event disappeared, the other string remained.
Any idea what could cause this? The xml file has around 500 lines, but even with one half the
size the behavior is the same. The documentation does not say anything about maximum length etc.

Is there another way to pass the content to the webview, as I cannot read the file from within the webview?

  • Ti.App.Property unfortunately does not work in a WebView, would be cool.

    But I found the solution. It was my fault, somehow I managed to convert the String into a Blob. After testing a few things a xmlContent instanceof String gave the right hint.
    The string value is perfectly passed with the event.
