      further I wrote a method for getting the right images:

      getImageFroDevice = function (imageName) {
    var path = 'images/';

    if(/android/i.test(Titanium.Platform.osname)) {
        path += 'android/default/' + imageName;
        //TODO: here we should also check for screen size
        // at the moment the default dir is in use.

    } else if(/iphone/i.test(Titanium.Platform.osname)) {
        path += 'iphone/' + imageName;
    }

    Ti.API.log('retuning image: ' + path);

    return path;
};

  • The contents of each folder are "effectively" copied over the root folder for each platform. So during the build you're putting images in different locations. If you don't need the images to support multiple densities but you want different images for each platform then simply put them in the same "place"

    Resources&#x2F;android&#x2F;bla.png and Resources&#x2F;iphone&#x2F;bla.png

    or

    Resources&#x2F;android&#x2F;images&#x2F;bla.png and Resources&#x2F;iphone&#x2F;images&#x2F;bla.png

  • As mentioned above, items are copied to the root folder for the platform, so for an image that you have stored at Resources/iphone/bla.png, you simply reference it as "bla.png" for iphone use. To do the exact same for Android, place the Android version of "bla.png" in Resources/android/ and regardless of platform, the image can be referenced as simply "bla.png"
