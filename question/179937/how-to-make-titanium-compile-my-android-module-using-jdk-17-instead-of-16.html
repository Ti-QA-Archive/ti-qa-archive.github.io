How to make Titanium compile my Android Module using JDK 1.7 instead of 1.6?

I have installed JDK 1.7 on my Mac and configured in Preference: Java : Compiler and Preference: Java : installed JREs respectively.

However, when packaging my android module project, it is still using JDK 1.6 for compilation.

I know because it keep prompting me the following errors:

cannot find symbol
symbol: constuctor ZipInputStream(java.io.InputStream in, java.nio.charset.Charset charset)

I need to use ZipInputStream(InputStream in, Charset charset) which is available in JDK1.7 but not in JDK1.6.

Anyone know how can I resolve this?
Thanks a lot in advance.

  • Hi

    You are "meant" to be able to use newer versions of the Java library now - however it appears there is a bug that is stopping that from happening.since it was meant to be an option.

    There is a related Jira ticket https://jira.appcelerator.org/browse/TISTUD-6155 that suggests it should have been fixed since 3.3. However from experience 3.4.2 failed to work with anything except 1.6.

    You do need to make sure any previous Java versions have been removed so that ONLY 1.7 is left.

    It is worth checking to see if the latest Ti SDK 3.5.0 RC works any better. This update (and those that follow) will be required to submit any new apps into the app store after Feb 1st and a couple of months later for any app updates.

    If you are still having problems, create a new Jira ticket detailing your entire environment and experience. Then post the link back here.

  • Urban Airship now requires JDK 1.7

    Android Migration

    I'm trying to figure out what can be done to compile a module with JDK 1.7.

    Below is a list of things I've been trying:

    1. Titanium Studio -> Preferences:
      • Java->Installed JREs
      • Make sure the JDK 7 is the default
      • Java->Compiler
      • Make sure 1.7 is selected (Also make sure your not overriding this in sub projects)
    2. Right Click Android Module -> Properties:
      • Java Build Path -> Libraries
      • Make sure all of these reference JDK 7 options

    Warning: These Next Steps are Messing with the Compiler's Build Scripts

    1. Titanium/mobilesdk/osx/3.5.1.GA/module/android/build.xml
      • Around Line 157: Change 1.6->1.7
      • Around Line 224: Change 1.6->1.7
    2. Titanium/mobilesdk/osx/3.5.1.GA/android/builder.py
      • Around Line 1618: Change 1.6->1.7
    3. Titanium/mobilesdk/osx/3.5.1.GA/android/cli/commands/_build.js
      • Around Line 928: Change 1.6->1.7

    The reason I started messing with the Compiler build scripts is because even with it pointing to the JDK 1.7 I would get warnings and errors like this:

    [javac] warning: java/lang/Object.class(java/lang:Object.class): major version 51 is newer than 50, the highest major version supported by this compiler.
[javac] It is recommended that the compiler be upgraded.

    I believe it was still compiling with the 1.6 compiler from the 1.7 JDK. Once, I modified all the aforementioned steps I compile the module and still get a different error:

    [javac] javac: invalid target release: 1.7
[javac] Usage: javac <options> <source files>

    This is where I got stuck.
