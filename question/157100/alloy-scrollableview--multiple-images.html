Alloy scrollableView : multiple images

Hello,
I would want to create a scrollableview with images stored in a table vues[] but by using Alloy. I do not manage to show the images in my scrollableView.
I am not sure that views = "vues" is taken into account.

Thank you in advance for your help.

tabGallery.xml

    <Tab id="tabGallery" title="Gallery">
        <Window>
            <ScrollableView id="portefolio" showPagingControl="true" currentPage="0" pagingControlHeight="30" views="vues">
                <!--ImageView id="image1" image='/img/1.jpg'/-->
            </ScrollableView>


        </Window>
    </Tab>

</Alloy>

tabGallery.js

var vues=[];
for(i=0;i<3;i++){
    var imageVue = Titanium.UI.createImageView({ image:'/img/'+i+'.jpg' });
    vues.push(imageVue);
};

//$.portefolio.addView(imageVue);
//$.portefolio.addView(vues);

Configuration
Titanium Studio, build: 3.1.2.201308091617 Android emulator 4.1.2(x86) Windows 7 jdk1.6.0_25

    Adding the string vues to the views attribute in XML does not do what you think it does. You could do it like this:

    <Tab id="tabGallery" title="Gallery">
        <Window>
            <ScrollableView id="portefolio" showPagingControl="true" currentPage="0" pagingControlHeight="30">
            </ScrollableView>
        </Window>
    </Tab>
</Alloy>

    var vues = [];
for (var i = 0; i < 3; i++) {
    var imageVue = Titanium.UI.createImageView({ image:'/img/' + i + '.jpg' });
    vues.push(imageVue);
}
$.portefolio.views = vues;

  • It's an other solution by using Alloy's data binding.

    tabGallery.xml

    <Alloy>
    <Collection src="Photos" />
    <Tab id="tabGallery" title="Gallery">
        <Window id="winPhoto">
            <ScrollableView id="portefolio" showPagingControl="true" currentPage="0" pagingControlHeight="30" dataCollection="Photos">
                <ImageView id="image1" image="{fichier}"/>

            </ScrollableView>


        </Window>
    </Tab>

</Alloy>

    Nothing in tabGallery.js but you must create a model (Photos.js) and fill the collection with data.

    Photos.js

    exports.definition = {
    config: {
        columns: {
            "libelle": "text",
            "fichier": "text"
        },
        defaults : {
            "libelle" : "-",
            "fichier" : "/img/0.jpg"
            },
        adapter: {
            type: "sql",
            collection_name: "Photos"
        }
    },
    extendModel: function(Model) {
        _.extend(Model.prototype, {
            // extended functions and properties go here
        });

        return Model;
    },
    extendCollection: function(Collection) {
        _.extend(Collection.prototype, {
            // extended functions and properties go here
        });

        return Collection;
    }
};

  • Hello, I'd suggest you could try with Alloy's data binding capabilities, since form Alloy 1.1.0 it is supported also for ScrollableView.

    You can find documentation at http://docs.appcelerator.com/titanium/3.0/#!/guide/Alloy_Data_Binding.

    With that approach you don't need to populate manually the views, just need a "vues" collection to bind your Scrollableview to.

    Hope it could help.
