  • Go to android-sdks&#x2F;platform-tools&#x2F;lib folder and search for current files within, in my case:

    egomezMBP:lib egomez$ ls -l
total 1680 
-rw-r--r--  1 egomez  staff  859105 Apr 13 12:30 dx.jar

    If that file is there copy it manually and put it under \android-sdk\platforms\android-8\tools\lib\

    Hopefully after rebooting Windows, Android will be able to find that file and complete build process. Otherwise you may want to re-install Android-sdks or better yet update them through out Manager to make sure everything has been set properly.
