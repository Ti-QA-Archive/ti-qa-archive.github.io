Android Parity: ImageView error event misses image property

Hi all,

when an ImageView fires an error event, there should be an image property as described in documentation. It works well on iOS but not on Android.

Example:

var win = Ti.UI.createWindow();
win.open();

var imageView = Ti.UI.createImageView({
    width:"auto",
    height:"auto",
    image: 'http://invalidusrlssdsdlsdkdjer.com7/rwerewrer'
});

imageView.addEventListener('error', function (event) {
    for (var prop in event) {
        Ti.API.info(prop + ' value: ' + event[prop]);
    }
});

win.add(imageView);

Steps to reproduce

Just place this in your app.js.

The event contains just the source and type property on Android.

Regards Markus
