Where is Ti SDK 3.1.3 installed?

In previous versions of the Ti SDK, they were installed on OSX in /Library/Application Support/Titanium/mobilesdk/osx, which has a list of directories for 3.1.2.GA, 3.0.2.GA, etc… However, 3.1.3 is installed but it's not in that folder. I did a search on my mac and couldn't find a 3.1.3 folder anywhere, so where is it??

The reason I need to know is because I build/launch from command line rather than from Ti Studio and certain apps of mine require certain Ti SDKs. A command like this:

titanium build --platform android --target device

only works if there is one Ti SDK in the osx folder. If there are multiple then it says:

INFO]  tiapp.xml <sdk-version> set to 3.1.1, but current Titanium SDK set to 3.1.3
[INFO]  Forking correct SDK command: /usr/local/bin/node /usr/local/bin/titanium build --sdk 3.1.1
Target platform [android,ios,mobileweb]:

Since the 3.1.3 folder isn't there, I can't remove it, so I'm stuck dealing with this annoying prompt, which slows you down on iOS and doesn't work at all on Android (it launches emulator when you told it to install to device.)

  • Never mind, I tackled the issue another way. Now I specify the Ti SDK in the build command itself, so I'm not at the mercy of titanium build guessing which Ti SDK to use.

    titanium build --platform android --target device --sdk 3.1.1
