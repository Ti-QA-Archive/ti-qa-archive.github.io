  • Hi

    You are using third-party code that aims to solve a problem, namely the ability to override the standard way tabs work - there is nothing wrong with that.

    That said as it is third-party code and as it does come from the Market Place there is very little that can be done without the original author making the changes or someone else trying to add that functionality to that persons code.

    This if often the case when using third-party code to solve one problem - you end up with another, no matter how great the original code was.

    I would suggest you either look for a different way to achieve your custom tabs requirement or determine a way for the app to not require 6 tabs and to use five instead as there are many code examples that achieve that well.

    Keep in mind when you have tabs each tab should have the same 'feature weight'. By this I mean that each tab should be a separate distinct function within the app. If the tabs have similar functions then join them together and use a tabBar to swap between the similar functionality. If one of the tabs is no more than supplementary information or a limited set of functionality ass this as a button or other link from within a 'home' or other generally focussed tab.
