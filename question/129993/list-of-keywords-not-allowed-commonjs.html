  • I'm not so sure that keywords are reserved in CommonJS standard. As far as I know, CommonJS just mentions "require" function and "exports" object in module 1.0 standard and it introduced "module" object later on.

    Since "exports" and "module" are objects from implementation point of view, I think there shouldn't be any restriction regarding property names in that objects (except for few that CommonJS standard will use in future specs versions like "module.id").

    Therefore, I think this should be considered as a bug in Titanium implementation and I think it exists because some interntal Titanium workings uses them.
