Update existing SQLite database of one app, from another app (both my apps) - Android

I was wondering if it is possible to update an app's database, from another apps query? So basically I created two apps, App A and App B. For App A, I have a database I created called App_A.db. So I go ahead and create another app called App B. In App B I want to update the database in App A (App_A.db). Is this possible? I know the databases are stored at /data/data/<app_name>/databases/<db name>. I have tried a bunch of different variations like: 

var db = Titanium.Database.install('/data/data/com.app_a.com/databases/App_A.db', 'App_A');

as well as: 

var db = Titanium.Database.open('/data/data/com.app_a.com/databases/App_A');

Is this even possible with Appcelerator? I even have my phone rooted and tried some "adb shell" commands and verified the database existed.

  • The basic problem is each app is "sandboxed" to its own application. When you try to give an absolute path in your open method, if you start your path with a leading / character that means the root of the current app's resource folder structure – not the root of the device. If you leave off the leading /, you get a path relative to where the current *.js file resides.

    In short, AFAIK, you can't reach the sandbox of another app even when you are the author of both.

    Since this is android, if the device has a storage card present, what you can do is have each app access the storage card at an agreed upon common location as a means of sharing data. But you can't directly open a database in another app's sandbox.
