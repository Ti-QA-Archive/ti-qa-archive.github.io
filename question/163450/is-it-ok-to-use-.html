  • It depends what you are actually trying to size.

    In my code I use mainly the dp units which translates to a consistent "real physical size" on every device.

    With dp I sometimes need to change from dp to px or vice-versa and I use those two functions for that
    (I use alloy, I am not sure but OS_ANDROID constant is not present without alloy)

    
/* Convert units
-------------------------------------------------------------- */
function pxToDp(pixels) {

    return OS_ANDROID ? pixels / Ti.Platform.displayCaps.logicalDensityFactor : pixels
}


function dpToPx(dpUnits) {

    return OS_ANDROID ? dpUnits * Ti.Platform.displayCaps.logicalDensityFactor : dpUnits
}

    I use % sometimes but mainly when I want to divide the screen. (says two elements that will take half of the screen)

  • Hi Daniel, you can also try the code below. I've tested it and works fine.

    First create a user define module ui.js and use the code below

    var base = {
    width : 320,
    height : 480
};
var orgWidth = Titanium.Platform.displayCaps.platformWidth;
var orgHeight = Titanium.Platform.displayCaps.platformHeight;
var divide = base['width'] / orgWidth;
exports.fit= function(px) {
    return Math.round(px / divide);
};

    Then in your every window use it like

    var ui = require('ui');
Titanium.UI.setBackgroundColor('#000');

var win= Ti.UI.createWindow({
 backgroundImage: '#000'
});

var anImageView = Ti.UI.createImageView({
    image : '/img.jpg',
    width : ui.fit(320),
    height : ui.fit(240),
    top : ui.fit(50),
    left : ui.fit(50)
});
win.add(anImageView);

win.open();

  • Hi

    As the others have already stated in various ways, using percentages is not the best way to go about creating your layouts.

    As you have noticed Android has many different screen sizes or resolutions which we have to deal with. The best way I have found in dealing with this is to use the Logical Density Factor to help calculate the different elements on your screen.

    LDF is used as a scaling factor for different displays to allow you to correctly scale your Density Independent Pixel (dip) units. This way you can make your app do all the hard work for you.

    In addition to this you should be making use of Ti.UI.SIZE , which constrains the element to the size of its contents, and also Ti.UI.FILL , which allows the element to fill to the size of its parent.

    In answer to your actual question, I don't know if there is a performance issue with using percentages as this would require a lot of testing to prove it against other methods, but whatever method is used, there will still be calculations made to define your sizes.

    At the end of the day it is your app so you can work with whatever unit or measurement you want.

    Hope this helps!
