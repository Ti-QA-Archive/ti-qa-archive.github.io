Does SQL's " FOREIGN KEY REFERENCES " work in Titanium (Mobile)?

On Titanium (Mobile) can you define a column in one titanium database table as a foreign key referencing a Primary key in another table?

Or are there any other elegant ways to ensure key integrity.

This doesn't seem to work:

//  tableOne
myApp.bvxDB.execute('CREATE TABLE IF NOT EXISTS tableOne ('+
                     'first_id TEXT PRIMARY KEY, ' +
                     'a_field TEXT)');

//  tableTwo
myApp.bvxDB.execute('CREATE TABLE IF NOT EXISTS tableTwo ('+
                     'second_id TEXT PRIMARY KEY, '+
                     'first_id TEXT FOREIGN KEY REFERENCES tableOne(first_id), '+
                     'another_field TEXT)');

Thanks very much.

  • According to the docs, Titanium databases implement SQLite 3. I'm not sure if that includes the foreign key references as you ask. On this page (http://www.sqlite.org/lang_createtable.html) it looks like the current incarnations of SQLite do. Maybe there are some syntax specifics for SQLite that you need to take into consideration?
