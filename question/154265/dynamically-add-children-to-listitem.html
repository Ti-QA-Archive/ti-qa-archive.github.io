Dynamically add children to ListItem

Hello!
I have a ListView that is populated dynamically.
Now, within each ListItem I want to display buttons, but the number of buttons will vary from item to item.

So, what I've tried to do is to, in the Template, define a 

{

    //... all the other elements that work fine

    type : 'Ti.UI.View',
    bindId : 'hiddenItemView',
    properties : {
        width : '100%',
        height : 0,
        layout : 'vertical',
        align : 'left',
    }, 
    childTemplates : []
}

and then, when populating the ListView, I've tried the following:

    buttons = [{
        type : 'Ti.UI.Button',
        properties : {
            title : 'test'
        }
    }];

    dataSet.push({

        //... all the other elements that work fine, i.e. a few labels

        hiddenItemView : {
            childTemplates : buttons
        }

    });

Now, this doesn't affect my hiddenItemView view at all. I've tried manually populating it with some Labels just to see if they disappear, but they don't. So, I'm guessing that childTemplates cannot be manipulated from the dataSet.

So, how can I instead dynamically add buttons to a View within my ListItem?

Thanks!
Erik

    Hi

    The short answer is that ListView is designed for data driven fixed layouts, whilst TableView is designed for view based more dynamic layouts.

    This straight from the ListView documentation;

    • You cannot add views to a ListItem using the add method, which could be done with a TableViewRow. To add views to a ListItem, you need to define an ItemTemplate, which is bound to a list data item using the template property.

    It might be possible depending on the complexity of your needs by creating multiple templates, but still sounds better handled in the TableView instead.
