    • Tomasz

      For Titanium Desktop SDK, we would expect to see the following in the diagnostic log:

      Titanium Desktop SDK Version: 1.1.0

Titanium Desktop SDK Location: /Library/Application Support/Titanium/sdk/osx/1.1.0

      And for Mobile SDK, something like this:

      Titanium Mobile SDK Version: 1.6.0

Mobile SDK Timestamp: 02/23/11 12:34

Mobile SDK Build Number: 9882e81

Titanium Mobile SDK Location: /Library/Application Support/Titanium/mobilesdk/osx/1.6.0

      From what you have provided, your TiStudio doesn't have either, which indicates that it is not finding any SDKs on the system.

      Can you tell me where your titanium directory is located? Note that this will contain a mobilesdk and sdk directory.

      Does the &#x2F;Library&#x2F;Application Support&#x2F;Titanium&#x2F;sdk&#x2F;osx&#x2F; folder exists, which is where we would usually expect to see it.

      Cheers

      — commented May 5th 2011 by Paul Dowsett

    • I had the same problem. It was problem with permissions: /Library/Application Support was not writable to do the logged user (me). I fixed it by creating /Library/Application Support/Titanium/ and giving my group (staff) write permissions there. Everything works OK now (choosing Android SDK and downloading SDKs)

      — commented May 10th 2011 by Marcin Gozdalik
