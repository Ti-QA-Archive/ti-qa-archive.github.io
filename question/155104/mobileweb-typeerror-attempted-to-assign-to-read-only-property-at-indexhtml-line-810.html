  • FYI Mobile Web is a little stricter than the other platforms when it comes to read only properties. On iOS, if you try to write to a read-only property (such as Ti.Platform.osname), it does nothing, but on Mobile Web it will throw this error.

    With regards to debugging, the index.html file contains the Mobile Web SDK itself, but you should be able to see app.js as it was originally written. The debuggers often show index.html first, so you may need to use the debug tools to navigate to your code. I find that it's helpful to add a "debugger" statement to my code so that you can easily get to debugging your code without having to step through the SDK.

    To diagnose this issue, I would recommend telling the debugger to break on all uncaught errors. Once this happens (right before the red screen is shown), it will halt inside the SDK. If you go up the call stack, you will eventually get to the offending code.
