  • While I would not call this a "good" answer, just thought I'd comment that I saw the same problem but was able to run the executable created in the dist folder from a bash shell by identifying the installed libraries which provide the missing functions and forcing them to be preloaded - ie

    bash$ export LD_PRELOAD="/lib/i386-linux-gnu/libgcrypt.so.11.6.0 /usr/lib/i386-linux-gnu/libgnutls.so.26.14.12"
bash$ full_path_to_executable

    The details would be slightly different on a different system.
