    >> It's impressive how well crafted and cool the App is. Is this possible only using a lot of html,css and webviews? Or am I missing something?

    Though it's possible to use html+css in webviews, this is not common practice in Titanium - for this just use PhoneGap ;-). The provided API allows you to achieve pretty much anything you want, with varying degrees of effort. You can either use standard native controls, or create customized views. Moreover, if you want to create specialized native components (e.g. for leveraging specific target OS features, or existing libraries) you can do it creating native modules.

    • I think Olivier has given you the best answer.

      Titanium at its basic level offers you native controls, something that html/css/phonegap combos will not. This native components will run faster than engineered solutions built with html/css in a web view (essentially what phonegap does).

      However, using prototyping, you can can create your OWN objects based upong NATIVE components and create a non-standard, custom design.

      Which is exactly what TripLingo did. When Vince says break the rules, he wasn't kidding. Their app works. Its UI works for what the app is supposed to do, facilitate language learning on a limited screen, in your hand. To do that, they had to create a custom UI that DOESN'T match the iOS design suggestions. However, they have 16 separate apps in the App Store based upon this design with thousands of happy customers. It must have worked.
