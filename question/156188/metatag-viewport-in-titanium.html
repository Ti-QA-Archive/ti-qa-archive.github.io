Metatag Viewport in Titanium

Hi,

i´m trying to give my app a fixed width of 320 px on any device. I´m looking for something like the HTML Metatag "Viewport" : 

<meta name="viewport" content="width=320px, initial-scale=1.0, user-scalable=no">

If I give my window a width, i scales to 320 px, but i doesnt fit the screen of my tablet (A501) anymore. 

var win = Titanium.UI.createWindow({  
    width:"320",

});

I hope anybody understood what i´m trying to do and can help me ;)

  • Accepted Answer

    Titanium isn't web development. You're not dealing with a browser canvas and setting a meta viewport width has no place in Titanium (unless you're rendering a WebView within your app). The behavior you saw on your Android device is expected – if you set the width explicitly less than the device's width the UI isn't going to fill the device.

    Can you explain why you're trying to set a fixed width? Maybe we can help you achieve the same results using a different technique than what you've tried.

    • The simplest technique is to use percentages and Ti.UI.SIZE/Ti.UI.FILL to make the app pretty much the same (just enlarged) on all devices. You can also use 'dp' (display independent pixels) for dimensions and font sizes so that they scale based on the device resolution. For example:

      myview.width = '100dp';
var label = Ti.UI.createLabel({
  width: Ti.UI.FILL,
  font: {
     fontSize: '20dp'
  }
}

      But the best design is a responsive UI pattern, where the layout changes depending on the screen size, especially when going between phones and tablets.

      Android in general gets a bad rap for encouraging developers to create "tablet apps" that are just enlarged phone apps. Apple does a better job of forcing developers into actually taking advantage of the differences in both screen size and typical usage. iPad apps almost always have different UI/UX than their iPhone counterparts.

      Think about it: most phone use is one-handed, on the go. Most tablet use is two handed, sitting and leaning back, tablet at arm's length, and more media consumption. Your UI/UX should account for those typical usage patterns.
