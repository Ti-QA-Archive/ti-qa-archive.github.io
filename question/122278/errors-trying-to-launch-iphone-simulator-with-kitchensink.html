  • I think I've figured out the problem; the clue is in the error message, which mentions that 'ADBannerContentSizeIdentifierPortrait' is undeclared. This apple develop page indicates that this symbol was only introduce in iOS SDK 4.2, so the compile is failing when it can't find it's definition.

    So we need to upgrade your iPhone/iOS SDK to 4.2 or higher for compiling against Titanium 1.7.1 to work. I'm now doing the 4GB download of the xcode+sdk right now and will be able to confirm once that's installed :) But it looks pretty certain that that is the issue.

    In the meantime, I'm going to code with with the Titanium 1.7.0 SDK which doesn't have this requirement.

  • I've developed a workaround after struggling with the same issue. Seems to be something to do with Titanium SDK 1.7.1. Downgrading to Titanium SDK 1.7.0 works as a workaround (go into the tiapp.xml file in the root of your project directory, and change the SDK version from 1.7.1 to 1.7.0), for me at least. I'm using iOS SDK 4.1 and xcode 3.4.2 on OSX 10.6.7
