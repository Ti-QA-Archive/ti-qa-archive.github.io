Using the Titanium CLI and getting error: You have not agreed to the Xcode license agreements, please run 'xcodebuild -license'

On OS X 10.8.4 using Titanium 3.1.1 CLI with Xcode 4.6.3, when I run the following command from the project directory, I get the following error:

titanium build -p ios -T simulator -Y iphone

[ERROR] You have not agreed to the Xcode license agreements, please run 'xcodebuild -license' (for user-level acceptance) or 'sudo xcodebuild -license' (for system-wide acceptance) from within a Terminal window to review and agree to the Xcode license agreements.

I have run and agreed to xcodebuild -license and sudo xcodebuild -license multiple times and I see plist files for Xcode stating that I have agreed to version 4.63 in both /Library/Preferences and ~/Library/Preferences.

If I manually run xcodebuild -sdk iphonesimulator in the build directory of my project, the application builds successfully.

If I run xcode-select -print-path I get &#x2F;Applications&#x2F;Xcode.app&#x2F;Contents&#x2F;Developer which appears correct to me.

Has anyone else run into this? I've closed and restarted my applications, rebooted, and pleaded with my computer to no avail…

Thanks!
