malformed database causes crash on iphone SDK < 4.2

I'm getting very close to finishing my first iphone app with titanium. It's working well in the iphone simulator with SDK 4.2, but when I launch it with 4.1 or below, it always crashes immediately with the below error:

[ERROR] Error occurred calling next on a PLSqliteResultSet. SQLite error: 'database disk image is malformed' for 'SELECT COUNT(*) as theCount FROM reviews where id = 11202'
[ERROR] The application has crashed with an unhandled exception.

I've tried commenting out various queries in my code. It occurs regardless of whether it's a select, delete, etc.

Sadly, I can't find a thing about it online. Any suggestions?

Thanks,
-Sean
