Declare Function, Not Execute

I am not sure if I am totally missing something here or its just my familiarity with Java and HTML, but I have a question regarding declaring a function vs. executing the function.

I have a code that lays out a table on the screen and then when a button is pressed I want it to execute a function I declared earlier in the file, but when I simulate on iOS it tells me that variable is undefined. This makes sense since I haven't actually created the table before the function is declared so I was wondering if there was a way in Titanium to declare a function for later use (via click event) without it executing as the UI is built. See below code for example (with comments).

// Want to declare this function, not execute
function calcPoints() {
    var proteinVAR = parseInt(textFieldsArray['PROTEIN'].value);
    var carbsVAR = parseInt(textFieldsArray['CARBS'].value);
    var fatVAR = parseInt(textFieldsArray['FAT'].value);
    var fiberVAR = parseInt(textFieldsArray['FIBER'].value);

    pointsCalculated = proteinVAR + carbsVAR + fatVAR - fiberVAR;

    var alertNotice = Ti.UI.createAlertDialog({
        buttonNames: ['OK'],
        title: 'Calculated Points',
        message: 'Points' + pointsCalculated
    });    

    alertNotice.show();
}

calcButton = Ti.UI.createButton({
    font:{fontSize:17, fontWeight:'bold'},
    height: 45,
    width: '45%',
    left: 10,
    bottom: 85,
    title: 'Calculate'
});
calcButton.addEventListener('click', calcPoints());

If this needs more clarification, please ask for it. Thank you in advance!

  • You can declare a function in two ways:

    1. function myFunc() {
...
};
2. var myFunc = function(){
...
}

    to reference a function

    var myReference = myFunc;

    to execute a function

    myFunc();
myFunc.call(contextRef, param1, param2, ...);
myFunc.apply(contextRef, arrayOfParams);

    so when adding to Event listeners you dont want to execute it immediatly, but give a reference.
