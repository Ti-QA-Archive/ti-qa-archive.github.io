Picker options don't appear (Ti 1.7.6)

Hi,

I'm doing some changes in an already made application that is using Titanium 1.7.6, and i'm with problems with the picker element on the android 2.2 emulator (haven't tested in other Android versions or devices).

It appears that a picker in a table row, doesn't work properly, i.e., when we click the picker, it's window with the options doesn't appear.

I've managed to reproduce the problem in the following test case:

    //create window instance
    var win = Ti.UI.createWindow({
        backgroundColor:'#ffffff',
        navBarHidden:true,
        exitOnClose:true
    });

    var table = Ti.UI.createTableView();

    //Header section
    var headerSection = Ti.UI.createTableViewSection();
    headerSection.headerTitle = 'Header Section';
    headerSection.add(Ti.UI.createTableViewRow({title:'Header section'}));

    //Picker section
    var pickerSection = Ti.UI.createTableViewSection();
    pickerSection.headerTitle = "Picker Section";
    var row = Ti.UI.createTableViewRow();
    var rowview = Ti.UI.createView();
    var picker = Ti.UI.createPicker({selectionIndicator: true,width: '100%'});
    var pickerData=[];
    pickerData[0]=Ti.UI.createPickerRow({title:'Bananas'});
    pickerData[1]=Ti.UI.createPickerRow({title:'Strawberries'});
    pickerData[2]=Ti.UI.createPickerRow({title:'Mangos'});
    pickerData[3]=Ti.UI.createPickerRow({title:'Grapes'});
    picker.add(pickerData);
    picker.addEventListener('change',function(e){
        alert("User selected: " + e.row.title);
    });
    rowview.add(picker);
    row.height = 'auto';
    row.add(rowview);
    pickerSection.add(row);

    //Footer Section
    var footerSection = Ti.UI.createTableViewSection();
    footerSection.headerTitle = 'Footer Section';
    footerSection.add(Ti.UI.createTableViewRow({title:'Footer section'}));

    table.data = [
        headerSection,
        pickerSection,
        footerSection];


    win.add(table);
    win.open();

Is this a known problem of this version(1.7.5 and 1.7.6) or am i doing something wrong?

    var rowview = Ti.UI.createView();
    var picker = Ti.UI.createPicker({selectionIndicator: true,width: '100%'});
    var pickerData=[];
    pickerData[0]=Ti.UI.createPickerRow({title:'Bananas'});

    pickerData[2]=Ti.UI.createPickerRow({title:'Mangos'});
    pickerData[3]=Ti.UI.createPickerRow({title:'Grapes'});
    picker.add(pickerData);
    picker.addEventListener('change',function(e){
        alert("User selected: " + e.row.title);
    });
    rowview.add(picker);
    row.height = 'auto';
    row.add(rowview);
    pickerSection.add(row);

    //Footer Section
    var footerSection = Ti.UI.createTableViewSection();

    footerSection.add(Ti.UI.createTableViewRow({title:'Footer section'}));

    var tableData = [
        headerSection,
        pickerSection,
        footerSection];

    var table = Ti.UI.createTableView({data: tableData});

  • Hi Andre

    Two questions;

    • Can you not upgrade to the latest version of the SDK (2.1.1 as of now), there have been significant improvements in all areas and you may be getting stuck by a problem that has been fixed.
    • Why are you using a picker in a tableviewrow as the parent for your picker? tableviewrows do an excellent job of showing formatted tabular data in a variety of ways, but some combinations are best left un-tried. You may be better off adding the picker to a scrollview or a headerview for the tableview

    If you cannot change the SDK version and you cannot use a more appropriate parent for the picker, provide a bit more detail into why your layout requires the picker in a row.

    Cheers
