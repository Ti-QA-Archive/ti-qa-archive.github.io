Tab Won't fire original function assigned.

In my app.js, I call a splash screen imageview, and then wait for the user to click. This takes them to a function that has two tabs in a tab group, one for Guide and the other for About Us. The Guide part works great. However, when I click the About Tab, it shows its HTML screen. If you click that same tab again, it jumps ALL the way back to showing the splash. I can't see a path for that:

//aboutus.js
var win = Titanium.UI.currentWindow;
var w = Ti.UI.createWindow();
w.orientationModes = [
    Titanium.UI.PORTRAIT,
    Titanium.UI.LANDSCAPE_LEFT,
    Titanium.UI.LANDSCAPE_RIGHT
];

w.barColor='#007C28';
w.navBarHidden=true;

var webview = null;
webview = Ti.UI.createWebView();
webview.url = 'html/aboutus.html';

w.add(webview);
win.tab.open(w);

In an earlier js, I set that tab to the aboutus.js.

//in main.js
var about_win = Titanium.UI.createWindow({ 
    title: 'About CSR', 
    url: 'aboutus.js',
    barColor: '#007C28'
});

var about_tab = Titanium.UI.createTab({ 
    icon:'images/KS_nav_ui.png',
    title: 'About Us',
    window:about_win
});

How can the path go all the way back to the original splash screen, unless that view is on that tab's stack (which it shouldn't be?

    you are creating an extra window and opening the window incorrectly with the tab object; this code should work

    //aboutus.js
var win = Titanium.UI.currentWindow;
win.barColor='#007C28';
win.navBarHidden=true;

var webview = null;
webview = Ti.UI.createWebView();
webview.url = '../html/aboutus.html';

win.add(webview);
