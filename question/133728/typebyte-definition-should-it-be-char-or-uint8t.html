TYPE_BYTE definition. Should it be char or uint8_t?

in TiUtils.m's encodeNumber function it has:

case TI_BYTE: {
            char byte = [data charValue];
            memcpy(bytes+position, &byte, size);
            break;
        }

should that be:

case TI_BYTE: {
            uint8_t byte = [data charValue];
            memcpy(bytes+position, &byte, size);
            break;
        }

to make byte unsigned?

As it stand now, its signed so 0xA0 turns into 0xFFFFFFA0 (-96 DEC) instead of (160 DEC).
