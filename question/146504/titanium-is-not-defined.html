    Hi

    You have not explained WHY you want to control the camera from within a webView rather than outside as the code other parties code has, why?

    If you are trying to access Ti/Titanium from within a webView and more specifically Titanium.UI.currentWindow there are more fundamental issues that need to be worked out first.

    For example Titanium.UI.currentWindow ONLY works with multiple context apps (using the URL property of a window) so that scope would need to be determined before anything else.

    If you simply wanted to trigger the block of camera code FROM the webView then that is very simple;

    Code for your mobile app;

    Ti.App.addEventListener('test', function (e) {
    alert(JSON.stringify(e));
});
var web = Ti.UI.createWebView({
    height: Ti.UI.FILL,
    url: 'test.html',
    width: Ti.UI.FILL
});
win.add(web);

    test.html file placed in the resources folder.

    <!DOCTYPE HTML>
<html lang="en-US">
<head>
    <meta charset="UTF-8">
    <title>Test</title>
<script type="text/javascript">
    function test(boolTest) {
        Ti.App.fireEvent('test', { test: boolTest });
        return false;
    }
</script>
</head>
<body>
    <p><a onclick="test(true);">Test Yes</a></p>
    <p><a onclick="test(false);">Test No</a></p>
</body>
</html>

    This will simply case an alert controlled within the app to be initiated from the webView, and more than one parameter can be returned, other code run - basically whatever you want.

    However your answers to the questions Stephen Feather (and myself) asked for versions and platforms may provide better insight.
