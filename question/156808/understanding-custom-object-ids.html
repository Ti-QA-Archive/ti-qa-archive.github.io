Understanding custom object ID's

I'm having trouble with the custom object update command. I understand the structure and can create custom objects, but I'm having trouble understanding the naming convention for the custom object "id" field, which recognizes the created custom object and updates it. The Appcelerator documentation isn't that clear as to how they created an id for the object. (i.e. savedCar1.id is not mentioned in the create object documentation). I have provided my own example below that I would like to figure out with regards to updating:

The id fields for the Cloud.Objects.update is left blank because I'm not sure what the id could be.


Cloud.Objects.create({
          classname : 'goingOutTonight',

fields : {
            going_out: 'No'
          }
        }, function(e) {
          if(e.success) {
          var goingOut = e.goingOutTonight[0];
            alert("created" + 'id:' + goingOut.id + 'going_out:' + goingOut.going_out);
          } else {
            alert('Error: ' + ((e.error && e.message) || JSON.stringify(e)));
          }
        }); 


        Cloud.Objects.update({
          classname : 'goingOutTonight',
          id: ,
          fields : {
            going_out: 'Maybe'
          }
        }, function(e) {
          if(e.success) {
            alert("Updated");
          } else {
            alert('Error: ' + ((e.error && e.message) || JSON.stringify(e)));
          }
        }); 


        Cloud.Objects.update({
          classname : 'goingOutTonight',
          id: ,
          fields : {
            going_out: 'Yes'
          }
        }, function(e) {
          if(e.success) {
            alert("Updated");
          } else {
            alert('Error: ' + ((e.error && e.message) || JSON.stringify(e)));
          }
        }); 
    }
