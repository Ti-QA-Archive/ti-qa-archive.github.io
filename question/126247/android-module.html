Android module

Hello,

When I tried to make a module project with android I will get an error like

"[ERROR] Couldn't find the google APIs r%s addon directory" % self.sdk.DEFAULT_API_LEVEL
AndroidSDK instance has no attribute 'DEFAULT_API_LEVEL'

Anyone know how to fix this?

    I ran into this issue when creating an Android module earlier today.

    Assuming the Google APIs are installed I still ran into an issue with the Titanium module create command:

    titanium create --platform=android --type=module --name=Foo --id=me.izen.FooBar --android=$ANDROID_SDK

    The best part is that the script reported "Created android module project" but there was none to be found on the file system.
    I did a bit of digging around and found that the Google Android boys did a bit of renaming as indicated in a previous answer. I traced the error down and made a few chanages.

    From Titanium&#x2F;mobilesdk&#x2F;osx&#x2F;1.7.5&#x2F;android&#x2F;androidsdk.py:

    93         def find_google_apis_dir(self):
94                 self.google_apis_dir = self.find_dir(self.api_level, os.path.join('add-ons', 'addon-google_apis-'))
95                 if self.google_apis_dir is None:
96                         self.google_apis_dir = self.find_dir(self.api_level, os.path.join('add-ons', 'addon-google_apis-google_inc_-'))

    Hope this helps…

  • i found that android sdk manager <r14 naming convention in add-ons subfolder is
    addon_google_apis_google_inc_7
    whereas sdk manager r15 is
    addon-google_apis-googleinc-7

    I renamed all my sdks to the naming convention like in <r14 for create to work
