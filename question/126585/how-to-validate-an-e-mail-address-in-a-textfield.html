    Something like this should get you started, you might want to double check the regEx code, it is not my strength

    emailInput.addEventListener('change', function(e){
    var reg = /^([A-Za-z0-9_\-\.])+\@([A-Za-z0-9_\-\.])+\.([A-Za-z]{2,4})$/;      

   if(reg.test(e.source.text) == false) {
        emailInput._valid_flag = false;
    } else {
        emailInput._valid_flag = true;      
    }
});

    basic concept is to attach a new property on the field and set it based ohe chars entered

  • Are you trying to validate the format of the email address is valid, or that the address itself is a valid email box?

    If the latter, you really can't prove an email address is valid without sending a registration type message and requiring a response. Appropriate for some web pages but not a mobile app.

    If the former, I suggest you use the regex in this stackoverflow thread instead of the one Aaron used in his example. It covers more of the subtle nuances which may occur in an email address.
