  • Marc, things can definitely be hard when first setting up your environment for Titanium. On top of that, I've found the Android platform to assume a lot about its users competency with installing SDKs, using the command line, using eclipse, etc. So you're doing just fine for a new user.

    titanium_1_wvga800 is most likely your android emulator. That was the name of my first one. I can't remember when I first started using Titanium Studio whether android was installed already or I had to download it separately. Either way, it sounds like you have it going…

    You should be able to create a new project in Titanium Studio. After that, you should see Run in the top left corner which has that green button with a white play button inside it. Right next to that, there should be another dropdown, it may say "Android Device" right now but if you select that you should see a dropdown with options. Go to Android Emulators and select titanium_1_wvga800. Click run to ensure the command gets sent and you should be good to go. I'm no support guy so I don't know if that will work but that's the basic workflow of compiling your app to run. What operating system are you on, by the way?

    • Hey Marc,

      Actually I mentioned this in my first answer, but it may have been overlooked since these links apply to after you get the emulator working. You're not the only one who thinks the Android Emulator is slow. There are a couple options to help you with that.

      Anyway, there are a lot of different options for this. Above I posted these two links:

      Speed up for android emulators

      Video on how to create a faster emulator (I like this one!)

      Another option would be Genymotion. I don't know a lot about it just yet, but I think its a different codebase for android emulators. Basically they're "the faster android emulator". I think they're seperate from the ones you create with the android SDK. Check it out

      A lot harder option, which I've used but haven't really perfected yet, is TiShadow. Its an app you install on your Emulator, and then instead of building your app to the emulator, you build the app to theTiShadow app inside the emulator that's already running and it automatically refreshes. No closing and starting a new emulator which takes all that time. I've found some weird kinks with it, but people seem to love it. Check it out
