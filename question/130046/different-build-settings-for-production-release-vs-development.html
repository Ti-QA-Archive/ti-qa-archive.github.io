Different build settings for production release vs development

The development version of my mobile iphone/android app needs to use different settings than the production version, specifically for the facebook app id and the server endpoint that it's hitting. Is there a way for me to have different builds that include different files (settings-dev.js vs settings-prod.js)?

  • Sure, 

    var production = false;

if (!production){
Ti.Include('developmentsettings.js');
} else {
Ti.Include('productionsettings.js');
};
