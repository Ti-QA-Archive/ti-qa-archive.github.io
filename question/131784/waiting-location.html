    • an indicator is a sort of "wait until a task is complete", that said you will need to hide it when your request is finished no matter if sucessful or not. The 'location' event will fire when the location has changed(!), if you need a single location then use

      Titanium.Geolocation.getCurrentPosition();

      see here => http://developer.appcelerator.com/apidoc/mobile/latest/Titanium.Geolocation.getCurrentPosition-method.html

      be aware, it returns the last valid position.

      if you are on android, add an android:back event and hide there the indicator.

  • You need to show an activityIndicator before you make the call and then hide it in the callback. 

    Titanium.Geolocation.getCurrentPosition

    Will execute only once.

    Titanium.Geolocation.addEventListener('location',

    Will execute repeatedly as your location changes, so you only need to use one or the other, depending on your needs
