Added platform target, mobile web, to existing app - build fails

I'm trying to add a platform target, mobile web, to my existing app. I added this to the tiapp.xml

    <mobileweb>
        <precache/>
        <splash>
            <enabled>true</enabled>
            <inline-css-images>true</inline-css-images>
        </splash>
        <theme>default</theme>
    </mobileweb>

and in the <deployment-targets> this 

<target device="mobileweb">true</target>

but the app fails while compiling with this console error

[INFO] :   - mobileweb/alloy/CFG.js
[ERROR] :  : ENOENT, no such file or directory '/Users/--path to Ti workspace --/config.xml'
[ERROR] :  Alloy compiler failed

What did I forget to do? I don't know anything about a config.xml file.

    Hi

    This is likely one of two things;

    • You have not cleaned your project before trying a re-build - usually a good safety net process, backup your code and then if this is Alloy also delete the contents of the resources folder ONLY if an Alloy project - DO NOT do this if this is not Alloy - think I have covered myself there.
    • Your app has elements that are not compatible with MobileWeb - if you are some time down the road you may have added methods and features that simply do not work or require changes to work with mobile web. Also you may have a dependency issue - is something required that is not automatically created - a Database for exmaple

    To test the last one - create a new test app and include MobileWeb from the start - does this work? If so then it is not your configuration but your code in some way. Take a look for the differences.
