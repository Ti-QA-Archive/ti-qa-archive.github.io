  • You can use CSS3 and jQuery within a webview, but Titanium apps are written in javascript and become native applications. Titanium apps are not web apps wrapped in a native wrapper like PhoneGap apps are, so CSS3 and jQuery are not usable tools for creating Titanium apps.

    • As Anthony said, any application that runs in a webView instead of native code will be 100% depending on the phone's browser speed.

  • Yes you can use it by create html file, as you would similar to creating a website. Then you simply put everything in a webbrowser in the phone, which acts like a native app. Although it is possible, it is not the best approach from performance and user experience point.

    Firstly, you'll need to create user interface yourself using KendoUI, Sencha or similar frameworks. And this interface should be different for different platforms.

    In my experience (maybe I just have an old phone), any javascript heavy webapp performs poorly on mobile devices, because of hardware abilities I guess.

    Conclusion, although it is possible, it is not recommended, and I doubt there will be a difference between PhoneGap performance.
    But you can run a quick test. Create or use existing PhoneGap index.html and create an app in Titanium:

    //create window
var win = Titanium.UI.createWindow({  
    backgroundColor:'#fff'
});

//create webview provide url to HTML file
var webview = Titanium.UI.createWebView({
    url: 'http://somewebsite.com/index.html',
    top: 0,
    left: 0,
    width: "100%",
    height: "100%"
});

//add webview to window
win.add(webview);

// open window
win.open();
