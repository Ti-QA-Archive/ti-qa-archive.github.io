Apple push certificate getting disabled in the ACS

It's not the first time I'm using ACS, so I'm 100% sure that all the steps are correct. I've obtained the required, perfectly valid Push development certificate from Apple, create a certificate and uploaded the key to ACS. However, the first time I try to send a push from Console, the Certificate changes its status to "Disabled" in Console. No error message is given, even the push status is "Successful" (although my phone never receives it).

Any thoughts???

    When certificate status changes to Disabled, it means we cannot make a successful connection to APNS by using app developer provided certificate. Mostly it should be an expired certificate, or an invalid certificate. Sometime developer generates a p12 file from a private key not public key, and sometime p12 file is expired.

    Please double check your p12 file, and make sure it is valid and not expired. If there is still a problem, please either email your appKey or your p12 file/password to plv@appcelerator.com, and I will take a look.
