Getting Bad Request on Android Device - 400

I am getting a 400 bad request when testing on my android device. It's an Asus Transformer tablet with 4.0 on it. When I test in the emulate it works fine. It also works fine on iPad and iPhone. This is the call where I get a 400 - Bad Request

function loadXML()
{
    xhr = Ti.Network.createHTTPClient({
        onload: function(e)
        {
            parseXML(this.responseXML);
            actInd.hide();
        },
        onerror: function(e)
        {
            Ti.API.debug(e);
            alert(e.error); // 400 - Bad Request
            actInd.hide();
        },
        timeout: 5000
    });

    xhr.open('GET',generateURL(win.vforumLocation + 'presentation.xml',15));
    xhr.send();
}
loadXML();

I verified generateURL(win.vforumLocation + &#39;presentation.xml&#39;,15) returns a valid URL. I'd post the URL but it expires after 15 seconds so it would be pointless, plus I verified it works on my android phone with 2.2.2, my iphone and my ipad. The new android tablet is the only one giving me issues.
