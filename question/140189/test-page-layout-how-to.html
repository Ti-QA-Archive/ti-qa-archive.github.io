  • Well, it all depends,

    • If you want a native look and feel, you will have to re-create your interface using Titanium native controls.
    • If you want to keep that "web page" look and feel, you can embed your page in a webview.

    Keep in mind though, that if you go the webview route, you will have a more limited set of features since your code will be sandboxed in the WebView.

    • Apps like Facebook are native and use rich labels.
      (there is free a module for that)

      You can also use webviews (much like a button or an image) in cases where you want to display formatted text. Although you cannot use them too intensively since the performance hit will be too big.

      But my best advice to you when developing a mobile application. Don't try to make it look like a website, make it look like an application. If users want to look at something that looks like a site, they will just go to the actual site.

      While it is a nice "edge" that you are very familiar with HTML5 and CSS3 and all that. But forcing to apply those same principles will be more of a handicap in this case.
