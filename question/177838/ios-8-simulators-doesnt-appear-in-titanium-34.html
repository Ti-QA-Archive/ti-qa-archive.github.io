  • Just for a coincidence I have realized that the new projects work fine!! The only problem was on tiapp.xm of the project i was developingl!!

    The TAG <sdk-version> was not automatically updated after changing the SDK (even the hundred times i have change this on the UI of the Run Configuration from Ti-UI). I have ran Help-Studio-Run Diagnostic Test, copy the version of the Ti SDK show there in the TAG sdk-version of the Tiapp.xml (3.4.0.v20140916181713 in my case) and…. everything is in its place!!! I am able to use simulators on iOS7.1 and iOS8.0…

    Thats all!! The TAG sdk-version in file tiapp.xml of the project!!! (now it sounds even obvious that I should have updated that tag by myself… :))

    I am happy, probably you (the world) don't care, but I am sooooo happy…

  • So I had the same issue and Tom's fix related to isolating the modules worked for me. I downloaded the source from the dk.napp GitHub repo and opened the xcodeproj in XCode6. I updated the titanium.xcconfig SDK to the following TITANIUM_SDK_VERSION = 3.4.0.v20140919162515 however it wasn't working. You have to click on the "social" project and clcik on then on the target that says "social". At the bottom is a section called user-defined. The TITANUM_SDK_VERSION needs to be manually set there as it isn't set by the xcconfig. Once that is done the module compiled with the build.py and I was able to installed 1.7.4 and get it working for iOS8.
