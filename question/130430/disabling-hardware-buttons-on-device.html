Disabling hardware buttons on device

Hello,

Our main concern is how to disable all the hardware buttons on the devices we'll be using like option/menu button. We would like to prevent from our users to exit or "minimize" our application without entering a password.

Is it even possible?

  • For back and search buttons, you should be able to listen to the android:back and android:search events, and just do nothing when the event is fired. The menu button won't do anything unless you program it.

    As for the home button, I don't know of anything you can do in Titanium, but there some options for native android which you might be able to use via custom module of your own development.

      var win = Ti.UI.createWindow({
    title:'test',
    backgroundColor:'ff0000',
    modal:false  //makes the window heavyweight
});

win.open();

win.addEventListener('android:back',function(e) {
    Ti.API.info('back clicked'); 
});

    • That's not what exitOnClose is for. Setting it to true on your root window in android just makes sure the app closes when the back button is pressed while that window open – it is a way to keep from showing the splash screen (and requiring a second back button click) when backing out of an app. It's probably not related to what the OP is trying to do.

      The back event code isn't working because you probably don't have thewindowset as a heavyweight window. See here. Lots of additional info on what a heavyweight window is in the Q&A.
