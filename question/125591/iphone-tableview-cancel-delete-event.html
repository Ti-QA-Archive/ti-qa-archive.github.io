iPhone TableView, cancel delete event?

I have a tableview in my code that includes multiple rows. Only one of the row will have hasCheck = true. I would like to prevent users from deleting this row.

When the tableview is in edit mode (iPhone), the delete indicator shows for all rows. Is there a way to have it not show for a row? If not, is there a way to cancel the delete of the row (see code below)?

var tv = Ti.UI.createTableView({});
tv.addEventListener('delete', function (e) {
    if (e.row.hasCheck)
    {
        // some code that would cancel the delete event???
    }
});

    When creating the data object or rows for your table, try setting moveable:false and/or editable:false. You can see an example of this in the Kitchen Sink Demo in the file: table_view_edit_and_move.js
