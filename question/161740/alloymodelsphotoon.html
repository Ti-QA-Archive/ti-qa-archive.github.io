Alloy.Models.photo.on

I'm doing a custom object query and wanting to use the photo URL to display the image inside a <TableViewRow> using <ImageView>. I get the following error:

"'undefined' is not an object (evaluating 'Alloy.Models.photo.on')";

<TableView dataCollection="provider">
    <TableViewRow id="row" onClick="doViewDetails">
        <ImageView id="rowImage" image="{photo.urls.square_75}" />
        <Label id="rowFirstName" text="{first_name}" />
        <Label id="rowLastName" text="{last_name}" />
    </TableViewRow>
</TableView>

  • Accepted Answer

    According to the error Alloy.Models.photo is not defined, which you are essentially referencing when you bind to photo.urls.square_75. Alloy binding does not currently support complex objects, but if you log a feature request in the Titanium Community Jira project I'll see that it gets in the schedule because Alloy's behavior in this case is confusing and the error message is not terribly informative. Be sure to label the ticket with "alloy" and to add a link to this Q&A question in the ticket.

    • Ok, the hack for this ended up being simple…

      In the backbone sync I looped through the collection and moved the URL from the photo object to a new attribute… Works fine now :)

      doc.square_75 = doc.photo.urls.square_75;

      now…

      <ImageView id="rowImage" image="{square_75}" />

      Unless someone has a better idea I'm going with this until it's fixed in Alloy 1.4.0.
