Fullscreen view at wrong position when setting it visible

Hi,

I have a window with a hidden fullscreen view (overlay):

<Window>
<View id="hidden"/>
</Window>

TSS:

#hidden:{top:0,left:0,right:0;bottom:0;width:Ti.UI.FILL, height:Ti.UI.FILL,backgroundColor:"#000",opacity:0.5;visible:false}

In my app I want to switch the view to visible from time to time:
$.hidden.visible=true;

Sometimes it happens - not all the time, so very hard to debug - that the view is not placed with top left corner at 0,0 but with the center. So I only see the bottom right corner of the fullscreen view. Even when I put a activityindicator inside the view it will be shifted to the top left corner.

I would say that 1 out of 10 times the bug appears.

  • System: iOS 7, simluator and device
  • Ti 3.2.3

    Hi

    This "might" sounds stupid, but can you use show & hide and not visible.

    It is possible that this is a bug I have seen before where for a very strange reason using visible affects the size or position of a view if the state of the view(s) is repeated.
