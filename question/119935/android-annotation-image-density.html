Android annotation image density

It appears as though a map annotation image does not take into account a device's density. I'm working with 1.7.0.RC1 and developing on a Nexus S. I'm using custom annotation images with them set in the appropriate res directories, but the images appear very small. This seems to be the only area I'm having problems with image density. I use tab icons and those load just fine. I hope I've been clear enough.

Is this a bug in 1.7.0.RC1?

  • I'm having the same problem. I went from 1.6.2 to 1.7.0 and the images in both the tabs and map annotations are micro sized. I am testing on a Samsung Infuse 4G. When I revert back to 1.6.2 all the tab and annotation images are correctly sized for the display.

    Looks like this issue made from the 1.7.0.RC1 to the final 1.7.0 release.

  • The problem exists in 1.6 even. I believe the reason it becomes apparent is in 1.7 the manifest defaults with <support-screens /> settings whereas 1.6 does not. If you set <support-screens /> and it's arguments within the tiapp.xml in the android manifest section then Titanium will use those settings.

    What I did as a temporary workaround was within code check the density setting (high, medium, low) and return the path to the image icon that is appropriate for the density.
