XML evaluate gives an NodeList of x items where there is only 1 item ( <space><minus> combination appears to be a problem)

Hello,

I got this XML reader to load data from the web. Now i've got the following problem, i do an XPath expression on <row>, this row has the following data:

<row>
   <product_id>6</product_id>
   <store_id>1</store_id>
   <product_name>Some name</product_name>
   <description>-Anti-fit jeans -loltext -moreText -And even more text</description>
</row>

So i use my xPath expression on this row to get all the data, and all works fine, but for one thing.
The xPath expression i use to get the description is the following:

var description = rows.item(i).evaluate('description/node()').item(0).nodeValue;

This same technique works on all other data i need to read, but on this one it doesnt seem to work like i want it.

Because what is in the nodeValue is only: '-Anti-fit jeans', nothing more, all other tekst parts are gone. I already figured out that it splits in another item for each ' -' (<space><minus>) combination the string has. I dont know why this happens, but i dont want it to happen, because i just want to have the full string, and not have to combine all items manually. Is there a way to do get the content of an xml element as it is, not splitted?

If not, would the solution be the following line of code:

var description = CombineList(rows.item(i).evaluate('description/node()'));

function CombineList(NodeList)
{
    var content = '';
    for(var i = 0; i < NodeList.length; i++)
    {
        content += NodeList.item(i).nodeValue + ' ';
    }
    return content;
}

I'd like to hear your opinions.

Sincerely,

Hylke Bron
