Search Bar value gets cleared on blur

I have a search bar attached to a table view. See code below. When I type a search term and select one of the results, which opens a child window, the search term and result set get cleared.

So when I return to the search window from the child window the search bar is empty and the full set of rows is displayed in the table. I would prefer and expect that the window is the way I left it - search term still in the search bar and only my search results displayed so I can pick a different one.

Is this behavior by design? It is not what I find in other iPhone apps.
In any case, how would I make it stick?

var searchBar = Ti.UI.createSearchBar({
                    hintText:'Search'
                });

var tableData = buildTableData();

var tableView = Ti.UI.createTableView({
                    searchHidden:false,
                    scrollable:true,
                    filterAttribute: 'search_text'
                });

tableView.setData(tableData.rows);
tableView.index = tableData.index;

win.add(tableView);
tableView.search=searchBar;
