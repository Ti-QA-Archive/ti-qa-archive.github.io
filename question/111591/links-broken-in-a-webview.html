I've found a bug with the webview module for titanium mobile. If a website loaded through the url property contains an html link such as

&lt;a href=&quot;test.html&quot;&gt;Test Link&lt;&#x2F;a&gt;

nothing happens when this link is clicked. If the link is changed to contain an absolute url, it works just fine:

&lt;a href=&quot;&#x2F;test.html&quot;&gt;Test Link&lt;&#x2F;a&gt;

OR

&lt;a href=&quot;http:&#x2F;&#x2F;www.somewebsite.com&#x2F;test.html&quot;&gt;Test Link&lt;&#x2F;a&gt;

If I load any of these through the html property it clicks just fine.

    You shouldn't really answer your own questions - either update the original question, post a comment beneath the original question, or post a comment response to another user's answer, if there is one. Thanks.

    It would be better if your page was valid. Try this instead:

    <!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Transitional//EN" "http://www.w3.org/TR/xhtml1/DTD/xhtml1-transitional.dtd">
<html xmlns="http://www.w3.org/1999/xhtml" xml:lang="en-GB">
<head>
<title>Link Test Page</title>
</head>
<body>
<h3><a href="page1.html">Page 1</a> link has an href value of "page1.html". </h3>
When Page 1 is pressed it will not work. If you press page 2 then come back, the link starts working 
<br />
<h3><a href="/titanium/page2.html">Page 2</a> link has an href value of "/titanium/page2.html"</h3>
</body>
</html>

    You can use the W3V Validator to verify your page when its live.

    Cheers
