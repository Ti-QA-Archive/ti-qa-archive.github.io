Titanium Facebook Module -- Possible to change the user?

I can successfully log in and out of the FB module, however, if I logout and then try to login again, it simply tries to re-link to the account I originally entered, it does not ask me to enter a username, so in essence, I cant change which account I want to use.

I was thinking if there is a way to clear all fb related data from my app, that would do the trick, but cant find that.

Any ideas?

  • It's a Facebook issue - not an issue with the module or your app. If an app uses SSO the Facebook SDK will use the credentials stored on the phone - i.e. the same user. if you use the web dialog login then you can switch users if the previous user did a Facebook logout in the browser - but I don't think you can control this. This is an issue for all apps. Facebook has an example of an app with multiple users - but even their own solution is problematic: Facebook switch user example

    • This answer is not wholly correct. If you stay connected on device to Facebook as the same user then I 100% agree.

      If you use the fbobject.logout() facility, the credentials in our app (and therefore module) should be forgotten and then renewed on the next login. This is not happening correctly. Try testing an FB connect on the simulator. Since SSO is technically not allowed, it will take you through the login form. If you issue a logout command and then login, you will not be asked for your details again. The same thing happened in the older version of the Facebook object Link to Q&A.
