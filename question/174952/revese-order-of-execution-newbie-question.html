Revese order of execution, newbie question

When i setup this code:

label.addEventListener('click', function(e) {

    alert(Ti.Filesystem.isExternalStoragePresent()); // true
    alert(Ti.Platform.name); // android

});

…i get first an alert with android and then an alert with true.
Is there an explanation for?

    Hi

    Yes - Android handles alerts differently than iOS and what would appear to be logic. The solution is to not stack one alert after the other. A better solution is to not use the short hand alert and instead use the alertDialog.

    • Ti.UI.AlertDialog

    Doing this means you can provide callbacks for any button presses- this way the next block of code will not run until the callback calls it. Plus you can give the alert a better title than 'Alert'.
