"an application restart is required" bug or only non-store app? - Revisited...

I don't know why the recent previous thread/question was locked/closed. I'm seeing the same restart thing. Developing with Titanium 3 with all the latest updates and on newer Android OSes like 4.1/4.2 I'm getting constant reports of people needing to "restart". App is installed via Google Play. I have tried all the various:

<property name="ti.android.bug2373.disableDetection" type="bool">true</property>
<property name="ti.android.bug2373.finishfalseroot" type="bool">true</property>
<property name="ti.android.bug2373.restartDelay" type="int">500</property>
<property name="ti.android.bug2373.finishDelay" type="int">0</property>
<property name="ti.android.bug2373.skipAlert" type="bool">true</property>
<property name="ti.android.bug2373.message">Initializing</property>
<property name="ti.android.bug2373.title">Restart Required</property>
<property name="ti.android.bug2373.buttonText">Continue</property>

And just:

<property name="ti.android.bug2373.finishfalseroot" type="bool">true</property>

Fixes to no avail. That multi-line paste is direct from my tiapp.xml, which is in my active APK on the store as I type this. I am NOT seeing this bug on older devices using 3.x and 2.x Android OSes. ???

Anyway… any tips for newer devices would certainly be appreciated. Thanks.

  • Hi

    This is a guess only as I have not tried it myself. Given the problem is 4.x only, it might be worth setting the minimum API level in the tiapp.xml to something high like; 14. This WILL mean the app will ONLY work on 4.0 and higher. But for the moment this is a ruling out test.

    If it does work then you may need to take advantage of the multiple APK solution Android now employs.

    Google has tried to solve the problem of having all assets and all features in one app - causing blotting and feature limitations when trying to have apps work 2.x and up - by allowing you to provide multiple APKs per app. When the user downloads your app, Market Place will provide the most suitable APK from however many you supplied.

    So you can target tablets in one APK variation full of high res images, but for smaller handhelds an APK that only has small and medium images, this way those smaller devices do not carry the larger images with bigger file size footprints it will never use.

    Just a thought.

  • Hani I owe you a beer! Putting the 'finishfalseroot' under the main 

    <ti:app xmlns:ti="http://ti.appcelerator.org">

    tag and NOT under

    <android xmlns:android="http://schemas.android.com/apk/res/android">

    tag in tiapp.xml worked!! No more restart bug on Droid 4.1+! This information should be part of every time this question comes up as it's not said anywhere else that I've found and noobs like me would never figure that out since it's a droid only issue so would make sense to put under the android tag. Thanks!!!
