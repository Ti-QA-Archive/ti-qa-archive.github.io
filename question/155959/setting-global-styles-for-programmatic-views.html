Setting global styles for programmatic views

Are styles in app.tss supposed to be for all elements, not just ones defined in xml?

My problem is that I'm dynamically generating a view, but the style that I'm setting in app.tss isn't being applied:

row.badge = Ti.UI.createLabel({
      className: 'row-badge'
    });

".row-badge": {
  "width": 35,
  "right": 15,
  "color": "white",
  "backgroundColor": "#8591AA",
  "shadowColor": "#8591AA",
  "textAlign": Ti.UI.TEXT_ALIGNMENT_CENTER,
  "verticalAlign": Ti.UI.TEXT_VERTICAL_ALIGNMENT_CENTER,
  "borderRadius": 10,
  "font": {
    "fontSize": 16,
    "fontWeight": "bold"
  },
  "shadowOffset": {
    "x": 5,
    "y": 5
 }
},

I can set the style in the code and it applies just fine, but not from app.tss.

  • Components you create in code (using the createFoo methods like you showed in your example) will need to be styled in code. Instead, what you want to do is create a view/controller/style set of files then instantiate that in code.

    1. Right-click on any of the views, styles, or controllers folders and choose New, Controller
    2. Enter a name for your new controller (e.g. myCustomLabel) without a file extension then click OK. That will create a new XML, TSS, and JS set of files for you
    3. Put your label code (or whatever) into the XML and style it with the TSS file. This view will also inherit from the app.tss file.
    4. Instantiate this custom view where you want to use it with code like this:
    var myCoolLabel = Alloy.createController('myCustomLabel');
$.win.add(myCoolLabel.getView());

    See this section of the guides for info on how you'd pass arguments to that controller.
