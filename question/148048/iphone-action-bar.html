    Hi

    Why are you wanting to create an ActionBar for an iOS app? Action Bars are designed to provide a navigation solution to Android whereas iOS has the Navigation Bar which handles all of that for you.

    I know people try to make an Android app act like iOS, but it less common to try and make the iOS app more like Android.

    A good app uses the standard UI for that platform, unless you have created something so great it needs to be shared as a new way. Using something so inherently Android for iOS is counter productive.

    • You have a scope problem.

      You create the dialog variable inside the click event but try to add the event listener outside the click event.

      As the click event will not have run before you try to assign the eventlistener to it, it see it as undefined.

      THis is what I mean;

      // dialog = undefined
var dialog ;
// dialog = undefined
$.moreIcon.addEventListener('click', function(e) {
// dialog = undefined 
    var opts = {
        cancel : 2,
        options : ['Confirm', 'Help', 'Cancel'],
        selectedIndex : 2,
        destructive : 0,
        title : 'Delete File?'
    };
 // dialog = undefined
 dialog = Ti.UI.createOptionDialog(opts);
// dialog = defined dialog.show();

});
// dialog = undefined
dialog.addEventListener('click', function(e) {
    Ti.API.info(JSON.stringify(e));
});
// dialog = undefined

      You would need to change the code to this;

      $.moreIcon.addEventListener('click', function(e) {

    var opts = {
        cancel : 2,
        options : ['Confirm', 'Help', 'Cancel'],
        selectedIndex : 2,
        destructive : 0,
        title : 'Delete File?'
    };

    var dialog = Ti.UI.createOptionDialog(opts);
    dialog.addEventListener('click', function(e) {
        Ti.API.info(JSON.stringify(e));
    });
    dialog.show();
});
