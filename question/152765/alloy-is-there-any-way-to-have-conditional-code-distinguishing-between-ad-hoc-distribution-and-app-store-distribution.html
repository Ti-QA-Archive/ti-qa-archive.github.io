Alloy: Is there any way to have conditional code distinguishing between ad-hoc distribution and app store distribution?

Hi,

as far as I understand it, the alloy documentation states that you can use ENV_PRODUCTION to determine if the app was build for distribution - no matter if ad hoc or app store. Correct?

("ENV_PRODUCTION : true if the current compiler target is built for production (running after a packaged installation)"; http://docs.appcelerator.com/titanium/latest/#!/guide/Alloy_Controllers-section-34636384_AlloyControllers-ConditionalCode)

Is there any way to distinguish between ad-hoc distribution and app store distribution? The reason is that I would like to use other analytics methods for ad-hoc users than for app store customers…
Thank you!

  • Hello,

    [In classic Titanium], there is no difference between adhoc or app store distribution in the optimization level, so, usually are the same (except maybe for the type of certficates).

    I would recommend you create your own variables set to true if you want to compile for ad hoc.

  • I talked with the lead CLI engineer and this can be determined by the underlying CLI but is not currently exposed in Alloy like say the other environment compiler directives like:

    • ENV_DEV
    • ENV_TEST
    • ENV_PROD

    If you would however like there to be something like:

    • DIST_IOS_ADHOC
    • DIST_IOS_APPSTORE

    you could log a Titanium Community Jira Project ticket as a feature request for Alloy. Please be sure to include all these details and label it with "alloy". From there I can work it into the Alloy release schedule.
