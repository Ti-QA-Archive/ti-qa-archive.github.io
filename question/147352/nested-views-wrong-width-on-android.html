Nested views: wrong width on Android

I have the following code with nested views inside a scrollview. But when I try to access the width of the view_final I'll receive wrong numbers. 

var win = Ti.UI.createWindow({
    backgroundImage : "images/background_leather.png", backgroundRepeat : true, fullscreen : true, navBarHidden : true, backgroundColor:"#f00"
});

var view_content = Ti.UI.createView({
    top : 48, bottom : 0, left : 0, right : 0,backgroundColor:"#ff0"
})

var view_card_scrollview = Ti.UI.createScrollView({
    top : 85, bottom : 0, left : 0, right : 0, scrollType : "vertical", contentHeight : 0, contentWidth : Ti.Platform.displayCaps.platformWidth,backgroundColor:"#f0f"
});

var view_final = Ti.UI.createView({
    top : 130, bottom : 0, left : 10, right : 10,backgroundColor:"#fff"
});


win.add(view_content);
view_content.add(view_card_scrollview);
view_card_scrollview.add(view_final);
win.open();

Ti.API.info("WIDTH: " + view_final.toImage().width);

output is:

WIDTH: 100 480

width should be 460. iOS displays the right width

Android 4, TI SDK 3.0.0 GA, iOS 6

    Hi

    I have tweaked your code, adding extra properties to make the code more reliable for different resolutions. I have also given you several ways to get the width answer you require.

    The event listener postlayout fires when the layout of a view has changed. This fires each time so you will get this event occurring several times.

    The answer using your method (which is no longer recommended) can be correctly found if there is a small delay. This is mostly to do with the different ways in which iOS and Android allow this information to be gained. On Android you cannot determine the dimensions of a view until it has been added to the UI and only then when the OS has actually completed that task - which may not be immediately after you coded the addition.

    Here is my version of you code and you will see the value 460 appears in each of my tests.

    var win = Ti.UI.createWindow({
    backgroundColor: "red",
    backgroundImage: "images/background_leather.png",
    backgroundRepeat: true,
    fullscreen: true,
    height: Ti.UI.FILL,
    navBarHidden: true,
    width: Ti.UI.FILL
});

var view_content = Ti.UI.createView({
    backgroundColor: 'yellow',
    bottom: 0,
    left: 0,
    right: 0,
    top: 48,
    width: Ti.UI.FILL
})

var view_card_scrollview = Ti.UI.createScrollView({
    backgroundColor: '#f0f',
    bottom: 0,
    contentHeight: Ti.UI.SIZE,
    contentWidth: Ti.UI.SIZE,
    left: 0,
    right: 0,
    scrollType: "vertical",
    top: 85,
    width: Ti.UI.FILL
});

var view_final = Ti.UI.createView({
    backgroundColor: '#fff',
    bottom: 0,
    left: 10,
    right: 10,
    top: 130,
    width: Ti.UI.FILL
});

win.add(view_content);
view_content.add(view_card_scrollview);
view_card_scrollview.add(view_final);
win.open();

Ti.API.info("width 1: " + view_final.toImage().width);

view_final.addEventListener('postlayout', function (e) {
    Ti.API.info('width 2: ' + view_final.getRect().width);
    Ti.API.info('width 3: ' + view_final.getSize().width);
});

setTimeout(function () {
    Ti.API.info("width 4: " + view_final.toImage().width);
    Ti.API.info('width 5: ' + view_final.getRect().width);
    Ti.API.info('width 6: ' + view_final.getSize().width);
}, 3000);

    • In answer to your "placing views inside the view_final" set the property layout to vertical, you may find you do not need the height or the width.

      var view_final = Ti.UI.createView({
    backgroundColor: '#fff',
    bottom: 0,
    layout: 'horizontal',
    left: 10,
    right: 10,
    top: 130,
    width: Ti.UI.FILL
});

      Then add your child views and images to that and use the top, bottom, left & right as if they were margins against the previous views. Play with it if you have it used this before.

      And if you want the child views to be centred against the parent view;

      var view_final = Ti.UI.createView({
    backgroundColor: '#fff',
    bottom: 0,
    layout: 'horizontal',
    top: 130,
    width: Ti.UI.SIZE
});

      Then view_final will size to the children and centre to the parent. All without a width or height required in the calculation.
