ItemTemplate and device performance

Calling Appcelerator gurus!!

I already tested the new listView performance regarding memory consumption with Instruments.
The row UI I used had a view with bordeRadios and 3 labels and an array of 5500 record.
I tried on both ListView and TableView.

The improvement was tremendous, same row UI configuration was taking 55Mb using a TableView and only 4.5Mb with the ListView, I was amazed with difference .. so that means the ListView is the way, right?

Now I'm developing an app that the itemTemplate has to be quite complex.
We are talking about 9 labels and 2 views with opacity.
The listView that will use this itemTemplate will no have more than 31 rows and my question is:

  • Is this itemRender too "heavy" for Android or iOS?
