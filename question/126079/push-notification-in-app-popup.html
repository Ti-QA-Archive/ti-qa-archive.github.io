Push Notification In App Popup

Hi All,

I'm building a great app ;) with push notification. Sending an id with that push message to open the specific detail window.

So when the app is closed, the push is received and when the user launches the app, it opens the specific id. Without any questions in the app itself.

But when the app is already running, and i'm sending a push with a specific id.
It opens the detail window without any question.

How could i see in the callback of the registerForPushNotifications that the app is already running so that I can ask the user if he want to open it.

Thank you in advance.

Louis

    Check out this solution here for keeping track of the state of your app. It should be able to help you determine whether or not your app is in the foreground: http://developer.appcelerator.com/question/123753/relave-app-status-foreground-or-background.

    Simple, untested example:

    var isInForeground = true;

Ti.App.addEventListener('pause', function(){
    isInForeground = false;
});

Ti.App.addEventListener('resume', function(){
    isInForeground = true;
});
