XHR POST request with redirect issue

I'm currently "porting" my project for iPhone from PhoneGap to Titanium (2.0.1) because I want to use some native UI features. So far I'd just created new project, added my HTML from PhoneGap project, created WebView and loaded local HTML into it.
And here I have this issue that is quite annoying - in my app user have to log into service using login and password. They are sent to the server using POST request. If everything is OK, server sets cookie and redirects client to page with data. Since it's WebView stuff, I used jQuery there, and in PhoneGap it worked ok. In Titanium for some reason it does not works, it just doesn't get's the cookies.
I tried changing jQuery.post to custom function created with Ti.Network.HTTPClient, but the client is as well if set to follow redirect does not handles the request correctly and misses the cookies.
The only way I'd found to work now is to create HTTPClient that does not follows redirects, save cookie and then manually follow redirect. But this is just totally stupid.
May be I missed something in the docs (or it's undocumented) and there's some flag that fixes cookie handling for WebView?
Thanks in advance.
