  • For those looking for the similar module, here's what I currently use - compile ACRA together with the app

    • Download ACRA jar file from http://code.google.com/p/acra/
    • Copy it to C:\Users\<username>\AppData\Roaming\Titanium\mobilesdk\win32\<version>\android\lib\
    • Edit App.java template C:\Users\<username>\AppData\Roaming\Titanium\mobilesdk\win32\<version>\android\templates\App.java
      ~~~
      // import module
      import org.acra.;
      import org.acra.annotation.      ;
      import static org.acra.ReportField.*;

    // add to class definition (you may need to read ACRA documentation
    // on how to get the <formkey>
    @ReportsCrashes(formKey="<formkey>")
    public final class ${config['classname']}Application extends TiApplication
    {

    // first line in onCreate() method
    ACRA.init(this);

    
- Add ACRA jar to build script C:\Users\<username>\AppData\Roaming\Titanium\mobilesdk\win32\<version>\android\builder.py

    // line 1213
    classpath = os.pathsep.join([classpath, os.path.join(self.support_dir, 'lib', 'acra-4.2.3.jar')])

    // line 1805
    dex_args.append(os.path.join(self.support_dir, 'lib', 'acra-4.2.3.jar'))
    ~~~

    • Follow the ACRA documentation to upload the template csv file to GoogleDoc, then full build the app. When the app crash you may see ACRA will send bug report by monitoring the LogCat output

  • There is a JIRA Module published by Appcelerator for $19.99 a month. Allows you to have users submit directly to your JIRA install.

    Atlassian Jira starts at around $10 for 10 users (logged in users, anon can submit but not count towards towards 10). But is free for non-profits and OS projects.
