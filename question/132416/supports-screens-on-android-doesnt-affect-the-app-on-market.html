supports-screens on Android doesnt affect the app on market

Hi,

I'm trying to limit to # of devices that support my Android app. Here's my tiapp.xml code:

 <android xmlns:android="http://schemas.android.com/apk/res/android">
        <tool-api-level>9</tool-api-level>
        <manifest>
            <supports-screens
            android:anyDensity="true"
            android:largeScreens="false"
            android:normalScreens="true"
            android:smallScreens="false"
            android:xlargeScreens="false"
            />
        </manifest>
</android>

The Manifest file is OK as well:

<supports-screens android:anyDensity="true" android:largeScreens="false" android:normalScreens="true" android:smallScreens="false" android:xlargeScreens="false"/>

As you can see, it's set to only run on 'normal' sized screens, but after I upload it to the market, I get:
Screen layouts: NORMAL LARGE XLARGE

Any idea what am I doing wrong?

Thanks,
Idan

  • OK I was able to isolate the problem but still not solve it. What happens is that once a screen size is set to 'true', all the larger screen sizes are also enabled, even if I set them to 'false'.
    Any idea how to work around it?

  • OK I figured it out:

    <supports-screens> specifies the minimum supported screen.

    if you want to support only specific screen sizes, use <compatible-screens> instead
