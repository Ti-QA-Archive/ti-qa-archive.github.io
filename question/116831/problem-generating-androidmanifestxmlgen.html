Problem generating AndroidManifest.xml.gen

When generating an Android app either on emulator, device or distribute the compiler doesn´t write correct code to AndroidManifest.xml.gen. Especially the header is causing big problems to me as it will not include correct encoding (UTF-8).

I tried several ways with 1.4. 1.5.x 1.6.x, always deleting all files in build/android/:

1.) Include an AndroidManifest.custom.xml in the /build/android folder generates an identical AndroidManifest.xml but different AndroidManifest.xml.gen.

2.) Include an AndroidManifest.xml in an additional /platform/android/ folder as suggested by the compiler generates an identical AndroidManifest.xml but different AndroidManifest.xml.gen.

Example:
AndroidManifest.custom or in platform/android/ folder:

<?xml version="1.0" encoding="utf-8"?>
<manifest xmlns:android="http://schemas.android.com/apk/res/android" …

generates AndroidManifest.xml.gen as following:

<?xml version="1.0" ?><manifest android:versionCode="1" …

Main problem is the missing encoding="utf-8", as my app is in german and all Umlaute are destroyed.
