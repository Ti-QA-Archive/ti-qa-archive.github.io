animation callback

titanium sdk: 2.1.1

os: mac os

platform: android

i have a imageview with its image loaded from the remote. and i want to fade in the image when the loading is finished, so i use opacity and animate to achieve my requirement, following is my code:

var imageView = Ti.UI.createImageView({
    opacity:0
});

...

//when the image is loaded
var a = Ti.UI.createAnimation({
                        opacity:1,
                        duration:2000
                    });
imageView.animate(a,function(){
    // question: is this necessary, because after the animation, the opacity is aready 1
       imageView.opacity = 1;
     Ti.API.info('image delegate call back works!');
});

considering the code, once the image is loaded, the image fades in, so it works. But after some operation, i.e. scroll the tableview to make this imageView hide, then scroll it back, or navigate to another view, then navigate back, the image disappears. so i guess, the opacity property turns back to 0 when i scroll the tableview or switch to another view. That's why i add imageView.opacity=1 in the animation callback. am i right?

the second thing is that the callback is never called, because i never get the logs in the ddms.

  • With animation object you should use the eventlistener. Try this instead:

    a.addEventListener('complete',function(e){
 //your code for callback function here.
});
