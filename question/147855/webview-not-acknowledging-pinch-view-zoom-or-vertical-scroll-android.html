Webview not acknowledging pinch-view zoom or vertical scroll (Android)

I have a window that displays a local html file. The only thing this html file displays is a google map. It also has functions to map my objects onto this map The reason I use a local html and webview instead of the native android maps is because they are too limited for what I need (custom markers, colors, tables, etc.).
Anyhoo I can get everything to map properly and the map displays fine, but I cannot scroll up or down or pinch-zoom which I NEED to have or else its essentially useless. It worked perfectly on the iphone version, which simply placed the webview within my main view (a scrollView).
Below is the code I have set in the Android version. It places the webview in a scrollView which is added to the window. I have tried playing with the touchEnabled, enableZoomControls, and scalesToFitPage, but to no avail.

    //create webview 
    mapScrollView = Titanium.UI.createScrollView({
    contentWidth: 'auto',
    contentHeight: 'auto',
    top: 0,
    bottom: 0,
    showVerticalScrollIndicator: true,
    showHorizontalScrollIndicator: true,
    maxZoomScale: 100,
    minZoomScale: 0.1,
    zoomScale: 1
}); 

    //create webview 
    wv = Ti.UI.createWebView({
        url: htmlFile,
        top:50,
        enableZoomControls:true,
        touchEnabled: true
    });

     // add web view to the main view
     mapping_win.remove(mappingFormView);

    mapScrollView.add(wv);
    mapping_win.add(mapScrollView);

    Hi

    Why are you adding the webView to a ScrollView? Have you tried without the scrollView?

    As an experiment please add the webView directly to the parent;

    //mapScrollView.add(wv);
//mapping_win.add(mapScrollView);
mapping_win.add(wv);

    Does this work any better?

    The scrollView will add a layer of interaction complexity that should not be required.
