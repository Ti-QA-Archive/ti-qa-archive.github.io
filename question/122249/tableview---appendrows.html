TableView - appendRows

Hi all.

Wouldn't be nice if there was "appendRows" function in "TableView" component that adds multiple rows at once?

I see that there is growing number of developers that are working with large set of data and that have problems with "TableView" performance because of that. "appendRows" would be great option for lazy row data loading and better user experience (if it can be implemented on Android/iOS). If you like the idea, please vote this question. :)

— asked July 8th 2011 by Ivan Škugor

  • @Paul - I have 500 rows that are loaded from database and it takes about 6-7 seconds. They're not just basic rows, there are several items on each row. Much more rows will come in the future. :) As for lazy row data loading … that means, to load rows 100 by 100, for example. Using "setData" (Micah!) you'll have to load and set all data at once ("setData" doesn't append rows to the table). Using "appendRow" is slow when you need to add 100 more rows. There is one solution that is OK (when user scrolls to the end of the table, more rows are loaded), but that approach also suffers from "appendRow". "appendRows" would solve all that problems and usage would be much more simpler.

    "Currently you can load in blocks/chunks with a custom function." - can you explain this in more detail?
