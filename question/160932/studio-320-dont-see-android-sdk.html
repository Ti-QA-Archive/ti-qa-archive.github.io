  • I had this problem and tried everything above. nothing worked. then got a clue from JIRA bug TIMOB-15504.

    It appears the installation either caused a bad AVD file (Android Virtual Device), or the new version simply can't handle a bad one. Open the android Virtual Device Manager. If there is a bad config there delete it (that's what worked for me).

    It appears from the defect that you may need at least one defined. So try removing them all and recreating them. You may want to make a backup first. The AVD configuration files on LINUX are ./android/avd

  • It's working for me now. This is what I found:

    I had a /opt folder with only Imagemagick installed in it but it was owned by root and had permissions: 700

    When I did 

    sudo chmod -R 755 /opt

    My SDK was detected (after restart Studio). And no, I didn't install the Android SDK in /opt but for some reasons it searches there? I don't think I had it in my PATH.

    BTW this is on osx Mavericks
