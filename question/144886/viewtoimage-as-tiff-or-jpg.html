View.toImage() as .tiff or .jpg

Is it possible to get another format from toImage() ?

var blob = myView.toImage();
Ti.API.error( blob.mimeType ); //Only works on iOS, on Android you would need blob.media.mimeType.

The output will always be image&#x2F;png.

How would one get the blob as another format or convert it to another format (e.g. .tiff or .jpeg)?

We have considered adding a conversion layer in our backend, but that would be as a last resort, as this could potentially disrupt other workflows. The best thing would be if the application could upload the image with the right encoding (that being .tiff or .jpeg).

I have tried saving the image to the device storage, with the desired file extension - which works and most image viewers parse the image just fine. The problem is that the actual binary data is PNG-encoded and the server reads the BLOB-data, thus seeing the image as .png and not e.g. .jpeg as specified in the file extension or MIME type.
