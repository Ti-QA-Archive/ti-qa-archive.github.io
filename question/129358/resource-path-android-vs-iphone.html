Resource path android vs iphone

Hello, just to understand, i'm a bit confused, i've read from the documentation that all specific resources for both android and iphone should be put inside: resources/android/images/res-xxx and resources/iphone and then i need to use only the base name while creating views. The problem here is that following this i can reach only android resources and only if i add /images to the base name like this images/myimage.png, no luck for iphone however, i have a red error message on the console telling me the resource is not found. Someone can clarify me how this works?
Thank you in advance!

  • for good practice to avoid this type of problem.. you can try this..

    //this will work on android and iphone both.
var resDir = Titanium.Filesystem.resourcesDirectory;

var view = Titanium.UI.createView({
backgroundImage:resDir+'images/someimage.png'
});
