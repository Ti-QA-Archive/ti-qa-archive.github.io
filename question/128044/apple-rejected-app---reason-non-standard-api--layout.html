Apple rejected app - reason: non standard API = layout

Just had a rejection email from Apple as follows:

"We found that your app uses one or more non-public APIs, which is not in compliance with the App Store Review Guidelines. The use of non-public APIs is not permissible because it can lead to a poor user experience should these APIs change.

We found the following non-public API/s in your app: layout"

For reference I used the following to build and submit my App
Titanium Studio, build: 1.0.6.201110251616 / SDK 1.7.5 / Xcode 4.2
Xcode reports that the Target iOS is version 5.0

When I searched the Xcode build for the term "layout" it returned 484 results in 47 files!

There were no issues reported by the Application Loader when I originally submitted the App to Apple.

Can anyone provide any insight at all?

Thanks
