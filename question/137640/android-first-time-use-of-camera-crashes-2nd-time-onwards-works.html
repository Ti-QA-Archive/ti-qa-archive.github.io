Android: first time use of camera crashes, 2nd time onwards works

When installing the app via the web (download the apk) on a fresh install, the app crashes when using the camera: the user can take a picture and click the Save button, but once the Save button is clicked the app crashes.

Once you restart the app and from then on, the camera save works fine.

The problem doesn't occur when it's installed from Eclipse to the Android device - it only shows up when installing by downloading an apk.

The crash happens before the success/cancel/error functions are called - that is, none of my Ti.API.info log messages show up.

I have a few guesses as to why - eg. perhaps the directory where the image is supposed to be saved doesn't exist - but I can't do much about it because the crash happens before I get into my code. It happens in showCamera.

Works fine on iPhone.

Any ideas?

Titanium.Media.showCamera({
        success: function(event) {
            Ti.API.info('--- Camera SUCCESS');
        },
        cancel: function() { 
            Ti.API.info('--- Camera CANCEL'); 
            window.close();
        },

        error:function(error) {
            Ti.API.info('--- Camera ERROR');
        }
);
