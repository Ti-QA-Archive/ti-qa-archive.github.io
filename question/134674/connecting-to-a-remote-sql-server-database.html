Connecting to a remote SQL Server Database

I have a remote SQL Database that I need to connect to to read a table. Is there a way of doing it without using a webservice? (ie, direct connection)

  • It's not possible to directly connect and query a remote database. The only way is to place between the mobile device and the remote database an application server (such as ColdFusion, PHP, .NET, JSP, etc) so that you can POST your requests and get through HTTPClient the result in either JSON or XML format (preferably JSON as it's smaller in size and remember.. in the mobile world size matters!) :-)

    • Just a thought but you would need to keep both your old and new server up for a while and on either app start or even on network error you make a webservice call to check the production server location. In the app you keep this stored in a database or property. If the return location from the webservice is different then the location store in the app you update. This way eventually everyone would have the new server info stored in the app. This info is what you use when you make your http connection.   • I have the exact same situation. I am using a c# web service (25+ methods) that connects to a SQL Server db. Also, my web service (and db) will be moving to a new hosting service at some point, as I'm moving to a cloud hosting provider.

    My strategy is to simply re-route my domain name to the new provider, when I make the switch. While that will result in an outage for the app, it won't require any changes or re-submission of the app.
