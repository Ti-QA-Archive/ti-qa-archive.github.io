  • In my case there was no build.log file.

    I solved the problem by installing the Command Line Tools for Xcode (or start XCode, in the menu Xcode > Open Developer Tool > More Developer Tools…).

    After restaring Titanium Studio I still had the same error. This time the problem were dashes in the project name, e.g. foo-bar. After I created my module project again only with alphanumeric and underscore characters in the name like foo_bar, it worked.

    • Also, the module project's build directory is not cleaned when you use the Project > Clean command in Titanium Studio. Deleting the build directory by hand helped me to get rid of error messages that complained about problems that I already fixed.

    • Since I don't have a build.log file to see what went wrong when the module was compiled, I run the build.xml file in the module project directory by hand: Open a terminal, cd to the module project directory where the build.xml file is located, then run ant. You will see the output in your terminal.
