    Ok, enough for comment for my beloved Titanium. Here is the solution I found:

    For those of you who don't want the details do this: ( Mac OS X or Linux, I don't know Windows command)

    1. In command line, run 'npm bin -g' (of course without ' )
    2. You should see current configuration setting for bin directory; make note of the return value, it is the path where npm program is being installed to.

    In my system this is /Applications/Titanium Studio/TitaniumStudio.app/Contents/MacOS/Password:/usr/local/bin

    Obviously, Titanium installation has changed my default npm installation prefix! And it is not in my PATH environment variable so the newly installed titanium package will never get called, but the old module which is still installed some where in the system is still intact!

    In your case this may be different, just follow on.

    1. Type 'which titanium' and note the path returned; that is the path of current titanium version you are using.

    2. If the path's value from 2) and 3) doesn't seem to match ( e.g. 2) = /usr/local/bin 3) = /some/other/dir ) I means you have two install location, which must be the case if after installing new version of titanium CLI but you still calling the old one.

    3. Run 'sudo npm config set prefix /usr/local' which will revert back the npm prefix for bin and lib/node_module (global package install location) to the default.

    4. Now install titanium again using 'sudo npm install titanium -g'

    5. This is optionally but you may want to delete whatever in and over path from 2) and related stuffs. It's old npm's installation after we have changed prefix in step 5)

    Hope this help.

  • In Studio, choose Help, Check for Updates (checks for Studio updates). Then, choose Help, Check for Titanium Updates (which checks for CLI, Alloy, and other updates).

    Or, at the terminal for just the CLI:

    sudo npm update titanium -g
ti --version // should report 3.2.0

    If that doesn't work, try uninstalling and reinstalling the CLI:

    sudo npm uninstall titanium -g
sudo npm install titanium -g
ti setup // and run the check option to confirm all is setup correctly
ti --version // should report 3.2.0

    • Jenna, so apparently you do not have titanium installed locally (in your user directory), which is good. However, I'm not sure why you cannot uninstall/reinstall/update titanium to 3.2.

      Try entering these commands (the part after // is what shows on my system):

      which node        // --> /usr/local/bin/node
which npm         // --> /usr/local/bin/npm

      Perhaps you need to remove node & npm and reinstall them. That's a bit of a sledgehammer approach. So, if someone else has other ideas hopefully they will chime in.

      These articles will help with the uninstall process:

      • SO: Uninstall and reinstall node
      • Uninstalling Node/NPM on OS X
      • SU: Can't install/remove NPM

      Then, visit http://nodejs.org/ to reinstall node (which includes npm). Then do 

      sudo npm install titanium -g

      To reinstall Titanium.
