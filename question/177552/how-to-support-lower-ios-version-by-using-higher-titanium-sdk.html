How to support lower iOS version by using higher Titanium SDK?

As of the release of TiSDK 3.4.0, iOS 6 support will be scrapped and anyone who wants their apps to support iOS 6 have to use earlier SDK such as 3.3.0.

My question is - what if I want to use SDK 3.3.0 to support iOS 6 but have bug fixes in SDK 3.4.0? No way of getting bug fixes on lower SDK??

As far as I know, Xcode allows developers to set iOS development target which designates the lowest iOS version the app is compatible with. However, this is impossible in Titanium.

  • Hi

    In short, in order to support the latest versions of iOS you will have to sacrifice the support for older versions.

    In order to use Ti SDK 3.4 you will need the latest version of Xcode, and this means only support for iOS7/8. You could run 2 versions of Xcode side by side, but you would still need to run two separate compiles to achieve what you want.

    Apple always kills off old support, in favour of the new, as things move on. In this case, the only devices that cannot use iOS7 are the iPhone 3GS and before.

    You might want to check out his link for one set of statistics in favour of ios7, to around 90% currently.

    http://david-smith.org/iosversionstats/

    When iOS8 is released this will sway even more in favour of iOS7 and 8.

    Hope this information is useful to you.

  • If you keep Xcode 5 installed, you will be able to install & run the older CLI (command-line) tools to build apps using the older SDKs.

    [sudo] npm install -g titanium@3.3.0.GA
[sudo] npm install -g alloy@1.4.1
# make sure your tiapp.xml still lists the older SDK version
ti build -p ios <other arguments here as necessary>

    As long as you're not implementing any of the newer SDK/UI components, Alloy 1.5 should also work. So, there may be no reason to install the older Alloy version.

    As Luke notes, you will need to compile separately for older & newer versions of iOS using different versions of the CLI. Also, Apple will at some point require all new app uploads to be compiled with Xcode 6. At that point, you might have to abandon older iOS versions altogether.
