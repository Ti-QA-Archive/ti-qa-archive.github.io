Consistent Cross Platform UI

I know this has been answered somewhere but in two days of searching off and on and rewatching the training videos I still cannot figure it out. Basically I want my application to look basically the same on iOS and Android at all resolutions. By the same I mean that my custom View based toolbar on the bottom in portrait mode shows six icons that are evenly spaced regardless of the resolution and platform. Thus, imagine the layout on a iPhone non-retina display the 6 evenly spaced 32x32 icons in a view that is 40dp heigh. Rotating landscape uses the same size icons but the spacing is greater of course. Now, how to translate this to the iPad or and Android tablet? Currently my design presents a tiny toolbar on the bottom of the iPad view (retina and not). I want proportionally the screen to be nearly identical on all devices. Secondly, I have tried to alter my layout in the postlayout event, however, it does not fire when the device rotates at least in the iOS simulator. Is there another event for this?

  • Hi Jason,

    This is an idea that comes to mind:

    If you put each icon themselves inside a view, you could specify the "fixed" icon width and height, but set the container view to have a percentage-based width. For example:

    Main View containing all buttons is set to width=Ti.UI.FILL and Layout horizontal.
    Then, if you have 6 buttons, each button (the View) has a width of 16% ( 100/6 )

    In theory, each icon will be centralized withing its parent view. Each view will be automatically resized, and place neatly one next to the other because of the horizontal layout.

    Hope this helps,

    R
