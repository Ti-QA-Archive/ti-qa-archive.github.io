    HI Navid,
    Looking good. :)

    I do have a couple of comments:

    • In your manifest file, change apiversion to 2. This is not the android SDK min-version, but an titanium CLI version. you only specify this in the timodule.xml, as you correctly did. This why Titanium Studio will recognize the module.

    • You specify min-sdk 10, but build with version 14. You might wanna change that as well - so they match up.

    • Any reason why you start your module at version 2.3.3? instead of 1.0

    That said, I tried to take your for a spin after recompiling it.

    • Can you tell me what did you exactly do step by step?
      This is what I did:

      1. Intellij Idea->New Project
      2. Add build.xml to ant build
      3. Added these libraries to intellij idea:
        • {PROJECT_ROOT}/lib/android-support-v4.jar
        • {PROJECT_ROOT}/lib/viewpager.jar
        • {TITANIUM_3.2.0_ROOT}/android/kroll-apt.jar
        • {TITANIUM_3.2.0_ROOT}/android/kroll-common.jar
        • {TITANIUM_3.2.0_ROOT}/android/kroll-v8.jar
        • {TITANIUM_3.2.0_ROOT}/android/titanium.jar
      4. Added Android SDK 4.0 to Project Structure ( Intellij IDEA option )
      5. Changed build.properties paths to correct paths
      6. Ran build-module.dist ant target with intellij idea

      That't it.
      Then it created com.navid.tabs-android-2.3.3.zip in dist folder then I extracted and copied to titanium module folder and use it in my program. Everything works.

      This is my build.properties:

      android.ndk=/Users/navid/android-ndk
android.platform=/Users/navid/android-sdk/platforms/android-14
google.apis=/Users/navid/android-sdk/add-ons/addon-google_apis-google-14
titanium.platform=/Users/navid/Library/Application Support/Titanium/mobilesdk/osx/3.2.0/android
