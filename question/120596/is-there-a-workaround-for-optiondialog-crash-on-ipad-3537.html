    • Hi,

      I went Through your codes and debugged them, the issue is not with the options dialog its an issue due to option dialog on a modal window.

      if i remove the modal:true the option dialog works , Now tell me under-what circumstances are you using a modal window ???

      Work Around 1:

      • Try to avoid the modal window if you ca.

      Work Around 2:

      • If your App really needs a Modal window kinda UI(the one with Nav Bar) you can fake it using
        ~~~
        // this sets the background color of the master UIView (when there are no windows/tab groups on it)
        Titanium.UI.setBackgroundColor('#000');

      // create tab group
      var tabGroup = Titanium.UI.createTabGroup();

      //
      // create base UI tab and root window
      //
      var win1 = Titanium.UI.createWindow({
      title:'Tab 1',
      backgroundColor:'#fff'
      });
      var tab1 = Titanium.UI.createTab({
      icon:'KS_nav_views.png',
      title:'Tab 1',
      window:win1
      });

      var btn = Ti.UI.createButton({
      height:50,
      width:200,
      title:'chick here'
      });

      var alertD = Titanium.UI.createOptionDialog({
      title:'Select Type',
      options: ['Choice 1', 'choice2','Cancel'],
      destructive:2
      });
      alertD.addEventListener('click', function(e) {
      Ti.API.info('Clicked '+e.index);
      });
      btn.addEventListener('click', function() {
      alertD.show();
      });
      win1.add(btn);

      //
      // add tabs
      //
      tabGroup.addTab(tab1);

      // open tab group
      tabGroup.open();
      ~~~
