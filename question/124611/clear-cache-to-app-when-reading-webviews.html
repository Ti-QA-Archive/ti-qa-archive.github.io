  • Accepted Answer

    Webviews are very expensive to create and maintain within your app. The more you webviews you create the slower you app will run. Think about using some native views where possible.

    On the data management side, open you webviews in windows that you can destroy to help free up memory and collect garbage when you close any given view.

    • Yea, this is very taxing as you are spawning web views over and over again. Try utilizing the same webview for each page that you have to load and just hide and show the window that the webview is in. That might help out with your app speed.

      If you must create more than 1 webview, make sure that you attach each view to its own window so that you can close the window that the view is attached to. When closing the window, try setting that window reference to a null or blank object to see if you can get some of that memory back.
