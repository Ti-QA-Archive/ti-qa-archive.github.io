I need to identify Cellular network off for my app to display appropriate message

Hello Experts,

Real Device: iPhone 6, OS 8.1.2, Ti: 3.4.1GA, OS X 10.9

I am testing on real device.I am validating 2 things

  1. Turn on Airport mode - If you see my code, It displays the alert message when there is no network.this is good.
  2. Disable cellular usage for my app from settings - here we have network, but user was disable the cellular usage for my app.

Even thought I have internet , it is showing showInternetDisableMessage() message.

Is my logic correct? Please suggest the solution or correct my answer..
For some of the functionality where we are pulling from the server I need to restrict the user not to open the window.



    //--- Internet Offline Message to the User
    function showOfflineMessage() {
        var alert = Titanium.UI.createAlertDialog({
            title : 'Warning!',
            message : 'Please connect to internet and try again',
            buttonNames : ['OK']
        });
        alert.show();
        return;
    }

    function showInternetDisableMessage() {
        var alert = Titanium.UI.createAlertDialog({
            title : 'Warning!',
            message : 'You have disabled the app to use cellular network, please enable to use this feature',
            buttonNames : ['OK']
        });
        alert.show();
        return;
    }

function BtnClick(e) {

        Ti.API.log('Title Value:' + e.source.title);
        if (Titanium.Network.networkType == Titanium.Network.NETWORK_NONE) {
            showOfflineMessage();
        } else if (Titanium.Network.networkType == Titanium.Network.NETWORK_MOBILE) {        
            showInternetDisableMessage();
        } else {
            alert("You have internet, you are good");
        }
    }
