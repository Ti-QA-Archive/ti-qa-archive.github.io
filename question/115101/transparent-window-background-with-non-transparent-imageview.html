Transparent window background with non-transparent imageView

Hi everybody,

I am coding a window which needs to host a imageView. The window itself has a background image and I set the windows's opacity < 1 hence the background image could be blended into the window's background color.

On top of the window I created an imageView, which the image hosted by it MUST NOT be transparent. However it seems the imageView always inherits the opacity setting from the underlying window, regardless I've explicitly set the view's opacity = 1. Now the pic from the imageView blends with the window's background image, and it looks ugly~~

Anyone please suggests if it is possible for the view to have distinct opacity value from its parent window? If the window is transparent then the views it hosts must be transparent as well?

Any help would be GREATLY appreciated.

Jimmy

    Setting a window opacity will make all it's children to have the same opacity.
    Set a normal window with a backgroundColor, add to this window a view with the backgroundImage the one you set for the window and set it's opacity to 0.5 let;s say. Add to the window you image that doesn't have opacity.

    Makes sense?
    Basically you emulate the background image of the window with another view that takes the opacity to obtain the blending you want.
