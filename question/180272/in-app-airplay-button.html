In-app AirPlay button

Hello,

Is it really true that Appcelerator doesn't support an in-app AirPlay button for streaming audio to airplay on iOS?

I know of the TvOut module, but thats only for Video. And I know that you can use the Native airplay button support when you slide the menu up from the buttom of your screen, but I want it to be a in-app button which does this.

Anyone?

Thanks!

    Hi

    Nope it is not true. There is a third-party module free module that performs the task you are after.

    To clarify;

    • Support is most commonly provided via the buttons found on system components such as the video player.
    • Those components have system provided UI's and automatically show/hide the button as it becomes aware of suitable AirPlay devices.
    • This is how most apps get around the issue of course.
    • The TVOut module does not support video, it actually allows you to define a view and assign it to the module. Whatever you then place within that 'AirPlay view' is shown on the Apple TV. Please note that the AirPlay mirroring option must have been selected before this 'AirPlay view' is shown. From there however you can do whatever you like with the contents of that view and they are updated in real-time (give or take a few frame rate buffers). I use this a lot and it works very well.

    You can download the module via;

    • gitt.io (good place to make sure modules are 64 bit)
    • github.com

    It has been recently updated to 64 bit by the looks of things and I can confirm (just checked) it compiles and works with Ti SDK 3.5 GA and performs correctly when swapping between the iOS device and AirPlay device when swapping between targets.
