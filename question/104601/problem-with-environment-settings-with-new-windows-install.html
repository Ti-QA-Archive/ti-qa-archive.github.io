Problem with Environment Settings with new Windows install

Hi all

I installed Andriod SDK Tools and Titanium Dev 1.2.2 today on a Windows box. I'm trying to import the Kitchen Sink project but when I try to enter my Android SDK Environment Settings I get the following error:

Couldn&#39;t find adb.exe or android.bat in your SDK&#39;s &quot;tools&quot; directory. You may need to install a newer version of SDK tools.

On looking, adb.exe is in the '\platform-tools\' directory, and android.bat is in the '\tools\' directory. So, everything is present, if not exactly where the error message states.

Some system/install info:

The SDK install is here:
C:\Program Files\Android\android-sdk-windows

Environment Variables path:
C:\Program Files\Java\jdk1.6.0_23\bin

Android SDK is a brand new install and up to date (i beleive) so does anyone have any advice please?

Thanks in advance
Ian

      As you are on WinXP and there is no mklink command, just copy the files in to your tools directory, as explained at the end of that section in the link I posted:
