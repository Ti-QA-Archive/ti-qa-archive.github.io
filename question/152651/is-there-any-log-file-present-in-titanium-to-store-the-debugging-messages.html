Is there any .log file present in Titanium to store the debugging messages

Is there any code to save the messages shown by Ti.API.info('message'); in a file ( say debug.log ) ?….

  • Unfortunately, due to a bug in the Titanium Command Line (CLI), the build.log file is not currently being generated like it used to. That file would be in the project root directory and would contain build information and logging output. Watch the ticket here: https://jira.appcelerator.org/browse/TIMOB-12169

    If you're building for Android, you can pipe the output of the logs to a file. Off the top of my head, something like this should work (on OS X/Linux):

    adb logcat | grep TiAPI > debug.log

    Same should work on Windows minus the | grep TiAPI portion (which limits the log output to just API logging statements)

    On the Mac building for iOS, you can use the Console utility (Applications, Utilities, Console) to view logging messages. I'm not sure if there's a save to file option or adb-like command to output the logs to a file automatically.
