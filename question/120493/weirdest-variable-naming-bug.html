weirdest variable naming bug

I think Titanium is buggy with variable names. I'm using a mobile/iPhone project on the latest Titanium and Simulator.

[INFO] Titanium SDK version: 1.6.2
[INFO] iPhone Device family: iphone
[INFO] iPhone SDK version: 4.3

This cost me 4 hours and I finally gave up:

In one window I create a table view

data = [{title:'Hello', hasChild:true, test:'./list.js', setid:1}];

var tableview = Titanium.UI.createTableView({
    data:data
});

tableview.addEventListener('click', function(e)
{
    if (e.rowData.test)
    {
        var newwin = null;
        newwin = Titanium.UI.createWindow({
                url:e.rowData.test,
                setid:e.rowData.setid
            });
        Titanium.UI.currentTab.open(newwin,{animated:true});
    }
});

I want to use setid in the next window, therefore I write there:

var win = Titanium.UI.currentWindow;
Titanium.API.info(win.setid);

But this doesnt work. 

win.setid equals to [object TiUIWindow]

However, if I rename setid to moo in data, and setid to huhu in the createWindow call, then it works.

I tried set_id, setId and all kinds of variations and combinations, but all failed. huhu and moo work splendid.

Pretty f* up if you ask me.

But why is it like this?
