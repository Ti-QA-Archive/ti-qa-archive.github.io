  • As far as I know, Titanium does not ship with a SMTP client. That means you can't send email from Titanium. You need to do it from a web service, either one that you have written yourself or one someone else offers to the public (which I imagine wouldn't be free).

    I have successfully routed emails through Gmail SMTP using PHP, so I know that works. That's how my apps send emails.

    Keep in mind that there are security risks with having a publicly-accessible web service that sends email from your servers. "To" email address should never be set by a POST variable (instead it should point to a record in a database that contains an email address), and you should do your best to authenticate calls to the web service as coming from your own apps. A good first step would be to use a unique ID from each installation of your app as an access key. This would also help you limit emails to 1 per x number of minutes.

    All that said, it sounds to me like you want a database to store stats from your app. You can't do that much with information sent via emails.
