tabBar tabGroup.activeTab broken on Android with SDK 1.5.1

[UPDATE]
Actually this has been fixed in git by dizzymonkey already:

https://github.com/appcelerator/titanium_mobile/commit/b5ac7e8103410d0b26520770c3f48b523164106a

I applied this code as a patch to git tag 1.5.1, rebuilt the sdk and now tabGroup.activeTab returns the active tab.

[/UPDATE]

If you create a new mobile project and set tab1 as active tab, tabGroup.activeTab is undefined. This works on SDK 1.4.2

This make TabBar currently unusable for most use cases.

If you modify the end of the default app like in this example:

...
//
//  add tabs
//
tabGroup.addTab(tab1);  
tabGroup.addTab(tab2);
tabGroup.setActiveTab(tab1);



// open tab group
tabGroup.open(); 

Ti.API.debug("tabGroup: " + tabGroup.activeTab)

This will result in:

[DEBUG] [4,1034] tabGroup: undefined
