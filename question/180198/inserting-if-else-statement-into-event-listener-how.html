  • Actually you don't.

    It is a slight misconception that you should build the function to execute inline of the event listener.

    In best practice You shouldn't.

    In your case you need to put an event listener on each object. you cannot string them together. To have a single function evaluating multiple view objects, simply give each view the same custom property which you can use to identify it.

    So you would have something like…

    
// view example ...

var imageView2 = Ti.UI.createImageView({
    // normal parameters.

    // Now lets add a custom one.
    imgNum: '2' // apply this to all image views giving different number,
});
imageView2.addEventListener('click', domytask);
imageView5.addEventListener('click', domytask);
function domytask(e) {
   Ti.API.info(e);  // this will show you the contents of the returned payload, which will indicate what was pressed.

  // one of those parameters would be imgNum
  // You can put whatever code here you like. but I have given a quick switch statement example

  switch (imgNum) {
    case '2': // do somthing here
        break;
    case '5' // do something here
                break;
    default: 'GO HOME'
                break
  }
}
