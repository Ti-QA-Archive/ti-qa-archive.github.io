iOS Packaging Option Missing from Launch Configurations in "Run Configurations" after 3.3.0 Update

I have just upgraded Titanium Studio from 3.2 to 3.3 and my iOS Ad Hoc/Enterprise Distribution launch configuration, along with the run configuration I had under there, has disappeared. I have checked the filters but I'm still not able to bring it back in. I have also re-installed Studio, in vain.

However, if I select "Package" on the toolbar's launch config icon (the one where you select Run/Debug/Package), then select the menu item "Configurations" from the next toolbar item's menu, the Run Configurations dialog shows all my configurations and from there I am able to create/edit/run my distribution run config.

Why am I no longer seeing my distribution configs from Run–>Run Configurations? How can I bring them back?

  • Solution using Titanium CLI

    If you had yet a provisioning profile and an iOS Distribution certificate :

    Open Terminal :

    -move into your project folder - using cd <project_folder>

    -get your provisioning profile and certificate with the command :

    ti info

    search : Ad Hoc iOS Provisioning Profiles - UUID

    => AAAAAAAA-BBBB-CCCC-DDDD-EEEEEEEEEEEE

    search : iOS Distribution Certificates

    => Firstname Lastname (AZERTYYUPO5)

    You should be able to write something like :

    ti build -p ios -T dist-adhoc -R "Firstname Lastname (AZERTYYUPO5)" -P "AAAAAAAA-BBBB-CCCC-DDDD-EEEEEEEEEEEE" -O ./dist

    A folder "dist" will be created in your project folder and a file .ipa created.

    Hope this help

    More documentation : Titanium CLI Tasks > iOS Ad Hoc Distribution
