Splashscreen : navBar & tabBar always visible on window.open() in app.js --> BUG ?

I don't know if this is a bug, but I'll try to explain my problem…

When I launch my app, in first I've got the Appcelerator image, then it executes the first page app.js. Here is my code for app.js :

Ti.UI.setBackgroundColor('#fff');

var menu = Titanium.UI.createWindow({
    url:'views/menu.js'
});

setTimeout(function() {
    menu.open({transition:Ti.UI.iPhone.AnimationStyle.FLIP_FROM_RIGHT});
}, 4000);

So, you see I ask to open the menu.js page with an animation, 4 seconds after the app is launched. Here is my code for menu.js :

var win = Ti.UI.currentWindow;

var tabGroup = Ti.UI.createTabGroup();

var win1 = Titanium.UI.createWindow({  
    title:'Gestion',
    backgroundColor:'#fff',
    url:'gestion.js'
});

var tab1 = Titanium.UI.createTab({  
    icon:"../images/icons/gestion@2x.png",
    title:'Gestion',
    window:win1
});

tabGroup.addTab(tab1);

tabGroup.open();

win.add(tabGroup);

Normally I should have a transition between my Appcelerator image and my menu.js page. I mean the flip effect should have the Appcelerator on the first side and the menu.js on the second side. But it's not working.

When I launch the app, I've got the Appcelerator logo for about 1 second (more or less). 4 seconds after the opening, I've got the menu.js page without any elements except navBar and tabBar, then my flip animation to the same menu.js with all elements (navBar, tabBar, background, buttons, tableView, etc…). Look at the image attached, it shows the transition :

How do you explain that ? My goal is to have a smooth transition between the first splashscreen and my first page (app.js or what you want). Thanks a lot !
