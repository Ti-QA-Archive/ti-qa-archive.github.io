    Hi

    This might be a simple scope situation. You are using a callback and you know it works as tests to win work (good test btw).

    However as the callback will likely not return the data about the image until after the current row is complete, the loop will have moved on to the the next row, the one after that or even after all have been completed. The variable row no longer links to the version inside the loop that you think it does.

    The likely solution is simple and a technique I use all the time if only for the ability to separate the code to create a row into a function.

    I am not at my dev machine so I cannot give you the full code back but this will explain what to do.

    function addRow(data) {
    var row = Ti.UI.createTableViewRow({
        height:'80dp',
        // backgroundImage    : 'images/tablebgl.png',
        backgroundColor: 'transparent',
        separatorColor:'transparent',
    });
      var  name = Ti.UI.createLabel({
            text: data.from.name,
            font:{
                fontSize:'17dp',
            fontWeight:'bold'
        },
        height:'20dp',
        left:'90dp',
        top:'9dp',
        color:'#eee',
        touchEnabled:true
        });
        row.add(name);
   // the rest of your code
   // including the callback
   return row;
}

    Then inside your loop

    var row;
for (i = 0; i < json.data.length; i++) {
    row = addRow(json.data[i]);
    tableData.push(row);
}

// rest of your code
table.setData(tableData);

    This should work by giving the call back a unique reference to come back to.

    This should give you enough to tweak your code to try this out, I have tried to keep the variable references the same so very little if anything needs tweaking.
