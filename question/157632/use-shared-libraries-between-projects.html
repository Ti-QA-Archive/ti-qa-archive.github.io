Use shared libraries between projects

Hello,

I'm currently developing several apps, all of them use the same basic structure and what changes is the content, styling, etc. For this I have been able to successfully achieve the creation of the base framework (which is a collection of javascript files) which read from custom variables to use the customization, this way I'm able to use the same files on every app making maintenance and support easy.

Now I'm thinking of a way to make things even easier, as it stands right now whenever I do a change on a file of the base library I need to update this file on every project, this has the inherent problem of human error, as I might forget to update a specific file, and makes version tracking a pain. So I'm looking for a way to actually put the libraries on a central place from which all projects can feed, but I'm not sure if that's even possible since it means that I would need to include .js files that are outside the actual Resources directory from the project.

Basically my question boils down to this: is it possible to Ti.include files outside the Resources directory?

If that's not possible I would like to hear any suggestion or ways any one with a similar situation has been able to do a workaround.

Thanks

  • Hi,

    Christian is right: short answer is NO.

    Long answer is that if you're using Alloy, you could write a JMK script that moves the common libraries to the project folder just before compilation. This could give you some ideas: https://gist.github.com/ricardoalcocer/6115571. If you're using Titanium Classic, then you'd have to create your own script, maybe using Node.js, that moves the files to the appropriate location and then call the build using the CLI.

    Hope this helps,

    R
