Android emulator, modules and titanium sdk 2.0.1

we had been using various modules(mostly 3rd party modules) in application, but now when we switched to titanium sdk 2.0.1 and tried run it on android emulator, it asks for new version of those modules with the 'apiversion' value 2 or greater in its manifest file.(might be due to this need some change in native code for that module).

My question is, Is there any way to use a simple way to make them work again?

I had tried this link but could not proceed further than 'ant' command, it says 'build.xml not found'.

  • Hi Nitin,

    apiversion is present in the manifest file of a module on path <Titanium App Directory>/modules/<Module Directory><Module version directory>. You can change its value to 2 by editing this file.

    I am not sure but this seems to describe what api version the module is using w.r.t titanium sdk. Also, its not sure whether the module will work after the change to its manifest file so do it at your own risk and i hope this helps.
