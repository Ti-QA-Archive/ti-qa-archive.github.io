TabGroup inside a NavigationWindow lacks title

SDK 3.3.0, iOS 7. I have an iOS app based on a NavigationWindow. One of the subwindows needs tabs, so I added a TabGroup inside the NavigationWindow. This works but I have still two issues with the TabGroup:

  1. The title of the tab windows is not visible.
  2. I can't change the text of the back button (currently the title of the previous window)

This is the index.xml:

<Alloy>
    <NavigationWindow id="navWin" platform="ios">
        <Window title="My Title">
            <Button Title="Open tabgroup" onClick="clickOpen" />
        </Window>
    </NavigationWindow>
</Alloy>

with index.js:

function clickOpen() {
    win = Alloy.createController('tabgroup').getView();
    $.navWin.openWindow(win);
}
$.navWin.open();

And tabgroup.xml:

<Alloy>
    <TabGroup title="Not used :(" backButtonTitle="Not used :(">
        <Tab title="Shows in tab">
            <Window title="Hardly visible" backButtonTitle="Not used">
               <Label color="#999">I am tab 1</Label>
            </Window>
        </Tab>
        <Tab title="Okay">
            <Window title="Window 2">
                <Label color="#999">I am tab 2</Label>
            </Window>
        </Tab>
    </TabGroup>
</Alloy>

The result of the tabbed window is:

|---------------------------------------------------|
| < My Title                                        |
|                                                   |

I expected:

|---------------------------------------------------|
| < Not used        Hardly visible                  |
|                                                   |

The only trick I can think of is to drop the NavigationWindow and make it a TabGroup with a single tab (tab bar hidden), and to add tabs in the inner window. Seems a real kludge to me and I'm not sure it will even work. Anyone a better/real solution?

    I think nesting a TabBar within a NavigationWindow is like a kind of anti-pattern. The iOS docs say:

    "Before creating a tab bar interface, you need to decide how you intend to use a tab bar interface. Because it imposes an overarching organization on your data,you should use one only in these specific ways:

    • Install it directly as a window’s root view controller.
    • Install it as one of the two view controllers in a split view interface. (iPad only)
    • Present it modally from another view controller.
    • Display it from a popover. (iPad only)"

    To solve this, you should either use it stand-alone by using the TabGroups built-in NavigationController or open the TabBar as a modal window.

    • Okay. The same docs say:

      "You can use the view controllers that the UIKit framework provides by themselves or in conjunction with other view controllers to create even more sophisticated interfaces. When combining view controllers, however, the order of containment is important; only certain arrangements are valid. The order of containment, from child to parent, is as follows:

      • Content view controllers, and container view controllers that have flexible bounds (such as the page view controller)
      • Navigation view controller
      • Tab bar controller
      • Split view controller"
        So embedding a NavigationWindow inside a TabGroup is legitimate, the other way around not.

      If our designer insists to keep the tabs, I think I'll use a ScrollableView with a Toolbar instead.
