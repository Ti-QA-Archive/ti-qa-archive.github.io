iTunes File Sharing and Ti.Filesystem

Hi All,

I want to selectively share some files with the user via the iTunes File Sharing UI. I have enabled this via the Info.plist key. iTunes File Sharing shares out files in the Application Data Directory (/Documents).

Previously I had stored application generated files in the Application Data Directory but there are a lot of these which will confuse the user so I want to put them elsewhere. Apple recommend using the Library directory which is also backed up and preserved across updates. See Apple Q&A

I haven't been able to access the directory using explicit paths.

I'd like to use Ti declared variables.

Unfortunately the Library directory is not declared, only the following:
Titanium.Filesystem.applicationDataDirectory
Titanium.Filesystem.applicationDirectory
Titanium.Filesystem.resourcesDirectory
Titanium.Filesystem.tempDirectory

Has anyone else encountered this?

— asked July 23rd 2011 by Nick The Geek

  • Hi,

    There is an iOS-only constant called Ti.Filesystem.applicationSupportDirectory. This points to the Library&#x2F;Application Support directory, so I think it should work for your use case.

    This constant is missing from the documentation. There's an open issue for this omission and I'm currently adding it to the docs for the next release (1.8.1).
