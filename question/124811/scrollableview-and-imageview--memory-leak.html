  • A quick thought without caching is that every time you open a window it puts it onto of the previous one, so if you have a few images each time then the app will have four lots of this in memory. This could be your out of memory issue.

    Without looking at some code, I would recommend that you modify your code to close the window after it has been finished with and then open it again when needed. This gets round the issue of stacking but adds the issue of automatic back buttons, which would have to be coded to resolve.

    It does sound like this could be the issue as it occurs pretty much constantly. try cutting down the amount of information on the screen and see how many time it would open then.

    Hope this helps.
