TSS not applied to all elements

I've got a question regarding the application of tss styles to elements:
In my app I need to define some icons in a loop in the javascript.
These icons are then put into a certain view.
In order to separate my design from the logic I wanted to not define the design of my newly created imageview directly in the js.
So I created a separate widget as container for the single icons, and I put all my new icons into this container.
In the tss of my widget I defined the design for all image-views in it.

When testing, I discovered that the tss is only applied for the image views defined directly in the xml (example.png),
but not for the image views I add in the javascript.

Is there any possibility to apply a design from a tss also to elements in the javascript?

Some parts of my code:

In the xml of my "main" view:

<Require type="widget" src="com.example.MyWidget" id="widgetId"></Require>

In the js of my "main" view:

for (var i=0; i<icns.length; i++) {
    $.widgetId.icons.add(Ti.UI.createImageView({
        image : '/images/icons/' + icns[i] + '.png' 
    }));
}

The widget's xml is:

<Alloy>
    <ScrollView id="icons" class="detailIcons" >
        <ImageView image="/images/example.png"></ImageView>
    </ScrollView>
</Alloy>

The widget's tss defines the style for image-views:

"ImageView": {
    borderColor: "#FF0000",
    borderWidth: "5dp",
    left: "5dp"
}

Any suggestions?

  • I tend to give most of my UI either ID's or Classes. But if I was adding a style after creation then I would create a style in my tss like so:

    ".image": {
    borderColor: "#FF0000",
    borderWidth: "5dp",
    left: "5dp"
}

    And then I would use the addClass method

    for (var i=0; i<icns.length; i++) {
    var img = Ti.UI.createImageView({
        image : '/images/icons/' + icns[i] + '.png' 
    });
    $.widgetId.icons.add(image);
    $.addClass(img, 'image');
}

    This will then add the .image class to each of the 'img' objects created.

    For more information check Dynamic Styles

    • Another thing to note is that TSS doesn't act like CSS and so styles will not filter down to children.

      You have to add styles directly to the object via their own reference.
