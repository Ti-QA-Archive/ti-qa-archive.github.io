  • You could try using the 'sp' unit instead of 'dp'. The "scale-independent pixel" unit is the correct one to use for fonts. It takes display density into account (like dp) but also the user's font size preferences as set in the Settings app.

    var label = Ti.UI.createLabel({
  text: "this is a label",
  font: {
    fontSize: '32sp'
  }
};

    • Check the Settings app on both devices to make sure that there haven't been adjustments made to font display settings. Check on other devices, because maybe one of your test devices is just an oddball.

      What you're describing is basically the opposite of what should be happening. If you were using "px" for units, for example, on the high dpi device the font should render smaller because the same number of pixels fills a smaller spot. That's why you use dp/sp, so that they're scaled appropriate to the actual dpi.

      You could get tricky and calculate a size:

      var baseFontSize = 32; // determine by experimentation
var calculatedSize = baseFontSize / Ti.Platform.displayCaps.logicalDensityFactor;
var label = Ti.UI.createLabel({
  text: "this is a label",
  font: {
    fontSize: calculatedSize + 'px'
  }
};

      (What's shown above is basically what dp/sp is supposed to do for you automatically.)

      This would let you fudge the size a bit by throwing some extra factors in there for different display classes. Check the displayCaps docs page for other properties that might assist.
