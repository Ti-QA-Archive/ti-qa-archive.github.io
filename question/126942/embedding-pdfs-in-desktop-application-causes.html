Embedding PDFs in Desktop Application causes

I am creating a desktop app that should display a PDF inside the current window (that is, no pop-up windows or launching external applications).

I am able to successfully embed a PDF in a Titanium desktop app using the following markup:

<embed src="http://www.adobe.com/studio/print/pdf/graphic_file_formats.pdf" width="100%" height="100%" />

The PDF loading progress bar displays as expected, however upon displaying the PDF the following messages appear in the IDE console:

2011-10-17 22:13:59.023 PDF_Embed_Demo[764:1407] *** WebKit discarding exception: <NSInvalidArgumentException> Unlocking Focus on wrong view (<CocoaAVGenericControl: 0x104fbe00>), expected <CocoaAVGenericControl: 0x10401b40>
2011-10-17 22:13:59.024 PDF_Embed_Demo[764:1407] unlockFocus called too many time.

  • OS: Mac OS X 10.7 (Lion)

  • This issue occurs using both the 1.1.0 SDK and 1.2.0RC4

  • Adobe Reader is installed on the system

I have uploaded an example project that demonstrates the issue to http://dl.dropbox.com/u/5300095/PDF_Embed_Demo.zip

Any help is appreciated.
