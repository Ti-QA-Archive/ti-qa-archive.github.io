mapView is not as details as the native Google Maps App

Hello all

I have this code:

var w = Titanium.UI.createWindow({
    backgroundColor:"#ffffff"
});
w.open();

var mapview = Titanium.Map.createView({
    mapType: Titanium.Map.STANDARD_TYPE,
    region:{latitude:31.74511, longitude:-82.38993, latitudeDelta:0.5, longitudeDelta:0.5},
    animate:true,
    regionFit:true,
    userLocation:true,
    width:320,
    height:320,
    top:0
});
w.add(mapview);

var b = Titanium.UI.createButton({
    title:"Change",
    height:20,
    width:40,
    top:0,
    left:0
});

b.addEventListener("click", function(e) {
    mapview.setLocation({longitude:34.851319, latitude:32.033262, latitudeDelta:0.5, longitudeDelta:0.5});
});
w.add(b);

The first waypoint when I create the map is somewhere in Atlanta, where the map is details and beautiful.

When I Click the button it centers somewhere in Israel, but I see only the country silhouette, no roads, no signs, no nothing. When I run the Native App of Google Maps on the iOS, I see the same exact point very detailed and beautiful as well.

Is Google downgrading the maps intentionally? or could it be some kind of bug?

    This seems to be an issue with google maps API and Israel, see these links for more info.

    http://stackoverflow.com/questions/5862986/show-details-for-israel-map-in-mkmapview

    http://groups.google.com/group/Google-Maps-API/browse_thread/thread/989b4a856ecf5745/6d8711aab32f2547?lnk=gst&q=israel+streets#6d8711aab32f2547

    http://groups.google.com/group/Google-Maps-API/browse_frm/thread/989b4a856ecf5745#

    So i suspect they do not have rights to allow that region to be delivered to third parties but can use it themselves.

    I'm guessing you maybe out of luck unless you can change to a different map service provider which maybe a whole bunch effort that may not be worth it unless Israel coverage is vital to your app.
