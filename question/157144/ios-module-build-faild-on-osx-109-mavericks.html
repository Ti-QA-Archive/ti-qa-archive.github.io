  • This problem occurs with Mavericks 10.9 and Xcode 5.0.1.

    Turns out, it’s NOT a problem with Mavericks, but a problem with an Xcode 5.0.1 update.

    By running “./build.py” from the command line and watching the Activity Monitory, I noticed the following problem, a loop between xcrun and lipo:

    http://stackoverflow.com/questions/18667916/installing-psycopg2-has-it-stuck-between-xcrun-and-lipo

    By replacing the copy of lipo in /usr/bin with one from a prior copy of Xcode, the build process now works correctly.
