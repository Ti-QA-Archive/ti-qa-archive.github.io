How to detect a scroll start/scroll on a tableView with the index at 0 and trying to scroll further up the table

Imagine the initial state of a TableView with a couple of rows and I dont want to scroll down the table, but up. My goal is to implement a pull-to-refresh. Most of it works like I want it to but:

As far as I know there is no 'scrollstart' event for Android in Titanium. My problem is, there seems to be no scroll event fired, when starting to scroll at the top end of a TableView without actually scrolling. There is a 'scrollend' event fired oddly enough, I need to detect the scroll start or scrolling though.

So how would I implement such a 'scroll start' detection, or is there an easy way I just didn't think about yet (I am a Titanium beginner)?

    What about the touchstart event? Attach it to your tableview. When the scroll event is not fired, it means that the user in your case tries to pull the container down to refresh it (pull-to-refresh).

    On the touchmove event you can detect what movement the user is doing. 

    var tableView({ .. });
var startY;

tableView.addEventListener("touchstart", function(e) {
    startY = e.y;
});
tableView.addEventListener("touchmove", function(e) {
    // you can decide how quick it's triggered, by changing the check number
    if(e.y - startY > 50) { // e.y has to be greater than the startY to be a 'drag-down' action
        // Do your refresh
    }
});

    Takes some fine-tuning to make it right, this is just a quick example.
