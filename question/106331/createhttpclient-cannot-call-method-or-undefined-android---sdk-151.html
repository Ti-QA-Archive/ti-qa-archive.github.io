    Brieuc

    If you have updated our Android SDK packages recently, close your emulators and delete all the existing AVDs using the AVD Manager, as explained in Preparing for Android development (around the red box).

    Also, delete all the contents of the immediate subdirectories of your project's build&#x2F;android directory. Don't delete the subdirectories themselves, though, as this may cause you problems. Then on the Edit Project screen, click the Save Changes button to update the timestamp on the tiapp.xml file, and then launch the app.

    This should resolve the error.
