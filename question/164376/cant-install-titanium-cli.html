Can't install Titanium CLI

First, I've searched and searched and searched…haven't found anything that specifically addresses my issue.

I'm having a problem getting the Titanium CLI installed.

npm install -g titanium goes through the entire npm http get process, appears to pull down the titanium modules, taking about a minute. I can provide the output of the `'npm install -g titanium' if needed.

After doing a 'npm install -g titanium', if I run titanium -v, it returns sh: titanium: command not found

I have uninstalled and reinstalled Titanium Studio twice, yanking all remnants I could find off my machine prior to reinstalling.

At this point, I'm lost. The problem results from Titanium not being able to find my Android SDK, which without I'm stuck.

  • Have you tried the ti alias? As in ti -v ? npm ls -g should list your globally-installed modules. Make sure Titanium is listed there.

    It could be that system paths aren't being set correctly. I believe global modules are installed to C:\Program Files\nodejs on Windows. Make sure that's in your PATH with no ending slash. You might need to add quotes or use the 8.3 name for Program Files because of the space in that path. Perhaps it's a permission problem in which your user account can't read from that folder?
