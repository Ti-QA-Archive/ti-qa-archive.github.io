  • I haven't actually had a need to do this, but I think I can point you in an alternative direction. In Android, you can launch "activities" in other applications if they are designed to make those available. An example which Appcelerator used is this blog was how to use the built-in ZXing client to read a QR scan code and return it to your application.

    The interesting thing here is by looking at the Java class code for that activity in ZXing, located here, you can see that you can change the com.google.zxing.client.android.SCAN in the example code to com.google.zxing.client.android.ENCODE to switch from scanning a barcode to generating a barcode and displaying it.

    Beyond that, you'll also have to modify the example to pass the data to encode and specify the format. Since I have not had any reason to do that, I can't give you complete working code. I'm just pointing you in an alternative direction for Android. Since you're in the top 10 Q&A list, I'm sure you can take it from here. :)

    It would not need a webview or any module. It would just utilize the barcode generation capabilities already inherent in Android.
