Take a picture, save it then use in imageView - headaches

Hi,

I am trying to do what a lot of threads on this forum seem to be asking.

I am trying to take a picture then save it, then use it in an imageView. Simples.

Here are the problems i get;-

When i take a pic using the following code

Titanium.Media.showCamera({
      success:function(event)
      {
    var cropRect = event.cropRect;
    var image = event.media;
    var d=new Date();
    var filename = d.getTime() + pictype + ".png";
    var f = Titanium.Filesystem.getFile(Titanium.Filesystem.applicationDataDirectory,filename);
    f.write(image);

    Titanium.API.info('taken picture.. path is;-');
    Titanium.API.info(image.nativePath);

the console says nativePath is null.

I am trying to do this twice for two images in one window.
The second time i go to the camera it is all black with the viewer of the camera off in the top left corner with only 25% of it showing.

I am copying the examples word for word so i cannot see whats going wrong.

I am also using a camera overlay and allow editing;-

overlay:overlayView,
allowEditing:true,
showControls:true

pictype is a variable of a string either 'my' or 'your' depending on the image i am currently taking.

Obviously i have the rest of the camera function there and there are no syntax errors.

What i want to do is save the path in my local sqlite db and recall it later, but i only get null values in the database.

Any ideas on this ?

Thanks in advance.

Jez

    You want f.natvePath, not image.nativePath as image is your picture contents and f is the name of the filesystem object.
