    OK That was a bit complicated - but I think I have your general question.

    You want to know IF you can ask the user for the security details (email, username, password) within the scope of the app and then open a web view sometime later (or straight away) and show the service already logged in as you have provided the login credentials at the same time as the URL. If this is wrong let me know.

    If it is right, then;

    In most cases no - not with a website, part of this reason is that the site uses SSL encryption and other techniques so the password is NOT sent in plain text and in order for you to pass it that is how you would have to send it.

    There are currently no additional properties on the webview to allow you to add the extra information -allow you can use this technique which will work with some websites, but not many of the ones you are likely to want to use this for.

    http://user:pass@secure.site.com

    This would be a website secure.site.com, username user and password pass. As I say this is rarely the technique used though.

    You can also call the web page view HTTPClient but you will then not be suitable to show the user in this scenario.

    With all the main players moving to oauth or similiar solutions - you never actually get to use or interact with the credentials as they are kept from you (the programmer) for good security reasons (not you specifically of course).

    Hope this gives you a better insight.
