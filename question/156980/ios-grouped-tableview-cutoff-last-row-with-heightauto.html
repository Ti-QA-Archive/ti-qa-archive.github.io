iOS Grouped TableView Cutoff Last Row with height:"auto"

The following code cuts the last row exactly in half. It looks like a header is present on the table. It does not appear to be compensated for since its cutoff exactly by the header height (20 in this example). Am I doing something wrong here?

index.js:

$.index.open();

index.xml:

<Alloy>
    <Window class="container" id="index">
        <Label> Stuff... </Label>
        <TableView platform="ios">
            <TableViewRow>
                <TextField id="tf1" hintText="textfield1"></TextField>
            </TableViewRow>
            <TableViewRow>
                <TextField id="tf1" hintText="textfield2"></TextField>
            </TableViewRow>
        </TableView>
        <Label> ...Stuff </Label>
    </Window>
</Alloy>

index.tss:

".container": {
    layout: "vertical",
    backgroundColor: "white"
},
"TableView[platform=ios]":{
    height:Ti.UI.SIZE,
    style: Ti.UI.iPhone.TableViewStyle.GROUPED,
    scrollable:false,
    backgroundColor:"yellow"
}, 
"TableViewRow[platform=ios]": {
    left:0,
    backgroundColor:"white",
    height:"40"
},
"TextField[platform=ios]":{
    width: Ti.UI.FILL,
    height: 40, left:20, right:20,
    borderStyle:Ti.UI.INPUT_BORDERSTYLE_NONE
}
