random app crash on iPhone device

I've been struggling with an issue in titanium mobile for ages. My app crashes for what seems to be no apparent reason on random sections of the app. I'm using the tabgroup but I'm not sure if thats the cause.

The app is very image intensive as it's for a retailer. When monitoring the memory I can see that it's leaking some, but I've limited this. The problem seems to occur mostly when navigating around quickly (i.e. jumping from product to product and page to page fast).

There are xhr calls that fetch the information for each page and having checked the crash log from the phone I've fond the EXC_CRASH (SIGABRT) error. I'm not sure if the error comes from the xhr calls not completing, if this is the case does anyone know of a workaround or further error check to cancel an uncompleted call on window close?

I'm really not sure of the issue here, any thoughts you may have regarding possible fixes etc would be greatly appreciated.

Excuse me for the irate ranting style of this post, but this really has been bugging me!

I'm using the titanium studio SDK 1.7 on an iPhone 3GS ver 4.3.4

  • You can see device logs in Xcode organiser.

    Careful with XHR population of things, if the XHR has not returned properly or completely and you are using the XHR data to populate things, there can be a race condition.

    For instance if you download an XML file and store it, then try and read it to populate the picker you need to make sure you do not call the population from anywhere else other than the onload of the XHR.

  • The code populating the picker and the xhr call got into a race condition which caused the picker to look at the data before it had returned properly.

    To solve the issue I made the xhr call run asynchronously, using the async:true argument, so that no other operations were attempted until the data was returned. This seems to have solved the issue.

    I'm pretty sure the memory leaks and general buggyness of Titanium etc aren't helping the issue, but this seems to have reduced its negative affect on user experience.
