Cannot add a photo to a Place after removing a photo for that Place

Application Type: HTML Website
Host Operating System: Appcelerator Cloud Services

I have a website hosted on my Web Server, which uses Appcelerator Cloud Services to send and receive data to and from the Appcelerator Cloud.

I make use of the "Places" APIS to Insert and update records inside the Places Collection for my ACS Application on the Appcelerator Cloud.

I also use cocoafish-1.2 to upload and delete photos.

PROBLEM IDENTIFIED

For each Places Record:

  • If I upload a photo for the first time, it works.

  • If I remove the photo I uploaded, it works.

  • If I replace the photo with another photo, it works.

  • If I try to add a photo after removing a photo, it fails.

When the ACS fails, the following Response is returned:

{
  "meta": {
    "status": "fail",
    "code": 400,
    "message": "Failed to upload photo: undefined method `embedded_photo' for nil:NilClass",
    "method_name": "updatePlace"
  }
}

Below is the code used to upload the photo using cocoafish-1.2:

var sdk = new Cocoafish(ACS_ID);
   var data = {
      place_id : IDVariable,
      name : nameVariable,
      photo : 'fileInput'
   };

sdk.sendRequest('places/update.json', 'PUT', data, callbackFunction);

SUMMARY

If I add a photo, remove the photo added, and try to add another photo, the ACS fails. It doesn't make sense, but it almost feels as if I am not allowed to add a photo to a Place when a photo for that place existed but was removed.
