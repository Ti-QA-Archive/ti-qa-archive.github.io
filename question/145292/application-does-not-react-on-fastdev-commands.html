Application does not react on FastDev commands

Hello, i'm trying to get working fastdev on Windows 7. Here is my configuration:

  1. Application type: mobile
  2. Titanium SDK: 2.1.5
  3. Platform & version: Android 2.2
  4. Device: Android emulator
  5. Host Operating System: Windows 7

I made following steps:

  • Started Android emulator
  • Started fastdev server in first shell window: titanium fastdev start
  • Got "Serving up files for… at 0.0.0.0:51264" message
  • Started application in second shell window: titanium run –platform=android
  • Got "connected: 192.168.0.97:51270" message in first shell
  • And "[DEBUG] app.titanium… installed? True" message in second shell

And that's all. I can't get on emulator any changes i made in sources after that. And i can't see any file requests in fastdev server log. titanium fastdev restart-app, application reopen and titanium run –platform=android commands do not work.

Any idea how to work make it work? :(
