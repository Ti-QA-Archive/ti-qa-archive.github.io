  • it might be a limitation of the ListView at this time. Since you need to specify the templates with a string I'm not sure if you an use a module. You may want to try specifying the full require() call in the string used in the template. Something like this (untested):

    // i don't remember what the require() actually is for styled label, or the create functions, so you may need to change it

var myTemplate = {
    childTemplates: [
        {       
            type: 'require("ti.styledlabel").Label', 
            bindId: 'label',  
            properties: {/* .. */}
        },
// ...

    Probably doesn't work. but its worth a shot. If it does not work, log a test case and all the details to the Titanium Community Jira Project.

  • Hello,

    You can, I did it, but not in Alloy. I am still trying to figure it out for Alloy. For native, i've followed this example : http://noodlecode.net/2014/04/use-view-from-custom-module-inside-titanium-listviewitem/

    iOS :
    Just add the module, no need to require it. 

    {
     type : 'TiStyledlabelLabel', // taken from the module's code, I don't remember the file exactly, @interface TiStyledlabelLabel (Private)
    ....
}

    Android :
    Add the module, and require it in app.js (I tried requiring it where it was needed, but it didn't work)

    var Styledlabel = require('ti.styledlabel'); // in app.js

{
    type : 'Styledlabel.Label' ,
    ....
}

    That's it :)
