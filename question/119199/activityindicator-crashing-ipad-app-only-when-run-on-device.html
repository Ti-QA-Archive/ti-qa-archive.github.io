ActivityIndicator crashing iPad app only when run on device.

I'm working on an iPad app that uses an activity indicator while content it being loaded. In the simulator, it works just fine, but when I install it on the device itself, it no longer works. My code is below:

var indicatorWin = Ti.UI.createWindow({ height:150, width:150 });

var indicatorView = Ti.UI.createView({
  height:150,
  width:150,
  backgroundColor:'#000',
  borderRadius:10,
  opacity:0.7
});
indicatorWin.add(indicatorView);

var activityIndicator = Ti.UI.createActivityIndicator({
  style: Titanium.UI.iPhone.ActivityIndicatorStyle.BIG
  height: 30,
  width: 30
});
indicatorWin.add(activityIndicator);

var message = Ti.UI.createLabel({
  text:'Loading...',
  color:'#fff',
  width:'auto',
  height:'auto',
  font:{fontSize:20,fontWeight:'bold'},
  bottom:20
});
indicatorWin.add(message);
indicatorWin.open();
activityIndicator.show();

I've confirmed that my simulator is using the same SDK version as my iPad device (4.3). Any ideas why this happens?

Additional Information: After installing the app on my device, the simulator version starts crashing with the following error: Terminating app due to uncaught exception 'org.tennessean.module', reason: 'invalid method (createActivityIndicator) passed to UIModule' The only way to get it to work again in the simulator is to delete the contents of the build/iphone/ folder and do a full rebuild.

  • I've been able to fix this problem myself. Originally, I had the code above in a function called showActivityIndicator() that I would call when I wanted to display an activity indicator. I moved the code out of the function and in to the main app.js file so that the activity indicator is created on start, then I just use functions to hide and show it. Works like a charm on the device now!
