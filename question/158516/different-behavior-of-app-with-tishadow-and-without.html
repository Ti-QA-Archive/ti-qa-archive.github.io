Different behavior of App with TiShadow and without.

I am trying to execute exact the same code with TiShadow and without it, just Install to Android device from Titanium.

$.table.addEventListener('longpress', function(e){
        var alertDialog = Ti.UI.createAlertDialog({
            title : "Do you want to remove this row?",
            message : e.index,
            buttonNames : ["Yes", "No"],
            cancel : 1
        });
        alertDialog.show();
        alertDialog.addEventListener('click', function(e){
            if(e.index == 0){
                var table = Alloy.createCollection("title");
                table.fetch({query: "SELECT * FROM title where id = " + tableViewEvent.rowId});
                table.at(0).destroy();
                Alloy.Collections.title.fetch();
            }   else if(e.index == 1) {
                alertDialog.hide();
            }        
        });
});

When I use TiShadow I am getting the following runtime error:

Location:
[101,32] undefined
Message:
Uncaught TypeError: Object #<Object> has no method 'createAlertDialog'
Source:
var alertDialog = __ui.createAlertDialog({

But if I execute it using Install to Android device from Titanium it works without any issues.
Is this rather TiShadow configuration issue or should I alter somehow my code with AlertDialog in another way?

Also, I tried that function without AlertDialog - 'longpress' worked well in both launches.

I use:
Titanium SDK - 3.1.3
Platform - Android 4.3
Device - Nexus 4
Host OS - Mac OS 10.8.5
Titanium Studio - build: 3.1.3.201309132423

  • It looks like you need to update the the TiShadow app.

    Run this command:

    tishadow app /path/to/the/tishadow/app/you/generated/in/the/past --upgrade
