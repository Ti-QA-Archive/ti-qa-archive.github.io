  • Accepted Answer

    Titanium Mobile Web 1.8 is really old. There's been significant improvements since 1.8 was released. The current version is 3.0.2.

    In any case, Titanium Mobile Web does not support Windows Phone 7. We do however "run" on Windows Phone 7.5 (Mango), but functionality is severely limited. The biggest limitation is that Windows Phone 7.5 doesn't support w3c touch events, so UI elements such as TableView, ScrollView, ScrollableView, and gestures don't work at all.

    Windows Phone 8 also doesn't support w3c touch events, so things aren't improved. However, Windows Phone 8 does have a new touch API Microsoft invented that Titanium Mobile Web does not currently support, but hopefully will some day.

    In short, it will be extremely difficult to create a decent mobile web experience for Windows Phone users regardless of what framework you use.

    • Titanium Mobile Web runs in mobile web browsers. Most mobile web browsers do not expose things like camera and bluetooth. Titanium Mobile Web does support geo-location provided the browser supports it.

      We are working on added native support to Titanium for the Windows Phone platform. It's not done yet and I don't know when it'll be available. Could be a while.

      As far as Windows Phone is concerned, given the major limitations of the Internet Explorer web browser, you will be able to provide a much better user experience in a native app.
