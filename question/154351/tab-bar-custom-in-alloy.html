tab bar custom in alloy

Hi, I'd like to ask which is the best strategy to implement a tab bar navigation inside a view, have 4 bottons, when user clicks on one of them I have to change a portion of my window by loading a view associated at the button. Here the view:

<Alloy>
    <Window>
        <View layout="vertical" class="container" id="viewMain">
            <View id="viewTopBar">
                <Button id="buttonInfo" onClick="buttonInfoClicked" right="45dp"></Button>
                <Button id="buttonHome" onClick="buttonHomeClicked"></Button>
            </View>

            <View id="viewNavBar" layout="horizontal">
                <Button id="buttonNavBarMap"></Button>
                <Button id="buttonNavBarList"></Button>
                <Button id="buttonNavBarCategory"></Button>
                <Button id="buttonNavBarSearch"></Button>
            </View>

            <View id="viewPlaceholderMap">
                <Require type="view" src="map" />
            </View>
            <View id="viewPlaceholderList">
                <Require type="view" src="table" />
            </View>
            <View id="viewPlaceholderCategory">
                <Require type="view" src="category" />
            </View>

        </View>
    </Window>
</Alloy>

and this is the controller


$.buttonNavBarMap.addEventListener('click', function(){
    $.viewMain.remove($.viewPlaceholderCategory);
    $.viewMain.remove($.viewPlaceholderList);
    $.viewMain.add($.viewPlaceholderMap);
});

$.buttonNavBarList.addEventListener('click', function(){
    $.viewMain.add($.viewPlaceholderList);
    $.viewMain.remove($.viewPlaceholderMap);
    $.viewMain.remove($.viewPlaceholderCategory);
});

$.buttonNavBarCategory.addEventListener('click', function(){
    $.viewMain.remove($.viewPlaceholderList);
    $.viewMain.remove($.viewPlaceholderMap);
    $.viewMain.add($.viewPlaceholderCategory);
});

the performance are not so good and I'm looking for a best solution, any hint? Thank you

  • Hi

    One way to achieve your goal would be to use the ScrollableView and in each "page" add your visual elements. Then each button would change the current page to the correct index.

    Example mark-up

    <Alloy>
    <Window id="win">
        <ScrollableView id="scrollableView" showPagingControl="true">
            <View id="view1" backgroundColor="#123" />
            <View id="view2" backgroundColor="#246" />
            <View id="view3" backgroundColor="#48b" />
        </ScrollableView>
    </Window>
</Alloy>

    You would then place whatever elements you needed inside each main child view, you could even use the require keyword to manage the "pages" seperately.

    Ideally you would use the native tabs to solve this problem - but if you do need your own custom version you will find this solution very easy to maintain and the performance should be more than suitable.
