Button with Image and Background Image Issues

I'm trying to create a button with a scaled foreground image and a stretched background image (image buttons…). Essentially I want to make generic button backgrounds for when events are fired, and have a scaled image icon as the foreground. I am trying to accomplish this with the following code

#optionsButton":{
    backgroundImage:'/images/ftest-bgs.png',
    backgroundSelectedImage:'/images/ftest-bgp.png',
    image: '/images/ftest.png',
    height: "100%",
    width: Ti.UI.SIZE
    }

Needless to say it does not work correctly. On an older android version (2.3.4) it will stretch the "backgroundImage" to the width of the "image", while keeping the aspect ratio and clipping the top and bottom of the image, what I would really like to happen is to have the "image" scaled at its original aspect ratio, with a smaller footprint than the total size of the background image at it's native aspect ratio.

On a new device (android 4.4.2) it will give the button the proper "backgroundImage" aspect ratio, however the "image" is small…

On ipad (iOS 7) it will do what it does on android 2.3.4 except the height is scaled to the area, so it is smushed.

At this point, my best idea is to create a widget and pass it the height, and then when the screen loads determine the size of the button and the size of the icon, create an imageview with the background and then another imageview with the icon and attach event listeners to the foreground and make it act like a button?

What is the proper approach to Image buttons in appcelerator? I can't believe that it is to create tons of individual images for every aspect ratio and load them situationally on a stretched button background…
