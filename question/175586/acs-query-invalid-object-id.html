ACS query invalid object Id

Hi,

when I tried to query for a user by its object id I got an exception (400) because one of the object Ids passed was not found.


cloud.Users.query({
        where:{
            id:{"$in": usersArray}
        }
    }, function(e) {
        //
    });

Can't the query just return a result that contains the found Ids only ?

— asked June 27th 2014 by Hani Hamadeh

  • query

  • Yes, it can if you give it the proper query parameters. ACS is returning an error because one or more of the IDs you've passed is malformed. When it comes to queries, ACS can accept 24-character hex strings for the user IDs, so make sure that each ID in this usersArray is this length. Also, don't include any placeholder values in the usersArray like -1 or a boolean.
