is there some other function than "createImageView" to solve this issue or any other kind of solution you can think of?

Here my code-example

var imgUrl = "http://XXXXX.XXXX.XXX/REST/auth/path/thumb/1530.jpg";
        Titanium.API.info(" ------------ " + imgUrl + "------------ ");
        var row = Titanium.UI.createTableViewRow();

        var flag =  Titanium.UI.createImageView({
            url: imgUrl,
            width:45,
            height:45,
            left:4,

        });

  • Osman

    Regardless of whether the server returns a static image or creates one on-the-fly, it's how it's sent/presented to the client that is important. If it sends it as a static image, you should be able to use an imageView as you normally would.

    Your URL of http:&#x2F;&#x2F;XXXXX.XXXX.XXX&#x2F;REST&#x2F;auth&#x2F;path&#x2F;thumb&#x2F;1530.jpg looks like a non-complex address that should not cause any problems.

    Your question is not really clear about what your problem is. If you are still having issues, please explain them. A usecase with a real URL, so that people can test the issue for themselves, would be helpful.

    Thanks
