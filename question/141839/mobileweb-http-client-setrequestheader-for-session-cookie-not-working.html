Mobileweb HTTP Client setRequestHeader for Session Cookie not working

On my mobileweb project a http client call for the login fails, because the session cookie that is sent after the login into the user interface is not sent for the second and third calls from my api.

So the api generates a new session cookie and the user is not logged in anymore.

When I try to setRequestHeader('Cookie', sessionCookie) nothing is sent to my api script.

Is this a bug in Titanium or is there any workaround for this?

Thanks for your help

  • Due to browser security limitations some headers cannot be set and as far as I know, no headers can be set on cross-domain requests. Luckily you can use a custom proxy server to set your session cookies. You might also be able to do a mobile web specific code block and manually add the session cookie using document.cookie.
