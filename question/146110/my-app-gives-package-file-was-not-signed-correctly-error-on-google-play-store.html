    For the record, this is not a Titanium specific problem. This is a JAVA tooling problem. This occurs frequently with mixing JDK and JRE tools on the system.

    DO NOT USE THE TOOLS FROM Java 7!

    Only use the tools from JDK 6.

    Optionally, we can waste time by having you paste the output from the following so that we both feel we have actually done something:

    which jarsigner

    jarsigner -verify -verbose -certs yourJar.jar

      Yes you CAN use 1.7!

It is very tricky to find info on the usage with 1.7, but once found it is reasonable simple:

For keytool include

-sigalg SHA1withDSA -keyalg DSA -keysize 1024

For jarsigner include

-sigalg SHA1withDSA -digestalg SHA1

(1024 is the maximum and works, less might do the trick)

  • thanks for your response Stephan Feather. As you said, the problem occurs because of jdk 7.
    I solved my problem by running below command before jarsigner -verify -verbose -certs my_app.apk

    jarsigner -verbose -sigalg MD5withRSA -digestalg SHA1 -keystore my-release-key.keystore
