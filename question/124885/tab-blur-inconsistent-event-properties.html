Tab blur inconsistent event properties

When a blur event is fired on a tabgroup, it's event properties relating to current tab and previous tab are inconsistent between Android and iOS. If previousTab (previousIndex) is suppose to be the tab that is blurred then Android has previousTab and tab (and previousIndex and index) values switched. My guess is that previousTab is the tab that was blurred and the tab value is actually the tab being focused.
