UPD Wifi Connection

Hi All,

I am building a app the connects to a wireless controller over UPD.

The only stumbling block i have from a usability perspective is the user has to go to settings find the network and connect to the device WIFI.

This is not the end off the world as i can display a popup with info displaying it is not connected and how to connect then check for connection, but i have been asked to make this process more user friendly.

Now a couple of things which i am pretty sure are not possible but i wanted to ask anyway to get a second opinion is.

  1. Can i redirect them to the wifi settings screen from the app, i know you used to be able to use pref:// but as i understand this is no longer possible is this true?

  2. Can i allow them to auto connect to the wifi within the app i have a set IP, i know this is a long shot but thought i would see if anything like this is possible.

  3. Any suggestion on a good way to do this.

If not ill just have to settle for a popup with directions but i know some users will find this frustrating and complicated?

Thanks

    For Android, there are these WiFi Access QR-Codes that encode the data for an access point, so there might be a way to configure the access point in native code. For iOS, this is not possible.

    However, you can offer to copy the password to the clipboard and open the settings app.

    For opening the settings under iOS use the all new Ti.App.iOS.getApplicationOpenSettingsURL. I don't know if theres a way to get to the wifi setting directly, though.
