Titanium.Filesystem.getFile() not working when in production state.

Titanium.Filesystem.getFile(&#39;filename.js&#39;); comes back with file does not exist in Android only when I deploy to production.

Otherwise it works fine. Any ideas?

I've also tried Titanium.Filesystem.getFile(Titanium.Filesystem.resourcesDirectory, &#39;filename.js&#39;); and Titanium.Filesystem.getFile(Titanium.Filesystem.resourcesDirectory, &#39;&#x2F;android_asset&#x2F;Resources&#x2F;filename.js&#39;); and many other things over the last couple hours. :-/

    In production mode, we compile your JS files into bytecode, so you won't be able to actually find them using the filesystem. You have a couple of options:

    If you just want to evaluate the code in filename.js, you can:

    Ti.include("filename.js");

    Alternatively, if you want to read the contents for some reason before evaluating the code, you can give the file some extension other than .js, and the compiler won't pre-compile it.
