    Hi Nicolò

    Quick tip put three ~ symbols on a fresh line before and after your code, makes it easier for us old folk to read - cheers.

    To the answer.

    You have a search bar in your code;

    var cerca = Ti.UI.createSearchBar({ value:'Cerca..', width:'auto', heigth:40, top:0 })

    But you are currently adding it to the window - seems like a smart idea - strangely - it is not (this one got me at the start.

    So remove this part;

    win.add(cerca);

    What we need to do now is add two properties to your existing table create;
    filterAttribute

    var myTableView = Ti.UI.createTableView({
    search: cerca, // this line is new
    filterAttribute: 'pizzatitle', // this line is new
    data:mydata,
    allowsSelection:'true'
});

    And a quick tweak to your row create

    row = Ti.UI.createTableViewRow({
    pizzatitle: rowText[i], // this line is new
    height:'70dp',
    backgroundImage:'images/brown.png', 
    backgroundRepeat:'true'
});

    The reason for the extra bit - may now be obvious - but just in case;

    myTableView

    • search this tells the table to use the search you created already (but did not add to the view) it gets assigned here instead.
    • filterAttribute this tells the table which property to use in the row INSTEAD of title.

    row

    • pizzatitle this stores the name of the pizza directly in the row - for the search to use

    Should work a treat.

    Cheers

    • Yes you could add a row called 'favourites' and include the extra row at the top of your other rows.

      You would make this row separately from the other rows as it will look different I would imagine.

      The key here though is to use another custom field - like we did with pizzatitle, add one to the existing rows in my original tweaks, called id as an example. Value for this will be menu.

      Then add another row favourites with an custom field id as well. Value for this will be favourite.

      Then inside your table view click event, wrap an if statement like;

      If (e.source.id === 'menu') {
  // do what you do now to open up extra table view window
} else {
  // show favourites table view window instead
}

      This way one table serves two jobs.
