Displaying maps for indoor navigation

We are in the process of creating an app which will feature indoor navigation. Logically this would require some kind of map to show the planned path on. However we are unsure what is the best way to show a custom map. A map of a single floor has the following requirements in our project:

  1. -It should have sufficient detail (not pixelated) to allow the user to understand where he/she is.
  2. -It should be fast (as little delay between touch gestures and map movements as possible)
  3. -We should be able to zoom and move the map.

With that said my first idea was to use an image in a scrollview. I believe this would work however there are several problems:

  1. -Large buildings will result in very large images. I'm not sure how an average mobile will deal with images with more than 10k*10k pixels. It may be slow and it will certainly use a lot of memory.
  2. -How will I draw a route on an image in an easy way?
  3. -Zooming is not directly supported, so we have to program something for that.

So an image in a scrollview may not exactly be ideal in our situation.

Our second idea is based on my previous experience with the ArcGIS sdk for Android. I could simply create a map on a desktop computer, store it on a server and allow the app to download parts of this map (at various resolutions) when needed. It also allowed for easy addition of features like a series of lines to represent a path.

Sadly appcelerator has no module for ArcGIS so at the time this is also not a solution. It is possible though to use a webview and the ArcGIS javascript library but this has proven to run quite slow.

The next thing we saw was the openstreetmap module, the features look a lot like those of ArcGIS. Sadly this will also not work since it does not support Android, only iPhone.

So to summarize: what would be the best (or atleast working) way to show a detailed map to the user which allows zooming and scrolling while also allowing us to draw other elements (lines or polygons) on it?

I know this may be a hard question to answer, but if you have any ideas please share. It may just help us
solve this problem.

  • If you're not open to the idea of creating a native module to add support for something like Micello then you're very limited. However, the only other way that I can think of besides the solutions you've already mentioned is a <canvas> element wrapped in a webview. The canvas element will give you zoom and pan functionality while also allowing you to create polylines. This may not be the most elegant solution, and the more elements you add to the canvas the slower it may become, but it will give you the functionality that you need.
