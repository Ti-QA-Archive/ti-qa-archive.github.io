Image Offset and Scale Problem

Hi,

I have a problem and can't solve it by myself.
I developed an app where the user chooses an image from the image gallery. I then display it and the user can scale (pinch) and move it into a shape. That part is working perfectly!

I then take the original image hires image, scale (event.media.imageAsResized) it to the same scale as the user did in the imageview. Thats also working. As I only need a the visible part of the image and not the whole image I try to crop it with the imageAsCropped function. The problem is that the cropped image is not the same part as I see on the screen.
I measure the movement on the screen but when I crop the image with the same offset as the image on the screen, the cropped image is not right.

For example: The user scales the image to 2.48 and moves it 100px to the left and 100px to the top. I then take the original image, scale it by 2.48 and then I take the middle of the image (width/2, height/2) and substract 100px from the middle.x and 100px from the middle.y. I then substract (960/2)px from the middle.y and (480/2)px from the middle.x.
image.imageAsCropped({x:middle.x,y:middle.y,height:960,width:480});
The result is another part of the image as I expected :-(

The problem must be the scale. When I leave the scale at 1.0 and move the image, the cropped image is right.

the .toimage function is not working for me because I need a higher resolution - even on older non-retina devices.

I hope someone can help me! If you need further information, I can provide them.

Regards,
Joscha

  • I solved my problem. Never measure the offset. Instead use the getLeft() and getTop() attributes of the image. My pinch + zoom App with hires images is now working!
