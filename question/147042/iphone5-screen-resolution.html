    Hi

    The default image is the only one that supports this naming convention so default-568h@2x.png will work for the splash/start-up screen but this is the only place.

    So your example is a background image, this will NOT be done automatically for you, you have to manage this yourself.

    iOS will swap between non-retina and retina images but not height adjusted images, you will need to swap those in yourself using logic to determine the device height.

    This is not a Titanium limitation it is an iOS one.

    • Here you go - I have create three images that you can use to test this (same ones I used to confirm it works).

      • back.gif
      • back@2x.gif
      • back-568h@2x.gif

      Save these and name these as shown and place them in the resources folder.

      This function;

      function h568(strFilename) {
    return (Ti.Platform.displayCaps.platformHeight === 568) ? strFilename.replace(/(\.[a-zA-Z]{2,4})/gim, '-568h$1') : strFilename;
}

      Will parse filenames and ONLY if the device has a tall height (568/iPhone 5) returns revised filename with the suffix tweaked. If the device does not have a tall screen the filename is returned untouched.

      This function does not care what file extension you use it handles them all

      This is how you use it.

      var view = Ti.UI.createView({
    backgroundImage: h568('back.gif'),
    height: Ti.UI.FILL,
    width: Ti.UI.FILL
});
win.add(view);

      My example code will add a background image EXACTLY the correct size for a view that fits inside a tabGroup with navBar and statusBar - basically a standard app.

      So for a non-retina device it will use back.gif, for retina devices back@2x.gif and for tall devices it will use back-568h.gif.

      Here is a code block if you want to run this stand-alone, simply create a blank new project and replace the contents of app.js with the code below. You only need to do this is the code above is not obvious enough for you.

  • Hi Orhan,

    I recently came across this problem when developing for a universal app.

    Apple introduces naming conventions for assets in your apps. As Malcolm rightly mentioned, default-568h@2x.png only works for the splash screen. It works as an indicator to the device that the app maybe optimized to run on the iPhone 5.

    Getting to the point, you can name the assets in this fashion and iOS will pick out the appropriate one, without having to write any custom code. 

    'button.png' //For Non-Retina devices
'button@2x.png' //For iPhone 4,4S
'button~568@2x.png' //For iPhone 5
//or
'button-568h@2x~iphone.png' //For iPhone 5, I can't recollect exactly which one works
'button~ipad.png' //For the iPad

    You can read more about these naming conventions here: http://developer.apple.com/library/ios/#documentation/iphone/conceptual/iphoneosprogrammingguide/App-RelatedResources/App-RelatedResources.html
    and here http://www.vigorouscoding.com/2012/03/naming-conventions-for-image-resources-in-ios/
