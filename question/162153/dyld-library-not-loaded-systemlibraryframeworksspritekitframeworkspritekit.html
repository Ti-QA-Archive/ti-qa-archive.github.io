  • It looks like you are building using the 6.1 Xcode simulator. The SpriteKit framework is only available since iOS 7.0, and it would seem the 3.0 version of StoreKit is linked against it. So, yes I would upgrade to latest Xcode. I think Apple have mandated that you build against the latest sdk for new Apps as of February anyway

  • Solved the problem.

    When you upgrade xcode to 5.0.2 you end up with sdk 7.0.3 Which Titanium 3.2 with sdk 3.1.3 and Cli 3.2 cannot see sdk 7.0.
    If you just use titanium sdk 3.2 then you can then just use sdk 7.0.3 but in my case I needed to be using titanium 3.1.3 ga because the app I have been working on for the past seven months uses ios Navgroup.

    To be able to use titanium 3.1.3 with xcode 5.2 ,sdk 7.0.3 you can downgrade the Titanium Cli 3.2 back to version 3.1 by typing this into the terminal

    sudo npm install -g titanium@3.1.0

    and you then get to use sdk 7.0.

    This however is a backward step and I found this gist with step by step instructions to be able to use titanium 3.1.3 together with titanium 3.2 and Cli 3.2 and xcode 5.2, plus I can also use titanium 3.2.0GA when I get round to removing the navgroup for my app.
