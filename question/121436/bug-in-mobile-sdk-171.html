BUG in Mobile SDK 1.7.1

Hello

It seems that the bugs signaled here, in the Q&A section, aren't very well analyzed and taken into account.
Anyway, I just want to make sure this is clear:

There is a bug in the 1.7.0 and 1.7.1 versions of the mobile SDK that prevents us to properly use classes in JSS.

To be more specific, I have this code:

var lblMin1 = Titanium.UI.createLabel({
    className: "labelsMin",
    id: "lblMin1"
});
mainView.add(lblMin1);

var lblMin2 = Titanium.UI.createLabel({
    className: "labelsMin",
    id: "lblMin2"
});
mainView.add(lblMin2);

var lblMin3 = Titanium.UI.createLabel({
    className: "labelsMin",
    id: "lblMin3"
});
mainView.add(lblMin3);

/* This is in the JSS file: */
.labelsMin {
    height: 'auto';
    width: 144px;
    top: 549px;
    color: blue;
    text-align: center;
    font-size: 23px;
    font-family: 'CustomName';
    font-weight: bold;
}
#lblMin1 {
    text: 'Min: -';
    left: 9px;
}
#lblMin2 {
    text: 'Min: -';
    left: 156px;
}
#lblMin3 {
    text: 'Min: -';
    left: 302px;
}

This worked correctly in 1.6.2, now some of the definitions in the class are not applied. More specific, these are the definitions that I must move to the ID-based definitions:

  • font-size
  • width
  • top
  • height (tested for images)

It is possible that any definition that has "px" won't work in the class group, but only in the ID-based group.

The issue appears only on Android.

Is this fixable?
