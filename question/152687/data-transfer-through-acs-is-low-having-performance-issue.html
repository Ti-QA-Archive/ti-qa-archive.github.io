Data transfer through ACS is low having performance issue

Hi

we have developer one application which fetches the around 700 records from the ACS on first load, and it takes time around 40-60 sec. which causing the performance issue.

we have use the query using modified date and all to give most latest date only but as first app install it required all the 700 records at once but as mention its causing time delay in loading

is the problem any one face or is it ACS issue ?.

can any one help to get idea about this issues.

    Hello,

    If you are starting the sync of your local db with the remote db (ACS or any other service) and the initial load is what you mentioned, it's likely to cause some issues. There is a lot of variables, like:

    • How much time the ACS will take to execute your query and respond.
    • How much time will take to your app to download that response, according to the type and speed of connection you have.

    Being said that, what i would recommend is start the sync in background, requesting just some elements (20? 40? 50? at a time) and continue to downloading the rest, so the user gets the feeling that the app is working and still responsive.

    Best,

    Mauro

    PS. BTW, 40-60 secs for a "setup" sounds like a dream, specially compared to the usual times that it takes to many apps to do the initial setup. IF you explain to the user what is happening and why is important, with an activitymonitor, should be reasonable.
