Splash screen appears during orientation change

Hi all,

I've got an app supporting orientation changes fine. But occasionally (not always), the splash screen will appear during the orientation change before the app has a chance to scale and reposition the UI elements. It looks like its the result of a lag.

Are there any solutions to this? I wouldn't mind a black background so much, but the splash screen really looks glitchy when it does that. This is on Android by the way. iOS transitions quite smoothly.

Thanks heaps.

    Hi

    Have you tried setting a backgroundColor or backgroundImage to either; the windows you have created or base Ti?

    Ti.UI.setBackgroundColor('#fff'); // sets the base **backgroundColor** for the app
// make this the first line of you **app.js** - for testing

    And for any windows;

    win.setBackgroundColor('#000'); // where win is the window variable you declared.
