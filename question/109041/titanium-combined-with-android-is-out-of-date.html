TITANIUM combined with ANDROID is Out of Date..

Hello,<br />
for android dev's (with a fresh / new android 2.1/2.2/2.3 version or higher) it could be a painful disaster, because there are some more updates required. First, the "adb" file is no longer located in the "tools" directory of android (the problematic path should be edited by appcelerator to "platform-tools/adb", or better, use a second search path ("platform-tools") or use a hard link instead, that means, use the terminal "ln [ANDROID_FOLDER]/platform-tools/adb [ANDROID_FOLDER]/tools/adb). You need also a installation / update by the sdk tool ("tools/android"), the following packages should be installed:<br />

  • Android SDK Tools, revision 9<br />
  • Android SDK Platform-Tools, revision 2<br />
  • SDK Platform Android 1.6, API 4, revision 3<br />
  • Google APIs by Google Inc., Android API 7 or higher

The next stuff is also problematic, because of missing write permissions on mac's. Be sure, that there are these 3 folders in your home directory named ".titanium", ".android" and ".deltafy". Use the terminal / commandline to solve this problem:<br />
sudo mkdir ~/.android ~/.titanium ~/.deltafy <br />
sudo chown [YOUR_USERNAME] ~/.android ~/.titanium ~/.deltafy

I hope that this also helps others, because it was like pain in the a** for me xD

But I have still the problem with kitchensink, the emulator can't install the app, I got only this meaningless message:
[ERROR] Failed installing com.appcelerator.kitchensink: pkg: /data/local/tmp/app.apk

There are no more tracing / debugging messages to solve this last problem :-(

[EDIT]: After a short while I regonized, that there is no API in the package included, this means that I have to download the api with the android manager ("tools/android"). In my opinion it could be a little bit more comfortable.

  • Seb

    Most of your post talks about the Android SDK, which is produced by Google. If you think Google should improve its product, then you would no doubt have better success by addressing your complaints directly to Google.

    In my opinion, your advice about chowning ~&#x2F;.android ~&#x2F;.titanium ~&#x2F;.deltafy is misinformation. If your underlying system is configured correctly, all files created by a user in their own home directory will be automatically owned by that user. Furthermore, if you have "missing write permissions" on your home directory, then it sounds like something is really amiss with your system in general, possibly (although, it's a guess) with the user's umask setting.

    Yes, I agree that it will be much better when the adb bug is fixed, but the workaround is explained several times in the docs, not least in the Getting Started guide, which I trust you have followed.
