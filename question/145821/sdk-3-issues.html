  • Hi

    I had the same issue. You can clear this warning by selecting the active tab.
    Add this line before you open the tabGroup:

    tabGroup.setActiveTab(0);

    — answered January 10th 2013 by Niels Wille

  • Hi

    I have being seeing the exact same thing and it is easy to reproduce, this code will trigger the warning - and as you can see there is nothing complicated here.

    I am working on the basis it is a small bug that those nice people at Appcelerator will resolve. It does not appear to have any negative effect.

    Ti.UI.setBackgroundColor('#000');

var win1 = Ti.UI.createWindow({  
    backgroundColor: '#fff',
    height: Ti.UI.FILL,
    title: 'Test 1',
    width: Ti.UI.FILL
});
var tab1 = Ti.UI.createTab({  
    icon: 'KS_nav_views.png',
    title: 'Test 1',
    window: win1
});

var win2 = Ti.UI.createWindow({  
    backgroundColor: '#fff',
    height: Ti.UI.FILL,
    title: 'Test 2',
    width: Ti.UI.FILL
});
var tab2 = Ti.UI.createTab({  
    icon: 'KS_nav_views.png',
    title: 'Test 2',
    window: win2
});

var tabGroup = Ti.UI.createTabGroup();
tabGroup.addTab(tab1);
tabGroup.addTab(tab2);
tabGroup.open();

    Also, this is from a cleaned project created fresh in Ti SDK 3.0.0 GA

    And just to prove you can make things simpler;

    Ti.UI.setBackgroundColor('#000');
