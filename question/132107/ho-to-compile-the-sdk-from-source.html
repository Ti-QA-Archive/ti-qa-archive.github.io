    Hi, if I remember well Boydlee Pollentine http://developer.appcelerator.com/devlink/profile/259495/boydlee-pollentine once implemented that feature in a custom Ti SDK fork. I can't remember if he open sourced the solution, or if it's available somewhere.

    WRT building the SDK, you need to install the Scons build tool http://www.scons.org/. More info here. Once installed you can launch the scons command from the Ti SDK sources root directory.

    The command accepts some parameters. Without any argument it will try to build the complete SDK. If you want to build only the android part of the SDK, you can use:

    $ scons android=1 android_sdk=/path/to/android-sdk

    you can also provide a custom name for the generated SDK version, e.g. my-1.8.1 with:

    $ scons android_sdk=/path/to/android-sdk PRODUCT_VERSION=my-1.8.1

    once the sdk has been built, you'll find the zipped package in the dist directory, e.g. dist&#x2F;mobilesdk-my-1.8.1-osx.zip

    For installing the custom SDK you can follow the instructions available for installing continuous integration builds: https://wiki.appcelerator.org/display/guides/Installing+Titanium+SDK+Continuous+Builds
