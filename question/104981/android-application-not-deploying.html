Android Application not deploying

Dear community,

I am new to Appcelerator Titanium and cannot make it work on Windows 7 x64.
It works as a charm on Linux but I would like to use it on Windows too.

I installed, Android SDK, Scons, Git, Python 2.7, Titanium and set up the PATH and other system variables as described in the tutorial.

Titanium (1.2.2) detects the Android SDK and starts it.
But an error occurs when trying to deploy the Kitchen Sink application:
[TRACE] E&#x2F;PackageManager( 73): Package com.appcelerator.kitchensink requires unavailable shared library com.google.android.maps; failing!

The complete trace level log of the deployment is available here:
https://gist.github.com/796405

I have the Android SDK with the optional Google APIs installed (all of them).

For information, Rhodes framework on the same computer (similar to Appcelerator Titanium) succeed to start an application.

Any idea of the problem?

Best,

Thomas

Edit: Answer found: Select "API 2.3" instead of "2.3"…
