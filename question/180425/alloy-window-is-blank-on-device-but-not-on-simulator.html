Alloy Window is blank on device, but not on simulator

Hi,

Whenever I load a window in a TabGroup beyond the first window, the window is completely blank on my device, even though on the iPhone simulator it works as intended.

Here's what it looks like on my simulator: http://imgur.com/IZVRsMt

Here's what it looks like on my device: http://imgur.com/Abjd9mh

Titanium SDK: 3.2.2

Target Device: iPhone (iOS 8)

Here is my code:

rss_add.xml:

<Alloy>
    <Window id="win" title="Add RSS Feed" backgroundColor="#eee">
        <RightNavButton id="btnDone" systemButton="Titanium.UI.iPhone.SystemButton.DONE" onClick="save"></RightNavButton>
        <TableView id="table" width="100%">
            <TableViewSection>
                <TableViewRow title="Title">
                    <TextField hintText="Name of the RSS feed" id="input_title"/>
                </TableViewRow>
                <TableViewRow title="URL">
                    <TextField hintText="URL of the RSS feed" id="input_url"/>
                </TableViewRow>
            </TableViewSection>
        </TableView>
    </Window>
</Alloy>

rss_add.js (nothing happens here on initialization, so issue is not here - feel free to skip)

var controller = {

    init : function() {
    },

    save : function() {
        var _title = $.input_title.getValue();
        var _url = $.input_url.getValue();
        var _code = Math.random().toString(36).substring(10);
        var errors = [];

        var data = {
            title : _title,
            link : _url.toLowerCase(),
            code : _code
        };

        if (!data.title) {
            errors.push('You must enter the name of the RSS feed.');
            $.input_title.focus();
        }
        if (!data.link) {
            errors.push('You must enter the URL of the RSS feed.');
            $.input_url.focus();
        }

        if (errors.length) {
            alert(errors.join('\n'));
            return false;
        }
        else {
            Ti.API.info('save RSS info now.');
            Ti.API.info(data);
            var _id = FX.db.fxdata.saveRSS(data);
            data.id = _id;

            FX.cloud.saveRSS(data, function() {
                Ti.API.info(JSON.parse(this.responseText));                    
            });

            Ti.App.fireEvent('fx:news_reload');
            $.win.close();

        }

    }    
};

controller.init();

function save() {
    controller.save();
}

rss_add.tss

"#win" : {

}

Here's the code which opens the window:

var win_add = APP.getWindow('news/rss_add');
APP.tab_news.open(win_add, {animated : true});

APP.getWindow simply is a wrapper for Alloy.createController. APP.tab_news is a global reference to a tab in a tab group.

I tried all sorts of different things to solve the problem, but none of them worked:

  • set the width and height of the window to 100%
  • set the width and height of the window to Ti.UI.FILL
  • set the width and height of the window to Ti.UI.SIZE
  • did not specify any width and height for the window (default)
  • define window elements (TableView) with an alloy view
  • define window elements the old fashioned way (Ti.UI.createTableView) then add to the window
  • tried compiling with different versions of Ti SDK, same result.
  • replaced TableView with ListView

I'm pretty sure the issue is with the Window object, as I removed the entire TableView and just added a simple label, which also did not appear on my device.

Where am I going wrong? How can I fix this so that the Alloy elements appear on the device the same way they appear on the simulator?

thanks,

-Asrar

      Thank you for chiming in. APP.getWindow already calls getView(). Here is the code for APP.getWindow, which is defined in the main alloy.js file:

      var APP = {
    getWindow : function(path_to_controller, options) {
        options = options || {
            controller : false
        };
        var _controller = Alloy.createController(path_to_controller, options);
        var _win = _controller.getView();

        _win.setWidth('100%');
        _win.setHeight('100%');

        if (options.controller) {
            return _controller;
        }
        if (_win) {
            return _win;
        }
        else {
            Ti.API.error('Cannot find view: ' + path_to_controller);
        }
    }
};

      Either way, If I replace APP.getWindow("news/rss_add") with Alloy.createController('news/rss_add').getView(), I get the same result.

      Anyway, I was able to get the TableView to display in the window on my device. It seems like there is some race condition going on, as the only way I could get it to work was to add the table to the Window after window creation, and by delaying the adding of the rows to the table:

      var controller = {

    table : Ti.UI.createTableView({
        left: 0,
        top: 20,
        height: 140,
        width: '100%',
        style: Titanium.UI.iPhone.TableViewStyle.GROUPED
    }),

    rows : [
        Ti.UI.createTableViewRow({
            height: 45,
            title : 'Title',
            selectionStyle : Titanium.UI.iPhone.TableViewCellSelectionStyle.NONE
        }),    
        Ti.UI.createTableViewRow({
            height: 45,
            title : 'URL',
            selectionStyle : Titanium.UI.iPhone.TableViewCellSelectionStyle.NONE
        })    
    ],

    init : function() {
        var self = this;

        $.win_rss_add.add(self.table);
        self.rows[0].add($.input_title);
        self.rows[1].add($.input_url);
        Ti.API.info('added table');
        setTimeout(function() {
            self.table.setData(self.rows);
            Ti.API.info('added rows');
        },250); /* without this delay, the entire window would go blank */

    },

    save : function() {
        var _title = $.input_title.getValue();
        var _url = $.input_url.getValue();
        var _code = Math.random().toString(36).substring(10);
        var errors = [];

        var data = {
            title : _title,
            link : _url.toLowerCase(),
            code : _code
        };

        if (!data.title) {
            errors.push('You must enter the name of the RSS feed.');
            $.input_title.focus();
        }
        if (!data.link) {
            errors.push('You must enter the URL of the RSS feed.');
            $.input_url.focus();
        }

        if (errors.length) {
            alert(errors.join('\n'));
            return false;
        }
        else {
            Ti.API.info('save RSS info now.');
            Ti.API.info(data);
            var _id = FX.db.fxdata.saveRSS(data);
            data.id = _id;

            FX.cloud.saveRSS(data, function() {
                Ti.API.info(JSON.parse(this.responseText));                    
            });

            Ti.App.fireEvent('fx:news_reload');

            Ti.API.info('all done, close rss_add window now.');

            $.win_rss_add.close(); // getting notice: Window is opening. Ignoring this close call

        }

    }    

};

      The issue now is that when the window is closed with $.win_rss_add.close(), the window does not close, and I get this message in the console:

      Window is opening. Ignoring this close call

      This message seems to be coming from TiWindowProxy.m, line 286 here:
      https://github.com/appcelerator/titanium_mobile/blob/master/iphone/Classes/TiWindowProxy.m

      I think what is happening is that the Window is not opening properly, which causes the windowDidOpen (TiWindowProxy.m, line 99) event not to trigger, thus the "opening" attribute is never set to NO.

      It's still a mystery why the window is not opening properly. This buggy framework is really getting on my nerves.
