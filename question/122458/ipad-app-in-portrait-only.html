iPad app in Portrait only

I have an iPad app that I want to stay in Portrait orientation. It starts that way, but the orientation can still be changed. I have read many answers on this, but none of the solutions have helped or have been definitive.

I have the following in tiapp.xml:

<iphone>
        <orientations device="iphone">
            <orientation>Titanium.UI.PORTRAIT</orientation>
        </orientations>
        <orientations device="ipad">
            <orientation>Titanium.UI.PORTRAIT</orientation>                    
        </orientations>
    </iphone>

I have also put the following in each window creation:

orientationModes:Titanium.UI.PORTRAIT

I have even tried to globally set it with:

Titanium.UI.orientation = Titanium.UI.PORTRAIT;

at the very start of the app.

None of these work for me.

I am ready to submit my app except for this last issue (hopefully last). Any help will be much appreciated!

    Your orientationModes variable needs to be an array. Other than that, it should work just fine. I just tested this with 1.7.0 on the iPad:

    var win = Titanium.UI.createWindow();
win.orientationModes = [Ti.UI.PORTRAIT];
Ti.UI.orientation = Ti.UI.PORTRAIT;

var label2 = Titanium.UI.createLabel({
    color:'#999',
    text:'I am Window 2',
    font:{fontSize:20,fontFamily:'Helvetica Neue'},
    textAlign:'center',
    width:'auto'
});
win.add(label2);

win.open();

    Works beautifully.

    • Great! Ti.UI.orientation forces the orientation regardless of which orientation you are currently in. So, if someone had launched your app in landscape mode, that would force the landscape to change to portrait.

      Yes, since each window is its own entity, you have to define the oModes array for each one. Then, if for some reason you're currently not in portrait mode, you can simply force it with Ti.UI.orientation = Ti.UI.PORTRAIT.
