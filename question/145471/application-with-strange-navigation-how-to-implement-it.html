    Hi

    There is no native way of supporting the tab style you want on android. By design android tabs will slide out as soon as you add a new window to the stack.

    The navigation group suggested here is iOS only and has nothing to do with the tabs themselves.

    The problem you are trying to fix is that your design is iOS centric and in fact very simple on iOS. Android does not work the same way as iOS. Many android apps have moved towards the "action bar" context, this appears on the top of windows with a captio and/or icon with optional buttons. Even this has problems as there is no native support below android version 3. Titanium sdk 3.0 has support but only for the native version. BTW you still have to create a new action bar for each window, so they do not stay fixed either.

    A good app design should focus on the core design of that platform before trying to emulate the style of other platforms. Whilst there are many elements that share commonalities, many more do not.

    So play with some android apps and see how they provide the non-content areas, basically the navigation between windows.

    There maybe a custom module that solves your problem so worth hunting - not one I know of though (does not mean there are none).

    Good luck.

    BTW the design looks good, albeit a little too iOS centric for Android.
