Merging dynamic variable with static in xml view

Simple - I want to merge some wording in one of my views to:

<Label id="profileInfo" text="Logged in as {user.realname}" />

but that outputs exactly that instead of the dynamic realname. If I take out the 'Logged in as ' - then the dynamic works. How would I combine that so that I can do some static with dynamic?

  • You can do this in the controller using string formatting substitution. See the String.format method for that.

    You could set an Alloy.Globals variable with your string-substituted value (say, when they log in you'd build that "Logged in as __" value). You could set that label's text property equal to that global variable in the TSS or controller, though maybe not in the XML. Try it, maybe it would work in the XML.

    If you had a model and bound your label to that model, you could use substitution like you show in the XML. See the View/Model binding docs for a how-to on that.

    Those are only ways I can think of off the top of my head.
