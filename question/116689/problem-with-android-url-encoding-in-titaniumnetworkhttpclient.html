Problem with Android url encoding in Titanium.Network.HTTPClient

Hi, I am using a geo service to query info from latitude and longitude using OAuth lib authentication.

A query url is like this:

http://service.com/1.0/places/37.7645,-122.4294.json

With OAuth, I get the signed url that runs through Titanium.Network.createHTTPClient, the service responds OK on iphone, but on Android doesn't.

I have noticed, that url on Android is encoded like this:

http://service.com/1.0/places/37.7645%2C-122.4294.json

The , (comma) is encoded to %2C, and it seems that the service has a problem with this encoded url.

How can I solve this problem? Please help.

Thanks in advance.

    In the HTTPClient object setup, include the property:

    autoEncodeUrl:false,

    This property is a relatively recent addition, so be sure you are on SDK 1.6.x
