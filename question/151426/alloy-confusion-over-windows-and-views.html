    • You would get this error:

      > Uncaught Error: Requested module not found: alloy/controllers/reportlinks

      if you used the wrong name of the controller when using createController(). Are you sure you have a view/controller named "reportlinks"? It is case sensitive.

      I'm not sure I understand your question "why is one a window and why is one a view" because you created the controllers. They are that way because you chose to build them that way. Ti.UI.Window has a different API than Ti.UI.View and different overall usage. You should review those before choosing one for your controllers.

      One final note is that Alloy.createController(&#39;reportlinks&#39;).getView(); will simply return a reference to the top-level UI element in the "reportlinks" controller. If it is a view you'll need to add it to the view hierarchy:

      parent.add(Alloy.createController('reportlinks').getView());

      or if it was a window you'd open it:

      Alloy.createController('reportlinks').getView().open();

      I do highly suggest reading up on the APIs and the docs mentioned in original FAQ link as they will help make the high level concepts a little clearer.

  • Yes you can create a different window for each page, especially if you like to close these windows using the back button. in this case you need to create the windows with their modal property set to true.
    This way when the user is done with the page he can simply close it or press the back button and he'll be back to the index page

  • Windows are main containers for views and others UI objects.
    Try adding the reportlinks view to your index window with

    var report_links = Alloy.createController('reportlinks').getView();
$.index.add(report_links);

    in your Login_onClick() function
