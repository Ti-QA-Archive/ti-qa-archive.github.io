Android 'orientationchange' event not firing

When moving between tabs, the 'orientationchange' event is not fired when switching between portrait & landscape. Here's a short app.js to highlight the problem:

app.js - 30 lines

My win.js is simply this:

var detected = function(e) {
   alert('detected');
};

Ti.Gesture.addEventListener('orientationchange', detected);

Here's the reproduction steps:

  1. Switch to tab 2, and then enter landscape mode.
  2. Switch back to tab 1, rotate phone into portrait mode.
  3. The event does not occur. (Subsequent events fire as usual)

Anyone have an idea on a work-around? I am using 1.4.2, APIs 2.2. I am trying to release a large application, upgrading is not currently an option.

    • a function win.changeOrientatioon = function() {whatever} in each window
    • a global variable or property for orientation, that will be changed by each window if it detects such change through gesture listener
    • a "focus" event of tabgroup, that will call changeOrientation of the window that becomes active, to make sure, that it has updated its orientation (the window will read the global variable to know the orientation)
