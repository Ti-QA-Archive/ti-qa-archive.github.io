    Hi

    You are over complicating things here, your actual solution will likely need a scrollView and a better handle on how to use layout, Ti.UI.SIZE & Ti.UI.FILL. Also adding something that normally scrolls (tableView) to something that would normally scroll (scrollView) is always more complicated than you think.

    However there is a much easier way. Use the table you are working with and then set the header and footer views, these are designed to handle this specific job.

    var viewHeader = Ti.UI.createView({
    height: Ti.UI.SIZE,
    layout: 'vertical',
    width: Ti.UI.FILL
});
var viewFooter = Ti.UI.createView({
    height: Ti.UI.SIZE,
    layout: 'vertical',
    width: Ti.UI.FILL
});

var lblHeader = Ti.UI.createLabel({
    color: '#999',
    font: {
        fontSize: 20,
        fontFamily: 'Helvetica Neue'
    },
    height: Ti.UI.SIZE,
    text: 'I am Window 1',
    textAlign: 'center',
    width: Ti.UI.FILL
});
var lblFooter1 = Ti.UI.createLabel({
    backgroundColor: 'gray',
    height: Ti.UI.SIZE,
    text: 'Example Table',
    width: Ti.UI.FILL
});
var lblFooter2 = Ti.UI.createLabel({
    backgroundColor: 'gray',
    height: Ti.UI.SIZE,
    text: 'last label',
    width: Ti.UI.FILL
});

var tbl =  Ti.UI.createTableView({
    data: [
        { title: 'Row 1 - simple row' },
        { title: 'Row 2 -  with child', hasChild: true },
        { title: 'Row 3 -  with detail', hasDetail: true },
        { title: 'Row 4 -  with check', hasCheck: true },
        { title: 'Row 5 -  red background', backgroundColor: '#f00' }
    ],
    footerView: viewFooter,
    headerView: viewHeader,
    height: Ti.UI.FILL,
    width: Ti.UI.FILL
});
view.add(lastLabel);

win.add(tbl);

  • I think you need a height on the TableView for it to work in a vertical layout

    var w = Ti.UI.createWindow();
var view = Ti.UI.createView({
    layout : 'vertical',
});

var label1 = Titanium.UI.createLabel({
    color : '#999',
    text : 'I am Window 1',
    font : {
        fontSize : 20,
        fontFamily : 'Helvetica Neue'
    },
    textAlign : 'center',
    width : 'auto',
    height : 30,
    top : 0
});

var table1 = Titanium.UI.createTableView({
    data : [{
        title : "Row 1 - simple row"
    }, {
        title : "Row 2 -  with child",
        hasChild : true
    }, {
        title : "Row 3 -  with detail",
        hasDetail : true
    }, {
        title : "Row 4 -  with check",
        hasCheck : true
    }, {
        title : "Row 5 -  red background",
        backgroundColor : "#f00"
    }],
    top : 1,
    height : 200
});
tableFooter = Ti.UI.createLabel({
    text : 'Example Table',
    backgroundColor : 'gray',
    height : 30,
});

table1.footerView = tableFooter;

lastLabel = Ti.UI.createLabel({
    text : "last label",
    height : 30,
    width : Ti.UI.FILL,
    top : 0,
    color : 'white',
    backgroundColor : 'green'
});

view.add(label1);
view.add(table1);
view.add(lastLabel);

w.add(view);
w.open();

    • @Elisa I am glad you have solved your issue, but just to clarify, I provided a full solution and not a simple suggestion on fixing your existing method. Ti.UI.SIZE was always going to fix the problem on iOS (platform was not known when I provided answer). As you will note from my answer I said you needed a better handle on how to use those constants. What I provided was a better cross platform answer to the layout that utilises the features already available within the tableView rather than stitching things together.

      Keep in mind android will likely have a problem with this if you ever translate it - at which point my solution will solve that. Also keep in Ming a tableView is a scrollView with extras. So you have a scrollView inside a scrollView - fine for iOS but a problem for android.

      You should also add a property scrollable: false on the tableView to play it safe - something else android does not support.
