    iOS is pretty easy going about casting. To iOS true, 1, !false all mean the same thing. You can move back and forth from string values to numeric values and often get away without converting them. Android on the other hand, isn't so friendly. It expects color strings to be certain lengths, it expects a boolean to be a boolean.

    Looking at your errors, seem to be some casting issues going on.
