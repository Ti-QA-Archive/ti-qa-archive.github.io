Toggling tabs on and off in ios app

I'm using Titanium with alloy and have a tab based app for ios. In my "settings" tab I give the user the option to show or hide certain tabs. I want the tab to immediately appear or disappear as they make their choice. Their preference is stored in properties so it will be remembered when they restart the app.

The problem with what I have now, is that once the app is running and the tab is turned on (or off, then back on), when you go to that tab the view is now in front of everything so that you can't see the tabs and can't navigate anywhere.

<TabGroup id="tabGroup" backgroundColor="#e1e1e1"> 
    <!--Scenes TAB --> 
        <Tab title="Scenes" icon="/tabicons/scenes.png" id="scenesTab">
            <Require type="view" src="scenes" id="scenesTabWin"/> 
        </Tab>
   <!--Settings TAB  -->
        <Tab title="settings" icon="/tabicons/settings.png" id="settingsTab">            
            <Require type="view" src="settings" id="settingsTabWin"/>    
        </Tab>
</TabGroup>

and here's the controller that responds to the fire event from settings

//toggle scenes tab when switch changes in settings
Ti.App.addEventListener('toggleScenes', function() {
    var scenesTab = $.scenesTab;    
    if(!Ti.App.Properties.hasProperty('ShowScenes') || Ti.App.Properties.getString('ShowScenes')==0){
    $.tabGroup.removeTab(scenesTab);
}else{
    $.tabGroup.addTab(scenesTab);
}
});

I've tried making the scenes window smaller so you can see the tab bar, but the window stays there when you navigate to the other tabs. So then I tried closing the window when you leave the scenes tab, but then it's gone for good. These all seem like hacky solutions anyway, what's the right way to do this?
