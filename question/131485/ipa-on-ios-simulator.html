  • Hello,

    when you compile for the simulator, you will get code for intel processors, because will be actually running using your intel processor and can access most of your RAM. That's actually the reason the apps runs really fast in the simulator, and they almost never crash because of issues with memory.

    When you compile for device, you will get a binary for ARM, so, when you try to run that code in the simulator, it will simply won't work.
