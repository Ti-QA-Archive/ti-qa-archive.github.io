    What do you mean 'return back to the web view' ? If the link in the web view directs you to another url you are still 'inside' the web view.

    If you require browser type navigation, I think you have to roll your own using the goBack(), goForwards() etc functions of the webview, fired from a series of UI elements.
    In other words you need to fake the browser!
