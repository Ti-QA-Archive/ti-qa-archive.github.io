Abandoning Ti Studio

I've decided to go back to TI Developer, as I'm spending way too much time fighting with TI Studio. Since making the transition my productivity has gone way down and I'm giving up for now. Developer was too lean but Studio is definitely too bloated.

I'm finding the UI to be awkward and unintuitive. I like the integrated editor and code completion, but everything else really gets in my way. Some of the problems I've had:

1) I'm not getting any syntax error messages to display on the console.
2) Unable to run w/o the debugger. Any time I hit RUN, trying to avoid the debugger (which I dislike), I get a TI error indicating that its unable to connect.

Anyone else throwing in the towel with TI Studio?

  • Ditto, and to make things worse Developer is, from what I can tell, no longer available on the download page, though I think you can still grab it from github and build yourself.

    I am surprised that they are pushing the Ti Studio so hard considering, among other things, that all the training videos still only cover usage of Ti Developer. I think forcing on IDE on developers is a really bad idea that likely will not last long, especially if we all continue to voice our opinions. Lack of freedom to choose a text editor is often a deal breaker in selecting a job for many developers, many will just secretly just use their own. A favorite question of mine to ask in interviews is "what is you favorite text editor or IDE?", because good developers have a passion for theirs.

    Ultimately there is no reason the build and edit environments can't be separate. My feeling is that Appcelerator should either continue to support Ti Developer or at least provide full documentation to the command line build/test/deploy interface as an alternative to using Ti Studio.

    my 5¢
