Mapview on iOS7 crash on orientation change

This is kind of a two fold issue. I have Xcode5 with Titanium 3.1.3.

I am building for iOS7 and have a view which is a Ti.Map. I made it exactly off the 3.x documentation, including the onComplete function. If regionFit == true, the application crashes with an error along the lines of 

[ERROR] :  The application has crashed with an uncaught exception 'NSInvalidArgumentException'.
[ERROR] :  Reason:
[ERROR] :  Invalid Region <center:+37.78134027, -122.40641785 span:-0.00454454, +0.01774999>

If regionFit == false, I no longer get crashes on orientation change, however the map zooms out and shifts when I change orientations (I tested this by centering the user location and rotating device. the user location doesn't stay centered and wanders off screen after 1-2 orientation changes. The user location isn't actually moving of course, but rather the map is shifting around).

Any help would be great. It should be noted that this issue doesn't exist when we build for iOS 6.1, but rather just iOS7+ (including the map moving on orientation change, that's just very strange that we cannot reproduce in a 6.1 environment)
