ImageView 'image' instead of 'url' causing pixelation...

Hi,

I am building an app for iPhone.

I have created a tableView with custom rows. within these rows there is an imageView displaying a small icon.

I am receiving warnings stating that declaring the image path with 'url' is no longer supported and that I should use 'image' instead. However, if I change 'url' to 'image', the image looks heavily pixelated on the iPhone4's retina screen. It doesn't if I use 'url'.

Here is my code for the imageView:

var leftIcon =  Titanium.UI.createImageView({
        image:'../icons/face1.jpg',
        width:26,
        height:26,
        left:8,
        top:8
    });

The actual size of the JPG file is 120x120, so it's plenty big enough to look crisp on the retina display at 26x26. And it does if I use 'url' to point to it.

Can I keep using 'url'? or is there a way to use 'image' instead without it changing the quality?

Any advice on this would be really useful… Thanks…

  • I think I can answer my own question here. Renaming my JPG using iPhone's special '@2x' did the trick. It seems that when using 'image' rather than 'url', the iPhone4 will only render the graphic at double resolution if it has '@2x' at the end of it's name…

    var leftIcon =  Titanium.UI.createImageView({
        image:'../icons/face1@2x.jpg',
        width:26,
        height:26,
        left:8,
        top:8
    });

    seems so obvious now!..
