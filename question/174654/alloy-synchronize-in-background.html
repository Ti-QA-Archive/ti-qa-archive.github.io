    • There are three states of an app;

      • Foreground - app is the one currently being displayed to the user
      • Background - the device has gone to sleep or another app has become the one currently being displayed
      • Closed - the app has been removed from the task manager, shut down in some way or never been started

      Your code can run when in foreground, in background if it uses the background services but not when the app is closed. If the app is closed it is not in memory and so has no ability to run any code.

      So you if you mean closed as in not running, rather than closed means not in foreground then - No. You cannot run any code in your application when the app itself is not running.
