    David,

    The when a new version of a tool is released, it is best to check the compatibilty matrix to see which Mobile SDK version matches that tool. The 2.0.1 Mobile SDK is the only one to currently support xCode 4.3. May I suggest that you update your TiStudio to 2.0.x and your mobile sdk to 2.0.1 to use against your xCode 4.3 installation. Since the Titanium tools have been updated, it doesn't make much sense to me to be referencing 1 year old hack threads to resolve your current problems.

    • This has been a hard problem to keep track of as it can be:

      • Random occurrence for a particular developer.

      • Can be constant occurrence for a developer.

      • Is not an occurrence for every developer.

      The other problem that occurs is that the error doesn't always mean the same thing:

      • It could be as simple as a case problem in a file/path name

      • It could mean that xCode is looking to build into the wrong location.

      The problem is exacerbated by Apples tooling changes from 3.x -> 4.3:

      • In 3.x, builds by default went into one location
      • In 4.x, we had a nice drop down option in preferences to behave like 3.x
      • In 4.2, that option kind of/sort of disappeared.
      • In 4.3, the tooling install point moves.

      Lastly, if you have search the Q&A, you will find that the majority of the questions asked about this have been abandoned eventually by the seeker without all the information coming out or a resolution being found.
