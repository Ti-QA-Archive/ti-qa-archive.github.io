I'm very confused with the Backbone integration in Alloy

As an old user of Backbone.js, i'm confused with the Backbone integration in Alloy.

First of all : how can i create a model class by extending an other model class ?

Example in standard Backbone.js :

var MyClassA = Backbone.Model.extends({

    customProp : 'foo',

    doStuff : function() {
        return 'hello';
    };

    validate : functin() {
        return maybe();
    }

});

var MyClassB = MyClassA.extends({

    validate : function() {
        return MyClassA.prototype.call(this) || bar();
    }

});

I don't know how to do this in Alloy when i declare an Alloy Model.

I also have a little problem with the given examples : 

extendModel : function(Model) {
       _.extend(Model.prototype, {
           // Do your stuff here               
       });

       return Model;
    },
};

I don't understand why the example use _.extend instead of Model.extends.

  • The _.extend is referring to the underscore.js extend() method, used to extend the Model.prototype object with the properties you specify in the second argument.

    Alloy uses just the models/collections portion of backbone and provides some wrapper functionality to make those structures fit within the rest of our framework. You could check out our docs for more info, though they're admittedly a bit sparse on this topic. You could also look at our code to see what's up. https://github.com/appcelerator/alloy/.
