iOS 6.0 packaging issue

Hello,

Titanium Mobile SDK: 2.1.3GA

Platform & Version: iOS 6.0

Titanium Studio: 2.1.2.201208301612

Host Operating System: OS X 10.7.4

We are having an issue while trying to distribute an app to the app store. After packaging, when we try to validate, we get an error message that says

###"This bundle is invalid. The application-identifier entitlement is not formatted correctly; it should contain your 10-character App ID Seed, followed by a dot……."

I see that there is an open ticket for the same: Ticket Link

Is there any workaround for this until it gets fixed. We have apps that need to be out at app store and any help in this regard is greatly appreciated.

  • You're supposed to have a separate app id per app, I know things like Push certificates require these.

    Also, you don't have to change the app id for the distribution certificate. You just make a new provisioning profile for each new app, choose your existing certificate, and the app id you created for that app.

    — commented October 4th 2012 by Josh Lewis

We already have several apps in app store using the wild card distribution provisioning profile. It is only after updating to iOS 6 sdk that we are facing this issue.
