Alloy widget problem

I am testing the highchart with Alloy. Here is the sample project link

  • high chart Alloy widget

Here is the titanium blog :

  • interactive-charts-in-appcelerator-titanium

it runs fine inside iOS, however, it gives below exceptions in android with 4.4.2

[ERROR] :  AndroidProtocolHandler: Unable to open asset URL: file:///android_asset/Resources/html/com.alco.highcharts/jquery.min.js
[ERROR] :  AndroidProtocolHandler: Unable to open asset URL: file:///android_asset/Resources/html/com.alco.highcharts/highcharts.js
[ERROR] :  AndroidProtocolHandler: Unable to open asset URL: file:///android_asset/Resources/html/com.alco.highcharts/exporting.js

All the js files are under widgets/com.alco.highcharts/assets/html/ , they are referenced by a html file under the same directory

<script src="./jquery.min.js"></script>

<script src="./exporting.js"></script>

    Hi

    I have NOT run the app, this is a quick look and my aim is to give you two suggestions to check in order to see if either is the issue;

    • Suggestion 1: Relative file location

      In my experience relative urls on web sites can get confusing. This line of html code tells the file system where to look;

    <script src="./jquery.min.js"></script>

    The error thinks that location should be here;

    file:///android_asset/Resources/html/com.alco.highcharts/jquery.min.js

    To check, we need to reset everything;

    • BACKUP YOUR APP CODE
    • Run the Menu > Project > Clean > Choose project > Confirm choice
    • BACKUP YOUR APP CODE (just making sure)

    • ONLY ON ALLOY

      • delete the contents of the resources folder

      Then run the app. The same error should occur - if it does not - sorted (unlikely). Navigate to the following location;

    • {appname}/resources/html/com.alco.highcharts/

      Does it contain the file jquery.min.js? If not then that is the problem - the relative folder location. Find the file and adjust the locations.

    • Suggestion 2: Alloy is pre-processing the JavaScript files

      Alloy does some amazing things with your app JavaScript files, seriously - spend some time reviewing how your code is tweaked inside the processed resources directory one day. However it normally pre-processes all JavaScript files. So it may also be doing the same here.

      Solution; rename the file extension and update the file references.

      jquery.min.js becomes jquery.min.jsf (does not matter what extension you choose - just not .js

      Update the reference for the html page;

    // this
<script src="./jquery.min.js"></script>
// becomes
<script src="./jquery.min.jsf"></script>

    Just a couple of suggestions to try.
