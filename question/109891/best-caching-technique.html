Best caching technique

Hi, in my app the user can request details of a given place. Each request currently leads to an HTTP request and json response. To improve performance I'm considering storing the request/response pair locally as a sort of cache. I can use the Preferences API, filesystem or DB. The Preferences seem to be the most simple - are there any factors that would make you consider the other options?

  • Factors that would make me consider a database:

    • Performance. A properly indexed database can search very fast.
    • Scalability. Database performance can handle large volumes easily.
    • Cache TTL. With a database, it is easier to add a field to calculate an expiration timestamp based on some Time To Live criteria. While you could also put a TTL/expiration in a json response, with a database you can remove all expired entries easily with a single SQL DELETE statement.
    • Prepopulate. It would be easier to either install or download a set of common/default places.

    Those are just some reasons off the top of my head.
