    Hi, Here is the complete code:

    var array=[];

var txtfld = Titanium.UI.createTextField({
    top : 10,
    width :100,
    left : 10,
    hight : 50,
});

 var btn = Ti.UI.createButton({
        title: '"Click Me To Insert',
        top: 200,
        height: 50
    });

btn.addEventListener('click',function(e){
array.push(txtfld.value);
Titanium.App.Properties.setList('MyList',array);
});

    • @Muhammad; I would not avoid using any of the Ti.App.Properties methods because you think they will cause a memory leak. Whilst filling up a significant amount of them with a significant amount of data is not smart, it is unlikely to cause a memory leak - just a full device.

      There are many good reasons for their use and you would have to do something amazing for a memory leak to occur.

      As Sudeep has not provided any scope to the size of data he wants to store I am going to assume it will be low given a button press is the interval of new data - so whilst a database is very useful it "may" be overkill, but as I say - no idea until he provides a little more information.
