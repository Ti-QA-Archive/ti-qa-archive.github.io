Cannot Locate Custom Module on Lion?

I'm using Studio on Lion (latest version of everything). I'm trying to work on a project with ti.paint - but I seem to be running into a problem. I've moved over the ti.paint folder into:

/Library/Application Support/Titanium/modules/iPhone/

I then included this in the appxml:

<modules>
<module version="1.0">ti.paint</module>
</modules>

My app is still not picking up the module, however (I've forced a full rebuild already).

Question: since I'm working on an iPad app - does the module need to create an iPad directory instead of the iPhone one? Is there an issue with Lion not being able to read the Library directory? When I went to install the module originally, there was no "Titanium" folder inside Application Support (I had to create it all). Any help would be awesome. Thanks!

  • Doesn't sound too wrong.

    I guess I'd try to install the module again. So remove the ti paint folder from the iPhone folder and put the zip for the module in the /Library/Application Support/Titanium directory and let Titanium handle the install at the next run. Also, just to make sure I'd set the platform attribute for the module (even though it doesn't seem necessary here):

    <module platform="iphone" version="1.0">ti.paint</module>

    And then clean everything and build again.

    I also run Titanium Studio on Lion and my app is universal iPhone/iPad and I did not have problems with module handling.
