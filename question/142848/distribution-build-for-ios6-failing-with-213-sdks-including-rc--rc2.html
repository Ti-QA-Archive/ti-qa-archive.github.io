  • Turned out to be an obscure bug in the JavaScript compiler - the same app ran fine on the iPhone and Android emulators but failed when trying to package for iPhone because the JavaScript compiler couldn't handle this…

        var divisor = (boothGroup.evenhigh - boothGroup.evenlow + 1) 
            / boothGroup.rows;

    Apparently, the compiler parses the first line as a complete arithmetic statement and then the second line as a malformed comment and causes the resulting error messages that I posted.

    Seriously, this is what caused it. The formatter had split the statement on to two lines.
    This was in the smallest code file of a huge app - how we found it was a miracle.

    Had nothing to do with modules or anything. I had added the file with this code in between when it packaged OK and when it didn't, and as I said it ran fine on both emulators so that's why it was a real head-scratcher.
