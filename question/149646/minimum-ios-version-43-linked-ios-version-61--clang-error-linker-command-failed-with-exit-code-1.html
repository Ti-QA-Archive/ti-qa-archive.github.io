Minimum iOS version: 4.3 Linked iOS Version 6.1 / clang: error: linker command failed with exit code 1

I am using xcode 4.6 , iOS 6.1 and Titaniums SDK 3.0.2.GA release. I am porting a new module for titanium which requires linking of below 3 ios libs. The ios native module works perfectly with ios Native apps.

I am just trying to port the ios native module to Ti module so that I can use it with Ti.
CoreTelephony.framework
SystemConfiguration.framework
libz.dylib

I have added above libs and compile the module. It does generate the zip for Ti module. When I try to use this module in a sample titanium app (or simply use 'titanium run' i.e default Ti app of module), I gets below error and applications terminates.

[INFO] Minimum iOS version: 4.3 Linked iOS Version 6.1
[ERROR] clang: error: linker command failed with exit code 1 (use -v to see invocation) [ERROR]
[ERROR] Error: Traceback (most recent call last):

Any pointer?

  • This may help someone. I changed my tiapp.xml, min-ios-ver to 5.0 and made sure that my modules were loading the correct version, ie:

    <module version="2.0.2">ti.storekit</module>

    That fixed the issue for me. I was using SDK 3.1.1 and iOS SDK 6.1.
