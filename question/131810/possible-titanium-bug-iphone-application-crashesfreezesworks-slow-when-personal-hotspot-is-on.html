Possible Titanium BUG: iPhone application crashes/freezes/works slow when Personal Hotspot is ON.

I am developing an application that uses GPS, accelerometer and heading data.
I don't have SIM card in it so I am not on internet with my iPhone.
Nowhere in my app I don't use network data.

Now I've put SIM card in my iPhone and application started to act slow and also it freezes and crashes.

I know that I didn't change anything in my code so I started thinking what was different from the last usage (when application was working normally).
I remembered that I've put a SIM card in it so my iPhone is on internet now.
I went to settings and turned off 3G and EDGE and application was again working normally.

Eventually I would need network communication in my app so this was not the solution.

So I remembered that I had internet tethering (Personal Hotspot) turned ON. I turned it OFF but 3G and EDGE was still ON and my app is working normally.

This looks like a Titanium SDK bug because I don't see how Personal Hotspot could possibly influence my application.

iPhone 4

iOS 5.0

Titanium SDK 1.8.1
