  • Hello, Samie, According to the LINK here it states that "For assets, such as images, any references to the Resources folder in the Titanium SDK documentation should be replaced with the app/assets folder to use it for Alloy. For example, the Icons and Splash Screens guide tells you to place files in either the Resources/android or Resources/iphone folder. For Alloy, the files should be placed in either the app/assets/android or app/assets/iphone folder.".

    So when you create a new alloy project you would see a default assets folder which containg both ratina and non ratina images.

    If you use non ratina image as default and if the ratina image of that is in the same directory then in ratina devices it will take the ratina images. Like.

    var view = Titanium.UI.createView({
   backgroundImage:'/assets/iphone/bg_bar.png',
   width:320,
   height:45
});

    Then if bg_bar@2x.png is in there, then ratina device will use this image by default. Thanks.
