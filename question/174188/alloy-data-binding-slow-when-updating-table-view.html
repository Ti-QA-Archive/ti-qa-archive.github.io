Alloy Data Binding slow when updating table view.

It appears that when I call 'Alloy.Collections.library.trigger('change');' the entire table view is redrawn which can be slow and jerky on large collections.

In my code I used the infinite scroll widget (https://github.com/FokkeZB/nl.fokkezb.infiniteScroll/) to add 20 items to the collection at a time. I then call 'Alloy.Collections.library.trigger('change');' to update the view. I have 'silent : true' so backbone doesn't trigger change on every model that I added.

Is there a way to have alloy update a table view more intelligently by appending / inserting rows instead of recreating the entire table? Being able to define transition effects would also be nice.

Or is my only alternative to do this all by hand and not use the built in data binding feature?
