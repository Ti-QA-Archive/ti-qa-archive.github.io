Geocoding in iOS6

Hello,

There was(is?) a problem with Titanium geocoder – it would hit Google's daily limit since ultimately Titanium uses one account for all devs (or something like this, not sure exactly how it works).

This was easily solved by directly using Google's apis and not using Titanium built-ins for this.

This seemingly won't effect anything since those goecoding calls don't depend on os, be it 5 or 6, ios or android. However, Google states in its geocoding api policy, that they're only to be used for the purpose of displaying a poi/address on Google maps. So as soon as Google maps are no longer used on iPhone in any of the apps theoretically we'll be violating the rules.

Does Appcelerator plan a different approach to its built-in geocoding services? Will it still be using Google's apis and hit that daily limit, or will it work somehow differently so that we can stop using Google's apis and start using Titanium's?

Thanks,
Zhenya
