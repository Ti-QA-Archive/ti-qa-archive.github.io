  • I don't have a Linux box to test. So, this will be a bit of guessing.

    • Is ADB hung? Try killing & restarting with adb kill-server followed by adb devices There's no need to kill the emulator before running this command – in fact, don't shut down the emulator unless you have to. Sometimes it takes so long for the emulator to start that Titanium thinks the emulator is dead and gives up. Sometimes the ADB process hangs and cannot communicate with the emulator.
    • Can you run the emulator directly (from the android SDK manager tool)? If not, something is wrong with your Android / Java setup.

    If it comes to it, the build scripts create an installable APK file. You can use the adb install command to install the app manually to the emulator. Or, if you have a device, simply build to the device rather than the emulator. You'll get a faster-running app and a higher fidelity test on a device than the emulator.

    • Windows 7 - Titanium 3.1.3 - After much struggle, this answer put me on the right track, thanks. I had a problem running kill-server in the Command Line but I then found a fix here:

      http://stackoverflow.com/questions/15201344/how-to-write-adb-commands-in-windows-7-command-prompt

      following Sean O'Toole's advice and gradually the emulator began to open apps for me:

      Add adb to your PATH and you'll be able to use adb commands anywhere in cmd, which is helpful for building apps and killing/starting the server quickly.

      To do so, right-click on Computer in the start menu and click properties. Next, click on Advanced system

      settings on the left hand side of the window. Next, click Enviroment Variables on the window that pops up.

      From here you get a list of user variables and system variables. Add a new system variable called ADB_HOME, with the value as the path to the platform-tools folder for your android sdk. Next, find the Path variable and add %ADB_HOME% to this string of values.

      You'll now be able to use adb from anywhere.

      So I then Ran in Command Line:

      adb kill-server adb start-server

      and without restarting the already loaded Emulator, I hit Run in Emulator and it worked. I think what Tim Poulsen says here about 'Titanium thinking that the Emulator is dead and giving up' is right, not surprising on my slow computer.     • As another followup to your first point, at your shell, launch android which is the all-purpose Android development environment management tool. Make sure you have downloaded the most recent Android tools (checkboxes near the top) and the Android SDKs you want to use (which your initial comment says you probably have). Then, choose Tools, Manage AVDs. This is your virtual device (emulator) management tool. Use this tool to create and then run an emulator. If it doesn't run here, then the issues are likely to be with your Java or Android tools configurations, or perhaps permissions or path issues.

      Titanium creates its own AVDs, you might even see one listed if the build scripts got that far before throwing the errors. When Studio launches the AVDs, it does so on a custom port. So, you can't really launch the AVD from this tool, then build & install to it from Studio. But you can at least confirm that you can launch an AVD which means the Android tools are installed correctly.

  • The fix is to go into the Android SDK Manager and install "Android 3.0 (API 11)". Make sure the app uses emulator "Google APIs (Android 2.3.3)" and "WVGA854". I assume there's a Titanium bug because you have to install a higher API level (3.0) than is actually used (2.3.3). Using exactly these settings, Kitchen Sink works as expected.
