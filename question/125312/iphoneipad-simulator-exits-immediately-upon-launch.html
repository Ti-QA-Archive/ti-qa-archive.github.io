iPhone/iPad simulator exits immediately upon launch

I can't seem to get the iPhone/iPad Simulator to get up and running on my mac. The simulator window briefly comes up for less than a second, then immediately closes without warning or error messages. I'm suspecting the red "permission denied" warning is a good lead, but I don't know how to follow it. This is for an unchanged new Mobile Project.

Console output:

[INFO] One moment, building …
[DEBUG] Detecting modules in /Users/myusername/Documents/Titanium Studio Workspace/twitterapp/modules
[DEBUG] Detecting modules in /Users/myusername/Library/Application Support/Titanium/modules
[INFO] Titanium SDK version: 1.7.2
[INFO] iPhone Device family: universal
[INFO] iPhone SDK version: 4.3
[INFO] iPhone simulated device: iphone
[DEBUG] executing command: /usr/bin/killall iPhone Simulator
[DEBUG] No matching processes belonging to you were found
[DEBUG] finding old log files
[DEBUG] executing command: mdfind -onlyin /Users/myusername/Library/Application Support/iPhone Simulator/4.3 -name 50b54543-42ed-4b4c-8f55-54ec8c2cd5f3.log
[INFO] Launching application in Simulator
/bin/sh: /Users/myusername/Library/Application Support/Titanium/mobilesdk/osx/1.7.2/iphone/iphonesim: Permission denied
[INFO] Launched application in Simulator (3.79 seconds)
[INFO] Application has exited from Simulator

I'm running OS X 10.7.1 and Titanium build 1.0.4.201108101535 / SDK version 1.7.2

Assistance would be greatly appreciated.

    Does your iphonesim executable currently have execute permissions? Run this to make sure:

    chmod +x /Users/myusername/Library/Application Support/Titanium/mobilesdk/osx/1.7.2/iphone/iphonesim

    Do you get a permission denied error when yo urun the iphonesim manually as well, meaning not through Titanium Studio?
