Images display issue on version 3.0 in ios 6

I upgraded titanium to version 3.0 and using ios 6 for my app. My app was working fine on version 2.3. After up gradation it is giving a issue. In version 2.3, the images folder was getting integrated in XCODE and all images were displayed in simulator as well as iphone device. Now in 3.0 version, the images folder is not getting integrated in XCODE and all images are displayed in simulator but none on iphone device. Waiting for reply.

    Hi

    I wonder if this is related to HOW you reference images.

    If you do something like this;

    var img = TI.UI.createImageView({
    height: Ti.UI.SIZE,
    image: '/iphone/images/example1.png', // important line
    width: Ti.UI.SIZE
});

    It will not work on the devices, tweak it to the following.

    var img = TI.UI.createImageView({
    height: Ti.UI.SIZE,
    image: '/images/example1.png', // important line
    width: Ti.UI.SIZE
});

    This will display the image on both the simulator and the device.

    If this is not the problem, then please provide a code snippet showing the path of a typical image, like I have above and also simple tree structure of where that image actually appears, like the one below to illustrate the location of the image used in my code.

    /{appname}/resources/app.js
/{appname}/resources/iphone/
/{appname}/resources/iphone/images/
/{appname}/resources/iphone/images/example1.png // this is the physical location
