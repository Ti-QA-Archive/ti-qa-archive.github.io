  • Hi

    If you are looking for a completely different way to access the image so you can change it then create a new function that responds to your call from within the event handler, from here you can access everything;

    I provided this technique as an answer yesterday to a similar but different question tableviewrow-click-highlighting-children-views.

    This was the example I provided, you appear capable of understanding how to tweak for your own needs but ask if you need pointers given the differences.

    function addRow(obj) {
    var row = Ti.UI.createTableViewRow({
        type: 'row',
        className: 'standard',
        height: Ti.UI.SIZE,
        isSelected: false,
        selectionStyle: Ti.UI.iPhone.TableViewCellSelectionStyle.NONE,
        width: Ti.UI.FILL
    });
    var view = Ti.UI.createView({
        backgroundColor: '#fff',
        height: 60,
        left: 0,
        touchEnabled: false,
        width: Ti.UI.FILL
    });
    row.add(view);
    var img = Ti.UI.createImageView({
        height: Ti.UI.SIZE,
        image: 'checkoff.png',
        left: 10,
        touchEnabled: false,
        width: Ti.UI.SIZE        
    });
    row.add(img);    
    var lbl = Ti.UI.createLabel({
        color: '#000',
        height: Ti.UI.SIZE,
        highlightedColor: '#fff',
        left: 70,
        text: obj.title,
        textAlign: 'left',
        touchEnabled: false,
        width: Ti.UI.SIZE
    });
    row.add(lbl);
    var viewButtons = Ti.UI.createView({
        height: Ti.UI.SIZE,
        layout: 'horizontal',
        right: 10,
        width: Ti.UI.SIZE
    });
    row.add(viewButtons);
    var btn1 = Ti.UI.createButton({
        type: 'btn',
        height: 40,
        left: 10,
        title: 'One'
    });
    viewButtons.add(btn1);
    var btn2 = Ti.UI.createButton({
        type: 'btn',
        height: 40,
        left: 10,
        title: 'Two'
    });
    viewButtons.add(btn2);
    row.toggleSelected = function () {
        var isSelected = !row.isSelected;
        Ti.API.info('isSelected', isSelected);
        row.isSelected = isSelected;
        img.setImage(isSelected ? 'checkon.png' : 'checkoff.png');
        view.setBackgroundColor(isSelected ? '#ccc' : '#fff');
    };
    return row;
}
var data = [
    { title: 'One' },
    { title: 'Two' },
    { title: 'Three' },
    { title: 'Four' }
];
var intRow = 0, intRows = data.length, row, rows = [];
for (intRow = 0; intRow < intRows; intRow = intRow + 1) {
    row = addRow({
        title: data[intRow].title
    });
    rows.push(row);
}
var tbl = Ti.UI.createTableView({
    data: rows,
    height: Ti.UI.FILL,
    minRowHeight: 60,
    width: Ti.UI.FILL
});
tbl.addEventListener('click', function (e) {
    if (e.source.type === 'row') {
        e.source.toggleSelected();
    }
});
win.add(tbl);

    The important bits are here;

    function addRow(obj) {
    var row = Ti.UI.createTableViewRow({
        type: 'row',
        className: 'standard',
        height: Ti.UI.SIZE,
        isSelected: false,
        selectionStyle: Ti.UI.iPhone.TableViewCellSelectionStyle.NONE,
        width: Ti.UI.FILL
    });
...
    row.toggleSelected = function () {
        var isSelected = !row.isSelected;
        Ti.API.info('isSelected', isSelected);
        row.isSelected = isSelected;
        img.setImage(isSelected ? 'checkon.png' : 'checkoff.png');
        view.setBackgroundColor(isSelected ? '#ccc' : '#fff');
    };
    return row;
}
...
tbl.addEventListener('click', function (e) {
    if (e.source.type === 'row') {
        e.source.toggleSelected();
    }
});
...

    The function is attached on each row and you can then call that function from the listener, whilst my example does not show passing values in - this is very possible as well. I use this technique in most of my apps where complicated (detailed) row layouts that are easily updated after the fact.

    To pass variables in;

    ...
row.something= function (obj) {
    image.setImage(obj.image);
    lbl.setText(obj.title);
};
...
e.source.something({
    image: 'example1.png',
    title: 'title1'
});
...

    Enjoy

    • Hi

      This is not working because I do not think you understand exactly what source is.

      If you have a tableViewRow with an image and two labels then source will be WHICHEVER one you actually interact with, if you click the image then image is the source, if you touch an area of the row that is not covered by anything else then source is the tableViewRow.

      If you notice form the example I originally supplied the technique is dependant on the use of touchEnabled: false on each of the child views so that events from those are simply ignored because they are not needed, this way our source is ALWAYS the row.

      From here you can then call the test function toggleSelected I included to perform the work INSIDE the row by calling the function on the event listener.

      Follow the code I provided and you will see.
