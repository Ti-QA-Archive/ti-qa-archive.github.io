  • An update on our investigation into this issue.
    I believe the issue is related to the Titanium Studio "Install from URL"
    method for installing the SDK. I have opened a query into this with our
    Studio team. Until then a work around is to unzip the SDK manually. The command line operation goes like this for Mac OS X:

    unzip -o desktopsdk-1.2.0.RC3-mac.zip -d "~/Library/Application Support/Titanium"

    This should unzip the SDK correctly and override the
    copy Studio installed incorrectly. Please let me know if this does
    not resolve your problem.
