No iPhone Build Folder

Hello -

For some reason when I try to build my app to the iPad, I get an error message System Exit 65. I have seen multiple questions regarding this, and the answers are as varied as it comes. So, the next thing I was going to do was take my xproj out and just deliver it directly to xcode, and this led me to find that I have no build folder in my app. I used to have these iPhone build folders with the xcode translations, and then they just seemed to stop. Is there any clues as to what I did wrong, and how I can fix it?

Thanks!

  • Accepted Answer

    Go into your app root via terminal and type.

    rm -rf build/iphone

mkdir build/iphone

    Try running it again. You will first receive the Exit 65, then run it again, and it will successfully build. Hope this helps.

    • Try and leave comments to the answer, instead of posting a new answer. Members here kind of get annoyed by that. Anywho, your app root is where ever you created your application. Titanium Studio defaults to ~/Titanium. In terminal you can type:

      cd ~/Titanium

      Then in this directory type.

      ls

      You should see your project in there. Simply "cd" into your projects directory. You are now in it's root. Then run the above "rm -rf" command.

      Cheers.
