Pass touch event to new window - iOS

The aim is to allow a user to hold down a table row, a new tranlucent window will pop in with a slider placed in the same possition as the row.
What i need is to pass the touch event (that is currently on the tableview), straight to the slider which the user is now touching 'over'. However, by default, moving your finger on the screen moves the table underneath the newly opened window.

obviously letting go and then touching the slider works, but it is this step that i want to remove.

I dont want to bloat the table by adding a slider underneath every row etc etc.

The user should be able to hold down the row, slide the slider that has popped up and release to hide it, seemlessly and in one move.

Is this possible? Im at a loss with this one!

Im not sure examples of my current code would help here, it is a simple tableview with a window containing a simple slider opening on touch start. But if needed just give me a shout.

    What I would suggest is simulating the entire thing. Place all of the Sliders ontop of the TableViewRow on a View with 0.0 opacity, then when the user taps on that row it animates the SliderView to show, and then use the TableViewRow's touchstart and touchmove to calculate the difference between the start and current position of the touch (Only the X coordinate), and then move the Slider accordingly (After mapping it from 0-platformWidth to 0-1). When touchend is called, then hide the SliderView.
