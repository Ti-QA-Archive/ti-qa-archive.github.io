Scrollableview - view preload?

Hi,

I'm creating a book for iphone where each page is a view within a scrollableview object. The views just contain a background image.

I'm using iPhone 3G.

Everything is ok in the simulator. But on the device the page transition is not smooth. When I start to swipe the finger to change the image I can see the next image appearing. But when it gets to the middle of the screen it stops for a while. Then it continues with the transition. It is maybe loading the next views.

It is ok if I have just 4 views but in my case I have more views and it doesn't go nicely.

This is how I do it:

inline code

var win = Titanium.UI.currentWindow;

var view1 = Ti.UI.createView({
    backgroundImage:'../images/1.png',
    voice: '../sounds/1.mp3'
});

....

var view10 = Ti.UI.createView({
    backgroundImage:'../images/10.png',
    voice: '../sounds/10.mp3'
});


var scrollView = Titanium.UI.createScrollableView({
    views:[view1,view2,view3,view4,view5,view6,view7,view8,view9,view10],
    currentPage:0
});

win.add(scrollView);

This code lives in a separate js and belongs to a window.

var pages = Ti.UI.createWindow({
    url:'main_windows/pages.js',
    height:Ti.Platform.displayCaps.platformHeight,
    width:Ti.Platform.displayCaps.platformWidth,
    fullscreen:true,
    navBarHidden:true,
    orientationModes: [ Titanium.UI.LANDSCAPE_RIGHT ] 
});

So when pages window is opened the scrollableview is used.

Any ideas how can I improve the view transition on the device?
I think this topic has been discussed before but I couldn't find an answer how to fix it.

Thanks.

  • If this hasn't yet been answered, I would like to help. I have an app that has the need for multiple views in one window, so I used a scroll view with a table inside it. Then I just used a loop to create the rows. Looping is always better than typing 300 lines of code for multiple items.. for example:

    for(i=0;i>10;i++)
{
var view[];
    view[i] = Ti.UI.createTableViewRow({
    backgroundImage:'../images/'+i+'.png',
    voice: '../sounds/'+i+'.mp3'
});
}
