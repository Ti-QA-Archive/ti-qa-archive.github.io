Sending limited push notifications information to individual users

Hi,

I am currentrly writing a football live score app and im struggling to figure out how Im going to implement notifications, I have used this guide before http://iosmuncher.blogspot.co.uk/2012/06/implementing-apns-in-titanium-start-to.html and I can sucessfully send alerts to individual apps.

What I want to do is give the user the chance to recieve alerts only for certain games. so the user selects a game(s) in the app and only gets alerts when goals are scored in those games or its half or full time.
I could send all alerts to the app and then filter them at app level but that seems like the wrong technique to use, really I want to use a much more targetted approach and send alerts only for the games the user has selected to that phone

Anyone any ideas how I can implenment this please ?

Mike

  • Actually, I haven't used APNS before but ACS (Appcelerator) Push Notifications but the idea should be the same.

    The keypoint for accomplishing this is that you should assign a "channel" per game (that is just a unique alias of your choice) in which the user would subscribe and unsubscribe. So along with the payload you are sending now there must be a way to define which channel to use as a filter (a parameter similar to payload), so only users subscribed to this channel will receive the notification.

    From server side you would give a unique id in each game so you would know in which channel to publish.

    Every user subscribed to this channel would then receive the push notification you would send via your server side "app".
