picker positioning

Hey guys, I'm putting a picker to the screen, and using the right: 0 during creation expecting it will snap to the right side of the encompassing layout: horizontal type view, but instead it stacks up horizontally as if the right: 0 was never there. I've used the right: 0 on views and imageViews and it works as expected, but not with a picker?

Has this behavior been witnessed before? Or should I just do something else like: right: Ti.Platform.displayCaps.platformWidth - 128, I would really like to stay away from a hardcode like this…

  • the right property for a child control inside a parent control with layout horizontal works like right-spacing between the next child. If you want to position a child control to right, the parent control should be layout:&#39;composite&#39; and the property of the child right: 0
