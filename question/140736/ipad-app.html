    Hi Gurkan

    This is a technique known as parallaxing, in simple terms they have several views placed over the top of each other, when the user scrolls one view the app tells the other views to scroll as well but as a percentage of the distance the top one scrolls.

    So by moving the top layer the middle layer moves at the same time but only 80% of the distance travelled by the top layer, the bottom layer travels only 60% of the top layer. The % a guides only people use there own.

    By using this technique it gives that weird view - looks magic and special, but really easy to make yourself.

    It is certainly not the scrollView, just normal views or imageviews stacked on top of each other with the top layer responding to events touchstart, touchmove and touchend.

    This technique also pre-dates computers, look at old traditional cartoons, particularly when they are moving across the screen, the background moves slower than the grass layer, with the characters on top moving the fastest.

    Hope that explains it. Look forward to seeing your app in the store using the technique.

    • Hi

      I have not tried the code you mentioned, but if you want to learn the principle - you start by following a few steps.

      • Add three views; viewBottom, viewMiddle & viewTop (in this order)

      • Add an image to each of these views that has transparent region so you can easily see each layer through the other.

      • Add an event listener to viewTop for touchmove and use Ti.API.info(&#39;your debug code&#39;) to write out the x and y

      • set the x and y of the viewMiddle to be the SAME as the values you received.

      • move around the screen - both viewTop AND viewMiddle should be moving at the same time.

      • now play with the x&#x2F;y by multiplying by a factor like 80%.

      • test again, the viewMiddle should now be moving at a different speed.

      Continue down this path and see what you achieve.
