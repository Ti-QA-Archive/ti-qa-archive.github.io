  • Hi Varun,

    My main issue with Titanium and its error reporting is that the error messages provide no information and therefore make debugging almost impossible. This error message, for example, can be found in various posts with at least 4 completely different solutions… a pretty good sign that it is a useless error message. The causes i have found so far that are linked to this error include:

    1. clean project
    2. wrong android sdk
    3. wrong java jdk
    4. so short a timeout on starting emulator

    i have done all of these fixes:

    1. cleaned numerous times
    2. now running sdk 21
    3. running 32bit jdk (i think; not sure how to tell this)

      I found this link, it might help you to understand what is happening, and why your application is not installing on the emulator.       ti build -p android --debug-host localhost:8999 --target "device"
