Max length of requestHeader for basic auth using Titanium.Network.HTTPClient

Hi,

i'm trying to authenticate API requests with basic auth using Titanium.Network.HTTPClient and it's requestHeader.

var _consumerKey = '0123456789',
    _consumerPassword = '0123456789012345678901234567890123456789';

var client = Ti.Network.createHTTPClient();

client.onload = function(e) {
    console.log(JSON.stringify(e));
    console.log(JSON.stringify(this.responseText));        
};

client.onerror = function(e) {
    console.log('error: ', JSON.stringify(e));
};

client.open('GET', 'http://localhost:8888/test/htaccess', true);

var authString = 'Basic ' + Ti.Utils.base64encode(_consumerKey + ':' + _consumerPassword);
client.setRequestHeader('Authorization', authString);

client.timeout = 5000;    
client.send();

My problem here is, that the requestHeader seems to have a (undocumented?) max length, because when i add one char to username or password in my example above, the header doesn't get send any more. When i send the same credentials using PHP and curl to the server, it works. So this has to be something in Titanium.

I also tried to use client.username und client.password, but that seems not to work at all.

Is this a known restriction? Can i do anything about it? I want to use at least 32 char long consumerKeys and 48 char long consumerPasswords.

Thanks in advance, Florian

PS: This is on iOS using SDK 3.1.3
