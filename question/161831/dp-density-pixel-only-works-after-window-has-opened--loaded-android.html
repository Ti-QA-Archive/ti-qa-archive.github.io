'dp' Density pixel only works after window has opened / loaded (Android)

3.2.0 SDK - Android (although this has happened in previous SDKs too)

To achieve a consistent layout I'm using the 'dp' (I've also tried 'sp') for the fontSize property of my Window's labels.

I tried everything but when the window loads the fontSize is tiny, the 'dp' is being ignored. I noticed the strange behaviour because, for this window with lots of labels I added an orientationChange event to detect going from landscape to portrait so that the screen is re-drawn. Upon a re-draw all the fontSize DP values are correct and the window looks right.

I've since tried to do a better implementation of the label placement, however I can only get the fontSizes to display correctly if I wrap them in a setTimeout(). Then the window is loaded and then the labels are added / drawn to the window and everything works.

Why is this the case?

  • Well, there are two things here you need to address.

    • First, fonts need to use the sp scale in order to function properly on Android.
    • Second, without posting any code whatsoever with your question, it is difficult for anyone here to properly identify what your issue really is. But from what I'm reading, it seems that you are "placing" UI elements on the screen dynamically. But since the screen hasn't rendered completely yet, your code cannot do the proper calculations it needs to place its elements on the screen. For this, I recommend you use the postLayout event. That way, your code will be run once the whole layout is complete.

    Hope this helps
