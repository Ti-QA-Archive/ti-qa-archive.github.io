Need help on starting my first Appcelerator Titanium Alloy App with TableView

I am developing my first Appcelerator Titanium Alloy app, and I have the following code:

index.xml

<Alloy>
    <Collection src="songs" />
    <Window class="container" title="Songs">
        <TableView dataCollection="songs">
            <TableViewRow title="{title}" artist="{artist}" album="{album}" genre="{genre}" />
        </TableView>
    </Window>
</Alloy>

songs.js

exports.definition = {
    config: {
        "columns": {
            "title": "TEXT",
            "artist": "TEXT",
            "album": "TEXT",
            "genre": "TEXT"
        },
        "defaults": {
            "title": "-",
            "artist": "-",
            "album": "-",
            "genre": "-"
        },
            "adapter": {
            "type": "sql",
            "collection_name": "songs"
        }
    }
};

In my index.js file, I simply call $.index.open();. However, I don't see my TableView at all. When running as a Mobile Web app, I simply see the Appcelerator logo, while when running as an Android app, I see a blank white screen after seeing the Appcelerator logo.

Does anyone know what I am doing wrong here?

    Hi

    I think you also require the following inside index.js;

    var songs = Alloy.Collections.songs;
songs.fetch();

    Also you do not show any existing record entries in the question - if you do not have any yet add this code above the fetch block;

    var song = Alloy.createModel('songs', {
    title: 'Title 1',
    artist: 'Artist',
    album: 'Album',
    genre: 'Genre'
}); 
song.save();
