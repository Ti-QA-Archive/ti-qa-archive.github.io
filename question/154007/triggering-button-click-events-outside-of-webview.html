Triggering button click events outside of webview

Alloy: 1.0 Titanium SDK: 3.1.1 iOS: 6.0 iOS iPad Simulator: 6.0

I am trying to load a web view in a small window, within the confines of another window. So I have my main window, and then I am loading a webview which is within its own small window on top of that. Clicking a button executes the runReport function below.

Here is my code that creates the webview within the window:

function runReport() {

     var winWeb = Ti.UI.createWindow({
         top: 70,
         right: 0,
         width: 742,
         height: 607
     });

    var webView = Ti.UI.createWebView({
        url : '/recline/analytics.html'
    });

    winWeb.add(webView);

    setTimeout(function(){
        winWeb.open();
    }, 500);
}

The issue is after this webview has loaded, other buttons on the screen, that are attached to the underlying window, no longer fire their events. Is there a better way to set this up? I'm guessing putting a window on top of another is not a best practice, but the only way I know how to use a webview is to add it to a window. So I guess the better question is, how do you trigger button click events in the underlying window, when you have a webview window on top of it (but not completely covering it)?

  • Use a view, instead of a new window. Just create the webview and add it on top of everything. the visible buttons in the window, will work just fine.
