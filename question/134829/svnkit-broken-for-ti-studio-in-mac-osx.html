SVNKit Broken for Ti Studio in Mac OSX

This definitely seems to be a problem with Ti studio and the SVNKit plug-in…not the underlying OS. I've just completed wiping my mac and re-installing everything CLEAN (including Lion Internet Recovery) and I have exactly the same behavior.

OS X 10.7.3 with all latest updates
Xcode 4.2.1 with iOS SDKs
Android-SDK r17 (Installed to /Applications/android-sdk-macosx)
Relevant Android tools and environments installed using Android System Manager
Latest Ti Studio from the site (the 2/14/2012 build 1.0.9)

All of that appears sane…then I take the following steps:

Help => Install New Software.
Select the Eclipse (Helios) repo
Search for SVN
Install the "Subversive SVN Team Provider" and restart when prompted.
After restart, select the "SVN Repo Browsing" perspective
When prompted to select a connector I choose "SVNKit 1.3.5 Connector" and restart when prompted.

After restart I am prompted to select a connector again. It shows as installed from Help => Install New Software => Show installed software… however Ti Studio => Prefs => Team => Connectors has nothing in the drop down list.

Looking in About => Installation Details => Plug-Ins it does appear that the Polarion SVN Kit Plugin is unsigned - Are unsigned plug-ins black-listed? Is there a signed version? Is there a way around it?

What can I do to get SVN Integration back?

    Fixed

    I finally fixed it for anyone else having a similar problem…

    Believe it or not it was my firewall content filter. Apparently ClearOS was blocking some critical component from downloading at the edge of my network (maybe that unsigned .jar?).

    The boggling thing is that the installs never failed or threw an error about missing components - it just didn't install them and carried on about its happy way. I'm not sure what was missing, but after uninstalling the SVNKit components and then disabling the content filter I was able to install SVNKit again and now it works.

    Good luck!
