Setting WebView cookies

I am hitting a .Net http handler that handles the login for our app and displays folders and reports. When a report is clicked it uses a WebView to display the rendered report. Im assuming the WebView has used the cookie from the original call to the login handler as the server checks if the call is authenticated or not - this all worked fine in April last year (code hasn't been touched since).

The cookie seems to be no longer sent so Im wondering if I can set it for the WebView manually. I've seen references to updating a sqlite db in order to do this but cant find any docs about it.

TIA
Dave

  • I'm not sure if it still works, but the solution you are referring to is inserting the cookie into the webviews.db database.
    You might try simply setting the cookie with javascript when the onload event for the webview is fired. That should ensure that your cookie is set.

  • This turned out not to be Titanium related. In .Net 4.0 the default browser capability files dont have an entry for the iPhone Simulator user agent string so it falls through to the default which turns off cookies and callbacks (gasp!).

    So hitting a http handler that returns an auth cookie will in fact get used by a WebView provided you update the browser cap files.
