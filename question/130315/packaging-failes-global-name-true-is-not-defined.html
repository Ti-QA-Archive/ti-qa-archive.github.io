Packaging failes: global name 'true' is not defined

I have a serious problem: I can't package any of my desktop apps anymore. I always get the message "Packaging failed with message: global name 'true' is not defined" and it stops.

This happens with the "TiDev" compiler as well as with Titanium Studio.

It happens when building for any platform, but Win & Linux packages are created regardless if the error shows up, or not. Only mac packages aren't created at all.

I tried switching the compiler (TiDev / TiStudio) - changed nothing. I tried a solution mentioned in another Thread, switching the "Show Spash Screen" option to "Yes", which didn't help at all.

I have created completely new projects, copying the contents of my resources folder inside it - IT DIDN'T HELP.

I have no clue how to solve this problem and I'm really desperate…

  • My own experience:

    Build to Mac:
    Works fine. This will then give you the download page link.

    then, Build to Linux + Mac:

    You get the "global name 'true' is not defined" error, but when you visit the download page (though Studio will claim it doesn't exist - i had to go in via Ti Developer) you will see the Linux builds - so it's the mac one that fails.

    So, build and download the linux packages, and throw them up somewhere else for people to actually go and grab them.

    Then go and build Mac only, and grab that, and do the same thing- put it somewhere else for people to download. This is my current workaround. Can't comment on windows, since not targeting it at all.

    hope this helps someone.
