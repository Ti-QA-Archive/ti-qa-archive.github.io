  • Should not be too hard to figure out how an UI element is heavier then another.
    A label would eat more memory then a simple view because besides all the properties of the view it also had to hold the text and the formatting text.
    An image will consume even more memory because it has to hold all the pixels' information. A webview will be the worst as it's nothing else then a browser, and so on.

    If you really want to test this maybe you should run some tests with each component and the xcode profiler (not sure though how to do it :) )
