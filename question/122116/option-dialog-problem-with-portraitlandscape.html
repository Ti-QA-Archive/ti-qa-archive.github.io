Option dialog, problem with portrait/landscape

I've ran into a problem on iOS with the option dialog.

The problem lies in landscape mode, when i open the dialog in landscape mode, it gives me an scrollable list of options, with the destruction button on the top. But in portrait mode this destruction button is on the bottom.

I am using the clicked index to figure out what button is pressed. But this problem messes up the indexes, in landscape mode the destruction button is on the first index instead of the last, resulting in every other index to be +1 of the original index.

Can someone help me with this problem?
