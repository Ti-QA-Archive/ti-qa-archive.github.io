Deploy iOS ad-hoc using dev environment variables in config.json?

Hello,

Is there a way to force an iOS ad-hoc build to use env:development variables set in config.json? When I do an ad-hoc build today, it automatically uses env:production variables. I would like to distribute a build to my testers, which use env:development to point to dev backend services and test data. Perhaps there is a specific command line flag? Or a Studio build config I'm missing? Thanks!

My app runs Titanium 3.0.2.GA and uses Alloy 1.0.0.
The ad-hoc build is for iOS 6.1 on a physical iPhone 4-5.

I'm running OSX 10.8.3 and Xcode 4.6.1

    Alloy can't make a distinction between the deploy type and whether or not it's an ad-hoc build. This is determined by the CLI. That said perhaps we can look into a means of allowing this distinction to be specified. Can you log a feature request ticket to the Titanium Community Jira project with this information? Be sure to give it the labels "alloy" and "cli".

    • You'll have to do this from the command line, as there's no option for it in TiStudio, but you can define an explicit "deploytype" for your alloy compile process. For example, if I want to compile alloy for ios and a "development" environment regardless of what the CLI thinks, I would do this:

      alloy compile --config platform=ios,deploytype=development

      Notice that there's no space between the comma, it's all one string after –config
