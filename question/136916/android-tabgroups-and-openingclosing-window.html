Android tabgroups and opening/closing window?

My question trying to understand why Android is opening and displaying heavyweight windows the way it is.

I have created a standard tabgroup, and all screens or sub screens are heavyweight windows

eg:

Ti.UI.createWindow(pp.combine(pp.style.Window,{
            title:    'My Window',
        }));

I have also created a newer window stack manager method, that does not use a fireevent tyle process that crashes under Android.. (long story)

    // our window stack manager (opens a window and adds it to the stack)
    pp.ui.openTabWin = function(obj) {
        if(pp.app.osname == 'iphone') {
            obj.tab.open(obj.win,{animation:true});
            pp.ui.winStack.push(obj.win)
            Ti.API.info("Current Window Stack: "+pp.ui.winStack);
        } else {
            obj.tab.open(obj.win,{animation:true});
            pp.ui.winStack.push(obj.win)
            Ti.API.info("Current Window Stack: "+pp.ui.winStack);

            //Ti.API.info("Opening Android heavyweight window.");
            //obj.win.open();
        }
    };

Using the above logic under Android, it will open the window outside of the tabgroup (e.g no tabgroup tabs) AND if the user hits the back button it will walk the windows back to the home screen where it shows a tabgroup tabs. This is fine, however I would like to have the tabgroup on all windows. If I use the commented out code ("obj.win.open();") to open the window normally, outside of the tabgroup I would think, it opens it within the tabgroup or at least shows the tabgroup bar and the new window content. However when the back button is hit the entire app closes, not just the opened window.

So, short question is why does the tabgroup show when I do a normal window open, and when I open the window in a tabgroup it does not show the tabs.

  • My understanding on Android was that if you are a child of a tabgroup - then you are automatically a heavy-weight window. Also, the tabgroup metaphor on Android only relates to items in the tab group and not any windows opened "under" them (the tabgroup goes away at that point)
