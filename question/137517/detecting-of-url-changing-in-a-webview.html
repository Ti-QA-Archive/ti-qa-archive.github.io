Detecting of url changing in a webview

Hi,
for a oauth project I need to fish out a parameter from redirected URL

In this way I start a webview I try to listen to all redirects:

var webview = Ti.UI.createWebView({
    url : url
});
parent.add(webview);
var oldquery = null;
var cron = setInterval(function() {
    var query = webview.url.split('?')[1];
    if(oldquery != query) {
        Ti.API.log(query.split('&'));
        oldquery = query;
    }
    var vars = query.split('&');
    for(var i = 0; i < vars.length; i++) {
        var pair = vars[i].split("=");
        if(pair[0] == 'oauth_verifier')
        var verifier  = pair[1];  // found!!!
    }
},100);

This code runs last friday. This means the webview is opened, the user gives password/login and after a lot of redirects the parameter 'verifier' appears.
Today it shows only the first URl in log – all other changes of URL are not readabel by command 'query = webview.url.split('&')'

Any ideas? It is mystic!

Rainer
