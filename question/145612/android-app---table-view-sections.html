    Hi

    Are you talking about sections or section headers, your question only mentions headers but your content appears to lead towards section headers.

    If you mean sections then they should occupy the space equal to the height of the contained rows plus any section headers (and section footers if on iOS).

    If you mean section headers then you can use headerTitle if you want the default simple caption. But if you want more control then you can use headerView to build your own custom version. See example below.

    var viewHeader = Ti.UI.createView({
    backgroundColor: '#eee',
    height: 50, // this cannot currently auto size so you must fix or calc. this value
    width: Ti.UI.FILL
});
var lblHeader = Ti.UI.createLabel({
    color: '#444',
    height: Ti.UI.SIZE,
    left: 10,
    right: 10,
    text: 'This is a test',
    textAlign: 'left',
    width: Ti.UI.FILL
});
viewHeader.add(lblHeader);
var section = Ti.UI.createTableViewSection();
section.add(viewHeader);
...
// add rows to the section as normal
...
// perform any normal table creations
...

    This is an example only, you can add almost any views in the headerView, position them as you see fit and each sections header can be different.

    With iOS you can do the same to the footerView, but currently only footerText is supported normally on Android with footerView having limitations.

    • Hi

      Yes as the headerView accepts a standard view and views accept standard other views you can have anything you like. You could make it 150 pixels tall if you wanted and add a switch, three labels and a button. Or a simple label but styled the way you want it to look rather than accepting the default.

      As each section header can be different it can help you break up a table into visually different section groups.

      In the past I have used headers for showing a label on the left and a section refresh button on the right.

      I have also used headers to show table column headings, the rows as data cells and the footer as total columns. Using this technique allows the data to scroll up whilst the header and footer views are locked to the top and bottom of the screen, great for mini invoice displays or spreadsheet representations.

      You can achieve pretty much whatever you like.
