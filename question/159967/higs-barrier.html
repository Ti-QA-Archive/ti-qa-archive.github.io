HIGS barrier

I have an iOS radio app with 3 windows and 3 tabs to switch between each. My client wishes their user to be able to click a button in the 'Stations' window to select an audio stream and have the app immediately move back to the 'Player' window without the user having to click on the 'Player' page tab that would otherwise do the same job. Where this is perfectly possible using setActiveTab I was wondering if this would get passed at Apple as it would seem to take the responsibility for moving between tabs away from the user. The Human Interface Guidelines are quite complex, but also vague in some areas and I did have app rejected once for not following the HIGS. Has anybody had a similar issue - or any thoughts on this?

Thanks.

Mobile /Titanium SDK version 3.1.3.GA/ iOS 7/ Mac OSX 10.8.5
