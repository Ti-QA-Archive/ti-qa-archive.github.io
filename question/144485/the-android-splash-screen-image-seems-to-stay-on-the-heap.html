The Android splash-screen image seems to stay on the heap

I created a Titanium Mobile "Single Window Application" project, and replaced the default.png file with one of my own. The new PNG image size is 320x480. I built the project using Titanium SDK 2.1.3GA, and loaded it onto my Samsung Galaxy S3 phone, running Android OS 4.0.4.

After running the application (which does nothing other than display a single window that states "Welcome to Titanium!"), I used DDMS to create a heap dump, converted it, and viewed it in Eclipse Memory Analyzer.

The memory analyzer showed a large bitmap on the heap, which had a size (in bytes) that was exactly 16 times the number of pixels in my splash screen PNG. I then replaced default.png with another PNG of a different size, re-ran the test, this time making sure to force some garbage collections via the memory analyzer after the app had fully loaded. The memory analyzer again showed a bitmap that was exactly 16 times the number of pixels in the differently-sized PNG.

While I'm curious why the size of the bitmap on the heap is 16x the number of pixels in the original image (I don't know where 16 comes from, and I don't know why it's not 16x the number of pixels in the scaled-to-fit-the-whole-screen image), I'm more curious about why the bitmap doesn't appear to be removed from the heap after the application has loaded.

I ran this same test using one of my applications which does things that are a bit more complicated than displaying a single window, and found the same result… a bitmap which appears to be the splash screen is never cleared off the heap. This obviously becomes an issue as my app needs more of the heap.

Any information on how to clear the splash screen off the heap would be very appreciated. Thanks.
