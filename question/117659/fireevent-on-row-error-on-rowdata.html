FireEvent on row: Error on rowdata

Hello

When there is only 1 row in my tableView i try to trigger a click event on the row. When it is triggered i get an error and it says that e.rowData is undefined. Is there something I might be doing wrong?

Here's my code:

triggering the clickevent on the first row. I first generate an alert (which shows the correct sessionId!), right after that I generate the click alert.

if (rowCount <= 1) {
    alert(tableview.data[0].rows[0].sessionId);
    tableview.data[0].rows[0].fireEvent('click');
};

Then on the click event i do the following:

    tableview.addEventListener('click', function(e){
         if (e.rowData.sessionId) {
             openPage(e.rowData.sessionId);
         }
         else {
             alert('Error');
         };
    });

At this point the error occurs. It says that e.rowData is undefined. But if you look at the alert in the previous block of code, it returns the correct sessionId. However when there are more rows to click on, and i manually click on a row, it runs smootly and goes to the next page.

Can anyone tell me what I'm missing here?

Thanks!

  • hey usman ..
    as i understand it .. correct me if i'm wrong .. what u want is to 'recreate' the same image used on the row and show it in another window ..

    first of all again with custom rows don't try using e.rowData it won't work ..

    as u only need the imagePath/imagUrl do something like that, this is the easiest way i know and it works :)

    var imagePath = 'images/myimage.png';
var r = Ti.UI.createTableViewRow();
var img = Ti.UI.createImageView({image:imagePath});
r.add(img);
r.yourCustomProp= imagePath;

.. //add it to a table show the table whatever :)

    and in the eventhandler

    ...
var rowImage=Ti.UI.createImageView({image:e.row.yourCustomProp});
var subWindow=CreateSubWindow(rowImage);
subWindow.open();
...

    here it also maybe a good idea to move the creation of the image view to the 'createSubWindow' function as i'm not 100% sure but i suspect UI-objects created in anonymous functions to leak memory .. couldn't hurt :)

    if ur using standard rows like this

    var data = [{title:"Row 1"},{title:"Row 2"}];
var table = Titanium.UI.createTableView({data:data});
win.add(table);

    u can try with adding the imagePath property there and then try retrieving it via e.row.yourCustomProp or e.rowData.yourCustomProp .. didn't test that as i never use standard rows ..
