passing element object data as fireEvent param crashes app.

Ti.SDK 1.6.2
iOS 4.3

I am attempting to code a custom tab group for an iOS application.

In doing so I want to fire a custom eventlistener when a tab is clicked.

In testing, if I pass standard string or int data to the listener things work fine, but once I pass an object containing view elements the app crashes without any useful data to help debug.


var children = parentView.children;

Ti.App.fireEvent('fireTab', { 

    activeIndex: activeIndex,
    tabWidth: tabWidth,
     children: children // crashes once this is present

});

And, for those that are wondering, yes children does in fact have data:

(
    "[object Window1]",
    "[object Window2]",
    "[object Window3]",
    "[object Action1]",
)

Thanks,
Josey
