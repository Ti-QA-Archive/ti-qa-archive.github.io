Updated to Titanium 3.0.2 error in Alloy.createController

After update to Titanium 3.0.2 my project is not working properly. (It worked pefectly before updating)
After much debugging it lookes like its the Alloy.createController('').getView('').open() that doesnt fire.
Code:

Ti.API.info('Before');
Alloy.createController('home').getView().open();
Ti.API.info('After');

Log:
[INFO] : Before

Anybody else have this problem or have a solution?

    Yes, version 3.0.2 removes the support for the on eventListener for everything but alloy/backbone models.

    So for ui elements you have to use addEventListener.

  • Found the error and its not Alloy.getController().getView(), it was the eventListener i used.
    Normally i use

    $.stream.table.on('click', function(e) { showStream(e); });

    Instead i need to use

    $.stream.table.addEventListener('click', function(e) { showStream(e); });

    Anybody know if they removed support for .on eventListener?
