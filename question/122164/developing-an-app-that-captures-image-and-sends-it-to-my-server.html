Developing an app that captures image and sends it to my server

I'd like to develop a Titanium application that guides a user through taking a photo and sending the image to my server, along with location and date information. The Oil Reporter application, which was developed using Titanium, essentially does the same thing.

So, does Titanium give me easy access to the OS API's needed to capture and then send the image with the metadata? I found some threads on this forum that suggest that transferring images from the phone device to a remote server is not a very straightforward process.

——- Rick

  • I am currently in the finishing stages of an app that does exactly this - Except with 4 images instead of 1. You have the option of adding 1 - 4 images, as well as information about these images and some other information regarding the content of the app, and then in the end you can send these images via Email and/or to a remote MySQL Database and remote server, which is hosted by my company.

    It is essentially done through HTTPClients and a PHP script on my server. Depending on how many images there are it loops through the HTTPClient and sends the image to the server, which uses the PHP image functions to check the size of the image and then uploads it to a folder on the server.

    If you are looking to see some code, send me an Email at coltonarabsky@me.com and I can show you some code that I hope you would find useful.
