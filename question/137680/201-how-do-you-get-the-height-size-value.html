  • Hi Leo,

    since Titanium 2.0 you can ask for the rect property available on every UI.View. Rect is a dimension with height and width properties. See docs here and explanation here.

    If you need to update the view size as soon as possible, follow the documentation's example using postlayout event. But if you are pretty sure that the view is already drawn (i.e., because you are capturing a user event), there's no need to manage it.

    Note that postLayout event is called very often and for this reason you may want to remove the listener when your view is updated, as is done in the documentation example.

  • This is a useful guide:

    http://docs.appcelerator.com/titanium/2.0/index.html#!/guide/Transitioning_to_the_New_UI_Layout_System

    var postLayoutCallback  = function(e){
  Ti.API.info(String.format("Layout done, left: %f, width: %f", myView.rect.x, myView.rect.width));
  myView.removeEventListener('postlayout', postLayoutCallback);
}
myView.addEventListener('postlayout', postLayoutCallback);
myView.updateLayout({
  left: '25%',
  width: '25%'
});

    That way we don't have to "hack it" like before and have a while loop which waits until the size property is available on the object.

    Note: make sure you remove the postlayout event listener after you've gotten the dimensions, otherwise it will get called every time the screen redraws itself!!
