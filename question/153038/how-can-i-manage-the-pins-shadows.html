  • Hi

    Firstly and MOST importantly - unless you have already been issued an API key for this specific app id you CANNOT create an Android app using Maps v1.

    Google are no longer issuing API keys as v1 has been deprecated completely. You must swap over to using v2.

    • Google Maps v2 for Android
    • Modules.Map

    Irrespective of which version - if you are providing the annotation pins as images yourself - you are responsible for the shadows and unlike the web version they are not a separate image reference. You combine the shadow image into the pin image and supply as a single image as the annotation pin.

    • Yes, I did integrate leafletjs into my Ti project. It's really cool because you treat maps the same way for Android and iOS, and plus you can manage offline maps and lots of added plugins (like grouping pins and so on).

      There's a nice tutorial here:
      http://techblog.troyweb.com/index.php/2012/11/a-mobile-app-with-offline-maps-using-titanium/

      but there were few thing to tweak (like the fact that with android you can't save blobs in sqlite, nasty bug, so I had to convert all of the offline map blobs into base64 strings).

      In case you're interested in the offline part:

      1. Download TileMill and download the part of the world you want to show (there are lots of maps wheighing a matter of GBs)

      2. Import with PostgreSQL the maps and edit the css for the map (road colors and so on)

      3. Export it in mbtiles (don't zoom a lot)
