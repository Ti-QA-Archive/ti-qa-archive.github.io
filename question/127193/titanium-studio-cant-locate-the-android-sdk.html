    Hi Marcin

    This usually happens when the Android SDK path in TiStudio is configured to the location of the android or adb executables (ie the tools or platform-tools directories, respectively), rather than the parent of those directories. Hence, make sure that those two files exist in their respective directories, and that the TiStudio SDK path is configured with their common parent directory.

    If this does not work, please follow the instructions at Install Diagnostics for Windows. Copy the output and paste it to your preferred pastebin service, and paste the link to it here. It's important that you do not modify it in any way, as this could mask problems with your configuration. Do NOT paste the actual code here! This is because the system will change the formatting.

    Also, please create and share a screenshot of your android installed packages, using something like ImageShack

    State the processor and RAM installed to your system.

    Lastly, run the command systeminfo at the commandline and share it in a pastebin.

    Then I will help you further.

    • Also, you have JDK 1.7 installed, which is not supported by Titanium. Please install JDK 6 (aka 1.6).

      Refer to the Titanium Compatibility Matrix for this information.

      Lastly, you need to install the Google APIs package for each version you have installed. At the moment you have only got it installed for Android 4, which we don't yet support. Without it, you won't be able to boot an emulator that supports maps, and hence won't be able to run the KitchenSink, which uses them.
