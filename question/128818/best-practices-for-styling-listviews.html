Best practices for styling ListViews

Hi there!

I wonder which is the best practise for styling multiple ListViews (iOS) the same way.

I found some saying that making use of JSS and applying the class to each ListView in question, is more efficiently and the rendering is faster and less memory consuming. So I tend to using (inline) styling "the usual way" instead of using JSS.

And I found others saying that the JSS support is not yet stable enough. So I tend not to use JSS at all.

This is quite a dilemma, as I would prefer a stable and efficient App, of course, as this App I'm currently developing is mainly based on ListViews.

I highly appreciate your advice!

Best regards and thank in advance,

Chris

    The JSS in "incomplete" (as far as I know), some of the properties need to be set inside the constructor.
    A solution would be to extend the Ti APIs with your own function that accept a "theme", maybe something like:

    var createView = function(options, theme) {
    for (var attrname in options) { theme[attrname] = options[attrname]; }


    return Ti.UI.createView(theme);

}

    This is a proof of concept. The theme is extended and the same props are overwritten by the options so if at some point you will want to change some colors let's say you can simply add them in the options object and it will overwrite the "defaults" found in theme, leaving other props intact.

    You need though to take care in this case because the for in loop is very slow, so maybe some sort of caching of the resulting props object would help.
