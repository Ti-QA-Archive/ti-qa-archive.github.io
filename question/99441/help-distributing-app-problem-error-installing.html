  • This is getting really strange.

    ever since like 2 days topics like these keep popping up and Since then I suddenly have the same problem.
    I'm beginning to think there is something strange going on, I''ve seen more of these topics in the last 2 days then I've seen in last couple of months.

    Anyway a workaround is:

    1: click on install to device.

    2: in the build map this should make the release-build folder

    3: the error comes up

    4: open iTunes

    5: in iTunes click: file - add to library

    6: browse to the build folder and in the release build folder is a file with a .app extension (yourappname.app)

    7: add this file to the iTunes library

    if correct it should load the app into iTunes

    8: sync with the device

    I use this workaround now and it works.
    But it's still strange

  • I prefer not to use iTunes at all to transfer apps onto my devices. If you use XCode Organiser you can easily set up global Team Provisioning Profiles for all your apps and install apps to multiple devices without messing with iTunes syncing.

    The most common install error is due to the incorrect Provisioning Profile….make sure Titanium Developer is loading the correct Profile and that it is named correctly (check the build.log for details)

    Hope this helps.
