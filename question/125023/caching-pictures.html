Caching pictures

Hi everybody.

I've found a few ideas about caching pictures while searching the Q&A but nothing serving my exact needs.
Here is my problem. I have a webview in my application.
This webview will be filled with html content while the application is running. This content will be provided from a remote DB and can change anytime. I want to be able to see the latest content retrieved on my application when I am offline. So I am going to store this HTML string into the local DB of the device, the only problem will be the pictures.

I'm investigation several solution:

  • The first one would be to use the cache manifest feature of HTML5 to cache the pictures, and I would add the manifest in the beginning of my html file : <html manifest=''> .. But I can't get it working for the moment. Does someone know if this feature works with Titanium and/or someone has a working example ?
  • The second idea is to parse the HTML into XML, retrieve all IMG markup, get the src attribute, save the picture locally and change the link for the local file.
    Only problem is that I can't find a way to change the value of the attribute ..
var htmlXML = StringtoXML(htmlString);        
var my_items = htmlXML.getElementsByTagName("img");            
for(var i = 0; i < my_items.length; i++)
{
    my_items.item(i).removeAttribute('src'); // don't work
    my_items.item(i).setAttribute('src','/iphone/Default-Landscape.png'); // don't work
}

The value of my_items.item(i).getAttribute('src') is the same before and after the loop …
I found this topic http://developer.appcelerator.com/question/119420/cant-change-attribute-of-xml-document so apparently i'm not the only one stuck there..

So if anyone know how to get one of this solution working, or has another idea to get the pictures offlinable, thanks in advance !!

Romain
