new app project: globals, closures and location listeners - best practise?

hello,
this is not really a problem but rather a request for opinions: i'm in the process of planning an (android) app and trying to find a reasonable approach. basically all relevant functionality is triggered by geolocation updates, example:

var latitude, longitude, oldLatitude, oldLongitude, ...;

function func1(){ 
  alert('lat:' + latitude + ' lng:' + longitude);
  // some other things to do with new coordinates 
}
function func2(){ 
  alert('lat:' + oldLatitude + ' lng:' + oldLongitude); 
  // some other things to do with old coordinates 
}
...
Titanium.Geolocation.getCurrentPosition(function(e)
{
  if (e.success)
  {
    latitude = e.coords.latitude;
    longitude = e.coords.longitude;
    func1();
    func2();
    oldLatitude = latitude;
    oldLongitude = longitude;
  }
});

i know it's ugly procedural code, globals etc., however i've used it that way before without problems. question is what would be the best way to maintain a state (latitude, longitude and others) accross location updates, and, in general what would be a more useful and elegant approach to organize the event listener and functions? would it make sense to encapsulate all global functions and variables within closures, even if there are only a few? thanks for suggestions, uno
