Accessing webserver running within Eclipse from outside the workstation

I have a Spring Restful API Server running locally on my machine.

I start/stop it as a Tomcat App server via Eclipse's Server's tab.

I also have built an Appcelerator app that makes calls to this server.

When I run my app via the Android Emulator option in Titatium Studio it connects and retreives data fine from my local server.
The request is:

var TRANSACTIONS_URL = 'http://myipaddress:8080/PurchaseAPIServer/api/purchase';

I encounter a problem when I connect my android device to machine via a usb cable (in developer mode) and install/run the app on Android Device with Titanium Studio.

My locally running server is never been hit.

Is there a way to get around this?

Thanks

    i'm afraid your problem is not a Titanium related issue.

    Your main problem is that your device cannot reach the web server installed on your machine. Having the device connected with a USB cable does not magically make them appear on the same network.

    My best advice would be firstly to find a way for your device to reach your web server (you can easily test that using the phone's web browser and a dummy html page on the web server). Once you have a valid network connection between the two. Only then, will you be able to test your application.
