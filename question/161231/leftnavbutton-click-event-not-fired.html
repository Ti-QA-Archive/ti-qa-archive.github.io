LeftNavButton Click event not fired

I have a 'details' controller that shows a detailed view of a person's information. The view of this controller is a tab with some stuff in it. The controller gets created dynamically because they can be opened for any person from quite a list.

My problem used to be the following: the automatically generated button from NavigationGroup in MobileWeb would not go back when clicked. No events would fire. Because of this I switched to Tabs for the whole project.

Now my problem is similar. I have a Tab system, and the 'details' view is now a Tab. I keep the NavBar for navigation functionality and make the Tabs invisible by setting the height to 0 (any other options just wouldn't work). Now the LeftNavButton just won't trigger click events. I used to put it in Alloy but now I've tried doing it programmatically for more versatility. Here's what I've got:

var backButton = Titanium.UI.createButton({
    bubbleParent: false,
});

$.window.setLeftNavButton(backButton);
$.window.getLeftNavButton().addEventListener('click', function(e) {
    alert("WOOOOH");
});

No Wooh'ing whatsoever. I tried out my setup by firing a 'click' event programmatically and this dis trigger the event listener. I've therefore come to the conclusion that I simply can't get click events to come through. Mind you that this isn't only the case for my custom back button but also for the one automatically created by NavigationGroup.

Any new insights you could give me on this matter would be highly appreciated.

If it makes any difference I develop using GA 3.2.0.
