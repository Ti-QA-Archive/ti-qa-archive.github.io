Intents vs. openURL

What are the pros and cons of using intents versus openURL. I'm able to successfully use both:

        Titanium.Platform.openURL('http://maps.google.com/maps?saddr=&daddr='+placeObj.place.lat+','+placeObj.place.lng);

Titanium.Platform.openURL('google.navigation:q='+placeObj.place.city+', '+placeObj.place.state);

The behavior seems to be similar if I were to call similar intents, is that what happens under the covers? If so is OpenURL preferred as it's more of a cross-platform solution?

  • Frank,

    Intents are an Android only thing. And they're meant for more than just loading a URL (though you can use them). So, openURL is the cross-platform choice to make if you're looking to open a web page in the user's default browser.

    My limited understanding of intents is based on what was shown in the Building Native Android Apps webinar. I haven't tried using them myself. In their demo app code, they showed how you could call on the services of a barcode scanner by using an intent. You open an intent via the barcode scanner's full app name (com.whatever.barcodescanner) then you can pass data to it (such as the type of scan to take) and handle data it sends back. They showed another example using an intent to sent text to the user's default text messaging app.

    So long answer to agree that intents will do what you're trying, but openURL is probably better.

    HTH
