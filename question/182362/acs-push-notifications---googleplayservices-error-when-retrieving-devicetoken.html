ACS push notifications - GooglePlayServices error when retrieving deviceToken

Using Titanium Studio 3.4.1, SDK 3.5.1. GA

I'm trying to retrieve deviceToken to register the device for push notifications. Upon launching the app i get following errors:

[ERROR] :  TiApplication: (main) [1029,1212] Sending event: exception on thread: main msg:java.lang.NoClassDefFoundError: com.google.android.gms.common.GooglePlayServicesUtil; Titanium 3.5.1,2015/03/05 10:08,96875c9
[ERROR] :  TiApplication: java.lang.NoClassDefFoundError: com.google.android.gms.common.GooglePlayServicesUtil

The ti.cloudpush debug is on, just before the error i get a message in the console:

[INFO] :   APSCloudPush: receivedGCMSenderId: 5xxxxxxxxxxx

Which is by the way different then the sender id i have configured in my ACS management for both development and production environment.

My code in alloy.js

// Initialize the module

var CloudPush = require('ti.cloudpush');
CloudPush.debug = true;
CloudPush.enabled = true;

CloudPush.retrieveDeviceToken({
    success : function deviceTokenSuccess(e) {
        Ti.API.info('Device Token: ' + e.deviceToken);
    },

    error : function deviceTokenError(e) {
        Ti.API.info('Failed to get the token! ' + e.error);
    }
});

  • Hello, Michal,

    [ERROR] :  TiApplication: java.lang.NoClassDefFoundError: com.google.android.gms.common.GooglePlayServicesUtil

    This error occurs when your device does not have Google Play services APK available or nor up-to-date on this device.

    Try updating your Google Play services APK in your device. More info.
