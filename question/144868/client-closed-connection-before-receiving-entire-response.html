Client closed connection before receiving entire response

My application (which has worked fine for 6 months or so) now occasionally has a big problem whereby on doing a POST HTTPClient request to a REST Web Service, an error occurs even though the web service is correctly processing the data and returning the correct response to my app.

I am using Charles to look at the requests and responses and although everything is as it should be, very often the request has (shown inside of Charles) a status of "Complete" but with a Failure message of: 

"Client closed connection before receiving entire response"

I have set the HTTPClient timeout to 400,000 (which I think is 40 seconds?). Charles shows the failed response comes back after 20.85 seconds so I assume that the application is not timing out.

My application looks for a response code of "1" from the web service so that it knows the request was processed successfully but in these instances where the failure occurs, it obviously gets nothing back.

Does anyone have any idea what might be causing this?

Titanium SDK 2.1.3.GA
XCode 4.5.2
