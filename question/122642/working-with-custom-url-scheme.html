    What your app needs to do is check the value of Ti.App.getArguments(), which will contain the full URL used to launch the program. You need to check in both the app.js as part of startup processing, AND during the "resumed" event for the app (and NOT the "resume" event).

    However, there is a known problem in that getArguments() will return its previous value if you just pause and resume. Therefore it is impossible to tell the difference between a normal pause/resumed, and the app being relaunched using the same custom URL as the previous time.

    See also this thread.
