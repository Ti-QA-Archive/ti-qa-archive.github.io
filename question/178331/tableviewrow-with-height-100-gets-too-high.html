TableViewRow with height 100% gets too high

Problem

In my app, where there are empty tableviews, I show an empty-state-view (ESV) instead. The ESV is actually a TableViewRow with height:100%. The tableviews has pull-2-update implemented. When the tableviews is first shown, the ESV is much higher than 100%, causing the ESV content to be placed to far down. Also, it is possible to scroll in the tableview. Also indicating that the one TableViewRow (ESV) is too high. However, when I pull-2-update, and the update is finished, the ESV gets the correct height; I can't scroll anymore and the ESV content is centered vertically. Just like it should be.

What I've tried

I was thinking about what is actually going on when i perform a pull-2-update. What happens is that i set the contentinsets top value, and then when the update is finished, contentinsets top value is set back to 0. Therefore, i thought i would be able to solve this by doing the very same thing (setting contentinset back and forth) upon loading the tableview, but no difference what so ever.

One thing that i think could be the issue, is that the tableview height is determined before the entire application is finished rendering causing it to become to high. But i don't know. Does anyone have any idea?

Versions

Titanium SDK: 3.4.0.GA

iOS: 7.1 / 8.0
