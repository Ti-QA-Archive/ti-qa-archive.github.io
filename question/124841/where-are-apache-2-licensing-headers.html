    • Where did you find that? Titanium Studio is not open source, but the SDKs that it uses, the ones your potential apps would rely on, are.     • This is verbatim from Ti Studio. The language here effectively states that if you proceed with Ti Studio install, you are no longer working on an open source basis with ANY Appcelerator code (whether it is open source or not). This is clearly meant to extend to the SDKs as well. This nullifies any benefit of the Apache license. Not sure why the use of an IDE should in any way imply change to the existing relationship with Licence Agreements for SDKs. All I can say is that you must have your lawyers working overtime to crank this sort of thing out. I am not sure how many folks even read the Agreement but this is over the top from my point of view.

      In essence, wording prevents you from doing anything but selling your apps in marketplaces prescribed by Appcelerator and requires that any local changes you might make to ANY open source code (including SDKS) must be contributed back and incorporated in sources, otherwise you are not permitted to use the changes. This basically takes the SDK's under Apache 2 Licensing and turns relationship into LGPL by design. I say by design because support for Developer is gone and developers have few options but be coerced into this since many have been building code on the platform for some time.
      This feels underhanded.

      I would be happy if the wording pertained specifically to TiStudio intellectual property but it goes well beyond. I think an appropriate solution and right thing here is for Appcelerator to change the language and be clear that use of Ti Studio does not change relationship with Apache 2 licensed SDK's.
