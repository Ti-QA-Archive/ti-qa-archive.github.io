TableView.index + TableViewIndexEntry

Hi there!

I'm creating a TableViewIndexEntry array as well as a TableView with custom TableViewRows on the fly.

THe TableViewIndexEntry array looks something like this:

 [{"title":"A","index":0},{"title":"B","index":4},{"title":"C","index":31}]

The array is assigned to the TableView's index property and showing.

I'd expect to see all the objects in the array directly among themselves. However, the result is that there's a big vertical space between the titels depending on the index value. If I set to index to 1, 2, 3 etc. they are shown as I'd expect, but the index connection is wrong so I'll loose the functionality.

Now, is this the intended behavior or am I doing something wrong?

  • Your example means that you have "at least" 31 tableview rows in your tableview.

    1. You have items starting with an 'A' in rows 1 through to row 3
    2. You have items start with a 'B' in rows 4 through to row 30
    3. You have one or more items starting with 'C' starting at row 31

    Does this match your data you have loaded into the tableview ?
