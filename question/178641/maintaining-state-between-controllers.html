Maintaining state between controllers

I am developing an Android app using Titanium SDK 3.4.0.201409261227-26092014122818 on Windows 7, testing on a HTC One device. My Titanium Studio build info is: Titanium Studio, build: 3.4.0.201409261227 Build: jenkins-titanium-rcp-master-197 (origin/master) Date: 26 September 2014, 12:28:18.

I have an Alloy Window (A) with a form in it that I have filled in with data. I then move to another Window (B) using:

Alloy.createController("windowB", args).getView().open();

However, if I then click the back button in the Android action bar to go back to Window A which fires this code:

Alloy.createController("windowA", args).getView().open();

I am presented with an empty form. The Window has been generated afresh.

Is there a way to get back to Window A and have the form still filled in with the data I entered first time round? Should I be opening Views rather than creating controllers again?

I'd appreciate any advice anyone can offer.

Thanks.

  • Not sure if this is the right way to do it but it seems to work. Instead of re-opening the Window A using the command listed above I have used:

    $.windowB.close();

    which closes the current Window and reveals the previous Window in the stack (A in this case).
