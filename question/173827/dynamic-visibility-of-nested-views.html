Dynamic visibility of nested views

Hello,

I have a couple of views nested within a root view. Each of these views has a height adjusted to Titanium.UI.SIZE, and I am toggling visibility on both.

My problem is the root view, who's height is also Titanium.UI.SIZE is conserving the height of the hidden view.

It's almost as if even though the view is hidden, there is a placeholder for it.

    Hi

    When you hide/show something you are doing EXACTLY that you are making the thing that exists not be visible, the property show/hide adjusts is 'visible' so it is only toggling visability.

    This is why the parent view maintains the dimensions of the largest child element. It sounds like you wish the two child views to act as optional states. This is easy enough, especially when you note the different method name 'remove'.

    It is important to realise that 'remove' does NOT destroy the view, it simply removes it from the parent. If you created it in code then you still have the variable used to define it, use that to remove and that again to add it back in.

    So use;

    parent.remove(child2);
// later
parent.add(child2);

    To toggle add child1 and remove child2, on toggle swap them round.
