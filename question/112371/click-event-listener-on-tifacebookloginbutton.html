click event listener on Ti.Facebook.loginButton

My app only supports the landscape orientation on iPhone. There is a Facebook login button in the app, which opens up the login dialog in landscape mode. But logging into Facebook in landscape mode is very user unfriendly. You can't see what you're typing.

So I tried to "intervene" by adding an event listener on the FB button so that when the button is clicked I can change the orientation to portrait. But it doesn't work. The click event never gets fired.

So the questions are:

  1. Is there a way to force a portrait orientation when Facebook login dialog is created?
  2. Is there a way to capture the click and add your own event listener?

Ti SDK 1.5.1 and iOS SDK 4.2.

Or should I just go try Ti release candidate 1.6?

    • The problem with Ti 1.6 is that the new module requires the user to have the Facebook app installed. If the Facebook app is installed, the user bounces from your app to the FB app and comes back. But if the app is not installed, the user goes into Safari and never gets back to your app. I'd much rather have the in-app experience of Ti 1.5.1, wouldn't you?

      Having said that,,there is the benefit of single-sign-on for people who are already logged in..so that's pretty cool. But again..only if the FB app is installed.

    • Brian, the problem with that approach is that there is no way to actually trigger the Facebook login button's functionality without having the user click it. If you create a layer over the button, that button's click event will fire and will give you the opportunity to rotate the screen. So far so good.

      But after rotating the screen, there is no way for you to do something like fbButton.fireEvent('click') or anything like that to simulate the actual click :-(

      If there is a hack to work around this, please let me know.
