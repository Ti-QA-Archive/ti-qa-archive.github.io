SDK 3.x causes Android apps to "crash" without error message

This is not a question, but I believe there must be many developers out there having the same problem. SDK 3.x causes Android apps to close silently just a second after starting. No error message, exception or logging whatsoever.

This only happens if the user has "do not keep activities" checked in the developer options on his device, but that wasn't a problem with former Titanium SDKs. At the time of this writing it's not clear if the bug report at https://jira.appcelerator.org/browse/TIMOB-12939 gets accepted, as the previous one was marked invalid. Our users definitely see this as a bug in our app.

  • Test case: Create a new Titanium Project, Single Window Application, using SDK 3.0.2.GA in Titanium Studio 3.0.2. Don't change anything in the example code, then install to a connected Android device. Start it from the device and you'll see "Welcome to Titanium!". Everything fine so far.

    After that, close the app, open the settings app, navigate to "Developer Options" and check "do not keep activities". After that, open the example app again. It will open and go away within just one second.
    I justed tested this on RAZR i (Android 4.1.2), Galaxy S2 (4.0.4), LG-P990 (4.0.4), but bug reports came from a wide variety of devices with Android 4.x.

    I know it's a developer option and users shouldn't mess with them, but an astonishingly large percentage of users do - at least in Germany, where most of our users are located.
