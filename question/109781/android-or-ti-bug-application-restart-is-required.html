Android or TI bug "application restart is required"

I created an app and it is working fine in the emulator. But when I am testing on my devices (android) I keep getting this error "application restart is required".

I have tried it on my Mac and PC running, TI 1.2.2 and SDK 1.5 & 1.6.

My devices are a Droid X and Incredible running Android 2.2.

I have looked though the Q&A and have tried everything listed in the Q&A. I tried the nightly build 1.6.0, didn’t work I tried building from source (1.5 & 1.6). I tried the workaround documented in Lighthouse.

The issue is installing the "app" on the device. I put the .apk file on the device and install it. Once the install is complete I get this error "application restart is required".

  • Just in case somebody is still dealing with this issue, the solution is to add the following property tag to you tiapp.xml file.

    This fix was introduced on Titanium SDK 2.1.2 so make sure you're using that version as a minimum.

    <property name="ti.android.bug2373.finishfalseroot" type="bool">true</property>

    Titanium SDK builds can be found here:

    http://builds.appcelerator.com.s3.amazonaws.com/index.html

  • There are now more options available for dealing with the Google bug that we're working around. See TIMOB-4941 - Android: More graceful workaround for Android Bug 2373 & 5277 for configuration options.

    To provide a little more detail, this is a bug in the android market app that doesn't launch applications the same way as the home screen. Therefore, Android sees it as two different invocations of the app and you'll have multiple copies of your app running if you open it right after install, press home, and then launch the app again from the home screen.

    The new options allow you to disable this workaround and have exposure to the same bug as native Android apps or you can also more flexibly message and restart the app w/o user intervention. It's completely up to you now to deal with it as you see fit for your app.

    • It looks like google fixed the bug in Android 4, because I do not see the app closing and opening with the correct intent. I can't 100% confirm though. I can confirm that SDK 1.8.1 has the fix if you add the properties in tiapp.xml.
      This worked for me (no toast message):

      <property name="ti.android.bug2373.disableDetection" type="bool">false</property>
<property name="ti.android.bug2373.restartDelay" type="int">500</property>
<property name="ti.android.bug2373.finishDelay" type="int">0</property>
<property name="ti.android.bug2373.skipAlert" type="bool">true</property>
<property name="ti.android.bug2373.message"></property>
