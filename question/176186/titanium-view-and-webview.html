  • For me it sounds crazy and is an evil software pattern.

    It depends if the HTML comes from remote or local.

    In case of local it is simple:

    In you webview (inside HTML) you place a snippet like: 

    window.onload(function(){Ti.App.fireEvent('webapp',{})});

    and on native side:

    var web = Ti.UI.createWebView({url:LOCALWEBINRESOURCEFOLDER});
Ti.App.addEventListener('webapp',function(){
    // do what you want
})

    In case of remote HTML you should use evalJS after web loading (width setInterval) . In this webJS you test, if dom ready.
