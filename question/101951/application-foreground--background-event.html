Application Foreground & Background Event

  • Accepted Answer

    You are looking for the "resume" and "pause" events. Note these occur on iOS but not Android. Strangely, there is also an undocumented "resumed" event which must be used if you want to read the Ti.App.getArguments() value from a custom URL launch. But use "resume" for normal purposes.

    Ti.App.addEventListener( 'resume', function(e) {
    // do something here
}

Ti.App.addEventListener( 'pause', function(e) {
    // do something here
}
