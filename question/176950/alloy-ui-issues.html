  • You have lots of questions here! Please read through our Guides, as nearly everything you ask is covered there.

    When I click on "Hello, World", the new view register.xml is shown, but I still see the "Hello, World" Label

    You have a label in your registerGroup View in register.xml. Is that the cause?

    In register.xml View id="registerGroup":top="Alloy.Globals.top" does not work

    If that's the exact syntax, remove the colon before top. Overall, I suggest you put all the styling in the TSS. Just as you'd separate HTML structure and CSS, you should keep structure to the XML file and styling to the TSS.

    In register.tss: borderStyle property is not applied to Textfileds

    Uh, not sure. It looks like it should work with what you have. Have you tried the other border styles? Maybe bezel no longer works at the OS level?

    How to make the ui rotate with the device?

    For iOS, you need to edit the tiapp.xml to enable the orientations you want to support. See the Guides for more info.

    How to say in register.xml that Textfield.top is some value from another Textfield.bottom?

    Wrap the text fields in a View whose layout property is set to 'vertical' then set the top of each textfield to the amount you want as padding.

    What is best way to have a ui adapt to any device resolution and orientation? Work only with %? Work only with dp? An bottom up approach where all view adapt to the required size for text labels and buttons? Or a top down approach where everything relies on percentage from view borders?

    Search the Q&A, there are lots of threads on this. Look for answers from Malcolm Hollingsworth; I know he's covered this well and extensively. There are also plenty of tips in the Guides. This is too big a subject to answer here considering how well it's been covered before.
