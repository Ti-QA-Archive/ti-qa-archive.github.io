Change the application locale INSIDE the app, so that the WHOLE app gets translated

Hi guys,

I want to provide to the app users the ability of changing the language of the app.
Since the built-in i18n features of appcelerator gives me some functionalities like the strings.xml files and the L() function (https://wiki.appcelerator.org/display/guides/Internationalization) I'm using them.

But sadly this is not enough, I mean, the i18n works great, but it takes the OS-Langauge and that's it.
So, when I want to provide a function of changing the app language inside the app (without OS-language change), then I'm having some troubles.

What's the best approach for solving this problem?

I have seen some modules for localization (https://marketplace.appcelerator.com/apps/7210?restoreSearch=true#!overview and https://marketplace.appcelerator.com/apps/7209?restoreSearch=true#!overview but these two modules won't render the whole application after setting the locale, right?

I thought of something like using the module, and setting a property with the current language and then showing a message like: "The language change will be available after a restart of this application".

But I don't really like this solution.

So which way to go?

cheers
emre

  • Hi Emre,

    on iOS you can use Ti.Locale.setLanguage() to set the current Language. But you have to reset all text or variables again using the L() function

    Ti.Locale.setLanguage("en");
label.text = L('welcome');
