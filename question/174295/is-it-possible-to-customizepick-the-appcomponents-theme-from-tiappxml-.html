  • Hi

    If you mean that; action bars, buttons, colour, and more seem to have the same styles when an app is created by Titanium then you need to understand something fundamental;

    Android apps created using Titanium use the base Android theme - this is as defined by Android itself

    This is the expected behaviour. If we limit ourselves to Android 4.x for the moment then you need to be aware that Android apps all start using a theme call Holo. You can change this to Holo.Light by making a change in the tiapp.xml file.

    However you can change this if you wish - the same as you can in most other Android development systems. In fact there is very little you cannot change. However as there are so many variation you can have it is wise to start off by using a pre-rolled solution and then learn as you go. You will need to add extra properties and files to your app, again this is very normal.

    • Alloy Action Jackson: An Android Action Bar Demo App for Appcelerator Titanium
    • AlloyActionJackson
    • Android Holo Colors Generator
    • Android Action Bar Style Generator

    The links above will get you up and running very quickly.

    Just to be clear "i found that the app's appearance developed by Titanium was pretty similar when using the built-in components/ui/controls" is a choice by the developer of the Apps to leave the styles using the base Android styles, this is not something that Appcelerator forces or needs. You can go and make your app look and act whichever way you like.
