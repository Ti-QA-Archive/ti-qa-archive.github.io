Does Appcelerator Ignore Unused Images when Deploying/Packaging

Just wondering if Appcelerator is smart enough to ignore images not referenced in code, or do I need to manually remove images not used by the by the mobile platform Im deploying to?

  • It depends on where those images are. If they are under \Resources\android\images\ or \Resources\iphone\images\ (which is where they should be), then they will only be included by the respective platform build.

    • Stephen Feather's statement is incorrect! Images in \Resources\iphone\images\ will NOT be packaged into the APK file for Android. You can see this for yourself by opening the APK file with WinZip. Also, at least when buiding for iPhone, the build process reduces the size of PNG files (you can see it doing this in the console log).

      Gagan Tiwari's statement (text message) is correct, that any file placed in the \Resources\ directory will be packaged for both platforms (I think that's what he was trying to say). This is why platform-specific images should be placed in the \Resources\android\images\ or \Resources\iphone\images\ directories.

      Stuart, I'm not sure what you mean by "javascript files specific to Android". The general rule is that any file placed in the \Resources\ directory will be packaged for both platforms. I've never tried this, but if you have platform-specific js files, you can try placing them under the \Resources\android\ or \Resources\iphone\ directories respectively. If that doesn't work, you can try placing them under \Resources\platform\android\ or \Resources\platform\iphone\ respectively and see if that works. The easiest way to test this is to build for Android and then open the resulting APK file with WinZip and see if the iPhone files got included.
