iOS SearchBar Cancel Bug?

I have a SearchBar in a TableView that can include 1,000+ entries. I'm including a "Search" button as a NavButton so that if a user has scrolled down the page, they can quickly see the SearchBar without having to scroll all the way back to the top. However, if "Search" button is pressed (when the SearchBar is not visible) and you try to click the cancel button in the SearchBar, the cancel event does not fire.

Can anyone else recreate this? Is there a different/better way to do what I'm trying to do?

<LeftNavButton>
    <Button onClick="showSearch">Search</Button>
</LeftNavButton>
<TableView id="Table" onClick="events_click" search="SearchTable">
    <SearchBar id="Search"/>
</TableView>

function showSearch(){
    $.Search.focus();
}
