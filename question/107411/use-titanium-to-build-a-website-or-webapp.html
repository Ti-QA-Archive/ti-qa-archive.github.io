Use Titanium to build a website or webapp?

I'd like to build my app once for my website, mobile app, and desktop. I see all over Titanium to build for desktop and mobile, but what about for a website? Do I just strip out all of the Titanium APIs for the website version? What is the workflow to build for all three spaces? Should I build the website first, and then port it to the mobile app and desktop app with Titanium?

  • Bryce

    You would build your website/service first, with your audience probably using a web browser to access it. Your mobile devices can also use their web browser to do the same.

    However, where Titanium is useful is to develop a software client to your website that is able to utilize the powerful native capabilities of mobile devices. This gives a much more engaging, immersive and user-friendly experience for your users.

    Hope this clarifies it.

    Cheers
