  • showAsAction="Ti.Android.SHOW_AS_ACTION_IF_ROOM"

    Here, you are trying to put a MenuItem into the ActionBar as an ActionItem. Not sure, but this could probably cause the error.

    If you are trying to use an OptionsMenu without an ActionBar then you should use Ti.Android.SHOW_AS_ACTION_NEVER

    But the question comes to my mind, why you would use an optionsMenu without an ActionBar? Where do you want to show the menu?

    • First, the statement "nearly every Android device has an "menu" hardware button" is wrong (it was true like 5 years ago) - Menu buttons are prehistoric (and this is how your screenshot looks like)!

      With the introduction of Android Honeycomb and the ActionBar in early 2011, Android has removed the reliance on physical buttons. For backward compatibility they've provided a software menu button to run old Apps!

      “Beginning with Android 3.0, the Menu button is deprecated (some devices don't have one), so you should migrate toward using the action bar to provide access to actions and other options.“ (src)

      Apps that were developed after 2011 should implement OptionsMenu following Creating an Options Menu

      If you you still try to use the menu button in 2014, then STOP CODING now and get to know the Android Platform and read this: Say Goodbye to the Menu Button (posted in Jan. 2012)

      My recommendation is:

      • don't use Menus without the ActionBar
      • test your apps on latest version (maybe make it backward compatible to older versions) - not vice versa
      • for your case: Refresh, Compose, … these are ACTIONS - use an ActionBar and put in in there

      Best,
      Manuel
