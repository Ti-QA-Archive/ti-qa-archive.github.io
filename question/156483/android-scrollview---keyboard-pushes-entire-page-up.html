  • I figured it out - if I include this line of code the keyboard does not push up elements anymore:

    $.win.windowSoftInputMode = Ti.UI.Android.SOFT_INPUT_ADJUST_PAN;

    • the $. in this line is used in Alloy markup and refers to an element in the xml view pages. If you aren't using Alloy or you define a window in the .js controller pages instead, you don't need to add $. in front. The code would then simply read: win.windowSoftInputMode. (or whatever id you have given your window). I placed this line of code in my global stylesheet for Alloy (app.tss). But you can place this line at the top of your .js pages in titanium as well.
