  • Hi,

    Try pointing to your scripts with absolute urls, rather than relative ones. I had this issue with an iPhone app compared to an Android one. (Very weird, I know!)

    Create a variable to locate the Resources folder using Titanium.Filesystem.resourcesDirectory and build the location url in your 'test' properties using that.

    The way you're doing it should work, but alas, it doesn't work cross-platform. Most odd (and probably a bug)

    Hope this helps!
