iOS5 NavigationGroup and TabGroup bugs (animations, back, etc.)

This is related to this question but contains some other bugs and problems.

To me, this (back navigation broken, button as TitleControl of window missing) happens with a NavigationGroup inside a TabGroup.

BUT the bugs go away, when I select another tab, switch back to tab with the NavigationGroup and then open/close windows in the NavigationGroup.

Also, when the application starts with the tab containing the NavigationGroup not selected, the NavigationGroup is placed ~40px further up than it is supposed to. This hides the title bar.

All these bugs only affect iOS5 and also appear in the simulator. With 1.7.5 and the current master. Problem is, we have an app in the app store showing exactly that behaviour, people are not happy and I can't find a way to fix this.

Is there a way to "simulate" a tab change, so that whatever causes the bugs to go away when switching tabs happens until this is fixed in the SDK?

  • Ok, I have found a quick solution, but it involves getting rid of your navigationGroup completely, which probably won't help some of you.

    Instead of using navigationGroup.open(window), get rid of your navigationGroup and just add the window that was in the navigationGroup to a Tab. Then call tab.open(window), which also works like this: tab.open(window,{animated:true}).

    This fixed the "back" animation issue for me.
