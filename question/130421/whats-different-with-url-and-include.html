What's different with url and include

Hellow , I need to know what the different between using include and createWindow with url.

  1. in app.js to using Ti.include('ui/HotNews.js')
  2. in app.js to createWindow using url
    var WinHotNews = Titanium.UI.createWindow({  
 title:'????',
 backgroundColor:'#fff',
 url : 'ui/HotNews.js'
});
    If there's different which one is the best wan to performance?
    thanks for helping.

  • Accepted Answer

    "Ti.include" includes source code in current context (like it's copy-pasted in place where "include" is located), while using window's "url" property creates new Titanium/JS context. In short - use "include" or "require" and CommonJS since multicontext environment is unstable and unpractical since everything needs to be reincluded in every new context you create.

  • Yang. Best practice for Titanium development is to develop your application in a single javascript context. If you look around in the documentation you will find lots about this subject (and using the Require method as an alternative to the Include statement). The example you give above (which is similar to the method currently used in the Kitchen Sink app) would not be generally considered Best Practice. The Kitchen Sink application is great for looking at examples of how to make various API calls but it is not meant as a good example of application architecture/design.
