Alloy: Use <Require/> to wrap a child-element?

Hey there,

i'm currently starting to migrate my apps to use Alloy and it pretty cool already! But there is currently a problem to wrap a required UI element to its child views (e.g. <ListSection> and <ListItem>). Here my snippet:

(index.xml)

<Alloy>
  <Require src="section" type="view" id="mySection">
    <ListItem title="TestRow" />
  </Require>
</Alloy>

(section.xml)

<Alloy>
  <ListSection title="My Section Title"></ListSection>
</Alloy>

The wrapper is not recognized in the compiling-proccess. What am i do wrong here? Thank you!

  • You'll need to use a structure like the one provided by Larrie. Overall, I'm not sure of your goal in attempting to structure your code this way. The idea of a require is to divide up your code into more manageable, and possibly reusable, files. With a listview, you're going to do all the styling in the template, and data is added dynamically in the controller (usually). I'm not seeing the benefit you hope to get by breaking up the list view's XML into require files.

  • Accepted Answer

    You need to place the ListItem within the ListSection, so index.xml would be

    <Alloy>
  <Require src="section" type="view" id="mySection" />
</Alloy>

    and section.xml becomes:

    <Alloy>
  <ListSection title="My Section Title">
    <ListItem title="TestRow" />
  </ListSection>
</Alloy>
