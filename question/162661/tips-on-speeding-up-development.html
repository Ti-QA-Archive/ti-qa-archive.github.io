Tips on speeding up Development

I'm sure this has come up before, but with all the new releases (thanks for that BTW), are there any tips for Developers to speed up Development. When I need to make any change, it takes about 1 minute for everything to compile and get loaded on the simulator. When the app loads, I need to navigate to the right place to test my code.

iOS simulator loads faster than Android's. I actually connect a physical device to test on Android. Do folks do the same?

Any tips?

    Hi

    You are not going to like my answer. I have the advantage of being old and therefore having been writing software long before the modern computer.

    The average modern developer writes a few lines of code, compiles - test to see if it works and if it fails, tweaks one value, compile again and continues this process until something actually works.

    Back in my day (yes I am old enough to use that phase), we wrote code on paper, debugged on paper, ran test data sets against the written code. Once this all appears to work and only then did we enter the code into the computer. Now I am very well aware that things have moved on, the amount of code and the things that need to interconnect have all increased - so the older way of writing code on a sheet of paper first has come and gone. However the logic that it created can and should be taken into account.

    What I am suggesting is that modern developers should take much more care in creating the code they are writing. By not relying on a compile to confirm if code will work and instead test the process in your mind first will give you significant increases productivity not only by confirming your own code long before you actually run but also increasing your ability to remember all the keywords and parameters.

    So on to tips;

    • Read the documentation
    • Review to the documentation
    • Review the release notes for each update
    • Plan your app, plan everything, revisit the plan and then write code - not before
    • Wireframe your app before you write any code, it helps you work out which things are common - these you write once and re-use
    • Break up your code into lots of smaller manageable modules
    • Define what parameters each module should accept and what outputs the module can provide
    • Learn when to use CommonJS and when to use in-line functions
    • Use GenyMotion rather than the Android Emulator - it is much better and much faster
    • Use physical devices as much as possible as they better reflect the speed and memory of an actual device
    • Look into TiShadow
    • Always consider the code you are writing can and should be designed for re-use
    • Always consider the code you are writing can and should be used for more than one app
    • Learn about 'underscore' and benefit from it (included in Alloy apps)
    • Learn about 'backbone' and benefit from it (included in Alloy apps)
    • Learn about 'i18n' and make extensive use of it (only ever store string here)

    Alloy;

    • Use Alloy if you are not already doing so. If you are mid-way through a project wait until the next project
    • Use Widgets, if for nothing else, break up your app into sections and use a Widget for each section.
    • Always use TSS, never add properties to the XML views
    • Make good use of the 'app TSS'
    • Learn about 'momentjs' and benefit from it (included in Alloy apps)

    The obvious things;

    • Buy a faster computer
    • Buy as much RAM as possible
    • Buy an SSD as the primary drive
    • If you are using a SSD then try and get one that connects via PCI

    Everything else is app specific.
