Alloy Model Sync

I read all the documentation and watched a bunch of videos highlighting the new Alloy MVC framework but I can't figure out how to get model sync working. I expect from what I read that it will allow the controller to query a RESTful json web service and store it locally in the SQL database.

So Far I have this model

exports.definition = {

    config: {
        "columns": {
            "name":"String"
        },
        "adapter": {
            "type": "sql",
            "collection_name": "event"
        }
    },        

    extendModel: function(Model) {        
        _.extend(Model.prototype, {

            // extended functions go here

        }); // end extend

        return Model;
    },


    extendCollection: function(Collection) {        
        _.extend(Collection.prototype, {

            // extended functions go here            
            url: 'http://0.0.0.0:5000/events.json'
        }); // end extend

        return Collection;
    }

}

And this for the controller

$.index.open();

var eventList = Alloy.Collections.event;
eventList.fetch({
    error : function(collection, response) {
      console.log('error');
    },
    success : function(collection, response) {
      console.log('error');
    }
});

The migration was performed and it is reading from the event's table, but its not aware of the API its supposed to pull data from. I have a rails app with an events resource and I can get the information from that.
