  • In this case, this error is most likely caused because you originally built your app from Titanium Studio for Simulator, then opened the Xcode project and tried to build for device. There was a bug for this scenario that we fixed in TIMOB-11599. The fix has been merged, but hasn't been tested.

    If you're feeling adventurous, you can preview the fix by running:

    titanium sdk install --branch 3_0_X

    That will download and install the latest 3.0.X CI build. Again, this build has not been thoroughly tested and should not be used in production.

    From Studio, clean your project, then edit the tiapp.xml and select the new Titanium SDK version. Next build your app again, open the generated Xcode project, and should be able to build without any problems.

    • 3.0.0.GA has a bug where if any JavaScript files have a syntax error, the build will success, but won't run (i.e. app.js is not found). We fixed this shortly after 3.0.0.GA shipped, however we have not had a stable release since 3.0.0.GA. The upcoming 3.0.2 release will have the fix.

      In the meantime, you are welcome to try the latest 3.0.2 build by running the following:

      titanium sdk install --branch 3_0_X --default

      The 3.0.x branch is not 100% stable yet, so feel free to use it in development, but don't ship any production code with it quite yet.

      Also remember to update the <sdk-version> in the tiapp.xml to the latest 3.0.2 release that you just installed.

    • @Stephan, hmm, I bet I know what's going on. It's possible somehow for the symlinks to get messed up when installing the app in the iOS Simulator.

      Let's first try cleaning your project, then build for simulator. When the simulator starts up and assuming that it continues to say app.js not found, then from the "iOS Simulator" menu, select "Reset content and settings". Close the simulator and try building again.

      If it still errors, open a Terminal and run:

      ls -l ~/Library/Application\ Support/iPhone\ Simulator/<SIM_VERSION>/Applications/<IOS_GUID>/<APP_NAME>.app

      Obviously replace the tokens above with the appropriate values. The directory should contain a symlink to your app.js:

      lrwxr-xr-x   1 chris  staff       49 Jan 31 17:24 app.js -> /Users/chris/appc/workspace/test/Resources/app.js

      I've seen instances where this symlink isn't copied correctly. I have no clue what I did to my machine to fix this when I caused this error. I do recall having some filesystem corruption and had to run Disk Utility to verify the disk and eventually repair it. I'm not saying your hard drive has issues, but it is possible. Proceed with caution.

      I know you don't want to upgrade to 3.0.x, but I've added a new flag called –force-copy that will copy all files when building for simulator instead of symlinking them. Might come in handy when 3.0.2.GA is released and you're ready to upgrade.
