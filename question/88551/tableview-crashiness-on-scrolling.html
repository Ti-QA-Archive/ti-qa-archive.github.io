TableView crashiness on scrolling

Hi All,

I have a table heavy application which was working just fine until I added a table made up of rows comprising of multiple labels.

This table, unlike all my other my tables, creates each row individually using createTableRow and add 4 small formatted text labels.

When I scroll this table up and down it will eventually crash. This is guaranteed to happen. The number of rows displayed is inversely proportional to the time to crash. Ie. more rows, less time to crash.

All of my other tables, with many more entries do cause any crashes but they are all much simpler using title parameter to set the row title.

All by DB access is opened and closed correctly, the row construction is smooth.

Has anyone else faced this problem? Any suspects for the cause?

    Nick

    With complex rows, as opposed to simple ones that you have been using up until now, it's important to set the className property with a unique value for rows with the same layouts (including for simple rows).

    Secondly, would you run the KitckenSink > BaseUI > Views > Table Views > Table View (Layout 2) and report back about whether it exhibits the same behaviour?

    If the problem persists, please post your code, and someone will take a closer look.

    Cheers
