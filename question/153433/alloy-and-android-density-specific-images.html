Alloy and Android density-specific images

I am adapting an iPhone app to run on Android using the Alloy framework, and am encountering a problem with density-specific image folders when building. The result is that images within the density-specific folders (eg. app/assets/android/images/res-hdpi) are not being displayed, with the app reverting to the version in the default images folder.

On digging around in the built project I found that images are being renamed when copying to the build folder. Here is an example:

watermarkProjectIcon.png becomes watermarkprojecticon_3f6fdabcab.png

This is echoed in the build log:

[TRACE] COPYING FILE: /Users/julian/Documents/Mobile/Titanium/PROJECT/Resources/android/images/res-hdpi/watermarkProjectIcon.png => /Users/julian/Documents/Mobile/Titanium/PROJECT/build/android/res/drawable-hdpi/watermarkprojecticon_3f6fdabcab.png
[TRACE] COPYING FILE: /Users/julian/Documents/Mobile/Titanium/PROJECT/Resources/android/images/res-mdpi/watermarkProjectIcon.png => /Users/julian/Documents/Mobile/Titanium/PROJECT/build/android/res/drawable-mdpi/watermarkprojecticon_3f6fdabcab.png

Has anyone else encountered this behavior? I would like to determine if this is a bona fide issue and not just my missing something before posting it to Jira. Any suggestions on a fix or viable workaround would be much appreciated.

Environment

  • OS X 10.8.3
  • Mobile SDK 3.0.2.GA
  • Alloy 1.1.2
  • Ti Studio 3.1.0.201304151600
  • Testing on HTC Desire S, Android version 2.3.5

  • The file copy names are a red herring here. They are just an intermediary file and the name with the hex value does not impact the app. The images will be located where you expect, at &#x2F;images&#x2F;watermarkProjectIcon.png.

    To help reduce confusion, I added a test app to the alloy github repository that shows exactly how to handle android density specific images files. It's a very basic app to make it clear exactly how it works: https://github.com/appcelerator/alloy/tree/master/test/apps/advanced/android_density

    appcelerator Q&A

    Michael G posted his problem also in the issue tracker. There I found the advice to change the path to the image.

    As an example, instead of using
    image: &#39;header_logo.png&#39;
    use
    image: &#39;&#x2F;images&#x2F;header_logo.png&#39;

    It works, but who knows how long …
