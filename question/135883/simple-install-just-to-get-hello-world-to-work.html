Simple install just to get "hello world" to work

I've just spent hours attempting to get appC's Ti-Studio to install and compile the "hello world" app or the "LoanCalc" in "Appcelerator Titanium Smartphone App Development Cookbook" by Boydlee Pollentine.

Before going further, I'm on Windows XP with SP3 and all updates are current. The Java SDK version installed is 1.6 and the Android version was 2.2 with all the other models stated as required here:

Here's the problem, I keep getting errors/bugs. Here's a few examples:

  • [ERROR] Error locating JDK: set $JAVA_HOME or put javac and jarsigner on your $PATH (Believed I fixed this )
  • [Android SDK Home] Exception: No "android-8" or "android-2.2" in the Android SDK (Appears 2.2 is required, after I installed it via the Android model manager, the error went away, and the Ti-Prefs pulled in 2.2 as the version being used; I had installed the latest version of Android first.)
  • ImageMagick not installed, aborting icon conversion. (No idea, though this the error was related to the build for the "hello world" I uninstalled everything (Ti-Studio, Java-SDK, Android) and then reinstalled just Ti-Studio. This didn't make a difference, since I got the same error.)

So, here's the question…. Why is this so hard to install? It's got to be in the top three of most complex IDEs I've ever done. Meaning the is no way to tell way to simply test if a install is right for a given configuration, or trace where the build for them is breaking down. Which is to say ever time I fix one error/bug, it's completely unclear if the error message that just went away created another error, or the next error was unrelated, etc. Beyond that, every single error/bug I ran across and Googled had lots of other people with the same issues/errors/bugs/etc – believe one page even had 30000 views, which is amazing to me.

Am I missing something? Why is even getting the "hello world" desktop app to compile so hard?
