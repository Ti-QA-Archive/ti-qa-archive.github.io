Geolocation preferred provider on Android

Mac OSX 10.5
Studio 1.08
Titanium API 1.8.0.1

I have read through the postings about using Geolocation.preferredProvider and that if you don't set it, Android will figure out the 'best' provider. The issue I'm having is that if I don't set the provider, the device starts the GPS radio (you can see the icon in the status bar) and if you are in range of a satellite (ie. outside) it works fine. When I move inside and run the same app, the GPS radio starts but NEVER switches to the network as provider. If I change the code to chose the Network as the preferred provider, it works inside, but never switches to the GPS (I'm guessing it's not supposed to).

So the question is: How do you get Android to 'pick' the best provider automatically. It doesn't seem to do it by NOT setting preferredProvider… Are others seeing this? Is it a bug?

Thanks,

Ray

      3. Run Advanced Task Killer again to be sure that the program is not running anymore.
      4. Go outdoors and open the app again. You should see that the value "Updated location" a better accuracy level (indoors you will see something like 46, but outdoors with a real gps, you should get like 4 - the less the better).
