    • The way you started suggests exactly this. You need first to know what your app does, then split it coponents and each component split it in small modules.
      Something like this:

      Geolocation chat app.

      This will need a central server and the mobile clients (the app itself).

      The mobile client will need a way to detect the location, to identify itself (what user it's using it) and to communicate with the server

      the location- solution

      identify itself - this needs to be splited even more - login, sign in, logout, the UI of this forms and the communication with the server, etc

      communication with the server - solution

      etc…

      So as you see, if you split your idea in small bits, you will find yourself what you need to do and even pretty easy. Then your problems will be very specific and you will be able to ask very clear questions but of course only after you tried yourself to fix it.

      So DON'T ask for code, DON'T ask for ready made solutions, but ask HOW your problem can be solved AFTER you tried yourself and SAYING what you tried to do.
