ACS admin user

It appears that the Appcelerator guys have just recently introduced an admin user concept in ACS, making it possible to change custom objects that the admin may not be the owner of. Apart from that, there's not a lot of documentation about the admin role at this point, since it's so fresh. Does anyone know:

  • what privileges the admin user has? (A concrete example: I need to be able to delete users via the API - preferrably without logging them in, as I don't know their password, but if I can't do that, I'd like a way to get their passwords so I can log them in and THEN delete them.)

  • the parameters needed to use them? (In my above example, I am calling the REST interface.)

An example call would be awesome. Thanks in advance.

  • Accepted Answer

    Hi, We just added setting app user to admin on our website. we are still working on enabling admin access to all objects and adding documentation.

    Right now if you logged in as admin user, you can update/delete custom object and places. We will make it available for all objects soon.

    It currently can't delete other users but we will have it soon, basically you first login as admin user, then call user delete and pass in user_id of the user you want to delete. If you could email me at wkong@appcelerator.com and I can let you know as soon as it is available.
