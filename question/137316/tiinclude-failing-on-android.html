  • I have the same problem with android. In my use case I include the jsoauth-1.3.4.js in my own module. In iPhone it works fine.
    I think it is a problem with the "special" structur of this file.
    But there are an other version of jsoauth in commonJS manier, but there are not example how to embed it.

    Here my code:

    OA.prototype.start = function(_callback) {
    var self = this;
    Ti.include('/lib/jsoauth-1.3.4.js');  //  <====
    var oauth = new OAuth(this.oauth_options);
    oauth.fetchRequestToken(function(url) {
        self.web_options.url = url;
        var webview = Ti.UI.createWebView(self.web_options);
        webview.scalesPageToFit = true;
        var parent = self.web_options.parent;
        parent.add(webview);
    }, function() {
          // errorhandling
    });
};

    It works very fine on iMachine, but not on aMachine. Android give me the message 

    Location:
lib/poiauth.js  //line with constructor
Uncaught Reference Error: OAuth is not defined
