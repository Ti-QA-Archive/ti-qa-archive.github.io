Correct Titanium SDK folder location? Documentaion location C:\ProgramData\ is NOT CORRECT

Hi there

I am trying to use one of the new builds of the Titanium SDK for Android, and the location of the folder does not exist as per the installation instructions.

In
http://developer.appcelerator.com/guides/en/installing_on_windows.html#titanium_sdks

The guide says that once folder options are set to un-hide hidden folders;

"It will now be possible to navigate to the C:\ProgramData\Titanium\mobilesdk\win32 directory, where Titanium SDKs are stored"

But that folder does not exist.

My issue is that I want to use the latest mobile SDK build 1.6.0 - so that I can test an app build with a Samsung Galaxy device - and following the instructions at:

http://developer.appcelerator.com/blog/2010/12/festivus-celebration-day-five-festivus-miracles.html#titanium

It says I need to extract the 1.6.0 folder to:

/Library/Application Support/Titanium/mobilesdk/

Where is this path???

The conflicting documentation for Android installs makes using the Titanium platform very difficult.

Please let us know the right folder location to extract new mobile builds.

  • Gabriel,

    The problem is the multiplicity of platforms and OS levels. And beyond that stuff also depends on if the software was installed for all users or a specific user.

    All the references you see to paths which start /Library/…. or ~/Library/… are for OSX installations – and maybe *nix too – but not for Windows installs.

    You seem to be runinng some flavor of Windows. On my Win7 machine, the correct folder is in fact C:\ProgramData\Titanium\mobilesdk\win32

    I think the same is likely try for Vista, but for XP I don't think the C:\ProgramData folder existed.

    What OS are you using, XP? If nothing else, do a folder search for Titanium or mobilesdk. Somewhere you should find a path C:(unknown)\Titanium\mobilesdk\win32, and under that the various SDK levels you have installed.

    Put it alongside those.

    Doug
