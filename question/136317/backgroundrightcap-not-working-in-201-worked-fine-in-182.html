backgroundRightCap not working in 2.0.1 (worked fine in 1.8.2)

In 1.8.2 I had a button that was using backgroundLeftCap and backgroundRightCap. in 1.8.2 it worked perfectly, but in 2.0.1 the backgroundRightCap value seems to be ignored and the right cap is stretched.

The button is designed to be nearly full width by using left:5, right:5 and not specifying a width. According to the docs for 2.0 it appears that backgroundRightCap may be missing in 2.0.1 and the value specified appears to be ignored.

In the description of the backgroundLeftCap, it states …

This property specifies the size of the left end cap. The middle (stretchable) portion is assumed to be 1 pixel wide. The right end cap is therefore computed by adding the size of the left end cap and the middle portion together and then subtracting that value from the width of the image

but this doesn't appear to be the case. Instead, it appears to stretch the middle the width of the right cap, and the right cap the width the middle should be…

I've tried changing the width of the button to a static width, but that didn't help either. Am I missing something or is this a bug in 2.x?
