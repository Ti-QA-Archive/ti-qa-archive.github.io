Mapview in a ScrollView on Android not working

Hi all,

I have a Window ( tried modal or not modal ) with a ScrollableView:

    var scrollView = Ti.UI.createScrollableView({  // wrong variable name ;)
        height: '90%',
        width: '90%',
        borderColor: '#000000',
        borderWidth: 1,
        backgroundColor: '#ffffff',
        scrollingEnabled: true
    });

This ScrollView has two Views, the second is for a MapView:

    var mapView = Ti.UI.createView({
        width: '100%',
        height: '100%',
        backgroundColor: '#ffffff',
        borderColor: '#000000',
        borderWidth: 1,            
    });

In that second View I add a MapView:

    var mapVw = Ti.Map.createView({
        top: '40dp',
        left: '10dp',
        right: '10dp',
        height: '300dp',
        mapType: Titanium.Map.STANDARD_TYPE,
        region: {latitude:52.0774299, longitude:4.3160544, 
                 latitudeDelta:0.001, longitudeDelta:0.001},
        animate:true,
        regionFit:true,
        userLocation:false
    })

    scrollView.views = [firstView,mapView];

    mapView.add(mapVw);
    self.add(scrollView);

This shows only the Grid and Zoom buttons but no map.

Is this a bug or just not possible?

Ti SDK: 2.1.1.GA
Android: Google API 11 - 14
Happens on emulator and device

When adding the mapView direct to self, it works so no Maps API issue…

Thanks for any help!
