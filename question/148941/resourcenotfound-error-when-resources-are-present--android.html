ResourceNotFound Error when resources are present : Android

I have an application that works fine on Iphone and runs on Android Emulators. However when I come to try and run it on one of my many Android devices it fails.

Using Install on Android Device I am able to get the application onto the device so that it appears in the App tray.

When I run the application the splash screen appears, but at the point that it goes to display the first window it crashes.

Using DDMS I get an error of which this is an extract (feel free to ask for more):

02-28 17:58:07.747: E/AndroidRuntime(6638): FATAL EXCEPTION: main
02-28 17:58:07.747: E/AndroidRuntime(6638): java.lang.RuntimeException: Unable to start activity ComponentInfo{com.ipsos.beautyi60/org.appcelerator.titanium.TiActivity}: android.content.res.Resources$NotFoundException: Resource ID #0x7f020015
02-28 17:58:07.747: E/AndroidRuntime(6638): at android.app.ActivityThread.performLaunchActivity(ActivityThread.java:2202)
02-28 17:58:07.747: E/AndroidRuntime(6638): at android.app.ActivityThread.handleLaunchActivity(ActivityThread.java:2237)
02-28 17:58:07.747: E/AndroidRuntime(6638): at android.app.ActivityThread.access$600(ActivityThread.java:139)
02-28 17:58:07.747: E/AndroidRuntime(6638): at android.app.ActivityThread$H.handleMessage(ActivityThread.java:1262)

This indicates that a resource is missing with a reference of 7f020015.

I have looked up this resource in Java and found it refers to a landscape background image called bg_horizontal_da63af802c

I have looked in the three landscape folders for images in the Build and all three of them show this file.
This file does not exist in any of the portrait folders as it is a landscape image not a portrait image.

Can someone explain how to resolve this issue.
