Launch android app trough push notification, parse startup variables

Hi,

I would like to know if it is possible to add some startup variables to the app which is launched trough the push notification, so that this app knows what to show. What i have working now is a notification that when you click it, it just opens the app it needs to open, but what i would like to achieve is that when the app is launched from the notification, the app knows it was launched from the notification and then immediatly redirects the user to a predestined page for showing the information showed also in the notification. I am wondering how this is achieved without some expensive module.

this is the notification sending code i have now:


var intent = Titanium.Android.createIntent({
    action: Titanium.Android.ACTION_MAIN,
    className: 'app.id.AppActivity', 
    packageName: 'app.id'
});

intent.addCategory(Titanium.Android.CATEGORY_LAUNCHER);

var pending = Titanium.Android.createPendingIntent({
    activity: Titanium.Android.currentActivity,
    intent: intent,
    type: Titanium.Android.PENDING_INTENT_FOR_ACTIVITY, 
    flags: Titanium.Android.FLAG_ACTIVITY_NEW_TASK
});

var notification = Titanium.Android.createNotification({
    contentIntent: pending,
    contentTitle: 'A Notification',
    contentText: 'Time to feed the cat ...',
    tickerText: "Feeding time ...",
    when: new Date().getTime()
});

Ti.Android.NotificationManager.notify(1, notification);

what i would like to have is either an onclick event of the notification, so i know the user clicked the notification, or set some variables in the Ti.App object so when the app starts, i can check wether the app was started trough the notification or started the normal way.

I am hoping someone can help me on this, because i am out of options.

Thanks in advance,

Hylke Bron
