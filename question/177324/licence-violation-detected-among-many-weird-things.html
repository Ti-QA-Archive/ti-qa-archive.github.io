    I'm afraid you're going to have to remove all the modules from your app, add them one at a time, and see at which point you get the license violation message. We have tickets open to provide more useful information when the license violation is detected. But as of now, as best as I know, this is the only way to see which module is causing the issues.

    Changing the GUID of your app won't help. It's the GUID in the module that's the issue. What I suspect has happened, because it has happened in the past, is that someone takes a free module and submits it to the marketplace without changing the GUID. So now, you download from GitHub (gittio), the check is done to our servers, which finds a matching GUID in the list of published modules that you haven't bought (or downloaded) from the marketplace, and the violation error is thrown.

    The original author of the module will need to republish his/her module to GitHub with a new GUID to fix this. And, he/she should remove the version from the marketplace (if it's theirs) or contact the person who published the module to the marketplace to remove it from there.

    As for your log messages, **[WARN] : V8Object: Runtime disposed, cannot set property &#39;userAgent&#39; can be ignored. It happens all the time. The others you starred are probably all a result of the problematic module not loading. Fix that and I'll bet they'll go away.
