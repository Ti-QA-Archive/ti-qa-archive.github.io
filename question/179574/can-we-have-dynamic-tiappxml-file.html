  • You have a couple of options, depending on whether you're building an Alloy project or a Classic project.

    Alloy

    Using a JMK file, also called a build configuration file), you can effectively run a pre-compile script to modify the tiapp.xml. See the Maps sample app that ships with Alloy for an example of a JMK file that modifies the tiapp (to add map keys). You might need to look at an older version of Alloy because I think we removed that functionality in the current version. See the GitHub repo for the older branches.

    Classic

    Using a CLI plugin you could modify the tiapp using a hook. Off the top of my head, I think you'd hook in at the cli:pre-validate event.

  • Hi Aparna,

    It's funny that i'm in need of the same thing, i found this trick i can use.

    basicly you can have multiple tiapp.xml files, keep them in dedicated folder which i named : tiapps.

    for now i have only two tiapps files named doodleApp_tiapp.xml and testApp_tiapp.xml.

    I deleted the original tiapp.xml and replace it with an alias as a shortcut to point to one of my two (or even more in your case). just google unix alias file,
    i created one using the command :

    ln -s tiapps/doodleApp_tiapp.xml tiapp.xml

    i'm working on a Mac of course, but even on windows i believe we can use a shortcut named tiapp.xml that points to a desired tiapp.xml among multiple once.

    Hope this will help out :)
