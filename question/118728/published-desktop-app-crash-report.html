  • Accepted Answer

    Without knowing anything about the underlying libraries, it looks like the problem is a SIGABRT while trying to create a directory. My guess is that the app tried to create a directory and hit an assertion that aborted the program when it couldn't.

    I can't tell if your code is explicitly creating a directory or if it's being done as a side-effect of some Ti initialization. The first thing to look at would be whether or not you have write permission to the usual directories: $HOME, $HOME/Library, $HOME/Documents, etc. If that doesn't pan out try installing the app to a directory you know you have access to. Maybe it (or Ti) is trying to write to its install directory by mistake.
