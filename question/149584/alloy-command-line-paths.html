Alloy command line paths

I am following this doc and trying to build an alloy view component it works but the output path parameter seems to have no effect.

http://docs.appcelerator.com/titanium/latest/#!/guide/Alloy_Command-Line_Interface_Reference-section-35620083_AlloyCommand-LineInterfaceReference-ModelFormat

For instance when I run this "alloy generate view testView -o outputTest"

I am getting

[INFO] Generated view named testView
[INFO] Generated style named testView

But the view is on
app/views
shouldn't be on app/outputTest/

Probably I misunderstanding the –outputPath parameter ,
if so is there a way to organize my code on a HMVC file structure with the use of command line ?

  • The –outputPath is used only for the "alloy compile" command. Alloy 1.0.0 does not support subdirectories for the "generate" command.

    Once Alloy 1.1.0 is released, though, you can create views/controllers/styles in specific directories. Just list the full relative path when using the create command. So in your example above when you are attempting to create a view named "testView" in the folder "outputTest", you would do the following:

    alloy generate view outputTest/testView

    This would place the view "testView" in app&#x2F;views&#x2F;outputTest&#x2F;textView.

    For reference, the ticket where this was resolved is here: https://jira.appcelerator.org/browse/ALOY-546
