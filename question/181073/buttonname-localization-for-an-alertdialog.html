ButtonName localization for an AlertDialog

I'm creating an Alert Dialog in alloy and wanted to have the button names localized, but I can't find the appropriate attribute to set. When I used textid or titleid it did not work. here's a sample:

<AlertDialog id="AlertDialog" titleid="alert_title" messageid="alert_message" onClick="handleAlertDialog">
                <ButtonNames>
                    <ButtonName textid="upload_docs"/>
                    <ButtonName textid="localized_ok"/>
                </ButtonNames>
            </AlertDialog>

I can't find any documentation on button name, but it seems I may have to do this programmatically.

  • Hi Joshua ceaser,

    @Mr.harish it is possible to localize the title, message and button also of alert dialog..

    if you create alert dialog the you can use L() function which is the default localization and one more way you can it by using custom localization. see this custom localization using .js file..

    After implementing this thing you can create alert dialog like this..

    var alert = Ti.UI.createAlertDialog({
        title : Alloy.Globals.selectedLanguage.appTitle,
        message : Alloy.Globals.selectedLanguage.exitConfirm,
        buttonNames : [Alloy.Globals.selectedLanguage.ok, Alloy.Globals.selectedLanguage.cancelTitle]
    });
    alert.addEventListener('click', function(e) {
        if (e.index == 0) {
            //Ti.Android.currentActivity.finish();
            $.winHome.close();
        }
    });
    alert.show();

/*

Here appTitle, exitConfirm, ok and cancelTitle declare in both file english.js and arabic.js
