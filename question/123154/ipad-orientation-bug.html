iPad orientation bug?

Recently I found a bug in iPad, application "nbc" has the same bug:

If I want my iPad app orientation only support LANDSCAPE_LEFT and LANDSCAPE_RIGHT,

first, update the file "tiapp.xml"

&lt;orientations device="ipad"&gt;
&lt;orientation&gt;Ti.UI.LANDSCAPE_LEFT&lt;/orientation&gt;
&lt;orientation&gt;Ti.UI.LANDSCAPE_RIGHT&lt;/orientation&gt;
&lt;/orientations&gt;

second, update the file "Info.plist"

&lt;key&gt;UISupportedInterfaceOrientations~ipad&lt;/key&gt;
&lt;array&gt;
&lt;string&gt;UIInterfaceOrientationLandscapeLeft&lt;/string&gt;
&lt;string&gt;UIInterfaceOrientationLandscapeRight&lt;/string&gt;
&lt;/array&gt;

third, set window property

window.orientationModes = [Ti.UI.LANDSCAPE_LEFT,Ti.UI.LANDSCAPE_RIGHT]

but if the orientation of iPad is PORTRAIT at first, then launch the app, you will notice that it will show the startup image LANDSCAPE_LEFT or LANDSCAPE_RIGHT, then rotate to PORTRAIT until window opened.

Anyone has the same problem? How to resolve this?

  • I ran into something similar awhile ago. How I got around this issue was in my app.js I check if the orientation is LANDSCAPE_LEFT or LANDSCAPE_RIGHT. If not I set it to LANDSCAPE_LEFT

    After it was set the first time it seemed to take over. This doesn't resolve the issue of which default.png is displayed though.

    Hope this works for you.
