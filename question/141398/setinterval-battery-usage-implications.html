setInterval: battery usage implications?

I'm using setInterval to display a simple timer, updated every second (ie. the interval is 1000).

Two questions:

  • Does this negatively impact battery life in a tangible way?

  • How does setInterval interact with the app going in the background? I'm assuming (hoping) setInterval does not run while the app is in the background, and that having setInternal doesn't prevent the app from going into the background (doesn't have any negative battery usage implications). Does setInterval automatically restart when the app comes out of the background?

This is for both iOS and Android.

  • Accepted Answer

    • Running a timer to display something like a clock is not going to affect the battery in an overly negative way.
    • Your interval will not run if your app is not running unless you put it into a background service.
    • You can listen for the resume event of the app to restart an interval if that is required.
