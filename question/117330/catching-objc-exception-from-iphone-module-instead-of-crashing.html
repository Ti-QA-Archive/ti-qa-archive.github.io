Catching ObjC Exception from iPhone module instead of crashing…

I'd like to build an iPhone module with Objective-C. One function I created looks like this:

/* … */

@implementation MyModule

DEFINE_EXCEPTIONS;

/* … */

#pragma Public APIs

/* example method */
-(id)example:(id)arg
{
  ENSURE_SINGLE_ARG (args, NSDictionary);

  return @"Hello World";
}

/* … */

The JavaScript file in example&#x2F;app.js looks like this:

/* … */

try {
  mymodule.example();
} catch (e) {
  Ti.API.info (e);
}

/* … */

When I run the example app from the command line, my code throws the exception just as I expected… However, this exception does not get caught by my JavaScript code, instead it terminated the program with a stack trace…
