Bind values undefined in event handler

Hello.
I have problem with passing bind values to event handler on Android. I have simple Alloy controller with ListView.
XML:

<Alloy>
    <Window class="container" layout="vertical" onOpen="openWindowHandler">
        <ListView id="imageListView" defaultItemTemplate="template1">
            <Templates>
                <ItemTemplate name="template1" height="96dp">
                    <ImageView bindId="imageView" onLoad="imageLoadedHandler" />
                </ItemTemplate>
            </Templates>
            <ListSection id="section">
            </ListSection>
        </ListView>
    </Window>
</Alloy>

Code:

var args = arguments[0] || {};

function openWindowHandler() {
    var items = [];
    items.push({
        imageView: {
            image: "http://docs.appcelerator.com/titanium/latest/resources/images/appc_header.png",
        }
    });
    $.section.items = items;
}

function imageLoadedHandler(obj) {
    var imageView = obj.source;
    if (typeof(imageView.image) != 'undefined') {
        alert(imageView.image);
    } else {
        alert('Undefined!!!');
    }
}

When I run this code on iOS Simulator I get alert with URL string.
When I run this code on Android emulator or device I get alert with 'Undefined!!!' string.

In both cases image loaded and shown properly, but I can't get image URL in load handler on Android.

Additional info:

Titanium SDK: 3.2.2.GA

Platform & version: iOS 7.1, Android 4.3

Device: iOS simulator, Android emulator and Samsung Galaxy S3

Host Operating System: OSX 10.9

Titanium Studio: Titanium Studio, build: 3.2.1.201402041146

How I can resolve this issue? Is this bug in Titanium or I missed something?
