What is the best way to use iOS Push service in a chat app?

Hey guys, here is the thing, one of the projects I'm working on it right now is a chat app with PUSH Notifications.

I know how to make an iOS Push notification work, so the first question is not exactly a technical one, but rather a logistic one.

I have a chat app that has several users across several channels, they receive the messages through a live socket connection(PubNub), how is the best way to implement PUSH to work with that in your guys opinions?

I can't detect who is online and who is on background, so I thought in three options.

1 - Always send a PUSH Notification after each message (which I think would make an expensive service even more expensive)

2 - Create an in-App status for Users for each channel(yes, it's a chat with several channels), those who enter background mode, will send a message "telling" of that, same when resumed.

3 - I'm hoping you guys can give me an idea.

The other thing, is which is the best way to use PUSH? Currently I'm doing an in-App HTTP POST to a PHP Script on my backend, which then connects to the APNS for the PUSH, it's working, but is that really the best option for such chat app that will certainly send a lot of PUSH messages?
