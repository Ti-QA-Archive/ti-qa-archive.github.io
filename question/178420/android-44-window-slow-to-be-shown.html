Android 4.4 window slow to be shown

SDK 3.4.0.GA

I've noticed that on Android API19 (4.4.2, 4.4.4) windows are very slow to be shown.
Here how I open it

Alloy.createController('mywin', {}).getView().open();

I put a postLayout listener on the window

<Alloy>
    <Window onClose="cleanup" onPostlayout="postlayout">
        <View id="layer"></View>
    </Window>
</Alloy>

here is the code of postlayout

function loadList() {
    $.mywin.removeEventListener('postlayout', loadList);
    Ti.API.info('PostLayout');
};

All of this code are meant to show you that there is nothing heavy in that…

The app uses https://github.com/FokkeZB/nl.fokkezb.drawer skeleton-like.

What I can see is a delay from 1 to 5 second between the time I read "PostLayout" in the log and the moment I really see the window on the screen. The same app on an Android4.2.2 API 17 displays the windows instantly.
In both environment window is closed instantly (cleanup + disappear)

Is there any thing I forget? Is there any thing I should not do or be aware of? (maybe I should not use windows?)

Thank you for your help
