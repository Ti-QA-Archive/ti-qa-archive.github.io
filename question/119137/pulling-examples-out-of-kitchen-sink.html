    This took me a bit to figure out to so I thought I'd post a more explicit example.
    What I've done is navigated to the KitchenSink\resources\examples\ directory and copied the 'map_view.js' file to my app's resources directory. Then if I want to use that as a starting point for a window within a tab, I'd do something like this:

    //
// Map View
//
var mapWindow = Titanium.UI.createWindow({  
    title:'Map',
    backgroundColor:'#fff',
    url:'map_view.js' //Link to file copied from KitchenSink
});
var mapTab = Titanium.UI.createTab({  
    icon:'KS_nav_views.png',
    title:'Map',
    window:mapWindow
});

    From there you can edit the KitchenSink example file to your heart's content. I've also found it good practice for code organization, so if I want to edit the map window I just need to open 'map_view.js' instead of searching through a long app.js file for the right block of code.

    Just something I thought was helpful, hope you find it the same.

    Kitchensink doesn't have APIs that you "pull into" your app, it just uses the existing API that you can find here:

    http://developer.appcelerator.com/apidoc/mobile/latest

    Kitchensink is just sample code to give you ideas of what Titanium can do. What you want to do is go into Resources/samples and open up the js files to see sample code to do what you want.

    Good luck!
