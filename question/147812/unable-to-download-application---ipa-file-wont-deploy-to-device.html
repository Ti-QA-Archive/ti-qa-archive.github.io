Unable to Download Application - ipa file won't deploy to device

I am on the 3rd iteration of my in-house iPad app. After upgrading to TI 3.x, having not touched the code for awhile since releasing version 2, I can no longer deploy/install my app. I have an Enterprise distribution license (the $299 per year one). For previous versions I always went through hell trying to get the keys, dev profiles, and provisioning profiles all correct and aligned so that I could install my app on physical iPads. I had it all correct 6 months ago and deployed my app out to many devices, but now apparently I'm back to square one. I'm not sure what changed or could have caused this.

I was able to compile my app to an ipa file and I believe I have the manifest all correct, etc. When I try to download my app onto my ipad from a web page (this is allowed in Enterprise apple licensing), it starts downloading and installing. And the little progress bar inside the app icon makes it 95% across, then halts with the error "Unable to Download Appliation - 'Daily Report' could not be installed at this time". My app's name is 'Daily Report'.

When I try to install it through iTunes (from macbook air to ipad) I get a slightly different error message: "iTunes Sync - 'Daily Report' failed to install".

My best guess is that something's tangled with my development profile and/or distribution profile. Yet it all compiles fine and gives me what appears to be a valid ipa file.

Here is my pre-compile window:

http://www.zerogravpro.com/temp/precompile.png

I'm wondering if I was supposed to use my Distribution Profile for Provisioning Profile rather than my Development Profile. Even if that's it, I can't find any way to change my selected profile in that pulldown. When I click on it, I see no way to edit or select something different. And I don't know if that's even it but I can't figure what else it could be. Also, Titanium seems to look in my keychain rather than filesystem for the profile – in my keychain I have both a dev profile and a dist profile.

Please help.

  • I figured it out! In Titanium 3.0 there's a new little icon (looks like a brown box or folder) I didn't notice before for "Distribute - Ad Hoc/Enterprise". I went through the wizard, chose all the defaults, and it published. I was able to download and install the ipa on my first try. Sorry to bother everyone, but maybe this will help someone else.
