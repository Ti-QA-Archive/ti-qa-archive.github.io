Different global variable behavior on Android

At the beginning of app.js I set the variable:

var fps = require(&#39;&#x2F;app&#x2F;FilesPathService&#39;);

This module contains references to all other files, so when I am doing refactoring I can change this all in on place

//FilesPathService.js
exports.MAIN_MENU_WINDOW_VIEW_CONTROLLER = 'app/controllers/MainMenuWindowViewController';
exports.MAIN_MENU_WINDOW_VIEW = 'app/views/MainMenuWindowView';

On iOS I can reference that variable from every file:

var MainMenuWindowView = require(fps.MAIN_MENU_WINDOW_VIEW).MainMenuWindowView;

but on Android it says that fps variable does not exists??

Why is this different on Android?
