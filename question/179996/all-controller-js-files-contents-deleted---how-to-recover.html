All controller .js files contents deleted - how to recover?

I have no idea what happened but I did a build of an Android Alloy app in Titanium Studio that I'm working on for a client that's due tomorrow and after the build all the .js files' contents were deleted so every single .js file in the project under controllers and lib folders are now 0 bytes. On top of that all the .tss files in the styles folder are also 0 bytes. Only the .xml view files are intact. Didn't realize this had happened and was wondering why the app just hung on the splash screen on the Android test device until after I rebuilt the app a few times and restarted my Macbook Pro and then saw that all the files were now empty! I'm using Titanium Studio build: 3.4.1.201410281727 and Titanium SDK 3.4.1.GA and the Chariti framework. I of course had to forget to backup my code and "lucky" me this had to happen. Anyone experience this problem before or know of a way to recover the files? I've tried deleting a file and then recovering it in Titanium Studio but it just recovers the same 0 byte file. Any help would be appreciated!
