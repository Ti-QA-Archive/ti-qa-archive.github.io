iPhone - different orientation for tabs in tabgroup?

Hi

I'm building an iPhone app that should, ideally, have three tabs: two that are Portrait mode only, and one that works in all orientations (it simply displays a photo).

I have set the win.orientationModes of each in app.js, and this works perfectly in the simulator. However, if complied and run on real devices, the result is a mess: sometimes the window that should rotate to landscape will not, and other times the windows that should be locked to Portrait get rotated.

Does anyone know if it's possible to use different orientations on tabs on the iPhone? I'm wondering if I need to go back to the drawing board to avoid this issue.

Alternatively, is it possible to pop open a window 'outside' the tab group to handle the multi-orientation content?

Many thanks

Tom

  • Quick update: I managed to work around this.

    Solution was to open the rotating window as a modal window from app.js. The button that opens the window sets its orientationmodes to all, and the button that closes it flips the modal window to Portrait, closes it, then sets the main window orientation to Portrait only (again).

    With this solution, tapping the 'close' on the modal window in Landscape animates it into Portrait then immediately closes it, leaving the main window in portrait as it should be.
