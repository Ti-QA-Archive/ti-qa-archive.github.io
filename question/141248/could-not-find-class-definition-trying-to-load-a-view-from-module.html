"Could not find class definition" Trying to load a View from Module.

I already done a Custom Control in ObjectiveC (for iOS of course) using UiViewcontrollers and UIViews, so now I want to include it as a module in my Ti app.
I was reading the doc about iOS modules, and see that there is a sample to create a View in a Module from Titanium.
The problem is that when im trying to load the module, it gives me the error 

[WARN] Attempted to load MyModViewProxy: Could not find class definition.

Where my.mod is the identifier of my module.
Am doing exactly the same as in the sample from the documentation, but I cant figure it out why am running into this issue.

Am working with SDK 2.1.1.GA and my module is working with the same version.
I already try cleaning the project, reboot Titanium and reboot my Mac.

Please help me with this!

  • Accepted Answer

    You need to create two classes, a TiUIView subclass (which is the view itself) and a TiViewProxy subclass (which is the controller and the bind to the JS)

    Look at this module I made, it is very simple: https://github.com/pec1985/TiDraggable/tree/master/ios/Classes

    The files must be also named this way: ModuleIdSomeView.h and ModuleIdSomeViewProxy.h (and .m in both)

    When, in JS, you "create" something, the name of the proxy will be appended to that "create". For example in my module: Ti.Draggable.createView() <– it calls the ViewProxy in Ti.Draggable. So, it calls TiDraggableViewProxy and the proxy calls the View. Make sense?

    I hope this helps
