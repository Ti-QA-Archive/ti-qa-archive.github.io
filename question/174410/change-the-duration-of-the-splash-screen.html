  • Hi

    The duration of the splash is controlled by these things because of the time it takes to;

    • place the app infrastructure, any internal logic, Apis and more that the titanium core requires into memory.
    • your code, anything you have said should happen straight away, loading windows, adding images. Calling remote Apis. These things contribute to CPU time, memory allocation of those things network calls if required
    • how fast the device CPU is
    • how much memory is available to the app at that one time, this is determined by factors like; total memory of device, apps currently dp running, background processes
    • interrupt background processes from running to allow your app to start

    There is no simple setting you adjust.

    To shorten the time, load less at the start of your app and only load as and when required.

    Create a brand new default app install it on your device, the time that loads is a baseline (give or take) for your app. If it is similar then you are as fast as you can be, much slower - tweak as I said.
