How to override default location of Alloy database in Android.

Hi

I have just started exploring Appcelerator using the Alloy MVC framework.

By default the a model designed in Alloy will result in a database file being created in the /data/data/com.myDomain.myapp/.. folder.

Is there a way how to override this behavior so that the db file should be created and all operations to take place on the sd card for example?

I know from the the fighters.js sample. But that's only showing how to create a db file again in the data/data/ directory but this time it copies over the supplied db. Once that is done, all operations will be from the /data/data folder and not from the supplied path.

Any help would be greatly appreciated.

Thanks.

  • As it stands, no, there's not a way to do this. The underlying sql adapter makes use of the Ti.Database.install which always installs to internal storage. There are manual ways to install and open from an SD card, but they aren't built into Alloy.

    What you mention though seems a worthwhile improvement to Alloy. Can you log this as a Titanium Community Jira project ticket. Make it a feature request and be sure to label it "alloy".
