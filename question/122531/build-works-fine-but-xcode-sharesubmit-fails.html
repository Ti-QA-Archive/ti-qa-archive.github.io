Build works fine, but xCode share/submit fails

I have got a problem with one of the apps I made with Appcelerator. It does build fine for the simulator, for iOS device and also for distribution, so no problems here. After the build for distribution, xCode opens as expected, I also see the newly created archive, so I click on it, use share or submit and after selecting iOS App Store Package the right identity I get this error message:

"The operation couldn´t be completed. No such file or directory".

Since it works fine with other projects I made in the past, I wonder where I could start looking, but maybe somebody has a hint for me?

thanks!

  • I've had this happen when somehow the mobile provisioning file that is selected no longer exists in the MobileProvision folder. I used XCode to remove the mobile provisioning files and redownloaded them from Apple Dev Center, re-added them to XCode and then re-added them through the TI Studio package app screen.
