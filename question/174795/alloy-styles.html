    1. I think the ordering goes like this from lowest to highest :
    • View-Controller Style File
    • View-Controller Style File in a Theme
    • View-Controller Style File with Platform-Specific Styles
    • View-Controller Style File in a Theme with Platform-Specific Styles
    1. Yes, styles defined in TSS files are merged.
    • index.xml
      <Alloy>
  <Window class="container">
      <Label class="lbl" id="lbl1">Hello</Label>
  </Window>
</Alloy>
    • app/styles/index.tss
      "#lbl1": {
  font: {
      fontWeight: 'bold',
      fontSize: '12dp'
  }
}
    • app/themes/ios/styles/index.tss
      "#lbl1": {
  font: {
      fontFamily: 'Zapfino',
      fontSize: '32dp'
  }
}

    The final merged style for #lbl6 on ios will be

    font: {
    fontFamily: 'Zapfino',
    fontWeight: 'bold',
    fontSize: '32dp'
}

    • A small clarification: merging font properties isn't supported under Alloy 1.3.1 but will be under 1.4. Actually, that change will apply to any styles that accept an object … which off the top of my head I think is just font and gradient properties.

      Prior to 1.4, the last defined font (or gradient) setting would be what is applied. Given Feon's samples above, assuming the theme is enabled, and the priority order, under 1.3.1, the resulting font style would be: 

      font: {
    fontFamily: 'Zapfino',
    fontSize: '32dp'
}

      With Alloy 1.4, you'd get the font settings she described.
