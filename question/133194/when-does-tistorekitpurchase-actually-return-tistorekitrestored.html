When does Ti.Storekit.purchase() actually return Ti.Storekit.RESTORED?

If you have already purchased a non-consumable product and attempt to re-purchase it (maybe because you have not restored your purchases yet), Apple warns you, let's you "re-buy" it, and does not charge for it … BUT it appears that the method Ti.Storekit.purchase() does not return Ti.Storekit.RESTORED in this case, but rather Ti.Storekit.PURCHASED.

It would seem to make more sense that this should result in RESTORED not PURCHASED, thus giving one the ability to accurately utilize the initial purchase event ONCE.

So then when does that RESTORED value ever get returned, if not then? All of the Storekit examples show Ti.Storekit.RESTORED as a possible return "state" value for the method.

Has anyone else encountered this issue?

Is this a bug, and/or are there any workarounds for firing an event ONLY at the initial purchase (and not on ANY type of restore)?

Thanks.

  • Hi, the RESTORED value is returned when receiving restored receipts. See the In App Purchase Programming Guide from Apple for details on the types of items whose purchase can be restored.

    Regarding your peculiar issue, getting a PURCHASED response is right: you are not restoring the purchase, you are going through the same purchase process more than once (even if you don't get charged). You should simply make your app to be ready to receive that event multiple times.
