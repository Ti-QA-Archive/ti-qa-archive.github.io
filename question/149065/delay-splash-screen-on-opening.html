    • Without having all the code, the basic principle is there from my code.

      The setTimeout is only a place-holder though, I would suggest looking for an alternative to this if your need to delay the splash screen appearance is really a wish to have everything finishing loading and appear at once.

      If you are wanting a loadtime based delay rather than a simple time based delay then - consider adding load events to the windows, have them increment a counter that is available in the global scope (through whatever method you are currently using for global variables). Inside EACH of these load events call a function that decides to run ONLY if the counter has hit a fixed number. The value of the number is equal to the load events totals you are checking against.

      For example if you are loading 4 windows have it run only if counter === 4, the code that you run is the delayed open method(s). This way you can delay the appearance of everything until all loaded, leaving the splash screen visible until that happens. You can always add a little extra delay in seconds if you want afterwards.

      This way the delay is based on actual need and processing ability of the device it is on - rather than a guess of the potential delay required.
