Should I convert TableViews to ListViews

We have an existing project containing a number of TableViews. I'd like to convert some (if not all) of them to ListViews due to a number of minor issues with TableViews. However the functionality and layout of the tables cannot change significantly, so I need to know if ListViews can support things in the existing tables, namely:

  • filtering
  • section title text styling
  • rows that contain custom buttons
  • rows that can zebra (alternate background color according adjacent rows)
  • rows that contain variable-length text, such that the row height auto-adjusts to fit the text.

I'm starting to test this myself, but would appreciate feedback. Thanks.

  • I dont think there is a simple answer to this.

    ListView has many performance enhancements over TableViews…. However it is at the cost of functionality.

    IF you doing searches, paging or complex actions then you need to stick with tableViews.

    If your only displaying straight lists of data in one block, then yes you should switch it over to ListViews.

    To answer some of your specific questions.. A ListView is basically a glorified view, so yes you can additional views to it, that includes buttons, specific backgrounds and make the row heights as required.

  • Well, I'm actually trying out ListViews today for the first time. I previously converted my custom TableViews to ScrollViews due to the horrid TableView performance. So far so good - with one major exception: no row auto height on iOS (i.e. no Ti.UI.SIZE), which is a killer. Looks like I'll stick with ScrollViews until this is fixed, despite the memory issues they may cause when getting long.
