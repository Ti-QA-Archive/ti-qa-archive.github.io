    • Android applications run on android devices. However, Google makes an android emulator for both Windows and OSX, and has compilation tools for both Windows and OSX. In contrast, Apple only creates a simulator for OSX and compilation tools OSX, and does not make a Windows variety.

      Since titanium creates a native project which is compiled under the covers by the target platform's native development tools, appcelerator has no toolset to call under Windows. Note this is Apple's toolset restriction and choice; not Appcelerator.

      The only ways around that are:

      • Install OSX in a virtual machine (against Apple's OSX terms)
      • Have the project sent to a machine running OSX, compile it, and return

      The latter is what a couple of toolsets are now trying to implement. One major issue here is proper debugging of the project under iOS.
