Freeze package dependencies for an Alloy project

Is there a way to freeze the dependencies for an Alloy project like you can with Bundler (Ruby) or package.json (Node)? Or can we install more than one version of Alloy and set that version in tiapp.xml or somewhere else?

I have a specific version of Alloy that I need to keep around for a particular project, but don't want to overwrite the global version every time I compile the app.

  • There isn't a way to do this at this time. The best you can do is uninstall the current version of Alloy, install the old version, do your work, then reinstall the current Alloy version when you're done.

    sudo npm r alloy -g   // to uninstall the current Alloy version
sudo npm install -g alloy@1.x.x  // where 1.x.x is the version of Alloy you need

// do your work, build as necessary, etc.

// to go back to using the current Alloy version
sudo npm r alloy -g 
sudo npm cache clean
sudo npm install -g alloy

    You might have other version issues to deal with, like CLI & SDK versions. You may have to uninstall/install the CLI (titanium) versions and update your tiapp.xml in addition to swapping Alloy versions.
