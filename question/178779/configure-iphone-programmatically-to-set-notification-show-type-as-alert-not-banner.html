Configure iPhone programmatically to set notification show type as ALERT not banner

Am using 3.2.2 SDK, Alloy and developing app for iPhone. I need to force app to programmatically
to use Alert as notification show type. As in docs, we can do it for iOS 8 and later, but is it possible to use in lower versions of iOS programmatically without user intervention while installing app or at any point, as am creating Alarm app intentionally I need to show a snooze and dismiss button to user stating alarm is ringing. So far I got this code from docs for setting it programmatically,

if (Ti.Platform.name == "iPhone OS" && parseInt(Ti.Platform.version.split(".")[0]) >= 8) {
    Ti.App.iOS.registerUserNotificationSettings({
        types: [
            Ti.App.iOS.USER_NOTIFICATION_TYPE_ALERT,
            Ti.App.iOS.USER_NOTIFICATION_TYPE_SOUND,
            Ti.App.iOS.USER_NOTIFICATION_TYPE_BADGE
        ]
    });
}

Native Alarm also uses the same mechanism right?
