Determining file name from which a snippet is called

Hi, so I can see that I can retrieve a files name, without path, by making a snippets expansion equal the string "$TM_FILENAME". This is fine, but the result also inserts the file extension. Quite simply, I want to process the file name before inserting the result into the editor. In simple terms, lets say I have a script called "AnObject.js". I want to retrieve the file name ("AnObject.js") and remove the file extension, truncating it to "AnObject". Thing is, while making the expansion equal "$TM_FILENAME" or "${TM_FILENAME}", I cannot access the result before processing the string. What the hell am I doing wrong? How can I retrieve the file name of the script without interpolating the variable in the context of the scripts contents POST insertion? ENV['TM_FILENAME'] ALWAYS returns a Nil class, regular expressions are apparently ignored and result in empty strings, and the documentation is not very clear as to how to go about this. I can see use of ENV in commands, but not snippet. If possible, please explain the situation, and if its of any help, you can refer to and add to my stackoverflow question if that clarifies anything. Thanks a million!
