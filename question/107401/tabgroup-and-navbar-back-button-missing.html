Tabgroup and Navbar: back button missing

Hi,

Maybe i am doing something wrong, but i cannot see why a back button does not appear on the navbar at the top for windows that are part of a tabGroup. It is only windows opened from a table view with a 'haschild' that show the back button.

Can anyone point me to an example as i cannot find anything on the Docs about it.

I would like to post some code, but what? 

var tabGroup = Titanium.UI.createTabGroup({
    barColor:'#000000'
});

Then i createTab paired with createWin and i add all the tabs to the tabGroup.

But i must be missing something.
Any suggestions?

Thanks.

  • Each tab is in essence its own navigation group. The window associated with each is thus analogous to the root window of an application without tabs. As such, there is no back button because you are at the root of that group.

    Take a look at a standard application with tabs, such as the App Store, and you will notice that the root window of each tab does not have a back button either.

    That said, it won't only be windows opened from a table view with a 'hasChild' property that by default show a back button. It should be any non-tab window opened unless you use set navBarHidden to true.

    Or maybe I am just not understanding the question correctly?
