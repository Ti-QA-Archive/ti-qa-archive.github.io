Remove constraints of parent view with animation

Ti 3.1.3 / iphone simulator testing

Is there a way to take a View out of its parents constraints, so it can move/animate independently of its parent object/View.

The below code animates a View to a size bigger than its parent view, but once it reaches the edges of the parent View everything is hidden. The objective in my case is to enlarge the child View and place it at the centre of the Window object, like a pop up image.


var Window = Ti.UI.createWindow({
  backgroundColor: '#000'
});

var Container = Ti.UI.createView({
  backgroundColor: 'green',
  width: '150',
  height: '150'
});

var View = Ti.UI.createView({
  backgroundColor: 'blue',
  width: '100',
  height: '100'
});

Container.add(View);

Window.add(Container);

Window.addEventListener('click', function(ev) {
  var anim = Ti.UI.createAnimation({
    zIndex: 1,
    width: '200',
    height: '200',
    duration: 200,
    autoreverse: true
  });
  return ev.source.animate(anim);
});

Window.open();

    • The only way to not be constrained within a parent in the manner you suggest, is to not be a child inside the parents view hierarchy. You create a window object not attached to the view hierarchy and when you click your small image, show the new window on top of it and perform your animation on that.
