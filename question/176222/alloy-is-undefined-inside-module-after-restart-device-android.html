Alloy is 'undefined' (inside module) after restart device (Android)

My application starts a service Android when device finished the BOOT. But when I start the app by icon,the error 'Alloy is not defined' is showed. If I kill the app and start again, works fine. The error happens in just first start of the app after restart device. The nexts starts works fine.
If I don't start the service in Boot, the errors don't happens.

The referente for Alloy is inside my module js in lib folder. If I put "var Alloy = require('alloy')" inside that js, the error happens inside other module js, and so on.

Can someone help?

Thank you!

Android 4.2
Ti SDK 3.2.0
Device S3

    CommonJS modules that you add to your project (in the lib or vendor folders) do not automatically get a reference to Alloy (and its core libraries, like underscore and backbone). You have to create references yourself if you need them. You'd do this in each module.

    I can't tell from what you've written if this is what you've done, and the failure is happening only on initial launch (or resume from background).
