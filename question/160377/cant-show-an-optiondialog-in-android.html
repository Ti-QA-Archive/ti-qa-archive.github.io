Can't show an OptionDialog in Android

I'm using Titanium SDK 3.1.3 and deploying for both iOS and Android.

In one of my app's windows I show a controller that has a label, when the label is single tapped an OptionDialog should be shown, this works on iOS but on Android I get a java.lang.NullPointerException when trying the same thing.

function checkIfEmailInString(text) { 
    var re = /(([^<>()[\]\\.,;:\s@\"]+(\.[^<>()[\]\\.,;:\s@\"]+)*)|(\".+\"))@((\[[0-9]{1,3}\.[0-9]{1,3}\.[0-9]{1,3}\.[0-9]{1,3}\])|(([a-zA-Z\-0-9]+\.)+[a-zA-Z]{2,}))/;
    return re.test(text);
}

function validateFields()
{
    var okToSend = true;
    if(type === 'email')
    {
        if(!checkIfEmailInString($.data.value))
        {
            okToSend = false;
        }
    }
    else if(type === 'phone')
    {
        if(isNaN($.data.value))
        {
            okToSend = false;

    }
    return okToSend;
}

  • I found the problem:

    I made the mistake in this line 

    options.push(L('cancel_string'));

    instead of pushing 'cancel_string' into the array, I pushed L('cancel_string') in it. This caused that when I pushed the internationalized string into the array stringOptions it was no found and on Android this was replaced with a null value, since no null value can be used on a OptionDialog the whole app crashed.
