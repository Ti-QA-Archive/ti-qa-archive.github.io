Publishing Ad-Hoc/Enterprise app archive sometimes creates a Mac app. Also, missing icons

Sometimes, when packaging an app for archive, for pre-beta/testflight submission, Ti Studio seems to build a Mac app and not an IOS app.

Anyone know why this occurs and how to stop it?

Also, apps have been failing validation for missing icons , even though all the icon sizes requires are present in the /iphone folder.

Has anyone had this issue recently?

Thanks.

  • This happens when you run a "clean" of the project and immediately after this (without run the simulator) you try to create an Ad-Hoc/Enterprise app archive. In this case, it doesn't select the type of application therefore is created an archive not IOS.
