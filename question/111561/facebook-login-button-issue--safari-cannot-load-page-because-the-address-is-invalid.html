Facebook Login Button Issue- Safari cannot load page because the address is invalid.

In the iPhone simulator, using Ti.Facebook.createLoginButton, when Safari tries to load this URL: "http://www.facebook.com/connect/uiserver.php"– it gives me this alert box: "Cannot Open Page - Safari cannot load page because the address is invalid."

My code is pretty similar to what's going on in the facebook_login_logout.js file in the KitchenSink:

Titanium.Facebook.appid = "127651321xxxxx"; 
Titanium.Facebook.permissions = ['publish_stream', 'read_stream'];

fbLoginButton = Titanium.Facebook.createLoginButton({
    style: 'wide',
    bottom: 60
});
MainView.add(fbLoginButton);

I've tried building with 1.6.0RC1 and 1.5.1 with dramatically different results. In 1.5.1, clicking the Login button pops up a modal window from within the app where you can enter your login credentials and allow permissions. In 1.6.0RC1, clicking the same button pushes your app to the background and opens up the browser and ends up with the error I described above.

    I am getting a same error message.
    So,I add code below to "tiapp.xml".

    <property name="ti.facebook.appid">YOUR_FACEBOOK_APP_ID</property>

    and performing full rebuild.

  • In case this helps anyone, I had this issue because I my app needed custom fonts which required a custom Info.plist in my root folder to overwrite the plist file that Titanium writes. So I had simply forgot to add the FB id to the array in the Info.plist file.

        <array>
        <dict>
            <key>CFBundleURLName</key>
            <string>com.mycompany.myapp</string>
            <key>CFBundleURLSchemes</key>
            <array>
                <string>myapp</string>
                <string>fbYOURFBIDGOESHERE</string>
            </array>
        </dict>
    </array>
