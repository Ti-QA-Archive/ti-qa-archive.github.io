    Hey guys, since I built the Q&A, I'm probably the best one to respond.

    1) Yes, the search sucks. It sorta works. Since I originally added Apache Solr for full text search. I added a boost value for dates, but it doesn't work right. We had another guy look at it, but he wasn't able to improve it. Before Solr, we used Google, but despite paying Google to use their search API, we hit the maximum amount of queries per day by around 3pm. If we spent more time trying to fix Solr, I have to believe we could improve search and even add additional filter criteria.

    2) I think you used to be able to edit questions and answers, but when the Q&A was revamp in early Summer 2011, that was never added back in. We never had the ability to edit comments. I had every intention of adding back in editing capabilities, but higher priority projects kept coming up.

    3) I'm unaware of any bugs adding images. It's probably the shitty PHP markdown library we're using. Sorry.

    4) Yes, there's a ton of other things that need love. Navigation is jacked. Actual timestamps of questions/answers/comments would be nice. The format of emails is horrible. The voting system is borderline pointless. The list goes on and on.

    The Q&A hasn't received hardly any updates since it launched. I don't actively support it since I moved from the web dev team to the Titanium platform team. FWIW, we did in the past couple weeks address 2 performance issues that should noticeably speed up Q&A.

    There were talks about ditching Q&A and going with another solution. As far as I know, these talks have stalled. I'm pretty sure we'd lose all the points, top experts, and DevLink. I don't think that's an acceptable loss.

    So, we hear you guys loud and clear. I am personally taking the initiative to maintain and improve the dev center. Thanks for your patience. If you have more comments/complaints, fire away, we're listening.
