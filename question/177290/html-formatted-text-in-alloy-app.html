Html-formatted text in Alloy app

My application loads data from server - terms of use for example. This data will have html formatting. What I mean by that is that users will type text in cms, in tinymce textarea, so it will have html tags. Can I show it to my users so that if something will be &lt;b&gt;bold&lt;&#x2F;b&gt; (somehow parse html tags in my alloy application)?

    • Yes, it essentially loads a "chrome-less" browser within your app (meaning no buttons, address bar, etc.), and it is a heavyweight control. You should use it sparingly. However, there are few other options for displaying formatted HTML content.

      You didn't say which platform you're building for. Labels on Android have an html property which will show rendered HTML content. There are add-on modules for iOS in our Marketplace to add similar controls in iOS (though some I've looked at are in essence just webviews repackaged as if they were labels). You could do manual string parsing to strip out the HTML then display a large series of labels, each formatted per the original HTML, positioned dynamically next to each other, etc.. That would get extremely messy very quickly.
