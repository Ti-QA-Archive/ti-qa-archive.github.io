Android - Handle touches later

My app crashes when it receives a touch input while it is still loading some data.
Is there a way, while loading, to handle touches later ? or at least to disable touches while loading?
Thanks

  • Easiest solution: set touchEnabled to false while loading.

    Another soulution: cover your window with another transparent window wich is showing an ActivityIndicator. This is better because it shows that something is going on. When you block touch events, it seems that your app is not responding to user interactions.

    You could store touch events to handle them later but I'm not sure if this is the best idea because you may want to handle only one event.
