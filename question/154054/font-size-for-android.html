  • Hello Renato,

    I'll give it another try because my first answer wasn't very well explained but I'm still unhappy with the currently accepted answer.

    How to set fontSize in Android

    1. Screen differences

    At first, let's have a look on what screens we have to support. You can not just talk about phone and tablets or small devices an big devices, because the range is huge. Android distinguishs the actual screen size into four generalized sizes ( small / normal / large / extra-large ) and also four generalized densities for the actual screen density ( ldpi / mdpi / hdpi / xhdpi).

    That means, we theoretically have to support 16 categories of screens. As you know, nobody provides 16 layouts (or in this case 16 different fontSizes). Therefore Android provides a technique that lets us handle all those screens.

    2. Density independance

    To support all 16 screen sizes / densities with a single value, Android is abstracting the pixel to a "dp"-unit (density-indepenent pixel). This lets you define dimensions for your views easily, and Android does calculate the actual pixel values as appropriate for the current screen density. This way a button with a dimension of 120dp x 48dp has the same size on every screen.

    3. Bad examples

    fontSize: 16 // or 16px

    Defining the fonts size like this causes the font to be large on low density displays while it is very small on higher densities. Just as you described in your question.

    fontSize: "10%"

    This is even worse, although it seems to scale acceptable. It doesn't. There are several problems with that solution. First of all, relative measurements depend on the size of the parent view. Having a label with fontSize of 10% that looks okay in a full screen window, would be tiny in a nested view with a small size. Other problems are that this is not density independent too and you force the system to do unnecessary calculations.

    fontSize: (Titanium.Platform.displayCaps.platformHeight * 3) / 100;

    This seems to calculate a suitable value for the fontSize. Let's see what happens. Titanium.Platform.displayCaps.platformHeight is the dynamic factor. It returns the height of the display as an absolute pixel value. If you compare what this would be on a WQVGA Screen with a resolution of 240x400 to a xhdpi screen width 640x960 resolution. You get 12px fontSize on the first screen and 29px on the second screen. But both displays have the same physical size!

    fontSize: "16dp"

    This solution is actually not bad. Quite the reverse! You can go with that one.

    4. the right way

    sp - "Scale-independent Pixels - This is like the dp unit, but it is also scaled by the user's font size preference. It is recommend you use this unit when specifying font sizes, so they will be adjusted for both the screen density and the user's preference." (source: developer.android.com)

    Nothing more to say, apart from using sp is not only the recommended way of fontSize measurements, it also gives your users a better user experience. If someone increases the fontSize in the OS settings, then the OS will consider your sp defined fonts. This is also more barrier-free.

    conclusion

    set your fontSizes like this:

    fontSize: "16sp"

    I hope this helps somebody. thanks

  • You have to keep in mind the Ti.Platform.displayCaps.logicalDensityFactor on Android.

    I've used "dp" for fontSize and on devices which have set this to "2" (e.g. Samsung Galaxy S4), the fonts would be too small. So I've switched to "sp" and everything is fine on iOS, Android and all devices.

    And I realized, that, as soon as you use the font object on e.g. a label, you have to define a fontSize, otherwise the font would be too small.

    I've tested this on Titanium SDK 3.2.0.

    Examples:

    var Label = Ti.UI.createLabel({
    text : 'Jipiiiiee '
});
// -> the font size is good

    var Label = Ti.UI.createLabel({
    text : 'Jipiiiiee ',
    font: { fontWeight: bold }
});
// -> on logicalDensityFactor > 1 the font is too small (we use the font object with no fontSize definition

    var Label = Ti.UI.createLabel({
    text : 'Jipiiiiee ',
    font: { fontWeight: bold, fontSize: '12dp' }
});
// -> on logicalDensityFactor > 1 the font is still too small (because we're using "dp")

    var Label = Ti.UI.createLabel({
    text : 'Jipiiiiee ',
    font: { fontWeight: bold, fontSize: '12sp' }
});
// -> that's how to do it (using "sp" instead of "dp")! This works on any device!

    // Define this in app.js (in no object context)
var fontSizeDefault = '12sp';

var Label = Ti.UI.createLabel({
    text : 'Jipiiiiee ',
    font: { fontWeight: bold, fontSize: fontSizeDefault }
});
// -> that's how I do it! So I'm able to vary font sizes globally just by changing one variable. Very comfortable though...

    This also affects Ti.Platform.displayCaps.platformHeight and Ti.Platform.displayCaps.platformWidth - I've used that for example to position some elements.
    So let's say you would like to position an element on half of the screen height, you normally would use Ti.Platform.displayCaps.platformHeight/2.

    On Android with logicalDensityFactor of 2 Ti.Platform.displayCaps.platformHeight will be twice the height.
    So you always have to divide the result through the density factor. Here's some code:

    var labelDistance = Ti.Platform.displayCaps.platformHeight*0.5;    // position the label on half of the screen

// if the logicalDensityFactor is set, use it for calculation of the screen height 
if(Ti.Platform.osname == "android" && Ti.Platform.displayCaps.logicalDensityFactor != undefined && Ti.Platform.displayCaps.logicalDensityFactor > 1)
{
    labelDistance = labelDistance / Ti.Platform.displayCaps.logicalDensityFactor;
}

    -> This works on iOS and on Android…

  • Hi,

    To tackle with font size on every resolution and if you don't want to give user a chance increase the font size. I have done the following : The code is self explainatery

    
DisplayMetrics dm = new DisplayMetrics();
        try {
            getActivity().getWindowManager().getDefaultDisplay().getMetrics(dm);
        } catch (Exception ex) {
        }

        final int heightOfScreen = dm.heightPixels;
        final int widthOfScreen = dm.widthPixels;

        int baseHeight = getResources().getInteger(R.integer.base_height);
        int baseWidth = getResources().getInteger(R.integer.base_width);



final float textSize = Float.valueOf(getResources()
                .getInteger(R.integer.MEDorSF_TextSize))
                * widthOfScreen / baseWidth;

TextView medicineReminderDiseaseTextBox = (TextView) currentFragment
                .findViewById(R.id.ID_ReminderPage_MedicineReminder_Disease_TextBox);
        medicineReminderDiseaseTextBox.setTextSize(TypedValue.COMPLEX_UNIT_PX,
                textSize);

    This solved my problem. And when I saw the result on various screen, they were identical words by words and line by line.
