Background updating

My application needs to load data from a web service, but once every hour that data changes, what would be the best way to preform a scheduled update of that data?

Can I just do it with a timeout? Or is there another way?

— asked August 8th 2011 by Wouter De Loose

  • Accepted Answer

    Apple does not allow background updating of data like that… the best way to do that is to have a false timer also (setInterval for when the app is open, and the false timer when its closed) the "false timer" you just need to set a variable for when the next update should be (time/data) and check if its after that on app resume.

  • Perhaps the best way is to display a message a message whenever there is an update, not download it immediately. Or you could also just check it every time you run the app.

    And it's better with an interval instead of a timeout. An interval will perform a function ever x-minutes/hours (the one you give).

    How I do it is I set an interval every x-minutes and just check for updates, then I show a button that they an click to download the updates, I don't download it automatically, but let the user decide when.

    Hope this helps!
