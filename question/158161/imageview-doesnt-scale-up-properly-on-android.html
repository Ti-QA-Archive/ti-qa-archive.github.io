Imageview doesn't scale UP properly on Android

I'm trying to create an ImageView where the image will be scaled (up or down) to fill the device window width while maintaining it's aspect ratio. My understanding from reading the documentation is that if I specify a width and leave the height blank (or set it to 'auto'), the image will be scaled to fit the width provided. But that is not what I am seeing in the case where the image needs to be scaled up.

I'm using the 3.1.3 Titanium SDK.

For the example below I'm using an image with dimensions 640x360 on a device (galaxy s4) with dimensions of 1080x1920 (portrait mode).

If i create a window and put the following ImageView in it:

    var sizeImage = Ti.UI.createImageView({
        top: 20,
        right: 0,
        left: 0,
        image: 'http://placehold.it/640x360',
        backgroundColor: '#ff0000'   //Add a red background to show the imageview dimensions
    });

    //"self" is a reference to the window.    
    self.add(sizeImage);

I would expect my image to fill the width of the device window, and it's height be adjusted automatically to retain it's aspect ratio. However, what actually happens is that the ImageView's height gets limited to the height of the actual image (360px) and the width is limited to maintain the aspect ratio, while the remaining area to the left and right of the image is filled with the red background.

I get the same results if I explicitly set the ImageView's height to 'auto', or if I specify width: '100%' instead of left and right, or if I specify width: Ti.Platform.displayCaps.deviceWidth. Also, using a local image instead of remote doesn't change anything.

The image is scaled correctly on iOS (6 and 7). Is this a Titanium bug that affects android only? Am I doing something wrong?
