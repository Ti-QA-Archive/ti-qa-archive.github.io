Build Error Packaging for App Store

I've been testing my application in the emulator and on local physical devices and everything checks out perfectly. I'm now ready to package this for App Store Review, but i'm getting an error during packaging that isn't present at any other time. Unfortunately the new versions of everything came out the day before i was ready to go so i'm wondering if there is an issue in there as a result of that?

the error i get in the console is:

[ERROR] : BUILD FAILED
[ERROR] : The following build commands failed:
[ERROR] : CompileC build/ProjectName.build/Release-iphoneos/ProjectName.build/Objects-normal/armv7/main.o main.m normal armv7 objective-c com.apple.compilers.llvm.clang.1_0.compiler
[ERROR] : (1 failure)

The error makes no obvious sense to me, and as i mentioned, only happens at package time. app works perfectly fine on devices and emulators.

I've upgraded everything to the latest versions. I'm still using SDK 3.4.0 instead of 3.5.1 because the facebook integration has changed drastically, and i don't want to re-write the code now that it's passed user testing.

Additional information:
Mac OSX 10.10.3
App type: mobile
Titanium Command-Line Interface, CLI version 4.0.0, Titanium SDK version 3.4.0.GA
TitaniumStudio 3.4.1

  • Hi Jason, I believe this is because all new submissions to Apple require 64bit support. That being said, the minimum Ti SDK you can use is 3.5.0 GA.
