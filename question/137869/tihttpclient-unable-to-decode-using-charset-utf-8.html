TiHTTPClient Unable to decode using charset: UTF-8

Am trying to connect my app to a RESTful service.
I create two JSON responses with PHP. The first one is something like this:

  {
        "respWS": {
        "estatus": "OK",
        "respuesta": {
            "mensaje": "",
            "E_RUT": "09204804",
            "E_TVGRT_BEZEI": "Pepe Henriquez",
            "E_COND_PAGO": "Z030",
            "E_COND_PAGO_TEX": "30 días desde Fecha de Factura",
            "E_BEZEI_OF": "Of. Iquique",
            "E_SOC": "3003",
            "ORG_VENTA": "Z120"
        }
    }
}

And the second json file is the almost the same, but when I save the file I did change the name of it. And it look like this:

{ 
    "respWS": { 
    "estatus": "ERROR", 
    "respuesta": {
         "mensaje": "No existe Nro Pedido"
     }
     }
 }

When I execute my code, I have no problem with the second JSON file, but with the first JSON I get an error like this: 

[ERROR][TiHttpClient(  446)] (KrollRuntimeThread) [6,12666] Unable to decode using charset: UTF-8

My code for calling this service is this:

var client;
 functionexecuteCall(){

     client = Ti.Network.createHTTPClient({
         onload : function(e) {
            Ti.API.info("responseText: " + this.responseText);
            alert('success' + this.responseText);
         },
         onerror : function(e) {
             Ti.API.debug(e.error);
             alert('error');
         },
         timeout : 5000
     });
 client.open("GET","http://myAddress/respRest_OK.php");
 client.send();    
 };

I already try using "encodeURI" and setting the header with something like:

 client.setRequestHeader('Content-Type', 'application/json', 'charset=ansii');

I've been looking into the DOC, but with no success.
Any suggestion will be really appreciate.
BTW am using the Android emulator.

Thanks!

  • After doing some testing, I finally get it.
    Don't know why, but HTTPClient was giving me error for the UTF-8 encoding.

    I had change the encoding to ISO-8859-1 and everything is going right now.
