Views - png with "backgroundColor"

Hi, I'm new to Appcelerator and the whole app development world, perhaps this problem/question has been answered before but i've tryed google and searching the Q/A. I come from the web development world and considering was said in the guide a "view is like a div".

I'm using, Titanium Studio for Google android 2.3 mobile project

When applying a png background it shows up fine until i try an apply a underlying background color.

window({
        id:'AppWindow',
        backgroundColor:'#FFFFFF',
        layout: "vertical",
        fullscreen:true,
        statusBarHidden:true,
           navBarHidden:true,
            tabBarHidden:true, 

        orientationModes : [
        Titanium.UI.PORTRAIT,
        Titanium.UI.UPSIDE_PORTRAIT
        ]
});


view({
        width: ui.platformWidth,
        height: 596 / 2,
        backgroundColor:'#0278d0', //replaced by png...
        backgroundImage:"/images/whatsnew2_bg.png"
});

In this case my "View" backgroundColor (#0278d0) doesn't show, this would be typical with a web div. Instead it is completely replaced by the png, not stacking on top and I see the white window through it.

Am i doing something wrong? How is this typically handled?

  • So, if you try this in iOS or Mobile Web, you'll see what you expected–the image blends with the background color. Why doesn't it work on Android? The secret is that Android tries to optimize its drawing by assuming pixels are opaque.

    There is an Android-only window property called windowPixelFormat that can be used to force it to do blending. For example, if you create your window like this:

    var pixelFormat = "";
if (Ti.Platform.name === "android") {
    pixelFormat = Ti.UI.Android.PIXEL_FORMAT_TRANSLUCENT;
}
var win = Ti.UI.createWindow({
    title : "Transparency Test",
    fullscreen : false,
    backgroundColor : bgColor,
    windowPixelFormat: pixelFormat
});

    You'll find the images blend like you expect them to. There are a couple of other wrinkles:

    • Modal windows are always translucent on Android. So setting modal: true in your window creation args will also cause the colors to blend. Although it will also make it a modal window, which may be undesirable.
    • Image views always blend their images with the background color.
    • This issue doesn't seem to occur at all on the lastest CI builds (early builds for 2.1). I'm not sure if that's a bug or a feature, however.

    This should be made clear in the docs for backgroundColor and backgroundImage. I've logged a bug to make it so:

    https://jira.appcelerator.org/browse/TIDOC-648
