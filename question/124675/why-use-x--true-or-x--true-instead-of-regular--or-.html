Why use: X === True or X !== True, instead of regular == or !=

Title is my question, just wondering about a while back before summer.

Now i got back into tita using the TiDev community and what i see is that the code editor there gives an explanation mark if i use the == operator and should use === instead

But this is not always the case as with the following line:

// This must be ===
if(e.index === 0){ recentWebView.html = recentUpdatesHtml; }
// This can be == and doesnt give me the explanation mark for an extra =
else if(e.index == 1){ recentWebView.html = recentSocialHtml; }

So could someone explain me what is the difference and why i should use === instead of == , ?

    Hi Kami.

    First comparison is strict (===) and second one is type-converting comparison. The difference is that the first one checks if operands are of the same type, beside their value, and the second one does type converting if operands are not of the same type and then compare their value. The algorithm how are they compared and type-converted is described in ECMA-specification (JavaScript is implementation of that standard).

    Now … you can use both operators and they are valid to use in any case. Sometimes it's better to use "==", sometimes it's better to use "===". Algorithmically, they are very similar and performance difference between them is negligible.

  • Difference between == and === in JavaScript.

    Basically === is more strict than ==.

    If you were to use if(e.index == 0), then in the case that e.index actually has the value &#39;&#39; or false, that comparison would evaluate to true. using === ensures that it would only be true of e.index was 0 specifically.

    This is used with 0 in particular because it's the only number that evaluates to false in boolean operations in javascript.
