Ti.Android.getCurrentActivity() throws runtime error

Hi all,
I'm creating a simple app for a test case. When I try to assign the activity like this:

if (Ti.Platform.osname === 'android') {
    var activity = Ti.Android.getCurrentActivity();    
    activity.addEventListener('resume', function(e) {
        alert('hello');
    });
}

I get a runtime error as shown in this screenshot here. Anyone seen this happen before? If I comment out the code below, the app runs as expected.

    • My problem was not related to the listener. But, for those looking for resolution on the listener issue described, here's what worked:

      $.window.addEventListener('open', function(e) {
    if (Ti.Platform.osname === 'android') {
        var activity = Titanium.Android.currentActivity;
        activity.addEventListener('resume', function(e) {
            alert('hello');
        });
    }
});

      Remember, this code uses the new Alloy framework. So, if your app does not, then substitute $.window for your window name.
