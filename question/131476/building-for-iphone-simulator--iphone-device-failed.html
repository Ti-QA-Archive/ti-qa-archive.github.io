  • ###Fixed the problem by myself

    After reinstalling Titanium & xCode I noticed the problem had to be on the OS itself.
    The solution was to set a simlink for the Python.app in the MacPython-folder to a working version of Python. Somehow my MacPython installation wasn't properly set. But a clean installation is mandantory because xCode uses MacPython in order to use the iOS-Simulator and to perform the distributon.

    After this quick fix the simulator and the distribution to my iPad worked, as well as the debugging.

    Solution:

    If you are getting this error check the installation of MacPyton in the following location:

    /System/Library/Frameworks/Python.framework/Versions/2.6/Resources

    If there is no folder named Python.app you have to set a simlink to an working Python installation.
    Make sure the following line is executed in the location mentioned above folder:

    sudo ln -s /Library/Frameworks/Python.framework/Versions/2.6/Resources/Python.app /Python.app

    ###Only do this quick fix if your MacPython is not installed properly
