WebView behind NavBar in modal window when rotating

I asked this yesterday but since Q&A type "forums" tend to lose your questions very fast, I need to ask again since my question is at the bottom of the page now. I really dislike this style forum. Why can't we have a traditional forum??

Anyway….I have a web view in a modal window. Its width and height are 100%. First opening the webview is sized correctly. Its height is 100% and it starts right below the navbar. When I rotate the device, the webview then gets pushed up 30 pixels or so (however tall the navbar is) and the web view is now behind the navbar. This seems like a bug to me. I've tried quite a few different things and I cannot get it to stay positioned right below the navbar after rotation.

  • Hack/work around….

    Ti.Gesture.addEventListener('orientationchange',function(e)
{
    Ti.API.error("orientation: " + e.orientation);

    switch (e.orientation)
    {
        case 1:
        case 2:
            //portrait
            myWebView.top = 44;
            break;
        case 4:
        case 3:
            //landscape
            myWebView.top = 32;
            break;
    }
});
