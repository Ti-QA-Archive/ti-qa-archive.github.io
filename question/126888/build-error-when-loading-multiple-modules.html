  • I think I know what the problem is. There is a problem with using the iOS version of styledlabel that someone in appcelerator should fix. There is a duplicate symbol problem when you use styledlabel because the styledlabel module uses an objC category for NSData to add Base64 encoding. This makes the linker not compile when you use it with another module that uses the same Base64 Code for NSData. It's causing problems with my ParseAPI module I wrote because they too are using the same names for their Base64 Code and it's causing the linker to get confused.

    Someone at Appcelerator, can you please namespace your NSData category from (Base64) to something else and also namespace your NewBase64Encode and NewBase64Decode methods to something else.

    I have more information on this if you need it too.

    Right now the only alternative is to not use styledlabel for iOS until either flurry or styledlabel gets fixed.
