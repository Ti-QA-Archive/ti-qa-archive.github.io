Displaying a popover seems to mess up the navigation group stack

Here's the basic layout.

  1. I have a window (mainWindow) which contains a navigation group (navGroup).
  2. The navGroup is bound to another window called homeWindowP.
  3. homeWindowP has a ButtonBar set as the rightNavButton
  4. Clicking a ButtonBar button displays a popover containing a table.
  5. Clicking a row in the table does navGroup.open() to open another window.

This works perfectly the first time. The new window opens and a Back button is added to the navGroup.

The issue is when navGroup.open() is called the second time. Instead of getting the new window with the back button, I get a third window with a back button labeled (for example) Foo.

Clicking Foo takes me to the original new window with a Back button.

Doing the navGroup.open() multiple times adds a new window to the stack each time.

Here's what the stack looks like when the app starts

  • mainWindow
  • homeWindowP

After the popover fires navGroup.open() the first time:

  • mainWindow
  • homeWindowP
  • newWindow

After the popover fires navGroup.open() the second time:

  • mainWindow
  • homeWindowP
  • newWindow
  • newWindow

This only appears to happen if I hide the popover before opening the new window. If I leave the popover open the entire time, the navGroup behaves like it's supposed to.

Does anyone have any thoughts about why this is happening and how to fix it?
