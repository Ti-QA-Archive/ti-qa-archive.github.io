  • I traced this problem to a function where I create table rows after querying data from my local database. If I dynamically change the row's make-up (labels, images, etc) based on if there is data to populate a label, it will crash when I open my tabgroup and then return to the table in cases were an element is removed because there is no data. If I don't dynamically change the row and just put in 'No Data' so that all the rows in the table are uniform, it doesn't crash. I've removed all the dynamic row creation code and it appears fine now.

    I'm not sure what is happening when I show the window with the table again. Perhaps there is an event someone is aware of. Maybe a focus. The odd thing is that it works fine the first time and shows the data in the table with the dynamic rows as expected. The second time I just show the window with the table on it, it crashes. I do not close this window, it is always in the background waiting to be shown again.
