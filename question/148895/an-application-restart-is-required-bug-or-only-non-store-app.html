"an application restart is required" bug or only non-store app?

I'm using SDK 3.0.0(Android) and if i install the .apk which i downloaded from my own server(So not in the store yet) it says it needs to restart. Is this a bug, or will this message dissolve once it is in the store?

I've googled the message and got messages of bugreports from more than a year ago, so i'm assuming it's fixed. But what's the fix?

  • hi,

    you can fix this by using SDK 2.1.3 / 2.1.4. to solve this put the below line in tiapp.xml file then clean build project and test again.

    <property name="ti.android.bug2373.finishfalseroot" type="bool">true</property>

    but in SDK 3.0.0 , the problem is still exists. :(

  • I fixed it in Titanium SDK 3.0.0 the same way as one should in 2.1.3/2.1.4:

    <property name="ti.android.bug2373.disableDetection" type="bool">true</property>
    <property name="ti.android.bug2373.restartDelay" type="int">500</property>
    <property name="ti.android.bug2373.finishDelay" type="int">0</property>
    <property name="ti.android.bug2373.skipAlert" type="bool">true</property>
    <property name="ti.android.bug2373.message"/>Initializing</property>
    <property name="ti.android.bug2373.title">Restart Required</property>
    <property name="ti.android.bug2373.buttonText">Continue</property>
    <property name="ti.android.bug2373.finishfalseroot" type="bool">true</property>
