Top attribute for a label does not seem to work for IOS simulators

Hello,

When i add a label to a window with a specified top, it will not appear correctly on the window. This only counts for iPhone and iPad simulator.
On Android the label appears perfectly at the specified top.

here is my code: 

var numberOfScreensLabel = Ti.UI.createLabel({
    text: "Screen " + screenNumber + "/" + totalNumberOfScreens,
    top: 10,
    right: 20,
    color: "black"
});

Titanium.UI.currentWindow.add(numberOfScreensLabel);

The label appears in the middle of the screen, that's weird because i specified a top of 10.
So my question is: What am i doing wrong, and what can i do to place this label correctly on Android and iPhone/iPad.

I've already tried to try it with a top of -100, this works for iphone and ipad, but for Android it places the label outside the screen, like you'd expect.

    Try to set width&#x2F;height for label

    var numberOfScreensLabel = Ti.UI.createLabel({
    text: "Screen " + screenNumber + "/" + totalNumberOfScreens,
    top: 10,
    right: 20,
    height: 40,
    width: 100,
    color: "black"
});

  • Hi Elmar

    Basically the window will place all items in the center. So what you can do is Add one view inside the window and then add the label as per your need in that view. You can set the view size as the window height and width or you can keep that height and width to 100%.

    So instead you add the label in the window add one view in the window and add all the controls in that view as per requirement.
