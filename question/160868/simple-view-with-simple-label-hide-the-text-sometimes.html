</View>

  • Hello Ricardo,
    I did test your issue with Mobile SDK 3.2.0, the issue is that the L('date') is returning an empty string in Android. Check this out.

    Testcase

    index.js

    $.mod_title_date.setText("date");
$.mod_value_date.setText("test");
$.index.open();

    index.tss 

    ".module_row": {
    width:Ti.UI.FILL,
    height:"51dp"
},

".module_title": {
    left:"10dp",
    color:"#585858",
    width:Ti.UI.SIZE,
    font: {
        fontSize:"16dp"
    }
},

".module_value": {
    right:"10dp",
    color:"#585858",
    width:Ti.UI.SIZE,
    font: {
        fontSize:"16dp"
    }
}

    index.xml

    
<Alloy>
    <Window>
<View class="module_row">
    <Label class="module_title" id="mod_title_date"/>
    <Label class="module_value" id="mod_value_date"/> 
</View>
</Window>
</Alloy>

    Running

    When you run that testcase, it will show properly the string "date" in the android version.

    Solution

    Replace L('date') for 'date', so the label will be filled with a text. For constructing the i18n for your app, check this resource.
