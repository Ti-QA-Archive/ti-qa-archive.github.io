  • This constant has been deprecated in iOS 6. Group style table views still default to using the standard iOS group table view background texture if no background color is specified. However, specifying this constant explicitly on iOS 6 and above results in a black background color.

    Please see TIMOB-11030 for more information.

  • I have a brand new Macbook Air with OSX 10.8, Titanium 2.1.2 GA.

    I solved the problem for myself by downgrading to Xcode 4.4. Hopefully you can use 4.4 with your project.

    What I did:

    1. Download and install Xcode 4.4
    2. Open Xcode 4.4 (make sure you have 4.4 open and not 4.5 by going to Help, Release Notes)
    3. In Xcode 4.4 change "Command Line Tools" to Xcode 4.4.1.
    4. Go to Titanium Studio, Preferences, Titanium Studio,Titanium, iOS, iOS SDK Home and click refresh. You should see the path change to the 4.4 directory. Click save and reboot your workstation.
    5. When you are going to deploy to your device, make sure you choose iOS SDK 5.1.

    Note: For Step 5, this is one area where I downloaded 5.1 into Xcode 4.5 BEFORE I figured out this fix. So if you can't change from iOS 6.0 to SDK 5.1, go download 5.1 to Xcode 4.5 then try this part again)

    You should now be permanently pointed to 4.4.

    I hope this helps you and anyone else having this issue.
