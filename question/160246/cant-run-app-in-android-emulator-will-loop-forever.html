    Android (both emulator and devices) produce a lot of 'noise' in the logs. When it comes to android, I tend to use the android tools. monitor can be found in the tools directory of your android sdk. You can filter out the 'system' noise and just see what your app is having problems with.

    I uploaded a demo here:

    http://www.screencast.com/users/Stephen_Feather/folders/Jing/media/ff4a2b95-5397-4405-86d7-5383505d6ebc

You could try to launch the appropriate emulator image manually using the android utility. Then you know that its actually up and running before the script tries to push the binary to it. When in the android utility, I'd check to be sure that the image you are using actually has an sd card img associated with it.
