Issues with app imported from Developer to Studio

I'm noticing the app I imported from developer to studio is missing some features that new apps (created in studio) have. Most important a lot of the "run" features don't work and I need to create custom run configurations. The run button on top of the application tab is disabled as well. I'm also noticing lack of support for iPad even if I select it in tiapp.xml.

Can someone comment if this is a known limitation or if I need to do something to enable these features?

Thanks.

    Typically, the missing "run" features you describe mean the build directories aren't there. Check the build directory for the iphone, android, etc. directories and add them if they're not present. Check that the tiapp.xml file has all the required fields filled in. If none of that works, I'd create a new project in Studio then copy the Resources and i18n directories from your old project into the new one.

    • Frank, check the XML of the tiapp.xml file. Do you have an ipad tag like this?

      <deployment-targets>
  <target device="mobileweb">false</target>
  <target device="iphone">true</target>
  <target device="ipad">true</target>
  <target device="android">true</target>
  <target device="blackberry">false</target>
</deployment-targets>

      Also, an orientations section like this?

      <orientations device="ipad">
  <orientation>Ti.UI.PORTRAIT</orientation>
  <orientation>Ti.UI.UPSIDE_PORTRAIT</orientation>
  <orientation>Ti.UI.LANDSCAPE_LEFT</orientation>
  <orientation>Ti.UI.LANDSCAPE_RIGHT</orientation>
</orientations>
