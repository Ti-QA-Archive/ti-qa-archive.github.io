    Hi

    Both Android and iOS handle fonts very easily (compared to earlier versions) however they are referenced differently - that difference is important here.

    • iOS uses the Postscript family name
    • Android use the font name without the extension

    You no longer need to have any settings in the plist, tiapp.xml or anywhere else for this to work.

    You have not said if you are using Titanium with or without the Alloy Framework. Given you code style, I am guessing you are not using the Alloy Framework. However I have given the details for both platforms.

    Place you fonts inside these folders

    • Titanium with the Alloy Framework (duplicate the fonts into both)
    {appname}/app/assets/android/fonts/
{appname}/app/assets/iphone/fonts/
    • Titanium without the Alloy Framework
    {appname}/resources/fonts/

    How to reference the font family entry in your code, we will assume an example font Titillium-Thin-Regular.otf.

    • iOS (uses the postscript name)
    ".something": {
    font: {
        font-family: 'Titillium-Thin'
    }
}
    • Android (uses the file name without the extension)
    ".something": {
    font: {
        font-family: 'Titillium-Thin-Regular'
    }
}

    Now this can be REALLY annoying - as you have to declare things two different ways. So here is a cheat;

    • Rename Titillium-Thin-Regular.otf to Titillium-Thin
    • Done

    Now the file name matches the postscript name - so the font-family is the same for both Titillium-Thin, so;

    ".something": {
    font: {
        font-family: 'Titillium-Thin'
    }
}

    So - in short the same as the iOS format as the two now match.

    When testing make sure you clean the project between builds as fonts may not be available until you have performed a full re-build.

    • As a follow up based on the wording of your actual question I felt it worth mentioning there are places you should not go.

      Your app code makes me think your app does not use the Alloy Framework. So you ONLY work inside the resources folder for code.

      If you do use the Alloy Framework then you ONLY work inside the app folder and do NOT wander into the resources folder.

      No matter your choice to use or not use Alloy - you should NEVER play in the build folder unless you know what you are doing and then you are unlikely to as it rarely (some exceptions apply) makes sense. The build folder is where your code is compiled into for the process of using it with the simulators, emulators, devices and stores. This is an end result you should stay away from.
