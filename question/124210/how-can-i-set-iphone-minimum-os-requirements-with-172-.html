How can I set iPhone Minimum OS Requirements with 1.7.2 ?

Developing an app for iphone. How can I set a different value than 3.1 as the Minimum OS Requirements with Titanium 1.7.2 ? Apple suggests not to modify directly MinimumOSVersion field in the info.plist file.

I didn't find any possibility in the configuration of Titanium apart of modifying directly the IPHONEOS_DEPLOYMENT_TARGET field in the Library/Application Support/Titanium/mobilesdk/osx/1.7.2/iphone/builder.py but I don't know if it is a correct procedure or not and I usually don't like to modify those kind of scripts.

Anyone can advice on it?

Thanks

  • You can copy the info.plist form the build\iphone directory to the same directory as you tiapp.xml file and edit the contents as you like. It will then use this one when building your app.

  • Personally, I have modified the IPHONEOS_DEPLOYMENT_TARGET parameter in the build script you mentioned. We released our app requiring iOS 4 a few weeks back and haven't had any issues. Note that in iTunes connect, after you upload your application you can see the minimum required OS version in the Binary Details section. This can confirm your change took effect.

    • I did what Mike did (in my case I changed the IPHONEOS_DEPLOYMENT_TARGET from 3.1 to 3.2) in the Library/Application Support/Titanium/mobilesdk/osx/{SDKVersion}/iphone/builder.py file. It worked fine. I only needed to bump it up so that XCode validation (when deploying to the App Store) wouldn't complain about a missing app icon (launcher icon) file in the Resources directory, which is required if 3.1 support is desired. Instead, I have density-specific launcher icons (MyAppIcon.png and MyAppIcon@2x.png) in the Resources/iphone/ directory.

      It would be nice if Titanium would let us specify this.
