Close previous Window - new Window TableView click event not firing

Previous experience of developing involves using separate window js files for each page, i.e. Kitchen Sink. I have looked at the recommended project structure but I am struggling to understand how to implement a single js context file.

Essentially I'm developing a mobile app which has 3 tabs, 2 of them contain static pages, and the last tab contains a form page to start with.

The user process should be as follows:

  • User completes the form
  • Separate window opens (in the same tab) and the form data is passed to the new window
  • Form window closes
  • User interacts with TableView in new window

Currently, the TableView in the new window is displayed, but the click events don't fire after closing the form window. If I keep the form window open, then the click events do fire.

btnLogin.addEventListener("click", function(e){
  if(IsValid){
    var childWin = Titanium.UI.createWindow({
      backgroundColor:'#fff',
      title: response.title,
      url: 'info.js',
      _name: name.value // form input value
    });
    Titanium.UI.currentTab.open(childWin, {
      animated: true
    });
    winLogin.close();
  }
}

Any recommendations on how best to solve this issue?

System Info

  • Application type: mobile
  • Titanium SDK: 1.7.5
  • Platform & version: iOS 4.3
  • Device: iOS simulator
  • Host Operating System: OSX
  • Titanium Studio: 1.0.7.201112281340

    There are different ways you can do that. If you have a lot code already written then just hide the bottom bar of that window and when they click on the login button set the current tab to the one you like (without closing the window or opening the tab), or you can create a 2 tab tabgroup app and start it with a modal window that will open the tabgroup onces the login button get the click event.
