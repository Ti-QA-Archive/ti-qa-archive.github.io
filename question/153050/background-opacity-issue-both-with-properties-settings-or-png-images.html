Background opacity issue both with properties settings or png images

I want my View or Window (the issue concerns both) to have an 80% opacity, and I'm not able to make it work as it should (or as I think it should).
First case:

var win = Ti.UI.createWindow({
        width: Ti.UI.FILL,
        height: Ti.UI.FILL,
    opacity: 0.8,
        backgroundColor: 'black'
    });

var view = Ti.UI.createView({
    opacity: 1,
    //other properties
});

var some_other_stuff         //create some other stuff

win.add(some_other_stuff);
win.add(view);
win.open();

Everything has 80% opacity, even if i set they should have opacity 100%.
But f I make the Window heavyweight (for example with fullscreen property) it works as it should.

Second case:
I use a png image (1x1 pixel, 32bit) with transparency as a background image using the property backgroundImage.
A simple example could be this (image with black color, 80% opacity):

var win = Ti.UI.createWindow({
        width: Ti.UI.FILL,
        height: Ti.UI.FILL,
        backgroundImage: '/images/bgOpacity80.png'
    });

win.open();

It works: I see a semi-transparent black window.
But I make it heavyweight by setting (for example) the fullscreen property I see a completely black window, without any transparency.
I tried creating the image both with Paint.net and Gimp 2.8.
I'm using latest Titanium version on Android 4.2.1 (on Google Nexus phone.

I understood all this after many tests, and I know I can use first case or second case depending on if I need heavyweight or lightweight window, but of course it's not the best solution.

Are these real issues or is it me who's doing something wrong?
Thanks in advance.
