What's the deal with Ti.UI.orientation?

So I've got a really simple app that's supposed to work in any orientation.
I've set all four orientation options in tiapp.xml, and created my main window with all four orinetationModes.

I then install it to an iPad, start the app, and, even though the iPad is in landscape orientation (button-left), the launch image is in portrait mode, and my app reports that the value of Ti.UI.orientation equates to Ti.UI.PORTRAIT. It's not until I rotate the iPad that it picks up on the correct orientation. Switching away from the app and then resuming also corrects the orientation.

The API documentation says that Ti.UI.orientation is deprecated, and links to a non-existent entry for Titanium.Window.orientationModes for setting modes, but gives no information about determining the current orientation.

Now if I could only get a correct value for the current orientation, I could at least restrict the orinetationModes property of the window to the correct one, but I can't even do that as things stand.

Any suggestions? If this is a bug, any news on when it'll be resolved. I really don't want to publish an app where the user has to turn their iPad 90 degrees and back again every time they start it up…
