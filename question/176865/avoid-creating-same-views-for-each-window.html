Avoid creating same views for each window

Hello,

I'm building my first app here for android.
Since my app requires a number of views = controllers i would like to know if there is some kind of a way to "include" stuff.
I'm playing on 3.X so i read that include is deprecated.

To give you an example, and please correct me if i'm doing something unnecessary (i've only been playing with this for a week now).

I have my index file and i have styled it with a couple of views, say a view at the top to simulate a custom title bar.
Now if the user clicks on a button say he wants to see all the players that i have we go to a different view
in which case i have to create the window and top view again.
Now if i need to do a minor change on that particular view i'll have to go and change it in both files.
You understand that this can scale up pretty fast and become an issue.
So is there any way i can avoid the issue and "include" certain views or small scripts ?

Thank you
