Editable tableview not working

Hello everyone,
I recently rebuild my iOS app with Ti SDK 3.5.0 and I can't swipe tableview row to delete. I set editable:true in tableview constructor and also for each row, but it's not working now. It was working fine in older versions of Titanium. How can I fix this? Thank you.

  • Hi Marian Kucharcik ,

    Thanks for your concern .

    To disable right swipe event listener for this module you just need comment out rightWindow form drawer .In that case right button will be inactive .

    var drawer = NappDrawerModule.createDrawer({
    leftWindow: createAPIExampleWindow(),
    centerWindow: mainWindow,
    //rightWindow: Ti.UI.createWindow({backgroundColor:"#FFF"}),
    closeDrawerGestureMode: NappDrawerModule.CLOSE_MODE_ALL,
    openDrawerGestureMode: NappDrawerModule.OPEN_MODE_ALL,
    showShadow: false, //no shadow in iOS7
    leftDrawerWidth: 200,
    rightDrawerWidth: 120,
    statusBarStyle: NappDrawerModule.STATUSBAR_WHITE,  
    orientationModes: [Ti.UI.PORTRAIT, Ti.UI.UPSIDE_PORTRAIT]
});

  • Hello Marian Kucharcik,

    When we swipe the row right to left a delete button is appearing on the right-hand side of the row. And it works for 3.4.1.GA, 3.5.0.GA and 3.5.1.GA.

    Test Code

    Ti.UI.backgroundColor = 'white';
var win = Ti.UI.createWindow();

var tableData = [{
    title : 'Apples'
}, {
    title : 'Bananas'
}, {
    title : 'Carrots'
}, {
    title : 'Potatoes'
}];

var table = Ti.UI.createTableView({
    data : tableData,
    top : 10,
    editable : true,

});
win.add(table);
win.open();

    For more info Row Editing and Moving Modes(iOS only).

    If you have any concern on this issue, then let me know.

    Thanks
