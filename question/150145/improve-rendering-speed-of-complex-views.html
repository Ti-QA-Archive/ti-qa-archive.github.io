Improve rendering speed of complex views

I have an app where views (plain views, image view, labels etc) are added in about 6-7 levels of views and child views. On some levels I need to know the absolute size of the parent view, which is not available until postlayout event.

The problem is that the rendering is slow enough to cause a visible delay to the user (even on a fast iPhone). If I try and hard code the size on the view, the delay is not there.

My hunch is that the views I add are rendered as soon as I add them and not until all views have been added. This causes a layout/render more times than would have been required if I could use something like view.StartLayout(), then add more views and sub views, and finally a view.EndLayout(). No layout/rendering would be done between those calls.

I know that prior to 3.0 there were similar calls (startLayout() etc). They are no longer there, plus they were only useful when updating properties of a view, rather than adding child view.

So, what options do I have?

Hard coding sizes might be possible for iPhone, but for Android with all it's various screen sizes, I don't think that is an option.
