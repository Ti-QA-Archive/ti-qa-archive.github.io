Frustrating Blue Navbar Buttons Bug

Titanium Mobile is moving along at an impressive pace, but its becoming increasingly frustrating to see basic bugs unaddressed.

Example: No matter what colours I specify when adding a native iOS navbar button (normal or back buttons) they always show as blue. The only workaround is to create custom buttons. I want native buttons though, and I use Titanium Mobile exactly because I don't want to be "mimicing" native controls like you have to do in other javascript based mobile frameworks.

What are my options here? Is this a well known bug? Do we have an eta on a fix?

Would it be possible to fix the button colour issue manually in X-Code once the app is built?

— asked September 26th 2011 by Shealan Forshaw

  • As far as I know, the native navbar buttons will use the color of the navbar itself, which is set using the barColor property of the window.

    If you need the buttons to be any other color than the navbar, then you will need to make custom buttons.

  • My apologies, you are indeed correct. The colour is actually determined by the navbar itself. If that isn't set (because you are using an image as a bg for example) it will show as the default blue.

    I think the documentation should be updated to note this as its very unclear currently.

  • As far as I know the color of the buttons in the navbar is set by the iOs ( iOs not Titanium ) depending on the barColor this has. There is nothing you can do except setting a barColor you want the buttons to follow.

    Options:

    1. use images to create custom buttons
    2. use window.barColor to make the buttons take this as base and a window.barImage to set another color (will be in fact an image) to the navbar.
    3. use instead of normal buttons a buttonbar with one label only. Anyway this will make you loose the arrow shaped back button.
