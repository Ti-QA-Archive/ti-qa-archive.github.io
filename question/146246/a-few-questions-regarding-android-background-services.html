A few questions regarding Android background services

Hi, I am testing out background services to see if it will work for an app idea I have.

For testing purposes I have used the code from here
https://wiki.appcelerator.org/display/guides/Simple+Android+Services+in+Titanium
and simply replaced the TI.Api.info with Ti.Media.Vibrate so I know real time if the service is working or not.

A few issues I noticed:

  1. When the phone is asleep the service is very random. Sometimes it wont vibrate at all, others it does it in random intervals.

  2. If I kill the app by using the default task manager it also stops the background service, If I then go into Settings > Apps > Running and look at my app it says '0 processess and 1 service - Restarting'. It looks like its trying to restart the service? Not sure here..

Any help is appreciated.

Using Galaxy Nexus 4.2, SDK 3.0, TiStudio 3.1

    I fought with background services in android and iOS for quite some time. The short answer is that you can't just background your app and expect it to run indefinitely, on either platform.

    On iOS, you have to use local notifications, and on android, you have to use the alarm manager and notifications.

    See my tutorial app, GrillTime. In part 3 of the tutorial, I have an in-depth discussion of this topic, along with a solution.
