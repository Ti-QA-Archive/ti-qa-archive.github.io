  • Hi

    Pickers do NOT go inside OptionDialog. An OptionDialog is a list of textual values in an array that are displayed according the the number of items and platform they are on. They do not accept child views or other elements.

    This is a standard example;

    <Alloy>
    <Window id="win" onClick="showOptions" title="Click window to test"
        fullscreen="false" onExit="true" backgroundColor="white">

        <OptionDialog id="dialog" title="Delete File?">

            <!-- The Options tag sets the options property. -->
            <Options>
                <Option>Confirm</Option>
                <Option platform="ios">Help</Option>
                <Option>Cancel</Option>
            </Options>

            <!-- The ButtonNames tag sets the Android-only buttonNames property. -->
            <ButtonNames>
                <ButtonName>Confirm</ButtonName>
            </ButtonNames>

            <!-- Add a View for the androidView property here. -->

        </OptionDialog>
    </Window>
</Alloy>

    You have not said if this is for an Android app - if so there is an optional androidView that will allow you to change the message area of the dialog - but not the options themselves.

    If you need a custom look - you need to create your own window and add a picker to that.
