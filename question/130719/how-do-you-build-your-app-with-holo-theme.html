  • Here's how I achieved Theme.Holo.Light, falling back to Theme.Light on older devices.

    In tiapp.xml:

    <android xmlns:android="http://schemas.android.com/apk/res/android">
    <tool-api-level>14</tool-api-level>
    <manifest>
        <application android:theme="@style/LightTheme"></application>
    </manifest>
</android>

    Then in platform/android/res/values (alongside, NOT under Resources) put themes.xml:

    <?xml version="1.0" encoding="utf-8"?>
<resources>
    <style name="LightTheme" parent="@android:style/Theme.Light.NoTitleBar">
        <!-- Any customizations for your app running on pre-3.0 devices here -->
    </style>
</resources>

    In platform/android/res/values-v11 put themes.xml:

    <?xml version="1.0" encoding="utf-8"?>
<resources>
    <style name="LightTheme" parent="@android:style/Theme.Holo.Light">
        <!-- Any customizations for your app running on 3.0+ devices here -->
    </style>
</resources>

    And in platform/android/res/values-v14 put themes.xml:

    <?xml version="1.0" encoding="utf-8"?>
<resources>
    <style name="LightTheme" parent="@android:style/Theme.DeviceDefault.Light.NoActionBar">
        <!-- Any customizations for your app running on 4.0+ devices here -->
    </style>
</resources>

  • Updating your <uses-sdk> target SDK version should get you a newer theme. However, if you want to request a specific theme, specify something like this in your tiapp.xml file:

    <android xmlns:android="http://schemas.android.com/apk/res/android">
    <tool-api-level>14</tool-api-level>
    <manifest>
        <application android:theme="@android:style/Theme.Holo">
            <activity android:name=".ThemetestActivity" android:theme="@android:style/Theme.Holo">
                <intent-filter>
                    <action android:name="android.intent.action.MAIN"/>
                    <category android:name="android.intent.category.LAUNCHER"/>
                </intent-filter>
            </activity>
        </application>
    </manifest>
</android>

    A few notes: the SDK generates a launcher activity with the Titanium theme (which sets the splash screen). The name of this activity will correspond to your application. If you want, you can build your application once, then copy the <activity> element from the generated AndroidManifest.xml in the build/android folder into tiapp.xml, and change the theme.

    The tool-api-level of 14 is required to recognize the newer themes.

    You can also use this technique to specify custom themes. Android themes files should be placed in platform/android/values inside the project folder (not inside Resources). You can also specify version-specific themes by adding values-v<apiLevel> folders. For example, values-v11 for Honeycomb and values-v14 for ICS.

    Finally, note that the SDK hardcodes the "Translucent" theme for certain things (such as modal windows). You can see this in the generated AndroidManifest.xml file: 

    <activity android:configChanges="keyboardHidden|orientation" android:name="org.appcelerator.titanium.TiTranslucentActivity" android:theme="@android:style/Theme.Translucent"/>
<activity android:configChanges="keyboardHidden|orientation" android:name="org.appcelerator.titanium.TiModalActivity" android:theme="@android:style/Theme.Translucent"/>

    If you want to override this, you need to copy the generated AndroidManifest.xml to platform/android and modify it there. This custom AndroidManifest file is not overwritten by the SDK, so you'll need to make any further changes there, not in the tiapp.xml file.

    • Is it a modal window? On Android, modal windows and translucent windows are launched as unique activities so they can be assigned a different, translucent theme. However, this can also be overridden. To assign different themes, add code like this to your tiapp.xml file:

       <activity
    android:configChanges="keyboardHidden|orientation"
    android:name="org.appcelerator.titanium.TiTranslucentActivity" 
    android:theme="@style/Theme.MyTranslucent"/>
<activity
    android:configChanges="keyboardHidden|orientation"
    android:name="org.appcelerator.titanium.TiModalActivity" 
    android:theme="@style/Theme.MyModal"/>

      (If you look in your generated AndroidManifest.xml, you'll see that by default these activities use Android's built-in Translucent theme. So to override them, we need to declare a different theme here. The activity names are the same for all Titanium Apps, so you can cut and paste this into your code. You can change the theme names as you choose–but don't change the activity names!)

      So now you've substituted your own set of translucent themes–now you need to actually define them.

      I've done this using different definitions of the themes for different versions. In platform/android/res/values, I have the following themes.xml file:

      <?xml version="1.0" encoding="utf-8"?>
<resources>
<style name="Theme.MyTheme" parent="android:Theme">
    <item name="android:windowBackground">@drawable/background</item>
</style>
<style name="Theme.MyTranslucent" parent="android:Theme.Translucent">
</style>
<style name="Theme.MyTranslucent" parent="android:Theme.Translucent">
</style>
</resources>

      Our "custom themes" here are really just inheriting from existing themes. If you're familiar with Android themes, you can add your own styles and themes. But it's a mighty slippery business in my experience.

      This file basically defines the themes for pre-4.0 devices so they look like they would with the default Titanium themes. Then in platform/android/res/values-v14, I have the following themes.xml file:

      <?xml version="1.0" encoding="utf-8"?>
<resources>
    <style name="Theme.MyTheme" parent="@android:style/Theme.Holo">
        <item name="android:windowBackground">@drawable/background</item>
    </style>
    <style name="Theme.MyTranslucent" parent="@android:style/Theme.Holo">
    </style>
    <style name="Theme.MyModal" parent="@android:style/Theme.Holo.Dialog">
    </style>
</resources>

      From my experimentation, it seems like the Holo theme already allows translucent windows, so we can use the regular Holo theme for translucent windows. For modal windows, we default to the Holo.Dialog theme.

      If you want to, you can also define a separate set of themes for 3.0 devices. (in values-v11). I'm not sure whether the Holo.Dialog theme was supported in 3.0, so you'd need to do a little investigation.

      This is about as far as I've gotten into Android themes. Hope it helps.

    • Sorry. You need to set the tool API level so it'll build against the correct SDK. Add the following line before the manifest element:

      <tool-api-level>14</tool-api-level>

      You might also want to set a default theme for the application. So the whole thing might look something like this:

      <android xmlns:android="http://schemas.android.com/apk/res/android">
    <tool-api-level>14</tool-api-level>
    <manifest>
        <uses-sdk targetSdkVersion="14"/>
        <application android:logo="@drawable/applogo" 
                 android:theme="@style/Theme.MyTheme">
            <activity
                   android:configChanges="keyboardHidden|orientation"
                   android:label="ThemeTest"
                   android:name=".ThemetestActivity" 
                  android:theme="@style/Theme.MyTheme">
                   <intent-filter>
                       <action android:name="android.intent.action.MAIN"/>
                       <category android:name="android.intent.category.LAUNCHER"/>
                   </intent-filter>
            </activity>

            <activity
                android:configChanges="keyboardHidden|orientation"
                android:name="org.appcelerator.titanium.TiTranslucentActivity" 
                android:theme="@style/Theme.MyTranslucent" />
            <activity
                android:configChanges="keyboardHidden|orientation"
                android:name="org.appcelerator.titanium.TiModalActivity" 
                android:theme="@style/Theme.MyModal"/>
        </application>
    </manifest>
</android>

      If you want to simplify matters, and you don't care about pre-3.0 devices, you could just use @android:style/Theme.Holo and Holo.Dialog directly here instead of using the custom themes.

  • I spent some hours on this (using Titanium Studio 3.0.1):

    1. note that filename is theme.xml, not themeS.xml
    2. if you create platform/android/res/values/theme.xml you will overwrite default titanium theme, so the compiler will miss "Theme.Titanium" that is used for the main activity anyway
    3. anyway to use a custom theme seems you only need to put in tiapp.xml
      ~~~
      <android xmlns:android="http://schemas.android.com/apk/res/android">
       <tool-api-level>14</tool-api-level> <!-- usata per compilare -->
 <manifest>
     <application android:theme="@android:style/Theme.Holo"></application>
 </manifest>
      </android>

    ~~~
    and it will run on all devices
