Webview Performance vs Safari - Any Tips?

Hi

For part of an app that I'm working on, I have a webview that allows the user to interact with an external wwebsite. I know it's not ideal, but it works ok for the most part.

The website is powered with Backbone and my problem is that in one particular part, where I click on the page to reveal a new part, it take ages.

When I tried the same in Safari on the iPhone it worked really quickly.

So I wonder is there something that I can do to increase the responsiveness within the webview? Perhaps a way to increase the cache size?

I'm at a loss here, so any help would be greatly appreciated.

Thanks

James

    The biggest difference between a WebView and Mobile Safari is that Mobile Safari has JIT enabled and WebView does not. JIT (just in time) is the ability to dynamically compile logic and execute it. It's what makes JavaScript engines fast.

    Apple does not allow apps to execute dynamic native instructions in any app. Titanium's WebView suffers from the same performance limitations that PhoneGap does.

    If possible, you could optimized the web page you're loading in the WebView. This includes reducing the number of DOM nodes, reducing the amount of JavaScript, reducing the amount of CSS, defer execution of JavaScript, minify all JavaScript and CSS, enable content caching on the web server, and so on.
