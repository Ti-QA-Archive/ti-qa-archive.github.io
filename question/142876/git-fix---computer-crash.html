GIT Fix - Computer Crash

Hi Guys

I was running the iPhone simulator for SDK 6.0 when the power went off.
Now my project does not have version control, seems that it needs some fix for this project.

I have checked other projects in the studio, and they all work.

Titanium Studio, build: 2.1.2.201208301612

Using git version control system on mac.

Mac OS X Version: 10.8.2

Thanks.

  • I just had a problem where git did not work form the Ti UI when it had been previously working. Suddenly (though no power problems here…), there were unstaged changes that did not show up in the Project Explorer or the Commit dialog. I went into Preferences > Team > Git and the executable path was missing. I found the git executable path, applied it, restarted Ti Studio, and it worked; commit-able changes showed up. Don't know if that's the same "project does not have version control" problem you describe. I recently updated my Mac to 10.8.2.

    • I had same issue when I converted over to Mountain Lion.

      The issue actually was that git crashed in a locked state, and it couldn't write to git anymore.

      So I had to trash the project, and recreate the project, and copy the files back.

      Then init git for the new project, and all was good.
