Update image in an ImageView

Here is the process:

  1. User selects an image from the gallery OR there camera.
  2. Image is displayed in an image view as a preview of whats being uploaded.
  3. User wants to change the image before they upload, so they select a new one from the gallery or there camera.
  4. Same image is displayed in the image view, but new image is uploaded.

Any suggestions on how to fix this? I found A LOT Of questions about refreshing the image in an image view, but not good answers.

Thanks!

  • I wanted to think you both for your answers. But, I figured out my issue. I had the event listener for my options dialog INSIDE of the even listener for the button that makes the option dialog pop up. I added some code to have the filename of the image display in the console. When I would select an image the first time, I would see the file name. When I would select and image the second time, no name would appear. When I would try to do it a third time, the app would crash. Once I separated the event listeners, the image view is updating just by using the hide() and show() commands without the temp names. Thanks for your help!
