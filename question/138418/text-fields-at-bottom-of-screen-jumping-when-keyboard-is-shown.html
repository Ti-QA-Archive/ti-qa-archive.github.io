Text Fields at bottom of screen jumping when keyboard is shown

I have fields at the bottom of a screen which were getting obscured by the keyboard. Therefore, I put them in ScrollView and now the fields can be moved to the top. But now, when a one of the text fields receives focus, the screen jump down so that the field is immediately above the keyboard. Is there a way to have the window stay stationary?

There is mockup code below. It create a ScrollView with 10 text fields. First, click in one of the text boxes to get the keyboard to come up. Then, scroll the screen to expose one of the lower text fields. Then, give the lower text field focus. The text field will jump to be just above the keyboard.

It just seems awkward. I would like the user to be able to expose the fields to be filled in and then just tap in each one. Right now, it is tap, screen jump, enter data, scroll, tap, screen jump, enter data, etc.

Thanks for any insight.

Application Type: iPhone, Titanium SDK: 2.0.2 GA, Platform: iOS 5.1, Host OS: OSX, Studio: 2.0.2.201205311912

~~
var win = Ti.UI.createWindow({
backgroundColor: 'white',
layout: 'vertical',
title: 'Window'
})

var sview = Ti.UI.createScrollView({
    contentHeight: 'auto',
    layout: 'vertical',
    top: 0,
})

var fview = Ti.UI.createView({
    top: 0,
    height: 'auto',
    layout: 'vertical'
})

sview.add(fview);
win.add(sview);

var f = Ti.UI.createTextField({
    width: 100,
    height: 20,
    top: 0
})
fview.add(f);

var fld;
for(i = 0; i < 10; i++) {
    fld = Ti.UI.createTextField({
        title: 'Field ' + i,
        top: 5,
        width: 100,
        borderStyle: Ti.UI.INPUT_BORDERSTYLE_LINE
    })
    fview.add(fld);
}

win.open();

~~
