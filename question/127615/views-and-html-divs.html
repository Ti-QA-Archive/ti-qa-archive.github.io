Views and HTML Divs

Hi all,
I am using titanium studio sdk 1.7.2, build mobile apps.People say that views behave like divs.When we write three div tags like <div></div><div></div><div></div> in html these three divs will appear one below the other on the page and their height is dependent on the content within them i.e their height increases when height of divs inside them increases.My question:Is the same thing possible with views where I dont need to specify top and height for the views to come below each other.This mainly because not necessarily I may know the height of the content that is gonna be displayed within the view.

  • Every object your create in your JS file in Titanium translates into a native iPhone / Android object. It's not HTML - so don't expect things to work as they do on HTML - you're not building a web page. The comparison to HTML Div's is made so you could understand it's part in the UI - that doesn't means it is translated into a DIV.

    If you want to place the views one on top of the other use the top property of each view and the height property. I think views take up a 100% width and height by default - but also check the 'auto' value. Check out the view control documentation for more information about different properties.
