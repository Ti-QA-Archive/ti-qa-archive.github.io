In webView ,fireEvent() fire only first run!

Hi,there!I have two problem as follows:
In app.js,I use 

var wv=Ti.UI.creatWebView({
    url:"map.html";
});
Ti.APP.addEventListener('fromwebview',function(e){
    player=Ti.Media.createSound({url:"hello.mp3"});
    player.play();
});
Ti.APP.addEventListener('backTo',function(e){
    player.pause();
});

In webview "map.html"(map.html and data.html are in the same webview),I use 

<a href="data.html" onclick="get()"></a>

function get(){
    Ti.App.fireEvent("fromwebview",{msg:"1234"});
}

when first run there,it can frie the "fromWebview"event, play the music,and jump to
the "data.html".

in webview "data.html",I use

< a href="map.html" onclick="back()"></a>
function back(){
    Ti.App.fireEvent("backTo",{msg:""});
    alert("1234");
}

to fire "backTo" event,it can jump to the map.html,but the fire dose not work,app.js can't catch the event,and the alert("1234") can not alert;
at this time,in the "map.html",I click the link,it can jump to "data.html",but it can't fire the "fromwebView" event any more.

I use titanium SDK 3.0.this problems can't work in IOS and Android both .I have dubegged for a long time,and didn't find out where is wrong.can you help me?
