Camera overlay width when rotating using a 2d matrix transform

I'm just finishing off an AR app and trying to draw parts of the UI which are clickable.

The problem I'm having is that to handle the landscape orientation I've rotated the overlay view by an angle depending on the orientation - This works great and even though I can reposition parts of the UI using their top and left values, the container they're in (camera overlay) is still the original 320px width. This is causing problems for me as any of the UI that's position outside of this becomes unclickable.

Here's what I mean - the camera overlay is given a background color of green and the close button is within this and clickable

When I apply a 2d transform and rotate the camera overlay - I can reposition the elements but the width will always remain 320. The close button is not within this and no longer clickable.

I've tried setting the width, top and left values of the overlay with no luck.

Hows anyone ever tried anything like this and fixed it?

  • I have found out (the hard way) that the best way of getting a components to always stretch, is not to use width at all, but use right and left properties. for example:

    var view = Titanium.createView({left:0,right:0});

    will ensure the view will always resize to the full width.

  • That was the first thing I tried. The white boxes in the screenshot use only a left and right value with a height. It's also within a container value that I have to set the width to 320 or 480 depending on the orientation. If it's not in a container then it always remains 320 wide. Also bear in mind that this view is an AR overlay that's attached to the camera view.

    I'm pretty sure this is a bug and that probably few people have tried doing this so it's probably gone unnoticed. I just wish the appcelerator guys check these forums as often as they used too.

  • I don't think the camera overlay was ever intended to have any animation/2d transforms applied to it. The solution was to place everything in another container view which gets added to the camera overlay and it's this that gets rotated. Bit of a pain but it works.
