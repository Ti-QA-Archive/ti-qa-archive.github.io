How do I store all the references to all the different pages using Alloy

My project is becoming very large and I'd like to store all the references to each window globally so I can open a certain window on any page. For example this is currently how I store a Global reference to the examination window:

$.examination.open();
Alloy.Globals.examination = $.examination;

This is really causing my Alloy.Globals namespace to become large. I'd thought I could just add an additional name space like so: 

$.examination.open();
Alloy.Globals.Pages.examination = $.examination;

Is this the best way to do this. Should I store them in a large Global array? Could I store those references in the CFG object? Could I use a Model to store the references? What is the standard for doing this. Just asking for advice.

  • I've seen 2 practices most used for this situation.

    • Create a separate commonjs module in the "lib" folder that takes care of managing all your global windows. In that module I often see useful functions for opening, closing, ordering, etc…
    • Do what you are doing now by just keeping track of the references in the Alloy.Globals object. Be aware though that keeping all your windows open is making your app unnecessarily consume more memory than it needs to.

    • Ranjith,

      Thanks for the answer. Well, I have done exactly that…I have created horizontal scrollable views but a lot of my project is built up in widgets and I have to have global references to them in order to open them at certain points in the app.

      Here's what I decided to do. I decided to use Alloy.createController and/or Alloy.createWidget("widget_name") to create each page but to set the values of each textfields (and other various controls) based on what is currently in the properties data. This way I can always close the window and the state remains the same regardless of what point in the app the widget and/or controller is created. This works a lot better than having to store global references to each of my pages just to keep the values in the textfields values.

      For example, on one page I'm creating a new controller when a user presses a button with the following command: 

      
Alloy.createController('NewWindow');

      And in the NewWindow controller I've got the following code (I defaulted the values to be an empty string) so their values don't appear until those values are actually set: 

      
//Set references to Models that will be used on the Page
var student = Alloy.Models.Student;

$.student_textfield_first_name.value = student.get("firstname");
$.student_textfield_middle_name.value = student.get("middlename");
$.student_textfield_last_name.value = student.get("lastname");

      By the way, the model student is first created in alloy.js like so: 

      
Alloy.Models.Student = Alloy.createModel("Student");

      The Student model has the properties of firstname, middlename, and lastname. Each time a users leaves the NewPage (through a button click) the state of the firstname, middlename, and lastname are also updated like so: 

      
student.set({
   "firstname" : $.student_textfield_first_name.value,
   "middlename" : $.student_textfield_middle_name.value,
   "lastname" : $.student_textfield_last_name.value,
});

student.save();

      Using this method means that, regardless of when the widget or controller is created in the app, that widget and/or controller will contain the correct state and this will mean that I will not have a ton of Global references to other windows in my app in the Alloy.Globals namespace.
