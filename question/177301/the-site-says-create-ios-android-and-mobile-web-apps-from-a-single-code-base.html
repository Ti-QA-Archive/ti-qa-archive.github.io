    • I would not say our claim is hype, but I guess that's opinion and perspective. If you meet the development environment requirements as spelled out in our documentation, you can write an app for all platforms from a single codebase.

      We rely on the native development tools (the compilers and SDK libraries provided by Xcode, Android development kit, Blackberry development kit). The Titanium code you write is prepared by our tooling, but ultimately compiled for the target platform using those vendors' native tools. Thus, we are constrained by the limitations put forth by those vendors.

      Apple releases Xcode only for OS X (their choice, not ours). To run on Windows or something else, we'd have to re-create their SDK, their compiler, linker, debugger, code signing tools, publishing tools, etc. That's way beyond our mission and would introduce a whole new layer of potential bugs, missing features, development lag time, etc.

      There are hacks for running OS X on a Windows computer; I don't recommend any of them nor do we support any of them officially. You could subscribe to one of the many "mac in a cloud" hosting services to run Xcode on a remote Mac. I encourage you to try the other development tools (our competitors) – other than a few offering cloud-compile services, none of them let you publish an iOS app without a Mac at some point in the process.
