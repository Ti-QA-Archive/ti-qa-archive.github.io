  • In my company, we declared a NavBar.js file where we keep permanent components and take _args to pass to it. You could create a view in this NavBar.js, give it an image to use for the nav bar, and then define individual methods for taking arguments and giving event listeners to it. So, example for how you'd call this::

    Ti.include("/app/NavBar.js");

var navBar = navBar.createNavBar({
    leftButton : '/images/backButton',
    leftCallback : function(e) {
        win.close();
    },
});

    And leftButton, leftCallback, etc. would all be args your pass to your NavBar.js to use in constructing it.

    This is just my experience. Might not be what you need, but I'm sure some aspects could be easily applied
