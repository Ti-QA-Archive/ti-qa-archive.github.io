    Hi

    Sounds like you actually need scrollableView rather than scrollView;

    The scrollableView works with a paging metaphor, so each visible page is shown, then you press the left or right arrow (or swipe or in code) and the page moves out of frame and a new page appears.

    The pages as views that can hold the same information any other view can.

    Example code from the documentation on scrollableView;

    var view1 = Ti.UI.createView({ backgroundColor:'#123' });
var view2 = Ti.UI.createView({ backgroundColor:'#246' });
var view3 = Ti.UI.createView({ backgroundColor:'#48b' });

var scrollableView = Ti.UI.createScrollableView({
  views:[view1,view2,view3],
  showPagingControl:true
});

win.add(scrollableView);

    The views in this example could be imageViews or views with other child views, buttons, labels etc inside them.

    • To place something in the middle you normally place a smaller thing inside a bigger thing and do not set the left or right of the smaller thing. This way it automatically centres the smaller thing to the bigger thing.

      This only works though if you are adding either one thing or several vertically aligned things.

      If you are adding several things horizontally to a single parent you need to create an offset value for the left position of the imageViews.

      If you are placing things with a know width inside a fixed space then simply find the offset like this;

      var offset = parseInt(outerwidth - imagewidth) / 2, 10);
