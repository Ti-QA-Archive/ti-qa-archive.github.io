Array resets between windows

So I have the following code in config.js…

var cartItems=[];

And this code in functions.js…

function addItem(title, car, color, qty, price){
    var item = {title: title, car: car, color: color, qty: qty, price: price};
    cartItems.push(item);
}

When I'm on the specific product page I have the following code…

addItem('Hub Cap', 'Honda', 'Silver', '1', '25');

Which everything works. If I do an alert after, it shows the information successfully injected into the cartItems array. But if I go to cart.js and call the cartItems array, the array is now empty… How do I keep the data in the array?

    Hi Casey,

    Using the property you can keep data in array like this.

    Ti.App.Properties.setList('cartArray', cartItems);

    If you want to use that array than do like this.

    var cartArray = Ti.App.Properties.getList('cartArray');

    In cartArray you get all array object.
    Hope this will help you.

  • In Titanium, every new js file has its own context. This means, even though variables may have the same name, they are new and different variables.

    What you can do is set up a single-context application that puts variables into the same namespace. To achieve it, check out Forging Titanium Episode 6: A Single Context Tab Group Template.

    Using properties is a valid alternative, but you risk polluting the global namespace.

    Cheers.
