Program Flow -- Order of Execution

This may seem like a very simple question to the experts out there, and while I am by no means an newbie developer, I am having trouble coming to grips with the order of execution of Titanium javascript apps.

Code snippet to illustrate my confusion:

    myButton.addEventListener('click', function() {        
        var myVar = false;

        myFunction(myVar);
        Ti.API.info("myVar: " + myVar);

        ...other code statements not relevant ...

       }

I have a button with a 'click' event listener. When the button is clicked, the event listener takes over. The first I do is define a boolean variable and set it to false.

I then call a function "myFunction," passing my variable. In "myFunction" I set it to true and pass it back. I then write it to the console to verify it is set properly.

However, when stepping through the code in the debugger, if I set a breakpoint, the code runs through all the statements in my listener BEFORE jumping to my function to execute. I'm expecting it to switch execution to "myFunction" before executing the remaining statements in my listener, and when control is passed back from "myFunction" THEN the rest of the code in my listener function is executed.

So, I'm obviously not understanding how the order of execution works in a case like this.

I would appreciate some enlightening, or a pointer to a reference where I can clue myself in.

Thanks!

  • Without seeing all your code, and stepping through myself, I don't think I can tell you why execution is going the way you see it in the debugger. But, I think I can help with your underlying question:

    The function, not written by me, makes a json call to a web service, sending and retrieving data from a backend database. Upon completetion of my function I want to refresh a label on the screen

    Because you don't know how long that network call will take, you can't rely on a return value. You must instead pass a function to that network-function, which it will call for you when the data has been retrieved.

    See the HTTPClient guide for more information on how this works. You might to have to modify that other developer's function to accept a callback and execute it when the data is retrieved. The function you pass to it (your callback) will contain the statements necessary to update the label.
