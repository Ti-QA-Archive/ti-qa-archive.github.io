    Hi Wylene,

    L('captured') mean Ti.Locale.getString('captured') - which we call string Localization.

    In order to provide Internationalization to apps we use this feature. Take a look at this Documentation.
    http://docs.appcelerator.com/titanium/latest/#!/api/Titanium.Locale

    Also,
    http://docs.appcelerator.com/titanium/latest/#!/api/Titanium.Locale-method-getString

    We usually stores this files at this location &#x2F;i18n&#x2F;LANG&#x2F;strings.xml where LANG stands for Language for ex. English we use en and
    for Spanish we use es and so on..

  • Hi

    I am confused when you say the titles do appear in the emulator and then they do not appear in the emulator. But I have some suspicions.

    If your question is "why do the tab title appear with an uppercase first letter" then it is because you are using the L() macro. This automatically swaps out the key (value inside the L macro) with a value from your strings.xml located within your i18n folder. This is the way your app can provide different international phrases without having to create more than one app.

    Location of i18n folder;

    {appname}/i18n/en/strings.xml // en = one of the languages you support

    The reason you may not be seeing some titles on Android is that is you do not provide the second optional L macro parameter and no strings file exists or no entry matches in the strings file for the current country then nothing is shown.

    To test this swap one value as follows;

    L('captured', 'No Value found')

    See what happens, chances are the strings file is missing or in the wrong place.
