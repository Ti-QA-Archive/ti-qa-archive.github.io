How to manage orientation change in Titanium app

I'm doing an application in titanium. Different screens are a little complex, because they have a lot of images, views, and webview.

Is there any way to solve it and don't see the transition?

My application is for Android and IOS, and I'm working with Titanium sdk 3.1.3.GA

  • orientationchange is the only event for monitoring orientation changes. You should be using applyProperties() rather than setting new properties individually.

    // instead of this:
Ti.Gesture.addEventListener('orientationchange', function(e) {
   if(e.source.isPortrait()) {
      $.view1.top = 100;
      $.view1.left = 50;
   } else if(e.source.isLandscape()) {
      $.view1.top = 50;
      $.view1.left = 100;
   }
});

    // do this:
Ti.Gesture.addEventListener('orientationchange', function(e) {
   if(e.source.isPortrait()) {
      $.view1.applyProperties({
         top: 100,
         left: 50
      });
   } else if(e.source.isLandscape()) {
      $.view1.applyProperties({
         top: 50,
         left: 100
      });
   }
});

    With applyProperties, all of the changes are applied in a single UI repaint operation. When set individually (as with the first code sample), each property change is applied and the UI re-rendered. With a complex layout, you will see the elements move individually as properties are set.
