iOS : SMS/Messages-like tableview fix to bottom

Hey guys,

Ok so I'm trying to create a chat application (again) and am running into some fun stuff that I've researched before and haven't had any luck so I'm seeing if anything's changed :)

A long time ago I posted this question that describes pretty much what I'm trying to do again:
Stack Overflow Post

If you look at the way the messaging app works, when you open the app it loads up the tableview scrolled all the way to the bottom already. I'm assuming then that this means they have the messages stored on the phone and they are loading them into the tableview before the window actually renders and scrolling to the bottom without an animation (similar to the answer to my stack overflow post above). New messages would then be appended to the bottom and scroll to bottom is called again.

The issue I'm having is any time I do that in Titanium it loads the window and then snaps to the bottom… and that snap is not exactly ideal :) I'd LOVE to make it so we could just fix the tableview to the bottom instead if the top…

Does anyone have any thoughts on a good way to do this? If not, it looks like I'm going to have to create a custom tableview module, which is fine, I'm just double checking before I dive down that rabbit hole.
