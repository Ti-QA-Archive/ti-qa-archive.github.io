Audio queue error renders app useless!

This is problem is not easy to reproduce.

I have an iPad App, just got launched to the AppStore (called Radio HD). This is a radio streaming app.

Sometimes, when starting stream.play() I get an Audio queue error. When this error has popped up, the whole app is a wreck. Functions stop working, and all animations stop.

The fact that this error shows up randomly is bad enough, but it renders my app useless.

Someone else had the same problem here ( http://developer.appcelerator.com/question/78481/catch-error-audio-queue-creation-failed ), but I was unable to find any way to fix it.

When doing animations after the error (and that is a main feature in my app), the debugger goes crazy, and almost all variables are suddenly turned to undefined, even if they are created AFTER the error to begin with!

EDIT: The objects do move, but the animation doesn't happen. So if i click on one radio channel, I might be clicking on a totally other object. (The graphics dont update it seems)
