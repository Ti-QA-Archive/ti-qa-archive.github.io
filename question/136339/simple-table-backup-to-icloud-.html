"Simple" table backup to iCloud ?

Hi!

I've looked all over the documentation but can't seem to find anything regarding iCloud backup support (iOS).

In my app I have a couple of SQLite tables that really need to be backedup along with the app via the user's iTunes and/or iCloud. The tables contain text and integers only and the data is put there by the user.
Each user may put several hundred rows in the tables over time, maybe even thousands.

Since this setting exists

db.file.setRemoteBackup(false);

I figure there are some backup possibilities? =)

I don't need a great backend for my app, I just want to make sure the users don't loose their
data if their iPhone break or get stolen.

Is there an easy way of just telling the app to backup my tables when the user backups his/her iPhone to iCloud or iTunes? Note that there is no need to backup WHILE USING the app at this point.

If there is a good answer anywhere already, please point me in the right direction. =)

Thank you!

  • A couple months late, but if you just need to backup if your users lost their phones, iOS has those data backed up by default, given that the user restores their old data to the new phone.

    If you want to sync data across iDevices, then you're out of luck. Ti doesn't support iCloud (probably never will). You'll have to either deploy your own web service, or use Ti.cloud. Neither are as smooth and seamless as iCloud, since both ways will require your user to create some sort of account.
