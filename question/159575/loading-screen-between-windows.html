Loading screen between windows.

I'm trying to show a "Loading…" modal to show to the user in the transition between two different windows, so the user will not think the app crashed in some way.

Window1 —-> Loading… —-> Window2

But when I get back to Window1 from Window2, there is still the "Loading…". There is no way I can remove this modal after I open Window2. I tried with remove, using fire event in Window2 to remove the modal in Window1…

When I add and remove the modal in the same window, it works fine (appears and disappears). The problem is between windows.

Here is the code:

In Window1.js I have:

win.add(loadingModal);
var Window2 = require('ui/common/Window2');
var window2 = new Window2(tabGroup);
tabGroup.activeTab.open(window2);
win.remove(loadingModal);

The "Loading…" only disappears if I move the remove above the new Window2:

win.add(loadingModal);
var Window2 = require('ui/common/Window2');
win.remove(loadingModal);
var window2 = new Window2(tabGroup);
tabGroup.activeTab.open(window2);

But this is not the effect I want, because it takes one more moment to the new window open after the modal disappears, because I make a call to a webservice in Window2.

This is a problem of "parent-child" windows? I really don't know how to solve this.

Thanks in advance for any help.

Best regards.

    Thats what activity indicators are for…
    Fire the activity indicator before opening the window, and remove it when your window/callback is finished (be sure to keep in mind removing it if the xhr call fails too)
