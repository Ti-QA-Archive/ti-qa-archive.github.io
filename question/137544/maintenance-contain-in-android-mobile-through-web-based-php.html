    Well, it depends on how much customization you desire.
    Colors, text, images, easy enough to change. Have your application read in a json file that contains its configuration. You pull a new json file from the server and apply the settings from it in your application. You could cache the json file and read it at each startup until a new one is available, or save the settings into Ti.App variables. Extending this approach, it would also be possible to add additional windows/tabs. That gets a bit more difficult as the main structures would need to be declared at build time.
