Saving / Syncing ACS Objects locally on the phone

Hello,

I am creating an app where I will be making full use of ACS, however, I need the app to be able to have some of my custom objects saved locally, so the user can still have functionality when it is off line. The offline functioning of the app is quite important. The best solution I have found so far was in this blog post:
http://developer.appcelerator.com/blog/2010/06/how-to-persist-complex-javascript-objects.html

which is basically using JSON serialization and titanium properties. The issue with I have with this is if I am saving an array of custom objects like this, it seems you have to retrieve the entire array of objects, as opposed to just one of the items.

Essentially I want to have my custom objects available offline.

Any help? Thanks!

  • I use ACS and sync locally to an sqlite database. It isn't two way syncing (I basically update one way from ACS to the database) but it allows users to be offline and access their data. It works really well and because it is a database, it's fast and you can simply query what you want. You could try that approach.
