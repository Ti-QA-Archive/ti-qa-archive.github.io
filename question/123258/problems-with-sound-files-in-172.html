Problems with sound files in 1.7.2

My app partly consists of 20 photos on which you can tap to hear a voice telling what animal is in the photo. Everything worked well until I updated to SDK 1.7.2.

Now, completely random, the sound might not play at all or it will only play a small part of it. I tried going back to 1.6.2 and everything worked perfectly again.

I have tried with both mp3 and caf files, same result. Since it's so obvious the problem is the SDK version, does anyone know of this problem and if the bug has been repoted? Or if there is a workaround.. For now I have to stick with 1.6.2.

  • This is the event of 'singletap' on the image. The path to the file has been set before with the custom variable audiopath.

    function ImageTouch(e){    
    var image = e.source;
    image.removeEventListener('singletap', ImageTouch);

    var player = Ti.Media.createSound({
        url: image.audiopath
    });
    player.addEventListener('complete', function(e){
        image.addEventListener('singletap', ImageTouch);    
        player.release();
    });
    player.play();
}
