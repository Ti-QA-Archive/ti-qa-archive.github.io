Alloy: layout system for different screen sizes

Hi Tony!

I have completed a portion of my new project and running well in iPhone. Then I tried to run in android devices, and noticed that everything fits exactly the way that I lay out views according to iPhone's screen size, despite its a smaller android device. I am super impressed!!

what's the technology behind this? does Titanium calculate the space and gaps according to one device and apply to all devices?

    It depends entirely on how you create your layouts. It's really easy to make your layouts NOT cross-platform, so I'm glad you happened upon a good method unintentionally. Personally, I prefer to use constraints for determining the size and position of components so that they can adapt to the size of a given device. By constraints I mean left,right,top,bottom. If you started hard-coding values for width and height, then you'd start to see the UI looking differently between platforms.

    This is only one small part of creating cross-platform UIs. Considering platform, form factor, density, etc… are all relevant when trying to support as many platforms as possible.
