Studio won't find Android SDK

Hi- I'm having all kinds of trouble (about 6 hours in now) with Ti and Android's SDK manager differing on which SDK's are installed. I try to get Ti to install the SDK's it think should be there, it does and tells me I'm fine (from the Dashboard). I got to my project and no Android build options are present. I go back to Dashboard and I'm not "fine" anymore and am missing SDK's. The Preferences > Platform > Android options similarly will tell me I've pointed to a good directory and then tell me the next time I access the screen that it's no longer valid.

Android's SDK manager says I have the required SDK's installed. I'm completely lost as to what's going on. I've seen several QA sections about this but none fixed the issue.

Where is the mismatch here? Where is Android's SDK looking and where is Ti looking? Where is Ti installing the SDK's it downloads?

Titanium Studio Version: 3.3.0.201407100905
Latest Titanium SDK: 3.3.0.GA
Host OS: Mac OS X

  • Hello, Ben, The same problem happened to me and it's really irritating. The thing that solved for me is I downloaded and installed a fresh copy of the Ti SDK 3.3.0.GA again. To do that

    • Download the SDK.
    • Go to your studio/help/install titanium sdk/install from URL and there browse the sdk zip file path.

    Install the sdk and restart the studio. That worked for me. Hope it does for you. Thanks.
