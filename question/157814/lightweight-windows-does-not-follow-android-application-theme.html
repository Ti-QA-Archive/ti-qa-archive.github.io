Lightweight windows does not follow Android Application theme?

Hi all,

I'm using the latest release of Titanium (3.1.3) via the Alloy framework. Rummaging through the official and community docs I managed to set the base API level and theme for my test app:

//tiapp.xml
<android xmlns:android="http://schemas.android.com/apk/res/android">
        <tool-api-level>14</tool-api-level>
        <manifest>
            <uses-sdk android:minSdkVersion="14" android:targetSdkVersion="14"/>
            <application android:theme="@android:style/Theme.Holo.Light"/>
        </manifest>
    </android>

//index.xml
<Alloy>
    <Window class="container" fullscreen="true" layout="vertical" >
...
</Alloy>

However when my app closes the main window (index.js) and opens another one, I'm back to the stock window header that is black and thin. Here's the Alloy markup for my next window:

//page2.xml
<Alloy>
    <Window>
        <View class="container">
            <Label id="label">Cool!</Label>
        </View>
    </Window>
</Alloy>

I read that setting the theme at the <application> tag is supposed to make it application-wide. Or am I missing something? Is it caused by my suspicion that the new window is lightweight and follows the old convention?
