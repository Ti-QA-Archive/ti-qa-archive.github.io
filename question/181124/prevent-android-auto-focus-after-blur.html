Prevent Android auto focus after blur

Hi all,

I am developing a little widget and I wonder if anyone have noticed the weird behavior for Android auto-focus on textfield.
To show the issue, the code is really simple just as follow:

index.xml

<Alloy>
    <Window class="container">
        <TextField id="textfield"></TextField>
        <Label onClick="blurTextfield">blur textfield</Label>
    </Window>
</Alloy>

index.js

function blurTextfield(){
    $.textfield.blur();
}

$.textfield.addEventListener('blur', function(){
    console.log("textfield blurred")
})

$.textfield.addEventListener('focus', function(){
    console.log("textfield focus")
})

$.index.open();

I have one textfield in the page and a label when you click it, it will make the textfield blur. However, the really weird thing is it will again auto-focusing the textfield as soon as the blur is called, which is really annoying!
Below is shown in the console

[INFO] :   textfield blurred
[INFO] :   textfield focus
[INFO] :   textfield blurred
[INFO] :   textfield focus
[INFO] :   textfield blurred
[INFO] :   textfield focus

I notice that there is a discussion a few years ago about the auto-focusing issue on window open. However, the most of the workarounds in that discussion only consider the first time.

The only one seems promising is by inserting a dummy textfield out of the vision of the page, which is acceptable but somehow I hope there might be a more beautiful solution. Any thought on this issue? Thanks!!
