Position of tableView inside ScrollView

I have a ScrollView that contains a tableView. I want the tableView to take up 95% of the screen width (the scroll view gets 100%). And I want that the table view is positioned in the middle of the view.
Normally this is what automatically happens when I don't set left/right values. Also this works as expected on iPhone - the tableView is in the middle of the screen.

On Android however, the table view is put at the left side of the screen (as if giving it a left: 0 value).

This is my definition of the views:

    view.scrollView = Titanium.UI.createScrollView({
        width: '100%',
        contentWidth: '100%',
        contentHeight: 'auto',
        height: '100%'
    });

    view.tableView = Titanium.UI.createTableView({
        top : '3%',
        bottom : '3%',
        width : '95%',
        height: view.packageItems.length * view.packageWordListSettings.tableRowHeight,
        borderWidth : 2,
        borderRadius : 14,
        borderColor : '#aaa',
        separatorColor : '#aaa',
        backgroundColor : '#fff',
        scrollable: 'false'
    });

Is there anything I can do to get Android to position the tableView in the middle like it does on the iPhone?

(Tried this with Ti sdk 1.7.5 and 1.8.0.v20110828131421)

    Hi Kathrin. :)

    First thing … I must say that you (and everybody else) should post workable example. That means that it can be run without any modifications and from example it should be visible what the bug is. Otherwise, we have to do that instead of you.

    Now, about this bug you have … it could be related to this: http://jira.appcelerator.org/browse/TC-435

    You could try to wrap scroll view's content to regular view which has width and height set to 'auto' (so, add table view to view and view to scroll view).

    Also, using opposite position properties (top and bottom for example) affects dimension property on Android (height for example) and you should be aware of that.

    Hope this helps. ;)
