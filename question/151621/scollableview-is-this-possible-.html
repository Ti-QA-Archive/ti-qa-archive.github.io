    • I think I have worked out what you are trying to achieve and why it has been hard to work it out.

      You appear to want to know when the scrollableView is on the second page and then you want it to do something - in this case go back to the first page and then call your intent.

      The problem you are having is determining when you are on the second page. You appear to be trying to detect this by working out the left position of the row2 - which is the the view on the second page. However as the row2 sits inside a container, that left position is relative to that container and not the display - so it will always be "0" as you are not actually changing it.

      The thing you need to check is currentPage, this tells you the index of the current page and can be found in several ways. The easiest one is also the most likely you should use and the quickest for you to implement.

      scrollableView.addEventListener('scrollend', function(e) {
    if (e.currentPage === 1) {
         // do your intent stuff here
    }
});

      This uses the scrollend event as it allows the scrollable view and user to have finished there choice before is does something - otherwise things go a little crazy, based on what you are trying to do.

      You can also find out the current page at any time by doing this;

      alert(scrollableView.getCurrentPage());

      This returns an index value with the number of the current page index.
