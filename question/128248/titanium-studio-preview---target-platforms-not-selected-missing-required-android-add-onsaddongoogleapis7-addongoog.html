Titanium Studio Preview - Target platform(s) not selected (Missing required Android Add-Ons[addon.+google.+apis.+7, addon.+goog

Application type: mobile
Titanium Mobile SDK Version: 1.7.5
Mobile SDK Timestamp: 11/02/11 17:00
Mobile SDK Build Number: ab20af7
Platform & version: Android 2.1, iOS 4
Device: n/a
Host Operating System: OSX 10.6.8, Windows 7-64
Titanium Studio: Titanium Studio, build: 1.0.7.201111111017
Build: jenkins-titanium-rcp-release-72 (origin/release)

Date: 11 November 2011, 10:21:58

Problem description:

TiStudio Preview offers creation of Titanium Mobile Module Project (menu File->New->Other…)
First notice is that on both platform (OSX and Windows) the Deployment Targets (Android) is not active.
Filling in Project name and Module Id gives an error message in the dialog:
Target platform(s) not selected (Missing required Android Add-Ons[addon.+google.+apis.+7, addon.+google.+apis.+8])

I have checked if all necessary Google API's where installed. Also PATH/ALIAS references where ok. Because it was not working I tryed to create a module manually.

This is the result in OSX (same on Windows):

titanium create –platform=android –type=module –name=calc –id=org.appcelerator.calc –android=/Developer/Android/android-sdk-macosx Traceback (most recent call last): File "/Library/Application Support/Titanium/mobilesdk/osx/1.7.5/module/module.py", line 319, in <module> main(sys.argv) File "/Library/Application Support/Titanium/mobilesdk/osx/1.7.5/module/module.py", line 316, in main module = ModuleProject(config['platform'],project_dir,config) File "/Library/Application Support/Titanium/mobilesdk/osx/1.7.5/module/module.py", line 63, in init self.platform_delegate = ModulePlatform.create_platform(platform, project_dir, config, self) File "/Library/Application Support/Titanium/mobilesdk/osx/1.7.5/module/module.py", line 34, in create_platform return platform_module_class(project_dir, config, module) File "/Library/Application Support/Titanium/mobilesdk/osx/1.7.5/module/android/android.py", line 26, in init print "[ERROR] Couldn't find the Google APIs r%s add-on directory" % self.sdk.DEFAULT_API_LEVEL AttributeError: AndroidSDK instance has no attribute 'DEFAULT_API_LEVEL' Created android module project

Investigating further I noticed that the directory structure was changed in the new Android SDK r15 release.
So I follow the solution of Greg Berger, see http://developer.appcelerator.com/question/126699/creating-android-native-module-fail-creating-module-template

This solved the manually creation of a module. I had hoped it sorted out also the error in TiStudio but unfortunately it didn't.
I think som reference is wrong in the code of the Wizard for creating this module in TiStudio because of the syntax of the error message:

Target platform(s) not selected (Missing required Android Add-Ons[addon.+google.+apis.+7, addon.+google.+apis.+8])

It looks like a structure to add a directory reference addon + google + api -> addon google api dir reference?

Paul, I hope you have some more info now.

Goodluck,
Erik
