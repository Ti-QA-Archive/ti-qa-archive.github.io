Loading javascript files in WebView

I am trying to load a javascript library (dojo) inside a webview. The base of the library loads (dojo base) but the webview can not load additional files from the library so I can use additional components such as charting. If I load the library from an external web site instead of the local file system it works fine, but I don' want to download the library everytime I run the webview. Any ideas (nice to see this forum is using dojo) ?

    Ahhh, who doesn't love Dojo. Ok, so as you know, Dojo has some slick wizardry going on with their package/module/dependency management, so if you are getting any errors, exceptions, debug messages, core dumps, etc, those will be essential in getting to the bottom of your issue.

    Nonetheless, remember that Dojo loads everything based on where dojo is 'installed' (where dojo.js lives). Of course, this is relative to the applications webroot. I'm willing to bet that Dojo and a native webview don't necessarily agree on that location. That is why the external lib works just fine.

    If that is the issue, you can go about dealing with it in a couple different ways. You can consult Dojo Mobile and see what it can do for you (package is dojox.mobile), or you can grab yourself a copy of the Dojo source and create a custom build. Your custom build will be nicely packaged and compressed into a single file, which means you only need to know the location of 1 file rather than 20.

    I'm partial to custom builds myself, all the code you need, none of the code you don't, no worries about what files live where according to which application.
