I've seen this question asked before, but for some reason, the provided answers aren't working for me. My guess is I'm missing something.

    • I have Font Awesome working on my iPhone simulator no issues, so here are my settings and some things to make sure you do;

      • upper/lower case of filenames MUST be matched
      • there is no requirement to use info.plist, if you have;
        • only for the needs of fonts - backup and remove
        • for more than just fonts - backup, remove font section, backup again, create a new custom info.plist and merge back the non-font changes
      • Project > Clean very important*

      Sections of files related;

      tiapp.xml

      ...
<ios>
    <min-ios-ver>4.3</min-ios-ver>
    <plist>
        <dict>
            <key>UIStatusBarStyle</key>
            <string>UIStatusBarStyleBlackOpaque</string>
            <key>UIAppFonts</key>
            <array>
                <string>/fonts/fontawesome-webfont.ttf</string>
            </array>
        </dict>
    </plist>
</ios>
...

      Note the all lower case filename entry

      index.xml

      <Alloy>
    <Window class="container">
        <Label>Hello, World</Label>
        <Label text="\uf000"></Label>
        <Label text="\uf001"></Label>
        <Label text="\uf002"></Label>
        <Label text="\uf003"></Label>
        <Label text="\uf004"></Label>
        <Label text="\uf005"></Label>
    </Window>
</Alloy>

      index.tss

      ".container": {
    backgroundColor:"white",
    layout: "vertical"
}
"Label[platform=android]": {
    color: "#c60000",
    width: Ti.UI.SIZE,
    height: Ti.UI.SIZE,
    font: {
        fontSize: 36,
        fontFamily: "fontawesome-webfont"
    }
}
"Label[platform=ios]": {
    color: "#000099",
    width: Ti.UI.SIZE,
    height: Ti.UI.SIZE,
    font: {
        fontSize: 36,
        fontFamily: "FontAwesome"
    }
}

      This will result in a page containing;

      Hello, World
Cocktail Glass
Music Symbol
Search Icon
Envelope
Filled Heart
Filled Star

      Font file location

      // note all lower case filename
{appname}/app/assets/fonts/fontawesome-webfont.ttf

      This is tested as working.
