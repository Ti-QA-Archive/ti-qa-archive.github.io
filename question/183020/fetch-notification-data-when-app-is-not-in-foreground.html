Fetch notification data when app is not in foreground

Hi,
My app receives some Push Notification. I have to fetch the data located in the payload of Push and do something with it without the user intervention. I've already tried to create a background service but when the notification arrives and the app is in background i'm not able to trigger any event so that my background service can react to it. In other words, i'm looking for the eventListener that will executes some codes when a push notification arrives and the app is paused/closed. I'm not sure to use the good technique. If someone know the answer of my question or know another way to solve my problem i'll be very grateful.
Thanks

  • So, notifications don't fire anything to the app until they are 'touched' or 'acknowledged', so if you are trying to read the payload without the user doing anything, then you're out of luck. I would take a look at a service that monitors a socket then does something with the payload coming in. I haven't tried that exact approach myself, but just setting up the service to listen for a message would (should) let you react to it.
