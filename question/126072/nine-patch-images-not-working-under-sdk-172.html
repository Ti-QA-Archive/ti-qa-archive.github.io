Nine Patch Images not Working under SDK 1.7.2?

I've been trying to get Nine Patch images to work under Titanium SDK 1.7.2, but with no luck…

I am assuming that if I use the image URL "images/toolbarbg.png" for a backgroundImage property and then have a nine patch image file at Resources/Android/images/toolbarbg.9.png, Titanium/Android should use the nine patch image file as the stretchable background image. Do I have this wrong?

From reading other threads about nine patch image support, it looks like they were working under previous versions of the Titanium SDK.

    I finally figured it out (1.7.2). What I'm doing is creating a directory, "platform" in the root directory (same directory build and Resources directories are). Then, in the platform directory, create android directory. In android directory, create res directory. In there, I have drawable, drawable-hdpi, drawable-ldpi and drawable-mdpi. Inside of drawable, I have testing.9.png. Now in my code, I access it by saying: backgroundImage: "/images/testing.png". Good luck!
