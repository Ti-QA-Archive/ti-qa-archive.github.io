  • Accepted Answer

    java.lang.String cannot be cast to java.lang.Boolean;

    Which part of JAVA development do you fail to understand? You see, while JavaScript allows you to be lazy, JAVA on the other hand does not and in the end your JavaScript values need to match the type that JAVA is expecting. It would appear that somewhere, in your app, you, or the system is trying to shove a string (round peg) into a boolean (square hole).

    The answer is right there in the log. If you have trouble with an 'obvious' problem like this, you are in for a lot of disappointments.

  • As Stephen rightly says, this is telling you what the problem is - the fact it may or may not be associated with the 3.2 upgrade isn't relevant - it's telling you the error.

    I for example, have an Alloy app and a certain controller/view was crashing. The item in question showed a faded out background and white label (it was a help/tutorial popup). I got this error and it turned out to be me specifying "false" in an alloy tag for a particular view that I wanted to have NOT responding to touch events so

    touchEnabled="false"

    I removed this attribute and the view worked.

    So, it's telling you the problem - solve the problem and THEN identify if this is indeed an issue with 3.2 and if it is - raise a ticket!

  • I think the problem is somewhere in Dalvik.

    Offending line is:
    clickable = TiConvert.toBoolean(proxy.getProperty(TiC.PROPERTY_TOUCH_ENABLED),

    TiConvert.toBoolean is:
    public static boolean toBoolean(Object value, boolean def)

    which calls:
    public static boolean toBoolean(Object value)

    which checks type of object, and does the right thing.

    I think that Dalvik runtime does some weird optimization, and this fails.
