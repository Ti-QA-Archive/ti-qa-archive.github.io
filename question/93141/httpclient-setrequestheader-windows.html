httpClient, setRequestHeader, Windows

Okay, I've upgraded my desktop application to use the new 1.1.0 API, and so far, so good. I'm only running into one issue.

I'm using an httpClient object to connect to a server, authenticate, and then download a file. The server authenticates with a session ID, sent inside a cookie, like so:

var cookie = 'sessionid=foobar1234567890';

httpClient = Titanium.Network.createHTTPClient();
httpClient.setRequestHeader('Cookie', cookie);

// do stuff...

This works in OS X and Linux: I'm able to send the cookie, authenticate with the server, and download my file. In Windows, however, this doesn't seem to work, and I think (although I'm not 100% sure yet) that setRequestHeader() is the culprit.

Any ideas? Has anyone else run into this? Why would Windows behave differently?…

Thanks!
