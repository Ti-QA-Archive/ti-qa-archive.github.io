Build 64-bit (ARM64) iOS Module

Hi,

I have an existing module which was build using 3.2.0.GA. But when I build the application using 3.5.0.RC using the module built using 3.2.0.GA, the application gets stuck at the splash screen. In the log it shows that it is unable to load the module. But when building the same application using 3.4.0.GA, then it is working properly.

Any idea as to how can I build the same module using 3.5.0.GA and make it 64bit compliant, and also able to run on 32-bit devices (like iPhone 5)
