  • I have also been trying to figure out this problem to put FB social plugins into the app. The route I went with is logging in via the Appcelerator's Facebook API and then for some pages that I want these social plugins I am going with a webview.

    Currently there has to be a double login situation. They have to login via the app and then in the webview if they want to post to the social plugins such as likes or comments. Yeah i know this sucks. But havent figured a way around it.

    I think i have narrowed down where the problem is. The webview has it's own local cookie jar that is holding the FB session. If somehow you can take the authToken and place a cookie down that it recognizes then the social plugins would be logged in with that current user.

  • I'm working on the same thing at the moment but have been running into some problems. I've read that it's possibly to attach an access_token to an HTTP POST request and send it to a Graph URL, like 'https://graph.facebook.com/PAGE_ID/likes'– and that should work. But so far I haven't had much luck. It's certainly possibly to embed a WebView into one of your app's views and use a standard IFRAME or FBML implementation of the the Like the button (the kind you see all everywhere). But I'd prefer to avoid a WebView.
