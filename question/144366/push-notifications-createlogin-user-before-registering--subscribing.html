Push Notifications: Create/login user before registering & subscribing

I've been working back and forth with the Titanium API for push notifications, and need clarification on one thing.

Before we register (Titanium.Network.registerForPushNotifications) and subscribe to a channel (PushNotifications.subscribe) - do we need to create an individual user on that device using ACS (Users.create and then users.login)?

— asked November 5th 2012 by Zvi Band

  • Yes for some reason ACS does require you to login first before managing Push subscriptions. If you are only using push and not any other ACS functionality, you can just create a single user in the management console and always login with that user in the app. That way you don't have to create a user for everyone who installs the app.

    It worked for me, but I definitely wish they would remove that requirement, doesn't seem necessary if you only want Push.
