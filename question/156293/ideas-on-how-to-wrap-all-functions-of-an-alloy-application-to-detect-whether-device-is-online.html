Ideas on how to "WRAP" all functions of an alloy application to detect whether device is online

Just tossing out a question about approach. I am building an application where the client only wants it to work when the device is online. So I am trying to figure out the most efficient approach to this using the alloy framework. The best I have come up with is in the onLoad event of each window/view, we run a 'Check Online' function to see if we are online or not.

Not sure if this will get cumbersome or not, just wondering if anyone has done this kind of thing before and has any experience they can share about their approach.

  • Hello,

    you should be using ti.network.online when you launch the app. You can assume that you can open the app with network available in the beginning. YOu should be able to run like this just fine, since the 3G/4G data is usually going up and down if you are moving, so you don't want the app to stop if you randomly are without network if 5 seconds later you actually have signal.

    On the other hand, if you insist in doing this window by window, just add a open event in each window. In the event listener, check for the network, if it's not available, just throw another event blocking the app. This alternative, from the UI/UX point of view is not ideal. But if your customer wants this, well, you can provide it.
