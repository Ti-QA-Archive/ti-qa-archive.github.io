Issue with remote URL in ImageView

I am unable to find the issue with this code snippet. Though the image loads fine in the browser, it doesn't load in the Titanium Android app. Whereas, the same snippet works fine in Titanium iOS app.

var profilePicture = Titanium.UI.createImageView({
    image : 'http://m3.licdn.com/mpr/mprx/0_e0hbvuc9GQEH_UgPoViZvavZ850d342Pogq4vakwx8IJOyR1XJrwRmr5mIx9C0DxWpGMsWn6H8jQ',
    height : '85dp',
    width : '85dp',
    top:20
});

Please help me in finding the issue.

    Hi

    The URL you have provided does not contain a file extension, this is stored in JPEG format (I checked).

    If you save the file locally and add an extension the image works fine, if you add an extension to the remote file it will work fine.

    iOS is smart enough to determine the extension but Android appears to need the extension otherwise it reports the image as corrupted.

    Fix = add an extension.

    • You are welcome. I found the new URL also failed on Android but worked fine on iOS - for the same reason as before.

      I have uploaded your two example files to another server (they will be removed within 24 hours). Both have the extension appended and the name simplified (for my convenience only). I also made the image sizes correct for the images provided.

      Both of these work perfectly on Android and iOS without any issues or OS complaints of problems rendering.

      Try the revised snippets, one for each photo and see if you get the same results.

      var profilePicture = Titanium.UI.createImageView({
    image: 'http://eshotz.net/ah/photo1.jpg?' + Math.random(),
    height: '80dp',
    top: 20,
    width: '80dp'
});

      var profilePicture = Titanium.UI.createImageView({
    image: 'http://eshotz.net/ah/photo2.jpg?' + Math.random(),
    height: '80dp',
    top: 20,
    width: '80dp'
});

      The random element is there to force a download so we can rule out caching as an issue in anything.

      As you can see both work on both platforms.
