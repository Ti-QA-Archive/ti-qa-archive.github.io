The following log also appears in logcat when the keyboard type changes:

I/LatinIME( 145): InputType.TYPE_NULL is specified
W/LatinIME( 145): Unexpected input class: inputType=0x00000000 imeOptions=0x00000000

I am using SDK 3.0.2GA but checked that the problem happens also on SDK 3.1.0GA.

    Hi

    There has been a problem with adding textFields to tableViewRows in Android for some time, you may be running up against that.

    However you can try the following;

    var win = Ti.UI.createWindow({
    ...
    windowSoftInputMode: Ti.UI.Android.SOFT_INPUT_ADJUST_PAN
    ...
});

    Add the windowSoftInputMode entry to your window create and see if the situation improves.

    If that fails, add the same line to the tableView create;

    var tableView = Titanium.UI.createTableView({
    ...
    windowSoftInputMode: Ti.UI.Android.SOFT_INPUT_ADJUST_PAN
    ...
});

    Good Luck
