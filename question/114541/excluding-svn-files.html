    Seems this error is occuring when the build script copies image resources for android and your project contains .svn directories. Here is a quick fix for this case that works for me. Would be great to get this into the next build.

    Add the follow code at the beginning of the copy_resource_drawable() method at line 468 of mobilesdk/osx/1.6.2/android/builder.py

    if orig.find('.svn'):
   return

  • I found that the accepted answer was only partially correct. It needs to additionally include support for other source control systems such as CVS.

    builder.py defines a set of directories and files that are to be ignored as 2 global variables.

    ignoreFiles = ['.gitignore', '.cvsignore', '.DS_Store'];
ignoreDirs = ['.git','.svn','_svn', 'CVS'];

    We can use these variables to more generally ignore unwanted files while additionally allowing the list to be expanded in future releases.

    To leverage this knowledge,

    modify builder.py as follows:

    find the following line:

    fileset = []

    This is where the drawable fileset is built.

    modify the for loop to add the check for ignored files and directories and not add them to the fileset. The code block originally looks like:

    ORIGINAL:

        for root, dirs, files in os.walk(os.path.join(self.top_dir, "Resources")):
        for f in files:
            path = os.path.join(root, f)
            if is_resource_drawable(path) and f != 'default.png':
                fileset.append(path)

    MODIFY TO:

        for root, dirs, files in os.walk(os.path.join(self.top_dir, "Resources")):
        for name in ignoreDirs:
            if name in dirs: dirs.remove(name)
        for f in files:
            if f in ignoreFiles: continue
            path = os.path.join(root, f)
            if is_resource_drawable(path) and f != 'default.png':
                fileset.append(path)

    I have verified this works in versions 1.6.2 and 1.7.2 of the SDK, but should be very similar to fix in other versions.

  • If you can copy the project to a different folder when distributing, just delete the svn folders.
    You can do an svn export to a new folder, or you can simply run this command on Mac OSX or linux inside that folder: (warning. this will delete all the .svn folders inside that directory.)

    find . -type d -name '*.svn' -print0 | xargs -0 rm -rdf
