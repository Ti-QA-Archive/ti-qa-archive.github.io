Switching many images at one time

Is there a way to be able to switch many images to a different state.

I can have a dynamic number of images on the screen at any given time. Based on a selection they can either be in their 'highlighted' or 'unhighlighted' state. Looping over the images and changing them individually can cause a stutter effect that locks up the ui while each image changes individually.

Is there a way to take all the images and change them all at the same time.

  • Hi Paul

    You are doing it the right way, however there is something we can do about the 'stutter', which should do what you want by batching all the updates into one process.

    PLEASE NOTE; I have not tried this for image swaps. But given the tiny change to your code a simple test that costs you nothing.

    As you have seen, changing many properties (does not matter which), cause a redraw event on the item and anything near it. Do lots of changes and lots of redraws occur.

    There is a new pair of functions you can use that turn the process in to a kind of batch process.

    This example explains it in the most basic form;

    view.startLayout();  // you add this to the start of your changes
view.top = 50;
view.left = 50;
view.finishLayout(); // and this to the end

    By adding the startLayout and finishLayout calls BEFORE and AFTER your process - it switches of redrawing and simply does everything behind the scenes relating to calculations.

    You change view to whatever parent the images are in.

    Again - I have not tried your exact case - as I do not have to hand lots of images to try it on.

    Please report back either way.
