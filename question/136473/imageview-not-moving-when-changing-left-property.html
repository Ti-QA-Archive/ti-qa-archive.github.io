imageView not moving when changing left property

My app has several icons (ImageView) that need to change position on orientation change. So inside my event listener I have the following code: 

var str = '';
for (var i = 0; i < settings.items.length; i++) {
    tabBarItems[i].left = Math.floor(settings.width * i + (spacing * i) + (spacing / 2));
    str += tabBarItems[i].left + ',';
}
Ti.API.info('rotate: (' + background.width + ') ' + str);

tabBarItems is an array of ImageView objects and I'm trying to move them to their correct position by changing their "left" property. This works most of the time, but sometimes their position doesn't change.

The values output by Ti.API.info show the correct values for where the icons should be, every once in awhile the icons don't move. I can't really find a pattern to when it fails either.

Also tried using updateLayout() instead of just setting the "left" property, but no change.

Anyone got any ideas? Something stupid I'm doing?

Thanks for your help,
Brad
