webView with Image doesnt fit

Hi,

i have a webView with an image in it,- but the image doesnt fill
the entire space. There is a padding at top and left:

See how it looks like

Here is the code:

var webview = Titanium.UI.createWebView({
                size:{
                    width: '60dp',
                    height:'60dp'
                },
                html:"<img width='60'  height='60' src="+mitglied.profilbild+">",
                left:'0dp',
                center:{
                    x:0,
                    y:0
                }
            });

Later i add the webView to a tableViewRow:

row.add(webview);

What can i do ?

Greets

    Hi

    The answer to your question is simple, all browsers have some sort of margin on a web page so you need to add styles choose what margins you want even if they are zero. Nthe same is true of the web view which is a mini browser.

    The main problem as Anthony has pointed out is this is NOT the right way to achieve your goal and will be a perforce and memory hog.

    Use an imageView and include the property defaultImage which points to a local image as well as the property image that points to the remote file. If you have a connection problem then the default image is shown and for slow connections it will show this until the image downloads. This is the correct way to add images in a scroll view or table view.

    • Quick follow up, you are using size to set dimensions, this is a read-only property, it only reports back sizes once added to views and has finished laying out. Use the properties width and height at the root level of the object definition (same as your left).

      I am assuming you we're using center to try and fix the current offset, it will not as you have noticed and would not normally be used for this purpose anyway.
