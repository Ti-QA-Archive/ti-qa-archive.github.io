How don't show this message?

  • This has been discussed and argued here since I first came across Titanium. Appcelerator gives the answer that its a bug in Android and how the installer handles intents. That if you don't do it Appcelerators way (force a restart) there is a potential for running two instances of your app on a device.

    You can take a look at TIMOB-4941 which had the first set of documented work arounds (not really work arounds, but beautification).

    I want to say that in 1.8.0.1 we can now "dangerously" ignore the restart, but I can't find the documentation on that.

    We change the alert to something less offensive (Database has been updated) or (Your credit card information has been stolen) (j/k)
