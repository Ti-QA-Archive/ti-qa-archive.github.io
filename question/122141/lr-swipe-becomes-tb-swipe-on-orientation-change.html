l/r swipe becomes t/b swipe on orientation change??

the code below says that if i tap and drag the view (left or right) and release the view, it will snap back to original position. but it only works correctly in portrait view. if i rotate to landscape the same code only works if i tap and drag up or down and release.

is this correct behavior? shouldn't 'left and right' be 'left and right' no matter the orientation? i've only tested in the emulator, not on device.

var news_view = Ti.UI.createView({
    backgroundColor: 'orange',
    left: 320,
    right: 0,
    width: 450,
    top: 0,
    height: 1000
});
Ti.UI.currentWindow.add(news_view);

news_view.addEventListener('click', function(e) {
    origin = (e.globalPoint.x-320);
});
news_view.addEventListener('touchmove', function(e) {
    var offset = (e.globalPoint.x-origin);
    news_view.left = offset;
});
