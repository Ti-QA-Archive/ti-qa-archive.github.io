  • You could do this by working from the command line only.

    Titanium Developer downloads mobile SDK versions to /Library/Application Support/Titanium/mobilesdk/osx/<version>, or on windows, C:\ProgramData\Titanium...

    Inside the directory for the mobile SDK release you wish to use, there is a titanium.py file. This is a command line interface to many functions Titanium Developer usually provides. Run this script with no arguments for help. titanium.py is not a total replacement for Titanium Developer, however. If you want to learn how to do EVERYTHING Titanium Developer does, launch the Titanium Developer binary from the console with a –debug flag. This will print verbose debug logging to the console. In the console, you should be able to see what Python scripts are being invoked to do task X, Y, or Z. For example, if you want to package for distribution, you would do this through the UI, and then in the console, see which Python script was executed by Developer, and what arguments it requires.

    Eventually titanium.py will be a full-featured CLI to go along with Titanium Studio, but you can peek behind the veil with Titanium Developer today to circumvent it if you need to.
