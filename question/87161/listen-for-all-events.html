  • Theoretically you could extend the Titanium UI components you're working with and construct a method that takes into account all the possible events, but this not recommended. You should know what specific events you're looking for and what functionality you want to happen per event. This way you are creating specific functionality that won't "surprise" the user or yourself during use and especially debugging.

    — answered December 8th 2010 by Michael Szul

    • This may be the way to go. You certainly wouldn't want to use this in production code, but it is a good way to experiment with the API and get a good feel for what is going on behind the scenes without having to write a lot of code to capture every possible event.

      This is also valuable because the docs aren't 100% up to date and several objects can capture events that aren't noted in the API reference.
