  • I have installed and run the emulator at least three times on my computers. I have always found that the problem lay in my environment variables. Specifically the user variables. So I will copy and paste them:

    USER VARIABLE:

    PATH(only the word in variable name, after this is the variable value) %APPDATA%\Python\Scripts;C:\Program Files (x86)\Java\jdk1.6.0_43\bin;C:\Android\platform-tools;C:\Android\tools;

    SYSTEM VARIABLE:

    (Variable name): JAVA_HOME C:\Java\jdk1.6.0_43;

    (Variable name): PATH: %JAVA_HOME%\bin;%SystemRoot%\system32;%SystemRoot%; %SystemRoot%\System32\Wbem;%SYSTEMROOT%\System32\WindowsPowerShell\v1.0; C:\Python27\;C:\Python27\Scripts;C:\Program Files (x86)\Java\jdk1.6.0_43; C:\Program Files (x86)\nodejs;C:\Program Files (x86)\Git\cmd; C:\Android\platform-tools;C:\Android\tools;C:\Android\tools; C:\ANT_HOME;%android_sdk%;

    Obviously, the paths have to be set to the directory where you installed the Android SDK and Java. Use the command prompt to confirm installation by typing in "java -version" and "javac -version". Adding the bin file at the end as well as the semicolon after every directory may assist in solving this issue further. The above variables worked for my computer.
