Subsequent windows in tab fullscreen and not showing tabGroup (android)

I'm using the commonJS template for ApplicationTabGroup. The tabs are displaying fine with the first added window. However, I used the default button to open a window to open/layer another window over that tab. However with android it opens the new window fullscreen which covers the tab - essentially disappearing my "interface".

This is apparently default behaviour as seen in the Forging Titanium ep6 video.

However it makes no sense to me why you would have a TabGroup in Android if the windows keep fullscreening over eachother. Am I suppose to use views inside the first tab window or is there a workaround. There are other questions on this but they are over a year old.

self.containingTab.open(ChatWin);  //opens fullscreen in android

    "over a year"? Try 10 hours! :-)

    http://developer.appcelerator.com/question/145689/tab-bar-problem

    This is a fairly common question. The answer is that this is the way TabGroups work on android. By design. If you don't like it, you could code up a set of views that look similar to Tabs, and then have your views show and hide content as you see fit.

    It might end up being more work than just using TabGroups, but it would give you more freedom to control the look and feel of your app anyway…
