ti.map module cannot be found on build

Using the ti.map module in order to use the google map api v2 and whenever we're using the new map module we get a "Requested module not found:ti.map"

I know the map module was installed correctly as I added it via the green "+" button on the gui side of tiapp.xml which should mean the module is recognized by the sdk. Anyways any help would greatly be appreciate.

Heres some code so you know if I'm doing it right:

tiapp.xml:

<modules>
        <module platform="android">ti.map</module>
</modules>

view:

var MapModule = require('ti.map');
var win = Ti.UI.createWindow({backgroundColor: 'white'});


var mapview = MapModule.createView({
    mapType: MapModule.NORMAL_TYPE,
    region: {latitude: 37.389569, longitude: -122.050212, latitudeDelta: 0.2, longitudeDelta: 0.2},
});


win.add(mapview);
win.open();

  • Hi

    Check you have a folder called modules at the same level as resources, inside that should be at least android and inside that ti.map.

    If you do not have that structure it will not work. Ideally you should locate the modules local to the app and not system wide (in my opinion). This helps keep functions and versions fixed for each app you work on.

    If you went through the green button then you should not need to manually tweak the tiapp.xml.

    I would suggest you run the menu Project > clean on your app (close the emulator or test device first). As this might be a simple case of it still using cached module info.

    It does work and does work well BTW

    • @Allen

      There are two types of modules;

      • CommonJS modules - you write these if you need them and they are file based. To link to them you specify the path and the file name (without the initial back slash \. These are stored inside your own project folders; resources for classic and app\lib for Alloy
      • Native Modules - you did not create these - they have been created by someone else and you are using them. There are stored in the projects modules folder - or stored with titanium itself and referenced automatically. To link to a native module in code you simply specify the name of module and NOT the path.

      In the case of ti.map we are talking about a native module, so it does not matter which folder your script run from you simply reference the name of the module.

      So 

      var variablename = require('ti.map');

      Is the only way to call the require from any scrip in any file in any folder within your app.

      Hopefully this has answered your question - if not create a new question providing as much details as you can and a short snippet of code you are working with for maps. Then update this question with a link to your new question so I know where to look.

  • I have had the same issue.

    If you put the "ti.map-android…zip" into Resources directory, that file will be never unpacked because it's considered "assets".

    Instead, if you put the zip file at the same level of tiapp.xml, it will be unpacked by titanium at first run.
    Look and give a try:

    /Resources
ti.map-android-2.0.0.zip
...
tiapp.xml

    Remember that ti.map module works only on device and with Titanium SDK 3.0.2+.

    HTH

    • @Damian; there is a difference in the modules for Google Maps that may be related.

      Ti 3.0.2 uses the module and you must manually download it.
      Ti 3.1.0 uses the module but it comes as part of the installation.

      In both cases you must select it, but only one requires manual download.

      If you have used the map for Ti 3.0.2 and now upgraded to Ti 3.1.0 - it might require the local module be deleted (placed inside your app folder) .

      Clean the project form the main menu, close the project and back up your app. Delete the local module folder, then re-open the project. Go and add the module back again - this time it should access the global version which is Ti 3.1.0 compatible.

      Big guess - but likely the reason.
