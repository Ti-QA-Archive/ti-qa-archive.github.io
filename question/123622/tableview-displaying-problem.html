TableView displaying problem

I have a problem with displaying TableView.

My program is very simple:

I made a mainWindow with a button which has action to open tableWindow.
And tableWindow has a TableView.

My code is here:

//Create tabGroup
var tabGroup = Titanium.UI.createTabGroup();

//Create mainWindow
var mainWindow = Ti.UI.createWindow({
    title: 'test'
});
var button = Ti.UI.createButton({
    title: 'open'
});
button.addEventListener('click', function(e){
    Ti.UI.currentTab.open(tableWindow);
});
mainWindow.add(button);

//Create tableWindow
var tableWindow = Ti.UI.createWindow({
    title: 'table'
});
var dataArray = [];
for(var i=0;i<10;i++){
    dataArray.push({title: 'testRow'});
}
var table = Ti.UI.createTableView({
    data: dataArray
});
tableWindow.add(table);

//Create tab
var tab1 = Titanium.UI.createTab({
    window: mainWindow
});
tabGroup.addTab(tab1);

//Open tabGroup
tabGroup.open();

and now when I tap button to open tableWin, TableView is displayed.
but once I back to mainWin, TableView becomes empty. This is problem.

like this:

  1. mainWin

  2. open tableWin …table is displayed.

  3. back to mainWin

  4. open tableWin again …table is displayed, but looks just empty.

What does cause this problem? does anybody know why?

  • Expanding on Dan's answer, when you open a window from within another window like in your example, you always have to create it and open it from inside the event listener callback.

    In your example, the window gets created only once, and then is destroyed when you close it, hence the result you are getting, when you attempt to open it again and the object doesn't exist anymore. If you look at the logs you'll probably see an exception…

  • Well, when you close the window with the table view, you are freeing memory, which is very advantageous. Tableviews are quite heavy, specially if they have hundreds of rows and many views attached.

    If you want to keep the tableview, you'll have to set up your window in a more complex way, where you use a single window and you hide and show child UI objects.
