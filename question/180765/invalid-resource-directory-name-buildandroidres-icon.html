invalid resource directory name: /build/android/res Icon

I have no idea what changed, but now when I clean my project and build for the first time, I get a Failed to package application: invalid resource directory name: /[my folder]/[project name]/build/android/res Icon

I checked my build folder and indeed a blank text file named "Icon" is generated. On the second time I build this file is erased and everything runs fine, I can even see alloy overwriting a ton of files before this happens. I am running 3.4.1 and alloy 1.5.0-dev (we have to run this for our globalization capabilities).
