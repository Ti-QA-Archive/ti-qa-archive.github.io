Is there a way to use a specific iOS provisioning profile?

I can't figure out how to specify which provisioning profile to use when building my app. All of my provisioning profiles are showing in titanium's iOS app config window. I'm using the latest 3.1.3 and sdk.

I am trying to build with my ad-hoc profile and titanium is using one of the other ones, so when I distribute my app as an api file, the other person can't install it and is showing this error in the device console log:

install_embedded_profile: Could not install embedded profile

  • You will see an option to select a provisioning profile under the publish configurations panel, and make sure you've installed the provisioning profile. Remember for adhoc you need to 'publish' it not Run or Debug it.

    Take a look at this if you haven't already :

    Ti Docs : Build your app, embedding the profile within the app's bundle
