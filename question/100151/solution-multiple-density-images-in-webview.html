Solution: Multiple density images in webview

For my articles displayed using a webview I wanted to display the correct resolution images for iPhones with the retina display and lower resolution images for older iphones. The solution I came up with was to send the device dpi in the json request set to my server to get all of the data used for displaying the tableview of headlines and subsequent article webview. You can access the devices dpi using Ti.Platform.displayCaps.dpi

This approach should have worked but it would appear 1.4.2 which I am using has a bug where it report the dpi as 160 when it should report it as 300.

So eventually I achieved what I set out to by using Ti.Platform.model and checking whether it was "iPhone 4" when generating the json response on the server.
