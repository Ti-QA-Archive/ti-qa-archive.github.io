Move & scale a scrollview is very unpredictable

I'm trying to implement what's effectively a custom map view which is a scrollview (for pinch/zoom/pan) which contains various images which I fade in and out to change the 'map', and various imageviews for pins. This working fine, except that as I change map images I also need move the scrollview to different locations and scales, as the maps are of different sizes (intentionally).
The problem is that when I scroll the map (map.scrollTo(100, 100);) it's not moving the where I expect it too. E.g. if I place an imageview at 100x100 inside the scrollview, I'd expect scrollTo(100,100) to place the top left of the image at the top left of the visible scroll area? I've tried adjusting these values depending on the scall of the scrolview, but that doesn't seem to help.

How can I predictably scale and scroll a scrollview to show a specific region of the content?
(This is only going to be an iPad app, using 3.2.2.GA / iOS 7)

  • scrollTo as per documentation, Moves the specified coordinate of the scrollable region into the viewable area.

    Try content offset as per documentation X and Y coordinates to which to reposition the top-left point of the scrollable region
