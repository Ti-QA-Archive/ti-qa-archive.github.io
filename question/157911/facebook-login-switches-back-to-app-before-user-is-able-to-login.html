Facebook login switches back to app before user is able to login

I'm trying to integrate Facebook in my app, I've set up the correct ti.facebook.appid in my tiapp.xml file, but when I call .authorize(), Titanium opens Safari as expected, but before anything is even drawn on the screen, it switches back to my app. It happens in iOS 6 and 7 simulators. Here's what I'm using to log the user:

var facebook = require("facebook");
facebook.permissions = ["name", "id"];
facebook.addEventListener("login", function(event) {
    if (event.success) {
        //open my window, by the way this is never reached
    }
});

  • The issue was that I misunderstood the permissions attribute of the Facebook object. name and id are just properties that can be acessed via Graph, not the permissions to acess those.
