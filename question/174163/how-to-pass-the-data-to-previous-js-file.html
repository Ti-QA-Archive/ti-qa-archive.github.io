  • No No No No.

    Using app level event listeners to move data from a child window to its parent is a BAD BAD BAD idea.

    1. Use Alloy.
    2. When opening the child controller, pass in a callback.
    3. When done in the child, exec the callback with the data you want to send back to the parent.

    (I'd allow for substituting Alloy for a project with commonJS utilizing functions to build out your UI.)

    Let me head off a few possible answers:

    I dont want to be using Alloy <- not our fault

    Our project is too big to start over <- It will only get bigger

    You dont understand our project structure <- If we dont understand it, you shouldnt be doing it
