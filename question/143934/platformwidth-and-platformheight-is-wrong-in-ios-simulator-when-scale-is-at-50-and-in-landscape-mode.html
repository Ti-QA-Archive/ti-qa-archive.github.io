platformWidth and platformHeight is wrong in iOS Simulator when scale is at 50% and in landscape mode

There appears to be a bug in Titanium in which Ti.Platform.displayCaps.platformWidth returns the height and platformHeight returns the width when 'creating' the view in landscape mode. When getting the platformWidth and platformHeight after the view has been 'created' (i.e. a button click), the properties return the right values. Surprisingly this only occurs when the iOS Simulator is set to scale at 50%. Anyone else encounter this?

  • this happens because when you run the app on the simulator it opens in portrait and then changes to landscape. What you need to do is listen for an orientation change event and reset the width and height variable using Ti.Platform.displayCaps.platformWidth / platformHeight inside the event
