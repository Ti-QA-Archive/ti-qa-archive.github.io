Setting an image to a nested ImageView

I'm working with Titanium 3.2.0 on a Mac OS X 10.9.1 and I'm using a physical device for debugging. I am trying to set an ImageView image inside a parent view (the container). However, the image does not show up at all. This is a problem for Android only. Am I doing something wrong?

I have this in my xml file:

<View id="imageContainer" backgroundColor="black">
     <ImageView id="nestedImageView"></ImageView>
</View>

And in my js file:

$.nestedImageView.setImage(myImage);

The container view shows up but the image that I set in the nested ImageView doesn't. If I do not nest the ImageView, the image shows up so the image itself isn't the problem. The problem is that I need to have the ImageView nested in the container. I couldn't find any posts related to this so I'm wondering if there is a known solution to this.

Thank you very much!

    • So I managed to get the image to show up by using a roundabout method of setting the ImageView height and width:

      $.nestedImageView.width = $.nestedImageView.image.width;
$.nestedImageView.height = $.nestedImageView.image.height;
