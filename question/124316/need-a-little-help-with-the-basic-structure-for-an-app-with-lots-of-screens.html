Need a little help with the basic structure for an app with lots of screens

I'm not not 100% clear on how I should structure an application with lots of tabs, and lots of "drill down" screens inside each tab. For an app with multiple tabs and screens it seems that each "screen" inside a tab would be a new window, thus a new execution context, but that seems a bit "heavy" to me. In addition, a new execution context makes it really difficult to add common components like my custom sliding menu and application wide styles via global variables.

I suppose I could just create and destroy views instead of opening new windows, but then I'd also have to create my own navigation bar from scratch, keeping track of each view, creating a "Back" button that goes to the previous view, etc., etc. That also sounds like a lot of unnecessary work, so I'm wondering if I'm just missing something. It seems there should be a way to drill down a series of screens with the nav bar, without opening a new execution context for each one. Am I just missing something fundamental? If so, can someone straighten me out?
