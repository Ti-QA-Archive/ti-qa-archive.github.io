    • It is true that the KitchenSink can be a little hard to digest at first, but it is filled with great example on how to use the Framework.

      But from what I understand, you are looking for something simpler.

      ####Try the following in your app.js file

      // Create the main window
var win1 = Ti.UI.createWindow({
    title: 'First Window'
});

// Create the button
var button = Ti.UI.createButton({
    title: 'Click me',
});

// Handle the click event
button.addEventListener('click', function(e) {
    // Create a new window and opens it
    var win2 = Ti.UI.createWindow({
        title: 'Second Window',
        url: 'second.js'    // second.js, contains the code 
                            // relative to the second window
    });

    // Open the newly created window
    win2.open();
});

// Open the main window
win1.open();

    • That;s because your second window is created using CommonJS. (which is good, but the example I gave you used the classic way of doing things).

      If you want the code to work with your SecondWindow.js implementation, simply change the eventListener like so:

      // Handle the click event
button.addEventListener('click', function(e) {
    require('SecondWindow').open();
}

      Which is a lot more leaner in terms of line numbers and readability.

      — commented September 3rd 2012 by Christian Brousseau

    • I just noticed one more thing.

      In your SecondWindow function, make sure that your last statement is:

      return self;

      Otherwise, the call to the open function will fail later down the road.
