fireEvent and async window for UI rendering?

I'm attempting to make my app faster and more responsive. I'm instrumenting my code and it looks like lots of time is spent just in rendering the UI.

I can either render the UI pieces on app startup, which causes the app to startup slowly (ie. the initial app splash screen is shown for a long time) or render pieces of the UI when the user presses the corresponding buttons, causing the buttons to respond slowly (ie. there's a noticeable lag between the user pressing the button and the corresponding UI being shown on the screen).

I've already moved some code to an async background window to speeds things up (eg. interacting with a web service, filling out various DB fields). I'm wondering if I can take a similar approach for UI rendering:

I have my app UI pieces being recorded in a variable (ala Tweetanium): App.UI stores the pieces of the UI, and I have various createSomethingView calls to fill out the pieces of the UI.

I'm wondering if I can move the UI rendering to the async window so it gets rendered in the background instead of hindering app responsiveness:

var asyncThread = Ti.UI.createWindow ({ url: "async.js" });
asyncThread.open();

Ti.App.fireEvent('render:SomethingUI', {UI: App.UI} );

// In async.js
Ti.App.addEventListener('render:SomethingUI', function(data) {
    data.UI.somethingWindow = createSomethingView() 
});

Is this a reasonable approach? How do you prevent UI rendering from being a bottleneck to responsiveness?
