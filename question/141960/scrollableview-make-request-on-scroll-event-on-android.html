ScrollableView make request on scroll event on Android

I am making scrollable gallery with scrollable view. Every 30 views I make a new request to server from where I get another 30 views. On each scroll I am checking if it is the first or last view and then getting the request. On iPhone is working ok, but on Android when I scroll between two views in scrollable view the event fired multiple times. From there I get multiply request one after another (3 or 4 requests). Since the scrollable view for Android supports only scroll event, is there any other solution so that I can be sure that the request will be called only once? I tried with e.currentPageAsFloat which should give float number while scrolling and when scrolling stops it gives integer number, but on Android that number is always integer.
