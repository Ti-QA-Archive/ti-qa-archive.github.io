Open window, close window, open again

This question was asked four months ago and never really answered. It looks like it cannot be done. I.e., all the examples I found in the Kitchen Sink instantiate new windows in the event handlers, instead of reusing already-created instances. Sounds like a lot of memory churning, and wastes time, too.

Has anyone been able to successfully reuse a window instance?

  • Robb

    I fear you may be falling into the pre-optimization trap by wanting to do this. Be careful never to pre-optimize. I think these sentiments are best summarized in Scalability Is A War; And, Don't Pre-Optimize , which states, "you should never pre-optimize, because what you think is the performance or scalability bottleneck is almost surely wrong…the best thing you can do is to keep your architecture and implementation simple, elegant, and easy to change."

    There is nothing wrong with opening new windows, or more typically views, when your application warrants it. The benefit is that all the objects that they contain will be properly destroyed when they are closed, and common coding mistakes (ie race conditions etc) can be avoided.

    Be aware of the Lifecycle Note section of the 1.6.0 Release Notes / Changelog, which recommends that events be used to guarantee the correct execution order:

    The Android Lifecycle changes in 1.6.0 mean that your Window's URL (and app.js) are executed synchronously as part of the Activity's onCreate(). Try to keep your initial code light, and don't count on window.open() being synchronous, especially if the window is launching a new Activity. Use a listener on the Window's "open" event to ensure that code gets executed after the new Activity has been opened.

    Have you seen an example in one of the Appcelerator projects (tweetanium is a good place to start) where they have used the technique you have described? If so, great; you can use it as an example, and post a link here to benefit others. If not, then I would assume that even Appcelerator does not code in the way you have described.

    If you have a specific performance issue with your application, then please open another question and explain it with the help of a usecase, and I am certain someone will help you to resolve it.

    Hope this helps
