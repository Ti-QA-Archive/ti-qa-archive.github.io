[Share] I did some improves with ChariTi

Hi everybody, this is just a share post for improved the ChariTi. ChariTi is a great alloy framework, and it's open source! But no matter how powerful framework, each person’s habits is always a little different, or that we can not meet the needs of all, ChariTi is not exception. In the course I also found some deficiencies, and therefore had to improve it myself :)

As this framework after I improved, just named: ChariTi-CB, the following are some improved features:

  1. Switch language by manual in the app. The normal situation, if you want to use the i18n feature or support multiple languages in the app, you need to let the user to change the phone’s language setting. For this case in ChariTi-CB, you can just let user to change the language in the app, and it will not affect the phone’s language setting.

  2. Support the animations when switch the page. There is no animation in ChariTi when you switch the page, so I added 4 animations , they are slide left, right, top and bottom.

  3. You can change the database name in the core.js file for the app. ChariTi just hard code it in each place when you call the database.

  4. You can set whether you need to use the default “Settings” button. ChariTi will add a settings button in the menu or header bar, if there is no settings feature in your app, you can disable it.

  5. Improved the APP.log function. ChariTi just can print the text debug messages within APP.log function, I have improved it and support to print the object and you can add the description with it.

  6. By default, ChariTi will save all debug message into database, and I think when we are in the development stage , it don’t need to save all debug messages in database in each time, so I add a setting value for handle it, you can disable or enable this feature.

  7. Improved the auto update function. This is a useful feature in the ChariTi, if you have a server or a website, you can put the configuration file (app.json) into your website, and you just need to update this file, it will be check in the app and auto update the settings what you changed, if you just do some small changed, then the user don’t need to go to app store to download the app again, he can just update the changed by auto download this remote app.json file. There is a “minimumVersion” setting in the ChariTi, if this value is changed, then user cannot update the app by remote configuration file, but there is also no anyways to let user to do in this time, I suppose the app should be popup a box alert user to go to app store or google play to download the new app, so I added two setting field for improve this: “appStoreLink” and “googlePlayLink”. When the “minimumVersion” changed, user click the alter box’s comfirm button, it will auto redirect to app store or google play to download the app.

  8. Enhance the remove children function. Please consider the following scenarios:
    There are 4 screens, you need to use the APP.addChild to navigate them: screen1 -> screen2 -> screen3 -> screen4, after you go into screen4, and you want to back to screen2, by default ChariTi just can back one screen with the APP.removeChild() function, or you can use the APP.addChild to add a screen2, but in this way, the controller stack will keep to increase, so we should use the APP.removeChild for remove the screen4 and screen3 after switch back to screen2. In this case, I enhanced the APP.removeChild function to support remove to define screen.

Ok, these are the changed of ChariTi-CB, if you are interest it, please go the following link to have a look the details :)
