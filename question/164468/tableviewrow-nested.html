    Hello Victor,
    when you iterate the first time over the ResultSet rowsQuestions, with rowsQuestions.next(), the record iterator gets to the end of it; it will not restart from the beginning at second round, because (quote) "the ResultSet object maintains an internal record of the current row".

    So its likely that when you try to access it the second time, the index goes out of range.

    You should create -out of the main loop on rowsConcepts, an array of rows for the rowsQuestions, and use it to append its rows to the tableview after each rowConcept.

    hth.

    • Hi Victor,
      try adding the following line in your code, between yours .push(rowConcept) and the .next() lines, as:

          [..]
    concepts.push(rowConcept);

    // this line adds questions rows after the current rowConcept
    concepts = concepts.concat(questions);

    rowsConcepts.next();
}
