  • here is an example using node module in Alloy project:

    1.install q.js, which will create a folder named "node module" and contain some files :

    $ npm install q
$ find node_module
node_modules/
node_modules/q
node_modules/q/README.md
node_modules/q/queue.js
node_modules/q/package.json
node_modules/q/q.js
node_modules/q/LICENSE

    2.copy the q.js to your app/lib/ folder:

    $ mkdir app/lib
$ cp node_modules/q/q.js app/lib

    3.declare it in your Titanium file:

    // in app/alloy.js
Q = require('q')

    4.use it in your controller:

    // app/controllers/index.js:
var defer = Q.defer();

    refer to: http://developer.appcelerator.com/question/156642/is-it-possible-to-add-any-nodejs-module-into-a-titanium-app#answer-285205

      — commented February 10th 2015 by Siwei Shen

      • You can sym link node_modules (ln -s ..&#x2F;..&#x2F;node_modules&#x2F;q&#x2F;q.js Resources&#x2F;lib&#x2F;q.js)so that you don't have to re-copy every time npm updates.
      • This will work for bower packages that do not require a DOM.
      • None of the native node modules will work. execSync, events, streams, child_process, etc. These are node specific and will not carry over to the compiled applicaiton. If you need feature like this your better off using a native solution and Titanium's module API
