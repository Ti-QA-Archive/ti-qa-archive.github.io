iOS Module development: how to add graphic assets to module

Hi there,

I'm developing an iOS module. It has a few buttons with icons, and I can't figure out how to include the graphic icons in the module distribution. If I run the code as stand-alone xcode app, works fine. But when including the files in the module project, buttons does not load the icons.

I've read in in the iOS module development guide a few lines about assets folder, but I can't get it working.

So, here are my two questions:

1- is possible to hard-code include a few graphics files with the module, without using the titanium assets folder? Graphic files are included in the project. This way, I would load them with these few lines in objective-c:

UIImage *imageBtn = [UIImage imageNamed:@"Done-Button-H.png"];
[doneButton setBackgroundImage:imageBtn forState:UIControlStateNormal];

2- Let's suppose that this is not possible and I have to use the assets folder. As I read in documentation, I would have to do something like this

NSString *path = [NSString stringWithFormat:@"modules/%@/Done-Button-H.png", [self moduleId]];
NSURL *urlImageH = [TiUtils toURL:path proxy:self];
UIImage *imageBtn = [TiUtils image:urlImageH proxy:self];

[doneButton setBackgroundImage:imageBtn forState:UIControlStateNormal];

In this case, the problem is that I'm not using this code in the main module file (the one that extends TiModule), but in an another class, so I do not have access to the proxy reference, and I can't reference to proxy.self.

I'm stuck. Which would be the best strategy for this?

thanks,
javier

using xCode 4.2, sdk 1.8.0.1, iOS 5.
