Time Picker in Android gets wrong value

I have a picker with type Ti.UI.PICKER_TYPE_TIME that calls onTimePickerChanged() on change. Works fine on iOS but in Android, getValue() returns the previous picker value. For example, current value of picker is 1:30PM, when I scroll to 1:29PM or 1:31PM picker.getValue() still returns 1:30PM.

Titanium SDK : 3.3.0
Android Emulator : titanium_1_WVGA800
Android Version : 4.1.1

    Why don't you use the value that is returned by the change event? That one should be correct.

    I usually use time pickers like that (basically):

    var time = new Date();
Ti.UI.createPicker({
    value: time,
    type: Ti.UI. PICKER_TYPE_TIME,
    locale: 'de-DE'
}).showDatePickerDialog({
    callback : function(e) {
        if(!e.cancel) {
            time = e.value;
        }
    }
});
