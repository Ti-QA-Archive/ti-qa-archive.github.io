Push Notification sound file path in Alloy

According to this link (http://docs.appcelerator.com/platform/latest/#!/guide/Sending_and_Scheduling_Push_Notifications-section-37551726_SendingandSchedulingPushNotifications-NotificationFeatures), I have to place my notification sound path file in the /Resources/sound directory. However, in alloy, the Resources folder is regenerated everytime I build the project. So how can I have a custom push notification sound in alloy?

I`m currently sending a notification with this format:

{

registration_ids: ["deviceToken"],

data: {
        payload:{

          android: {
              title: Example,
              alert: Sample alert,
              icon: push_icon,
              badge: +2,
      sound: reply_sound,
              vibrate: true,
          },
          custom_field1: wigle wigle wigle,

        }    
}

}

  • You have two options,

    1. Create a lib directory inside app directory (if its not there) and put your sound files/folder inside it. All the contain are copied into Resource directory while alloy compiles a project fro you.

    2. Put it inside assets directory. Its contain also copied to Resource directory after compile.

    Now access the files in the same way like how you are accessing in classic titanium.

    Hope this helps.
