iPad Distribution Build Producing .app Instead Of .ipa File

I am running Titanium Studio v1.0.7.201112080131 with SDK v1.8.0.v20111114103905. I am able to build successfully for the iOS Simulator and to my iPad with development credentials. Now I am trying to build for distribution. I selected "Distribute - App Store" in the project tab (I don't see an option for other types of distribution builds), and confirmed the enterprise provisioning profile and certificate. The project built successfully. I found a new "Release-iphoneos" directory in my build directory. But that directory contained a .app file rather than a .ipa file. I tried changing the file extension, but that didn't work. iTunes complains on install that the "app" is not the right kind.

Did I miss a step or setting somewhere that has left me building desktop apps in my Titanium Mobile project?

    Build for distribution.
    Then in xcode, open the Organizer, and then Archives.
    Choose the last build archives under your project name.
    You can then send that off to iTunes for release, or if it is adhoc distro, you can "share". That then builds the ipa file.
