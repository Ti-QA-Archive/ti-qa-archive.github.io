Is there a way to programmatically tell if an object is a Ti.UI.Picker/Ti.UI.TextField?

I would love to be able to manage a group of TextFields, TextAreas, Pickers, etc using a class that behaves like a form. The form class would need to be able to get the value of whichever type of input is added to it. Programmatically I am unsure how best to be sure which type of input each object is. typeof simply outputs 'Object' for all of them. What would be the best way to determine which class each object is (assume I cannot add a property to the input objects myself in order to accomplish this).

    Hi Dustin, don't check the typeof in that case…

    simply check either like ((Ti.UI.Picker).toString() === [object TiUIPicker])

    or assign an id with each object and check that id when you want to check the type of object.

  • hi,

    recently i have also faces same problem and my solution is little bit same like Ashish's answer but something different

    this is for ios

    
if(<object name>.valueOf() == '[object TiUIPicker]'){
    Ti.API.info('Picker');
}else if(<object name>.valueOf() == '[object TiUITextField]'){
    Ti.API.info('TextField');
}else if(<object name>.valueOf() == '[object TiUIView]'){
    Ti.API.info('View');
}

    this is for android

    
if(<object name>.valueOf() == '[object Picker]'){
    Ti.API.info('Picker');
}else if(<object name>.valueOf() == '[object TextField]'){
    Ti.API.info('TextField');
}else if(<object name>.valueOf() == '[object View]'){
    Ti.API.info('View');
}
