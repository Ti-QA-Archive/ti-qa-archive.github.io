Incorrect values in Titanium compatibility matrix

According to the following document, Titanium apps created with the SDK 3.1.3 would only run on iOS versions 6.1.x up to 7.0.x
http://docs.appcelerator.com/titanium/3.0/#!/guide/Titanium_Compatibility_Matrix-section-29004837_TitaniumCompatibilityMatrix-iOSSDK/TargetiOSPlatform

I have just performed some experiments with Ti SDK 3.1.3 apps and they are running on iOS 5.1.1
That's a big problem for my company as we would like to reliably tell our customers what iOS versions our apps support.

What's the correct compatibility for Ti SDK 3.1.3 then ?
We are porting our apps to Ti SDK 3.4.0 soon, is it going to support iOS 5.1.1 at all ?

Thanks,

Joseandro Luiz

  • Titanium apps might run on a lower iOS version. But it's hard to test for older iOS version, e.g. as no iOS 5 simulator is available with the latest XCode version.

    That's why I always define the minimum iOS version in tiapp.xml

    <ios>
    <min-ios-ver>6.0</min-ios-ver>
</ios>

    BTW: interesting iOS version stats can be found here: http://david-smith.org/iosversionstats/

    I personally won't support iOS 5 and iOS 6 in the future.

  • 3.4.0 will not support iOS 5.x or 6.x at all. You will be able to build for iOS 7 and 8 using 3.4.0. However, by using the CLI, and by keeping a copy of Xcode 5.x installed, you will be able to build apps using the 3.3.0 and earlier SDKs to target those earlier iOS versions.
