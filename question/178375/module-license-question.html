"Module License" question

I've been using the great Securely module from Ben Bahrenburg in an app.
When launching the app, it's apparently connecting to "http://api.appcelerator.net" to verify the license:

[WARN] :   TiVerify: (Timer-0) [4995,4995] Verifying module licenses...
[WARN] :   TiVerify: (Timer-0) [174,5169] Error Verifying License

When there's no network connection it logs an error:

[WARN] :   TiVerify: (Timer-0) [4995,4995] Verifying module licenses...
[WARN] :   TiVerify: (Timer-0) [174,5169] Error Verifying License
[WARN] :   TiVerify: org.apache.http.conn.HttpHostConnectException: Connection to http://api.appcelerator.net refused

While the module works fine even without a verified license, I don't understand why the app is trying to connect to api.appcelerator.net to verify the license.

I verified this in a new app that doesn't have any other modules or custom code.

I read in another Q&A that you could change the module's GUID in the manifest file to the guid of the app, but that didn't help.

The module is Apache Public Licensed and not from the marketplace. Am I missing something? Why is the app trying to connect to appcelerator.net?
