Android Theme - A 'type' attribute is required for <item>

Application type: Mobile Titanium

SDK: 3.3.0

Platform:Android 4.x

Device: Android GenyMotion Emulator & Sony Xperia E

OS: OSX10.9

Titanium Studio: 3.3.0

After upgrading to 3.3.0 I have numerous problems with Genymotion and getting my app to work on the new SDK. I just fixed an issue where the app was crashing when it started and it was caused by my Theme setup which I had modified to remove the actionbar. The app is failing to build now and throwing up an error.

The below code has now been updated to use AppCompat theme, I was still using Holo.

<?xml version="1.0" encoding="utf-8"?>
<resources>
    <style name="Theme.NoActionBar" parent="Theme.AppCompat.Light.DarkActionBar"/>
     <!-- Depending on the parent theme, this may be called android:windowActionBar instead of windowActionBar
        <item name="android:windowActionBar">false</item> -->
        <item name="android:windowNoTitle">true</item>
    </style>
</resources>

The problem I am facing now is though is that I get the below error when I am trying to build it.

A &#39;type&#39; attribute is required for &lt;item&gt;

I looked on Android docs and the only reference to type is the item name which tends to be the type, I've tried other items but the same error is displayed.

Anybody got any ideas?
