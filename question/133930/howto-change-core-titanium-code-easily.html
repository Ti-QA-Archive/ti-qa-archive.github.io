  • We have some very custom situations to meet client needs, and have SDK forks dedicated to them.

    Editing the files in the build directory won't do you much good as they will disappear when you clean a project.

    If however you edit the file in the actual sdk directory structure, those changes should carry over into the build tree.

    $HOME&#x2F;Library&#x2F;Application Support&#x2F;Titanium&#x2F;mobilesdk&#x2F;osx&#x2F;{sdk_version}&#x2F;iphone&#x2F;

    For example, we have a local organization. we need something particular for them, so they have their own sdk directory:

    $HOME&#x2F;Library&#x2F;Application Support&#x2F;Titanium&#x2F;mobilesdk&#x2F;osx&#x2F;1.8.2-myr&#x2F;iphone&#x2F;

    Changes to the their specific sdk carry over into their builds just fine.

  • In my opinion, your best bet would be to create a custom module which extends TiUIWebView to remove the security restrictions on running scripts. I did something similar to add some features that I needed to the standard TiUIImageView class. By using a module, you don't have to maintain a custom SDK build as new versions are released.

    If you do decide to build a custom SDK or just want to take a look at the code, you can get it from Github here:

    https://github.com/appcelerator/titanium_mobile

    The Titanium mobile SDK is definitely open-source – it says so right on their Github page – so you can hack away to your heart's content.
