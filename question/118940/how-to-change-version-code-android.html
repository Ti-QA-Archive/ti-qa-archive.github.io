  • Accepted Answer

    You Need to bump both android:versionCode and android:versionName

    <manifest xmlns:android="http://schemas.android.com/apk/res/android"
    package="com.appcelerator.aidori"
    android:versionCode="5"
    android:versionName="1.4"
    >

  • Hey guys, technically there are a hybrid of answers above to make a working solution but it is in-fact possible to just edit the tiapp.xml file.

    The initial code of line reads:

    <android xmlns:android="http://schemas.android.com/apk/res/android"/>

    what you need to replace it with is:

    <android xmlns:android="http://schemas.android.com/apk/res/android">
   <manifest android:versionCode="101" android:versionName="1.0.1"/>
</android>

    versionCode must be an integer and versionName should be a float or integer.

  • in some cases make sure you don't have <manifest> tag after that tag you used to
    for example 

    <android xmlns:android="http://schemas.android.com/apk/res/android">
   <manifest android:versionCode="2" android:versionName="1"/>


<manifest >
   <application android:theme="@style/Theme.mytheme" android:icon="@drawable/appicon" />
</android>

    DONT DO USE PREVIOUS CODE ! in this case second manifest will override first one and titanium builder won't be able to see versioncode number

    the correct form is 

    <android xmlns:android="http://schemas.android.com/apk/res/android">
   <manifest android:versionCode="2" android:versionName="1" >
     <application android:theme="@style/Theme.mytheme" android:icon="@drawable/appicon"/>
</manifest>
</android>
