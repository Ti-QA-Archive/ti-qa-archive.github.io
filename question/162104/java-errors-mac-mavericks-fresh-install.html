Java errors Mac Mavericks Fresh Install

I have just done a fresh install of Mavericks and installed the latest JDK from oracle 1.7.0_51

When trying to start Titanium Studio i get an alert

The JVM shared library "/Library/Java/JavaVirtualMachines/jdk1.7.0_51.jdk/Contents/Home/bin/../jre/lib/server/libjvm.dylib" does not contain the JNI_CreateJavaVM symbol.

When running from the comment line the following is reported

2014-01-30 16:51:02.671 TitaniumStudio[44489:507] Error loading /Library/Java/JavaVirtualMachines/jdk1.7.0_51.jdk/Contents/MacOS/libjli.dylib: dlopen(/Library/Java/JavaVirtualMachines/jdk1.7.0_51.jdk/Contents/MacOS/libjli.dylib, 265): no suitable image found. Did find: /Library/Java/JavaVirtualMachines/jdk1.7.0_51.jdk/Contents/MacOS/libjli.dylib: mach-o, but wrong architecture

After several hours of googling, I have added the JVM Capabilities to (and rebooted)

<key>JVMCapabilities</key>
<array>
        <string>CommandLine</string>
<!-- added the following -->
        <string>JNI</string>
        <string>BundleApp</string>
        <string>WebStart</string>
        <string>Applets</string>

</array>

I have tried adding (above and below -vmargs)
-vm
/System/Library/Frameworks/JavaVM.framework/Versions/Current/Commands/java

Any assistance would be appreciated

Titanium Build is 3.2.0
