Android build Error while messing with SVN control files

Hi,

I have a very strange error when I build my Android verion of my mobile app:

[ERROR] IOError: [Errno 2] No such file or directory: '/Users/myUSer/Documents/Titanium Studio Workspace/myProject/build/android/res/drawable-long-land-hdpi/_20f791a761.svn/all-wcprops'

What's going on here?
It's been a while since that project is under Subversion control but only today all my builds are failing.
I don't know how to solve. Any Ideas?

  • Hi Alain.

    This is well known issue that has been ignored by Appcelerator team for some time.

    Workaround is trivial, modify "builder.py" script that is located in Titanium SDK folder (for example: "C:\Documents and Settings\All Users\Application Data\Titanium\mobilesdk\win32\1.7.1\android" on Windows XP machine). Add following lines to "make_resource_drawable_filename" function (line 484):

    if orig.find('.svn'):
    return

    That should fix your problem.
