Pull to Refresh - deprecated event 'dragEnd' causes bug in new Window opening

In my code I have a Pull to refresh on a TableView:

myTableView.addEventListener('dragEnd',function(e){
    if (pulling && !reloading && offset < -80){
              pulling = false;
              reloading = true;
              labelStatus.text = L('updating');
              imageArrow.hide();
              actInd.show();
              e.source.setContentInsets({top:80}, {animated:true});
              setTimeout(function(){
                       checkUpdates(); // calls the server for updates
              }, 2000);
         }
});

This code is from the documentation, and it works well. Though when I open a new Window (due to a click on a row of the TableView) the TableHeader locks to the Updating… state, continuously and cannot hide it back.

I imagine this is due to the deprecated dragEnd event. Are there any work arounds?

  • SOLVED: The issue was from my end, it was not the 'dragEvent' at all. I simply forgot to add the resetPullHeader() in the example here: http://docs.appcelerator.com/titanium/latest/#!/guide/TableView_Refresh_with_headerPullView if anybody else stumbles on a similar issue.
