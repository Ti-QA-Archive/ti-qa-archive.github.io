ti desktop - use unzip codec to win7 user directory - executables get stripped

Is this some sort of win7 security? My app tries to unzip a companion app - so far I have not been able to get it to do so anywhere but the win 7 user data area, and then, it does so minus all the executables.

The thing works so well on XP!

  • This really sounds like a security restriction issue. Installing the app as user x and starting it as user y is always a bad idea. You are already using the user data area. Enter %APPDATA% in the Windows Explorer. Is this really happening there? Is the user allowed to copy an exe into your folder using Windows Explorer? You can try to switch off User Account Control. Is this a customer computer or your own? Some company admins do weird things to secure computers, try to talk to the admin.
