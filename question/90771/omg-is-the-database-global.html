    Yak

    The database is local. I can't anticipate there to be any way that one user could see the local data of another, bearing in mind they are on two completely separate devices, unless…

    1. You are distributing a pre-populated database, and are inadvertently including user-related data

    2. Your web service is not managing user sessions properly, and so the XML page is serving content to one user that is meant for another

    3. The XML page could be being cached by the server, meaning that they aren't regenerated for the next user's request.

    That's all I can conclude right now.

    Cheers
