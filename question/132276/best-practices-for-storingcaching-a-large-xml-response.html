Best Practices for storing/caching a large XML response

I have an API that spits back about 500 results. This is obviously too big to just parse and throw into a tableview. I'm looking for the best way to load the XML once, cache it and then parse the local version. Would this be the most efficient way to do this, or would looping through the results and adding them to a DB be the best way?

Any help is appreciated.

    Chris,

    the best way here, is do a service in the middle changing your xml into json, and then store all values in the local db. In addition to that, if it's data that have versions, you can put an md5 version into your json, and then every time your app starts, just download the json file as a file, and then parse it and update the local db.

    Sending xml to a mobile app is very bad for two reasons: it's big, so you will use more network connection time, and battery. It's complex to parse, therefore you will be using more cpu time and battery.

    Best,

    Mauro
