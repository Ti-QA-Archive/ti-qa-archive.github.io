Significant memory leaks with windows, views and more

Hi!

I'm struggling with memory leaks since days and I haven't found the right answer (here in the Q&As) until now. I have an app in the app store since ~1 year. Because of tiny bugs I now switched to API 1.8.0.1 (that is much more robust!). The app update is nearly finished but unfortunately I now have severe memory leaks.

My app is a dice game (App Store - Mia HD) opening some windows, showing/hiding views (sometimes using timeouts), and so on. With 1.8.0.1 I can play the game for only 10 minutes because my iPhone runs out of memory.

I cannot imagine that it is the intention of the Appcelerator team to provide an easy-to-use framework for developers providing the possibility to develop apps only with Javascript knowledge on the one hand, and on the other hand leaving them alone with their memory leak problems only providing a few hints how to solve parts of the problems with some workarounds. At least that is my feeling after reading several posts about this issue.

I already had a look at this post: Why is there still the problem with memory leak ?. But it did not work for me.

Does anyone have an idea or a solution?

Regards
