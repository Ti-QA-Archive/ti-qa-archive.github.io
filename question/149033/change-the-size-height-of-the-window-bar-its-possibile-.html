  • Hi

    Longer answer, the size of the standard iOS UI elements are generally fixed, there are human interface guidelines you can read to understand why. Attempting to make such changes will likely cause your app to fall foul of the verification and review process.

    As others have already said here, you can make your own version - BUT you have to ask yourself "is a little extra height worth the pain and trouble of creating a whole new implementation?".

    If you are looking to add additional information to the navigation bar, the it is likely you are simply better off either;

    • using the tileprompt property if it is text only
    • adding a view in the main window under the title/navigation bar fixed in position and adjusting the height of whatever else is there to allow for it. Let me know if you need a quick code snippet to show you what I mean.
