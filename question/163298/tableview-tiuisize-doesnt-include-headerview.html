tableView Ti.UI.SIZE doesn't include headerView

I'm using Titanium Studio 3.2.1 for iOS.

After populating a standard TableView with rows, I resize the table height in order to remove any "blank" rows that might show up if the screen size is bigger than the number of rows. I use this code to accomplish that:

myTableView.height = Ti.UI.SIZE;

That works great until I add a HeaderView to the TableView. As soon as I do, the Ti.UI.SIZE property still thinks that the size does not include the HeaderView even though it's part of it. Therefore, the new size of the table is smaller than the total number of rows.

How to I get the dynamic size of the TableView INCLUDING the Header View? And to clarify, my row heights are constantly changing so I can't simply multiply the number of rows by a standard RowHeight.

    I found a workaround, that removes the blank lines at the bottom of a table. You can create a FooterView and then the empty rows disappear. In the following example, I create a single pixel line to simulate the bottom line of the last row which doesn't appear when using a footer:

        var emptyFooterView = Ti.UI.createView({
            width:Ti.UI.FILL,
            height:1,
            backgroundColor:'#cccccc',
    });
    myTableView.footerView = emptyFooterView;
