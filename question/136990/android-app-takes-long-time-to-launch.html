Android App takes long time to launch

Hi guys i just developed sample app using titanium and it takes long time to launch on the emulator. Can anybody explain whey and how we can avoid it.

  • The Android Emulator is known to be very slow, so Rainer's suggestion to test directly on your device makes more sense.

    One trick to shave some time off starting the emulator is to enable Snapshot use.

    In AVD MAnager, simply edit your AVD instance (usually, there is only one). Then click the Enable checkbox.

    This will create some kind of save state of the Emulator's ROM every time you close the emulator. And this same state will be restored when the emulator is restarted. Which saves a lot of time.

    Hope this helps.

  • The question is, does it load fast enough on the actual device. Like Christian said in his answer, android emulator is extremely slow, so you need to test you app on the device as soon and as often as possible to get a realistic view on your app performance.

    One mistake a lot of Titanium developers do is require / include a lot (or all) of the application's JavaScript files or modules when the application loads, which means all of your JavaScript code is evaluated when the application starts, meaning you will get very slow loading speed.

    This is both unnecessary and inefficient, so you should design your application in such a way that you can evaluate only the minimum necessary code that's needed for your application to start up and present the initial user interface. For example, if you are using the CommonJS module pattern for your app, make sure you are requiring your modules at the point where you need them, not when your application is starting.

    Make sure you read and use Titanium best practices when developing your app to make sure it's fast and efficient.
