Problem reloading data

I have a tableview that will frequently be empty. Its rows also change frequently so I am reloading the data every minute. My problem is that when the tableview is reloaded, the new rows will show up but then disappear until the tableview is updated several more times. Then the new rows continue to appear until another update takes place, at which time I have the same problem. Also, when I have an empty tableview, I cannot get rid of the last rows without killing the app and reloading. Do I really need to delete rows before every reload?

If someone could point out the error of my ways (see code at https://gist.github.com/987691) it would be greatly appreciated.

Thanks! Heidi

  • The first thing that jumps out at me is you are recreating the tableview object in each xhr onload iteration.

    I'd suggest moving the createTableView() to prior to the xhr stuff, then just using tableview.setData(data) inside the onload function.

  • Hi Heidi,

    Put var xhr = Ti.Network.createHTTPClient(); within the loadData() function because all the xhr calls in the titanium are asynchronus and you don't know the state of last xhr call, it might be possible that your last call is still not processed and waiting for server response!

    Also implement the Doug's suggestion.
