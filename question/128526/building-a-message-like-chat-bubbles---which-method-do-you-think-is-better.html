  • I am also looking at building something like this.

    If you go the webview route i feel it will be less intense on the client side and better for battery life but I feel like you want have as much control and it wont look as native..

    If you go native (views,imageviews,labels) you will be able to achieve more control but what if you have lots of images/long conversations? will you have the client constantly hitting server for the data or will you have push functionality?

    If you have a webview I would assume you could create an ajax site and auto update the content (not sure about this as i haven't tested it yet).
