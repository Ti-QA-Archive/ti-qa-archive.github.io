  • node.js 0.10 (the current stable version) added much stricter type checking to path.join() and the CLI wasn't fully compliant with it. I haven't seen this problem with 3.1.0, but it's possible that we missed an edge case somewhere. We will look into fixing it, but in the meantime if you downgrade to node 0.8 then this problem will go away. If you don't want to uninstall node 0.10, then you can use the n npm package to manage your node versions.

    — answered April 22nd 2013 by Bryan Hughes

  • This error is caused in part by one of your provisioning profiles not being parseable by our library. We probably should do a better job error checking, though you are the first person to report this specific issue.

    I recommend going through your "~/Library/MobileDevice/Provisioning Profiles" directory and try to determine which provisioning profile file is bad.
