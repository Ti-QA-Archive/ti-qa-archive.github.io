Facebook 3.0 iOS 6 Authorization [NSError boundBridge:withKrollObject:]: unrecognized selector

Hello there – I'm using Facebook's iOS 6 authorization through Parse.com's SDK and all works well if I do fast-switching, however when using iOS 6 native-auth, if I touch "Don't Allow" I get this message:

[NSError boundBridge:withKrollObject:] unrecognized selector sent to instance

And when my iOS module's -(void)resumed:(id)note { … } is called, the KrollCallback that I had retained is now null… but it's not null when I resume via fast-switching method of Facebook auth.

I saw that other people were having this same error with Ti.StoreKit a few versions ago. Perhaps this has something to do with ACAccountStore and Titanium?

Thanks for any help… appreciate it and have a great day.
