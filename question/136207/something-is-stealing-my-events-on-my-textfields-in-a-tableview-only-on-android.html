Something is stealing my events on my TextFields in a TableView, only on Android

Strange behavior, only on Android (the same code works fine on iPhone): touching the TextFields in my TableView cause an initial focus on the TextField with the correct keyboard shown, then an immediate switch of focus to something else (not sure what), with an incorrect keyboard showing up.

The touch event is getting handled by something I can't find, and is causing the change in focus (it's as if the correct behavior takes place, then an extra unwanted behavior takes place switching focus away to something else).

If I touch the input a second time it gains focus and acts as expected. However, if I touch another TableRow, then touch the row with the TextField I see the undesired behavior again.

The same code behaves as expected on the iPhone: touching the TextField causes it to gain and retain focus with the correct keyboard.

Any thoughts on what could be stealing my events? Code looks like this:

root.menu = Ti.UI.createTableView({
            backgroundColor:'transparent',
            left:0, right:0, top: UI.topbar.height_dp,
});


var row = Ti.UI.createTableViewRow({ className: 'tableRow', height: '60dp' });
row.addEventListener("click", function(e){ Titanium.API.info('HERE') });

root.notesInput = Titanium.UI.createTextField({
            hintText:'Your Notes', left: UI.edge, width: '250dp',
            returnKeyType: Titanium.UI.RETURNKEY_DONE,
            borderWidth: 0, borderColor: 'white', backgroundImage: '/images/spacer.gif'
});
row.add(root.notesInput);
root.menu.appendRow(row);
