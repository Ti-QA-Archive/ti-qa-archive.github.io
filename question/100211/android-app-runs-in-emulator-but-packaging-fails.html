Android app runs in emulator but packaging fails

I am working on my second Android app. The first one works in the emulator and correctly generates the .apk file to be installed on my device. In general, I like Titanium and believe it will enable me to be very productive building cross-platform apps.
My second app is no more complex than the first, and runs correctly in the emulator, however, when I try to create the android package it fails. There is no indication of failure - the Ti Dev window says 'Creating android distribution', and after a few seconds that message disappears. No .apk file has been generated, however. I see no logging information or other indication of failure.
Can someone just point me in the correct direction to figure out what is going on? The lack of useful feedback from Ti Dev is very frustrating.

Platform is Win7 64-bit, up-to-date Android SDK, using API 2.2, and Ti Dev 1.2.2. Tried with several API versions and emulator settings, all run on the emulator, but no .apk.

Thanks,

Herb

    • I figured out the solution. I deleted the keystores file and regenerated all of my android package keys, and then the packaging worked. I can only assume that I mistyped the alias during the original key creation. I could find no evidence in the trace or any log file that there was no matching entry in the keystores file. It was very frustrating to spend time on such a trivial mistake.
