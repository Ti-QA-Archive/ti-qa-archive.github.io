Forward & Reverse Geocoders

What underlying service does the forward and reverse geocoders use? I've noticed a few limitations:

  1. I can't forward geocode a zipcode.
  2. Often reverse geocoding doesn't return a valid city (for relatively well known places).
  3. Reverse geocoding doesn't return a separate state field.

Are any of these planned to be addressed in 1.6?

Thanks.

  • Hi Frank

    I think Google provides all this now, doesn't it? Try using Titanium.Network.HTTPClient to retrieve the following URL:

    http://maps.google.com/maps/api/geocode/json?latlng=37.32,-122&sensor=true

    This returns json, which includes the US State.

    Also see this document and the Google Geocoding API.

    Hope this helps
