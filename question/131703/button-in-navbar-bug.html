Button in Navbar Bug

I have come across this bug in Titanium which I really need some help on. The code below will open a tabgroup window and add a button on the right hand side of the navigation bar.

The bug is you can actually fire the button click event by clicking the window just under the button… so dont actually click on the button or the nav bar but just below it.

This is causing me major grief because in my app when the user tries to click on the first row in the tableview its firing the button event. Its like the height of the navbar and the button and too high and its overflowing onto the window/tableview.

I have tried setting the height of the button but its not working. 

Titanium.UI.setBackgroundColor('#000');

var tabGroup = Titanium.UI.createTabGroup();

var win = Titanium.UI.createWindow({  
    title:'Window',
    backgroundColor:'#fff'
});
var tab1 = Titanium.UI.createTab({  
    icon:'KS_nav_views.png',
    title:'Tab',
    window:win
});

var label = Titanium.UI.createLabel({
    color:'#999',
    font:{fontSize:20,fontFamily:'Helvetica Neue'},
    textAlign:'center',
    width:'auto'
});
win.add(label);

var myButton = Titanium.UI.createButton({
    title:'Click Me',
    style : Ti.UI.iPhone.SystemButtonStyle.BORDERED
});
win.setRightNavButton(myButton, {
    animated : false
});

myButton.addEventListener('click',function() {
    label.text='You Clicked The Button';
});
win.addEventListener('click',function() {
    label.text='You Clicked The Window';
});

tabGroup.addTab(tab1);  
tabGroup.open();

Does anyone know how to correct this ?
