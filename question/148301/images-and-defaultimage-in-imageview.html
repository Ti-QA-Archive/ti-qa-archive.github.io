Images and DefaultImage in ImageView

I'm trying to figure out how to make an animated loading image show up while a remote image is being loaded in an imageview.

I know that i can set defaultImage and that will display until the remote image is fetched.

I also know that i can set the images array and start() the animation.

However, can the images array work with defaultImage somehow?

Can I do something along the lines of:

var image = Ti.UI.createImageView({
  image: remoteUrl,
  images: animationArray,
  defaultImage: images
});

image.start();

    Hi

    The two properties image and images are mutually exclusive, using one stops you being able to use the other.

    Depending on your platform there are a number of ways to achieve this goal.

    • iOS

      • You can use an activity indicator placed at the same position as the image. Add an event listener to the image for load so you can determine the remote image has loaded. You then hide the activity indicator and reveal the image.

    • iOS and Android

      • You can use two images, one with the animation, one with the remote image. Add the same event listener of load to hie one when the remote one has loaded.

    Try those.
