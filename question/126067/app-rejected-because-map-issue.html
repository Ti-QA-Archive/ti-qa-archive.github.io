App rejected because Map issue

Hi Friends,

My app has location feature which i used the exact titanium sample. Its working fine for my iphone4,iphone 3 and ipad2. It takes 2 seconds loading time with data connection and instantly loading with wifi. But my app got rejected because the following reason. They said they cant see the map, instead they see a blank white page.
Error message from apple:

"When a location is selected from a list,the app does not display the location. Instead, a blank page"

Is there any one came across a similar situation? May be they dont like the 2 seconds loading time? If so is there any progress loader i can use before loads the map?

Im using Titanium Developer.Should i use Titanium Studio instead?

This is a travel App and i spend 3 -4 months to complete it and really cant loose it.
Any suggestions please?

    Yes, Apple will reject apps if there is too much dead time where there's no activity. Take a look at Ti.UI.ActivityIndicator. That should help you fill that dead time and get you approved.
