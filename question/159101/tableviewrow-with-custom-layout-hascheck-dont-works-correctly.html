TableViewRow (with custom layout) hasCheck don't works correctly

A TableViewRow (with custom layout) hasCheck don't works correctly in iOS7.

The hasCheck property changed (I see in log), but the symbol remains in screen.

custom_tableviewrow.xml

<Alloy>
    <TableViewRow id="row" onClick="select">
        <ImageView id="image" class="brandDefault" />
        <Label id="titleLabel" class="h3"/>
    </TableViewRow>
</Alloy>

custom_tableviewrow.js

var args = arguments[0] || {};
$.row.hasCheck   = false;
$.titleLabel.text = args.get("name");
$.image.image  = args.get("imagePath");
function select(e) {
    if($.row.hasCheck == false) {
        $.row.setHasCheck(true);
    } else {
        $.row.setHasCheck(false);
    }
}

If I try use only the title property in row (removing the layout) it's works fine. :/

SDK version 3.1.3.GA
iOS 7.0.3: Simulaor & Device

  • Hi Eduardo,
    I got the same issue. Setting the hasCheck in the click event listener changes the value but the check mark does not appear. I found it to be some refresh problem because if I scroll after clicking a row (enough to move the clicked row out of sight) and then scroll back to it, the mark is there.
    The only solution I found so far is to call the following just after the hasCheck property is set:

    $.myTable.setData($.myTable.data);
