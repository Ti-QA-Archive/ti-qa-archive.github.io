Ti.UI.PICKER_TYPE_DATE_AND_TIME only shows date part when change listener present

Something weird happening here.

I'm using a date/time picker but somehow its value was ignoring the date part, just providing me the time picked and considering the date was always for today.

It only started working when I added a change event listener to the code. Just this:

cal_picker.addEventListener('change',function(e)
{
     Ti.API.info('new date: '+ e.value.toLocaleString());
});

And everything started working again. The picker's value started to provide time AND date values properly.

This is not a question at all. Just something for someone facing the same issue.
