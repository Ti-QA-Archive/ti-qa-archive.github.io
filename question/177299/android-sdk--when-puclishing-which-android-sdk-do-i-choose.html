Android SDK ~ when puclishing, which Android SDK do i choose?

So in the titanium GUI, I can choose to publish my app.

In the list of choices are things like the key store, the password, the alias - but also the "Android SDK"

In my setup, I have four options:

  1. Google APIs (Android 4.4.2) [armeabi-v7a]
  2. Android 4.4.2 [armeabi-v7a]
  3. Android 4.4.2 [x86]
  4. Android 4.0 [no ABIs.]

I have nfi what these mean.

If I want my app to be run with this configuration:

<uses-sdk android:minSdkVersion="14" android:targetSdkVersion="19"/>

Then which of the four confusing choices do i make? Also, can someone tell me how i find out what the differences are between the choices?

Titanium have an actual tutorial for deploying the app, but they don't bother to address this particular option ~ see http://docs.appcelerator.com/titanium/3.0/#!/guide/Distributing_Android_apps ~ which is a shame, as this is a presumably straightforward setting.

    According to your Specification "<uses-sdk android:minSdkVersion="14" android:targetSdkVersion="19"/>" this means you can choose any one of them. It's better to Choose a lower version of SDK when Publishing.

    Reason is if you choose "Android 4.0 [no ABIs.]" in here , your app will run all the other higher version of android. But your app will not run on any lower android versions then Android 4.0. So choose a lower version. Also check if your app utilities support the version. or else your app will malfunction.
