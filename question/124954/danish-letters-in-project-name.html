Danish letters in project name

Hello community.

I'm having a basic problem in naming an app. If I include the Danish letter æ in the project name, it fails in compilation. Even with a new empty project. When I exclude the æ, it compiles fine. This is for the app name. Is there a way to have the folder called something else than the text that goes on the icon for iPhone?

    Project name must not contain special chars nor white spaces.

    If you are going to use internationalization (the i18n folder), you can assign one name for each language. For this, inside each language folder, create a file called app.xml with this content:

    <?xml version="1.0" encoding="UTF-8"?>
<resources>
    <string name="appname">My title app in my locale language</string>
</resources>

    Appart of this, you can specify a bundle name (the name under the icon) and an app name different to the project name. Edit info.plist file inside your project build folder and copy this file to the root of your project (where tiapp.xml is). This way, the file will be copied automatically after a clean build.
