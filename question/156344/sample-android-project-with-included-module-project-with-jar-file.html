  • Motiur. A jar file is a archived file of java classes and other files, images, etc.The one I have is basically the API for controlling a card reader built by another company. It's the only reason I need a module file. According to the docs, if one needs to access files from a jar file in a titanium android mobile project, it must be contained in a module project. The docs further specify that this .jar file is placed in the /lib folder of the mobile project and a 'classpathentry' must be added to the .classpath file pointing to the .jar file. I understand that marketplace modules may be downloaded and placed, for example, in the root of the project file but that doesn't apply here.
