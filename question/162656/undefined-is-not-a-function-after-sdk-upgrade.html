"undefined is not a function" after sdk upgrade

Hi Guys,
I built an deployed an app a couple of years ago. I'm trying to run it from my new environment and this is what I get on the first screen (last time/last environment I deployed it just worked fine):

Script Error {
[ERROR] :      backtrace = "#0 () at :0";
[ERROR] :      line = 20;
[ERROR] :      message = "'undefined' is not a function (evaluating 'Titanium.UI.iPhone.createNavigationGroup({\n   window: winLogin,\n})')";
[ERROR] :      name = TypeError;
[ERROR] :      sourceId = 309385728;

I've tried cleaning and rebuilding the project to no avail.
winLogin is defined indeed, as so:

// in an include
var winLogin = Titanium.UI.createWindow({
   // backgroundColor:"red",
    title:"Login",
    navBarHidden:true
});


// in the main view
// create nav group
var nav = Titanium.UI.iPhone.createNavigationGroup({
   window: winLogin,
});


// eventually, open app
nav.open(winLogin,{animated:true,navBarHidden:true});
win.add(nav);
win.open();

I'm on mac osx X.8.4
Titanium Studio, build: 3.2.1.201402041146
Any idea?
