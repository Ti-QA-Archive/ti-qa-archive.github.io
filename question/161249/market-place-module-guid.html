Market Place module GUID

I have a question regarding downloading updated modules in the market place and the verification process TiVerify in an App.

This follows on from this thread where I posted a comment http://developer.appcelerator.com/question/160130/unlicensed-modules-detected

I downloaded and used a module (Scandit Bar code reader) a while ago at a certain version (I think around 1.2). Now if I want to use the updated module I go to the Marketplace for My Products and download again (this give the latest version).

The problem with this is that it gives an Unlicensed Module error in a compiled App. This effectively means you cannot use any updates that are published by the module developer (not a good situation) and also means that if you did not keep the original version of the module you downloaded, basically your stuck (unless you create a new Ti account and start again).

So I have found a way around this and thought I would share (btw. I am sure this is not really what should be done but it did work).

As it seems from the above thread the licensing checks the GUID of your App against the GUID of the module you originally obtained from the marketplace (note originally),you need to make sure you have that original GUID from the modules manifest file (if you no longer have the version of the module your originally obtained then you may have a problem).

Use the original GUID and update the manifest file for the updated download and replace it with your original. Compile the app and it should now work OK.

My question is , can this be fixed so that when you get an updated module either:

  1. It keeps the same module GUID that was originally assigned
  2. The new module GUID is added (not replacing) the list of valid GUID's for that App/GUID lookup

It can't replace it as this would mean that any existing apps using the original download would then fail.

  • Marketplace modules are tracked based on your user id (that you log into Studio/DevCenter with) and the module's GUID. If either changes, you'll get the unlicensed-module error. I don't know if or why Scandit would have changed their module's GUID, but perhaps that's the case.

    (Note: the GUID of your app is generated based on your user id. So, the thread you reference saying a comparison is made between your app's GUID and the module's GUID is not wrong, just incomplete.)

    Rather than some hackish workaround, I suggest you contact the Marketplace support team (link in Marketplace). Unless there are licensing restrictions placed by the publisher, I imagine that our support team would fix the problem by issuing you a license for the updated module version.
