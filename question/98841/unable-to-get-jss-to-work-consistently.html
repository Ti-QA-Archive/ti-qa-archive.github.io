Unable to get JSS to work consistently

Hello everyone - I've looked around and haven't found an item on JSS that's helped so starting a new thread.

I am trying to test out JSS to see if its feasible to use within my application - for the life of me i can not get any type of consistent results that would make it useable.

Now my first caveat here is that admittedly, I haven't used JSS in the past, but I am extremely familiar with CSS and the paradigm.

Background Info

  1. iPhone SDK 4.2
  2. Titanium 1.5.1
  3. New Project created through Titanium Developer

Scope

Below is just a very basic example of what I am trying to and I can't get it working.

I have created a global.jss file which i am placing in the Resources Directory alongside app.jss. It is only creating one window style as shown:

// global.jss

#win {
  background-color: "red";
}

Within the app.jss file, I have the following code to create a simple window with a button:

//app.js

var win1 = Titanium.UI.createWindow({  
    id:'win',
    title:'Window1'
});

var button1 = Titanium.UI.createButton({
    title:'I am a Button',
    width:150,
    height: 25
});

win1.add(button1);
win1.open();

Observations

First - i was able to get this to work one time, when I actually had the background color set to #000. Since then, the background has remained #000 no matter what the current assigned value.

Second - I have noticed that even when navigating to the simulator application directory and deleting the stylesheet.plist file from my .app package, it is getting replaced, but the data doesn't seem to ever change (even if I change it and recompile from scratch)

Third - Manually updating the stylesheet.plist file of course doesn't work, because it is getting recopied.

Fourth - I haven't tried pushing this to the device yet.

Conclusion

Has anyone else seen this behavior? Am I doing something completely wrong? Is there a fix I am unaware of?

Right now JSS just doesn't seem to be a valid use case for the iPhone unless the mistake is on my side.

I welcome any and all feedback, even if its to tell me I am way wrong. Thanks in advance!

  • First of all global.jss doesnt work on the iPhone, so dont try using it on the iPhone. To use JSS on the iPhone you will have to have the exact same of the JSS file as the JS file you are working on.
    For example: if you want to handle design for window.js you will have to have window.jss in the same place as the JS file.

    Second. The id proprierty is not working so you have to use classes.

    For example:

    //window.js

    var win1 = Titanium.UI.createWindow({
    className: 'win1',
    title:'Window1'
    });

    var button1 = Titanium.UI.createButton({
    title:'I am a Button',
    width:150,
    height: 25
    });

    win1.add(button1);
    win1.open();

    //window.jss

    .win1{
    background-color:#000000; /or use background instead of background-color/
    }

    This should do the trick.

  • I haven't used the JSS stuff yet. One thing to note is that the style rules use appcelerator object properties not CSS type properties. So, the background color is backgroundColor not background-color. During their recent webinar, they said that JSS is basically a way to expose the properties of the UI elements rather than an implementation of the CSS spec.

    You might also want to check out the tweetanium app they just released. It uses an included styles.js file as an alternative to JSS. For various reasons, I might use that app model instead.

  • In my experience, JSS will sometimes work and sometimes not, and it is very hard to figure out what the cause is or how to fix it when jss is not applied.

    1. Not all files are imported
      My app uses @import statements to import different jss files. Some of these files seem not to be used at all by Titanium, although I can see them being imported when I debug with iOS simulator.

    2. Location within a jss file matters
      Some classes were not applied when I had them near the top of the jss file, but they worked fine once I moved them near the bottom, or vice versa.

    3. The order of defining and adding elements matters
      Sometimes it helps to create a view and add it immediately to its parent before creating and adding other children of that parent, sometimes it helps to create all children first, then add them all to the parent.

    4. Sometimes it's about the jss properties
      In specific cases, setting a top, left, right, or bottom property seems to be necessary for the other class properties to be applied.

    My suspicion is that many problems are caused during parsing of the classes and are caused things like commented lines, unquoted or quoted values, and other stuff that should work fine.
