  • As Matt already mentioned, doing so is against Apple's guidelines. If you try to submit it to the store, Apple would reject it if they caught the fact it did this. If they didn't, it would get removed if/when they were made aware of it.

    So regardless of technical abililty, it begs the question on whether you intend a version to be available in the app store or not. If android only or if only installed via ad-hoc or enterprise profiles, then the app store guidelines may not matter to you.

    There was a product mentioned in the Q&A several months ago which let you do exactly this. But I can't remember the name nor find it with a few search attempts now. I also saw very little mention of it in the Q&A, and I suspect a major reason for that is because Apple would reject any apps found to be doing this.
