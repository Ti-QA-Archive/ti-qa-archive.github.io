Please explain tableview classname

I've read in the API that you should set the classname on each row that has the same layout.

I'm a little confused about this statement. Does layout refer to the row's controls and positions or just controls?

For example, let's say row 0 contains a Label at Left 5 and row 1 contains a Label at Right 10.

Should each row have a unique name because the labels are in different positions?

I realize I may be over analyzing, but just want to be sure I understand.

  • Accepted Answer

    Think on className as a template name.
    Each different layout needs to have a different className, and all the rows that share the same layout to have the same className.
    This way the Os will simply reuse a row and change only it's content, fact that leads to a very good rendering process.

  • I believe these would be separate classes, as I understand them - if the layout changes (including position), it's a different class. Content, of course, can vary with each row. It does make a difference in table view creation speed if the view has a lot of rows that share layout, and they do not have to be consecutive. I'll defer to Appcel to confirm the position assumption.
