How to allow/impersonate 'swipe' to work with WebView?

Hi,
Trying to replicate a common user experience using Titanium and am stuck, looking for advice:

I'd like to use WebView for viewing simple Blog-like articles with inline images, because we can control font formatting and such. Am using WebView.html=&#39;&#39; style to add my own locally crafted content to the view. This works and looks great, and uses the internal scrolling of webview perfectly.

So now we'd like allow the user to Swipe left/right over the screen to change to next/previous article. This is common ux in many reader apps, of course.

HOWEVER, as documented in the WebView docs (sneaked into the fine print, I'd add), and as Doug Handy points out in this answer (developer.appcelerator.com/question/122502/adding-click–swipe-event-listeners-to-a-webview), WebView looses it's internal scrolling behavior as soon as you add a Titanium 'swipe' event listener to it. Bummerdeal, dead in the water.

Q: What would people suggest doing to accomplish this? It doesn't seem like we have another html capable view or control available in Ti, and the regular text/labels don't have enough formatting control or inline images/etc.

New to Titanium so feel free to slap me around with alternate approach. ;)
THANK YOU!

  • There are special Parameters in HTML header to control the behaviour.

    <meta name="apple-mobile-web-app-capable" content="yes" />
    <meta name="apple-mobile-web-app-status-bar-style" content="black" />
    <meta name="viewport" content="width=device-width, minimum-scale=1, maximum-scale=1">

    Hope it helps.
