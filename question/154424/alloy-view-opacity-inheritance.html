Alloy View Opacity inheritance

TiStudio: 3.1.1.201306112235, Alloy: 2.2.0.201306112154,SDK: 3.1.0.GA (IOS App)

I have a query about the opacity of child views in Alloy. My example is

<View id="parentView" opacity="0.5">
    <View id="childView" opacity="1.0"/>
</View>

I am assuming that the parentview should define the opacity of the child view (i.e. the child inherits the value), but surely if I specify the childview opacity it should be set correctly (i.e. in the code above the child view should be opaque)?

Currently this is not the case, it seems the child view inherits but cannot override a value?

Is this a bug or corrrect behaviour?

Thanks,

Steve.

    No, the behavior is as expected. Child elements cannot decrease the opacity applied to their parents.
