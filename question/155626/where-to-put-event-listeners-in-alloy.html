Where to put event listeners in Alloy

Just a general best practice question in Alloy:

I am looking through the ToDo sample and I notice that there a many event listeners that are inline amid the XML markup. I seem to have been trained on the web that listeners should be placed in the javascript, but I see "on__" events used here for example. Is this a recommended practice, or would it be preferable to add IDs to the buttons and listen for each of the elements' events via add.js (and why or why not)?

    It really is a matter of personal preference. Whatever you do, be consistent or you'll end up looking for events or property settings all over the place.

    Although you can set pretty much all properties in your views, I personally find they're easier to track if kept in the controllers and tss files

    • I started moving the code where I set my listeners to the controller, for the reason that there seem to be differences between platforms ( such as when the change event fires for text fields seems to differ between iOS and Android ). In such cases I can better control the flow when I use the controller to set the event listener as opposed to the XML markup.
