ListView: Hold the deleteView active until being clicked

Hey there,

i'm currently facing a problem when deleting items on iOS with "swipe to delete". Once i swipe, the red deleteView is shown. In my case, i want to click the "deleteView", display an alertDialog and if the user clicks OK, the row needs to be deleted.

Perfect example: Exiting WhatsApp groups from the chat-list.

Currently in Titanium, once i click the deleteView, the delete event is fired and the row is deleted automaticly. So my question: a) Is there a way to manage the whatsapp-like way? b) If not, can i open a jira ticket for that?

Thank you!

    Hi

    Short answer - no.

    Longer anger answer;

    From a user experience perspective, the aim is to make sure that any destructive actions are not performed accidentally. By swiping the row and then pressing the button the expectation is that those two required staged actions make sure that the delete is not accidental. This is also balanced with allowing the user the opportunity to perform the action quickly if the choose to perform it on many rows one after the other.

    This is the idea of balance - enough protection from accidental usage combined with only enough steps so it does not become a pain for the user.

    As to "could you create a work around" - technically yes. But you will have to put the work in as the automatic delete option will have to go and you will be adding your own (chances are how whatsapp have achieved it).

    • You remove the delete option from rows
    • you add a delete red view with a label inside and anchor it to the right of the row
    • create an internal row wrapper and fix the width to the device width NOT Ti.UI.FILL
    • add an event to that view for sliding action via touch start and touch end or swipe
    • on the event you move the wrapper to the left so part of it is not seen and it then reveals the red view
    • add you confirm action here

    HOWEVER you will find it hard to place your red view flush right, so it may not look identical to the ios7 version.

    Should you create a jira? That one is up to you, no reason why not - there might be a hook we are currently not exposed to.
