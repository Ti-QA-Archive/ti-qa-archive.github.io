Keyboard Resets on Key Press (Alloy)

I have a textfield. The keyboard comes up fine, but if the users switches to the alpha/punc and presses a key then it switches back to default. This is very annoying. How can I make sure the keyboard stays in the last user state while on the same field?

  • Found the issue…looks like someone else had the problem and found this to be the solution..

    The issue appears to be that with every character you type, when it repaints the textField, it also refocuses it as well, causing it to revert to the initial alphabet mode. In order to stop this from refocusing, you need to fully define both the width AND the height of the textField. This stops it from 'resizing' all the time(even when it's not) which stops it from having to repaint the entire component and thus stops it from refocusing.
