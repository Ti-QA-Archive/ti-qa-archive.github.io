  • Hi,

    OTHER_LDFLAGS =$(inherited) -F"PATH TO YOUR FRAMEWORK" -framework myFramework

    PATH TO YOUR FRAMEWORK will be your framework folder path, where your myFramework.framework is residing.
    example suppose my myFramework.framework is in abc in root directory directory then

    "PATH TO YOUR FRAMEWORK" will be "/abc"

    in this case your module will be having reference of the framework and library of it.

    • Hi, ok so for your conceptual understanding, when you add library directly then libraries are combined into one product library and when you link as a framework then it just take a reference.

      linking like framework is done for Apple provided framework as they are available in the systems, but for custom 3r party framework if you do like framework liking then you have to ship the framework along with your module and tell the developer to place them in the path from where you are linking.
      That is not a developer would love to do, so better you add them directly.

      but again its module developer choice, i say both are ways to do the same task. I prefer which reduces other effort once i am going to don something.
