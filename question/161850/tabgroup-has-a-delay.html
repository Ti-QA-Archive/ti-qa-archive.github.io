TabGroup has a delay

I created a new project using the default two tab alloy project template… I added a TableView /w TableViewRow. In the controller (index.js) to listen for click events and open a new detail window using the Alloy.createController. I'm seeing a small delay in the details window and a label during the transition.

Here is an example that reproduces the issue…

https://github.com/awamser/TwoTab.git

    Any time view rendering requires compositing, there's going to be a delay. The default for Window background is transparent. That means that nothing can be rendered to the screen until the compositing engine figures out what pixels are "shining through" the transparent background. Add some more transparent elements within the window and there's further compositing to see what shines through them.

    • If possible, avoid background gradients because they require extra compositing passes
    • You should always set background colors to match the background color of the parent view unless you really need an element to be transparent. You can use a global style in the app.tss file to set app-wide background colors then override as needed in the controller-specific TSS files.
    • On OS X with the iOS Simulator, you can display which layers are being composited by choosing Debug, Show Blended Layers. Red shows layers that are being blended/composited

  • By adding the backgroundColor="white" to the details.xml (view) the issue is resolved!

    <Alloy>
    <Window backgroundColor="white">
        <Label text="Details" />
    </Window>
</Alloy>
