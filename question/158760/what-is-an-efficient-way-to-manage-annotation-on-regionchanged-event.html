What is an efficient way to manage annotation on regionchanged event

Hi,

I am new to this whole map/annotation programming. Just wondering if someone can share some idea on how to manage and avoid duplicated annotation (backend/frontend) when the regionchanged event is triggered.

I am getting records from db by supplying coordinates and the given radius ie within 10 miles. Now the trouble I am running into is that when there is no significant region changed (region overlap), this resulted in retrieving the duplicated records for the overlapped region

What approach should I take to make this as efficient as possible?
What kinds of changes do I need to change both front and back end to make this work?

Any suggestion is appreciated!

  • When ever there is a regionchanged before you create the map view with the annotations remove the existing annotations using removeAnnotations method.

    Also, I don't think removeAnnotations works as expected, there is some bug in that method(give it a try). Instead use removeAnnotation by looping through all the Annotations that you created previously.

  • Another solution which would give a more fluid user experience would be to just maintain an array of the latitude/longitudes that have been plotted and build two arrays (de-duplicating as you go) from that of the new ones to add and those that no longer need to be plotted. Then just go through each array adding the new and removing the old.
