Highlight some labels when user "mouseover" it

Hi.

I'm having some troubles to detect and change color of labels I have.

I'm trying to do a wordsearch game and I thought on do a board with labels, for example, 5x5 labels. I want to "highlight" the words when the user touches and swipe the finger over the letters.

But I'm only getting the first label the user touches, I'm not able to get the other labels to change the color.

This is my current try:

... //Print board
    for (var row=0; row<settings.gridSize; row++) {
        for (var col=0; col<settings.gridSize; col++) {
            var letter = Ti.UI.createLabel({
                row: row,
                col: col,
                top: (row * letterWidth),
                left: (col * letterWidth),
                height: letterWidth,
                width: letterWidth,
                textAlign: 'center',
                text: matrix[row][col].letter
            });
            letter.addEventListener('touchstart', onMousedown);
            letter.addEventListener('touchmove', onMouseover);
            letter.addEventListener('touchend', onMouseup);

            $.boardView.add(letter);
        }
    }

// User interaction
function onMousedown(e) {
    Ti.API.info("Mousedown");
}

function onMouseover(e) {
    Ti.API.info('Mouseover: {x:' + e.x + ', y:' + e.y + '}');
    e.source.backgroundColor = 'white';
}

function onMouseup() {
    Ti.API.info("Mouseup");
}

With that code, only the first touched letter detects the events and goes white. When the finger changes from the first label to the adjacent label, no event is triggered. I also tested with "click", "swipe", … same results.

How can I get every label I pass over?

    Hi Manuel,

    See… you can not directly fetch touchMove event of other controller. touchMove event fire only those object on which you start touch.. for example there is three labels according to your problem. label1, label2 and label3.
    now you touch start from label1 and move to label2 direction that time touchMove event is fire for label1 only even you move over label1 or label2 or label3 because you start from label1.

    even if you want to check then check you log x and y point is also according to your label1. if you move on left side of label1 it display minus(-) points..

    but for that i have one solution if you want to get touch Moved coordinate according to parent view.. then after you need to do something manually find label on particular position.

    for find parent coordinate using convertPointToView method.

    i share some code with you. so you can test it

    var view = Ti.UI.createView({
    width : "100%",
    height : 300,
    backgroundColor : "gray"
});
$.winIndex.add(view);
var lable = Ti.UI.createLabel({
    text : "Test",
    width : Ti.UI.SIZE,
    height : Ti.UI.SIZE,
    top : 50,
    backgroundColor : "red"
});
view.add(lable);

lable.addEventListener("touchmove", function(e) {
    Ti.API.info('Original = ' + JSON.stringify(e));
    var point = e.source.convertPointToView({
        x : e.x,
        y : e.y
    }, view);
    Ti.API.info('CONVERTED POINTS = ' + point.x + " " + point.y);
});

    you can check on log both points original and converted…

    Then after you need to do something for identifying view at converted point…
