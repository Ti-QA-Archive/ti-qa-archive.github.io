Android Memory Leaks SDK 1.8.1 with V8 and tableview

I am developing an Android Application using SDK 1.8.1 with V8.

Testing on Samsung Galaxy ACE (android 2.2.3)

I've noticed some serious memory leaks.

Having read the following Wiki:
Managing Memory and Finding Leaks
, with DDMS I've isolated the part affected by the memory leaks.

I have a table, contents of which keep being cancelled and reloaded again.

My code is similar to the example in KitchenSink (Base
UI-Views-Table Views-Refresh Table Views file:table_view_refresh.js)

I've tried to debug this function also in KitchenSink and seems that the
line (referring to the KitchenSink example)

……

tv.setData([]);

….

doesn't release the memory. In fact, with DDMS I can see that allocated
memory continues to grow until it crashes the app.

I don't have this problem with SDK 1.8.1 and Rhino (which I'd prefer not
to use because of poor performance)

Are you aware of this problem? Do you have any suggestions on how to fix
this?

Thank you in advance!
