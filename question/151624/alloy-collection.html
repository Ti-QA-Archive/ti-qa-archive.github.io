  • Take a look at the "sql_queries" test app in the github repo: https://github.com/appcelerator/alloy/tree/master/test/apps/models/sql_queries

    In this app I use the model Id of a particular row in a data bound table to determine how I will render the next screen of content. Essentially it's a data-driven master/detail sample. You should be able to determine how to translate that into your own similar situation.
