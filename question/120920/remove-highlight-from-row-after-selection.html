Remove highlight from row after selection

I'm having some trouble with the following table view. By default the rows turn blue when selected. In my app, when the user selects a row another table is opened in the view. When they return to the original table via the back button, the row still has a highlight. I'd like to remove it. Also, if the user selects another row, sometimes the highlight is removed from the original row and sometimes both are highlighted.
Is there an easy way to have a row highlight to respond to a touch and then be immediately deselected? Or at least a way to get rid of the default highlight?
Thanks!

var TableView = Titanium.UI.createTableView({
    style:Titanium.UI.iPhone.TableViewStyle.PLAIN,
    separatorStyle:Ti.UI.iPhone.TableViewSeparatorStyle.NONE,
    width:304,
    borderRadius:0,
    backgroundColor:'transparent',
    rowBackgroundColor:'transparent',
    borderWidth:0,
    allowsSelection:true,
    top:105,
    zIndex:0,
    data:memberRowData
    });
    View.add(TableView);
row = Titanium.UI.createTableViewRow({
        height:60,
        backgroundImage: '../images/groups_popup_bg.png',
         hasChild:true
});

  • You can set the TableViewCellSelectionStyle:

    row = Titanium.UI.createTableViewRow({
        height:60,
        backgroundImage: '../images/groups_popup_bg.png',
        hasChild:true,
    selectionStyle: Titanium.UI.iPhone.TableViewCellSelectionStyle.NONE,
});

    see documentation: http://docs.appcelerator.com/titanium/latest/#!/api/Titanium.UI.iPhone.TableViewCellSelectionStyle
