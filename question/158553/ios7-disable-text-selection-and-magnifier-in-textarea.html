iOS7 Disable text selection (and magnifier) in TextArea....

Hi,

I'm using several TextAreas to just display text (no editing) and make use of the autoLink feature. But I need to make the text in the TextArea not selectable (no "Copy | Define" popover, and no magnifying glass).

This old Q&A suggests that setting enabled:false and editable:false should accomplish this, but it does not seem to work with SDK 3.1.3GA.

Here's the code:

index.js

<Alloy>
    <Window backgroundColor="#fff">
        <TextArea 
            enabled="false" 
            editable="false"
            autoLink="Ti.UI.AUTOLINK_ALL"
            value="This text should not be selectable. www.apple.com">
        </TextArea>
    </Window>
</Alloy>

Am I missing something? … I can't disable touchEnabled because that prevents the links in the text from working.

  • @Edward M is there any reason why you are using textArea? Generally the reason for using a text area is so a user can edit the content and type in it. Could you use a regular label? Like so: 

    var appleLink = Ti.UI.createLabel({
    color: 'white',
    left : "24%", top : "10%", right : 5, height : "14%",
    font:{fontFamily:'Comic sans', fontColor: 'black', fontSize: '17sp'},    
    text : 'This text should not be selectable. www.apple.com'
});

var appleURL='https://www.apple.com';

appleLink.addEventListener('click', function(e) {
    Titanium.Platform.openURL(appleURL);
});
