HTTPClient method setRequestHeader sometimes doesn't work

Hello,

My question: when I create a HTTPClient, open it and then use the method setRequestHeader some headers are being set correctly and some are not. I use fiddler to monitor the request that is done:

Please look at this example code:

var url = "https://raw.github.com/appcelerator/Documentation-Examples/master/HTTPClient/data/json.txt";
var xhr = Ti.Network.createHTTPClient();
xhr.setEnableKeepAlive(true);
xhr.open("GET", url);
xhr.setRequestHeader('Accept-Encoding','gzip, deflate');
xhr.setRequestHeader("Accept","text/html, application/xhtml+xml, */*");
xhr.setRequestHeader("Accept-Language","nl-NL");
xhr.setRequestHeader("Connection","Keep-Alive");
xhr.setRequestHeader("User-Agent","test");
xhr.send();

Actually I have two questions:

  1. In fiddler (a http sniffer) I can see the headers for Accept-Language and User-Agent are being set.
    The Accept-Encoding header is still gzip (and not gzip, deflate what I expect) and the Connection header is still close (and not Keep-Alive what I expect). What is the cause of this?

  2. For the webservice I have to call (not the same as the example) I need a persistent connection. To accomplish this I use the method setEnableKeepAlive(true) and I set the Connection header to Keep-Alive. This is not working. How can I force the request to set Keep-Alive for the connection?

I saw some simular questions in the Q&A but I couldn't find a proper answer so far. Can someone help me?

  • I think there might be a couple of issues…

    Firstly set the keep alive, within the creation of the xhr object. Found a bug in titanium a few weeks ago where trying to set certain properties just doesn't work unless they are set at creation time. This may be one of them.

    Secondly define the headers before the open.
