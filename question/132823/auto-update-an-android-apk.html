Auto-update an Android APK

After a small research , we found out that our target users don't update applications. Our application updates are very important , because we are working with an app that requires new things added.

So the android market updates aren't forced.. So , is there any way to download an APK from our server and install it in the background?

SDK: 1.7
OS: Windows
Target Platform: Android 2.1+
Device: HTC Desire / Samsung Galaxy S2 (testing devices)

  • You could host a webservice that returns the current version of the app, wire up open and resume events to check the webservice and prompt the user. You can also look into Krooshal. It is a single file drop-in SDK for Titanium, which you can configure to check for updates. Krooshal DOES NOT bypass your distribution channel (ad-hoc or store). It will prompt the user about the update and will take them right into the store.
