  • I'm not aware of a way to automatically do this. You could probably manage it yourself by tying into all your controls' event listeners and updating a timestamp. Tie that into a timeout, and you could automatically trigger the login window. But this could be very tricky to capture every way in which the user could interact with your app. For example, does scrolling around a table view but not clicking in it count? You'd have to capture a LOT of events.

    If your app is client/server, you could keep track of the last time you ran a network operation. Each time you prepare to run a network operation, check the elapsed time since the last operation. If it's been too long, bring up the login screen.
