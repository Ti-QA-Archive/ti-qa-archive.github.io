App crashes low memory and very slow with scroll view background image

Hi experts,
I'm trying to build a view like the iBooks app.

So I combined a scrollable view with several scrollviews.
In order to have a background I first tried to set a background to the scrollable view but this didn't work because then the background is fixed.
So I put a background image to each of my scrollviews (with repeat = true). That made the app horrible slow.

The next idea was to put normal views in the background, each with one tile of the background image.
This gave a reasonable speed for the app.
But now if I open one of the associated pdf files within a web view, the app crashes sometimes because of low memory.

The background image is not very big, just 120k of jpeg-image.
Without any image the app is very fast and doesn't crash.

What am I doing wrong?
Why does the app get so slow if I use a tiled background?

I tried with SDK 2.1.4 and SDK 3.0.0 but that makes no difference.

TIA Stephan
