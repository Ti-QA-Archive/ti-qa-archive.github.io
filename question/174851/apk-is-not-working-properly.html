  • Hi

    I have a wild, out of the box thought - it is wild and out of the box purely because there is insufficent information provided to give any real informed answers.

    • Are the sections of code that process these specific things inside CommonJS or other related files external to the current calling file?
    • Does the filename of the requested ComnonJS file (inside require) match that of the file name and path on the server. Including upper and lower case matches?

    If you are calling something 'like' this;

    var x = require('Folder/File');

    But the physical file is located as follows;

    // alloy example
{appname}/app/lib/folder/file.js
// non-alloy example
{appname}/resources/folder/file.js

    The require will not find this file, the upper and lower case do not match even thought all the letters are the same. The operation of this rule differs from emulators and simulators to those on a physical device.
