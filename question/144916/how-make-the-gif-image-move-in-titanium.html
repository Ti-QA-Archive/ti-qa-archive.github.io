How make the GIF image move in Titanium

Hello, there

I am developing a communication tool that sends emoticons. I use GIF images for the emoticons, but when I add the GIF to the imageView, it doesn't have actions or move. Do you have any idea how to solve this issue?

  • Hi

    Not supported I am afraid, you can achieve a similar effect using the imageView and having several image frames display in a sequence automatically.

    This will however require exporting the GIF files into individual images, possible in applications like Photoshop and fireworks. This will also only reproduce the sequence effect and not any custom timings or actions within the files, although hopefully with emoticons this problem will be limited.

    Obviously the webView will show them, but I would not expect this to be a suitable solution at all given your likely usage.

  • You can use the Ti.UI.createImageView({ images: [image1, image2, image3]})

    From Doc:

    images : String[]/Titanium.Blob[]/Titanium.Filesystem.File[]
    Array of images to animate, defined using local filesystem paths, File objects, remote URLs (Android only), or Blob objects containing image data. Blob and File objects are not supported on Mobile Web.
