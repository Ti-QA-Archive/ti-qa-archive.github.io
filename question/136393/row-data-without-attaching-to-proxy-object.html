Row data (without attaching to proxy object)

Just out of curiosity, what kind of solution do you use when the user selects a row from a TableView and you need to respond based on the selected row.
At first I always just set a property on the TableViewRow which I could then retrieve from the event.row object, but I was told that it's not a good idea to attach custom properties to the proxy objects.

    • ok, then you can do something like that….

      for(var i = 0; i < 10; i++) {
    var star = Ti.UI.createImageView({
        height : 24,
        width : 24,
        left : 5 + (i * 25),
        backgroundColor : 'white'
    });

    (function() {
        //we need to capture the value of "i"
        //for use in click handler functions
        //on our image views.  To do this, we
        //use a closure (this self-calling function)
        var index = i;

        star.addEventListener('click', function() {
            alert(index);
        });

    })();

    win.add(star);

    // stars.push(star);

}
