    • Yes it is on the page;

      • Writes to app/assets instead of Resources.

      If you know what app/assets means then you are using Titanium with the Alloy framework. It appears you do not know what it means then you would uncheck this option.

      Titanium with the Alloy framework has many new folders and does not use the resources folder directly. Instead imagery and other files are places inside this folder only.

      You are clearly not using Titanium with the Alloy framework for your app, so you are using what is now called "Titanium without the Alloy framework". This places all source files inside the resources folder. So when you answer that question.

      • Writes to app/assets instead of Resources.

      You say NO, this way it using the resources folder.

      BTW using or not using the Alloy framework is a CHOICE and one you make at the start of a project, do not think anything here is meant to say you have done anything wrong - you haven't. In case you are wondering "should I use that Alloy framework" - then the answer is Yes, but on your next app - NOT this one. There is no benefit to making that change now when you are about to push the app into the store. You may decide later benefit from the framework in this app - but expect a lot of work first. The Alloy framework saves you lots and lots of time - but only if you start a project with it.

    • appicon-60.png (60x60)
    • appicon-60@2x.png (120x120)
    • appicon-60@3x.png (180x180)
    • appicon-72.png (72x72)
    • appicon-72@2x.png (144x144)
    • appicon-76.png (76x76)
