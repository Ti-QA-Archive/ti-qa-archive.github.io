  • It might be a bug. I would try a couple of things first.

    Try setting the scrollview's contentHeight to Ti.UI.SIZE or omitting that property. 'auto' is deprecated and might cause unusual behavior in current SDKs. I'd also try using the 3.1.1GA version of the SDK rather than the nightly build version. It might be introducing regressions. You don't have a left value set for your scrollview. Maybe it's being drawn off screen on rotation?

    Also, you're relying on calculated values for the width & height of the views inside your scroll view. You might need to add an orientationchange event listener to reapply (and thus recalculate) those automatic heights/widths when the device is rotated.

    If you do decide to file a bug, please try to be as specific as possible about what causes and works around the issue to help our support team triage the bug and our engineers to fix it.
