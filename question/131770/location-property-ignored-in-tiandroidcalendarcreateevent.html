Location property ignored in Ti.Android.Calendar.createEvent

If you look at the example in the API docs,

// Create the event
var eventBegins = new Date(2010, 11, 26, 12, 0, 0);
var eventEnds = new Date(2010, 11, 26, 14, 0, 0);
var details = {
    title: 'Do some stuff',
    description: "I'm going to do some stuff at this time.",
    begin: eventBegins,
    end: eventEnds
};

var event = calendar.createEvent(details);

It is curious that there is no line

location : "4 Parkfield Crescent, Fairview, CT"

and indeed if you add "location" property to details, nothing is done and
both event.location and event.getLocation() return null.

I suppose you could describe this technique as "implied documentation" for lack of better terms.
