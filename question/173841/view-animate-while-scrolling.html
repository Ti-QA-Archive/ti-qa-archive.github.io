    Hi

    There are a few animations going on there so having to guess which one - assuming it is the parallax one that appears first.

    The user scrolls the scrollView upwards and the header image at the top moves at a different speed to the scrollView itself. Although the second animation is similar as it does not animate the top header image but does animate a top button until it locks in place.

    The logical thought here would be that the header image and the lockable button are NOT linked to the scrollView and are in fact at the same parent level as the scrollView. When the scrollVIew scrolls you listen for the 'scroll' event and then update the position of the header view and button to show those effects.

    • Parallax

      Simply shows two or more views that move an different rates to give the effect the top most moves over the next one down faster than the top most one.

    • Locked button

      This is moved at the same speed (y positions match when scrolling) until the scrollView reaches a certain value (would be a minus number - 100 to 150 on that video) when it reaches that number or lower then the button stops moving and is locked to the top position, when scrolled down again if higher than that number simply anchor back as it was before.

    The scrollView would likely have a layout property set to vertical to allow the other child elements display. The very first child object would have a top (margin) equal to the image header header in the first example and equal to both image header and button height in the second example.
