Splash screen present for 20 seconds then app quits

  • App type: mobile
  • Titanium SDK: 3.0.1.201212181159
  • Platform: iOS 5.1.1
  • Device: iPhone 4S (Jailbroken)
  • Host OS: Mac OSX Mountain Lion

I'm working on an app that loads information from a server asynchronously in app.js. I'm using a jailbroken 5.1.1 device to test my applications (build then drag and drop to iTunes to deploy). In some cases, the app is built, and I'll run it on the device and I get an "app.js not found" error, which is weird - but in most cases after reinstalling the app, it just opens and sits at the splash screen for around 20 seconds, then quits. What is really strange is that the app works perfectly in the simulator I ran an analysis on the app to see if it was burning through memory, but it looks like it peaks around 3MB, so that can't be it. I'm not sure providing code will be of too much use, but I'm wondering if anyone else has been in a similar situation, and what you did to fix it. I'll keep looking myself, thanks for any information you may be able to provide.

  • When trying to track down similar issues, here is the approach I've taken:

    1. Insert copious Ti.API.info calls into your code. Since you think you are getting hung up running the code in app.js, go through your code and insert info calls with specific information to let you know where your code is executing/failing.
    2. Launch xCode, run the app on your phone and use the Organizer view in xCode to view the logs for the device. Then you can see the output from your Ti.API.info calls and figure out where the code is failing… fix the code… repeat.

    Ray
