Global scope

Should I be worried about polluting the global scope, when defining variables outside of functions in .js pages that are not 'alloy.js'?

e.g.

var win = this.getView();

function clickListener(e)
{
win.setTitle("…");
}

functionClickListener2(e)
{
win.close();
}

    commonJS modules have their own scope, so when using require, the scope of a variable outside of a function in the required module, unless exported is global but private to that module only.

    Hope this makes sense.
