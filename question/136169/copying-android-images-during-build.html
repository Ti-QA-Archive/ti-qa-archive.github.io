Copying Android Images During Build?

I have a situation where I have a few images located in Resources&#x2F;android&#x2F;images&#x2F;res-long-port-hdpi&#x2F;tabs. These include tab icons themselves as well as images I'm using to create a custom "navbar". When building for the emulator, the console reports the following warnings:

[WARNING] [2012-04-27 16:33:45,929] get images/tabs/home.png: path not found
[WARNING] [2012-04-27 16:33:46,047] get images/tabs/navbar_home.png: path not found
[WARNING] [2012-04-27 16:33:46,074] get images/tabs/navbar_home.png: path not found
[WARNING] [2012-04-27 16:33:46,092] get images/tabs/navbar_home.png: path not found
[WARNING] [2012-04-27 16:33:46,107] get images/tabs/navbar_home.png: path not found
[WARNING] [2012-04-27 16:33:46,499] get images/tabs/locations.png: path not found
[WARNING] [2012-04-27 16:33:46,549] get images/tabs/search.png: path not found
[WARNING] [2012-04-27 16:33:46,598] get images/tabs/video_library.png: path not found

My confusion is that the tab icons (home.png, locations.png, search.png and video_library.png) are displayed even though they couldn't be found. The navbar_home.png image is not displayed though it does exist in the same directory.

Any thoughts? I'm not sure what to do with the seemingly contrary information being reported.

Thanks.

  • When you reference an image on android, do you preface the path with a &#x2F;?

    Resources&#x2F;android&#x2F;images&#x2F;* would be image: &#39;&#x2F;images&#x2F;myimage.png&#39;
