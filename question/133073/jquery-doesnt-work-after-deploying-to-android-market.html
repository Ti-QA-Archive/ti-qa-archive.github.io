jQuery doesnt work after deploying to Android market

In my html file I have the following:

<html>
<head>
    <script language="JavaScript" type="text/javascript" src="jquery.js"></script>
</head>
<body>
$(document).ready(function(){
    alert("doument ready");
});
</body>
</html>

If I RUN this in the emulator or RUN it via a device, it works fine. However, if I DEPLOY it (then install the .apk file that gets created on my device) its doesn't work.

jquery.js isnt the minified version, its the full source code.

This is only a problem for Android Market.

Has anyone come across this before, and/or knows of a solution?

  • Have you tried prefixing your resource url with app:&#x2F;&#x2F;? This prefix maps to the root of your Resource directory.

    Like so:

    <script language="JavaScript" type="text/javascript" src="app://jquery.js"></script>

    Provided your jquery.js file is at the root of your Resource directory.

    • I forgot to mention something in my response. By default, Titanium processes all JavaScript in the resources directory on Android to optimize for native apps. For a hybrid app today, you must prevent this behavior to keep your application HTML’s directory paths intact. To accomplish this, add the following line to your tiapp.xml:

      <property name="ti.android.compilejs" type="bool">false</property>

  • The problem is that Titanium is removing all .js-files on distribution, and includes the JavaScript in the "real" Java-code instead. Titanium will think that your jquery.js is Titanium-code, so it will be removed as well.

    I simple solution would be to rename your JS-file to have another extension instead, like jquery.jsx
