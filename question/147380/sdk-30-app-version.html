SDK 3.0 app version

Hi guys,

I've noticed that SDK 3.0 adds a 13 character number to the app version on iOS. In app.xml the app version is: 1.1.2 while inside the app it's shown like: 1.1.2.1359096820446
How can I make sure that in the distribution ready app the version will be displayed correctly?

Thanks,

— asked January 25th 2013 by Lucian Pacurar

  • Accepted Answer

    We only add the timestamp to the end of the version number for device builds to trick iTunes into thinking that it's a newer version of your app, otherwise iTunes wouldn't initiate the sync properly.
