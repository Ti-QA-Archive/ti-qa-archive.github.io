Opening a new controller, UIModule error

Hello everyone,

I'm new to titanium and I'm starting an Alloy project using the New Project creator.
For information (I'm on MacOS10.8 on a MBAir 2013, with Titanium Studio 3.1.3.GA fresh installed.

I'm able to load the "Two Tab Project" without any problem in the iOS Simulator.
Then I would like to create a new page and navigate to that one.

So I add a controller using menu "New > Alloy controller".
That controller will be named "accueil".

In 'index.js" I add the following code to open the controller as said in the tutorial.
function onWinLabelClick(){
var accueil = Alloy.createController('accueil').getView();
accueil.open();
}

As far it's ok, I can lauch the simulator and go to my accueil controller by clicking on a label.

But then I would like to modify accueil.xml and in my view I add:
<label>TEST</label>

once I did that I'm stuck getting the error:
"invalid method (createLabel) passes to UIModule at accueil.js"

I tried for several hour testing all solution available on google, stackoverflow etc…
like cleaning the project, rebuild all etc…

Even using that hack: https://gist.github.com/tzmartin/1372475
that should preload all my needed module.

If someone could help me on that it would be very appreciated.
I'm all new with titanium and being stuck on an 'hello world' for hours drive me nuts.

Regards,
Antoine

    Is the xml tag for Ti.UI.Label capitalized in your code as in this snippet?

    <Label> test </Label>

    If I remember well from some previous experience, you could get problem if not. Hth.
