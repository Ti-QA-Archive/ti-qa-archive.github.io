1.6.0 Text being cut off on TableViewRow with Label

Not sure where to log this issue,

In version Mobile sdk 1.4.2 for iphone when I load dynamic text into a label which is placed in a tableviewrow it looks great and resizes the rows accordingly.

In every version of Mobile sdk after 1.4.2 even up to now 1.6.0 depending on the amount of characters it cuts off and places a (…) at the end of the text. Resulting in lost wording.

Example 1.4.2 ( good)
What is the process known as when water-soluble products are introduced into the skin by using electric current?

Example 1.6.0 ( bad)
What is the process known as when water-soluble products are…

When I load the same code on iPad using 1.6.0 the issue does not exist, I am assuming because of extra spacing…

Code looks good and i have no other issues, the text cut off issue shows itself when building with any version after 1.4.2

If anyone needs anymore information let me know and I can get screenshots and code.

Thanks in advanced!

  • Try adding a "minimumFontSize" property to your Label. I don't know exactly when that option was added in, but if the label doesn't fit into the height & width of it's holder it's fontSize will scale accordingly until it hits this value, at which it will stop. So if your regular fontSize is 18, I would suggest something between 14 and 15. Such as:

    var label = Ti.UI.createLabel({
text:"BLAH BLAH TEXT GOES HERE.",
font:{fontSize:18},
height:'auto',
width:'auto',
minimumFontSize:14
});
