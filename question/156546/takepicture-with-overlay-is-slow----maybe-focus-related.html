takePicture with overlay is slow -- maybe focus related?

I've got a camera app where I want to allow the user to snap off a bunch of photos in a row. Currently testing on iOS. It has a custom overlay, autohide: false, etc. When you hit the Take Picture button instead of snapping the photo the app instead seems to start focusing. This delays the picture up to 5 seconds.

Any suggestions on speeding up the picture taking? Is there a way to take multiple pictures in a row with the standard UI (instead of a custom overlay?).

    • Anybody looking for an answer - I reasked this again in a different way recently and finally figured it out.

      Then have a timer running in the background that constantly checks in memory for photos to process.
