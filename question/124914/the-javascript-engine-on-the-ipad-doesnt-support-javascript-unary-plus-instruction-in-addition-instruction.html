The Javascript engine on the iPad doesn't support Javascript unary plus instruction in addition instruction.

In the iPad Simulator, the following instructions ALL compile and execute, producing the answers shown in the comments


var v1, v2, ans;

v1 = 1;
v2 = 1;

ans = v1  +  v2;                         //  = 2

v1 = '1';
v2 = '1';

ans = v1  +  v2;                         //  = 11
ans = parseInt(v1) + parseInt(v2);         //  = 2
ans = Number(v1) +Number(v2);         //  = 2
ans = +v1  +  +v2;                     //  = 2

On the iPad itself (iPad 2, software version 4.3.5 build 8L1), the final line "ans = +v1 + +v2;" causes a red screen 'Parse Error' and claims the error at a line number that's different from the true instruction line.

  • I'm not sure if that is correct or not.

    In ECMAScript, tokens are generated as longest possible sequence of characters from left to right. When we apply that rule on the last line:

    ans = +v1  +  +v2;

    tokens would be:

    (ans) (=) (+v1) (++v2)

    because "++v2" is valid token. "+" is also valid token, "+v2" is also valid token, but "++v2" is the longest valid token.

    Now the problem is that I tested that line on multiple environments and it passes with no exception thrown, so I'm not sure what implementation got that right. I hope someone with greater knowledge of ECMAScript will bring some light. :)

    • I checked this, I was wrong about my conclusion about token generation. The whitespace makes the difference, so that statement is valid ECMAScript statement and the tokens would be:
