TableView and low memory exceptions

I have this window that displays a 'Back' button and a TableView. This TableView displays data that I retrieved from a remote server. This TableView is set to display 30 rows at most plus a little more if a complete data set is not displayed. Each row either consists of a single label or 2 label labels. There is also a button on the last row to view more data which replaces the rows the user is viewing with a new set of rows which are next in line.

My problem is that sometimes when scrolling fast it eats up a TON of memory and my app inevitably crashes; even if I close all of my windows back to the login page. I have a global listener that alerts me when my app is using more than 12mb and it only ever goes off on that TableView.

This is only an issue with Android. My iPhone version displays 100+ rows of the same sort at once and never crashes. I have read that Titanium is worse with Android garbage collection. Anyone have any experience with this?
