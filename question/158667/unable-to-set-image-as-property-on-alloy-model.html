Unable to set image as property on Alloy model

Env: Ti 3.1.3 | Alloy 1.2.2

I'm using Alloy models, and I don't seem to be able to set an image (TiBlob) as a property on the model.

Here's how this plays out. A user sets an image using the camera or gallery, which is loaded into $.avatar.

Then:


 currentUser.set({
        fullName : Alloy.Globals.tmpProfile.fullName,
        username : Alloy.Globals.tmpProfile.username,
        password : Alloy.Globals.tmpProfile.password,
        email : Alloy.Globals.tmpProfile.email,
        image : $.avatar.image
});

console.log("image: "+$.avatar.image);    
console.log("currentUser: "+JSON.stringify(currentUser));

currentUser.save({
...});

The first of these console.log calls shows "Ti Blob", so I know I'm reading in the image correctly with the selector. The second of the console.log calls shows the following:

currentUser: {"bio":"","fullName":"test9","username":"test9","password":"test","email":"test@test9.com","image":{}}

When I inspect the HTTP request, the POST data also contains an empty object for the image property.

Any ideas of what's going on here? In case it matters, I'm using the Napp REST Api adapter for persistence on the server.

Thanks!
