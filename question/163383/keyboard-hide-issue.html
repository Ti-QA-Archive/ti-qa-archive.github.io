  • Hi Patel,
    I don't have lot of experience in Titanium but i am working on latest SDK 3.2.1 and below code hides keyboard once window opened.

    var loginWin = Titanium.UI.createWindow({

            navBarHidden : true,

        exitOnClose : true,

        url : '/ui/android/LoginWin.js',

        top : 0,

        windowSoftInputMode : Ti.UI.SOFT_INPUT_STATE_HIDDEN
    });

    loginWin.open({
        fullscreen : true
    });

    use windowSoftInputMode:Ti.UI.SOFT_INPUT_STATE_HIDDEN as property when you open window.

    I have tested in many android devices and it works for me.

    Jitendra

  • Muhammad's suggestion for using the focus event probably works. I didn't test. You can also solve this by adding the text field during the postlayout event, as long as you then remove the associated listener. Something like this:

    // it's fine to define your text fields outside of the event handler
// so, assuming you have two fields, username and password:

function addFields() {
  win.removeEventListener('postlayout', addFields);
  win.add(username);
  win.add(password);
}
win.addEventListener('postlayout', addFields);
