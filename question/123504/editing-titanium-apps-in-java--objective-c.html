Editing Titanium apps in Java / Objective C

Is it possible to start an app in Titanium, compile it and then work on it later in the native Java/Objective C environments? Would this be easy to do or does much of the code remain in Javascript?

    Aside from how feasible it is, this strikes me as making future project maintenance incredibly more complicated than necessary. If you want to do this so that you can incorporate things you can do in the native environment which the Titanium SDK does not yet expose, then IMHO a far better alternative is to code just those functions in a native module which you call from your Titanium project.

    You should read more about creating native ios and android modules.

    To more directly answer your question, no it would not be easy to do. It is a common misconception that all of the javascript gets converted into native code for each platform. That isn't quite the way it works. Two of the best explanations I have seen are here and here.

    I suggest you build both an iOS and android distribution build for a project, then go look at the respective build folders and examine the source produced. Yes, you can edit it and use the ./build.py script to recompile. But I really don't think you want to go there.

    If the intent is to accomplish things you can't in the current Titanium SDKs, I strongly suggest you instead consider putting those in custom modules.

    YMMV.
