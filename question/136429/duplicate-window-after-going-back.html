Duplicate window after going back

Device: iPhone

I'll try to make this as simple as possible:

I have a tab group with 5 tabs. The first tab is opened by default when the app loads.

Inside the first tab I have a table view with an event listener which opens reportWin when clicking on a row:

var reportWin = Titanium.UI.createWindow({
title: 'Report',
url:'report.js',
backButtonTitle: 'Select Location'
});

tableView.addEventListener('click', function(e)
{
var rowNum = e.index;
requestWin.djID = rowNum;
Titanium.UI.currentTab.open(requestWin,{animated:true}, {modal:true});
});

Two questions:

1) After selecting an item on the table view, it pushes me over the the reportWin. If in the reportWin I press the back button (labeled as Select Location), it takes me back to my original table view. Now, if I click on an item in the table view again, it pushes me over to the reportWin, however when I try to scroll, I see that there is a duplicate reportWin underneath the one that scrolls! Why could this be happening? It's as if the previous reportWin is not being closed?

2) If I don't put the backButtonTitle attribute when defining reportWin, the system does not create a default back button. Shouldn't there be a way to create a default back button? Am I doing this right?
