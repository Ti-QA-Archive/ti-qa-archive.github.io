  • Mario – I think you are fine building your app with the classic approach. I have been building Titanium apps for about 2 1/2 years, and I prefer the classic technique. We are building our biggest app ever using classic, and I feel like the app is well structured, with many reusable components. The resulting codebase should be quite easy to maintain.

    I must admit that I am biased against frameworks like Alloy that make it look like you can build an entire GUI application with a few simple configuration directives. My experience in 20 years of software development tells me that such tools might help you whip together an app that works 85% of the way you want, but finishing the last 15% will be a frustrating battle with the framework and its quirks.

    Maybe Alloy would surprise me if I gave it a chance. :-)

  • I built this app using both:

    https://github.com/kgividen/xControl

    It was my first real Titanium app and I used the traditional approach first. I then re-wrote it in Alloy a while later and was much happier with Alloy. I'm not saying my code is perfect but I would strongly recommend Alloy.

    It took me a while to understand models since I wasn't familiar with backbone but even if you don't use models it was much easier for me to work with and turned out to be cleaner code. Once I did somewhat understand models though it also cleaned up my code.

    I don't see any time I would not use Alloy over the traditional way in the future. There might be times to use traditional code inside of an Alloy app but I have no plans on going back to a full traditional app. Here are some reasons I liked Alloy:

    • Widgets. They are great! You can leverage a lot of reusable code from others or make your own. This is a great resource: http://gitt.io/

    • MVC. It was hard for me to separate styling from code in the traditional model. My code was all mixed together and even when I tried to modulize it became difficult to track through and messy.

    • It was much easier to add a theming mechanism with Alloy.

    • I got rid of most of my SQL statements with models and cleaned them up. My tables and data driven objects are much easier to maintain and understand now.

    • I now have many less lines of code to maintain because Alloy handles a lot.

    • Coding up your views, controls, etc. over and over vs just adding an XML doc and style takes longer for me. But I'm also more comfortable in an MVC environment or in an HTML, CSS, Javascript environment which breaks things out.

    There are probably more reasons but there you go. Just my opinion.
