Help required using Objective-C from iPhone app in native module

Hi,

I was wondering if anyone could give me some advice on using some code from an objective c iPhone app to create a native module for my app.

I am currently trying to add to an app the ability to view an MJPEG stream from an IP camera.

I have tackled this problem for weeks and first implemented this feature using a webview to view the mjpeg stream, but performance was really bad and the frame rate low as this whole webview is constantly re-rendered.

I then searched the web for a native objective c solution and found an open source iPhone app that used an imageview rather than a webview to display the MJPEG stream. I have stripped this app down as much as possible and I am now attempting to use this code to form a native module.

he app can be found at my github:

https://github.com/Matt-Marsland/Motion-JPEG-Image-View-for-iOS

My knowledge of creating native modules is pretty limited. I have looked at the moddevguide module to try to understand how to put this mjpeg code into a module, but I am struggling.

If anyone could give me some advice or pointers it would be greatly appreciated.

Thanks,

Matt.

  • Hi,
    mjpeg: this is a simple sequence of jpegs, mosten used by webcams.

    Display in Titanium:

    1. on iOS you can request this mjpg with a simple webview (html with embeded image tag)
    2. on all devices:
      Often the web cam offers also single jpegs.
      You can request the image with a xhr client, onload you fill the imageview by callback and call the same module again. In this case you can totally control the motionjpg: you cam abort connection or try it again by resuming of app or an other events.
