Android - Layout issues when using Multiple TableViews within a View

Hi all,

when I want to display multiple TableView's within a View, the View's height is not calculated correct. Depending on the screen size only some TableViewRow's will be visible/scrollable. If you rotate the device to landscape even less rows will be visible/scrollable. The same example works fine on iOS and MobileWeb.

The problem is that the height of .container View is not calculated correct. When simply removing .container everything will be displayed correct and everything is scrollable.

Unfortunately, I need the .container view because of Alloy's data binding feature. Imagine .container gets a dataCollection attribute and each TableView element equates to a Require element.

I know one could simulate a TableView by using a simple View instead, but this results in very slow rendering.

Example:

Controller

$.index.open();

Style

"Window": {
  backgroundColor: 'white',
}

".container": {
  layout: 'vertical',
  height: Ti.UI.SIZE,
}

"ScrollView": {
  layout: 'vertical',
  contentHeight: 'auto'
}

"TableView": {
  height: Ti.UI.SIZE
}

View

<Alloy>
    <Window id="index">
        <ScrollView>
            <View class="container">
                <TableView>
                    <TableViewRow title="1"/>
                    <TableViewRow title="2"/>
                    <TableViewRow title="3"/>
                    <TableViewRow title="4"/>
                </TableView>
                <TableView>
                    <TableViewRow title="5"/>
                    <TableViewRow title="6"/>
                    <TableViewRow title="7"/>
                    <TableViewRow title="8"/>
                </TableView>
                <TableView>
                    <TableViewRow title="9"/>
                    <TableViewRow title="10"/>
                    <TableViewRow title="11"/>
                    <TableViewRow title="12"/>
                </TableView>
                <TableView>
                    <TableViewRow title="13"/>
                    <TableViewRow title="14"/>
                    <TableViewRow title="15"/>
                    <TableViewRow title="16"/>
                </TableView>
                <TableView>
                    <TableViewRow title="17"/>
                    <TableViewRow title="18"/>
                    <TableViewRow title="19"/>
                    <TableViewRow title="20"/>
                </TableView>
            </View>
            <View height="10"/>
        </ScrollView>
    </Window>
</Alloy>

How the XML actually looks:

<Alloy>
    <Window id="index">
        <ScrollView>
            <View class="container" dataCollection="mymodel">
                <Require src="mywidget"/>
            </View>
            <View height="10"/>
        </ScrollView>
    </Window>
</Alloy>
