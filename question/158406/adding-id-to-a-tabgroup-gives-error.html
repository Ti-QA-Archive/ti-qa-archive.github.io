adding id to a TabGroup gives error

I am working on a mobile app on titanium alloy 1.2.2. I have made some code which is working fine.

<Alloy>
    <TabGroup> 
        <Tab title="Login">
            <Require type="view"  src="login"></Require>
        </Tab>

        <Tab title="SignUp">
            <Require type="view" src="signUp"></Require>>
        </Tab>

        <Tab title="home">
            <Require type="view" src="home"></Require>
        </Tab>

        <Tab title="partialHome">
            <Require type="view" src="partialHome"></Require>
        </Tab>
    </TabGroup>
</Alloy>

Now I want to add a menu bar to it. So just when I assign the TabGroup some id it shows an error:

message:
cannot call method 'open' of undefined

source:
$.index.open();

can someone told me why is this happening? My index.js file has just one line.:- $.index.open();

    The first child object that is defined in the Alloy tag, is given the same id as the name of the js file, in this case the TabGroup is given the id 'index' and is referenced in the js file using $.index.

    If you give that object a different id, the $.index object is undefined, so doesn't exist anymore which means you either change the $.index to your id, or remove your id from the XML markup and use $.index.
