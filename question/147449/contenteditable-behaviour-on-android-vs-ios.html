contenteditable behaviour on Android vs. iOS

This is more of a platform question than a Titanium API question. I'm trying to figure out if contenteditable is broken on Android vs. iOS. (Or, maybe, just broken for me, for some reason.)

I'm using it in a webview. On iOS the functionality can be tested in Safari, while on Android it has to use the built-in Browser (as opposed to Chrome or another third-party browser). That way you should, as I understand it, be seeing the same behaviour as a Ti.UI.WebView on the relevant platform.

The test is here.

(I figured a JSFiddle was easier than sample Titanium code which you'd have to pop into a test app yourself to accomplish, essentially, the same thing.)

Basically, tap the HTML Results pane to activate the contenteditable. Then select a word in the second line, and tap Backspace on the keyboard to delete it.

Doing the selection on Android can be tricky enough, but in my testing (on a Nexus S running 4.1.2), deleting something from the second line will delete the range from the first line, starting at the same character offset and for the same length as the intended second-line word. Basically, trying to delete from any line deletes from the first line in the contenteditable instead.

Does anyone else see this same behaviour?
