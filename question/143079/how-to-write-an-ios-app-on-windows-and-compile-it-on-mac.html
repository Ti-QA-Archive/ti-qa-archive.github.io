How to write an IOS App on Windows and compile it on Mac

I would like to develope a cross platform app on Windows. However my app must run on an iPhone. For personal reasons I cannot develope on a Mac. So how can I set up a project which I develope on Windows and give it to a second developer to test it on Apple products?

I have already set up a project for Android only, now it seems to be a problem to add the BlackBerry support later. I don't want to run in the same problem for the Apple products.

Do you have some ideas/experiance?

  • Hi

    In order to develop iOS apps, you must use a Mac. This is not a limitation of Titanium, this is the requirements for running the Apple SDKs. One way to get around this is to use a service called MacInCloud. Its exactly what it sounds like, an OSX machine in the cloud that you can connect to via your Windows box. The great part is that it comes pre-installed with Titanium and everything necessary to start building your apps. ** Make sure that you have an Apple Developer Account.

  • The best approach to achieve this would be to create the entire project on the Mac. So every project setting and dependencies are well set up. Once the project is created, you can import this project on you Windows machine. (there will be warnings when you import the project, but it will still work)

    You can (of course) test your code using Android on the Windows machine, this can help clear some bugs and avoid going back and forth between your two environments.

  • Hi,

    you can do this by creating your project and develop it on windows. once you want to run on MAC,

    Approach1:

    import your titanium project on MAC titanium studio and edit your tiapp.xml file.

    in tiapp.xml file add iPhone also in your project tiapp.xml by changing false to true in iPhone target.

    <deployment-targets>
        <target device="mobileweb">false</target>
        <target device="iphone">false</target>
        <target device="ipad">true</target>
        <target device="android">true</target>
        <target device="blackberry">false</target>
    </deployment-targets>

    Approach 2:
    or you can create another project in MAC and copy paste your JS files.

    in approach 1 don't forget to clean your imported project if reported any issue.
