    • It is a property;

      var parent = Ti.Filesystem.applicationDataDirectory;  // property
var parent = Ti.Filesystem.getApplicationDataDirectory();  // method

      These two lines return the same value, the first as a property, the second as a method of the object. Note the prefix of get, this is part of a getter and setter pair (where both are available).

      Personally I prefer to use properties when defining objects/views and the like and anything outside the create I use the getters and setters. Be aware there are still a few view properties, that do not have both even though they are documented as having both. If you get an unexpected response, try the other one.
