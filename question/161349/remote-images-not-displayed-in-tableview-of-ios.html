    • Sorry I thought I had - I certainly typed it out - must have closed the window.

      Firstly I should have said image: res[i].logo rather than image: url, I took the end point rather than the starting point.

      The problem you have with your chosen method is that the data returned is not one of the three accepted parameters;

      • local file path to the device
      • url of local or internet server address ending in a file
      • blob data

      Your routine is simply taking the raw data from the response and pushing that to the image - as it is not one of the three above it will not work. If you then saved the raw data as a file and passed back that local file path it would work. This is not recommend though.

      But if you are presenting an actual url to your routine and that url end with a normal file then you should simply pass that url to the imageView property image. This way the OS will manage the in memory caching and the external net calls to gain the image s- which will be done as they are about to be displayed not all at once. This is not only the normal behaviour it the most sensible behaviour on all fronts. You would have to have a very specific reason to not use this built in behaviour and reading your code there is nothing distinct about it.
