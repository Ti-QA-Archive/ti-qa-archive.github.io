Programmatic access to module version number at runtime

I've created an Objective-C iOS module using the 2.1.3.GA SDK. At runtime when my module is loaded, I would like to programmatically access the module version number that was specified in the manifest file of my module. Is that version number available at runtime, and if so, how do I access it programmatically? Thanks.

    Hi Art,

    it's not programmatically possible to have the values specified in the manifest available.

    I think you might of course just put a method in your module indicating also there the version and just change it there too when your module version changes, but it depends of course on which is your use case.
