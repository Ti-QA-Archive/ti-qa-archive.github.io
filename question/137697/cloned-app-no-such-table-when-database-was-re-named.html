Cloned App: "No Such Table" when database was re-named

I just finished my first app, which is to be the light version. Now I want to create the full version which will be identical to the light but with some extra features. (In retrospect, I should have done this the other way around, but too late now.) I cloned the light app and got it working as a perfect replica, database and all. But then I changed the name of the database:

var db = Titanium.Database.install('/data/LightDB.sqlite', 'LightDB');

to

var db = Titanium.Database.install('/data/FullDB.sqlite', 'FullDB');

As soon as I did that, I started getting errors everywhere saying "No Such Table". I've checked all the obvious things….

  • Opened up /data/FullDB.sqlite in SQLLight Manager from inside the Resource directory of the full app. The tables DO exist.

  • Deleted the Build folder

  • Deleted this folder: user/library/Application Support/iPhone Simulator / 5.1

  • in the emulator, did a "Reset Contents and Settings"

I'm not sure what else to do?

Application type: mobile. Titanium SDK: 2.0.1. Platform & version: iOS 5.x. Device: iOS simulator. Host Operating System: OSX 10.7.3. Titanium Studio: 2.0.1
