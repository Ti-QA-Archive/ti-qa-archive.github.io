    Hi, generally speaking it is rather bad practice to add a scrollview to a table view. Mostly because it's really not required. I would highly recommend refactoring so that you are adding TableView rows to your "spotlights" section. Instead of adding the views to the ScrollView inside one row. You are not taking advantage of the tableview.

    You have it structured better for the lower TableView Section. Add rows to TableView Sections instead of adding ScrollViews to TableView rows.

      What I found was that I did not get any benefit from using the tableview since I was not listening for a click event listener and I was not trying to inject rows or anything like that. The only downside of that is that I had to rebuild the whole scrollview to change the data in any meaningful way. But for displaying and interacting it worked quite well.
