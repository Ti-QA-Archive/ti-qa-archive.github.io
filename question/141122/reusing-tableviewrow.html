Reusing TableViewRow

Hi All,

In my application I am facing difficulty with one scenario mentioned below.

My application hits the web-server and get 400-500 records from server Database.

I want to show this records on screen with the help of TableView.

I am following the kitchensink demo app.In this app there is table_view_layout2.js module by studying this module I got how do I create the TableViewRow and add it to TableView.

But the problem is adding 400 TableViewRow in TableView is not good approach it makes my app slow and sluggish.

Is there any way so that I can reuse the TableViewRow objects already added to the TableView?
Please help I google a lot but Got nothing.

Thank You.

    Hi Ashwin

    Unless you are using an old android device or maybe a iPhone 3g and your tables are complicated - 400-500 records should not be a real problem.

    There is a built in caching method you can invoke by using the property className when you define a row. This tells the OS that it can re-use this row style and just swap data. You should use this on all rows that you add any views to. Also if you have more than one style of presenting the row then have more than one className in the table - but usually you should not need more than 3 or 4 - most of the time just one.

    Example;

    var row = Ti.UI.createTableViewRow({
...
    className: 'standard',
...
});

    If this does not solve the problem, please provide you tableview and tableviewrow code for us to test, also note which devices you are targeting. Also note if the speed is the initial adding of rows or the traversing of the tableview rows (scrolling up and down).

    • Hi Ashwin

      Most certainly. The idea of the className property is to tell the table that this row is the same layout as other rows - even if the content is different.

      So imagine you had a row style that includes a title label on the left side and a caption on the right, to include these you have to make your own tableviewrow. You then add lots of rows using this layout. By adding the property 'className' you tell the table that it does not have to recreate each row from fresh as they are displayed when scrolling up and down - instead the table can simply swap out the title and label into a previous cached row.

      Example;

      var row = Ti.UI.createTableViewRow({
    height: Ti.UI.SIZE,
    className: 'standard',
    width: Ti.UI.FILL
});
var lblTitle = Ti.UI.createLabel({
    bottom: 5,
    height: Ti.UI.SIZE,
    left: 5,
    text: 'This is the title',
    top: 5,
    width: Ti.UI.SIZE
});
row.add(lblTitle);
var lblCaption = Ti.UI.createLabel({
    bottom: 5,
    height: Ti.UI.SIZE,
    right: 5,
    text: 'This is the caption',
    top: 5,
    width: Ti.UI.SIZE
});
row.add(lblCaption);

      You then add as many rows as you need using this 'template' in a loop and then add those to the table.

      By using className: &#39;standard&#39;, you tell the table this rows share the same style.

      Now imagine you ALSO had another few rows but they has a different internal layout.

      var row = Ti.UI.createTableViewRow({
    height: Ti.UI.SIZE,
    className: 'other',
    width: Ti.UI.FILL
});
var viewTitles = Ti.UI.createView({
    bottom: 5,
    height: Ti.UI.SIZE,
    left: 5,
    top: 5,
    width: Ti.UI.SIZE
});
row.add(viewTitles);
var lblTitle = Ti.UI.createLabel({
    height: Ti.UI.SIZE,
    text: 'This is the title',
    width: Ti.UI.SIZE
});
viewTitles.add(lblTitle);
var lblSubtitle = Ti.UI.createLabel({
    color; '#666',
    height: Ti.UI.SIZE,
    text: 'This is the subtitle',
    top: 5,
    width: Ti.UI.SIZE
});
viewTitles.add(lblTitle);
var lblCaption = Ti.UI.createLabel({
    bottom: 5,
    height: Ti.UI.SIZE,
    right: 5,
    text: 'This is the caption',
    top: 5,
    width: Ti.UI.SIZE
});
row.add(lblCaption);

      We used a different className for this style as it had a different layout to the previous row style. Now the table knows there are two different row layouts to cache automatically.

      It is as simple as that. You could have a set of rows like this and the table would automatically swap data out into the cached layouts for you.

      standard
standard
standard
standard
other
other
standard
standard
other
standard
standard
other

      Hopefully that explains it a little better.

    • Re: …can I provide any string for className property?

      Yes, anything you like as long as it is text and numbers, I try and use something descriptive so I remember that layout in the future. Also you can create a library of these row layouts so you can re-use in future projects.
