Installing a non-market module

So I've been attempting to get a Bluetooth module from HERE to run and be used with Titanium.

Now, I downloaded the zip file from github and put it into the correct titanium modules folder but titanium was unable to find it. After some searching I found that what I downloaded was not the actual module but what appears to be the Xcode project or something like that.

So… How do I make the thing I downloaded into a usable and understandable module for Titanium? OR, is there someplace I can download the actual module zip? OR, is there another bluetooth module out there that I overlooked that would do a better job?

Any help is appreciated.

    Where to begin…

    First you want to open terminal and cd into the root directory of the module project. Secondly, if it's iOS then you do "titanium create", if it's Android then you do "ant". These commands will build the modules and generate zip files. See the build output in the terminal for where these zips are created. In Android it's in the dist folder, I forget for iOS.

    Note: For iOS, you will need to add titanium to your environment path, and it should point to the SDK version that you're targeting (probably 1.8.0.1 currently.)

    Next you will drop the zip file into the root of your project. You will also open your tiapp.xml and add the modules in there too. Here is an example from one of my projects:

    <modules>
        <module platform="iphone" version="1.0">ti.udp</module>
        <module platform="android" version="2.1">ti.udp</module>
        <module platform="android" version="1.0">org.appcelerator.sygic</module>
    </modules>

    Clean your project and rebuild, you will notice that the module zip disappears, in its place you will see a modules folder created with the necessary files instead.

    Then you can use the module within your project. See the module project's example/app.js for usage.
