Device-Family parameter seems to be ignored when I build

When I use command line to build my project the -F (–device-family) parameter seems to get ignored. When I install the app onto the ipad it displays like an iphone. But if I build in Titanium Studio the app shows up correctly.

Is there another parameter that I need to be entering?

ti build -p ios -b -F universal
cd build/iphone

xcodebuild -target MyApp -configuration Debug -sdk iphoneos clean build
xcrun -sdk iphoneos PackageApplication "$IPA_PATH/MyApp.app" -o "$IPA_PATH/$IPA_APP_NAME" --sign "iPhone Developer: MyName" --embed "$PROVISION_PATH/$PROVISION_NAME"

    Hi

    Have you tried targeting iPad specifically?

    ti build -p ipad -b -F universal

    Looking at the available and optional flags in the documentation here and you don't appear to be missing anything obvious.

    You could also try removing -b so it removes the --build-only flag and should compile to device to see if it's part of the manual process you are doing.

    Just a couple of suggestions, hope they help.

    • The following did work 

      ti build -p ios "--developer-name" "<DEVELOPER>" "--target" "device" "--pp-uuid" "<PROVISION>"
