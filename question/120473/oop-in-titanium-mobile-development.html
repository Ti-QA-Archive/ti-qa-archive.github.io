  • I think that using JavaScript's prototypes is better (and faster!) than simulating classes. It requires different way of thinking, so it can be difficult for someone coming from class-based OOP languages.

    If your project is medium or large size, you should organize your source code so that maintenance is easy to do. I like MVC approach in other programming languages, but also in JavaScript. Titanium JavaScript environment is different from browser's, but it can also benefit from MVC approach. For example, you can build UI components as "View" part of MVC, add events to that components, making them "Controller" part of MVC and use models (database, xml, …) as "Model" part of MVC. That way you have decomposed different parts of application and it's much easier to maintain application if it's organized that way.
