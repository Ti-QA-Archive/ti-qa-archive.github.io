Dropbox JS iPad/iPhone (screenshot)

In my application, developed for the iPhone and iPad, I'm using this API https://github.com/aaronksaunders/ti-dropboxjs.

It works perfectly, but now I have a doubt. I read that Apple has rejected many applications that use Dropbox because the login page is a link to sign up.
Now I noticed that if I start the application with the iphone, it should work fine, but if you start with the iPad, you'll see a different login page, which also shows other links, so I fear that it will be rejected.

What can I do?

  • Consulting the SDK iOS Dropbox, and more specifically the file for DBSession in iOS, I found that in API url is formatted in the format "%@://%@/%@/connect_login?k=%@&s=%@&easl=1%@"
    If, therefore, in webview uses url https:&#x2F;&#x2F;www.dropbox.com&#x2F;m&#x2F;connect_login?cont=https:&#x2F;&#x2F;www.dropbox.com&#x2F;1&#x2F;oauth&#x2F;authorize?...etc instead of https:&#x2F;&#x2F;www.dropbox.com&#x2F;1&#x2F;oauth&#x2F;authorize?...etc, you will see the same login page of the iPhone, then no other link :)
