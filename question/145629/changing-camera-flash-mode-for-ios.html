Changing camera flash mode for iOS

When I open the camera using Ti.Media.openCamera() with overlay, I don't seem to be able to change the flash mode (auto, on, off) using a custom button. Even the example in the kitchen sink does not work.

Here's my code inside the addEventHandler 

if (Ti.Media.cameraFlashMode == Ti.Media.CAMERA_FLASH_AUTO) {
    Ti.Media.setCameraFlashMode(Ti.Media.CAMERA_FLASH_ON);
    button.title = "on";
} else if (Ti.Media.cameraFlashMode == Ti.Media.CAMERA_FLASH_ON) {
    Ti.Media.setCameraFlashMode(Ti.Media.CAMERA_FLASH_OFF);
    button.title = "off";
} else {
    Ti.Media.setCameraFlashMode(Ti.Media.CAMERA_FLASH_AUTO);
    button.title = "auto";
}

When the button is pressed, it will always switch it to ON mode because it is always AUTO no matter what. After calling setCameraFlashMode, when I print out cameraFlashMode on the console, it did not change. For example,

Ti.Media.setCameraFlashMode(Ti.Media.CAMERA_FLASH_ON);
Ti.API.debug(Ti.Media.cameraFlashMode); 
// This produces "0" even though Ti.Media.CAMERA_FLASH_ON == 1. 0 is for AUTO.

Anyone had this problem before? Since it's not working in kitchen sink either, it might be a bug in Titanium. I'm using 2.1.4 GA (in kitchen sink, instead of calling setCameraFlashMode, it just assigns the value to cameraFlashMode property, which I already tried)

Anyone found a workaround?
