Alloy: where to put events? xml or js

Hi,

couldn't find it in the Alloy Best Practices list so I quickly ask:
What is the best place to put e.g. click events? Directly inside the xml:

<Button id="mybtn" onClick="clickFn"/>

or in the js:

$.mybtn.addEventListener("click",clickFN);

Should be the same after compiling or does it make any difference? If I put it in the xml it will save me one line in the js (cleaner code) but I have logic in the view-xml-file (so not all code/logic is inside the js-file). Any tips/recommendations?

  • that onClick or in this case doClick (they seem to be aliases at one point. not sure if doClick still works) ends up generating this:

        doClick ? $.__views.t.addEventListener("click", doClick) : __defers["$.__views.t!click!doClick"] = true;

    so they both are event listeners at the end of the day.
