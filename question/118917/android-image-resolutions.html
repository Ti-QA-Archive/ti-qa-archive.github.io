Android Image Resolutions

I'm trying to understand how different image files are resolved based on resolution. I've got an android project and modified the default code to replace the backgroundColor property with a backgroundImage:

var win1 = Titanium.UI.createWindow({  
    title:'Tab 1',
    backgroundImage:'images/default.png'
});

I chose 'images/default.png' because the default project files include 'default.png' in several sizes under 'android/images/<orientation_and_resolution>/'. Alas, the image is never found and my window is left blank.

The closest answer I've found is in the documentation about Using density-specific resources on Android, but this page refers to using directories named 'low', 'medium', and 'high', and doesn't address screen orientation, or whether it's a 'long' screen or not (i.e. 854 vs 800 pixels). The directories that come with the project seem to infer that such categorization is recognized and supported by Titanium.

Any idea where I can find more information about how the platform is resolving these images?
