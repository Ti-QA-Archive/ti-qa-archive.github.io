  • Accepted Answer

    Ti.UI.currentWindow isn't broken, but if you are using one context as many of the replies here are doing, you won't be able to use it, because you are already having a reference to the window that you have opende. As previously written, currentWindow only works with multiple contexts such as using the url property when creating a new window.

    • I am not sure if I understand you correclty. When you open a window using the url like url: 'mynewwindow.js' it opens what appcelerator calls a 'new context' (it runs in a different process). So if you plan to add a new view you have to that with different contexts in mind. Lets use my former example:

      Option 1: adding a view on to mySuperCoolWindow in app.js:

      novaVisita.addEventListener('click', function (e) {
          var mySuperCoolWindow = Titanium.UI.createWindow({
            url: 'mynewwindow.js',
            title: 'This is My super dupper window',
            mySuperCoolCustomVariable: VariableValue,
            mySuperCoolCustomVariable2: VariableValue2,
            orientationModes: [Titanium.UI.PORTRAIT, Titanium.UI.UPSIDE_PORTRAIT]
        });
        var mySuperNewView = Ti.UI.createImageView({
           backgroundImage: 'cal.png',
           top: 5,
           right: 7,
           height: 70,
           width: 60,
           border: 1
           });
// Here we add mySuperNewView
mySuperCoolWindow.add(mySuperNewView);  
mySuperCoolWindow.open();
myBoringAppJSWindow.close();
    };
});

      Option 2: adding a view to mySuperCoolWindow in mynewwindow.js:

      
var mySuperCoolWindow = Titanium.UI.currentWindow;
// Just to check if it sees the real window
mySuperCoolWindow.backgroundImage = 'images/fundo_landscape.png';
Ti.include("configdb.js");
// we define the new view
var mySuperNewView = Ti.UI.createImageView({
           backgroundImage: 'cal.png',
           top: 5,
           right: 7,
           height: 70,
           width: 60,
           border: 1
           });
// Here we add mySuperNewView
mySuperCoolWindow.add(mySuperNewView);

      If you need to pass variables from one window to another there are several ways to do it.

      Cheers, hope it helps
