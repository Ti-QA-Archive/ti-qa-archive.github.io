Android: WebView does not disappear after null-ing it

I have something similar to the following code:

var win = Ti.UI.createWindow();
var webView = Ti.UI.createWebView({ … });

win.add(webView);

win.addEventListener('close', function() {
    webView = null;
});

My webview contains a page which uses javascript (inside this page) to refresh itself. On my server logs I can see that the js refreshes the webview's content AFTER the window is closed, and what is more, event after THE APP CLOSES. I have to kill my app with some task killer to stop it.

Is it a bug? Is it reported? Is there a fix for this?
