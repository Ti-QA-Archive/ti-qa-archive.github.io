Multithreading - General Question

We have this app which connects to the internet to fetch feeds, which can take a while.

function callRuby(url, callback) {
    Ti.API.info("callRuby with this:" + url);
    sync_log += "feed:" + url + "<br />";
    var startTime = startTimeW();
    // network_activity = true;'
    if (!Titanium.Network.online)
        callback("failed");
    var client = Ti.Network.createHTTPClient({
        // function called when the response data is available
        onload : function(e) {
            var endTime = elapsedTimeW(startTime);
            // var feedData = JSON.parse(this.responseText);
            // var feed_length = feedData.length;
            // Ti.API.info("feed_length:" + feed_length);
            Ti.API.info("time taken:" + endTime);
            sync_log += "Time Taken:" + endTime + "<br />";
            callback(this.responseText);
        },
        // function called when an error occurs, including a timeout
        onerror : function(e) {
            // Ti.API.info("url:" + url);
            // Ti.API.info("callRuby:" + e.error);
            // alert('callRuby error:' + e.error+',url:'+url);
            // my_timer.stop();
            sync_log += "Feed Failed:" + e.error + "<br />";
            callback("failed");
        },
        timeout : 5000 /* in milliseconds */
    });
    // Prepare the connection.
    client.open("GET", url);
    // Send the request.
    client.send();
}

So we use a function as seen above to fetch feed data and process it.
We also use this type of thing to download say images.

Now if we go into the app, which this is being done the UI is not responsive, or lags.

Is the network fetches being done in a multithreaded setup be default or is there a way to do.

As I have been looking at blocks in xcode, and how they are used to make sure that the network fetches can be done without affecting the main thread where the UI stuff is run.
