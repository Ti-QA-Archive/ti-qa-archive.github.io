  • Aaron, the problem with rolling your own file download with httpClient is file size. I can't download the same files I could in iOS, even before the .file support. And now filesize is not an issue with iOS due to the direct download.

    In my case I need to be able to download files over 10MB when zipped, and over 20MB unzipped.

    My downloads fail now in android OS; they work in iOS. (Plus in iOS I use the ZipFile module and download the zip, then extract. But it also is iOS specific.)

    So it is not just a convenience thing. It is a make / break deal for me.

    On a related note, is there a way we can use Java's built in zip support to decompress a file on android without needing a module?
