Debugging the backgroundColor Property of a Button - Not Showing Through

Hi all-

I have an Alloy Button:

<Button id='login'>Login</Button>

That is styled as such

'#login': {
  backgroundColor: '#0098db',
  borderRadius: 5,
  color: '#8F9191',
  top: 150,
  height: 35,
  width: 300,
  font: {
    fontSize: 10,
    fontFamily:  'Gotham'
  }
}

The issue I am having is with the backgroundColor property. I can see the turquoise blue background peeking through the rounded corners of the button, but it for some reason is not showing on the button. The button remains white. the background color of my page is white. If I remove the property of the button, the button stays white still (not the default button color)…as if the backgroundColor is being set. I'm just not sure how this property is being overridden….as if it has a button being set atop this button background color.

Is there a way to debug the backgroundColor property, or make sure my property is set on the button? Not sure why my color is not being set on the button properly.

Thanks for any assistance.

Chris
