  • Hello Vince,

    From my side suggestion, You need to use a remote database.

    When user download the app and going for registration then just stored contact number in the remote database.
    Suppose, One user deleted this app and download again then you just need to check that the current number is available on remote database or not.

    If YES, it mean user have downloaded the app, if NO , it mean user have not downloaded this app atleast once.

    This idea is for the only contact num mean if you want to check that current number is registered or not.

    If you want that like current device have this app then I think you need to go for UDID or any unique ID which is unique for the Device.
