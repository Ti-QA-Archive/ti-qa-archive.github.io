google maps support for android 2.3.7

hi,
i was using the android 2.2.1 on my REAL DEVICE, my app was working correctly, and since i upgraded my phone to 2.3.7(unofficial version), my app don't work and i have a message in the logcat "instal_failed_missing_shared_library", and it s due to the missing of the library of google Maps on the phone. how to add library to the phone, or how can i resolve the issue from the manifest.xml??!

  • Hi Moez

    Does your tiapp.xml file have an entry for;

    <property name="ti.android.google.map.api.key.development">long code</property>
<property name="ti.android.google.map.api.key.production">long code</property>

    Your app needs to have the api key you obtain from google for Android Map usage. And it is placed in ti.android.google.map.api.key.production, the ti.android.google.map.api.key.development can be left as it is.

    • My Problem was a missing "com.google.android.maps.jar" from my android phone (2.3.7 unofficial ROM). So i"m unable to launch apps using google maps on the phone. to solve this problem that appear on the CM7 ROM. You have to download "ROOT EXPLORER" or similar application from google play and apply this instructions
      I've included a maps-framework-kfire.zip modified zip (maps-framework-kfire) file with only the files you need. The files are in the folders necessary inside the zip. As we can't flash zips from recovery like other devices yet (that I know of) you'll have to put the files in manually. As this uses google services you probably need to have GoogleServicesFramework.apk installed already in your /system/apps directory (to do this if you haven't either install the market which will include this step or transfer the GoogleServicesFramework.apk there and chmod to 644 like the others below [do this before you reboot as it will be required])

      1. Extract the zip file on your computer
      2. Copy the extracted files to your device
      3. Copy com.google.android.maps.jar into /system/framework via ADB or Root Explorer
      4. Chmod (long hold select file in Root Explorer and select permissions) to 644 (owner read/write, group read, others read)
      5. Copy com.google.android.maps.xml to /system/etc/permissions the same way
      6. Chmod that file to the same permissions
      7. Restart your Fire to allow it to recognize the new framework
      8. Install this APK Google_Currents_1.0.apk (or any app using google maps)

      This should also allow any other apps to install that require this framework.
