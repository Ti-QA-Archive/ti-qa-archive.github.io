Issues with Ti.Platform.displayCaps.platformWidth on device orientation.

Hi all..

I'm trying to implement a sliding menu using a widget found here .It worked fine, But I’ve a small issue with “Ti.Platform.displayCaps.platformWidth” as when i rotate the display the width is not changing automatically. I mean that the same horizontal width is measured even if the device held vertically. Any idea to solve it? I’m trying to deploy it in android.

    Hi

    This is because the answer you are being provided is platformWidth, the width of the platform and not a compensated orientation of width.

    If you want to determine the value with a compensation for orientation, simply determine the orientation and swap the values over.

    • If you do not need to know the actual answer then this is almost correct. It would be better if you wrote;

      width: Ti.UI.FILL

      I prefer this to 100% as if can be more flexible when used with other dimensions.

      This for example;

      left: 100,
right: 50,
width: Ti.UI.FILL

      This does NOT add the three things together it removes the left and right from the actual width leaving the object to fill the remaining space - can be very useful if you need to leave a fixed space but allow the balance to span the remaining area. This works for height as well.
