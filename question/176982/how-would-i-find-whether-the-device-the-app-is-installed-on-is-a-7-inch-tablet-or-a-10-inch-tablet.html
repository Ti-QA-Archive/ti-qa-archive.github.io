  • You're doing a ton of calculations to solve what the system will do for you automatically. Both iOS and Android support "display independent pixels" for both View dimensions and font sizes. You should not calculate these for yourself directly as you're doing. By doing so your app is tied to the resolution and display sizes available when your app is published. As you've found, you need to consider many variations and test across many devices to hand-tune all the parameters. Any devices you miss, or newer devices released after publishing will not get the benefit of your then-outdated calculations. All that, when the OS will just do it for you.

    With display-independent sizing, the dimensions/font sizes will be scaled appropriately for the physical pixels of the actual device. Use them like this:

    styles/app.tss or some other TSS file

    ".smallFont": {
   font: {
      fontSize: '10dp' /* use dp for display-independent pixel units */
   }
}

// you can do platform-specific variants
".smallFont[platform=android]": {
   font: {
      fontSize: '12sp' 
      /* sp is the Android-native unit for font sizes, though dp works too */
   }
}

    Starting with Alloy 1.4.0, you can also do custom queries in your TSS declarations. You can use these to override styles for specific devices, when the global styles aren't optimal

    alloy.js

    Alloy.Globals.isSomeSpecificDevice = true; // some logic to return a boolean

    in any of the tss files

    ".smallFont[if=Alloy.Globals.isSomeSpecificDevice]": {
   font: {
      fontSize: '40dp' 
   }
}
