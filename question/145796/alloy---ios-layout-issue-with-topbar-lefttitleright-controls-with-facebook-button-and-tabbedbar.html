[Alloy - iOS] Layout issue with TopBar Left/Title/Right controls with Facebook button and TabbedBar

Hi,

I'm facing a curious bug using Alloy and i don't if it's me or a primilary bug of this new great framework…
I'm would like to add control into LeftNavButton / RightNavButton / TitleControl.

I've tried to reproduce the quick example showed in Alloy Doc below and it doesn't work… (first deception)

<Alloy>
    <Window>
        <RightNavButton>
            <Button title="Back" onClick="closeWindow" />
        </RightNavButton>
    </Window>
</Alloy>

So I've worked around and use this solution which work for the moment

<Alloy>
    <Window id="myWindow">
            <Button id="add" title="Back" top="-50dp" onClick="closeWindow"></Button>
    </Window>
</Alloy>
  • this in the controller:
$.myWindow.setRightNavButton($.add);

But when i try to do the same thing with a Facebook login button (Ti.Facebook.LoginButton) or a TabbedBar (Ti.UI.iOS.TabbedBar)…. Using Alloy, it places the component like 100px below the position it should be. Causing a horrible layout bug because the button is below the Title Bar and overlay the window main content area.

To avoid that problem, for these 2 specific components, I had to create/code them into the controller.js.
So they are well place without any layout bug….

Does anyone faced that problem ? or is it an known Alloy issue which will be fixed in the next release ?
