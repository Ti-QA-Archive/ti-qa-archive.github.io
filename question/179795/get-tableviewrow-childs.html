get TableViewRow Childs

Greetings

I need to evaluate all the view's children. I use the getChildren() method.

<Alloy>
    <View class="subheader" id="subheader">
        <TableView class="subheadertableview" id="subheadertableview">
            <TableViewRow class="subheadertable" id="subheadertable">
                <ImageView class="btnMain" id="btnMain">
                </ImageView>
                <ImageView class="imagen" id="imagen">
                </ImageView>
                <ImageView class="btnCorner" id="btnCorner" >
                </ImageView>
                <Label class="namelabel" id ="namelabel" >
                </Label>
                <ImageView class="triangulo" id="triangulo" >
                </ImageView>
            </TableViewRow>
            <TableViewRow height="0">
            </TableViewRow>
        </TableView>
    </View>
</Alloy>

This view is content within another window call: "Tipologia".

So the JS code of Tipologia is:

var subheader=Alloy.createController('subheader', {}).getView();
$.Tipologia.add(subheader);

When I execute getChildren() code, it returns:

1.subheader
1.1 subheadertableview

But nothing beyond that; when I code getChildren() of subheadertableview it returns nothing.

Anyone has any idea why is this happening?

    • If you do subheadertableview.getChildren() it does indeed not return the section or rows.
      The TableView works differently for it's sections and rows. You don't get them with .getChildren(). However if you do want the sections through your TableView, do: $.subheadertableview.getSections(). This returns the sections inside the TableView
