iPad navigation controls slow to appear when going back

Hi guys

I have a tab group, which works just great except for one thing:

I have two windows:

  • window1
  • window2

From window1 I open window2 like this: myTabGroup.activeTab.open( window2 );

This works as expected. However, from window2 I have a button that closes the window (not using the back button in the upper left corner) using: window2.close();

It does close the window, however, I have a left nav button in window1 which only appears when the close animation is completely done. Why is this? When using the back button it's really smooth but when I use win.close(); it doesn't look smooth at all.

— asked September 15th 2011 by John Johnson

  • When you use the back button you get a built in animation and cleanup for free .. when you just close the window, there is no smooth animation attached as you didn't attach any. So, if you are going to use the close() method on the window, you may want to create some transition animation for your windows. Check out the Kitchen Sink demo for many examples of this.
