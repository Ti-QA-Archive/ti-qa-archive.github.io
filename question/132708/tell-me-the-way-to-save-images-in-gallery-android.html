    Please refer to the following URL:
    http://stackoverflow.com/questions/2170214/image-saved-to-sdcard-doesnt-appear-in-androids-gallery-app

    Use:Titanium.Media.Android.scanMediaFiles()

    After write
    Ti.Media.Android.scanMediaFiles([temp_target.nativePath], null, function(e){});

    • Hi Alessandro, there may be any problem we can not guess.

      If you are getting a file in external storage then, only one thing remains to test, it is not writing proper content in file. if paintView's background color is black, change it to white and use same code. this will clear problem a bit.

      If your paintView is different from your drawingView(on which you are drawing), try using parentView.toImage() or window.toImage() instead of paintView.image(), in that case.

      Here parentView is view which contains paintView and drawingView added in it.

      — commented August 13th 2012 by nitin soni

    • Thank you Nitin, it was the paintView background. Even if not documented, I set the background white as you suggested and worked. The parent background was white so the image was looking good.
