I have started a new app and I am unable to make a titlebar appear on my window. Do I need to use tab groups to get a title bar? I do not need tabs for this app. I am testing in iPhone simulator and see the same results across 3.2-4.2


var indexWin = Titanium.UI.createWindow({
    title: 'Index'
});

var data = [{title:"Row 1"},{title:"Row 2"}];
var itemList = Titanium.UI.createTableView({
    data: data,
    top: 60
});

indexWin.add(itemList);
indexWin.open();

    I finally found the single property that was needed.

    var indexWin = Titanium.UI.createWindow({
    title:'Index',
    modal:true
});

    Yep, for iPhone atleast, and a simple window, modal:true was the key. navBarHidden, fullscreeen, id, tabBarHiddden, etc made no difference. Am I going to run into any problems by making this window modal?

  • you can do it like this - UPDATED, basically you need to have the tabGroup

    // create tab group
var tabGroup = Titanium.UI.createTabGroup();

var indexWin = Titanium.UI.createWindow({
    title: 'Index',
    tabBarHidden:true

});

var data = [{title:"Row 1"},{title:"Row 2"}];
var itemList = Titanium.UI.createTableView({
    data: data,
    top: 0
});

indexWin.add(itemList);

var tab1 = Titanium.UI.createTab({  
    icon:'KS_nav_ui.png',
    title:'Tab 2 Title',
    window:indexWin
});

tabGroup.addTab(tab1);  


// open tab group
tabGroup.open();

  • This is a pretty old question, but thought I might answer it in case anybody else struggled with this.
    The tabGroup doesn't create by default a window element, it creates a navigation window, your solution would be to create a navigation window, and then add the window object after it.

    Right now we use alloy, so a basic structure in alloy xml would be:

    <Alloy>
    <NavigationWindow>
        <Window title="Window">
        </Window>
    </NavigationWindow>
</Alloy>
