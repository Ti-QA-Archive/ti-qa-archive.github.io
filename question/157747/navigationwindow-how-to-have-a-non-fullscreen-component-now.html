NavigationWindow. How to have a non-fullscreen component now?

Hello,

For our apps we have config panel which uses the NavigationGroup on iOS.
On iPad this NavigationGroup lives in a non-fullscreen view with some buttons next to it (which are not in the NavigationGroup).
What I now understand how the new NavigationWindow works, this is can not be done anymore (from 3.2.0) and the NavigationWindow has to be the window you open instead of having it as a child.

It's hard to believe I'm the only one who uses the NavigationGroup non-fullscreen.
So does anyone else also has this problem or am I missing something how to fix this?

Thanks.

  • Yes you can do that. Now that you say that, I'm probably not clear enough.

    You don't have to use the NavigationWindow as a fullscreen window, but what I want is some elements outside of the NavigationWindow in a non-fullscreen window.

    Let's do some art:

    |-----------------------|
|///////|_B_|            |
|///////|_B_|            |
|///////|_B_|  NavWin   |
|///////|_B_|            |
|///////|_B_|            |
|-----------------------|

    The window starts at where the first button is (|_B_|) but the navigationWindow should start next to those buttons. The buttons are used to control that NavigationWindow, so it's sort of like a non-fullscreen SplitWindow.
    The &#x2F;&#x2F;&#x2F; part is not part of the window (that's a semi transparent view where you see the window below).
