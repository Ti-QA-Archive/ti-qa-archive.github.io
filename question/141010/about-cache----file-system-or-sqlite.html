  • It's not clear what your requirements are for storage, whether you need read and write or just read, if you are dealing with a small are large amount of data, etc., so I'll just put down some other options that you might want to consider.

    If you aren't dealing with too much data and want to be able to read and write, consider keeping your data in JSON format and then save it to a property, like this.

    If you need more functionality and have more complex data requirements, you could go with SQLITE, or do a NoSQL option like JSONDB or CouchDB.
