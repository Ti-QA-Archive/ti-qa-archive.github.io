Button click to open a new window

I'm still on the steep, upward slope of the learning curve and I have yet another basic question. I've built a home screen that has a button on it. I'd like to attach a click event to that button that loads a new window. I've "borrowed" some code from the Kitchen Sink app that I thought might do the trick, but I'm getting an error. Here's what I have:

btn_er_app.addEventListener( 'click', function( e ) {
  var win = Ti.UI.createWindow({
    url: 'erapp.js',
    title: 'ER App',
  });
  Ti.UI.currentWindow.open( win,{ animated: true } );
});

The home screen is presented in my app.js file and the erapp.js file exists in the same directory. The error I get is:

message = "Result of expression 'Ti.UI.currentWindow' [null] is not an object.";

Is this a sequence error? Does my event listener have to be created after adding the button to the current window, maybe?

Thanks.

    win.open({ animated: true });

      Just one more explanation.

      If you do want to use "url" property anyway, then in "erapp.js" file you can reference current window (that window is stored in "win" variable - it's window that you created and opened with "url" property) with "currentWindow" property.

      If you don't want to use "url" property (which I highly recommend), then you need to include "erapp.js" file so you can use code inside it. You can include files with "Ti.include" or with CommonJS function "require".

      Check this out: http://wiki.appcelerator.org/display/guides/Building+Reusable+Factories
      It's good place to start (although it might become limited depending on your needs).

    • Yes, you got it (except proper code formating :D , you need to use 3 "~").

      To make this clear:

      //app.js
//app namespace
var MyApp = {};
//ui namespace
MyApp.ui = {};
//don't use underscore first
Ti.include('ui.js');

//don't use this since this create "view" global variable
var view = MyApp.ui.doSomething();

win.add(view);


//ui.js
MyApp.ui.doSomething = function() {
    return Ti.UI.createView();
}
