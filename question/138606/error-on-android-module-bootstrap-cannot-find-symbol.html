Error on Android module bootstrap: cannot find symbol

Hi,
I'm facing a problem on developing a module on Titanium 2.0.2.GA, both Linux (Ubuntu 12.04 64bit) and Windows (XP, 32 bit).

I created an Android module and an empty project using Titanium Studio wizards.
Here is my project's app.js:

Titanium.UI.setBackgroundColor('#000');

var tabGroup = Titanium.UI.createTabGroup();

var win1 = Titanium.UI.createWindow({  
    title:'Tab 1',
    backgroundColor:'#fff'
});
var tab1 = Titanium.UI.createTab({  
    icon:'KS_nav_views.png',
    title:'Tab 1',
    window:win1
});

var label1 = Titanium.UI.createLabel({
    color:'#999',
    text:'I am Window 1',
    font:{fontSize:20,fontFamily:'Helvetica Neue'},
    textAlign:'center',
    width:'auto'
});

win1.add(label1);

var win2 = Titanium.UI.createWindow({  
    title:'Tab 2',
    backgroundColor:'#fff'
});
var tab2 = Titanium.UI.createTab({  
    icon:'KS_nav_ui.png',
    title:'Tab 2',
    window:win2

var label2 = Titanium.UI.createLabel({
    color:'#999',
    text:'I am Window 2',
    font:{fontSize:20,fontFamily:'Helvetica Neue'},

    width:'auto'

win2.add(label2);

tabGroup.addTab(tab1);  
tabGroup.addTab(tab2);  

tabGroup.open();
// I just added these lines; without them the app works
var module = require('com.modules.module');
Ti.API.info("module is => " + module);

and here is my error trying to launch it on emulator: 

[ERROR] Error(s) compiling generated Java code

  • I struggled with this same problem all day long yesterday and no matter what I tried I was not able to get a module working. Eventually I decided to remove both versions of Python that I had on my machine (3.3 and 2.7) and to uninstall Titanium Studio. After that I installed the Titanium Studio. I was able to create a regular app, but not a module project. At that time it was complaining about not being able to find Python. I went ahead to install Python 2.7 since 3.3 was giving me some issues yesterday. I also made sure that my environment Path had this Python location in it.

    Even after doing all of that I was not able to create a module project (to create it now, not the problem that I was experiencing yesterday, which is the same problem that was reported in this post). This time around it was complaining about gperf, so I ended up downloading that as well and putting the location in the environment variables path. You can download gperf from this location: http://sourceforge.net/projects/gnuwin32/files/gperf/3.0.1/gperf-3.0.1-bin.zip/download?use_mirror=voxel&download=

    I should also mention that originally I was on 2.something version of Titanium Studio and after this latest re-install I an running 3.0.1.201212181159.

    I am always very reluctant to uninstall everything from my computer and start over, but by doing this I saved myself from another wasted day. I hope this will help you.
