How to fix a Titanium bug for Android compiling (ClassNotFoundException)

Alright, here I posed a question:
http://developer.appcelerator.com/question/120003/android-app-wont-run-once-signed-really-desperate

Nobody answered and after 30 hours of debugging I found out that Titanium Studio/Developer have a bug that can really prevent you from building your distribution on Android. Both programs have problems with the size of javascript files. In my case I found out that app.js wasn't compiled into a class because it's too big. Titanium Developer doesn't even catch the problem for the user: the .apk file is produced, alas it won't work. Titanium Studio will warn with a "64k limit" error that doesn't prevent the application from building the apk and for which I have found no information in the KB.

So, first of all, if running a packaged app (signed and zipaligned) you get an error like "Couldn't find pre-compiled class for script: app.js", Titanium simply didn't compile the js file in a class. Go back to your .js file, split it including part of it in an external file (obviously I would suggest including a whole activity or part of the program in the external file, not just arbitrary code to cut it short) and use the Titanium.include() method to include it. That's gonna fix it.

One quick note. Before I said that the problem is with the size of the .js file. To be precise, it's with the size of uncommented code. Any commented code won't count toward some kind of limit.

Now, I would really find it useful if someone from the staff here at Appcelerator could evaluate this information because something in my opinion must be done. Either allowing bigger .js files, warning explicitely the user and not producing the .apk file (which is not gonna work anyway) or documenting the issue. But someone should really address this because I threw away a weekend on this and it's not good publicity for a very good application.

Hope this helps someone…
