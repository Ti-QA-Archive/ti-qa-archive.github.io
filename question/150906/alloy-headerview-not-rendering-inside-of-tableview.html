  • Accepted Answer

    Ah, shoot, I should have caught this earlier in your code. The elements used to contain proxy properties, things like headerView, footerView, rightNavButton, etc… are all abstract containers. The <HeaderView> element itself is not rendered, only the content inside of it. This is due to some of the weirdness in the naming of some Titanium APIs. For example, although rightNavButton seems to indicate that it should be a Ti.UI.Button, it can actually be any basic Titanium View, so I make no assumptions about it being any particular type.

    Long story short, just do this and you should be all set:

    <TableView id="recipes" class="detail">
    <HeaderView>
        <View id="detail_header">
            <ImageView id="search_icon" onClick="search_icon_click" />
            <TextField id="search" hintText="Search for a recipe" onChange="search_change" />
            <ImageView id="search_cancel_icon" onClick="search_cancel_icon_click" />
        </View>
    </HeaderView>
</TableView>
