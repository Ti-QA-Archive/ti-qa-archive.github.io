Changes not visible in Android emulator, without deleting the whole Android build directory

Using Titanium SDK version: 1.6.1 (03/15/11 11:45 2fdc0c5)

Created a new project, and ran it in the Android and iPhone emulators to validate the dev environment.

Changed the application - using the Pizza ordering tutorial. At the end of step 1 ran the changed code. iPhone emulator shows the expected results. Android emulator still shows the default "two tab" default application.

  • Deleted the class files from the Android build directory - no change
  • Restarted the emulator and Ti Developer numerous times - no change
  • Uninstalled the app (through Android settings) - no change
  • Have deleted the .apk file prior to a new launch (with build) - no change

I can see that the .apk file is being updated - its time stamp changes.

It seems that the only way to get the Android runtime build to reflect my changes I have to delete the whole Android build directory. Is this right? \

The iPhone environment works as I would expect it. Why doesn't the Android development process work the same way?
