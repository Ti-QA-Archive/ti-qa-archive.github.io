  • it depends on many things.

    If you need to use a captcha I would suggest to include an existing webapp page, that will sort all type of troubles, without getting things too messy. Then you will need to send from your backend some data containing all of the credentials that you'll need to use in your Titanium app. [You can fire Titanium events from a webpage that can be caught by listeners inside your code].

    In case you want to use a Titanium page, you will need to expose webservices from your backend, and implement the captcha code remotely (imageview with the captcha image, the reload button and eventually checking if it's correct).
