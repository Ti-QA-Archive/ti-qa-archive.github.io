Application Flow Control - Multi-Threaded Issue

I'm new to asynchronous development and I'm having issues figuring out how to pause/delay/sleep my application so it waits for a download to complete before loading the UI components.

My mobile application uses a JSON file downloaded from the internet to define the values used to populate Table Data; however, the table is being loaded before the JSON file and I don't know how to slow the program down wait for the JSON file to load.

Is there a way to implement a EventListener on the JSON file load activity?

Any suggestions would help. Thanks.

    Here are the steps I would recommend you follow

    1. draw table
    2. display activity indicator, to let user know table is being updated
    3. start download
    4. finish download, when onload is called from httpClient, fireEvent to update table data
    5. update table data
    6. hide activity indicator
