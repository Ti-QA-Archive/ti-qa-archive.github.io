Rotated Image resets rotation using touchmove

In Android, a rotated Image change default rotation(0 degree) when rotated Image dragged using touchmove.
I set move action by dragging and rotate action by tap.

This is my code.

var isDrag = false;
var currentRotate = 90;
baseView.rotate(currentRotate);

baseView.rotate = function(angle){
    baseView.transform = Ti.UI.create2DMatrix().rotate(angle);
}

baseView.addEventListener('touchstart', function(e){
    //

});

baseView.addEventListener('touchmove', function(e){
isDrag = true;
// moving action
});

baseView.addEventListener('touchend', function(e){
if(!isDrag){
baseView.rotate(currentRotate + 90);
}
isDrag = false;
});
~~~

I use Titanium Mobile 1.7.5, Android 2.2.

  • After all, the problem is "createAnimation" with dragging.
    As solution,
    in the case of using transform, use like this code,

    view.transform = Ti.UI.create2DMatrix().rotate(angle);

    In the case of animation, don't use transform, use width, height and so on.
