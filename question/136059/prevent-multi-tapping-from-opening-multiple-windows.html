Prevent multi tapping from opening multiple windows

Hi guys,

I have an app which works connected to web services. For demo purposes I've added a demo mode to the app. In the module that makes the requests to the service I've added a flag which is triggered to true/false before making the request and at the end of each callback. This way I won't trigger multiple requests when users are taping multiple times on a button/table view row, and it seems to be working fine.

In demo mode the app uses a local database. And if the app is running in demo mode the request data and callbacks are passed on to a method which runs the database queries and at the end it calls the callback. Using the same flags, described in the previous paragraph, I can't prevent the user opening a window multiple times. I assume that the database queries are running very fast and then the flag is set to false/not busy but the rendering of the new layout takes longer and this is the moment when the user has some time to click the button/row again and it opens a second or even third window with the same data.

Does anybody have a better idea on how to tackle this?

Thanks!
