Minification / Combining Javascript files (Juicerize) performance

Hi,

I recently ran a few tests on the speed differences between the Ti.include vs combining all the JS files into one and thought people might be interested.

This was just for a basic shell framework (5 tabs a bit of code setting up windows/tabs etc).

Running this on the simulator the quickest boot time was 60ms.

If I combined the 12 JS files into one using the Juicerize tool and that combo file is app.js the quickest boot time went to about 56ms.

So it gave about a 6% improvement in the startup sequence.

Nothing amazing, but the 12 files were not particularly heavy lifters so YMMV.

Thought it'd be an interesting test.

Now I'm not sure on the full release build if any form of minification is included on the files before they're packaged/sucked into Titanium proper. I'd assume so, at least things like stripping all whitespaces, comments etc.

Anyway, perhaps the gurus at Appcelerator can shed a bit of light.
