iOS Module: Setting Proxy Values

I'm having a problem following the Module Developer Guide for iOS from the wiki part of the site.

In the "Setting Proxy Values", after I put this code on app.js:

var my_module = require('com.test');
var foo = my_module.createFoo({
  "bar":"123"
});

and create ComTestFooProxy .h and .m on Xcode:

#import "TiProxy.h"
#import "TiUtils.h"

@interface ComTestFooProxy : TiProxy {
}

@end

#import "ComTestFooProxy.h"

@implementation ComTestFooProxy

- (void)setBar:(id)value {
    NSString *str = [TiUtils stringValue:value];
}

-(id)bar {
    return @"456";
}

@end

When I run the project from Titanium, I get this error message pro iOS Simulator: Application Error - invalid method (createFoo) passed to ComTestModule at app.js (line 85).

Anyone?

  • Yes, there isn't.
    That's why:

    Titanium will automatically correctly define and dispatch proxy factory creation method like above.
You simply need to define a Proxy with the same name as your module and the name of your method plus the Proxy suffix.

ComTestFooProxy.h
ComTestFooProxy.m

Your module does not need to define the createFoo method.
Using this convention, Titanium already knows how to do that for you.
