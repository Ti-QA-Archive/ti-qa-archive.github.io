[Ti 3.0.2 + Alloy 1.0] Textfield/Password issue, user can't type more than 1 char

Strange behaviour… i couldn't find anyone facing the same issue.

Symptom:

Textfield/Password do not let the user type more than 1 character

Context:

The textfield is inside a TableViewRow

BAD Code:

 <TableViewRow>
         <TextField id="txtPassword" passwordMask="true" suppressReturn="false" hintText="Password" maxLength="16" />
</TableViewRow>

Fix

Just wrap the Textfield inside a View OR turn passwordMask=false

GOOD Code

 <TableViewRow>
          <View>
               <TextField id="txtPassword" passwordMask="true" suppressReturn="false" hintText="Password" maxLength="16" />
          </View>
</TableViewRow>

  • For a temporary workaround simply specify a height on the textField and it will work. For example:

    var confirmPasswordField = Ti.UI.createTextField({
    left               : 100,
    right              : 10,
    hintText           : 'Confirm',
    textAlign          : "right",
    maxLength          : 30,
    height             : 35,
    passwordMask       : true,
    autocapitalization : false,
    borderStyle        : Ti.UI.INPUT_BORDERSTYLE_NONE
});
