  • Short version:
    We currently do not support debugging Android apps in Genymotion emulators.

    Long version:
    In order for debugging to work, we write a file called "deploy.json" to the Android device's SD card. This is a legacy thing because old Android devices had limited internal storage whereas external storage was more plentiful. Since Genymotion emulators by default do not have SD cards, there's no place for us to put the "deploy.json".

    Unfortunately, you can't just create a folder "sdcard" in the root of the Genymotion emulator filesystem. We check the mount points for "/sdcard" and "/mnt/sdcard". You may be able to edit the VirtualBox VM and add a second drive and try to mount it as "/sdcard", but I have no clue if that's actually possible.

    I created a ticket a while ago to fix it: https://jira.appcelerator.org/browse/TIMOB-16132. This requires changes in both the Android build scripts and the Titanium Android code. It is currently scheduled for release in Titanium SDK 3.3.0, but that may change.
