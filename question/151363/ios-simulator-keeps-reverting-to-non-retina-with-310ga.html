iOS simulator keeps reverting to non-retina with 3.1.0GA

Prior to 3.1.0.GA I could set the iOS simulator's hardware version by opening XCode and setting it there then returning to Titanium and restarting. I believe I saw somewhere that this is now supposed to be fixed so you can set the version without going through XCode.

But I find that Titanium Studio sets the iOS simulator back to "hardware Device" -> "iPhone" each time. It doesn't matter if I change the setting while running an app from TiStudio or from if I've shut Ti Studio, gone to XCode, changed the hardware and returned to Ti Studio.

This is kind of a big deal for me. Anyone else see this regression? Any workarounds?

(Running latest Ti Studio 3.1.0.201304151600, latest Titanium 3.1.0.GA, on a Mac running Mountain Lion targeting iOS development.)

Note this all worked FINE before 3.1.0.GA.

thx
