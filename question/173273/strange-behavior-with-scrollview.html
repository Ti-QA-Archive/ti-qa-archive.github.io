Strange behavior with scrollview

I have a scroll bar in the following window. When you first navigate to the page it won't scroll. There are several text inputs that get added to the view called "NameHolder" from the controller. If you click in one of the text boxes, the keyboard pops up, then as soon as you dismiss the keyboard the scroll will begin to work. WTH? 

<Alloy>
    <Window title="Setup" layout="vertical"  navBarHidden="true">        
        <Label class="titleBar">Setup</Label> 
        <ScrollView id="scrollView" showVerticalScrollIndicator="true" showHorizontalScrollIndicator="true" height="Ti.UI.SIZE" width="Ti.UI.SIZE">
            <View height="Ti.UI.SIZE">
                <View height="40">     
                         <Label top="20">Names</Label>
                </View>                             
                <View class='wrapper' id="NameHolder" layout="horizontal">
                </View>    
            </View>
        </ScrollView>                    
    </Window>        
</Alloy>

    So it seems that my problem was having a vertical layout on the window. I removed the layout property, then gave my elements absolute positions and now it works.

  • Hi

    If the width and height of the contents of the scrollView exceeds the width and height of the actual scrollview then it will scroll - if they are less then - no scrolling.

    The keyboard situation is an is quirk that by reducing the external height means the internal height exceeds the external height - this triggers the scrolling, this remains even after a keyboard is dismissed.
