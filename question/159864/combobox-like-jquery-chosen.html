  • Dropdown lists are not part of the iOS UI component set, nor are they part of the Android component set. Both the Apple Human Interface Guidelines and Android Design Guides point out that dropdown lists are troublesome on mobile because of the small click areas and limited screen real estate. That's why both platforms offer the spinner controls.

    On Android, the spinner can appear somewhat like a dropdown list. However, iOS does not offer that sort of native control. There's a small chance that rigging up one of your own would keep Apple from approving your app for the App Store.

  • Hi

    There is a right way to do this, in fact there are two right ways; one for iOS and one for Android.

    We now use these methods in all our apps as do many of the big names apps.

    • iOS

      Modal window that pops up showing a table that an entry can be selected - shows a tick and allows the user to cancel or confirm the choice. The window is then closed updating the user interface to show the new choice.

      There is a great example of this in the Apple Store app (not App Store app that is different). Download the app if you do not already have it and then select any product that offers a choice - iPad case for example, press the initial colour and the modal window displays with the other choices for you to select from.

    • Android

      OptionDialog; this is the accepted way to display a mobile UI list of choices.

    • How

      In our apps we create a single CommonJS module (classic Titanium style) or Widget (Alloy style) that accepts the list of choices as an array and the currently selected item. It then uses the correct UI for the platform and then returns back the selected option value which is usually a title and id in our apps.

      By making this modular you can then use it anywhere you like in your apps because it becomes a CommonDialog you re-use.
