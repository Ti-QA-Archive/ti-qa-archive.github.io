  • You can trigger push notifications however and whenever you like. If you only want to send it to the app's current user, and trigger it from within the app code, local notifications might be better suited.

    The difficult part here would be in identifying when these things have occurred in Facebook. For example, when you say all content is from 'Facebook page' you mean HTML in a webView, or are you using the Facebook Graph API. If you're strictly dealing with HTML, that's likely not going to be sufficient. You probably want to look at the below to find out the best way to subscribe to Facebook updates. Then when you receive the update that you're looking for, create the notification and send it.

    http://developers.facebook.com/docs/reference/api/realtime/
