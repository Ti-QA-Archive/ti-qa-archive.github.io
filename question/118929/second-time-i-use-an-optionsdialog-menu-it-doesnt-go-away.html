Second time I use an optionsDialog menu, it doesn't go away

I'm experiencing this bug in two separate circumstances. I'll use the options dialog once and it will be fine, and the next time I use it it will trigger the event but wont hide() itself after a click.

Anyone know how to fix this?

  • I encountered the same problem when declaring the dialog VAR and EVENT LISTENER inside a function. Within the same code block I had also set up several nested VIEWS and noticed the bugginess started when calling the options dialog from nested content.

    SOLUTION: I lifted the options dialog code out of the main function that populates the UI with data. The Options Dialog now lives on the outer layer of code attached to the currentWindow. Individual selections are saved to Ti.App.Properties. Other dependent elements first GET properties from Ti.App.Properties and then do something. So this is how I pass variables in and out of the main function, and how i store them across user sessions.
