Android OS by the numbers

I am starting a new app and wanted to look into using the new Android ActionBar available to us in mobilesdk 3.0. I looked and the native ActionBar is available in Android 3.0 (API level 11) and above.

This got me thinking. If I use this feature and require API 11 and above in my app would I be alienating a large portion of users? I found this report from last month: linky

So it looks like I would be missing out on over 60% of all Android users. I know I could check which OS version a user is using in my app and either use the new Android ActionBar or use my current custom ActionBar i use in all my previous apps. Right now, I don't think it's worth handling both scenarios.

Just thought I'd share…. Anyone have a differing opinion on not bothering to use this until more users can take advantage?
