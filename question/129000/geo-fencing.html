  • You can solve your problem in this way:
    You test if your location is in a polygon or not. Here is the code:

    function isPointInPoly(poly, pt){
    for(var c = false, i = -1, l = poly.length, j = l - 1; ++i < l; j = i)
            ((poly[i].y <= pt.y && pt.y < poly[j].y) || (poly[j].y <= pt.y && pt.y < poly[i].y)) && (pt.x < (poly[j].x - poly[i].x) * (pt.y - poly[i].y) / (poly[j].y - poly[i].y) + poly[i].x) && (c = !c);
        return c;
    }

    Hope this helps.

    • Unfortunately this does not work if the app gets pushed out of memory. I eventually switched to writing my apps in Obj C to accomplish what I needed. The problem is that, at the point where you need to handle the geofence trigger (assuming your app is out of memory), the Titanium framework hasn't fully loaded yet. So what you need to do it handle the trigger with Obj C code that gets executed before the framework finishes loading. If you don't want to completely switch to Obj C, you could modify the Titanium framework and add the handling code directly into the framework.
