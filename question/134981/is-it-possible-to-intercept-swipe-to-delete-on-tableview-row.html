Is it possible to intercept swipe to delete on tableview row?

iOS, 1.8.2

I have a tableview with custom rows that include image views along the right side of each row. When swiping a row to signify I wish to delete, I'd like my image to disappear to make way for the delete button instead of being moved over to the left, which is what is happening now. I know that an event fires when the row actually gets deleted, but is there any way to catch the start of the action- when the row is swiped?
