  • Hi,

    We can actually create separate user interface files for our iOS and our Android implementations. Because in Android we use the Back button to navigate back through Windows. Whereas in iOS, we click through, and then the Back button is automatically provided by the tabGroup.It's one of the reasons why we're using the tabGroup to begin with. So, on Android it's much more idiomatic to use the Back button, because regardless of what framework was used to code in Android applications, they are all coded to use what's called the back stack.
