Android and different screen resolutions (texts and positioning)

Hi, I'm developing both Android and iPhone app. When I run the app in the emulators (both iPhone and Android) they look fine. Objects are positioned where I need them and text size looks as I want.

However, when I run the app on my Samsung Galaxy the text is much much smaller that it is in the emulator. If I make the text bigger so it looks normal on my Galaxy device then it looks way too big on the emulator and the iPhone.

I know that there are a lot of different Andoird devices out there in different sizes and shapes with different resolutions. I know (in theory - but would be happy to get more info) that there are different settings for the emulator - currently I run on HVGA.

As for positioning - I place things on screen with percentage. That does the trick. Trying to do the same with font size doesn't do the trick.

Is there a way I can know what screen resolution the app is running on (like I know osname and set settings accordingly?) ?

Thanks

    The Titanium.Platform api gives you a number of properties to help identify the current platform including osname, model, and maybe most importantly in your case DisplayCaps. Read the docs for each api for more information.

  • Another way to make the view of both the emulator and the phone view the same by adding the following in tiapp.xml

    <android xmlns:android="http://schemas.android.com/apk/res/android">
        <manifest>
            <supports-screens android:anyDensity="false"/>
        </manifest>
</android>
