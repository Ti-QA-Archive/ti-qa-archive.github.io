Button size not according to backgroundImage

Hi
Maybe this is not such a good question but I will ask it anyway…
I would like to create a button that inherits its backgroundImage size. Having button icons with different sizes for each device DPI (ldpi, mdpi, hdpi and so on) I would like the button to automatically inherit backgroundImage size… resize itself according to its content.

Something like this:

var button = Titanium.UI.createButton({
    backgroundImage: 'myImage.png',
    backgroundSelectedImage: 'myImageSelected.png',
    width: Ti.UI.SIZE,
    height: Ti.UI.SIZE
})

Somehow this approach does not resize the button according to backgroundImage. Instead it seems that the button has a default size value and the backgroundImage get stretched, beeing bigger than it really is. I don't know how to solve this using Ti.UI.SIZE. Do you even use Ti.UI.SIZE on buttons?

Platform: Android

Than you in advance.

    Hi

    Setting Ti.UI.SIZE for width and/or height tells the button to occupy the space based on the contents of the button. This is based on the title and image if provided. Setting a backgroundImage to any view will not affect the contents space so will not affect the width and height. The backgroundImage will only fill the space that has been created by the actual contents.

    If you are using a backgroundImage with a specific size then set the width and height to those dimensions yourself.

    This is the normal behaviour of this view and all other views that allow backgroundImage as it does not contribute to the content space only reacts to the space that has already been defined.
