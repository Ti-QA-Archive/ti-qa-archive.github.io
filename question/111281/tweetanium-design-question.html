Tweetanium Design Question

The UI approach used by Tweetanium seems to assume that users will not change orientation when using it. If I change orientation from Portrait to Landscape I actually see a "tab and a half" at a time. A similar issue is plaguing my app as I used some of the UI concepts of Tweetanium as a starting point.

Anyone have any ideas on how to make this work better?

  • I should probably make Tweetanium smarter about orientation over time, but the short answer is to have all of your UI components listen for an orientation change event and redraw themselves based on the new layout. It ends up being a great deal of work, unfortunately, since Titanium native layouts are not quite as robust as we might like them to be (we know this and are working on new features to address that).

    Also, to the extent that you can avoid specifying heights/widths/absolute positions and rely on the default positioning behavior of views, that will help your cause for orientation changes.

    Another thing you can do to "cheat" is to only allow orientation changes for certain windows in your application (I am going to pursue this for Tweetanium, since the only screen where supporting landscape really makes sense is on the "New Tweet" window).
