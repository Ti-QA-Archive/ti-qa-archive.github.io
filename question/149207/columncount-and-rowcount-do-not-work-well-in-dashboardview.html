  • This is not a bug in Alloy only, it also appears with the old coding style.

    I have managed to make a workaround with editing TiUIDashboardView.m:

    static const NSInteger kDashboardViewDefaultRowCount = 2;

    Don't ask me why, rowCount should override this value but it does not properly.

    Ti.UI.createDashboardView must also have a proper height to get my custom grid (e.g. 2x3) to work.

    Yeah, just a workaround, real solution would be nice.
