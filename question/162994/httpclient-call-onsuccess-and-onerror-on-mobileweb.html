HttpClient call onsuccess and onerror on MobileWeb

My project is set to Android and Mobile Web platforms, on Android works fine, show the alert just once on debug and run.
The problem is when I run on Mobile Web, my HttpClient ignores my timeout value and when the timeout of the request ends with no answer it calls onsuccess with empty responseText and onerror together.
I don't know how to proceed, just didn't find anyone with this problem. Am I doing something wrong?

function createLogoutPost() {
   return Titanium.Network.createHTTPClient({
      onload : function(event) {
         alert("success");
         $.activityIndicator.hide();
         showMessage("mimimi.");
         toggleButtons(true);
      },
      onerror : function(event) {
         alert("error");
         $.activityIndicator.hide();
         showMessage("mimimi.");
         toggleButtons(true);
      },
      timeout: 5000
   });
}
