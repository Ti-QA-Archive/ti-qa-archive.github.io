Weird behavior on tapping again the active tab

My problem is: if a tab loads a second window and I tap on that active tab again, it goes back to the first window.

So, here's the code:

app.js

var firstWin = Ti.UI.createWindow({url:'firstwin.js'});
var myTab = Ti.UI.createTab({window:firstWin});
var tabGroup = Ti.UI.createTabGroup({backgroundColor:'#fff'});
tabGroup.addTab(myTab);  
tabGroup.open();

firstWin.js

[some other code...]
var secondWin = Ti.UI.createWindow({url:'secondWin.js'});
Ti.UI.currentTab.open(secondWin,{animated:true});

So if I tap on the tab when the secondWin is opened, it's like if I pressed the Back button.
Is it a bug? can I disable that feature?

iPhone, iOS 4.2, Ti SDK 1.6.1

    Hi there Alberto,

    That "tap active tab" behavior is standard/default behavior in iOS, so users would notice more if it didn't do that.

    You may be able to customise the tab click event to do something else, unless it occurs before the event is fired.
