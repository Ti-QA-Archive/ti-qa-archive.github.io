  • I am also interested to know whether 2.0 will address this issue. But I am not going to be too hopeful on this since there is no indication yet for Appcelerator to do this.

    One thing for sure you can do is to compile the core yourself, and strip whatever libraries that are not needed. If you open the apk file using 7zip for an example, under lib folder there are another two subfolders which are quite identical:armeabi and armeabi-v7a. I think these are v7 and rhino libraries and they shouldn't be in together in the apk. Probably the changes needed from Appcelerator is if user chose v8, then don't embed the rhino libraries. I could be wrong though, v8 may still refer to some symbols in rhino - so for this, there is a need to restructure the libraries so v8 will never depend on rhino in future.
