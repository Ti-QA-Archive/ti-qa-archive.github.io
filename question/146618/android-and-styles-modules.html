Android and Styles Modules

Heyah Folks-

I am working on porting my iOS app to Android and I have run into a snag. Based on examples I've seen posted on how to build a CommonJS app, I was storing all style info in a separate module and referencing that module in my layout code.

For example this worked perfect in iOS:

File One:

var showStyles = require('modules/styles/shows');
var searchView = Ti.UI.createView(showStyles.searchView);

File Two:

exports = {
searchView: {top:0,left:1,height:'50dp',layout:'absolute'}
}

In android, however, it doesn't seem to recognize any of the style info I have in the second file.

This works fine:
var searchView = Ti.UI.createView({top:0,left:1,height:'50dp',layout:'absolute'});

…but obviously this is what I was trying to avoid in the first place.

Could anyone give me some guidance on how to fix this without merging my layout and style info into a single file?

Thanks!

Ben
