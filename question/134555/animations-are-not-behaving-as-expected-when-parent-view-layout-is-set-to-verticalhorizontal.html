Animations are not behaving as expected when parent view layout is set to vertical/horizontal.

Here is the code where I add 2 labels in a parent view

var container = Ti.UI.createView({
    width : 50,
    height : 50
    layout : 'vertical'
});

var labelUp = Ti.UI.createLabel({
    top : 0,
    height : 20,
    width : 40
});

var labelDown = Ti.UI.createLabel({
    top : 0,
    height : 20,
    width : 40
});

container.add(labelUp);
container.add(labelDown);

Here top property of labels represent offset from the view added before.
LabelUp is first and LabelDown is below it.

If I do animation on LabelDown where I scale it:

var matrix = Ti.UI.create2DMatrix()
matrix = matrix.scale(1.5, 1.5);
var a = Ti.UI.createAnimation({
    transform : matrix,
    duration : 500,
    autoreverse : true,
});
labelDown.animate(a);

It first places label on top of the view, then scales it, then reveres it (back to top of the view) and the it just places it down on init position.

This looks like a bug to me since animation should started and finished from the LabelDown init position, not from the top of the parent view.
