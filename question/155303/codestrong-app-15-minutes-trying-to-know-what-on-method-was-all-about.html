Codestrong App: 15 minutes trying to know what ".on" method was all about

Hi,

I've been analysing Codestrong app, and to be honest I found myself lost when I saw dozens of ".on" method calls $.object.on. I've looked at titanium docs and no word about ".on" method. Take a look:

$.login.on('click', function() {
    $.index.parent.add($.loading.getView());
    $.loading.start();
    $.email.blur();
    $.password.blur();
    User.login($.email.value, $.password.value, function(e) {
        $.trigger('loginSuccess', e);
        $.password.value = '';
        $.loading.stop();
        $.index.parent.remove($.loading.getView());
    }, function() {
        $.loading.stop();
        $.index.parent.remove($.loading.getView());
        showError();
    });
});

Well, look at this code It looks a lot like adding an event listener… but in any part at titanium docs says that .on method exists. So I was guessing what the hell is that. Tried to find if it was a custom method added to the classes, but it wasn't.

So after a while I found what the hell was ".on" about… http://nodejs.org/docs/latest/api/events.html#emitter.on

It's a node emitter, or the same as addListener…

I guess this kind of stuff should have been documented… I'm titanium rookie, but I think this kind of undocumented aliases brings some confusion when decoding others work (great work)… in this case Tony's work!

My 2 cents ;)

Joao
