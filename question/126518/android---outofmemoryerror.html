Android - OutOfMemoryError

I am running Titanium Studio, build: 1.0.2.201107130739 on Mac OSX. I am experiencing a problem where my Anrdoid app crashes after opening several windows in the same tab, and than opening and closing the last window twice. Error report:

E/TiUncaughtHandler( 1007): (kroll$9: app://views/jobs/jobInfo.js) [1438,144605] Sending event: exception on thread: kroll$9: app://views/jobs/jobInfo.js msg:java.lang.OutOfMemoryError; Titanium 1.7.1,2011/06/21 14:28,293a6d
E/TiUncaughtHandler( 1007): java.lang.OutOfMemoryError

(jobs.js) simply displays a table view with labels in each row and that loads ok. When you click a row a new window (jobInfo.js) will open in the current tab which shows some information with a webView, 3 buttons, and a few labels. I have completely removed all code out of jobInfo.js leaving only the following

Titanium.include('../../models/__models.js');

var win = Titanium.UI.currentWindow;
win.backgroundColor = '#F7F7F7';
var job = win.job;

After removing the majority of the code in JobInfo.js I am able to open and close that window via the back button on Android more times than before but it will still crash after around the sixth time. Here is the code used to open JobInfo.js

var win = Titanium.UI.createWindow({
url:'jobInfo.js',
job:jobs[e.index],
title:jobs[e.index].category,
});
Titanium.UI.currentTab.open(win,{animated:true});

There seems to be a some memory that is not being freed up when closing JobInfo.js. I've spent many hours fidgeting with code and reading forums. Can someone help me please? This is a very urgent need. Thanks!
