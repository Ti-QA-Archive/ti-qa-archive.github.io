Titanium Studio auto-complete functionality for "function" JavaScript keyword

Hi.

I'm using latest nightly build of Titanium Studio (build: 1.0.0.201106040720, on Windows XP) and there is new auto-complete "feature" that annoys me.

When I type "function" keyword in Titanium Studio JavaScript editor, "Function" keyword shows in auto-complete dialog as first option and when I come to "(" character that follows "function" keyword, it gets overwritten by "Function" keyword.

For example:

var myFunc = function()...
//this gets overwritten by this after I type "(":
var myFunc = Function()...

Note: "function" and "Function" are different things in JavaScript.

The solution is to put "function" keyword first in auto-complete options, "Function" is rarely used (I never use it).

Please change this behavior.

Thanks.
