Security in Rest Api

I've got some quedtions about security. I've created a Rest Server with basic auth. In my app i call with Ti.Network and i specified user and pass in headers, question is, can anybody read this credentials when my app is in AppStore and copy them to use for their own? I mean can anybody read my source code or monitor network request¿

Or how it's best way to securize my api¿ Thanks

  • Nobody can read your source code.

    I guess it's possible to monitor a network request via the iPhone somehow, but regardless you'll be safe enough if you replace the user/pass being sent in your headers with an encrypted token instead. Then, on your server, decrypt your token to get the information you need out of it. Alternatively, to be more secure, make the token encryption one-way and store it along with the users details in a database or something, and authenticate them that way.

    Secure your REST API with SSL for additional security if it's that important.

    • Nobody can read your source code- I would actually say ALWAYS assume they can.

      Titanium only base64's your code into an Object C file it does not encrypt it, so some one with the correct knowledge could extract.

      I would suggest that if it is a username and password that is typed in when your app is running then if it is always over SSL then your headers will be encrypted as well so you should be fine, BUT if you are saying that you have a master username and password hardcoded into the header then this is a BAD idea.

      Boydlee's suggestion about using one way tokens in this case would be slightly more secure, but again if the token is the same all the time it's a bit like MD5'ing the same phase (not a lot of point).

      If you encrypt on your app and then decrypt on the server, again be careful how you store the encryption key (and how you identify it within your code - i.e. var secretkey=1234567890) - that would be easy to extract.

      At the end of the day the security of you application and server is a direct measure of how confidential your service is and how long it would take someone to discover how to get into it.

      Hope this helps as I have had a few apps PEN tested (penetration) with external security companies and this a some of the feedback.
