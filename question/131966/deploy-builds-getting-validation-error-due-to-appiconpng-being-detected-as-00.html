'deploy' builds getting validation error due to appicon.png being detected as 0,0

Since updating to v1.0.8 of Studio, each ad-hoc or app store build I create using studio fails validation through XCode 4.

The error I get is:

"iPhone/iPod Touch: appicon.png: icon dimensions (0 x 0) don't meet the size requirements. The icon file much be 57 x 57 pixels, in .png format"

I've double checked both my appicon.png and appicon@2x.png (neither of which have been changed since I last submitted a built) and both are 24-bit pngs of the correct dimensions.

I've tried deleting my 'build' folder so that this would have to be recreated; I've tried compiling against older versions of the SDK; I've tried completely deleting the project from Studio and re-importing it - none of this makes the slightest bit of difference.

The only thing I haven't actually tried is attempted to revert back to an older version of Studio. But thought I'd better check on here first to see if anyone get help or is getting a similar issue?
