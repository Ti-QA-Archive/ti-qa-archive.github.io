Android app crashes "randomly"

I've developed an app that runs on the same code for iOS as for Android. The app works pretty much fine for Android, however sometimes the app crash and restarts.
I am very well aware of what im doing when the app crashes, but i don't see any reason for it to crash.

The app shows some events in a tableview. The user can select events per date. When the app is launched, the events for the current date is shown. Then the user is able to change the selected date in a "filterview". The filter view is actually HTML-buttons rendered in a webview. When a button is pressed, it fires an event to Ti.App, containing the new filter data.

The user is only able to select dates from today and 7 days forward (a week in total). First time a date is selected, an XHR request is made to the server. Then the data is "cached" local in the phone. The app contains a login screen that is presented before the events screen. When the events screen is closed the "cache" is cleared. To be able to reproduce this issue i have to close the events screen, enter it again (at least one time) and then change the date in the filter view.

Mi first thought was something like that the phone couldn't handle multiple XHR requests, but i implemented a queue which makes sure only one request can be performed at a time, and it didn't solve the issue.
I've used Fiddler to check if there is any bad data returned from the server when the app crashes (the server returns JSON) but all that is fine. Sometimes the app crashes before the XHR request is made. Sometimes after.

I have used logcat to find the reason why it's crashing, but when the app crashes all i get is:

D/CrashAnrDetector(  783): processName: se.kodfabriken.myapp
D/CrashAnrDetector(  783): broadcastEvent : se.kodfabriken.myapp data_app_crash

I though maybe debugging the app would help me. However, Titanium tells me absolutely nothing when the app crashes.

How should i continue debugging my app to be able to solve this?
