Redux causes crash on Android device when using Titanium mobile sdk 1.8

When including redux.js in a project using Titanium mobile sdk 1.8, the app always crashes at startup on an Android device. It runs fine in the simulator.

I have narrowed it down to this line in redux.js:

redux.fn.addNaturalConstructor(this, Ti[i3], redux.data.types[i3][j3]);

in particluar

Ti[i3]

It makes no difference whether it is using the v8 or Rhino engine.

I have tried 1.8.0.1 and the latest nightly build. It runs ok in 1.7.5.

The bug can be replicated by trying to build the Jab11 (https://github.com/appcelerator-titans/jab11) project with 1.8 sdk and then run it on an Android device.

I'm wondering if anyone knows if this is this a bug with sdk 1.8 or redux?

I would be very interested in any workarounds.
