How to detect if an app is loading data from the network

mobile, Titanium Studio, build: 3.4.1.201410281727, iOS 8.1.2, iPhone 5c and Simulator, OSX latest.

I am trying to do a custom ProgressBar to display when data is loading in a webview. The data is internet traffic so I have no control over how it is structured. I need to know when to start and stop the custom progress bar.

But the 'beforeload' and 'load' events are no good as they seem to fire with some ajax calls and not with others. To make matters worse, some calls trigger 'beforeload' but never trigger 'load'…

So next I attempted to poll the WebView.loading property to get full control over when the webview was actually loading data. But that just returns 'undefined' even if the webview is clearly loading data.
I have filed a ticket on that here https://jira.appcelerator.org/browse/TC-5164

Is there any other way to detect if the app have active network connections? Maybe from higher up?

  • Accepted Answer

    For data loading inside a webview, I'd use code in the webview itself to create the progress bar. You could iframe the contents if you're loading remote content over which you don't have control. Otherwise, use any of the jQuery-style progress bar components just as you would if creating a web page.

    I suppose you might be able to inject code (with evalJS) to listen for readystate levels and fire a Ti.App.fireEvent call which you'd listen for at the Titanium layer to hide a Titanium progress bar. My hunch is that this wouldn't work reliably (or at all for an AJAX-heavy page).
