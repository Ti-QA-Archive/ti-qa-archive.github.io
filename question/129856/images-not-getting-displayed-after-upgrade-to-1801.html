Images not getting displayed after upgrade to 1.8.0.1

In my application, images are stored in <project>\Resources\images folder, whereas
appicon and default.png files are in <project>\android folder. In the code I specify image URLs as 

url:'images/xyz.png'

This used to work in 1.7.5. I upgraded to 1.8.0.1 and images are not showing up any more.

I also tried placing the images in <project>\Resources\android\images\res-long-port-hdpi folder. Still it is not working. Any help is appreciated.

Thanks!

  • Correcting my previous post: start using "/".

    Ex: Use image:&#39;&#x2F;images&#x2F;header.png&#39; for an image in "…/project_name/Resources/images/header.png".

    This problem only occurred on android.

  • I was having the same issue with Android and 1.7.5 vs 1.8.0.1. Solved the issue…

    Works: backgroundImage:&#39;&#x2F;images&#x2F;square_button.png&#39;

    Doesn't work: backgroundImage:&#39;images&#x2F;square_button.png&#39;

    Doesn't work: backgroundImage:&#39;.&#x2F;images&#x2F;square_button.png&#39;
