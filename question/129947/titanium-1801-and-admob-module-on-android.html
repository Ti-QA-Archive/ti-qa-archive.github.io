  • For anyone who's having the 

    [ERROR] Error generating R.java from manifest
String types not allowed (at 'configChanges' with value 'keyboard|keyboardHidden|orientation|screenLayout|uiMode|screenSize|smallestScreenSize').

    error, all you need to do is add

    <tool-api-level>14</tool-api-level>

    to the "android" section of your tiapp.xml file:

    <android xmlns:android="http://schemas.android.com/apk/res/android">
    <manifest android:versionCode="1" android:versionName="1.0"/>
    <tool-api-level>14</tool-api-level>
</android>

  • version "2.0" indicates that the module is 1.8.0.1.RC3 compatible which means it will not work with pre-1.8.0.1.RC3 versions of Titanium SDK.

    so, if your tiapp.xml has the following declaration:

    <module platform="android" version="2.0">ti.admob</module>

    than make sure you're building your project using 1.8.0.1.RC3 SDK.

    according to the bev. blog post (see below), the same marketplace package should have version 1.x of the same module, which you can use with pre-1.8.0.1.RC3.

    in case you did not know about the breaking changes which were distributed several days ago, read this post:

    http://developer.appcelerator.com/blog/2011/12/upgrading-android-modules-to-1-8-0-1.html

    take a look at this one, also:

    http://developer.appcelerator.com/question/129904/if-youre-using-modules-from-the-titanium-marketplace—you-must-know-this
