Titanium appears to be masking my window setter method

I have created a Titanium.UI.Window object and defined a custom setter on it for a property I want to be able to set from elsewhere in my app. I've named that function "setWindowRecord". It is not being called as I'd expect via myWin.setWindowRecord(r). Instead, when I inspect the window, I see that typeof(setWindowRecord) is in fact a function. However, there is also now a property defined on the object named "windowRecord".

If I change the name of the function to "doSomething", it is called as I'd expect. So Titanium appears to be doing something magical with the setter I've defined. Is that behavior documented somewhere? I wasn't able to find it. Is it possible to opt out of that behavior?
