iOS handles relative images differently than Android?

Hi,

I have images in my Resources folder across different subfolders. In Android I can make a imageview like this:

var boardPreview = Ti.UI.createImageView({
    backgroundImage: "previews/cross.png",
    //backgroundColor: "#fff"
});

This script is in Resources/boards/cross.js. It's included in app.js
The stupid thing is that it works in Android, but on iOS the image is not found. Pointing it to a non-relative path works. Is there anything I can do to make it work on iOS? I don't want to use different paths for iOS and Android.

  • You can use Resource folder relative paths since 1.8.x. Put a / in front of all your paths and they will be relative to Resources

    var boardPreview = Ti.UI.createImageView({
    image: Ti.Filesystem.resourcesDirectory + "previews/cross.png",
    //backgroundColor: "#fff"
});
