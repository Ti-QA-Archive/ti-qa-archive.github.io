Custom fonts with Alloy and Themes

Type: Mobile
SDK: v3
Platform: iPhone iOS6
Host OS: OSX
TS: v3.0.1.2

I'm having no luck getting custom fonts to show up using Alloy.
I've made a copy of info.plist and added the fonts(otf and ttf) pathing them to /fonts directory. I have placed the fonts in my themes/[theme name]/assets/fonts folder. When I run it, I can see /fonts copies into the "Resources" folder which then reflects the path that is used in the info.plist. I've quit XCode and "Cleaned" the project in a number of different combinations too.
Not sure where to go from here.

Thanks.

  • Hello!

    Now with SDK 2.1.4+, it's possible to declare your info.plist into tiApp.xml with this sample code (I use it with SDK 3.0.0.GA / Alloy):

    <ios>
        <orientations device="iphone">
            <orientation>Ti.UI.PORTRAIT</orientation>
            <orientation>Ti.UI.UPSIDE_PORTRAIT</orientation>
            <orientation>Ti.UI.LANDSCAPE_LEFT</orientation>
            <orientation>Ti.UI.LANDSCAPE_RIGHT</orientation>
        </orientations>
        <orientations device="ipad">
            <orientation>Ti.UI.PORTRAIT</orientation>
            <orientation>Ti.UI.UPSIDE_PORTRAIT</orientation>
            <orientation>Ti.UI.LANDSCAPE_LEFT</orientation>
            <orientation>Ti.UI.LANDSCAPE_RIGHT</orientation>
        </orientations>
        <plist>
            <dict>
                <key>UIAppFonts</key>
                <array>
                    <string>fonts/Lobster1.4.otf</string>
                    <string>fonts/BebasNeue.otf</string>
                    <string>fonts/Entypo.otf</string>
                    <string>fonts/Bariol.otf</string>
                    <string>fonts/BAM.otf</string>
                </array>
            </dict>
        </plist>
        <min-ios-ver>4.3</min-ios-ver>
    </ios>

    It work for me with custom fonts in app&#x2F;assets&#x2F;fonts and for my custom widgets (add fonts in assets&#x2F;fonts of the widget) but I don't test it on theme folder but I'm sure it work if you have custom font on primary assets folder…
