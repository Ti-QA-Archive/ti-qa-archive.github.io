Paypal module button on iPhone always disabled, always says network error; works fine on simulator

iphone, ios5 / ios4.3, SDK 1.8.0.xxx, paypal module 1.0 / 1.3, Lion (osx 10.7)

When using Paypal module 1.0 or 1.3 on simulator with SDK 1.8.0.xxx (I've tried every version, including some 1.7's) everything works fine with all environments (None, Sandbox, Live). Perfectly, in fact.

When I try to run it on the device, everything works up until the paypal button, which stays disabled on Sandbox AND Live, and always gives me the error: 'Network error. Pay with PayPal not available. Please try again later.' It works fine with the None environment (Demo)

I've created my app on x.com and know that works correctly, because when I comment my appID out, I get the message that I have an invalid one. I know I have connectivity, and can put a check beforehand that says I have connectivity, even to paypal.com!

I'm at a loss at this point. I've read someone else's q&a that says it works with Snow Leopard but not Lion, but I don't have that available to me right now! Anybody have any idea about where to start with this? I've been at it for days straight and it's the last thing I need to do to get it in the app store!
