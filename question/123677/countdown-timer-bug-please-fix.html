Countdown timer bug. Please fix!

So, apparently Kenn from this post and I live in the same timezone. This bug returns a different offset per timezone! I released an app with a timer using the fix that Kenn released and that I then modified to fix for long times. But now I find out from bug reports that the offset being returned is different per offset!

Its not that each timezone is 1 hr different from the other. That would be easy enough to fix. They are off by odd amounts. So, for instance, in New York I'm off by some amount, and an hour over in Memphis, its off not an hour more, just 54 mins.

Spectacular.
