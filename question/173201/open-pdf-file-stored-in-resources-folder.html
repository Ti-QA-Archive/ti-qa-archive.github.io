  • Hi Shekar,

    Try below code..

    This works on ios..

    var win = Ti.UI.createWindow();

var strFileName = '/abc.pdf';  // pdf file path in resource directory
Ti.API.info('Ti.Filesystem.getResourcesDirectory()-->' + Ti.Filesystem.getResourcesDirectory());

var source = Ti.Filesystem.getFile(Ti.Filesystem.getResourcesDirectory(), strFileName);
if (source.exists()) {
    Ti.API.info('source.exists-->' + source.exists());
    var webview = Ti.UI.createWebView({
        top : 0,
        url : args.attach,
        scalesPageToFit : true,
    });
    win.add(webview)

}
Ti.API.info('source-->' + source);
win.open();

  • if your file is under resources folder then no need to set resources path just use file name to read file.

    
var win = Ti.UI.createWindow();

var strFileName = '/test.pdf';

var f = Ti.Filesystem.getFile(strFileName);
if (f.exists()) {
    var webview = Ti.UI.createWebView({
        top : 0,
        data : f.read(),
    //or
    url : f.nativePath,
        scalesPageToFit : true,
    });
    win.add(webview)

}

win.open();

    action : Ti.Android.ACTION_VIEW,
    type : 'application/pdf',
    data : f.getNativePath()

  • Actually, you need to consider permission sandboxing. If the pdf is in your Resources folder, it is sandboxed. Other apps do not have permission to access that folder. If you wish to use another viewer app to read the file, you will need to move the file to a location that the other app has has permission to read.
