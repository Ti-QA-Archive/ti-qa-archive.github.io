creating a context without a window / project structure / include paths

hi

im currently in the process of understanding how its best to structure a titanium (mobile) app.

im using this structure

  • Resources/components -> components created with helium
  • Resources/config -> stuff like helium psets (presets for objects)
  • Resources/contexts -> the contexts opened with url from windows
  • Resources/images -> all images
  • Resources/lib -> library stuff like helium and my own functions

now i have the idea that in app.js i call the contexts/main.js in some way so all of my code starts there. for example i want to build my tabgroup in main.js

the problem now is that all paths in my application need to start from Resources/contexts which of course does not work if i use the app.js or include main.js in the app.js.

solutions to this problem would be:

  • dont use the url parameter for windows and have everything in the same contexts -> not so great because stuff like Ti.UI.currentTab does not work without seperate contexts.

  • remove the contexts folder and put all files in Resources -> not so great. i like clear structures.

  • write some complex code that prefixes every path according to if im calling from app.js or not -> not so great either because i would have to call a function on each path i use.

  • create a seperate context for main.js without making it a window

the last 2 are the best solution i think and i guess i have to go with the prefixing but it would be great to make a special context for main.js.

so does this work or what are your takes on this problem?
