  • if you seek docs for "build app distribution" you will find nothing. The keyword is package. So you can find THIS

    So, in order to build an app for distribution you have to click over project explorer with right button and select publish.

    But publish is completely disable if you have not configure run&#x2F;debug settings.

    So, you have to click with right button over your project name and select properties and then run&#x2F;debug settings.

    Create a new launch configuration selecting Titanium Android App Store Distribution

    If you want to use the same private key, so that you can upgrade your current app, you should configure keystore location to something like .titanium&#x2F;mobilesdk&#x2F;linux&#x2F;3.2.1.GA&#x2F;android&#x2F;dev_keystore with the password tirocks and àlias` tidev.

    Now you can use publish option described above.

    Run it and you will have a production APK lying where you have defined the distribution location

    Hope that helps.
