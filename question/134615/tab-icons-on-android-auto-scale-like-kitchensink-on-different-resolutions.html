Tab Icons on Android, auto scale like kitchensink on different resolutions?

Hi!

I'm currently developing a cross-platform mobile App. I'm using a tabbed interface using tabgroup. Now my problem is that on Android the icons i use on my tabs seems to display using exact pixel size instead of the "virtual" pixel size.
Since IOS handles tabs a different way, no problem there.

On big screens the icons are very small, on medium they are just fine and on small screens the icons dont fit.

Just had a look at the kitchensink app and I noticed that the icons on the tabs actually do scale correctly using only ONE image.
I can't find any property that gets set to enable this in the kitchensink source.

What am I missing?

I tried the approach using different images using the Android/images/xxxx folders but it seems to be a big pain since i do NOT have higher resolution icons anyway and Kitchensink seems to work fine using only one image for all sizes.

    Hi,

    edit your tiapp.xml and replace the android element (at the end of the document) with the following:

        <android xmlns:android="http://schemas.android.com/apk/res/android">
        <manifest>
            <supports-screens android:anyDensity="false"/>
        </manifest>
    </android>

    For more information, read this.
