    Hi

    It sounds like you wish to make an app that simply show a website? If so then you have to be aware of a few things;

    • iOS will very likely reject your app as it breaks at least one of the requirements to publish and app, including; must not simply display a web site and must provide native device functionality beyond simply showing something simple.
    • Android will let anything in
    • Windows phone - is not an available option just yet - but will be later this year
    • I am not sure you users will be that interested in a app that can easily be reproduced on the actual website
    • Your users will require a constant internet connection to use the app - one of the benefits of the app is to lessen that requirement and make the whole experience much snappier and friendly
    • You only have to pay the annual fees you stated to publish an app

    You can create a window that displays a full screen window and only includes a single webView linked to one URL. That information can be found inside the documentation;

    • Window
    • webView

    As the code to achieve what you want is a handful of lines and is found on those pages I feel it would be a good idea to learn at least those few lines if your aim is to publish an app yourself.

    However I would strongly advice you to think of something more challenging to yourself and better designed for your potential users - if of course the store allow the app in.

    • Ideally you would create an API to list that information and display it in a tableView - update the list each time the app loads and then show a webView when any is selected to show that article.

      You should include features such as notification (a good one), have windows spawn new windows (as mentioned above), ideally you should make use of some of the other features - not for the sake of it, but you should try to tick off a few;

      • Camera
      • Compass
      • Network (geo)
      • Network (api)
      • Map
      • Push Notifications
      • Local Notifications
      • Multiple windows
      • Manage login details
      • Store configuration details

      Again - I am NOT suggesting adding any of these just to pad out the app, but you need to differentiate your app from a simple app that only shows a web page that could so easily have been accessed by a mobile browser.

      For example when you are using push notifications you could store the users preferences to what sort of push notifications they wish to be informed of, this would make a great window with a few labels each one with a switch and those choices stored to update the push service.

      If your app has some sort of premium access option then you could include an in app purchase option to gain that extra level - I do NOT mean to charge more for each new info, simply a way of saying this information is only available if you have paid the IaP of (example) £2 per app - otherwise you get the standard track of info. Likely this is not suitable but you get the idea of how features can be used to both increase the app functionality and at the same time show that your app is not a simple web page in disguise.

    • If you mean pay someone to do the app then good question, depending on the final features and need to learn third-party things (if there are any) could range from £500 (if you get a one man band trying to get into the game basically an amateur) to £20k or more. Likely somewhere in the middle.

      But still it is not that big an app to do yourself. If you have learnt c++ then structurally JavaScript is similar, the language you need to learn is identical to JavaScript (as it is JS) and has additional functions due to the Titanium API.

      The best way to start is to pick one thing to learn, start with adding a label and changing properties, then add a button and make it react to your touch. Add a tableView with some rows and then get each row click to open a window and pass data around from each row. Keep doing this for a short while and you have the basic parts of any app . From there it is working what you need to put together and when, fill in the extra things like switches and toolbars if and when you need them.

      This is how I started and I now build apps for very large companies. As with anything it will take some effort - but Titanium is very simple and rather forgiving - plus the feeling of success when your app does something you tell it to and you show a friend - so cool. You are creating something from nothing.
