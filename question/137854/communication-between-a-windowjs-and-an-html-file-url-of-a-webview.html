communication between a window.js and an html file url of a WebView

In a window of my app, I have a button that opens a new window with a webview. The url of this webview is an html file of the Resources folder. This button must pass the parameters that must be read from a script that is located in the html file. I tried with a fireEvent. In the simulator works, but no in device … you know why? This is my code:

win.js

var win = Ti.Ui.createWindow();
var readBtn = Ti.UI.createButton({
 title : "Read"
});
win.add(readBtn);

readBtn.addEventListener("click", function(){
    var newWin = Ti.UI.createWindow({
        title : "First Window",
        backgroundColor : "transparent"
    });            
    var winWV = Ti.UI.createWebView({
        height :  "100%",
        width : "100%",
        top: 0,
        url: 'read.html',
        backgroundColor : "transparent"
    });
    newWin.add(winWV);
    newWin.open();
    setTimeout(function(){
        Ti.App.fireEvent("read",{name : "Aaaaa", surname : "Bbbbb"});
    },2000);

});

read.html

    I just solved. The problem was in the path relative to a js file to include in the html file. The file is located in / Resouces In the html file, it also located in / Resources, I used 

    <script src="code.js"> </script>

    and worked in the simulator, but not in the device. I tried to change it so

    <script src="../code.js"> </ script>

    and now works in both the simulator and in the device :)
