Cross platform multi-threading within Titanium Mobile

What is the best way to write multi-threaded applications using Titanium mobile? I see some posts on creating multiple execution contexts by opening new windows. Is that the best way? Wouldn't they have a visible impact in the UI? How do I create a backend service which fetches data and fires events when there is new data in order to update the UI?

I am writing a IM client and having a seamless implementation of this functionality is necessary.

  • You have two options for background processing:

    • Use the setInterval() function to have the JavaScript environment call a polling function at intervals. The polling function could fire an app event with the new data or just a flag that tells the app that there is new data to fetch. This approach should work across platforms and is probably what you want to do.
    • Write a custom module that does the multithreading and fires events back to the JavaScript code. The downside here is that you need to write native code for each platform you want to support.
