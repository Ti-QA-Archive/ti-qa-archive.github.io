  • When create navigation group.

    Before

    var navGroup = Ti.UI.iPhone.createNavigationGroup({
    window : windowToOpen
});

var containerWindow = Ti.UI.createWindow();
containerWindow.add(this.navGroup);
containerWindow.open();

    After

    var navGroup = Ti.UI.iOS.createNavigationWindow({
    window : windowToOpen
});

navGroup.open();

    And when you stack new window.

    Before

    navGroup.open(newWindow);

    After

    navGroup.openWindow(newWindow);

    navGroup variable may be renamed to navWIndow.^^

  • Regarding why the navigationWindow is a top level object and NavigationGroup is not, that is due to the fact that they represent different UI Objects. The NavigationWindow is a Window while the Navigation Group is a View (which can be added to other Views/Windows). Maintaining the NavigationGroup has become increasingly difficult since we can not reliably focus/blur windows (and as such maintain accessibility elements) and can not guarantee that rotation and appearance methods will be forwarded to managed ViewControllers (and you really need this for some ViewControllers to behave correctly).

    Regarding why the ability to add Windows as subviews of other objects was removed, it was a conscious design decision. One it lets us support iOS7 and future versions more easily, two we really did not want Windows behaving as Views since Windows do a lot more than displaying content on screen (Status bar appearance, orientation, accessibility features to name a few)

    We have notified our module developers of the architecture change in base Titanium Windows and hope they will update their modules as soon as possible.

    Please note that while the NavigationGroup class is deprecated in 3.1.3 GA, it is still a supported API on this release (though we do not plan to add any new features to it). The API will be dropped in 3.2.0 GA release. That should give you sufficient time to migrate your applications.

    As always, your feedback is appreciated.

  • Arjan, I made a new project and then used the app.js example file from v1.3.2 of the module. In this example I simply changed the navController from create NavigationGroup to createNavigationWindow on line 135 in the example.

    Before

        var navController = Ti.UI.iPhone.createNavigationGroup({
        window : win
    });

    After

        var navController = Titanium.UI.iOS.createNavigationWindow({
        window : win
    });

    Everything seems to be working as normal.

    I hope that this helps.
