  • Accepted Answer

    The issue is that any Titanium Mobile SDK less than 2.0.0 does not support the "Distribute - Ad Hoc" function. Titanium Studio should prevent you from using that option if your SDK is set incorrectly.

    If you need to do an Ad Hoc distribution on an earlier version, the Xcode approach is the easiest way.

  • For lesser than 2.0.0 Titanium SDK ad-hoc distribution :
    1) Click on publish menu, select publish to itunes option,
    2) Distribute window will pop up configure your distribution, provisioning certificate and SDK version
    3) Now system will ask for user name and password for system to access keychain(Dont panic if it asks for more than once)
    4) X-code will open automatically, from corner of window go to organizer menu and then archive menu.
    5) Your new ipa will be visible there
    6) Now distribute it with the option in right it will ask for three options select ad-hoc for deployment
    7) Select next to give code signing authority on next it will ask for location to save ipa
    8) Thats it drop the ipa in itunes under apps sync your iphone and drag the ipa to iphone's app in itunes.
