  • There is something known as "tweetanium coding style", that refers to the way tweetanium is coded. In this app, all style coding is isolated on a style.js file (pure javascript, not jss). There isn't a clear response about which is better (jss or tweetanium style), but I think is important to know both and choose in each case.

    If you don't know tweetanium, I suggest you to do it. Also, are other conversations about this subject, like this http://developer.appcelerator.com/question/122778/titanium-ans-jss

    http://developer.appcelerator.com/blog/2011/01/tweetanium-mobile.html

  • I have a pair of iPad apps completely done from an indesign document, with buttons, bullets, layers, animations and so on. Works fine and is not very difficult. You have to convert each piece to png, as you would to build a web from PS.

    Was easy because the app was only for iPad in landscape orientation, but a multi-platform/multi-screen/multi-resolution app would be probably a lot harder.
