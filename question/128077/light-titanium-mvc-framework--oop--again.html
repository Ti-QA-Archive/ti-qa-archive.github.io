Light Titanium MVC framework + OOP ... again

Hi everybody,
I apologize for asking a question already posted a lot of times but I already red all the posts and tryed all the framework suggested in the answers (Backbone, PureMVC, Adamantium, mcv,… ) without really finding what I am searching for.

I am looking for a lighweight MVC framework to better structure my Titanium Mobile applications.

I think my requirements are very common:

  • I want to have a framework that extendly uses the CommonJs pattern
  • I need to have a hierarchical controller structure with a main controller that take care to route actions to sub controllers
  • A controller must be able to easily have a reference on every element of the view and its model
  • I need an OOP approach that enables me to multi instantiate a view in different context and attaching it to different controllers
  • I need a navigation controller that easily integrates with the main application controller and can access and modify the stack of the open windows

I think that more than just the MVC framework itself I need some architectural patterns and blueprint to apply to my apps.

Right now I developed a pretty large app, refactoring it a couple of time according with the increasing amount of info (community, tutorials, …) that I continously gather around but I still experience some memory leaks, unpredictable crashes and so on.

Does anybody have some suggestions to share? I also read somewhere that a solution on this matter (mvc + oop patterns) is in development from the Appcelerator team. Am I wrong?

Thank you

Regards,

Alessandro

  • Hi Alessandro. I don't know is anyone from Appcelerator working on OOP + MVC features, but it would be nice to have something like that. If they are, then it's pity they didn't ask people from community for ideas and feedback.

    Anyway, in the past there was couple of ideas how to bring OOP feature to the Titanium and most of them are similar ideas (you can search Q/A with new search mechanism, I'm sure you'll find interesting stuff). You can see basic idea here (idea is to wrap Titanium object with JS native ones):

    https://gist.github.com/1333884

    Here is one more idea (mine :) ):

    http://pastie.org/2799582

    it's basically the same idea, but mine solves the problem of getting particular element (without using slow linear search through all elements of some component).

    To build (M)VC, you need to separate component creation logic and functional logic. So, in view part you just create dummy UI look, in controller part you add functionality (using events) to that dummy UI look. Simple as that.
