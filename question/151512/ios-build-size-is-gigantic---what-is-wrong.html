iOS Build Size is Gigantic - What is Wrong?

I've successfully submitted several apps to the iOS and Android stores. My most recent iOS submission was rejected because the file size was too large. It was over 148MB! I couldn't believe it because my app has very little actual content (most of the heavy lifting is done by downloading and playing videos after the app is installed).

I went back to look at the file size of my app. My total app is less than 14MB. Every time I clean my project and then build it for local and/or distribution the build folder grows exponentially. I removed cloud features, etc… and I've gotten the build down to 90MB but that's still too large for my liking. Why is the build so large if my entire Titanium folder for the app is less than 14MB.

I'm using Titanium Studio 3.0.2 on a Mac Book Pro OSX 10.7.5. I'm using TItanium SDK 2.1.1GA. It is a universal iPad/iPhone app.

Please help!

-Tim

  • Hi

    What is the size of the {appname}.IPA file when you build for device?

    Make sure the only files and folders inside your app folder are related to the app directly and not things that helped you make the app like; PSDs etc.

    If you have any folders at this level move them out, make sure you ONLY have things you need inside the resources folder.

    Always "Clean" before builds for this SDK version and physically check the folder is cleaned. Close the simulator before cleaning of course.

    Make sure you have removed any modules from your local app folder if you are not using them, plus any old versions (should not be required but just-in-case).

    Your app folders normally will not have more than these root folders (may not have all of these)

    • {app name}\app
    • {app name}\build
    • {app name}\i18n
    • {app name}\modules
    • {app name}\platform
    • {app name}\plug-ins
    • {app name}\resources
