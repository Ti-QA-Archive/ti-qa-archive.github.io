Windows by url in SDK 3.2.0

Hi there,
I installed SDK 3.2.0 then I re-builded my app for Android.

I understand that url in windows generates the crash of my app.

The code organization is very easy, and this works perfectly in old sdk release (also for ios)

Titanium.UI.setBackgroundColor('#000');

// create tab group
var tabGroup = Titanium.UI.createTabGroup();

//
// create base UI tab and root window
//
var win1 = Titanium.UI.createWindow({  
    url:'home.js',
    exitOnClose: true,
    title:'Cerca con agriturismi.it',
    backgroundColor:'#fff'
});
var tab1 = Titanium.UI.createTab({  
    icon:'/images/home.png',
    title:'home',
    window:win1
});


var win2 = Titanium.UI.createWindow({  
    url: 'map.js',
    title:'Near',
    backgroundColor:'#fff'
});
var tab2 = Titanium.UI.createTab({  
    icon:'/images/compass.png',
    title:'Near',
    window:win2
});


//
//  add tabs
//
tabGroup.addTab(tab1);  
tabGroup.addTab(tab2);  

// open tab group
tabGroup.open();

I read that "url" in createWindow is deprecate, the question is:
I must rewrite my all application in alloy? There is a easy way for correct this simple code organization?

PS: In the official documentation, this code is accepted http://docs.appcelerator.com/titanium/latest/#!/api/Titanium.UI.Window

  • You dont have to use Alloy, you can still use 'classic' Titanium code, however there are many benefits from using a MVC architecture for your app (which Alloy provides). What you need to do is go through the App windows that are currently being referenced in the url property and make them commonJS modules that return a Window object you can then reference in your TabGroup or open directly. There's no 'easy' way of doing this, you just have to take the hit and get it done. It will improve the stability and readability of your App code.
