    Coming with SDK 3.3, is a new event on the ListView, noresults which you can listen for and perform appropriate actions when the set is empty - i.e. add a no results row.

  • Hi

    Personally I prefer do add a view equal to the height of the table (or different, change to suit needs) with a label inside as the footer view. This is then set to the table as the footer view when there are no rows/items to add to a TableView or ListView.

    This works in conjunction with any existing footer view as you simply swap them out as needed. This has an additional advantage that anything else the TableView or ListView needs - like a header view - can persit as the it is still visible.

    Ultimately - it is personal preference. Nicks suggestion of Hiding/Showing solves the problem as well.
