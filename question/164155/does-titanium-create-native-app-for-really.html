Does Titanium create native app for really?

Hi folks, just a simple question: if Titanium creates native app it means that, in the case of Android, Titanium generates a real APK file similar to what I can get using only the Google stuff. So, if it is correct why in the APK file created by Titanium Studio there is a javascript intepreter? (a very huge file that increases enormously the size of the APK) Why I need a js intepreter if the native app is in Java? So it is not true that Titanium creates native app because part of the program remains in Javascript… How this stuff works? Thanks to everyone who can clarify this point.

— asked April 2nd 2014 by mario rossi

  • Hi,
    In Titanium JS code is compiled almost one to one into the representative symbols in nativeland. There's still an interpreter running in interpreted mode otherwise things like dynamic code wouldn't work. However, its much faster, much more compact and it's about as close to pure native mapping as you can get.
    For this reason the titanium compiled api is a bit larger than apk build using bogle stuff only.

  • Ti generates a native app. Everything related to the UI and other native capabilities (like gps, media, etc, well everything) is native.
    However, the logic of the app is in JS ( like click that button it opens that window), so this is why you need the V8 JS engine behind.
