getDirectoryListing() on an iPad returns blank

Hi all, I'm wondering if there is a different way to call getDirectoryListing() on an iPad? On the emulator and on an iPhone, it returns the correct directory listing, but on an iPad it returns blank. It doesn't even return an empty directory listing, it just returns absolutely nothing. Not even false. I've tried it on 3 separate iPads just in case, one with iOS5, two with iOS6.

Here is my code:
Works perfectly on Emulator and iPhone, not on iPad

// this path does exist, if I alert using .exists() it returns true
var dir = Ti.Filesystem.resourcesDirectory + "data/apps/icons";
var dir_list = Ti.Filesystem.getFile(dir).getDirectoryListing();

alert (dir_list);
// returns a listing on emulator and iPhone,
// returns blank on all iPads
