Using TiQuickGame2D. Strange behavior when changing rotation center

I've got a problem using this module when I rotate a sprite indicating the rotation center and after that I rotate it again using another rotation center:


// quicktigame2d is the module object, previously loaded

var mySprite = quicktigame2d.createSprite({
    image : 'graphics/image.png',
    width: 2000,
    height: 2000
});

// Rotate to angle 30, rotation center  (150, 150)
mySprite.rotateFrom(30, 150, 150);


// Now we make a new rotation from a different center (80, 80)
mySprite.rotateFrom(90, 150, 150);

Doing that, when the second rotation is applied the sprite seems to change its position (apart from the rotation itself). If I check the x and y position they are OK, they haven't changed, but the sprite 'seems' to have moved.

I've tried many ways to do it with the same result. I've also done it using the transform object, clearing the tranforms and creating a new one:

// quicktigame2d is the module object, previously loaded

var mySprite = quicktigame2d.createSprite({
    image : 'graphics/image.png',
    width: 2000,
    height: 2000
});

var transform = quicktigame2d.createTransform();
// Rotate to angle 30, rotation center  (150, 150)
transform.rotateFrom(30, 150, 150);
mySprite.transform(transform);

// Clear previous tranforms
mySprite.clearTransforms();

// Create a new transform
transform = quicktigame2d.createTransform();

// Now we make a new rotation from a different center (80, 80)
transform.rotateFrom(90, 150, 150);
mySprite.transform(transform);

but it did not work.

I'm not sure if I'm doing something wrong or is a bug.

Thanks
