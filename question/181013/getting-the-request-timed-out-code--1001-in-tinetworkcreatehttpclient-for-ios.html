    • I don't think it's either, actually. If CWP isn't even showing an outgoing request, then for some reason Titanium isn't sending a request. It would be highly unlikely that you have found an error in httpconnection by making just a basic POST. It's more likely a variable scope issue in your code. However, with your real code, that would be hard for any of us to diagnose.

      Make sure that this.url actually has the value you expect. If no request is sent out AND onerror is getting triggered, I would think that Titanium can't send a request, and the URL seems the obvious culprit, right?
