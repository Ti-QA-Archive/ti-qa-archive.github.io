Android - Button Background and Border Colors Reversed

Hello

I'm having some weird issues with the button properties. It seems as though the button border colors and background color properties are reversed in titanium. Could someone else confirm this?

Here is my code:

   var btn1 = Ti.UI.createButton({
       backgroundColor: '#white',
       backgroundImage: 'none',
       borderColor: 'blue',
   });

   win1.add(btn1);

When it displays, the background color is blue, and the border is white!. The colors work on iphone, but not Android. I'm using mobile sdk 1.7.0, but it doesn't work on older versions either…

    "Happy ness", or may I call you 'ness? ;)

    The following issues are causing the behavior you have observed:

    • setting the backgroundImage property any value, even "none", will negate the backgroundColor. Be mindful of the fact that Titanium object properties do not work in the same way as CSS properties
    • '#white' is not a valid color. Either use 'white' or '#fff'
    • to use borderColor, you must set a borderWidth, otherwise the border will not be visible

    The following code works as you expect:

    var button = Titanium.UI.createButton({
  backgroundColor: 'blue',
  color: 'white',
  borderColor: 'red',
  borderWidth:2,
  title: 'Hello',
  top:20
});

    Hope this helps
