Non-English chars in iOS develop certivicate trouble

Hello,

My iOS develop certificate contain Russian chars. When I try to upload my application to iPad it shows error:

Code Sign error: The identity &#39;iPhone Developer: \320\224\320\265\320\275\320\270\321\201 Degtyarev (PP5JBTZ87D)&#39; doesn&#39;t match any valid, non-expired certificate&#x2F;private key pair

My certificate and key are configured right and valid. Xcode compile and upload apps to iPad with that certificate without errors. Trouble is in Titanium studio.

By the way previous versions of Titanium shows my certificate correct (with russian chars, not with \320\224…)

Is there any way to correct my iOS develop certificate name? Or sign code from command line? Or compile, sign and upload to device at once from comand line?

Titanium Studio, build: 3.0.1.201212181159, Titanium SDK version 3.0.0.GA, iOS SDK version 6.0

  • Hi Denis,

    this is a known problem.

    Other developers have also noticed this as well:

    • Umlauts in iOS Provisioning Profile (here you will find the installation of bug fixing)
    • Issues with Ti 3.0 Studio + 3.0 SDK

    The problem is documented in jira (CLI: Developer names with Chinese characters appear as octal escape sequence in Titanium Info) as error and marked as resolved.

    In the first point of my list (Umlauts in iOS Provisioning Profile) is the link for, how to do a bug fix installation for node_appc.

    Please read very carefully to the end, the conversation by Chris Barber and Florian Bühlmann. There will be explained, how the installation is carried out

    Hope that helps,

    Matthias

    • Hi Denis,

      also this seems to be a known problem.

      • Appcelerator Developer Blog - Titanium Studio 3.0.1
      • iOS: Unable to select mobile provisioning profile
      • Have Error : Unable to find any iOS Distribution Certificates on Export ipa file step
      • iOS Development Certificate is NOT found (Ti 3.0 Studio, SDK 3.0.0.GA)

      So this is either a python or SDK problem.

      The keychain problem (last point from list) should have been fixed with Titanium Studio 3.0.1 already.

      You can download and install the latest SDK CI build from this site: Continuous Builds - Mobile Builds.

      Or you use the Titanium CLI to download/update/install the latest SDK branch.

      titanium sdk update --branch 3_0_X --default

or

titanium sdk install --branch 3_0_X --default
