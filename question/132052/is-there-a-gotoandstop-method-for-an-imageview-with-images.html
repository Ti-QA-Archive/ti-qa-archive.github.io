Is there a GoToAndStop() method for an ImageView with Images

I've got a sequence of images loaded up in an ImageView. I'd love to be able to say:

editView.goToAndStop(5); //shows 005.jpg
editView.goToAndStop(542); //shows 542.jpg

I only have 30 images, but need the user to be able to touch and drag a virtual slider on the screen and "play" the correct image based on where they're currently touching. Perhaps switching in and out the image property on the ImageView is better, but I thought an image sequence would be more efficient.

Any thoughts? Cheers!

actual code would look something like:

    editView.addEventListener("touchmove", function(e){
        editView.goToAndStop(e.x);  //math.round function omitted for clarity
});

  • Hello Forrest,

    There are already multiple options available to achieve this. Why you want do everything in Image view. Here are available options you can go for that and you can write your a little logic to achieve what you want,

    1) TableView : Table View having property scrollToIndex which allows you to scroll to a specific row. You can fill your table view's row with images and you can do your job.

    2) ScrollableViews : having scrollToView property which will allow you to jump to a particular indexed view.

    3) ScrollBar : having scrollTo property which allows to jump to particular x , Y position.
