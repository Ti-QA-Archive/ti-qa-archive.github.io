    When having an apparent problem, it is best to tell us the SDK level you are trying to use. Also more info than "did not work" would be really helpful. We are just shooting in the dark here.

    But since you mention it is a mp3 file, one possibility is the size of the file may be creating a memory overflow issue if you are trying to save the file using the f.write(xhr.responseData); method in your code.

    With iOS it is easy to avoid that problem using the "file" property of the HTTPClient object. However it is not supported in the android version of the HTTPClient, at least as of SDK 1.7.2 anyway.

    A user did come up with a method to achieve the same end with android however, and it is described here. Try integrating that method into your code and see if it solves your problem. If not, please tell us the SDK level you are using, the relevant portion of your code, and what errors or symptoms are occurring.
