Building Android modules for a certain SDK 'level'

I've been looking at the Android module development guidelines here : http://wiki.appcelerator.org/display/guides/Module+Developer+Guide+for+Android

.. and have been able to create a new template module fine. But does anyone have any idea how I can set it to target a higher 'level of the Android SDK?

Even though I have platform levels 2 to 13 installed, the build.properties file created by Titanium seem to only be using level 7


titanium.platform=/Library/Application Support/Titanium/mobilesdk/osx/1.7.2/android
android.platform=/opt/android-sdk/platforms/android-7
google.apis=/opt/android-sdk/add-ons/google_apis-7_r01

I need it to be at least level 8 for a module I'm trying to create.

Is this possible? Or is level 7 the highest that Titanium Android modiles support?

I edited my ~/.bash_profile to make sure that titanium is using the latest 1.7.2 Titanium SDK, but cannot see any way to target the Android platform level I need.

Anyone have any ideas?

thanks.

  • Seems like you need to specify it via the tiapp.xml file, like this.. 

        <android xmlns:android="http://schemas.android.com/apk/res/android">
        <manifest android:versionCode="140" android:versionName="1.4.0">
            <uses-sdk android:maxSdkVersion="11" android:minSdkVersion="4" android:targetSdkVersion="8"/>
            <supports-screens android:anyDensity="false" android:largeScreens="false" android:normalScreens="true" android:smallScreens="false"/>
        </manifest>
    </android>
