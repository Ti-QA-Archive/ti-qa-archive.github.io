TaffyDB or SQLite?

In my Ti App I need to have use a database. Most of the DB accesses will be to read data with very few writes. The query I need to use is fairly complex. The SQL equivalent would be

SELECT name FROM table WHERE (cellVal = 'cellvalue') AND ((abs(cellX - x) < limX) AND (abs(cellY - y) < limY) LIMIT nRecords

Basiclly, box cellX,cellY values in a rectangle where the cellVal is cellvalue and return a maxium of ten results.

The DB size in JSON is currently around 20Kb. I don't expect it to get any bigger than 100 Kb.

My questions

a. Am I better of using SQLite for the job or can I use Taffy (I do rather like it)?
b. If I do use Taffy will I be able to store the DB data locally with Ti.App.Property.setString or will this get too tricky with a DB that is as big as 100 Kb.

I'd much appreciate any help.
