Best practice for event processing between pages

I have a data entry screen that can open other windows - eg. clicking on "Category" opens another window which has a table of categories:

Window A:               Window B:
-----------             -----------
Category       ----->   Category 1
-----------             -----------
Date                    Category 2
-----------             -----------
...

What's the best way to pass the selection of a category in Window B to window A? That is, when the user clicks on "Category 1" in Window B, closing Window B and going back to A, how do I tell window A to update what it displays for Category?

One option would be to manually set the value of Window A's category from within Window B's event listener, but that would require me to know window A's category variable. Is there a way to do this cleanly so I don't have to know about a global variable name?

I'm looking for something like a return value that's passed back when a window is closed, or a custom event that can be used to trigger user-defined updates.

    Place the creation of Window B in Window A.
    When Window A receive a click in tableView (or what you used) execute the creation of Window B and set a custom property for the new win, for example window_b.type='category'.
    Than, when window B will closed you can use this code:

    Window B (place that in the close event listener of your window)
    Ti.App.fireEvent(&#39;updateCategory&#39;,{type : win.type});

    In Window A

    Ti.App.addEventListener('updateCategory`,function(data){
    alert(data.type);  //<- this will contain "category"
}

    It's ok?
