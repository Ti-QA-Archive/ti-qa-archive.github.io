Can't Load Apps on Android (em or device) with Ti Studio and 1.7

I was running apps fine on both iOS and Android prior to upgrading to Ti Studio and 1.7 (on Mac OS 10.6.7). iOS still works fine. Below is what I've tried and some of the messages I get. Any help would be much appreciated!

I've tried:

  • Updating all Android SDKs, APIs and Add-ons
  • Deleting all AVDs and allowing Ti Studio to create one (it seems to create a new one whether you had any already or not)
  • Running the Android native sample apps outside of Ti - they work (i.e., my core Android install seems to work)
  • Updating Ti Studio and SDK (last night's pushed updates)
  • Rebooting several times
  • Tried creating a new Ti project, tried running with KiSk 1.7, tried running my prior working apps

    Brian,

    Your error usually happens when you are not using the Google APIs of the Android SDK. When selecting your Android version, the dropdown will have 2 different types of selections. "API 2.x" and "Google API 2.x" … you need to choose the Google version of the APIs in order to run some functionality in your app.
