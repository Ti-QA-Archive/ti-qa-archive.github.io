Acceptable TableView Add Parameters

The documentation for adding to the TableView states that it accepts "the view to add to this views hiearchy".

Does this mean that any "view" object, such as another TableView, can be added to the TableView via the add method?

  • Joe

    A moving UI object on top of another moving UI object, such as a scrollView on a scrollView or a tableView on a tableView or a combination of the two, is very difficult for the user to navigate, and therefore should be avoided.

    Have you considered using a Titanium.UI.TableViewSection instead?
