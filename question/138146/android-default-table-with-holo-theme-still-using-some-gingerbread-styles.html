Android: Default table with holo theme still using some gingerbread styles

Run the code and notice that when you click the table row you have a orange glow instead of a blue glow.
Please fix this so that the orange glow is not hardcoded into the SDK.

app.js

var win1 = Titanium.UI.createWindow({  
    title:'Tab 1',
    backgroundColor:'#000'
});

var opttable = Ti.UI.createTableView({
    data: [ {title: 'Option1'}, {title: 'OPtion1'}, {title: 'Option3'}, {title: 'Option4'} ]
});

win1.add(opttable);

win1.open();

tiapp.xml
you need to change the name and label of the activity depending on your project

    <android xmlns:android="http://schemas.android.com/apk/res/android">
        <tool-api-level>11</tool-api-level>
        <!-- This tells us what Android API version to compile against -->
        <manifest>
            <application>
                <activity
                    android:configChanges="keyboardHidden|orientation"
                    android:label="test_code"
                    android:name=".Test_codeActivity" android:theme="@android:style/Theme.Holo.NoActionBar">
                    <intent-filter>
                        <action android:name="android.intent.action.MAIN"/>
                        <category android:name="android.intent.category.LAUNCHER"/>
                    </intent-filter>
                </activity>
            </application>
        </manifest>
    </android>
