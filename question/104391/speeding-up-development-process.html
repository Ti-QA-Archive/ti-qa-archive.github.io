  • As strange as it may sound for Android development, I find development much faster to code, debug, and test most of the project targeting the iPhone, then periodically build for Android and just tweak what needs to change. This is true even when I don't need an iphone version.

    Advantages to prototyping in iphone:

    • More *.js lint warnings detected which could point out unintentional typos or bugs in your code
    • Simulator starts WAY faster than android emulator
    • Simulator runs code much faster
    • Aside from app.js, external *.js context windows can be tested dynamically

    Let me explain the last point because it is the source of a MAJOR improvement in productivity and the crux of why I develop/debug for iphone even when I want an android target.

    For an iphone simulator build, the .js files are run directly by the simulator without going through the compile step needed for a distribution build. While this saves some time by itself, the real advantage is that the simulator will dynamically use whatever changes you make to a .js file when you navigate to a window using an external .js reference (i.e. the url property). So changes to app.js still need to relaunch the project. But for windows opened later, you can navigate to the window to see how it looks or test code, then just hit the back button in the navigation bar, tweak the .js, and navigate back to the desired window and immediately see the new layout or test the code changes.

    This makes tweaking UI layout stuff incredibly fast compared to the android emulator, not to mention code/debug cycles for some *.js logic is as quick as backing up a screen, revising the code, and showing the screen again. Then when you get the logic worked out, switch to android and retest.

    Obviously, this does not work for android specific portions of the SDK and the UI layouts are not exactly the same, but IMHO can still lead to some significant increases in productivity.

    YMMV, but I thought I'd toss in what could sound like a completely off topic suggestion you may not have considered.

  • Frank

    JS validation is highly recommended when working with Titanium, and as an android developer it will provide your biggest time-saver. To enable Javascript validation for eclipse, you need to install the wtp plugin (Web Tools Platform).

    The time it takes to boot an Android emulator or launch an app to an emulator is almost entirely dependent on Google's SDK. However, with this in mind and to give you an example for comparison, on my Ubuntu (i7 processor, 6GB memory) machine it takes 40 seconds to launch an application to an already-booted emulator, as this video demonstrates.

    The adb process should not "hang every few minutes causing me to kill the process and restart".
