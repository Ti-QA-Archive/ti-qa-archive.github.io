Titanium framework consistency

Hi all.

This is related to this issue: http://jira.appcelerator.org/browse/TIMOB-4945 but can be applied on number of other issues.

Now … what bugs me is that change in behavior is considered normal in minor releases. Before SDK 1.7.2 we had one behavior, 1.7.2 (minor release) introduced different behavior. Can anyone explain to me is Titanium versioning different from common practice?

Also, I have concern about informing users about this kind of changes. The thing is that I (and I suppose others also) think this is a bug and for every this kind of situation, I need to isolate the issue so I can report it. Sometimes this is trivial thing to do, but most of the time it is very hard thing to do because applications can be very complex and debugging options are very limited. So, I spend a lot of time in isolating bugs and, regarding that, I think that it would be nice if Titanium core developers would be more informative about changes in behavior. JIRA is useful, but it's hard to track important changes.

    • Paul,

      as I said, this behavior changed after 1.7.2 release.

      I did some testing now and I've found some new info about this issue, but the principle is the same. I discovered that "modal" property has influence on navbar, not "fullscreen". Anyway …

      Before, setting "modal" property to "true" would hide navbar, after 1.7.2 release you needed to explicitly define "navBarHidden" property to hide navbar.

      Test with this example (Android 2.2, Windows XP):

      var win = Ti.UI.createWindow({
    backgroundColor: '#fff',
    modal: true
});

win.open();

      1.6.3 -> navbar hidden

      1.7.1 -> navbar hidden

      1.7.2 -> navbar shown

      It's like default setting changed.

      There are more examples, but I don't have time to find them and present them to you.

      My other concern is this:

      1.7 release: http://developer.appcelerator.com/blog/2011/06/introducing-titanium-studio.html

      blog post about new layout mechanism: http://developer.appcelerator.com/blog/2011/06/new-defaults-for-android-layouts-in-1-7.html

      See the dates … 11 days users didn't know about new layout mechanism in Android. Anyone who upgraded SDK did have "WTF" moment. I think blog post about new layout mechanism should be posted before SDK was released, not after so people can prepare and make changes in their code if they are necessary. That would save a lot of developer's time and nerves.

      See other software products and frameworks and how are they releasing versions, this is common practice, I'm not asking nothing that is strange or new.

      I hope that now you understand my concerns. :)
