Android emulator won't install application

I'm having a hard time with this, I just updated the Android SDK (SDK Tools 22.0.1) and now all my apps that I use to run on the Android emulator won't install.

Setup:

  • Ti CLI: 3.1.1
  • Ti SDK: 3.1.1.GA and 3.2.0.v20130625113003 for testing
  • Ti Studio : 3.1.1.201306112235
  • node : v0.8.22
  • Android SDK: from 2.2 to 4.2.2
  • Mac OS X 10.8.4

Using the CLI and Ti Studio I tried to run the application, the emulator open just fine, but the apk is never created or installed.

I tried with different configurations for the emulators and different Ti SDK's, but the result is the same.

Also I tried using the -b option on the CLI and it tries to open the emulator and don't build the apk file.

This is a log from Ti Studio: gist link

As an extra information, I tried with the CLI using the legacy mode --legacy and the apk is created and installed in the emulator and I don't have right now any devices to test directly.

    Hello Reymundo,

    The error "hwcomposer module not found" leads me to believe your emulator target might be too high for your application. Check that your emulator's API level is <= android:minSdkVersion in your manifest, this should solve your problem. The update to the new API might have automatically set your emulator's API to too high a level. Let me know if this solves your problem.

    Regards,

    Carter
