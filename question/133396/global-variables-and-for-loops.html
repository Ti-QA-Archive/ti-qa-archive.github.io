Global variables and for loops

I have a xml file that I am parsing through and then putting the contents into global variables. Everything is working fine until I reach a section that has multiple values within its namespace. If I remove the TI.App line, I can loop through the XML fine and set standard variables. It's just when I try and set them as global. Can someone tell me what I'm doing wrong?

XML

<timetaken>
    <time>
        <Date>01/02/2012</Date>
        <Hours>8</Hours>
        <Reason>Holiday</Holiday>
    </time>
    <time>
        <Date>01/16/2012</Date>
        <Hours>8</Hours>
        <Reason>Holiday</Holiday>
    </time>
    <time>
        <Date>02/14/2012</Date>
        <Hours>8</Hours>
        <Reason>Birthday</Holiday>
    </time>
</timetaken>

  • Invalid property id (for line)

    • I don't understand what are trying to do. But it seems that you need an array, instead of object.

      var data = [], dataObject;
for (...) {
    dataObject =  {};
    dataObject[_date + c] = item.getElementsByTagName("Date").item(c).text;
    ...
    data.push(dataObject);
}

      To implement globally available data, I recommend that you write simple CommonJS module that does that (CommonJS modules are singletons and stateful):

      //global.js
module.exports = Object.create(null); //creates empty object

      and then you can "require" it where you want to use it:

      var globalModule = require('global');

globalModule.Something = 42;


//somewhere else 
var globalModuleStoredInOtherVariable = require('global');
globalModuleStoredInOtherVariable.Something; //42
