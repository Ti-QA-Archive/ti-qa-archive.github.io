Titanium sdk 3.2.0 - problem with tab icon

this morning I upgrade my titanium to the latest version (sdk 3.2.0) but i find a bad surprise: my app doesn't display the tab icon anymore.
Others images still work, so it shoud not be a folder issue.
this is the code:

<Alloy>
    <TabGroup>
        <Tab titleid="homeTab" icon="/home.png" id="homeTab">
            <Require type="view" src="intro" id="homeID"></Require>
        </Tab>
        <Tab titleid="mapTab" icon="/map.png" id="mapTab">
            <Require type="view" src="map" id="mapID"></Require>
        </Tab>
        <Tab titleid="settingsTab" icon="/settings.png" id="settingsTab">
            <Require type="view" src="settings" id="settingsID"></Require>
        </Tab>
        <Tab titleid="aboutTab" icon="/about.png" id="aboutTab">
            <Require type="view" src="about" id="aboutID"></Require>
        </Tab>
    </TabGroup>
</Alloy>

Yesterday tabs are displaied fine, please help me!
I'm working on an Android device.

  • I'm not using alloy, but I have similar problem.
    If I set navBarHidden to true (in the createTabGroup), there is no image.
    The tabs fit in the screen (the title is animeted and scrolling in the slected tab)

    But if the navbar is shown, the tabs can be scrolled to left, and right, (no animated scroll on selected tab) and there is images in front of the title. (and there is an empty navBar above the tabgroup)

    In the 3.1.3 SDK I have navBarHidden set to true, and the tabs fit in the screen with animated scrolling title (if not fit on the tab) and the images located above the title.
