  • I have been having the same problems. Here is how I managed to get the new version on my app on my phone:

    1. Click Install in the Run on Device tab as usual
    2. Shout expletives and such at your computer when the error message appears. Then ignore it.
    3. Open iTunes
    4. File > Add To Library
    5. Go to the root of the mobile repository
    6. Open /build/iphone/build/Release-iphoneos/YourAppNameHere
    7. Click Replace when prompted in iTunes
    8. Start Sync

    No clue how to fix the underlying source of the bug, though. I've reported it as a premium support ticket so we'll see if they pay attention.

    Credit: Jonatan Melzer's Answer to "Problem Installing to Device (iTunes)"

  • I'm having the same problem, iOS SDK 4.2 and Titanium 1.5. Did any of you guys just installed a new module? The problem first started for me just after registering a Titanium module (in Application Support/Titanium etc), I'm not sure if its a related issue or not. Emptying the build/iPhone folder did not help me either.

    EDIT:

    Ok guys I managed to fix it, it seems to be related to tiapp.xml, here's what I did:

    1. Create a new empty mobile Titanium project.
    2. Go to the new project's folder and copy tiapp.xml
    3. Go back to your app's folder (the app that presents the build problem)
    4. Rename tiapp.xml to tiapp.xml.temp and copy new app's tiapp.xml
    5. Build again, your project will work now.
    6. Delete new app's tiapp.xml
    7. Rename tiapp.xml.temp back to tiapp.xml
    8. Delete the build/iPhone folder
    9. Bump the App's version (not sure if this is necessary but it doesn't hurt)
    10. Build again.

    Let me know if this works.

  • I had a similar problem after fiddling with GIT and dropbox. Not sure what caused it. Reinstalling Xcode and Titanium didn't work. All projects had the same problem. Changing to a new workspace didn't help either. Nor did cleaning the build with project/clean menu item.

    Apparently the tiapp.xml points to the SDK in the library of a particular user: "/Library/Application Support/Titanium/mobilesdk/osx/". I had two versions 2.1.1.GA and 2.1.2.GA. In my projects tiapp.xml 2.1.1.GA was set. Changing to 2.1.2.GA. Solved the problem. The IOS simulators now started up without problems.
