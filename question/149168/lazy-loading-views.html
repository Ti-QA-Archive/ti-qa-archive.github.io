Lazy Loading Views

In Alloy (in the docs) it describes using Require to include a view within another view. Is there a way (and is it preferable) to instantiate the view at a later time and perhaps unload a previous one? In my current design I load my main view and it contains a "View" header and and footer with open space that in the center that I planned to fill with one view at a time, unloading each previous one to save resources. All I could find in the Alloy docs is the Require methodology that would result in loading all views and hiding the unwanted ones having them all always in memory?

  • It might help if you gave us the skeleton code for this layout. That said, if you define this central view that can be moved in and out as its own controller, you can then just call Alloy.createController() in your code whenever you need to create it then move it into view.
