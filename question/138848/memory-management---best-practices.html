    • Ivan,

      I agree that in theory, Titanium should take care of most memory management for you .. as long as you stick to some basic coding best practices. Memory management is a problem however because a lot of folks don't follow those best practices. Also, people fail to recognize the limited memory space available on mobile devices and underestimate the memory impact of their app components. (Like loading many images, which get uncompressed at runtime consuming very large memory footprints.)

      Your blog post is informative and describes some of the ways JavaScript can help or trip you up. I added a comment to clarify app-level event listeners and the purpose of setting objects to null.
