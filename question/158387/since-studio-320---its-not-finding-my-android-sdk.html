  • I have solved this. It's actual pretty simple. The cause of the problem is that Titanium Studio is not calling things by their name. What caused the confusion for me when I first encountered this problem was that a message was saying: An Android SDK is missing. Titanium requires Android platform 5.0

    1. There is not such thing called Android platform 5.0
    2. In the error message, they really mean, Android SDK API 5. Not that I didn't write API 5.0
    3. Because of point 2 above, the 5.0 will mislead people and make then thing of the Android version which is currently at 4.4.x for KitKat.
    4. If Titanium know which Android SDK API Level is absolutely required, why not make it available in the list of checkboxes and check it by default? That's a no-brainer.

    THE SOLUTION: Install the Android API Level 5 using the Android SDK Manager using the following steps …

    1. Open the Android SDK Manager. This is done from the command line by running the script tools/android in the Android sdk folder. Since my Android SDK is located at ~/android-sdk-macosx, I had to do the following

      $ ~/android-sdk-macosx/tools/android

    2. IMPORTANT: The Android API 5 is deprecated so you will not see it in the list. To solve that, make sure that you select Sort by: API Level and check the Obsolete checkbox, then scroll in the list of packages and select and install everything under Android 2.0 (API 5).

    3. Restart Titanium studio and check thet status of the Android SDKs. If you already had the Android SDKs, you can simply specify the path of the previously installed Android SDK and Voilà. If no, you can start downloading whatever version you like. It's much more pleasant to work with once you get passed that nightmare.

    Have fun.
