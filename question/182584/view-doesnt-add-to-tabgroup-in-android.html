View doesn't add to tabgroup in android

Titanium SDK 3.4.2 Alloy 1.5.1
iOS: SDK 8.1, iPhone 6 simulator
Android: Samsung Galaxy S3 - 4.1.1 - API 16 - 720x1280

I am trying to add a login view controller to a tabgroup if a user is not logged in. When I launch in the iphone simulator, the login view displays immediately as it should. When I launch the android emulator, the login view does not display, but instead shows the first tab of the tabgroup. Is there something android specific that I'm not including here?

index.js

if (authToken === null || authToken === "") {

    var loginController = Alloy.createController("login");
    $.tabGroup.add(loginController.getView());
    $.tabGroup.open();

} else {

    $.tabGroup.open();
}

login.xml

<Alloy>
    <View class="container">
            <View id='loginView'>
                <Label id='loginText'></Label>
                <View id="lvContainer" >
                    <TextField id="email" class='tfWrapper' />
                    <TextField id="password" class='tfWrapper' />
                    <View class="centerHelper">
                        <Button id="doLoginBtn" title="Login" />
                        <Button id="forgotPasswordBtn" title="Forgot Password" />
                    </View>
                </View>
            </View>
    </View>
</Alloy>

  • I got it to work, but I had to create a separate login view for android that was a "window", while for iOS it was just a "view". For android I just opened the view controller window, while for iOS I could just add the view to the tabgroup, see my code below:

    index.js

    if (OS_ANDROID) {

    Alloy.Globals.openAndroidLogin();

    } else {

    Alloy.Globals.openiOSLogin();

}

    alloy.js

    Alloy.Globals.openiOSLogin = function() {
    var loginController = Alloy.createController("login");
    Alloy.Globals.tabGroup.add(loginController.getView());
};

Alloy.Globals.openAndroidLogin = function() {
    var loginController = Alloy.createController("loginAndroid");
    loginController.getView().open();
};

    This may not be the best practice for doing this, but hey it works! If someone knows of a better way, please correct me!

    CodeStrong!

      1. create a separate tabbed window, & a separate login view

      4. if already logged in, then only open the tabbed window, but make sure you dont call this code $.index.open() in this case because your tabbed window will be opened from index.js file….

      • few more points to make app more perfect*
