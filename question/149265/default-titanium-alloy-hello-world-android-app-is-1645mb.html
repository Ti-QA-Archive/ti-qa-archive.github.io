Default Titanium Alloy Hello World Android App is 16.45MB

Hey all,

I'm new to Titanium. I figured I would start with the latest approach using Alloy to create a simple App. After I created the Default Alloy Project (no tabs), I installed it to my Android phone and noticed that the size is rather massive for a Hello World application.

I saw other posts where people were wondering about what else the developer had added to their project to get the crazy sizes the were worried about, but this is as clean a project as one should be able to create. Others mentioned that their Apps are coming around 4 MB.

If I just look at the total size of the files under all resources, including iOS which shouldn't be loaded, there is 5.69 MB of files. Android is only a little over 1 MB. Even assuming that non-Android files are getting loaded, which hopefully they aren't, that leaves me with a 10.9 MB "Hello World" app. Ouch.

Is there some additional code being loaded that I'm just not using that can be filtered out somehow?
Is this a defect that is being addressed?
Other?

Here is my current setup:

Windows 7 64 bit
Titanium SDK: 3.0.2.GA
Java JDK: 1.6
Installing to a Samsung Galaxy Note II running Android 4.1.1

Thanks,
Todd

    • The APK is generated prior to deployment. That's how Android apps work in any form. The V8 engine is required so that Titanium's javascript bindings to native code can be executed. iOS does not have this problem because V8 is not used, the default jscore engine is used.
