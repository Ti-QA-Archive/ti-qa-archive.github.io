Viewing the current exception in Titanium Studio

Am I missing something or possibly being a bit stupid, but I am confused as to how the debugger and exceptions are handled with Titanium Studio (version 1.0.2.201107130739).

When an exception occurs, the debugger kicks in and jumps into debug mode with the cursor on the line where the exception has been raised. I can also see the stack trace in the window above (Debug tab).

However, I can't actually see what the exception is. The only way I've been able to do this is to wrap the code in a try {} catch(e) {} and output the object e to the console. Surely there is a way to see what the exception is that has been raised.

I've included a screen shot of what happens when an exception is raised at runtime, yet I can't see what the exception is. http://imageshack.us/photo/my-images/121/exceptionz.png/

Your advice is appreciated!
