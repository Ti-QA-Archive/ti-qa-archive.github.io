Distribute - Apple iTunes Store creates "Mac App Store" archive instead of "iOS App Store"

I'm in the final stages of submitting my first app to the iTunes App Store, and it has the status of "Waiting for Upload" in iTunes Connect.

However, when I create an archive by using the "Distribute - Apple iTunes Store" link in Titanium, it appears to create a "Mac App Store" archive instead of an "iOS App Store" archive. The reason I believe this to be true is because when I "Validate" the archive in Xcode, it only comes up with "Mac App Store" and "Direct Distribution" as the only two options. And when I "Validate" using the "Mac App Store" option, it comes up with the error "No suitable application records were found". When I called Apple support, they said it should've given me an option for "iOS App Store"

The project is set up as "Titanium Mobile" and it works fine on iPhones, iPads, etc. during development. And I've created a Distribution profile with a Distribution Certificate, etc. which all compiles fine without errors. It's the archive in Xcode that seems to be having problems.

  • I found the problem: When I added the multiple icon filenames to the Info.plist file in the Build director, I also copied it to the root directory also. Once I deleted the second copy of the Info.plist file (and left the modified one in the Build directory), and re-packaged it, it then loaded in Xcode as a proper iOS archive.
