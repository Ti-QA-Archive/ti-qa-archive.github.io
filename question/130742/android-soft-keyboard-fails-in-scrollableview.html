Android soft keyboard fails in ScrollableView

Can't seem to find any solution to this.

If I add a textfield to a view, then that view to a ScrollableView then that ScrollableView to an OptionDialog, the soft keyboard will not show, regardless of it's settings. The minimum use case is as follows:


Ti.UI.setBackgroundColor('#000');

var win = Ti.UI.createWindow();

var field = Titanium.UI.createTextField({
    width : 180,
    softKeyboardOnFocus: Ti.UI.Android.SOFT_KEYBOARD_DEFAULT_ON_FOCUS
});

var view = Ti.UI.createView({ width: 100, height: 100 });

view.add(field);

var scrollView = Titanium.UI.createScrollableView({
    views : [view],
    showPagingControl : true
});

var dialog = Ti.UI.createOptionDialog({
    buttonNames : [ "OK"],
    title : "Testing",
    androidView : scrollView
});

var button = Titanium.UI.createButton({
    title : 'Open',
    height : 40,
    width : 145,
    left : 10,
    top : 55
});

button.addEventListener('click', function()
{
    dialog.show();    
});

win.add(button);
win.add(dialog);
win.open();

Now if you change the OptionDialog androidView to the initial view that has the textfield in it, it works just fine. i.e.


var dialog = Ti.UI.createOptionDialog({
    buttonNames : [ "OK"],
    title : "Testing",
    androidView : view
});

Also, if the ScrollableView is just added to the window (with the view and textfield in it) it works fine. It also works fine in iOS.

Anyone else faced this problem?

Titanium Mobile SDK Version: 1.8.0.1

Mobile SDK Build Number: fbdc96f

Android 3.2 Emulator on OS X
