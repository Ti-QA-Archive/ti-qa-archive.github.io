Cool trick to make a slow execution context load "faster" using closure

So I have a tab group where one tab needs to be a separate execution context. Unfortunately this tab also needs to load up some sound files, which is an unfortunately slow process. This meant that my UI for this tab wouldn't display until that process completed. To make it faster, I used a javascript closure to effectively delay loading the audio files:

var loadSoundLater = function(itemIndex){
  return (function(){
    sounds[itemIndex].play();
    sounds[itemIndex].pause();
  });
};

for (var i=0; i < 4; i++){
  sounds[i] = Ti.Media.createSound({sound:soundFile});
  var funcRef = loadSoundLater(i);
  setTimeout(funcRef, 1);
};

This allowed me to pass parameters to the setTimeout for which sound to "preload". Now my UI comes up instantaneously!
