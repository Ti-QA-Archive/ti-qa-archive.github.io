Ad Hoc Build Always Forces Production Environment Variables

I've been using the following command in an attempt to force alloy to build my application using the development environment variables set in config.json for an adhoc build 

alloy compile --config platform=ios,deploytype=development

And when creating the Titanium build I through in -D=development for good measure.

ti build -p ios -f -T dist-adhoc -R "..." -P "..." -O "build" -D "development"

However, the resulting build always using the production settings. Is there any other way to get around this, or is the assumption that that any adhoc build (or build using certificates) must be a production build?

Thanks
James

  • Hi

    As far as I was aware you are not able to choose the type of setting, as this is only based on the actual type of compile you are after.

    • ad-hoc and app store builds; will always be production
    • on device for testing - will always be test
    • on simulator or emulator - will always be development

    If anyone knows anything different then update me as well.
