[WARN] New layout set while view [object TiUIView] animating: Will relayout after animation.

I have encounter this problem. I still not resolve. The problem is meaning the animation show, but view have not create. I move picker animation position. But the problem still exist. How to resolve this problem? thank you very much.

app.js have a button. When user click this button, that will require picker from other js.

trace:

[WARN] New layout set while view [object TiUIView] animating: Will relayout after animation.

app.js

Titanium.UI.setBackgroundColor('#000');

var win = Titanium.UI.createWindow({
    title: 'Tab 1',
    backgroundColor: '#fff'
});

var btn = Ti.UI.createButton({
    title:'show',
    width:100,
    height:30,
    top:100,
    left:100
})
btn.addEventListener('click', function(e){

    win.add( require('showPicker').showPicker());
})
win.add(btn);
win.open();

showpicker.js

exports.showPicker = function(){
    var slideIn =  Titanium.UI.createAnimation({bottom:130});
    var slideOut =  Titanium.UI.createAnimation({bottom:250})

    var pickerHoldAllView = Ti.UI.createView(
        {
            width: 320,height: 640,top:0, left:0,
            opacity: 1
        }
    )

    var holdView = Ti.UI.createView({
        width: 320,
        height: 640,
        top:0, left:0, opacity:.5,
        backgroundColor:'#000',

    })


     var pickerView = Ti.UI.createView(
         {
             height: 300,
             bottom:-250        
         }
     );


     var dayPicker = Ti.UI.createPicker({
        type:Ti.UI.PICKER_TYPE_DATE,
        maxDate:new Date(),
        value:new Date()
    });

    pickerHoldAllView.add(holdView);
    pickerHoldAllView.add(pickerView);
     pickerView.add(dayPicker);

    pickerView.animate(slideIn);


     return pickerHoldAllView;
}

    • had same issue, setting a delay fixed it.

      error: 

      window.animation(slide in);
gaugeNeedleA.animate(rotate);
gaugeNeedleB.animate(rotate);

      fixed:

      window.animation(slide in);
setTimeout(function() 
{
    gaugeNeedleA.animate(rotate);    
},50);

setTimeout(function() 
{
    gaugeNeedleB.animate(rotate);
},2150);     //depends on the rotate animation speed, mine was set to 2 seconds

      — commented February 20th 2013 by Gopi Reddy

    • I'm cleaning up warnings in my app and came across this post. Thanks for the tidbit. Instead of executing the views right after each other I embedded the second view animation within the first…….. and Voila! no more animation warnings, woohoo!!! My modified code below….. Cheers!

      // Close the view when the user makes a selection
      thisView.animate(
      {
      bottom:-150,
      opacity:0.0,
      duration:200
      }, function(){
      thisotherView.animate({
      opacity:0.0,
      duration:200
      });
      });
