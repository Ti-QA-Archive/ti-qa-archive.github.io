    First, using roger's script, you set up your analytics as a website, not as a mobile app. Essentially faking website pageviews to map traffic within the app.

    We have just recently switched to using Matt Tuttle's module with GA v2 support.
    https://marketplace.appcelerator.com/apps/5081?867626504

    With it, you setup your analytics as a mobile app.

    Analytics should be working on the simulator. It can take a day or two for the results to show up.
    If using a V1 type setup (the script) you can sometimes choose today's date even if its greyed out in the Analytics Dashboard and get data early. (sometimes)

    The v2 stuff, we had one situation where nothing was showing up on the GA Dashboard for about 3 days. We created a new id (a -2) and using it, it all worked fine. No change to our code, so must have been broken on the big G's side.
