Force a push notification to disappear

Hi, I'm developing an app for iOS and Android. I successfully send push notifications when a new Task is coming. When the user (accessing from a web page, or the app in other device) read the task, I need to cancel the push notification, I mean, force it to disappear from the notification center.
I know that the Gmail or Facebook application do such thing.
My question is, Is this possible using ACS? Is there a way that i can send a push notification to cancel another? thanks in advance

  • For anyone interested here is my solution:

    if the app badge != 0 : Send a push notification with no text and badge 0.

    if the app badge == 0 : Send a push notification with no text and badge 1, then send another with badge 0.

    This is working just for iOS. I'm still hoping that someone helps me out with android. thanks

  • My android solution: Change the cloudpush.jar that titanium provides.

    • unpack the cloudpush.jar file located in Library/Application Support/Titanium/modules/android/ti.cloudpush/your_version

    • open the CustomReceiver.class file using some proper software like JD-GUI

    • in eclipse, modify this file to process the push notification yourself. In mi case I only had to add some ifs before calling the CloudpushModule funciton

    • compile the file to generate another CustomReceiver.class. You need to place it into a dummy project, the package name has to be ti.cloudpush, and you may need to generate a dummy CloudpushModule.class, empty, just to fix the compilation errors.

    • replace the CustomReceiver.class with your own-recently generated file, generate a .jar and replace the original one.

    It is a hard process that would take you ay least a few hours, but it is the only way I could figure out.
    I've spent a few days searching for a solution with no success, then I figure out how to do it.
    I hope that this post help someone to achieve this sooner, or achieve it at all.
