Adjusting TableViewRow height in Android

Hello fellow developers.
I have a tableView that is populated with data coming from a server. Im using data binding. Inside each row I have a View and inside that View I have among other stuff a ImageView that has its image property pointing to a url. The goal is to have all the pictures with the same width(the width of the its parent View) and the height of each image proportional to its ratio. Thats part of my code:

<Alloy>
    <TableViewRow id="row" class="row">
                <View id="view" class="view">
                    <!--stuff 1 -->

                    <ImageView id="picture" class="picture" image={pictureUrl}></ImageView>

                    <!--stuff 2-->
                </View>

    </TableViewRow>
</Alloy>

".picture": {
    left: 10,
    right: 10,
    width: Titanium.UI.FILL,
    decodeRetries: 10,
}

".view": {
    backgroundColor: "#ffffff",
    objName: "bubbleView",
    layout: "vertical",
    height: Titanium.UI.SIZE,
    zIndex: 300,
    left: 5,
    right: 5,
    borderRadius: 5,
    bottom: 0,
    top: 10,
}

".row": {
    height: Titanium.UI.SIZE,
    className: 'itemRow',
}

On Iphone it works fine, whenever the picture is loaded, the view adjusts its height so it can fit the image inside of it. However, in Android the View doesnt adjust its height when the image is loaded. The fun fact is that when the row loses focus it adjusts its components heights and it`s all good when I focus the row again. Has anybody ever seen this behavior in TableViews in android before? Is there a workaround?
