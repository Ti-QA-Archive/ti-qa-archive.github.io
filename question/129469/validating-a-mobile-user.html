Validating a Mobile User

I wish to create a php page that will receive variables and post them for viewers.
That's the easy part thanks to the many examples and documents online.

I'm not sure how to validate the user without asking for a username/password.
What's the easiest method or variable I can look for to validate that a POST is coming from a valid user on a legit mobile device?

I'm looking for something that a phone has that an emulator or random spoofer couldn't replicate, be it a single or combination of elements.

Ideally I would like to avoid requiring any credentials from the user specifically, making the interaction as transparent as possible.

    Hi Dooley

    when the user first time install the app, at the first time you need to ask for login or registration at that time you can save the device ID of the mobile. After that on each time the user login just take the device id and send it server and get the response based on that. I dont know what the actual scenario you need but this is something i implemented for one of my app.

    Check this Titanium.Platform.id this will return you the unique device id for the iphone, ipad and android.

    • well its all depend on your requirement, you need to just check that the request came from a mobile device then in that case you can get the platform.model which gives you iphone or android. while if you working on emulator then it gives you iphone simulator or android simulator.

      var runningInSimulator = (Titanium.Platform.model.indexOf("Simulator") > -1) ||
                             (Titanium.Platform.model.indexOf("x86_64") > -1) ? true : false;

      above code works in 1.7.5 the variable runningInSimulator if got true then the app is running on simulator else on the real device.
