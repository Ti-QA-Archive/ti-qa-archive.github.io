Reset password email not working?

Hi.

I'm trying to send a reset password email but I'm not able to receive nothing in my email. I'm doing it directly from the ACS console, and also by code, but nothing in the last 2 hours.

I tried different emails to see if was a problem with that, but not.

I have the SMTP well configured because I can send emails from my cloud app (I have a contact form, and I can send emails from that, which arrives inmediately), so ACS is sending email right, the only part that don't work is the reset password email.

Is there any issue relative to this?

  • More info about this problem. I remember now that I had a problem to send my first email from ACS with my provider, 1and1, so I wrote this JIRA: https://jira.appcelerator.org/browse/TIDOC-1424.

    Now, by code, I tested this, only to see if the problem was the same:

    ACS.Users.requestResetPassword({
    email: 'registered@email.com',
    from: 'my@email.com'
}, function (e) {
    if (e.success) {
        alert('Success: Reset Request Sent');
    } else {
        alert('Error:\n' + ((e.error && e.message) || JSON.stringify(e)));
    }
});

    and… bingo, I got the email. Again, the problem comes from the provider, 1and1. It seems that, maybe for check purposes, it needs the FROM field when sending an email.

    So I figure that ACS console is not sending the email with a FROM field inside and that is why I'm not receiving it. But by code I can set the FROM.

    Would be interesting to put a reference to this in docs, and maybe a fix in ACS to be able to set the from field, or will not work for some email providers.
