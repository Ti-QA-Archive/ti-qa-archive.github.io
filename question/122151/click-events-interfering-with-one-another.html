click events interfering with one another

I have a view that i can tap and drag left and right using a click event(along with touchmove and touchend). when i add a tableview to that view, with a click event, it stops working. if i remove the tableview's click event it starts working again but i need to be able to click the tableview to fire other events.

i'm thinking that all the events in the API aren't supported. like 'singletap', i've tried using that instead of 'click' and no response.

Any ideas? thx!

sidenote:if anyone is familiar with Twitter's ipad app i'm trying to mimic the functionality of being able to drag l/r the window holding the Timeline window (tableview) but still being able to click a tweet in the timeline and open a new window.

    Did you try to use a 'touchstart' event rather than a 'click' event in your view? Doing so, you could still use the 'click' event listener in your tableview. And these events wouldn't collide anymore?

  • 2 others ideas:

    1) Replace 'touchstart', 'touchmove' and 'touchend' by 'swipe' and see what happens.

    2) Set the zIndex property of the tableView to, say, 10. Set the zIndex property of the view to, say, 11 (which means the view will be 'above' the tableView - maybe you'll have to make it transparent). Use 'touchstart', 'touchmove' and 'touchend' (or 'swipe'?) to scroll the view. If you detect a 'click' in the view, send it to the tableView (fireEvent) with the x, y coordinates you got.
