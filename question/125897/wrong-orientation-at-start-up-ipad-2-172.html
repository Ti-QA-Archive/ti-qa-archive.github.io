Wrong orientation at start up (iPad 2, 1.7.2)

Hi,
I have a window with orientationModes = [Ti.UI.LANDSCAPE_LEFT,Ti.UI.LANDSCAPE_RIGHT];

iPad 2,

  • I had the same issue. I wanted to restrict the app to landscape left and right. I found that by alerting the Ti.UI.orientation value, LANDSCAPE_RIGHT (3) was always returned regardless of device position.

    I know this is cheating a bit, but what I had to do was obtain a value that the application could not argue. The x value from Ti.Accelerometer. In my app.js file (before opening any UI components) I added the following…

    var adjustOrientation = function(e) {

    if(e.x > 0.5) {
        Ti.UI.orientation = Ti.UI.LANDSCAPE_LEFT;
    }
    else if(e.x < -0.5) {
        Ti.UI.orientation = Ti.UI.LANDSCAPE_RIGHT;
    }
};

Ti.Accelerometer.addEventListener('update', adjustOrientation);

    I utilized this from the KitchenSink example. So, by getting the actual physical orientation of the device, I was able to change the orientation accordingly.

    Remember, since this is only required on startup, be sure to remove the event listener with something like…

    var myStartupWindow = Ti.UI.createWindow({
    orientationModes: [Ti.UI.LANDSCAPE_LEFT, Ti.UI.LANDSCAPE_RIGHT]
});

myStartupWindow.addEventListener('open', function() {
    Ti.Accelerometer.removeEventListener('update', adjustOrientation);
});

myStartupWindow.open();

    Hope this helps someone out.
