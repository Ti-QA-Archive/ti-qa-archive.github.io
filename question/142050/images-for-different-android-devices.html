images for different android devices

hi, im trying to create an app for which looks nice on different android devices. i tried to use the file structure which was given in titanium mobile project. the problem is, that the images are only shown on emulator and device if i give them an absolute path, for example 'images/res-notlong-port-hdpi/image_b.png but i want them to be chosen depending on which device i'm using. my file structure looks like this.

Resources:

/images
    /res-notlong-port-hdpi
        /image_a.png
        /image_b.png
        /image_c.png
    /res-notlong-port-mdpi
        /image_a.png
        /image_b.png
        /image_c.png
    /res-notlong-port-ldpi
        /image_a.png
        /image_b.png
        /image_c.png
/common.js
/app.js

i saw examles where the path was like this:
backgroundImage: '/images/image_b.png' but thats not working. so how should i name the path correctly?
or is there something else wrong?

thanks for answering!

  • Accepted Answer

    You have to put the files in the Android specific folder. So the path should be:

    /android/images/res-notlong-port-hdpi/image_a.png

    You would then access that image via:

    images/image_a.png

    Please check the docs for more info on using device specific images: http://docs.appcelerator.com/titanium/2.1/index.html#!/guide/Using_density-specific_resources_on_Android
