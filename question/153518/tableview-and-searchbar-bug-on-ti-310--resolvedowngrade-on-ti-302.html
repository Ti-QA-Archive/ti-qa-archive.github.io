TableView and SearchBar Bug on TI 3.1.0 => resolve/downgrade on TI 3.0.2

Environment:

  • iOS 6.1.4 or Simulator 6.0

  • Titanium SDK 3.1.0 -> BUG

  • Titanium SDK 3.0.2 -> NO BUG

  • Alloy Project

Console Error:

[DEBUG] :  2013-06-12 11:28:46.499 Freightplus[10532:c07] *** Assertion failure in -[UITableView _endCellAnimationsWithContext:], /SourceCache/UIKit_Sim/UIKit-2380.17/UITableView.m:909
[ERROR] :  The application has crashed with an uncaught exception 'NSInternalInconsistencyException'.
[ERROR] :  Reason:
[ERROR] :  attempt to insert row 0 into section 0, but there are only 0 rows in section 0 after the update

Context:

  • Inside a NavigationGroup, Windows A open Window B.

  • Window B contains a TableView with a SearchBar

  • When clicking on a TableViewRow, Window B is closed and Window A come back -> NO BUG

  • When using the SearchBar, then click on the TableViewRow matching; Window B is closed, Window A come back… the app hang and crash -> BUG

–> Downgrading to SDK 3.0.2 resolve the problem.

is anybody facing this issue ?

  • After many test, it's definitely the timelag between the moment when the search screen disappear (after click) go back to the table view then close the NavGroup

    In my case, the App crashed because the operation Search + Click + Close the Window/NavGroup was executed to quickly without letting the time to the App to redraw correctly the TableView after using the search bar.
    In the iOS simulator, we can see this lag by looking the TableViewRow title disappears and then reappears

    Adding a timeout() of 250ms before closing the NavGroup resolve my issue
