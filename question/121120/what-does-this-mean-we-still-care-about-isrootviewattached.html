What does this mean: We still care about isRootViewAttached!!!!!!!

I'm building a mobile app in the new Titanium Studio with SDK v1.7 and having an issue coming up with the best way to use pickers to set some data values. I'm currently testing in the iPhone simulator. Here's what's happening…

Main Primary Nav (4 buttons across top of screen)
Main Tab Group (4 tabs across bottom of screen)
Third Tab (sell) selected, so win3 is active
Scroll Group is in a view on top of win3
Third View in Scroll Group is active and needs to call a picker to change the value of a button.

I was using a new window to hold the picker and then close the window when done. Upon returning to the Scroll Group and Third View, I get this in my console and nothing is clickable on the View:
[WARN] We still care about isRootViewAttached!!!!!!!

Any thoughts what would be causing that?

Here's the code for the event listener on the button:


    priceTypeButton.addEventListener('click', function(e){

        var saleTypePicker = Ti.UI.createPicker();

        var data = [];
        data[0]=Ti.UI.createPickerRow({title:'For Sale',item_index:'s'});
        data[1]=Ti.UI.createPickerRow({title:'For Rent',item_index:'r'});
        data[2]=Ti.UI.createPickerRow({title:'For Free',item_index:'f'});

        saleTypePicker.selectionIndicator = true;

        saleTypePicker.add(data);

        saleTypePicker.setSelectedRow(0,priceTypeButton.item_index,false);

        var w = Ti.UI.createWindow({
            backgroundColor:'#666'
        });
        var b = Ti.UI.createButton({
            title:'Close',
            width:100,
            height:30,
            bottom:50

        });
        w.add(b);
        b.addEventListener('click',function(e)
        {
            w.close();
        });

        w.add(saleTypePicker);
        saleTypePicker.addEventListener('change',function(e)
        {
            priceTypeButton.title = saleTypePicker.getSelectedRow(0).title;
            priceTypeButton.item_index = saleTypePicker.getSelectedRow(0).item_index;
            newItem.priceType = saleTypePicker.getSelectedRow(0).title;
        });

        w.open({modal:true,modalTransitionStyle:Ti.UI.iPhone.MODAL_TRANSITION_STYLE_PARTIAL_CURL,modalStyle:Ti.UI.iPhone.MODAL_PRESENTATION_PAGESHEET,navBarHidden:true});

    });

  • I was actually having this issue earlier. I was setting an index property on something in a tableview, but a tableview helper library was later using variable named index. Once I changed the name of my index property, it was fine.

    You can see the error in Titanium source code here: https://github.com/appcelerator/titanium_mobile/blob/master/iphone/Classes/TiWindowProxy.m

    My guess is it's got something to do with modalTransitionStyle. Try this maybe:

    w.modal = true;
w.open(modalTransitionStyle:Ti.UI.iPhone.MODAL_TRANSITION_STYLE_PARTIAL_CURL,modalStyle:Ti.UI.iPhone.MODAL_PRESENTATION_PAGESHEET,navBarHidden:true});
