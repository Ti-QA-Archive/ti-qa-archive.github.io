Alloy XML Markup - event handler executed twice

This is tested on iOS. An example markup below:

<Alloy>
    <Popover id="win1" width="350" height="350" formFactor="tablet">
        <View id="view1" onClick="view1Click"/>
    </Popover>
    <NavigationWindow id="nav1" formFactor="handheld" modal="true">
        <Window id="win1">
            <View id="view1" onClick="view1Click"/>
        </Window>
    </NavigationWindow>
</Alloy>

So the idea of this markup is that if it's on iPad, it's a popover. If it's on iPhone, it's a modal window. Everything works fine except that view1Click is always executed twice every time I click on view1. I know it's something to do with this markup, but I can't find out why. Any ideas? What's the proper way to do this?

  • Yeah, I've experienced the same days ago… seems like a bug, it also happens when using platform.
    The workaround is to assign 2 different functions that calls the same general function.

    <Alloy>
    <Popover id="win1" width="350" height="350" formFactor="tablet">
        <View id="view1" onClick="view1ClickTablet"/>
    </Popover>
    <NavigationWindow id="nav1" formFactor="handheld" modal="true">
        <Window id="win1">
            <View id="view1" onClick="view1ClickHH"/>
        </Window>
    </NavigationWindow>
</Alloy>

    function view1ClickTablet(e){
    view1Click(e);
}

function view1ClickHH(e){
    view1Click(e);
}

    This solved the issue at least for me. Good luck
