  • Hi

    In short YES and NO.

    500 simple tableViewRows are can be easily displayed on a tableView. However the more complexity you add to the views - by using a custom tableViewRow and adding your own views, labels etc you reduce the potential performance benefits.

    • I would recommend you include the property className to each tableViewRow and set the same name to all rows that share the same basic layout. If you use two or more basic layouts within the same table then use the same className for each type.
    • I would strongly recommend you provide a "more items" solution and provide the data in bite size chunks, this makes it better for the user in data usage and performance - both real and perceived.
