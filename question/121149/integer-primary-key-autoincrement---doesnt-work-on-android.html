INTEGER PRIMARY KEY AUTOINCREMENT - doesnt work on android?

Hi Guys,
Im creating a table on an app for android.
I want an ID field that auto increments as a primary key so it starts at 1, then 2, then 3 and so on…

db.execute('CREATE TABLE IF NOT EXISTS drivers (id INTEGER PRIMARY KEY AUTOINCREMENT, title TEXT, fn TEXT, sn TEXT, dob TEXT, phone TEXT, add1 TEXT, add2 TEXT, add3 TEXT, pcode TEXT, selected INTEGER)');

I insert a row to the table as so…

db2.execute('INSERT INTO drivers (id,title,fn,sn,dob,phone,add1,add2,add3,pcode,selected) VALUES(?,?,?,?,?,?,?,?,?,?,?)',null,valueData.text,fnText.value,snText.value,dateData.text,phoneText.value,add1Text.value,add2Text.value,add3Text.value,pcodeText.value,0);

Note that i set the ID to null so that it will auto increment a value.

This works great in iOS but when i try and get the id from the row is returns 'null' in Android?

Any ideas?

  • Try skipping the ID column altogether when doing your insert. If it's not specified, sqlite will populate the value for you.

    Here is an example based on your code:

    db2.execute('INSERT INTO drivers (title,fn,sn,dob,phone,add1,add2,add3,pcode,selected) VALUES(?,?,?,?,?,?,?,?,?,?)',valueData.text,fnText.value,snText.value,dateData.text,phoneText.value,add1Text.value,add2Text.value,add3Text.value,pcodeText.value,0);
