hide tabGroup in Android with Ti SDK 3.2.0

before SDK 3.2.0 I used to hide the tabGroup in Android with an animation (see code)

var win1 = Ti.UI.createWindow({
    top : 0,
    left : 0,
    bottom : 0,
    right : 0,
    backgroundColor : '#c0c0c0'
});
var win2 = Ti.UI.createWindow({
    top : 0,
    left : 0,
    bottom : 0,
    right : 0,
    backgroundColor : '#336699'
});

var tab1 = Titanium.UI.createTab({
    title : 'Title One',
    window : win1
});

var tab2 = Titanium.UI.createTab({
    title : 'Title Two',
    window : win2
});

tabGroup.addTab(tab1);
tabGroup.addTab(tab2);

//nascondo la tabGroup per android - tablet
tabGroup.addEventListener("open", function() {
    tabGroup.animate({
        top : -64,
        duration : 0
    });
});

now with SDK 3.2.0 the same code show the tabGroup on top and a black line below.

is the a permanent to hide a tabGroup in Android? it freaks me out.

  • after many trials, I realized that my approach was wrong.
    programming with Titanium SDK is like a Zen exercise: you have to change your mind, discover new way of thinking. you have to think like Android OS.

    Android OS has ActionBar. Android OS has activities. ActionBar can be hidden by Titanium. So… nirvana!

    //only with Android API >=11).
tabGroup.addEventListener("open", function() {
    var actionBar = tabGroup.getActivity().actionBar;
    if (actionBar) {
        actionBar.hide();
    }
});

    namaste.

  • To throw my 2p in here, there is a lot of confusion because the tab group has now changed its origins and is part of the actionbar.

    The above answers all seem to be correct as long as you realise which version of the tabs you are using.

    If you are using a theme which uses an actionbar then the tabs themselves cannot be hidden, however the actionbar can be hidden as mentioned above by Danielo and in turn this seems to hide the tab group.

    If you are on a phone with v2.3.3 then the actionbar is not available, in this instance it is possible to animate the tab group off the top of the screen. I would go a step further to the demo above and use duration:0 so that you don't see it sliding off.

    Thirdly you can choose a theme which does not have an actionbar, such as: 

    <application android:theme="@android:style/Theme.Holo.Light.NoActionBar">

    this then makes your tab groups act in the same way as v2.3.3 on all devices.

    Hope that helps sombody as the info was a little confusing to find!
