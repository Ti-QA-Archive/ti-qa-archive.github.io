Images on android

Hi I have a simple question:

To manage the deferent sizes of android devices i put the images in the good resolutions in the ldpi or hdpi … folders.

For example I need to display a view with background like this :

var navBarContainer = Ti.UI.createView({
        height: 50,
        backgroundImage: '/images/topBar.png',
        top: 0
    });

    var backButton = Ti.UI.createButton({
        backgroundImage: '/images/btnBack.png',
        width: 55,
        height: 30,
        color: 'Black',
        font:{fontSize: 12, fontFamily:'Helvetica'},
        left: 10
    });

    navBarContainer.add(backButton);

I tried to use Ti.UI.SIZE instead of giving a pixel height it work in Rows but in views the result is not good.

So what is the bset practice to manage all the different sizes of images on android ? and should I use Ti.UI.SIZE instead of giving values ?

  • There's a guide for this.

    I recommend to use density-independent pixel values like this:

    var backButton = Ti.UI.createButton({
        backgroundImage: '/images/btnBack.png',
        width: '55dp',
        height: '30dp',
        color: 'Black',
        font:{fontSize: '12sp', fontFamily:'Helvetica'},
        left: '10dp'
);

    This way, you can be sure your views have the same size across all devices. If you want to dive a little deeper, you should read the Android guide too
