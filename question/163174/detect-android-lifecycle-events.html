Detect Android lifecycle events

I have an application for iOS, and now I want to create the android app. The application shows data and I have to update it each 10 minutes.

I have also three different windows, that I open / close to navigate in the app.

For iOS, I have an interval, and when the app is resumed I clear the interval, check if 10 minutes has been passed and then init the interval again.

For android, it doesn't work, because resumed event is not fired.

I've tried with two devices:

  • Android 2.3 it seems to work, I don't know how, maybe app is always working in background?
  • Android 4.4 it works sometimes. I'm not able to detect the casuistry. I mean, for example: I open the app, 10 minutes later data hasn't been updated, but 45 minutes later (for example), data has been updated.

Why does it behave like this? How can I be sure that data will be updated although android kill the activity?

How is the best way to implement it?

I've tried to add listener to activity but it doesn't seem to work:

var androidEvents = function() {
    var activity = Ti.Android.currentActivity;
    ['create', 'destroy', 'pause', 'resume', 'start', 'stop'].forEach(function(e) {
        activity.addEventListener(e, function() {
            Trace.info((new Date()) + " Activity: " + e + " HIT!");
        });
    });
};

And I call this function each time I change the window.
