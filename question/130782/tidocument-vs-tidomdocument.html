Ti.Document vs TiDOMDocument

Can anyone tell me the difference between the two objects. I am currently working on a project for both android and iphone. I was trying to create a text node. This works on android but not on iphone

// I read xmlString from file as 'xmlstring' variable and then 
var xmlDoc;
if(Ti.Platform.osname == 'android'){
    xmlDoc = Ti.XML.parseString(xmlstring).childNodes.item(0);
} else if(Ti.Platform.osname == 'iphone'){
    xmlDoc = Ti.XML.parseString(xmlstring);
}
// and then create a textNode
var textNode = xmlDoc.createTextNode('hello');

The only difference i have noticed is that xmlDoc object is shown as Ti.Document in android debug and TiDOMDocument in iphone debug. Can anybody please help me out.

Thanks in advance

  • In android you are returning a fragment of the overall document (.childNodes.item(0)) , in IOS you are returning the whole document.
