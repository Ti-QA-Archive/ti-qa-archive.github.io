Difficulty deploying / distributing an iPad application

I'm getting to the business end of our project and I need to start deploying our application to iOS devices for testing. I would ideally like to use TestFlight to do this, however I'm having some issues producing a valid IPA file. Eventually I'll need to distribute the final app to a limited number of devices too (Adhoc?).

Is there any up to date documentation on how to produce an IPA file using Titanium Studio and how to deploy apps? The only way I can see it how to do it is using Titanium Developer which I don't think is in circulation anymore (e.g. this video http://vimeo.com/10278960)? I've read this good Wiki article (https://wiki.appcelerator.org/display/guides/Deploying+to+iOS+devices) but it refers to creating an IPA by bundling a Development Provisioning Profile with the app, and I can only find an option (the small box icon next to the 'insect' and 'play' icons in Studio) to distribute or bundle the app with a Distribution Provisioning Profile (key word being Distribution and not Development). I've tried this option with a Distribution Provisioning Profile (to see if this will work) but the console logs some messages and then sticks on "[INFO] Performing clean build". I can still create a IPA by using this method (http://developer.appcelerator.com/question/123862/application-loader-where-is-the-ipa-located-to-upload-to-app-store) but when I deploy it on my iPad the app just hangs. If I check the logs for the device it says -[NSNull krollObjectForBridge:]: unrecognized selector sent to instance 0x2dd0cd8. Also if I run the app in the simulator I get the same message (even though if I clean the build and run using the debug or normal options - rather than packaging - it runs fine without issue).

So basically my app runs fine in the simulator but when I try and build it to put on the iPad it doesn't work. I tried dragging the .mobileprovisioning file onto the iPad along with the IPA file but that didn't work, and if I distribute the IPA using TestFlight it doesn't work either. My guess is I'm not producing a valid IPA file?

Any help would be greatly appreciated! Please don't assume that I know much about any of this, so detailed instructions would be great.

  • Problemo solved! From the console messages I could see that there was perhaps an issue with Helium. So I searched the Q&A library for issues relating to Helium and I found this post: https://github.com/kwhinnery/Helium/issues/4. I followed Matt Lanham's instructions to amend Helium (Matt is incidentally someone I know well and have worked with in the past) and it worked perfectly. Perhaps Helium should be updated so others don't encounter this issue…
