Global variable - again

I still don't get it. Please be nice!

I can not access my variable "thisfoobar" in "win2.js" created in "config.js"

app.js

Ti.include('config.js');
var tabGroup = Titanium.UI.createTabGroup({});
var win1 = Titanium.UI.createWindow({  
});
var tab1 = Titanium.UI.createTab({  
    title:'Tab 1',
    window:win1
});
var win2 = Titanium.UI.createWindow({  
    url:'win2.js'
});
var tab2 = Titanium.UI.createTab({  
    title:'Tab 2',
    window:win2
});
tabGroup.addTab(tab1);  
tabGroup.addTab(tab2);  
tabGroup.open();

config.js

(function(){
    var thisfoobar = "ok foo bar";
})();

win2.js

var win = Titanium.UI.currentWindow;
var l1 = Titanium.UI.createLabel({
    color:'#FFF',
    text: 'show me =>' + thisfoobar,
});
win.add(l1);

    When included via the url property as you're doing, each window is its own execution context. That means it has its own global variable and that of the caller is not directly available. Structuring an app with a single or multiple contexts is somewhat a matter of preference. But, it can be easier to access and pass variables when using a single context.

    See these guides for more info:

    • Titanium JavaScript Environment (passing info between contexts)
    • Building Reusable Factories (alternate coding style that can be used to create a single context app)
    • and the Cross Platform Applications video at the bottom of the Zero-to-App page

    Another resource to check out is the Tweetanium app

  • Within app.js you need to attach a reference to win2. So you need to add something like: 

    win2.foobar = thisfoobar;

    You might also want to look at the data passsing documentation.

    It's not really recommended, but alternatively you can create and append your own variables onto Ti.App, which exists as a single context within all windows.
