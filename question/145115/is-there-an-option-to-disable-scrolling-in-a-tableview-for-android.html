    • I don't normally reply to year-old threads. But since someone revived this with a comment today, and since my answer a year ago was wrong. I'll add a comment here. Mitul's answer is the correct one. The Android TableView does not support the scrollable property. Setting the table's height equal to its contents' height will work though.

      The other techniques described here might work. Though, they're a little hack-ish and might have other downsides. For example, setting touchEnabled false might stop scrolling. But you won't be able to click rows and get events at the table level.

  • Here's the solution:

    For iOS:

    touchEnabled = false;

    That will allow swipes, though…

    For Android: create a view in the same spot (absolute positioning). Give it an opacity of 0. Make sure it's in front of the one you want to disable. If it's still clickable, make sure it's touchEnabled and then listen to the touchstart event. In it, just

    return false;
