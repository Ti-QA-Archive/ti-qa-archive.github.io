File or Directory?

With Ti Mobile 2.0.2 on Android, there is a reliable way to find out if a Ti.Filesystem.File object is a file or a directory?

For me, right now every object is reporting as a file and all have the getDirectoryListing function. Directories and files are all looking the same!

  • Since you specifically mention Android, you can use the isDirectory() method after getting a File object reference to the file. Be aware that method does not (yet?) exist for iOS builds. In that case, AFAIK the best you can do is try getDirectoryListing() on the object to see what gets returned. The caveat is I am not sure if an empty folder will return a string array of zero elements, or a null. You'd have to experiment.
