  • Second issue RESOLVED – [ERROR] Error in compiler. , 'utf8' codec can't decode byte 0xa9 in position 1311

    If you are new to MAC dev and coming from Windows, you are probably clicking the ALT or Ctrl keys instead of APPLE. (AutoSave, Copy/Paste, Toggle between Windows)… this will pump a bunch of ß and ƒ inside the source code an result in the error mentioned.

    How I hacked a fix for this problem.

    1. Create a new Titan Project

    2. Copy and paste the App.js file, renaming the files to the correct file names needed for your project.

    3. (1-by-1) Copy text of all your js files from the ORIGINAL project to TextMate and SAVE with "FILE ENCODING" = 'ISO-8859-1 (Latin 1)"

    4. Paste the text from the text mate project into the new ".js" file you created.

    5. Rebuild your project and it should run.

    Hope this hack helps someone.
