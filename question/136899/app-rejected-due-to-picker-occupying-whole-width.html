App rejected due to picker occupying whole width

titanium mobile sdk 2.0
iphone sdk 5.1

Hi,

I used the api example on creating a simple picker but my app was rejected because of this. Here's what apple said:

10.3

We found your app uses system-provided items in a non-standard manner, which is not in compliance with the App Store Review Guidelines.

As recommended by the iOS Human Interface Guidelines, Pickers should not take up the entire width of the screen.

  • I had the same problem.

    How you solve it depends on how you implemented the picker.

    In my case I opened a transparent pop-up window as a modal dialog and placed the picker inside a container view, with a couple of buttons.

    All I did was specify the width of the container view that contains the picker and it was restricted in width, while the overall modal dialog effect remained.

    I hope that helps
