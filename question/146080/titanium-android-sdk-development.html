  • Hi

    Once you have everything installed developing for Android is as simple and quick as iOS, I am not sure why you are having to re-build your code every time.

    Once you have some test code you can just run by selecting the Android Emulator from the run menu.

    The first time you run your app it will/should perform a full-rebuild and if you have manually cleaned the project using the menu Project > Clean, but all other times the app should simply take your new code into account and start from there.

    Typical times for me are;

    • Emulator open and start (from first run) around 2 mins - not Appcelerators fault
    • Second run, emulator still running - no need to close it - 10 - 20 seconds, longer if it is a VERY large app.

    I run a Mac Mini, OSX 10.8, i5 quad, 4Gb of RAM and all files over a network share to a drobo (not the fastest choice).

    Please try and create a very basic project;

    • In Ti Studio
    • File menu
    • New
    • Titanium Project
    • Choose "Default Project" - should be selected already
    • Next
    • Provide a project name (example; test1)
    • Leave default location unless you want to put it elsewhere
    • Enter a suitable App ID (yours)
    • Finish

    Choose the run Android Emulator from the toolbar, wait for the app to open - time this.

    ONCE loaded, leave the emulator running and repeat the "run Android Emulator from the toolbar" process - this should be much faster.
