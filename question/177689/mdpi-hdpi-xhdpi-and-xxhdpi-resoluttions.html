  • Hi,

    "ldpi : 240 x 320 mdpi: 320X480 hdpi: 480X 800 xhdpi: 640X960
    xxhdpi:960X1600"

    Where did you get these values ???

    I think you are messing up resolution and density. These density classifications (mdpi, hdpi, xhdpi, etc.) describe a relation to abstract pixels (dp - density-independent pixels). It's got nothing to do with screen size or screen resolution. Tablets (or in fact, every display) uses the same density classes.

    Read about this here and also have another look at Malcolms answer in the link that Ricardo provided. He's explaining it very well. You should also show this to your graphic designer as he needs to provide the correct graphic resources.

    When you are trying to design for multiple resolutions, you are doing it wrong.

    You have to :

    • design for multiple form-factors (normal, large, xlarge)
    • adopt resources for multiple densities (mdpi, hdpi, xhdpi, etc.)

    You should start to think in density-independent pixels. It makes everything a lot of easier.

      Phones:

      • small (320 dp width)
      • normal (384 dp width)

      Tablets:

      • xlarge (1280 dp x 800 dp)
