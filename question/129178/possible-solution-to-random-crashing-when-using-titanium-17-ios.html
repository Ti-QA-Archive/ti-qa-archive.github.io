Possible Solution to Random Crashing when Using Titanium 1.7.* (IOS)

Hi all,

This is not a question but a possible solution from my own experience that may help someone who is experiencing random crashing when using 1.7.*.

For my case I found out that these random crashes are related to memory leaks.

It is not just the leak which is causing the issue. I'm suspecting the zombie window which still exist when it is closed may be listening to global event listeners and be responding to them which can cause weird behaviors.

This is an excellent video on how to profile your app for leaks :-
http://search.vimeo.com/29804284#

There is a memory leak issue when using XHR with multiple execution context window. This seems to be a bug with Ti since this leak also shows up in the KitchenSink. So if you are using XHR at all, use only single execution context window.

Apart from XHR and multiple execution context windows, the following also can cause leaks :-

  • Not calling removeEventListeners to global events such as on Ti.App or Ti.Gestures
  • Not shutting down timers when window exits.

So just by converting my app to using only single execution context and removing removing the memory leak, the random crashes are now totally gone.

Hope this helps.
