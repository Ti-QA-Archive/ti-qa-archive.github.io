Alloy 1.0 SQLite data type issues

Hi everybody, I get these messages but according to the sqlite data types on the Alloy Docs Page [http://docs.appcelerator.com/titanium/latest/#!/guide/Alloy_Collection_and_Model_Objects] it does not have to happen:

"datetime" is not a valid sqlite field, using TEXT instead
"boolean" is not a valid sqlite field, using INTEGER instead

So, how i see the current version of SQLITE? Alloy docs said datetime and boolean are valid data types but shows to me these messages…

App type: mobile
Ti SDK: 3.0.2
Device: iOS simulator
Host OS: Mac OSX
Ti Studio: 3.0.2.201302191606
Alloy Version: 1.0.0

  • As answered in the google group, this is an issue with the docs. Check the SQLite docs on the topic. Your app will continue to work because Alloy will make the translation behind the scenes, but ti would make your schema definition more accurate if you use the actual type names used by SQLite.
