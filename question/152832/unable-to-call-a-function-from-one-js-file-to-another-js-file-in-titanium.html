  • Hi

    You need to use CommonJS this allows you to wrap up functionality into modules that can be called from anywhere in your app from any of your script files.

    A quick example;

    testmodule.js

    function asFullName(strFirstName, strLastName) {
    return strFirstName + ' ' + strLastName;
}
exports.asFullName = asFullName;

    In any of your other files you use this as follows;

    var strFullName = require('testmodule').asFullName('Malcolm', 'Hollingsworth');

    So this module now has a single function that takes two parameters and does something with them before returning the answer. You functions can do whatever your normal code does, just export any functions that are public and do not export any functions that are simply helpers for the public functions.

    A note about the require parameter;

    require('testmodule')

    This means there is a JavaScript files named testmodule.js in the root of your files, the .js is NOT included. If you want to group your modules together in folders (good idea) then the path would be;

    require('somefolder/testmodule').

    Note the LACK of forward slash prefix - this is NOT included.

    You should group common code together into one of more modules (more is better) this way you are defining objects of code that have specifics purposes. Titanium will then run those files ONLY if required and cache them in memory for fast re-use.

    It is very important to use these modules as things that do NOT rely on knowing anything about the world outside them. So in my example - I passed to parameters and received one parameter back, my test module did not know why I want the information it just processed the request.
