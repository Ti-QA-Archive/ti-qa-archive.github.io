TableViewRow with hasChild = true not aligned correctly inside TableView

Titanium SDK: 3.1.3.GA
Platform & version: ios7
Device: iPhone simulator
Host Operating System: Mac OS.X 10.9
Titanium Studio build: 3.1.3.201309132423

Hi all,
I would like to have a table view whose rows will be populated dynamically. Each rows contains 2 labels. My problem is that if the table is populated with a large amount of rows, vertical align seems to be broken every 10 ~ 15 rows, like following screenshot:

here is the code (which generates the screenshot):

var win = Ti.UI.createWindow({
    title:'Test',
    backgroundColor:'white',
    exitOnClose: false,
    layout : 'vertical'
});

var table = Ti.UI.createTableView({

});

table.setData([]);
for (var i = 0; i < 100; i++) {
    var row = Ti.UI.createTableViewRow({
        left : 0,
        right : 0,
        hasChild : true
    });
    var label = Ti.UI.createLabel({
        left : 20,
        height : 20,
        top : 2,
        font : {
            fontSize : 15
        },
        text : 'line 1'
    });
    var label2 = Ti.UI.createLabel({
        left : 20,
        height : 20,
        top : 22,
        font : {
            fontSize : 15
        },
        text : 'line 2'
    });

    row.add(label);
    row.add(label2);
    table.appendRow(row);
}


win.add(table);

win.open();

If I remove the hasChild =true, the align is ok, but I need the right arrow symbol on each row.

Any suggestions?

Thanks.
