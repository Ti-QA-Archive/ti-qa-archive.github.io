Hide menu button on android?

This question is basically the same as this one but I hope the accepted answer of "not possible" is actually incorrect.

On my HTC One X test device with 4.1.1 the menu button always shows up at the bottom of the screen in my app. Has Titanium exposed a way to hide or disable this for specific windows or an entire app in general? There are plenty of apps that disable this button (facebook, instagram, etc).

    Hi

    Yes this is an easy one (once you know of course).

    Edit your tiapp.xml file and either amend or insert the following entries;

    <android xmlns:android="http://schemas.android.com/apk/res/android">
    <tool-api-level>14</tool-api-level>
    <manifest>
        <uses-sdk android:minSdkVersion="14" android:targetSdkVersion="14"/>
    </manifest>
</android>

    If you have an empty single line android entry replace with the above example. If you already have the android and manifest section simply add the tool-api-level and uses-sdk entries.

    These 3 values determine which SDK is used to build, which target Android version is the lowest you will allow your app to run on and which is the main one you aim for.

    Until you have some experience with these the above settings are fine and will remove the menu button from the normal 4.x three menu buttons. This is because this says my minimum SDK is lvel 14 which is 4.0, this then adds no support for the old menu hardware button in the software emulation.

    You can find out which Android OS versions equal which API levels in the Android <uses-sdk> documentation, you then simply swap out the level for the minimum you are targeting. Make sure the tool-api-level is equal to or greater than the minimum in my experience.

    This has nothing to do with how you build your window or app - just the tiapp.xml.

    BTW even though the big shift came in Android 3.0 (API Level 11) the menu button did not actually leave until 4.0 so 14 is the lowest to get rid of the menu.

    Keep in mind this will mean your app will not run on earlier versions of Android. If you want to run on both and lose the menu button - this is possible;

    You build two versions of your app; one with the api level at 14 and the other with a level of 10, this creates two APK files - when you submit your app provide both APKs and the store delivers the correct one to the correct devices. This also means you could choose to only place smaller resolution images in the api 10 version as it did not support larger screens and in the api 14 one do not place smaller imagery. Something to play with if you need it.
