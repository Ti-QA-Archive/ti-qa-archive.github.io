Desktop showDialog Recommendation

I'd like to make a recommendation for the Titanium.UI.showDialog method.

In the parameters object, allow a "close" callback that gets called before the dialog actually closes that would allow access to the dialog's DOM so you could grab things from a form or options or whatever the user had done on the dialog. This would also be a nice parameter for "createWindow." This should function similar to the callback set for "openFileChooserDialog", for instance.

A similar "open" parameter that fires before the dialog/window actually opens might be useful too though I can't think of a specific reason that I'd use it.

— asked December 11th 2011 by Mike Casto
