  • I had this exact same error in my code. There are probably many ways this could happen, but in my case, it was due to a timeout I was setting in my "open" event handler.

    To get the size of the window, I put an event listener on the "open" event and then use self.size.width and self.size.height to get the usable window size. Works great on iPhone. However, on android, I sometimes get a width, height of 0. This problem was intermittent. Obviously, though, you can't lay out an interface without valid dimensions.

    I had added a "recheck" mechanism that would use a timeout to call the open event listener again after 200ms.

    I think that was to blame. When I removed it, the problem went away.

    My workaround so far is to use the new "postlayout" event; I still sometimes get 0 width and height, but I just bail out of my event handler function if I get 0; postlayout will fire again with the right dimensions.

    Note that if you use this technique, you have to removeEventListener() to prevent an infinite loop.
