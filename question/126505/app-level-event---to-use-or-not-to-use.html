App level event - to use or not to use?

I see that many people encourage others to use app-level event (Ti.App.fireEvent). I think that is not good practice for the similar reason global variables are bad practice.

For example, event name can cause collision:

//component1.js
Ti.App.addEventListener('createComponent', function(e) {
    var component1 = new Component1();
    var win = Ti.UI.createWindow()
    win.add(component1);
    win.open();
});

//component2.js
Ti.App.addEventListener('createComponent', function(e) {
    var component2 = new Component2();
    var win = Ti.UI.createWindow()
    win.add(component2);
    win.open();
});

One of solutions that people suggest is prefixing event name to mimic namespace:

Ti.App.addEventListener('Component1::createComponent', function(e) {});

Well, that solves one problem, but introduces other. Component level events does not have this kind of problem, since they are bind to the component (so other components can't receive their events and namespaces are not necessary).

So, I think component level events should be preferred usage of events, while app-level events should be used in some cases where it makes sense.

What are your opinions?

  • Application level events have their uses, but you are right, in most cases component level events are preferable. One very important thing to note is that using application level events is the primary way for native Titanium code to communicate bidirectionally with Titanium.UI.WebView components. It basically affords you the opportunity to integrate a web experience with the power of the Titanium API.
