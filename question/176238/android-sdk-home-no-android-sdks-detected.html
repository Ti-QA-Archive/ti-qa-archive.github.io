  • If the command "ti info -t android" doesn't return valid Android SDKs, or invalid Android SDK location, then I would run the command "ti config android.sdkPath <android_path>" to set the Android sdk Path.
    Run again "ti info -t android" to make sure it recognizes the C:\android-sdk location and shows the valid SDKs and emulators.
    Once you open Titanium Studio, make sure the Android SDK location in the preferences also reference to the same location.

  • I faced the same problem on windows 7 and here is how I resolved it step by step:

    ####1. Upgrade normally from an older version (Don't install the new version as a standalone exe installation, you have to upgrade).

    ####2. Close Titanium

    ####3. open the folder name ".titanium" under the "Administrator" folder.

    ####4. Rename the config.JSON file to whatever

    ####5. open titanium, then open terminal and type: "ti setup" then [enter], then type "1" then press [enter] which is the number of the first choice.

    ####6. Fill each request and after each filling press [enter], make sure you give correct path of android SDK when prompted, the last request will show you the GA versions available, so type [1] to select the latest 3.3 update then press [enter] button.

    ####7. Exit titanium again, then open the command prompt and go to the titanium directory from there as follows:

    c> cd\
c> cd C:\Program Files (x86)\Appcelerator\Titanium Studio
C:Program Files (x86)\Appcelerator\Titanium Studio>TitaniumStudio.exe -clean

    That's it! your emulator should appear if you already added one, if not just open the AVD manager to add a new emulator device for testing, the launch the emulator, any other errors that might appear like JAVA_HOME is resolvable through "my computer environment variable".
