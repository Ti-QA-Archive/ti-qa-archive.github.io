API Documenation Rework RFC or "Is 12px really enough?"

Hi,

today i saw this little 12x12 greyscale (apple/android) Platform-Icons the first time at the API-Documentation pages. 12 Pixel?????? Greyscale???? In my little (but bigger than 12px) world greyscale means disabled! :-)

I think, the API-Documenation really is worth a rework. Everytime developer search for details, they visit the api documentation - and many direct visit the forum after they dont find the right informations there.

API-Documentation optimizations

1. Replace 12px images with better and bigger Platform-Icons!

I think, its a good idea that a developer see at the first sight if his platform is supported or not!

Whats about these colorized: http://developer.appcelerator.com/assets/img/getting_started_tabs.png

2. Connect API docs and Forum / Usergenerated content

As an alternative to a new comment function its possible to create API forum threads and connect them with the api pages!

So one API-Documenation page have one API-Forum thread!

So everybody registered in the forum can write Comments inside the API Forum.

3. Make the API Docs filterable by Platform/Device

Filtered views for each supported Platform of Titanium Mobile will help to set the right track for the developers. This will minimize Questions lik "Method x dont work on Android! - Why?"

Example Api-Doc homepage:

Show me the Mobile-API for [ANDROID] [BLACKBERRY] [IOS] [All Platforms]

Every Method, Constant, Parameter should be tagged with the supported platforms/devices. (via annotations?) - small icons in front of the method/constant/parameter name indicates the platform/device!

At the moment, this is done in plaintext… so i read a complete text only to the the last words… only on ios! Great… so why i read this????

4. Take care of namesaces

I think, some parts of the API (and Documenation) needs to be reviewed and maybe refactured.

Here my point of view!

  • The namespace Titanium.UI.Android contains android objects/methods!
  • The namespace Titanium.UI.iOS contains android objects/methods!
  • The namespace Titanium.UI contains objects/methods available for all platforms/devices!

Hmm… whats about the Titanium.UI.DashboardView ? Is this a generic class or not?

Maybe Titanium 1.8.0 should move Methods to the right place (Titanium.UI.iOS.DashboardView) , add some "aliases" and mark the wrong classes like Titanium.UI.DashboardView as deprecated in the documentation! Titanium SDK 2.0.0 than remove all deprecated!
