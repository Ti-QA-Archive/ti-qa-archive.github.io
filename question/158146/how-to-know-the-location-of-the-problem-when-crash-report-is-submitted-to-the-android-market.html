  • Probably, yes. The stack trace refers an ActivityIndicator proxy that may not be available somewhere at the app. If you are not familiar with logs these points can be a good start:

    1. Is the progress of an operation taking too long that might prevent ActivityIndicator to either show or hide? Try out to debug app to narrow it down.
    2. Are ActivityIndicator's methods called consistently (does the activity indicator loads, unloads as per correctly?) Try out to show an activity indicator while some code is running and hide it on completion. Does the problem occurs?
    3. The more devices under test the better. Android Market does not provide full details like OS and device type where the crash actually happened. If you find anything unusual (inconsistent behavior for instance) please post logs, TiSDK and Android OS to be able to help out here. Thanks.
