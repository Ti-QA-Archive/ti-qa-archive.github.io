Titanium Studio Android App crashes upon running

Titanium studio compiles and installs the app with no errors at all. However, when I test it on my physical device (Nexus 5, running latest Kit Kat OS), the app crashes and says "Unfortunately, the App has stopped". The IDE isn't showing any fatal errors at all. How can I trace to where the problem is coming from?

NOTE: I am using Titanium Studio 3.1.3GA and my physical device is running Android 4.4.3, if that helps.
Now the reason why I am using 3.1.3GA is because if I were to use the latest version of Titanium Studio 3.2.x I get errors stating that the New Relic will not be able to properly instrument Android. So I am following the docs by using this work around http://docs.newrelic.com/docs/mobile-monitoring-installation/titanium-installation-and-configuration-windows#titanium_3.2.x

Unfortunately, there's no go. (But the iOS, specifically the iPad version, is working and running just fine).

    First, SDK 3.1.3.GA does not support Android 4.4.3. So, there's a strong chance you won't get this working if you must use that SDK. If New Relic's module isn't updated to support new SDKs, you may need to remove it.

    If you want to see the log messages associated with the crash, you'll need to use the Monitor application (included with the Android development tools) to view the device's logs.

      — commented June 23rd 2014 by Ebad Saghar

    • Monitor is an android utility that will output any errors in a log that you can view. It is NOT a piece of Titanium. To run it, find your Android Runtime directory. You can check it in Studio by going to Studio/preferences in the menu, then expand Studio/Platforms/Android in the tree and the directory in the 'Android SDK Home' field is the home directory for android. When you go to that directory, there is a 'Tools' subdirectory and the monitor executable/batch file will be there. Connect your Android phone to your pc, then run the app again. You will be able to see (monitor) all of the messages/errors generated in real time.
