    Steps I used to setup and test in Genymotion

    Genymotion makes android development using Titanium realistic without using a real device. The emulator is blazing fast!

    ###Setting up Genymotion to work Correctly

    1. Open Genymotion and click on the settings gear on the top
    2. Under ADB connection settings, check the "When a virtual device starts, connect it automatically to ADB"
    3. Point the path to android SDK to where you store your SDK
    4. Click "OK" to save
    5. Create or start up an already created device
    6. Wait for the device to boot

    ###Setting up and building using Titanium CLI

    1. Make sure that Titanium CLI is installed

      npm install titanium -g

    2. Make sure you are logged into titanium through titanium CLI

      titanium login

    3. Browse to your project root in terminal

      cd /your/project/root/location

    4. Build the project using CLI for android (make sure the –sdk is set to the Titanium SDK version you have installed)

      titanium build --platform android --sdk 3.1.3.GA --build-only

    ###Installing project on Genymotion

    1. Make sure no other android devices are attached to your computer and that the Genymotion emulator is running.

    2. Install the newly built app onto Genymotion emulator

      adb install -r build/android/bin/app.apk

    3. Launch the newly installed app on the Genymotion emulator

    ###Optional

    • To debug your app at an info level
      Terminal: adb logcat TiAPI:I *:S
    • To debug your app at a debug level
      Terminal: adb logcat TiAPI:D *:S
    • To exit use "Control-C"

    ###To speed up the entire process, here is a shell script to do all of this in one line of code

    #!/bin/bash
# My example bash script to compile and run in Genymotion

titanium build --platform android --sdk 3.1.3.GA --build-only;

adb install -r build/android/bin/app.apk;

if [ $# -eq 0 ]
then
exit 1
else
    if[ $1 -eq "debug" ]
        adb logcat TiAPI:D *:S;
    fi
    if[ $1 -eq "info" ]
        adb logcat TiAPI:I *:S;
    fi
fi

  • This is great, thank you Joseph Burger!
    My SDK was not in the path, as Larrie Wilson commented on above. Rather than put it in my path, I edited the script to include the full path to the SDK:

    #!/bin/bash
# My example bash script to compile and run in Genymotion

titanium build --platform android --sdk 3.1.3.GA --build-only;

/Applications/AndroidSDK/platform-tools/adb install -r build/android/bin/app.apk;

if [ $# -eq 0 ]
then
exit 1
else
    if[ $1 -eq "debug" ]
        /Applications/AndroidSDK/platform-tools/adb logcat TiAPI:D *:S;
    fi
    if[ $1 -eq "info" ]
        /Applications/AndroidSDK/platform-tools/adb logcat TiAPI:I *:S;
    fi
fi

    Of course, you have the have the right path to your SDK..

    Also of note for those that are not familiar with shell scripts, you can add ' info' or ' debug' (without the quotes) to the end of the command line call to the script to get got those options.. (In other words, when you run the script at the command line, type in the script name, followed by a space, followed by either info or debug..
