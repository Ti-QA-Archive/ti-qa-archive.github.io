Inactivity timeout - slowing the app down

I have set the app inactivity timeout to 2 minutes, i.e. if the user does not click any buttons in the app for 2 minutes the app throws an alert to indicate timeout is called…

On click of OK, I can continue using the app features, when I do this the app slows down in terms of scrolling and loading data for different windows. If I do not encounter the timeout and continue using the app, it works fine.

Here is the code we use, can someone point out what might be causing the app to slow down after a timeout?

function SetInactivityTimeOut() {
    var TimeoutInterval = parseInt(pm.view.globals.INACTIVITY_TIMEOUT, 10);

    var PreviousTimeStamp = Math.round(new Date().getHours() * 60 + new Date().getMinutes());
    pm.view.globals.TIME_STAMP = PreviousTimeStamp;    

    if (TimeoutInterval === 0)
        return;

    TimeoutInterval = TimeoutInterval * 60 * 1000;
    //Converting to milisecond
    var TimeOutObj = pm.view.globals.Timeout_Object; 
    if (TimeOutObj != null && TimeOutObj != undefined) {
        clearTimeout(TimeOutObj);
    }
    //Ti.API.info('TimeOutObj---'+TimeOutObj);
    TimeOutObj = setTimeout(function() {
        alert('Timeout has occured');
    }, TimeoutInterval);
    pm.view.globals.Timeout_Object = TimeOutObj;    
}
