Alloy Model creation at runtime

I am trying to create Alloy models at runtime in a controller, using Backbone APIs.

As explained in the documentation, this is fine as far as I create instances of models in memory and not trying to persist them:

var Book = Backbone.Model.extend();
var b = new Book({title: "alloy", author: "tony lukesavage"});

So I was trying to set the config adapter and columns and change the default Backbone.sync, with:

b.config = {};
b.config.adapter = {type: "sql", collection_name: "books"};
var sql = require("alloy/sync/sql");
b.sync = sql.sync;
b.save();

but I have got an error (Invalid SQL statement). This is probably due to the fact that no "books" table is found in the "alloy" default db nor any primary key is created.

I am wondering if there is a easy solution to create Alloy models at runtime currently with sql persistence support
