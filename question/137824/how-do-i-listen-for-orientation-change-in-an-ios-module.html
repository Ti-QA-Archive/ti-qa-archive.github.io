How do I "listen" for orientation change in an iOS module

I am writing an iOS module (in xcode) and want to implement didRotateFromInterfaceOrientation so my module can react to orientation changes. However, you usually implement this function in your implementation of a UIViewController. I was not seeing an easy way to do this because I am not using a UIViewController in my module.

Any suggestions?

    did you try to add an observer for the orientation change in your module UIDeviceOrientationDidChangeNotification

    • That is what I was looking for. Here is the sample code that is firing when I rotate the device:

      -(void)startup
{
    // this method is called when the module is first loaded
    // you *must* call the superclass
    [super startup];

    NSLog(@"[INFO] %@ loaded",self);

    [[NSNotificationCenter defaultCenter] addObserver: self
                                             selector: @selector(orientationDidChange:)
                                                 name: UIApplicationDidChangeStatusBarOrientationNotification
                                               object: nil];
}

- (void) orientationDidChange: (NSNotification *) note
{
    UIInterfaceOrientation orientation = [[[note userInfo] objectForKey: UIApplicationStatusBarOrientationUserInfoKey] integerValue];
    NSLog(@"Rotated!");
}
