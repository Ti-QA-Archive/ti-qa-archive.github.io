Using GPS in Webview (remote URL) on Android

On Android, when using a web view to display a remote website, the user is never prompted to allow GPS and therefore the webpage hangs.

On iOS, loading this url="http://www.towing.com/mobile/" in a web view works perfect. It prompts for permission to access your location, you accept, and immediately the page loads with relevant data.

I found an article describing the need to enable JS and passing a "Chrome UI" to the web view.
http://turbomanage.wordpress.com/2012/04/23/how-to-enable-geolocation-in-a-webview-android/

Anyone come across this issue and find a solution within Titanium?
