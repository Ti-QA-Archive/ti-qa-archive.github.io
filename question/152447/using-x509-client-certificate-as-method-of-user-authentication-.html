Using X.509 client certificate as method of user authentication ?

Hello, is there any way to use Titanium.Network.HTTPClient or another known module, Tiitanium API, etc. to open a SSL connection to a server and authenticate the user via a X.509 client certificate ?

My question relates to Titanium SDK 3.1 since there seems to be a module (SealSign CryptoLibrary Module), which handles these kind of things for SDK 1.8 - 2.1, but does currently not support 3.x .

    the SDK 3.x support might be added soon.

    Also I think I found some undocumented properties related to this problem in TiNetworkHTTPClientProxy.m in the source files of the 3.1 SDK. It seems the SDK developers added some way to add client certificates to httpClients. Just look at the code below the line containing:

    //TIMOB-11728. Expose setClientCertificates and setClientCertificateIdentity for HTTPClient

    In controller code it is possible to use these enhancements in this (exemplary) way:

    var conn = Ti.Network.createHTTPClient();
conn.setClientCertificates(["asdasd"]);
conn.setClientCertificateIdentity("asdasd");

    So, I'm relatively new to developing in Titanium and have no experience in XCode. So far it seems to me, that if did embed some certifcate file within the app, manage to read it in the controller code and pass it in the appropriate format to setClientCertifcates it might be able to work out.

    It would be really helpful, if there would exist some form of documentation regarding the use of setClientCertificates…
