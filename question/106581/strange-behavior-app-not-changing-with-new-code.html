Strange behavior. App not changing with new code.

I am working on my first app and everything was working a couple of days ago, then I did not work on it for a few days.
now I am back to it but I've run into some strange behavior.

As I am editing the code and launching on titanium to test the app is not changing with the code.
I am changing titles on buttons and trying to add labels but the app remains the same even when the code is completely different.

I am saving the project.
I have closed the emulator and restarted.
Closed titanium and restarted.
Deleted project off titanium and imported it back.
nothing seems to be working.

anyone have any ideas on what I did to cause this and how to fix?

  • Which platform(s) and SDK levels are you using? Neither the question nor tags specified. It sounds as if it is still running an older version of the app which is already on the emulator. Although titanium should normally be able to just replace an older version, my suggestion would be to force the simulator to delete the prior version before you launch again.

    If using the iphone, an easy way to do that is from the simulator menu bar, using iPhone Simulator > Reset Contents and Settings.

  • Thank you for the reply.

    Problem is fixed

    I am running

    Titanium 1.2.2
    SDK APIs 2.2

    Here is what i did that fixed the problem even though i am unsure why it fixed it.

    Inside my resources folder i had another folder called "Main_windows" in which i had more .js files.
    i was having trouble making my app.js file inside resources open the .js inside main_windows.
    so i copied all the files from main_windows and put them into resources and did away with '../main_windows/filename.js' and replaced it with 'filename.js'

    this worked for calling the file.
    this is where i ran into the strange behavior i suppose.
    i never deleted main_windows.

    after deleting main_windows the app started updating after each launch as it is supposed to.
    I guess simply having duplicate files was for some reason causing this behavior.

    problem is no resolved after deleting main_windows folder.
