Resizing Images and Wiritng for Android

I'm trying to take an image from the resource directory, resize it to the desired size (based off screen dimensions, etc..), then write it to the ApplicationDataDirectory. Then all future runs of the application will use this new image (rather then resizing everytime on the fly). I gather the following should work (doesnt for android, does for iOS).

function getImage( originalName, resizeName, width, height ) {
    var file = Ti.Filesystem.getFile(Ti.Filesystem.applicationDataDirectory, 
        resizeName);
    if (!file.exists()) {
        var originalFile = 
            Ti.Filesystem.getFile(Ti.Filesystem.resourcesDirectory, 
            originalName);
        var blob = originalFile.read();
        blob = blob.imageAsResized(width, height);
        file.write(blob);
    }
    return Ti.Filesystem.getApplicationDataDirectory() + resizeName;
}

originalFile doesn't exists when running on android(orignalFIle.exists() == false). This leads me to believe that I cannot access the original image resources. Is there a specific flag in the manifest to enable writing to internal storage?

Any ideas?
