App crashes on device but sometimes runs in simulator

I have an app that has been working perfectly for a long time that all of a sudden crashes on launch. It crashes consistently on the device, but sometimes I can get it to work on the simulator, but it crashes there too at times.

The error is 'Ti.UI.iPhone.systemButton [undefined] is not an object.' That line is a simple function to create a flexible space button to reuse throughout the UI. If I comment that one out, I get a an error of 'Invalid Method (createTabGroup)' when it tries to create the tab group. Both functions are unchanged from any prior version that was working.

I've searched all over and the only tips I've found involve deleting the build folder, which I've done about 100 times now along with running Clean both before and after. I've tried the nightly builds for 1/23 and 1/24 as well as 1.8.0.1. I've tried restoring the file from a previous version to see if there was some seemingly innocuous change that's causing the error to appear. All to no avail. Please help! I'm dead in the water with this app right now and I'm actually really close to finishing it. SO FRUSTRATED.

  • I seem to have discovered the cause of the problem. I am using ShareKit 2.3 and tried to include the .plist file that says what services to show. Commenting that out (i.e. sticking with the defaults) seems to allow things to work again. No idea why that causes the app to throw those errors, but there it is.
