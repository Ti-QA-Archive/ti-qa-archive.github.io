Is there any trick to use natively vector graphic instead of bitmap type graphic?

HI Folks, to reduce the size of the app and to offer a better visual experience (totally independent from devices) it could be good to use vector graphic (realized, for example, using Illustrator) instead of bitmap type graphic avoiding also memory issues. In my case, for example, I need to create very simple graphic elements (a lot of them) that in vector style they'd be very very small in size and memory occupacy but in bitmap mode (e.g. jpg, png or gif) they don't. Do you know if there is a possibility in that direction? Thanks guys.

  • First, your question failed to address what platforms you were targeting. Because of that, I'm going to only address iOS and if you come back with 'what about android' I'll mark your question down. Fair warning for skipping the instructions that were on the question submission page.

    Even in xCode 6, while you can have vector assets for iOS, xCode still generates and packages PNGs at build time from them. So, at the core, iOS is only using static bitmapped based graphics.

    You could look at wrapping a native library such as SVGKit (but they are a cocoapods based distro)
