Can an app like Flipboard be developed in Titanium?

I'm currently evaluating Titanium for an internal project for my company. They are looking for a magazine like application for displaying their internal content on their staff iPads. So basically something like Flipbook. Content will be streamed to the iPad app in a custom XML file and text, images and video will be displayed on the screen in various magazine like layouts. Can someone tell me if this is possible with Titanium and if so what methods/classes would one use to do this (not necessarily the XML part, but being able to split the screen/page up into various sections that contain a summary of text and then when tapped open up a larger window that shows more text/images etc.).

Many thanks!

  • Thanks!

    I think I'm going to go with a different approach and that is to render all my pages on the server and serve them up as just html. So here's some more details of my requirements. Just so you know I posted this on PhoneGap because I'm currently evaluating both of these tools and I apologize for the length of the post but I really do need to know if this is possible before I start any coding, hope you understand :)

    Here are my requirements, note I
    refer to my content as a "magazine", and there may be more than one
    that can be accessed from the app:

    1. When app opens up user can tap on one or more thumbnails (which
      will be the cover for the magazine - a simple jpg screenshot that is
      cached from our server). Tapping will open up a webview that contains
      all the html for the cover page of the magazine. If you've seen
      Flipboard app, this is similar to how they display "channels" on their
      content page.

    2. User will have ability to read previous issues of the magazine,
      therefore the content needs to be cached locally. User will be a able
      to pull up pop-up list and select previous issues from the list.

    3. As above, when user opens app up for first time it will connect to
      our server and pull down all the pages of the magazine, so this all
      needs to be cached. Pages will need to be stored in a folder structure
      that can be easily accessed ie app/magazinename/issuedate/pages /
      images etc.

    4. Pages can be read by swiping left/right, each swipe loads page x
      (no page turn animation necessary). Perhaps some sort of caching will
      be needed to speed up the response. I've thought about whether all the
      content can be stuffed in one html page and use javascript to show/
      hide each page but this might take too long to load or slow down any
      type of animation. Pages will be simple DIVs re-sized into various
      sections on the page. If I were doing this for web delivery, I would
      just use a mouse click event on the DIV and expand the DIV using some
      jquery code, but the animation needs to be smooth (like in Flipbook).

    5. When a DIV is tapped on this will be expanded to fill the screen so
      all the content can be read. There will be a button within this DIV
      that when the user taps, opens a link to our website and displays our
      website within a modal window that can be closed. There will need to
      be some sort of alert if no internet connection is available.

    6. When user taps on icon on status bar at bottom a small webview
      window will slide up which will display a carousel of thumbnails
      representing each page - definitely doable in jquery. User will be
      able to swipe through images and tap on the image to take them
      directly to that page.

    7. Will potentially need search feature which will search through all
      issues of the magazine and display thumbnails representing search
      results (not required for release 1).

    8. Some pages will contain embedded video which will play within the
      DIV, or be able to be viewed full screen.

    On our server we'll be generating the pages every month and there
    could be as many as 100 pages per issue, each containing html, images,
    links, embedded video etc.

    Hope this helps. I'm looking forward to see if all of this is possible
    with Titanium.
