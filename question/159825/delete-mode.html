  • If you are trying to obtain a user confirmation before the delete event, on iOS tableViews, I tried and gaveup, whit the idea that can't be done. I've been making a test, where I used a confirm-dialog on the delete event as workaround. It allows you to reintegrate the 'not-deleted' row in the tableview if user cancels the operation, or in case of confirmation persist it on the dataset as needed.
