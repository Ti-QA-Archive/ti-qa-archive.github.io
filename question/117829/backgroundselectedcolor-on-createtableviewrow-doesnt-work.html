backgroundSelectedColor on createTableViewRow doesn't work?

Hi all,

I'm doing the following to prevent the default blue (default iPhone) color on mouse down for my row, however, it doesn't work. The color doesn't change. So, what am I doing wrong?

var row = Ti.UI.createTableViewRow({
        height:40,
        backgroundSelectedColor:'#006837'
    });

  • Not sure if it has to do with a newer Titanium SDK (using 1.7.1 currently), but I managed to get it to work on Android.
    What got me on the right track was noticing that the property worked when using only the standard table row properties imageLeft and title.

    It did not work for customized rows that consist of views, image views, labels..

    what did the trick was setting

    touchEnabled: false

    for all views that make up the table row view.

    • Obviously, this thread is very old, but the issue still exists - touchable views inside TableViewRows on android cause the backgroundSelectedColor to not show up on touch events.
