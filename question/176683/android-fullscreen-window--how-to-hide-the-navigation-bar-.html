  • You have two choices, hide the action bar programmatically or set it in the theme. Hiding it programmatically means that when the window is first opened, the action bar will be present. Users will see the bar appear, then go away, which is probably not what you want.

    The theme method described in our documentation is what you want. But as of version 3.3.0 it would apply to all activities in your app. An upcoming release will let you specify the theme style to use on each activity by adding a property to your window.

    To use this now, you will need to download a CI version of the Titanium SDK to use this property. Just set the property to the name of the theme style resource. The code below has been tested with 3.4.0.v20140718070912.

    app.js

    var win = Ti.UI.createWindow({theme: "Theme.NoActionBar"});
win.open();

    (This should work for Alloy, too. If not, please log a ticket.)

    platform/android/res/values/mytheme.xml

    <?xml version="1.0" encoding="utf-8"?>
<resources>
    <style name="Theme.NoActionBar" parent="@style/Theme.AppCompat">
        <item name="windowActionBar">false</item>
        <item name="android:windowNoTitle">true</item>
    </style>
</resources>
