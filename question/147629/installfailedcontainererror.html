INSTALL_FAILED_CONTAINER_ERROR

Hi Guys

I'm attempting to install my app into the default emulator, but I get the following error message:

[INFO] Installing application on emulator
[ERROR] Failed installing remunerator.com.au: pkg: /data/local/tmp/app.apk

INSTALL_FAILED_CONTAINER_ERROR

I'm using the latest Titanium (3.0.1), deploying against Google APIs Android 4.1.2 (armeabi-v7) Screen HVGA, using the default settings (which I think gives it a 512mb memory)

The generated app.apk is 41.2 MB

I'm using <manifest android:installLocation="preferExternal"> to try to use the sd card.

According to the settings, I have 42 mb free (I presume that's the onboard memory?). From the settings I think I have 62 mb on the sd card.

Is there a way of setting my SD card to have more memory? Would that solve it anyway? I can't see a way to change it in the file which Titanium creates and supplies to the emulator.

— asked January 30th 2013 by Alister Pate

  • Run in a terminal cd <sdk directory>/tools and execute ./android avd

    Select the AVD your using and click on Edit, then set the SD Card size to 512 MiB, click on Edit AVD and you are done.
