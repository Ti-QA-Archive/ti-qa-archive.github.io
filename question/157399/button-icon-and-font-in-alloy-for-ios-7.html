Button icon and font in Alloy for iOS 7

I am trying to customize the font on NavBar icons in iOS 7. However, changing the font family and size seems to have no effect. Does anyone know if this is a bug or expected behavior?

My end goal is to use font-awesome icons in my NavBar buttons with this Alloy widget. I know that it's possible to add an image to the buttons, but that won't help accomplish my goal.

I also tried creating a label and converting it to a blob and add the blob to the NavBar button like this:

index.xml

<Alloy>
    <Button id="myButton"/>
</Alloy>

index.xml

var dummy = Ti.UI.createLabel({
    text: "test",
    font: {
        fontFamily: 'Helvetica Neue',
        fontSize: 40
    },
    textAlign: Ti.UI.TEXT_ALIGNMENT_CENTER
});
var blob = dummy.toImage();
$.myButton.applyProperties({image: blob});

If anyone has any thoughts or tips on using a custom font in NavBar buttons in iOS 7, I'd appreciate it.

Studio 3.1.3
Alloy 1.2.2
OS X 10.8.5
Xcode 5.0

  • I hit the same issue when trying to use Font Awesome for right and left nav bar buttons. What worked for me was to use a Label element, instead of a button for the views. In order to get a "pressed" look when touched all you need to do is change the label color on touchstart and touchend.
