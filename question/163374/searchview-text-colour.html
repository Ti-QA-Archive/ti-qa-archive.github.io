    • Thanks for reaching out, Ricardo and Md.Zahidul.

      Explicitly setting the theme to Holo causes the action bar and search view to become even darker. This would be OK - perhaps even better than before - if it wasn't for the fact that the text is still black, making it impossible to read.

      (A workaround to make the text visible is to change theme to Holo.Light. Needless to say, that's not always desirable.)

      Code for creating the searchView:

      var search;

if (Ti.Platform.name == 'android' && Ti.Platform.Android.API_LEVEL >= 11) {
    // Use action bar search view
    search = Ti.UI.Android.createSearchView({
        hintText: "hintText",
        //backgroundColor: '#fff'
    });
    var searchAsChild = false;

    tabGroup.addEventListener("open", function(e) {
        var activity = tabGroup.getActivity();
        activity.onCreateOptionsMenu = function(e) {
            var menuItem = e.menu.add({
                title : "Testtitle",
                actionView : search,
                icon: (Ti.Android.R.drawable.ic_menu_search ? Ti.Android.R.drawable.ic_menu_search : "search_icon.png"),
                showAsAction: Ti.Android.SHOW_AS_ACTION_IF_ROOM | Ti.Android.SHOW_AS_ACTION_COLLAPSE_ACTION_VIEW
            });
        };
        activity.invalidateOptionsMenu();
    }); 
}
else {
    // Use search bar
    search = Ti.UI.createSearchBar({
        hintText: "hintText"
    });
    searchAsChild = true;
}

      By the way: will setting the theme to Holo(.Light) in the tiapp.xml as suggested above cause any inconveniences on units with API level 10 (Android 2.3.3 & 2.3.4)? I don't have a unit with that API level available at the moment, and I don't trust the emulator on these issues.

  • Hi. I'm assuming that by explicitly setting the Holo Theme you inherited some styling that was not part of the Titanium default Theme.

    One more thing I would suggest is to create your own Holo Theme following the instructions here: https://gist.github.com/ricardoalcocer/8152522. I know this is a long shot, but I'd like to see if by doing this you inherit even more styles that are not part of your app right now.

    As for 2.3.3 and 2.3.4, Holo Themes and the ActionBar are only available after 3.0, so no, your app will not run on those devices. ActionBar support on old devices is slated for Titanium SDK 3.3.0.

    A workaround for pre 3.0 devices could be to use the ActionBar Clone Alloy Widget, but you'll have to hack in the Search Bar yourself.

      if (Activity.ActionBar) {
// there's an ActionBar
}else{
// no actionbar
}
