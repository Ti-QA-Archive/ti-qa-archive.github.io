  • I would recommend reading our wiki guides located here and checking out the best practices and common JS modules section as a starting point. There are some good example apps here.

    I would not recommend using the structure used by the Kitchen Sink as it does not represent best coding practices.

    In app.js in the kitchen sink, the tabs are created and added to the tabGroup. Each tab will have a root window that is opened on that tab. When we do tabGroup.open() we see the tabs added to the screen. The windows created in app.js have the url property set pointing to other js files that have the window contents defined in them. Creating windows in this way causes the window to run in a different context, this is not recommended.

    :) Check out the wiki!

  • Titanium doesnt let you have access to their root win (this is the window that contains the red splash image). You can open your first window on top of that one and that becomes the root window of the app. In the case of the tabGroup, in the hello world example, that is your root window and then you add windows to the tabGroup in form of tabs
