  • Ok, its a cart/horse thing with webviews. Couple of different approaches.

    You can listen to the webviews 'load' event which is fired when the document is loaded.

    We use webviews for graphs in a ton of our apps. Originally, we would open the webview with the graph library, then fire an event to push the data to the webviews functions. That was frustrating, because the web view may not have loaded yet.

    So we tried to have the webview fire an event that would in turn fire an event with the data in it. That worked ok, unless the data was coming from an XHR call, then we had to wait for that to finish.

    Most of what we do now is load a template file in, inject data into the template, then just load the template as html into the webview.
