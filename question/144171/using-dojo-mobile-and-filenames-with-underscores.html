Using Dojo Mobile and Filenames with underscores

Hi,

I am looking to use Dojo Mobile and a webview to build a mobile application. This is the first time I have used Dojo. I have downloaded the Dojo toolkit release and I have built an 'Hello World' type test which works fine in a browser. The problem is that some of the files in Dojo start with an underscore (_base.js), so that when I try and compile it as an android app I get the following error:

_base.js is an invalid filename. Android will not package assets whose filenames start with underscores. Fix and rebuild.

I tried removing the files that begin with underscore, but not surprisingly this breaks the app.

There are a lot of files beginning with underscore so it looks like it would be a big job to go through the code and change them all.

Does anybody know of a workaround for this issue ? Is anybody else using Dojo Mobile in this way ?

TIA,
JC

  • The problem isn't Dojo specific. It must be a bug in Android's WebView. As a workaround and best practice, you should perform a Dojo "build" to combine all the Dojo files into a single dojo.js file. This will speed the web page inside the WebView considerably. For more info about how to perform Dojo builds, check out the reference guide here: http://dojotoolkit.org/reference-guide/1.8/build/buildSystem.html.
