iPhone Module Won't Link to 3rd Party Framework

I've read lots of posts on this, but none of them help me.

I've created a custom iOS module that's dependent on a 3rd party framework: SpeechKit

I've got the SpeechKit framework added to my XCode project and per the docs I've modified the module.xconfig file to reference the framworks the module is dependent in (2 or system frameworks, the 3rd is the custom SpeechKit framework).

– module.xconfig
OTHER_LDFLAGS=$(inherited) -framework AudioToolbox -framework SystemConfiguration -framework SpeechKit

The Xcode project builds. The module builds (build.python). I can package the module in the Eclipse IDE and it is copied into the project I use for testing it.

When I attempt to build the Titanium project I get a linker error saying, "ld: framework not found SpeechKit"

QUESTIONS:

Does it matter where I put the SpeechKit.framework directory? If I put it somewhere "special" will Xcode find it when linking automatically?

Why doesn't this work like some people's posts claim in the module.xconfig: -framework -F"/path/to/SpeechKit.framework/ SpeechKit

Do I need to include the SpeechKit framework differently in my Xcode project? I'm using Xcode 4.52 on Mountain Lion with Titanium 3.0.0GA (also tried 2.1.4GA with same result). I've tried including SpeechKit with an absolute URL and that didn't do anything.

Is there a way to bundle the SpeechKit framework inside the resulting module static library so I don't have to care about the external linked dependency

Any help would be greatly appreciated.
