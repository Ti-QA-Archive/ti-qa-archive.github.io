  • I'm sure it is the use of the same object in two listeners without a "var" declaration. This would cause them to refer to the same global object. The name can be the same, but both must be declared with "var" to make them local to the enclosing function.

    Your order of declaration is fine - you add the event listener before opening the object, which should ensure the listener is ready to respond immediately.
