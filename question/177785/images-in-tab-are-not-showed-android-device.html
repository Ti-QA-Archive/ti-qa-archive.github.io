    • Hola Javier. I'm replying here so we can keep the thread organized.

      If you're now getting the images right on the device and huge on the emulator, it is because you need to create individual images for different resolutions, and then let Android choose the best one for the device it's running on. It's not really that hard and this answer explains it very well.

      Here's a summary of what you need to know.

      If working with Alloy (you should) go to your /app/assets/android/images folder. If working with Classic, go to /Resources/android/images

      Inside that folder create the following folders:

      • res-mdpi
      • res-hdpi
      • res-xhdpi
      • res-xxhdpi

      MDPI is considered "normal" and that tab icon should be 32x32, also known as 100% of the size. The size for the rest of the images is calculated starting from this size, multiplied by the local density factor, which is

      • res-mdpi = 1
      • res-hdpi = 1.5
      • res-xhdpi = 2
      • res-xxhdpi = 3

      Doing that math gives you this:

      • res-mdpi = 32x32
      • res-hdpi = 48x48
      • res-xhdpi = 64x64
      • res-xxhdpi = 96x96

      Create those file versions, store them in the appropriate directory and you should be set. BTW, this is not a Titanium-specific behavior, but the way Android works. Also, there are other possible folders like ldpi for lo-res, and xxxhdpi for super hi-res. It depends on what you're targeting.

      You don't really have to do anything to your code, Android will take care of picking the right one. You can however at any point get the current LocalDensityFactor from this API Call.

      Hope this makes sense. If anyone finds any error or omission, please jump in!
