Installing an application on Windows issues an unknown exception

Hi,

I just installed Titanium developer 1.1.2 on OS X and used it to create a desktop application (made by html & javascript).

I created the package using the installer type: network

Everything runs fine if I install the package on a OS X virtual machine (VirtualBox).

When I try to install the package on both Windows XP Pro (different pc) and Windows 7 (VirtualBox), the installer, at a certain point, shows an application error: "The exception unknown software exception (0xc000000d) occurred in the application at location 0x10040e2b
Click OK to terminate the program".

When I click on the OK button, the installation finishes after a while.

I then tried to launch the application and the runtime is downloaded and (I guess) installed. As a matter of fact the application window is shown and it looks like the app is running.

However, when I try to use the app, I get some strange behaviors on the Windows systems while, on the virtual OS X box everything works properly.

Does any other user faced the same issue?

Is Appcelerator going to investigate and, possibly, fix it? If you need further info please let me know.

  • For those who are still experiencing the same issue, please try to build the package with 'Bundled' option rather than 'Network' option.

    In my case, the installer size increased to about 3 times bigger (for Windows), but it fixed the 'Unknown Software Exception' error. Although it's not ideal to have large installers but it's better than having errors popping up.
