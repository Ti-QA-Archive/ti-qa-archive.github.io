iOS Push notifications only send when I update certificate

Hi all,

I am currently working on a native iOS app using Appcelerator.

The app works fine when in development mode - the certificate is on Appcelerator and push notifications are received pretty much as soon as I send them from the console.

I have created my Production certificate on Apple's Provisioning Portal and uploaded that to Appcelerator on the Production side of things. Here's the weird part:

When I try to send a push notification via Appcelerator's push console, it does not get received, at all. However, when I go back to Appcelerator Settings, remove the certificate then re-add the same certificate again, all of a sudden the last push notification I sent from the console is received on all devices built for Production (i.e. the Ad-Hoc devices).

I can consistently produce this… send a push notification and it only gets received when I remove and re-add the certificate again.

Does anyone have any suggestions? I am thinking that obviously the certificate is correct and valid because if not, I would not be receiving any push notifications…it seems like something wrong with Appcelerator to me, but please prove me wrong.

Thanks.
