Does Sound object uncompress mp3 file each time play is called?

I know that mp3 files are not the preferred format for the iphone but for space reasons, I have chosen to use a mp3 file instead of some uncompressed data format. My question is this: if I create a Ti.Media.Sound object and keep calling Play on that same sound object during the lifetime of the application, am I decompressing the mp3 file just the first time when I call play() or each time? Thanks in advance for any help.

    Not sure what your asking ?

    Just for clarity….an MP3 is not compressed in the same way as a ZIP file for instance. The file doesn't decompress and get larger when you hit play. It stays the the same. The clever bit is done when you convert the audio in the first place to make it smaller. It uses a process called perceptual encoding to make the files smaller. The phone has the required codec built in to be able to play these files. Hope that makes sense.
