How many images?

I want to support multiple devices, iphone, ipad, android multiple phones and tablets. I've read most of the docs and questions on the subject but I think I've confused myself.

For a given image that I want to look similar on all devices (in terms of the % of the screen it takes up), it doesn't have to be exact though, that is 100x50 on a standard iphone (non-retina) - how many sizes of the image should I create?

My thinking is:

- iphone - 100x50
- iphone retina - 200x100
- ipad - 200x100 (I can almost get away with reusing the retina)
- android phone ldpi - 66 x 33
- android phone mdpi - 100 x 50
- android phone hdpi - 133 x 66
- android tablets basically 2x the phone

(I was planning on using a cutoff based on resolution for phone vs. tablet).

Am I way off here or is this the general approach?

  • I have not much idea with tablets in android and iPad (But most probably UI of tablets are little bit different than phone because of big screen sizes. Most of the apps having this still i don't know exactly.) I will let you know my knowledge about phones.

    Let's just talk about main screen size (Based on screen pixel size you can configure your image sizes)

    • iPhone : 320 X 480
    • iPhone Retina : 640 X 960 ( Double )
    • mdpi Android Phone : 320 X 480 ( Pixel Density 1.0 )
    • ldpi Android Phone : 240 X 360 ( Pixel Density 0.75 )
    • hdpi Android Phone : 480 X 640 ( Pixel Density 1.5 ) (I think you have taken wrong here 133 X 66)
    • xLarge screen in android : 720 X 960
