Error message line number

Hello,

In producing an error message in the IOS simulator, we are given a line number for the offending code. However, the line number given never really matches the actual line number in the controller.

is there a way to know what the actual line number of the error is that is being given in the simulator?

Thanks!

  • If you're talking about Alloy, the line number most likely is accurate…. except that it's the line number in the compiled code. Look at the file path in your error message… you'll notice that the path is not the same as your source.
