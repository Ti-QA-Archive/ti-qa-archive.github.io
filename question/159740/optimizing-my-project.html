    Hi

    You should read; Managing Memory and Finding Leaks this will give you a strong starting point for handling memory.

    Alloy does manage much of the memory handling for you until you start adding your own code in controllers. When you do then the notes in the above link come into play.

    If you manually create something then you should clean it up. If you add an event listener you should remove it yourself, if you use setTimeout or setInterval then these persist unless you clear them.

    Your example;

    function test(){
    var a = 10;
    var b = 10;
    var c = a + b;
    alert(c);
}

    Variables; a, b & c will no longer persist once the alert has been confirmed. The function test will no longer persist once whatever created it has been nulled. Keep that point in mind the parent must be nulled - so if the parent nulling fails dependant persistent code will remain.

    Your tableViewRow example will clean itself based on the same rules - the parent that created it must be nulled, this will happen automatically if your code utilises Alloy logic throughout - however it can persist based on how it was created.

    Use the instruments to show HOW your views are created and if they are destroyed, if they fail to destroy then you add clean up code to handle the proper clean-up.
