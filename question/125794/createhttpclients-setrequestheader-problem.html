  • Joe is right about the colon, comma thing, but for what its worth, I have experienced problems with custom headers myself.

    I have discovered that webservers can be configured to block 'inappropriate' headers. E.g. extension headers should start with X and should use - (dash) as a seperator instead of _ (underscore). Thus any in between proxy/webserver could potentially block your header based on the violation of the above rules/guidelines.

    When using HTTPS, I experienced additional problems on SDKs 2.0.1, 2.0.2 and 2.1 in combination with an nginx web server. Not sure yet where the problem is, but if I use SDK 1.8.2 there is no problem.
