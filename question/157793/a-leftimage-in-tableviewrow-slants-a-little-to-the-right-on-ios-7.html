  • My workaround seems to prevent this bug, i set the image after the view is built, not a very elegant solution but seems to work:

     row.addEventListener("postlayout", function(e) {
            row.removeEventListener("postlayout", function(e) {
            });
            if (app.os == "iPhone OS") {
            row.rightImage = Ti.Filesystem.resourcesDirectory + '/images/startArrow.png';
        } else {
            row.rightImage = '../../images/startArrow.png';
        }
