Best way to sync databases - local and remote

Right now I have an app that uses a local sql lite database. I also have an iPad version as well.

I'm looking at in the next update, adding an optional user registration so people can have the information on my website if they want, so they can enter the information and have it sync with either the iphone, ipad, or both.

Does anyone have a best practice, maybe fields I should add to the db, so deletes/updates/inserts are all in sync?

  • We have an app that contains a local DB and a remote DB, where the local one is always in sync with the remote one. There's really not too much to worry about except for the UI (interrupts, loading, etc). Also, make sure that the sync only happens one way.
