screen swipes laggy

Not sure laggy is a word, however I am trying to make sure that when I swipe the motion is smooth. I am going to build a project that has separate HTML view files and for testing created simple HTML files that load a single full page graphic per html file. Whenever I start the app, the first swipe to the left never works. Page slightly moves but stops. Second time it works. Swipes after that usually work, but are laggy. Below is the simple code from app.js.

var win = Ti.UI.createWindow();

var view1 = Ti.UI.createWebView({ url: "index.html"});

var view2 = Ti.UI.createWebView({ url: "index2.html"});

var view3 = Ti.UI.createWebView({ url: "index3.html"});

var view4 = Ti.UI.createWebView({ url: "index4.html"});

var view5 = Ti.UI.createWebView({ url: "index5.html"});

var view6 = Ti.UI.createWebView({ url: "index6.html"});

var scrollableView = Ti.UI.createScrollableView({
  views:[view1,view2,view3,view4,view5,view6],
  showPagingControl:false,
});

win.add(scrollableView);
win.open();

Here is also a sample HTML file. They are all the same just a different image:

<!DOCTYPE html>
<html>
<head>
<meta charset="utf-8" /> 

    <meta name="viewport" content="width=1024" />

<meta name="viewport" content="initial-scale = 1.0, user-scalable = no" />

<meta name="apple-mobile-web-app-capable" content="yes" />

<title>Test</title>

</head>

<body>

<img src="images/bg-home.png">

</body>

</html>

Note I have also tried the cache option on createScrollableView but same deal. Any thoughts?

As a side issue, this application is set to landscape only however I can still bouncy things vertically and see white space on the top and bottom.

Excuse the spacing and the character changes. This forum was wrapping carriage returns.

Thoughts?
