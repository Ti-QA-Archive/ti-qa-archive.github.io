ListItem changes style after being clicked on IOS

Hi all,

I have a listItem which is displayed differently before and after being clicked on IOS only.

Here is the xml:

    <ListView id = 'listview' defaultItemTemplate = 'template'>
            <Templates>
                <ItemTemplate id = 'template' name = 'template'>
                    <Label bindId = 'title' id = 'title' />
                    <View id = 'centerLine'/>
                    <Label bindId = 'subtitle' id = 'address' />
                    <Label bindId = 'dist' id = 'distance' />
                    <View id = 'bottomLine' />
                    <ImageView id = 'icon' />
                </ItemTemplate>
            </Templates>
            <ListSection id = 'section'>
                <ListItem/>

            </ListSection>
        </ListView>

Here are associated styles:

'#listview' : {
    separatorColor : 'transparent',
    top : '13%',
    left : '3%',
    height : '74%',
    width : '94%',
    backgroundColor: 'transparent'
},
'#template' : {
    height : 100,
    backgroundColor: "white"
  }

After being clicked, views centerLine and bottomLine disappears and the backgroundColor of the item itself changes (from white to grey).

I tried to set selectedBackgroundColor on the ItemTemplate and to explicitly set the zIndex in case that would change after selecting without success.

Do you have any ideas why and how to fix that?

for info, my env is:

  • You have to set the SelectionStyle in your template. Options include Titanium.UI.iPhone.ListViewCellSelectionStyle.NONE if you want no styling to be applied when a user selects an item.
