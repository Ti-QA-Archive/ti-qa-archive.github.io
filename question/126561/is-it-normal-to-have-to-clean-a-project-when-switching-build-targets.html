Is it normal to have to Clean a project when switching build targets?

EVery project I have, whenever I have to switch from build for simulator (iOS) to build for device or back again, I get this error:

[ERROR] Error: Traceback (most recent call last):
  File "/Library/Application Support/Titanium/mobilesdk/osx/1.7.2/iphone/builder.py", line 1305, in main
    execute_xcode("iphoneos%s" % iphone_version,args,False)
  File "/Library/Application Support/Titanium/mobilesdk/osx/1.7.2/iphone/builder.py", line 1066, in execute_xcode
    output = run.run(args,False,False,o)
  File "/Library/Application Support/Titanium/mobilesdk/osx/1.7.2/iphone/run.py", line 39, in run
    sys.exit(rc)
SystemExit: 78

Fortunately, doing a clean allows me to build. Is this normal behavior?

Titanium Studio, build: 1.0.5.201109301412, SDK 1.7.2, OS 10.7

  • Yes, this is quite normal. When there are build files already in the build directory, in order to make the build quicker, the build scripts will use what exists already. When changing your build target, the build files that existed for other targets will cause the new target to build improperly, so just make sure to clean before building for a new target.
