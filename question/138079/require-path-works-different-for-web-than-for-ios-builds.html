require() path works different for web than for iOS builds

Hi,

I just imported the Community App to get started.
I managed to compile for the iOS simulator. But when I tried to export in HTML5 I got a red screen that tells me:

Failed to load module "/ui/bootstrap": 404 at /index.html (line 1168)

So it seems like this (default) require path is not accepted when building in HTML5:

var app = require('/ui/bootstrap');

I created a test file to require and called it "test.js". It worked to include it this way: require(&#39;test&#39;). But it failed again doing it this way: require(&#39;&#x2F;test&#39;).

Is the initial backslash interpreted different in HTML5 mode than in iOS mode??

I know that there is actually no bootstrap file in the ui directory of the Cummunity App. So I created one but it still doesn't run. I'm also interested about why the compiling for iOS uses "/iphone/ui/bootstrap" instead of "/ui/bootstrap"… anyone knows where to set this?

Thanks in advance!

Balz

    Check out the comment on this Jira Ticket about absolute paths. This, along with the info on CommonJS paths in the docs is a little confusing to me, but I'm guessing that when the latest mobile web was introduced, the oversight was fixed, but now we are stuck with the incorrect behavior, at least for a time. Just take out the leading slash '/' and you will hopefully be good across all platforms.

    As for using /iphone or /android or /mobileweb in the file path, these are called platform build directories, and are a way to override files in your project per platform. There is nothing to 'set', you can opt to use the structure or not. Just note that the files are ignored if they don't apply to that platform. Read more about the platform build directories in the Community App documentation.
