Spotify link within webView not working on Android

I' building an app for a music festival and they have a spotify playlist I want to be reachable from within my app.

Very simple code for this:

var win1 = Titanium.UI.currentWindow;

var webview = Ti.UI.createWebView({
    title:'Spotify playlist',
    url: 'spotify:user:username:playlist:playlistID'
    });

win1.add(webview);

This works on IOS from within the app, opening the window automatically opens the spotify app but not on Android. If I open the built-in browser on Android and paste the exact same url it opens the spotify app and opens the playlist.

How can i fix this on Android? Is there some other way of calling installed apps? Maybe calling the "open with" dialog in Android somehow?

  • I have seen websites that don't show up properly on Android (such as twitter) unless you set a custom user agent. Try adding this property to your webview:

    webview.setUserAgent(&#39;custom user agent&#39;);

    that should work.
