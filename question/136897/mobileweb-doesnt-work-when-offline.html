Mobileweb doesn't work when offline

Hi, so yesterday I was flying home and I thought it would be a great moment to work on migrating my app to mobileweb but when I hit run I got a blank screen and a few log errors:

[INFO] Appcelerator Titanium 2.1.0.v20120504170331 Mobile Web
GET http://connect.facebook.net/en_US/all.js The Internet connection appears to be offline.
GET http://maps.gstatic.com/intl/en_us/mapfiles/api-3/8/11/main.js The Internet connection appears to be offline.
POST https://api.appcelerator.net/p/v2/mobile-web-track?callback=296221&output=html The Internet connection appears to be offline.

I first thought maybe this happened because of something I'm using something in my project so I tried a normal default Titanium project and got the same errors.

Is there any way to test mobileweb offline?

— asked May 14th 2012 by Dario Marcelino

  • This is a known problem (although curiously enough a JIRA ticket hadn't been created). I created a ticket that you can follow on JIRA.

    In the meantime as a hack, you can open up <titanium SDK dir>/mobilesdk/<platform>/<sdk version>/mobileweb and open up compiler.py. If you scroll down you will see a long list of package names in an array. If you comment out Ti/Map and Ti/Map/* then the app will load fine.
