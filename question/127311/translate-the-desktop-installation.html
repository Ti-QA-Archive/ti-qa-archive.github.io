  • Hi, Allan,

    i´ve changed the language in the win32_app-Phyton-File in zhe sdk-Directory (1.2RC2 and above) the sel.language-value at line 169 and 188 :

            def get_app_language(self):
                self.language = 'en-us'
                self.get_tiapp_element_as_prop('language', 'language')
                langs = {
                        "cs-cz": ("1029", "1250"),
                        "nl-nl": ("1043", "1252"),
                        "en-us": ("1033", "1252"),
                        "fr-fr": ("1036", "1252"),
                        "de-de": ("1031", "1252"),
                        "hu-hu": ("1038", "1250"),
                        "it-it": ("1040", "1252"),
                        "ja-jp": ("1041", "932" ),
                        "pl-pl": ("1045", "1250"),
                        "ru-ru": ("1049", "1251"),
                        "es-es": ("3082", "1252"),
                        "uk-ua": ("1058", "1251")
                }
                if self.language in langs:
                        return langs[self.language]
                else:
                        return langs['en-us']

    hope, this helps, for other operating systems I´d never tried this.

  • The way i solved this problem of changing the language on the msi installer was the following on the file win32_app.py that is located on Titanium\sdk\win32\1.1.0\ I added the line '-cultures:es-es' 

    self.env.run([
p.join(wix_bin_dir, 'light.exe'),
'-cultures:es-es',
'-ext', ' WixUIExtension',
'-out', '%s' % target,
'%s.wixobj' % wxs_path
])

    This way when you run your .bat which is posted on a comment before this by Udo Hempel you force light.exe to use the -cultures attribute. A list of currently supported languages by WiX can be found here
