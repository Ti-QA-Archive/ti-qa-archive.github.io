Dashboard Allowed for iPhone?

I'm putting the finishing touches on my first app that'll be released on the iPhone first then Android. So I've started comparing my app with Apple's guidelines on what they look for when reviewing an app. My app passes all points (from what I can tell), except one possibility, this one:

10.4 Apps that create alternate desktop&#x2F;home screen environments or simulate multi-app widget experiences will be rejected

To make things nice and organized I used a dashboard setup to launch each feature of the app. Each feature is related to the same industry; so for example the News feature only has news related to the industry and the Calculator feature only has calculators related to the industry. So it's not like each feature is different from one another, but rather they all complement each other. In testing all testers loved how easy it was to switch between each feature, but because of the rule above I'm worried that the app will be rejected.

Many apps on the App Store use Dashboard setups to launch their different features (ex. Applets, uAlberta, GameBox), but do those slip through the cracks or is using a dashboard setup acceptable in certain instances?

  • I was just rejected by apple for using the dashboard and here was there reasoning. So apparently you can't use rounded icons in the dashboard, trying to get clarification. They rejected it by just looking at one of teh screenshots I uploaded and didn't even bother testing the app.

    8.3: Apps which appear confusingly similar to an existing Apple product or advertising theme will be rejected

    We found that your app includes a feature, or features, that mimic an iOS interface or behavior, which is not in compliance with the App Store Review Guidelines.

    Your app and/or application description contains icons with rounded corners that look similar to iOS springboard icons. We've included screenshots to help identify the feature/s in question.

    It would be appropriate to remove or revise these features to make them distinctly different from the iOS behaviors and interfaces to avoid causing user confusion. While the guideline specifies iPod interfaces, the spirit of the guideline includes all iOS interfaces and behaviors.
