ScrollView issue with vertical scrolling with horizontally positioned children

Hi, i'm new but have been making a LOT of progress with titanium. But I'm stuck and need some help with my ipad implementation.

I am trying to have a 3 column list of views, laid out horizontally like floated div's and have them scrollable up and down. sounds simple. I have a window, and inside that a scrollView to which i'm adding all of my views. the problem its the scrollView won't scroll vertically. i've tried several approaches to this… but i need the child views to be laid out horizontally, but the window to scroll vertically.

    var win = Ti.UI.createWindow({
        layout:'vertical',
        height:'auto'
    }); 

    var scrollView = Ti.UI.createScrollView({
        top:0,
        left:0,
        height:'auto',
        width:'auto',
        contentHeight:'auto',
        contentWidth:'auto',
        layout:'horizontal',
        showHorizontalScrollIndicator:false,
        showVerticalScrollIndicator:true
    });
    win.add(scrollView);

    var view = Ti.UI.createView({
        top:0,
        left:0,
        height:330,
        width:256,
        backgroundColor:'#f1f1f1'
    });
    scrollView.add(view);

the problem it seems is that the scrollable container won't adjust it's content height. it works if i add a fixed content height… but it seems like it should be automatic.

any help would be really really appreciated

  • Hi Eric,

    If you want, vertical scrollView, use the property contentHeight:'auto' only. Adjust the child views in a row with top, left properties. first child view: top, left will be 0,0. second child: top: -first view height, left: first view width, third child: top: - (first view height+second view height, left: (first view width+second view width).

    one more thing is in-order to get the scrolling, scrollView content height should be more than your page. then only scrolling appears.

    The best practice is to use scrollView with child as another normal View (scrollViewContainer).
    Add the child views to scrollViewContainer. With this approach, when data is getting added/deleted dynamically in the scrollView,
    there will not be any flickering.
