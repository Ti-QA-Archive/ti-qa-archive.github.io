Android TableView updates: wrong row gets updated, fixed on repaint

I'm using a TableView to display various items. Clicking on each item opens a new window that lets the user update the value (eg. Click on "Amount" and you go to an Amount page where you can update the value).

Each row contains a Label that displays the value. When the user updates the value via the update window, the update window closes and I update the label.text to reflect the new value.

This works fine on the iPhone and works somewhat on the Android. The problem on Android is that sometimes the update shows up in the wrong row - Eg. if you update the Amount value the new value shows up in the Name row instead of in the Amount row. The values are displaced.

If you scroll the rows off the screen and scroll them back in (which I assume does a repaint) then everything is fixed - the correct values are displayed at the correct locations. So it looks like it's a display issue, not a logic issue.

The code looks like this:

var amountValueLabel = amountRow.valueLabel;
amountValueLabel.text = new_amount;

I've seen table.setData() used to deal with this kind of thing, but since my table is a complex view I don't know if I can do that. Is there another way to force a repaint of the table on Android? Or any other work-around?
