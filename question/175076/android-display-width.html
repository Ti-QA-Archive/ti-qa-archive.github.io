Android display width

Hi all,

I have a question about display widths on Android. I request them with:

var w = Titanium.Platform.displayCaps.platformWidth;

But when i make a view with width w, it's never exactly the size of the screen on Android. But it is on iOS.

I need it for a formula to scale a row with background image. The background should be (a scale of) 320x140

Current formula:

var w = Titanium.Platform.displayCaps.platformWidth;
var rowheight = 140 * (w / 320);

The formula is right, but 'w' is not exactly the with of the screen.

Questions:
Is this not the way to do this? Should I use another method in the future?
Is there a way to receive the real width, so I can fix my app?

Thanks!
Marc

  • But when i make a view with width w, it's never exactly the size of the screen on Android

    See, not all pixels are created equal. You need to account for the density of the screen.

    • By default, Titanium.Platform.displayCaps.platformWidth returns pixels.

      A quick, dirty fix is to set the default unit on your android to DP.

      <property name="ti.ui.defaultunit" type="string">dp</property>
