How to get response headers in "onerror" of httpClient?

Hello.

Application type: mobileweb

Titanium sdk 3.3.0.v20131212111946

Ubuntu 12.04

HttpClient does not return headers when application run by "Mobile web preview in browser".

    var xhr = Ti.Network.createHTTPClient({
        async : true
    });

    xhr.onerror = function() {
        Ti.API.debug('Headers!!!!!');
        Ti.API.debug('Content Type', xhr.getResponseHeader("Content-Type"));
    };

    xhr.open('GET', 'http://10.0.0.62:8180/users/profile');
    xhr.send();

Response

  • Status Code:401 Unauthorized
  • Connection:keep-alive
  • Content-Length:20
  • Content-Type:application/json; charset=utf-8

  • Hi Ivan,

    if you read this API-documentation:

    getResponseHeader( String name ) : String
Returns the value of the specified response header.
Parameters
    name : String
    Name of the header to retrieve.
Returns    String

    This API runs on all platforms, you must know the name of header. On Android you can use getAllResponseHeaders(). On the other hand you can install a web proxy to sniff network traffic. My choise is Burb suite.
