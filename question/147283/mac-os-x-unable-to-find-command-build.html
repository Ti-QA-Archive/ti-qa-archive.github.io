    That's very strange. I'm guessing you have a incomplete Titanium SDK installation. I recommend you try reinstalling the Titanium CLI and Titanium SDK.

    Open the Terminal app and run the following:

    sudo npm install -g titanium

    titanium sdk install 3.0.0.GA --force --default

    That should get you back to normal. Let me know if the problem persists.

    • Hmm. I thought this would be an easy fix. :)

      I notice that you have 3.0.0.GA installed in "/Library/Application Support/Titanium". While this should be fine, this is confusing because the "titanium sdk install" command from above should have installed to "/Users/<your name>/Library/Application Support/Titanium".

      I'm thinking that despite reinstalling Titanium 3.0.0.GA, it didn't overwrite the bad install. This should be immediately remedied by removing the 3.0.0.GA folder:

      rm -rf /Library/Application\ Support/Titanium/mobilesdk/osx/3.0.0.GA

      You may need to run that command as "sudo".

      Next, let's see what Titanium SDKs that Titanium is finding:

      titanium sdk

      We should see a list of all installed SDKs. You should still see 3.0.0.GA in the list despite having just removed one of the copies, but this time it'll be in the user's Library folder.

      On a side note, if you see that you have any 2.2.X SDKs installed, you need to delete all 2.2.X SDKs. 2.1.X are OK, but 2.2.X SDKs will cause serious problems since they contain unfinished CLI-related code.

      Finally, try building your app again. :)

      — commented January 23rd 2013 by Chris Barber

    • The easiest way to change the owner is to use the Terminal:

      sudo chown -R `whoami` ~/Library/Application\ Support/Titanium

      If that doesn't work, then maybe you have permission issues too. That can be fixed by running:

      sudo chmod 755 ~/Library/Application\ Support/Titanium
sudo chmod 755 ~/Library/Application\ Support/Titanium/mobilesdk

      That should do it.

    • @Jorge, you have a badly configured setup. You have many weird things happening. I suspect you have run the "titanium" command as sudo at some point. I also think that your SDK might be corrupt. I highly recommend doing a complete uninstall of the Titanium SDK. You don't have to uninstall Titanium Studio.

      To completely reinstall the Titanium CLI and latest Titanium SDK, run the following:

      sudo npm uninstall -g titanium

sudo rm -r ~/.titanium

rm /Users/<username>/Library/Application\ Support/Titanium/mobilesdk/osx/3.0.1.v20121227155121

      You must remove ALL 2.2.X builds and I highly recommend you remove all 3.0.1 or newer builds.

      Next, we reinstall the CLI:

      sudo npm install -g titanium

      3.0.0.GA is the latest Titanium SDK. To reinstall it, run:

      titanium sdk install --force --default

      If you want to test drive the latest and greatest with all of the bug fixes, then run:

      titanium sdk install --branch 3_0_X --force --default

      Note that the 3.0.X branch is not production ready. It's still undergoing testing and there may still be bugs.

      You should be good to go now. You'll be able to create projects with the titanium create command.

      If you have any existing projects, remember to update the <sdk-version> in your project's tiapp.xml to the appropriate 3.X Titanium SDK, then build.
