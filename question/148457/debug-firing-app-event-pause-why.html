[DEBUG] Firing app event: pause. Why???

Hello,

I have a strange problem.
In my iPhone Titanium App I have a tabbar and a webview on one tab. Now when the app is starting up it displays the webview. When I change to another tab while the webview is loading suddenly the event "pause" is called.

I've registered the pause event like so:

    Ti.App.addEventListener('pause', function() {
        Ti.API.info('paused');
    });

But why does that event get called? I never call it myself and I do not press the home button that makes the app getting paused.
This is a serious issue for me, as I do other stuff in the pause function and I don't see a reason why the pause event is getting called.

Thanks for any help!!
