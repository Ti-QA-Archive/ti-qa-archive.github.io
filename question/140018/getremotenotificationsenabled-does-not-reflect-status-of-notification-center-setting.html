getRemoteNotificationsEnabled does not reflect status of Notification Center setting

It seems that getRemoteNotificationsEnabled( ) returns only the status of registerForPushNotifications and not the setting of the Notification Center. For example, if at the start of the app, the user accepts notifications and registerForPushNotifications is successful, getRemoteNotificationsEnabled return TRUE. However, if later the user changes the notification setting to OFF in the Notification Center, getRemoteNotificationsEnabled still returns TRUE. The reverse is the same if at first the user DECLINES notifications (getRemoteNotificationsEnabled = FALSE) and later turns notifications ON, getRemoteNotificationsEnabled still = FALSE.

So what function do you use to get the status of the Notification Setting, so the app can be in sync with the user's settings? If there is no way of getting this status, do you have to ask the user to ACCEPT or DECLINE notifications each time the app is started?

— asked July 23rd 2012 by Eric Folbrecht
