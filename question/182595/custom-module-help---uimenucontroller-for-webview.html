Custom Module Help - UIMenuController for WebView

I am trying my hand at creating a custom module that will add UIMenuItems to the UIMenuController when text is selected inside a WebView.

This is proving to be a challenge.

I have seen this existing module example working with the UIMenuController here:
https://github.com/alexshive/TiPopup/blob/master/Classes/ComAlexshivePopupPopupMenu.m

This is no good with a WebView however.

I have made a module that extends the TiUIWebView. The module is very similar in structure to:
https://github.com/mwfire/titanium-module-extended-webview/blob/master/iphone/Classes/TiUIWebViewProxy%2BExtend.m

Inside the class that extends the WebView (TiUIWebView+Extend.m) I have a "setCustomMenu" method that is adding a new UIMenuItem to the UIMenuController. But it has no affect on the items in the menu that appears.

Here is the method.

-(void)setCustomMenu_:(id)value
{
    ENSURE_UI_THREAD(setCustomMenu_,value);

    [self becomeFirstResponder];

    NSLog(@"Custom Menu called");

    UIMenuItem *highlightMenuItem = [[UIMenuItem alloc] initWithTitle:@"Highlight" action:@selector(highlight)];

    [[UIMenuController sharedMenuController] setMenuItems:[NSArray arrayWithObject:highlightMenuItem]];
}

This is just experimental of course as I don't really know what I am doing :) However I can see the log entry coming up "Custom Menu called", which means the code is successfully being called.

But when the menu appears the items are still "Copy" and "Define" - i.e. the default entries. I get no errors. But I see no change from the code.

Is anybody able to provide some tips on this? Hopefully the question makes sense.

Matthew
