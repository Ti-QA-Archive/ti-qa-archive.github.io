Google Maps v.2 on Ti 3.0.2

Ti Studio 3.1 - Ti SDK 3.0.2 - Mac OSX - Android App

In the Documentation it says:
For Titanium SDKs prior to 3.1, download the ti.map module from https://marketplace.appcelerator.com/.
Unpack the module to your Titanium SDK home directory, for example, /Users/…/Library/Application Support/Titanium, which make the module available to all your projects.
Alternatively, you can copy the ZIP file to your Resources folder, which makes the module available to a single project.

So in my case, I am using Ti SDK 3.0.2 (since there is a bug I'm getting with the TextFields in 3.1), and have downloaded the Module for Google Maps v. 2.

The Issue is, in tiapp.xml, when I select SDK 3.0.2, I get:

  • I'll explain clearer what I'm doing step by step.

    Using Ti Studio 3.1, SDK 3.1.0GA (just updated), Running on Mac OSX 10.8, Target Google API 4.0.

    Downloaded the distribution from here: https://github.com/appcelerator/titanium_modules/blob/master/map/mobile/android/dist/ti.map-android-2.1.0.zip

    In tiapp.xml

    <modules>
   <module platform="android">ti.map</module>
</modules>

    also

    // In Manifest node
<!-- Allows the API to download data from Google Map servers -->
            <uses-permission android:name="android.permission.INTERNET"/>
            <!-- Allows the API to cache data -->
            <uses-permission android:name="android.permission.WRITE_EXTERNAL_STORAGE"/>
            <!-- Use GPS for device location -->
            <uses-permission android:name="android.permission.ACCESS_FINE_LOCATION"/>
            <!-- Use Wi-Fi or mobile connection for device location -->
            <uses-permission android:name="android.permission.ACCESS_COARSE_LOCATION"/>
            <!-- Allows the API to access Google web-based services -->
            <uses-permission android:name="com.google.android.providers.gsf.permission.READ_GSERVICES"/>
            <!-- Specify OpenGL ES 2.0 as a requirement -->
            <uses-feature android:glEsVersion="0x00020000" android:required="true"/>
            <!-- Replace com.domain.appid with your application ID -->
            <uses-permission android:name="com.domain.myappandroid.permission.MAPS_RECEIVE"/>
            <permission
                android:name="com.domain.myappandroid.permission.MAPS_RECEIVE" android:protectionLevel="signature"/>
            <supports-screens android:anyDensity="true"/>
// in application node
<!-- Replace "PASTE YOUR GOOGLE MAPS API KEY HERE" with the Google API key you obtained -->
<meta-data android:name="com.google.android.maps.v2.API_KEY" android:value="MY_KEY_I_GOT"/>

    In My Code:

    var MapModule = require('ti.map');

var userAnnotation = MapModule.createAnnotation({    // Ti.Map.createAnnotation
      latitude:myLat,
      longitude:myLon,
      //pincolor:MapModule.ANNOTATION_RED,
      title:user.first_name + ' ' + user.last_name,
      subtitle:user.custom_fields.address,
      image: 'assets/icons/pin.png',
      animate:false,
});

// GMaps vers. 2
var mapview = MapModule.createView({    
      top: '467dp', left:0, right:0, height:'180dp',
      mapType: MapModule.NORMAL_TYPE,  
      region: {latitude:myLat, longitude:myLon, 
                  latitudeDelta:0.01, longitudeDelta:0.01},
      animate:true,
      regionFit:true,
      userLocation:false,
      annotations:[userAnnotation]
});
scrollView.add(mapview);

    As the SplashScreen is Loading, the App Crashes (the famous Android:"App stopped responding").

    If I compile with Ti SDK 3.0.2 and remove <module platform="android">ti.map</module> because it gives an error as described above, the App runs, but the will hit the Runtime Error (due to the fact it's looking for ti.map.

    Any way of using Google Maps Version 2 with Ti SDK 3.1. I've followed the steps listen here carefully:
    http://docs.appcelerator.com/titanium/latest/#!/api/Modules.Map
