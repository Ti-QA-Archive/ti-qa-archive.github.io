How to delete a Row of a TableView in Android?

I'm trying to convert my first app (EsoClassics, already present in App Store for iPhone and iPad) to the Android platform, but I'm experiencing a problem: the app stores a list of bookmarks and in iPhone, if I want to delete an old bookmark, there is a perfect implementation of native function that allows to delete the single row I wish, simply swiping or putting the tableview in editing mode and then tapping the button on the row.

Well, this is not possible in Android, because 'editable' and 'editing' are not available for this platform and I'm concerned for this thing. For completeness, I (still) don't own an android device, so I don't know how apps are resolving this problem.

I know there is a page in kitchensink where you can delete a row in android, but this is only for the first row and this is not suitable for my app.

So can someone tell me a way to resolve this or explain the use of the line below in Android (I have not read it's only for iPhone)?

tableview.addEventListener('delete', function(e) {
     ... 
    });

Thanks!

  • The feature you are trying to emulate doesn't exist on Android (or at least, the implementation is different)

    What is usually done on Android is that:

    • You "swipe" the table view in edit mode. (usually done using a menu).
    • From this moment, every row now has a checkbox next to it (hasCheck=true)
    • The user selects selecting which items are to be deleted.
    • The user confirms the deletion by clicking a button. (wherever it is located)

    The deletion confirmaiton button code could go someting like this:

    confirmButton.addEventListener('click', function() {
    for (var i = 0; i < table.data.length; i++) {
        if    (table.data[i].hasCheck == true) {
            try {
                tableview.deleteRow(i);
            } catch (E) {
                Ti.UI.createNotification({ message: E.message }).show();
            }
        }
    }
});

  • Ok, since the method with 'hasCheck=true' doesn't work for me (because I'm not able to delete from the database), this is how I've resolved the problem:

    • created a button to enter in so-called 'edit mode', where the original tableview hides and a new tableview shows
    • added a click eventListener that deletes the row in the table when it's clicked
    • added a database execution to delete the record (this was a great problem)
    • added a button to return to the original tableview, but without the deleted rows.

    If someone is interested this is the code for the tableview:

    delTab.addEventListener('click', function(e)
{
    delTab.deleteRow(e.index);
    var db = Ti.Database.open('bmdb');
    db.execute("DELETE FROM bmdb WHERE header = ?", e.rowData.header);
    delete bm[e.index];
    delTab.setData(bm);
// bmdb is the bookmarks database which is best ordered by headers
});
