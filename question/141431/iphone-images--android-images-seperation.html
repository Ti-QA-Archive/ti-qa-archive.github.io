iPhone images & Android images seperation

I've seen that when building my Android app, also all the @2x images in the /images folder are also added to the package, even though these are not used by Android. That results in very big apk files.

But when I put all the iOS images in the iphone/-folder, and all android versions only in the subfolders of the android/ folder (resulting to no images left in the images/ folder), the images are no longer displayed on iOS.

I've tried this:

var logo = Ti.UI.createImageView({
        width : "176dp",
        height : "70dp",
        image : "images/test.png"
});

and this

var logo = Ti.UI.createImageView({
        width : "176dp",
        height : "70dp",
        image : "/images/test.png"
});

Neither of them works.

How can I accomplish a smaller Android APK, not including the iOS retina images?

    You need to put the images in

    app_folder/Resources/android/images  <-- for android

app_folder/Resources/iphone/images  <-- for iphone

    and use

       image : "/images/test.png"

    This works correctly only with tisdk > 2.0
