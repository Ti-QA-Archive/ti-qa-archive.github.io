ACS Custom Objects in Rails

I'm using the Cocoafish Ruby library https://github.com/cocoafish/cocoafish-ruby to create a website that will manage content for an ACS based mobile app.

So far it's been working great but I've hit a wall with Custom Objects, no matter what I do I can't get one to create in ACS using the Ruby library. The API call is simple as follow:

result = Cocoafish::Client.post("objects/app/create.json", {:appname => "name of app", :user => "502a7a33b68553743d0e3238", :object_user_id => "502bcdd4567afe48e900385f"})

but it just keeps saying '400 bad request - missing fields'.

Can anyone from Appcelerator or otherwise confirm whether what I'm trying to achieve is actually possible or not?

  • Ahh, thanks Wei, that makes sense now! Got it working as follows:

    result = Cocoafish::Client.post("objects/app/create.json", {:fields => {:appname => "hi", :user => "502a7a33b68553743d0e3238", :object_user_id => "502bcdd4567afe48e900385f" }})

  • Example how ACS web uses Cocoafish client:

    result = Cocoafish::Client.post("objects/#{params[:class_name]}/create.json", { :user_id => params[:user_id], :acl_id => params[:acl_id], :photo => params[:photo], :photo_id => params[:photo_id], :fields => params[:fields] } )

    :user_id(owner of the object) and :fields(json object for custom fields) are required fields.

    If you use update.json, :id => params[:id] is also a required field.

    So for what I'm understanding in your case(if you were trying to set the object belong to :user):

    result = Cocoafish::Client.post("objects/app/create.json", { :user_id => "502a7a33b68553743d0e3238", :fields => {:appname => "name of app", :object_user_id => "502bcdd4567afe48e900385f"} } )
