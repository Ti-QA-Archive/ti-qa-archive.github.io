window doesn't add view

Hi everyone,
i have this strange problem which maybe is very simple: in my application i have 2 windows, a button andone view.
At the beginning the view is added to the first window.
I'd like that pressing the button the second window adds the view and then opens itself.
The problem is that when the second window opens the view is not added, but, if i press the android-back-button and re-press the button the view magically appare in the second window.
Why at the first press the view isn't added?
Here the following code:

var win1 = Ti.UI.createWindow({title:'win1', backgroundColor:'white'});

var view = Ti.UI.createView({top:'0%', width:'100%', height:'13%', backgroundColor:'black'});

var win2 = Ti.UI.createWindow({title:'win2', backgroundColor:'white'});

var btn = Ti.UI.createButton({title:'win2'});

btn.addEventListener('click', function () {

    win2.add(view);
    win2.open();
});

win1.add(btn);

win1.open();

Conversely, if i create a second view when i open the second window, the view's added and i don't know why i have to create a second view instead of using the same as before.

Thanks in advance for your help.

  • You cannot use the same view in multiple places. A view can only have a single parent. You can create a factory function that will create duplicate views for you, if thats what you want to.

  • Stephen is correct in his approach… create the object for each view. In theory, you could remove the view and add it to a different view…

    btn.addEventListener('click', function () {
    win1.remove(view);
    win2.add(view);
    win2.open();
    win2.addEventListener('close',function(){
        win2.remove(view);
        win1.add(view);
    })
});

    No guarantees that would work, though. You'll also probably have issues about how each platform handles such code. You're better off creating a Widget (if you're using Alloy) or create your own CommonJS module to create instances of a view.
