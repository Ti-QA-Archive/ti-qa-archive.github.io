iOS module including a 3rd party library

Hello, all. I am attempting to use a 3rd Party library within an iOS module and could do with some assistance on the best way to go about this. The library I am attempting to use is this iOS Pusher library https://github.com/lukeredpath/libPusher

The library’s readme, recommends using CocoaPods dependency manager. However, it appears that iOS Titianium modules don't support CocoaPods. Or maybe they do, I just can’t find any information on how to configure it to get it to work. Closest thing I could find is this link, but there isn’t a definitive answer.
https://developer.appcelerator.com/question/162277/ios-module-fails-to-build-after-pod-installed-brightcove-sdk

I searched the module docs for instructions on how to add a 3rd party library, but the only thing I could find was this link for adding a 3rd party framework.
http://docs.appcelerator.com/titanium/3.0/#!/guide/iOS_Module_Project-section-43288810_iOSModuleProject-AddaThird-PartyFramework

In the end, I tried adding the Pusher Library as a sub-project. Finding this link very useful. http://www.cocoanetics.com/2011/12/sub-projects-in-xcode/

I can now get my module to build. However it only builds and runs on physical devices. On the simulator I get the following error in the console, so it must not be building it correctly for the simulator architecture:

Undefined symbols for architecture x86_64:"_OBJCCLASS$_PTPusher"

Can anybody offer any help on the best way to go about this? Thanks all!
