Weird: objects disappear after longpress on a tableviewrow

Demo of the problem: http://www.youtube.com/watch?v=RnFVqOhENeI&feature=youtu.be

This is happening to all of our apps. Some are in alloy, some in commonjs.

Remote data loaded into a simple tableview, that contains some elements like a thumbnail, title, category. When you do a longpress, some of the elements disappear.

Example of the articleRow view (xml):

<Alloy>
    <TableViewRow id="articleRow">
        <View class="articleContainer">

            <ImageView id="thumbnail" class="thumbnail">
                <View class="categoryContainer" id="categoryContainer">
                    <Label class="category" id="category"></Label>
                </View>                    
            </ImageView>

            <Label id="title" class="title" />

        </View>
    </TableViewRow>
</Alloy>

TIA.

    Seems like a bug to me. The only way I can prevent the background colour of the label from disappearing is to set 

    selectionStyle : Ti.UI.iPhone.TableViewCellSelectionStyle.NONE

    on the tableViewRow.
