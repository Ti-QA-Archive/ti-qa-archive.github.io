Yet Another Password Security Question

I've been looking into this situation for a while and I haven't been able to find an answer that fits my scenario yet.

I need to be able to store the users password to our login system on the device. The traditional method of using a hash with salt won't work because it needs to be reversable, our authentication system (while over SSL) requires the original password (not hashed or encrypted).

The app itself is going to be released on both iOS and Android, so using two different methods to do the same thing seems silly to me (e.g. Keychain on iOS or dMarie's SQLite encryption module for Android).

I'm currently storing the users password in the devices properties space but it's being stored un-changed (clear text).

What is the best option to secure the password on BOTH platforms with the same (or at least as similar as possible) process? If an encryption process was used that required a key, is the key accessible in the app once released?
