Android and language xml files - no spaces in name attribute

I learned this the hard way, but your app will crash and have to be forced closed if you have a space in your name value in one of your language xml files. It'll run just fine on an iPhone but won't work at all on the Android emulator. You'll also see an error in a file called R.java.

Do this

&lt;string name="contact_us"&gt;Contact Us&lt;/string&gt;

Don't do this

&lt;string name="contact_us"&gt;Contact Us&lt;/string&gt;

— asked February 9th 2011 by Kevin McCallum
