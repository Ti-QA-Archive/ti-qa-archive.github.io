Disable Context Menu in a WebView?

I would like to completely disable the default Context Menu that appears when a user selects text in a WebView. Note that I still want the user to be able to select text - I just don't want the system menu with "Copy", "Define" etc appearing.

Does anybody know how to do this?

Note that the WebView is loading local HTML that I have complete control over.

I have tried putting this into the HTML of the WebView:

html {
   -webkit-touch-callout: none;
}

Which is theoretically supposed to do exactly what I want. But it does nothing.

Others point to:

html {
   -webkit-user-select: none;
}

Sure that completely disables the context menu - because the user can no longer select anything. Which is not what I want.

So I am looking for other options at this point. Any clues appreciated.

Matthew

PS: As a side comment the lack of support in Titanium for working directly with the UIMenuController is proving to be extremely frustrating.
