Keep getting invalid binary in itunes connect...

I validated the project just fine, and uploaded just fine. but later on, I find that itunes connected says "Invalid Binary". In the email I got:

"Invalid Signature - Make sure you have signed your application with a distribution certificate, not an ad hoc certificate or a development certificate. Verify that the code signing settings in Xcode are correct at the target level (which override any values at the project level). Additionally, make sure the bundle you are uploading was built using a Release target in Xcode, not a Simulator target. If you are certain your code signing settings are correct, choose "Clean All" in Xcode, delete the "build" directory in the Finder, and rebuild your release target."

I have verified I'm using the distribution certificate, and that I user RELEASE as the build option… how can I overcome this?

    • Yaniv,

      My experience is that the Keychain is VERY difficult to get setup properly. Until you get it setup properly, it will not be able to sign properly. I feel your frustration, or shall I say I have felt it before! :-(

      It might have to do with being or not being root certificates, seems I recall that something needs to be a root cert to work properly and be able to sign.

      I know you need your developer cert which is signed by Apple, as well as the Apple Worldwide Developer Relations Certificate. Also, there is something about the triangles inside the Keychain Assistant, you should see them next to the certs when you look in the Keychain Assistant.

      My best suggestion is to go to the Apple Developer site and search for the doc on setting up the keys and follow it closely, after removing the keys you have in place already. I don't know if there is a way to get it working with the keys you have in place, there might be.

      Maybe one of the mobile guys will jump in with some info on setting up the keys. I know it's tricky, as I mentioned above.
