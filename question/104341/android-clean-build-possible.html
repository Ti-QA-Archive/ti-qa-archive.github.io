Android Clean Build possible?

Android, Ti mobile SDK 1.6, on Mac.

My problem is that for some reason when I change images in my resources (ie: modify the graphic icon for a tab bar in an image editor) resave etc it doesn't make it to the emulator.

When I check in the build/android/bin/assets/Resources/images it still has the old one.

I can't seem to get a "build clean" to force it to rebuild all the assets… I've tried a bunch of stuff, but deleting parts of the build directory just throw Titanium into a wobbly.

How do I force Titanium to do a clean rebuild so that update files from my source (basically Resources/images) directories into the build….

?

  • The only thing that should be necessary is to visit the Edit tab and click "Save changes", even if you don't change anything like the version number. The "Save changes" will cause the timestamp of the the tiapp.xml file to change (ie be "touched") and that should trigger a full rebuild.

    That has always worked for me for Android; for iPhone sometimes I find it helpful to delete all of the contents in the project/build/iphone folder (leaving an empty folder). For Android, in my experience that has not been necessary. In fact, like you, it seems to cause problems.
