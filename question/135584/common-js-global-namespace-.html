    • No. CommonJS module creates new JS context - that means, it is isolated and clean JS environment in which you can do whatever you can do in other JS contexts. In that context, there is one global scope and there are scopes created by functions.

      If you declare a variable in CommonJS module (to declare a variable, "var" keyword must be used), it will be local variable declared in that scope. If that scope is global scope, that variable will be visible in that module, but not in other modules because other modules are new JS context (which their global scope).
