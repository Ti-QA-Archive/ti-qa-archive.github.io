How do you deal with different screen sizes?

A UX-architect sent me a project where in the "home" there are 5 buttons. I grouped them in one view, then I compiled for iPhone simulators 5/6/6+. Of course the view remains the same fixed size in any platform but in iOS6/6+ there's more space around it. I'm NOT talking about bitmap graphics but coded view and buttons with no graphics inside.

How do you deal with this? Is there any "magic" way to scale the content of the view or should I code by hand, calculating top, left, width and height of views and buttons for every device?

    • The problem with your solution, (hardcoding the width) is that if you get a smaller screen, your UX will break.

      For example, if you have a screen that's 300 width, and your videw is 300, PLUS the margins (let's say 10 left and 10 right) you're already off screen.

      Using only margins, your views will always adjust to the current screen.

      I develop mainly for android and iOS both, so i can have lots and lots of different screen sizes.
      If you develop only for iOS exclusively, not too much because you already know the devices out there and their resolutions.

  • Hi London Moe,

    You can scale UI controller according to device resolution by calculating based on screen. for example 100x35 button is showing perfect in iPhone 5s so you can calculate it size for iPhone 6 and 6+ based on iPhone 5s resolution.

    e.g

    Alloy.Globals.GetWidth = function(value) {
    var temp = (value * 100) / 320;
    return parseInt((Alloy.Globals.platformWidth * temp) / 100);
};
Alloy.Globals.GetHeight = function(value) {
    var temp = (value * 100) / 568;
    return parseInt((Alloy.Globals.platformHeight * temp) / 100);
};

    And use it in your controller. for example as per earlier discussion calculate it

    e.g

    var density;
if(OS_IOS){
    density = "dp";
} else {
    density = "px";
}
var btnView = Ti.UI.createView({
    width : (Alloy.Globals.GetWidth(100) + density),
    height : (Alloy.Globals.GetHeight(35) + density),
});

    Here i define density depends on platform because when you calculating it by using device resolution it return in pixel so you need to assign it "px" for android. and for ios it return non retina resolution so for that need to assign "dp".
