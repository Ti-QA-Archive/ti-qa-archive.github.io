Require causes table view to be hidden if it's not the first child in the required view. - Alloy

Hello,

In my application I am busy creating dynamic table views with collection data.

But for some reason, if the table view is specified inside a required view (As in: <Require src="<viewname" /> and is not the first child, it stays hidden.

Test case:

index.xml:

<Alloy>
    <Window id="index">
        <Require src="test">
    </Window>
</Alloy>

test.xml:

<Alloy>
    <View> <!-- No matter what kind of view -->
    </View>
    <TableView>
        <TableViewSection>
            <TableViewRow title="Test 1"></TableViewRow>
            <TableViewRow title="Test 2"></TableViewRow>
            <TableViewRow title="Test 3"></TableViewRow>
        </TableViewSection>
        </TableView>
</Alloy>

index.js: 

$.index.open();

Causes table view to stay hidden.

If the <View> element is removed inside test.xml, the table is shown.

Does anyone know what's causing this?

Thanks in advance

Patrick van Vuuren

    Can't find it looking in the docs, but a required module should only return a single view, you need to wrap your view and tableview in another view:

    <Alloy>
    <View>
        <View> <!-- No matter what kind of view -->
        </View>
        <TableView>
            <TableViewSection>
            <TableViewRow title="Test 1"></TableViewRow>
            <TableViewRow title="Test 2"></TableViewRow>
            <TableViewRow title="Test 3"></TableViewRow>
            </TableViewSection>
        </TableView>
    </View>
</Alloy>

  • Allright figured it out:

    When you directly require a view component using require in alloy xml, it may have multiple views directly attached to the <Alloy> element.

    When you add a view using Alloy.createController("<viewname>", {}).getView(), it must have one view, and not more than one. It always picks the first element, so you won't see the others.
