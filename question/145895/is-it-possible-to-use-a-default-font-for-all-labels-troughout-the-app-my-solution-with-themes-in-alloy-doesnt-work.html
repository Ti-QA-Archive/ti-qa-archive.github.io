Is it possible to use a default font for all labels troughout the app? my solution with themes in alloy doesnt work

Hello,

What i want to achieve is to create some default style sheet, in which i define the default style of all elements. I got this working, because i use alloy and i am using themes. All works as i suppose it would work. i create a theme, and all elements in the app correspond to the attributes in the theme if they have not set the attributes themselves.

HOWEVER, this does not work for fonts, as i feared. i have the following dummy tss in my folder themes -> mytheme -> styles -> app.tss

"Label": {
    color: "green",
    width: Ti.UI.SIZE,
    height: Ti.UI.SIZE,
    font: {fontFamily : "serif", fontSize : 32}
},
"View": {
    backgroundColor: "#3F00",
    width: Ti.UI.SIZE,
    height: Ti.UI.SIZE
},
"Window": {
    backgroundColor: "gray"
}

The only thing you actually should look at is the label, the font attribute. I say it should use the fontFamily serif with a default fontSize of 32.

When i just make a simple label, it becomes green with font family serif and fontsize 32. But as you can understand i dont always want the fontsize to be 32, that depends very much. but i do want to make the fontFamily allmost always serif, thats why i set it to serif.

So when i create a label which has the following stylesheet:

".label": {
    font: {fontSize : 12}
}

the label will have a fontSize of 12, but it wont have the fontFamily serif anymore, it just switches back to the default fontFamily for android.

I already tried to do the following:

".label": {
    font.fontSize : 12
}

but this gives an error.

I assume that you cant really change only the fontSize of a font defined in a theme, but i would like to be sure there is no possibility to make it work. And if not, i would like to suggest this as a feature for one of the following releases.

maybe an solution would be font: {fontSize: 12, fontFamily: &#39;theme&#39; }
in which theme could also be something like parent or auto or whatever, but not default.

I still have a little hope that there is already a solution to this, and for the time being, i guess the fontFamily should just be stored in the Ti.Alloy.Global.

Sincerely,

Hylke Bron

    I understand your problem, but as it stands, no, there isn't a way for alloy to merge individual objects while merging styles. It does bring up a good point though that perhaps the Titanium API should expose discrete properties for the more common complex objects, like font. I would say that I could investigate having alloy look into each property and attempt to determine if it should merge objects, but I don't this is intuitive behavior in all cases and I think most would expect that each individual property would be overwritten, not do a deep dive into each property of an object of the property.

  • The best way here is to become aware that this is part of a bigger issue that you run into much more often: you want to customize Titanium's elements. There are elegant solutions, but it does take a little effort to use them.

    Once you realize that there's a bigger problem to be solved, you'll be prepared to invest that effort.

    The solution below also happens to be one that works equally well on all platforms: in fact, it allows you to fix recurring platform-specific issues with UI elements in a single place.

    To do this…

    Create special modules

    • create a module called UI.js
    • create an Elements folder
    • in that folder, create one module for every Ti object you want to customize

    Note: in this case, keep it simple and start with label.js. This module and all others will have only one module.export (I'll describe the Label example later).

    • make UI.js require all the files in that folder
    • give UI.js a .create[something] export object for each of the files

    Merge your custom element builders with Ti.UI

    • then loop through all .create[something] methods of the Ti.UI object
    • for each method in Ti.UI, if your UI does not have it, set Ui[method] to Ti.UI[method]

    Use what you built

    From now on, everywhere in your app, use only UI.createLabel, UI.createEtc when creating UI elements. If you're smart, use search and replace right now to turn every Ti.UI.create[] into UI.create[]
    If there's a custom creator in your Elements folder, your UI.js will use that; if not, then it automatically picks the regular Ti.UI creator for your desired element.

    The Label example

    Your Label.js file should preferably accept the same parameters as Ti.UI.createLabel: in this case as so often, a single options object. Its code can be simple: it should create a label using Ti.UI.createLabel, but before you pass along the options object, you edit its font property to whatever you want.

    Hope that helps!
