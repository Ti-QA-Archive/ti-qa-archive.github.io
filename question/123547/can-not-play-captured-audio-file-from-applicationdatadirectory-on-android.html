can not play captured audio file from applicationDataDirectory on android

I am having trouble playing back an audio file (in the android emulator) that is captured to android's 'applicationDataDirectory' or 'app-private' location.

Is there some limitation about read/writing from this path? If I modify my path to use the external storage the file, there is no problem with the capture or playback. This failure occurs using either the Ti.Media.Sound or Ti.Media.AudioPlayer objects. I also found that there is a local file reading bug in the 1.7.2 SDK that prevents, so I have a sneaking suspicion that these may be related.

works:
file:///sdcard/com.app.mediapriori/20110701090059.3gp
fails:
file:///data/data/com.app.mediapriori/app_appdata/20110701085342.3gp

Although I have solved my problem by using the sdcard (i.e. Titanium.Filesystem.externalStorageDirectory) , I would prefer to keep data in my local app's directory at first.

Thanks,
Eric
