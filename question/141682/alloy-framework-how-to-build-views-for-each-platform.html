Alloy Framework: How to build Views for each platform?

Hi, i’m testing the Alloy and it’s awesome! But i’m a little bit confused with views for each platform… the documentation is very poor and i hope that will be released with the anounciment.
But, how can i build view platform specific? The app folder is at the root project folder… can i make an app folder at Resources/android folder?

I’m building an application after months of plan ans specifications… i choose Titanium and Alloy for MVC. But if these docs won’t be updated soon (and soon is at september 1st week)… i’ll choose another MVC framework because i have time to build my app.

My platform: mobile for Android/ipad/iphone, sdk 2.1.1GA and Alloy MVC Framework

  • The app folder must remain at the project root.

    A way you can make platform specific markup is to use the "platform" attribute in your view's XML. This gist shows you how you can use it: https://gist.github.com/3529762

    We are currently structuring a way to have platform-specific files as well for your views, controllers, styles, etc… We will keep you posted on that.

    Finally, you can still use platform-specific files and assets by adding them to the app&#x2F;assets folder. So if you had android-specific items, you could add them to app&#x2F;assets&#x2F;android, just as you would in Resources. We are having discussions on this structure currently as well, with the aim of making it as intuitive as possible.

    Additional comments with regards to this question can be found with your original mention of these issues on the blog: http://developer.appcelerator.com/blog/2012/08/alloy-framework-is-ready-for-pre-release.html
