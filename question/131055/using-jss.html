Using JSS

I'm trying to refactor my app to take advantage of JSS, but I'm having some trouble. I haven't found much documentation that's terribly useful and, on top of that, my app is a little different in that every window is created by a factory (e.g. createLoginView()).

Because there's no "context" (as it's referred to in some of the docs), I've created a single app.jss file in Resources&#x2F; and am creating platform specific settings by setting id/class values as Ti.Platform.osname + &#39;_location_icon1 (Right now, I'm only dealing with one Android display). Then, in my app.jss file, I have the following definition:

  #android_location_icon1 { bottom: '100dp'; right: '5dp'; }
  #iphone_location_icon1 { bottom: '100dp'; right: '5dp'; }

Two problems:

  • TitaniumStudio complains about the bottom and right values if they have no unit or if the unit value is anything other than px. I'm not sure of the rules for defining these styles, but since JSS extends the object definition, I guess I assumed that the same values would be accepted.
  • Nothing seems to actually work. I think my selector names are correct and I think the app.jss file should be getting loaded, but something is going wrong.

The state of JSS documentation (that I've been able to track down) leaves me making a lot of assumptions and guesses, but they've seemed reasonable enough given what I've read. Nonetheless, things aren't coming together. Is there something I'm missing in how all of this comes together?

Thanks.

  • I tried using JSS when it first was released and ran into the same issues as you. A couple other options:

    Follow the tweetanium model for styling, where you have a styles.js, and then do something like:

    var label = new Ti.UI.createLabel(tt.combine($$.mylabelclass, { //custom properties }));

    Or use titanium redux, which uses "RJSS".
