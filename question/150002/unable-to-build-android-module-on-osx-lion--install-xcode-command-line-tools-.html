  • Accepted Answer

    I was able to get around this by changing the Android.mk to use the android ndk platform include instead.

    In Android.mk I changed -I$(SYSROOT)/usr/include to -I$(NDK_PATH)/platforms/$(TARGET_PLATFORM)/arch-arm/usr/include

    In the main build.xml I had to export the ndk path by adding <arg value="NDK_PATH=${ndk.path}/"/> to build.ndk macro definition.

    I've updated the bug with these details so hopefully this can get resolved in the main Titanium build

  • In my case the problem was a bunch of different things that might be useful for others to check.

    1. Use the right workspace. When you create a module for Android, you're warned if your workspace path contains spaces. Android SDK can't handle that. So you create a new workspace. But sometimes you don't get warned, and you end up with a module that will never work because it's in the wrong workspace. Other times, I restarted Titanium because I changed some other setting and figured restarting might help; of course I forgot that I needed to use the new workspace, so I kept fixing the wrong module.

    Also, a lot of preferences vary by workspace. For example, you need the Android NDK to build your module. So if you didn't have it set already, you download the NDK and point Titanium to it via Preferences > Platforms > Android. But then you switch workspaces, and that preference is empty again. Really, really check if everything you've tried to do was applied to the correct workspace.

    1. In this tutorial and others, you're advised to edit [yourmodule]/android/build.properties. That's great advice, but you do not need to set the google.apis property. If you do, you may get new warnings about Google APIs missing certain directories. That might be a problem later on, but you don't need Google APIs to build your module. So don't bring this grief upon yourself just now.

    2. Somewhere, I forgot where, I found a mention that you cannot use Android NDK higher than version 8 because Titanium cannot handle higher versions. Get the right NDK (android-ndk-r8e).
