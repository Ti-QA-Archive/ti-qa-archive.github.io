    Once you have the new sqlite3 file created in your php script, you can download to iOS or android using the techniques I describe in this among other threads. I also have more information here.

    If the files are very "large" (or even if not), instead of using the older typical httpClient method of writing the responseData to a file, you should consider having it download directly into a file.

    The iOS SDK supports that via the .file property, which must be set between the xhr .open() and .send() calls. An example is in this thread or the kitchen sink xhr_filedownload sample. The Android mobile SDK does not yet support that property, however this thread shows a work-around which seems to work very well for me.
