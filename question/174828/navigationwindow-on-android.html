  • NavigationWindow isn't supported on Android, as you've found. In the case of tabs, you don't need one. (See additional note below for iOS.) You can simply call the tab's open() method to open new window within that tab. The user will be able to tap their device's Back button to close that new window and return to the previous one.

    If you're working outside of a tab group, you might consider Fokke Zandbergen's cross-platform UI code at http://fokkezb.nl/2013/10/21/cross-platform-ui/

    Please note that there's no reason to include a NavigationWindow within a tab on iOS. That functionality is already built into tabs. You should remove the <NavigationWindow> elements from the iOS version. Again, to open new windows in a tab, you'd call the tab's open() method. iOS users would see a back button in the tab's navbar.
