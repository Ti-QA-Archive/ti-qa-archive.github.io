SQLite Triggers problem

Hi All

Let`s say I have a table A in mydatabase.sqlite sitting in project resources root folder. A trigger has been created through SQLiteManager GUI. So far so good. When I install mydatabase through

Titanium.Database.install("mydatabase.sqlite", "mydatabase");

while app is running, triggers do not get copied from mydatabase.sqlite to mydatabase.sql.

So, any trigger test queries failed.

I wonder why triggers are not being copied?

Thanks

  • Is your database kept within the resource folder. if its directly in resource folder then it will work, but if its within and sub directory, then you need to give its proper path.

    If we do not provide proper path then it create the new database, and will not override the database which we have created.
