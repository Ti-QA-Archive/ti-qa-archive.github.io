  • If I understand the question correctly, it is whether or not you can use XCode as the IDE to develop Titanium apps. Not how to extend Titanium using objective C.

    While you could use any editor you want (including XCode or TextMate or DashEdit or whatever), what you lose is the integration with the debugger and build scripts. If you want to go that route, you may want to consider Titanium Developer as the build mechanism after you edit your code in the editor of your choice. Ti Developer was the precursor to Ti Studio, but Appcelerator end-of-life'd it once Studio was complete so they could concentrate resources on Studio and the various SDKs, etc. While it is no longer available from Appcelerator, it was an open source project so other users have taken it, made various enhancements, and it is available from the links in this thread.

    IMHO, using Studio is "better" and XCode is less "desirable" just due to the integration features:

    • Code completion already knows about Titanium classes
    • Debugger integration with breakpoints, single-step, etc.
    • Easy to download new mobile SDKs when available
    • Built-in form editor for most common Tiapp.xml properties
    • Built-in script support for building projects

    PS - "Modules" are a way you can extend Titanium when you need a feature which does not exist in the Titanium SDK yet. Modules are platform specific and written in the native language for that platform. Thus for iOS you do use XCode and obj-C to write those. Ivan supplied the link for more information on that. But I don't think that is what you were asking here, if I interpret the question correctly.
