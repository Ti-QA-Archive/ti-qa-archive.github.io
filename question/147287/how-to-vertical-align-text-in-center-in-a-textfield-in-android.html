  • Hi

    I think you will find it is working - just your current dimensions and font size choices mean that the internal padding (which you cannot change) is forcing the text towards the top and clipping it as you have seen.

    Here is my test code for you where I have extended the dimensions and used a border and background colour - so it stands out. All other values you provided are the same though.

    You will see the text is vertically aligned within the box. I must say that I have only been able to test Ti.SDK 2.1.3 and Ti SDK 3.0.0 but they both work. I have no knowledge of any Ti SDK 2.1.1 specific issues.

    Try the sample code and see what happens - if it sits in the middle (as in between the top and the bottom) then it is about the height and font size - once you realise that the top and bottom padding is there -which you cannot change.

    var whereField = Ti.UI.createTextField({
    backgroundColor: '#fff',
    backgroundImage: '/images/textfield.png',
    borderColor: '#000',
    borderWidth: 1,
    color: '#000',
    height: '100dp',
    hintText: 'Where ?',
    keyboardType: Ti.UI.KEYBOARD_DEFAULT,
    paddingLeft: 5,
    returnKeyType: Ti.UI.RETURNKEY_DEFAULT,
    top: 10,
    //verticalAlign: Ti.UI.TEXT_VERTICAL_ALIGNMENT_TOP,
    verticalAlign: Ti.UI.TEXT_VERTICAL_ALIGNMENT_CENTER,
    //verticalAlign: Ti.UI.TEXT_VERTICAL_ALIGNMENT_BOTTOM,
    width: 252
});

    I have left the top and bottom values commented so you can quickly swap them in and out.

    Let me know.
