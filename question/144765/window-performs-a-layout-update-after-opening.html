Window performs a layout update after opening?

Hey all, first post here, so thanks for taking a look :)

Using Studio 3.0.1 and Alloy 0.3.1…

I can't seem to figure out if there is a nice way to get a window to ensure it's own layout before opening.

i.e. I have a somewhat basic tabbed application. In one of the tabs, when you click on something, it opens another window in that tab. However, on both the simulator and on my iphone 4s, the window opens without the layout calculated for an instant so it looks garbled. Very quickly, the actual desired layout gets rendered.

Basically, it seems that the "non-styled" view renders, then an instant later it actually gets applied.

I feel like there is probably a way to hack the window to render by opening it not visible and then sliding it onto the screen or something like that but there has to be a better way….

Any ideas? Here is the code…

The view is simple enough:

<Alloy>
<Window class="container">
<ScrollView id="view">
<ImageView id="pic" />
<Label id="name"></Label>
</ScrollView>
</Window>
</Alloy>

The controller is simple too:
var args = arguments[0] || {};

$.pic.image = args.pic;
$.name.text = args.name;

And the .tss:
".container": {
backgroundColor: "#111",
height: "35dp",
left: 0,
top: 0
},
"#pic":{
height: "100dp",
width: "100dp",
left: "5dp"
},
"#name":{
color: "white",
left: "110dp",
font : {
fontSize: "30dp"
}
}

And the creation of the window:
myButton.on("click", function(){
var userDataArgs = {};
userDataArgs.pic = someUrl;
userDataArgs.name = "test";

// create controller from above
Alloy.createController("myController", userDataArgs).getView();

$.myTab.open(artistProfile)

})

  • Found the problem. I had the .container Window at a height of 35dp…. when it loaded the page, everything was scrunched into 35dp, then upon loading, it would re-layout to what i expected. Kind of weird behavior, but I'm glad I found it!
