  • Accepted Answer

    1. The SDK contains modules that let you write a program that interfaces with different components / capabilities on the device (you have stuff like db modules to handle database connection, ui components…).

    2. You write your code in Javascript. you can write and compile it from command line (if you like working hard) - but basically you would write it in Titanium Studio. Titanium Studio is an eclipse based development environment specialized for writing, running and distributing Titanium based applications.

    3. Write iPhone and Android apps (later on Blackberry and mobile web apps), and writing cross platform desktop applications.

    4. Depends on what you need to accomplish and what you need. And mostly - depends on you! like any other platform - it has it's advantages and disadvantages.

    5. The JS code you write translates to a native app with SDK libraries that knows how to run your JS program and generate native UI components.

    6. sqlLite databases are supported in Titanium.

    7. See answer # 4.

    8. See answer # 7.

    9. Using Titanium Studio is the easiest way to do that.

    10. You can write native modules in objective-c and java for Titanium.

    11. http://developer.appcelerator.com/apidoc/mobile/latest/Titanium.Map-module

    12. Yes.
