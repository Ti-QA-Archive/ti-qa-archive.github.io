  • Ti.Geolocation.getCurrentPosition is asychronous, which is why you pass a callback function to it in order to get the lat/long.

    Even with using it in another function here, you'll still need to account for the delay. You can go with passing your own callback function to App.func.locate(), or you can also use a custom event if you prefer to go that route.
