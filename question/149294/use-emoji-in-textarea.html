  • I solved this problem using a simple trick using plain old javascript (I didn't have permissions to change the mySQL server's encoding to "utf8mb4". As Nick rightly mentioned, utf8 is needed irrespectively.

    In your app, do something like this:

    textarea.addEventListener('return', function() {
    var str = encode_utf8(textarea.getValue());

    //Send the above str to your API/Server for storing in your mySQL db
});

function encode_utf8(string) {
  return unescape( encodeURIComponent(string));
}


//Use the following function to show the same string elsewhere, eg. Label

var label = Ti.UI.createLabel({
    text: decode_utf8(stringFromDB)
});

function decode_utf8(string) {
  return decodeURIComponent(escape(string));
}

    Neither of those two functions are my own creation, so I can't take any credits for it. This works seamlessly, and I'm currently using it in one of my apps (in development).
