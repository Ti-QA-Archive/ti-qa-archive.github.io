Label height/new line restriction

Im getting very weird behaviour with labels added to views. Just wanted to know if it is a normal issue, a bug, or if I'm the only person to experience it.

I have a large label with a few paragraphs of text. (this is the only way to add large blocks of text isnt it?). Depending on the amount of text in the label and how it is displayed, sometimes certain labels (more often with longer amounts of text, or where I have used the return key to enter on a new line) appear to overlap both above and below it. The 'top' and 'bottom' values seem to be completely ignored.

Has anyone had problems with labels overlapping?

This is always on labels that have a height:&#39;auto&#39; property. For this there is no option to have a fixed height.

I'm wondering if it is because I use the enter key to go onto a new line, as all this text is being taken from an sqlite database. I have tried using \n for new lines but this doesnt work when text is pulled from a database.

Thanks!

  • I think the problem lies in how you defined the view_content
    setting top and bottom in the same time you force it's height to be the height of the parent view minus the top minus the bottom.
    Try to remove the bottom setting of the view_content.

    • You set the view with layout:vertical. I'll try to make a simple scheme to show u how to achieve this:

      view_content ( height:auto,top:0, layout:vertical) 
    lbl1 ( top:10, height:auto)
    lbl2 ( top:10, height:auto)
    lbl3 ( top:10, height:auto)
    lbl_last ( top:10,bottom:10, height:auto)

      So you will get 4 labels even spaced between them and the margins of the parent view. Makes sense?
