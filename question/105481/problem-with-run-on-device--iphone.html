    • It's the AppID, not the Apple Account. If the AppID is a wild-card app id, it will contain a '*' character. This causes the problem. Use an explicit AppID instead of a wild-card AppID, and things work.

      — commented February 17th 2011 by Michael Cohen
    0 Votes

  • We ran into the same problem and found a solution. The issue was that our AppID (as specified in Apple's iOS Provisioning Portal) was a wildcard app-id of the form ABCDE12345.*, instead of an explicit app-id of the form ABCDE12345.com.domain.appname.

    Once we created a new AppID and generated a new provisioning profile for that AppID, things worked fine.
