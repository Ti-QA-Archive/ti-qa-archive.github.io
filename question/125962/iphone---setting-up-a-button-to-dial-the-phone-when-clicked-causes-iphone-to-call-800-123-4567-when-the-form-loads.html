iPhone - Setting up a button to dial the phone when clicked causes iPhone to call (800) 123-4567 when the form loads

I've searched the Q&A for an answer, but haven't found it, so here goes.

I create a TI.UI.window, then place a view on the window and add labels and buttons to the view, one of which is a simple image button to place a call to the phone number in the database for that record; however, when the page actually loads (and before the page appears), Titanium suspends the app and tries to dial +1 (800) 123-4567.

If I exit he call and return to the app, the call button works as expected, placing a call to the number from the database. Here's the offending code:

inline code

if (winery.phone) {
    var btnCall = Ti.UI.createButton({
        image:'../images/call_icon.png',
        top:curTop,
        left:curLeft,
        height:32,
        width:32
    });
    sView.add(btnCall);
    btnCall.addEventListener('click', function(){
        var phoneNum = "tel:" + winery.phone;
        Ti.Platform.openURL(phoneNum);
    });
    curLeft -= btnCall.width + borderSize;
}

The variable "winery.phone" is just a string of numbers like "8005551212", so that shouldn't be a problem.

If I comment the code out, no call is made, and the button does not appear (obviously), and the screen works as expected, so this is definitely the offending code.

Any thoughts?
