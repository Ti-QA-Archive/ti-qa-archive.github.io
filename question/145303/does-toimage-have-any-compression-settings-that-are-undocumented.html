    Hi

    toImage() creates a blob which is in the PNG format, which by design is loseless data format - which means it save every single pixel of data.

    So I am surprised you are seeing a poor quality image when using this routine. Are you stretching the image prior to using toImage() or when you use it elsewhere?

    Can you provide a screen shot of the section you are using toImage() on and the save portion used by toImage(). Also provide the code where you make the image and how you re-use in the app or save it to the file system.
