Vertical position 'top' property on view ignored when height not explicitly set

I'm new to Titanium and noticed that adding an image view to the window results in a vertical center position even if the 'top' property is set to a value not being the center. So for example top: 10, results in the view being positioned at the vertical center of the window. Finally I found the reason to be the absence of a value for the 'height' property of the image view. Only setting the height property to an explicit value resulted in a correct vertical position. Why is the height property mandatory on an image view in order to have the top (or bottom for that matter) property to work as expected. Can't the image view determine its own dimensions based on the actual image file which it represents.
Just curious why.

  • The image view can determine its own dimensions. You don't have to set the height property to a point value, but if you want it to determine its own height, you do have to set the height value to 'auto'. Doing this will ensure that no matter what layout system you are using in your parent view, layout properties like top and left will calculate properly.
