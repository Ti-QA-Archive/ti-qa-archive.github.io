BUILD FAILED CodeSign when building for iOS

Titanium Studio, build: 3.4.1.201410281727 Build: jenkins-titanium-rcp-master-202 (origin/master)
OSX Yosemite

I am trying to package a distribution for the app store.
First I had some cert and provision problems but I got that all cleared up.
But now I get this :

[INFO] :   Invoking xcodebuild
[INFO] :   Image optimization complete
[ERROR] :  ** BUILD FAILED **
[ERROR] :  The following build commands failed:
[ERROR] :      CodeSign build/Release-iphoneos/Project_Name_Here.app
[ERROR] :  (1 failure)

But if I go and do the CodeSign on the .app file manually from the terminal it is successful…

cbair:Release-iphoneos CB$ CodeSign -s "iPhone Distribution: Identifier here" Project_Name_Here.app

It does prompt me to for keychain access, but I have done all I can to make the keychain accessible to other applications. + Added studio to Sec & Priv Accessibility and all that

I am at a loss and I have spend all day on this one thing, I just need to get this packaged and in to iTunes Connect.

Is there some other way to do this? Or can I find more info on the error somehow?

If any of you guys have a rabbit loaded hat, now would be a very good time to pull one out for me :)

  • OK, I got it working, I had created a new keychain with all the right stuff and pointed titanium studio towards it.
    But in my default keychain I had an old version of the profile that was invalid.

    So most of the build process ran smooth, but it seems like the CodeSign command was trying to use the default keychain.

    I am not sure that my assessment is correct, but putting all the right stuff in the default keychain did the trick this time, what a day!
