Menu performance in Android

Hi all,
is there any material specifically about menus/actionbar and menu updating for Android in Titanium?

I see very little material about the issue and nothing about how to update a menu.

My personal case, I have a menu with 4 different actions that will become visible in different lightweight windows of the app. Also, the title in the action bar will change according to the lightweight window we are in.

I manage to do it after a lot of trial and error since I didn't find any material about the subject (only the documentation pages of Titanium, but the examples there don't cover anything advanced).

Still, although the performance of the rest of the app is quite good, the performance of the menu updates is really bad (more than 1 second in a Nexus 7 to change from one menu icon to another).

I know I must be doing something wrong, but some tips would be welcome.

I define all the menu buttons in the masterView window:

var addBtnId = 1, editBtnId = 2, saveBtnId = 3, infoBtnId = 4;
winMasterView.activity.onCreateOptionsMenu = function(e) {
var menu = e.menu;
        // ********** ADD Button **********
        var addBtn = menu.add({...});
        // ********** EDDIT Button **********
        var editBtn = menu.add({...});
...

Then, I have an onPrepareOptionsMenu in each window that sets the menu icons visibility and the actionbar title string:

// In every window something like:
var activity = winMasterView.activity
var actionBar = winMasterView.activity.actionBar;
actionBar.setTitle(L('app_name'));
activity.onPrepareOptionsMenu = function(e) {
    menu.findItem(editBtnId).setVisible(false);
    menu.findItem(addBtnId).setVisible(true);
    ...
}    
winMasterView.activity.invalidateOptionsMenu();
