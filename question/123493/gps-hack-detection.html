GPS Hack Detection

Is there a way to detect if a user is running a 3rd party app that changes the GPS values?

    • A 3rd party app on a jailbroken iPhone can hardcode the GPS values "pretending" that a user is at a location that he really isn't. If you have an app that relies on a users location, this can be exploit I would like to protect against.

  • You can't see what other apps are installed.

    All you can work with are the GPS coordinates. You could monitor them (poll regularly) and see if they jump unreasonably, e.g. 10 km in one minute (which is 600 km/h).

    — answered July 31st 2011 by Christian Siegert
    permalink

    • The problem would be someone would close my app, fire up their 3rd party app to change their location and then open my app which wouldn't know the difference.

      — commented August 2nd 2011 by Aaron Benzick

    • The problem here is that jailbroken phones can have programs perform tasks that a properly sandboxed application cannot do. I am almost positive you would never truly detect such an exploit from a stock Titanium application. And I question if you could even pull it off with a custom module using objC code.

      That said, Christian may be onto as good as you may be able to reasonably achieve. However, that will never be 100% foolproof either, as there will be times when the GPS is off for extended periods of time. If it was off because they were in airplane mode and really traveling, you could have a significant shift in GPS coordinates that is legitimate.

      — commented August 2nd 2011 by Doug Handy

    • Android has a built in feature called "ALLOW_MOCK_LOCATION" and you can use this to detect if another app is using a mock location feature. I'm at a loss for something clever I could check on the iPhone.
