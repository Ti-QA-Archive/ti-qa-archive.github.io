Is it possible to scroll two tableview at the same time ? (Grid TableView Layout)

Good day everyone. I am trying to create a brick grid style Tableview (like Pinterest) with appcelerator.

At the early stage, I was tried to create a Grid-style tableview with 2 data container contained within 1 tableViewRow. But It is not able to create the brick-style layout even both container are with different height. Like This

And Then I tried to implement this layout with two ways:
i. Create Two TableView containing in one view and then draw it into a ScrollView. Like This. This case seems worked with iOS platform (seems buggy at android), but since this case seems not able to make use of tableview functionality, so it would cause lag and memory issue.

ii. This time I would like to just create Two TableView within a window Like This. But these two tableview would scroll and works individually. So my question is: Is it possible to handle two tableview scroll event so i could make them able to scroll together at the same time? or is there any other ways to create this grid style tableview ?

    Hi

    They use a ScrollView with two/three columns side by side and manage the flow (to average heights) as they add the pictures. They also will be managing the memory actively, especially for Android as it is not as capable as iOS automatically.

    Your TableView solution is probably safest for now. You do NOT need two tables, simple create a single table and add custom rows. Each row had two images, that simple. No need to go side by side with two tables or try and match their positions which will not be ideal on iOS and level harder on Android.

    You will need to keep memory issues in mind as lots of images are always going to be an issues, especially on Android.

    BTW it may look like I am putting Android done, but the simple fact is when it comes to image memory management - iOS is significantly better (although more recent Android SDKs have improved), it just means you may have to do so e work yourself if you have many rows.

    A common memory management technique is to unload the imageView for any rows out of view (with a safety margin) and when they are getting close add them back in. Only do this if it becomes a problem though.
