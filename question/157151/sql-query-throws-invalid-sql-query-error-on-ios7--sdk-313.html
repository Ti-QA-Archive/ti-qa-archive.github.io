SQL query throws "invalid SQL query" error on iOS7 / SDK 3.1.3

I'm doing a simple SQL query as follows:

SELECT * FROM view__mobile_companies_ WHERE (customername like ? OR customercity like ?)

which works without any problem on with iOS SDK 6.1 and Titanium SDK 3.1.3v20130912132758.

On iOS SDK 7 (GM seed, downloaded 09-13) the same code throws an "invalid SQL query" error.
Is this a know issue?
Is it iOS 7's problem or Titanium's problem?

  • Answering myself: the iOS7 simulator uses an Applications folder of it's own. So the SQL database was not the same when it ran in the iOS7 simulator as it was in the iOS6.1 simulator.
    After copying the SQL file from the App's library/private documents folder to the iOS7 simulator the problem was solved.
