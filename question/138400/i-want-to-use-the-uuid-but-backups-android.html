I want to use the UUID, but backups.. (android)

I want to encrypt a database server-side, then send it to a device after an in-app payment and have it work only for that person. The part I am stuck on is how do I do this while still letting them restore backups and have multiple devices? My original idea was to just have the UUID as part of the encryption key. On the other hand, if I tie it to a generated key or a server-side account, how do I know they aren't installing it on friends devices? I guess I can make it so it needs to be re-paid for if you get a new device, but that seems cruel. I am in need of guidance.

-Kyle

  • I think what your app needs is an user account and associate purchases to users, not devices.

    You may want to use the UUID to limit the number of devices (i.e. the user can install the database in up to 5 devices), but not as a part of the encryption key, but as your own internal control

  • Also, Apple is phasing out developer access to the real device UDID so you really don't want this used as part of your encryption key. At some point you'd hit an iOS update or patch which would cause the device to not return the UDID, and you'd have existing installed applications that could no longer display their own data.

    The good news is Appcelerator is hiding this complication from us, see this blog and Ti.Platform.id
