Action bar back/home button action ignored on Nexus android 4.3

I have a weird behaviour on an nexus with android 4.3

if I click on the left upper corner on the action bar it doesn't call the onHomeIconItemSelected event

It works on phones but not on this tablet, don't know if it's the android sdk

I'm using titanium 3.1.1 

win.addEventListener("open", function (evt) { 
    var actionBar = win.activity.actionBar; 
        actionBar.displayHomeAsUp = true;

        actionBar.backgroundImage = 'images/bg_navbar_short.png';

        actionBar.onHomeIconItemSelected = function () {      
               //doesn't call this line 
               win.close();
            }; 
});

  • Hi

    I can confirm the same experience on a Nexus 4, but it appears to work correctly on other devices.

    I took your event code, add it to to a test Android app supporting the action bar and tabs, I opened a new window and attached this event code (and several variations) from a button event fro the main code.

    Whilst other Android devices correctly dismiss the window the Nexus 4 does not, nor does it actually respond to any code inside this event.

    My test Nexus 4 was running 4.3 and I also tried it after very recent update I had yet to perform and still the same experience.

    Sorry it is not better news, for example a work around.

    I would suggest you register this as a bug in the jira ticket system.
