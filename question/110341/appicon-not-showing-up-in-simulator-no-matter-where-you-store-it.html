Appicon not showing up in Simulator no matter where you store it

Hi guys! I created a nice appicon.png and put it in the Resources/iPhone location, but it's not showing up in the Simulator.

Similarly, I tried putting it in the build/iphone/Resources and that didn't work either.

I took another thread's advice and deleted the contents of the build/iphone directory and let it build from scratch. It just populates the folder with the default titanium icon, and then doesn't use them when running.

There might be a bug. If you clear out the build, and then go and change the info.plist, putting your appicon in there, it saves fine, but the next time Titanium Developer runs your app, it overwrites the info.plist with the template. So, it seems, in essence you can't put your icon in the emulator.

This might work fine on the iPhone though.

This is a terribly frustrating bug. I'm about ready to package and I can't get the simple icon to display!

    When you create your project in titanium developer there's a field for the app icon.
    By default it is set to: default_app_logo.png (which is a larger image)
    I updated that image with my icon and it worked.. but I think there might be a bug, because I tried changing the path to another image, and it still uses that one.

    Sometimes try emptying the build/iphone folder and try again
