setData doesn´t update TableView correctly

Hi Guys,

I wonder if someone is able to help me with this problem. I made a autocomplete/autosuggest search, utilizing createSearchBar and createTableView. This works like a charm but there is one problem: if I enter a number into the search field, the table view gets updated but the changes are not show, until I press the cancel button of the searchbar. If I enter a search term "Vienna", everything works like expected, if I enter "1100" as search term, in the background I can see via Ti.API.info() that the whole functions get executed, but the table view stays empty, displaying "no results", until I hit the cancel button in the search bar, after that the list magically displays correctly. It seems a lot like for some reason the table view does get updated but changes are just not displayed until the cancel button is pressed in the search bar. Maybe this is a bug within Appcelerator?

I am using Titanium Mobile SDK 1.7.2 by the way, have also tried 1.6.1 and a few others, every time the same problem.

I wonder how this happens and maybe one of you guys had this or maybe some one is able to help me track down the error here?

Thanks!

Kind regards,
Daniel
