problem with ios simulator after installing xcode 4.5.2

I installed the latest version of Titanium Studio - 3.0.1, Xcode - 4.5.2 and ios sdk 6.0. I am running this on mac osx 10.7.5.

But when I launch my app on the ios simulator, the simulator crashes with the following message. 

dyld: Library not loaded: @rpath/DevToolsFoundation.framework/Versions/A/DevToolsFoundation
  Referenced from: /Applications/Xcode.app/Contents/Developer/Platforms/iPhoneSimulator.platform/Developer/Library/PrivateFrameworks/iPhoneSimulatorRemoteClient.framework/Versions/A/iPhoneSimulatorRemoteClient
  Reason: image not found

I tried all the solutions listed in the forum. When I tried to execute the

sudo ln -s /Applications/Xcode.app/Contents/OtherFrameworks/DevToolsFoundation.framework /Developer/Library/PrivateFrameworks/

I get an error saying 

/Developer/Library/PrivateFrameworks/: No such file or directory

  • I finally fixed this problem! The problem was that the Tiapp.xml inside my project was still pointing to the old version of Titanium sdk (1.7.5), I changed this to 3.0.0GA and it worked!
