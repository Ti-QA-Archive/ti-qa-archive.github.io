android UI Controls style issue

Hello,

  1. Application Type: mobile
  2. Titanium SDK : 3.2.3.GA
  3. Platform: Android 4.0 & +
  4. Device: Samsung Galaxy S3 / Android 4.3
  5. Operating System: Mac OS X 10.8.5
  6. Titanium Studio: 3.2.3.201404181442

I am a bit confused because I didn't find out how to get the Android UI Controls style as we can see in the android developer doc.

I am looking for the switch button to take the toggleButton style like this : ToggleButton and for the Spinner control to be like this : Spinner

Here is the code I have for the toggleButton :

<Switch id="mySwitch" value="false" />

Here is the code I have for the spinner control : 

<Picker id="myPicker" selectionIndicator="true" useSpinner="true">
    <PickerColumn>
        <PickerRow title="Home"/>
        <PickerRow title="Work"/>
    </PickerColumn>
</Picker>

In order to fix this issue, I tried to set up a theme : AndroidThemes

Here is my tiapp.xml : 

<android xmlns:android="http://schemas.android.com/apk/res/android">
    <manifest>
        <application android:theme="@style/LightDarkBar"/>
        <uses-sdk android:minSdkVersion="14" android:targetSdkVersion="14" android:maxSdkVersion="19"/>
    </manifest>
</android>

./platform/android/res/values/builtin_themes.xml : 

<?xml version="1.0" encoding="utf-8"?>
<resources>
    <!-- Available for Android 4.0.x/API Level 14 and later -->
    <style name="LightDarkBar" parent="@android:style/Theme.Holo.Light.DarkActionBar"/>    
    <!-- Available for Android 3.0.x/API Level 11 and later -->
    <style name="Light" parent="@android:style/Theme.Holo.Light"/>
    <style name="Dark" parent="@android:style/Theme.Holo"/>
    <!-- Available for all Android versions -->
    <style name="OldLight" parent="@android:style/Theme.Light"/>
    <style name="OldDark" parent="@android:style/Theme.Black"/>
</resources>

Does anybody have a solution ?

Thanks for helping !

  • General

    At first, you don't need to provide an own theme to do this. As you are on SDK 3.2.X, you are fine (you'll get the latest Android SDK selected by default). So, the first thing you do is, remove all the theming stuff from your project and also remove the manifest part from your tiapp.xml.

    Especially this line should be removed, because it blocks devices with API Levels <14 and >19 and this is not what you want! Also, if you specify a targetSdkVersion, then apply the latest sdkVersion that you have tested your app against (should be 19 atm)

    <!-- DON`T DO THIS -->
<uses-sdk android:minSdkVersion="14" android:targetSdkVersion="14" android:maxSdkVersion="19"/>

    Spinner

    Don't get confused by Titaniums infelicitous naming of this component. The (android) Spinner is represented as the (titanium) Picker whereas useSpinner should not be set to true (because it will replace the native picker [/Spinner] with "non-native Android control, with a spinning wheel that looks and behaves like the iOS picker" as described in the docs). Instead you should use

    type: Ti.UI.PICKER_TYPE_PLAIN

    Switch

    The Switch is currently not supported by Titanium as explained in this questions answers but fortunately the great David Bankier has just released a module for this: RealSwitch
