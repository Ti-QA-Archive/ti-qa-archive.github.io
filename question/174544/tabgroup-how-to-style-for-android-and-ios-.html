    Hi

    Short answer - No (not tab group anyway)

    Longer answer;

    You do not have to leave the platform, just choose more appropriate objects.

    • Create a view, set the bottom value to something you like
    • Add four labels (or as many as you want) to the view, give them a blue background and white text colour
    • Add a scrollableView to the remaining part of the screen and add a view for each of the tabs

    Then on the label clicks, change the current page of the scrollable view.

    Sorted.

    The tabGroups have built in UI conversions that are not always easy to change to exactly what you want but also they will not work identically as the two operating systems handles tabs differently. Android in particular as it has two main ways to display them i 2.3 and 4.x. Also the 4.x Android versions are always being tweaked for each version bump.

    Think of things as more like building blocks. If the thing you want does not quite exist - make your own.

    • You can get close to your wishes with iOS, but nearly close enough. Android certainly nowhere near close enough. It is not a limitation of Alloy or even Titanimum - these are due to the specific UI goals of the things we know as tab groups.

      But as I mentioned, those are simply one take on changing from one app section to another. If we take the Facebook style menu as an example, it also navigates the user from one app section to another. It does not visually look like a tab group, but it does perform the exact same purpose as one. It has no internal relationship to a tab group either.

      So if you like your visual example and I can see nothing wrong with the principle (I can imagine what you are going to do with it beyond the mock up) then that is what you should build.

    • This will get you moving in the right direction;

      index.xml

      <Alloy>
  <Window>
    <View id="tabgroup">
      <View class="tab">
        <Label class='tabtitle'>One</Label>
      </View>
      <View class="tab">
        <Label class='tabtitle'>Two</Label>
      </View>
      <View class="tab">
        <Label class='tabtitle'>Three</Label>
      </View>
      <View class="tab">
        <Label class='tabtitle'>Four</Label>
      </View>
    </View>
  </Window>
</Alloy>

      index.tss

      "#tabgroup": {
    backgroundColor: 'blue',
    bottom: 20,
    height: 50
    left: 20,
    right: 20,
    width: Ti.UI.FILL
},
".tab": {
    height: 50,
    width: '25%'
},
".tabtitle": {
    color: '#fff',
    height: 50,
    textAlign: 'center',
    width: Ti.UI.FILL
}

      The id of tabgroup and the class of tab are representative of what you are using them for - not related to the actual tab group or tabs.

    • Something like this will do it;

      index.js

      // additions to the file
var tabsHandler = function (e) {
    $.pages.setCurrentPage(e.source.index);
};
$.tabgroup.addEventListener('click', tabsHandler);

      index.tss

      "#pages": {
    bottom: 70,
    height: Ti.UI.FILL
    top: 0,
    width: Ti.UI.FILL
}
"#tabgroup": {
    backgroundColor: 'blue',
    bottom: 20,
    height: 50,
    layout: 'horizontal',
    left: 20,
    right: 20,
    width: Ti.UI.FILL
},
".tab": {
    height: 50,
    width: '25%'
},
".tabtitle": {
    color: '#fff',
    height: 50,
    textAlign: 'center',
    width: Ti.UI.FILL
}

      index.xml

      <Alloy>
  <Window>
    <ScrollableView id="pages" showPagingControl="false">
        <View id="view1" index="0" backgroundColor="#123">
            <Label>Hello World</Label>
        </View>
        <View id="view2" index="1" backgroundColor="#246">
            <Button>Does nothing</Button>
        </View>
        <View id="view3" index="2" backgroundColor="#48b" />
        <View id="view4" index="3" backgroundColor="#6ad" />
    </ScrollableView>
    <View id="tabgroup">
      <View class="tab">
        <Label class='tabtitle'>One</Label>
      </View>
      <View class="tab">
        <Label class='tabtitle'>Two</Label>
      </View>
      <View class="tab">
        <Label class='tabtitle'>Three</Label>
      </View>
      <View class="tab">
        <Label class='tabtitle'>Four</Label>
      </View>
    </View>
  </Window>
</Alloy>

      This is a very quick example based on my previous code. Enough to get you started though.

  • Hi khodour fouani,

    For android and iOS tabgroup show different behavior. By default it goes to bottom for iOS and top for Android. But if you want to bottom for android you have to use some tricks like module. So you have to learn about this. I am posting some link and example code you should follow this carefully for getting your expected behavior.

    1.TabGroup Link

    2.Tab link

    If you want to change the style you have to use Theme for android

    3.Theme Link

    Simply paste this code in view and controller folder.

    ###Index.xml

    <Alloy>
    <TabGroup tabsBackgroundColor="blue" navBarHidden="true">
        <Tab id="tab1" title="Tab 1">
            <Window id="win1" backgroundColor="gray">
                <Label id="label1" color="#000">I am Window 1</Label>
            </Window>
        </Tab>
        <Tab id="tab2" title="Tab 2">
            <Window id="win2" backgroundColor="gray">
                <Label id="label2" color="#000">I am Window 2</Label>
            </Window>
        </Tab>
        <Tab id="tab2" title="Tab 2">
            <Window id="win3" backgroundColor="gray">
                <Label id="label2" color="#000">I am Window 3</Label>
            </Window>
        </Tab>
        <!-- Use the Require tag to include external Ti.UI.Tab views -->
    </TabGroup>
</Alloy>

    ###index.js

    $.index.open();
