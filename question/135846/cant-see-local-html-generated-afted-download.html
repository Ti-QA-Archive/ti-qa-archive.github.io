Can't see local HTML generated afted download

Dears,

I'm developing an app that download HTML code (with responsetext) and images (with response data). It create a folder in ApplicationDataDirectory and store several html files with related images.

The structure is:

DEVICE\ApplicationDataDirectory()\test:

|_ 1.html

|_ 2.html

|_ …html

|_ myimage.jpg

|_ myimage2.jpg

|_ …jpg

I verified on both IOS and Android devices, in their app folders i find all my html and image files (prevously downloaded).

Problem is, when i try read it by webView on IOS i reach my goal, on Android i can read HTML correctly, images are with question mark (seem like the was not founded).

Inside HTML I tried with:

<IMG SRC="/myimage.jpg">

<IMG SRC="myimage.jpg"> (without slash)

in both cases i've the correct result on IOS and question mark on image layout on Android.

This the simple code to see what i downloaded in first HTML

parent = Titanium.Filesystem.getApplicationDataDirectory()
parent = parent + 'test/'

mywebview = Ti.UI.createWebView({ backgroundColor:'white',url:parent "1.html" });


win.add(mywebview)

Any experience on topic??

Many Thanks
Steve

    • According API docs applicationDataDirectory: On Android, files in this directory cannot be accessed by other applications, so it should not be used for files that must be used by another application (for example, a video file that will be passed to the system's video player application). These files should be stored in the externalStorageDirectory.

      May I suggest you to store the downloaded files to externalStorageDirectory?
