Rebuild and Test Approach

Hi, I'm pretty new to appcelerator, I'm currently building an app for Android on Windows. Each time I make a change and want to retest what is the most efficient way to do that? It seems like clicking launch takes 3 to 4 minutes each time, makes frequent testing difficult.

— asked December 15th 2010 by Frank A

  • Make sure you're not restarting the emulator with each run. You'll also get better performance if you make sure to have 1.5 to 2GB memory before starting up the emulator and TiDev. The emulator works best w/ about 1GB available to it. At least that's what we've found in our daily use.
