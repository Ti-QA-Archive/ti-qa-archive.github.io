Multiple Criteria on a Alloy dataFilter

Hi there-

This should be an easy question…is it possible to place multiple criteria on a dataFilter to filter a collection? All examples I have seen filter collections by only 1 criteria of data.

If I have a collection that is bound to a table:

<Alloy>
    <Collection src="book" />
    <Window class="container">
        <TableView dataCollection="book" dataTransform="transformFunction" dataFilter="filterFunction">
            <!-- Also can use Require -->
            <TableViewRow title="{title}" />
        </TableView>
    </Window>
</Alloy>

and within my filterFunction I would like to return/show all results where the books have an author of Mark Twain or Samuel Clemens, I know I could do:

function filterFunction(collection) {
    return collection.where(
    {
        author:  'Mark Twain'
    });
}

..but is there a way to put an or, or/and, or any other compound argument into the author fields? I've even seen in the Fugitive/Captured examples, it is only one criteria ever returned. I am allowing a user for my app to display multiple types of orders within a table based on a filter, thereby requiring a compound filter if possible.

Thanks in advance for any help.
Chris

    You can do whatever you want in the filter function as long as you return an array of models. You don't even need to use the where() function. You can iterate over the models in the Backbone.Collection.models array and return a subset of that array in any way you choose.

    • For others out there, just to tie up how I ended up doing the implementation, I did the following. I have several settings which are controlled by Switch controls. Based on the switches, I wanted to hide or show returned table rows that are bound within my TableView. So, within my dataFilter, to do my multiple condition, I simply used the filter function as follows:

      function filterFunction(collection) {
  var collection = collection.filter(function(order) {
    if (Alloy.Globals.Configuration.myFilters.indexOf(order.get('Status')) != -1)
      return order;
  });
  return collection;
}
