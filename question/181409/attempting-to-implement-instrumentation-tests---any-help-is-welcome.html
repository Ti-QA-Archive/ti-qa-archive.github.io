Attempting to implement instrumentation tests - any help is welcome!

Type: Mobile
Titanium SDK: Titanium Command-Line Interface, CLI version 3.3.0, Titanium SDK version 3.2.3.GA
Platform: Android

I'm trying to find any kind of ability to do instrumentation testing for android through Titanium/Appcelerator. I've tried everything I could think of; building a native instrumentation test project in appcelerator, importing the appcelerator app into eclipse, changing the debug keystore, etc. The target package in the manifest is the correct package for the app, but at this point the furthest I can get is Appcelerator failing with either a "Package signatures don't match" error or a "No instrumentation class for test package" error.

At this point, I'm willing to use non-native instrumentation, but after looking around I can't find anyone who is actually doing instrumentation testing, at least no one in the past 2 years. I know about SOASTA, but that's not instrumentation; that's record/playback, which is not what I want.

So is there a way to get up to your elbows in the app with testing? Or am I limited to SOASTA and whatever I can do to dependency modules in Eclipse?

  • This may or may not be what you're looking for, but hopefully it helps somebody.

    My team is doing a deep dive into appium with some success. We've been able to build a set of tests for our app that run on android and ios. The appium server is compatible with Selenium WebDriver, so you can use any language that has webdriver bindings. We're using Javascript and the wd library.

    Authoring the tests is a bit tricky. We have not found a way to put meaningful IDs or labels on our views, so we have to find all the views via an XPath-like mechanism. This can be a bit fragile if you don't construct those xpath queries very carefully. And since the xpaths use the native UI views (e.g. "UIAWebView" vs. "android.webkit.WebView"), finding the views is different on iOS and android.

    Our plan is to take the tests we're developing and run them on a cloud-based testing platform, like AppThwack.
