Setting tableview data in 1.8

Hey, all –

I'm doing some refactoring and I have a situation where I need create the shell of a table and add data later (after an async process). Essentially, each row represents a location. I have to geocode each location so that I can apply a distance to each location from the user's current location.

What I do is generate the list of locations and run each through a method that returns a distance and create a tableViewRow that is composed of labels. Each row is added to a rows[] collection.

When I finish processing the last location, I sort the rows[] collection by distance and then want to set the table's data to the value of the sorted rows:

  if( last ) {
    alert( rows );
    var sortedRows = _( rows ).sortBy( function( row ) {
      return row.distance * 1; 
    });

    alert( sortedRows );
    table.setData( rows );
    alert( table.data[0].rows );
  }

The first 2 alerts indicate an array of TiUITableViewRow objects like I would expect (perhaps an incorrect expectation?). I get an error, though, when I try to alert table.data[0].rows just to be sure it "took":

}
Terminating in response to SpringBoard's termination.

Any idea what I'm doing wrong here? I'm using the 1.8 SDK.

Any insight would be much appreciated. Thanks.

  • Tim,

    Once you set a table's data, the structure it creates a table structure is something like this:

    Table

    • Section
      • Rows
    • Section
      • Rows

    If you want to examine the structure, use the debugger and take a look at it. You can verify that your rows are 'SET' and you can use the expression builder to test getting data from the structure to make sure you have the right syntax. I just tested this and your syntax seems correct (ie. tableview.data[0].rows) returns an array of rows for me. So maybe debugging will give you a hint as to what is going on because it looks like it should work.

    BTW, I'm using 1.8.0.1
