1D Barcode Support

Hello,

Does the Titanium+ Barcode module support 1D barcodes? The module documentation states "We use Google's ZXing (Zebra Crossing) library, an open-source multi-format 1D/2D barcode image processing library.", yet I'm unable to get 1D scanning working.

Here's the code which I'm using to successfully scan QR codes, but unsuccessfuly recognize 1D codes.

To create a sample UPC code, you can use this URL: http://www.barcoding.com/upc/

Here's sample code which I'm using to implement a simple barcode scanning button:

var win = Ti.UI.createWindow({
    backgroundColor: 'white'
});

var barcode = require("ti.barcode");
var scan = Ti.UI.createButton({
    width: 200,
    height: 44,
    title: 'Scan!'
});

scan.addEventListener('click', function() {
    barcode.capture({
        success: function(data) {
            Ti.API.error(JSON.stringify(data));
            alert('Scanned!');
        },
        cancel: function(event) {
            alert("Cancel!");
        },
        error: function(event) {
            alert("Error: " + event.message);
        }
    });
});

win.add(scan);
win.open();

Please clarify what the state of 1D support is – is this a bug, or is this a known limitation of the ZXing implementation?

Thanks!
Adam Meghji

  • > "Why not use TiBar module from appcellerator? It's availble on sourceforge. loog for TiBar in this forum."

    I've integrated TiBar and it works for 1D, but unfortunately it doesn't support auto-detection of QR codes, so the user experience just isn't as nice as ti.barcode. Also, integrating it into a project is quite intrusive given the 5 additional frameworks you need to add into the project, which makes builds a manual and error-prone process. Moreover, on their website, the first thing stated is that it's experimental and not suitable for a production environment, so given their disclaimer, I'm hesitant to put that into use with real users.

    ti.barcode looks awesome so far, we just really need 1D / UPC support.
