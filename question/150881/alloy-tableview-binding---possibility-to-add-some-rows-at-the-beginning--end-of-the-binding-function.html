Alloy Tableview Binding - Possibility to add some rows at the beginning / end of the binding function

Hi,

I really like Alloy's data binding feature but I can use it only for very few screens. Because I often need some additional views or rows to be rendered before the other views/rows are rendered by the binding feature.

Example

  • Row "Previous"
  • Data binding row
  • Data binding row
  • Data binding row
  • Row "Next"

I cannot use the data binding feature because I need to render one row "Previous" before all rows are created by the binding feature and one row "Next" afterwards.

Does somebody know how to work around this without using something like "appendRow"? Because it's pain to use and pretty slow. Maybe there is already a practical solution for this?

Maybe a solution could be to define a function in XML view markup like dataPreBinding=&#39;myfunction1&#39; and dataPostBinding=&#39;myfunction2&#39;. Those functions would return some additional rows or views which have to be added before or after all other generated views.

Example

function myfunction1 () {
  return [Ti.UI.createTableViewRow({title: 'Previous'})];
}
function myfunction2 () {
  return [Ti.UI.createTableViewRow({title: 'Next'})];
}
