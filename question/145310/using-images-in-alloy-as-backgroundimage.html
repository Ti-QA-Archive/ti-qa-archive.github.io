Using images in alloy as backgroundImage

Hi eveyone!

i have started using alloy with the tss properties, controllers and everything else. I use some images as background images and are placed as follows:

iphone\
   images \
    global\
         background.png
         background@2x.png
android\
   large\
    global\
         background.png
   medium\
    global\
         background.png
   low\
    global\
         background.png

the .tss file has the properties of the object which is :

".window": {
    top:"0dp",
    left:"0dp",
    backgroundImage:"images/global/background.png",
    layout:"vertical",
    width:"80%",
    zIndex:0
},

I have tried modifing the backgroundImage property with

backgroundImage:&#39;&#x2F;images&#x2F;global&#x2F;background.png&#39;

backgroundImage:&#39;&#x2F;global&#x2F;background.png&#39;

backgroundImage:&#39;images&#x2F;global&#x2F;background.png&#39;

backgroundImage:&#39;global&#x2F;background.png&#39;

and even moving the files to other folders, but nothing of what i did work…

Does anyone knows what i'm doing wrong to load images on android as backgroundImages?

Thank you for your help.

  • FWIW, this appears to be an issue with fastdev. When I added this to my tiapp.xml file, images worked as expected (specifying '/images/test.png' for backgroundImage properly loaded the right image out of the right density directory).

    <property name="ti.android.fastdev" type="bool">false</property>

    Thanks to this post.
