Let's put an end to the "abnormal process termination" error!

It appears that I'm not the only one, who experienced the abnormal process termination error, when trying to run an app in an Android emulator. The exact error is:

[ERROR] Application Installer abnormal process termination. Process exit value was 1

I saw one solution available on this forum here. However, being a new user, I am not able to apply it. I have just downloaded the Titanium Studio, did the vanilla install, tried the "Hello" example and got this error.

Can someone, who resolved it before, please guide me and tell me what version of SDK I need to install to make it go away.

So far, I installed Android 4.4.2 and Android 2.3.3 (just in case). I have Studio 3.2.1 running on Windows 7. I figured how to turn the trace on, but was unable to pick up any clues, like "KeyError: 15", mentioned in the previous solution. My trace log doesn't have that.

I really appreciate your help!

    "Let's put an end to the "abnormal process termination" error!"

    I agree! The sooner folks stop pretending that Windows is a viable platform upon which mobile development takes place, the faster this will occur!

    Btw, this isn't a problem with Titanium, its a problem with the crappy android emulator enhanced by a crappy host os. Many of us use genymotion instead of the android emulator.

    You also failed to show your android tooling, if its 22.x, well, thats part of the problem as well.

    The results of ti info from a command prompt would be helpful.

    • Solved: I am able to run an app in the Android emulator now! The trick was to start the Android emulator ahead of time and wait for about 5 minutes for it to fully initialize. Only then, hit Run in Titanium. Also prior to that, I had to create an Android device definition in Run Configurations of the project. And before that, I had to reinstall the right version of the Android tooling and reinstall Android SDK. All in all, this was probably one of the most involved installation processes I ever had to do - it took about 4 days (for 2 platforms). I'm surprised there's not more documentation on these issues. Thank you Stephen for helping!
