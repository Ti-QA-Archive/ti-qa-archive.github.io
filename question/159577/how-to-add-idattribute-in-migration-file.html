  • SQLite tables can have only one autoincrementing primary key. You're attempting to add another with this sort of change. You won't be able to switch primary keys using the technique you're using. I guess in theory you could copy all of the data to a temp table, drop the old table, re-create it with your gID primary key, and copy the data back, then drop the temp table. I'm still not sure you could set the idAttribute to the new column in the migration file. I haven't tried that. Maybe it would work.

    Wouldn't it be easier to just update your code to use alloy_id as the field name of the primary key?
