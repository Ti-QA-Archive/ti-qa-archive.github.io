Retina Graphics for Repeated (Tiled) Backgrounds and Backgrounds with Caps (9 sliced)

I am working with the 2.0.1.GA2 SDK and have found that while most retina graphics (@2x) work fine, tiled backgrounds and backgrounds using caps do not.

For tiled or Repeated Backgrounds, I have found that the retina image (@2x) is not scaled down to 1x size, it is kept at 2x it's size. So the tile appears at twice the size it should on a retina device.

For backgrounds with caps, I have found that the 2x graphic is not scaled down to 1x dimensions, so if the cap amount is 10, it is not creating a cap that is 20 for the 2x graphic.

Has anyone else experienced this?

  • Hey Mike,

    I haven't seen (or tried) the repeated background image problem, but I have had problems with background caps on retina displays.

    Retina display support for background caps seems to be highly dependent on the type of object Ti Object. For example, it seems to always work on buttons even as far back as 1.7 SDK (my previous work around was to use a button with touchEnabled: false when needed).

    In the 2.0.1.GA2 SDK however, background caps seem to be supported on labels as well for retina devices. So, I would recommend using labels in place of standard views where background caps are needed. Background caps might work on other objets as well, but as most display objects are just views with added functionality, a label seems like a good base object to work with.

    tl;dr: background caps work on labels, use them in place of views.
