ListView with ItemTemplates how to set "searchableText" property

Hi,

I have SearchBar inside ListView. And I am using ItemTemplates to have various types of ListItem structure. So how to set "searchableText" property ?? Or is there any alternative to provide automatic search functionality in this case??

Following is my code snippet.

    <View id="view">
        <ListView id="widgetList" >
            <SearchBar barColor="#F8F8F8" borderColor="#D3D3D3" borderRadius="2dp" />
            <Templates defaultItemTemplate="template1_CA">
                <!-- template1 : LIST1 : L -->
                <ItemTemplate name="template1_CA" height="50dp" accessoryType="Ti.UI.LIST_ACCESSORY_TYPE_DISCLOSURE">
                    <Label bindId="lhs" left="5%" width="93%" height="18dp" minimumFontSize="20dp" textAlign="left"/>

                </ItemTemplate>
            </Templates>
            <ListSection  id="widgetListSection" ></ListSection>
        </ListView>
    </View>
</Alloy>

  • Would you please file a Jira ticket for this? Provide the smallest working app that demonstrates the issue. Please include a link back to this Q&A thread.

    A couple of things:

    1. You need to associate the search bar with the list by using the searchView attribute. 

      <ListView id="widgetList" searchView="search">
<SearchBar id="search" barColor="#F8F8F8" borderColor="#D3D3D3" borderRadius="2dp" height="Ti.UI.SIZE" />

    2. Here's some controller code that should set the searchableText property for the list items.
      ~~~
      var names = [

       "Gandalf",
 "Legolas",
 "Frodo",
 "Bilbo",
 "Samwise",
 "Elrond",
 "Galadriel"

      ],
      numitems = names.length;

    var data = [];
    for(var i=0; i<50; i++) {
    var thetext = names[Math.floor(Math.random()*numitems)];
    var item = {
    lhs: {text: thetext, searchableText: thetext.toLowerCase()},
    template: (i%2) ? 'template1' : 'template1_CA'
    };
    data.push(item);
    }
    $.widgetListSection.setItems(data);

    $.index.open();
    ~~~

    1. There is a ticket that's perhaps related TIMOB-16157 It's closed and will be included in the next release. However, in my tests, using the nightly CI builds of Titanium & Alloy, searching still doesn't work correctly. Searching never returns a match.

    I'm not sure if this is an issue with Alloy or the underlying platform.
