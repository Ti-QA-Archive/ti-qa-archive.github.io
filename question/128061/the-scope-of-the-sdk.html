Hi, I've never used this SDK before questions:

#1. How hard is it to develop with? Does it matter which language you use?

#2. Is it possible to have access to network function such as bluetooth wifi?

#3. How hard is it debug?

#4. Is is possible to get moodle type functionality?

#5. Can you access /integrate adobe flash / adobe impress?

#6. Anything else an expert can tell me ?

  • A majority of these questions can be answered by going to the home page of the wiki, where many of these topics are detailed. For this reason I suggest you go to the following link: http://wiki.appcelerator.org/display/guides/Home

    My answers will be brief as you will be better served going to the wiki

    1) Titanium for mobile use a pure Javascript API, desktop uses HTML/JS/CSS. Appcelerator prides itself on the speed with which you can develop an app using Titanium.

    2) Yes, you can access wifi, and extend Titanium apps to use bluetooth via native modules

    3) Titanium Studio, the sanctioned IDE for Titanium development, has an integrated step debugger. In addition, the Titanium.API module has functions for logging

    4) Moodle is a web based CMS if I'm not mistaken. You could replicate much of its functionality in titanium from a front end perspective.

    5) At this moment we don't have a means of integrating flash content

    6) Again, be sure to check the wiki and developer blog for info

  • Some more stuff:

    • While Ti uses JS your "maybe very good js knowledge" won't be enough if you worked with in an DOM only environment. You will have to forget about DOM and your coding experience will be closer to a server one.
      The speed you can reach to build apps is amazing. The Ti APIs + some smart autocompletion + using a snippets manager (and of course experience) will allow you to make ready to submit apps in matter of days.

    • You have access to many features natively and for modules there are few sources you can get them from, like github, appcelerator marketplace(free and $), codecanyon ($) or even create modules by yourself or pay somebody.

    • Flash, despite is popularity is sort of dying (imho). I wouldn't rely on it too much.

    • You will need patience, if you have issues come to QA and ask specific questions (not like "how to make a rss reader. provide me code. it's urgent.") :). the best way to find how to do stuff is KitchenSink where you have virtually all the APIs inside the examples.

    • The cross-platform ratio is not quite 100% :) but like 80% , there are differences between android and ios that cannot be normalized. Most of them are on the UI side.
