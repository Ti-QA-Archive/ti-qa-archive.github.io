Loading a model inside a service

I have this model that is working well on controllers:

exports.definition = {

    config : {
        "columns" : {
            "active" : "Bool",
            "name" : "String"
        },
        "adapter" : {
            "type" : "properties",
            "collection_name" : "stuff"
        }
    },
...

On myservice.js (called with Ti.Android.createServiceIntent):

var items = Alloy.createCollection('stuff');
Ti.API.info(items.length);

It shows "0".

But if I do that:

Ti.API.info(Ti.App.Properties.listProperties());

It shows a list that contains "stuff-nnnnn".

If I can access my "stuff" with Ti.App.Properties why can't I access it with Alloy.createCollection?
It's an Alloy bug inside services?

    You never populated the collection. Try fetching.

    var items = Alloy.createCollection('stuff');
items.fetch();
Ti.API.info(items.length);
