Titanium.Facebook.requestPermission is always returning True, even when the user declines the permission

Anyone else running into this?

  • I'm running into this problem also, targeting iPhone.

    I'm using the following (very straightforward) code to ask for birthday, email, and location data. I'm building a wine application, so it's really important that we ensure our users are 21+.

        Titanium.Facebook.requestPermission('email,user_location,user_birthday', function(evt) {
            if(evt.success){
                Ti.API.info("Permission granted.");
            } else {
                Ti.API.info("Permission not granted.");
            }
        });

    Unfortunately, evt.success always comes back true, even if the user declines the additional permissions.

    Worse yet, more calls to requestPermission don't cause a dialog window to appear. Has anyone else encountered this problem or found a workaround?

  • Okay, I've figured it out.. just a tiny bit more searching was required. It's because of a bug in Titanium dealing with responses from Facebook, I think. I suppose a fix on this probably isn't coming since it'll all be superseded by the new Graph API implementation of Facebook in 1.6.0. But this works!

    Install the patch described in https://appcelerator.lighthouseapp.com/projects/32238/tickets/2792-ios-facebook-permissions-return-false-positives to /Library/Application Support/Titanium/mobilesdk/osx/1.5.1/iphone/Classes/FBConnect/FBDialog.m.

    Then delete your build/ directory in your app, and re-launch it. And then it actually works! Phew.
