Webview only loads once

I have a window that is shown when a user clicks a "more info" button. The window contains a web view, with a local html file displayed. There is a toolbar with a back button, which closes this window and returns you to where the more info button was.

This works fine the first time it's called, but if you close the window and click "more info" again, the window opens but the html view is empty.

What am I missing? view code in pastie

    I agree with Javier. Below I am posting a code example:

    function showIntroText() {
    var iw = Ti.UI.createWindow({
        backgroundColor: '#d4d6d5'
    });

    iw.add(introWebView);
    iw.add(introToolbar);
    var introWebView = Ti.UI.createWebView({
        url: 'html/intro.html'
    })

    // more info 'close' button
    b = Ti.UI.createButton({
        title: 'Back'
    });

    b.addEventListener('click', function() {
        iw.close();
    });
    var flexSpace = Titanium.UI.createButton({
        systemButton:Titanium.UI.iPhone.SystemButton.FLEXIBLE_SPACE
    });

    var introToolbar = Ti.UI.createToolbar({
        items: [b,flexSpace],
        barColor: '#d63d29',
        bottom: 0
    })

    iw.open({
        modal:true,
        modalTransitionStyle:Ti.UI.iPhone.MODAL_TRANSITION_STYLE_FLIP_HORIZONTAL,
        modalStyle:Ti.UI.iPhone.MODAL_PRESENTATION_FULLSCREEN,
        navBarHidden:true
    });
};

  • You add the webView to a window that is destroyed when you close it. You can:

    a) declare introWebView component inside showIntroText() function, so is created each time the function is called.

    b) call iw.remove(introWebView) before the window is closed (capture the close event) to release the UI component from its parent (the window)

    I don't see why you would want to leave in memory the webView component when the window is closed, so I suggest you the first option.
