Developer keeps downloading and installing 1.2.2 update

Windows 7. Titanium Developer 1.2.2

Every time I launch developer a dialog opens saying it is downloading and installing application update (1.2.2)

How can I stop this?

    • Interesting. Prior to the 1.2.2 "update", for fun I used developer to import "Titanium Developer" as a project and then build it, just to hack around the code.
      Tried that.
      If using the 1.0.0 Desktop SDK: I get an "Error: Missing installer and application has additional modules that are needed."
      If using the 1.1.0 Desktop SDK: I get the same behavior… "downloading and installing …"
      However, I re-launched the "Titanium Developer" project again and the message went away.
      I'm not saying that Virus scan didn't cause a problem, but turning it off didn't resolve it.

      Steps:
1. Launch Titanium Developer (Ti Dev)
2. Import Titanium Developer as a project into Ti Dev. 
3. Under Edit, confirm that 1.1.0 Titanium SDK is selected.  Save changes.
4. Under Test&Package, click Launch app.
5. Get dialog box about 1.2.2 update. Let it finish.
6. Shutdown/exit/kill the launched Ti Dev project app.
7. Click Launch app again. ( I did not get the update dialog )
8. Shut down/exit/kill the launched app
9. Shut down/exit Ti Dev.
10 Restart Ti Dev.  ( I did not get the dialog again)

      FYI: I just checked and McAfee services were still disabled during the build process.
