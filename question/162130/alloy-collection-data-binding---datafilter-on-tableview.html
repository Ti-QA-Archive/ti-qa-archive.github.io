Alloy Collection Data Binding - dataFilter on TableView

I have a collection (stored as sqlite db) that I am trying to bind to a table view.

The db schema is id, type, name where id is auto increment integer, type is text and name is text.

Let say my data is as follow: 

id  | type  |  name
0   | AA    |  Joe
1   | AA    |  Marie
2   | MA    |  Steve
3   | SA    |  John
4   | SA    |  Mark

My view XML is as follow 

<Alloy>
    <Collection src="contacts">
    <Window id = "ContactsWindow" title ="Contacts">
        <TableView id="ContactsTable" dataCollection="contacts" dataFilter="filterByType">
            <TableViewRow>
                <Label id="name" text="{type}"></Label>
            </TableViewRow>
        </TableView>
    </Window>
</Alloy>

My intention is to create the tableView to first display only 

AA
MA
SA

Then use can click to one of them, for example, if the user clicks AA, then it will open another tableView with 

Joe 
Marie

as the entry.

The question is, how do I do a distinct dataFilter ?

I tried to do the following

function filterByType(collection) {
    return _.pluck(collection, 'type');
};

But it gave me error message

message = "'undefined' is not an object (evaluating 'value[key]')";

    The function you assign to dataFilter expects you to return an array of models. You are returning an array of values from those models, which is invalid.

    Why don't you just use a query in your fetch(), then you don't need a filter or transform at all on the data (untested):

    Alloy.Collections.contacts.fetch({
    query: 'SELECT DISTINCT type FROM contacts'
});

  • The collection object is a complex object and values aren't available directly as you're using them. Here's how filtering is done in the training app AlloyHunter

    <TableView id="table" dataCollection="Fugitive" dataFilter="dofilter" >
    <Require src="FugitiveRow"/>
</TableView>

    function dofilter(_collection) {
    return _collection.filter(function(criminal){
        // match only those with captured == false
        return !criminal.attributes.captured;
    });
}

    In this case, filtering on the captured (boolean) property of the models. You should be able to adapt this to your need, where you're filtering on a value passed to the second table's controller.
