ListItem: Dynamically setting "canEdit" not working

Hey there,

i found a problem when dynamically setting the ListItem-property "canEdit" when defining its properties in a controller. Sample code:

(index.xml)

<Alloy>
    <Window>
    <ListView>
        <ListSection>
            <ListItem title="My Sample Item" id="myItem1" />
            <ListItem title="My Sample Item" id="myItem2" />    
        </ListSection>
    </ListView>
    </Window>
</Alloy>

(index.js)

$.myItem1.canEdit = false;
$.myItem2.canEdit = true;

$.index.open();

Expected behaviour: The first item can be swiped to show the deleteView, the second cannot be swiped because it is not made editable.
Given behaviour: Both items cannot be edited, because the property is ignored.

Note: When setting the property in the xml e.g. canEdit="true/false", everything works?!

Am i doing something wrong or it is a bug?

Thank you!

    You cannot directly access listItems. Read here

    You cannot explicitly set properties or bind events for a ListItem. You have to set them using the properties dictionary of a ListDataItem or ItemTemplate and the events dictionary of the ItemTemplate or ViewTemplate, respectively.

    You will need to set a template and then call updateItemAt
