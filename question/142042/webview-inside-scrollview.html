  • It is sometimes tempting to put a webview inside a scrollview, but since the webview has its own scrolling, the two views don't play well with each other. You are strongly discouraged from doing this. Perhaps you can build out the rest of this window's UI inside the webview and drop the scrollview. That's what I ultimately ended up doing.

    With some clever CSS and javascripting, you can build a pretty solid UI in the webview. In the app where I use a webview, I use it to display a mix of images and text (which is hard to do without a webview). Beneath that formatted text, I had a tableview, and I tried to put all of that into a scrollview. That was a disaster. I ended up building a tableview-like interface in HTML and using only the webview.

  • Put it inside a scrollview, and do this:

    // Alter height of webview dynamically
$.webView.addEventListener('load', function(e){
    actualHeight = e.source.evalJS("document.height;");
    e.source.height = actualHeight;
});

$.webView.addEventListener("touchmove", function(e) {
    e.preventDefault();
});

    • I didn't see any viewport specification in your markup.

      I use this:

      <meta name="viewport" content="width=' + vw + ',user-scalable=no,initial-scale=1.0,maximum-scale=1.0,minimum-scale=1.0,target-densitydpi=device-dpi"/>

      where vw is equal to Ti.Platform.displayCaps.platformWidth.

      I did a lot of testing with the viewport settings, and I had all sorts of bad behavior with different settings. This was the setting that worked for me.

      Maybe it will work for you.
