Elements moved off screen when editing (TextField in TableView in ScrollView)

I have got a simple profile edit dialog consisting of several TextFields. The TextFields are layed out within a non-scrollable TableView (split into three sections) which is added to a ScrollView (as well as other elements like buttons, …). When editing textfields that are at the bottom of the screen they move up automatically in order to not be hidden by the keyboard. Unfortunately, instead of "scrolling" the contents of the ScrollView, the content of the TableView is scrolled instead. Thus the first elements of the tableview are not visible anymore and between the last element of the TableView and the element that used to be right below this last element is suddenly a huge gap.

This is partly fixable by calling scrollToTop(0) on the TableView after the TextField loses focus, but as long as the TextField is not blurred the layout is destroyed.

Any ideas/workarounds?
