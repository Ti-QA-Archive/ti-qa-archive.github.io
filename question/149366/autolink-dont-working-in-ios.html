AutoLink don't working in iOS

var local = Ti.UI.createTextArea({
    editable: false,
    backgroundColor:'transparent',
    value:'Rua Silva Jardim 151, Porto Alegre, BR',
    font: { fontSize: 16 },
     bottom: 90,
     left: 40,
     autoLink: Titanium.UI.AUTOLINK_MAP_ADDRESSES
})
win.add( local )

In this example my textArea only link Rua Silva Jardim 151, Porto… the rest of my sentence are cut from autoLink…
anyone knows why?

    Hi

    As it is an auto-linking property which tries to match what it believes is a valid match, it looks within the text and finds a match, so if it finds a match for an address it will mark it as such.

    You have simply provided more address detail than it required.

    BTW if you go to Google Maps and enter Rua Silva Jardim 151, Porto, it will also be found without problem.

      var map = Ti.Map.createView({
    animate: true,
    height: Ti.UI.FILL,
    mapType: Ti.Map.STANDARD_TYPE,
    region: {
        latitude: -30.02321,
        longitude: -51.190619, 
        latitudeDelta: 0.005,
        longitudeDelta: 0.005
    },
    regionFit: true,
    userLocation: true,
    width: Ti.UI.FILL
    //annotations: [mountainView]
});    
win.add(map);

    • Sorry - should have read closer.

      Show a map with one point

      Ti.Platform.openURL('http://maps.google.com/maps?q=-30.02321,-51.190619');

      Show a map with a route

      Ti.Platform.openURL('http://maps.google.com/maps?saddr=-30.02321,-51.190619&daddr=-31.02321,-52.190619');

      The google reference works for all versions of iOS even those that use Apple maps.
