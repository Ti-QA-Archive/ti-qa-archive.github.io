iOS 7 Modal Windows in Landscape orientation issue

Anyone else having issues with opening modal windows when in landscape mode?

When I open a Modal window, from my App which is setup in Landscape orientation the Modal window opens using only about 60% of the screen height, and display thinks it's wider than it is, as the view no longer displays in the center, but to the right. This occurs in either iPhone or iPad.

See an example screenshot….
http://tinypic.com/view.php?pic=2z9cwgz&s=8#.UysOPa1_s7k

(you can only see as far down as the "date of birth" field).

I'm using the new createNavigationWindow API, as documented in the iOS7 Migration guide.

I'm using:

Titanium SDK: 3.2.2.GA and
iOS SDK: 7

Issue is reproduceable with the following code:

var winRegister = Titanium.UI.createWindow(
            {
                url:'/ui/common/account/register.js',
                   title: 'Register'
            }
         );

var navWin = Ti.UI.iOS.createNavigationWindow({
                modal: true,
                window: winRegister
            });

winRegister.navWin = navWin;
navWin.open();

By the way my tiapp.xml contains the following code to force it into Landscape…

<orientations device="ipad">
       <orientation>Ti.UI.LANDSCAPE_LEFT</orientation>
       <orientation>Ti.UI.LANDSCAPE_RIGHT</orientation>
 </orientations>

I've tried opening the window using the different modalStyles, but get the same issue, where the window doesn't use up all the available screen real estate

e.g.

navWin.open({
    modalTransitionStyle: Ti.UI.iPhone.MODAL_TRANSITION_STYLE_CROSS_DISSOLVE,
    modalStyle: Ti.UI.iPhone.MODAL_PRESENTATION_FORMSHEET
});

Any one else seen this?

Has to be a bug with the navigationWindow API.

Thanks!

    Hi

    Just a thought could this be something simple like it decides the width and height prior to the orientation change starting? So in this case the width is the height and the height is the width, you saying around 60% sounds close enough.

    ".yourmodalwin": {
    height: Ti.Platform.displayCaps.platformWidth,
    width: Ti.Platform.displayCaps.platformHeight
}

    This would map each height and width to the opposite property.

    Worth a go?

    • Hi Malcolm,

      Thanks for that.

      It didn't work as you suggested, but from following your suggestion was to set the height = platformHeight and width = platformWidth. This now makes the modal window use fullscreen. (Previously I had tried setting height: 'auto', width: 'auto', without any success).

      Now it open's full screen but the screen hides behind the top status bar, and window bar. To work around this I had to set top: 0

      e.g….

      width: Ti.Platform.displayCaps.platformWidth,
height: Ti.Platform.displayCaps.platformHeight,
top: 0

      For anyone else coming up with this issue here's my complete createWindow code:

      
var winRegister = Titanium.UI.createWindow(
            {
                url:'/ui/common/account/register.js',
                title: 'Register',
        width: Ti.Platform.displayCaps.platformWidth,
        height: Ti.Platform.displayCaps.platformHeight,
        top: 0
            }
         );

      Thanks - this is working now.
