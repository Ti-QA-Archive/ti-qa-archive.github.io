FIX - If you are packaging for Distribution and iTunes Connect warns about "icon dimensions (0 x 0) don't meet size requirement"

I went to Validate my app against iTunes Connect this morning and got the following error:

Icon.png: icon dimensions (0 x 0) don&#39;t meet the size requirements. The icon file must be 57x57 pixels, in .png format

It turns out that the fix for this is to install a newer version of the Application Loader from here:
https://itunesconnect.apple.com/apploader/ApplicationLoader_2.5.1.dmg

Thanks to this thread for providing the working answer:
http://www.iphonedevsdk.com/forum/iphone-sdk-development/54784-icon-dimensions-0-x-0-dont-meet-size-requirements.html

Thought this might help somebody as I couldn't find anything in this forum…
