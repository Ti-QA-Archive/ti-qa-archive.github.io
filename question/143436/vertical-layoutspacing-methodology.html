Vertical layout/spacing methodology

Hi all

Ok, so in Titanium you explicitly set the position of each added element - very different to the way I'm used to laying content out with HTML/CSS where everything is positioned for you in order with block elements.

What kind of methods do people use to stagger content without having to manually set everything? Furthermore, if one needs to switch content on/off and have content that flows after it shift position accordingly, how is this best achieved?

When I say content, I am referring to various labels, buttons, etc, which will be placed one after the other in a scrollable view.

Thanks.

  • Accepted Answer

    Ok, so you don't have to statically position everything. You can set the layout property on any view to give it either a vertical or horizontal value to lay things out similar to the way that elements lay out in html. Vertical would be like stacking block level elements on top of each other and Horizontal would be like setting float:left on a bunch of images and stacking them from left to right, wrapping when they get to the end of the view.

    With that knowledge you should be able to get your content laid out the way that you want to.

    When you need to relayout content, the best idea is to create a function that handles nothing but the layout of whatever it is that you need specific dynamic layouts for. When something in that view changes, call that layout function again to relayout that view.

    As for the space at the end of the scrollview, you can either set the bottom property on the last item to the space you need, or create a white view with the space you need. The white view will be undetectable as it will look like the background of the scrollview.
