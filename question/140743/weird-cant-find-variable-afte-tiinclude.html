  • Its very hard to tell you what is wrong with the little bit of info that you've provided, however, if you're trying to find a way to globalize variables, you really should encompass them in a commonjs module and require() them instead of using Ti.include and possibly polluting the global namespace. You'll find this is much easier to keep track of as opposed to including a file inside another file inside a third file that you may or may not need to include in the first file.
