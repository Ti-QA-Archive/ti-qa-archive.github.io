  • I'm aware of the app.tss issue in 1.1.0. It is logged here: https://jira.appcelerator.org/browse/ALOY-625

    The issue has been resolved and we are working on posting a 1.1.1 version that will make app.tss files function as expected. In fact, if you want to try it out right now, you can install the candidate release from npm and test it with this fix. If you do, please report back and confirm that it solves your issue, I would be most grateful:

    sudo npm install -g alloy@1.1.1-cr

    The quick and dirty work around for this issue, if you don't want to install yet, is that app.tss files don't apply to any views that don't have their own view-specific style. This means that, for example, index.xml will not be styles by app.tss unless index.tss exists, even if it is blank. Adding a blank view-specific TSS will get app.tss to style your views in the meantime.

    Sorry for not understanding the root of this issue sooner. 1.1.1-cr is available now and 1.1.1 official should be around by the end of the week to resolve it completely and removing the need for the aforementioned workaround.
