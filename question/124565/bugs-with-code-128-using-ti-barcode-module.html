Bugs with Code 128 using Ti barcode module

Hi, I see some Q&A regarding Code 128 barcodes missing characters, and I am seeing those bugs, additionally I have noticed one other bug that I would like to bring to the attention of those addressing these bugs.

If you have two (or more) valid Code 128 barcodes in the viewer at one time, the module (incorrectly) attempts to resolve both of them into one scan returning one gibberish text string from them both.

This does not happen with Code 3of9 where the module picks one of them and gives its text which is understandable and acceptable.

I am using
Titanium Studio, build: 1.0.4.201108101535
Ti barcode module 1.2 for iphone
Using the example app.js supplied with the module
