Build Issues with Titanium Studio 3.2 and SDK 3.2.0.v20131209

I cannot build any project after updating Titanium Studio and installing the latest preview of the SDK. I am getting this error when running from command line.

  var localNSMap = config.localNSMap;
  TypeError: Cannot read property 'localNSMap' of undefined

Anybody have any ideas? Also, I can't seem to build using the 3.1.3GA SDK anymore as it says invalid iOS version 7.0.3, must equal 7.0

    • 'Sound Ideas' let me first say, your question is incomplete.

      What you FAILED TO MENTION that is rather important, is that the error comes from the build process, not actually your js files. By asking an incomplete question, you waste volunteer time.

      I believe that you left the following off the beginning of your error.

      /Users/username/Library/Application Support/Titanium/mobilesdk/osx/{your_sdk_version}/node_modules/xmldom/sax.js:92

      This would tell us where the error was actually occurring during the build.

      You also left off the name of the file that was most likely to have been Symlinked like so:

      [DEBUG] Symlinking /development/project/Resources/myfile.ext => /development/project/build/iphone/build/Debug-iphonesimulator/project.app/myfile.ext

      Had you mentioned this, someone would have been able to say, 

      Hey, take a look at the next file to be processed, and check for errors in the markup!

      Ie, something as simple as

      <H2>my header</h2>

      will break the builds at this point.

      — commented December 12th 2013 by Stephen Feather
