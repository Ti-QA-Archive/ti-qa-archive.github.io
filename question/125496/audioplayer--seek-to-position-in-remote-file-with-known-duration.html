AudioPlayer / Seek to position in remote file with known duration

Is it somehow possible to seek to a certain position of a remote MP3 with the AudioPlayer on iOS? I know it's not documented and in order to do this the AudioPlayer needs the duration of the file. I know the duration in milliseconds for every file i want to stream.

Has anyone an idea for this? A hack maybe?

I've allready seen another thread where it is recommended to use the VideoPlayer to do this, but i think with the VideoPlayer it's not possible to continue streaming when the app is closed or the device goes to sleep. Right?

  • There is a forked version of the 1.7.0 mobile SDK on GitHub, where all of the missing features (seeking, remote control, volume control) are added to the AudioPlayer on iOS. The author (gerry3) also opened a pull request for Appcelerator to include his changes in the original mobile SDK, but it seems they did not. I'll try to hack the changes into the current SDK myself. If it works, I'll post a guide here.

    Fork: https://github.com/instantautomatic/titanium_mobile

    Pull request with all relevant changes: https://github.com/appcelerator/titanium_mobile/pull/86

    Files that have to be changed:

    • iphone/Classes/AudioStreamer/AudioStreamer.h
    • iphone/Classes/AudioStreamer/AudioStreamer.m
    • iphone/Classes/TiMediaAudioPlayerProxy.h
    • iphone/Classes/TiMediaAudioPlayerProxy.m
    • iphone/Classes/MediaModule.h
    • iphone/Classes/MediaModule.m
