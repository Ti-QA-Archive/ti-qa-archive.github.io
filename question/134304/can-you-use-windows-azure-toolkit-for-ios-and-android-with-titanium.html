Is it possible or is there some other better way of implementing the back-end server than using Windows Azure Toolkit for Devices?

  • Ti framework includes the httpClient. This allows for any sort of http based interaction with the server side. It can consume both XML and JSON payloads. So, any server technology that can produce this is suitable to use. If you are specifically using Microsoft tech on the server side then you can use either Web Services (asmx) or WCF. WCF has easier ways to produce xml and JSOP output
