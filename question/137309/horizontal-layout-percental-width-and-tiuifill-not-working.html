Horizontal layout, percental width and Ti.UI.FILL not working

I want to place labels across a view. From left to right. The leftmost label should be aligned to the left of the parent view, the rightmost label should be aligned to the right. The labels should be evenly spaced. I use horizontal layout, cause I don't have to calculate the left position. I use percental width, cause I don't have to bother orientation changing. To avoid rounding errors (which are leading to a small gap on the right side), I set the width of the rightmost label to Ti.UI.FILL. I expected the label to fill the remaining space. But instead this label is not displayed. Has anyone an idea why?

I've setup a simple demo. Just create a new project and replace the original content of app.js with the following code:

var win1 = Titanium.UI.createWindow({
    navBarHidden: true,
    backgroundColor: '#000000',
    layout: 'vertical',
    exitOnClose: true
});

var view = Titanium.UI.createView({
    left: 0,
    top: 0,
    width: Ti.UI.FILL,
    height: 50,
    backgroundColor: '#ff0000',
    layout: 'horizontal'
});

var columns = 4;

var spacing = 1/2.4;
var remainingWidth = 100 - (columns -1)*spacing;
var columnWidth = remainingWidth/columns;

for (var n = 1; n <= columns; n++) {
    var cell = Titanium.UI.createLabel({
        left: (n == 1)? 0:spacing.toString() + '%',
        top: 0,
        // This works, but leaves a small gap on the right side.
        // width: columnWidth.toString() + '%',
        // This doesn't work. The view will not be displayed.
        width: (n == columns)? Titanium.UI.FILL:columnWidth.toString() + '%',
        height: 50,
        backgroundColor: '#00ff00',
        text: n.toString()
    });
    view.add(cell);
}

win1.add(view);

win1.open();

Application type: mobile. Titanium SDK: 2.0.1.GA2. Platform & version: Android 2.3.3. Device: Android emulator. Host Operating System: OSX. Titanium Studio: 2.0.2.201204051557.
