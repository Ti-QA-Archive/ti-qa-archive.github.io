  • Is there a question in here somewhere? With 2.0 we released some layout updates that changed the way widths and heights were handled and it required some updating. These changes were made to better the entire SDK and improve UI as well as keep up with the layout systems of both platforms. All this is laid out in the release notes for anyone porting their apps from before 2.0 release. These updates were never considered 'problems'.

    With everything else, you have not provided any error logs, output, messages, or anything that might help someone help you with any issues that you're having.

    People are putting out quality products with these same tools every single day. Keeping on the heels of the constantly updating platforms is usually the best way to stay ahead in this game.
