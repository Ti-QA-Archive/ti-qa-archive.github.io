ImageView does not completely fill in width

I have this simple Alloy view:

<Alloy>
    <Window class="container">
        <ImageView id="header" />
        <ScrollView id="content" />
    </Window>
</Alloy>

and its style:

".container": {
    backgroundColor:"white",
    width: Ti.UI.FILL,
    height: Ti.UI.FILL,
    layout: 'vertical',
    fullscreen: true,
    navBarHidden: true,
    exitOnClose: true
},
"#header": {
    image: Alloy.Globals.Images.HomeHeader, // 1024x381 pixel
    width: Ti.UI.FILL
},
"#content": {
    height: Ti.UI.FILL,
    contentHeight: 'auto'
}

The ImageView does not completely fill in width and on the left and on the right appear a border (something like this CSS: border: 1px solid #FFF 0 1px 0 1px).

If I set manually the width (in DP units full width of my smartphone is 360) with 361, the borders disappear.

The app was tested on two Android 4.2 smartphones with different resolution (Xperia SP and HTC One) and compiled with Titanium 3.2.0, SDK 3.2.0.GA and Alloy 1.3.0 on Windows 8.1 64 bit.

— asked January 18th 2014 by Filippo Neri
