Issue with remote image using imageView on iOS but not Android

Application type: mobile
Titanium SDK: 3.2.1 GA
Platform & version: iOS 7.0
Device: iPad & iPhone 4-5
Host OS: OSX 10.9.2
Titanium Studio: 3.2.1

I am having an issue trying to use an imageView on a remote image with iOS and not on Android. I am currently working on an app that shows an image on the home screen. Here is an example:


 var self = Ti.UI.createWindow ({
 backgroundColor: 'white',
});


var page_pic = Ti.UI.createImageView({
        top: '5dp',
        width: Ti.UI.SIZE,
        height: Ti.UI.SIZE,
        image: "http://www.fleetwoodusa.net/css/images/Raftery_Exterior.jpg",
        borderWidth: 0,
    });

self.add (page_pic);

self.open();

On Android it works just fine but on iOS I get that mountain place holder. I checked our server logs and it shows that it is "touching" our server but I get an 404 error. Here is an example of one of our log entries:

03/Mar/2014:11:08:54 -0500] "GET /css/images/Raftery_Exterior.jpg HTTP/1.0" 404 - "-" "Appcelerator Titanium/3.2.1.GA (iPhone/7.0.6; iPhone OS; en_US;)"

If anyone can help enlighten me on what may be the problem I would greatly appreciate it.
