  • Though the alloy update coincided with the problem, I don't think it's actually associated with alloy. It looks to be a configuration issue or corruption of your android emulator.

    Are you able to get the android emulator running at all? You may want to delete the emulator you use for titanium apps (they will have a "titanium" prefix) from the AVD listing by manually running android avd then delete it from the list and let TiStudio create a new one for you when you attempt to run your android app again. Or you can target a different emulator, if you have more than one, by going to Run -> Run Configurations…. In there, expand the list under Titanium Android Emulator in the left pane. Select your app and in the right pane select a different google API, click "apply", and then "run". This should attempt to launch your app in a new emulator.
