    • No, global variables like that will not 'automatically' make all your textColor (or whatever) of every object you created inherit that value. Where did you read that ? You can place Alloy.Globals values into .tss files but for what reason ? The global .tss file is the way to proceed. That's what it was invented for.

  • Hi Suraj,

    Thanks for your concern.

    I've checked Nick Milner's answer and it's working for me.

    I think you are little confused about the answer.

    Please try the following code.

    Create a app.tss file like

    app.tss

    "Window":{
    backgroundColor: 'red'
}

    index.tss

    ".container": {
    //backgroundColor:"white"
},
"Button": {
    width: Ti.UI.SIZE,
    height: Ti.UI.SIZE,
    color: "#fff",
    borderColor: 'green',
    backgroundColor:'black'

}

    firstWin.tss

    ".new": {
    //backgroundColor:"white"
},
"#lbl": {
    width: Ti.UI.SIZE,
    height: Ti.UI.SIZE,
    color: "#000"
}

    I've checked this way, common background color in the app.tss file. I don't use the background color for other tss file and it's working. I've checked it on android platform and only for the window common background color.
    Do the same thing for label or whatever you need,

    Hope it will help you.

    Thanks
