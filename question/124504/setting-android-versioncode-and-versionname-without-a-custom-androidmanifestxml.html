Setting Android VersionCode and VersionName without a custom AndroidManifest.xml

Hey all,

This isn't a question, I just wanted to let others know about something I just found out. When you update your app in the Android market, you have to increment the Version Code, however by default the AndroidManifest.xml sets the Version Code to 1. Traditionally we got around this by creating a custom AndroidManifest.xml, however then you have to maintain that custom file, which can be a pain when upgrading the Ti SDK version.

Fortunately, there is another way since Ti SDK 1.5, although I didn't know until now. In the tiapp.xml you can do this:

<android xmlns:android="http://schemas.android.com/apk/res/android">
<manifest android:versionCode="5" android:versionName="5">

This will copy the versionCode and versionName over to the generated AndroidManifest.xml so there is no need to create a custom one.

I apologize if everyone else already knew this but I never got the memo so I bet some others haven't either.

  • correct is:

    <android xmlns:android="http://schemas.android.com/apk/res/android">
        <manifest android:versionCode="INTEGER" android:versionName="STRING"/>
</android>

    f.e.

    <android xmlns:android="http://schemas.android.com/apk/res/android">
        <manifest android:versionCode="5" android:versionName="1.0.5"/>
</android>
