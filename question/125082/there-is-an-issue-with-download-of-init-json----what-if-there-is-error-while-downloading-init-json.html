  • I suppose the init json is something you want to download from a remote site when your app starts.

    In this case, the xhr request has the onerror event that triggers when the connections fails for some reason.
    If the json is downloaded but is faulty, you can try to process it in the onload event function of the xhr with a try/catch trap and inside json.parse

    something like this (pseudocode):

    
xhr.onload =  function() {

    try {
        JSON.parse(this.responseText);
    }
    catch (err) {
        alert('we have an error on processing the json');
    }
};

xhr.onerror =  function() {
    alert('we have an error with the internet connection')
}

    This is what you want?
