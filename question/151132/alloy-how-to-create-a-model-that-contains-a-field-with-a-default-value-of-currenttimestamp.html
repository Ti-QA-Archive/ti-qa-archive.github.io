  • Work-around

    config: {
      "columns": {
            "date_created": "REAL"
      },
      "defaults": {

      },
      "adapter": {
            "type": "sql",
            "collection_name": "Table1"
      }
}

    You can then insert Javascript TimeStamp new Date().getTime() to date_created Column.

  • Hi Mark

    You can try pre-loading the sqlite schema to achieve this

    Specify a database file to preload

Define the db_file key-value pair in the config.adapter object to specify the database file ('myfile.sqlite') to preload. Place this file in the app/assets directory of your Alloy project. Alloy creates a database using the name of the database file minus the file extension if one does not exist.

  • "defaults" in this case refers to the default values that will be used to populate the Backbone model, not whatever the back-end data storage is, in this case SQLite. To set a default for SQLite in this case, you need to add the default directly to the column definition.

    In your initial answer, your 2nd attempt looks valid to me. What behavior are you seeing in that case? Errors? Bad data? Please be specific.
