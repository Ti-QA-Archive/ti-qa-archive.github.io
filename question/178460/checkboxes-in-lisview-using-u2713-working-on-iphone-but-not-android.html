Checkboxes in Lisview using '\u2713' working on iPhone but not Android?

Hi All,

I'm new to Titanium Alloy Development and this is my first time posting to this forum, so if I break protocol I apologize.

I'm currently trying to build a list view with a custom template. The template is made of two labels and an image, the first label is going to be a checkbox using '\u2713'. When I run the code on any of the iOS simulators, it works as I intended with each individual label being updated when it is checked or unchecked. However, when I run this on my Galaxy S4, the empty boxes are never filled, but the alert pops up saying that e.source.text was in fact updated. I also notice that e.source.text doesn't seem to be referring to the label that was just clicked because e.source.text will alternate between '\u2713' and an empty string regardless of which box is checked. Is there some different way to access the Label text for $.check? Or is the functionality I'm attempting not possible?

function checkClicked(e) {

    var check = e.source.text || '';
    if(check != ''){
        e.source.text = "";
    }
    else{
        e.source.text = "\u2713";
    }
    alert(e.source.text);
}

<ListView id="list">
    <Templates>
        <ItemTemplate name="rewardTemplate" height="50" layout="horizontal">
            <Label id="check" bindId="check" onClick="checkClicked"></Label>
            <ImageView id="leftImage" bindId="leftImage"/>
            <Label id="name" bindId="name"></Label>
        </ItemTemplate>
    </Templates>
    <ListSection>
        <ListItem template="rewardTemplate" check:text="" leftImage:image="/images/alien.png" name:text="A short description"/>
        <ListItem template="rewardTemplate" check:text="" leftImage:image="/images/predator.png" name:text="Testing"/>
        <ListItem template="rewardTemplate" check:text="" leftImage:image="/images/pod.png" name:text="Some test"/>
        <ListItem template="rewardTemplate" check:text="" leftImage:image="/images/predator.png" name:text="Testing"/>
        <ListItem template="rewardTemplate" check:text="" leftImage:image="/images/pod.png" name:text="Yet another string"/>
    </ListSection>
</ListView>

  • android requires a capital 'U'
    ios requires a lowercase 'u'

    I've had this same issue and previously reported it to Jira. Unless something has changed, you need to branch the logic

    OS_IOS ? "\u2713" : "\U2713"

    https://jira.appcelerator.org/browse/TIMOB-16815
