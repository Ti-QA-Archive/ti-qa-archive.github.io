Push Question

I have a question that is probably pretty simple for those who have used Push in their applications.

I know I can register with Urban Airship, but my app is going to maybe be used statewide, and some counties might send push for different events, and so I don't want everyone to receive push.

-a user will login with their account, which will authenticate with the system
-I'm supposing I can then store the device's id in my MySQL database
-I can then run a script on my server that will search users by email (sort of like a cron job) and then send a push to those device id's

Is that possible using one app and one Urban Airship registration?

  • I am unfamiliar with Urban Airship, but if what you are saying is the correct workflow, then yes it's possible.

    Using the createHTTPClient (see docs) you can send XHR (POST or GET) requests to wherever you need, get the data back and process it on the app.

    There are a few different methods of storing info, if you want to store just a few variables you can use Titanium.App.Properties.setInt('varname',123) and Titanium.App.Properties.getInt('varname'). Otherwise check out the database method in the docs.
