Image zoom doesn't work after updating to SDL 1.8

Hello Appcelerator,

I have a small image gallery in my iPhone apps, with the SDK 1.7.5, the zoom works just fine but after upgrading to 1.8, i can't zoom any more.

I just have a scrollableView that contains an array of images, each image is an normal imageView like this:
view = Ti.UI.createImageView({
backgroundColor : '#000',
image : images[z].url,
defaultImage:'images/img_not_found.png',
width : 320,
height : 460
});

Please tell me if i did something wrong or there's any property to add that i miss..?

  • The scrollableView is as follow:

    photosView = Ti.UI.createScrollableView({
    backgroundColor:'#000',
    width : 320,
    height : 460,
    top : 0,
    showPagingControl : false
    maxZoomScale : 2.0,
    currentPage : 0
});
...
...
view = Ti.UI.createImageView({
    backgroundColor : '#000',
    image : images[z].url,
    defaultImage:'images/img_not_found.png',
    width : 320,
    height : 460,
    top : 0,
    photoId : z
});

    With the SDK 1.7.5, everything worked fine.

    • I am seeing a similar issue. Comparing the online apidoc between 1.7.5 and latest, I see that maxZoomScale is no longer listed as a property for Ti.UI.ScrollableView.

      Can someone say if this functionality was removed??

      — commented January 20th 2012 by Kevin Coffman

    • yes, this functionality has been remove recently. The scale event is no longer supported in ScrollableView. To be able to zoom in a scrollableView, you have to add a scrollview inside a scrollable view.
      Best,
