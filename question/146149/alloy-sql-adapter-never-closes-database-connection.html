Alloy sql adapter never closes database connection

In the quick start guide it recommends that database connections be closed promptly with the following explanation:

In a mobile app, you're dealing with a single-user, memory constrained environment in which connection pooling is not important. The time to open/close connections is less of a hit that consuming memory by keeping connections open.

Yet the alloy sql adapter NEVER closes it's database connection. Is this just an oversight in the coding of the sql adapter, or is it intentional?

Also I was wondering if data can be lost due to the database connection not being closed properly?
