Multithreading in Titanium

Hi everybody!

I am creating an app in which I want some network-related things to happen in the backround and keep the UI responsive. Can somehow threads be used? I have seen that others recommend setTimeOut or setInterval, others say that creating a separate window, we create a new sub-context and it will behave like a thread, but does it really behave like a real thread? Is there some kind of trick to implement threads?

Thanks a lot!

  • There is no need to use multithreading since XHR calls are asynchronous.

    var xhr = Ti.Network.createHTTPClient();

xhr.onload = function() {
    //when XHR request finishes, you can do another XHR request
};

xhr.open("GET","http://www.appcelerator.com/");
xhr.send({ hackme: true });

//rest of your logic, add, remove buttons, or what ever you want.

    Anyway, in Titanium, UI is implemented as separate thread, so you don't need to worry about responsiveness of the UI. Just use what ever JavaScript provides natively, there is really no need to create overhead with simulating threads.
