    • @Dan, Titanium Studio ships with a "HTML5-based Application" which despite the horrible name, is a native Titanium Mobile app for iOS/Android that does a bunch of WebView boilerplate stuff and you just add your HTML, CSS, and JavaScript.

    • @Stephen I don't agree that the Mobile Web platform has muddied anything. It's given developers the ability to target platforms that won't don't yet natively support and platforms that only allow HTML-based apps such as Blackberry, Windows Phone, Firefox OS, and so on.

      Parity is a hard problem to solve. We will never achieve 100% parity across ALL platforms. There are features that simply cannot be implemented. There are plenty of things that iOS supports that Android does not and vice versa.

      I understand that Mobile Web has major limitations. For example, Titanium Mobile Web will never support Ti.Database. Maybe someday we'll have a new database API which will be async and key/value based, not SQL based. Mobile Web supports Ti.Media.VideoPlayer. It will never support the iOS specific MusicPlayer. AudioPlayer support is coming later this year (fingers crossed).

      @Zane Mobile Web only supports Windows Phone 7.5, and even then we have limited support. IE on Windows Phone 7.5 does not support touch events, so you can't scroll ScrollViews, TableViews, or ScrollableViews. Windows Phone 8 (or whatever they call it) will supposedly support touch events. In other words, you will need to put a bunch of up/down/left/right buttons all over your app to get it to somewhat work on Windows Phone 7.5. Ugh, I know.
