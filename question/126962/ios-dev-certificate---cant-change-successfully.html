iOS Dev Certificate - Can't Change Successfully

I renewed by Apple dev package and got my 'new' Dev certificate. Installed it per the instructions, and it appears in KeyChain and Xcode organizer. Created my App ID and dev prov profile and Ti Studio seems to recognize it. I try to run my app to device and get 'System Exit 65'. Occasionally (every few run attempts) an error appears in the Studio console saying I have more than one Dev certificate. I check KeyChain and the old, expired one (same id as new one) is there, so I delete it. I run again and get Exit 65. I check Xcode Organizer and it shows two of them, but you cannot select either one to edit or delete. Occasionally after a few more run attempts, my deleted certificate has returned to KeyChain. This cycle has happened few times where the old cert reappears - cannot get app to run to device in any of the runs.

Any ideas? Thanks.

(Ti Studio 1.0.4.2, SDK 1.7.2, Xcode 4.02, SDK 4.3)

  • Okay, resolved it. When I ran the project in Xcode (which I should have tried along time ago) it gave me a more specific error message…telling me exactly where the two dupe certificates were…one in my 'logins' keychain and the other, old one in my 'system' keychain. Previously, I had just deleted the old one that was appearing in 'login'. Once I deleted the 'system' keychain's old certs, that did the trick. Serenity now.
