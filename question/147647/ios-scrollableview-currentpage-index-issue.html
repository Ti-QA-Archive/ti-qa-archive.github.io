    Hi

    If you add an event listener you would only see 1, 2, 3 - as the first page would not report as it would have already been shown. Obviously this is not what you are getting right now, but worth pointing out.

    I have knocked up a quick app for you to show how it should work. The currentPage property is pushed to the console for review.

    To test this create a new default project and then replace the contents of app.js with the code below.

    var win = Ti.UI.createWindow({  
    backgroundColor: '#fff',
    title: 'Tab'
});
var tab = Ti.UI.createTab({
    icon: 'KS_nav_ui.png',
    title: 'Tab',
    window: win
});

var view1 = Ti.UI.createView({ backgroundColor:'#123' });
var view2 = Ti.UI.createView({ backgroundColor:'#246' });
var view3 = Ti.UI.createView({ backgroundColor:'#48b' });
var view4 = Ti.UI.createView({ backgroundColor:'#c60000' });

var scrollableView = Ti.UI.createScrollableView({
    showPagingControl: true,
    views: [
        view1,
        view2,
        view3,
        view4
    ]
});
scrollableView.addEventListener('scrollend', function (e) {
    Ti.API.info(e.currentPage);
});
win.add(scrollableView);

var tabGroup = Ti.UI.createTabGroup();
tabGroup.addTab(tab);
tabGroup.open();

    Notice the currentPage property is returned from the event listener and not the original object.

    This will show;

    • Forwards; 1, 2, 3
    • Backwards; 2, 1, 0

    • There is some better news though. Whilst not an ideal fix I have a work-around.

      var currentPage = -1;
scrollableView.addEventListener('scroll', function (e) {
    currentPage = e.currentPage;
});
scrollableView.addEventListener('scrollend', function (e) {
    Ti.API.info('currentPage: ' + currentPage);
});

      This uses the fact that the scroll event DOES fire the correct index back (I checked). It uses a local variable to cache the result and then the scrollend event to show the answer.

      Let me know what you think.
