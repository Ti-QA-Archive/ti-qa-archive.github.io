    Hi

    Sorry but no.

    If you set userLocation to true then the display associated is controlled by the system.

    The closest you can get is to set userLocation: false and then place an image marker where the userLocation annotation would have been. You will not get the cool blue pulse with this technique so you have to choose which is more important.

    I find the answer for me is app dependant as there is not one clear winner for me.

    • I have been doing some more research and although my original answer is still true - you cannot do this in Titanium - there is hope for the future.

      It seems there is a JIRA ticket for Titanium that is proposing this feature. This is still OPEN so not implemented yet.

      Ti.map.view add direct access to user's currentlocation

      The proposed plan is to provide an event, this may also include the ability to set the caption, and if that is true - maybe the left and right images and click events.

      You could always write a custom module for this - but you would then need to understand ObjectiveC and Titanium Module Creation first.
