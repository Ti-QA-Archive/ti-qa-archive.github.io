  • Accepted Answer

    I had the same problem just now, and I was able to fix it by using an older version of the Android NDK.

    Having problems with the ndk-r9 installed I found that Titanium states here it only supports revision 8 of the NDK. So I tried installing ndk-r8e which seemed to work, but trying to test the module on a device gave me another error witch lead me to this question.

    I have now installed ndk-r8 and it seems to run fine now.

    Downloading Old NDKs:

    To download an old version however you have to type in the exact address of the file as explained here: http://stackoverflow.com/…/where-do-i-find-old-versions-of-ndk

    I have tested it briefly, and these links seem to work:

    Windows (x86): android-ndk-r8-windows.zip

    Mac OSX: android-ndk-r8-darwin-x86.tar.bz2

    Linux (x86): android-ndk-r9-linux-x86.tar.bz2

    Don't forget to change the address of the "android.ndk=" address in the "build.xml" file for it to work.
