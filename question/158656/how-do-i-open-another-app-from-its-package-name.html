How do I open another app from its package name?

Im currently making two apps, and I'm trying to implement switching between them without using the devices window managers.

On iOS i can do this by doing:

if (Titanium.Platform.canOpenURL('appName://')){
    Titanium.Platform.openURL('appName://');
}

This switches me to the other app window, I've got an else which tells them the other app was uninstalled too.

Now on android I think I need to use an intent, however Id normally specify the component name and open it, as the app has a action.MAIN and the launcher category. But I can't find how to specify the component name in titanium.

So I'm trying to use this:

var intent = Ti.Android.createIntent({
    action: Ti.Android.ACTION_MAIN,
    className:'appNameActivity',
    packageName:'com.org.appName'
});
intent.addCategory(Ti.Android.CATEGORY_LAUNCHER);
intent.setFlags(0x10200000);
Ti.Android.currentActivity.startActivity(intent);

But this gives me an unable to find activity error.

Any Ideas?

    While you can open apps via an intent, you can also do this with Ti.Platform.openURL() as you do on iOS. Though you need to configure the target app's URL scheme in its tiapp.xml. I have a working example in the training labs.

    • Source app (has buttons to open the other app): 09_source
    • Target app (the app that gets opened): URLSchemes

    Note that passing launch parameters as shown in those apps is not working on Android due to recent changes in the platform's underlying handling of activities. I'll update those training apps once all the changes are merged into the master branch.
