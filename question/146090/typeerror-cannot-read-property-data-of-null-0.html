  • Accepted Answer

    This is a known bug with 3.0.0.GA. You must either have an <ios><plist> section in your tiapp.xml or a custom Info.plist file that contains a setting who's value is a <dict>, which is probably CFBundleIcons. If you comment out those lines in your plist, it should work.

    Alternatively, you can download and install the latest untested 3.0.x build by running:

    titanium sdk update --branch 3_0_X --default

    You'll need to update the <sdk-version> in the tiapp.xml. Keep in mind that this 3.0.x build is NOT production ready. It hasn't been thoroughly tested. You should be fine for development.
