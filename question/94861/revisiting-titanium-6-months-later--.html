Revisiting titanium 6 months later :-(

I have outlined in a different post how frustrating it is to revisit titanium after 4-5 months and see that the old problems are still there.

Would like to sum some of these up here (as of December 29th 2010) and like to see these get solved as soon as possible.

I am developing for Android with titanium developer 1.2.2 and mobile SDK 1.5.1 - on a fast laptop. Windows 7.

(1) Update function of Ti Dev. This works only in a row - when you get back to Titanium after some months Ti Dev prompts for the installation of the latest Desktop SDK first, then eventually (after several minutes) the installation of the latest mobile SDK and again after tenths of minuts the installation of the next version of Ti Developer itself. This works, but on each restart of Ti Developer the download process is repeated for unknown reason. This has been the same problem in previous releases, as I vaguely remember. I'm not getting into this again, there might be something in Q/A, but I don't care. Don't post a workaround on some hidden Q/A place - solve it !

If you click on the right upper titanium button (it seems to be one) in Ti Developer nothing happens. There should be an immediate output saying 'all up-to-date' or 'New SDK's available'….

Ti Dev is slower than 6 months ago - just not very responsive. Clicking the links or the images is cumbersome.

(2) 'Compiling' for the emulators takes way too long. Compilation of a small project with two *.js files takes between 2-3 minutes, sometimes longer, sometimes hangs, which can only get solved by pressing 'Launch' again. 'Stop' sometimes works, sometimes it is greyed out.

(3) APK Installation. Installation of an apk on the emulator by TI Dev is a matter of luck. somtimes it works, other times it hangs. Restarting of Ti Dev and the emulator helps but is not professional.

(4) Website

There is still (!) no permanent login to the website. A cookie that is recognized preventing that you have to login all over again each day. Could somebody tell me one time why this is not done ? Implementation takes about 15 minutes. Let me do it for you :-)

Q/A has not changed much. You neeed to put the posts in logical order (!) within a question. latest posts at the end.
Q/A without categories is not useful at all. This post is a good example. It should not be on top of all posts, because it is negative (although constructive) feedback to your product, which could keep off potential pro customers. This post should go into a category 'suggestions' and stay away from the rest. Pls go and create categories !

API documentation is sparse. I miss working, complete code examples.

And: I just do NOT want to look at your kitchensink example. When do you get over this ? You want us to download it again and again in the 24th version just to find out ourself what has changed ? No way …

Feel free to add to this list !

Best regards

rudolf

  • Android emulator is actually not titanium software, it comes from the Android SDK. It is a well known fact that you need to kill all atb.exe processes running from previous emulators for Android. If you run Eclipse or Titanium makes no difference… Kill all atb.exe in processmanager, and start emulator first, and keep it running. It solves a lot of the problems described above.

  • Hello R B and thanks for checking out Titanium again.

    So, there are three main issues that you're raising - Developer, Q&A, and Docs. Our focus during the period you describe (for that matter, 2010) has been on building out the core API set. The release notes for each version show the major improvements in the API… and they are material. On the PC, this is especially true for Android. I'd certainly encourage you to look at the most recent release notes for Android as that's been a major investment/focus area.

    On Developer, well, that experience can be improved. All I can say on that one is that we'll be making some significant announcements in January that will go a long way to resolve those issues.

    As for Q&A/login, those are good suggestions and we are looking to refresh Q&A in Q1.

    Finally, on docs, I'd certainly encourage you to check out our new Titanium Guides section. It has over 30 new guides on all areas of Titanium development. We can do more, but this too is an area we're working on.

    In any case, thanks for coming back and we look forward to seeing what cool new app you come up with ;-)
