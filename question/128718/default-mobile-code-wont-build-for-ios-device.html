  • hey guys .. we finally managed to solve the problem for us ..

    somehow a wrongly encoded whitespace character squeezed into our js .. in titanium studio it looked like a normal whitespace but opening it with netbeans it showed up as something cryptic … removed this character (whatever it was) and everything was fine

    so how to find such a thing ..

    as normal titanium output was not helpful at all we rechecked the build log files in xcode and went up a pretty good bunch of lines until we noticed a class name we knew which was definitly ours .. we got lucky as it was one of the smaller classes .. and right in the second function we found it …

    may be if someone has a small tool that checks a file for 'valid' utf8 characters, which means characters that make sense in a js content .. maybe post a link here or something ..

  • Hi Brad,

    I got that error message this morning after making some improvements to my app… I tried everything, to no avail… until I read something about invalid characters on image filenames… Since I had just added some new .png files, I decided to try… after some testing (I could build it successfully after deleting my image folders), I found out a .psd (Photoshop) file I had forgotten in the Resources folder… after getting rid of it, it worked.

    So please check your filenames; if you don't see anything wrong, try deleting some files/folders in your Resources directory and building again and again to see where the problem may be.
