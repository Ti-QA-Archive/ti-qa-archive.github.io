  • Most Mach-O link errors usually relate to not specifying the library/frameworks in the module.config file.

    If the library/framework is a 3rd party one, the previous machine may have been set up with these libraries/frameworks in a certain location. Moving the module over without addressing the dependencies will definitely result in CLANG Linker errors.
