Titanium to iPhone Fails

For some reason I can't get the application that I've made to successfully port to the iPhone. The process runs just fine, it installs to the device but when I launch the application it fails and does not run. I have tried building the application (with success) in xCode, but the resulting application (on the device) gives the same problem.

If, after attempting to install the application through iTunes, I try and build the application in Titanium, the simulator launches and errors out saying it can't find the UIButton madness.

I have no error messages to provide because there are none except for the missing class files after the a fluke install.

Any help would be greatly appreciated.

    I would store app configuration values differently. Or, at least I've never tried what it looks like you're doing–you're setting the js file name and path as part of the API.config namespace: url : Ti.API.config.windows.home

    I'd create something within your app's namespace, like yourapp.config.home. See the tweetanium source for a good example of creating your own namespace.

    I guess where I'm going is that the Ti.API.config namespace might differ on device compared to in-simulator, but your app's namespace would match.

  • There are no errors coming up in the console. I figured out what the problem is though. Although I still have not found a solution. It appears that the application (when on the device) cannot find my JS file. Here is the code:

    var testWindow = Ti.UI.createWindow({ 
    title : 'Nearby',
    url : Ti.API.config.windows.home,
    barImage : Ti.API.config.windowBarImage
});

this.tabGroup.addTab(Ti.UI.createTab({ 
    title : 'Nearby',
    icon : Ti.API.config.icons.nearby,
    window : testWindow
}));

    Everything works just fine in the simulator, this is very frustrating.
