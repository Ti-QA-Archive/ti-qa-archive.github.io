Titanium Mobile 1.7 not dying on errors correctly

Using either the Titanium Developer or Titanium Studio causes this issue. The issue is when you have a variable that doesn't exist in a call back function as part of an ajax call. In 1.6.2 you'd get the exception correctly thrown back to the console/logs. In 1.7.1 you see nothing and changing the logging level does nothing to change this. It would be as if the async thread running the xhr calls isn't able to send it's error messages out. Since my apps do a fair amount of xhr calls this is a pain. I now I have to first build my code with 1.6.2 to ensure all the variable names and functions actually exist then recompile with 1.7.1 to make sure nothing broke between 1.6.2 and 1.7.1.

Please fix this Appcelerator. I'm sure I'm not the only one who think's he's going crazy from error messages magically not showing up in the logs when using 1.7. Perhaps this issue is related to the requirement to pay for debugging in 1.7 with Titanium Studio? If so then you'd be best served to give out debugging for free.
