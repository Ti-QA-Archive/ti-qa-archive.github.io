How to include 72x72 iPad appicon?

I am trying to build for both iPhone/iPad (Using Titanium 1.3.0), but when I build for distribution and upload via Apple's Application Loader, I get an error saying that I need a 72x72 .png in my resources folder, when I have both a 57x57, 72x72, and 114x114 file in my Resources folder.

Do I have to define it somewhere? I looked through the files but this new XCode 4 interface is just… Annoying. Especially when I was so used to the old one.

    In Xcode, expand your project and drag your icons into the Resources folder there. I had the problem too, and that seemed to fix it. It might put it at the top level of your Xcode project, if it does, just move them.

  • A better way to do it is to create an info.plist file in the root of your project. You need to add:

    <key>CFBundleIconFiles</key>
    <array>
        <string>appicon.png</string>
        <string>appicon@2x.png</string>
        <string>appicon-72.png</string>
    </array>

    The info.plist file will now be used during building for distribution to appstore and the different icons will be correctly used in the binary.

  • This made me crazy for just a little bit..so here's what worked for me. Save the files separately (outside of Titanium) as appicon.png, appicon@2x.png, appicon-72.png, and appicon-144.png. Add appicon.png to the Titanium project and make sure all of these icons are registered in the tiapp.xml by adding them to the icons array. i.e.
    <icon>appicon.png</icon>
    <icon>appicon@2x.png</icon>
    <icon>appicon-72.png</icon>
    <icon>appicon-144.png</icon>
    Now let Titanium clean build its project. Open the proj in xcode, drag the files onto their appropriate "No Image Specified" box in the Universal app project Summary area in Xcode and they should pop up in the box after you drag them. You will get errors if htey aren't sized properly. Now the icons are actually in the project for release. When you do another clean build, you will have to do it again… Anyone tell me if any of these steps are not necessary - for example, not sure if you have to add them to the tiapp.xml file
