Tail of request at Ti.Network.createHTTPClient

I wanted to request HTTP GET url like "http://foo.com/?key=…..", but when I used Ti.Network.createHTTPClient , "%0D" character was added to a tail of the request.
Since the server app checked tail strings as hash value, I had to got an error response.

var url = "http://foo.com/xml?Key=AAA";
var xhr = Ti.Network.createHTTPClient();
xhr.open('GET', url);
xhr.onload  = function() {
    var xml = this.responseText;
    Ti.API.info('URL     :' + targetUrl);
    Ti.API.info('location:' + this.location);    
    Ti.API.info('result  :' + xml);
}
xhr.send();

Result is

[INFO] URL :http://foo.com/xml?Key=AAA

[INFO] location:http://foo.com/xml?Key=AAA%0D

Why is "%0D" inserted? Is this the cause of problem?
