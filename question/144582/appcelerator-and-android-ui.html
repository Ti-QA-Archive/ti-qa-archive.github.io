    • Well, something like Roambi will be pretty hard to pull on Android, even in native Code. But if you really need to push the envelope like that, I can only assume that you are pretty well rounded with Android development and its intricate zBuffering. So my advice to you would be: Stay with native if you already have deep knowledge.

      As for Wunderlist, as simple as it may look, it's quite a complex app. While it is true they had to move away from Android (the app was aweful to use back then). It was largely due to the fact that the first version of Titanium was using the Rhino Javascript engine. Which was very slow, it has since been replace with the V8 engine which is much faster. But since it was not ready back then, it made perfect sense for them rewrite the thing natively instead of having to wait for a better Titanium (with V8).
