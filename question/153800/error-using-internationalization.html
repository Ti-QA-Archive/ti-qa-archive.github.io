    Hi

    Normally internationalisation is much easier to do than you would first think - so hopefully this is something simple.

    Firstly you need to create a folder if one does not exist as follows;

    [appname}/i18n/en/

    This is the English file (not the use of the en to denote this. To make sure you support all the languages you want to correctly, I suggest you toggle your phones international setting and display the value of this property in your app to confirm the correct folder name for each language;

    Ti.Locale.currentLanguage;

    Assuming you have sorted out other languages - we will concentrate on the en (English) one.

    Create a file in the folder we confirmed/created above called strings.xml and inside it place the following content;

    <?xml version="1.0" encoding="UTF-8"?>
<resources>
    <!-- app -->
    <string name="test">English</string>
</resources>

    Do the same thing in another country folder, but change English to the correct country name for easy detection. Then somewhere in your app add a label to confirm this works - something like this;

    var lbl = Ti.UI.createLabel({
    text: L('test')
});
win.add(lbl);

    Please make sure you close the simulator, emulator or physical device and then;

    • Menu
    • Project
    • Clean
    • Select project
    • Confirm

    This should display the correct language without error and different on each supported language you have set your phone to.

    This is the basic procedure.

    • You are welcome.

      App name for iOS is very easy, create another file in the same location and call it app.xml, then paste this inside and save the file.

      <?xml version="1.0" encoding="UTF-8"?>
<resources>
    <string name="appname">Some Great Name</string>
</resources>

      Clean the app as before and there you go, icon name is now changed.

      The Android process is a little more complicated (not much though) so best to leave this to the step-by-step in the documentation;
