Local push notification - passing parameters

Hi,

I'm trying to set a local push notification to a specific date&time in the future:

My app.js file has a date&time picker. The idea is to let the user pick a date and time and set a local push notification to that date.

Here's the code in my main app.js file:

var value = new Date();
var timePicker = Ti.UI.createPicker({
  top:180,
  useSpinner:true,
  type:Ti.UI.PICKER_TYPE_TIME,
  transform:transformPicker,
  value:value
});

var service = Ti.App.iOS.registerBackgroundService({
  url: 'localpush.js'
});
~~

And the code in the localpush.js file:

Titanium.App.iOS.scheduleLocalNotification({
alertBody:"Are you sure ?",
alertAction:"Launch!",
userInfo:{"Hello":"World"},
date:<——— this is where I'm stuck
});
~~~

My question is - how can I access the date&time that were picked in app.js from the localpush.js file? I tried simply using the variable "value" that stored the date&time but it didn't work.

Thanks
Idan

    You could use a few different ways to pass the parameters to localpush.js, but I think you're doing this the wrong way to begin with.

    You don't need to run a background service (and really shouldn't unless you have a very good reason) in order to create a local push notification.

    So in your example, get rid of the registerBackgroundService call and just call Titanium.App.iOS.scheduleLocalNotification in it's place inside your app.js file.
