ImageView won't remove from Window

Hi guys,

I've got an app that will display a nice friendly image view if there is either no data connection or when there is an error in parsing the data via XML.

For iOS (iPhone) using the latest SDK.

Here is the function:

// Hide the activity indicator when network traffic complete
            xhr.onerror = function () {
                MainActInd.hide();

                //alert("Test error popup if needed");

                // Load an image into the view for a nicer experience for the user
                errorDataImage = Ti.UI.createImageView({
                    image: "images/ERROR.png",
                    width: 320,
                    height: 480
                });
                window.add(errorDataImage);

                // Action to reload data if the data wasn't parsed correctly
                errorDataImage.addEventListener('click', function (e) {

                    // reload the whole view
                    loadAll();
                    // remove the graphics from the screen
                    window.remove(errorDataImage);

                });


            }

The issue I have is that it's not actually removing the image using window.remove, the tablewview content just displays over the top. Technically I could get round this by setting the tableview background colour to white. But that seems a bit like cheating!

I then get an error in the console:

[WARN] Called remove for [object TiUIImageView] on [object TiUIWindow], but [object TiUIImageView] isn't a child or has already been removed.

Any ideas what i might be doing wrong here?

This is the only place that image is set/removed in my code.

Can post the entire JS file if needed.

Simon

  • Hello Simon,

    does the loadAll() call do something like re-adding the imageview ? Because it's really strange that neither remove() nor hide() doesn't, well, hide the imageview.

    Anyway, to avoid memory leaks I always try not to use "external" vars in events handlers.
    You could for example rewrite the window.remove(errorDataImage); line with e.source.parent.remove(e.source); and maybe remove the reference to the imageview with errorDataImage = null; &#x2F;&#x2F; or false or 42 or whatever you want.

    Hope it helps,

    Marco
