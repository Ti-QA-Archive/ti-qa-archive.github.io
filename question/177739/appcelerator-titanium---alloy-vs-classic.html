Appcelerator Titanium - Alloy vs Classic

I'm about to start a new Titanium project. So far, what I've written is what's now considered as "Classic Project" (I've been writing Titanium apps since before Alloy).

I was wondering weather or not it's worth while to switch to Alloy for my new project. I know that basically, Alloy compiles to "Classic" Titanium code before compiling down to the java project or objective-c project. I also know about a big advantage in that "pre-compilation" process, which is to mark in alloy code that are relevant only for Android or iPhone and not including the other in the compiled "Classic" code.

My main concerns are flexibility - as with "Classic" I feel like I have more control over the code and what's going on.

Second concern, I have controls (like open sourced sliding menu) that are not written for alloy - how will I be able to have them as part of my project?

I started asking myself rather to use Alloy or not after finding (introduced in TiConf 2014) TI UX (https://github.com/jaraen/Ti.UX.Templates) which is written for Alloy.

few other questions:

how can I have old UI code run inside Alloy?
Can I have controls written for Alloy run in a "Classic" project?
Can "Classic" project also map out code not belonging to iOS/Android?
is there a difference in performance between Alloy projects and Classic projects? what about other flexibility issues?
Thanks

  • Tim has you sorted but I want to clear up a misunderstanding that persists; Alloy and "Classic" are not two competing things;

    • Titanium App without the Alloy Framework (classic is the common phrase but is wrong)

      This is Titanium

    • Titanium App with the Alloy Framework

      This is also Titanium

      However you gain all the things Tim will have said, plus control over things you never had before.

    There is a small learning curve to get used to the extra abilities of Alloy, plus how to best manage a project that requires structure (my previous projects were in different states of organisation). The time this take to understand is much less than the time you will save on the project. THe second project will be even faster.

    The only real way to see how much better you and Titanium can be by using Alloy is to actually start using it. There is no way I would start a new project without the Alloy Framework.

  • I posted an answer last week, possibly the week before on this same topic. Please do a search to find the reasons I gave.

    Maybe a few I didn't address specifically:

    • how can I have old UI code run inside Alloy? You put it in a controller (JS file)
    • Can I have controls written for Alloy run in a "Classic" project? Yes
    • Can "Classic" project also map out code not belonging to iOS/Android? Probably not as efficiently as Alloy. Alloy pre-compiles your app, removing all code and assets not used for the target platform. None of the if/then branching options in the Classic SDK offer that. So, you'd have to come up with your own techniques.
    • is there a difference in performance between Alloy projects and Classic projects? Yes, Alloy apps are typically faster
