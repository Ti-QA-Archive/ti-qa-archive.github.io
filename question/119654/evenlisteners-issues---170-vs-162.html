evenListeners issues - 1.7.0 vs 1.6.2

I\\'ve been having problems accessing the .listeners property in 1.7.0-continuous. It doesn\\'t seem to work as expected (as in 1.6.2).

In previous versions listeners property allowed the possibility of accessing all of the listeners attached to a view by type, but it is no longer the case.

It was very useful property, since it allowed operations like removing all events of a certain type, without having to have to listener reference on hand (useful if you use anonymous functions for listeners).

for example :

var openers=myView.listeners && myView.listeners.open || [];
openers.forEach(function(f){
    myView.removeEventListener(\\\'open\\\',f);
});

In 1.6.2, myView.listeners would give you an object with key/value pairs equating to the listener type/array of callback functions bound as Listeners to the view.

In 1.7.0, myView.listeners gives you an object, however the value is now an integer, which seems to be equal to the length of what the callback array would be.

Ex: create the view below:

var myView=Ti.UI.createView();
myView.addEventListner(\\\'eventName\\\', function(){/* eventListener*/});

1.6.2 : Ti.API.info(myView.listeners) -> 

{ \\\'eventName\\\' : [<KrollCallback: 0x#####>] }

1.7.0 : Ti.API.info(myView.listeners) ->

{ \\\'eventName\\\' : 1 }

Has anyone successfully tried something along these lines in 1.7.0? It\\'d be a shame to loose this functionality… but with all of the memory issues of 1.6.2, i\\'m going to have to switch to 1.7.0 once it\\'s officially released.

— asked May 13th 2011 by Russell Munson
