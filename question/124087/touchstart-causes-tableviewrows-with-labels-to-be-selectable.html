'touchstart' causes TableViewRows with labels to be selectable?

I believe this is a bug (Titanium SDK 1.7.2). When I assign a 'touchstart' listener to a Window, TableViewRows containing Labels unexpectedly become selectable, despite having given allowsSelection:false in createTableView(). Here is example code:

var win = Titanium.UI.createWindow();

win.addEventListener('touchstart', function(){});

var row = Ti.UI.createTableViewRow({ height: 40 });
var label = Ti.UI.createLabel({ text:'Table Row Text' });
row.add(label);

var table = Ti.UI.createTableView({
    data:[row],
    height: 40,
    top:0,
    allowsSelection:false
});

win.add(table);

win.open();

When I run this in the iPhone Simulator the table row, the row is selectable (and can't be unselected, either.) When I remove the addEventListener(), the table row is not selectable. Also, if I use TableViewRow.title instead of a child Label, the table row is not selectable.
