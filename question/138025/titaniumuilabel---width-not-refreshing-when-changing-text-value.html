Titanium.UI.Label - width not refreshing when changing text value

I'm creating a new label as follows:

var dueDateLabel;
dueDateLabel = Titanium.UI.createLabel({
  height: 40,
  top: 0,
  right: 5,
  font: {
    fontSize: 15,
    fontFamily: 'Helvetica',
    fontWeight: 'normal'
  },
  color: '#0047B3',
  width: 'auto',
  text: 'None'
});

This is then added to a row in a Titanium.UI.TableViewSection.

Later, when the user selects a date, I update the value of the label text as follows:

dueDateLabel.setText(this.getDueDateLabel(this.dueDate));

The problem is, on an iPhone (physical device), the width of the label does not change, so the date is not properly visible. If I touch the row containing the label, it then repaints and has the correct width (i.e. increased to hold the date). Note that this problem is not present on the iPhone simulator.

Is there any way around this?

Thanks

    Rather than making the width an 'auto' value that will have to resize, set it to fill the remaining width of the row, that way it will always be as wide as possible for any value that you may need to change it to and it won't have to resize.
