Studio & Simulator Not Catching Simple Parse Errors. (works on simulator, not on device)

Hi All,

I've been working on an update of my app. Once I had made the changes and tested thoroughly in the iPhone simulator, i deployed to device (which on separate note always requires manual work iTunes).

The app failed immediately, when launching on device. Throwing a parse error.

Looking at location and the lines of code around it I found nothing.

I cleaned the build, tried again. still no love. I cleaned, opened in Xcode, cleaned, built for device, still no love.

Deleted build directory, ran on device from studio, still same error. Again tried with Xcode, again.

I reviewed all 2756 lines of code the file reported with parse error.

I found three missing ; and two strings broken across lines with a joining +.

Neither of these cause Studio to report an error or caused any errors in the simulator.

Once I corrected these, app ran beautifully on device.

Nick
