CommonJS Module on Android Fails.

Hy all,

I am trying to use a CommonJS Module with Titanium. I started to test the technology by creating a file MyModule.js (under the folder modules).

MyModule.js:

exports.sayHello = function(str){
    alert("Hello "+str);
}

app.js :

var my = require('modules/MyModule');
my.sayHello('World');

When I use the iPhone simulator, everything works fine. The problem is with the Android Emulator (2.3.3), It throws the following error :

E/TiJSError( 1425): (kroll$1: app://app.js) [4,1108] - In file:///android_asset/Resources/app.js:11,0
E/TiJSError( 1425): (kroll$1: app://app.js) [4,1112] - Message: Wrapped java.lang.IllegalArgumentException: size < 0 (file:///android_asset/Resources/app.js#11)

Can someone help me with this please?

  • I'm seeing the same issue using sdk 1.7.2. There are a few open JIRA tickets regarding Android and CommonJS here and here.

    As a workaround trying a 1.8x continuous integration build. That worked for me, but note other issues described in the jira tickets.

    Also, I think you are missing a semicolon at the end of your exports: 

    exports.sayHello = function(str){
    alert("Hello "+str);
};
