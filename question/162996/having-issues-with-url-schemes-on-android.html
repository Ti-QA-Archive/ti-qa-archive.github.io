Having issues with URL Schemes on Android

I setup a URL scheme for my app no problem for iOS. It was a breeze.

Android is a different story, as it always seems to be.

So I have a url like so:

<a href="vforummobile://foo=bar&hello=world">URL scheme test link</a>

I click that in my browser on my S4 and chrome comes back saying page not found. I've looked at numerous posts on here and stackoverflow and I still cannot get a link to launch my app using android. Below is my tiapp.xml file (the android portion).

<android xmlns:android="http://schemas.android.com/apk/res/android">
    <tool-api-level>11</tool-api-level>
    <manifest android:versionCode="9" android:versionName="1.5.0">
        <uses-sdk android:targetSdkVersion="11"/>
        <supports-screens android:anyDensity="false"
            android:largeScreens="true" android:normalScreens="true"
            android:smallScreens="true" android:xlargeScreens="true"/>
        <application android:theme="@style/HoloLight"/>
        <activity android:name=".VforumMobileActivity">
            <intent-filter>
                <action android:name="android.intent.action.MAIN" />
                <category android:name="android.intent.category.LAUNCHER" />
            </intent-filter>    
            <intent-filter>
                <data android:host="dopserv1.dop.com" android:scheme="vforummobile"/>
                <action android:name="android.intent.action.VIEW"/>
                <category android:name="android.intent.category.DEFAULT"/>
                <category android:name="android.intent.category.BROWSABLE"/>
            </intent-filter>
        </activity>
    </manifest>
</android>

I am not using any Activities in my app so getting the

<activity android:name=".VforumMobileActivity">

part of the XML was weird. I found it by opening the AndroidManifest.xml file in the resources&#x2F;build&#x2F;android directory and placed it inside my tiapp.xml file.

I am using TiSDK 3.2.1 and testing on my galaxy S4. Any idea what I am doing wrong here?
