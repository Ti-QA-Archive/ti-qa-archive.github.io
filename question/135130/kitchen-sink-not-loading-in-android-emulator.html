Kitchen Sink not loading in Android Emulator

I just downloaded Titanium Studio (build 1.0.9.20120214208) on a Mac running Mac OS X 10.5.8, and the installation seems to be good. I have the Android SDK installed, and Titanium Studio reports that it is installed correctly.

When I try to run Kitchen Sink, the project compiles, the Android Emulator launches, and there is no Kitchen Sink app! I can navigate in the Emulator, and check the directory of apps, run other apps, etc. but Kitchen Sink is not there.

I have tried clean a rebuild, reloading Titanium Studio, and restarting the computer, with no difference in the results.

Any idea what I need to do to get it to load?

  • Are there any errors in the build.log file?

    Typically when this happens I check the following:

    1.Am I using a mapview? If so you need to make sure you have your emulator set to a Google APIs emulator version.

    2.Try cleaning your project, then add an empty line to the bottom of your tiapp.xml file

    3.Sometimes the emulator gets stuck on load and you need to press the "run" or "debug" buttons again.

    4.If none of these fix it, I try to see if there is a difference between run and debug
