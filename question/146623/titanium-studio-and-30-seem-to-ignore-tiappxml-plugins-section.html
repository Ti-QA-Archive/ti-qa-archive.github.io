Titanium Studio and 3.0 seem to ignore tiapp.xml plugins section

I currently use a few plugins to handle my build process and they are ignored in titanium 3.0 both from the command line and Studio. Alloy Run (which I guess uses builder.py) still respects them.

Is there a new way to have pre/post compile plugins in Titanium 3.0?

  • Accepted Answer

    Titanium 3.0 only supports legacy Python-based plugins for Android builds only. iOS and MobileWeb do NOT support Python-based plugins. You will need to port them to the new CLI plugin architecture.

    CLI plugins are now written in JavaScript and consist of one or more "hooks" and/or "commands". Unfortunately, CLI plugins are not documented.

    For an example of a CLI plugin hook, take a look at how Alloy works.
