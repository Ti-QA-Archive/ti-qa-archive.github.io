  • Hi

    A common technique in all development platforms for mobile and desktop is to load a single image that has all of the actual images drawn within it. This does require a fixed aspect image but has significant benefits over your existing plan of loading each image separately.

    • one single file, or at least a small number of merged tile page images
    • one image, so less resources
    • one I age so a single event listener
    • less overall memory
    • significantly faster

    You add a single event listener to determine the x/y of the touch and respond accordingly. Just place all the icons into an I age by using an image editor.

    This is how the pro developers handle your problem.
