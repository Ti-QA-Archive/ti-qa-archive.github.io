Remove ActionBar from transparent windows

Hi,

I would like to completely remove the ActionBar from all the windows, I succeeded doing this by creating a theme like this:

<?xml version="1.0" encoding="utf-8"?>
<resources>
    <style name="Theme.MyApp" parent="@style/Theme.Titanium">
        <item name="android:windowNoTitle">true</item>
        <item name="android:windowContentOverlay">@null</item>
        <item name="android:windowIsTranslucent">true</item>
        <item name="android:windowActionBar">false</item>
    </style>
</resources>

But still I get the ActionBar when my window have a transparent or semi transparent backgroundColor.

Any ideas on how to also remove it from this kind of windows?

  • You need use this:

    styles.xml

    <?xml version="1.0" encoding="utf-8"?>
    <resources>
    <style name="Theme.MyTheme" parent="@style/_MyTheme"/>
    <style name="_MyTheme" parent="Theme.AppCompat.Light.NoActionBar">
    </resources>

    AndroidManifest.xml

    android:theme="@style/Theme.MyTheme"

  • Hi Smits,

    Thanks for your concern.

    Till Ti SDK 3.2.3.GA you can remove ActionBar by setting <navbar-hidden>true</navbar-hidden> in tiapp.xml and setting the window property navBarHidden:true

    Since Release 3.3.0, due to changes to support the appcompat library, navBarHidden property has no effect. By default, the action bar is always displayed.

    For more, check the link

    There are two ways to hide the ActionBar in Ti SDK 3.3.0. You can either hide the activity's Action Bar in your JavaScript code or modify the default theme to hide the Action Bar.

    In your JavaScript code, get the activity's action bar instance and call the hide method. For example:

    // Wait for the Window (or TabGroup) to render
win.addEventListener('open', function(){
    // Grab the window's action bar instance and call the hide method
    win.activity.actionBar.hide();
});

    To modify the theme to hide the action bar:

    1. Add a custom theme file to your project:

    platform/android/res/values/custom_theme.xml

    <?xml version="1.0" encoding="utf-8"?>
<resources>
    <style name="Theme.NoActionBar" parent="@style/Theme.Titanium">
        <!-- Depending on the parent theme, this may be called android:windowActionBar instead of windowActionBar -->
        <item name="windowActionBar">false</item>
    </style>
</resources>
    1. Modify your tiapp.xml file to use the modified theme:
     <android xmlns:android="http://schemas.android.com/apk/res/android">
    <manifest>
        <application android:theme="@style/Theme.NoActionBar"/>
        <!-- Need to specify at least API level 11 for Titanium SDK 3.2.x and prior -->
        <uses-sdk android:minSdkVersion="14" android:targetSdkVersion="19"/>
    </manifest>
</android>

    Please check the link of Android ActionBar

    Ti SDK 3.2.3.GA and 3.3.0.RC

    Android version 4.4.3

    Appcelerator Studio, build: 3.2.3.201404181520

    Mac OS X 10.9

    • solution was posted to a jira ticket:

      upgrade to 3.4.0.v20140917120000,

      add a style:

      <style name="Theme.NoActionBarTransparent" parent="@style/Theme.AppCompat.Translucent.NoTitleBar">
    <item name="android:windowActionBar">false</item>
    <item name="android:windowNoTitle">true</item>
    <item name="android:windowActionBarOverlay">false</item>
</style>

      and when you open the window use: window.open({theme:"Theme.NoActionBarTransparent"})

      see: http://docs.appcelerator.com/titanium/3.0/#!/guide/Android_Themes-section-34636181_AndroidThemes-TitaniumThemes
