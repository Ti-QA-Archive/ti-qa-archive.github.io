Very annoying loop crash

I'm finding a few problems with very quick loops. I had to scrap an XML parsing script because it randomly crashed the app in the simulator, and now I'm having problems even with simply checking if a file exists. The code is very simple:

var local_images_array = [];
for(var x = 0; x < images_array.length; x++){
    var imagename = images_array[x]; //Let's take an image name frmo the general array
    if(imagename){
        var file = Titanium.Filesystem.getFile(Titanium.Filesystem.applicationDataDirectory, imagename+'.png'); //Checking if file exists
        if(file.exists()) { 
        Ti.API.info('file exists'); 
        } else { //It doesn't, let's add it to the local images array
        Ti.API.info(imagename+'.png is missing'); 
        local_images_array.push(imagename+'.png');
        }
    }
}

This simple loop that checks for roughly 3,000 images is crashing randomly, sometimes after 200 iterations, sometimes 600, sometimes 1800, etc. I went around it for now but I really need to understand why such somple loops crash the app without even outputting a single line of error in the debugger. I'm using Titanium Studio 1.7.1 on iOs.

Has anybody already dealt with this kind of problem? Thanks in advance.
