  • Hi

    Here you go, this should get you going.

    Event listener

    var win = Ti.UI.currentWindow;
var searchBar = Titanium.UI.createSearchBar({
    borderStyle: Ti.UI.INPUT_BORDERSTYLE_ROUNDED,
    hintText : "Case ID",
    top:110,
    height:60,
    width:450,
    showCancel:false
});
searchBar.addEventListener('return', function (e) {
    // fires on the return/search key being pressed on the keyboard
    Ti.API.info(e.value);
});
searchBar.addEventListener('change', function (e) {
    // fires on any key being pressed on the keyboard
    Ti.API.info(e.value);
});
win.add(searchBar);

    • Working with the database - you have two ways to handle this;

      • Install an existing database
      var db = Ti.Database.install('filenameofdatabase', 'nametorefertodb');
db.close();

      This will take a local copy of your database filenameofdatabase and install it into the devices local storage (location dependant on platform) and then set a name to refer to the database later nametorefertodb.

      Then later on you refer to the database like;

      var db = Ti.Database.open('nametorefertodb');
var rows = db.execute('SELECT fieldname FROM tablename;');
while (rows.isValidRow()) {
  Ti.API.info('field: ' + rows.fieldByName('fieldname');
  rows.next();
}
rows.close();
      • Create the database locally and create the tables locally;
      var db = Ti.Database.install('mydb1', 'mydb1Installed');
db.execute('DELETE FROM people');
db.execute('CREATE TABLE IF NOT EXISTS people (name TEXT, phone_number TEXT, city TEXT)');

var thisName = 'Arthur';
var thisPhoneNo = '1-617-000-0000';
var thisCity = 'Mountain View';
db.execute('INSERT INTO people (name, phone_number, city) VALUES (?, ?, ?)', thisName, thisPhoneNo, thisCity);

var personArray = ['Paul','020 7000 0000', 'London'];
db.execute('INSERT INTO people (name, phone_number, city) VALUES (?, ?, ?)', personArray);

var rows = db.execute('SELECT rowid,name,phone_number,city FROM people');
db.close();

while (rows.isValidRow())
{
  Ti.API.info('Person ---> ROWID: ' + rows.fieldByName('rowid') + ', name:' + rows.field(1) + ', phone_number: ' + rows.fieldByName('phone_number') + ', city: ' + rows.field(3));
  rows.next();
}
rows.close();

      This is the example from the documentation; execute method.

    • You only need to install the database once. From then on you only use the open method.

      To open another window depends on what else you have as an interface. You have not specified if you are writing for iOS or Android, there are some differences.

      If you are writing based on the code above (no tabBar and no navigationController) then;

      searchBar.addEventListener('return', function (e) {
    // fires on the return/search key being pressed on the keyboard
    Ti.API.info(e.value);
    var newWin = Ti.UI.createWindow({
        height: Ti.UI.FILL,
        title: e.value,
        url: 'somefile.js',  // your coding style makes me think you use this property
        width: Ti.UI.FILL
    });
    newWin.open();
});

      If you were opening as a child of a tab then this line;

      newWin.open();

      Becomes;

      tab1.open(newWin, { animated: true});

      or (if you are using the url property

      var tab = Ti.UI.Tab.currentTab;
tab.open(newWin, { animated: true});

    • I would recommend NOT installing in the return event listener, but instead adding a simple latch when the app first runs.

      Something like;

      if (Ti.App.Properties.getBool('AppInstalled', false) === false) {
    // install your database here ready for when needed

    // this makes sure THIS block of only runs on first install
    Ti.App.Properties.setBool('AppInstalled', true);
}

      From that point onwards you should only use open to access the database. This will gracefully manage the first install of the DB.
