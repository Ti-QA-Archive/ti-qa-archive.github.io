  • I think .text only works on a node with a text value, not child nodes. what does like xmlDoc.documentElement.getElementsByTagName(&#39;return&#39;).item(0).getElementsByTagName(&#39;Name&#39;).item(0).text give you? My guess is "Hallo".

    Remember that even using the Suds client, we're not thinking about your SOAP web service returning "Objects", just verbose XML. For examples on how to work with XML documents, check here:

    https://github.com/appcelerator/KitchenSink/blob/master/Resources/examples/xml_dom.js

    SOAP is just a bulky wrapper around XML over HTTP, so it is more helpful to think about the SOAP envelope you get back as an XML document you are working with (at least in this case - a more sophisticated client might parse a WSDL and serialize XML into objects, etc.).
