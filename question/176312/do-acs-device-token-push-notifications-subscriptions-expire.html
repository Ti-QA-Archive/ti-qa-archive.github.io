Do ACS Device Token Push Notifications subscriptions expire?

I am developing an alloy Android app using Titanium Studio, build: 3.3.0.201407100905. The app is using device token GCM push notifications with ACS.

Once a device has successfully subscribed to a notification channel, will the subscription automatically expire at some point in time, and need to be renewed? Should the device re-subscribe every time the app is loaded, perhaps via a call in alloy.js, or is this unnecessary?

Thanks for your help.

    Couple of ways to think about this. (we re-register regularly, but the question is 'when')

    1. Is the push tied to a specific user? Meaning, do you have user accounts in your app? If so, then you reregister and add user id, or some other var so you can send a push JUST to the device that user is on.

    2. If your app is not user specific, then reregister on start.

    While some services allow you to simply update a var associated with a user, this approach works across different push providers.
