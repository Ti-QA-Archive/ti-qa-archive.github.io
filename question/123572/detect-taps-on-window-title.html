  • I tried doing this recently in an iOS app of mine, and experimented trying to use a label or other object types to hold the title and setting the object using win.setTitleControl. Although undocumented in the Window module, I found that method used around line 115 of the kitchen sink tabbedbar.js example.

    Alas, I could not get it to work with 100% consistency in detecting taps, even playing with various z-index values for the title control vs the left and right nav buttons (when present). So I ended up moving to plan B and a different UI paradigm.

    I still think using setTitleControl is probably one of the keys to making this work. I just didn't have time to work out the details. Maybe this information will help you out though.

  • instead of setting the title property for the window, please use the titleControl property and add the Label and set the text property for window's title.

    Then add the event Listener for label control which was added to the window.

    var titleLabel = Titanium.UI.createLabel({
    text : "window's title",
    font:{
    fontSize : 14,
    fontWeight : "Bold"
    },
    color: "white"
    })

    var win=Ti.UI.createWindow({
    // title:'Window Title', < ——– Comment out this line
    backgroundColor:'#fff',
    url:'lessonreport.js',
    titleControl : titleLabel
    });

            win.open();
