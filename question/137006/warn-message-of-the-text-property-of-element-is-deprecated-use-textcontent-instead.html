  • Accepted Answer

    It is quite simple. In a future version of the sdk, the text property of that particular object type will no longer be valid and has been replaced with textContent instead. I'm guessing you are parsing XML.

    The solution is simple: change the text property to textContent and the errors will go away.

    I would say that the error could be a bit more helpful, possibly actually telling you what kind of object is generating the error.
