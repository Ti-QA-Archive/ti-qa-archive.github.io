Sliding window open animation not working properly

Creating an iOS app in which press a button open a window by sliding it to the right. However, if I go from portrait mode to landscape or vice-versa; and then I if I press the button, the window slides in from top left for the first time instead of sliding form the left.

How can I fix this ?

Code : 

 var win = Ti.UI.createWindow({left:0});

 var slideLeft = Titanium.UI.createAnimation();
   slideLeft.left = 0; 
   slideLeft.duration = 200;

  var slide_it_right = Titanium.UI.createAnimation();
    slide_it_right.left = -320; 
    slide_it_right.duration = 300;

  button.addEventListener('click',function(){
              win.open(slideLeft);
  });
  win.addEventListener('swipe',function(){
          win.close(slideRight);
  });

  • Hi

    I do this all the time, I use a slightly different approach.

    • Create the window but set it to appear outside the main screen space - -320 left for your example
    • Open the window with NO animation
    • Then animate the window itself

    Syntax;

    win.animate(slideLeft);

    • If you are still using the code above the a likely issue is that you are not defining the width and height of the window to the physical width and height of the device. Do not use Ti.UI.FILL for either width or height as this will not work as you expect.

      — commented August 9th 2013 by Malcolm Hollingsworth

    • @malcolm - thanks for your comment. I was having a similar problem - I could animate views in from the left, but not from the right. Adding the width and height to the view before adding it solved the problem!

  • function open(e) {
    $.winID.animate({
        right : 150,
        duration : 400,
        curve : Ti.UI.ANIMATION_CURVE_EASE_IN_OUT
    });
}
