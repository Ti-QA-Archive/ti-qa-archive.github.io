    Nick is actually correct on this. The difference between the 2 validators is the way that they handle unrecognized text. If you view the json document in a browser like FireFox, even the browser will tell you that the json may not be well formed.

    The problem is literally right here:

    See the larger version here

    So this actually doesn't have anything to do with the size of the file. My advice would be to find a way to properly handle unrecognized text during the json creation process so that you don't have to worry about it during the network transfer process.
