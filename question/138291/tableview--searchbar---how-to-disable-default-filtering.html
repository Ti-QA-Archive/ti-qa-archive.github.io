TableView + SearchBar - how to disable default filtering?

I'm making an iOS mobile app, and using the TableView and SearchBar together to allow people to look through their Facebook friends. That list can get very long, so I'm storing the full list in local SQL, then only loading in so many at a time. Because of this, I have an event hooked up to the 'change' event of the SearchBar that then calls setData on the TableView.

I'm also doing slightly custom filtering- it takes the text in the search box and checks both the first and last names of a person for a match. The issue is that the 'default' filtering of a SearchBar-enabled TableView is getting in my way- it only checks the first name. How can I turn off this default filtering behaviour and let my own work by itself?

  • You'll have to go all custom here…

    I'm assuming the searchBar is not linked to the table view i.e. it's an independent view.

    Method 1:
    Firstly, keep a record of which name is at which index in an array. A key:name pair would work.

    On the change listener for the searchBar, filter through the aforementioned array, and display only the results. Since your searchBar is not linked to the tableView, you can write a quick function that sorts through the array. Gather the result indexes and show those rows of the tableView.

    Method 2:
    For the change listener, sort through your database. Get the results from out, create a dictionary consisting of UITableViewRows and append the dictionary to the tableView using setData();
