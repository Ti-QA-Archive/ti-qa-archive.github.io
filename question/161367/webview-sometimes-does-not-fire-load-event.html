  • Had a similar problem with Twitter authorization. If the webview url does not change (e.g. due to an ajax call), the "load" event does not fire again.

    There is a workaround, if you are brave enough to edit the mobilesdk source code in TiUIWebView.m

    Search for this code (line 717):

    if ([self.proxy _hasListeners:@"load"]) {
    if (![urlAbs isEqualToString:lastValidLoad]) {
        NSDictionary *event = url == nil ? nil : [NSDictionary dictionaryWithObject:[self url] forKey:@"url"];
        [self.proxy fireEvent:@"load" withObject:event];
        [lastValidLoad release];
        lastValidLoad = [urlAbs retain];
    }
}

    Now uncomment the second if-statement and "load" event should fire every time on page change.

    if ([self.proxy _hasListeners:@"load"]) {
    // if (![urlAbs isEqualToString:lastValidLoad]) {
        NSDictionary *event = url == nil ? nil : [NSDictionary dictionaryWithObject:[self url] forKey:@"url"];
        [self.proxy fireEvent:@"load" withObject:event];
        [lastValidLoad release];
        lastValidLoad = [urlAbs retain];
    // }
}
