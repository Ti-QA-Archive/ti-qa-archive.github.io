Titanium SDK version Vs. iOS used devices

I have to build an iOS device compatible with most used iOS devices, according the graph of this Apple page most used iOS version are 7 & 8. The problem I have is that according the Titanium compatibility matrix if I use the Titanium SDK 3.4.0 my app won't work on devices with iOS less than 7.1.0, and if I use the Titanium SDK 3.3.0 the app won't work on 7.1.0 and upper… Am I understanding things right?

I have doubts about which Titanium SDK version should I use :(

Any help would be appreciated…

Thank you!

  • Accepted Answer

    If you are writing a new app, then you have no choice. Apple is requiring all new apps come with 64bit support, and only Ti 3.5.0 and newer provide that.

    If you are updating an existing app, Apple's 64bit deadline is like June 1… so you're still going to need to migrate to 3.5+ shortly.

    That said, you have made an assumption that you need to target 7.0 devices. Apple's chart says "7.x". Looking at the stats I have for an app I've built, only 0.88% of all users have used something older than 7.1 in the past few months. You should be plenty safe just targeting 7.1 and newer. There are 0 devices out there that support 7.0 but not 7.1. Those users should update their devices.
