orientationModes bug / issue in "more windows" on iOS

I've developed my app just like the tweetanium app (namespaces). I am displaying some tableviews behind each tab, and when you click on a row a new window is created with the content (and a video).

In the new window I set orientationModes :


tableView.addEventListener('click', function(e)
{
    app.ui.createMediaView({
        data: somedata
    });    
}

(function() 
{
    app.ui.createMediaView = function(_arg) {

        var win = Titanium.UI.createWindow( {
            backgroundColor: '#fff'
        });

        win.orientationModes = [
            Titanium.UI.PORTRAIT,
            Titanium.UI.LANDSCAPE_LEFT,
            Titanium.UI.LANDSCAPE_RIGHT
        ];

        // get some rowdata,add player to window etc..
        // ...

        // open window
        currentTab.open(win,{animated:true});

    }
})();

Problem: this works fine on all tabs EXCEPT when I access an tableview from the "more tab" in iOS. So, when an window is opened from within the more tab (auto generated by iOS), the orientation change isn't detected/active. Every window jumps to PORTRAIT.
