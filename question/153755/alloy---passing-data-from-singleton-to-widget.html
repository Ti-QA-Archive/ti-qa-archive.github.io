Alloy - Passing data from singleton to widget

Hello,

I'm trying to implement a widget so I can reuse my code.
The widget needs a string as an argument. I am trying to databind that string but it does not work.

<Alloy>
     <Model src="MyModel" id ="MyModelId" instance="true">
     <Window>
          <Require type="widget" src="com.mycompany.mywidget" id="MyWidget" customStatic="test" customBound="{$.MyModelId.Name}" />
     </Window>
</Alloy>

Then in my widget js file:

alert(arguments[0]);

In the alert box I can see that there is an argument "customStatic" but no argument customBound.
Can anyone help?
Tks.

ps: I'm using an alert because somehow, breakpoints are not hit in the widget js file…

    That's not exactly how the <Require> tag works. The properties you list as its attributes will only be processed once, when it is created, regardless of any binding you attempt. If you want to do binding, you need to do it within the widget itself. If you need to pass a dynamic value to the widget when it is created, you should do that programmatically, or give the widget an initialization function of some kind that can be called later to give it the necessary value.
