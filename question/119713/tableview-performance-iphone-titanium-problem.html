  • TableView speed is fine, but adding 500 rows at once does take some time yea.

    Why dont you add the first 50 rows and then add rows to the tableview when its allready loaded.

    use this method to add rows on an ecisting tableview.

    // index is the number it should be added before, 
// so if you set this at 0 every row will be inserted before the first one
// Nice way to test it out at first
tableview.insertBefore( insertingRow, index );

// OR

// index is mostly the last index in the tableview, 
// something what you should use
tableview.insertAfter( insertingRow, index);

    i am using the second method to add rows to an excisting tableview, initially it loads data which is important to show at first, then let a function load the rest along the way.
