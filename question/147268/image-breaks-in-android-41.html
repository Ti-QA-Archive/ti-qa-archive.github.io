Image Breaks in Android 4.1

I just installed Titanium 3 and all the latest Android SDKs. First think I did was test my app with Android v4.1. I noticed my imageview was not showing my logo, but the background image worked. Then I re-targeted for Android 2.3 and the logo was there again. Is there special syntax to make the image view work with Android v4.1?

Here is my code:

/**
 * Logo
 */
var image = Titanium.UI.createImageView({
    url:'/images/logo.png',
    height : 'auto',
    width : 'auto',
    top: 20
}); 
scrollview.add(image);

    Hi

    Easy one this. You have found an old example of how to add an image, as the url property has been deprecated you are seeing inconsistent results.

    Change your code as follows and try both Android version again - it will work with both.

    /**
 * Logo
 */
var image = Titanium.UI.createImageView({
    Image:'/images/logo.png', // correct property
    height : 'auto',
    width : 'auto',
    top: 20
}); 
scrollview.add(image);

    Let me know how it goes.
