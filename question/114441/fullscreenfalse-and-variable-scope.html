fullscreen:false and variable scope

Hi all,

I'm trying to write a bit of code with a variable shared between a master file and an included child (as below). Here's my problem; when the initial window has a fullscreen value set, the included file can't see the 'm' variable (when I comment out the fullscreen line, it can see it). Any clues?

Cheers,
Mark

app.js

var m="hi";

var win1 = Titanium.UI.createWindow ({
    backgroundColor: 'blue',
    //fullscreen:false
})

var button = Titanium.UI.createButton ({
    title: 'TestMe'
})

button.addEventListener ('click', function () {
    Ti.include ("test2.js")
})

win1.add(button);
win1.open();

  • Each window is executed in it's own namespace ( sandbox ). I suppose you are on android and without a fullscreen setting ( true or false) you open a lightweight window (that is similar to a view).

    The best practice is to pass the variable to the new window in it's constructor.

    var m="hi";

var win1 = Titanium.UI.createWindow ({
    backgroundColor: 'blue',
    m:m

    //fullscreen:false
})

    test.js

    alert(Titanium.UI.currentWindow.m);
