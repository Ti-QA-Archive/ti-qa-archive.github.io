  • A possible error that can be received when launching a mobile app with Titanium SDK 1.5.1 is:

    [ERROR] Error generating R.java from manifest

    This error can be thrown by not having the correct syntax for the Application ID. The Application ID must contain at least 1 ".", ie:

    Valid names: "com.zaks.tester", "anyword.anotherword", "this.that"

    Invalid names: "Mytest", "MyFirstApp", "ThisIsBad"

  • Regarding the apkbuilder.bat error, you can safely ignore it. Read apkbuilder.bat - THIS TOOL IS DEPRECATED for more information.

    Note that "APIs 2.3" is not yet supported by Titanium. Use "APIs 2.2" instead.

    Have you tried to delete your build files and force a full rebuild?

    Caution, though; currently the process for Android and iOS is different. This is due to ticket #2852.

    Hence:

    For iOS/iphone, delete the contents of the project's build&#x2F;iphone directory, select Edit Project in Titanium Developer and click the Save Changes button to force a full rebuild. Then launch the app.

    For android, only delete the contents of the immediate subdirectories of &#x2F;build&#x2F;android, otherwise you may encounter problems. Then click the Save Changes button to force a full rebuild and launch the app, as above.

    Lastly, what processor and RAM do you have installed on your system?

    • I've seen a lot of problems with this but haven't run across a definitive answer, I think we've found it. When trying to Launch a mobile app with the Titanium SDK 1.5.1 you may receive this error:

      [ERROR] Error generating R.java from manifest

      The problem is the application id format. The application id has to have a "." (dot) in it. So when I create a new project called "mytest1" it won't work, gives the R.java error if I use SDK 1.5.1 (not with 1.2.0). If I rename the id "mytest1.asdf" then it will launch and load on the emulator.

      This isn't perfect - it will load and be available on the emulator but you will get a Force Close error.
      The best thing to do is to restart with a new name, new application id (with a "." in it) and launch - then it's OK.

      So I think the next version or next quick fix for the SDK 1.5.x should have a parser that doesn't need a "." or at least warns you that you don't have one in the app id.

      This works on the Mac and Windows. Don't have a Linux running so I couldn't test it there.

      I want the hours back I've got into this … but I hope it helps someone else.

  • I thought I'd come back and put the answer in just in case someone else runs into this problem or something similar. It turned out that it had nothing to do with my installation, what version of Windows I as running, it wasn't 32 or 64 bit or anything close to that. It was the Application ID. It seems in the Titanium Developer the ID MUST be in a format like the reverse-DNS system. (here's a wiki reference to it: http://en.wikipedia.org/wiki/Reverse-DNS)

    I wasted hours on this, it's not documented anywhere that it has to be in this format. I found one of the tutorial videos that stated "You can name it what you want, I use xxx.xxx.xx". Wrong. It will throw the manifest error if the ID isn't in that format.

    I've not tried it in the new Titanium Studio. These are the kinds of things that make the frustration level creep up with a product.

    Hope this helps someone. Be aware of problems as out of line as this.
