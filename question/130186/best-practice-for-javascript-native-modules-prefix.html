Best Practice for JavaScript Native Modules' Prefix

I wrote a JavaScript Native Module, and consider where should the module be put at my project.

If the js native module is only for iOS, it should be /modules/iphone/[id]/[ver]/assets/[name].js .
https://wiki.appcelerator.org/display/guides/iOS+Module+Development+Guide#iOSModuleDevelopmentGuide-BuildingJavaScriptNativeModules

However, My js native module is for iOS/Android.
The module have resources(icons, etc) so same problem is for resources.

put:
  /modules/[id]/[ver]/[name].js
  /modules/[id]/[ver]/Resources/icon.png

load:
  var module = require('modules/[id]/[ver]/[name]');

…

Is anyone knows the best practive about it? thanks.

  • Even though they will have the same name, you really have 2 modules, 1 for android and 1 for ios, as they are separate module projects with separate code. So when all is said and done, you should have modules/android/modulename/1.0 and modules/iphone/modulename/1.0.
