HTTPClient Call with Unique Port Fails

Hey Folks-

When a xhr call is made with a url with a unique port (say 3000 if you are using a rails dev environment), the call fails. It actually works fine on iOS and fails on Android. At least in the emulators.

Example:


var xhr = Ti.Network.createHTTPClient();

xhr.onload = function(){    
  alert(JSON.parse(this.responseText).venues);
};

var url = "http://www.gwahir.com:3000/api/venues";
xhr.open('GET', url);
xhr.send();

Any ideas on how to make it work?

Ben
