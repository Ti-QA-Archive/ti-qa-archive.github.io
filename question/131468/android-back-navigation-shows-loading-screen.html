Android "back" navigation shows loading screen

Hi guys,

I'm using the app namespace architecture as in Tweetanium, so I have app.js which loads bootstrapper.js, which loads everything.

On Android, I had to set some windows' navBarHidden to false to be able to hit the physical "back" button.

Here's the thing: I have 3 windows in my navigation group, the first one is a simple hello message, the second one is a button and the third one shows some info.

When the user is on the third screen, he hits back, it works, he sees the second screen.

When the user is on the second screen, he hits back, it DOESN'T work, he sees the loading screen and nothing.

  • In Android the Titanium window object has a property of exitOnClose. Set to true, this will close the app, when clicking the back button from that window. Have you looked into whether that is applied to your window anywhere?

    If you put this property on the window that you want to close the app when the hardware back button is pushed, that should work just fine. I have used this sucessfully in Android apps.
