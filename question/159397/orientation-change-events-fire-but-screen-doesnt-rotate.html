Orientation change events fire, but screen doesn't rotate

I've recently taken over an iPad application that was written in an older version of Appcelerator (1.7 something) I had added orientation handling, but it wasn't working well so we locked the app to portrait.

I've recently updated the project to the latest (3.1.3.GA) version and I'm trying to add back in the orientation switching. I've added the available orientations to the app.xml file and orientation change events ARE firing, as I can see top and left for screen elements changing, however the actual screen doesn't rotate, so the keyboard still comes out from the physical bottom of the iPad, and text inputs stay locked in portrait.

It seems like I'm missing something really simple here that changed from Appcelerator 1.7 to 3.0, but I'm at a lose as to what it could be.

Anyone have any ideas as to where to start looking?

  • Found it:

    Apparently this used to work, but now allows orientation handlers to fire, but doesn't rotate screen:

    <orientations device="ipad">
            <orientation>Ti.UI.PORTRAIT</orientation>
            <orientation>Ti.UI.LANDSCAPE</orientation>
</orientations>

    changed to

    <orientations device="ipad">
            <orientation>Ti.UI.PORTRAIT</orientation>
            <orientation>Ti.UI.LANDSCAPE_LEFT</orientation>
            <orientation>Ti.UI.LANDSCAPE_RIGHT</orientation>
</orientations>

    and all is well
