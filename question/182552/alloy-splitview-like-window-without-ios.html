Alloy SplitView-like window without ios?

Is it possible to create a non-iOS SplitView window in Alloy?
I would like to create a simple master-detail window for tablet sized screens, in which the user may select from a tableview on the left and load the detail view on the right, without opening a new window. Something like this.

<Window platform="!ios" formFactor="tablet">
        <Require src="master" type="view"/>
        <Require src="detail" type="view"/>
</Window>

Nearest I've seen would be a drawer menu widget that stays open, (that which I am about to try dissembling).

    So here is the complete code to achieve this task:

    index.xml

    <Alloy>
    <Window width="100%" height="100%" backgroundColor="white">
        <View id="leftView" layout="vertical">
            <View name="childView1" backgroundColor="green" onClick="setDetailPageOnRightView" class="leftViewChilds">
                <Label text="open green color view" color="white" touchEnabled="false" />
            </View>

            <View name="childView2" backgroundColor="#a0a" onClick="setDetailPageOnRightView" class="leftViewChilds">
                <Label text="open magenta color view" color="white" touchEnabled="false" />
            </View>

            <View name="childView3" backgroundColor="blue" onClick="setDetailPageOnRightView" class="leftViewChilds">
                <Label text="open blue color view" color="white" touchEnabled="false" />
            </View>

            <View name="childView4" backgroundColor="red" onClick="setDetailPageOnRightView" class="leftViewChilds">
                <Label text="open red color view" color="white" touchEnabled="false" />
            </View>
        </View>

        <View id="rightView">

        </View>
    </Window>
</Alloy>

    index.tss

    ".leftViewChilds" :{
    width : "100%",
    height : "25%"
}

"#leftView":{
    left : 0,
    width : "30%",
    height : Ti.UI.FILL,
}

"#rightView":{
    right : 0,
    width : "70%",
    height : Ti.UI.FILL,
    backgroundColor : "#aaa"
}

    index.js

    function setNewPage(){
    // store right view in temp variable before removing the childs
    var temp = Alloy.createController(''+currentPage).getView();
    $.rightView.add(temp);

    // set the opacity to 1 again to make right view visible
    $.rightView.animate({
        duration : 250,
        opacity : 1
    });

    // release the variable for clean up
    temp = null;
}

function setDetailPageOnRightView(e){
    // e.source.name property will hold the name property we had set on each left child view,
    // we are setting this property just to know the controllers name to set on right side view without
    // using if-else condition ....

    // set views name to call it in above function
    currentPage = ""+e.source.name;

    // remove previous views from right side to make app memory efficient
    try{
        $.rightView.removeAllChildren();
    }catch(e){}    

    // animate the view to make changing of right views more appealing, set opacity to fade out the view
    // after fading out the view, call back the setNewPage function
    $.rightView.animate({
        duration : 200,
        opacity : 0.5
    },setNewPage);
}

$.index.open();

    **Note down that i have not defined the variable currentPage anywhere before using it, so Javascript assumes it as a global variable for the current Javascript context, so we don't need to initialise it.

    Since it is a global variable now, so you might need to set it to null when you do not use index.js file just to release the memory.

    But i have not done this because to make SplitView work, i always need a variable to store the current page name, so it is upto you how you manage this variable**

  • Hi K.E, as there is no such iOS like SplitView for Android, i always create a custom one for myself.

    You could achieve it like this:

    • Create a window taking layout as horizontal
    • Set 2 child views, & set their width according to your requirement with height = 100% for both
    • Left child would work like master view
    • Right child would be like detail view then.
    • Create tableview in left child, & set the right view's content according to the selected option through left view using.

    I know it is quite time taking but you will get used to after creating your own designs.
