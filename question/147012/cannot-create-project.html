    • Ha, wow. Epic Studio fail.

      OK, so we need a little bit of info.

      What version of Titanium Studio are you running?

      Are you running Windows Vista or XP or older? Titanium 3 only support Windows 7 or newer. It may work on older OSes, but it's not supported.

      From the command line, find out the version of node.js installed. It should be 0.8 or newer. 0.6 won't cut it.

      node -v

      Next run the following:

      titanium status

      This will tell us 3 important pieces of information. The first line says what version of the CLI you're using. It should be 3.0.22. Directly after the CLI version is the active Titanium SDK version which is probably 3.0.0.GA. If it does not output the Titanium SDK version, then see below. Lastly the status command indicates if you're logged into the Appcelerator Network. You must be logged in to create projects or build.

      If you're not logged in, then run:

      titanium login

      If it did not show the Titanium SDK version, then you need to select an SDK. Start by running:

      titanium sdk

      This will print out all SDKs that are installed. To activate a particular SDK, run:

      titanium sdk select 3.0.0.GA

      You can only select 3.0 or newer SDKs. Don't worry if you still have 2.x projects… the 3.0 SDKs will detect your project is a 2.x app from the tiapp.xml and automatically switch to the old 2.x compiler.

    • I reinstalled Titanium Studio v3.0.1.201212181159 on my Windows 7 VM. I logged into Studio and proceeded to create a project. What's interesting is Studio only looks in your home directory's AppData\Roaming\Titanium directory for installed SDKs. I had to go into Studio's preferences and change this to "C:\ProgramData\Titanium" for Studio to see 3.0.0.GA in the list of installed SDKs. After I set that up, I was able to create and build a project just fine.

      Verify that you have a "C:\ProgramData\Titanium" directory on your computer and that it contains "mobilesdk\win32\3.0.0.GA", then set the Titanium Home SDK in the preferences to "C:\ProgramData\Titanium" and see if that fixes it.

  • We could turn on logging to see the exact command Studio is calling to create the project and check if running the same command from the command line works:

    1. Open Preferences > Titanium Studio > Troubleshooting page, change Debug Level to All, and make the entry com.aptana.core/debug/core checked;
    2. Hit OK and restart Studio;
    3. Try to create the project again to generate the error;
    4. Open the Studio log file via Help > Titanium Studio > View Log File;
    5. Look toward the end of the log to find the command to create the project; it should be something like "node <titanium_path> –no-banner … create …". Then copy-and-paste the command into the command line to see what output it generates.

    Hope this helps.
