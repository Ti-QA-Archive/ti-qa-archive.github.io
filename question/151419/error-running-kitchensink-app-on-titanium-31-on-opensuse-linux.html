    My logs are showing the same errors than yours:

    console:

    sh: mc: line 1: syntax error: unexpected end of file
sh: error importing function definition for `mc'

    build.log:

    [ERROR] Exception occured while building Android project:
[ERROR] Traceback (most recent call last):
[ERROR]   File "/home/alfredozn/.titanium/mobilesdk/linux/3.1.0.GA/android/builder.py", line 2518, in <module>
[ERROR]     builder.run_emulator(avd_id, avd_skin, avd_name, avd_abi, add_args)
[ERROR]   File "/home/alfredozn/.titanium/mobilesdk/linux/3.1.0.GA/android/builder.py", line 621, in run_emulator
[ERROR]     avd_name = self.create_avd(avd_id, avd_skin, avd_abi)
[ERROR]   File "/home/alfredozn/.titanium/mobilesdk/linux/3.1.0.GA/android/builder.py", line 540, in create_avd
[ERROR]     available_avds = avd.get_avds(self.sdk)
[ERROR]   File "/home/alfredozn/.titanium/mobilesdk/linux/3.1.0.GA/android/avd.py", line 23, in get_avds
[ERROR]     for line in run.run([sdk.get_android(),'list','target'],debug=False).split("\n"):

    Debugging the python files I found that the file <myhome>/.titanium/mobilesdk/linux/3.1.0.GA/android/avd.py is calling the command android list target to get the available emulators, but the command is returning in standard error the error about sh: mc: line 1: syntax error: unexpected end of file so the python module that executes the command is returning NONE for the android emulators list regardless that the command also returns the correct output in standard output.

    So in order to ignore the error, I changed the file <myhome>/.titanium/mobilesdk/linux/3.1.0.GA/android/avd.py line number 23 from:

    for line in run.run([sdk.get_android(),'list','target'],debug=False).split("\n"):

    to:

    for line in run.run([sdk.get_android(),'list','target'],debug=False,ignore_error=True).split("\n"):

    and is now working, but this is not the definitive solution, I think that the error is associated with the shell that is executing the python commands "sh" as you can see in the error message sh: mc: line 1: syntax error: unexpected end of file but I don't know how to solve it.

    Hope this help you a little.
