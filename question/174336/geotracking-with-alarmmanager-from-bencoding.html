Geotracking with AlarmManager from BenCoding

Hi,

I follwed the instruction/example of Ben Bahrenburgs Alarmmanager.

In app.js I'm starting the manager:

var alarmModule = require('bencoding.alarmmanager');
var alarmManager = alarmModule.createAlarmManager();
alarmManager.addAlarmService({
    service : 'de.appwerft.cm.Geo_serviceService',
    minute : 1, 
    interval : 300000 
});

In a geo.service.js I realize the communication with apiomat-cloud:

var service = Ti.Android.currentService;
var intent = service.getIntent();
if (Ti.Geolocation.locationServicesEnabled) {
    Ti.Geolocation.preferredProvider = Ti.Geolocation.PROVIDER_GPS;
    Ti.Geolocation.getCurrentPosition(function(e) {
        if (!e.error) {
            Ti.App.Apiomat.setPosition({
                latitude : e.coords.latitude,
                longitude : e.coords.longitude
            });
        }
    });
}

In my tiapp.xml I have added the following code into application section:

<activity android:alwaysRetainTaskState="true"
                    android:configChanges="keyboardHidden|orientation"
                    android:label="CriticalMass"
                    android:launchMode="singleTop"
                    android:name=".CriticalMassActivity" android:theme="@style/Theme.Titanium">
                    <intent-filter>
                        <action android:name="android.intent.action.MAIN"/>
                        <category android:name="android.intent.category.LAUNCHER"/>
                    </intent-filter>
</activity>

All runs fine, but I see two icons on my home screen. One is the real one and I can start and the other goes in the hell.

Any ideas, whats going wrong?

    OK, obviously the issue is that there are two launcher activities. And you can see this in the generated manifest.

    The first one (the right one) was generated by Titanium:

    <activity android:name=".CriticalmassActivity" android:label="@string/app_name" android:theme="@style/Theme.Titanium" android:configChanges="keyboardHidden|orientation|screenSize">
   <intent-filter>
    <action android:name="android.intent.action.MAIN"/>
    <category android:name="android.intent.category.LAUNCHER"/>
   </intent-filter>
</activity>

    The second one is hooked in by tiapp.xml:

    <activity android:alwaysRetainTaskState="true" android:configChanges="keyboardHidden|orientation|screenSize" android:label="CriticalMass" android:launchMode="singleTop" android:name=".CriticalMassActivity" android:theme="@style/Theme.Titanium">
   <intent-filter>
      <action android:name="android.intent.action.MAIN"/>
      <category android:name="android.intent.category.LAUNCHER"/>
   </intent-filter>
</activity>

    The intention of overriding the actual launcher activity failed because it has a different name:
    .CriticalMassActivity should be .CriticalmassActivity

    You can try to change it in tiapp.xml but you should probably also clean the project.
