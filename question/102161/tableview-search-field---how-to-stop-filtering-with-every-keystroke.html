TableView search field - how to stop filtering with every keystroke?

What is the best way of preventing every change in the search bar from filtering the TableView? We load the top n records into the TableView and the filter function confuses the issue since it is only working with the loaded data.

In our case the search button issues a new db query, which is as intended. Since the built-in searchbar hides nicely and has other things going for it, it seems like a good idea to use it.

Seems like the opportunity to stop the filtering is in this event handler:

searchBar.addEventListener('change', function(e)   {return 0;  });

but returning 1, 0, -1, null, or e doesn't do the trick. If you've found a way - what is the solution to stop filtering on keystrokes?

iPad/iPhone 4.2, Ti: 1.2.2, SDK: 1.6.x
