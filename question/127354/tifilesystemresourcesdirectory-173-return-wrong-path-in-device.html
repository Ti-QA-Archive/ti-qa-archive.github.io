Ti.Filesystem.resourcesDirectory @1.7.3 Return Wrong Path in Device

hi there,

when i run the following code in the device

var tmpFile = Titanium.Filesystem.getFile(Ti.Filesystem.resourcesDirectory, 'data/v12.js');
alert( Ti.Filesystem.getFile(Ti.Filesystem.resourcesDirectory, 'data/v12.js').exists() );

it showed me like following:

'file://localhost/var/mobile/Applications/xxxxxxxx/xxx.app/data/v12.js'

there should be like

'file://localhost/var/mobile/Applications/xxxxxxxx/xxx.app/Resources/data/v12.js'

right ?

eric

  • No, the resources directory of your app is the root level of your application. So when you request the Resources directory in a compiled app, you're actually requesting the root folder of your app.
