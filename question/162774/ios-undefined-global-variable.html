    This creates an array, not an object:

    Ti.API.GVAppData.device = [];

    You are not assigning stuff as if its an array, but as though its an object:

    Ti.API.GVAppData.device.osname          =   Ti.Platform.osname;

    You are extending the Ti.API namespace (not a good thing):

    Ti.API.GVAppData = [];

    • As Stephen says, don't modify the Titanium proxies (any Ti.* objects) with custom properties like this. You're right, it works on Android. On iOS, it's going to cause an app crash. Unlike standard JavaScript objects, the Ti proxies represent native objects. Unlike JS objects, the native objects are not designed to be mutable.

      You would be wise to explore the CommonJS pattern, callbacks, and even better, Alloy (which uses both of the prior).
