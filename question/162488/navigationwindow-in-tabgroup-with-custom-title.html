NavigationWindow in TabGroup with custom title

I'm using Titanium Studio 3.2.0 in iOS 7.x platform with Alloy framework.

My question is:

is it possible create a NavigationWindow child of a Tab (child of TabGroup, of course) with a custom title without two Navigation bars?

For "custom title" I mean a title with a custom image and a value programatically changed in .js controller.

In the following example I have a TabGroup with 3 tabs:

  • tab 1 has a custom title in Navigation Window but there are also the Navigation Bar with title "tab 1".
    I don't want this Navigation Bar "tab 1", I want only Navigation bar with custom title "window 1".
  • tab 2 it has the same problems of tab 1, the only difference is in the definition of the title.
  • tab3 has only one Navigation Bar, but it is the Navigation Bar wrong (it appears the title "tab 3", but I want the navigation bar "window 3" with barImage.

I have seen other questions similar, but none with a definitive response on this topic.

In
http://developer.appcelerator.com/question/159792/tabgroup-from-navigationgroup
it seems to me that the answer is simply "no, it is not possible", but is mentioned "NavigationGroup", not "NavigationWindow".

In https://developer.appcelerator.com/question/157425/navigationwindow-in-tabgroup
the accepted answer is not exactly the same things, because the solution proposed by Eric does not use NavigationWindow.

NOTE: in 3.1.4. with NavigationGroup there was not this problem, and setting the navBarHidden="true" in the Window father of NavigationGroup it was fine.

Now I have an app already submitted and accepted in App Store, but I cannot do another version with 3.1.2 for this problem.

Thanks for any hints/solution,

Gabriele

    • Hi Stephen,

      good news.

      I chose to stay in 3.2.

      I remove all NavigationWindow (because its were child of TabGroups).

      I used the approach suggested by Eric in the post reported above but adapted with Alloy (all my app uses Alloy).

      Now all works fine and the migration task - now that the architecture of the widgets is clear for me - was simple, just some hours of coding included the testing.

      Lessons learned:

      1) For developers: Don't use NavigationWindow as child of TabGroups.

      2) For you/project team of Titanium :-): write more clearly in the migration guide to 3.2 that NavigationWindow "cannot" be child of TabGroups (if the App doesn't want show two NavigationBars).

      Just for future reference for future Titanium developers I report the Eric example adapted in Alloy.

      Thanks for your support,

      gabriele

      index.xml

      <Alloy>

  <TabGroup id="tabs">

      <Tab title="tab 1">
          <Window id="win1" title="win 1" class="window">
              <Button onClick="doOpen" title="open(click)"/>
          </Window>      
      </Tab>

      <Tab title="tab 2">
          <Window id="win2" title="win 2" class="window">
            <Label text="window 2">                        
          </Window>
      </Tab>

  </TabGroup>

</Alloy>

      index.js

      $.tabs.open();

function doOpen() {
    $.tabs.getActiveTab().open(Alloy.createController('child').getView());
}

      child.xml

      <Alloy>

    <Window id="win3" title="win 3" class="window">
        <Label text="window 3, child"/>
    </Window>      

</Alloy>

      app.tss

      "Window": {
    barColor:"red",
    backgroundColor:"white"
}
