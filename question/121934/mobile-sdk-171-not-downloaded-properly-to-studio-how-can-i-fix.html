mobile SDK 1.7.1 not downloaded properly to studio...(?) how can I fix?

Since updating studio to mobile sdk 1.7.1 I keep getting the following error when trying to build my project to run on the iPhone emulator:

/usr/bin/python: can't open file '/Library/Application Support/Titanium/mobilesdk/osx/1.7.1/iphone/builder.py': [Errno 2] No such file or directory

When I look in /Library/Application Support/Titanium/mobilesdk/osx/1.7.1/iphone/ that file is indeed missing - but in all my previous mobile SDK versions (e.g. 1.7.0 & 1.6.2) it is present.

In fact there's quite a lot less files in 1.7.1 than in previous builds so it looks to me as if the 1.7.1 update didn't download properly.

Any idea how I fix this? I've been searching around for the correct URL so I could try 'Install Titanium SDK from URL…' but can't find the correct path anywhere.

Sorry if this is a bit obvious, I've not really used Eclipse-based IDEs that much before.

    I had the same problem, but here's what happened to me:

    1. SDK Update found…do you want to update? Yes (started the SDK update)
    2. Titanium Studio Update found…do you want to update? Yes (started Titanium Studio update)
    3. Titanium Studio update finishes FIRST and asks to reboot. I click yes, but the SDK update had not indicated its completion.
    4. I try to run an emulator with 1.7.1, the new SDK, and it can't find some .py files…hmmm.

    Thoughts:
    When you do the SDK update, you have to wait awhile. There will be a pop-up acknowledging its success. (I did not do this the first time by restarting T. Studio in the middle of it, but did the second time, and now it works)

    Solution:

    1. Quit Titanium Studio
    2. Delete the 1.7.1 folder in (HDD)/Library/Application Support/Titanium/mobilesdk/osx
    3. Start Titanium Studio - it should now say again that it needs an update
    4. Start the Titanium SDK update
    5. WAIT until the popup acknowledgement

    Voila! It should work now.
