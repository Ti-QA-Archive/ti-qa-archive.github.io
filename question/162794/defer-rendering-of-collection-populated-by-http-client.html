Defer rendering of collection populated by HTTP client

In my Titanium (3.2.1) Alloy project, targeted for iOS6/7, I have a collection which is populated by an HTTP client request upon the page opening that returns an array. The HTTP client loops through the array creating a model for each item and subsequently adding the model to the collection and saving the collection.

Since at this point the view has already been rendered, the models are rendered into the view individually with a noticeable delay between each item. How can I prevent the models being rendered individually and instead populate the collection and then render the entire collection at once?

  • It would help if you provide the code, but since you didn't, I can give high level advice. Instead of looping through the array of models and adding them one by one to the collection, just add them all at once. This will trigger only a single binding event and should get rid of that delay. For reference, the collection's add(http://backbonejs.org/#Collection-add) function can take either a single model, or an array of models.
