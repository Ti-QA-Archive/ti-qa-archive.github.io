Permission Denied - error while executing source \~/.bash_profile in terminal

Team,

I'm trying to created iOS module for titanium application. I'm using MAC LION OS. Titanium version is 1.7.3

I followed the http://wiki.appcelerator.org/display/guides/iOS+Module+Development+Guide.
While i execute the following command from terminal, i'm getting permission denied exception.

source ~/.bash_profile

Kindly let me know how to solve this.
I didnt face any issue while executing the same in MAC OS 10.6

    Hi Sathya

    All that line does is re-evaluate your bash profile after you added the alias in the previous step. You can actually just close the terminal and re-open it to do the same.

    It looks like there is a formatting error in the docs, which I will correct. Would you try the following instead, please?

    source ~/.bash_profile

    Let me know if that helps
