  • You would have to store a list of included files somewhere, and then check against it whenever you try to include something. But you should never need "include_once" in PHP, nor should you need it in JS. An include_once-type function is just extra overhead that slows your application down unnecessarily. If you design your application well, you'll always know what has been included and what hasn't. Be careful trying to write JS the way you write PHP – they're very different languages and require your applications to be designed very differently.

  • LABjs is probably the most popular JavaScript script loader.

    yepnope is a good lightweight option.

    Both work like PHP's include_once.

    I don't think there would be any problems including one of those in a Titanium project. I don't know, however, how they would behave with Titanium's app:// pseudoprotocol. If you are into editing third party code, you could alter one of those to use Titanium.Filesystem to load local scripts.

    It also shouldn't be too difficult to make your own script loader that works like include_once using Titanium.Filesystem. You load scripts into a document by inserting a script tag with desired src attribute. You only need to keep an array or some other structure of filesystem paths for the scripts you've already loaded and check against it when trying to load a new script.
