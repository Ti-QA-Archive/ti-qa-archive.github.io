Cache a local web page inside a webview in Android

I have an application in which I have to cache local web pages inside a webview in Android. The source code of the HTML page is saved in SQLite database. I am fetching the HTML code from the database and setting it inside the webview like this:

web.html=html_code;

The html_code variable contains the source code for creating the webpage. web is the name of the webview which is getting populated from the source code.

The problem is that, whenever I am moving out of the page containing the webview, the content is getting lost. Whenever, I come back to this page containing the webview, I am getting the error that, resource not found. Where am I going wrong?

  • Hi

    Firstly you have not provided any code to show HOW you are caching the pages and how you are retrieving that cached file. We need to see that information before anyone can assist you.

    You also need to define what you mean by a cached web page - as the html content of a web page is far from sufficient information to reproduce an off-line version of a web page unless there are no supporting files referenced by the specific web page.

    These sorts of files are commonly also part of a normal web page;

    • One or more CSS files
    • One or more (usually many) images
    • One or more JS files
    • Any video content
    • Any audio content

    Have you made provisions to cache this - as this is a MUCH harder job.

    When you provide the code so we can see how and where to help you, please also include an example URL or HTML content to see what level of caching you are attempting.
