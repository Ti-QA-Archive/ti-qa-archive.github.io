Cannot Select Provisioning Profile When Distributing to Apple App Store

I'm having issues with not being able to choose a provisioning profile from the drop down when trying to distribute to the apple app store.

Titanium SDK 2.1.3 or 2.1.4
ios sdk 4.3
OS 10.8.6
Titanium Studio, build: 3.1.0.201304151600

Distribution certificate, keychain and sdk all show up properly in the "Distribute" window. If I choose "Manage Settings" I am able to browse to the profile file and add it. It will show up in that preferences window and then when I hit "Apply" or "OK" and return to the distribution window, it still is not there. I've deleted and re-created all of apple's dev certificates. I've used titanium sdk's 2.1.3 and 2.1.4, and neither work. I can't use 3.1.0 because I am running Snow Leopard right now.

I know this was an issue with Titanium Studio 3.0, but was addressed in 3.1 according to everything I've read. To be certain, I installed pyopenssl and pycrypto (a workaround to this initial issue) and nothing has changed.
