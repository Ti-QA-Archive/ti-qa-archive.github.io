Quickest way to find dimensions of an image file in Appcelerator

Hi,

i am a lazy bum and never know the dimensions of my image files.

How do you address this? Do you take notes, or do you store the dimensions of the image in the filename (e.g. "myImage_300_200.png")?

Ideally, i'd like to have a function which returns the correct dimensions for a given image file.

For this to work, one would probably have to create a Ti.UI.createImageView() with width = "auto" and height = "auto", or is there any quicker way, let's say for instance by creating a BLOB from a file's content?

How can I get the quickest the dimensions of an image file I don't know nothing about?

Thanks in advance,
Christophe

  • I did some profiling, it is quickest to do the blob file method, it is twice as fast as the imageView thing:

    [INFO] BLOB method: 54ms
    [INFO] ImageView method: 116ms

    Here's the code i used:

    function blobMethod () {
    var testFile = Ti.Filesystem.getFile("images/img_1024x748.jpg").read();
    return {width:testFile.width, height:testFile.height};

}

function imageViewMethod () {
    var imgView = Ti.UI.createImageView({image:"images/img_1024x748.jpg", width:"auto", height:"auto"});
    return {width:imgView.width, height:imgView.height};
}

    just for the records, this works as well:

    var testFile = Ti.Filesystem.getFile("images/imageFile.jpg"); // get the file

    var dataBlob = testFile.read(); // read in the binary data

    trace(dataBlob.width); // undocumented properties for width and height
    trace(dataBlob.height);
