iPhone iOS View - backgroundImage or backgroundGradient only one will work!

Hi guys,

I'm having an odd problem with the iPhone version of my game…

The following works just fine on Android but on iPhone I only see the backgroundImage if I remove the gradient:


myView = Ti.UI.createView({
        width: tileSize,
        height: tileSize,
        left: tileSide,
        top: upperTileTop,
        borderColor: '#532d1f',
        borderRadius: 3,
        borderWidth: 4,
        zIndex: 3,
        visible: amIVisible,
        backgroundImage: "/Images/MyCharacter" + characterEvent + " Small.png",
        backgroundGradient: myGradient
    });

Just in case, I tried defining the gradient first and the backgroundImage second but it made no difference. (nor should it… but you never know!)

However, the moment I removed the gradient property altogether, it works and I see the image!

Is this a known issue as a search brought back nothing! (remember - it works just fine on Android)

In case this was one of those weird bugs like 'sounds' where you have to declare certain properties outside of the initial object creation I have tried all permutations of setting the image and/or background both inside and outside of the initial view definition.
ie

myView = Ti.UI.view({ etc });
myView.backgroundGradient = myGradient;
myView.backgroundImage = "/Images/MyCharacter" + characterEvent + " Small.png";

So basically it appears that a Ti.UI.view can have ONLY a backgroundImage OR a backgroundGradient but not both! (So I will have to overly a second view on top with the image!) :(

Many thanks in advance…

Oh yeah and…

Mavericks 10.9.1 (13B42)
The latest GA 3.2.0.201312191547
Compiling to iPhone simulator, SDK & Simulator Version 7.0.3, iphone retina 3.5"
