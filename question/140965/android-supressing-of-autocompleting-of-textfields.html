Android: supressing of autocompleting of textfields

Hi,
I have a login area. Both elements (login & password) are textfields.
If I want type 'mc-public' then automaticly a name of a fastfood "restaurant" appears.
How can I avoid or suppress?

In API I only found autocapitalization and autocorrect. Both properties are default 'false'.
What can I do? Is it a system property?

Rainer

    You'll have to set this to suppress both of them.
    They are by default false/none in the SDK but for some reason it doesn't kick in.

    I believe that on android side, if no value is set, the OS defaults are used, and this depends on each producer implementation.

    Is better on android not to assume anything, but set everything the way you want it to be, as with the huge fragmentation you'll never know what the "default" is…

    autocorrect:false,
autocapitalization:Titanium.UI.TEXT_AUTOCAPITALIZATION_NONE
