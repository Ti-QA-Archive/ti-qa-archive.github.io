  • Although not directly what you meant, I achieve this by using different GIT branches. In my case, one branch for the "full" version of my app, one for the "lite" version and one for the "indev" version. By using git checkout <branchname> in the console I can effectively switch the contents of my resource folders and project files and deploy each.

      1. Make a baseline project with everything in.

      2. Then create a new blank project and use symbolic links to references the parts of the baseline project you want uniquely in this project.

      3. Repeat for the "other" project.

      You can now set your own appId or anything else in its own TiApp.xml
