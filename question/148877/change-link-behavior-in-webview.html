  • Mitul's solution will not work for a remote HTML document (he did say "local html file" in his initial response). If the HTML is originating from an external server, the Javascript in the page cannot access the Ti API at all. So it can't fire the application-level event.

    One approach is this: instead of having the WebView load the page, use an HTTPClient to download the HTML, process it to inject some javascript that will intercept clicks on links and fire the app event, and then load the resulting HTML/Javascript into the WebView as a local document. This will allow the document's Javascript to access the Ti API and fire the app event.

    Another alternative is the "link queue" technique that Beau Liddell outlines in this post. He lets the webview load the document directly, and then uses evalJS() to run some Javascript within the document to alter all the hyperlinks. He then uses evalJS() every 1/3 second to check whether a link has been clicked. It looks like a particularly clever technique, but I've never tried it myself.
