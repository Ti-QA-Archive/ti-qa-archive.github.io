[Android] Scrollview inside TableViewRow not show added sub-views

Hello, I have the following sample code: 

    var win = Ti.UI.createWindow({
        backgroundColor : 'white',
        layout : 'vertical'
    });
    var tableView = Ti.UI.createTableView({
        top : 0,
        left : 0,
        right : 0
    }), row = Ti.UI.createTableViewRow({
        layout : 'vertical',
        height : Ti.UI.SIZE
    }), photoScrollView = Ti.UI.createScrollView({
        width : '100%',
        backgroundColor : 'blue',
        height : 260,
        contentWidth : 'auto',
        contentHeight : 'auto',
        scrollType : 'horizontal'
    }), button = Ti.UI.createButton({
        title : 'Add image'
    })
    length = 0;

    row.add(photoScrollView);
    row.add(button);
    tableView.setData([row]);
    win.add(tableView);

    button.addEventListener('click', function(e) {
        var imageView = Ti.UI.createImageView({
            left : length * 250 + 10,
            image : '/images/shop.png',
            height : 250,
            width : 250
        });
        photoScrollView.add(imageView); ++length;
    });

    win.open();

When deploy on my device (Galaxy Nexus with JB 4.2), the imageView was not shown inside the photoScrollView after I clicked the button.

Please help me if you know any work-around for this issue!

Thank you very much.

  • Hi

    Adding a scrollView to a tableViewRow is not always as easy as you would like;

    If the row you are using is either the first or last I would strongly consider adding is as the headerView or footerView to that tableView.

    If you are not sure how - let me know I will give you some code.

  • As referenced by Ti Doc

    Tables and Scroll Views

As a table view inherently scrolls, it creates a very poor user experience when one contains other scrolling views, such as a ScrollableView or TextArea. Thus, this layout is strongly discouraged.
