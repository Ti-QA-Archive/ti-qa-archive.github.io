Titanium 1.7.2 tableview and searchbar shows keyboard onload

I have noticed an uncertain keyboard behavior when there is a tableview and searchbar/text field on the same window. The keyboard always pops up on window load even if you try to hide it by writing 

softKeyboardOnFocus : Ti.UI.Android.SOFT_KEYBOARD_HIDE_ON_FOCUS

I faced same issue when I was working with sdk 1.6.2. I tried all possible ways to hide the keyboard on load but it just din't work. On emulator it works fine but on my android 2.2 device it doesnt.

Thanks

  • Try to hide the keyboard on window open 

    currentWindow.addEventListener('open', function(){
    currentWindow.softKeyboardOnFocus = 0;
});

    This should suppress the keyboard.

    • Actually, You probably have tried this, but have you also tried a click event on the text field and then sending a blur event? Because you need to take focus off of that field on window open. Once that is set, you can use

      softKeyboardOnFocus : Ti.UI.Android.SOFT_KEYBOARD_HIDE_ON_FOCUS

      to take care of the textfield when it's focused.

      //make sure to change activeWindow
//to the window the textfield is in

activeWindow.addEventListener('open', function()
{
    textField.blur();
});

      Make sure you have all your code set up properly to fire in the proper order as well.
