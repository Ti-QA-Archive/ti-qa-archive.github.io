[ERROR] : Invalid "--ios-version" value "7.0" 7.0.3

Hi,
I update the Titanium SDK to 3.2.0.v20131107155450, but when I start application on iPad Simulator i get the error:
[ERROR] : Invalid "–ios-version" value "7.0"
7.0.3
Accepted values:
7.0.3
For help, run: titanium help build
[ERROR] : Invalid "–sim-version" value "7.0".

My XCode version is 5.0.2 and in Xcode.app/Contents/Developer/Platforms I have:

  1. iPhoneOS.platform -> Developers -> SDKs -> iPhoneSimulator7.0.sdk
  2. iPhoneSimulator.platform -> Developers -> SDKs -> iPhoneOS7.0.sdk

I try to change the version number manually with 7.0.3 :-) and the simulator shortly is appear and i get alert with message "iOS Simulator quit unexpectedly".

Can anyone help me how i solve this problem?

Thanks in advance.

    Hope this helps you.

  • The right thing to do would be to switch to 3.2.0.GA.

    However, if you have the need (like i had) to keep going with 3.1.3.GA, you can go on the terminal and write the following:
    sudo npm install –g titanium@3.1.0
    This should overwrite the CLI 3.2.0, which is the one causing problems.

    You may also need to install the previous version of Alloy (1.2.2), with the following command line:
    sudo npm install –g alloy@1.2.2

    This should do the trick, at least it did for me!

  • I had the same problem. I took a bit of a path to my solution, so I am not sure which path fixed it. First I noticed an update to OS X had occurred in addition the simulator not working - it launched but nothing happened. In light of that I guessed new components for Xcode had also been installed. After launching Xcode, it did indeed have an update so I did that.

    In addition to this issue, I had the command line version issue (mentioned in other posts) when building so I had decided to re-install studio.

    My guess is the Xcode update fixed the iOS issue and the re-install fixed the CLI issue.
