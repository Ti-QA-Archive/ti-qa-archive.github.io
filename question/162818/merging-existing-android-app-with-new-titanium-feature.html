Merging existing Android app with new Titanium feature

Scenario: I have an already existing mobile Android platform and I was asked to implement a new independant feature.

Situation: This occurs due to a thesis I'm elaborating, and I found AppCelerator Titanium which caught my attention, and made me want to implement this new feature using this technology.

Problem: So I have the Android app, very well built for a long time now, and running on the market, and I want to integrate my AppCelerator feature on this Android App… Is this possible by some way?

Tip: I know it is possible to convert native Android to AppCelerator modules. However, being this an Android App (in other words, the main part of the application) and not just one simple feature, would it be wise to convert it to a module and then implement any new needed feature through AppCelerator?

The "Tip" was the only possible solution I found, although I'm not sure and might not even make sense… I was wishing I could get some hints/solutions on this please! Thanks so much in advance! :)

    Sorry Tadeu, I know of no way to do what you're asking. The "tip" about creating modules is meant for specific functionality that's not available directly in our API. For example, if you wanted to expose device functionality or an API that our Titanium SDK doesn't expose, you could create a module for that to be used within your Titanium app.
