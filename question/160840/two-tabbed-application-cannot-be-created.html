  • I also came across this error with a brand new installation, attempting to run the quickstart

    • Titanium 3.2.0.201312191547
      on

    Error message inside of Titanium Studio looks like the following:
    http://www.screencast.com/t/nQdTqCU2ubm

    The workaround involves the use of the command-line:
    First, run a couple of checks to see if the SDK is installed correctly

    titanium -version
3.2.0
alloy -version
1.3.0
node --version
v0.10.17 (v0.10.13 should work)
npm --version
1.3.8 (1.3.2 should work)

    Then change directory to your workspace and use the "titanium" and "alloy" commands

    cd /Users/burr/Documents/Titanium_Studio_Workspace (using your workspace directory)
titanium create --name Hello
    • hit return on target platforms
    • app id: com.yourdomain.hello
    • hit return for directory to place project
    alloy new Hello default
cd Hello
titanium build

    The build command may be unnecessary, I just wanted to see if everything else (XCode, etc) was configured correctly.

    In Titanium Studio, File -> New -> Import, Titanium -> Existing Mobile Project and select Hello

    Note: Titanium 3.2 requires XCode 5 which requires Mavericks, presently upgrading the OS/XCode on my other Mac

    Burr
