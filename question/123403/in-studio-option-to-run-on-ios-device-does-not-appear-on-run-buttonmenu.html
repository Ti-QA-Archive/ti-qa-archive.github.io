In Studio: option to run on iOS device does not appear on "run" button/menu

I've a project that's been successfully built/deployed via Appcelerator Developer.

I've now created a new project in Studio and migrated the code over, and can run it in the Simulator, by clicking the "run" button (green "play" button) and selecting a simulator configuration from the menu that drops down from it.

HOWEVER, I see no options there to run on an actual device (even though one is plugged in). This appears to be the case for any/all projects I create in Studio.

Any help…?

Many thanks in advance.

  • I am seeing quite a few people posting this issue on the forum.

    I came across this when I first installed studio and on a couple of projects. have you tried creating a new project and running with the defaults. This should run ok.

    Now a thought as I Type. In studio there are two ways to view the contents of a project. One is where you see all projects the other is project specific. I remember that I hit the problem and went into the individual view and the options appeared.

    also try opening the tiapp.js file unselecting all the platform options save it, then select the platform options again.

    hopefully one of these will solve the issue.

  • I too had this issue, I found that if I right click on my project from the Project explorer tab. There is a properties option, when I clicked on that select the Project Natures option. I found that somehow the Titanium Mobile Nature was unchecked. Once I checked that option I was able to see the run on device options again.
