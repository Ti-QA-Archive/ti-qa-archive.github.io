Help with UI Orientation on iPad

I use the following code to create a nice toolbar with custom buttons that works great! It uses fixed width to properly space the Title and buttons where I want them.

I am having problems with orientation on the iPad changing to Landscape mode and need advice. I know how to get the orientation and setup a listener for the change. What I need help with is the best way to redo my UI when the orientation changes…if that makes sense. Do I hide the old toolbar and create a new one each time the orientation changes?

var toolbar2 = Titanium.UI.createToolbar({
                items:[
                mapButton, 
                Ti.UI.createButton({systemButton: Ti.UI.iPhone.SystemButton.FIXED_SPACE, width:15 }),
                helpButton, 
                Ti.UI.createButton({systemButton: Ti.UI.iPhone.SystemButton.FIXED_SPACE, width:185 }),
                 mainTitleLabel,
                 Ti.UI.createButton({systemButton: Ti.UI.iPhone.SystemButton.FIXED_SPACE, width:120 }),
                 navigationButton,
                 flexSpace, envelopeButton, flexSpace, settingsButton, flexSpace, refreshButton, flexSpace, infoButton ],
                top:0,
                borderTop:true,
                borderBottom:true,
                barColor:'#222'
            });

Help?

    The approach you outlined of removing and re-creating on orientation change should work. Please note I believe you have to win.remove(toolbar2) then set it to null, then re-create and add.

    Another option might be to use a series of views. You find alot of the iPad apps use views instead of the native toolbar and header controls.
