Ti.Map - map shows only after "Clear data", otherwise is empty/black

I am having a strange problem. I am not sure when it started, but we noticed it only these days. It is
possible that it is the result of recent updates to Android SDK.

I have a map, created with the module Ti.Map (using the 2.1.2 version, but changing to 2.1.1 does not fix the problem).

I am compiling with SDK 3.1.2GA, but compiling with 3.1.1GA does not fix the problem.

I am on Android 4.2, but it also occurs on 4.1 (another device), so it's probably generic.
The version of Google Play services I have on my device is 3.2.25

I have Holo theme and Action Bar in my application. It is a single-window app, no tabgroup.

As I have said, I use a MapView created with ti.map, Google Maps v2. The mapview is inside the lightweight window, and inside an additional view. However, adding it directly to a heavy-weight window does not fix it.

The first time I install and run my app, the map shows correctly.

However, when I force-stop and restart my app, the map shows only a black/empty view, with only userLocation and zoom buttons visible.

After I do "Clear Data" from settings, the map shows fine again.

I tried it all: adding map directly into a heavy weight window, I created another api key, I tried adding map on focus. It did not change the behavior.

I also tried running the apk compiled 18/07/2013 (probably with sdk 3.1.1GA and ti.map 2.1.1), the problem occurs there too.

The only suspicious messages in log are those, but they also occur when the map shows fine:
