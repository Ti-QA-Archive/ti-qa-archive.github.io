$.resetClass() crashes app

Using $.resetClass() on views/objects created on the fly will apply styling to said views on an iOS device but will cause an Android device to crash. Why does $.resetClass crash an Android device but not an iOS one? I fixed the problem by switching to $.addClass but I find the logic backwards. The $.resetClass should cause the same problem on iOS or $.resetClass should work on Android. Is there any documentation to support this?
