How to speed up Android Module Development

Hey folks,

Been developing an Android module for Titanium 1.7 and 1.8, and the process has me never wanting to touch Titanium's Android module environment again. I spend most of my time waiting for the module to build and deploy to the emulator (which I keep running). It routinely takes 3 minutes from calling the ant script until the module launches. This doesn't take into account the recent frustrations that 1.8 modules couldn't even be BUILT on windows!

Is this the norm for building modules on Android for Titanium?

  • Hi Michael..
    I just been through the process of building an Android module. Yeah it takes time, but not 3 minutes for me. Do you close the Emulator each time?
    You can deploy your module directly to the emulator.

    But I found it easier to deploy it to a zip file, move the zip into a regular module project, and run the app again using fastdev.

    I have around 30 sec turnaround time with this method. hope it helps you
