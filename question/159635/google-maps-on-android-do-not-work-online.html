Google Maps on android do not work online

Hello ,
I am developing on mac os mavericks, on Titanium Studio, build: 3.1.3.201309132423 for android an application using alloy and google maps v3. When I test the application locally everything works fine but when I upload the application on google play and test it google maps do not show the tiles. I dont get the map images but I do get the annotations location and image. Can you please help me?
Thanks in advance

    It sounds like you didn't create the API key correctly for the Android maps module. You must generate it using the same certificate you use for publishing (you need the SHA1 key from the publication certificate), and you must modify your tiapp.xml with not only that key but the app ID used for publishing. If you used test values for either of those, or if you entered the text in the tiapp.xml incorrectly, you'd see the "missing tiles" issue you're experiencing.
