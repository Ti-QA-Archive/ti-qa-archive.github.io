WebView load event has wrong HTML

Android emulator 4.0.3.
Ti SDK 2.1.4.GA

I am working on a OAuth solution and we have a custom callback page that I need to execute an evalJS on to retrieve some application specific configuration. The OAuth is working fine and the callback page is being displayed. The 'load' event if firing as expected and I am able to detect when our callback URL is present.

But in about 90% of tests when I execute evalJS on the callback web page I do not get the results from my embedded javascript function. Examination of the 'source.html' of the load event argument shows that the HTML is for the previous web page and not the callback page. In the other 10% this works as expected.

Providing a code sample is not feasible. Also, I am not able to change SDK versions.

Richard
