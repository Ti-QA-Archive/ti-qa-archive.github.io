TableViewRow overlaps a containing view on tap

I have got a bunch of table rows being created like this:

var row = Ti.UI.createTableViewRow({
    backgroundColor: '#fff',
    selectedBackgroundColor: '#ddd',
    height: '40dip',
    hasChild: hasChild,
    hiddenText: title
});

var rowLabel = Ti.UI.createLabel({
    text: title,
    color: '#000',
    font: {
        fontSize: '14dip'
    },
    width: windowWidth - 50,
    height: '40dip',
    textAlign: 'left',
    left: 35,
    top: 0
});

var rowColour = Ti.UI.createView({
   width: 20,
   height: '40dip',
   backgroundColor: colour,
   top: 0,
   left: 0 
});

row.add(rowColour);
row.add(rowLabel);

It basically puts a small view with a background colour the the left of the label, both of which are inside the row. The only problem is, that when I tap on the row the selectedBackgroundColour (#ddd) overlaps the coloured view (rowColour). I assumed the #ddd would go behind the coloured the view since it's the child. Is there any way to push the coloured view to the front of the stack?
