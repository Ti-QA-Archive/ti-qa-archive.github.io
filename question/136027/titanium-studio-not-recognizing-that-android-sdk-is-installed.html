Hi,

When I try to run a sample hello world project, I get this error: An Android SDK is missing. Titanium requires Android platform 2.2.

I click on the Install or Update Android SDK button, and the Android SDK Manager pops up, and spends about an hour downloading and installing a bunch of packages. The finished result looks like the image below.

But Titanium Studio doesn't recognize the install. I still get the same error message, and if a restart Titanium Studio it simply goes through the whole process again, re-downloading everything. I've done this cyle 3 times now with no success.

How can I get studio to recognize the android SDK? I am using the most recent 2.0 version on win7.

    • Please new 'User Variables' in Computer > Properties > Advanced System Settings > Environment Variables

      ANDROID_SDK
      C:\Program Files (x86)\Android\android-sdk

      JAVA_HOME
      C:\Program Files (x86)\Java\jdk1.6.0_29

      PATH
      %JAVA_HOME%\bin; %ANDROID_SDK%/tools; %ANDROID_SDK%/platform

  • To anyone out there who still has this problem.

    I struggled with it for about two weeks. Even tried it on Ubuntu, still the very same problem. Just found a solution. Edit your USER environment variables in windows. Not system, user. Add the location to your android tools, platform and platform-tools to the PATH directory IN USER. Everything worked like a charm from then on.

    As an addition, make sure that in your SYSTEM environment variables, specifically in PATH, you have the bin directories for node, python and java.
