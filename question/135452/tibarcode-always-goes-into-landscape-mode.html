Ti.barcode always goes into landscape mode

Hello,

In my mobile application for both Android and iOS I use the barcode module, provided by Titanium+.

It works fine on iOS, however, on Android, the camera always goes into landscape mode. This is a problem because I want to make the application only in portrait mode due to positioning issues.

Does anyone know why this happens and how to fix it?

Initialization of the barcode variable:

var Barcode = require('ti.barcode');
Barcode.displayedMessage = '';
Barcode.useLED = false;
Barcode.allowRotation = false;

Opening the barcode scanner:

Barcode.capture({
  // animate: true,
  showCancel: true,
  showRectangle: true,
  keepOpen: false,
  acceptedFormats: [
     Barcode.FORMAT_QR_CODE
  ]
});

Can anyone help? We payed for this module, so this should work.

  • I already opened a ticket for that problem in the help desk. Currently this is not supported (Android) and will not be supported. That was the answer I got. The ZXing module that does the work for the Titanium module is just built this way, even though there is a hack to make ZXing work in portrait mode, this cannot be built into the module for some internal reasons.
