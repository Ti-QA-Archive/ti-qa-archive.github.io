Titanium module creation and interaction with XCode Build Phases

hello all, my question is regarding titanium module creation and how it interacts with XCode's "build phases" settings.

In my regular Xcode iphone app, the script copies over relevant data files as the last build phase item. The files are copied over into the iphone 5.1 simulator directory so that when I run the simulator, it interacts with those files.

The script is basically a bash script that creates some directories and copies the files over:

snippet (this is working just fine in XCode):

if [ -d ${TARGET_BUILD_DIR}/${PRODUCT_NAME}.app/data ]; then
echo "${TARGET_BUILD_DIR}/${PRODUCT_NAME}.app/data exists"
else
mkdir ${TARGET_BUILD_DIR}/${PRODUCT_NAME}.app/data
fi

cp -r ../../data/common ${TARGET_BUILD_DIR}/${PRODUCT_NAME}.app/data

I'm trying to replicate this process with Titanium, but it's not working. I created a module, brought in all the code from the XCode project. When I try to put the same build phase script as the last action item, it gives errors parsing that script. So what I do to test my code and the module itself is copy those relevant data files after XCode builds successfully, then do build.py and "titanium run" in the command line. Everything seems to work as it used to in the XCode environment.

So I would just like to automate this file copy process so that when the app runs in the simulator, the data is already copied over and ready to be interacted with. Do I implement this in XCode during build time? Or somewhere inside the "build.py" or "titanium run" scripts?

Thanks, I appreciate your time and advice.
