  • I can't really give my whole code here as it would be too complex but I try to give you more details here:

    1. I create a coverflowView 

      var coverflowView = Titanium.UI.createCoverFlowView({
});

    2. Then I create an array images[] that I fill with my images. I suppose this is similar to your images array but only with images, no text or title.

    3. I attach the images array to the coverflowView

      coverflowView.images = images;

    4. Next, I create a label for the title:

      var titleLabel = Ti.UI.createLabel();

    5. I add both views (coverflow and label) to my window (or another view).

      currentWindow.add(coverflowView);
currentWindow.add(titleLabel);

    6. Then, I use the coverflowView change listener to detect when the image changes. Within, this function, I get the index of the image within the array using e.index.
      Then I retrieve the corresponding title of the image to display it in the label.

      coverflowView.addEventListener('change',function(e) {
...        titleLabel.text = getTitle(e.index);
...

    7. The function getTitle will search for the corresponding title. In my code, I actually execute a SQL select query to retrive names and other stuff. As I am using the same query to fill my images table and to retrieve the names, I know that I can use e.index to make sure that I get the right title.

    The only thing you have to manage is the display of the title for the first image, before it is changed as the "change" listener is executed only when you switch image. When you add the label view, you need first to add the text of the first image displayed.

    I hope this is clear enough. If not, let me know which step is not.

    There might be other solutions but this one works for me.
