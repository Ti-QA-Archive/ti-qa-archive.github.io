using dip in iPad for positioning

Hi ,
Im creating one iPad application . For that i have created button with positioning left and right using 'dip' . Actually giving position using 'dip' is actually correct ? whether its hardcoded?

    Hi

    In iOS all measurements are consider in dips unless you say otherwise already.

    A good example would be the iPad and the iPad Retina - only taking the width into account here when you say this

    var view = Ti.UI.createView({
    width: 384
});

    You are saying create a view equal to half the width of the device. This is true for both iPad and iPad retina because the default is dip.

    • iPad width = 768 as 1 dip = 1 pixel
    • iPad retina width = 768 as 1 dip = 2 pixels

    So the previous code will create the same layout on either device.

    Also these things are equal;

    var view = Ti.UI.createView({
    width: 384
});
// same as
var view = Ti.UI.createView({
    width: '384dip'
});
// and even the shorthand version
var view = Ti.UI.createView({
    width: '384dp'
});

    This makes it easier to create one layout and not have to worry about the units of measurement.

    • If you wanted to access every single retina pixel you would use px.

      So on ipad this…

      var view = Ti.UI.createView({
    height: '384px',
    width: '384px'
});

      …would be a square measuring half the width of the standard iPad.

      However on an iPad retina is would cover only a quarter of the width.

      Unless you had a good reason to play with specific pixels on iPad retina only then you should maintain the use of dips and just type numbers. There are good reasons why you would use specific pixels measurements, games for example - but for most of the time retina graphics and dip positioning provide exactly what most apps require.
