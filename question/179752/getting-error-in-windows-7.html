  • Hello Anil,

    The primary problem seems to involve Android tooling version problems. Apparently some developers are having troubles when using Android tools R23. This ticket has been resolved, which could be the cause of this problem.

    Open the android app (part of the Android SDK) and see what it reports on the "Android SDK Tools" line. If it says R23, you have three choices: 1) downgrade your Android tools (a manual and not very easy process), 2) use the nightly build of the SDK which includes the fix for TIMOB-17286, or 3) Get the latest SDK release which officially contain this fix.

    To use the nightly build: visit http://builds.appcelerator.com/ then next to "install" under the heading, copy the ti SDK install command. Paste that into your terminal window and press Enter. Update your tiapp.xml to use the resulting SDK version.

    Also more info on the issue is here. Hope this helps. Thanks.
