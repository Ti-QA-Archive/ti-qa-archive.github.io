    Base64 is NOT encryption in any sense of the word, other than a VERY weak "security by obscurity".

    Base64's sole purpose is to allow non-text data to get passed via a text based protocol such as SMTP.

    There is no key, because it is not encryption. It is just an alternate way to "encode" but not "encrypt" data. See the wiki definition of base64.

    • Thanks for moving the comment back to here. It will help rank future searches by other users.

      To answer the question, there are a few ways you can encrypt data using javascript:

      • Javascript implementation of AES
      • Block Tiny TEA
      • Code a module and encrypt using native algorithms instead of js

      One obvious caveat is when you encrypt a field you store in a db, that value becomes useless as an index field or ORDER BY clause.

      However, if you can store your keys in clear text and just encrypt the values, the above may work for you.

      Appcelerator does not currently support one of the implementations of SQLite which stores the entire DB encrypted, with full support for indexes and ORDER BY etc to work on encrypted columns.
