    It might be the Java 1.7 (aka 7) as Titanium does not support this. You have to use 1.6 (aka 6).

    • Titanium Compatibility Matrix - Java Development Kit

    If a system has Java 1.7 then this is often the cause of the problems. Hopefully everything will work once that is sorted.

  • After hours of debugging through the 'node-inspector' tool, I'm able to make Titanium Studio 3.3.0 works on Windows 7 64 bit.
    Here the detailed procedure:

    NOTE: EVERYTHING YOU INSTALL MUST BE 32 BIT, REGARDLESS YOU HAVE A 64 BIT WINDOWS!

    0) INSTALL JAVA SDK (if you haven't already done)

    0.1) Download the 32 bit version (I have used the 1.7) of java sdk;

    0.2) Install it.

    0.3) Set the environment variable JAVA_HOME to the installed dir;

    0.4) Add to the path %JAVA_HOME%\bin.

    1) CLEAN UP EVERYTHING ABOUT TITANIUM AND NODE.JS:

    1.1) From Control Panel, remove Titanium Studio (if installed);

    1.2) Remove Node.js (if installed. Before this make sure that no 'node.exe' process are in execution from Task Manager: kill them'all!);

    1.3) Open a command prompt and do the following:

    1.3.1) cd Users\<User> (where <User> is your logged username)

    1.3.2) rd .android /s (remove the android)

    1.3.3) rd .titanium /s (remove old titanium settings)

    1.3.4) cd AppData\Roaming

    1.3.5) rd npm /s (to delete npm)

    1.3.6) rd npm-cache /s (to delete npm cache)

    1.3.7) rd Appcelerator /s (to remove residual Titanium Studio files)

    1.3.8) cd \ProgramData

    1.3.9) rd Titanium /s (to fully remove any SDK)

    DONE! Titanium is now SERIOUSLY gone away.

    2) INSTALL ANDROID SDK

    2.1) Download the latest Android SDK Tools (or ADT) and install it.

    2.2) Add to the PATH the platform-tools and tools directory of the installed Android SDK Tools.

    2.3) Run "SDK Manager", install the package, re-run it (when requested) and continue installing Android API 16 (or higher);

    3) INSTALL NODE.JS

    3.1) Download the latest 32 bit version of Node.JS (0.10.29 at time of this writing) and run it.

    4) INSTALL TITANIUM

    4.1) From command prompt type:

    4.2) npm install titanium -g

    4.3) ti sdk install –default

    4.4) FIX TITANIUM ISSUES

    4.4.1) First of all remove the file 'C:\ProgramData\Titanium\mobilesdk\win32\3.3.0.GA\node_modules\titanium-sdk\lib\emulators\genymotion.js', if it's present (I have read this here: https://jira.appcelerator.org/browse/TIMOB-16074);

    4.4.2) NOW, the magic. After hours of debugging, I discover the reason why Node.js hangs (at least on my machine). Edit the following file with a text editor:

    C:\ProgramData\Titanium\mobilesdk\win32\3.3.0.GA\node_modules\titanium-sdk\lib\windows.js

    From line 376, you'll see the following piece of code:

    'powershell': function (next) {
    if (!selectedVisualStudio) return next();

    var args = [
            '&&',
            config.get('windows.executables.powershell', 'powershell'),
            '-command',
            path.resolve(__dirname, '..', '..', '..', 'node_modules', 'titanium-sdk', 'bin', 'test_permissions.ps1')
        ],
        child = spawn(selectedVisualStudio.vcvarsall, args, {
            cwd: path.join(selectedVisualStudio.path, 'VC')
        });
    child.stdout.on('data', function () {});
    child.stderr.on('data', function () {});
    child.on('close', function (code) {
        if (code) {
            results.issues.push({
                id: 'WINDOWS_PHONE_POWERSHELL_SCRIPTS_DISABLED',
                type: 'error',
                message: __('Executing PowerShell scripts is disabled.') + '\n' +
                    __('In order to build Windows Hybrid apps for the Windows Store (winstore), you must change the execution policy to allow PowerShell scripts.') + '\n' +
                    __('To enable PowerShell scripts, search __PowerShell__ in the __Start__ menu, right click the icon, select __Run as administrator__, then run:') + '\n' +
                    '    __Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser__'
            });
        }
        next(null, { enabled: !code });
    });

    This routine detect the 'powershell' and for some reason, it hangs the script. More in details, the 'close' is never triggered.
    I put a breakpoint on that instruction and from the node-inspector command line, I have tried to run manually (from command prompt) the command executed by the 'spawn' and it return correctly. But for some odd reason, the 'close' event is never triggered from the script. Mistery. So, if you don't want to develop for Windows Phone 8, you can fix this bug simply replacing:

    if (!selectedVisualStudio) return next();

    with:

    return next();

    (without the if)

    Save the file and now the system doesn't hangs anymore! Now we can continue with the setup.

    4.5) ti setup

    4.5.1) select 'quick' and setup titanium.

    5) CHECK THAT ALL IS GOOD

    5.1) From command prompt, type:

    ti info

    6) INSTALL TITANIUM STUDIO

    6.1) NOW, you can download and install Titanium Studio 3.3.0.

    6.2) Install it, make a new project and try to run it. In my case, it works.

    I hope this guide can help other people, I spent more than 10 hours to figure out the problem.

    Enjoy it.

    P.S.: I hope to haven't miss any step. Let me know if this procedure works also for you.
