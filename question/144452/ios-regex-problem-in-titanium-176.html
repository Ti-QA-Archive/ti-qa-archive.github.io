iOS regex problem in Titanium 1.7.6

Hi,

I'm having problems with a regular expression to validate a password that must have 6 alphanumeric characters. For this i'm using the following regular expression:

/(?!^[0-9]*$)(?!^[a-zA-Z!#$()*+,.:;=?@|{}\/\[\]\\]*$)(?!^[0-9!#$()*+,.:;=?@|{}\/\[\]\\]*$)^([a-zA-Z0-9!#$()*+,.:;=?@|{}\/\[\]\\]{6,16})$/

In the application that i'm upgrading, i'm using Titanium 1.7.6, and to reproduce this issue i'm using the following lines of code that should return "true","false","false":

var rgx = new RegExp(/(?!^[0-9]*$)(?!^[a-zA-Z!#$()*+,.:;=?@|{}\/\[\]\\]*$)(?!^[0-9!#$()*+,.:;=?@|{}\/\[\]\\]*$)^([a-zA-Z0-9!#$()*+,.:;=?@|{}\/\[\]\\]{6,16})$/);
Ti.API.debug("rgx.test('12345a') = " + rgx.test('12345a'));
Ti.API.debug("rgx.test('123456') = " + rgx.test('123456'));
Ti.API.debug("rgx.test('abcdef') = " + rgx.test('abcdef'));

Executing this code in several Android versions, always give me the correct output:

D/TiAPI   (  706): (kroll$2: file:///android_asset/Resources/tests/test_regex_problem.js) [10819,13665] rgx.test('12345a') = true
D/TiAPI   (  706): (kroll$2: file:///android_asset/Resources/tests/test_regex_problem.js) [13,13678] rgx.test('123456') = false
D/TiAPI   (  706): (kroll$2: file:///android_asset/Resources/tests/test_regex_problem.js) [42,13720] rgx.test('abcdef') = false

Executing the equivalent code in Google Chrome, gives me the expected output:

rgx.test('12345a') = true
rgx.test('123456') = false
rgx.test('abcdef') = false

Executing the testing code in iOS (Xcode 4.4.1), gives me the following output:

[DEBUG] rgx.test('12345a') = true
[DEBUG] rgx.test('123456') = true
[DEBUG] rgx.test('abcdef') = true

I've tested this code with the latest version of titanium, and the output is the expected, but for now, updating the app to latest Titanium version is not an option.

What am i missing here? Is this a known titanium problem for this version? Is there a workaround to this problem?
