since it converts titanium app into xcode after pre compilation.

  • Short answer is: nope, you cannot.

    You can, however, create your very own modules in Objective-C and import them into your Ti Project.

    See here for more info on how exactly that works and how to build one: http://developer.appcelerator.com/blog/2011/10/module-development-guide-reference-module-for-ios-and-android.html

    Hope it helps.

    • Actually, that's correct. You can add functions or stuff like that to existing .m files in the Class directory.

      For example, to add functionality to your UIView, go to build/iphone/Classes/TiUIView.m and do your thing.

      However, to create new classes or add new functionalities to the Ti API, I'd advice you to create Ti Modules.

      — commented November 22nd 2011 by Bernardo Oliveira

    • Oh, plus, if you add your methods to any existing .m files in the Classes directory, remember to back that up! Otherwise, you'll lose them on the first clean rebuild you do.
