Android orientation change trigger window close event

I've notived that on android for some window with an close event handler, the orientation change trigger this event without closing the window (it is still on the screen).

The log shows this

[DEBUG] : Window: Window is closed because the activity is forced to destroy by Android OS.
[DEBUG] : Window: Checkpoint: postWindowCreated()

Android 4.2, 4.4
Ti 3.4.1.GA

It seems that it is because of the nature of the window (lightweight, heavyweight).
Is there a way to prevent this close triggering?

If not, is there any clean way to display a "popover" window anywhere in the app on android? Thank you

    By default, Android restarts activities on various system events, such as orientation changes. You can turn this off by adding a setting to your manifest.

    <activity 
  android:configChanges="keyboardHidden|orientation|screenSize">
...

    The key in the above is making sure that orientation is included in the list. If it's there, it signals the OS to preserve the activity rather than restart it. I believe our default is to include that value in the generated AndroidManifest.xml. Perhaps you've made a customization that's removed it?

    There are steps for editing the manifest (adding entries to tiapp.xml) in the Orientation handling guide, though in your case it sounds like you're not actually limiting orientation changes.

    BTW, since roughly SDK 3.3, all Ti.UI.Windows have been heavyweight. There are no more lightweight windows. Use full-screen Views if you need "lightweight windows."
