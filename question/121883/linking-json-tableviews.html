My question is should I have all the company data in the one searchable feed (which will contain a huge amount of data), or if I create a separate feed for each company, how would I go about linking it to the filter results table, so when I click a row in the filter table it loads the info from that companies individual feed ?

    You really do confuse me terry, but hope this will awnser everything now.

    Step 1 send an request to the webservice for only the names and unique id's from every company

    Step 2 create tableview with the data the webservice send you, make it as simple as possible, so create a row with only the title property set for the company name. Give the row an property for the Id.

    Step 3 click an row, create an event that gets the unique Id you added to the row as a property. Now create a new request to the webservice to get the full data from the company. You send the unique Id to the webservice and it will return you full details verry quickly.

    This is the best way in my opinion, loading full data of 2000 rows will take minutes depending on the size of the full data.

    Even if you have 2 different JSON results, 1 containing the names and 1 containing the full data. If you send the request at the same time, the moment you click a row you havent gotten the full data yet. With the above you will get full data of only the company you need, reducing the loading time drasticly
