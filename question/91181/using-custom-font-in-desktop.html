  • Currently, there is a bug with custom fonts, so it won't work at all in Windows. And of course this is seen as a low priority bug that will not be fixed in the forseeable future. (Though hopefully this will fix itself with the webkit update in the upcoming 1.1 SDK.)

    Anyway, if you want to use custom fonts in the Mac and Linux version of your apps, these are the steps:

    1. Get the font you want, in otf or ttf format. NOTE: You must have the rights to distribute the font! It's probably best to stick to fonts with an open license. For this example I will use Lobster

    2. Copy the font in your resources folder, for example Resources&#x2F;fonts&#x2F;Lobster.ttf.

    3. Now create the @font-face rule at the beginning of your CSS file to make the font usable:

    @font-face {
    font-family: 'Lobster';
    src: url('fonts/Lobster.ttf');
}

    Adjust the path depending on where your css file is located.

    1. Now you can use 'Lobster' as a value for font-family in all other CSS rules. For example:
    body {
    font-family: 'Lobster';
    font-size: 16px;
}
