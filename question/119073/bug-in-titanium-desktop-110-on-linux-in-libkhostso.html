    • @Stuart: It's because Titanium Studio is distributed as a zip but zip doesn't support symlinks. Whichever compression tool they've used has converted the symlinks to text files with the name of the file the symlink is supposed to point at as their content.

      Whomever packaged it clearly didn't test it.
      Anyway, you can fix it by executing the following bit of shell script:

      cd ~/.titanium/runtime/linux/1.1.0/;
for i in $(find -size -100c); do
    mv $i $i.bak;
    ln -s $(cat $i.bak) $i;
done;

      If anyone from appcelerator is reading perhaps you should consider tgz to package the linux versions in the future.
