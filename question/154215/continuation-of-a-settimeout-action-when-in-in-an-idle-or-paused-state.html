Continuation of a setTimeout action when in in an Idle or Paused state

Hi all-

I have a activity timer which will automatically logout a user after a specified period of time. I place the timer into a setTimeout action, which calls my logout functionality to cleanup the app and log the user out as the timer runs out (and essentially the token at the server expires).

What I have seen is that if the user places an iOS app into a paused state, the setTimeout function also seems to pause until I capture a resume state. If in the paused state I call my logout function, the logout actually will trigger my app cleanup properly, so I know functionality will work within my paused handler.

Here is the question…is there a way to continue my setTimeout while in the paused state so I can continue the timer countdown and properly logout the user when in paused or active state?

As of now, here is what I have going on:

Info.plist:

Application does not run in background: NO

Alloy.Globals.Configuration.tokenExpiration = 300000;
Alloy.Globals.Configuration.activityTimer = _.delay(clickLogoutEvt, Alloy.Globals.Configuration.tokenExpiration);


Ti.App.addEventListener('pause', function(e) {

  Ti.API.info('APP RUNNING IN BG');

  // This does execute if I hit the home button, so I simply need to continue the _.delay/setTimeout within this pause stream
  // clickLogoutEvt

}

Ti.App.addEventListener('resume', function(e) {

  Ti.API.info('APP RESUMED');
}
