translucent window background color (not transparent)

Hi there,

i would like to show a config dialog view over an existing screen and would like to dim the background when my dialog is open in foreground.

I noticed that one can apparently set the backgroundColor to include an Alpha value by providing something like

win.backgroundColor = '#AA000000' (note that these are 4 pairs instead of 3

Two questions here:

  • apparently, when adding alpha, the FIRST pair is the alpha (AA), is this correct?

  • any chance to make a fade-in here somehow? For the time being, the dialog appears, the background IS dimmed, I would like to softly animate it if possible

  • is there any shortcut for this using 4 values (like one can set the color with '#AF0" for a window - can one do #AAF0? This is however not working for me for the time being, i get a malformed color hint in the console.

Thanks in advance,
Christophe

  • Well, #ff000000 is a solid black and not a transparent red. So, ff is opacity (100%) and 000000 the color (black). Try #00000000 and you'll have a transparent background.

    Opacity, instead, affects the entire element, so, for example, if you are building a label with opacity: 0.0, it will all become transparent, text included.
