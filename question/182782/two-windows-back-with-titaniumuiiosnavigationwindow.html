Two windows back with Titanium.UI.iOS.NavigationWindow

Hi, I use a Titanium.UI.iOS.NavigationWindow with three windows: from the first I open the second, from the second I open the third.

In some cases, when the third is the currently opened window I need to go back to the first without displaying the second.
So, when I click the back button in the navigation bar of the third window, I must close either the third and the second.

Is this possible? How?

Thanks in advance

  • I must say I did something a little crazy that (in simulator) seems to work: I store a reference of the second window in the third. When I close the third (with animation) I previously close (without animation) the second by calling a function I have in the second. This way, when the third is closing, I can see the first underneath. I would be glad if there was some solution less "tricky".

    — commented June 26th 2015 by London Moe

  • I've done this before by storing an array of references to the opened windows, then 'go home' by looping through and closing them off without animation.
