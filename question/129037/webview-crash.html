Webview Crash

Hi,

I'm experiencing issues with a Webview running on an iPad app I'm developing.

The Webview is simply used to load an external site. Everything works perfectly on the iPad simulator but it's very temperamental when running on a device (first generation iPad) and causes the App to crash when using the webview.

There error is persistent on SDK 1.75 - 1.8 builds (several of which I've tried).

I'm unable to post my complete code, but to give you an idea it consists of:

4 Scrollviews (1 Parent horizontal scrollview that contains 3 vertical scrollviews as children, obviously each one has it's own content and are almost entirely comprised of images).

1 webview (used to load the external site)

These are added to a Window and I use animate view/transitions to switch between the parent scrollview and the webview.

Webview:

var webApp = Ti.UI.createWebView({
    url : 'http://www.robchalmers.info/temp/index.html',
    //url : 'http://www.google.com/',
    width : 1024,
    height:768
});

Transitions:

wrapper.animate({
    view : webApp,
    transition : Ti.UI.iPhone.AnimationStyle.FLIP_FROM_RIGHT
});

There really isn't anything intensive running on the URL that I could foresee as an issue and I've tried other URL's and I'm still experiencing the problem. I can only assume this is a memory issue caused by the scrollviews or something that I've missed.

Any suggestions / thoughts would be appreciated,

Cheers,

Rob
