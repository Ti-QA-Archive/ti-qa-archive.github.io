Missing dev_keystore file?

Hi all,

I seem to be missing any Library/Application Support files anymore pertaining to Titanium. I'm on OSX running Mavericks. I recently (couple months ago) looked at the dev_keystore file, as when I initially sent my Android app in I'd not used a keystore of my own, which made the app use the default.

Now I've got an app update, but I can't do what I did last time, which was to manually specify the file. So if anyone can explain to me how I can get those files back, or can provide the dev_keystore file for me, that would be fantastic.

Thanks!

  • You published an app to the Play store using the development certificate? Not a good idea. The keystore should be unique to the publisher and/or the app. You should make sure to save backup copies of the keystore file in secure locations so you can publish updates. Admonishments out of the way…

    The keystore is distributed with the SDK and may not be the same across release versions. Though, it's probably the same file each time.

    1. Run ti sdk to find the various locations where your SDKs are installed.
    2. Find the one you want to use, that version's keystore will be at SDK_PATH_HERE/android/dev_keystore

    If you don't have the correct version on your system and the closest version you do have doesn't work for your app, you might be able to find the old version on builds.appcelerator.com.
