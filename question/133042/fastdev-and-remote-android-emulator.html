Fastdev and remote android emulator

I connect the Android emulator running on windows to my linux notebook, where I want to develop. Using titanium.py with the run command works and I can interact with the app, but I intended to use fastdev.

With titanium fastdev started, the app stops on the 'powered by titanium' splash screen when I do titanium run --platform=android

I suspect this is caused by the fastdev server not running on localhost, from the the apps POV. But I'm not quite sure how fastdev really works.

Maybe there some config where I can set the remote IP? I skimmed the python sources but couldn't find anything about the application side part of fastdev. Where is that located?

  • I changed the IP address

    private static final String EMULATOR_HOST = "10.0.2.2";

    in

    android/runtime/common/src/java/org/appcelerator/kroll/common/TiFastDev.java

    to the IP of the host where fastdev is started, recompiled the SDK and redeployed.

    Seems to work. A solution using tunneling or some setting in the android emulator to reroute 10.0.2.2 would be nicer though.
