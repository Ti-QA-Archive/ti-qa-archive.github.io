How to perform NTLM authentication between a Desktop Titanium app and a REST service

Within a Titanium Desktop app, I would like to reuse existing REST services hosted in an ASP.NET app requiring Windows (NTLM) Authentication.
If review the jQuery Ajax calls issued by WebKit through Fiddler (HTTP Proxy debugging tool), I see that the response is always a HTTP/1.1 401 Unauthorized.
I have even placed a crossdomain.xml in the root of the web app. When I compare successful browser based exchanges to the one issued by WebKit, I can see that WebKit is not passing the Authorization: NTLM XYZ header.
Does anyone have any suggestion for this? Or is this a limitation in Titanum-WebKit integration?
