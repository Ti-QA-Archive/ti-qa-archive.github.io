    Hi

    You have not said what fails to occur with this code that you believe should be happening.

    • You have provided a backgroundImage, but this does not form part of the height calculation - it only responds to the height required.
    • You have no views, images, labels or buttons added to the row in your code, so size will not be calculated
    • You have no default title property so this will also not adjust the row height

    I have no reason to assume you have a minRowHeight property on your table either.

    So as far as your provided code shows - a height of 0 should be determined.

    As I say - you have not said what problem you are seeing so I can only assume you are expecting it to have a height.

    • Keep in mind that the row heights set to SIZE do expand to the contents, so if you are using the techniques I explained in the link - then the child objects should be sized to the correct density for you and using numeric dimensions will accounts for spacing.

      Provided you place the correctly sized graphics in the correct folder using the correct scale factors then they will fit as you would hope - everything should fit into place.

      A couple of other tips;

      • If I know the size of the images or child views I use those for dimensions and positions
      • All numerical values are based on a scale factor of 1 (100%)
      • I always aim to use values that are divisible by 4, this way 75%, 100%, 150%, 200% ALWAYS ends up with a integer pixel value.
