Showing large number of ImageView(s) in ScrollableView

Hi everyone,

I need to make something like iOS default Photo app and I need to put a large number of photos to be shown in a ScrollableView. The problem is I need to store those images in array to be used by ScrollableView and the user may have few thousands of photos which means I will need to store few thousands of ImageView(s) in the array.

I haven't experienced this before but my logic says this will cause the app to crash because of excessive amount of memory used by the array. The problem is I don't have a device to test it.

I'm thinking to create on-demand loading but it's very slow since there are a lot of checking need to be done(image filename, image original album source, image date) whenever the image scroll to certain point, so I'm looking for faster alternative.

If anyone has some tips, please share your thoughts. Thanks in advance =)
