Android SDK OSX

I can't get Titanium Studios to recognize my installation of the Android sdk. Every time I try and point the IDE to my working android sdk directory, it complains that it can't find it. I was tasked with testing out this IDE and I can't even get the thing to work. I've done plenty of native development for both Android and iOS and think this platform is a joke but I have to work with it as it is expected from the people who make decisions.   • A temporary work around for anyone who's interested…

    Make a symbolic link from any of the existing platforms in your directory to android-7. If you're like me and your platforms folder looks more like 'android-2.1_r02-macosx' you will need to make a symbolic link for each of these platforms to the naming convention that Titanium Studio is expecting it to be stored under.

    Making a symbolic link in OSX is as simple as firing up the terminal app, and typing the following command :
    sudo ln -s /path/to/real/directory /name-titanium-expects

    This should only be used as a temporary workaround until they can fix this mess. When you're ready to get rid of the symbolic links, just navigate to the platforms directory again and type:
    sudo rm -f /name-titanium-expects
    This will only delete the link and it won't have any effect on the real directory it was pointing to.
