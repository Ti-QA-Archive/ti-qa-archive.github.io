Passing a controller (By ID) to a custom function -- error

Hi Q&A Community.

Just a quick question:

I have an external function that handles all my XHR requests. For now it is just an empty function.

/* XHR Helper - Returns parsed Response
--------------------------------------*/
require('Alloy')
exports.init_xhr = function(callback, failcallback, url, oController)
{

};

The plan is to manipulate the current controller by adding Activity Indicators etc in this function rather than adding them to every Window they may use this functionality.

The issue I have is calling it.

I have a simple view XML

<Alloy>
    <Window class="container" title="Home" id="home_window">
    </Window>
</Alloy>

And the Controller that calls the function looks like this:

/* = Page Includes
====================== */
Ti.include('lib/xhr.js');


/* = Callbacks
======================= */
function xhr_success(api_json) {
    alert("It Worked!!")
}

function xhr_fail(response) {
    alert("Server Error! Please try again");        
}

init_xhr(xhr_success, xhr_fail, Alloy.CFG.API_BASE_URL + '&method=get_affiliates', $.home_window);

Whenever I run this in the emulator I get the following error:

message = "*** -[__NSArrayM objectAtIndex:]: index 1 beyond bounds [0 .. 0]";

The error seems to be attached to the controller being passed and was wandering if anyone could shed any light on it. I probably doing something simple that I can't see but thought I would ask the experts.
