    • UPDATE: Regarding Slim's first issue with building an iOS app, there is a bug in Titanium 3.0's iOS build script that affects people with <ios><plist> sections in their tiapp.xml file or custom Info.plist where there are settings that have <dict> values such as CFBundlePrimaryIcon.

      A fix is pending review and release in v3.0.1.

      Diff here -> https://github.com/cb1kenobi/titanium_mobile/commit/2e21d9e1cafe47c8d9ab5e3eec102e38632f4962

      Ticket here -> https://jira.appcelerator.org/browse/TIMOB-12105

    • @Slim So you removed the <ios><plist> section of the tiapp.xml and it still errors? Are you using a custom Info.plist?

      Studio does log errors and debug info. You have to go into Preferences > Titanium Studio > Troubleshooting, then change Debug Level to "All" and check the "Debug specific components" box. From the list of components, select "com.aptana.core/debug/shell". That will show debug info for the CLI commands that get run.

    • @Slim, the code that tries to detect if a provisioning profile is a "distribution" or "adhoc" profile is based on whether or not it has any devices associated with it. This isn't a great way to determine the type, especially since enterprise adhoc provisioning profiles can have associated devices, but don't have to. We're gonna work on improving that.

    • @Frank, there is a bug (TIMOB-12105) in v3.0.0 where if the plist has a value of type <dict>, it will break stuff. The only key that appears to have a <dict> value that I know of is CFBundleIcons. If you have that in your custom Info.plist file, try removing it and see if it works. The bug has been fixed and we're waiting to get it reviewed and tested for the upcoming 3.0.1 release.
