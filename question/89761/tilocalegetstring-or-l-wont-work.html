Ti.Locale.getString() or L() won't work

Hello,

I just found out about the localization system of the 1.5.0 SDK but I can't make it work.

I did exactly what Jeff tells us to do in this blog post: http://developer.appcelerator.com/blog/2010/08/dealing-with-multiple-screens-and-multiple-languages.html

But the app won't find the string.

My code:

app.js

//
// home
//
var win1 = Ti.UI.createWindow({  
    title: L("hotel"),
    backgroundImage: 'images/win1/bg.png',
    url: 'windows/hotel/win.js',
    barColor: '#1b1b1b'
});

/i18n/en/strings.xml

<?xml version="1.0" encoding="UTF-8"?>
<resources>
    <string name="hotel">Hotel</string>
    <string name="media">Media</string>
    <string name="situation">Location</string>
    <string name="reservation">Book</string>
    <string name="contact">Contact</string>
</resources>

I'm using the latest build of 1.5.0, Titanium 1.2.2, did a full rebuild and got this message in the console:
> [INFO] Compiling localization files

But nothing, just getting the key "hotel" as result instead of Hotel, Bla or whatever I tried.

    If think I've found the problem, and it seems there's a BUG (I use a mac, with iOs sdk 4.2, Ti sdk 1.5.1 and ti 1.2.2) :

    i18n doesn't work if there's a space inside your project name. I've made 2 test projects. One named "nospace", and the other named "one space".

    i18n works with the first one (nospace), but doesn't work with the second one (one space). You can see the resulting problem here :

    http://img268.imageshack.us/img268/2029/1101130001.png

    there's 2 generated folders inside the build directory ("One Space.app" and "OneSpace.app"), what is the problem : there should be only one…

    So, because built ressources are splitted into several directories -> niet, kaput, nada, no localization…

    • If you're trying to fix an already created project, there's 3-4 things to check :

      1) the path name of the project's root directory (no space)

      2) the <name> tag inside the tiapp.xml file (no space too)

      3) the #appname: line inside the manifest file (once again, no space)

      4) remove and re-import this project inside the titanium developer app

      It's only after having done this, that i18n finally worked for me (even if I'm not sure they are all mandatory…)

      — commented January 13th 2011 by Christophe Soudron

    • And also, delete your build/iphone/build directory, to force a full rebuild…

  • I was having this same issue when trying to convert an application initially created on the 1.4.x SDK- looks like it could be a problem with the build directory as well as the tiapp.xml.

    Here are the steps to fix it:

    • Create a new project using the 1.5.x SDK
    • Open the new tiapp.xml file in your favorite text editor
    • Open your existing tiapp.xml file for your application as well
    • Copy non-app related tags from the newly generated tiapp.xml to your project tiapp.xml
    • Save your tiapp.xml
    • Delete the ../build/iphone directory
    • Rebuild your application

    This worked for me - hope it helps!
