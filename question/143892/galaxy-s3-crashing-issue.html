Galaxy S3 crashing issue

I have an Android app which misbehaves on a particular Galaxy S3 device, unfortunately that of the person I am writing it for. It does not misbehave on his wife's S2, my Arnova 7G3 tablet or the emulator. What happens is the app quits shortly after it's launched with the log showing this error:

2012-10-23 10:38:25.755 D 26071/TiUIView: (main) [180,4777] Nativeview is null

The app actually downloads a JSON set from a server via the HTTPClient, parses it for image names and then attempts to download any it has not already got within the app's resourcesDirectory. We then build a table using these image. It's at this point the app appears to quit.

I've looked on the web for what 'Nativeview is null' means and why the app should be doing 'Scheduling application restart' on my clients Galaxy S3 (but not on other devices as I've check their logs) but am none the wiser. So has anyone any pointers as to the cause of these crashes which appear to be device specific?
