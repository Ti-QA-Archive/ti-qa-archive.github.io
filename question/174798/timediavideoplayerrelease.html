Ti.Media.VideoPlayer.release()

My app is done in alloy.

In a window I have a VideoPlayer. Is it true that once the window is closed, the VideoPlayer is released from memory? Do I have to null the VideoPlayer's reference, and call the release() function when the window is closed?

  • Hello Shawn,

    Thanks for your concern.

    release() is not usually necessary but can help if you no longer need to use the player after it is used to help converse memory.

    Check the link for details: Ti.Media.VideoPlayer.release()

    You can use the release() method in your complete event like,

    videoPlayer.addEventListener("complete",function(e){
            Ti.API.info("Reason: " + translateCompleteReason(e.reason) );
            videoPlayer.stop();
            videoPlayer.release();
            container.remove(videoPlayer); // container is the window
            container.close();
            Titanium.UI.orientation = Titanium.UI.PORTRAIT;
        }
    );

function translateCompleteReason(reason){
        switch(reason){
            case Ti.Media.VIDEO_FINISH_REASON_PLAYBACK_ENDED : return "PLAYBACK_ENDED"; break;
            case Ti.Media.VIDEO_FINISH_REASON_PLAYBACK_ERROR : return "PLAYBACK_ERROR"; break;
            case Ti.Media.VIDEO_FINISH_REASON_USER_EXITED    : return "USER_EXITED";    break;
            default : return "UNKNOWN"; break;
        }
    }

    Hope it will help you.

    Thanks
