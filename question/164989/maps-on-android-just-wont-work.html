Maps on android... just won't work!!!

Hi Guys.
So.. ti.maps plugin is a breeze in iOS 7, it just works!
I come over to Android and its just a world of pain!

Ive followed the guides, updated my SDK, installed the google play services
' Extras folder, select Google Play services, then click Install X Packages….'

Ive gone into my run configs and created a new keystore, set a password.
Then i ran the 'keytool -list -v -keystore dev_keystore_new'
Took to SHA1 code and popped it into google API and regenerated a new key.

Ive added the module correctly, included it in the tiapp.xml file.
Ive also added the correct manifest snippet and changed it to my app id and API key.

now when i try to install the app on my device i get:

[ERROR] :  Failed to install apk on "TA92902IWQ"
[ERROR] Application Installer abnormal process termination. Process exit value was 143
[ERROR] Application Installer abnormal process termination. Process exit value was 143

And thats it…
If i remove the keystone, it installs on the device but won't run the maps and tells me the play services are not installed…

Whats going on???
Thanks

Adam.

  • Hi,
    Are you building the app with the same keystore you generated a key with? If not then make sure they are same. Also if you are testing during development just use the following way to generate SHA 1

    keytool -list -v -keystore ~/Library/Application\ Support/Titanium/mobilesdk/osx/3.1.0.GA/android/dev_keystore

    and then create a api key using above.

    Hope it helps.
