Titanium SDK version 3.5.1.GA, removeButton function only takes one argument after upgrade ?

Application type: mobile
Titanium SDK: 3.5.1 GA (3.5.1.96875c9)
Platform & version: iOS 8.1, OSX 10.10.2
Device: iOS simulator, iPhone 6
Host Operating System: OSX 10.10.2
Studio: Titanium Studio, build: 3.4.1.201410281727

Hi community

Since upgrading the SDK to 3.5.1 GA I have noticed an odd behaviour regarding the removeButton function.
This little piece of code, show the problem - it only removes the first argument in win.removeButton().

var win = Ti.UI.createWindow({
    backgroundColor:'white',
});
var button1 = Ti.UI.createButton({
    title: 'Button 1',
    top: 100
});
var button2 = Ti.UI.createButton({
    title: 'Button 2',
    top: 200
});
win.add(button1,button2);

function rem_buttons() {
    win.remove(button1,button2);
    clearInterval(timerInterval);
}
win.open();
timerInterval = setInterval(rem_buttons,2000);

If I change the code, to remove one button at a time, it works well.

function rem_buttons() {
    win.remove(button1);
    win.remove(button2);
    clearInterval(timerInterval);
}

Have you guys experienced the same thing ?
Best regards

    The SDK has never, to my knowledge, supported removing more than one view at a time using the remove method. If it did work, it was certainly undocumented behaviour. The current version of the source specifically checks for a single argument
