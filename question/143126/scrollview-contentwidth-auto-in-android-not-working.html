Scrollview contentwidth 'auto' in android not working

Application type : mobile

Titanium SDK : 2.1.3.GA

Platform : Android 2.2

OS : Mac OS X

Hello,

I'm trying to create an horizontal scrollview with an variable number of views in it. On iOS this works fine, but on Android it only shows the first three views, the other items are placed under the three views.

Here's the code i use:

var win = Titanium.UI.createWindow({
    backgroundColor : 'white',
    fullscreen : false
});    
var scrollView = Titanium.UI.createScrollView({
    contentWidth : 'auto',
    contentHeight : '90dp',
    height : '90dp',
    width : 'auto',
    top : '0dp',
    left : '0dp',
    layout : 'horizontal',
    backgroundColor : 'gray',
    scrollType : 'horizontal',
    showHorizontalScrollIndicator : true,
});
for (var i = 0; i < 10/*variable*/; i++) {
    var view = Titanium.UI.createView({
        height : '80dp',
        width : '90dp',
        top : '5dp',
        left : '5dp',
        bottom : '5dp',
        backgroundColor : 'yellow',
    });
    scrollView.add(view);
}
win.add(scrollView);
win.open();

Is this is a bug or am i doing something wrong?

    auto is deprecated. Use Ti.UI.SIZE instead. See this document for more details. That said, I don't know if that's really the problem here or not.

    Have you tried setting the horizontalWrap property on your ScrollView to false?
