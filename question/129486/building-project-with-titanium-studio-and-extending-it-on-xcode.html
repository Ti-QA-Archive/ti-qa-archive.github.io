      — commented December 16th 2011 by Ajeet pratap Maurya

    • When you run a titanium project you build, if you have a look in you project directory, in it you'll find a "Build" directory. In that directory you'll find the different platforms you ran your application on - one of them will be iPhone.

      In that iPhone directory you'll find xCode project. if you open xCode and run it though xCode it will run. You can browse around the project and change things - but that is not what you are supposed to do. What you are actually seeing in that project is the Titanium sdk that is being "joined" as one xCode project in order to evaluate you JS files and build native code out of them.

      It's perfectly ok for you to want to extend Titanium with xCode and do some things you can't do with the Titanium sdk. In that case what you would like to look into is how to extend titanium via native modules. What that means is that you write a module for titanium using native objective-c code, compile it, and bring it into your titanium project.

      have a look at this guide here: https://wiki.appcelerator.org/display/guides/iOS+Module+Development+Guide
