  • as a wowza user, your question is very interesting to me. I did a little digging, and I found the LivU app. The help page for this app gives some insight into how it works:

    • hardware h.264 encoder on iphone
    • ffmpeg 1.1 for RTSP streaming to wowza

    I think that you're going to have to go the route of a native module to do this. I don't know if there are any commercial modules already out there. This sounds like a lot of heavy lifting.

    I don't know much about OpenTok, but it looks like it uses its own protocol ("Rumor") over WebSockets. I guess if you were to use OpenTok to send the video to a server which then translates to RTSP, RTMP, or RTP, you could relay to a wowza server.
