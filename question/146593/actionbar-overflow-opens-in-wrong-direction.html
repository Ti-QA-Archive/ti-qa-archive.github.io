  • Codewise it's now like this:

    this.tab.addEventListener('focus', function(e) {
    var activity;

    if (core.isAndroid) {
        activity = m.ui.main.tabGroup.getActivity();

        activity.onCreateOptionsMenu = function(e) {
            var menu = e.menu;

            var helpItem = menu.add({
                title : L('help_label'),
                showAsAction : Ti.Android.SHOW_AS_ACTION_NEVER,
                icon: '/images/help.png',
                order: 2
            });

            helpItem.addEventListener('click', function(e) {
                m.ui.help.createWindow().open();
            });
        };
        activity.invalidateOptionsMenu();
    }
});

    As you might see i changed the place where I create the optionsMenu a bit, from the position where I acutally create the tabGroup to the tab.focus event. This also allows me to display different actionItems for each tab, dependent to the particular content.

    Another difference is, that i call activity.invalidateOptionsMenu() at the end, which is in my case necessary because of the changing actionBar content. Even if your actionItems are static all over your app i suggest you to try out putting this function at the end, maybe the re-render process will position the optionsMenu correctly.

    Also if you use a custom AndroidManifest.xml check your <application android:theme="xyz"> and <activity android:theme="xyz">
    I had some weird layout-bugs because of those and maybe they are also responsible for the actionBar to act like this.

  • Just figured it out. The cause is this in the manifest:

    <supports-screens android:anyDensity="false"/>

    Guess what the solution is! ;-)

    <supports-screens android:anyDensity="true"/>
