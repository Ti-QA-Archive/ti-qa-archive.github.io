Building simple app takes hours

After updating Snow Leopard to 10.6.8, XCode to 4.2 and Titanium Studio to 1.0.9 with MobileSDK 1.8.2 a clean build of a simple iPhone universal app with Appcelerator takes several hours (before it was updated it built clean within 2-3 minutes)! Even when compiling with XCode directly. It doesn't matter whether it's a clean simulator or release build.

A switch of used Mobile SDK in project settings hasn't changed anything. Also cleaning build folder, moving files to new project and so on.

The Titanium Console Window shows "[Info] Compiling localization files" during this time. Alternatively in XCode it shows preheader files compilation. The CPU consumption of the xcodebuild process (when using Appcelerator) is about 65%.

The build.log tail looks like the following without change for hours:

Any suggestions whats going wrong? Thanks a lot!
