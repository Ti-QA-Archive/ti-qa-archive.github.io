SDK 3.4.0-RC Build Fails

My organization has been developing successfully using 3.1.3.GA and Xcode 5, for iOS 7.x.

We use a crypto module (es.smartaccess.cryptolibrary) to manage the certificates for the encrypted web service. Have not had a problem for well over a year.

In preparation for iOS 8 we setup a sandbox to test compatibility of our code with iOS 8.

When trying to build as we have been for iOS 7, the following happens:

[ERROR] :  ** BUILD FAILED **
[ERROR] :  The following build commands failed:
[ERROR] :      Ld build/Debug-iphonesimulator/LV\ Dev1.app/LV\ Dev1 normal i386
[ERROR] :  (1 failure)

If I remove the reference to the crypto module, the project builds successfully but doesn't run because the crypto module is obviously needed.

So, other than the difference between the Ti SDK, Ti, Studio, and Xcode, our source is the same.

QUESTION: Why would the project build successfully under 3.1.3.GA but not for 3.4.0-RC? All signs point to the crypto module being the culprit, but it appears I am the only having a problem. Or maybe the only one using certificate?

Help would be very appreciated!
