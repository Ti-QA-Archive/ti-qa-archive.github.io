Memory footprint: Image Filesize vs Image Dimensions

Hi,

Is it more memory efficient to load smaller filesize images or smaller dimension images? I see the Image Best Practice section states "(640 x 480 x 3) / 1024 = 900 KB". Is the filesize irrelevant once loaded into memory? e.g. loading a smaller dimensions image is more efficient even if it's filesize is larger than a similarly sized higher dimension image?

http://docs.appcelerator.com/titanium/2.1/#!/guide/Image_Best_Practices-section-30082525_ImageBestPractices-Loadingandunloadingimagestomanagememoryuse

Cheers,

    Hi

    The file size is only relevant when you trying to save lots of images with little space.

    When displaying an image all OS have to,load the image into memory and that is the uncompressed version. So if you have two images with the same dimensions, but one is simple gradient and the other a detailed landscape - they both require the same amount of memory to reside in.

    Whilst there are some operating system performance tricks that are done when playing with a loaded image, a big image is a big image no matter how much detail.

    Btw keep in mind a PNG calc would be x4 rather than x3 due to opacity.
