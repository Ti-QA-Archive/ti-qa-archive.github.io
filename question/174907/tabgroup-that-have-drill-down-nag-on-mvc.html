    Hi

    I have quickly made the changes - should work - but let me know if there are any syntax issues.

    We have a few changes to make;

    dashboard.xml

    <Alloy>
    <Tab id="dashboard" title="dashboard">
        <Window id="dashboardWin">
            <TableView id="dashboardTable" onClick="openSub"> // moved
                <TableViewRow id="firstRow" action="anything"> // new property
                    // no need for touch
                    <Label id="firstLabel" touchEnabled="false" /> 
                </TableViewRow>
                <TableViewRow id="secondRow" action="something">
                    <Label id="secondLabel" touchEnabled="false" />
                </TableViewRow>
            </TableView>
        </Window>
    </Tab>
</Alloy>

    dashboard.js

    function openSub(e) {
    // you will want to know which is which
    // I have used a custom property 'action'
    // you can use what you want
    switch (e.source.action) {
    case 'anything':
        // do something specific
        break;
    case 'something':
        // do something specific
        break;
    }
    $.dashboardWin.open(Alloy.createController('firstSub', {
        // use this to pass properties across
        action: e.source.action
    }).getView());
}

    firstSub.xml

    // you are opening a window into an existing tab
// you do not provide a new tab - just the window
<Alloy>
    <Window id="firstSubWin", title="I will change">
        <TableView id="firstSubTable">
            <TableViewRow id="firstSubRow">
                <Label id="firstSubLabel"/>
            </TableViewRow>
        </TableView>
    </Window>
</Alloy>

    // firstSub.js

    var args = arguments[0] || {};
// none of this is needed

$.firstSubWin.open({
    transition: Ti.UI.iPhone.AnimationStyle.FLIP_FROM_LEFT
});
$.firstSubWin.setTitle(args.action);

    • I have put all this code into a tested app and here are the results that gives you a two tab app with the first tab including a table with two rows, clicking any row opens a new window, depending on which row the title of the sub folder is different - this demonstrates passing parameters.

      index.xml

      <Alloy>
    <TabGroup>
        <Require src="dashboard" />
        <Tab id="tab2" title="tab 2">
            <Window id="win2" title="win2">
                <Label>Hello</Label>
            </Window>
        </Tab>
    </TabGroup>
</Alloy>

      dashboard.xml

      <Alloy>
    <Tab id="dashboard" title="dashboard">
        <Window id="dashboardWin" title="dashboard">
            <TableView id="dashboardTable" rowHeight="50">
                <TableViewRow id="firstRow" action="anything" hasChild="true">
                    <Label id="firstLabel" touchEnabled="false" text="Row 1 - anything" left="20" /> 
                </TableViewRow>
                <TableViewRow id="secondRow" action="something" hasChild="true">
                    <Label id="secondLabel" touchEnabled="false" text="Row 2 - something" left="20"  /> 
                </TableViewRow>
            </TableView>
        </Window>
    </Tab>
</Alloy>

      firstSub.xml

      <Alloy>
    <Window id="firstSubWin" title="I will change">
        <TableView id="firstSubTable" rowHeight="50">
            <TableViewRow id="firstSubRow">
                <Label id="firstSubLabel" text="Sub Window" left="20" />
            </TableViewRow>
        </TableView>
    </Window>
</Alloy>

      index.js

      $.index.open();

      dashboard.js

      function openSub(e) {
    var title = 'Nothing Matched';
    switch (e.source.action) {
    case 'anything':
        title = 'Red';
        break;
    case 'something':
        title = 'Green';
        break;
    }
    $.dashboard.open(Alloy.createController('firstSub', {
        title: title
    }).getView());
}
$.dashboardTable.addEventListener('click', openSub);

      firstSub.js

      var args = arguments[0] || {};

$.firstSubWin.setTitle(args.title);

      I have not added any TSS classes but I have added a couple of in-line styles - I do NOT recommend this for a real app. Always use suitable TSS files.
