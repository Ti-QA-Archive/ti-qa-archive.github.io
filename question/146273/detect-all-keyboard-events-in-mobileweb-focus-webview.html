Detect all keyboard events in mobileweb, focus WebView?

This question has been asked in previous years, but I'm hoping someone has an answer now.

I have a mobile web Titanium/Alloy app, and I want the app to respond to keyboard events, for example going to the home screen when hitting escape, or responding to function keys, with the function keys having different functions when ALT is pressed. Although it's easy to do this in Javascript in a normal web page, I can't find any way to bind the keydown/keyup events in Titanium.

I've found one mechanism, using a WebView with keydown and keyup handlers on the document, but that only works when the WebView is focussed, that is you have clicked on it. So how do you programmatically focus a WebView? Or even detect if it is focussed?

Or is there some other way to detect keydown and keyup in Titanium?

  • Titanium doesn't have a formal spec for keyboard events. Furthermore, I'm unaware of ANY mobile browsers that actually fire keyboard events, so it's pointless. You'll have to listen for the blur event and perform a dirty check to see if the value changed.
