  • Hi John! Sorry for the delay, I've been out on holiday.

    One way to do this is to build a Mobile Web module. Unfortunately, there is no documentation and I don't have bandwidth to code you up a sample.

    In a nutshell, create a Mobile Web module and only use a single .js file. Multiple JS files do not work right now (it is supposed to work, but there's a bug). Next I would copy Ti/UI/TextBox.js from the Mobile Web platform directory. You'll need to rework it so that the domNode is a <form> tag, then place an <input> tag inside it.

    You will probably want to do some CSS tricks to make the field presentable. I'm not sure about mobile browsers, but in desktop browsers, I recall placing the <input> on top of a button and setting its opacity to zero, so nobody sees the field part of the file input and they think they are clicking the button below, but they're actually clicking the non-visiable "choose file" button.

    I have no clue how to invoke a custom module like this from Alloy, so this may not be the best method.
