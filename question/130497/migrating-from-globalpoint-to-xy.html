Migrating from GlobalPoint to X,Y

I have a drag script that works great (based on TiDrop), except it uses globalPoint. In Ti 1.8, globalPoint is now deprecated. So I was looking at switching to the new event.x and event.y. In my code, there is really no logic difference between using globalPoint and using e.x,e.y. I just calculate the difference between the starting point and the new point and then add this different to my view's location.

The problem is that where globalPoint dragged very smoothly, using e.x,e.y has a very "jumpy" look and the view does not keep up with the pointer position. It's almost like it takes e.x,e.y a short amount of time to update itself to the new position where globalPoint could update itself very quickly.

Has anyone else seen this happen?
