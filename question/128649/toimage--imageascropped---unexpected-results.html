toImage & imageAsCropped - Unexpected Results

I am trying to take a cropped screenshot on my iPhone app but it is giving me unexpected results. The screenshot comes out flipped (or reflected). Now the image that i am capturing in the window is a MaskedImage and this behaviour only exists with this Image Type. If i replace var image = Titanium.UI.createMaskedImage with var image = Titanium.UI.createImageView then the screenshot is exactly as on screen and not flipped. Code is below which can be placed straight into an app.js file to see what i mean.

Has anyone got any ideas on this? Unfortunately i need to keep the image as a createdMaskImage for later use.

Thank you in advance

// Create Window
window = Titanium.UI.createWindow({});

//Create loaded image
var image = Titanium.UI.createMaskedImage({image:'image.png', height:124, width:160, top:40, left:80});

//Create loaded image
var copiedImage = Titanium.UI.createImageView({height:124, width:160, top:230});

//Create rotate button
var rotateBtn = Ti.UI.createButton({title:'Rotate', width:100, height:35, top:385, left:170}); 

//Create Snap button
var takeSnapBtn = Ti.UI.createButton({ title:'Snap', width:100, height:35,top:385,left:50 });

//Rotate image function
function rotateImage(e){
    var t = Ti.UI.create2DMatrix();
    var spin = Titanium.UI.createAnimation();
    var degrees = -45;
    t = t.rotate(degrees);
    spin.transform = t;
    image.animate(spin);
}

//Rotate Clockwise Button function 
rotateBtn.addEventListener('click',function(e){ 
    rotateImage();
}); 

// Take Screenshot function 
takeSnapBtn.addEventListener('click',function(e){ 
    var screenshot = window.toImage();
    copiedImage.image = screenshot.imageAsCropped({x:80,y:40,width:160,height:124});
}); 

//Add image and buttons to window
window.add(image);
window.add(copiedImage);
window.add(rotateBtn);
window.add(takeSnapBtn);

//Open window
window.open();

    Hi,

    Here is fix that worked for me. (I had a different problem where the label added to the Masked image was upside down.)

    If you are using SDK 1.7.3 do the following. Modify the following file (assuming OS X):
    &#x2F;Library&#x2F;Application Support&#x2F;Titanium&#x2F;mobilesdk&#x2F;osx&#x2F;1.7.3&#x2F;iphone&#x2F;Classes

    Change line 71 from "YES" to "NO": 

    - (void)configurationSet
{
    [[self layer] setGeometryFlipped:NO];
}

    Change around line 137 (inserting the Translate and Scale lines):

        if (image != nil)
    {
      CGContextTranslateCTM(ourContext, 0, bounds.size.height);
      CGContextScaleCTM(ourContext, 1.0, -1.0);
      CGContextDrawImage(ourContext, bounds, [image CGImage]);
    }

    Clean and build your project and it should fix your problem.

    If you are interested see the following link for the explanation.

    In a nut shell instead of fixing the problem it looks like they just flipped the whole Geometry which had side effects that you and I experienced.
