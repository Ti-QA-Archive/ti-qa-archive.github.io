Javascript on Remote Webview not updating?

I'm using a webview on Android/iOS for my App built in Titanium SDK 3.5.0.

The webview is pointed to a remote location.
I found some issues with javascript which were fixed by changing the extension from ".js" to ".jsf" or something else. No idea why but it was mentioned in another Q&A to get javascript on a remote webpage to load that way.

Anyways, while the javascript loads, sometimes for some strange reason the Javascript is not updated.
Say if I want to alert something on the remote javascript and I go to view the app on android, the alert won't show up on the webview.

The alert will show up however if I reinstall the app. This happens about every few days or so.
I'm thinking it's some kind of caching issue.

How do I fix this? I always want it to load the most recent javascript file on the remote web server but sometimes it just doesn't load and it must be loading an old cached version or something.

The javascript is loaded from a simple <script> tag on the remote webview.

I do not want users to have to reinstall the app every time I make a change to the javascript file.

  • If you add random parameters after the request, that will make the browser think the request is unique. (ie cache-busting). eg

    http://www.mysite/com/myjsfile.js?v=32482374239478

    Then next time, change the value of v.
