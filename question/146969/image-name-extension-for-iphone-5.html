    Hi

    When using iOS there are two image densities you should supply and there is a simple convention for the naming (once you get used to it).

    If you have an image called example1.png with dimension of 300x300 suitable for iPhone that are not retina then you would ALSO create another version with a higher resolution called example1@2x.png with dimensions of *600x600 and place this file in the SAME folder.

    Your code will include the image file reference as below.

    var img = Ti.UI.createImage({
    height: Ti.UI.SIZE,
    image: 'example1.png', // JUST this
    width: Ti.UI.SIZE
});
win.add(img);

    None retina iPhones will show the file example1.png and retina versions will AUTOMATICALLY use example1@2x.png if the file exists if not it will use the lower resolution version. This is automatic behaviour of iOS.

    You can of course specify the dimensions if you wish; my code would change to;

    var img = Ti.UI.createImage({
    height: 300,
    image: 'example1.png', // JUST this
    width: 300
});
win.add(img);

    Having a file with the suffix @2x simply tell iOS that this is the file to use for retina if a matching prefix exists.

    The file extension does not matter as long as the match, so you could have used example1.jpg and example1@2x.jpg.

    • Just to clarify;

      • iPhone 4
      • iPhone 4s
      • iPhone 5

      All use the retina images @2x. All earlier iPhones used non-retina images.

      The iPhone 5 is taller but the only difference you will see is the splash screen uses a file name convention of default-568h@2x.png - so;

      • iPhone 4 = default@2x.png
      • iPhone 4s = default@2x.png
      • iPhone 5 = default-568h@2x.png

      All earlier iPhones simply use default.png.

    • As I said you either have retina or not retuna so iPhone is treated the EXACT same way as iPhone 4 and iPhone 4s

      Adding this code (from before);

      var img = Ti.UI.createImage({
    height: 300,
    image: 'example1.png', // JUST this
    width: 300
});
win.add(img);

      Place two images in your app example1.png and example1@2x.png (this one has twice the width and twice the height of the normal one.

      This code will then work for EVERY single iPhone it runs on. As far as images go the iPhone 5 is NOT a different case EXCEPT for the splash screen as explained above.

      The iPhone 5 does have a taller height so you have more space to play with though.

      if (Ti.Platform.DisplayCaps.platformHeight === 480) {
   // LAYOUT code if not iPhone 5

   // LAYOUT code if IS iPhone 5
