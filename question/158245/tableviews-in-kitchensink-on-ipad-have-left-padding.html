TableViews in KitchenSink on iPad Have Left Padding

Hello,

I'm reviewing the newest (or almost newest) KitchenSink on an iPad. I've noticed that the `TableView' widgets, for example the TableView on the home "Base UI" tab, have a little bit of space on the left between the window edge and the edge of the TableView itself.

Visually, the bottom border of each row in the TableView does not extend to the window edge. I think this looks bad and certainly does not look "native." It makes sense that the row label is padded a little, so the text doesn't touch the window edge. But the bottom border of each row should extend to the left window edge, just like the bottom border extends to the right window edge.

Can anybody explain why it is like that, and if there is a way to fix it?

Thank you!!!

-Ryan
