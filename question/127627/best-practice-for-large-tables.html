Best Practice for large tables

Hi, i'm new on Titanium and thinking of the best way to show a large (>1000 rows) table. The data should come via HTTP (JSON). As webdeveloper i would load the rows if he scrolls down, but what do you do in an app ?

  • There are a few options here.

    You can load the rows as they scroll down, which is a standard way, you can add paging so left and right, not quite so standard.

    But the key is not to download 1000 rows at once, and not to have 1000 rows in memory so if you had 10 on the screen you could have the previous and next 10 in memory to scroll to then load the rest etc.

    You only have limited resources on the phone and more than that could kill the app.

    Also as per other posts use classname to skin each row so that performance is increased.

    Hope this helps

    T.
