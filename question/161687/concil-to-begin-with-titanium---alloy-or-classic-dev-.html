  • Hello Jerome, when using Alloy MVC framework you could still develop 'classic' in your controllers, so the first does not exclude the latter.

    I'd suggest to go along with it, as it gives code a sane structure keeping logic well separated from presentation.

    Plus it offers backbone models and collections, sync adapters for data persistence, out of the box bindings for most of the user interface data-presenting elements, re-usable widgets […] and it is quite well documented.
