Working with git branches - How to compile different branch

I'm starting to use git to manage different versions of my app. Master branch is my release version and I have one branch (v1) for the next version. Inside this branch I'll do the developing but when there is a bugfix I want to merge it inside my master branch.

When I select v1 and edit my files and switch back to master (using the TI dropdown next to the project name) I don't see different files. All my changes are already in the master tree. How can I say I want to see the master branch without the changes from v1 or how can I compile the master branch and not v1?

    It sounds like you're not switching branches correctly. When you switch branches, the files & contents should reflect the state of the active branch. You didn't mention what you're using as a git client. Perhaps you should investigate some different tools. SourceTree (from Atlassian) is free; GitTower not free, and Git for Mac/Win from GitHub are three tools you might want to look into. Also, check out some git guides on branching, e.g. https://help.github.com/articles/branching-out
