Problem loading main app with storekit when not online

titanium mobile sdk 2.1

iphone sdk 5.1

Hi,

I have a problem with my apps with storekit on it. When there is no or low network connection, it takes a very long time for the app to load and then error shows 'unable to talk to apple' which is from storekit. It is only then that the main view of the app is displayed.

Can anyone please tell me what's the best way to handle this. I've seen other apps that has storekit. what it does it displays the main page first and then load the storekit after. I tried moving the 'include' to the later part of the app.js but this didn't work. It would appear that titanium attempts to load everything first before viewing the main page.

any help is greatly appreciated.

    You don't actually need storekit until a purchase is to be made, correct?

    Do all your purchase/storekit calls in a commonJS module that isn't loaded/required until an event takes place (button press, etc).
