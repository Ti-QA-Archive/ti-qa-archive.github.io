Android not loading right language

I have an app that uses strings.xml. I have two languages in my app: English and norwegian.
My folder structure looks like this

app
   platform
      android
         res
            values
               strings.xml
            values-no
               strings.xml

On iOS the strings.xml files are placed inside of the i18n folder. Everything works fine on iOS. On Android however, the app chooses english as language even if the phones's language is norwegian. Why is it not choosing norwegian instead?

Worth noting is that all my strings in the app is located in strings.xml, but even before i had added the platform folder to my app the app did load the english strings on Android, probably from the i18n folder.

  • Hi Anton Gildebrand,

    Make sure you have cleaned the project since you have started using the platform folder and updated strings as these may not become available until you have.

    Menu > Project > Clean > Choose Project > Confirm.

    There is an internationalisation guide you can follow this for language support.
    Be aware that the strings will ONLY display the correct language files when the device is set to use that language - this is not something you set, you simply react to the users choice of language.

    Thanks.
