Failed Validation in iTunes Connect with URL scheme?

This appears be something iTunes Connect has changed? Nothing in our app has changed other than selecting the latest Ti SDK (2.1.3.GA) so our app runs on iOS6.
The validation is shown as "The following URL schemes found in your app are not in the correct format: [actionpacked_marketing]… Please see RFC1738 for more detail."

Can anyone shed some light on what we might do to correct this? The only URL's in our app are simple links to our other apps in the iTunes Store. Not sure what else it could be.

    This Q&A will be relevant to you.

    In a nutshell, the url scheme allow for inter-app communication (see the above link). It is in defined in your Info.plist file which is generated for you. See the build/iphone subdirectory. You will need to change that. You can copy and edit the Info.plist file to you project directory.

    • Thanks David. That corrected the URL validation issue I was having. After reading your suggestion, I noticed that in the info.plist, my app name did not match the bundle URL scheme. So I copied the info.plist to the project root, then changed the following:
      From this….

      <key>CFBundleURLName</key>
    <string>com.mysite.myappname</string>
        <key>CFBundleURLSchemes</key>
        <array>
            <string>my_appname</string>    
        </array>

      to this…

      <key>CFBundleURLName</key>
    <string>com.mysite.myappname</string>
        <key>CFBundleURLSchemes</key>
        <array>
            <string>myappname</string>    
        </array>
