Strange module/event error: "Result of expression 'this.trivnet.restart' [OK] is not a function.";

I've got an iOS app that incorporates an Objective C module. For various reasons, I've got Javascript layers that interact via Ti.API events, which eventually end up in a line of Javascript like this:

this.trivnet.restart();

"restart" is an Objective C method defined in the module:

- (void)restart { [httpServer dealloc]; httpServer = nil; [self startServer:nil]; }

Ordinarily, the app runs fine, with lots of events and API calls back and forth between the Javascript and the module.

However, there seems to be an edge case where the above function call results in this runtime error:

[WARN] Exception in event callback. { expressionBeginOffset = 3659; expressionCaretOffset = 3679; expressionEndOffset = 3681; line = 127; message = &quot;Result of expression &#39;this.trivnet.restart&#39; [OK] is not a function.&quot;; name = TypeError; sourceId = 276072968; sourceURL = &quot;file:&#x2F;&#x2F;localhost&#x2F;Users&#x2F;user1&#x2F;Library&#x2F;Application%20Support&#x2F;iPhone%20Simulator&#x2F;5.0&#x2F;Applications&#x2F;085464A7-C04B-4DA8-8801-DFEF14A2F294&#x2F;trivially-console-v2.app&#x2F;generated_js&#x2F;http_server.js&quot;; }

Where the source file/line in question point to the Javascript line above ("this.trivnet.restart").

So, I've run into "is not a function" error messages before… but this one caught my eye because of the "[OK]" part… usually, it's "[Undefined]":

Result of expression &#39;this.trivnet.restart&#39; [OK] is not a function

No part of my code is trying to execute, or return "[OK]"… so it seems that something weirder than usual is going on here.

This happens both in the simulator (iOS 4+ and 5) and on the device (iOS 5).

Has anyone seem this sort of thing before? Ring a bell, anyone? Anyone?? Ferris??

Thanks much in advance - Steve

Titanium 1.8.0.1
MacOS Snow Leopard
Titanium Studio, build: 1.0.7.201112281340
