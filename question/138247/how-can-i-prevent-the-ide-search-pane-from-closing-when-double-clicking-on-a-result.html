How can I prevent the IDE search pane from closing when double-clicking on a result?

I think this is more of an Eclipse question, but here goes: I use find-in-files a lot to find occurrences of a text string across my project. When I single-click on a particular search result, it jumps to the file and line ONLY if that file hasn't yet been opened in a tab. If the file is already open, I have to double-click on the search result in order to jump to the specified file and line, but the double-click ends up closing the search pane, which is the incorrect behavior about 95% of the time. Generally, I want to investigate the context of the search result, and then continue looking at other search results.

This quirk of the search pane is rapidly melting my brain. Is there a setting I can change to prevent this behavior, or another way I can approach this?

I'm using Titanium Studio 2.0.1.201204132053.

    When you double click on a search result and the Search View goes away, look for the search icon around the corners of the Studio window. Next to the icon, you should see the 'double window' icon that says 'Restore' when you hover the mouse over it. Click the restore button, and the search view will be shown again. Now, try double clicking on a result. The search view should stay where it is.

    When you are ready to close the search view, just click the x on the tab. This will remove the view. The next time you do a search, the view should be restored to this previous location. From here on out, close the Search view with the X when you are finished using it.

    If you want to revert to the Search view minimizing each time you double click a result, look for the single horizontal bar icon on the right side of the search view. Click that to minimize.

    If you have the search view in a pane with other views, like console or terminal, you can drag it to other panes or create a new one for it just by dragging and dropping the view's tab.
