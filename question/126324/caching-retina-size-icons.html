Caching retina size icons

Hi,

I have an iPhone app that uses custom tableview rows with icons in imageViews within the rows.

I currently have a function that saves these icons into a cache folder. It does this using the onLoad event for the imageView. I'm saving the image using the myImageView.toImage(); function.

Here's the issue, I'm using the hires: true parameter so that when the imageView loads from a URL it displays in glorious retina resolution on the iPhone4. however, when it caches, it saves the image in the standard resolution for the imageview.

for example, take this imageView:

var myImageView = Ti.UI.createImageView({
    url: 'http://www.mysite.com/myimage_60@2x.png',
    height: 30,
    width: 30,
    hires: true
});

The PNG is 60px square. The imageView displays it at it's full size in a 30px box for the retina screen when loading from the URL (that's what hires: true is doing as far as I'm aware.)

When I save this image using myImageView.toImage(); it saves it at 30px square instead of 60px square. I've tried using myImageView.toBlob(); but this doesn't seem to work at all.

Basically, I just want to save icons at retina size, without doing a manual HTTP request for every icon (hence using the onLoad event for the imageView)

Any help would be greatly appreciated. Thanks!

  • Seems like the easiest way to address it would be to create it at fullsize, save it with toImage(), then shrink it down.

    var myImageView = Ti.UI.createImageView({
    url: 'http://www.mysite.com/myimage_60@2x.png',
    height: 60,
    width: 60,
    hires: true
});

// save with myImageView.toImage()

myImageView.height = 30;
myImageView.width = 30;

// add myImageView to your view hierarchy
