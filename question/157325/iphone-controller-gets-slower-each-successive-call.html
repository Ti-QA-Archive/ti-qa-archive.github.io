iPhone Controller Gets Slower Each Successive Call

Really hard to show the code as this is basically a finished app now.. but the problem is I have a controller which loads a TableView, and each time it gets called, it takes longer and longer each time to open. (it has been closed prior to each successive call)

Does that point to any common problems that I should look into? I've ripped up the code many different ways now to try to figure out what piece of code is causing it, but I can't seem to find what's doing it.

Just looking for an advice on what this could mean and any links or tips that might help me get to the bottom of it! Thanks!

    I'm 80% sure you are not freeing the memory the right way, the table stays there even if you close the window.

    however:

    • are you loading the table based on an app-wide event? Do you remove the event or the table loads +1 times each time you load the window
    • are you altering the table data somehow? so each time you open the window you actually load a multiple of the initial number of rows?

    the best way to find what it is, is to open instruments and see where it leaks :)
