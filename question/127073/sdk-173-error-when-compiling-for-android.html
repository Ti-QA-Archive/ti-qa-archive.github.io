  • If you try to compile an already existing app, then it throughs this type of error (I faced also this).

    What I did was to:

    • backup the build/android/* contents.
    • delete all contents from build/android folder
    • compile the app again.
      This will rebuild the app and should run normaly.

  • Thanks for your answer, but I already tried this without success.

    I resolved my problem by "reseting" the rights on my home folder (including hidden folder) (I'm on MacOS)

    I used this command:

    sudo chown -R user .[!.]*

    Problem solved!
