  • [SOLVED] MacOSX Moutain Lion 10.8.5

    From documentation:

    npm set [key] [value] --global

    What i did,

    sudo npm set prefix &#39;&#x2F;usr&#x2F;local&#x2F;etc&#39; --global

    thats all.

    reference: https://npmjs.org/doc/config.html

  • Solved:

    All I did was export 

      `PREFIX=/usr/local/etc`

    in my ~&#x2F;.bashrc. Right after doing that I ran 

      `npm config get prefix`

    and it gave me 

      `/usr/local/etc`.

    The inspiration to set the PREFIX variable came from here.

    Curious: The "prefix" key appears in my &#x2F;usr&#x2F;local&#x2F;etc&#x2F;npmrc, with a long value starting with [sudo] password for… and going several directories deep… the funny thing is, the directory it points to does not exist.

    But the update worked!

  • Here is the fix if you into this problem, having the update pop up continuously appearing asking you to update what you already updated.
    Then getting the error message : 

    Error encountered during update
  Failed to get value of npm config key prefix
  Failed to get value of npm config key prefix
  Failed to get value of npm config key prefix

    I applied Rohit Chatterjee solution, but that wasnt enough.
    Just to clarify, here s what you need to do :
    edit the file ~/.bashrc
    and add at the end :
    export PREFIX=/usr/local/etc

    then logout and login into your session.

    It was not enough for me, I also had to do that :

    echo ''>/usr/etc/npmrc

    to empty this file, which seems to hold some random config for nodejs.
    Anyway, that solved it and it stopped asking me to update what already updated !

  • Despite the fact that node and npm were previously installed on my system, and I was able to do things like "node -v" and "titanium -v" from a "normal" linux terminal, Titanium was not recognizing the existence of any of these programs. I tried various combinations of inserting an export statement into my bash profile, uninstalling and reinstalling node, (via apt-get and manually extracting the tarball, etc) and so on, but nothing seemed to work.

    What I ended up doing was launching my Titanium project, and I ran "node -v". It gave me version 0.6.blah.blah. I am still confused why this is… but to solve it, using the terminal WITHIN titanium, I issued all the following commands:

    (before I go on, if you're going to try this yourself, this answer might be incomplete or slightly inaccurate: i'm sorry that I didn't write it down exactly as i went… but this is close. The point to this solution is that I had to do it FROM WITHIN titanium).

    sudo apt-get remove npm
    sudo apt-get remove node
    wget http://nodejs.org/dist/v0.10.18/node-v0.10.18.tar.gz
    tar -zxf node-v0.10.18.tar.gz
    cd node-v0.10.18
    ./configure && make && sudo make install

    RESTART TITANIUM then, again in a titanium terminal, run

    npm install titanium
    npm install alloy

    Those last 2 commands are what really got me over the hump. My hello world application, and my android emulator, now successfully launch. Hope that helps someone. If anyone can explain to me why I had to do it this way, I'd like to understand it. Otherwise, thanks & good luck!
