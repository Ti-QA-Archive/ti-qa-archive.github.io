Defining ImageView in JS or XML in alloy

I have six image views need to be displayed in one single screen & image sizes are very large.
Now My question is, should we create the image views in js file or should we define it into xml in alloy, as far as memory issue is concerned?

  • No difference at all. I've seen questions along a similar vein before so let me explain the difference between 'in js' or 'in commonjs' vs 'in alloy'.

    Alloy is a structure that is used to define the components if a titanium application. The information that is stored in Alloy controllers, models, widgets, etc are all run through a pre-compile that creates an output that is pure javascript just like a 'classic' application. You can see/confirm this by looking at the resources directory that is created when you compile an Alloy app. The additional benefit of the Alloy structure is that the code that is generated follows the commonjs approach to code writing. So you get the benefit of good code structure.

    So, since they are compiled down to THE SAME code, there is NO DIFFERENCE between defining an object in Alloy or js.

    Ray
