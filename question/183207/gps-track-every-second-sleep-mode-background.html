GPS track every second, sleep mode, background

I'm creating an app for android and iPhone, to track time and position every second, ex:

  • time: 2015/10/02 - 11:21
  • latitude: 37.78583526611328
  • longitude:-122.40641784667969

But when the app goes to background or the screen goes to sleep, this geolocation event's stop working until I turn on the screen, or select the app again.

So I added this to my tiapp.xml file:

 <key>UIBackgroundModes</key>
<array>
    <string>fetch</string>
</array>
<key>NSLocationAlwaysUsageDescription</key>
<string>Get Always Locations</string>

It's that enough?
I've found this for android, should I add this to my tiapp.xml file?

    <android xmlns:android="http://schemas.android.com/apk/res/android">
        <manifest android:versionName="1.0.0">
            <uses-permission android:name="android.permission.WAKE_LOCK" />
        </manifest>
        <services>
            <service type="interval" url="myservice.js"/>
        </services>
    </android>

This will block the power button function or home button?

    Ti.App.idleTimerDisabled = true; // to disable going to sleep
    Ti.App.idleTimerDisabled = false;

    For iOS you should add this to your tiapp.xml file 

    <key>UIBackgroundModes</key>
<array>
    <string>location</string>
</array>

    In android is not necessary to add anything to the tiapp.xml file. For as long as your app is updating location it shouldn't pause even if you send it to the background or turn the screen off. At least it works that way for me.
