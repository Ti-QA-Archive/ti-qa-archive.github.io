Having a problem submitting my app to the app store

Hey guys,

I'm beaten and bruised from this app submission process. Finally after getting all of my certificates and everything in place to build the app for app store distribution, I'm now stuck on the actual upload process.

I was reading that you have to use this Application Loader program to upload it, but it won't allow me to select the .app file or anything. I just tried zipping it and it'll let me submit that, but then comes back with a bunch of errors.

The weird thing is that whatever folder I select to output the distribution file to, nothing shows up.

I also noticed in the XCode organizer that whenever I do a build it shows up under "Archived Applications" and has a line item for each time I build. When I try to submit through there, it requires that I enter my login information and then after a few seconds throws this error.

Uncaught Exception:

Stack Backtrace:

Does anyone have any idea what I'm doing wrong? I'm trying to get this app uploaded before they shut down the approval process for the holidays. Any help is greatly appreciated. :)

  • I'm using the latest version of Titanium Mobile and the iOS 4.2 SDK. For some reason XCode wouldn't submit the app, so what I had to do was go into the Library -> Mobile Devices -> [GUID] -> and compress the appname.app file into a zip and upload that using the separate Application Loader app.

    The weird thing is that TM's info.plist.template automatically puts a icon@2x.png requirement in the info.plist so I had to create a second icon for it to go through. But either way it worked and my app is now submitted and will hopefully be approved soon!
