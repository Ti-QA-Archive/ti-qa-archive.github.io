    Hi GCM means google cloud messaging and its used for android platform to send push notification:

    Implement Google Cloud Messaging (GCM) :-

    1 . Creating a Google API project –

    a. Open the Google Cloud Console.

    b. If you haven't created an API project yet, click Create Project

    c. Supply a project name and click Create.

    Once the project has been created, a page appears that displays your project ID and project number. For example, Project Number: 73450094152.

    d. Copy down your project number. You will use it later on as the GCM sender ID.

    2 . Enabling the GCM Service –

    a. In the sidebar on the left, select APIs & auth.

    b. In the displayed list of APIs, turn the Google Cloud Messaging for Android toggle to ON.

    3 . Obtaining an API Key

    a. In the sidebar on the left, select APIs & auth > Credentials.

    b. Click on Create New Key under Public API Access.

    4 . Credentials : The IDs and tokens that are used in different stages of GCM to ensure that all parties have been authenticated, and that the message is going to the correct place.

    a. Sender ID : A project number you acquire from the API console. The sender ID is used in the registration process to identify a 3rd-party application server that is permitted to send messages to the device.

    b. Application ID : The Android application that is registering to receive messages. This ensures that the messages are targeted to the correct Android application.

    c. Registration ID : An ID issued by the GCM servers to the Android application that allows it to receive messages. Once the Android application has the registration ID, it sends it to the 3rd-party application server, which uses it to identify each device that has registered to receive messages for a given Android application. In other words, a registration ID is tied to a particular Android application running on a particular device.

    d. Google User Account : For GCM to work, the mobile device must include at least one Google account if the device is running a version lower than Android 4.0.4.

    e. Sender Auth Token : An API key that is saved on the 3rd-party application server that gives the application server authorized access to Google services. The API key is included in the header of POST requests that send messages.

    5 . Download the GCM module from the link.

    6 . Write these properties tag in tiapp.xml

    <property name="com.activate.gcm.sender_id" type="string">409351601301</property>  <!-- put your sender_id here -->

    <property name="com.activate.gcm.icon" type="int">2130837504</property> <!-- get this from R.java, but converted to decimal, 0x7f020000 give you 2130837504 (type "0x7f020000 to decimal" in google) -->
    <property name="com.activate.gcm.component" type="string">gcm.gcm/gcm.gcm.GcmActivity</property> <!-- [package name]/[package name].[activity name] -->

    7 . Include module tag in tiapp.xml

    <modules>

<module platform="android" version="0.5">com.activate.gcm</module>

</modules>

    Let me know if you want anything else.
