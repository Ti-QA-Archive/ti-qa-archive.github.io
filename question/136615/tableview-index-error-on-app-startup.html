TableView Index Error On App Startup

Hi everyone,

I'm brand new to Titanium and mobile development in general. I'm working a iPhone app using the 1.8.2 Titanium SDK. I have a two tab application. The first tab is the main app, and the second tab is an options screen that contains a table view.

I am doing some application testing on my iPad 2 (only iOS device I own besides the simulator), and about 15% of the time on app startup I get the following error:

'Result of expression tableView.data[0] [undefined] is not an object at app.js……. line X.

Which leads me to believe that I'm trying to use the data in the tableView before it's done initializing or something. Here's the offending code:


var inputData = [
    {title:'Lowercase'},
    {title:'Uppercase'},
    {title:'Uppercase & Lowercase'},
    {title:'Match Lowercase & Uppercase'},
    {title:'Vowels'}
];
var tableView = Titanium.UI.createTableView({
    data:inputData,
    style:Titanium.UI.iPhone.TableViewStyle.GROUPED,
    backgroundColor:'transparent',
    rowBackgroundColor:'white'
});

tableView.setHeaderView(headerView);
tableView.data[0].rows[0].setHasCheck(true);

As I mentioned, this error is very sporadic. More often than not the app starts up just fine. Other times not so much. I looked at the docs for tableView and I didn't see any obvious 'initialized' event. I'm hoping you guys can give me some direction on the proper way to do this initialization.

My end goal is just to get the first item in the tableView to be checked on app startup. Thanks for your help.

    Do this on your json array

    var inputData = [
    {title:'Lowercase', hasCheck: true},
    {title:'Uppercase'},
    {title:'Uppercase & Lowercase'},
    {title:'Match Lowercase & Uppercase'},
    {title:'Vowels'}
];
