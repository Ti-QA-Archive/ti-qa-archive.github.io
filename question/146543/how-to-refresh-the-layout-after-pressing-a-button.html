  • First, you really need to provide all the info about your dev environment when posting a question like this. If you have any questions regarding what info needs to be posted please see the Ask A Question Page for more info.

    Second, your code is quite scary. You are drilling down into children of children of views to remove things and there is no way for anyone reading this to know what is what. The screen orientation change does not refresh the view, what is most likely happening is that the orientation change moves some left over parent out of the way of the view you are trying to click on.

    The best thing you can do is come up with a small test case that reproduces your problem. The code should be easily copied and pasted into a new project so that it can be tested in correlation with your dev environment.
