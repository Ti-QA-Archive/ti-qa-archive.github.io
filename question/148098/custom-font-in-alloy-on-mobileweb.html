Custom font in Alloy on MobileWeb

I have an Alloy project that is delivered to Mobile Web. It has a custom font installed in

app/assets/fonts/Test Font.ttf

I am using the following in the .tss file, but get nothing. If I change it to a font that Titanium knows about, eg Helvetica, it works. But any other font, whether or not it is in the project or installed in the operating system, does not work.

font: {fontSize:"40", fontFamily:"Test Font"}

So how do you do custom fonts in Mobile Web with Alloy? How do I tell Titanium that a custom font exists?

BTW, the HTML rendered by Titanium looks like this, with no reference to font:

<div class="TiUIElement TiUIView TiLayoutsComposite" data-widget-id="Ti.UI.View:19" style="white-space: normal; color: rgb(56, 198, 100); font-size: 40px; left: 0px; top: 0px; width: 364px; height: 46px; ">Test text here</div>

But with another font that Titanium DOES know about, the font is included:

<div class="TiUIElement TiUIView TiLayoutsComposite" data-widget-id="Ti.UI.View:58" style="white-space: normal; color: rgb(56, 198, 100); font-size: 12px; font-family: 'American Typewriter'; text-align: right; left: 0px; top: 0px; width: 200px; height: 14px; ">DATE MODIFIED:</div>

  • Accepted Answer

    Titanium Mobile Web only supports otf and woff fonts. At the time I built it, there was little-to-no support for ttf fonts, so I never even thought to implement them.

    I've created a ticket to add support for ttf fonts and have scheduled it for the 3.1.0 release. It missed the 3.0.2 release by a couple weeks. The ticket is here: https://jira.appcelerator.org/browse/TIMOB-12664.

    — answered February 11th 2013 by Chris Barber

  • ..and here is the answer: just as in CSS, a font name with spaces must be quoted. Luckily, single quotes will do, so you can use this (of course the &quot; is a double quote):

    font: {fontFamily:&quot;&#39;Perfect DOS VGA 437&#39;&quot;}
