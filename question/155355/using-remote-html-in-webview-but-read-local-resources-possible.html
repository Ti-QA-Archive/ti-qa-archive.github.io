Using remote html in webview but read local resources, possible?

Ok, so we wish to use a webview, read the html from a website (since it is dynamic) and use webview.setHtml() on the webview so we can use titanium events (as it is local then) directly from the html code to access app functions.

What we want is to have all images and heavy stuff reside in the Resource directory of the app instead of reading it from the website. Is this possible?

We've created a Resource/html directory and put our images in there.

Suppose our web is at ourweb.com/page.html, we read that into a string. Then call webview.setHtml(page)

In our page, this all works fine if all urls to images etc are fullpath to the website such as http://ourweb.com/map.png.

If we put that map.png into our Resource/html directory in the app, how do we write the url in the webpage to be able to access that? I've tried almost everything, nothing seems to work.
img src="map.png"
img src="html/map.png"
etc.

I just get errors in titanium logcat such as:

Error loading /html/map.png
Error loading /Resources/html/map.png
Error loading /map.png

Any idea what to write in the image src on the webpage?

  • After some messing around i found a solution i think:

    On iphone, the links should be set as:
    app://Resources/html/map.png

    On android the links should be:
    file:///android_asset/Resources/html/map.png

    It seems to work.
