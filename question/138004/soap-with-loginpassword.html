SOAP with login/password

Hi,
in my case the communication to server uses Basic Authorization.For this I patched suds.js. This was easy ;-)

if (config.username && config.password) {
     xhr.setRequestHeader('Authorization', 'Basic ' 
           + Ti.Utils.base64encode(config.username + ':' + config.password));
}

Next step: Login into server on soap level. The server returns a sessionID for further comminication.
