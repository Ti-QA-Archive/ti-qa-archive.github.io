Empty/Null YQL response while working fine on YQL Console

I tried to run the following code on iOS 64 bit simulator using Ti 3.1.3 and iOS 7

Ti.Yahoo.yql('select * from xml where url="http://news.yahoo.com/rss/topstories"', function(e){
    alert(e);
});

The alert popped-up, but I received empty response back. It looks like the "alert dialog" has a lot of content/data in it because the height of the alert dialog fills the screen and it seems that I can scroll the alert dialog view but without any data in it.

I checked my YQL statement on YQL Console and I did got a response back with data in it. Here's a link to the YQL console with the same YQL statement: YQL Console Link

I then changed the YQL statement to something different, for example, I tried to pull some data out of flickr

Ti.Yahoo.yql('select * from flickr.photosets.photos where photoset_id ="PHOTOSET_ID" and api_key = "MY_API_KEY"', function(e){
    alert(e);
});

and this time I did receive a nice response back with a matching data to the one I did through YQL console.

Any thought?

  • Hi

    You are trying to show e which is the response object for YQL, this is not the actual answer. You can show the entire object in an alert if you wish - but it will be big and you need to convert to a string to see it correctly.

    // instead of
alert(e);
// try
alert(JSON.stringify(e));

    The results are actually pass on the data property;

    alert(e.data);
// might be easier to read
alert(JSON.stringify(e.data));

    Personally I prefer to use httpClient with the YQL string instead of the YQL method as you can control the timeout and better handle failures.
