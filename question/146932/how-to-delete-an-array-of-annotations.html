How to delete an array of Annotations

Hi,

I'm having lots of trouble removing an array of annotations on my mapview. I already asked about it in this thread earlier today, but decided to start a fresh thread.

Basically, what the map section of my app does is displaying various locations on a map. Throughout my app, and inside the database, these locations are divided into different categories. Each category has a given ID, called categoryID. On my mapview, I want to display all annotations of all the categories on the map, but I want the user to hide and add each category of annotations to get a more details view.

I've managed to split the annotations into different arrays, dividing them into the different categories. I now want to be able to hide and show each annotation array individually. It's the hiding part that doesn't seem to work. Simply placing mapview.removeAnnotations(foodAnnotations); into my button event listener doesn't seem to work. I AM able to display an individual annotation array, but only AFTER I've removed all annotations. So basically, adding individual annotation arrays works, removing an individual annotation array doesn't. I've tried multiple solutions but nothing seems to work. Any care to help? Thanks!

  • There is a stupid bug in removeAnnotations objective-C code:

    Lines 312-331 of TiMapViewProxy.m must be:

    -(void)removeAnnotations:(id)arg
{
    ENSURE_SINGLE_ARG(arg,NSArray)
    for (id ann in arg) {
        if ([ann isKindOfClass:[TiMapAnnotationProxy class]]) {
            [self forgetProxy:ann];
        }
    }

    if ([self viewAttached]) {
        TiThreadPerformOnMainThread(^{
            [(TiMapView*)[self view] removeAnnotations:arg];
        }, NO);
    }
    else {
        for (id annotation in arg) {
            [self removeAnnotation:annotation];
        }
    }
}

    I just changed line ENSURE_ARRAY(arg, NSArray) to correct ENSURE_SINGLE_ARG(arg,NSArray). Otherwise arg is array with ONE element which is array of annotations. :) :) :) I think it is VERY VERY OLD BUG.
