Webview not allowing for file access using camera

I have simple HTML element that displays button and when clicked the camera window appears and allows taking pictures.
<input type = "file" id="capcam" capture="camera"" accept="image/*">

Now running a titanium app (on Android device) that has an embedded WebView, it does not work, Clicking on button does nothing.

But If I run the Android web browser directly, it works perfectly. The camera appears and I can take a picture. The problem is related to the Titanium WebView. Is there a permission issue?

  • Hi

    This "might" be that as the app itself doe not use the camera (a webView does), the camera permissions are not being added automatically.

    You should add these - at least the first one to the tiapp.xml > Android > Manifest section;

    <uses-permission android:name="android.permission.CAMERA"/>
<uses-permission android:name="android.permission.FLASHLIGHT"/>

    If the app does not HAVE to have the camera then you should also include this;

    <uses-feature android:name="android.hardware.camera" android:required="false"/>

    This was devices that do not have a camera or a suitable compatible camera can still use the app - rather than being excluded. Just keep in mind you will have to make sure your code handles the lack of a camera if you allow for optional camera usage.
