ListView does not have method setHeaderView() on Android

Hi,

I've just found out that method setHeaderView() does not exist for ListView when you run app on Android (on iOS the same code works fine).

As api doc describes this method should be available but when I run app on Android I get the following error (Android 4.2):

[ERROR] :  TiExceptionHandler: (main) [0,4429] - In alloy/controllers/index.js:27,16
[ERROR] :  TiExceptionHandler: (main) [1,4430] - Message: Uncaught TypeError: Object #<ListView> has no method 'setHeaderView'
[ERROR] :  TiExceptionHandler: (main) [0,4430] - Source:     $.listView.setHeaderView(view);
  • index.js
    ~~~
    var view = Alloy.createController('header').getView();

$.listView.setHeaderView(view);
//$.listSection.setHeaderView(view);

$.index.open();

- index.xml

<Alloy>
<Window class="container">
<ListView id="listView">
<ListSection id="listSection"></ListSection>
</ListView>
</Window>
</Alloy>
~~~

Am I missing something or there is mistake in api docs so ListView for Android does not support method setHeaderView() at all.

Curios note:
When I am trying to setHeaderView for listSection everything works fine.

  • Hi

    OK this is confusing, setHeaderView was not available until Ti SDK 3.2, so as long as you have that set in your tiapp.xml everything should be fine.

    But is seems likely you do have it set to Ti SDK 3.2 as you say that the iOS version is fine and that also gained this method at the same time.

    Please check anyway and report back.

    Also can you try using the direct property just in case that is any different (I also use the getters/setters - but just in case).

    //$.listView.setHeaderView(view);
$.listView.headerView = view;
