Best Design Patterns to use when Creating an App

About a year ago I created an app using titanium, it was rather procedural and there wasn't really any OOP in place. I moved away from titanium due to work constraints etc and now I'm back having another go, but a lot has changed. It seems to be written in commonJS? (the starter templates are anyway). At work I use an object literal approach when making websites so I'd prefer to use this if possible.

Basically I'd like to know what is best suited for this, or whether it matters at all?? I want to go down the best route even if it does mean learning a different way of programming…

Cheers, James

  • Well, in general you want to take an MVC approach. We have pre-release of Alloy which creates a framework for you and allows you to use xml for layout and .tss for styling, then you just use javascript for your controllers: http://developer.appcelerator.com/blog/2012/08/alloy-framework-is-ready-for-pre-release.html. If you would prefer to use commonJS and pure javascript, you can start with this scaffold app as an example: https://github.com/alanleard/scaffold-demo
