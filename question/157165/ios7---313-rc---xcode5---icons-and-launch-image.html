iOS7 - 3.1.3 RC - xCode5 - Icons and launch image

I've been reading and merging some docs from Apple and Appcelerator and although everything is working fine, I have some doubts about icon and launch image naming and size.

I need to support both iOS6 and iOS7 in my apps, so this is what I have working now.
Icons in resources/iphone folder:

appicon@2x.png / appicon.png  (114px / 57px)
appicon-60@2x.png / appicon-60.png  (120px / 60px)
appicon-72@2x.png / appicon-72.png  (144px / 72px)
appicon-76@2x.png / appicon-76.png  (152px / 76px)
appicon-Small-40@2x.png / appicon-Small-40.png ( 80px / 40px)

All 10 icons are included in the info.plist file.

There are some other icons that Apple recommends:

Icon-Small.png    29 x 29    iPhone 3+, iPad
Icon-Small@2x.png    58 x 58    iPhone 4+, iPad Retina
Icon-Small-50.png    50 x 50    iPad
Icon-Small-50@2x.png    100 x 100    iPad Retina

I tried to add these icons to my folder, but they were not included in the info.plist at build time, so I think Titanium does not take them into account.

So, the first question is: Are those 10 icons the only ones needed for every device? (for ios6 and ios7)

About the launch image, I have these files in resources/iphone folder:

Default.png (320x480)
Default@2x.png (640x960)
Default-568h@2x.png ( 640x1136)
Default-Portrait.png (768x1004)
Default-Portrait@2x.png (1536x2008)
Default-Landscape.png (1024x748)
Default-Landscape@2x.png (2048x1496)

I noticed that when the app runs on iOS7 devices, the launch image stretches a little.

This is stated in the Titanium iOS7 Migration guide I think:

"Update the launch image to include the status bar area if it doesn’t already do so. This is a difference of 20 pixels in height, that is, the image is stretched 20 more pixels if you do nothing."

So the second question is:

What should I do here? My apps are fullscreen… should I add 20px? substract 20px? If I do, will the image stretch in iOS6 devices?

Is there a way to have launch images for iOS7 and others for iOS6 ?

Thank you very much!

  • I had thought that landscape launch images were supported for the iPhone, but I guess that is not true. According to Apple's documentation, you have everything covered.

    iOS 6 and below: Human Interface Guidelines

    iOS 7: Human Interface Guidelines

    I wouldn't see a need for a non-retina version of appicon-60@2x.png, since iOS 7 does not support anything below the iPhone 4, but the other non "@2x" icons in iOS 7 are needed for the iPad 2 and the iPad Mini. All iOS 6 icons should be included, if you plan to support each platform.

  • Yes! I think newest versions of Ti.SDK (latest 3.x's) automatically add them in the build of your app. However if you are like me and still work with older versions of the Ti.SDK you need to change some stuff in the ~&#x2F;Library&#x2F;Application Support&#x2F;Titanium&#x2F;mobilesdk&#x2F;osx&#x2F;SDK_VERSION&#x2F;tiapp.py :

    Find the line which says: 

    iconorder = list(iconname+".png" ... etc

    and complete this with the following new icons (60.png, 60@2x.png, 76.png, 76@2x.png): 

    iconorder = list([iconname+".png",iconname+"@2x.png",iconname+"-60.png",iconname+"-60@2x.png",iconname+"-72.png",iconname+"-72@2x.png",iconname+"-76.png",iconname+"-76@2x.png",iconname+"-Small-50.png",iconname+"-Small-50@2x.png",iconname+"-Small.png",iconname+"-Small@2x.png"])

    This part in the tiapp.py generates the bundleiconfiles property in the info.plist file. IT DOESN'T WORK if you add them manually to a custom info.plist file as this tiapp.py will still overwrite them. So do it (carefully) in the tiapp.py file.

    Now be sure to add the new icon files (with maybe a flat design, that's up to you) to your project Resources/-root or project's Resources/iphone folder.

    Good luck!
