MVC Revisted

Well, after checking various articles and best practices, I ended up into a good structure for mobile apps.

First of all, I need to state that a mobile application is usually a compact application meaning that the code is compact and app developers don't (always) have to face the problems we have in desktop development.

I believe that in some cases MVC can make simple things complicated depending on the case of course. For this reason I would prefer a simple, custom yet well-structured model for my apps.

As you know most mobile apps could be represented by a tree-stucture diagram. Each node on the tree can represent a window containing it's views. For this reason I thought it could be a good idea to create this tree-structure diagram in folders under my Resources folder. So, let's say we have an eCommerce mobile app. I would do something like this:

Resources
- iphone
- android
- src
-- assets
-- common
--- ui
--- lib
-- app
--- home
--- products
--- services
--- contact us
--- members
---- registration
---- forgot my password
---- login

Now, the best thing of this is that below each node I always have three files.

main.js
ui.js
handlers.js

main.js contains include calls, usually ui.js first and handlers.js follows. In some cases where authentication is required it also contains the appropriate logic. In some others where timers are needed, I include them here also.

ui.js contains the obvious. If certain blocks are reused throuought the app, I place these in the common/ui and just call them with various arguments.

handlers.js includes all the logic. Button actions, form validations, calls to remote web services, etc.

So far I have developed 2-3 applications using this model and I have to admit that I find it very easy to use and maintain. Do you see any upcoming problems with that?

thank you
