Hello, can i use scripts that have been put inside the iphone and android resource folder in order to have 2 different implementation for the same named jsmodule? for instance, i've one menu.js for iphone and one menu.js for android, i'd like to put theme rispectively one into the iphone folder and one into the android one, then i just use require('menu.js') and have the right implementation based on the build device target. Any answer is really appreciate. It seems to work on the simulator but crashes on device.

  • in app.js, find the platform first and define the related require files directory. Test for syntax. I did not test this code.

    If (Ti.platform.name == 'android' ) {
    var jsPath = '/resources/android/'; }
    else {
    var jsPath = '/resources/iphone/';
    }

    where you are placing the require funciton, use
    require(jsPath+'menu.js');

    Actually, why do you want to keep android related JS file for iphone build? Actually, you just keep only platform related files when you are building/deploying the app. You may not need to manipulate the code if you keep only platform related files.

    Hope this helps.
