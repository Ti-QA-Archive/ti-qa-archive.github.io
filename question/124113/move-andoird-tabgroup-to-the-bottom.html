  • Accepted Answer

    As Paul said in the first Q&A you linked, the Android SDK native tab control always appears at the top (whereas the iOS native tab control always appears at the bottom). Neither SDK lets you easily change that – it is just the native "look and feel" for each platform.

    That said, people have come up with some inventive ways to coerce android tabs to the bottom of the screen when programming native apps in Java but they aren't perfect either. For example, the gray line at the base of the tabs which normally divides the tabs from the views is still at the base of the tabs (i.e. bottom of window) instead of above the tabs to separate from the views above the tabs. Here is one way of doing it in a native Java project.

    But I wouldn't really expect Appcelerator to implement something like that hack as a standard feature of Titanium.

    Aside from the workaround you mentioned, another alternative is how the sample app Tweetanium provides its own pseudo tabgroup which looks the same on both platforms.
