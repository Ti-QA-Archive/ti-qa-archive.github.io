Can't understand how Ti.include() works

From what I gather through the documentation, Ti.include will just copy paste the entire code of the file mentioned in the argument to this function.

However, it seems to be behaving in the opposite way.
In a file (lets say a.js), I'm including 2 files, which just do some processing and assign values to certain variables. These 2 included files do not have any windows.

According to these assigned values, I display stuff on a map in a.js
However, it seems to be that it executes a.js before assigning the values. I can verify this because I'm executing a.js several times.
So each time, the stuff displayed on the map is according to the values assigned the PREVIOUS time. It seems it is executing a.js first, and then assigning the values in the included files.

Any ideas on this?

  • Accepted Answer

    include just places the contents of the includes file into the calling script.

    a.js - 

    function addSome(n) {
  n += 10;
  return n;
}

    b.js -

    Ti.include('a.js')
alert(addSome(40))

    IMO an included file is something that sould only be included once. As opposed to 'code to be inserted in a function' - include the file at the top of your code, and call it's functions as needed.
