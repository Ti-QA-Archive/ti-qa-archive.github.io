Upgrading an old app (3 years old) to Titanium sdk 3.3.0 causes the app to crash upon launching on Android

My app was originally created about three years ago. When I tried to upgrade it from Titanium sdk 3.2.2 to Titanium sdk 3.3.0, I get an unhanded exception and the app force closes upon launching. I have tested on an HTC Incredible running Android 2.3.4 and a Nexus 7 running Android 4.4. I am using Alloy for most of the user interface. All of my initialization code executes without errors, but as soon as that code finishes, I get the dialog:
"Sorry! The application ___ has stopped unexpectedly. Please try again."
I dug through your release notes and I didn't find any red flags about converting any code or generated files. In my android manifest.xml, I have the following line:

<uses-sdk android:minSdkVersion="10"/>

I do not have any references to the android sdk in my tiapp.xml.

I have installed: Android SDK Tools 23.0.2 Android SDK Platform-tools 20 Android SDK Build-tools 20

Once I upgraded the sdk tools, I can no longer build with Titanium sdk 3.2.2 - it gives me an error about not finding zipalign.

— asked August 11th 2014 by Lisa Stump
