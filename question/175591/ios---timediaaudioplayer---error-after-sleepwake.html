iOS - Ti.Media.AudioPlayer - error after sleep/wake

I use Ti.Media.AudioPlayer to play remote mp3s. If I pause the player, lock the screen, wake it up and resume the playback, I'll get the error: "File Error: Unable to configure network read stream." This is a big issue. Any fix?

SDK 3.2.3 on iOS 7.

  • Do you have it set for background play?

    if (Ti.Platform.osname != 'android') {
    Ti.Media.audioSessionMode = Ti.Media.AUDIO_SESSION_MODE_PLAYBACK;
}

var audioPlayer = Ti.Media.createAudioPlayer({
    allowBackground : true
});
