A white box shows where my searchbar is hidden

I want to have my search bar hidden at the start of my app, not showing unless clicked to be shown by a button I have made. So I set my search bar -visible:false.

However, there is still a white box with the height of my search bar, above my view table.

var search = Titanium.UI.createSearchBar({
    backgroundColor:'black',
    showCancel:false,
    hintText:'Search course name',
    height:100,
    visible:false
});

//the data table
var TheTable = Titanium.UI.createTableView({
    top: mySessionsLabel.height,
    width:windowWidth,
    search:search,
    searchHidden:true
});

//show or hide the search bar
searchBtn.addEventListener('click',function(e){
    if(!search.visible)
    {
        search.show();
    }
    else
    {
        search.hide();
    }
});

  • Hi

    So far you have not mentioned your Platform target, so providing a definite answer is harder right now. Your screen shot gives away nothing about the platform either.

    As you are using the searchBar as a table filter rather than a separate search feature (nothing wrong with this) you are going to some limitations/unexpected outcomes.

    If you are targeting iOS then using ONLY the property you have already found; searchHidden is the right way to go. This brings your app in line with standard iOS behaviours. The is no need to have a show/hide button as that property will initially set the searchBar to touch under the top edge and only be revealed when pulled down. Users will already be familiar with this interaction as many built-in apps already exhibit this behaviour.
