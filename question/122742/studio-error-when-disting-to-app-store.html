  • This usually means that your provisioning profile is not correct.
    Go through the developer.apple.com process and make sure you have the appropriate profile for the app.
    If it is for distribution then you need a distribution certificate first and also a distribution profile.
    Then when you download the profile double click it to open Xcode, this will add it to xcode.
    Then you can use it in studio and it should work.

    • Hmm, things i would check are;-

      Does the profile show in Xcode organiser without an error logo? (if there is an error there then it is the profile at fault)

      Have you done a clean before a build from Studio?

      Does your development profile still work?

      Are you using the right certificate in the the actual build dialog?

      Did you make the profile with the right app id and certificate (rather than mix it up with a development cert).

      You probably have done all of these things, but failing all that all i can suggest is give it a break for an hour or two / maybe overnight and try again. Provisioning is a painful process for the first few times.

  • John,

    I've been there and had all the same problems. Here is how to fix assuming the provisioning file is all good.

    So you referenced it in TitaniumStudio.

    • The "No such file or directory" issue is solved by Double-clicking the provisioning file itself so it loads right in Xcode.

    • The "SystemExit: 1" issue (which doesn't say much) is solved by just cleaning the build/iphone directory, i.e. delete the content of the directory so you get a clean build.

    Hope that fixes it.
