ImageView / View rotation problem

I was trying to build a simple demo of accelerometer API with an Android robot who will wave it's hands and feet when accelerometer was triggered. By using the Titanium.UI.create2DMatrix().rotate(), I found that the robot's hands and feet rotated by the center of the hands or feet itself. I have tried to change the anchorPoint but it didn't do any good. How could I rotate the ImageView by the point / coordinate I specified? or it just couldn't?

  • Are you setting the anchorPoint correctly?

    Remember that it is not using the x,y pixel values.

    top left : anchorPoint:{x:0,y:0}
    top right : anchorPoint:{x:1,y:0}

    center : anchorPoint:{x:0.5,y:0.5}

    bottom left : anchorPoint:{x:0,y:1}
    bottom right : anchorPoint:{x:1,y:1}

    If these don't work, then maybe there's an issue with Android.

If we look at the description about anchorPoint on API reference, we'll find that "anchor specifies the position by which animation should occur".
