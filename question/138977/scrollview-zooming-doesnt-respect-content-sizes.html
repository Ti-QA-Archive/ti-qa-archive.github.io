ScrollView Zooming doesn't respect content sizes

I seem to have run into an issue. When you zoom in content into a scrollview, it seems that the content size used is the width and height of the scrollview scaled instead of using the content size scaled.

Here is some code


var win = Ti.UI.createWindow({
    backgroundColor : '#000',
});

var scrollView = Ti.UI.createScrollView({
    backgroundColor: 'yellow',
    width: 320,
    height: 460,
    contentWidth:'auto',
    contentHeight:'auto',
    showHorizontalScrollIndicator: true,
    showVerticalScrollIndicator:true,
    minZoomScale:1,  
    maxZoomScale:10, 
    zoomScale:1,
    top:0
});


var image = Ti.UI.createView({
    backgroundColor: 'red',
    width: 320,
    height: 320,
});

scrollView.add(image);

win.add(scrollView);

win.open()

Try to zoom even just a little. You will see the vertical and horizontal sliders appear. While it is right that the horizontal one appears since the content width is bigger than the width of the scrollview. It doesn't seem right that the vertical one should appear.

For the right behaviour try the Photo app on the iphone. You should be able to zoom in and the photo remains centered until it stretches beyound the edges.

Is anyone else having this issue? How do you get around this?

Thanks for your help.
