About Titanium

I know it's a little late to ask such a question but here it goes:

Why is Titanium the way it is ? Why did it depend on a shipped Javascript engine instead of translating javascript code to native source code that does not depend on an external framework that may suffer from performance issues and has a bigger footprint ?

— asked June 14th 2013 by Hani Hamadeh

    Historically, it was because writing an interpreter is hard, a compiler is harder, and it's tough enough just using someone else's. Javascript is very dynamic, which is a nice way of saying it doesn't map well to pure machin code.

    It turns out that the language can be complex enough that not even that would be immune from reinterpretation as different objects and values are passed, leading to a very minimal ROI. Consider the following function:

    Function(a, b) {
If (!a) return b;
Return a+b;
}

    This function, mapped to machine code, would be over a dozen different functions based on the values and types of a and b. This is part of why JIT is Just in Time, as opposed to compile once: The methods often have to be recompiled based on inputs.
