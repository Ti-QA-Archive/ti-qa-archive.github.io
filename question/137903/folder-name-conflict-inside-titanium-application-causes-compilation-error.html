Folder name conflict inside Titanium application causes compilation error

This is a problem concerning the application linker (ld) and the names of folder inside your app.

I changed the application name and the compilation process started failing diring the linking fase with some copy error code and exit code 65.
No matter what i changed.
Finally i discovered the problem and i want to share it with you.

If you have inside the Resource folder a folder witch have the same name of your application and inside this folder (or any subfolder) you have some file which are not js file.

For example you have this structure:

Application1/Resources/app.js

Application1/Resources/application1/main.js

Application1/Resources/application1/image.png

The folder application1 will be copied inside the root of your application package.
Inside that folder the compiler will create the executable file which will be called Application1
This will cause an error in the linker (ld). This problem is case insensitive and make me have big headhacke.
It may be considered a bug of titanium compilation process? In any case there should be a clear message.

Application Type:mobile
Titanium SDK: 1.8.2 (but i think all)
Platform: iOS 5.0 (but i think all)
Device: Simulator, but event final build
Titanium studio 2.0
