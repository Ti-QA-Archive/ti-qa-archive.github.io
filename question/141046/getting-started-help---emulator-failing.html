    • Try to use the comments features when responding to an answer, it makes the thread easier to follow.

      Now, as for the SDK issue, fire up your Android SDK manager. You can do this easily by running the command android from the android-sdk directory and then running the SDK manager. From there, make sure you've installed the apis and tools (its always good to not only have the latest ones, but a few back for testing). Also, make sure you've got the Google APIs version for each tool. In this case, you'll need to make sure you have Android 2.2 (API 8).

      You can also change the default Android API for the project in the project settings if you want to use Android 4.1 as your default Android version. Again, make sure that you have the Google APIs version before you try to use it as a default.
