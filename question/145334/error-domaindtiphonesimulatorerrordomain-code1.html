  • For me, it turned out to be because there was no application present at the given path to be launched.

    That was because my Xcode settings were specifying a default build location, as described in this answer at Stack Overflow: XCode 4 and Titanium: TARGET_BUILD_DIR is incorrectly set (a setting I had applied long ago) so the binaries to be executed were not present within the build/iphone/Debug-iphonesimulator/ tree.

    The clue came from running the xcodebuild on the command line and looking closely at the output, and realising that the binary was being written elsewhere.
