iOS-specific constants raise a warning on Android

We use Titanium 1.7.5 SDK on Android 2.3 on Linux.

We use a single codebase for both Android & iOS. For TableViews on iOS, we used constants like Ti.UI.iPhone.TableViewSeparatorStyle.NONE for separatorStyle. On Android, we use separatorColor: 'transparent' in JSS to achieve the same effect. But since the code is the same, while running on Android we see warnings like:

> Ti.UI.iPhone.TableViewSeparatorStyle.NONE property isn't supported on Android

We get similar warnings for other iOS-specific constants like Ti.UI.iPhone.TableViewStyle.GROUPED, Ti.UI.iPhone.TableViewCellSelectionStyle.NONE etc.

Is there any best practice for using such constants in a single codebase on both platforms? I presume these constants can't be moved to the iOS JSS? Can such warnings lead to other issues? Is there any way out at all?
