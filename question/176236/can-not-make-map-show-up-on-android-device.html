    • Google Play Services are indeed device only. If you're using the Genymotion emulator, the paid versions come with Play Services installed. There are hacks to add them to the free version.

      The other issue, and probably the issue here, is that the maps API key is tied to the certificate used to build and package the application. That means you can't use Studio's run on device option stock out of the box – by default, that uses our certificate not yours. You need to do one of the following:

      1. Use the Package for Store option, use your certificate, then install the resulting APK on your device.
      2. After using Run on Device once (which you've done, and which will fail as you're seeing) open the Run Configurations dialog. In that dialog, choose your keystore and certificates, etc. Future builds will now use your certificate (and maps will hopefully work).

      To open Run Configurations, either: choose Run, Run Configurations; or, above Project Explorer on the left, click the down arrow next to the run target (e.g. your device, selected simulator, etc.) and choose Run Configurations

  • I ran into this issue as well. You were on the right track adding that API key in Google's Developer Console… but you added the wrong key. The key you added is only used when building the distribution APK. During development builds, there's another key file being used that's stored in the Titanium SDK.

    {Titanium}/mobilesdk/{os}/{sdk version}/android/dev_keystore

    Add a second entry for your app using this key and you should be good to go.
