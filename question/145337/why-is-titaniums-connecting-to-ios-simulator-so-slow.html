    • Create a new blank project and DO NOT ADD ANY CODE.

      Build the app for the first time. Close the simulator.

      Build for the second time and note the time it takes. This will remove ANY code or modules you have added as part of any delays. If this is still slow then something else is at fault, if not then it is likely the additional modules you have added.

      If it appears to be one of the modules then make a choice, do you care - as you cannot remove one I am guessing. If it is for scientific benefits then add one and re-time, remove that one and re-time - then finally add both and re-time.

      At least you will know if it is the extra modules.
