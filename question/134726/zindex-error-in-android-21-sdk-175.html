zIndex error in Android 2.1 (SDK 1.75)

Morning All,
I have an app which I created for Android 2.2+ using SDK1.8.2, which works perfectly. In the interest of making the app available to as many users as possible, I want to make it backwards compatible to Android 2.1 (using SDK 1.75).

All the code works fine EXCEPT a silly little facebook login button, in particular that it's placed onstage with a zindex.

// facebook login button
var login = Titanium.Facebook.createLoginButton({
    top:302,
    left:220,
    zIndex:2000,
    visible: false,
});

If I comment out the zIndex, the whole app works fine, but if I put the zIndex back in, the app crashes. The error is: 


"E/AndroidRuntime(  579): Uncaught handler: thread main exiting due to uncaught exception
E/AndroidRuntime(  579): java.lang.NumberFormatException: unable to parse '2000.0' as integer"

I know it;s the zIndex causing the problem becuase if I change the value to something arbitrary e.g. 3281, the error becomes: 'unable to parse '3281''.
Unfortunately I need the zIndex, so I can't just remove it.

Can anyone suggest a solution please?
Thanks,
Lee
