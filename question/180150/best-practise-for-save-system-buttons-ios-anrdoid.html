Best practise for Save System buttons ios, anrdoid

hello,

in the app there is profile page where the user can save some values. what is the best practise to achieve the saving? in ios, i have added a toolbar in the profile.xml. i guess this is what an apple user would expect.

<Toolbar platform="ios" bottom="0" borderTop="true" borderBottom="false">
           <!-- The Items tag sets the Toolbar.items property. -->
           <Items>
                <Button id="updateButton" systemButton="SAVE" onClick="profile_update"/>
           </Items>
           <!-- Place additional views for the Toolbar here. -->
</Toolbar>

how would i implement this in android?

closely related, i read in one slide (unfortunately i cant find it anymore) to generally avoid EventListeners. Is that true? So it would be better to add the update function to the xml view as opposed to:


$.updateButton.addEventListener("click", function(){
update user profile stuff
)}

    You can mock up an Android toolbar using a view with buttons inside. Event listeners are essential for handling events (eg like 'click') so cannot be avoided. I suspect you read about using Application eventlisteners which if not managed can cause memory leaks (These are events added to Ti.App.addEventListener)
