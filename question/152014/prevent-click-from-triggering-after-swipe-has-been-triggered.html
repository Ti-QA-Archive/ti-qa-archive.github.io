Prevent click from triggering after swipe has been triggered

I'm trying to show a view after a user has swiped on the screen, that works. The problem however is, that the click event is also triggered after the swipe has occured, causing another action to occur (in my case opening a window which shouldn't happen after swipe).

Is there a way to prevent the click event from triggering when a swipe has occurred?

    You could consider listening to the singletap instead of the click event. The singletap does not fire on swipe.

  • Hi

    This depends on the order the events are fired. If the swipe event is called BEFORE the click event (I think it does), then you call the cancelBubble method.

    Example;

    view.addEventListener('swipe', function (e) {
    e.cancelBubble = true;
    // your code
});
