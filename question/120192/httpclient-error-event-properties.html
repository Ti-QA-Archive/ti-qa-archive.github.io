HTTPClient error event properties

Does the error raised in the httpclient onerror event have any specific properties ?

for example e.error returns a lot of information about the error, but is that it?

Does e.error have other properties such as e.error.code or e.error.message etc?

    Steve,

    When processing errors I use this.status to get the error code.

    I found e.error was too much for iPhone, but you can use the following to parse out the interesting bits.

    var match_arr = e.error.match(/\"([^\"]+)\"/);
var message;
if (e.error.indexOf("ASIHTTPRequestError") > -1 && match_arr.length > 0) { // Catches iPhone
    message = match_arr[1];
} else {
    message = "Default Error Message...";
}

    You could also use this.responseText if you want to get the html response.
