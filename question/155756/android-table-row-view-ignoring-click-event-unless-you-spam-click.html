Android Table row view ignoring click event (unless you spam click)

Hi all,

I'm having an on going Android issue where complicated table rows ignore a click event on the table itself, but is working normally in iOS. Removing the contents of the table arrow allows the click event to occur normally 100% of the time, where as before it required several fast clicks to trigger the event. Would anyone have any suggestions on how to resolve this problem? I have already attempted to add a wrapping view containing the contents of the tablerow and add a listener to it, but the result is the same.

Below is a snippet of the loop to build the table rows.

    for (var i=0; i<numRows; i++)
    {        
        var row = Ti.UI.createTableViewRow({
             height:                    "75dp" ,
             className:                "alertRow",
             backgroundSelectedImage:     '/row_edge_pressed.png',
              backgroundSelectedColor:     'transparent',
              selectedBackgroundImage:      '/row_edge_pressed.png',
              backgroundColor:             "#fff"
        });

        //wrap tablerow contents
        var wrapperView = Titanium.UI.createView({
            layout:    "composite",
            left:          0,
            right:        0,
            width:       "100%",
            height:     "75dp"
        });

        var icon =  Ti.UI.createImageView({
                left:        "15dp",
            top:        "13dp",
             image:    '/alert_icon.png'
        });    

        wrapperView.add(icon);    

        var distance = Ti.UI.createLabel({
             left:            "15dp",  
             top:            "54dp", 
             text:            tableRows[i].distance, 
             textAlign:        "Ti.UI.TEXT_ALIGNMENT_CENTER",
              color:        "#b91e43"
        });

        wrapperView.add(distance);    

        var merchant = Ti.UI.createLabel({
            left:        "70dp" ,
            top:        "12dp" ,
            width:    "60%",
            text:        tableRows[i].merchant_name,
            textAlign:    "Ti.UI.TEXT_ALIGNMENT_LEFT",
             color:    "#b91e43"
        });    

        wrapperView.add(merchant);    

        //add wrapper to row
        row.add(wrapperView);

        //push row into data array
        rowData.push(row);

Environment:

OSX: 10.8.3
Android 4.1.2 (Samsung S2)
Titanium SDK 3.1.1.GA

  • Instead of listening to the click event on the tableviewrow, subscribe to the click event on the tableView itself.

    — answered August 7th 2013 by Nick Milner
    permalink
    2 Comments

    • Additionally, add the property touchEnabled with a value of false to each child object that should NOT receive touch events. Even if you do not listen for the events they still are enabled so things below may not receive the events.

      Having said all that, Nick is on the money.

      — commented August 7th 2013 by Malcolm Hollingsworth

    • Thank you for your comments.

      I currently have the listener set to the tableview rather than the row. I implemented Malcolm's suggestion of using touchEnabled and found it worked well on less complicated row structures used elsewhere within the app, but not with the above example.

      While it's not an ideal solution I did find that changing the click to a touchend event provided a good level of interactivity without accidental touches from scrolling the tableview.
