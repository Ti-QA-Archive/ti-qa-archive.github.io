KitchenSink Does Not Install on iPad

Hello,

I'm using a Mac with OS X version 10.8.4. I have XCode with the iOS SDK 7 installed.

I'm trying to install the latest (I think) KitchenSink on an iPad 2 that has iOS 7. Titanium builds the app and installs it to iTunes, but the app is never fully installed on the iPad. The app icon appears on the iPad, but it's a dark gray color. When I tap it to launch the app, the title "KitchenSink" changes to "Installing…" I restarted the iPad and it exhibits the same behavior.

At first I thought the problem was because I had an older XCode installed (6.1 SDK) but I get the same behavior after installing the latest XCode.

KitchenSink seems to run OK on the iPhone simulator.

In iTunes if I remove the KitchenSink app, the button returns to saying "Install" but the icon is not removed from the iPad. I have to tap and hold on the iPad icon, then tap the X to remove it from the iPad.

According to iTunes I have over 16 GB of space available, so it's not a space issue on the iPad.

I've tried this several times and I can't get it to work. Any ideas?

  • stop using iTunes if that gives you a problem (I stopped using it a long time ago). Compile it as ad hoc, and install it manually on your device, either with an OTA link or with the iPhone configuration utility. You'll get a better feedback on what you may be doing wrong.

    Things to look at:

    • Is your ad hoc certificate valid?
    • Is your device included in the ad hoc certificate?
    • Is the certificate valid for that bundle?

  • It looks like I was using an incorrect provisioning profile (the profile did not include an App ID that matched the KitchenSink app). I created a new provisioning profile that included the KitchenSink App ID, installed the profile, rebuilt using that new profile, and I get the result. The app is pushed to the iPad, but can't be run.

  • click on the little brown box next to the debug button, then select Distribute Ad Hoc Enteprise. select where the ipa should be saved to, select the right provisioning (and make sure that the udid of your device is included in it), and save it. Then download the iPhone Configuration Utility here and install it through that on your device.

  • I finally figured out the problem. The UDID for the iPad I was using was incorrect. I used an app to give me the UDID, but it was incorrect. I used the UDID that was listed in iTunes, which also match the ID displayed using the iPhone Configuration Utility. Once I created and downloaded a new provisioning profile that included the correct UDID for my iPad, it installed correctly.

    Thank you to everyone for the help!!!
