SQLite Transactions using alloy and backbone

Just started using alloy for a project that occasionally needs to insert 1,000s of records into an SQLite table. Previously when I've needed to do this I've used transactions which speeds up the database write time considerably but I can't find any information on how to do this in alloy or backbone (the closest I found was a technique to add a save function to a collection). Does anyone have any experience/best-practice with this (or advice on a better way to do this)?

    Yes, this is doable, but we need to know more about your setup. That said, given the assumption that you didn't change the "db_name" or "db_file" property of the model's adapter config you would do something like this (untested):

    // _alloy_ is the default database name
var db = Ti.Database.open('_alloy_');

db.execute('BEGIN;');
// execute your series of inserts
db.execute('COMMIT;');
