How to deploy build package for Enterprise Distribution?

After countless hours of trial/error and failure, I finally was able to successfully create a build from Titanium's "Distribute - Apple iTune's Store". But I am not sure what to do now because it didn't seem to create an ipa file.

Note: I have an Enterprise program from apple, rather than standard iOS developer account. My app is for in-house distribution only, and won't go to the app store.

Previously when I did a dev build (targeting specific apple device IDs), it created an ipa file. Then with the help of the BetaBuilder app, I was able to create a series of files (ipa, plist, html) that I copied to a web server for download and install. But now I don't see an ipa file and I'm not sure what/how to deploy this to the web server.

Instead of the IPA, Titanium created some kind of package named the same as my app in this location: build\iphone\build\Release-iphoneos\Daily Report. The "Daily Report" (my app) is shown as Kind = Application in finder. I can look inside this app/package and see a bunch of folders and files, but no ipa. I suspect this entire package gets deployed, but how? Do I still use BetaBuilder? Can this package be deployed on its own to the web server and downloaded?

    You can deploy an enterprise package with Titanium, that is not a problem. Once you made the so called Archive with Titanium (by using Distribute - Apple iTunes Store), you have to switch to xCode though. Open xCode -> Window -> Organizer, in Organizer click on Archives, look for your application, click on it and use "Distribute", xCode will ask you what to do, one of the possible selections is Enterprise distribution. Thats about it, it will save a ipa file and also the other necessary files, upload those and enjoy enterprise distribution.
