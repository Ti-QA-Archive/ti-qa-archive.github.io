Get current date-time and not device date-time

Hi, I am making use of appcelerator cloud in my app which involves some date manipulations. So i made use of moment.js for the same. Now when i change the device date time to a future date moment.js returns me the date according to the device date-time whereas i want to make use of actual current date-time. I tried using the Date() object too but the same result. How can i get the current date-time and not the device date-time. I am on 3.3.0 GA. Thanks

  • Any of the date functions, whether the moment library or native Date() constructor, will pull the date from the system clock. This would be true of a desktop computer or mobile device. If you alter the system's clock, naturally those functions would get the manually configured date/time.

    If you need something else, you'll need to hit a network service to retrieve the actual date/time. Depending on the accuracy you need, you'll have to do some manipulations to account for propagation delays.

    The responses of a GPS location query typically include the time. So, if you're already doing a GPS location lookup, that could be your external source. I'm not sure that a network-based location response would include anything other than the device's timestamp. You'd have to test that.
