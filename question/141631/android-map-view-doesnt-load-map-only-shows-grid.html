Android map view doesn't load map, only shows grid.

Hi all, I know, i know.. there are many simmilar questions here, I have read them all ..

Ti SDK: 2.1.1.GA
Android API: tried from 10 till 14 ( all Android Google API's)

I try with and without &lt;property name=&quot;ti.android.google.map.api.key.development&quot;&gt;xXxXxXx&lt;&#x2F;property&gt; in TiApp.xml. Tried custom AndroidManifest …

Nothing works: no map, just a view width the grid. And a message in my log:

[ERROR][MapActivity(  487)] Couldn't get connection factory client

    mapView.add(mapVw);

I see that people are in other threads talking about the 'dev' api-key from the KitchenSink, but is that still used in the 'new' (CommonJS) KitchenSink? I can run the KitchenSink fine in my emulator.

    • There are two things at play here…

      1. In order for an android app to run on a phone, it has to be signed. When you Distribute, you're signing it with your certificate. When you hit "Run on Device" or "Run in Emulator", Titanium uses it's own certificate.

      2. In order for maps to show up, you must use the API key for your certificate. So, if you specify your API key for the Titanium-signed app, it won't work. Also, if you don't specify a production API key, it won't work.
