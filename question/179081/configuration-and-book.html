  • Hello Mario,

    Thanks for your inquiry. There are some steps of configuration and quite a few software’s are needed to configure titanium for developing mobile applications. Since you mention you are using Mac OS X. I will try to provide every configuration steps for using Titanium with Mac platform.

    ###Installing Oracle JDK

    To start with, you need to have “Oracle's Java Development Kit (JDK)” installed. Oracle JDK is installed by default on Mac OS X 10.6 (Snow Leopard) and earlier. You can also go to Apple Developers and search 'java' to download the installer. Install JDK if you already don’t have. Run the following commands from the terminal to determine whether JDK has been installed and configured correctly:

    java -version
javac –version

    ###Installing Node

    To run all Appcelerator components, you must have Node.js 0.10.13 or later. Download and install node.js from this link here

    ###Installing Titanium Studio

    I presume you have installed titanium studio properly. For any farther details please follow this link

    ###Launching Titanium Studio

    1. The first time you launch Studio you need to select a workspace, Select a workspace of your choosing.
    2. Log in to studio with your Appcelerator account.
    3. When studio opens, go to Help>Check for Titanium Updates or Help> Check for Updates.

    4. This will ask you to install

      • Titanium Studio
      • Titanium SDK
      • Titanium CLI
      • Node.js
      • Required NPM packages

    Install them.

    ###Installing platform SDKs

    To develop native applications with Titanium you need the SDKs and tools for those native platforms installed on your system. For example, to develop Android applications you need the Android SDK and toolchain installed.

    The first time you launch Studio the Platform Configuration dialog opens. This dialog indicates which platforms SDK are installed and which is not, and lets you easily install individual SDKs. Select the platforms you want to install. Click Configure to start the installation process. Studio begins downloading and installing each SDK you selected, according to the settings you specified. In the case of iOS, Studio launches the App Store application installed on your Mac to the XCode download page.

    After installation complete go to your Studio Dashboard>Get Started to see if your Native SDKs are properly configured.

    As of now your Mac is configured for Titanium platform. If you missed any of the steps and couldn’t follow through Please follow this link here. Everything is described there.

    ###Running Android Application with Genymotion

    To run android application faster, you can use Genymotion virtual devices (or emulators) to test Android applications. It is an alternative to the Android SDK's native Android Virtual Device manager. Genymotion emulators run a lot faster than the native Android emulators.

    In order to download Genymotion and Genymotion virtual devices, you will need a Genymotion Cloud account. To sign up for an account, visit. You will need to verify your e-mail address before downloading files. Download the Mac OS X installer (DMG file) from Genymotion.

    1. The Genymotion application starts. A dialog prompts you to "Add a first virtual device". Click Yes.
    2. In order to add a virtual device, you need to connect to the Genymotion Cloud service. Click Connect.
    3. You are prompted to enter your Genymotion Cloud account credentials. Input your credentials, then click Connect.
    4. After your account has been verified, select a device and click Next.
    5. Optionally change the name of the device. Click Next.
    6. Click Finish after the download completes or click Add to download more virtual devices.

    To enable support for Genymotion with the Titanium toolchain, you need to configure the Titanium CLI. If you have not modified the default installation of VirtualBox and Genymotion, you only need to set the genymotion.enabled value to true with the ti config command. Restart Studio after updating the CLI configuration.

    ti config genymotion.enabled true

    For further details please follow this link here.
