Change TableViewRow backgroundImage on click event not working

I know this has been asked many times on forums, and there are various answers…most questions never get a solid answer. The best answer that I've seen is to use table.updateRow(). That got me a little closer, but then began to produce some bizarre behavior. Rows that should not have their backgroundImage changed, would, and random rows would have the backgroundImage change. Using Ti.API.info, the index was alwasy correct so there was no reason for the backgroundImage of the other rows changing. To reproduce, download the following widget:
com.controlenvy.flyoutmenu

Create a new Alloy project and add the widget to the widgets folder under app so it should be "yourproject"/app/widgets/com.controlenvy.flyoutmenu. You may have to create the widgets directory.

Next edit your /app/config.json file and add 

"dependencies": {
    "com.controlenvy.flyoutmenu": "1.0"
}

You index.xml file should look like this:

<Alloy>
    <Window class="container">
        <Widget id="fly" src="com.controlenvy.flyoutmenu" />
    </Window>
</Alloy>

You index.js file should look like this:

$.fly.init({});
$.fly.add({});
$.fly.add({});
$.fly.add({});

$.index.open();

Now start clicking around on the buttons. Eventually several buttons will be selected and you'll start seeing some patterns.

Why is this happening? If you look at the widget.js file, you will see some commented lines where I have tried many things.

I'm running

Titanium Studio, build: 3.1.0.201302181614
titanium: 3.0.24
Alloy: 1.0.0
