Error loading provisioning profiles in Ti Studio

Hi,

Ti studio is no longer able to load my provisioning profiles. Screenshot of Ti studio is at https://www.evernote.com/shard/s9/sh/34f97151-6e83-435f-a4cb-7fddf54bfff6/da32a810af9f14df65395492787554df. I can see profiles in xcode & keychain. I am also able to install the app on devise using command line

titanium build --pp-uuid &#39;my provisioning profile uuid&#39; -T device -p iphone -f -V &#39;developer name&#39;

I have tried using Ti Studio 2.1, 3.0 RC, Beta & nightly build. Also used Ti sdk versions 2.1.3.GA & 3.0.

Tried uninstalling & reinstalling studio. Tried removing the following directories

~/Library/Application\ Support/Titanium
~/Library/Caches/com.appcelerator.titanium
Titanium workspace/.metadata
project/build

I have tried with both alloy & normal titanium apps.

I could see that other developers are also having the same issue.

http://developer.appcelerator.com/question/144167/titanium-studio-can-no-longer-load-provisioning-profiles

Platform Details:

XCode: 4.5.1
OS: Mac OS X 10.7.5
Ti Studio: 2.1 & 3 (RC, Beta & nightly)
Ti SDK: 2.1.3.GA & 3.0.0.v20121030170824

Ti Studio Installation Details is at https://gist.github.com/4004876
