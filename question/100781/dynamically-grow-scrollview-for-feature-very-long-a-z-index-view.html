dynamically grow scrollView (for feature: very long A-Z index view)

Hi all!

Is there any way to dynamically add space to the TOP of a scrollView? Unfortunately, if I increase the contentHeight the space usually will always be added to the bottom.

If not - is there any way to move the focus of a scrollView (contentOffset) immediately, i.e. WITHOUT animation? I though about circumventing the problem by increasing the contentHeight (and thereby adding space to the bottom), then moving an embedded view container downwards (by increasing its "top" property) and finally jumping the focus downwards (by applying "scrollTo"). However the scrolling animation is really disturbing…

For your background: The final aim of all this is to create an A-Z-index table of possibly thousands of entries (derived from a database) inside a scrollView. It should grow dynamically while the user scrolls upwards or downwards. Preloading the index is not an alternative since creating an tableView of e.g. 20.000 rows takes much too much time… :-(

Any help appreciated!

Simon

I do not see why this would not work to "insert" content at the top of the parent view - although I have to admit I have not tried it there, yet.
