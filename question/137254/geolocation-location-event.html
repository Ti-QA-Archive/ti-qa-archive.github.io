Geolocation location event

Hello community,

The iOS documentation suggests that the system wakes up a sleeping app when it detects a significant location change event. Is this not what should happen if I code Geolocation along the following lines

Ti.Geolocation.purpose = 'Monitor location changes';
 Ti.Geolocation.distanceFilter = 10;
 Ti.Geolocation.accuracy = Ti.Geolocation.ACCURACY_NEAREST_TEN_METERS;
 Ti.Geolocation.addEventListener('location',uponLocationChange);

This works just fine when the app is in the foreground but from what I can see it does nothing at all when I change location with the app backgrounded, or killed. Any helps and tips with this would be much appreciated.

— asked May 21st 2012 by Et Kit

  • The Appcelerator API does not support significant location change events. There are alternate approaches to solving this problem including developing a module.
