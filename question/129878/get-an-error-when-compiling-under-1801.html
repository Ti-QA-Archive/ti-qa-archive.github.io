    Oh I experienced the same issue today.

    Scenario:

    • I have an file ui.js under Resources/ui/
    • In app.js, I use require('ui/ui') to load my ui.js file.
    • Build/Run project, I got error: [ERROR] Error: Traceback (most recent call last):
    • Show Package Contents of myproject.app ( under myproject/build/iphone/build/Debug-iphonesimulator/myproject.app ) I notice that ui.js did not copy to there.
    • Last lines in build.log ( under path_to_myproject/build/iphone/build/build.log )
    ** BUILD FAILED **


The following build commands failed:
    Ld build/Debug-iphonesimulator/UI.app/UI normal i386
    GenerateDSYMFile build/Debug-iphonesimulator/UI.app.dSYM build/Debug-iphonesimulator/UI.app/UI
(2 failures)
EXIT CODE WAS: 65

Exception detected in script:
Traceback (most recent call last):
  File "/Library/Application Support/Titanium/mobilesdk/osx/1.8.0.1/iphone/builder.py", line 1274, in main
    execute_xcode("iphonesimulator%s" % link_version,["GCC_PREPROCESSOR_DEFINITIONS=__LOG__ID__=%s DEPLOYTYPE=development TI_DEVELOPMENT=1 DEBUG=1 TI_VERSION=%s %s %s" % (log_id,sdk_version,debugstr,kroll_coverage)],False)
  File "/Library/Application Support/Titanium/mobilesdk/osx/1.8.0.1/iphone/builder.py", line 1180, in execute_xcode
    output = run.run(args,False,False,o)
  File "/Library/Application Support/Titanium/mobilesdk/osx/1.8.0.1/iphone/run.py", line 39, in run
    sys.exit(rc)
SystemExit: 65

    Work-around:

    • Rename ui folder to other name, i.e: Resources/modules/ui.js
    • Update require('ui/ui') to: require('modules/ui')
    • Delete all file/folder under path_to_myproject/build/iphone
    • Build/Run Successfully.

    Hope that help.

      I actually did the same thing but still faced the same error. It finally worked after I changed the name of another folder. Apparently this error happens when a folder name is the same as a file name in the project (a folder: "anything/" and a file: "anything.js").
