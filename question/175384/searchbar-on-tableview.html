SearchBar on TableView

Hello,

i try to implement a rest search on a table view. So far i got this code to fetch Events from the remote
service by using the Alloy Collection. The table is also refreshed in back ground but when i try to searchsomething i get this autocomplete List saying no results.

How can i disable this autocomplete list and connect the searchbar to my TableView instead?

<Alloy>
    <Tab id="eventsTab" title="Events" icon="tabbaricons/events.png">
        <Window id="eventsList" title="Events" barImage="/images/barimage.png" class="container">
            <Require src="_header"/>

            <TableView dataCollection="events">
                <SearchBar id="searchBar" height="43" top="0" hintText="Search Event ..." />
                <TableViewRow color="black" onClick="showEventDetail" leftImage="{Flag}">
                    <View class="box" height="100">
                        <Label top="10" left="20" class="date" text="{Date}"></Label>
                        <Label top="25" left="20" text="{Title}"></Label>
                    </View>
                </TableViewRow>
            </TableView>
        </Window>
    </Tab>
</Alloy>

$.searchBar.addEventListener('change', function(e){
    var searchFor = e.value;

    if(searchFor.length  > 2){
        events.fetch({search: searchFor});
    }

    return false;
});

    Because you have placed the searchBar within the tableView that automatically wires up the inbuilt search. because you have not populating the neccessary fields for it to do this job, that is why you see the 'no results' message. You need to move the searchBar OUTSIDE of the tableView to prevent the inbuilt search from executing

    • If you want build an external search (by REST) you have to separate your search from tableview, like Nick explained. In my old projects (without listview) I have played with dummy tableview.       Or: you avoid search view and use simple textfield and link this input with REST servive etc.
