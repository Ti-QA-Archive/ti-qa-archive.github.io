TableView with TableViewRows: click event x and y inconsistent

Hi,

I'm having some trouble with a TableView that includes some relatively complex TableViewRows (with leftImage, rightImage and a TextField). The 'click' event is set on the TableViewRow. When I click on the rightImage, it will return an e.x=0 and e.y=0 when I click on it the first time but if I click elsewhere, the back to the rightImage area, the e.x value will be close to the maximum (e.g. e.x=263, etc.). Any idea why this is?

I've also tried setting the event on the TableViewRow itself but it won't pass the clicks that are on the images.

Thanks in advance,

Mark

  • It being a case of need, I decided to create separate buttons rather than use the leftImage/rightImage. This allows you to add specific listeners to the buttons themselves and not have to worry about the coordinates returned by the click. The example I followed was in Kitchen Sink, table_view_layout_3.js
