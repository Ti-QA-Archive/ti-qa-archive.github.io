Difference between Button with image, ImageButton, and clickable ImageView?

Is there any difference between Button with image, ImageButton or clickable image? Functionality, or looks?

  • Please compare these components in the API documentation for the full details. In a broad sense, you can accomplish very similar things using an image, a view with a background, and a button. Buttons have additional properties to change their appearance as the user clicks or selects them. Buttons would have some accessibility advantages, especially on Android where users could Tab to the next button.

    If a label, image, or view gets the job done, you can use any of them as if they were buttons by simply adding a click or tap event listener. But in general, you should probably use Ti.UI.Button for things in your app that should be treated like buttons.

  • Well, both can achieve pretty much the same behavior, but they differ on one fundamental level.

    • A Button with image is basically just what it says. It's a button (which is clickable) with an image. That's all it does. Also, you can specify a different look and feel when the button is pressed down and when it is released.
    • A View with a background image, while it can act as a button, it can do a lot of other things. (swipe, animate, contain other components, etc.). But it is not a JUST a button.

    So if, you need to create a clickable button without anything else then clicking it. Then I recommend you go with the Ti.UI.Button component.

    An ImageButton is simply something that doesn't exist.
