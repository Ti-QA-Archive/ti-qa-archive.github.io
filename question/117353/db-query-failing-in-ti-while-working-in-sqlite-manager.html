    Alexander

    SQLite's support for autoincrement is explained in the official SQLite documentation.

    There is a recommended way to insert data using the Titanium API. See the example in the Storing Data section of the Working with Local Data guide.

    Thus, combining these two approaches, there is no need to specify any value, even null, for the autoincrement field when inserting data. Although I haven't tried it myself, I suggest that you could retrieve the unique ROWID key value using the following code (not tested):

    var db = Ti.Database.open('weatherDB');
var cityWeatherRS = db.execute('SELECT ROWID AS mykey,id,name,continent FROM city');
while (cityWeatherRS.isValidRow())
{
  var myKey = cityWeatherRS.fieldByName('mykey');
  var cityId = cityWeatherRS.fieldByName('id');
  var cityName = cityWeatherRS.fieldByName('name');
  var cityContinent = cityWeatherRS.fieldByName('continent');
  Ti.API.info(cityId + ' ' + cityName + ' ' + cityContinent);
  cityWeatherRS.next();
}
cityWeatherRS.close();
db.close();

    Try this out, and post your updated code if you can't get it to work.

    Cheers

      When you post a question to the Q&A, you will receive much more accurate answers more quickly by posting a proper "usecase". A usecase is a self-contained piece of code that demonstrates your issue in the least possible number of lines. Some additional advantages of providing a usecase are that it is much kinder to those wishing to help you, because they won't have to spend considerable time replicating your issue on their systems, that you will gain a much great knowledge of how Titanium works, and lastly that the process of creating a usecase can often simplify the problem and consequently reveal the answer. You can read more about this subject in the official documentation.

      I had no idea what the solution to your problem was when I first read your question, but by creating a usecase I was able to deduce the solution.

      Hence, the following code works, with the functionality you require. Simply paste it into app.js of a new project.

      var db = Ti.Database.open('autoincrement_test');
db.execute('CREATE TABLE IF NOT EXISTS table1 (fieldA INT, fieldB TEXT)');
db.execute('DELETE FROM table1');
db.execute('INSERT INTO table1 (fieldA, fieldB) VALUES (?,?)', 123,'this is row 1');
db.execute('INSERT INTO table1 (fieldA, fieldB) VALUES (?,?)', 234,'this is row 2');
db.execute('INSERT INTO table1 (fieldA, fieldB) VALUES (?,?)', 345,'this is row 3');
db.execute('INSERT INTO table1 (fieldA, fieldB) VALUES (?,?)', 456,'this is row 4');

var testRS = db.execute('SELECT ROWID,fieldA,fieldB FROM table1');
while (testRS.isValidRow())
{
  var myKey = testRS.fieldByName('ROWID');
  var fieldA = testRS.fieldByName('fieldA');
  var fieldB = testRS.fieldByName('fieldB');
  Ti.API.info('row data = ' + myKey + ' / ' + fieldA + ' / ' + fieldB);
  testRS.next();
}
testRS.close();
db.close();

      Hope it helps! :)
