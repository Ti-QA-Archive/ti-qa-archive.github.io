because it always appears as a red heart

var lbl=Ti.UI.createLabel({
    text: String.fromCharCode(9829),

    Hi

    In order to add unicode symbols you encode in the unicode format;

    var lbl=Ti.UI.createLabel({
    text: '\u2764'
});

    Unicode is encoded using hex values (base 16) rather than decimal (base 10)

    • I have found the reason, others with the issue - and Kosso was correct with the reason.

      It seems that iOS has over-ridden many of the non-emoji characters with the emoji equivalents. This has not been greeted with much love by a group here;

      • How to disable emoji in a UILabel?

      They have spent some time trying to reverse this OS enforced behaviour. There is a suggested fix, but not that I have managed to repeat, do try and see what happens for you.

      The suggestion is to use the fontFamily; Hiragino Mincho ProN, the understanding is that this font does not render the special characters with the emoji overrides.

      As I say - this has not worked for me, let me know how you get on.

      Otherwise the FontAwesome route does work, you would do this;

      var lbl=Ti.UI.createLabel({
    font: {
        fontFamily: 'FontAwesome'
    },
    text: '\uf004'
});
