  • Guys, please be aware of the following tickets:

    • Install Titanium SDK from URL downloads but does not install SDK
    • Incorrect permissions for Titanium SDK files after "Install Titanium SDK from URL"

    So, for me, the resolution was:

    • delete the existing Titanium 1.7 SDK from the filesystem and then "Install Titanium SDK from URL"
    • set execute permissions on all .py files inside the SDK directory installed in the previous step
    • close emulator
    • restart TiStudio

    I'd be interested if this works for you.

    Cheers

    • launch an app using Titanium 1.7. Make a note of the Ti SDK build version / hash / date
    • close the simulator
    • make sure your environment adheres to the iOS section of the Titanium Compatibility Matrix
    • uninstall TiStudio, and reinstall ensuring that only the Stable Updates site is enabled. Update TiStudio to the latest Stable build
    • install the latest 1.7.X Continuous Build using the "Install Titanium SDK from URL" option
    • launch the app using the newly installed 1.7, and make sure its hash and date have changed since step 1 above to confirm it was correctly installed
    • try to use the debugger

  • If you accidentally clicked debug and don't use debug

    To disable debug:

    • 1) open you .xcodeproj in XCode > select Project.
    • 2) Select your project in middle pane
    • 3) Select Build Settings in right half of middle pane
    • 4) Under Apple LLVM compiler x.x - Preprocessing expand _Other C Flags
    • 5) Double click text right of Debug
    • 6) Delete line -DEBUG or -DDEBUG or the like.
    • 7) Clean via XCode (Product > Clean)
    • 8) Quit XCode
    • 9) Build and run App in Titanium
