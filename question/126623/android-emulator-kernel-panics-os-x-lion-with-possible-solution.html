Android emulator kernel panics OS X Lion (with possible solution)

After installing OS X Lion, I started getting kernel panics on both my desktop iMac and my MBA just about every time the Android emulator fired up. I ran across this issue on Google's site: http://code.google.com/p/android/issues/detail?id=19099 and this issue on Apple's Java dev mailing list: http://www.mailinglistarchive.com/html/java-dev@lists.apple.com/2011-08/msg00018.html Another user suggested increasing the heap limit for the Java VM, and so far that seems to have solved the problem for me (I've been using the emulator heavily today and have yet to see a single panic). The only way I could find to do this in Titanium Studio was by going into the application bundle and increasing the limits in TitaniumStudio.ini, but it's possible there's a simpler way.

This obviously appears to be a generalized problem with Java on Lion rather than a problem with Android, Titanium Studio or Aptana/Eclipse - note that the Apple thread describes a similar problem with NetBeans, and of course user-mode code shouldn't be capable of causing a kernel panic anyway. I thought I'd post this here nonetheless in case other Ti users are experiencing the problem – it was driving me nuts. :-)
