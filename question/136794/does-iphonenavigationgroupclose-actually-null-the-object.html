Does iPhone.NavigationGroup.close() actually null the object?

I'll preface this by saying I'm relatively new to Titanium.

I'm using a iPhone.NavigationGroup to organize my windows in order to get those built-in "back" buttons that close the child window and take you back to the parent window.

I'm wondering if iPhone.NavigationGroup's close() method actually nullifys the object, or if that's something I'd have to handle myself somehow. The one problem with Titanium I keep hearing about is memory leaks, so I just want to make sure I'm doing all I can do garbage collect objects that aren't being used.

Is there something extra I could be doing to force these window objects to be garbage collected, or is the close() method sufficient?
