  • Apple's lock you up, provisioning requiring security system can be a buggar at times indeed.

    Try to create and build a new project to the device, see if you get the provisioning prompts from inside of Titanium.

    In each project folder is a hidden folder called .settings

    Inside of it is a com.appcelerator.titanium.core.prefs file. It contains provisioning to provisioning profiles mappings for your projects. I had one get set as read only (I blame OSX) and when it came time to use a different dev key (handoff to client) we couldn't get it to change.

    Now, I can't remember if we deleted that file or if we moved the contents into a newly created project folder and rebuilt. Either way, suffered similar to what you are.

  • I had a similar issue, caused by changing provisioning profiles. In the Titanium_workspace directory (where your project lives if using default settings), there is a hidden folder called .metadata.

    There are a few files with the old provisioning profile number, which should be updated to the new number:
    ./.plugins/org.eclipse.core.runtime/.settings/com.appcelerator.titanium.mobile.prefs (twice on line 6)
    ./.plugins/org.eclipse.debug.core/.launches/–YourRunConfigName–.launch (line 18)

    You can find the new number in XCode –> Organizer. Click export profile & the file name is the provisioning number (it's actually an alphanumeric value: XXXXXXXX-XXXX-XXXX-XXXX-XXXXXXXXXXXX)
