Camera View Overlay rotating to Landscape Android

So I think this might be for someone at Appcelerator but not sure.

Version 2 of studio and SDK, running on Mac and Android V8 & rhino specific.

I have an application which uses the camera view with an overlay.

On IOS no issues, but on Android it rotates that overlay 90degrees from Portrait mode to Landscape mode.

I have come across this before, and raised a Jira for it. I had confirmation it had been fixed, so am wondering if it has crept back in.

Anybody know if this is the case or am I missing something ?.

  • Yes, I can confirm this is an issue. After finishing the iOS version of our app that makes use of the camera, and overlays where we need to take photos in both portrait and landscape orientation, when I got the Android version working on a Galaxy tablet, when the live camera preview screen appears with the overlay, the orientation of the preview view immediately rotates to landscape.
    The Android emulator also does exactly the same thing when the preview view is displayed.
    I should note that this is using 1.8.2 SDK. I will try to go ahead and test in 2.0.1 if nothing else breaks.
