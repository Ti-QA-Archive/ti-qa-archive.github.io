applicationDataDirectory Vs getresourcesDirectory

Where is it best to save file to..

getresourcesDirectory

or

applicationDataDirectory

I need to do a standard fetch image and save to the phone request, but not sure where to store the images.. data directory or resource directory.. and to be honest I am not sure what the difference is as is not really very clear in the docs…

Could anyone shine some light on this for me? Will one directory remain in tact following a 'clear app data' call by the client or something?

    Hi,
    There is major difference between both the directory. getresourcesDirectory will give you access of resource directory and this is your project resource directory where your images, .js files are kept. This directory is read only , Means you cannot write or make changes in resources directory.

    Where as application directory is read and write directory, where on runtime you can download images and other file and store. this is possible because application directory is read and write both.

  • The data directory is for storing data that is not part of your original application files. The resources directory is where your application files go. In your case, use the data directory.
