AudioPlayer vs Sound

Ti.Media.AudioPlayer is for streaming sounds where as Ti.Media.Sound is for playing smaller files locally as it caches the sound into memory before playing.

What I don't understand is Ti.Media.Sound contains all the properties and methods you would want to use for an mp3 player based app whereas Ti.Media.AudioPlayer hardly contains anything.

For starters, I have to use PHP to determine the duration of the MP3. If I was to use Ti.Media.Sound, it has a duration property, but I can't because waiting for the file to fully cache is silly.

Second, Ti.Media.Sound has a seek method. AudioPlayer does not. Why?
