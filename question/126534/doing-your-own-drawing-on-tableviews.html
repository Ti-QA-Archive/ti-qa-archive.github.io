  • Hi,

    Here you have a tip to improve a lot TableViews performance: borderRadius and transparency are truly EVIL. Yes, I know… a 4px borderRadius makes rectangles look beautiful, as so having a gradient background behind labels and images.

    In my case I had a TableView with 30 rows with an image (actually quite big: 100x100) and 2 labels (long TextField with such an username + TextField with user's detail) each. Rows background was white. There's a view containing the image and labels, since somewhere I read that having a view container helps -not sure it helps in the end…-.
    The first time I scrolled down it was jerky, choppy, whatever… you know.
    What REALLY helped was setting the background to both the view and labels. Now, it performs very very good, no more jerky stuff :)

    Actually, there's an opened ticket at jira for the next 1.9 SDK which lead me to try this.

  • The tableviews can be a bit pesky, but here is a way to speed up the scrolling performance (though resource costly so use with caution):

    • iOS: Cache the images as thumbnails in memory blobs and use these when setting the image property. I am using this technique for a tableview with around 90rows and the scrollling performance is great.
    • Android: Tableview scrolling performace on android is pain in the *, however if you are not using many tableviews in your app, try wrapping it into a scrollview, set the height of the tableview and the contentheight of the scrollview to the total height of all rows. This makes the scolling nice and smooth. Tested with SDK 2.0.

    Again, the above techniques can be resource consuming so use with caution.
