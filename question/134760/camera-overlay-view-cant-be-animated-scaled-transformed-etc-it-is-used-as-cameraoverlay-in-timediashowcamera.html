Camera overlay view can't be animated (scaled, transformed etc) it is used as cameraOverlay in Ti.Media.showCamera()

I opened a few questions with specific problems on iOS: 1, 2 and I think I've got to conclusion.

I am debugging this for 5h now and here is my conclusion: overlay view that is used for camera overlay can't be modified/resized/rotated (animated) etc.

It rearranges labels, buttons etc just the way they were in PORTRAIT mode and you can't see it's top and it's end since they don't fit now.

I tried adding that overlay to a Window and then open window instead of camera and it works as expected. But when that camera overlay view is passed as argument to Ti.Media.showCamera() it looks like you can't do anything to it (transformations).

If you let system to handle orientation changes it will always look at height, width properties.

So if you said something like this:

var view = Ti.UI.createCiew({
    width : platformWidth,
    height : platformHeight
})

And if you don't handle orientation changes then you will on landscape mode see view that is the same size as in PORTRAIT mode just rotated by 90.

But if you say this:

var view = Ti.UI.createCiew({
    width : '100%',
    height : '100%'
})

then your view will always fill the whole screen.

Whole those scenarios were tested in two ways. First one is that that view is added to a window object and that window is opened. And second is that that view is added to Ti.Media.showCamera.

So I think that view can't be animated/scaled etc because it does not have any parent view nor window so probably in Titanium SRC that was not considered and it does not know how to modify size an rotation of that cameraOverlay view.
