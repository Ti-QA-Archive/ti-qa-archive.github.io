    • You can create wildcard app ids in itunes connect, so you can use the same provisioning profile for multiple apps.

      Nevertheless your apps need a unique id, but can use the same PP.
      And with a profile using a wildcard app id, you can't use several services like In-App Purchase, iCloud, and Game Center

      — commented February 20th 2013 by Florian Walter

    • With Titanium 2.X and 3.0.0, there was only app id validation during project creation, but not at build time. That didn't make any sense.

      Shortly after 3.0.0.GA shipped, I added tiapp.xml validation during build operations. As apart of that effort, I added the same validation from project creation to build operation.

      To make things more complicated, Android's app id validation is stricter than iOS and Mobile Web. For example, com.appcelerator.final.123abc is totally valid on iOS and Mobile Web, but not on Android. Android cannot have parts that are Java reserved words or that start with numbers (i.e. 123abc).
