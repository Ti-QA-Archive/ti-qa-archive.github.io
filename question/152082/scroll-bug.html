Scroll bug?

This code tries to display a set of rows in an area with scroll vertically.

//Create some data
    var rows = [];
    for (var n = 1; n < 50; n++) {
        rows.push(Ti.UI.createTableViewRow({
            title : n,
            height : 'auto'
        }));
    }

    //Create the tableView
    var table = Ti.UI.createTableView({
        width : '100%',
        height : Ti.UI.SIZE -> fails on phisical device, works on emulator.
    });

    //Create the ScrollView
    var scrollView = Ti.UI.createScrollView({
        contentWidth : 'auto',
        contentHeight : 'auto',
        showVerticalScrollIndicator : true,
        height : '100%',
        width : '100%'
    });

    //And the windows
    var win = Ti.UI.createWindow();

    //Link
    scrollView.add(table);
    win.add(scrollView);
    win.open();

    //And set data
    table.setData(rows);

In a emulator works fine. But when installing in a device (Samsung Galaxy SII & Tab10) don't scroll.

I've tried to assign other 'height' properties to table object as '100%' 'auto' or none. Don't work. Only if assign a value as 5000 works, butI I want to adjust the height of the table to its actual content.

Any ideas?
