How to properly add a iOS Framework to a Titanium project?

Hello,

I'm trying to use this module ti_keychain (my fork: https://github.com/fmcypriano/ti_keychain) and this module requires the Security.framework. As you can see on GitHub the module is correctly configured even with the module.xcconfig:

OTHER_LDFLAGS=$(inherited) -framework Security

This is what I do add this module to a Titanium project:

$ cd <path to ti_keychain>/mobile/ios/
$ ./build.py
$ cp com.obscure.keychain-iphone-1.0.zip ~/Library/Application\ Support/Titanium/

The process completes successfully. And I add the module to tiapp.xml:

    <modules>
        <module version="1.0" platform="iphone">com.obscure.keychain</module>
    </modules>

When I try to run the Titanium I get a build error. That I solve by opening the build project in Xcode and manually adding the Security.framework.

But this isn't a good solution. It's manual and I've to repeat it every time I do a clean build. Is there a proper way to add a framework to a project?
