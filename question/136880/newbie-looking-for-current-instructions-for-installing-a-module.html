Newbie looking for current instructions for installing a module

I'm using Titanium 2.0 and need to install the StoreKit module on a Mac OSX machine I'm using to develop my app. Other posts in this forum talk about modifying the xml in the tiapp.xml file, but it looks like in the latest version that can also be done on the Overview tab.

I've read posts that talk about putting the module in the Library/Application Support/Titanium/Modules directory. When I expand the directories in the Library, I see files with a dylib extension. I don't see a dylib file in the download I got for StoreKit. Included in the download are folders for documentation and examples, which clearly won't be needed for my production app.

Other information I've read talks about putting things in the modules folder in the Titanium project.

Can someone help me understand the current process needed to download and install a module? Right now what I'm seeing is an Info message that says "[INFO] Detected third-party module: ti.storekit/1.5", but when the app loads in the simulator I get a red screen of death that says "Couldn't find module: ti.storekit".

Thanks!

    • for modules you want available to all projects, place in Library.
    • for modules unique to a project that you will never use anyplace else, place in {project]/modules
    • The scripts detect modules that are available and tells you that (what you see above)
    • module need to be declared in the tiapp.xml, but the tistudio 2.0 has a nice interface to help with that now
