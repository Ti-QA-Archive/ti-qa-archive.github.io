Alloy Model Collection SQL Sync Adapter configuration

Can I refer downloaded sqlite database file in configuring Alloy Model sync adapter configuration parameter "db_file" ?
for example
exports.definition = {
config: {
"adapter": {
"type": "sql",
"collection_name": "contacts",
"db_file" : "/app_appdata/databases/mydownloadedDB.sqlite",
"db_name": "contacts",
"idAttribute": "id",
"remoteBackup": false
}
}
};

Seems to me that alloy is looking for the sqlite file under "Resources/alloy/sync" as I am getting the following exception on the console:
W/System.err: java.io.FileNotFoundException: Resources/alloy/sync/mydownloadedDB.sqlite

It works perfectly when the database file is packaged into the App and under app "assets" or "assets/Resources" folder and use the following config

"db_file" = "/myprepackagedDB.sqlite". My application is built using alloy collections and I need to download the database from remote server, and I can't download or copy the file under app resources directory as it is readonly. Any suggestions?
