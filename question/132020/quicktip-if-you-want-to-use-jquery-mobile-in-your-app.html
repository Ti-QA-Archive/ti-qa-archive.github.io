QuickTip if you wan't to use jQuery Mobile in your app

Hello!

I've worked with an app recently where i used the jQuery Mobile navigation inside a webview, where the HTML is downloaded from a server and stored locally. All jQuery and images is also stored locally. But i ran into a problem! I couldn't use the "Back" button within the jQuery Mobile navigation, it just wouldn't work. So i found that window.history.back() doesnt work within a webview for local pages.

To get it working, i simply replaced the row that says

window.history.back();

with the following

window.location="#" + h.getPrev().url;

And you should get it working :)
Best solution? Probably not. Does it work? Yes :)

  • I encounter the same problem and this post really help. Thanks, Anton.

    I am using the latest jquery moible version 1.1.1, the amendment is a bit different, so post here for those encounter the same problem:

    In jquery.mobile-1.1.1.min.js, change the following line of code
    From:

     if (!this._isClosed) this._isClosed = true, a.mobile.hashListeningEnabled ? c.history.back() : a.mobile.changePage(a.mobile.urlHistory.getPrev().url)

    To:

    if (!this._isClosed) this._isClosed = true, a.mobile.hashListeningEnabled ? c.location=c.location.pathname+"#"+a.mobile.urlHistory.getPrev().url : a.mobile.changePage(a.mobile.urlHistory.getPrev().url)
