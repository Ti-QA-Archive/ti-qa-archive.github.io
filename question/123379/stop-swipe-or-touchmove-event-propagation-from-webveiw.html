Stop swipe or touchmove event propagation from WebVeiw

Hi,

My application shows several WebViews in a ScrollableView component. The HTML in the WebViews can contain image galleries. These galleries can be scrolled by swipe. The problem is that when a user swipes horizontally over a gallery inside the HTML, the swipe event is propagated to the ScrollableView component and the currently visible WebView changes.

Is it possible to stop that event propagation from the WebView?
Maybe I could mark the event somehow in the JavaScript included by the HTML and then check that marker and ignore the event in Appcelerator's code? (I have full control over the HTML/JS rendered in WebViews)
