Android/iOS Icons Structure and Use (not appicons/splashscreens)

Currently i try to fix my blurred icons in the android version of my titanium project.
I read a lot of docs and know that there's a best-practice directory-structure for the different android resolutions. unfortunatly i can't get it to work so far, so any help would be appreciated. I'm using Titanium SDK 3.1.2

I reference my icons something like this:

    image: "/images/icons/xyz.png",
});

My assets are in:
/Resources/android/images/icons..
/Resources/iphone/images/icons..

So anyone can tell me how to use the right icon for the right resolution in android of the icon.pngs?

    There are really a couple of ways of doing this.

    1) The cheat but very effective method.

    Make sure you have dp selected in you tiapp.xml file.

    Create all your images in high res. within your current android/images/icons directory and size them on the image object to work on a 320 x 480 screen. The other Android device will adjust the image accordingly. This only works for images where you can set the size and be specific.

    2) The proper way

    look HERE

    However you will increase the size of you app as you have much more actual images.

    You have to use this method if you are using icons on a tab group for example, as you cannot set the size.
