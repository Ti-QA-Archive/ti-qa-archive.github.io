  • Hi

    It is less likely that the state of the build folder is affecting your app and more likely that you have either got one of the following issues;

    • You have previously added one or more tables to the database since it was installed, this is why it works on the emulator and not the device.
    • You have a case problem with the table/column names when on the device. Consider checking the original database to make sure that for example a table name "Test" is not referenced in the code as "test" or "TEST", all three are considered different. The emulator and devices can handle this type case differently.

    If the build folder had stopped part way through you would end up with an app that would fail on more than a single table in the app.

    Consider editing the database on your computer and reviewing the tables and columns match exactly as you are programming for.

  • I don't know why this problem happened but I figured out a way around it…

    I used a sqlite database browser to export all my tables and data to .csv files. I then deleted all tables and then imported the CSV file to recreate the tables again. A clean is probably not a bad idea (I did so many I can't remember/know which one fixed it or when to run it). After rebuilding the DB, I was able to install and run my app on an android device as well as them emulator. Hope this helps someone else.
