    I think what you are referring to as global.jss is actually app.jss (this mirrors the app.js file that always gets loaded when a TiApp is run.

    In this way you could possibly include other styles. But I've found I need to create .jss files that mirror/match the name of the file that is requesting a style.

    eg. if the filename is mywidget.js then I will need a mywidget.jss file for any referenced styles to work.

    The documentation really does need better explanation.

  • I think perhaps the documentation is out-of-date as the KitchenSink demo has no global.jss either. It appears the best way forward is to simply "include" the common.jss as required from all other *.jss files. This is how the KitchenSink demo does it as well.

    For example:

    // common.jss
.window { 
    background-image:'/image/myimage.png'; 
}

// widget.jss
@import 'common.jss'; /*** Note this line ***/
.widget_label { 
    text:'helloworld'; 
}
