Is it possible to set the quality for ti.barcode module in android?

Hello,

I just installed ti.barcode and although the module works it needs barcodes to be really big and clear to be able to read them. I can see that the module is not using the full resolution of the camera, in my case I am testing with a Samsung Galaxy S4.

I am wondering is there any setting or anything I can set to tell the module to use the best quality? Right now as it is it is almost useless for a real project.

Thanks in advance for any tip.

  • I had the same problem. You can get the source and edit CameraConfigurationManager.java. You will need to update the:
    private static final int MAX_PREVIEW_PIXELS = 2048*1152

    • which is the smallest HD selection for the Samsung Galaxy S4.

    • you are quite right, you also have to edit the

      private static final int MAX_FRAME_WIDTH

      in the CameraManager.java file to expand the scanning area, else it still tries to use the postage stamp in the middle of high res phones. Try fitting it on a LG g3…

  • We ended up using the scanditsdk module for our last project requiring bar code scanning. Needed code 39, so it wasn't free, but the barcodes our client uses are long and narrow, and the ti.barcode wouldn't recog them.

    We tested it and redlaser, both were decent regarding recog and acquisition. Don't remember why we chose scandit over the other though.
