Android imageview width percentage

App Type: Android
SDK: 3.2.0
Device: Galaxy S3 i9300
Version: Android CM 4.3.1
Host: OSX 10.9.1
Alloy

Just added imageview to xml:

<Alloy>
    <Window>
        <ImageView image="/images/logo.png" id="logo"></ImageView>        
    </Window>
</Alloy>

Then tried to change the width of image to 50% on tss

"#logo": {
    width: "50%"
}

It's perfect on iOS but Android is ignoring the width, any clues?

  • I'm having the exact issue. The jira bug here:

    https://jira.appcelerator.org/browse/TIMOB-15747

    addresses the issue but it has not been resolved.

    If you only set the image width on Android, the image will not scale properly to the width specified. It is constrained by the image's height.

    A height value must be set for the width to be correct, which both defeats the purpose and also is not what the documentation states.
