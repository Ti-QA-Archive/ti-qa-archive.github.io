Does Alloy handle events from ListViews differently in Android vs iOS?

I was having an issue with a ListView and decided to test the same principal using Tony Lukasavage's github code.

I changed the sections.xml found here to:

Modified sections.xml

<Alloy>
    <Tab title="sections">
        <Window title="sections">
            <ListView id="list" onItemclick="onItemClick">
                <Templates>
                    <ItemTemplate name="template1">
                        <Label class="brief" bindId="brief" left="10dp"/>
                        <Label class="url_text" bindId="url_text" left="200dp"/>
                    </ItemTemplate>
                </Templates>
                <ListSection headerTitle="title 1">
                    <ListItem template="template1" brief:text="test text 1" url_text:text="test text 2" title="row 1"/>
                    <ListItem title="row 2"/>
                    <ListItem title="row 3"/>
                </ListSection>
                <ListSection headerTitle="title 2">
                    <ListItem title="row 1"/>
                    <ListItem title="row 2"/>
                    <ListItem title="row 3"/>
                </ListSection>
                <ListSection headerTitle="title 3">
                    <ListItem title="row 1"/>
                    <ListItem title="row 2"/>
                    <ListItem title="row 3"/>
                </ListSection>
                <ListSection headerTitle="title 4">
                    <ListItem title="row 1"/>
                    <ListItem title="row 2"/>
                    <ListItem title="row 3"/>
                </ListSection>
            </ListView>
        </Window>
    </Tab>
</Alloy>

Unmodified sections.js

function onItemClick(e) {
    // get the clicked section
    var section = $.list.sections[e.sectionIndex];

    // get the clicked item from that section
    var item = section.getItemAt(e.itemIndex);

    // print the item's title
    Ti.API.info('sectionIndex: ' + e.sectionIndex);
    Ti.API.info('itemIndex: ' + e.itemIndex);
    Ti.API.info('itemclick: ' + item.properties.title);
}

In iOS it reports back the sectionIndex, itemIndex, and title when you click on "test text 1" or "test text 2" but on Android you get nothing. However, if you click on the "empty" space within the text item it reports properly. Consequently you can't access the bindId of the ListItem.

  • Two things that will probably work:

    1. Set touchEnabled: false for the labels
    2. Execute e.cancelBubble = true; at the beginning of the event handler.

    • I tried both solutions alone and together.

      • touchEnabled: false solves the triggering of an event but doesn't report the bindId.

      • e.cancelBubble = true; doesn't seem to have any affect.

      I did more research last night and found this jira ticket that seems to be related. So I reverted back to 3.1.1 and voila. It worked. I was able to trigger an event and get the bindId of the view that was clicked on both Android and iOS without setting touchEnabled to false or cancelBubble to true.
