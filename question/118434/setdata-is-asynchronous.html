setData() is asynchronous?

Is TableView.setData() asynchronous? If so, how do I know when the data is loaded? Can I make it asynchronous?

Example:

// prints all my rows
Titanium.API.info(td);

// set the table data
tableView.setData(td);

// prints NULL
Titanium.API.info(tableView.data);

And before you ask, yes, the tableView works and displays data. It just that its "data" property doesn't seem to be populated right away. I don't see a TableView event for data being loaded or anywhere to insert a callback.

Help!

  • It's not asychronous per se, its getting populated on a different thread (the UI thread I think) and your javascript continues on its own so because it takes some time to render the tableview your code may race past that happening. I dont think there's an answer other than making sure you optimise the tableview load time or just make the situation more bearable for your user by putting up an activityIndicator and guestimate the time it needs for the rows to load before removing it again

    Coming back to answer my own question MUCH later. This still continues to be a problem wit hTableView, but the new ListView API gives you access to the its "data" right away, allowing you to manipulate it like any other Javascript array. I'd highly suggest using ListView for anyone who encounters a similar situation.
