Textfield and soft-keyboard bug

I'm developing an iOS app.

I have 2 elements, a textfield at bottom and a conversation (tableView) above. They're wrapped by a View (temporary called A).

As you know, If A is non-scrollView, textfield got hidden by soft-keyboard when focused. So A must be a scrollView. But I still got trouble. Because A is a scrollView, the textfield is also scrollable as all elements inside A. It's got hidden by soft-keyboard again. After that, I type some text, the textfield scrolls up. But there is a space between it and soft-keyboard.

To understand further, please watch this video http://youtu.be/ev4qO17OKFA.

Help!!!

  • SOLVE!

    For everybody have same problem:

    1.Make 2 non-scrollView, call them A and B. A contains B. B contains 2 elements that I describe above.

    2.Add 2 event listener for textfield.

    • focus: animation slide up B.

    • blur: animation slide down B.

    (bottom : 216px in PORTRAIT and 162px in LANDSCAPE)
