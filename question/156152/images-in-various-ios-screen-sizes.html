    Hi

    Yes perfectly now.

    In iOS you have two image densities; standard and retina. Both images are placed in the same location - but the retina one gains a suffix so the operating system can tell them apart.

    Example;

    • image.png standard
    • image@2x.png retina

    Note the suffix @2x before the file name. This way both files are placed in the same folder.

    So if you created an image that was 200x200 standard size - you would create a 400x400 version and add the suffix.

    There is an additional suffix you can include which let's you provide images for the tall retina screens; -568h@2x. This says this image is used if the screen is retina AND tall. This image is optional and only required if you wish a specific graphic to be used for tall. It is very useful if you supply a background image;

    • background.png standard
    • background@2x.png retina
    • background-568h@2x.png retina tall

    Let me know if you need more.

    • I assume you mean it automatically uses the @2x versions of each file rather than you adjusting the filename?

      Your description also states the smaller images are 20x2 but the bigger ones are 40x40, is this a typo? Did you mean 20x20 rather than 20x2?

      I am surprised that you have this strange movement - if your normal size images are 20x20 then a top of 20 on the second image should provide the correct gap.

      I would always define the width and height of images;

      var image1 = Ti.UI. createImageView({
    height: Ti.UI.SIZE,
    image: "image1.png",
    left:0,
    top:0,
    width: Ti.UI.SIZE
});
var image2 = Ti.UI. createImageView({
    height: Ti.UI.SIZE,
    image: "image2.png",
    left:0,
    top:20,
    width: Ti.UI.SIZE
});

      If this does not fix the issue then try to add a parent view and layout, this will think the next for the previous view as they are added.

      var view = Ti.UI.createView({
    height: Ti.UI.SIZE,
    layout: 'vertical',
    top: 0,
    width: Ti.UI.FILL
});
win.add(view);
var image1 = Ti.UI. createImageView({
    height: Ti.UI.SIZE,
    image: "image1.png",
    left:0,
    top:0,
    width: Ti.UI.SIZE
});
var image2 = Ti.UI. createImageView({
    height: Ti.UI.SIZE,
    image: "image2.png",
    left:0,
    top:0,
    width: Ti.UI.SIZE
});
view.add(image1);
view.add(image2);

      Either way, plus confirm the images are 20x20 and 40x40 and you are not doing anything to adjust the filename yourself.
