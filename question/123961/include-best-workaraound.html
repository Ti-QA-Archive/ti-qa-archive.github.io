include, best workaraound

Hello everyone, I need some advice about the include workaround.

Situation:

My app have a single window, without tabs, the executed filename is always index.js.

In the first launch I check if is set a global var named "global_var"

If global var is set index.js include "file_name.js" (file_name is the name of file contained in the var global_var).

If global_var isn't set user must choose from 2 possibility, then, on click, the correct file is included with the same method.

So, this snippet of my index.js is something like this

if(global_bar) { Ti.include(global_var + &#39;.js&#39;) } else { var view = choose_view.create(); &#x2F;&#x2F;THERE ARE 2 BUTTONS ADDED TO VIEW &#x2F;&#x2F;ONCLICK BUTTONS SET var_global TO page_one OR page_two AND THEN INCLUDE THE CORRECT PAGE Ti.Include(var_global + &#39;.js&#39;); }

Ok, for now my code work perfectly.

Now, I made a modal window to make possible change the user first choice.

When the modal is opened user can change the first choice and than, when the modal is closed, the user return in index.js

This is my problem: When the user return in index.js after closing the modal window, there is the previously included file, so how I can "remove" this and include the new one? Is there a best workaround to make what I have to do in a single page?

Thank you, I hope everything is clear.
Stefano.
