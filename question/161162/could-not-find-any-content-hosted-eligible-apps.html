  • Hello Stephen….

    my X-code version is 5.0, IOS version is 7.0

    I created test user and test products in in app purchase option under my application in itunes connect.

    I want to display content for each test products when user buy my free products.

    For displaying content i followed the below process:

    Creating App Store Hosted Content:

    In order to have the App Store host your content, the content must first be put in a package. This guide will show how to create this package.
    Notes:

    Before starting the steps below, login to iTunes Connect, create a "Non-Consumable" in-app purchase, and select "Yes" for "Hosting Content with Apple" for that purchase. In for the purposes of this guide, the "Product ID" of that purchase will be "DownloadablePop".
    What is the Package?

    The package is a structured folder, that contains a ContentInfo.plist in the root level along with the content. The ContentInfo.plist has two keys:

    ContentVersion - The version number of the content.
IAPProductionIdentifier - The product identifier of the product that the content is associated with. It is the identifier you defined in iTunes Connect.

    The package must not contain executable code or be larger than 2GB.
    Creating the Package.

    The best way to make this package is using Xcode.
    Step 1: Create a New Project in Xcode

    "File" > "New" > "Project"
Under "iOS" select "Other" on the left hand panel of the template screen.
Select "In-App Purchase Content" on the right hand side.
Click "Next".

    New Project Template
    Step 2: Enter project info

    Product Name - Enter the product identifier that you defined in iTunes Connect.
Organization Name - The Organization name.
Company Identifier - The Company identifier.
Click "Next".

    Project Info
    Step 3: Where to save the project

    Select the location to save the project.
Click "Create".

    Step 4: Edit the ContentInfo.plist

    Expand the "Supporting Files" section of the project navigator panel and select ContentInfo.plist.
Update the two keys in this file.
    ContentVersion - The default version is 1.0, update this as necessary.
    IAPProductIdentifier - Change this so that it matches the Product Identifier you defined in iTunes Connect.

    ContentInfo.plist
    Step 5: Adding Contents

    Using Finder, select the file(s) that you want to add to the package.
Drag and drop the files onto the "Supporting Files" section.
Click "Finish"."

    Adding Contents
    Step 6: Archive the Project

    Select "Project" > "Archive" from the menu.

    Step 7: Validate the Archive

    Select "Window" > "Organizer" to open the Xcode Organizer.
Select "Archives" at the top.
Select your "Archive" on the left.
Select the version you want on the right.
Click "Validate" on the right.
Fill in your iTunes login info.
Select your In-App Purchase Content from the drop down.
Click "Validate".
If there is a validation error, resolve the issue and repeat the above steps until validation passes
(Validation Succeeded) Click "Finish".

    Archive in Organizer
    Step 8: Submit the Content

    Still in the Organizer, Click "Distribute".
Select "Submit in-app purchase content".
Click "Next".
Fill in your iTunes login info.
Click "Next".
Select the purchase that your content is for.
Click "Submit".
Click "Finish".

    Submit
    Alternate Step 8:

    Still in the Organizer, Click "Distribute".
Select "Export Package".
Click "Next".
Select a place to save the pkg.
Click "Save".
Click "Finish".
Use Application Loader, a free tool provided by Apple, to upload the pkg. Application Loader can be found under "Xcode.app/Contents/Applications/Application Loader.app".

    Export Package
