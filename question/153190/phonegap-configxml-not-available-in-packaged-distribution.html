PhoneGap config.xml not available in packaged distribution

Hi,

I'm generating native iOS + Android applications as well as a mobile web application from my Titanium project.

I want to use the mobile web application to publish native/hybrid apps on Windows Phone + BlackBerry using PhoneGaps Build service. This service can be configured using a config.xml. Therefore, I've placed a config.xml in the root directory of my Titanium project because the config.xml file needs to be in the root folder. But when packaging mobileweb for distribution, there is no config.xml in the generated package/folder. Guess it's ignored by the build process.

Am I doing something wrong? Is it bold to ask for such a feature (not ignoring config.xml)?
