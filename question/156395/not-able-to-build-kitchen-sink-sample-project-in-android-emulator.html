    • I have a suspicion as to why it does not work on the emulator. Can you see this error in the console?

      • requires unavailable shared library com.google.android.maps

      The latest Kitchen Sink app provides revised support for Google Maps v2 for Android, by doing so it may well be stopping the KitchenSink app from working on your emulator. This is due to restrictions with the Google Maps v2 for Android, as found in the documentation;

      • http://docs.appcelerator.com/titanium/latest/#!/api/Modules.Map

      It shows the following requirements;

      • Google Maps API key (required for both development and production)
      • Google Play services SDK installed using the Android SDK manager
      • This module only works on Android devices. This module is not supported on the Android emulator

      It appears the KitchenSink app really does need to be used in conjunction with a physical device when testing.
