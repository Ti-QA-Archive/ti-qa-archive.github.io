Gcm not working on Android 2.3.x

As part of the discussion in the link below, there is a problem with Gcm and the ACS push notifications for android, version 2.3.x.
https://developer.appcelerator.com/question/157032/cloudpushretrievedevicetoken-error

I hoped this was solved with SDK 3.2.0, but unfortunately I still have the same problems. Until now we solved it with using MQTT, but that is being discontinued as of December 31.

Today, I have setup gcm in our app again and it still works perfectly on Android 4.x, but however when testing it on Android 2.3.6 I have this problem:

  • When calling retrieveDeviceToken: the app shows an alert on device with error: SERVICE_NOT_AVAILABLE

  • In log of phone during startup: dalvikvm / Could not find class 'android.app.Notification$Builder', referenced from method ti.cloudpush.CloudpushModuleGeneric.showTrayNotification

Obviously, subscribing to a channel and receiving push messages do not work.

Versions:

  • SDK version: 3.2.0.GA

  • Cloudpush version: 3.2.0

  • Device: Samsung GT-I9001 (android 2.3.6)

I am curious if there is anybody out there have a working version of acs push notifications by gcm on android 2.3.x. Thanks!

    This is still not resolved in 3.2.0? :(

    Ok, I have GCM working in 2.3.6 with the ti.cloudpush 2.3.3 version (you must download it, I don't remember where I got it). Just copy it to modules folder and select the 2.3.3 version in the tiapp.xml (right-up corner).
