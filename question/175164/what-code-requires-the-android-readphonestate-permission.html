What code requires the Android READ_PHONE_STATE permission

Does anyone know what causes the Android permission READ_PHONE_STATE to be included in the AndroidManifest.xml? I'd like to not include this permission.

Rather than make a custom manifest excluding it and risking some code failing in the app, I'd like to, if possible, stop using the code that's causing it to be included.

I thought it might be Ti.Platform.id but after removing that code, cleaning the app, and rebuilding, the permission is still being included in AndroidManifest.xml.

I searched the github Titanium Mobile source for READ_PHONE_STATE but only found it in xml files, not .java files.

Any help appreciated.

  • According to the docs you can prevent the permission being included 

    Note: If both your minSdkVersion and targetSdkVersion values are set to 3 or lower, the system implicitly grants your app this permission. If you don't need this permission, be sure your targetSdkVersion is 4 or higher.
