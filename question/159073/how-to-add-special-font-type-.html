    • I did this today on one of my own projects;

      • You do not need to edit the info.plist

      • You do not need to go anywhere near the

        {appname}/resources/ folder

      • Copy your font into the following folder; (should be and otf or ttf - I tend only to use ttf)

        {appname}/app/assets/iphone/fonts/yourfont.ttf

      • Close the simulator

      • Use the clean menu option
        • Menu > Tools > Clean > Select Project > Confirm
      • Run the app into the simulator again.

      You MUST reference the font inside the app using the PostScript name which can be found by checking the font details in *Font Book".

      For example a font file called TitilliumText22L001-webfont.ttf actually uses a PostScript name of TitilliumText22L-Light this is the name used inside the tss.

      // example usage
".lbl": {
    font: {
        fontFamily: 'TitilliumText22L-Light'
    }
}

      The fonts will not be used until you have cleaned the project folder. If it still fails, un-install the app from the simulator in the same way you would do on your own device. Clean again and see what happens.
