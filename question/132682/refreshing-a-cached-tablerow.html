Refreshing a cached tablerow?

On Android with SDK 1.8.1

I am using several tables throughout my app. All the rows have a 'favorite' icon that works like an on/off toggle.

I am using a standardview with a background image, the image gets toggled to a bright or a dark star, depending on state.

However, when using cached rows (ie. giving the rows a 'className') - they do not update on click, I have to scroll them out of view and back in, in order to see the effect of the changed background image.

In order words: The user has to refresh them by pulling them out of view and back in.

Strange thing is that this 'bug' only applies to the rows that are out of view initially. Meaning that the first rows, the ones that are immediately visible, seems to work all right.

The only solution I've found to this, is avoiding className, but some of the tables are quite heavy.

One of my tables has a search box, and if I use className on that table, a favorite can appear 'unfavorited' as soon as the filter kicks in.

Is there a way I can both have my interactivity AND my caching?

  • I don't think you should avoid use of class name as it may result in crashing of your table view in android. So you should use classname property always with tableview in android.

    What are you doing while updating your table view row ? simply setting like img.image ='on.png' ? If yes than you can give a try to update your row by using tblview.updateRow() function. This may result in what you are expecting.
