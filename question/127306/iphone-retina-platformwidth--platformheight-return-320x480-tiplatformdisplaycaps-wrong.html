iPhone Retina: platformWidth & platformHeight return 320x480 (Ti.Platform.displayCaps) WRONG!

An iPhone 4 Retina display is 640x960, but Ti.Platform.displayCaps.platformWidth and Ti.Platform.displayCaps.platformHeight are returning 320x480.

platformWidth and platformHeight are supposed to return the actual screen dimentions, in pixels. This seems to work fine on Android devices, and on my iPod Touch 3G. But on the iPhone 4 I just installed the app to, it reports 320x480, with a dpiof 320. This is incorrect.

Can anyone else confirm this is a bug?

  • This is actually correct. The size of any iphone screen in pixels is 320x480, regardless of model. On the retina display, the dpi will return 320 and on a normal display, the dpi will return 160. On Android, the dpi is device specific.

    The idea behind this is that the retina display simply crams twice as many points into the same amount of space to create a higher density image. So while an image for a retina display may be 640x960 pixels in size, it is actually displayed in a 320x480 space.

    • You get the actual dimensions of the screen in pixels on iPhone as well, just as you do on Android. When you need to deal with pixels on the screen, you will always need to deal with the same surface area, the dpi will not make a difference when placing something 10 pixels from the left edge of the screen, it will just be 10 pixels.

      Ideally cross-platform gives you exactly what is given by the default platform in this instance. If we were to change that, we'd move out of Apple's own guidelines for describing the device environment. "if iphone" tells you almost everything you need to know in 1 statement. Dealing with all of the different Android densities and screen sizes does not mean that it should be as difficult on the iphone side, as there are far less variables to deal with.
