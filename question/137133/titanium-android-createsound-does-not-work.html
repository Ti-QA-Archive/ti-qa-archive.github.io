  • Is your code in a subdirectory? On Android, relative paths are interpreted relative to the current file. So if this code is in a subdirectory of Resources, you might try '../1.mp3', as in the kitchen sink sound sample. However, that doesn't explain why '/1.mp3' isn't working, so you could be seeing a bug here.

    You might also try accessing the MP3 file as a file and checking the nativePath property. This should be what you need for the Sound URL. Something like this:

    var soundURL = Titanium.Filesystem.getFile(
      Titanium.Filesystem.resourcesDirectory,'1.mp3').nativePath;
