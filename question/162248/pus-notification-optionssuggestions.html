    • Depends on your needs and plans for the future.
      My main priority would be to have a Titanium module - this in case you are not ready to build one.
      Then how the price will scale if you have lots of pushes.

      So some solutions would be:

      ACS - Appcelerator's own platform, you get lots of free pushes/month/app - like 5M api calls to share among all the features and the module is included in the SDK.

      UrbanAirship -module: https://github.com/appcelerator/titanium_modules/tree/master/urbanairship/mobile

      pushwoosh has some free plan, not sure they have a module for android, you'll have to check

      parse.com not sure thee is a module for Ti, you'll need to find this

      Another way would be to use your own server to send the pushes and use in iOS the already existent registerForPushNotification and the gcm.js module for android.

      For the server you'll have to search yourself something that suits you (php, nodejs, etc)

      There is also Amazon that has push notifications with a very good price, however you will need to write an adapter for it so for the device to register with it (in javascript of course), but it should be pretty easy - I tested with gcm.js on android from the AWS console and it works like a charm, and it should be the same with iOs.

      There might be other solutions, so I'd just use google a little longer :)
