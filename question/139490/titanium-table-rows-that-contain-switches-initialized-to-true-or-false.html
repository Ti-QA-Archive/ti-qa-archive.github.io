Titanium Table rows that contain Switches, initialized to true or false

I have no problem in including a true/false initialized Switch in table rows, using Titanium.UI.createSwitch,
but how can I retrieve their values? The thisSwitch.addEventListener, I employed here, does not seam to do the job
!
Some of the data I need from a clicked on row can be easily retrieved by clicking on the left side of a
row that does not contain the clickable Switch icon. Here the table.addEventListener works jut fine, however, I would like to retrieve all three piece of data (the name, the id of the row and the Switch value) by just by clicking on the Switch icon of the row.

Can someone please help me solve this problem? I would greatly appreciate it!

A simplified test code that demonstrates the problem is listed below.

1) Application Type: Mobil
2) Titanium SDK version: 2.0.1 (04/12/12 16:33 999c68a)
3) IOS 5.0, Xcode 4.3
4) IOS Simulator
5) Host Operating System: Mac OS X version 10.7
6) Ti Studio build: 2.1.0.201206251749, Date: 25/6/2012

———————————— test.js ————————————————-

var defaultColor = '#000';
var win = Titanium.UI.currentWindow;
var ilen = 6; // hardwired for test purposes

var tableData = [];

for (var i=0; i<ilen; i++) {
var thisLabelName = Ti.UI.createLabel({
color:'#000',
text:"Text Line ("+i+")",
objectId:i,
objectName:"T-Line-"+i,
touchEnabled:true,
font:{fontSize:14, fontFamily:'Verdana',fontWeight:'bold'},

    value:'true',

    value:e.value,

    });

    layout:"vertical",
    textAlign:"center",
    selectedColor:"black",
    left:1,
    height:50,
    top:1
    });

thisRow.add(thisLabelName);

    Each object on the row is a child of the row, these are myrow.children[].

    First object is myrow.children[0]

    Total number of children is myrow.children.length

    Your switch is the 2nd children or myrow.children[1]

    Should be able to access its value as myrow.children[1].value
