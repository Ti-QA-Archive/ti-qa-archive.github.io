Focusing TextArea, Focused Event then Blurred Event Called

Hi,

I have a textarea with callbacks to the focus and blur events. However, when I select it, the focused event is called, then ~100ms after, the blurred event is called. The textarea is still focused, and it wouldn't be a problem with this event being called. However, the blur event is now intervening with my app logic.

At the moment, I have a global variable called finishedFocus, which is set to false when focusing, and then set to false after 1s with a timeout. This, in combination with a global variable activeTextAreaId, can cancel the blur event if it wasn't meant to be called.

However, this is super hacky. Can I fix this another way?
