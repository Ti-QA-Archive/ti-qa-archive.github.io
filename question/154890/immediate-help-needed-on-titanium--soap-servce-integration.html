  • SOAP WebServices are usually not recommended when using mobile applications. Mostly for performance reasons. An XML file should normally be more verbose when serving information. Where with JSON, it is basically a simple dictionary.

    Also, the XML parsing (while still fast) will never be as fast as loading a JSON object; especially with Titanium.

    Of course, while using JSON over XML is the recommended way, there is absolutely nothing stoping you from using XML for your WebServices. This is usually the case when you already have a web service in place and want to leverage that actual service. But if you have the chance to develop the service from scratch, then JSON would be your best bet.
