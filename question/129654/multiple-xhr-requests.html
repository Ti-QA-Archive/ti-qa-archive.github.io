Multiple XHR Requests

Hi! I'm building an app that uses REST API to make several calls to a server, so after getting that server data I can use it on my app…

My problem is that I need to do various calls (I require several objects) at the same time and I understand xhr calls are asyncronous… So what I want to do is:

firstCallToServer();
secondCallToServer();

but I NEED secondCallToServer() to execute AFTER firstCallToServer() ends. Is there a way of knowing this without having to nest the functions?

    Quick answer yes…

    Long answer.. wait for the onload to complete then run the next occurrence.

    Longer answer.

    you can allocate the http request to an array element for each occurrence then null out that element when the request has completed to save running out of memory. http requests keep the request open until it automatically closes, and if you try to set a second request using the same variable name it will use the existing request. So you are correct in creating separate requests.

    To see an example of this have a look at this Q & A.. I have posted a solution, read the comments though I got the onload and onerror case sensitive…

    http://developer.appcelerator.com/question/129564/running-out-of-memory-when-downloading-multiple-files-with-httpclient-in-android

    Hope this helps
