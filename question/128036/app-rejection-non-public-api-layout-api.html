    • While we can't be absolutely sure what Apple is detecting, we have a good guess now, and the latest change reflects this:

      Your app does not have to use the 'layout' property for it to appear in the code. We have hardcoded locations which use the string "layout", a method called setLayout:, and a data structure with a 'layout' member. It appears that Apple was picking up on the last one; it has been renamed and this will hopefully avoid future conflicts. To be safe, we have also obfuscated setLayout: and all of the "layout" hardcoded strings as well.
