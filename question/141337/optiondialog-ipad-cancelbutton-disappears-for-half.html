Optiondialog ipad cancelbutton disappears for half!

Application type : mobile

Titanium SDK : 2.1.1.GA

Platform : iOS 5.1

Device : iPad simulator

OS : Mac OS X

Hello,

I'm trying to create an optiondialog with more items than can be displayed and an cancel button. But when i run it on the iPad simulator, the cancel button disappears for half!

Is the use of a cancel button not common for iPad (I know you can also close the optiondialog by clicking next to it, but I like to use the cancel button) or is this a bug?

Here is the code i use:

  var optionDialog = Titanium.UI.createOptionDialog({
    title : "title",
    options : ['option1', 'option2', 'option3', 'option4', 'option5', 'option6', 'option7', 'option8', 'option9', 'option10', 'option11', 'option12', 'option13', 'option14', 'option15', 'option16', 'option17', 'etc', 'cancel'],
    cancel : 18,
});

    On iPad the cancel button shouldn't even show up. I'm pretty sure the fact that you can see even half of it is the bug. You can see this in the docs under ipad here: http://docs.appcelerator.com/titanium/2.1/index.html#!/api/Titanium.UI.OptionDialog
