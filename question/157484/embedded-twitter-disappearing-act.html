Embedded Twitter disappearing act

I have an embedded Twitter timeline in my app (iOS and Android) which for the past few days appears to be coming and going. I'm loading an html page containing the latest the latest Twitter embed code into a webView. Sometimes it loads the timeline and sometimes it doesn't. Given their code creates its own DOM elements this can't be a case of the DOM not being ready.

And indeed in a browser test the Twitter code is loading it's iframe to which it adds the class name 'twitter-timeline twitter-timeline-rendered' - only the <body> in it's associated doc is often empty! Then a while later a page refresh produces the timeline as it should.

This is happening in both iOS and Android (which use the same html doc) so it's not platform specific.

Is there some weird blacklisting of my ip address going on here which clears a while later? Or are there known problems with Twitter's servers? Their status page does show a less than 100% record of being up over the last few days.

Mobile /Titanium SDK version 3.1.3.GA/ Android & iOS 7/ Mac OSX

  • I think I've worked out what's going on. I'd mistakenly included Twitter's 'widgets.js' in the head of my html doc (mainly as I'd seen it in a page running on a website) and completely missed the fact that it's the embed script which adds 'widgets.js' programmatically. So I guess with 'widgets.js' already there and my embed code trying add it again a recipe for disaster was being created!
