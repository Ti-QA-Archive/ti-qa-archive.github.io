Webview Disappears after clicking Back and then reclicking window containing webview

Hi,

I know I had seen a solution for this but had come across it while searching for a different issue (that I can't remember). Now that I am searching for an answer to this I can't seem to find it, so sorry if it is repetitive.

I have a window that contains a webview, and when I click "Back" on the webview page, go back to the Home page, and then click on that same webview window again, the screen is blank.

I am not really sure if posting code will help, but I will post my window call for the page:

gg.ui.createFAQWindow = function(){
    Ti.API.info('FAQ function called');
        var win = Titanium.UI.createWindow({
            title:'FAQ',
            backgroundColor:'#000000',
            barColor:'#fff',
            backButtonTitle:'BACK',
            titleimage:'iphone/appicon.png',

            });

        var webview = Titanium.UI.createWebView({url:'faq.html'});
        win.add(webview);
        return win;
};

Thanks for any help!

    Call the function that creates the window in the click event of the element you use to open it.
