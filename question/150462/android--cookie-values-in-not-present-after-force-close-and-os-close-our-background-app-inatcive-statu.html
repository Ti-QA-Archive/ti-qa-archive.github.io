Android : Cookie values in not present after force close and OS close our background app (inatcive statu)

Hi,

About my issue:
I have to login into a website and it will set cookie values. for this I used httpclient with username/password in post request.

But, When my app is background and/ or (user or OS) force close the app, or app crashed.

And Open the app again it's again asking for login (trying to access to resource which need authorization, getting unauthorized httpcode). If it unauthorized response from server I am redirecting to login page.

If I close the app by using Back button (h/w) and reopen. And try acceess that same resource is works fine (just for checking user is still authorized or not).

Same code works fine in iOS even force close the app or crashed app re-open the app works fine. No need to enter username/password again until cookie expire.

Thank you,

      my suggestion is Save Cookie(s) values in App properties. When app is opened again. Please read that Cookies values check with time expire. If it is not expired set "cookie" request header.

      xhr.setRequestHeader(&quot;Cookie&quot;, Ti.App.Properties.getString(&#39;MYCOOKIE&#39;));
