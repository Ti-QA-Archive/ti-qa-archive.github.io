  • Accepted Answer

    Ideally if the data isn't going to change anymore often than you would update your app, then storing locally will give you better performance. The complexity and size of your stored data should determine whether you should store it as sqlite or with simple JSON.

    If the dataset is modest enough that you could load the whole thing in memory, storing it in JSON and loading it all up would be sure to give you your best performance. You'd simply be accessing an array of Javascript objects at that point, instead of having to query sqlite. If the dataset is large and requires some more complex logic to query, perhaps sqlite would be your best path.
