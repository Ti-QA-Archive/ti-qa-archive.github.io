Signed Android App Launch Timeout problem

Hi There,

I have a couple different physical Android devices. An older Samsung Galaxy S and two Samsung Galaxy s3 's. My app works fine on both when I build and deploy to the device from Titanium Studio. It also works on my Samsung Galaxy S when I publish the app and email it to myself and then install. It doesn't work when I do the same thing (publish and email, or publish to Play and download) on the 2 different S3 's.

When I tap on the app icon, my splash screen appears but nothing else happens. I looked in DDMS and I get a device log entry stating the Application Launcher timed out, but I can't figure out the root cause. Has anyone seen this issue? Does anyone have any hints how to further trouble shoot this and/or what my be causing it to time out?

I'm using the 3.0.2 GA SDK.
Both the S3 s are running Jelly Bean.
I'm building this on Mac OSX 10.7.5.
I have set the build to use Google APIs Android 4.2.2 [armeabi-v7a].

Thanks…

    • You don't need to do anything, it will work, even with published/signed apps. All you need is to have DDMS running, your phone connected and your device highlighted in the list of devices…

      If you're not seeing any of the info statements, you should probably try and isolate your startup in app.js. For example, not sure how many external files you are referencing or loading, but commenting out all or most external files and then adding them back in one at a time, kind of a pain (because you need functionality from the other modules), but it's difficult to troubleshoot stuff like this.

      Are you using any third party libraries/modules? Could be that one of them is having trouble…

      Best advice I can give is simplify as much as possible, see if it works, then start adding stuff back in. If you want to post your app.js, might be able to help with some ideas… Your call.

    • Thanks. … I was never able to get Ti.API.info output in DDMS from my failing app or my successful test app. In trying many different things I was able to get the install to fail consistently on all 3 of my test devices. I haven't determined the root cause of the issue but I suspect it's something to do with how the app is signed and modules I'm using. Working thru the differences between what happens when I "Deploy to Android" versus "Publish" I noticed there is a difference in the way the app gets signed.

      Deploy to Android:

      jarsigner -sigalg MD5withRSA -digestalg SHA1 -storepass ******* -keystore ...

      Publish:

      jarsigner -sigalg SHA1withRSA -digestalg SHA1 -storepass ********** -keystore

      The Android guidelines found here indicates that the app should be signed using MD5withRSA in their example. I've also found some reference to the Google Maps module and an issue with it loading when being published. I'm not using the Google Maps module but I am using the payment module and would guess that's the one causing the problem (didn't bother removing that completely from my code to confirm this but simply commenting out that module doesn't solve the issue). So for now, I'm manually building my app for Publish because I don't think it's possible to override/configure the defaults. Does anyone know if it's possible to change the jar signing default.
