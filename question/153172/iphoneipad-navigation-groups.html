  • Hi

    To go back from from one window to the previous one in a nav group (same with auto nav group in tabs) using code, you need to use the close method. The information below is based on your use of the window property url.

    Inside AppContacto.js

    var win = Ti.UI.currentWindow;
// then in an event
win.close();

    The thing to take into account is that there is no automatic way to return to the root window.

    • If you are creating platform specific folders then you should be creating them in the folders that already exist for this;

      {appname}/resources/iphone/
{appname}/resources/android/

      You would then NOT include the platform name in the file path as this is collapsed when combined.

      And example using your folders would be as follows;

      {appname}/resources/android/handheld/AppMenu.js
{appname}/resources/iphone/handheld/AppMenu.js
// you would then reference the file as follows;
...
url: '/handheld/AppMenu.js'
...

      I would suggest you normalise the filenames to all lowercase - or be very carefuly that the written name and the actual filename match EXACTLY in case.

      If you are breaking out your files to this sort of level - you should consider moving your project to a CommonJS one, or even Alloy if you are just starting out as it handles much of this for you, but be aware there will be a learning curve.
