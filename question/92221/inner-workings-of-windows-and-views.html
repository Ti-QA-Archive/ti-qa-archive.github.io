Inner workings of windows and views

As far as I understand, in an objective c app using cocoa, apps have 1 window, and then each 'screen' is created using a view as a container. In this light, internally is a window in titanium a view natively, or are they literally multiple windows being created?

I'd like to create an app with multiple screens, but it's not very clear which container is best used for this. Ideally I would imagine using 1 window and multiple views in that window as each screen, but it seems that windows have better support for being animated on and off the viewport.

Anyone have any suggestions? Thanks!

  • If you need one window only ( and not tabs ) I would go with a navigationGroup.
    This allows you to open a "drill down" suite of windows without the need to take care of the animation, hiding the views or memory management.
