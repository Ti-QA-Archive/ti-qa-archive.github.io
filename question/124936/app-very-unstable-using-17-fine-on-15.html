  • Don't know if this has anything to do with what your seeing, but when I upgraded to the 1.7 series the biggest change that I saw that broke my apps was how files are accessed. 

    var file = Titanium.Filesystem.applicationDataDirectory + '/filename';

    for files in your application directory and

    var file = '/filename';

    for accessing files under your Resources directory (note the slash)

  • I also face these random crashes, but I don't know if they are the same as the ones you are seeing.

    The iPhone simulator crash logs are stored in: ~&#x2F;Library&#x2F;Logs&#x2F;DiagnosticReports. The device crash logs can be accessed using the XCode Organizer (I think these are also available in iTunes).

    To get better debug information open the generated project in XCode (it's in build/iphone under your project directory), and run it under XCode until it crashes.

    I've documented my crashes in http://developer.appcelerator.com/question/123736/reproducible-crash-and-errors-with-database-access-in-172 and the Titanium (Community) ticket #220

    You should post a crash log so we can see what is happening.
