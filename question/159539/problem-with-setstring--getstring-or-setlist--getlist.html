Problem with setString / getString or setList / getList

So i use "Ti.App.Properties.setString /getString " or Ti.App.Properties.setList / getList" that is the question ?

var row = Ti.UI.createTableViewRow( {title:'Add name product '});

  • Saving app state is always a cumbersome problem.

    JSON.stringify would generate a string so one would probably use Ti.App.Properties.setString.

    Ti.App.Properties.setList is for saving arrays.

  • Your example is using a list (array) to store a string.

    I have a feeling that you can't store the 'rows' like that in Properties, as they are effectively UI elements.

    Try using the rowData of each row and Properties.set/getObject without the stringify.

    Does your example work when you use set/getString? It should return data, but not actual 'rows', even after parsing the JSON string.
