  • What version of the Android tools do you have installed? We've had some reports that using Android tools R23 causes errors like you're seeing, but downgrading to the R22.3 tools resolves the issue. This ticket has been resolved, which could be the cause of this problem.

    Open the android app (part of the Android SDK) and see what it reports on the "Android SDK Tools" line. If it says R23, you have three choices: 1) downgrade your Android tools (a manual and not very easy process), 2) use the nightly build of the SDK which includes the fix for TIMOB-17286, or 3) wait for our next SDK release which will officially contain this fix.

    To use the nightly build: visit http://builds.appcelerator.com/ then next to "install" under the Windows heading, copy the ti sdk install command. Paste that into your terminal window and press Enter. Update your tiapp.xml to use the resulting SDK version.
