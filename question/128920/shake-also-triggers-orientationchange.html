  • I couldn't find a real solution for this, but found a small workaround.

    You can validate the orientation value of the event to catch the false event:

    Just add:
    if ( e.orientation == 0 || e.orientation == 5 ) { return false; }

    All shake events have an orientation value of 0 or 5, while the real orientation values are 1-4.
