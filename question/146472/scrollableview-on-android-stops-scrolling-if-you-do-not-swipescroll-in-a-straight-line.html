ScrollableView on Android stops scrolling if you do not swipe/scroll in a straight line

Hi Everyone,

On Android there seems to be an issue with the ScrollableView if you do not swipe/scroll in a perfect horizontal line.

Using the sample code from the latest documentation, run the code on iPhone, you will notice that the scrollableview will continue to scroll even if you are not swiping/scrolling in horizontal line. If you do the same on Android the scrollableview stops scrolling and the view snaps back

Steps to Reproduce

1. Copy the code to create the ScrollableView from the latest documentation
2. iPhone - Swipe/scroll in horizontal line and not horizontal line -  the scrollableview will continue to scroll.
3. Android - Swipe/scroll in horizontal line and not horizontal line -  the scrollableview will only work if you scroll in a horizontal line. If you scroll in non-horizontal line (slight angle) the scrollableview will stop scrolling the view will snap back.

This is not usable on Android because if you are swiping with your thumb this will cause issues because is almost impossible to swipe in a straight line. This is even tough to do with your pointer finger.

A good example of being able to swipe/scroll in a non-horizontal line would be the Twitter App. On the "ME" tab you can swipe/scroll between your profile pic and details even at various angles the scroll works perfect

Application type: mobile
Titanium SDK Version: 3.0.0
Titanium Studio, build: 3.0.1.201211202043

Sample Code from Documentation

var win = Ti.UI.createWindow();

var view1 = Ti.UI.createView({ backgroundColor:'#123' });
var view2 = Ti.UI.createView({ backgroundColor:'#246' });
var view3 = Ti.UI.createView({ backgroundColor:'#48b' });

var scrollableView = Ti.UI.createScrollableView({
  views:[view1,view2,view3],
  showPagingControl:true
});

win.add(scrollableView);
win.open();

To me this seems like a bug on Android.

Thanks for any feedback or advice on how to handle this.

Sean

  • Hi

    Having tried your code on Android I cannot see the same result you are experiencing.

    • Swipe horizontally - works
    • Swipe down around 15 degrees from edge - works
    • Swipe down around 30 degrees from edge - works
    • Swipe down around 40 degrees from edge - works
    • Swipe down around 45+ degrees from edge - fails (to be expected - as no longer a a distinct action)
    • Swipe horizontally from 40% in from either edge - works
    • Swipe down from 40% in from either edge and around 15 degrees - works
    • Swipe down from 40% in from either edge and around 30 degrees - works
    • Swipe down from 40% in from either edge and around 40 degrees - works
    • Swipe down from 40% in from either edge and around 45+ degrees - fails (same reason as before)

    Repeat all the downward direction test above in an upwards direction instead - all results matched those of the downward direct.

    You did not specify an Android version for your test scenario, I have used 4.1.2, let me know what you used if different and I will check that.

    Have you tried on both the emulator/simulator AND physical devices?

      Your summary above suggests that you start on the first view; if you start from the top left and drag/scroll down diagonally towards the lower right then you expect to end up on the second view.

      However if you are on the first view and start from left to right then the current view would move right and IF there was a view to the left of the current one then that would be shown. However as we started on the first view there is no view prior to the current one.

      So if your scenario is correct - it will NEVER work.
