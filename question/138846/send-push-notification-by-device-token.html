Send Push notification by device token

Hi,

I checked the API document and found Push Notifications: notify for sending notification.
If I am not wrong we can send notification to the particular channel. This way we can send notification to all the users within the channel. We can also pass "to_ids" to send notification to specific users within the channel.

However, I didn't see any relation between the Create user and Subscribe web services. Means while creating the user we cannot assign user to channel. And while Subscribe we do not have user id.

Query -

  1. How this is managed in ACS?
  2. Is it possible to send different notification to individual user within the channel?
  3. Is it possible to send notification by device token instead of Channel and to_ids.

Let me know if I am not clear…

    • Hi Greg,

      There is no way to send notification by device token. So the way we implement it was by creating user in ACS before subscribing. The User -> Create API will return you ACS UserId. We used this ID to send notification.

      Please let me know if you need any further information.

    • HI Greg,

      The ACS User Id cannot be same. This is a auto generated filed. When you create a user using "https://api.cloud.appcelerator.com/v1/users/create.json" API in the response you get the value of "id" for this user. You can use this id to send notification.
