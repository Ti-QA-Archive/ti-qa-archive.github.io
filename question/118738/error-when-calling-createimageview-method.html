    Leray

    There isn't anything wrong with the code you have included in your question. Are you certain you have not misnamed the createImageView function, although, it is indeed correct here? Note that methods are case-sensitive, so a spelling such as createImageview (note the v of view) would throw an exception.

    I really doubt this suggestion will help, as the exception message doesn't seem related, but some variable names are reserved, and so will cause an exception. Try changing view to view1 or imageView to ensure it does not clash.

    Copy and paste the following into an empty app.js file, and run it. I've tested it, and it works on my system:

    Titanium.UI.setBackgroundColor('#000');

var win = Ti.UI.createWindow();

var view = Ti.UI.createImageView({
//    image:"img_test.png",
    backgroundColor:"red",
    width:40,
    height:40
    });

win.add(view);
win.open();

    Lastly, it's always worth clearing your build files and forcing a full rebuild when anything unexpected is happening. Hence, for iOS/iphone, delete the contents of the project's build&#x2F;iphone directory, select Edit Project in Titanium Developer and click the Save Changes button to force a full rebuild. Then launch the app.

    Hope this helps
