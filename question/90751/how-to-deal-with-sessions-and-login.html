  • Don't know if this will help or not, but in my (desktop) app, I did something very similar to what the tutorial above outlines, but instead of storing the username and password in the database (storing passwords locally is a sensitive issue here), I store the username in the database, and a “session_id” field.

    If the session_id is null, then my app displays a login dialog, and the username and password are submitted over https to a script on the server that, pending the correct password, responds with a session_id (on the server, the scripts handle session_id handling, timeouts, etc. - if you’re using PHP for example, then its standard session handling methods will suffice).

    I then store that session_id locally, and use that for any http connections to my server from then on (either passed in as a url variable, e.g., url = 'https://foo.com/app.json?sid=' + user.session_id + '&bar=baz', or set as a cookie in a HTTPClient object, like this: httpClient.setRequestHeader('Cookie', 'sid=' + user.session_id);, depending on what I’m doing).

    If the session_id is not blank in the local database, then on launch my app attempts to authenticate with the server using that first. If the server responds with a “Session expired” error code, then my app displays the login dialog as above, and resets the session_id field in the local database on successful login.

    I hope that helps a little!
