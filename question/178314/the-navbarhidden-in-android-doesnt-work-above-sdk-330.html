The navBarHidden in android doesn't work above SDK 3.3.0

After I upgrade to SDK 3.3.0 or 3.4.0, it's seems the navBarHidden cannot work on android, even I set this in the tiapp.xml and also failed:

<fullscreen>true</fullscreen>

<navbar-hidden>true</navbar-hidden>

The android status bar will always shows, but if I use SDK 3.2.0, and it's fine, is this a bug in the Titanium SDK?

    • Hi, to clear up the answer, you can hide the nav bar/ action bar in your Java script code or using a custom theme.

      Method 1. Add Custom Theme:

      Add a custom theme file to the project:

      [Project]/platform/android/res/values/custom_theme.xml

      <?xml version="1.0" encoding="utf-8"?>
<resources>
    <style name="Theme.NoActionBar" parent="@style/Theme.Titanium">
        <!-- Depending on the parent theme, this may be called android:windowActionBar instead of windowActionBar -->
        <item name="windowActionBar">false</item>
    </style>
</resources>

      Then, modify the tiapp.xml file to use custom theme:

      <android xmlns:android="http://schemas.android.com/apk/res/android">
    <manifest>
        <application android:theme="@style/Theme.NoActionBar"/>
        <!-- Need to specify at least API level 11 for Titanium SDK 3.2.x and prior -->
        <uses-sdk android:minSdkVersion="14" android:targetSdkVersion="19"/>
    </manifest>
</android>

      Method 2. Hide NavBar In JavaScript Code:

      win.addEventListener('open', function() {
        win.activity.actionBar.hide();
});

      N.B: navBarHidden property is deprecated from the release of The SDK 3.3.0. So to hide nav bar in 3.3.0 and 3.4.0, there is no use of navBarHidden property.

  • Taken from AppC Docs: Since Release 3.3.0, due to changes to support the appcompat library, this property has no effect. By default, the action bar is always displayed. To hide the action bar, see the Android Action Bar guide.

    Link to doc

    I fixed it just adding this in my Tiapp.xml

     <android xmlns:android="http://schemas.android.com/apk/res/android">
        <manifest>
            <application android:theme="@style/Theme.Titanium">
            <!-- Add whatever else you need here -->
        </application>
        </manifest>
</android>

    "@style/Theme.Titanium" creates a theme build/android/res/values where the actionBar is hidden by default.

    <style name="Theme.Titanium" parent="@style/Theme.AppCompat">
        <item name="android:windowBackground">@drawable/background</item>
        <item name="android:windowActionBar">false</item>        
        <item name="android:windowNoTitle">true</item>
        <!-- AppCompat Compatibility -->
        <item name="windowActionBar">false</item>
</style>

    This worked for me. Hope this helps for you too.
