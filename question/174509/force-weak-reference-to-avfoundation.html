Force weak reference to AVFoundation

Ti SDK 3.2.3GA
iOS SDK 7.1
BenCoding.Utterance 0.2
Mac OSX 10.9.3

I'm trying to use the BenCoding Utterance module and enable the App to still run on iOS 6.x

The problem I have is that when you deploy on iOS 6 devices the App crashes. XCode indicates that a required method in AVFoundation.framework that is used by Utterance is not found. I've found that if I manually amend the generated XCode project and set a weak reference to AVFoundation (ie make it optional) then I can build fine.

I would rather not have to perform this step manually each time I wish to build. Does anyone have any idea how I can force the build process to include AVFoundation.framework as optional (weak referenced) ?

Thanks

    Each module has a config file, module.xcconfig, that specifies libraries and frameworks to link.

    You could try setting the AVFoundation to a -weak_framework instead of a -framework
