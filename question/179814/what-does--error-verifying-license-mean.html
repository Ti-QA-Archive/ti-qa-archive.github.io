  • Hi Devanjali Singh,

    Ti.Verify makes a call to a backend server to verify the licenses. If the connection to that server is not available or the backend server is having an issue, then you got the error.

    The error may state that a license violation has been detected. If you encounter this error you need to navigate to the Appcelerator Product Market and ensure you have downloaded the modules (i.e. you are subscribed to them).

    Possible Workaround:

    Instead of installing the module in the Titanium folder, install the module locally to your mobile project.

    Mobile Project Folder/modules/iphone/com.modulename.xxx

    Then inside the module folder you will find a 'manifest' file. Open this in a text editor and change the 'guid' string to be the SAME guid as you find in Tiapp.xml (near the top).

    In Titanium studio, go to Project > Clean and clean your project (emptying the /Build folder) and re-build. If it's going to work and this is a free trial with no issues then it should now work.

    Another trick to use for a long long time is to actually change the guid of my app in tiapp.xml and then also do the same for my modules. Clean & Rebuild you should be good to go. Failing that you will need to speak to the module developer for assistance.

    Hope it helps.
