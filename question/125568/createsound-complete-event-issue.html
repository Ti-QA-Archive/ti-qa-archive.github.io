createSound complete event Issue

Hi,
I am working on an app for iPhone on Titanium Studio ,SDK version 1.7.2.
I am using the createSound() control to play a recorded file.
While the file is playing, a timer is displayed indicating the file's progress and then once the file's duration is ended I stop the timer in the 'complete' event of createSound().However, the complete event of createSound() is not firing at all for some reason.

Is this an issue with appcelerator?
Any help is appreciated.

thanks

    I found the solution so thought should put it here.

    The problem was I was doing sound.release() because of which complete event wasnt firing.However, sound.release() is important to prevent caching else no matter how many times you change url of sound it plays the first file only.
    The solution is after sound.release() you do sound=null and then again create the sound object as sound = Ti.Media.createSound({

    });

  • This code performs as expected, utilizing the Sound complete event on iPhone sim 4.3 using mobile SDK 1.7.2 in Titanium Studio 1.0.4.

    Link to beep-1.mp3 for your testing

    var win = Ti.UI.createWindow();

var button = Ti.UI.createButton({
    title:'play sound',
    width:100,
    height:50,
    top: 60
});
var sound = Ti.Media.createSound({
    url: 'beep-1.mp3'
});
sound.addEventListener('complete', function(e) {
    alert('completed');
});
button.addEventListener('click', function(e) {
    sound.play();
});

win.add(button);
win.open();
