In-app purchases & credits

Hi all!

I'd appreciate help/feedback regarding the following scenario:

When a customer purchases a license of my app I want to include say 3 credits. The credits are personal and allows the app to call a method on my server 3 times, the method returns data. When the credits are up, I want the customer to be able to purchase new credits without leaving the app, preferably by using the already existing ways to pay already in AppStore/AndroidMarket.

This is how I'd ultimately would like it to work:

  • Customer buys the App in AppStore/AndroidMarket. The IMEI is communicated to my server and 3 credits are associated with it.
  • Customer uses the app, and for each call to the server the IMEI is communicated and credits are decremented.
  • When credits are empty the customer is presented with a "Buy more credits" dialog and completes the purchase via existing AppStore/AndroidMarket infrastructure. The IMEI and purchase are communicated to my server which fills credits.

.

Is this possible? Is there another/better way to achieve this?

Thanks :)
