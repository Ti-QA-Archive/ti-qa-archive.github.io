  • Jaime,

    Since you have manualMode set to true then you need to add a location provider for your configuration to work: This is the relevant section of the documentation:

    "Manual configuration involves managing providers and rules:

    Location providers, such as GPS, provide location updates.
    Location rules filter the results returned by location providers.
    Configuring geolocation manually involves three steps:

    Enabling manual mode.
    Enabling location providers.
    Adding location rules (optional)."

    You should check and see if you require manual mode or not for your particular project. The full documentation on geoLocation in Android is here.

    • Jaime,

      I think you might be confused as to what location services does/is doing. You are checking to see if location services is enabled (good), but if you want to get the current lat/lon, then you MUST get them from somewhere. If you are using Android and try getCurrentPosition then it WILL NOT WORK. You can see from the documentation, that getCurrentPosition returns a cached location. On Android, the radios are not turned on to update the location, and a cached location is used.

      So, if you want the current lat/lon, you MUST use the location event handler. There is also a problem with the event handling with location where you MUST throw away the first result it returns and take the second one (on Android).

      So your code would look something like this:

                  var ignore = true;
            Ti.Geolocation.addEventListener('location', function(e) {
                if (OS_ANDROID) {
                    if (ignore == true) {
                        ignore = false;
                        return;
                    }
                }

                if (!e.success || e.error) {

                    //alert("error!");

                    e.coords = {};
                    e.coords.latitude = 0;
                    e.coords.longitude = 0;
                    indicator.close();
                    return;
                }


                Ti.Geolocation.removeEventListener('location', arguments.callee);
                callback(e);
            });

      Ray
