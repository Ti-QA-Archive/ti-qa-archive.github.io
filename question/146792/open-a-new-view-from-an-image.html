    Quick thought; You have used the image reference /android/helpdesk.jpg when not using Alloy you would not include the /android part as this is a virtual path that root resolves itself. You should be able to simply have /helpdesk.jpg.

    As I say I have not checked this with Alloy, but without it and certainly on iOS the virtual folders are there for platform specific files but you would not include that prefix in the folder path.

    For example, if you had an app for both iOS and Android and creted images specific for each you would add them like this for example;

    /android/example1.png
/iphone/example1.png

    But you would only reference them using;

    /example1.png

    As the virtual folders collapse when on the device.

    Hi

    Change this bit…

    var more = function(){
    var win = $.mo.getView('win2');
    win.open();
}

    …to this;

    function more() {
    var win = $.mo.getView('win2'); // assumes this bit works
    win.open();
}

    This create a function rather than a variable that contains a function, this should respond to the onclick part of this;

    <ImageView id='img05' icon='/android/helpdesk.jpg' bottom='10%' right='10%' onClick='more'/>
