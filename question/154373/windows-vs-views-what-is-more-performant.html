Windows vs Views, what is more performant?

Is more performant to create "n" windows or windows with "n" view that overlap?

For example, in a list of news that the tap opens the detail with the text of the news, you should create two separate windows or windows with a single view containing the list of news and another view that contains the details of the news?

  • Hi

    If you are creating an app that has a list of things in a table and then the user selects one of those things and a whole screen full of information appears, I would create a second window to handle the extra data and show it.

    Basically a master > detail approach.

    You have not specified if you are target Android or iOS, but each has its own way of handling the user wanting to move backwards from one window to the previous one - this is normal with windows but not with views.
