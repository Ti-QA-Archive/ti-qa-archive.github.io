    Please, always state the mobile SDK version and the Titanium SDK version you are using. The question tags are the best place for this information. Without this, it's difficult to answer your question.

    Are you debugging using an emulator or a physical device?

    You would be better off using logcat rather than ddms to obtain debug info. Thus, when your app is running in an emulator, type the following at the local terminal:

    adb -e logcat

    In the Debugging section of our Third-party tools guide, there is a script listed that I use called "Colored Logcat" that I would heartily recommend.

    Once you have provided all the relevant details, I will take another look.

    • Hi David

      Unfortunately, the Android emulator 2.3.X is quite broken with regards to webviews. This is explained in the note below the table at Android SDK / Target Android Platform. Note that this fault cannot be attributed to Titanium itself. The solution is to run your app on an Android APIs 2.2 or below emulator. Fortunately, physical Android devices do not suffer from the same issue.

      What was the error you received when you ran the adb command? It may just be because your system does not know where the adb executable is. If this is the case, you will need to add it's directory to your system path.

      Hope this helps
