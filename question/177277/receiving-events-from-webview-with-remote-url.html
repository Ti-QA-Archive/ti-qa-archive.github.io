Receiving events from webview with remote url

So, it is not possible to call Ti.App code from pages loaded with a remote url in the webview. We're okay with that to help with security. But we need a way to work around it.

As an example we may have a button in the remote html page that should trigger a certain action in the app. Since we cannot do this directly from javascript in the html we have tried this approach:

In the html javascript we added an array of events to trigger. When the user presses something, the html javascript will push something into this array.

In the app, we added a setInterval for 50 ms that will call the webview with evalJS to fetch an event from the array in the html. Based on the event we then trigger certain actions.

This will make the window holding the webview poll with evalJS with 50ms interval… will this cause any horrible battery or other issues for the app?

AND is there a better way to do this?
