Do I need semaphores for server?

Hi,
I have a server setup with sockets where I am listing for incoming connections and function calls. I am wondering if a second function call can appear while I am processing another function call. If yes then I would need semaphores to assure that the data I am manipulating does not get broken.

In the example below, if a client calls function1 and it sets gameData=1 and then another client calls function2, will function2 set gameData=3 immediately or will gameData=2 happen always before that so I do not need to handle semaphores myself?

Alex

var app = require('http');
var server = app.createServer();
var sio = require('socket.io');
var io = sio.listen(server);

var gameData;

io.sockets.on('connection', function (socket) {          
             socket.on('function1', function (data) {
                gameData=1;
                gameData=2;
             });
             socket.on('function2', function (data) {
                gameData=3;
             });
    };
