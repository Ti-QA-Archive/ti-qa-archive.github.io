Facebook iOS publish_actions and Submission/Review Process

Mobile app with Titanium SDK 3.2.3.GA for iOS 7 (working on a Mac OS)

Hello guys I really don't know more what to do and need your help.
Actually, to create an application that the user can upload and share some image at Facebook, the app must be previously approved by Facebook that will grant access to the "publish_actions" permission. Without this permission the users will not see the "This app can publish at your behalf" screen.
Ok great, the Facebook had implemented a cool review system and everything works.

Here's what I'm facing: submit my simulator version of the app. The Facebook guys had created a tutorial explaining how to build and send an iOS Simulator version of the app. Here's the link:

https://developers.facebook.com/docs/ios/creating-ios-simulator-build-for-review/

Ok, great, I followed and achieved it twice, however at the two submissions I received an answer from the tester telling that they can run my app on Simulator but it crashes and closes at the start.

I'm almost certain that this is a little issue related with Titanium build, and haven't any idea how to fix it.

To achieved the version, I built my app and ran it on Simulator. Then, I went to build folder (created by Titanium) -> iPhone and found the "app.xcodeproj" file. This is the file that I'm using to create the Simulator Release that will be send to Facebook. Am I doing it right?

I'm also appreciate if somebody can explain to me how Facebook Testers runs an app at the iOS Simulator, so this way I can test my own app before send it for them and certify that the app is really crashing.

I can't find anything on web about this (recent) issue and appreciate any suggestion or solution =]

Thank you

    I'm also appreciate if somebody can explain to me how Facebook Testers runs an app at the iOS Simulator, so this way I can test my own app before send it for them and certify that the app is really crashing.

    um…

    Before uploading, test your simulator build by running ios-sim launch <path to .app> and verify that the app opens in the simulator.
