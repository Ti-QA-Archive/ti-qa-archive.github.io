internationalization with alloy projects

Hello!

I wish to internationalize my Alloy project, so - i've created the following folders: ProjectRoot/i18n/en
inside that i made a string.xml file:
<?xml version="1.0" encoding="UTF-8"?>
<resources>
<string name="test">Test</string>
</resources>

and I tried to use it as follows:
alert(L('test'));

and it didn't work.

I've scaned the Web for answers, and found many related topics but none of them helped me, such as:

  1. clean your project - I pressed project->Clean…
    my alloy projects are not exists at the project list, so i pressed the "Clean all projects"

  2. the i18n works only in the tabbed project because the manifest is configured that way - that's correct the i18n works when I use tabbed project but there is no major diff between the manifest or tiapp.xml.

  • Hi

    Internationalisation is used and configured identically in both Alloy and non-alloy projects.

    The problem you appear to have is that you have spelt the name of the file incorrectly.

    {appname}/i18n/en/string.xml
// should be 
{appname}/i18n/en/strings.xml // note the "strings.xml" not "string.xml"

    The strings file was correct;

    <?xml version="1.0" encoding="UTF-8"?>
<resources>
  <string name="test">Test</string>
</resources>

    As was your example;

    alert(L('test'));
