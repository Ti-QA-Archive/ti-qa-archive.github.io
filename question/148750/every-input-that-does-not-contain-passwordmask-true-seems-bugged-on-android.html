Every input that does not contain passwordMask: true seems bugged on Android

Hi,

I have a regular login form with an email and password input type. Everything works as expected on iOS, but in the Android simulator the email input does not fire any event handlers and displays a different font than the field with passwordMask set to true. I have tested it with setting the property passwordMask for the input field to true and then everything works as expected. I have also tried to remove all event listeners, and removing / adding several other properties but only passwordMask works.

When focussing on the email field the info log shows:
[WARN][LatinIME( 165)] Unexpected input class: inputType=0x00080020 imeOptions=0x08000005

Operating system: OSX Mountain Lion
Titanium SDK version: 3.0.2
Android tool-api-level: 16
Android API: Google APIs Android 4.0.3
Android screen: HVGA
navController

This is the code for both textfields:

        title: 'Login',
        layout: 'vertical',
        navBarHidden: true,
        exitOnClose: true,
        fullscreen: true
    });

    // Email input field
    Ti.App.ui.loginField = Ti.UI.createTextField({
        id: 'emailfield',
        top: 28,
        width: 220,
        height: 35,
        hintText: 'Email address',
        paddingLeft: 8,

        font: { fontSize: 13 }, // fontFamily: 'monospace'
        returnKeyType: Ti.UI.RETURNKEY_NEXT,
        keyboardType: Ti.UI.KEYBOARD_EMAIL,
        clearButtonMode: Ti.UI.INPUT_BUTTONMODE_ONFOCUS,
        autocapitalization: Ti.UI.TEXT_AUTOCAPITALIZATION_NONE
    });
    win.add(Ti.App.ui.loginField);

    // Password input field
    Ti.App.ui.passwordField = Ti.UI.createTextField({
        id: 'passwordfield',
        top: 14,
        width: 220,
        height: 34,
        hintText: 'Password',

        paddingRight: 8,

        suppressReturn: false,
