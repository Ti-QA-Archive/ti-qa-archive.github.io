      For android, only delete the contents of the immediate subdirectories of &#x2F;build&#x2F;android, otherwise you may encounter problems. Then click the Save Changes button to force a full rebuild and launch the app, as above.

      If you have recently added or removed android packages, close your emulator, delete any existing AVDs, and launch your app again.

    • You seem to have two versions of java installed. I would remove java altogether, and reinstall jdk (which will also install the correct version of java).

      You have duplications in your path, and the order is not optimal. You are missing a semi-colon after C:\Programmi\TortoiseHg\ which may prevent it from working properly. Lastly, D:\Downloads\DevelopMobile\android-sdk\platforms\android-8\tools; is not required. Hence, change it to the following:

      C:\WINDOWS\system32;
C:\WINDOWS;
C:\WINDOWS\System32\Wbem;
C:\Programmi\Java\jdk1.6.0_21\bin;
D:\Downloads\DevelopMobile\android-sdk\tools;
D:\Downloads\DevelopMobile\android-sdk\platform-tools;
C:\Programmi\TortoiseHg\

      Note: make sure the jdk path is correct (with the right version number) in the above path.

      Please configure your ANDROID_SDK variable, so that android will run when you next run the diagnostics.

      Note that python, git and scons are only required when you need to compile the source code. However, I would recommend you install git, so that you can easily keep your copy of the KitchenSink up to date.

      Read Installing on Windows 7 if you need any guidance with any of this.
