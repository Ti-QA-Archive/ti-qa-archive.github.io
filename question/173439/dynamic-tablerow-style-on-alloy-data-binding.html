Dynamic TableRow Style on Alloy Data Binding

The scenario:
I have a table view that is connected to a collection through data binding. How do I make the first row of the table view to have a height that's different than the rest of the rows.

This is the set up I have currently.

news.xml

<Alloy>
    <Collection src="News" />
    <TableView dataCollection="News" dataTransform="transformData">
        <TableViewRow model="{id}" height="Ti.UI.SIZE">
            <View height="{rowHeight}">
                <Label text="{title}" />
            </View>
        </TableViewRow>
    </TableView>
</Alloy>

news.js

var News = Alloy.Collections.News; 
var rowIndex = 0;
News.fetch();

function transformData(model) {
    var transform = model.toJSON();

    if (rowIndex == 0) {
        console.log('this is the first model within the collection');
        transform.rowHeight = 230;
    } else {
        transform.rowHeight = 63;
    };

    rowIndex++; 

    return transform;
};

With the set up above, I was able to see the string "this is the first model within the collection" on the console 1 time, which signaling that the counter works.

When I run the app however, I saw for a split second that the first row's height was bigger but it changes immediately to be the same with the rest of the rows. It seems that there's another process that overwrites my custom height for the very first row of the table.

Anyone how any thought on how to achieve this?
