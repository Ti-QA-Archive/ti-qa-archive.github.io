Ti.include fails in Android emulator

I'm seeing calls to Ti.include have no apparent effect.

For, e.g., a file foo.js whose content is simply Ti.API.debug(&quot;in foo.js&quot;), the print out never happens.

I'm seeing this in the Android emulator, using the APIs 2.3 SDK and Ti Developer v.1.2.2. It is not consistent - a few files usually can be included before it starts to fail.

It does not appear to be specific to an individual file - I've tried commenting out the contents of files, and commenting out includes of previous files, etc. No permutation seems to work.

Has anyone else encountered this? Is there a solution?

edit: As requested, a simple example.

In session.js

Ti.API.debug("session.js: at top of file");

In app.js

Ti.API.debug("before");
Ti.include("session.js");
Ti.API.debug("after");

before and after print. at top of file does not.

  • I'm seeing the same behavior with APIs 2.3.3, TiDev1.2.2 & MobileSDK 1.6.2.

    This seemed to be the relevant portion of the trace:
    http://pastie.org/1882899

    I used the solution provided in this post: http://developer.appcelerator.com/question/116828/too-deep-recursion-while-parsing-when-including-files-on-android

    … which instructs to add the directive:

    <property name="ti.android.threadstacksize" type="int">32768</property>

    to tiapp.xml (posted again for posterity )

    Worked for me. Hooray!
