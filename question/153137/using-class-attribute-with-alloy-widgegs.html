    The <Widget> is not technically a UI element, it is a container for UI elements, just like a controller. For this reason, no, adding a class to the <Widget> element will have no direct effect on the widget's UI. You need to style the elements within the widget with TSS files contained in the widget.

    • Though you could make your widget use the passed in properties to style itself. Setting the background style in your app.tss on the class myclass will actually pass that style into the 'arguments' of your widget. You would just have to pull it out of the arguments and then set the background color programmatically.

      We use this pattern to style all our widgets at the app.tss level. We only include structural styles in the widget itself - this way it's a true reusable widget.
