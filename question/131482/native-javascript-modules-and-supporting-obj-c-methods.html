Native javascript modules and supporting Obj-C methods

Is it possible to write a javascript module - but also have methods in Obj-C?

For example, I'd like to update http://bytespider.github.com/jsOAuth/ to use more of the titanium framework that I currently have it doing. Specifically I'd like to write a HMAC_SHA1 method in Obj-C but be able to call that method from my existing javascript code.

Whilst I realise that I could:

  1. rewrite jsOAuth exclusively in Obj-C
  2. leave it alone as it works

I feel that there are advantages to this approach.
Firstly there are already OAuth libraries in Obj-C, I don't want to reinvent the wheel.
Secondly, I think there would be a performance boost by having the native crypto libraries in iOS to the hashing work.

What do you think?

  • You can create your own native iOS module with the methods you need and call them from a javaScript module. This is possible, if this is what are you asking. But note that you can't mix both languages, you still need a native module done with Obj-C and compiled using the titanium build.py for modules and require it from titanium.
