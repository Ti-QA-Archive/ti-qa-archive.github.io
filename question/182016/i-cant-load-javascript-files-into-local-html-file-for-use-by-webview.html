I can't load javascript files into local html file for use by webview

I am trying to use the <script> tag to load rapheal-min.lib (note I did read the docs
and have changed the extension) into my local html file like this:

local.html

<!doctype html>
<html>
<head>
    <title>local html</title>
    <script src="/raphael-min.lib" type="text/javascript"></script>
</head>
<body>
    <div id="chartDiv"> </div>
    <script>r = new Raphael("chartDiv")</script>
</body>
</html>

This runs fine in the web preview mode, but when I run it on the emulator I get the error

"Uncaught ReferenceError: Raphael is not defined"

I just want to draw a simple graph and I'm sure I can manage that without raphael, but this
is a serious long term issue, in that if I can't use third party libs in my local html files using
web view, then I am left with a very limited product, especially as Raphael has not built in
drawing library. Is their a fix to this?

  • Having read all through these forums I have noticed many people have had the same problem
    and found no resolution. So here is the solution I found. For some reason the android packager
    does not package these files onto the main path as you would expect so the file is apparently
    not at /raphael-min.lib. When I packaged it in the same directory as my html file and used a
    relative path i.e. <script src="raphael-min.lib"> it was able to find it. Go figure.
