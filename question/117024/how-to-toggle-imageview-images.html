How to Toggle ImageView Images

What's the best way to toggle an imageView between two images (urls)?

On Android & SDK 1.6:

  1. Updating the imageView.url property - this doesn't seem to take any affect.

  2. Remove the imageView from parent view and adding a new imageview with the second image in it's place. Doing so numerous times, however, results in some kind of android exception.

I haven't tried animation, so if that's the proper way, I'd appreciate if someone could post some sample code. Logic would be: ImageView defaults to first image. Some event causes the ImageView to toggle. Some other event causes the ImageView to toggle back.

  • I actually got this to work on the DROID. (Worked fine on iOS).

    I had to put full relative path when the images were in the same folder. If I used "ui/image1.png" and "ui/image2.png", 1 would show, but swapping to 2 would not show. When I changed to include Titanium.Filesystem.resourcesDirectory+<path/image> then I can swap back and forth on DROID.

    Also, you cannot do image1.add(image) on DROID… I am making a simple TIC-TAC-TOE on top of a board.png just as a proof of concept for a more complex multi-layer image product.

    Hope that helps you guys!
