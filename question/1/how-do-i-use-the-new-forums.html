How do I use the new forums?

Great question. Glad you asked.

Welcome to the first Q&A for the new community forums. This post is designed to help show you how to create your own first post.

First things first - the title

Set the title. Make it descriptive. But not too long. Your titles the first impression others will have.

Got Code?

If you have code you need reviewed, please be considerate of crappy looking web pages when you paste your codes. Also, make sure you use the special wiki-formatting (called Markdown) when you paste your code. It's really super easy. No excuses for bad pastes. Here's how. Just start and end your code with the ~ mark - three times. Look over to the right on the sidebar (near the bottom) and you'll see an example.

Got lots of Codes?

If you got more than a few lines of code or log output - please, for the love of God - please don't paste it here. Please use Gist or Pastie for your code snippets. It will be so much easier to read, fork and comment.

Tags

Please tag your posts – and separate them with commas. For example:

iphone, animation

Your tags are also a way that others can find your post and also a better way of classifying content. If you post good tags, the community will love you for it.

Vote!

Not quite American Idol, but your vote still counts - especially around here. If you vote up (or down if it sucks), it will help you get more attention and more attention means better answers.

Lend a hand

We give and take. More takers than givers. Do your share if you can and try and help out. The earth will love you for it and we will too.

Close up shop

After you ask a question and get answers, please mark the best answer - assuming you have one. Once you mark an answer at the best, you'll both get Appcelerator reward points and it will help others find quality content.

Be courteous

Nobody likes a prick. Take a deep breath. Ask your question clearly and post a reduced test case, if possible. If you're answering a dumb question, consider that maybe it's a n00b question and that's OK - we all do it from time to time. You gotta start somewhere.

— asked March 8th 2010 by Jeff Haynie

  • we have pushed some fixes this morning to improve the ordering of answers.

    the ordering is now in the following:

    • best answer is always first
    • highest votes are next
    • chronological (newest first) are third

    The goal here is to make sure that if the answer has been marked "best" by the person who asked the question - that that answer will always been on top and be highlighted. The second goal is to make sure that even if a best answer hasn't yet been chosen, that the ones with the highest community votes will rise to the top. In the case there are no votes, the latest answers will then go up.

    We'd like to encourage community voting as a way to encourage better answers between peers and rankings.

    The model we're following is very similar to Stackoverflow which works quite well. Since this is a new capability, it will take a little bit of time to work through some of the issues and get the system trained a bit and the correct behaviors working.

    So why do this on our own? For the past 3 years, we've had now about 4 different forum systems. They all have had major issues for us mainly and none of them allowed us to do what we really want to do here. So, we tried to make it work with off the shelf and we haven't. We're now going to try and make it work ourselves so we can customize things the way we need to. Also, we're planning on a bunch more integrated capabilities here soon and we needed the ability to control how things workflowed for that reason.

    Please bear with us…
