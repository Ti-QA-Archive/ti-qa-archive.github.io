  • Hello,

    Please note this issue has now been fixed in the 1.9.x CI builds available here http://builds.appcelerator.com.s3.amazonaws.com/index.html

    Make sure when downloading you select the master branch.

    Other things to keep in mind.

    1. If you where using app.jss as a bootstrap you will now need to use global.jss instead. Just copy the contents from app.jss into a file call global.jss

    2. If you have file paths in your JSS such as imagines or other assets. You will need to test them as their relative path has changed. I just avoided this issue all together an use app://your/file/path/goes/here
