iCloud Backup

Hello All,

I am currently trying to get my app to be published onto the iOS App Store but keep coming across iCloud Backup issues. I am currently working with the iOS 6.1 SDK and I am testing with an iPhone 5.
The message I get from Apple App Store is the following:

We found that your app does not follow the iOS Data Storage Guidelines, which is required per the App Store Review Guidelines.

The iOS Data Storage Guidelines indicate that only content that the user creates using your app, e.g., documents, new files, edits, etc., should be backed up by iCloud.

Temporary files used by your app should only be stored in the /tmp directory; please remember to delete the files stored in this location when the user exits the app.

I have browsed the forum to look for what this issue could be and found an article which explains that you need to use a remoteBackup = false; attribute. Here is what I currently put in my app to try and stop the iCloud backup:

var sPath1 = Titanium.Filesystem.resourcesDirectory+Titanium.Filesystem.separator+'packaged_data'+Titanium.Filesystem.separator;

sPath1.remoteBackup = false;

var sPath2     = Titanium.Filesystem.applicationDataDirectory+Titanium.Filesystem.separator;

sPath2.remoteBackup = false;

However this does not seem to change what is backed up to iCloud.

Any help and assistance on this would be greatly appreciated.

Thanks

    • i think you are calling the function in a right way, you don't have to put a separator at the end…

      just try this when you don't need iCloud Backup at all (copy the code into js file named iCloud.js and put it in your Resources directory)

      require('iCloud').disableBackupForFolder(Ti.Filesystem.applicationDataDirectory);

      — commented August 21st 2013 by Moritz Knecht
