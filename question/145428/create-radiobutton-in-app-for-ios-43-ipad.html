    For selecting an option from multiple different items, you can use optionBar UI and store the changed value like this.

    var tb = Titanium.UI.iOS.createTabbedBar({
    style:Titanium.UI.iPhone.SystemButtonStyle.BAR,
    // some other options here
});

tb.index = selected; //selected is the initial value in integer
bb1.value = selected;

tb.addEventListener("click", function(e){
    tb.value = e.index;
});

// get the current value by accessing the value property like this
var currentValueIndex = tb.value;

    Another option is creating your own control like a toggle button

    
var iv = Ti.UI.createImageView({
    mi: "images/male.png",
    fi: "images/female.png",
    value: initialValue // initialValue is the default option (i)
});
iv.backgroundImage = initialValue==0?iv.mi:iv.fi; // assuming 0 is male and 1 is female
iv.addEventListener("click", function(e){
    if(iv.value==1){
        iv.value = 0;
        iv.backgroundImage = iv.mi;
    } else{
        iv.value = 1;
        iv.backgroundImage = iv.fi;
    }
});

  • Hi

    The switch object does not currently support caption changes for iOS - unfortunately. Also not available in the upcoming Ti 3.0 SDK.

    You are left with a few choices;

    • Create your own, but you will have to handle swapping images between states and provided artwork in at least two sizes.
    • Use a different controls (not ideal), for example; TabbedBar
    • If you were adding the switch to a table, consider grouping Male and Female and indicate with hasCheck
    • Use a module from the market place SliderSwitch
    • Create your own module - if your new this is most likely left for another day.

    The third one is my preference.

    I would imagine this is due in part to the multiple language support that that means there is very little space available for other language variations. For example the French equivalent of the Yes/No switch is |/0 - not ideal.
