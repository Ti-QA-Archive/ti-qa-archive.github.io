Android webview - PDF links don't work

Hi, I have a webview which loads an external webpage. The webpage happens to have a link on it to a PDF. The link opens fine in iPhone/iPad but on Android the link highlights but does not open. If I open the webpage directly in the Android browser the PDF link opens fine.

var webview = Titanium.UI.createWebView({url:'http://www.webpage_contains_a_link_to_a_PDF.com'});
var window = Titanium.UI.createWindow();
window.add(webview);
window.open({modal:true});

Any ideas? Many Thanks.

  • The answers below do nothing to address the situation where you're loading an external page in the webview (one that you don't have control over the code for that webpage and is not stored locally on the device) and that external webpage contains hyperlinks to PDF files.

    There's no way that I can see to handle that situation with a webview. When a PDF link is followed in the webview it does not fire any titanium events. No "beforeload", no "load", not even "error". It's impossible to inject javascript onto the remote webpage with evalJS() that fires events that are catchable by the Titanium platform so there's no way to handle it.
