    • I suspect the problem is related to the webView as this is a VERY memory intensive view.

      Can you try your exact same code but DO NOT add the webView at all are place it with a very simple label as below;

      /*
var contentlabel = Titanium.UI.createWebView({
    html:"<html><head></head><meta http-equiv=\"Content-Type\" content=\"text/html; charset=utf-8\" /><body style='width:250px;overflow:hidden;font-family:Arial;'>"+Message+"</body></html>",
    width:'86%',
    bottom:'5dip',
    height:'100%',
    left:'7%',
    font:{fontSize:'10dip'},
    color:'black',
    scalesPageToFit:false,
    className:"Content"+ClassNameCount // this should not be here
});
*/
// this will ONLY show the caption 'message' and NOT the value 'message'
var contentlabel = Titanium.UI.createLabel({
            html:"message",
            width:'86%',
            bottom:'5dip',
            height:'100%',
            left:'7%',
            font:{fontSize:'10dip'},
            color:'black'
        });

      Does doing this stop the error?

      Notes

      • You can add a webView into another view - but you should really not use many and ideally only one per window.

      • You have the property className on the webView this should be on the tableViewRow

      • Your use of the property is also wrong, the idea of the className property should be the SAME for any row with the same style, this allow the OS to speed up the display by caching the row structure of similar row layouts.
      tablerow[dataCount]= Ti.UI.createTableViewRow({
    className: "Standard"
});
