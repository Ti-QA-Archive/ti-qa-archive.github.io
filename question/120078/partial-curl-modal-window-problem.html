Partial curl modal window problem

I have a button on the toolbar which shows a modal iphone window with the partial page curl style.

I noticed these dialogs can be dismissed by tapping on the page curl.

However, if this is done, the next time the dialog is invoked it is blank. If I create a button that calls the dialog's .close() method, then it works fine.

Why would this be, and is there anything I can do about it, if the user doesn't use the button but "closes" the page curl by tapping on it?

  • well i stumpled upon this today. I open a modal window with a partial curl and if the user clicks on the partial window, the partial window will uncurl, but you will not be able to open a model window again.
    There seems to be a listener on the click event of your model window per default. This event closes the window once you clicked outside of the modal window, which is only possible if you hit the partial part.

    My workaround for this situation is an eventlistener for the blur event of you window and close the window within that event listener. That solves the problem for me.
