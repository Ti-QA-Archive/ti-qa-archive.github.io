  • i know it is after 11 months… but you can change package name in tiapp.xml.

    <?xml version="1.0" encoding="UTF-8"?>
<ti:app xmlns:ti="http://ti.appcelerator.org">
    <id>com.example.app</id>
    <name>Example</name>
    <version>1.0.0</version>

    The package name is tagged as

    <id>com.example.app</id>

    Change this to whatever you want. > clean project > delete contents of build folder in project. This should give you app with new package name.

    BEAWARE however! Changing package name will result in:

    • as Manuel partially said (you still can change it :P), if you published app in google play, you wont be able to update it anymore,
    • if you didn't upload to google, but got the app on some devices, note, that it wont be updated anymore. It won't be deleted by system, when you install this app after change. You'll get 2 apps that look and are named the same (one with old package name and the one with new),
    • if you had some references with package names in your code, changing packade name will break the code. You'll have to track down all package name references to fix this.

    This is, as far as i know, everything you have to know about changing your package name. Hope this helps others in making decision to change or not. :)
