TableView Black Corner Edges When Using backgroundImage

I'm testing on the iphone simulator using the Grouped TableViewSytle. The top and bottom rows of the table have rounded corners. When I set a backgroundColor everything looks good. When I set a backgroundImage I see black triangles outside all the rounded corners.

Any ideas on how to fix this?

  • To do this, set the background on your window, and then set the background of the table view to transparent. You may have to set background on your rows to make things right, but it works and there is an example of it in the Kitchen Sink.

  • I've seen this bug too…the backgroundImage appears to be not "in the background" but on top of the TableViewRow.

    The only way I found to fix this was to use a different backgroundImage with the corners transparent for the top and bottom rows. A complete pain but the only way to solve it! (Or don't use the Grouped style!)
