passwordMask does not work when TextField inside TableViewRow?

I'm trying to create a login form, and I noticed that the password field doesn't allow for more than 1 character to be inserted if the passwordMask is set to true. But same code works fine when I move the textfield out of the TableViewRow. Is there a way to get the expected behavior?
Here is my TableView code - 

<TableView style='Titanium.UI.iPhone.TableViewStyle.GROUPED' background='transparent'>
    <TableViewRow>
        <Label id='lblUsername' text='Username'/>
        <TextField id='txtUsername' width='200dp' hintText='Username'></TextField>
    </TableViewRow>
    <TableViewRow>
        <Label id='lblPassword' text='Password'/>
        <TextField id='txtPassword' passwordMask='true' width='200dp' hintText='Required' onReturn="closeKeyboard"></TextField>
    </TableViewRow>
</TableView>

Titanium Studio, build: 3.0.2.201302191606
Titanium SDK: 3.0.2.GA
Platform: iOS
Device: Simulator+iPhone5
