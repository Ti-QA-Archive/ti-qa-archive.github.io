registerForPushNotifications callback() Problem - No Notifications in App

Hi guys

I am developing an app that relies heavily on push notifications.
I'm using the apns-php library (no UrbanAirship) to send notifications to Apple on the server side and until yesterday everything went perfect.

But suddenly, the app can't receive push notifications while running.
When in background or closed, push notifications are received as usual with the correct alert text and custom sound.
What doesn't work, however, is the process defined in the callback(e) function of registerForPushNotifications.
As soon as a notification is received while the app is open, the Xcode console prints the following error:

Nov 12 10:50:12 unknown UIKitApplication:com.XXX.YYY[0x2f50][3957] <Notice>: [WARN] Accessing APS keys from toplevel of notification is deprecated

I couldn't find ANYTHING about this issue, this warning seems to be quite unusual. Could you tell me what's going on here? :)

I use custom payload, sending a variable msgType to determine the kind of notification and userId to identify the user who sent the message.

  • Titanium SDK 1.7.5 (error reproduced in 1.7.2 & 1.7.3 as well)
  • No modules or other javascript libraries used
  • iOS 5.0
  • apns-php r100
  • Test devices: iPhone 4, iPhone 4S, iPad 2

I'd really appreciate your help!

All the best,
Luis

  • If you check the obj-c source (this shouldn't be necessary) you'll see it's just a debug statement with a false alarm. Based on the code it will happen on every push notification while your app is running but has no impact other than unnecessary logging.
    If you are not receiving the push notification in the app I would think that is indicating some other problem but it is making it to the device if you see this log.
    If you plan on using titanium you should get familiar with the source and building it. Unfortunately you can't survive in Javascript land alone.
