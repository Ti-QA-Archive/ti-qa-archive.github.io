Multiple NameSpaces or Sub NameSpaces definition

I use several namespaces, App, Db. Network, UI…

(function() {

    var App = {
        title:<title>,
        autor:<author>,
        ....
    };

    var db = {
        init: function() {},
        ....
    };
})();

the question is whether it is right to be independent as the example, or defined within each other like this, or doesn't matter

(function() {

    var App = {

        title:<title>,
        autor:<author>,
        ....,

        db: {
            init: function() {},
            ....
        },

        ui:{
        },

        <etc>
    };
})();

    Hi Xabi,

    Every namespaces declaration should be independent of each other. If you define the namespace within other, then it will consider as element of the parent namespaces.
