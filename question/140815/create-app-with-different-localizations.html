    Hi

    This could not be easier, There is a command you call L(&#39;phrase&#39;, &#39;fallback&#39;), which looks up country specific XML files, that find that phrase and then swaps it for the one in the list.

    Docs here

    Then when the device is running on a phone with Italian as the default language it uses that xml file, and as for English - it uses that file.

    So you would do;

    var lbl = Ti.UI.createLabel({
  height: Ti.UI,SIZE,
  text: `L('greeting', 'hi')`,
  width: Ti.UI.SIZE
});

    Then it looks up the xml file (example)

        <?xml version="1.0" encoding="UTF-8"?>
    <resources>
      <string name="phrase" formatted="false">You say %2$s and I say %1$s!</string>
      <string name="greeting">hello</string>
      <string name="signoff">goodbye</string>
    </resources>

    If it finds a match, it uses it, if not it uses the fallback.

    You can use the L() anywhere you have text, you can even format as above and in the docs.

    Happy lingustics .

    • You have spelt strings.xml two different ways in you comment, can I check you have not also done the same in the actual filenames.

      Also make sure you have done a clean project build Project > Clean after closing the simulator(s) and then try again.

      The others things to make sure are, the location of you i18n folder is at the same level as your resources, not lower - example

      [appname]
  resources
    app.js
  i18n
    en
      strings.xml
    it
      strings.xml

      Also you must have the device or simulator set up for that region or it will not work at all.

      One more thing, always provide a fallback - just in case.

      L("prova", "hello")
