How serious are [WARN]'s?

Hello,

I have been trying for a while to find a way to check if a view exists in a window before removing the view from the window. I have not been successful. (If someone wanted to help me out there, I'd be grateful, but that's not what this question is for.) What I ended up doing instead is removing all possible views from the window and then adding back the one I need. Naturally, this means I am often removing views from a window that don't exist. When that happens I get this warning:

[WARN] called remove for [object TiUIView] on [object TiUIWindow], but [object TiUIView] isn't a child or has already been removed

This is to be expected. Sometimes I also get warnings that look like this:

[WARN] [object TiUITableViewRow] has an auto height value of 0, meaning this view may not be visible.

So my question is: How seriously should I pay attention to these warnings? Should I view them as show-stoppers and make sure they are all resolved? Will they cause memory problems once the app is in the device? Or… am I okay letting them slide?

Thanks!

  • First warning should be serious, you tried to remove a view that does not exist inside a window. So, that's your error. Second warning is not so important, just set default height value to table view row (like Ti.UI.SIZE) and it should be gone.
