// Funçao que checa a colicao
function fingerSlideCollider(originObj, targetObj, e){

    var pointRelativeToTarget = originObj.convertPointToView({x:e.x, y:e.y}, targetObj),
        x = pointRelativeToTarget.x,
        y = pointRelativeToTarget.y,

        onX = (x > 0) && (x < targetObj.rect.width),
        onY = (y > 0 ) && (y < targetObj.rect.height);
    return (onX && onY);    
};

  • You will want to add eventListener for (touchstart) touchmove (and touchend) events on objectA.

    In the touchmove callback, check the if x, y position of the event source (objA) overlaps the area occupied by objectB (see its rect property) to infer a collision.

    I could try to write down a testing example, if you find problems with it. Regards

    • "touchmove
      Fired as soon as the device detects movement of a touch.
      Event coordinates are always relative to the view in which the initial touch occurred"

      This means you have to add the listener to the parent of objA and B, where the touch event bubbles to.

      • With touchStart you can know if the finger was over objA position

      • While touchMove you can infer a collision with objB

      • On touchEnd you will know that the finger left the screen and act accordingly.

      If i manage to put together a working example I'll put it here.
