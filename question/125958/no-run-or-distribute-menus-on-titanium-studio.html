No Run or Distribute menus on Titanium Studio

We are ready to do a final build and distribution of an app, but when I imported the project into a coworker's computer, none of the menus for building the app are available. I know about the trick (which should not be necessary, by the way) of making sure there is an iPhone/ folder under the build/ folder, but that has not made a difference.

The project itself was imported from a git repository. Everything was imported correctly as far as I can tell, it just simply has no way to build.

Creating a test project from scratch does build, so there is no problem with Titanium seeing xcode. We're under a tight today deadline, so any help is greatly appreciated.

  • I've had that problem on occaison as well when importing a project directly from a github repository. The simplest way to work around this is the do a git clone YOUR_REPO_URL on the command line, then import the resulting project directory into Titanium Studio as you would any other existing project. I've never had the issue you've described importing projects in this manner.

  • You could try the following:

    1. With your project selected in App Explorer view, right-click on the empty space and select Properties;

    2. In the new dialog, select Project Natures on the left;

    3. Make sure Titanium Mobile is selected on the right and set to be the primary nature;

    4. Apply the change and see if the Run and Distribute menus are enabled and contain the necessary actions.
