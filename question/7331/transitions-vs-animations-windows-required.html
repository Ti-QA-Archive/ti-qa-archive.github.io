Transitions vs. Animations: Windows required?

Except for native transitions (iPhone: curl up/down, flip right/left; Android: wink), are additional windows for animation are required? (one for each view?)

The KS "view animation" animates a window, not a view. Other discussions on related topic suggest the same thing. I understand that windows are "lightweight," but this does represent a major difference in transitioning between views/windows. This is especially true when sliding text views left/right (where you need to move both left & right sides to prevent damage to the contents).
