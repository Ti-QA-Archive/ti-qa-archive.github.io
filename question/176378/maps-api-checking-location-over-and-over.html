Maps API checking location over and over?

I have an app using Google Maps. Titanium 3.2.3.GA

There are a couple places I call the geolocation services to get the users position. And the map itself is going to make some calls.

But in an idle state, I am finding that the device continues to poll the GPS. It halts polling when sent to the background, but anytime it is the focused app, it starts polling. It will hit the GPS for about 3-4 seconds, then stop for 2-3 seconds, then hit it again for 3-4 seconds. Looping indefinitely.

During this loop, my app isn't making GPS calls.

Any ideas? In this case, it is a Galaxy 4 running 4.4.3, but other devices have exhibited this behavior.

  • You must remove the event listener for geolocation. Depending on platform you link this on Ti.App or on Ti.Android.currentActivity.

    — commented July 22nd 2014 by Rainer Schleevoigt

  • All my Geolocation calls are made with closures.
    Ti.Geolocation.getCurrentPosition(function(e){ … });

    I don't have any event listeners tied directly to the Geolocation call. Within my function, I do use the Ti.App.fireEvent to call a registered listener whom handles geo changes. Or maybe in the future more listeners.

    However, I'm just calling a state change.
