iOS Enterprise Production - Xcode thinks my app is a mac app

Hi

I'm trying to build an app for Enterprise Distribution but I'm having problems with it. I've seen some duplicate questions on here but I think the answers might be outdated at this stage.

The answer here says to package the app for iTunes distribution and then in Xcode to click Distribute and then select Enterprise distribution. However this option is not available to me and the first option is to "Distribute to the Mac App Store", so it seems that Xcode thinks I've built a mac app!?

Has anyone come across this before? Am I doing something wrong in the earlier steps?

Any help would be hugely appreciated as we're getting pretty close to a release deadline!

Thanks

James

(Mac OSX Mountain Lion, TiStudio 3.1.2, iOS)

  • I found a fix for this myself. Since I was releasing a beta build, I had set the version in TiApp.xml to 0.11.0. Apparently Xcode or Titanium doesn't like this and when I set the major version to 1, e.g. 1.11.0 it archived it okay and recognised it as an iOS app. No idea why, but it worked fine after I changed this…
