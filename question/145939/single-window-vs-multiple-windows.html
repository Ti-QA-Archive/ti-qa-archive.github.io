Single window vs Multiple windows

Hi.

I just wanted to ask what would be considered better practice and if there are any real advantages to using multiple windows, aside from the ability to transition between them on iOS.

My first Titanium project was a 2D game which really only needed a single window. Objects within the window would transform appropriately and be nulled out when they are no longer needed. My second app was a more full-scale application but I kept the same 'single window' paradigm. Whenever the user moves from one screen to another, I swap the root view with a 'loading' view, null out the root view to free up memory and then re-instantiate it. Are there any real disadvantages to doing it this way as opposed to using multiple windows?

Cheers,
Jay.

  • There certainly are advantages and disadvantages in both ways.

    When you need to create a tabbed project for example with lists that have children and get you to new windows on click, the best way is to use multiple windows. This way you keep the possibility to go BACK without having to recreate the previous window.

    When you need to create simple applications that won't go far creating many windows (and most important, they don't have relation to the previous window) then you can use one window.

    Your choice.
