Table View Row Font Size revisited

Under the Documentation for Ti.UI.TableView, properties listed include "data", "font-size", "font-style", "font-weight", etc. However, when you attempt to create a TableView with the font properties clearly documented, the compiler balks.

I'm sure I am misunderstanding something, but here's what I tried:

var tableview = Titanium.UI.createTableView ({
    font-style : 'italics',
    data : data
});

I'm obviously missing something obvious and I'm just not seeing it. The compiler doesn't seem to like the hyphen between font and style! But isnt this a property?

    In my limited attempts in the past, I was never able to do it at the table view or even row level for a row title itself. When I want to affect the font of the main text on a table row, I leave off the title property and add a label object, where I have full control.

    There may be an easier way. But sometimes I just move on to a technique that I know I can make work…

  • font-style like that is not valid javascript. In such cases you should wrap it in quotes:

    var tableview = Titanium.UI.createTableView ({
    'font-style' : 'italics',
    data : data
});
// or instead of dot notation:
tableview['font-style'] = 'italics';
