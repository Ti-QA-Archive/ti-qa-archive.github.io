Android views ui: the width, the height and the ugly.

Let's suppose i'm creating a view like this:

{ width: 100, height: 100, left: 10, right: 10, top: 10, bottom: 10 }

Then I add this into an horizontal layout view container. With iOS i'm getting what I expect: every box has same distance from the others and height and width are set to 100.

On Android for some reason I get the box height stretched (100% parent view) and width fixed.

Why is that? Why they don't behave same way? How do I get the same render on both platforms?

Thanks to anyone will help.

    My guess is that when you specify a height, and a top and bottom, it will take the top and bottom, and will ignore the height. When you leave the bottom away, i think it will display just fine.

  • Elmar is right. In short, don't use opposite positioning properties at the same time. What you see is a side affect on Android (width/height is ignored). Just use one position property in combination with width/height.
