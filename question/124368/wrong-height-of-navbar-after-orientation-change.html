  • I really would like to see the issue fixed as well, but I do have a minor work around to share. Listen to the orientation change and redraw.

    It is a little obnoxious having to hide/show the nav bar. Sometimes, if the page is open, but wasn't the current page you have to hide & show the navBar on the focus event as well.

    var currOrientation = Ti.Gesture.orientation;
Ti.Gesture.addEventListener('orientationchange', function (e) {
    if (laf.device.isiPhone()) {
        //iOS not always redrawing nav bar to account for new orientation 
        setTimeout(function () {
            //Wait & validate the change was actually desired
            if (win != undefined && win.orientationModes.length > 1 && Ti.Gesture.orientation != currOrientation && Ti.Gesture.orientation == e.orientation) {
                currOrientation = Ti.Gesture.orientation;
                win.hideNavBar();
                win.showNavBar();
            }
        }, 600);
    }
}

    • Good to hear. Just giving you an update on my work around. We had a number of users complain that the display was too jittery so I dived into the Titanium source looking for a fix, which I didn't find. However, I did find that the hide/show navBar methods can be called with an animated property which reduces the jitter considerably. I would say the work around is pretty much acceptable for now.

      var currOrientation = Ti.Gesture.orientation;
Ti.Gesture.addEventListener('orientationchange', function (e) {
    if (laf.device.isiPhone()) {
        //iOS not always redrawing nav bar to account for new orientation 
        if (win != undefined && win.orientationModes.length > 1 && currOrientation != e.orientation) {
            currOrientation = e.orientation;
            win.hideNavBar({
                animated: false
            });
            win.showNavBar({
                animated: false;
            });
        }
    }
}

  • The above workaround with hiding/showing the navbar doesn't work for me on SDK 1.7.5, instead i found this hack to be working:

    var currOrientation = Ti.Gesture.orientation;
Ti.Gesture.addEventListener('orientationchange', function (e) {
    if (api.utils.platform.isiOS()) {
        //iOS not always redrawing nav bar to account for new orientation 
        if (win != undefined && win.orientationModes != undefined && win.orientationModes.length > 1 && currOrientation != e.orientation) {
            currOrientation = e.orientation;
            var img = win.barImage;
            win.barImage = null;
            win.barImage = img;
        }
    }
});
