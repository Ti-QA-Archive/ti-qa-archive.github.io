borderWidth and borderColor on Titanium.UI.View in Android broken?

I'm attempting to create a view with a 3 pixel gray border. The result I am getting though is unexpected. The view instead shows up with the entire background being the same color as the border, in this case all gray. Is there something I'm missing? Android API 2.3.3, Titanium Mobile API 1.6.1

var view = Titanium.UI.createView({
        borderColor: "#C0C0C0",
        borderWidth: 3,
        backgroundColor: "#FFFFFF",
        width:"90%"
    });

  • Turns out this is an issue with using nested views and percentage widths. I changed the widths on my views to be fixed and this issue went away. More in this post here: http://developer.appcelerator.com/question/116905/percentage-dimensions-in-nested-views
