"Max retries reached, giving up decoding image source"

When loading a remote image into an ImageView, I get these errors:

[WARN][TiDrawableReference(  771)] (main) [19820,219160] Bitmap bounds could not be determined. If bitmap is loaded, it won't be scaled.
[ERROR][TiUIImageView(  771)] (main) [27,219187] Max retries reached, giving up decoding image source: http://www.worldweatheronline.com/images/wsymbols01_png_64/wsymbol_0002_sunny_intervals.png?rnd=0.5698938753921539

As you can see, the image in question loads fine.

This only happens on Android. Works fine on iPhone.

  • Just to follow up on the comments made so far, I'm also seeing a similar sort of thing on Android 4.2.x with TiSDK 3.0.2 with images from world weather online.

    Example:

    http://www.worldweatheronline.com/images/wsymbols01_png_64/wsymbol_0001_sunny.png

    From DDMS:

    02-21 14:44:36.481: 
W/TiDrawableReference(20393): 
(AsyncTask #1) [2,1736474] Bitmap bounds could not be determined. If bitmap is loaded, it won't be scaled.

02-21 14:44:36.481: 
D/skia(20393): 
--- SkImageDecoder::Factory returned null

02-21 14:44:36.481: 
E/TiUIImageView(20393): 
(main) [3,1736477] Max retries reached, giving up decoding image source: http://www.worldweatheronline.com/images/wsymbols01_png_64/wsymbol_0001_sunny.png

    The ImageView's error event does fire but doesn't return anything useful, not even the url for the image which I would have expected to be in the .image property of the event if the docs are to be believed.

    I had a quick look at the PNG with a hex editor & it looks kosher. I can see the size of the image defined in the PNG's IHDR chunk. In fact downloading the image & referencing it locally displays the image fine.

    Needless to say it displays fine on iOS.
