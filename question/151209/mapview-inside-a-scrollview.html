MapView Inside a ScrollView

Essentially, I building an app, that has a page with a small map view in it, and a list of stores, I need to be able to scroll the entire page, so I must create a scrollview, when I wrap everything in a scrollview though, i cannot pan north and south on the map at all, it just move the scrollview up and down, is there any way to have the entire page scroll, and still be able to pan the map? My page layout:

<Alloy>
    <Collection src="result" />

    <Window class="container" id="results">
        <ScrollView id="mainScroller">            
            <View id="mapview" ns="Ti.Map" animate="true" userLocation="true"
                mapType="Ti.Map.STANDARD_TYPE">
            </View>        
            <View id="listOfResults" dataCollection="result">
                <Require src="resultItem" id="includedResultList" />
            </View>
        </ScrollView>
    </Window>
</Alloy>

Thanks for any help!

  • I don't think this one will be solved by Titanium, though I've been wrong before. This is the nature of trying to nest components that have the same vertical scrolling behavior, the parent is generally going to win. This is not unique to Titanium. If you check for similar problems in native development as well it rears its head. For example, this one in native Android dev that required a developer to write a custom Map class and override its touch event handling.

    I would submit your test case above to the Titanium Community Jira Project and be sure to include additional detail like the target platform, Titanium SDK, etc…
