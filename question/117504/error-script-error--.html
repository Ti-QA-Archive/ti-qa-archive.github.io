  • When you post a question to the Q&A, you will receive much more accurate answers more quickly by posting a proper "usecase". A usecase is a self-contained piece of code that demonstrates your issue in the least possible number of lines. Some additional advantages of providing a usecase are that it is much kinder to those wishing to help you, because they won't have to spend considerable time replicating your issue on their systems, that you will gain a much great knowledge of how Titanium works, and lastly that the process of creating a usecase can often simplify the problem and consequently reveal the answer. You can read more about this subject in the official documentation, under the section Creating Good Use-cases.

    Note that, if the code is long, you may want to create a Git Gist or pastebin rather than paste the code here directly.
