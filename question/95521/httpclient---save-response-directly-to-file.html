HTTPClient - save response directly to file

Hi All,
I've seen comments indicating that Ti Mobile supports storing a HTTP response directly to a file (I'm trying to get around the memory leaks in the HTTP client when pulling multiple large files)

It appears that setFile calls setDownloadDestinationPath which sets downloadDestinationPath , which has a comment of:

    // When downloadDestinationPath is set, the result of this request will be downloaded to the file at this location
    // If downloadDestinationPath is not set, download data will be stored in memory

However, its not actually working - calling setFile doesn't actually set the file, and checking the .file property doesn't report back the value set with setFile.

Can anyone give some guidance?

(also, is there some super-secret documentation where all this stuff lives? The public API is missing a ton of information, and I'm not sure if thats intentional to spur community-organized documentation efforts, or if I'm missing something)
