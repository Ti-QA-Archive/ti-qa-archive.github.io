App in Genymotion crashes if not running TiShadow

  • Genymotion: 2.3
  • Ti SDK: 3.4.0.GA
  • Mac OSX 10.9.5

I've been using tishadow with Genymotion for a few months now with no problems. While debugging my app yesterday, I needed to run the app outside the tishadow environment. So, I launched it directly from Genymotion. However, the app never proceeds farther than the splash screen before it crashes.

  • I can run the app from within tishadow in genymotion.
  • I can run it without either tishadow or genymotion in a standard android emulator.
  • I checked the app through android device monitor without finding anything that points to a cause.
  • A new test app runs in genymotion with no problems, so it has to be in my app itself.

I don't know when this started because, as I said, I haven't had to test without tishadow. I have no idea what set of conditions would need to exist to recreate this scenario. I can recreate the problem on two different iMacs, one with a licensed version of Genymotion, the other with the free version. I'm just wondering if anyone else has experienced this behavior. If so, please share if you found a solution.
