Play Sound from background service

Hi!
In my app, I play a sound when an event occurs, using the following:

var player = Ti.Media.createSound({url:"/audio/new_message.wav"}); // Create sound
player.play(); // Play sound
player = null;

This works perfect, but, using the exact same code on an android service, it throws the following exception:

  • Accepted Answer

    The Titanium Sound proxy requires a window since it implements the activity-lifecycle events. That means it won't work from a service because the activity reference is null then as you can see in the error log. It might be possible to implement this without the dependency on a window. Therefor, you might want to request this feature as a new JIRA ticket.

    • Thanks Manuel.
      Your answer helped me to find a work around.

      The workaround is to use the default notification settings when creating the notification like this:

      var notification = Ti.Android.createNotification({
//... settings
defaults: Ti.Android.DEFAULT_ALL // Plays default sound, default vibration and default lights.
});
