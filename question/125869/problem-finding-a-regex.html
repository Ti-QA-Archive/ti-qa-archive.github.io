Problem finding a regex

Hello everyone,

I am stuck right now. I have a webview with remote content stored locally (basically some text with html tags and a little css). Everything works fine except that some of the images and links are relative to the website and so the app can't reach them.
Basically I find myself having these tags : 

<a title="ttl" href="/path/to/page.html">
 and
<img src="/path/to/img">

I would like to transform these relative links into absolute links but I just can't figure out the regex that would do that.

Any help would be really appreciated.

Vincent

  • I may just have found the answer if anyone if interested.
    I added in the head tag of my webview a tag <base href="www.yoursite.com" /> as it was the case in the original website. That arranged my problem but the local css file wasn't loading anymore since it had been linked with a local url, relative to the application at first but then relative to the base url. I solved that by prepending the local url with 'app://com.vincentghyssens.myapp/' and it works well for the moment
