Some filters not working on Node.ACS

Hi.

I have this strange issue with filters.

I have some filters defined some time ago that are working well, but the news I'm including now aren't.

The code is so simple:

function validateSession(req, res, next) {
  if(!req.session.user) {
    res.send('Unauthenticated!');
  } else {
    console.log(req.session.user);
    next();
  }
}

And my config.json:

{
  "routes":
  [
    {"path": "/", "callback": "application#index"},
    {"path": "/login", "method":"post", "callback": "useraccess#login"},
    {"path": "/logout", "callback": "useraccess#logout"},
    {"path": "/home", "callback": "application#home"},
    {"path": "/about", "callback": "application#about"},
    {"path": "/contact", "callback": "application#contact"},
    {"path": "/contact-end", "method":"post", "callback": "application#contactFinish"},
    {"path": "/error", "callback": "application#errorInfo"},
    {"path": "/admin", "callback": "application#home"},
    {"path": "/admin/users", "callback": "users#list"},
    {"path": "/messages", "callback": "messages#list"},
    {"path": "/changePwd", "method":"get", "callback": "users#changePassword"},
    {"path": "/changePwd", "method":"post", "callback": "users#changePassword"},
  ],
  "filters":
  [
    {"path": "/admin", "callback": "session_filter#validateAdmin"},
    {"path": "/admin/*", "callback": "session_filter#validateAdmin"},
    {"path": "/logout", "callback": "session_filter#validateSession"},
    {"path": "/changePwd", "callback": "session_filter#validateSession"},
    {"path": "/messages", "callback": "session_filter#validateSession"},
    {"path": "/messages/*", "callback": "session_filter#validateSession"}
  ],
}

Well, all the older filters (admin, logout, messages) are working well and, if the user is not loged in, throws the error.

But I now included the changePwd, copying it from messages (that is working) and nothing happens, it simply renders the page. Even I have included that console.log(req.session.user); to see if something is saved, but it seems that simply it is not executing the filter, because there is nothing in the console after executing the call.

What I'm doing wrong?

  • Sh*t!!

    Viewing the code from a higher perspective I see only a difference: the uppercase letters in the filter… and was that!!!

    I changed the path from changePwd to changepwd and now is working!!!
