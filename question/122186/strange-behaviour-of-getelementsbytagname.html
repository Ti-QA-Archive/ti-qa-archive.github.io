Strange behaviour of getElementsByTagName()

Hello folk,

I ran into an issue when parsing XML.

first of all: i'm running the Titanium 1.7.1 SDK and developing for both android and iphone

here's my code (which actually works): 

function getImage(item, feed){
            if(feed == 'bip'){
                return item.getElementsByTagName('media:content').item(0).getAttribute('url');
            }else{

                Ti.API.debug('trying to retrieve image');
                var img = '/images/logo_med.png';

                var thumbnail = item.getElementsByTagName('media:thumbnail');
                // tests
                if (Ti.Platform.name == 'android'){
                    try{
                        img = thumbnail.item(0).getAttribute('url');
                    }catch(e){
                        Ti.API.debug('No image found');
                    }
                }else{
                    Ti.API.debug('iphone');
                    if (null !== thumbnail){
                        Ti.API.debug('type of thumbnail '+thumbnail);
                        img = thumbnail.item(0).getAttribute('url');
                    }    
                }

                return img;
            }
        }

i've had to split the process of getting the url attribute since the behaviours of OS's aren't equal:

in iOS, if the element with tag doesn't exists the value of thumbnail is null. In Android it is Ti.NodeList

In Android I had to add a try-catch not to get a nullPointerException at runtime.

I also had to test which platform i'm on, because Android will crash at runtime parsing the statement img = thumbnail.item(0.getAttribute(&#39;url&#39;) if not surrrounded by try-catch and throws the nullPointerException if thumbnail is actually null in if (null !== thumbnail)

The strangest behaviour i noticed is that the app freezes (in iOS simulator) if I don't put the debug statement Ti.API.debug(&quot;&#39;type of thumbnail &#39;+thumbnail); before I try to get the url!!! (is that a bug ?)

Hope this could help some of you, and also wanted to know if anyone ran into the same strange issue?
