Getting the correct image in a webview on Android

Hi,

I have the following structure in my Resources directory:
android/images/high/image.png
android/images/medium/image.png
android/images/low/image.png

I am trying to reference image.png in a webview, e.g.:
html = "<html><body><h1>Image should be below</h1><img src='images/image.png'></body></html>";

I've tried this as well:
imgsrc = Ti.Filesystem.getFile(Titanium.Filesystem.resourcesDirectory, "images/image.png").nativePath;
html = "<html><body><h1>Image should be below</h1><img src='" + imgsrc + "'></body></html>";

Neither work, both return a broken image. Does anybody know how to do this or if it is even possible?

Thanks!

  • No, the webview does not support the density/resolution specific folders for android. You could however manage this yourself. You can use the values returned by Ti.Platform.displayCaps.dpi or Ti.Platform.displayCaps.density to determine which image you would like to use. You could then pass this value to your webview, and let the webview create the appropriate URL to your density/resolution specific images.
