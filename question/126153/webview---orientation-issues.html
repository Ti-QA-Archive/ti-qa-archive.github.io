Webview - orientation issues

I have an event on my webview:

  window.onresize = function(event){  resetColumns(); }

This simply will reset column widths when the device is rotated. This generally works great.

However, if I start the webview off in Landscape mode, and change the orientation, the window.width never gets updated! Meaning that the columns stay in the same way as in landscape mode - width 1024 instead of 768.

I also tried to use Titanium.Gesture.addEventListener(&#39;orientationchange&#39;,function(){ Ti.App.fireEvent(&#39;resetColumns&#39;)); - This also worked fine in most cases, but not when we start the webview off in Landscape mode… Very frustrating.

On the event firing, debugger spits out:

[ERROR] listener callback is of a non-supported type: NSDecimalNumber

What?? Any ideas? Everything else has been smooth sailing.

  • You could try to detect the launch orientation like this in your app.js : 

    
if(Titanium.Platform.displayCaps.platformWidth<Titanium.Platform.displayCaps.platformHeight){

    Ti.API.info("Launched in PORTRAIT");    

} else {

    Ti.API.info("Launched in LANDSCAPE");    
}
