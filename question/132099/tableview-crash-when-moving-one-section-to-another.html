TableView Crash When Moving One Section to Another

QUESTION: How can I make it so that I can move a row in one section to the last row location of another section?

BACKGROUND: I have a table with several sections and multiple rows in each. There is no problem when moving rows within their parent section or when a row is moved from a different section to a location that is not the last index spot of the section. The problem is that it crashes when a row is moved from one section to another and the new location is the last index of the section. From the output I am guessing that the it can't insert a row at index 3 when it only goes to 2…

TRIED: I did find to semi-solution in which a footerTitle is added to the section so that it creates an area for a row to be moved to. But it still crashes.This solution example is found at http://davemeehan.com/technology/moving-uitableviewcell-between-sections-in-grouped-uitableview

  • You need to:

    • copy section row references to array
    • copy section reference to local variable
    • remove section and it's rows from table view
    • re-add section with rows that you stored in a variable and an array

    You can use similar principles described here: https://gist.github.com/1485751
