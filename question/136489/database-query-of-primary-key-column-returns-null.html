Database: Query of primary key column returns null

Strangely, I can query all columns from a table with the exception of the primary key, which always returns null.

Here is the code used to create and fill the table:

CREATE TABLE "mytable" ("id" INTEGER PRIMARY KEY  NOT NULL  UNIQUE , "name" TEXT NOT NULL , "nr" INTEGER NOT NULL )

INSERT INTO mytable VALUES (1, "Testname1", 1)
INSERT INTO mytable VALUES (2, "Testname2", 5)

Querying the table's data with SQLite Manager shows that the table was set up properly and contains data. However, if I install the database and query its values, the row id (primary key) always returns null.

Here's my test code:

var db = Ti.Database.install('myDB.sqlite', 'myDB');
var rows = db.execute('SELECT * FROM mytable');
while (rows.isValidRow()) {
    var id = rows.fieldByName('id'),
         name = rows.fieldByName('name'),
         nr = rows.fieldByName('nr');
    Ti.API.info(id + ': Name=' +  name + ' Nr=' + nr);
    rows.next();
}


--------- OUTPUT -----------
null: Name=Testname1 Nr=1
null: Name=Testname2 Nr=5

Why can't I query the primary key column?

Working with Titanium SDK: 2.0.1.GA2; Android: 2.3.2 (on device)

  • Thank you all for your suggestions! I have just found the reason for this issue.

    The problem was that changes in the database scheme (for example renaming a column) are not properly recognized by the app/Titanium. The column was obviously found with the new name (I didn't get a 'no such column error'), but any request for data of this column resulted in 'null'.

    If someone has the same or a similar issue, try the following after changing the database scheme (worked for me):

    • If running on simulator: completely clean the simulator and reinstall the app
    • If running on device, uninstall the app from the device and then reinstall it

    The only way to update the database after a schema change is to completely uninstall the old app version and then install the new one.
