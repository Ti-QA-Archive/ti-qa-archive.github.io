The use of prototype

Hi all.
What I need to achieve is to have a class that I can use across all my project files that holds information regarding the current user.

SO in a file called UserClass.js I have written this:

var UserClass = function(){};

UserClass.prototype.data = {
    id:'',
    name:'',
};

UserClass.prototype.loggedIn = false;
UserClass.prototype.valid = false;
UserClass.prototype.blocked = false;

module.exports = UserClass;

Then I can call anywhere like this:

var UserClass = require('classes/UserClass');
var user = new UserClass();

Can also set variables values like this:

var UserClass = require('classes/UserClass');
var user = new UserClass();
user.data.name = 'My Name'

And this value will be available through all my project files.

BUT, is this acceptable? I'm I gonna run into memory problems later since I need to call that that class a number of times?

    This is perfectly acceptable and is exactly what I do. I have not run into any memory issues doings things this way - the 'class' code itself is not being re-included over and over again due to it being CommonJS. I have written apps that have hundred of screens using this exact same style.
