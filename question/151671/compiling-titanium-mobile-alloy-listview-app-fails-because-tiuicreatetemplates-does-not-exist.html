Compiling Titanium Mobile Alloy ListView app fails because Ti.UI.createTemplates does not exist

I'm writing an Alloy app that contains a ListView. I'd like to use a different template to format the various ListItems in it. I can't however, seem to compile the app. First here's some info:

  • Application Type: mobile
  • Titanium SDK: 3.1.0
  • Titanium Studio build: 3.1.0.201304151600
  • Platform & Version I'm targeting: Android 2.3 (Emulator)
  • Device: Haven't tested
  • Host OS: Mac OS X 10.8.3

Bit of source code that causes this problem:

<Alloy>
    <Window class="container" layout="vertical">
        <TextField id="search_term" hintText="Search ..." ></TextField>
        <Button id="search_button" onClick="doSearch" >Search</Button>
        <ListView id="listview" defaultItemTemplate="template1">
            <Templates>
                <ItemTemplate name="template1">
                    <Label id="label1" bindId="heading" />
                    <Label id="label2" bindId="description" />
                </ItemTemplate>
            </Templates>
            <ListSection id="mainSection">

            </ListSection>
        </ListView>
    </Window>
</Alloy>

The compile logs are as follows:

E/TiExceptionHandler(  655): (main) [206,2534] ----- Titanium Javascript Runtime Error -----
E/TiExceptionHandler(  655): (main) [1,2535] - In alloy/controllers/index.js:61,34
E/TiExceptionHandler(  655): (main) [0,2535] - Message: Uncaught TypeError: Object #<UI> has no method 'createTemplates'
E/TiExceptionHandler(  655): (main) [1,2536] - Source:     $.__views.__alloyId1 = Ti.UI.createTemplates({
E/V8Exception(  655): Exception occurred at alloy/controllers/index.js:61: Uncaught TypeError: Object #<UI> has no method 'createTemplates'

It seems this error is accurate since the Ti.UI does not have a createTemplates() method. Is there anyone else who has seen this before?
