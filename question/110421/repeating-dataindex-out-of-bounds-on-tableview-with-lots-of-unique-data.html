Repeating data/Index out of bounds on tableview with lots of unique data

I have a 'standard' TableView with more than 50 rows, each containing rather unique data but arranged in the same format. The rows are rendered with a bunch of labels and an image view. All the labels have unique content, but there's only about 15 different images that are displayed.

If i specify a unique class name for each one of the rows, the view renders properly, but eventually i get a crash similar to the one here: https://appcelerator.lighthouseapp.com/projects/32238/tickets/2621-android-app-crashes-with-paged-tableview-15x-regression

If i specify a class name unique to the different image views (e.g. i have 15 rows with the same image, but different labels), the label content will be repeated after about 4-5 rows (usually about 1 off the visible views on the screen)

If i specify one class name for all the rows, only about 5 rows of unique data are rendered, and repeated for the 50 or so rows in the list.

It seems from that lighthouse ticket that there's only about 29 unique rows that i can generate. Is there any way to go over this limit?
