  • There are several reasons for this behavior.

    1. Maybe you have defined an expression in the debug expression tab which may cause an application crash,
      to fix this clear your expressions
    2. You have set a breakpoint within an event handler, which sometimes might not work at all
    3. Clear all breakpoints use the "Remove all" function and set your breakpoint again

    If this all does not help, create an new app,
    pick the newly created tiapp.xml file and replace the tiapp.xml
    of the project you are currently working on.
    Then do necessary amends like adding modules …

    Now your breakpoint should be hit and the application should stay alive.
