Create android emulator with 360dp width?

I'm developing an application for android devices (2.1+) using titanium sdk 2.1.4GA, titanium studio build: 2.1.2.201208301612, running on Mac OSX.

When adding images and other elements to the app, I'm specifying element width in DP, i.e. 300dp. The total screen width is considered to be 320dp.

However, some devices, such as HTC Radar and HTC Sensation 4G have screen width = 360dp and on these devices some elements of my application are positioned incorrectly (as some of the testers complained). I do not personally have any of these devices, and would like to test the behaviour in android emulator. However, all titanium emulators have 320dp width. I'm wondering if it is possible to create an emulator with screen width = 360dp and test my application in that emulator. Would appreciate if you give me a link/instruction on how to do that.

    • Just found a way to do that, so in case anyone else needs to do the same:

      1. In AVD manager, select the device and click "Details" button.
      2. It will show AVD details, among which there would be specified Path variable
      3. Go to that folder and edit config.ini file, in that file set hw.lcd.density as needed (it corresponds to Abstracted LCD density). May be the same should be done with hardware-qemu.ini file, I'm not sure (looks like the latter is overwritten every time the AVD starts from the config file).

      After that it is possible to start project as usual from the Titanium Studio. I was able to reproduce my original issue on the emulator!
