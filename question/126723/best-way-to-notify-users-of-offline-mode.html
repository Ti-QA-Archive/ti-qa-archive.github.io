Best way to notify users of offline mode?

Hi!

We've developed an iPad app that communicates with a web service via the Internet to function properly. We're adding some synchronization routines to the app so it replicates some basic data on the local device. When the app is unable to communicate with the web service we're going to put it in Offline mode so it knows to use the local DB instead of the web service.

Sorry for the story, but I thought the background would be helpful. I'm curious how you think it would be best to notify the user throughout the app (all open windows and tabs) that they are in offline mode. I don't want it to be intrusive, but I do want to make sure they're aware of it and that it stays visible until the app is able to communicate with the web service once again.

Any ideas?
