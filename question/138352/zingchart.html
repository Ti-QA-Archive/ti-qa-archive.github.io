    Shouldn't have any problems with ZingChart (disclosure: I'm on the team). Renders SVG and Canvas so it works across both platforms. You'd be better served to use SVG than Canvas where possible on mobile for performance. Glad to help: support (at) zingchart.com or Twitter @ZingChart .

  • I've tried embedding it although I am admittedly probably doing something wrong.

    I just dropped the zingchart/ directory in my resources folder (as if it were a web server /var/www/html/ dir) I created a simple html page using the zingchart quickstart guide that looks like this:

    <html>
<head>
    <!--<script src="flash_scripts/zingchart-1.1.js"></script> <!-- Flash Library -->
    <!-- or -->
    <script src="/html5_scripts/zingchart-html5-min.js"></script> <!-- HTML5 Library -->
    <script>
    window.onload = function() {
    zingchart.render({
    id : 'zingchart',
    output : 'canvas', // or use 'flash' or 'svg', Internet Explorer versions 6,7 and 8 will render using VML
    width : "100%",
    height : "50%",
    dataurl : 'graphdata.txt',
    //liburl : 'flash_scripts/zingchart.swf' // in case Flash Library is used
    });
    };
    </script>
</head>

        <div id="zingchart">
        </div>
</html>

    And I have a simple json file with the data called graphdata.txt - also from the example:

    {
    "graphset" : [
        {
        "type" : "line",
            "series" : [
                {
                "values" : [5, 10, 15, 5, 10, 5]
                }
            ]
        }
    ]
}

    and I'm calling it in a simple webview container:

    var chartView = Ti.UI.createWebView({
            url: "/zingchart/chart.html"
        })

        self.add(chartView);

    I just get a White background and this message logged in the console:

    [INFO][TiWebChromeClient.console(  265)] (main) [968,9781] TypeError: Result of expression 'zingchart.render' [undefined] is not a function. (48:file:///android_asset/Resources/zingchart/chart.html)

    Maybe a relative path issue? All of this works fine hosted on a standard web server - if I point the webview to that hosted url rather than locally.
