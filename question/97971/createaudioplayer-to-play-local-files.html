createAudioPlayer to play local files

I am willing to know if I can use Titanium.Media.createAudioPlayer (instead of createSound) to play local (not remote) files.
I have tried the following code and the iOS simulator gives the following alert: "File Error. Unable to configure network read stream" and the sound does not play.

var SoundButton = Titanium.UI.createButton({
title:'Play me',
height:40,
width:145,
left:10,
top:10
});

SoundButton.addEventListener("touchstart", function()
{
var Sound = Titanium.Media.createAudioPlayer({
url:"Tom.mp3",
preload:true,
});
Sound.start();
});

  • Just a few ideas to fix this problem (as I managed to fix it a few times now).
    1) Make sure you're loading an MP3 file.
    2) Make sure url has .mp3 end (might make a difference - not sure now if this was an issue)
    3) If you're using a cookie to get the file then check there isn't a hidden new line at the end.
    4) Check the path is correct

    In my case this cropped up when the MP3 file wasn't in the correct format.

  • I have posted a solution at http://developer.appcelerator.com/question/133414/how-to-get-timediacreateaudioplayer-to-play-local-files#answer-263837

    This error has nothing to do with the file name or file format. In fact, most media players won't care about the file extension. Give them a file, and they will analyze the file header and move on with it.

    In our case, we used the video player to play the MP3. You can then use TSS to resize and hide the control empty video box. Make sure that it's not small or else you will loose the touchable surface area of the playback controls.

    In general, most video players can also play audio files.

    Have fun.
