How to wrap an existing working iphone app in titanium, or simply call the objective-c methods?

I skimmed through the documentations page and the closest thing I found related to my needs is the Module Developer Guide for iOS (http://wiki.appcelerator.org/display/guides/Module+Developer+Guide+for+iOS)

But it's not clear whether the module should be written in Titanium and consumable in XCode, or the other way around. I already have a proof of concept app written in objective-c, and I wish I could simply call it from javascript in titanium.

The app uses the iphone camera (UIImagePickerController) to get an image, spins of a thread to do some image processing, then calls back with the result. Is this possible to be imported into titanium as a library that I can use?

Thanks in advance

  • Accepted Answer

    When you write a module, you write it in objective C in XCode. You open up your objC methods to titanium through the use of KrollProxy and KrollModule.

    The Module class the template creates for you when you run titanium create in the console is your starting point. You can wrap your objC methods and then call them from Titanium by including your generated module that you build with .&#x2F;build.py and then adding the module to your tiapp.xml file.

    The KrollModule class let's you open up functions to titanium but if you need to return a class to use as an object in Titanium, you can use KrollProxy.
