APP vs API

I am using 3.1.3.GA sdk, Alloys and Android 2.3.4 Emulator. I created an app with login, I have to save cookies after logging in. I need to know in which module we have to save and which is best, 

    Ti.App.properties.setString(''Cookie',cookie);

or

    Ti.API.cookie = cookie;

I am good in API and App properties. As far I know, API is used for logging and with some api level methods. And App properties will persist even mobile switched off and updates too. I thought I have to save it in App properties. And will API variables also persists or what?. Where we have to use API and App. Please share some views on it and explain it. lead me in correct way of appcelian. I need to make login persist even after my app updates from play or app store.

    Definitely use the first. It is perfect for your needs.

    Ti.App.properties.setString(''Cookie',cookie);

    The second method will not persist the value between restarts of the app. What you are doing is adding your own custom property onto a reserved namespace 

    Ti.API

    And while this is possible because of the way that javascript works, its a very bad practice

      — commented October 25th 2013 by Dont Know What to do

    • They are both just namespaces to logically divide up functions.

      Ti.API contains some methods that relate to logging messages to the console and Ti.App contains methods that are specific to maintaining the current App. In the case of Ti.App.Properties it provides you methods to save and retrieve data that is persisted between App restarts. Your first example is precisely this
