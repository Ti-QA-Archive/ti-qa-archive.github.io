Debug App Store submission

Hi,

I have a problem with an App Store submission of an App.
Apple rejected it with the following reason:


We found that your app exhibited one or more bugs, when reviewed on iPad running iOS 8.1 and iPhone 5s running iOS 8.1, on both Wi-Fi and cellular networks, which is not in compliance with the App Store Review Guidelines.

Specifically, we were not able to access the contents of your app.

The steps to reproduce are:

1) Launch app
2) Freezes on the splash page.

Please see the attached screen shot(s) for more information.

The screenshot shows the splash screen of my app.
The problem is I never experienced such an issue with my app, neither in the simulator nor on real devices. Is there a possibility to debug such a behavior?

In my alloy.js i detect the first app start and install the database in this case. Other than that
I just execute a method to remove all files & folders from the iTunes backup mechanism, because
last time my App was rejected because of that.

Alloy.Globals.disableBackupForFolder = function(folderName) {

    function walk(folder) {
        var dir = Ti.Filesystem.getFile(folder);
        var dir_files = dir.getDirectoryListing();
        for (var l = 0; l < dir_files.length; l++) {
            var file = Ti.Filesystem.getFile(folder, dir_files[l]);
            file.setRemoteBackup(false);
            var nativePath = file.nativePath;
            if (nativePath.lastIndexOf('/') == nativePath.length - 1) {
                walk(file.nativePath);
            }
        }
    }
    walk(folderName);
};

The method works fine on all my devices. What else could block the App at the splash screen and how can I remotely find the cause?
