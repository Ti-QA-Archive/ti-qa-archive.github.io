Freeing Memory of dynamically TableViewRow's creation

Greetings

I am trying to my app's free memory, I use a tableView with a variable number of Rows.
When I search for my View children, I can not see the rows, only the tableView.

If I remove the tableView and later set to null; it also removes the rows within the tableView?

I will be grateful if anyone can help me.

Thanks

  • The rows are accesible through the tableviewsection that contains the rows by section.getRows() (There is always a present section even if you dont define it). For a safe removal do:

    tableView.setData([]);
parentName.remove(tableView)
