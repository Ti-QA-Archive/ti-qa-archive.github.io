Images not showing up in Android Emulator

Hi all,

I've been working on an Android App for a while now and it's getting near complete. Just working on some multi-layout multi-firmware debugging at the moment. I have a few actual devices I can test on (luckily) -

  1. my own Samsung Galaxy S (Gingerbread 2.3.3)
  2. My Girlfriend's old Samsung Galaxy Spica (Eclair 2.1 update 1)
  3. My Girlfriend's new HTC Cha Cha (Gingerbread 2.3.3 I think)

I haven't done any testing on the Cha Cha yet but that's last on my list. Currently I have an app working pretty well on the Samsung Galaxy S and for some reason it's crashing the first time I test it on the Galaxy Spica (I'll start a seperate question about that). I've set the Android SDK to version 1.5 to try to accommodate older phones if possible.

Anyway, my main issue here is I want to try to use the emulator do do multi-res testing as it's probably the best way to do it (despite the slowness)

I tested the app in the emulator and noticed two things -

  1. First, the images (buttons) aren't showing up. tabs are fine, custom fonts fine.
  2. Second, the tab group buttons seem grey at the top instead of blue on my Galaxy S.

So my question is this - does anybody know why the imageviews aren't showing up on the emulator ? (and is it running an older firmware

  • I have seen the fastdev server cause issues with updating images. You can turn it off by putting 

     <property name="ti.android.fastdev" type="bool">false</property>

    In tiapp.xml

    Ray
