delete app arguments possible?

Is there a possibility to remove arguments that where injected into the app and can be received with Titanium.App.getArguments()?

My problem is that my app can be opened by other apps using the name of my app as protocol. This way some arguments are injected into my app and it is started in a custom state.
But because in IOS 4 the apps can keep on running I also listen to the "resumed" event to see if another app has injected some arguments while my app was still running in the background. Works great, but this way I cannot tell in the "resumed" callback if the arguments that I get with Titanium.App.getArguments() are newly injected, or whether they are old.
So right now every time I return to the app it starts in the last injected state instead of just starting in a custom state if there where new arguments injected.

Is there something like Titanium.App.deleteArguments() somewhere hidden?

Greetings Lars

  • What I do is keep track of the last value returned by getArguments() and in the resumed event, if it has not changed presume it was not called by a custom URL schema. That can't detect two consecutive launches using the same URL, but for my purposes it gets me close enough.
