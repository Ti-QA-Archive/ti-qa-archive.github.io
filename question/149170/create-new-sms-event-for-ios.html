Create new SMS event for iOS

Hi All,

This is specifically for iOS.
I'm trying to determine if there is a way to create a new SMS from scratch (creating a deliver_sm PDU), and pass it onto the phone asif the SMS came in via GSM, so that the default ringtones and notifications on the phone are triggered. We've successfully done this (outside of Titanium, in a native java app) for Android, by creating the raw PDU and passing it onto the SmsReceiverService intent service.
Is there something similar that can be done using Titanium SDKs? We're not tied to any specific SDK right now, so any version will do, and this should probably work on iOS 5 and up.
Also, I'd like to port our native app to Titanium, so if this can be done using SDKs for Android, it's a bonus, but not required for this question.

Thanks
-Aubrey

  • As far as in-built functionality goes, i'm afraid you're out of luck. The API exposed by Titanium does not include any such features by itself. For Android, if you have already achieved this in native java code then it should be possible to convert to a Ti.Module. For iOS, although I've seen code that allows you to spoof SMS to an iPhone on a jail-broken device, I dont think that its possible otherwise. Also, Apple will almost certainly reject such an App from the Store for attempting this.
