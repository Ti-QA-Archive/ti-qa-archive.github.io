HTML5 canvas performance problems

I'm using a canvas, along with a framework called kineticjs, to create some simple interactive graphs. Basically, my script places draggable horizontal and vertical lines over top of images of graphs, to help users read the graphs. This script works perfectly on the desktop and safari mobile, but the performance is absolutely horrific in the titanium app. My suspicion is that the safari mobile version works well because of nitro, but there has got to be a way to make it work in titanium, right? I see in appcelerator's docs that people have built games with the canvas, so surely I can drag simple lines without lag? Does anyone have any tips? Is there something obvious that I am missing?

  • Hi

    I have no direct experience with your specific issue, but there is an overriding problem all apps have - they are hobbled by Apple.

    Safari uses an engine designed to get the very best performance out of the browsing and javascript world. These benefits are not provided to the "WebView".

    Worth reading the Daring Fireball article on the subject.
