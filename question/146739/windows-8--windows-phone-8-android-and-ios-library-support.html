  • Hi

    Whilst support for windows 8 and windows phone 8 are coming, currently this is not the case. It may be a few months before a public release is available.

    As to which libraries will be available - that is not known until the actual release.

    Creating platform independent code for true cross platform apps is possible even though as you say the windows version are so different in appearance.

    Take a look at the similarities;

    • tables are there
    • windows are there
    • animations and transitions as well
    • buttons, switches, sliders and alerts all there
    • etc

    They may look different and some may have some differences but when you remove the visual aspect you are looking at the same basic view/control. Take a table, it is still the sum of its rows which is the sum of its contents which react to touch events - same for all.

    The underlying logic code - this also remains the same, a function to work out the distance between two points would be the same on both.

    So we are now down to MOSTLY visual differences, we have those with android and iOS, maybe not as starkly obvious but still we already work through those.

    To start getting ready I would suggest you start looking into and using

    • CcmmonJS; specifically placing code in platform specific folders, this is where you create the same functions with the same calling and return parameters but each platform specific version does the best for that platform visually
    • Alloy; this is where things are going for titanium, true MVC so you logic, views and styles are separated, this means you can leave the differences to the visuals only in most cases.

    I hope this answers your questions.
