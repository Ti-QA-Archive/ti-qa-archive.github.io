  • I had the same issue. A new project with no modifications wouldn't compile for Android.
    Turns out it was the ApplicationId that I'd used for the project.
    I initially used 'com.domain.projectname' (which is a fairly standard approach I think?). After I changed it to 'domain.projectname' it worked fine.
    On reflection, it makes sense that my first attempt caused problems in the generated java classes. Unfortunately, it took a little while to realise my mistake.

    — answered October 7th 2011 by Duncan Pratt

    • Actually, the issue was not caused by the inclusion of 'com.' It was because the domain begins with a number (eg com.4dx…). Changing to use com.fourdx… worked.
