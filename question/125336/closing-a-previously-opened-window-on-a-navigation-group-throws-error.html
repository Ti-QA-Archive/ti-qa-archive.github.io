Closing a Previously Opened Window on a Navigation Group Throws Error

I am using a navigationGroup in the detailView of a splitView to open new windows when an item is selected from the masterView. I only want 1 window in the navigationGroup stack at a time. That is, even though I'm using a navGroup, there is no need for navigation, and I don't want the back button in the navBar. So, I am closing the previously opened window prior to opening a new window.

I have the following code:

var myNewDetailWindow = Ti.UI.createWindow({
    // properties here
});

if (detailNavigationGroup._detailWindow) {
    detailNavigationGroup.close(detailNavigationGroup._detailWindow);
}

detailNavigationGroup.open(myNewDetailWindow, {animated: false});
detailNavigationGroup._detailWindow = myNewDetailWindow;

After about 2-3 windows are opened and closed, the app crashes with the following errors:

nested push animation can result in corrupted navigation bar
Finishing up a navigation transition in an unexpected state. Navigation Bar subview tree might get corrupted.

Am I doing something wrong or this is a bug?

— asked September 8th 2011 by Slim McKinsley

  • After further testing, I think the crash is being caused by the WebLock error, as reported here:
    http://developer.appcelerator.com/question/123881/odd-crash-with-webview

    myNewDetailWindow contains a WebView. If I remove the WebView, I do not get the error. So, seems like a fix for the WebLock error will resolve this.
