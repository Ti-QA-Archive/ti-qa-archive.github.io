Android Window Open/Close Flashing

All,

With the following configuration:
Titanium Studio, build: 1.0.7.201112281340
Titanium SDK 1.8.0.1
Android Runtime V8
Google APIs Android 4.0.3

I am getting flashing whenever I switch windows. I am using the following call to open a window

btnMap.addEventListener('click', function()
    {
        app.loadPage(Titanium.Platform.osname + '/CustomerMap');
        win.close();
    });

and the following to close a window

    btnMainMenu.addEventListener('click', function()
    {
        win.remove(mapview);
        mapview = null;
        app.loadPage(Titanium.Platform.osname + '/MainMenu');
        win.close();
    });

I have put debug code on the open and close events for the windows on both pages, and they appear to fire correctly. The behavior is odd though. What actually happens is the screens flash once for each time they have been accessed. When I first go to the map screen from the main menu, it flashes once. When I return to the main menu, the menu flashes once. When I return to the map page again, it flashes twice. When I return to the main menu again, it too flashes twice. The pattern is consistent with however many opens and closes I perform, and the whole time, the open and close debug messages only show one open/close combination.

Any suggestions? It appears am doing this correctly, but as my wife always says, I have been wrong before…

Any and all help greatly appreciated,
Jax
