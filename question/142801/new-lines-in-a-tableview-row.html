  • Here is something that I did. type and data are custom attributes that I added to my table row so that I can get the alertDialog to show up properly. Yes it's repeating the data in the row, but I was too lazy to split the string.

    jsonAddressDataArr is a variable that holds JSON data that I got from my company's www site.

    The only snag I'm running into is that my boss would like for this to only fire if the row data is too long for the window. Ideas?

    // an example header an tableViewRow
var sectionPropertyData = Ti.UI.createTableViewSection({ headerTitle: 'Property Details'});
sectionPropertyData.add(Ti.UI.createTableViewRow({ title: 'Bedrooms: ' + jsonAddressDataArr.bed, type: 'Bedrooms', data: jsonAddressDataArr.bed}));

// Here is the Event Listener for the table
tableJSONAddress.addEventListener('click', function(e){
        // On the smaller screen of the iPhone or Android
        // you may not be able to see the entire row of
        // of the table.  This spits it out to a Message
        var PLATFORM = Ti.Platform.name;
        var type;
        var data;
        if (PLATFORM === 'android'){
            type = e.row.type;
            data = e.row.data;
        }
        else{
            type = e.rowData.type;
            data = e.rowData.data;
        }
        var descMessage = Ti.UI.createAlertDialog({
            title:type,
            message:data
        }).show();
    });
