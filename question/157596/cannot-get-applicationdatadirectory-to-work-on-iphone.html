Cannot get applicationDataDirectory to work on iPhone

Hi. I'm writing an app targeting iPhone, iOS 7, SDK 7, Titanium Studio build 3.1.3

One of the things I need to do is store a file locally.

I try to do so using this:

var savedFile = Titanium.Filesystem.getFile(Titanium.Filesystem.applicationDataDirectory, '/testfile');

It seems that the path is all wrong; with the doubling of the path and all that.

    • I found something really weird now.

      So, apparently, my problem is a bug that happened with iOS 7, where iOS treats the filesystem a bit differently. Apparently, file system operations should no longer search for "file:///localhost/" and instead search directly for "file://". However, this was apparently fixed in Titanium 3 months ago!

      If I check the FilesystemModule.m file in my Titanium SDK I have the fixed version where the reference is to "file://", but when I check in the build of my app, the FilesystemModule.m is the old version with "file://localhost/". This persists even if I delete the file and manually rebuild the project.

      Obviously, my Titanium Studio must get the FilesystemModule.m from somewhere other than the SDK but I just can't find another copy of the file on my computer.

  • Solved it!

    Apparently, my project was set to use an old version of the SDK, rather than the new one where this iOS 7 bug was present.

    However, here I found another bug. Even when I changed the project properties to use the latest SDK version it would still use the old. This persisted through rebuilds, and I even manually deleted the "build" folder from the project, but this didn't help.

    Finally, I solved it through simply deleting the old SDK versions from the SDK folder (~&#x2F;Library&#x2F;Application Support&#x2F;Titanium&#x2F;mobilesdk) and then again reforcing build in Titanium Studio. Finally, the new SDK was used and the file system works as it should.
