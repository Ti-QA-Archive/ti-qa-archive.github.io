Google Maps v3 on Android 4.4 = just blank

Could anyone let me know how to get the most basic example of Google Maps to work on an Android 4.4 device (in a WebView). The HTML-example is here:
Simple Map

app.js:

var win = Ti.UI.createWindow();
var webview = Ti.UI.createWebView({
    url: 'data.html',
});
win.add(webview);
win.open();

where data.html = copy/paste the html from the google-link.

This example used to work just fine on all Android versions prior to 4.4. I tried to update to all latest SDKs. I previously was on Titanium 3.1.2, Android SDK 14. Now compiled with 3.2.1GA / Android SDK 19.

The only info I get from the Titanium-console window is "Async pixel transfers not supported", not sure if this is related.

  • Hello Roel,

    thanks for pointing this out. I did tested this and I was totally able to reproduce the issue.

    Test Case

    index.html

    <!DOCTYPE html>
<html>
  <head>
    <title>Simple Map</title>
    <meta name="viewport" content="initial-scale=1.0, user-scalable=no">
    <meta charset="utf-8">
    <style>
      html, body, #map-canvas {
        height: 100%;
        margin: 0px;
        padding: 0px
      }
    </style>
    <script src="https://maps.googleapis.com/maps/api/js?v=3.exp&sensor=false"></script>
    <script>
var map;
function initialize() {
  var mapOptions = {
    zoom: 8,
    center: new google.maps.LatLng(-34.397, 150.644)
  };
  map = new google.maps.Map(document.getElementById('map-canvas'),
      mapOptions);
}

google.maps.event.addDomListener(window, 'load', initialize);

    </script>
  </head>
  <body>
    <div id="map-canvas"></div>
  </body>
</html>

    app.js

    var win = Ti.UI.createWindow();
var webview = Ti.UI.createWebView({
    url:'index.html',
});
win.add(webview);
win.open();
