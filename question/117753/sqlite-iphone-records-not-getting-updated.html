  • Please see the replies in this thread for the full explanation. The short answer is that the Resources folder is read-only on a device. Thus when you use the install() method it copies the resource db to a location which can be updated. When you restart your app later, if the install() method sees the updatable version of the database already exists, it does not recopy the version from the resources folder. My reply in the above thread explains the options for dealing with that, depending on your intentions.
