Child views on Buttons not rendered

I spiced up the ordinary Button with a Label as a child view. This way I have more control over how the label renders (i.e. a shadow for instance) as compared with setting the text directly on the button.

This all works fine except that I started noting that the label isn't always being rendered - resulting in empty buttons. Typically this happens if the button is not visible when the button is created (for instance further down on a View which is seen through a scrollable view … or on a view which is being scrolled in from the side).

If I change the Button to an ordinary View it all works as expected - the label is rendered perfectly even though it happens off-screen so to speak. But changing to a View from a Button makes me lose functionality I also need, those typical for a Button … so that is something I would like to avoid.

Why is there a difference for Button and View on how (and when) children are being rendered?
I guess this is a bug?

Anyone with similar experiences which has a suggested workaround?

I am very thankful for any insights into this!

    Hi Tobias

    As you have not supplied any example code (please do in future) I am going to guess that you have created a button and added a label directly to that. Something like this;

    var button = Ti.UI.createButton({
    title: ''
});
var lbl = Ti.UI.createtLabel({
    text: 'Press Me'
});
button.add(lbl);

    If this is the case, this is where the problem lies. Whilst most controls are subsets of views, not all controls are container views. So whilst something may work enough to make you think they should work, others do not so they confuse or annoy.

    Instead use a real view as the container for both;

    var view = Ti.UI.createView({
    height: Ti.UI.SIZE,
    width: Ti.UI.SIZE
});
var button = Ti.UI.createButton({
    title: ''
});
view.add(button);
var lbl = Ti.UI.createtLabel({
    text: 'Press Me'
});
view.add(lbl);

    As both the label and button are not in a container view, this is now set up so the Titanium can process things as it expects.
