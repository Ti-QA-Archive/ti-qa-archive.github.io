Permission Denied - TFS files not checked out

I'm attempting to use Eclipse with the Team Explorer Everywhere plugin for our TFS server. I'm running into a problem when I try to build:

IOError: [Errno 13] Permission denied: u'/Users/seth/Documents/workspace/MyApp/build/iphone/build/Debug-iphonesimulator/MyApp.app/Default.png'

All of my source files that I haven't checked out are marked as read-only. After getting the project from TFS the first time, the build is successful. I suspect this is because the build directory is empty, and on the first build it is only writing files to the build directory. All subsequent builds fail with the above error message. All of the resources not checked out are read-only and cannot be overwritten.

Is there any known workaround for this? I can't check out the entire project just to be able to build. Right now the only workaround I have is either A) delete the contents of the build folder before every build, or B) Force read-write access on the entire build folder prior to every build.

Either option is cumbersome. I would appreciate any suggestions!

Possible solution: When the build script copies files from the Resources directory, it should change the file permissions to writable?

UPDATE: My current workaround is an AppleScript that deletes the content of the build/iphone folder. Before every build, I run this AppleScript. Running the script isn't time consuming - performing a complete rebuild every time is, though.
