What's the persistence of user data in an iPhone app?

I have a nav group in which each screen has a table view where the use makes selections. When the next button is clicked, I save the selections as an array in user data.

As each screen loads, I check the user data array for that step to see if selections have been made and display prior selections.

It works fine for a couple of forward/back uses, but after about 3 times when I go forward the screens saved user data is blank. Why would the user data have been cleared?

  • Are you talking about Ti.App.Properties?
    If yes, you are doing a mistake somewhere because Properties are "forever" (as long the app is not deleted from the device or the info is not overwriten/deleted by you :) )

    • In the next button handler I save the selected items: 

      nextBtn.addEventListener('click', function(e) {
    chosenItems = [];
    for (i=0; i<tv1.data[0].rows.length;i++) {
        if ( tv1.data[0].rows[i].isChosen ) {
            chosenItems.push(tv1.data[0].rows[i].clickName);
        }
    }
    setArr('step1chosen', chosenItems);
    currWindow.navGroup.open(step2win);
});

      Where the setArr function maps to 

      function setArr(name, value) {
    Titanium.App.Properties.setList(name, value);
}

      And then I load it with 

      function getArr(name, defaultvalue) {
    return Titanium.App.Properties.getList(name);
}

      As I said, this is working twice, but the third time through the array is empty.
