  • The red warning there can be ignored. The android team just needs to compile the SDK against some updated libraries. (which, if it bugs you, you could do too)

    Does the emulator launch?
    Does adb ever find the device?

    [DEBUG] Waiting for device to be ready …

    [TRACE] adb devices returned 1 devices/emulators

    Does the apk get copied to the device?

    [DEBUG] /android-sdk-mac_86/platform-tools/adb -e install -r "/path/to/yourapp.apk"

    Does adb send the start command?

    [DEBUG] /android-sdk-mac_86/platform-tools/adb -e shell am start -a android.intent.action.MAIN -c android.intent.category.LAUNCHER -n com.yourname.yourapp/.yourappActivity

  • Andrew,

    If you are using the Kitchen Sink demo for the 1.7.0 release, you may need to update your Android SDKs. Once updated, open your AVD Manager and delete any devices in the list and relaunch the KS demo.

    If you are using the KS demo from a previous release, then you may need to build using the mobile SDK that matches that version of KS.
