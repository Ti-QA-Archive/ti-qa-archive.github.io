  • Strange as it may seem, iOS does not support the ability for an application to even close itself, let alone force a restart of the application. In Apple's infinite wisdom, iOS application are not allowed to have a close button (unlike OSX applications) – it is considered inappropriate (in Apple's view) and simply say the user must close the app with the home button or gesture (i.e. 4 or 5 finger pinch).

    The closest you can do is set a plist property that tell iOS you don't want to allow the application to suspend but rather to exit when the home button is pressed. This of course has the (IMHO negative) side effect of not being able to multi-task to another app then return to where you were. However if you want the user to exit on on the home button then see this thread.
