MobileWeb: Define root window size.

Is there a way to define the overall width and height used in mobile web builds?

I'm developing an iPhone app for a study project and would like to provide my fellow students, which aren't software developers but provide icons/pictures/etc. and general comments about the design, with the possibility to quickly take a look at the current state of development by simply pointing them to a web site they can open on their desktop computer.

In other words: I would like the mobile web build to always use a width of 320 pixels and height of 480 pixels regardless of the browser's window size.

I didn't find anything about this in the documentation. Is it possible at all?

Thanks for any replies,
Robert.
