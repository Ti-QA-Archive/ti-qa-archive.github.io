Tiapp.xml, a module and 'require' in a project. Ti3.0, Android 4.2, Win7

I've created a module and modified the tiapp.xml file in the project I want to have access to the module functionality. In Tiapp.xml I have added a line of the form "<modules><module version="1.01" platform="android" (app name) </modules>. However, I can't seem to find the correct form of the app name. One document gives "ti.(app name)" as the correct form without explaining what the "ti" means. Another document uses the app id, e.g., "org.appcelerator.appname". To access module functionality both documents say to add "var modulefile = require('ti.app name")" or var modulename = require('org.appcelerator.app name) respectively. The error I receive hen running the project is "ti.app name" or "org.appcelerator.app name" NOT FOUND. Any help would be appreciated.

  • Its the name used by the module developer, and ends up being the same as the folder name in the appropriate sub folder of the modules folder.

      If no version is listed, then the newest version available is used. An example of this is that version 1.3 of the quicktimegame2d module has problems, so we force the loading of version 1.2 even though 1.3 is installed.
