    Hi

    Short answer - yes.

    You have the ability to create an entry into the settings section of iOS, this is done by creating a series of file that talk to each other that are hybrid xml/array files.

    You achieve this by creating these files in a specific folder location;

    {appname}/platform/iphone/Settings.Bundle/

    Then creating a main file Root.plist. You MUST maintain the specific upper/lower case of the file name and folder names for this to work.

    This is a very basic example Root.plist file;

    <?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
    <key>StringsTable</key>
    <string>Root</string>
    <key>PreferenceSpecifiers</key>
    <array>
        <dict>
            <key>Type</key>
            <string>PSGroupSpecifier</string>
            <key>Title</key>
            <string>Test</string>
        </dict>
        <dict>
            <key>Type</key>
            <string>PSTitleValueSpecifier</string>
            <key>Key</key>
            <string>Version</string>
            <key>Title</key>
            <string>Version</string>
            <key>DefaultValue</key>
            <string>1.0</string>
        </dict>
</dict>
</plist>

    Save this in the folder mentioned above, do a project > clean and then re-run your app, it will show in the simulator and on the device so it does not matter how you are testing. Once your app is running, simply exit the app and go to the correct section in the iOS Settings app. Under the name of your app you will see a single page with a group name Test and an entry Version.

    You can extend this to show groups, items, child pages, text boxes, switches, footer text and more.

    When you start playing with this - start slowing as it can be confusing what is going on until you get the hang of it. The two example entries above have the very same first two lines, but the second line of each denotes what type of information is about to be displayed, and then additional type specific entries are shown underneath to finish the information.
