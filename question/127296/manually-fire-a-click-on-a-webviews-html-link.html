Manually fire a Click on a WebView's html link.

Hey guys,

I have Scroll View with Web Views in it that link to YouTube videos. To be able to scroll the Scroll View without scrolling the Web View, I did this:

webViewVideos.addEventListener('touchmoved', function(e) {

    e.preventDefault();
}

This issues me the following WARNING when I scroll the Scroll View:

Result of expression &#39;e.preventDefault&#39; [undefined] is not a function.;

And works. As I am no longer able to scroll the Web View anymore. The problem is that this also removes the click event from my Web View and I can't click the YouTube play button.

I was thinking about placing an invisible view in front of the WebView, to avoid using e.preventDefault() and adding an eventListener to this view to manually fire a click to play the YouTube video in the Web View.

I know this is not the best option, but I'm short on time and it's the only thing I could think of right now.

Any ideas how to make this work? Or another workaround to my issue?

Thanks!

  • I'm afraid, that is not a warning, that is JS exception. In Titanum event system you can't prevent default action. "preventDefault" is DOM only, you can use it in web view, but not outside web view. I think that's why your "click" event is not working …

    • No, you need use app-level events to trigger something in web view. So, capture overlay view's click event and then, using app-level event, trigger something in web view. Also, note that { e } is not valid JS object, you can do this:

      myBackgroundWebView.fireEvent('click', e);

      since "e" is object in your case.

      Anyway, you'll need to track where did overlay window got clicked (see "e.x" and "e.y" properties) and according to that, you'll need to fire particular event in web view. If you can build same layout in overlay view as it is in web view, you can map particular areas in web view to corresponding areas in overlay view (so, overlay view would consist of more views that constructs layout similar to the one in webview). I don't know will you understand what I mean. :)

      Anyway, it's a bit of a work, but I can't think of any other solution. Maybe you should wait for some time, maybe someone will come with better solution.
