    Hi

    You have not actually said WHAT is wrong with the widths. The image shows four equally spaced tabs inside a tabGroup that occupies the bottom portion of the display. This is normal.

    What do you think it is doing wrong?

    A tabgroup will take up the entire width of the device in either orientation. It can have to one or more tabs added and depending on the available space an automatic more tab will appear to allow the user to access additional secondary tabs that could not be shown. An iPhone will usually show a maximum of 5 tabs. More than that will cause the more tab to appear in the fifth space, the first four appearing from the left, additional tabs appear in the automatic tab overflow UI when the more button is pressed.

    Each button spaces are worked out this way;

    • 1 button = 100%
    • 2 buttons = 50% each
    • 3 buttons = 33% each
    • 4 buttons = 25% each
    • 5 buttons = 20% each
