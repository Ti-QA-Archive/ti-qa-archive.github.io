  • Hi

    This works for me, using your own file information - this should get you moving.

    var activeMovie = Ti.Media.createVideoPlayer({
    url: 'http://fms.rrz.uni-hamburg.de:1935/vod/_definst_/mp4:8l2gelbmin/GXGrv5N0BAleqC9b8J3OtQxx.mp4/playlist.m3u8',
    backgroundColor: '#000',
    mediaControlStyle: Ti.Media.VIDEO_CONTROL_DEFAULT,
    scalingMode: Ti.Media.VIDEO_SCALING_ASPECT_FILL,
    fullscreen: true,
    autoplay: true
});

    The internals of the returning text file simply points to different quality versions if available. This is because the protocol allows mid-stream increases and decreases in bandwidth to be met with increases and decreases in quality on the fly.

    iOS support HLS on the devices natively so all normal video interactions work including fullscreen, rotations and also AirPlay.
