  • Unlike the iOS simulator, you do NOT have to shut down the simulator to try a new compile build. The slowest part is the initial android simulator load. Once loaded, leave it there instead of hitting Stop.

    Just make your code changes and hit Launch, and it will replace the existing version with the new build and start it. Still slower than for iOS, but MUCH more tolerable the a full restart of the simulator each time.

  • Hen

    JS validation is highly recommended when working with Titanium, and as an android developer it will provide your biggest time-saver. To enable Javascript validation for eclipse, you need to install the wtp plugin (Web Tools Platform), or use Aptana that has it built-in.

    The time it takes to boot an Android emulator or launch an app to an emulator is almost entirely dependent on Google's SDK. However, with this in mind and to give you an example for comparison, on my Ubuntu (i7 processor, 6GB memory) machine it takes 40 seconds to launch an application to an already-booted emulator, as this video video demonstrates.
