Serious memory issue with remotely loaded images for iOS

I read some post about memory issues related with UIImageViews. I think I'm facing the same problem, in the current app I work on I load remotely images in a tableView as thumbnails (imageViews) the tableView is placed in a Window and pushed in a NavigationGroup. Since it are not a few images and the quality is guaranteed for retina I see the memory decreasing at least 3 to 5 MB for each pushed window. Whenever I close (backbutton) a window, so removing it from the NavigationGroup, the memory is not released… does anyone has a solution or workaround (I tried even in objective-c ImageLoader).
