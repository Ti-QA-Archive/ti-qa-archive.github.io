Android targetSdkVersion="11" causes crash on 3.0 beta

Titanium SDK: 3.0.0.v20121109144656
Platform & version: Google APIs Android 4.1.2 [armeabi-v7a]
Device: Android emulator
Host Operating System: OS X 10.8.2
Titanium Studio: build: 3.0.1.201211131826
Test case:
File -> New -> Titanium project -> Choose tabbed application.
Edit tiapp.xml

  • Select sdk version: 3.0.0.v20121109144656
  • Change android settings to:
    <android xmlns:android="http://schemas.android.com/apk/res/android">
  <tool-api-level>11</tool-api-level>
  <manifest>
      <uses-sdk android:targetSdkVersion="11"/>
      <!-- other manifest entries -->
  </manifest>
</android>
  • Run in Android emulator. The Android emulator will display an alert with text: "Unfortunately, yourappnamehere has stopped".

  • This combination does not work for me either (crashes the app with the same error message as yours):

    1. android:targetSdkVersion > 9
    2. Titanium SDK 3.0.0 or 3.0.2
    3. Android emulator 4.2.2
    4. ti.android.fastdev true

    Disabling fastdev, using an older Android emulator (I use 4.0.3), switching to an older Titanium SDK, or lowering the target SDK version makes the app running again. So I guess your best choice is to switch to an Android 4.0.3 device. Sigh.
