Location of Strings.xml in Android Application

I am looking for the strings.xml file in an Android application so that I can loop through and parse the name values rather than using L() or local.getString(), both of which work correctly. Appcelerator documentation states that this file is located in the resources folder, however, when I try to retrieve this file, I get an "java.io.FileNotFoundException: Resources/i18n/en.xml".

I am trying to get the strings.xml file with:

var locale = 'en';
var f = Ti.Filesystem.getFile(Ti.Filesystem.resourcesDirectory, 'i18n/' + locale + '.xml');

Doing the following for iOS retrieves the file (albeit in a different format):

var locale = 'en';
var f = Ti.Filesystem.getFile(Ti.Filesystem.resourcesDirectory, locale + '.lproj/Localizable.strings');

If I list the contents of the resources folder by doing:

var resourcesDir = Titanium.Filesystem.getResourcesDirectory();
var dir = Titanium.Filesystem.getFile(resourcesDir);
var dir_files = dir.getDirectoryListing();

for (var i=1;i<dir_files.length;i++){ 
                Ti.API.log('file CM: '+ dir_files[i].toString() );
            }

The returned list does not list a i18n folder or a strings.xml file. The strings.xml file is in the correct directory (i18n -> en -> strings.xml) and works for iOS.

So, does anyone know where I can find the strings.xml file in Android so that I can parse it for use outside of the typical localization functionality?

      var dir = Titanium.Filesystem.getFile("/platform/android/res/values-en/string.xml");
