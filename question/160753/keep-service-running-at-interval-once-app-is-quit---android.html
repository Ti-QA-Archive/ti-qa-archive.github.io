Keep service running at interval once app is quit - Android

What I have been trying to Google but have been coming up with useless results is how to keep a service running at an interval even after the application is quit on android. At the moment I have a service that runs perfectly fine and sends the notification when I want it too. The problem now is that when I exit the application and destroy it completely the service does not carry on.

Since I have come from a native android background my instincts would be to use the Android Alarm Manager but I do not know how to implement this on Titanium using the API system. I would also prefer the implementation to be without the use of any 3rd party libraries. Any bright cookie out there kind enough to lend a hand?

In general about the service I want the service to send periodic notifications so if there is a different work around that would be great!
