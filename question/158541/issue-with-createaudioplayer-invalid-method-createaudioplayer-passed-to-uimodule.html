Issue with createAudioPlayer, invalid method (createAudioPlayer) passed to UIModule

There seems to be an issue with the alloy <AudioPlayer id="player"/> tag, it resolves to

$.views.player = Ti.UI.createAudioPlayer({
id: "player"
});
$.views.employeeFlow.add($.__views.player);

which in turn throws the invalid method (createAudioPlayer) passed to UIModule error on both simulator and device running ios7.

    You need to give the appropriate namespace.

    <AudioPlayer ns="Ti.Media" id="player"/>
