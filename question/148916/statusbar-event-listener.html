StatusBar event listener?

Hi!

On iPhone, I'd like to be able to detect tap in the top bar and do some custom action form it.

The docs seem incomplete on it or maybe I am taking the wrong approach here?

    Hi

    The documentation is not incomplete on it I am afraid - there simply are no events that are provided to listen out for.

    Whilst iOS does support the "click on status bar" to instantly scroll any scrollView or tableView to the top of the screen, the action is built-in to the operating system and not an exposed event. For those two though you can turn off that feature;

    var tbl = Ti.UI.createTableView({
    data: data,
    height: Ti.UI.FILL,
    scrollsToTop: false, // default is true
    width: Ti.UI.FILL,
});

    As the docs point out - there are no events. This is because the OS handles those things - particularly; the notification panel sliding.

    • Here goes, think outside the box. There is a back-story, feel it deserves telling.

      What we need to do is "detect" the statusbar click event, but that does not mean we have to actually hook into the event. Maybe there is a way to filter something that we can use to be the event itself.

      Could we use an event that occurs in ONE circumstance elsewhere and then filter it so we only respond to something that we can use for our needs.

      The answer actually came because of the code snippet I gave earlier. So boring story done.

      CommonJS module statustouch.js place in resource root (you can relocate later)

      // do not worry about ANY of this code you do not need to edit
// it or ever care about it again
var scrollView;
function create(o) {
    var obj = o || {};
    scrollView = Ti.UI.createScrollView({
        backgroundColor: 'red',
        contentHeight: 1000,
        contentWidth: 500,
        scrollsToTop: true,
        showVerticalScrollIndicator: true,
        showHorizontalScrollIndicator: true,
        height: Ti.UI.FILL,
        left: 100,
        visible: false,
        width: Ti.UI.FILL
    });
    win1.add(scrollView);
    scrollView.add(Ti.UI.createLabel({ top: 0, color: '#fff', height: Ti.UI.SIZE, text: 'Top', width: Ti.UI.SIZE }));
    scrollView.add(Ti.UI.createLabel({ bottom: 0, color: '#fff', height: Ti.UI.SIZE, text: 'Bottom', textAlign: 'center', width: Ti.UI.SIZE }));
    scrollView.addEventListener('scroll', function (e) {
        if (e.y === 0) {
            if (obj.action) {
                obj.action({
                    type: 'statusbar'
                });
            }            
            scrollView.setContentOffset({ x: 0, y: 500 }, false );
        }
    });
    setTimeout(function () {
        scrollView.setContentOffset({ x: 0, y: 500 }, false );    
    }, obj.delay || 1000);
    return scrollView;
}
exports.create = create;

      Usage 

      var status = require('/statustouch').create({
    //timeout: 1000, // optional
    action: function (e) {
        Ti.API.info(JSON.stringify(e));
    }
});
// MUST be added to the window you wish to detect the event on
win.add(status);

      You add your response code to the action callback, I have added a debug console line to show you it working.

      There is one caveat you NEED to pay attention to. NO OTHER scrollableView or tableView can use the property scrollsToTop with true, it MUST be set to FALSE.

      Example;

      var tbl = Ti.UI.createTableView({
    data: data,
    height: Ti.UI.FILL,
    scrollsToTop: false, // MUST be set
    width: Ti.UI.FILL
});
win.add(tbl);

      This STOPS the statusbar normal operation so this table will NOT then move the view/table back to the top. But as always there is a way of putting it back, you simply call this inside the event I have created for you;

      tbl.scrollToIndex(0);
