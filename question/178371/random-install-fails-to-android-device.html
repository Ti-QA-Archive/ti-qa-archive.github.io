random install fails to android device

mac mavericks, studio: 3.2.3.2, xcode 5.1.1 ios 7.1sdk

i was having problems trying to install an app to a sony experia j (4.1.2) - so i created a new basic default project and didnt change anything - ran it and it installed on the phone fine. (the 2 window one)

i ended the app running, cleaned the project, ran it and it failed to install (standard "failed to install apk on "kjhkjhk")

so i uninstalled the app, cleaned the project and ran it again - again it failed to install -

cleaned and tried again and i got a 143 error

cleaned and tried again - standard failed to install notice

no code was touched inbetween

is there some sort of knack/trick of doing this ? ;)

i had been using the emulator fine on the pc and uploading the apk to my site and used the browser to install and thats fine but the simulator is sooooo slow i've borrowed a friends android to speed up the development and doing it on my mac laptop but its taking longer trying to install direct because of random fails than doing it via pc emulator build which is a bit ridiculous (installing on the mac emulator always fails)

eg, does anyone use a 3rd party file manager type app for either the android phone or for the mac to transfer the apk ?

  • after much faffing around with file managers etc i realised that the storage space was near the limit so i changed the tiapp.xml to this

        <android xmlns:android="http://schemas.android.com/apk/res/android">
        <manifest android:installLocation="preferExternal">
            <uses-sdk android:minSdkVersion="16"/>
        </manifest>
    </android>

    and it installs everytime direct from studio :)
