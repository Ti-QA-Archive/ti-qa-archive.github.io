  • I had a similar problem, By the way Im using build tools 22 without any stress.
    My issue turns out to be a problem with a malformated splash screen resource. But all the feedback i was getting was misleading.

    I ran the packing command on command line in order to get some feedback on what was happening and at first sight all i saw was 20 messages stating that resources some did not match. Something related with the themes I though.

    After scrolling all way back up i found out that the nodpi splash screen had a malformated error. I've deleted that image and i was able to create the package again.

    No idea why but this only start happening after i've been working for sometime and successfully pack the app. Maybe because i clean the solution who knows…
