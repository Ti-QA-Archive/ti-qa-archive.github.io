Hi guys, i have this simple code related to sqlite database. Firstly, i have 'id' column as primary key with auto-increment feature. it's simply adding up every time we add a new record to the db, but the question is, the last 'id' stays and the next new record will continue from the last one and so on.

for example:

brandID    brandName
---------------------------------
1        Top King
2        Twins Special

even if i have deleted brandID='2' record, the next addition will start from brandID='3' and not replacing the one that just removed

    No, autoincrement keys will not 'fill in the gaps' they will always return a higher number than the last id. To do this you would have to manage it manually yourself. This is a terrible idea though, as the of your table increases then the time it takes to work out the first lowest unused id will take longer and longer… Is there any real reason why your ids' must be consecutive ?
