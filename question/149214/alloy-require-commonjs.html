Alloy: require commonJS

I started a new project with alloy, (the Resources folder is hidden from the App and Project Explorer.)
now i added a js file to project with path: app/models/person.js.

In person.js i have:

var models = (function() {
    alert('hi joli');
});
exports.Model = models;

then in the index.js file i try to require the file:

var joli = require('/joli').connect('myDB'); // opening up database
var Models = require('models/models').Models; // export from database file

$.index.open();

i went through (CommonJS Modules in Titanium) and tried to specify the path in many ways ("/models", "../models", "models/models" etc…) but nothing works!!!!

i get these errors:

  • Script Error = Couldn't find module: models/models at app.js (line 0).
  • NSCFString out of bounds error

what is the correct way of requiring a file? need help please!!

  • Alloy model files are special files that get processed at compile time. They are not accessible as-is, but are instead part of a composite model file that gets created. There's more details on that format here: http://docs.appcelerator.com/titanium/latest/#!/guide/Alloy_Collection_and_Model_Objects
