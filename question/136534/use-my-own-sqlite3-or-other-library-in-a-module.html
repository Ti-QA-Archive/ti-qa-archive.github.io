Use my own sqlite3 or other library in a module

I wish to use my own sqlite3 library in a module with some of my own extension function and compilation options.

Env:
Titanium Studio 2.0.1
SDK 2.0.1 GA2
Building for iOS on OSX 10.6.6

I created a new Appcelerator module in Studio.

In the …Module.m file I added:


#include "sqlite3.h"

...

-(NSString *)getSqliteVersion:(id)args
{
    return [NSString stringWithFormat:@"%s", sqlite3_libversion()];
}

-(NSString *)getSqliteVersionHdr:(id)args
{
    return @SQLITE_VERSION;
}

I have added the sqlite.c to the xcode project and I can see it being complied in the build process.

In the test app.js file I have:

var versions = '\nSqlite version: ' + sqlitesecure_ios.getSqliteVersion() + ' [' + sqlitesecure_ios.getSqliteVersionHdr() + ']'
    + '\nSqlite Src Id: ' + sqlitesecure_ios.getSqliteSourceId();

Ti.API.info(versions);

When the test app is run the versions are different. I.e. I am using 3.7.10, but the library version being reported is 3.7.7.

So I assume somewhere the system library is being picked up. I have checked the build strings and there is not a -lsqlite type arg and the framework is NOT included in the xcode project. So I have to assume Appcelerator is including the sqlite lib somewhere in the final build.

How do I get around this??

I assume it must be possible as there is a module in marketplace that provides encryption via SQLChiper and so they must be rolling their own sqlite build.
