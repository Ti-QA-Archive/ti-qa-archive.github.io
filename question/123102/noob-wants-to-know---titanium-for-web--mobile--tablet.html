NOOB Wants To Know - Titanium for Web + Mobile + Tablet?

Guys,

I'm a product manager, NOT a developer or architect, so I'm a bit unclear about how to think about the value of Titanium as a tool for building a new cross platform mobile/web product we have mind. Our product plans include the following:

We want to provide highly configurable wizard driven survey forms on both Mobile (iPhone/Android), Tablet (iPad/Honeycomb) and browser that will be used by a subject matter expert doing facilitated observations. These guided surveys will be configurable and will allow data capture to feed to a cloud database. From this database we drive analytics out to both the browser, mobile and tablet with value added alerts, etc.

So in the scenario above, how should I position Titanium as a tool for helping my team build this product? Mobile/Tablet only + a Rails/Java browser app feeding a cloud data store? Or is it possible to deploy Titanium to an HTML5 browser like Chrome as well?

Thanks!

Brad

  • I'd usually suggest the HTML5 route, however our HTML5 mobile web sdk is currently in beta. Any paid subscription has access to this beta, though while in beta, we usually advise against using the sdk in production. When this is officially released however, you will be able to build and deploy for mobile/tablet and mobile web with minimal code.

    Personally, in projects like the one you describe, I will always recommend building the application around a webservice so that you can ultimately deploy to any medium.
