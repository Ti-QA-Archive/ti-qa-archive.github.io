HTTP POST request being sent as OPTIONS request

In MobileWeb/Browser with Ti 3.1.3.GA, this request is being performed as an "OPTIONS" request. How can I force it to be a "POST"?


var xhr = Ti.Network.createHTTPClient();

xhr.onload = function(e) {
    alert(xhr.status);
};
xhr.onerror = function(e) {
    Ti.API.error(e);
};

var url = 'http://beigerecords.com/joe/api/system/connect';
Ti.API.debug("POSTing to url "+url);

xhr.open("POST", url);
xhr.send({test:'true'});

To see the expected result of the POST, use this in a terminal:

curl http://beigerecords.com/joe/api/system/connect -d ""

    This is called CORS. This is expected behavior in most cases. The first request checks to see if the web browser is permitted to make a request/send information across domains. If the first connection is successful, then the actual POST request is sent.

    The server hosting the web app must have its security configuration enabled to acknowledge and response to the OPTIONS request.

    http://en.wikipedia.org/wiki/Cross-origin_resource_sharing
