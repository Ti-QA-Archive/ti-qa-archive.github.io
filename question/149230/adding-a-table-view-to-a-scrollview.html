Adding a table View to a scrollView

Is it good to add a TableView to a scroll View?
.if Yes what is the advantage and is it something that by using it ,i am going to have any memory issues.
If no what are the Disadvantages.

vat table = Ti.UI.createTableView({
backgroundColor:black
});

var win = Ti.YI.createWindow();

var demoScrollView = Ti.UI.createScrollView();

win.add(demoScrollView );
demoScrollView .add(table);

    Do not put a tableview, which is an enhanced scrollview inside a scroll view.

    • OK expanding a bit..

      A table view is a high resource user. A scroll view is the same.

      If you do put a table view in a scroll view you will hit performance and usage issues.

      400 rows ????… thats a lot of data to put in at once. maybe consider paging it, if it is just a tile maybe ok but if it is images or lots of data on each row, then page it..

      Look at the tableview optimisation docs using a classname etc

      freeing the memory is simply null out the table view variable, after removing it from the window or when the window is closed…
