Handle error response in JSON. Please help!

Hello,

Titanium SDK version: 1.6.1
iPhone SDK version: 4.2

I get this response back from the API I am consuming and I want a popup
to show up on each error. For example: Desc can´t be blank.

This is the output in JSON.

{"desc":"can't be blank","value_1":"can't be blank"}

I tried this but it outputs every character, one by one.

for (var thekey = 0; thekey < response.length; thekey++) {

    alert(response[thekey]);

};

How can I output the errors?

  • Is this your own API which is responding?

    If you have the power, then a better error response, including a 'status' and/or 'error' value in the returning JSON would be much better and more useful.

    I am assuming (since you didn't provide any of your code) that you are responding to an HTTPClient request in your app.

    The JSON response in an 'onload' event would have the value 'this.responseText' which you appear to have assigned to a variable called 'response'.

    Since this is JSON, you can simple parse this by using :

    var my_response_json = JSON.parse(response);

    Then you can get your 'desc' and 'value_1' values by using simple dot notation do do something like :

    var desc = my_response_json.desc;
var value_1 = my_response_json.value_1;

if(desc=="can't be blank" || value_1=="can't be blank"){

    alert('there is an error');

}

    As I said, if it your scripts responding with this JSON, then it would be much better to return an error with an 'error' property and value in the returning JSON.

    Hope this helps.
