    • Well, it kinda is there in the docs. For a better insight, here is an idea on how I would do it. Create a function that returns an Animation object with randomly calculated x and y coordinates that are +/- 5 of their current position. The Animation object has a property called autoReverse. If you set this to true, then the original x/y position will always be the same. Use the animation callback to call the function for creating a new random animation and animate the view again in the callback. This will loop your animation giving the look of a continuous animation but always returning back to the same position, like its floating in space.
