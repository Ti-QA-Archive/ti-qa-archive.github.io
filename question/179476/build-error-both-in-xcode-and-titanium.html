  • My answer from extensive search:

    (This answer is only if you have have clang failed with exit code 254 with segmentation fault 11)

    What causes “clang failed with exit code 255” in Xcode?

    CASE 1 :

    If you're using genuine mac and if you directly updated mac from app store, Its just a problem of clearing old cache in Xcode. Your previous developed ios app's target kernel version has mismatch with newly installed command line tools (in this case LLVM compiler) OR Your ios app's old commnandline tools has mismatch with newly updated kernel OR both (in case you updated both xcode and mac). You just need to clear old cache of Xcode, clean and build. Mere restart should do the job.

    Follow this:

    ####http://miraclechow.blogspot.in/2014/02/deleting-xcode-cache-cleaning-build.html

    ####http://stackoverflow.com/questions/5714372/how-to-empty-caches-and-clean-all-targets-xcode-4

    CASE 2 :

    If you're using h4ck1nt0sh system, you may not have updated kernel version. Make sure you are at least running kernel version 13.3.0 on macosx 10.9.4 or kernel version 13.4.0 on macosx 10.9.5. You will get this error whenever there is a target kernel version mismatch with command line tools (in this case LLVM compiler 6.0 or similar)

    Compatibility list of kernel versions with llvm compiler

    To see your kernel version (in terminal type): uname -r
