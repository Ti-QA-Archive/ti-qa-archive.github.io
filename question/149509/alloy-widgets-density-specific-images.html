Alloy Widgets Density Specific Images

Hi, how do you reference density specific images in alloy widgets and what should be the folder structure for them?

  • Let's say we have an image name image.png we want to have multiple density versions of and we want to access it in our widget.

    First you'd add image.png to your widget's assets/android/images folders, with the appropriate density of image in each of the folders in that directory. Pretty simple and very much like you would do it in a traditional Titanium app.

    Now when you want to access the image, all you need to do is request it via your widget's generated runtime path. If your widget was named my.widget you could create markup in your widget's widget.xml file like this:

    <Alloy>
    <ImageView image="/my.widget/image.png"/>
</Alloy>

    If your image.png was instead in a more complex spot, like my/path/to/image.png, you would place it in the android folders above just the same. For example, to place a high density image, you put it in:

    app/widgets/my.widget/assets/android/images/res-hdpi/my/path/to/image.png

    And then to reference it in the same markup as above…

    <Alloy>
    <ImageView image="/my/path/to/my.widget/image.png"/>
</Alloy>

    Notice that the name of you widget is always the last path before the actual file name. This allows alloy to slip the platform-specific images into Titanium's platform-specific folders at compile time while avoiding any possible collisions.
