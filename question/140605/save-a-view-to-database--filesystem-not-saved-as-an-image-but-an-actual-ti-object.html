Save a view to database / filesystem? Not saved as an image, but an actual Ti object.

App Type: Mobile
SDK version 2.1.0
Platform: iPad (all versions)

Overview:
I have a lot of content read into an app dynamically, this content is then used to create around 100 views each containing labels, animations, images etc.

Processing/creating the views takes a bit of processing time (iPad 1) so I'd like to cache the views as blob objects either in a database or local file system.

Note I do NOT want to save an image of the view, but it's actual view object including state etc.

The following page says:
https://wiki.appcelerator.org/display/guides/Filesystem+Access+and+Storage
"A few of the ways you might use the filesystem include:
Novel uses, such as saving the contents and state of a view as a blob in a file"

Ti.UI.View doesn't seem to have a toBlob() function, and directly writing the view to a file fails.

Can anyone point me in the right direction, or is this not a possibility?

If it is possible, I'll also need to load the blob to a view again…

Thanks
