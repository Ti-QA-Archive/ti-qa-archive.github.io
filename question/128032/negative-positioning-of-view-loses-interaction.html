Negative positioning of view loses interaction

When a view is positioned with a negative value, it loses interaction. I think it might be better to explain it with a test case:

We have a container, that is positioned on the right side, off the viewport
(so left: +platformWidth).

Inside this container we have a view positioned with a negative value
(left:-platformWidth), so the view is in screen. Inside this view there is a button and a textfield, but both are not clickable.

I was wondering if any of you were experiencing something similar, if yes, I will submit a bug report.

I am using:
Titanium SDK version: 1.7.3, 1.7.5 and 1.8.0 (10/26/11 11:42 6dd8a4b…)
Platform & version: iPhone SDK version: 5.0
Device Details: iOS simulator
Host Operating System: OSX 10.6.7, Windows x, Ubuntu x.x
Titanium Studio version: Titanium Studio, build: 1.0.6.201110251616

This is the code for testing:

    This is not a bug… but how it works.

    If you have a view on screen then the clickable area is of the container. If that container i.e. a button is part of a view off screen and is then displayed over the view of the current screen it will not be active.

    This sounds a little weird the way I have described it. But

    Basically …. The parent view of the item you wish to add an event listener too is its active area, displaying the item outside of the parent view will make it in active.

    Hope this explains it …

    T…
