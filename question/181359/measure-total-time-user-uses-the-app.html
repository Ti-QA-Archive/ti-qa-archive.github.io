Measure total time user uses the app

Hi guys.

Is there any easy way to measure the total time a user is interacting with our apps? Not just the time when the app was opened, but the time when user uses the app (i.e., is in foreground).

I was thinking on create a timer when the app starts, detect when it goes to background to stop timer and continue timer when goes to foreground, but I have some questions with that:

  1. If I have 5 different windows, I must program all this in the 5 windows, or events like Ti.App.Event… will work always (once called in the index.js), no matter in what window the user leaves the app?
  2. Because app can die, I will need to save current value every time it die and retrieve it on load to continue the measurement, true?
  3. Is there any simpler method?

I need it to gave my users, individually, the "total time of use" statistic.

    Hi Manuel Conde Vendrell,

    As per my suggestion it's not good practice to start timer and continue during application running.. Because earlier i did same thing for change banners but it's create problem for me when i navigate to detail screen. (specially in android).

    but i have one idea if you want to implement it..

    At the time of application launch set current time into one property and at the time of application pause (when go to background) calculate the difference of current time and started time and store it into property. and at the time of resume override start time property and set new time and at the time close app calculate new time + old total time (which is stored in property).
