  • If you look at this error, this is because it is unable to find the variable app in app.js line 10, so rather than handling this type of error, it should be fixed, this is because when error comes then may be the logic which you want to implement, may not work and app crashes while running.

    So best way is not to run away from this problems and try to fix this problem. This error was due to developer fault during and its not an run time error.

    As it is good way to use try catch block where error may come in the app. But this may not be handeled automatically.

    There is one way to avoid this red screen comes into the app. This can be done when app is released for production so that user can't be able to view this red screen:

    For this you need to :

    Change Xcode source in main.m change 

    TI_APPLICATION_DEPLOYTYPE = @"development";

// to 

TI_APPLICATION_DEPLOYTYPE = @"production";

    This will never provide you red screen in production build. I think this may have provided you answer of your question.

  • Hi Iñaki Abete
    It is possible to handle the errors at runtime, you can use the try{}catch(e){} clause . It is used like this.

    try{
//The code you think can cause error.
}catch(e){
// catch the error here you can also print the message as e is the object with properties like message etc
Ti.API.info(e.message);//Will give you the message also
Ti.API.info(JSON.stringify(e)); // This will give you all the properties inside the e object

    Please feel free to ask if any confusion, or mark the answer if problem is solved.
