    • Also followed this process to allow using Titanium 1.8 with Xcode 4.3 but still the same problem. Although Xcode 4.3 should now be supported by Titanium 2.0

      1. Live on the edge with Xcode 4.3
        So as I stated, isn’t officially supported by Titanium yet, but that doesn’t mean it’s not possible to use. In fact, I’m using it myself. That said, there are a few things you’ll have to do manually to be able to use it. A combination of community contributions and Xcode bug squashing will give you the opportunity to use the latest Xcode before it hits an official Titanium release.
        How to use Xcode 4.3 with Titanium 1.8
        Upgrade Mac OSX to 10.7.3 from “? -> Software Update”
        Download and install Xcode 4.3.1
        Run Xcode 4.3.1 and install any necessary components if prompted. It is not required to delete prior Xcode versions, but you can if you want to.
        Install Xcode 4.3.1 command line tools from “Xcode -> Preferences -> Downloads”
        sudo xcode-select -switch /Applications/Xcode.app/Contents/Developer
        Download and extract ios-sim-xcode4.3.tar.gz
        Copy the new ios-sim executable to any Titanium SDK paths you are using. For example, if you are testing with Titanium 1.8.1 and 1.8.2, you would do the following:
        cp /path/to/ios-sim ~/Library/Application\ Support/Titanium/mobilesdk/osx/1.8.1/iphone/
        cp /path/to/ios-sim ~/Library/Application\ Support/Titanium/mobilesdk/osx/1.8.2/iphone/
        Create a new project, or clean an existing one, and build for the iPhone/iPad simulator and you’ll have your Titanium app running against Xcode 4.3 and iOS 5.1!

  • I am now able to work successfully with Xcode Version 4.3.2 (4E2002) and Titanium 2.0.1GA2.

    What worked for me was the following. Hopefully someone can come up with a more concise explanation, but this worked for me and I can get back to developing …

    1. Take a backup of your existing Titanium project directory.

    2. Fully uninstall Xcode and Titanium - make sure there are no files are directories remaining.

    3. Reinstall Xcode and the Titanium.

    4. I imported the projects from the backed up directory into Titanium.

    Can now build in Simulator, Device and as a distributable ipa file. I have not tested in Android.

    I also looked more closely at the messages in the console and one of my projects worked fine in the simulator but would not build for device or file. The problem was with a png file. Once removed the project built correctly.

    It could be some tolerances that were acceptable in previous versions are no longer ok.

    Hope this is useful for others.
