  • Hi

    You are right to be confused - it is VERY confusing.

    When developing for Android you can ignore the vast majority of these errors and it will take time to work out what is an error that is something you should care about ans something that it system generated and just bounced on the screen.

    If you are using Studio to debug - only worry about the ones that are blatant or that actually stop you app from running and require you to intervene in the code.

    Things that mentioned your code files and lines within those files are also of interest.

    Always make sure you have the latest versions of all Android APIs and this can sometimes reduce the count of the error.

    Just one of the many joys of working with Android.
