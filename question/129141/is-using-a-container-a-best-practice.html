Is Using a Container a "Best Practice"

I know it's situational, but I'm looking for generalization in this case. For a while, I had one in my app, but then it seemed like it wasn't doing anything except adding complexity. Now I'm starting to wonder whether that was a bad idea.

Within this app, most of my windows are a stacked (vertical) layout and after removing the container (which was a ScrollView), the windows which extend out of the viewport no longer scroll. I guess I just assumed that the window would do what it needs to do to access its content.

What are experienced Titanium folks doing with respect to containers and, perhaps most importantly…why? I'm looking for something that I can adopt as my default approach, but I'd like to understand why one way is better than another.

Thanks.

  • Accepted Answer

    IMHO you are doing the right thing if you want the content to scroll beyond the screen size. So use a scrollview container in that instance. Else, you need to ensure that you content fits inside the viewport. Regards the second comment, I would use a container for both views and then I can control how much vertical space each uses by setting the heights of the inner views using percentages.
