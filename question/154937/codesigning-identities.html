Codesigning Identities

Hello, I have been building apps with PhoneGap/Xcode for a while and I am interested in switching to Titanium Studio. I am trying to test the development workflow by simply running an app on my test device (iPhone 5 iOS 7), using the "HTML-based Application" template from the TI Dashboard "Develop" tab.

No matter what I do, I keep getting the following errors when I try to run on my iPhone:

[ERROR] : Error details: Code Sign error: No codesigning identities found: No codesigning identities (i.e. certificate and private key pairs) that match the provisioning profile specified in your build settings (“HomeOffice Profile”) were found.

[ERROR] : Error details: CodeSign error: code signing is required for product type 'Application' in SDK 'iOS 7.0'

This makes no sense to me. My profiles and certificates are valid, and I can switch over to Xcode from Titanium and run an app using the same cert/profile/device. I have tried various combinations of the following several times

  • Remove cert/profile from my keychain
  • Restart computer
  • Reinstall cert/profile

Any time I do this, Xcode can successfully build apps, but never Titanium Studio. From Titanium Studio, I can only run apps in the iPhone/iPad simulator, but this is useless to me since the first app I will be building has a lot of real-world GPS scenarios and I want to test outside of the simulator environment.

Here is the other info from the requested list for support tickets:

  1. Application type: mobile
  2. Titanium SDK: Titanium SDK version 3.1.1.GA
  3. Platform & version: iOS 7.0
  4. Host Operating System: OSX 10.8.4
  5. Titanium Studio: Titanium Studio, build: 3.1.1.201306112235
  6. a simple test case: Build any app to iOS device (I have tried several, including blank templates from the "Develop" tab in the dashboard)
  7. Logs: See next
  8. Android and iOS Development: duplicated from earlier:

[ERROR] : Error details: Code Sign error: No codesigning identities found: No codesigning identities (i.e. certificate and private key pairs) that match the provisioning profile specified in your build settings (“HomeOffice Profile”) were found.

[ERROR] : Error details: CodeSign error: code signing is required for product type 'Application' in SDK 'iOS 7.0'

Please help! I like the look of the studio and would like to be able to really test the workflow and capabilities!

  • OK, I thought I'd take the time to post a workaround that I discovered today as courtesy to anyone else out there who wants to use Titanium for some development, but is also testing development for iOS 7 in other SDKs.
    As Stephen Feather said, you cannot currently publish to the iOS 7 SDK. However, if you have already installed, don't waste your time asking questions and getting downvoted for it, just do the following:

    sudo xcode-select –switch /Applications/Xcode.app/Contents/Developer

    Then restart Titanium Studio. This switches you back to the previous version of Xcode's SDK, provided that you didn't uninstall it.

    If you want to switch back to Xcode 5 for testing iOS 7 development in other SDKs, reverse the command:

    sudo xcode-select –switch /Applications/Xcode5-DP3.app/Contents/Developer

    Note that switching to Xcode.app only allows you to publish to iOS 6.1 and lower, but I successfully published to an iOS 7 device and it worked just fine, it just doesn't have the new UI.
