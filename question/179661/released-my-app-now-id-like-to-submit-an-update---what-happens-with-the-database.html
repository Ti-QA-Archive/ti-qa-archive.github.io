Released my app, now i'd like to submit an update - what happens with the database?

So i released my app to android and amazon , you can see the app

here

Anyway, I'd now like to push an update to the app. The update has nothing to do with the database. If a user has the app downloaded, and then updates the app, will that update

  1. erase all the database data they have and then reinstall it?

  2. not touch the database data, and leave it as-is?

Is there documentation of how this works somewhere? I plan to push to iOs soon, is database-updating on app-update consistent across all platforms?

    The database should be left untouched. If you haven't changed the sqlite file or removed it, or included any code that modifies that database in the new version, then it will continue to be used as it was before. About the only place I know where soemthing like this is documented is in our best practices guide where it recommends including a version number to make DB updates easier.
