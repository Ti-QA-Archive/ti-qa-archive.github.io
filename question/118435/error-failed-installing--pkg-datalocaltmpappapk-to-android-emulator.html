[ERROR] Failed installing ...: pkg: /data/local/tmp/app.apk to Android emulator

Have seen others ask this question… I have a larger application where this is happening - about 32MB - small test applications work fine with Android emulator.

Using Titanium SDK 1.6.1 with Google APIs 2.2 / WVGA800 on MacOS X 10.6.6
Have increased Heap size 128 and Device RAM size 512 or 1024

Android packages installed snapshot (up to date!)
[http://img156.imageshack.us/i/screenshot20110412at125.png]

Android device config snapshot
[http://img694.imageshack.us/i/screenshot20110412at125.png]

Application does install to real hardware HTC Inspire which is Android 2.2 WVGA800 with 1024MB. Thank goodness we got real hardware to test on.

Mostly get the [ERROR] Failed installing - sometimes get pernicious timeouts trying to connect to Android simulator - and rarely get the "run out of device memory" error. Very rarely the install actually succeeds and the app gets launched.

Have tried all suggestions could find on the board…

  1. deleting /tmp/Android

  2. deleting $HOME/.android

  3. deleting $HOME/.titanium

  4. deleting build/android (although this is somewhat dangerous - it appears that 1.6.1 does not properly re-create all of the expected files in build/android - and in any case it appears that our app is properly built, it just fails on the install)

  5. recreating device in AVD with larger memory

  6. restarting Titanium and killing adb

Basically question is whether to abandon use of Android emulator or just stick with the device? Makes debugging a bit trickier but it sounds like an easier proposition.

And, to that point, anyone with suggestions on remote debugging of Android applications on real hardware?

tia, Joe
