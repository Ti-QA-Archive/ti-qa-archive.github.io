Keyboard covering scrollview

I have a registration type page that requires a lot of fields all put in a ScrollView. When I start entering info and the the keyboard pops up, it covers the bottom fields. I can still access them by clicking the next button on the keyboard, but it won't let me scroll down to them. It's like the ScrollView still continues all the way to the bottom of the screen when I want it to only go to the top of the keyboard. Anyone have a solution to this?

Application type: mobile
SDK 4.0
Android 4.0
Galaxy s5
Windows 7

  • You can detect height of onscreen keyboard by using Ti.App.addEventListener('keyboardFrameChanged'). It gives keyboardframe dictionary object containing width and height of keyboard. But unfortunately keyboardframe dictionary is available only for iOS. For your Scenario you can use tableView to contain all your form data and it will automatically scrolls upword to display bottom textFields.
