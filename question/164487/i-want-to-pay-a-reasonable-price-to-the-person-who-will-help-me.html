  • 1) The Android-emulator is known for being painfully slow. One solution if you really need to use an emulator is using an x86-based one, like Genymotion. This might cause some issues with modules that aren't built for x86-processors.

    I personally prefer to run my apps on a real device. It's easy, it's quick and you get a much better idea on how it will perform on real devices. If you don't have a device yet, get one. You should definitely test on a real device before the app is finished.

    2) On Android you won't get any smaller APKs. The reason for this is that Titanium bundles the V8 JavaScript-runtime that's used to actually run your code. With 15 years of experience you shouldn't be surprised that runtimes takes a bit of space.

  • Mario,

    A couple of things:

    1) if you expect your APK to be smaller, it won't. If this is a deal breaker for someone to help you, them it's going to be a problem. Titanium bundles the v8 engine in with the APK. You need it, there is no way around it and it means your APK starts at over 6mb+.
    2) have you tried this tutorial on setting up tishadow? It might help.

    3) did you ever get genymotion installed and configured? You MAY find that is good enough without tishadow.

    You can find the genymotion download here

    I would start with genymotion, give it a try and see if that helps you enough. If you do need more help, let me know and I'll see what I can do.
