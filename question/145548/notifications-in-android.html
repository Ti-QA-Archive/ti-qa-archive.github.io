  • Since you said that you've already handled the chat portion, it sounds to me like your big challenge is to reliably provide notifications to the user, even if your app is not in the foreground (or is not running at all).

    One approach that might work: use the AlarmManager to periodically check for messages on the server, and if it finds a message, trigger a notification.

    Using the alarm manager requires a third party module, bencoding.AlarmManager.

    I have a tutorial app, GrillTime, where I detail the use of AlarmManager and notifications (for a different purpose, but I think you will find some valuable discussion). Look to part 3 of the tutorial, and download the source code so you can examine the code while you read the article.
