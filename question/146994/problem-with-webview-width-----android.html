problem with webview width --- Android

Hi,

I am using a webview in my app.

In my tiapp.xml:

<supports-screens android:anyDensity="true"/>

I have the samsung google Nexus S (480 x 800px 233dpi)

This is my webview:

    var mywebView = Ti.UI.createWebView({
    url : '../../local.html',
    width:480,     /*exactly the same as my nexus s*/
    backgroundColor:'#000000'
    });

That code produces a webview too large on my device. I have a horizontal scroll bar. Why raw integer values are not corresponding to real pixels on the screen?

Android 4.1.2

thanks

  • Hi

    As you a designing an app that will fit on multiple devices (not just your test device) you should code so that all devices will have the appropriate sized views.

    If you need your webView (or any view) to fit the width of the device you can tweak your code as follows;

    var mywebView = Ti.UI.createWebView({
    backgroundColor: '#000000',
    url : '../../local.html',
    width: Ti.UI.FILL // this will adjust for all device widths automatically.
});

    You can do the same for the height as well.

    • If the problem is the contents of the web view html (this is not very clear) then you adjust your HTML as follows;

      In the head section of your HTML content add this line;

      <meta name="viewport" content="width=device-width, initial-scale=1, maximum-scale=1">

      Fuller example;

      <!DOCTYPE HTML>
<html lang="en-US">
<head>
    <meta name="viewport" content="width=device-width, initial-scale=1, maximum-scale=1">
    <meta charset="UTF-8">
    <title></title>
</head>
<body>
    Your html mark-up here
</body>
</html>

      This would be because the browser (or webView) doe snot tell the HTML page what size tobe, it is the web page that reacts to the device width.
