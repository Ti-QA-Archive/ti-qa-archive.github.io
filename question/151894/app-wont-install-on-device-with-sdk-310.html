App won't install on device with SDK 3.1.0

I've been using SDK 3.0.0 for a while, but started getting App.js not found randomly. Obviously App.js is there. So I read that it could be an SDK issue, so I went up to 3.1.0 which gives me this error during build:

[ERROR] :  ** BUILD FAILED **
[ERROR] :  The following build commands failed:
[ERROR] :      PhaseScriptExecution Pre-Compile "build/POS Anywhere.build/Debug-iphoneos/POS Anywhere-universal.build/Script-DABB369F12E8CB280026A6EA.sh"
[ERROR] :  (1 failure)

I don't really care if it's 3.0.0 or 3.1.0 .. I just need one of them to work. I should say that this app will install and work just fine on the simulator in either sdk.

I'm trying to instal on an iPod touch, not sure of generation.

Tried deleting and redownloading the provisioning profile, which didn't do anything. I do have an ad hoc certificate and a development certificate for this app. I'm trying to install with the Development certificate. And it was working fine until about an hour ago.

  • My similar issue was fixed by spending hours combing over files for commas and semicolons, as explained here, http://developer.appcelerator.com/question/146809/device-error-script-error–invalid-method-createtextfield-passed-to-uimodule .

    Though what differed in my own project was that the problem seemed to be caused by an extremely long, pre-defined array that I was using to supply temporary data to my app for testing,
    ie var myArray = [&#39;string1&#39;, string2&#39;, etc.. ];
    The array was long enough that the strings in the array stopped changing color when encapsulated by quotation marks.

  • I figured it out. For me, I was missing a closing brace on a function. You'd think that Titanium would've complained and not let me build it with a syntax error. It neither showed up as a red flag nor told me where the error was. I went through all the trouble of upgrading my xcode and ios sdk to 6.1 only to find out it was a stupid mistake like that. Eventually found it through trying to build in xcode which gave me the file and line that was broken. Thanks for everyone's help.
