Local JQuery in Android

Hi everyone. I'm currently working on an app for both iOS and Android. The app needs to work without an internet connection.

In my project I have a folder that contains the HTML pages, the javascript pages that load the HTML in a webview and a folder called jquery-mobile. Inside the jquery-mobile folder, i have all the files you need for it to work locally. On an iPhone, with airplane mode on so I know I'm not getting an internet connection, it works fine.

On an Android device with wifi off, it just displays like a webpage with no CSS. I tried moving the jquery-mobile folder to the Android folder to see if that would work (like with images in Android) but no luck.

WIth wifi on, and links to the same files on the Jquery server, the html pages display properly. Am I missing something here? Does JQuery only work on Android if it's not local? Any help would be much appreciated. Thanks.

  • Does JQuery only work on Android if it's not local?

    No, works fine. You do however…

    it just displays like a webpage with no CSS

    …need to be sure your references within your html are correct, understanding that iOS and Android handle those references differently.

    And, *.js files used within a webview, need to be renamed to something like .ljs so they are not processed at build time and are available to the webview at runtime.
