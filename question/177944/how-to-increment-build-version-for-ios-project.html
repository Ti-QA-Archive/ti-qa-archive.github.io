How to increment build version for iOS project?

I recently uploaded a build to iTunes Connect but realized there was an issue so I rejected the binary. Now I have it fixed and am ready to upload again but cannot. iTunes Connect is ready for a 1.0.5 build but wont let me delete the current build that is up there nor will it allow me to upload the new build with the same version number (I get a 'Redundant Binary Upload' error). So from the below article the only fix is to increment the build version and upload another build.

http://stackoverflow.com/questions/25699404/new-itunes-user-interface-unable-to-find-ready-to-upload-binary

I tried grabbing the info.plist from the build/iphone folder and increment the bundleversion property and then placing it into the root of my project and rebuilding but that did not seem to work. I still got the 'Redundant Binary Upload' error.

Any suggestions on how to increment the build version so I can submit this app?

    • You can't replace a binary once it was uploaded to the "new" itunes connect.

      You have to increment either the Build or Version number of the binary. Since I don't know a way how to change the Build number with Titanium, I suggest you to increment the Version number. You can then go ahead and add a new build to itunes connect, upload it and select it before submitting for review.
