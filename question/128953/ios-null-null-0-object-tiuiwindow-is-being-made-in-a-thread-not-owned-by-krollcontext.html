iOS: (null)->(null) [0]->[object TiUIWindow] is being made in a thread not owned by <KrollContext

iOS, Ti 1.7.5/1.8

Does anyone have any idea what this means or what sort of thing I'm doing that could be causing it?

(null)->(null) [0]->[object TiUIWindow] is being made in a thread not owned by <KrollContext

At some point I experience a mysterious crash and I'm trying to trace down any antecedents for it. (It seems to me that more recent versions of the SDK provide much less in the way of helpful error messages.)

  • Accepted Answer

    Generally, this is caused when you create a new window with a file URL, and then call a function or property that was declared in a different window (execution context). Because of how our JS interpreter runs, this creates unsafe conditions sometimes, so tons of warnings are printed as a result of calls like this, and crashes can frequently occur.
