Strange behavior with httpClient between TiDev 1.5.1 and 1.6

Hi all,

I've been struggling with this very strange problem and have been unable to solve it for days now and was hoping someone in the community can assist or help determine if it is a bug with Ti 1.6.

My code accesses the Google Docs List API to export online Google Document contents to a webview in the app. It works perfectly in TiDev 1.5/1.5.1 but is just not working the same in TiDev 1.6.

To replicate the problem you can follow these steps with this code:

  1. Copy and paste the code from here to your app.js (http://pastie.org/1626702). You can start with a new project in TiDev and just overwrite the existing app.js content with this.

  2. Create a new file called "parse.js" and paste the code from here http://pastie.org/1626711.

  3. If you have a Google account, login using your desktop and create a few dummy documents (these must be of 'Document' type).

  4. Build the code in Titanium Developer using SDK v1.5 or 1.5.1 and login to your Google account through the app. Once logged in, click on a document in the list to see the document display in the webview.

  5. Repeat step 4 but this time use SDK v1.6. You should see that the Google page that comes up now is not the right one.

So, same code, but different results when built using v1.5 and v1.6. Would greatly appreciate any assistance or insight from the community.

Thanks in advance.

  • Eureka moment here! I've finally solved this! Google was implementing a redirect to different webservice URL (return code 302 moved temporarily) and this was not being handled properly in 1.6 unlike 1.5. Once I realized what the correct URL endpoint should be, it works like a dream. It's a good thing to know that 3rd parties can always change a source URL on you anytime so hard-coding things like that may cause issues for your users when things change.
