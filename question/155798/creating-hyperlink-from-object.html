    Hi

    First thing I would do is get rid of this pattern;

    dealerArea.add(dealerData,resultsHead,dealerCount,dealerContact);

    You are going to get killed when/if you try to cross platform this code - there are incompatabilities with adding more than one item in an add method.

    You also appear to be adding all your labels into an array and then adding those in the add method - again not a great idea.

    You should simplify your code and create a single view to take your labels that handles the top position automatically for you.

    var view = Ti.UI.createView({
    height: Ti.UI.SIZE,
    layout: 'vertical',
    width: Ti.UI.FILL
});
dealerWin.add(view);
while (rows.isValidRow()) {
    view.add(Ti.UI.createLabel({
        text: rows.fieldByName('Name'),
        top: 0,
        left: 250,
        width: 500,
        height: 30,
        font: {fontSize: 20, fontWeight: 'normal', fontFamily: 'Source Sans Pro'},
        color: '#000'
    }));

    view.add(Ti.UI.createLabel({
        text: rows.fieldByName('Address'),
        top: 0,
        left: 250,
        width: 500,
        height: 30,
        font: {fontSize: 20, fontWeight: 'normal', fontFamily: 'Source Sans Pro'},
        color: '#000000'
    }));
    ...

    var urlDealer = Ti.UI.createLabel({
        data: {
            url: rows.fieldByName('Url')
        },
        text: 'URL: ' + rows.fieldByName('Url'),
        left: 250,
        width: 600,
        height: 30,
        font: {fontSize: 20, fontWeight: 'normal', fontFamily: 'Source Sans Pro'},
        color: '#000000'
    });
    urlDealer.addEventListener('click', function (e) {
        alert(e.source.data.url);
        // do somthing with this here
    });
    view.add(urlDealer);
...

    You will also notice the link is handled with an event and the url is passed in as a custom property called data with a child property of url - keeps things neat.

    • Source is a special property that lets you access the original "thing" that the event was fired off.

      In this case source represents the label we have attached the event to. This way you can access the custom property I added directly from the label "source" that fired the event.

      You will find this property available on all events that fire from a; view, label, button, tableView etc.

    • Here is something else you can do when you "get" how source works and means your code can become much more manageable.

      Imagine you have three labels, each displaying a URL as a caption, you want each of these to react to a touch event. You would normally add an event to each one of these labels and the content of the event would be the same - which is a shame. You would normally think they are different because each one has a different URL - thus the slightly different code.

      But I can do it in one function and make your code much easier to read at the same time. This is example has simplified object creation but process is perfectly formed.

      // data for example code only
var url1 = 'http://www.apple.com/';
var url2 = 'http://www.microsoft.com/';
var url3 = 'http://www.google.com/';
var handleClick = function (e) {
    Ti.Platform.openURL(e.source.data.url);
};
var lblURL1 = Ti.UI.createLabel({
    data: {
        url: url1
    },
    text: 'URL: ' + url1
});
var lblURL2 = Ti.UI.createLabel({
    data: {
        url: url2
    },
    text: 'URL: ' + url2
});
var lblURL3 = Ti.UI.createLabel({
    data: {
        url: url3
    },
    text: 'URL: ' + url3
});
lblURL1.addEventListener('click', handleClick);
lblURL2.addEventListener('click', handleClick);
lblURL3.addEventListener('click', handleClick);

      Here we create a generic function handleClick that accepts the same parameters that the event listener uses, using the source attribute we are able to reference which of the three labels actually responded to the click event and use that labels URL.

      Makes life easier.
