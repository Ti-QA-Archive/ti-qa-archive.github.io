Setting width of createimageview is ignored on Firefox

I have an 400x400 pixel image. When I scale it down and it runs on Firefox, the height is properly scaled, but the width tag is ignored and the image is squashed.

On Chrome it is rendered correctly. 

var profileImage = Titanium.UI.createImageView({
        top:510,
        left:510,
        width:64,
        height:64,
        image: 'f4.jpg'
});

Explicitely scaling, a la profileImage.setWidth(64); also does not work.

I would like to use Firefox for debugging - does anyone know how to resolve this?

  • I found a similar problem with a mobile app – it worked fine in iOS, but on Android, my images were squeezed. I had to explicitly calculate the height and width using the aspect ratio of the image.

    Assuming you have the width and height of the image, you can do it like this:

    iv_w = 64;
iv_h = 64;
if (image_w > 0 && image_h > 0)
{
    iv_w = (image_w > image_h) ? 64 : parseInt (64 * image_w / image_h);
    iv_h = (image_h > image_w) ? 64 : parseInt (64 * image_h / image_w);
}

    image_w and image_h are the width and height of the image; iv_w and iv_h are the width and height of the imageview.

    I'm not sure exactly how you could dynamically get the height and width of the image; in my case, those values are passed as part of the data feed I'm receiving.
