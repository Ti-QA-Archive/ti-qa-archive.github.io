Errors Building for iPhone

I've been having a very positive experience with Titanium so far. But I've hit a brick wall trying to get my app on to my iPhone. All the various certs are set up and Titanium says I'm ready to go, but when I build I get the following error:

Error: Traceback (most recent call last):
File "/Library/Application Support/Titanium/mobilesdk/osx/1.5.1/iphone/builder.py", line 1143, in main
execute_xcode("iphoneos%s" % iphone_version,args,False)
File "/Library/Application Support/Titanium/mobilesdk/osx/1.5.1/iphone/builder.py", line 925, in execute_xcode
output = run.run(args,False,False,o)
File "/Library/Application Support/Titanium/mobilesdk/osx/1.5.1/iphone/run.py", line 31, in run
sys.exit(rc)
SystemExit: 1

The app fails to install. What's more, when I go back to the simulator it no longer runs on there either! It fails with the following error:

[WARN] attempted to load: TiUITabGroupProxy
[ERROR] The application has crashed with an unhandled exception. Stack trace:

I have found that the warning appears with whatever the first command in the app.js is. So it's not actually to do with TiUITabGroupProxy.

The only way to get my app running in the simulator again is to copy out the resources, delete the project, create it again and put the resources back in.

Any ideas what's going on?

Sean

  • I ran into a similar issue a few days ago. It turned out that my provisioning profile had expired.

    I renewed it on Apple, downloaded the new profile, and then selected it in the compiler.

    At no time did the compiler tell me that I was trying to use an expired profile. Might be an area to check into.
