Home Button without Back button at the header

Hello, am a Ti newbie. I am trying to create a home button at the header at the right near the back button. Is there a simple way to create a home button after say two pages have been navigated without showing the back button.

Here is my code:

var homeButton=Ti.UI.createButton({title:"Home"});
homeButton.addEventListener("click",function(e){
    var win=Ti.UI.createWindow({
        url:"../homePage.js"
    });
    Ti.UI.currentTab.open(win,{animate:true})
});

win.setRightNavButton(homeButton);

Though it goes to the home button (homePage.js) it still shows the back button which is not a normal user experience. Please help .Thanks.

  • Hi

    I am going to assume you are talking about iOS as you mention a back button.

    Firstly, if you are opening windows as children of the specific tab then the correct user experience is to see a back button on each new window so the user can return back to the start.

    It is not common to add a home button on a child window of a stack, but it is possible, just not the way you are doing it. The problem is that you have should be removing the previous windows in code yourself if you want the user to go back to the start, as you are using the url property of each window that is very difficult to achieve.

    If you are presenting a series of windows one after the other but wish have the ability to simply close the windows quickly without the messy clean up then you should create a modal window as the first window and then add a navigation controlled to that, open the windows as you have been with the tabs. When it comes to closing the windows you should close the modal window.

    But as I say having a close/home option at the end of a stack or windows is not the normal experience.
