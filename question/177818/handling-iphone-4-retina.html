Handling iPhone 4 (Retina)

I am currently developing a cross-platform application. I have assets for the following android densities:
res-long-port-ldpi, res-long-port-mdpi, res-long-port-hdpi, res-long-port-xhdpi, res-long-port-xxhdpi & not-long versions. I have non-retina and retina assets for iOS. I use DP for measurements. The layout is fine for Android. On iOS the layout is fine when running the iPhone 5 simulator and on a 3GS device. When running the app on the iPhone 4 retina simulator the retina images are obviously still used but are way to big for the screen. I'm wondering how to solve this issue? The retina assets seem to be designed for iPhone 5. I was given the assets and can no longer contact the person who passed the assets down.

Any help would be appreciated, thanks…

  • Hi John,

    just put your retina images in the same folder as your non-retina images, a respect this naming convention

    imagename.png <--- normal version
imagename@2x.png <-- your retina version

    in js just use the plain name, without any suffix

    var view = Ti.UI.createView({
    backgroundImage:"/path/to/imagename.png"
});

    iOS will automatically search for a file with the same name and a "@2x" suffix, it will display that.
    The same naming convention is also used for Splashscreens, btw. since iOS 8, you can also apply @3x images for higher resolution devices.

    have a nice day

    Moritz
