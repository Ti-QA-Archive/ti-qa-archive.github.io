Error: EPERM, operation not permitted Thumbs.db

I am working on an android app using Titanium 3.4.1GA and Alloy 1.5.1 on a Win 7 PC
I was able to compile with out any errors and went to make some changes to the images in app/assets and apps/assets/android
Now when ever I attempt to compile I get:

fs.js:427
return binding.open(pathModule._makeLong(path), stringToFlags(flags), mode);

            ^

Error: EPERM, operation not permitted 'C:\Documents\Titanium_Studio_Workspace\IMOK\IMOK2\Resources\android\Thumbs.db'

at Object.fs.openSync (fs.js:427:18)

I have tried compiling other projects and they work.
I have searched the internet and there is a number of posts that explain you need to delete the Thumbs.db file if you have made changes to the images. I cannot find the Thumbs.db file in any of my directories (file options is set to show hidden files).

Is there a setting that inadvertently got changed ?

  • After much searching and combining answers, here is what I did:

    Disable the system from using Thumbs.db:

    • Go into Windows Explorer and Alt+f go into Tools/Folder Options View tab, click on "Always show icons, never thumbnails".

    • Using a cmd window, goto the project's app/assets directory, find Thumbs.db by using the dir (show hidden files) command:

      dir /Ah

    • unhide the file by using the attrib command from the command prompt

      attrib *.db /S /H

    • then delete the file

      del *.db

    I had to do the samething with Thumbs.db in the app/assets/android directory

    Was able to build the project without any issues.

  • Hello Greg,

    Thanks for your inquiry. First, check Your image file isn't open on an another software. Second, you can safely delete any thumbs.db file although Windows will automatically recreate it unless thumbs.db is disabled.

    Here's how you can disable thumbs.db:

    For Windows Vista and Windows 7:

    • Click the Start button

    • Click on Control Panel

    • Click on Folder Options

    • Click on the View tab

    • Under the Files and Folders section check the button next to "Always show icons, never thumbnails" Click on Apply

    • Click on OK

    You may need to run Disk Cleanup in Vista or Windows 7 to remove any thumbs.db files that have already been created.

    Hope this clears your problem. Feel free to response. Thanks.
