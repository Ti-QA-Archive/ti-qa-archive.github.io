ACS Web interface editing not working with my data format.

I have a CURL statement that creates an object in the custom objects section of an ACS app, using the following Syntax:

curl -b cookies.txt -X POST --data-urlencode 'fields={"Name":"Pasta Fresco and The Black Tomato","Locations":{"address":{"addr":"862 Heritage Blvd West","city":"Lethbridge","prov":"AB","postal":"T1K 7V5","phone":"403-380-6225","fax":"403-380-6212","lat":49.6970894,"lon":-112.8923772}},"orgLink":{"link":{"website":"www.pasta-fresco.com","twitter":"www.twitter.com/blacktomatola","facebook":"www.facebook.com/groups/177288495644472"}},"pdid":2,"active":1,"Category":"Dining"}' https://api.cloud.appcelerator.com/v1/objects/organization/create.json?key=APPCELERATOR_APP_KEY

When I run this, I get the object created in ACS. Great. The problem is, when I look at the record in ACS, the "Locations" and "orgLink" fields have the colons (:) that used to be between the values replaced with the following characters (=>). So, through the web interface, the fields read:

Locations field Value is {"address"=>{"addr"=>"862 Heritage Blvd West", "city"=>"Lethbridge", "prov"=>"AB", "postal"=>"T1K 7V5", "phone"=>"403-380-6225", "fax"=>"403-380-6212", "lat"=>49.6970894, "lon"=>-112.8923772}}
orgLink field Valus is {"link"=>{"website"=>"www.pasta-fresco.com", "twitter"=>"www.twitter.com/blacktomatola", "facebook"=>"www.facebook.com/groups/177288495644472"}}

Now, I retrieve the value of these fields in my app and it works fine… however when I try to Edit the fields in the ACS web interface, it doesn't see these fields as valid JSON, so won't let me save them as a Dictionary field type. This breaks my app, because the web editor is changing the data format. The web editing worked until the new interface was introduced a few weeks ago.

If I try replacing the >= values in the field with a colon (:) it will save the value as JSON, but when I try to retrieve it in the application, there is a leading backslash in front of each quotation… This breaks my app.

Here is how the app retrieves if I use CURL to submit it:

{"address":{"addr":"400 4th Ave S","city":"Lethbridge","prov":"AB","postal":"T1J 4C9","phone":"403-381-3222","fax":"","lat":49.6936717,"lon":-112.8407795}}

if I change it to valid JSON through the web interface and retrieve it, then I get the following value in the app:

{\"address\":{\"addr\":\"862 Heritage Blvd West\", \"city\":\"Lethbridge\", \"prov\":\"AB\", \"postal\":\"T1K 7V5\", \"phone\":\"403-380-6225\", \"fax\":\"403-380-6212\", \"lat\":49.6970894, \"lon\":-112.8923772}}

So I'm trying to figure out is there any way to fix this? Right now the web editing interface is unusable for me and I need to make updates using CURL… a real PITA. The only way I can update is to use CURL and submit valid JSON.

Ray

    Hi Ray, sorry about the bug.

    This is a bug that our engine, Rails interpret the json as hash, which instead of colon(:) it shows with arrow(=>).
    Which means that website just showing malformed data that is actually correct on the backend.
    For now you should update dictionary formatted data with CURL.

    This should be fixed on 2.1.3 release.

    Sorry about the inconvenience.
