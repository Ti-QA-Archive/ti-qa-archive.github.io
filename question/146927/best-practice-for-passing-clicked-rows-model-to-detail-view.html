Best practice for passing clicked row's model to detail view?

Consider a Collection of custom Models that has been fetched and is displayed in a TableView:

<Alloy>
    <Collection src="transaction" />
    <Window title="History" id="win">
        <TableView id="tableView" dataCollection="transaction"> 
            <TableViewRow id="row" hasDetail="true">
                <Label id="amount" text="{amount}"/>
            </TableViewRow>
          </TableView>
    </Window>
</Alloy>

We need to open a detail view for the Model when the hasDetail button is clicked. What is the best way to communicate the Model to the detail view? Here is the start of a detail view:

<Alloy>
    <Model src="transaction">
    <Window id="win">
        <Label id="date" text="{date}"/>
        <Label id="amount" text="{amount}"/>
    </Window>
</Alloy>

    • That does it just fine.

      To get the model instance of the clicked row:

      tableView.addEventListener('click', function(e) {
   var myModelInstance = myCollection.at(e.index)
}

      If you pass {model: myModelInstance} when creating the controller, and the controller marks up a model instance in the XML, this line will make the local model instance reflect the passed model:

      var args = arguments[0] || {};
$.detailModel.set(args.model.attributes);

      Here is the XML for the detail view:

      <Alloy>

   <Model src="ModelClass" instance="true" id="detailModel"/> 

   <Window model="$.detailModel">
      <Label text="{$.detailModel.}"/>
   </Window>

</Alloy>

      BONUS! To provide a transform function for the local model in the detail view, do this before that other stuff:

      $.detailModel = _.extend({}, $.detailModel, {
    transform : function() {
        return transformFunction(this);
    }
});
