Breakpoints being ignored in the new Appcelerator Studio

Hi all-

I am currently running the new Appcelerator Studio, with the 3.2.2 SDK. When placing breakpoints into my controllers and running the app in Debug mode, I've noticed that no breakpoints are hit. Other messages similar relate to issues in the SDK, but this seems to have resolved itself in the latest SDK. I was wondering if there were any other tricks or tips to get my breakpoints to trigger in App Studio?

Thanks in advance for any guidance.

Chris

  • You need to use the debugger keyword option in the debug configuration.

    Select the relevant debug configuration, in the debug tab on the right there's a checkbox marked 'suspend on debugger keyword'. Select that and you're good to go, anywhere you add the line debugger; in your code, you'll effectively get a breakpoint.
