Distribution Issues

Tried to submit an applicaton and got the following error:

This bundle is invalid. The key CFBundleShortVersionString in the Info.plist file must contain a higher version than that of the previously updated version.

Not sure what is going on…

  • Here is the solution to this problem:

    1. Compile the application for distribution.
    2. Right click the archived application and show in finder.
    3. Right click the xcode archive and select show package contents.
    4. Navigate to Products > Applications, right click the application and select show package contents.
    5. Copy the Info.plist file and paste into the folder /platform/iphone in your projects resource directory.
    6. Open the Info.plist file and edit the Bundle versions string, short and increment the value.
    7. Recompile the application and submit.

  • Forgive me if you know all of this already, just want to cover some of the basics.

    Open the xproj file in build/iphone, which will open your project in XCode. You'll see a version in there, which you can increment from 1.01 to 1.02 or w/e it was.

    Obviously you also want to do the following things:

    1. Set the target to device instead of simulator and either universal, ipad, or iphone.
    2. Edit the scheme and set it to release.
    3. In the build settings set the provisioning profile to your app store version.
    4. Drag n drop in your icons and splash screens.

    After all of these things are set, you can do Archive, which will generate an ipa that should work for submitting to the app store.
