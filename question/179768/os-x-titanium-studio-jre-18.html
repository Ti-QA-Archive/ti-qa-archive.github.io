OS-X Titanium Studio JRE 1.8

I'm running Titanium Studio 3.4.1.201410281727on OS-X Yosemite.

Is it possible to run Studio on JRE 1.8?

I notice that if I change the TitaniumStudio.ini file to point to a 1.8 JRE using this setting:

-vm
/Library/Java/JavaVirtualMachines/jdk1.8.0_25.jdk/Contents/Home/bin/java

Then it will no longer startup. I get this error:

The JVM shared library "/Library/Java/JavaVirtualMachines/jdk1.8.0_25.jdk/Contents/Home/bin/../jre/lib/server/libjvm.dylib"
does not contain the JNI_CreateJavaVM symbol.

Basically it seems to have to run on the Apple Java, which is based on Java 1.6.

Curious if you can run Studio on 1.8 since I run all my other Java tools are using 1.8

Thanks for your advice.

Beren

    • Give this link a look;

      • Titanium Command-Line Interface Reference - SDK Install

      This may seem scary - but it is not. Run terminal and then a command like this at the prompt to update yourself;

      // optionally prefix with sudo
// ti sdk install [<version>] [--default]
ti sdk install 3.4.1.GA
// to make this the default SDK suffix the optional param
ti sdk install 3.4.1.GA --default

    • Moving things back up here. Cool so no need to lead you by the nose. Take a look here for the latest builds of everything. You can either use the CLI or using the Studio Help menu;

      • Titanium Mobile Continuous Builds

      The GA is in then - if you are feeling ambitious then the 3.5.0RC is also available there (or on the Appcelerator Blog) - this is the SDK version that supports 64 bit apps - you will need this come Feb 1st for brand new apps and a couple of months afterwards for any updates to existing apps.

      Keep in mind Ti SDK 3.5.0 really requires all iOS modules to be 64 bit. A good source of modules and their status can be found @ gitt.io - this is the 64 bit version list.
