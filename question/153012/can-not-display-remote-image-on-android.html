Can not display remote image on Android

I am developing a Titanium App for iOS and Android. The iOS app is working as expected. I access a local database that holds the name of the image I want to display. When I store the image locally, it displays fine. Here is a snippet of my code:

var clientlogo = Ti.UI.createImageView({
    image: 'images/'+rows.fieldByName('clientlogo')+'',
    width:  'auto',
    height: '150dp',
    top:    '1dp'
    });

However, I have many images and storing all of them locally will take up far too much space on the device, so I want to store them remotely. I have been unsuccessful in getting it to display with the following snippet:

var clientlogo = Ti.UI.createImageView({
    image:  'http://mydomain.com/app/uploads/thumb/'+rows.fieldByName('clientlogo')+'',
    width:  'auto',
    height: '150dp',
    top:    '1dp'
    });

The image displays properly when viewed through a web browser and I have re-uploaded the image just in case there was some kind of corruption happening to the file. Can someone please help me? I am using Titanium SDK 3.1.0, I am testing on an HTC EVO running Android 2.3.5

  • Hi

    There is nothing obviously wrong with your code, so lets rule some things out;

    • Confirm the URL you think is being called is actually the one that is being called, add this condole line;
    Ti.API.info('URL: ' + 'http://mydomain.com/app/uploads/thumb/'+rows.fieldByName('clientlogo')+'');
// review the URL to make sure it is what you thought
// then confirm this can be accessed on the device
    • Tweak the code just in-case the issue is dimensions;
    var clientlogo = Ti.UI.createImageView({
    image:  'http://mydomain.com/app/uploads/thumb/'+rows.fieldByName('clientlogo')+'',
    width:  Ti.UI.SIZE,
    height: Ti.UI.SIZE,
    top:    '1dp'
});
    • Obviously confirm that the actual domain is accessible via the device without the image url
    • If the mydomain.com is a local web server make sure the device has access to this local server
