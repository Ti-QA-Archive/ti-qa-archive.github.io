TableView data disappears after changing windows in NavigationGroup

I have a NavigationGroup setup with 2 windows. The second window has an HTTP client that loads JSON from a remote server, then adds rows and finally adds the tableview to the window. This works great and shows up the first time you load the window.

However, once you are on that page and click Back in the navbar, then once again return to table view page, all of the data is gone.

I have solved the problem by adding an event listener to the window for the focus event, but it doesn't seem like it should be truncating all of the tableview rows between window changes.

Am I doing something wrong?

    Hi,

    Whenever you go out a window, the data is discarded to avoid memory problems.
    When you return the array of data in tableview now is null.

    One way to avoid this is to create your array into a global variable (in the first screen or in app.js) and populate it with your data from remote server.
    Then populate your tableview with this array.

    Where is called the second window, you reassign the array to the date of Tableview like this:

    
myTableview.data = myArray;
myNav.open(win2);
