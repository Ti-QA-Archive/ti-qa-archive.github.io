TextAreas not rendering properly within a ScrollableView

I have an iPad application that contains a number of "pages" within a ScrollableView. Into each view of the ScrollableView I put a TextField, ImageView and TextArea for the user to enter data into - kind of like a basic book.

My problem is that this all works fine but for some reason the TextAreas don't render properly on the views outside of Index 0. As you can see from the screenshot here http://i.imgur.com/IAFF3.png - it renders the first couple of words in the TextArea but no more than that. If you focus the TextArea and type into it for awhile then it sort of kicks to life and shows up all the text again. This happens on both the simulator and on the device.

Here's a bit of code, it's pretty straight forward and is just looping some database records and adding a new scrollableView view with the TextArea in it.

  • I've applied Janne's suggestion for a workaround which resolves the problem for me. I have a view containing a ScrollView that contains the TextArea. Just before returning the main view I scroll the ScrollView 1 px.

    
function createMyView(){
// Build up the main view and components including the ScrollView containing a TextArea
    .
    .
    mainView.add(myScrollView);
    myScrollView.scrollTo(0,1);
    return mainView;
}
