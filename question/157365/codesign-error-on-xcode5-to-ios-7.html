CodeSign error on Xcode5 to iOS 7

Hello all,

After a update to new Titanium Studio Today, Alloy and SDK 3.1.3, i cannot compile any app.
whenever I try to compile, I get the following error message:

[ERROR] :  Error details: Code Sign error: No codesigning identities found: No codesigning identities (i.e. certificate and private key pairs) that match the provisioning profile specified in your build settings (“YOUR_APP”) were found.
[ERROR] :  Error details: CodeSign error: code signing is required for product type 'Application' in SDK 'iOS 7.0'

Someone is having this same problem and managed to solve?

  • Did you also update your xcode sdk 7.0? Also open xcode click window -> organizer , click provisioning profiles then on bottom right click refresh. This will download the latest certs and provisioning profiles from Apple developer portal.

    Also in Titanium Studio check Titanium Studio -> preferences expand > Studio > Platforms click iOS
    What sdk are you using? Since your Titanium Studio install is trying to build for ios 7.0.

    — commented September 19th 2013 by Zeshan Bilal

  • Hello Zeshan, thanks for the help, but I had already tried that.

    I just solve my problem as follows:

    REMOVING all mobileprovisioning and Certificates from my MAC and from Apple Dev Portal.

    I Recreate all again and works perfect.

    — commented September 19th 2013 by Adriano Paladini
