Enable customized white splash on touch event

in previous versions of the SDK it was possible to get feedback on click event listeners by showing a white splash where the touch event happened. Details of the splash are available here.

TIMOB-7034 has made it possible to disable the "splash". However I am now not able to Enable the splash.

Below is the code that shows the "splash" when touched in version 1.8.2 of the SDK. In version 2.1.3 the splash has gone. Any help would be soooo awesome.

var win = Titanium.UI.createWindow({
    backgroundColor:'#fff'
});  

var button = Ti.UI.createButton({
    width:207, 
    height:80, 
    top:0, 
    left:0,
    backgroundColor:'#000',
    color:'#000',
    backgroundImage: '/testButton.png'
});

button.addEventListener('click',function(){
    Ti.API.info('clicked button');
});

win.add(button);
win.open();

To disable the "splash" it says:

When both the "backgroundImage" and the "backgroundSelectedImage" are set in iOS button, the customized white splash for visual feedback should be disabled.

One would assume that by only including the backgroundImage that the "splash" would be enabled, but I have not been able to get the lovely "splash" to show unless I use SDK 1.8.2

Any help would be appreciated
