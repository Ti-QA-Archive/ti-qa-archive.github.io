      Yes I did finally get it to work .. SDK 3.1.2 titanium and android 4.0 at least (I don't remember if I tried it with anything higher)

      here is what you need to do
      get these files
      http://barcode-coder.com/en/barcode-jquery-plugin-201.html

      your going to have to create two files (one for android one for IOS because they handle things differently)

      i created a directory under assets called local
      there is a code39.js file in the files from the website.
      rename this to code39.jslocal
      (if you leave it with the JS ext. titanium will attempt to compile it with the rest of the program and it will not work).

      here is some of the code

      I made this file ANDROIDcode39.html

      
<HTML>
<HEAD>
<script language="JavaScript" src="code39.jslocal"></script>
</HEAD>
<BODY style="font-family:Verdana; font-size:11pt">

<script language="JavaScript">

function sku(barcodesku) {
    var skudisplay = barcodesku;

    document.open();
    document.write(skudisplay);
    Code39("","",60,10,skudisplay,0);
    document.close();
};


</script>

</BODY>
</HTML>

      and I named this file IOScode39.html

      <HTML>
<HEAD>
<script language="JavaScript" src="code39.jslocal"></script>
</HEAD>
<BODY style="font-family:Verdana; font-size:11pt">

<script language="JavaScript">




    document.open();
    document.write(sku);
    Code39("","",60,10,sku,0);
    document.close();



</script>

</BODY>
</HTML>

      While they look the same how android and IOS deal with it is different.

      I had to create a check to see what platform I was on in order to get to the right file

      if (Titanium.Platform.name == "android"){
var weburl = "/local/ANDROIDjscode39.html";
} else {
    var weburl = "/local/IOSjscode39.html";
};

      so to create the web view i did this 

      var barcode =Ti.UI.createWebView({
    top:"30%",
    left:"35%",
    height:"45%",
    width:"30%",
    url: weburl,
    backgroundColor: "white"
})

Ti.API.info("sku = " +barcodesku);

if (Titanium.Platform.name == "android"){

barcode.addEventListener('beforeload', function()
{    


    barcode.evalJS("sku('"+barcodesku+"');");
    Ti.API.info('sku after evaljs = ' +barcodesku);

});

} else {
    barcode.addEventListener('beforeload', function(){
        barcode.evalJS("var sku='"+barcodesku+"';");
    });
};

      so regardless of what platform they are on it will include the right html file.

      couple of things to note

      in the local directory you have to have
      code39.jslocal
      b.gif
      w.gif
      ANDROIDcode39.html
      IOScode39.html

      in the area where I have named it weburl (so that the web view gets the correct location of the phones in the directory it needs to look exactly how I put it in here to ensure cross compatibility.

      what you should be able to do is change the barcodesku variable to your text field variable and then it should populate. I did put in some error checking for the console in it (all the TI.API statements)

      ALSO * this is important to note. If you use the same machine for android and IOS programming when you switch from one emulator to the other you will need to load the program twice in order to see it work on both sides (took me an hour to figure that out) Android and IOS … otherwise you will get a sku var not defined problem.

      the code in the HTML files will also show you the barcode or the sku numbers sequence in blue at the top… if you take out the document.write(skudisplay) Android / document.write(sku) IOS in the html files you will get a pure bar code.
