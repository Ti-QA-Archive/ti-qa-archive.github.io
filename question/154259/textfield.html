  • Hi

    Androids have a particular issue with a textField inside a tableViewRow.

    Many people would simply use a scrollView with the layout set to vertical with child views taking the part of the tableViewRow.

    However these is something yu can try first to try and get Android to understand this is a normal request. Set the following property on the tableView in your associated tss file.

    • softKeyboardOnFocus

    If this does not work, then try adding it INSTEAD to the window in the tss file.

    If this fails then use the solution above that many others do.

    • If you are only having one tableViewRow you really should move to a solution without a table as I am not sure what benefit you are gaining from the tableView.

      You are MUCH better off with something simpler, use this as a starting point.

      *xml

      <ScrollView id="tablei">
    <Label id="labelp">Please Login</Label>
    <TextField id="email" hintText="Email" onReturn="closeKeyboard"/>
    <TextField id="pwd" hintText="password" onReturn="closeKeyboard" passwordMask="true"/>
    <Button id="Loginbutton" onClick="Loginbutton">Log in</Button>
    <Label id="labelf" text="link" onClick="openlink">Forgot Password?</Label>
    <Label id="labeld">Don't have an account?</Label>
    <Button id="signupbutton" onClick="signupbutton">Free Sign Up</Button>
</ScrollView>

      tss

      "tablei" : {
  contentHeight: Ti.UI.SIZE,
  contentWidth: Ti.UI.FILL,
  height: Ti.UI.FILL,
  showHorizontalScrollIndicator: false,
  showVerticalScrollIndicator: true,
  width: Ti.UI.FILL
}

      You can fill in the individual styles for the textfields, buttons and labels.

    • You really should not be putting textFields in tableViewRows in Android the two do not mix together well at all as you have experienced.

      You were only using one row in a table anyway so an ideal use of a tableView.

      If you need tableViewRows underneath or above where you require textFields, use a normal scrollView and place it as the headerView or footerView of the tableView - it gives you more control.

      Whilst tableViews may appear to give you lots of flexibility - you should only use then for the things they are meant for. If this was for iOS I could understand the wish to use the grouped view style with textFields, but Android doe snot have this, so the tableView and scrollView look the same anyway.

      Last point; a tableView is a scrollView with a few extra methods to it. - so they are basically the same thing anyway.
