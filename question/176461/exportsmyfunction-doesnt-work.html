    Hi

    Much like Allen; I read your code s working fine - it is after all a simple example of CommonJS. I have run the code and seen that it does work correctly;

    {appname}/controllers/index.js

    ...
$.btn.addEventListener('click', function (e) {
    var utils = require("utils");
    utils.prueba();
});
...

    {appname}/lib/utils.js

    exports.prueba = function(){
    console.log("Hello world");
};

    This works without failure.

    When the error returns undefined' is not a function (evaluating 'utils.prueba()') this means it is the left side that is at fault - so in the case utils is the thing it think is not defined. However it is - certainly is if my even listener triggered version is basically your example. If however you wrote the script something like this;

    // somewhere in the app
var utils = require("utils");

    Then

    // somewhere else in the app with NO scope to the 'utils' variable
utils.prueba();

    This would fail as shown in the error message. But I trust this is not how you have your code.

    So we are going to try an old school - "turn it off and on again" approach - this 'may' work.

    • Clean
      • In Studio
      • Menu
      • Projects
      • Clean
      • Select Project
      • Confirm
    • Backup your project

    This stage can ONLY be performed if this is a Titanium with Alloy framework - I see yours is. For everyone else ONLY if using Alloy - seriously ONLY if using Alloy

    • Locate the {appname}/resources folder
    • Remove the contents of this folder

    As this is Titanium with the Alloy Framework; the resources folder is used to cache the framework version of your app and is rebuilt on each compile. However it does not remove what was there before. So this 'housekeeping' should be done every once in a while and ideally between Android and iOS real tests.

    It is ONLY safe to remove the contents from resources because you do not code in here. Anyone not using Alloy MUST not perform this stage.

    This will now have the project fully cleaned and ready from the compile process - what happens now? I really hoping it all starts working.
