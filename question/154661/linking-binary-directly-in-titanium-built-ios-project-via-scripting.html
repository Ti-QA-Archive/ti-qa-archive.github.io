Linking Binary directly in Titanium built iOS Project via Scripting

I've built an iOS module that uses an external .a library. When I use the module in a project it doesn't link the .a library required by the library and therefore runs into a compile-time error. The work-around now is to open the Xcode project after Titanium Studio is done building and manually linking the binary .a library. Then everything works great! However this is cumbersome, is there any way to script the linking of the .a library? I've tried using the -framework parameter in OTHER_LDFLAGS but it doesn't seem to work for .a libraries.
