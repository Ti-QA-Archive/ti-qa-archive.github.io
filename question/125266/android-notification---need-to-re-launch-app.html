Android Notification - Need to re-launch app...

Hi,
I wrote a program that gives a notification to the device. I see the notification but when I click on it I need the App to open - I succeeded in doing that with the following code:

var intent = Ti.Android.createIntent({
                action: Ti.Android.ACTION_MAIN,
                flags: Ti.Android.FLAG_ACTIVITY_NEW_TASK | Ti.Android.FLAG_ACTIVITY_RESET_TASK_IF_NEEDED,
                className: 'com.povsw.com.TestingActivity',
                packageName: 'com.povsw.com'
            });
            intent.addCategory(Ti.Android.CATEGORY_LAUNCHER);

            // This is fairly static: Not much need to be altered here
            var pending = Ti.Android.createPendingIntent({
                activity: Ti.Android.currentActivity,
                intent: intent,
                type: Ti.Android.PENDING_INTENT_FOR_ACTIVITY,
            });

            var notification = Ti.Android.createNotification({
                contentIntent: pending,
                contentTitle: 'New message',
                contentText: e.data.message,
                tickerText: "New message"
            });

            Ti.Android.NotificationManager.notify(1, notification);

This code is placed inside

callback:function(e)

for google c2dm service.

The problem is when the application is close (I close it from the device task manager). What happens is that when a message is received the device pops up an alert that the application (with my app id) encountered an error and needs to be closed.

I need to be able to receive a notification and start the application when the notification is clicked even if it's close.

How can I do that?
Thanks
