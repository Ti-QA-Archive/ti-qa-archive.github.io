Application size in Titanium

In my Titanium Application I need to use large no of image files and sound files(each 500). The maximum file size will be 70kb. Is it possible to use large no of image and audio files in Titanium. Whether it will cause any problem during deployment?

I need to develop this App for both iOS and Android.

Any suggestion welcome.

    If you had 500 image and 500 sound files, and each approached 70KB then you'd have up to 70MB of media files, not counting the application. Although technically it could be compiled in Titanium (at least AFAIK), you would run afoul of app store guidelines. Or at least Apple limits app store sizes to 20MB last I knew. Even if 70KB was the largest, you'd have to average well under 20KB to fit under the Apple guidelines.

    If you do not install via the app store/Google Play (for example using an Apple Enterprise certificate) then it may technically be doable. But you also need to consider that anything you bundle will be part of the Resources folder (or a sub-folder), and on the device this is read-only. That means that your app would have to use them directly from the Resources folder. If you copied them to another location you'd essentially use twice the space on the device, because you cannot delete from the Resources folder once copied because the folder is read only.

    I'd highly recommend NOT bundling more than a core sampling of media files – if that – and make the user download them (or a subset of them) upon first use of the application. That keeps your install package size within submission requirements, and lets you put them in the folder of your choice instead of needing them to be in the Resources folder or below.

    YMMV

    • Yes, that is very possible and in fact is what I was trying to suggest when I said "make the user download them (or a subset of them) upon first use of the application". The only "problem" it could cause would be if the user never tried to launch your program until after they no longer had internet connectivity.

      When your application starts, check if you have downloaded the files yet. If not, put up a message saying they must be downloaded and give an approximate size and suggest they may want to use wifi (or 4G-LTE) and not 3G. I'd give them the option to not continue with the download at that time.

    Looks like for Google Play it's 50MB for the apk + 2 * 2GB expansion files. But it doesn't look like Titanium supports expansion files yet.
