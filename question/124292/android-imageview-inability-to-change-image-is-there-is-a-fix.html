Android ImageView inability to change image: is there is a fix?

I'm running into an issue where changing the image on an imageView that sits in a tableRow doesn't cause the displayed image to change:

var myImageView = Titanium.UI.createImageView({
    image: 'images/camera.png', width: 45
});
myRow.add(receiptView);

myImageView.image = 'images/settings.png';
myImageView.height = 300; myImageView.width = 300;

This works fine on the iPhone, but on the device on Android the image never changes from camera.png to settings.png .

I did some searching and found a few references to similar issues. One thread suggested using the CI 1.8 build, which I did to no avail.

Is there a fix or work-around for this?
