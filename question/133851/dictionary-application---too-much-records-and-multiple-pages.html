Dictionary application - Too much records and multiple pages

Hello all, i'm developing a dictionary application. The first page "WelcomePage.js" contains the tabs and each tab is attached to a window and every window has a URL.
So per default, the first tab is enabled, and "allwords.js" is the page where tableview of 8000+ records is loaded. This is the code of the link between the first tab and all words.js :

var main = Ti.UI.createWindow({  
    title:'Dictionary',  
    url:'allwords.js',
   // tabBarHidden: true  // ONLY FOR IPHONE AND IPAD
});  

// create main tab  "Dictionary"
var tab = Ti.UI.createTab({  
    icon:'images/tabs/32-32px/dictionary_icon.png',  
    title:'Dictionary',

    window:main  
});

When the user clicks on a word from the tablebview, the page "meanings.js" will be opened where the meaning of the clicked word will be shown. The problem is that when I click on a word, and the meaning will be show, and then click back again, this takes so much time because "WelcomePage.js" will be loaded again. Anyone can help me to let this page always loaded and when we press back, the user can access the words rapidly? 10x all

    Multiple context apps using the url property will get re-evaluated each time they are loaded. the best thing would be not to use multiple contexts (even the wiki states this). However, I think you can place the data dictionary within the context of the main app.js and it will be available to the other contexts because it is "global"
