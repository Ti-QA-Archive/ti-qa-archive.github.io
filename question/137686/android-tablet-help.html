Android Tablet Help

I'm having some issues getting my application to look correct on Android Tablets. The first issue is that compatibility mode is always on, even if i take the anyDensity tag out of the tiapp.xml. This is a big issue and i'm not exactly sure how to fix it and let android know that this is a tablet app as well. I'm also unsure of what folder the resources need to be in for android to recognize the resources for a screen size of over 1024x768.

Currently i have a resources folder of:
res-large-land-mdpi
res-large-port-mdpi

I've tried this, but get errors:
res-xlarge-land-mdpi
res-xlarge-port-mdpi

My Manifest looks like this:

<android xmlns:android="http://schemas.android.com/apk/res/android">
        <manifest>
            <supports-screens 
                android:anyDensity="true"
                    android:xlargeScreens="true"
                    android:largeScreens="true"
                android:normalScreens="true"
                android:smallScreens="true"
                />
            <tool-api-level>9</tool-api-level>
        </manifest>

And i get this error:
Error generating R.java from manifest
error: No resource identifier found for attribute 'xlargeScreens' in package 'android'

  • This is what i ended up doing to get my app to work. Here is the code for the tiapp.xml

    <android xmlns:android="http://schemas.android.com/apk/res/android">
        <tool-api-level>15</tool-api-level>
        <manifest android:installLocation="auto" android:versionCode="6" android:versionName="1.3">
            <uses-sdk android:minSdkVersion="4" android:targetSdkVersion="15"/>
            <supports-screens android:anyDensity="false"
                android:largeScreens="true" android:normalScreens="true"
                android:smallScreens="true" android:xlargeScreens="true"/>
        </manifest>
        <icon>appicon.png</icon>
    </android>

    I have the following android image specific folders in my resources –> android –> images folder

    • Unordered
    • res-large-land-mdpi
    • res-large-port-mdpi
    • res-long-land-hdpi
    • res-long-land-ldpi
    • res-long-port-hdpi
    • res-long-port-ldpi
    • res-notlong-land-hdpi
    • res-notlong-land-ldpi
    • res-notlong-land-mdpi
    • res-notlong-port-hdpi
    • res-notlong-port-ldpi
    • res-notlong-port-mdpi
    • res-xlarge-land-mdpi
    • res-xlarge-port-mdpi

    This is not an all inclusive list of the resolutions supported and you may not need all of these for your app. But with the combination of files and manifest settings, this worked for my phone/tablet app.
