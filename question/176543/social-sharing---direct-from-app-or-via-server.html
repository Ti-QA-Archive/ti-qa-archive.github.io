Social sharing - Direct from app or via server?

Just a curiosity for now, but if I wanted to implement social sharing options for various sites such as Facebook, Twitter, Google+, Pinterest, etc., would it be better to implement that functionality directly into the app or to offload it to a remote server and send requests from the app using an HTTPClient?

Naturally sharing via a remote server might increase the response time, but the app is already set up to interface with a CodeIgniter application, so the framework in which to build a simple OAuth library is already in place. On the other hand, building it into the app might provide a quicker end-user experience but may be more troublesome to implement and maintain.

Are there any significant advantages or disadvantages for either approach?

  • When you add a server share there is an extra step the user has to do: authorise your app to access the social service. If your app doesn't need facebook or twittter to work (so the user is already logged in) you will lose a lot of potential posts - the user is lazy or he doesn't want to give you access.

    Sharing from device is faster and seamless - iOs has fb/tw already baked in, on android you can use intents.
    The drawback - for other networks you need to find/build the sharing part yourself and you won't be able to control the text - on server you can always add "via myAwsemeApp" :)
