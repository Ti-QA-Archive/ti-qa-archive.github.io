  • @Darren Wallace

    I was having the same issues last night (same error on build, even same line of code in the .py files)

    What took hours to find in the end was that I had mucked up my provisioning profiles. In the Apple Dev center, it showed my profile as having a generated code before my ID, e.g: ZKSJNFVSKRBG.com.appname.app.

    I then used that ID in Titanium (because that's what Apple Dev Center claimed it was). I decided to change it back to what I originally entered i.e. com.appname.app, then:

    • deleted build/iphone folder
    • clean project
    • rebuild to iOS Device

    And suddenly it worked perfectly…

  • Problem Solved

    Well finally identified the problem, and it's not where I expected it to be. I was packaging some web content as part of the app. The web content had it's own js library which included modernizer.js. Now modernizer wasn't actually called in any page or script so I never expected it to cause a problem. But obviously Titanium must evaluate every file in some way during build and modernizer was doing things it didn't like. I deleted it from the folder and everything's working great.

    Phew!

    Is this a gotcha worth posting somewhere? I know the goal of Titanium is to do more than just package web content but we're converting our web stuff to native in a phased plan, which I'm guessing is quite a common development path. Worth identifying where web js can interfere with the titanium project build.
