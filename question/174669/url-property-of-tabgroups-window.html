URL property of Tabgroup's Window

Why can't Tabgroup's Window set URL property ? My both js file located in controller folder.

<Alloy>
<TabGroup id="tab1">
    <Tab id="findtab">
        <Window id="window1" url="home.js"></Window>
    </Tab>
    <Tab id="tab2">
        <Window id='window2' url="home2.js"></Window>
    </Tab>
 </TabGroup>
</Alloy>

    Hi

    Do you mean this windows.url property?

    If so - then this is not how you do things in Alloy. That creates an old school multiple context app - that is no longer the accepted pattern.

    Tweak as follows;

    index.xml

    <Alloy>
<TabGroup id="tab1">
    <Tab id="findtab">
        <Require src="home" id="window1" />
    </Tab>
    <Tab id="tab2">
        <Require src="home2" id="window2" />
    </Tab>
 </TabGroup>
</Alloy>

    home.xml

    <Alloy>
    <Window id="window1">
        <!-- stuff -->
    </Window>
</Alloy>

    home2.xml

    <Alloy>
    <Window id="window2">
        <!-- stuff -->
    </Window>
</Alloy>

    This allows the separation of code into additional controllers, views and tss files.

    Require takes a valid controller set and inserts it correctly inside the structure you have created.

    NEVER use the .js file extension inside any src entries, not required and will not work anyway.
