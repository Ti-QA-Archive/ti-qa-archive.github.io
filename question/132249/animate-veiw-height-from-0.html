Animate Veiw Height From 0

I'm having serious issues with what should be a very straight forward problem in Titanium Mobile 1.8.1.

I have a view with height 0. I then wish to animate that view to open up and display the contents. If this was jQuery, I'd be able to use .slideDown(), or .animate({height: 0px}) with no trouble. For whatever reason, I'm unable to recreate this effect in Titanium Mobile?

When the height either starts as 0, or ends up that way after an animation from open to close, I'm unable to get the height to anything from 0. The only way I'm able to even approximate this is using a height of 1, which then causes the view to slide to the top of its container and is not a height of 0 which is what I ideally need?

Surely I can't be the only one to try and recreate jQuery's slideDown animation?

Any/all help is appreciated!

  • Solved it! I need to animate the top to it's original position, which in my case is 0 as I'm using layout vertical. This then animates the height of the box and keeps it where it is at the same time!

    wrapper.animate({
    height: fullHeight
});
wrapper.animate({
    top: originalTopPosition
});

  • Just a quick update for whoever may stuck in this problem.

    If you have other element whose position depends on the one to be collapsed, you may find that element suddenly jump to its final position without moving gradually with the one performing animation.
    That's because animation will not actually check the property of the element(if you check the element.height, you will find it doesn't change that value, even the animation finishes!!) So other elements' position will be reset at the moment the animation starts.

    I ended up completely rewrite my own animation function using setTimeout, instead of using animation function… Hope there will be better solution for it though.
