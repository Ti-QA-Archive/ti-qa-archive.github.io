<Module> xml markup. "has no method createView"

App Type: Android SDK: 3.2.0 Device: Galaxy S3 i9300 Version: Android CM 4.3.1 Host: OSX 10.9.1 Alloy

I'm trying to use appcelerator barcode module within Alloy (without, it's ok)

<Module id="barcode" module="ti.barcode"></Module>

Reading the documentation, I can see that Titanium always call the createView method (I don't know why), but, lookes like barcode module doesn't have this method implemented, any workaroud?

    As described in the docs on the modules tag, you can specify the module method to call by adding the method attribute to your tag. For example:

    <Module id="barcode" module="ti.barcode" method="createModalBarCodeWindow" />

    (Hopefully obvious, I made up that method call name. Check the barcode module's docs for the actual method name used to open the modal scanner window.)

    If you don't specify a method name, createView() will be used by default. Of course, as Manuel said, you can do this all in the controller instead.
