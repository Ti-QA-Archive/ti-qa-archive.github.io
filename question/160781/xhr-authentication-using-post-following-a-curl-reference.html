xhr authentication using POST following a cURL reference

mobile application using Titanium 3.1.3 for iOS

Hello everybody,
I need to authenticate on a webservice but the backend guy only gave me a example of how to connect
using curl. I'm new jn the authentication stuff and need some help. This is the curl:

curl -v -X PUT http://informedbynature.org/apiauth/index -d'{"email":"raajphp@gmail.com","password":"test123"}'

If I just copy/paste this curl on my terminal it retrieves me the (right) message:
{"success":true,"app_id":"APP_ID","api_key":"API_KEY","message":"Please use application id and API key to access our science fair project API"}

Otherwise in my titanium project I'm trying to run the xhr this way


    var xhr = Titanium.Network.createHTTPClient();
var url = 'http://informedbynature.org/apiauth/index/';
xhr.open('POST',url);

xhr.onload = function(){
    Titanium.API.info(this.responseText);
};

xhr.send({"email":"raajphp@gmail.com","password":"test123"});

This is what I get:
{"success":false,"message":"Either email address or password provided by you is incorrect."}

I noticed that if I try to run the curl without the "-d" command on URL it doesn't work. I already did search it on the google but can't find what this -d command means. I'm wondering how to input this on the url using Javascript.

Thanks for your attention guys!

Best regards

    try 

    
var xhr = Titanium.Network.createHTTPClient();

xhr.onload = function(){
    Titanium.API.info(xhr.responseText);
};

var url = 'http://informedbynature.org/apiauth/index/';

xhr.open('POST',url);



xhr.send(JSON.stringify({"email":"raajphp@gmail.com","password":"test123"}));
