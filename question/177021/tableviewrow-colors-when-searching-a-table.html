TableViewRow colors when searching a table

Developing in Titanium Alloy on iOS, I have a tableView with a table search bar and many TableRows. My rows have alternating background colors. When searching, the table filters out the rows that don't match a search, sometimes this messes with the alternating row colors and will put two rows with the same colors next to each other.

I developed a method that turns all the rows white when a search is being performed to solve that problem. The problem is that the method iterates through each row in the table and sets its background color to white. The method works great and even sets the alternating colors back after the search is cleared. The problem arises when I have ALOT of rows, 100+ or so, when the search box is focused and the method is fired there is a delay before the user can type in the search term because the method is busy going through each table row and setting its colors.

Is there a better way, something possibly built in or just a better way to handle it? Thanks in advance !
