Adding same button with different attributes multiple times to the same window only last one shows

When I add the same button with different attributes multiple times to a Window only the last button added shows. Not sure what I'm doing wrong. Android emulation. Code below demonstrates problem:

button_postit_00 = Titanium.UI.createButton({
        font:{fontFamily:'Marker Felt'},
        color:'#31041D',
        backgroundImage:'../images/postit_00.png',
        backgroundSelectedImage:'../images/postit_hilite_00.png',
        height:32,
        width:32,
        title:'1',
        top:33,
        left:39
    });


win.add(button_postit_00);

button_postit_00.title = '2';
button_postit_00.top = 33;
button_postit_00.left = 79;
win.add(button_postit_00);

button_postit_00.title = '3';
button_postit_00.top = 33;
button_postit_00.left = 119;
win.add(button_postit_00);

  • Its because its the same button over and over. Just because you change attributes doesn't make it a different button. You need to create a factory function that creates a new button for each instance of the button you want to show.

    Sidenote: please use the markdown code to make posts more readable:

    function createBtn() {
    var button_postit_00 = Titanium.UI.createButton({
        font:{fontFamily:'Marker Felt'},
        color:'#31041D',
        backgroundImage:'../images/postit_00.png',
        backgroundSelectedImage:'../images/postit_hilite_00.png',
        height:32,
        width:32,
        title:'1',
        top:33,
        left:39
    });
    return button_postit_00;
}

win.add(createBtn());
win.add(createBtn());
//etc, etc

    Of course this needs to be modified a bit so that you can track your buttons and pass in attributes if you want. But you should get the gist of it here.

  • Hi Martin

    When you set the properties of button_postit_00 individually, you are simply modifying the original object:

    button_postit_00.title = '2';
button_postit_00.top = 33;
button_postit_00.left = 79;

    Then, when you add the object to the window for the second time, you are actually adding the exact same object, which the window add() method will not allow you to do:

    win.add(button_postit_00);

    You can't really "reuse" objects in this way, but you can remove them from one view, add them to another and modify their properties (as you have) to make them appropriate to their new visual context. Whether this is really necessary is dependent on your situation.

    I hope this clarifies it.
