Problem with building package for android store (maps)

Hi there!

Preinfo:

After developing so much titanium, finally I got my first mobile app finished. It is already in the iOS and Play Store. Unfortunatelly I didn't use the built-in packaging mechanism of Titanium Studio for building the android package. (But I did it for iOS). I just searched for my APK after I built it for running the app, and submitted this apk to the play store.

The problem:

It is not in production mode, it is in development mode.

So I tried out the Package -> Android App Store functionality and it generated me also an valid APK.
Opening the app on my Android device showed me a blank (white) google maps view. So something waswrong with the signatures or so for the map. Maybe because of a newly generated keystore? Or maybe because of the target version is not the typical google apis version (I created a new run configuration and selected it, but didn't work too).

The Queston:

How to fix this issue, because I want to built the package in production mode, otherwise I'm getting some Titanium Error Windows (with the red background) or some alerts.

Sources:

I followed http://docs.appcelerator.com/titanium/3.0/#!/guide/Distributing_Android_apps.

Thanks for the help.
Emre

  • Ok, after trying it out again, this solved my problem

     ti build -p android  -T dist-playstore -K ~/Desktop/dev_keystore -P tirocks -L tidev -O ~/Desktop/

    (I copied the built-in titanium dev_keystore to my Desktop to have easier access to it)
