"Assertion failure" at secondary tableSection. when I call .deleteRow()

var table = Ti.UI.createTableView({
    data:[ {title:'Row 1'}, {title:'Row 2'} ] 
});
$.testView.add(table);

function createTableSection(){
    var tableSection  = Ti.UI.createTableViewSection({headerTitle : "dd"}); 
    for (var i = 0; i < 10; i++) {
        var tableRow = Ti.UI.createTableViewRow({
            title : "row  "+ i
        });
        tableRow.addEventListener("click", function(e){
            Ti.API.info("index: " + e.index);
            table.deleteRow(e.index);
        });
        tableSection.add( tableRow  );
    }
    return tableSection;
}

$.testButton.addEventListener("click", function(e){
    Ti.API.info("testButton clicked");
    table.insertSectionAfter(0, createTableSection() );
});

when every click "testButton", table section insert to the table. good.
and every click table row, that row will be deleted. good.
but "table.deleteRow(e.index)" will work only last time added section's row.
at other section's row, assertion failure occur.

[4992:c07] *** Assertion failure in -[UITableView _endCellAnimationsWithContext:], &#x2F;SourceCache&#x2F;UIKit_Sim&#x2F;UIKit-2380.17&#x2F;UITableView.m:1070

for example, you click button twice. section added twice.
and then click first time added section's row. assertion failure will occur. (instead of deletion of row)

**i'm using Titanium SDK 3.0.0.GA with iPhone
