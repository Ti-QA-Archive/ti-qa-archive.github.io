Module licensing issue

Hi all,
I am having licensing issues with titanium detecting an unpaid module. I currently have 3 modules in my app

  • Google maps for iOS (Paid)
  • ImageFactory (Open Source)
  • Star rating module (Paid)

I downloaded the first two off my account and the last off a colleagues account. (We both needed the same module for different apps and were paying with the same card).
In the past I have read to change the guid in my ti.app.xml to be the same as the guid in the module's manifest, however, since I have 3 modules, this cannot be the solution as I can only use one of the module's guid.

I also read to use a random guid generator online to solve the issue, but this hasn't worked fully either. It gets rid of the error initially, but it resurfaces after a couple of days.

How can I fix this issue? Any help appreciated

  • NO NO NO NO

    You ONLY change the GUID of the MODULE IF the module WAS in the marketplace and THEN was open sourced and the author didn't change the guid themselves.. You should NEVER EVER EVER change the GUID of a PURCHASED module.

    I am not sure where this errant information is coming from.

    Second, if by change you purchased a module after you started a project, SOMETIMES the new module license isnt applied to your project (contact support) and you MIGHT be able to work around this by changing the GUID of your PROJECT and fooling the system into seeing the project as new and applying the proper licensing to it.

    IF the module was bought on your coworkers account, you must login to ti as the coworker, work on a project created by the coworker.

    • account + purchase + module id + app id combine to allow you to use an app, in your project, with your account.

      purchased modules

      Purchased module can only be used when building under the account it was purchased for and (near as I can tell) for projects created by that account.

      Purchase all modules under one account, and if working on a shared/common project, both devs log in as the same user.

      Sometimes you need to email support to have the licensing reset.
