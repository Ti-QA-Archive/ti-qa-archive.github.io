    Hi

    This is not meant as a solution but a potential way to test your tests as if it was a store delivered app.

    Create an adhoc certificate and compile for adhoc (rather than app store). The resulting app "should" be the same as one that goes through the app store compile process.

    • I am assuming you can at least compile for iTunes - if not then you this will not help you.

      If you can compile to iTunes then do so and then;

      • Locate the app in iTunes
      • Right click the app and choose Show in Finder
      • Click the filename so you can edit it
      • Rename ipa to zip
      • Agree to the confirmation for this action
      • Double click the newly typed zip file (or choose open)
      • A single file will be displayed, right click this
      • Select Show Package Contents

      This will show you everything inside the ipa file that has been compiled.

      The problem is however that this shows you the development version and not the app store version - there may be some differences.

      It would be worth starting the simulator and using the Reset Content and Settings… option from the iOS Simulator menu. Keep in mind this will RESET all properties, remove apps and wipe the simulator back to empty - only do this if you are happy that nothing will be there afterwards.

      Once you have rest the simulator run the app again - this will now have no previous installs in the simulator that may previously have pushed files or settings that right now may be causing false positives on file and setting entries.

      Obviously if you have no access to the certificates for distribution - you cannot compile for distribution - there is no real way around that.
