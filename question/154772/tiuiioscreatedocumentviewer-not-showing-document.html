Ti.UI.iOS.createDocumentViewer not showing document

I am trying to use the createDocumentViewer to display file. The filename does appear at the top of the window, but the file contents are not displayed. I have tried this with both a pdf and a text file. I am not sure what form the url parameter takes, i.e. is it "file:///file.pdf" or or "app://file.pdf" or just "file.pdf" or does it need to be a fully qualified path? When I use the fully qualified path (returned from file.resolve()) I get an error in the console:

[INFO] : Couldn't issue file extension for path: /Users/daryl/Library/Application Support/iPhone Simulator/6.1/Applications/54ADDE9C-E072-47F6-9E88-C9C4CD1FAE96/DocAthalon.app/Users/daryl/Library/Application Support/iPhone Simulator/6.1/Applications/54ADDE9C-E072-47F6-9E88-C9C4CD1FAE96/Documents/uftest.log.txt

I'm not sure what that means. If someone could explain it, I sure would appreciate it. I'm also wondering what type of documents a DocumentViewer can show?

My environment is as follows:

Application type: mobile

Titanium SDK: version 3.1.1.GA

Platform: iOS, API 6.1

Device: iPhone emulator

Host OS: Mac OSX 10.8.4

Titanium Studio, version 3.1.1.2013061122350

Any help would be greatly appreciated.

Thanks,

Daryl

  • This worked for me:

        var file = Ti.Filesystem.getFile('Library/Application Support/iPhone Simulator/7.0.3/Applications/276C4837-4618-4040-908C-738036E92C80/Documents/SomeFile.pdf');
    var docViewer = Ti.UI.iOS.createDocumentViewer({
        url : file.nativePath
    });
    docViewer.show();
