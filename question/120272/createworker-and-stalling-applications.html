createWorker and stalling applications

Hi there,

I have a very simple question in relation to how web workers operate in the 1.1 version of Titanium Desktop. Say I have code which looks like this (obviously a contrived example):

        var worker = Titanium.Worker.createWorker(function() {
            var _sleep = function(millis) {
              var date = new Date();
              var curDate = null;
              do { curDate = new Date(); }
              while(curDate-date < millis);
            };
            _sleep(20000);
        });
        worker.start();

I'd expect this worker to just stall and not affect the rest of my JavaScript execution. However, when I execute this code, my application becomes unresponsive, my cursor beachballs and ActivityMonitor shows the application chewing up 100% of my CPU. I've tried externalizing this sleep method into it's own separate .js file, but I still get the same result.

Am I not using the createWorker API correctly, or do I have a unrealistic understanding of how web workers actually work?

Sean
