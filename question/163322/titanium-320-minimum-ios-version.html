    • Check this real-time chart for iOS version installed base.

      • iOS Adoption

      Whilst you can still submit apps targeted at iOS 5 it is no longer considered a main stream OS version with on average currently less than 2% of the share. Whilst that does mean that percentage actually translates into a larger number of devices - it also means those devices cannot access more modern API sets.

      So you can use an older Ti SDK for older apps and a newer one for current iOS apps (as Nick points out) you will have to branch your code into two development versions.

      The app store will provide an earlier version of your app to those who do not have the later OS versions that would allow newer versions it does mean you need to be smart about your app if it requires external API calls. If your app uses external API calls you provide then you should change the end point of each main app versions to another URL or path within a URL - otherwise older apps will conflict with changes to newer APIs you create - just worth noting.
