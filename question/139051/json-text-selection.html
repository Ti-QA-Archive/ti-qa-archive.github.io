  • Assuming your label is added and sized correctly to view, you would want to stringify the JSON object first.

    var client = Titanium.Network.createHTTPClient();
client.onload = function() {
    var jsonObject = JSON.parse(this.responseText);

    // See the result
    locationLabel.text = 'Response=' + JSON.stringify(jsonObject);
};

client.open("GET", "http://api.wunderground.com/api/45ee78ced37759a1/geolookup/conditions/forecast/q/CA/San_Francisco.json");
client.send();

    Though it would be such a big object might be more readable in the console

    Ti.API.info(JSON.stringify(jsonObject));

    And if you truly just want to see the response, you could just output the responseText.

  • Apologies in advance for posting A LOT of code, however, this code has always let me know what I am doing wrong when it comes to createhttpClient.

    I used your parameters toward the end of the code and only provided output to the console based on a good/valid response. What you do with the data is now up to you.

    I tested this code on a iphone and android simulator. Again, this code only puts data out to the console. You need to add your own code to parse it and/or display it in your app. The code you need to add/modify would go in to the onload method.

    The response I get in the console with this code shows me the entire process and server responses of the app interacting with the server and then finally the response. The response is the same as when I copy and paste your url in the browser.

    Before copying and pasting this code in to your app, I would suggest reading through it to gain a better understanding.

    I hope this helps.

    
var xhr = Ti.Network.createHTTPClient({

    /*this.responseText holds the returned payload as raw text
    this.responseXML holds the payload as an XML document instance
    this.responseData holds the payload as a BLOB (binary data)*/

    onload: function(e) {
    // onload function called in readyState DONE (4)

    Ti.API.info(this.responseText);

    }, // end of onload

    onerror: function(e) {
        // function called in readyState DONE (4)
        // bad url caught here or no web server response ( if this.status = 0 then display timeout error)

        Ti.API.info('onerror called, readyState = '+ this.readyState);
        Ti.API.info(this.responseText); // contains any text data that was returned with your error response
        Ti.API.info(this.status); // contains the returned status code of the error.
        Ti.API.info(e.error); // contains additional error information included in the event object e.
          // capture this.status = 401 Authentication needed
          // capture this.status = 500 Internal Server Error
          // test for other errors too

          var alrt_Window = Titanium.UI.createAlertDialog({
        title: 'ERROR!',
        message: 'this.status = ' + this.status + ' this.responseText ' + this.responseText + ' e.error ' + e.error,
        buttonNames: ['OK'],
        });

    }, // end of onerror

    ondatastream: function(e) {
        // function called as data is downloaded
        Ti.API.info('ondatastream called, readyState = '+this.readyState);
    }, // end of ondatastream

    onsendstream: function(e) {
        // function called as data is uploaded
        Ti.API.info('onsendstream called, readyState = '+this.readyState);
    }, // end of onsendstream

    onreadystatechange: function(e) {

        /*
        UNSENT (numeric value 0) – The object has been constructed. Titanium doesn't report on this state with the onreadystatechange handler.
        OPENED (numeric value 1) – The open() method has been successfully invoked. During this state request headers can be set using setRequestHeader() and the request can be made using the send() method.
        HEADERS_RECEIVED (numeric value 2) – All redirects (if any) have been followed and all HTTP headers of the final response have been received. Several response members of the object are now available.
        LOADING (numeric value 3) – The response entity body is being received.
        DONE (numeric value 4) – The data transfer has been completed or something went wrong during the transfer (e.g. infinite redirects).
        */

        switch(this.readyState) {
            case 0:
                // after HTTPClient declared, prior to open()
                // though Ti won't actually report on this readyState
                Ti.API.info('case 0, readyState = '+this.readyState);
            break;
            case 1:
                // open() has been called, now is the time to set headers
                Ti.API.info('case 1, readyState = '+this.readyState);
            break;
            case 2:
                // headers received, xhr.status should be available now
                Ti.API.info('case 2, readyState = '+this.readyState);
                Ti.API.info('Headers: ' + this.responseText);
                Ti.API.info('Headers Received: ' + this.HEADERS_RECEIVED);

            break;
            case 3:
                // data is being received, onsendstream/ondatastream being called now
                Ti.API.info('case 3, readyState = '+this.readyState);
            break;
            case 4:
                // done, onload or onerror should be called now
                Ti.API.info('case 4, readyState = '+this.readyState);
                //if a session doesn't exist, the following statement will get session data in the header
                //if the same session exists, the following statement will return undefined
                //if a new session is started, the new session data will be sent

                var cookie = xhr.getResponseHeader("Set-Cookie");
                Ti.API.info('cookie-' + cookie);
                Ti.API.info('Headers - ' + 'Content Type - '+ xhr.getResponseHeader("Content-Type")
                 + ' Date - '+ xhr.getResponseHeader("Date") + ' Vary - '+ xhr.getResponseHeader("Vary")
                 + ' Content Encoding - '+ xhr.getResponseHeader("Content-Encoding")
                 + ' Content Encoding - '+ xhr.getResponseHeader("Connection")
                 + ' Content Encoding - '+ xhr.getResponseHeader("Content-Encoding")
                 + ' Content Encoding - '+ xhr.getResponseHeader("Transfer-Encoding"));
            break;
        }; // end of switch

    }, //end of onreadystatechange

    enableKeepAlive:false,

    }); // end of createHTTPclient

    xhr.setTimeout(5000); // in milliseconds

    xhr.validatesSecureCertificate = false;


    xhr.open("GET", "http://api.wunderground.com/api/45ee78ced37759a1/geolookup/conditions/forecast/q/CA/San_Francisco.json");
    xhr.send();

});

  • Hi to all,

    The reason the label text is not changing is due to a synchronical issue, you are trying to change a label's text while running a xhrClient, it seems fine, but what's the problem. At runtime your application runs all of javascript's code no matter what's the order, so when you call the send method on the xhr, the label has changed to the original value of this.responseText.

    To make it simple, you need to add an event listener inside the xhr so you can return data to a view. Check this code: 

    var xhr = Ti.Network.createHttpClient({
    onload: function(e){
        var foo = this.responseText;
        Ti.App.fireEvent('customEvent',foo:foo);
    }
});
xhr.open('GET','YOURWEBPAGE');
xhr.open();

Ti.App.addEventListener('customEvent',function(e){
    Ti.API.info(e.foo);
    // Now you can use foo to change the Label Text
});

    What's happening here? We are running a segment of code exactly after we have the information on our variable, this way you prepare the info and send it synchronously.
