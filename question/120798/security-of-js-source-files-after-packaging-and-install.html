Dear all,

  • You can't store credentials in the app. For example in the case of android, the .apk file is just a zip file. Anyone can unzip it and see your app.

    http://stackoverflow.com/questions/2421572/is-there-any-facility-to-protect-the-contents-of-an-apk-from-user-modification

    What you have to do is run some type of proxy to the webservice on a webserver that only you (and people you trust) have control over. Then in your mobile app, connect to your server instead of directly to the webservice that needs credentials.

    Then you have to have some type of authentication between your proxy and your mobile app so not just anybody can use your proxy. Perhaps by username/password, or giving the device/user a client certificate.

    It is impossible to store the credentials to the third party service on the device with 100% security.

  • Here is what I would recommend:

    1. Send the username / pass over https.
    2. From the server respond with either false or a token value.
    3. Store the token on the device, use the token value from there on to authenticate any action for all other web services. You could also set an expire time on the device & server.
