How to detect Memory Leaks on iPhone?

Hi there,

I wanted to know is there an efficient way to detect the memory leaks with Titanium Mobile 1.6.2 for iPhone iOS 4.2.x / 4.3.x.

The result between an analysis with Instruments, and the tracking of Ti.Platform.availableMemory variable are kind of different so I really don't know if we are leaking memory or not.

Instruments doest not find anything irregular (with the Allocations, Leaks and Zombies analysis).
But Ti.Platform.availableMemory makes us really worried since the app seems to take more a lot of memory (~150Mo on iPhone4)

Not to mention that the display of the available memory via SBSettings on jailbroken iphone is also different… But kind of close of the analysis of Ti.Platform.availableMemory.

So I hope someone can tell us can give us reliable informations about the correct way to manage memory use with those UI components which seems leaky:

  • Tableviews with background Image and thumbnails.
  • Full image on scrollview (like photo gallery)
  • MapView (which seems to be the most leaky component of our app)

Regards
