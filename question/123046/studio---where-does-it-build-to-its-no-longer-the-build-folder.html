Studio - where does it build to? Its no longer the build folder?

Hi,

Maybe i am not used to the studio tool, but i have noticed an issue a few times in the last few weeks when using it.

When i build an APK file with studio, the apk file in the build folder does not get updated.

Previously with developer, i did a build of the app and then in there was an apk file ready to share for testing.
Now, i do the build in studio, see that my changes are shown in the app (they are) then i look in the build folder but the apk file still have an old timestamp.
It was first shown to be by a colleague who I sent the apk file to, only for them to say no changes were there, yet the build i looked at in the emulator was clearly showing the latest changes.

Any ideas?

Thanks,

Jez

  • FastDev only pushes the changes to the phone, not a new APK.
    Clean the project and force a full rebuild to update the APK.

  • I think this is a real bug.
    Here are some steps.

    1) App is fully functional in emulator and on device
    2) Perform clean from Studio
    3) Check that build folder is empty of all APK files (well its empty of everything after a clean).
    4) Rebuild
    5) NO APK FILE!!!!!!!

    There is definitely some confusion between Studio's workspace (which was defined as this folder) and the real world.

    Does anyone else experience this? It is quite perplexing when there is clearly an app running but apparently no binary that it is running from.
