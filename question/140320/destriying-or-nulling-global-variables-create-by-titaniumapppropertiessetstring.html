Destriying or nulling global variables create by Titanium.App.Properties.setString

How can one either destroy, null out or overwrite a global variable which was created using Titanium.App.Properties.setString?

I created an array:
var x = [];

then I populate it via a for loop with data.
Next, this data was read via Titanium.App.Properties,getString then displayed in a new window of my iPhone app. All the array items were there as it was expected..
I went back to the first window and I attempted to over-write the x array with new data. It seam to work (no errors reported), however when I looked at the data in the new page, something unexpected happened: the old date showed up, not the new. Repeating this the second and the third time the new data shows up, but why not the fist time? I tried to null out the array before re-populating it, but it did not work either.

Can anyone tell me what I am doing wrong?

1) Application Type: Mobil
2) Titanium SDK version: 2.0.1 (04/12/12 16:33 999c68a)
3) IOS 5.0, Xcode 4.3
4) IOS Simulator
5) Host Operating System: Mac OS X version 10.7
6) Ti Studio build: 2.1.0.201206251749, Date: 25/6/2012

    If you want to destroy properties created by Titanium.App.Properties,then you can use
    Titanium.App.Properties.removeProperty( String property );
    Removes a property if it exists, or does nothing otherwise.
    Parameters
    property : String
    Name of property.
    Returns
    void

    for more details you can refer this link

    http://docs.appcelerator.com/titanium/2.0/#!/api/Titanium.App.Properties

    I hope this will help you.
