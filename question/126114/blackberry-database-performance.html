Blackberry Database Performance

Hello, let me start by saying I realize the blackberry - titanium sdk is in a 'preview' state and all that.

That said, I am having terrible performance with the database on the blackberry. Not on the simulators but on the device itself. I have a Curve 8530, and after doing a number of different variations and tests, I'm averaging about 2700ms per query on device, which when you need to update or insert a couple hundred rows is a real problem. The device is clearly slower than the simulator with everything, but not by a factor of 100 like with the database calls.

Has anyone else had that experience? I've written database apps for iOS and android using Titanium, so I'm confident I'm not doing anything bizarre with the code. It may be just that Titanium's database functions for BB are slow at this point, but I wanted to check with everyone else.

Thanks

    • Thanks for responding. Nice to know it's not just my problem. For the record, I was able to do my on device debugging (benchmarking) via the Blackberry JDE debugger.

      Just connect the device via USB, make sure the settings on device allow everything that needs to be allowed, open the JDE, go to debug -> attach to device. It can be touchy, sometimes you need to unplug and replug the device, and its looking for a bunch of files it can't find (you can show it where the titanium files you want are though), but it does work.
