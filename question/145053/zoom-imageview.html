  • Hi Djamel

    This code will demonstrate how to use the scrollView and Zooming, create a blank project and paste this inside to try it.

    Ti.UI.setBackgroundColor('#000');

var tab1, tab2, tab3;
(function () {
    var win = Ti.UI.createWindow({  
        title:'Tab 1',
        backgroundColor:'#fff'
    });
    tab1 = Ti.UI.createTab({  
        icon: 'KS_nav_views.png',
        title: 'Tab 1',
        window: win
    });
    // main section start
    var scrollView = Titanium.UI.createScrollView({
        contentHeight: Ti.UI.SIZE,
        contentWidth: Ti.UI.SIZE,
        showHorizontalScrollIndicator: true,
        showVerticalScrollIndicator: true,
        maxZoomScale: 10,
        minZoomScale: 0.25,
        top: 0,
        zoomScale: 0.6
    });
    win.add(scrollView);

    var img = Ti.UI.createImageView({
        height: Ti.UI.SIZE,
        image: 'http://placehold.it/2500x2000',
        width: Ti.UI.SIZE
    });
    scrollView.add(img);
    // main section end
})();

var tabGroup = Ti.UI.createTabGroup();
tabGroup.addTab(tab1);
tabGroup.open();

    Or as you know what you are doing you can just add the main section commented above.

    This takes advantage of the iOS properties minZoomScale, maxZoomScale & zoomScale (initial zoom). These are NOT available on Android.

    The part that is not obvious to start with is the zoom is based on the ratio of the content within the scrollView and the size of the scrollView itself. So to make this work at its optimum you will want to calculate the zoom ratios scale figures dependant on actual image dimensions themselves and the allocated space for the scrollView.

    You can of course then put this into a scrollableView to show several 'pages' - as in the image gallery views.

    • Hi

      It is because you are assuming the current zoom will equal exactly 1, which is unlikely.

      This method will determine if the current zoom is more min or more max and then swap to the max or min as needed.

      I have also tweaked the example to better illustrate the difference.

          var scrollView = Titanium.UI.createScrollView({
        contentHeight: Ti.UI.SIZE,
        contentWidth: Ti.UI.SIZE,
        showHorizontalScrollIndicator: true,
        showVerticalScrollIndicator: true,
        maxZoomScale: 2,
        minZoomScale: 0.5,
        top: 0,
        zoomScale: 1
    });

    scrollView.addEventListener('doubletap', function (e) {
        var dblCurrentZoom = e.source.getZoomScale();
        var dblMidWay = ((e.source.getMaxZoomScale() - e.source.getMinZoomScale()) / 2) + e.source.getMinZoomScale();
        if (dblCurrentZoom > dblMidWay) {
            scrollView.setZoomScale(e.source.getMinZoomScale());
        } else {
            scrollView.setZoomScale(e.source.getMaxZoomScale());
        }
    });
    win.add(scrollView);

    var img = Ti.UI.createImageView({
        height: Ti.UI.SIZE,
        image: 'http://placehold.it/768x1024',
        touchEnabled: false, // added this to allow the events to be ignored **
        width: Ti.UI.SIZE
    });
    scrollView.add(img);

      ** Not likely to cause problems but unless you need a specific event on this object I would always turn off the touch event handling.

      BTW

      Your if statement coding style is likely to cause you debugging problems in the future.

      if(scrollView.zoomScale == 1)
    scrollView.zoomScale = 10;
else
    scrollView.zoomScale = 1;

      Consider changing style to

      if (scrollView.zoomScale === 1) {
    scrollView.zoomScale = 10;
} else {
    scrollView.zoomScale = 1;
}

      This way when you come back to edit this block later you will not accidentally end up with code that only runs if your if is false. You will save yourself a lot of future pain.
