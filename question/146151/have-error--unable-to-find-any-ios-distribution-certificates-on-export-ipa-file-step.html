  • Wittawoot gave me a strong clue about updating (thank you). You can read my comments, I initially did have a certificate problem. Here are some details for how I got my iOS dev and ad-hoc builds working: Downloaded latest Titanium Studio from main appcelerator site. Then on builds.appcelerator.com (which currently resolves to http://builds.appcelerator.com.s3.amazonaws.com/index.html) chose the latest build under branch: master. By choose, I mean right click the OS X link (for me this was http://builds.appcelerator.com.s3.amazonaws.com/mobile/master/mobilesdk-3.1.0.v20130107133402-osx.zip) and copy its location, don't download and unzip anything. Then, in Titanium Studio go "Help" -> "Install Specific Titanium SDK". Then, in the field it offers, paste in the build url you copied from builds.appcelerator.com. Let it install. This will not change the version of Titanium Studio it will add an additional Titanium SDK version. Then, in your TiApp.xml (use GUI interface), change Titanium SDK to the version you downloaded. Mine now reads, "3.1.0.v0130107104203". Doing that made my error, "Unable to find any iOS Distribution Certificates" go away. When doing an ad-hoc build, for "specify location" I specify my desktop folder, it doesn't have to be under your app's build folder, that' folder seems to be used for temp files. Note that for me, using this process, Wittawoot's specific build version didn't work for me. Once I tried the latest version (at present, mobilesdk-3.1.0.v20130107133402-osx.zip), it worked.

    • Seems it was a problem with my cert not being signed with the correct private key. Even though the cert is found in xcode and titanium it was invalid but only xcode showed it as invalid and even at that it was very subtle. If I highlighted my team on the left in the xcode organizer, I could see my dev and distribution certificates. one had a tiny green checkmark, the other didn't. when I rolled over the one that didn't, my distribution one, it said something about not being able to find or not having a private key. So, I went into keychain, selected All Items, did a search in keychain for distribution, deleted both keys and certificates, then revoked the distribution cert in the developer portal. My first attempt at recreating the certificate was to issue a certificate signing request via keychain. This time, as the portal recommended, I did it through xcode which entails simply clicking refresh (you have to have devices->profiles selected in the Organizer AND have deleted the certs from keychain – i also deleted all my profiles that were associated with that cert before I revoked it. By letting XCode do everything I ended up with the little green checkbox on my distribution certificate and now my IPA builds and in fact, it seems to be preferred not to use "build/iphone/build/Release-iphoneos" for the "specify location" field, I just specified my ~/tmp folder. Lastly in testing all this, I used Titanium Studio 2.1.2 rather than 3 (downgraded). I'll check v3 again soon, maybe it'll work now too.
