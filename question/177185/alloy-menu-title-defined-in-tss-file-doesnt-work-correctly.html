Alloy menu title defined in tss file doesn't work correctly

Ti sdk 3.3.0, Android, Windows, Titanium Studio, build: 3.3.0.201407100905
Example:
index.xml

<Alloy>
    <Window>
        <Menu/>
        <TextField id="loginField">
    </Window>
</Alloy>

index.tss

"Menu":{
    title:L('title")    
},
"#loginField": {
    hintText:L('loginField')
}

After the compilation and deployment the actionbar title is "_ALLOYEXPR–L("title")".
But the textfield has a correct localized hint.

Does anybody know wtf?

  • Hi

    You should add a menu item, it is the menu item that you set the title of: Titanium.Android.Menu

    XML
------

<Alloy>
    <!-- Create a heavyweight window to use the Android menu. -->
    <Window id="win" fullscreen="true" onOpen="doOpen">

        <!-- The Menu tag adds the Android menu. -->
        <Menu id="menu" platform="android">

            <!-- Cannot specify node text.  Use attributes only. -->
            <MenuItem id="menuItem" title="Item 1" icon="item1.png" onClick="doClick" 
            showAsAction="Ti.Android.SHOW_AS_ACTION_IF_ROOM" />
        </Menu>
    </Window>
</Alloy>

TSS
------

"#menuitem": {
    title: L('title')
}

    Hope this helps
