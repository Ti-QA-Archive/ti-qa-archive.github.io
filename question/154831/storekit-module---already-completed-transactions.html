Storekit module - already completed transactions

Hi there!

I want my application to check whether the user had already purchased any of the available inApp-Purchases on a different device (with the same appleID), so I created a button that is supposed to check that - Therefore, I am using the method:

"restoreCompletedTransactions", which is a method of the Storekit global object.

The only thing I keep receiving in the event "restoredCompletedTransactions" as a result, is:
[INFO] : {
[INFO] : bubbles = 1;
[INFO] : cancelBubble = 0;
[INFO] : source = "[object ti.storekit]";
[INFO] : type = restoredCompletedTransactions;
[INFO] : }

What am I doing wrong? Where do I get a list of the products the user have already purchased in the past?

The event "transactionState" is not being fired, after I call restoreCompletedTransactions.
So I can't catch anything from there.

I use a Non-Renewing Subscription as a type for our inApp Purchases.
Could that be the reason why it is not working?

/ STATS
Titanium Studio with 3.1.1 SDK.
Most recent version of Titanium Storekit version /
