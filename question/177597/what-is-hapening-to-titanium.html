    • Good new you have it back up and running.

      I would like to point out that ideally you should only use pre-release unstable versions on test machines or dual boot setups. As the names imply "pre-release" - not ready to be released & "unstable" - not stable, these should not be used on production machines ever.

      During testing of; Yosemite, Ti 3.4, xcode 6 and iPhone 6 I have an external HD connected to my dev machine that acts as a dual boot device - this way I simply swap boot drives from preferences (Apple OS X solution) without a second thought - the only cost is a reboot and a cheap external HD.

      Using this method means I get everything I want to test on - without any of the risks. I learnt the hard way - not with Titanium with software long ago, you then work out how not to ever get into that situation again.

      So this question can be marked as answered now I hope.
