Read files from device is slow, alternatives?

I've built an application that gets an XML file using a XHR request. The XML is stored on the devices file system so a request isn't needed everytime the app needs to look at this data.

The problem is, even if the file is just a few kb, it takes about 1 second to load. It's just enough for the user to notice the delay..

Is there any faster way than to store the XML file and read it when needed. Is there some sort of memory or cache that is faster and could be used for this?

Example code for reading:

if(file.exists()){
   var content = file.read();
}

Thank you

    Did you ever find the cause or a solution to this?

    I ran into similar problems a year ago where my app was trying to download large xml content from various web services I had set up. Everything ran smoothly in the emulator but when I deployed the app to my mobile devices (a 2nd gen iphone and Nexus) it took ages for both to process the streamed XML.

    In the end I had no choice but to give up and ended up updating the webservice to return CSV. This made a world of difference performance wise but returning CSV from a WS is frankly a horrid great design.
