  • Accepted Answer

    If you choose GET on the app (httpClient) then you have to use $_GET on the backend, same applies to POST. Both ways are able to Transport data, but POST is better for many other reasons.

    Short Story: use POST on both Client & Server, use a js object as params and put a var_dump($_POST); die(); at the top of your PHP script. Add an onload Event to your httpClients objct and add Ti.API.info(this.responseText);

  • If you do an alert on your params, are your values actually there?

    I would change the code in your js to

    var params = {
"newpassword" : newpassword.value,
"username" : username.value

ValidReq.open("POST","http://localhost/edit.php"); 
ValidReq.send(params);
};

    and in your php file

    extract($_POST);
.
//db code
.
$sql = "UPDATE candidat SET password='$newpassword' where username = '$username'";
$query = mysql_query($sql);

    Using EXTRACT assigns the POST (or GET if you go that route) to variables of the same name, without having to manually set them up.

    Also, you're missing a ";" after your alert.

    And finally, I dont think you can use "localhost" in your JS code, at the least you need to use the IP.
